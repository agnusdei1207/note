# 시스템 모니터링 (System Monitoring)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 모니터링은 **"시스템**의 **상태**를 **수집**, **분석**, **알림**하는 **활동\\\"**으로, **Metrics**(메트릭)**, **Logs**(로그)**, **Traces**(트레이스)**를 **수집**하고 **Visualization**(시각화)**와 **Alerting**(알림)**을 **제공**한다.
> 2. **Prometheus**: **Pull **Based**(풀 **기반)** **메트릭 **수집**으로 **Time **Series **DB**(시계열 **DB)**에 **저장**하고 **PromQL**(Prometheus **Query **Language)**로 **쿼리**하며 **Alertmanager**가 **알림**을 **전송**한다.
> 3. **3 Pillars of Observability**: **Metrics**(숫자), **Logs**(이벤트), **Traces**(요청 **경로)**가 **observability**(관측 **가능성)**의 **3대 **기둥**이고 **Grafana**가 **통합 **대시보드**를 **제공**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
시스템 모니터링은 **"가시성 확보 활동"**이다.

**모니터링 계층**:
| 계층 | 내용 | 예시 |
|------|------|------|
| **Infrastructure** | 서버, 네트워크 | CPU, Memory |
| **Application** | 서비스 성능 | Latency, QPS |
| **Business** | 비즈니스 지표 | Revenue, DAU |

### 💡 비유
모니터링은 ****자동차 **계기판 ****과 같다.
- **속도계**: 처리량
- **연료 게이지**: 리소스
- **경고등**: 알림

### Metric Types

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Prometheus Metric Types                                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    1. Counter (Monotonically increasing):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # Total requests (only increases)                                                      │  │
    │  http_requests_total{method="GET"} 100                                                 │  │
    │  http_requests_total{method="GET"} 101  (+1)                                            │  │
    │  http_requests_total{method="GET"} 105  (+4)                                            │  │
    │                                                                                         │  │
    │  Query: rate(http_requests_total[5m])                                                    │  │
    │  → Requests per second (over last 5 minutes)                                             │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    2. Gauge (Can go up or down):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # Memory usage (fluctuates)                                                             │  │
    │  memory_usage_bytes{host="server1"} 1073741824  (1 GB)                                  │  │
    │  memory_usage_bytes{host="server1"} 2147483648  (2 GB)                                  │  │
    │  memory_usage_bytes{host="server1"} 1073741824  (1 GB)                                   │  │
    │                                                                                         │  │
    │  Query: avg(memory_usage_bytes) by (host)                                                │  │
    │  → Average memory usage per host                                                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    3. Histogram (Count + Sum of values):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # Request latency distribution                                                           │  │
    │  http_request_duration_seconds_bucket{le="0.1"} 1000                                    │  │
    │  http_request_duration_seconds_bucket{le="0.5"} 1500                                    │  │
    │  http_request_duration_seconds_bucket{le="1.0"} 1900                                    │  │
    │  http_request_duration_seconds_bucket{le="+Inf"} 2000                                   │  │
    │  http_request_duration_seconds_sum 250  (total latency)                                  │  │
    │  http_request_duration_seconds_count 2000                                              │  │
    │                                                                                         │  │
    │  Query: histogram_quantile(0.95, http_request_duration_seconds_bucket)                     │  │
    │  → 95th percentile latency (~0.5s)                                                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    4. Summary (Similar to histogram, client-side aggregation):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # RPC latency across services                                                           │  │
    │  rpc_duration_seconds{service="api",quantile="0.5"} 0.05                                 │  │
    │  rpc_duration_seconds{service="api",quantile="0.9"} 0.10                                 │  │
    │  rpc_duration_seconds{service="api",quantile="0.99"} 0.50                                │  │
    │  rpc_duration_seconds_sum{service="api"} 100                                             │  │
    │  rpc_duration_seconds_count{service="api"} 1000                                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

    │  │  )                                                                                    │  │  │

## Ⅴ. 기대효과 및 결론

### 모니터링 기대 효과

| 지표 | 도입 전 | 도입 후 | 개선 |
|------|--------|--------|------|
| **MTTD** | 30분 | 5분 | 83% |
| **MTTR** | 2시간 | 30분 | 75% |
