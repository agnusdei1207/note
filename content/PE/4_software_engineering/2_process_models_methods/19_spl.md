# SPL (Software Product Line)

## 정의

**SPL(Software Product Line)**은 **공통된 특성과 가변적 특성을 관리**하여 **제품군을 체계적으로 개발**하는 소프트웨어 공학 접근법이다. 핵심 자산(Core Asset)을 재사용하여 여러 제품을 효율적으로 생산한다.

│   │   │     핵심 자산 (Core Assets)                     │    │ │

## 핵심 개념

### 1. 공통점과 가변점 분석

```
┌─────────────────────────────────────────────────────────────────┐

### 2. 특징 모델 (Feature Model)

```
┌─────────────────────────────────────────────────────────────────┐
│           특징 모델 (Feature Model)                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   특징 다이어그램 표기법:                                 │ │
│   │                                                         │ │
│   │                      ○── 필수 (Mandatory)               │ │
│   │                     ╱                                   │ │
│   │                ───●                                    │ │
│   │                   ╲                                     │ │
│   │                    ○── 선택 (Optional)                  │ │
│   │                                                         │ │
│   │                      ○    ○                            │ │
│   │                     ╱      ╲                            │ │
│   │                ───●────────●── 선택 중 1 (Alternative)  │ │
│   │                                                         │ │
│   │                      ○    ○    ○                       │ │
│   │                     ╱      ╲    ╱                       │ │
│   │                ───●────────●── 1개 이상 (OR)            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   예시: 모바일 앱 특징 모델:                                     │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │                         모바일 앱                        │ │
│   │                            │                            │ │
│   │            ┌───────────────┼───────────────┐            │ │
│   │            │               │               │            │ │
│   │            ▼               ▼               ▼            │ │
│   │         인증 ◆          기능 ◇          플랫폼 ◇        │ │
│   │            │               │               │            │ │
│   │       ┌────┴────┐    ┌────┴────┐    ┌────┴────┐       │ │
│   │       │         │    │         │    │         │       │ │
│   │      ▼         ▼    ▼         ▼    ▼         ▼       │ │
│   │    로그인   로그아웃  대시보드  리포트   iOS     Android │ │
│   │      ◆        ○      ◆        ○     ◇        ◇       │ │
│   │    (필수)  (선택)  (필수)  (선택)  (OR 중 1개)         │ │
│   │                                                         │ │
│   │   ◆: 필수, ○: 선택, ◇: 복수 선택 가능                   │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   가변점 의존성:                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Requires: A가 선택되면 B도 필수                        │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │   5G ──requires──▶ 5G 안테나                   │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   Excludes: A와 B는 동시 선택 불가                       │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │   소형 배터리 ──excludes──▶ 고성능 CPU         │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. SPL 구현 기법

```
┌─────────────────────────────────────────────────────────────────┐
│           가변성 구현 메커니즘                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   컴파일 타임 가변성:                                     │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   ① 전처리기 (Preprocessor):                    │    │ │
│   │   │      #ifdef FEATURE_A                          │    │ │
│   │   │        // Feature A 코드                        │    │ │
│   │   │      #endif                                    │    │ │
│   │   │                                                 │    │ │
│   │   │   ② 조건부 컴파일:                               │    │ │
│   │   │      if (config.hasFeatureA) {                 │    │ │
│   │   │        featureA.run();                         │    │ │
│   │   │      }                                         │    │ │
│   │   │                                                 │    │ │
│   │   │   ③ 템플릿/제네릭:                               │    │ │
│   │   │      template<typename Storage>                │    │ │
│   │   │      class DataManager { ... };                │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   런타임 가변성:                                          │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   ① 상속/오버라이딩:                             │    │ │
│   │   │      abstract class Feature {                  │    │ │
│   │   │        abstract void execute();                │    │ │
│   │   │      }                                         │    │ │
│   │   │      class FeatureA extends Feature { ... }    │    │ │
│   │   │                                                 │    │ │
│   │   │   ② 전략 패턴 (Strategy Pattern):               │    │ │
│   │   │      interface Algorithm {                     │    │ │
│   │   │        void process();                         │    │ │
│   │   │      }                                         │    │ │
│   │   │      // 런타임에 구현체 교체                     │    │ │
│   │   │                                                 │    │ │
│   │   │   ③ 플러그인 아키텍처:                           │    │ │
│   │   │      // 동적 로딩으로 기능 확장                  │    │ │
│   │   │      PluginManager.load("feature-a.jar");      │    │ │
│   │   │                                                 │    │ │
│   │   │   ④ 설정 파일:                                  │    │ │
│   │   │      // config.xml로 가변점 구성                │    │ │
│   │   │      <feature name="A" enabled="true"/>        │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   비교:                                                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   구분        │ 컴파일 타임     │ 런타임       │    │ │
│   │   │   ────────────┼────────────────┼─────────────  │    │ │
│   │   │   성능        │ 높음           │ 낮음         │    │ │
│   │   │   유연성      │ 낮음           │ 높음         │    │ │
│   │   │   바이너리    │ 제품별 다름    │ 동일         │    │ │
│   │   │   메모리      │ 최적화 가능    │ 오버헤드     │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │

## 기술사 시험 포인트

### 1. 핵심 정의

> **SPL**: 공통점과 가변점 관리를 통한 제품군 개발 방법론
> **Core Asset**: 재사용 가능한 핵심 자산
> **Feature Model**: 제품 특징의 계층적 모델링

### 2. 빈출 키워드

- **Commonality/Variability**: 공통점/가변점
- **Domain Engineering**: 도메인 공학, 핵심 자산 개발
- **Application Engineering**: 응용 공학, 개별 제품 개발
- **Feature Diagram**: 특징 다이어그램, 필수/선택/대안/OR

### 3. 예상 문제

**Q1. SPL의 도메인 공학과 응용 공학의 차이점과 상호 관계를 설명하시오.**

**Q2. 특징 모델(Feature Model)의 구성 요소와 가변점 표기법을 설명하시오.**

**Q3. SPL에서 가변성을 구현하는 방법을 컴파일 타임과 런타임 관점에서 비교하시오.**
