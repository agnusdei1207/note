# PSP/TSP (Personal/Team Software Process)

## 정의

**PSP(Personal Software Process)**는 개인 개발자의 **소프트웨어 개발 프로세스**로, 자신의 작업을 측정하고 개선하는 방법론이다. **TSP(Team Software Process)**는 PSP를 기반으로 **팀 단위 개발 프로세스**를 체계화한 방법론이다. 둘 다 SEI(CMU)의 Watts Humphrey가 개발했다.

```
┌─────────────────────────────────────────────────────────────────┐
│           PSP/TSP 계층 구조                                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │                    CMMI (조직 성숙도)                    │ │
│   │                         │                               │ │
│   │            ┌────────────┴────────────┐                  │ │
│   │            ▼                         ▼                  │ │
│   │   ┌────────────────┐      ┌────────────────────┐        │ │
│   │   │      TSP       │      │   조직 프로세스    │        │ │
│   │   │  (팀 프로세스)  │      │                    │        │ │
│   │   │                │      │                    │        │ │
│   │   │      PSP       │      │                    │        │ │
│   │   │  (개인 프로세스)│      │                    │        │ │
│   │   └────────────────┘      └────────────────────┘        │ │
│   │                                                         │ │
│   │   관계:                                                  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • PSP: 개인의 작업 습관 개선                   │    │ │
│   │   │ • TSP: PSP 기반 팀 협업 체계화                  │    │ │
│   │   │ • CMMI: 조직 수준의 프로세스 성숙도             │    │ │
│   │   │                                                 │    │ │
│   │   │   PSP → TSP → CMMI                             │    │ │
│   │   │   개인 → 팀 → 조직                              │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. PSP 측정 지표

```

│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │

│   │   │   • 페어 시간 (개입 시간, 차단 시간)             │    │ │

│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   크기 측정:                                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   LOC (Lines of Code):                         │    │ │
│   │   │   • 신규 LOC, 수정 LOC, 삭제 LOC               │    │ │
│   │   │   • 재사용 LOC                                 │    │ │
│   │   │                                                 │    │ │
│   │   │   프록시 기반 추정:                             │    │ │
│   │   │   • 기능 포인트                                │    │ │

│   │   │   • 화면/보고서 수                              │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   결함 측정:                                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   결함 로그:                                    │    │ │
│   │   │   ┌─────────────────────────────────────┐     │    │ │
│   │   │   │ 번호 │ 타입 │ 삽입 │ 제거 │ 수정시간│     │    │ │
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **PSP**: 개인 소프트웨어 프로세스 - 자신의 작업 측정/개선
> **TSP**: 팀 소프트웨어 프로세스 - PSP 기반 팀 협업 체계
> **측정 기반 개선**: 데이터 수집 → 분석 → 프로세스 개선

### 2. 빈출 키워드

- **PSP Level**: PSP0 → PSP3 진화 과정
- **Time/Size/Defect Logging**: 시간/크기/결함 기록
- **TSP Launch**: 9단계 런칭 프로세스
- **LOC/Hour, DRE**: 생산성 및 품질 지표

### 3. 예상 문제

**Q1. PSP의 단계별 진화 과정과 각 단계의 특징을 설명하시오.**
