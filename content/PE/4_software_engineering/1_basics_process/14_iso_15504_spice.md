# ISO/IEC 15504 (SPICE)

## 정의

**ISO/IEC 15504(SPICE)**는 소프트웨어 **프로세스 평가를 위한 국제 표준**이다. 조직의 소프트웨어 개발 프로세스 역량을 체계적으로 평가하고 개선하기 위한 프레임워크를 제공한다. **SPICE**(Software Process Improvement and Capability dEtermination)라는 별칭으로도 불린다.

```
┌─────────────────────────────────────────────────────────────────┐
│           SPICE 개요                                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   목적:                                                  │ │
│   │   ① 프로세스 개선 (Process Improvement)                  │ │
│   │      - 조직의 프로세스 역량 향상                         │ │
│   │   ② 프로세스 역량 결정 (Capability Determination)        │ │
│   │      - 공급자 선정 시 역량 평가                          │ │
│   │                                                         │ │
│   │   특징:                                                  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • ISO/IEC 12207 기반                           │    │ │
│   │   │ • 프로세스 중심 평가                           │    │ │
│   │   │ • 6단계 역량 수준 (Capability Level)           │    │ │
│   │   │ • 객관적이고 측정 가능한 평가                   │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 역량 수준 (Capability Levels)

```
┌─────────────────────────────────────────────────────────────────┐
│           SPICE 6단계 역량 수준                                   │

│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │

│   │   └───────────────────────────────────────────────┘    │ │

│   │   Level 4: 예측 가능 (Predictably)                      │ │

│   │   │ • 정량적 프로세스 관리                         │    │ │
│   │   │ • 성능 예측                                   │    │ │

│   │   └───────────────────────────────────────────────┘    │ │
│   │                      ▲                                  │ │
│   │   Level 2: 관리 (Managed)                               │ │

│   │   └───────────────────────────────────────────────┘    │ │

│   │   Level 1: 수행 (Performed)                             │ │

│   │   │ • 목표 달성 실패                               │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
```

### 3. 평가 등급 (Rating Scale)

```
┌─────────────────────────────────────────────────────────────────┐
│           속성 달성 등급                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   N (Not Achieved):    0% ~ 15% 달성                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 속성이 거의 달성되지 않음                    │    │ │
│   │   │ • 체계적 접근 없음                             │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   P (Partially Achieved): 15% ~ 50% 달성                │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 속성이 부분적으로 달성됨                     │    │ │
│   │   │ • 일부 체계적 접근 존재                        │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   L (Largely Achieved): 50% ~ 85% 달성                  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 속성이 상당히 달성됨                         │    │ │
│   │   │ • 대부분 체계적 접근 존재                      │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   F (Fully Achieved): 85% ~ 100% 달성                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 속성이 완전히 달성됨                         │    │ │
│   │   │ • 완전한 체계적 접근 존재                      │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   수준 달성 조건:                                        │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 해당 수준의 모든 속성이 L 또는 F 등급         │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **SPICE (ISO/IEC 15504)**: 소프트웨어 프로세스 평가 국제 표준
> **Capability Level**: 6단계 (0~5) 역량 수준
> **Process Attribute**: 각 수준별 평가 속성

### 2. 빈출 키워드

- **6 Capability Levels**: 불완전~최적화
- **Process Attributes (PA)**: 각 수준의 달성 기준
- **N/P/L/F Rating**: 속성 달성 등급
- **Process Categories**: CUS, ENG, SUP, MAN, ORG

### 3. 예상 문제

**Q1. SPICE(ISO/IEC 15504)의 6단계 역량 수준을 설명하고 각 수준의 특징을 기술하시오.**

**Q2. SPICE 평가에서 속성 달성 등급(N/P/L/F)의 의미를 설명하시오.**

**Q3. SPICE와 CMMI의 차이점을 비교 설명하시오.**
