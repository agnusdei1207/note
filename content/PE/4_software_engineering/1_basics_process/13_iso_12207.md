# ISO/IEC 12207 (소프트웨어 생명주기 공정 표준)

## 정의

**ISO/IEC 12207**는 소프트웨어의 **획득, 공급, 개발, 운영, 유지보수**를 포함한 전 생명주기 프로세스를 체계적으로 정의한 국제 표준이다. 1995년 처음 제정되었으며 2008년 개정되어 시스템 수명주기 프로세스(ISO/IEC 15288)와 통합되었다.

│   │   │            기본 프로세스 (Primary)               │  │ │
│   │   │  ┌─────────┐ ┌─────────┐ ┌─────────┐            │  │ │

│   │                                                         │ │
│   │   ④ 검증 (Verification):                                │ │
│   │   • "제품을 올바르게 만들고 있는가?"                    │ │

│   │   "Did we build it right?"                              │ │

## 기술사 시험 포인트

### 1. 핵심 정의

> **ISO/IEC 12207**: 소프트웨어 생명주기 프로세스 국제 표준
> **3대 프로세스 그룹**: 기본, 지원, 조직 프로세스
> **V&V**: 검증(Verification)과 확인(Validation)

### 2. 빈출 키워드

- **Primary Processes**: 획득, 공급, 개발, 운영, 유지보수
- **Support Processes**: 문서화, 형상관리, 품질보증, 검증, 확인
- **Organizational Processes**: 관리, 인프라, 개선, 인력훈련

### 3. 예상 문제

**Q1. ISO/IEC 12207의 3대 프로세스 그룹과 각각의 구성 프로세스를 설명하시오.**

**Q2. 검증(Verification)과 확인(Validation)의 차이점을 기술하시오.**

**Q3. 소프트웨어 개발 프로세스의 주요 활동을 ISO/IEC 12207에 따라 설명하시오.**
