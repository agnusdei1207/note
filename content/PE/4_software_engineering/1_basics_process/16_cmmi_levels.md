# CMMI 5단계 (CMMI Maturity Levels)

## 정의

**CMMI 5단계**는 조직의 프로세스 성숙도를 **초기, 관리, 정의, 정량적 관리, 최적화**의 5단계로 분류한 모델이다. 각 단계는 특정 Process Area를 달성해야 하며, 하위 단계를 충족해야 상위 단계로 진행할 수 있다.

```
┌─────────────────────────────────────────────────────────────────┐
│           CMMI 5단계 성숙도 모델                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Level 5: 최적화 (Optimizing) ─────────────────── 5    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 지속적 프로세스 개선                         │    │ │
│   │   │ • 프로세스 혁신 및 배포                        │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                      ▲                                  │ │
│   │   Level 4: 정량적 관리 (Quantitatively Managed) ──── 4  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 정량적 목표 설정 및 관리                      │    │ │
│   │   │ • 통계적 프로세스 관리                         │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                      ▲                                  │ │
│   │   Level 3: 정의 (Defined) ────────────────────────── 3  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 조직 표준 프로세스                           │    │ │
│   │   │ • 엔지니어링 프로세스                          │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                      ▲                                  │ │
│   │   Level 2: 관리 (Managed) ────────────────────────── 2  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 프로젝트 관리                                │    │ │
│   │   │ • 기본 지원 프로세스                           │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                      ▲                                  │ │
│   │   Level 1: 초기 (Initial) ────────────────────────── 1  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 프로세스 정의 없음                           │    │ │
│   │   │ • 성공이 개인에 의존                           │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **5단계**: 초기 → 관리 → 정의 → 정량적관리 → 최적화
> **Level 2**: 기본 프로젝트 관리 (REQM, PP, PMC, CM 등)
> **Level 5**: 지속적 개선 (OID, CAR)

### 2. 빈출 키워드

- **Process Area per Level**: L2(7개), L3(11개), L4(2개), L5(2개)
- **Statistical Process Control**: 정량적 관리 단계의 통계적 관리
- **CAR (Causal Analysis and Resolution)**: 원인 분석 및 해결
- **OID (Organizational Innovation and Deployment)**: 조직 혁신

### 3. 예상 문제

**Q1. CMMI 5단계 성숙도 모델의 각 단계별 특징과 주요 Process Area를 설명하시오.**

**Q2. Level 3(정의)와 Level 4(정량적 관리)의 핵심 차이점을 기술하시오.**

**Q3. CMMI Level 5(최적화) 단계의 지속적 개선 활동을 설명하시오.**
