# CMMI (Capability Maturity Model Integration)

## 정의

**CMMI(Capability Maturity Model Integration)**는 조직의 **프로세스 성숙도와 역량**을 평가하고 개선하기 위한 프레임워크다. 카네기멜론대학교 SEI(Software Engineering Institute)가 개발했으며, 단계형(Staged)과 연속형(Continuous) 두 가지 표현 방식을 제공한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           CMMI 개요                                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   CMMI의 통합:                                           │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • CMM (Software CMM)                          │    │ │
│   │   │ • SECM (Systems Engineering CMM)              │    │ │
│   │   │ • IPD-CMM (Integrated Product Development)    │    │ │
│   │   │ • Supplier CMM                                │    │ │
│   │   │                   ↓                            │    │ │
│   │   │              CMMI 통합                         │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   두 가지 표현 방식:                                     │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ ① 단계형 (Staged Representation)              │    │ │
│   │   │    - 5단계 성숙도 수준                         │    │ │
│   │   │    - 조직 전체의 성숙도 평가                   │    │ │
│   │   │                                                 │    │ │
│   │   │ ② 연속형 (Continuous Representation)          │    │ │
│   │   │    - 6단계 역량 수준 (0~5)                     │    │ │
│   │   │    - 개별 프로세스별 역량 평가                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 단계형 표현 - 5단계 성숙도

```
┌─────────────────────────────────────────────────────────────────┐
│           CMMI 5단계 성숙도 (Maturity Levels)                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Level 5: 최적화 (Optimizing)                          │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 지속적 프로세스 개선                         │    │ │
│   │   │ • 프로세스 혁신 및 최적화                       │    │ │
│   │   │ Process Area: OID, CAR                         │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                      ▲                                  │ │
│   │   Level 4: 정량적 관리 (Quantitatively Managed)         │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 정량적 목표 설정                             │    │ │
│   │   │ • 통계적 프로세스 관리                         │    │ │
│   │   │ Process Area: OPP, QPM                         │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                      ▲                                  │ │
│   │   Level 3: 정의 (Defined)                                │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 조직 표준 프로세스 사용                      │    │ │
│   │   │ • 프로세스 테일러링                            │    │ │
│   │   │ Process Area: RD, TS, PI, VER, VAL, OPF, OPD  │    │ │
│   │   │                 OT, DAR, RSKM                  │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                      ▲                                  │ │
│   │   Level 2: 관리 (Managed)                                │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 프로젝트 계획 및 관리                         │    │ │
│   │   │ • 요구사항 관리                                │    │ │
│   │   │ Process Area: REQM, PP, PMC, SAM, MA, PPQA    │    │ │
│   │   │                 CM                             │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                      ▲                                  │ │
│   │   Level 1: 초기 (Initial)                                │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 프로세스가 정의되지 않음                      │    │ │
│   │   │ • 개인 의존적                                  │    │ │
│   │   │ • 예측 불가능                                  │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   💡 각 수준은 하위 수준의 Process Area 달성이 전제              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. Process Area (프로세스 영역)

```
┌─────────────────────────────────────────────────────────────────┐
│           주요 Process Area                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Level 2 (관리):                                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   REQM - 요구사항 관리 (Requirements Management)        │ │
│   │   PP   - 프로젝트 계획 (Project Planning)               │ │
│   │   PMC  - 프로젝트 모니터링 및 통제                       │ │
│   │   SAM  - 공급자 협약 관리 (Supplier Agreement Mgmt)     │ │
│   │   MA   - 측정 및 분석 (Measurement and Analysis)        │ │
│   │   PPQA - 프로세스 및 제품 품질 보증                     │ │
│   │   CM   - 형상 관리 (Configuration Management)           │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   Level 3 (정의):                                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   RD   - 요구사항 개발 (Requirements Development)       │ │
│   │   TS   - 기술 솔루션 (Technical Solution)               │ │
│   │   PI   - 제품 통합 (Product Integration)                │ │
│   │   VER  - 검증 (Verification)                            │ │
│   │   VAL  - 확인 (Validation)                              │ │
│   │   OPF  - 조직 프로세스 초점 (Org Process Focus)         │ │
│   │   OPD  - 조직 프로세스 정의 (Org Process Definition)    │ │
│   │   OT   - 조직 훈련 (Organizational Training)            │ │
│   │   DAR  - 의사결정 분석 (Decision Analysis & Resolution) │ │
│   │   RSKM - 리스크 관리 (Risk Management)                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   Level 4 (정량적 관리):                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   OPP  - 조직 프로세스 성능 (Org Process Performance)   │ │
│   │   QPM  - 정량적 프로젝트 관리                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   Level 5 (최적화):                                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   OID  - 조직 혁신 및 전개 (Org Innovation & Deployment)│ │
│   │   CAR  - 원인 분석 및 해결 (Causal Analysis & Resolution)│ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 연속형 표현 - 6단계 역량 수준

```
┌─────────────────────────────────────────────────────────────────┐
│           연속형 표현의 역량 수준                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   CL 0: 불완전 (Incomplete)                             │ │
│   │   • 프로세스가 수행되지 않거나 목표 미달성              │ │
│   │                                                         │ │
│   │   CL 1: 수행 (Performed)                                │ │
│   │   • 프로세스가 수행됨                                    │ │
│   │   • 특정 목표 달성                                       │ │
│   │                                                         │ │
│   │   CL 2: 관리 (Managed)                                  │ │
│   │   • 계획, 수행, 모니터링, 검토                          │ │
│   │   • 작업 산출물 관리                                     │ │
│   │                                                         │ │
│   │   CL 3: 정의 (Defined)                                  │ │
│   │   • 조직 표준 프로세스 사용                             │ │
│   │   • 테일러링 적용                                        │ │
│   │                                                         │ │
│   │   CL 4: 정량적 관리 (Quantitatively Managed)             │ │
│   │   • 정량적 목표 사용                                     │ │
│   │   • 통계적 관리                                          │ │
│   │                                                         │ │
│   │   CL 5: 최적화 (Optimizing)                             │ │
│   │   • 지속적 개선                                          │ │
│   │   • 성능 최적화                                          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   단계형 vs 연속형 선택 기준:                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   단계형:                                                │ │
│   │   • 조직 전체 성숙도 평가                                │ │
│   │   • 인증/계약 요구                                       │ │
│   │   • 명확한 로드맵                                        │ │
│   │                                                         │ │
│   │   연속형:                                                │ │
│   │   • 특정 프로세스 집중 개선                              │ │
│   │   • 유연한 적용                                          │ │
│   │   • 비즈니스 목표 중심                                   │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```
│   │   │ 3. 데이터 수집 (인터뷰, 문서 검토)             │    │ │
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **CMMI**: 프로세스 성숙도/역량 평가 및 개선 프레임워크
> **5단계 성숙도**: 초기 → 관리 → 정의 → 정량적관리 → 최적화
> **Process Area**: 각 수준에서 달성해야 할 프로세스 영역

### 2. 빈출 키워드

- **Staged vs Continuous**: 단계형(5단계) vs 연속형(0~5)
- **Process Area**: REQM, PP, CM, OPD, OID 등
- **SCAMPI**: CMMI 공식 평가 방법
- **Goal/Practice**: 목표(Goal)와 실천(Practice)

### 3. 예상 문제

**Q1. CMMI의 5단계 성숙도 모델을 설명하고 각 단계의 특징을 기술하시오.**

**Q2. CMMI의 단계형 표현과 연속형 표현의 차이점을 비교하고 선택 기준을 설명하시오.**

**Q3. CMMI Level 2와 Level 3의 주요 Process Area를 설명하시오.**
