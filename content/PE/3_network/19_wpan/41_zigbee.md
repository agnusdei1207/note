# 지그비 (Zigbee)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 지그비는 **"저전력**, **저속**의 **무선 **개인 **통신망**(WPAN)**을 **제공**하는 **기술\\\"**으로, **IEEE 802.15.4** **PHY/MAC**를 **기반**으로 **하고 **Zigbee **Alliance**가 **네트워크 **계층**(Network **Layer)**과 **응용 **계층**(Application **Layer)**를 **정의**한다.
> 2. **네트워크**: **Coordinator**(코디네이터)**, **Router**(라우터)**, **End **Device**(단말 **장치)**로 **구성**되고 **Mesh **Networking**(메시 **네트워킹)**로 **다중 **홉**(Multi-**Hop)** **통신**을 **지원**하며 **자가 **복구**(Self-**Healing)**가 **가능**하다.
> 3. **보안**: **AES-128** **암호화**와 **Network **Key**(네트워크 **키)**, **Trust **Center**(신뢰 **센터)**로 **키 **분배**를 **관리**하며 **Joining**(네트워크 **가입)** 시 **인증**을 **수행**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
지그비는 **"IoT용 저전력 무선 표준"**이다.

**지그비 버전**:
| 버전 | 주요 특징 | 주파수 | 데이터 전송률 |
|------|-----------|--------|---------------|
| **Zigbee 3.0** | 통합 표준 | 2.4 GHz | 250 Kbps |
| **Zigbee Pro** | 대규모 | 868/915 MHz | 20/40/100 Kbps |

### 💡 비유
지그비는 ****소방관 **알림 ****과 같다.
- **감시자**: Coordinator
- **연락**: Router
- **가정**: End Device

```
└─────────────────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Zigbee Coordinator (ZC):                                                             │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • Form network (starts PAN)                                                          │  │  │

    │  │  • Stores network keys                                                                 │  │  │
    │  │  • Manages Trust Center                                                               │  │  │
    │  │  • Typically: Mains-powered (gateway, hub)                                             │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  2. Zigbee Router (ZR):                                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • Routes messages                                                                    │  │  │
    │  │  • Allows devices to join                                                             │  │  │
    │  │  • Battery or mains-powered                                                           │  │  │
    │  │  → Extends network range                                                              │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  3. Zigbee End Device (ZED):                                                             │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • Communicates with parent only (coordinator or router)                              │  │  │
    │  │  • Sleep mode (battery-powered)                                                       │  │  │
    │  │  • Cannot route messages                                                             │  │  │
    │  │  → Low power consumption                                                              │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Mesh Networking

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Zigbee Mesh Networking                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Mesh Example:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Coordinator                                      Router 3                             │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  ┌────────────┐                  Router 1          Router 2         ┌────────────┐   │  │  │
    │  │  │  ZC       │◄───────────────────────┼───────────────────┼──────────────►│  ZR 3     │   │  │
    │  │  │(Gateway)  │                      │                   │               │(Lamp)    │   │  │
    │  │  └────────────┘                      ▼                   ▼               └────────────┘   │  │  │
    │  │                                    │                   │                                 │  │  │
    │  │                         ┌────────────┐      ┌────────────┐                            │  │  │
    │  │                         │  ZED 1     │      │  ZED 2     │                            │  │  │
    │  │                         │(Sensor)    │      │(Switch)    │                            │  │  │
    │  │                         └────────────┘      └────────────┘                            │  │  │
    │  │                                                                                       │  │  │
    │  │  Path: ZED 1 → ZR 1 → ZR 3 → ZC (multi-hop)                                         │  │  │
    │  │  → If ZR 1 fails: ZED 1 → ZR 2 → ZR 3 → ZC (alternate route)                        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Routing:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Route Discovery (AODV - Ad hoc On-Demand Distance Vector)                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Source (ZED 1) wants to send to Destination (ZC)                                    │  │  │
    │  │  │                                                                                   │  │  │
    │  │  1. Broadcast Route Request (RREQ)                                                    │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  ZED 1 → ZR 1: "Find route to ZC"                                                   │  │  │  │  │
    │  │  │  ZR 1 → ZR 2, ZR 3: "Find route to ZC" (rebroadcast)                              │  │  │  │  │
    │  │  │  ZR 3 → ZC: "Find route to ZC" (rebroadcast)                                      │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  2. Unicast Route Reply (RREP)                                                         │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  ZC → ZR 3: "Route found: ZC ← ZR 3" (path cost: 1)                               │  │  │  │  │
    │  │  │  ZR 3 → ZR 1: "Route found: ZC ← ZR 3 ← ZR 1" (path cost: 2)                      │  │  │  │  │
    │  │  │  ZR 1 → ZED 1: "Route found: ZC ← ZR 3 ← ZR 1 ← ZED 1" (path cost: 3)            │  │  │  │  │
    │  │  │  → Each router updates routing table                                              │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  3. Data flows along discovered route                                                   │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### ZCL (Zigbee Cluster Library)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Zigbee Cluster Library (ZCL)                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Cluster (Functional Group):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Endpoint: Logical interface (device can have multiple endpoints)                        │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Endpoint 0: ZDO (Zigbee Device Object) - management                                 │  │  │
    │  │  Endpoint 1: On/Off Light (clusters: 0x0006 On/Off, 0x0008 Level Control)            │  │  │
    │  │  Endpoint 2: Temperature Sensor (cluster: 0x0402 Temperature Measurement)              │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Common Clusters:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  │  Cluster ID  │  Name                    │  Description                                 │  │
    │  │  ─────────────────────────────────────────────────────────────────────────────────────│  │
    │  │  0x0000      │  Basic                   │  Device information, attributes             │  │
    │  │  0x0003      │  Identify                │  Locate device (blink, beep)               │  │
    │  │  0x0006      │  On/Off                  │  On/off control                             │  │
    │  │  0x0008      │  Level Control           │  Brightness (0-255)                        │  │
    │  │  0x0300      │  Color Control           │  RGB color                                 │  │
    │  │  0x0402      │  Temperature Measurement │  Temperature value (°C/°F)                 │  │
    │  │  0x0405      │  Humidity Measurement    │  Humidity value (%)                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    ZCL Frame:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Frame Control (1 byte) │  Sequence Number (1 byte) │  Command (1 byte) │  Data (...)      │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  │  Frame Type  │  Direction  │  Disable Default Response  │                          │  │  │
    │  │  │  (Profile/     │  Client→    │  (General: 0, Cluster: 1)  │                          │  │  │
    │  │  │   Cluster)     │  Server     │                                                           │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 스마트 홈 조명 제어
**상황**: 50개 전구, 10개 스위치, 1개 게이트웨이
**판단**: Zigbee 3.0 + Coordinator

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Smart Home Lighting Zigbee Network                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Network Design:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Devices:                                                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • 1 Coordinator (Smart Home Hub, mains-powered)                                     │  │  │
    │  │  • 10 Router Lights (mains-powered, extend network)                                  │  │  │
    │  │  • 40 End Device Lights (battery-powered switches, sensors)                          │  │  │
    │  │  • 10 Switches (mains-powered, control lights)                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Binding (Direct Device-to-Device):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Switch 1 controls Light 5 (without coordinator intervention)                            │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Binding Process:                                                                    │  │  │
    │  │  1. Switch 1 (Endpoint 1, Cluster 0x0006) binds to Light 5 (Endpoint 1)               │  │  │
    │  │  2. Switch stores binding table: {target: Light 5, cluster: 0x0006}                   │  │  │
    │  │  3. User presses Switch 1:                                                           │  │  │
    │  │     └─> Switch sends On command directly to Light 5                                 │  │  │
    │  │     └─> Light turns on (response: Success)                                           │  │  │
    │  │  → Fast response (no coordinator hop)                                                 │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Groups (Multicast):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  "Scene" control: All lights in group 0x5001 turn on warm white                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. Lights 1-10 join group 0x5001 (add group to endpoint)                           │  │  │
    │  │  2. User activates "Movie" scene:                                                      │  │  │
    │  │     └─> Coordinator sends multicast: Group 0x5001, Command: On, Level: 50             │  │  │
    │  │     └─> All lights in group respond (no collision, sequential ACK)                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 지그비 기대 효과

| 지표 | Zigbee | BLE Mesh | WiFi |
|------|---------|----------|------|
| **배터리 수명** | 2-5년 | 1-2년 | 주간 |
| **커버리지** | 100m+ | 50m | 50m |
| **장치 수** | 65K+ | 제한 | 50 |
| **지연시간** | 30-100ms | 10-50ms | 10ms |

### 모범 사례

1. **Router**: Mains-powered
2. **End Device**: Battery
3. **채널**: 15, 20, 25
4. **PAN ID**: 고유한 값

### 미래 전망

1. **Matter**: 통합 표준
2. **Zigbee 3.0**: 단일 표준
3. **Green Power**: 에너지 하베스팅
4. **Sub-GHz**: 더 긴 범위

- **Zigbee Alliance**: Specification
- **IEEE**: 802.15.4
- **Matter**: CSA Standard
