# 비동기식 전송 - 시작/정지 비트 (Start/Stop Bit)

## 정의

**비동기식 전송(Asynchronous Transmission)**은 송수신자 간 **공통 클럭 없이** 데이터를 전송하는 방식이다. 각 문자(바이트) 앞뒤에 **시작 비트(Start Bit)**와 **정지 비트(Stop Bit)**를 추가하여 동기를 맞춘다. UART, 직렬 포트 등에서 사용된다.

```
┌─────────────────────────────────────────────────────────────────┐
│           비동기식 전송 프레임 구조                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Idle │Start│  D0 │  D1 │  D2 │  D3 │  D4 │  D5 │  D6 │  D7 │Parity│Stop│ Idle │
│   │   (1)  │ (0) │     │     │     │     │     │     │     │     │ (P) │(1) │  (1) │
│   │        │     │     │     │     │     │     │     │     │     │     │    │      │
│   │   ─────┐     ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌────────  │
│   │        │     │   │ │   │ │   │ │   │ │   │ │   │ │   │ │   │ │   │ │          │
│   │        └─────┘   └─┘   └─┘   └─┘   └─┘   └─┘   └─┘   └─┘   └─┘   └─┘          │
│   │         ↑                                                           ↑          │
│   │      시작 비트                                                   정지 비트    │
│   │     (Start Bit)                                               (Stop Bit)    │
│   │                                                         │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   전송 순서: LSB first (D0부터 전송)                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 시작 비트와 정지 비트의 역할

```
┌─────────────────────────────────────────────────────────────────┐
│           시작/정지 비트 동작 원리                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   시작 비트 (Start Bit):                                        │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 항상 0 (Low)                                        │ │
│   │   • Idle 상태(1)에서 0으로 떨어지는 시점 감지           │ │
│   │   • 수신기의 타이머 시작                                │ │
│   │   • 수신기가 비트 타이밍에 맞춰 샘플링 시작             │ │
│   │                                                         │ │
│   │   Idle ────┐                                            │ │
│   │    (1)     │                                            │ │
│   │            │← 시작 비트 감지 시점                       │ │
│   │   Start  ──┘                                            │ │
│   │    (0)                                                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   정지 비트 (Stop Bit):                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 항상 1 (High)                                       │ │
│   │   • 문자 전송 종료 표시                                 │ │
│   │   • 수신기가 다음 시작 비트 대기 상태로 전환            │ │
│   │   • 1비트, 1.5비트, 2비트 가능                          │ │
│   │                                                         │ │
│   │   D7 ────┐    ┌──────────── Idle                        │ │
│   │          │    │          (1)                            │ │
│   │   Stop ──┘    │                                         │ │
│   │    (1)        │← 정지 비트 구간                          │ │
│   │               │                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   동기화 메커니즘:                                               │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   1. 수신기는 Idle 상태에서 High(1) 대기                │ │
│   │   2. High → Low 전이 감지 시 시작 비트로 인식           │ │
│   │   3. 1.5 비트 시간 후 첫 데이터 비트 샘플링             │ │
│   │   4. 1 비트 간격으로 나머지 비트 샘플링                 │ │
│   │   5. 정지 비트 확인 후 프레임 종료                      │ │
│   │                                                         │ │
│   │   💡 짧은 프레임 내에서는 클럭 오차 허용 가능           │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 비동기 전송 파라미터

```
┌─────────────────────────────────────────────────────────────────┐
│           비동기 전송 설정 파라미터                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   일반적 UART 설정:                                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 전송 속도 (Baud Rate): 9600, 19200, 38400, 115200   │ │
│   │   • 데이터 비트: 7비트, 8비트                            │ │
│   │   • 패리티: None, Even, Odd, Mark, Space                │ │
│   │   • 정지 비트: 1, 1.5, 2                                 │ │
│   │                                                         │ │
│   │   표기 예: "9600-8-N-1"                                 │ │
│   │   = 9600 bps, 8데이터 비트, 패리티 없음, 1 정지 비트    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   프레임 크기 계산 (8-N-1 설정):                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   시작(1) + 데이터(8) + 패리티(0) + 정지(1) = 10비트    │ │
│   │                                                         │ │
│   │   9600 bps일 때:                                        │ │
│   │   • 문자당 10비트 필요                                  │ │
│   │   • 초당 960자 전송 가능                                │ │
│   │   • 문자당 전송 시간: 10/9600 ≈ 1.04ms                 │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   오버헤드:                                                      │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   데이터 8비트당 최소 2비트 오버헤드 (25%)              │ │
│   │   • 8비트 데이터 + 1 시작 + 1 정지 = 10비트             │ │
│   │   • 효율: 8/10 = 80%                                    │ │
│   │                                                         │ │
│   │   패리티 추가 시: 8/11 ≈ 73%                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 클럭 허용 오차

```
┌─────────────────────────────────────────────────────────────────┐
│           클럭 동기화와 허용 오차                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   샘플링 타이밍:                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   비트 위치:   Start  D0   D1   D2   D3   D4   D5   D6  D7  Stop│
│   │                                                         │ │
│   │   이상적 샘플:  0.5   1.5  2.5  3.5  4.5  5.5  6.5  7.5 8.5 9.5│
│   │   (비트 시간 중간)                                       │ │
│   │                                                         │ │
│   │   오차 누적:                                             │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 송신 클럭과 수신 클럭이 약간 다를 수 있음    │    │ │
│   │   │ • 마지막 비트로 갈수록 오차 누적               │    │ │
│   │   │ • 일반적으로 ±2~3% 오차 허용                  │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   최대 허용 오차 계산:                                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 최대 오차 = ±0.5 / (N + 0.5)                  │    │ │
│   │   │ N = 총 비트 수                                 │    │ │
│   │   │                                                 │    │ │
│   │   │ 10비트 프레임: ±0.5/10.5 ≈ ±4.76%             │    │ │
│   │   │ (실제로는 마진 고려하여 ±2% 권장)              │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   오버샘플링 (Oversampling):                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   수신기는 보통 16배 오버샘플링 사용                    │ │
│   │                                                         │ │
│   │   • 9600 bps → 수신 클럭 153.6 kHz (×16)               │ │
│   │   • 시작 비트 감지 후 24클럭(1.5비트) 후 첫 샘플        │ │
│   │   • 이후 16클럭(1비트) 간격으로 샘플링                  │ │
│   │   • 노이즈 필터링 가능 (다수결)                         │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 프레이밍 에러

```
┌─────────────────────────────────────────────────────────────────┐
│           에러 검출                                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   프레이밍 에러 (Framing Error):                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   정지 비트 위치에서 1이 아닌 0이 검출됨                │ │
│   │                                                         │ │
│   │   원인:                                                  │ │
│   │   • 클럭 불일치                                         │ │
│   │   • 노이즈                                              │ │
│   │   • 전송 속도 설정 오류                                  │ │
│   │                                                         │ │
│   │   정상:   ... D7 │ Stop(1) │ Idle(1) │                  │ │
│   │                   └─ 1 감지                             │ │
│   │                                                         │ │
│   │   에러:   ... D7 │ Stop(0) │ ← 프레이밍 에러!           │ │
│   │                   └─ 0 감지 (비정상)                    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   오버런 에러 (Overrun Error):                                  │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   수신 버퍼가 비워지기 전에 새 데이터 도착              │ │
│   │   CPU가 이전 문자를 읽기 전에 다음 문자 수신            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   패리티 에러 (Parity Error):                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Even 패리티: 1의 개수이 짝수가 되어야 함              │ │
│   │   Odd 패리티: 1의 개수이 홀수가 되어야 함               │ │
│   │                                                         │ │
│   │   데이터: 01000001 (1의 개수: 2개)                      │ │
│   │   Even 패리티 비트: 0 (2+0=2, 짝수)                     │ │
│   │   Odd 패리티 비트: 1 (2+1=3, 홀수)                      │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **비동기식 전송**: 공통 클럭 없이 시작/정지 비트로 동기화
> **시작 비트**: 항상 0, 프레임 시작 알림
> **정지 비트**: 항상 1, 프레임 끝 알림

### 2. 빈출 키워드

- **UART**: 범용 비동기 송수신기
- **Framing Error**: 정지 비트 위치에서 0 감지
- **Oversampling**: 정확한 샘플링을 위한 고속 클럭 (×16)
- **Overhead**: 시작/정지 비트로 인한 20~25% 오버헤드

### 3. 예상 문제

**Q1. 비동기식 전송에서 시작 비트와 정지 비트의 역할을 설명하시오.**

**Q2. 비동기식 전송의 오버헤드를 계산하고 동기식 전송과 비교하시오.**

**Q3. 프레이밍 에러가 발생하는 원인과 해결 방안을 기술하시오.**
