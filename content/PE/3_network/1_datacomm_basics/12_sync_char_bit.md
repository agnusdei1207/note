# 동기식 전송 - 문자 동기 / 비트 동기 (Character/Bit Synchronization)

## 정의

**동기식 전송(Synchronous Transmission)**은 송수신자가 **공통 클럭**을 사용하여 데이터를 전송하는 방식이다. **문자 동기방식**은 특수 문자(SYN)로 동기를 맞추고, **비트 동기방식**은 플래그(Flag) 비트 패턴으로 프레임 구간을 식별한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           동기식 전송 개요                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   동기식 전송의 특징:                                            │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 연속적인 비트 스트림 전송                           │ │
│   │   • 시작/정지 비트 없음 → 오버헤드 감소                 │ │
│   │   • 공통 클럭 필요 (별도 클럭선 또는 클럭 복원)         │ │
│   │   • 대량 데이터 전송에 효율적                           │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   문자 동기방식 (Character-Oriented):                   │ │
│   │   │ SYN │ SYN │ STX │ 데이터 │ ETX │ BCC │             │ │
│   │                                                         │ │
│   │   비트 동기방식 (Bit-Oriented):                         │ │
│   │   │ Flag │ Address │ Control │ Data │ FCS │ Flag │     │ │
│   │   │ 01111110 │          ...              │ 01111110 │   │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 문자 동기방식 (BSC)

```
┌─────────────────────────────────────────────────────────────────┐
│           BSC (Binary Synchronous Communication)                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   프레임 구조:                                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   │ SYN │ SYN │ STX │    데이터    │ ETX │ BCC │       │ │
│   │   │ 16  │ 16  │ 02  │             │ 03  │    │       │ │
│   │   └──┬──┴──┬──┴──┬──┴─────────────┴──┬──┴────┘       │ │
│   │      │     │     │                    │                 │ │
│   │      │     │     │                    └─ 블록 검사 문자  │ │
│   │      │     │     └─ 텍스트 시작 (Start of Text)          │ │
│   │      │     └─ 동기 문자 2개 (Synchronize)                │ │
│   │      └─ 동기 문자 (ASCII 16H)                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   제어 문자:                                                     │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   SYN (16H): 동기 문자, 수신기 동기화                   │ │
│   │   STX (02H): 텍스트 시작                                │ │
│   │   ETX (03H): 텍스트 종료                                │ │
│   │   SOH (01H): 헤더 시작                                  │ │
│   │   EOT (04H): 전송 종료                                  │ │
│   │   DLE (10H): 데이터 링크 이스케이프                     │ │
│   │   ETB (17H): 블록 전송 종료                             │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   동기화 과정:                                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   1. 수신기는 SYN 문자 패턴(0010110)을 검색            │ │
│   │   2. 2개 연속 SYN 감지 → 문자 경계 인식                │ │
│   │   3. 이후 8비트씩 끊어서 문자로 해석                    │ │
│   │   4. STX ~ ETX 사이를 데이터로 처리                     │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   투명성 (Transparency):                                        │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   데이터에 제어 문자가 포함된 경우:                     │ │
│   │   DLE STX ... DLE ETX                                   │ │
│   │                                                         │ │
│   │   DLE 뒤의 제어 문자는 데이터로 취급                    │ │
│   │   DLE 자체는 DLE DLE로 표현                             │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 비트 동기방식 (HDLC)

```
┌─────────────────────────────────────────────────────────────────┐
│           HDLC (High-level Data Link Control)                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   프레임 구조:                                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ┌────────┬─────────┬─────────┬───────┬─────┬────────┐│ │
│   │   │ Flag   │ Address │ Control │ Info  │ FCS │ Flag   ││ │
│   │   │01111110│ 8/16비트│ 8/16비트│ 가변  │16/32│01111110││ │
│   │   └────────┴─────────┴─────────┴───────┴─────┴────────┘│ │
│   │                                                         │ │
│   │   Flag: 프레임 시작/종료 구분 (01111110)                 │ │
│   │   Address: 수신국/송신국 주소                            │ │
│   │   Control: 프레임 타입, 시퀀스 번호                      │ │
│   │   Info: 데이터 필드 (I-프레임만)                         │ │
│   │   FCS: 프레임 검사 시퀀스 (CRC)                          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   제어 필드 프레임 타입:                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   I-프레임 (Information):                               │ │
│   │   │ N(R) │ P/F │ N(S) │ 0 │   사용자 데이터 전송       │ │
│   │                                                         │ │
│   │   S-프레임 (Supervisory):                               │ │
│   │   │ N(R) │ P/F │  S   │ 0 1 │  흐름/에러 제어          │ │
│   │   RR, RNR, REJ, SREJ                                    │ │
│   │                                                         │ │
│   │   U-프레임 (Unnumbered):                                │ │
│   │   │  M   │ P/F │  M   │ 1 1 │  링크 제어               │ │
│   │   SABM, UA, DISC, DM, FRMR                              │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   비트 스터핑 (Bit Stuffing):                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   규칙: 연속된 1이 5개 나오면 0을 삽입                   │ │
│   │   목적: 데이터 내 01111110(Flag) 패턴 방지               │ │
│   │                                                         │ │
│   │   원본 데이터: 01111110                                 │ │
│   │   스터핑 후:    011111010  (5번째 1 뒤에 0 삽입)        │ │
│   │                                                         │ │
│   │   수신측: 연속 1이 5개 + 0이면 0 제거                   │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 문자 동기 vs 비트 동기 비교

```
┌─────────────────────────────────────────────────────────────────┐
│           문자 동기방식 vs 비트 동기방식                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분          │ 문자 동기 (BSC)     │ 비트 동기 (HDLC)  │ │
│   ├────────────────┼─────────────────────┼───────────────────┤ │
│   │ 동기 단위      │ 문자 (8비트)        │ 비트              │ │
│   │ 프레임 구분    │ 제어 문자 (SYN/STX) │ 플래그 (01111110) │ │
│   │ 투명성         │ DLE 사용            │ 비트 스터핑       │ │
│   │ 전송 효율      │ 낮음                │ 높음              │ │
│   │ 에러 제어      │ BCC                 │ CRC (FCS)         │ │
│   │ 전이중 지원    │ 제한적              │ 완전 지원         │ │
│   │ 복잡도         │ 낮음                │ 높음              │ │
│   │ 활용           │ 레거시 시스템       │ 현대 네트워크     │ │
│   └────────────────┴─────────────────────┴───────────────────┘ │
│                                                                 │
│   전송 효율 비교:                                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   BSC: 1000바이트 데이터 전송 시                        │ │
│   │   SYN(2) + STX(1) + DATA(1000) + ETX(1) + BCC(1)       │ │
│   │   = 1005바이트 → 효율 99.5%                             │ │
│   │                                                         │ │
│   │   HDLC: 1000바이트 데이터 전송 시                       │ │
│   │   Flag(1) + Addr(1) + Ctrl(1) + DATA(1000) +           │ │
│   │   FCS(2) + Flag(1) = 1006바이트 → 효율 99.4%            │ │
│   │                                                         │ │
│   │   💡 비트 스터핑 오버헤드 약 1.25% 추가                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **문자 동기방식**: SYN 문자로 동기화 (BSC 프로토콜)
> **비트 동기방식**: Flag 비트 패턴으로 프레임 구분 (HDLC, SDLC)
> **비트 스터핑**: 5개 연속 1 뒤에 0 삽입

### 2. 빈출 키워드

- **SYN Character**: 동기 문자 (ASCII 16H)
- **Flag Pattern**: 01111110 (HDLC)
- **Bit Stuffing**: 투명성 확보 기법
- **FCS (Frame Check Sequence)**: CRC 에러 검출

### 3. 예상 문제

**Q1. 문자 동기방식(BSC)과 비트 동기방식(HDLC)의 차이점을 비교 설명하시오.**

**Q2. HDLC에서 비트 스터핑(Bit Stuffing)이 필요한 이유와 동작 방식을 설명하시오.**
