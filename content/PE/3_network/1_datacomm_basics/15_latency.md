# 지연 (Latency)

## 정의

**지연(Latency)**은 데이터가 **출발지에서 목적지까지 도달하는 데 걸리는 총 시간**이다. 전파 지연, 전송 지연, 처리 지연, 큐잉 지연의 합으로 구성되며, 네트워크 성능의 핵심 지표다.

```
┌─────────────────────────────────────────────────────────────────┐
│           지연 기본 개념                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   총 지연 (Total Latency) = 전파 + 전송 + 처리 + 큐잉    │ │
│   │                                                         │ │
│   │   송신자                    네트워크                수신자│ │
│   │   ┌─────┐              ┌─────────────┐            ┌─────┐│ │
│   │   │     │──전송지연──▶│             │──전파지연──▶│     ││ │
│   │   │     │              │   처리지연   │            │     ││ │
│   │   │     │              │   큐잉지연   │            │     ││ │
│   │   └─────┘              └─────────────┘            └─────┘│ │
│   │                                                         │ │
│   │   ┌───────────────────────────────────────────────────┐ │ │
│   │   │                                                     │ │ │
│   │   │   전송 지연     큐잉 지연   처리 지연   전파 지연   │ │ │
│   │   │   (Transmission)(Queuing) (Processing)(Propagation)│ │ │
│   │   │   ──────────────────────────────────────────────── │ │ │
│   │   │   │◀──────────────── 총 지연 ────────────────────▶│ │ │
│   │   │                                                     │ │ │
│   │   └───────────────────────────────────────────────────┘ │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 지연의 4가지 구성 요소

```
┌─────────────────────────────────────────────────────────────────┐
│           지연 구성 요소                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 전파 지연 (Propagation Delay):                       │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   d_prop = 거리 / 전파 속도                     │    │ │
│   │   │                                                 │    │ │
│   │   │   • 물리적 거리에 비례                          │    │ │
│   │   │   • 전파 속도: 빛의 속도 (~2×10⁸ m/s in fiber)  │    │ │
│   │   │                                                 │    │ │
│   │   │   예: 1000km 광케이블                           │    │ │
│   │   │       d_prop = 10⁶ / 2×10⁸ = 5ms              │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ② 전송 지연 (Transmission Delay):                      │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   d_trans = 패킷 크기 / 대역폭                  │    │ │
│   │   │                                                 │    │ │
│   │   │   • 링크 대역폭에 반비례                        │    │ │
│   │   │   • 패킷 크기에 비례                            │    │ │
│   │   │                                                 │    │ │
│   │   │   예: 1500바이트 패킷, 1Gbps 링크              │    │ │
│   │   │       d_trans = (1500×8) / 10⁹ = 12μs         │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ③ 처리 지연 (Processing Delay):                        │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   • 라우터/스위치에서 패킷 처리 시간             │    │ │
│   │   │   • 헤더 분석, 라우팅 테이블 조회               │    │ │
│   │   │   • TTL 감소, 체크섬 검증                       │    │ │
│   │   │                                                 │    │ │
│   │   │   일반적으로 수 μs ~ 수 ms                      │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ④ 큐잉 지연 (Queuing Delay):                           │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   • 출력 포트 대기열에서 대기 시간              │    │ │
│   │   │   • 네트워크 혼잡도에 따라 가변                  │    │ │
│   │   │   • 트래픽 부하에 가장 큰 영향                   │    │ │
│   │   │                                                 │    │ │
│   │   │   ┌─────────────────────────────────────┐     │    │ │
│   │   │   │ 출력 큐: ▣▣▣▣▣│      │     │    │ │
│   │   │   │           ↑                        │     │    │ │
│   │   │   │       패킷 대기 중                  │     │    │ │
│   │   │   └─────────────────────────────────────┘     │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. RTT (Round-Trip Time)

```
┌─────────────────────────────────────────────────────────────────┐
│           왕복 시간 (RTT)                                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   정의: 패킷이 송신자에서 수신자까지 갔다가              │ │
│   │         다시 송신자로 돌아오는 총 시간                    │ │
│   │                                                         │ │
│   │   클라이언트                          서버               │ │
│   │   ┌─────────┐                      ┌─────────┐          │ │
│   │   │         │────── 요청 ──────────▶│         │          │ │
│   │   │         │                      │         │          │ │
│   │   │         │◀───── 응답 ──────────│         │          │ │
│   │   └─────────┘                      └─────────┘          │ │
│   │        │◀─────────── RTT ───────────▶│                  │ │
│   │                                                         │ │
│   │   RTT = 2 × (전파 지연 + 전송 지연 + 처리 지연 + 큐잉)   │ │
│   │                                                         │ │
│   │   측정 방법:                                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   ping 명령:                                    │    │ │
│   │   │   $ ping google.com                            │    │ │
│   │   │   64 bytes: time=15.2 ms                       │    │ │
│   │   │                      │                         │    │ │
│   │   │                      └── RTT                   │    │ │
│   │   │                                                 │    │ │
│   │   │   traceroute:                                   │    │ │
│   │   │   $ traceroute google.com                      │    │ │
│   │   │   1  router.local  1.2 ms  1.1 ms  1.3 ms     │    │ │
│   │   │   2  isp.gateway  5.4 ms  5.2 ms  5.5 ms      │    │ │
│   │   │   ...                                           │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   TCP 성능과 RTT:                                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • TCP 연결 설정: 3-way 핸드셰이크 = 1.5 × RTT         │ │
│   │   • TCP 윈도우 크기: BDP = 대역폭 × RTT                 │ │
│   │   • TCP 재전송: RTO (Retransmission Timeout) ≥ RTT      │ │
│   │                                                         │ │
│   │   💡 RTT가 클수록 TCP 처리량 제한                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 지연 vs 처리량 관계

```
┌─────────────────────────────────────────────────────────────────┐
│           지연-처리량 상관관계                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   파이프라인 비유:                                               │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   파이프 용량 = 대역폭 × 지연 (BDP)                      │ │
│   │                                                         │ │
│   │   ┌─────────────────────────────────────────────────┐  │ │
│   │   │                                                   │  │ │
│   │   │   물 파이프:                                      │  │ │
│   │   │                                                   │  │ │
│   │   │   ▶──[════════════════════════]──▶               │  │ │
│   │   │       │◀──── 파이프 길이 ────▶│                   │  │ │
│   │   │                                   │               │  │ │
│   │   │   단면적 = 대역폭                   │               │  │ │
│   │   │   길이 = 지연 (RTT)                 │               │  │ │
│   │   │   용량 = 단면적 × 길이 = BDP        │               │  │ │
│   │   │                                                   │  │ │
│   │   └─────────────────────────────────────────────────┘  │ │
│   │                                                         │ │
│   │   예시:                                                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   ① 위성 링크 (높은 지연, 제한된 대역폭):       │    │ │
│   │   │      대역폭: 1 Mbps                            │    │ │
│   │   │      RTT: 600ms                                │    │ │
│   │   │      BDP = 1×10⁶ × 0.6 = 600kb                │    │ │
│   │   │                                                 │    │ │
│   │   │   ② 광케이블 (낮은 지연, 높은 대역폭):          │    │ │
│   │   │      대역폭: 10 Gbps                           │    │ │
│   │   │      RTT: 20ms                                 │    │ │
│   │   │      BDP = 10×10⁹ × 0.02 = 200Mb              │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   Little's Law:                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   L = λ × W                                             │ │
│   │                                                         │ │
│   │   L: 시스템 내 평균 요청 수                              │ │
│   │   λ: 도착률 (요청/초)                                    │ │
│   │   W: 평균 대기 시간 (지연)                               │ │
│   │                                                         │ │
│   │   예: 초당 1000 요청, 평균 50ms 지연                     │ │
│   │       L = 1000 × 0.05 = 50개 동시 처리                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **지연**: 출발지→목적지 총 소요 시간
> **4가지 구성**: 전파 + 전송 + 처리 + 큐잉
> **RTT**: 왕복 시간, TCP 성능의 핵심 요소

### 2. 빈출 키워드

- **Propagation/Transmission Delay**: 전파/전송 지연
- **Queuing Delay**: 큐잉 지연, 혼잡도 영향
- **BDP (Bandwidth-Delay Product)**: 대역폭×지연, 파이프 용량
- **RTT (Round-Trip Time)**: 왕복 시간

### 3. 예상 문제

**Q1. 네트워크 지연의 4가지 구성 요소를 설명하고 각각의 특징을 기술하시오.**

**Q2. 대역폭-지연 곱(BDP)의 의미와 TCP 성능에 미치는 영향을 설명하시오.**
