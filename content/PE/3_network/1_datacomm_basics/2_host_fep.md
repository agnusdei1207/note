# 정보처리장치 (Host Computer, Front-End Processor FEP)

## 정의

**정보처리장치**는 데이터통신 시스템에서 데이터의 수집, 가공, 저장, 분배를 담당하는 중앙 처리 시스템으로, **호스트 컴퓨터(Host Computer)**와 **전단 처리기(Front-End Processor, FEP)**로 구성된다. 호스트는 비즈니스 로직과 데이터 처리를 수행하고, FEP는 통신 관련 부하를 호스트로부터 분리하여 전체 시스템 효율을 극대화한다.

```
+------------------------------------------------------------------+
|              정보처리장치 아키텍처 개요                             |
+------------------------------------------------------------------+

                         +------------------------+
                         |    호스트 컴퓨터       |
                         |    (Host Computer)     |
                         |                        |
                         |  - 비즈니스 로직       |
                         |  - 데이터베이스 관리   |
                         |  - 애플리케이션 실행   |
                         +-----------+------------+
                                     |
                                     | 고속 채널
                                     | (Bus & Tag, ESCON, FICON)
                                     |
                         +-----------v------------+
                         |   전단 처리기 (FEP)    |
                         | Front-End Processor    |
                         |                        |
                         |  - 통신 프로토콜 처리  |
                         |  - 회선 제어          |
                         |  - 오류 검출/정정     |
                         |  - 메시지 버퍼링      |
                         +-----------+------------+
                                     |
                    +----------------+----------------+
                    |                |                |
               +----v----+      +----v----+     +----v----+
               |   DCE   |      |   DCE   |     |   DCE   |
               +----+----+      +----+----+     +----+----+
                    |                |                |
               [통신 회선]       [통신 회선]      [통신 회선]
```

## 핵심 개념

### 1. 호스트 컴퓨터 (Host Computer)

**정의**: 데이터통신 시스템의 중추 역할을 수행하는 고성능 중앙처리장치로, 사용자 요청을 처리하고 데이터를 저장 및 관리하는 핵심 컴퓨팅 자원이다.

**호스트 컴퓨터 계층 구조**:
```
+------------------------------------------------------------------+
|                    호스트 컴퓨터 내부 구조                         |
+------------------------------------------------------------------+

+------------------------------------------------------------------+
|                     사용자 영역 (User Space)                      |
|  +----------------------------------------------------------+   |
|  |            애플리케이션 계층 (Application Layer)          |   |
|  |  - 업무 프로그램                                          |   |
|  |  - 트랜잭션 처리 모니터 (CICS, IMS)                      |   |
|  +----------------------------------------------------------+   |
|  +----------------------------------------------------------+   |
|  |            데이터 관리 계층 (Data Management)            |   |
|  |  - DBMS (DB2, Oracle, IMS/DB)                           |   |
|  |  - 파일 시스템                                            |   |
|  +----------------------------------------------------------+   |
+------------------------------------------------------------------+
|                     커널 영역 (Kernel Space)                      |
|  +----------------------------------------------------------+   |
|  |            운영체제 (Operating System)                   |   |
|  |  - 메모리 관리, 프로세스 스케줄링                        |   |
|  |  - I/O 서브시스템, 보안 제어                             |   |
|  +----------------------------------------------------------+   |
|  +----------------------------------------------------------+   |
|  |            하드웨어 추상화 계층 (HAL)                    |   |
|  |  - CPU, 메모리, 채널 인터페이스                          |   |
|  +----------------------------------------------------------+   |
+------------------------------------------------------------------+
|                     하드웨어 영역 (Hardware)                       |
|  +----------------------------------------------------------+   |
|  |            중앙처리장치 (CPU)                             |   |
|  |  - 다중 코어, 파이프라인, 캐시                            |   |
|  +----------------------------------------------------------+   |
|  +----------------------------------------------------------+   |
|  |            주기억장치 (Main Memory)                      |   |
|  |  - RAM, 가상 메모리 관리                                  |   |
|  +----------------------------------------------------------+   |
|  +----------------------------------------------------------+   |
|  |            채널 (I/O Channel)                            |   |
|  |  - FEP 연결, 디스크 연결, 테이프 연결                    |   |
|  +----------------------------------------------------------+   |
+------------------------------------------------------------------+
```

**호스트 컴퓨터 핵심 기능**:
```
+-------------------+-------------------------------------------+
|     기능 영역      |                 세부 기능                  |
+-------------------+-------------------------------------------+
| 연산 처리          | 비즈니스 로직 실행                         |
|                   | 트랜잭션 처리 (ACID 보장)                  |
+-------------------+-------------------------------------------+
| 데이터 관리        | 데이터베이스 CRUD 연산                     |
|                   | 데이터 무결성, 보안 관리                   |
+-------------------+-------------------------------------------+
| 자원 관리          | CPU, 메모리, 저장장치 할당                 |
|                   | 프로세스 스케줄링, 동시성 제어             |
+-------------------+-------------------------------------------+
| 사용자 관리        | 인증(Authentication), 인가(Authorization) |
|                   | 세션 관리, 감사(Audit) 로그                |
+-------------------+-------------------------------------------+
| 통신 서비스        | FEP와의 인터페이스                         |
|                   | 요청 분배, 로드 밸런싱                     |
+-------------------+-------------------------------------------+
```

### 2. 전단 처리기 (Front-End Processor, FEP)

**정의**: 호스트 컴퓨터와 통신망 사이에 위치하여 통신 관련 모든 기능을 전담하는 전용 프로세서로, 호스트의 통신 부하를 경감시키고 통신 효율을 최적화하는 장치이다.

**FEP 핵심 역할**:
```
+------------------------------------------------------------------+
|              FEP (Front-End Processor) 구조                       |
+------------------------------------------------------------------+

                    [호스트 컴퓨터 측]
                          |
                   +------v------+
                   |  호스트 채널 |
                   |  인터페이스  |
                   +------+------+
                          |
    +---------------------v---------------------+
    |              FEP 내부 구조                 |
    |  +-------------------------------------+  |
    |  |         제어 프로세서 (CPU)          |  |
    |  |   - 통신 제어 프로그램 실행          |  |
    |  |   - 프로토콜 처리                   |  |
    |  +-------------------------------------+  |
    |  +-------------------------------------+  |
    |  |         공유 메모리 (RAM)           |  |
    |  |   - 송수신 버퍼                     |  |
    |  |   - 라우팅 테이블                   |  |
    |  +-------------------------------------+  |
    |  +-------------------------------------+  |
    |  |      라인 어댑터 (Line Adapter)     |  |
    |  |   - 다중 회선 인터페이스            |  |
    |  |   - RS-232C, V.35, X.21            |  |
    |  +-------------------------------------+  |
    +---------------------+---------------------+
                          |
              +-----------+-----------+
              |           |           |
         +----v----+ +----v----+ +----v----+
         | 회선 1  | | 회선 2  | | 회선 N  |
         +---------+ +---------+ +---------+

                    [통신망 측]
```

**FEP 핵심 기능 상세**:
```
+-------------------+-------------------------------------------+
|     기능 영역      |                 세부 기능                  |
+-------------------+-------------------------------------------+
| 프로토콜 변환      | BSC <-> HDLC, SNA <-> TCP/IP              |
|                   | 서로 다른 통신 규약 간 변환               |
+-------------------+-------------------------------------------+
| 코드 변환         | ASCII <-> EBCDIC                          |
|                   | 문자 인코딩 자동 변환                     |
+-------------------+-------------------------------------------+
| 속도 변환         | 저속 회선 <-> 고속 채널                   |
|                   | 버퍼링을 통한 속도 매칭                   |
+-------------------+-------------------------------------------+
| 오류 제어         | CRC 검사, ARQ 재전송                      |
|                   | 오류 발생 시 자동 복구                    |
+-------------------+-------------------------------------------+
| 회선 제어         | 다중 회선 집중 관리                       |
|                   | 폴링(Polling), 셀렉팅(Selecting)          |
+-------------------+-------------------------------------------+
| 메시지 처리       | 메시지 조립/분해                          |
|                   | 우선순위 큐잉, 로드 밸런싱                |
+-------------------+-------------------------------------------+
| 보안 기능         | 접근 제어, 암호화/복호화                  |
|                   | 방화벽 기능 (일부 FEP)                    |
+-------------------+-------------------------------------------+
```

### 3. 호스트-FEP 연결 아키텍처

#### 채널 연결 방식
```
+------------------------------------------------------------------+
|              호스트-FEP 채널 연결 방식                              |
+------------------------------------------------------------------+

[1] Bus & Tag 방식 (구형 메인프레임)

    +-------------+                         +-------------+
    |   호스트    | ==== Bus & Tag 케이블 ==== |    FEP      |
    | (메인프레임)|    (병렬, 4.5MB/s)       |             |
    +-------------+                         +-------------+

    특징: 병렬 전송, 최대 122m, 케이블 다수 필요


[2] ESCON (Enterprise Systems Connection)

    +-------------+                         +-------------+
    |   호스트    | ====== ESCON 광케이블 ====== |    FEP      |
    | (메인프레임)|    (직렬, 17MB/s)        |             |
    +-------------+                         +-------------+

    특징: 광섬유, 최대 43km, 리피터 사용 시 100km+
          2x 더 빠른 전송, 단일 케이블


[3] FICON (Fiber Connection)

    +-------------+                         +-------------+
    |   호스트    | ====== FICON 광케이블 ====== |    FEP      |
    | (메인프레임)|    (직렬, 1-8Gbps)       |             |
    +-------------+                         +-------------+

    특징: 광섬유, 최대 100km, 파이버 채널 기반
          ESCON의 8배 이상 속도, 현대 표준
```

#### 채널 방식 비교
```
+----------+-----------+----------+----------+---------------+
| 방식     | 전송속도   | 거리     | 매체      | 비고          |
+----------+-----------+----------+----------+---------------+
| Bus&Tag  | 4.5 MB/s  | 122m     | 구리     | 병렬 전송     |
| ESCON    | 17 MB/s   | 43km     | 광섬유   | 직렬 전송     |
| FICON    | 1-8 Gbps  | 100km    | 광섬유   | FC 기반       |
+----------+-----------+----------+----------+---------------+
```

## 상세 설명

### 1. FEP의 동작 메커니즘

#### 데이터 수신 처리 흐름
```
+------------------------------------------------------------------+
|                 FEP 데이터 수신 처리 시퀀스                        |
+------------------------------------------------------------------+

    [원격지 DTE]           [DCE]           [FEP]           [Host]
         |                   |               |               |
         | == 데이터 프레임 ==>               |               |
         |                   | == 아날로그 ==>|               |
         |                   |    신호       |               |
         |                   |               |               |
         |                   |   +----------v-----------+    |
         |                   |   | 1. 신호 복원         |    |
         |                   |   |    (아날로그->디지털)|    |
         |                   |   +----------+-----------+    |
         |                   |              |                |
         |                   |   +----------v-----------+    |
         |                   |   | 2. 동기 확인         |    |
         |                   |   |    (SYN/FLAG 검출)  |    |
         |                   |   +----------+-----------+    |
         |                   |              |                |
         |                   |   +----------v-----------+    |
         |                   |   | 3. 오류 검사         |    |
         |                   |   |    (CRC 검증)       |    |
         |                   |   +----------+-----------+    |
         |                   |              |                |
         |                   |   +----------v-----------+    |
         |                   |   | 4. 프로토콜 변환     |    |
         |                   |   |    (HDLC->SNA)      |    |
         |                   |   +----------+-----------+    |
         |                   |              |                |
         |                   |   +----------v-----------+    |
         |                   |   | 5. 코드 변환         |    |
         |                   |   |    (ASCII->EBCDIC)  |    |
         |                   |   +----------+-----------+    |
         |                   |              |                |
         |                   |   +----------v-----------+    |
         |                   |   | 6. 버퍼링            |    |
         |                   |   |    (속도 조정)      |    |
         |                   |   +----------+-----------+    |
         |                   |              |                |
         |                   |              | === 채널 ====> |
         |                   |              |                |
         |                   |              |   +----------v-----------+
         |                   |              |   | 7. 애플리케이션     |
         |                   |              |   |    처리             |
         |                   |              |   +----------------------+
```

#### 폴링(Polling) 및 셀렉팅(Selecting) 제어
```
+------------------------------------------------------------------+
|              FEP의 회선 제어: 폴링 vs 셀렉팅                       |
+------------------------------------------------------------------+

[폴링(Polling)]: FEP가 단말기에게 수신할 데이터가 있는지 묻는 과정

    FEP                                           단말기
     |                                              |
     | -- Poll 메시지 (ENQ) ----------------------->|
     |    "데이터 보낼 것이 있습니까?"               |
     |                                              |
     | <-- 응답 (ACK + 데이터) --------------------|
     |    "네, 데이터를 보냅니다"                   |
     |                                              |
     | -- ACK (수신 확인) ------------------------->|
     |                                              |


[셀렉팅(Selecting)]: FEP가 단말기에게 수신 준비를 요청하는 과정

    FEP                                           단말기
     |                                              |
     | -- Select 메시지 (ENQ) --------------------->|
     |    "데이터를 보내려 합니다, 준비되셨나요?"    |
     |                                              |
     | <-- 응답 (ACK) -----------------------------|
     |    "네, 준비되었습니다"                      |
     |                                              |
     | -- 데이터 전송 ----------------------------->|
     |                                              |
     | <-- ACK (수신 확인) ------------------------|
     |                                              |
```

### 2. FEP와 CCU의 비교

```
+------------------------------------------------------------------+
|                    FEP vs CCU 비교 분석                            |
+------------------------------------------------------------------+

+-------------------+-------------------------------------------+
|     구분          |         FEP                    | CCU       |
+-------------------+-------------------------------------------+
| 정의              | 독립형 통신 전용 컴퓨터        | 호스트 내장 통신 카드|
+-------------------+-------------------------------------------+
| 위치              | 호스트 외부, 별도 하드웨어     | 호스트 내부 슬롯   |
+-------------------+-------------------------------------------+
| 처리 능력         | 강력 (전용 CPU, 대용량 메모리)  | 제한적 (전용 컨트롤러)|
+-------------------+-------------------------------------------+
| 지원 회선 수      | 수백 ~ 수천 회선               | 수십 회선         |
+-------------------+-------------------------------------------+
| 프로그래밍        | 범용 프로그램 가능             | 고정 기능 중심    |
+-------------------+-------------------------------------------+
| 비용              | 고가                           | 저가              |
+-------------------+-------------------------------------------+
| 가용성            | 이중화 용이                    | 이중화 어려움     |
+-------------------+-------------------------------------------+
| 주 용도           | 대형 온라인 시스템              | 중소형 시스템     |
+-------------------+-------------------------------------------+

[발전 과정]

    CCU (1960~70년대)
      |
      v
    FEP (1970~90년대)
      |
      v
    통신 서버 / TP Monitor (1990~2000년대)
      |
      v
    WAS / 미들웨어 / API Gateway (현재)
```

## 실무 적용

### 1. 현대적 아키텍처로의 진화

```
+------------------------------------------------------------------+
|          전통적 Host-FEP → 현대적 아키텍처 진화                     |
+------------------------------------------------------------------+

[전통적 아키텍처 (1980~1990년대)]

    +----------+     +----------+     +----------+
    | Terminal |<--->|   FEP    |<--->|  Host    |
    | (3270)   | SDLC| (3745)   |SNA  | (S/390)  |
    +----------+     +----------+     +----------+


[현대적 아키텍처 (2000년대 이후)]

    +----------+     +----------+     +----------+     +----------+
    | Web/     |<--->| API      |<--->| App      |<--->| DB       |
    | Mobile   | HTTP| Gateway  |REST | Server   |SQL  | Server   |
    | Client   |/TCP | (Kong)   |/gRPC|(Spring)  |/ORM | (Oracle) |
    +----------+     +----------+     +----------+     +----------+

    FEP 역할이 다음으로 분산:
    - API Gateway: 라우팅, 프로토콜 변환, 인증
    - Load Balancer: 트래픽 분산, 헬스 체크
    - Message Queue: 비동기 처리, 버퍼링
```

### 2. FEP 기능의 현대적 구현

```
+-------------------+-------------------------------------------+
|     FEP 기능      |         현대적 구현 기술                   |
+-------------------+-------------------------------------------+
| 프로토콜 변환      | API Gateway, Protocol Translator         |
|                   | (Kong, Apigee, AWS API Gateway)          |
+-------------------+-------------------------------------------+
| 코드 변환         | Character Encoding Library                |
|                   | (iconv, ICU, charset normalization)      |
+-------------------+-------------------------------------------+
| 속도 변환/버퍼링   | Message Queue, Stream Processing          |
|                   | (Kafka, RabbitMQ, Redis Streams)         |
+-------------------+-------------------------------------------+
| 오류 제어         | Circuit Breaker, Retry Pattern            |
|                   | (Hystrix, Resilience4j)                  |
+-------------------+-------------------------------------------+
| 회선/세션 제어    | Connection Pooling, Session Manager       |
|                   | (HikariCP, Redis Session Store)          |
+-------------------+-------------------------------------------+
| 보안 기능         | WAF, DDoS Protection, SSL Termination     |
```

## 기술사 시험 포인트

### 1. 핵심 암기 사항

```
+------------------------------------------------------------------+
|                Host vs FEP 핵심 비교                               |
+------------------------------------------------------------------+
| 구분          | 호스트 컴퓨터           | FEP                   |
+---------------+-------------------------+-----------------------+
| 주 역할       | 비즈니스 로직 처리      | 통신 처리             |
| CPU 집중도    | 높음 (연산 중심)        | 낮음 (I/O 중심)       |
| 메모리 용도   | 앱, DB 캐시            | 통신 버퍼             |
| 연결 방식     | 채널, 버스              | 다중 회선             |
| OS            | 범용 OS (z/OS, Unix)   | 전용 펌웨어/NCP       |
+------------------------------------------------------------------+

FEP 5대 핵심 기능:
1. 프로토콜 변환 (Protocol Conversion)
2. 코드 변환 (Code Conversion)
3. 속도 변환 (Speed Conversion)
4. 오류 제어 (Error Control)
5. 메시지 버퍼링 (Message Buffering)
```

### 2. 시험 출제 포인트

1. **FEP 도입 효과**: 호스트 부하 경감, 통신 효율 증대, 확장성 향상
2. **FEP 기능 서술**: 5대 핵심 기능 (변환x3, 오류제어, 버퍼링)
3. **채널 방식 비교**: Bus&Tag, ESCON, FICON 특징 및 속도
4. **CCU vs FEP**: 위치, 처리능력, 비용, 용도 비교
5. **SNA 구조**: 호스트-VTAM-FEP(NCP)-단말기 계층 구조

### 3. 기술사 답안 작성 요령

```
[문제] 정보처리장치의 구성요소인 호스트 컴퓨터와 FEP의 역할을 설명하고,
      FEP 도입의 필요성을 기술하시오.

[답안 구성]
1. 개요
   - 정보처리장치의 정의 및 데이터통신에서의 위치

2. 구성요소별 상세 설명
   가. 호스트 컴퓨터 (Host Computer)
      - 정의, 핵심 기능, 하드웨어 구조
   나. 전단 처리기 (FEP)
      - 정의, 5대 핵심 기능

3. FEP 도입 필요성 (효과)
   가. 호스트 부하 경감 (CPU 이용률 향상)
   나. 통신 효율 증대 (전용 처리)

   라. 가용성 향상 (FEP 이중화)

4. 호스트-FEP 연결 방식
   - 채널 방식: Bus&Tag, ESCON, FICON 비교

5. 현대적 진화 및 결론
   - FEP 기능의 API Gateway, Load Balancer 등으로 분산
```
