# 처리량 (Throughput)

## 정의

**처리량(Throughput)**은 네트워크나 시스템에서 **단위 시간당 실제로 전송된 데이터 양**이다. 이론적 최대 대역폭과 달리 프로토콜 오버헤드, 에러, 혼잡 등의 실제 환경 요소를 반영한 측정값이다.

```
┌─────────────────────────────────────────────────────────────────┐
│           처리량 기본 개념                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   대역폭 vs 처리량 vs 굿풋:                               │ │
│   │                                                         │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   대역폭 (Bandwidth):                           │    │ │
│   │   │   ┌───────────────────────────────────────┐   │    │ │
│   │   │   │ 이론적 최대 전송 용량                   │   │    │ │
│   │   │   │ 예: 1 Gbps 이더넷                      │   │    │ │
│   │   │   └───────────────────────────────────────┘   │    │ │
│   │   │                      ▼                         │    │ │
│   │   │   처리량 (Throughput):                         │    │ │
│   │   │   ┌───────────────────────────────────────┐   │    │ │
│   │   │   │ 실제 전송된 데이터 + 오버헤드          │   │    │ │
│   │   │   │ 프로토콜 헤더, 에러 복구 포함          │   │    │ │
│   │   │   │ 예: 950 Mbps                          │   │    │ │
│   │   │   └───────────────────────────────────────┘   │    │ │
│   │   │                      ▼                         │    │ │
│   │   │   굿풋 (Goodput):                              │    │ │
│   │   │   ┌───────────────────────────────────────┐   │    │ │
│   │   │   │ 응용 계층에 전달된 유용한 데이터만      │   │    │ │
│   │   │   │ 예: 900 Mbps                          │   │    │ │
│   │   │   └───────────────────────────────────────┘   │    │ │
│   │   │                                                 │    │ │
│   │   │   대역폭 ≥ 처리량 ≥ 굿풋                        │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 처리량 영향 요소

```
┌─────────────────────────────────────────────────────────────────┐
│           처리량에 영향을 미치는 요소                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 프로토콜 오버헤드:                                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   ┌─────────────────────────────────────┐     │    │ │
│   │   │   │ 이더넷 프레임 (1518바이트)           │     │    │ │
│   │   │   ├─────────────────────────────────────┤     │    │ │
│   │   │   │ 프리앰블 │ 헤더 │ 페이로드 │ FCS    │     │    │ │
│   │   │   │   8B    │ 14B  │  1500B   │  4B    │     │    │ │
│   │   │   └─────────────────────────────────────┘     │    │ │
│   │   │                      │                        │    │ │
│   │   │                      ▼                        │    │ │
│   │   │   ┌─────────────────────────────────────┐     │    │ │
│   │   │   │ IP 헤더 (20B) + TCP 헤더 (20B)       │     │    │ │
│   │   │   │ = 40B 오버헤드                       │     │    │ │
│   │   │   └─────────────────────────────────────┘     │    │ │
│   │   │                                                 │    │ │
│   │   │   유효 데이터 비율 = 1500 / (1518+20+20)       │    │ │
│   │   │                   ≈ 94.8%                      │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ② 전송 지연과 RTT:                                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   처리량 = TCP 윈도우 크기 / RTT               │    │ │
│   │   │                                                 │    │ │
│   │   │   예: 윈도우=64KB, RTT=100ms                   │    │ │
│   │   │       처리량 = 65536 / 0.1 = 655KB/s          │    │ │
│   │   │                = 5.24 Mbps                     │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ③ 패킷 손실과 재전송:                                 │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   손실률이 높으면:                              │    │ │
│   │   │   • TCP 혼잡 윈도우 감소                       │    │ │
│   │   │   • 재전송으로 인한 대역폭 낭비                 │    │ │
│   │   │   • 처리량 급격히 저하                          │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. TCP 처리량 공식

```
┌─────────────────────────────────────────────────────────────────┐
│           TCP 처리량 계산                                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   기본 공식:                                                     │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Throughput = Window Size / RTT                        │ │
│   │                                                         │ │
│   │   • Window Size: TCP 수신 윈도우                        │ │
│   │   • RTT: 왕복 시간 (Round-Trip Time)                    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   혼잡 제어 고려 공식 (Mathis 공식):                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Throughput = (MSS / RTT) × (1 / √p)                  │ │
│   │                                                         │ │
│   │   • MSS: 최대 세그먼트 크기                             │ │
│   │   • p: 패킷 손실 확률                                   │ │
│   │                                                         │ │
│   │   예: MSS=1460B, RTT=100ms, p=0.01                     │ │
│   │       Throughput = (1460/0.1) × (1/0.1)                │ │
│   │                 = 14600 × 10                           │ │
│   │                 = 146000 B/s = 1.17 Mbps               │ │
│   │                                                         │ │
│   │   💡 손실률이 1%일 때 처리량이 크게 저하됨              │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   윈도우 스케일링:                                               │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   기본 윈도우: 16비트 = 최대 65535바이트 (64KB)          │ │
│   │   스케일링: 14비트 확장 = 최대 1GB 윈도우                │ │
│   │                                                         │ │
│   │   고속 네트워크 예시:                                    │ │
│   │   • 대역폭: 10 Gbps                                     │ │
│   │   • RTT: 100ms                                          │ │
│   │   • 필요 윈도우: 10×10⁹ × 0.1 = 125 MB                  │ │
│   │   • 윈도우 스케일링 필수                                 │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **처리량**: 단위 시간당 실제 전송된 데이터 양
> **Throughput ≤ Bandwidth**: 처리량은 대역폭을 초과할 수 없음
> **Goodput ≤ Throughput**: 굿풋은 처리량보다 작거나 같음

### 2. 빈출 키워드

- **Bandwidth vs Throughput**: 이론 vs 실제
- **TCP Window Size**: 처리량 결정 핵심 요소
- **Mathis Formula**: 손실률과 처리량 관계
- **Goodput**: 유용 데이터 전송률

### 3. 예상 문제

**Q1. 대역폭, 처리량(Throughput), 굿풋(Goodput)의 차이점을 설명하시오.**

**Q2. TCP 처리량에 영향을 미치는 요소들을 기술하고 최적화 방안을 제시하시오.**

**Q3. 대역폭-지연 곱이 큰 네트워크에서 처리량을 최적화하는 방법을 설명하시오.**
