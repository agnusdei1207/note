# 데이터통신 시스템 구성요소 (DTE, DCE, CCU)

## 정의

**데이터통신 시스템(Data Communication System)**은 정보를 생성, 전송, 수신하는 하드웨어 및 소프트웨어 요소들의 유기적 결합 체계로, 단말장치(DTE), 데이터회선종단장치(DCE), 통신제어장치(CCU)의 세 가지 핵심 구성요소로 구성된다.

```
+------------------------------------------------------------------+
|                    데이터통신 시스템 구성도                        |
+------------------------------------------------------------------+

    [정보원]                    [전송 매체]                  [정보목적지]
        |                            |                            |
        v                            v                            v
+---------------+            +---------------+            +---------------+
|    DTE        |<---인터페이스--->|     DCE     |<---회선--->|     DCE        |
| (단말장치)     |            | (회선종단장치) |            | (회선종단장치) |
+---------------+            +---------------+            +---------------+
        |                            |                            |
        |                            |                            |
        v                            v                            v
+---------------+            +---------------+            +---------------+
|    DTE        |            |   전송로      |            |    DTE        |
| (단말장치)     |            | (통신망)      |            | (단말장치)     |
+---------------+            +---------------+            +---------------+

    [송신 측]                                                [수신 측]
```

---

## 핵심 개념

### 1. DTE (Data Terminal Equipment, 단말장치)

**정의**: 사용자가 데이터를 입력하거나 수신된 데이터를 출력하는 장치로, 데이터의 시작점과 끝점을 담당하는 최종 사용자 인터페이스 장비이다.

**핵심 특성**:
- 데이터의 생성(Source) 및 싱크(Sink) 역할
- 디지털 신호 처리 및 사용자 인터페이스 제공
- 일반적으로 DCE와 직렬 인터페이스로 연결

**주요 장비**:
```
+---------------------------+---------------------------+
|        DTE 유형 분류       |          예시 장비         |
+---------------------------+---------------------------+
| 입력 전용 DTE              | 키보드, 카드 리더기, 스캐너  |
| 출력 전용 DTE              | 프린터, 디스플레이, 플로터   |
| 입출력 겸용 DTE            | PC, 터미널, 워크스테이션    |
| 특수 목적 DTE              | POS 단말기, ATM, 산업용 PLC |
+---------------------------+---------------------------+
```

**DTE의 핵심 기능**:
1. **데이터 입력 기능**: 사용자 데이터의 디지털 변환
2. **데이터 출력 기능**: 수신 데이터의 사용자 표현
3. **오류 제어 기능**: 데이터 무결성 검증
4. **흐름 제어 기능**: 송수신 속도 조절

---

### 2. DCE (Data Circuit-terminating Equipment, 데이터회선종단장치)

**정의**: DTE와 전송 매체(통신 회선) 사이에서 신호 변환 및 회선 제어 기능을 수행하며, 데이터 통신망에 접속하기 위한 인터페이스를 제공하는 장비이다.

**핵심 특성**:
- 디지털 신호와 아날로그 신호 간 변환 (Modulation/Demodulation)
- 전송 매체와의 물리적/전기적 인터페이스 제공
- 회선의 설정, 유지, 해제 기능

**DCE 상세 구조**:
```
+----------------------------------------------------------+
|                    DCE 내부 구조                          |
+----------------------------------------------------------+

    [DTE 측]                                              [회선 측]
        |                                                    |
        v                                                    v
+---------------+      +---------------+      +---------------+
| 인터페이스     | --->| 신호 변환기    | --->| 회선 구동기    |
| 제어부        |      | (Modulator/   |      | (Line Driver) |
|               | <---| Demodulator)  | <---|               |
+---------------+      +---------------+      +---------------+
        |                    |                      |
        |                    v                      |
        |            +---------------+              |
        |            | 타이밍/동기   |              |
        +----------->| 제어부        |<-------------+
                     | (Clock Gen)   |
                     +---------------+
```

**주요 DCE 장비**:
```
+---------------------------+-------------------------------------------+
|       DCE 유형             |                 상세 설명                  |
+---------------------------+-------------------------------------------+
| 모뎀 (Modem)               | 아날로그 전화선에서 변복조 수행             |
|                           | MOdulator + DEModulator                   |
+---------------------------+-------------------------------------------+
| DSU (Data Service Unit)    | 디지털 전용선에서 신호 변환 및 회선 종단    |
|                           | 디지털 회선의 B8ZS, AMI 등 부호화 처리     |
+---------------------------+-------------------------------------------+
| CSU (Channel Service Unit) | T1/E1 등 디지털 회선의 유지 보수 기능      |
|                           | 루프백 테스트, 성능 모니터링               |
+---------------------------+-------------------------------------------+
| DSU/CSU                   | DSU와 CSU 기능을 통합한 복합 장비          |
|                           | 디지털 데이터 회선의 표준 DCE              |
+---------------------------+-------------------------------------------+
| NT (Network Termination)   | ISDN, xDSL 등의 망 종단 장치               |
|                           | 가입자 장비와 통신망의 경계점              |
+---------------------------+-------------------------------------------+
```

**DCE의 핵심 기능**:
1. **신호 변환**: 디지털 <-> 아날로그 신호 변환
2. **부호화/복호화**: 회선 부호화 방식 적용 (NRZ, AMI, HDB3 등)
3. **동기화**: 클럭 신호 생성 및 동기 타이밍 제공
4. **회선 제어**: 통신 회선의 설정 및 해제
5. **진단 기능**: 루프백 테스트, 오류 모니터링

---

### 3. CCU (Communication Control Unit, 통신제어장치)

**정의**: 중앙처리장치(CPU)와 통신 회선 사이에서 데이터 전송을 제어하고, 통신 프로토콜을 처리하며, 오류 검출 및 정정 기능을 수행하는 전용 하드웨어 장치이다.

**CCU 계층 구조**:
```
+----------------------------------------------------------+
|              CCU (통신제어장치) 계층 구조                   |
+----------------------------------------------------------+

    [Host CPU]                                             [통신 회선]
        |                                                    |
        v                                                    v
+----------------------------------------------------------+
|                  통신 제어기 (CCU)                         |
+----------------------------------------------------------+
|  +----------------------------------------------------+  |
|  |              프로토콜 처리부                         |  |
|  |  - HDLC/SDLC 프레임 처리                            |  |
|  |  - 동기 문자 삽입/검출 (SYN, FLAG)                  |  |
|  +----------------------------------------------------+  |
|  +----------------------------------------------------+  |
|  |              오류 제어부                            |  |
|  |  - CRC 생성/검출 (CRC-16, CRC-CCITT)               |  |
|  |  - 재전송 요구 (ARQ) 처리                           |  |
|  +----------------------------------------------------+  |
|  +----------------------------------------------------+  |
|  |              버퍼 관리부                            |  |
|  |  - 송수신 버퍼링                                    |  |
|  |  - 흐름 제어 (X-ON/X-OFF)                          |  |
|  +----------------------------------------------------+  |
|  +----------------------------------------------------+  |
|  |              회선 인터페이스부                      |  |
|  |  - 다중 회선 제어 (최대 256회선)                    |  |
|  |  - RS-232C, V.35 등 물리적 인터페이스              |  |
|  +----------------------------------------------------+  |
+----------------------------------------------------------+
```

**CCU의 핵심 기능**:
```
+-------------------+-------------------------------------------+
|     기능 영역      |                 세부 기능                  |
+-------------------+-------------------------------------------+
| 회선 제어          | 회선 접속/해제, 통신 모드 설정             |
|                   | (단방향/반이중/전이중)                     |
+-------------------+-------------------------------------------+
| 동기 제어          | 문자 동기 (SYN 문자), 비트 동기 (플래그)   |
|                   | 클럭 추출 및 타이밍 관리                   |
+-------------------+-------------------------------------------+
| 오류 제어          | 패리티 검사, CRC 생성/검출                |
|                   | ARQ (Stop-and-Wait, Go-Back-N, Selective) |
+-------------------+-------------------------------------------+
| 흐름 제어          | 버퍼링, X-ON/X-OFF 제어                   |
|                   | 윈도우 기반 흐름 제어                      |
+-------------------+-------------------------------------------+
| 프로토콜 처리      | 프레임 구성/분해, 제어 문자 처리           |
|                   | BSC, HDLC, SDLC 등 프로토콜 실행           |
+-------------------+-------------------------------------------+
```

## 상세 설명

### 1. DTE-DCE 인터페이스 표준

#### RS-232C (EIA-232) 인터페이스
```
+----------------------------------------------------------+
|              RS-232C 핵심 신호선                          |
+----------------------------------------------------------+

    [DTE]                                          [DCE]
       |                                              |
       |---- TD (Transmit Data, 핀 2) -------------->|
       |<--- RD (Receive Data, 핀 3) ---------------|
       |---- RTS (Request to Send, 핀 4) ----------->|
       |<--- CTS (Clear to Send, 핀 5) -------------|
       |---- DTR (Data Terminal Ready, 핀 20) ------>|
       |<--- DSR (Data Set Ready, 핀 6) ------------|
       |<--- DCD (Data Carrier Detect, 핀 8) -------|
       |                                              |

    핸드셰이크 순서:
    1. DTE -> DTR -> DCE (단말기 준비 완료)
    2. DCE -> DSR -> DTE (회선종단장치 준비 완료)
    3. DTE -> RTS -> DCE (전송 요청)
    4. DCE -> CTS -> DTE (전송 허가)
    5. DTE -> TD -> DCE (데이터 전송)
```

#### 주요 인터페이스 표준 비교
```
+----------+--------+--------+----------+----------+----------+
| 표준     |속도(bps)|거리(m) | 커넥터    | 용도      | 비고     |
+----------+--------+--------+----------+----------+----------+
| RS-232C  | 20K    | 15     | DB-25/9  | 단거리    | -12V~+12V|
| RS-422   | 10M    | 1,200  | DB-37    | 장거리    | 차동신호  |
| RS-423   | 100K   | 1,200  | DB-37    | 장거리    | 단독신호  |
| RS-485   | 10M    | 1,200  | DB-37    | 다중통신  | 32노드   |
| V.35     | 2M     | 60     | AMP      | WAN       | 동기식   |
| X.21     | 2M     | -      | DB-15    | 공중망    | 디지털   |
+----------+--------+--------+----------+----------+----------+
```

### 2. CCU의 프로그램 제어 방식

#### 프로그램 제어 I/O (Polling 방식)
```
+----------------------------------------------------------+
|              폴링(Polling) 기반 데이터 수신                |
+----------------------------------------------------------+

    CPU                                              CCU
     |                                                |
     | -- 1. 상태 폴링 (Status Poll) ---------------> |
     | <-- 상태 응답 (Data Ready/Not Ready) --------- |
     |                                                |
     | -- 2. 데이터 요청 (Read Command) ------------> |
     | <-- 데이터 전송 ----------------------------- |
     |                                                |
     | (반복 - 폴링 주기마다 상태 확인)                |

    장점: 구현 단순, 하드웨어 비용 저렴
    단점: CPU 오버헤드 큼, 실시간성 낮음
```

#### 인터럽트 제어 I/O
```
+----------------------------------------------------------+
|            인터럽트 기반 데이터 수신                       |
+----------------------------------------------------------+

    CPU                                              CCU
     |                                                |
     | (다른 작업 수행 중...)                          |
     |                                                |
     | <--- 인터럽트 요청 (INTR) -------------------- |
     |                                                |
     | -- 1. 인터럽트 응답 (INTA) -------------------> |
     | <-- 2. 인터럽트 벡터 ------------------------- |
     |                                                |
     | -- 3. ISR 진입 (Interrupt Service Routine)      |
     | -- 4. 데이터 읽기 ----------------------------> |
     | <-- 데이터 전송 ----------------------------- |
     |                                                |
     | (원래 작업 재개)                                |

    장점: CPU 효율 높음, 실시간 응답
    단점: 하드웨어 복잡도 증가
```

#### DMA (Direct Memory Access) 제어
```
+----------------------------------------------------------+
|                 DMA 기반 데이터 전송                       |
+----------------------------------------------------------+

    CPU                          CCU               Memory
     |                             |                   |
     | -- 1. DMA 초기화 ---------->|                   |
     |    (시작주소, 길이, 방향)    |                   |
     |                             |                   |
     | (다른 작업 수행 가능)        |                   |
     |                             |                   |
     |                             | == 데이터 전송 ==>|
     |                             |    (직접 메모리)   |
     |                             |                   |
     | <--- 2. DMA 완료 인터럽트 --|                   |
     |                             |                   |

    장점: CPU 개입 최소화, 고속 전송
    적용: 대용량 데이터, 고속 통신 회선
```

## 실무 적용

### 1. 현대적 DTE/DCE 진화

```
+-------------------+-------------------------------------------+
|     구분          |         과거 (Legacy)                     |
+-------------------+-------------------------------------------+
| DTE               | 더미 터미널, 카드 리더                    |
| DCE               | 아날로그 모뎀 (300bps ~ 56Kbps)          |
| CCU               | 독립형 하드웨어 (전용 캐비닛)              |
+-------------------+-------------------------------------------+

+-------------------+-------------------------------------------+
|     구분          |         현재 (Modern)                     |
+-------------------+-------------------------------------------+
| DTE               | 스마트폰, IoT 센서, 서버, 클라우드 VM      |
| DCE               | 광 모뎀, DSL 모뎀, 셀룰러 모듈            |
| CCU               | NIC 내장 펌웨어, SDN 컨트롤러             |
+-------------------+-------------------------------------------+
```

### 2. 기업 환경 구성 사례

**시나리오: 대형 금융사 데이터 센터**
```
+------------------------------------------------------------------+
|              금융 데이터 센터 네트워크 구성                         |
+------------------------------------------------------------------+

    [지점 단말]                  [데이터 센터]
         |                            |
    +----v----+                 +-----v-----+
    |  DTE    |                 |   DTE     |
    | (ATM)   |                 | (서버)    |
    +----+----+                 +-----+-----+
         |                            |
    +----v----+                 +-----v-----+
    |  DCE    |                 |   DCE     |
    |(DSU/CSU)|                 | (DSU/CSU) |
    +----+----+                 +-----+-----+
         |                            |
    +----v----------------------------v----+
    |         전용선 (T1/E1, 2.048Mbps)    |
    +--------------------------------------+

    설계 포인트:
    - 이중화된 DCE 구성 (Active-Standby)
    - DSU/CSU의 자동 백업 스위치
    - CCU 기능은 서버 NIC에 통합
```

### 3. 성능 최적화 지표

```
+-------------------+-------------------+-------------------+

+-------------------+-------------------+-------------------+
```

## 기술사 시험 포인트

### 1. 핵심 암기 사항

```
+------------------------------------------------------------------+
|                    DTE vs DCE vs CCU 비교                         |
+------------------------------------------------------------------+
| 구분      | DTE                  | DCE                  | CCU       |
+-----------+----------------------+----------------------+-----------+
| 위치      | 사용자 단말          | 회선 종단            | 호스트 측  |
| 주기능    | 데이터 생성/소비      | 신호 변환            | 프로토콜   |
| 신호      | 디지털               | 디지털+아날로그      | 디지털    |
| 인터페이스| RS-232C DTE 핀배열   | RS-232C DCE 핀배열   | 내부 버스  |
| 예시      | PC, 터미널           | 모뎀, DSU            | 통신카드   |
+------------------------------------------------------------------+
```

### 2. 시험 출제 포인트

1. **DTE/DCE 구분 문제**: RS-232C 커넥터의 핀 배열에 따른 DTE/DCE 식별
2. **CCU 기능 서술**: 회선제어, 동기제어, 오류제어, 흐름제어 4대 기능
3. **인터페이스 표준**: RS-232C, V.35, X.21의 특징 및 용도 비교
4. **모뎀 원리**: 변복조(Modulation/Demodulation) 과정 설명
5. **DSU vs CSU**: 디지털 회선에서 DSU와 CSU의 기능 차이

### 3. 기술사 답안 작성 요령

```
[문제] 데이터통신 시스템의 구성요소를 설명하고, DTE와 DCE의 차이점을 기술하시오.

[답안 구성]
1. 개요
   - 데이터통신 시스템의 정의 및 구성 개요

2. 구성요소별 상세 설명
   가. DTE (Data Terminal Equipment)
      - 정의, 기능, 예시
   나. DCE (Data Circuit-terminating Equipment)
      - 정의, 기능, 예시
   다. CCU (Communication Control Unit)
      - 정의, 4대 제어 기능

3. DTE vs DCE 비교 (표 작성)
   - 위치, 기능, 신호, 핀배열, 전압레벨 등

4. 실무 적용 및 결론
   - 현대적 변화와 통합 추세
```
