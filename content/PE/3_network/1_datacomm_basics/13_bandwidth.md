# 대역폭 (Bandwidth)

## 정의

**대역폭(Bandwidth)**은 통신 채널이 **단위 시간당 전송할 수 있는 데이터의 최대 양**이다. 주파수 영역에서는 신호가 차지하는 주파수 범위(Hz)를 의미하며, 데이터 통신에서는 초당 전송 비트 수(bps)로 표현한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           대역폭 개념                                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   주파수 영역 대역폭:                                            │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   전력                                                   │ │
│   │    ↑                                                     │ │
│   │    │     ┌─────────────────┐                            │ │
│   │    │     │                 │                            │ │
│   │    │     │   통과 대역     │                            │ │
│   │    │     │                 │                            │ │
│   │    │     └─────────────────┘                            │ │
│   │    │     │←── 대역폭 ──→│                               │ │
│   │    └─────┼───────────────┼──────────▶ 주파수             │ │
│   │          f1              f2                              │ │
│   │                                                         │ │
│   │   대역폭 B = f2 - f1 (Hz)                                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   데이터 전송 대역폭:                                            │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   대역폭 = 데이터 전송률 (bps)                           │ │
│   │                                                         │ │
│   │   예:                                                    │ │
│   │   • 이더넷: 1 Gbps (기가비트)                           │ │
│   │   • Wi-Fi: 300 Mbps (802.11n)                          │ │
│   │   • 5G: 10 Gbps                                         │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 대역폭 관련 용어

```
┌─────────────────────────────────────────────────────────────────┐
│           대역폭 관련 용어                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   처리량 (Throughput):                                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 실제 데이터 전송률                            │    │ │
│   │   │ • 오버헤드, 에러, 혼잡 등 고려                  │    │ │
│   │   │ • 항상 대역폭 ≤ 처리량                          │    │ │
│   │   │ • 단위: bps, Mbps, Gbps                        │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   굿풋 (Goodput):                                         │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 응용 계층에 전달된 유용한 데이터 전송률        │    │ │
│   │   │ • 프로토콜 오버헤드 제외                        │    │ │
│   │   │ • 재전송 데이터 제외                            │    │ │
│   │   │ • 굿풋 ≤ 처리량 ≤ 대역폭                        │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   관계:                                                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   대역폭 ≥ 처리량 ≥ 굿풋                        │    │ │
│   │   │                                                 │    │ │
│   │   │   예: 100Mbps 링크                              │    │ │
│   │   │   • 대역폭: 100 Mbps                            │    │ │
│   │   │   • 처리량: 85 Mbps (프로토콜 오버헤드)          │    │ │
│   │   │   • 굿풋: 75 Mbps (TCP 헤더, 재전송)             │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 대역폭-지연 곱 (Bandwidth-Delay Product)

```
┌─────────────────────────────────────────────────────────────────┐
│           대역폭-지연 곱 (BDP)                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   정의:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   BDP = 대역폭 × RTT (Round-Trip Time)                  │ │
│   │                                                         │ │
│   │   • 링크에 채울 수 있는 최대 데이터 양                   │ │
│   │   • TCP 윈도우 크기 결정에 중요                          │ │
│   │   • "파이프 용량"이라고도 함                             │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   예시:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   1Gbps 링크, RTT 100ms:                                │ │
│   │   BDP = 1×10^9 × 0.1 = 100×10^6 bits                   │ │
│   │       = 12.5 MB                                         │ │
│   │                                                         │ │
│   │   💡 TCP 윈도우가 최소 12.5MB 이상이어야                  │ │
│   │      링크를 완전히 활용 가능                             │ │
│   │                                                         │ │
│   │   ┌───────────────────────────────────────────┐         │ │
│   │   │                                             │         │ │
│   │   │   송신자        [파이프(BDP)]             수신자    │ │ │
│   │   │                                             │         │ │
│   │   │   ◀───── RTT ─────────────────────────▶   │         │ │
│   │   │                                             │         │ │
│   │   │   ┌─────────────────────────────────────┐ │         │ │
│   │   │   │         BDP만큼의 데이터            │ │         │ │
│   │   │   │         (12.5 MB)                  │ │         │ │
│   │   │   └─────────────────────────────────────┘ │         │ │
│   │   │                                             │         │ │
│   │   └───────────────────────────────────────────┘         │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   고속 장거리 링크에서 BDP가 매우 커짐                           │
│   → TCP 윈도우 스케일링 필요                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 대역폭 효율 (Spectral Efficiency)

```
┌─────────────────────────────────────────────────────────────────┐
│           대역폭 효율                                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   정의:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   η = 데이터 전송률 / 대역폭 (bps/Hz)                    │ │
│   │                                                         │ │
│   │   • 단위 대역폭당 전송 가능한 비트 수                     │ │
│   │   • 주파수 자원 활용 효율                                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   변조 방식별 효율:                                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  변조 방식 │ 대역폭 효율 │ 이론적 한계                   │ │
│   ├───────────┼────────────┼─────────────────────────────    │ │
│   │  BPSK     │  1 bps/Hz  │ 1 bit/symbol                  │ │
│   │  QPSK     │  2 bps/Hz  │ 2 bits/symbol                 │ │
│   │  8-PSK    │  3 bps/Hz  │ 3 bits/symbol                 │ │
│   │  16-QAM   │  4 bps/Hz  │ 4 bits/symbol                 │ │
│   │  64-QAM   │  6 bps/Hz  │ 6 bits/symbol                 │ │
│   │  256-QAM  │  8 bps/Hz  │ 8 bits/symbol                 │ │
│   └───────────┴────────────┴─────────────────────────────    │ │
│   │                                                         │ │
│   │   💡 높은 효율 → 높은 SNR 필요                           │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   Shannon 한계:                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   C = B × log₂(1 + S/N)                                 │ │
│   │                                                         │ │
│   │   최대 효율: η_max = log₂(1 + S/N) bps/Hz               │ │
│   │                                                         │ │
│   │   S/N = 20dB (100) → η_max ≈ 6.6 bps/Hz                │ │
│   │   S/N = 30dB (1000) → η_max ≈ 10 bps/Hz                 │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **대역폭**: 단위 시간당 최대 데이터 전송 용량
> **BDP**: 대역폭 × 지연 (링크에 채워진 데이터 양)
> **Spectral Efficiency**: 단위 대역폭당 전송 비트 (bps/Hz)

### 2. 빈출 키워드

- **Throughput/Goodput**: 처리량 vs 유용 데이터 전송률
- **Bandwidth-Delay Product**: TCP 윈도우 크기 결정
- **bps/Hz**: 대역폭 효율 단위
- **QoS**: 대역폭 품질 관리

### 3. 예상 문제

**Q1. 대역폭, 처리량(Throughput), 굿풋(Goodput)의 차이점을 설명하시오.**

**Q2. 대역폭-지연 곱(BDP)의 의미와 TCP 성능에 미치는 영향을 기술하시오.**

**Q3. 변조 방식에 따른 대역폭 효율의 차이와 Shannon 한계를 설명하시오.**
