# IP 멀티캐스트 (IP Multicast)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IP 멀티캐스트는 **"하나의 **송신자**가 **여러 **수신자**에게 **동시**에 **데이터**를 **전송**하는 **기술\"**로, **Class D** **주소**(224.0.0.0/4)**를 **사용**하고 **IGMP**(Internet Group Management Protocol)**가 **그룹 **관리**를 **하며 **PIM**(Protocol Independent Multicast)**가 **라우팅**을 **담당**한다.
> 2. **구조**: **Source**(송신자) → **Last-Hop Router**(마지막 **홉)** → **Rendezvous Point**(RP) → **Distribution Tree**(배포 **트리)** → **Receivers**(수신자)**로 **데이터**가 **전파**되고 **TTL**(Time **to **Live)**로 **범위**를 **제어**한다.
> 3. **라우팅**: **Dense Mode**(PIM-DM, **DVMRP)**는 **모든 **라우터**에 **상태**를 **유지**하고 **Sparse Mode**(PIM-SM)**는 **on-demand**로 **라우팅**을 **설정**하며 **SSM**(Source-Specific Multicast)**는 **PIM-SSM**으로 **구현**된다.

## Ⅰ. 개요 (Context & Background)

### 개념
IP 멀티캐스트는 **"일대다 통신"**이다.

**전송 방식 비교**:
| 유형 | 송신자 | 수신자 | 예시 |
|------|--------|--------|------|
| **Unicast** | 1 | 1 | 웹 |
| **Broadcast** | 1 | All | ARP |
| **Multicast** | 1 | Group | IPTV |
| **Anycast** | 1 | Nearest | CDN |

### 💡 비유
멀티캐스트는 ****라디오 **방송 ****과 같다.
- **방송국**: 송신자
- **주파수**: 그룹
- **수신기**: 청취자

## Ⅱ. 아키텍처 및 핵심 원리

### Class D 주소

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Class D Address Range                                               │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    IPv4 Multicast Address:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1110     │  │                                     │                                             │  │
    │  ────────  │  Class D                              │  Group ID                                   │  │
    │  4 bits   │  (224-239)                            │  (28 bits)                                  │  │
    │           │                                       │                                             │  │
    │  Range: 224.0.0.0 - 239.255.255.255                                                     │  │
    │  Reserved: 224.0.0.0/24 (Local), 239.0.0.0/8 (Administrative)                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    MAC Mapping:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  IPv4: 224.1.2.3                                                                      │  │
    │  → MAC: 01:00:5E:01:02:03  (01:00:5E:XX:XX:XX)                                               │  │
    │                                                                                         │  │
    │  → Low 24 bits of IPv4 multicast address mapped to MAC                                   │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### IGMP (Internet Group Management Protocol)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         IGMP Host Membership                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Host Join Process:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Host                                                  Router                           Multicast Source     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. Host wants to join group 239.1.1.1                                                │  │  │
    │  │  ──────────────────────────────────────────────────────────────────────────────────────>  │  │
    │  │                           │  IGMP Membership Report (Join 239.1.1.1)               │  │  │
    │  │                           │                                                       │  │  │
    │  │                           ▼                                                       │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  Router maintains group membership list                                       │  │  │  │
    │  │  │  • Interface eth0: {Host A, Host B, Host C}                                    │  │  │  │
    │  │  │  • Enable multicast forwarding for 239.1.1.1 on eth0                         │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                           │  IGMP Membership Query (Periodic: every 60s)           │  │  │
    │  │  <──────────────────────────────────────────────────────────────────────────────────────  │  │  │
    │  │                           │                                                       │  │  │
    │  │  ▼ Host responds with IGMP Membership Report                                        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │  │
    │  → If Host doesn't respond to 3 consecutive queries: Remove from group                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### PIM Sparse Mode (PIM-SM)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         PIM-SM Distribution Tree                                                │
└─────────────────────────────────────────────────────────────────────────────────────────

    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Source: 10.1.1.1 (sending to 239.1.1.1)                                              │  │
    │  RP (Rendezvous Point): Core Router with PIM-SM enabled                                      │  │
    │  Receivers: 192.168.1.10, 192.168.2.20, 192.168.3.30                                        │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
            │
            ▼
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Step 1: Source encapsulates multicast packet in PIM Register                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  PIM Register:                                                                         │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  PIM Version = 2                                                                    │  │  │  │  │
    │  │  │  Type = Register                                                                   │  │  │  │  │
    │  │  │  Group Address = 239.1.1.1                                                       │  │  │  │  │
    │  │  │  Source Address = 10.1.1.1                                                         │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │  │
    │  │  │  Multicast Data Packet (encapsulated)                                            │  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │            │ Unicast to RP                                                                 │  │
    │            ▼                                                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  RP receives PIM Register, decapsulates, forwards down distribution tree                 │  │  │
    │  │  │  Downstream Interface 1: Has members → Forward multicast                             │  │  │  │
    │  │  │  Downstream Interface 2: No members → Prune (don't forward)                          │  │  │  │
    │  │  │  Downstream Interface 3: Has members → Forward multicast                             │  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                                  │  │
    │            ▼                                                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Last-Hop Router: PIM Join/Prune to signal RP "I have receivers"                     │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │
    │  │  │  PIM Join: "I have receivers for 239.1.1.1"                                       │  │  │  │
    │  │  │  ┌────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  │  • RP adds this interface to distribution tree                                 │  │  │  │  │
    │  │  │  │  • RP starts forwarding multicast data                                       │  │  │  │  │
    │  │  │  └────────────────────────────────────────────────────────────────────────────────┘  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 멀티캐스트 라우팅 프로토콜

| 프로토콜 | 모드 | 범위 | 복잡도 | 사용처 |
|--------|------|------|--------|--------|
| **PIM-DM** | Dense | 소규 | 낮음 | LAN |
| **PIM-SM** | Sparse | 글로벌 | 높음 | WAN, IPTV |
| **DVMRP** | Dense | 소규 | 낮음 | 레거시 |
| **MOSPF** | Dense | AS | 중간 | OSPF 네트워크 |

### PIM-SM vs PIM-DM

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         PIM Sparse vs Dense Mode                                              │
└─────────────────────────────────────────────────────────────────────────────────────────

    PIM-DM (Dense Mode):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • All routers maintain full routing state                                                     │  │
    │  • Flood-and-Prune: Periodically flood PIM Prune messages                                  │  │
    │  → Good for: LANs with many receivers on each subnet                                         │  │
    │  → Bad for: WAN (scales poorly, high overhead)                                             │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    PIM-SM (Sparse Mode):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Only routers with downstream members maintain state                                     │  │
    │  • Explicit Join/Prune: Routers explicitly signal interest                                  │  │
    │  → Good for: WAN, IPTV (scales well)                                                      │  │
    │  → Bad for: Frequent join/leave churn                                                           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: IPTV 방송
**상황**: 10만 시청자
**판단**: PIM-SM + IGMPv3

```
┌─────────────────────────────────────────────────────────────────────────────────────────────────┐
│                         IPTV Multicast Deployment                                              │
└─────────────────────────────────────────────────────────────────────────────────────────

    Network:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Source (IPTV Encoder)                                                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Stream: udp://239.1.1.1:5004 (Channel 1)                                         │  │  │
    │  │  → Encodes video to MPEG-TS, sends to 239.1.1.1 (multicast group)                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                             │  │
    │            ▼ Backbone (PIM-SM)                                                        │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Core routers form distribution tree from RP                                            │  │  │
    │  │  • Only branches with active viewers receive traffic                                     │  │  │
    │  │  • Efficient bandwidth usage (single copy per branch)                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────  │  │
    │            │                                                                             │  │
    │            ▼ Last Mile (Access Network)                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  DSLAM switches enable IGMP snooping                                                      │  │  │
    │  │  • Viewer 1 changes channel → IGMP Leave 239.1.1.1, Join 239.1.1.2                  │  │  │
    │  │  • Switch stops forwarding 239.1.1.1 to Viewer 1's port                            │  │  │
    │  │  • Starts forwarding 239.1.1.2 to Viewer 1's port                                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Configuration:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # Enable PIM-SM on interface                                                             │  │
    │  interface eth0                                                                            │  │
    │      ip pim sparse-mode                                                                 │  │
    │                                                                                             │  │
    │  # Configure RP statically or dynamically                                              │  │
    │  ip pim rp-candidate                                                                       │  │
    │                                                                                             │  │
    │  # TTL scoping (prevent multicast from leaving local domain)                               │  │
    │  access-list 101 permit ip 224.0.0.0 0.0.0.255 any                                        │  │
    │  access-list 101 deny   ip any any                                                   │  │
    │  interface eth0                                                                          │  │
    │      ip multicast boundary 101                                                         │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 멀티캐스트 기대 효과

| 방식 | 대역폭 | 효율 | 복잡도 |
|----|--------|------|--------|
| **Unicast** | N× | 낮음 | 낮음 |
| **Broadcast** | 1× | 최악 | 낮음 |
| **Multicast** | 1× | 높음 | 높음 |

### 모범 사례

1. **TTL**: 범위 제한
2. **Pruning**: 불필요 전송 방지
3. **RP**: 안정적 선정
4. **S,G**: Source-Specific

### 미래 전망

1. **BIER**: Bit Indexed Explicit Replication
2. **SRM**: Stateless RPM
3. **L2**: Multicast over VXLAN
4. **QUIC**: HTTP/3 multicast

### ※ 참고 표준/가이드
- **RFC 1112**: Host Extensions
- **RFC 2365**: Administratively Scoped IP
- **RFC 4607**: PIM-SM
