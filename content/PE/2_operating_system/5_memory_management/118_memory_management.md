# 메모리 관리 (Memory Management)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 관리는 **"프로세스**에 **가상 **메모리**(Virtual **Memory)**를 **할당**하고 **물리 **메모리**(Physical **Memory)**로 **매핑**하는 **기법\\\"**으로, **Paging**(페이지 **단위 **매핑)**과 **Segmentation**(세그먼트 **단위 **매핑)**이 **있고 **페이지 **교체**(Page **Replacement)**로 **부족**한 **메모리**를 **해결**한다.
> 2. **Paging**: **가상 **주소**(Virtual **Address)**를 **페이지**(Page)**와 **오프셋**(Offset)**으로 **나누고 **Page **Table**로 **물리 **프레임**(Physical **Frame)**을 **찾으며 **TLB**(Translation **Lookaside **Buffer)**가 **매핑**을 **캐싱**하여 **성능**을 **향향**시킨다.
> 3. **교체 알고리즘**: **LRU**(Least **Recently **Used)**, **FIFO**, **Clock**알고리즘이 **있고 **Working **Set**과 **Thrashing**을 **모니터링**하여 **페이지 **폴트**(Page **Fault)**를 **최소화**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
메모리 관리는 **"가상 메모리 → 물리 메모리 매핑"**이다.

**메모리 관리 목표**:
| 목표 | 설명 |
|------|------|
| **Abstraction** | 프로세스마다 독립적인 주소 공간 |
| **Protection** | 프로세스간 메모리 격리 |
| **Sharing** | 코드 공유 (공유 라이브러리) |
| **Virtualization** | 물리 메모리보다 큰 가상 메모리 |

### Paging 아키텍처

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Paging Architecture (x86-64)                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Virtual Address Structure (48-bit):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  │  47    39  │    38    30  │    29    21  │    20    12  │    11     0  │           │  │
    │  │  PML4Idx   │  PDPTIdx     │    PDIdx     │    PTIdx     │   Offset     │           │  │
    │  │  (9 bits)  │  (9 bits)    │  (9 bits)    │  (9 bits)    │  (12 bits)   │           │  │
    │  └────────────┴──────────────┴──────────────┴──────────────┴──────────────┘           │  │
    │                                                                                         │  │
    │  → Each index selects entry in page table level                                        │  │
    │  → Page Size: 4096 bytes (2^12)                                                         │  │
    │  → Virtual Address Space: 256 TB (2^48)                                                 │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Page Walk (4-level):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Virtual Address: 0x0000_7fff_f123_4567                                                  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  CR3 (Page Table Base) = 0x1000                                                       │  │  │

    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │

    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  Address: 0x2000 + PDPTIdx * 8 = 0x2000 + 0xff * 8 = 0x27f8                       │  │  │  │  │
    │  │  │  Entry: 0x3000 (PD base address, present=1)                                       │  │  │  │  │

    │  │  │  Entry: 0x4000 (PT base address, present=1)                                       │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
```

### TLB (Translation Lookaside Buffer)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         TLB (Translation Lookaside Buffer)                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    TLB Structure:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  │  Tag (VPN)  │  Valid  │  Dirty  │  Permissions  │  PFN (Physical Frame Number)  │    │  │
    │  │  ────────────────────────────────────────────────────────────────────────────────────│  │
    │  │  0x12345     │  1      │  0      │  RW-U         │  0xabc                         │  │  │
    │  │  0x12346     │  1      │  1      │  RW-U         │  0xdef                         │  │  │
    │  │  0x12347     │  0      │  -      │  -            │  -                             │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────────┘
    │  → Cache of recent virtual → physical translations                                        │  │
    │  → TLB Hit: ~1 cycle, TLB Miss: ~10-100 cycles (page walk)                               │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    TLB Operation:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  CPU reads from virtual address 0x0000_7fff_f123_4567                                     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Step 1: Check TLB for VPN (Virtual Page Number) = 0xf123_4                          │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  TLB Hit: Found entry with VPN 0xf123_4 → PFN 0xabc                              │  │  │  │  │
    │  │  │  Physical Address = PFN * Page Size + Offset = 0xabc000 + 0x567 = 0xabc567       │  │  │  │  │
    │  │  │  → Access memory (1-2 cycles)                                                     │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                                                                       │  │  │
    │  │  OR:                                                                                   │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  TLB Miss: Entry not found                                                        │  │  │  │  │
    │  │  │  → Perform Page Walk (read 4 page table levels from memory)                        │  │  │  │  │
    │  │  │  → Update TLB with new entry                                                       │  │  │  │  │
    │  │  │  → Retry memory access                                                            │  │  │  │  │
    │  │  │  → Penalty: ~10-100 cycles                                                         │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 메모리 관리 기법 비교

| 기법 | 단위 | 장점 | 단점 | 사용처 |
|------|------|------|------|--------|

### Thrashing

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Thrashing (쓰레싱)                                                 │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Definition: System spends more time paging than executing
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Cause: Working Set > Physical Memory                                                   │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Process 1: Needs 100 pages, has 50                                                  │  │  │
    │  │  Process 2: Needs 100 pages, has 50                                                  │  │  │
    │  │  Process 3: Needs 100 pages, has 50                                                  │  │  │
    │  │  → Total: 300 pages needed, 150 available                                            │  │  │
    │  │  → Constant page faults, disk I/O bound                                              │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Working Set Model:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Working Set W(t, Δ): Set of pages referenced in last Δ time units                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  If |W(t, Δ)| > Physical Frames:                                                     │  │  │
    │  │     → Thrashing likely                                                                │  │  │
    │  │  Solution: Swap out processes until working sets fit                                   │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Huge Pages

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Huge Pages (대용량 페이지)                                         │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Comparison:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Standard 4KB Pages:                                                                     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1 GB memory = 262,144 page table entries (PTEs)                                     │  │  │
    │  │  TLB covers ~128 pages (512 KB) per process                                          │  │  │
    │  │  → High TLB miss rate for large databases                                            │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Huge 2MB Pages:                                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1 GB memory = 512 page table entries (PTEs)                                         │  │  │
    │  │  TLB covers ~32,768 MB per process                                                   │  │  │
    │  │  → Lower TLB miss rate, better performance                                           │  │  │
    │  │  → Used for: Database buffer pools, VM memory                                        │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 데이터베이스 서버 메모리 최적화
**상황**: 512GB RAM, PostgreSQL 400GB SGA
**판단**: Huge Pages + Swappiness 최적화

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Database Memory Optimization                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Configuration:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # Enable 2MB Huge Pages (Linux)                                                        │  │
    │  /proc/sys/vm/nr_hugepages = 200000  # 200000 * 2MB = 400GB                            │  │
    │                                                                                         │  │
    │  # PostgreSQL Configuration                                                             │  │
    │  shared_buffers = 400GB    # Use huge pages                                             │  │
    │  huge_pages = on           # Request huge pages from OS                                 │  │
    │                                                                                         │  │
    │  # Reduce swap usage (but don't disable completely)                                     │  │
    │  vm.swappiness = 1          # Only swap if absolutely necessary                          │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Benefits:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Reduced Page Table Overhead: 262M → 512K entries (512x reduction)                    │  │
    │  2. Improved TLB Hit Rate: 512KB → 64GB coverage                                        │  │
    │  3. Reduced Page Faults: Huge pages pinned in memory                                     │  │
    │  4. Lower Memory Latency: Fewer page walks                                               │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Monitoring:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # Check page fault rate                                                                 │  │
    │  $ ps -o min_flt,maj_flt -C postgres                                                     │  │
    │                                                                                         │  │
    │  # Check swap usage                                                                      │  │
    │  $ free -h                                                                               │  │
    │  $ vmstat 1                                                                              │  │
    │                                                                                         │  │
    │  # Check TLB misses (perf)                                                              │  │
    │  $ perf stat -e dTLB-load-misses,dTLB-store-misses -p $(pidof postgres)                  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 메모리 관리 기대 효과

| 기법 | 성능 향상 | 비고 |

1. **Huge Pages**: DB, VM
2. **mlock**: 중요 프로세스
3. **Swappiness**: 1 (서버)
4. **Overcommit**: 비활성화

1. **Persistent Memory**: NVDIMM
2. **Unified Memory**: CPU+GPU
3. **CXL**: Memory Expansion
4. **Secure**: Encryption
