# CPU 스케줄링 알고리즘 (CPU Scheduling Algorithms)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 스케줄링은 **"준비 **큐**(Ready Queue)**에서 **다음 **프로세스**를 **선택**하여 **CPU**에 **할당**하는 **기법\"**으로, **Preemptive**(선점 **강제)**와 **Non-preemptive**(비선점)**가 **있고 **FCFS**, **SJF**, **Round Robin**, **Multilevel Feedback Queue**가 **대표적**이다.
> 2. **평가 기준**: **CPU Utilization**(사용률), **Throughput**(처리량), **Turnaround Time**(반환 **시간), **Waiting Time**(대기 **시간), **Response Time**(응답 **시간)**으로 **평가**하며 **Fairness**(공정성)**과 **Starvation**(기아 **현상)** **방지**가 **중요**하다.
> 3. **알고리즘**: **MLFQ**(Multilevel Feedback Queue)**가 **현대 **OS**에서 **널리 **사용**되며 **Priority Boost**( Aging)**로 **Starvation**을 **방지**하고 **Time Slice**(시간 **조각)**을 **동적**으로 **조정**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
CPU 스케줄링은 **"CPU 시간 할당"**이다.

**스케줄링 목표**:
| 목표 | 설명 | 우선순위 |
|------|------|----------|
| **Fairness** | 공정한 분배 | 높음 |
| **Throughput** | 처리량 최대화 | 높음 |
| **Response Time** | 대기 시간 최소화 | 대화형 |
| **CPU Utilization** | 사용률 최대화 | 배치형 |

### 💡 비유

- **비행기**: 프로세스
- **활주**: CPU
- **대기**: Ready Queue

## Ⅱ. 아키텍처 및 핵심 원리

### 스케줄링 알고리즘 비교

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Scheduling Algorithms Comparison                                        │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Example Processes:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Process │  Arrival │  Burst │  Priority  │  Description                         │  │
    │  ─────────────────────────────────────────────────────────────────────────────────────│  │
    │  P1      │  0        │  8      │  2 (low)    │  CPU-bound                            │  │
    │  P2      │  1        │  4      │  1 (high)   │  I/O-bound                            │  │
    │  P3      │  2        │  9      │  2 (low)    │  CPU-bound                            │  │
    │  P4      │  3        │  5      │  1 (high)   │  I/O-bound                            │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    1. FCFS (First-Come First-Served):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  P1 (0-8) → P2 (8-12) → P3 (12-21) → P4 (21-26)                                        │  │  │
    │  │                                                                                       │  │  │
    │  │  Average: TT = (8+12+21+26)/4 = 16.75, WT = (0+8+12+21)/4 = 10.25                     │  │  │
    │  │  → Convoy effect (short process waits for long)                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    2. SJF (Shortest Job First):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  P1 (0-8) → P2 (8-12) → P4 (12-17) → P3 (17-26)                                        │  │  │
    │  │                                                                                       │  │  │
    │  │  Average: TT = (8+12+17+26)/4 = 15.75, WT = (0+8+12+17)/4 = 9.25                      │  │  │
    │  │  → Optimal for minimizing average TT (requires knowing burst time)                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    3. Round Robin (Quantum = 4):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  P1(0-4) → P2(4-8) → P3(8-12) → P4(12-16) → P1(16-20) → P2(20-24)                      │  │  │
    │  │  → P3(24-28) → P4(28-29) → P1(29-33) → P3(33-37)                                     │  │  │
    │  │                                                                                       │  │  │
    │  │  TT = 33, WT = 24 + 18 + 20 + 24 = 21.75                                                    │  │  │
    │  │  → Fair, no starvation, predictable                                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Multilevel Feedback Queue (MLFQ)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Multilevel Feedback Queue                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Structure:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Queue 0 (Highest Priority)                                                             │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Time Quantum: 8ms                                                                    │  │  │
    │  │  • Interactive processes (I/O-bound)                                                 │  │  │
    │  │  • Short CPU bursts                                                                  │  │  │
    │  │  → If time quantum not used: Promote? No, stay in Q0                                 │  │  │
    │  │  → If time quantum used: Demote to Q1                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                             │  │
    │            ▼                                                                             │  │
    │  Queue 1                                                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Time Quantum: 16ms                                                                   │  │  │
    │  │  • Interactive with longer bursts                                                    │  │  │
    │  │  → If time quantum used: Demote to Q2                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │                                                                             │  │
    │            ▼                                                                             │  │
    │  Queue 2 (Lowest Priority)                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Time Quantum: Round Robin (FCFS)                                                    │  │  │
    │  │  • CPU-bound processes                                                                │  │  │
    │  │  → Stay in Q2                                                                         │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Aging (Prevent Starvation):                                                            │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • Periodically boost all processes to Q0                                            │  │  │
    │  │  • Ensures fairness                                                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### 알고리즘 비교

| **RR** | 중간 | 공정 | O | X |

### Priority Scheduling

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Priority Scheduling with Aging                                          │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Problem: Starvation (low priority processes never run)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Priority: 1 (highest) ... 10 (lowest)                                                     │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Process A (Priority 1, CPU-bound): Always runnable                                   │  │  │
    │  │  Process B (Priority 10): Never gets CPU                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Solution: Aging
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Periodically increase priority of waiting processes                                    │  │
    │  • Example: Every second, priority = max(1, priority - 1)                                  │  │
    │  • Eventually, low priority process reaches highest priority                               │  │
    │  → Prevents starvation                                                                  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 리눅스 CFS 스케줄러
**상황**: 데스크톱 + 서버 혼합
**판단**: CFS (Completely Fair Scheduler)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Linux CFS (Completely Fair Scheduler)                                     │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Key Concept: Virtual Runtime (vruntime)
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • Each task maintains vruntime = actual runtime / weight                                   │  │
    │  • Scheduler picks task with smallest vruntime                                              │  │
    │  • Ensures fair CPU allocation based on weight                                               │  │
    │                                                                                         │  │
    │  Example:                                                                              │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Task A (weight=1024): vruntime = 100ms → 100/1024 = 0.098                              │  │  │
    │  │  Task B (weight=1024): vruntime = 200ms → 200/1024 = 0.195                              │  │  │
    │  │  Task C (weight=512):  vruntime = 150ms → 150/512 = 0.293                                │  │  │
    │  │                                                                                       │  │  │
    │  │  Scheduler picks Task A (smallest normalized vruntime)                                 │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Real-time Scheduling (Linux RT):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • SCHED_FIFO: Real-time FIFO (runs until blocked or yields)                               │  │
    │  • SCHED_RR: Real-time Round Robin (preemptible, time slice)                               │  │
    │  • Priority: 1 (highest) to 99 (lowest)                                                      │  │
    │  • Use case: Audio processing, industrial control                                           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 스케줄링 기대 효과

| 워크로드 | 최적 알고리즘 | 이유 |
|----------|-----------------|------|
| **Batch** | FCFS/SJF | 최소화 TT |
| **Interactive** | RR/MLFQ | 빠른 응답 |
| **Real-time** | EDF/RM | 데드라인 보장 |

### 모범 사례

1. **I/O**: Priority boost
2. **CPU-bound**: Lower priority
3. **시분할**: 짧은 Quantum
4. **모니터링**: run queue 길이

### 미래 전망

1. **Energy-aware**: 전력 절감
2. **Thermal**: 온도 기반
3. **ML**: 예측 스케줄링
4. **Real-time**: Linux PREEMPT_RT
