# 인터럽트 벡터 (Interrupt Vector)

## 정의

**인터럽트 벡터(Interrupt Vector)**는 각 인터럽트 소스에 대응하는 **인터럽트 서비스 루틴(ISR)의 시작 주소**를 저장하는 테이블의 엔트리다. CPU는 인터럽트 발생 시 벡터 번호를 통해 해당 ISR의 위치를 찾아 실행한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           인터럽트 벡터 기본 개념                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   인터럽트 발생 과정:                                     │ │
│   │                                                         │ │
│   │   외부 장치              CPU                  메모리     │ │
│   │   ┌─────┐              ┌─────┐              ┌─────┐    │ │
│   │   │ I/O │──인터럽트──▶│     │              │     │    │ │
│   │   │장치 │   신호       │ CPU │              │ RAM │    │ │
│   │   └─────┘              │     │              │     │    │ │
│   │                        │     │   벡터 조회   │     │    │ │
│   │                        │     │──────────────▶│ IDT │    │ │
│   │                        │     │              │     │    │ │
│   │                        │     │◀───ISR 주소──│     │    │ │
│   │                        │     │              └─────┘    │ │
│   │                        │     │                          │ │
│   │                        │     │   ISR 실행               │ │
│   │                        │     │──────────────▶           │ │
│   │                        └─────┘                          │ │
│   │                                                         │ │
│   │   💡 벡터 = 인터럽트 번호 → ISR 주소 매핑                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 인터럽트 벡터 테이블 (IVT)

```
┌─────────────────────────────────────────────────────────────────┐
│           인터럽트 벡터 테이블 구조                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   x86 Real Mode IVT (위치: 0x00000):                            │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   주소    │ 벡터 │ 용도                                  │ │
│   │   ────────┼──────┼────────────────────────────            │ │
│   │   0x0000  │  0   │ Divide Error (0으로 나누기)            │ │
│   │   0x0004  │  1   │ Debug Exception                       │ │
│   │   0x0008  │  2   │ NMI (Non-Maskable Interrupt)          │ │
│   │   0x000C  │  3   │ Breakpoint                            │ │
│   │   0x0010  │  4   │ Overflow                              │ │
│   │   0x0014  │  5   │ BOUND Range Exceeded                  │ │
│   │   0x0018  │  6   │ Invalid Opcode                        │ │
│   │   0x001C  │  7   │ Device Not Available                  │ │
│   │   ...     │ ...  │ ...                                   │ │
│   │   0x0080  │ 32   │ IRQ0 - Timer                          │ │
│   │   0x0084  │ 33   │ IRQ1 - Keyboard                      │ │
│   │   ...     │ ...  │ ...                                   │ │
│   │   0x03FF  │ 255  │ 사용자 정의 가능                       │ │
│   │                                                         │ │
│   │   각 엔트리: 4바이트 (IP 2바이트 + CS 2바이트)             │ │
│   │   총 256개 벡터 × 4바이트 = 1024바이트 (1KB)              │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   x86 Protected Mode IDT (Interrupt Descriptor Table):          │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   각 엔트리: 8바이트 (Descriptor)                        │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │  │15       Offset Low      0│15    Segment    0│    │ │
│   │   │  ├─────────────────────────┼──────────────────┤    │ │
│   │   │  │31       Reserved        │ P DPL 0  Type    │    │ │
│   │   │  ├─────────────────────────┼──────────────────┤    │ │
│   │   │  │63       Offset High                    32  │    │ │
│   │   │  └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   • P (Present): 유효한 엔트리 여부                       │ │
│   │   • DPL (Descriptor Privilege Level): 접근 권한           │ │
│   │   • Type: 인터럽트 게이트 / 트랩 게이트                    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

│   │               │ • 14: Page Fault    │                   │ │

│   │    5  │  37  │ LPT2              │ 13  │  45  │ FPU     │ │
│   │    6  │  38  │ Floppy            │ 14  │  46  │ HD Primary │ │
│   │    7  │  39  │ LPT1              │ 15  │  47  │ HD Secondary│ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │

### 3. 인터럽트 디스크립터 (Protected Mode)

```
┌─────────────────────────────────────────────────────────────────┐
│           인터럽트 게이트 디스크립터                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   8바이트 디스크립터 구조:                                        │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   63        48 47  45  40 39     32 31      16 15     0│ │
│   │   ┌───────────┬───┬───┬───┬───────┬─────────┬────────┐ │ │
│   │   │ Offset    │ P │DPL│ 0 │ Type  │ Reserved│Segment │ │ │
│   │   │ [31:16]   │   │   │   │       │         │Selector│ │ │
│   │   └───────────┴───┴───┴───┴───────┴─────────┴────────┘ │ │
│   │                                                         │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │  P (Present, 1 bit):                          │    │ │
│   │   │    1 = 유효한 게이트                          │    │ │
│   │   │    0 = 무효, 접근 시 예외 발생                │    │ │
│   │   │                                               │    │ │
│   │   │  DPL (Descriptor Privilege Level, 2 bits):    │    │ │
│   │   │    00 = Ring 0 (커널)                         │    │ │
│   │   │    11 = Ring 3 (사용자)                       │    │ │
│   │   │    소프트웨어 인터럽트 접근 권한               │    │ │
│   │   │                                               │    │ │
│   │   │  Type (4 bits):                               │    │ │
│   │   │    0xE = Interrupt Gate (IF=0)               │    │ │
│   │   │    0xF = Trap Gate (IF 유지)                  │    │ │
│   │   │    0x5 = Task Gate (태스크 전환)              │    │ │
│   │   │                                               │    │ │
│   │   │  Segment Selector:                            │    │ │
│   │   │    코드 세그먼트 선택자                       │    │ │
│   │   │                                               │    │ │
│   │   │  Offset:                                      │    │ │
│   │   │    ISR의 시작 주소 (32비트)                    │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   Interrupt Gate vs Trap Gate:                                  │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Interrupt Gate:                                       │ │
│   │   • 진입 시 IF=0 (인터럽트 비활성화)                     │ │
│   │   • 하드웨어 인터럽트에 사용                             │ │
│   │   • ISR 실행 중 다른 인터럽트 차단                       │ │
│   │                                                         │ │
│   │   Trap Gate:                                             │ │
│   │   • 진입 시 IF 유지 (인터럽트 상태 보존)                  │ │
│   │   • 소프트웨어 인터럽트/예외에 사용                       │ │
│   │   • ISR 실행 중 인터럽트 허용 가능                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **인터럽트 벡터**: ISR 주소를 저장하는 테이블 엔트리
> **IVT/IDT**: 인터럽트 벡터/디스크립터 테이블
> **벡터 번호**: 인터럽트 식별용 0-255 번호

### 2. 빈출 키워드

- **IVT (Interrupt Vector Table)**: Real Mode, 4바이트 엔트리
- **IDT (Interrupt Descriptor Table)**: Protected Mode, 8바이트 엔트리
- **Vector 0-31**: CPU 예외용으로 예약
- **Vector 32-255**: IRQ 및 소프트웨어 인터럽트

### 3. 예상 문제

**Q1. Real Mode IVT와 Protected Mode IDT의 차이점을 설명하시오.**

**Q2. 인터럽트 게이트와 트랩 게이트의 차이점과 각각의 용도를 기술하시오.**

**Q3. x86에서 인터럽트 벡터 번호 할당 체계를 설명하시오.**
