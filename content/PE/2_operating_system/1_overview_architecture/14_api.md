# API (Application Programming Interface)

## 정의

**API(Application Programming Interface)**는 응용 프로그램이 **운영체제나 라이브러리의 서비스를 사용**할 수 있도록 제공하는 함수, 프로토콜, 데이터 구조의 집합이다. 프로그램 간 상호작용을 위한 표준화된 인터페이스다.

```
┌─────────────────────────────────────────────────────────────────┐
│           API 계층 구조                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │              ┌─────────────────────┐                    │ │
│   │              │   응용 프로그램      │                    │ │
│   │              └──────────┬──────────┘                    │ │
│   │                         │                               │ │
│   │                         ▼                               │ │
│   │   ┌─────────────────────────────────────────────────┐  │ │
│   │   │                    API 계층                     │  │ │
│   │   │  ┌──────────┐  ┌──────────┐  ┌──────────┐      │  │ │
│   │   │  │ 표준 라이 │  │ 시스템   │  │ 서드파티 │      │  │ │
│   │   │  │ 브러리   │  │   API    │  │ 라이브러리│      │  │ │
│   │   │  │ (libc)   │  │(Win32 API)│ │ (OpenSSL)│      │  │ │
│   │   │  └──────────┘  └──────────┘  └──────────┘      │  │ │
│   │   └─────────────────────────────────────────────────┘  │ │
│   │                         │                               │ │
│   │                         ▼                               │ │
│   │              ┌─────────────────────┐                    │ │
│   │              │      운영체제        │                    │ │
│   │              │    (커널 서비스)     │                    │ │
│   │              └─────────────────────┘                    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. API와 시스템 호출의 관계

```
┌─────────────────────────────────────────────────────────────────┐
│           API vs 시스템 호출                                      │

│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   응용 프로그램                                          │ │
│   │        │                                                │ │
│   │        │ printf("Hello");  ← API 호출                   │ │
│   │        ▼                                                │ │
│   │   ┌─────────────────────────────────────┐               │ │
│   │   │           표준 C 라이브러리 (libc)   │               │ │
│   │   │                                     │               │ │
│   │   │   printf() → vfprintf() → write()   │               │ │
│   │   │                          │          │               │ │
│   │   │                          ▼          │               │ │
│   │   │                      시스템 호출     │               │ │
│   │   └─────────────────────────┬───────────┘               │ │
│   │                             │                            │ │
│   │   ══════════════════════════╪═════════════════════════  │ │
│   │                  사용자/커널 경계                        │ │
│   │   ══════════════════════════╪═════════════════════════  │ │
│   │                             ▼                            │ │
│   │   ┌─────────────────────────────────────┐               │ │
│   │   │              커널                    │               │ │
│   │   │         sys_write() 수행             │               │ │
│   │   └─────────────────────────────────────┘               │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   💡 API는 시스템 호출을 감싸는 래퍼(Wrapper) 함수              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. API 설계 원칙

```
┌─────────────────────────────────────────────────────────────────┐
│           좋은 API 설계 원칙                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 추상화 (Abstraction):                               │ │
│   │   • 내부 구현 숨기기                                    │ │
│   │   • "무엇"을 제공, "어떻게"는 숨김                      │ │
│   │                                                         │ │
│   │   ② 일관성 (Consistency):                               │ │
│   │   • 명명 규칙 통일                                      │ │
│   │   • 매개변수 순서 일관                                  │ │
│   │   • 반환 값 규칙 통일                                   │ │
│   │                                                         │ │
│   │   ③ 완전성 (Completeness):                              │ │
│   │   • 모든 필요 기능 제공                                 │ │
│   │   • 에러 처리 포함                                      │ │
│   │                                                         │ │
│   │   ④ 안전성 (Safety):                                    │ │
│   │   • 입력 검증                                           │ │
│   │   • 버퍼 오버플로우 방지                                │ │
│   │   • 예외 처리                                           │ │
│   │                                                         │ │
│   │   ⑤ 성능 (Performance):                                 │ │
│   │   • 최소 오버헤드                                       │ │
│   │   • 불필요한 시스템 호출 회피                           │ │
│   │                                                         │ │
│   │   ⑥ 이식성 (Portability):                               │ │
│   │   • 플랫폼 독립적 설계                                  │ │
│   │   • 표준 준수                                           │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   API vs ABI:                                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   API: 소스 코드 레벨 인터페이스                        │ │
│   │        함수 원형, 데이터 구조, 상수 정의                │ │
│   │                                                         │ │
│   │   ABI: 바이너리 레벨 인터페이스                         │ │
│   │        호출 규약, 데이터 정렬, 이름 맹글링              │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **API**: 프로그램이 OS/라이브러리 서비스를 사용하는 인터페이스
> **POSIX**: 유닉스 호환 OS를 위한 표준 API
> **libc**: 표준 C 라이브러리, 시스템 호출 래퍼 제공

### 2. 빈출 키워드

- **POSIX API**: 이식 가능한 유닉스 API 표준
- **Win32 API**: Windows 전용 API
- **Wrapper Function**: 시스템 호출을 감싸는 라이브러리 함수
- **Standard Library**: 표준 라이브러리 (libc)

### 3. 예상 문제

**Q1. API와 시스템 호출의 관계를 설명하고 API가 제공하는 이점을 기술하시오.**

**Q2. POSIX API와 Win32 API의 주요 차이점을 비교하시오.**

**Q3. 좋은 API 설계를 위한 원칙을 설명하고 그 중요성을 기술하시오.**
