# ABI (Application Binary Interface)

## 정의

**ABI(Application Binary Interface)**는 컴파일된 **바이너리 코드 간의 인터페이스** 규약이다. 함수 호출 규약(Calling Convention), 데이터 정렬, 레지스터 사용, 이름 맹글링 등 바이너리 레벨 호환성을 정의한다. API가 소스 레벨이라면 ABI는 바이너리 레벨이다.

```
┌─────────────────────────────────────────────────────────────────┐
│           API vs ABI 계층                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │              소스 코드 (Source Code)                     │ │
│   │                    │                                    │ │
│   │                    │ API                                │ │
│   │                    ▼                                    │ │
│   │   ┌──────────────────────────────────────────────────┐ │ │
│   │   │              컴파일러 (Compiler)                  │ │ │
│   │   └──────────────────────────────────────────────────┘ │ │
│   │                    │                                    │ │
│   │                    │ ABI                                │ │
│   │                    ▼                                    │ │
│   │              바이너리 코드 (Binary)                      │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분    │ API                    │ ABI                 │ │
│   ├──────────┼────────────────────────┼─────────────────────┤ │
│   │ 레벨     │ 소스 코드              │ 바이너리 코드       │ │
│   │ 대상     │ 프로그래머             │ 컴파일러/링커       │ │
│   │ 내용     │ 함수 원형, 구조체      │ 호출규약, 정렬     │ │
│   │ 호환성   │ 소스 호환              │ 바이너리 호환       │ │
│   │ 변경     │ 재컴파일 필요          │ 재컴파일 불필요     │ │
│   └──────────┴────────────────────────┴─────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 호출 규약 (Calling Convention)

```
┌─────────────────────────────────────────────────────────────────┐
│           주요 호출 규약 (x86-64)                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   System V AMD64 ABI (Linux/macOS):                             │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   정수/포인터 인자 전달 순서:                            │ │
│   │   RDI, RSI, RDX, RCX, R8, R9 (처음 6개)                 │ │
│   │   나머지는 스택으로 전달                                 │ │
│   │                                                         │ │
│   │   부동소수점 인자: XMM0 - XMM7                           │ │
│   │                                                         │ │
│   │   반환값: RAX (정수/포인터), XMM0 (부동소수점)           │ │
│   │                                                         │ │
│   │   Callee-saved: RBX, RBP, R12-R15                       │ │
│   │   Caller-saved: RAX, RCX, RDX, RSI, RDI, R8-R11         │ │
│   │                                                         │ │
│   │   예시: int func(int a, int b, int c, int d)            │ │
│   │   a → RDI, b → RSI, c → RDX, d → RCX                    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   Microsoft x64 Calling Convention (Windows):                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   정수/포인터 인자 전달 순서:                            │ │
│   │   RCX, RDX, R8, R9 (처음 4개)                           │ │
│   │   나머지는 스택으로 전달                                 │ │
│   │                                                         │ │
│   │   반환값: RAX                                            │ │
│   │                                                         │ │
│   │   Callee-saved: RBX, RBP, RDI, RSI, R12-R15             │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   32비트 호출 규약:                                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   cdecl: 인자를 스택으로, 호출자가 스택 정리             │ │
│   │   stdcall: 인자를 스택으로, 피호출자가 스택 정리         │ │
│   │   fastcall: 처음 2개 인자를 레지스터로                   │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 데이터 정렬과 바이트 순서

```
┌─────────────────────────────────────────────────────────────────┐
│           데이터 정렬 (Alignment)                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   자연스러운 정렬 (Natural Alignment):                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • char:  1바이트 정렬                          │    │ │
│   │   │ • short: 2바이트 정렬                          │    │ │
│   │   │ • int:   4바이트 정렬                          │    │ │
│   │   │ • long:  8바이트 정렬 (64비트)                 │    │ │
│   │   │ • pointer: 8바이트 정렬 (64비트)               │    │ │
│   │   │ • double: 8바이트 정렬                         │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   구조체 패딩 예시:                                      │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ struct Example {                              │    │ │
│   │   │     char c;     // 1바이트                    │    │ │
│   │   │     // 3바이트 패딩                           │    │ │
│   │   │     int i;      // 4바이트                    │    │ │
│   │   │     double d;   // 8바이트                    │    │ │
│   │   │ };  // 총 16바이트                            │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   메모리 레이아웃:                                       │ │
│   │   ┌───┬───┬───┬───┬───┬───┬───┬───┬───────────────┐    │ │
│   │   │ c │pad│   int    │      double               │    │ │
│   │   └───┴───┴───┴───┴───┴───┴───┴───┴───────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│           바이트 순서 (Endianness)                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   0x12345678을 메모리에 저장:                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Little Endian (x86, x86-64):                          │ │
│   │   ┌────┬────┬────┬────┐                                │ │
│   │   │ 78 │ 56 │ 34 │ 12 │  (낮은 주소 → 높은 주소)        │ │
│   │   └────┴────┴────┴────┘                                │ │
│   │                                                         │ │
│   │   Big Endian (네트워크, 일부 RISC):                     │ │
│   │   ┌────┬────┬────┬────┐                                │ │
│   │   │ 12 │ 34 │ 56 │ 78 │  (낮은 주소 → 높은 주소)        │ │
│   │   └────┴────┴────┴────┘                                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

│                                                                 │

│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ✗ 기존 라이브러리와 호환성 상실                        │ │

│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │

│   ┌─────────────────────────────────────────────────────────┐ │

│   │   • 새 시스템 호출은 추가만 가능 (삭제 불가)            │ │

└─────────────────────────────────────────────────────────────────┘

## 기술사 시험 포인트

### 1. 핵심 정의

> **ABI**: 바이너리 코드 간 인터페이스 규약
> **Calling Convention**: 함수 호출 시 인자 전달, 반환, 레지스터 사용 규칙
> **ABI vs API**: ABI는 바이너리 레벨, API는 소스 레벨

### 2. 빈출 키워드

- **Calling Convention**: 인자 전달 방식 (레지스터/스택)
- **Endianness**: 바이트 저장 순서 (Big/Little Endian)
- **Alignment**: 데이터 정렬 요구사항
- **Name Mangling**: 심볼 이름 변환 (C++)

### 3. 예상 문제

**Q1. API와 ABI의 차이점을 설명하고 각각의 중요성을 기술하시오.**

**Q2. x86-64 System V ABI의 함수 호출 규약을 설명하시오.**

**Q3. ABI 호환성이 깨지는 경우와 그 영향을 설명하고 ABI 안정성을 유지하는 방안을 기술하시오.**
