# 인터럽트 (Interrupt)

## 정의

**인터럽트(Interrupt)**는 CPU에게 **외부 또는 내부 이벤트의 발생을 알리는 신호**다. CPU는 현재 작업을 중단하고 인터럽트 서비스 루틴(ISR)을 실행한 후 원래 작업으로 복귀한다. 운영체제의 핵심 메커니즘으로 비동기적 이벤트 처리를 가능하게 한다.

```

│           인터럽트 처리 개요                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   CPU: 사용자 프로그램 실행 중...                        │ │
│   │        │                                                │ │
│   │        │  ① 인터럽트 발생 (하드웨어/소프트웨어)          │ │
│   │        ▼                                                │ │
│   │   ┌────────────────────────────────────────────┐        │ │
│   │   │  ② 현재 상태 저장 (PC, PSW, 레지스터)      │        │ │

│   │   └────────────────────────────────────────────┘        │ │

│   │        │  ⑤ 저장된 상태 복원                           │ │
│   │        ▼                                                │ │
│   │   사용자 프로그램 계속 실행...                          │ │
```

## 핵심 개념

### 1. 인터럽트 종류

```
┌─────────────────────────────────────────────────────────────────┐
│           인터럽트 분류                                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 하드웨어 인터럽트 (Hardware Interrupt):              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 외부 장치에서 발생 (비동기적)                │    │ │
│   │   │ • I/O 완료, 타이머 만료, 키보드 입력          │    │ │
│   │   │ • INTR 핀을 통해 CPU에 전달                    │    │ │
│   │   │                                                 │    │ │
│   │   │ 예: 디스크 읽기 완료, 네트워크 패킷 도착       │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ② 소프트웨어 인터럽트 (Software Interrupt):            │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 프로그램이 명시적으로 발생                   │    │ │
│   │   │ • 시스템 호출 (INT 0x80, syscall)             │    │ │
│   │   │ • 동기적 발생                                  │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ③ 예외/트랩 (Exception/Trap):                         │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 프로그램 실행 중 오류 발생                   │    │ │
│   │   │ • 동기적 발생                                  │    │ │
│   │   │                                                 │    │ │
│   │   │ 종류:                                          │    │ │
│   │   │ • Fault: 복구 가능 (페이지 폴트)               │    │ │
│   │   │ • Trap: 의도적 (브레이크포인트)                │    │ │
│   │   │ • Abort: 복구 불가 (하드웨어 오류)             │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   비교:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분      │ 하드웨어 인터럽트 │ 소프트웨어 인터럽트     │ │
│   ├────────────┼───────────────────┼────────────────────────┤ │
│   │ 발생 원인  │ 외부 장치         │ 프로그램 명령          │ │
│   │ 동기성     │ 비동기            │ 동기                   │ │
│   │ 발생 시점  │ 언제든지          │ 특정 명령 실행 시      │ │
│   │ 예시       │ I/O 완료          │ 시스템 호출            │ │
│   └────────────┴───────────────────┴────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 인터럽트 처리 과정

```
┌─────────────────────────────────────────────────────────────────┐
│           인터럽트 처리 상세 과정                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 인터럽트 요청 (IRQ):                                 │ │
│   │      장치가 인터럽트 컨트롤러에 신호 전송                │ │
│   │                                                         │ │
│   │   ② 인터럽트 컨트롤러 우선순위 판별:                     │ │
│   │      PIC/APIC가 여러 인터럽트 중 우선순위 결정           │ │
│   │                                                         │ │
│   │   ③ CPU 인터럽트 확인:                                   │ │
│   │      명령어 사이클마다 인터럽트 신호 확인                │ │
│   │                                                         │ │
│   │   ④ 하드웨어 자동 저장 (CPU):                            │ │
│   │      ┌───────────────────────────────────────────────┐ │ │
│   │      │ • PC (Program Counter): 다음 실행 주소        │ │ │
│   │      │ • PSW (Program Status Word): 상태 플래그      │ │ │
│   │      │ • 스택 포인터 전환 (사용자 → 커널)            │ │ │
│   │      └───────────────────────────────────────────────┘ │ │
│   │                                                         │ │
│   │   ⑤ 인터럽트 벡터 조회:                                  │ │
│   │      ┌───────────────────────────────────────────────┐ │ │
│   │      │ IVT/IDT에서 핸들러 주소 조회                   │ │ │
│   │      │ Vector = IRQ + Base                            │ │ │
│   │      │ Handler = IDT[Vector]                          │ │ │
│   │      └───────────────────────────────────────────────┘ │ │
│   │                                                         │ │
│   │   ⑥ ISR (Interrupt Service Routine) 실행:              │ │
│   │      ┌───────────────────────────────────────────────┐ │ │
│   │      │ ISR Entry:                                     │ │ │
│   │      │   push rax, rbx, rcx...  // 레지스터 저장     │ │ │
│   │      │   call handler            // 실제 처리        │ │ │
│   │      │   pop rcx, rbx, rax...   // 레지스터 복원     │ │ │
│   │      │   iret                    // 인터럽트 복귀     │ │ │
│   │      └───────────────────────────────────────────────┘ │ │
│   │                                                         │ │
│   │   ⑦ 인터럽트 복귀 (IRET):                                │ │
│   │      저장된 PC, PSW 복원 → 원래 프로그램 계속 실행       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────┐

│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **인터럽트**: CPU에게 이벤트 발생을 알리는 신호
> **ISR**: 인터럽트 서비스 루틴, 인터럽트 처리 함수
> **IDT**: 인터럽트 디스크립터 테이블, 핸들러 주소 저장

### 2. 빈출 키워드

- **IRQ (Interrupt Request)**: 인터럽트 요청 번호
- **Interrupt Vector**: 인터럽트 핸들러 주소를 가리키는 인덱스
- **Context Save/Restore**: 인터럽트 처리 시 상태 저장/복원
- **Nested Interrupt**: 인터럽트 중첩 처리

### 3. 예상 문제

**Q1. 하드웨어 인터럽트와 소프트웨어 인터럽트의 차이점을 설명하고 각각의 예시를 들어 기술하시오.**

**Q2. 인터럽트 처리 과정을 단계별로 상세히 설명하시오.**

**Q3. 인터럽트 우선순위와 중첩 인터럽트의 개념을 설명하고 운영체제에서의 활용을 기술하시오.**
