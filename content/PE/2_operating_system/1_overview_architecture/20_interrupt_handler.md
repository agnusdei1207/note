# 인터럽트 핸들러 (Interrupt Handler)

## 정의

**인터럽트 핸들러(Interrupt Handler)**는 특정 인터럽트가 발생했을 때 **실행되는 함수 또는 루틴**이다. ISR과 유사하지만 핸들러는 더 일반적인 용어로, 이벤트 처리, 예외 처리, 하드웨어 응답 등 다양한 컨텍스트에서 사용된다.

## 핵심 개념

### 1. 핸들러 등록과 디스패치

```
┌─────────────────────────────────────────────────────────────────┐
│           핸들러 등록 및 호출 과정                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Linux 커널 핸들러 등록:                                        │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   request_irq() 함수:                                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   int request_irq(                              │    │ │
│   │   │       unsigned int irq,           // IRQ 번호   │    │ │
│   │   │       irq_handler_t handler,      // 핸들러 함수│    │ │
│   │   │       unsigned long flags,        // 플래그     │    │ │
│   │   │       const char *name,           // 장치 이름  │    │ │
│   │   │       void *dev                   // 장치 데이터│    │ │
│   │   │   );                                           │    │ │
│   │   │                                                 │    │ │
│   │   │   예: 키보드 핸들러 등록                        │    │ │
│   │   │   request_irq(1, keyboard_handler,             │    │ │
│   │   │               IRQF_SHARED, "keyboard", dev);    │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   핸들러 함수 형식:                                       │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   irqreturn_t handler(int irq, void *dev_id) { │    │ │
│   │   │       // 인터럽트 처리                          │    │ │
│   │   │       return IRQ_HANDLED;  // 또는 IRQ_NONE    │    │ │
│   │   │   }                                            │    │ │
│   │   │                                                 │    │ │
│   │   │   반환값:                                       │    │ │
│   │   │   • IRQ_HANDLED: 처리 완료                     │    │ │
│   │   │   • IRQ_NONE: 이 장치의 인터럽트 아님           │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   핸들러 디스패치 과정:                                          │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   1. 인터럽트 발생 → IRQ 번호 획득                        │ │
│   │   2. IRQ 디스크립터에서 핸들러 체인 조회                  │ │
│   │   3. 등록된 각 핸들러 순차 호출                           │ │
│   │   4. IRQ_HANDLED 반환 시 중단                            │ │
│   │   5. EOI(End of Interrupt) 전송                          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 공유 인터럽트 핸들러

```
┌─────────────────────────────────────────────────────────────────┐
│           공유 인터럽트 (Shared Interrupt)                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   개념: 여러 장치가 동일한 IRQ 라인 공유                  │ │
│   │                                                         │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   IRQ 11 (공유)                                 │    │ │
│   │   │        │                                        │    │ │
│   │   │        ▼                                        │    │ │
│   │   │   ┌─────────────────────────────────────┐    │    │ │
│   │   │   │        핸들러 체인 (Action List)     │    │    │ │
│   │   │   │  ┌─────────┐  ┌─────────┐  ┌────────┐│    │    │ │
│   │   │   │  │USB      │─▶│ Sound   │─▶│ NIC    ││    │    │ │
│   │   │   │  │Handler  │  │ Handler │  │Handler ││    │    │ │
│   │   │   │  └─────────┘  └─────────┘  └────────┘│    │    │ │
│   │   │   └─────────────────────────────────────┘    │    │ │
│   │   │                                                 │    │ │
│   │   │   💡 각 핸들러가 자신의 장치 인터럽트인지 확인     │    │ │
│   │   │      (ISR status 레지스터 확인)                  │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   공유 핸들러 작성:                                       │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   irqreturn_t shared_handler(int irq,          │    │ │
│   │   │                            void *dev_id) {      │    │ │
│   │   │       struct my_dev *dev = dev_id;             │    │ │
│   │   │                                                 │    │ │
│   │   │       // 자신의 장치 인터럽트인지 확인            │    │ │
│   │   │       if (!(read_status(dev) & INT_PENDING))   │    │ │
│   │   │           return IRQ_NONE;  // 다른 장치임      │    │ │
│   │   │                                                 │    │ │
│   │   │       // 인터럽트 처리                          │    │ │
│   │   │       handle_interrupt(dev);                   │    │ │
│   │   │                                                 │    │ │
│   │   │       return IRQ_HANDLED;                      │    │ │
│   │   │   }                                            │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **인터럽트 핸들러**: 특정 인터럽트 이벤트 처리 함수
> **Shared Interrupt**: 여러 장치가 공유하는 IRQ
> **Threaded IRQ**: 스레드 컨텍스트에서 수행하는 인터럽트 처리

### 2. 빈출 키워드

- **request_irq()**: 핸들러 등록 함수
- **IRQ_HANDLED/IRQ_NONE**: 핸들러 반환값
- **Shared IRQ Handler**: 공유 인터럽트 핸들러
- **spin_lock_irqsave**: 인터럽트 동기화

### 3. 예상 문제

**Q1. 인터럽트 핸들러 등록 과정과 request_irq() 함수의 매개변수를 설명하시오.**

**Q2. 공유 인터럽트(Shared Interrupt)의 개념과 핸들러 작성 시 주의사항을 기술하시오.**

**Q3. Threaded IRQ의 구조와 기존 인터럽트 처리 방식에 비해 갖는 장점을 설명하시오.**
