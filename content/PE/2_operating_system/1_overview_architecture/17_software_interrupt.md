# 소프트웨어 인터럽트 / 트랩 / 예외 (Software Interrupt / Trap / Exception)

## 정의

**소프트웨어 인터럽트(Software Interrupt)**는 프로그램이 **명시적으로 발생시키는 인터럽트**다. 시스템 호출 구현에 주로 사용된다. **트랩(Trap)**은 특정 조건 발생 시 자동으로 생성되는 소프트웨어 인터럽트이며, **예외(Exception)**는 프로그램 실행 중 오류나 특수 조건에 의해 발생하는 동기적 인터럽트다.

## 핵심 개념

### 1. 소프트웨어 인터럽트 (INT)

```
┌─────────────────────────────────────────────────────────────────┐
│           소프트웨어 인터럽트                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   특징:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   • 프로그램이 명시적으로 발생 (INT n 명령)               │ │
│   │   • 동기적 발생 (명령 실행 순서에 따라)                   │ │
│   │   • 시스템 호출 구현에 사용                               │ │
│   │   • 사용자 모드 → 커널 모드 전환                         │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   x86 INT 명령:                                                  │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   INT n    ; n = 0~255 인터럽트 벡터 번호                │ │
│   │                                                         │ │
│   │   예시:                                                  │ │
│   │   INT 0x80  ; Linux 시스템 호출 (32비트)                 │ │
│   │   INT 0x21  ; DOS 시스템 호출                            │ │
│   │   INT 3    ; 디버깅 브레이크포인트                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   처리 과정:                                                     │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   1. 프로그램: INT n 실행                               │ │
│   │   2. CPU: 플래그, CS, IP을 스택에 저장                   │ │
│   │   3. CPU: IF(인터럽트 플래그) 클리어                    │ │
│   │   4. CPU: IDT[n]에서 핸들러 주소 로드                   │ │
│   │   5. 핸들러: 해당 서비스 수행                           │ │
│   │   6. IRET: 저장된 상태 복원, 프로그램 계속               │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 트랩 (Trap) / 예외 (Exception) 종류

```
┌─────────────────────────────────────────────────────────────────┐
│           예외(Exception) 종류                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① Fault (결함):                                        │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 복구 가능한 오류                              │    │ │
│   │   │ • 문제를 수정하고 해당 명령 재실행              │    │ │
│   │   │ • 반환 주소: 문제 발생 명령                      │    │ │
│   │   │                                                 │    │ │
│   │   │ 예:                                             │    │ │
│   │   │ • Page Fault (페이지 부재)                      │    │ │
│   │   │ • Segment Fault (세그먼트 위반)                 │    │ │
│   │   │ • General Protection Fault                      │    │ │
│   │   │ • Divide Error (0으로 나누기)                   │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ② Trap (트랩):                                          │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 의도적인 예외                                  │    │ │
│   │   │ • 다음 명령부터 계속 실행                        │    │ │
│   │   │ • 반환 주소: 다음 명령                           │    │ │
│   │   │                                                 │    │ │
│   │   │ 예:                                             │    │ │
│   │   │ • Breakpoint (INT 3)                           │    │ │
│   │   │ • Overflow                                      │    │ │
│   │   │ • System Call (INT 0x80)                       │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ③ Abort (중단):                                         │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 심각한 하드웨어 오류                           │    │ │
│   │   │ • 복구 불가능                                    │    │ │
│   │   │ • 프로세스 종료 필요                             │    │ │
│   │   │ • 반환 주소: 의미 없음                           │    │ │
│   │   │                                                 │    │ │
│   │   │ 예:                                             │    │ │
│   │   │ • Double Fault                                  │    │ │
│   │   │ • Machine Check                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

│   ├──────┼───────────────────────┼────────┼────────────────┤  │
│   │  0   │ #DE Divide Error      │ Fault  │ 0으로 나누기   │  │
│   │  1   │ #DB Debug             │ Fault/ │ 디버깅         │  │
│   │      │                       │ Trap   │                │  │

│   │ 14   │ #PF Page Fault        │ Fault  │ 페이지 부재    │  │
│   │ 16   │ #MF x87 FPU Error     │ Fault  │ FPU 오류       │  │
│   │ 17   │ #AC Alignment Check   │ Fault  │ 정렬 오류      │  │

### 4. 예외 처리 과정

```
┌─────────────────────────────────────────────────────────────────┐
│           페이지 폴트(Page Fault) 처리 예시                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   상황: MOV EAX, [0x00400000]  ; 페이지가 메모리에 없음   │ │
│   │                                                         │ │
│   │   처리 과정:                                             │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 1. CPU가 #PF(Page Fault) 발생                  │    │ │
│   │   │                                                 │    │ │
│   │   │ 2. 하드웨어 자동 저장:                          │    │ │
│   │   │    • SS, ESP, EFLAGS, CS, EIP를 스택에 Push     │    │ │
│   │   │    • CR2에 문제 발생 주소(0x00400000) 저장      │    │ │
│   │   │    • Error Code를 스택에 Push                   │    │ │
│   │   │                                                 │    │ │
│   │   │ 3. Page Fault 핸들러 실행                       │    │ │
│   │   │    • CR2에서 문제 주소 확인                      │    │ │
│   │   │    • 페이지 테이블 확인                          │    │ │
│   │   │    • 디스크에서 페이지 로드                      │    │ │
│   │   │    • 페이지 테이블 갱신                          │    │ │
│   │   │                                                 │    │ │
│   │   │ 4. IRET로 복귀                                  │    │ │
│   │   │    • 저장된 레지스터 복원                        │    │ │
│   │   │    • 문제 발생 명령 재실행 (Fault 특성)          │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   💡 Fault는 문제 해결 후 원래 명령 재실행              │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **소프트웨어 인터럽트**: 명시적 INT 명령으로 발생
> **Fault**: 복구 가능, 문제 명령 재실행
> **Trap**: 의도적, 다음 명령 계속
> **Abort**: 복구 불가, 프로세스 종료

### 2. 빈출 키워드

- **Synchronous Interrupt**: 동기적 인터럽트
- **Fault/Trap/Abort**: 예외의 세 가지 유형
- **Page Fault**: 페이지 부재, 메모리 로드 후 재실행
- **INT n / IRET**: x86 인터럽트 명령어

### 3. 예상 문제

**Q1. 소프트웨어 인터럽트와 하드웨어 인터럽트의 차이점을 설명하시오.**

**Q2. Fault, Trap, Abort의 차이점을 비교하고 각각의 예시를 기술하시오.**

**Q3. 페이지 폴트(Page Fault)의 처리 과정을 단계별로 설명하시오.**
