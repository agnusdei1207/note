# 시스템 호출 (System Call)

## 정의

**시스템 호출(System Call)**은 사용자 프로그램이 **커널의 서비스를 요청**하기 위한 프로그래밍 인터페이스다. 사용자 모드에서 커널 모드로 전환하여 운영체제의 기능(파일 입출력, 메모리 할당, 프로세스 관리 등)을 안전하게 수행한다.

### 2. 시스템 호출 처리 과정

```
┌─────────────────────────────────────────────────────────────────┐
│           시스템 호출 상세 처리 과정                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 사용자 프로그램에서 API 호출                         │ │
│   │      count = read(fd, buffer, size);                    │ │
│   │                                                         │ │
│   │   ② 라이브러리 함수가 시스템 호출 번호 설정               │ │
│   │      %eax = __NR_read  (시스템 호출 번호)               │ │
│   │      %ebx = fd                                          │ │
│   │      %ecx = buffer                                      │ │
│   │      %edx = size                                        │ │
│   │                                                         │ │
│   │   ③ 소프트웨어 인터럽트 발생 (Trap)                      │ │
│   │      int 0x80 (x86)  또는  syscall (x86-64)            │ │
│   │                                                         │ │
│   │   ④ CPU가 커널 모드로 전환                              │ │
│   │      - 모드 비트: 1 → 0 (커널 모드)                     │ │
│   │      - 사용자 스택 → 커널 스택                          │ │
│   │      - 레지스터 저장                                    │ │
│   │                                                         │ │
│   │   ⑤ 시스템 호출 핸들러 진입                              │ │
│   │      system_call()                                      │ │
│   │      - 시스템 호출 번호로 디스패치                       │ │
│   │      - sys_read() 함수 호출                             │ │
│   │                                                         │ │
│   │   ⑥ 커널 서비스 수행                                     │ │
│   │      - 파일 시스템 접근                                  │ │
│   │      - 디바이스 드라이버 호출                            │ │
│   │      - 데이터 버퍼에 복사                                │ │
│   │                                                         │ │
│   │   ⑦ 결과 반환 및 사용자 모드 복귀                       │ │
│   │      - 반환값 레지스터 설정                              │ │
│   │      - iret 명령으로 복귀                               │ │
│   │      - 사용자 프로그램 계속 실행                         │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 시스템 호출 vs 함수 호출

```
┌─────────────────────────────────────────────────────────────────┐
│           시스템 호출 vs 일반 함수 호출                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분        │ 일반 함수 호출      │ 시스템 호출         │ │
│   ├──────────────┼─────────────────────┼────────────────────┤ │
│   │ 실행 모드    │ 사용자 모드 유지    │ 커널 모드 전환      │ │
│   │ 메모리 공간  │ 같은 주소 공간      │ 커널 주소 공간      │ │
│   │ 호출 방식    │ CALL 명령           │ INT/SYSCALL        │ │
│   │ 오버헤드     │ 낮음                │ 높음               │ │
│   │ 권한         │ 제한 없음           │ 커널 권한 필요      │ │
│   │ 안전성       │ 사용자 책임         │ 커널 보장           │ │
│   └──────────────┴─────────────────────┴────────────────────┘ │
│                                                                 │
│   모드 전환 오버헤드:                                            │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   사용자 → 커널:                                        │ │
│   │   • 레지스터 저장/복원                                   │ │
│   │   • TLB 플러시 가능                                     │ │
│   │   • 캐시 오염                                           │ │
│   │   • 수백~수천 사이클 소요                                │ │
│   │                                                         │ │
│   │   💡 시스템 호출 최소화가 성능 향상의 핵심                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **시스템 호출**: 사용자 프로그램이 커널 서비스를 요청하는 인터페이스
> **Trap**: 사용자 모드에서 커널 모드로 전환하는 메커니즘
> **시스템 호출 번호**: 각 시스템 호출의 고유 식별자

### 2. 빈출 키워드

- **User Mode → Kernel Mode**: 시스템 호출 시 모드 전환
- **Software Interrupt**: INT 0x80, syscall 명령
- **System Call Handler**: 커널 내 시스템 호출 처리 루틴
- **Overhead**: 시스템 호출의 비용

### 3. 예상 문제

**Q1. 시스템 호출의 개념과 처리 과정을 단계별로 설명하시오.**

**Q2. 시스템 호출을 5가지 카테고리로 분류하고 각각의 예시를 들어 설명하시오.**

**Q3. 시스템 호출과 일반 함수 호출의 차이점과 시스템 호출 오버헤드를 줄이는 방안을 기술하시오.**
