# 인터럽트 서비스 루틴 (ISR)

## 정의

**인터럽트 서비스 루틴(ISR, Interrupt Service Routine)**은 인터럽트 발생 시 **실행되는 처리 함수**다. 하드웨어나 소프트웨어 인터럽트에 대한 구체적인 처리 로직을 구현하며, 인터럽트 핸들러라고도 한다. ISR은 가능한 짧고 빠르게 실행되어야 한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           ISR 기본 개념                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   인터럽트 처리 흐름:                                     │ │
│   │                                                         │ │
│   │   정상 실행         인터럽트            ISR 실행          │ │
│   │   ┌─────────┐                        ┌─────────────┐   │ │
│   │   │ 명령어  │     인터럽트 요청       │ 컨텍스트    │   │ │
│   │   │ 실행    │───────────▶           │ 저장        │   │ │
│   │   │         │                        └──────┬──────┘   │ │
│   │   │         │                               │          │ │
│   │   │         │                        ┌──────▼──────┐   │ │
│   │   │         │                        │ ISR 실행    │   │ │
│   │   │         │                        │ (이벤트 처리)│   │ │
│   │   │         │                        └──────┬──────┘   │ │
│   │   │         │                               │          │ │
│   │   │         │                        ┌──────▼──────┐   │ │
│   │   │  ◀──────┴──────── 복귀 ─────────│ 컨텍스트    │   │ │
│   │   │                                 │ 복원        │   │ │
│   │   │                                 └─────────────┘   │ │
│   │   │                                                   │ │
│   │   └─────────┘                        └─────────────┘   │ │
│   │                                                         │ │
│   │   💡 ISR은 인터럽트된 프로그램에 영향 없이 실행되어야 함    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. ISR 구조와 작성 원칙

```
┌─────────────────────────────────────────────────────────────────┐
│           ISR 작성 원칙                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   핵심 원칙:                                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   ① 신속성 (Speed):                            │    │ │
│   │   │      • 최대한 짧게 실행                         │    │ │
│   │   │      • 긴 작업은 지연 처리(Deferred)           │    │ │
│   │   │                                                 │    │ │
│   │   │   ② 원자성 (Atomicity):                        │    │ │
│   │   │      • 컨텍스트 저장/복원                       │    │ │
│   │   │      • 레지스터 보존                            │    │ │
│   │   │                                                 │    │ │
│   │   │   ③ 비동기성 (Asynchronous):                   │    │ │
│   │   │      • 언제 발생할지 모름                       │    │ │
│   │   │      • 재진입 가능해야 함                       │    │ │
│   │   │                                                 │    │ │
│   │   │   ④ 최소화 (Minimal):                          │    │ │
│   │   │      • 최소한의 작업만 수행                     │    │ │
│   │   │      • 블로킹 호출 금지                         │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   금지 사항:                                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   ✗ sleep() / delay() 호출                     │    │ │
│   │   │   ✗ 메모리 할당 (kmalloc 등)                   │    │ │
│   │   │   ✗ I/O 대기                                   │    │ │
│   │   │   ✗ 세마포어 대기 (blocking)                   │    │ │
│   │   │   ✗ 사용자 공간 접근                           │    │ │
│   │   │   ✗ 긴 루프 실행                               │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. ISR 실행 과정

```
┌─────────────────────────────────────────────────────────────────┐
│           ISR 상세 실행 과정                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   하드웨어 자동 수행 (CPU):                               │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   1. 인터럽트 신호 감지                         │    │ │
│   │   │      • IRQ 라인 활성화 확인                     │    │ │
│   │   │                                                 │    │ │
│   │   │   2. 현재 명령어 완료                           │    │ │
│   │   │      • 실행 중인 명령 완료 후 응답               │    │ │
│   │   │                                                 │    │ │
│   │   │   3. 컨텍스트 저장 (하드웨어):                   │    │ │
│   │   │      ┌───────────────────────────────────┐    │    │ │
│   │   │      │ 스택에 Push:                       │    │    │ │
│   │   │      │ • EFLAGS (플래그 레지스터)         │    │    │ │
│   │   │      │ • CS (코드 세그먼트)              │    │    │ │
│   │   │      │ • EIP (명령어 포인터)             │    │    │ │
│   │   │      │ • Error Code (일부 예외)          │    │    │ │
│   │   │      └───────────────────────────────────┘    │    │ │
│   │   │                                                 │    │ │
│   │   │   4. 인터럽트 비활성화                         │    │ │
│   │   │      • IF 플래그 클리어 (Interrupt Gate)       │    │ │
│   │   │                                                 │    │ │
│   │   │   5. ISR 주소 로드                             │    │ │
│   │   │      • IDT에서 핸들러 주소 획득                 │    │ │
│   │   │      • CS:EIP 설정                             │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   소프트웨어 수행 (ISR):                                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   ISR_Entry:                                    │    │ │
│   │   │     pusha           ; 범용 레지스터 저장        │    │ │
│   │   │     push ds         ; 세그먼트 레지스터 저장    │    │ │
│   │   │     push es                                    │    │ │
│   │   │     ...                                        │    │ │
│   │   │     call ISR_Handler   ; 실제 핸들러 호출       │    │ │
│   │   │     ...                                        │    │ │
│   │   │     pop es          ; 세그먼트 레지스터 복원    │    │ │
│   │   │     pop ds                                    │    │ │
│   │   │     popa            ; 범용 레지스터 복원        │    │ │
│   │   │     iret            ; 인터럽트 복귀             │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. Top Half vs Bottom Half

```
┌─────────────────────────────────────────────────────────────────┐
│           Linux 인터럽트 처리 계층                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   인터럽트 처리를 두 단계로 분리:                          │ │
│   │                                                         │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   Top Half (상반부):                            │    │ │
│   │   │   ┌───────────────────────────────────────┐   │    │ │
│   │   │   │ • ISR 자체 (즉시 실행)                  │   │    │ │
│   │   │   │ • 인터럽트 비활성화 상태로 실행          │   │    │ │
│   │   │   │ • 하드웨어 처리, 상태 저장              │   │    │ │
│   │   │   │ • 최소한의 작업만 수행                  │   │    │ │
│   │   │   │ • Bottom Half 스케줄링                  │   │    │ │
│   │   │   │                                         │   │    │ │
│   │   │   │ 예: NIC 인터럽트                        │   │    │ │
│   │   │   │   • 패킷 도착 확인                      │   │    │ │
│   │   │   │   • 패킷을 커널 버퍼로 복사             │   │    │ │
│   │   │   │   • SoftIRQ 스케줄링                   │   │    │ │
│   │   │   │   • 즉시 복귀                          │   │    │ │
│   │   │   └───────────────────────────────────────┘   │    │ │
│   │   │                      │                        │    │ │
│   │   │                      ▼                        │    │ │
│   │   │   Bottom Half (하반부):                        │    │ │
│   │   │   ┌───────────────────────────────────────┐   │    │ │
│   │   │   │ • 지연된 작업 처리                      │   │    │ │
│   │   │   │ • 인터럽트 활성화 상태로 실행            │   │    │ │
│   │   │   │ • 더 오래 걸리는 작업 가능              │   │    │ │
│   │   │   │                                         │   │    │ │
│   │   │   │ 구현 방식:                              │   │    │ │
│   │   │   │ • SoftIRQ (가장 우선순위 높음)          │   │    │ │
│   │   │   │ • Tasklet (SoftIRQ 기반)               │   │    │ │
│   │   │   │ • Workqueue (프로세스 컨텍스트)         │   │    │ │
│   │   │   │ • Threaded IRQ (커널 스레드)           │   │    │ │
│   │   │   │                                         │   │    │ │
│   │   │   │ 예: 패킷 처리                           │   │    │ │
│   │   │   │   • 프로토콜 스택 처리                  │   │    │ │
│   │   │   │   • 데이터 복사                        │   │    │ │
│   │   │   │   • 소켓 버퍼 처리                     │   │    │ │
│   │   │   └───────────────────────────────────────┘   │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   Bottom Half 비교:                                             │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   메커니즘    │ 컨텍스트    │ 선점 가능 │ 병렬 실행      │ │
│   │   ────────────┼────────────┼──────────┼─────────────    │ │
│   │   SoftIRQ     │ 인터럽트    │ X        │ O (다중 CPU)    │ │
│   │   Tasklet     │ 인터럽트    │ X        │ X (동일 유형)   │ │
│   │   Workqueue   │ 프로세스    │ O        │ O              │ │
│   │   Threaded    │ 스레드      │ O        │ O              │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **ISR**: 인터럽트 발생 시 실행되는 처리 함수
> **Top Half**: 즉시 실행되는 필수 작업 (ISR 본체)
> **Bottom Half**: 지연 처리되는 나머지 작업

### 2. 빈출 키워드

- **Context Save/Restore**: 컨텍스트 저장/복원
- **Top Half/Bottom Half**: 빠른 처리와 지연 처리 분리
- **SoftIRQ/Tasklet/Workqueue**: Linux 지연 처리 메커니즘
- **Nested Interrupt**: 인터럽트 중첩, 우선순위 기반 선점

### 3. 예상 문제

**Q1. ISR 작성 시 준수해야 할 원칙과 금지 사항을 기술하시오.**

**Q2. Linux의 Top Half와 Bottom Half 개념을 설명하고 각각의 역할을 기술하시오.**

**Q3. 인터럽트 중첩(Nested Interrupt)이 가능한 조건과 처리 과정을 설명하시오.**
