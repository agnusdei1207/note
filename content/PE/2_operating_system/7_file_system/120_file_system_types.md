# 파일 시스템 유형 (File System Types)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 시스템은 **"파일**과 **디렉터리**를 **저장**하고 **관리**하는 **구조\\\"**로, **Ext4**(Extended **File **System **4)**, **XFS**(XFS **File **System)**, **Btrfs**(B-**Tree **File **System)**, **ZFS**(Zettabyte **File **System)**가 **대표적**이다.
> 2. **구조**: **Inode**(아이 **노드)**가 **파일 **메타데이터**를 **저장**하고 **Data **Block**(데이터 **블록)**이 **실제 **내용**을 **저장**하며 **Journaling**(저널링)**이 **파일 **시스템 **복구**를 **빠르게 **한다.
> 3. **특징**: **Ext4**는 **안정적**이고 **XFS**는 **대용량**에 **최적화**되었으며 **Btrfs/ZFS**는 **Snapshot**, **Compression**, **Deduplication**을 **지원**한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
파일 시스템은 **"디스크 상의 파일 구조"**이다.

**파일 시스템 비교**:
| 파일 시스템 | 최대 크기 | 최대 파일 | 저널링 | 사용처 |
|-------------|-----------|-----------|--------|--------|
| **Ext4** | 1 EB | 16 TB | O | 일반 |
| **XFS** | 16 EB | 8 EB | O | 서버 |
| **Btrfs** | 16 EB | 16 EB | O | 데스크톱 |
| **ZFS** | 256 ZB | 16 EB | O/C | 스토리지 |

### 💡 비유
파일 시스템은 ****책장 **책 ****철 ****과 같다.
- **Inode**: 색인
- **Data Block**: 내용
- **Journal**: 변경 로그

## Ⅱ. 아키텍처 및 핵심 원리

### Ext4 (Fourth Extended Filesystem)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Ext4 Structure                                                      │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Disk Layout:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Boot Block      │  Superblock  │  Group Descriptors  │  Data Block Bitmaps             │  │
    │  (1 KB)          │  (1 KB)      │  (many)            │  Inode Bitmaps                 │  │
    │  ────────────────┴───────────────┴────────────────────┴──────────────────────────────────  │  │
    │  Inode Table     │  Data Blocks                                                          │  │
    │  (many)          │  (user data)                                                         │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
    │  → Partition divided into block groups (each ~8GB default)                               │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Block Group:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Block Group (8GB)                                                                      │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Superblock backup (copy of main superblock)                                         │  │  │
    │  │  Group descriptors (block group metadata)                                            │  │  │
    │  │  Data block bitmap (track used/free data blocks)                                     │  │  │
    │  │  Inode bitmap (track used/free inodes)                                               │  │  │
    │  │  Inode table (inodes for this group)                                                 │  │  │
    │  │  Data blocks (actual file data)                                                      │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Inode Structure:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  struct ext4_inode {                                                                     │  │
    │      __le16  i_mode;        // File type and permissions                                  │  │
    │      __le16  i_uid;         // Owner UID                                                │  │
    │      __le32  i_size;        // File size in bytes                                       │  │
    │      __le32  i_atime;       // Access time                                              │  │
    │      __le32  i_ctime;       // Change time                                              │  │
    │      __le32  i_mtime;       // Modification time                                         │  │
    │      __le32  i_dtime;       // Deletion time                                            │  │
    │      __le16  i_gid;         // Group GID                                                │  │
    │      __le16  i_links_count; // Hard link count                                          │  │
    │      __le32  i_blocks;      // Number of 512-byte blocks                                 │  │
    │      __le32  i_flags;       // File flags                                               │  │
    │      __le32  i_block[15];   // Pointers to data blocks                                   │  │
    │      // ...                                                                             │  │
    │  };                                                                                      │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Inode Block Pointers

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Ext4 Inode Block Pointing                                           │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Direct/Indirect Pointers:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  i_block[0]     →  Direct block 0        (4 KB data)                                   │  │
    │  i_block[1]     →  Direct block 1        (4 KB data)                                   │  │
    │  i_block[2]     →  Direct block 2        (4 KB data)                                   │  │
    │  ...                                                                                     │  │
    │  i_block[11]    →  Direct block 11       (4 KB data)                                   │  │
    │  ──────────────────────────────────────────────────────────────────────────────────────  │  │
    │  i_block[12]    →  Singly Indirect Block                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  [Block 0]  →  Data Block 0                                                          │  │  │  │  │
    │  │  [Block 1]  →  Data Block 1                                                          │  │  │  │  │
    │  │  ... (1024 pointers)                                                                 │  │  │  │  │
    │  │  → Max file size: 12 direct + 1024 indirect = 1036 blocks (4 MB)                     │  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  ──────────────────────────────────────────────────────────────────────────────────────  │  │
    │  i_block[13]    →  Doubly Indirect Block                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  [Indirect 0]  →  [Block 0..1023]  →  Data Blocks                                   │  │  │  │  │
    │  │  [Indirect 1]  →  [Block 1024..2047]  →  Data Blocks                                │  │  │  │  │
    │  │  ... (1024 indirect pointers)                                                       │  │  │  │  │
    │  │  → Max file size: ~4 GB                                                               │  │  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  ──────────────────────────────────────────────────────────────────────────────────────  │  │
    │  i_block[14]    →  Treely Indirect Block                                                 │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  [Doubly Indirect 0]  →  [Indirect 0..1023]  →  [Blocks]  →  Data                  │  │  │  │  │
    │  │  → Max file size: ~16 TB                                                             │  │  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
    │  → Extents (modern approach): Store contiguous block ranges instead of indirect pointers  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### XFS (XFS File System)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         XFS Structure (Allocation Groups)                                  │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Disk Layout:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Allocation Groups (AGs)                                                                │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  AG 0           AG 1           AG 2           AG 3                                 │  │  │
    │  │  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐                            │  │  │
    │  │  │Superblock│  │ AGF+AGI  │  │ Free     │  │ Free     │                            │  │  │
    │  │  │+ AGF+AGI │  │          │  │ Space    │  │ Space    │                            │  │  │
    │  │  └──────────┘  └──────────┘  └──────────┘  └──────────┘                            │  │  │
    │  │  (metadata)    (metadata)    (data)        (data)                                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  → Each AG is independent (can allocate/free blocks in parallel)                            │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    B+Tree Indexing:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Inode B+Tree (root → internal → leaf)                                                    │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Root Node                                                                             │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  [Inode 1000-1999]  →  Internal Node                                             │  │  │  │  │
    │  │  │  [Inode 2000-2999]  →  Internal Node                                             │  │  │  │  │
    │  │  │  [Inode 3000-3999]  →  Internal Node                                             │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                  │                                                     │  │  │
    │  │                                  ▼                                                     │  │  │
    │  │                          Internal Node                                                │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  [Inode 1000-1099]  →  Leaf Node                                                 │  │  │  │  │
    │  │  │  [Inode 1100-1199]  →  Leaf Node                                                 │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │                                  │                                                     │  │  │
    │  │                                  ▼                                                     │  │  │
    │  │                          Leaf Node                                                   │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  Inode 1000: /etc/passwd                                                           │  │  │  │  │
    │  │  │  Inode 1001: /etc/shadow                                                           │  │  │  │  │
    │  │  │  Inode 1002: /etc/hosts                                                            │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  → Fast lookup: O(log n) instead of linear scan                                           │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### ZFS (Zettabyte File System)

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         ZFS Structure (Copy-on-Write Transactional Model)                     │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Pool → VDEV → Disk:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  ZFS Pool: tank                                                                          │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  VDEV 0 (Mirror)                     VDEV 1 (RAIDZ)                                  │  │  │
    │  │  ┌──────────────────────────────────────────────────────────────────────────────────┐  │  │  │  │
    │  │  │  Disk A  │  Disk B  │          │  Disk C  │  Disk D  │  Disk E  │                 │  │  │  │  │
    │  │  │  (data)  │  (mirror)│          │  (data)  │  (parity)│  (data)  │                 │  │  │  │  │
    │  │  └──────────────────────────────────────────────────────────────────────────────────┘  │  │  │
    │  │  → VDEV types: mirror, RAIDZ1/2/3, draid                                             │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  → Dataset: zvol, filesystem, snapshot                                                  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Copy-on-Write (CoW):
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Original File:                                                                         │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Block A  Block B  Block C  Block D                                                  │  │  │
    │  │  (data)   (data)   (data)   (data)                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │            │ Modify Block B                                                              │  │
    │            ▼                                                                             │  │
    │  After Modification:                                                                    │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  Block A  Block B  Block C  Block D  Block E' (new)                                  │  │  │
    │  │  (old)    (old)    (old)    (old)    (modified)                                    │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │  → Old blocks kept until all snapshots freed                                            │  │
    │  → Never overwrites live data (corruption-resistant)                                     │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

### Journaling

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Journaling (Metadata Consistency)                                   │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Ordered Mode Journaling:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  Write Operation Steps:                                                                 │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  1. Write metadata to journal (transaction begins)                                   │  │  │
    │  │  2. Write data blocks to disk (ordered: data before metadata commit)                 │  │  │
    │  │  3. Commit journal (mark transaction complete)                                       │  │  │
    │  │  4. Write metadata to final location                                                 │  │  │
    │  │  5. Checkpoint journal (mark journal space free)                                     │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  Crash Recovery:                                                                         │  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  On mount, check journal:                                                              │  │  │
    │  │  • If transaction complete (committed):                                               │  │  │
    │  │      → Replay metadata (write to final location)                                      │  │  │
    │  │  • If transaction incomplete (not committed):                                         │  │  │
    │  │      → Discard transaction (no corruption)                                            │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 기술사적 판단 (실무 시나리오)

#### 시나리오: 서버 파일 시스템 선택
**상황**: 10TB 데이터베이스 서버
**판단**: XFS + AIO

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                         Database Server Filesystem Configuration                             │
└─────────────────────────────────────────────────────────────────────────────────────────┘

    Requirements:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  • 10TB database files                                                                   │  │
    │  • High sequential I/O (scans, backups)                                                  │  │
    │  • Data integrity (checksums)                                                           │  │
    │  • Fast recovery (crash consistency)                                                     │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Solution: XFS with AIO
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  # Create XFS filesystem with large inode size (for extended attributes)                   │  │
    │  mkfs.xfs -f -i size=512 -l size=128m,lazy-count=1 /dev/sdb1                             │  │
    │                                                                                         │  │
    │  # Mount with optimal options                                                             │  │
    │  mount -o noatime,nodiratime,allocsize=4M,attr2,largeio /dev/sdb1 /data                   │  │
    │                                                                                         │  │
    │  /etc/fstab entry:                                                                       │  │
    │  /dev/sdb1  /data  xfs  defaults,noatime,nodiratime,allocsize=4M  0 0                    │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘

    Benefits:
    ┌──────────────────────────────────────────────────────────────────────────────────────────┐
    │  1. Large File Performance:                                                               │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • Extent-based allocation (reduces fragmentation)                                    │  │  │
    │  │  • Allocation groups (parallel allocation)                                            │  │  │
    │  │  • Large I/O (up to 1MB per request)                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  2. Metadata Performance:                                                                │  │  │
    │  ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • B+Tree indexing (fast inode lookup)                                               │  │  │
    │  │  • Delayed allocation (group writes)                                                  │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    │                                                                                         │  │
    │  3. Reliability:                                                                         │  │  │
    ┌──────────────────────────────────────────────────────────────────────────────────────┐  │  │
    │  │  • Journaling (metadata consistency)                                                  │  │  │
    │  │  • Checksums (metadata only, data via external)                                      │  │  │
    │  │  • Online resize/defragment                                                           │  │  │
    │  └──────────────────────────────────────────────────────────────────────────────────────┘  │  │
    └──────────────────────────────────────────────────────────────────────────────────────────┘
```
