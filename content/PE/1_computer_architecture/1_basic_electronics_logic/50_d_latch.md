# D 래치 (D Latch)

## 정의

**D 래치(Data Latch)**는 단일 데이터 입력(D)과 Enable 신호로 구성된 래치다. SR 래치의 **금지 상태(S=R=1)를 제거**한 구조로, Enable이 활성화될 때 D 입력이 Q 출력으로 전달된다. 투명 래치(Transparent Latch)라고도 한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           D 래치 기본 구조                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   D ─────┐                                              │ │
│   │          │                                              │ │
│   │          │    ┌─────────────────┐                       │ │
│   │          └───▶│                 │                       │ │
│   │               │     D 래치      │───▶ Q                 │ │
│   │   EN ────────▶│   (Transparent) │                       │ │
│   │               │                 │                       │ │
│   │               └─────────────────┘                       │ │
│   │                        │                                │ │
│   │                        └──▶ Q' (반전 출력)              │ │
│   │                                                         │ │
│   │   특징:                                                  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 단일 데이터 입력 D                             │    │ │
│   │   │ • 금지 상태 없음                                  │    │ │
│   │   │ • EN=1: Q=D (투명 모드)                          │    │ │
│   │   │ • EN=0: Q 유지 (래치 모드)                        │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. D 래치 특성 테이블

```
┌─────────────────────────────────────────────────────────────────┐
│           D 래치 특성 테이블                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   EN │  D  │  Q(t+1)  │  동작                          │ │
│   │   ───┼─────┼──────────┼────────────────────────         │ │
│   │    0 │  X  │  Q(t)    │  상태 유지 (Hold)              │ │
│   │    1 │  0  │  0       │  Reset                         │ │
│   │    1 │  1  │  1       │  Set                           │ │
│   │                                                         │ │
│   │   💡 X = Don't Care (EN=0일 때 D 무시)                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   논리식:                                                       │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Q(t+1) = EN·D + EN'·Q(t)                             │ │
│   │                                                         │ │
│   │   EN=1: Q = D      (투명 모드)                          │ │
│   │   EN=0: Q = Q(t)   (래치 모드)                          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   타이밍 다이어그램:                                             │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   EN  ────┐     ┌─────┐     ┌─────┐                    │ │
│   │            └─────┘     └─────┘     └─────               │ │
│   │            │←1─→│     │←1─→│                            │ │
│   │            │투명│     │투명│                             │ │
│   │                                                         │ │
│   │   D   ────┬─────┬─────┬─────┬────┐                     │ │
│   │            │     │     │     │    │                     │ │
│   │          D1│   D2│   D3│   D4│ D5 │                     │ │
│   │                                                         │ │
│   │   Q   ────┬─────┬─────┬─────┐                          │ │
│   │            │     │     │     │                          │ │
│   │          D1│   D2│   D3│ D4 ┴── D4 유지                 │ │
│   │            │           │                                │ │
│   │         EN=1 구간    EN=0                               │ │
│   │         D 따름      상태 유지                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. D 래치 회로 구현

```
┌─────────────────────────────────────────────────────────────────┐
│           D 래치 회로 구현                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   SR 래치 기반 구현:                                             │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   D ───┬──▶ [NOT] ──┬──▶ R                             │ │
│   │        │             │                                  │ │
│   │        │             └──▶ [SR 래치] ──▶ Q               │ │
│   │        │                     ▲                          │ │
│   │        └─────────────────────┘                          │ │
│   │                      S                                   │ │
│   │                                                         │ │
│   │   💡 D=1 → S=1, R=0 (Set)                              │ │
│   │      D=0 → S=0, R=1 (Reset)                            │ │
│   │      S=R=1 불가능 → 금지 상태 제거                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   게이트드 D 래치 (Gated D Latch):                               │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   D ───┬──▶ [AND] ───┐                                  │ │
│   │        │      ↑       │                                  │ │
│   │        │      EN      │                                  │ │
│   │        │              │                                  │ │
│   │        │         ┌────┴────┐                            │ │
│   │        │         │  SR     │                            │ │
│   │        │         │  래치   │────▶ Q                     │ │
│   │        │         │         │                            │ │
│   │        └──[NOT]──│         │                            │ │
│   │                  │         │                            │ │
│   │   EN ───[AND]────┘         │                            │ │
│   │        ↑                   │                            │ │
│   │        D                   │                            │ │
│   │                            │                            │ │
│   │   논리: EN=1일 때 S=D, R=D'                              │ │
│   │        EN=0일 때 S=R=0 (Hold)                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   멀티플렉서 기반 구현:                                          │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   D ──────────────────▶ [2:1 MUX] ──▶ Q                │ │
│   │                              ▲                          │ │
│   │   Q (피드백) ───────────────┘                           │ │
│   │                              │                          │ │
│   │   EN ────────────────────(Select)                       │ │
│   │                                                         │ │
│   │   EN=1: D 선택 → Q=D                                   │ │
│   │   EN=0: Q 선택 → Q=Q (유지)                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 투명성과 타이밍

```
┌─────────────────────────────────────────────────────────────────┐
│           D 래치 투명성과 타이밍 제약                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   투명성 (Transparency):                                        │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   EN=1 구간에서 D가 변하면 Q도 즉시 변화                  │ │
│   │                                                         │ │
│   │   EN  ────┐     ┌─────┐                                 │ │
│   │            └─────┘     └───                             │ │
│   │            │←── EN=1 ─→│                                │ │
│   │                                                         │ │
│   │   D   ────┬───┬───┬───┬───┐                           │ │
│   │            │   │   │   │   │                           │ │
│   │        D1 ┘ D2┘ D3┘ D4┘ D5                             │ │
│   │                                                         │ │
│   │   Q   ────┬───┬───┬───┬───┐                           │ │
│   │            │   │   │   │   │                           │ │
│   │        D1 ┘ D2┘ D3┘ D4┘ D5                             │ │
│   │            │              │                              │ │
│   │         모든 D 변화가 Q에 반영                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   타이밍 파라미터:                                               │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Setup Time (tsu):                                     │ │
│   │   • EN이 1→0이 되기 전 D가 안정되어야 하는 시간          │ │
│   │   • EN 하강 에지 전 D 확정 필요                         │ │
│   │                                                         │ │
│   │   Hold Time (th):                                        │ │
│   │   • EN이 1→0이 된 후 D가 유지되어야 하는 시간            │ │
│   │   • 일반적으로 매우 짧음                                 │ │
│   │                                                         │ │
│   │   Propagation Delay (tpd):                               │ │
│   │   • D 또는 EN 변화 후 Q가 변하는 시간                    │ │
│   │                                                         │ │
│   │         ┌─── tsu ───┐                                   │ │
│   │   D ───┬────────────┴──────                             │ │
│   │        │                                                 │ │
│   │   EN ──┘                                                 │ │
│   │        ↑                                                 │ │
│   │      샘플링                                              │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **D 래치**: 단일 데이터 입력의 투명 래치
> **투명 모드**: EN=1일 때 Q=D (입력이 그대로 출력으로)
> **금지 상태 제거**: SR 래치와 달리 S=R=1 불가능

### 2. 빈출 키워드

- **Transparent Latch**: 투명 래치 (EN=1일 때 투명 동작)
- **Hold Mode**: EN=0일 때 상태 유지
- **Address Latch**: 주소 래칭 (ALE)
- **74LS373**: 8비트 투명 래치 IC

### 3. 예상 문제

**Q1. D 래치의 동작 원리를 설명하고 SR 래치와 비교하시오.**

**Q2. D 래치의 투명성(Transparency)이란 무엇이며 어떤 문제를 야기하는가?**

**Q3. 마이크로프로세서에서 주소 래칭에 D 래치가 사용되는 이유를 설명하시오.**
