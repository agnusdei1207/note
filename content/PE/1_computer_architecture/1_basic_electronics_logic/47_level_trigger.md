# 레벨 트리거 (Level Trigger)

## 정의

**레벨 트리거(Level Trigger)**는 클럭 신호가 **특정 레벨(High 또는 Low)을 유지하는 동안** 지속적으로 입력을 샘플링하는 동기 방식이다. 에지 트리거와 달리 활성 레벨 구간 내에서 입력 변화가 즉시 출력에 반영된다. 주로 래치에서 사용된다.

```
┌─────────────────────────────────────────────────────────────────┐
│           레벨 트리거 동작                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   클럭 신호 (Enable):                                    │ │
│   │                                                         │ │
│   │   전압                                                   │ │
│   │    Vdd ──┐     ┌─────┐     ┌─────┐                     │ │
│   │           │     │     │     │     │                     │ │
│   │    0V  ──┘     └─────┘     └─────┘                     │ │
│   │           │←High→│     │←High→│                          │ │
│   │           │ 구간 │     │ 구간 │                          │ │
│   │           │      │     │     │                           │ │
│   │           │ 투명 │     │ 투명 │  (Transparent Mode)      │ │
│   │           │ 모드 │     │ 모드 │                          │ │
│   │           │      │     │     │                           │ │
│   │   D 변화 반영  D 변화 반영                               │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   High 레벨 트리거: Enable=1일 때 동작                          │
│   Low 레벨 트리거: Enable=0일 때 동작                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. D 래치의 레벨 트리거 동작

```
┌─────────────────────────────────────────────────────────────────┐
│           D 래치 (Level Triggered)                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   구조:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   D ─────┐                                              │ │
│   │          │                                              │ │
│   │          │    ┌─────────────────┐                       │ │
│   │          └───▶│    D 래치       │───▶ Q                 │ │
│   │               │  (High Level)  │                        │ │
│   │   EN ────────▶│   Triggered    │                        │ │
│   │               └─────────────────┘                       │ │
│   │                                                         │ │
│   │   EN = Enable (클럭 역할)                               │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   동작 모드:                                                     │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   EN = 1 (High): 투명 모드 (Transparent)                │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • Q = D (입력이 그대로 출력으로)                │    │ │
│   │   │ • D가 변하면 Q도 즉시 변화                       │    │ │
│   │   │ • 지속적인 샘플링                                │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   EN = 0 (Low): 래치 모드 (Latched)                     │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • Q = 이전 상태 유지                            │    │ │
│   │   │ • D 변화 무시                                   │    │ │
│   │   │ • 메모리 동작                                   │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   타이밍 다이어그램:                                             │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   EN  ────┐     ┌─────┐     ┌─────┐                    │ │
│   │            └─────┘     └─────┘     └─────               │ │
│   │            │←─── T ───→│                             │ │
│   │            │  투명    │  래치                         │ │
│   │                                                         │ │
│   │   D   ────┬─────┬─────┬─────┬────┐                    │ │
│   │            │     │     │     │    │                    │ │
│   │   D1──────┘ D2 ─┘ D3 ─┘ D4 ─┘ D5 ┘                    │ │
│   │                                                         │ │
│   │   Q   ────┬─────┬─────┬─────┐                          │ │
│   │            │     │     │     │                          │ │
│   │   D1──────┘ D2 ─┘ D3 ─┘ D4 ┴── D4 유지                 │ │
│   │            │           │                                │ │
│   │        EN High 구간    EN Low                           │ │
│   │        D 따라 변화     마지막 값 유지                    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 투명성 (Transparency) 문제

```
┌─────────────────────────────────────────────────────────────────┐
│           레벨 트리거의 투명성 문제                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   문제: 활성 구간 내 입력 변화가 모두 출력에 반영                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   EN  ────┐     ┌─────┐                                 │ │
│   │            └─────┘     └───                             │ │
│   │            │←── High ─→│                                │ │
│   │                                                         │ │
│   │   D   ────┬───┬───┬───┬───┐                           │ │
│   │            │   │   │   │   │                           │ │
│   │        D1 ┘ D2┘ D3┘ D4┘ D5                             │ │
│   │                                                         │ │
│   │   Q   ────┬───┬───┬───┬───┐                           │ │
│   │            │   │   │   │   │                           │ │
│   │        D1 ┘ D2┘ D3┘ D4┘ D5                             │ │
│   │                                                         │ │
│   │   💡 의도하지 않은 중간 상태가 출력에 반영될 수 있음       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   해결:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 에지 트리거 사용 (플립플롭)                          │ │
│   │      • 특정 순간에만 샘플링                              │ │
│   │                                                         │ │
│   │   ② 마스터-슬레이브 구조                                 │ │
│   │      • 입력과 출력 분리                                 │ │
│   │                                                         │ │
│   │   ③ 입력 안정화 후 Enable                               │ │
│   │      • 타이밍 제어                                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 레벨 트리거 회로 구현

```
┌─────────────────────────────────────────────────────────────────┐
│           D 래치 회로 구현                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   D 래치 논리 회로:                                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Q = EN·D + EN'·Q                                      │ │
│   │                                                         │ │
│   │   D ─────┬──[AND]────┐                                  │ │
│   │          │     ↑      │                                  │ │
│   │   EN ────┴──────────┐│                                  │ │
│   │                     ││                                  │ │
│   │                [OR]──┴──▶ Q                             │ │
│   │                     ││                                  │ │
│   │   Q ───[AND]────────┘│                                  │ │
│   │         ↑            │                                   │ │
│   │   EN'───┘            │                                   │ │
│   │                      │                                   │ │
│   │   💡 EN=1: Q=D, EN=0: Q=Q (유지)                        │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   게이트 레벨 구현:                                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │              ┌──────────────────────┐                   │ │
│   │   D ────────▶│                      │                   │ │
│   │              │    멀티플렉서 기반    │────▶ Q            │ │
│   │   Q ────────▶│     2:1 MUX          │                   │ │
│   │              │                      │                   │ │
│   │   EN ───────▶│   (Select = EN)      │                   │ │
│   │              └──────────────────────┘                   │ │
│   │                                                         │ │
│   │   EN=1 → D 선택, EN=0 → Q 선택 (피드백)                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **레벨 트리거**: 활성 레벨 구간 내 지속적으로 입력 반영
> **투명 모드**: Enable 활성 시 입력이 출력으로 직접 전달
> **래치 모드**: Enable 비활성 시 상태 유지

### 2. 빈출 키워드

- **Transparent Mode**: 투명 모드 (EN=1일 때 Q=D)
- **Latched Mode**: 래치 모드 (EN=0일 때 상태 유지)
- **Transparency Problem**: 투명성 문제 (입력 변화가 모두 반영)
- **Level-Sensitive**: 레벨 민감형

### 3. 예상 문제

**Q1. 레벨 트리거의 동작 원리와 투명성 문제를 설명하시오.**

**Q2. 레벨 트리거와 에지 트리거의 차이점을 비교하고 각각의 적합한 응용을 기술하시오.**

**Q3. D 래치의 회로를 설계하고 동작을 설명하시오.**
