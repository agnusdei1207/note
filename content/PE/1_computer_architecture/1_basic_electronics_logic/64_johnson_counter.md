# 존슨 카운터 (Johnson Counter)

## 정의

시프트 레지스터의 **마지막 출력의 반전(Q')을 첫 번째 입력에 연결**한 카운터. n비트 존슨 카운터는 **2n개의 상태**를 가지며, 링 카운터보다 효율적이다. 모아비(Moebius) 카운터라고도 한다.

```
┌─────────────────────────────────────────────────────────────┐
│         4비트 존슨 카운터 (Johnson Counter)                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                         ┌───────────────┐                  │
│                         │  NOT Gate     │                  │
│                         │  Q3 → Q3'     │                  │
│                         └──────┬────────┘                  │
│                                │                            │
│                                ▼                            │
│  ┌─────────┐     ┌─────────┐     ┌─────────┐     ┌────────┐│
│  │  D FF   │────▶│  D FF   │────▶│  D FF   │────▶│  D FF  ││
│  │   Q0    │     │   Q1    │     │   Q2    │     │  Q3    ││
│  └────┬────┘     └────┬────┘     └────┬────┘     └───┬────┘│
│       │               │               │               │      │
│       ▼               ▼               ▼               ▼      │
│      Q0              Q1              Q2              Q3     │
│                                                             │
│              ┌─────────────────┐                           │
│              │       CLK       │  ← 공통 클럭              │
│              └─────────────────┘                           │
│                                                             │
│  피드백: Q3' (반전 출력) → D0                               │
│                                                             │
│  상태 순서 (초기 0000):                                     │
│  클럭 │ Q0 Q1 Q2 Q3 │ 10진수 │ 패턴                        │
│  ────┼─────────────┼────────┼─────────                     │
│   0  │  0  0  0  0 │   0    │ 0000                         │
│   1  │  1  0  0  0 │   8    │ 1000                         │
│   2  │  1  1  0  0 │  12    │ 1100                         │
│   3  │  1  1  1  0 │  14    │ 1110                         │
│   4  │  1  1  1  1 │  15    │ 1111                         │
│   5  │  0  1  1  1 │   7    │ 0111                         │
│   6  │  0  0  1  1 │   3    │ 0011                         │
│   7  │  0  0  0  1 │   1    │ 0001                         │
│   8  │  0  0  0  0 │   0    │ 0000 (반복)                  │
│                                                             │
│  특징: 2n개 상태 (n=4 → 8상태)                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4비트 존슨 카운터 구조 상세 설명

존슨 카운터는 링 카운터와 유사하지만, 피드백에 **반전 출력(Q')**을 사용한다는 점이 다르다. 이로 인해 n비트 존슨 카운터는 2n개의 상태를 가진다.

**피드백 연결**이 핵심 차이이다. 링 카운터는 Q3를 D0에 연결하지만, 존슨 카운터는 Q3'(반전 출력)를 D0에 연결한다. NOT 게이트를 사용하거나, FF의 Q' 출력을 직접 사용할 수 있다.

**상태 순환**을 분석하면, 초기 상태 0000에서 Q3'=1이므로 D0=1이 된다. 다음 클럭에서 1000이 된다. 이후 1100→1110→1111→0111→0011→0001→0000으로 8 클럭 주기로 순환한다.

**2n 상태 원리**를 이해하려면 상태 패턴을 관찰해야 한다. 0000에서 시작하여 1이 왼쪽에서 오른쪽으로 채워진다(1000→1100→1110→1111). 그 후 0이 왼쪽에서 오른쪽으로 채워진다(0111→0011→0001→0000). n비트를 채우는 데 n 클럭, 다시 비우는 데 n 클럭이 걸려 총 2n 상태가 된다.

```
┌─────────────────────────────────────────────────────────────┐
│         존슨 카운터 상태 전이 다이어그램                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                    0000                                     │
│                      │                                      │
│                      ▼                                      │
│                    1000                                     │
│                      │                                      │
│                      ▼                                      │
│                    1100                                     │
│                      │                                      │
│        1이 채워짐    ▼                                      │
│     (Shift Right   1110                                     │
│      with D0=1)      │                                      │
│                      ▼                                      │
│                    1111                                     │
│                      │                                      │
│                      ▼                                      │
│                    0111                                     │
│                      │                                      │
│        0이 채워짐    ▼                                      │
│     (Shift Right   0011                                     │
│      with D0=0)      │                                      │
│                      ▼                                      │
│                    0001                                     │
│                      │                                      │
│                      ▼                                      │
│                    0000 ──┐                                 │
│                      ▲    │                                 │
│                      └────┘ (8상태 순환)                    │
│                                                             │
│  상태 수 = 2n = 2×4 = 8                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 상태 전이 다이어그램 분석

상태 전이 다이어그램은 존슨 카운터의 8상태 순환을 명확히 보여준다. 두 개의 위상(1 채우기, 0 채우기)으로 구성된다.

**1 채우기 위상**에서는 D0=Q3'=1이므로 1이 왼쪽에서 오른쪽으로 시프트된다. 0000→1000→1100→1110→1111로, n=4 클럭 동안 1이 채워진다.

**0 채우기 위상**에서는 D0=Q3'=0이므로 0이 왼쪽에서 오른쪽으로 시프트된다. 1111→0111→0011→0001→0000으로, n=4 클럭 동안 0이 채워진다.

**순환 특성**을 분석하면, 존슨 카운터는 0000과 1111 두 극단 상태를 모두 포함한다. 이는 링 카운터(1000, 0100, 0010, 0001)와 다른 점이다. 2n개의 상태는 모두 인접 상태와 한 비트만 다르다(그레이 코드 특성).

```
┌─────────────────────────────────────────────────────────────┐
│         존슨 카운터 디코딩                                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  존슨 카운터의 8상태 디코딩 (2-input AND 게이트만 사용)     │
│                                                             │
│  상태 │ Q0 Q1 Q2 Q3 │ 디코딩 논리                           │
│  ────┼─────────────┼──────────────────                     │
│   0  │  0  0  0  0 │ Q0'·Q3'                              │
│   1  │  1  0  0  0 │ Q0·Q1'                               │
│   2  │  1  1  0  0 │ Q1·Q2'                               │
│   3  │  1  1  1  0 │ Q2·Q3'                               │
│   4  │  1  1  1  1 │ Q0·Q3                                │
│   5  │  0  1  1  1 │ Q0'·Q1                               │
│   6  │  0  0  1  1 │ Q1'·Q2                               │
│   7  │  0  0  0  1 │ Q2'·Q3                               │
│                                                             │
│  특징:                                                      │
│  - 각 상태를 2-input AND 게이트로 디코딩 가능               │
│  - 인접한 두 비트만 고려                                    │
│  - 디코딩 스파이크 없음                                     │
│                                                             │
│  디코딩 회로 예시 (State 0):                                │
│                                                             │
│  Q0' ─────────┐                                             │
│               │                                             │
│               ├──▶ [AND] ──▶ State 0                       │
│               │                                             │
│  Q3' ─────────┘                                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 존슨 카운터 디코딩 분석

존슨 카운터의 가장 큰 장점은 **간단한 디코딩**이다. 각 상태를 2-input AND 게이트만으로 디코딩할 수 있다.

**디코딩 논리**를 분석하면, 각 상태에서 "변화가 발생하는 지점"의 두 비트를 AND하면 된다. State 0(0000)에서 Q0'=1, Q3'=1이므로 Q0'·Q3' = 1. State 1(1000)에서 Q0=1, Q1'=1이므로 Q0·Q1' = 1.

**스파이크 없는 디코딩**이 가능하다. 존슨 카운터는 한 번에 한 비트만 천이하므로(그레이 코드와 유사), 디코딩 게이트 입력이 동시에 변하지 않는다. 따라서 디코딩 스파이크가 발생하지 않는다.

**게이트 수**를 비교하면, 8상태 2진 카운터(3비트)는 3-8 디코더(8개 3-input AND)가 필요하지만, 4비트 존슨 카운터는 8개 2-input AND만 필요하다. 게이트 복잡도가 낮다.

```
┌─────────────────────────────────────────────────────────────┐
│         존슨 카운터 vs 링 카운터 vs 2진 카운터                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌────────────────┬────────────┬────────────┬────────────┐ │
│  │     특성        │  2진 카운터 │  링 카운터  │ 존슨 카운터│ │
│  ├────────────────┼────────────┼────────────┼────────────┤ │
│  │ n비트 상태 수   │    2^n     │     n      │     2n     │ │
│  │ 4비트 예시      │     16     │     4      │     8      │ │
│  │ FF 효율        │    최고    │    최저    │    중간    │ │
│  │ 디코딩 복잡도  │    높음    │    없음    │    낮음    │ │
│  │ 디코딩 게이트  │  n-input   │   없음     │  2-input   │ │
│  │ 스파이크       │    가능    │    없음    │    없음    │ │
│  │ 초기화        │   불필요   │    필요    │    필요    │ │
│  │ 오류 상태      │    없음    │    가능    │    가능    │ │
│  │ 주파수 분주    │    2^n    │     n      │     2n     │ │
│  └────────────────┴────────────┴────────────┴────────────┘ │
│                                                             │
│  FF 효율 비교 (상태 수 / FF 수):                            │
│  - 2진: 2^n / n = 2^(n-1) × (2/n) → 최고                  │
│  - 링: n / n = 1 → 최저                                    │
│  - 존슨: 2n / n = 2 → 중간                                 │
│                                                             │
│  4비트 기준:                                                │
│  - 2진: 16/4 = 4.0 상태/FF                                 │
│  - 링: 4/4 = 1.0 상태/FF                                   │
│  - 존슨: 8/4 = 2.0 상태/FF                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 세 가지 카운터 비교 분석

**FF 효율**에서 2진 카운터가 압도적이다. 4비트에서 2진은 16상태, 존슨은 8상태, 링은 4상태를 가진다. 하지만 효율만으로는 카운터 선택이 결정되지 않는다.

**디코딩 복잡도**에서 링 카운터가 가장 단순하다(디코딩 불필요). 존슨 카운터는 2-input AND로 간단한 디코딩이 가능하다. 2진 카운터는 n-input AND나 디코더가 필요하다.

**주파수 분주** 관점에서, n비트 2진 카운터는 2^n 분주, 링 카운터는 n 분주, 존슨 카운터는 2n 분주를 제공한다. 예를 들어, 100Hz를 12.5Hz로 분주하려면 8 분주가 필요한데, 3비트 2진 카운터 또는 4비트 존슨 카운터가 적합하다.

```
┌─────────────────────────────────────────────────────────────┐
│         존슨 카운터 자동 초기화                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  문제: 잘못된 상태에 빠질 수 있음                            │
│  4비트 존슨 카운터의 정상 상태: 8개                          │
│  가능한 전체 상태: 16개 (4비트)                              │
│  잘못된 상태: 8개 (예: 0100, 1010, 0101, ...)               │
│                                                             │
│  자동 초기화 방법 1: 홀수 패리티 검사                        │
│                                                             │
│  정상 상태의 특징: 인접한 1의 그룹이 하나만 존재             │
│  0000: 1 그룹 = 0개 (정상)                                  │
│  1100: 1 그룹 = 1개 (정상)                                  │
│  1010: 1 그룹 = 2개 (비정상!)                               │
│                                                             │
│  자동 초기화 방법 2: n 클럭 후 강제 리셋                    │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  추가 카운터로 n 클럭 카운트                         │   │
│  │  n 클럭 도달 시 강제로 0000 또는 1000으로 리셋       │   │
│  │                                                     │   │
│  │  ┌─────────┐                                       │   │
│  │  │ Johnson │     ┌─────────┐                       │   │
│  │  │ Counter │────▶│ n-bit   │───▶ Reset            │   │
│  │  │         │ CLK▶│ Counter │    Signal            │   │
│  │  └─────────┘     └─────────┘                       │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 자동 초기화 방법

존슨 카운터도 링 카운터와 마찬가지로 잘못된 상태에 빠질 수 있다. 4비트 존슨 카운터의 정상 상태는 8개지만, 전체 가능한 상태는 16개이다. 0100, 1010, 0101 등은 비정상 상태이다.

**홀수 패리티 검사**는 비정상 상태를 감지하는 방법이다. 정상 상태에서는 1이 연속된 구간이 하나만 존재한다. 0000(0개), 1100(1개), 1111(1개) 등. 하지만 1010에서는 1 구간이 2개이므로 비정상이다.

**n 클럭 강제 리셋**은 별도의 카운터를 사용하여 n 클럭마다 존슨 카운터를 리셋하는 방법이다. 정상 동작 중에는 2n 클럭 주기이므로, n 클럭마다 리셋하면 잘못된 상태에서도 정상 상태로 복구된다.

```
│                                                             │

```

### 응용 분야 상세 설명

**주파수 분주기**로 존슨 카운터는 2n 분주를 제공한다. 2의 거듭제곱이 아닌 짝수 분주에 유리하다. 예를 들어, 6분주는 3비트 존슨 카운터로, 10분주는 5비트 존슨 카운터로 구현할 수 있다.

**3상 클럭 생성**에서 존슨 카운터가 활용된다. 3비트 존슨 카운터의 Q0, Q1, Q2는 120° 위상차를 가진 3개의 클럭 신호를 생성할 수 있다. BLDC 모터 구동, 3상 전력 제어에 사용된다.

## 특징

### 장점
1. **2n 상태**: 링 카운터보다 2배 효율적
2. **간단한 디코딩**: 2-input AND만 필요
3. **스파이크 없음**: 한 비트만 천이
4. **짝수 분주**: 2의 거듭제곱이 아닌 분주 구현

### 단점
1. **중간 FF 효율**: 2진 카운터보다 낮음
2. **초기화 필요**: 잘못된 상태 가능
3. **홀수 분주 불가**: 2n 분주만 가능

## 기술사 시험 포인트

1. **존슨 카운터 정의**: Q' 피드백, 2n 상태
2. **상태 순환**: 1 채우기 n클럭 + 0 채우기 n클럭
3. **디코딩**: 2-input AND로 각 상태 디코딩
4. **비교**: 링(n상태) < 존슨(2n상태) < 2진(2^n상태)
5. **응용**: 주파수 분주, 다상 클럭, 모터 구동
6. **자동 초기화**: 패리티 검사 또는 강제 리셋
