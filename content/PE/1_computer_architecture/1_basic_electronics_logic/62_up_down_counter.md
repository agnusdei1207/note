# 업/다운 카운터 (Up/Down Counter)

## 정의

제어 신호에 따라 **증가(Up) 또는 감소(Down)** 카운팅을 선택할 수 있는 카운터. 양방향 위치 제어, 가감산 카운팅, 펄스 폭 측정 등에 활용된다.

```

│         3비트 업/다운 카운터 구조                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Up/Down Control                                            │
│       │                                                     │
│       │    1 = 증가 (Up)                                    │
│       │    0 = 감소 (Down)                                  │
│       ▼                                                     │
│  ┌────────────────────────────────────────────────────────┐│
│  │                 제어 논리 (Control Logic)               ││
│  │                                                        ││
│  │   Up/Down=1 (증가):                                    ││
│  │   J0=K0=1 (항상 토글)                                  ││
│  │   J1=K1=Q0 (Q0=1일 때 토글)                            ││
│  │   J2=K2=Q0·Q1 (Q0=Q1=1일 때 토글)                      ││

│          │              │              │                    │

│     ┌─────────┐    ┌─────────┐    ┌─────────┐              │

│        └────┬────┘         │              │                │

```

### 3비트 업/다운 카운터 구조 상세 설명

업/다운 카운터는 단일 제어 신호로 카운팅 방향을 선택하는 다기능 카운터이다. 위 다이어그램은 동기식 JK 플립플롭 기반 3비트 업/다운 카운터의 구조를 보여준다.

**Up/Down 제어 신호**가 핵심이다. Up/Down = 1이면 증가 모드로, 0이면 감소 모드로 동작한다. 이 신호는 제어 논리(Control Logic)에 입력되어 각 플립플롭의 J, K 입력에 적절한 값을 생성한다.

**증가 모드(Up Mode)**에서의 토글 조건을 분석하면, 2진수 증가에서 "각 비트는 모든 하위 비트가 1일 때만 토글된다"는 규칙이 있다. FF0은 항상 토글(J0=K0=1), FF1은 Q0=1일 때 토글(J1=K1=Q0), FF2는 Q0=Q1=1일 때 토글(J2=K2=Q0·Q1)한다.

**감소 모드(Down Mode)**에서의 토글 조건을 분석하면, 2진수 감소에서 "각 비트는 모든 하위 비트가 0일 때만 토글된다"는 규칙이 있다. FF0은 항상 토글(J0=K0=1), FF1은 Q0=0일 때 토글(J1=K1=Q0'), FF2는 Q0=Q1=0일 때 토글(J2=K2=Q0'·Q1')한다.

**공통 클럭**은 모든 FF가 동시에 천이함을 보장한다. 동기식 구조이므로 리플 지연이 없으며, 모드 전환 시에도 스파이크 없이 안정적으로 동작한다.

### 동작 순서표 분석

동작 순서표는 증가 모드와 감소 모드에서의 카운트 변화를 명확히 보여준다. 두 모드 모두 3비트이므로 0~7 범위에서 순환한다.

**증가 모드**에서 0→1→2→...→7→0으로 순차적으로 증가한다. 7(111) 다음 클럭에서 오버플로우(Overflow)가 발생하여 0(000)으로 되돌아간다. 오버플로우는 Carry 출력으로 외부에 알릴 수 있다.

**감소 모드**에서 7→6→5→...→0→7로 순차적으로 감소한다. 0(000) 다음 클럭에서 언더플로우(Underflow)가 발생하여 7(111)로 되돌아간다. 언더플로우는 Borrow 출력으로 외부에 알릴 수 있다.

**모드 전환**은 언제든 가능하다. 예를 들어, 현재 3(011)인 상태에서 Up/Down를 1→0으로 변경하면, 다음 클럭에서 2(010)로 감소한다. 모드 전환 시에도 동기식 구조 덕분에 스파이크 없이 깔끔하게 천이한다.

```
┌─────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────┤

│         └───────────┘                                       │
│                                                             │

│  논리식:                                                   │
│  J1 = K1 = Up/Down·Q0 + Up/Down'·Q0'                       │

│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 업/다운 제어 논리 구현

각 플립플롭의 J, K 입력은 2:1 MUX를 통해 생성된다. Up/Down 신호가 MUX의 선택 입력이 되어, 증가 모드와 감소 모드에 맞는 토글 조건을 선택한다.

**FF1의 J, K 입력**은 Up/Down=1이면 Q0를 선택하고, Up/Down=0이면 Q0'를 선택한다. 논리식으로 표현하면 J1 = K1 = Up/Down·Q0 + Up/Down'·Q0'이다. 이는 Up/Down 신호와 Q0를 사용하여 필요한 조건을 생성한다.

**FF2의 J, K 입력**은 Up/Down=1이면 Q0·Q1을 선택하고, Up/Down=0이면 Q0'·Q1'을 선택한다. 논리식으로 표현하면 J2 = K2 = Up/Down·(Q0·Q1) + Up/Down'·(Q0'·Q1')이다.

**FF0의 J, K 입력**은 항상 1이다. FF0은 증가/감소 모드 모두에서 매 클럭마다 토글된다. 이는 2진수 증감에서 LSB가 항상 변경됨을 의미한다.

```
┌─────────────────────────────────────────────────────────────┐
│         Carry & Borrow 출력                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌────────────────────────────────────────────────────────┐│
│  │                4-bit Up/Down Counter                   ││
│  │                                                        ││
│  │  Up/Down ────▶ [Counter] ────▶ Q3 Q2 Q1 Q0            ││
│  │      │                          │                      ││
│  │      │                          │                      ││
│  │      ▼                          ▼                      ││
│  │  ┌────────┐                ┌──────────┐               ││
│  │  │ Carry  │                │ Borrow   │               ││
│  │  │ Logic  │                │ Logic    │               ││
│  │  └────┬───┘                └────┬─────┘               ││
│  │       │                         │                      ││
│  │       ▼                         ▼                      ││
│  │     Carry                     Borrow                   ││
│  │  (오버플로우)               (언더플로우)               ││
│  │                                                        ││
│  └────────────────────────────────────────────────────────┘│
│                                                             │
│  Carry = Up/Down·Q3·Q2·Q1·Q0                               │
│        (증가 모드에서 1111 → 0000 전이 시 활성화)           │
│                                                             │
│  Borrow = Up/Down'·Q3'·Q2'·Q1'·Q0'                         │
│         (감소 모드에서 0000 → 1111 전이 시 활성화)          │
│                                                             │
│  다단계 카운터 확장:                                        │
│  Carry ───▶ 다음 단계 Enable                               │
│  Borrow ──▶ 다음 단계 Enable                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Carry & Borrow 출력

다단계 카운터 구성을 위해 Carry와 Borrow 출력이 필요하다. 이들은 각각 오버플로우와 언더플로우를 외부에 알린다.

**Carry 출력**은 증가 모드에서 모든 비트가 1일 때(예: 1111) 활성화된다. 다음 클럭에서 0000으로 되돌아가며, 이때 Carry가 1이 되어 다음 단계 카운터에 "증가하라"는 신호를 보낸다. 논리식: Carry = Up/Down·Q3·Q2·Q1·Q0

**Borrow 출력**은 감소 모드에서 모든 비트가 0일 때(예: 0000) 활성화된다. 다음 클럭에서 1111(또는 11111111)로 되돌아가며, 이때 Borrow가 1이 되어 다음 단계 카운터에 "감소하라"는 신호를 보낸다. 논리식: Borrow = Up/Down'·Q3'·Q2'·Q1'·Q0'

**다단계 확장**을 통해 8비트, 16비트, 32비트 업/다운 카운터를 구성할 수 있다. 4비트 카운터 4개를 직렬로 연결하면 16비트 카운터가 된다. 하위 단계의 Carry/Borrow 출력이 상위 단계의 Enable 입력으로 연결된다.

### 응용 분야 상세 설명

**위치 제어 시스템**에서 업/다운 카운터는 모터의 회전을 카운트하여 현재 위치를 추적한다. 정방향 회전 시 Up 모드로 증가하고, 역방향 회전 시 Down 모드로 감소한다. 스테퍼 모터나 엔코더와 함께 사용되어 정밀한 위치 제어를 가능하게 한다.

**재고 관리 시스템**에서는 상품 입고 시 Up 카운트, 출고 시 Down 카운트를 수행한다. 실시간으로 현재 재고량을 파악할 수 있으며, 재고가 일정 수준 이하로 떨어지면 자동으로 발주 신호를 생성할 수 있다.

**주차장 관리 시스템**에서는 입구와 출구에 센서를 설치하여 차량 진입 시 Up, 진출 시 Down 카운트한다. 현재 주차 대수와 빈 자리 수를 실시간으로 표시할 수 있다.

## 특징

### 장점
1. **양방향 카운팅**: 증가/감소 모두 지원
2. **모드 전환**: 실행 중 모드 변경 가능
3. **다기능**: 다양한 응용 분야에 활용

### 단점
1. **복잡한 제어 논리**: MUX 또는 추가 게이트 필요
2. **오버플로우/언더플로우 처리**: 별도 로직 필요
3. **지연 증가**: 제어 논리로 인한 추가 지연

## 기술사 시험 포인트

1. **업/다운 카운터 정의**: 제어 신호로 증가/감소 선택
2. **토글 조건**: 증가(하위 비트 모두 1), 감소(하위 비트 모두 0)
3. **제어 논리**: MUX로 Up/Down에 따른 조건 선택
4. **Carry/Borrow**: 오버플로우/언더플로우 감지, 다단계 확장
5. **응용 분야**: 위치 제어, 재고 관리, 주차장 관리
