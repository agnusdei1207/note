# 클럭 (Clock)

## 정의

**클럭(Clock)**은 디지털 시스템에서 **동기(Synchronization)**를 위한 주기적인 펄스 신호다. 모든 순차 회로 요소가 클럭 신호에 맞춰 동작하며, 시스템 전체의 타이밍을 조정한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           클럭 신호의 기본 형태                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   전압                                                   │ │
│   │    Vdd ──┐   ┌──┐   ┌──┐   ┌──┐   ┌──┐                │ │
│   │          │   │  │   │  │   │  │   │  │                │ │
│   │    0V  ──┘   └──┘   └──┘   └──┘   └──┘   ──▶ 시간     │ │
│   │          │←─→│                                    │     │ │
│   │         Tlow  Thigh                                │     │ │
│   │          │←───────── T (주기) ────────────────────→│     │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   주요 파라미터:                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   • 주기 (Period, T): 한 사이클의 시간                   │ │
│   │   • 주파수 (Frequency, f): 1초당 사이클 수, f = 1/T     │ │
│   │   • 듀티 사이클 (Duty Cycle): Thigh / T × 100%          │ │
│   │   • 상승 시간 (Rise Time): Low → High 전이 시간         │ │
│   │   • 하강 시간 (Fall Time): High → Low 전이 시간         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   주파수 단위:                                                   │
│   • Hz (헤르츠): 초당 사이클                                    │
│   • MHz (메가헤르츠): 10^6 Hz                                   │
│   • GHz (기가헤르츠): 10^9 Hz                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 클럭의 역할

```
┌─────────────────────────────────────────────────────────────────┐
│           클럭의 역할과 중요성                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 동기화 (Synchronization):                           │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 모든 플립플롭이 같은 시점에 상태 변경        │    │ │
│   │   │ • 데이터 전송 타이밍 보장                       │    │ │
│   │   │ • 경쟁 상황(Race Condition) 방지               │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ② 타이밍 제어 (Timing Control):                       │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 명령어 실행 단계 구분                         │    │ │
│   │   │ • 파이프라인 스테이지 동기                      │    │ │
│   │   │ • 버스 전송 타이밍                              │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ③ 성능 결정 (Performance):                            │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 클럭 주파수 ↑ → 처리 속도 ↑                  │    │ │
│   │   │ • CPI × 클럭 주기 = 명령어 실행 시간           │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 클럭 에지와 트리거

```
┌─────────────────────────────────────────────────────────────────┐
│           클럭 에지와 트리거 방식                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   클럭 신호:                                             │ │
│   │                                                         │ │
│   │        ┌──┐     ┌──┐                                    │ │
│   │   ────┘  └─────┘  └────                                │ │
│   │        ↑     ↑     ↑                                    │ │
│   │      상승  하강  상승                                    │ │
│   │      에지  에지  에지                                    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   에지 트리거 (Edge Trigger):                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   상승 에지 (Rising Edge / Positive Edge):              │ │
│   │   • 0 → 1 전이 시점에 동작                              │ │
│   │   • 가장 일반적인 방식                                  │ │
│   │                                                         │ │
│   │   하강 에지 (Falling Edge / Negative Edge):             │ │
│   │   • 1 → 0 전이 시점에 동작                              │ │
│   │   • 일부 시스템에서 사용                                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   레벨 트리거 (Level Trigger):                                  │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   High 레벨: 클럭이 High일 때 동작                      │ │
│   │   Low 레벨: 클럭이 Low일 때 동작                        │ │
│   │                                                         │ │
│   │   💡 에지 트리거가 더 안정적 (한 번만 샘플링)            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 클럭 타이밍 제약

```
┌─────────────────────────────────────────────────────────────────┐
│           클럭 타이밍 제약조건                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ┌─────┐         ┌─────────┐         ┌─────┐          │ │
│   │   │ FF1 │────────▶│조합 논리 │────────▶│ FF2 │          │ │
│   │   └─────┘  tpd    └─────────┘  tcomb  └─────┘          │ │
│   │     ▲                                          │        │ │
│   │     │                                          │        │ │
│   │    CLK ─────────────────────────────────────┘        │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   Setup Time (tsu):                                             │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   클럭 에지 전에 데이터가 안정되어 있어야 하는 시간      │ │
│   │                                                         │ │
│   │   데이터 ──────┬────────────────┐                       │ │
│   │                │←── tsu ────────│                       │ │
│   │                │                │                       │ │
│   │   CLK ─────────┘                │                       │ │
│   │                    ↑ 에지       │                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   Hold Time (th):                                               │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   클럭 에지 후에도 데이터가 유지되어야 하는 시간         │ │
│   │                                                         │ │
│   │   데이터 ────────────────┬─────────────────             │ │
│   │                     │←── th ────│                       │ │
│   │                     │           │                       │ │
│   │   CLK ──────────────┘           │                       │ │
│   │                     ↑ 에지      │                       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   Clock-to-Q Delay (tclk-q):                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   클럭 에지 후 출력이 유효해질 때까지의 시간             │ │
│   │                                                         │ │
│   │   CLK ──────┐                                           │ │
│   │             │                                           │ │
│   │             │←── tclk-q ──→│                           │ │
│   │             │                │                           │ │
│   │   Q ────────┘                ──── (변화)                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   최대 클럭 주파수:                                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Tclk ≥ tclk-q + tcomb(max) + tsu                     │ │
│   │                                                         │ │
│   │   fmax = 1 / (tclk-q + tcomb(max) + tsu)               │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 클럭 분배와 스큐

```
┌─────────────────────────────────────────────────────────────────┐
│           클럭 분배 네트워크                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   클럭 스큐 (Clock Skew):                                       │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   같은 클럭 신호가 서로 다른 플립플롭에                  │ │
│   │   서로 다른 시간에 도달하는 현상                         │ │
│   │                                                         │ │
│   │   CLK 소스 ──▶ FF1 (도착: t1)                           │ │
│   │              │                                          │ │
│   │              └──▶ FF2 (도착: t2)                        │ │
│   │                                                         │ │
│   │   Clock Skew = |t1 - t2|                                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   H-Tree 클럭 분배:                                             │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │                   CLK 소스                              │ │
│   │                      │                                  │ │
│   │            ┌─────────┴─────────┐                        │ │
│   │            │                   │                        │ │
│   │      ┌─────┴─────┐       ┌─────┴─────┐                  │ │
│   │      │           │       │           │                  │ │
│   │   ┌──┴──┐     ┌──┴──┐  ┌──┴──┐     ┌──┴──┐            │ │
│   │   FF  FF     FF  FF   FF  FF     FF  FF            │ │
│   │                                                         │ │
│   │   💡 모든 경로 길이 동일 → 스큐 최소화                   │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   스큐의 영향:                                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   Positive Skew (FF2가 늦게 도착):                      │ │
│   │   • Setup 위험 감소, Hold 위험 증가                     │ │
│   │                                                         │ │
│   │   Negative Skew (FF2가 빨리 도착):                      │ │
│   │   • Setup 위험 증가, Hold 위험 감소                     │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **클럭**: 디지털 시스템 동기를 위한 주기적 펄스 신호
> **주파수**: 단위 시간당 클럭 사이클 수 (f = 1/T)
> **에지 트리거**: 클럭 전이 시점에 동작하는 방식

### 2. 빈출 키워드

- **Setup Time**: 클럭 에지 전 데이터 안정 시간
- **Hold Time**: 클럭 에지 후 데이터 유지 시간
- **Clock Skew**: 클럭 도달 시간 차이
- **Duty Cycle**: High 구간 비율

### 3. 예상 문제

**Q1. 클럭 신호의 주요 파라미터와 타이밍 제약조건을 설명하시오.**

**Q2. Setup Time 위반과 Hold Time 위반이 발생하는 원인과 해결 방안을 기술하시오.**

**Q3. 클럭 스큐(Clock Skew)가 시스템 성능에 미치는 영향과 H-Tree 분배 방식을 설명하시오.**
