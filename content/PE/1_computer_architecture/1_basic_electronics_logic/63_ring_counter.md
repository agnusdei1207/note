# 링 카운터 (Ring Counter)

## 정의

시프트 레지스터의 **마지막 출력을 첫 번째 입력에 연결**하여 하나의 1(또는 0)이 순환하는 카운터. n비트 링 카운터는 n개의 상태를 가지며, 원-핫(One-Hot) 상태 인코딩에 활용된다.

```
┌─────────────────────────────────────────────────────────────┐
│         4비트 링 카운터 (Ring Counter)                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                    ┌─────────────────────┐                  │
│                    │    Feedback Loop    │                  │
│                    │   (마지막→첫 번째)   │                  │
│                    └──────────┬──────────┘                  │
│                               │                             │
│                               ▼                             │
│  ┌─────────┐     ┌─────────┐     ┌─────────┐     ┌────────┐│
│  │  D FF   │────▶│  D FF   │────▶│  D FF   │────▶│  D FF  ││
│  │   Q0    │     │   Q1    │     │   Q2    │     │  Q3    ││
│  └────┬────┘     └────┬────┘     └────┬────┘     └───┬────┘│
│       │               │               │               │      │
│       ▼               ▼               ▼               ▼      │
│      Q0              Q1              Q2              Q3     │
│                                                             │
│              ┌─────────────────┐                           │
│              │       CLK       │  ← 공통 클럭              │
│              └─────────────────┘                           │
│                                                             │
│  초기 상태: Q0=1, Q1=Q2=Q3=0 (프리셋 필요)                  │
│                                                             │
│  상태 순서:                                                  │
│  클럭 │ Q0 Q1 Q2 Q3 │ 상태                                  │
│  ────┼─────────────┼────────                                │
│   0  │  1  0  0  0 │ State 0 (초기)                        │
│   1  │  0  1  0  0 │ State 1                               │
│   2  │  0  0  1  0 │ State 2                               │
│   3  │  0  0  0  1 │ State 3                               │
│   4  │  1  0  0  0 │ State 0 (반복)                        │
│                                                             │
│  특징: 하나의 1이 링을 따라 순환                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4비트 링 카운터 구조 상세 설명

링 카운터는 시프트 레지스터의 출력을 입력에 피드백하여 순환 구조를 만든다. 위 다이어그램은 4비트 링 카운터의 기본 구조와 상태 순서를 보여준다.

**피드백 루프(Feedback Loop)**가 핵심이다. 마지막 FF(Q3)의 출력이 첫 번째 FF(Q0)의 D 입력에 연결된다. 이로써 데이터가 Q0 → Q1 → Q2 → Q3 → Q0 → ... 순서로 무한히 순환한다. 링(Ring)이라는 이름은 이 순환 구조에서 유래했다.

**초기 상태 설정**이 중요하다. 링 카운터는 프리셋(Preset) 또는 리셋(Reset) 신호로 초기 상태를 설정해야 한다. 일반적으로 Q0=1, Q1=Q2=Q3=0으로 설정하여, 하나의 1만이 존재하게 한다. 이를 **원-핫(One-Hot)** 상태라 한다.

**상태 순환**을 분석하면, 초기 상태 1000에서 클럭마다 1이 오른쪽으로 시프트된다. 1000 → 0100 → 0010 → 0001 → 1000으로 4 클럭 주기로 순환한다. n비트 링 카운터는 n개의 상태를 가지며, n 클럭 주기로 순환한다.

**공통 클럭**은 모든 FF가 동시에 천이함을 보장한다. 이는 동기식 시프트 레지스터 기반이므로 리플 지연이 없다.

```
┌─────────────────────────────────────────────────────────────┐
│         링 카운터 타이밍 다이어그램                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│        ▲     ▲     ▲     ▲     ▲     ▲     ▲     ▲         │
│  CLK  ─┘ ┌──┘ ┌──┘ ┌──┘ ┌──┘ ┌──┘ ┌──┘ ┌──┘ ┌──┘ ┌──      │
│           │     │     │     │     │     │     │     │       │
│  Q0   ────┐     ┌─────────────────────────────┐     ┌────   │
│           └─────┘                             └─────┘       │
│           │                                           │     │
│  Q1   ────┼─────┐     ┌─────────────────────────────┼───┐  │
│           └─────┘     └─────────────────────────────┘   └──│
│                 │                                           │
│  Q2   ──────────┼─────┐     ┌─────────────────────────────┐│
│                 └─────┘     └─────────────────────────────┘│
│                       │                                     │
│  Q3   ────────────────┼─────┐     ┌─────────────────────── │
│                       └─────┘     └──────────────────────── │
│                                                             │
│  State:  0     1     2     3     0     1     2     3       │
│                                                             │
│  각 출력이 순차적으로 1이 됨                                │
│  디코딩 불필요 (직접 상태 출력 가능)                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 타이밍 다이어그램 분석

타이밍 다이어그램은 링 카운터의 동작을 명확히 보여준다. 각 출력이 순차적으로 1이 되며, 어느 시점에도 정확히 하나의 출력만 1이다.

**Q0 파형**은 초기 상태에서 1이고, 1 클럭 후 0이 되며, 4 클럭 후 다시 1이 된다. Q0의 주기는 4 × T_CLK이다.

**Q1, Q2, Q3 파형**은 각각 1, 2, 3 클럭 지연된 Q0 파형이다. 모든 출력의 주기는 동일하게 4 × T_CLK이다.

**상태 해석**이 직관적이다. Q0=1이면 State 0, Q1=1이면 State 1, Q2=1이면 State 2, Q3=1이면 State 3이다. 별도의 디코더 없이 FF 출력을 직접 상태로 사용할 수 있다. 이를 **디코딩 없는 상태 표시(Decoding-Free State Indication)**라 한다.

```
┌─────────────────────────────────────────────────────────────┐
│         링 카운터 vs 2진 카운터 비교                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [2진 카운터 - 4비트]                                       │
│  상태 수: 16개 (0000 ~ 1111)                                │
│  FF 활용률: 높음 (2^n 상태)                                 │
│  디코딩: 필요 (2진 → 10진/상태)                             │
│  스파이크: 디코딩 시 가능                                   │
│                                                             │
│  [링 카운터 - 4비트]                                        │
│  상태 수: 4개 (1000, 0100, 0010, 0001)                      │
│  FF 활용률: 낮음 (n 상태)                                   │
│  디코딩: 불필요 (직접 상태)                                 │
│  스파이크: 없음 (한 비트만 천이)                            │
│                                                             │
│  비교표:                                                    │
│  ┌────────────────┬──────────────┬────────────────┐        │
│  │     특성        │  2진 카운터   │   링 카운터    │        │
│  ├────────────────┼──────────────┼────────────────┤        │
│  │ n비트 상태 수   │    2^n       │      n         │        │
│  │ FF 효율         │    최고      │     낮음       │        │
│  │ 디코더 필요     │    예        │     아니오     │        │
│  │ 속도            │    중간      │     빠름       │        │
│  │ 스파이크        │    가능      │     없음       │        │
│  │ 전력 소모       │    높음      │     낮음       │        │
│  │ 용도            │ 카운팅       │ 순차 제어      │        │
│  └────────────────┴──────────────┴────────────────┘        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 링 카운터 vs 2진 카운터 비교 분석

**FF 활용률**에서 2진 카운터가 압도적이다. n비트 2진 카운터는 2^n개의 상태를 가지지만, 링 카운터는 n개의 상태만 가진다. 4비트에서 2진은 16상태, 링은 4상태이다. 상태당 FF 비용은 링 카운터가 훨씬 높다.

**디코딩**에서 링 카운터가 우위이다. 2진 카운터는 0000, 0001, ... 등의 상태를 10진수나 특정 상태로 변환하기 위해 디코더가 필요하다. 반면 링 카운터는 Q0=1, Q1=1, ... 이 바로 상태를 의미하므로 디코더가 필요 없다.

**속도**에서 링 카운터가 빠르다. 디코딩 게이트 지연이 없으며, 한 비트만 천이하므로 전파 지연도 최소이다. 고속 순차 제어 회로에 적합하다.

**스파이크**에서 링 카운터가 안전하다. 한 번에 정확히 한 비트만 1→0, 다른 한 비트만 0→1로 천이하므로 일시적인 잘못된 상태가 발생하지 않는다.

```
┌─────────────────────────────────────────────────────────────┐
│         링 카운터 초기화 회로                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  문제: 전원 인가 시 임의 상태가 될 수 있음                   │
│  예: 1100, 1010, 0111 등 (잘못된 상태)                      │
│                                                             │
│  해결: 자동 초기화 회로 추가                                 │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                                                     │   │
│  │  ┌─────────┐     ┌─────────┐     ┌─────────┐       │   │
│  │  │  D FF   │────▶│  D FF   │────▶│  D FF   │─┐     │   │
│  │  │   Q0    │     │   Q1    │     │   Q2    │ │     │   │
│  │  └────┬────┘     └────┬────┘     └────┬────┘ │     │   │
│  │       │               │               │      │     │   │
│  │       │               │               │      │     │   │
│  │       └───────────────┴───────────────┘      │     │   │
│  │                       │                      │     │   │
│  │                       ▼                      │     │   │
│  │              ┌───────────────┐               │     │   │
│  │              │     NOR       │               │     │   │
│  │              │ (Q1'+Q2'+Q3') │               │     │   │
│  │              └───────┬───────┘               │     │   │
│  │                      │                       │     │   │
│  │                      ▼                       ▼     │   │
│  │               ┌───────────────────────────────┐   │   │
│  │               │  첫 번째 FF의 D 입력 (MUX)    │   │   │
│  │               │  NOR=0: Q3 (정상 순환)        │   │   │
│  │               │  NOR=1: 1 (자동 초기화)       │   │   │
│  │               └───────────────────────────────┘   │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  동작: Q1=Q2=Q3=0이면 NOR=1, D0=1 (자동으로 1000 복구)     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 자동 초기화 회로 분석

링 카운터의 치명적 약점은 잘못된 상태(예: 1100, 1010)에 빠질 수 있다는 것이다. 이를 방지하기 위해 자동 초기화 회로를 추가한다.

**문제 상황**을 분석하면, 전원 인가 시 FF가 임의의 상태로 초기화될 수 있다. 정상적인 링 카운터는 정확히 하나의 1만 가져야 하지만, 1100(두 개의 1), 0000(0개의 1), 1111(네 개의 1) 등의 잘못된 상태가 될 수 있다.

**자동 초기화 논리**는 Q1, Q2, Q3가 모두 0인지 검사한다. NOR 게이트로 Q1'+Q2'+Q3'를 계산하여, Q1=Q2=Q3=0이면 NOR=1을 출력한다. 이를 첫 번째 FF의 D 입력에 MUX로 결합한다.

**동작 원리**는 다음과 같다. 정상 순환 중에는 NOR=0이므로 D0 = Q3(기존 피드백)이다. 하지만 Q1=Q2=Q3=0인 잘못된 상태에서는 NOR=1이 되어 D0 = 1이 된다. 다음 클럭에서 Q0=1, Q1=Q2=Q3=0이 되어 정상 상태 1000으로 복구된다.

```
┌─────────────────────────────────────────────────────────────┐
│         링 카운터 응용 분야                                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 순차 제어 (Sequential Control)                          │
│     각 출력이 순차적으로 활성화되어 단계별 동작 제어         │
│     예: 세탁기 사이클, 자판기 동작 순서                      │
│                                                             │
│  2. 타이밍 생성 (Timing Generation)                         │
│     n분주 주기 내에서 각 단계에 해당하는 타이밍 신호 생성    │
│     예: CPU 명령어 사이클 타이밍                            │
│                                                             │
│  3. 스캐닝 (Scanning)                                       │
│     키보드 스캔, 디스플레이 다이내믹 스캔                    │
│     각 행/열을 순차적으로 선택                              │
│                                                             │
│  4. 원-핫 상태 머신 (One-Hot State Machine)                 │
│     각 상태를 하나의 FF로 표현                              │
│     상태 천이 로직 단순화                                   │
│                                                             │
│  5. LED 순차 점등 (LED Chaser)                              │
│     간단한 회로로 LED 순차 점등 효과                        │
│                                                             │
│  6. PWM 생성 (Pulse Width Modulation)                       │
│     링 카운터 + 디코딩으로 다양한 듀티 사이클 생성          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 응용 분야 상세 설명

**순차 제어**는 링 카운터의 가장 직관적인 응용이다. 예를 들어, 세탁기에서 Q0=급수, Q1=세탁, Q2=헹굼, Q3=탈수 단계를 순차적으로 제어할 수 있다. 각 단계는 정확히 하나의 출력만 활성화되므로 충돌이 발생하지 않는다.

**키보드 스캐닝**에서 링 카운터가 키보드 매트릭스의 행을 순차적으로 선택한다. Q0=1이면 첫 번째 행, Q1=1이면 두 번째 행... 이 방식으로 n개의 행을 순차적으로 스캔하여 키 입력을 감지한다.

**원-핫 상태 머신**은 각 상태를 하나의 FF로 표현하는 FSM 설계 기법이다. 링 카운터는 자연스럽게 원-핫 인코딩이 되므로, 상태 머신의 각 상태를 링 카운터의 출력으로 직접 사용할 수 있다. 상태 천이 로직이 단순해진다.

## 특징

### 장점
1. **디코딩 불필요**: FF 출력이 직접 상태 표시
2. **고속**: 한 비트만 천이, 디코딩 지연 없음
3. **스파이크 없음**: 일시적 오류 상태 불가
4. **저전력**: 한 번에 하나의 FF만 1

### 단점
1. **낮은 FF 효율**: n비트로 n상태만 표현 (vs 2^n)
2. **초기화 필요**: 프리셋/자동 초기화 회로 필수
3. **오류 상태**: 잘못된 상태(0000, 1100) 가능

## 기술사 시험 포인트

1. **링 카운터 정의**: 출력→입력 피드백, 하나의 1 순환
2. **n비트 특성**: n개 상태, n 클럭 주기
3. **장점**: 디코딩 없음, 고속, 스파이크 없음
4. **단점**: 낮은 FF 효율, 초기화 필요
5. **자동 초기화**: NOR 게이트로 Q1=Q2=Q3=0 감지
6. **응용**: 순차 제어, 타이밍 생성, 원-핫 FSM
