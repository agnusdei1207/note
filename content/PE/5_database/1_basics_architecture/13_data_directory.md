# 데이터 디렉터리 (Data Directory)

## 정의

**데이터 디렉터리(Data Directory)**는 시스템 카탈로그 중 **시스템만 접근 가능한 부분**이다. 물리적 저장 위치, 내부 포맷, 버퍼 관리 정보 등 DBMS 내부 관리용 메타데이터를 저장한다. 사용자는 직접 접근할 수 없으며 DBMS 내부에서만 사용된다.

```
┌─────────────────────────────────────────────────────────────────┐
│           시스템 카탈로그 vs 데이터 디렉터리                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │              시스템 카탈로그 (System Catalog)            │ │
│   │   ┌─────────────────────────────────────────────────┐  │ │
│   │   │                                                 │  │ │
│   │   │  ┌───────────────┐    ┌───────────────┐         │  │ │
│   │   │  │ 데이터 사전   │    │ 데이터 디렉터리│         │  │ │
│   │   │  │ (Data         │    │ (Data         │         │  │ │
│   │   │  │  Dictionary)  │    │  Directory)   │         │  │ │
│   │   │  └───────┬───────┘    └───────┬───────┘         │  │ │
│   │   │          │                    │                  │  │ │
│   │   │          ▼                    ▼                  │  │ │
│   │   │   사용자 접근 가능      시스템만 접근 가능       │  │ │
│   │   │   (SELECT 조회)        (직접 접근 불가)          │  │ │
│   │   │                                                 │  │ │
│   │   └─────────────────────────────────────────────────┘  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 데이터 사전 vs 데이터 디렉터리

```
┌─────────────────────────────────────────────────────────────────┐
│           데이터 사전 vs 데이터 디렉터리 비교                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분        │ 데이터 사전        │ 데이터 디렉터리      │ │
│   ├──────────────┼─────────────────────┼────────────────────┤ │
│   │ 접근 권한    │ 사용자 조회 가능    │ 시스템 전용        │ │
│   │ 내용         │ 논리적 스키마       │ 물리적 저장 정보   │ │
│   │ 사용자       │ DBA, 개발자, 사용자 │ DBMS 내부 모듈     │ │
│   │ 예시         │ 테이블 정의, 권한   │ 파일 경로, 블록    │ │
│   │ SQL 접근     │ SELECT 가능        │ 직접 불가          │ │
│   │ 목적         │ 정보 제공          │ 내부 관리          │ │
│   └──────────────┴─────────────────────┴────────────────────┘ │
│                                                                 │
│   데이터 사전 예시 (사용자 접근):                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   SELECT table_name, column_name, data_type             │ │
│   │   FROM INFORMATION_SCHEMA.COLUMNS                       │ │
│   │   WHERE table_schema = 'mydb';                          │ │
│   │                                                         │ │
│   │   → 사용자가 SQL로 조회 가능                            │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   데이터 디렉터리 예시 (시스템 전용):                             │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   내부 포인터:                                           │ │
│   │   • File ID: 5, Page: 123, Slot: 4                      │ │
│   │   • Buffer Pool: Pool_A, Frame: 45                      │ │
│   │   • Log Sequence Number: 0x12345678                     │ │
│   │                                                         │ │
│   │   → 사용자 접근 불가, DBMS 내부에서만 사용               │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **데이터 디렉터리**: 시스템만 접근 가능한 메타데이터 영역
> **데이터 사전**: 사용자가 조회 가능한 메타데이터 영역
> **물리적 저장 정보**: 파일 경로, 블록 주소 등 내부 관리 정보

### 2. 빈출 키워드

- **Physical Storage Info**: 물리적 저장 위치 정보
- **Buffer Pool Metadata**: 버퍼 관리용 메타데이터
- **System-only Access**: 시스템 전용 접근
- **Internal Pointer**: 내부 포인터 (File ID, Page ID)

### 3. 예상 문제

**Q1. 데이터 사전(Data Dictionary)과 데이터 디렉터리(Data Directory)의 차이점을 설명하시오.**

**Q2. 데이터 디렉터리에 저장되는 정보의 유형을 분류하고 각각의 예시를 기술하시오.**

**Q3. DBMS 내부 모듈이 데이터 디렉터리를 활용하는 방식을 설명하시오.**
