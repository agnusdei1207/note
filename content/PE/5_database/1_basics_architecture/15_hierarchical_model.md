# 계층형 데이터 모델 (Hierarchical Model)

## 정의

**계층형 데이터 모델(Hierarchical Model)**은 데이터를 **트리(Tree) 구조**로 표현하는 데이터 모델이다. 1960년대 IBM의 IMS(Information Management System)에서 처음 구현되었으며, 부모-자식 관계가 1:N으로 구성되는 것이 특징이다.

```
┌─────────────────────────────────────────────────────────────────┐
│           계층형 모델 구조                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │                        [루트]                           │ │
│   │                       학과                               │ │
│   │                        │                                │ │
│   │            ┌───────────┼───────────┐                    │ │
│   │            │           │           │                    │ │
│   │           ▼           ▼           ▼                    │ │
│   │        [자식1]     [자식2]     [자식3]                  │ │
│   │         교수        학생         과목                   │ │
│   │          │          │            │                     │ │
│   │      ┌───┴───┐  ┌───┴───┐    ┌───┴───┐               │ │
│   │      ▼       ▼  ▼       ▼    ▼       ▼               │ │
│   │   [손자] [손자] [손자] [손자] [손자] [손자]             │ │
│   │    연구   강의   성적   출석  수강   과제               │ │
│   │                                                         │ │
│   │   특징:                                                  │ │
│   │   • 부모:자식 = 1:N (일대다)                             │ │
│   │   • 루트에서 리프까지 단일 경로                          │ │
│   │   • 계층적 접근 (루트 → 자식 → 손자)                     │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 계층형 모델 용어

```
┌─────────────────────────────────────────────────────────────────┐
│           계층형 모델 주요 용어                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   세그먼트 (Segment):                                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 계층형 모델의 논리적 레코드                   │    │ │
│   │   │ • 관계형 모델의 튜플/행에 해당                  │    │ │
│   │   │ • 필드(Field)들의 집합                         │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   노드 (Node):                                            │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 트리 구조의 각 세그먼트                       │    │ │
│   │   │ • 부모-자식 관계 존재                          │    │ │
│   │   │ • 루트 노드, 내부 노드, 리프 노드               │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   부모-자식 관계 (Parent-Child Relationship):             │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 부모는 여러 자식 가능                        │    │ │
│   │   │ • 자식은 하나의 부모만 가능                    │    │ │
│   │   │ • 1:N 관계만 표현 가능                         │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   계층 경로 (Hierarchical Path):                          │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 루트에서 특정 노드까지의 경로                 │    │ │
│   │   │ • 데이터 접근 시 경로 명시 필요                 │    │ │
│   │   │ • 예: 학과 → 학생 → 성적                       │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 데이터 접근 방식

```
┌─────────────────────────────────────────────────────────────────┐
│           계층형 모델 데이터 접근                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   순차적 접근 (Sequential Access):                               │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   트리 순회 순서로 접근 (전위 순회)                       │ │
│   │                                                         │ │
│   │   학과1 → 교수1 → 연구1 → 연구2 → 교수2 → 학생1 → ...   │ │
│   │                                                         │ │
│   │   ┌─────────┐                                           │ │
│   │   │ 학과1   │                                           │ │
│   │   └────┬────┘                                           │ │
│   │   ┌────┴────┐                                           │ │
│   │   ▼         ▼                                           │ │
│   │ ┌─────┐  ┌─────┐                                        │ │
│   │ │교수1│  │학생1│  ← 부모 → 자식 순서                    │ │
│   │ └──┬──┘  └──┬──┘                                        │ │
│   │    ▼        ▼                                           │ │
│   │ ┌─────┐  ┌─────┐                                        │ │
│   │ │연구1│  │성적1│                                        │ │
│   │ └─────┘  └─────┘                                        │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   직접 접근 (Direct Access):                                     │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   계층 경로 명시로 직접 접근                              │ │
│   │                                                         │ │
│   │   GET UNIQUE 학과 WHERE 학과코드 = 'CS'                  │ │
│   │   GET NEXT 학생 WITHIN 학과                              │ │
│   │   GET NEXT 성적 WITHIN 학생                              │ │
│   │                                                         │ │
│   │   💡 상위 노드를 통해 하위 노드 접근                      │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 계층형 모델의 장단점

```
┌─────────────────────────────────────────────────────────────────┐
│           계층형 모델 장단점                                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   장점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 단순성: 트리 구조로 이해하기 쉬움                    │ │
│   │   ② 빠른 접근: 계층 경로를 통한 효율적 검색               │ │
│   │   ③ 데이터 무결성: 부모-자식 관계로 참조 무결성 보장      │ │
│   │   ④ 1:N 관계에 최적화                                   │ │
│   │   ⑤ 성능: 특정 유형의 쿼리에 빠른 응답                   │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   단점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① N:M 관계 표현 불가                                   │ │
│   │      • 학생이 여러 학과에 소속 불가                      │ │
│   │      • 논리적/물리적 복제로 해결 (중복 문제)             │ │
│   │                                                         │ │
│   │   ② 구조 경직성                                           │ │
│   │      • 스키마 변경 어려움                                │ │
│   │      • 새 관계 추가 시 트리 재구성                       │ │
│   │                                                         │ │
│   │   ③ 데이터 중복                                           │ │
│   │      • M:N 관계 표현 시 데이터 복제                      │ │
│   │      • 동기화 문제                                       │ │
│   │                                                         │ │
│   │   ④ 프로그래밍 복잡성                                     │ │
│   │      • 경로 명시 필요                                    │ │
│   │      • 네비게이션형 쿼리                                 │ │
│   │                                                         │ │
│   │   ⑤ 루트 없는 데이터 불가                                 │ │
│   │      • 모든 데이터는 루트 아래 존재해야 함                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. IMS (Information Management System)

```
┌─────────────────────────────────────────────────────────────────┐
│           IBM IMS                                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   개요:                                                  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 1966년 IBM 개발                              │    │ │
│   │   │ • Apollo 우주선 계획 지원                       │    │ │
│   │   │ • 최초의 상용 DBMS                             │    │ │
│   │   │ • DL/I (Data Language/I) 인터페이스 사용       │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   DL/I 호출 유형:                                         │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ GU (Get Unique): 조건에 맞는 첫 번째 세그먼트   │    │ │
│   │   │ GN (Get Next): 다음 세그먼트 순차 검색         │    │ │
│   │   │ GNP (Get Next within Parent): 부모 내 다음     │    │ │
│   │   │ GHU/GHN/GHNP: 갱신용 검색 (Hold)               │    │ │
│   │   │ ISRT (Insert): 세그먼트 삽입                   │    │ │
│   │   │ DLET (Delete): 세그먼트 삭제                   │    │ │
│   │   │ REPL (Replace): 세그먼트 갱신                   │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   현대적 활용:                                            │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 대형 메인프레임 환경에서 여전히 사용          │    │ │
│   │   │ • 금융, 항공, 보험 등 레거시 시스템             │    │ │
│   │   │ • 고성능 트랜잭션 처리                          │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **계층형 모델**: 트리 구조로 데이터를 표현하는 모델
> **1:N 관계**: 부모-자식 관계가 일대다로 제한
> **IMS**: IBM의 계층형 DBMS, 최초 상용 DBMS

### 2. 빈출 키워드

- **Tree Structure**: 트리 구조, 루트-노드-리프
- **Parent-Child**: 부모-자식 관계 (1:N만 가능)
- **Segment**: 계층형 모델의 논리적 레코드
- **IMS/DL/I**: IBM IMS, Data Language/I

### 3. 예상 문제

**Q1. 계층형 데이터 모델의 구조적 특징과 장단점을 설명하시오.**

**Q2. 계층형 모델에서 N:M 관계를 표현하는 방법과 그 한계를 기술하시오.**

**Q3. 계층형 모델과 관계형 모델의 차이점을 비교 설명하시오.**
