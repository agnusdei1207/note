# 망형 데이터 모델 (Network Model)

## 정의

**망형 데이터 모델(Network Model)**은 데이터를 **그래프(Graph) 구조**로 표현하는 데이터 모델이다. 계층형 모델의 제약을 극복하여 **N:M(다대다) 관계**를 표현할 수 있으며, 레코드가 여러 부모를 가질 수 있다. CODASYL DBTG 모델로도 불린다.

```
┌─────────────────────────────────────────────────────────────────┐
│           망형 모델 구조                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │                    ┌─────────┐                          │ │
│   │                    │  학과   │                          │ │
│   │                    └────┬────┘                          │ │
│   │                         │                               │ │
│   │            ┌────────────┼────────────┐                  │ │
│   │            │            │            │                  │ │
│   │            ▼            ▼            ▼                  │ │
│   │       ┌────────┐  ┌────────┐  ┌────────┐               │ │
│   │       │  교수  │  │  학생  │  │  과목  │               │ │
│   │       └───┬────┘  └───┬────┘  └───┬────┘               │ │
│   │           │           │           │                     │ │
│   │           │     ┌─────┴─────┐     │                     │ │
│   │           │     │           │     │                     │ │
│   │           ▼     ▼           ▼     ▼                     │ │
│   │       ┌────────────┐   ┌────────────┐                   │ │
│   │       │   수강     │   │   개설     │   ← M:N 관계      │ │
│   │       └────────────┘   └────────────┘   표현 가능       │ │
│   │                                                         │ │
│   │   특징:                                                  │ │
│   │   • 그래프 구조 (사이클 가능)                            │ │
│   │   • N:M 관계 표현 가능                                   │ │
│   │   • 다중 부모 허용                                       │ │
│   │   • Set 타입으로 관계 정의                               │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 망형 모델 용어

```
┌─────────────────────────────────────────────────────────────────┐
│           망형 모델 주요 용어                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   레코드 타입 (Record Type):                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 노드(Node)에 해당                            │    │ │
│   │   │ • 데이터 항목들의 집합                         │    │ │
│   │   │ • 관계형 모델의 테이블과 유사                   │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   세트 타입 (Set Type):                                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 두 레코드 타입 간의 1:N 관계                  │    │ │
│   │   │ • 오너(Owner)와 멤버(Member)로 구성             │    │ │
│   │   │ • 오너: 1개, 멤버: 여러 개                      │    │ │
│   │   │ • 계층형 모델의 부모-자식 관계                   │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   오너/멤버 (Owner/Member):                               │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 오너: 세트 타입의 부모 레코드                 │    │ │
│   │   │ • 멤버: 세트 타입의 자식 레코드                 │    │ │
│   │   │ • 한 레코드는 여러 세트의 멤버 가능             │    │ │
│   │   │   → 다중 부모 허용                              │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   세트 모드 (Set Mode):                                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • CHAIN: 연결 리스트 방식                       │    │ │
│   │   │ • POINTER ARRAY: 포인터 배열                   │    │ │
│   │   │ • 레코드 간 연결 방식 지정                       │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 세트 타입 구조

```
┌─────────────────────────────────────────────────────────────────┐
│           세트 타입 (Set Type) 구조                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   학과-학생 세트 타입 예시:                               │ │
│   │                                                         │ │
│   │        ┌─────────────┐                                  │ │
│   │        │    학과     │  ← Owner (1)                     │ │
│   │        │ (CS학과)   │                                   │ │
│   │        └──────┬──────┘                                  │ │
│   │               │                                          │ │
│   │     ┌─────────┼─────────┐                               │ │
│   │     │         │         │                               │ │
│   │     ▼         ▼         ▼                               │ │
│   │  ┌─────┐  ┌─────┐  ┌─────┐                              │ │
│   │  │학생1│  │학생2│  │학생3│  ← Members (N)               │ │
│   │  └─────┘  └─────┘  └─────┘                              │ │
│   │     │         │         │                               │ │
│   │     └─────────┴─────────┘  (체인으로 연결)              │ │
│   │                                                         │ │
│   │   N:M 관계 표현 (수강):                                  │ │
│   │                                                         │ │
│   │   ┌──────┐           ┌──────┐                           │ │
│   │   │ 학생 │──(수강1)──│ 과목 │                           │ │
│   │   └──────┘           └──────┘                           │ │
│   │       ↑                  ↑                               │ │
│   │       │                  │                               │ │
│   │   학생-수강 세트    과목-수강 세트                        │ │
│   │   (1:N)            (1:N)                                │ │
│   │                                                         │ │
│   │   💡 두 개의 1:N 세트로 N:M 관계 표현                     │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 데이터 조작 (DML)

```
┌─────────────────────────────────────────────────────────────────┐
│           망형 모델 DML 명령어                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   CODASYL DML 명령어:                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   검색 명령:                                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ FIND     - 레코드 위치 찾기                     │    │ │
│   │   │ GET      - 현재 레코드 가져오기                 │    │ │
│   │   │ FIND NEXT - 다음 레코드 찾기                    │    │ │
│   │   │ FIND OWNER - 오너 레코드 찾기                   │    │ │
│   │   │ FIND MEMBER - 멤버 레코드 찾기                  │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   갱신 명령:                                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ STORE    - 새 레코드 저장                      │    │ │
│   │   │ MODIFY  - 레코드 수정                          │    │ │
│   │   │ ERASE    - 레코드 삭제                          │    │ │
│   │   │ CONNECT  - 세트에 연결                          │    │ │
│   │   │ DISCONNECT - 세트에서 분리                      │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   예시:                                                  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ // 학과 CS-01의 첫 번째 학생 찾기               │    │ │
│   │   │ FIND 학과 WHERE 학과코드 = 'CS-01'             │    │ │
│   │   │ FIND FIRST 학생 WITHIN 학과-학생세트           │    │ │
│   │   │ GET 학생                                       │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 망형 모델 장단점

```
┌─────────────────────────────────────────────────────────────────┐
│           망형 모델 장단점                                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   장점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① N:M 관계 표현: 계층형보다 유연한 관계 표현            │ │
│   │   ② 다중 부모: 한 레코드가 여러 부모 가능                 │ │
│   │   ③ 데이터 중복 감소: 복제 없이 관계 표현                 │ │
│   │   ④ 접근 경로 다양: 여러 경로로 데이터 접근               │ │
│   │   ⑤ 성능: 포인터 기반 빠른 접근                          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   단점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 복잡성: 구조와 사용이 복잡                           │ │
│   │      • 프로그래머가 접근 경로 알아야 함                  │ │
│   │                                                         │ │
│   │   ② 프로그래밍 부담:                                     │ │
│   │      • 네비게이션형 프로그래밍                           │ │
│   │      • 포인터 관리 필요                                  │ │
│   │                                                         │ │
│   │   ③ 구조 변경 어려움:                                    │ │
│   │      • 스키마 수정 시 프로그램 재작성                    │ │
│   │                                                         │ │
│   │   ④ 표준화 부족:                                         │ │
│   │      • 구현마다 차이 존재                                │ │
│   │                                                         │ │
│   │   ⑤ 학습 곡선:                                          │ │
│   │      • 개념 이해와 사용이 어려움                         │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   계층형 vs 망형 비교:                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분     │ 계층형 모델         │ 망형 모델              │ │
│   ├───────────┼─────────────────────┼───────────────────────┤ │
│   │ 구조      │ 트리 (단일 경로)    │ 그래프 (다중 경로)    │ │
│   │ 관계      │ 1:N만 가능          │ N:M 가능              │ │
│   │ 부모      │ 단일 부모           │ 다중 부모 가능        │ │
│   │ 데이터    │ 중복 발생 가능      │ 중복 감소             │ │
│   │ 복잡도    │ 상대적 단순         │ 복잡                  │ │
│   └───────────┴─────────────────────┴───────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **망형 모델**: 그래프 구조로 N:M 관계 표현 가능
> **Set Type**: Owner(1) - Member(N) 관계 정의
> **CODASYL DBTG**: 망형 모델의 표준화 기구

### 2. 빈출 키워드

- **Graph Structure**: 그래프 구조, 사이클 가능
- **N:M Relationship**: 다대다 관계 표현
- **Set Type (Owner/Member)**: 세트 타입의 구성
- **CODASYL DML**: FIND, GET, STORE, CONNECT

### 3. 예상 문제

**Q1. 계층형 데이터 모델과 망형 데이터 모델의 차이점을 비교 설명하시오.**

**Q2. 망형 모델에서 N:M 관계를 표현하는 방법을 기술하시오.**

**Q3. 망형 데이터 모델의 장단점과 현대 DBMS에서의 활용을 설명하시오.**
