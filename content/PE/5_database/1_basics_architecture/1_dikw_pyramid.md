# DIKW 피라미드 (Data-Information-Knowledge-Wisdom)

## 정의

**DIKW 피라미드**는 데이터(Data), 정보(Information), 지식(Knowledge), 지혜(Wisdom)의 4단계 계층 구조를 통해 데이터의 가치 창출 과정을 체계적으로 설명하는 정보 과학의 핵심 모델이다. 1989년 Russell Ackoff가 체계화하였으며, 데이터베이스 및 정보 시스템 분야에서 데이터의 수집부터 의사결정 지원까지의 가치 사슬을 정의하는 이론적 틀로 활용된다.

```
                    +-------+
                    | Wisdom|  ← 판단, 통찰, 미래 예측
                    +-------+
                  /           \
               +---------------+
               |   Knowledge   |  ← 맥락, 패턴, 규칙
               +---------------+
             /                   \
          +-----------------------+
          |      Information      |  ← 의미, 관계, 구조
          +-----------------------+
        /                           \
     +-------------------------------+
     |            Data               |  ← 원시, 사실, 기록
     +-------------------------------+
```

## 핵심 개념

### 1. 데이터 (Data) - 피라미드 기저

```
┌──────────────────────────────────────────────────────────────┐
│                        DATA (데이터)                         │
├──────────────────────────────────────────────────────────────┤
│  정의: 객관적 사실이나 관찰 결과의 원시 형태                  │
│                                                              │
│  특성:                                                       │
│    - 맥락(Context) 부재                                      │
│    - 독자적 의미 부재                                        │
│    - 정성적/정량적 기호                                      │
│    - 가공되지 않은 원재료                                    │
│                                                              │
│  예시: 37.5, "2024-03-15", "Seoul", A001                    │
│        (숫자, 날짜, 문자열, 코드 등)                         │
└──────────────────────────────────────────────────────────────┘
```

**데이터의 분류 체계**

```
                    ┌─────────────────────┐
                    │       Data          │
                    └─────────────────────┘
                              │
           ┌──────────────────┼──────────────────┐
           ▼                  ▼                  ▼
    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
    │  Structured │    │Semi-struct  │    │ Unstructured│
    │   (정형)    │    │  (반정형)   │    │  (비정형)   │
    └─────────────┘    └─────────────┘    └─────────────┘
           │                  │                  │
    - RDB 테이블        - JSON, XML         - 텍스트
    - 스프레드시트       - YAML, CSV         - 이미지
    - 고정 스키마       - 로그 파일          - 오디오/비디오
                                           - 센서 신호
```

### 2. 정보 (Information) - 제2계층

```
┌──────────────────────────────────────────────────────────────┐
│                    INFORMATION (정보)                        │
├──────────────────────────────────────────────────────────────┤
│  정의: 데이터에 맥락(Context)과 의미(Meaning)를 부여한 것    │
│                                                              │
│  변환 과정:                                                  │
│    Data + Context + Relationship = Information               │
│                                                              │
│  핵심 질문: Who, What, Where, When                           │
│                                                              │
│  예시:                                                       │
│    - "홍길동의 체온은 37.5°C이다"                           │
│    - "2024년 3월 15일 서울 지역 매출: 1,000만 원"           │
│    - "제품 A001의 재고: 50개"                                │
└──────────────────────────────────────────────────────────────┘
```

**데이터 → 정보 변환 메커니즘**

```
     Raw Data              Processing              Information
  ┌────────────┐        ┌────────────┐         ┌────────────┐
  │    37.5    │        │            │         │ 환자 체온   │
  │ 2024-03-15 │  ───►  │  Context   │  ───►   │ = 37.5°C   │
  │   Seoul    │        │  Mapping   │         │ 측정일:     │
  │   A001     │        │            │         │ 2024-03-15 │
  └────────────┘        └────────────┘         └────────────┘

  가공 기법:
  ┌────────────────────────────────────────────────────┐
  │ 1. 분류 (Classification) - 카테고리 그룹핑        │
  │ 2. 정렬 (Sorting) - 순서 부여                     │
  │ 3. 집계 (Aggregation) - 요약 통계                 │
  │ 4. 필터링 (Filtering) - 조건부 추출               │
  │ 5. 연관 (Association) - 관계 형성                 │
  └────────────────────────────────────────────────────┘
```

### 3. 지식 (Knowledge) - 제3계층

```
┌──────────────────────────────────────────────────────────────┐
│                     KNOWLEDGE (지식)                         │
├──────────────────────────────────────────────────────────────┤
│  정의: 정보 간의 관계, 패턴, 규칙을 파악하여 구조화한 것     │
│                                                              │
│  변환 과정:                                                  │
│    Information + Pattern + Rule + Context = Knowledge        │
│                                                              │
│  핵심 질문: How, Why                                         │
│                                                              │
│  예시:                                                       │
│    - "체온 37.5°C는 미열 범위로, 감기 초기 증상일 수 있다"  │
│    - "서울 지역 3월 매출은 전년 대비 15% 증가하는 경향"     │
│    - "제품 A001은 재주문 주기가 30일이다"                   │
└──────────────────────────────────────────────────────────────┘
```

**지식의 구조화 모델**

```
    ┌─────────────────────────────────────────────────────┐
    │                  KNOWLEDGE HIERARCHY                │
    └─────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
  ┌───────────┐        ┌───────────┐        ┌───────────┐
  │ Explicit  │        │  Tacit    │        │ Cultural  │
  │ Knowledge │        │ Knowledge │        │ Knowledge │
  │ (형식지)  │        │  (암묵지) │        │ (문화지)  │
  └───────────┘        └───────────┘        └───────────┘
        │                     │                     │
  - 문서화 가능          - 경험 기반            - 조직 고유
  - 전수 용이            - 개인화됨             - 관행/규범
  - DB 저장 가능        - 암묵적 이해           - 공유 가치
  - 매뉴얼, 규정        - 노하우, 감           - 기업 문화

  ┌─────────────────────────────────────────────────────┐
  │              SECI Model (Nonaka & Takeuchi)         │
  ├─────────────────────────────────────────────────────┤
  │                                                     │
  │    Socialization      Externalization               │
  │    (암묵지→암묵지)    (암묵지→형식지)               │
  │          ▲                   │                      │
  │          │                   ▼                      │
  │    Internalization    Combination                   │
  │    (형식지→암묵지)    (형식지→형식지)               │
  │                                                     │
  └─────────────────────────────────────────────────────┘
```

### 4. 지혜 (Wisdom) - 피라미드 정점

```
┌──────────────────────────────────────────────────────────────┐
│                      WISDOM (지혜)                           │
├──────────────────────────────────────────────────────────────┤
│  정의: 지식을 바탕으로 미래를 예측하고 최적의 의사결정을    │
│        내리는 통찰력과 판단력                                │
│                                                              │
│  변환 과정:                                                  │
│    Knowledge + Judgment + Morality + Foresight = Wisdom      │
│                                                              │
│  핵심 질문: What is best? What should we do?                 │
│                                                              │
│  예시:                                                       │
│    - "환자의 미열 증상을 고려하여 휴식과 수분 섭취 권장,    │
│       증상 악화 시 병원 방문 권고"                           │
│    - "3월 매출 증가 패턴을 활용하여 충분한 재고 확보 및     │
│       마케팅 캠페인 집행 결정"                               │
└──────────────────────────────────────────────────────────────┘
```

**지혜의 의사결정 프레임워크**

```
    ┌─────────────────────────────────────────────────────┐
    │            WISDOM-BASED DECISION MAKING             │
    └─────────────────────────────────────────────────────┘
                              │
         ┌────────────────────┼────────────────────┐
         ▼                    ▼                    ▼
  ┌─────────────┐      ┌─────────────┐      ┌─────────────┐
  │  Predictive │      │  Prescript  │      │   Ethical   │
  │  Insight    │      │  Action     │      │  Judgment   │
  └─────────────┘      └─────────────┘      └─────────────┘
         │                    │                    │
  - 미래 시나리오        - 행동 규범           - 도덕적 판단
  - 위험 평가           - 실행 계획           - 사회적 책임
  - 기회 식별           - 자원 배분           - 지속가능성
```

### DIKW와 데이터베이스 설계 원칙

```
┌─────────────────────────────────────────────────────────────────────┐
│                DIKW-DRIVEN DATABASE DESIGN                          │

   └───────────────────────────────────────────────────────────┘

   ┌───────────────────────────────────────────────────────────┐

   │ • 메타데이터 관리 체계 구축                               │
   │ • 데이터 품질 관리 (DQM)                                  │
   │ • 표준화 및 분류 체계 정의                                │
   └───────────────────────────────────────────────────────────┘

3. 지식 계층 설계 원칙
   ┌───────────────────────────────────────────────────────────┐
   │ • 비즈니스 규칙 저장소 (Rule Repository)                  │
   │ • 지식 그래프 (Knowledge Graph) 구축                      │
   │ • 머신러닝 모델 관리 (MLOps)                              │
   │ • 패턴 분석 파이프라인                                    │
   └───────────────────────────────────────────────────────────┘

4. 지혜 계층 설계 원칙
   ┌───────────────────────────────────────────────────────────┐
   │ • 의사결정 지원 시스템 (DSS) 아키텍처                     │
   │ • 전문가 시스템 (Expert System) 연동                      │
   │ • 자율 의사결정 엔진                                      │
   │ • 윤리적 AI 거버넌스 프레임워크                           │
   └───────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 핵심 암기 사항

```
┌─────────────────────────────────────────────────────────────────────┐
│                    DIKW PYRAMID - KEY EXAM POINTS                   │
└─────────────────────────────────────────────────────────────────────┘

1. 계층별 정의 (4단계 암기)
   ┌───────────────────────────────────────────────────────────────┐
   │ • Data: 객관적 사실, 맥락 없는 원시 자료                     │
   │ • Information: Data + Context + Meaning                       │
   │ • Knowledge: Information + Pattern + Rule + Structure         │
   │ • Wisdom: Knowledge + Judgment + Foresight + Ethics           │
   └───────────────────────────────────────────────────────────────┘

2. 계층별 핵심 질문
   ┌───────────────────────────────────────────────────────────────┐
   │ • Data: "What?" - 사실 자체                                   │
   │ • Information: "Who, What, Where, When?" - 맥락               │
   │ • Knowledge: "How?" - 방법과 원리                             │
   │ • Wisdom: "Why? What is best?" - 판단과 선택                  │
   └───────────────────────────────────────────────────────────────┘

3. 가치 특성 (역삼각형)
   ┌───────────────────────────────────────────────────────────────┐
   │ • 상향식: Volume 감소, Value 증가, Scarcity 증가              │
   │ • 하향식: Volume 증가, Value 감소, Processing 용이성 증가     │
   └───────────────────────────────────────────────────────────────┘

4. IT 시스템 매핑
   ┌───────────────────────────────────────────────────────────────┐
   │ • Data Layer: DBMS, Data Lake                                │
   │ • Information Layer: DW, BI, OLAP                            │
   │ • Knowledge Layer: Data Mining, ML, KM System                │
   │ • Wisdom Layer: AI, DSS, Expert System                       │
   └───────────────────────────────────────────────────────────────┘

5. SECI 모델 (지식 경영)
   ┌───────────────────────────────────────────────────────────────┐
   │ • Socialization: 암묵지 → 암묵지 (경험 공유)                  │
   │ • Externalization: 암묵지 → 형식지 (지식 표현)                │
   │ • Combination: 형식지 → 형식지 (지식 체계화)                  │
   │ • Internalization: 형식지 → 암묵지 (학습/체득)                │
   └───────────────────────────────────────────────────────────────┘
```
