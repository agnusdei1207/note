# 데이터 모델 (Data Model)

## 정의

**데이터 모델(Data Model)**은 현실 세계의 정보를 **데이터베이스로 표현하는 방법**을 정의한 추상화된 모델이다. 데이터 구조(Structure), 연산(Operation), 제약조건(Constraint)의 세 가지 구성 요소를 가지며, 데이터베이스 설계의 기초가 된다.

```
┌─────────────────────────────────────────────────────────────────┐
│           데이터 모델 개요                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   현실 세계 ──▶ 개념 세계 ──▶ 논리 세계 ──▶ 물리 세계   │ │
│   │                   │             │             │          │ │
│   │                   ▼             ▼             ▼          │ │
│   │              개념적 모델   논리적 모델   물리적 모델      │ │
│   │              (ER 모델)    (관계형)     (저장 구조)       │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   데이터 모델의 3대 구성 요소:                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐  │ │
│   │   │   구조      │   │   연산      │   │  제약조건   │  │ │
│   │   │ (Structure) │   │ (Operation) │   │ (Constraint)│  │ │
│   │   └─────────────┘   └─────────────┘   └─────────────┘  │ │
│   │                                                         │ │
│   │   • 구조: 데이터 타입, 관계, 스키마                       │ │
│   │   • 연산: 데이터 조작 방법 (검색, 삽입, 수정, 삭제)        │ │
│   │   • 제약조건: 무결성 규칙, 제약 사항                      │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 데이터 모델의 3대 구성 요소

```
┌─────────────────────────────────────────────────────────────────┐
│           데이터 모델 구성 요소 상세                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 구조 (Structure):                                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 정적 성질: 데이터 구조 자체                   │    │ │
│   │   │ • 데이터 타입: 정수, 문자열, 날짜 등           │    │ │
│   │   │ • 데이터 구조: 테이블, 튜플, 속성              │    │ │
│   │   │ • 관계: 개체 간의 연관성                       │    │ │
│   │   │ • 스키마: 데이터베이스의 논리적 구조           │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ② 연산 (Operation):                                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 동적 성질: 데이터 조작                        │    │ │
│   │   │ • 검색(Retrieval): SELECT                      │    │ │
│   │   │ • 삽입(Insertion): INSERT                      │    │ │
│   │   │ • 삭제(Deletion): DELETE                       │    │ │
│   │   │ • 수정(Update): UPDATE                         │    │ │
│   │   │ • 관계형 대수: Selection, Projection, Join     │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ③ 제약조건 (Constraint):                               │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 데이터 무결성 보장                            │    │ │
│   │   │ • 개체 무결성: 기본키 NOT NULL, UNIQUE         │    │ │
│   │   │ • 참조 무결성: 외래키 규칙                      │    │ │
│   │   │ • 도메인 무결성: 데이터 타입, 범위             │    │ │
│   │   │ • 사용자 정의 무결성: CHECK, TRIGGER           │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 데이터 모델 종류

```
┌─────────────────────────────────────────────────────────────────┐
│           데이터 모델 분류                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   개념적 데이터 모델 (Conceptual):                       │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 사용자 중심의 고수준 모델                     │    │ │
│   │   │ • E-R 모델 (Entity-Relationship)               │    │ │
│   │   │ • 개체, 속성, 관계로 표현                       │    │ │
│   │   │ • 데이터베이스 설계 초기 단계                   │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   논리적 데이터 모델 (Logical):                          │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • DBMS 구현 관점의 모델                        │    │ │
│   │   │ • 관계형 모델 (Relational Model)               │    │ │
│   │   │ • 계층형 모델 (Hierarchical Model)             │    │ │
│   │   │ • 망형 모델 (Network Model)                    │    │ │
│   │   │ • 객체지향 모델 (Object-Oriented Model)        │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   물리적 데이터 모델 (Physical):                         │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 저장 장치 관점의 모델                        │    │ │
│   │   │ • 레코드 포맷, 인덱스 구조                      │    │ │
│   │   │ • 파티셔닝, 클러스터링                          │    │ │
│   │   │ • 성능 최적화 중심                              │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   추상화 수준:                                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   높음   개념적 모델 ──▶ 논리적 모델 ──▶ 물리적 모델  낮음│ │
│   │         (What)          (How)          (Where)          │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 관계형 모델 상세

```
┌─────────────────────────────────────────────────────────────────┐
│           관계형 데이터 모델 (Relational Model)                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   구조:                                                  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 릴레이션(Relation): 테이블                   │    │ │
│   │   │ • 튜플(Tuple): 행(Row)                         │    │ │
│   │   │ • 속성(Attribute): 컬럼(Column)                │    │ │
│   │   │ • 도메인(Domain): 속성 값의 집합               │    │ │
│   │   │ • 스키마(Schema): 릴레이션 구조                 │    │ │
│   │   │ • 인스턴스(Instance): 실제 데이터              │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   연산 (관계 대수):                                       │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 일반 집합 연산:                                 │    │ │
│   │   │ • 합집합(∪), 교집합(∩), 차집합(-), 카티션곱(×)│    │ │
│   │   │                                                 │    │ │
│   │   │ 순수 관계 연산:                                 │    │ │
│   │   │ • 선택(σ): 행 선택                              │    │ │
│   │   │ • 투영(π): 열 선택                              │    │ │
│   │   │ • 조인(⋈): 관계 연결                            │    │ │
│   │   │ • 디비전(÷): 나눗셈                             │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   제약조건:                                               │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 키 제약: 기본키, 후보키, 대체키, 외래키       │    │ │
│   │   │ • 개체 무결성: 기본키는 NULL 불가              │    │ │
│   │   │ • 참조 무결성: 외래키는 부모 키 또는 NULL      │    │ │
│   │   │ • 도메인 무결성: 속성값은 도메인 내 값         │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   개념적: E-R 다이어그램, 개체-관계도                    │ │
│   │   논리적: 릴레이션 스키마, 정규화된 테이블               │ │

## 기술사 시험 포인트

### 1. 핵심 정의

> **데이터 모델**: 현실 세계를 DB로 표현하는 추상화 모델
> **3대 구성요소**: 구조(Structure), 연산(Operation), 제약조건(Constraint)
> **관계형 모델**: 테이블 기반의 가장 널리 사용되는 논리적 모델

### 2. 빈출 키워드

- **Structure/Operation/Constraint**: 데이터 모델 3대 구성요소
- **Conceptual/Logical/Physical**: 개념/논리/물리적 모델
- **Relational Model**: 관계형 모델, E.F. Codd 제안
- **Relational Algebra**: 관계 대수 (선택, 투영, 조인)

### 3. 예상 문제

**Q1. 데이터 모델의 3대 구성 요소를 설명하고 각각의 역할을 기술하시오.**

**Q2. 개념적, 논리적, 물리적 데이터 모델의 차이점을 비교 설명하시오.**

**Q3. 관계형 데이터 모델의 구조와 제약조건을 설명하시오.**
