# DBMS 언어 (DDL, DML, DCL, TCL)

## 정의

**DBMS 언어**는 데이터베이스를 정의, 조작, 제어하기 위한 **SQL(Structured Query Language) 명령어들**을 분류한 것이다. DDL(정의), DML(조작), DCL(제어), TCL(트랜잭션 제어)로 구분된다.

```
┌─────────────────────────────────────────────────────────────────┐
│           DBMS 언어 분류                                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │                     SQL (Structured Query Language)      │ │
│   │                              │                           │ │
│   │        ┌─────────┬──────────┼──────────┬─────────┐      │ │
│   │        ▼         ▼          ▼          ▼         ▼      │ │
│   │   ┌────────┐┌────────┐┌────────┐┌────────┐┌────────┐   │ │
│   │   │  DDL   ││  DML   ││  DCL   ││  TCL   ││  DQL   │   │ │
│   │   │데이터  ││데이터  ││데이터  ││트랜잭션││데이터  │   │ │
│   │   │정의어  ││조작어  ││제어어  ││제어어  ││질의어  │   │ │
│   │   └────────┘└────────┘└────────┘└────────┘└────────┘   │ │
│   │                                                         │ │
│   │   DDL: CREATE, ALTER, DROP, TRUNCATE                    │ │
│   │   DML: INSERT, UPDATE, DELETE, SELECT                   │ │
│   │   DCL: GRANT, REVOKE                                    │ │
│   │   TCL: COMMIT, ROLLBACK, SAVEPOINT                      │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. DDL (Data Definition Language)

```
┌─────────────────────────────────────────────────────────────────┐
│           DDL - 데이터 정의 언어                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   CREATE: 객체 생성                                      │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   -- 테이블 생성                                │    │ │
│   │   │   CREATE TABLE 학생 (                          │    │ │
│   │   │       학번   VARCHAR(10) PRIMARY KEY,          │    │ │
│   │   │       이름   VARCHAR(20) NOT NULL,             │    │ │
│   │   │       학과   VARCHAR(20),                      │    │ │
│   │   │       학년   INT CHECK (학년 BETWEEN 1 AND 4)  │    │ │
│   │   │   );                                           │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 인덱스 생성                                │    │ │
│   │   │   CREATE INDEX idx_학과 ON 학생(학과);          │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 뷰 생성                                    │    │ │
│   │   │   CREATE VIEW 컴퓨터학생 AS                     │    │ │
│   │   │       SELECT 학번, 이름 FROM 학생               │    │ │
│   │   │       WHERE 학과 = '컴퓨터';                    │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ALTER: 객체 수정                                       │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   -- 컬럼 추가                                  │    │ │
│   │   │   ALTER TABLE 학생 ADD 평점 DECIMAL(3,2);      │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 컬럼 수정                                  │    │ │
│   │   │   ALTER TABLE 학생 MODIFY 이름 VARCHAR(30);    │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 컬럼 삭제                                  │    │ │
│   │   │   ALTER TABLE 학생 DROP COLUMN 평점;           │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 제약조건 추가                              │    │ │
│   │   │   ALTER TABLE 학생                              │    │ │
│   │   │       ADD CONSTRAINT fk_학과                    │    │ │
│   │   │       FOREIGN KEY (학과) REFERENCES 학과(코드); │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   DROP: 객체 삭제                                        │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   DROP TABLE 학생;        -- 테이블 삭제       │    │ │
│   │   │   DROP INDEX idx_학과;    -- 인덱스 삭제       │    │ │
│   │   │   DROP VIEW 컴퓨터학생;   -- 뷰 삭제           │    │ │
│   │   │                                                 │    │ │
│   │   │   💡 DROP은 객체와 데이터 모두 삭제              │    │ │
│   │   │      자동 COMMIT (DDL은 암시적 커밋)             │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   TRUNCATE: 데이터 삭제 (구조 유지)                       │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   TRUNCATE TABLE 학생;                         │    │ │
│   │   │                                                 │    │ │
│   │   │   특징:                                         │    │ │
│   │   │   • 모든 데이터 삭제                            │    │ │
│   │   │   • 테이블 구조 유지                             │    │ │
│   │   │   • DELETE보다 빠름 (로깅 최소화)               │    │ │
│   │   │   • 롤백 불가능                                  │    │ │
│   │   │   • 자동 COMMIT                                 │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. DML (Data Manipulation Language)

```
┌─────────────────────────────────────────────────────────────────┐
│           DML - 데이터 조작 언어                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   INSERT: 데이터 삽입                                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   -- 단일 행 삽입                               │    │ │
│   │   │   INSERT INTO 학생 (학번, 이름, 학과, 학년)      │    │ │
│   │   │   VALUES ('1001', '홍길동', '컴퓨터', 3);       │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 모든 컬럼 지정 시 컬럼명 생략 가능          │    │ │
│   │   │   INSERT INTO 학생                              │    │ │
│   │   │   VALUES ('1002', '김영희', '전자', 2);         │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 서브쿼리를 이용한 다중 행 삽입              │    │ │
│   │   │   INSERT INTO 학생                              │    │ │
│   │   │   SELECT * FROM 예비학생 WHERE 합격 = 'Y';       │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   UPDATE: 데이터 수정                                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   -- 단일 컬럼 수정                              │    │ │
│   │   │   UPDATE 학생 SET 학년 = 4                      │    │ │
│   │   │   WHERE 학번 = '1001';                          │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 다중 컬럼 수정                              │    │ │
│   │   │   UPDATE 학생                                   │    │ │
│   │   │   SET 학년 = 4, 학과 = '컴퓨터'                  │    │ │
│   │   │   WHERE 학번 = '1001';                          │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 서브쿼리 이용 수정                          │    │ │
│   │   │   UPDATE 학생 SET 학과 =                        │    │ │
│   │   │       (SELECT 코드 FROM 학과 WHERE 이름='AI')    │    │ │
│   │   │   WHERE 학번 = '1001';                          │    │ │
│   │   │                                                 │    │ │
│   │   │   💡 WHERE 생략 시 모든 행 수정 (주의!)           │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   DELETE: 데이터 삭제                                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   -- 조건부 삭제                                │    │ │
│   │   │   DELETE FROM 학생                              │    │ │
│   │   │   WHERE 학번 = '1001';                          │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 전체 삭제                                   │    │ │
│   │   │   DELETE FROM 학생;                             │    │ │
│   │   │                                                 │    │ │
│   │   │   DELETE vs TRUNCATE:                           │    │ │
│   │   │   ┌─────────────────────────────────────┐     │    │ │
│   │   │   │ DELETE        │ TRUNCATE           │     │    │ │
│   │   │   ├───────────────┼────────────────────┤     │    │ │
│   │   │   │ DML          │ DDL                │     │    │ │
│   │   │   │ ROLLBACK 가능│ ROLLBACK 불가       │     │    │ │
│   │   │   │ WHERE 가능   │ WHERE 불가          │     │    │ │
│   │   │   │ 느림         │ 빠름               │     │    │ │
│   │   │   │ 로깅 O       │ 로깅 최소화         │     │    │ │
│   │   │   └─────────────────────────────────────┘     │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   SELECT: 데이터 조회 (DQL)                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   SELECT [DISTINCT] 컬럼명                      │    │ │
│   │   │   FROM 테이블명                                 │    │ │
│   │   │   [WHERE 조건]                                  │    │ │
│   │   │   [GROUP BY 컬럼명]                             │    │ │
│   │   │   [HAVING 그룹조건]                             │    │ │
│   │   │   [ORDER BY 컬럼명 [ASC|DESC]]                  │    │ │
│   │   │   [LIMIT n];                                    │    │ │
│   │   │                                                 │    │ │
│   │   │   예:                                           │    │ │
│   │   │   SELECT 학과, COUNT(*) as 학생수               │    │ │
│   │   │   FROM 학생                                     │    │ │
│   │   │   WHERE 학년 >= 2                               │    │ │
│   │   │   GROUP BY 학과                                 │    │ │
│   │   │   HAVING COUNT(*) >= 5                          │    │ │
│   │   │   ORDER BY 학생수 DESC;                         │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. DCL (Data Control Language)

```
┌─────────────────────────────────────────────────────────────────┐
│           DCL - 데이터 제어 언어                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   GRANT: 권한 부여                                        │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   -- 시스템 권한 부여                            │    │ │
│   │   │   GRANT CREATE SESSION TO 사용자1;              │    │ │
│   │   │   GRANT CREATE TABLE TO 사용자1;                │    │ │
│   │   │   GRANT CREATE ANY TABLE TO 사용자1;            │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 객체 권한 부여                              │    │ │
│   │   │   GRANT SELECT, INSERT ON 학생 TO 사용자1;       │    │ │
│   │   │   GRANT ALL PRIVILEGES ON 학생 TO 사용자1;       │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 권한 전달 (WITH GRANT OPTION)              │    │ │
│   │   │   GRANT SELECT ON 학생 TO 사용자1               │    │ │
│   │   │       WITH GRANT OPTION;                        │    │ │
│   │   │       -- 사용자1이 다른 사용자에게 권한 부여 가능  │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 역할(Role) 이용                             │    │ │
│   │   │   CREATE ROLE 교수;                             │    │ │
│   │   │   GRANT SELECT, UPDATE ON 성적 TO 교수;          │    │ │
│   │   │   GRANT 교수 TO 사용자1;                        │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   REVOKE: 권한 회수                                       │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   -- 객체 권한 회수                              │    │ │
│   │   │   REVOKE SELECT, INSERT ON 학생 FROM 사용자1;    │    │ │
│   │   │   REVOKE ALL PRIVILEGES ON 학생 FROM 사용자1;    │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 시스템 권한 회수                            │    │ │
│   │   │   REVOKE CREATE TABLE FROM 사용자1;             │    │ │
│   │   │                                                 │    │ │
│   │   │   -- CASCADE vs RESTRICT                       │    │ │
│   │   │   REVOKE SELECT ON 학생 FROM 사용자1 CASCADE;   │    │ │
│   │   │   -- CASCADE: 사용자1이 부여한 권한도 회수        │    │ │
│   │   │   -- RESTRICT: 다른 사용자에게 부여된 권한이      │    │ │
│   │   │   --         있으면 회수 실패                    │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   주요 권한 종류:                                         │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   시스템 권한          │ 객체 권한              │    │ │
│   │   │   ────────────────────┼───────────────────     │    │ │
│   │   │   CREATE SESSION      │ SELECT               │    │ │
│   │   │   CREATE TABLE        │ INSERT               │    │ │
│   │   │   CREATE VIEW         │ UPDATE               │    │ │
│   │   │   CREATE PROCEDURE    │ DELETE               │    │ │
│   │   │   CREATE SEQUENCE     │ EXECUTE              │    │ │
│   │   │   ALTER ANY TABLE     │ REFERENCES           │    │ │
│   │   │   DROP ANY TABLE      │ INDEX                │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. TCL (Transaction Control Language)

```
┌─────────────────────────────────────────────────────────────────┐
│           TCL - 트랜잭션 제어 언어                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   트랜잭션 특성 (ACID):                                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   A (Atomicity): 원자성                        │    │ │
│   │   │      모두 성공 또는 모두 실패                   │    │ │
│   │   │                                                 │    │ │
│   │   │   C (Consistency): 일관성                      │    │ │
│   │   │      트랜잭션 전후 DB 상태 일관                  │    │ │
│   │   │                                                 │    │ │
│   │   │   I (Isolation): 격리성                        │    │ │
│   │   │      동시 실행 트랜잭션 간 간섭 없음             │    │ │
│   │   │                                                 │    │ │
│   │   │   D (Durability): 지속성                       │    │ │
│   │   │      커밋된 트랜잭션은 영구히 저장               │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   COMMIT: 트랜잭션 확정                                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   START TRANSACTION;                           │    │ │
│   │   │   UPDATE 계좌 SET 잔액 = 잔액 - 10000          │    │ │
│   │   │   WHERE 계좌번호 = 'A';                        │    │ │
│   │   │   UPDATE 계좌 SET 잔액 = 잔액 + 10000          │    │ │
│   │   │   WHERE 계좌번호 = 'B';                        │    │ │
│   │   │   COMMIT;  -- 변경사항 영구 반영                │    │ │
│   │   │                                                 │    │ │
│   │   │   💡 COMMIT 후에는 ROLLBACK 불가                 │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ROLLBACK: 트랜잭션 취소                                 │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   START TRANSACTION;                           │    │ │
│   │   │   DELETE FROM 학생 WHERE 학과 = '컴퓨터';       │    │ │
│   │   │   -- 실수 발견!                                 │    │ │
│   │   │   ROLLBACK;  -- 변경사항 취소                   │    │ │
│   │   │                                                 │    │ │
│   │   │   💡 마지막 COMMIT 이후로 되돌림                  │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   SAVEPOINT: 부분 복구 지점                               │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   START TRANSACTION;                           │    │ │
│   │   │   INSERT INTO 주문 VALUES (1, ...);            │    │ │
│   │   │   SAVEPOINT sp1;                               │    │ │

│   │   │   SAVEPOINT sp2;                               │    │ │
│   │   │   INSERT INTO 주문상세 VALUES (2, ...);         │    │ │
│   │   │                                                 │    │ │
│   │   │   -- sp2로 롤백 (마지막 INSERT만 취소)           │    │ │
│   │   │   ROLLBACK TO SAVEPOINT sp2;                   │    │ │
│   │   │                                                 │    │ │
│   │   │   -- 전체 롤백                                  │    │ │
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **DDL**: 데이터 구조 정의 (CREATE, ALTER, DROP, TRUNCATE)
> **DML**: 데이터 조작 (SELECT, INSERT, UPDATE, DELETE)
> **DCL**: 권한 제어 (GRANT, REVOKE)
> **TCL**: 트랜잭션 제어 (COMMIT, ROLLBACK, SAVEPOINT)

### 2. 빈출 키워드

- **DDL 암시적 커밋**: DDL 실행 시 자동 COMMIT
- **DELETE vs TRUNCATE**: DML vs DDL, 롤백 가능 여부
- **GRANT/REVOKE**: WITH GRANT OPTION, CASCADE
- **ACID**: 원자성, 일관성, 격리성, 지속성

### 3. 예상 문제

**Q1. SQL 명령어를 DDL, DML, DCL, TCL로 분류하고 각각의 특징을 설명하시오.**

**Q2. DELETE와 TRUNCATE의 차이점을 비교하시오.**

**Q3. 트랜잭션의 ACID 특성을 설명하고 COMMIT, ROLLBACK, SAVEPOINT의 사용법을 기술하시오.**
