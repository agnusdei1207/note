# 관계형 모델 (Relational Model)

## 정의

**관계형 모델(Relational Model)**은 데이터를 **2차원 테이블(릴레이션)** 형태로 표현하는 데이터베이스 모델이다. 1970년 E.F. Codd가 제안했으며, 수학적 집합론과 관계 대수에 기반한다. 현재 가장 널리 사용되는 데이터베이스 모델이다.

```
┌─────────────────────────────────────────────────────────────────┐
│           관계형 모델 기본 개념                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   릴레이션 (Relation) = 테이블 (Table):                  │ │
│   │                                                         │ │
│   │   ┌─────────────────────────────────────────────────┐  │ │
│   │   │  학생 (Student)                                  │  │ │
│   │   ├──────┬────────┬────────┬────────┬─────────────┤  │ │
│   │   │ 학번 │  이름  │  학과  │ 학년   │   평점      │  │ │
│   │   ├──────┼────────┼────────┼────────┼─────────────┤  │ │
│   │   │ 1001 │ 홍길동 │ 컴퓨터 │  3     │   3.8       │  │ │ ← 튜플(Tuple)
│   │   │ 1002 │ 김영희 │ 전자   │  2     │   4.0       │  │ │
│   │   │ 1003 │ 이철수 │ 컴퓨터 │  4     │   3.5       │  │ │
│   │   └──────┴────────┴────────┴────────┴─────────────┘  │ │
│   │        │                            │                 │ │
│   │        └── 속성(Attribute) ─────────┘                 │ │
│   │               = 컬럼(Column)                          │ │
│   │                                                        │ │
│   │   용어 대응:                                            │ │
│   │   ┌─────────────────────────────────────────────┐     │ │
│   │   │  이론적 용어      │  SQL 용어                │     │ │
│   │   │  ─────────────────┼─────────────────────     │     │ │
│   │   │  Relation         │  Table                  │     │ │
│   │   │  Tuple            │  Row / Record           │     │ │
│   │   │  Attribute        │  Column / Field         │     │ │
│   │   │  Domain           │  Data Type              │     │ │
│   │   │  Degree           │  Number of Columns      │     │ │
│   │   │  Cardinality      │  Number of Rows         │     │ │
│   │   └─────────────────────────────────────────────┘     │ │
│   │                                                        │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 릴레이션의 구성 요소

```
┌─────────────────────────────────────────────────────────────────┐
│           릴레이션 구성 요소                                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   스키마 (Schema) vs 인스턴스 (Instance):                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   스키마 (정적 구조):                                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ Student(학번, 이름, 학과, 학년, 평점)          │    │ │
│   │   │          │                                    │    │ │
│   │   │          └── 속성들의 집합 (고정)              │    │ │
│   │   │                                                 │    │ │
│   │   │   Degree (차수): 속성의 개수 = 5               │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   인스턴스 (동적 데이터):                                 │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 특정 시점의 튜플 집합                          │    │ │
│   │   │                                                 │    │ │
│   │   │   Cardinality (카디널리티): 튜플의 개수        │    │ │
│   │   │   시간에 따라 변함                              │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   도메인 (Domain):                                               │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   각 속성이 가질 수 있는 값의 집합:                       │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   학번 도메인: 1000~9999 (정수)                 │    │ │
│   │   │   이름 도메인: 문자열 (최대 20자)               │    │ │
│   │   │   학년 도메인: {1, 2, 3, 4}                    │    │ │
│   │   │   평점 도메인: 0.0~4.3 (실수)                  │    │ │
│   │   │                                                 │    │ │
│   │   │   💡 원자값 (Atomic Value):                    │    │ │
│   │   │      도메인의 각 값은 더 이상 분해 불가         │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 키 (Key)의 종류

```
┌─────────────────────────────────────────────────────────────────┐
│           키의 종류와 역할                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 슈퍼키 (Super Key):                                  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 튜플을 유일하게 식별하는 속성 집합              │    │ │
│   │   │                                                 │    │ │
│   │   │ 예: {학번}, {학번, 이름}, {주민번호}            │    │ │
│   │   │     {학번, 학과, 이름} 등                       │    │ │
│   │   │                                                 │    │ │
│   │   │ 💡 유일성 O, 최소성 X                          │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ② 후보키 (Candidate Key):                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 슈퍼키 중 최소성을 만족하는 것                  │    │ │
│   │   │                                                 │    │ │
│   │   │ 예: {학번}, {주민번호}                         │    │ │
│   │   │                                                 │    │ │
│   │   │ 💡 유일성 O, 최소성 O                          │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ③ 기본키 (Primary Key):                                │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 후보키 중 선택된 대표 키                       │    │ │
│   │   │                                                 │    │ │
│   │   │ • NULL 불가 (NOT NULL)                         │    │ │
│   │   │ • 고유해야 함 (UNIQUE)                         │    │ │
│   │   │ • 변경 불가 (Immutable)                        │    │ │
│   │   │                                                 │    │ │
│   │   │ 예: 학번을 기본키로 선택                        │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ④ 대체키 (Alternate Key):                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 기본키로 선택되지 않은 후보키                   │    │ │
│   │   │                                                 │    │ │
│   │   │ 예: 주민번호 (학번이 기본키인 경우)             │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ⑤ 외래키 (Foreign Key):                                │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 다른 릴레이션의 기본키를 참조하는 속성          │    │ │
│   │   │                                                 │    │ │
│   │   │   수강 (학번, 과목코드, 성적)                   │    │ │
│   │   │          │                                      │    │ │
│   │   │          └── 학생 릴레이션의 기본키 참조        │    │ │
│   │   │                                                 │    │ │
│   │   │   참조 무결성 제약조건                          │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 무결성 제약조건

```
┌─────────────────────────────────────────────────────────────────┐
│           무결성 제약조건 (Integrity Constraints)                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 개체 무결성 (Entity Integrity):                      │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   • 기본키는 NULL이 될 수 없음                  │    │ │
│   │   │   • 기본키는 유일해야 함                        │    │ │
│   │   │                                                 │    │ │
│   │   │   ┌─────────────────────────────────┐         │    │ │
│   │   │   │ 학번 (PK) │ 이름 │ 학과         │         │    │ │
│   │   │   ├───────────┼──────┼─────────────┤         │    │ │
│   │   │   │  NULL     │ 홍   │ 컴퓨터      │ ✗       │    │ │
│   │   │   │  1001     │ 김   │ 컴퓨터      │ ✓       │    │ │
│   │   │   │  1001     │ 이   │ 전자        │ ✗       │    │ │
│   │   │   └─────────────────────────────────┘         │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ② 참조 무결성 (Referential Integrity):                 │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   • 외래키는 참조하는 기본키와 일치하거나 NULL  │    │ │
│   │   │                                                 │    │ │
│   │   │   학생            수강                          │    │ │
│   │   │   ┌─────┐        ┌───────────┐                 │    │ │
│   │   │   │학번 │◀───────│학번 (FK)  │                 │    │ │
│   │   │   │ PK  │        │과목코드   │                 │    │ │
│   │   │   └─────┘        │성적       │                 │    │ │
│   │   │                   └───────────┘                 │    │ │
│   │   │                                                 │    │ │
│   │   │   위반 시 동작:                                  │    │ │
│   │   │   • CASCADE: 연쇄 삭제/수정                     │    │ │
│   │   │   • RESTRICT: 삭제/수정 금지                    │    │ │
│   │   │   • SET NULL: NULL로 설정                       │    │ │
│   │   │   • SET DEFAULT: 기본값 설정                    │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ③ 도메인 무결성 (Domain Integrity):                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   • 속성값은 정의된 도메인 내의 값이어야 함      │    │ │
│   │   │   • 데이터 타입, 범위, 형식 등                  │    │ │
│   │   │                                                 │    │ │
│   │   │   CHECK 제약조건으로 구현                       │    │ │
│   │   │   CHECK (학년 BETWEEN 1 AND 4)                 │    │ │
│   │   │   CHECK (성별 IN ('M', 'F'))                   │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 관계 대수 연산

```
┌─────────────────────────────────────────────────────────────────┐
│           관계 대수 (Relational Algebra) 연산                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   일반 집합 연산:                                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   합집합 (Union): R ∪ S                                 │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ R 또는 S(또는 둘 다)에 있는 튜플               │    │ │
│   │   │                                                 │    │ │
│   │   │   R: {a, b, c}                                 │    │ │
│   │   │   S: {b, c, d}                                 │    │ │
│   │   │   R ∪ S: {a, b, c, d}                          │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   교집합 (Intersection): R ∩ S                          │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ R과 S 모두에 있는 튜플                         │    │ │
│   │   │ R ∩ S: {b, c}                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   차집합 (Difference): R - S                            │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ R에는 있고 S에는 없는 튜플                     │    │ │
│   │   │ R - S: {a}                                    │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   카티션 곱 (Cartesian Product): R × S                  │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ R의 각 튜플과 S의 각 튜플의 모든 조합           │    │ │
│   │   │ R(m행) × S(n행) = m×n행                       │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   순수 관계 연산:                                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   선택 (Selection): σ조건(R)                            │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 조건을 만족하는 튜플 선택 (행 추출)             │    │ │
│   │   │                                                 │    │ │
│   │   │   σ학과='컴퓨터'(학생)                         │    │ │
│   │   │   → 컴퓨터과 학생만 선택                        │    │ │
│   │   │                                                 │    │ │
│   │   │   SQL: SELECT * FROM 학생 WHERE 학과='컴퓨터'  │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   투영 (Projection): π속성(R)                           │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 지정된 속성만 추출 (열 추출)                    │    │ │
│   │   │                                                 │    │ │
│   │   │   π이름,학과(학생)                             │    │ │
│   │   │   → 이름과 학과만 추출                          │    │ │
│   │   │                                                 │    │ │
│   │   │   SQL: SELECT 이름, 학과 FROM 학생              │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   조인 (Join): R ⨝ S                                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 공통 속성 기반으로 두 릴레이션 결합             │    │ │
│   │   │                                                 │    │ │
│   │   │   학생 ⨝ 수강 (학번 기준)                       │    │ │
│   │   │                                                 │    │ │
│   │   │   SQL: SELECT * FROM 학생                       │    │ │
│   │   │        JOIN 수강 ON 학생.학번 = 수강.학번       │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **관계형 모델**: 수학적 관계론 기반의 테이블 형태 데이터 모델
> **릴레이션**: 속성과 튜플로 구성된 2차원 테이블
> **무결성 제약조건**: 데이터 정확성 보장 규칙

### 2. 빈출 키워드

- **Tuple/Attribute**: 행/열 (레코드/필드)
- **Primary/Foreign Key**: 기본키/외래키
- **Entity/Referential Integrity**: 개체/참조 무결성
- **Selection/Projection/Join**: 선택/투영/조인 연산

### 3. 예상 문제

**Q1. 관계형 모델의 구성 요소와 무결성 제약조건을 설명하시오.**

**Q2. 슈퍼키, 후보키, 기본키, 외래키의 차이점을 비교하시오.**

**Q3. 관계 대수의 선택(Selection), 투영(Projection), 조인(Join) 연산을 SQL과 연계하여 설명하시오.**
