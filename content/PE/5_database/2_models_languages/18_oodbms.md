# 객체지향 DBMS (OODBMS)

## 정의

**객체지향 DBMS(Object-Oriented Database Management System)**는 **객체지향 프로그래밍의 개념을 데이터베이스에 적용**한 시스템이다. 객체, 클래스, 상속, 캡슐화, 다형성 등 OOP 특성을 지원하며, 복잡한 데이터 구조와 객체 간 관계를 자연스럽게 표현할 수 있다.

```
┌─────────────────────────────────────────────────────────────────┐
│           OODBMS 기본 개념                                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   RDBMS vs OODBMS:                                       │ │
│   │                                                         │ │
│   │   RDBMS                    OODBMS                        │ │
│   │   ┌─────────────────┐     ┌─────────────────────────┐   │ │
│   │   │ Table (테이블)   │     │ Class (클래스)          │   │ │
│   │   │ Row (행)        │     │ Object (객체)           │   │ │
│   │   │ Column (열)     │     │ Attribute (속성)        │   │ │
│   │   │ Foreign Key     │     │ Reference (참조)        │   │ │
│   │   │ JOIN            │     │ Navigation (탐색)       │   │ │
│   │   │ SQL             │     │ OQL / OML               │   │ │
│   │   └─────────────────┘     └─────────────────────────┘   │ │
│   │                                                         │ │
│   │   OODBMS 특징:                                            │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 객체를 그대로 저장 (Impedance Mismatch 없음) │    │ │
│   │   │ • 복잡한 데이터 구조 표현 용이                  │    │ │
│   │   │ • 객체 간 관계를 참조로 표현                    │    │ │
│   │   │ • 상속, 다형성 지원                             │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 객체지향 개념 매핑

```
│   │   │                                                 │    │ │
│   │   │   OID (Object Identifier):                     │    │ │
│   │   │   • 시스템 전체에서 유일한 식별자               │    │ │
│   │   │   • 객체 생성 시 자동 할당                      │    │ │

│   │   │   ┌─────────────────────────────────────┐     │    │ │

│   │   │   │  │ Private Data                   ││     │    │ │

│   │   │   │  ├─────────────────────────────────┤│     │    │ │

│   │   │   └─────────────────────────────────────┘     │    │ │
│   │   │                                                 │    │ │
│   │   │   • 데이터와 메서드를 하나로 묶음                │    │ │
```

### 2. 객체 참조와 탐색

```
┌─────────────────────────────────────────────────────────────────┐
│           객체 참조와 탐색 (Navigation)                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   참조 기반 관계:                                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   RDBMS (외래키 + JOIN):                                 │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ 학생 테이블          수강 테이블               │    │ │
│   │   │ ┌─────┬──────┐     ┌──────┬──────┬──────┐   │    │ │
│   │   │ │학번 │ 이름 │     │학번  │과목코드│성적 │   │    │ │
│   │   │ ├─────┼──────┤     ├──────┼──────┼──────┤   │    │ │
│   │   │ │1001 │홍길동│     │1001  │CS101 │ A    │   │    │ │
│   │   │ └─────┴──────┘     └──────┴──────┴──────┘   │    │ │
│   │   │        │                     ▲               │    │ │
│   │   │        └───── JOIN (학번) ────┘               │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   OODBMS (객체 참조):                                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   Student 객체          Course 객체            │    │ │
│   │   │   ┌────────────┐        ┌────────────┐       │    │ │
│   │   │   │ OID: @100  │        │ OID: @200  │       │    │ │
│   │   │   │ name: 홍   │        │ code:CS101 │       │    │ │
│   │   │   │ courses ───┼───────▶│ name: DB   │       │    │ │
│   │   │   │            │   ┌───▶│ students   │       │    │ │
│   │   │   └────────────┘   │    └────────────┘       │    │ │
│   │   │                    │                          │    │ │
│   │   │   💡 직접 참조로 탐색 (Navigation)             │    │ │
│   │   │      student.courses[0].name → "DB"          │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   탐색 질의 예시 (OQL):                                   │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   // 학생의 수강 과목 이름 조회                 │    │ │
│   │   │   SELECT c.name                                 │    │ │
│   │   │   FROM Student s, s.courses c                  │    │ │
│   │   │   WHERE s.name = '홍길동'                       │    │ │
│   │   │                                                 │    │ │
│   │   │   // 참조 탐색 방식                              │    │ │
│   │   │   student.courses.name                         │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. ODMG 표준

```
┌─────────────────────────────────────────────────────────────────┐
│           ODMG (Object Database Management Group) 표준           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ODMG 3.0 구성 요소:                                     │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   ① ODL (Object Definition Language):          │    │ │
│   │   │      • 객체 스키마 정의 언어                     │    │ │
│   │   │      • 클래스, 속성, 관계, 연산 정의             │    │ │
│   │   │                                                 │    │ │
│   │   │   ② OQL (Object Query Language):               │    │ │
│   │   │      • 객체 질의 언어                           │    │ │
│   │   │      • SQL과 유사하나 객체 탐색 지원             │    │ │
│   │   │                                                 │    │ │
│   │   │   ③ OML (Object Manipulation Language):        │    │ │
│   │   │      • 객체 조작 언어                           │    │ │
│   │   │      • C++, Java, Smalltalk 바인딩              │    │ │
│   │   │                                                 │    │ │
│   │   │   ④ OIF (Object Interchange Format):           │    │ │
│   │   │      • 객체 교환 형식                           │    │ │
│   │   │      • 데이터 덤프/로드                         │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ODL 예시:                                               │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   interface Person {                           │    │ │
│   │   │     attribute string name;                     │    │ │
│   │   │     attribute string address;                  │    │ │
│   │   │   };                                            │    │ │
│   │   │                                                 │    │ │
│   │   │   class Student extends Person {               │    │ │
│   │   │     attribute string studentId;                │    │ │
│   │   │     relationship Set<Course> courses           │    │ │
│   │   │       inverse Course::students;                │    │ │
│   │   │     float getGPA();                            │    │ │
│   │   │   };                                            │    │ │
│   │   │                                                 │    │ │
│   │   │   class Course {                                │    │ │
│   │   │     attribute string code;                     │    │ │
│   │   │     attribute string name;                     │    │ │
│   │   │     relationship Set<Student> students         │    │ │
│   │   │       inverse Student::courses;                │    │ │
│   │   │   };                                            │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. OODBMS 장단점

```
┌─────────────────────────────────────────────────────────────────┐
│           OODBMS 장단점                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │

│   ┌─────────────────────────────────────────────────────────┐ │

│   │   │ • 복잡한 ORM 없이 직접 저장                     │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ② 복잡한 데이터 모델링:                                 │ │
│   │   ┌───────────────────────────────────────────────┐    │ │

│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • JOIN 없이 참조 탐색                          │    │ │
│   │   │ • 캐싱 효율 높음                                │    │ │
│   │   │ • 특정 도메인에서 높은 성능                     │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   단점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 표준화 부족:                                          │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • SQL만큼 보편화되지 않음                       │    │ │
│   │   │ • 벤더 간 호환성 문제                           │    │ │

│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ② 학습 곡선:                                            │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 새로운 질의 언어 학습 필요                    │    │ │
│   │   │ • RDBMS와 다른 사고방식 요구                    │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ③ 생태계:                                               │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │ • 도구, 라이브러리 부족                          │    │ │
│   │   │ • 인력 확보 어려움                              │    │ │
│   │   │ • 상용 제품 선택지 제한                          │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **OODBMS**: 객체지향 개념을 적용한 DBMS
> **OID**: 객체 식별자, 유일하고 불변
> **ODMG**: OODBMS 표준화 단체 (ODL, OQL, OML)

### 2. 빈출 키워드

- **Encapsulation/Inheritance**: 캡슐화/상속 지원
- **Object Reference**: 객체 참조, JOIN 없는 탐색
- **Impedance Mismatch**: OOP-DB 매핑 문제 해결
- **OQL (Object Query Language)**: 객체 질의 언어

### 3. 예상 문제

**Q1. OODBMS와 RDBMS의 차이점을 비교하고 각각의 적합한 응용 분야를 기술하시오.**

**Q2. OODBMS의 객체지향 특성(캡슐화, 상속, 다형성)이 데이터베이스에 어떻게 적용되는지 설명하시오.**

**Q3. ODMG 표준의 구성 요소와 OQL의 특징을 설명하시오.**
