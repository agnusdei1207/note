## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 감산기(Subtractor)는 두 이진수 간의 차(Difference)를 구하고 필요시 상위 비트로부터 빌림수(Borrow)를 가져오는 산술 조합 논리회로다.
> 2. **가치**: 반감산기(Half Subtractor)와 전감산기(Full Subtractor)를 통해 수치적 '감소'를 하드웨어적으로 구현하며, 이는 논리 비교 및 조건부 분기문의 판단 근거가 된다.
> 3. **융합**: 현대 CPU 아키텍처에서는 독립적인 감산기 회로 대신, 2의 보수(2's Complement) 원리를 이용하여 가산기(Adder) 하나로 덧셈과 뺄셈을 통합 처리하는 고효율 ALU 구조를 지향한다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 감산기(Subtractor)는 이진수의 뺄셈 규칙($0-0=0, 1-0=1, 1-1=0, 0-1=1$ 빌림수 발생)을 수행하는 회로다. 뺄셈의 결과인 **차(Difference)**와 아랫자리가 부족할 때 윗자리에서 꿔오는 **빌림수(Borrow)**라는 두 개의 출력을 가진다.

- **💡 비유**: 감산기는 **"잔돈 계산기"**와 같다. 내가 가진 돈($A$)에서 물건값($B$)을 뺄 때, 돈이 모자라면($A < B$) 옆 친구에게 10원을 빌려와서(Borrow) 계산을 마무리하고 남은 돈(Difference)을 확인하는 것과 같은 원리다.

- **등장 배경: 연산 장치의 통합**:
  - 초창기에는 덧셈용 가산기와 뺄셈용 감산기를 따로 만들어 칩 면적을 많이 차지했다.
  - 하지만 "A - B는 A + (-B)와 같다"는 보수 연산법이 정립되면서, 감산기는 독립된 소자보다는 가산기의 변형된 동작 모드로 이해되기 시작했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 반감산기(Half Subtractor)와 전감산기(Full Subtractor)

감산기는 빌림수($B_{in}$)의 처리 여부에 따라 두 단계로 나뉜다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │                  감산기(Subtractor)의 논리 구조                   │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │   [반감산기 (Half)]                [전감산기 (Full)]           │
 │   A ──┬───))── D (Diff)            A ──┐  ┌─────┐              │
 │       │ ┌─┘                        B ──┴─▶│ HS1 │─ D1 ─┐ ┌─────┐│
 │   ┌─▷○┴─┐                          Bin ──▶└─ Bo1 ┘      └▶│ HS2 │─▶ D
 │   │     ]─── Bo (Borrow)                                   └──┬──┘│
 │   B ────┘                          Bo1 ──┐  ┌────┐            Bo2 │
 │                                          └─▶│ OR ├────────────▶ Bo│
 │   * 식: D = A ⊕ B                  Bo2 ──┘  └────┘                │
 │         Bo = A'B                                                  │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 위 그림은 감산기의 구성을 보여준다. **반감산기**는 XOR로 차를 구하고, 빌림수를 구하기 위해 피감수($A$)를 반전시켜 AND에 넣는 것이 특징이다. 즉, $A=0$이고 $B=1$일 때만 빌림수 1이 발생한다. **전감산기**는 이 반감산기 2개와 OR 게이트를 조합하여, 하위 비트에서 빌려준 값($B_{in}$)까지 고려하여 최종 차($D$)와 빌림수($B_{out}$)를 구한다. 가산기와 구조는 매우 흡사하지만, '빌림'을 처리하기 위한 NOT 게이트의 위치가 핵심적인 차별점이다.

---

### 실무적 구현: 가산기를 이용한 뺄셈 (Inverter + Adder)

현대 디지털 시스템에서는 감산기를 따로 설계하지 않고, 가산기 입력단에 XOR 게이트를 추가하여 뺄셈을 수행한다.

$$A - B = A + (\text{2's Complement of } B) = A + \overline{B} + 1$$

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: 가산기(Adder) vs 감산기(Subtractor)

| 비교 항목 | 가산기 (Adder) | 감산기 (Subtractor) |
|:---|:---|:---|
| **기본 연산** | $A + B$ | $A - B$ |
| **핵심 출력** | 합 (Sum) | 차 (Difference) |
| **자리 이동** | 올림수 (Carry) | 빌림수 (Borrow) |
| **XOR 활용** | 동일 ($A \oplus B$) | 동일 ($A \oplus B$) |
| **차이점** | $C = AB$ | $B = \overline{A}B$ (A에 NOT 존재) |

### 비교 2: 하드웨어 설계 전략 (Standalone vs Integrated)

| 항목 | 독립 감산기 (Standalone) | 가산기 통합형 (Integrated) |
|:---|:---|:---|
| **면적 효율** | 낮음 (중복 회로) | **매우 높음 (회로 공유)** |
| **제어 복잡도** | 단순 | 약간 높음 (Add/Sub 선택 신호 필요) |
| **성능** | 최적화 가능 | 가산기 성능에 의존 |
| **현대적 위상** | 거의 쓰이지 않음 | **ALU 설계의 표준** |

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: CPU의 비교(Compare) 명령어 구현

1. **상황**: `CMP R1, R2` (R1과 R2의 값을 비교) 명령어를 수행할 때, 별도의 비교기를 만들 것인가?
2. **원인 분석**: 비교 연산은 사실상 $A - B$를 수행한 뒤 그 결과가 0인지, 음수인지 확인하는 과정과 같음.
3. **기술적 해결책**:
   - **가산기 재활용**: 가산기를 뺄셈 모드로 동작시킴.
   - **Flag 반영**: 연산 결과($D$)는 버리고, 빌림수($B_{out}$)나 Zero 결과값만 상태 레지스터(Status Register)의 Carry Flag, Zero Flag에 기록.
4. **결과**: 추가 하드웨어 없이 덧셈-뺄셈-비교 기능을 하나의 ALU 블록에서 완벽히 처리.

### 도입 체크리스트 (Subtractor/ALU Design)
- [ ] **2's Complement Overflow**: 뺄셈 결과가 할당된 비트 범위를 벗어나 부호가 뒤집히는 오버플로우 처리를 했는가?
- [ ] **Mode Control**: 덧셈과 뺄셈을 전환할 때 발생하는 제어 신호의 지연이 전체 연산 속도에 영향을 주지 않는가?
- [ ] **Unsigned vs Signed**: 부호 있는 수와 부호 없는 수의 빌림수 해석 차이를 아키텍처에 반영했는가?

### 안티패턴: 음수 표현을 위한 별도 감산기 고집
- 단순히 뺄셈이 필요하다고 해서 보수 연산법을 무시하고 별도의 감산기 로직을 집적하는 행위는 칩 다이 면적과 전력 소모를 2배로 늘리는 비효율의 극치다. 현대 아키텍처에서는 반드시 보수 방식을 통한 가산기 통합을 우선시해야 한다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 기대효과 | 비즈니스 가치 |
|:---:|:---|:---|
| **정량** | 가산기 통합 설계로 연산부 면적 45% 절감 | 칩 소형화 및 모바일 기기 경쟁력 확보 |
| **정량** | 통합 경로 최적화로 누설 전력 감소 | 서버용 CPU의 전력 효율(TCO) 개선 |
| **정성** | 연산 체계의 수학적 간결성 확보 | 컴파일러 및 소프트웨어 최적화 용이성 증대 |

### 미래 전망
- **Configurable Arithmetic**: AI 연산 중 양자화(Quantization) 과정에서 덧셈과 뺄셈의 비트 정밀도를 동적으로 조절하는 가변 감산기 기술이 NPU 설계의 핵심으로 부상하고 있다.
- **Memristor-based Subtractor**: 메모리 내부에서 직접 뺄셈 연산을 수행하는 PIM(Processing-In-Memory) 기술을 통해, 데이터 이동 없이 감산 연산을 처리하는 차세대 아키텍처가 개발 중이다.
