# 464. 메모리 미러링 (Memory Mirroring)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 미러링은 RAID 1처럼 메모리를 두 세트로 나누어 동일 데이터를 동시 저장하고, 한 쪽 고장 시 다른 쪽으로 서비스를 지속하는 고가용성 기법이다.
> 2. **가치**: 50% 용량 손실이 있지만, 메모리 고장이나 다중 비트 오류(Uncorrectable Error) 발생 시에도 서비스 중단 없이 동작하여 99.99% 이상의 가용성을 달성한다.
> 3. **융합**: 인텔 Xeon, IBM Power 서버에서 지원하며, 금융, 통신, 미션 크리티컬 시스템에서 ECC와 함께 필수적으로 적용된다.

---

### I. 개요 (Context & Background)

#### 개념 정의

**메모리 미러링(Memory Mirroring)**은 시스템 메모리를 두 개의 독립된 영역으로 나누어, 동일한 데이터를 두 영역에 동시에 저장하는 기술이다. 한 쪽 메모리 채널/랭크/디밈에 고장이 발생해도 다른 쪽에서 데이터를 읽어 서비스를 지속한다.

```
┌─────────────────────────────────────────────────────────────────┐
│                 메모리 미러링 구조                               │
└─────────────────────────────────────────────────────────────────┘

일반 메모리 (Non-Mirrored):
┌─────────────────────────────────────────────────────────────────┐
│  ┌─────────────────────────────────────────────────────────────┐│
│  │              모든 메모리를 데이터 용도로 사용               ││
│  │                 (예: 256GB 전부 사용 가능)                  ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  용량: 100%  |  이중화: 없음  |  가용성: 일반                    │
└─────────────────────────────────────────────────────────────────┘

메모리 미러링 (Mirrored):
┌─────────────────────────────────────────────────────────────────┐
│  ┌───────────────────────────┐ ┌───────────────────────────┐   │
│  │      Primary Set          │ │      Secondary Set        │   │
│  │      (채널 A, C)          │ │      (채널 B, D)          │   │
│  │                           │ │                           │   │
│  │    [128GB 데이터]         │ │    [128GB 데이터]         │   │
│  │         ▲                 │ │         ▲                 │   │
│  │         │   동일 데이터    │ │         │                 │   │
│  └─────────┼─────────────────┘ └─────────┼─────────────────┘   │
│            │                             │                      │
│            └─────────────────────────────┘                      │
│                       미러링                                    │
│                                                                 │
│  용량: 50% (128GB)  |  이중화: O  |  가용성: 고가용성           │
└─────────────────────────────────────────────────────────────────┘
```

#### 비유

> **메모리 미러링은 "같은 내용을 두 공책에 동시에 적는 것"과 같다.**
>
> 중요한 필기를 할 때, 한 공책을 잃어버리면 큰일 나니까 두 공책에 똑같이 적어둔다. 한 공책이 망가져도 다른 공책이 있으니까 안전하다!

### II. 아키텍처 및 핵심 원리 (Deep Dive)

#### 메모리 미러링 동작

```
┌─────────────────────────────────────────────────────────────────┐
│               메모리 미러링 동작 시나리오                       │
└─────────────────────────────────────────────────────────────────┘

쓰기 동작 (Write):
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   CPU ──▶ 데이터 ──┬──▶ Primary Memory (채널 A)               │
│                    │                                            │
│                    └──▶ Secondary Memory (채널 B)               │
│                                                                 │
│   · 두 메모리 세트에 동시 쓰기                                  │
│   · 쓰기 지연 시간: 일반 메모리와 유사                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

읽기 동작 (Read):
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   정상 시: Primary Memory에서 읽기                              │
│   CPU ◀── 데이터 ◀── Primary Memory                            │
│                                                                 │
│   Primary 고장 시: Secondary에서 읽기                           │
│   CPU ◀── 데이터 ◀── Secondary Memory                          │
│                                                                 │
│   · 기본적으로 Primary에서 읽기                                 │
│   · Primary 고장/오류 시 자동으로 Secondary 전환                │
│   · 전환 시간: 마이크로초 단위                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

고장 감지 및 복구:
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   1. ECC 오류 감지                                              │
│      Primary에서 UE (Uncorrectable Error) 발생                  │
│                                                                 │
│   2. 자동 전환                                                  │
│      Memory Controller가 Secondary로 전환                       │
│                                                                 │
│   3. 서비스 지속                                                │
│      시스템 리셋 없이 계속 동작                                 │
│                                                                 │
│   4. 고장 격리                                                  │
│      고장 난 DIMM/채널 로그 기록                                │
│      OS에 이벤트 통지                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### III. 융합 비교 및 다각도 분석

#### 메모리 보호 기법 비교

| 기법 | 용량 손실 | 정정 능력 | 가용성 | 비용 |
|------|-----------|-----------|--------|------|
| **ECC** | 12.5% | 1-bit | 일반 | 낮음 |
| **Chipkill** | 12.5% | 1-chip | 높음 | 중간 |
| **Memory Mirroring** | 50% | 전체 | 매우 높음 | 높음 |
| **Memory Sparing** | 6~12% | 1-rank | 높음 | 중간 |

### IV. 실무 적용 및 기술사적 판단

#### 실무 시나리오: 인텔 서버 메모리 모드

```
┌─────────────────────────────────────────────────────────────────┐
│           Intel Xeon Memory Modes                               │
└─────────────────────────────────────────────────────────────────┘

1. Independent Mode (기본)
   · 모든 메모리 채널 독립적 사용
   · 최대 용량, 최대 대역폭
   · 보호: ECC만

2. Lockstep Mode
   · 채널 2개를 하나로 묶음
   · 128-bit 데이터 경로
   · 보호: 향상된 ECC

3. Mirror Mode
   · 채널 2쌍이 서로 미러
   · 용량 50%, 99.999% 가용성
   · 보호: 완전 이중화

BIOS 설정:
┌─────────────────────────────────────────────────────────────────┐
│  Memory Configuration                                           │
│  ├── Memory Mode: [Independent / Lockstep / Mirror]            │
│  ├── Mirror Mode Configuration                                  │
│  │   ├── Inter-Socket Mirroring: [Disabled / Enabled]          │
│  │   └── Partial Mirroring: [Disabled / 25% / 50% / 75%]       │
│  └── Sparing: [Disabled / Rank Sparing / Bank Sparing]         │
└─────────────────────────────────────────────────────────────────┘
```

### 어린이를 위한 3줄 비유 설명

**메모리 미러링은 "같은 숙제를 두 공책에 적어두는 것"과 같아요!**

1. 중요한 숙제를 한 공책에만 적어두면, 그 공책을 잃어버리거나 찢어지면 큰일 나요! 그래서 두 공책에 똑같이 적어두면 안전하죠.

2. 물론 두 공책에 적어야 하니까 시간이 더 걸리고, 공책도 두 개가 필요해요. 하지만 안전하니까 그만큼 가치가 있어요!

3. 은행 컴퓨터나 중요한 서버도 메모리를 두 배로 써서, 하나가 고장 나도 다른 하나가 있으니까 돈이나 게임 데이터가 사라지지 않아요!
