# 452. 가용성 (Availability)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가용성은 시스템이 언제든지 사용자에게 서비스를 제공할 수 있는 상태로 존재할 확률로, MTBF와 MTTR의 함수인 A = MTBF/(MTBF+MTTR)로 정량화되며 비즈니스 연속성의 핵심 지표이다.
> 2. **가치**: 99.999%(Five 9s) 가용성은 연간 다운타임 5.26분을 의미하며, 금융/통신/헬스케어 등 미션 크리티컬 분야에서 필수적이고, 가용성 9가 하나 추가될 때마다 인프라 비용은 2~3배 증가한다.
> 3. **융합**: 가용성은 하드웨어 이중화, 소프트웨어 페일오버, 네트워크 다중화, 지리적 분산, Auto Scaling 등 모든 계층에서 종합적으로 구현되어야 달성 가능하다.

---

### I. 개요 (Context & Background)

#### 개념 정의

**가용성(Availability)**은 시스템이나 서비스가 필요한 시점에 정상적으로 작동하여 사용자의 요청을 처리할 수 있는 상태로 존재할 확률을 의미한다. 전통적으로 백분율(%)로 표현되며, "9의 개수"(Number of nines)로도 자주 인용된다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    가용성의 본질적 정의                         │
└─────────────────────────────────────────────────────────────────┘

                    MTBF (평균 무고장 시간)
    Availability = ─────────────────────────────
                   MTBF + MTTR (평균 수리 시간)

    또는

                    MTTF (고장까지 평균 시간)
    Availability = ─────────────────────────────
                   MTTF + MTTR

해석:
· 시스템이 "살아있는 시간" / "전체 시간"
· 0% = 항상 다운
· 100% = 절대 다운되지 않음 (현실적으로 불가능)
```

#### 비유

> **가용성은 "편의점이 문을 여는 시간"과 같다.**
>
> - **99% 가용성**: 1년 365일 중 약 3.6일 동안 문을 닫음 (연휴에 쉬는 편의점)
> - **99.9% 가용성**: 연간 약 8.7시간 문을 닫음 (하루 8시간 영업하는 편의점)
> - **99.99% 가용성**: 연간 약 52분 문을 닫음 (거의 항상 열려있는 편의점)
> - **99.999% 가용성**: 연간 약 5분 문을 닫음 (사실상 24시간 365일 열려있는 편의점)
>
> 고객 입장에서는 언제 가도 물건을 살 수 있는 편의점이 좋다. 마찬가지로 언제든 서비스를 이용할 수 있는 시스템이 높은 가용성을 가진다.

#### 등장 배경 및 발전 과정

1. **1960-70년대: 메인프레임 시대**
   - 은행, 항공 예약 시스템에서 24시간 가동 요구
   - 초기 가용성 개념: "업타임 비율"

2. **1980-90년대: 인터넷 등장**
   - 웹 서비스의 등장으로 24×7 요구사항 보편화
   - SLA(Service Level Agreement) 계약에 가용성 명시

3. **2000년대: 클라우드 서비스**
   - AWS, Azure 등 "99.9%~99.99% 가용성" 보장
   - Multi-AZ, 지리적 분산으로 고가용성 실현

4. **2010년대~현재: Zero Downtime**
   - Blue-Green, Canary 배포로 무중단 배포
   - Kubernetes, Service Mesh로 자동 복구
   - 금융권 99.999%, 통신사 99.9999% 목표

#### 가용성 계산 심층 분석

```
┌─────────────────────────────────────────────────────────────────┐
│               가용성 계산 다이어그램                             │
└─────────────────────────────────────────────────────────────────┘

기본 공식:
    ┌─────────────────────────────────────────────────────┐
    │                                                     │
    │                 MTBF                                │
    │  A = ────────────────────────                       │
    │                MTBF + MTTR                          │
    │                                                     │
    └─────────────────────────────────────────────────────┘

    시각화:
    ┌────────────────────────────────────────────────────────┐
    │                                                        │
    │  시간 → ══════════════════╳═══════════════════════════│
    │         ├─────────────────┤├─────────────────────────┤│
    │               MTBF                 MTTR               │
    │            (정상 동작)           (수리 중)            │
    │                                                        │
    │  전체 시간 = MTBF + MTTR                               │
    │  가용성 = 정상 시간 / 전체 시간 = MTBF / (MTBF+MTTR)   │
    │                                                        │
    └────────────────────────────────────────────────────────┘

예시 계산:
┌────────────────────────────────────────────────────────────────┐
│                                                                │
│  Case 1: MTBF = 1,000시간, MTTR = 1시간                       │
│  A = 1000/(1000+1) = 0.9990 = 99.90%                          │
│                                                                │
│  Case 2: MTBF = 10,000시간, MTTR = 1시간                      │
│  A = 10000/(10000+1) = 0.9999 = 99.99%                        │
│                                                                │
│  Case 3: MTBF = 1,000시간, MTTR = 0.1시간 (6분)               │
│  A = 1000/(1000+0.1) = 0.9999 = 99.99%                        │
│                                                                │
│  ※ MTBF 10배 증가 ≈ MTTR 10배 감소 (가용성 관점)              │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

#### 직렬/병렬 시스템 가용성

```
┌─────────────────────────────────────────────────────────────────┐
│             직렬 시스템 (Series System)                         │
└─────────────────────────────────────────────────────────────────┘

    입력 ──▶ [A] ──▶ [B] ──▶ [C] ──▶ 출력

    모든 컴포넌트가 작동해야 시스템이 작동

    A_series = A1 × A2 × A3 × ... × An

    예시: 3개 컴포넌트, 각각 99% 가용성
    A_series = 0.99 × 0.99 × 0.99 = 0.9703 = 97.03%

    ※ 컴포넌트가 많을수록 시스템 가용성 급격히 하락!

┌─────────────────────────────────────────────────────────────────┐
│             병렬 시스템 (Parallel System)                       │
└─────────────────────────────────────────────────────────────────┘

         ┌──▶ [A1] ──┐
    입력 ├───▶ [A2] ───┼──▶ 출력
         └──▶ [A3] ──┘

    하나라도 작동하면 시스템이 작동

    A_parallel = 1 - (1-A1) × (1-A2) × ... × (1-An)

    예시: 3개 컴포넌트, 각각 99% 가용성
    A_parallel = 1 - (0.01 × 0.01 × 0.01) = 0.999999 = 99.9999%

    ※ 이중화로 가용성 획기적 향상!

┌─────────────────────────────────────────────────────────────────┐
│          N+1 중복 시스템 가용성 비교                             │
└─────────────────────────────────────────────────────────────────┘

    ┌────────────────┬────────────────────────────────────────────┐
    │  구성          │  시스템 가용성 (단일 컴포넌트 99% 가정)    │
    ├────────────────┼────────────────────────────────────────────┤
    │  단일 (1×)     │  99.00%  (연간 3.65일 다운)               │
    │  1+1 이중화    │  99.99%  (연간 52분 다운)                 │
    │  1+2 3중화     │  99.9999% (연간 31초 다운)                │
    │  1+3 4중화     │  99.999999% (연간 0.3초 다운)             │
    └────────────────┴────────────────────────────────────────────┘
```

#### 핵심 코드: 가용성 계산기

```python
#!/usr/bin/env python3
"""
Availability Calculator
- 가용성 계산 및 분석
- 직렬/병렬 시스템 가용성
- 다운타임 예측
"""

import math
from dataclasses import dataclass
from typing import List, Literal

@dataclass
class SystemComponent:
    name: str
    availability: float  # 0.0 ~ 1.0

    @property
    def unavailability(self) -> float:
        return 1 - self.availability

    @property
    def nines(self) -> float:
        """9의 개수 계산"""
        if self.availability >= 1:
            return float('inf')
        return -math.log10(1 - self.availability)

    def annual_downtime(self) -> dict:
        """연간 다운타임 계산"""
        hours_per_year = 365.25 * 24
        downtime_hours = hours_per_year * self.unavailability

        return {
            "hours": downtime_hours,
            "minutes": downtime_hours * 60,
            "seconds": downtime_hours * 3600,
            "human_readable": self._format_downtime(downtime_hours)
        }

    def _format_downtime(self, hours: float) -> str:
        if hours >= 24:
            return f"{hours/24:.1f}일"
        elif hours >= 1:
            return f"{hours:.1f}시간"
        elif hours >= 1/60:
            return f"{hours*60:.1f}분"
        else:
            return f"{hours*3600:.1f}초"


def calculate_system_availability(
    components: List[SystemComponent],
    config: Literal["series", "parallel"]
) -> SystemComponent:
    """
    시스템 전체 가용성 계산
    """
    if config == "series":
        # A_system = A1 × A2 × ... × An
        system_availability = 1.0
        for comp in components:
            system_availability *= comp.availability

    elif config == "parallel":
        # A_system = 1 - (1-A1) × (1-A2) × ... × (1-An)
        system_unavailability = 1.0
        for comp in components:
            system_unavailability *= comp.unavailability
        system_availability = 1 - system_unavailability

    else:
        raise ValueError(f"Unknown config: {config}")

    return SystemComponent(
        name=f"System ({config})",
        availability=system_availability
    )


def availability_from_mtbf_mttr(mtbf_hours: float, mttr_hours: float) -> float:
    """MTBF와 MTTR로부터 가용성 계산"""
    return mtbf_hours / (mtbf_hours + mttr_hours)


def mttr_for_target_availability(
    target_availability: float,
    mtbf_hours: float
) -> float:
    """목표 가용성 달성을 위한 필요 MTTR 계산"""
    # A = MTBF / (MTBF + MTTR)
    # MTTR = MTBF × (1-A) / A
    return mtbf_hours * (1 - target_availability) / target_availability


def generate_availability_report(components: List[SystemComponent]) -> str:
    """가용성 보고서 생성"""
    lines = []
    lines.append("=" * 70)
    lines.append("                     가용성 분석 보고서")
    lines.append("=" * 70)
    lines.append("")

    # 개별 컴포넌트
    lines.append("┌────────────────────────────────────────────────────────────────┐")
    lines.append("│                     개별 컴포넌트                              │")
    lines.append("├─────────────┬───────────┬─────────────┬──────────────────────┤")
    lines.append("│   이름      │  가용성   │    9의 개수 │    연간 다운타임     │")
    lines.append("├─────────────┼───────────┼─────────────┼──────────────────────┤")

    for comp in components:
        downtime = comp.annual_downtime()
        lines.append(
            f"│ {comp.name[:11]:11} │ {comp.availability*100:7.3f}% │ "
            f"{comp.nines:7.2f} 9s │ {downtime['human_readable']:>18} │"
        )

    lines.append("└─────────────┴───────────┴─────────────┴──────────────────────┘")
    lines.append("")

    # 직렬 시스템
    series_system = calculate_system_availability(components, "series")
    series_downtime = series_system.annual_downtime()
    lines.append("┌────────────────────────────────────────────────────────────────┐")
    lines.append("│                 직렬 시스템 (모두 작동 필요)                   │")
    lines.append("├────────────────────────────────────────────────────────────────┤")
    lines.append(f"│  가용성: {series_system.availability*100:.6f}% ({series_system.nines:.2f} 9s)")
    lines.append(f"│  연간 다운타임: {series_downtime['human_readable']}")
    lines.append("└────────────────────────────────────────────────────────────────┘")
    lines.append("")

    # 병렬 시스템
    parallel_system = calculate_system_availability(components, "parallel")
    parallel_downtime = parallel_system.annual_downtime()
    lines.append("┌────────────────────────────────────────────────────────────────┐")
    lines.append("│             병렬 시스템 (하나라도 작동하면 OK)                 │")
    lines.append("├────────────────────────────────────────────────────────────────┤")
    lines.append(f"│  가용성: {parallel_system.availability*100:.9f}% ({parallel_system.nines:.2f} 9s)")
    lines.append(f"│  연간 다운타임: {parallel_downtime['human_readable']}")
    lines.append("└────────────────────────────────────────────────────────────────┘")

    return "\n".join(lines)
