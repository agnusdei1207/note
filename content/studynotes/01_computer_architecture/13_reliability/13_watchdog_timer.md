# 461. 워치독 타이머 (Watchdog Timer)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 워치독 타이머는 시스템이 정해진 시간 내에 응답(Kick/Pet)하지 않으면 자동으로 리셋하는 하드웨어/소프트웨어 메커니즘으로, 시스템 Hang/Deadlock을 감지하고 복구한다.
> 2. **가치**: 임베디드 시스템, 서버, 산업 제어 등에서 소프트웨어 오류로 인한 무한 루프나 데드락을 자동 복구하여 무인 운영을 가능하게 한다.
> 3. **융합**: 하드웨어 WDT(칩 내장), 소프트웨어 WDT(별도 스레드), 분산 WDT(Heartbeat) 등으로 확장되며, Kubernetes Liveness Probe도 WDT의 일종이다.

---

### I. 개요 (Context & Background)

#### 개념 정의

**워치독 타이머(Watchdog Timer, WDT)**는 시스템이 정상적으로 동작하는지 감시하는 타이머 장치다. 소프트웨어가 주기적으로 타이머를 리셋(Kick the dog)해야 하며, 리셋하지 않으면 시스템 리셋이 발생한다.

```
┌─────────────────────────────────────────────────────────────────┐
│                  워치독 타이머 동작 원리                        │
└─────────────────────────────────────────────────────────────────┘

         ┌─────────────────────────────────────────────────────┐
         │                    Software Loop                    │
         │                                                     │
         │   while(True) {                                     │
         │       do_work();                                    │
         │       watchdog_kick();  // 타이머 리셋 ("개 밥주기")│
         │   }                                                 │
         │                                                     │
         └──────────────────────┬──────────────────────────────┘
                                │
                    주기적 Kick │
                                ▼
         ┌─────────────────────────────────────────────────────┐
         │              Watchdog Timer Hardware                │
         │                                                     │
         │   Counter: 0 ──▶ 1 ──▶ 2 ──▶ ... ──▶ TIMEOUT      │
         │            ▲                                        │
         │            │                                        │
         │       Kick 시 0으로 리셋                             │
         │                                                     │
         │   TIMEOUT 도달 → System Reset                       │
         │                                                     │
         └─────────────────────────────────────────────────────┘

시나리오:
1. 정상: 1초마다 Kick → Counter 리셋 → 리셋 발생 안 함
2. 고장: 무한 루프/Hang → Kick 안 함 → Counter TIMEOUT → 시스템 리셋
```

#### 비유

> **워치독 타이머는 "집사가 출근하면서 강아지에게 밥을 주는 타이머"와 같다.**
>
> 강아지에게 "1시간마다 밥 그릇을 비우면 자동으로 채워진다"는 기계가 있다. 집사가 정상적으로 돌아다니면 1시간마다 밥 그릇을 비울 수 있다. 하지만 집사가 쓰러져서 움직이지 못하면 밥 그릇이 비워지지 않고, 기계가 "무언가 잘못됐다"고 판단해 비상 연락을 보낸다.
>
> 소프트웨어가 "1초마다 워치독 밥 주기(Kick)"를 하면 정상. 못 하면 시스템 리셋!

#### 핵심 코드: 워치독 타이머 구현

```c
/*
 * Hardware Watchdog Timer Driver (Linux)
 */

#include <stdio.h>
#include <stdlib.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/ioctl.h>
#include <linux/watchdog.h>

#define WDT_DEVICE "/dev/watchdog"

int main() {
    int fd;
    int timeout = 10;  // 10초 타임아웃

    // 워치독 장치 열기
    fd = open(WDT_DEVICE, O_WRONLY);
    if (fd < 0) {
        perror("Failed to open watchdog device");
        return 1;
    }

    // 타임아웃 설정
    ioctl(fd, WDIOC_SETTIMEOUT, &timeout);
    printf("Watchdog timeout set to %d seconds\n", timeout);

    // 메인 루프
    while (1) {
        // 실제 작업 수행
        if (do_critical_work() != 0) {
            printf("Work failed, not kicking watchdog\n");
            // 실패 시 kick 하지 않음 → 리셋 발생
            sleep(timeout + 1);
        }

        // 워치독 킥 (밥 주기)
        if (write(fd, "\0", 1) != 1) {
            perror("Failed to kick watchdog");
        } else {
            printf("Watchdog kicked\n");
        }

        sleep(5);  // 5초 대기
    }

    // 종료 시 Magic Close (워치독 비활성화)
    write(fd, "V", 1);
    close(fd);
    return 0;
}
```

### III. 융합 비교 및 다각도 분석

#### 워치독 타이머 비교

| 유형 | 구현 | 신뢰성 | 응답 시간 | 적용 |
|------|------|--------|-----------|------|
| **하드웨어** | 칩 내장 | 높음 | ms | 임베디드, 서버 |
| **소프트웨어** | 별도 스레드 | 중간 | ms~s | 애플리케이션 |
| **분산** | Heartbeat | 높음 | s | 클러스터 |

### IV. 실무 적용 및 기술사적 판단

#### 실무 시나리오: Kubernetes Liveness Probe

```yaml
# Kubernetes Liveness Probe = 일종의 워치독
apiVersion: v1
kind: Pod
metadata:
  name: app-pod
spec:
  containers:
  - name: app
    image: myapp:latest
    livenessProbe:
      httpGet:
        path: /health
        port: 8080
      initialDelaySeconds: 30
      periodSeconds: 10    # 10초마다 체크
      timeoutSeconds: 5
      failureThreshold: 3  # 3회 실패 시 재시작
    # 30초 동안 응답 없으면 Pod 재시작
```

### 어린이를 위한 3줄 비유 설명

**워치독 타이머는 "1시간마다 주인에게 신호 보내는 반려견"과 같아요!**

1. 강아지가 1시간마다 "멍!" 하고 짖으면 주인이 "응, 살아있구나!" 하고 밥을 줘요. 만약 강아지가 1시간 동안 짖지 않으면, 무슨 일이 생긴 거예요!

2. 컴퓨터도 마찬가지예요. 프로그램이 1초마다 "나 잘 작동해!"라고 신호를 보내요. 신호가 안 오면 컴퓨터가 "이상하다, 다시 시작해야지!" 하고 재부팅해요.

3. 이게 워치독이에요. "개를 지켜보는 타이머"라는 뜻인데, 컴퓨터가 멈추면 자동으로 다시 살려내는 거예요!
