# 455. TMR (Triple Modular Redundancy, 삼중 모듈 중단)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: TMR은 동일한 기능을 수행하는 3개의 모듈을 병렬로 배치하고, 다수결 투표(Voting)를 통해 한 모듈의 고장을 허용하는 대표적인 고장 허용 기법이다.
> 2. **가치**: TMR 신뢰도는 R_TMR = 3R² - 2R³으로, 단일 모듈 신뢰도 99%일 때 TMR 신뢰도는 99.97%로 향상되며, 항공우주, 원자력, 철도 신호 등에서 필수적으로 사용된다.
> 3. **융합**: TMR은 NMR(N-Modular Redundancy)로 확장 가능하며, 소프트웨어에서는 N-Version Programming으로, 분산 시스템에서는 합의 알고리즘으로 응용된다.

---

### I. 개요 (Context & Background)

#### 개념 정의

**TMR(Triple Modular Redundancy, 삼중 모듈 중단)**는 고장 허용 시스템 설계에서 가장 널리 사용되는 하드웨어 이중화 기법이다. 3개의 동일한 모듈이 독립적으로 동일한 연산을 수행하고, 그 결과를 투표기(Voter)가 비교하여 다수결로 최종 결과를 결정한다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    TMR 기본 구조                                │
└─────────────────────────────────────────────────────────────────┘

                    입력 (Input)
                        │
          ┌─────────────┼─────────────┐
          ▼             ▼             ▼
     ┌─────────┐  ┌─────────┐  ┌─────────┐
     │ Module1 │  │ Module2 │  │ Module3 │
     │  (M1)   │  │  (M2)   │  │  (M3)   │
     └────┬────┘  └────┬────┘  └────┬────┘
          │            │            │
          └────────────┼────────────┘
                       ▼
              ┌────────────────┐
              │     Voter      │
              │   (투표기)     │
              │                │
              │  다수결 판정   │
              └────────┬───────┘
                       ▼
                   출력 (Output)

동작 원리:
· M1=O, M2=O, M3=O → 출력 O (모두 동의)
· M1=O, M2=O, M3=X → 출력 O (M3 고장 허용)
· M1=O, M2=X, M3=O → 출력 O (M2 고장 허용)
· M1=X, M2=O, M3=O → 출력 O (M1 고장 허용)
· M1=X, M2=X, M3=O → 출력 X (2개 고장 = 실패)
· M1=X, M2=X, M3=X → 출력 X (모두 고장)
```

#### 신뢰도 공식

```
┌─────────────────────────────────────────────────────────────────┐
│                    TMR 신뢰도 분석                              │
└─────────────────────────────────────────────────────────────────┘

단일 모듈 신뢰도: R
TMR 시스템 신뢰도: R_TMR = 3R² - 2R³

유도:
· 3개 모두 정상: R³
· 2개 정상, 1개 고장: 3 × R² × (1-R) = 3R² - 3R³
· 총 신뢰도: R³ + 3R² - 3R³ = 3R² - 2R³

┌─────────────────────────────────────────────────────────────────┐
│  단일 모듈 R  │  TMR 신뢰도 R_TMR  │    향상폭    │             │
├───────────────┼────────────────────┼──────────────┤
│    0.80       │      0.896         │   +9.6%      │
│    0.90       │      0.972         │   +7.2%      │
│    0.95       │      0.9928        │   +4.3%      │
│    0.99       │      0.9997        │   +0.97%     │
│    0.999      │      0.999997      │   +0.097%    │
└───────────────┴────────────────────┴──────────────┘

※ 단일 모듈 신뢰도가 높을수록 TMR 효과 ↑
```

#### 비유

> **TMR은 "3명의 심판이 점수를 매기는 체조 경기"와 같다.**
>
> 체조 경기에서 3명의 심판이 독립적으로 점수를 매긴다. 한 심판이 실수를 해도 나머지 2명이 정확한 점수를 주면 최종 점수는 올바르다. TMR도 이와 같이 3개의 모듈이 독립적으로 연산하고, 투표기가 다수결로 정답을 결정한다.

#### 등장 배경 및 발전 과정

1. **1960년대: 우주 탐사**

   - 우주 환경에서 방사선에 의한 일시적 고장 허용

2. **1970-80년대: 항공 전자**
   - Fly-by-Wire 시스템
   - 보잉 777: 3중화 비행 제어 컴퓨터

3. **1990-2000년대: 산업 제어**
   - 원자력 발전소 제어 시스템
   - 철도 신호 시스템

4. **2010년대~현재: 자율주행**
   - 자율주행 자동차의 센서 퓨전
   - 듀얼/TMR 컨트롤러

### II. 아키텍처 및 핵심 원리 (Deep Dive)

#### Voter 구현 방식

```
┌─────────────────────────────────────────────────────────────────┐
│                 Voter 구현 방식 비교                            │
└─────────────────────────────────────────────────────────────────┘

1. Bit-wise Majority Voter (비트 단위 투표)
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  M1:  1 0 1 1 0 1 0 1                                          │
│  M2:  1 0 0 1 0 1 0 1   ← bit 2 오류                           │
│  M3:  1 0 1 1 0 1 0 1                                          │
│       ─────────────────                                        │
│  Out: 1 0 1 1 0 1 0 1   ← 각 비트별 다수결                     │
│                                                                 │
│ · 장점: 단일 비트 오류까지 정정 가능                            │
│ · 단점: 복잡한 하드웨어                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

2. Word-wise Majority Voter (워드 단위 투표)
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  M1: 0x1234                                                     │
│  M2: 0x1235   ← 오류                                            │
│  M3: 0x1234                                                     │
│                                                                 │
│  if (M1 == M2) return M1;                                      │
│  else if (M1 == M3) return M1;                                 │
│  else if (M2 == M3) return M2;                                 │
│  else return ERROR;  // 모두 다름                               │
│                                                                 │
│ · 장점: 구현 간단                                               │
│ · 단점: 워드 전체가 틀리면 정정 불가                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

3. Median Voter (중간값 선택)
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  M1: 100                                                        │
│  M2: 150   ← 오류 (너무 높음)                                   │
│  M3: 102                                                        │
│                                                                 │
│  Out: median(100, 150, 102) = 102                               │
│                                                                 │
│ · 장점: 아날로그/연속값에 적합                                  │
│ · 단점: 정확한 값이 아닐 수 있음                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### TMR 변형 기법

```
┌─────────────────────────────────────────────────────────────────┐
│                    TMR 변형 기법                                │
└─────────────────────────────────────────────────────────────────┘

1. TMR with Spare (TMR/S)
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  Active:  [M1] [M2] [M3] ──▶ Voter ──▶ Output                   │
│  Spare:   [M4]                                                 │
│                                                                 │
│  · M1 고장 감지 시 M4로 교체                                   │
│  · 2개 고장까지 허용                                           │
│  · 신뢰도: R = 12R² - 16R³ + 6R⁴ (단, R < 0.5에서는 효과 ↓)   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

2. NMR (N-Modular Redundancy)
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  N=5 (5중화):                                                   │
│  [M1] [M2] [M3] [M4] [M5] ──▶ Voter ──▶ Output                  │
│                                                                 │
│  · 2개 고장까지 허용 (과반수 유지)                              │
│  · 신뢰도: R_NMR = Σ C(n,k) × R^k × (1-R)^(n-k), k > n/2       │
│  · N↑ → 신뢰도↑, 비용↑                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

3. Hybrid TMR (이종 TMR)
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  [M1: CPU-A] [M2: CPU-A] [M3: CPU-B] ──▶ Voter ──▶ Output       │
│                                                                 │
│  · 서로 다른 하드웨어/소프트웨어 사용                           │
│  · 공통 모드 고장(Common Mode Failure) 방지                     │
│  · 예: 보잉 777 (3개의 서로 다른 프로세서)                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 핵심 코드: TMR 시뮬레이션

```c
/*
 * TMR (Triple Modular Redundancy) Simulator
 * 신뢰도 분석 및 고장 시뮬레이션
 */

#include <stdio.h>
#include <stdlib.h>
#include <math.h>

typedef struct {
    int id;
    double reliability;  // 0.0 ~ 1.0
    int is_working;      // 1: 정상, 0: 고장
} Module;

typedef struct {
    Module modules[3];
    int (*voter)(int, int, int);
} TMRSystem;

// 다수결 투표 함수
int majority_vote(int v1, int v2, int v3) {
    if (v1 == v2) return v1;
    if (v1 == v3) return v1;
    if (v2 == v3) return v2;
    return -1;  // 모두 다름 (에러)
}

// TMR 신뢰도 계산
double tmr_reliability(double single_reliability) {
    double R = single_reliability;
    return 3 * R * R - 2 * R * R * R;
}

// NMR 신났도 계산
double nmr_reliability(double single_reliability, int n) {
    double R = single_reliability;
    double reliability = 0.0;
    int majority = (n / 2) + 1;

    // 이항 계수 계산
    for (int k = majority; k <= n; k++) {
        double coeff = 1.0;
        // C(n,k) 계산
        for (int i = 0; i < k; i++) {
            coeff *= (double)(n - i) / (i + 1);
        }
        reliability += coeff * pow(R, k) * pow(1 - R, n - k);
    }

    return reliability;
}

// 몬테카를로 시뮬레이션
int simulate_tmr(TMRSystem *system, int input, int iterations) {
    int successes = 0;

    for (int i = 0; i < iterations; i++) {
        int outputs[3];

        // 각 모듈 실행 (확률적 고장)
        for (int m = 0; m < 3; m++) {
            double rand_val = (double)rand() / RAND_MAX;
            if (rand_val < system->modules[m].reliability) {
                outputs[m] = input * 2;  // 정상 결과
            } else {
                outputs[m] = -999;  // 고장 (잘못된 결과)
            }
        }

        // 투표
        int result = system->voter(outputs[0], outputs[1], outputs[2]);
        if (result == input * 2) {
            successes++;
        }
    }

    return successes;
}

int main() {
    printf("=============================================\n");
    printf("  TMR Reliability Analysis\n");
    printf("=============================================\n\n");

    // 이론적 신뢰도 비교
    printf("Theoretical Reliability Comparison:\n");
    printf("------------------------------------------------\n");
    printf("Single R   |  TMR R      |  5MR R     | Improvement\n");
    printf("------------------------------------------------\n");

    double R_values[] = {0.8, 0.9, 0.95, 0.99, 0.999};
    for (int i = 0; i < 5; i++) {
        double R = R_values[i];
        double R_tmr = tmr_reliability(R);
        double R_5mr = nmr_reliability(R, 5);
        printf("  %.3f    |  %.5f   |  %.5f  |  +%.2f%%\n",
               R, R_tmr, R_5mr, (R_tmr - R) * 100);
    }

    printf("\n");

    // 몬테카를로 시뮬레이션
    printf("Monte Carlo Simulation (1,000,000 iterations):\n");
    printf("------------------------------------------------\n");

    srand(12345);  // 시드 고정

    TMRSystem system = {
        .modules = {
            {1, 0.95, 1},
            {2, 0.95, 1},
            {3, 0.95, 1}
        },
        .voter = majority_vote
    };

    int successes = simulate_tmr(&system, 42, 1000000);
    double simulated_reliability = (double)successes / 1000000;

    printf("Single Module Reliability: 0.95\n");
    printf("Theoretical TMR Reliability: %.5f\n", tmr_reliability(0.95));
    printf("Simulated TMR Reliability: %.5f\n", simulated_reliability);

    return 0;
}
```

### III. 융합 비교 및 다각도 분석

#### TMR vs. 다른 고장 허용 기법

| 기법 | 허용 고장 수 | 비용 | 복잡도 | 적용 분야 |
|------|--------------|------|--------|-----------|
| **DMR** | 0 (감지만) | 2배 | 낮음 | 오류 감지 |
| **TMR** | 1개 | 3배 | 중간 | 항공, 우주 |
| **5MR** | 2개 | 5배 | 높음 | 원자력, 국방 |
| **Hot Standby** | 1개 | 2배 | 낮음 | 일반 서버 |

### 어린이를 위한 3줄 비유 설명

**TMR은 "3명의 선생님이 같은 시험지를 채점하는 것"과 같아요!**

1. 한 선생님이 실수로 틀린 점수를 줄 수 있어요. 하지만 3명의 선생님이 각자 채점하고, 2명 이상이 같은 점수를 주면 그게 정답이에요!

2. 한 선생님이 채점을 잘못해도 나머지 2명이 올바른 점수를 주면, 학생은 공정한 점수를 받을 수 있어요. 이게 TMR의 마법이에요!

3. 비행기나 우주선의 컴퓨터도 3개가 같은 계산을 해요. 하나가 고장 나도 나머지 둘이 정답을 찾아내니까 비행기는 안전하게 날 수 있어요!
