# 459. 페일 세이프 (Fail-Safe)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페일 세이프는 시스템 고장 발생 시 자동으로 안전한 상태(Safe State)로 전이하여 인명 피해나 치명적 손실을 방지하는 설계 철학이다.
> 2. **가치**: 원자력 발전소 비상 정지, 엘리베이터 비상 브레이크, 자동차 ABS 등에서 페일 세이프는 생명과 직결되며, "고장 나면 멈춘다"는 원칙으로 안전을 보장한다.
> 3. **융합**: 페일 세이프는 하드웨어(퓨즈, 릴레이), 소프트웨어(Watchdog, Circuit Breaker), 보안(Fail-Secure) 등 다층으로 구현된다.

---

### I. 개요 (Context & Background)

#### 개념 정의

**페일 세이프(Fail-Safe)**는 시스템이 고장 났을 때 자동으로 안전한 상태로 전이하는 설계 방식이다. "고장 나더라도 안전하게"라는 철학으로, 서비스 지속보다 안전을 우선한다.

```
┌─────────────────────────────────────────────────────────────────┐
│                페일 세이프 vs 페일 소프트 vs 페일 톨러런트      │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  Fail-Safe (고장 시 안전):                                      │
│  ┌──────────┐      ┌──────────┐      ┌──────────┐              │
│  │  정상    │ ───▶ │  고장    │ ───▶ │  안전    │              │
│  │  동작    │      │  발생    │      │  정지    │              │
│  └──────────┘      └──────────┘      └──────────┘              │
│  예: 원자력 비상 정지, 엘리베이터 브레이크                       │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Fail-Soft (고장 시 축소 동작):                                 │
│  ┌──────────┐      ┌──────────┐      ┌──────────┐              │
│  │  정상    │ ───▶ │  고장    │ ───▶ │  축소    │              │
│  │  100%   │      │  발생    │      │  기능    │              │
│  └──────────┘      └──────────┘      └──────────┘              │
│  예: 8코어 CPU 1코어 고장 → 7코어로 계속 동작                   │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Fail-Tolerant (고장 허용):                                     │
│  ┌──────────┐      ┌──────────┐      ┌──────────┐              │
│  │  정상    │ ───▶ │  고장    │ ───▶ │  정상    │              │
│  │  동작    │      │  발생    │      │  유지    │              │
│  └──────────┘      └──────────┘      └──────────┘              │
│  예: 이중화 서버, TMR                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 비유

> **페일 세이프는 "자동문이 고장 나면 열린 채로 멈추는 것"과 같다.**
>
> - 자동문이 고장 났을 때 닫힌 채로 멈추면 사람이 갇힐 수 있다 → 위험!
> - 열린 채로 멈추면 사람이 나갈 수 있다 → 안전!
>
> 이게 페일 세이프다. "고장 나더라도 안전한 쪽으로 실패하라"

### II. 아키텍처 및 핵심 원리 (Deep Dive)

#### 페일 세이프 설계 원칙

```
┌─────────────────────────────────────────────────────────────────┐
│                  페일 세이프 설계 원칙                          │
└─────────────────────────────────────────────────────────────────┘

1. Default Safe (기본 안전)
   · 고장 시 기본 상태 = 안전 상태
   · 예: 전원 차단 시 브레이크 자동 작동

2. Fail-Safe Position (FSP)
   · 밸브, 스위치 등의 안전 위치 사전 정의
   · 예: 솔레노이드 밸브 → 전원 차단 시 "닫힘" 또는 "열림"

3. Watchdog Timer
   · 일정 시간 응답 없으면 자동 리셋/정지
   · 예: 5초 내 신호 없으면 비상 정지

4. Redundant Safety (이중 안전)
   · 안전 장치 자체도 이중화
   · 예: 비상 정지 버튼 2개 직렬 연결

5. Graceful Degradation (우아한 저하)
   · 기능 축소하더라도 안전하게 동작
   · 예: ABS 고장 → 일반 브레이크로 동작
```

#### 대표적 페일 세이프 사례

| 시스템 | 고장 시 동작 | 안전 상태 |
|--------|--------------|-----------|
| **엘리베이터** | 정전 시 브레이크 작동 | 가장 가까운 층에 정지 |
| **원자력** | 이상 감지 시 제어봉 투입 | 원자로 정지 |
| **자동차 ABS** | ABS 고장 시 일반 브레이크 | 제동 가능 |
| **가스 밸브** | 화재 시 밸브 자동 차단 | 가스 누출 방지 |
| **철도 신호** | 전원 차단 시 적신호 | 열차 정지 |

### III. 융합 비교 및 다각도 분석

#### Fail-Safe vs Fail-Secure

```
┌─────────────────────────────────────────────────────────────────┐
│           Fail-Safe vs Fail-Secure                              │
└─────────────────────────────────────────────────────────────────┘

Fail-Safe (안전 우선):
· 고장 시 → 개방 (Open)
· 예: 비상구 문 (전원 차단 시 열림)
· 목적: 인명 안전

Fail-Secure (보안 우선):
· 고장 시 → 폐쇄 (Locked)
· 예: 전자 도어락 (전원 차단 시 잠김)
· 목적: 보안 유지

┌────────────────┬────────────────┬────────────────┐
│     상황       │  Fail-Safe     │  Fail-Secure   │
├────────────────┼────────────────┼────────────────┤
│  화재 시       │  문 열림 ✓    │  문 잠김 ✗    │
│  침입 시       │  문 열림 ✗    │  문 잠김 ✓    │
└────────────────┴────────────────┴────────────────┘
```

### IV. 실무 적용 및 기술사적 판단

#### 실무 시나리오: Circuit Breaker 패턴

```python
# Circuit Breaker - 소프트웨어 페일 세이프
class CircuitBreaker:
    def __init__(self, failure_threshold=5, timeout=60):
        self.failure_count = 0
        self.failure_threshold = failure_threshold
        self.timeout = timeout
        self.state = "CLOSED"  # CLOSED, OPEN, HALF_OPEN
        self.last_failure_time = None

    def call(self, func, *args):
        if self.state == "OPEN":
            if time.time() - self.last_failure_time > self.timeout:
                self.state = "HALF_OPEN"
            else:
                raise Exception("Circuit is OPEN - Fail-Safe!")

        try:
            result = func(*args)
            self.on_success()
            return result
        except Exception as e:
            self.on_failure()
            raise e

    def on_success(self):
        self.failure_count = 0
        self.state = "CLOSED"

    def on_failure(self):
        self.failure_count += 1
        self.last_failure_time = time.time()
        if self.failure_count >= self.failure_threshold:
            self.state = "OPEN"  # Fail-Safe: 차단
```

### 어린이를 위한 3줄 비유 설명

**페일 세이프는 "비상구 문이 고장 나면 열린 채로 멈추는 것"과 같아요!**

1. 비상구 문이 전기로 작동하는데, 정전이 되면 어떻게 될까요? 문이 잠겨서 못 나가면 위험해요! 그래서 비상구 문은 전기가 끊기면 자동으로 열리게 만들어요.

2. 이게 페일 세이프예요. "고장 나더라도 안전하게"라는 뜻이에요. 엘리베이터도 전기가 끊기면 브레이크가 자동으로 작동해서 안전하게 멈춰요.

3. 컴퓨터도 마찬가지예요. 중요한 시스템이 고장 나면, 문제를 키우지 않고 안전하게 멈추거나, 위험한 일을 하지 않게 만들어요!
