# 460. 페일 소프트 (Fail-Soft)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페일 소프트는 시스템 고장 발생 시 완전히 정지하지 않고, 기능을 축소(Degraded)하거나 비핵심 기능을 제외하고 핵심 기능만 지속하는 방식이다.
> 2. **가치**: 8코어 CPU에서 1코어 고장 시 7코어로 계속 동작, 디스크 일부 고장 시 RAID에서 축소 용량으로 운영 등 서비스 중단 없이 부분적 기능 유지가 가능하다.
> 3. **융합**: Graceful Degradation, Circuit Breaker, Bulkhead 패턴, Kubernetes의 Pod Disruption Budget 등에서 페일 소프트 개념이 활용된다.

---

### I. 개요 (Context & Background)

#### 개념 정의

**페일 소프트(Fail-Soft)**는 시스템의 일부 구성 요소가 고장 났을 때, 완전히 정지하지 않고 기능을 축소하여 서비스를 지속하는 방식이다. "부분 고장은 부분 서비스로"라는 철학이다.

```
┌─────────────────────────────────────────────────────────────────┐
│              페일 소프트 동작 예시                               │
└─────────────────────────────────────────────────────────────────┘

정상 상태 (8코어 CPU, 64GB RAM, 4TB 스토리지):
┌─────────────────────────────────────────────────────────────────┐
│  ┌──────┐┌──────┐┌──────┐┌──────┐┌──────┐┌──────┐┌──────┐┌──────┐│
│  │Core 1││Core 2││Core 3││Core 4││Core 5││Core 6││Core 7││Core 8││
│  └──────┘└──────┘└──────┘└──────┘└──────┘└──────┘└──────┘└──────┘│
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                  64GB Memory (All Available)                ││
│  └─────────────────────────────────────────────────────────────┘│
│  ┌───────────────┐┌───────────────┐┌───────────────┐┌─────────┐ │
│  │   Disk 1      ││   Disk 2      ││   Disk 3      ││ Disk 4  │ │
│  │   (OK)        ││   (OK)        ││   (OK)        ││ (OK)    │ │
│  └───────────────┘└───────────────┘└───────────────┘└─────────┘ │
│                                                                 │
│  성능: 100%  |  용량: 100%  |  기능: 전체                      │
└─────────────────────────────────────────────────────────────────┘

고장 상태 (Core 3 고장, Disk 2 고장):
┌─────────────────────────────────────────────────────────────────┐
│  ┌──────┐┌──────┐┌──────┐┌──────┐┌──────┐┌──────┐┌──────┐┌──────┐│
│  │Core 1││Core 2││ ✗    ││Core 4││Core 5││Core 6││Core 7││Core 8││
│  └──────┘└──────┘└──────┘└──────┘└──────┘└──────┘└──────┘└──────┘│
│  ┌─────────────────────────────────────────────────────────────┐│
│  │              64GB Memory (Degraded - Bad Page Offline)      ││
│  └─────────────────────────────────────────────────────────────┘│
│  ┌───────────────┐┌───────────────┐┌───────────────┐┌─────────┐ │
│  │   Disk 1      ││   ✗          ││   Disk 3      ││ Disk 4  │ │
│  │   (OK)        ││  (Failed)    ││   (OK)        ││ (OK)    │ │
│  └───────────────┘└───────────────┘└───────────────┘└─────────┘ │
│                                                                 │
│  성능: 87.5% | 용량: 75%   | 기능: 축소 (핵심만)               │
│  ↓                                                              │
│  Fail-Soft Mode (Degraded Operation)                           │
└─────────────────────────────────────────────────────────────────┘
```

#### 비유

> **페일 소프트는 "다리가 하나 부러진 의수 의자"와 같다.**
>
> 4개 다리 의자의 하나가 부러지면, 의자는 쓰러지지 않고 3개 다리로 비틀거리지만 계속 앉을 수 있다. 이게 페일 소프트다. 완벽하지는 않지만 기능은 한다.

### II. 아키텍처 및 핵심 원리 (Deep Dive)

#### 페일 소프트 구현 기법

```
┌─────────────────────────────────────────────────────────────────┐
│                 페일 소프트 구현 기법                           │
└─────────────────────────────────────────────────────────────────┘

1. Graceful Degradation (우아한 저하)
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  Web Service 예시:                                              │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ 정상: 검색 + 추천 + 댓글 + 좋아요 + 공유                    ││
│  │ 저하: 검색 + 추천 (댓글/좋아요/공유 비활성화)               ││
│  │ 심각: 검색만 (핵심 기능만)                                  ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                 │
│  구현:                                                          │
│  if system_load > threshold:                                    │
│      disable_feature("comments")                                │
│      disable_feature("recommendations")                         │
│  if system_load > critical_threshold:                           │
│      enable_readonly_mode()                                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

2. Circuit Breaker (서킷 브레이커)
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  상태 전이:                                                     │
│  ┌─────────┐   실패 임계치 초과   ┌─────────┐                  │
│  │ CLOSED  │ ──────────────────▶ │  OPEN   │                  │
│  │ (정상)  │                      │ (차단)  │                  │
│  └─────────┘                      └────┬────┘                  │
│       ▲                                │                        │
│       │           타임아웃 후          │                        │
│       │        ┌──────────────┐       │                        │
│       └────────│  HALF_OPEN  │◀──────┘                        │
│                │  (반개방)    │                                 │
│                └──────────────┘                                 │
│                                                                 │
│  Fail-Soft: OPEN 상태에서 fallback 함수 실행                   │
│  def get_recommendations():                                     │
│      return cached_recommendations  # 페일 소프트              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

3. Feature Flags (기능 플래그)
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  features:                                                      │
│    search: true           # 핵심 - 항상 켜짐                    │
│    recommendations: true  # 부가 - 부하 시 끌 수 있음           │
│    comments: true         # 부가 - 부하 시 끌 수 있음           │
│    analytics: true        # 비핵심 - 문제 시 즉시 끔           │
│                                                                 │
│  if system.is_degraded():                                       │
│      features.recommendations = False                           │
│      features.comments = False                                  │
│      features.analytics = False                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### III. 융합 비교 및 다각도 분석

#### Fail-Safe vs Fail-Soft vs Fail-Tolerant

| 특성 | Fail-Safe | Fail-Soft | Fail-Tolerant |
|------|-----------|-----------|---------------|
| 고장 시 동작 | 정지 | 축소 동작 | 계속 동작 |
| 서비스 | 중단 | 부분 유지 | 완전 유지 |
| 복잡도 | 낮음 | 중간 | 높음 |
| 비용 | 낮음 | 중간 | 높음 |
| 예시 | 비상 정지 | 7/8코어 동작 | 이중화 |

### IV. 실무 적용 및 기술사적 판단

#### 실무 시나리오: E-커머스 페일 소프트

```
┌─────────────────────────────────────────────────────────────────┐
│           E-커머스 서비스 페일 소프트 레벨                      │
└─────────────────────────────────────────────────────────────────┘

Level 0 (정상): 모든 기능 사용 가능
├── 검색, 상품 상세, 장바구니, 결제
├── 추천, 리뷰, 쿠폰, 포인트
└── 실시간 재고, 배송 추적

Level 1 (부하): 부가 기능 제한
├── 검색, 상품 상세, 장바구니, 결제 ✓
├── 추천 (단순화), 리뷰 (읽기 전용)
└── 쿠폰/포인트 일시 중단

Level 2 (장애): 핵심 기능만
├── 검색 (캐시만), 상품 상세 ✓
├── 장바구니 (로컬 저장)
└── 결제 (대체 PG)

Level 3 (심각): 읽기 전용
├── 상품 조회만 가능
└── 주문 접수 (나중에 처리)
```

### 어린이를 위한 3줄 비유 설명

**페일 소프트는 "다리 하나가 부러져도 3개로 서 있는 의자"와 같아요!**

1. 의자 다리가 4개인데 하나가 부러지면 어떻게 될까요? 의자가 쓰러질까요? 아니면 비틀거리지만 앉을 수 있을까요?

2. 페일 소프트는 비틀거리지만 계속 앉을 수 있는 거예요. 완벽하지는 않지만, 아예 못 쓰게 되는 것보다 낫죠!

3. 온라인 게임도 마찬가지예요. 서버에 문제가 생기면 채팅은 안 되더라도 게임은 계속할 수 있게 만들면, 친구랑 못 떠들어도 게임은 즐길 수 있어요!
