# 465. 락스텝 (Lockstep)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락스텝은 두 개 이상의 프로세서가 완전히 동기화되어 동일한 클럭, 동일한 명령어, 동일한 데이터로 실행하며, 결과를 비교하여 고장을 감지하는 기법이다.
> 2. **가치**: 락스텝은 CPU 사이클 단위에서 고장을 감지하여, 단일 비트 오류나 연산 오류도 즉시 발견하여 미션 크리티컬 시스템(항공, 우주, 원자력)에서 필수적이다.
> 3. **융합**: 메인프레임, Stratus 서버, 우주선 컴퓨터 등에서 사용되며, TMR과 함께 가장 엄격한 고장 허용 방식이다.

---

### I. 개요 (Context & Background)

#### 개념 정의

**락스텝(Lockstep)**은 두 개 이상의 CPU가 완전히 동기화된 상태로 실행되는 기법이다. 모든 CPU가 동일한 클럭, 동일한 명령어, 동일한 데이터로 동작하며, 각 사이클마다 결과를 비교하여 불일치 시 고장으로 판단한다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    락스텝 아키텍처                               │
└─────────────────────────────────────────────────────────────────┘

                    입력 (Input)
                        │
          ┌─────────────┼─────────────┐
          ▼             ▼             ▼
     ┌─────────┐  ┌─────────┐  ┌─────────┐
     │  CPU A  │  │  CPU B  │  │  CPU C  │
     │(Primary)│  │(Primary)│  │(Primary)│
     └────┬────┘  └────┬────┘  └────┬────┘
          │            │            │
          │    동일한 클럭/명령어   │
          │            │            │
          └────────────┼────────────┘
                       ▼
              ┌────────────────┐
              │    Comparator  │
              │   (비교기)      │
              │                │
              │ if A==B==C: OK │
              │ else: ERROR    │
              └────────┬───────┘
                       ▼
                   출력 (Output)

특징:
· 사이클 단위 동기화
· 모든 CPU가 완전히 동일하게 실행
· 결과 불일치 = 즉시 고장 감지
· TMR과 유사하지만 더 엄격한 동기화
```

#### 비유

> **락스텝은 "군인들의 제식 훈련"과 같다.**
>
> 세 명의 군인이 완벽하게 동기화되어 행진한다. 발 맞추고, 팔 흔들고, 모든 동작이 동시에. 한 사람이라도 발이 틀리면 즉시 "틀렸다!"라고 알 수 있다.

### III. 융합 비교 및 다각도 분석

#### 락스텝 적용 사례

| 시스템 | 구성 | 용도 |
|--------|------|------|
| **IBM Mainframe** | CPU Pair Lockstep | 고가용성 |
| **Stratus ftServer** | 4-way Lockstep | 미션 크리티컬 |
| **HP NonStop** | CPU Pair | OLTP |
| **Space Computer** | Triple Lockstep | 우주 탐사 |

### 어린이를 위한 3줄 비유 설명

**락스텝은 "세 명이 똑같이 행진하는 군인"과 같아요!**

1. 세 명의 군인이 완벽하게 동시에 발을 맞춰서 걸어요. 하나, 둘, 셋! 누가 한 발이라도 틀리면 바로 알 수 있어요!

2. 컴퓨터도 세 개의 CPU가 완벽하게 동시에 계산해요. 하나, 둘, 셋! 하나라도 다른 결과가 나오면 "이상하다! 어디 고장 났다!"라고 바로 알 수 있어요.

3. 비행기나 우주선 컴퓨터는 아주 중요하니까, 이렇게 똑같이 동작하는 컴퓨터를 여러 개 써서 조그만 실수도 바로 잡아내요!
