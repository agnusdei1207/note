# 457. 핫 스탠바이 (Hot Standby)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 핫 스탠바이는 실시간으로 데이터를 동기화하며 대기하는 예비 시스템으로, Active 시스템 고장 시 즉시(수 초~수 분 내) 서비스를 인계받는 고가용성 구성 방식이다.
> 2. **가치**: 핫 스탠바이는 페일오버 시간(RTO)을 30초~5분으로 단축하며, 데이터 손실(RPO)을 0~수 초로 최소화하여 99.9%~99.99% 가용성을 달성한다.
> 3. **융합**: DB 복제(MySQL Replication, PostgreSQL Streaming), 스토리지 미러링, VM/컨테이너 마이그레이션 등에서 핫 스탠바이가 핵심적으로 활용된다.

---

### I. 개요 (Context & Background)

#### 개념 정의

**핫 스탠바이(Hot Standby)**는 Active(활성) 시스템과 실시간으로 데이터를 동기화하며 항상 서비스 인계 준비가 된 상태로 대기하는 예비 시스템이다. 콜드 스탠바이와 달리, 핫 스탠바이는 이미 부팅되어 있고, 데이터가 최신 상태이며, 즉시 서비스 가능한 상태로 대기한다.

```
┌─────────────────────────────────────────────────────────────────┐
│                핫 스탠바이 vs 콜드 스탠바이                     │
└─────────────────────────────────────────────────────────────────┘

Hot Standby (핫 스탠바이):
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  ┌─────────────┐              ┌─────────────┐                  │
│  │   Active    │   Sync Rep   │    Hot      │                  │
│  │  (Primary)  │ ──────────▶ │   Standby   │                  │
│  │             │   Real-time  │  (Ready)    │                  │
│  └─────────────┘              └─────────────┘                  │
│        ▲                             │                          │
│        │         Failover            │                          │
│        └─────────────────────────────┘                          │
│              (30초 ~ 5분)                                        │
│                                                                 │
│  특징:                                                          │
│  · 항상 실행 중                                                  │
│  · 실시간 데이터 동기화                                          │
│  · 즉시 페일오버 가능                                            │
│  · RPO: 0~수 초                                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

Cold Standby (콜드 스탠바이):
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  ┌─────────────┐              ┌─────────────┐                  │
│  │   Active    │   Periodic   │    Cold     │                  │
│  │  (Primary)  │   Backup     │   Standby   │                  │
│  │             │ ──────────▶ │  (Offline)  │                  │
│  └─────────────┘              └─────────────┘                  │
│        ▲                             │                          │
│        │         Failover            │                          │
│        └─────────────────────────────┘                          │
│              (수 시간 ~ 수 일)                                    │
│                                                                 │
│  특징:                                                          │
│  · 전원 Off 상태                                                 │
│  · 주기적 백업만 보유                                            │
│  · 부팅 및 복구 필요                                              │
│  · RPO: 수 시간 ~ 1일                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 비유

> **핫 스탠바이는 "경기장에서 대기하는 교체 선수"와 같다.**
>
> - **핫 스탠바이**: 경기장 옆에서 유니폼 입고 몸 풀며 대기 → 즉시 투입 가능
> - **콜드 스탠바이**: 집에서 TV로 경기 보는 중 → 투입되려면 시간 걸림
>
> 핫 스탠바이 선수는 경기 상황을 계속 보고 있어서, 주전 선수가 부상당하면 즉시 뛰어들어갈 수 있다.

### II. 아키텍처 및 핵심 원리 (Deep Dive)

#### 핫 스탠바이 아키텍처

```
┌─────────────────────────────────────────────────────────────────┐
│              핫 스탠바이 데이터베이스 아키텍처                   │
└─────────────────────────────────────────────────────────────────┘

                        Application
                             │
                             ▼
                    ┌────────────────┐
                    │   Virtual IP   │
                    │  (10.0.0.100)  │
                    └────────┬───────┘
                             │
          ┌──────────────────┼──────────────────┐
          │                  │                  │
          ▼                  ▼                  ▼
   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐
   │   Primary   │   │  Hot Standby│   │  Hot Standby│
   │   (Active)  │   │   (Node 2)  │   │   (Node 3)  │
   │  10.0.0.1   │   │  10.0.0.2   │   │  10.0.0.3   │
   └──────┬──────┘   └──────┬──────┘   └──────┬──────┘
          │                  │                  │
          │    Streaming     │    Streaming     │
          │    Replication   │    Replication   │
          └──────────────────┴──────────────────┘
                             │
                    ┌────────┴────────┐
                    │  Shared Storage │
                    │   (or WAL Log)  │
                    └─────────────────┘

페일오버 시나리오:
1. Primary 고장 감지 (Heartbeat timeout)
2. Virtual IP를 Standby로 이동
3. Standby를 Primary로 승격
4. 애플리케이션 자동 재연결 (VIP 통해)
5. 총 소요 시간: 30초 ~ 2분
```

#### 핫 스탠바이 기술 비교

| 기술 | 동기화 방식 | RPO | 페일오버 | 적용 |
|------|-------------|-----|----------|------|
| **MySQL Replication** | 비동기/반동기 | 0~수 초 | 수동/자동 | MySQL |
| **PostgreSQL Streaming** | 비동기/동기 | 0~수 초 | 수동/자동 | PostgreSQL |
| **Oracle Data Guard** | 비동기/동기 | 0 | 자동 | Oracle |
| **SQL Server AlwaysOn** | 동기 | 0 | 자동 | SQL Server |

### III. 융합 비교 및 다각도 분석

#### Standby 유형 비교

| 유형 | 상태 | 동기화 | 페일오버 | 비용 | 가용성 |
|------|------|--------|----------|------|--------|
| **Hot** | 실행 중 | 실시간 | 30초~5분 | 높음 | 99.9%~99.99% |
| **Warm** | 실행 중 | 주기적 | 5~30분 | 중간 | 99%~99.9% |
| **Cold** | 정지 | 백업 | 수 시간 | 낮음 | ~99% |

### V. 기대효과 및 결론

| 지표 | Cold Standby | Hot Standby | 개선 |
|------|--------------|-------------|------|

### 어린이를 위한 3줄 비유 설명

**핫 스탠바이는 "경기장 옆에서 대기하는 교체 선수"와 같아요!**

1. 야구 경기에서 교체 선수가 벤치에 앉아 있다가, 주전 선수가 다치면 바로 뛰어들어갈 수 있죠? 이게 핫 스탠바이예요!

2. 교체 선수가 집에 있으면 부를 때까지 시간이 걸려요(콜드 스탠바이). 하지만 경기장에서 대기하면 바로 뛰어들어갈 수 있어요!

3. 컴퓨터도 마찬가지예요. 예비 컴퓨터가 켜져 있고 데이터를 계속 받아오면, 메인 컴퓨터가 고장 나도 바로 일을 이어받을 수 있어요!
