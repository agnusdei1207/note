# 456. 이중화 (Redundancy / Dual Redundancy)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이중화는 시스템 구성 요소를 2개 이상 배치하여 하나가 고장 나더라도 다른 것으로 서비스를 지속하는 고장 허용의 가장 기본적이고 효과적인 기법이다.
> 2. **가치**: Active-Active 이중화는 즉시 페일오버(0~1초)를 제공하고, Active-Passive는 비용 효율적(1.5~2배)으로 가용성 99%→99.9% 향상을 달성한다.
> 3. **융합**: 이중화는 전원(UPS), 네트워크(이중 ISP), 스토리지(RAID 1), 서버(클러스터), 데이터센터(Multi-AZ) 등 모든 계층에서 적용된다.

---

### I. 개요 (Context & Background)

#### 개념 정의

**이중화(Redundancy)**는 시스템의 신뢰성과 가용성을 높이기 위해 핵심 구성 요소를 2개 이상 배치하는 기법이다. 하나가 고장 나면 다른 하나가 자동으로 업무를 인계받아 서비스 중단을 방지한다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    이중화 유형                                  │
└─────────────────────────────────────────────────────────────────┘

1. Active-Passive (Hot Standby)
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│    ┌─────────┐                    ┌─────────┐                  │
│    │ Active  │ ◀── Heartbeat ──▶ │ Passive │                  │
│    │ (Primary)│                    │(Standby) │                  │
│    └────┬────┘                    └────┬────┘                  │
│         │                               │                       │
│         ▼                               ▼                       │
│    ┌─────────┐                    ┌─────────┐                  │
│    │트래픽 O │                    │트래픽 X │                  │
│    └─────────┘                    └─────────┘                  │
│                                                                 │
│    Active 고장 → Passive가 Active로 승격                        │
│    페일오버 시간: 30초 ~ 5분                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

2. Active-Active (Load Balanced)
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│              ┌─────────────────────┐                            │
│              │   Load Balancer     │                            │
│              └──────────┬──────────┘                            │
│                   ┌─────┴─────┐                                  │
│                   ▼           ▼                                  │
│              ┌────────┐  ┌────────┐                             │
│              │Active 1│  │Active 2│                             │
│              │  50%   │  │  50%   │                             │
│              └────────┘  └────────┘                             │
│                                                                 │
│    한 쪽 고장 → 다른 쪽이 100% 처리                             │
│    페일오버 시간: 즉시 (0~1초)                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 비유

> **이중화는 "예비 타이어"와 같다.**
>
> 자동차에 예비 타이어가 있으면, 타이어가 펑크 나더라도 교체해서 계속 운전할 수 있다. 이중화도 마찬가지로, 시스템에 "예비 부품"을 미리 준비해두어 고장 시 즉시 교체할 수 있게 한다.

### II. 아키텍처 및 핵심 원리 (Deep Dive)

#### 이중화 구성 비교

| 구성 | 설명 | 비용 | 페일오버 시간 | 가용성 |
|------|------|------|---------------|--------|
| **단일** | 이중화 없음 | 1x | N/A | ~99% |
| **Active-Passive** | 1개 활성, 1개 대기 | 2x | 30초~5분 | ~99.9% |
| **Active-Active** | 2개 모두 활성 | 2x | 0~1초 | ~99.99% |
| **N+1** | N개 활성, 1개 대기 | (N+1)/N | 30초~5분 | ~99.9% |

### III. 융합 비교 및 다각도 분석

#### Active-Passive vs Active-Active

| 항목 | Active-Passive | Active-Active |
|------|----------------|---------------|
| **자원 활용** | 50% (대기 유휴) | 100% |
| **페일오버** | 30초~5분 | 즉시 |
| **복잡도** | 낮음 | 높음 (동기화 필요) |
| **비용** | 2x | 2x |
| **적용** | DB, 스토리지 | 웹서버, 앱서버 |

### IV. 실무 적용 및 기술사적 판단

#### 실무 시나리오: DB 이중화

```
MySQL InnoDB Cluster (Active-Active 읽기, Active-Passive 쓰기):

┌─────────────────────────────────────────────────────────────────┐
│                          Application                            │
└───────────────────────────────┬─────────────────────────────────┘
                                │
                    ┌───────────┴───────────┐
                    │   MySQL Router        │
                    │  (Read/Write Split)   │
                    └───────────┬───────────┘
                                │
            ┌───────────────────┼───────────────────┐
            ▼                   ▼                   ▼
     ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
     │   Node 1    │    │   Node 2    │    │   Node 3    │
     │  (Primary)  │◀──▶│ (Secondary) │◀──▶│ (Secondary) │
     │   Read/Write│    │  Read Only  │    │  Read Only  │
     └─────────────┘    └─────────────┘    └─────────────┘
            │                   │                   │
            └───────────────────┴───────────────────┘
                     Group Replication

특징:
· 쓰기: Primary 1개에만
· 읽기: 모든 노드 분산
· Primary 고장 → 자동 선출
```

### 어린이를 위한 3줄 비유 설명

**이중화는 "양손으로 자전거를 타는 것"과 같아요!**

1. 한 손으로 자전거를 타다가 손이 아프면 균형을 잃을 수 있어요. 하지만 두 손으로 타면 한쪽이 아파도 다른 쪽으로 균형을 잡을 수 있죠!

2. 컴퓨터도 마찬가지예요. 중요한 컴퓨터가 하나뿐이면 고장 나면 게임이 멈춰요. 그래서 똑같은 컴퓨터를 2대 준비해두면, 하나가 아파도 다른 하나가 일을 할 수 있어요!

3. 은행이나 게임 회사들은 항상 컴퓨터를 2대 이상 써요. 그래야 고장 나도 우리가 게임을 하거나 돈을 보낼 수 있거든요!
