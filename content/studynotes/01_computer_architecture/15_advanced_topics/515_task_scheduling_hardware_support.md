# 작업 스케줄링 하드웨어 지원 (Task Scheduling Hardware Support)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 운영체제의 스케줄러 성능을 향상시키기 위해 CPU가 제공하는 하드웨어 기능들로, 컨텍스트 스위칭 오버헤드 감소, 스레드 우선순위 관리, 캐시 친화적 스케줄링을 지원한다.
> 2. **가치**: 컨텍스트 스위칭 시간을 30-50% 단축하고, 캐시 지역성을 고려한 스케줄링으로 전체 시스템 처리량을 10-25% 향상시킨다.
> 3. **융합**: 운영체제 스케줄러, 캐시 계층, 전력 관리, 멀티코어 아키텍처와 밀접하게 연동된다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
작업 스케줄링 하드웨어 지원은 운영체제의 태스크/스레드 스케줄링 효율을 높이기 위해 CPU 하드웨어가 제공하는 기능들의 총칭이다. 여기에는 빠른 컨텍스트 스위칭을 위한 레지스터 저장/복원 메커니즘, 캐시 친화적 스케줄링을 위한 정보 제공, 우선순위 기반 선점을 위한 인터럽트 메커니즘, 그리고 에너지 효율적 스케줄링을 위한 전력 상태 관리 등이 포함된다.

### 💡 비유
작업 스케줄링 하드웨어 지원은 "자동차의 자동 변속기"와 같다. 운전자(OS 스케줄러)가 가속 페달(태스크 실행)을 밟으면 변속기(하드웨어)가 자동으로 최적의 기어를 선택한다. 엔진 회전수(부하), 연료 효율(전력), 도로 상황(캐시 상태)을 모두 고려하여 부드럽고 효율적인 주행을 돕는다.

### 등장 배경 및 발전 과정

#### 1. 기존 기술의 치명적 한계점
- **컨텍스트 스위칭 오버헤드**: 수천 사이클 소요
- **캐시 콜드 미스**: 스위칭 후 캐시 재구축 비용
- **전력 비효율**: 스케줄링과 전력 관리의 분리

#### 2. 패러다임 변화의 역사
- **1990년대**: 기본 컨텍스트 스위칭 메커니즘
- **2000년대**: Hyper-Threading, 멀티코어 스케줄링
- **2010년대**: Cache QoS, Intel RDT, ARM big.LITTLE
- **2020년대**: Heterogeneous scheduling, AI 기반 스케줄링

### 심층 동작 원리

#### ① 빠른 컨텍스트 스위칭
```
전통적 컨텍스트 스위치 (수천 사이클):
1. 인터럽트/시스템 콜 진입
2. 범용 레지스터 저장 (R0-R31): ~32 cycles
3. 부동소수점 레지스터 저장 (F0-F31): ~32 cycles
4. SIMD 레지스터 저장 (Z0-Z31): ~100+ cycles (SVE)
5. 시스템 레지스터 저장: ~50 cycles
6. 주소 공간 전환 (TLB flush 가능): ~100+ cycles
7. 새 레지스터 복원: 위의 역순

하드웨어 가속 컨텍스트 스위치:
1. Banked Register File: 포인터만 교체 (~5 cycles)
2. XSAVEOPT: 변경된 부분만 저장 (~100 cycles)
3. Lazy FP/SIMD: 실제 사용 시 복원 (지연)
4. PCID/ASID: TLB flush 회피

개선 효과:
- 전체 스위칭 시간: 3000 cycles → 500 cycles
- 처리량: +15-25%
```

#### ② 캐시 친화적 스케줄링
```
캐시 온도(Cache Temperature) 기반 스케줄링:

1. 캐시 모니터링 (Intel CMT):
   - 각 스레드의 L3 캐시 사용량 추적
   - 캐시 미스율 측정
   - 스케줄러에 정보 제공

2. 스케줄링 결정:
   - 높은 캐시 사용 스레드 → 같은 코어 유지
   - 낮은 캐시 사용 스레드 → 마이그레이션 가능
   - 캐시 핫 스레드 → 버디 스케줄링

3. 캐시 분할 (CAT):
   - 중요 스레드: 전용 캐시 영역 할당
   - 배경 스레드: 제한된 캐시

효과:
- 캐시 미스율: -20~40%
- IPC: +10~20%
```

### 핵심 알고리즘 & 실무 코드 예시

#### 컨텍스트 스위치 벤치마크
```c
#include <stdio.h>
#include <stdint.h>
#include <time.h>
#include <pthread.h>
#include <sched.h>

#define NUM_SWITCHES 1000000
#define USE_XSAVE 1

// 컨텍스트 스위치 시간 측정
void measure_context_switch() {
    struct timespec start, end;
    cpu_set_t cpuset;

    // 단일 CPU에 고정
    CPU_ZERO(&cpuset);
    CPU_SET(0, &cpuset);
    sched_setaffinity(0, sizeof(cpuset), &cpuset);

    clock_gettime(CLOCK_MONOTONIC, &start);

    for (int i = 0; i < NUM_SWITCHES; i++) {
        // 시스템 콜을 통한 컨텍스트 스위치 유발
        sched_yield();
    }

    clock_gettime(CLOCK_MONOTONIC, &end);

    uint64_t ns = (end.tv_sec - start.tv_sec) * 1000000000ULL +
                  (end.tv_nsec - start.tv_nsec);

    printf("컨텍스트 스위치 %d회\n", NUM_SWITCHES);
    printf("총 시간: %lu ns\n", ns);
    printf("평균 스위치 시간: %.2f ns (%.2f cycles @ 3GHz)\n",
           (double)ns / NUM_SWITCHES,
           (double)ns / NUM_SWITCHES * 3.0);
}

// XSAVE vs 전통적 방식 비교
void compare_save_methods() {
    printf("\n=== 레지스터 저장 방식 비교 ===\n");

    // 전통적 방식 시뮬레이션
    struct timespec start, end;

    clock_gettime(CLOCK_MONOTONIC, &start);
    for (int i = 0; i < 10000; i++) {
        // 전통적 저장: 모든 레지스터
        __asm__ volatile (
            "push %rax\npush %rbx\npush %rcx\npush %rdx\n"
            "push %rsi\npush %rdi\npush %rbp\npush %r8\n"
            "push %r9\npush %r10\npush %r11\npush %r12\n"
            "push %r13\npush %r14\npush %r15\n"
            "pop %r15\npop %r14\npop %r13\npop %r12\n"
            "pop %r11\npop %r10\npop %r9\npop %r8\n"
            "pop %rbp\npop %rdi\npop %rsi\npop %rdx\n"
            "pop %rcx\npop %rbx\npop %rax\n"
        );
    }
    clock_gettime(CLOCK_MONOTONIC, &end);

    uint64_t traditional_ns = (end.tv_sec - start.tv_sec) * 1000000000ULL +
                              (end.tv_nsec - start.tv_nsec);

    printf("전통적 방식: %.2f ns per save/restore\n",
           (double)traditional_ns / 10000);

#ifdef USE_XSAVE
    // XSAVE 방식 (AVX 레지스터 포함)
    clock_gettime(CLOCK_MONOTONIC, &start);
    for (int i = 0; i < 10000; i++) {
        // XSAVE: 변경된 부분만 저장
        char xsave_area[4096] __attribute__((aligned(64)));
        __asm__ volatile ("xsave %0" : "=m"(xsave_area) : "a"(-1), "d"(-1));
        __asm__ volatile ("xrstor %0" : : "m"(xsave_area), "a"(-1), "d"(-1));
    }
    clock_gettime(CLOCK_MONOTONIC, &end);

    uint64_t xsave_ns = (end.tv_sec - start.tv_sec) * 1000000000ULL +
                        (end.tv_nsec - start.tv_nsec);

    printf("XSAVE 방식: %.2f ns per save/restore\n",
           (double)xsave_ns / 10000);
    printf("개선: %.1f%%\n",
           100.0 * (traditional_ns - xsave_ns) / traditional_ns);
#endif
}

int main() {
    printf("=== 컨텍스트 스위치 성능 측정 ===\n");
    measure_context_switch();
    compare_save_methods();
    return 0;
}
```
