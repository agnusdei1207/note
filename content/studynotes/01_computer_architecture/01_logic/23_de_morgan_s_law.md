# 23. 드모르간의 법칙 (De Morgan's Law)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 드모르간의 법칙은 AND와 OR 연산 간의 이중성(Duality)을 설명하는 부울 대수의 핵심 법칙으로, "논리곱의 부정은 부정의 논리합"이고 "논리합의 부정은 부정의 논리곱"이라는 원리를 수학적으로 증명한다.
> 2. **가치**: NAND 게이트만으로 모든 논리 회로를 구현할 수 있게 해주며, 복잡한 OR 회로를 AND와 NOT으로, 또는 그 반대로 변환하여 게이트 수를 20~40% 절감하고 회로 설계 유연성을 극대화한다.
> 3. **융합**: 프로그래밍의 조건문 최적화, SQL 쿼리 변환, 검색 엔진의 복합 필터, 암호학의 S-box 설계 등 모든 논리 연산 분야에서 핵심적인 변환 도구로 활용된다.

### I. 개요 (Context & Background)

#### 개념 정의

**드모르간의 법칙(De Morgan's Law)**은 19세기 영국의 수학자 어거스터스 드모르간(Augustus De Morgan)이 제안한 부울 대수의 두 가지 핵심 법칙이다. 이 법칙은 AND와 OR 연산 사이의 이중성(duality)을 설명하며, NOT 연산과 결합했을 때 발생하는 논리적 동치 관계를 수학적으로 정립했다.

두 가지 법칙:
- **제1법칙**: (A AND B)' = A' OR B' → "논리곱의 부정은 각각 부정한 것의 논리합"
- **제2법칙**: (A OR B)' = A' AND B' → "논리합의 부정은 각각 부정한 것의 논리곱"

이 법칙은 논리 회로 설계에서 AND 게이트를 OR 게이트로(또는 그 반대로) 변환할 때 필수적인 도구이며, 특히 NAND 또는 NOR 게이트만으로 모든 논리 회로를 구현할 수 있다는 이론적 근거를 제공한다.

#### 비유

> **드모르간의 법칙은 "양면 거울의 법칙"과 같다.**
>
> 어떤 그림을 거울에 비추면 좌우가 반전된다. 드모르간의 법칙도 이와 비슷하게, NOT이라는 "거울"을 통해 보면 AND가 OR로, OR가 AND로 서로 바뀌어 보인다. "모두가 찬성(AND)하는 것의 반대(NOT)"는 "누군가가 반대(OR NOT)하는 것"과 같다. 예를 들어 "철수와 영희가 모두 싫어하는 음식"의 반대는 "철수가 좋아하거나 영희가 좋아하는 음식"이다.

#### 등장 배경 및 발전 과정

1. **1847년: 드모르간의 정립**
   - George Boole의 논리 체계를 확장하며 형식화
   - 수학적 논리학의 기초 확립
   - 집합론에서의 여집합과 분배 법칙의 관계 정리

2. **1938년: 섀넌의 회로 응용**
   - Claude Shannon이 스위칭 회로에 부울 대수 적용
   - 드모르간 법칙을 통한 회로 변환의 이론적 기반 마련
   - 디지털 회로 설계의 수학적 도구로 확립

3. **1960년대: TTL 및 CMOS 설계**
   - NAND 게이트만으로 전체 회로 구현 (NAND가 Universal Gate)
   - CMOS에서 NAND/NOR의 구현 효율성 입증
   - VLSI 설계 자동화의 핵심 알고리즘

4. **현대: HDL 및 합성 도구**
   - Verilog/VHDL 논리 합성에서의 자동 최적화
   - Technology Mapping에서의 게이트 변환
   - FPGA LUT 매핑 최적화

#### NAND와 NOR의 범용성 증명

```
┌──────────────────────────────────────────────────────────────────┐
│         NAND/NOR을 이용한 기본 게이트 구현                       │
│         (드모르간 법칙의 실제 응용)                               │
└──────────────────────────────────────────────────────────────────┘

1. NAND만으로 NOT 구현
   ┌─────────────────────────────────────────────────────────────┐
   │  A ────┬─── NAND ──── A'                                    │
   │         │                                                   │
   │         └─────────────────┘                                 │
   │  A NAND A = (A * A)' = A'                                  │
   └─────────────────────────────────────────────────────────────┘

2. NAND만으로 AND 구현
   ┌─────────────────────────────────────────────────────────────┐
   │  A ─┐                                                        │
   │      ├─ NAND ─┬─ NAND ─── A * B                             │
   │  B ─┘         │                                              │
   │               └────────────────┘                            │
   │  (A NAND B) NAND (A NAND B) = ((A*B)')' = A*B             │
   └─────────────────────────────────────────────────────────────┘

3. NAND만으로 OR 구현 (드모르간 법칙 활용)
   ┌─────────────────────────────────────────────────────────────┐
   │  A ── NAND ─┐                                               │
   │              ├─ NAND ─── A + B                              │
   │  B ── NAND ─┘                                               │
   │  (A NAND A) NAND (B NAND B) = A' NAND B' = (A'*B')'       │
   │                            = A'' + B'' = A + B             │
   └─────────────────────────────────────────────────────────────┘

4. NOR만으로 NOT 구현
   ┌─────────────────────────────────────────────────────────────┐
   │  A ────┬─── NOR ──── A'                                     │
   │         │                                                   │
   │         └─────────────────┘                                 │
   │  A NOR A = (A + A)' = A'                                   │
   └─────────────────────────────────────────────────────────────┘

5. NOR만으로 OR 구현
   ┌─────────────────────────────────────────────────────────────┐
   │  A ─┐                                                        │
   │      ├─ NOR ─┬─ NOR ─── A + B                               │
   │  B ─┘         │                                              │
   │               └────────────────┘                            │
   │  (A NOR B) NOR (A NOR B) = ((A+B)')' = A+B                 │
   └─────────────────────────────────────────────────────────────┘

6. NOR만으로 AND 구현 (드모르간 법칙 활용)
   ┌─────────────────────────────────────────────────────────────┐
   │  A ── NOR ─┐                                                │
   │             ├─ NOR ─── A * B                                │
   │  B ── NOR ─┘                                                │
   │  (A NOR A) NOR (B NOR B) = A' NOR B' = (A'+B')'            │
   │                            = A'' * B'' = A * B              │
   └─────────────────────────────────────────────────────────────┘
```

                                │

                                │
                                ▼

        ┌────────────────────────────────────────────────────┐

        │      ↓                                            │

        ┌────────────────────────────────────────────────────┐
        │ F = (A' + B') * (C' + D')                         │
        │   = A'C' + A'D' + B'C' + B'D'                     │
        │                                                   │
        │ 검증: 진리표로 원식과 결과식 비교                 │
        └────────────────────────────────────────────────────┘

#### 핵심 코드: 드모르간 법칙 적용 함수

```python
#!/usr/bin/env python3
"""
드모르간 법칙 적용 및 논리식 변환
- NOT-AND ↔ NOT-OR 상호 변환
- NAND/NOR 기반 회로 변환
"""

from typing import List, Tuple
from enum import Enum

class Gate(Enum):
    AND = "AND"
    OR = "OR"
    NOT = "NOT"
    NAND = "NAND"
    NOR = "NOR"

def de_morgan_transform(gate: Gate, inputs: List[str]) -> Tuple[Gate, List[str]]:
    """
    드모르간 법칙을 적용하여 게이트 변환

    Args:
        gate: 원본 게이트 타입
        inputs: 입력 변수 리스트

    Returns:
        변환된 게이트와 (부정된) 입력 리스트
    """
    if gate == Gate.AND:
        # (A AND B)' = A' OR B'
        return Gate.OR, [f"NOT({inp})" for inp in inputs]

    elif gate == Gate.OR:
        # (A OR B)' = A' AND B'
        return Gate.AND, [f"NOT({inp})" for inp in inputs]

    elif gate == Gate.NAND:
        # (A NAND B) = (A AND B)' = A' OR B'
        return Gate.OR, [f"NOT({inp})" for inp in inputs]

    elif gate == Gate.NOR:
        # (A NOR B) = (A OR B)' = A' AND B'
        return Gate.AND, [f"NOT({inp})" for inp in inputs]

    return gate, inputs

def convert_to_nand_only(expression: str) -> str:
    """
    임의의 논리식을 NAND만으로 구현하도록 변환

    AND: A AND B → NOT(NOT(A AND B)) → (A NAND B) NAND (A NAND B)
    OR: A OR B → (NOT(A) NAND NOT(B))
    NOT: NOT(A) → A NAND A
    """
    # 이것은 개념적 구현 - 실제로는 파서 필요
    conversions = {
        "AND": "(A NAND B) NAND (A NAND B)",
        "OR": "(A NAND A) NAND (B NAND B)",
        "NOT": "A NAND A"
    }
    return f"NAND-only 변환: {expression}"

def verify_de_morgan(a: bool, b: bool) -> dict:
    """
    드모르간 법칙 검증
    """
    # 제1법칙: (A AND B)' == A' OR B'
    law1_left = not (a and b)
    law1_right = (not a) or (not b)

    # 제2법칙: (A OR B)' == A' AND B'
    law2_left = not (a or b)
    law2_right = (not a) and (not b)

    return {
        "입력": (a, b),
        "제1법칙": {
            "(A AND B)'": law1_left,
            "A' OR B'": law1_right,
            "일치": law1_left == law1_right
        },
        "제2법칙": {
            "(A OR B)'": law2_left,
            "A' AND B'": law2_right,
            "일치": law2_left == law2_right
        }
    }

# 예시 실행
if __name__ == "__main__":
    print("=== 드모르간 법칙 검증 ===")
    for a in [False, True]:
        for b in [False, True]:
            result = verify_de_morgan(a, b)
            print(f"\n입력 A={a}, B={b}")
            print(f"  제1법칙: {result['제1법칙']}")
            print(f"  제2법칙: {result['제2법칙']}")

    print("\n=== NAND만으로 기본 게이트 구현 ===")
    print("NOT: A NAND A = (A*A)' = A'")
    print("AND: (A NAND B) NAND (A NAND B) = ((A*B)')' = A*B")
    print("OR: (A NAND A) NAND (B NAND B) = A' NAND B' = (A'*B')' = A+B")
```

### III. 융합 비교 및 다각도 분석

#### 게이트 변환 비교표

| 원래 게이트 | 드모르간 변환 | 동등 게이트 | 게이트 수 변화 | CMOS 트랜지스터 |
|-------------|---------------|-------------|----------------|-----------------|
| AND + NOT | NOT + OR | NAND | 동일 | 6 → 4 (절감) |
| OR + NOT | NOT + AND | NOR | 동일 | 6 → 4 (절감) |
| AND (2입력) | NAND + NOT | NAND | +1 → 0 | 6 → 4 |
| OR (2입력) | NOR + NOT | NOR | +1 → 0 | 6 → 4 |
| NAND | NOT + OR | 동등 | 2 → 2 | 4 → 6 (증가) |
| NOR | NOT + AND | 동등 | 2 → 2 | 4 → 6 (증가) |

#### 프로그래밍 언어별 드모르간 표현

| 언어 | 원래 식 | 드모르간 변환 |
|------|---------|---------------|
| **Python** | `not (a and b)` | `(not a) or (not b)` |
| **Python** | `not (a or b)` | `(not a) and (not b)` |
| **C/C++** | `!(a && b)` | `(!a) || (!b)` |
| **C/C++** | `!(a || b)` | `(!a) && (!b)` |
| **SQL** | `NOT (A AND B)` | `(NOT A) OR (NOT B)` |
| **Java** | `!(a && b)` | `(!a) || (!b)` |
| **JavaScript** | `!(a && b)` | `(!a) || (!b)` |

### IV. 실무 적용 및 기술사적 판단

#### 실무 시나리오

**시나리오 1: CMOS 레이아웃에서 AND 게이트 제거**
```
요구사항: 복잡한 조건 판별 회로 설계
원래 식: F = (A AND B AND C) OR (D AND E)

문제: AND 게이트는 CMOS에서 6개 트랜지스터 필요
      OR 게이트도 6개 필요
      총 6*5 + 6 = 36개 트랜지스터

드모르간 적용:
F = (A AND B AND C) OR (D AND E)
F' = ((A AND B AND C) OR (D AND E))'
F' = (A AND B AND C)' AND (D AND E)'
F' = (A' OR B' OR C') AND (D' OR E')

NAND로 재표현:
F = NAND(NAND(A,B,C), NAND(D,E))

CMOS 구현:
- 3입력 NAND: 6개 트랜지스터
- 2입력 NAND: 4개 트랜지스터
- 최종 NAND: 4개 트랜지스터
- 총 14개 트랜지스터 (61% 절감!)
```

**시나리오 2: SQL 쿼리 최적화**
```
원래 쿼리 (인덱스 활용 불가):
SELECT * FROM products
WHERE NOT (category = 'A' AND price > 1000);

드모르간 변환:
SELECT * FROM products
WHERE category != 'A' OR price <= 1000;

효과:
- category에 인덱스가 있으면 활용 가능
- 쿼리 실행 계획 개선
- 실행 시간 80% 단축
```

**시나리오 3: 프로그래밍 조건문 최적화**
```
원래 코드 (복잡):
if (!(user.isActive && user.hasPermission)) {
    return "Access Denied";
}

드모르간 변환:
if (!user.isActive || !user.hasPermission) {
    return "Access Denied";
}

효과:
- Short-circuit evaluation 활용
- user.isActive가 false면 두 번째 조건 평가 안 함
- 가독성 향상
```

#### 도입 시 고려사항 체크리스트

```
[회로 설계 관련]
  □ NAND/NOR의 범용성 활용 검토
  □ CMOS 트랜지스터 수 최적화
  □ Critical Path에 미치는 영향 분석
  □ Fan-in/Fan-out 제약 확인

[소프트웨어 관련]
  □ 조건문 복잡도 분석
  □ Short-circuit evaluation 활용
  □ 코드 가독성 vs 성능 트레이드오프
  □ 테스트 커버리지 유지

[데이터베이스 관련]
  □ 인덱스 활용 가능성 검토
  □ 쿼리 실행 계획 비교
  □ NOT 연산자의 성능 영향
```

#### 안티패턴 및 주의사항

```
[Anti-pattern 1] "무조건 NAND로 변환"
   → 단계 수 증가로 지연 시간 늘어날 수 있음
   → Critical path는 AND/OR 유지 고려

[Anti-pattern 2] "프로그래밍에서 과도한 NOT"
   → 이중 부정으로 가독성 저하
   → if (!(!a || !b)) 대신 if (a && b) 사용

[Anti-pattern 3] "SQL에서 NOT 인덱스 오해"
   → NOT은 인덱스 활용 어려움
   → 드모르간으로 OR 패턴 변환 고려

[Anti-pattern 4] "변환 후 검증 생략"
   → 진리표로 반드시 검증
   → Corner case 테스트 필수

[Anti-pattern 5] "부정 연산자 우선순위 오해"
   → NOT이 AND/OR보다 우선순위 높음
   → 괄호로 명확히 표현
```

### 어린이를 위한 3줄 비유 설명

**드모르간의 법칙은 "반대말 놀이의 비밀"과 같아요!**

1. "철수와 영희가 모두 좋아하는 음식"의 반대말을 생각해보세요. "철수와 영희가 모두 싫어하는 음식"이 아니라, "철수가 싫어하거나 영희가 싫어하는 음식"이에요! "모두"가 "또는"으로 바뀌는 마법!

2. 컴퓨터에서도 똑같아요. "A와 B 둘 다 참이다"의 반대는 "A가 거짓이거나 B가 거짓이다"예요. 이 규칙을 알면 복잡한 조건도 쉽게 반대로 만들 수 있어요!

3. 이 규칙 덕분에 컴퓨터 칩 안에서 NAND라는 특별한 스위치 하나만 있어도 모든 계산을 할 수 있어요. "NAND"는 "AND의 반대"라는 뜻인데, 드모르간 법칙 덕분에 이것만 있으면 모든 논리 회로를 만들 수 있답니다!
