# 69. 데일리 스텐드업 (Daily Scrum)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데일리 스탠드업은 스크럼 팀이 **매일 정해진 시간, 정해진 장소에서 15분 이내**로 진행하는 짧은 동기화 미팅으로, **어제 한 일, 오늘 할 일, 장애물**을 공유하여 팀의 투명성과 협업을 극대화합니다.
> 2. **가치**: 일일 동기화로 **커뮤니케이션 오버헤드 50% 감소, 이슈 조기 발견 3배 향상, 팀 결속력 강화** 효과가 있으며, 스크럼의 5가지 이벤트 중 가장 빈번하게 수행되는 핵심 의식입니다.
> 3. **융합**: 카오스 엔지니어링의 **Postmortem**, 린의 **Stand-up Meeting**, XP의 **Stand-up Meeting**과 개념적으로 동일하며, 원격 근무 시에는 **비동기 스탠드업(Async Standup)** 도구와 결합합니다.

---

## I. 개요 (Context & Background)

### 1. 명확한 개념 및 정의

**데일리 스탠드업(Daily Scrum 또는 Daily Standup)**은 스크럼 프레임워크의 5가지 이벤트 중 하나로, 개발 팀이 **매일 같은 시간, 같은 장소**에서 15분 동안 진행하는 짧은 미팅입니다. 모두가 서서 진행하는 관습에서 "스탠드업"이라는 이름이 유래했습니다.

**3가지 핵심 질문 (The Three Questions)**:

| # | 질문 | 목적 |
|:---:|:---|:---|
| 1 | 어제 스프린트 목표를 위해 무엇을 했나요? | 진척 공유 |
| 2 | 오늘 스프린트 목표를 위해 무엇을 할 건가요? | 계획 공유 |
| 3 | 스프린트 목표 달성을 방해하는 장애물이 있나요? | 이슈 식별 |

**스크럼 가이드 2020 개정**:
- 3가지 질문이 **권장사항에서 필수사항으로 변경**되지 않음
- "개발 팀이 다음 24시간 동안의 작업을 계획"하는 것이 핵심
- 형식보다 **목적(동기화)**이 중요

### 2. 비유: 야구 팀의 더그아웃 미팅

```
[데일리 스탠드업 = 야구 팀 더그아웃 미팅]

야구 경기                          스크럼 프로젝트
─────────────────                 ─────────────────
매 이닝 시작 전                    매일 아침
선수들이 모여                      팀원들이 모여
"어제 경기 어떻게 됐어?"          "어제 뭐 했어?"
"오늘 상대 투수 공략법?"          "오늘 뭐 할 거야?"
"부상 있어?"                      "막히는 거 있어?"

──────────────────────────────────────────────────────
특징:
- 짧다 (1~2분)                    - 짧다 (15분)
- 전원 참여                       - 전원 참여
- 서서 한다 (빠르게)              - 서서 한다 (집중)
- 감독은 듣기만                   - 스크럼 마스터는 촉진만
- 선수끼리 소통                   - 개발팀끼리 소통

──────────────────────────────────────────────────────
절대 하지 않는 것:
- 30분씩 경기 분석 X              - 30분씩 기술 토론 X
- 감독이 잔소리 X                 - 스크럼 마스터가 지시 X
- 선수 빼놓고 회의 X              - 팀원 빼놓고 회의 X
```

### 3. 등장 배경 및 발전 과정

**1) 1993년: 스크럼의 탄생**
- Jeff Sutherland, Ken Schwaber가 스크럼 개발
- 일일 미팅을 "Daily Scrum"으로 명명

**2) 2001년: 애자일 선언문과 대중화**
- 스크럼이 애자일 방법론의 대표주자로 부상
- 데일리 스탠드업이 필수 실천법으로 정착

**3) 2010년대: 다양한 변형**
- **Walkaround Standup**: 서서 걸으며 칸반 보드 앞에서
- **Async Standup**: 슬랙, 이메일로 비동기 진행
- **Lean Coffee**: 주제 중심 자유 토론

**4) 2020년대: 원격 근무 시대**
- 줌(Zoom) 화상 스탠드업
- Range, Standup Alice 등 비동기 도구 활용

### 3. 심층 동작 원리: 5가지 안티패턴과 해결

```
|                   DAILY SCRUM ANTI-PATTERNS & SOLUTIONS                       |
================================================================================

    ─────────────────────────────────────────────────────────────────────────
    증상: "어제 DB 쿼리 최적화했는데, 원래는 인덱스를 타야 하는데..."

    • "간단히 요약해 주세요"
    • "기술적 세부사항은会后에 별도로"
    • 타임키퍼 역할 지정

    ─────────────────────────────────────────────────────────────────────────

    안티패턴 2: 스크럼 마스터 중심 (Scrum Master Centric)
    ─────────────────────────────────────────────────────────────────────────
    증상: 스크럼 마스터가 "A씨, 어제 뭐 했어요?" 일일이 질문

    문제: 팀원 간 소통이 아니라 스크럼 마스터에게 보고

    해결:
    • 팀원끼리 서로에게 말하도록 유도
    • 공 순서 돌리기 (Pass the Token)
    • 스크럼 마스터는 침묵

    ─────────────────────────────────────────────────────────────────────────

    안티패턴 3: 문제 해결 시도 (Problem Solving)
    ─────────────────────────────────────────────────────────────────────────
    즉상: "서버 에러나요" → "로그 봤어? 포트 확인했어? 방화벽?"

    문제: 2명만 남고 나머지 13명이 지루해짐

    해결:
    • "会后에 따로 보죠"
    • "오후 2시에 기술 세션 열까요?"
    • 문제는 기록만 하고 넘어감

    ─────────────────────────────────────────────────────────────────────────

    안티패턴 4: 불참 또는 지각 (Absenteeism)
    ─────────────────────────────────────────────────────────────────────────
    증상: "B씨는 아직 안 오셨네요? 기다릴까요?"

    문제: 정시 시작 안 하면 습관화됨

    해결:
    • 정시 시작, 정시 종료 (불참자는 못 듣는다)
    • 반복적 불참 시 팀 규칙으로 제재
    • 비동기 스탠드업 도구로 보완

    ─────────────────────────────────────────────────────────────────────────

    안티패턴 5: 상태 보고 (Status Report)
    ─────────────────────────────────────────────────────────────────────────
    증상: "관리자님, 제가 어제 8시간 일해서..."

    문제: 팀 동기화가 아니라 관리자 보고

    해결:
    • 관리자는 참관만 하고 발언 안 함
    • "팀원들에게 말하세요"
    • 스프린트 목표 중심으로 발표

================================================================================
```

### 4. 핵심 코드: 비동기 스탠드업 도구

```python
"""
Async Daily Standup Bot
비동기 데일리 스탠드업 자동화 봇 (Slack 연동 예시)
"""

from dataclasses import dataclass, field
from typing import List, Optional
from datetime import datetime, time
from enum import Enum

class StandupStatus(Enum):
    PENDING = "대기중"
    COMPLETED = "완료"
    MISSED = "누락"

@dataclass
class StandupResponse:
    """개인별 스탠드업 응답"""
    user_id: str
    user_name: str
    date: datetime
    yesterday: str  # 어제 한 일
    today: str      # 오늘 할 일
    blockers: str   # 장애물
    status: StandupStatus = StandupStatus.PENDING
    timestamp: Optional[datetime] = None

@dataclass
class StandupSummary:
    """일일 스탠드업 요약"""
    date: datetime
    responses: List[StandupResponse] = field(default_factory=list)
    blockers_identified: List[str] = field(default_factory=list)

    @property
    def completion_rate(self) -> float:
        """참여율"""
        if not self.responses:
            return 0.0
        completed = sum(1 for r in self.responses
                       if r.status == StandupStatus.COMPLETED)
        return (completed / len(self.responses)) * 100

    def get blockers_summary(self) -> str:
        """장애물 요약"""
        blockers = [r.blockers for r in self.responses
                   if r.blockers and r.blockers.lower() != "없음"]
        if not blockers:
            return "오늘 식별된 장애물이 없습니다 🎉"
        return "\n".join([f"• {b}" for b in blockers])

class AsyncStandupBot:
    """비동기 스탠드업 봇"""

    def __init__(self, team_name: str, standup_time: time):
        self.team_name = team_name
        self.standup_time = standup_time
        self.team_members: List[str] = []
        self.summaries: List[StandupSummary] = []

    def add_team_member(self, user_id: str, user_name: str):
        """팀 멤버 추가"""
        self.team_members.append(user_id)

    def collect_response(self, response: StandupResponse):
        """응답 수집"""
        response.status = StandupStatus.COMPLETED
        response.timestamp = datetime.now()

        # 오늘 요약에 추가
        today = datetime.now().date()
        summary = self._get_or_create_summary(today)
        summary.responses.append(response)

        if response.blockers and response.blockers.lower() != "없음":
            summary.blockers_identified.append(response.blockers)

    def _get_or_create_summary(self, date) -> StandupSummary:
        """해당 날짜 요약 반환 또는 생성"""
        for s in self.summaries:
            if s.date.date() == date:
                return s

        new_summary = StandupSummary(date=datetime.now())
        self.summaries.append(new_summary)
        return new_summary

    def generate_daily_report(self) -> str:
        """일일 스탠드업 리포트 생성"""
        today = datetime.now().date()
        summary = self._get_or_create_summary(today)

        report = [
            f"📋 **{self.team_name} 데일리 스탠드업 리포트**",
            f"📅 {today.strftime('%Y-%m-%d')}",
            f"👥 참여율: {summary.completion_rate:.0f}%",
            "",
            "---",
            "**오늘의 할 일:**",
        ]

        for r in summary.responses:
            if r.status == StandupStatus.COMPLETED:
                report.append(f"• @{r.user_name}: {r.today}")

        report.extend([
            "",
            "**⚠️ 장애물:**",
            summary.blockers_summary,
            "",
            "---",
            "*스탠드업 시간이 지났는데 아직 입력하지 않으신 분은 지금 입력해주세요!*"
        ])

        return "\n".join(report)

    def send_reminder(self, pending_users: List[str]) -> str:
        """미입력자 리마인더"""
        if not pending_users:
            return ""

        mentions = " ".join([f"<@{u}>" for u in pending_users])
        return (
            f"⏰ 스탠드업 리마인더!\n"
            f"{mentions} 아직 스탠드업을 입력하지 않으셨습니다.\n"
            f"빠른 시간 내에 입력해주세요!"
        )


# 사용 예시
if __name__ == "__main__":
    bot = AsyncStandupBot("백엔드 팀", time(9, 30))

    # 샘플 응답
    responses = [
        StandupResponse(
            user_id="U001", user_name="김개발",
            date=datetime.now(),
            yesterday="로그인 API 개발 완료",
            today="로그아웃 API 개발",
            blockers="서버 접근 권한 필요"
        ),
        StandupResponse(
            user_id="U002", user_name="이테스트",
            date=datetime.now(),
            yesterday="테스트 케이스 20개 작성",
            today="통합 테스트 수행",
            blockers="없음"
        ),
        StandupResponse(
            user_id="U003", user_name="박디자인",
            date=datetime.now(),
            yesterday="메인 페이지 UI 완성",
            today="로그인 화면 디자인",
            blockers="가이드라인 문서 대기 중"
        ),
    ]

    for r in responses:
        bot.collect_response(r)

    print(bot.generate_daily_report())
```

## III. 융합 비교 및 다각도 분석

### 1. 심층 기술 비교: 동기 vs 비동기 스탠드업

| 비교 항목 | 동기 스탠드업 (실시간) | 비동기 스탠드업 (Async) |
|:---|:---|:---|
| **참여 시간** | 정해진 시간 필수 | 24시간 내 언제든 |
| **소요 시간** | 15분 (전원 동시) | 3~5분 (개별) |
| **동기화** | 즉시 | 지연 가능 |
| **원격 팀** | 시차 문제 | 시차 무관 |
| **도구** | 줌, 구글 미트 | 슬랙, Range, Standup Alice |
| **참여율** | 높음 (동료 압력) | 변동 가능 |
| **기록** | 수동 | 자동 기록 |

### 2. 과목 융합 관점

**데일리 스탠드업 + 형상 관리**
```
[스탠드업에서 형상 관리로]

어제 한 일:
"기능 A 브랜치에서 작업 완료"

오늘 할 일:
"기능 B 브랜치 생성해서 작업"

장애물:
"머지 충돌 해결 중"

───────────────────────────────────────
→ 스탠드업 내용이 Git 활동과 연동
→ "어제 5개 커밋, 오늘 PR 예정"
→ 자동으로 형상 관리와 연계되는 트레이스
```

## IV. 실무 적용 및 기술사적 판단

### 1. 기술사적 판단 (실무 시나리오)

**[시나리오] 분산 팀의 스탠드업 운영**

*   **상황**:
    - 팀원 8명: 서울 4명, 런던 2명, 샌프란시스코 2명
    - 시차: 최대 17시간
    - 실시간 스탠드업 불가능

*   **기술사적 판단**: **하이브리드 스탠드업**

*   **실행 전략**:
    ```
    1. 비동기 스탠드업 (매일):
       - 슬랙 봇으로 24시간 내 입력
       - 자동으로 요약 리포트 생성

    2. 주간 동기 스탠드업 (주 1회):
       - 겹치는 시간대에 30분 화상 회의
       - 깊은 토론, 팀 빌딩

    3. 장애물 즉시 공지:
       - 중요한 장애물은 슬랙 채널에 즉시 포스팅
       - 스탠드업까지 기다리지 않음
    ```

### 2. 도입 시 고려사항

**시간대 선정**:
- [ ] 전원 참여 가능한 시간 (원격 팀)
- [ ] 업무 시작 직후 (동기화 목적)
- [ ] 회의가 없는 시간

**형식 결정**:
- [ ] 3가지 질문 vs 자유 형식
- [ ] 스크럼 보드 앞 vs 원격
- [ ] 동기 vs 비동기

### 3. 주의사항

*   **15분 엄수**: 초과하면 일상화되어 의미 없어짐
*   **전원 참여**: 빠지면 동기화 깨짐
*   **장애물 추적**: 보고만 하고 해결 안 하면 신뢰 잃음
