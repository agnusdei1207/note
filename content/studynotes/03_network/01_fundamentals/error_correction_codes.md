# 오류 정정 코드 (Error Correction Codes)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오류 정정 코드(Forward Error Correction, FEC)는 송신측에서 추가적인 중복 비트(패리티, 체크 비트)를 전송하여, 수신측에서 재전송 요청 없이 전송 중 발생한 오류를 자체 수정할 수 있게 하는 채널 부호화 기술입니다.
> 2. **가치**: 위성 통신, 딥 스페이스 통신, 광통신, 5G NR 등 재전송이 불가능하거나 지연이 치명적인 환경에서 필수적이며, 현대 통신 시스템의 신뢰성을 99.999%까지 향상시킵니다.
> 3. **융합**: 5G는 제어 채널에 폴라 코드(Polar Code), 데이터 채널에 LDPC를 채택하여 Shannon 한계에 0.1dB 이내로 근접하며, 양자 통신에서도 오류 정정이 핵심 요소입니다.

---

## Ⅰ. 개요 (Context & Background)

오류 정정 코드는 1940년대 해밍 코드로 시작되어, 오늘날 5G, Wi-Fi. 디스크 드라이브. 위성 통신 등 모든 디지털 통신/저장 시스템의 필수 요소입니다.

**💡 비유**: 오류 정정은 **'짝수 맞추기 퍼즐'**과 같습니다.
- **원본**: 완성된 퍼즐 그림
- **데이터**: 일부 조각이 손실되거나 변형된 퍼즐
- **패리티 비트**: 남는 조각들로, 원본을 추정할 수 있는 단서
- **오류 정정**: 손실된 조각을 주변 조각과 패턴 분석으로 복원

**핵심 개념**:
- **부호율(Code Rate)**: k/n (정보 비트 / 전송 비트), 높을수록 효율적이나 오류 정정 능력 저하
- **해밍 거리(Hamming Distance)**: 두 코드워드 간 다른 비트 수
- **최소 거리(minimum distance, d_min)**: 모든 코드워드 쌍 중 최소 해밍 거리

**오류 정정 능력**:
- 검출: d_min - 1 비트까지 오류 검출
- 정정: floor((d_min - 1) / 2) 비트까지 오류 정정

## Ⅱ. 아키텍처 및 핵심 원리

### 블록 코드 vs 길쌈 코드

| 특성 | 블록 코드 | 길쌈 코드 |
|------|----------|----------|
| 구조 | 고정 길이 블록 단위 | 연속 비트 스트림 |
| 예시 | 해밍, Reed-Solomon, LDPC | 터보, 길쌈(Viterbi) |
| 메모리 | 제한적 | 무한 (상태 유지) |
| 지연 | 낮음 | 높음 (터보) |
| 적용 | 디스크, CD/DVD, QR | 딥 스페이스, DVB |

### 주요 FEC 코드 비교

| 코드 | 연도 | d_min | 오류 정정 능력 | 부호율 | 복잡도 | 주요 적용 |
|------|------|-------|---------------|--------|--------|----------|
| **해밍(7,4)** | 1950 | 3 | 1비트 | 4/7 | 낮음 | 메모리 ECC |
| **Reed-Solomon** | 1960 | n-k+1 | 바스트 오류 | 가변 | 중간 | CD/DVD, QR |
| **BCH** | 1959 | 2t+1 | t비트 | 가변 | 중간 | 위성 |
| **길쌈** | 1957 | 무한 | 연속 | 1/2~1/3 | 높음 | 딥 스페이스 |
| **터보** | 1993 | Shannon 근접 | 우수 | 가변 | 높음 | 3G/4G |
| **LDPC** | 1962/1996 | Shannon 근접 | 우수 | 1/2~9/10 | 중간 | 5G, Wi-Fi |
| **폴라** | 2009 | 우수 | 제어 채널 최적 | 가변 | 중간 | 5G NR |

### 핵심 코드: 해밍(7,4) 인코더/디코더

```python
def hamming_74_encode(data_4bit):
    """해밍(7,4) 인코딩"""
    d1, d2, d3, d4 = data_4bit

    # 패리티 비트 계산
    p1 = d1 ^ d2 ^ d4       # 위치 1, 2, 4
    p2 = d1 ^ d3 ^ d4       # 위치 1, 3, 4
    p3 = d2 ^ d3 ^ d4       # 위치 2, 3, 4

    # 결과: p1 p2 d1 p3 d2 d3 d4
    return [p1, p2, d1, p3, d2, d3, d4]

def hamming_74_decode(received_7bit):
    """해밍(7,4) 디코딩 및 오류 정정"""
    p1, p2, d1, p3, d2, d3, d4 = received_7bit

    # 신드롬(Syndrome) 계산
    s1 = p1 ^ d1 ^ d2 ^ d4
    s2 = p2 ^ d1 ^ d3 ^ d4
    s3 = p3 ^ d2 ^ d3 ^ d4

    error_pos = s1 * 1 + s2 * 2 + s3 * 4  # 오류 위치

    if error_pos == 0:
        return [d1, d2, d3, d4]  # 오류 없음

    # 오류 정정
    received_7bit[error_pos - 1] ^= 1
    p1, p2, d1, p3, d2, d3, d4 = received_7bit
    return [d1, d2, d3, d4]
```

## Ⅲ. 융합 비교

### 5G NR 채널 부호화

| 채널 유형 | 부호화 방식 | 부호율 | 이유 |
|----------|-----------|--------|------|
| **PDSCH/PUSCH** | LDPC | 1/3~9/10 | 높은 처리량, 낮은 지연 |
| **PDCCH/PUCCH** | 폴라 코드 | 1/6~1/2 | 작은 블록, 높은 신뢰성 |
| **PBCH** | 폴라 코드 | 1/3 | 제어 정보 |

## Ⅳ. 실무 적용

### 5G NR LDPC 설계 고려사항
1. **Base Graph 선택**: BG1(큰 블록) vs BG2(작은 블록)
2. **Shift 값 최적화**: 병렬 처리 위한 순환 이동
3. **Rate Matching**: 펑처링/반복으로 정확한 전송 크기 맞춤

## Ⅴ. 기대효과 및 결론

### 정량적 효과
- **BER 향상**: 10^-3 → 10^-6 (3자릿수 개선)
- **재전송 감소**: 90% 이상 감소
