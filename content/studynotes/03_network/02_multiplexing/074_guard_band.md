# 074. 보호 대역 (Guard Band)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 주파수 분할 다중화(FDM) 시스템에서 인접 채널 간 간섭을 방지하기 위해 채널 사이에 배치하는 사용되지 않는 주파수 대역으로, 필터 특성의 한계와 주파수 드리프트를 보상하는 완충 공간
> 2. **가치**: 인접 채널 간 간섭(ICL)을 99% 이상 억제하여 전체 시스템의 신호 대 잡음비(SNR)를 10-20dB 개선하고 통신 품질 보장
> 3. **융합**: 무선통신(셀룰러, Wi-Fi), 위성통신, 케이블 TV, 광통신(WDM) 등 모든 주파수 기반 통신 시스템에서 필수 요소

## Ⅰ. 개요 (Context & Background)

### 개념
보호 대역(Guard Band)은 주파수 분할 다중화(FDM) 또는 파장 분할 다중화(WDM) 시스템에서, 인접한 두 통신 채널 사이에 존재하는 비어 있는 주파수(또는 파장) 영역을 의미한다. 이 대역은 데이터 전송에 사용되지 않으며, 송수신 필터의 완벽하지 못한 차단 특성(Transition Band)과 주파수 원천의 불안정성(Drift)으로 인한 채널 간 간섭을 방지하는 완충 지대 역할을 수행한다.

### 💡 비유
6차선 고속도로를 상상해 보자. 각 차선은 하나의 통신 채널이다. 그런데 차선 사이에는 흰색 점선이나 중앙분리대가 있다. 이 분리대가 바로 보호 대역이다. 차선끼리 너무 가까우면 옆 차가 침범할 수 있듯, 주파수 채널도 너무 가까우면 신호가 섞여버린다. 보호 대역은 "이 공간은 아무도 사용하지 마세요"라는 안전지대인 셈이다.

### 등장 배경 및 발전 과정

#### 1. 기존 기술의 치명적 한계점
- **필터의 비이상적 특성**: 현실의 필터는 무한한 경사도를 가질 수 없어, 차단 대역에서도 일부 신호가 누설됨
- **주파수 드리프트**: 발진기의 온도 변화, 노화 등으로 인해 송신 주파수가 시간에 따라 변동 (±10ppm ~ ±100ppm)
- **도플러 효과**: 이동통신에서 이동국의 속도로 인해 수신 주파수가 변화 (최대 ±1kHz at 100km/h)
- **국제 전파 규제**: 인접 국가 간 전파 간섭 방지를 위한 협약 필요

#### 2. 혁신적 패러다임 변화
- 1930년대 FM 라디오 도입 시 채널 간격 200kHz, 보호 대역 약 25kHz 설정
- 1980년대 셀룰러 이동통신(AMPS)에서 30kHz 채널 간격, 10-15kHz 보호 대역
- 1990년대 디지털 이동통신(GSM)에서 200kHz 채널, 45kHz 보호 대역
- 2000년대 OFDM 기술로 보호 대역 최소화 (CP로 대체)

#### 3. 비즈니스적 요구사항
- **주파수 자원의 희소성**: 전파는 유한 자원이므로 보호 대역 최소화가 스펙트럼 효율 핵심
- **서비스 품질 보장**: 채널 간 간섭으로 인한 통화 품질 저하 방지
- **국제 로밍**: 글로벌 표준 주파수 대역 할당을 위한 보호 대역 합의

### 심층 동작 원리

#### ① 송신측 스펙트럼 형성
```
1. 기저대역 신호 생성 (음성, 데이터 등)
2. 변조 수행 (GMSK, QPSK, QAM 등)
3. 스펙트럼 마스크 필터링
   - 3GPP TS 45.005 규정 마스크 준수
   - 인접 채널 누설전력(ACL) 제한
4. 주파수 업컨버전
```

#### ② 전파 전파 과정
```
자유 공간 전파 중:
- 감쇠 (Path Loss): 20log(4πd/λ) dB
- 페이딩 (Fading): 다중 경로로 인한 진폭 변동
- 도플러 시프트: 이동체 속도에 따른 주파수 변화
  Δf = f₀ × v/c (v: 속도, c: 빛의 속도)
```

#### ③ 수신측 채널 분리
```
1. 대역통과 필터로 원하는 채널 추출
   - 중심주파수 fc ± 대역폭/2
   - 인접 채널은 필터에 의해 감쇠됨
2. 보호 대역이 있어 인접 채널 신호의 일부가
   필터 Transition Band에 걸쳐도 영향 최소화
3. 복조 및 데이터 복원
```

#### ④ 인접 채널 간섭 (ACI) 계산
```
ACI (Adjacent Channel Interference) 공식:

ACI(dB) = Padj - Pwant - ACLR

여기서:
- Padj: 인접 채널 송신 전력 (dBm)
- Pwant: 희망 채널 수신 전력 (dBm)
- ACLR: 인접 채널 누설비 (Adjacent Channel Leakage Ratio)

일반적 요구사항:
- GSM: ACLR > 30 dB
- LTE: ACLR > 45 dB
- 5G NR: ACLR > 45 dB
```

### 핵심 알고리즘/공식 & 실무 코드 예시

#### 보호 대역 크기 산정 공식
```python
import numpy as np

def calculate_guard_band(
    channel_bandwidth: float,  # Hz
    filter_transition_band: float,  # Hz (필터 -3dB ~ -60dB 구간)
    frequency_drift_ppm: float,  # ppm (발진기 정밀도)
    center_frequency: float,  # Hz
    doppler_shift_hz: float = 0,  # Hz (이동통신)
    safety_margin_percent: float = 10  # %
) -> float:
    """
    필요한 보호 대역 크기 계산

    Returns:
        guard_band: 필요한 보호 대역 (Hz)
    """
    # 1. 필터 Transition Band의 절반 (양쪽 채널이 나눠 가짐)
    filter_margin = filter_transition_band / 2

    # 2. 주파수 드리프트 (발진기 불안정성)
    frequency_drift = center_frequency * frequency_drift_ppm / 1e6

    # 3. 도플러 효과 (이동통신)
    # doppler_shift_hz는 이미 Hz 단위로 입력

    # 4. 총 필요 마진
    total_margin = filter_margin + frequency_drift + doppler_shift_hz

    # 5. 안전 여유율 적용
    guard_band = total_margin * (1 + safety_margin_percent / 100)

    return guard_band

# 예시: LTE 10MHz 채널
gb = calculate_guard_band(
    channel_bandwidth=10e6,
    filter_transition_band=1e6,
    frequency_drift_ppm=0.1,  # 고정국 (TCXO)
    center_frequency=2.1e9,
    doppler_shift_hz=500,  # 시속 250km 차량
    safety_margin_percent=20
)
print(f"필요 보호 대역: {gb/1e3:.1f} kHz")

# 스펙트럼 효율 계산
def spectral_efficiency(
    data_rate_bps: float,
    channel_bandwidth: float,
    guard_band: float
) -> float:
    """
    스펙트럼 효율 계산 (bps/Hz)
    """
    total_bandwidth = channel_bandwidth + guard_band
    return data_rate_bps / total_bandwidth

# GSM 예시
se_gsm = spectral_efficiency(
    data_rate_bps=270.833e3,
    channel_bandwidth=200e3,
    guard_band=0  # GSM은 채널 간격 = 채널 대역폭
)
print(f"GSM 스펙트럼 효율: {se_gsm:.2f} bps/Hz")

# LTE 20MHz 예시
se_lte = spectral_efficiency(
    data_rate_bps=150e6,  # 2x2 MIMO, 64QAM
    channel_bandwidth=18e6,  # 유효 대역폭
    guard_band=2e6  # 보호 대역
)
print(f"LTE 스펙트럼 효율: {se_lte:.2f} bps/Hz")
```

#### MATLAB/Octave 필터 시뮬레이션
```matlab
% 대역통과 필터 설계 및 보호 대역 분석
fc = 1e9;           % 중심 주파수 1 GHz
bw = 10e6;          % 채널 대역폭 10 MHz
fs = 100e6;         % 샘플링 주파수 100 MHz
order = 6;          % 필터 차수

% 버터워스 대역통과 필터 설계
f_low = fc - bw/2;
f_high = fc + bw/2;
[b, a] = butter(order, [f_low f_high]/(fs/2), 'bandpass');

% 주파수 응답 분석
[h, w] = freqz(b, a, 2048, fs);
H_dB = 20*log10(abs(h));

% 인접 채널 감쇠량 계산
adj_channel_center = fc + 1.5 * bw;  % 인접 채널 중심
adj_atten = interp1(w, H_dB, adj_channel_center);

fprintf('인접 채널 감쇠량: %.1f dB\n', adj_atten);

% 필요 보호 대역 계산 (-40dB 지점)
idx_40dB = find(H_dB <= -40, 1, 'first');
guard_band_40dB = w(idx_40dB) - (fc + bw/2);
fprintf('-40dB 기준 필요 보호 대역: %.2f MHz\n', guard_band_40dB/1e6);
```

| 5G NR 100MHz | 97.2 MHz | 2.8 MHz | 100 MHz | 2.8% | 30 bps/Hz |
| OFDM (DVB-T) | 7.6 MHz | 0 | 8 MHz | 0%* | 4 bps/Hz |

*OFDM은 보호 대역 대신 Cyclic Prefix(CP) 사용

### 보호 대역 vs 대역 간섭 비교

| 간섭 유형 | 원인 | 영향 | 해결책 |
|----------|------|------|--------|
| **인접 채널 간섭 (ACI)** | 인접 채널 신호 누설 | 신호 품질 저하 | 보호 대역 확대, 필터 개선 |
| **동일 채널 간섭 (CCI)** | 동일 주파수 재사용 | SINR 저하 | 셀 계획, 주파수 재사용 거리 |
| **상호 변조 간섭 (IM)** | 비선형 소자 | 스퓨리어스 신호 | 선형 증폭기, 필터링 |
| **대역 외 간섭 (OOB)** | 원격 송신기 | 수신기 포화 | 입력 필터, AGC |

### 과목 융합 관점 분석

#### 1. [무선통신] 주파수 재사용과 보호 대역
```
주파수 재사용 거리 D:
D = R × √(3 × N)

여기서:
- R: 셀 반경
- N: 클러스터 크기

보호 대역이 클수록 N 감소 가능 (용량 증가)
하지만 스펙트럼 효율 저하
=> 최적 균형점 탐색 필요
```

#### 2. [신호처리] 필터 설계와 보호 대역
```
필터 차수 vs 보호 대역 트레이드오프:
- 고차원 필터: 급격한 롤오프 → 보호 대역 축소
- 단점: 위상 왜곡, 구현 복잡도 증가

예시:
- 4차 필터: -60dB 도달에 2MHz 필요
- 8차 필터: -60dB 도달에 0.5MHz 필요
```

#### 3. [네트워크 경제성] 스펙트럼 가치
```
주파수 경매 가격 (한국 2018년 기준):
- 3.5GHz 대역: 약 1조원 / 100MHz
- 단위당 가격: 100억원 / MHz

보호 대역 10MHz 감소 = 1,000억원 가치
=> 최소 보호 대역 설계가 경제적 핵심
```

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 시나리오)

#### 시나리오 1: 사설 LTE 망 구축
**상황**: 스마트팩토리용 사설 LTE (Band 46, 5GHz 비면허 대역)
**문제**: 20MHz 채널 4개를 80MHz 대역에 배치해야 함

**분석**:
- 이론적 최대: 20MHz × 4 = 80MHz (보호 대역 0)
- 실제 필요: 20MHz + 1MHz GB × 3 + 20MHz = 83MHz
- 대역 부족: 3MHz 초과

**의사결정**:
1. 채널 대역폭을 15MHz로 축소 (속도 25% 감소)
2. 또는 3개 채널만 사용 (용량 25% 감소)
3. 추천: 15MHz × 4 = 60MHz + GB 12MHz = 72MHz < 80MHz ✓

#### 시나리오 2: 위성 통신 시스템 설계
**상황**: Ku-band (14GHz上行/12GHz下行) 위성 트랜스폰더
**문제**: 36MHz 트랜스폰더에서 보호 대역 결정

**분석**:
- 위성 필터: 6차 버터워스, 롤오프 3MHz
- 주파수 드리프트: 위성 ±5ppm, 지구국 ±1ppm
- 도플러: LEO 위성의 경우 최대 ±100kHz

**의사결정**:
- 총 필요 마진: 1.5 + 0.07 + 0.1 = 1.67MHz
- 안전 여유 30% 적용: 2.17MHz
- 실제 트랜스폰더: 33.83MHz 유효 대역폭

#### 시나리오 3: 5G NR 동적 스펙트럼 공유 (DSS)
**상황**: LTE와 5G NR이 동일 주파수 대역 공유
**문제**: 서로 다른 파형, 보호 대역 요구사항 상이

**분석**:
- LTE: CP-OFDM, 10% 보호 대역
- 5G NR: CP-OFDM + DFT-s-OFDM, 2-5% 보호 대역
- 상호 간섭 방지 필요

**의사결정**:
- MB-SCS (Multiple Bandwidth-Specific Configuration) 적용
- LTE와 NR의 심볼 경계 정렬
- 공통 보호 대역: 5% 사용

### 도입 시 고려사항 (체크리스트)

#### 기술적
- [ ] 필터 차단 특성 측정 (삽입 손실, 롤오프)
- [ ] 발진기 정밀도 확인 (TCXO, OCXO, GPS)
- [ ] 환경 요인 (온도 범위, 노화율)
- [ ] 인접 채널 전력 레벨 차이 (최대 30dB)
- [ ] 스펙트럼 마스크 규정 준수

#### 운영/규제적
- [ ] 국가별 주파수 할당 정책
- [ ] 인접 국가와의 협조 (국경 지역)
- [ ] 전파 감시 결과 분석
- [ ] 주파수 재할당 시 마이그레이션 계획

### 주의사항 및 안티패턴 (Anti-patterns)

#### ❌ 안티패턴 1: 보호 대역 과소 설계
```
잘못된 설계:
- 이론적 최소값만 적용
- 여유 마진 0%

결과:
- 온도 변화 시 간섭 발생
- 필터 노화 후 품질 저하
```

#### ❌ 안티패턴 2: 일률적 보호 대역 적용
```
잘못된 설계:
- 모든 채널에 동일한 보호 대역
- 채널별 특성 무시

올바른 설계:
- 대역 경계: 더 큰 보호 대역
- 내부 채널: 최소 보호 대역
- 동적 할당: 트래픽 패턴 기반
```

#### ❌ 안티패턴 3: 필터 성능 과신
```
잘못된 설계:
- 데이터시트 값만 신뢰
- 양산 편차 무시

올바른 설계:
- Worst Case 기준
- 온도/전압 변동 고려
- 실측 데이터 기반 검증
```

## 👶 어린이를 위한 3줄 비유 설명

### 🎵 피아노 건반 사이의 "빈 공간"이에요
피아노를 칠 때, 건반과 건반 사이에는 작은 틈이 있어요. 그 틈을 실수로 누르면 이상한 소리가 나죠? 보호 대역은 그 틈과 같아요. 옆 건반(다른 채널) 소리가 섞이지 않게 막아주는 안전한 공간이에요.

### 🛣️ 차선 사이의 "흰색 실선"이에요
고속도로에서 차가 옆 차선으로 넘어가지 못하게 선이 그어져 있죠? 그 선 덕분에 차끼리 부딪히지 않아요. 보호 대역도 라디오 채널들이 서로 부딪히지 않게 하는 "전파 도로의 흰색 선"이에요.

### 📻 덕분에 라디오를 선명하게 들을 수 있어요
보호 대역이 없으면 100.1MHz와 100.3MHz 방송이 섞여서 무슨 말인지 모르게 돼요. 하지만 보호 대역이 있어서 각 방송국마다 자기 자리를 깔끔하게 지킬 수 있답니다!
