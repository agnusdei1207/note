# 055. 커넥션 풀 (Connection Pool)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커넥션 풀은 데이터베이스 연결(Connection)을 미리 생성하여 풀(Pool)에 보관하고, 요청 시 재사용함으로써 연결 생성/해제 오버헤드를 제거한다.
> 2. **가치**: 연결 생성 시간(수십~수백 ms)을 절약하여 응답 속도를 10~100배 향상시키고, 최대 연결 수 제한으로 DB 보호한다.
> 3. **융합**: HikariCP, c3p0, Apache DBCP 등의 라이브러리와 WAS(Thread Pool)가 협력하여 동시성을 관리한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의
**커넥션 풀(Connection Pool)**은 데이터베이스 연결 객체들을 미리 생성하여 풀에 저장해두고, 애플리케이션 요청 시 재사용하는 디자인 패턴이다.

### 💡 비유
커넥션 풀을 **택시 대기줄**에 비유할 수 있다:
- **연결 생성**: 택시 호출 (10분 대기)
- **커넥션 풀**: 택시 대기줄 (즉시 탑승)
- **반납**: 택시 대기줄로 복귀

### 연결 생성 비용
| 단계 | 시간 |
|:---|:---|
| TCP 연결 | 10~50ms |
| 인증 | 5~20ms |
| 세션 초기화 | 5~30ms |
| **총합** | **20~100ms** |

### 2. 주요 설정 파라미터
| 파라미터 | 설명 | 권장값 |
|:---|:---|:---|
| **maximumPoolSize** | 최대 연결 수 | CPU 코어 * 2 + 스파이크 |
| **minimumIdle** | 최소 유휴 연결 | maximumPoolSize와 동일 |
| **connectionTimeout** | 연결 대기 시간 | 30000ms |
| **idleTimeout** | 유휴 연결 종료 시간 | 600000ms |
| **maxLifetime** | 연결 최대 수명 | 1800000ms (30분) |

### 3. 풀 크기 계산 공식
```
connections = (core_count * 2) + effective_spindle_count

예시: 8코어 서버, SSD 1개
connections = (8 * 2) + 1 = 17
```

## Ⅲ. 융합 비교 및 다각도 분석

### 1. 커넥션 풀 라이브러리 비교
| 라이브러리 | 성능 | 특징 |
|:---|:---|:---|
| **HikariCP** | 최고 | Spring Boot 기본, 가장 빠름 |
| **c3p0** | 중간 | 안정적, 설정 다양 |
| **Apache DBCP2** | 중간 | Apache 프로젝트 |
| **Tomcat JDBC** | 높음 | Tomcat 내장 |

### 2. Thread Pool과의 관계
```text

│                    THREAD POOL vs CONNECTION POOL                           │
├─────────────────────────────────────────────────────────────────────────────┤

│   ┌────┬────┬────┬────┬────┬────┬────┬────┬────┬────┐                     │

│      │    │    │           │                                             │

│           │                                                                │
│           ▼                                                                │
```

## Ⅳ. 실무 적용

### HikariCP 설정 (Spring Boot)
```yaml
# application.yml
spring:
  datasource:
    hikari:
      maximum-pool-size: 20
      minimum-idle: 10
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000
      pool-name: MyAppPool
```

### 모니터링
```java
// HikariCP 메트릭
HikariPoolMXBean poolProxy = jmxProxy.getHikariPoolMXBean("MyAppPool");

log.info("Active: " + poolProxy.getActiveConnections());
log.info("Idle: " + poolProxy.getIdleConnections());
log.info("Waiting: " + poolProxy.getThreadsAwaitingConnection());
```

## 👶 어린이를 위한 3줄 비유
1. **택시 대기줄**: 택시를 부를 때마다 기다리는 것보다, 택시 여러 대를 미리 대기시켜두면 바로 탈 수 있죠. 커넥션 풀도 연결을 미리 만들어둬요.

2. **장난감 대여소**: 장난감을 빌릴 때마다 새로 사는 것보다, 대여소에서 빌려 쓰고 반납하면 되죠. 커넥션도 빌려 쓰고 반납해요.

3. **전화선**: 전화를 걸 때마다 선을 연결하는 것보다, 여러 개의 선을 미리 연결해두면 바로 통화할 수 있죠. 데이터베이스와도 미리 연결해둬요!
