# 056. 데이터 딕셔너리 캐시 (Data Dictionary Cache)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 딕셔너리 캐시는 시스템 카탈로그의 메타데이터를 메모리에 캐싱하여, SQL 파싱/최적화 시 디스크 I/O 없이 빠르게 접근한다.
> 2. **가치**: DDL이 실행될 때만 캐시를 무효화하고, 일반 DML은 캐시된 메타데이터로 빠른 처리가 가능하다.
> 3. **융합**: Oracle의 Shared Pool, MySQL의 Table Definition Cache가 이에 해당하며, SGA의 핵심 구성 요소다.

---

## Ⅰ. 개요

### 개념 정의
**데이터 딕셔너리 캐시(Data Dictionary Cache)**는 테이블, 인덱스, 컬럼, 권한 등의 메타데이터를 메모리에 캐싱하여 빠른 접근을 가능하게 하는 영역이다.

### 💡 비유
**도서관 카드 목록의 복사본**에 비유할 수 있다:
- 원본 목록(시스템 카탈로그)은 창고에
- 자주 쓰는 목록은 책상 위에(캐시)

## Ⅲ. 핵심 내용

### 캐시 무효화 시점
| 이벤트 | 캐시 영향 |
|:---|:---|
| DDL (CREATE, ALTER, DROP) | 관련 메타데이터 무효화 |
| DML (INSERT, UPDATE, DELETE) | 영향 없음 |
| 통계 정보 갱신 (ANALYZE) | 관련 통계 무효화 |
| 권한 변경 (GRANT, REVOKE) | 권한 캐시 무효화 |

## 👶 어린이를 위한 3줄 비유
1. **책상 위 메모**: 자주 보는 정보는 책상 위에 적어두면 매번 찾을 필요가 없죠. 메타데이터도 메모리에 적어둬요.

2. **단어장**: 모르는 단어를 사전에서 찾는 것보다, 단어장에 적어두면 빨리 볼 수 있죠. 테이블 정보도 캐시에 적어둬요.

3. **암기 과목**: 시험 볼 때 공식을 외워가면 빨리 풀 수 있죠. 자주 쓰는 정보는 외워둬서(캐시) 빨리 써요.
