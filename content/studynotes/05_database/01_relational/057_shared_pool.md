# 057. 공유 풀 (Shared Pool)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공유 풀은 Oracle SGA의 핵심 구성 요소로, Library Cache(SQL/실행계획)와 Dictionary Cache(메타데이터)를 포함한다.
> 2. **가치**: 동일 SQL의 재실행 시 파싱 생략, 메타데이터 캐싱으로 성능 향상, 세션 간 공유 가능하다.
> 3. **융합**: LRU 알고리즘으로 관리되며, 크기 부족 시 ORA-04031 에러가 발생할 수 있다.

---

## Ⅰ. 개요

### 개념 정의
**공유 풀(Shared Pool)**은 Oracle SGA(System Global Area)에서 SQL 문장, 실행 계획, 데이터 딕셔너리 정보 등을 캐싱하는 메모리 영역이다.

### 💡 비유
공유 풀을 **회의실의 화이트보드**에 비유할 수 있다:
- 여러 사람이(세션) 같은 화이트보드를(공유 풀) 사용
- 중요 내용은 지우지 않고 유지(LRU)
- 꽉 차면 오래된 내용부터 지움

### 튜닝 파라미터
```sql
-- 공유 풀 크기 설정
ALTER SYSTEM SET shared_pool_size = 500M;

-- 자동 메모리 관리
ALTER SYSTEM SET sga_target = 2G;
ALTER SYSTEM SET memory_target = 4G;
```

## 👶 어린이를 위한 3줄 비유
1. **공용 화이트보드**: 학급에 있는 큰 화이트보드처럼, 모든 학생이 같은 내용을 볼 수 있어요. 한 사람이 쓴 걸 다른 사람도 쓸 수 있죠.

2. **학급 게시판**: 학급 게시판에 중요한 공지를 붙여두면 매일 말할 필요가 없죠. SQL도 한 번 적어두면 다시 안 적어도 돼요.

3. **공용 도서**: 학교 도서관에 있는 책은 모든 학생이 빌려 볼 수 있죠. 공유 풀도 모든 프로그램이 같이 써요.
