# 038. 관계 대수 (Relational Algebra)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 관계 대수는 관계형 데이터베이스에서 데이터를 조작하기 위한 절차적 언어로, 일반 집합 연산자와 순수 관계 연산자를 통해 릴레이션을 입력받아 새로운 릴레이션을 생성하는 연산 체계다.
> 2. **가치**: SQL의 이론적 기반을 제공하며, 옵티마이저의 쿼리 변환 규칙(Transformation Rules)과 동등성 증명(Equivalence Proof)의 수학적 토대가 된다.
> 3. **융합**: 셀렉트(Selection), 프로젝션(Projection), 조인(Join) 연산은 SQL의 WHERE, SELECT, JOIN 절로 직접 매핑되며, 쿼리 최적화의 핵심 원리다.

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

**관계 대수(Relational Algebra)**는 1970년 E.F. Codd가 제안한 관계형 모델의 데이터 조작 언어로, 릴레이션(Relation)을 조작하여 새로운 릴레이션을 생성하는 연산자들의 집합이다. 관계 대수는 **"어떻게(How) 데이터를 얻을 것인가"**를 명시하는 절차적(Procedural) 언어다.

### 💡 비유

관계 대수를 **레고 조립 설명서**에 비유할 수 있다:
- **입력**: 레고 블록들 (릴레이션들)
- **연산자**: 조립 방법 (셀렉트, 프로젝트, 조인 등)
- **출력**: 완성된 레고 작품 (새로운 릴레이션)

각 연산은 하나의 명확한 동작을 수행하고, 여러 연산을 조합(Composite)하면 복잡한 결과를 만들 수 있다.

### 등장 배경

1. **데이터 조작의 수학적 기반 필요**: 파일 시스템의 임의적 접근 방식을 탈피하여, 집합론에 기반한 체계적 데이터 조작 방법 필요
2. **쿼리 언어의 이론적 토대**: SQL 같은 선언적 언어가 내부적으로 어떻게 수행되는지 정의
3. **쿼리 최적화의 근거**: 동등한 결과를 내는 여러 대수식 중 최적 선택

│  │                                                                       │ │

### 2. 연산자 상세 분석

#### 2.1 셀렉션 (Selection, σ)

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                    SELECTION (σ) - 수평적 부분집합                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  정의: σ_condition(R)                                                       │
│  조건을 만족하는 튜플(행)들만 선택                                          │
│                                                                             │
│  입력 릴레이션 R:                         결과 σ_salary>50(R):              │
│  ┌────────────────────────────┐          ┌────────────────────────────┐    │
│  │ ID │ Name  │ Salary │ Dept │          │ ID │ Name  │ Salary │ Dept │    │
│  ├────┼───────┼────────┼──────┤          ├────┼───────┼────────┼──────┤    │
│  │ 1  │ Alice │ 60000  │ D1   │   ───►   │ 1  │ Alice │ 60000  │ D1   │    │
│  │ 2  │ Bob   │ 45000  │ D2   │          │ 3  │ Carol │ 70000  │ D1   │    │
│  │ 3  │ Carol │ 70000  │ D1   │          └────────────────────────────┘    │
│  │ 4  │ Dave  │ 40000  │ D3   │                                              │
│  └────────────────────────────┘          2 rows selected                    │
│                                                                             │
│  SQL 매핑: SELECT * FROM R WHERE salary > 50000                             │
│                                                                             │
│  특성:                                                                      │
│  • 카디널리티 감소 (|σ(R)| ≤ |R|)                                          │
│  • 차수 보존 (deg(σ(R)) = deg(R))                                          │
│  • 교환법칙: σ_c1(σ_c2(R)) = σ_c2(σ_c1(R)) = σ_c1∧c2(R)                    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 2.2 프로젝션 (Projection, π)

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                    PROJECTION (π) - 수직적 부분집합                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  정의: π_attributes(R)                                                      │
│  지정한 속성(열)들만 선택, 중복 제거                                        │
│                                                                             │
│  입력 릴레이션 R:                         결과 π_Name,Dept(R):              │
│  ┌────────────────────────────┐          ┌──────────────────┐              │
│  │ ID │ Name  │ Salary │ Dept │          │ Name  │ Dept     │              │
│  ├────┼───────┼────────┼──────┤          ├───────┼──────────┤              │
│  │ 1  │ Alice │ 60000  │ D1   │   ───►   │ Alice │ D1       │              │
│  │ 2  │ Bob   │ 45000  │ D2   │          │ Bob   │ D2       │              │
│  │ 3  │ Carol │ 70000  │ D1   │          │ Carol │ D1       │              │
│  │ 4  │ Dave  │ 40000  │ D3   │          │ Dave  │ D3       │              │
│  └────────────────────────────┘          └──────────────────┘              │
│                                                                             │
│  SQL 매핑: SELECT DISTINCT Name, Dept FROM R                                │
│                                                                             │
│  특성:                                                                      │
│  • 차수 감소 (deg(π(R)) ≤ deg(R))                                          │
│  • 카디널리티 감소 가능 (중복 제거)                                         │
│  • 주의: 프로젝션은 자동으로 중복 제거!                                     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 2.3 조인 (Join, ⋈)

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                         JOIN (⋈) - 관계 결합                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  정의: R ⋈_condition S                                                      │
│  조건을 기반으로 두 릴레이션 결합                                           │
│                                                                             │
│  ┌─────────────────┐        ┌─────────────────┐                            │
│  │ R (Employee)    │        │ S (Department)  │                            │
│  ├────┼────┼───────┤        ├────┼────────────┤                            │
│  │ ID │Name│ DeptID│        │ ID │ DeptName   │                            │
│  ├────┼────┼───────┤        ├────┼────────────┤                            │
│  │ 1  │ Kim│  10   │        │ 10 │ Sales      │                            │
│  │ 2  │ Lee│  20   │        │ 20 │ Marketing  │                            │
│  │ 3  │ Park│ 10   │        │ 30 │ Finance    │                            │
│  └────┴────┴───────┘        └────┴────────────┘                            │
│                                                                             │
│  Natural Join (R ⋈ S) on DeptID = ID:                                       │
│  ┌───────────────────────────────────────┐                                 │
│  │ ID │ Name │ DeptID │ DeptName        │                                 │
│  ├────┼──────┼────────┼─────────────────┤                                 │
│  │ 1  │ Kim  │  10    │ Sales           │                                 │
│  │ 2  │ Lee  │  20    │ Marketing       │                                 │
│  │ 3  │ Park │  10    │ Sales           │                                 │
│  └────┴──────┴────────┴─────────────────┘                                 │
│                                                                             │
│  종류:                                                                      │
│  • Theta Join (θ-join): 일반 조건 조인                                     │
│  • Equi Join: 등호(=) 조건만 사용                                          │
│  • Natural Join: 동일 속성명 기준 자동 등호 조인, 중복 컬럼 제거           │
│  • Outer Join: Left ⟕, Right ⟖, Full ⟗                                   │
│                                                                             │
│  SQL 매핑: SELECT * FROM R JOIN S ON R.DeptID = S.ID                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 2.4 디비전 (Division, ÷)

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                       DIVISION (÷) - 포함 관계 추출                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  정의: R ÷ S                                                                │
│  R의 속성 중 S의 모든 속성값과 관련된 튜플 추출                             │
│                                                                             │
│  예시: "모든 과목을 수강한 학생 찾기"                                       │
│                                                                             │
│  R (수강):                              S (과목):                           │
│  ┌───────────────┐                     ┌────────┐                          │
│  │ 학생 │ 과목   │                     │ 과목   │                          │
│  ├──────┼────────┤                     ├────────┤                          │
│  │ Kim  │ Math   │                     │ Math   │                          │
│  │ Kim  │ English│                     │ English│                          │
│  │ Lee  │ Math   │                     └────────┘                          │
│  │ Lee  │ English│                                                         │
│  │ Lee  │ Science│                     R ÷ S 결과:                         │
│  │ Park │ Math   │                     ┌────────┐                          │
│  └──────┴────────┘                     │ 학생   │                          │
│                                         ├────────┤                          │
│  Kim: Math, English (모두 포함) ──►    │ Kim    │                          │
│  Lee: Math, English, Science (모두) ──►│ Lee    │                          │
│  Park: Math만 (English 없음) ──X       └────────┘                          │
│                                                                             │
│  SQL 구현: NOT EXISTS / NOT IN 패턴                                         │
│  SELECT DISTINCT 학생 FROM R R1                                             │
│  WHERE NOT EXISTS (                                                         │
│      SELECT * FROM S                                                        │
│      WHERE NOT EXISTS (                                                     │
│          SELECT * FROM R R2                                                 │
│          WHERE R2.학생 = R1.학생 AND R2.과목 = S.과목                       │
│      )                                                                      │
│  );                                                                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3. 관계 대수 vs SQL 매핑

| 관계 대수 | SQL | 설명 |
|:---|:---|:---|
| σ_condition(R) | SELECT * FROM R WHERE condition | 조건 필터링 |
| π_attrs(R) | SELECT attrs FROM R | 컬럼 선택 |
| R ∪ S | SELECT ... FROM R UNION SELECT ... FROM S | 합집합 |
| R ∩ S | SELECT ... FROM R INTERSECT SELECT ... FROM S | 교집합 |
| R - S | SELECT ... FROM R EXCEPT/MINUS SELECT ... FROM S | 차집합 |
| R × S | SELECT ... FROM R, S | 카티션 프로덕트 |
| R ⋈ S | SELECT ... FROM R JOIN S ON ... | 조인 |
| R ÷ S | NOT EXISTS 패턴 | 디비전 |

### 4. 관계 대수 동등성 규칙 (Optimization Rules)

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                 RELATIONAL ALGEBRA EQUIVALENCE RULES                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. Selection Cascade (셀렉션 연쇄)                                         │
│     σ_c1(σ_c2(R)) ≡ σ_c1∧c2(R) ≡ σ_c2(σ_c1(R))                             │
│                                                                             │
│  2. Projection Cascade (프로젝션 연쇄)                                      │
│     π_A(π_A,B(R)) ≡ π_A(R)                                                  │
│                                                                             │
│  3. Selection-Projection Commute (셀렉션-프로젝션 교환)                     │
│     π_A(σ_c(R)) ≡ σ_c(π_A(R))  (단, c는 A의 속성만 참조)                   │
│                                                                             │
│  4. Selection-Join Commute (셀렉션-조인 교환)                               │
│     σ_c(R ⋈ S) ≡ σ_c(R) ⋈ S  (단, c는 R의 속성만 참조)                     │
│                                                                             │
│  5. Join Associativity (조인 결합 법칙)                                     │
│     (R ⋈ S) ⋈ T ≡ R ⋈ (S ⋈ T)                                              │
│                                                                             │
│  6. Join Commutativity (조인 교환 법칙)                                     │
│     R ⋈ S ≡ S ⋈ R                                                           │
│                                                                             │
│  옵티마이저는 이 규칙들을 활용하여 동등한 쿼리 중 최적 선택                │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 관계 대수 vs 관계 해석

| 특성 | 관계 대수 | 관계 해석 |
|:---|:---|:---|
| **언어 유형** | 절차적 (Procedural) | 비절차적 (Non-procedural) |
| **질문 형태** | "어떻게 찾을 것인가?" | "무엇을 찾을 것인가?" |
| **기반** | 연산자 조합 | 술어 논리 (Predicate Logic) |
| **표현** | σ, π, ⋈ 등 기호 | {t \| P(t)} 형식 |
| **SQL과의 관계** | 실행 계획 수립 기반 | SQL WHERE 절 기반 |

### 2. 관계 대수 완전성

**관계적으로 완전(Relationally Complete)**: 어떤 질의 언어가 관계 대수의 모든 연산을 표현할 수 있으면 관계적으로 완전하다고 한다. SQL은 관계적으로 완전하며, 이는 SQL이 관계 대수로 표현 가능한 모든 질의를 수행할 수 있음을 의미한다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 시나리오: 쿼리 최적화에 관계 대수 적용

**문제 상황**: 복잡한 SQL 쿼리의 성능 최적화 필요

**기술사적 접근**:
1. SQL을 관계 대수로 변환
2. 동등성 규칙 적용으로 대수식 단순화
3. 비용 기반 최적화로 최적 실행 계획 도출

```text
원본 쿼리:
SELECT name FROM employee WHERE dept_id IN (
    SELECT dept_id FROM department WHERE location = 'Seoul'
)

관계 대수 변환:
π_name(σ_dept_id∈π_dept_id(σ_location='Seoul'(Department))(Employee))

최적화 (조인으로 변환):
π_name(Employee ⋈ σ_location='Seoul'(Department))
```

## 👶 어린이를 위한 3줄 비유 설명

1. **관계 대수는 데이터 요리법이에요**: "야채를 씻고(Selection), 껍질을 벗기고(Projection), 섞어서(Division) 샐러드를 만드세요"처럼, 데이터를 어떻게 가공할지 순서대로 알려주는 레시피예요.

2. **레고 블록 조립법 같아요**: 어떤 블록을 선택하고(Selection), 어떤 면을 보여주고(Projection), 어떻게 연결할지(Join) 정하는 조립 설명서예요.

3. **SQL의 비밀 원리예요**: 우리가 쓰는 SQL은 편한 말로 되어 있지만, 사실 컴퓨터 안에서는 이 관계 대수라는 수학 공식으로 바뀌어서 실행돼요!
