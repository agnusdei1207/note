# 041. 셀렉트 (Select, σ) - 수평적 부분집합

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 셀렉트(Selection, σ)는 릴레이션에서 지정한 조건(Predicate)을 만족하는 튜플(행)들만 추출하여 수평적 부분집합을 생성하는 관계 대수 연산자다.
> 2. **가치**: SQL의 WHERE 절로 직접 매핑되며, 조건 필터링은 쿼리 최적화의 가장 기본적인 연산으로 인덱스 활용의 핵심이다.
> 3. **융합**: 셀렉션은 조인 전에 수행(Pushdown)되면 처리 데이터량을 획기적으로 줄여 성능을 최적화할 수 있다.

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

**셀렉트(Selection, σ)**은 릴레이션 R에서 주어진 조건(술어, Predicate)을 만족하는 튜플들의 부분집합을 반환한다. "수평적 부분집합"이라고도 하며, 행(Row) 단위의 필터링을 수행한다.

**주의**: SQL의 SELECT 문과 관계 대수의 Selection(σ)은 다르다!
- 관계 대수 σ: WHERE 절에 해당 (행 필터링)
- SQL SELECT: 관계 대수의 Projection(π)에 해당 (열 선택)

### 💡 비유

셀렉트를 **사람 구별하기**에 비유할 수 있다:
- 전체 모임 사람들 중에서
- "안경 쓴 사람만 앞으로 나오세요"
- 조건에 맞는 사람들만 선택

### 수학적 정의

```
σ_p(R) = {t | t ∈ R ∧ p(t) = true}

여기서:
- σ: 셀렉트 연산자
- p: 술어(조건)
- R: 입력 릴레이션
- t: 튜플
```

### 2. 조건 표현식

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                    SELECTION CONDITIONS (조건 표현식)                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  비교 연산자:                                                               │
│  • = : 같다                                                                 │
│  • ≠ : 같지 않다                                                            │
│  • <, > : 미만, 초과                                                        │
│  • ≤, ≥ : 이하, 이상                                                        │
│                                                                             │
│  논리 연산자:                                                               │
│  • ∧ (AND) : 논리곱                                                         │
│  • ∨ (OR) : 논리합                                                          │
│  • ¬ (NOT) : 부정                                                           │
│                                                                             │
│  예시:                                                                      │
│  • σ_salary>50000(Employee)                                                 │
│  • σ_dept='IT'∧salary>60000(Employee)                                       │
│  • σ_dept='IT'∨dept='Sales'(Employee)                                       │
│                                                                             │
│  SQL 매핑:                                                                  │
│  • σ_c(R) → SELECT * FROM R WHERE c                                         │
│  • σ_c1∧c2(R) → SELECT * FROM R WHERE c1 AND c2                             │
│  • σ_c1∨c2(R) → SELECT * FROM R WHERE c1 OR c2                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3. 셀렉트 연산 법칙

```python
# 셀렉트 연산 법칙 (Optimization Rules)

# 1. 교환 법칙 (Commutativity)
# σ_c1(σ_c2(R)) = σ_c2(σ_c1(R))

# 2. 연쇄 법칙 (Cascading / Conjunction)
# σ_c1(σ_c2(R)) = σ_c1∧c2(R)

# 3. 분배 법칙 (Distribution over Union)
# σ_c(R ∪ S) = σ_c(R) ∪ σ_c(S)

# 4. 분배 법칙 (Distribution over Difference)
# σ_c(R - S) = σ_c(R) - σ_c(S)

# 5. 조인에 대한 분배 (Distribution over Join)
# σ_c(R ⋈ S) = σ_c(R) ⋈ S  (c가 R의 속성만 참조할 때)

# 실무 활용: 조인 전에 셀렉트를 수행하면 데이터량 감소!
```

## Ⅲ. 융합 비교 및 다각도 분석

### 1. 셀렉트 vs 프로젝트

| 특성 | 셀렉트 (σ) | 프로젝트 (π) |
|:---|:---|:---|
| **방향** | 수평 (행) | 수직 (열) |
| **SQL** | WHERE | SELECT |
| **중복** | 발생하지 않음 | 발생 가능 (자동 제거) |
| **결과 크기** | 행 수 감소 | 열 수 감소 |

### 2. 인덱스와 셀렉트

| 상황 | 인덱스 없음 | 인덱스 있음 |
|:---|:---|:---|
| **검색 방식** | Full Table Scan | Index Range Scan |
| **시간 복잡도** | O(n) | O(log n) |
| **I/O** | 모든 페이지 | 인덱스 + 필요 페이지 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 시나리오: 쿼리 최적화

**원본 쿼리**:
```sql
SELECT * FROM orders o
JOIN customers c ON o.customer_id = c.id
WHERE o.order_date >= '2024-01-01' AND c.status = 'ACTIVE';
```

**최적화 (Selection Pushdown)**:
```sql
-- 옵티마이저가 내부적으로 수행
SELECT * FROM
    (SELECT * FROM orders WHERE order_date >= '2024-01-01') o
JOIN
    (SELECT * FROM customers WHERE status = 'ACTIVE') c
ON o.customer_id = c.id;
```

## 👶 어린이를 위한 3줄 비유

1. **사람 고르기**: 모임에 온 사람들 중에서 "안경 쓴 사람만 남으세요!"라고 하면 안경 쓴 사람들만 남는 것처럼, 셀렉트는 조건에 맞는 데이터만 골라요.

2. **필터 역할**: 커피 필터가 찌꺼기는 걸러내고 맑은 커피만 내려주듯, 셀렉트는 원하는 조건의 데이터만 걸러줘요.

3. **WHERE 절의 친구**: SQL에서 WHERE 뒤에 적는 조건이 바로 이 셀렉트 연산이에요. "나는 20살 이상만 볼래!"라고 말하는 것과 같아요.
