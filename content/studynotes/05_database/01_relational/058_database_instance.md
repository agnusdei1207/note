# 058. 데이터베이스 인스턴스 (Database Instance)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터베이스 인스턴스는 메모리 구조(SGA)와 백그라운드 프로세스의 결합으로, 데이터베이스에 접근하는 실행 단위다.
> 2. **가치**: 인스턴스는 데이터베이스(디스크 데이터)와 분리되어, 하나의 데이터베이스에 여러 인스턴스가 접근 가능하다(RAC).
> 3. **융합**: Oracle RAC, MySQL 다중 인스턴스 등 고가용성 아키텍처의 기반이다.

---

## Ⅰ. 개요

### 개념 정의
**데이터베이스 인스턴스(Database Instance)** = **SGA(System Global Area)** + **백그라운드 프로세스(Background Processes)**

### 💡 비유
데이터베이스 인스턴스를 **음악 연주팀**에 비유할 수 있다:
- **데이터베이스**: 악보(데이터 파일)
- **인스턴스**: 연주팀(메모리 + 연주자들)
- **SGA**: 연주 공간(메모리)
- **백그라운드 프로세스**: 각 악기 담당자

### 백그라운드 프로세스 역할
| 프로세스 | 역할 |
|:---|:---|
| **DBWn** | Dirty Page를 디스크에 기록 |
| **LGWR** | Redo Log Buffer를 로그 파일에 기록 |
| **CKPT** | 체크포인트 수행, 헤더 갱신 |
| **SMON** | 시스템 복구, 공간 통합 |
| **PMON** | 프로세스 복구, 세션 정리 |
| **ARCn** | Redo Log 아카이빙 |
| **RECO** | 분산 트랜잭션 복구 |

## Ⅲ. 융합 분석

### Instance vs Database
| 구분 | Instance | Database |
|:---|:---|:---|
| **구성** | SGA + 프로세스 | 데이터 파일 |
| **위치** | 메모리 | 디스크 |
| **생명주기** | 시작/종료 | 영구 존재 |
| **관계** | N:1 (RAC) | 1:N (다중 DB) |

## 👶 어린이를 위한 3줄 비유
1. **연주팀**: 악보(데이터베이스)는 한 권이지만, 연주팀(인스턴스)은 여럿이 쓸 수 있어요. 동시에 여러 팀이 같은 악보로 연주할 수 있죠.

2. **주방 팀**: 식재료(데이터)는 창고에 있고, 주방 팀(인스턴스)이 요리해요. 주방이 많으면 동시에 많은 요리를 할 수 있죠.

3. **독서실**: 책(데이터)은 도서관에 있지만, 독서실(인스턴스)에서 읽어요. 독서실이 넓으면(메모리가 크면) 더 많은 책을 펴놓고 볼 수 있죠.
