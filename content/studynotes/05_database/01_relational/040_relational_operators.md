# 040. 순수 관계 연산자 (Relational Operators)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 순수 관계 연산자는 관계형 데이터 모델에 고유한 연산자로, 셀렉션(Selection, σ), 프로젝션(Projection, π), 조인(Join, ⋈), 디비전(Division, ÷)을 포함하며 릴레이션의 구조적 특성을 활용한다.
> 2. **가치**: 셀렉션은 행(튜플) 필터링, 프로젝션은 열(속성) 선택, 조인은 릴레이션 결합, 디비전은 포함 관계 추출을 수행하여 SQL 질의의 이론적 기반이다.
> 3. **융합**: 이 연산자들은 SQL의 WHERE, SELECT, JOIN, NOT EXISTS 구문으로 직접 매핑되며, 옵티마이저의 쿼리 변환과 최적화 규칙의 핵심이다.

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

**순수 관계 연산자(Relational Operators)**는 관계형 데이터 모델의 구조적 특성(릴레이션 = 튜플의 집합)을 활용하여 데이터를 조작하는 연산자들이다. 일반 집합 연산자(Union, Intersection, Difference, Cartesian)와 달리, 관계형 모델에 특화된 연산을 제공한다.

### 💡 비유

**순수 관계 연산자를 엑셀 시트 조작**에 비유할 수 있다:
- **셀렉션(Selection)**: 행 필터링 ("급여가 5000만 원 이상인 직원만 보기")
- **프로젝션(Projection)**: 열 선택 ("이름과 부서만 보기")
- **조인(Join)**: 시트 결합 ("직원 시트와 부서 시트를 부서ID로 연결")
- **디비전(Division)**: 포함 관계 ("모든 필수 과목을 이수한 학생 찾기")

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 셀렉션 (Selection, σ) - 수평적 부분집합

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                    SELECTION (σ) - 수평적 부분집합                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  정의: σ_condition(R) = {t | t ∈ R ∧ condition(t)}                         │
│  조건을 만족하는 튜플(행)들의 부분집합                                      │
│                                                                             │
│  ┌────────────────────────────────────────────────────────────────────┐    │
│  │                      원본 릴레이션 R                                │    │
│  │  ┌────┬──────┬────────┬───────┐                                   │    │
│  │  │ ID │ Name │ Salary │ Dept  │                                   │    │
│  │  ├────┼──────┼────────┼───────┤                                   │    │
│  │  │ 1  │ Kim  │ 45000  │ D1    │ ← 제외                            │    │
│  │  │ 2  │ Lee  │ 65000  │ D2    │ ← 선택 ✓                          │    │
│  │  │ 3  │ Park │ 70000  │ D1    │ ← 선택 ✓                          │    │
│  │  │ 4  │ Choi │ 55000  │ D3    │ ← 선택 ✓                          │    │
│  │  │ 5  │ Jung │ 40000  │ D2    │ ← 제외                            │    │
│  │  └────┴──────┴────────┴───────┘                                   │    │
│  └────────────────────────────────────────────────────────────────────┘    │
│                           ↓ σ_Salary>50000(R)                               │
│  ┌────────────────────────────────────────────────────────────────────┐    │
│  │                      결과 릴레이션                                  │    │
│  │  ┌────┬──────┬────────┬───────┐                                   │    │
│  │  │ ID │ Name │ Salary │ Dept  │                                   │    │
│  │  ├────┼──────┼────────┼───────┤                                   │    │
│  │  │ 2  │ Lee  │ 65000  │ D2    │                                   │    │
│  │  │ 3  │ Park │ 70000  │ D1    │                                   │    │
│  │  │ 4  │ Choi │ 55000  │ D3    │                                   │    │
│  │  └────┴──────┴────────┴───────┘                                   │    │
│  └────────────────────────────────────────────────────────────────────┘    │
│                                                                             │
│  SQL 매핑: SELECT * FROM R WHERE Salary > 50000                             │
│                                                                             │
│  특성:                                                                      │
│  • 카디널리티: |σ(R)| ≤ |R| (행 수 감소 또는 동일)                         │
│  • 차수: deg(σ(R)) = deg(R) (열 수 동일)                                   │
│  • 교환법칙: σ_c1(σ_c2(R)) = σ_c2(σ_c1(R))                                │
│  • 연쇄법칙: σ_c1(σ_c2(R)) = σ_c1∧c2(R)                                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2. 프로젝션 (Projection, π) - 수직적 부분집합

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                    PROJECTION (π) - 수직적 부분집합                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  정의: π_attributes(R) = {t[attributes] | t ∈ R}                           │
│  지정한 속성(열)들만 선택, 중복 자동 제거                                   │
│                                                                             │
│  ┌────────────────────────────────────────────────────────────────────┐    │
│  │                      원본 릴레이션 R                                │    │
│  │  ┌────┬──────┬────────┬───────┐                                   │    │
│  │  │ ID │ Name │ Salary │ Dept  │                                   │    │
│  │  ├────┼──────┼────────┼───────┤                                   │    │
│  │  │ 1  │ Kim  │ 45000  │ D1    │                                   │    │
│  │  │ 2  │ Lee  │ 65000  │ D2    │                                   │    │
│  │  │ 3  │ Park │ 70000  │ D1    │                                   │    │
│  │  │ 4  │ Choi │ 55000  │ D3    │                                   │    │
│  │  └────┴──────┴────────┴───────┘                                   │    │
│  └────────────────────────────────────────────────────────────────────┘    │
│                           ↓ π_Name,Dept(R)                                  │
│  ┌────────────────────────────────────────────────────────────────────┐    │
│  │                      결과 릴레이션                                  │    │
│  │  ┌──────┬───────┐                                                │    │
│  │  │ Name │ Dept  │                                                │    │
│  │  ├──────┼───────┤                                                │    │
│  │  │ Kim  │ D1    │                                                │    │
│  │  │ Lee  │ D2    │                                                │    │
│  │  │ Park │ D1    │                                                │    │
│  │  │ Choi │ D3    │                                                │    │
│  │  └──────┴───────┘                                                │    │
│  └────────────────────────────────────────────────────────────────────┘    │
│                                                                             │
│  SQL 매핑: SELECT DISTINCT Name, Dept FROM R                                │
│  (관계 대수는 자동 중복 제거, SQL은 DISTINCT 필요)                          │
│                                                                             │
│  특성:                                                                      │
│  • 카디널리티: |π(R)| ≤ |R| (중복 제거로 감소 가능)                        │
│  • 차수: deg(π(R)) ≤ deg(R) (선택한 열 수)                                 │
│  • 연쇄법칙: π_A(π_A,B(R)) = π_A(R)                                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3. 조인 (Join, ⋈) - 관계 결합

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                         JOIN (⋈) - 관계 결합                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  정의: R ⋈_condition S = σ_condition(R × S)                                │
│  카티션 프로덕트 + 조건 필터링                                              │
│                                                                             │
│  조인 유형:                                                                 │
│                                                                             │
│  1. Theta Join (θ-Join): 임의 조건 사용                                    │
│     R ⋈_R.A > S.B S                                                        │
│                                                                             │
│  2. Equi Join: 등호(=) 조건만 사용                                         │
│     R ⋈_R.ID = S.ID S                                                      │
│                                                                             │
│  3. Natural Join (⋈): 동일 이름 속성 자동 등호 조인, 중복 제거             │
│     R ⋈ S ≡ π_distinct(σ_R.A=S.A(R × S))                                   │
│                                                                             │
│  예시 (Natural Join):                                                       │
│  ┌────────────────────────────────────────────────────────────────────┐    │
│  │  R (Employee)              S (Department)                          │    │
│  │  ┌────┬──────┬────────┐   ┌────────┬──────────┐                    │    │
│  │  │ ID │ Name │ DeptID │   │ DeptID │ DeptName │                    │    │
│  │  ├────┼──────┼────────┤   ├────────┼──────────┤                    │    │
│  │  │ 1  │ Kim  │ 10     │   │ 10     │ Sales    │                    │    │
│  │  │ 2  │ Lee  │ 20     │   │ 20     │ IT       │                    │    │
│  │  │ 3  │ Park │ 10     │   │ 30     │ HR       │                    │    │
│  │  └────┴──────┴────────┘   └────────┴──────────┘                    │    │
│  └────────────────────────────────────────────────────────────────────┘    │
│                           ↓ R ⋈ S                                           │
│  ┌────────────────────────────────────────────────────────────────────┐    │
│  │  ┌────┬──────┬────────┬──────────┐                                 │    │
│  │  │ ID │ Name │ DeptID │ DeptName │                                 │    │
│  │  ├────┼──────┼────────┼──────────┤                                 │    │
│  │  │ 1  │ Kim  │ 10     │ Sales    │                                 │    │
│  │  │ 2  │ Lee  │ 20     │ IT       │                                 │    │
│  │  │ 3  │ Park │ 10     │ Sales    │                                 │    │
│  │  └────┴──────┴────────┴──────────┘                                 │    │
│  └────────────────────────────────────────────────────────────────────┘    │
│                                                                             │
│  SQL 매핑:                                                                  │
│  SELECT * FROM R NATURAL JOIN S;                                            │
│  SELECT * FROM R JOIN S ON R.DeptID = S.DeptID;                             │
│                                                                             │
│  특성:                                                                      │
│  • 교환법칙: R ⋈ S = S ⋈ R                                                 │
│  • 결합법칙: (R ⋈ S) ⋈ T = R ⋈ (S ⋈ T)                                    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4. 디비전 (Division, ÷) - 포함 관계 추출

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                       DIVISION (÷) - 포함 관계 추출                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  정의: R ÷ S = {t | ∀s ∈ S, (t, s) ∈ R}                                    │
│  S의 모든 튜플과 관련된 R의 튜플                                            │
│                                                                             │
│  직관적 이해: "S의 모든 것을 가진 R의 대상 찾기"                            │
│                                                                             │
│  ┌────────────────────────────────────────────────────────────────────┐    │
│  │  R (수강)                          S (필수 과목)                    │    │
│  │  ┌──────┬────────┐                ┌────────┐                       │    │
│  │  │ 학생 │ 과목   │                │ 과목   │                       │    │
│  │  ├──────┼────────┤                ├────────┤                       │    │
│  │  │ Kim  │ Math   │                │ Math   │                       │    │
│  │  │ Kim  │ English│                │ English│                       │    │
│  │  │ Lee  │ Math   │                └────────┘                       │    │
│  │  │ Lee  │ English│                                                 │    │
│  │  │ Lee  │ Science│                R ÷ S:                          │    │
│  │  │ Park │ Math   │                ┌──────┐                         │    │
│  │  └──────┴────────┘                │ 학생 │                         │    │
│  │                                    ├──────┤                         │    │
│  │  Kim: Math, English ✓ (모두 포함)│ Kim  │                         │    │
│  │  Lee: Math, English, Science ✓   │ Lee  │                         │    │
│  │  Park: Math만 ✗ (English 없음)   └──────┘                         │    │
│  └────────────────────────────────────────────────────────────────────┘    │
│                                                                             │
│  SQL 구현 (이중 NOT EXISTS 패턴):                                           │
│  SELECT DISTINCT 학생 FROM R R1                                             │
│  WHERE NOT EXISTS (                                                         │
│      SELECT * FROM S                                                        │
│      WHERE NOT EXISTS (                                                     │
│          SELECT * FROM R R2                                                 │
│          WHERE R2.학생 = R1.학생 AND R2.과목 = S.과목                       │
│      )                                                                      │
│  );                                                                         │
│                                                                             │
│  해석: "S에 있는 어떤 과목도 수강하지 않은 학생이 없는 학생"               │
│        = "S의 모든 과목을 수강한 학생"                                      │
│                                                                             │
│  특성:                                                                      │
│  • 차수: deg(R ÷ S) = deg(R) - deg(S)                                      │
│  • 디비전은 조인의 역연산 성질                                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5. 순수 관계 연산자 비교 요약

| 연산자 | 기호 | 방향 | SQL 매핑 | 주요 용도 |
|:---|:---|:---|:---|:---|
| 셀렉션 | σ | 수평 (행) | WHERE | 조건 필터링 |
| 프로젝션 | π | 수직 (열) | SELECT | 컬럼 선택 |
| 조인 | ⋈ | 결합 | JOIN | 테이블 결합 |
| 디비전 | ÷ | 포함 | NOT EXISTS | "모든 것을 가진" 질의 |

### 6. 연산자 조합 (Composite Operations)

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                    COMPOSITE OPERATIONS (복합 연산)                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  실제 질의는 여러 연산자를 조합하여 표현:                                   │
│                                                                             │
│  질의: "IT 부서 직원의 이름과 급여를 조회하시오"                            │
│                                                                             │
│  단계별 표현:                                                               │
│  1. 부서 테이블에서 IT 부서 찾기: σ_DeptName='IT'(Department)               │
│  2. 직원과 조인: Employee ⋈ (1의 결과)                                      │
│  3. 이름, 급여만 선택: π_Name,Salary((2의 결과))                            │
│                                                                             │
│  관계 대수식:                                                               │
│  π_Name,Salary(Employee ⋈ σ_DeptName='IT'(Department))                      │
│                                                                             │
│  SQL:                                                                       │
│  SELECT e.Name, e.Salary                                                    │
│  FROM Employee e                                                            │
│  JOIN Department d ON e.DeptID = d.DeptID                                   │
│  WHERE d.DeptName = 'IT';                                                   │
│                                                                             │
│  ┌────────────────────────────────────────────────────────────────────┐    │
│  │  Employee ──⋈──┬─────────────────────────────────────────────────┐ │    │
│  │                │                                                 │ │    │
│  │  Department ──σ_DeptName='IT'────────────────────────────────────┘ │    │
│  │                                                                     │    │
│  │                           ↓                                         │    │
│  │                        π_Name,Salary                                │    │
│  │                           ↓                                         │    │
│  │                        결과                                         │    │
│  └────────────────────────────────────────────────────────────────────┘    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 1. 셀렉션 vs 프로젝션

| 특성 | 셀렉션 (σ) | 프로젝션 (π) |
|:---|:---|:---|
| **조작 대상** | 행 (튜플) | 열 (속성) |
| **부분집합 유형** | 수평적 | 수직적 |
| **결과 크기** | 행 수 감소 | 열 수 감소 |
| **중복 발생** | 없음 | 발생 가능 (자동 제거) |
| **SQL** | WHERE | SELECT |

### 2. 조인 vs 카티션 프로덕트

| 특성 | 카티션 프로덕트 (×) | 조인 (⋈) |
|:---|:---|:---|
| **결과 크기** | \|R\| × \|S\| | ≤ \|R\| × \|S\| |
| **조건** | 없음 | 존재 |
| **관계** | 조인 = 카티션 + 셀렉션 | |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 시나리오: 최적화 가능한 쿼리 작성

**문제 상황**: 복잡한 쿼리의 성능 최적화

**관계 대수 기반 최적화 예시**:
```
원본: σ_c1(π_A,B(σ_c2(R ⋈ S)))

최적화 규칙 적용:
1. 셀렉션을 조인 내부로 이동 (Selection Pushdown)
   = π_A,B(σ_c1(σ_c2(R) ⋈ S))

2. 조인 전에 데이터량 감소 → 성능 향상
```

**SQL로 표현**:
```sql
-- 비최적화
SELECT A, B FROM R JOIN S ON ... WHERE c1 AND c2;

-- 최적화 (옵티마이저가 자동 수행)
SELECT A, B FROM (SELECT * FROM R WHERE c2) R2 JOIN S ON ... WHERE c1;
```

## 👶 어린이를 위한 3줄 비유 설명

1. **셀렉션은 친구 고르기**: 반 친구 중에서 "키가 150cm 이상인 친구"만 골라내는 거예요. 조건에 맞는 친구들만 남기는 것이죠.

2. **프로젝션은 정보 선택**: 친구들의 명단에서 "이름과 반 번호만" 적어내는 거예요. 키, 몸무게, 주소 같은 건 빼고 필요한 것만 보여주는 것이죠.

3. **조인은 두 명단 연결**: 우리 반 명단과 동아리 명단을 연결해서 "같은 동아리 친구"를 찾는 거예요. 두 명단에 같은 이름이 있으면 연결해 주는 것이죠.
