# 060. 서버리스 데이터베이스 (Serverless Database)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 서버리스 데이터베이스는 컴퓨팅과 스토리지가 완전히 분리되어, 사용량에 따라 자동으로 스케일 업/다운하고 유휴 시 비용이 발생하지 않는 클라우드 네이티브 DB다.
> 2. **가치**: Amazon Aurora Serverless, Azure SQL Database Serverless 등이 있으며, 트래픽 급증 시 자동 확장, 유휴 시 0 ACU로 축소된다.
> 3. **융합**: 마이크로서비스, 개발/테스트 환경, 간헐적 워크로드에 최적화되어 있다.

---

## Ⅰ. 개요

### 개념 정의
**서버리스 데이터베이스(Serverless Database)**는 사용자가 서버를 프로비저닝하거나 관리할 필요 없이, 워크로드에 따라 자동으로 리소스를 할당/해제하는 클라우드 데이터베이스 모델이다.

### 💡 비유
서버리스 DB를 **택시 앱**에 비유할 수 있다:
- **기존 DB**: 차량 소유 (항상 유지비)
- **서버리스 DB**: 택시 호출 (탈 때만 비용)
- 탑승객이 많으면 대형 차량, 없으면 소형 차량

### 특징
1. **자동 스케일링**: 워크로드에 따른 ACU(컴퓨팅 단위) 자동 조절
2. **Pay-per-use**: 사용한 만큼만 과금
3. **Zero Administration**: 서버 관리 불필요
4. **High Availability**: 기본적으로 다중 AZ

### 3. 주요 서버리스 DB 비교
| 제품 | 제공사 | 특징 |
|:---|:---|:---|
| **Aurora Serverless v2** | AWS | MySQL/PostgreSQL 호환, 밀리초 스케일링 |
| **Azure SQL Serverless** | Microsoft | SQL Server 호환, 자동 일시 중지 |
| **Cloud SQL Serverless** | GCP | MySQL/PostgreSQL, 자동 스케일링 |
| **PlanetScale** | PlanetScale | MySQL 호환, Vitess 기반 |
| **Neon** | Neon | PostgreSQL 호환, 브랜치 기능 |
| **Turso** | Turso | SQLite 호환, 엣지 배포 |

## Ⅲ. 융합 분석

### 1. 서버리스 DB vs 기존 DB
| 특성 | 기존 DB (Provisioned) | 서버리스 DB |
|:---|:---|:---|
| **용량 계획** | 필요 | 불필요 |
| **스케일링** | 수동 | 자동 |
| **유휴 비용** | 발생 | 최소화/없음 |
| **콜드 스타트** | 없음 | 있을 수 있음 |
| **최대 성능** | 높음 | 제한적 |
| **비용 예측** | 용이 | 변동적 |

## Ⅳ. 실무 적용

### Aurora Serverless v2 구성
```text
# Terraform 예시
resource "aws_rds_cluster" "serverless" {
  engine              = "aurora-mysql"
  engine_version      = "8.0"
  database_name       = "mydb"

  # Serverless v2 설정
  engine_mode         = "provisioned"  # v2는 provisioned 모드 사용
  serverlessv2_scaling_configuration {
    min_capacity = 0.5  # 최소 ACU
    max_capacity = 1    # 최대 ACU
  }
}

resource "aws_rds_cluster_instance" "serverless" {
  cluster_identifier = aws_rds_cluster.serverless.id
  instance_class     = "db.serverless"
  engine             = aws_rds_cluster.serverless.engine
  engine_version     = aws_rds_cluster.serverless.engine_version
}
```

### 비용 비교 예시
```text
시나리오: 월 720시간 중 200시간만 활성

기존 Aurora (db.r5.large, $0.17/hr):
  720시간 × $0.17 = $122.40/월

Aurora Serverless v2 (0.5-4 ACU, 평균 1 ACU, $0.06/hr):
  활성 200시간 × $0.06 = $12.00/월
  유휴 520시간 × $0.03 = $15.60/월
  총: $27.60/월

절감: 77% 비용 절감
```

## 👶 어린이를 위한 3줄 비유
1. **택시 앱**: 내 차를 사는 게 아니라 필요할 때만 택시를 타는 것과 같아요. 서버리스 DB도 필요할 때만 써요.

2. **전기 요금**: 24시간 불을 켜는 것보다, 필요할 때만 켜면 전기세를 아낄 수 있죠. 서버리스도 쓸 때만 비용이 들어요.

3. **레고 대여**: 레고를 사는 게 아니라 필요할 때만 빌려 쓰는 것과 같아요. 많이 필요하면 많이 빌리고, 안 쓰면 반납하죠. 서버리스도 이래요!
