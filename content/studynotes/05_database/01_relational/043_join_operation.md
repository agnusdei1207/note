# 043. 조인 (Join, ⋈)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 조인(Join, ⋈)은 두 개 이상의 릴레이션을 공통 속성이나 조건을 기준으로 결합하여 새로운 릴레이션을 생성하는 관계 대수 연산자다.
> 2. **가치**: 정규화로 분리된 테이블들을 논리적으로 재결합하며, Nested Loop, Sort Merge, Hash Join 등 다양한 구현 알고리즘이 존재한다.
> 3. **융합**: 조인은 관계형 데이터베이스의 핵심 연산으로, 옵티마이저의 실행 계획 수립과 성능 튜닝에서 가장 중요한 요소다.

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

**조인(Join, ⋈)**은 두 릴레이션 R과 S를 결합하여 새로운 릴레이션을 생성하는 연산이다. 수학적으로는 카티션 프로덕트(R × S)에 조건을 적용한 것과 같다: R ⋈_condition S = σ_condition(R × S)

### 💡 비유

조인을 **두 명함첩 연결**에 비유할 수 있다:
- 명함첩 A: 직원 명함 (이름, 부서ID)
- 명함첩 B: 부서 명함 (부서ID, 부서명)
- 조인: 같은 부서ID를 가진 명함끼리 옆에 붙이기

### 등장 배경

정규화 과정에서 하나의 엔티티가 여러 테이블로 분리된다. 이를 다시 결합하여 의미 있는 정보를 얻기 위해 조인이 필요하다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 조인 유형 분류

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                         JOIN TYPES (조인 유형)                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                    INNER JOIN (내부 조인)                            │   │
│  │  • Equi Join: 등호(=) 조건만 사용                                   │   │
│  │  • Natural Join: 동일 속성명 자동 조인, 중복 제거                    │   │
│  │  • Theta Join: 임의 조건 사용                                        │   │
│  │  • Self Join: 자기 자신과 조인                                       │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                    OUTER JOIN (외부 조인)                            │   │
│  │  • Left Outer Join (⟕): 왼쪽 테이블 모든 행 유지                     │   │
│  │  • Right Outer Join (⟖): 오른쪽 테이블 모든 행 유지                  │   │
│  │  • Full Outer Join (⟗): 양쪽 테이블 모든 행 유지                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                    SPECIAL JOINS (특수 조인)                         │   │
│  │  • Cross Join (×): 카티션 프로덕트                                   │   │
│  │  • Semi Join: 조인 조건 만족하는 한쪽만 반환                         │   │
│  │  • Anti Join: 조인 조건 불만족하는 한쪽 반환                         │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2. 내부 조인 (Inner Join)

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                         INNER JOIN (내부 조인)                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  R (Employee)              S (Department)       R ⋈ S (Inner Join)          │
│  ┌────┬──────┬────────┐   ┌────────┬───────┐   ┌────┬──────┬────────┬───────┐│
│  │ ID │ Name │ DeptID │   │ DeptID │ DName │   │ ID │ Name │ DeptID │ DName ││
│  ├────┼──────┼────────┤   ├────────┼───────┤   ├────┼──────┼────────┼───────┤│
│  │ 1  │ Kim  │ 10     │   │ 10     │ Sales │   │ 1  │ Kim  │ 10     │ Sales ││
│  │ 2  │ Lee  │ 20     │   │ 20     │ IT    │   │ 2  │ Lee  │ 20     │ IT    ││
│  │ 3  │ Park │ 10     │   │ 30     │ HR    │   │ 3  │ Park │ 10     │ Sales ││
│  │ 4  │ Jung │ 40     │   └────────┴───────┘   └────┴──────┴────────┴───────┘│
│  └────┴──────┴────────┘                                                      │
│                                                  3 rows                      │
│  Jung (DeptID=40)은 S에 없어서 제외                                         │
│  HR (DeptID=30)은 R에 없어서 제외                                           │
│                                                                             │
│  SQL:                                                                       │
│  SELECT * FROM Employee R                                                   │
│  INNER JOIN Department S ON R.DeptID = S.DeptID;                            │
│                                                                             │
│  또는                                                                       │
│  SELECT * FROM Employee R JOIN Department S ON R.DeptID = S.DeptID;         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

│                        OUTER JOIN (외부 조인)                               │
├─────────────────────────────────────────────────────────────────────────────┤

│  LEFT OUTER JOIN (R ⟕ S):                                                   │
│  ┌────┬──────┬────────┬───────┐                                            │

│  ┌────┬──────┬────────┬───────┐                                            │

│  ├────┼──────┼────────┼───────┤                                            │

│  │ 2  │ Lee  │ 20     │ IT    │                                            │
│  │ 3  │ Park │ 10     │ Sales │                                            │
│  │ 4  │ Jung │ 40     │ NULL  │                                            │
│  │NULL│ NULL │ 30     │ HR    │ ← S의 모든 행도 유지                       │

│                                                                             │
│  SQL: SELECT * FROM R FULL JOIN S ON R.DeptID = S.DeptID;                   │
│                                                                             │

### 4. 조인 알고리즘 비교

| 알고리즘 | 시간 복잡도 | 적합한 상황 | 특징 |
|:---|:---|:---|:---|
| **Nested Loop** | O(M×N) | 소량 데이터, 인덱스 있음 | 단순, 인덱스 활용 |
| **Sort Merge** | O(M log M + N log N) | 정렬된 데이터, 대량 | 정렬 비용 |
| **Hash Join** | O(M+N) | 대량, 등가 조인 | 메모리 많이 필요 |

## Ⅲ. 융합 비교 및 다각도 분석

### 1. Natural Join vs Equi Join

| 특성 | Natural Join | Equi Join |
|:---|:---|:---|
| **조건** | 동일 이름 속성 자동 | 명시적 조건 |
| **중복 컬럼** | 자동 제거 | 유지됨 |
| **SQL** | NATURAL JOIN | JOIN ON a=b |

### 2. 조인과 성능

```sql
-- 조인 순서 최적화 예시
-- 작은 테이블을 드라이빙 테이블로

-- 비효율
SELECT * FROM large_table l
JOIN small_table s ON l.id = s.id;

-- 효율 (small_table 드라이빙)
SELECT * FROM small_table s
JOIN large_table l ON s.id = l.id;
```

## Ⅳ. 실무 적용

### 시나리오: N+1 문제 해결

**문제**: ORM에서 연관 엔티티 조회 시 N+1 쿼리 발생

**해결**: JOIN FETCH 사용
```sql
-- N+1 문제 발생
SELECT * FROM orders;
-- 각 order마다
SELECT * FROM customers WHERE id = ?;

-- 해결 (JOIN FETCH)
SELECT o, c FROM orders o JOIN FETCH o.customer c;
```

## 👶 어린이를 위한 3줄 비유

1. **짝 찾기**: 두 반의 학생들이 서로 같은 번호를 가진 친구끼리 짝을 이루는 것과 같아요. 1번 학생은 1번 학생과, 2번은 2번과 짝이 되죠.

2. **퍼즐 맞추기**: 두 조각 퍼즐이 서로 맞는 부분끼리 딱 맞춰지는 것처럼, 조인은 두 테이블의 같은 값을 가진 데이터끼리 연결해요.

3. **명함 합치기**: 한 명함첩에는 이름과 부서번호가 있고, 다른 명함첩에는 부서번호와 부서명이 있을 때, 같은 부서번호를 가진 명함을 옆에 붙여서 하나로 만드는 거예요!
