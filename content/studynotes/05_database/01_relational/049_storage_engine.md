# 049. 스토리지 엔진 (Storage Engine)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스토리지 엔진은 DBMS에서 데이터의 물리적 저장, 검색, 갱신을 담당하는 핵심 모듈로, MySQL의 InnoDB, MyISAM이 대표적이다.
> 2. **가치**: 동일한 SQL 인터페이스 아래에서 서로 다른 저장 방식과 성능 특성을 제공하여, 워크로드에 맞는 최적의 엔진 선택이 가능하다.
> 3. **융합**: 스토리지 엔진은 버퍼 풀, 로깅, 인덱싱, 트랜잭션 관리 등 DBMS의 핵심 기능을 통합 구현한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

**스토리지 엔진(Storage Engine)**은 데이터베이스 관리 시스템(DBMS)에서 데이터의 실제 저장, 검색, 갱신을 담당하는 소프트웨어 모듈이다. 상위 계층(SQL Parser, Optimizer)과 분리되어 있어, 같은 DBMS에서도 여러 스토리지 엔진을 선택적으로 사용할 수 있다.

### 💡 비유

스토리지 엔진을 **자동차 엔진**에 비유할 수 있다:
- **차체(SQL 인터페이스)**: 같은 운전대, 페달
- **가솔린 엔진(InnoDB)**: 안정적, ACID 보장
- **디젤 엔진(MyISAM)**: 단순, 빠른 읽기

### 2. InnoDB vs MyISAM 비교

| 특성 | InnoDB | MyISAM |
|:---|:---|:---|
| **트랜잭션** | O (ACID) | X |
| **락킹** | Row-level | Table-level |
| **외래키** | O | X |
| **MVCC** | O | X |
| **클러스터드 인덱스** | O (PK) | X |
| **Full-text Search** | O (5.6+) | O |
| **압축** | O | O |
| **복구** | 자동 복구 | 수동 복구 |
| **적합한 용도** | OLTP | 읽기 중심 |

### 4. InnoDB 핵심 기능

| 기능 | 설명 |
|:---|:---|
| **Buffer Pool** | 데이터/인덱스 페이지 캐싱 (기본 128MB) |
| **Change Buffer** | Secondary Index 변경 지연 처리 |
| **Adaptive Hash Index** | 자주 접근하는 페이지 자동 해시 인덱싱 |
| **Redo Log** | WAL 기반 복구 로그 |
| **Undo Log** | MVCC, 롤백 지원 |
| **Doublewrite Buffer** | 페이지 손상 방지 |

## Ⅲ. 융합 비교 및 다각도 분석

### 1. 워크로드별 스토리지 엔진 선택

| 워크로드 | 추천 엔진 | 이유 |
|:---|:---|:---|
| **전자상거래** | InnoDB | 트랜잭션, 동시성 |
| **로그 분석** | MyISAM/Archive | 읽기 중심, 압축 |
| **세션 저장** | Memory | 빠른 접근, 휘발성 허용 |
| **데이터 웨어하우스** | Column Store | 분석 쿼리 |

### 2. Oracle vs PostgreSQL 스토리지

| DBMS | 구조 | 특징 |
|:---|:---|:---|
| **Oracle** | 통형 아키텍처 | Tablespace, Segment, Extent |
| **PostgreSQL** | MVCC + TOAST | 다중 버전, 대형 데이터 분리 |

## Ⅳ. 실무 적용

### 시나리오: 엔진 선택

```sql
-- 테이블별 스토리지 엔진 지정
CREATE TABLE orders (
    id INT PRIMARY KEY,
    user_id INT,
    amount DECIMAL(10,2)
) ENGINE=InnoDB;  -- 트랜잭션 필요

CREATE TABLE access_logs (
    id INT PRIMARY KEY,
    access_time DATETIME,
    path VARCHAR(255)
) ENGINE=MyISAM;  -- 읽기 전용, 쓰기 빈번

-- 엔진 변경
ALTER TABLE logs ENGINE=Archive;
```

### 성능 튜닝

```sql
-- InnoDB Buffer Pool 크기 조정
SET GLOBAL innodb_buffer_pool_size = 4G;

-- 로그 파일 크기
SET GLOBAL innodb_log_file_size = 512M;

-- Flush 방식
SET GLOBAL innodb_flush_method = O_DIRECT;
```

## 👶 어린이를 위한 3줄 비유

1. **자동차 엔진**: 자동차마다 엔진이 다르듯, 데이터베이스에도 데이터를 저장하는 방식이 달라요. 가솔린 엔진(InnoDB)은 안정적이고, 경주용 엔진(Memory)은 빠르지만 위험해요.

2. **옷장 종류**: 옷장에도 여러 종류가 있죠? 일반 옷장(InnoDB)은 모든 기능이 있고, 압축 옷장(Archive)은 공간을 적게 써요.

3. **요리 도구**: 프라이팬, 냄비, 오븐이 각각 다른 요리에 쓰이듯, 스토리지 엔진도 어떤 데이터를 다루느냐에 따라 다른 걸 써요!
