# 061. 릴레이션 (Relation)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 릴레이션은 관계형 데이터 모델의 핵심 데이터 구조로, 2차원 테이블 형태로 표현되며 속성(Attribute)과 튜플(Tuple)로 구성된다.
> 2. **가치**: 수학적 집합론에 기반하여 중복 제거, 무순서성, 원자성 등의 특성을 가지며, SQL 테이블의 이론적 기반이다.
> 3. **융합**: 릴레이션 스키마(내포, Intension)와 릴레이션 인스턴스(외연, Extension)로 구분된다.

---

## Ⅰ. 개요

### 개념 정의
**릴레이션(Relation)**은 관계형 데이터 모델에서 데이터를 저장하는 기본 단위로, 행(Tuple)과 열(Attribute)로 구성된 2차원 테이블 구조다.

### 💡 비유
릴레이션을 **엑셀 시트**에 비유할 수 있다:
- **속성(Attribute)**: 컬럼(열) - 이름, 나이, 주소 등
- **튜플(Tuple)**: 행(Row) - 한 사람의 데이터

### 수학적 정의
```
릴레이션 R: D1 × D2 × ... × Dn의 부분집합

여기서:
- D1, D2, ..., Dn: 도메인 (속성이 가질 수 있는 값의 집합)
- n: 차수 (Degree) - 속성의 개수
- |R|: 카디널리티 (Cardinality) - 튜플의 개수
```

## Ⅱ. 아키텍처 및 핵심 원리

### 1. 릴레이션 구조
```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                    RELATION STRUCTURE                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│                    Relation: STUDENT (학생)                                  │
│                                                                             │
│     Attribute (속성) →                                                      │
│   ┌────────┬────────┬────────┬────────┬────────┐                           │
│   │  학번   │  이름   │  학과   │  학년   │  GPA   │                           │
│   ├────────┼────────┼────────┼────────┼────────┤                           │
│   │ 2024001│ 김철수  │ 컴퓨터  │   1    │  3.5   │ ← Tuple 1                │
│   ├────────┼────────┼────────┼────────┼────────┤                           │
│   │ 2024002│ 이영희  │ 전자    │   1    │  3.8   │ ← Tuple 2                │
│   ├────────┼────────┼────────┼────────┼────────┤                           │
│   │ 2024003│ 박민수  │ 컴퓨터  │   2    │  3.2   │ ← Tuple 3                │
│   └────────┴────────┴────────┴────────┴────────┘                           │
│       ↑                                                                     │
│   Domain (도메인)                                                           │
│                                                                             │
│   • Degree (차수): 5 (속성 개수)                                            │
│   • Cardinality (카디널리티): 3 (튜플 개수)                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2. 릴레이션의 특성
| 특성 | 설명 |
|:---|:---|
| **튜플의 무순서** | 행의 순서는 의미 없음 |
| **속성의 무순서** | 열의 순서는 의미 없음 |
| **튜플의 유일성** | 중복 튜플 불가 (키로 식별) |
| **속성의 원자성** | 각 셀은 하나의 값만 가짐 |

### 3. 스키마 vs 인스턴스
```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                    SCHEMA vs INSTANCE                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   Relation Schema (릴레이션 스키마) = Intension (내포)                       │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  • 릴레이션 이름                                                     │  │
│   │  • 속성 이름과 도메인                                                │  │
│   │  • 제약조건                                                          │  │
│   │  • 정적 (잘 변하지 않음)                                              │  │
│   │                                                                      │  │
│   │  STUDENT(학번: CHAR(10), 이름: VARCHAR(20), ...)                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   Relation Instance (릴레이션 인스턴스) = Extension (외연)                   │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  • 특정 시점의 실제 데이터                                            │  │
│   │  • 튜플의 집합                                                        │  │
│   │  • 동적 (계속 변함)                                                   │  │
│   │                                                                      │  │
│   │  {(2024001, 김철수, 컴퓨터, 1, 3.5),                                │  │
│   │   (2024002, 이영희, 전자, 1, 3.8),                                   │  │
│   │   ...}                                                               │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 용어 매핑

| 이론적 용어 | SQL 용어 | 설명 |
|:---|:---|:---|
| Relation | Table | 데이터 구조 |
| Attribute | Column | 속성/열 |
| Tuple | Row | 레코드/행 |
| Domain | Data Type | 값의 범위 |
| Degree | Column Count | 속성 개수 |
| Cardinality | Row Count | 튜플 개수 |
| Schema | Table Definition | 구조 정의 |
| Instance | Table Data | 실제 데이터 |

## 👶 어린이를 위한 3줄 비유
1. **반 명부**: 반 명부에는 학생 이름, 번호, 주소 등이 적히죠? 이게 릴레이션이에요. 한 줄이 한 학생이고, 항목들이 속성이에요.

2. **엑셀 표**: 엑셀에서 표를 만들면 행과 열이 있죠? 릴레이션도 이와 같아요. 열은 무엇(이름, 나이)이고, 행은 누구(철수, 영희)예요.

3. **도서 목록**: 도서관에 어떤 책이 있는지 목록이 있죠? 책 제목, 저자, 위치가 적혀 있어요. 이 목록 전체가 릴레이션이에요.
