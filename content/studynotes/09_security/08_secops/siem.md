# SIEM (Security Information and Event Management)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이기종 이기종 보안 장비(방화벽,, IDS, IPS, WAF 등)와 애플리케이션(웹 서버, DB, ERP 등)에서 발생하는 로그를 중앙 집계, 상관 분석하여 보안 위협을 실시간 탐지하는 SOC 핵심 플랫폼입니다.
> 2. **가치**: "단일 창문"에서 조망 가능한 통합 보안 가시성 확보, 평균 탐지 시간(MTTD) 80% 단축, 분석가의 피로도 감소, 규정 준수(PCI DSS 10.5)를 달성합니다.
> 3. **융합**: SOAR와 결합하여 자동 대응 플레이북 실행, 위협 인텔리전스(TI)와 연동하여 실시간 인텔리전스 제공, UEBA와 통합하여 이상 행위 탐지합니다.

---

### Ⅰ. 개요 (Context & Background)

#### 1. 개념 및 기술적 정의
**SIEM(Security Information and Event Management)**은 이기종 로그 수집, 정규화, 상관 분석, 알림을 제공하는 통합 보안 관제 플랫폼입니다. 1990년대 후반에 등장하여 현재 SOC(Security Operations Center) 운영의 핵심 인프라입니다.

```
SIEM 핵심 기능:
1. 로그 수집 (Log Collection)
   - 다양한 소스에서 로그 수집
   - 에이전트, Syslog, API 등 지원

2. 정규화 (Normalization)
   - 이기종 로그를 공통 포맷으로 변환
   - CEF(Common Event Format), LEEF, JSON 등

3. 인덱싱 (Indexing)
   - 빠른 검색을 위한 인덱싱
   - Elasticsearch, Splunk 등

4. 상관 분석 (Correlation)
   - 다중 이벤트 간 패턴 매칭
   - 규칙 기반 탐지

5. 알림 (Alerting)
   - 임계값 초과, 패턴 매칭 시 알림
   - 이메일, SMS, 티켓 시스템 연동

6. 대시보드 (Dashboard)
   - 시각화
   - 실시간 모니터링
```

#### 2. 비유를 통한 이해
SIEM은 **'통합 관제 센터(CCTV 관제실)'**에 비유할 수 있습니다.

- **개별 CCTV(로그 소스)**: 각각의 건물, 도로, 주차장 카메라
- **SIEM**: 모든 화면을 한곳에서 모니터링
  - "3층 회의실에서 5명 이상 이상 행동" 탐지
  - "지하 주차장에서 낯선 사람이 30분 이상 머무름" 경고

#### 2. SIEM 주요 벤더 비교

| 벤더 | 제품 | 강점 | 약점 |
|:---|:---|:---|:---|
| **Splunk** | Splunk Enterprise/SIEM | 강력한 검색, 유연성 | 높은 비용 |
| **Elastic** | Elastic Security | 오픈소스 기반 | 운영 복잡성 |
| **Microsoft** | Sentinel | Azure 통합 | Azure 종속 |
| **IBM** | QRadar | 엔터프라이즈 | 구형 아키텍처 |
| **Google** | Chronicle | 클라우드 네이티브 | 제한적 온프레미스 |
| **Sumo Logic** | Cloud SIEM | SaaS | 커스터마이징 제한 |

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

#### 1. SIEM vs SOAR 비교

| 특성 | SIEM | SOAR |
|:---|:---|:---|
| **주 기능** | 로그 분석 | 자동화 대응 |
| **입력** | 로그, 이벤트 | SIEM 알림 |
| **출력** | 알림, 대시보드 | 자동 조치 |
| **운영** | Tier 1 (탐지) | Tier 2 (대응) |

#### 2. 과목 융합 관점

**네트워크 보안과 융합**
- 방화벽/IDS/WAF 로그 수집
- 네트워크 플로우 분석

**애플리케이션 보안과 융합**
- 웹 앱 로그 분석
- 인증 실패 패턴 탐지

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 기술사적 판단 (실무 시나리오)

**시나리오 1: SIEM 도입 ROI 분석**
- 비용: Splunk 연간 라이선스 $500K+
- 효과: MTTD 80% 감소, 분석가 2명 감소
- ROI: 2년 내 회수

**시나리오 2: 클라우드 SIEM 전환**
- 온프레미스 Splunk → Azure Sentinel
- 비용 절감: 30%
- 확장성 향상

#### 2. 안티패턴 (Anti-patterns)

```
취약한 운영 (금지!)

1. 로그만 수집
   - 분석 없음 → SIEM 의미 없음

2. 과도한 알림
   - 알림 피로 → 중요 알림 무시

3. 규칙 미튜닝
   - False Positive 다수 → 신뢰도 하락

올바른 운영:

1. 상관 규칙 설계
   - 단일 이벤트 → 알림 지양

2. 알림 티어 운영
   - Tier 1: 트리아지
   - Tier 2: 조사
   - Tier 3: 헌팅

3. 지속적 튜닝
   - False Positive 제거
   - 새 위협 패턴 추가
```
