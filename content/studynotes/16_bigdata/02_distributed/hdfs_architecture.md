# HDFS (Hadoop Distributed File System)

## 핵심 인사이트 (3줄 요약)

> 1. **본질**: HDFS는 대용량 파일을 여러 노드에 분산 저장하는 하둡의 핵심 저장소로, 128MB 블록 단위 저장과 3중 복제를 통해 내결함성을 보장한다.
> 2. **가치**: HDFS는 페타바이트 규모의 데이터를 저비용으로 저장할 수 있어, 빅데이터 처리의 근간이 되는 기술이다.
> 3. **융합**: MapReduce, Spark, Hive, HBase와 결합하여 빅데이터 생태계의 저장 계층을 담당한다.

## Ⅰ. 개요 (Context & Background)

### 개념 정의

HDFS(Hadoop Distributed File System)는 Apache Hadoop의 핵심 구성 요소로, 상용 하드웨어로 구성된 클러스터에서 대용량 파일을 분산 저장하기 위해 설계된 파일 시스템이다. Google File System(GFS) 논문(2003)을 기반으로 Doug Cutting이 개발했다.

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    HDFS 아키텍처                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                    NameNode (마스터 노드)                        │   │
│  │  ┌─────────────────────────────────────────────────────────────┐│   │
│  │  │  메타데이터 관리                                             ││   │
│  │  │  - 파일 시스템 트리 (fsimage)                               ││   │
│  │  │  - 편집 로그 (edits log)                                    ││   │
│  │  │  - 블록 매핑 정보                                           ││   │
│  │  │  - DataNode 하트비트 관리                                    ││   │
│  │  └─────────────────────────────────────────────────────────────┘│   │
│  └───────────────────────────┬─────────────────────────────────────┘   │
│                              │                                        │
│         ┌────────────────────┼────────────────────┐                   │
│         │                    │                    │                    │
│         ▼                    ▼                    ▼                    │
│  ┌─────────────┐    ┌─────────────────┐    ┌─────────────┐           │
│  │ DataNode 1  │    │ DataNode 2      │    │ DataNode N  │           │
│  │ ┌─────────┐ │    │ ┌─────────────┐ │    │ ┌─────────┐ │           │
│  │ │ Block 1 │ │    │ │ Block 1     │ │    │ │ Block 2 │ │           │
│  │ │ Block 2 │ │    │ │ Block 3     │ │    │ │ Block 3 │ │           │
│  │ │ Block 3 │ │    │ │ Block 2     │ │    │ │ Block 1 │ │           │
│  │ └─────────┘ │    │ └─────────────┘ │    │ └─────────┘ │           │
│  │ 128MB 블록  │    │ 3중 복제        │    │ 랙 인식    │           │
│  └─────────────┘    └─────────────────┘    └─────────────┘           │
│                                                                         │
│  핵심 특징:                                                             │
│  - 블록 크기: 128MB (기본값, 변경 가능)                                 │
│  - 복제 계수: 3 (기본값, 변경 가능)                                     │
│  - 랙 인식: 같은 랙에 복제본 분산                                       │
│  - 순차 읽기: 대용량 파일에 최적화                                      │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 💡 비유

HDFS는 "분산 창고"에 비유할 수 있다. 하나의 거대한 창고 대신, 여러 작은 창고(DataNode)에 짐(데이터 블록)을 나누어 저장한다. 관리자(NameNode)는 어떤 짐이 어느 창고에 있는지 기록한다. 한 창고에 불이 나도 다른 창고에 예비 짐이 있어 안전하다.

### 등장 배경

1. **2003년**: Google GFS 논문 발표
2. **2006년**: Apache Hadoop 프로젝트 시작
3. **2008년**: Yahoo! 프로덕션 환경 도입
4. **2012년**: Hadoop 2.0 (YARN) 출시
5. **2018년**: Hadoop 3.0 (Erasure Coding) 출시

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### HDFS 핵심 구성 요소

| 구성 요소 | 역할 | 상세 기능 | 고가용성 |
|-----------|------|-----------|----------|
| **NameNode** | 마스터 | 메타데이터 관리, 블록 매핑 | HA NameNode |
| **DataNode** | 슬레이브 | 블록 저장, 하트비트 | 다중 복제 |
| **Secondary NameNode** | 보조 | 체크포인트 생성 | HA에서 불필요 |
| **JournalNode** | 쓰기 로그 | 편집 로그 공유 (HA) | 3개 이상 |
| **ZooKeeper** | 코디네이터 | 리더 선출 (HA) | 홀수 개 |

### HDFS 파일 읽기/쓰기 흐름

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    HDFS 파일 쓰기 흐름                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 클라이언트 -> NameNode: 파일 생성 요청                              │
│  2. NameNode -> 클라이언트: DataNode 목록 반환                          │
│  3. 클라이언트 -> DataNode: 파이프라인 쓰기 (복제본 3개)                │
│  4. DataNode -> NameNode: 블록 수신 확인                                │
│                                                                         │
│  파이프라인 쓰기:                                                       │
│  Client -> DN1 -> DN2 -> DN3 -> ACK                                    │
│                                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                    HDFS 파일 읽기 흐름                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 클라이언트 -> NameNode: 파일 메타데이터 요청                        │
│  2. NameNode -> 클라이언트: 블록 위치 목록 반환                         │
│  3. 클라이언트 -> 가장 가까운 DataNode: 블록 읽기                       │
│  4. DataNode -> 클라이언트: 블록 데이터 전송                            │
│                                                                         │
│  최적화:                                                                │
│  - 랙 인식: 같은 랙의 DataNode 우선                                    │
│  - 로컬리티: 클라이언트과 같은 노드 우선                                │
│  - 캐싱: 자주 읽는 블록은 메모리 캐시                                  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### HDFS 블록 배치 전략

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    3중 복제 블록 배치 전략                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  랙 1                                      │   │
│  │  ┌─────────┐   ┌─────────┐                                      │   │
│  │  │ Node A  │   │ Node B  │                                      │   │
│  │  │ [복제1] │   │         │   클라이언트가 Node A에서 쓰기 요청   │   │
│  │  └─────────┘   └─────────┘                                      │   │
│  └─────────────────────────────┬───────────────────────────────────┘   │
│                                │                                        │
│  ┌─────────────────────────────┴───────────────────────────────────┐   │
│  │  랙 2                                      │   │
│  │  ┌─────────┐   ┌─────────┐                                      │   │
│  │  │ Node C  │   │ Node D  │                                      │   │
│  │  │ [복제2] │   │ [복제3] │   다른 랙에 2개 복제본 분산          │   │
│  │  └─────────┘   └─────────┘                                      │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  배치 규칙:                                                             │
│  1. 복제본 1: 쓰기 요청한 클라이언트와 같은 노드                        │
│  2. 복제본 2: 다른 랙의 노드                                            │
│  3. 복제본 3: 복제본 2와 같은 랙의 다른 노드                            │
│                                                                         │
│  장애 내성: 랙 1 또는 랙 2 전체 장애 시에도 데이터 사용 가능            │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### HDFS vs 기타 분산 파일 시스템

| 구분 | HDFS | GFS | Ceph | Amazon S3 |
|------|------|-----|------|-----------|
| **개발사** | Apache | Google | Red Hat | Amazon |
| **일관성** | 강한 일관성 | 강한 일관성 | 강한 일관성 | 결과적 일관성 |
| **블록 크기** | 128MB | 64MB | 4MB | 객체 |
| **복제 방식** | 3중 복제 | 3중 복제 | 복제/EC | 다중 AZ |
| **POSIX** | 부분 지원 | 아니오 | 예 | 아니오 |
| **오픈소스** | 예 | 아니오 | 예 | 아니오 |

### HDFS 최적화 기법

| 기법 | 설명 | 효과 |
|------|------|------|
| **Erasure Coding** | XOR 기반 패리티 | 스토리지 50% 절감 |
| **Short-Circuit Read** | 로컬 직접 읽기 | 지연 시간 50% 감소 |
| **Centralized Cache** | NameNode 캐시 관리 | 반복 읽기 가속 |
| **Striping** | 블록 분산 저장 | 대역폭 확장 |

│                                                                         │

│  - 랙 수: 10개 (각 10 DataNode)                                        │

│  예상 비용:                                                             │

│  - 네트워크: $200K                                                      │
│  - 소프트웨어: $0 (오픈소스)                                            │

## 👶 어린이를 위한 3줄 비유

**1단계 (무엇인가요?)**: HDFS는 커다란 파일을 여러 컴퓨터에 나누어 저장하는 거예요. 한 컴퓨터에 넣기엔 너무 큰 파일을 작은 조각(블록)으로 나눠서 여러 곳에 저장해요.

**2단계 (어떻게 안전하게 보관하나요?)**: 같은 조각을 3곳에 복사해 놓아요. 한 컴퓨터가 고장 나도 다른 곳에 똑같은 게 있어서 걱정 없어요. 또 다른 방(랙)에도 복사해 놔서 방에 문제가 생겨도 안전해요.

**3단계 (왜 중요한가요?)**: 요즘은 사진, 동영상, 로그 같은 게 너무 많아서 한 컴퓨터에 다 못 넣어요. HDFS를 쓰면 싼 컴퓨터 여러 대를 묶어서 슈퍼컴퓨터처럼 쓸 수 있어요!
