# HDFS (Hadoop Distributed File System)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: HDFS는 대용량 파일을 블록(기본 128MB) 단위로 분할하여 여러 데이터노드에 분산 저장하는 하둡의 분산 파일 시스템으로, 복제(Replication)를 통해 고가용성과 내결함성을 제공합니다.
> 2. **가치**: 저가형 범용 하드웨어로 페타바이트 규모의 데이터를 안정적으로 저장할 수 있어, 데이터 레이크의 핵심 스토리지 기술입니다.
> 3. **융합**: 구글 GFS 논문을 기반으로 개발되었으며, 현재는 클라우드 객체 스토리지(S3, ADLS)와 함께 또는 대체되어 사용됩니다.

---

### Ⅰ. 개요

#### 1. 개념 및 정의
**HDFS(Hadoop Distributed File System)**는 하둡의 분산 파일 시스템으로, 대용량 파일을 블록 단위로 쪼개어 여러 서버(데이터노드)에 분산 저장합니다. 마스터-슬레이브 구조로, 네임노드(메타데이터 관리)와 데이터노드(실제 데이터 저장)로 구성됩니다.

#### 2. 핵심 특성
- **블록 저장**: 128MB(또는 256MB) 단위 블록으로 분할 저장
- **복제**: 기본 3-way 복제로 내결함성 보장
- **랙 인지**: 복제본을 서로 다른 랙에 분산 배치
- **순차 읽기**: 한 번 쓰고 여러 번 읽는 패턴에 최적화

---

### Ⅱ. 아키텍처

```text
<<< HDFS Architecture >>>

+------------------+
|    NameNode      |  (Master)
| - 메타데이터      |
| - 블록 맵핑       |
| - 네임스페이스    |
+--------+---------+
         |
    +----+----+----+----+
    |    |    |    |    |
    v    v    v    v    v
+----++----++----++----++----+
| DN || DN || DN || DN || DN |
+----++----++----++----++----+

[블록 복제 예시]
파일: 500MB
블록 크기: 128MB
블록 수: 4개 (128+128+128+116)
복제 계수: 3
총 저장 공간: 500MB x 3 = 1.5GB
```

---

### Ⅲ. 핵심 원리

**복제와 랙 인지**:
- 복제본은 서로 다른 랙에 배치
- 한 랙 전체 장애 시에도 데이터 보존
- 읽기 시 로컬 랙 우선 선택

**데이터 지역성**:
- 연산을 데이터가 있는 노드로 이동
- 네트워크 전송 최소화

---

### Ⅳ. 실무 적용

**HDFS 명령어**:
```bash
# 디렉토리 생성
hdfs dfs -mkdir /user/data

# 파일 업로드
hdfs dfs -put local_file.txt /user/data/

# 파일 목록
hdfs dfs -ls /user/data/

# 복제 계수 변경
hdfs dfs -setrep 5 /user/data/file.txt
```

---

### Ⅴ. 결론

HDFS는 빅데이터 스토리지의 기반 기술이며, 현재는 클라우드 객체 스토리지와 함께 사용되거나 대체되고 있습니다.
