# 네임노드와 데이터노드 (NameNode/DataNode)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네임노드는 HDFS의 마스터 노드로 파일 시스템 메타데이터(파일명, 블록 위치, 권한)를 관리하고, 데이터노드는 실제 데이터 블록을 저장하는 워커 노드입니다.
> 2. **가치**: 마스터-슬레이브 구조로 분산 저장을 관리하며, 네임노드는 SPOF(Single Point of Failure) 위험이 있어 고가용성(HA) 구성이 필요합니다.
> 3. **융합**: Secondary NameNode, Standby NameNode 등의 HA 구성으로 발전했으며, 클라우드에서는 관리형 서비스로 제공됩니다.

---

### Ⅰ. 개요

#### 1. 네임노드 (NameNode)
- **역할**: 마스터 노드, 메타데이터 관리
- **저장 정보**: 파일 시스템 트리, 블록 맵핑, 접근 권한
- **메모리 기반**: 모든 메타데이터를 RAM에 로드
- **SPOF 위험**: 단일 장애점 → HA 구성 필요

#### 2. 데이터노드 (DataNode)
- **역할**: 슬레이브 노드, 실제 데이터 저장
- **동작**: 블록 저장, 복제, 하트비트 전송
- **다수 구성**: 수십~수천 대로 확장

---

### Ⅱ. 아키텍처

```text
<<< NameNode/DataNode Interaction >>>

+------------------+
|    NameNode      |
| - fsimage        | ←---- 하트비트 ----+
| - edits log      | ←---- 블록 리포트 --+
| - in-memory meta |                    |
+--------+---------+                    |
         |                              |
    +----+----+----+----+               |
    |    |    |    |    |               |
    v    v    v    v    v               |
+----++----++----++----++----+          |
| DN || DN || DN || DN || DN | ---------+
+----++----++----++----++----+

[하트비트 프로세스]
1. DN이 3초마다 NN에게 하트비트 전송
2. 하트비트에 블록 리포트 포함
3. NN이 30초 이상 응답 없으면 DN을 데드로 표시
4. 데드 DN의 블록을 다른 DN으로 재복제
```

---

### Ⅲ. HA 구성

**Active/Standby NameNode**:
```text
+------------------+     +------------------+
| Active NameNode  |     | Standby NameNode |
| - 서비스 처리     |     | - 대기 상태       |
+--------+---------+     +--------+---------+
         |                        |
         +--------+---------------+
                  |
         +--------v---------+
         |   ZooKeeper      |
         | - 리더 선출       |
         | - 장애 감지       |
         +------------------+
```

---

### Ⅳ. 실무 적용

**메타데이터 관리**:
- FsImage: 파일 시스템 스냅샷
- EditLog: 변경 이력
- 체크포인트: Secondary NameNode가 주기적으로 병합

---

### Ⅴ. 결론

네임노드는 HDFS의 핵심 관리자이며, HA 구성을 통해 안정성을 확보해야 합니다.
