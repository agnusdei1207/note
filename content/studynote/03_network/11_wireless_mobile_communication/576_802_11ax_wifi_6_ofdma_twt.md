# 576. 802.11ax (Wi-Fi 6) - OFDMA 도입, 양방향 MU-MIMO, TWT(Target Wake Time)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 802.11ax (Wi-Fi 6)는 단순히 최대 속도를 올리는 헛된 뻥튀기(Capacity) 경쟁을 멈추고, 경기장이나 공항처럼 수천 명이 밀집한 고밀도(High-Density) 환경에서 '체감 속도가 떨어지지 않게' 만드는 **실효 처리율(Efficiency) 개선**에 사활을 건 표준 규격이다.
> 2. **가치**: 기존 무선랜의 태생적 한계였던 1:1 경쟁 방식(CSMA/CA)의 비효율을 타파하기 위해, 이동통신(LTE/5G) 기지국의 핵심 기술인 **OFDMA(직교 주파수 분할 다중 접속)**를 공유기 안으로 통째로 가져와, 주파수 자원 블록(RU)을 잘게 쪼개 여러 명에게 동시에 쏴주는 셀룰러 스케줄링의 기적을 이뤄냈다.
> 3. **융합**: 다운로드(DL)에만 머물렀던 MU-MIMO를 업로드(UL)까지 확장(양방향 MU-MIMO)하고, 스마트홈 IoT 기기들의 배터리 보존을 위해 잠드는 시간을 예약하는 **TWT (Target Wake Time)** 기술을 융합하여 진정한 사물인터넷 시대의 초연결 백본망으로 진화했다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: IEEE 802.11ax는 2019년에 공식 승인된 무선랜 표준으로, **Wi-Fi 6**로 불린다. 11ac(Wi-Fi 5)가 5GHz 전용이었던 것과 달리, 2.4GHz와 5GHz 대역을 모두 완벽하게 지원하며 1024-QAM 변조 방식을 채택해 이론상 최대 9.6Gbps의 속도를 지원하는 HEW (High-Efficiency Wireless) 규격이다.
- **필요성**: 11ac 공유기는 혼자서 속도를 잴 땐 1Gbps가 넘지만, 지하철역이나 코엑스에서 100명이 동시에 접속하면 인터넷이 완전히 멈춰버렸다(Ping 수백 ms 폭발). 원인은 와이파이 특유의 '눈치 보기(CSMA/CA)' 때문이었다. 1명이 말할 때 나머지 99명이 숨죽여 기다려야 했고, 누군가 작은 카톡 하나를 보낼 때도 20MHz짜리 넒은 도로를 혼자 전세 내서 쓰는 엄청난 자원 낭비가 발생했다. 공유기가 경찰관처럼 "너는 1차선, 너는 2차선"하고 통제해 주는 중앙 스케줄링 아키텍처가 절실했다.
- **등장 배경**: ① 공공 와이파이 혼잡도 증가로 인한 사용자 불만(체감 속도 저하) 폭발 → ② 4G LTE가 채택하여 대성공을 거둔 OFDMA 다중 접속 기술의 도입 요구 → ③ BSS Color(공간 재사용), TWT, UL MU-MIMO 등 트래픽 분산과 전력 관리에 특화된 혁신 기술들을 모아 802.11ax 제정.

```text
┌─────────────────────────────────────────────────────────────┐
│             기존 OFDM(11ac) vs OFDMA(11ax)의 트럭 배송 낭비 시각화   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [과거: OFDM 방식 - "빈 트럭 운행의 비효율"]                        │
│   (20MHz라는 거대한 트럭 한 대가 한 번에 한 명에게만 출발함)               │
│   [트럭 1] ──▶ 카톡 한 줄 (트럭 짐칸의 1%만 쓰고 99%는 텅 빈 채 배달!)    │
│   (기다림)                                                    │
│   [트럭 2] ──▶ 웹서핑 그림 하나 (짐칸 5%만 씀, 95% 빈 공간 낭비!)        │
│   => 결과: 사람은 많은데 짐이 작아도 무조건 트럭 전체를 전세 내야 해서 길이 막힘.│
│                                                             │
│   [혁신: OFDMA 방식 - "합승 밴(Carpool) 방식의 극강 효율"]             │
│   (20MHz 트럭 한 대의 짐칸을 수십 개의 작은 조각(RU, Resource Unit)으로 나눔)│
│                                                             │
│   [ 거대 트럭 1대 출발! ]                                         │
│   ├─▶ (RU 1번 조각) ─▶ A 고객의 카톡 한 줄 전달!                   │
│   ├─▶ (RU 2번 조각) ─▶ B 고객의 웹서핑 그림 전달!                   │
│   └─▶ (RU 3번 조각) ─▶ C 고객의 유튜브 영상 조금 전달!                │
│                                                             │
│   => 결과: 트럭 한 대가 출발할 때 한 방에 3~9명의 짐을 빈 공간 없이 꽉꽉 │
│            채워서(합승) 배달함! 눈치 보며 기다리는 대기 줄이 완전히 증발함!  │
└─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** Wi-Fi 6의 심장인 OFDMA(직교 주파수 분할 다중 접속)의 철학을 가장 잘 나타내는 그림이다. 과거 11ac는 주파수 채널(도로)을 쪼갤 줄을 몰랐다. 무조건 도로 전체 폭(20MHz, 40MHz, 80MHz)을 한 사람이 혼자 썼다(OFDM). 반면 11ax 공유기는 20MHz 대역폭을 최소 2MHz 폭을 가진 **RU(Resource Unit)** 단위로 잘게 썰어버린다. 공유기가 폰들에게 "너희들 각자 필요한 짐의 크기가 얼마야?" 물어본 뒤, A에게는 1개 RU, 짐이 많은 B에게는 4개 RU를 할당해서 거대한 트럭 짐칸에 테트리스 하듯 꽉꽉 끼워 맞춰 한 번에 쏘아 보낸다. 이것이 11ax가 혼잡 구역에서도 체감 지연(Ping)을 혁신적으로 낮출 수 있었던 진짜 마법이다.

- 📢 **섹션 요약 비유**: 11ac는 짜장면 한 그릇을 배달시킬 때마다 큰 배달 트럭이 한 대씩 나가야 해서 차가 엄청 밀렸습니다. 11ax는 커다란 트럭의 짐칸을 수십 개의 작은 보관함(RU)으로 쪼갠 뒤, 짜장면, 피자, 치킨을 한꺼번에 꽉 채워서 한 번에 출발하는 똑똑한 쿠팡맨 새벽 배송 시스템입니다. 배달을 기다리는 대기 줄이 0이 됩니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 11ax(Wi-Fi 6)의 4대 물리적/MAC 아키텍처 혁신

Wi-Fi 6는 단순히 속도만 올린 것이 아니라, 무선 통신의 "눈치 보기(경쟁)" 규칙 자체를 근본적으로 해킹했다.

| 혁신 기술 | 기존 Wi-Fi 5 (11ac) 한계 | Wi-Fi 6 (11ax) 해결책 | 극복 효과 및 융합 포인트 |
|:---|:---|:---|:---|
| **OFDMA** (주파수 분할 접속) | 한 번에 한 명만 전체 주파수 독식 (OFDM) | 20MHz 대역폭을 최대 9명의 RU로 쪼개서 **다중 합승 동시 전송** | 작은 패킷(IoT, 카톡, 게임)이 몰릴 때의 지연(Latency) 완벽 해소. LTE와 기술 철학 동일. |
| **양방향 MU-MIMO** | 다운로드(DL) 시에만 여러 명에게 빔 찢어 쏨 | **업로드(UL)** 할 때도 여러 폰이 동시에 공유기를 향해 빔을 쏨! | 사람들이 일제히 인스타에 사진을 올릴 때 멈추던 카페 와이파이 업로드 병목 타파. |
| **BSS Color** (공간 재사용) | 옆집 공유기가 말하면, 우리 집 공유기는 입 다물고 양보함(CSMA/CA) | 옆집 파동(빨간색)과 우리 집 파동(파란색)에 **컬러 꼬리표**를 달아 옆집이 말해도 그냥 내 할 말 쏨! | 밀집 아파트에서 공유기들이 서로 눈치 보느라 속도 0 되는 간섭(OBSS) 셧다운 방어. |
| **TWT** (타겟 웨이크 타임) | 공유기가 아무 때나 폰을 깨워서 배터리 소모 큼 (짧은 DRX) | "너는 1분 뒤에 일어나서 나랑 0.1초만 데이터 주고받자"고 **취침 시간 예약** | 스마트홈 IoT(온도계 등)가 배터리 하나로 수년 버틸 수 있는 극단적 전력 다이어트. |

### UL MU-MIMO와 OFDMA의 기적: Trigger Frame (트리거 프레임)

Wi-Fi 5 시절 업로드 MU-MIMO가 불가능했던 이유는 "스마트폰 여러 대가 동시에 전파를 쏘면, 공유기 안테나에서 도저히 역계산으로 분리할 수가 없었기" 때문이다(폰끼리 박자가 안 맞아서). Wi-Fi 6는 이를 **Trigger Frame(방아쇠 신호)**이라는 무자비한 중앙 통제 프로토콜로 뚫어냈다.

```text
┌───────────────────────────────────────────────────────────────┐
│               Wi-Fi 6 업로드(UL) OFDMA 및 MU-MIMO 동기화 과정        │
├───────────────────────────────────────────────────────────────┤
│                                                               │
│   [상황: 스마트폰 A, B, C가 동시에 사진을 올리고 싶어 함]                 │
│                                                               │
│   1. 공유기: (CSMA/CA 경쟁을 중지시키고 독재자로 돌변)                 │
│      "모두 입 다물어! 지금부터 내가 정해주는 시간에만 쏜다!"              │
│                                                               │
│   2. 공유기 ──(Trigger Frame 발사!)──▶ A, B, C 단말기               │
│      "자! A는 1번 빔으로 쏘고, B는 2번 주파수 조각(RU)으로 쏘고,         │
│       C는 3번 조각으로 쏴! 준비하시고~ 쏘세요!!!"                      │
│                                                               │
│   3. 단말기 A, B, C ──(Trigger 신호 수신 후, 완벽히 동시에 0.0001초도    │
│                        안 틀리고 일제히 업로드 데이터 발사 🚀)           │
│                                                               │
│   4. 공유기: "오케이! 너희들이 100% 동시에 쐈으니까 파동이 깔끔하게 들어왔네!│
│              내 칩셋(DSP)으로 분리(역행렬/필터) 성공! 다 받았음!"          │
└───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 이 Trigger Frame 아키텍처야말로 Wi-Fi가 그동안의 "자유방임주의(Ad-hoc)" 시장통 룰을 버리고, 이동통신(LTE/5G) 기지국의 **"스케줄링(Scheduling)"** 철학을 완벽하게 베껴온 융합의 결정체다. 공유기는 폰들에게 '언제, 어떤 주파수, 어느 정도의 힘(출력)으로 쏠지'를 명시한 방아쇠(Trigger) 편지를 쏜다. 폰들은 이 편지를 받자마자 자신의 내부 시계를 공유기 시계와 나노초 단위로 동기화한 뒤, 일제히 합창하듯 데이터를 쏘아 올린다. 발사 타이밍과 주파수(RU)가 칼같이 분리되어 있으므로, 허공에서 파동들이 엉키지 않고 공유기 안테나에 순수하게 도달하여 업로드 병목이 기적처럼 해소된다.

- 📢 **섹션 요약 비유**: 시장 바닥에서 상인들(폰)이 서로 자기가 먼저 말하겠다고 소리치던 난장판을 끝냈습니다. 공유기가 오케스트라 지휘자가 되어 지휘봉(Trigger Frame)을 딱 올리고 "바이올린(A폰), 첼로(B폰), 피아노(C폰) 지금 이 악보(RU)대로 동시에 연주 시작!"이라고 통제하니, 잡음이 완벽한 화음(업로드 다중화)으로 변하는 기적입니다.

## Ⅲ. 융합 비교 및 다각도 분석

### QAM 밀도의 극한: 256-QAM(11ac) vs 1024-QAM(11ax)의 한계 돌파

Wi-Fi 6는 OFDMA로 딜레이만 줄인 게 아니다. 데이터의 덩치(용량)를 한 번 더 욱여넣기 위해, 11ac의 256-QAM을 **1024-QAM**으로 진화시켰다.

| 변조 방식 (QAM) | 한 번 쏠 때 전송 비트 | 과녁판 점의 개수 | 최고 체감 속도 (이론치) | 방어 및 안정성 제약 |
|:---|:---|:---|:---|:---|
| **64-QAM** (11n) | 6 비트 | 64개 | 일반적 (최대 수백 Mbps) | 거실에서 안방으로 가도 잘 터짐 (안전함) |
| **256-QAM** (11ac)| 8 비트 | 256개 | 빠름 (최대 1Gbps 돌파) | 벽 하나 뚫거나 조금만 멀어져도 에러 나서 강등됨 |
| **1024-QAM** (11ax)| **10 비트** | **1,024개** | **초고속 (약 25% 추가 뻥튀기)**| **미친 예민함.** 공유기 앞 3미터 이내의 무결점 청정 환경에서만 작동! |

과녁판 하나에 점을 1,024개(32 x 32) 찍어놓았다고 상상해 보라. 수전증(노이즈)이 0.1mm만 있어도 바로 옆 칸으로 데이터가 넘어가 에러가 터진다. 즉, 1024-QAM은 도서관이나 거실처럼 "공유기가 눈앞에 떡하니 보이고 간섭이 1도 없는 뻥 뚫린(LOS) 환경"에서만 잠깐 발동되는 치트키다. 거리가 조금만 멀어져 에러 벡터(EVM)가 흔들리면, 공유기는 자비 없이 256-QAM, 64-QAM으로 속도를 낮춰버린다(Fallback). 따라서 11ax의 진짜 위대함은 1024-QAM의 속도 뻥튀기가 아니라 앞서 말한 **OFDMA의 '안 멈추는 부드러움'**에 있다.

```text
┌───────────────────────────────────────────────────────────────┐
│               BSS Color: 이웃집 간섭(OBSS) 눈치 보기 타파의 원리         │
├───────────────────────────────────────────────────────────────┤
│                                                               │
│   [과거: OBSS (Overlapping BSS) 현상 - 이웃집 지옥]                 │
│   우리 집 AP(채널 6) <--(벽)--> 옆집 AP(채널 6)                     │
│   내 폰: "나 유튜브 쏠게!" (허공 스캔)                               │
│   내 폰 귀: "앗? 옆집에서 채널 6번으로 넷플릭스 쏘는 전파가 들리네?"          │
│   내 폰 행동: [CSMA/CA 룰 발동] "누가 쓰고 있으니 나 10초 쉴게 ㅠ"        │
│   => 결과: 내 집인데 옆집 와이파이 쓴다고 내가 인터넷을 못 쓰는 바보 같은 상황! │
│                                                               │
│   [혁신: BSS Color (공간 재사용, Spatial Reuse) 마법]              │
│   우리 집 AP (채널 6, 파란색 꼬리표 부착)                              │
│   옆집 AP (채널 6, 빨간색 꼬리표 부착)                                │
│                                                               │
│   내 폰: "유튜브 쏠게!" (허공 스캔)                                  │
│   내 폰 귀: "옆집에서 6번 전파 들리는데, 꼬리표를 까보니 '빨간색'이네?"       │
│   내 폰 행동: "빨간색은 남의 집 전파니까, 무시하고 나도 파란색 달고 그냥 쏜다!"│
│   => 결과: 옆집이 쏘든 말든, 우리 집도 똑같은 채널로 동시에 데이터를 쏴버림!   │
└───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** Wi-Fi의 기본 철학인 CSMA/CA("듣고 비어있을 때 쏜다")는 아파트나 도심 상가처럼 남의 공유기 전파가 난무하는 곳에서는 끔찍한 올가미다. 내 폰은 전파가 우리 집 건지 옆집 건지 구별할 지능이 없어서 무조건 쉬어버렸다. 11ax는 모든 프레임 헤더에 6비트짜리 **Color(색상 코드, 1~63번)**를 박아 넣었다. 내 폰은 귓가에 들리는 전파가 내 공유기 색깔(파랑)이면 조용히 기다리지만, 옆집 공유기 색깔(빨강)이면 그 전파 세기(Threshold)를 억지로 낮춰 듣는(무시하는) **공간 재사용(Spatial Reuse)** 기능을 켠다. 덕분에 아파트 단지 전체의 와이파이 체감 속도가 극적으로 회복되었다.

- 📢 **섹션 요약 비유**: 독서실에서 남이 떠들면 조용해질 때까지 기다렸다가 내 말을 하는 게 옛날 와이파이였습니다. 11ax(BSS Color)는 귀마개를 끼는 것과 같습니다. "저 멀리 옆자리(빨간색) 사람이 떠드는 소리는 무시하고, 내 앞에 앉은 친구(파란색)랑 동시에 말하자!"며 눈치를 안 보고 통신해 버리는 뻔뻔하지만 효율적인 룰입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 지하철 공용 와이파이 및 핫스팟의 Dense Environment(고밀도) 설계

1. **상황**: 지하철 객차 안에 통신사가 설치한 802.11ac(Wi-Fi 5) 공유기가 있다. 승객 50명이 타고 있는데, 모두가 인스타 릴스나 카카오톡을 한다. 대역폭은 충분한데 인터넷을 누르면 모래시계만 5초 돌다가 안 넘어가는 극심한 병목 현상이 발생했다.
2. **원인 (CSMA/CA의 충돌 지옥)**: 카카오톡 메시지는 크기가 수십 바이트밖에 안 되는 엄청나게 작은 패킷이다. 하지만 50명이 이 작은 패킷 하나를 보내기 위해 "나 보낼게!"라며 1차선 좁은 문(무선 채널)으로 수천 번씩 동시에 돌진했다. 결과적으로 허공에서 패킷들이 꽝꽝 부딪혀 깨지고(Collision), 다시 보내느라 기다리고(Backoff), 결국 실제 데이터 전송보다 줄 서서 눈치 보는 데 시간의 90%를 다 써버렸다.
3. **의사결정 및 조치 (Wi-Fi 6 OFDMA 강제 적용)**:
   - 통신 아키텍트는 객차 공유기를 **802.11ax (Wi-Fi 6)** 모델로 전면 교체하고, **OFDMA 기능**을 활성화한다.
   - 공유기는 50명의 폰에게 "너 카톡 보낼 거 작은 거지? 너네 10명은 2MHz짜리 작은 조각(RU) 하나씩 딱딱 맡아서 한 번에 싹 다 보내!"라고 스케줄링(Trigger Frame)을 해준다.
   - **결과**: 줄 서서 싸우는 충돌(Collision)이 0%로 소멸했다. 10명이 보낼 카톡이 단 한 번의 전파 송신(1개의 프레임)으로 한 방에 처리되며, 모래시계 도는 지연 시간(Ping)이 500ms에서 20ms로 기적처럼 사라졌다. 이것이 Wi-Fi 6가 '기가비트'가 아니라 '체감 속도(Efficiency)'의 제왕인 이유다.

### 도입 체크리스트 및 안티패턴
- **구형 단말기(Legacy Client) 혼재에 따른 OFDMA 무력화 검증**: 11ax 공유기를 달아도, 그 안에 접속한 폰 중 절반이 구형 11ac/11n 폰이라면 기적이 일어나지 않는다. 구형 폰들은 OFDMA(주파수 쪼개기)를 알아듣지 못하고 옛날 방식대로 20MHz 폭 전체를 혼자 점유하려고(CSMA/CA) 덤벼든다. 공유기는 어쩔 수 없이 구형 폰이 말할 때는 OFDMA 스케줄링을 멈추고 기다려줘야 한다. 진정한 11ax의 힘은 망 내의 모든 단말기가 11ax 칩셋을 달고 있는 **'Greenfield 환경'**에 근접할수록 100% 폭발한다. (이것이 기업 환경에서 주기적으로 단말기 교체 주기를 강제하는 기술적 이유다).
- **안티패턴 (불필요한 160MHz 채널 본딩 고집)**: 일반적인 카페나 사무실에서 속도 뽕을 뽑겠다고 160MHz 채널 본딩을 강제로 켜두는 짓. 160MHz는 5GHz 대역의 거의 1/3을 혼자 삼킨다. 옆 부서 사무실 공유기와 필연적으로 주파수가 겹치게 되고, 결국 BSS Color가 아무리 뛰어나도 감당할 수 없는 엄청난 층간 소음(CCI)이 터진다. 고밀도 환경에서는 차선을 넓히는 게(160MHz) 아니라 차선을 20MHz나 40MHz로 좁게 쪼개어 AP 개수를 늘리고 각자 간섭 없이 통신하게 만드는 셀 플래닝이 100배 현명한 아키텍처다.

- 📢 **섹션 요약 비유**: 막히는 톨게이트(지하철 와이파이)에 차들을 그냥 들이밀면 병목(충돌)이 생깁니다. Wi-Fi 6는 카풀(합승) 전용 차선을 만들어, 목적지가 비슷한 10대의 소형차 승객들을 대형 버스(OFDMA) 한 대에 꽉꽉 채워 한 번에 톨게이트를 무정차로 쏴버리는 궁극의 교통 체증 해결책입니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 802.11ac (Wi-Fi 5) | 802.11ax (Wi-Fi 6) | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (밀집 지역 지연율)** | 100명 접속 시 Ping 수백 ms, 패킷 드랍 폭발 | OFDMA 스케줄링으로 합승 배달 | 경기장/공항 고밀도(Dense) 환경에서 **체감 지연(Latency) 75% 감소** |
| **정량 (배터리 보존, TWT)** | 공유기 맘대로 깨워서 배터리 낭비 | 1분/1시간 단위로 자는 시간(TWT) 예약 합의 | 폰 및 IoT 기기의 **대기 전력 소모 최대 3배 절감** |
| **정성 (전파 간섭 극복)** | 옆집 공유기가 말하면 무조건 기다려야 함 | BSS Color로 옆집 전파 강제 무시 | 아파트 등 OBSS 간섭 환경에서의 **실효 데이터 처리율(Throughput) 4배 펌핑** |

### 미래 전망 및 진화 방향
- **Wi-Fi 6E (6GHz 대역의 해방)**: Wi-Fi 6의 기술은 너무 훌륭하지만, 2.4GHz와 5GHz 도로는 이미 구형 11a/b/g/n/ac 차들로 꽉 막혀있어 제 속도를 낼 수 없었다. 그래서 등장한 확장 규격이 **Wi-Fi 6E (577번 문서)**다. 6E는 아예 아무도 밟지 않은 신대륙인 6GHz 대역(1200MHz 광활한 영토)으로 이사가서, 오직 Wi-Fi 6(ax) 칩셋을 단 최신 스포츠카들끼리만 거칠 것 없이 160MHz 풀 액셀을 밟아 진정한 11ax의 잠재력을 폭발시키고 있다.
- **5G NR-U (Unlicensed)와의 정면승부**: Wi-Fi 6는 OFDMA와 트리거 프레임을 도입하며 사실상 '통신사 없는 미니 5G 기지국'이 되어버렸다. 이에 위협을 느낀 3GPP 셀룰러 진영은 5G 전파(NR)를 공짜 와이파이 대역(Unlicensed Band, 5G/6GHz)에서 쏘아버리는 NR-U 표준을 만들었다. 미래의 실내 스마트 팩토리망 패권을 두고, 누구나 꽂으면 되는 Wi-Fi 6/7 진영과 완벽한 통제력의 5G 특화망(이음 5G) 진영 간의 거대한 기술 전쟁이 벌어지고 있다.

### 참고 표준
- **IEEE 802.11ax-2021**: High Efficiency (HE) WLAN. 11ac의 VHT를 넘어 '고효율(HE)'이라는 철학을 이름에 박아넣고, OFDMA와 TWT, BSS Color를 처음 규정한 Wi-Fi 6의 마스터 표준.
- **WFA (Wi-Fi Alliance) Wi-Fi 6 인증**: 제조사들이 OFDMA나 UL MU-MIMO를 대충 빼먹고 팔아먹는 걸 막기 위해, 이 핵심 스펙들을 모두 탑재해야만 'Wi-Fi 6 Certified' 로고를 주도록 강제한 소비자 보호 프로그램.

802.11ax (Wi-Fi 6)는 20년 넘게 무선랜이 버리지 못했던 "무식하게 차선만 넓히고 파워만 올리면 빠르겠지"라는 철학(Capacity)에 사망 선고를 내렸다. 속도는 덤일 뿐, 본질은 100명이 좁은 방에 모여 있어도 단 한 명의 통화도 튕기지 않도록 주파수를 레고 블록(RU)처럼 쪼개어 핀셋으로 입에 넣어주는 "스케줄링의 위대한 정밀함(Efficiency)"에 있다. 이로써 와이파이는 통신사 기지국 부럽지 않은 완벽한 뇌(Brain)를 가지게 되었다.

- 📢 **섹션 요약 비유**: 이전의 Wi-Fi 5가 넓고 시원한 8차선 고속도로를 깔았지만 신호등이 없어 차가 밀릴 땐 엉망진창이었다면, Wi-Fi 6는 그 고속도로에 완벽한 AI 인공지능 신호등과 교통경찰(OFDMA)을 투입해, 100대의 차가 한 치의 오차도 없이 1차선, 2차선을 칼같이 나누어 최고 속도로 빠져나가게 통제하는 궁극의 '스마트 교통 시스템'입니다.

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **OFDMA (직교 주파수 분할 다중 접속)** | Wi-Fi 6의 가장 위대한 알파이자 오메가. 20MHz 도로를 혼자 쓰던 낭비를 버리고, 도로를 최대 9개의 조각(RU)으로 쪼개 9명의 카톡을 한 번에 배달하는 합승의 기적이다. |
| **RU (Resource Unit, 자원 블록)** | OFDMA가 주파수를 쪼개어 만든 최소 단위의 레고 블록. 스마트폰의 데이터가 적으면 작은 레고 조각을 주고, 영화를 받으면 큰 레고 덩어리를 몰아주는 유연한 배분 시스템이다. |
| **BSS Color (공간 재사용)** | 아파트 옆집 와이파이(OBSS)와 채널이 겹쳐서 인터넷이 멈추는 걸 막기 위해, 옆집 파동(빨강)과 우리 집 파동(파랑)에 색칠을 해서 옆집 소리는 귀마개로 씹어버리는 방어술이다. |
| **TWT (Target Wake Time)** | 스마트홈 온도계나 워치가 "나 1시간 뒤에 딱 1초만 깰게!"라고 공유기와 예약 시간을 잡고, 나머지 59분은 기절(Deep Sleep)해버려 배터리를 10년 쓰게 만드는 극강 절전 기술이다. |
| **Trigger Frame (방아쇠 프레임)** | 업로드 시 폰들이 눈치 싸움(CSMA) 하느라 막히는 걸 부수기 위해, 공유기가 "너네 지금 다 같이 이 주파수로 쏴!"라고 0.001초 타이밍을 맞춰주는 마법의 오케스트라 지휘봉이다. |
