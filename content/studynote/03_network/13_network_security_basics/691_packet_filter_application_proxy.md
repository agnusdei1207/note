# 691. 패킷 필터 (Packet Filter 라우터/L3,L4) 및 애플리케이션 프록시 방화벽

> **핵심 인사이트**: 인터넷 초기엔 편지 겉봉투의 주소(IP)와 우표(포트 번호)만 보고 검사하는 '패킷 필터'가 방화벽의 전부였다. 그러나 해커가 정상 주소로 위장한 폭탄을 편지 안에 숨겨 보내기 시작하자, 겉봉투만 보는 경비원으로는 막을 수 없게 되었다. 결국 편지를 뜯어서 내용물(애플리케이션 데이터)까지 끝까지 까보는 깐깐한 경호원, '애플리케이션 프록시'가 등장하게 된다.

## Ⅰ. 1세대 방화벽: 패킷 필터링 (Packet Filtering)
- **작동 계층**: OSI 7계층 중 **네트워크 계층(L3, IP)**과 **전송 계층(L4, TCP/UDP)**에서 동작합니다.
- **검사 원리**: 지나가는 모든 데이터 패킷의 헤더(Header)만 쓱 훑어보고 통과/차단을 결정합니다.
  - **검사 항목**: 출발지 IP, 목적지 IP, 출발지 포트, 목적지 포트, 프로토콜 종류(TCP/UDP/ICMP).
  - **라우터 ACL (Access Control List)**: 우리가 흔히 시스코 라우터나 AWS 보안 그룹(Security Group)에서 설정하는 `allow tcp 192.168.0.1 80` 같은 규칙이 전형적인 패킷 필터링입니다.
- **장점**: 헤더만 보고 빠르게 넘기므로 처리 속도가 미친 듯이 빠르고, 라우터 자체 기능만으로도 구현 가능하여 비용이 쌉니다.
- **치명적 단점**: 
  - **멍청함 (Stateless)**: 이전 통신의 맥락(문맥)을 전혀 기억하지 못합니다. 내가 네이버에 접속 요청을 한 적도 없는데, 뜬금없이 네이버 IP를 달고 응답(ACK) 패킷이 들어와도 "어? 네이버 IP 허용이네?" 하고 그냥 통과시켜 버립니다. (IP 스푸핑에 속수무책)
  - **내용물(Payload) 맹인**: 포트가 80번(정상 웹사이트 포트)이기만 하면, 그 안에 바이러스나 SQL 인젝션 코드가 들어있어도 전혀 보지 못하고 통과시킵니다.

## Ⅱ. 3세대 방화벽: 애플리케이션 프록시 (Application Proxy Firewall) 🌟
패킷 필터링의 무능함을 극복하기 위해 등장한 '대리인(Proxy)' 방식의 방화벽입니다.

- **작동 계층**: OSI 7계층 중 최상단인 **응용 계층(L7, Application)**에서 동작합니다.
- **검사 원리 (Deep Packet Inspection)**:
  - 외부의 클라이언트가 사내 서버와 직접(Direct) TCP 연결을 맺는 것을 원천 차단합니다. 
  - 대신, **방화벽(프록시 서버)이 중간에서 클라이언트와 먼저 TCP 연결을 맺어 데이터를 다 넘겨받습니다.**
  - 방화벽은 이 택배 상자(패킷)를 칼로 다 찢어발기고, 안에 있는 응용 계층 데이터(HTTP URL, 바이러스 서명, 금지된 명령어)를 엑스레이 찍듯 완벽히 뜯어봅니다.
  - 내용물에 문제가 없으면, 방화벽이 새 택배 상자에 내용물을 다시 예쁘게 포장하여 사내 서버로 새롭게 TCP 연결을 맺고 전달해 줍니다.
- **장점**: 보안성이 현존 최고 수준입니다. 외부와 내부가 물리적으로 완전히 단절(직접 통신 불가)되므로 해커가 내부 서버의 IP나 구조를 유추할 수 없습니다. 
- **단점**: 택배 상자를 일일이 다 뜯어보고 다시 포장해야 하므로 **방화벽 CPU 부하가 엄청나고, 네트워크 속도가 매우 느려지는 병목 현상(Bottleneck)**이 발생합니다.

> 📢 **섹션 요약 비유**: 
> - **패킷 필터 (1세대)**: 우편물 분류 알바생입니다. 겉봉투에 '발신자: 친구', '수신자: 나'라고 적혀 있으면 그냥 내 책상에 올려놓습니다. 안에 폭탄 가루(악성코드)가 있어도 모릅니다.
> - **애플리케이션 프록시 (3세대)**: 깐깐한 수석 경호원입니다. 친구가 보낸 편지라도 절대 나에게 바로 주지 않습니다. 경호원이 먼저 편지를 뜯어 화학 성분(Payload)을 다 검사하고 폭탄이 아님을 확인한 뒤에야, 새 봉투에 다시 넣어서 내 책상 위에 안전하게 놓아주는 철통 방어 시스템입니다. 속도는 느리지만 가장 확실합니다.
