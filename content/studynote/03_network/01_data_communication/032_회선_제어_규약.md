> **핵심 인사이트 3줄**
> 1. 회선 제어 규약(Line Control Protocol)은 데이터 링크 계층에서 회선 접근·동기화·오류 제어·흐름 제어를 담당하는 프로토콜 집합이다.
> 2. 문자 지향(BSC)→비트 지향(HDLC·SDLC) 발전으로 전송 효율과 신뢰성이 크게 향상되었다.
> 3. HDLC(High-level Data Link Control)는 ISO 표준으로 현재도 ISDN·위성·WAN 링크에서 슬라이딩 윈도우 기반 ARQ를 구현한다.

---

## Ⅰ. 회선 제어 규약의 개요와 분류

회선 제어 규약(Line Control Protocol)은 **물리 링크 위에서 데이터 전송을 신뢰성 있게 수행**하기 위한 규약이다. 크게 문자 지향(Character-Oriented)과 비트 지향(Bit-Oriented) 방식으로 분류된다.

| 분류          | 대표 프로토콜 | 특징                          |
|-------------|-------------|-------------------------------|
| 문자 지향     | BSC         | 제어 문자(STX/ETX) 기반        |
| 비트 지향     | HDLC, SDLC  | 플래그(01111110) 기반 프레임   |
| 바이트 카운트 | DDCMP       | 헤더에 바이트 수 명시          |

### 회선 제어 5대 기능

1. **회선 접속** — 논리적 연결 설정 (Setup)
2. **동기 제어** — 비트/문자/프레임 동기 맞춤
3. **흐름 제어** — 수신자 버퍼 넘침 방지
4. **오류 제어** — CRC 검출 + ARQ 재전송
5. **회선 해제** — 전송 완료 후 논리 연결 해제

📢 **섹션 요약 비유**: 회선 제어 규약은 도로 교통 법규와 같다. 신호(동기), 차선(흐름 제어), 사고 처리(오류 제어), 진입/진출(접속/해제) 규칙이 모두 포함된다.

## Ⅱ. BSC (Binary Synchronous Communication)

BSC는 IBM이 1960년대 개발한 **문자 지향 반이중 프로토콜**이다.

```
[STX] [데이터] [ETX] [BCC]
 시작   본문   종료  오류검사
```

| 제어 문자 | 의미                    |
|----------|------------------------|
| STX      | 본문 시작 (Start of Text) |
| ETX      | 본문 종료 (End of Text)  |
| ENQ      | 회선 확인 요청            |
| ACK      | 긍정 응답                |
| NAK      | 부정 응답 (재전송 요청)   |
| SYN      | 동기 문자                |

**BSC 한계**: 반이중, 포인트-투-포인트만, 다중점 제한, 에러 제어 취약

📢 **섹션 요약 비유**: BSC는 무전기 통신이다. "오버"를 외쳐야(ENQ/ACK) 상대가 말할 수 있고, 한 번에 한 방향만 통신된다.

## Ⅲ. HDLC (High-level Data Link Control) — 핵심

HDLC는 ISO가 정의한 **비트 지향 전이중 프로토콜**로, X.25·ISDN·PPP의 기반이 된다.

### HDLC 프레임 구조

```
┌────────┬──────────┬────────────┬──────────┬────────┐
│ Flag   │ Address  │  Control   │  Data    │  FCS   │
│01111110│  8 bit   │  8/16 bit  │ 가변 길이 │ 16/32b │
└────────┴──────────┴────────────┴──────────┴────────┘
```

### 프레임 종류

| 프레임  | 기호 | 기능                            |
|--------|------|---------------------------------|
| I-Frame | 000  | 정보 전송 + 피기백 ACK/NAK      |
| S-Frame | 01   | 감시 (RR/RNR/REJ/SREJ)          |
| U-Frame | 11   | 비번호 (설정·해제·초기화)        |

### HDLC 스테이션 모드

- **NRM (Normal Response Mode)**: 마스터-슬레이브, 폴링 방식
- **ARM (Asynchronous Response Mode)**: 슬레이브 자발 전송 가능
- **ABM (Asynchronous Balanced Mode)**: 양방향 대등 전송 → LAP-B, PPP

📢 **섹션 요약 비유**: HDLC는 양방향 고속도로다. BSC가 일방통행 국도였다면, HDLC는 차선별로 다른 종류의 신호(I/S/U 프레임)를 동시에 보낼 수 있는 자동차 전용도로다.

## Ⅳ. 슬라이딩 윈도우와 ARQ 제어

### 슬라이딩 윈도우 (Sliding Window) 프로토콜

```
송신측 윈도우: [ 3 | 4 | 5 | 6 | 7 ]  (미확인 전송 가능 프레임)
수신측:        ACK3 수신 → 윈도우 이동 → [ 4 | 5 | 6 | 7 | 8 ]
```

| ARQ 방식    | 재전송 방식            | 효율          |
|------------|----------------------|---------------|
| Stop-and-Wait | 1프레임씩 ACK 대기  | 낮음 (1/RTT)  |
| Go-Back-N   | 오류 이후 전부 재전송 | 중간           |
| Selective Repeat | 오류 프레임만 재전송 | 높음 (버퍼↑) |

**윈도우 크기**: n비트 순서번호 → Go-Back-N: 2ⁿ-1, SR: 2ⁿ⁻¹

📢 **섹션 요약 비유**: 슬라이딩 윈도우는 택배 기사가 확인 전화 없이 여러 집에 미리 배달하는 것과 같다. 반품(NAK)이 오면 Go-Back-N은 그 이후 전부 다시 배달하고, SR은 그 집 것만 다시 배달한다.

## Ⅴ. PPP와 현대 링크 계층 프로토콜

### PPP (Point-to-Point Protocol)

HDLC 기반에서 발전한 PPP는 인터넷 다이얼업·DSL·LCP/NCP 협상을 지원한다.

```
PPP 연결 설정 과정:
Dead → Establish (LCP) → Authenticate → Network (NCP) → Open
```

| 프로토콜    | 계층  | 특징                         |
|------------|------|------------------------------|
| PPP        | L2   | 다중 프로토콜, 인증(PAP/CHAP) |
| PPPoE      | L2   | 이더넷 위 PPP (DSL)           |
| MLPPP      | L2   | 다중 링크 묶음                |
| L2TP       | L2   | VPN 터널링                    |

📢 **섹션 요약 비유**: PPP는 표준 전화잭처럼 어떤 기기든 연결되도록 만든 규격이다. 기기 종류(NCP)와 연결 설정(LCP)을 협상 후 데이터를 전달한다.

## 📌 관련 개념 맵

```
회선 제어 규약
├── 문자 지향
│   └── BSC (Binary Synchronous Communication)
│       ├── STX/ETX 프레임
│       └── ACK/NAK 응답
├── 비트 지향
│   ├── HDLC (High-level Data Link Control)
│   │   ├── I-Frame / S-Frame / U-Frame
│   │   └── NRM / ARM / ABM 모드
│   └── SDLC (IBM 독점 → HDLC 표준화)
├── 흐름/오류 제어
│   ├── 슬라이딩 윈도우 (Sliding Window)
│   ├── Go-Back-N ARQ
│   └── Selective Repeat ARQ
└── 현대 응용
    ├── PPP / PPPoE
    ├── L2TP
    └── X.25 / ISDN 링크
```

```
핵심 키워드 연결:
BSC → HDLC → PPP → L2TP → IPSec VPN
  ↓       ↓       ↓
문자동기  비트동기  NCP 협상
  ↓       ↓
CRC     슬라이딩 윈도우 ARQ
```

## 👶 어린이를 위한 3줄 비유 설명

1. 회선 제어 규약은 편지 배달 규칙이다 — 봉투 형식, 주소 쓰는 법, 반품 처리 방법이 모두 정해져 있다.
2. HDLC 프레임은 특급 우편 봉투다 — 앞에 보내는 사람 주소, 뒤에 분실 방지 스티커가 붙어있다.
3. 슬라이딩 윈도우는 편의점 배달 앱이다 — 확인 전화 없이 여러 건 미리 배달하고, 반품만 다시 가져간다.
