> **핵심 인사이트**
> 1. 맨체스터 인코딩(Manchester Encoding)은 각 비트 주기의 중간에 전이(Transition)를 강제하는 자기 클록킹(Self-Clocking) 방식으로, DC 성분이 없고 클록 복원이 쉬워 이더넷(10BASE-T)에서 채택했다.
> 2. 차분 맨체스터(Differential Manchester)는 비트 경계에서 전이 여부로 비트를 표현 — 전이 없음=1, 전이 있음=0 (또는 반대) — 절대 전압 레벨이 아닌 상대적 변화로 인코딩하여 극성 반전에 강하다.
> 3. 두 방식 모두 비트당 2배 대역폭을 소비하는 단점이 있어, 고속 이더넷(100BASE-TX 이상)은 4B/5B + MLT-3 또는 8B/10B 등 더 효율적인 인코딩으로 전환했다.

---

## I. 맨체스터 인코딩 원리

```
맨체스터 (Manchester):
  비트 중간에 반드시 전이 발생
  
  비트 1: 낮음 -> 높음 (↑ 상승 전이)
  비트 0: 높음 -> 낮음 (↓ 하강 전이)
  
  (IEEE 802.3 이더넷 표준)
  
시간 축:
  비트 경계   중간   비트 경계
    |         |         |
    +--+  or  +--+
       |          |
    1  |       0  |

파형 예시 (비트열: 1 0 1 1 0):
  1:   ___|‾‾‾|
  0:   ‾‾‾|___|
  1:   ___|‾‾‾|
  1:   ___|‾‾‾|
  0:   ‾‾‾|___|

특징:
  자기 클록킹: 중간 전이로 클록 추출 가능
  DC 성분 없음: 장거리 전송에 유리
```

> 📢 **섹션 요약 비유**: 비트가 심장 박동처럼 중간에 반드시 뛰어야 함 — 1은 위로, 0은 아래로 뛰는 방향이 다를 뿐.

---

## II. 차분 맨체스터 인코딩

```
차분 맨체스터 (Differential Manchester):
  비트 경계에서 전이 여부로 0/1 결정
  비트 중간: 항상 전이 (클록 추출용)
  
  비트 0: 비트 경계에서 전이 있음
  비트 1: 비트 경계에서 전이 없음
  (IEEE 802.5 토큰링 표준)

비교:
  맨체스터: 절대 레벨 변화 (↑=1, ↓=0)
  차분 맨체스터: 상대 변화 (전이 없음=1, 있음=0)

파형 예시 (비트열: 0 1 0 0 1):
  초기 상태: 낮음
  0 (경계전이): ↑___↓  
  1 (경계무전이): 계속↓___↑ (중간전이)
  0 (경계전이): ↓___↑
  ...

장점:
  극성 반전(케이블 연결 실수)에 강함
  중간 전이 항상 있어 클록 추출 확실
```

> 📢 **섹션 요약 비유**: 차분 맨체스터는 방향이 아닌 "방향 전환 여부"로 코드화 — 직진하면 1, 회전하면 0 (출발 방향은 중요하지 않음).

---

## III. 비교 분석

```
인코딩 방식 비교:

                    맨체스터   차분맨체스터  NRZ-L
자기 클록킹:          Yes        Yes         No
DC 성분:             없음        없음        있음
극성 반전 민감도:     높음        낮음        높음
비트당 전이:          1~2회      2회(고정)   0~1회
대역폭 효율:         50%         50%         100%
표준:               10BASE-T    토큰링      RS-232

대역폭 효율:
  맨체스터: 10Mbps 전송 시 20MHz 대역 필요
  -> 고속에서 불리 -> 4B/5B(80%), 8B/10B(80%)
  
10BASE-T (이더넷):
  맨체스터 사용
  10Mbps 데이터 -> 20MHz 신호 필요
  UTP 케이블 최대 주파수: 충분
  
100BASE-TX (패스트 이더넷):
  4B/5B + NRZI + MLT-3
  맨체스터 사용 시 200MHz 필요 -> 불가
```

| 방식         | 표준           | 대역폭 효율 | 특징        |
|------------|--------------|---------|-----------|
| 맨체스터      | 이더넷 10BASE | 50%     | 자기 클록킹   |
| 차분 맨체스터  | 토큰링 802.5  | 50%     | 극성 강인    |
| 4B/5B      | 100BASE-TX   | 80%     | 대역폭 개선  |
| 8B/10B     | 기가비트 이더넷  | 80%     | DC 균형     |

> 📢 **섹션 요약 비유**: 맨체스터는 명확하지만 채널 낭비(50%) — 고속화가 필요해지자 더 효율적인 4B/5B로 교체한 것.

## IV. 클록 복원 원리

```
자기 클록킹 (Self-Clocking) 메커니즘:

문제: 데이터와 클록을 별도 선으로 전송하면
      두 선 간 지연 차이(Skew) 발생
      -> 에러 가능성

맨체스터 해결책:
  클록 정보를 데이터에 내포
  수신측 PLL (Phase-Locked Loop):
    전이 타이밍 감지 -> 클록 주파수 추출
    -> 클록 없이 데이터 복원 가능

클록 복원 단계:
  1. 수신 파형에서 전이 감지
  2. PLL로 클록 주파수/위상 동기화
  3. 비트 중간 샘플링으로 데이터 추출

장거리 전송:
  동기화 손실 없이 장거리 데이터 전송
  10BASE-T: UTP 100m 최대
```

> 📢 **섹션 요약 비유**: 신호 안에 메트로놈(클록)을 숨겨 보내는 것 — 받는 쪽이 신호에서 박자를 스스로 찾아낼 수 있음.

## V. 실무 시나리오 — 이더넷 PHY 계층

```
10BASE-T 이더넷 PHY 동작:

송신:
  MAC에서 비트 스트림 수신
  맨체스터 인코더:
    1 -> 상승 전이 (01 패턴)
    0 -> 하강 전이 (10 패턴)
  10MHz 클록 * 2 = 20MHz 신호 출력
  
수신:
  20MHz 신호 수신
  PLL 클록 복원 (20MHz 추출)
  비트 중간 샘플링 -> 1/0 판별
  MAC에 비트 스트림 전달

실제 PHY IC (예: Realtek RTL8019):
  내부: 맨체스터 인코더/디코더
  출력: TX+/TX- (차동 신호, 극성 반전 허용)
        -> 차분 방식의 장점 활용

문제 해결:
  이더넷 크로스오버(MDI-X) 자동 감지
  자동 극성 교정 -> 케이블 거꾸로 꽂아도 동작
```

> 📢 **섹션 요약 비유**: 이더넷 PHY는 디지털 번역가 — MAC의 0과 1을 케이블이 이해하는 맨체스터 파형으로 변환하고, 받을 때 역변환.

## 📌 관련 개념 맵

```
맨체스터 / 차분 맨체스터
+-- 공통 특징
|   +-- 자기 클록킹
|   +-- DC 성분 없음
+-- 차이점
|   +-- 맨체스터: 절대 레벨 (↑=1, ↓=0)
|   +-- 차분 맨체스터: 경계 전이 여부 (없음=1)
+-- 한계
|   +-- 대역폭 효율 50%
|   +-- 고속(100Mbps+)에서 비실용적
+-- 후계 기술
    +-- 4B/5B + NRZI (100BASE-TX)
    +-- 8B/10B (기가비트 이더넷)
```

## 👶 어린이를 위한 3줄 비유 설명

1. 맨체스터 인코딩은 신호가 항상 비트 중간에 방향을 바꾸어서, 1은 위로 올라가고 0은 아래로 내려가는 방식이에요.
2. 차분 맨체스터는 방향 자체가 아닌 "방향을 바꾸었는가 안 바꾸었는가"로 0과 1을 구별해서 케이블이 거꾸로 꽂혀도 올바르게 동작해요.
3. 예전 이더넷(10Mbps)은 맨체스터를 썼지만, 더 빠른 인터넷이 필요해지면서 대역폭을 절반이나 낭비하는 맨체스터 대신 더 효율적인 방식으로 바뀌었어요!
