> **핵심 인사이트**
> 1. NRZ(Non-Return to Zero)는 신호가 비트 주기 중 0V로 복귀하지 않는 라인 코딩 방식으로, NRZ-L(Level)은 비트 값 자체를 전압 레벨로, NRZ-I(Inverted)는 1 비트마다 전압을 반전시켜 표현한다.
> 2. NRZ-L의 핵심 문제는 긴 연속 0 또는 1이 발생할 때 클락 동기화가 불가능한 기저선 천이(Baseline Wandering)와 DC 성분 발생이며, NRZ-I는 연속 1에서는 해결되지만 연속 0 문제는 여전히 존재한다.
> 3. 이 한계 때문에 실제 물리 계층에서는 맨체스터(Ethernet), 4B/5B+NRZI(FastEthernet), 8B/10B(Gigabit Ethernet), PAM4(400GbE)처럼 클락 내장 코딩으로 발전해 왔다.

---

## I. NRZ 코딩 기본 원리

```
NRZ-L (Non-Return to Zero Level):
  데이터: 1  0  1  1  0  0  1
  신호:   +V -V +V +V -V -V +V
  
  규칙: 1 = 높은 전압(+V), 0 = 낮은 전압(-V)
  
NRZ-I (Non-Return to Zero Inverted):
  데이터: 1  0  1  1  0  0  1
  신호:   V+ V+ V- V+ V+ V+ V-
           (1이 나올 때마다 전압 반전)
  
  규칙: 1 = 이전 신호에서 반전, 0 = 변화 없음
```

| 구분    | 기준          | 1 비트 처리     | 0 비트 처리  |
|-------|-------------|--------------|------------|
| NRZ-L | 전압 레벨     | 높은 전압 유지   | 낮은 전압 유지|
| NRZ-I | 전압 변화     | 전압 반전       | 전압 유지    |

> 📢 **섹션 요약 비유**: NRZ-L은 신호등처럼 빨강=0, 초록=1 고정 — NRZ-I는 1이 올 때마다 색깔을 바꾸는 방식.

---

## II. NRZ의 장단점

```
NRZ-L/NRZ-I 공통 장점:
  + 단순한 회로
  + 높은 대역폭 효율 (RZ 대비 2배)
  + 구현 용이

NRZ-L 문제:
  연속 0 or 1이 길면:
  1. 기저선 천이 (Baseline Wandering)
     -> AC 결합 회로에서 기준 레벨 이동
  2. DC 성분 발생 (평균 전압 ≠ 0)
     -> 변압기/AC 결합 라인에 통과 불가
  3. 클락 동기화 불가
     -> 수신측이 언제 새 비트인지 알 수 없음

NRZ-I 개선:
  연속 1 문제 해결 (반전이 클락 역할)
  But 연속 0 문제 여전히 존재
```

> 📢 **섹션 요약 비유**: NRZ-L은 "빨강만 30초 계속" — 운전자가 신호가 고장인지 멈춘 건지 모른다. NRZ-I는 1마다 바뀌니 연속 1은 해결되지만 연속 0은 동일 문제.

---

## III. 클락 동기화 문제

```
문제 상황:
  NRZ-L 송신: 1 1 1 1 1 1 1 1 (8개의 1)
  신호: +V +V +V +V +V +V +V +V (변화 없음)
  
  수신측: "한 개의 긴 +V인가? 여러 개의 1인가?"
  -> 비트 경계(Bit Boundary)를 알 수 없음
  -> 클락 복원 불가

해결책들:
  1. 별도 클락 선 사용 (비효율)
  2. 자기 클락킹 코딩 사용:
     맨체스터: 각 비트 중앙에서 반드시 전환
     차분 맨체스터: 0이면 비트 시작에 전환
  3. 4B/5B: 4비트를 5비트로 변환 (연속 0 제거)
```

> 📢 **섹션 요약 비유**: 수신기가 심장박동기처럼 규칙적인 신호를 기대하는데, NRZ에서 긴 1이 오면 "아직 살아있나?" 확인 불가 — 자기 클락킹 코딩이 해결책.

## IV. 발전된 라인 코딩 비교

```
NRZ-L: 가장 단순, 긴 런(run)에 취약
NRZ-I: 연속 1 처리, USB/HDMI에 사용
맨체스터: 각 비트 중앙 전환, 10BASE-T Ethernet
차분 맨체스터: 0에서 시작 전환, Token Ring
4B/5B: 연속 0 5개 이상 방지, FastEthernet
8B/10B: 신호 균형, DC 성분 제거, Gigabit Ethernet
128B/130B: 8B/10B 오버헤드 감소, 10GbE+
PAM4: 4레벨 펄스, 400GbE 이상
```

| 코딩          | 최대 연속 동일 신호 | 사용처              |
|-------------|--------------|-------------------|
| NRZ-L       | 제한 없음      | 단순 RS-232        |
| NRZ-I       | 연속 0 문제   | USB 2.0           |
| 맨체스터     | 최대 1비트     | 10BASE-T          |
| 4B/5B+NRZI  | 최대 3비트     | 100BASE-TX        |
| 8B/10B      | 최대 5비트     | Gigabit Ethernet  |

> 📢 **섹션 요약 비유**: 라인 코딩의 역사는 단순함(NRZ)에서 신뢰성(맨체스터)으로, 다시 효율(4B/5B)로, 또 고속(PAM4)으로 진화하는 과정이다.

## V. 실무 시나리오 — USB 2.0 NRZI 사용 이유

```
USB 2.0 데이터 인코딩:
  NRZI + 비트 스터핑 (Bit Stuffing)

NRZI (NRZ-I 변형):
  1 = 신호 변화 없음, 0 = 신호 반전
  (NRZ-I의 0/1 정의 반대)

연속 1 문제 해결: 비트 스터핑
  6개의 연속 1이 오면 자동으로 0 삽입
  수신측에서 6개 연속 1 이후 0은 자동 제거
  -> 최대 연속 1 = 6개로 제한

결과:
  DC 성분 최소화
  클락 동기화 보장
  별도 클락 선 불필요
```

> 📢 **섹션 요약 비유**: USB가 비트를 보내다가 1이 6개 연속되면 강제로 0을 끼워 넣는 것 — 너무 길게 같은 신호가 지속되는 것을 막는 안전장치.

## 👶 어린이를 위한 3줄 비유 설명

1. NRZ-L은 "빨강=1, 파랑=0"처럼 색깔로 데이터를 표현하는 방법이에요.
2. 같은 색이 너무 오래 지속되면 수신자가 "색이 바뀌지 않은 건지, 아직 전송 중인 건지" 알 수 없는 문제가 생겨요.
3. 그래서 현대 인터넷 케이블은 더 영리한 방법(맨체스터, 4B/5B)을 써서 항상 신호 변화가 있도록 해요!
