> **핵심 인사이트**
> 1. 라인 코딩(Line Coding)은 디지털 비트를 물리적 신호(전압 패턴)로 변환하는 방식으로, DC 성분 제거·동기화·오류 검출 기능을 신호 자체에 내포시켜야 한다.
> 2. 블록 코딩(Block Coding)은 mB/nB 방식으로 원래 m비트를 n비트(n>m)로 확장해 DC 균형 유지와 타이밍 복구를 위한 여유 비트(Redundant Bit)를 추가한다.
> 3. 현대 이더넷(10GBASE-R)은 8B/10B 또는 64B/66B 블록 코딩 + NRZ-I 라인 코딩을 조합해 효율·신뢰성을 동시에 달성한다.

---

## Ⅰ. 라인 코딩 종류와 특성

```
코딩 방식     신호 파형 예시 (0과 1이 교대)
────────────────────────────────────────
NRZ-L:      ─┐  ┌─ ─┐  ┌─    (Low=0, High=1)
             └──┘   └──┘

NRZ-I:      ─┐  ──┐  ──┐     (변화=1, 유지=0)
             └──   └──   └──

Manchester: ─┐ ┌┐ ┌┐ ┌─      (하강엣지=1, 상승엣지=0)
             └─┘ └─┘ └─

4B/5B+NRZ-I: 4비트를 5비트로 변환 후 NRZ-I
```

| 방식       | DC 성분 | 동기화 | 효율      | 사용처        |
|------------|---------|--------|-----------|---------------|
| NRZ-L      | 있음    | 불량   | 100%      | 단거리        |
| NRZ-I      | 있음    | 보통   | 100%      | USB 2.0       |
| Manchester | 없음    | 우수   | 50%       | 이더넷 10BASE |
| AMI        | 없음    | 보통   | 100%      | T1 라인       |
| MLT-3      | 적음    | 보통   | 100%      | Fast Ethernet |

> 📢 **섹션 요약 비유**: 모스 부호처럼 비트를 전압 패턴으로 번역 — 받는 쪽이 타이밍과 값을 동시에 읽을 수 있어야 한다.

---

## Ⅱ. 블록 코딩 — mB/nB 원리

```
원본 데이터 (m비트)
      │ 변환 테이블 참조
      ▼
부호화된 코드워드 (n비트, n > m)
      │ 라인 코딩 적용
      ▼
물리 신호 전송
```

### 4B/5B 코딩 (Fast Ethernet)

| 4B 입력 | 5B 출력 | 의미          |
|---------|---------|---------------|
| 0000    | 11110   | 유효 코드     |
| 0001    | 01001   | 유효 코드     |
| ...     | ...     | (32개 중 16개 사용) |
| 11111   | —       | (연속 1 금지) |

- 5비트 중 연속 0이 3개 이상인 조합을 제외 → 동기화 보장
- 전송 오버헤드: 25% (m=4→n=5)

### 8B/10B 코딩 (Gigabit Ethernet, Fibre Channel)

- 8비트 → 10비트 (오버헤드 25%)
- **RD (Running Disparity)**: +RD/-RD 상태 교대로 DC 균형 유지
- 최대 연속 동일 비트: 5개 이하 보장

### 64B/66B 코딩 (10G+ Ethernet)

- 오버헤드 3.125% (8B/10B 대비 대폭 감소)
- 2비트 헤더(10 또는 01)로 프레임 경계 표시

> 📢 **섹션 요약 비유**: 택배 박스에 여분 패딩재를 채우는 것 — 물건은 안전하게 전달되고, 흔들리지 않는다(DC 균형 = 박스 안 빈 공간 최소화).

## Ⅲ. DC 성분과 동기화 문제

```
연속 0만 전송 시 문제:
물리 신호 = 계속 낮은 전압
    → 수신측 클럭 회로가 엣지 감지 불가 → 동기 손실
    → 직류 결합(DC coupling) 회로 포화

해결책:
① 라인 코딩으로 전압 변화 강제 (Manchester)
② 블록 코딩으로 연속 동일비트 패턴 제거 (8B/10B)
③ 스크램블링(Scrambling): XOR 기반 의사 랜덤 패턴
```

> 📢 **섹션 요약 비유**: 말을 안 해도 상대가 잠들지 않도록 — 신호가 없는 구간에도 "나 여기 있어"라는 엣지를 만들어줘야 한다.

---

## Ⅳ. 스크램블링 vs 블록 코딩

| 비교 항목   | 블록 코딩 (8B/10B) | 스크램블링 (PRBS) |
|------------|-------------------|-------------------|
| 방식        | 변환 테이블        | XOR 패턴          |
| DC 균형    | 보장               | 확률적            |
| 오버헤드   | 25%               | 0%                |
| 복잡도     | 높음               | 낮음              |
| 사용처     | Fibre Channel, PCIe | 100G+ Ethernet   |

> 📢 **섹션 요약 비유**: 블록 코딩은 확실한 규칙, 스크램블링은 통계적 확률 — 속도가 빠를수록 스크램블링이 유리하다.

---

## Ⅴ. 실무 적용 — 이더넷 세대별 코딩

| 세대             | 라인 코딩         | 블록 코딩  | 효율     |
|-----------------|-------------------|-----------|---------|
| 10BASE-T        | Manchester        | —         | 50%     |
| 100BASE-TX      | MLT-3             | 4B/5B     | 80%     |
| 1000BASE-T      | PAM-5             | 8B/10B    | 80%     |
| 10GBASE-R       | NRZ               | 64B/66B   | 96.875% |
| 400GBASE        | PAM-4             | RS-FEC    | ~97%    |

> 📢 **섹션 요약 비유**: 도로가 넓어질수록 포장 방식도 효율적으로 진화 — 초기엔 안전 위주, 이제는 속도와 안전 동시 추구.

## 📌 관련 개념 맵

```
디지털 부호화
├── 라인 코딩 (물리 신호 변환)
│   ├── NRZ-L / NRZ-I
│   ├── Manchester (이더넷 10BASE)
│   ├── AMI / B8ZS / HDB3 (T1/E1 라인)
│   └── MLT-3 / PAM-4 (고속 이더넷)
├── 블록 코딩 (비트 확장)
│   ├── 4B/5B → Fast Ethernet
│   ├── 8B/10B → GbE, Fibre Channel, PCIe Gen1/2
│   └── 64B/66B → 10G+ Ethernet
├── 스크램블링 (PRBS)
│   └── 100G+ Ethernet (코딩 없이 DC 균형)
└── 공통 목표
    ├── DC 성분 제거
    ├── 타이밍 동기화
    └── 오류 검출 용이성
```

## 📈 관련 키워드 및 발전 흐름도

```
[초기 디지털 통신]
NRZ-L: 단순 전압 변환, DC 문제 심각
      │
      ▼
[Manchester 코딩]
엣지 = 클럭 정보 내포, 오버헤드 50%
      │
      ▼
[4B/5B + NRZ-I]
Fast Ethernet 100Mbps, 오버헤드 25% 절충
      │
      ▼
[8B/10B]
GbE / Fibre Channel / PCIe, RD 균형, 25% 오버헤드
      │
      ▼
[64B/66B 스크램블]
10G+ Ethernet, 오버헤드 3%, 스크램블 병용
      │
      ▼
[PAM-4 + RS-FEC]
400G/800G 이더넷, 물리층 고밀도 변조 + FEC 조합
```

## 👶 어린이를 위한 3줄 비유 설명

1. 라인 코딩은 컴퓨터의 0과 1을 전선의 전압 변화로 바꾸는 '번역'이에요.
2. 같은 숫자가 너무 많으면 수신기가 혼란스러워서, 블록 코딩으로 규칙을 추가해요.
3. 더 빠른 인터넷일수록 더 영리한 번역 방법을 써서 실수 없이 빠르게 전달해요!
