> **핵심 인사이트**
> 1. RZ(Return to Zero)는 각 비트 주기의 절반에서 신호가 반드시 0V로 복귀하는 라인 코딩으로, 신호 전환 자체가 클락 정보를 내포해 별도 클락 선 없이 동기화가 가능한 자기 클락킹(Self-Clocking) 코딩이다.
> 2. RZ의 핵심 단점은 대역폭 비효율성 — 신호가 비트 주기의 50%만 실제 데이터를 전달하고 나머지 50%는 0으로 복귀하므로, 동일 데이터 전송에 NRZ 대비 2배의 대역폭이 필요하다.
> 3. 현대 디지털 통신에서 RZ는 직접 사용되지 않지만, 광섬유 통신의 RZ-OOK(On-Off Keying)와 OTDM에서 초단 펄스 구현에 활용되며, 맨체스터 코딩의 개념적 전신이다.

---

## I. RZ 코딩 원리

```
RZ (Return to Zero):
  각 비트 주기 절반: 데이터 전압
  나머지 절반:      0V로 복귀

신호 파형:
  데이터: 1  0  1  1  0  1
  
  RZ 단극성 (Unipolar RZ):
  |¯| |¯|¯| |¯|
  0               (1=+V 전반, 0=0V 내내, 0V로 복귀)
  
  RZ 양극성 (Bipolar RZ / AMI 변형):
  |¯|  |-|¯|  |-|
       (1=교대로 +V/-V, 0=0V)
```

| 구분      | 1 표현          | 0 표현    | 동기화     |
|---------|---------------|--------|----------|
| NRZ-L   | +V 유지        | -V 유지  | 불가      |
| RZ 단극성 | +V → 0V       | 0V 유지  | 가능      |
| RZ 양극성 | ±V → 0V 교대   | 0V 유지  | 가능      |

> 📢 **섹션 요약 비유**: NRZ가 도로 신호등(초록/빨강 계속 유지)이라면, RZ는 점멸 신호등 — 항상 중간에 한 번씩 꺼졌다가 켜져서 리듬을 알 수 있다.

---

## II. RZ vs NRZ 대역폭 비교

```
동일 데이터(1Mbps)를 전송할 때:

NRZ:
  비트 주기 = 1/1Mbps = 1μs
  신호 전환 최대 = 1MHz (1Mbps에서)
  대역폭 요구 ≈ 0.5MHz

RZ:
  비트 주기 = 1μs이지만
  신호 전환 = 비트 당 2번 (1MHz → 0V, 다음 비트)
  대역폭 요구 ≈ 1MHz  (NRZ의 2배!)

결론:
  RZ = NRZ 대비 2배 대역폭 소비
  -> 대역폭이 비싼 유선 통신에서 비효율적
  -> 광섬유처럼 대역폭이 풍부한 곳에서 활용
```

> 📢 **섹션 요약 비유**: RZ로 같은 데이터를 보내려면 도로가 2배 넓어야 한다 — 자동 리듬은 좋지만 비용이 두 배.

---

## III. 변형 RZ 코딩

```
1. 단극성 RZ (Unipolar RZ):
   1 = +V (절반) + 0V (절반)
   0 = 0V 내내
   문제: DC 성분 존재 (1 > 0이면 양전하 편향)

2. 양극성 RZ (Bipolar RZ = AMI + RZ):
   1 = 교대로 +V/-V (절반) + 0V (절반)
   0 = 0V
   장점: DC 균형 (양전하와 음전하 균등)
   = AMI (Alternate Mark Inversion)의 RZ 버전

3. NRZ-S (Space)와의 관계:
   RZ의 개념이 맨체스터 코딩으로 발전
   맨체스터: 비트 중앙에서 반드시 전환
   (1=하강 엣지, 0=상승 엣지)
```

> 📢 **섹션 요약 비유**: 양극성 RZ는 양수와 음수를 번갈아 사용해서 전체 평균을 0으로 유지 — 직류 전압이 축적되지 않는다.

---

## IV. 광통신에서의 RZ 활용

```
광섬유 RZ-OOK (On-Off Keying):
  빛이 있음 = 1, 빛 없음 = 0
  RZ: 각 1비트마다 빛을 짧은 펄스로 쏨
  
  장점:
    초단 펄스(피코초 수준)로 초고속 전송 가능
    펄스 간 공간으로 OTDM(광 시분할 다중화) 구현
    
  OTDM (Optical Time Division Multiplexing):
    여러 채널의 초단 펄스를 시간축으로 인터리빙
    단일 광섬유에서 테라비트 전송 가능

NRZ vs RZ in Fiber Optics:
  장거리 (>100km): NRZ 선호 (낮은 스펙트럼 폭)
  초고속 OTDM: RZ 필수 (펄스 구분 필요)
```

> 📢 **섹션 요약 비유**: 광섬유에서 RZ는 아주 짧은 섬광을 쏘는 것 — 섬광 사이 공간에 다른 채널의 섬광을 끼워 넣으면 여러 채널을 동시에 쓸 수 있다.

## V. 실무 시나리오 — 라인 코딩 선택 기준

```
응용별 최적 라인 코딩:

이더넷 (10BASE-T):
  맨체스터 코딩 (Manchester)
  이유: RZ 기반 자기 클락킹, 단순 구현

FastEthernet (100BASE-TX):
  4B/5B + NRZI
  이유: NRZ 효율 + 동기화 보장

USB 2.0:
  NRZI + 비트 스터핑
  이유: 클락 없는 연결, 저비용

광섬유 (장거리 WDM):
  NRZ (또는 Duobinary)
  이유: 좁은 스펙트럼, 색분산 최소화

광섬유 (OTDM/초고속):
  RZ
  이유: 펄스 인터리빙 가능
```

> 📢 **섹션 요약 비유**: 라인 코딩 선택은 포장 방식 선택 — 속도, 거리, 비용, 동기화 요건에 따라 최적 포장이 다르다.

## 📌 관련 개념 맵

```
RZ (Return to Zero)
+-- 특성
|   +-- 자기 클락킹 (Self-Clocking)
|   +-- 대역폭 2배 소비
+-- 변형
|   +-- 단극성 RZ (DC 문제)
|   +-- 양극성 RZ / AMI
+-- 발전
|   +-- 맨체스터 코딩 (개념적 후속)
|   +-- 광섬유 RZ-OOK, OTDM
+-- 관련 코딩
    +-- NRZ-L, NRZ-I
    +-- 맨체스터, 차분 맨체스터
    +-- AMI (Alternate Mark Inversion)
```

## 👶 어린이를 위한 3줄 비유 설명

1. RZ 코딩은 신호가 항상 중간에 0으로 잠깐 돌아오는 방식이에요 — 이 덕분에 수신자가 박자를 쉽게 맞출 수 있어요.
2. 하지만 같은 정보를 보내는 데 NRZ보다 도로(대역폭)가 2배 필요해서 보통 유선 통신에서는 잘 안 써요.
3. 광섬유 통신에서는 아주 짧은 빛 펄스를 빠르게 쏠 때 RZ 방식을 써서 수십 테라비트를 전송할 수 있어요!
