# 516. GNN (그래프 신경망) 기반 추천 시스템

> ⚠️ 이 문서는 "나와 비슷한 나이의 사람들이 산 물건"을 추천해 주던 1차원적인 통계(협업 필터링)를 넘어, **그래프 DB(Neo4j)에 얽혀있는 복잡한 인간관계와 구매 내역의 연결 고리 자체를 인공지능이 통째로 학습하여 소름 돋게 정확한 추천을 해주는 'GNN 아키텍처'**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 점(Node)과 선(Edge)으로 이루어진 그래프 데이터 자체를 딥러닝 인공지능망에 집어넣어 학습시키는 차세대 머신러닝 기술이다.
> 2. **가치**: 기존 추천 시스템은 '단절된 표(Table)'로만 학습했지만, GNN은 '내 친구가 좋아요를 누른 글에 댓글을 단 사람이 산 신발'처럼 **복잡하게 얽힌 다차원적 관계(Topology)의 패턴을 스스로 깨우친다.**
> 3. **기술 체계**: 주로 **그래프 DB(468번 문서)**를 원천 데이터로 사용하며, 넷플릭스 영화 추천, 우버이츠 음식 추천, 아마존 상품 추천 등 글로벌 IT 기업 추천 시스템의 표준으로 자리 잡았다.

---

### Ⅰ. 개요: 표(Table)로 세상을 이해할 수 없다 (Context & Necessity)
과거 쇼핑몰의 추천 알고리즘은 **협업 필터링(Collaborative Filtering)**이었다.
- "A 고객과 B 고객이 둘 다 콜라를 샀네? 그럼 B가 산 사이다를 A한테 추천해 주자!"
- 한계: 이 방법은 사용자와 상품의 **'단순 구매 기록'**만 본다. A 고객이 콜라를 사기 전에 어떤 유튜브 영상을 봤는지, 어떤 친구와 카톡을 했는지 같은 '복잡한 맥락(관계)'은 전혀 고려하지 못한다.

인간의 삶은 엑셀 표(RDBMS)처럼 단절되어 있지 않고, 거미줄처럼 얽힌 **그래프(Graph)**다.
천재들은 생각했다. **"이 거미줄(관계망) 전체를 통째로 인공지능한테 외우게(학습시키게) 할 순 없을까?"**
이 미친 발상을 수학적으로 구현해 낸 것이 **GNN (Graph Neural Network, 그래프 신경망)**이다.

> 📢 **섹션 요약 비유**: 기존 추천이 이력서 한 장만 보고 "너 농구 동아리 했네? 체대 가라!"라고 조언하는 것이라면, GNN 추천은 **'나의 SNS 친구 관계도, 카톡 대화망, 내가 다녀간 동선'**이라는 거대한 관계 지도를 전부 분석해서 "너랑 친하게 지내는 애들이 다 미술을 좋아하네? 넌 미대에 어울려!"라고 숨겨진 취향을 소름 돋게 찾아내는 것입니다.

---

### Ⅱ. GNN의 핵심 작동 원리: 이웃의 목소리 듣기 ★

GNN이 어떻게 거미줄(그래프)을 학습하는지 직관적으로 이해해 보자. 이를 **메시지 패싱(Message Passing)**이라고 부른다.

#### 1. 내 정보와 이웃의 정보를 합치기 (Aggregation)
- 우주 공간에 '나'라는 점(Node)이 있다. 
- 1단계 학습: 나와 선(Edge)으로 직접 연결된 1촌 친구들의 정보(프로필, 구매 내역)를 가져와서 내 정보와 섞는다. (내 취향 = 내 취향 + 친구들 취향의 평균)
- 2단계 학습: 그 1촌 친구들도 자신의 1촌(나에게는 2촌)들의 정보를 섞어서 나에게 전달해 준다.
- 이 짓을 몇 번 반복하면, **'나'라는 점(Node) 하나에 내 친구들, 친구의 친구들까지의 거대한 네트워크 분위기(문맥)가 전부 스며들게 된다.**

#### 2. 임베딩 (Embedding) 생성
- 이렇게 섞인 나의 최종 정보를 1,024개의 숫자 배열(**벡터 - 485번 문서**)로 꾹 눌러 담아 만든다.
- 1,000만 명의 유저와 100만 개의 상품이 모두 이 복잡한 관계망이 반영된 '숫자 좌표(임베딩)'를 갖게 된다.

#### 3. 유사도 검색 (추천하기)
- 내 좌표와 가장 가까운 거리에 있는 상품 좌표를 찾아서(코사인 유사도 - 486번 문서) 화면에 뿌려준다. 

---

### Ⅲ. 실무 아키텍처: Graph DB와의 융합

GNN을 돌리려면 애초에 원천 데이터가 예쁜 '그래프' 모양이어야 한다. 그래서 **그래프 DB(Neo4j 등 - 468번 문서)**와 GNN은 환상의 짝꿍이다.

1. **데이터 저장 (Neo4j)**: 모든 회원 가입, 친구 맺기, 상품 결제 로그를 Node와 Edge 형태로 그래프 DB에 실시간 저장한다.
2. **그래프 추출**: 매일 밤, 이 거대한 그래프 DB의 데이터(수십억 개의 점과 선)를 통째로 퍼 올린다.
3. **GNN 학습 (DGL, PyG)**: 퍼 올린 그래프를 딥러닝 프레임워크(Deep Graph Library, PyTorch Geometric)에 넣고 며칠 동안 빙글빙글 돌려서(Message Passing) 모든 유저의 최신 벡터(좌표)를 구워낸다.
4. **벡터 DB 서빙 (Milvus, Pinecone)**: 다 구워진 벡터들을 벡터 데이터베이스에 넣고 실시간 추천 API로 쓴다.

```text
┌──────────────────────────────────────────────────────────────┐
│           GNN(그래프 신경망) 기반 추천 시스템의 파이프라인 시각화         │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ [ 🕸️ 1단계: 그래프 DB (Neo4j) ]                                 │
│  (나) ─[팔로우]─▶ (친구) ─[구입]─▶ (나이키 신발)                      │
│   └────[클릭]───────────────▶ (아디다스 신발)                     │
│                                                              │
│ [ 🧠 2단계: GNN 딥러닝 학습 (Message Passing) ]                   │
│  "나의 좌표에 친구의 취향(나이키)과 나의 행동(아디다스)을 섞어서 버무려라!"     │
│   ──▶ 내 최종 벡터 좌표: [0.8, 0.2, 0.9...] 생성 완료!                 │
│                                                              │
│ [ 🎯 3단계: 벡터 DB 추천 (Milvus) ]                              │
│  내 좌표 [0.8, 0.2, 0.9...] 와 가장 가까운 상품을 찾아라!               │
│   ──▶ 추천 결과: "뉴발란스 한정판 운동화" 👟                        │
└──────────────────────────────────────────────────────────────┘
```

### Ⅳ. 결론

"데이터를 넘어서, 맥락(Context)을 이해하는 지능의 탄생."
GNN은 기존의 RDBMS나 데이터 웨어하우스가 가진 2차원적인 표(Table)의 한계를 완벽하게 찢어버린 혁명이다. "유유상종(비슷한 사람끼리 모인다)"이라는 인간 사회의 진리를 수학 알고리즘으로 가장 완벽하게 구현한 것이 바로 GNN의 메시지 패싱(Message Passing)이기 때문이다. 현재 글로벌 빅테크 기업들의 인프라는 단순히 데이터를 '저장'하는 그래프 DB의 시대를 지나, 그 저장된 거미줄을 통째로 뇌(AI)에 집어넣어 고객의 무의식적인 욕망마저 예측해 내는 '그래프 AI' 시대로 진입했다.

### 📌 관련 개념 맵
* **원천 데이터베이스**: Graph DB (그래프 DB, Neo4j - 468번 문서)
* **결과물 데이터베이스**: Vector DB (벡터 DB - 485번 문서)
* **딥러닝 프레임워크**: DGL (Deep Graph Library), PyTorch Geometric (PyG)
* **핵심 알고리즘**: GraphSAGE, GCN (Graph Convolutional Network)
