## 핵심 인사이트
> 1. **본질**: ITIL v3는 5단계 수명주기(SS→SD→ST→SO→CSI)로 프로세스를 구조화하고, ITIL 4는 SVS(Service Value System)로 가치 흐름 중심의 유연한 거버넌스를 추구한다.
> 2. **가치**: v4의 SVC(Service Value Chain)는 애자일·DevOps를 포용해 변화 속도를 따라가고, 4가지 차원 모델로 전체론적 서비스를 관리한다.
> 3. **판단 포인트**: 기존 v3 기반 조직이 v4로 전환 시 프로세스→실천(Practice)으로 용어가 바뀌며, 37개 실천이 v3 26개 프로세스를 흡수·확장한다.

## Ⅰ. 개요 및 필요성

ITIL(IT Infrastructure Library)은 영국 AXELOS가 관리하는 IT 서비스 관리(ITSM) 국제 표준 프레임워크다. v3(2007/2011)는 5권 서적 기반으로 서비스 전략(SS), 서비스 설계(SD), 서비스 전환(ST), 서비스 운영(SO), 지속적 서비스 개선(CSI)이라는 수명주기를 따른다. 각 단계마다 프로세스가 정의되어 있어 직관적이지만 단계 간 경계가 엄격해 애자일 환경에 적용하기 어렵다.

ITIL 4(2019)는 SVS(Service Value System) 중심으로 재설계되었다. SVS는 기회/수요 → SVC(Service Value Chain) → 가치 산출물이라는 흐름을 가지며, 거버넌스·실천·가이딩 원칙·지속적 개선이 SVC를 둘러싼다. 4가지 차원(조직·정보기술·파트너·가치흐름) 모델이 서비스를 입체적으로 지원하며 DevOps·애자일·린을 프레임워크 내에 수용한다.

📢 **섹션 요약 비유**: v3는 조립 라인처럼 단계마다 부품이 이동하는 공장이고, v4는 각 팀이 자율적으로 협업해 제품을 완성하는 스타트업 스튜디오다.

## Ⅱ. 아키텍처 및 핵심 원리

| 구분 | ITIL v3 (2007/2011) | ITIL 4 (2019) |
|:---|:---|:---|
| 핵심 구조 | 5단계 수명주기 | SVS + SVC |
| 단위 개념 | 프로세스 (26개) | 실천 (Practice, 37개) |
| 가이드 문서 | 5권 서적 | 4권 + Foundation 1권 |
| 애자일·DevOps | 별도 확장판 필요 | SVC 활동에 내재화 |
| 고객 가치 | 단계별 산출물 | 공동 가치 창출 (Co-creation) |
| 지속적 개선 | CSI 단계 | SVC 모든 활동에 내장 |

```
┌─────────────────────────────────────────────────────────┐
│                   ITIL 4 SVS 구조                        │
│  기회/수요                                               │
│     │                                                    │
│     ▼                                                    │
│  ┌──────────────────────────────────────────────────┐   │
│  │             Service Value Chain (SVC)            │   │
│  │  계획(Plan) → 참여(Engage) → 설계/전환(D&T)      │   │
│  │  공급(Obtain) → 제공(Deliver) → 개선(Improve)    │   │
│  └──────────────────────────────────────────────────┘   │
│     │                                                    │
│  [거버넌스]   [37개 실천]   [가이딩 7원칙]              │
│     │              │               │                     │
│     └──────────────┴───────────────┘                    │
│                     │                                    │
│                  가치 산출물                             │
└─────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: SVC의 6가지 활동은 스타트업의 기획·영업·개발·조달·배포·개선 팀이 동시에 돌아가는 것이며, 어떤 팀이든 '가치'라는 목표 하나를 공유한다.

## Ⅲ. 비교 및 연결

| 항목 | v3 수명주기 단계 | v4 대응 실천/SVC 활동 |
|:---|:---|:---|
| 서비스 전략 | SS (Service Strategy) | SVC Plan + 포트폴리오 관리 실천 |
| 서비스 설계 | SD (Service Design) | SVC Design & Transition |
| 서비스 전환 | ST (Service Transition) | SVC Design & Transition + 변경 실천 |
| 서비스 운영 | SO (Service Operation) | SVC Deliver & Support |
| 지속적 개선 | CSI (Continual Service Improvement) | SVC Improve + 지속개선 실천 |
| 인시던트 관리 | SO 프로세스 | 인시던트 관리 실천 |

📢 **섹션 요약 비유**: v3 프로세스는 엄격한 직무 기술서고, v4 실천은 역할이 유연한 조직 내 역량(Competency) 카드다.

## Ⅳ. 실무 적용 및 기술사 판단

IT 서비스 관리 조직이 ITIL v4로 전환할 때 가장 중요한 점은 프로세스 소유자(Process Owner)에서 실천 소유자(Practice Owner)로 책임 구조를 재편하는 것이다. SVC 6활동 중 'Engage(참여)'는 고객·사용자와의 접점을 일원화하며 SPOC(Single Point of Contact) 서비스 데스크와 직결된다. 기술사 논술에서는 v3 조직의 전환 로드맵 제시, 37개 실천의 관리·기술·일반 3개 범주 분류, SVC 활동별 KPI 설계가 차별화 포인트다.

📢 **섹션 요약 비유**: v3→v4 전환은 부서별 칸막이 사무실을 오픈 스페이스로 바꾸는 것처럼, 경계를 허물되 역할은 더 명확하게 가져가는 개혁이다.

## Ⅴ. 기대효과 및 결론

ITIL 4 SVS는 애자일·DevOps·린을 내재화해 IT 서비스 조직이 비즈니스 변화에 신속히 대응하도록 한다. 가이딩 7원칙(가치에서 시작, 현재 상태 유지, 반복·피드백, 협력·가시성 등)은 조직 문화 변화까지 유도하며, co-creation 중심의 가치 창출로 IT가 비용 센터에서 전략 파트너로 도약할 수 있다.

📢 **섹션 요약 비유**: ITIL 4는 IT 부서를 '수리 공장'에서 '혁신 파트너'로 바꾸는 설계도다.

### 📌 관련 개념 맵
| 개념 | 설명 | 연관 키워드 |
|:---|:---|:---|
| SVS (Service Value System) | ITIL 4의 전체 가치 창출 시스템 | SVC, 37 Practices |
| SVC (Service Value Chain) | 수요→가치 변환 6단계 활동 체계 | Plan, Engage, D&T, Deliver |
| 가이딩 원칙 | ITIL 4의 7가지 의사결정 원칙 | 가치 집중, 반복·피드백 |
| 실천 (Practice) | v4의 역량·절차·도구 묶음 37개 | 인시던트, 변경, 서비스 데스크 |
| CSI (Continual Service Improvement) | v3의 지속 개선 단계 = v4 Improve 활동 | PDCA, 개선 레지스터 |

### 👶 어린이를 위한 3줄 비유 설명
1. ITIL v3는 공장처럼 물건이 컨베이어 벨트 위에서 단계별로 만들어지는 방식이에요.
2. ITIL 4는 여러 팀이 동시에 협력해 고객이 원하는 것을 바로 만드는 유연한 작업실이에요.
3. 둘 다 IT 서비스를 잘 관리하려는 목표는 같지만, v4가 더 빠르고 유연하게 움직일 수 있어요.
