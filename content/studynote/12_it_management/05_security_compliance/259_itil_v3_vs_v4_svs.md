## 핵심 인사이트
> 1. **본질**: ITIL v3은 5단계 서비스 수명주기 모델, ITIL 4는 애자일/DevOps를 흡수한 SVS(Service Value System) 가치 중심 모델로 진화했다.
> 2. **가치**: ITIL 4 SVS의 서비스 가치 사슬(SVC)은 6개 활동의 유연한 조합으로 DevOps 파이프라인과 통합 가능하다.
> 3. **판단 포인트**: ITIL 4의 4차원 모델(조직·정보기술·파트너·가치흐름)은 서비스 설계 시 전방위적 관점을 요구한다.

## Ⅰ. 개요 및 필요성

ITIL (IT Infrastructure Library)은 영국 OGC(Office of Government Commerce)가 제정한 ITSM 베스트 프랙티스 라이브러리다. v3(2007)는 서비스 전략→설계→전환→운영→CSI의 선형 수명주기로 IT 운영을 구조화했다. 그러나 클라우드·DevOps·애자일이 확산된 환경에서 v3의 프로세스 중심 접근은 민첩성이 부족하다는 비판을 받았다.

ITIL 4(2019)는 이를 SVS(Service Value System)로 대체하여 서비스 가치 사슬, 거버닝 원칙, 4차원 모델을 포괄하는 통합 프레임워크를 제시했다.

📢 **섹션 요약 비유**: ITIL v3가 조립라인(고정된 순서로 만들기)이라면, ITIL 4는 레고 블록(필요에 따라 조합)이다.

## Ⅱ. 아키텍처 및 핵심 원리

```
┌─────────────────────────────────────────────────────────┐
│  ITIL v3 수명주기          │  ITIL 4 SVS                 │
├────────────────────────────┼─────────────────────────────┤
│  SS 서비스 전략             │  거버닝 원칙(7가지)          │
│  SD 서비스 설계             │  서비스 가치 사슬(SVC 6활동)│
│  ST 서비스 전환             │  34개 관리 관행(Practice)   │
│  SO 서비스 운영             │  지속적 개선                 │
│  CSI 지속적 서비스 개선     │  4차원 모델                  │
└────────────────────────────┴─────────────────────────────┘
```

| 구분 | ITIL v3 | ITIL 4 |
|:---|:---|:---|
| 구조 | 5단계 수명주기 | SVS(가치 시스템) |
| 초점 | 프로세스 준수 | 가치 공동 창출 |
| 애자일 통합 | 미흡 | DevOps/애자일 통합 |
| 관행 수 | 26개 프로세스 | 34개 관리 관행 |

📢 **섹션 요약 비유**: ITIL v3는 교과서 챕터별 공부법, ITIL 4는 프로젝트 기반 학습법이다.

## Ⅲ. 비교 및 연결

| ITIL 4 SVC 활동 | 의미 |
|:---|:---|
| Plan(계획) | 전략·방향 설정 |
| Improve(개선) | CSI 지속 개선 |
| Engage(참여) | 이해관계자 소통 |
| Design & Transition | 서비스 설계·전환 |
| Obtain/Build | 서비스 구성 요소 획득·구축 |
| Deliver & Support | 서비스 제공·지원 |

📢 **섹션 요약 비유**: SVC 6활동은 피자 가게의 메뉴기획→재료조달→조리→배달→고객응대→개선의 순환과 같다.

## Ⅳ. 실무 적용 및 기술사 판단

DevOps 팀에서 ITIL 4를 적용하면 변경 관리를 "Change Enablement(변경 지원)"로 재정의하여 CI/CD 파이프라인의 자동화된 변경을 허용하면서도 중대 변경은 CAB 승인을 거치는 하이브리드 모델을 구성한다. 기술사 시험에서 "ITIL v3 vs v4 진화 방향을 논하라"는 단골 출제다.

📢 **섹션 요약 비유**: ITIL 4는 기존 규정집(v3)을 폐기하지 않고, 스타트업 문화를 흡수해 업그레이드한 버전이다.

## Ⅴ. 기대효과 및 결론

ITIL 4 도입으로 ① 애자일/DevOps 환경에서의 ITSM 적용 가능성 확대, ② 가치 공동 창출(Co-creation)을 통한 비즈니스-IT 협업 강화, ③ 34개 관리 관행의 선택적 조합으로 조직에 맞는 ITSM 구현이 가능해진다.

📢 **섹션 요약 비유**: ITIL 4는 iOS 업데이트처럼 기존 앱(프로세스)을 유지하면서 새 기능(DevOps 통합)을 추가한 것이다.

### 📌 관련 개념 맵
| 개념 | 설명 | 연관 키워드 |
|:---|:---|:---|
| SVS | 서비스 가치 시스템 | ITIL 4 |
| SVC | 서비스 가치 사슬 6활동 | Plan/Improve/Engage |
| 4차원 모델 | 서비스 관리 4가지 관점 | 조직·기술·파트너·흐름 |
| CSI | 지속적 서비스 개선 | PDCA |

### 👶 어린이를 위한 3줄 비유 설명
1. ITIL v3는 학교 교과서처럼 정해진 순서대로 배우는 방법이에요.
2. ITIL 4는 스마트폰처럼 필요한 앱(기능)을 골라서 쓰는 더 유연한 방법이에요.
3. 둘 다 IT 서비스를 잘 관리하려는 목표는 같아요.
