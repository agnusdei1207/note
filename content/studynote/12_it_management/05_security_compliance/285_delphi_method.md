## 핵심 인사이트
> 1. **본질**: 델파이 기법 (Delphi Method)은 다수의 전문가 의견을 익명으로 수집하고 반복적 피드백을 통해 합의에 도달하는 구조적 미래 예측·의사결정 방법론이다.
> 2. **가치**: 전문가 그룹의 경험과 직관을 체계적으로 수집하면서, 권위·지위에 의한 압력 없이 순수한 전문 의견 합의를 도출한다.
> 3. **판단 포인트**: 기술사 시험에서 델파이 기법은 주로 IT 프로젝트 규모 산정(코스트 추정), 신기술 도입 타당성 평가, 리스크 식별에 적용 방법으로 출제된다.

## Ⅰ. 개요 및 필요성

델파이 기법은 랜드 연구소(RAND Corporation)가 1950년대 개발한 기법으로, 전문가 패널에게 설문을 반복 실시하고 전 라운드 결과를 익명으로 피드백하여 의견을 수렴시키는 방식이다.

📢 **섹션 요약 비유**: 이 개념은 조직의 기초 체력 훈련이다. 당장은 힘들지만 나중에 빛난다.

## Ⅱ. 아키텍처 및 핵심 원리

```
┌─────────────────────────────────────────┐
│  델파이 기법 수행 절차                   │
│  1라운드: 개방형 설문 (자유 의견)        │
│     ↓ 집계·분석                         │
│  2라운드: 구조화 설문 + 1라운드 결과 제공│
│     ↓ 집계·분석                         │
│  3라운드: 재평가 + 이탈 의견 설명 요구  │
│     ↓ 합의 도달                         │
│  최종: 전문가 합의 결과 도출            │
└─────────────────────────────────────────┘
```
| 특징 | 설명 |
|:---|:---|
| 익명성 | 권위자 영향 차단 |
| 반복성 | 의견 수렴 촉진 |
| 피드백 | 타인 의견 학습 기회 |

📢 **섹션 요약 비유**: 이 구조는 잘 설계된 공장 라인이다. 각 단계가 명확하고 흐름이 끊기지 않는다.

## Ⅲ. 비교 및 연결

| 구분 | 델파이 | 브레인스토밍 |
|:---|:---|:---|
| 형식 | 익명 서면 | 대면 토론 |
| 반복 | 여러 라운드 | 1회 |
| 편향 | 낮음(익명) | 높음(동조) |

📢 **섹션 요약 비유**: 이 비교는 여러 도로 중 목적지에 가장 빠른 길을 찾는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

IT 시스템 규모 산정 시 기능 점수(FP) 추정에 델파이 기법을 적용하면 3~5명의 전문가가 독립적으로 추정하고, 이탈 의견의 근거를 공유하며 수렴하는 과정에서 추정 정확도가 향상된다.

기술사 시험에서는 이 개념의 정의·구성 요소·실무 적용·기대 효과를 논술하는 형태로 출제된다.

📢 **섹션 요약 비유**: 실무 적용은 지도를 보며 실제로 길을 걷는 것이다. 이론과 현실의 간극을 메우는 능력이 핵심이다.

## Ⅴ. 기대효과 및 결론

델파이 기법 적용으로 전문가 편향 최소화, 의견 합의 도달 시간 단축, IT 투자 타당성·규모 산정의 신뢰도 향상이 달성된다.

📢 **섹션 요약 비유**: 이 접근법은 나침반이다. 방향을 잃지 않고 목표를 향해 나아갈 수 있게 한다.
