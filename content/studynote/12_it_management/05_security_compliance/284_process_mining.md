## 핵심 인사이트
> 1. **본질**: 프로세스 마이닝 (Process Mining)은 정보 시스템의 이벤트 로그(Event Log)를 분석하여 실제 업무 프로세스를 시각화하고 병목(Bottleneck), 루프, 편차를 자동 진단하는 데이터 기반 BPR 방법론이다.
> 2. **가치**: 기존 BPR이 인터뷰·설문 등 주관적 분석에 의존했다면, 프로세스 마이닝은 ERP·ITSM·BPM의 실제 로그 데이터를 기반으로 객관적이고 정밀한 프로세스 개선 근거를 제공한다.
> 3. **판단 포인트**: 프로세스 마이닝의 3대 분석 유형: 프로세스 발견(Discovery, 실제 흐름 자동 도출)·적합성 검사(Conformance, 규정과 실제 비교)·개선(Enhancement, 병목 제거)이 순서대로 적용된다.

## Ⅰ. 개요 및 필요성

이벤트 로그는 '케이스 ID + 활동 + 타임스탬프'의 구조로, SAP의 이벤트 로그를 프로세스 마이닝 도구(Celonis, UiPath Process Mining)로 분석하면 이상적 프로세스와 실제 실행 간의 편차를 수천 개의 케이스에서 자동으로 탐지한다.

📢 **섹션 요약 비유**: 이 개념은 조직의 기초 체력 훈련이다. 당장은 힘들지만 나중에 빛난다.

## Ⅱ. 아키텍처 및 핵심 원리

```
┌────────────────────────────────────────┐
│  프로세스 마이닝 3대 분석              │
├──────────────┬─────────────────────────┤
│  발견         │ 이벤트 로그 → 실제 흐름│
│ (Discovery)  │ 프로세스 맵 자동 도출  │
├──────────────┼─────────────────────────┤
│  적합성 검사  │ 규정 프로세스 vs 실제  │
│ (Conformance)│ 편차 케이스 탐지       │
├──────────────┼─────────────────────────┤
│  개선         │ 병목 제거 시뮬레이션   │
│ (Enhancement)│ KPI 예측              │
└──────────────┴─────────────────────────┘
```
| 분석 유형 | 질문 | 도구 |
|:---|:---|:---|
| 발견 | "실제로 어떻게 처리되나?" | Alpha 알고리즘 |
| 적합성 | "규정대로 하는가?" | 토큰 재생 |
| 개선 | "어디서 시간이 가장 오래 걸리나?" | 시뮬레이션 |

📢 **섹션 요약 비유**: 이 구조는 잘 설계된 공장 라인이다. 각 단계가 명확하고 흐름이 끊기지 않는다.

## Ⅲ. 비교 및 연결

| 구분 | 전통 BPR | 프로세스 마이닝 |
|:---|:---|:---|
| 데이터 소스 | 인터뷰, 설문 | 이벤트 로그(객관) |
| 소요 시간 | 수개월 | 수일~수주 |
| 정확도 | 주관적 | 100% 데이터 기반 |

📢 **섹션 요약 비유**: 이 비교는 여러 도로 중 목적지에 가장 빠른 길을 찾는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

제조 기업에서 프로세스 마이닝으로 구매 오더 처리 프로세스를 분석, 전체 케이스의 35%가 승인 없이 우회 처리됨을 발견하여 컴플라이언스 위반을 자동 탐지한 사례가 있다.

기술사 시험에서는 이 개념의 정의·구성 요소·실무 적용·기대 효과를 논술하는 형태로 출제된다.

📢 **섹션 요약 비유**: 실무 적용은 지도를 보며 실제로 길을 걷는 것이다. 이론과 현실의 간극을 메우는 능력이 핵심이다.

## Ⅴ. 기대효과 및 결론

프로세스 마이닝 도입으로 BPR 분석 기간 단축(6개월→2주), 프로세스 편차 자동 탐지, RPA 자동화 대상 업무 발굴, 컴플라이언스 감사 자동화가 달성된다.

📢 **섹션 요약 비유**: 이 접근법은 나침반이다. 방향을 잃지 않고 목표를 향해 나아갈 수 있게 한다.
