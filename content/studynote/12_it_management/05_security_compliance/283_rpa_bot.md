## 핵심 인사이트
> 1. **본질**: RPA (Robotic Process Automation)는 규칙 기반의 반복적 디지털 업무(데이터 입력, 조회, 복사-붙여넣기)를 소프트웨어 로봇(Bot)이 자동 수행하도록 하는 기술이다.
> 2. **가치**: IT 시스템 통합(API) 없이도 사용자 인터페이스(UI) 수준에서 자동화가 가능하여 레거시 시스템을 포함한 빠른 업무 자동화가 가능하다.
> 3. **판단 포인트**: RPA는 단순 반복 자동화(Attended/Unattended Bot), AI 결합 지능형 자동화(IPA, Intelligent Process Automation), 초자동화(Hyperautomation)로 성숙도가 발전한다.

## Ⅰ. 개요 및 필요성

RPA 봇은 화면 UI를 통해 사람처럼 로그인, 데이터 입력, 조회, 복사, 보고서 발송 등의 작업을 24시간 무오류로 수행한다. 기존 API 통합이 불가능한 레거시 시스템에도 적용 가능한 것이 핵심 장점이다.

📢 **섹션 요약 비유**: 이 개념은 조직의 기초 체력 훈련이다. 당장은 힘들지만 나중에 빛난다.

## Ⅱ. 아키텍처 및 핵심 원리

```
┌───────────────────────────────────────────┐
│  RPA 성숙도 모델                           │
│  Level 1: Attended Bot (인간 감독 필요)    │
│  Level 2: Unattended Bot (완전 자율 실행)  │
│  Level 3: IPA (AI+NLP+OCR 결합)           │
│  Level 4: Hyperautomation (전사 통합)      │
└───────────────────────────────────────────┘
```
| RPA 대상 업무 | 예시 | 효과 |
|:---|:---|:---|
| 데이터 이동 | ERP→보고서 복사 | 오류 제거 |
| 청구서 처리 | OCR+자동 입력 | 시간 80% 절감 |
| 규제 보고 | 데이터 수집·보고 | 정확도 100% |

📢 **섹션 요약 비유**: 이 구조는 잘 설계된 공장 라인이다. 각 단계가 명확하고 흐름이 끊기지 않는다.

## Ⅲ. 비교 및 연결

| 구분 | RPA | API 통합 |
|:---|:---|:---|
| 방식 | UI 레벨 자동화 | 시스템 간 직접 연동 |
| 속도 | 빠른 구현(주~월) | 느린 구현(월~년) |
| 안정성 | UI 변경 취약 | 안정적 |

📢 **섹션 요약 비유**: 이 비교는 여러 도로 중 목적지에 가장 빠른 길을 찾는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

금융 기관에서 RPA로 대출 심사 서류 검토(매일 500건)를 자동화하여 처리 시간을 8시간→30분으로 단축하고, 직원을 고부가가치 상담 업무에 재배치한 사례가 있다.

기술사 시험에서는 이 개념의 정의·구성 요소·실무 적용·기대 효과를 논술하는 형태로 출제된다.

📢 **섹션 요약 비유**: 실무 적용은 지도를 보며 실제로 길을 걷는 것이다. 이론과 현실의 간극을 메우는 능력이 핵심이다.

## Ⅴ. 기대효과 및 결론

RPA 도입으로 반복 업무 처리 시간 70~90% 절감, 오류율 제로, 직원 만족도 향상(단순 반복 업무 해방), 24시간 운영이 달성된다.

📢 **섹션 요약 비유**: 이 접근법은 나침반이다. 방향을 잃지 않고 목표를 향해 나아갈 수 있게 한다.
