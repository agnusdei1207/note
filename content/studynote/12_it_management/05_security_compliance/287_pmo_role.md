## 핵심 인사이트
> 1. **본질**: PMO (Project Management Office)는 조직 내 여러 프로젝트의 표준화된 관리 방법론·도구·거버넌스를 제공하고, 프로젝트 간 자원 조정 및 포트폴리오 최적화를 지원하는 전담 조직이다.
> 2. **가치**: 개별 프로젝트 관리자(PM)의 역량에만 의존하는 비표준화 운영에서 벗어나 조직 차원의 프로젝트 성공률을 높이고, IT 포트폴리오의 전략 정합성을 확보한다.
> 3. **판단 포인트**: PMO 유형에 따라 역할이 다르다. 지원형(Supportive): 도구·템플릿 제공. 통제형(Controlling): 방법론 준수 감독. 지시형(Directive): PM 직접 통제. 조직 성숙도에 맞는 PMO 유형 선택이 중요하다.

## Ⅰ. 개요 및 필요성

PMO는 기업의 모든 IT 프로젝트를 단일 거버넌스 하에 관리한다. PMBOK·PRINCE2 방법론 표준화, 프로젝트 헌장 승인, 리스크 레지스터 통합 관리, 이해관계자 커뮤니케이션 체계 구축이 핵심 기능이다.

📢 **섹션 요약 비유**: 이 개념은 조직의 기초 체력 훈련이다. 당장은 힘들지만 나중에 빛난다.

## Ⅱ. 아키텍처 및 핵심 원리

```
┌─────────────────────────────────────────┐
│  PMO 유형과 권한 수준                    │
│  ┌──────────────────────────────────┐  │
│  │  지시형 (Directive)  - 최고 권한  │  │
│  ├──────────────────────────────────┤  │
│  │  통제형 (Controlling) - 중간 권한 │  │
│  ├──────────────────────────────────┤  │
│  │  지원형 (Supportive)  - 낮은 권한 │  │
│  └──────────────────────────────────┘  │
└─────────────────────────────────────────┘
```
| PMO 기능 | 설명 |
|:---|:---|
| 방법론 표준화 | PMBOK 기반 PMP 템플릿 |
| 자원 최적화 | 프로젝트 간 인력 배분 |
| KPI 모니터링 | EVM 기반 진척도 관리 |

📢 **섹션 요약 비유**: 이 구조는 잘 설계된 공장 라인이다. 각 단계가 명확하고 흐름이 끊기지 않는다.

## Ⅲ. 비교 및 연결

| 구분 | PMO | PM |
|:---|:---|:---|
| 범위 | 전체 포트폴리오 | 단일 프로젝트 |
| 역할 | 방법론·거버넌스 제공 | 프로젝트 실행 |
| 권한 | 조직 수준 | 프로젝트 수준 |

📢 **섹션 요약 비유**: 이 비교는 여러 도로 중 목적지에 가장 빠른 길을 찾는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

대기업 그룹의 디지털 전환 포트폴리오 관리: PMO가 30개 IT 프로젝트의 EVM 데이터를 통합 대시보드로 모니터링하고, 리소스 병목 시 프로젝트 우선순위를 재조정하여 포트폴리오 ROI를 향상시켰다.

기술사 시험에서는 이 개념의 정의·구성 요소·실무 적용·기대 효과를 논술하는 형태로 출제된다.

📢 **섹션 요약 비유**: 실무 적용은 지도를 보며 실제로 길을 걷는 것이다. 이론과 현실의 간극을 메우는 능력이 핵심이다.

## Ⅴ. 기대효과 및 결론

PMO 구축으로 프로젝트 성공률 향상(평균 30%), 자원 낭비 감소, 프로젝트 포트폴리오의 전략 정합성 확보, 기관학습(Lessons Learned) 축적이 달성된다.

📢 **섹션 요약 비유**: 이 접근법은 나침반이다. 방향을 잃지 않고 목표를 향해 나아갈 수 있게 한다.
