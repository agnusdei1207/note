## 핵심 인사이트
> 1. **본질**: CPM (Critical Path Method, 주공정법)은 프로젝트 네트워크 다이어그램에서 여유 시간(Slack/Float)이 0인 가장 긴 경로를 임계 경로(Critical Path)로 식별하여 프로젝트 최단 완료 시간을 결정하는 일정 관리 기법이다.
> 2. **가치**: 임계 경로상의 어느 활동이 하루 지연되면 프로젝트 전체가 하루 지연되므로, PM은 임계 경로 활동에 자원과 리스크 관리를 집중한다.
> 3. **판단 포인트**: 자유 여유(Free Float)는 후속 활동의 최조 시작일에 영향 없이 지연 가능한 시간, 총 여유(Total Float)는 프로젝트 완료일에 영향 없이 지연 가능한 시간이다.

## Ⅰ. 개요 및 필요성

CPM은 작업의 선후 관계와 소요 시간을 PDM(Precedence Diagram Method)으로 표현하고, 전진계산(Forward Pass: 최조 시작/완료)과 후진계산(Backward Pass: 최만 시작/완료)을 수행하여 각 활동의 여유 시간을 산출한다.

📢 **섹션 요약 비유**: 이 개념은 조직의 기초 체력 훈련이다. 당장은 힘들지만 나중에 빛난다.

## Ⅱ. 아키텍처 및 핵심 원리

```
┌──────────────────────────────────────────────┐
│  CPM 네트워크 다이어그램 예시               │
│       [A:3일]→[C:4일]                        │
│  시작→               →종료 (임계경로: A-C)   │
│       [B:2일]→[D:2일]                        │
│  경로1: A+C = 7일 ← Critical Path           │
│  경로2: B+D = 4일 (Total Float = 3일)        │
└──────────────────────────────────────────────┘
```
| 용어 | 정의 |
|:---|:---|
| ES (Early Start) | 최조 시작일 |
| EF (Early Finish) | 최조 완료일 |
| LS (Late Start) | 최만 시작일 |
| TF (Total Float) | 총 여유 = LS-ES |

📢 **섹션 요약 비유**: 이 구조는 잘 설계된 공장 라인이다. 각 단계가 명확하고 흐름이 끊기지 않는다.

## Ⅲ. 비교 및 연결

| 구분 | CPM | PERT |
|:---|:---|:---|
| 활동 시간 | 결정론적(1개 추정치) | 확률론적(3점 추정) |
| 목적 | 임계 경로 파악 | 통계적 완료일 예측 |
| 활용 | 건설, IT 프로젝트 | 연구개발, 불확실 프로젝트 |

📢 **섹션 요약 비유**: 이 비교는 여러 도로 중 목적지에 가장 빠른 길을 찾는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

임계 경로 압축(Crashing): 임계 경로 활동에 자원을 추가하여 프로젝트 기간을 단축. 패스트 트래킹(Fast Tracking): 순차 활동을 병렬 수행. 두 기법 모두 추가 비용·리스크가 발생한다.

기술사 시험에서는 이 개념의 정의·구성 요소·실무 적용·기대 효과를 논술하는 형태로 출제된다.

📢 **섹션 요약 비유**: 실무 적용은 지도를 보며 실제로 길을 걷는 것이다. 이론과 현실의 간극을 메우는 능력이 핵심이다.

## Ⅴ. 기대효과 및 결론

CPM 적용으로 프로젝트 최단 완료 일정 파악, 자원 집중 투자 지점 명확화, 일정 압축 의사결정 지원이 달성된다.

📢 **섹션 요약 비유**: 이 접근법은 나침반이다. 방향을 잃지 않고 목표를 향해 나아갈 수 있게 한다.
