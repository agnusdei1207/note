## 핵심 인사이트
> 1. **본질**: 데이터 스튜어드십 (Data Stewardship)은 현업 부서의 실무 담당자(데이터 스튜어드)가 데이터의 정의, 품질, 생명주기를 직접 책임지는 분산형 데이터 관리 체계다.
> 2. **가치**: IT 부서만으로는 전사 데이터 품질을 감당할 수 없다. 데이터를 가장 잘 아는 현업이 스튜어드 역할을 맡아야 SSOT(Single Source of Truth)가 실현된다.
> 3. **판단 포인트**: 데이터 스튜어드의 권한과 책임이 명확히 정의되지 않으면 형식적 역할에 그친다. CDO(Chief Data Officer)가 전체 거버넌스를 총괄하고, 스튜어드가 도메인별로 실행해야 한다.

## Ⅰ. 개요 및 필요성

데이터 스튜어드는 도서관 사서다. 책(데이터)이 어디 있고, 어떻게 분류되고, 언제까지 보관하는지를 책임지는 사람이다.

금융 기업에서 고객 데이터의 '잔고' 필드 정의가 팀마다 달라 분석 결과가 다르게 나오는 문제가 빈번하다. 데이터 스튜어드가 '잔고'의 공식 정의, 형식, 허용 값을 정의하고 데이터 카탈로그에 등록하면 이를 해결할 수 있다.

📢 **섹션 요약 비유**: 데이터 스튜어드십이 없으면 '고객이 몇 명이냐'는 질문에 팀마다 다른 숫자가 나온다.

## Ⅱ. 아키텍처 및 핵심 원리

```
┌──────────────────────────────────────────┐
│        데이터 거버넌스 조직 구조          │
├──────────────────────────────────────────┤
│  CDO (Chief Data Officer) - 전략 총괄    │
│           ↓                              │
│  데이터 거버넌스 위원회 - 정책 결정      │
│           ↓                              │
│  도메인별 데이터 스튜어드 - 실무 책임    │
│  (고객 도메인, 상품 도메인, 재무 도메인) │
└──────────────────────────────────────────┘
```

| 역할 | 책임 | 도구 |
|:---|:---|:---|
| CDO | 데이터 전략·정책 | 거버넌스 위원회 |
| 데이터 스튜어드 | 도메인 데이터 품질 | 데이터 카탈로그 |
| 데이터 엔지니어 | 파이프라인 구축 | ETL/ELT |

📢 **섹션 요약 비유**: 데이터 스튜어드는 축구팀의 포지션별 주장이다. 전체 감독(CDO) 아래 각 포지션(도메인)의 데이터를 책임진다.

## Ⅲ. 비교 및 연결

| 구분 | 데이터 오너 | 데이터 스튜어드 |
|:---|:---|:---|
| 레벨 | C레벨·임원 | 현업 실무자 |
| 책임 | 전략적 데이터 소유 | 운영적 품질 관리 |
| 의사결정 | 정책 승인 | 정의·표준화 |

📢 **섹션 요약 비유**: 데이터 오너가 토지 주인(전략)이라면, 데이터 스튜어드는 실제 농사짓는 농부(운영)다.

## Ⅳ. 실무 적용 및 기술사 판단

MDM(Master Data Management) 프로젝트에서 고객 ID 통합 시 각 시스템(ERP, CRM, SCM)의 스튜어드가 모여 '고객'의 공식 정의와 중복 제거 규칙을 합의해야 한다.

📢 **섹션 요약 비유**: 데이터 스튜어드십 없는 MDM은 지도 없이 보물 찾기다. 누가 무슨 데이터를 가지고 있는지 아무도 모른다.

## Ⅴ. 기대효과 및 결론

데이터 스튜어드십 체계 구축으로 ①데이터 품질 향상(DQ Score 개선), ②규제 컴플라이언스(개인정보 생명주기 관리), ③분석 신뢰성 확보가 달성된다.

📢 **섹션 요약 비유**: 데이터 스튜어드는 정원사다. 각자 맡은 구역의 식물(데이터)을 돌봐야 전체 정원(전사 데이터)이 아름답다.

### 📌 관련 개념 맵
| 개념 | 설명 | 연관 키워드 |
|:---|:---|:---|
| MDM | 마스터 데이터 관리 | SSOT 달성 |
| 데이터 카탈로그 | 데이터 자산 탐색 | 스튜어드 등록 |
| DQ 지표 | 데이터 품질 측정 | 완전성·정확성 |

### 👶 어린이를 위한 3줄 비유 설명
1. 데이터 스튜어드는 학교 도서관 도우미 같아요. 어느 책이 어디 있는지, 낡은 책은 어떻게 할지 담당해요.
2. 각 부서 담당자가 자기 부서 데이터를 올바르게 관리하면 회사 전체 데이터가 믿을 수 있게 돼요.
3. CDO(교장 선생님)가 방향을 정하면, 스튜어드(각 반 담임)가 직접 실행해요.
