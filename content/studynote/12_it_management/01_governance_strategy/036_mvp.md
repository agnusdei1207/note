> **핵심 인사이트**
> 1. MVP (Minimum Viable Product)는 핵심 가치 가설을 검증하는 데 필요한 최소 기능만 갖춘 제품으로, 린 스타트업(Lean Startup) 방법론의 핵심 개념이다 — 가능한 빠르게 학습하는 것이 목표다.
> 2. "최소(Minimum)"는 빈약하거나 불완전한 것을 의미하지 않는다. 고객이 핵심 가치를 경험하고 피드백을 제공할 수 있는 최소한의 완성도를 의미한다.
> 3. 구축-측정-학습(Build-Measure-Learn) 피드백 루프를 가능한 짧은 시간에 반복하는 것이 MVP의 본질이며, 피벗(Pivot)이나 지속 여부 결정의 근거가 된다.

---

## I. MVP의 본질

```
린 스타트업 피드백 루프:

아이디어 -> [BUILD] -> 제품(MVP)
                          |
                       [MEASURE] -> 데이터
                          |
                       [LEARN]
                          |
                          +-> 피벗 (방향 전환)
                          +-> 지속 (계속 발전)
                          +-> 종료 (실패 인정)
```

| 구분        | 설명                             |
|-----------|----------------------------------|
| Minimum   | 학습에 필요한 최소 기능만           |
| Viable    | 고객이 실제로 사용할 수 있는 완성도 |
| Product   | 실제 시장에 출시 가능한 형태        |

> 📢 **섹션 요약 비유**: 자동차를 만들기 전 스케이트보드 -> 킥보드 -> 오토바이 -> 차 순으로 이동 수단의 핵심 가치를 검증하는 것.

---

## II. MVP 유형

```
1. 랜딩 페이지 MVP:
   실제 제품 없이 관심 있는 고객 수를 측정
   Dropbox 설명 영상 -> 7만 명 신청 확인

2. 컨시어지 MVP:
   자동화 없이 수동으로 서비스 제공
   Zappos: 신발 가게 사진을 온라인에 올리고
           주문 오면 실제 가게서 구매 후 발송

3. 마법사의 이면 (Wizard of Oz) MVP:
   고객은 자동화된 것처럼 보이지만 뒤에서 수동 처리
   초기 AI 추천 서비스: 실제로 사람이 추천

4. 프로토타입 MVP:
   핵심 기능만 포함한 최소 버전
   Facebook: 하버드 학생만 사진 업로드·연결
```

> 📢 **섹션 요약 비유**: 레스토랑 메뉴 전체를 개발하기 전, 한 가지 메뉴로 팝업 식당을 열어 반응 테스트 — 실패해도 손실이 최소화된다.

---

## III. MVP vs MMP vs MDP

```
MVP (Minimum Viable Product):
  목적: 가설 검증 (학습 우선)
  고객: 얼리 어답터
  완성도: 최소

MMP (Minimum Marketable Product):
  목적: 시장 출시 (매출 발생)
  고객: 일반 사용자
  완성도: 상업적 판매 가능

MDP (Minimum Delightful Product):
  목적: 고객 감동 (WOW 경험)
  고객: 주류 시장
  완성도: 감동을 주는 수준
```

| 단계 | 목적         | 완성도     | 고객층      |
|-----|------------|-----------|------------|
| MVP | 학습·검증    | 최소       | 얼리 어답터 |
| MMP | 시장 진출    | 상업적     | 대중         |
| MDP | 고객 감동    | 탁월       | 주류 시장   |

> 📢 **섹션 요약 비유**: MVP는 레시피 테스트, MMP는 정식 메뉴 출시, MDP는 미슐랭 별 수준 — 단계별로 완성도와 목표가 다르다.

---

## IV. MVP 설계 원칙

```
MVP 설계 체크리스트:

1. 핵심 가치 가설 정의
   "우리는 [고객]이 [문제]를 [해결책]으로 해결한다고 믿는다"

2. 최소 기능 범위 결정
   MoSCoW: Must Have / Should Have / Could Have / Won't Have

3. 성공 지표(KPI) 사전 정의
   "7일 재방문율 30% 이상이면 성공"

4. 시간 제한 설정
   "2주 스프린트 내 출시"
```

> 📢 **섹션 요약 비유**: 시험 범위를 미리 정하고 그것만 완벽히 공부하는 것 — 모든 것을 다 하려다 아무것도 못 하는 것보다 낫다.

## V. 실무 시나리오 — B2B SaaS MVP

| 단계           | 내용                                      |
|-------------|-------------------------------------------|
| 가설 정의     | "중소기업 HR 담당자는 엑셀 대신 자동화 급여 계산 도구를 원한다" |
| MVP 범위     | 급여 계산 + 이메일 발송 (관리자 기능 없음) |
| 기간          | 3주 개발 + 2주 테스트                      |
| 타깃 고객    | 파일럿 기업 5개사 무료 제공               |
| 성공 지표    | 3개 기업이 두 번 이상 사용                 |
| 결과 반영    | 피드백으로 MMP 기능 우선순위 결정          |

> 📢 **섹션 요약 비유**: 5개 회사에 무료 체험을 주는 것이 500개 회사에 팔 제품을 만드는 것보다 훨씬 빠르고 확실한 검증이다.

## 👶 어린이를 위한 3줄 비유 설명

1. MVP는 완벽한 제품을 만들기 전에 핵심 기능만으로 사람들이 좋아하는지 먼저 확인하는 방법이에요.
2. 모든 기능을 다 만들었다가 아무도 안 쓰면 시간과 돈 낭비니까, 작게 시작해서 반응을 보는 거예요.
3. Dropbox가 실제 앱 없이 영상 하나로 7만 명의 관심을 확인한 것이 대표적인 MVP예요!
