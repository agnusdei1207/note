# 472. BSIMM (Building Security In Maturity Model)

> ⚠️ 이 문서는 "우리 회사의 소프트웨어 보안 실력은 구글이나 애플에 비해 몇 점이나 될까?"라는 의문을 해결하기 위해, **전 세계 130개 이상의 일류 기업들이 실제로 하고 있는 보안 활동들을 관찰(관찰적 모델)하여 통계를 내고, 우리 회사의 보안 성숙도를 벤치마킹할 수 있게 만든 평가 기준표, 'BSIMM'**을 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: BSIMM(비심)은 이론가들이 책상머리에서 "이렇게 보안을 하세요!"라고 명령하는 처방적(Prescriptive) 모델이 아니다. "잘나가는 애들은 진짜로 뭘 하고 있지?"를 관찰한 **관찰적(Descriptive) 통계 모델**이다.
> 2. **가치**: 우리 회사가 시큐어 코딩은 잘하고 있는지, 모의 해킹은 남들만큼 하고 있는지 4개 영역 12개 실무(Practice) 항목으로 점수를 매겨(Maturity), 타 회사(경쟁사)와 벤치마킹하고 약점을 객관적으로 채우는 데 쓰인다.
> 3. **기술 체계**: 거버넌스(조직/교육), 인텔리전스(보안 기준/위협 모델링), SSDL 터치포인트(아키텍처 리뷰/코드 리뷰), 배포(모의 해킹/방화벽)라는 4개의 큰 뼈대로 구성되어 있다.

---

### Ⅰ. 개요: 남들은 어떻게 하고 있을까? (Context & Necessity)
우리가 토스(Toss) 같은 핀테크 스타트업 보안팀이라고 치자.
개발팀에게 "코딩할 때 보안 테스트 좀 빡세게 합시다"라고 말하면 개발팀은 반발한다. "아니, 어느 회사가 그렇게 빡세게 해요? 기준이 뭔데요?"

이때 보안팀장이 책상에 탁 던지는 문서가 바로 **BSIMM (비심)**이다.
"이것 봐! 전 세계 금융 회사 50개를 조사해 보니까, 90%의 회사가 코드 커밋할 때 자동화된 정적 보안 분석(SAST)을 돌리고 있어! 우리만 안 하고 있는 거라고!"

BSIMM은 2008년 게리 맥그로(Gary McGraw) 등의 보안 학자들이 MS, 구글, 뱅크오브아메리카 같은 일류 기업들의 **'실제 소프트웨어 보안 훈련(SSDL) 현황'**을 직접 발로 뛰며 관찰하여 만든 데이터베이스다. 
"이렇게 해야 한다"는 잔소리가 아니라, "남들은 이렇게 하고 있다"는 명백한 통계치이므로 회사의 경영진과 개발자를 설득하는 가장 강력한 무기(성숙도 평가 모델)가 된다.

> 📢 **섹션 요약 비유**: 수능 공부를 할 때 "하루에 10시간 공부해라"라고 선생님이 잔소리(처방적 모델)를 하면 잘 안 듣습니다. 하지만 BSIMM은 "전국 1등부터 100등까지의 학생들을 관찰해 보니, 95명이 오답 노트를 쓰고 80명이 기출문제를 5번 풀더라(관찰적 모델)"라는 통계 데이터입니다. 학생(우리 회사)은 이 표를 보고 자신의 부족한 공부법을 깨닫게 됩니다.

### Ⅱ. BSIMM의 4대 핵심 영역 (Framework)

BSIMM은 기업의 보안 수준을 4개의 큰 덩어리(Domain)와 그 아래 12개의 실천 항목(Practice)으로 쪼개서 채점한다.

#### 1. 거버넌스 (Governance) - 뼈대 세우기
- **의미**: 보안을 개발자 개인의 책임이 아니라 회사 차원의 제도로 만들었는가?
- **항목**: 보안 임원(CISO)이 있는지, 신입 개발자에게 시큐어 코딩 교육을 의무적으로 시키는지 측정한다.

#### 2. 인텔리전스 (Intelligence) - 두뇌 탑재
- **의미**: 회사만의 명확한 보안 규정과 공격자 지식(해커의 수법)을 보유하고 있는가?
- **항목**: 암호화는 SHA-256을 쓰라는 사내 코딩 표준 가이드가 있는지, 기획 단계에서 **위협 모델링(Threat Modeling)**을 수행하여 해커의 공격을 미리 예상하는지 평가한다.

#### 3. SSDL 터치포인트 (SSDL Touchpoints) - 실전 방어 (가장 중요)
- **의미**: 개발 과정(SDLC) 곳곳에 직접 개입하여 코드를 뜯어보고 흠집을 잡는가?
- **항목**: 아키텍처 보안 리뷰를 하는가? 사람이 직접 코드를 까보는 **코드 리뷰(Code Review)**와, 기계가 검사하는 정적 분석(SAST)을 파이프라인에 구축해 두었는가?

#### 4. 배포 (Deployment) - 실전 운영 방패
- **의미**: 코드가 서버에 올라가기 직전, 그리고 운영 중일 때 어떻게 방어하는가?
- **항목**: 외부 화이트해커를 불러 **모의 해킹(Penetration Test)**을 정기적으로 돌리는가? 웹 방화벽(WAF)이나 실시간 모니터링 체계가 잘 돌아가는가?

┌──────────────────────────────────────────────────────────────────────────────────────────┐
│           BSIMM의 '거미줄(Spider Web)' 성숙도 벤치마킹 시각화                            │
├──────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                          │
│       (거버넌스)  100점                                                                  │
│           |      /                                                                       │
│          70 ─ 🟢 ─ (인텔리전스)                                                          │

### Ⅲ. BSIMM vs SAMM (면접 단골 비교)

BSIMM과 항상 세트로 나오는 것이 OWASP 재단에서 만든 **SAMM (Software Assurance Maturity Model)**이다.

* **BSIMM (비심) = 관찰적(Descriptive) 모델**
  - "다른 회사들은 이렇게 하고 있어요." (현실의 팩트)
  - 우리 회사가 업계 평균 벤치마킹을 통해 "우물 안 개구리"임을 깨닫고 자극받는 데 최고다. 매년 새로운 데이터(BSIMM 13, 14...)가 업데이트된다.
* **SAMM (샘) = 처방적(Prescriptive) 모델**
  - "너네 보안 성숙도를 올리려면 1단계 땐 이거 하고, 2단계 땐 이거 해라!" (잔소리 가이드)
  - 이제 막 보안 조직을 꾸린 스타트업이 "무엇부터 시작해야 할지" 로드맵을 짤 때 아주 훌륭한 나침반 역할을 한다. 오픈소스이므로 공짜다.

### Ⅳ. 결론

"보안은 보이지 않는 위협과의 싸움이지만, 보안 관리는 철저히 데이터와의 싸움이다."
우수한 소프트웨어를 만드는 기업은 해커가 침입했을 때 뛰어난 해킹 방어 스크립트 하나로 막아내는 회사가 아니다. 개발자에게 꾸준히 보안 교육을 시키고, 설계도에 위협 모델링을 강제하며, 배포 전 모의 해킹 프로세스를 파이프라인에 시멘트처럼 굳혀버린 회사(거버넌스)다. BSIMM(Building Security In Maturity Model)은 이러한 '보안 내재화(Building In)'의 수준을 숫자로 엑스레이 찍어주는 도구다. CEO나 임원들에게 "해커가 위험해요"라는 막연한 공포 대신, "경쟁사인 B은행은 터치포인트 영역에서 우리보다 40점이나 앞서 있습니다"라는 객관적인 지표를 들이밀 때, 회사의 보안 예산은 비로소 결재 도장을 받게 된다.

### 📌 관련 개념 맵
* **상위 개념**: Secure SDLC (소프트웨어 개발 보안 생명주기)
* **비교/경쟁 모델**: OWASP SAMM (처방적 보안 성숙도 모델)
* **내부 평가 지표 4대장**: Governance, Intelligence, SSDL Touchpoints, Deployment
* **주요 평가 항목(Practices)**: 모의 해킹(Pen Test), 시큐어 코드 리뷰, 위협 모델링

### 👶 어린이를 위한 3줄 비유 설명
1. 내가 수학 공부를 잘하고 있는지 알아보려고, 옆집 똑똑한 철수나 앞집 영희는 매일 수학 문제집을 몇 장 풀고 학원을 몇 번 가는지 몰래 조사했어요 (관찰).
2. 조사해 보니 1등 하는 친구들은 매일 오답 노트를 3장씩 쓰고 있었어요. 하지만 나는 오답 노트를 한 장도 안 쓰고 있었죠! (약점 발견)
3. 이렇게 나보다 훌륭한 전국 1등 기업들이 평소에 '보안 연습'을 얼마나 빡세게 하고 있는지 몰래 조사해서 통계표로 만들어 두고, 우리 회사의 성적과 비교해 보는 성적표가 바로 **BSIMM**이랍니다!
