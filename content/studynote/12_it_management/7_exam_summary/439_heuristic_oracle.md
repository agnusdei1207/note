# 439. 휴리스틱 오라클 (Heuristic Oracle)

> ⚠️ 이 문서는 복잡한 비즈니스 로직이나 빅데이터 렌더링처럼, 모든 테스트 결과에 대해 수학적으로 100% 떨어지는 정답(샘플링)을 만들기 불가능할 때, **일부 핵심 입력값만 정확한 정답으로 채점하고, 나머지 수많은 값들은 테스터의 '경험과 직관(어림짐작)'으로 "음, 이 정도면 맞겠지"라고 통과시키는 타협형 오라클**을 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 휴리스틱(Heuristic)은 '발견법, 직관적 어림짐작'을 뜻한다. 휴리스틱 오라클은 수학 공식처럼 딱 떨어지는 정답지가 없을 때, 인간(테스터)의 노하우와 직감을 믿고 결과의 참/거짓을 어림잡아 판정하는 기준이다.
> 2. **가치**: 빅데이터 통계 화면, 날씨 예측 모델, 복잡한 게임의 데미지 계산처럼 "완벽한 정답은 모르겠지만 100~150 사이로 나오면 대충 정상이야"라고 유연하게 판단해야만 테스트가 전진할 수 있는 늪에서 빛을 발한다.
> 3. **기술 체계**: 완전한 추측은 위험하므로, **"샘플링 오라클(엄격한 정답 10개) + 휴리스틱(나머지 90개는 직관적 추정)"** 형태로 결합하여 엄격함과 효율성을 동시에 잡아내는 하이브리드 방식으로 주로 쓰인다.

---

### Ⅰ. 개요: 엑셀로 계산할 수 없는 세계 (Context & Necessity)
우리가 쇼핑몰 장바구니 합계 금액을 테스트한다면 `10,000원 + 5,000원 = 15,000원`이라는 완벽한 샘플 오라클(정답)을 만들 수 있다.

하지만 넷플릭스의 **'영화 추천 알고리즘'**을 테스트한다고 치자.
* 입력: "내가 로맨스 영화를 3편 봤고, 액션을 1편 봤어."
* 기대 결과(정답지): "이 알고리즘이 나한테 어떤 영화를 첫 번째로 추천해 주는 게 100% 정답일까?"

알 수 없다! 완벽한 정답(참 오라클)은 존재하지 않고, 엑셀로 샘플을 만들 수도 없다.
이때 시니어 기획자나 테스터가 화면을 본다. "음.. 타이타닉이나 노트북 같은 로맨스가 상위 5위 안에 떴네. 그럼 알고리즘이 대충 정상적으로 작동하는 거로 합격(Pass) 줍시다!"

이처럼 계산기처럼 딱 떨어지지 않는 복잡계 소프트웨어에서, 결과가 **합리적인 범주(Reasonable Range)** 안에 들어왔는지를 인간의 직관(Heuristic)으로 판정하는 것이 바로 **휴리스틱 오라클**이다.

> 📢 **섹션 요약 비유**: 미술 대회 심사를 할 때 수학 시험처럼 "답이 4번이니까 10점!" 하고 채점(샘플링 오라클)할 수는 없습니다. 심사위원(테스터)이 자신의 오랜 미적 감각과 경험(휴리스틱)에 비추어 "음, 구도와 색감이 이 정도면 합격점이군!" 하고 어림잡아 채점하는 예술적인 판단법입니다.

---

### Ⅱ. 휴리스틱 오라클의 동작 원리와 한계

휴리스틱은 무당처럼 막 찍는 것이 아니다. "어림짐작의 근거(경험)"가 매우 중요하다.

1. **상식적 경계(Common Sense Boundary)의 활용**
   - 테스트 결과: 고객 나이가 `-5살`로 계산되어 화면에 찍혔다.
   - 테스터: "정확한 나이가 35살인지는 내 알 바 아니지만, 사람 나이가 마이너스가 나올 순 없잖아? 버그다(Fail)!"
   - 이처럼 정확한 값은 몰라도, 상식의 범위를 벗어났을 때 칼같이 잘라내는 것이 휴리스틱의 훌륭한 예시다.
2. **이전 경험의 의존성**
   - 휴리스틱 오라클의 신뢰도는 테스터의 도메인(Domain) 지식에 100% 의존한다. 은행 시스템을 10년 굴린 테스터의 직관(휴리스틱)은 거의 참 오라클에 가깝지만, 신입 테스터의 직관은 치명적인 버그를 통과시키는 재앙이 된다.

#### [ 샘플링 오라클과의 하이브리드 결합 ]
가장 훌륭한 실무 테스팅은 두 오라클을 섞어 쓰는 것이다.
- `A 그룹 (10건)`: 가장 중요한 VIP 결제 금액 계산. $\rightarrow$ 이건 1원도 틀리면 안 되니 **샘플링 오라클**로 엑셀 수식 돌려서 100% 깐깐하게 검증한다.
- `B 그룹 (90건)`: 월별 매출 통계 막대그래프 높이. $\rightarrow$ 눈으로 보기에 대충 우상향하고 있고, 가장 큰 막대가 매출 1위인지만 슥 보고 넘긴다. (**휴리스틱 오라클**)

```text
┌────────────────────────────────────────────────────────────────────────────┐
│           오라클의 엄격성과 비용의 트레이드오프 곡선 시각화                │
├────────────────────────────────────────────────────────────────────────────┤
│                                                                            │
│ 100% 👑 [ 참 오라클 ] : 모든 경우의 수 완벽 계산 (돈/시간 무한대 파탄)     │
│  │                                                                         │
│  │  📊 [ 샘플링 오라클 ] : 핵심 샘플 10개만 수학적으로 완벽 계산           │
│  │                                                                         │
│  │     🤔 [ 휴리스틱 오라클 ] : 정확한 수치 계산 포기, 대충 범위만 눈대중  │
│  0%                                                                        │
│     (낮음) < ────────── 비용 / 정확도 ────────── > (높음)                  │
│                                                                            │
│ ★ 실무 타협점: 중요한 곳엔 샘플링을 쓰고, 정답이 애매하고 복잡한 화면이나  │
│              통계 UI 등에는 휴리스틱을 써서 프로젝트 속도를 높인다!        │
└────────────────────────────────────────────────────────────────────────────┘
```

### Ⅲ. 휴리스틱이 빛을 발하는 현대적 환경

AI(인공지능) 시대가 오면서 휴리스틱 오라클의 가치가 폭등했다.
* **LLM (챗GPT 등) 테스팅**: "한국의 수도는?"이라는 질문에 챗GPT가 "아름다운 도시 서울입니다"라고 대답했다. 완벽한 정답 "서울"과 글자가 다르다고 실패(Fail)시킬 것인가? 문맥상 의미가 맞으니 인간 테스터가 "음, 통과(Pass)!"라고 휴리스틱하게 판정해야만 한다.
* **추천 알고리즘 / 자율주행**: 정답이 없는(Non-deterministic) 무한한 아웃풋을 뱉는 현대 소프트웨어에서는, 어림짐작으로라도 정답의 궤도를 판정해 내는 인간의 직관(Heuristic)이야말로 유일한 검증 툴이다.

---

### Ⅳ. 결론

"수학으로 증명할 수 없는 세계를 경험과 직관으로 정복하라."
휴리스틱 오라클(Heuristic Oracle)은 수학적 엄밀함을 포기하는 대신, 테스트 프로젝트가 진흙탕에 빠져 멈추는 것을 방지하는 윤활유 역할을 한다. 모든 결괏값에 대해 완벽한 예상 결과를 엑셀로 타이핑하는 데 3달을 허비하는 팀보다, 주요 골격은 샘플링 오라클로 단단히 잡고 나머지 변수들은 도메인 전문가의 휴리스틱(직관)으로 빠르게 쓸고 지나가며 1주일에 한 번씩 배포하는 팀이 시장에서 승리한다. 테스팅은 결국 품질에 대한 '신뢰'를 얻는 과정이며, 훌륭한 전문가의 휴리스틱은 때론 1만 줄의 자동화 스크립트보다 더 강력한 신뢰를 제공한다.

### 📌 관련 개념 맵
* **상위 개념**: 테스트 오라클 (Test Oracle - 정답지)
* **결합되는 오라클**: 샘플링 오라클 (Sampling Oracle - 보통 휴리스틱과 섞어서 씀)
* **관련 테스팅 기법**: 경험 기반 테스팅 (Experience-based Testing), 탐색적 테스팅 (Exploratory Testing)
* **적용 사례**: 빅데이터 시각화 렌더링, AI/머신러닝 알고리즘 결과물 테스트

### 👶 어린이를 위한 3줄 비유 설명
1. 계산기에서 `5+5`를 눌렀을 때 `10`이 나오는지 확인하는 건 100% 딱 떨어지는 정답이 있죠 (샘플링).
2. 하지만 일기예보 앱에서 "내일 비 올 확률 60%"라고 떴을 때, 그게 100% 완벽한 정답인지 우리는 죽을 때까지 알 수가 없어요.
3. 이때 기상 캐스터 아저씨(테스터)가 하늘을 보고 구름과 바람을 느낀 다음 "음, 어제 데이터랑 내 경험상 60% 정도 뜬 거면 프로그램이 정상적으로 잘 돌아간 거네!"라고 대충 맞게 어림잡아 합격시켜 주는 것이 **휴리스틱 오라클**이랍니다.
