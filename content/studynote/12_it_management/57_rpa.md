# RPA (Robotic Process Automation)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: RPA(Robotic Process Automation)는 소프트웨어 로봇(Robot)을 활용하여 컴퓨터 기반의 반복적·정형화된 업무 프로세스(입력, 데이터 처리, 시스템 간 전송 등)를 자동으로 수행하게 하는 기술이며, 기존 시스템의 변경 없이前台 레벨에서业务流程自动化을実現한다.
> 2. **가치**: 단순 반복 업무를 자동화하여人力을 핵심 창작적 업무에 집중하게 하며, 24시간 운영, 오류 감소, 처리 속도 향상, 규정 준수 강화 등의 효과를実現한다.
> 3. **융합**: OCR(광학 문자 인식), AI(자연어 처리), BPM(业务流程 관리) 등과 결합되어 단순 정형 처리에서 인지적 업무로 영역을 확장하고 있다.

---

### Ⅰ. 개요 및 필요성 (Context & Necessity)

RPA(Robotic Process Automation)의登場背景에는 기업들이 직면한 몇 가지 구조적 문제가 있다. 첫째, 기업 내には 많은数の既存システムが共存하고 있으며, 이들 시스템을 모두 API 기반으로 연결하는 것은 기술적·비용적으로 实现하기 어렵다. 둘째,deskilling(단순화)된 반복 업무는员工의 창의성을抑制하고 사기저하의原因이 된다. 셋째, 이러한 단순 업무는人力コスト의大部分을 차지하면서도業務적価値가 낮다.

전통적 자동화 방식인 ETL(Extract-Transform-Load)이나 시스템 연계(EAI/ESB)는 개발 기간이 길고 비용이 높으며, 기존 시스템의 변경이required되었다. 그러나 RPA는 '표면层次的自动化(Front-End Automation)'로,人类가 PC를 사용할 때와 동일한画面操作을 소프트웨어 로봇이模倣하여 수행한다. 즉, 기존 시스템에 API가 없더라도 화면을 통해 데이터를 입력하고, 버튼을 클릭하고, 정보를抽出하는 것이 가능하다.

실무에서 RPA가 효과적인 업무의 특징은 다음과 같다.
*   **규칙 기반(Rule-Based)**: 판단 기준이 명확하고 예외 상황이 적음
*   **반복적(Repetitive)**: 동일한 작업이 빈번하게 반복됨
*   **대량 데이터 처리**: 많은 양의 데이터를同一 순서로 처리해야 함
*   **정형화된 입력**: 구조화된 데이터(스프레드시트, forms 등)에서 입력을 받아 처리
*   **다중 시스템 걸침**: 여러 시스템을 오가며 데이터를 입력/조회/전달

반면, RPA에 부적합한 업무의 특징은 다음과 같다.
*   **예외 상황 多发**: 매번 판단이異なりルール化하기 어려움
*   **비정형 데이터 처리**: 이메일 본문, 문서 등構造화되지 않은 정보 handling
*   **자연어 생성/이해**: 복잡한 글이나 대화의 이해·생성
*   **자주 변하는 프로세스**:ルール이 수시로 변경되어 manutenção이频繁

📢 **섹션 요약 비유**: RPA는 마치 거실에 있는 리모컨과 같습니다. TV, 에어컨, 조명, 오디오 등 각각의 기기가 저마다 다른 리모컨을 가지고 있지만,万能 리모컨 하나만으로 모든 기기를 자동화된 程序대로 조작할 수 있습니다. 기기 자체를 바꿀 필요 없이 리모컨(로봇)이 그 역할을 대신합니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

RPA의구현을위해서는 플랫폼 아키텍처, 로봇 유형, 개발 프레임워크에대한 이해가 필요하다.

RPA 플랫폼은通常 다음의 구성 요소로 이루어진다.

| 구성 요소 | 역할 | 설명 |
|:---|:---|:---|
| **Studio/Designer** | 프로세스 설계 및 로봇 스크립트 개발 |drag-and-drop 기반의直观적 개발 환경 |
| **Orchestrator** | 로봇의 실행 관리 및 모니터링 | centrally한 管理 控制塔, 작업 할당, 모니터링 |
| **Robot (Attended/Unattended)** | 실제 업무 수행 | Attended는 사람과 함께, Unattended는完全 무인 자동화 |
| **Control Room** | 전체 RPA 환경의 중앙 관리 | 사용자 관리, 라이선스, 감사 로그 |

로봇의유형은 다음과 같이 구분된다.
*   **Unattended Robot (무인 로봇)**: 백그라운드에서没有人관리로 가동, batch 처리,大批量 업무에 적합
*   **Attended Robot (유인 로봇)**:人間の PC에서 작동,部分 자동화,Desktop automation
*   **Hybrid Robot**: Unattended + Attended 조합

RPA 개발 시 핵심적인 고려사항은 다음과 같다.
1. **로봇의 견고성(Robustness)**: 화면 변경,弹出ウィンドウ,네트워크遅延 등에 대비한 예외 처리 설계
2. **로봇의 재사용성(Reusability)**: 공통 업무는library로 만들어 재사용, 설정値은config로 관리
3. **로봇의 감사 가능성(Auditability)**:操作 로그를詳細하게 기록, 감사 대응 가능
4. **로봇의 관리 용이성(Maintainability)**: 프로세스 변경 시 영향範囲을 최소화하는 모듈식 설계

📢 **섹션 요약 비유**: RPA의 Orchestrator는大型物流倉庫의 중앙控制실과 같습니다. 여러대의 무인 로봇(AGG: Automated Guided Vehicle)이 склаاد물을 운반하고, 控制실이 언제 어떤 로봇에게 어떤 작업을 할당하고, 현재 상황을 모니터링하며, 문제가 생기면 다른 경로로 라우팅합니다. 물류仓库와의 차이점은 '물리적 물체'가 아닌 '데이터'를 처리한다는 점입니다.

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

RPA는 다양한 주변 개념과比較되며, 실무에서는 이를 명확히 구분하여 적용 범위를設定해야 한다.

#### 1. RPA vs 전통 자동화 vs AI 기반 자동화 비교
| 항목 | RPA (로봇 절차 자동화) | ETL/EAI (시스템 연계) | AI 기반 자동화 (지능형) |
|:---|:---|:---|:---|
| **자동화 수준** | 규칙 기반, 정형 데이터 | 데이터 처리, 배치 | 인지/판단/예측 |
| **대상 업무** | 반복적 screen 작업 | 백엔드 데이터 연동 | 비정형 데이터 처리 |
| **개발 난이도** |低 (수 주) | 中 (수 개월) | 高 (수 개월~년) |
| **예시** | 송장 입력, 데이터 entry | 데이터 warehouse 적재 | 고객 이메일 분류, 계약서 분석 |

#### 2. RPA와 BPM의 관계
*   **공통점**:业务流程自动化Goal
*   **차이점**: BPM은业务流程을模型화하고 redesign까지 포함하지만, RPA는 기존流程을 그대로自动化하는 데重点
*   **병행 사용**: BPM으로流程을 개선·최적화하고, RPA로 개선된流程을 자동화하는组合이 효과적

#### 3. RPA와 DevOps의 관계
*   **공통점**:自动化, 빠른 배포, 품질 향상
*   **적용 영역 차이**: DevOps는 소프트웨어 개발/배포에, RPA는 бизнес 업무 처리에 적용
*   **融合趋向**: RPA도 CI/CD 파이프라인처럼 robot 개발/버전 관리/배포를 Pipeline화하는 RPA DevOps 접근이 확산

```text
이 도식은 업무 자동화 수준별 적용 기술의 차이를 보여준다.

[자동화 스펙트럼: 규칙 기반 → 지능형]

[Level 1: 수동 (Manual)]
   │
   │ 인간이 全過程 수동 처리
   ▼
[Level 2: RPA (Rule-Based)]
   │
   │ 규칙을 程序대로 따르는 로봇
   │ 예: 데이터 entry, form 채우기, 시스템 간 데이터 전송
   ▼
[Level 3: RPA + OCR/ICR (문자 인식)]
   │
   │ 스캔 문서, 영수증 등에서 텍스트를 추출하여 처리
   │ 예: 종이 송장 → OCR → RPA 입력
   ▼
[Level 4: RPA + NLP/AI (지능형)]
   │
   │ 비정형 텍스트 처리, 판단·예측 기능 추가
   │ 예: 고객 이메일 의도 파악 → 적절한 부서 라우팅
   ▼
[Level 5: 자동화 + 오토메이션 (엔드투엔드 자동화)]
   │
   │ 인간 개입 없이 End-to-End业务流程全自动处理
   │ 예: 주문受理 → 결제 → 재고 확인 → 배송 → 고객 통보
   ▼
[Level 6: 자율적 최적화 (Self-Optimizing)]
   │
   │ AI가 프로세스 병목를 분석하고 스스로 개선
   │ 예: ML 기반 최적 라우팅, 예측적 유지보수
```

이 스펙트럼의要点은 RPA가 자동화의 初始点이며, OCR, NLP 등 AI 기술을 결합하면서 점차 고도화되어간다는 것이다. 그러나 모든 업무를最高 수준까지的高度화할 필요는 없으며, 업무의 가치와 자동화 개발·유지 비용을 비교하여 적절한 수준을 선택해야 한다.

📢 **섹션 요약 비유**: 자동화 스펙트럼은 자동차의 자율주행 단계와 비교할 수 있습니다. Level 1은긴급 브레이크만 自动으로, Level 2는车道 유지辅助, 그리고 最终적으로完全 자율주행을 实现하는 것과 같습니다. 모든 차종에完全 자율주행을配备할 필요 없이,用途와 비용을 고려하여 적절한 단계를 선택합니다.

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

RPA를 실무에 적용할 때는 다음의 단계를 따르는 것이 일반적이다.

#### RPA 적용 5단계 프로세스
1. **업무 선정 (Process Selection)**
   * 자동화 적격 업무 선별 (규칙 기반, 빈번 반복,大批量)
   * 자동화 효과 분석 (시간 절감, 오류 감소,コスト削減)
   * 우선순위 결정 (높은volume + 높은effect = 먼저自动化)

2. **세부 설계 (Detailed Design)**
   * 현재业务流程 AS-IS 분석
   * 예외 처리 시나리오 정의
   * 바람직한 프로세스 TO-BE 설계
   * RACI 매트릭스 작성

3. **개발 (Development)**
   * Studio에서流程設計/개발
   * 단위 테스트 (Unit Test)
   * 통합 테스트 (Integration Test)

4. **운영 준비 (Deployment Preparation)**
   *Pilot 운영 (일부 사용자/일부 거래)

5. **지속적 운영 (Continuous Operations)**
   * 모니터링 및 성과 측정
   * 예외 사항 검토 및 대응
   * 정기적 개선 (Change Request)
   * 로봇 버전 관리 (RPA DevOps)

#### 자동화 적격 업무 선별 기준
다음의표를 활용하여 자동화 대상을 선정한다.

| 기준 | 설명 | 점수 |
|:---|:---|:---|
| **빈번도** | 업무 수행 빈도 (일별, 주별, 월별) | 높을수록 高점 |
| **정형성** | 프로세스의 표준화程度 | 규칙이 명확할수록 高점 |
| **볼륨** | 처리해야 할 거래 건수 | 많을수록 高점 |
| **시간 소비** | 업무 1건당 평균 소요 시간 | 길수록 高점 |
| **정확성** | 현재 오류율 | 높을수록 高점 |
| **变更 빈도** | 프로세스 변경 빈도 | 적을수록 高점 |

📢 **섹션 요약 비유**: RPA 도입은 집터에家を건축하는 것과 같습니다. 먼저 어떤 집을 지을지設計하고(프로세스 분석), 설계도에問題가 없는지 검토하고(自动化適格성 판단), 그리고 건축을 시작합니다(개발). 집을 지은 후에도 관리하고 수리해야 하듯이(유지보수), 로봇도 정기적인 관리와 개선이 필요합니다.

### Ⅴ. 기대효과 및 결론 (Future & Standard)

RPA는 기업에 즉각적인 효율성 향상과 비용 절감의 효과를提供할 수 있으며, 조직이更高水準의 자동화로 나아가는 위한跳一跳可以实现更高的目标。

#### 정량적/정성적 기대효과
| 구분 | 기대효과 내용 | 측정 지표 예시 |
|:---|:---|:---|
| **정량적** | 업무 처리 시간 단축,人力コスト 절감, 오류율 감소, 처리량 증가 | FTE (Full-Time Equivalent) 절감, 처리 시간 단축율, 오류율 감소율, Cycle Time |
| **정성적** |员工 만족도 향상, 규정 준수 강화, 서비스 수준 안정화 | employee engagement, 감사 적합성, SLA 달성률 |

#### 미래 전망
*   **지능형 RPA로의 진화**: OCR, NLP, Machine Learning과 결합하여 비정형 데이터 처리와 판단 기능이 필요한 업무로 영역 확장
*   **RPA의 Democratization**: 전문 개발자 없이 business用户在 자체적으로 로봇을 개발하는 'Citizen Developer' 확대
*   **슈처 RPA (Swarm RPA)**: 다수의 작은 로봇이collaboratively协作하여 복잡한 업무를 처리하는 개념
*   **Process Mining/RPA의 융합**: Process Mining으로 실제业务流程을발견·분석하고, 이를 기반으로 자동화 대상을精准하게 선별하는 결합

RPA는 자동화의 初期 단계이지만, 기업 文化로서의 자동화 사고를根づかせ고, 더高度な 자동화로 도약하기 위한 기반을 마련하는 데重要한 역할을 한다. 그러나 RPA만으로는数字化 전환의 최종 목표에 도달할 수 없으며, BPM, AI, 클라우드 등의 advanced 기술과 함께End-to-End 자동화를 추구해야 한다.

📢 **섹션 요약 비유**: RPA는 마치 사무실에配置된 '물건 정리 로봇'과 같습니다. 매일 아침마다 책상 위의 서류를分类하여 정리하는无聊한 작업을 대신해주면, 사원은 그 시간을 고객 대응이나 창의적인 기획에 사용할 수 있습니다. 그러나 로봇은 정해진 위치의 물건만 정리할 수 있으므로, 사무실 arrangements가 자주 바뀌면 로봇의 설정도それに合わせて 변경해야 합니다.
