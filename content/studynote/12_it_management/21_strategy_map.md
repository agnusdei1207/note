# 전략 체계도 (전략 맵: Strategy Map)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전략 체계도(Strategy Map)는 BSC의 4개 관점 간 인과 관계를 가시화하여"어떤 활동이最终적으로戦略目標에 기여하는지"를 명확히 하는 도구이다.
> 2. **가치**:抽象적인 전략을 조직 구성원 모두가 이해할 수 있는 공통 언어로 변환하여, 전략의执行力과 소통 효율성을 높인다.
> 3. **구조**: 학습/성장 → 내부 프로세스 → 고객 → 재무 순서의 인과 관계 사슬로 구성된다.

---

### Ⅰ. 개요 및 필요성 (Context & Necessity)

전략 체계도(Strategy Map)는 2000년 로버트 카플란(Robert Kaplan)과 데이비드 노튼(David Norton)이 저서 "[[The Strategy Maps]]"에서 처음 소개한 도구이다. BSC의 4개 관점을 단순히 나열하는 것이 아니라, 관점 간의 인과 관계를 화살표로 연결하여 전략의論理을視覚적으로 표현한다.

전략 체계도가 필요한 이유는 다음과 같다. 첫째,"전략과 실행 사이의 격차(Gap)"를 해소하기 위해서이다. 많은 기업이优秀的 전략을 세우지만, 이를 구성원에게 전달하지 못해 실행이 부진하다. 전략 체계도는 전략을 한눈에 파악할 수 있는図解로 제공하여 전달의 효울성을 높인다. 둘째, 전략의 논리적 근거를 명확히 하기 위해서이다. "왜 이 전략 목표를追求해야 하는가?"에 대한 답을 관점 간 인과 관계를 통해明确히 한다. 셋째, 구성원들의 전략적 이해를 높이기 위해서이다. 전사 전략부터 부서 전략, 개인 목표까지 연결할 때, 전략 체계도는共同의思考框架として機能한다.

전략 체계도의 핵심 원리는"연쇄적인 인과 관계"이다. 학습/성장의 투자가 내부 프로세스 개선을 이끌고, 이는 고객 만족도 향상을 거쳐 최종적으로 재무적 성과로 귀결된다.

```text
이 도식은 전략 체계도의 기본 구조를 보여준다.

[기업 전략: 지속 가능한 성장과 가치 창조]

                         [학습/성장]
                        (미래 동력)
                             │
                ┌────────────┼────────────┐
                │ 핵심 인재   │ 기술 역량   │ 학습 문화
                │ 확보       │ 강화       │ 조성
                └────────────┴────────────┘
                             │ enables
                             ▼
                ┌────────────────────────────┐
                │      내부 프로세스           │
                │  (가치 제공의 원천)          │
                ├─────────────┬──────────────┤
                │ 운영 효율성   │ 혁신 프로세스 │
                │ 비용 절감    │ 신시장 개척   │
                └─────────────┴──────────────┘
                             │ improves
                             ▼
         ┌────────────────────────────┐
         │         고객               │
         │   (가치의 受容者)          │
         ├─────────────┬──────────────┤
         │ 고객 만족    │ 시장 점유율  │
         │ 및 유지     │ 확대        │
         └─────────────┴──────────────┘
                             │ delivers
                             ▼
                ┌────────────────────────────┐
                │         재무               │
                │  (전략의终极적 목표)         │
                │   매출 성장    수익성 개선   │
                └────────────────────────────┘
```

Kaplan과 Norton이 제시한 이 인과 관계 사슬에서, 각 관점의 목표는 단순히 나열된 것이 아니라"이것이 왜 중요한가?"의 답을次の 관점에서 찾는다.

📢 **섹션 요약 비유**: 전략 체계도는'장보기 목록의 이유'와 같습니다. "우유를 사와야 한다"(학습/성장)는 목표만으로는 부족하고,"우유를 사야 하는 이유 → 요리를 제대로 해야 하기 위해 → 요리를 잘해야 손님이满意하기 위해 → 손님이再来하면 가게가 됑되기 위해"라는 연쇄적 이유(인과 관계)를 보여주는 것이 전략 체계도입니다.

### Ⅱ. 전략 체계도 작성 방법론 (Deep Dive)

전략 체계도는 체계적인 단계를 거쳐 작성해야 한다.

#### 전략 체계도 작성 단계

| 단계 | 활동 | 산출물 |
|:---|:---|:---|
| **1단계: 전략 정의** | 비전, 미션, 전략 수립 | 전략 서사 |
| **2단계: 재무 관점 목표** | 전략에서 도출된 재무적 목표 설정 | 재무 관점 목표 |
| **3단계: 고객 관점 목표** | 고객 관점의 가치 제안 정의 | 고객 관점 목표 |
| **4단계: 내부 프로세스 목표** | 가치 제공을 위한 핵심 프로세스 식별 | 내부 프로세스 목표 |
| **5단계: 학습/성장 목표** | 핵심 역량 및 기반 구축 | 학습/성장 목표 |
| **6단계: 인과 관계 연결** | 관점 간 연결고리 설정 | 전략 체계도 |
| **7단계: 검증 및 공유** | 논리적 일관성 검증, 조직 공유 | 검증된 전략 체계도 |

#### 관점별 목표 설정 원칙

| 관점 | 목표 설정 원칙 | 예시 |
|:---|:---|:---|
| **재무** | 전략의 최종成果물 | "매출 500억 원 달성" |
| **고객** | 재무 목표를위한 고객 가치 제안 | "고객 만족도 90%" |
| **내부** | 고객 가치를創出하는 프로세스 | "주문 처리 시간 50% 단축" |
| **학습** | 내부 프로세스 실행을위한 기반 | "핵심 인력 90% 유지" |

```text
이 도식은 전략 체계도 작성 시 인과 관계 검증 방법을 보여준다.

[인과 관계 검증 질문]

1. [학습/성장 → 내부 프로세스]
   "역량 강화를 통해 프로세스가 실제로 개선되는가?"
   예: 교육 시간 증가 → 기술 수준 향상 → 프로젝트 품질 향상?

2. [내부 프로세스 → 고객]
   "프로세스 개선이 고객 만족도 향상에 직접 연결되는가?"
   예: 처리 시간 단축 → 고객 대기 시간 감소 → 만족도 향상?

3. [고객 → 재무]
   "고객 만족/유지가 재무 성과로 연결되는가?"
   예: 고객 만족 향상 → 재방문율 증가 → 매출 증가?
```

각 연결고리에 대해 이러한 질문을 통해 논리적 일관성을 검증해야 한다. 만약"教育 시간 증가"이"고객 만족도 향상"으로 직접 연결되지 않는다면, 사이사이에 필요한 중간 단계(프로세스 개선 등)를 추가해야 한다.

📢 **섹션 요약 비유**: 전략 체계도 작가는'多米諾骨牌 배열'과 같습니다. 첫 번째 블록(학습/성장)을 넘어뜨리면 마지막 블록(재무)까지 연쇄적으로 넘어져야 합니다. 어느 하나라도 빠지면(논리적 연결缺失) 연쇄 반응이 끊어집니다.

### Ⅲ. 전략 체계도와 BSC/KPI의 연계 (Comparison & Synergy)

전략 체계도는 BSC와 KPI와 밀접하게 연결되어 하나의 성과 관리 체계를 形成한다.

#### 3가지 도구의 관계

| 도구 | 역할 | 산출물 |
|:---|:---|:---|
| **전략 체계도** | 인과 관계의 시각화 |"왜"를 설명하는 그림 |
| **BSC** | 4개 관점의 성과 측정 프레임워크 |"무엇을" 측정할지 정의 |
| **KPI** | 구체적 지표 |"얼마나" 달성했는지 숫자 |

```text
이 도식은 3가지 도구의 통합 구조를 보여준다.

[전략 체계도]           [BSC]              [KPI]
      │                   │                  │
      │"왜?"             │"무엇을?"         │"얼마나?"
      ▼                   ▼                  ▼
 [학습/성장] ──────▶ [ BSC 관점 ] ──────▶ [KPI 예시]
                           │
                           │ 재무        │ 매출 500억
                           │ 고객        │ CSAT 90%
                           │ 내부        │ 처리 시간 50% 단축
                           │ 학습        │ 교육 시간 100h
```

전략 체계도가"왜 이 목표가 중요한가?"를回答한다면, BSC는"무엇을 측정해야 하는가?"를回答하고, KPI는"얼마나 달성했는가?"를回答한다. 이 세 가지는 분리해서 사용할 수 있지만, 함께 사용할 때 비로소 전략 관리의 完全한体系가 된다.

#### 전략 체계도 활용 영역

| 영역 | 활용 방법 |
|:---|:---|
| **전략 수립** | 관점 간 인과 관계를 통해 전략의 논리적 타당성 검증 |
| **전략 전달** | 경영진 → 부서장 → 팀장 → 구성원へと階層적으로 전달 |
| **성과 계약** | BSC 관점별 목표를 부서/개인 BSC로 분해하여 계약 |
| **전략 점검** | 정기적 모니터링을 통해 인과 관계의 유효성 검증 |

📢 **섹션 요약 비유**: 전략 체계도, BSC, KPI의 관계는'건축 설계도'와 같습니다. 전략 체계도는"왜 이 건물을 지어야 하는가"(인과 관계), 설계도는"건물이 어떻게 생겨야 하는가"(BSC), 공사 점검은"현재 공사가 설계도에 맞게 진행되고 있는가"(KPI)입니다.

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

전략 체계도를 실무에 적용할 때 주의해야 할 점과 흔한 함정을 분석한다.

#### 전략 체계도 작성 시 흔한 실수

**1. 인과 관계의 부재**
* **문제**: 관점 간 단순罗列으로, 연결고리(화살표)가形骸化
* **예시**:"교육 시간 증가"(학습) →"고객 만족도 향상"(고객)처럼 直接적 연결이 不可能한 경우
* **해결**: 중간 단계(프로세스 개선 등)를 반드시 포함

**2. 재무 관점偏重**
* **문제**: 고객, 내부, 학습/성장의 관점이 재무의 하위 개념으로 전락
* **예시**: 학습/성장 목표가 모두"비용 절감"으로 귀결
* **해결**: 4개 관점을 均霑하게 취급

**3. 정적 도구로 전락**
* **문제**: 한 번 작성 후 방치되어 환경 변화에 대응하지 못함
* **해결**: 분기별 정기적 검토 및更新

```text
이 도식은 전략 체계도 품질 점검 체크리스트를 보여준다.

[전략 체계도 품질 점검]

□ 인과 관계가 논리적으로 타당한가?
□ 각 관점의 목표가 전략 전체와 연결되어 있는가?
□ 목표 간 상호 의존성이 واضح한가?
□ 목표가 측정 가능하고 도전적인가?
□ 구성원들이 전략 체계도를 이해하고 있는가?
□ 정기적으로更新되고 있는가?
```

📢 **섹션 요약 비유**: 전략 체계도는'등산 코스'와 같습니다.頂上(재무)에서 내려다보면 길이 보이지만,実際に 걸어갈 때는 각 체크포인트(관점)를 순차적으로 지나야 합니다. 어느 체크포인트를 스킵하면顶에 오르는 게 아니라 헤매게 됩니다.

### Ⅴ. 기대효과 및 결론 (Future & Standard)

전략 체계도를 체계적으로 활용하면 다음과 같은 효과를 기대할 수 있다.

#### 정량적/정성적 기대효과

| 구분 | 기대효과 내용 | 측정 지표 예시 |
|:---|:---|:---|
| **정량적** | 전략 달성률 향상, 의사결정 효율성 개선 | 전략 목표 달성률 |
| **정성적** | 전략 이해도 제고, 조직 정렬 강화, 소통 효율성 향상 | 전략 인식도, employee engagement |

#### 전략 체계도의 미래

* **디지털화된 전략 체계도**: 온라인 협업 도구를活用한 실시간 업데이트 및 공유
* **AI 기반 분석**: 인과 관계의 유효성을 AI가 분석하여 개선점 제안
* **동적 전략 체계도**: 실시간 KPI 데이터를 반영하여 자동 업데이트

📢 **섹션 요약 비유**: 전략 체계도는'오케스트라의 지휘|score'와 같습니다. 지휘자는 악보(전략 체계도)를 보며 전체 오케스트라(조직)를统领하고, 각 연주자(부서/팀)는 자신의 파트를 연주하며 最终적으로交响曲(전략 목표 달성)을完成합니다.

### 📌 관련 개념 맵 (Knowledge Graph)
* BSC (균형 성과기록표) | 4개 관점에서 전략을 평가하는 프레임워크.
* KPI (핵심 성과 지표) | 전략 목표의 달성도를 측정하는 구체적 지표.
* CSF (핵심 성공 요인) | 전략 목표 달성을 위한 핵심 영역.
* 전략 관리 (Strategic Management) | 전략의 수립, 실행, 통제를全过程管理하는 활동.
* 인과 관계 (Causality) | 한 사건이 다른 사건의 원인이나 결과가 되는関係.

### 👶 어린이를 위한 3줄 비유 설명
1. **개념**: 전략 체계도는'레고 조립 설명서'와 같습니다. 먼저 무슨 모양을 만들지(재무)를 정하고, 그次に 필요한 블록들(고객, 내부, 학습)을 순서대로 쌓아 나가는 방법을 보여줍니다.
2. **원리**: 기초 블록(학습/성장)을 먼저 쌓고, 그 위에 블록(내부 프로세스)을 올리고, 색깔(고객)을 입히면 마지막에完成形(재무)이 나옵니다.
3. **효과**: 설명서를 보면 아이들 모두가 같은 모양을 만들 수 있듯이, 전략 체계도로 全構成원이 같은 방향을 향해 갈 수 있습니다!
