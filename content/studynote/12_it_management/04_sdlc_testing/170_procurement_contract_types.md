## 핵심 인사이트
> 1. **본질**: SW 조달 계약은 FP (Fixed Price, 고정가)와 CR (Cost Reimbursable, 실비정산) 양극단 사이에 위치하며—계약 방식 선택의 핵심은 "범위 불확실성을 누가 감수하는가"다.
> 2. **가치**: 계약 방식 선택이 잘못되면 발주자는 과도한 리스크를 부담하거나, 공급자는 손실을 보면서 일하게 된다—계약 구조가 인센티브 정렬(Incentive Alignment)을 결정한다.
> 3. **판단 포인트**: 범위가 명확하고 변경 가능성이 낮으면 FFP (Firm Fixed Price), 범위가 불명확하고 탐색적 개발이면 T&M (Time and Material) 또는 CR 계약이 적합하다.

---

## Ⅰ. 개요 및 필요성

조달 계약 방식은 PMBOK 조달 관리(Procurement Management) 지식 영역의 핵심이다. 발주자와 공급자 간 리스크 배분, 비용 산정, 인센티브 구조를 결정하는 법적 구속력 있는 합의다.

계약 방식이 중요한 이유는 **인센티브 구조**를 결정하기 때문이다. 고정가(FP) 계약에서 공급자는 예산 내 완료에 강한 인센티브가 있지만, 이를 위해 범위를 최소화하거나 품질을 타협할 유인도 생긴다. 실비정산(CR) 계약에서는 발주자가 모든 비용을 부담하므로 공급자의 비용 절감 유인이 약해진다.

SW 프로젝트의 특성—요구사항 변경, 기술 불확실성, 무형의 산출물—은 전통 건설이나 제조업과 달리 계약 구조 설계를 더 복잡하게 만든다. 특히 Agile 개발 환경에서는 범위가 스프린트마다 변화하므로, 전통적 FFP 계약이 맞지 않는 경우가 많다.

📢 **섹션 요약 비유**: 조달 계약 방식 선택은 택시 vs 렌터카 vs 월 정기권의 선택과 같다. 거리가 확실하면 택시(FFP), 시간이 예측 불가능하면 렌터카(T&M), 매달 정기적으로 쓴다면 월정기권(CPFF)이 적합하다.

## Ⅱ. 아키텍처 및 핵심 원리

### 계약 방식 전체 분류

```text
┌──────────────────────────────────────────────────────────────────┐
│                  조달 계약 방식 분류                              │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  FP (Fixed Price, 고정가)                                        │
│  ├── FFP  (Firm Fixed Price, 완전 고정가)                       │
│  ├── FP-EPA (Fixed Price with Economic Price Adjustment)         │
│  └── FPIF (Fixed Price Incentive Fee)                            │
│                                                                  │
│  CR (Cost Reimbursable, 실비정산)                                │
│  ├── CPFF (Cost Plus Fixed Fee, 실비+고정 수수료)               │
│  ├── CPIF (Cost Plus Incentive Fee, 실비+인센티브 수수료)        │
│  └── CPAF (Cost Plus Award Fee, 실비+성과 보상금)               │
│                                                                  │
│  T&M (Time and Material, 시간+자재)                              │
│  └── 중간형: FP 요소 + CR 요소 혼합                              │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### FP (Fixed Price, 고정가) 상세

```text
1. FFP (Firm Fixed Price, 완전 고정가):
   - 총 계약금액 사전 확정
   - 비용 초과 시 공급자 부담
   - 발주자: 최저 리스크
   - 공급자: 최고 리스크
   - 적합: 범위 완전 명확, 안정적 기술
   
2. FP-EPA (Fixed Price with Economic Price Adjustment):
   - 물가 지수, 환율에 연동해 가격 자동 조정
   - 장기 계약 (3년 이상)에 적합
   - 인플레이션 리스크를 양측이 분담
   
3. FPIF (Fixed Price Incentive Fee):
   - 목표 비용, 목표 수수료, 가격 상한선 설정
   - 실제 비용 < 목표 비용이면 절감 분 공유
   - 공식: 최종 가격 = 실제 비용 + 목표 수수료 + 조정
   - 공급자에게 비용 절감 인센티브 제공
   - PTA (Point of Total Assumption): 공급자가 100% 비용 부담하는 전환점
```

### CR (Cost Reimbursable, 실비정산) 상세

```text
1. CPFF (Cost Plus Fixed Fee, 실비+고정 수수료):
   - 실제 비용 전액 + 사전 합의된 고정 수수료
   - 수수료는 비용 증감과 무관하게 고정
   - 발주자: 최고 비용 리스크
   - 적합: 연구·개발, 불확실성 매우 높은 프로젝트
   
2. CPIF (Cost Plus Incentive Fee, 실비+인센티브):
   - 실제 비용 + 성과 기반 인센티브 수수료
   - 목표 비용 달성 시 인센티브 최대화
   - 공급자에게 비용 절감 동기 부여
   
3. CPAF (Cost Plus Award Fee, 실비+성과 보상금):
   - 실제 비용 + 발주자의 주관적 평가 기반 보상금
   - 평가 기준: 품질, 일정, 협력 태도 등
   - 발주자 재량 높음, 공급자 예측 어려움
```

### T&M (Time and Material, 시간+자재)

```text
T&M 계약:
  - 시간: 시간당 요율 (Man-hour Rate)
  - 자재: 실제 자재비 + 간접비
  
  특징:
  - FP 요소 (시간당 요율 고정) + CR 요소 (시간 무제한)
  - 범위가 명확하지 않지만 빠른 시작이 필요할 때
  - 발주자의 감시·통제 중요 (시간이 늘어날 수 있음)
  
  리스크:
  - 시간이 증가할수록 총 비용 증가 (발주자 리스크)
  - 최대 비용 상한선(Not to Exceed) 설정 권장
  
  적합:
  - Agile 초기 스프린트
  - 규모 미확정 컨설팅
  - 긴급 유지보수
```

### 리스크 분배 다이어그램

```text
발주자 리스크 ◄──────────────────────────────► 공급자 리스크

    낮음                 중간                   높음
  발주자                                        발주자
     │                                            │
     ├── CPAF ─── CPIF ─── CPFF ─── T&M ─── FPIF ─── FP-EPA ─── FFP
     │
  높음                  중간                    낮음
  공급자                                        공급자
```

📢 **섹션 요약 비유**: FP vs CR 선택은 식당 선택과 같다. FFP는 코스 요리(가격 고정, 메뉴 고정), CPFF는 뷔페(먹은 만큼 냄, 다양한 선택), FPIF는 "10만원 안에서 다 먹으면 1만원 돌려줌" 구조다.

## Ⅲ. 비교 및 연결

### 계약 방식 선택 기준

| 기준 | 적합한 계약 방식 |
|:---|:---|
| 범위 완전 명확, 변경 없음 | FFP |
| 범위 명확, 장기 계약 | FP-EPA |
| 범위 명확, 비용 절감 인센티브 필요 | FPIF |
| 범위 불명확, R&D | CPFF |
| 범위 불명확, 성과 동기 부여 필요 | CPIF |
| 성과 기준 복합적, 주관적 평가 | CPAF |
| 범위 불명확, 빠른 착수 | T&M |
| Agile 지속 개발 | T&M 또는 CPIF |

### SOW vs PWS vs SOO

```text
계약 작업 기술서 유형:
  SOW (Statement of Work, 작업 기술서):
    → 방법·절차까지 지정 (FP 계약에 적합)
    → 발주자가 요구 방법을 알 때
    
  PWS (Performance Work Statement, 성과 작업 기술서):
    → 결과만 기술, 방법은 공급자 재량 (CR/CPIF 적합)
    → 공급자 혁신 장려
    
  SOO (Statement of Objectives, 목표 기술서):
    → 목표만 제시, 방법과 성과 기준 공급자 제안
    → 최대 유연성 (초기 탐색 단계)
```

### FP vs CR 주요 특성 비교

| 구분 | FP 계약 | CR 계약 |
|:---|:---|:---|
| 비용 리스크 | 공급자 부담 | 발주자 부담 |
| 범위 명확성 | 높을 때 | 낮을 때 |
| 감시 필요성 | 낮음 | 높음 |
| 행정 부담 | 낮음 | 높음 |
| 적합 환경 | 반복 가능 작업 | R&D, 신기술 |
| 공급자 이익 | 원가 절감 | 수수료 |

📢 **섹션 요약 비유**: SOW와 PWS의 차이는 "이 길로 가세요"(SOW)와 "목적지만 알려드릴게요"(PWS)의 차이다. PWS는 공급자가 더 좋은 방법을 스스로 찾을 수 있는 자유를 주지만, 결과에 대한 책임도 커진다.

## Ⅳ. 실무 적용 및 기술사 판단

### Agile 환경의 계약 방식

```text
Agile 프로젝트 계약 문제:
  - 스프린트마다 요구사항이 변경
  - FFP는 범위 고정을 요구 → Agile과 충돌
  
해결 방안:
  1. T&M + Not to Exceed (NTE, 상한선)
     → 유연성 유지, 비용 통제
     
  2. Sprint-based FFP
     → 스프린트 단위로 고정가 계약
     → 각 스프린트 범위를 스프린트 시작 시 확정
     
  3. CPIF + Agile 성과 지표
     → 비용+인센티브를 팀 속도(Velocity) 기반으로
     
  4. Inception Deck + T&M
     → 초기 이해관계 정렬 후 T&M으로 진행
```

### 기술사 시험 핵심 암기

```text
FP 계약 3종:
  FFP: 완전 고정, 공급자 최대 리스크
  FP-EPA: 물가 연동, 장기 계약
  FPIF: 인센티브 공유, PTA 개념

CR 계약 3종:
  CPFF: 고정 수수료, 비용 절감 유인 없음
  CPIF: 인센티브 수수료, 비용 절감 장려
  CPAF: 주관적 보상금, 발주자 재량

T&M: 중간형, 시간당 요율 고정 + 시간은 유동
```

### FPIF PTA (Point of Total Assumption) 계산

```text
PTA = [(목표 가격 - 공급자 목표 비용) / 구매자 분담비율] + 목표 비용

예시:
  목표 비용: 100만원, 목표 수수료: 10만원
  목표 가격: 110만원 (= 목표 비용 + 목표 수수료)
  가격 상한선: 130만원
  비용 분담비율: 80/20 (발주자 80%, 공급자 20%)
  
  PTA = [(130-110) / 0.80] + 100 = 25 + 100 = 125만원
  
  PTA 이상 비용 발생 시: 공급자가 초과분 100% 부담
  → FFP처럼 동작 (PTA 이상에서 공급자 최대 리스크)
```

📢 **섹션 요약 비유**: FPIF의 PTA는 대출 한도와 같다. 일정 금액까지는 은행(발주자)이 리스크를 분담하지만, 한도 초과분은 대출자(공급자)가 전적으로 책임져야 한다.

## Ⅴ. 기대효과 및 결론

적절한 계약 방식 선택으로:

1. **리스크 최적 배분**: 불확실성을 더 잘 관리할 수 있는 당사자가 해당 리스크를 부담하게 한다.
2. **인센티브 정렬**: 계약 구조가 공급자의 성과 향상 동기를 만들어 프로젝트 성공을 유도한다.
3. **분쟁 최소화**: 명확한 계약 방식이 예산·범위 분쟁의 근거를 미리 제거한다.
4. **Agile 친화성**: T&M 또는 스프린트 기반 계약으로 Agile 개발의 유연성을 계약에 반영한다.

조달 계약의 핵심 교훈은 "완벽한 계약 방식은 없다—프로젝트 특성과 불확실성 수준에 맞는 최선의 선택만 있을 뿐"이다. 발주자와 공급자 모두 지속 가능한 파트너십을 위해 공정한 리스크 분배가 장기적으로 최선이다.

📢 **섹션 요약 비유**: 계약 방식은 팀 스포츠의 선수 계약과 같다. 스타 플레이어(검증된 공급자)에게는 FFP 수준의 연봉을 보장하고, 신인(불확실한 공급자)에게는 기본급+인센티브(FPIF) 구조가 양쪽 모두에게 공정하다.

### 👶 어린이를 위한 3줄 비유 설명

1. FFP는 피자 한 판 2만원처럼 가격이 정해진 것이에요 — 재료가 더 비싸도 가게(공급자)가 손해를 봐요.
2. CPFF는 뷔페처럼 먹은 만큼 내는 것이에요 — 발주자(손님)가 얼마를 먹을지 모르지만, 가게(공급자)는 수수료를 받아요.
3. T&M은 시간당 알바비처럼 일한 시간에 따라 내는 것이에요 — 몇 시간이 걸릴지 모를 때 쓰는 방식이에요.
