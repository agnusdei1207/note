## 핵심 인사이트
> 1. **본질**: SV (Schedule Variance, 일정 차이) = EV - PV로, 완료된 작업의 가치와 계획된 작업의 가치 차이를 나타낸다—양수(+)는 일정 선행, 음수(-)는 일정 지연을 의미하며, 화폐 단위로 일정을 측정한다.
> 2. **가치**: SV는 전통적 일정 지연 측정(날짜 비교)보다 더 객관적이다—완료된 작업의 계획 가치(EV)가 이 시점까지 완료되어야 했던 가치(PV)보다 적으면 지연이다.
> 3. **판단 포인트**: SV의 결정적 한계—프로젝트 완료 시점에는 EV = BAC = PV가 되어 SV = 0으로 수렴한다. 이로 인해 후반부의 일정 지연이 숨겨질 수 있으므로 SPI와 함께 ES (Earned Schedule) 이론으로 보완해야 한다.

---

## Ⅰ. 개요 및 필요성

SV (Schedule Variance, 일정 차이)는 EVM (Earned Value Management)에서 일정 성과를 측정하는 지표다. 기존 일정 지연 측정은 "계획일 vs 실제일"을 직접 비교하지만, SV는 "이 시점까지 완료되어야 할 작업의 가치(PV) 대비 실제로 완료된 가치(EV)"로 측정한다. 이 방식은 비용 기준선(PMB)과 통합되어 있어 일정과 비용을 동일한 측정 체계에서 다룰 수 있다.

SV가 유용한 이유는 일정 지연의 **경제적 의미**를 전달하기 때문이다. 단순히 "2주 지연"이라는 말보다 "SV = -100만원"이라고 하면, 이해관계자는 지연의 규모를 금액으로 직관적으로 이해한다. 또한 SV는 WBS Work Package 수준에서 어느 작업이 지연되고 있는지 분석하는 데도 활용된다.

SV의 가장 중요한 특성은 **프로젝트 완료 시 SV = 0 수렴**이다. 프로젝트가 완료되면 EV = BAC이고, PV도 BAC이므로 SV = EV - PV = 0이 된다. 이는 프로젝트 후반부의 지연 상황이 SV로는 추적되지 않는다는 의미다—SPI와 ES (Earned Schedule) 이론으로 보완이 필요하다.

📢 **섹션 요약 비유**: SV는 학교 시험의 중간 점검과 같다. "지금까지 100문제 중 40문제를 풀어야 했는데 30문제밖에 못 풀었다"면 SV = -10문제(지연)이다—완료 시 모든 문제를 다 풀면(EV=BAC) SV는 0이 된다.

## Ⅱ. 아키텍처 및 핵심 원리

### SV 공식과 해석

```text
SV = EV - PV

예시:
  BAC = 1,000만원
  계획 완료율 = 60% → PV = 600만원
  실제 완료율 = 45% → EV = 450만원

  SV = 450 - 600 = -150만원 (일정 지연)

  해석:
  → 이 시점까지 600만원어치 작업이 완료되어야 했음
  → 실제로는 450만원어치만 완료됨
  → 150만원어치(15% 분량)의 작업이 아직 미완

SV% = (SV / PV) × 100
     = (-150/600) × 100 = -25%
  → 계획 대비 25% 지연
```

### SV 상태 분류

```text
┌──────────────────────────────────────────────────────────────┐
│  SV 상태 분류                                                 │
├──────────────────────────────────────────────────────────────┤
│  SV > 0: 일정 선행 (Ahead of Schedule) ✅                    │
│    → EV > PV: 계획보다 많은 작업이 완료됨                   │
│    → 주의: 품질 타협 또는 낙관적 EV 측정일 수 있음          │
│                                                              │
│  SV = 0: 계획대로 (On Schedule) ✅                           │
│    → EV = PV: 정확히 계획된 만큼 완료됨                     │
│                                                              │
│  SV < 0: 일정 지연 (Behind Schedule) ❌                     │
│    → EV < PV: 계획보다 적은 작업이 완료됨                   │
│    → 주공정(Critical Path) 상의 지연 여부 확인 필요          │
└──────────────────────────────────────────────────────────────┘
```

### SV의 한계와 ES (Earned Schedule)

```text
SV의 한계:
  프로젝트 완료 시 EV = PV = BAC → SV = 0 수렴
  → 후반 지연이 SV로 추적 불가

ES (Earned Schedule) 이론으로 보완:
  ES = EV의 누적 가치가 PV 커브상 어느 시점에 해당하는가
  
  예: 10개월 프로젝트, 7개월째 EV=PV[5개월]에 해당
    → ES = 5개월 (실제 진도는 5개월째 수준)
    → SV(시간) = AT(Actual Time) - ES = 7 - 5 = 2개월 지연
    
  SPI(t) = ES / AT = 5 / 7 = 0.71
    → 표준 SPI보다 더 직관적인 시간 기반 지표
```

📢 **섹션 요약 비유**: SV의 한계는 마라톤 완주 후에는 모든 선수의 SV가 0이 된다는 것이다. 42km를 완주했다면 SV=0이지만, 어떤 선수는 5시간, 어떤 선수는 2시간이 걸렸다—완주 시간의 차이(일정 지연)는 ES 이론으로 추적해야 한다.

## Ⅲ. 비교 및 연결

### SV vs SPI 비교

| 구분 | SV (Schedule Variance) | SPI (Schedule Performance Index) |
|:---|:---|:---|
| 공식 | EV - PV | EV / PV |
| 단위 | 화폐 | 없음 (비율) |
| 기준 | 0 (양수/음수) | 1 (크면 선행, 작으면 지연) |
| 상대적 크기 | 프로젝트 규모 의존 | 규모 독립적 |
| 종료 시 | SV → 0 (수렴) | SPI → 1 (수렴) |

### SV와 비주공정의 관계

```text
중요 포인트:
  SV < 0이어도 주공정(Critical Path)이 지연되지 않으면
  납기에 영향 없을 수 있음

  분석 순서:
  1단계: 프로젝트 전체 SV 확인
  2단계: Work Package별 SV 분해
  3단계: 지연된 Work Package가 주공정 상에 있는지 확인
  4단계: 주공정 지연이면 일정 압축 기법 검토
         비주공정 지연이면 슬랙(Float) 범위 내면 허용
```

📢 **섹션 요약 비유**: SV 분석은 축구 전반전 점수 확인이에요. 전반전 득점(SV)이 마이너스여도, 후반에 따라잡을 수 있는 체력(슬랙)이 있다면 걱정할 필요가 없을 수 있어요. 하지만 주공정이 지연됐다면 즉시 전술(일정 압축)을 바꿔야 해요.

## Ⅳ. 실무 적용 및 기술사 판단

### 기술사 시험 계산 포인트

```text
문제 유형: SV와 SPI를 계산하고 해석하시오.

예시:
  BAC=800만원, 기간 8개월 (현재 4개월 경과)
  계획 완료율: 50% → PV=400만원
  실제 완료율: 40% → EV=320만원
  AC=360만원

  SV = EV - PV = 320 - 400 = -80만원 (지연)
  SPI = EV/PV = 320/400 = 0.80 (20% 지연)
  
  해석:
  → 계획 대비 20% 일정 지연 중
  → 현재 페이스 지속 시 8개월 × (1/0.80) = 10개월로 완료 예상
  → 주공정 지연 여부 확인 필요
  → Fast Tracking 또는 Crashing 검토
```

### SV 개선 전략

| 전략 | 방법 | 적용 조건 |
|:---|:---|:---|
| Fast Tracking | 순서 활동 병렬화 | 의존성이 낮은 활동 존재 |
| Crashing | 주공정에 자원 추가 | 예산 여유 있을 때 |
| 범위 축소 | 저우선순위 기능 제거 | 발주자 협의 가능 |
| 팀 역량 강화 | 훈련, 전문가 지원 | 시간 여유 있을 때 |

📢 **섹션 요약 비유**: SV 개선 전략은 지각 위기의 학생이 취할 수 있는 방법들이다. 뛰어가기(Crashing-추가 노력), 지름길 이용(Fast Tracking-병렬화), 일부 과목 생략(범위 축소)—선택은 상황과 비용에 따라 다르다.

## Ⅴ. 기대효과 및 결론

SV를 체계적으로 관리하면:

1. **일정 지연 조기 감지**: EVM 주기마다 SV 추이를 확인해 지연이 누적되기 전에 조치한다.
2. **비용과 연계된 일정 분석**: SV와 CV를 함께 보면 "일정도 지연되고 비용도 초과"인 최악의 상황을 빨리 인식한다.
3. **자원 배분 우선순위**: SV가 나쁜 Work Package, 특히 주공정 상의 지연 작업에 자원을 우선 배분한다.
4. **이해관계자 기대 관리**: SV 수치로 일정 지연의 규모를 객관적으로 전달한다.

SV는 EVM의 일정 관리 지표이지만, 한계를 이해하고 SPI, ES 이론, 그리고 CPM의 주공정 분석과 함께 활용할 때 진정한 가치가 발휘된다.

📢 **섹션 요약 비유**: SV는 항법 장치의 "예상 경로 대비 현재 위치" 표시와 같다. 하지만 목적지에 도착한 후에는 모든 SV가 0이 되므로—여정 중에 지속적으로 모니터링하고 조기에 경로를 수정하는 것이 핵심이다.

### 👶 어린이를 위한 3줄 비유 설명

1. SV는 "계획보다 빨리 가고 있나, 늦게 가고 있나"를 돈으로 재는 것이에요 — -100만원이면 그만큼 할 일을 아직 못 했다는 뜻이에요.
2. SV가 0으로 끝나는 한계가 있어요 — 마라톤을 6시간에 완주하든 2시간에 완주하든 "완주"라는 사실(SV=0)은 같거든요.
3. 그래서 SPI나 ES 이론도 함께 봐야 해요 — "얼마나 늦게 완주했나"를 시간으로 측정하는 더 정확한 방법이에요.
