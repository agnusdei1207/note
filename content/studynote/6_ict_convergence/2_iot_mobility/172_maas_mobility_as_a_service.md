# 172. 마스 (MaaS, Mobility as a Service)

> **핵심 인사이트**: 집에서 강릉 여행을 갈 때, 마을버스 앱, KTX 앱, 쏘카 앱, 택시 앱을 각각 켜서 예약하고 결제해야 한다. MaaS는 이 모든 이동 수단을 하나의 앱에 모아서 "집 ➔ 버스 ➔ KTX ➔ 킥보드 ➔ 펜션" 경로를 한 번에 짜주고 단 한 번만 결제하게 해주는 모빌리티의 넷플릭스다.

## Ⅰ. MaaS (Mobility as a Service)의 개념
기존에 개별적으로 운영되던 다양한 이동 수단(버스, 지하철, 기차, 택시, 렌터카, 전동 킥보드, 자전거 등)을 **하나의 통합된 플랫폼에서 경로 검색, 예약, 결제까지 원스톱(One-Stop)으로 제공하는 서비스**입니다. 
시민들은 이동 수단을 '소유(차량 구매)'하는 대신, 구독이나 통합 결제를 통해 언제 어디서든 최적의 모빌리티 서비스를 끊김 없이 '이용'할 수 있게 됩니다.

## Ⅱ. MaaS의 발전 단계 (Level 0 ~ 4)

스웨덴의 샬머스(Chalmers) 공대에서 정의한 통합 수준에 따른 5단계 모델입니다.

| 단계 | 수준 | 설명 및 예시 |
|:---|:---|:---|
| **Level 0** | 통합 없음 | 각 회사가 알아서 독자적 서비스 제공 (단순 버스 앱, 단순 택시 앱) |
| **Level 1** | **정보 통합** | 목적지를 치면 **경로와 비용 정보만 하나의 앱에서 보여줌** (카카오맵, 네이버 지도 길찾기) |
| **Level 2** | **예약/결제 통합**| 경로 검색 후 각기 다른 교통수단(기차+택시)을 **하나의 앱에서 연속해서 예약하고 결제**할 수 있음 |
| **Level 3** | **서비스 패키지 통합** | 개별 요금이 아니라, 월 10만 원을 내면 대중교통 무제한 + 쏘카 10시간 + 킥보드 무제한 등 **정기 구독 패키지(Bundle)** 로 통합 제공 |
| **Level 4** | **정책 통합** | 국가/지자체의 인프라 정책과 완벽히 연동되어, 교통 체증 시 대중교통 요금을 깎아주거나 마일리지를 주어 도시 전체의 교통 흐름을 최적화 |

## Ⅲ. MaaS 구현을 위한 핵심 기반 기술
MaaS가 완벽히 동작하려면 방대한 데이터가 0.1초의 지연 없이 융합되어야 합니다.
1. **오픈 API 및 데이터 허브**: 코레일, 버스회사, 킥보드 스타트업의 예약/결제 API가 하나의 클라우드 허브로 통합되어야 합니다.
2. **AI 기반 라우팅**: 사용자의 성향(최단 시간 선호, 최저가 선호, 환승 최소화 선호)을 학습하여 수만 가지 이동 수단 조합 중 최적의 경로를 AI가 실시간으로 뽑아내야 합니다.

## Ⅳ. 기대 효과
자가용 소유 욕구를 줄여 도심의 주차난과 교통 체증을 해결하고, 대중교통이 부족한 외곽 지역의 라스트 마일(Last Mile, 집 앞까지 킥보드로 이동) 문제를 완벽히 해결합니다. 핀란드의 '윔(Whim)' 앱이 전 세계 최초이자 대표적인 성공 사례입니다.

> 📢 **섹션 요약 비유**: 과거에는 해외여행을 갈 때 항공사 사이트, 호텔 사이트, 렌터카 사이트를 일일이 돌아다니며 각각 결제했습니다. 하지만 익스피디아나 스카이스캐너 같은 앱이 나오면서 클릭 한 번에 비행기, 호텔, 렌터카를 패키지로 묶어버렸듯, MaaS는 일상생활의 모든 교통수단을 '클릭 한 번짜리 패키지'로 만들어버린 교통계의 혁명입니다.
