# 214. SDDC (Software Defined Data Center) - 컴퓨트 스토리지 네트워크 가상화 인프라 소프트웨어 통제 클라우드 데이터센터 자동화 프로비저닝

> **핵심 인사이트**: (모든 기계 쇳덩어리를 코드로 짓밟아버린 최후의 성전) 옛날 지하 전산실 관리자는 비참했다. 서버 CPU 모자라면 용산 가서 쇳덩어리(컴퓨터)를 사 오고, 랜선 꽂을 구멍 모자라면 시스코(Cisco) 스위치 쇳덩어리 사서 드라이버로 나사 조이고, 외장하드 용량 모자라면 거대한 스토리지 쇳덩어리 사다 지게차로 날랐다. "야 ㅆㅂ!! 21세기에 언제까지 드라이버 돌리고 랜선 꽂으며 생고생할래!! 당장 전산실에 있는 'CPU(컴퓨터)', '랜선과 공유기(네트워크)', '하드디스크(스토리지)' 이 3대 쇳덩어리들을 몽땅 다 망치로 박살 내버려!! **그리고 거대한 깡통 x86 컴퓨터 1만 대만 깔아놓고, 그 깡통 위를 덮는 거대한 '소프트웨어 마술막(SDDC)'을 쳐!! 이 마술막 위에서 마우스로 딸깍! [서버 생성], [가상 랜선 꽂기], [가상 100TB 하드 연결] 버튼만 누르면 소프트웨어가 밑바닥의 깡통 기계들의 자원을 싹 다 긁어모아 0.1초 만에 허공에 100% 가상의 데이터센터를 완벽하게 창조(프로비저닝)하게 만들어라!!**" 무겁고 더러운 하드웨어의 노예 시절을 끝장내고 데이터센터 전체를 텍스트 코드로 지배해 버린 궁극의 진화, SDDC다.

## Ⅰ. 사일로(Silo) 인프라의 하드웨어 지옥
- 과거의 데이터센터: 서버 담당자 따로, 네트워크(랜선) 담당자 따로, 스토리지(하드디스크) 담당자가 따로 있었습니다(사일로 현상). 
- 회사가 쇼핑몰 하나를 오픈하려면, 이 3명의 담당자가 무거운 전용 쇳덩어리 기계들(IBM 서버, Cisco 라우터, EMC 스토리지)을 낑낑대며 사 와서 한 달 내내 랜선 꽂고 나사 조이며 수동으로 세팅해야 간신히 불이 켜졌습니다. 

## Ⅱ. SDDC (Software Defined Data Center)의 개념 🌟
- **Software Defined (소프트웨어에 의해 완벽하게 정의되고 통제되는)** 
- **개념**: 데이터센터를 구성하는 3대 핵심 물리적 쇳덩어리 하드웨어 자원(서버, 네트워크, 스토리지)을 몽땅 다 가상화(Virtualization) 기술로 껍데기를 벗겨내어 거대한 하나의 자원 웅덩이(Pool)로 통합하고, **이 모든 자원의 할당과 통제, 보안, 배포(프로비저닝)를 중앙의 단일 '소프트웨어 관리 화면'에서 마우스 클릭이나 코드로 100% 자동 제어할 수 있게 만든 차세대 클라우드 데이터센터 아키텍처**입니다.

## Ⅲ. SDDC를 빚어내는 3대 가상화 흑마법 🌟 핵심 기출 🌟
"모든 것을 소프트웨어로 씹어먹겠다!" SDDC를 지탱하는 3개의 기둥입니다. (215, 216번 단독 문서를 관통합니다.)

### 1. SDC (Software Defined Compute) - "서버 쇳덩어리의 가상화"
- 190번에서 배운 내용입니다. 용산에서 사 온 거대한 x86 쇳덩어리 서버에 **하이퍼바이저(VMware 등)** 마술막을 씌웁니다.
- 기계 1대를 소프트웨어로 쪼개어 가상 머신(VM) 100개를 허공에 0.1초 만에 쑴풍쑴풍 찍어내어 CPU와 메모리를 자유자재로 통제합니다.

### 2. SDN (Software Defined Networking) - "물리적 랜선과 공유기의 가상화"
- 옛날엔 네트워크 쪼개려면 랙에 들어가서 직접 노란 랜선을 뽑아 파란 구멍에 다시 꽂았습니다(노가다).
- SDN 마법을 부리면, 거대한 가상의 네트워크 콘트롤러(소프트웨어) 화면을 엽니다. 마우스로 드래그 앤 드롭해서 점과 점을 이으면(가상 랜선 연결), **밑바닥 쇳덩어리 기계들끼리 0.001초 만에 논리적인 전용 도로망과 방화벽이 허공에 완벽하게 세팅(격리)됩니다.** (215번에서 딥하게 팝니다.)

### 3. SDS (Software Defined Storage) - "외장하드 창고의 가상화"
- 옛날 1억짜리 비싼 특수 외장하드(SAN/NAS) 장비들을 다 쓰레기통에 버립니다.
- 그냥 10만 원짜리 싸구려 하드디스크가 달린 깡통 컴퓨터 수천 대를 연결해 놓고, **그 위에 SDS 소프트웨어를 쫙 덮어씌웁니다.** 소프트웨어가 싸구려 하드디스크 수천 개의 용량을 하나로 합쳐서(Pool), "너 C드라이브 10TB 써라!" 하고 논리적으로 뚝 떼어내어 할당해 주는 마법입니다. (216번에서 딥하게 팝니다.)

## Ⅳ. 왜 이게 AWS(아마존 클라우드)의 심장인가? (결론)
- 203번에서 **IaC(테라폼 코드로 서버 만들기)**를 배웠죠? "마우스도 귀찮아! 그냥 메모장에 텍스트 코드로 `서버 10대, 하드 100GB, 방화벽 열어` 라고 치고 엔터 딱 치면 클라우드에 인프라가 1초 만에 생성되는 마법!"
- 이 IaC라는 텍스트 코드 마법이 어떻게 물리 세계에 적용될까요? **아마존 데이터센터 전체 밑바닥이 바로 이 SDDC(모든 것이 소프트웨어로 조종되는)로 100% 떡칠되어 구축되어 있기 때문입니다.** SDDC 뼈대가 없으면, 우리가 누리는 클라우드의 1초 오토스케일링 클릭 마법은 아예 물리학적으로 존재할 수 없습니다.

> 📢 **섹션 요약 비유**: **SDDC(소프트웨어 정의 데이터센터)**는 과거의 시끄러운 **'아날로그 녹음 스튜디오 장비'를 맥북 1대의 '작곡 프로그램(소프트웨어)'으로 모두 쓸어 담은 혁명**과 완벽히 똑같습니다. 옛날 음악가(전통적 전산실)는 드럼 치는 사람(서버 SDC), 기타 치는 사람(네트워크 SDN), 녹음 테이프 릴 기계(스토리지 SDS) 등 거대하고 무거운 실제 물리적 쇳덩어리 장비들을 방 안에 가득 채워놓고 선을 이리저리 꽂아대며 생고생을 해서 녹음했습니다(사일로 장비의 고통). 드럼 소리를 키우려면 드러머한테 뛰어가서 "크게 쳐!"라고 해야 했습니다. **SDDC 스튜디오**는 이 무거운 아날로그 장비들을 모조리 엿장수에게 팔아버립니다. 오직 거대한 고성능 깡통 PC 딱 1대만 책상에 덜렁 둡니다. 그리고 화면에 **'로직 프로(Logic Pro)' 같은 만능 작곡 소프트웨어 화면(SDDC 관리 도구)**을 띄웁니다. 작곡가(클라우드 엔지니어)는 마우스 클릭 1번으로 가상 드럼(가상 머신)을 화면에 띄우고, 마우스 드래그로 가상 기타에 리버브 이펙터(방화벽)를 연결하며, 마우스 휠을 돌려 가상 하드디스크 용량(스토리지)을 무한대로 펌핑합니다. 무겁고 먼지 나는 진짜 쇳덩어리 악기(하드웨어)들을 모조리 0과 1의 소프트웨어 클릭질 영역으로 압축 귀속시켜, 마우스와 키보드 하나로 거대 오케스트라 1만 명을 손가락 하나로 통제하는 현대 클라우드 인프라의 최종 완성체입니다.
