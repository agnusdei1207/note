# 197. K8s 컴포넌트 - 마스터 노드 컨트롤 플레인 워커 노드 API Server etcd Kubelet Kube-proxy 도커 쿠버네티스 아키텍처

> **핵심 인사이트**: (196번 쿠버네티스의 내장 해부) 쿠버네티스(K8s)가 컨테이너 10만 개를 부활시키고 자동 조종하는 AI 지휘자라고 했다. 근데 이 지휘자의 머리통을 빠개보면 뇌가 하나가 아니다. "야! K8s는 거대한 하나의 군대야!! 뒤에서 뒷짐 지고 에어컨 바람 쐬면서 지시만 툭툭 내리는 절대 권력의 사령부 '마스터 노드(뇌)'가 있고, 마스터 노드의 명령을 받아 현장에서 총알 받받으며 진흙탕 구르면서 컨테이너 박스를 실제로 띄우고 부수는 최전방 노가다꾼 '워커 노드(손발)' 수천 대가 끝없이 펼쳐져 있는 거야!! 마스터 노드(뇌) 안에도 명령을 받는 입(API Server), 기억을 저장하는 뇌세포(etcd), 노가다꾼을 배정하는 스케줄러가 정교하게 돌아가며 이 미친 거대 군단을 하나의 톱니바퀴처럼 굴려버리는 거라고!!" 뇌와 손발이 철저하게 분리된 마이크로서비스 제국의 지휘 통제소, K8s 컴포넌트 아키텍처다.

## Ⅰ. 쿠버네티스의 양분된 세계 (Control Plane vs Worker Node)
- 컴퓨터 1대에서 도커를 돌릴 땐 뇌와 손발이 짬뽕되어 있었습니다.
- K8s 클러스터(군대)는 서버 기계가 수백 대입니다. 
- 이 중 3~5대는 오직 **'뇌(마스터 노드)'** 역할만 담당하며 절대로 실제 컨테이너를 돌리는 잡일을 하지 않습니다. 나머지 995대의 기계는 오직 명령만 따르는 **'손발(워커 노드)'**이 되어 노가다 짐을 집어던집니다.

## Ⅱ. 마스터 노드 (Master Node / Control Plane) 🌟 사령부 🌟
군대를 지휘하는 브레인입니다. 4대 핵심 내장 기관으로 돌아갑니다. (외우세요!)

### 1. kube-apiserver (대뇌 피질 & 입구)
- K8s 세계의 **모든 통신이 무조건 거쳐 가야 하는 절대적인 중앙 통제 입구**입니다.
- 개발자가 "결제 텐트 3개 쳐줘!"라는 YAML 문서를 쏘면, 무조건 이 API Server의 입으로 꽂힙니다. 또한 워커 노드들이 "대장님! 1번 텐트 불탔습니다!" 보고하는 것도 무조건 이놈의 귀로 들어옵니다.

### 2. etcd (해마 / 기억 저장소) 🌟 핵심 🌟
- K8s의 **유일한 데이터베이스(DB)**입니다. Key-Value 형태의 초고속 뇌세포입니다.
- 클러스터의 모든 상태 정보("결제 텐트 3개 유지할 것", "현재 노드 1번 살아있음")가 여기에 100% 영구 기록(백업)됩니다. 만약 이 `etcd`가 날아가면 쿠버네티스 전체가 기억상실증에 걸려 뇌사 판정을 받습니다.

### 3. kube-scheduler (스케줄러 / 배정관)
- API Server가 "야, 새로운 텐트(Pod) 하나 쳐!"라고 소리칩니다.
- 스케줄러가 현미경으로 995대의 워커 노드(서버)들을 쫙 스캔합니다. "어? 1번 서버는 CPU 꽉 찼네 패스. 오 45번 서버 널널하네! **새 텐트는 45번 서버에 박아라!!**" 하고 최적의 자리를 기가 막히게 찍어주는(스케줄링) 놈입니다.

### 4. kube-controller-manager (컨트롤러 / 감시자)
- 끝없는 잔소리꾼입니다. "결제 텐트 3개 유지"라는 명령이 etcd에 적혀있습니다.
- 24시간 내내 현실 세계의 워커 노드들을 감시하며, "야! 왜 결제 텐트 2개밖에 없어?! 당장 1개 더 쳐서 3개 맞춰!!" 라며 **이상(Desired)과 현실(Current)의 갭을 끝없이 맞춰내는(자가 치유) K8s의 진짜 심장**입니다.

## Ⅲ. 워커 노드 (Worker Node) 🌟 최전방 노가다꾼 🌟
마스터의 명령을 받아 실제로 도커 컨테이너를 띄우는 기계들입니다.

### 1. Kubelet (큐블렛 / 현장 반장) 🌟 핵심 🌟
- 각 워커 노드 기계마다 1명씩 무조건 설치되어 있는 마스터의 스파이(에이전트)입니다.
- 마스터 노드의 `API Server`와 통신하며 "야, 텐트 1개 띄워!" 명령을 받습니다. 명령을 받으면 밑의 도커(컨테이너 런타임) 멱살을 잡고 "야 도커! 빨리 컨테이너 띄워!"라고 채찍질하며 현장을 감독하고 상태를 마스터에게 보고합니다.

### 2. Kube-proxy (큐브 프록시 / 네트워크 교통경찰)
- 각 노드마다 설치되어, 수만 개의 텐트(Pod)들로 들어오고 나가는 네트워크 트래픽(IP 주소와 포트)을 귀신같이 라우팅해주고 쪼개주는(로드밸런싱) 트래픽 경찰입니다.

### 3. 컨테이너 런타임 (Container Runtime / 노가다 인부)
- 큐블렛(반장)의 채찍을 맞고 **진짜로 철제 박스(컨테이너)를 띄우고 삭제하는 맨바닥의 런타임 프로그램**입니다. (옛날엔 Docker가 지배했으나, 지금은 가벼운 containerd 나 CRI-O가 표준 노가다꾼으로 쓰입니다.)

> 📢 **섹션 요약 비유**: **K8s 컴포넌트 아키텍처**는 거대한 **'초대형 피라미드 건설 현장'**의 완벽한 분업 시스템입니다. 시원한 에어컨이 나오는 **마스터 노드(사령부 막사)** 안에는 4명의 간부가 있습니다. 건축가가 도면(YAML)을 접수처인 **API Server(소통 창구)**에 던집니다. 서기는 그 도면을 **etcd(금고장부)**에 소중히 보관합니다. **스케줄러(배치 반장)**가 망원경으로 현장을 보다가 "저기 45구역 놈들이 놀고 있네! 저기에 돌덩어리 올려!"라고 자리를 찍어줍니다. 그러면 **컨트롤러(감찰관)**가 24시간 내내 망원경을 보면서 "야! 도면에 피라미드 3층이어야 하는데 왜 2층이야! 빨리 올려!"라고 무한 감시(자가 치유)를 때립니다. 이 사령부의 명령은 무전기를 타고 뜨거운 사막 현장의 **워커 노드(노동자 구역)**로 날아갑니다. 각 구역마다 심어진 **Kubelet(현장 십장)**이 무전을 받고 몽둥이를 들고 뛰어다니며 **컨테이너 런타임(노가다 인부)**들을 때려 실제로 돌덩어리(컨테이너)를 띄우게 만듭니다. **Kube-proxy(교통경찰)**는 자재를 실은 트럭들이 서로 부딪히지 않게 교통정리를 해줍니다. 뇌(마스터)가 무너지면 군대 전체가 마비되고, 손발(워커)이 부족하면 건물이 안 올라가는, 뇌와 손발이 극단적으로 분리되어 통제되는 기적의 클라우드 식민지 지휘 체계입니다.
