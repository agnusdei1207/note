# 215. SDN (Software Defined Networking) - 제어 평면 데이터 평면 분리 중앙 집중식 네트워크 컨트롤러 클라우드 오픈플로우(OpenFlow)

> **핵심 인사이트**: (214번 SDDC의 네트워크편, 라우터들의 반란) "야! 전국에 깔려있는 인터넷 공유기랑 라우터(네트워크 쇳덩어리) 1만 대를 봐라! 길을 뚫어주려면 관리자가 1번 기계 찾아가서 명령 100줄 치고, 2번 기계 찾아가서 명령 100줄 치고 이 미친 노가다를 1만 번을 해야 해!! 왜 이딴 개고생을 하냐고?! **기계(라우터) 1만 대의 대가리를 빠개봐!! 머릿속에 '어느 길로 보내지 고민하는 뇌(제어부)'랑 '진짜 패킷을 던져주는 손발(데이터부)'이 한 몸통에 무식하게 섞여 있잖아!! 당장 이 1만 대 기계의 뇌(제어부)를 다 도끼로 적출해 뜯어내서 쓰레기통에 버려!! 그리고 기계들은 뇌가 없는 멍청한 '단순 패킷 셔틀(데이터부)'로 만들어!! 그 뜯어낸 1만 개의 뇌 역할을, 하늘 꼭대기에 띄워놓은 거대한 '중앙 통제 인공지능 서버(SDN 컨트롤러)' 딱 1개에다가 모조리 합쳐 넣어!! 그럼 넌 중앙 서버에서 마우스로 선 한 번 쭉 긋기만 하면, 밑바닥 뇌 없는 기계 1만 대가 알아서 동시에 길을 뚫고 데이터를 던지는 기적의 군단이 완성되잖아!!**" 똑똑했던 쇳덩어리 라우터들을 뇌사 상태로 만들어 중앙 1인 독재(소프트웨어)로 찍어 누르는 궁극의 교통 통제망, SDN이다.

## Ⅰ. 분산된 뇌(라우터)들의 끔찍한 비효율
- 기존 네트워크 장비(Cisco 라우터 등)는 하나하나가 독립된 똑똑한 개체입니다. 
- 장비 안에 **길을 뚫을 규칙을 고민하는 뇌(제어부, Control Plane)**와, **그 규칙대로 진짜 편지(패킷)를 옆으로 휙 던져주는 손발(데이터부, Data Plane)**이 찰싹 결합되어 한 쇳덩어리 기계 안에 박혀있었습니다.
- **재앙**: 사장님이 "보안 룰 바꼈어! 특정 게임 사이트 다 막아!" 지시하면, 관리자는 전국에 깔린 이 똑똑한 쇳덩어리 라우터 1,000대에 일일이 터미널로 접속(SSH)해서 타자로 룰을 1,000번 반복해서 쳐넣어야 합니다. (중앙 통제 불가능)

## Ⅱ. SDN (Software Defined Networking)의 개념 🌟
- **개념**: 기존 네트워크 장비에서 **'길을 찾는 뇌(Control Plane 제어 평면)'를 강제로 뜯어내어 쇳덩어리로부터 완벽히 분리**시킨 후, 이 뇌들을 논리적, 중앙 집중식으로 통치하는 거대한 **'소프트웨어 컨트롤러(SDN Controller 서버)' 하나에 몽땅 합쳐버려(중앙 집중화), 관리자가 중앙 컨트롤러 화면에서 클릭 한 번으로 수만 대의 멍청해진 하단 네트워크 스위치(Data Plane)들에게 동시에 명령을 내리고 길을 조종할 수 있게 만든 혁명적 네트워크 아키텍처**입니다.

## Ⅲ. SDN을 굴리는 3계층과 마법의 API 🌟 핵심 기출 🌟
뇌와 손발이 도끼로 두 동강 났으니, 둘 사이를 이어줄 무선 통신(API) 룰이 무조건 필요합니다!

### 1. 인프라 계층 (Infrastructure Layer) - "멍청해진 손발 장비들"
- 현장에 깔린 스위치, 라우터 쇳덩어리들입니다. 얘네들은 이제 길을 찾는 지능(뇌)이 1도 없습니다.
- 위에서 사령관이 내려준 쪽지(포워딩 테이블)만 멍때리며 보고, 편지가 오면 "아, 이쪽 구멍으로 던지라고 하셨지" 라며 **순수하게 패킷을 던져주는 노동(Data Plane)만 미친 속도로 수행**합니다.

### 2. 제어 계층 (Control Layer) - "절대 권력의 뇌 (SDN 컨트롤러)"
- 쇳덩어리들 위에 떠 있는 단 하나의 거대 소프트웨어 지휘 센터입니다. 이놈 혼자 전국의 트래픽 지도를 다 내려다보고 최적의 길을 계산합니다.
- **★ 사우스바운드 API (Southbound API) 🌟 가장 중요 🌟**:
  - 이 '뇌(컨트롤러)'가 밑바닥의 '멍청한 쇳덩어리(스위치)'들을 밧줄로 조종하며 "야! 이런 패킷 오면 3번 구멍으로 버려!"라고 룰을 심어줄 때 쓰는 **밑으로(South) 내려가는 통신 언어 규약**입니다.
  - 이 통신 언어의 전 세계 1등 표준 프로토콜 이름이 바로 시험 단골인 **`오픈플로우(OpenFlow)`** 입니다.

### 3. 애플리케이션 계층 (Application Layer) - "개발자의 마우스 클릭"
- 컨트롤러 꼭대기에 붙어있는 화면(앱)입니다. 개발자가 방화벽, 로드밸런싱 같은 프로그램을 짜서 올립니다.
- **노스바운드 API (Northbound API)**:
  - 위쪽의 개발자 프로그램(앱)이 "방화벽 룰 바꿔줘!" 라며 중간의 '뇌(컨트롤러)'에게 찌를 때 쓰는 **위로(North) 올라가는 통신 언어**입니다. (주로 편한 REST API를 씁니다.)

## Ⅳ. 왜 클라우드의 심장인가?
- 214번 SDDC 데이터센터를 구축하려면, 고객이 "서버 10대, 가상 방화벽 1개 세팅 완료!" 버튼을 누를 때 0.1초 만에 논리적인 네트워크 도로망이 쫙 깔려야 합니다. 
- SDN이 쇳덩어리의 뇌를 장악하고 중앙 통제를 하지 않는다면, 사람이 뛰어다니며 랜선을 꽂아야 해서 이 1초 컷 클라우드 가상화 세팅은 아예 물리학적으로 달성이 불가능합니다.

> 📢 **섹션 요약 비유**: **SDN(소프트웨어 정의 네트워킹)**은 복잡한 10만 대의 교차로 신호등을 조종하는 **'지능 적출과 중앙 AI 관제탑의 1인 독재 마술'**입니다. 옛날 바보 같은 네트워크 교통망(기존 라우터)은, 전국 10만 개의 교차로 신호등 안에 각각 **'어느 차를 보낼지 판단하는 교통경찰(뇌, 제어 평면 Control Plane)'**과 **'빨간 불을 켜는 전구(손발, 데이터 평면 Data Plane)'**가 한 세트로 박혀있었습니다. 사고가 나서 1번 교차로를 막으려면 경찰청장이 1번 교차로까지 차 타고 달려가 그 안의 경찰에게 "야! 수동으로 불 꺼!"라고 일일이 명령해야 했습니다(개별 수동 조작). SDN은 도끼를 가져와 10만 개 신호등 안에 있던 교통경찰(뇌)들의 목을 다 잘라서 쓰레기통에 버립니다. 이제 신호등은 아무 생각 없이 불만 깜빡이는 **'멍청한 전구 깡통(데이터 평면)'**이 되었습니다. 그리고 하늘 꼭대기 경찰청 상황실에 전국의 모든 신호등을 한눈에 내려다보는 **'슈퍼컴퓨터 AI 관제탑(SDN 컨트롤러, 제어 평면)'** 딱 1대를 세웁니다. 이 AI는 밑바닥의 10만 개 멍청한 전구들과 거대한 광케이블(사우스바운드 API, OpenFlow)로 다이렉트로 연결되어 있습니다. 경찰청장이 "강남구 전체 3분간 빨간불!" 이라고 버튼을 1번 누르는 찰나의 순간, AI가 광케이블에 전기를 쏴 강남구 수만 개의 신호등이 동시에 빨간색으로 찰칵 잠깁니다. 하드웨어 장비의 지능을 뺏어 뇌사 상태로 만들고, 모든 판단력을 중앙의 거대 소프트웨어 뇌 하나로 집중시켜 마우스 클릭만으로 전 세계의 도로망(네트워크 트래픽)을 0.1초 만에 뜯어고치고 방어하는 클라우드 인프라 최고의 교통지휘 아키텍처입니다.
