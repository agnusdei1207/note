# 247. 오픈 API (Open API) 및 API 게이트웨이 보안 스로틀링

> ⚠️ 이 문서는 구글 지도, 네이버 날씨처럼 우리 회사의 귀중한 자산(데이터)을 전 세계 개발자들이 가져다 쓸 수 있도록 대문에 구멍을 뚫어주는 **'오픈 API' 생태계와, 이 구멍으로 악성 해커나 미친 봇(Bot)이 1초에 100만 번의 트래픽을 융단폭격하여 우리 회사 서버를 뻗게 만드는 디도스(DDoS)급 테러를 막기 위해 대문 앞을 지키고 서서 초당 호출 횟수를 통제하고 목을 조르는 'API 게이트웨이와 스로틀링(Throttling)' 기술**을 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 오픈 API는 회사의 서버 문을 활짝 열어주는 행위다. 누구나 들어와서 데이터를 퍼갈 수 있지만, 아무런 문지기 없이 열어두면 트래픽 과부하로 본진 서버가 1시간 만에 불타버린다.
> 2. **가치**: API Gateway라는 막강한 톨게이트를 앞에 세워둠으로써, 인증(API Key), 초당 요청 수 제한(Rate Limiting), IP 차단 등을 중앙에서 한 방에 처리하여 뒤에 숨어있는 나약한 백엔드 서버(WAS)들을 완벽하게 보호한다.
> 3. **기술 체계**: 가장 중요한 무기는 **스로틀링(Throttling, 목 조르기)**이다. "1명의 개발자(API Key)는 1초에 딱 10번까지만 호출할 수 있다. 11번째부터는 `429 Too Many Requests` 에러를 뿜으며 쫓아내라!"라고 기계적으로 트래픽의 목줄을 죈다.

---

### Ⅰ. 오픈 API의 양날의 검: 플랫폼의 확장과 폭발
대문을 열면 손님도 오지만, 폭주족도 오토바이를 몰고 들어온다.

1. **오픈 API 생태계의 힘**:
   - 토스(Toss)나 배달의민족 같은 거대한 앱들은 자기들이 100% 모든 코드를 다 짜지 않는다. 지도는 카카오맵 오픈 API를 끌어오고, 결제는 토스 페이먼츠 API를 끌어와서 레고 블록처럼 뚝딱 조립(Mash-up)한다.
   - 기업은 자기 데이터를 오픈 API로 열어줌으로써(API Economy), 수만 명의 서드파티 개발자들이 알아서 우리 회사의 생태계를 확장해 주는 기적을 맛본다.
2. **트래픽 폭주의 재앙**:
   - 기상청이 '날씨 조회 오픈 API'를 열었다. 어떤 초보 대학생 개발자가 자기 날씨 앱에 버그를 내서, `while(true)` 무한 루프를 돌며 기상청 서버를 1초에 1만 번씩 찔러대기 시작했다.
   - 순식간에 기상청 백엔드 서버(DB)의 CPU가 100%를 치고 사망했다. 진짜 중요한 대국민 기상 특보 사이트마저 같이 뻗어버렸다. 대문을 열어준 대가로 본진이 털린 것이다.

📢 섹션 요약 비유: 오픈 API는 백화점 창고 뒷문을 열어두고 동네 상인들에게 "와서 물건(데이터) 마음껏 떼어 가세요!"라고 허락한 것입니다. 상인들이 알아서 물건을 팔아주니 좋지만, 어떤 미친 트럭 100대가 한 번에 후진으로 돌진해 와서 1초 만에 창고 물건을 다 싹쓸이해 가려고 하면 백엔드 창고가 완전히 붕괴되어 뼈대도 남지 않는 무방비 상태의 위험성이 도사립니다.

### Ⅱ. API Gateway: 중앙 통제소와 톨게이트
뒤에 있는 백엔드 서버는 멍청하다. 똑똑한 문지기를 앞에 세워라.

1. **게이트웨이의 탄생 (AWS API Gateway, Kong 등)**:
   - 이제 날씨 서버(백엔드) 앞에는 무조건 **API Gateway**라는 철통같은 요금소(톨게이트) 기계를 하나 떡 하니 세워둔다.
   - 외부 개발자들은 진짜 기상청 서버 IP 주소는 평생 알 수 없다. 무조건 이 게이트웨이 주소로만 찔러야 한다.
2. **권한 인가와 과금 (API Key)**:
   - 게이트웨이는 톨게이트 직원이다. 찌르는 사람에게 묻는다. "너 누구야? 신분증(API Key나 Bearer Token) 까봐."
   - 토큰이 유효하면 통과시키면서 엑셀(DB)에 '정상 호출 1회'라고 틱! 줄을 긋는다. 나중에 이 장부를 보고 "너 이번 달에 100만 번 썼으니까 10만 원 내놔"라고 과금(Billing) 시스템과 찰떡같이 연동된다.
3. **오프로딩 (Off-loading)의 미학**:
   - 백엔드 서버(Java, Python) 안에서 토큰이 맞는지 DB를 뒤지고 비밀번호를 검사하는 무거운 짓(Authentication)을 싹 다 지워버렸다. 이 짓은 앞단에 서 있는 게이트웨이가 100% 다 처리하고, 백엔드는 그냥 깨끗한 데이터만 내어주면 되는 완벽한 분업화(오프로딩)가 달성된다.

📢 섹션 요약 비유: 놀이공원(백엔드 서버) 입구에 거대한 톨게이트(API 게이트웨이)를 세운 것입니다. 옛날엔 놀이기구(API)마다 직원이 서서 표를 검사(인증)하느라 놀이기구가 멈췄습니다. 지금은 톨게이트에서 자유이용권(API Key)을 싹 다 검사하고 과금 처리까지 완벽하게 끝낸 다음, 진짜 놀이공원 안쪽으로는 "검증 끝난 안전한 손님"만 쏙 밀어 넣어주어 안쪽 직원들(백엔드)은 오직 기구 돌리는 데만 집중할 수 있게 해주는 만능 수문장입니다.

### Ⅲ. 최후의 방패 스로틀링 (Throttling): 트래픽 목 조르기
"네가 아무리 VIP라도 1초에 10번 이상은 밥을 줄 수 없다."

1. **스로틀링 (Rate Limiting)의 작동 원리**:
   - 게이트웨이의 필살기다. 특정 API Key 단위나 IP 주소 단위로 **'1초당 최대 호출 가능 횟수(TPS/RPS)'**를 칼같이 정해버린다.
   - "A 개발자의 토큰은 1초에 10번(10 RPS)까지만 허용!"
   - 초보 대학생이 버그 낸 앱이 1초에 1만 번을 찔렀다. 게이트웨이는 앞에 들어온 10개의 요청만 백엔드로 스윽 넘겨주고, 나머지 **9,990개의 요청은 백엔드로 보내지도 않고 그 자리에서 즉시 `429 Too Many Requests` 상태 코드를 뺨에 갈겨버리며 튕겨낸다 (Drop).**
2. **버스트(Burst) 트래픽의 허용 (토큰 버킷 알고리즘)**:
   - 무조건 1초에 10번으로 막으면 좀 억울할 수 있다. "저 평소엔 1번도 안 쏘다가 방금 이벤트 터져서 잠깐 30번 쏜 건데요?"
   - 그래서 **토큰 버킷(Token Bucket)** 알고리즘을 쓴다. 평소에 API를 안 쓰면 바구니에 여유 토큰(Burst)을 50개 정도 모아둔다. 그러다 갑자기 30번이 훅 들어오면 "오케이, 너 모아둔 토큰 있으니까 한 번 봐줄게!"라며 일시적인 트래픽 폭주(Spike)는 부드럽게 흡수해 통과시켜 주는 지능형 유연성도 발휘한다.

📢 섹션 요약 비유: 스로틀링은 톨게이트 직원이 차단기 바(Bar)를 쥐고 트래픽의 '목을 조르는(Throttling)' 행위입니다. 아무리 하이패스 잔액(돈)이 많아도 "이 차선은 1초에 1대씩만 지나가라!"라고 세팅해 두면, 미친 폭주족 100명이 덤벼들어도 톨게이트 차단기가 1대만 통과시키고 99대의 앞유리는 다 부숴버립니다(`429 에러`). 덕분에 톨게이트 뒤편 고속도로(백엔드 서버)는 1년 365일 체증 없이 시속 100km로 평온하게 달릴 수 있는 궁극의 평화를 얻게 됩니다.
