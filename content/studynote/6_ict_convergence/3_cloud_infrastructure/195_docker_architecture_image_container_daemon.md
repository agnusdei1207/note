# 195. 도커 (Docker) 아키텍처 - 도커 데몬 도커 이미지 레이어 컨테이너 레지스트리 가상화 클라우드 배포

> **핵심 인사이트**: (194번 컨테이너의 실체화) 옛날엔 내 노트북(Mac)에서 1달 밤새워 완벽하게 돌아가게 짠 자바 코드를 서버(Linux)에 배포하면 100% 에러가 나며 터졌다. "야! 내 컴퓨터에선 분명히 잘 됐다고!!" 빡친 솔로몬 파익스(도커 창시자)가 거대한 항구의 '철제 화물 컨테이너'를 보고 소리쳤다. "야!! 앞으로 네가 짠 소스 코드만 덜렁 서버에 던지지 마!! 네가 개발하던 그 노트북의 자바 버전, 우분투 OS 환경, 라이브러리 세팅까지 모조리 다 긁어모아서 거대한 '철제 박스(Docker Image)' 하나에 시멘트로 꽝꽝 얼려서 찍어내(Build)!! 그리고 이 철제 박스만 화물선에 실어서 아마존 서버든 구글 서버든 던져버려!! 서버에서는 그냥 이 박스를 까기만 하면(Run), 전 세계 어디서든 똑같은 환경이 0.1초 만에 뿅 튀어나와 100% 에러 없이 돌아가잖아!!" "내 자리에선 되는데요?"라는 개발자의 변명을 지구상에서 멸종시켜버린 클라우드 항구의 혁명, 도커다.

## Ⅰ. 의존성 지옥(Dependency Hell)의 종말
- 기존 배포의 끔찍함: 내 PC는 `Java 8`을 쓰고, 서버는 `Java 11`이 깔려있습니다. 배포하면 라이브러리 충돌이 터져 서버가 죽습니다. 서버 환경을 내 PC랑 똑같이 맞추려면 서버 관리자가 리눅스 터미널을 열고 3시간 동안 미친 듯이 타이핑을 해야 합니다.

## Ⅱ. 도커 (Docker)의 개념 🌟
- **개념**: 애플리케이션 소스 코드와 그 코드가 실행되는 데 필요한 모든 런타임, 라이브러리, 환경 설정(OS 껍데기)을 **단 하나의 투명한 '표준화된 패키지 상자(컨테이너 이미지)'로 포장하여, 윈도우든 리눅스든 아마존이든 도커 엔진만 깔려 있다면 어디서든 0.1초 만에 100% 동일하게 실행되도록 보장해 주는 오픈소스 컨테이너 플랫폼**입니다.

## Ⅲ. 도커 제국을 지탱하는 4대 절대 컴포넌트 🌟 핵심 기출 🌟

### 1. 도커 이미지 (Docker Image) - "얼어붙은 붕어빵 틀"
- **개념**: 실행 중인 컨테이너가 아닙니다! 앱을 실행하는 데 필요한 모든 파일과 소스코드가 완벽하게 박제된 **'읽기 전용(Read-Only)'의 불변하는 템플릿(설계도) 파일**입니다.
- **특징 (레이어 구조 🌟)**: 이미지는 통짜가 아닙니다. 밑바닥에 `우분투 OS 레이어`, 그 위에 `Java 설치 레이어`, 그 위에 `내 소스코드 레이어`가 양파 껍질처럼 차곡차곡 쌓여있는 겹겹 구조(Layered Architecture)입니다. 코드 1줄 바꾸고 새로 빌드하면 전체 1GB를 다 새로 굽는 게 아니라, 맨 위층 1MB짜리 코드 레이어만 싹 갈아치우는 기적의 속도 최적화를 이뤄냅니다.

### 2. 도커 컨테이너 (Docker Container) - "찍혀 나온 따끈한 붕어빵"
- **개념**: 죽어있는 박제 파일인 **도커 이미지(틀)**에 생명(CPU, RAM)을 불어넣어 **살아서 펄떡이며 돌아가고 있는 독립된 프로세스(실행 환경)**입니다.
- 이미지 1개로 똑같은 컨테이너를 1,000개든 1만 개든 1초 만에 찍어낼 수 있습니다(194번 초고속 스케일링).

### 3. 도커 데몬 (Docker Daemon, dockerd) - "항구의 총괄 매니저"
- **역할**: 호스트 리눅스 서버 밑바닥 깊은 곳에서 24시간 윙윙 돌아가며 상주하는 백그라운드 서버 프로그램(심장)입니다.
- 개발자가 터미널에 `docker run(실행해!)` 이라고 치면, 이 데몬이 명령을 낚아채서 허공에 텐트(네임스페이스)를 치고 이미지를 끌고 와서 컨테이너를 쾅쾅 구워내는 실제 막노동꾼입니다.

### 4. 도커 레지스트리 (Docker Registry / Docker Hub) - "클라우드 설계도 창고"
- **역할**: 내가 예쁘게 구워낸 '도커 이미지(철제 박스)'를 100MB짜리 압축 파일로 만들어서, 전 세계 누구나 다운받을 수 있게 인터넷 허공에 올려두는 **클라우드 파일 저장소(앱스토어 같은 역할)**입니다.
- 제일 유명한 공용 창고가 **Docker Hub**이며, MySQL, Nginx 등 전 세계의 모든 유명한 프로그램들이 이미 이미지로 박제되어 올라와 있어서 `docker pull mysql` 한 줄만 치면 10초 만에 완벽 세팅된 DB가 내 컴퓨터에 떨어집니다.

## Ⅳ. 왜 클라우드는 도커에 열광하는가?
- **불변 인프라 (Immutable Infrastructure 204번 스포일러)**: 옛날엔 서버 코드가 맘에 안 들면 관리자가 몰래 들어가서 소스코드 한 줄을 수정(Patch)하고 퇴근했습니다. 누가 언제 수정했는지 몰라서 나중에 서버가 꼬여버립니다. 도커는 이 짓을 금지합니다. 컨테이너는 일회용 종이컵입니다. 맘에 안 들면 고쳐 쓰는 게 아니라 **가차 없이 컨테이너를 폭파해(Kill) 버리고, 새로 1줄을 고친 완벽한 새 이미지를 가져와 다시 깨끗한 새 컨테이너로 1초 만에 엎어 쳐서(Replace) 띄웁니다.** 서버 환경이 영원히 더러워지지 않는 완벽한 무균실 배포의 완성입니다.

> 📢 **섹션 요약 비유**: 기존의 배포 환경은 **'유명 셰프가 다른 나라 식당에 요리법(소스 코드)만 종이에 덜렁 적어서 우편으로 보내는 짓'**이었습니다. 프랑스 주방장이 이탈리아 식당으로 레시피를 보냈더니, 이탈리아 식당에는 냄비도 없고 가스레인지 불 온도(운영체제 환경)도 달라서 요리가 다 타버리고 폭망합니다("내 주방에선 잘됐는데?"). 이를 부숴버린 **도커(Docker) 아키텍처**는 셰프가 **자신이 쓰던 가스레인지, 프라이팬, 조미료 세트, 심지어 주방 공기(OS 환경)까지 모조리 긁어모아 '거대한 투명 진공 포장 박스(Docker Image)' 하나에 꽁꽁 얼려버리는 기적**입니다. 셰프는 이 진공 박스를 헬기(Docker Registry)에 띄워 전 세계 아마존이든 구글이든 아무 식당에나 떨굽니다. 이탈리아 식당 주방장(Docker Daemon)은 그냥 박스 껍데기를 칼로 뜯기만(Run) 하면 끝납니다! 진공 박스가 열리자마자 셰프가 쓰던 완벽한 주방 환경이 허공에 1초 만에 똑같이 펄럭하고 펼쳐지며(Container 실행), 어떤 열악한 식당 환경이라도 프랑스 셰프의 주방과 100% 동일한 요리가 에러 없이 뿜어져 나오는 궁극의 밀키트 캡슐 배포 혁명입니다.
