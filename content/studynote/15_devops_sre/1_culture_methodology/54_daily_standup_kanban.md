## 핵심 인사이트 (3줄 요약)
1. **데일리 스탠드업**은 팀원들이 작업 현황을 공유하고 병목 지점을 조기에 발견하여 하루의 계획을 조율하는 15분 이내의 짧은 미팅이다.
2. **칸반 보드**는 업무의 흐름을 시각화하고 진행 중인 작업(WIP)을 제한하여 팀의 처리량을 극대화하는 린(Lean) 매니지먼트 도구이다.
3. 시각화된 보드와 면대면 미팅의 결합은 조직의 투명성을 높이고, 사일로 현상을 방지하는 데브옵스 문화의 핵심 실천 양식이다.

### Ⅰ. 개요 (Context & Background)
- **배경**: 개발 단계에서 누가 무엇을 하는지 모르는 정보 비대칭 문제를 해결하고, 작업의 정체 구간(Bottleneck)을 빠르게 찾아내야 한다.
- **필요성**: 텍스트 기반 보고는 맥락 전달에 한계가 있으므로, 시각적 도구와 짧은 대화를 통해 팀 전체가 동일한 상황 인식을 공유해야 한다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)
- **미팅의 3가지 질문**:
  1. 어제 무엇을 완료했는가?
  2. 오늘 무엇을 할 계획인가?
  3. 앞을 가로막는 장애물(Blocker)은 무엇인가?
- **칸반의 핵심 원리**:
  - **WIP (Work In Progress) Limit**: 동시에 진행하는 작업 개수를 제한하여 리소스 낭비 방지
  - **Pull System**: 자원이 준비되었을 때 다음 작업을 가져오는 흐름 최적화

```text
[Daily Standup & Kanban Interaction]

+-----------------------------------------------------------+
|               Kanban Board (Visualization)                |
|  [ To Do ]     [ In Progress ]      [ Testing ]    [ Done ]|
|  +-------+     +------------+      +---------+    +------+ |
|  | Card1 |     | Card3 (WIP)|      | Card2   |    | Card0| |
|  |       |     |   [Block!] |      |         |    |      | |
|  +-------+     +------------+      +---------+    +------+ |
+-----------------------------------------------------------+
       ^               ||                 ^             ^
       |               \/                 |             |
+-----------------------------------------------------------+
|           Daily Standup (15-min Meeting)                  |
|  - "Card3 is blocked by DB access issues!" (Blocker)      |
|  - "I'll help resolve that today." (Collaboration)        |
+-----------------------------------------------------------+
```

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

| 비교 항목 | 데일리 스탠드업 (Scrum 기반) | 칸반 미팅 (Flow 기반) |
| :--- | :--- | :--- |
| **주요 관점** | 사람(Person) 중심의 현황 공유 | 작업(Work) 중심의 흐름 최적화 |
| **미팅 주기** | 매일 아침 정해진 시간 | 흐름에 이상이 생길 때 수시로 가능 |
| **주요 목표** | 스프린트 목표 달성 여부 확인 | 리드 타임 단축 및 병목 제거 |
| **강점** | 팀워크 및 유대감 형성 | 프로세스 효율성 및 데이터 기반 개선 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)
- **실무 적용**: 지라(Jira)나 트렐로(Trello)를 활용하여 보드를 구성하고, 미팅 시에는 보드를 보며 '오른쪽(완료에 가까운 쪽)부터 왼쪽'으로 진행 상태를 점검하는 것이 효율적이다.
- **기술사적 판단**: 스탠드업은 '상태 보고'가 아니라 '계획 수립'이다. 기술적인 상세 토론은 미팅 직후 관심 있는 사람끼리만 남아서 진행하는 '16번째 분(16th Minute)' 규칙을 적용하여 시간 낭비를 최소화해야 한다.

### Ⅴ. 기대효과 및 결론 (Future & Standard)
- **기대효과**: 문제의 조기 발견(MTTD 단축), 불필요한 이메일/회의 감소, 팀 자율성 강화 및 지속적인 프로세스 개선(Kaizen).
- **결론**: 데일리 스탠드업과 칸반 보드는 현대적인 애자일 인프라의 양대 축이며, 이를 통해 팀은 '바쁜 상태'가 아닌 '가치를 인도하는 상태'로 거듭날 수 있다.

### 📌 관련 개념 맵 (Knowledge Graph)
1. **WIP Limit**: 시스템 내 과부하를 막기 위한 진행 중 작업 수 제한
2. **Blocker/Impediment**: 작업 진행을 방해하는 모든 외부 요인
3. **Cumulative Flow Diagram (CFD)**: 작업 흐름의 안정성을 보여주는 누적 흐름도

### 👶 어린이를 위한 3줄 비유 설명
1. **데일리 스탠드업**: 운동 경기를 시작하기 전, 선수들이 모여서 "너는 수비해, 나는 골을 넣을게!"라고 작전을 짜는 짧은 미팅이에요.
2. **칸반 보드**: 우리 집 냉장고에 붙여둔 메모지판처럼, 해야 할 일과 다 한 일을 한눈에 볼 수 있게 정리한 보드예요.
3. **결론**: 매일 아침 서로의 생각을 맞추고, 보물 지도를 보듯 할 일을 확인하며 즐겁게 일하는 방법이에요.
