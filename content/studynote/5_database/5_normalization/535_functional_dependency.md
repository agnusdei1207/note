# 함수 종속성 (Functional Dependency)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 함수 종속성 (Functional Dependency)은 관계형 데이터베이스에서 한 속성 값이 다른 속성 값을 유일하게 결정하는 관계로, 정규화의理论基础이다.
> 2. **가치**: 함수 종속성을 분석하면 테이블의構造的 문제점 (부분 종속, 이행적 종속 등)을 발견하고, 적절한 분해를 통해 정규형을 적용할 수 있다.
> 3. **융합**: Armstrong의 공리 ( réflexivité, augmentation, transitivité 등)를 利用하여 함수 종속성의閉包 (Closure)를 계산하고, 최소 덮개 (Minimal Cover)를 구하는 알고리즘이 정규화 설계에 활용된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념 정의

함수 종속성 (Functional Dependency)은 **관계 R에서 속성 X의 값이 속성 Y의 값을 유일하게 결정할 때, "X → Y"로 표기하는 관계**다.

### 정의

```
┌─────────────────────────────────────────────────────────────────────┐
│                    함수 종속성 (Functional Dependency) 정의                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   관계 R의 두 투플 t1, t2에 대해,                                    │
│   만약 t1[X] = t2[X]이면, t1[Y] = t2[Y]이다.                       │
│   이를 "X가 Y를 함수적으로 결정한다" 또는 "Y가 X에 종속된다"라 한다.   │
│                                                                     │
│   표기: X → Y                                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 비유

함수 종속성은 **학번과 학생 이름의 관계**와 같다. 학번이 결정되면 학생 이름이 유일하게 결정된다. 따라서 "학번 → 학생 이름"이라는 함수 종속성이 성립한다.

- **📢 섹션 요약 비유:**학번을 알면 이름을 알 수 있는 것처럼 (학번 → 이름), 컴퓨터에서도 특정 값이 다른 값을決定する関係를 분석하면 올바른 테이블 설계를 할 수 있어요.

## Ⅱ. 함수 종속성의 유형

### 1. 완전 함수 종속성 (Full FD)

X → Y에서 X의 어떤 진부분集合真부분集合叫做부분集合도 Y를 결정하지 않는 경우

```
┌─────────────────────────────────────────────────────────────────────┐
│                    완전 함수 종속성 예시                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   [주문항목 테이블]                                                 │
│   기본 키: (주문ID, 상품ID)                                        │
│                                                                     │
│   (주문ID, 상품ID) → 수량  ← 완전 종속 (复合键 전체에 종속)         │
│   (주문ID, 상품ID) → 상품명  ← 부분 종속! (상품ID에만 종속)        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2. 이행 함수 종속성 (Transitive FD)

X → Y이고 Y → Z이면, X → Z (이행적 종속)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    이행적 함수 종속성 예시                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   고객 → 도시이고, 도시 → 도시코드이면,                               │
│   고객 → 도시코드 (이행적 종속)                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **결정자** | 다른 속성을 결정하는 속성으로, X → Y에서 X가 결정자다. |
| **종속자** | 결정되는 속성으로, X → Y에서 Y가 종속자다. |
| **부분 종속성** | 복합 기본 키의 일부에만 종속되는 것으로, 2NF에서 제거한다. |
| **이행적 종속성** | 기본 키가 아닌 속성을 경유하는 간접 종속으로, 3NF에서 제거한다. |

## 👶 어린이를 위한 3줄 비유 설명

1. 함수 종속성은 **학번과 앉은 자리 관계**와 같아요. 학번이 1번이면 반드시 1번 책상에 앉는다면, "학번 → 앉은 자리"라는 관계가 있어요.
2. 하지만 반이 정해지면 반 안에서 번호를 정하는 경우, "반 + 번호"가 이름을 결정하지만, 반만으로는 이름을 모르는 것은 부분 종속이에요.
3. 컴퓨터에서도 이러한 관계를 분석하면 데이터를 더 잘 정리할 수 있어요!
