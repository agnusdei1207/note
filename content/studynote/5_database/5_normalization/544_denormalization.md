# 반정규화 (Denormalization)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 반정규화 (Denormalization)는 성능 향상을 위해 의도적으로 정규화된 테이블에 데이터 중복을 허용하는 설계 기법이다.
> 2. **가치**: 복잡한 JOIN으로 인한 성능 저하를 해소하고, 읽기 성능을 향상시키지만, 갱신 이상의 위험이 증가한다.
> 3. **융합**: 데이터 웨어하우스, 읽기 집중형 시스템, Reporting 시스템에서 널리 활용되며, 정규화와 트레이드오프를 통한 실용적 설계를実現한다.

---

## Ⅰ. 개요 및 필요성

### 개념 정의

반정규화 (Denormalization)는 **정규화된 테이블 구조를 의도적으로 깨뜨려, 데이터 중복을容忍하여 조회 성능을 높이는 기법**이다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    반정규화 예시                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   [정규화 상태]                                                      │
│   customers: (고객ID, 이름, 도시)                                    │
│   orders: (주문ID, 고객ID, 금액)                                    │
│   → 조인 필요                                                         │
│                                                                     │
│   [반정규화 상태]                                                    │
│   order_summary: (고객ID, 고객명, 총주문금액)                       │
│   → 조인 불필요, 빠른 조회                                           │
│   → 데이터 중복: 고객명이 orders에 중복 저장                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 비유

반정규화는 **자주使う 파일을 책상 위에 펼쳐두는 것**과 같다. 서랍에서 찾지 않고 바로 쓸 수 있어 빠르지만, 책상이 복잡해지고 여러 곳을 更新해야 할 수 있다.

- **📢 섹션 요약 비유:**요리사가 요리할 때 소금, 후추 등을 각각 서랍에서 찾지 않고 한쪽에 펼쳐두면 빨리 요리할 수 있어요. 하지만 소금을 다른 것으로 바꾸면 여러 곳을 모두 바꿔야 해요.
