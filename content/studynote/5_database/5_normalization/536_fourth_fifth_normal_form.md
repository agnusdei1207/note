# 제4정규형과 제5정규형 (4NF, 5NF)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 4NF는 다중값 종속성 (Multi-valued Dependency)을 제거하고, 5NF는 결합 종속성 (Join Dependency)을 제거하여 궁극적 정규화를 달성한다.
> 2. **가치**: 4NF와 5NF는 특수한情况에서만 필요하며, 대부분의 실무에서는 3NF 또는 BCNF 수준으로 충분하다.
> 3. **융합**: 4NF/5NF는複雑な業務 관계를정확히 모델링해야 하는 시스템 (예: 다양 한 商品属性 관리)에서 활용된다.

---

## Ⅰ. 제4정규형 (4NF)

### 다중값 종속성 (Multi-valued Dependency)

A →→ B는 "A가 결정되면 B는 독립적으로 결정된다"는 관계다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    다중값 종속성 예시                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   [고객-취미-연락처 테이블] (4NF 미충족)                               │
│   ┌────────┬────────┬────────┐                                   │
│   │ 고객ID  │ 취미    │ 연락처   │ ← 고객ID가 취미와 연락처 모두 결정    │
│   ├────────┼────────┼────────┤                                   │
│   │ C001   │ 등산    │ 010-1  │                                   │
│   │ C001   │ 등산    │ 010-2  │ ← 중복!                           │
│   │ C001   │ 독서    │ 010-1  │ ← 중복!                           │
│   │ C001   │ 독서    │ 010-2  │ ← 중복!                           │
│   └────────┴────────┴────────┘                                   │
│                                                                     │
│   문제: 고객ID가 취미와 연락처를 동시에 결정하지만,                     │
│         취미와 연락처 사이에는 관계가 없음 → 독립적                     │
│         → 데이터 중복 발생                                             │
│                                                                     │
│   4NF 분해:                                                         │
│   • 고객_취미 (고객ID, 취미)                                        │
│   • 고객_연락처 (고객ID, 연락처)                                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 제5정규형 (5NF)

### 결합 종속성 (Join Dependency)

R이 (R1, R2, ..., Rn)의 자연조인의 결과여야 하는 관계다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    결합 종속성 예시                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   [주문-상품-공급자 테이블] (5NF 미충족)                               │
│   ┌────────┬────────┬──────────┐                               │
│   │ 주문ID  │ 商品ID │ 공급자ID │                               │
│   ├────────┼────────┼──────────┤                               │
│   │ O001   │ P001   │ S001     │                               │
│   │ O001   │ P002   │ S002     │                               │
│   │ O002   │ P001   │ S001     │                               │
│   │ O002   │ P002   │ S003     │ ← 조합의 의미:                 │
│   │ O003   │ P001   │ S001     │ "O001은 P001,P002를 주문"      │
│   │ O003   │ P002   │ S002     │ "P001은 S001에게 공급"         │
│   └────────┴────────┴──────────┘ "P002는 S001,S002에게 공급"     │
│                                                                     │
│   문제: 세 테이블로 분해했다가 다시 조인하면 중복 tuples 발생!           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **다중값 종속성** | A →→ B로 표기되며, A가 결정되면 B가 독립적으로 결정된다. |
| **결합 종속성** | 테이블이 여러 투플의 조인으로만 표현되는 종속성이다. |

| **5NF** | 결합 종속성을 제거한 궁극적 정규형이다. |

## 👶 어린이를 위한 3줄 비유 설명

1. 4NF는 **학생이 여러 취미와 여러 연락처를 가질 때**, 취미와 연락처를 함께 두면 중복이 생기는 것처럼, 이를 분리하는 거예요.
2. 5NF는 **주문, 상품, 공급자**처럼 세 가지가관련될 때, 조합의 의미가 중요해서 더 세분화해야 하는 거예요.
3. 하지만 보통은 3NF까지만으로 충분해서, 4NF/5NF는特殊情况에서만 필요해요!
