# 278. 컬럼 패밀리 저장소 (Column Family / Wide-Column Store)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컬럼 패밀리 저장소는 와이드 컬럼(Wide Column) 모델을 사용하여, 각 행이 다른 수의 컬럼을 가질 수 있으며, 대량 읽기/쓰기 및 시계열 데이터 처리에 최적화된 NoSQL 모델이다.
> 2. **가치**: 대용량 쓰기 처리, 압축 효율, 시계열/로그 데이터 특화, Cassandra Query Language (CQL)를 통한 SQL 유사 인터페이스 등의 장점이 있다.
> 3. **융합**: Cassandra, HBase, 시계열 데이터, LSM-Tree, 가용성 우선(AP) 시스템과 밀접하게 연관된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념 정의
컬럼 패밀리 저장소(Column Family Store)는 와이드 컬럼(Wide Column) 모델을 사용하는 NoSQL 데이터베이스이다. 각 행은 행 키(Row Key)로 식별되며, 각 행은 다양한 컬럼(Column)을 가질 수 있다. 컬럼은 패밀리(Column Family)로 그룹화된다.

### 필요성
시계열 데이터, 로그 데이터, IoT 센서 데이터 등은 대량의 쓰기와 시간 기반 범위 查询가 필요하다. 컬럼 패밀리 저장소는 이러한 작업에 최적화된 저장 구조와 압축 기능을 제공한다.

### 비유
컬럼 패밀리 저장소는시간순으로 기록되는장부와 같다. 각 페이지(행)에 날짜별로 거래 내역을 기록하며, 같은 날의 거래가 같은 페이지에 몰려 있어 시간대별 확인이 쉽다. 새 거래가 발생하면 마지막 페이지에 추가한다.

📢 섹션 요약: 컬럼 패밀리 저장소는 와이드 컬럼 모델로 대량 쓰기와 시계열 데이터에 최적화된 NoSQL 모델이다.

### Cassandra vs HBase 비교

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Cassandra vs HBase 비교                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌────────────────┬────────────────────────┬────────────────────────┐      │
│  │     항목        │       Cassandra        │        HBase          │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 아키텍처       │ 피어 투피어 (P2P)     │ 마스터-슬레이브       │      │
│  │              │ (모든 노드가 동일 역할)  │ (Master coordination)  │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 쓰기 경로     │ CommitLog → Memtable  │ WAL → MemStore →      │      │
│  │              │ → SSTable             │  → HFile              │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 읽기 경로     │ Bloom Filter →        │ Row Cache →          │      │
│  │              │ SSTable merge         │ MemStore → HFile      │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 쿼리 언어     │ CQL (SQL 유사)        │ 스캔 (Scan)          │      │
│  │              │                        │ (범위 查询)           │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 일관성 수준   │ Tunable (ANY ~ ALL)   │ 강철 일관성 (기본)    │      │
│  │              │ (설정 가능)            │                      │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ CAP 분류      │ AP (가용성 우선)      │ CP (일관성 우선)      │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 적합 용도      │ 글로벌 분산, 쓰기     │ Hadoop 생태계,       │      │
│  │              │ 집중, 다중数据中心     │ 대량 배치 처리        │      │
│  └────────────────┴────────────────────────┴────────────────────────┘      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]**
Cassandra와 HBase는 대표적인 컬럼 패밀리 저장소이지만 아키텍처가 다르다.
Cassandra는 완전한 P2P 아키텍처로 모든 노드가 동일 역할을 하며, Tunable
Consistency를 지원한다. HBase는 마스터-슬레이브 구조로 Hadoop 생태계와
紧密结合되어 있으며 강철 일관성을 기본으로 한다.

📢 섹션 요약: Cassandra와 HBase는 대표적인 컬럼 패밀리 저장소로, 각각 P2P와 마스터-슬레이브 아키텍처, tunable consistency와 강철 일관성 등 다른 특성을 가진다.

---

## Ⅲ. 결론

컬럼 패밀리 저장소는 대량 쓰기, 시계열 데이터, 로그 수집 등에 최적화된 NoSQL 모델이다. Cassandra는 글로벌 분산과 쓰기 집중 시나리오에, HBase는 Hadoop 생태계와 배치 처리 시나리오에 적합하다.

📢 섹션 요약: 컬럼 패밀리 저장소는 대량 쓰기/시계열 데이터에 특화된 NoSQL 모델로, Cassandra와 HBase가 대표적이고 각각 다른 아키텍처를 가진다.

## 참고
- 컬럼 패밀리 저장소는 와이드 컬럼 모델을 사용한다.
- 대량 쓰기, 시계열/로그 데이터에 최적화되어 있다.
- Cassandra와 HBase가 대표적이다.
- Cassandra는 P2P 아키텍처, HBase는 마스터-슬레이브이다.
