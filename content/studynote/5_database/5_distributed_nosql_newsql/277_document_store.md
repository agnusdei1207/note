# 277. 문서 저장소 (Document Store)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 문서 저장소는 JSON, XML 형태의 문서를 기본 단위로 저장하며, 문서 내부 필드에 대한 쿼리와 색인이 가능하여 반정형 데이터 처리에 최적화된 NoSQL 모델이다.
> 2. **가치**: 유연한 스키마, 복잡한 데이터 구조 표현, 개발 생산성 향상, Restful API와의 자연스러운 통합 등의 장점이 있다.
> 3. **융합**: MongoDB, CouchDB, BSON, JSON, 스키마 온 리드, 분산 쿼리와 밀접하게 연관된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념 정의
문서 저장소(Document Store)는 JSON, XML, BSON 형태의 문서를 기본 저장 단위로 사용하는 NoSQL 데이터 모델이다. 각 문서는 자체적인 구조를 가질 수 있어 유연한 스키마를 지원하며, 문서 내부 필드에 대한 쿼리와 색인이 가능하다.

### 필요성
반정형 데이터는 고정된 스키마를 가지지 않아 관계형 데이터베이스로 저장하기 어렵다. 문서 저장소는 이러한 반정형 데이터를 자연스럽게 표현하고, 문서 단위로 저장하여 개발 생산성을 높인다.

### 비유
문서 저장소는여러 서랍을 가진文件 캐비닛과 같다. 각 서랍에는 다양한 크기와 형태의 문서를 넣을 수 있으며, 필요할 때 서랍 내에서 색인을 통해 빠르게 찾을 수 있다.固定された形式ではなく、自由な形式で情報を保存できる.

📢 섹션 요약: 문서 저장소는 JSON/XML 문서를 기본 단위로 저장하며, 유연한 스키마와 필드 쿼리가 가능한 NoSQL 모델이다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 문서 저장소 동작 구조

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    문서 저장소 동작 구조                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [문서 구조]                                                               │
│                                                                             │
│  {                                                                       │
│    "_id": "1",                      ← 문서 ID (자동 생성)                  │
│    "name": "Kim",                                                            │
│    "email": "kim@test.com",                                                  │
│    "address": {                        ← 중첩 문서                         │
│      "city": "Seoul",                                                         │
│      "zip": "12345"                                                           │
│    },                                                                       │
│    "orders": [                       ← 배열                               │
│      {"order_id": "O001", "amount": 100},                                  │
│      {"order_id": "O002", "amount": 200}                                   │
│    ],                                                                        │
│    "tags": ["vip", "premium"]        ← 문자열 배열                         │
│  }                                                                           │
│                                                                             │
│  [색인 및 쿼리]                                                             │
│                                                                             │
│  db.users.find({ "address.city": "Seoul" })                               │
│  → address.city 필드로 문서 조회                                             │
│                                                                             │
│  db.users.find({ "orders.amount": { $gt: 150 } })                        │
│  → orders 배열 내 금액이 150 초과인 문서 조회                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### MongoDB vs CouchDB 비교

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    MongoDB vs CouchDB 비교                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌────────────────┬────────────────────────┬────────────────────────┐      │
│  │     항목        │        MongoDB         │       CouchDB         │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 데이터 포맷   │ BSON (바이너리 JSON)    │ JSON                  │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 쿼리 방식     │ Mongodb Query Language  │ MapReduce, Views      │      │
│  │              │ (SQL-like)              │ (JavaScript)           │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 색인 지원     │ 다양한 색인 (단일,복합,  │ View 기반              │      │
│  │              │ 텍스트, 지리적)         │                        │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 일관성 모델   │ 유연한 일관성           │ Eventually Consistent  │      │
│  │              │ (설정 가능)             │ (AP 시스템)             │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 충돌 해결     │ Last Write Wins        │ 문서 버전 관리          │      │
│  │              │                        │ (복잡한 구조 가능)      │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 복제          │ 레플리카셋             │ CouchDB Replication     │      │
│  │              │                        │ (멀티 마스터)           │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 적합 용도      │ 범용 문서 저장,       │ 오프라인 우선,         │      │
│  │              │ 웹 애플리케이션,       │ 충돌 해결 중요,         │      │
│  │              │ 실시간 분석           │ CouchDB Mobile          │      │
│  └────────────────┴────────────────────────┴────────────────────────┘      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]**
MongoDB는 범용 문서 저장소로 다양한 쿼리와 색인을 지원하며, 유연한 일관성
설정이 가능하다. CouchDB는 충돌 해결에 강점을 가지며, 특히 오프라인 우선
애플리케이션이나 복제가 중요한 시나리오에 적합하다.

📢 섹션 요약: MongoDB와 CouchDB는 대표적인 문서 저장소로, 각각 다른 특장점을 가지므로 사용 사례에 맞게 선택해야 한다.

## Ⅲ. 결론

문서 저장소는 반정형 데이터 저장에 최적화된 NoSQL 모델로, MongoDB와 CouchDB가 대표적이다. 유연한 스키마와 개발 생산성 향상이 장점이며, inúmer用途에 적합하다.

📢 섹션 요약: 문서 저장소는 반정형 데이터에 최적화된 NoSQL 모델로, MongoDB와 CouchDB가 대표적이고 유연한 스키마가 장점이다.
