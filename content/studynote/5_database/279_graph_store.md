# 279. 그래프 저장소 (Graph Store)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 그래프 저장소는 노드(Node), 엣지(Edge), 속성(Property)로 데이터를 표현하며, 복잡한 관계 탐색(친구 친구 찾기, 최단 경로 계산 등)에 최적화된 NoSQL 모델이다.
> 2. **가치**: 소셜 네트워크, 추천 시스템, 사기 탐지, 지식 그래프, 네트워크 분석 등 관계 중심 작업에 뛰어난 성능을 보인다.
> 3. **융합**: Neo4j, Cypher, Gremlin, SPARQL, PageRank, GNN과 밀접하게 연관된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념 정의
그래프 저장소(Graph Store)는 데이터를 노드(Node), 엣지(Edge), 속성(Property)의 그래프 구조로 표현하는 NoSQL 데이터 모델이다. 노드는 실세계 개체를, 엣지는 노드 간의 관계를, 속성은 노드와 엣지의 정보를 저장한다.

### 필요성
관계형 데이터베이스에서 소셜 네트워크나 추천 시스템과 같은 복잡한 관계 데이터를 처리하려면 여러 JOIN 연산이 필요하여 성능이 저하된다. 그래프 저장소는 이러한 관계 중심 작업에 최적화된 구조를 제공한다.

### 비유
그래프 저장소는소셜 네트워크의 친구 관계도와 같다. 각 사람은 노드이고, 친구 관계는 엣지이다. "친구의 친구를 찾기"는 단순히 연결된 노드를 따라가면 되며, 지도를 펼치지 않고도 빠르게 찾을 수 있다.

📢 섹션 요약: 그래프 저장소는 노드, 엣지, 속성으로 데이터를 표현하며, 복잡한 관계 탐색에 최적화된 NoSQL 모델이다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 그래프 저장소 동작 구조

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    그래프 저장소 동작 구조                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [그래프 구조]                                                             │
│                                                                             │
│       (Kim) ────friend───▶ (Lee)                                          │
│         │                      │                                           │
│       friend                  friend                                         │
│         ▼                      ▼                                           │
│       (Park) ────friend───▶ (Choi)                                        │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐ │
│  │  노드 (Node)            │  엣지 (Edge)            │  속성 (Property) │ │
│  ├─────────────────────────┼─────────────────────────┼─────────────────┤ │
│  │ Kim                     │ Kim → Lee (friend)     │ Kim: age=30     │ │
│  │ Lee                     │ Kim → Park (friend)     │ Lee: age=25     │ │
│  │ Park                    │ Park → Choi (friend)    │ Park: age=28    │ │
│  │ Choi                    │ Lee → Choi (friend)     │ Choi: age=35    │ │
│  └─────────────────────────┴─────────────────────────┴─────────────────┘ │
│                                                                             │
│  [그래프 쿼리 예시 (Cypher - Neo4j)]                                       │
│                                                                             │
│  -- 친구의 친구를 찾기                                                       │
│  MATCH (p1:Person)-[:friend]->(p2:Person)-[:friend]->(p3:Person)         │
│  WHERE p1.name = 'Kim'                                                     │
│  RETURN p3.name                                                            │
│                                                                             │
│  -- 최단 경로 찾기                                                          │
│  MATCH path = shortestPath((a:Person {name:'Kim'})-[:friend*]-(b:Person {name:'Choi'})) │
│  RETURN path                                                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Neo4j vs Amazon Neptune 비교

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Neo4j vs Amazon Neptune 비교                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌────────────────┬────────────────────────┬────────────────────────┐      │
│  │     항목        │        Neo4j          │     Amazon Neptune     │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 저장소 엔진    │ Native Graph (자체)   │ 범용 저장소            │      │
│  │              │                      │ (VPC, SSD 지원)        │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 쿼리 언어     │ Cypher                │ Gremlin, SPARQL       │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 배포 방식     │ 온프레미스/클라우드    │ 완전 관리형 (AWS)     │      │
│  │              │ 모두 지원              │                      │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 확장성        │ 단일 서버 → 복제셋     │ 클러스터링 지원        │      │
│  │              │ (수백만 노드/엣지)    │                      │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ ACID 트랜잭션 │ 지원                  │ Support           │      │
│  │              │                      │ (완화된 일관성)       │      │
│  ├────────────────┼────────────────────────┼────────────────────────┤      │
│  │ 적합 용도      │ 범용 그래프,         │ 대규모 그래프,        │      │
│  │              │ 소셜 네트워크,        │ AWS 생태계 통합,     │      │
│  │              │ 추천 시스템          │ 완전 관리형 선호      │      │
│  └────────────────┴────────────────────────┴────────────────────────┘      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]**
Neo4j는 가장 인기 있는 그래프 데이터베이스로, 자체 Native Graph 저장소 엔진과
Cypher 쿼리 언어를 사용한다. Amazon Neptune은 AWS 완전 관리형으로,
Gremlin과 SPARQL 등 여러 그래프 쿼리 언어를 지원한다.

📢 섹션 요약: 그래프 저장소는 Neo4j와 Amazon Neptune이 대표적이며, 각각 Cypher와 Gremlin/SPARQL 등 다른 쿼리 언어를 사용한다.

## Ⅲ. 결론

그래프 저장소는 복잡한 관계 탐색에 최적화된 NoSQL 모델이다. 소셜 네트워크, 추천 시스템, 사기 탐지, 지식 그래프 등에 뛰어나며, Neo4j와 Amazon Neptune 등 다양한 구현이 있다.

📢 섹션 요약: 그래프 저장소는 관계 중심 작업에 특화된 NoSQL 모델로, Neo4j와 Neptune이 대표적이고 소셜 네트워크, 추천 시스템 등에 효과적이다.
