## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 세이브포인트(Savepoint)는 트랜잭션 내부에 중간 지점을 설정하여 해당 시점 이후의 작업만 선택적으로 롤백할 수 있는 메커니즘이다.
> 2. **가치**: 긴 트랜잭션에서 전체 롤백 대신 부분 롤백이 가능하여 작업 효율성을 크게 향상시킨다.
> 3. **융합**: 소프트웨어 버전 관리의 커밋 히스토리와 유사하며, 운영체제의 프로세스 체크포인트와 같은 원리다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념
세이브포인트(Savepoint)는 트랜잭션 내부에 설정하는 중간 저장점으로, 설정 이후의 연산만 롤백할 수 있게 한다. 전체 트랜잭션 롤백 대신 부분 롤백이 가능하므로, 긴 트랜잭션에서 유용하다.

### 필요성
복잡한 비지니스 로직에서는 트랜잭션 중간에某些 연산이 실패해도 처음부터 시작하고 싶지 않은 경우가 있다. 예컨대 쇼핑몰 주문에서 결제 실패 시 장바구니 비우기까지 롤백하고 싶지 않고, 해당 상품만 건너뛰고 싶을 수 있다.

### 비유
세이브포인트는 게임의 자동 저장과 같다. 보스 전에서 저장하고, 전투에서 패배하면 그 지점부터 다시 시작할 수 있다.

### 섹션 요약 비유
게임 자동 저장처럼, 세이브포인트는 긴 트랜잭션에서 "이곳까지 괜찮았다"를 표시하는 것이다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 동작 구조

```text
  ┌───────────────────────────────────────────────────────────────────────┐
  │                    세이브포인트 동작 예시                               │
  ├───────────────────────────────────────────────────────────────────────┤
  │
  │   BEGIN TRANSACTION;                                                │
  │       INSERT INTO orders ...;        -- 연산 1                         │
  │       INSERT INTO inventory ...;    -- 연산 2                         │
  │       SAVEPOINT sp1;               -- 세이브포인트 설정               │
  │       UPDATE stock ...;             -- 연산 3                         │
  │       INSERT INTO payment ...;      -- 연산 4                         │
  │       ROLLBACK TO SAVEPOINT sp1;   -- 연산 3,4만 롤백                 │
  │       INSERT INTO payment ...;      -- 연산 4 재시도                   │
  │   COMMIT;                                                            │
  │
  │   결과: 연산 1, 2, 4(재시도)만 완료, 연산 3은 롤백됨                   │
  │
  └───────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 세이브포인트는 롤백의粒度를 조절하는 메커니즘이다.ROLLBACK TO SAVEPOINT는 세이브포인트 이후의 연산만 취소하고 이전 연산은 유지한다. 이 예시에서 연산 3, 4가 실패하더라도 sp1 이전의 연산 1, 2는 유지되므로 처음부터 다시 시작할 필요가 없다. 그러나ROLLBACK TO는 세이브포인트를 설정한 시점으로만 돌아갈 수 있고, 설정 이전으로 돌아가려면 전체 ROLLBACK이 필요하다.

### 세이브포인트 제한

| 제한 | 설명 |
|:---|:---|
| **이름 중복** | 동일 트랜잭션 내에서 같은 이름의 세이브포인트 중복 설정 시 마지막 것이 우선 |
| **자동 해제** | 트랜잭션 종료 시 모든 세이브포인트가 자동으로 해제됨 |
| **ネスト禁止** | 대부분의 DBMS에서 세이브포인트의 세이브포인트는 지원되지 않음 |

### 섹션 요약 비유
세이브포인트는 마라톤比赛的折り返し地点と 같다.全部戻す 대신 중간 지점부터 다시 달릴 수 있다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교: Savepoint vs Nested Transaction

| 구분 | Savepoint | Nested Transaction |
|:---|:---|:---|
| **独立性** | 낮음 (부모에 종속) | 높음 (독립적 COMMIT/ROLLBACK) |
| **복구 범위** | 부모 트랜잭션 롤백 시 전체 | 자식만 롤백 가능 |
| **지원 DB** | 대부분의 RDBMS | PostgreSQL 등 제한적 |

### 섹션 요약 비유
Savepoint는同一个文件中不同版本的 책갈피와 같고, Nested Transactionは章ごとに別々の 파일로 저장하는 것과 같다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

배치処理에서 일부 레코드 처리 실패 시 전체를 롤백하는 대신 해당 레코드만 건너뛰고 계속 진행하는 경우. 이 경우 처리 전 세이브포인트를 설정하고, 실패 시 해당 레코드만 롤백 후 다음 레코드로 진행한다.

### 섹션 요약 비유
배치 处理에서의 세이브포인트 활용은 시험에서 답안지를 넘기며 部分的に уверен함을 유지하는 것과 같다.

## 👶 어린이를 위한 3줄 비유 설명
1. 세이브포인트は학창 시절의 **`저장`** 버튼과 같아서,万一를 위해 중간중간 저장하는 거예요.
2. 장거리 버스에서 2시간마다 휴게소에立ち寄る 것처럼, 긴 트랜잭션도 중간중간 저장하면全部 다시 할 필요가 없어요.
3. 게임에서 보스 전에セーブ하면 그 곳부터 다시 시작하듯, 세이브포인트도 실패해도 처음부터 하지 않아도 돼요!
