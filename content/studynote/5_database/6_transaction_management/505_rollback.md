## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 롤백(Rollback)은 트랜잭션 실패 시 UNDO 로그를 사용하여 모든 변경 사항을 원래 상태로 되돌리는 복구 연산이다.
> 2. **가치**: 부분 완료된 트랜잭션이 데이터베이스에 불일치를 초래하는 것을 방지하고, 시스템 장애 시 무결성을 보장한다.
> 3. **융합**: 롤백은 운영체제의 프로세스 복구나 버전 관리 시스템의 이전 版本还原과 동일한 원리의 응용이다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념
롤백(Rollback)은 실행 중이거나 부분 완료된 트랜잭션을 취소하여 데이터베이스를 트랜잭션 시작 전 상태로 되돌리는 연산이다. 롤백은 명시적(사용자가 ABORT 요청) 또는 암시적(제약조건 위반, 교착 상태 감지, 시스템 크래시)으로 수행될 수 있다.

### 필요성
롤백이 없으면 실패한 트랜잭션의 부분 완료된 변경 사항이 데이터베이스에 그대로 남게 되어 불일치가 발생한다. 예컨대 계좌에서 100원이 차감되었지만 입금 계좌에 반영되지 않은 상태가 영구적으로 남게 된다.

### 비유
롤백은 Photoshop에서 작업 후 저장하지 않고 닫는 것과 같다. 모든 변경 사항이 사라지고 이전 상태로 돌아간다.

### 섹션 요약 비유
롤백은 연필로 쓴 글씨를 지우개로 지우는 것과 같다. 쓴 글(변경 사항)을 지우고 원래的白紙(원래 상태)로 되돌린다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### UNDO 로그 기반 롤백 동작

```text
  ┌───────────────────────────────────────────────────────────────────────┐
  │                    UNDO 로그 기반 롤백 동작                            │
  ├───────────────────────────────────────────────────────────────────────┤
  │
  │   [UNDO 로그 구조]                                                    │
  │   ┌─────────────────────────────────────────────────────────────┐    │
  │   │ <UNDO, T1, X, old_value>  ← T1이 X를 old_value로 되돌려라  │    │
  │   └─────────────────────────────────────────────────────────────┘    │
  │
  │   [롤백 수행 과정]                                                    │
  │   1. 트랜잭션 T1의 로그 레코드 역순으로 처리                           │
  │   2. 각 로그 레코드에 대해:                                           │
  │      - 해당 페이지를 버퍼에서 로드 (이미 있으면 사용)                   │
  │      - old_value로 데이터 복원                                        │
  │      - CLR (Compensation Log Record) 기록                            │
  │   3. 트랜잭션 테이블에서 T1 제거                                      │
  │
  │   [실제 예시: 계좌 송금]                                              │
  │   T1: A -= 100, B += 100                                             │
  │
  │   UNDO 로그:                                                         │
  │   <UNDO, T1, A, 1000>  ← A를 1000으로 복원                           │
  │   <UNDO, T1, B, 500>   ← B를 500으로 복원                           │
  │
  │   롤백 순서: B 먼저 복원 (Xid大的为先) → A 그 다음                   │
  │
  └───────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** UNDO 로그는 "이 값을 이전 값으로 되돌려라"는 명령이다. 롤백 시 로그 레코드를 역순으로 처리하는 이유는 순방향 실행의 반대 순서로 원래 상태를 복원해야 하기 때문이다. 계좌 송금 예시에서 A에서 차감하고 B에 입금했으므로, 롤백 시에는 B에서 차감하고 A에 입금해야 한다. 또한 CLR(Compensation Log Record)을 기록하는 이유는 롤백 자체도 하나의 연산으로서 재복구 가능성을 열어두기 때문이다. 롤백 도중再度 장애가 발생하면 CLR을 보고 "여기까지 롤백됨"을 인지하고 그 이후부터 계속할 수 있다.

### 롤백 발생 원인

| 원인 | 설명 | 예시 |
|:---|:---|:---|
| **명시적 ABORT** | 사용자 또는 애플리케이션이 의도적으로 취소 | 트랜잭션 중간에 고객이 주문 취소 |
| **제약조건 위반** | CHECK, UNIQUE, FK 위반 | 중복 키 삽입 시도 |
| **교착 상태** | Deadlock 감지기에 의한 강제 중단 | 두 트랜잭션이 서로의 잠금 대기 |
| **시스템 크래시** | 장애 복구 과정에서 판단 | 로그에 COMMIT 없는 트랜잭션 |
| **_RESOURCE LIMIT** | 메모리/디스크 부족 | 대량 삽입 중 디스크 full |

### 섹션 요약 비유
롤백 발생 원인은 firefighters 출동 사유와 같다. 의도적 취소(가정용사), 규칙 위반(방화 예방 기준), 교착 상태(서로 돕는 구조), 시스템 장애(불꽃探测기) 등 다양하다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교: UNDO vs REDO vs UNDO+REDO

|モード | 사용 상황 | 로그 필요 여부 | 디스크 기록 시점 |
|:---|:---|:---|:---|
| **UNDO** | 변경 중인 페이지가 유실될 때 | 필요 | 즉시 |
| **REDO** | 변경 페이지가 디스크에 먼저 기록될 때 | 필요 | 나중 |
| **UNDO+REDO** | 즉시/지연 모두 가능한 경우 | 필요 | Flexible |

### 섹션 요약 비유
UNDO는 되돌리기(Undo Ctrl+Z), REDO는 다시하기(Redo Ctrl+Y)와 같다. 롤백은 UNDO의 연속 적용이다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

교착 상태로 인해 트랜잭션이 롤백될 때, 해당 트랜잭션이 이미 수신한 외부 메시지(송금 완료 통보 등)는 어떻게 처리하는가? 이는 애플리케이션 레벨의 보상 트랜잭션으로 처리해야 한다.

### 섹션 요약 비유
교착 상태에 빠진 두 트랜잭션의 롤백은 서로 동시에 한 발짝 물러서는 것과 같다. 그러나 한쪽이 먼저 물러서면(잠금 해제) 다른 쪽도 자연히 진행할 수 있게 된다.

---

## Ⅴ. 기대효과 및 결론

### 섹션 요약 비유
롤백은 뒷功夫와 같다. 잘 못 내리면 다시 처음부터 올바른 발톱을练习한다. 시스템도 마찬가지로 잘못된 연산은 되돌리고 다시 시도한다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **UNDO 로그** | 롤백의 물리적 기반으로, 이전 값(old_value)을 기록하여 복구时被用到。 |
| **WAL** | 로그를 먼저 기록하는 프로토콜로, 롤백 가능성을 보장한다. |
| **CLR** | Compensation Log Record로, 롤백 자체를 기록하여 재복구 가능성을 제공한다. |
| **교착 상태** | 롤백의 주요 원인 중 하나로, 두 트랜잭션이 서로의 잠금을 기다리며 무한 대기하는 상태다. |

## 👶 어린이를 위한 3줄 비유 설명
1. 롤백은 **折纸 되돌리기**와 같아서, 잘못 접으면 다시 처음부터 접어야 해요.
2. 은행에서 잘못된 송금을 취소하면 돈이 돌려오잖아? 그것이 바로 롤백이에요.
3. 게임에서 버그가 생기면 저장한 곳부터 다시 시작하듯, 컴퓨터도 문제가 생기면 이전 안전한 상태로 돌아가는 거예요!
