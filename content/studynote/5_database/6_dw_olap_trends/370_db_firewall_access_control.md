# 370. 접근 통제 정책 기반 방화벽 (DB 방화벽)

> ⚠️ 이 문서는 데이터베이스를 외부 공격과 내부자의 악의적인 유출로부터 보호하기 위해, **단순히 네트워크 접속을 막는 수준을 넘어 SQL 명령어의 내용까지 뜯어보고 접근을 제어하는 지능형 통제 시스템인 'DB 방화벽'**을 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 일반 방화벽(L4)이 IP와 포트 번호만 본다면, DB 방화벽(L7)은 데이터베이스로 날아오는 'SQL 쿼리문' 자체를 뜯어보고 허용/차단 여부를 결정하는 깐깐한 문지기다.
> 2. **가치**: 해커의 SQL 인젝션(SQL Injection) 공격을 실시간으로 막아낼 뿐만 아니라, 개발자나 유지보수 직원이 새벽에 몰래 접속하거나 실수로 `DROP TABLE`을 날리는 대참사도 완벽하게 차단한다.
> 3. **기술 체계**: **IP/포트/시간 제어**, **명령어 기반 통제(DML, DDL 제한)**, **마스킹 연동** 기능을 제공하며, 주로 DB 앞단에 프록시(Proxy)나 스니핑(Sniffing) 형태로 설치된다.

---

### Ⅰ. 개요: 털리고 나서 후회하지 마라 (Context & Necessity)
"우리 회사 서버는 일반 방화벽이 있어서 안전해요."
- 하지만 일반 방화벽은 포트(예: 3306)가 열려있고 정상적인 IP에서 접근하면, 그 안에 담긴 내용이 `SELECT`인지 `DROP DATABASE`인지 구별하지 못하고 통과시킨다.

DB 방화벽은 이 맹점을 노린다. **"누가(Who), 언제(When), 어디서(Where), 어떤 쿼리(What)를 날렸는가?"**를 완벽하게 분석한다.
- "퇴사 예정인 A대리가 새벽 3시에 고객 정보 테이블을 통째로 덤프(Dump) 뜨려고 하네? 차단해!"
이것이 DB 방화벽이 제공하는 핵심 가치인 **'세밀한 접근 통제(Fine-grained Access Control)'**다.

> 📢 **섹션 요약 비유**: 일반 방화벽이 아파트 정문에서 **'주민등록증(IP)만 보고 들여보내 주는 경비 아저씨'**라면, DB 방화벽은 내 집 현관문 앞에서 **'가방을 열어보고 도둑질할 물건(악성 SQL)이 있는지, 집 안에 들어갈 권한(접근 정책)이 있는지 깐깐하게 검사하는 개인 경호원'**과 같습니다.

---

### Ⅱ. DB 방화벽의 3대 핵심 통제 정책

DB 방화벽은 관리자가 설정한 룰(Rule)에 따라 작동한다.

#### 1. 환경적 통제 (IP / 포트 / 접근 시간)
- **정책**: "웹 서버 IP(`10.0.0.1`)에서만 접속 가능. 유지보수 직원은 평일 오전 9시~오후 6시에만 접속 가능."
- **효과**: 해커가 외부에서 직접 붙거나, 내부 직원이 야심한 밤에 악의적인 행동을 하는 것을 원천 차단한다.

#### 2. 명령어 및 객체 통제 (SQL/오브젝트 제어)
- **정책**: "개발팀 IP로는 `SELECT`만 가능하고, `UPDATE/DELETE/DROP` 명령어는 절대 차단. 급여 테이블(`Salary_TB`)은 아예 조회 불가."
- **효과**: 개발자의 실수로 인한 데이터 삭제(Human Error)를 막고, 직무 분리(SoD) 원칙을 강제한다.

#### 3. 패턴 및 결과 통제 (SQL 인젝션 방어 / 마스킹)
- **정책**: "쿼리문에 `1=1` 같은 이상한 패턴이 들어오면 차단(SQL 인젝션 방어). 1번에 1만 건 이상 데이터를 조회하려 하면 대량 유출로 간주하여 차단."
- **효과**: 지능적인 해킹 공격을 막고, 개인정보가 노출될 때는 별표(*)로 마스킹해서 보여준다.

---

### Ⅲ. DB 방화벽의 2가지 아키텍처 (구축 방식)

설치 위치에 따라 장단점이 극명하게 갈린다.

#### 1. 프록시 (Proxy / In-line) 방식
- **구조**: 애플리케이션과 DB 사이에 방화벽이 물리적으로 끼어든다. 모든 트래픽은 방화벽을 통과해야만 DB에 도달한다.
- **장점**: 이상한 쿼리가 들어오면 DB에 도달하기 전에 100% 튕겨낸다 (적극적 차단).
- **단점**: 방화벽 장비가 고장 나면 DB 연결도 통째로 끊긴다(SPOF). 속도가 미세하게 느려진다.

#### 2. 스니핑 (Sniffing) 방식
- **구조**: 네트워크 스위치에서 트래픽을 거울처럼 복사(Mirroring)해서 방화벽으로 던져준다.
- **장점**: 방화벽이 죽어도 원래 DB 통신에는 아무런 지장이 없다 (무중단 보장).
- **단점**: 복사본을 보고 뒤늦게 "어? 이거 해킹이네!" 하고 차단 신호를 보내기 때문에, 첫 번째 악성 쿼리는 DB에 실행되어 버릴 위험이 있다 (소극적 차단).

```text
┌──────────────────────────────────────────────────────────────┐
│           DB 방화벽(프록시 방식)의 접근 통제 프로세스 시각화             │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ [ 👨‍💻 악의적 내부자 ]                                             │
│        │ "DROP TABLE Users;" (금요일 밤 11시)                  │
│        ▼                                                     │
│ 🛡️ [ DB 방화벽 (Proxy) ]                                        │
│   ├── 1. 시간 검사: 밤 11시? (근무 시간 외 ❌)                       │
│   ├── 2. 권한 검사: 개발자 IP? (권한 없음 ❌)                        │
│   ├── 3. 쿼리 검사: DROP 명령어? (위험 명령어 ❌)                    │
│   │                                                          │
│   ▼ (차단 및 로깅)                                             │
│ 🚨 "접근 통제 정책 위반으로 차단되었습니다!" (경고 알람 발생)              │
│                                                              │
│ ★ 특징: 위험한 쿼리는 원본 DB 근처에도 가지 못하고 문전박대당함.         │
└──────────────────────────────────────────────────────────────┘
```

### Ⅳ. 결론

"보안의 최종 방어선은 결국 데이터가 잠들어 있는 곳이다."
네트워크 방화벽, 웹 방화벽(WAF)이 아무리 튼튼해도 결국 해커의 최종 목표는 DB에 있는 고객 데이터다. DB 방화벽은 이 마지막 골목을 지키는 가장 정밀한 체(Filter)다. 오늘날 개인정보보호법(PIPL)과 ISMS 등 각종 규제 컴플라이언스에서는 단순한 접속 기록(로그)을 넘어, "누가 어떤 데이터를 건드렸는지 통제하고 증명할 수 있는가?"를 핵심 의무로 규정하고 있으므로, 엔터프라이즈 환경에서 DB 방화벽은 선택이 아닌 필수 생존 장비가 되었다.

### 📌 관련 개념 맵
* **보안 공격**: SQL Injection (371번 문서)
* **네트워크 보안**: WAF (웹 방화벽), IPS/IDS
* **데이터 보안**: DB 암호화 (365번), 데이터 마스킹 (367번), DB 감사(Auditing - 369번)
* **아키텍처 용어**: Proxy, Sniffing, Port Mirroring

---

### 👶 어린이를 위한 3줄 비유 설명
1. 일반 방화벽은 우리 집 대문에서 "이름표(IP)"만 확인하고 들여보내 주는 문지기예요. 도둑이 가짜 이름표를 달고 오면 뚫리죠.
2. DB 방화벽은 내 보물상자(DB) 바로 앞에 서 있는 아주 깐깐한 경호원이에요.
3. 내 방에 들어오려는 사람이 누구인지, 몇 시에 왔는지, 그리고 손에 망치(나쁜 쿼리)를 들고 있는 건 아닌지 머리부터 발끝까지 다 검사해서 나쁜 짓을 완벽하게 막아낸답니다!
