# 386. 데이터 클린룸 (Data Clean Room) 보안 파티션 교환

> ⚠️ 이 문서는 "우리 회사 고객 데이터랑 너네 회사 고객 데이터를 합쳐서 타겟팅 광고를 해보자!"라고 할 때 발생하는 개인정보보호법 위반 문제를 해결하기 위해, **서로의 원본 데이터를 절대로 보지 못하게 암호화된 방(Clean Room)에 가둬두고 오직 교집합의 '통계 결과'만 뽑아갈 수 있게 만드는 B2B 데이터 공유 아키텍처**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 두 회사가 데이터를 공유할 때 서로의 원본 데이터베이스(Raw Data)를 직접 주고받지 않고, 제3의 안전한 클라우드 공간(Data Clean Room)에서 데이터를 매칭시키는 보안 기술이다.
> 2. **가치**: 고객의 민감한 개인정보(이메일, 전화번호)를 완벽하게 보호하면서도, 두 회사가 가진 고객의 교집합이나 구매 패턴을 합쳐서(Join) 강력한 타겟 마케팅을 합법적으로 수행할 수 있게 해준다.
> 3. **기술 체계**: **Snowflake**, **AWS Clean Rooms** 같은 글로벌 데이터 웨어하우스가 이 기능을 플랫폼 차원에서 제공하며, 개인정보는 해시(Hash) 처리되고 쿼리 결과는 익명화(Anonymization)되어 반환된다.

---

### Ⅰ. 개요: 섞고 싶지만 보여줄 순 없어 (Context & Necessity)
"디즈니플러스"와 "아디다스"가 제휴 마케팅을 하려고 한다.
- 아디다스: "우리 신발 산 사람들 중에 디즈니플러스 보는 사람한테만 디즈니 신발 광고를 보내고 싶어. 너희 고객 DB 좀 줄래?"
- 디즈니플러스: "미쳤어? 개인정보보호법 위반으로 감옥 가. 우리 고객 이메일을 어떻게 너한테 줘?"

이 딜레마를 해결하는 마법의 방이 **데이터 클린룸(Data Clean Room)**이다.
두 회사는 각자의 고객 이메일을 암호화(Hash)해서 클린룸에 밀어 넣는다.
클린룸 안에서는 암호화된 이메일끼리 비교해서 "양쪽 다 가입한 사람은 1만 명입니다"라는 결과만 아디다스에게 알려준다. 아디다스는 디즈니 고객이 정확히 누군지(이름, 이메일)는 절대 볼 수 없지만, 그 1만 명에게 광고를 쏠 수는 있게 된다.

> 📢 **섹션 요약 비유**: 클린룸은 **'유리벽으로 가로막힌 경찰서 면회실'**과 같습니다. A와 B는 룸 안에서 서로 대화하고 정보를 합칠 수는 있지만, 서로의 물건(원본 데이터)을 직접 만지거나 훔쳐서 룸 밖으로 가져나갈 수는 없습니다.

---

### Ⅱ. 데이터 클린룸의 작동 원리와 보안 제어

클린룸의 생명은 "원본 유출 방지"다. 이를 위해 3단계 보안이 들어간다.

#### 1. 해싱 (Hashing & Encryption)
- 데이터를 클린룸에 넣기 전에 `abc@gmail.com`을 `df8a9b...` 같은 알아볼 수 없는 해시값으로 바꾼다. (양쪽 회사가 똑같은 암호화 규칙을 써야 매칭이 가능함)

#### 2. 제한된 쿼리 (Restricted Query)
- 클린룸 안의 데이터를 조회할 때 `SELECT *` 같은 무식한 쿼리는 원천 차단된다.
- 오직 `COUNT`, `SUM`, `AVG` 같은 **통계 함수(Aggregation)만 허용**된다.

#### 3. 차등 프라이버시 (Differential Privacy) & 최소 임계값
- "통계만 허용한다고? 그럼 내가 쿼리를 좁혀서 '디즈니+를 구독하는 서울 사는 20대 김철수의 신발 구매 횟수(SUM)'를 조회하면 김철수 개인의 데이터를 알 수 있겠네?"
- 이걸 막기 위해 클린룸은 **"조회 결과가 100명 이하일 경우 아예 에러를 뱉어버리는(최소 임계값)"** 룰을 강제하거나, 결과값에 의도적인 **가짜 노이즈(차등 프라이버시, 362번 문서)**를 섞어서 특정 개인을 유추할 수 없게 만든다.

### Ⅲ. 아키텍처 구현: Snowflake Data Clean Room

최근 데이터 웨어하우스의 절대 강자인 Snowflake가 이 분야를 평정하고 있다.

- **특징**: 물리적으로 데이터를 복사해서 남의 서버로 보내는 게 아니다.
- **원리 (Secure Data Sharing)**: Snowflake는 클라우드 안에서 데이터 저장소와 쿼리 연산기(Compute)가 분리되어 있다. A회사의 데이터는 A회사 DB에 가만히 놔두고, B회사에게 **'안전하게 통제된 논리적인 접근 뷰(View)'**만 열어주는 방식을 쓴다. (데이터 이동 비용 = 0원)

```text
┌──────────────────────────────────────────────────────────────┐
│           데이터 클린룸 (Data Clean Room) 쿼리 처리 시각화             │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ [ 🏢 아디다스 DB ]                     [ 🏢 디즈니+ DB ]      │
│  (이메일 Hash)                          (이메일 Hash)         │
│         │                                      │             │
│         ▼                                      ▼             │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ 🛡️ Data Clean Room (안전 지대)                           │ │
│ │                                                          │ │
│ │  SELECT COUNT(*)                                         │ │
│ │  FROM 아디다스_DB JOIN 디즈니_DB ON Email_Hash               │ │
│ │  WHERE 디즈니.시청시간 > 10시간                                │ │
│ └──────────────────────────────────────────────────────────┘ │
│         │ (결과 리턴: "15,000명 입니다")                        │
│         ▼                                                    │
│ [ 👨‍💻 아디다스 마케터 ] "좋아, 이 15,000명 덩어리에게만 광고 쏴!"         │
│                                                              │
│ ★ 특징: 아디다스는 15,000명이 구체적으로 누구인지 절대 알 수 없음!         │
└──────────────────────────────────────────────────────────────┘
```

### Ⅳ. 결론

"데이터는 나눌수록 커지지만, 보안은 나눌수록 취약해진다."
서드파티 쿠키가 소멸(385번 문서)하면서 다른 회사의 데이터를 훔쳐보는 꼼수는 끝났다. 이제 기업들은 합법적인 '데이터 동맹(Data Partnership)'을 맺어야만 살아남을 수 있다. 데이터 클린룸은 이 동맹을 가능하게 하는 유일한 기술적 신뢰의 장이다. 데이터를 복사하지 않고 권한만 교환하는 이 우아한 아키텍처를 이해하는 데이터 엔지니어는 B2B 데이터 비즈니스의 판도를 바꿀 수 있을 것이다.

### 📌 관련 개념 맵
* **관련 기술**: Snowflake, AWS Clean Rooms, Google Ads Data Hub
* **보안 기술**: Differential Privacy (차등 프라이버시), Hashing, Homomorphic Encryption (동형 암호)
* **대결 구도**: Third-Party Cookie (사망) vs Data Clean Room (부상)
* **관련 직무**: Data Engineer, 마테크(MarTech) 아키텍트

### 👶 어린이를 위한 3줄 비유 설명
1. 클린룸은 두 친구가 서로의 일기장을 보여주지 않고 비밀을 알아내는 마법의 상자예요.
2. A친구가 상자에 "나 오늘 피자 먹었어"라고 쓰고, B친구가 "나 오늘 피자 먹었어"라고 쓰면...
3. 상자가 "두 친구 모두 피자를 먹었네요!"라고만 알려주고, 일기장 전체 내용은 절대 서로 못 보게 막아주는 똑똑한 상자랍니다!
