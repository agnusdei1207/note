# 무결성 제약조건 (Integrity Constraints)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 무결성 제약조건 (Integrity Constraints)은 데이터베이스에 저장될 수 있는 데이터의 유효성과 일관성을 보장하기 위한 규칙이다.
> 2. **가치**: 제약조건을 통해 잘못된 데이터의 삽입/수정/삭제를 방지하고, 데이터의 정확성을 유지하며, 응용 程序의 검증 부담을 줄인다.
> 3. **융합**: 무결성 제약조건은 DBMS 차원에서 자동으로 적용되어, 데이터 품질을 보장하고 비즈니스 규칙을 안전하게 집행한다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념 정의

무결성 제약조건 (Integrity Constraints)은 **데이터베이스에서数据的 정확성과 일관성을 보장하기 위해 데이터에 부과되는 제약 (Constraints)**이다.

### 비유

무결성 제약조건은 **체크리스트**와 같다. 호텔 체크인 시身份证確認,reservation Gage, 만료일 확인 등을 통해 잘못된 check-in을 방지하는 것처럼, 제약조건도 데이터베이스에 유효하지 않은 데이터가 들어가는 것을 방지한다.

- **📢 섹션 요약 비유:**주민등록证의有効期限처럼, 데이터베이스에서도 데이터가 유효한 범위 내에 있는지自動的に検証する必要があります.

---

## Ⅱ. 주요 무결성 제약조건

### 1. 개체 무결성 (Entity Integrity)

기본 키 (Primary Key)는 NULL일 수 없고, 중복될 수 없다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    개체 무결성 예시                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   customers 테이블:                                                  │
│   ┌────────┬────────┬────────┐                                    │
│   │ 고객ID* │ 이름    │ 도시   │                                    │
│   ├────────┼────────┼────────┤                                    │
│   │ C001   │ 김철수  │ 서울   │ ← 유효                            │
│   │ NULL   │ 이영희  │ 부산   │ ← 위반! (NULL 불가)               │
│   │ C001   │ 박민수  │ 서울   │ ← 위반! (중복 불가)                │
│   └────────┴────────┴────────┘                                    │
│       * = 기본 키                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

---

### 2. 참조 무결성 (Referential Integrity)

외래 키 (Foreign Key)는 참조하는 기본 키 값과 일치하거나 NULL이어야 한다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    참조 무결성 예시                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   customers (기준 테이블):                                           │
│   ┌────────┐                                                     │
│   │ 고객ID* │ ← 기본 키                                           │
│   │ C001   │                                                     │
│   │ C002   │                                                     │
│   └────────┘                                                     │
│                                                                     │
│   orders (참조 테이블):                                            │
│   ┌────────┬────────┐                                            │
│   │ 주문ID* │ 고객ID† │ ← 외래 키                                 │
│   ├────────┼────────┤                                            │
│   │ O001   │ C001   │ ← 유효 (customers에 존재)                  │
│   │ O002   │ C003   │ ← 위반! (customers에 존재하지 않음)         │
│   │ O003   │ NULL   │ ← 유효 (NULL 허용)                          │
│   └────────┴────────┘                                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

---

### 3. 도메인 무결성 (Domain Integrity)

속성 값이 정의된 도메인 (값의 범위)에 속해야 한다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    도메인 무결성 예시                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   성별 열의 도메인: {'남', '여'}                                   │
│                                                                     │
│   ┌────────┬────────┐                                            │
│   │ 고객ID* │ 성별    │                                            │
│   ├────────┼────────┤                                            │
│   │ C001   │ 남      │ ← 유효                                     │
│   │ C002   │ 여      │ ← 유효                                     │
│   │ C003   │ 남자    │ ← 위반! ('남'이 아님)                      │
│   │ C004   │ NULL   │ ← NULL는特殊情况으로 허용 가능                │
│   └────────┴────────┘                                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 👶 어린이를 위한 3줄 비유 설명

1. 무결성 제약조건은 **게임의 규칙**과 같아요. 축구에서 공을 손으로 만지면 안 되듯이, 데이터에도 지켜야 할 규칙이 있어요.
2. 기본 키가 NULL이면 (개체 무결성 위반) 학생증 없이 학교에 가는 것과 같고,
3. 다른 반 학생의 번호를 적으면 (참조 무결성 위반) 그 반에 정말 있는지 확인 안 한 것과 같아요.
