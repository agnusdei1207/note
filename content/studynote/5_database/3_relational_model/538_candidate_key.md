# 후보 키와 대체 키 (Candidate Key / Alternate Key)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 후보 키 (Candidate Key)는 투플을 유일하게 식별할 수 있는 속성의 최소 집합이며, 이 중 선택된 것이 기본 키 (Primary Key), 선택되지 않은 것은 대체 키 (Alternate Key)다.
> 2. **가치**: 후보 키를 분석하면 테이블의 키 구조를 명확히 하고, 정규화 과정에서 중요한 판단 기준이 된다.
> 3. **융합**: 슈퍼 키 (Super Key)는 후보 키의 상위 집합으로, uniqueness는 보장하지만 minimality는 보장하지 않는다.

---

## Ⅰ. 개요 및 필요성

### 개념 정의

후보 키 (Candidate Key)는 **투플을 유일하게 식별할 수 있는 속성의 집합**으로, 유일성 (Uniqueness)과 최소성 (Minimality)을 만족한다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    후보 키와 기본 키의 관계                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   [후보 키 (Candidate Key)]                                        │
│   • 고객ID: 1001, 1002, ... (유일)                                │
│   • 주민등록번호: 123456-1234567, ... (유일)                        │
│   • 이메일: kim@test.com, lee@test.com, ... (유일)                 │
│                                                                     │
│   세 가지 모두 후보 키가 될 수 있음 → 3개의 후보 키                    │
│                                                                     │
│   [기본 키 (Primary Key)]                                           │
│   → 이 중 선택된 하나                                                │
│   → 일반적으로 인조 식별자 (auto-increment)를 선택                    │
│                                                                     │
│   [대체 키 (Alternate Key)]                                         │
│   → 선택되지 않은 나머지 후보 키                                     │
│   → UNIQUE 제약조건으로 유지                                        │
│                                                                     │
│   [슈퍼 키 (Super Key)]                                             │
│   • (고객ID, 이름): 유일성은 만족하지만 최소성 불만족                   │
│   • (고객ID, 이메일): 유일성 + 최소성 → 후보 키                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **기본 키** | 후보 키 중 선택된 것으로, 테이블의 주요 식별자다. |
| **슈퍼 키** | 유일성은 만족하지만 최소성은 만족하지 않는 키다. |
| **대체 키** | 후보 키이지만 기본 키로 선택되지 않은ものだ. |
| **유일성** | 키 값이 테이블 내에서 중복되지 않는 성질이다. |
| **최소성** | 키를 구성하는 속성을 하나라도 제거하면 유일성이 깨지는 성질이다. |

## 👶 어린이를 위한 3줄 비유 설명

1. 후보 키는 **학생증을 만들 수 있는 여러 방법**과 같아요. 학번으로도, 주민등록번호로도,指纹으로도 학생을唯一하게 식별할 수 있어요.
2. 이 중에서 가장 편리한 것을 선택해서 실제 학생증에 넣는 것이 기본 키예요.
3. 선택되지 않은 나머지 방법은 학생증에 기입되지 하지만,指纹으로도 학생을 식별할 수 있는 것처럼备用할 수 있어요!
