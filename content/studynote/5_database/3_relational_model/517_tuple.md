# 투플 (Tuple)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 투플 (Tuple)은 관계에서 모든 속성 값을 포함하는 순서쌍으로, 관계형 데이터베이스에서 행 (row)에 해당한다.
> 2. **가치**: 투플은 데이터베이스에서 하나의 완전한 레코드를 의미하며, 키 속성을 통해 유일하게 식별된다.
> 3. **융합**: 관계 대수와 관계 해석에서 투플 변수를 사용하며, SQL의 SELECT 결과로返される。

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념 정의

투플 (Tuple)은 **관계에서 모든 속성 값의 순서쌍**이다. 관계에 포함된 각 행 (row)을 의미하며, 하나의 완전한 데이터 레코드를 나타낸다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    투플 (Tuple)의 예시                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   고객 관계 (Relation):                                              │
│   ┌────────┬────────┬────────┬────────┐                           │
│   │ 고객ID  │ 이름    │ 나이   │ 도시   │                           │
│   ├────────┼────────┼────────┼────────┤                           │
│   │ C001   │ 김철수  │ 35     │ 서울   │ ← 투플 #1                 │
│   │ C002   │ 이영희  │ 28     │ 부산   │ ← 투플 #2                 │
│   │ C003   │ 박민수  │ 42     │ 서울   │ ← 투플 #3                 │
│   └────────┴────────┴────────┴────────┘                           │
│                                                                     │
│   투플 #1: <C001, 김철수, 35, 서울>                                │
│   투플 #2: <C002, 이영희, 28, 부산>                                │
│   투플 #3: <C003, 박민수, 42, 서울>                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 투플은 관계의 모든 속성 값을 순서대로 포함한다. 고객ID=C001, 이름=김철수, 나이=35, 도시=서울인 투플은 "<C001, 김철수, 35, 서울>"로 표현할 수 있다. 그러나 실제로는 속성명이 중요하며, 순서쌍 표현은 수학적 정의에 가까우므로 실무에서는 속성명-값의 조합으로 이해하는 것이 더 적절하다.

### 비유

투플은 **학생 건강 기록표의 한 행**과 같다. 각横行에는学번, 이름, 키, 몸무게 등의 정보가 모두 포함되어 있으며, 이는 하나의 student 레코드에 해당한다.

- **📢 섹션 요약 비유:**스프레드시트에서横行 하나가 특정 사람의 모든 정보를 담고 있듯이, 투플도 관계에서 하나의 완전한 데이터 레코드를 나타냅니다.

## Ⅱ. 투플의 특성

### 1. 원자값

투플을 구성하는 각 속성의 값은 **원자값 (Atomic Value)**이다. 즉, 더 이상 분할할 수 없는 단일 값이다.

### 2. 순서의 부재 (수학적 관점)

수학적으로 투플은 순서가 있지만, 관계형 데이터베이스에서는 속성 간 순서는 논리적으로 의미가 없다. 그러나 실무적으로는 CREATE TABLE 문에서의 열 순서가 참조의 기준이 된다.

### 3. 유일성

관계에서 동일한 투플은 존재하지 않는다. 중복 투플이 필요하면 명시적 제약조건 (UNIQUE)을 추가해야 한다.

### 4. 키에 의한 식별

각 투플은 기본 키 (Primary Key)에 의해 유일하게 식별된다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    투플의 키에 의한 식별                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   고객 관계:                                                        │
│   ┌────────┬────────┬────────┬────────┐                           │
│   │ 고객ID* │ 이름    │ 나이   │ 도시   │                           │
│   ├────────┼────────┼────────┼────────┤                           │
│   │ C001   │ 김철수  │ 35     │ 서울   │ ← 투플 #1                 │
│   │ C002   │ 이영희  │ 28     │ 부산   │ ← 투플 #2                 │
│   │ C003   │ 박민수  │ 42     │ 서울   │ ← 투플 #3                 │
│   └────────┴────────┴────────┴────────┘                           │
│        * = 기본 키 (Primary Key)                                   │
│                                                                     │
│   투플 #1의 고객ID=C001 → 유일하게 식별 가능                         │
│   투플 #2의 고객ID=C002 → 유일하게 식별 가능                         │
│   투플 #3의 고객ID=C003 → 유일하게 식별 가능                         │
│                                                                     │
│   → 고객ID가 없으면 (NULL이면) 투플 식별 불가                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유:**학생证的学번이 각 학생을 유일하게 구분하듯이, 투플도 기본 키에 의해 유일하게 식별됩니다.

## Ⅲ. 실무 적용

### SELECT 결과로서의 투플

SQL의 SELECT 문 결과로返されるデータの単位이 투플이다.

```sql
SELECT customer_id, name, age, city
FROM customers
WHERE city = '서울';
```

위 쿼리의 결과는 도시가 '서울'인 투플들의 집합이다.

### 투플 변이

관계 대수에서 투플 변수는 투플을 대표하는 변수다.

```
π_name, city (σ_city='서울' (customers))
```

위 관계 대수 식에서 σ는 선택 (selection) 연산, π는 프로젝션 (projection) 연산이다.
