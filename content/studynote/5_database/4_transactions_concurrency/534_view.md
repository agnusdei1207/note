# 뷰 (View)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 뷰 (View)는 하나 이상의 테이블에서 유도되는 가상 테이블로, 실제 데이터는 저장하지 않고 SELECT 문으로 정의된다.
> 2. **가치**: 뷰를 통해 복잡한 쿼리를 단순화하고, 데이터 접근을 제한하며, 외부 스키마를 제공하여 데이터 독립성을 실현한다.
> 3. **융합**: updatable view, materialized view, CTE 등 다양한 확장이 있으며, 보안과 성능 최적화에 활용된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념 정의

뷰 (View)는 **하나 이상의 테이블 (또는 다른 뷰)에서 유도되는 가상 테이블**로, 데이터를 저장하지 않고 SELECT 문으로 정의된다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    뷰의 구조                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   [기본 테이블: customers]                                           │
│   ┌────────┬────────┬────────┐                                    │
│   │ 고객ID  │ 이름    │ 급여   │                                    │
│   ├────────┼────────┼────────┤                                    │
│   │ C001   │ 김철수  │ 5000  │                                    │
│   │ C002   │ 이영희  │ 3000  │                                    │
│   │ C003   │ 박민수  │ 7000  │                                    │
│   └────────┴────────┴────────┘                                    │
│                                                                     │
│   [뷰: high_salaries]                                             │
│   CREATE VIEW high_salaries AS                                      │
│   SELECT 고객ID, 이름, 급여                                         │
│   FROM customers                                                   │
│   WHERE 급여 > 5000;                                               │
│                                                                     │
│   [뷰 호출 결과]                                                   │
│   SELECT * FROM high_salaries;                                    │
│   ┌────────┬────────┬────────┐                                    │
│   │ 고객ID  │ 이름    │ 급여   │                                    │
│   ├────────┼────────┼────────┤                                    │
│   │ C001   │ 김철수  │ 5000  │                                    │
│   │ C003   │ 박민수  │ 7000  │                                    │
│   └────────┴────────┴────────┘                                    │
│                                                                     │
│   ※ 뷰는 데이터를 저장하지 않음 → 가상 테이블                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 비유

뷰는 **안경 처방전**과 같다. 처방전 자체가視力を改善하지는 않지만, 안경을掛면 보이는 것처럼, 뷰를 사용하면 기본 테이블을 다르게 볼 수 있다.

- **📢 섹션 요약 비유:**같은 데이터도 용도에 따라不同的视图로 볼 수 있는 것이ビューです。例えは、 급여 정보는本人만 볼 수 있도록 뷰를限制할 수 있어요.

## Ⅱ. 뷰의 유형

### 1. 단순 뷰 (Simple View)

하나의 테이블에서 유도되고, DML (INSERT/UPDATE/DELETE) 操作이 가능

### 2. 복합 뷰 (Complex View)

여러 테이블 JOIN, GROUP BY, 함수가 사용され、DML 操作이 제한적

### 3. 물리적 뷰 (Materialized View)

실제로 데이터를 저장하는 뷰로,Refresh 시점에서 갱신

```
┌─────────────────────────────────────────────────────────────────────┐
│                    물리적 뷰 vs 가상 뷰                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   [가상 뷰 (Virtual View)]                                         │
│   • 데이터 미저장                                                   │
│   • 호출 시 마다 SELECT 실행                                        │
│   • 항상 최신 데이터                                                │
│                                                                     │
│   [물리적 뷰 (Materialized View)]                                    │
│   • 실제 데이터 저장                                                │
│   • 주기적 또는 수동 Refresh                                       │
│   • 빠른 조회 (실행 불필요)                                         │
│   • 데이터 일관성Latency 존재                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 👶 어린이를 위한 3줄 비유 설명

1. 뷰는 **같은 데이터에 대해 다른 색안경**을 쓰는 것과 같아요. 붉은색 안경을掛면 모두 붉게 보여요.
2. 급여 정보에 대해 "관리자만 모든 급여 볼 수 있음"이라는 뷰를 만들면, 다른 부서는 급여 열 자체를 볼 수 없어요.
3. 하지만 안경 자체가 눈을改善하지 않듯이, 뷰도 실제 데이터를變하지는 않아요!
