# 222. 타임스탬프 순서 (Timestamp Ordering) 기법

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 타임스탬프 순서 기법은 락(Lock)을 사용하지 않고, 트랜잭션이 시스템에 진입한 순서대로 부여된 **타임스탬프를 기준으로 연산의 선후 관계를 강제하는 비관적/낙관적 혼합형 동시성 제어 기술**이다.
> 2. **가치**: 교착 상태(Deadlock)가 발생하지 않는다는 강력한 장점을 지니며, 데이터 항목마다 마지막 읽기/쓰기 시점을 기록하여 나중에 온 트랜잭션이 먼저 처리되는 모순을 원천 차단한다.
> 3. **융합**: 분산 시스템 환경에서 글로벌 일관성을 유지하기 위한 핵심 알고리즘으로 차용되며, 토마스 기록 규칙(Thomas Write Rule)과 결합하여 성능을 보정한다.

---

### Ⅰ. 타임스탬프 순서 기법의 원리

- **정의**: 각 트랜잭션 $T$에 부여된 고유 식별자 $TS(T)$를 기반으로, 모든 연산이 이 시간 순서를 따르도록 관리합니다.
- **데이터 항목별 타임스탬프**:
    - **W-TS(Q)**: 데이터 항목 Q를 성공적으로 쓴 가장 늦은 트랜잭션의 시간.
    - **R-TS(Q)**: 데이터 항목 Q를 성공적으로 읽은 가장 늦은 트랜잭션의 시간.

---

### Ⅱ. 동작 매커니즘 (ASCII Flow)

트랜잭션 $T$가 데이터 $Q$를 읽거나 쓰려 할 때의 판단 로직입니다.

```text
[Timestamp Ordering Decision Logic]

  Case: T tries to Read(Q)
  ┌────────────────────────────────────────────────────────┐
  │ IF TS(T) < W-TS(Q) THEN                                │
  │    "거절! 이미 미래의 트랜잭션이 덮어씀" ──▶ ROLLBACK     │
  │ ELSE                                                   │
  │    "허용! R-TS(Q)를 MAX(R-TS(Q), TS(T))로 업데이트"       │
  └────────────────────────────────────────────────────────┘

  Case: T tries to Write(Q)
  ┌────────────────────────────────────────────────────────┐
  │ IF TS(T) < R-TS(Q) OR TS(T) < W-TS(Q) THEN             │
  │    "거절! 미래의 누군가 이미 읽었거나 씀" ──▶ ROLLBACK    │
  │ ELSE                                                   │
  │    "허용! W-TS(Q)를 TS(T)로 업데이트"                    │
  └────────────────────────────────────────────────────────┘
```

---

### Ⅲ. 특징 및 한계

- **장점**: 락을 걸지 않으므로 **Deadlock이 절대 발생하지 않습니다.**
- **단점**: 트랜잭션이 충돌할 경우 대기하지 않고 즉시 롤백(Rollback)되므로, 충돌이 잦은 환경에서는 자원 낭비가 심할 수 있습니다. (Starvation 위험)

- **📢 섹션 요약 비유**: 타임스탬프 기법은 **'번호표가 있는 은행 창구'**와 같습니다. 10번 손님이 업무를 보는 도중 9번 손님이 뒤늦게 달려와서 "내가 먼저 할래!"라고 하면, 직원은 "이미 시간이 지났습니다!"라며 9번 손님을 돌려보내는(Rollback) 냉정한 규칙입니다.

---

### Ⅳ. 개념 맵 및 요약

- **[직렬 가능성]**: 시간 순서대로만 실행되므로 항상 보장됨.
- **[Thomas Write Rule]**: 늦게 온 쓰기가 과거의 것이라면 그냥 무시하고 진행하는 성능 최적화 기법.
- **[분산 DB]**: 물리적 시계(TrueTime 등)와의 융합 지점.

📢 **마무리 요약**: **Timestamp Ordering**은 락의 복잡함을 수학적인 시간 논리로 풀어낸 기법입니다. 대기 없는 빠른 처리가 필요한 분산 환경의 필수 알고리즘입니다.
