# 201. 동시성 제어(Concurrency Control)의 목적

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 동시성 제어는 다수의 사용자가 동시에 데이터베이스에 접근하여 트랜잭션을 수행할 때, 데이터의 **일관성(Consistency)과 무결성**을 유지하기 위한 트랜잭션 조율 기술이다.
> 2. **가치**: 시스템 전체의 **처리량(Throughput)을 극대화**하면서도, 마치 혼자서 DB를 사용하는 것과 같은 **고립성(Isolation)**을 보장한다.
> 3. **융합**: 이는 운영체제의 세마포어(Semaphore)나 뮤텍스(Mutex) 개념이 데이터베이스 트랜잭션 단위로 확장된 것이며, 현대의 고성능 서버 아키텍처의 필수적 근간이다.

---

### Ⅰ. 동시성 제어의 개념 및 필요성

- **개념**: 다중 사용자 환경에서 여러 트랜잭션이 동시에 실행될 때, 데이터베이스의 일관성을 해치지 않도록 트랜잭션들 간의 상호작용을 제어하는 매커니즘입니다.
- **필요성**:
    - **데이터 일관성 보장**: 동시 접근 시 발생할 수 있는 데이터 모순 방지.
    - **자원 활용도 최적화**: CPU 및 디스크 I/O를 병렬로 사용하여 응답 시간 단축.
    - **사용자 편의성**: 여러 사용자가 기다림 없이 동시에 작업 가능.

---

### Ⅱ. 동시성 제어의 주요 목적 (Core Goals)

#### 1. 데이터베이스의 일관성 유지 (Consistency)
트랜잭션 실행 전후의 데이터베이스 상태가 항상 모순 없는 일관된 상태여야 합니다.

#### 2. 다중 사용자 처리량(Throughput) 극대화
데이터베이스 자원을 효율적으로 사용하여 단위 시간당 처리되는 트랜잭션의 수를 높입니다.

#### 3. 고립성(Isolation) 보장
실행 중인 트랜잭션이 다른 트랜잭션의 중간 결과에 간섭받지 않도록 격리합니다.

```text
[동시성 제어의 논리적 구조]

  User A ──▶ [ Transaction 1 ] ──┐
                                 ├─▶ [ Concurrency Control ] ──▶ [ Consistent DB ]
  User B ──▶ [ Transaction 2 ] ──┘          (Manager)
```

---

### Ⅲ. 동시성 제어 부재 시 발생 가능한 문제점

| 문제점 | 설명 |
|:---|:---|
| **Lost Update** | 두 트랜잭션이 동시에 수정 시 이전 수정값이 사라짐 |
| **Inconsistency** | 일관성 없는 데이터를 읽어 발생하는 모순 |
| **Dirty Read** | 커밋되지 않은 데이터를 다른 트랜잭션이 읽음 |

- **📢 섹션 요약 비유**: 동시성 제어는 복잡한 교차로의 **'신호등'**과 같습니다. 차들이(트랜잭션) 멈추지 않고 빨리 지나가게 하면서도(처리량 극대화), 서로 충돌하지 않게(일관성 유지) 조절하는 필수 장치입니다.

---

### Ⅳ. 개념 맵 및 요약

- **[트랜잭션 ACID]**: 고립성과 일관성을 지키기 위한 기본 규칙.
- **[Locking 기법]**: 동시성 제어를 구현하는 물리적 방법 중 하나.
- **[MVCC]**: 최신 DB에서 성능을 위해 사용하는 다중 버전 제어 기술.

📢 **마무리 요약**: 동시성 제어는 **"안전한 병렬 처리"**를 가능케 하는 데이터베이스의 핵심 엔진입니다. 이를 통해 신뢰할 수 있는 대규모 서비스 운영이 가능해집니다.
