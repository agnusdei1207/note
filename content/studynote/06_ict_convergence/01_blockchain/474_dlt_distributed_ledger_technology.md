## 핵심 인사이트 (3줄 요약)

> 1. **본질**: DLT(Distributed Ledger Technology, 분산 원장 기술)는 중앙 서버 없이 다수 노드가 **동일한 원장 복사본을 공유·합의**하는 기술의 총칭으로, 블록체인은 DLT의 한 구현 형태다.
> 2. **가치**: CAP 정리(Consistency·Availability·Partition Tolerance 동시 불가)에서 DLT는 상황에 따라 CP(강한 일관성) 또는 AP(가용성 우선)를 선택하는 **설계 트레이드오프**를 명시화한다.
> 3. **판단 포인트**: 블록체인(선형 체인)·DAG(Directed Acyclic Graph)·HashGraph 등 구조 차이가 처리량(TPS)·최종성(Finality) 성능을 결정하므로, 활용 목적에 맞는 DLT 선택이 핵심이다.

---

## Ⅰ. 개요 및 필요성

### 중앙화 원장의 한계

기존 금융 시스템의 중앙화 원장(Centralized Ledger)은 단일 기관이 데이터를 독점 관리한다. 이 구조는 단일 장애점, 내부자 조작, 오프라인 시 서비스 중단 문제를 내포한다.

DLT는 원장을 **여러 노드에 분산·복제**하여 한 노드가 실패해도 서비스가 지속되고, 다수 노드가 합의해야만 원장을 수정할 수 있어 내부자 조작이 어렵다.

### DLT 분류 체계

```
┌─────────────────────────────────────────────┐
│             DLT 분류                         │
│                                             │
│  ┌─────────────────────────────────────┐   │
│  │  블록체인(Blockchain)               │   │
│  │  : 순차적 블록 연결, 해시 포인터     │   │
│  │  예) Bitcoin, Ethereum              │   │
│  └─────────────────────────────────────┘   │
│                                             │
│  ┌─────────────────────────────────────┐   │
│  │  DAG(Directed Acyclic Graph)        │   │
│  │  : 트랜잭션이 직접 서로 검증        │   │
│  │  예) IOTA Tangle, Nano              │   │
│  └─────────────────────────────────────┘   │
│                                             │
│  ┌─────────────────────────────────────┐   │
│  │  HashGraph                          │   │
│  │  : 가십 프로토콜 + 가상 투표        │   │
│  │  예) Hedera Hashgraph               │   │
│  └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

📢 **섹션 요약 비유** — *"중앙 은행 금고 하나에 모든 돈 장부가 있는 것 vs 전국 지점 모두가 동일한 장부 사본을 갖는 것이다."*

## Ⅱ. 아키텍처 및 핵심 원리

### DLT 주요 구현 비교

| 항목 | 블록체인 | DAG | HashGraph |
|:---|:---:|:---:|:---:|
| **구조** | 선형 블록 체인 | 비순환 방향 그래프 | 가십 이벤트 그래프 |
| **TPS** | 7~30 (BTC/ETH) | 수천 | 10,000+ |
| **최종성** | 확률적 | 확률적 | 결정적 |
| **에너지** | 높음(PoW) | 낮음 | 낮음 |
| **허가 여부** | 공개/비허가 | 공개/비허가 | 허가형 |

### CAP 정리와 DLT 설계

```
         C (Consistency, 일관성)
              /\
             /  \
            /    \
           /  CA  \
          /--------\
         / CP   AP  \
        /______________\
   A (Availability)    P (Partition Tolerance)
```

- **CP 모델**: PBFT(Practical BFT) 기반 Hyperledger Fabric → 파티션 시 가용성 희생, 일관성 보장
- **AP 모델**: 비트코인 → 일관성 최종성을 확률적으로 처리, 가용성 유지
- **실질적 분산 시스템**: P(Partition Tolerance)는 필수 → CA는 단일 노드에만 해당

📢 **섹션 요약 비유** — *"음식점 주방 메모판(중앙)을 없애고 모든 직원이 메모를 복사해 갖는 것. 한 명이 실수해도 나머지가 수정해 준다."*

## Ⅲ. 비교 및 연결

### 주요 DLT 플랫폼

| 플랫폼 | 유형 | 특징 |
|:---|:---|:---|
| **R3 Corda** | 허가형(Permissioned) | 금융 기관 전용, 트랜잭션 당사자만 공유 |
| **Hyperledger Fabric** | 허가형 | 채널(Channel) 격리, 체인코드(Chaincode) |
| **IOTA** | 비허가형 DAG | IoT 마이크로 트랜잭션, 수수료 없음 |
| **Hedera Hashgraph** | 허가형 HashGraph | 고TPS, 가십 프로토콜 |
| **Ethereum** | 비허가형 블록체인 | 스마트 컨트랙트 범용 플랫폼 |

**비블록체인 DLT의 차이점**: DAG는 블록 단위가 없으며 새 트랜잭션이 이전 트랜잭션 2개를 직접 검증하는 방식으로 합의한다. 트랜잭션 수가 많을수록 검증 속도가 빨라지는 **자기 확장성(Self-Scaling)**이 특징이다.

📢 **섹션 요약 비유** — *"블록체인은 기차처럼 한 줄로 가고, DAG는 고속도로처럼 여러 차선이 동시에 달린다."*

## Ⅳ. 실무 적용 및 기술사 판단

### 활용 도메인별 DLT 선택 기준

1. **금융 결제·무역 금융**: R3 Corda, Hyperledger Fabric (허가형, 프라이버시 보호)
2. **IoT 마이크로 결제**: IOTA DAG (수수료 없음, 높은 TPS)
3. **공개 DeFi(탈중앙화 금융)**: Ethereum, Solana (비허가형, 스마트 컨트랙트)
4. **공공 행정**: 허가형 블록체인 (데이터 접근 제어, 감사 Trail)

### 기술사 핵심 판단
- **"왜 블록체인만이 DLT가 아닌가?"**: DAG, HashGraph의 구조적 차이와 장단점 명시
- **CAP 트레이드오프**: 금융은 CP, IoT는 AP 선택이 합리적
- **Finality 종류**: 확률적 최종성(비트코인, k번 확인) vs 결정적 최종성(PBFT, Tendermint)

📢 **섹션 요약 비유** — *"은행 장부(R3 Corda)와 공개 게시판(Bitcoin)은 모두 분산 원장이지만, 쓰임새와 보안 모델이 완전히 다르다."*

## Ⅴ. 기대효과 및 결론

| 효과 항목 | 내용 |
|:---|:---|
| **신뢰 비용 절감** | 중개기관(Escrow, Clearing) 없는 P2P 거래 |
| **감사 가능성** | 변경 불가한 기록으로 규제 보고 자동화 |
| **글로벌 금융 포용** | 은행 계좌 없이도 DLT 지갑으로 금융 서비스 이용 |
| **데이터 주권** | 개인이 자신의 데이터를 직접 관리(SSI와 연계) |

DLT는 블록체인보다 넓은 개념이며, 사용 목적·허가 수준·합의 알고리즘에 따라 다양한 형태가 존재한다. 기술사는 DLT 종류를 용도에 맞게 선택하고 CAP 트레이드오프를 명확히 설명할 수 있어야 한다.

📢 **섹션 요약 비유** — *"'자동차'가 승용차·트럭·버스를 포괄하듯, DLT는 블록체인·DAG·HashGraph를 포괄하는 큰 범주다."*

### 👶 어린이를 위한 3줄 비유 설명

1. 반 친구 모두가 같은 공유 노트를 가지고 있어서, 선생님 없이도 내용을 수정하려면 반 전체 동의가 필요합니다.
2. 블록체인은 노트가 한 줄씩 이어지는 형태고, DAG는 여러 줄을 동시에 쓸 수 있는 형태예요.
3. 중앙에서 혼자 관리하면 그 사람이 거짓말해도 모르지만, 분산하면 다 같이 확인하니까 속이기 어렵습니다.
