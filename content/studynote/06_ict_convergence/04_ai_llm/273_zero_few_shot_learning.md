## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Zero-shot은 예시 없이 지시만으로 작업을 수행하는 것이며, Few-shot은 소량의 예시를 제공하여 모델의 성능을 보완하는 학습 기법이다.
> 2. **가치**: 모델 재학습 없이 프롬프트 구성만으로 즉각적인 성능 향상을 꾀할 수 있으며, 특히 복잡하거나 특수한 작업에서 Few-shot이 강력한 위력을 발휘한다.
> 3. **판단 포인트**: 모델이 이미 잘 아는 일반적 작업은 Zero-shot으로 효율을 높이고, 정밀한 형식이나 복잡한 논리가 필요한 경우 Few-shot을 선택해야 한다.

---

## Ⅰ. 개요 및 필요성

인간은 "개"와 "고양이" 사진을 한두 장만 봐도 새로운 사진을 구분할 수 있지만, 과거의 AI는 수만 장의 사진을 학습해야만 했다. Zero-shot과 Few-shot Learning은 AI가 인간처럼 적은 정보(또는 정보 없이도)만으로 새로운 문제를 해결할 수 있게 하려는 시도에서 시작되었다.

특히 대규모 언어 모델(LLM) 시대에 들어서면서, 사전 학습 과정에서 쌓은 방대한 지식을 인출하는 방법으로서 이 기법들은 프롬프트 엔지니어링의 표준이 되었다. 이는 데이터 수집 및 학습 비용을 획기적으로 줄여주는 실무적 도구다.

```text
┌──────────────────────────────────────────────────────────┐
│          Shot의 차이: 정보를 얼마나 주는가?                     │
├──────────────────────────────────────────────────────────┤
│ [Zero-shot] : "이거 해봐." (지시만)                         │
│ [One-shot]  : "이렇게 하는 거야, 이제 이거 해봐." (예시 1개)    │
│ [Few-shot]  : "이렇게, 저렇게 하는 거야. 이제 이거 해봐." (예시 N개)│
└──────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: Zero-shot은 천재에게 바로 문제를 던지는 것이고, Few-shot은 천재에게 기출문제 몇 개를 보여준 뒤 시험을 보게 하는 것이다.

---

## Ⅱ. 아키텍처 및 핵심 원리

이 기법들은 모델의 파라미터를 수정하지 않는 **In-context Learning**의 구체적인 구현 방식이다.

### 1. 프롬프트 구조 비교

| 기법 | 구조 예시 | 특징 |
|:---|:---|:---|
| **Zero-shot** | "다음 문장을 긍정/부정으로 분류해: 오늘 날씨 너무 좋아!" | 가장 간결함, 모델의 기본 성능에 의존 |
| **One-shot** | "감정 분류 예시: '슬퍼' -> 부정. 이제 다음을 분류해: '신나!'" | 최소한의 가이드 제공 |
| **Few-shot** | "예시1: '슬퍼'->부정, 예시2: '좋아'->긍정. 이제 다음을 분류해..." | 패턴 인식 극대화, 정확도 높음 |

### 2. 성능 결정 요인

*   **모델의 크기**: 모델이 클수록 Zero-shot 성능이 비약적으로 좋아진다.
*   **예시의 품질**: Few-shot에서 예시의 정답이 틀리더라도 패턴(형식)은 학습하지만, 정답까지 맞으면 성능이 최대로 발휘된다.
*   **유사성**: 예시가 실제 질문과 유사할수록(Semantic Similarity) 학습 효과가 크다.

- **📢 섹션 요약 비유**: Zero-shot은 맨몸으로 전장에 나가는 것이고, Few-shot은 든든한 장비를 챙겨 나가는 것이다.

## Ⅲ. 비교 및 연결

Zero-shot과 Few-shot은 비용과 성능의 트레이드오프 관계에 있다.

| 항목 | Zero-shot | Few-shot |
|:---|:---|:---|
| 토큰 소모 | 낮음 (경제적) | 높음 (예시만큼 비용 증가) |
| 지연 시간 (Latency) | 낮음 (빠름) | 상대적으로 높음 (입력이 길어짐) |
| 정확도 | 일반적 작업에서 우수 | 복잡한 작업, 특수 형식에서 압도적 |
| 편향성 | 낮음 | 예시의 편향에 영향을 받을 수 있음 |

최근에는 질문과 가장 유사한 예시를 벡터 DB에서 동적으로 찾아 프롬프트에 넣어주는 **Dynamic Few-shot** 기법이 RAG 시스템과 결합되어 자주 사용된다.

- **📢 섹션 요약 비유**: Zero-shot은 즉석 요리이고, Few-shot은 재료를 미리 손질해둔 밀키트 요리다.

---

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 Few-shot을 설계할 때는 예시의 개수보다 '예시의 다양성'이 중요하다. 비슷한 예시만 넣으면 모델이 편향된 답변을 낼 수 있다.

### 체크리스트
1. 모델이 질문의 의도를 자꾸 오해하는가? (Few-shot 전환 고려)
2. 출력 형식이 JSON 등 특정 포맷이어야 하는가? (Few-shot으로 형식 고정 필수)
3. 예시를 너무 많이 넣어서 질문이 잘리지는 않는가? (3~5개가 적당)
4. Zero-shot으로도 충분한 성능이 나오는가? (비용 절감을 위해 Zero-shot 유지)

### 안티패턴
- 예시의 정답 형식이 제각각인 경우 (예: '긍정', 'Positive', 'YES' 혼용)
- 질문과 전혀 상관없는 도메인의 예시를 넣는 경우

- **📢 섹션 요약 비유**: 똑똑한 아이라도 가이드가 엉망이면 엉뚱한 답을 하듯, 예시의 일관성이 곧 답변의 품질이다.

---

## Ⅴ. 기대효과 및 결론

Zero-shot과 Few-shot은 대규모 언어 모델을 '프로그래밍 가능한 지능'으로 만들었다. 코드를 짜서 로직을 구현하는 대신, 말과 예시로 로직을 주입하는 시대가 된 것이다. 이는 비전문가도 AI를 쉽게 활용할 수 있게 하는 핵심 기술이다.

앞으로 모델의 성능이 올라갈수록 Zero-shot의 영역이 넓어지겠지만, 아주 정교한 도메인 작업에서는 Few-shot이 여전히 최고의 성능을 보장하는 최후의 보루 역할을 할 것이다.

- **📢 섹션 요약 비유**: Zero-shot은 AI의 천부적인 재능이고, Few-shot은 그 재능을 꽃피우는 짧은 훈련이다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| In-context Learning | Zero/Few-shot이 작동하는 근본적인 학습 메커니즘 |
| 프롬프트 엔지니어링 | 좋은 Shot을 설계하기 위한 상위 기술 체계 |
| 창발적 능력 | 모델이 커질수록 Few-shot 학습 능력이 폭발적으로 증가함 |
| CoT (Chain of Thought) | Few-shot 예시에 '풀이 과정'을 추가한 응용 기법 |

### 👶 어린이를 위한 3줄 비유 설명

1. Zero-shot은 공부 안 하고도 문제를 척척 푸는 천재 로봇이에요.
2. Few-shot은 어려운 문제를 풀기 전에 비슷한 문제 몇 개를 먼저 풀어본 로봇이에요.
3. 문제 유형이 생소하고 어려울수록 Few-shot 로봇이 훨씬 더 점수가 높답니다.
