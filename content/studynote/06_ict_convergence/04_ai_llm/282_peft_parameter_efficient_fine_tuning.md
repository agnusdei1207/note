## 핵심 인사이트 (3줄 요약)
> 1. **본질**: PEFT (Parameter-Efficient Fine-Tuning)는 거대 모델의 수천억 개 파라미터 전체를 업데이트하는 대신, 극히 일부(1% 미만)만 학습시켜 효율을 극대화하는 파인 튜닝 기술군이다.
> 2. **가치**: 모델 학습에 필요한 GPU 메모리, 컴퓨팅 자원, 저장 공간을 획기적으로 줄이면서도 전체 파인 튜닝과 대등한 성능을 유지한다.
> 3. **판단 포인트**: 기업 내부의 한정된 자원으로 여러 개의 특화 모델을 구축해야 하거나, 하드웨어 비용 절감이 절실할 때 필수적으로 선택해야 하는 전략이다.

---

## Ⅰ. 개요 및 필요성

대규모 언어 모델(LLM)이 커질수록 전체 파인 튜닝(Full Fine-tuning)의 장벽은 높아진다. 175B(1,750억 개) 파라미터 모델을 튜닝하려면 수백 대의 GPU가 필요하고, 저장 공간 문제로 인해 다양한 파생 모델을 관리하기가 불가능에 가깝다.

PEFT는 이러한 '비용과 규모의 문제'를 해결하기 위해 등장했다. 모델의 대부분은 고정(Freeze)하고, 아주 일부의 매개변수만 새로 추가하거나 학습시킴으로써 소량의 데이터와 자원만으로도 모델을 특정 목적에 맞게 진화시킨다.

```text
┌──────────────────────────────────────────────────────────┐
│           PEFT의 핵심: "최소 자원으로 최대 효과"                │
├──────────────────────────────────────────────────────────┤
│ [기존 방식] : 엔진 전체를 분해해서 새로 조립 (무겁고 비쌈)      │
│ [PEFT 방식] : 엔진은 그대로 두고 연료 필터만 교체 (가볍고 빠름)    │
└──────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: PEFT는 큰 도서관의 책들을 다 새로 쓰는 대신, 책 뒤에 짧은 부록(새 지식)만 추가하여 도서관 전체의 지식을 업데이트하는 것과 같다.

## Ⅱ. 아키텍처 및 핵심 원리

PEFT는 접근 방식에 따라 크게 네 가지 카테고리로 분류된다.

### 1. PEFT의 주요 기술 분류

| 카테고리 | 특징 | 대표 기술 |
|:---|:---|:---|
| **가산적 (Additive)** | 모델에 새로운 레이어나 파라미터 추가 | Adapters, Prompt Tuning |
| **선택적 (Selective)** | 기존 모델의 일부 레이어만 학습 | BitFit |
| **재매개변수화 (Reparameterization)** | 가중치 행렬을 저차원으로 분해 | **LoRA** |
| **소프트 프롬프트 (Soft Prompts)** | 입력 앞단에 학습 가능한 벡터 배치 | Prefix Tuning |

### 2. 학습 효율성 비교 (예시)

```text
[전체 파인 튜닝]
■■■■■■■■■■■■■■■■ (100% 학습) -> 500GB VRAM 필요

[PEFT 적용]
□□□□□□□□□□□□□□□■ (0.1% 학습)  -> 40GB VRAM으로 가능
```

- **📢 섹션 요약 비유**: 집 전체를 새로 짓는 대신, 필요한 방 한 칸만 도배해서 새 집 같은 기분을 내는 전략이다.

---

## Ⅲ. 비교 및 연결

PEFT는 단순히 비용 절감 도구가 아니라 모델의 생태계를 구축하는 기반 기술이다.

| 항목 | Full Fine-tuning | PEFT (LoRA 등) |
|:---|:---|:---|
| 모델링 능력 | 이론상 최강 | 전체 튜닝의 95~99% 수준 도달 |
| 하드웨어 요구사항 | 하이엔드 서버 클러스터 | 단일 워크스테이션 또는 일반 GPU |
| 망각 현상 | 심각할 수 있음 (전체 왜곡) | 원본 지식 보존 능력이 뛰어남 |
| 배포 유연성 | 낮음 (모델 통째로 배포) | 높음 (작은 어댑터 파일만 배포) |

PEFT의 등장으로 인해, 하나의 거대 모델을 공통으로 쓰면서 각 부서나 서비스마다 작은 '어댑터'만 갈아 끼워 사용하는 **Multi-Adapter 서빙** 구조가 가능해졌다.

- **📢 섹션 요약 비유**: Full Fine-tuning이 모델을 통째로 복사해서 전달하는 것이라면, PEFT는 원본은 두고 '수정 사항 리스트'만 전달하는 것이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

실무에서는 어떤 PEFT 기법을 쓸지 결정해야 한다. 현재 업계 표준은 **LoRA**이지만, 입력 길이가 짧고 단순한 작업이라면 **Prompt Tuning**이 더 빠를 수 있다.

### 체크리스트
1. 가용 GPU 메모리가 모델 전체를 올리기에 부족한가? (PEFT 필수)
2. 실시간으로 여러 작업을 전환하며 처리해야 하는가? (PEFT 유리)
3. 원본 모델의 범용 지식을 반드시 지켜야 하는가? (PEFT가 더 안전함)
4. 학습 데이터가 아주 적은가? (PEFT가 과적합 방지에 유리)

### 안티패턴
- PEFT 성능이 조금 낮다고 해서 무작정 전체 튜닝을 시도하다 프로젝트 자금을 탕진하는 경우
- 특정 PEFT 기법(예: Prefix Tuning)이 모델의 입력 한도(Context Window)를 줄이는 것을 계산하지 못한 경우

- **📢 섹션 요약 비유**: 닭 잡는 데 소 잡는 칼(Full Fine-tuning)을 쓸 필요가 없다. 작은 칼(PEFT)로도 충분히 훌륭한 요리를 할 수 있다.

---

## Ⅴ. 기대효과 및 결론

PEFT는 '지속 가능한 AI'를 위한 필수 기술이다. 환경적, 비용적 측면에서 거대 모델의 전체 학습은 더 이상 지속 가능하지 않다. PEFT를 통해 중소기업이나 개인 연구자들도 최첨단 LLM의 성능을 자신의 도메인에 이식할 수 있게 되었다.

앞으로는 모델의 크기가 조 단위로 커져도 PEFT를 통해 개별 스마트폰이나 엣지 기기에서도 실시간 파인 튜닝이 가능한 시대가 올 것이다.

- **📢 섹션 요약 비유**: PEFT는 AI라는 거대한 불꽃을 우리가 원하는 작은 등불로 옮겨 담아 실생활에 유용하게 쓰는 기술이다.
