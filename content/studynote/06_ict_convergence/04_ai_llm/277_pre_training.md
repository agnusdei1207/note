## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사전 학습(Pre-training)은 거대한 규모의 말뭉치를 사용하여 언어의 구조, 문맥, 지식을 모델에 스스로 학습시키는 초기 단계다.
> 2. **가치**: 모델이 '세상이 어떻게 돌아가는지'에 대한 기초 지능을 형성하며, 이후 수행할 모든 미세 조정(Fine-tuning)의 토대가 된다.
> 3. **판단 포인트**: 엄청난 컴퓨팅 자원과 데이터가 필요하므로 일반 기업은 직접 수행하기보다 잘 만들어진 'Pre-trained 모델'을 선택하는 것이 핵심 전략이다.

---

## Ⅰ. 개요 및 필요성

인간이 태어나서 언어를 배우고 세상을 이해하는 과정처럼, AI 모델도 본격적인 업무(번역, 요약 등)를 배우기 전에 '언어 그 자체'를 마스터해야 한다. 사전 학습은 레이블(정답)이 없는 방대한 인터넷 데이터를 스스로 읽고, 다음에 올 단어를 맞히는 과정을 통해 지식을 습득한다.

이 과정이 없다면 모델은 문장의 맥락을 이해하지 못하고 단순한 패턴 매칭만 수행하게 된다. 사전 학습을 통해 형성된 '파운데이션 모델(Foundation Model)'은 강력한 범용성을 가지며, 아주 적은 추가 학습만으로도 수많은 작업을 수행할 수 있는 잠재력을 갖춘다.

```text
┌──────────────────────────────────────────────────────────┐
│          사전 학습의 역할: "기초 지능"의 형성과 지식 축적        │
├──────────────────────────────────────────────────────────┤
│ [학습 데이터] : 위키피디아, 책, 코드, 뉴스 (수조 개의 단어)      │
│ [학습 방식]   : 다음 단어 맞히기 (Self-supervised Learning)  │
│ [결과물]      : 문법, 상식, 논리력을 갖춘 기초 모델            │
└──────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: 사전 학습은 학교에 가기 전 집에서 수만 권의 책을 읽어 세상 물정을 다 깨우친 아이와 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

사전 학습은 주로 자기지도 학습(Self-supervised Learning) 방식을 사용하며, 대규모 병렬 처리가 가능한 트랜스포머 아키텍처를 기반으로 한다.

### 1. 사전 학습의 주요 방식

| 방식 | 설명 | 대표 모델 |
|:---|:---|:---|
| **Causal LM (Decoder)** | 이전 단어들을 보고 다음 단어를 예측 (Autoregressive) | GPT 시리즈 |
| **Masked LM (Encoder)** | 문장 중간에 빈칸을 뚫고 주변 맥락으로 맞히기 | BERT |
| **Seq2Seq (Encoder-Decoder)** | 입력 문장을 읽고 새로운 문장으로 생성 | T5, BART |

### 2. 학습 파이프라인

```text
  [데이터 수집] : 웹 크롤링, 도서 데이터, 코드 (Raw Data)
         │
         ▼
  [데이터 정제] : 중복 제거, 저품질 텍스트 필터링, 개인정보 삭제
         │
         ▼
  [토큰화] : 텍스트를 숫자로 변환 (Byte Pair Encoding 등)
         │
         ▼
  [대규모 학습] : 수천 개의 GPU를 동원하여 수개월간 연산
         │
         ▼
  [Pre-trained Model] : 완성된 파운데이션 모델
```

- **📢 섹션 요약 비유**: 엄청난 양의 밀가루(데이터)를 반죽하고 발효시켜서, 어떤 빵이든 만들 수 있는 최고의 '기본 반죽(사전 학습 모델)'을 만드는 과정이다.

## Ⅲ. 비교 및 연결

사전 학습과 파인 튜닝은 '학습'이라는 이름은 같지만 규모와 목적이 완전히 다르다.

| 항목 | 사전 학습 (Pre-training) | 파인 튜닝 (Fine-tuning) |
|:---|:---|:---|
| 데이터 성격 | 정답 없는 방대한 데이터 (Raw) | 정답 있는 정제된 데이터 (Labeled) |
| 데이터 규모 | 테라바이트(TB) 단위 | 메가바이트(MB) ~ 기가바이트(GB) 단위 |
| 소요 자원 | 수만 개의 GPU, 수개월 | 수개~수십 개의 GPU, 수시간/수일 |
| 목적 | 범용 지능 형성 | 특정 작업(Task) 전문화 |

최근에는 사전 학습을 마친 모델을 그대로 가져와서 파인 튜닝조차 하지 않고 쓰는 **In-context Learning** 기법도 널리 쓰이고 있다. 이는 사전 학습 모델의 능력이 그만큼 강력해졌음을 의미한다.

- **📢 섹션 요약 비유**: 사전 학습이 대학 공통 교양 과정을 이수하는 것이라면, 파인 튜닝은 전공 심화 과정을 듣는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서는 사전 학습을 직접 하기보다는 기업의 목적에 맞는 '베이스 모델'을 잘 고르는 것이 더 중요하다. 모델의 파라미터가 크다고 무조건 좋은 것이 아니라, 학습 데이터의 언어 비중과 도메인을 확인해야 한다.

### 체크리스트
1. 한국어 능력이 중요한가? (한국어 데이터 비중이 높은 모델 선택)
2. 코딩 능력이 중요한가? (Code-specific 사전 학습 모델 선택)
3. 상용 서비스로 배포할 것인가? (추론 속도를 위해 경량화된 모델 고려)
4. 라이선스가 자유로운가? (Llama 3, Mistral 등 오픈소스 라이선스 확인)

### 안티패턴
- 데이터 정제 없이 무작정 많은 양의 데이터를 사전 학습에 쏟아붓는 경우 (Garbage In, Garbage Out)
- 사전 학습 단계에서 편향된 데이터(혐오 표현 등)가 섞여 들어가는 것을 방치하는 경우

- **📢 섹션 요약 비유**: 재료가 상해 있으면 아무리 훌륭한 요리사라도 맛있는 음식을 만들 수 없다. 데이터 정제가 사전 학습의 80%다.

---

## Ⅴ. 기대효과 및 결론

사전 학습은 현대 AI 혁명의 심장이다. 이 단계에서 축적된 지능이 있었기에 챗GPT와 같은 혁신적인 서비스가 가능했다. 최근에는 텍스트뿐만 아니라 이미지, 영상, 센서 데이터까지 한꺼번에 사전 학습하는 '멀티모달 사전 학습'으로 진화하고 있다.

개별 기업은 직접 사전 학습을 하기보다는, 오픈소스 파운데이션 모델을 활용하여 자신들만의 독자적인 가치를 더하는 전략을 취하는 것이 가장 효율적이다.

- **📢 섹션 요약 비유**: 사전 학습은 AI가 인간의 문명을 통째로 읽고 이해하여 거대한 지식의 산맥을 쌓는 장엄한 과정이다.
