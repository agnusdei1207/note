# LIME / SHAP 지표 (부분적 / 게임이론 전역적 변수 기여 해석)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: LIME과 SHAP은 블랙박스 AI 모델(Random Forest, 딥러닝 등)의 의사결정 근거를 사후에 해석(Post-hoc)하는 XAI의 양대 산맥 알고리즘으로, 각 입력 변수(Feature)가 예측 결과에 얼마나 기여했는지를 정량화하여 보여준다.
> 2. **가치**: LIME은 단일 예측 결과 주변(Local)에 단순한 선형 모델을 근사시켜 매우 빠르게 해석을 제공하므로 실시간 고객 응대에 적합하며, SHAP은 게임이론 기반의 섀플리 값(Shapley Value)을 통해 이론적 일관성을 갖춘 전역적(Global)·지역적 기여도 분석이 가능해 기업의 AI 규제 컴플라이언스 준수와 공정성 입증에 필수적이다.
> 3. **융합**: 실무 파이프라인에서는 처리 속도가 빠른 LIME을 실시간 트래픽 환경의 단위 디버깅용으로, 연산량이 방대하지만 수학적 완결성이 보장되는 SHAP을 배치(Batch) 성격의 전체 모델 공정성 검증 및 리포팅 시스템과 융합하여 상호 보완적으로 운용한다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: 기계학습 모델의 설명 가능성(XAI)을 확보하기 위해 가장 널리 쓰이는 두 가지 사후 해석 도구(Model-Agnostic Tools)이다. 모델의 종류(구조)에 상관없이 입출력 데이터만 주어지면 작동한다. `LIME` (Local Interpretable Model-agnostic Explanations)은 특정 데이터 근처에서 선형 근사(Linear Approximation)를 통해 특징 중요도를 도출하고, `SHAP` (SHapley Additive exPlanations)은 협조적 게임이론을 차용하여 모든 변수 조합의 기여도를 공평하게 분배하는 수학적 방법론이다.
- **필요성**: 은행 신용평가 AI가 고객 A의 대출을 한도 3천만 원으로 거절했을 때, 상담원은 고객에게 이유를 설명해야 한다. 블랙박스 모델은 "거절 확률 85%"라는 결괏값만 내놓는다. 이때 LIME이나 SHAP을 돌리면 "거절 이유: 연봉 변수가 거절 확률을 40% 높였고, 최근 연체 이력 변수가 30%를 높였음"이라는 구체적인 변수별 기여도(Feature Contribution) 수치를 얻을 수 있어 비즈니스 신뢰도와 규제 준수성을 동시에 확보한다.
- **💡 비유**: 팀(변수들)이 축구 경기에서 승리(최종 예측치)했을 때, 팀원 각자가 승리에 얼마나 기여했는지 연봉 협상을 위해 평가하는 과정입니다. LIME은 방금 뛴 단 한 경기(특정 데이터)의 활약상만 보고 빠르게 평가하는 코치라면, SHAP은 선수가 뛰었을 때와 뛰지 않았을 때의 모든 경기 기록을 수학적으로 분석해 가장 공평한 공헌도를 계산하는 깐깐한 데이터 분석가와 같습니다.
- **등장 배경**:
  1. **딥러닝과 앙상블 모델의 대중화**: 트리기반 앙상블 기법(XGBoost 등)과 딥러닝이 높은 성능을 냈지만 변수 간 비선형적 상호작용으로 인해 기존의 중요도(Feature Importance) 지표가 왜곡되는 현상 발생.
  2. **LIME의 등장 (2016)**: 논문 "Why Should I Trust You?"를 통해 모델 구조에 독립적으로 단일 예측을 설명할 수 있는 돌파구가 제시됨.
  3. **SHAP의 등장 (2017)**: LIME의 지역적 불안정성(샘플링 변화에 따라 설명이 달라짐)을 극복하고자, 1950년대 노벨 경제학상을 받은 로이드 섀플리의 '게임이론'을 머신러닝에 접목하여 수학적 일관성을 증명한 SHAP 제안.

LIME과 SHAP이 어떻게 변수 기여도를 분배하는지 대출 거절 사례의 Waterfall 차트 개념으로 시각화하면 직관적인 이해가 가능하다.

```text
  ┌─────────────────────────────────────────────────────────────┐
  │         변수 기여도 시각화 예시 (SHAP / LIME Force Plot 개념)     │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │  [상황: 고객 A의 대출 승인 점수가 기준선(Base) 50점에서 시작하여   │
  │   최종 30점으로 예측(거절)된 이유 분석]                             │
  │                                                             │
  │  기준(Base Value): 평균 승인 점수 50점                          │
  │                                                             │
  │            (점수 하락 요인 ◀)   (▶ 점수 상승 요인)             │
  │       20      30      40      50      60      70           │
  │   ────┼───────┼───────┼───────┼───────┼───────┼─────       │
  │           ▲              │                                 │
  │      최종 예측값(30)      │                                 │
  │                           │                                 │
  │  █████████████████████◀── │ ──▶██████████                    │
  │   연체 이력 있음 (-25점)        고소득자 (+10점)                 │
  │                                                             │
  │  █████◀──────────────────                                     │
  │  재직 기간 짧음 (-5점)                                          │
  │                                                             │
  │  * 해석: 이 고객은 '고소득(+10)'이라는 긍정적 요인이 있었으나,       │
  │    '연체 이력(-25)'과 '재직 기간 부족(-5)'이라는 부정적 요인이      │
  │    이를 압도하여 최종적으로 평균(50)보다 낮은 30점으로 대출이 거절됨.  │

```

**[다이어그램 해설]** 이것은 실무에서 XAI를 도입할 때 가장 많이 구축하는 Force Plot 시각화의 전형적인 구조다. 블랙박스 모델이 "이 사람의 점수는 30점"이라고 던져준 결과를 두고, XAI(LIME/SHAP)는 모델의 베이스라인(Base Value, 모델의 평균 예측치)에서 출발하여 각 특징(Feature)이 점수를 음(-)의 방향으로 밀었는지, 양(+)의 방향으로 밀었는지 힘겨루기(Force) 형태로 분해한다. 고객에게 거절 사유를 명확한 수치 근거 기반으로 설명할 수 있으므로, 금융/공공 분야에서 컴플라이언스(GDPR 등의 설명 요구권)를 만족시키는 강력한 도구가 된다.

- **📢 섹션 요약 비유**: 성적표에 '총점 60점'만 찍혀 나오는 것이 아니라, '수학 때문에 -20점 깎였고, 영어 덕분에 +10점 올랐다'는 식으로 총점의 가감 요소를 항목별로 영수증처럼 뽑아주는 기능입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### LIME과 SHAP의 핵심 메커니즘 비교

| 지표명 | LIME (부분적/지역적 선형 근사) | SHAP (전역적/게임이론 기반 기여도) |
|:---|:---|:---|
| **기반 이론** | 대상 데이터 근처를 무작위 교란(Perturbation) 시켜 수집한 점들로 가중치 기반 선형 회귀 생성 | 협조적 게임이론의 섀플리 값 (변수 조합의 부분집합을 모두 계산하여 한계 기여도 평균 도출) |
| **장점** | 연산 속도가 매우 빠름, 비전/텍스트 직관적 설명 좋음 | 수학적 일관성(정확한 분배) 보장, Local과 Global 설명 모두 가능 |
| **단점 / 한계**| 난수 샘플링에 의존해 매번 결과가 조금씩 달라질 수 있음(불안정성), 비선형성이 심하면 정확도 하락 | 모든 변수 조합(2^N)을 계산해야 하므로 연산량(계산 비용)이 폭발적으로 높음 |

---

### 심층 원리 1: LIME (Local Interpretable Model-agnostic Explanations)

비선형적이고 복잡한 모델이라도, "아주 좁은 국소 부위(Local)"만 잘라내어 돋보기로 들여다보면 직선(선형 모델)으로 근사할 수 있다는 미적분학의 원리에서 착안했다.

```text
  ┌──────────────────────────────────────────────────────────────────┐
  │                 LIME의 국소적 선형 근사 메커니즘                     │
  ├──────────────────────────────────────────────────────────────────┤
  │                                                                  │
  │  [블랙박스 모델의 결정 경계 (비선형의 복잡한 곡선)]                     │
  │                                                                  │
  │   Y 변수                                                         │
  │    │           복잡한 경계선 (빨간영역/파란영역)                     │
  │    │     🔴 🔴 /         🔵                                      │
  │    │       🔴 /     ┌─(해석 대상 데이터 X)─┐                      │
  │    │         /      │          🔵          │                      │
  │    │       /        │  🔵     🌟      🔵   │                      │
  │    │      /         │          🔵          │                      │
  │    │     /          └──────────────────────┘                      │
  │    │    / 🔴           ↖ 이 좁은 박스 안에서는 곡선이 직선처럼 보임     │
  │    └──────────────────────────────────────────────────────── X 변수 │
  │                                                                  │
  │  [LIME 작동 순서]                                                  │
  │  1. 해석하고자 하는 타겟 데이터(🌟) 선택                             │
  │  2. 주변에 무작위 변동(Perturbation)을 주어 가짜 데이터(🔵) 생성         │
  │  3. 블랙박스 모델로 이 가짜 데이터들의 결과를 모두 예측                   │
  │  4. 타겟(🌟)과 가까울수록 높은 가중치를 주어 단순한 '선형 회귀선' 적합       │
  │  5. 도출된 선형 회귀선의 가중치(기울기)를 각 변수의 중요도로 제공           │
  └──────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 전체 데이터 공간(Global)에서 보면 딥러닝 모델의 결정 경계는 심하게 휘어진 곡선(블랙박스)이다. 하지만 LIME은 해석하려는 단일 타겟 데이터(🌟) 근처(Local)에 집중한다. 타겟 변수값을 조금씩 바꾼 가짜 데이터들을 무수히 생성(Perturbation)하고, 이 근방에서만 예측 결괏값을 바탕으로 가장 잘 맞는 단순한 직선(선형 회귀, White-box)을 긋는다. 선형 모델의 특성상 각 변수의 계수(Coefficient)가 곧 기여도를 뜻하므로 결과를 즉시 해석할 수 있다. 연산이 가벼운 대신, 타겟 주변 범위를 어떻게 설정하느냐에 따라 매번 그려지는 직선의 각도가 달라지는 안정성(Stability) 이슈가 있다.

### 심층 원리 2: SHAP (SHapley Additive exPlanations)

SHAP은 노벨상 수상 알고리즘인 섀플리 값(Shapley Value)을 기반으로 한다. "변수 집합에서 변수 A가 추가되었을 때 예측값이 얼마나 변하는가(한계 기여도)를, 모든 가능한 변수 조합에 대해 구한 뒤 평균을 낸 값"이다.

```text
  ┌──────────────────────────────────────────────────────────────────┐
  │               SHAP: 게임이론 기반 섀플리 값 계산 원리                 │
  ├──────────────────────────────────────────────────────────────────┤
  │                                                                  │
  │  [상황: 아파트 가격 예측. 변수는 3개 (면적, 층수, 역세권)]                │
  │   - 베이스라인 집값: 5억 원                                        │
  │   - 대상 아파트 예측치: 8억 원 (+3억 기여분을 세 변수가 나눠 가져야 함)     │
  │                                                                  │
  │  [역세권(변수 C)의 한계 기여도(Marginal Contribution) 계산]          │
  │   변수 C가 합류할 수 있는 모든 팀(조합) 시나리오:                        │
  │                                                                  │
  │   1. 아무도 없을 때 C 혼자 참여:  V({C}) - V(∅)                      │
  │   2. '면적(A)'만 있을 때 C 참여: V({A,C}) - V({A})                  │
  │   3. '층수(B)'만 있을 때 C 참여: V({B,C}) - V({B})                  │
  │   4. 'A,B' 둘 다 있을 때 C 참여: V({A,B,C}) - V({A,B})              │
  │                                                                  │
  │  [SHAP Value 도출 공식의 핵심]                                      │
  │  SHAP(C) = 위 4가지 시나리오의 기여도를 조합 비율에 맞춰 '가중 평균'한 값     │
  │                                                                  │
  │  * 결과: 3억 원 초과분 중 [면적: 1.5억, 역세권: 1억, 층수: 0.5억] 식으로     │
  │    단 1원의 오차도 없이 완벽하게(Additive property) 분배됨.             │
  └──────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 변수가 3개면 조합의 수는 2^3 = 8개다. SHAP은 특정 변수(여기선 역세권)가 포함되지 않았을 때의 예측값과, 포함되었을 때의 예측값 차이(한계 기여도)를 모든 가능한 조합 상황을 고려하여 평균 낸다. 이 수학적 철저함 덕분에 SHAP은 "각 변수의 SHAP 값을 모두 더하면 반드시 (최종 예측값 - 베이스라인 값)과 일치한다"는 Additive 특성(완결성)을 갖게 된다. LIME은 이런 일관성이 없다. 다만 변수가 100개면 2^100번 계산을 해야 하므로 연산 비용이 지수적으로 폭발하며, 실무에서는 이를 근사치로 계산하는 TreeSHAP이나 KernelSHAP을 사용한다.

- **📢 섹션 요약 비유**: LIME이 특정 지역(Local)의 지도를 대충 그려 길을 빨리 찾는 나침반이라면, SHAP은 지구 전체(Global)를 인공위성으로 정밀 스캔하여 오차율 0%로 영토 경계를 정확히 나눠주는(분배) 측량 시스템과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### LIME vs SHAP 실무 관점 상세 비교 매트릭스

| 비교 항목 | LIME | SHAP | 의사결정 시사점 |
|:---|:---|:---|:---|
| **수학적 견고성** | 낮음 (휴리스틱한 샘플링 기반) | **매우 높음** (협조적 게임이론 기반) | 규제/법적 증빙에는 SHAP 필수 |
| **연산 속도** | **매우 빠름** | 지수적 증가 (느림, TreeSHAP으로 보완) | 실시간 스트리밍 대용량 트래픽엔 LIME |
| **분석 범위 (Scope)** | Local 전용 (단일 건 해석) | **Local 및 Global (전체 데이터) 동시 해석** | 피처 중요도(전체 모델 시야) 분석 시 SHAP |
| **일관성 (Consistency)**| 떨어짐 (동일 모델/입력에 결과 바뀔 수 있음) | 완벽함 (Additive, Symmetry 속성 보장) | 의료, 금융 등 보수적 도메인은 SHAP 우위 |

최근의 기술 트렌드는 비즈니스 요건에 맞춰 두 기술을 융합하는 것이다. Tree 앙상블 모델(LightGBM, XGBoost)을 사용할 경우 SHAP의 연산 복잡도 문제를 트리 전용 알고리즘인 `TreeSHAP`으로 극복할 수 있어 SHAP의 점유율이 압도적으로 높아졌다. 반면, 복잡한 CNN 기반 이미지 처리나 자연어 처리 파이프라인 중 즉석 디버깅 시에는 빠르고 가벼운 LIME이 여전히 강점을 가진다.

- **📢 섹션 요약 비유**: 빠르고 간단하게 오늘 하루 매출이 오른 이유를 유추할 때는 LIME(일일 보고서)을 쓰고, 연말에 직원 보너스를 지급하기 위해 1년 치 성과를 1원 단위로 정확하고 공평하게 분배할 때는 SHAP(회계 감사)을 씁니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 적용 시나리오 및 최적화 전략

1. **시나리오 — 데이터 사이언티스트의 모델 디버깅 및 편향(Bias) 진단**: 100개의 피처(Feature)를 넣어 집값 예측 XGBoost 모델을 훈련시켰다. 이 모델이 특정 변수(예: '성별' 또는 '우편번호')에 과도하게 의존하는 편향성이 있는지 검증해야 한다.
   - **판단**: LIME으로는 불가능하다. 데이터 전체의 경향성을 봐야 하므로 **SHAP의 Summary Plot (Global Interpretation)**을 추출한다. X축을 SHAP Value, Y축을 변수로 두고 모든 데이터를 점으로 찍었을 때, 특정 민감 변수의 점들이 크게 분포하면 해당 변수를 제거하고 모델을 재학습해야 한다. 이를 통해 Data Leakage나 과적합(Overfitting)을 사전에 색출할 수 있다.

2. **시나리오 — 실시간 이상 탐지(FDS) 시스템의 관제 대시보드 연동**: 수만 건의 결제 트래픽 중 이상 결제로 탐지된 건에 대해 관제 요원이 10초 안에 승인/차단 여부를 눈으로 보고 판단해야 한다.
   - **판단**: 연산 지연(Latency)이 치명적이므로 일반 KernelSHAP은 사용할 수 없다. 모델이 딥러닝이라면 **LIME**을 비동기(Asynchronous)로 돌려 빠른 피드백을 제공하거나, 모델이 XGBoost 계열이라면 연산 속도가 최적화된 **TreeSHAP**을 활용하여 즉각적인 Force Plot을 UI에 띄워주는 아키텍처를 구성한다.

SHAP을 통해 글로벌(전역) 해석을 수행하여, 실무진이 모델의 변수 중요도를 파악하는 Summary Plot의 의미를 시각화한다.

```text
  ┌───────────────────────────────────────────────────────────────────┐
  │              SHAP Summary Plot (글로벌 변수 중요도 시각화)           │
  ├───────────────────────────────────────────────────────────────────┤
  │                                                                   │
  │   변수명 (Y축: 영향력이 큰 순으로 정렬)    SHAP Value (X축: 모델 예측 영향력) │
  │                                    ◀ 예측값 감소 │ 예측값 증가 ▶    │
  │                                                                   │
  │  1. 연체 기간 (Days_Past_Due)           🔵🔵 🔴🔴│     🔴🔴🔴🔴🔴      │
  │                                              │                    │
  │  2. 연소득 (Annual_Income)      🔵🔵🔵🔵       │🔵🔴🔴              │
  │                                              │                    │
  │  3. 나이 (Age)                        🔵🔵🔴 │ 🔴🔵               │
  │                                                                   │
  │  [색상(Color) 범례: 변수의 실제 값]                                    │
  │  🔴 빨간색 점 = 해당 변수의 실제 값이 큼 (예: 고소득, 연체 김)             │
  │  🔵 파란색 점 = 해당 변수의 실제 값이 작음 (예: 저소득, 연체 짧음)          │
  │                                                                   │
  │  * 해석 1: '연체 기간' 변수가 예측(대출 거절)에 가장 큰 영향을 미침.       │
  │  * 해석 2: 연체 기간이 길수록(🔴) 우측(거절 점수 증가)에 넓게 분포하며,     │
  │           연소득이 높을수록(🔴) 좌측(거절 점수 감소, 즉 승인)에 기여함을   │
  │           한눈에 파악하여 모델의 논리가 정상적임을 검증 가능.            │
  └───────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** SHAP Summary Plot은 XAI가 단순한 사후 설명을 넘어, 피처 엔지니어링(Feature Engineering)과 모델 튜닝의 나침반 역할을 함을 보여준다. Y축에 변수들이 중요도 순으로 나열되므로 기존의 단순 피처 중요도 그래프와 유사해 보이지만, SHAP은 각 데이터 포인트(점)의 색상을 통해 "값의 높낮이가 양의 방향으로 작용했는지 음의 방향으로 작용했는지" 상관관계까지 입체적으로 보여준다. 만약 어떤 변수가 값이 큰데(🔴) 좌우(🔵,🔴)로 마구잡이로 찍혀 있다면, 그 변수는 노이즈일 확률이 높아 모델에서 제거(Pruning)하는 의사결정을 내릴 수 있다.

### 안티패턴
- **SHAP의 무분별한 딥러닝 적용 (KernelSHAP 오남용)**: 수백 개의 피처를 가진 무거운 DNN 모델에, 백그라운드 샘플 데이터까지 1000개 이상 설정한 채로 KernelSHAP 연산을 돌리면 서버 리소스가 고갈되고 OOM(Out of Memory) 에러로 시스템이 다운된다. 딥러닝에는 `DeepSHAP`을 쓰거나 가벼운 LIME으로 선회해야 한다.
- **인과관계와의 혼동**: SHAP 값이 크다고 해서 "이 변수를 억지로 바꾸면 결과도 그만큼 확실히 바뀐다(인과성)"고 단정하면 안 된다. SHAP과 LIME은 관측된 데이터 기반의 '상관적 민감도'일 뿐이다.

- **📢 섹션 요약 비유**: SHAP 대시보드는 비행기 조종석의 '계기판'과 같습니다. 복잡한 엔진(모델)의 내부 상태를 고도, 속도, 연료량(변수 기여도)으로 변환해 주어, 기장이 비행기가 정상 항로로 날고 있는지(편향 여부) 한눈에 확인하고 안심하게 해줍니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 지표/효과 | 기존 Feature Importance 방식 | LIME / SHAP 적용 시 |
|:---|:---|:---|:---|
| **설명 해상도** | 글로벌(Global) | 전체 모델 경향성만 파악 (건별 원인 불명) | 전체 경향(SHAP) + **건별 맞춤형 해설(Local)** 동시 제공 |
| **수학적 신뢰성**| 일관성/완결성 | 트리 깊이에 따라 중요도 왜곡 발생 가능 | **섀플리 값 보장**으로 오차 없이 100% 분배 정합성 |
| **비즈니스 활용**| 고객 경험(UX) | "AI가 거절했습니다" 수준의 통보 | 명확한 수치(+30점 요인 등) 제공으로 투명성 제고 및 민원 억제 |

### 미래 전망
- **알고리즘 최적화**: SHAP의 최대 단점인 느린 연산 속도를 극복하기 위해, 트리 구조에 한정된 TreeSHAP을 넘어 신경망(Deep Learning)에 특화된 고속 DeepSHAP, 텍스트 모델 특화 TextExplainer 등의 최적화 파이프라인 개발이 XAI 프레임워크 생태계(Captum, Alibi 등)를 중심으로 고도화되고 있다.
- **LLM과 XAI 결합**: SHAP/LIME이 계산해 낸 배열 데이터를 대형 언어 모델(LLM)에 먹여, "당신의 대출은 A가 30% 영향을 주어 거절되었습니다"라는 딱딱한 텍스트를 "고객님, 최근 3개월 연체 기록 때문에 심사 점수가 부족했습니다. 내년에 다시 신청해 주세요"라는 부드럽고 자연스러운 언어로 자동 해설해 주는 생성형 XAI 솔루션이 차세대 핀테크 표준으로 자리 잡을 것이다.

- **📢 섹션 요약 비유**: 아무리 화려한 마술(AI 알고리즘)이라도 사람들을 오래 설득하려면 트릭의 비밀을 풀어주는 해설서(SHAP/LIME)가 필요합니다. 이 두 기술은 블랙박스라는 거대한 그림자를 걷어내는 가장 실용적인 손전등입니다.

## 📌 관련 개념 맵 (Knowledge Graph)

- **XAI (Explainable AI)** | 블랙박스 인공지능 모델의 예측 결과에 대한 인간의 이해와 해석을 돕기 위해 원인과 근거를 제시하는 제반 기술의 총칭.
- **피처 중요도 (Feature Importance)** | 트리기반 모델에서 각 변수가 노드를 분할하여 불순도를 낮추는 데 얼마나 기여했는지 전체 관점에서 측정한 고전적 지표, SHAP 이전의 표준.
- **섀플리 값 (Shapley Value)** | 게임 이론에서 협력 게임에 참여한 플레이어들이 전체 이득에 기여한 한계 공헌도를 조합론적으로 계산해 성과를 공정하게 분배하는 공식. SHAP의 뿌리 이론.
- **의사결정 나무 (Decision Tree)** | LIME/SHAP 같은 사후 해석기 없이도, 조건 분기(If-Then 룰) 자체로 완벽한 설명(White-Box)이 가능한 본질적 설명 가능 모델.
- **데이터 드리프트 (Data Drift)** | 운영 중인 AI 시스템에 들어오는 입력 데이터의 분포가 학습 당시와 달라져 성능이 저하되는 현상. SHAP 글로벌 분석으로 원인 변수를 조기 탐지할 수 있다.

## 👶 어린이를 위한 3줄 비유 설명
1. 5명의 친구가 함께 커다란 피자를 만들었는데, 누가 제일 맛있는 재료를 많이 가져왔는지 싸움이 났어요.
2. 이때 LIME은 방금 만든 피자 한 판만 보고 "이번 피자는 치즈(A친구)가 제일 중요했어!"라고 빨리 말해주는 심판이에요.
3. 반면에 SHAP은 지난 1년 동안 친구들이 피자를 만들 때마다 맛이 어땠는지 전부 계산해서, 한 치의 오차도 없이 공평하게 상을 나눠주는 아주 공정한 수학 선생님이랍니다!
