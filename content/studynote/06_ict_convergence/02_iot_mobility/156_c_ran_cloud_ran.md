# 156. C-RAN (Cloud RAN)

> **핵심 인사이트**: 기지국의 '안테나'와 '두뇌'를 분리하는 기술이다. 옥상마다 있던 기지국 처리 장비(DU)를 중앙의 데이터센터(클라우드)로 모으고, 현장에는 안테나(RU)만 남겨 운영 효율성과 확장성을 극대화한다.

## Ⅰ. C-RAN (Cloud Radio Access Network)의 개념
C-RAN은 기지국의 데이터 처리부(BBU/DU)를 한 곳의 클라우드 센터에 풀링(Pooling) 형태로 모아서 중앙 집중적으로 관리하고, 전파를 송수신하는 무선 신호 처리부(RRH/RU)만 셀 사이트(엣지)에 분산 배치하는 차세대 기지국 아키텍처입니다.

## Ⅱ. 기존 D-RAN과 C-RAN의 구조 비교

```text
[ 기존 D-RAN (Distributed RAN) ]
┌─────────────────────────┐
│ 옥상 / 철탑             │
│  [ 안테나(RU) ]         │
│       │                 │
│  [ 처리장비(DU) ]       │ ◄─ 각각의 기지국마다 냉방, 전력, 임대료 발생
└───────┬─────────────────┘
        │ Backhaul
     [ 코어망 ]

[ C-RAN (Cloud RAN) ]
┌─────────────────────────┐   Fronthaul (CPRI/eCPRI 광케이블)
│ 옥상 A: [ 안테나(RU) ]  │ ──────┐
│ 옥상 B: [ 안테나(RU) ]  │ ──────┤   ┌──────────────────────────┐
│ 옥상 C: [ 안테나(RU) ]  │ ──────┼──▶│ 중앙 집중국 (Cloud 센터) │
└─────────────────────────┘       │   │ [ DU Pool (가상화) ]     │
                                  └──▶│                          │
                                      └────────────┬─────────────┘
                                                   │ Backhaul
                                                [ 코어망 ]
```

## Ⅲ. C-RAN의 핵심 장점 및 도입 효과

| 구분 | 세부 내용 |
|:---|:---|
| **비용 절감 (CAPEX/OPEX)** | 현장 장비가 최소화되어 기지국 임대료, 전력 소모, 냉방 비용이 획기적으로 감소합니다. |
| **자원 효율성 (Pooling)** | 지역별 트래픽 편차(낮에는 오피스, 밤에는 주거지)에 따라 중앙의 컴퓨팅 자원을 유연하게 할당할 수 있습니다. |
| **간섭 제어 및 성능 향상** | 여러 기지국의 신호를 중앙에서 한 번에 처리하므로, 셀 간 간섭 제어(CoMP)와 핸드오버 처리가 훨씬 매끄럽습니다. |

## Ⅳ. 프론트홀 (Fronthaul) 요구사항
RU와 집중국의 DU를 연결하는 구간을 **프론트홀**이라고 합니다. 기저대역(Baseband)의 원시 아날로그 데이터를 디지털화(CPRI 등)하여 전송하므로 **막대한 대역폭과 초저지연(광케이블)** 이 필수적으로 요구됩니다.

> 📢 **섹션 요약 비유**: 매장마다 요리사를 두던 프랜차이즈 식당(D-RAN)이 중앙 센트럴 키친(클라우드 DU)에서 요리를 다 해서 각 매장(RU)에는 배식구만 두는 배달 전문점(C-RAN)으로 바뀐 것과 같습니다. 주방 유지비가 줄고 요리사 관리가 편해집니다.
