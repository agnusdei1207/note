## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 5G의 핵심 가치는 빠른 속도(eMBB)만이 아니라, 1ms 초저지연(uRLLC)과 km²당 100만 기기 동시 접속(mMTC)이라는 서로 상충하는 세 가지 요구사항을 단일 물리망 위에서 네트워크 슬라이싱(Network Slicing)으로 동시 충족하는 데 있다.
> 2. **가치**: 네트워크 슬라이싱은 하나의 물리 5G 인프라를 산업별 맞춤 가상망(NSI)으로 분리함으로써, 통신사는 B2B 서비스를 산업 수요에 맞게 판매하고 기업은 전용 품질(SLA)을 보장받는다.
> 3. **판단 포인트**: SA(Standalone) 아키텍처는 5G 코어(5GC)와 gNB가 완전 분리되어 네트워크 슬라이싱·URLLC 등 5G 고유 기능을 완전히 지원한다. NSA(Non-Standalone)는 LTE 코어를 공유해 전환 비용은 낮지만 5G 핵심 기능이 제한된다.

---

## Ⅰ. 개요 및 필요성

**5G ITU-R IMT-2020 표준 3대 서비스**

| 서비스 | 영문 | 핵심 지표 | 주요 사용처 |
|:---:|:---:|:---:|:---:|
| eMBB | Enhanced Mobile Broadband | 최대 20Gbps | VR·AR·4K 스트리밍 |
| uRLLC | Ultra-Reliable Low-Latency | 지연 1ms, 신뢰도 99.999% | 자율주행·원격 수술·공장 제어 |
| mMTC | massive Machine-Type Communication | 100만 기기/km² | 스마트시티·스마트팜·미터링 |

**4G(LTE) 한계와 5G 등장**: LTE는 단일 파이프(단순 고속) 구조로 서로 다른 QoS 요구를 동시에 충족 불가. 5G는 가상화(NFV)·소프트웨어 정의(SDN) 기반으로 유연한 다중 QoS 관리를 실현한다.

📢 **섹션 요약 비유**: 4G는 **단일 도로**, 5G는 **세 개의 전용 차선**이다. eMBB는 고속차선(빠른 차), uRLLC는 앰뷸런스 전용차선(항상 빈 길), mMTC는 오토바이 전용 넓은 갓길. 이 세 차선이 서로 침범하지 않는다.

## Ⅱ. 아키텍처 및 핵심 원리

```
┌──────────────────────────────────────────────────────────┐
│           5G 네트워크 슬라이싱 아키텍처                    │
├──────────────────────────────────────────────────────────┤
│  [물리 인프라]  gNB(기지국) + 5G Core(5GC) 하드웨어        │
│        │                                                 │
│  [가상화 계층]  NFV(네트워크 기능 가상화) / SDN             │
│        │                                                 │
│  [슬라이스 관리]  NSSF(Network Slice Selection Function)  │
│  ┌─────────────────────────────────────────────────┐    │
│  │  NSI-1 (eMBB 슬라이스)   │ 넷플릭스·통신사 서비스  │    │
│  ├─────────────────────────┤──────────────────────┤    │
│  │  NSI-2 (uRLLC 슬라이스) │ 자율주행·원격 수술     │    │
│  ├─────────────────────────┤──────────────────────┤    │
│  │  NSI-3 (mMTC 슬라이스)  │ 스마트시티·IoT 센서    │    │
│  └─────────────────────────┴──────────────────────┘    │
│                                                          │
│  [SLA 보장]  각 슬라이스별 독립 QoS 보장                   │
└──────────────────────────────────────────────────────────┘
```

### 네트워크 슬라이싱 핵심 개념

| 개념 | 설명 |
|:---|:---|
| NSI(Network Slice Instance) | 특정 서비스를 위해 생성된 독립 가상 네트워크 인스턴스 |
| NSSF(Network Slice Selection Function) | 단말 요청에 따라 적절한 슬라이스를 선택·배정 |
| S-NSSAI | 슬라이스 식별자. SST(슬라이스 서비스 유형)+SD(분류자) |
| NF(Network Function) | 5G 코어의 기능 단위. AMF·SMF·UPF 등을 슬라이스별 인스턴스화 |

📢 **섹션 요약 비유**: 네트워크 슬라이싱은 **구름 소프트웨어로 도로를 실시간 재배치하는 것**이다. 아침(출근 시간, eMBB 증가)엔 고속차선을 넓히고, 수술 시간(uRLLC 급증)엔 앰뷸런스 차선을 더 확보한다. 물리 도로는 그대로인데 소프트웨어로 분배만 바꾼다.

## Ⅲ. 비교 및 연결

**SA(Standalone) vs NSA(Non-Standalone)**

| 항목 | NSA (Non-Standalone) | SA (Standalone) |
|:---:|:---:|:---:|
| 코어 네트워크 | LTE EPC 공용 | 5G 전용 코어(5GC) |
| 기지국 | LTE + gNB 앵커링 | gNB 단독 |
| 슬라이싱 지원 | 제한적 | 완전 지원 |
| uRLLC 지원 | 제한적 | 완전 지원 |
| 도입 비용 | 낮음 (기존 LTE 활용) | 높음 |
| 진화 경로 | 과도기 | 5G 최종 목표 |

**5G SBA(Service-Based Architecture)**: 5G 코어는 마이크로서비스 구조로 설계. AMF(접속 관리), SMF(세션 관리), UPF(사용자 평면), PCF(정책), NSSF(슬라이스 선택) 등 기능별 독립 NF로 분리.

📢 **섹션 요약 비유**: NSA는 **새집(5G 기지국)을 짓되 구형 전기 설비(LTE 코어)를 그대로 쓰는 것**이다. 돈은 적게 들지만 새 가전제품(uRLLC·슬라이싱)을 다 쓸 수 없다. SA는 전기 설비까지 전부 교체한 완성형이다.

## Ⅳ. 실무 적용 및 기술사 판단

**산업별 슬라이스 설계 예시**

| 산업 | 슬라이스 유형 | 핵심 요구사항 |
|:---|:---:|:---|
| 자율주행 | uRLLC | 1ms 지연, 99.999% 신뢰도 |
| 스마트팩토리 | uRLLC + private 5G | 결정론적 통신, 데이터 로컬 처리 |
| 미디어·방송 | eMBB | 20Gbps, 대용량 |
| 스마트시티 IoT | mMTC | 초저전력, 대량 기기 |

**기술사 핵심 판단**

1. 네트워크 슬라이싱은 동일 물리 망에서 SLA가 다른 다수 고객을 동시 지원하는 **멀티테넌시(Multi-tenancy)** 구현 기술.
2. 슬라이스 격리(Isolation): 슬라이스 간 트래픽 누출 방지 → 보안 및 성능 보장.
3. 프라이빗 5G(Private 5G): 기업 전용 5G 네트워크. uRLLC 슬라이스를 자사 공장 내에서 독점 운용.

📢 **섹션 요약 비유**: 슬라이스 격리는 **아파트 분리벽**이다. 옆집(다른 슬라이스) 소음(트래픽)이 내 집(내 슬라이스)에 들어오지 않도록 벽(격리)을 튼튼하게 세워야 한다.

## Ⅴ. 기대효과 및 결론

5G의 3대 특성과 네트워크 슬라이싱은 통신 산업을 B2C 중심에서 B2B 중심으로 전환하는 핵심 엔진이다. SA 구조 완성과 슬라이싱 상용화가 진행될수록 자율주행·스마트팩토리·의료 등 미션 크리티컬 서비스의 무선화가 가속된다.

📢 **섹션 요약 비유**: 5G + 슬라이싱은 **만능 임대 공간**이다. 하나의 건물(물리 인프라)에서 식당(eMBB)·수술실(uRLLC)·창고(mMTC)를 동시에 운영할 수 있다. 수술실과 식당은 서로 절대 방해받지 않는다.

### 👶 어린이를 위한 3줄 비유 설명

1. 5G의 세 가지 특성은 **빠른 차선(eMBB), 앰뷸런스 전용차선(uRLLC), 오토바이 전용도로(mMTC)**예요.
2. 네트워크 슬라이싱은 **하나의 도로를 소프트웨어로 세 개의 전용차선으로 분리하는 마법**이에요.
3. SA 아키텍처는 **새 건물에 새 전기 설비까지 다 새로 설치하는 것**이어서 비싸지만, 5G의 모든 기능을 100% 쓸 수 있어요.
