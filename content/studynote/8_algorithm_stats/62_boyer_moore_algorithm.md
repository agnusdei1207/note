# 62. 보이어-무어 알고리즘 (Boyer-Moore Algorithm)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 보이어-무어(Boyer-Moore) 알고리즘은 패턴의 마지막 문자부터 왼쪽으로 비교하여 불일치가 발생하면 큰 점프를 수행하는 문자열 검색 算法이다.
> 2. **가치**: 실제 텍스트에서 평균 O(N/M) (패턴 길이 대비 매우 빠름)에 동작하며, 패턴이 길수록 유리하다.
> 3. **융합**: 텍스트 편집기, compilers의 lexical 分析, 대규모 텍스트 처리 등에서 가장 널리 사용되는 문자열 검색算法이다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

보이어-무어(Boyer-Moore) 알고리즘은 1977년 Robert Boyer와 Strother Moore이 개발한 문자열 검색 알고리즘이다. KMP가 텍스트의 현재 위치에서 패턴을左から右へ比較하는 반면, 보이어-무어는 패턴의마지막 문자부터右から左へ비교한다는 점이最大的 차이다.

보이어-무어가 중요한 이유는 **비교 횟수의大幅 감소** 때문이다. 패턴의 마지막 문자부터 비교를 시작하면, 불일치가 발생하는 순간 "텍스트의 현재 문자가 패턴에是否存在하는지"를 利用하여 큰 점프를 할 수 있다. 만약 불일치한 텍스트 문자가 패턴 전체에 없다면, 패턴 길이만큼 한 번에 점프할 수 있다.

보이어-무어는 **실용적으로 가장 빠른 문자열 검색 알고리즘** 중 하나이다. 특히 패턴이 길고 알파벳이 큰 경우에 극도로 효율적으로 동작한다. 많은 텍스트 편집기와 유닉스 grep 명령어에서도 보이어-무어의 변형이 사용된다.

```text
[보이어-무어 vs 다른 알고리즘 비교]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [보이어-무어: 뒤에서 앞으로 비교]                             │
│  ────────────────────────────────────────────                │
│                                                              │
│  텍스트:   Q S A B C D A B A B C D A B                       │
│  패턴:       A B C D A B                                     │
│                          ↑ 비교 시작 (마지막 문자)            │
│                                                              │
│  Step 1: 패턴 끝(D)과 텍스트 C 비교 → 불일치                  │
│          텍스트 C는 패턴에 없음 → 패턴 전체 건너뛰기          │
│                                                              │
│  Step 2:   Q S A B C D A B A B C D A B                       │
│                  패턴:     A B C D A B                       │
│                                    ↑ 다시 마지막 문자부터     │
│                                                              │
│  [점프 거리 계산]                                            │
│  ────────────────────────────────────────────                │
│                                                              │
│  불일치 문자 = "C"                                            │
│  Bad Character Rule:                                        │
│    C가 패턴에서 존재하는가?                                   │
│    - 예: C는 인덱스 2에 존재 → 2칸 이동                      │
│    - 아니오: 패턴 전체 건너뛰기                                │
│                                                              │
│  [성능 비교]                                                  │
│  ────────────────────────────────────────────                │
│  브루트 포스: O(N × M)                                       │
│  KMP: O(N + M)                                              │
│  Boyer-Moore: 평균 O(N / M), 최악 O(N × M)                  │
│  → 패턴이 길수록 / 텍스트가ランダム할수록 빠름                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 보이어-무어는 텍스트의 앞쪽보다 뒤쪽 문자를 먼저 확인한다.
- **원인**: 불일치한 문자를 利用하여 최대한 큰 점프를 하기 위해서이다.
- **결과**: 텍스트의 모든 문자를 비교할 필요가 없어진다.
- **판단**: 패턴이 길고 알파벳이 큰 경우 보이어-무어가 가장 효율적이다.

📢 **섹션 요약 비유**: 보이어-무어는 **图书馆 도서 검색**과 같습니다. 책의 맨 뒤 페이지(마지막 문자)부터 비교를 시작하면, 만약 "Z"로 시작하는 책을 찾는데 현재 책이 "A"로 시작하면, "Z"가 없는 모든 책(A-X)을 건너뛸 수 있다. 뒤에서부터 확인하면 불필요한 책을 빠르게 걸러낼 수 있다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

보이어-무어 알고리즘의 핵심은 두 가지 **휴리스틱(Heuristic)**이다. **Bad Character 휴리스틱**: 불일치가 발생한 텍스트 문자를 "bad character"라 하고, 이 문자가 패턴에서 가장 오른쪽出现 위치를 利用하여 점프 거리를 결정한다. **Good Suffix 휴리스틱**: 부분적으로 일치했던 접미사(good suffix)를 利用하여 점프한다.

**Bad Character Rule**: 불일치가 발생한 위치 i에서 텍스트 문자 T[j]와 패턴 문자 P[i]가 불일치했다고 가정하자. 패턴에서 T[j]가 나타나는 가장 오른쪽 위치를 찾는다. 만약 존재하면 패턴을 이동하여 그 위치와 T[j]를对齐한다. 존재하지 않으면 패턴을 넘어간다.

**Good Suffix Rule**: 패턴의 접미사 T[j-M+1..j]가 패턴의 다른 위치와도 일치하는 경우, 그 위치로 이동한다. 단순히 가장 긴 일치 접미사를 찾는 것이 아니라, 일치 접미사之后的第一个不匹配字符位置까지 고려한다.

```text
[보이어-무어 상세 알고리즘]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [Bad Character Rule 구현]                                    │
│  ────────────────────────────────────────────                │
│                                                              │
│  preprocessing: pattern의 각 문자별 가장 오른쪽 위치         │
│                                                              │
│  function badCharShift(pattern, mismatch_char, mismatch_idx):│
│      // mismatch_char: 텍스트에서 불일치한 문자               │
│      // mismatch_idx: 패턴에서 불일치가 발생한 위치 (오른쪽부터)│
│                                                              │
│      // 패턴에서 mismatch_char의 가장 오른쪽 위치 탐색        │
│      for i in range(mismatch_idx, -1, -1):                  │
│          if pattern[i] == mismatch_char:                    │
│              return mismatch_idx - i                         │
│                                                              │
│      // 없으면 패턴 전체 길이만큼 이동                        │
│      return len(pattern)                                    │
│                                                              │
│  [Good Suffix Rule 구현]                                      │
│  ────────────────────────────────────────────                │
│                                                              │
│  // 복잡한 구현 (패턴의 접미사와 일치하는 부분 탐색)         │
│  // 접미사가 패턴의 접두사로도 나타나는 위치로 이동            │
│                                                              │
│  [Boyer-Moore 검색]                                           │
│  ────────────────────────────────────────────                │
│                                                              │
│  function BoyerMoore(text, pattern):                         │
│      n, m = len(text), len(pattern)                         │
│      result = []                                             │
│                                                              │
│      // 패턴을 텍스트에 배치                                   │
│      i = 0                                                    │
│      while i <= n - m:                                      │
│          // 패턴의 마지막 문자부터 비교                        │
│          j = m - 1                                           │
│          while j >= 0 and text[i + j] == pattern[j]:       │
│              j -= 1                                          │
│                                                              │
│          if j < 0:  // 패턴 전체 일치                         │
│              result.append(i)                                │
│              // 다음可能出现 위치로 이동                       │
│              if i + m < n:                                   │
│                  shift = max(badCharShift(text[i + m],      │
│                                   pattern, m - 1),           │
│                              goodSuffixShift(pattern, m))   │
│              i += shift if exists else 1                    │
│          else:                                               │
│              // 불일치가 발생한 경우                           │
│              shift = badCharShift(text[i + j], pattern, j) │
│              i += shift                                       │
│                                                              │
│      return result                                           │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 보이어-무어는 패턴의 오른쪽 끝에서 왼쪽으로 비교한다.
- **원인**: 불일치가 발생하면 그 위치의 텍스트 문자를 利用하여 최대 점프를 하기 위해서이다.
- **결과**: 텍스트의 각 위치에서 패턴의 모든 문자를 비교하지 않아도 된다.
- **판단**: 패턴이 길고 텍스트가 큰 경우 보이어-무어가 가장 실용적이다.

📢 **섹션 요약 비유**: 보이어-무어의 양방향 비교 전략은 **.crossworduzzle 해결**과 같습니다. 단어를 찾을 때 앞에서부터 찾는 대신, 단어의 끝 글자부터 확인하면 더 빠르게 일치 여부를 판단할 수 있다. 끝 글자가 맞지 않으면 다음 위치로 빠르게 이동할 수 있다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

보이어-무어 알고리즘의 실무 적용은 다음과 같다. **텍스트 편집기의 찾기**: VSCode, Sublime Text 등의 에디터에서 사용된다. **시스템 수준 문자열 검색**: 유닉스 grep, find 명령어에서 활용된다. **컴파일러의 lexical 分析**:源代码에서 토큰을 인식하는 데 사용된다.

**구현 최적화**: 실제 구현에서는 Good Suffix Rule을省略하고 Bad Character Rule만 사용하는 경우가 많다. 이는 구현이 간단하고, 대부분의場合で十分な 성능을 제공하기 때문이다. 이를 **Boyer-Moore-Horspool** 알고리즘이라 한다.

```text
[Boyer-Moore-Horspool (단순화된 버전)]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  // Good Suffix Rule를省略하고 Bad Character Rule만 사용    │
│  ────────────────────────────────────────────                │
│                                                              │
│  function BoyerMooreHorspool(text, pattern):               │
│      n, m = len(text), len(pattern)                         │
│                                                              │
│      //预处理: 패턴의 각 문자별 오른쪽 위치                    │
│      skip = {}                                              │
│      for i in range(m - 1):                                  │
│          skip[pattern[i]] = m - i - 1                       │
│                                                              │
│      i = 0                                                    │
│      while i <= n - m:                                      │
│          j = m - 1                                           │
│          while j >= 0 and text[i + j] == pattern[j]:       │
│              j -= 1                                          │
│                                                              │
│          if j < 0:                                           │
│              return i  // 발견                                │
│          else:                                               │
│              // 텍스트의 현재 위치 문자 기준 점프              │
│              c = text[i + m - 1]                             │
│              i += skip.get(c, m)  // 없으면 패턴 전체 길이    │
│                                                              │
│      return -1  // 미발견                                    │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: Boyer-Moore-Horspool은 **음식점의 와인 리스트 확인**과 같습니다. 손님이 "2015년 와인을 찾는데"라는 요청을 하면, 전체 목록을 처음부터 보는 대신 가장 최근年份(문자열의 끝)부터 확인하여 빠르게 위치을 찾을 수 있다.

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

보이어-무어 알고리즘의 품질 관리에서 가장 중요한 것은 **점프 거리 계산의 정확성**과 **알파벳 처리**이다.

**품질 관리 체크리스트**: bad character table이 정확한지 확인해야 한다. 패턴의 마지막 문자에서 불일치할 때의 처리르 정확히 해야 한다. 빈 패턴이나 빈 텍스트에 대한 예외 처리가 필요하다.

📢 **섹션 요약 비유**: 보이어-무어 품질 관리는 **자동 Doors 센서調整**과 같습니다. 센서가 문이 완전히 닫혔는지(패턴 끝)부터 확인하고, 닫히지 않았으면(불일치) 어느 정도 열려있는지를 利用하여 정확한 개방 범위를 판단한다.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

보이어-무어 알고리즘의 최신 동향은 **더 빠른 검색**과 **메모리 최적화**와 관련되어 있다. **병렬 보이어-무어**: SIMD 명령어를 利用하여 한 번에 여러 문자를 비교한다. **FM-Index**: BWT(Burrows-Wheeler Transform)를 利用한 압축 기반 문자열 인덱싱.

보이어-무어 알고리즘은 "실용적인 문자열 검색의 대표 알고리즘"으로서, 단순하면서도 매우高效적인 특성을 가지고 있다.

📢 **섹션 요약 비유**: 보이어-무어의 발전은 **스마트 도서관 검색 시스템**과 같습니다. 초기에는 사서에게 처음부터 차례대로 물어봤지만(브루트 포스), 이제는 컴퓨터로 ISBN의 끝자리(패턴의 끝)부터 확인하여 빠르게 도서를 찾고, 연관된 다른 도서도 추천해준다.
