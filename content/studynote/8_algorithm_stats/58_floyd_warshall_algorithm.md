# 58. 플로이드-워샬 알고리즘 (Floyd-Warshall Algorithm)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 플로이드-워샬(Floyd-Warshall) 알고리즘은 모든 노드 쌍 사이의 최단경로를 한 번에 찾는 동적 프로그래밍 기반 알고리즘이다.
> 2. **가치**: O(V³) 시간에 모든 쌍 최단경로를 계산하며, 음수 가중치도 처리하고, 동시에 경로 재구성도 지원한다.
> 3. **융합**: 거리 벡터 라우팅, 그래프 전이闭包 계산, 경쟁 관계 分析 등 모든 쌍 관계가 중요한 문제에 활용된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

플로이드-워샬(Floyd-Warshall) 알고리즘은 1962년 Robert Floyd와 Stephen Warshall에 의해 독립적으로 발표되었다. 이 알고리즘은 **모든 노드 쌍 사이의 최단경로**를 한 번에 계산한다. 다익스트라나 벨만-포드가 하나의 시작점에서 다른 모든 점까지의 최단경로를 찾는 반면, 플로이드-워샬은 모든 가능한 시작-끝점 쌍에 대해 최단경로를 구한다.

플로이드-워샬 알고리즘이 중요한 이유는 **동시성**과 **완전성** 때문이다. V개의 노드에 대해 V번의 다익스트라(또는 벨만-포드)를 실행하는 것보다 한 번의 플로이드-워샬이 더 간단하고, 경우에 따라 더 효율적이다. 또한, 모든 노드 쌍의 거리를 미리 계산해두면, 실시간으로 어떤 두 노드 사이의 최단경로도 O(1)에 질의할 수 있다.

플로이드-워샬은 **동적 프로그래밍** 접근법을 사용한다. 경유 노드 k를 통해 오는 경로와 직접 가는 경로를 비교하여 더 짧은 것을 선택한다. 이 과정을 모든 k에 대해 반복하면 모든 노드 쌍의 최단경로가 완성된다.

```text
[플로이드-워샬 동작 원리]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [3x3 그래프 예시]                                            │
│  ────────────────────────────────────────────                │
│                                                              │
│    dist[i][j] 초기 상태:                                      │
│           A    B    C                                        │
│    A      0    4   INF                                       │
│    B    INF   0    1                                         │
│    C      2  INF   0                                         │
│                                                              │
│  [동적 프로그래밍 과정]                                        │
│  ────────────────────────────────────────────                │
│                                                              │
│  k = A (경유지 A 허용):                                       │
│    dist[B][C] = min(inf, dist[B][A]+dist[A][C]) = min(inf,inf)=inf
│    dist[C][B] = min(inf, dist[C][A]+dist[A][B]) = min(inf,inf)=inf
│                                                              │
│  k = B (경유지 A,B 허용):                                     │
│    dist[A][C] = min(inf, dist[A][B]+dist[B][C]) = 5         │
│    dist[C][A] = min(2, dist[C][B]+dist[B][A]) = 2 (유지)    │
│                                                              │
│  k = C (경유지 A,B,C 허용):                                   │
│    dist[A][B] = min(4, dist[A][C]+dist[C][B]) = min(4,inf)=4
│    dist[B][A] = min(inf, dist[B][C]+dist[C][A]) = 3         │
│                                                              │
│  최종 결과:                                                  │
│           A    B    C                                        │
│    A      0    4    5                                        │
│    B      3    0    1                                        │
│    C      2    6    0                                        │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 플로이드-워샬은 3중 중첩 루프로 모든 경유지 k에 대해 모든 쌍 (i, j)를更新한다.
- **원인**: 경유지를 늘려가며 더 짧은 경로를 발견하면 更新하는 동적 프로그래밍의 특성을利用한다.
- **결과**: 모든 노드 쌍 사이의 최단거리가 한번의 알고리즘 실행으로 완성된다.
- **판단**: 모든 쌍 최단경로가 필요하면 플로이드-워샬이 가장 간단한解决方案이다.

📢 **섹션 요약 비유**: 플로이드-워샬은 **도시 간 모든航班 연결表**编制와 같습니다. 서울에서 도쿄까지 직접 가는航班이 없으면, 도쿄행航班이 있는 다른 도시를 경유지로 고려하고, 여러 경유지를 탐색하여 가장 빠른旅程을 찾는다. 모든 도시 쌍에 대해 이러한 분석을 한 번에 수행한다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

플로이드-워샬 알고리즘의 핵심은 **경유지 확장(Intermediate Vertex)** 개념이다. 경유지를 0개에서 V개까지 늘려가면서 각 단계에서 최단거리를 更新한다. 경유지가 0개일 때: 직접 연결된 간선만 허용. 경유지가 1개일 때: 노드 1을 경유지로 허용. 경유지가 2개일 때: 노드 1, 2를 경유지로 허용. ... 경유지가 V개일 때: 모든 노드를 경유지로 허용.

**시간 복잡도**: O(V³) - V번의 경유지에 대해 V×V개의 쌍을 확인하므로 V³. 이는 희소 그래프에서는 다익스트라 V번(O(V² log V))보다 느리지만, 밀집 그래프에서는 더 효율적일 수 있다.

**공간 복잡도**: O(V²) - 거리 행렬을 저장해야 하므로.

```text
[플로이드-워샬 상세 알고리즘]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  function FloydWarshall(graph):                              │
│      V = graph.num_vertices                                  │
│                                                              │
│      // 1. 거리 행렬 초기화                                    │
│      dist = [[infinity] * V for _ in range(V)]              │
│      for i in range(V):                                      │
│          dist[i][i] = 0                                      │
│      for each edge (u, v, w) in graph.edges:                │
│          dist[u][v] = w                                      │
│                                                              │
│      // 2. 경유지 확장을 통한 최단거리 계산                     │
│      for k in range(V):  // 경유지                            │
│          for i in range(V):  // 출발지                         │
│              for j in range(V):  // 목적지                    │
│                  if dist[i][k] + dist[k][j] < dist[i][j]:   │
│                      dist[i][j] = dist[i][k] + dist[k][j]   │
│                                                              │
│      return dist                                             │
│                                                              │
│  [경로 재구성을 위한 이전 노드 배열]                           │
│  ────────────────────────────────────────────                │
│  next = [[None] * V for _ in range(V)]                      │
│  for each edge (u, v, w):                                   │
│      next[u][v] = v                                         │
│                                                              │
│  // 경유지 확장과 함께:                                        │
│  if dist[i][k] + dist[k][j] < dist[i][j]:                  │
│      dist[i][j] = dist[i][k] + dist[k][j]                   │
│      next[i][j] = next[i][k]                                 │
│                                                              │
│  [경로 재구성]                                               │
│  ────────────────────────────────────────────                │
│  function get_path(i, j):                                    │
│      if next[i][j] is None:                                 │
│          return []  // 경로 없음                             │
│      path = [i]                                             │
│      while i != j:                                          │
│          i = next[i][j]                                      │
│          path.append(i)                                      │
│      return path                                             │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 경유지 k를 추가할 때마다 더 짧은 경로가 발견될 수 있다.
- **원인**: k를 경유지로 허용한다는 것은 이전에 허용된 경유지들加上k도 사용할 수 있다는 의미이기 때문이다.
- **결과**: k=0에서 k=V까지 확장하면 모든 가능한 경유지 조합을 고려한 최단경로가 된다.
- **판단**: 밀집 그래프에서 모든 쌍 최단경로가 필요하면 플로이드-워샬이 효율적이다.

📢 **섹션 요약 비유**: 플로이드-워샬의 경유지 확장은 **국제 배송 경로 최적화**와 같습니다. 초기에 국내 물류망만 이용 가능하고, 점차 아시아 물류망, 유럽 물류망, 글로벌 물류망으로 확장하면서 더 짧은 배송 경로를 발견한다. 최종적으로 모든 가능한 중계 항구/공항을 고려한 최적 경로가 완성된다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

플로이드-워샬 알고리즘의 실무 적용은 다음과 같다. **네트워크 라우팅**: 모든 라우터 쌍 사이의最短거리를 미리 계산하여 라우팅 테이블을构建한다. **그래프 전이闭包 계산**: 어떤 노드에서 어떤 노드로 갈 수 있는지를判断한다(도달 가능성). **스포츠 토너먼트 분석**: 팀들 사이의 상대 전적을 분석하여 등급을 매긴다.

**응용: 무방향 그래프 vs 방향 그래프**: 무방향 그래프에서는 dist[i][j] = dist[j][i]이므로 행렬의 절반만 계산하면 된다. 방향 그래프에서는 모든 쌍을個別적으로 계산해야 한다.

```text
[플로이드-워샬 실무 활용]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [도달 가능성 분석 (Transitive Closure)]                       │
│  ────────────────────────────────────────────                │
│                                                              │
│  reach[i][j] = True/False (i에서 j로 도달 가능한지)          │
│                                                              │
│  for k in range(V):                                         │
│      for i in range(V):                                     │
│          for j in range(V):                                 │
│              reach[i][j] = reach[i][j] or (reach[i][k] and reach[k][j])
│                                                              │
│  → SSO(Small Swept Orchestra) 같은 도달 가능性问题에 활용     │
│                                                              │
│  [신경망의 은익층(hidden layer) 길이 계산]                     │
│  ────────────────────────────────────────────                │
│  각 뉴런 쌍 사이의 연결 강도를 가중치로 보아,                   │
│  두 뉴런 사이의최장 경로 길이를 계산                           │
│                                                              │
│  [행렬 곱셈과의 관계]                                          │
│  ────────────────────────────────────────────                │
│  FloydWarshall은 동적 프로그래밍이지만,                       │
│  행렬 곱셈의 반복과 유사한 구조를 가진다:                      │
│  dist^{(k)} = min(dist^{(k-1)}, dist^{(k-1)} × dist^{(k-1)}) │
│  (단, ×는 행렬 곱셈이 아닌 Min-Plus 곱셈)                    │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 플로이드-워샬의 실무 활용은 **세계 도시铁路망 지도**와 같습니다. 도시들 사이의 직접 열차 노선뿐 아니라, 여러 번 환승을 고려한 모든 가능한旅程 중 가장 빠른旅程을 동시에 보여준다.旅行者는 출발地和目的地만 알면,算法이自動的に最適경로를 찾아준다.

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

플로이드-워샬 알고리즘의 품질 관리에서 가장 중요한 것은 **오버플로우 방지**와 **음수 사이클 检测**이다.

**품질 관리 체크리스트**: 거리의 합이 자료형의 범위를 넘지 않도록 64비트 정수를 사용해야 한다. 음수 사이클이 존재하면 거리가 무한히 작아질 수 있으므로 检测 로직이 필요하다. INF(infinity)는 충분히 큰 값으로 설정하되, 합산 시 overflow가 발생하지 않도록 주의해야 한다.

📢 **섹션 요약 비유**: 플로이드-워샬의品質 管理는 **은행間の取引 Relationship 분석**과 같습니다. 은행 A가 은행 B에 돈을 보내고, B가 C에 보내고, C가 다시 A에 보내는交易이 있으면(음수 사이클), 각 거래의 잔액이 계속 줄어드는 문제가 발생할 수 있다. 이러한異常 거래 패턴을 检测하여システム保全에 활용한다.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

플로이드-워샬 알고리즘의 최신 동향은 **병렬화**와 **approximate 버전**과 관련되어 있다. **GPU 가속**: O(V³) 연산을 GPU에서 병렬화하여 대규모 그래프를 빠르게 처리한다. **행렬 분할**: 큰 행렬을小块로 분할하여 분산 처리한다.

플로이드-워샬 알고리즘은 "모든 쌍 최단경로 문제의 대표적 알고리즘"으로서, 비교적 작은 수의 노드(V < 1000)에서 모든 쌍 관계가 필요한 상황에 적합하다.

📢 **섹션 요약 비유**: 플로이드-워샬은 **전 세계 항공사聯合 시스템**과 같습니다. 모든 도시 pair 사이에直航이 없더라도,算法은 여러 경유지를 고려하여 최적의航班 연결을 찾는다. 여행객은 출발地和目的地만 입력하면, 최적의旅程이自動的に計算되어 표시된다.
