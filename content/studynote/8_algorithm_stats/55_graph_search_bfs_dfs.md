# 55. 그래프 탐색 - BFS/DFS (Breadth-First Search / Depth-First Search)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: BFS(너비 우선 탐색)는 가까운 노드부터 탐색하고, DFS(깊이 우선 탐색)는 한 경로를 끝까지 탐색한 후 백트래킹하는 그래프 탐색 알고리즘이다.
> 2. **가치**: BFS는 최단경로 보장(비가중 그래프), DFS는 경로 탐색과 토폴로지 정렬에 유리하다.
> 3. **융합**: 두 알고리즘은 길찾기, 소셜 네트워크 분석, 미로 탈출, 의존성解析 등 광범위한 문제에 활용된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

그래프 탐색은 그래프에서 특정 노드를 찾거나, 모든 노드를 방문하는 기본적인 작업이다. BFS(너비 우선 탐색, Breadth-First Search)와 DFS(깊이 우선 탐색, Depth-First Search)는 가장 기본적인 두 가지 그래프 탐색 전략이다.

BFS는 시작 노드에서 가까운 노드부터 우선적으로 방문한다. 이는 친구의 친구(2단계 연결)보다 바로 친구(1단계 연결)를 먼저 방문하는 것과 같다. BFS는 **큐(Queue)** 자료구조를 사용하여 구현되며, FIFO(선입선출) 특성으로 인해 레벨별로 탐색이 이루어진다.

DFS는 시작 노드에서 한 경로를 따라 가능한 깊이까지 탐색한 후, 더 이상 진행할 수 없으면 백트래킹하여 다른 경로를 탐색한다. 이는 미로에서 한 길로 쭉 가다가 막히면 가장 최근 갈림길로 돌아가 다른 길을试试보는 것과 같다. DFS는 **스택(Stack)** 또는 **재귀(Recursion)**를 사용하여 구현된다.

```text
[BFS vs DFS 탐색 비교]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [그래프 구조]                                                │
│  ────────────────────────────────────────────                │
│                                                              │
│         A (시작)                                              │
│        / \                                                    │
│       B   C                                                  │
│      /   / \                                                 │
│     D   E   F                                                │
│                                                              │
│  [BFS 탐색 순서: 레벨별]                                      │
│  ────────────────────────────────────────────                │
│  1단계: A → [A]                                              │
│  2단계: B, C → [A, B, C]                                     │
│  3단계: D, E, F → [A, B, C, D, E, F]                         │
│                                                              │
│  → 가장 가까운 노드부터 순차적으로 방문                        │
│                                                              │
│  [DFS 탐색 순서: 경로별]                                      │
│  ────────────────────────────────────────────                │
│  경로 1: A → B → D (leaf 도달, 백트래킹)                      │
│  경로 2: A → C → E (leaf 도달, 백트래킹)                      │
│  경로 3: A → C → F (leaf 도달, 완료)                          │
│                                                              │
│  → 한 경로를 끝까지 가고, 막히면 돌아가서 다른 경로探索       │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: BFS와 DFS는 동일한 그래프에서도 다른 탐색 순서를 보인다.
- **원인**: BFS는 FIFO 큐로 "먼저 온 것을 먼저 처리"하고, DFS는 LIFO 스택으로 "가장 최근 것을 먼저 처리"하기 때문이다.
- **결과**: BFS는 시작점에서 모든 최단경로를 찾고, DFS는 모든 경로를探索한다.
- **판단**: 문제의 성격에 따라 BFS 또는 DFS를 선택해야 한다.

📢 **섹션 요약 비유**: BFS와 DFS는 각각 **파도타기**와 **등산**과 같습니다. BFS는 해변에서 시작하여 파도가치는 물결처럼 한 단계씩同心円적으로 펼쳐나가는 것이고, DFS는 한 길로 끝까지 올라가山顶에 도달하면 돌아와 다른 길을試みる 것이다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

BFS의 핵심 원리는 **레벨별 탐색**과 **최단경로 보장**이다. BFS는 시작점에서 거리가 1인 모든 노드를 방문한 후, 거리가 2인 노드를 방문하는 식으로 진행한다. 이를 통해 임의의 노드에 처음으로 도달했을 때의 경로가 항상 최단경로(비가중 그래프 기준)임이 보장된다.

BFS의 시간 복잡도는 O(V + E)이며, 여기서 V는 노드의 수, E는 간선의 수이다. 각 노드를 한 번, 각 간선을 한 번씩 방문하기 때문이다. 공간 복잡도는 O(V)이며, 탐색 큐와 방문 배열이 필요하기 때문이다.

DFS의 핵심 원리는 **경로 추적**과 **백트래킹**이다. DFS는 한 경로를 따라 leaf 노드까지 도달한 후, 더 이상 진행할 수 없으면 가장 최근의 분기점으로 돌아가서 다른 경로를試みる. 모든 경로를 완벽하게 탐색할 수 있다.

DFS에는 세 가지主要的 방식이 있다. **전위 순회(Pre-order)**: 노드 방문 → 왼쪽 서브트리 → 오른쪽 서브트리. **중위 순회(In-order)**: 왼쪽 서브트리 → 노드 방문 → 오른쪽 서브트리. **후위 순회(Post-order)**: 왼쪽 서브트리 → 오른쪽 서브트리 → 노드 방문.

```text
[BFS/DFS 구현 및 탐색 순서]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [BFS 의사코드]                                               │
│  ────────────────────────────────────────────                │
│  function BFS(start):                                        │
│      queue = [start]                                         │
│      visited = {start}                                       │
│                                                              │
│      while queue is not empty:                              │
│          node = queue.pop(0)  // 앞쪽 원소 추출             │
│          visit(node)                                        │
│                                                              │
│          for each neighbor of node:                         │
│              if neighbor not in visited:                    │
│                  visited.add(neighbor)                       │
│                  queue.append(neighbor)                      │
│                                                              │
│  [DFS 의사코드 (재귀)]                                        │
│  ────────────────────────────────────────────                │
│  function DFS(node, visited):                               │
│      if node is null:                                       │
│          return                                              │
│      visit(node)                                            │
│      visited.add(node)                                      │
│                                                              │
│      for each neighbor of node:                              │
│          if neighbor not in visited:                        │
│              DFS(neighbor, visited)                         │
│                                                              │
│  [DFS 의사코드 (스택)]                                        │
│  ────────────────────────────────────────────                │
│  function DFS_iterative(start):                             │
│      stack = [start]                                        │
│                                                              │
│      while stack is not empty:                              │
│          node = stack.pop()  // 마지막 원소 추출            │
│          if node not in visited:                            │
│              visit(node)                                    │
│              visited.add(node)                               │
│                                                              │
│              // 역순으로 스택에 추가 (순서 유지)              │
│              for each neighbor of node (reverse order):     │
│                  if neighbor not in visited:                │
│                      stack.append(neighbor)                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: BFS는 큐를, DFS는 스택(또는 재귀)을 사용한다.
- **원인**: 큐의 FIFO는 "동일 레벨 먼저 처리"를, 스택의 LIFO는 "가장 최근 것 먼저 처리"를 의미하기 때문이다.
- **결과**: BFS는 최단경로, DFS는 모든 경로探索에 적합하다.
- **판단**: 문제의 요구사항(최단경로인지, 모든 경로인지)에 따라 선택한다.

📢 **섹션 요약 비유**: BFS/DFS의 자료구조 선택은 **식사 순서**와 같습니다. BFS는 모든 사람에게 같은 종류의 음식을 먼저 주고(1단계 레벨), 그 다음 다른 음식을 주는 것과 같다. DFS는 한 사람이 음식을 다 먹을 때까지(leaf 도달) 다음 사람에게 주지 않고, 먹고 나면 돌아와서 다른 음식 조합을試보는 것과 같다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

BFS의 실무 적용: **최단경로 탐색(비가중 그래프)**: GPS 네비게이션에서 두 지점 사이의 최단 거리 경로를 찾는 데 활용된다. **소셜 네트워크 분석**: SNS에서 "친구의 친구" 레벨의 사용자를 찾거나, 특정 인물과의 연결 경로를 찾는 데 사용된다. **미로 탈출**: 가장 짧은 탈출 경로를 찾는 문제에서 BFS가 필수적이다.

DFS의 실무 적용: **경로 탐색**: 모든 가능한 경로를 탐색해야 하는 문제(퍼즐, 게임 등)에서 활용된다. **연결 요소 분할**: 그래프에서 연결된 모든 노드를 찾는 데 사용된다. **토폴로지 정렬**: 작업의 우선순위를 결정하는 데 사용된다. **사이클 검출**: 그래프에서 사이클이是否存在하는지를 확인한다.

```text
[BFS/DFS 실무 활용 예시]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [미로 탈출 - BFS 활용]                                       │
│  ────────────────────────────────────────────                │
│                                                              │
│  미로:                                                       │
│  S # # # #                                                   │
│  # . . . #                                                   │
│  # # # . #                                                   │
│  # . . . E                                                   │
│                                                              │
│  S=시작, E=종료, #=벽, .=통로                                 │
│                                                              │
│  BFS로 발견한 최단경로:                                       │
│  S → ↓ → → → ↓ → ↓ → E (8 steps)                           │
│                                                              │
│  [컴포넌트 분할 - DFS 활용]                                   │
│  ────────────────────────────────────────────                │
│                                                              │
│  그래프:                                                     │
│    1 --- 2      5 --- 6                                      │
│    |     |                                                   │
│    3 --- 4      7                                            │
│                                                              │
│  DFS로 탐색하면 2개의 연결 컴포넌트 발견:                      │
│  컴포넌트 1: {1, 2, 3, 4}                                    │
│  컴포넌트 2: {5, 6, 7}                                       │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: BFS/DFS 실무 활용은 **도시 탐험 가이드**와 같습니다. BFS 가이드는 "가장 가까운 명소부터 차례로 가보자. 그러면 가장 빠른 시간에 주요 명소를 모두 찍을 수 있어요"라고 말할 것이고, DFS 가이드는 "이 길로 가서 막히면 돌아와서 다른 길을探索하자. 모든 가능한 경로를試 볼 수 있어요"라고 말할 것이다.

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

BFS/DFS의 품질 관리에서 가장 중요한 것은 **방문 배열의 정확한 관리**와 **그래프 유형에 따른 처리**이다.

**품질 관리 체크리스트**: 모든 노드가 한 번씩만 방문되었는지 확인해야 한다. 연결되지 않은 그래프에서 모든 노드가 방문되었는지 확인해야 한다. 사이클이 있는 그래프에서 무한 루프에 빠지지 않았는지 확인해야 한다.

📢 **섹션 요약 비유**: BFS/DFS의品質 管理는 **여행 루트確認**과 같습니다. 모든 도시를 정확히 한 번씩 방문했는지(방문 배열), 어느 도시간에도 연결되지 않은 고립된 도시가 없는지(연결 컴포넌트), 같은 도시를 두 번 방문하지 않았는지(사이클 방지)를 확인해야 한다.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

BFS/DFS의 최신 동향은 **병렬화**와 **그래프 데이터베이스**와 결합되어 발전하고 있다. 대규모 그래프(소셜 네트워크, 추천 시스템)에서分布式 BFS/DFS가 연구되고 있으며, Neo4j 같은 그래프 데이터베이스에서 Cypher 쿼리 언어로 BFS/DFS 기반 연산이 활용된다.

BFS와 DFS는 "그래프 탐색의 양대 산맥"으로서, 각각의 고유한 특성을 이해하고 문제에 맞게 선택하는 것이 핵심이다.

📢 **섹션 요약 비유**: BFS와 DFS는 **폭발물 탐지견 훈련**과 같습니다. BFS 방식의 탐지견은 가장 가까운 장소부터 체계적으로 확인해 나가고, DFS 방식의 탐지견은 의심스러운匂い가 있는 방향으로 깊이 파고들어 확인한 후, 없으면 돌아와서 다른 방향을試みる. 두 방법 모두 중요하며, 상황에 따라 적절한 방법을 선택해야 한다.
