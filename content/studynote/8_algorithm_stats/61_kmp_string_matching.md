# 61. KMP 문자열 매칭 (Knuth-Morris-Pratt Algorithm)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: KMP(크누스-모리스-프랫) 알고리즘은 문자열 검색에서 불일치가 발생했을 때 패턴의 일부를再利用하여 효율적으로 검색을 수행하는 算法이다.
> 2. **가치**: 브루트 포스 O(N×M)을 O(N+M)으로 크게 향상시키며, 텍스트를 되돌아가지 않고 전진한다.
> 3. **융합**: 텍스트 편집기의 찾기 기능, plagiarism 检测, bioinformatics의 DNA 패턴 매칭 등에 활용된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

KMP(크누스-모리스-프랫, Knuth-Morris-Pratt) 알고리즘은 1977년 Donald Knuth, Vaughan Pratt, James Morris가 독립적으로 개발한 문자열 검색 알고리즘이다. 이 알고리즘은 텍스트에서 패턴(Pattern)의 모든出現 위치를 찾는 문제를 해결한다.

KMP가 중요한 이유는 **비교 횟수의 혁신적 감소** 때문이다. 전통적인 브루트 포스(Brute Force) 방식은 불일치가 발생하면 패턴을 한 칸만 이동하고 텍스트의 현재 위치로 돌아간다. 이로 인해 최악의 경우 O(N×M) 시간이 소요된다. KMP는 "실패 함수(Failure Function)"를 활용하여 불일치 발생 시 패턴을 효율적으로 이동하고, 텍스트를 되돌아가지 않는다.

KMP의 핵심 아이디어는 **부분 패턴의再利用**이다. 예를 들어, "ABABC"라는 패턴으로 검색 중 "ABAB"까지 일치하고 "C"에서 불일치했다고 가정하자.这时候 "AB"가 패턴의 접두사로도出现하므로, 다음 검색 위치를 바로 "AB"부터 시작할 수 있다.

```text
[KMP vs 브루트 포스 비교]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [브루트 포스: 불일치 시 1칸만 이동]                           │
│  ────────────────────────────────────────────                │
│                                                              │
│  텍스트:   ABABABCABABCABABCABABC                            │
│  패턴:     ABABC                                             │
│                                                              │
│  시도 1: ABABABC...  (5번째 문자 불일치)                      │
│  시도 2:  ABABABC...  (1번째 문자 불일치)                     │
│  시도 3:   ABABABC...  (5번째 문자 불일치)                    │
│  ... (효율 매우 낮음)                                        │
│                                                              │
│  [KMP: 실패 함수를利用하여 점프]                               │
│  ────────────────────────────────────────────                │
│                                                              │
│  텍스트:   ABABABCABABCABABCABABC                            │
│  패턴:     ABABC                                             │
│                                                              │
│  시도 1: ABABABC...  (5번째 문자 불일치)                      │
│          → 실패 함수: "ABAB"에서 "AB"가 longest proper prefix-suffix  │
│          → 패턴을 3칸 이동 (다음 "AB"부터 시작)               │
│                                                              │
│  시도 2:     ABABCAB...  (모두 일치! 검색 완료)               │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: KMP는 텍스트를 되돌아가지 않고 항상前進한다.
- **원인**: 실패 함수 덕분에 패턴의 어느 위치에서 다시 시작해야 할지를 이미 알고 있기 때문이다.
- **결과**: 시간 복잡도가 O(N+M)으로大幅 개선된다.
- **판단**: 긴 텍스트에서 패턴을 여러 번 검색해야 할 때 KMP가 필수적이다.

📢 **섹션 요약 비유**: KMP의 실패 함수 활용은 **압축된地图**과 같습니다. 서울에서 부산까지 가는 동안 "광주 찍고 나면 다음 큰 도시는 목포"이라는 정보를 압축하여 저장해두면, 길이 끊겨도(불일치) 빠르게 다시 진행할 수 있다. 다시 처음부터 찾는 것이 아니라 압축된 정보로 점프한다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

KMP 알고리즘의 핵심은 **실패 함수(Failure Function)**, 또는 **LPS(Longest Proper Prefix which is also Suffix) 배열**이다. 이 배열은 패턴의 각 위치 i에서, 위치 i까지의 부분 문자열 중에서 proper prefix이자 proper suffix인 가장 긴 문자열의 길이를 저장한다.

**실패 함수 계산**: 패턴 P에 대해 lps[i]를 계산한다. lps[0] = 0 (첫 문자는 항상 0). i를 1부터 M-1까지 증가시키며: 현재 문자 P[i]와 P[len]을 비교한다. 불일치하면 len = lps[len-1]로 되돌아가서 다시 비교한다. 일치하면 len += 1, lps[i] = len.

**KMP 검색**: 텍스트 T와 패턴 P를 비교하며, 불일치 발생 시 P의 lps[위치-1] 값으로 이동한다. 텍스트의 인덱스는 절대 되돌아가지 않는다.

```text
[실패 함수 (LPS) 계산 과정]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [패턴: ABABC]                                                │
│  ────────────────────────────────────────────                │
│                                                              │
│  인덱스:    0   1   2   3   4                                  │
│  문자:     A   B   A   B   C                                  │
│                                                              │
│  lps[0] = 0 (항상)                                            │
│                                                              │
│  i = 1 (B):                                                   │
│      len = 0, P[1]=B != P[0]=A, len = lps[0] = 0             │
│      P[1]=B != P[0]=A, lps[1] = 0                           │
│                                                              │
│  i = 2 (A):                                                   │
│      len = 0, P[2]=A == P[0]=A, len = 1, lps[2] = 1         │
│                                                              │
│  i = 3 (B):                                                   │
│      len = 1, P[3]=B == P[1]=B, len = 2, lps[3] = 2         │
│                                                              │
│  i = 4 (C):                                                   │
│      len = 2, P[4]=C != P[2]=A, len = lps[1] = 0            │
│      len = 0, P[4]=C != P[0]=A, lps[4] = 0                  │
│                                                              │
│  최종 lps: [0, 0, 1, 2, 0]                                   │
│                                                              │
│  [KMP 검색 과정]                                              │
│  ────────────────────────────────────────────                │
│                                                              │
│  텍스트:  ABABDABABABD                                        │
│  패턴:    ABABD                                              │
│  lps:     [0, 0, 1, 2, 0]                                    │
│                                                              │
│  i=0: T[0]=A == P[0]=A, i=1                                  │
│  i=1: T[1]=B == P[1]=B, i=2                                  │
│  i=2: T[2]=A == P[2]=A, i=3                                  │
│  i=3: T[3]=B == P[3]=B, i=4                                  │
│  i=4: T[4]=D == P[4]=D, 패턴 전체 일치!                       │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: lps[i]는 "이 위치까지 일치했을 때, 어디부터 다시 비교하면 되는지"를 알려준다.
- **원인**: proper prefix와 proper suffix가 동일하다는 것은, 이미 일치한 부분을再利用할 수 있다는 의미이다.
- **결과**: 텍스트를 되돌리지 않고 패턴만 이동하여 검색을 계속한다.
- **판단**: 실패 함수의 정확한 계산이 KMP의 효율성을 결정한다.

📢 **섹션 요약 비유**: KMP의 실패 함수 계산은 **계산서 항목 매칭**과 같습니다. "물품 A, 물품 B, 물품 A, 물품 B, 물품 C"라는 계산서를 쓸 때, "물품 A, 물품 B"가 반복되는 것을 알아차리면(Proper prefix-suffix) 이후 같은 패턴이 올 때 빠르게 매칭할 수 있다. 사전에 이러한 반복 구조를 분석해두면 검색이 빠르다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

KMP 알고리즘의 실무 적용은 다음과 같다. **텍스트 편집기의 찾기 기능**: 긴 문서에서 특정 문자열을 찾는 데 활용된다. **침입 检测 시스템**: 네트워크 패킷에서 특정 패턴(악성 코드 서명)을 찾는 데 활용된다. ** біоінформатика**: DNA 시퀀스에서 특정 유전자 패턴을 찾는 데 활용된다.

**접두사-접미사 활용**: KMP는 문자열의 접두사와 접미사 관계를 이용하는 문제에 응용된다. 문자열의 모든 접두사와 일치하는 모든 접미사를 찾는 문제, 주기적인 패턴 검출 등에 활용된다.

```text
[KMP 구현]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  function computeLPS(pattern):                                │
│      M = len(pattern)                                        │
│      lps = [0] * M                                           │
│      length = 0  // 이전 lps 길이                              │
│      i = 1                                                   │
│                                                              │
│      while i < M:                                            │
│          if pattern[i] == pattern[length]:                   │
│              length += 1                                    │
│              lps[i] = length                                 │
│              i += 1                                         │
│          else:                                               │
│              if length != 0:                                │
│                  length = lps[length - 1]                    │
│              else:                                           │
│                  lps[i] = 0                                  │
│                  i += 1                                      │
│                                                              │
│      return lps                                              │
│                                                              │
│  function KMP(text, pattern):                                │
│      N = len(text)                                           │
│      M = len(pattern)                                        │
│      lps = computeLPS(pattern)                               │
│                                                              │
│      i = 0  // text 인덱스                                    │
│      j = 0  // pattern 인덱스                                │
│                                                              │
│      while i < N:                                            │
│          if text[i] == pattern[j]:                          │
│              i += 1                                         │
│              j += 1                                         │
│              if j == M:                                     │
│                  return i - j  // 패턴 발견                   │
│          else:                                               │
│              if j != 0:                                      │
│                  j = lps[j - 1]  // 패턴 이동                │
│              else:                                           │
│                  i += 1  // 다음 문자부터 시작               │
│                                                              │
│      return -1  // 패턴 미발견                                │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: KMP 실무 활용은 **금속 탐지기**와 같습니다. 특정 금속 물체가 지나갈 때마다 신호가 바뀌는 패턴을 인식하여, 별도의 보편적 신호 분석 없이도 빠르게 특정 물체를 탐지한다. 문자열에서도 마찬가지로 사전에 패턴을 분석해두면 텍스트를 한 번만 훑으면서 패턴을 찾을 수 있다.

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

KMP 알고리즘의 품질 관리에서 가장 중요한 것은 **실패 함수의 정확성**과 **경계 조건 처리**이다.

**품질 관리 체크리스트**: lps[0] = 0이 정확한지 확인해야 한다. 모든 i에 대해 lps[i]가 proper prefix-suffix 길이임을 검증해야 한다. 텍스트나 패턴이 비어있을 때의 예외 처리가 필요하다.

📢 **섹션 요약 비유**: KMP 품질 관리는 **계산기 내장 회로 검사**와 같습니다. 각 논리 게이트(실패 함수 계산)가 정확한 출력을 내는지, 전체 회로(알고리즘)가 정확한 결과를 산출하는지를 확인해야 한다.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

KMP 알고리즘의 최신 동향은 **더 빠른 문자열 매칭**과 **병렬 처리**와 관련되어 있다. **Boyer-Moore**: 패턴의 마지막 문자부터 비교하여 더 빠르게 점프하는 알고리즘. **Rabin-Karp**: 해시를利用한 알고리즘으로 여러 패턴 동시 검색에 활용된다. **Suffix Array/Tree**: 더 복잡한 문자열 질의에 사용되는 자료구조.

KMP 알고리즘은 "문자열 검색의 기본 알고리즘"으로서, 이후 수많은 문자열 알고리즘의 기반이 되었다.

📢 **섹션 요약 비유**: KMP 알고리즘의 발전은 **우체국 자동 분류 시스템**과 같습니다. 초기에는 물건을 하나하나 확인했지만(브루트 포스), 패턴을 분석하면 "우편번호 123-"으로 시작하면 특정 집배원에게 전달하는 것처럼, 사전 분석을 통해 분류 속도를 혁신적으로 높였다.
