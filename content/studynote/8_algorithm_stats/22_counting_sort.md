# 22. 계수 정렬 (Counting Sort)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 계수 정렬(Counting Sort)는 정렬할 원소들의 값 범위(최댓값-최솟값)가Known하고 비교적 작을 때, 각 값이 몇 번出現하는지를 세어(Build Count Array) Cumulative 합을求해 정렬을完成하는 O(N+K) 시간 복잡도의 算法이다.
> 2. **가치**: 비교 연산을 하지 않으므로(comparison-free) 이론적 하한 O(N log N)보다 빠른 O(N+K)에 정렬이 가능하며, 특히 值가 작은 범위内の整数에 대해 매우 효율적이다.
> 3. **융합**: 계수 정렬은 기수 정렬(Radix Sort)의 하위 단계로 활용되며, 범주형 데이터(카테고리, 학점 등)의 정렬에서 효율적으로 사용된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

계수 정렬(Counting Sort)은 1954년 Harold H. Seward가 개발한 것으로, 비교 기반 정렬의 이론적 하한인 O(N log N)에 의존하지 않는希土한 정렬算法이다. 비교 기반 정렬은 원소들 사이의大小関係만을利用하여 정렬하지만, 계수 정렬은 원소들의 值そのものの情報を利用하여 정렬의 위치를 직접計算한다.

계수 정렬이 중요한 이유는 **비교 없는 정렬**이라는革命的な 아이디어에 있다. 원소의 值가 배열의 인덱스가 되어直接 접근하면, 원소들 사이의 비교 없이도 정렬이 가능하다. 이것은 值 범위 K가Known하고 N에 비해 크지 않을 때 매우 효율적인 정렬을可能하게 한다.

> 이 도식은 계수 정렬의 작동을 보여준다.

```text
[계수 정렬 (Counting Sort) 과정]

┌──────────────────────────────────────────────────────┐
│                                                      │
│  [입력 배열] A = [4, 2, 2, 6, 1, 3, 4]            │
│  ────────────────────────────────────                │
│  K = 6 (최댓값, 值 범위 0~6)                      │
│                                                      │
│  [Step 1: Count 배열 구성]                          │
│  ────────────────────────────────────                │
│  count[0] = 1 (값 0이 1번)                         │
│  count[1] = 1 (값 1이 1번)                         │
│  count[2] = 2 (값 2가 2번)                         │
│  count[3] = 1 (값 3이 1번)                         │
│  count[4] = 2 (값 4가 2번)                         │
│  count[5] = 0 (값 5가 0번)                         │
│  count[6] = 1 (값 6이 1번)                         │
│                                                      │
│  [Step 2: 누적 합(Cumulative Sum) 계산]             │
│  ────────────────────────────────────                │
│  count[0] = 1                                      │
│  count[1] = 1+1 = 2                                │
│  count[2] = 2+2 = 4                                │
│  count[3] = 4+1 = 5                                │
│  count[4] = 5+2 = 7                                │
│  count[5] = 7+0 = 7                                │
│  count[6] = 7+1 = 8                                │
│                                                      │
│  [Step 3: 정렬된 배열 구성]                         │
│  ────────────────────────────────────                │
│  A를 뒤から 훑으며, count[A[i]]을 위치 결정에 사용    │
│  result[count[A[i]]-1] = A[i]                      │
│  count[A[i]]-- (다음 同値을 위해)                   │
│                                                      │
│  결과: [1, 2, 2, 3, 4, 4, 6]                      │
│                                                      │
└──────────────────────────────────────────────────────┘
```

- **관찰**: 계수 정렬의 핵심은 "원소의 值を 인덱스로利用하여 직접 정렬 위치를 결정한다"는 것이다.
- **원인**: 值 범위 K가有限하면, 각 值가 나타난 횟수를 세어 누적하면 바로 정렬이 가능하다.
- **결과**: 비교 연산이 없어 O(N+K)라는驚異的인 속도를 달성한다.
- **판단**: 值 범위 K가 N에 비해 크지 않을 때(예: K = O(N)) 매우 효율적이다.

📢 **섹션 요약 비유**: 계수 정렬은 投票紙整理와 같습니다.投票紙를 개수별로 山을 만들고, 각 山의 수를 세어累计하면, 다시 山의 순서대로投票紙를 나열하면自動的に整列が完了합니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

계수 정렬의 핵심 원리는 **누적 합(Cumulative Sum)**을利用한 위치 결정이다. 각 值 v가 전체 정렬에서 차지하게 될 시작 위치를 count 배열의 누적 합으로 구한다.

**안정성(Stability)**이 계수 정렬에서 중요한 이유는, 동일 값들을 가진 원소들의 상대적 순서가 지켜져야 하는 경우(예: Stable Sort가要求される場面)에 필수적이기 때문이다. 뒤から 훑으며 위치를 결정하면, 동일 값 사이의 순서가 보존된다.

```text
[계수 정렬: 안정성 확보 방법]

┌──────────────────────────────────────────────────────┐
│                                                      │
│  [동일 값의 순서를 보장하는 이유]                        │
│  ────────────────────────────────────                │
│                                                      │
│  입력이 [(3,'A'), (1,'B'), (3,'C'), (1,'D')]     │
│  (값, 원래 순서) 쌍으로 구성                          │
│                                                      │
│  뒤에서부터 처리:                                     │
│  1) (1,'D') → count[1]=2 → result[1]='D'        │
│  2) (3,'C') → count[3]=5 → result[4]='C'        │
│  3) (1,'B') → count[1]=1 → result[0]='B'        │
│  4) (3,'A') → count[3]=4 → result[3]='A'        │
│                                                      │
│  결과: [(1,'B'), (1,'D'), (3,'A'), (3,'C')]     │
│  → 동일 값(1)인 B가 D보다 앞에, 동일 값(3)인 A가 C보다 앞에 │  │
│  → 원래 순서 유지! (안정성 보장)                      │
│                                                      │
│  [시간 복잡도: O(N + K)]                             │
│  ────────────────────────────────────                │
│  count 배열 구성: O(N)                              │
│  누적 합 계산: O(K)                                  │
│  정렬된 배열 구성: O(N)                              │
│  → 총 O(N + K)                                     │
│                                                      │
└──────────────────────────────────────────────────────┘
```

- **관찰**: 계수 정렬을安定的に動作하게 하려면, 입력 배열을 뒤から 훑어야 한다.
- **원인**: 누적 합이 "현재 값以下の 개수"를 나타내므로, 뒤에서부터 처리하면 이전에処理된 동일 값들의 위치가 덮어씌워지지 않는다.
- **결과**: 동일 값 사이의 상대적 순서가 보존된다.
- **판단**: 기수 정렬(Radix Sort)의 각 자리수를 정렬할 때 계수 정렬을 사용하면, 안정성으로 인해 전체 정렬의安定性が確保される。

📢 **섹션 요약 비유**: 계수 정렬의安定性은 同じ点数の答案を提出顺序で整理하는 것과 같습니다.後ろから処理하면、同じ点数의 学生들의 原先顺序が自然に 保存됩니다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

계수 정렬의 실무 적용은 다음과 같다. **만세投票(단일 후보)**: 0/1 같은 이진 값의 개수를 세는 것은 계수 정렬의 특수한 경우이다. **학점 처리**: A/B/C/D/F那般 범위가 제한된categorical data에 적합하다. **기수 정렬의構成要素**: 각 자리수를 정렬할 때 계수 정렬을使用한다.

**제한사항**: 值 범위 K가 너무 크면(예: K > N²) 메모리 낭비가 심해진다. 원소가整数나小さくまとめられた 值여야 한다(문자열 등에는直接 적용 곤란).

```text
[계수 정렬 의사코드]

┌──────────────────────────────────────────────────────┐
│                                                      │
│  function counting_sort(A):                          │
│      max_val = max(A)                               │
│      k = max_val + 1                               │
│      count = [0] * k  // count 배열                │
│                                                      │
│      // 1단계: 빈도수 세기                             │
│      for x in A:                                    │
│          count[x] += 1                             │
│                                                      │
│      // 2단계: 누적 합 계산                            │
│      for i in range(1, k):                         │
│          count[i] += count[i-1]                    │
│                                                      │
│      // 3단계: 뒤에서부터 훑으며 정렬                  │
│      result = [0] * len(A)                          │
│      for x in reversed(A):                         │
│          result[count[x] - 1] = x                  │
│          count[x] -= 1                             │
│                                                      │
│      return result                                   │
│                                                      │
│  [시간/공간 복잡도]                                  │
│  ────────────────────────────────────                │
│  시간: O(N + K)                                     │
│  공간: O(N + K)                                     │
│                                                      │
└──────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 계수 정렬은 靴屋の靴整理と 같습니다.靴のサイズ별로棚を分け、各棚の靴の数をカウントし、棚の順序で靴を配置하면自動的にサイズ順に並んでいます.

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

계수 정렬의 품질 관리에서 가장 중요한 것은 **값 범위 K의 적절한 설정**이다.

**품질 관리 체크리스트**: K(최댓값)가 메모리budget 내에서 적절한지 확인해야 한다. 음수 값은 처리 불가능하므로offset 조정이 필요하다. 누적 합 계산에서 오버플로우(특히 고정 크기 정수형)를 방지해야 한다.

📢 **섹션 요약 비유**: 계수 정렬의品質 管理は ビルの強度検査と 같습니다. 強度(값 범위)가 설계 기준(메모리budget)을 넘으면 ビルの 안전性(시스템 안정성)이確保できません.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

계수 정렬의 최신 동향은 **기수 정렬(Radix Sort)**과의 결합应用이다. 계수 정렬을 각 자리수별로 적용하면, 문자열이나大きな整数도 O(N × M) 시간에 효율적으로 정렬할 수 있다. 여기서 M은 문자열의 길이이다.

계수 정렬은 "값 범위을 아는 경우, 비교 없이 정렬 가능하다는 revolutionary한 발상"으로, 비교 기반 정렬의 한계를 극복하는重要な算法学적 milestone이다.

📢 **섹션 요약 비유**: 계수 정렬은 投票의革命と 같습니다. 投票紙를 비교해서 순서를 매기는 대신, "몇 번째로 많이 받은候选人か"를 Countして並べると、瞬時に有序な結果得られます.
