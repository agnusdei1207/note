# 23. 기수 정렬 (Radix Sort)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 기수 정렬(Radix Sort)은 정렬할 원소들을 자리수(Digit)별로 분리하여 각 자리수를 낮은 자리부터 또는 높은 자리부터 순차적으로 정렬하는 비비교 정렬 알고리즘이다.
> 2. **가치**: 비교 연산을 사용하지 않으므로 O(N × K) 시간에 정렬할 수 있으며, 여기서 K는 원소의 최대 자리수이다. 특히 정수로 표현된 데이터에서 효율적이다.
> 3. **융합**: 기수 정렬은 계수 정렬(Counting Sort)을 각 자리수 정렬에 활용하며, 문자열 정렬이나 바이트 단위 정렬에서도 유용하게 사용된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

기수 정렬(Radix Sort)은 1887년 Herman Hollerith가 개발한 것으로, 비교 기반 정렬의 이론적 하한인 O(N log N)에 의존하지 않는 비비교(non-comparison) 정렬 알고리즘이다. 기수 정렬의 핵심 아이디어는 \"원소를 자리수별로 나누어 정렬하면, 전체 정렬이 완성된다\"는 것이다.

기수 정렬이 중요한 이유는 \"비교 없이도 정렬이 가능하다\"는革命적인 발상이다. 원소의 값을 직접 비교하는 대신, 각 자리수의 값을 사용하여 정렬의 위치를 결정한다. 이는 값 범위 K가 큰 경우에도 비교 기반 정렬보다 효율적으로 동작할 수 있다는 것을 의미한다.

> 이 도식은 기수 정렬의 과정을 보여준다.

```text
[기수 정렬: LSD (Least Significant Digit) 방식]

┌──────────────────────────────────────────────────────┐
│                                                      │
│  입력: [170, 45, 75, 90, 802, 24, 2, 66]          │
│                                                      │
│  [1단계: 1의 자리수 기준 정렬 (Counting Sort)]         │
│  ────────────────────────────────────                │
│  1의 자리: 170→0, 45→5, 75→5, 90→0, 802→2, 24→4, 2→2, 66→6│
│  0: [170, 90]                                       │
│  2: [802, 2]                                        │
│  4: [24]                                            │
│  5: [45, 75]                                        │
│  6: [66]                                            │
│                                                      │
│  중간 결과: [170, 90, 802, 2, 24, 45, 75, 66]       │
│                                                      │
│  [2단계: 10의 자리수 기준 정렬]                        │
│  ────────────────────────────────────                │
│  10의 자리: 170→7, 90→9, 802→0, 2→0, 24→2, 45→4, 75→7, 66→6│
│  0: [802, 2]                                        │
│  2: [24]                                            │
│  4: [45]                                            │
│  6: [66]                                            │
│  7: [170, 75]                                       │
│  9: [90]                                            │
│                                                      │
│  [3단계: 100의 자리수 기준 정렬]                       │
│  ────────────────────────────────────                │
│  100의 자리: 170→1, 90→0, 802→8, 2→0, 24→0, 45→0, 75→0, 66→0│
│  0: [90, 802, 2, 24, 45, 75, 66]                    │
│  1: [170]                                           │
│  8: [802]                                           │
│                                                      │
│  결과: [2, 24, 45, 66, 75, 90, 170, 802]           │
│                                                      │
└──────────────────────────────────────────────────────┘
```

- **관찰**: 기수 정렬은 각 자리수를 독립적으로 정렬한다.
- **원인**: 높은 자리수의 값이 낮은 자리수보다 우선하므로, 모든 자리수가 정렬되면 전체가 정렬된다.
- **결과**: 계수 정렬을 각 자리수에 적용하면 O(N × K) 시간에 정렬이 완료된다.
- **판단**: 정수의 자리수가 제한된 경우(예: 고정 너비 정수) 매우 효율적이다.

📢 **섹션 요약 비유**: 기수 정렬은 학번으로 학생을 정렬하는 것과 같습니다. 먼저 일의자리(1번자리)로 묶어 정리하고, 그다음 십의자리로 묶어 정리하면, 자연스럽게 학번순으로 정렬됩니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

기수 정렬의 핵심 원리는 **자리수별 정렬(LSD vs MSD)**과 **안정성 유지**이다. LSD(Least Significant Digit) 방식은 가장 낮은 자리수부터 정렬하며, MSD(Most Significant Digit) 방식은 가장 높은 자리수부터 정렬한다. 각 자리수 정렬에는 안정 정렬(보통 계수 정렬)을 사용해야 한다.

**LSD vs MSD**: LSD는 오른쪽 끝 자리부터 처리하며, 정렬이 끝나면 전체가 완전히 정렬된다. MSD는 왼쪽 끝 자리부터 처리하며, 각 그룹 내에서 재귀적으로 MSD를 적용한다.

```text
[기수 정렬: LSD vs MSD 방식]

┌──────────────────────────────────────────────────────┐
│                                                      │
│  [LSD (Least Significant Digit) 방식]                │
│  ────────────────────────────────────                │
│  입력: [45, 12, 73, 28, 91]                        │
│                                                      │
│  1의 자리 정렬 → [12, 73, 45, 28, 91]              │
│  10의 자리 정렬 → [12, 28, 45, 73, 91] (완료!)      │
│                                                      │
│  [MSD (Most Significant Digit) 방식]                  │
│  ────────────────────────────────────                │
│  입력: [45, 12, 73, 28, 91]                        │
│                                                      │
│  10의 자리로分组: [45, 12] [73] [28, 91]           │
│  각 그룹 내에서 1의 자리 정렬                         │
│  → [12, 45] [73] [28, 91]                          │
│  → [12, 28, 45, 73, 91] (결합)                      │
│                                                      │
│  [시간 복잡도: O(N × K)]                            │
│  ────────────────────────────────────                │
│  K = 최대 자리수 (基数)                             │
│  각 자리수 정렬: O(N)                               │
│  총 시간: O(N × K)                                 │
│                                                      │
│  [공간 복잡도: O(N + K)]                           │
│  ────────────────────────────────────                │
│  각 자리수 정렬에 계수 정렬 사용 시                   │
│                                                      │
└──────────────────────────────────────────────────────┘
```

- **관찰**: 기수 정렬의 효율은 자리수 K와 원소 수 N의 비율에 따라 달라진다.
- **원인**: K가 고정되어 있으면 O(N)에 가까운 성능을 낼 수 있다.
- **결과**: 원소의 값 범위가 크지 않고 자리수가 제한적인 경우 매우 효율적이다.
- **판단**: 문자열 정렬(lexicographic sort)에서 MSD 기수 정렬이 널리 사용된다.

📢 **섹션 요약 비유**: 기수 정렬의 자리수별 정렬은 電話番号码を整理하는 것과 같습니다.まず市外局番(高い 자리수)で分组し、各グループ内で市内局番(低い 자리수)を整理하면，全市中の電話番号が 체계적으로 정리됩니다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

기수 정렬의 실무 적용은 다음과 같다. **고정 너비 정수 정렬**: IP 주소(32비트 정수)나 날짜(YYYYMMDD)처럼 고정된 자리수를 가진 데이터에 적합하다. **문자열 정렬**: 문자열을 바이트 단위로 MSD 기수 정렬하면 O(N × M) 시간에 정렬할 수 있다. **바이트 단위 정렬**: 네트워크 프로토콜이나 이진 데이터 정렬에 활용된다.

```text
[기수 정렬 의사코드]

┌──────────────────────────────────────────────────────┐
│                                                      │
│  function radix_sort(A, d):  // d = 최대 자리수       │
│      for i = 0 to d-1:                             │
│          // i번째 자리수를 기준으로 안정 정렬 수행     │
│          stable_sort(A, lambda x: (x // 10^i) % 10)  │
│                                                      │
│  function counting_sort_by_digit(A, digit):         │
│      k = 10  // 자리수 범위 0-9                      │
│      count = [0] * k                               │
│                                                      │
│      // 1단계: 빈도수 세기                            │
│      for x in A:                                    │
│          d = (x // 10^digit) % 10                   │
│          count[d] += 1                             │
│                                                      │
│      // 2단계: 누적 합 계산                           │
│      for i = 1 to k-1:                             │
│          count[i] += count[i-1]                    │
│                                                      │
│      // 3단계: 뒤에서부터 훑으며 정렬                  │
│      result = [0] * len(A)                          │
│      for x in reversed(A):                         │
│          d = (x // 10^digit) % 10                   │
│          result[count[d] - 1] = x                  │
│          count[d] -= 1                             │
│                                                      │
│      return result                                   │
│                                                      │
└──────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 기수 정렬은 圖書館の藏書目錄整理と 같습니다. 먼저 著者名の最終文字(가장 낮은 자리수)로並べ,次にその前の文字(다음 자리수)로 정리하면,最終적으로完全に整列した目録 얻을 수 있습니다.

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

기수 정렬의 품질 관리에서 가장 중요한 것은 **안정성 확보**와 **자리수 처리**이다.

**품질 관리 체크리스트**: 각 자리수 정렬 시 안정 정렬을 사용해야 한다(일반적으로 계수 정렬). 음수 처리가 필요한 경우 별도의 offset 관리가 필요하다. 문자열 정렬 시 길이가 다른 문자열 처리를 위해 패딩(padding)을 고려해야 한다.

📢 **섹션 요약 비유**: 기수 정렬의品質 管理는 精密機械の組立と 같습니다. 各部品(자리수)을 올바른順序で组装해야 하며, 하나라도 빠지거나 순서가 바뀌면 전체製品(정렬 결과)가 불량品이 됩니다.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

기수 정렬의 최신 동향은 **병렬 기수 정렬**과 **GPU 가속**이다. 멀티코어 CPU나 GPU를 활용하면 기수 정렬의 각 단계를 병렬화하여 대규모 데이터 정렬 성능을 크게 향상시킬 수 있다. 또한 SSD나 HDD 같은 secondary storage에서 외부 기수 정렬을 활용하는 연구도 진행 중이다.

기수 정렬은 \"값을 직접 비교하지 않고 자리수별로 분류하면 정렬이 가능하다\"는 직관적이며 강력한 발상으로, 비교 기반 정렬의 한계를 극복하는 중요한 비비교 정렬 알고리즘이다.

📢 **섹션 요약 비유**: 기수 정렬は 手指の指を曲げて数を数える方法に似ています. 一の位から順に整理하면，最終的には指清洁な数列表現将达到できます.
