# 33. 외부 정렬 (External Sort)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 외부 정렬(External Sort)은 주 메모리(RAM)에 담을 수 없는 대용량 데이터를 디스크 등의 외부 저장장치에서 정렬하는 알고리즘이다.
> 2. **가치**: 메모리 제약으로 모든 데이터를 한꺼번에 처리할 수 없는 상황에서, 데이터를 Chunk로 나누어 정렬 후 합병하는 방식으로 동작한다.
> 3. **융합**: 외부 합병 정렬(External Merge Sort)이 가장 대표적이며, 멀티웨이 합병, 배분 합병, 하이퍼 합병 등의 변형이 있다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

외부 정렬(External Sort)은 1950년대부터 사용되어 온 대용량 데이터 정렬을 위한 알고리즘이다. 컴퓨터의 주 메모리(RAM)는 한계가 있으므로, 수기가바이트(GB) 이상의 데이터를 정렬해야 할 때에는 디스크와 같은 외부 저장장치를 활용해야 한다.

외부 정렬이 중요한 이유는 **메모리 제약의 극복**이다. 100GB의 데이터를 정렬해야 하는데 RAM이 8GB뿐인 상황을 생각해보면, 전체 데이터를 메모리에 올리고 내부 정렬 알고리즘을 적용하는 것은 불가능하다. 외부 정렬은 이 제약을 넘는다.

```text
[외부 정렬의 기본 원리]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [문제 설정]                                                  │
│  ────────────────────────────────────────────                │
│  RAM 크기: 8GB                                               │
│  정렬할 데이터: 100GB                                        │
│  디스크 I/O 비용 >> 메모리 접근 비용                          │
│                                                              │
│  [외부 정렬 2단계 접근]                                        │
│  ────────────────────────────────────────────                 │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐     │
│  │  Stage 1: 생성 (Generation)                        │     │
│  │  ───────────────────────────────────────          │     │
│  │  1-1. 데이터를 RAM 크기의 Chunk로 분할             │     │
│  │  1-2. 각 Chunk를 내부 정렬 알고리즘으로 정렬       │     │
│  │  1-3. 정렬된 Chunk를 디스크에 임시 파일로 저장      │     │
│  │       → N개의 정렬된 부분 파일(Run) 생성          │     │
│  └─────────────────────────────────────────────────────┘     │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐     │
│  │  Stage 2: 합병 (Merge)                             │     │
│  │  ───────────────────────────────────────          │     │
│  │  2-1. 정렬된 Chunk들을 멀티웨이 합병               │     │
│  │  2-2. 최종 정렬된 데이터를 출력                     │     │
│  └─────────────────────────────────────────────────────┘     │
│                                                              │
│  [예시]                                                      │
│  ────────────────────────────────────────────                 │
│  100GB 데이터를 8GB RAM로 정렬:                                │
│  Stage 1: 100GB / 8GB = 13개의 Chunk 생성                    │
│          각 8GB Chunk 내부에서 정렬 (8GB 씩)                  │
│          → 13개의 정렬된 Run 파일 생성                       │
│  Stage 2: 13개의 Run 파일을 합병                              │
│          → 최종 100GB 정렬된 파일 생성                        │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 외부 정렬은 "분할 정복" 원리를 적용한다.
- **원인**: 전체 데이터를 한 번에 처리할 수 없으므로, 작게 나누어 처리한다.
- **결과**: 메모리 제약에서도 대용량 데이터 정렬이 가능하다.
- **판단**: 데이터베이스, 파일 처리, 과학 계산 등 대용량 데이터 처리에서 필수적이다.

📢 **섹션 요약 비유**: 외부 정렬은 **도서관 전집 정리**와 같습니다. 전체 전집(100GB)을 한꺼번에 책상(8GB)에 올릴 수 없으므로, 전집을 챕터(Chunk)별로 나누어 각각 정렬한 후, 최종的に全卷을 올바른 순서로 並べる한다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

외부 정렬의 핵심은 **Stage 1: 생성**과 **Stage 2: 합병**으로 나뉜다. 생성 단계에서는 큰 데이터를 메모리 크기의 청크로 나누고 각 청크를 내부 정렬하고, 합병 단계에서는 정렬된 청크들을 하나의 정렬된 시퀀스로 합병한다.

**생성 단계(Generation)**: 입력 데이터를 읽을 수 있는 만큼씩 읽어 내부 정렬(일반적으로 힙 정렬이나 퀵 정렬)을 수행한 후, 정렬된 결과를 디스크에 저장한다. 이 과정을 모든 데이터가 처리될 때까지 반복한다.

**합병 단계(Merge)**: 정렬된Runs(Run files)들을 읽어 가장 작은 원소를 선택하는 방식으로 합병한다. 멀티웨이 합병(M-way merge)은 M개의 정렬된 시퀀스를 동시에 합병하며, 힙(優先순위 큐)을利用하여 효율적으로最小값을 선택한다.

```text
[외부 합병 정렬 상세 과정]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [멀티웨이 합병 (M-way Merge)]                                │
│  ────────────────────────────────────────────                │
│                                                              │
│  정렬된 Runs:                                                 │
│  Run 1: [1, 4, 9, 15, ...]                                  │
│  Run 2: [2, 5, 8, 12, ...]                                  │
│  Run 3: [3, 6, 11, 20, ...]                                 │
│  ...                                                         │
│  Run M: [..., 100, 200, ...]                                │
│                                                              │
│  우선순위 큐 (Min-Heap)에 각 Run의 첫 원소 삽입:                │
│  Heap: [1(Run1), 2(Run2), 3(Run3), ..., first(M)]           │
│                                                              │
│  합병 과정:                                                   │
│  1. Heap에서 최솟값 추출 → 출력 버퍼에 기록                    │
│  2. 추출한 값의 Run에서 다음 원소를 Heap에 삽입                │
│  3. 모든 Run이 소진될 때까지 1-2 반복                         │
│                                                              │
│  [디스크 I/O 최적화]                                          │
│  ────────────────────────────────────────────                 │
│                                                              │
│  버퍼링: 한 번에 여러 레코드씩 읽기/쓰기                      │
│  선독(read-ahead): 다음 데이터를 미리 읽어 놓음               │
│  기화(write-behind): 출력 버퍼를 모아 한꺼번에 쓰기            │
│                                                              │
│  [시간 복잡도 분석]                                            │
│  ────────────────────────────────────────────                 │
│                                                              │
│  N = 전체 레코드 수                                           │
│  B = 디스크 블록 크기                                          │
│  M = 사용 가능한 메모리 크기                                    │
│                                                              │
│  Stage 1 (생성): O(N)  // 각 레코드 한 번씩 읽기/쓰기          │
│  Stage 2 (합병): O(N log M)  // M-way 합병                   │
│                                                              │
│  디스크 I/O 관점:                                              │
│  읽기: N/B + (N/M) * log(M/B) ...                            │
│  쓰기: N/B + (N/M) * log(M/B) ...                            │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 외부 정렬의 성능은 디스크 I/O 횟수에 의해 결정된다.
- **원인**: 디스크 접근은 메모리 접근보다 수만 배에서 수십만 배 느리다.
- **결과**: 디스크 I/O를 최소화하는 것이 최적화의 핵심이다.
- **판단**: 버퍼링, 선독, 기화 등의 기법으로 I/O 효율을 높인다.

📢 **섹션 요약 비유**: 외부 정렬은 **大型建筑的施工**と似ています。全体建物を一度に建設할 수 없으므로、층별(Chunk)로 나누어 건설하고、各층을 먼저完成시킨後、階層を積み上げる(합병)方式进行施工한다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

외부 정렬의 실무 적용은 **대규모 데이터베이스**, **파일 시스템**, **과학 시뮬레이션** 등에서 필수적이다.

**DBMS에서의 활용**: ORDER BY 연산, JOIN 연산 등에서 외부 정렬이 사용된다. PostgreSQL, MySQL, Oracle 등의 데이터베이스는 대용량 결과셋을 정렬할 때 외부 정렬을 활용한다.

```text
[외부 정렬 구현]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [외부 합병 정렬 의사코드]                                      │
│  ────────────────────────────────────────────                │
│                                                              │
│  function external_sort(input_file, output_file, M):         │
│      // M: RAM 크기 (레코드 수)                               │
│                                                              │
│      // Stage 1: 생성                                        │
│      runs = []                                               │
│      while input has more records:                          │
│          chunk = read_records(input, M)                      │
│          sorted_chunk = internal_sort(chunk)  // 힙 정렬     │
│          run_file = create_temp_file()                       │
│          write_records(run_file, sorted_chunk)               │
│          runs.append(run_file)                              │
│                                                              │
│      // Stage 2: 합병                                         │
│      output_buffer = create_output_buffer()                  │
│      min_heap = MinHeap()                                    │
│                                                              │
│      // 각 Run의 첫 레코드 삽입                               │
│      for i, run in enumerate(runs):                        │
│          record = read_record(run)                          │
│          min_heap.insert(record, i)                         │
│                                                              │
│      while not min_heap.is_empty():                         │
│          record, run_idx = min_heap.extract_min()           │
│          write_record(output_buffer, record)                 │
│                                                              │
│          if run has more records:                          │
│              next_record = read_record(runs[run_idx])       │
│              min_heap.insert(next_record, run_idx)           │
│                                                              │
│      flush_output_buffer(output_buffer)                     │
│                                                              │
│  [실무 응용]                                                 │
│  ────────────────────────────────────────────                 │
│   1. 데이터베이스: ORDER BY, GROUP BY, JOIN                    │
│   2. 파일 처리: 대용량 로그 파일 정렬                          │
│   3. 과학 시뮬레이션: 원자 시뮬레이션 결과 정렬                 │
│   4. 검색 엔진: 인덱스 정렬                                   │
│   5. 외부 정렬 기반 병합 조인(External Merge Join)            │
│                                                              │
│  [최적화 기법]                                                │
│  ────────────────────────────────────────────                 │
│   1. 덮어쓰기 방지: 각 Run 파일은 한 번만 읽음                 │
│   2. 버퍼 관리: 디스크 I/O 효율화                              │
│   3. 합병 차수 결정: M(합병 수)에 따른 trade-off             │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 외부 정렬は **物流倉庫の大型商品整理**と似ています。大型商品を一度に運ぶことが不可能なため、小さな区域に分けて運び(MRU)、各区域内で先に順序を守りながら(정렬)，最後に全体の順序를 완성한다(합병).

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

외부 정렬의 최신 동향으로는 **병렬 외부 정렬**, **SSD 최적화 외부 정렬**, **클라우드 환경에서의 외부 정렬** 등이 연구되고 있다.

**병렬 외부 정렬**: 여러 머신에서 동시에 외부 정렬을 수행하여 대용량 데이터를 더 빠르게 처리한다. MapReduce 등의 분산 컴퓨팅 프레임워크에서 활용된다.

**외부 정렬의Positions**: 외부 정렬은 대용량 데이터 처리에서 여전히 핵심적인 기술이다. 데이터베이스, 파일 시스템, 과학 컴퓨팅 등 다양한領域에서 필수적이며, 하드웨어 발전(SSD, NVMe)과 함께 지속적으로 최적화되고 있다.

📢 **섹션 요약 비유**:外部 정렬は **時間旅行の計画**と似ています。すべてを同時に処理する代わりに、小さな塊に分けて处理し、最終的に全期間的正确な順序を确保する。
