# 29. 카운팅 정렬 (Counting Sort)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 카운팅 정렬(Counting Sort)는 원소의 값을 직접 비교하지 않고, 각 원소의 출현 횟수를 세어 정렬을 완료하는 비비교 정렬(Non-comparison Sort) 알고리즘이다.
> 2. **가치**: 비교 기반 정렬의 하한 O(N log N)을 넘을 수 있어, 특정 조건에서 O(N) 시간에 정렬이 가능하다.
> 3. **융합**: 정수나 고정 범위의 데이터에서 효율적이며, 기수 정렬(Radix Sort)의 핵심 구성 요소로 활용된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

카운팅 정렬(Counting Sort)은 1954년 Harold H. Seward에 의해 개발된 가장 오래된 비비교 정렬 알고리즘 중 하나이다. 전통적인 비교 기반 정렬(버블 정렬, 합병 정렬, 퀵 정렬 등)은 원소 간 비교를 통해 순서를 결정하지만, 카운팅 정렬은 원소의 값을 **카운트(개수 세기)**하여 정렬한다.

카운팅 정렬이 중요한 이유는 **시간 복잡도의 벽을 깰 수 있기** 때문이다. 비교 기반 정렬은理論적으로 Ω(N log N)보다 빠를 수 없지만, 카운팅 정렬은 원소의 범위가 제한적인 경우 O(N)에 정렬할 수 있다. 이것은 "정렬은 N log N보다 빠를 수 없다"는 잘못된 신념을 반박하는 중요한 알고리즘이다.

```text
[카운팅 정렬 전체 과정]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [입력 배열]                                                  │
│  ────────────────────────────────────────────                │
│  입력: [4, 2, 4, 3, 1, 4, 2, 3]                              │
│                                                              │
│  [Step 1: 카운트 배열 생성]                                    │
│  ────────────────────────────────────────────                │
│  값:    0   1   2   3   4    (범위: 0~4)                     │
│  카운트: 0   1   2   2   3                                    │
│          ↑   ↑   ↑   ↑   ↑                                   │
│          0개  1개 2개 2개 3개                                  │
│                                                              │
│  [Step 2: 누적 합 계산]                                       │
│  ────────────────────────────────────────────                │
│  값:    0   1   2   3   4                                    │
│  카운트: 0   1   2   2   3                                    │
│  누적:  0   1   3   5   8  ← 각 값의 마지막 위치+1            │
│                                                              │
│  [Step 3: 출력 배열 구성]                                     │
│  ────────────────────────────────────────────                │
│  입력 순회:                                                    │
│  4 → 누적[4]=8 → 출력[8-1]=4 → 누적[4]-- → 7                 │
│  2 → 누적[2]=3 → 출력[3-1]=2 → 누적[2]-- → 2                 │
│  4 → 누적[4]=7 → 출력[7-1]=4 → 누적[4]-- → 6                 │
│  3 → 누적[3]=5 → 출력[5-1]=3 → 누적[3]-- → 4                 │
│  1 → 누적[1]=1 → 출력[1-1]=1 → 누적[1]-- → 0                 │
│  4 → 누적[4]=6 → 출력[6-1]=4 → 누적[4]-- → 5                 │
│  2 → 누적[2]=2 → 출력[2-1]=2 → 누적[2]-- → 1                 │
│  3 → 누적[3]=4 → 출력[4-1]=3 → 누적[3]-- → 3                 │
│                                                              │
│  [최종 출력]                                                  │
│  ────────────────────────────────────────────                │
│  출력: [1, 2, 2, 3, 3, 4, 4, 4]                              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 카운팅 정렬은 원소의 값을 직접 비교하지 않고, 출현 횟수를 利用한다.
- **원인**: 원소의 범위(0부터 k까지)가 제한적이라면, 각 값을 直接 세는 것만으로 정렬이 가능하다.
- **결과**: 시간 복잡도 O(N + K), 공간 복잡도 O(K) (K는 원소 범위)
- **판단**: 원소 범위가 작고 정수인 경우, 비교 기반 정렬보다 卡우다.

📢 **섹션 요약 비유**: 카운팅 정렬은 **박물관 입장객 집계**와 같습니다. 각 나이대(0세부터 100세까지)에 몇 명이 입장했는지 세고, 0세부터 순서대로 어린 나이부터場に配置한다. 나이를 비교하지 않고, 세는 것만으로 정렬이 완료된다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

카운팅 정렬의 핵심은 **카운트 배열(Count Array)**과 **누적 합(Prefix Sum)**이다. 이 두 가지 개념을 이해하면 카운팅 정렬의 모든 단계가 명확해진다.

**카운트 배열**: 입력 원소의 범위(일반적으로 0부터 K까지)에 대해 각 값이 출현한 횟수를 저장하는 배열이다. 예를 들어, 값이 3인 원소가 2개 있다면 카운트[3] = 2이다.

**누적 합**: 각 값의 시작 위치를 계산하기 위해 카운트 배열을 누적합으로 변환한다. 누적 합 배열의 값은 "해당 값以下的 원소들을 정렬했을 때, 마지막 위치의 인덱스 + 1"을 의미한다. 이를 통해 같은 값을 가진 원소들을 올바른 위치에 配置할 수 있다.

```text
[카운팅 정렬 상세 알고리즘]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [카운팅 정렬 의사코드]                                        │
│  ────────────────────────────────────────────                │
│                                                              │
│  function counting_sort(A, k):  // A: 입력, k: 최대값        │
│      n = length(A)                                          │
│      B = array of size n  // 출력 배열                       │
│      C = array of size k+1  // 카운트 배열, 0으로 초기화      │
│                                                              │
│      // Step 1: 각 원소 카운트                                  │
│      for i = 0 to n-1:                                      │
│          C[A[i]] = C[A[i]] + 1                              │
│                                                              │
│      // Step 2: 누적 합 계산                                   │
│      for i = 1 to k:                                        │
│          C[i] = C[i] + C[i-1]                               │
│                                                              │
│      // Step 3: 출력 배열 구성 (역순으로 순회하여 안정성 확보)   │
│      for i = n-1 downto 0:                                  │
│          B[C[A[i]] - 1] = A[i]                              │
│          C[A[i]] = C[A[i]] - 1                              │
│                                                              │
│      return B                                               │
│                                                              │
│  [시간 복잡도 분석]                                            │
│  ────────────────────────────────────────────                 │
│  Step 1 (카운트): O(N)                                       │
│  Step 2 (누적 합): O(K)                                      │
│  Step 3 (배치): O(N)                                         │
│  총 시간: O(N + K)                                           │
│                                                              │
│  [공간 복잡도 분석]                                            │
│  ────────────────────────────────────────────                 │
│  출력 배열 B: O(N)                                          │
│  카운트 배열 C: O(K)                                         │
│  총 공간: O(N + K)                                          │
│                                                              │
│  [안정성 (Stability)]                                         │
│  ────────────────────────────────────────────                 │
│  역순으로 순회하는 이유:                                       │
│  같은 값을 가진 원소들이 원래 순서를 유지하기 위해서            │
│  ( 안정 정렬 )                                               │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 카운팅 정렬의 시간 복잡도는 O(N + K)이며, K가 N보다 작으면 O(N)이다.
- **원인**: 원소의 범위 K가 제한적이라면, K에 비례하는 작업만으로 정렬이 완료된다.
- **결과**: 비교 기반 정렬의 하한 O(N log N)보다 빠를 수 있다.
- **판단**: 원소 범위가 N에 비례하는 경우(K = O(N)), 카운팅 정렬은 선형 시간에 정렬된다.

📢 **섹션 요약 비유**: 카운팅 정렬은 **도서관 Books整理의自動化**와 같습니다. 책의 주제(값)를 확인하고, 각 주제별로棚に何人いるか数え、上位부터 순서대로棚에 배치한다. 책의 제목을 直接比較하지 않고, 주제별人数만 세면 된다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

카운팅 정렬의 실무 적용은 원소 범위가 제한적인 경우에 효과적이다. 특히 **정수 정렬**, **빈도수 기반 정렬**, **기수 정렬의 한 단계** 등으로 활용된다.

**기수 정렬(Radix Sort)과의 관계**: 카운팅 정렬은 기수 정렬의 개별 자릿수를 정렬하는 데 사용된다. 기수 정렬은 각 자릿수를 독립적으로 정렬하며, 각 자릿수 정렬에 카운팅 정렬을 활용한다.

```text
[카운팅 정렬 구현]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [Python 구현]                                               │
│  ────────────────────────────────────────────                │
│                                                              │
│  def counting_sort(arr, max_val):                           │
│      n = len(arr)                                           │
│      count = [0] * (max_val + 1)                            │
│      result = [0] * n                                       │
│                                                              │
│      # Step 1: 카운트                                        │
│      for val in arr:                                        │
│          count[val] += 1                                     │
│                                                              │
│      # Step 2: 누적 합                                        │
│      for i in range(1, max_val + 1):                        │
│          count[i] += count[i - 1]                            │
│                                                              │
│      # Step 3: 역순으로 배치 (안정성 확보)                     │
│      for val in reversed(arr):                               │
│          result[count[val] - 1] = val                       │
│          count[val] -= 1                                     │
│                                                              │
│      return result                                           │
│                                                              │
│  [실무 응용]                                                 │
│  ────────────────────────────────────────────                 │
│   1. 정수 정렬: 나이, 점수, 카운트 등 정수 데이터             │
│   2. 범위 제한 데이터: 0-100 사이의 성적 정렬                 │
│   3. 기수 정렬의 자릿수 정렬: LSD/MSD 기수 정렬               │
│   4. 채널 도메인: 네트워크 패킷의 우선순위 정렬               │
│                                                              │
│  [주의사항]                                                   │
│  ────────────────────────────────────────────                 │
│   1. 원소 범위 K가 너무 크면 공간 비효율 발생                  │
│   2. 음수 처리 시 offset 조정이 필요                           │
│   3. 실수에는 직접 적용 불가 (Bucket Sort 사용)              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 카운팅 정렬은 **체육관 출석부 정리**와 같습니다. 각 반(0반부터 6반까지)에 몇 명이 있는지 세고, 0반부터 순서대로 배치한다. 학생 이름을 하나하나 비교하지 않고, 반별人数만 세면 된다.

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

카운팅 정렬의品質 管理에서 가장 중요한 것은 **안정성(Stability)**과 **범위 밖 원소 처리**이다.

**품질 관리 체크리스트**: 같은 값을 가진 원소들이 원래 순서를 유지하는지(안정성) 확인한다.입력 배열의 최대값이 카운트 배열 크기보다 큰 경우를 처리한다.음수가 포함된 경우 offset 처리가 올바른지 확인한다.

```text
[카운팅 정렬 테스트 케이스]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [테스트 시나리오]                                             │
│  ────────────────────────────────────────────                 │
│                                                              │
│  1. 기본 정렬 테스트                                          │
│     - 입력: [4, 2, 4, 3, 1, 4, 2, 3]                         │
│     - 출력: [1, 2, 2, 3, 3, 4, 4, 4]                         │
│                                                              │
│  2. 중복 원소 안정성 테스트                                     │
│     - 입력: [(3,'a'), (1,'b'), (3,'c'), (1,'d')]              │
│     (값, 원래 순서)                                           │
│     - 출력: [(1,'b'), (1,'d'), (3,'a'), (3,'c')]             │
│     (같은 값에서 원래 순서 유지)                              │
│                                                              │
│  3. 경계 조건 테스트                                          │
│     - 빈 배열: []                                            │
│     - 원소 1개: [5]                                          │
│     - 모든 원소가 동일한 값: [3, 3, 3, 3]                     │
│     - 이미 정렬된 배열: [1, 2, 3, 4, 5]                       │
│                                                              │
│  4. 범위 테스트                                               │
│     - 최댓값이 예상보다 큰 경우: [1, 100, 50]                  │
│     - 음수 포함: [-2, 0, 3, -1, 2]                            │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 카운팅 정렬의品質 管理는 **인구普查의 정확도**와 같습니다. 각 연령대에 몇 명이 있는지 세고, 연령순으로 정리한다. 같은 연령대의 사람들이 순서대로 배열되어야 하며, 범위를 벗어난 데이터는 누락되지 않아야 한다.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

카운팅 정렬의 최신 동향으로는 **기수 정렬과의 결합**, **병렬 구현**, **메모리 최적화** 등이 연구되고 있다.

**기수 정렬(Radix Sort)**: 카운팅 정렬을 각 자릿수에 적용하여 O(d(N + k)) 시간에 정렬한다. 여기서 d는 자릿수, k는 기수(보통 10 또는 16)이다. 문자열, 날짜 등 다양한 데이터 형식에 적용 가능하다.

**적용 범위**: 카운팅 정렬은 원소 범위가 N에 비례하는 상황에서 가장 효과적이다. 원소 범위 K가 N보다 훨씬 크면, 공간 효율성 문제로 인해 다른 알고리즘이 더 적합할 수 있다.

📢 **섹션 요약 비유**: 카운팅 정렬は **色の棒を长度順に並べる**と似ています。まず、各色の棒的数量を数え、最も少ない色から始めて、長い棒の配置することで、比較なしで完成できます。
