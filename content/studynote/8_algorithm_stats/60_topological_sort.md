# 60. 위상 정렬 (Topological Sort)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 위상 정렬(Topological Sort)은 방향 비순환 그래프(Directed Acyclic Graph, DAG)에서 모든 간선의 방향을 따라 nodes를 선형으로 배치하는 算法이다.
> 2. **가치**: 작업의 우선순위(어떤 작업이 먼저 수행되어야 하는지)를 결정하거나,Course 时间표 작성, 빌드 系统 등에서 활용된다.
> 3. **융합**: 프로젝트 일정 관리, Course 수강 순서 결정, 데이터 처리 파이프라인, 의존성解析 등 선후 관계가 있는 문제에 활용된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

위상 정렬(Topological Sort)은 방향 비순환 그래프(Directed Acyclic Graph, DAG)에서 수행되는 정렬이다. DAG는 사이클이 없는 방향 그래프로, 모든 간선이 어떤 노드에서 다른 노드로의 "선행 관계"를 나타낸다. 위상 정렬은 이러한 선행 관계를尊重하여 nodes를 선형으로排列한다.

위상 정렬이 중요한 이유는 **선후 관계의 표현과 해결** 때문이다. 예를 들어, 대학 Course의 선수과목을 생각해보자. "자료구조"를 들으려면 "프로그래밍 기초"를 먼저 들어야 하고, "알고리즘"을 들으려면 "자료구조"를 먼저 들어야 한다. 이러한 선수과목 관계를 그래프로 표현하고, 위상 정렬을 통해 Course를 이수할 수 있는 순서를 찾을 수 있다.

위상 정렬은 **항상 존재하는 것이 아니다**는 점에 유의해야 한다. 사이클이 존재하면 "선행 관계"가 순환하게 되어 위상 정렬이 불가능하다. 따라서 위상 정렬을 수행하기 전에 반드시 그래프가 DAG인지 확인해야 한다.

```text
[위상 정렬 예시: Course 선수과목]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [Course 의존성 그래프]                                        │
│  ────────────────────────────────────────────                │
│                                                              │
│    프로그래밍 기초                                             │
│         │                                                     │
│         ▼                                                     │
│    자료구조 ◄──── 알고리즘                                     │
│         │                   ▲                                 │
│         ▼                   │                                 │
│    운영체제              고급 알고리즘                          │
│                                                              │
│  [위상 정렬 결과]                                              │
│  ────────────────────────────────────────────                │
│                                                              │
│  순서 1: 프로그래밍 기초 → 자료구조 → 알고리즘 → 고급 알고리즘│
│  순서 2: 프로그래밍 기초 → 자료구조 → 운영체제 → 알고리즘     │
│  (여러 유효한 순서가 존재할 수 있음)                          │
│                                                              │
│  [핵심 용어]                                                  │
│  ────────────────────────────────────────────                │
│  • 진입 차수 (In-degree): 해당 노드로 들어오는 간선의 수       │
│  • 진출 차수 (Out-degree): 해당 노드에서 나가는 간선의 수     │
│  • DAG: 사이클이 없는 방향 그래프                              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 위상 정렬의 결과는 유일하지 않을 수 있다. 여러 유효한 순서가 존재할 수 있다.
- **원인**: 동시에 여러 작업이 수행 가능할 때(진입 차수 0인 노드가 여러 개일 때) 어떤 것을 먼저 처리해도 정렬이 유효하기 때문이다.
- **결과**: 위상 정렬이 존재하면 모든 노드를 나열할 수 있고, 존재하지 않으면 사이클이 있다는 것을 알 수 있다.
- **판단**: 작업 간 선후 관계가 있는 문제에서 먼저 DAG인지 확인해야 한다.

📢 **섹션 요약 비유**: 위상 정렬은 **요리レシピ**와 같습니다. "반죽을 만든 후 오븐에 넣고 굽는다"는 순서가 있는 레시피에서, 독립적인 작업(양파 자르기, 재료 준비)을 먼저 할지 말지는 취향에 따라 다를 수 있으나, "반죽 → 굽기"의 순서는 반드시 지켜야 한다. 위상 정렬은 이러한 선후 관계를 만족하는 다양한 조리 순서를 찾아준다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

위상 정렬에는 두 가지主要的 알고리즘이 있다. **칸나흐(Kahn) 알고리즘**은 BFS 기반으로, 진입 차수가 0인 노드부터 큐에 넣고 처리한다. **DFS 기반 알고리즘**은 DFS를 수행한 후, 노드 방문을完了한 역순으로 결과를 수집한다.

**칸나흐 알고리즘**의 과정은 다음과 같다. 첫째, 모든 노드의 진입 차수를計算한다.둘째, 진입 차수가 0인 노드를 큐에 넣는다.셋째, 큐가 empty할 때까지: 현재 노드를 큐에서_pop하고 결과에 추가한다. 현재 노드의 모든 이웃 노드에 대해 진입 차수를 감소시킨다. 진입 차수가 0이 된 이웃 노드를 큐에 추가한다.

**DFS 기반 알고리즘**의 과정은 다음과 같다. 첫째, 모든 노드를 unmarked로 초기화한다.둘째, 각 unmarked 노드에 대해 DFS를 수행한다.셋째, DFS에서 노드의 모든 이웃을 처리한 후, 노드를 결과 리스트의 앞에 추가한다.

```text
[위상 정렬 알고리즘 상세]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [칸나흐 (Kahn) 알고리즘]                                      │
│  ────────────────────────────────────────────                │
│                                                              │
│  function KahnTopologicalSort(graph):                       │
│      // 1. 진입 차수 계산                                     │
│      in_degree = [0] * V                                     │
│      for each edge (u, v) in graph:                         │
│          in_degree[v] += 1                                   │
│                                                              │
│      // 2. 진입 차수 0인 노드 큐에 추가                        │
│      queue = []                                              │
│      for i in range(V):                                      │
│          if in_degree[i] == 0:                               │
│              queue.append(i)                                 │
│                                                              │
│      // 3. BFS 기반 처리                                      │
│      result = []                                             │
│      while queue:                                            │
│          node = queue.pop(0)                                 │
│          result.append(node)                                 │
│                                                              │
│          for each neighbor of node:                         │
│              in_degree[neighbor] -= 1                        │
│              if in_degree[neighbor] == 0:                   │
│                  queue.append(neighbor)                      │
│                                                              │
│      if len(result) != V:                                   │
│          return "사이클 존재 - 위상 정렬 불가"                │
│      return result                                           │
│                                                              │
│  [DFS 기반 알고리즘]                                          │
│  ────────────────────────────────────────────                │
│                                                              │
│  function DFSTopologicalSort(graph):                        │
│      visited = [False] * V                                   │
│      result = []                                             │
│                                                              │
│      function DFS(node):                                     │
│          visited[node] = True                                │
│          for each neighbor of node:                         │
│              if not visited[neighbor]:                      │
│                  DFS(neighbor)                               │
│          result.append(node)  // 모든 이웃 처리 후 추가       │
│                                                              │
│      for i in range(V):                                      │
│          if not visited[i]:                                  │
│              DFS(i)                                          │
│                                                              │
│      return result.reverse()  // 역순 반환                   │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 칸나흐 알고리즘은 "진입 차수 0 = 선행 작업 없음"을 利用한다.
- **원인**: 진입 차수가 0이라는 것은 어떤 작업에도 선행되지 않는 작업이라는 의미이므로 먼저 수행해도 안전하다.
- **결과**: 모든 노드가 처리되면 선후 관계를 만족하는 순서가 완성된다.
- **판단**: 사이클이 존재하면 어떤 노드의 진입 차수도 0이 되지 않아 모든 노드를 처리할 수 없다.

📢 **섹션 요약 비유**: 칸나흐 알고리즘의 진입 차수 개념은 **요리 준비 과정**과 같습니다. 계란을 打つ前に 양파를 다져야 하고(양파 진입 차수 > 0), 계란을 打つ 것은 어떤 작업의 선행 조건도 없으므로(진입 차수 = 0) 가장 먼저 진행할 수 있다. 준비된 재료(진입 차수가 0이 된 재료)를 가지고 다음 요리를 진행한다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

위상 정렬의 실무 적용은 다음과 같다. **빌드 시스템**: Makefile에서 어떤 파일을 먼저 컴파일해야 하는지 결정한다. 파일 간 의존성을 그래프로 표현하고 위상 정렬로 컴파일 순서를 결정한다. **Course 시간표**: 선수과목을 고려하여 수강 가능한 순서를 찾는다. **데이터 처리 파이프라인**: ETL(Extract-Transform-Load) 과정에서 데이터 처리 순서를 결정한다.

**활용: 작업 스케줄링**: 여러 작업이 있고,某些 작업은 다른 작업 완료后才能 시작할 때, 위상 정렬로 실행 가능한 순서를 찾는다. 각 작업에 소요 시간을 고려하면 крити路径 분석도 가능하다.

```text
[위상 정렬 실무 활용]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [빌드 시스템 예시]                                            │
│  ────────────────────────────────────────────                │
│                                                              │
│  파일 의존성:                                                 │
│    main.o : main.cpp app.h                                   │
│    app.o : app.cpp app.h lib.h                              │
│    lib.o : lib.cpp lib.h                                    │
│    app : main.o app.o lib.o                                 │
│                                                              │
│  위상 정렬 결과 (한 가지 가능한 순서):                         │
│  1. lib.cpp → lib.o (lib.h 먼저 컴파일)                      │
│  2. app.cpp → app.o (app.h, lib.h 의존)                    │
│  3. main.cpp → main.o (main.h, app.h 의존)                  │
│  4. main.o app.o lib.o → app (링크)                         │
│                                                              │
│  [데이터 처리 파이프라인]                                      │
│  ────────────────────────────────────────────                │
│                                                              │
│  데이터 소스 → 정제 → 변환 → 적재 → 분석                      │
│      │         │        │        │                         │
│      ▼         ▼        ▼        ▼                          │
│  정제 전에   변환前   적재前   분석前                         │
│  소스 완료   정제 완료 변환 완료 적재 완료                     │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 위상 정렬 실무 활용은 **大型 건团 프로젝트 계획**과 같습니다. "설계 →地基 시공 → 구조물 건설 → 내부 시설 설치 → 인테리어" 순으로, 각 단계가 완료되어야 다음 단계가 시작될 수 있다. 위상 정렬은 이러한 선후 관계를 자동으로分析하여 최적의 프로젝트 진행 순서를 찾아준다.

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

위상 정렬의 품질 관리에서 가장 중요한 것은 **사이클 检测**과 **모든 노드 처리 확인**이다.

**품질 관리 체크리스트**: 결과 리스트의 길이가 노드 수와 같은지 확인하여 모든 노드가 처리되었는지 검증해야 한다. 사이클이 존재하면 위상 정렬이 불가능하므로 이를 사용자에게 알려야 한다. 입력 그래프가 DAG인지 먼저 확인하면 에러를 미리 방지할 수 있다.

📢 **섹션 요약 비유**: 위상 정렬 품질 관리는 **의존성.circular参照 검사**와 같습니다. "A는 B를 참조하고, B는 C를 참조하고, C는 A를 참조한다"는 순환 참조가 있으면, 어떤 작업부터 시작해야 할지決定할 수 없다. 이를检测하여 순환 참조가 있음을 보고해야 한다.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

위상 정렬의 최신 동향은 **병렬 처리**와 **실시간 스케줄링**과 관련되어 있다. **병렬 빌드 시스템**: 위상 정렬로 결정된 작업을 여러 코어에서 병렬로 실행하여 빌드 시간을 단축한다. **ワークフロー 엔진**: Airflow, Luigi 등의 워크플로우 엔진에서 작업 실행 순서를 결정하는 데 활용된다.

위상 정렬은 "선후 관계가 있는 문제를 해결하는 기본 도구"로서, 컴퓨터 과학의 다양한 영역에서 활용된다.

📢 **섹션 요약 비유**: 위상 정렬의 최신 트렌드는 **로켓 발사 시퀀스 자동화**와 같습니다. 수백 개의 확인 절차를 선후 관계에 따라 정리하고, 자동으로 오류를 检测하며, 가능한 작업을 병렬로 실행하여 발사 준비 시간을 단축한다.
