# 36. Floyd-Warshall 알고리즘

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 플로이드-워샬(Floyd-Warshall) 알고리즘은 모든 정점 쌍 간의 최단 경로를 한 번에 구하는 동적 프로그래밍 기반 알고리즘이다.
> 2. **가치**: O(V³) 시간 복잡도로 모든 쌍 최단 경로를 구할 수 있으며, 음수 가중치도 처리가 가능하고 음수 사이클 검출도 가능하다.
> 3. **융합**: 경유지 개념을 활용하여 점진적으로 최단 거리를 更新하며, 결과적으로 각 정점 쌍 간의最短경로를 담은 행렬을 생성한다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

플로이드-워샬(Floyd-Warshall) 알고리즘은 1962년 Robert Floyd와 Peter Ingerman에 의해 독립적으로 개발되었다. 모든 정점 쌍 간의 최단 경로(All Pairs Shortest Path) 문제를 해결하는 알고리즘으로, V개의 정점에 대해 O(V³) 시간에 모든 쌍 최단 거리를 구한다.

플로이드-워샬 알고리즘이 중요한 이유는 **전체 쌍 최단 경로의 단일Shot 계산** 때문이다. 다익스트라를 V번 실행하면 O(V² log V + VE)이고, Bellman-Ford를 V번 실행하면 O(V²E)인데, 플로이드-워샬은 일관된 O(V³)으로 더 간단하며, 중간 정점 개념을 활용한 동적 프로그래밍으로 구현이 명료하다.

```text
[플로이드-워샬 알고리즘 개요]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [그래프 예시]                                                │
│  ────────────────────────────────────────────                │
│                                                              │
│        A ---4--- B                                          │
│       /|         \                                         │
│      1 |          2                                         │
│     /  |           \                                        │
│    S---3--- C ---5--- D                                     │
│              |                                               │
│              6                                               │
│              |                                               │
│              E                                               │
│                                                              │
│  [플로이드-워샬 동작 원리]                                     │
│  ────────────────────────────────────────────                 │
│                                                              │
│  핵심 아이디어: "중간 정점을 경유하여 더 짧은 경로 찾기"         │
│                                                              │
│  D[k][i][j] = 정점 0~k-1를 중간 정점으로만 利用하여             │
│                  i에서 j까지의最短경로 거리                   │
│                                                              │
│  전이 관계:                                                   │
│  D[k][i][j] = min(D[k-1][i][j], D[k-1][i][k] + D[k-1][k][j]) │
│                                                              │
│  [단계별 예시]                                                │
│  ────────────────────────────────────────────                 │
│                                                              │
│  D[0]: k=0 사용 전 (직접 연결만)                              │
│        S   A   B   C   D   E                                 │
│    S [ 0,  1,  4,  3,  ∞,  ∞]                               │
│    A [ 1,  0,  4,  ∞,  ∞,  ∞]                               │
│    B [ 4,  4,  0,  5,  2,  ∞]                               │
│    C [ 3,  ∞,  5,  0,  6,  ∞]                               │
│    D [ ∞,  ∞,  2,  6,  0,  ∞]                               │
│    E [ ∞,  ∞,  ∞,  ∞,  ∞,  0]                               │
│                                                              │
│  D[1]: k=1(A)을 중간 정점으로 이용                            │
│    S→B: min(4, 1+4)=4 (변화 없음)                           │
│    S→C: min(3, 1+∞)=3 (변화 없음)                           │
│    ...                                                       │
│                                                              │
│  D[2]: k=2(B)을 중간 정점으로 이용                            │
│    S→D: min(∞, 4+2)=6  ← 경유地发现                         │
│    C→E: min(∞, 5+2)=7  ← 경유地发现                         │
│    ...                                                       │
│                                                              │
│  [최종 결과]                                                 │
│  ────────────────────────────────────────────                 │
│  모든 정점 쌍 간 최단 거리 행렬 완성                           │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 플로이드-워샬은 동적 프로그래밍을 활용하여 모든 정점 쌍의 최단 거리를 구한다.
- **원인**: "정점 k를 중간에 경유할 수 있다면, k를 경유하지 않는 기존 경로와 비교한다"는 원리.
- **결과**: V개의 정점에 대해 O(V³) 시간에 모든 쌍 최단 경로를 구한다.
- **판단**: 전체 쌍 최단 경로가 필요하면 플로이드-워샬이 효율적이다.

📢 **섹션 요약 비유**: 플로이드-워샬 알고리즘은 **도시 간 모든 경로의最短거리를 계산하는 것**과 같습니다. 1번 도시를 경유하는 경우를 계산하고, 2번 도시를 추가 경유하는 경우를 계산하며, 모든 도시를 경유할 때까지 반복한다. 각 단계에서 더 짧은 경로를 찾으면更新한다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

플로이드-워샬 알고리즘의 핵심은 **동적 프로그래밍 테이블 갱신**과 **중간 정점 경유 개념**이다.

**동적 프로그래밍 원리**: D[k][i][j]는 "정점 0부터 k까지의 정점만을 중간에 경유할 수 있을 때, i에서 j까지의 최단 거리"를 의미한다. k가 증가할수록 더 많은 정점을 중간 경유지로 사용할 수 있게 된다.

**점화식**: D[k][i][j] = min(D[k-1][i][j], D[k-1][i][k] + D[k-1][k][j]) — k번째 정점을 경유하지 않는 경로와 k번째 정점을 경유하는 경로 중 더 짧은 것을 선택한다.

```text
[플로이드-워샬 알고리즘 상세]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [플로이드-워샬 의사코드]                                      │
│  ────────────────────────────────────────────                │
│                                                              │
│  function Floyd-Warshall(graph):                             │
│      V = number of vertices                                  │
│      dist = 2D array of size V x V                           │
│                                                              │
│      // Step 1: 초기화                                        │
│      for i = 0 to V-1:                                      │
│          for j = 0 to V-1:                                  │
│              if i == j:                                     │
│                  dist[i][j] = 0                             │
│              else if edge(i, j) exists:                     │
│                  dist[i][j] = weight(i, j)                  │
│              else:                                           │
│                  dist[i][j] = INFINITY                      │
│                                                              │
│      // Step 2: 동적 프로그래밍                               │
│      for k = 0 to V-1:  // 경유지 정점                       │
│          for i = 0 to V-1:  // 출발 정점                     │
│              for j = 0 to V-1:  // 도착 정점                │
│                  if dist[i][k] + dist[k][j] < dist[i][j]:  │
│                      dist[i][j] = dist[i][k] + dist[k][j]  │
│                                                              │
│      // Step 3: 음수 사이클 검출                              │
│      for i = 0 to V-1:                                      │
│          if dist[i][i] < 0:                                 │
│              return "음수 사이클 존재"                         │
│                                                              │
│      return dist                                             │
│                                                              │
│  [시간 복잡도 분석]                                            │
│  ────────────────────────────────────────────                 │
│  3중 루프: O(V³)                                             │
│                                                              │
│  [공간 복잡도 분석]                                            │
│  ────────────────────────────────────────────                 │
│  거리 행렬: O(V²)                                            │
│  (경로 재구성 시 path 행렬 추가: O(V²))                      │
│                                                              │
│  [다익스트라/V번 vs Floyd-Warshall 비교]                      │
│  ────────────────────────────────────────────                 │
│                                                              │
│  │ 방법              │ 시간 복잡도      │ 적합 상황           │
│  ────────────────────────────────────────────                  │
│  │ 다익스트라 V번    │ O(V² log V + VE) │ 희소 그래프 (E << V²)│
│  │ Bellman-Ford V번 │ O(V²E)          │ 음수 가중치 포함    │
│  │ Floyd-Warshall   │ O(V³)          │ 밀집 그래프 (E ≈ V²)│
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 플로이드-워샬은 3중 루프로 모든 정점 쌍을 처리한다.
- **원인**: 경유지 k가 증가함에 따라 사용 가능한 중간 정점이 늘어나고, 이를 동적 프로그래밍으로 更新한다.
- **결과**: 모든 정점 쌍 간의 최단 거리를 O(V³)에 구할 수 있다.
- **판단**: 밀집 그래프에서 모든 쌍 최단 경로가 필요하면 플로이드-워샬이 효율적이다.

📢 **섹션 요약 비유**: 플로이드-워샬 알고리즘은 **국제 항공편的所有 경로 분석**과 같습니다. 각 공항을 경유지로段階적으로追加하여, 더 짧은 경로가 발견되면更新한다. 모든 공항을 경유지로 사용할 수 있는 상황이 되면 최종적인 모든 경로의最短거리가 确定된다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

플로이드-워샬 알고리즘의 실무 적용은 **경로 계획**, **네트워크 분석**, **게임 개발** 등에서 활용된다.

**네트워크 라우팅**: 각 라우터 간의遅延을 최소화하는 경로를 찾을 때 사용된다.

```text
[플로이드-워샬 알고리즘 구현]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [Python 구현]                                               │
│  ────────────────────────────────────────────                │
│                                                              │
│  def floyd_warshall(vertices, edges):                       │
│      V = len(vertices)                                       │
│      dist = [[float('inf')] * V for _ in range(V)]          │
│                                                              │
│      for i in range(V):                                      │
│          dist[i][i] = 0                                      │
│                                                              │
│      for u, v, w in edges:                                  │
│          dist[u][v] = w                                      │
│          dist[v][u] = w  # 무방향 그래프의 경우               │
│                                                              │
│      # 동적 프로그래밍                                        │
│      for k in range(V):                                      │
│          for i in range(V):                                  │
│              for j in range(V):                              │
│                  if dist[i][k] + dist[k][j] < dist[i][j]:   │
│                      dist[i][j] = dist[i][k] + dist[k][j]    │
│                                                              │
│      # 음수 사이클 검출                                        │
│      for i in range(V):                                      │
│          if dist[i][i] < 0:                                  │
│              return None  # 음수 사이클 존재                  │
│                                                              │
│      return dist                                             │
│                                                              │
│  [경로 재구성 구현]                                           │
│  ────────────────────────────────────────────                │
│                                                              │
│  def floyd_warshall_with_path(vertices, edges):             │
│      V = len(vertices)                                       │
│      dist = [[float('inf')] * V for _ in range(V)]          │
│      next_node = [[None] * V for _ in range(V)]             │
│                                                              │
│      for i in range(V):                                      │
│          dist[i][i] = 0                                      │
│                                                              │
│      for u, v, w in edges:                                  │
│          dist[u][v] = w                                      │
│          next_node[u][v] = v                                 │
│          dist[v][u] = w                                      │
│          next_node[v][u] = u                                 │
│                                                              │
│      for k in range(V):                                      │
│          for i in range(V):                                  │
│              for j in range(V):                              │
│                  if dist[i][k] + dist[k][j] < dist[i][j]:   │
│                      dist[i][j] = dist[i][k] + dist[k][j]   │
│                      next_node[i][j] = next_node[i][k]      │
│                                                              │
│      return dist, next_node                                  │
│                                                              │
│  def get_path(next_node, start, end):                       │
│      if next_node[start][end] is None:                      │
│          return None  # 경로 없음                            │
│          path = [start]                                      │
│          current = start                                     │
│          while current != end:                               │
│              current = next_node[current][end]              │
│              path.append(current)                            │
│          return path                                         │
│                                                              │
│  [실무 응용]                                                 │
│  ────────────────────────────────────────────                 │
│   1. 네트워크 라우팅: 모든 라우터 쌍 간 최단 경로            │
│   2. 게임 개발: NPC 간 최단 경로 계산                         │
│   3. 운송 네트워크: 모든 도시 쌍 간最短거리                   │
│   4. робототехника: 다중 로봇 경로 planning                  │
│   5. 사회 네트워크 분석: 유력도中心성 계산                    │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 플로이드-워샬 알고리즘은 **철도 네트워크 전체 분석**과 같습니다. 모든 역 쌍 간의最短거리를 계산하기 위해, 각 역을 차례로 경유지로 추가하며 더 짧은 경로를 찾으면更新한다. 모든 역을 경유지로 사용할 수 있으면 전체 네트워크의 최단 경로가 完成된다.

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

**품질 관리 체크리스트**: 자기 자신으로의 거리가 0인지 확인한다.대칭 그래프에서 거리 행렬이 대칭인지 확인한다.음수 사이클이 있을 때 대각선이 음수인지 확인한다.모든 정점 쌍의 거리가 올바른지 확인한다.

│     - 입력: 음수 가중치 포함 그래프                            │

📢 **섹션 요약 비유**: 플로이드-워샬 알고리즘의品質 管理는 **국제 무역 협정 분석**과 같습니다. 모든 국가 쌍 간의 무역 비용을 분석하고, 순환하면 이익이 되는 상황(음수 사이클)을 탐지한다. 대각선(자국 무역)이 0보다 작으면 차익거래 기회가 있는 것이다.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

플로이드-워샬 알고리즘의 최신 동향으로는 **병렬 구현**, **GPU 가속**, **희소 그래프 최적화** 등이 있다.

**병렬 플로이드-워샬**: 3중 루프의 특성을 利用하여 행렬 연산으로 변환하면, GPU에서 대량 병렬 처리가 가능하다.

**플로이드-워샬의Positions**: O(V³) 시간 복잡도로 인해 수천 개 이상의 정점에는 실용적이지 않을 수 있지만, 수백 개 수준의 정점에서는 모든 쌍 최단 경로를 구하는 가장 간단한 방법이다. 밀집 그래프에서 특히 효율적이다.

📢 **섹션 요약 비유**: 플로이드-워샬 알고리즘は **宇宙探査の全航线分析**と似ています。すべての星系間の最短距離を一度に計算し、経由地を追加しながら路径を改善していく。シンプルだが强大的な算法です。
