# 59. 크루스칼/프림 알고리즘 (Kruskal's / Prim's Algorithm)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 크루스칼(Kruskal)과 프림(Prim) 알고리즘은 주어진 그래프에서 최소 비용 신장 트리(Minimum Spanning Tree, MST)를 찾는 그리디 알고리즘이다.
> 2. **가치**: 두 알고리즘 모두 그리디 접근법을 사용하지만, 크루스칼은 간선 기반, 프림은 노드 기반으로 MST를 구축한다.
> 3. **융합**: 네트워크 설계(通信网, 도로망), 집합 커버, 클러스터링 등 범위가 있는 문제에 광범위하게 활용된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

최소 신장 트리(Minimum Spanning Tree, MST)는 그래프의 모든 노드를 포함하되, 사이클 없이 최소 비용으로 연결하는 트리이다. 예를 들어, 여러 도시를 전화선으로 연결할 때 모든 도시를 연결하되 전화선의 총 길이를 최소화하는 문제가 MST에 해당한다.

크루스칼(Kruskal) 알고리즘은 1956년 Joseph Kruskal이 제안했으며, **간선을 비용순으로 정렬한 후, 사이클을 형성하지 않는 간선을 순차적으로 선택**한다. 프림(Prim) 알고리즘은 1957년 Robert Prim이 제안했으며, **시작 노드에서 출발하여 현재 트리에 연결된 최소 비용 간선을 선택하며 트리를 확장**한다.

두 알고리즘이 중요한 이유는 **네트워크 설계의 기본 도구**이기 때문이다. 통신网, 도로망, 전력망 등을 설계할 때 최소 비용으로 모든 지점을 연결해야 하는 문제는 MST로 모델링된다. 또한, 클러스터링, 집합 커버 등의 근접 문제도 MST를 통해 해결할 수 있다.

```text
[최소 신장 트리 (MST) 개요]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [그래프 예시]                                                │
│  ────────────────────────────────────────────                │
│                                                              │
│         4          3                                          │
│    A ------ B ------ C                                        │
│    |   2   |   1   |                                        │
│  6 |        |        | 2                                       │
│    |        |        |                                        │
│    D ------ E ------ F                                        │
│         5          7                                         │
│                                                              │
│  [MST 결과 (총 비용: 4+2+1+3+5 = 15)]                        │
│  ────────────────────────────────────────────                │
│                                                              │
│    A ------ B ------ C                                        │
│    |                  |                                        │
│  6 |               2  |                                       │
│    |                  |                                        │
│    D                  F                                        │
│    |                  |                                        │
│    D ------ E         |                                        │
│         5             |                                       │
│    (E-F 간선은 선택되지 않음)                                   │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: MST는 V-1개의 간선을 가진다(V는 노드 수).
- **원인**: V개의 노드를 연결하는 트리는 최소한 V-1개의 간선이 필요하고, 더 많으면 사이클이 발생하기 때문이다.
- **결과**: MST의 총 비용은 그래프의 모든 경로 중 최소이다.
- **판단**: 모든 지점을 최소 비용으로 연결하는 문제에서 MST가 활용된다.

📢 **섹션 요약 비유**: MST는 **도시 상하수도布置**와 같습니다. 모든 가구를 수도원으로 연결하되, 파이프의 총 길이를 최소화하고, 사이클이 없도록(한 방향으로만 흐르도록) 배관한다. 각 Household에 água가 도달하는 동시에 파이프 사용량을 최소화하는 것이 목표이다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**크루스칼 알고리즘**의 핵심 원리는 **유니온-파인드(Union-Find)** 자료구조를利用한 사이클 检测이다. 간선을 비용순으로 정렬한 후, 각 간선에 대해: 해당 간선이 연결하는 두 노드가 이미 같은 집합에 속해 있으면(유니온-파인드로 检测) 사이클이므로 건너뛰고, 다른 집합에 속해 있으면 MST에 추가하고 두 집합을 합친다.

**프림 알고리즘**의 핵심 원리는 **분리된 두 집합(탐색된/미탐색)** 유지와 **우선순위 큐** 활용이다. 시작 노드를 트리 집합에 추가한 후, 현재 트리에서 인접한 간선 중 최소 비용 간선을 선택하여 트리를 확장한다. 이를 모든 노드가 트리에 포함될 때까지 반복한다.

```text
[크루스칼 vs 프림 알고리즘 비교]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [크루스칼 알고리즘 과정]                                      │
│  ────────────────────────────────────────────                │
│                                                              │
│  간선 정렬: (A,D,2), (B,E,1), (B,C,3), (C,F,2), (D,E,4)...  │
│                                                              │
│  Step 1: (A,D,2) 선택 → A,D 합침                              │
│  Step 2: (B,E,1) 선택 → B,E 합침                              │
│  Step 3: (B,C,3) 선택 → B-E 집합과 C 결합                     │
│  Step 4: (C,F,2) 선택 → C-F 추가 (사이클 없음)                │
│  Step 5: (D,E,4) 건너뜀 (A,D,B,E,C,F 모두 같은 집합)         │
│  Step 6: (D,E,5) 선택 → 트리 완성                             │
│                                                              │
│  [프림 알고리즘 과정]                                          │
│  ────────────────────────────────────────────                │
│                                                              │
│  시작: A                                                     │
│  트리 = {A}, 경계 간선 = {(A,B,4), (A,D,2)}                  │
│                                                              │
│  Step 1: (A,D,2) 선택 → 트리 = {A,D}                         │
│          경계 = {(A,B,4), (D,E,4), (D,E,5)}                  │
│                                                              │
│  Step 2: (A,B,4) 선택 → 트리 = {A,D,B}                       │
│          경계 = {(B,C,3), (D,E,4), (B,E,1)}                   │
│                                                              │
│  Step 3: (B,E,1) 선택 → 트리 = {A,D,B,E}                     │
│          경계 = {(B,C,3), (D,E,4), (E,F,7)}                  │
│                                                              │
│  ... (반복)                                                  │
│                                                              │
│  [시간 복잡도 비교]                                            │
│  ────────────────────────────────────────────                │
│  │ 알고리즘   │ 시간 복잡도    │ 적합한 그래프          │   │
│  ├───────────┼────────────────┼─────────────────────────┤   │
│  │ 크루스칼  │ O(E log E)     │ 희소 그래프 (E << V²)  │   │
│  │ 프림      │ O(E log V)     │ 밀집/희소 모두 OK      │   │
│  │ (힙 사용) │                │                         │   │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 크루스칼은 전체 간선 중에서 선택하고, 프림은 현재 트리에서 확장한다.
- **원인**: 크루스칼은 "전체에서 최소"를, 프림은 "현재에서 최소"를 선택하는 그리디 접근법이기 때문이다.
- **결과**: 둘 다 동일한 MST(총 비용)를 보장한다.
- **판단**: 희소 그래프에서는 크루스칼, 밀집 그래프에서는 프림이 더 효율적이다.

📢 **섹션 요약 비유**: 크루스칼과 프림의 차이는 **책장 정리 전략**과 같습니다. 크루스칼은 방바닥에 흩어진 모든 책 중 가장 작은 책을 먼저拾い上げ(간선 선택), 이미 정리된 책과 같은 자리에 있으면 버리고(사이클 检测), 아니면 책장에 추가한다. 프림은 책장 왼쪽부터 채워가며(시작 노드), 현재 책장 근처에 있는 가장 작은 책을追加していく.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

MST 알고리즘의 실무 적용은 다음과 같다. **通信网 설계**: 전화선, 광케이블 등을 최소 비용으로 모든 가정에 연결하는 문제에 활용된다. **도로망 설계**: 모든 도시를 연결하는 highway를 최소 비용으로 건설하는 문제에 활용된다. **전기 power grids**: 전력망을 최소 비용으로 구축하는 문제에 활용된다.

**클러스터링**: K-클러스터링은 MST를 구성한 후, 가장 긴 K-1개 간선을 제거하여 K개의 클러스터로 분할한다. 이 방법은 계층적 클러스터링에 활용된다.

```text
[크루스칼 알고리즘 구현]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  function Kruskal(graph):                                   │
│      // 1. 간선 정렬                                          │
│      edges = sort(graph.edges, by=weight)                   │
│                                                              │
│      // 2. Union-Find 초기화                                   │
│      uf = UnionFind(graph.num_vertices)                      │
│      mst = []                                                │
│                                                              │
│      // 3. 간선 순회하며 MST 구축                              │
│      for each edge (u, v, w) in edges:                      │
│          if uf.find(u) != uf.find(v):                        │
│              mst.append((u, v, w))                          │
│              uf.union(u, v)                                  │
│                                                              │
│          if len(mst) == V - 1:                              │
│              break                                           │
│                                                              │
│      return mst                                              │
│                                                              │
│  [프림 알고리즘 구현]                                          │
│  ────────────────────────────────────────────                │
│                                                              │
│  function Prim(graph, start):                               │
│      pq = PriorityQueue()                                   │
│      visited = [False] * V                                  │
│                                                              │
│      pq.add((0, start, -1))  // (weight, node, parent)       │
│                                                              │
│      while pq is not empty:                                 │
│          w, u, parent = pq.pop()                            │
│                                                              │
│          if visited[u]:                                     │
│              continue                                        │
│          visited[u] = True                                  │
│                                                              │
│          if parent != -1:                                    │
│              mst.append((parent, u, w))                      │
│                                                              │
│          for each neighbor (v, weight) of u:                │
│              if not visited[v]:                              │
│                  pq.add((weight, v, u))                     │
│                                                              │
│      return mst                                              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: MST 실무 활용은 **아파트 광 Fiber 통신망 설계**와 같습니다. 각 세대(노드)에光纤 케이블을 연결하되, 전체 케이블 길이의 합을 최소화하면서도 모든 세대가 연결되도록 설계한다. 크루스칼은 가장 짧은 케이블부터 먼저 깔고, 프림은 현재 연결된 집합에서 가장 가까운未接続 세대에 연결하는 방식으로 진행한다.

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

MST 알고리즘의 품질 관리에서 가장 중요한 것은 **사이클 检测의 정확성**과 **연결 그래프 여부**이다.

**품질 관리 체크리스트**: Union-Find의 find와 union 연산이 정확하게 동작해야 한다. 그래프가 연결되어 있지 않으면 MST를 구성할 수 없으므로 예외 처리가 필요하다. 동일한 비용의 간선이 여러 개 있을 때 다른 MST가 될 수 있으나, 총 비용은 동일하다.

📢 **섹션 요약 비유**: MST 품질 관리는 **도시計画 Validate**와 같습니다. 모든 도시가正しく 연결되어 있는지(연결 그래프), 순환 도로가 없는지(사이클 없음), 총 도로 길이가 최소화되었는지(최소 비용)를 확인해야 한다.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

MST 알고리즘의 최신 동향은 **대규모 그래프 처리**와 **approximate MST**와 관련되어 있다. **병렬 크루스칼**: 정렬된 간선을 병렬로 처리하여 대규모 그래프에서 성능을 향상시킨다. **K-클러스터링**: MST를利用した階層的 클러스터링이 기계학습에서 활용된다.

크루스칼과 프림은 "MST 찾기의 양대 알고리즘"으로서, 각각의 장점(크루스칼: 구현 간단, 희소 그래프 효율적 / 프림: 밀집 그래프 효율적)을 이해하고 상황에 맞게 선택하는 것이 중요하다.

📢 **섹션 요약 비유**: MST 알고리즘의 발전은 **마을水道 시스템 최적화**와 같습니다. 초기에는 간단한 greedy 접근법으로 최소 비용을 찾았지만, 현대에는 컴퓨터 알고리즘으로 대규모 마을 전체의水道망을瞬時に 최적화할 수 있다. 이는 주민들에게 저렴한 비용으로 서비스를 제공할 수 있게 했다.
