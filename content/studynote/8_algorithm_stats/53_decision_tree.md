# 53. 결정 트리 알고리즘 (Decision Tree Algorithm)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 결정 트리(Decision Tree)는 데이터를 특징 based的特征으로 분할하여 분류 또는 회귀를 수행하는 트리 기반 머신러닝 알고리즘이다.
> 2. **가치**: 결과가 해석하기 쉽고, 수치형/범주형 데이터를 모두 처리하며,.feature 선택과 불순도 측정을 통해 최적의 분할을 학습한다.
> 3. **융합**: 결정 트리는 앙상블 방법론인 랜덤 포레스트와 그래디언트 부스팅의 기반이 된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

결정 트리(Decision Tree)는 머신러닝에서 가장 직관적이고 해석 가능한 알고리즘 중 하나이다. 트리 구조로 표현되며, 각 내부 노드는 하나의 특징에 대한 테스트를, 각 leaf 노드는 최종 예측 결과를 나타낸다. 새로운 데이터에 대해 root에서 leaf까지 조건들을 따라가며 예측을 수행한다.

결정 트리가 중요한 이유는 **높은 해석 가능성**과 **전처리 불필요** 때문이다. 의학적 진단이나 금융 규제 준수 같이 왜 그렇게 판단했는지 설명해야 하는 영역에서 결정 트리의 투명성은 매우 중요하다. 또한, 정규화나 표준화가 필요 없고, 범주형과 수치형 데이터를 모두 직접 처리할 수 있다.

결정 트리는 **정보 이득(Information Gain)** 또는 **지니 불순도(Gini Impurity)**를 기준으로 최적의 분할 특징과 임계값을 선택한다. 불순도가 낮아지는 방향으로 트리를 성장시키며, 이를 통해 데이터의 패턴을 자동으로 발견한다.

```text
[결정 트리 구조]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [결정 트리 예시:是否会购买商品？]                              │
│  ────────────────────────────────────────────                │
│                                                              │
│                    [나이 >= 30?]                               │
│                    /         \                                │
│                  Yes          No                              │
│                  ▼              ▼                              │
│           [소득 >= 5000만?]    [흥미 있는 카테고리?]          │
│            /        \           │                             │
│          Yes        No         │                             │
│          ▼            ▼        ▼                             │
│      [구매함]      [흥미 >= 높?]  ...                         │
│                        /    \                                 │
│                       Yes    No                               │
│                       ▼       ▼                               │
│                   [구매함]   [구매않함]                        │
│                                                              │
│  [용어 정리]                                                  │
│  ────────────────────────────────────────────                │
│  • 루트 노드 (Root Node): 전체 데이터 포함, 첫 분할 기준     │
│  • 내부 노드 (Internal Node): 특정 특징으로 분할 수행        │
│  • leaf 노드 (Leaf Node): 최종 예측 결과 (클래스 또는 값)    │
│  • 가지 (Branch): 분할 조건에 따른 부분 집합                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 결정 트리는 단계적으로 데이터를 분리하여 순수한(homogeneous) leaf 노드를 만든다.
- **원인**: 각 분할에서 불순도를 최대화 감소시키는 방향으로 선택하기 때문이다.
- **결과**: 새로운 데이터에 대해 root에서 leaf까지 조건을 확인하면 예측이 완료된다.
- **판단**: 해석 가능성이 중요한 문제에서 결정 트리가 적합하다.

📢 **섹션 요약 비유**: 결정 트리는 **의료 진단 flowchart**와 같습니다. 의사는 환자에게 일련의 질문(나이, 증상, 검사 결과 등)을 하고, 각 답변에 따라 다음 질문을 선택하며, 최종적으로 특정 진단에 도달한다. 이 과정은 결정 트리와 완전히 동일하며, 왜 그렇게 진단했는지 단계별로 추적할 수 있다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

결정 트리의 핵심은 **불순도 측정**과 **최적 분할 선택**이다. 불순도(Impurity)는 현재 노드의 데이터가 얼마나 혼합되어 있는지를 나타낸다. 대표적인 불순도 측정 방법으로 지니 불순도와 엔트로피가 있다.

**지니 불순도(Gini Impurity)**는 다음과 같이 계산된다. 완전히 순수한 노드(한 클래스만 존재)의 지니 불순도는 0이고, 완전히 섞인 노드에서는最大值에 가까워진다. **정보 이득(Information Gain)**은 분할 전 불순도와 분할 후 가중平均 불순도의 차이다. 정보 이득이 최대가 되는 분할을 선택한다.

**가지치기(Pruning)**는 과적합을 방지하기 위한 기법이다. 사전 가지치기는 최대 깊이나 최소 샘플 수를 제한하고, 사후 가지치기는 전체 트리를 만든 후 정보 이득이 작은 분할을 제거한다.

```text
[불순도 측정 및 분할 선택]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [지니 불순도 계산]                                            │
│  ────────────────────────────────────────────                 │
│  Gini(D) = 1 - Σ(pi)²                                        │
│                                                              │
│  예: 10개 샘플 [6개 클래스A, 4개 클래스B]                     │
│  Gini = 1 - ((6/10)² + (4/10)²)                              │
│       = 1 - (0.36 + 0.16)                                    │
│       = 0.48                                                 │
│                                                              │
│  [정보 이득 계산]                                              │
│  ────────────────────────────────────────────                 │
│  IG(D, f) = Gini(D) - Σ(|Dᵢ|/|D|) × Gini(Dᵢ)               │
│                                                              │
│  │Dᵢ│ = 분할 후 각 자식 노드의 샘플 수                          │
│  |D|  = 분할 전 부모 노드의 샘플 수                            │
│                                                              │
│  [특징 A로 분할 vs 특징 B로 분할]                              │
│  ────────────────────────────────────────────                 │
│  특징 A의 정보 이득: 0.25                                     │
│  특징 B의 정보 이득: 0.15                                     │
│  → 특징 A 선택 (더 높은 정보 이득)                            │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 정보 이득이 최대인 분할이 가장 순수한 leaf 노드를 만든다.
- **원인**: 분할 후 각 자식 노드의 불순도가 낮을수록 정보 이득이 커지기 때문이다.
- **결과**: 트리가 성장할수록 training 데이터에 과적합될 수 있다.
- **판단**: 일반화 성능을 위해 적절한 가지치기가 필요하다.

📢 **섹션 요약 비유**: 불순도 측정은 **색상 혼합도 측정**과 같습니다. 순수한 빨간색 물감만 있는 통은 불순도가 0이지만, 빨강, 파랑, 노랑이 섞인 통은 불순도가 높다. 결정 트리는 이 혼합도를 줄이는 방향으로 물감을 분리해 나간다. 가장 효과적으로 색상을 분리하는 기준(특징)을 선택하여 각 통을 순수한 색상으로 만들어 간다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

결정 트리의 실무 적용은 다양하다. **의학적 진단**에서 결정 트리는 증상과 검사 결과를 기반으로 질병을 진단하는 데 사용된다. 의사들은 트리의 각 결정 과정을 따라갈 수 있어 진단 근거를 명확히 설명할 수 있다. **금융 심사**에서 대출 신청자의 신분정보, 재정 상태 등을 기반으로 승인/거절을 결정한다.

**고객 세분화**에서 고객의 구매 패턴, 인구통계학적 특성을 기반으로 고객을 세분화하여 마케팅 전략을 수립한다. **스팸 필터링**에서 이메일의 특징(발신자, 단어 빈도, 링크 포함 여부 등)을 기반으로 스팸 여부를 분류한다.

```text
[결정 트리 분류 의사코드]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  function build_tree(data, features):                        │
│      // 종료 조건 확인                                        │
│      if all same class:                                     │
│          return Leaf(class)                                  │
│      if no features or max_depth reached:                   │
│          return Leaf(majority_class)                         │
│                                                              │
│      // 최적 분할 선택                                         │
│      best_gain = 0                                           │
│      best_feature = None                                     │
│      best_threshold = None                                  │
│                                                              │
│      for each feature in features:                           │
│          for each possible threshold:                        │
│              gain = calculate_info_gain(data, feature, th)   │
│              if gain > best_gain:                            │
│                  best_gain = gain                           │
│                  best_feature = feature                      │
│                  best_threshold = threshold                  │
│                                                              │
│      // 분할 수행 및 재귀적 트리 구축                           │
│      left = data[feature <= threshold]                        │
│      right = data[feature > threshold]                       │
│                                                              │
│      return Node(feature, threshold,                          │
│                   build_tree(left, features'),               │
│                   build_tree(right, features'))              │
│                                                              │
│  function predict(tree, sample):                             │
│      if is_leaf(tree):                                       │
│          return tree.class                                   │
│      if sample[tree.feature] <= tree.threshold:              │
│          return predict(tree.left, sample)                   │
│      else:                                                   │
│          return predict(tree.right, sample)                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 결정 트리 구현은 **퀴즈节目 전략**과 같습니다. 제한된 질문 수로 참가자를 올바르게 분류하려면, 각 질문이 가장 효과적으로 그룹을 나누는 질문을 선택해야 한다. "타고난 talent 있나요?"보다는 "연기를 해본 적이 있나요?"가 더 효과적으로 배우와 가수를 구분할 수 있듯이, 결정 트리도 정보 이득이 최대인 특징을 선택하여 효율적으로 분류한다.

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

결정 트리의 품질 관리에서 가장 중요한 것은 **과적합 방지**이다. 트리가 training 데이터에 너무 맞춰지면 새로운 데이터에 대한 성능이 떨어진다. **교차 검증**을 통해 일반화 성능을 평가하고, **정확도, 정밀도, 재현율, F1 점수** 등의 지표를 확인해야 한다.

**품질 관리 체크리스트**: 최대 깊이를 제한하여 트리의 복잡성을 제어해야 한다. 최소 leaf 노드 샘플 수를 설정하여 과적합을 방지해야 한다.葉 노드의 순수도가 100%인 경우가 많으면 과적합을 의심해야 한다.

📢 **섹션 요약 비유**: 결정 트리의 품질 관리는 **시험 문제 출제**와 같습니다. 너무 구체적인 문제(과적합)는 출제자가 아니면 풀 수 없고, 너무 일반적인 문제는 누구나 풀 수 있어 구별이 안 된다. 적절한 수준의 난이도(일반화 성능)가 유지되도록 출제 기준(파라미터)을 조정해야 한다.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

결정 트리의 최신 동향은 **앙상블 방법론**의 발전과 함께 이루어지고 있다. **랜덤 포레스트**는 다수의 결정 트리를 병렬로 학습하여 예측 안정성을 높이고, **그래디언트 부스팅**은 순차적으로 트리를 학습하여 잔차를 보정한다. XGBoost, LightGBM, CatBoost 등의 라이브러리가 실제로 가장 널리 사용되는 머신러닝 도구 중 하나이다.

결정 트리는 "머신러닝의 시작점"으로서, 단순하지만 강력한 앙상블 기법의 기반이 된다.

📢 **섹션 요약 비유**: 결정 트리의 발전은 **개별 전문가에서 위원회로**와 같습니다. 한 명의 전문가(단일 결정 트리)의 판단에는 편향과 오류가 있을 수 있지만, 여러 전문가(랜덤 포레스트)가 각자 독립적으로 판단을 내리고 최종 투표를 하면 더 정확한 결과를 얻을 수 있다.
