# 56. 다익스트라 알고리즘 (Dijkstra's Algorithm)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다익스트라(Dijkstra) 알고리즘은 주어진 시작 노드에서 다른 모든 노드까지의 최단경로를 찾는 그리디(greedy) 알고리즘이다.
> 2. **가치**: 각 단계에서 가장 짧은 거리의 노드를 선택하여 전역적 최단을 보장하며, 희소 그래프에서 효율적이다.
> 3. **융합**: GPS 네비게이션, 네트워크 라우팅, 게임 경로finding 등 최단경로 문제가 필요한 모든領域에서 활용된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

다익스트라 알고리즘은 1956년 네덜란드의 컴퓨터 과학자 에츠허르 다익스트라(Edsger Dijkstra)가 발표했으며, 그래프에서 하나의 노드에서 다른 모든 노드까지의 최단경로를 찾는 알고리즘이다. 이 알고리즘은 **그리디 알고리즘**의一種로, 각 단계에서 현재 가장 짧은 거리의 노드를 선택하여 전역적 최단을 보장한다.

다익스트라 알고리즘이 중요한 이유는 **실용성**과 **효율성** 때문이다. 도로 네트워크, 통신 네트워크, 게임 맵 등 실생활에서 발생하는 수많은 최단경로 문제를 해결하는 데 사용된다. 시간 복잡도가 O((V + E) log V)로 효율적이며, 특히 간선이 노드보다 훨씬 적은 희소 그래프(sparse graph)에서 뛰어난 성능을 보인다.

다익스트라 알고리즘은 **비마이너스 가중 그래프**에서만 동작한다. 모든 간선의 가중치가 양수(>= 0)이어야 한다. 음수 가중치가 존재하면 벨만-포드 알고리즘을 사용해야 한다. 이 제약 조건은 알고리즘의 핵심 가정인 "더 짧은 경로는 항상 더 좋은 경로다"를 보장하기 위한 것이다.

```text
[다익스트라 알고리즘 동작 원리]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [그래프 예시]                                                │
│  ────────────────────────────────────────────                │
│                                                              │
│         2          1                                         │
│    A ------ B ------ C                                        │
│    |        |        |                                        │
│  4 |      3 |      6 |                                        │
│    |        |        |                                        │
│    D ------ E ------ F                                        │
│         5          2                                         │
│                                                              │
│  [다익스트라 알고리즘 진행]                                     │
│  ────────────────────────────────────────────                │
│                                                              │
│  시작: A                                                     │
│  초기 거리: A=0, 나머지=∞                                     │
│                                                              │
│  Step 1: A 선택 (거리 0)                                      │
│    → B=2, D=4 업데이트                                        │
│                                                              │
│  Step 2: B 선택 (거리 2)                                      │
│    → C=3, E=5 업데이트 (기존 ∞보다 작음)                      │
│                                                              │
│  Step 3: C 선택 (거리 3)                                      │
│    → F=9 업데이트 (기존 7보다 큼, 유지)                       │
│                                                              │
│  Step 4: E 선택 (거리 5)                                      │
│    → F=7 업데이트 (이 경로가 더 짧음)                         │
│                                                              │
│  Step 5: D 선택 (거리 4)                                      │
│    → 변화 없음                                               │
│                                                              │
│  최종 거리: A=0, B=2, C=3, D=4, E=5, F=7                      │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 다익스트라는 각 단계에서 아직 선택되지 않은 노드 중 가장 짧은 거리의 노드를 선택한다.
- **원인**: 더 짧은 거리의 노드를 선택해도 이미 발견된 경로보다 짧아질 수 없기 때문이다.
- **결과**: 모든 노드가 선택되면 각 노드에 대한 최단거리가 확정된다.
- **판단**: 음수 가중치가 없는 그래프에서 최단경로 문제가 있으면 다익스트라가 적합하다.

📢 **섹션 요약 비유**: 다익스트라 알고리즘은 **지도를 보며 가장 짧은 경로 찾기**와 같습니다. 현재 위치에서 가장 가까운 목적지를 먼저 확인하고(greedy 선택), 그 목적지에 도달하면 거기서부터 새로운 목적지까지의 거리를 更新한다. 가장 가까운 것을 먼저 보는 것이 결국 전체 최적경로로 이어진다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

다익스트라 알고리즘의 핵심은 **최소 힙(또는 우선순위 큐)**을 사용한 효율적인 노드 선택이다. 매 단계에서 아직 방문하지 않은 노드 중 거리가 가장 짧은 노드를 선택해야 하는데, 이를 선형 탐색하면 O(V) 시간이 소요된다. 최소 힙을 사용하면 O(log V)로 가장 짧은 노드를 선택할 수 있다.

알고리즘의 동작 과정은 다음과 같다. 첫째, 시작 노드의 거리를 0으로, 나머지 노드의 거리를 무한대로 초기화한다.둘째, 방문하지 않은 노드 중 거리가 가장 짧은 노드를 선택한다(우선순위 큐에서 추출).셋째, 선택한 노드를 경유지로 하여 인접한 노드들의 거리를 갱신한다(경유지가 더 짧으면).넷째, 모든 노드가 방문될 때까지 2-3 과정을 반복한다.

**시간 복잡도**: O((V + E) log V) - 각 노드를 한 번씩 선택하고(O(V log V)), 각 간선을 한 번씩 확인한다(O(E log V)).

```text
[다익스트라 알고리즘 상세 의사코드]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  function Dijkstra(graph, start):                            │
│      // 거리 배열 초기화                                      │
│      distance = {}                                          │
│      for each node in graph:                                │
│          distance[node] = infinity                           │
│      distance[start] = 0                                     │
│                                                              │
│      // 우선순위 큐: (거리, 노드)                              │
│      pq = PriorityQueue()                                    │
│      pq.add((0, start))                                     │
│                                                              │
│      // 방문 집합                                             │
│      visited = set()                                         │
│                                                              │
│      while pq is not empty:                                 │
│          current_dist, current_node = pq.pop()              │
│                                                              │
│          // 이미 방문한 노드면 무시                           │
│          if current_node in visited:                        │
│              continue                                       │
│          visited.add(current_node)                           │
│                                                              │
│          // 인접 노드 거리 갱신                                │
│          for neighbor, weight of current_node:              │
│              new_dist = current_dist + weight                │
│              if new_dist < distance[neighbor]:              │
│                  distance[neighbor] = new_dist              │
│                  pq.add((new_dist, neighbor))                │
│                                                              │
│      return distance                                        │
│                                                              │
│  [우선순위 큐 동작]                                            │
│  ────────────────────────────────────────────                │
│  초기: [(0, A)]                                              │
│  A 추출: [(2, B), (4, D)]                                     │
│  B 추출: [(3, C), (4, D), (5, E)]                             │
│  C 추출: [(4, D), (5, E), (9, F)]                             │
│  ...                                                         │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 우선순위 큐에서 노드가弹出될 때의 거리가 최종 최단거리이다.
- **원인**: 모든 경유지가 더 긴 거리를 가질 수밖에 없기 때문이다.
- **결과**: BFS와 달리 가중 그래프에서도 정확한 최단경로를 찾을 수 있다.
- **판단**: 힙 자료구조의 활용이 다익스트라의 효율성을 결정한다.

📢 **섹션 요약 비유**: 다익스트라의 우선순위 큐 활용은 **항공사贵宾室 서비스**와 같습니다. 모든乗客을 받는 것이 아니라, 누적 마일리지(거리)가 높은乗客(짧은 거리 노드)부터 우선 서비스한다. 그러면 각 단계에서 가장 중요한 고객이 먼저 처리되어 효율적인 서비스가 가능하다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

다익스트라 알고리즘의 실무 적용은 광범위하다. **GPS 네비게이션**: 출발지에서 목적지까지의 최단경로를 계산한다. 도로 구간의 거리를 가중치로 사용하여 실시간으로 경로를 탐색한다. **네트워크 라우팅**: OSPF, IS-IS 등 인터넷 라우팅 프로토콜에서 사용된다. 라우터 간 링크 비용을 기반으로 최적 경로를 결정한다. **게임 개발**: 유닛의 이동 경로planning, 장애물 회피 등에 활용된다.

**응용: 경로 재구성**: 최단거리 배열만으로는 경로가 무엇인지 알 수 없다. 각 노드의 "이전 노드(previous node)"를 저장하면 경로를 재구성할 수 있다. 노드 v의 이전 노드를 u라 하면, start에서 v까지의 최단경로는 start에서 u까지의 최단경로 + 간선(u, v)이다.

```text
[다익스트라 경로 재구성]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [이전 노드 배열 사용]                                        │
│  ────────────────────────────────────────────                │
│                                                              │
│  previous = {}  // 각 노드의 이전 노드 저장                    │
│  // A에서 다른 노드까지 최단경로 중 마지막 이전 노드           │
│                                                              │
│  알고리즘 수행 중:                                            │
│  A → B (거리 2), previous[B] = A                             │
│  A → C (거리 3 via B), previous[C] = B                      │
│  A → D (거리 4 via A), previous[D] = A                      │
│  A → E (거리 5 via B), previous[E] = B                      │
│  A → F (거리 7 via E), previous[F] = E                      │
│                                                              │
│  [경로 재구성: F까지의 최단경로]                               │
│  ────────────────────────────────────────────                │
│  F ← previous[F] = E ← previous[E] = B ← previous[B] = A    │
│  역순: A → B → E → F                                         │
│                                                              │
│  [응용: K-번째 최단경로]                                      │
│  ────────────────────────────────────────────                │
│  일반 다익스트라는 1번째 최단경로만 구한다.                    │
│  K-번째 최단경로를 구하려면:                                   │
│  - 노드를 여러 번 방문 허용 (k번까지)                          │
│  - 상태 확장: (노드, 방문 횟수)                               │
│  - A* 알고리즘과 결합하여 효율적으로 탐색                      │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 다익스트라의 실무 활용은 **배달 경로 최적화**와 같습니다. 배달 기사님은 여러 물류 거점을 지나면서 각 거점间的 거리와 교통 상황을 고려하여 전체 배달 시간이最短이 되는 경로를 선택한다. 다익스트라 알고리즘은 이러한 판단을 자동화하여 컴퓨터로 수행하는 것이다.

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

다익스트라 알고리즘의 품질 관리에서 가장 중요한 것은 **음수 가중치 처리**와 **오버플로우 방지**이다.

**품질 관리 체크리스트**: 모든 간선 가중치가 비마이너스인지 확인해야 한다. 힙 연산에서 거리가 갱신될 때优先순위가正しく更新되었는지 확인해야 한다. 큰 가중치의 합이 정수 범위를 넘지 않도록 64비트 정수를 사용해야 한다.

📢 **섹션 요약 비유**: 다익스트라의品質 管理는 **항공사 운항 계획**과 같습니다. 연료가 부족한 비행기(음수 간선)는 계획에 포함할 수 없으며, 각 구간의 연료 소비量(거리)이 정확하게 계산되어야 전체 여정 계획이有効하다.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

다익스트라 알고리즘의 최신 동향은 **대규모 데이터 처리**와 **실시간 응용**과 관련되어 있다. **다익스트라 + A* 조합**: 목표 지점을 알고 있으면 A* 알고리즘으로 더 빠르게 탐색할 수 있다. **다이나믹 라우팅**: 실시간 교통 정보를 반영하여 경로를 동적으로更新한다.

다익스트라 알고리즘은 "그래프 최단경로 문제의 기본 알고리즘"으로서, 이후 수많은 알고리즘의 기반이 되었다.

📢 **섹션 요약 비유**: 다익스트라는 **우체부 경로 최적화**와 같습니다. 모든 집을 최소 거리로delivery하려면, 이미 확인한 범위 내에서 가장 가까운 미방문 집을 찾아拜访하고, 그 집을 중심으로 범위를 확장하는 과정을 반복한다. 결국 모든 집을 가장 짧은総距離로 방문할 수 있다.
