## 핵심 인사이트 (3줄 요약)
- 위상 정렬은 방향 비순환 그래프(DAG)에서 모든 간선의 방향을 거스르지 않도록 정점들을 나열하는 알고리즘입니다.
- 주로 작업의 선후 관계, 컴파일 의존성, 대학 선수 과목 등 순서가 있는 작업을 스케줄링할 때 사용됩니다.
- 진입 차수(In-degree)를 이용한 Kahn's 알고리즘과 깊이 우선 탐색(DFS) 기반 알고리즘 두 가지 방식으로 구현 가능하며 시간 복잡도는 O(V+E)입니다.

### Ⅰ. 개요 (Context & Background)
위상 정렬(Topological Sort)은 방향 비순환 그래프(DAG, Directed Acyclic Graph)의 각 정점을 선형으로 정렬하는 기법으로, 그래프 내의 모든 간선 (u, v)에 대해 정점 u가 정점 v보다 먼저 오도록 배열하는 과정입니다. 사이클이 존재하는 그래프에서는 불가능하며, 작업 간의 의존성(Dependency)을 해결하기 위한 필수적인 알고리즘으로 활용됩니다. 정보통신기술사 및 시스템 아키텍트 관점에서는 태스크 스케줄링 및 빌드 시스템 최적화의 핵심 개념으로 다루어집니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)
위상 정렬은 진입 차수(In-degree)를 관리하는 Kahn의 알고리즘과 DFS 역순 추적 기반 알고리즘으로 나뉩니다.
Kahn 알고리즘의 원리는 진입 차수가 0인 노드를 큐(Queue)에 삽입하고, 큐에서 하나씩 빼며 인접한 노드의 진입 차수를 감소시키는 과정을 반복하는 것입니다.

```text
[ Kahn's Algorithm Flow / Kahn 알고리즘 흐름 ]

+-----------+      (In-degree=0)      +-----------+
|  Node A   | ----------------------> |  Queue Q  | 
+-----------+                         +-----------+
      |                                     |
      | Remove Edges (간선 제거)              | Pop (추출)
      v                                     v
+-----------+      (Update In-degree) +-----------+
|  Node B   | <---------------------- | Sorted    |
| (In-deg--)|                         | List      |
+-----------+                         +-----------+
```

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

| 비교 항목 | Kahn's 알고리즘 (Kahn's Algorithm) | DFS 기반 알고리즘 (DFS-based) |
| :--- | :--- | :--- |
| **기반 구조** | 큐 (Queue), 진입 차수(In-degree) 배열 | 스택 (Stack), 방문(Visited) 배열 |
| **작동 방식** | 진입 차수가 0인 노드부터 순차 처리 | DFS 탐색 후 반환 시 스택에 삽입 (역순) |
| **시간 복잡도** | O(V + E) | O(V + E) |
| **사이클 판별** | 정렬된 노드 수가 전체 V보다 작으면 사이클 | 역방향 간선(Back Edge) 탐지로 사이클 확인 |
| **장점** | 순차적 의존성 및 다중 시작점 처리가 직관적 | 기존 DFS 코드 재사용이 용이 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)
실무에서는 CI/CD 파이프라인의 빌드 순서 결정, 패키지 매니저(npm, Maven)의 의존성 해결, 데이터베이스의 외래키 제약조건 분석 등에 위상 정렬이 사용됩니다. 기술사적 관점에서, 병렬 처리가 가능한 태스크를 분별하기 위해서는 진입 차수가 0인 작업들을 동시에 스케줄링하는 기법(Level-based Topological Sort)을 적용하여 분산 시스템의 처리 효율을 극대화해야 합니다.

### Ⅴ. 기대효과 및 결론 (Future & Standard)
위상 정렬은 단순한 알고리즘을 넘어 분산 컴퓨팅 환경의 워크플로우 엔진(Apache Airflow 등)에서 DAG 기반 스케줄링의 표준 원리로 자리잡았습니다. 향후 복잡도가 증가하는 클라우드 네이티브 환경에서도 서비스 의존성 트리를 빠르고 안전하게 해석하여 장애 전파를 막고 자원 활용을 최적화하는 데 핵심 역할을 수행할 것입니다.
