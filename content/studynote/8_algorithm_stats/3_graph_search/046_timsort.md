# 046. 팀 정렬 (Timsort)

## 핵심 인사이트 (3줄 요약)
> 1. 시간 복잡도가 무조건 $O(N \log N)$ 이며, 심지어 입력된 데이터가 운 좋게 거의 다 정렬되어 있을 때는 단 $O(N)$ 선형 시간만 터치하고 끝나버리는 완전 무결한 실무(Real-world) 최적화 알고리즘이다.
> 2. 병합 정렬(Merge Sort)과 삽입 정렬(Insertion Sort)의 기막힌 융합으로 개발. 배열을 런(Run) 이라는 연속된 오름차순/내림차순 조각 덩어리로 스캔하여 미리 묶어버림으로써 합병 트리 연산을 무자비하게 단축한다.
> 3. 안정성(Stable)을 절대 보장하므로 파이썬의 핵심 `sort()`, 자바 7 이상의 `Arrays.sort()`, 안드로이드, V8(자바스크립트) 엔진 등 지구상 모든 메이저 산업용 언어의 기본 내장 정렬 표준기(Standard Default)로 장악 등극했다.

---
## Ⅰ. 아키텍처 및 원리
```text
  [ Timsort 의 Run 기반 지능형 타겟 스캔 ]
    배열: [ 1, 3, 5,  2, 4, 6,  7, 9, 8 ]
    1. Run 식별 (오름/내림차순 덩어리 절삭):  (1,3,5), (2,4,6), (7,9,8 역방향뒤집기)
    2. 너무 작은 Run 은 그 자리에서 바로 삽입 정렬로 강제 묶음 
    3. 메모리 스택 구조로 비슷한 크기의 Run 들끼리만 최적 트리 머지(Merge)를 시켜 합침! 
```
---
## Ⅱ. 실무 적용 및 결론
개발자가 파이썬이나 자바에서 `리스트.sort()` 를 쳤을 때 벌어지는 마법의 정체다. 실험실 속 난수로 뒤섞인 데이터에선 퀵 정렬이 제일 빠를 수 있으나, 인간의 데이터(게시판 리스트, 로깅, 타임스탬프)는 언제나 '일부분씩 이미 예쁘게 정렬'되어 있는 특징을 공학적 직관으로 캐치 해낸 극단적 실용주의 컴퓨팅 대가(Tim Peters)의 위대한 발명품이다.
