# 052. 이분 탐색 (Binary Search)

## 핵심 인사이트 (3줄 요약)
> 1. 사전 지식으로 **"데이터의 바닥이 완벽하게 오름차순(또는 내림차순)으로 깔려있다"**는 룰을 이용해서 중간 위치(Mid)를 찔러 탐색할 범위를 무조건 50%씩 단두대로 날려버리는 분할 정복 탐색(Divide and Conquer)이다.
> 2. 데이터가 40억 개(전 세계 인구급)라도 최악의 경우 단 32번의 `if` 검사 찌르기 만으로 값을 찾아내는 압도적인 수학 지수로 타임 복잡도 $O(\log N)$ 의 우월성을 증명한다.
> 3. 만약 배열 데이터가 자주 삭제되거나 중간에 새로운 값이 삽입(Insert) 빈도가 높다면, 이 배열을 계속 '정렬된 채 유지재건($O(N)$)' 하는 비용이 탐색해서 얻은 이득속도를 작살내버리므로 구조적 결함에 처한다.

---
## Ⅰ. 아키텍처 및 원리
```text
  [ 배열 포인터 트리 수축망 파이프라인 ]
   타겟 = 7  / 데이터 = [ 1, 3, 5, 7, 9, 11, 13]
   1단계: Left=1(L), Right=13(R), Mid=7번 
      -> (찾았네? 1방 컷! => 만약 Mid 5을 찔렀으면, "어 타겟(7) 더 커!" 라면 Left 포인터를 Mid+1 오른쪽으로 버림 이동)
```
---
## Ⅱ. 실무 적용 및 결론
데이터베이스 인덱스의 최하단 B-Tree 잎(Leaf Node) 검색 과정의 가장 말단 모듈에서 구동되는 핵심 서치 엔진이다. 또한 알고리즘 코딩 구현에서 파라메트릭 서치(Parametric Search, 최적화 문제를 "이 크기가 가능할까?"라는 예/아니오 결정 문제로 바꾸어 이분 탐색하는 고급 기법) 논리로 활용되어, 제한 시간이 빡빡한 기계적 최댓값/최솟값 추론 파이프에서 신과 같이 군림한다.
