# 30. 기수 정렬 (Radix Sort)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 기수 정렬(Radix Sort)은 각 자리수를 기준으로 반복적으로 정렬하는 비비교 정렬(Non-comparison Sort) 알고리즘이다.
> 2. **가치**: 원소 간 직접 비교 없이 자릿수를 하나씩 처리하여 O(d(N + k)) 시간에 정렬할 수 있다.
> 3. **융합**: LSD(최하위 자릿수 먼저) 방식과 MSD(최상위 자릿수 먼저) 방식이 있으며, 각 자릿수 정렬에는 일반적으로 카운팅 정렬이 활용된다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

기수 정렬(Radix Sort)은 1887년 Herman Hollerith가 천공卡片 정렬에 사용한 알고리즘에서 유래했다. 기수(Radix)는 "밑(Base)"을 의미하며, 10진법에서는 10, 2진법에서는 2를 기수로 사용한다. 기수 정렬의 핵심 아이디어는 **원소를 한 번에 비교하지 않고, 자릿수별로 순차적으로 정렬**하는 것이다.

기수 정렬이 중요한 이유는 비교 기반 정렬의 한계를 넘을 수 있기 때문이다. 원소의 값이 d개의 자릿수로 표현될 수 있고, 각 자릿수가 k개의 값을 가질 수 있다면(일반적으로 k=10), 전체 시간 복잡도는 O(d(N + k))이다. d가 상수라면 이것은 O(N)이다.

```text
[기수 정렬 전체 과정]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [입력 배열]                                                  │
│  ────────────────────────────────────────────                │
│  입력: [170, 45, 75, 90, 802, 24, 2, 66]                     │
│                                                              │
│  [LSD (Least Significant Digit) 기수 정렬]                    │
│  ────────────────────────────────────────────                 │
│                                                              │
│  Round 1: 1의 자리 (k=10)                                    │
│  원래:    [170, 45, 75, 90, 802, 24, 2, 66]                  │
│  1의 자리: [0,   5,   5,   0,   2,   4,   2,   6]          │
│  정렬 후: [170, 90, 802, 2, 24, 45, 75, 66]                  │
│           (0)  (0)  (2)  (2) (4)  (5)  (5)  (6)             │
│                                                              │
│  Round 2: 10의 자리 (k=10)                                   │
│  현재:    [170, 90, 802, 2, 24, 45, 75, 66]                  │
│  10의 자리: [7,   9,   0,   0,   2,   4,   7,   6]         │
│  정렬 후: [802, 2, 24, 45, 66, 170, 75, 90]                  │
│           (0)  (0)  (2)  (4)  (6)  (7)  (7)  (9)           │
│                                                              │
│  Round 3: 100의 자리 (k=10)                                  │
│  현재:    [802, 2, 24, 45, 66, 170, 75, 90]                  │
│  100의 자리:[8,   0,   0,   0,   0,   1,   0,   0]         │
│  정렬 후: [2, 24, 45, 66, 75, 90, 170, 802]                  │
│           (0)  (0)  (0)  (0)  (0)  (1)  (0)  (8)            │
│                                                              │
│  [최종 출력]                                                  │
│  ────────────────────────────────────────────                │
│  출력: [2, 24, 45, 66, 75, 90, 170, 802]                     │
│                                                              │
│  [시간 복잡도]                                                │
│  ────────────────────────────────────────────                 │
│  d = 자릿수 (3 for above example)                            │
│  k = 기수 (10 for decimal)                                   │
│  시간: O(d(N + k)) = O(3(8 + 10)) = O(54) ≈ O(N)            │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 기수 정렬은 원소를 자릿수별로 순차적으로 정렬한다.
- **원인**: 높은 자릿수가 낮은 자릿수보다 더 중요한 영향을 미치므로, 낮은 자릿수부터 정렬하면 전체 순서가 맞춰진다.
- **결과**: 각 라운드에서 안정적인 하位 정렬을 보장하면, 모든 자릿수 처리 후 전체 정렬이 완료된다.
- **판단**: 정수나 문자열처럼 자릿수가 제한적인 데이터에서 효율적이다.

📢 **섹션 요약 비유**: 기수 정렬은 **우편 번호 정렬**과 같습니다. 먼저 1의 자리(끝 자리)를 보고分组하고, 그 다음 10의 자리를 보고 그룹을 나누며, 마지막으로 100의 자리를 본다. 각 단계에서 같은 그룹에 있으면 순서가 유지된다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

기수 정렬에는 두 가지 주요 방식이 있다. **LSD(Least Significant Digit, 최하위 자릿수 먼저)** 방식과 **MSD(Most Significant Digit, 최상위 자릿수 먼저)** 방식이다.

**LSD 기수 정렬**: 가장 낮은 자릿수(1의 자리)부터 시작하여 높은 자릿수로 진행한다. 모든 라운드가 완료되면 정렬이 완료된다. 1의 자리부터 정렬해도 문제가 없는 이유는 각 라운드에서 **안정적(Stable)** 정렬을 사용하기 때문이다.

**MSD 기수 정렬**: 가장 높은 자릿수부터 시작하여 낮은 자릿수로 진행한다. 이 방식은 문자열 정렬에서 특히 유용하며, 접두사 기반으로 분할하여 처리한다.

**각 라운드에서의 정렬**: 일반적으로 각 자릿수를 정렬할 때 **카운팅 정렬**이 사용된다. 자릿수의 범위(기수 k)가 제한적이기 때문이다.

```text
[기수 정렬 상세 알고리즘]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [LSD 기수 정렬 의사코드]                                      │
│  ────────────────────────────────────────────                │
│                                                              │
│  function radix_sort(A, d):  // A: 입력, d: 자릿수           │
│      for i = 1 to d:                                        │
│          // 자릿수 i에 대해 안정적 정렬 수행                  │
│          use stable_sort(A, i)  // 보통 카운팅 정렬 사용      │
│                                                              │
│  function stable_sort(A, digit):                            │
│      // digit 위치의 값으로 정렬 (0=1의 자리, 1=10의 자리...) │
│      k = 10  // 기수                                          │
│      n = length(A)                                          │
│                                                              │
│      // 카운팅 정렬 수행                                      │
│      count = array of size k                                │
│      for j = 0 to k-1:                                      │
│          count[j] = 0                                        │
│                                                              │
│      for j = 0 to n-1:                                      │
│          index = get_digit(A[j], digit)                     │
│          count[index] = count[index] + 1                    │
│                                                              │
│      for j = 1 to k-1:                                      │
│          count[j] = count[j] + count[j-1]                   │
│                                                              │
│      result = array of size n                                │
│      for j = n-1 downto 0:                                  │
│          index = get_digit(A[j], digit)                     │
│          result[count[index] - 1] = A[j]                     │
│          count[index] = count[index] - 1                    │
│                                                              │
│      return result                                           │
│                                                              │
│  [시간 복잡도 분석]                                            │
│  ────────────────────────────────────────────                 │
│  d = 자릿수                                                   │
│  k = 기수 (보통 10)                                           │
│  N = 원소 수                                                  │
│                                                              │
│  각 라운드: O(N + k) 카운팅 정렬                               │
│  총 시간: O(d(N + k))                                        │
│                                                              │
│  [공간 복잡도]                                                │
│  ────────────────────────────────────────────                 │
│  출력 배열: O(N)                                              │
│  카운트 배열: O(k)                                            │
│  총 공간: O(N + k)                                            │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **관찰**: 기수 정렬의 각 라운드는 O(N + k)이며, 총 O(d(N + k))이다.
- **원인**: 각 자릿수는 k개의 가능한 값을 가지므로, 카운팅 정렬의 시간 복잡도가 적용된다.
- **결과**: d가 상수라면 전체 시간 복잡도는 O(N)이다.
- **판단**: d가 작은 경우(예: 32비트 정수), 기수 정렬은 매우 효율적이다.

📢 **섹션 요약 비유**: 기수 정렬은 **도서관 Books를年代순으로整理**하는 것과 같습니다. 먼저 출판 연도 끝자리(1의 자리)로 분류하고, 그 다음 10의 자리로 분류하며, 마지막으로 100의 자리로 분류한다. 각 단계에서 같은 값이면 이전 순서가 유지된다.

## Ⅲ. 구현 및 실무 응용 (Implementation & Practice)

기수 정렬의 실무 적용은 **긴 데이터 형식의 정렬**, **문자열 정렬**, **날짜 정렬** 등에서 효과적이다.

**LSD vs MSD 선택**: 데이터의 길이가 균일하면 LSD가, 데이터의 길이가 가변적이면 MSD가 더 적합하다. MSD는 접두사 기반으로 빠르게 분류할 수 있기 때문이다.

```text
[기수 정렬 구현]

┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  [Python 구현 - LSD 기수 정렬]                                │
│  ────────────────────────────────────────────                │
│                                                              │
│  def get_digit(num, d):                                     │
│      return (num // (10 ** d)) % 10                          │
│                                                              │
│  def counting_sort_by_digit(arr, d):                        │
│      n = len(arr)                                            │
│      k = 10  // 기수                                          │
│      count = [0] * k                                         │
│      result = [0] * n                                        │
│                                                              │
│      # 자릿수 d의 값 카운트                                    │
│      for val in arr:                                        │
│          digit = get_digit(val, d)                           │
│          count[digit] += 1                                   │
│                                                              │
│      # 누적 합                                                │
│      for i in range(1, k):                                   │
│          count[i] += count[i-1]                              │
│                                                              │
│      # 역순 순회 (안정성)                                      │
│      for val in reversed(arr):                               │
│          digit = get_digit(val, d)                          │
│          result[count[digit] - 1] = val                     │
│          count[digit] -= 1                                   │
│                                                              │
│      return result                                           │
│                                                              │
│  def radix_sort(arr):                                       │
│      max_val = max(arr)                                      │
│      num_digits = len(str(max_val))                         │
│                                                              │
│      for d in range(num_digits):                             │
│          arr = counting_sort_by_digit(arr, d)                │
│                                                              │
│      return arr                                              │
│                                                              │
│  [실무 응용]                                                 │
│  ────────────────────────────────────────────                 │
│   1. 정수 정렬: 32비트, 64비트 정수                           │
│   2. 문자열 정렬: 사전순 정렬 (ASCII 코드 활용)               │
│   3. 날짜 정렬: 연, 월, 일 순서로 처리                        │
│   4. IP 주소 정렬: 8비트 옥텟 단위 처리                       │
│   5. 큰 데이터베이스: 외부 기수 정렬 (외부 정렬과 결합)        │
│                                                              │
│  [주의사항]                                                   │
│  ────────────────────────────────────────────                 │
│   1. 음수 처리가 어려움 (offset 변환 필요)                    │
│   2. 실수는 직접 처리 불가 (지수 부분 분리 필요)              │
│   3. d가 크면 다른 알고리즘이 더 효율적일 수 있음              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 기수 정렬은 **전화번호부 정리**와 같습니다. 먼저 마지막 4자리(1의 자리부터)를 보고分类하고, 이어서 앞쪽 번호를 본다. 우편 번호로 분류하고, 그 안에서 시/군/구를 보고, 다시 이름으로分類하는 것과 유사하다.

**품질 관리 체크리스트**: 각 라운드에서 안정적 정렬이 수행되는지 확인한다.자릿수 추출 함수가 정확한지 확인한다.최대값의 자릿수보다 작은 경우도 처리해야 한다.음수나 실수가 포함된 경우의 처리 로직을 확인한다.

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

기수 정렬의 최신 동향으로는 **GPU 가속 기수 정렬**, **대규모 데이터 처리를 위한 외부 기수 정렬**, **병렬 구현** 등이 연구되고 있다.

**기수 정렬의Positions**: 기수 정렬은 비비교 정렬이라는特的 범주에 속하며, 원소 범위나 자릿수에 따라 성능이 달라진다. d가 작고 k가 크지 않은 경우 매우 효율적이며, 문자열 정렬이나固定 형식 데이터에서 널리 사용된다.

**기수 정렬 vs 비교 기반 정렬**: 비교 기반 정렬은 Θ(N log N)이 최선이지만, 기수 정렬은 O(N)까지 가능하다. 그러나 원소의 특성(자릿수, 기수)에 따라 실제 성능이 달라지므로, 데이터 특성을 고려하여 선택해야 한다.

📢 **섹션 요약 비유**: 기수 정렬は **指紋認識システム**と似ています。各指紋의 고유한 특징을 하나씩 추출하여(자릿수별로)分類하고, 이를 결합하여 전체 분류를完成한다.
