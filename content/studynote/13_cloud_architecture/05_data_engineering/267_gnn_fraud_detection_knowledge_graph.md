## 핵심 인사이트 (3줄 요약)
> 1. **본질**: GNN(Graph Neural Network) 기반 FDS(Fraud Detection System)는 개별 거래의 특성이 아닌 사용자-가맹점-계좌 간의 관계망(그래프) 패턴을 학습하여, 기존 규칙 기반·독립 트랜잭션 ML이 놓치는 공모 사기·신규 패턴을 탐지한다.
> 2. **가치**: 사기범은 개별 거래를 정상처럼 위장하는 데 능숙하지만, 여러 계좌에 걸친 자금 흐름 그래프 패턴(링 구조, 레이어링 등)까지 위장하는 것은 극히 어렵다 — GNN은 이 그래프 구조적 특징을 학습한다.
> 3. **판단 포인트**: 실시간 FDS에서 GNN 추론 지연(수 ms~수십 ms)이 결제 승인 SLA를 충족해야 하며, 그래프 업데이트(신규 거래 반영)와 모델 재훈련 주기를 어떻게 관리할지가 핵심 설계 과제다.

---

## Ⅰ. 개요 및 필요성

전통적인 FDS는 "새벽 3시 해외 거래", "5분 내 연속 소액 결제" 같은 규칙 기반 또는 개별 트랜잭션의 특성(금액, 위치, 시간)을 분류하는 ML 모델이었다. 이 접근법의 한계는 **관계적 패턴**을 보지 못한다는 것이다.

실제 금융 사기는 고립된 사건이 아니다. 보이스피싱은 수십 개의 대포통장 계좌망이 연루되고, 보험 사기는 특정 병원-환자 그룹이 공모한다. 이런 **네트워크 구조** 패턴을 잡는 것이 GNN 기반 FDS의 본질이다.

```
[사기 거래 그래프 패턴 예시]

정상 거래 패턴:               사기 거래 패턴 (링 구조):
A ──▶ 가맹점                  A ──▶ B ──▶ C
B ──▶ 가맹점                   └──▶ D ──▶ C
C ──▶ 가맹점                 (A, B, D가 C에 자금 집중 후 인출)

각 거래는 개별적으로 정상    그래프 구조 자체가 이상 패턴
→ 규칙 기반/단순 ML 통과    → GNN이 관계망 패턴으로 탐지
```

📢 **섹션 요약 비유**: 규칙 기반 FDS는 "범인의 얼굴을 보고 잡는 것"이고, GNN FDS는 "범인 일당의 인맥 지도를 보고 잡는 것"이다. 개인 위장은 가능해도 조직 전체의 연결 패턴을 숨기기는 어렵다.

## Ⅱ. 아키텍처 및 핵심 원리

### 금융 지식 그래프 구조

```
[금융 트랜잭션 지식 그래프]

노드 유형:
  (U) 사용자/계좌
  (M) 가맹점
  (D) 기기(Device)
  (IP) IP 주소

엣지 유형:
  U ──결제──▶ M (거래 관계)
  U ──사용──▶ D (기기 사용)
  D ──접속──▶ IP (IP 접속)
  U ──이체──▶ U (계좌 이체)

사기 패턴 탐지:
  같은 IP에서 여러 계좌가 서로 다른 가맹점에 결제
  → GNN이 IP-계좌-가맹점 서브그래프 패턴 학습
```

### GNN 작동 원리 (Message Passing)

```
[GNN Message Passing 메커니즘]

노드 A의 특성 계산:

1단계: 이웃 노드 메시지 수집
   A ← [B의 특성, C의 특성, D의 특성]

2단계: 집계(Aggregation)
   A_new = Aggregate(A_old, Neighbors(A))
   = MEAN/SUM/MAX(B, C, D 특성)

3단계: 업데이트
   A의 새 임베딩 = Update(A_old, aggregated_msg)

→ K번 반복: K-hop 이웃까지 정보 전파
→ 3-hop이면 친구의 친구의 친구 관계까지 반영
```

### FDS에서 자주 쓰이는 GNN 모델

| 모델 | 특징 | FDS 활용 |
|:---|:---|:---|
| **GraphSAGE** | 샘플링 기반, 귀납적 학습 | 신규 노드(새 계좌) 지원 |
| **GAT** (Graph Attention Network) | 이웃 중요도 가중치 | 핵심 연결 관계 강조 |
| **GIN** (Graph Isomorphism Network) | 구조 구분 능력 최대 | 그래프 구조 패턴 학습 |
| **Temporal GNN** | 시간 정보 반영 | 시간에 따른 행동 변화 |

📢 **섹션 요약 비유**: Message Passing은 마을 소문 전파와 같다. 내 이웃이 어떤 사람들인지(이웃 노드 특성)가 나에 대한 평판(노드 임베딩)에 영향을 준다. 나쁜 이웃들과 연결된 계좌는 자신이 정상 거래를 해도 의심받게 된다.

## Ⅲ. 비교 및 연결

### FDS 접근법 비교

| 방법 | 탐지 기반 | 장점 | 단점 |
|:---|:---|:---|:---|
| **규칙 기반** | 전문가 정의 규칙 | 설명 가능, 빠름 | 신규 패턴 미탐지, 유지 비용 |
| **단일 트랜잭션 ML** | 거래 단위 특성 | 구현 용이 | 관계 패턴 미반영 |
| **시계열 ML** | 시간 시퀀스 패턴 | 행동 패턴 학습 | 관계 네트워크 미반영 |
| **GNN** | 그래프 관계 구조 | 공모·네트워크 사기 탐지 | 높은 구현 복잡성, 설명 어려움 |
| **하이브리드** | 규칙 + ML + GNN | 다층 방어 | 시스템 복잡성 최대 |

### 실시간 vs 배치 FDS 아키텍처

```
[하이브리드 FDS 아키텍처]

실시간 스트림 (< 100ms):
트랜잭션 ──▶ 규칙 엔진 ──▶ 단순 사기 즉시 차단
            + 경량 ML (RandomForest 등)

준실시간 (수분~수시간):
트랜잭션 ──▶ GNN 서빙 ──▶ 그래프 기반 사기 탐지
            (사전 계산된 노드 임베딩 활용)

배치 (일간):
전체 그래프 ──▶ GNN 재훈련 ──▶ 신규 패턴 학습
                              모델 업데이트
```

📢 **섹션 요약 비유**: FDS 계층 방어는 공항 보안과 같다. 1차(규칙)는 금속 탐지기로 빠르게 통과, 2차(ML)는 수하물 X-ray로 세부 검사, 3차(GNN)는 승객 네트워크(동행자, 예약 패턴) 분석이다. 각 계층이 서로 다른 유형의 위협을 잡는다.

## Ⅳ. 실무 적용 및 기술사 판단

### GNN FDS 구축 아키텍처

```
[실시간 GNN FDS 파이프라인]

거래 발생
    │
    ▼
[Kafka 스트리밍]
    │
    ▼
[그래프 DB 업데이트] ── Neo4j / Amazon Neptune
(신규 노드·엣지 추가)      (실시간 그래프 유지)
    │
    ▼
[노드 임베딩 서비스] ── 사전 계산된 임베딩 캐시 (Redis)
(GraphSAGE 추론)        ← 주기적 배치 업데이트
    │
    ▼
[FDS 분류 모델] ── 승인/거절/모니터링 결정
    │
    ▼
[결과 저장 + 설명] ── 왜 사기로 탐지됐는지 그래프 서브
                         패턴 시각화
```

### 성능 최적화 전략

1. **임베딩 캐싱**: 자주 변하지 않는 노드 임베딩을 Redis에 캐싱 → GNN 추론 비용 절감
2. **샘플링**: GraphSAGE의 이웃 샘플링으로 대규모 그래프에서 효율적 추론
3. **배치 추론**: 실시간 요청을 마이크로 배치로 묶어 GPU 활용률 극대화
4. **그래프 파티셔닝**: 대규모 그래프를 분산 저장하여 조회 지연 최소화

### 기술사 시험 판단 포인트

- **GNN의 핵심 차별점**: "개별 노드 특성" + "그래프 구조(토폴로지)" 동시 학습
- **설명 가능성 문제**: GNN은 XAI(설명 가능 AI) 기법(GNNExplainer)을 추가해야 규제 요건 충족
- **금융규제 연계**: 금융 FDS는 탐지 결과에 대한 설명 의무 → GNN의 설명 가능성이 핵심 과제

📢 **섹션 요약 비유**: GNN FDS의 딜레마는 "정확하지만 설명하기 어렵다"는 것이다. "이 계좌가 3단계 연결을 통해 사기 계좌와 연결되어 있습니다"라는 설명을 고객과 규제기관에 납득시키는 것이 기술 구현만큼 어려운 숙제다.

## Ⅴ. 기대효과 및 결론

### 기대효과

| 영역 | 효과 |
|:---|:---|
| **탐지율 향상** | 관계 패턴 기반 탐지로 공모 사기·자금 세탁 탐지율 향상 |
| **오탐율 감소** | 관계 맥락 반영으로 단순 규칙보다 정상 거래 차단 감소 |
| **신규 패턴 학습** | 배치 재훈련으로 새로운 사기 패턴 지속 학습 |
| **다분야 확장** | 보험 사기, 의료 청구 사기, 사이버 보안 이상 탐지에 동일 적용 |

### 한계 및 주의사항

- **설명 가능성**: GNN 결정 이유를 규제기관에 설명하기 어려움 — XAI 기법 필수
- **그래프 구축 비용**: 실시간 대규모 그래프 유지·업데이트 인프라 비용 높음
- **레이블 불균형**: 실제 사기 데이터는 전체의 0.1% 미만 — SMOTE, 오버샘플링 필요
- **Privacy 우려**: 그래프 데이터에 개인 관계 정보가 포함 → 개인정보 처리 기준 명확화 필요

📢 **섹션 요약 비유**: GNN FDS는 최신 예방접종이다. 기존 백신(규칙 기반)이 알려진 바이러스(패턴)만 막았다면, GNN은 바이러스의 유전자 네트워크(관계 구조)를 분석하여 변형 바이러스까지 탐지한다. 단, 새 백신 개발(모델 재훈련)에 계속 투자해야 한다.

### 👶 어린이를 위한 3줄 비유 설명
1. GNN FDS는 범인을 혼자가 아니라 "친구들까지" 보는 탐정이야. 범인 혼자는 정상처럼 보여도 친구들이 다 나쁜 사람이면 의심받아.
2. 돈이 A → B → C → 인출 순서로 빙빙 돌면, 각 거래는 정상이지만 전체 흐름이 이상한 거야. 이걸 그래프로 보고 찾아내는 거야.
3. 마치 연락처 앱처럼 "이 전화번호가 사기꾼 X와 3단계 연결"이라는 걸 AI가 알아내서 차단하는 거야!
