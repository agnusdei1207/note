## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연합 학습(Federated Learning)은 원본 데이터를 중앙 서버로 전송하지 않고 각 기기/노드에서 로컬 모델 훈련 후 모델 파라미터(그래디언트)만 집계하여, 데이터 프라이버시를 보존하면서 분산 ML을 가능하게 한다.
> 2. **가치**: 병원들이 환자 데이터를 공유하지 않고도 공동으로 질병 예측 모델을 훈련할 수 있고, 스마트폰이 사용자 키보드 입력 패턴으로 자동완성 모델을 개선하면서 개인 데이터는 기기 밖으로 나가지 않는다.
> 3. **판단 포인트**: Horizontal FL(데이터 행 기준 분산) vs Vertical FL(데이터 컬럼 기준 분산) vs Federated Transfer Learning 3가지 유형을 구분하고, 통신 비용(모델 파라미터 전송)과 비IID 데이터(데이터 이질성)가 핵심 기술적 과제임을 이해해야 한다.

---

## Ⅰ. 개요 및 필요성

의료 AI 개발의 딜레마: 정확한 암 진단 모델 훈련에는 수십만 건의 의료 영상이 필요하지만, 각 병원은 환자 개인정보 보호 때문에 데이터를 공유할 수 없다.

전통적 해결책은 데이터를 중앙 서버로 수집하는 것이었지만:
- GDPR, 개인정보보호법 위반 위험
- 데이터 주권 침해
- 보안 사고 시 전체 데이터 노출

**연합 학습(Federated Learning, FL)**은 "데이터는 이동하지 않고, 학습만 이동한다"는 혁신적 패러다임이다.

```
[연합 학습 vs 전통 중앙집중 학습]

전통 중앙집중 학습:
병원 A 데이터 ──┐
병원 B 데이터 ──┼──▶ 중앙 서버 ──▶ 모델 훈련
병원 C 데이터 ──┘    (데이터 노출 위험!)

연합 학습:
병원 A: 로컬 데이터로 훈련 ──▶ 파라미터(그래디언트)만 전송 ──┐
병원 B: 로컬 데이터로 훈련 ──▶ 파라미터만 전송 ──────────────┼──▶ 중앙 집계 ──▶ 글로벌 모델
병원 C: 로컬 데이터로 훈련 ──▶ 파라미터만 전송 ──────────────┘
(원본 데이터는 절대 병원 밖으로 나가지 않음!)
```

📢 **섹션 요약 비유**: 연합 학습은 요리 경연대회에서 레시피(모델 파라미터)만 공유하는 것이다. 각 셰프(기기)는 자신의 비밀 재료(데이터)로 요리를 만들고, 맛 평가 결과(그래디언트)만 제출한다. 비밀 재료는 절대 공개하지 않는다.

## Ⅱ. 아키텍처 및 핵심 원리

### FedAvg (Federated Averaging) 알고리즘

```
[FedAvg 동작 과정]

1라운드:
중앙 서버: 글로벌 모델 w_0 초기화
        │ w_0 배포
        ▼
클라이언트 k=1,2,...,K 중 일부 선택

각 클라이언트:
w_k ← LocalUpdate(w_0, local_data_k)
(로컬 데이터로 E번의 에포크 훈련)
        │
        ▼ w_k 업로드 (파라미터만, 데이터 아님)

중앙 서버:
w_1_new = Σ(n_k/n) × w_k  (데이터 크기로 가중 평균)
(n_k = 클라이언트 k의 데이터 수, n = 전체 합계)
        │
        ▼ 다음 라운드에 w_1_new 배포
...반복...
```

### 연합 학습 유형

| 유형 | 특징 | 예시 |
|:---|:---|:---|
| **Horizontal FL** | 동일 특성, 다른 샘플 (행 기준 분산) | 병원들이 같은 환자 속성, 다른 환자들 보유 |
| **Vertical FL** | 다른 특성, 같은 샘플 (열 기준 분산) | 은행(거래 내역) + 보험사(의료 기록) = 같은 고객 |
| **Federated Transfer Learning** | 특성+샘플 모두 다름 | 도메인이 다른 기업 간 협력 |

### 비IID 데이터 문제

```
[비IID 문제: 클라이언트 데이터 이질성]

IID (Independent and Identically Distributed):
이상적 상황 - 모든 클라이언트 데이터가 동일 분포

비IID (현실):
스마트폰 A: 영어 사용자 데이터
스마트폰 B: 한국어 사용자 데이터
병원 A: 노인 환자 중심
병원 B: 소아 환자 중심

→ 로컬 모델이 자신의 데이터에 과적합(Overfitting)
→ 집계된 글로벌 모델 성능 저하

해결책:
- FedProx: 로컬 모델이 글로벌 모델에서 너무 벗어나지 않도록 제약
- SCAFFOLD: 클라이언트 드리프트 보정
- Personalized FL: 글로벌 + 로컬 개인화 혼합
```

📢 **섹션 요약 비유**: 비IID 문제는 전국 학교 표준 시험 출제의 어려움이다. 강남 학생과 농촌 학생의 학습 환경이 다르면, 어느 한쪽에만 맞는 모델(교사)이 전국 표준이 될 수 없다. FedProx는 각 선생님이 너무 지역 특성에만 맞추지 않도록 제약하는 것이다.

## Ⅲ. 비교 및 연결

### 연합 학습 vs 프라이버시 보존 기법 비교

| 기법 | 원본 공유 | 계산 위치 | 적합 사례 |
|:---|:---|:---|:---|
| **연합 학습** | 공유 안 함 | 로컬 | 모바일 기기, 의료 협업 |
| **차분 프라이버시** | 부분 (통계) | 중앙 | 통계 공개 |
| **동형 암호** | 암호화된 데이터 | 중앙 | 매우 민감한 데이터 |
| **안전한 다자 계산** | 공유 안 함 | 분산 | 기업 간 협력 |
| **연합 학습 + DP** | 공유 안 함 | 로컬 + 노이즈 | 최강 프라이버시 |

### 실제 적용 사례

| 기업/분야 | 연합 학습 활용 | 성과 |
|:---|:---|:---|
| **Google Gboard** | 스마트폰 키보드 자동완성 개선 | 개인 타이핑 패턴 서버 전송 없음 |
| **Apple** | Siri, 이모지 추천 개선 | On-device 프라이버시 |
| **NHS (영국 의료)** | COVID-19 예측 모델 | 13개 병원 협력, 데이터 비공유 |
| **금융권** | 사기 탐지 모델 공동 훈련 | 경쟁사 간 데이터 비공유 |

📢 **섹션 요약 비유**: 연합 학습은 각 주방장이 비밀 레시피를 숨기고 요리 평가단에게만 평가(그래디언트)를 제출하는 요리학교다. 학교(중앙 서버)는 모든 평가를 종합하여 더 나은 요리 지침서(글로벌 모델)를 만들지만, 어떤 비밀 재료를 썼는지는 절대 알 수 없다.

## Ⅳ. 실무 적용 및 기술사 판단

### 연합 학습 프레임워크

| 프레임워크 | 개발 주체 | 특징 |
|:---|:---|:---|
| **TensorFlow Federated (TFF)** | Google | TF 기반, 연구·프로덕션 |
| **PySyft** | OpenMined | 프라이버시 보존 ML 중심 |
| **Flower (flwr)** | Flower Labs | 프레임워크 불가지론적 |
| **FATE** | WeBank | 엔터프라이즈, Vertical FL 강점 |

### 통신 비용 최적화

```
[연합 학습의 주요 오버헤드: 통신 비용]

문제: 대규모 딥러닝 모델 (GPT: 수십억 파라미터)
→ 매 라운드마다 수 GB의 파라미터 업로드

최적화 방법:
1. 압축 (Compression):
   - 파라미터 양자화 (32bit → 8bit)
   - 희소 업데이트 (변화 큰 파라미터만 전송)

2. 라운드 수 감소:
   - 로컬 에포크 수 증가 (한 번 통신으로 더 많이 학습)

3. 클라이언트 선택:
   - 배터리·네트워크 상태 좋은 기기만 참여
```

### 기술사 시험 판단 포인트

- **연합 학습의 한계**: 데이터가 기기를 떠나지 않아도 그래디언트에서 원본 데이터 역추론 가능 (Gradient Inversion Attack)
- **FL + DP 조합**: 그래디언트에 차분 프라이버시 노이즈를 추가하여 역추론 방어
- **Vertical FL 특수성**: 같은 사용자의 다른 특성이 다른 서버에 있을 때 — PSI(Private Set Intersection) 기법으로 사용자 매칭

📢 **섹션 요약 비유**: 연합 학습도 완벽하지 않다. 셰프가 "평가 결과"만 제출해도 전문 미식가(공격자)가 평가 패턴을 분석하여 레시피(데이터)를 역추론할 수 있다. 이것이 Gradient Inversion Attack이다. 차분 프라이버시로 노이즈를 추가하는 것이 방어책이다.

## Ⅴ. 기대효과 및 결론

### 기대효과

| 영역 | 효과 |
|:---|:---|
| **프라이버시 보존** | 원본 데이터를 중앙 서버로 전송하지 않음 |
| **데이터 주권** | 각 기관이 자신의 데이터에 대한 통제권 유지 |
| **규제 준수** | GDPR, HIPAA, 개인정보보호법 부담 완화 |
| **모델 품질** | 단일 기관 대비 더 많은 데이터로 훈련된 강한 모델 |

### 한계 및 주의사항

- **통신 비용**: 많은 라운드의 파라미터 업로드가 대역폭 소비
- **비IID 데이터 성능 저하**: 클라이언트 데이터 이질성이 심할수록 글로벌 모델 성능 하락
- **그래디언트 공격**: 파라미터에서 원본 데이터 역추론 가능 — DP와 결합 필수
- **조율 복잡성**: 수천 개 기기의 훈련 스케줄링, 장애 처리, 버전 관리 어려움

📢 **섹션 요약 비유**: 연합 학습은 데이터 국경을 넘지 않는 AI 외교다. 각 나라(기기/기관)는 자국 데이터(정보)를 내주지 않지만, 공동 AI 모델(조약)을 함께 만들어나간다. 국경을 넘지 않는 만큼 안전하지만, 협상(통신)과 조율(집계)에 비용이 든다.

### 👶 어린이를 위한 3줄 비유 설명
1. 연합 학습은 각 반에서 수학 숙제를 따로 하고, 선생님한테는 "이번 주 이렇게 이해했어요"(그래디언트)만 제출하는 거야.
2. 선생님은 모든 반의 이해도를 합쳐서 더 좋은 설명 방법(글로벌 모델)을 만들어줘. 각 반의 숙제 내용은 절대 다른 반에 공개 안 해!
3. 핸드폰이 내 타이핑 패턴을 학습하는 것도 이 방식이야. 내가 뭘 입력했는지 삼성이나 애플 서버로 전송되지 않아!
