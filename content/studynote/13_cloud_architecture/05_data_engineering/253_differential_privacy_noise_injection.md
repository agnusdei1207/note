## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터베이스 쿼리 결과에 수학적으로 보정된 미세 노이즈를 주입하여, 공격자가 특정 개인의 데이터 포함 여부를 파악할 수 없게 만드는 통계적 프라이버시 보호 기법이다.
> 2. **가치**: 데이터를 직접 마스킹하거나 삭제하지 않고도 통계적 유용성을 유지하면서 개인 정보를 보호 — Apple iOS 사용 통계, Google Chrome 사용 데이터 수집에 실제 적용.
> 3. **판단 포인트**: ε(엡실론, 프라이버시 예산)이 작을수록 프라이버시 강하고 정확도 낮아짐. ε = 0은 완전한 프라이버시이나 결과는 무의미. ε과 정확도(유용성)의 트레이드오프가 핵심 설계 변수다.

## Ⅰ. 개요 및 필요성

"이 병원 데이터셋에 암환자가 몇 명인가?"라는 통계 쿼리에 정확한 숫자 42를 답하면, 공격자는 자신이 아는 41명의 암환자 목록과 비교하여 나머지 1명이 누구인지 역추적할 수 있다. 이것이 **통계적 재식별 공격(Statistical Re-identification Attack)**이다.

차분 프라이버시(Differential Privacy, DP)는 이 문제를 수학적으로 해결한다. 핵심 아이디어는 단순하다: **답에 노이즈를 더해서 공격자가 특정 개인의 포함 여부를 파악하지 못하게 만든다.**

```
[차분 프라이버시 직관]

실제 데이터셋 D:   {Alice, Bob, Carol, Dave, Eve ... 42명의 암환자}
데이터셋 D':       {       Bob, Carol, Dave, Eve ... 41명 (Alice 제외)}

D  쿼리 결과: 42 + 노이즈 → 43.2
D' 쿼리 결과: 41 + 노이즈 → 41.8

공격자: "Alice가 포함됐는지 구별할 수 없다 → 개인정보 보호!"
```

### 수학적 정의

알고리즘 M이 (ε, δ)-차분 프라이버시를 만족하려면:

```
Pr[M(D) ∈ S] ≤ e^ε × Pr[M(D') ∈ S] + δ

D, D': 한 레코드만 다른 두 데이터셋
ε (엡실론): 프라이버시 예산 (작을수록 강한 프라이버시)
δ: 허용 확률적 실패 마진 (일반적으로 극소값)
```

📢 **섹션 요약 비유**: 차분 프라이버시는 시험 성적 통계에서 한 학생의 점수를 숨기는 방법이다. "평균 75점"이라고 공개할 때 실제 74.3점에 0.7점 노이즈를 더해서, 내 점수가 포함됐는지 아닌지 결코 알 수 없게 만든다.

## Ⅱ. 아키텍처 및 핵심 원리

### 노이즈 메커니즘

| 메커니즘 | 수식 | 적용 분야 |
|:---|:---|:---|
| **라플라스 메커니즘** | Noise ~ Laplace(0, Δf/ε) | 수치형 쿼리 (평균, 합계) |
| **가우시안 메커니즘** | Noise ~ N(0, σ²) | 고차원 데이터, 근사 DP |
| **지수 메커니즘** | 확률적 출력 선택 | 범주형 최적 선택 |
| **랜덤화 응답** | 동전 던지기로 응답 결정 | 개인 로컬 DP 설문 |

### 전역 DP vs 로컬 DP

```
전역 차분 프라이버시 (Central DP)
개인 데이터 ──▶ 신뢰 집계 서버 ──▶ [노이즈 주입] ──▶ 공개 통계

개인     데이터     서버
[Alice] ────────▶ |        |
[Bob]   ────────▶ | 원본   | ──▶ 결과+노이즈 ──▶ 외부
[Carol] ────────▶ | 수집   |
                  |        |
                  (서버 신뢰 필요!)

로컬 차분 프라이버시 (Local DP)
개인 데이터 ──▶ [개인 기기에서 노이즈 주입] ──▶ 노이즈 포함 데이터 전송

개인     데이터     로컬 노이즈         서버
[Alice] ────────▶ [Alice+noise] ──▶ |        |
[Bob]   ────────▶ [Bob+noise]   ──▶ | 집계   | ──▶ 결과
[Carol] ────────▶ [Carol+noise] ──▶ |        |

(서버 신뢰 불필요 → Apple, Google 방식)
```

### ε(엡실론) 프라이버시 예산

```
ε 값     프라이버시   정확도    적용 사례
─────────────────────────────────────────
ε ≈ 0.01  매우 강함   매우 낮음  극히 민감한 의료 데이터
ε ≈ 1     강함        낮음       Apple 로컬 DP 기본값
ε ≈ 10    중간        중간       Google RAPPOR 프레임워크
ε ≈ 100   약함        높음       실용적 비즈니스 분석
```

📢 **섹션 요약 비유**: 엡실론은 "진실을 얼마나 흐리게 말하느냐"의 정도다. 0이면 완전한 거짓말(무작위), 무한대면 완전한 진실. 실용에서는 그 중간 어딘가에서 균형을 잡는다.

## Ⅲ. 비교 및 연결

### 차분 프라이버시 vs 다른 프라이버시 기법

| 기법 | 보호 방식 | 유용성 | 재식별 위험 | 적합 용도 |
|:---|:---|:---|:---|:---|
| **데이터 마스킹** | PII 직접 제거/치환 | 높음 (구조 유지) | 중간 (조합 공격) | 내부 개발 환경 |
| **k-익명성** | k명 이상 동일 속성 보장 | 중간 | 낮음 | 데이터 공개 |
| **차분 프라이버시** | 노이즈 주입으로 통계 모호화 | 통계적으로 높음 | 매우 낮음 | 통계 쿼리, ML 훈련 |
| **동형 암호** | 암호화된 채로 연산 | 낮음 (성능 저하) | 없음 | 매우 민감한 데이터 |

### 실제 적용 사례

| 기업 | 적용 방식 | ε 값 |
|:---|:---|:---|
| **Apple** | iOS 사용 통계 (로컬 DP) | ε = 1 (기기당 하루) |
| **Google** | Chrome URL 통계 수집 RAPPOR | ε ≈ 2~8 |
| **Microsoft** | Windows 진단 데이터 | ε = 0.01~5 |
| **US Census Bureau** | 2020 인구 조사 공개 | ε = 17.14 |

📢 **섹션 요약 비유**: k-익명성이 "군중 속에 숨기"라면, 차분 프라이버시는 "목소리에 화이트노이즈 추가하기"다. 목소리(데이터)는 있지만 개인을 특정할 수 없게 만든다.

## Ⅳ. 실무 적용 및 기술사 판단

### ML 훈련에서의 차분 프라이버시 (DP-SGD)

```
[차분 프라이버시 적용 모델 훈련]

일반 SGD:
배치 데이터 ──▶ 그래디언트 계산 ──▶ 모델 업데이트

DP-SGD (Differentially Private SGD):
배치 데이터 ──▶ 그래디언트 계산
                    │
                    ▼
              [그래디언트 클리핑] ← 개별 그래디언트 크기 제한
                    │
                    ▼
              [가우시안 노이즈 주입] ← ε 예산에 따라 노이즈 크기 결정
                    │
                    ▼
              모델 업데이트
```

### TensorFlow Privacy / PyTorch Opacus

```python
# TensorFlow Privacy를 이용한 DP 모델 훈련 (개념)
from tensorflow_privacy.privacy.optimizers import dp_optimizer

optimizer = dp_optimizer.DPKerasSGDOptimizer(
    l2_norm_clip=1.0,         # 그래디언트 클리핑
    noise_multiplier=1.1,     # 노이즈 배율 (엡실론 결정)
    num_microbatches=256      # 마이크로 배치 크기
)
model.compile(optimizer=optimizer, ...)
```

### 기술사 시험 판단 포인트

- **핵심 질문**: "특정 개인이 데이터셋에 포함됐는지 공격자가 알 수 있는가?" → DP는 이를 수학적으로 불가능하게 함
- **ε 예산 합성**: 여러 DP 쿼리를 수행하면 예산이 소진됨 (합성 정리, Composition Theorem)
- **글로벌 vs 로컬 DP**: 신뢰할 수 있는 집계 서버가 있으면 글로벌 DP가 정확도 우수, 없으면 로컬 DP

📢 **섹션 요약 비유**: DP로 모델을 훈련하면 모델이 "개인의 기억"을 갖지 않는다. 모든 사람의 데이터를 골고루 흡수하되, 누구 한 명의 특징이 두드러지지 않도록 노이즈가 평균화시킨다.

## Ⅴ. 기대효과 및 결론

### 기대효과

| 영역 | 효과 |
|:---|:---|
| **통계 공개** | 인구·의료 통계를 안전하게 외부 공개 가능 |
| **ML 프라이버시** | 모델이 훈련 데이터를 기억하지 않음 (Memorization 방지) |
| **데이터 공유** | 규제 기관에 실 데이터 제공 없이 통계 쿼리 허용 |
| **법규 준수** | GDPR 프라이버시 by Design 원칙 충족 |

### 한계 및 주의사항

- **유용성-프라이버시 트레이드오프**: 강한 프라이버시는 분석 결과의 정확도를 낮춤
- **고차원 데이터 취약**: 데이터 차원이 높을수록 필요한 노이즈량이 급증 (차원의 저주)
- **ε 예산 관리 복잡성**: 복수 쿼리 수행 시 예산 추적이 필요하며, 고갈 시 프라이버시 보장 불가
- **구현 복잡성**: 노이즈 파라미터 잘못 설정 시 의미 없는 결과 또는 프라이버시 침해 가능

📢 **섹션 요약 비유**: 차분 프라이버시는 "완벽한 방패"가 아니라 "확률의 안개"다. 공격자가 안개 속을 헤매게 만들지만, 충분한 자원을 가진 공격자가 오랫동안 시도하면 안개가 걷힐 수도 있다. ε 예산이 그 한계를 수학적으로 정의한다.

### 👶 어린이를 위한 3줄 비유 설명
1. 차분 프라이버시는 친구들에게 "우리 반 평균 키"를 알려줄 때, 내 정확한 키는 감추기 위해 살짝 틀린 평균을 말하는 것이야.
2. 틀렸지만 "거의 맞는" 숫자라서 반 아이들의 통계는 제대로 쓸 수 있어, 근데 내 키만큼은 절대 알 수 없어!
3. 애플 아이폰이 "몇 명이 이 앱을 쓰나요?"를 물어볼 때 이 방법을 써서, 애플도 내가 어떤 앱 쓰는지 정확히 모르게 해.
