# 언더레이 네트워크 (Underlay Network) - 클라우드 아키텍처의 견고한 물리적 기반

> ⚠️ 이 문서는 가상 네트워크(Overlay)가 원활하게 동작할 수 있도록 밑바닥에서 패킷 전송을 책임지는 물리적 기반 인프라인 '언더레이 네트워크'를 분석합니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 언더레이 네트워크(Underlay Network)는 서버, 스토리지 및 네트워크 장비들을 연결하는 실제 물리적 인프라(L2 스위치, L3 라우터, 광케이블)를 의미한다.
> 2. **역할**: 오버레이 네트워크(Overlay)의 터널링 패킷을 목적지까지 가장 빠르고 안정적으로 전달(Transport)하는 전송망 역할을 수행하며, 단순하고 견고한 설계를 지향한다.
> 3. **가치**: 클라우드의 성능과 확장성은 언더레이의 대역폭과 안정성에 좌우되며, 최근에는 Spine-Leaf 아키텍처를 통해 데이터센터 내부의 트래픽 정체를 해결한다.

---

### Ⅰ. 개요 (Context & Background)

최근 클라우드 기술이 오버레이(VXLAN 등 가상망)에 집중되다 보니, 그 밑에 깔린 실제 기계들에 소홀해지기 쉽습니다. 하지만 아무리 화려한 가상 네트워크도 밑바닥의 물리 선이 끊기거나 스위치가 느리면 제 성능을 낼 수 없습니다.

- **Pain Point**: 전통적인 계층적 네트워크(3-Tier)는 데이터센터 내 서버 간 트래픽(East-West)이 급증할 때 상단 스위치에 병목이 발생하고 경로가 비효율적입니다.
- **도입 목적**: 가상화된 트래픽을 지연 없이 전송하고, 하드웨어 장애 시에도 경로를 즉시 우회하여 높은 가용성을 보장하기 위함입니다.

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

현대 클라우드 데이터센터의 언더레이는 주로 **Spine-Leaf 아키텍처**를 채택합니다.

```text
┌─────────────────────────────────────────────────────────────┐
│               [ Underlay Network Architecture ]             │
│                                                             │
│      [ Spine Switch 1 ]           [ Spine Switch 2 ]        │
│             ▲    ▲                     ▲    ▲               │
│             │    └───────────┬─────────┘    │               │
│      ┌──────┴──────┐         │         ┌────┴────────┐      │
│      │ Leaf Switch │         │         │ Leaf Switch │      │
│      └──────┬──────┘         │         └──────┬──────┘      │
│             │                │                │             │
│      ┌──────┴──────┐  ┌──────┴──────┐  ┌──────┴──────┐      │
│      │   Server 1  │  │   Server 2  │  │   Server 3  │      │
│      └─────────────┘  └─────────────┘  └─────────────┘      │
│                                                             │
│  ※ Spine-Leaf: Every Leaf is connected to every Spine.      │
│  ※ Protocol: BGP, OSPF (IP-based Routing)                   │
└─────────────────────────────────────────────────────────────┘
```

#### 1. 핵심 구성 및 원리
1. **Leaf Switch (액세스 계층)**: 서버들이 직접 연결되는 지점입니다. 모든 Leaf는 상단의 모든 Spine과 연결됩니다.
2. **Spine Switch (백본 계층)**: Leaf 스위치들 사이의 패킷을 고속으로 중계합니다. Spine끼리는 서로 연결되지 않는 것이 특징입니다.
3. **L3 라우팅 (IP Fabric)**: 과거에는 L2(MAC) 기반의 루프 방지 기술(STP)을 썼으나, 현재는 L3(IP) 라우팅 프로토콜(BGP, OSPF)을 사용하여 모든 경로를 동시에 사용(ECMP, Equal-Cost Multi-Pathing)함으로써 대역폭을 극대화합니다.

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

언더레이와 오버레이의 상호 관계를 비교합니다.

| 비교 항목 | 언더레이 (Underlay) | 오버레이 (Overlay) |
| :--- | :--- | :--- |
| **구성 요소** | 물리 스위치, 라우터, 케이블 | 가상 스위치, 터널링 (VXLAN) |
| **주요 역할** | 패킷의 물리적 전달 (Transport) | 논리적 격리 및 가상 망 구성 |
| **주소 체계** | 물리 IP (VTEP IP) | 가상 IP (VM/Container IP) |
| **관심 지표** | 처리량(Gbps), 지연(ms), 패킷 손실 | 테넌트 격리, 이동성, 유연성 |

- **시너지**: 언더레이는 최대한 단순하게(MTU 크기 확대 등) 유지하고, 모든 복잡한 비즈니스 로직(VLAN 분리 등)은 오버레이에 맡기는 것이 현대 클라우드 네트워크의 정석입니다.

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**1. 기술사적 판단 및 설계 가이드**
- **MTU 설정의 중요성**: 오버레이 패킷은 원본 패킷에 헤더를 덧붙이므로(Encapsulation), 언더레이 네트워크의 MTU(Maximum Transmission Unit)를 일반적인 1,500보다 큰 **9,000(Jumbo Frame)**으로 설정해야 패킷 파편화와 성능 저하를 막을 수 있습니다.
- **가용성 설계**: 특정 스위치나 라인이 죽어도 성능 저하가 최소화되도록 Spine-Leaf의 밀도를 조정해야 합니다.

**2. 실무 적용 사례**
- **Hyper-scale 데이터센터**: 구글, 페이스북 등의 데이터센터는 수천 대의 Spine 스위치를 엮은 패브릭 구조로 무한 확장이 가능하게 설계되어 있습니다.
- **SD-WAN**: 본사와 지사 사이의 물리 망(인터넷, 전용선)이 언더레이가 되고, 그 위에 보안 터널을 뚫는 것이 오버레이가 됩니다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

견고한 언더레이는 클라우드 서비스 안정성의 8할을 차지합니다.

- **미래 전망**: 하드웨어 자체에 지능을 부여하는 **P4 프로그래밍**이 가능한 스위치가 도입되어, 언더레이 수준에서도 특정 트래픽을 감지하고 제어하는 능력이 강화되고 있습니다.
- **결론**: 오버레이의 유연성은 언더레이의 견고함 위에서만 존재합니다. **"Underlay는 단순하게, Overlay는 지능적으로"**가 네트워크 현대화의 핵심 원칙입니다.

---

### 👶 어린이를 위한 3줄 비유 설명
1. 언더레이 네트워크는 "실제로 차가 다니는 튼튼한 고속도로"와 같아요.
2. 오버레이(가상망)가 그 위를 달리는 자동차라면, 언더레이는 구멍 난 곳 없는 아스팔트 길인 거죠.
3. 고속도로가 넓고 튼튼해야 자동차들이 쌩쌩 달릴 수 있듯이, 언더레이가 좋아야 클라우드도 빨라진답니다.
