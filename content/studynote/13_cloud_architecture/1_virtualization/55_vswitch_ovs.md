# 가상 스위치 (vSwitch) 및 OVS - 가상 세계의 신경망을 잇는 소프트웨어 다리

> ⚠️ 이 문서는 물리적인 네트워크 장비 없이 하이퍼바이저 내부에서 VM 간의 통신과 외부 네트워크 연결을 제어하는 가상 스위치(vSwitch)의 메커니즘과, 그 표준인 OVS(Open vSwitch)를 심층 분석합니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가상 스위치(vSwitch)는 하이퍼바이저 내부에 위치하여 가상 머신(VM)들의 가상 NIC(vNIC)를 상호 연결하고, 물리 NIC와 브리징하여 데이터 패킷을 라우팅하는 소프트웨어 기반 네트워크 장치다.
> 2. **OVS 가치**: Open vSwitch(OVS)는 오픈 소스 기반의 다계층 가상 스위치로, 흐름 기반 포워딩(Flow-based Forwarding)과 SDN(소프트웨어 정의 네트워크) 제어 프로토콜(OpenFlow)을 지원하여 클라우드 네트워킹의 유연성을 제공한다.
> 3. **융합**: SDN/NFV 환경의 핵심 요소로서, 오버레이 네트워크(VXLAN, Geneve) 구성과 세밀한 보안 정책(ACL) 적용을 가능하게 하여 동적인 클라우드 환경을 뒷받침한다.

---

### Ⅰ. 개요 (Context & Background)

서버 하나에 수십 개의 VM이 돌아가는 가상화 환경에서, 각 VM이 물리 스위치에 직접 연결되는 것은 불가능합니다. 하이퍼바이저 내부에서 VM끼리 대화를 나누고 밖으로 나갈 수 있게 해주는 '논리적인 스위치'가 반드시 필요합니다.

- **Pain Point**: 물리적 네트워크 구성만으로는 클라우드의 급격한 VM 생성/이동/삭제에 따른 네트워크 설정을 실시간으로 반영하기 어렵습니다.
- **도입 목적**: 소프트웨어적으로 네트워크 구성을 변경(Programmable)하고, VM 간 트래픽 가시성을 확보하며, 가상화의 탄력성을 네트워크 계층까지 확장하기 위함입니다.

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

가상 스위치는 하이퍼바이저 커널 또는 유저 공간에서 동작하며 물리 NIC와 vNIC 사이의 다리 역할을 합니다.

```text
┌─────────────────────────────────────────────────────────────┐
│                [ vSwitch / OVS Architecture ]               │
│                                                             │
│   [ VM A ]       [ VM B ]       [ VM C ]                    │
│   (vNIC)         (vNIC)         (vNIC)                      │
│      │              │              │                        │
│   ┌──▼──────────────▼──────────────▼──┐                     │
│   │        Virtual Switch (vSwitch)   │                     │
│   │  [ Forwarding Table / Flow Table ]│                     │
│   │  [ VLAN / VXLAN Tagging ]         │                     │
│   └──┬────────────────────────────────┘                     │
│      │                                                      │
│   ┌──▼──────────────────┐                                   │
│   │   Physical NIC      │  <── Bridge Mode / NAT Mode       │
│   └──────────┬──────────┘                                   │
│              │                                              │
│      [ Physical Switch ]                                    │
│                                                             │
│  ※ OVS Feature: OpenFlow Support & Overlay Tunneling        │
└─────────────────────────────────────────────────────────────┘
```

#### 1. Open vSwitch (OVS)의 주요 특징
1. **Flow-based 스위칭**: 단순히 MAC 주소를 보고 전달하는 것을 넘어, IP/포트/프로토콜 등 다양한 조건(Flow)에 따라 패킷을 처리(Drop, Forward, Modify)합니다.
2. **오버레이 터널링**: 물리 네트워크를 건드리지 않고 가상 머신들만의 독립된 L2망을 만들기 위해 VXLAN, GRE, Geneve와 같은 터널링 프로토콜을 사용합니다.
3. **SDN 연동**: 외부의 SDN 컨트롤러(OpenDaylight 등)와 OpenFlow 프로토콜로 통신하여 중앙 집중식 네트워크 관리가 가능합니다.

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

표준 vSwitch와 고성능 가속 기술을 비교합니다.

| 구분 | 리눅스 브리지 (Linux Bridge) | Open vSwitch (OVS) | OVS with DPDK |
| :--- | :--- | :--- | :--- |
| **복잡성** | 단순함 (기본 기능) | 높음 (다양한 기능) | 최고 (설정 복잡) |
| **확장성** | 낮음 (L2 위주) | 높음 (SDN, 터널링) | 높음 |
| **성능** | 보통 (커널 인터럽트) | 보통 (커널 모드) | 매우 높음 (User-space 가속) |
| **주요 용도** | 단순 가상화 환경 | OpenStack, 대규모 클라우드 | 고속 패킷 처리 (NFV) |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**1. 기술사적 판단 및 디자인 패턴**
- **East-West 트래픽 최적화**: 동일 호스트 내 VM 간 통신(East-West)은 물리 네트워크로 나가지 않고 vSwitch 내부에서 처리되므로 성능이 매우 빠릅니다. 기술사는 이를 고려하여 연관된 서비스를 동일 호스트에 배치(Affinity)하는 설계를 고려해야 합니다.
- **가시성 확보**: vSwitch는 NetFlow, sFlow 등을 지원하므로, 보이지 않는 가상 공간 내의 침입 탐지 및 트래픽 분석에 적극 활용해야 합니다.

**2. 실무 적용 시나리오**
- **멀티 테넌시**: 각 고객(Tenant)마다 독립적인 가상 네트워크를 제공하기 위해 OVS의 VLAN/VXLAN 기능을 사용하여 네트워크를 격리합니다.
- **보안**: vSwitch 레벨에서 보안 그룹(Security Group)이나 ACL을 설정하여 악성 트래픽의 전파를 차단합니다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

가상 스위치는 하드웨어 중심의 네트워크를 소프트웨어 중심의 유연한 서비스로 변화시켰습니다.

- **미래 전망**: eBPF(extended Berkeley Packet Filter) 기술을 활용하여 커널을 수정하지 않고도 초고속 패킷 처리와 관찰을 수행하는 방향으로 발전하고 있습니다. 또한 SmartNIC으로 vSwitch 기능을 오프로딩하여 CPU 부담을 줄이는 기술이 보편화되고 있습니다.
- **결론**: vSwitch는 클라우드의 혈관과 같습니다. **단순한 연결을 넘어 지능적인 트래픽 제어와 보안의 시작점**으로 이해해야 합니다.

### 👶 어린이를 위한 3줄 비유 설명
1. 가상 스위치는 "컴퓨터 안에 들어있는 보이지 않는 길 안내원"과 같아요.
2. 가상 머신들이 서로 편지를 주고받고 싶을 때, 누구에게 보내야 하는지 길을 알려주죠.
3. 실제 기계는 아니지만 소프트웨어로 똑똑하게 길을 만들어주는 마법 같은 장치랍니다.
