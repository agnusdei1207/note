## 핵심 인사이트 (3줄 요약)
> 1. **본질**: FTP는 TCP/IP 네트워크에서 파일을 전송하는 애플리케이션 프로토콜로, 제어 연결(21번 포트)과 데이터 연결을 분리하여 동작한다.
> 2. **가치**: 대용량 파일 전송, 디렉터리 浏览, 파일 권한 관리 등을 지원하며, 웹 개발, 시스템 관리 등 광범위하게 사용되었다.
> 3. **융합**: 현재는 FTPS, SFTP, 클라우드 스토리지 등으로 대체되어가고 있으나,legacy 시스템과의 호환성으로 여전히 일부 사용된다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

**개념**: FTP(File Transfer Protocol)는 RFC 959로 표준화된 파일 전송 프로토콜로, TCP 21번 포트로 제어 연결을 수립하고, 데이터 전송을 위해 별도의 데이터 연결을 만든다. FTP는 두 가지 모드(활성/수동)를 지원하며, ASCII와 Binary 두 가지 전송 모드를 제공한다. FTP는 Stateful 프로토콜로, 연결 수립 후 현재 디렉터리, 전송 모드 등의 상태를 유지한다.

**필요성**: 초창기 인터넷에서 파일을 공유하는 주요 수단이었으며, 대규모 파일 전송, 디렉터리 구조화된 관리, 접근 권한 관리 등의 기능이 필요했다. 현재는 보안 문제(평문 전송)로 인해逐渐淘汰되어, FTPS/SFTP, 클라우드 스토리지 등으로 대체되고 있다.

**비유**: FTP는 **식당 주문 시스템**과 같다. 손님이 웨이터에게「메뉴판 주세요」(명령 연결),「 steak 주세요」(요청), 웨이터가厨房에 가서 steak를 가져다주면(데이터 연결). 명령과 음식이 따로 오는 것과 같다.

**등장 배경**: 1971년 RFC 114에서 처음 정의되었으며, 1985년 RFC 959로 현재의 표준이 되었다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### FTP 활성 vs 수동 모드

```
┌───────────────────────────────────────────────────────────────────────┐
│                    FTP 활성 vs 수동 모드                                  │
├───────────────────────────────────────────────────────────────────────┤
│                                                                       │
│  【활성 모드 (Active Mode, PORT)】                                     │
│                                                                       │
│  Client                                                          Server │
│    │                                                                  │
│    │  ① TCP 21: CONNECT (제어 연결 수립)                              │
│    │ ──────────────────────────────────────────────────────────────▶ │
│    │                                                                  │
│    │  ② Client → Server: PORT 명령 (클라이언트 데이터 포트)           │
│    │     예: PORT 192,168,1,100,4,1 (IP + ports)                    │
│    │                                                                  │
│    │  ③ Server: TCP 20 → Client 데이터 포트 (데이터 연결)              │
│    │ ◀───────────────────────────────────────────────────────────── │
│    │                                                                  │
│  ⚠ 문제: Client Firewall가 외부에서 오는 연결을 차단할 수 있음         │
│                                                                       │
│  【수동 모드 (Passive Mode, PASV)】                                   │
│                                                                       │
│  Client                                                          Server │
│    │                                                                  │
│    │  ① TCP 21: CONNECT (제어 연결 수립)                              │
│    │ ──────────────────────────────────────────────────────────────▶ │
│    │                                                                  │
│    │  ② Client → Server: PASV 명령                                   │
│    │                                                                  │
│    │  ③ Server: 227 Entering Passive Mode (IP, Port)                │
│    │     예: 227 Entering Passive Mode (192,168,1,10,15,40)         │
│    │                                                                  │
│    │  ④ Client → Server: 데이터 포트 (데이터 연결)                    │
│    │ ──────────────────────────────────────────────────────────────▶ │
│    │                                                                  │
│  ✓ 장점: Firewall/NAT 환경에서 더 잘 동작                              │
│                                                                       │
└───────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** FTP의 활성/수동 모드 차이는 데이터 연결의 방향에 있다. 활성 모드에서 서버가客户端에 데이터 연결을发起하므로,客户端 Firewall이 Inbound 연결을 차단하면 동작하지 않는다. 수동 모드에서는客户端가 서버의 데이터 포트에 연결하므로,Firewall/NAT 환경에서 더 안전하다.因此、现代 FTPクライアント는 기본적으로 수동 모드를 사용한다.

### FTP 명령어

| 명령어 | 설명 |
|:---|:---|
| **USER** | 사용자 이름 전송 |
| **PASS** | 비밀번호 전송 |
| **LIST** | 디렉터리 목록 보기 |
| **RETR** | 파일 다운로드 |
| **STOR** | 파일 업로드 |
| **DELE** | 파일 삭제 |
| **MKD** | 디렉터리 생성 |
| **RMD** | 디렉터리 삭제 |
| **CWD** | 현재 디렉터리 변경 |
| **PWD** | 현재 디렉터리 경로 표시 |
| **QUIT** | 연결 종료 |

## Ⅲ. 융합 비교 및 다각도 분석

### FTP vs SFTP/FTPS

| 항목 | FTP | FTPS | SFTP |
|:---|:---|:---|:---|
| **보안** | 없음 (평문) | TLS 암호화 | SSH 기반 |
| **포트** | 21 + 별도 데이터 | 21 + 별도 데이터 | 22 (SSH) |
| **프로토콜** | 자체 규격 | FTP + TLS | SSH subsystem |
| **현재 상태** | 비권장 | 사용 가능 | 권장 |

## 👶 어린이를 위한 3줄 비유 설명

1. FTP는 **웨이터가 따로 주문(명령)을 받고, 요리를 따로 가져다주는** 식당 방식과 같아요. 주문을 받는端口(21번)과 음식을 받는 port가 다르다.
2. 활성 모드는 **厨房에서 직접 손님께 음식을 가져다주는** 거예요. 문제: 손님이 단지에 있으면 (Firewall) 받을 수 없어요.
3. 수동 모드는 **손님이 직접厨房에 가서 음식을 가져가는** 거예요. 이러면Firewall이 있어도 잘 돼요!
