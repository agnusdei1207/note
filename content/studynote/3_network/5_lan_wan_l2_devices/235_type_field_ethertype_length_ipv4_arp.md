## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이더넷 프레임 헤더의 목적지/출발지 MAC 주소 바로 다음에 위치하는 2바이트(16비트) 공간은, Ethernet II 프레임에서는 **Type(Ethertype)**으로 쓰이고, IEEE 802.3 프레임에서는 **Length(길이)**로 쓰여 데이터의 정체를 밝히는 다중화(Multiplexing)의 핵심 키(Key)다.
> 2. **Ethertype (Type)**: 페이로드(내용물)에 담겨 있는 상위 계층 3계층 프로토콜이 무엇인지(예: IPv4는 `0x0800`, ARP는 `0x0806`, IPv6는 `0x86DD`)를 수신측 랜카드에 명시적으로 알려주어, 랜카드가 이 데이터를 운영체제의 어느 소프트웨어 모듈(IP 모듈? ARP 모듈?)로 넘길지 결정하게 해 준다.
> 3. **0x0600 판별법**: 수신측은 이 2바이트 필드의 10진수 값이 1536(`0x0600`) 이상이면 프로토콜 종류를 나타내는 Type 필드로 해석하고, 1500(`0x05DC`) 이하면 데이터의 바이트 길이를 나타내는 Length 필드로 해석하여 프레임 포맷을 자동 구분한다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: 이더넷 통신에서 MAC 주소 필드가 "누구에게(Who)"를 나타낸다면, Type/Length 2바이트 필드는 "무엇을(What)" 혹은 "얼마나(How much)"를 나타낸다. 현재 대부분의 통신은 Ethernet II 기반이므로 이 필드는 **Ethertype (이더타입)**이라고 널리 불린다.
- **필요성**: 컴퓨터의 네트워크 인터페이스(NIC)는 케이블에서 0과 1의 전기 신호를 받아 조립한 뒤, 껍데기(MAC 헤더)를 벗기고 알맹이(페이로드)를 CPU(운영체제)로 올려보내야 한다. 그런데 알맹이가 IP 주소 기반의 데이터인지, 아니면 MAC 주소를 묻는 ARP 요청인지 구별할 수 없다면 운영체제는 이 데이터를 처리할 수 없다. Ethertype은 운영체제에게 "이건 IP 관련 부서로 보내라"라고 지시하는 라우팅 태그 역할을 한다.

- **💡 비유**: Ethertype은 택배 박스 겉면에 붙어있는 **"취급 주의 / 내용물 표시 스티커"**와 같습니다. 박스에 '신선 식품(`0x0800` IPv4)' 스티커가 붙어 있으면 우편집중국(운영체제)은 이를 냉장 창고(IP 처리 모듈)로 보내고, '가전제품(`0x0806` ARP)' 스티커가 붙어 있으면 일반 창고(ARP 처리 모듈)로 분류해 줍니다.

> 📢 **섹션 요약 비유:** Ethertype은 병원의 **"접수창구 진료과목 분류표"**입니다. 환자(데이터)가 도착하면 안내 데스크(MAC 계층)가 내과(IPv4)로 보낼지, 외과(ARP)로 보낼지 분류해 주는 완벽한 교통정리 시스템입니다.

## Ⅱ. Ethertype의 주요 식별 코드와 다중화 원리 (Deep Dive)

### 1. 상위 프로토콜 다중화 (Demultiplexing)
데이터 링크(L2) 계층에서 네트워크(L3) 계층으로 데이터를 올려보낼 때, Ethertype은 **역다중화(Demultiplexing)**의 스위치 역할을 한다. 하위 계층(이더넷 하나)이 상위 계층(IP, ARP, IPv6 등 여러 개)을 지원할 수 있게 해 주는 아키텍처의 핵심이다.

```text
 ┌─────────────────────────────────────────────────────────────┐
 │                 Ethertype에 의한 상위 계층 역다중화            │
 ├─────────────────────────────────────────────────────────────┤
 │                                                             │
 │   OSI 3계층 (네트워크)        [ IPv4 모듈 ]  [ ARP 모듈 ]  [ IPv6 모듈 ] │
 │                                   ▲            ▲            ▲      │
 │                                   │            │            │      │
 │                            0x0800 │     0x0806 │     0x86DD │      │
 │                                   │            │            │      │
 │   OSI 2계층 (데이터 링크)      [    MAC 계층 (Ethertype 확인)    ] │
 │                                   ▲                                │
 │                                   │                                │
 │   OSI 1계층 (물리)             [ 이더넷 케이블 (010101...) ]         │
 │                                                             │
 └─────────────────────────────────────────────────────────────┘
```

### 2. 시험 및 실무 필수 Ethertype 16진수 코드
이더타입은 IEEE에서 표준으로 관리하며, 실무에서 패킷을 덤프 뜨거나 방화벽(ACL) 룰을 짤 때 이 16진수 코드를 직접 다뤄야 할 때가 많다.

| Ethertype (Hex) | 설명 (프로토콜) | 비고 |
|:---|:---|:---|
| **`0x0800`** | **IPv4** (Internet Protocol version 4) | 현재 인터넷 트래픽의 대다수 |
| **`0x0806`** | **ARP** (Address Resolution Protocol) | IP로 MAC을 찾기 위한 필수 프로토콜 |
| **`0x86DD`** | **IPv6** (Internet Protocol version 6) | 차세대 IP 주소 체계 |
| **`0x8100`** | **VLAN Tag** (IEEE 802.1Q) | 스위치 포트 가상화 (VLAN) 태그 삽입 시 |

### 3. Length 필드로서의 동작 (IEEE 802.3의 경우)
만약 2바이트 값이 `0x05DC` (10진수로 1500) 이하라면, 이는 내용물이 무엇인지 알려주는 Type이 아니라 페이로드의 크기(바이트)를 의미하는 Length 필드로 동작한다. 이 경우 "내용물이 무엇인가?"에 대한 정보는 페이로드 내부에 숨어 있는 802.2 LLC(Logical Link Control) 헤더의 DSAP/SSAP 필드를 열어봐야만 알 수 있다. (현대에는 Spanning Tree Protocol 등 구형 네트워킹 제어 메시지에서나 드물게 볼 수 있다.)

> 📢 **섹션 요약 비유:** Type이냐 Length냐를 숫자의 크기(1500 기준)로 구분하는 것은, **"키가 150cm 이하이면 초등학생(Length 룰 적용), 153cm 이상이면 성인(Type 룰 적용)"**이라고 자동 분류 게이트를 만들어 둔 것과 같은 극강의 호환성 꼼수입니다.
