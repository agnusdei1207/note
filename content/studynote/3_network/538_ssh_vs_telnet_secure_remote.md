# 538. SSH (Secure Shell) 포트 22 / Telnet (원격 접속) 포트 23 비교

> **핵심 인사이트**: 원격지 서버에 접속해서 명령을 내릴 때, 텔넷(Telnet)은 엽서에 비밀번호를 적어 보내는 것이고, SSH는 튼튼한 금고에 비밀번호를 넣고 자물쇠를 채워 보내는 것이다. 오늘날 보안 문제로 Telnet은 멸종되었고 SSH가 지배하고 있다.

## Ⅰ. Telnet (텔넷) - TCP 23
- **개념**: 네트워크를 통해 원격 컴퓨터에 접속하여 터미널 명령을 실행할 수 있게 해주는 가장 오래된 표준 원격 접속 프로토콜입니다.
- **치명적 한계**: 모든 데이터를 **평문(Cleartext)**으로 전송합니다. 따라서 네트워크 중간에서 Wireshark 같은 패킷 스니핑 툴을 사용하면 관리자의 아이디, 비밀번호, 실행한 명령어 내역이 그대로 노출됩니다.
- **현황**: 심각한 보안 취약점 때문에 실무(인터넷 구간)에서는 절대 사용하지 않으며, 외부와 완전히 단절된 폐쇄망 내의 구형 라우터 설정 등 극히 제한적인 용도로만 남아있습니다.

## Ⅱ. SSH (Secure Shell) - TCP 22
- **개념**: Telnet의 평문 전송 취약점을 해결하기 위해, **강력한 암호화(Encryption)**를 적용하여 안전한 원격 접속 터널을 제공하는 프로토콜입니다.
- **보안 원리**:
  1. **비대칭키(공개키/개인키)**: 접속 초기, 서버가 올바른 서버인지 인증하고, 세션 키를 안전하게 교환하는 데 사용합니다. (예: RSA, ECDSA)
  2. **대칭키**: 교환된 세션 키를 이용해 이후 발생하는 모든 터미널 명령어와 응답 데이터를 빠르게 암호화/복호화합니다. (예: AES)
  3. **해시 함수**: 전송 중 데이터가 위변조되지 않았음을 무결성으로 검증합니다. (예: SHA-256)
- **인증 방식**: 전통적인 패스워드 입력 방식 외에도, 클라이언트의 공개키를 서버에 미리 등록해 두는 **Key-based Authentication(키 기반 인증)**을 지원합니다. 이를 통해 비밀번호 입력 없이도 안전하게 자동화 스크립트(Ansible 등)를 실행할 수 있습니다.

## Ⅲ. SSH의 확장 기능 (터미널 밖의 활용)
SSH는 단순한 원격 접속(CLI)을 넘어서, 만들어진 안전한 암호화 터널을 다른 용도로도 빌려줍니다.
1. **SFTP (SSH File Transfer Protocol) / SCP**: SSH 암호화 채널을 그대로 활용하여 파일을 안전하게 전송합니다. 불안전한 FTP(TCP 21)를 대체합니다.
2. **SSH Port Forwarding (Tunneling)**: SSH 터널 안에 다른 암호화되지 않은 프로토콜(예: HTTP, VNC, DB 접속) 패킷을 감싸서 안전하게 통과시키는 기술입니다. 원격지 내부망에 우회 접속하거나 방화벽을 통과할 때 유용하게 쓰입니다.

## Ⅳ. 요약 비교표

| 구분 | Telnet | SSH (Secure Shell) |
| :--- | :--- | :--- |
| **사용 포트** | TCP 23 | **TCP 22** |
| **데이터 전송 방식** | **평문 (Cleartext)** | **암호화 (Encrypted)** |
| **보안성** | 매우 취약 (스니핑 노출) | 매우 강력 |
| **주요 용도** | 폐쇄망 구형 장비 접속 테스트 | 리눅스 서버 원격 관리, 클라우드 인스턴스 접속 |

> 📢 **섹션 요약 비유**: 외국에 있는 별장에 들어갈 때, Telnet은 "비밀번호는 1234야"라고 투명한 유리병에 쪽지를 넣어 바다에 띄우는 것이고, SSH는 그 쪽지를 강철 금고에 넣고 나와 별장만 열 수 있는 특수 자물쇠를 채워 택배로 보내는 것입니다. 중간에 해커가 패킷을 가로채도 SSH 금고는 절대 열어볼 수 없습니다.
