# 75. 시분할 다중화 (TDM, Time Division Multiplexing)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시분할 다중화(TDM)는 하나의 고속 전송 선로를 물리적으로 쪼개는 대신, 시간을 아주 짧은 조각(Time Slot)으로 썰어서 **여러 사용자가 찰나의 순간마다 교대로 고속 선로를 독점하여 사용하는 디지털 다중화 기법**이다.
> 2. **해결점 (FDM 한계 극복)**: 아날로그 FDM이 빈 공간(가드 밴드)을 비워두느라 대역폭을 20%나 낭비했던 것과 달리, TDM은 시간 단위로 딱딱 끊어서 100% 디지털 펄스로 쏘기 때문에 낭비 없이 대역폭 효율을 극대화했다.
> 3. **실무 융합**: TDM은 우리가 사용하는 구리선(전화망 E1/T1 라인)과 현대 디지털 인프라의 모든 뼈대를 이루며, 나아가 이더넷(Ethernet)과 스위치 허브가 데이터를 패킷 단위로 쏘고받는 근본적인 물리적 철학이 되었다.

---

## Ⅰ. TDM의 개요 및 필요성 (Context & Necessity)

- **개념**: 
  - 1개의 굵고 빠른 통신 파이프(Link)를 여러 개의 느린 채널들이 나누어 쓸 때, 각 채널에 고유한 '주파수'를 주는 대신 고유한 **'시간표(Time Slot)'**를 배정하는 방식.
  - 다중화기(MUX)가 입력된 데이터들을 스위치를 돌려가며 1조각씩 뽑아 하나의 줄(프레임)로 합쳐 고속 발사한다.
- **필요성**: 
  - 과거 FDM(주파수 쪼개기)은 치맛자락이 겹치는 간섭 현상 때문에 가드 밴드(Guard Band)라는 빈 공간을 무지막지하게 비워둬야 해서 돈 낭비가 심했다.
  - 엔지니어들은 데이터를 PCM(펄스 부호 변조)을 통해 디지털(0과 1)로 바꾸는 데 성공하자, **"어차피 0과 1의 작은 덩어리인데, 굳이 주파수를 쪼갤 필요 없이 고속도로에 A 데이터 1초, B 데이터 1초, C 데이터 1초씩 차례대로 쏴버리면 간섭도 없고 깔끔하잖아!"**라는 진리를 깨달았다.

- **💡 비유**: **TDM은 '놀이공원 회전목마 탑승'**과 같다.
  - **FDM (기존)**: 놀이공원에 바이킹, 롤러코스터, 범퍼카 기둥을 아예 따로 세워서 사람들이 각각 논다 (땅값이 엄청 듦).
  - **TDM (시간 쪼개기)**: 놀이기구는 거대한 회전목마 단 1개(고속 링크)뿐이다. 대신 의자가 30개 있다. 1초 동안 A팀 30명이 우르르 타서 휙 돌고 내린다. 그다음 1초는 B팀 30명이 타고 돈다. 엄청나게 빠르게 타고 내리기를 반복하면, 각 팀은 마치 자기들 전용 기구를 타는 것처럼 착각하게 된다.

## Ⅱ. 핵심 아키텍처 및 메커니즘 (Deep Dive)

### 1. 라운드 로빈(Round-Robin) 스위칭 구조
TDM 장비(MUX) 앞에는 미친 듯이 빠르게 돌아가는 회전 스위치(Commutator)가 달려 있다.
- **MUX (송신)**: 1번 채널 선에서 1바이트 쏙 빼서 올리고, 2번에서 1바이트 쏙 빼고, 3번에서 1바이트 빼서 기차(Frame)를 하나 만든 뒤 고속도로에 던진다. 이걸 1초에 8,000번 반복한다.
- **DeMUX (수신)**: 기차가 도착하면 수신기 스위치가 똑같은 박자로 돌면서 1번 칸은 1번 PC에, 2번 칸은 2번 PC에 탁탁 떨어뜨려 준다. 송수신기의 스위치가 0.001초도 안 틀리고 완벽히 박자가 맞아야 하는 **동기화(Synchronization)**가 TDM의 생명이다.

### 2. TDM의 타임 슬롯(Time Slot) 프레임 시각화

```text
  ┌────────────────────────────────────────────────────────────────────────┐
  │                   [TDM (시분할 다중화) 프레임 직렬화 구조]                  │
  ├────────────────────────────────────────────────────────────────────────┤
  │                                                                        │
  │ [저속 입력 채널들]                                                         │
  │ 채널 A ── A1 A2 A3 ─▶ ┌─────────┐                                       │
  │ 채널 B ── B1 B2 B3 ─▶ │ 다중화기│     [초고속 TDM 링크 (파이프)]            │
  │ 채널 C ── C1 C2 C3 ─▶ │  (MUX)  │ ━━▶ [A1|B1|C1] [A2|B2|C2] [A3|B3|C3] ─▶│
  │                       └─────────┘     (프레임 1)  (프레임 2)   (프레임 3)  │
  │                                                                        │
  │ * [핵심 메커니즘]:                                                       │
  │   - 각 채널의 데이터가 자기 차례(Time Slot)에 한 칸씩 올라탄다.               │
  │   - 프레임 1개가 통째로 날아가며, 수신기는 "첫 번째는 무조건 A꺼"라는 규칙을 안다.│
  └────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** TDM 파이프는 주파수 낭비(가드 밴드)가 거의 없다. 대신 A1 짐이 A 트럭에 제대로 실리기 위해, A1 짐 뒤에 빈 시간(가드 인터벌)을 아주 살짝 주어 데이터가 엉키는 것을 막는다. 시간이라는 도메인은 아날로그 주파수 롤오프처럼 비스듬히 깎이지 않고 디지털 스위치로 칼같이 끊을 수 있기 때문에 효율이 압도적으로 높다.

## Ⅲ. 실무 융합 및 트러블슈팅

### 1. 실무 표준 인프라: E1 / T1 회선 (디지털 전화망의 뼈대)
TDM의 가장 완벽하고 위대한 실무 적용 사례는 전 세계 땅 밑에 깔린 기업용 전용선(T-Carrier / E-Carrier)이다.
- **E1 라인 (유럽/한국 표준)**: 2.048Mbps 고속 케이블 1가닥.
  - PCM 음성 데이터 1개는 64kbps 속도를 가진다.
  - TDM 스위치가 **32개의 타임 슬롯**을 만든다. (30개는 고객 통화용, 1개는 동기화용, 1개는 제어/시그널링용).
  - 즉, 선 1가닥으로 30명의 직원이 동시에 밖으로 전화를 걸 수 있게 시간을 쪼개는 마법을 부린 것이다.

### 실무 시나리오

1. **시나리오 — 구형 TDM 전용선(T1/E1)의 동기화 실패 (Slip 에러)**:
   A 지사와 B 지사를 E1 라인으로 연결했는데, 주기적으로 라우터 콘솔에 `Slip Error`나 `Framing Error` 로그가 미친 듯이 올라오면서 전화가 뚝뚝 끊긴다.
   **[해결책]** **TDM 스위치의 박자(Clock)가 어긋난 대참사**다. 송신기가 1번, 2번, 3번 차례대로 데이터를 던졌는데, 수신기 시계가 0.0001초 빠르거나 느려서 2번 데이터를 1번 통화자 전화기로 쑤셔 넣어버린 것이다 (프레임 붕괴). TDM은 이 박자 맞추기가 생명이다. 양쪽 라우터 세팅에 들어가서 한쪽을 **마스터 클럭(Master, 통신사 회선 클럭 수신)**으로 잡고, 반대쪽 라우터를 무조건 **슬레이브(Slave, Line Clock)**로 복종하게 만들어 두 장비의 톱니바퀴 박자를 마이크로초 단위로 완벽하게 일치시켜야 에러가 멈춘다.

2. **시나리오 — TDM 기반 구형 네트워크망의 고질적 대역폭 낭비**:
   야간에 콜센터 직원들이 다 퇴근하고 서버실에서 DB 백업용으로 E1 라인을 쓰려고 한다. 그런데 E1 전체 대역폭 2Mbps 중, 음성에 할당된 타임 슬롯 30개는 텅텅 비어있음에도 불구하고 DB 백업 서버는 자신에게 할당된 딱 1개의 타임 슬롯(64kbps) 속도밖에 쓰지 못해서 백업에 밤새워 시간이 걸린다.
   **[해결책]** 이것이 바로 **동기식 TDM(Synchronous TDM)의 치명적 안티패턴**이다. 동기식 TDM은 방에 사람이 있든 없든 무조건 예약된 좌석(Time Slot)을 비워둔 채 버스를 출발시킨다. 유연성이 제로다. 이 한계를 부수기 위해 탄생한 것이 "데이터가 있는 놈만 짐칸에 먼저 쑤셔 넣어 보낸다"는 개념의 **통계적 TDM (Statistical TDM, 비동기식)**이며, 이 철학이 진화하여 오늘날 우리가 쓰는 패킷 스위칭(인터넷 IP망)으로 완전 교체되는 계기가 되었다.

## Ⅳ. 시스템 비교 및 TDM의 우수성

| 구분 | FDM (주파수 분할) | TDM (시분할) |
|:---|:---|:---|
| **신호 종류** | 주로 아날로그 (아날로그 필터 필요) | 완벽한 디지털 (0과 1 펄스) |
| **자원 분할** | 주파수를 쪼갬 (차선 나누기) | 시간을 쪼갬 (교대로 도로 독점) |
| **낭비 요소** | 가드 밴드 (Guard Band) - 주파수 낭비 | 가드 인터벌 (빈 타임 슬롯) - 시간 낭비 |
| **회로 복잡도** | 변/복조기 수십 개 필요 (부피 크고 비쌈) | 디지털 스위치(MUX) 1개면 됨 (싸고 작음) |
| **미래 진화** | OFDM으로 부활 (5G, Wi-Fi) | 통계적 TDM $\rightarrow$ 패킷 교환망(인터넷)으로 진화 |

## Ⅴ. 결론

시분할 다중화(TDM)는 아날로그 시대의 비효율적이고 거대한 통신 장비들을, 작고 차가운 디지털 반도체 칩셋(스위치) 하나로 욱여넣은 디지털 혁명의 상징이다. 비록 자리가 비어있어도 무조건 출발해야 한다는 '동기식 TDM'의 비효율성 때문에 현대의 유연한 인터넷 패킷망(비동기식)에 자리를 내주었지만, **"모든 데이터를 잘게 쪼개어 번갈아 가며 한 줄로 쏘자"**라는 TDM의 철학은 여전히 컴퓨터 내부의 CPU 버스, 메모리 입출력 등 IT 인프라 최하단 물리 계층의 가장 강력한 지배 원리로 남아있다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **동기화 (Synchronization)** | TDM 송신기와 수신기가 동일한 박자로 스위치를 돌리기 위해 맞추는 시계(Clock). 이게 틀어지면 A의 데이터가 B에게 배달되는 대형 사고가 터진다. |
| **프레임 (Frame)** | TDM 다중화기(MUX)가 각 채널에서 뽑아낸 짐들을 차곡차곡 한 줄로 엮어서 포장한 하나의 기차 단위. 이 기차가 1초에 수천 번씩 날아간다. |
| **PCM (펄스 부호 변조)** | TDM이 실어 나르는 내용물. 아날로그 음성을 0과 1로 만든 PCM 데이터야말로 TDM 스위치가 가장 처리하기 좋아하는 완벽한 디지털 블록이다. |
| **통계적 TDM (비동기식 TDM)** | 빈자리(비어있는 타임 슬롯)를 그대로 날려 보내는 바보 같은 짓을 멈추고, 짐이 있는 애들만 먼저 태워서 효율을 100% 쥐어짜는 진화된 영리한 TDM. |

## 👶 어린이를 위한 3줄 비유 설명
1. 장난감 기찻길이 딱 하나밖에 없는데, 3명의 아이가 서로 자기가 만든 기차를 굴리고 싶어서 싸우고 있어요. (선로 부족)
2. 그래서 아빠(MUX)가 규칙을 정했어요. "기찻길을 반으로 쪼갤 순 없으니, **1번 아이 1초, 2번 아이 1초, 3번 아이 1초씩 번갈아 가면서 기차를 올리자!**"
3. 이렇게 시간을 아주 잘게 쪼개서 공평하게 기찻길(네트워크) 하나를 나눠 쓰는 똑똑한 디지털 방법이 바로 **TDM(시분할 다중화)**이랍니다!
