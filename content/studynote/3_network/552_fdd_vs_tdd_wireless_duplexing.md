# 552. 주파수 분할 방식(FDD) vs 시분할 방식(TDD) 무선 환경 적용

> **핵심 인사이트**: 무선 통신에서 내가 말하는 것(업로드)과 듣는 것(다운로드)이 동시에 일어나는 도로(차선)를 어떻게 나눌 것인가? FDD는 '상행선과 하행선 도로를 아예 따로 까는 것'이고, TDD는 '도로 하나만 깐 다음, 1초는 상행선, 다음 1초는 하행선으로 번갈아 쓰는 것'이다.

## Ⅰ. 무선 통신에서의 이중화(Duplexing) 개념
통신은 송신(Uplink)과 수신(Downlink) 양방향으로 이루어져야 합니다. 무선 환경에서는 같은 공기라는 매질을 통과하므로, 이 두 신호가 부딪히지 않도록 나누는 방식이 필요합니다.

## Ⅱ. FDD (Frequency Division Duplexing, 주파수 분할 이중화)
- **개념**: 송신용 주파수 대역과 수신용 주파수 대역을 **물리적으로 완전히 분리**하여 동시에 데이터를 주고받는 방식입니다. (예: 업로드는 800MHz, 다운로드는 900MHz 사용)
- **특징**:
  - 두 대역이 간섭하지 않도록 중간에 완충 지대인 **보호 대역(Guard Band)**을 두어야 하므로 주파수 자원 낭비가 큽니다.
  - 데이터 송수신이 '끊김 없이 동시에' 이루어지므로 실시간 음성 통화(VoLTE)에 매우 안정적입니다.
  - 기존 3G, 4G LTE 망의 전 세계적인 주력 방식입니다.

## Ⅲ. TDD (Time Division Duplexing, 시분할 이중화)
- **개념**: **동일한 주파수 대역 1개**를 사용하되, 시간을 아주 잘게(밀리초 단위) 쪼개어 "지금은 업로드 시간", "지금은 다운로드 시간"으로 **번갈아가며** 사용하는 방식입니다.
- **특징**:
  - 업/다운로드 전환 시 전파 충돌을 막기 위한 **보호 시간(Guard Period)**이 필요합니다.
  - **비대칭 트래픽에 강함 🌟**: 현대 인터넷은 다운로드(유튜브 시청 등)가 업로드보다 훨씬 많습니다. TDD는 전체 10초 중 다운로드에 8초, 업로드에 2초를 할당하는 식으로 자원 비율을 유연하게 조절할 수 있어 효율적입니다.
  - 주파수가 하나만 있어도 서비스가 가능하므로, 5G 시대 광대역 주파수 확보에 유리하여 최근 전 세계 5G의 주력 방식(예: 3.5GHz TDD)으로 채택되었습니다.

## Ⅳ. FDD vs TDD 요약 비교

| 구분 | FDD (주파수 분할 이중화) | TDD (시분할 이중화) |
| :--- | :--- | :--- |
| **자원 분할 방식** | 주파수(도로)를 2개로 분할 | 시간(신호등)을 분할하여 번갈아 사용 |
| **필수 보호 요소** | Guard **Band** (주파수 완충 대역) | Guard **Period** (시간 완충 구간) |
| **트래픽 비대칭성** | 유연성 낮음 (상/하행 대역 고정) | **매우 유연함 (다운로드 몰아주기 가능)** |
| **주 사용처** | 전통적인 음성 통화, LTE 기본망 | **비대칭 데이터 통신, 5G 주력 (광대역)** |
| **장점** | 커버리지가 넓고 실시간 지연이 적음 | 단일 주파수 대역으로 구현 가능, 자원 효율 극대화 |

> 📢 **섹션 요약 비유**: FDD는 강남대로에 상행선 4차로, 하행선 4차로를 콘크리트 중앙분리대(Guard Band)로 완벽히 분리해 놓은 것입니다. 출근 시간에 상행선만 막혀도 하행선 차로를 빌려 쓸 수 없어 비효율적입니다. 반면 TDD는 1차선짜리 외길 터널 양 끝에 신호등을 달아놓고, 출근 시간에는 상행선 파란불을 40초, 퇴근 시간에는 하행선 파란불을 40초 켜주는(유연한 비대칭 할당) 스마트한 가변차로 터널입니다.
