# 556. 핸드오버 (Handover) / 핸드오프 (Handoff) 종류 개념

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 핸드오버 (Handover)는 이동통신 가입자가 통화나 데이터 세션을 유지한 채 한 기지국 (Cell) 영역에서 다른 기지국 영역으로 이동할 때, 서비스 끊김 없이 연결을 자동으로 전환해 주는 핵심 이동성 관리 (Mobility Management) 기술이다.
> 2. **가치**: 모바일 환경에서 '이동 중 통신(Cellular 개념의 핵심)'을 가능하게 하는 결정적 요소이며, 핸드오버 실패율(Call Drop Rate)은 통신 품질(QoS)과 사용자 체감 품질(QoE)을 결정짓는 가장 중요한 지표다.
> 3. **융합**: CDMA 방식의 소프트 핸드오버(Soft Handover), LTE의 하드 핸드오버(Hard Handover)를 거쳐 5G/6G 환경에서는 이종망(Wi-Fi, 위성 등) 간의 수직적 핸드오버(Vertical Handover)와 엣지 컴퓨팅(MEC) 기반의 빠른 세션 전환으로 기술이 진화하고 있다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: 이동통신망은 제한된 주파수를 효율적으로 쓰기 위해 서비스 지역을 작은 벌집 모양의 셀(Cell)로 나누어 각 셀마다 기지국을 설치한다. 단말기(UE)가 이동하여 현재 접속된 '서빙 기지국(Serving Cell)'의 신호가 약해지고 '대상 기지국(Target Cell)'의 신호가 강해질 때, 통신 채널을 새로운 기지국으로 부드럽게 넘겨주는 과정이 핸드오버(Handover)다. (미국 방식에서는 주로 Handoff라 부른다.)

- **필요성**: 만약 핸드오버 기능이 없다면, 사용자가 차를 타고 옆 동네로 넘어갈 때마다 전화가 끊어지고 새로 걸어야 한다. 기지국 커버리지가 수백 미터에서 수 킬로미터에 불과한 현대 이동통신에서 고속 이동 중 끊김 없는 연결을 유지하려면 1초 이내에 기지국을 갈아타는 정교한 통제 메커니즘이 필수적이다.

- **💡 비유**: 핸드오버는 올림픽 육상 '계주(릴레이)'와 같다. 앞 주자(현재 기지국)가 뒷 주자(다음 기지국)에게 바톤(통신 연결)을 넘겨주는 짧은 구간에서, 바톤을 떨어뜨리지 않고 달리기의 속도를 유지하는 것이 기술의 핵심이다.

- **등장 배경 및 발전 과정**:
  1. **초기 아날로그 통신 (1G)**: 기지국이 단말기의 신호 세기를 측정하여 중앙 교환기(MSC)가 일방적으로 핸드오버를 결정했다. 처리 시간이 길어 통화 끊김(Call Drop)이 잦았다.
  2. **디지털 통신 (2G/3G)**: 단말기가 직접 주변 기지국 신호를 측정해 보고하는 MAHO (Mobile Assisted Handoff) 방식이 도입되었고, CDMA 기술과 함께 끊기지 않는 '소프트 핸드오버(Soft Handoff)'가 대세가 되었다.
  3. **초고속 패킷 통신 (4G/5G)**: IP 패킷 기반 망(LTE/5G)으로 전환되면서, 다시 '하드 핸드오버(Hard Handoff)'로 회귀하되 그 절체 시간(Interruption Time)을 밀리초 단위로 줄이는 X2/Xn 인터페이스 기반의 고속 핸드오버 방식으로 발전했다.

- **📢 섹션 요약 비유**: 라디오 채널을 들으며 서울에서 부산으로 갈 때, 지역마다 주파수가 달라도 라디오가 스스로 알아서 잡음 없이 다음 지역 주파수로 맞춰주는 자동 튜닝 기능과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 핸드오버의 3단계 프로세스

핸드오버는 기지국(BS)과 단말(UE)의 긴밀한 협력 하에 3단계로 진행된다. 현대 이동통신은 단말기가 주변을 측정하고 기지국이 결정하는 **MAHO (Mobile Assisted Handoff)** 방식을 표준으로 사용한다.

| 단계 | 명칭 | 주체 | 동작 내용 |
|:---|:---|:---|:---|
| 1 | **측정 및 보고 (Measurement)** | 단말 (UE) | 현재 서빙 셀과 주변 셀의 신호 강도(RSRP) 및 품질(RSRQ)을 주기적으로 측정하여 임계치 초과 시 기지국에 Measurement Report 전송 |
| 2 | **결정 (Decision)** | 서빙 기지국 | 단말의 보고서를 분석하여 핸드오버 여부 판단, 타겟 기지국과 핸드오버 준비 (자원 할당, 컨텍스트 전달) |
| 3 | **실행 (Execution)** | 단말, 기지국 | 단말에게 타겟 기지국으로 접속하라는 명령(Handover Command) 하달, 단말은 타겟 셀과 동기화 후 연결 확립 (Random Access) |

---

### 핸드오버 트리거링 원리 (히스테리시스 마진)

핸드오버 시점을 결정하는 것은 신호 세기의 역전이다. 그러나 두 기지국 사이의 경계 지역(Cell Edge)에서는 신호 세기가 엎치락뒤치락하며 핸드오버가 불필요하게 반복되는 **핑퐁 효과(Ping-Pong Effect)**가 발생할 수 있다. 이를 방지하기 위해 '히스테리시스(Hysteresis) 마진'과 'Time-to-Trigger(TTT)' 개념을 적용한다.

```text
  ┌───────────────────────────────────────────────────────────────┐
  │         히스테리시스 마진을 적용한 핸드오버 트리거링             │
  ├───────────────────────────────────────────────────────────────┤
  │                                                               │
  │ 신호 강도                                                     │
  │   ▲                                                           │
  │   │  서빙 기지국(A) 신호           타겟 기지국(B) 신호         │
  │   │  ───────.                       .───────              │
  │   │          . ↘                   ↗ .                     │
  │   │           .  ↘               ↗  .                      │
  │   │            .   ↘           ↗   .                       │
  │   │             .    ↘       ↗    .                        │
  │   │              .     ↘   ↗     .         ↑                │
  │   │               .      ╳       .          │ Hysteresis      │
  │   │                .   ↗   ↘   .            │ Margin          │
  │   │                 . ↗      ↘.             ↓                │
  │   │                  *─────────*──────────────                │
  │   │   (신호 역전) ───┘         └─▶ 핸드오버 발생              │
  │   │                                                           │
  │   └──────────────────────────────────────────────▶ 이동 거리  │
  │                                                               │
  │  조건: [타겟 기지국 신호] > [서빙 기지국 신호] + [Hysteresis]     │
  │        그리고 이 상태가 [Time-to-Trigger (TTT)] 이상 유지될 때   │
  └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 단말기가 기지국 A에서 B로 이동할 때, 신호 역전이 일어나는 교차점(`╳`)에서 즉시 핸드오버를 수행하면 신호의 미세한 흔들림(Fading)으로 인해 A와 B를 반복해서 오가는 핑퐁 현상이 발생한다. 이를 막기 위해 타겟 B의 신호가 A보다 단순히 큰 것이 아니라, 설정된 '히스테리시스 마진'만큼 더 커지고 그 상태가 'TTT(Time-to-Trigger)' 시간 동안 유지될 때만 확정적으로 핸드오버를 지시한다. 마진을 너무 크게 잡으면 핸드오버가 늦어져 통화가 끊어지고, 너무 작게 잡으면 핑퐁이 발생해 기지국 부하가 폭증하므로 최적화(SON)가 필수적이다.

---

### 핸드오버의 유형 구분 (분류 체계)

핸드오버는 제어 방식, 통신망 종류, 주파수 대역에 따라 다양하게 분류된다.

| 분류 기준 | 유형 | 설명 |
|:---|:---|:---|
| **물리적 연결 제어** | **Hard Handover** | 기존 연결을 먼저 끊고, 새 연결을 맺음 (Break-before-make) |
| | **Soft Handover** | 새 연결을 먼저 맺고, 기존 연결을 끊음 (Make-before-break) |
| **망간 이동 (수평/수직)** | **Horizontal Handover** | 동일한 통신망 내에서의 이동 (예: LTE 기지국 간 이동) |
| | **Vertical Handover** | 서로 다른 이종망 간의 이동 (예: Wi-Fi망에서 LTE망으로 이동) |
| **주파수 대역** | **Intra-frequency** | 서빙 셀과 타겟 셀이 동일한 주파수를 사용할 때 |
| | **Inter-frequency** | 서빙 셀과 타겟 셀이 서로 다른 주파수 대역을 사용할 때 |
| **코어망 경계** | **Intra-MME/SGW** | 단말이 동일한 코어 장비(MME/SGW) 관할 구역 내에서 기지국만 바뀔 때 |
| | **Inter-MME/SGW** | 이동 범위가 커서 코어 장비(MME나 SGW) 자체가 교체되어야 할 때 |

- **📢 섹션 요약 비유**: 핸드오버는 앞 차(현재 기지국)에서 내리기 전에 뒤 차(다음 기지국)의 문을 먼저 잡고 탈 것인지(소프트), 완전히 내린 후 폴짝 뛸 것인지(하드)를 결정하는 승차 방식의 차이입니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: 수평적 핸드오버 vs 수직적 핸드오버

5G 시대 이후 망의 융합이 가속화되면서 수직적 핸드오버(Vertical Handover)의 중요성이 급부상했다.

| 비교 항목 | 수평적 핸드오버 (Horizontal Handover) | 수직적 핸드오버 (Vertical Handover) |
|:---|:---|:---|
| **이동 환경** | 4G ↔ 4G, 5G ↔ 5G (동종망) | 4G ↔ 5G, Wi-Fi ↔ 5G, 위성 ↔ 지상망 (이종망) |
| **핵심 목적** | 사용자의 물리적 이동에 따른 연결 유지 | 통신 품질, 비용, 트래픽 분산을 위한 지능적 망 선택 |
| **트리거 기준** | 신호 세기 (RSRP), 신호 품질 (RSRQ) | 신호 세기 + 대역폭, 요금, 배터리 잔량, 앱 요구사항 |
| **기술적 난이도** | 표준화되어 있으며 비교적 단순 | MAC 계층 구조가 다르고 IP 주소 변경 이슈로 복잡함 |
| **관련 기술** | X2/Xn 핸드오버 | MPTCP (Multipath TCP), ANDSF, 패스포트(Passport) |

```text
  ┌─────────────────────────────────────────────────────────────┐
  │        수평적 (Horizontal) vs 수직적 (Vertical) 핸드오버      │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │  [수평적 핸드오버]                                            │
  │                                                             │
  │       기지국 1 ◀────────▶ 기지국 2                          │
  │        (5G)     (이동)     (5G)                             │
  │        / \                 / \                              │
  │       /   \               /   \                             │
  │      단말기 ──────────────▶ 단말기                            │
  │      (동종 통신 방식 내에서의 공간 이동)                        │
  │                                                             │
  │  [수직적 핸드오버]                                            │
  │                                                             │
  │       집/카페 ◀────────▶ 길거리                            │
  │       (Wi-Fi)   (이동)    (5G 망)                          │
  │         │                   │                              │
  │         ▼                   ▼                              │
  │      단말기 ──────────────▶ 단말기                            │
  │      (게임 중 Wi-Fi 영역을 벗어나면 5G로 IP 세션 무손실 전환)    │
  └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 수평적 핸드오버는 우리가 흔히 아는 기지국 간 이동이다. 기술적 방식이 동일하므로 제어 메시지와 사용자 데이터의 컨텍스트를 넘겨주기만 하면 된다. 반면 수직적 핸드오버는 전혀 다른 통신 프로토콜(예: IEEE 802.11과 3GPP NR) 간의 전환이다. 이 경우 할당받은 IP 주소가 바뀌어 TCP 세션이 끊어지는 문제가 발생하므로, 전송 계층에서 복수의 경로를 유지하는 MPTCP(Multipath TCP)나, 애플리케이션 계층에서의 세션 복구 메커니즘(QUIC)을 통한 융합이 필수적이다.

### 과목 융합 관점
- **운영체제 (OS) / 네트워크 (NW)**: MPTCP나 QUIC 프로토콜은 전송 계층(L4)에서 IP 주소 변경(수직적 핸드오버)에 면역성을 갖도록 설계되어, 이동통신 하위 계층(L2/L3)의 핸드오버 부담을 크게 줄여준다.
- **인공지능 (AI) / 데이터엔지니어링**: 과거에는 히스테리시스 마진이나 TTT를 엔지니어가 수동으로 고정했지만, 현재는 AI/ML 기반의 자율 최적화 네트워크(SON, Self-Organizing Network)가 단말의 이동 패턴과 트래픽을 학습해 셀 경계마다 최적의 파라미터를 동적으로 자동 설정(MRO, Mobility Robustness Optimization)한다.

- **📢 섹션 요약 비유**: 수평적 핸드오버가 1호선 전철에서 앞칸에서 뒷칸으로 걸어가는 것이라면, 수직적 핸드오버는 환승역에서 1호선에서 내려 KTX로 갈아타는(전혀 다른 시스템) 것과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 고속철도(KTX) 구간에서의 집단 핸드오버 (Handover Storm)**: 시속 300km로 달리는 KTX 내에는 수백 명의 승객이 있다. 열차가 기지국 경계를 통과할 때마다 수백 대의 단말이 동시에 핸드오버를 요청하면, 타겟 기지국의 제어 채널(PRACH)에 심각한 병목이 발생해 대규모 호 드랍(Call Drop)이 발생한다.
   - **아키텍트의 해결책**: 고속철도 전용으로 여러 개의 기지국을 하나의 논리적 셀로 묶어 핸드오버 자체를 발생시키지 않는 **단일 셀(Single Cell) 솔루션**을 적용하거나, 열차 옥상에 외부 안테나(이동형 릴레이)를 달아 열차가 대표로 기지국과 통신하고 승객들은 열차 내 Wi-Fi/펨토셀에 접속하는 백홀 구조를 도입하여 핸드오버 폭풍을 원천 차단해야 한다.

2. **시나리오 — 핑퐁 현상(Ping-Pong)으로 인한 배터리 및 네트워크 코어 부하**: 도심지 빌딩 숲에서 5G 매크로 셀과 스몰 셀의 신호가 난반사되면서 단말기가 1초에도 몇 번씩 핸드오버를 반복하는 상황.
   - **아키텍트의 해결책**: 네트워크 엔지니어는 MRO (Mobility Robustness Optimization) 파라미터를 튜닝해야 한다. 히스테리시스 마진을 높이거나 TTT(Time-To-Trigger)를 증가시켜 잦은 스위칭을 억제하고, 단말의 이동 속도(Doppler Shift 측정)를 인지하여 고속 이동 단말은 반경이 좁은 스몰 셀로의 핸드오버를 금지하고 매크로 셀에 묶어두는 계층적 셀 구조(Hierarchical Cell Structure) 전략을 구사한다.

### 도입 체크리스트
- **기술적**: 코어망 교체를 동반하는 Inter-MME 핸드오버 시, 데이터 패킷 유실을 막기 위해 소스 기지국과 타겟 기지국 간 데이터 포워딩(Data Forwarding) 터널이 정상적으로 확립되었는가?
- **운영·경영적**: 핑퐁 및 핸드오버 실패율(HOFR) 모니터링 시스템을 구축하고, 특정 지역(음영/경계)에서의 KPI 저하 시 자동으로 알람을 발생시키는 폐쇄 루프(Closed-loop) 최적화 체계가 적용되었는가?

### 안티패턴
- **단일 히스테리시스 마진 일괄 적용**: 도심, 고속도로, 실내 환경에 구분 없이 동일한 핸드오버 파라미터 세트를 일괄 적용하는 행위. 환경에 따라 페이딩(Fading) 특성이 다르므로 반드시 지역별 분리 튜닝이나 AI/ML 기반의 자동화 파라미터 적용(SON)을 해야 한다.

- **📢 섹션 요약 비유**: 시속 300km로 달리는 기차 안의 500명이 동시에 옆 칸으로 넘어가려 하면 문이 막히듯(핸드오버 폭풍), 상황과 환경에 맞춰 문을 넓히거나 이동 자체를 안 해도 되도록 묶어주는 설계 지혜가 필요합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | AI 기반 자율 핸드오버 최적화 전 | AI 기반 자율 최적화 (SON MRO) 후 | 개선 효과 |
|:---|:---|:---|:---|
| **정량** | 핸드오버 핑퐁 발생률 5% | 핑퐁 발생률 0.5% 이하로 통제 | 무선 제어 채널 부하 **90% 절감** |
| **정량** | Call Drop Rate (절단율) 1.2% | Call Drop Rate 0.1% 대 진입 | 통화 끊김 **10배 이상 감소** |
| **정성** | 음영/경계 지역 민원 발생 후 수동 조치 | AI가 파라미터를 실시간 조정해 사전 방어 | 운영 비용 (OPEX) 감소 및 사용자 체감 품질(QoE) 극대화 |

### 미래 전망
- **위성-지상망 간 무결절성 수직 핸드오버**: 6G 시대에는 저궤도(LEO) 위성 통신(예: 스타링크)과 지상 5G/6G 기지국이 통합(NTN, Non-Terrestrial Network)된다. 사용자가 해상이나 사막으로 이동할 때 지상망에서 위성망으로 자동으로 수직 핸드오버가 일어나는 심리스(Seamless) 모빌리티가 구현된다.
- **AI/ML 기반 예측형 핸드오버 (Predictive Handover)**: 단말기의 과거 이동 경로, 네비게이션 목적지, 센서 정보(방향, 가속도)를 결합하여 "3초 뒤 터널에 진입할 것"을 AI가 예측하고, 신호가 끊어지기 전에 미리 가장 적합한 셀로 선제적 핸드오버를 수행하는 기술이 연구되고 있다.

### 참고 표준
- **3GPP TS 38.300 / TS 38.331**: 5G NR 환경에서의 핸드오버 절차 및 아키텍처 표준
- **3GPP TS 36.331**: LTE (E-UTRAN) 무선 자원 제어(RRC) 핸드오버 규격

이동통신 기술의 패러다임은 주파수 효율성 극대화를 넘어 **"사용자가 망의 변화를 전혀 눈치채지 못하게 하는 것(Zero Interruption)"**으로 진화하고 있다. 기술사는 단순한 기지국 간 전환을 넘어, MPTCP 기반의 이종망 결합, MEC(엣지 컴퓨팅) 세션의 실시간 마이그레이션, AI 예측 모델을 아우르는 '종합적 이동성 관리(Holistic Mobility Management)' 관점에서 아키텍처를 설계해야 한다.

- **📢 섹션 요약 비유**: 미래의 핸드오버는 운전자가 앞을 보며 차선을 바꿀지 말지 고민하기도 전에, 내비게이션(AI)이 미리 덜 막히는 차선으로 차를 부드럽게 옮겨놓는 자율주행 차선 변경 기술과 같습니다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **소프트/하드 핸드오버** | 핸드오버를 물리적 회선 연결 순서에 따라 구분한 하위 상세 기술 개념. |
| **SON (Self-Organizing Network)** | 핸드오버 파라미터(마진, TTT)를 인간 대신 AI 알고리즘이 자동으로 최적화하여 핑퐁을 막는 핵심 운용 프레임워크. |

| **MPTCP (Multipath TCP)** | L4 전송 계층에서 다중 IP 경로를 동시 지원하여, 수직적 핸드오버 시 L3 패킷 끊김을 앱 계층이 모르게 방어하는 기술. |
| **NTN (Non-Terrestrial Network)** | 지상망 커버리지를 벗어났을 때 위성 기지국으로 수직적 핸드오버를 확장하는 6G 핵심 아키텍처. |
