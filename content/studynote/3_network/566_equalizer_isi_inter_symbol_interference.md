# 566. 등화기 (Equalizer) - ISI 상쇄 역필터

> **핵심 인사이트**: 동굴에서 "안~녕~하~세~요"라고 너무 빨리 말하면, 앞글자 '안'의 메아리가 다음 글자 '녕'을 덮어버려 무슨 말인지 알 수 없게 된다. 이를 ISI(기호 간 간섭)라고 한다. 등화기는 무선 채널이라는 찌그러진 거울의 모양을 수학적으로 예측해, 그와 반대되는 거울(역필터)을 덧대어 원본 신호의 모양을 평평하게(Equal) 펴주는 마법의 필터다.

## Ⅰ. 무선 채널의 심각한 문제: ISI 현상
- 초고속 데이터 통신(LTE, Wi-Fi)을 할 때 데이터를 1, 0, 1, 0으로 매우 촘촘하고 빠르게 쏩니다.
- 하지만 다중경로 페이딩 때문에 앞서 보낸 '1' 신호의 꼬리(메아리)가 지연되어 날아오다가, 바로 뒤이어 쏜 '0' 신호와 겹쳐서 뭉개지는 현상이 발생합니다. 
- 이를 **ISI (Inter-Symbol Interference, 기호 간 간섭)**라고 하며, 고속 데이터 통신의 속도를 올리지 못하게 막는 가장 치명적인 적입니다.

## Ⅱ. 등화기 (Equalizer)의 개념
등화기는 전파가 무선 공간(채널)을 날아오면서 특정 주파수 대역이 깎이거나 찌그러진(왜곡된) 것을 수신단에서 원래 모양대로 평탄하게(Equalize) 펴주는 **신호 처리 필터(상쇄 필터)**입니다. 

## Ⅲ. 등화기의 동작 원리 (역필터링) 🌟
1. **채널 추정 (Channel Estimation)**: 
   - 기지국은 데이터 사이에 미리 약속된 연습용 신호(Training Sequence, 파일럿 신호)를 섞어 보냅니다.
   - 단말기는 수신된 이 연습용 신호가 얼마나 찌그러졌는지를 보고, 현재 나와 기지국 사이의 공기(채널)가 어떤 모양의 요철을 가지고 있는지 수학적 함수(채널 응답 함수 $H(f)$)로 계산해냅니다.
2. **역필터 적용 (Inverse Filtering)**:
   - 채널이 신호를 $H(f)$만큼 왜곡시켰다면, 등화기는 단말기 내부에서 수신된 신호에 그와 정반대인 $1/H(f)$라는 역필터를 곱해버립니다.
   - 마치 오목거울에 반사되어 홀쭉해진 내 얼굴을, 볼록거울로 다시 비춰 정상 얼굴로 펴내는 것과 같은 원리입니다.
   - 이렇게 하면 신호 꼬리가 겹치는 ISI 현상이 깨끗하게 상쇄됩니다.

## Ⅳ. 레이크 수신기와 등화기의 차이
- **레이크 수신기 (CDMA 주력)**: 반사파(메아리)를 '긁어모아서 오히려 에너지를 키우는' 방식. 저속/중속 통신에 유리하지만, 광대역 고속 통신에서는 메아리가 너무 많아져 갈퀴로 다 모을 수 없게 되어 한계가 명확합니다.
- **등화기 (LTE/Wi-Fi 등 광대역 주력)**: 반사파에 의해 발생한 찌그러짐 자체를 수학적 필터로 '상쇄(무효화)'시켜버리는 방식. 고속 통신에서 발생하는 심각한 ISI를 잡는 데 필수적입니다. (현대의 OFDM 기술은 이 등화기를 매우 간단하게 구현할 수 있어 대세가 되었습니다.)

> 📢 **섹션 요약 비유**: 등화기는 고급 오디오의 이퀄라이저 조절기와 같습니다. 지하실에서 음악을 틀었더니 구조 탓에 베이스(저음) 소리가 다 죽어서 찌그려져 들립니다(채널 왜곡). 이때 이퀄라이저 기계를 켜서 의도적으로 베이스(저음) 증폭 레버만 팍 올려버리면(역필터 적용), 원래 스튜디오에서 녹음했던 원본 음악처럼 밸런스가 평탄하게(Equal) 복원되는 원리입니다.
