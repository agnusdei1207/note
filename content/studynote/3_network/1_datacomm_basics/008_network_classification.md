## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크는 지리적 범위에 따라 LAN (Local Area Network, 수 km 이내), MAN (Metropolitan Area Network, 도시 규모, 수십 km), WAN (Wide Area Network, 국가/대륙 규모, 수백 km 이상)로 분류되며, 각 분류는 고유한 기술, 프로토콜, 구축 비용, 관리 주체를 가진다.
> 2. **가치**: 네트워크 범위에 따라 적합한 전송 기술, 프로토콜, 장비 선택이 달라지며, 이를 이해해야 비용 효율적이고 성능 최적화된 네트워크를 설계할 수 있다.
> 3. **융합**: 현대 네트워크에서 LAN, MAN, WAN의 경계는 점점模糊되어 가고 있으며, SD-WAN (Software-Defined WAN) 기술은 WAN을 LAN처럼 유연하게 관리하게 함으로써 전통적인 분류를 재정의하고 있다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: 네트워크 분류는 네트워크가 coverage하는 지리적 범위에 따라 결정된다. LAN은 건물, 캠퍼스, 공장 등 제한된 지역 내의 네트워크로, 단일 조직이 소유하고 관리한다. Ethernet (IEEE 802.3), Wi-Fi (IEEE 802.11) 등이 LAN 기술의 대표적이다. MAN은 도시 규모의 네트워크로, 여러 LAN을 상호接続하여 광역 서비스를 제공한다.城域Ethernet, DQDB (Distributed Queue Dual Bus) 등이 MAN 기술이며, 주로 통신사업자가 제공한다. WAN은 국가, 대륙, 전 세계 규모의 네트워크로, 인터넷이 가장 대표적인 WAN이다.leased line, MPLS (Multiprotocol Label Switching), 위성 통신 등이 WAN 기술이며, 통신사업자(ISP, Telecom Carrier)가 서비스를 제공한다.

- **필요성**: 네트워크를 구축할 때 지리적 범위에 따라 적합한 기술을 선택해야 한다.同一 건물 내에서는 이더넷 스위치와 UTP 케이블로 충분하지만, 도시 之间 통신에는 광섬유 기반의 MAN 기술이 필요하고, 국가 간 통신에는 위성 또는 해저 광섬유 기반의 WAN 기술이 필요하다. 또한, 각 분류는 보안, 관리, 비용 측면에서 다른 특성을 가진다. 예를 들어, LAN은 단일 관리 도메인이므로security 관리가 용이하지만, WAN은 여러 조직과 서비스 プロバイダ를 경유하므로 security 관리의 복잡성이 높다.

- **💡 비유**: 네트워크 범위는 집, 동네, 나라의 관계와 같다. 우리 집 안( LAN)은 내가管理하고 바로修理할 수 있지만, 동네 전체( MAN)를 연결하는 도로는_municipal官청이 관리하며, 나라와 나라 사이( WAN)는中央政府와 국제 기구가管理합니다. 각 범위에 맞는 도로 설계와 유지보수 체계가 필요한 것처럼, 네트워크도 범위에 맞는 기술과 관리 체계가 있다.

- **등장 배경 및 발전 과정**:
  1. **1970년대:LAN의 탄생**: Xerox PARC에서 이더넷(Ethernet)을 개발하여,同一 건물 내 컴퓨터들을低成本으로 상호接続하는 LAN 기술이 등장했다.
  2. **1980년대:MAN과 WAN의 발전**: 도시 규모의 네트워크(MAN)를 연결하는 기술과 함께, ARPANET에서 발전한 TCP/IP 기반의 WAN(인터넷)이 확산되기 시작했다.
  3. **2000년대 이후:경계의模糊**: VPN (Virtual Private Network), SD-WAN, 클라우드 컴퓨팅의 등장으로, 사용자는物理적 네트워크 범위를 느끼지 않고全球 어디서나 사내 네트워크에 접근할 수 있게 되었다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소 및 기술 비교

| 분류 | 범위 | 전형적 속도 | 주요 기술 | 관리 주체 | 비용 |
|:---|:---|:---|:---|:---|:---|

### LAN, MAN, WAN의 계층적 관계

LAN, MAN, WAN은 계층적으로 구성되며, LAN이 WAN의 기본 단위가 된다. 이 구조는 인터넷의 global hierarchical architecture를 형성하며, 각 수준에서 사용되는 장비와 프로토콜이 달라진다.

```text
  ┌─────────────────────────────────────────────────────────────────────┐
  │              LAN-MAN-WAN 계층 구조와 상호관계                          │
  ├─────────────────────────────────────────────────────────────────────┤
  │
  │                         ┌─────────────────────┐                      │
  │                         │   Global Internet   │  ← WAN (전 세계)     │
  │                         │      (WAN)          │                      │
  │                         └──────────┬──────────┘                      │
  │                                    │                                   │
  │                         ┌──────────▼──────────┐                      │
  │                         │  ISP 백본 네트워크   │  ← WAN (사업자)      │
  │                         │     (MAN/WAN)        │                      │
  │                         └──────────┬──────────┘                      │
  │                                    │                                   │
  │           ┌───────────────────────┼───────────────────────┐          │
  │           │                       │                       │          │
  │  ┌────────▼────────┐  ┌──────────▼──────────┐  ┌────────▼───────┐   │
  │  │   기업 A 본사    │  │    기업 A 지사      │  │   기업 A 지사   │   │
  │  │  Campus LAN     │  │    Branch LAN       │  │   Branch LAN   │   │
  │  │  (다수의 LAN)   │  │  (소규모 LAN)      │  │  (소규모 LAN)  │   │
  │  └─────────────────┘  └─────────────────────┘  └────────────────┘   │
  │           │                       │                       │          │
  │           └───────────────────────┼───────────────────────┘          │
  │                                   │                                   │
  │                         WAN 링크 (MPLS / Leased Line)               │
  │
  │  ※ 기업은 WAN 서비스를 통신사업자로부터 구매하여,                      │
  │    여러 지사의 LAN을 상호接続한다                                      │
  │
  └─────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 네트워크의 계층적 구조에서 LAN은 가장 기본적인 단위이며, 각 조직(기업, 학교 등)이 독립적으로構築運用한다. MAN은 이러한 LAN들을 도시 수준으로 aggregated하여, 주로 통신사업자가 제공하는 backbone 네트워크로 연결된다. WAN은さらに 확장되어 국가, 대륙, 글로벌 규모를cover하며, ISP (Internet Service Provider)와通信 Carrier가 backbone을運用한다. 기업은 ISP로부터 WAN 서비스를 구매하여 본사와 지사의 LAN을相互 연결한다. 이러한 계층적 구조는 확장성(Scalability)과 관리 용이성을 제공하며, 각 수준에서 적절한 기술과 프로토콜이 사용된다.

## Ⅲ. 융합 비교 및 다각도 분석

### LAN vs MAN vs WAN 핵심 비교

| 비교 항목 | LAN | MAN | WAN |
|:---|:---|:---|:---|
| **전파 지연** | 매우 낮음 (µs~ms) | 낮음 (ms) | 높음 (ms~수초) |
| **대역폭** | 높음 (100Mbps~100Gbps) | 중간~높음 | 낮음~중간 (서비스 수준에 따라) |
| **비용 구조** | 초기 투자 높음, 운용 낮음 | 중간 | 월정액 (bandwidth 기반) |
| **보안** | 관리 용이 (단일 도메인) | 중간 (사업자 관여) | 주의 필요 (多事業자 경유) |
| **전송 매체** | 구리 케이블, 광섬유, 무선 | 광섬유 (주로) | 광섬유, 위성, 무선 |
| **대표 프로토콜** | Ethernet, Wi-Fi |，城域Ethernet, FDDI | IP, MPLS, PPP, HDLC |

### 과목 융합 관점

- **클라우드 컴퓨팅**: AWS, Azure, GCP 등의 공인 클라우드는全球에 분산된 데이터 센터를WAN로 연결하여 사용자에게 제공하며, 사용자의 온프레미스 LAN과 클라우드 사이의 연결(Direct Connect, ExpressRoute)도 WAN 기술로 구성된다.
- **사물인터넷 (IoT)**: IoT 디바이스는 근처의 게이트웨이(LAN 기술: Wi-Fi, Zigbee, BLE)로 연결되고, 게이트웨리는蜂窝網(WAN: 4G/5G) 또는 Wi-Fi (LAN/WAN 경계)를 통해 클라우드에 연결된다.

- **📢 섹션 요약 비유:** LAN, MAN, WAN의 관계는 집, 동네, 나라의 도로망과 같습니다. 집 안 복도(LAN)는 맘대로 사용할 수 있고, 동네 길(MAN)은_municipal가 관리하며, 고속도로(WAN)는정부와 통신사업자가管理하여 모두에게 共用됩니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 글로벌 기업 WAN 설계**: 서울 본사와 도쿄, 미국 지사 간 실시간 VoIP와 파일 전송이 가능해야 한다. WAN 기술로 MPLS (Multiprotocol Label Switching)를 선택하여, 통신사업자가 제공하는 SLA (Service Level Agreement)로 음성 품질과 가용률을 보장받는다. 각 지점은 WAN 라우터로 본사 VPN 게이트웨이에 연결되어, 안전한 터널을 통해 통신한다.

2. **시나리오 — SD-WAN으로 전통적 WAN 대체**: 전통적 MPLS WAN의 높은 비용과缺乏弹性를 해결하기 위해, SD-WAN (Software-Defined WAN)을 도입한다. SD-WAN은 MPLS뿐만 아니라 일반 인터넷(DSL, Cable, 4G/LTE)을 통합하여 사용하며, 애플리케이션별 최적 경로를 동적으로 선택한다. 이를 통해 WAN 비용을 50% 이상 절감하면서도 애플리케이션 성능을 개선한 사례가 많다.

```text
  ┌─────────────────────────────────────────────────────────────────────┐
  │              전통적 WAN vs SD-WAN 아키텍처 비교                       │
  ├─────────────────────────────────────────────────────────────────────┤
  │
  │  [전통적 WAN]                                                      │
  │
  │  Campus LAN ◀──▶ MPLS Cloud ◀──▶ Branch LAN                        │
  │                    │                                                │
  │                 단일 경로                                            │
  │                 (제한된弹性)                                         │
  │                 높은 비용 (MPLS)                                     │
  │
  │  [SD-WAN]                                                          │
  │
  │  Campus LAN ◀──▶ SD-WAN Controller ◀──▶ Branch LAN                  │
  │                     │                    │                         │
  │                  MPLS + 인터넷           인터넷                      │
  │                  (복수 경로)              (복수 경로)                 │
  │                  동적 경로 선택           자동 failover             │
  │                  비용 절감               애플리케이션 인식 라우팅    │
  │
  │  ※ SD-WAN은Overlay 네트워크로, 물리적 WAN 인프라 위에                │
  │    논리적 네트워크를 소프트웨어로 정의한다                           │
  │
  └─────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** SD-WAN의 핵심 가치는 물리적 WAN 인프라에 대한抽象화다. 전통적 WAN에서 MPLS는 신뢰성이 높지만 비용이 높고 경로 변경이 어렵다. SD-WAN은 복수의 물리적 WAN 연결(MPLS, 인터넷 등)을 aggregation하여 하나의 논리적 WAN으로 제공한다. SD-WAN 컨트롤러가 애플리케이션의 종류(VoIP, 이메일, 웹 등)와 현재 WAN 상태(대역폭, 지연, 패킷 유실률)를 기반으로 최적 경로를 동적으로 선택한다. 예를 들어, VoIP 트래픽은 지연이 낮은 MPLS로, 대역폭 위주 트래픽은 비용이 낮은 인터넷으로 자동 배분된다. 이러한 software-defined 접근은 WAN 운영의 Flexibility와 비용 효율성을 크게 향상시킨다.

### 도입 체크리스트
- **기술적**: 애플리케이션의 지연/대역폭 요구사항을 분석하고, 이를 만족하는 LAN, MAN, WAN 기술을 선택했는가?
- **운영·보안적**: WAN 경유 시 보안 정책(encryption, VPN)이 적용되어 있는가?

### 안티패턴
- **LAN 기술을 WAN에 잘못 적용**: 이더넷 스위치를 장거리 WAN 전송에 사용하면, 물리적 매체 제한과 프로토콜 한계로 인해 정상 동작하지 않는다.
- **불필요한 WAN 대역폭 구매**: 실제 사용량이 적당한 인터넷 대역폭으로 충분한데, 과도한 MPLS 대역폭을 구매하여 비용을 낭비한다.

- **📢 섹션 요약 비유:** WAN 설계는 물류 허브 선정과 같습니다. 소량이면 택배(인터넷 SDN-WAN)로 충분하지만, 대량 화물은 전용 화물선(MPLS WAN)이 필요하며, 어느 쪽이어도 고객(애플리케이션)이满意해야 합니다.

### 미래 전망
- **SD-WAN 대세**: 전 세계 SD-WAN 시장은 빠르게 성장しており, 전통적 MPLS WAN을 대체하는 추세다.
- **Cloud-Native Networking**: 클라우드 서비스와 Native統合된 네트워킹으로, 전통적인 LAN/MAN/WAN 분류가進一步模糊될 것으로 예상된다.

- **📢 섹션 요약 비유:** 네트워크 분류의進化は都市計画の進化와 같습니다. 처음에는城中全域が一つ 도로로 연결되었지만(단일 네트워크), 도시가 커지면서功能별로 도로가分かれ(분류 발생), 결국에는 고속도로와 지하철이 모두 연결된 복합 교통 체계(통합 네트워킹)가 되었습니다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **Ethernet (IEEE 802.3)** | LAN의 대표 전송 기술로, 고속 이더넷(10Gbps, 100Gbps)이 데이터 센터에서 주로 사용된다. |
| **MPLS (Multiprotocol Label Switching)** | WAN에서 사용되는 Label-based 라우팅 기술로, 경로 설정 시간 단축과 QoS 보장이 가능하다. |
| **SD-WAN** | 소프트웨어로 WAN을 정의하여 복수 WAN 연결을 통합 관리하는 기술이다. |
| **VPN (Virtual Private Network)** | 공인 네트워크 위에 터널을 생성하여 사설 네트워크처럼 사용하는 기술로, WAN 보안 통신에 필수적이다. |
| **ISP (Internet Service Provider)** | 사용자에게 WAN 인터넷 접속 서비스를 제공하는通信사업자다. |

## 👶 어린이를 위한 3줄 비유 설명
1. 네트워크 분류는 놀이공원太大了划分과 같습니다. 우리 집 뒤뜰( LAN)은 직접管理하고, 동네 놀이터( MAN)는 Municipal Offices管理하며,Disneyland 같은 대규모 테마파크(WAN)는Separate 회사가管理해요.
2. 집 뒤뜰에好玩 것은 바로 놀 수 있지만(낮은 지연), Disneyland은 차를 타고 가야 하죠(높은 지연). 그래도 Disneyland엔 집 뒤뜰에 없는 놀이기가 있듯이, WAN에도 LAN에 없는 글로벌 서비스가 있어요.
3. 그래서 회사에서도 업무용으로는 사무실 LAN을 쓰면서도, 글로벌 회의에는 WAN으로 연결된 화상 시스템을 쓰는 거예요!
