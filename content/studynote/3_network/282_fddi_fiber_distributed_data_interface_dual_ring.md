## 핵심 인사이트 (3줄 요약)
> 1. **본질**: FDDI(Fiber Distributed Data Interface)는 기존 구리선 기반의 느린 토큰 링(4~16Mbps)이 가진 한계를 극복하기 위해, **전송 매체를 빛(광케이블)으로 바꾸어 100Mbps의 초고속 스피드를 구현한 1990년대의 최첨단 백본(Backbone) 네트워크 규격**이다.
> 2. **이중 링(Dual Ring) 구조**: 토큰 링의 치명적 약점인 '선이 끊기면 망 전체가 죽는 현상'을 막기 위해, 시계 방향의 메인 링(Primary)과 반시계 방향의 예비 링(Secondary) 두 겹으로 망을 구성하여 **완벽한 하드웨어적 결함 허용(Fault Tolerance)**을 달성했다.
> 3. **역사적 퇴장**: 대학교 캠퍼스나 대기업 건물들을 묶어주는 100Mbps 백본망의 절대 강자였으나, 설치비가 너무 비싸고 유지 보수가 까다로웠으며, 무엇보다 훨씬 싸고 빠른 **기가비트 이더넷(1Gbps)**의 등장으로 한순간에 멸망한 비운의 기술이다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: ANSI(미국국립표준협회)와 ISO가 제정한 100Mbps 속도의 광섬유 토큰 링 LAN/MAN 프로토콜. 토큰 패싱(Token Passing) 알고리즘을 사용하므로 충돌이 발생하지 않는다.
- **필요성**: 1980년대 말 이더넷(10Mbps)과 일반 토큰 링(16Mbps)은 PC 수십 대를 연결하기엔 충분했지만, A건물, B건물, C건물의 거대한 네트워크들을 하나로 묶어주는 메인 척추(Backbone) 역할을 하기에는 속도가 너무 느리고, 거리가 짧았으며(구리선의 한계), 한번 끊어지면 대형 사고가 났다. 따라서 "수십 km를 가면서도 속도는 10배 빠르고, 포크레인이 선을 파먹어도 즉각 복구되는 궁극의 튼튼한 척추"가 절실했다.

- **💡 비유**: FDDI는 건물을 잇는 **"지하철 순환선(2호선)"**과 같습니다. 동네 마을버스(이더넷)가 모아온 승객(데이터)들을 한꺼번에 싣고 다른 동네로 초고속(광케이블)으로 실어 나릅니다. 내선 순환과 외선 순환(이중 링)이 다 있어서 한쪽 터널이 무너져도 반대 방향 열차로 어떻게든 승객을 실어 나를 수 있습니다.

> 📢 **섹션 요약 비유:** 일반 이더넷 랜선이 집 안에서 쓰는 **"플라스틱 수도관"**이라면, FDDI는 도시 전체의 물탱크를 연결하기 위해 땅속 깊숙이 묻어둔 **"절대 터지지 않는 초대형 강철 이중 배관(백본)"**입니다.

## Ⅱ. FDDI의 아키텍처와 Self-Healing 메커니즘 (Deep Dive)

### 1. 이중 링(Dual Ring)과 노드 종류
FDDI 네트워크는 물리적으로 2개의 광케이블 링으로 구성된다.
- **Primary Ring (주 링)**: 평상시에 모든 데이터와 토큰이 시계 방향으로 쌩쌩 도는 메인 도로.
- **Secondary Ring (부 링)**: 평상시엔 텅 비어있다가, 주 링이 끊어지면 반시계 방향으로 데이터를 돌려 살려내는 스페어 도로.

노드(라우터나 대형 스위치)가 이 링에 어떻게 물리느냐에 따라 두 종류로 나뉜다.
- **DAS (Dual Attachment Station)**: 메인 링과 예비 링 양쪽 모두에 랜카드가 꽂혀 있는 비싼 녀석. (백본망의 주요 기둥들)
- **SAS (Single Attachment Station)**: 돈이 없어서 메인 링에만 꽂혀 있는 녀석. 중간에 배선기(Concentrator)를 거쳐서 간접적으로 연결된다.

### 2. 셀프 힐링(Self-Healing): 무중단 복구의 마법
포크레인이 땅을 파다가 노드 B와 노드 C 사이의 케이블을 완전히 절단 냈다고 가정해 보자.
1. 노드 B와 C는 자신들 사이의 빛 신호가 사라진 것을 0.001초 만에 감지한다.
2. 노드 B는 오른쪽(C 방향)으로 주 링을 쏘지 못하게 되자, 주 링의 데이터를 스위치 내부에서 꺾어 **부 링(반시계 방향)으로 U턴** 시켜버린다. (Wrap 과정)
3. 노드 C 역시 왼쪽(B 방향)으로 쏘지 못하자, 부 링에서 오던 데이터를 주 링으로 U턴 시킨다.
4. 끊어진 부분을 제외한 거대한 C자 모양의 새로운 하나의 링이 완성되며 통신 단절 없이 네트워크가 100% 복구된다.

```text
 ┌─────────────────────────────────────────────────────────────┐
 │                 FDDI의 이중 링 자가 복구(Self-Healing)         │
 ├─────────────────────────────────────────────────────────────┤
 │                                                             │
 │   [ 평상시 ]                          [ 장애 발생 시 (U턴) ]       │
 │                                                             │
 │     ▶ 주 링(시계) ▶                      ▶ 주 링(시계) ▶        │
 │  ┌─ A ────── B ─┐                 ┌─ A ────── B ┐       │
 │  │   ◀ 부 링 ◀   │ 포크레인 절단!     │   ◀ 부 링 ◀   │ U턴!   │
 │  │              │    ====X====▶   │              X       │
 │  │   ◀ 부 링 ◀   │                 │   ◀ 부 링 ◀   │ U턴!   │
 │  └─ D ────── C ─┘                 └─ D ────── C ┘       │
 │     ▶ 주 링(시계) ▶                      ▶ 주 링(시계) ▶        │
 │                                                             │
 │   결과: 링이 끊어졌지만, B와 C가 내부에서 링을 꼬아서 이어버려       │
 │        거대한 "하나의 말굽 자석(U자) 모양"으로 망이 유지된다!        │
 └─────────────────────────────────────────────────────────────┘
```

### 3. FDDI의 소멸과 이더넷의 승리
아무리 완벽해도 도태될 수밖에 없었다. 광케이블 랜카드 2개(DAS)를 꽂아야 하는 FDDI 라우터는 한 대에 수천만 원이 넘었다.
그 사이 이더넷 진영(IEEE 802.3)은 UTP 구리선 8가닥을 다 꼬아 쓰는 기염을 토하며 가격이 10분의 1도 안 되는 **"패스트 이더넷(100Mbps)"**과 **"기가비트 이더넷(1Gbps)"**을 시장에 쏟아부었다. 신뢰성은 스위치 이중화(STP/LACP)로 때워버렸다. FDDI는 가성비에서 압살당하며 2000년대 초반에 흔적도 없이 사라졌다.

> 📢 **섹션 요약 비유:** FDDI는 타이어에 펑크가 나도 100km/h로 계속 달릴 수 있는 **"최첨단 방탄 군용 장갑차"**였습니다. 하지만 민간인(기업)들은 그 비싼 장갑차를 사는 대신, 펑크 나면 타이어를 빨리 갈아 끼우면(STP) 되는 **"10배 빠르고 저렴한 스포츠카(기가 이더넷)"**를 선택했습니다.
