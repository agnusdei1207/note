# 536. NTP (Network Time Protocol) - 계층적 시간 동기화, Stratum 레벨 모델 (UDP 123)

> **핵심 인사이트**: 네트워크 장비 수백 대의 시간이 1초라도 다르면, 방금 일어난 해킹 사고의 로그 순서가 뒤죽박죽이 되어버린다. NTP는 전 세계 컴퓨터의 시계를 기준 시계(원자시계)와 똑같이 맞추는 다단계 시간 동기화 피라미드 시스템이다.

## Ⅰ. NTP (Network Time Protocol)의 개념
인터넷이나 로컬 네트워크상의 컴퓨터들이 서로 시간을 동기화하기 위한 프로토콜입니다. 
- **포트 및 통신**: 기본적으로 빠르고 오버헤드가 적은 **UDP 123번 포트**를 사용합니다.
- **필요성**: 시스템 로그 분석 시 이벤트 발생 순서 정확성 보장, 트랜잭션 처리(분산 DB), 보안 인증서(SSL/TLS) 유효 기간 검증 등에 있어 정확한 시스템 시간은 필수적입니다.

## Ⅱ. Stratum (스트라텀) 계층 모델
NTP는 서버의 부하를 분산하고 계층적으로 시간을 전달하기 위해 피라미드 형태의 **Stratum(계층)** 구조를 가집니다.

- **Stratum 0**: 세슘 원자시계, GPS 시계 등 초정밀 하드웨어 시간 소스입니다. (네트워크로 직접 서비스하지 않고 물리적으로 연결됨)
- **Stratum 1**: Stratum 0 장비와 직접 연결된 최상위 타임 서버(Primary Time Server)입니다.
- **Stratum 2**: Stratum 1 서버와 네트워크로 동기화된 서버입니다. 일반적인 클라이언트나 기업 내 내부 NTP 서버들이 주로 여기에 접속합니다.
- **Stratum 3 ~ 14**: 하위 계층으로 내려갈수록 Stratum 숫자가 증가합니다.
- **Stratum 15**: 동기화 가능한 최하위 계층입니다.
- **Stratum 16**: 시계가 동기화되지 않은(Unsynchronized) 신뢰할 수 없는 상태를 의미합니다.

## Ⅲ. 통신 방식과 오차 보정 알고리즘
네트워크를 통해 시간을 받아올 때는 '패킷 이동 시간(Delay)' 때문에 필연적으로 오차가 발생합니다.
1. 클라이언트가 서버에 시간 요청 패킷을 보낼 때 타임스탬프(T1)를 기록합니다.
2. 서버가 요청을 받고(T2), 응답을 보낼 때(T3) 다시 타임스탬프를 찍습니다.
3. 클라이언트가 패킷을 수신한 시간(T4)을 바탕으로 왕복 지연시간(Round-Trip Delay)과 오프셋(Offset)을 계산합니다.
4. 클라이언트는 계산된 오프셋만큼 시스템 시간을 미세 조정(Slew)하거나 강제 설정(Step)하여 서버 시간과 정확히 일치시킵니다.

## Ⅳ. 실무 시나리오
금융권 트레이딩 시스템, 보안 로그(Syslog, SIEM) 통합 분석, 분산 데이터베이스(Cassandra, Spanner)의 트랜잭션 순서 보장에 필수적으로 활용됩니다. 내부망(폐쇄망) 환경에서는 외부에 연결된 Stratum 1/2 서버로부터 시간을 받아오는 사내 전용 NTP 서버(Stratum 3)를 구축하여 전사 장비들을 동기화합니다.

> 📢 **섹션 요약 비유**: 전국 학교 교실의 벽시계(컴퓨터)를 맞추는 과정입니다. 교장 선생님(Stratum 0)이 차고 있는 롤렉스 시계를 보고, 교감 선생님(Stratum 1)이 손목시계를 맞춘 뒤, 담임 선생님들(Stratum 2)이 교감 선생님 시계를 보고 맞추고, 학생들(Stratum 3)이 담임 선생님 시계를 보고 각자의 전자시계를 동기화하는 '다단계 시간 맞추기' 작전입니다.
