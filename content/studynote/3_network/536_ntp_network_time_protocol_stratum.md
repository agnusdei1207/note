# 536. NTP (Network Time Protocol) - 시간 동기화

> **핵심 인사이트**: 전 세계의 서버들은 시계가 제각각이면 에러 로그의 선후 관계를 맞출 수도 없고, 결제 시간도 꼬여서 대혼란이 발생한다. NTP는 원자시계라는 절대적인 기준시를 꼭대기(Stratum 0)에 두고, 피라미드처럼 아래로 내려오며 전 세계 컴퓨터의 시간을 밀리초(ms) 단위로 똑같이 맞추는 시계태엽공이다.

## Ⅰ. NTP (Network Time Protocol)의 개념
네트워크에 연결된 수많은 컴퓨터, 서버, 라우터들의 **시스템 시계(Clock)를 밀리초(ms) 단위의 정밀도로 동기화**시키기 위한 인터넷 표준 프로토콜입니다. (UDP 포트 123번 사용)

## Ⅱ. 시간 동기화가 필수적인 이유
- **로그 및 트러블슈팅**: 방화벽, 웹 서버, DB 서버의 시계가 다르면, 해킹 사고 발생 시 어떤 일이 먼저 일어났는지 원인 추적이 불가능합니다.
- **보안 및 인증**: 타임스탬프 기반의 인증(Kerberos, 일회용 비밀번호 OTP)은 서버와 클라이언트 간의 시계가 5분 이상 차이 나면 무조건 인증을 거부합니다.

## Ⅲ. Stratum (스트라텀) 계층 모델
NTP는 시간의 정확도를 보장하기 위해 피라미드 형태의 다단계 계층(Hierarchical) 구조를 가집니다. 숫자가 작을수록 최상위 권위자입니다.

```text
[ Stratum 0 ] 원자시계, GPS 시계 (네트워크 연결 안 됨, 가장 정확함)
     │
[ Stratum 1 ] (1차 서버) Stratum 0과 직접 시리얼 케이블로 연결된 서버 (국가 표준시 등)
     │
[ Stratum 2 ] (2차 서버) Stratum 1을 통해 시간을 받아오는 인터넷의 대형 공개 타임 서버들
     │
[ Stratum 3 ] (회사 내부 서버) Stratum 2로부터 시간을 받아 사내 망에 뿌려주는 로컬 서버
     │
[ Stratum 4 ] (일반 PC, 스마트폰)
```
- 하위 계층은 상위 계층에게 시간을 물어보며, 최대 Stratum 15까지 확장 가능합니다. (16은 동기화 안 된 상태)

## Ⅳ. 시간 동기화 알고리즘 (네트워크 지연 보상)
단순히 "지금 몇 시야?"라고 묻고 답을 받으면, 그 패킷이 인터넷을 타고 날아오는 동안 이미 0.1초가 지나버려 오차가 생깁니다.
- **NTP의 핵심**: 패킷이 갈 때 걸린 시간과 올 때 걸린 시간(Round Trip Delay)을 계산하여, **네트워크 전송 지연 시간만큼을 더하고 빼서 오차를 스스로 보정**하는 수학적 알고리즘이 탑재되어 있습니다.

> 📢 **섹션 요약 비유**: 수만 명의 마라톤 대회 참가자들이 각자 자기 손목시계를 보고 출발하면 난장판이 됩니다. 대회 주최 측(Stratum 0)이 출발 총성을 쏘고, 주변 스피커(Stratum 1)가 이를 중계하며, 멀리 있는 스피커(Stratum 2)는 소리가 도달하는 지연 시간까지 계산해서 타이머를 완벽하게 똑같이 맞추는(NTP) 시스템입니다.
