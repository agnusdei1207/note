# 70. 주파수 도약 확산 스펙트럼 (FHSS, Frequency Hopping Spread Spectrum)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: FHSS는 넓은 주파수 대역을 수십~수백 개의 좁은 채널로 쪼갠 뒤, 데이터를 보낼 때 송신자와 수신자가 미리 약속한 난수 패턴(PN)에 따라 **1초에 수천 번씩 채널을 이리저리 점프(Hopping)하며 통신하는 회피 기동 아키텍처**다.
> 2. **가치 (항재밍의 극의)**: 특정 주파수에 악성 노이즈나 재밍 폭탄이 터져도, 내 신호는 이미 0.001초 만에 다른 주파수로 도망가버리기 때문에 전체 데이터 손실을 원천 차단하는 생존력을 자랑한다.
> 3. **실무 융합**: 2.4GHz ISM 대역처럼 간섭이 미친 듯이 득실대는 더러운 전파 환경에서, 기기들이 서로 부딪히지 않고 공존할 수 있게 만든 **블루투스(Bluetooth)의 핵심 코어 기술**이다.

---

## Ⅰ. FHSS의 개요 및 탄생 배경 (Context & Necessity)

- **개념**: 
  - 반송파 주파수($f_c$)를 고정하지 않고, 의사 난수(PN) 생성기의 출력에 따라 반송파 주파수를 매우 빠른 속도로 계속 바꾸는(도약, Hopping) 대역 확산 기술.
  - 넓은 대역을 쓰긴 하지만, 특정 순간(찰나)만 보면 아주 좁은 협대역(Narrowband)을 사용한다.
- **탄생 배경**: 
  - 군용 통신에서 적군이 특정 주파수를 감청하거나 방해 전파(Jamming)를 쏠 때, 이에 맞서 싸우기보다 **"도망가는 게 상책"**이라는 게릴라 전술에서 착안했다. 
  - 송신자와 수신자가 피아노 건반을 무작위로 치는 순서(도-미-솔-파-시...)를 미리 맞춰놓고, 그 순서대로 건반(주파수)을 치며 통신하면 적군은 어느 건반에서 소리가 날지 예측할 수 없게 된다.

- **💡 비유**: **FHSS는 '메뚜기 뛰기로 징검다리 건너기'**와 같다.
  - 강물 위에 80개의 징검다리(주파수 채널)가 있다.
  - 암살자(노이즈, 재밍)가 3번 다리를 폭파시켰다.
  - 만약 내가 3번 다리만 고집해서 걸었다면(기존 통신) 나는 죽는다. 
  - 하지만 나는 1초에 1,600번씩 80개의 다리를 무작위로 미친 듯이 뛰어다닌다(블루투스). 3번 다리를 밟을 찰나의 순간에 약간 비틀거릴 수는 있어도, 나머지 79개의 다리에서 멀쩡히 뛰어갔으므로 목적지에 100% 안전하게 도착한다.

## Ⅱ. 핵심 아키텍처 및 메커니즘 (Deep Dive)

### 1. 호핑(Hopping) 메커니즘의 3요소
- **채널 세트**: 전체 대역폭을 잘게 쪼갠 수많은 채널 (예: 블루투스는 2.4GHz 대역을 1MHz 간격으로 79개로 쪼갬).
- **PN 코드 (의사 난수)**: 다음에 몇 번 채널로 뛸 것인지를 결정하는 비밀 지도. 수신기도 똑같은 코드를 돌려야 따라다니며 받을 수 있다.
- **도약 속도 (Hopping Rate)**: 
  - **Fast FHSS**: 데이터 1비트를 보내는 동안 주파수를 여러 번 바꿈. (극한의 보안, 드론 조종기)
  - **Slow FHSS**: 주파수 한 번 바꿨을 때 그 채널에서 여러 개의 비트를 우르르 보내고 튐. (블루투스: 1초에 1,600번 뜀).

### 2. 주파수-시간 도메인 호핑 시각화

```text
  ┌─────────────────────────────────────────────────────────────┐
  │              [FHSS의 주파수 호핑(도약) 시간축 시각화]             │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │ 주파수(Hz)                                                   │
  │  [Ch 4] ─────────────────■■───────────■■──────────         │
  │                                                             │
  │  [Ch 3] ──────■■───────────────────────────────────         │
  │                        (적군 재밍/간섭)                        │
  │  [Ch 2] ─────────────────────XXXX──────────────────         │
  │                                                             │
  │  [Ch 1] ──■■───────────────────────────────────■■─         │
  │           │   │   │   │   │   │   │   │   │   │   │         │
  │        ── t1  t2  t3  t4  t5  t6  t7  t8  t9  t10 ──▶ 시간(t) │
  │                                                             │
  │ * [방어 원리]: t5~t7 구간에 Ch 2번에서 노이즈(X)가 폭발함. 하지만 내 │
  │   신호는 t5에 이미 4번 채널로 도망갔으므로 1%의 피해도 받지 않음!      │
  └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** FHSS의 핵심은 **시간과 공간(주파수)의 분산**이다. 특정 주파수(Ch 2)에 강력한 간섭(Wi-Fi 공유기 등)이 상주하고 있더라도, 내 신호가 그 채널에 머무는 시간은 고작 0.0006초(625$\mu s$)에 불과하다. 그 찰나의 데이터가 깨지더라도 통신 계층에서 FEC(에러 정정)나 ARQ(재전송)로 슬쩍 때워버리면, 사용자는 끊김을 전혀 느끼지 못한다.

## Ⅲ. 실무 융합: DSSS와의 비교 및 응용

### 1. DSSS (직접 수열) vs FHSS (주파수 도약) 완벽 비교
대역 확산(Spread Spectrum)의 양대 산맥이지만, 싸우는 방식은 완전히 반대다.

| 비교 관점 | DSSS (Direct Sequence) | FHSS (Frequency Hopping) |
|:---|:---|:---|
| **에너지 분산 방식** | 암호 코드로 곱해서 **동시에 전체 대역**에 얕게 깔아버림 | 순간순간은 좁은 대역을 쓰지만, **시간에 따라 넓은 대역을 돌아다님** |
| **재밍 방어 논리** | 재밍을 맞아도 수학적으로 갈아 마셔버림 (정면 돌파) | 재밍이 있는 곳을 그냥 피해서 도망감 (회피 기동) |
| **근거리 간섭(Near-Far)** | 치명적임 (가까운 놈이 크게 쏘면 다 죽음, 전력 제어 필수) | 문제없음 (어차피 채널이 달라서 안 부딪힘) |
| **하드웨어 복잡도** | 칩셋이 비싸고 전력 소모가 큼 | 구조가 매우 단순하고 배터리를 덜 먹음 (IoT 최적) |
| **대표 실무 적용** | 3G CDMA, GPS, Wi-Fi(802.11b) | **블루투스**, 군용 무전기, RC 조종기 |

### 2. 블루투스의 진화 (AFH: 적응형 주파수 도약)
- **문제**: 2.4GHz 대역은 쓰레기통이다. Wi-Fi가 1, 6, 11번 채널에 거대한 대역폭으로 알박기를 하고 있다. 초창기 블루투스(FHSS)는 눈치 없이 Wi-Fi가 쏘고 있는 채널로 점프해 들어갔다가 머리통이 깨져서 서로 통신이 먹통 되는 일이 잦았다.
- **AFH (Adaptive FHSS)의 등장**: 블루투스 1.2부터는 똑똑해졌다. 79개의 채널을 뛰어다니다가, 특정 채널(예: Wi-Fi가 있는 곳)에서 패킷이 계속 깨지면 해당 채널을 **'Bad Channel'**로 블랙리스트에 올린다. 그리고 다음부터는 호핑을 뛸 때 그 채널은 아예 건너뛰어 버리는 갓-알고리즘을 탑재했다. 덕분에 오늘날 무선 이어폰이 만원 지하철에서도 끊기지 않는 것이다.

## Ⅳ. 시스템 도입 체크리스트 및 안티패턴

### 도입 시나리오 (산업용 드론 통신)
공사 현장에서 드론 영상을 전송받아야 한다. 현장에는 이미 수십 개의 Wi-Fi와 무전기가 난립하고 있다.
- **안티패턴**: 넓은 대역을 한 번에 쓰는 일반 Wi-Fi(OFDM) 모듈로 드론을 조종하려 하면, 크레인 뒤에 숨거나 현장 노이즈가 터질 때 드론 조종 신호가 0.1초 멈춰서 드론이 추락한다.
- **해결책**: 영상(고용량)은 OFDM으로 쏘더라도, **조종/제어 신호(Control Link)** 만큼은 절대로 끊기면 안 되므로 무조건 **고속 FHSS(수천 번 호핑)** 칩셋을 분리하여 장착해야 한다. 노이즈가 터져도 채널을 피해서 조종 신호를 살려내는 극강의 회피력을 활용해야 한다.

### 체크리스트
- **동기화(Synchronization) 지연**: FHSS 수신기는 송신자가 지금 79개 중 어느 채널에 있는지 정확한 '타이밍'을 잡기 위해 처음에 페어링(Pairing) 시간이 걸린다. 기기가 잠에서 깨어나 즉시 10ms 이내에 데이터를 쏴야 하는 초저지연 IoT 환경이라면 호핑 동기화 시간(Preamble 헌팅)이 병목이 되지 않는지 점검했는가?

## Ⅴ. 결론
FHSS는 **"모두가 싸우는 전쟁터에서 어떻게 하면 나 혼자 요리조리 피해가며 살아남을 수 있을까?"**라는 질문에 대한 가장 완벽한 물리 계층의 해답이다. DSSS가 거대한 파도를 정면으로 맞고 부수는 항공모함이라면, FHSS는 파도 사이를 요리조리 피해서 달리는 쾌속정이다. 2.4GHz라는 무법지대(ISM 대역)가 파멸하지 않고 오늘날 수백억 개의 기기가 평화롭게 공존할 수 있는 이유는 전적으로 이 FHSS의 회피 기동 아키텍처 덕분이다.
