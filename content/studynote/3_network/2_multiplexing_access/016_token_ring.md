## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 토큰 링(Token Ring, IEEE 802.5)은 네트워크上的 노드들이 하나의 논리적 고리(Logical Ring)를 형성하고, 특수한 제어 프레임인 "토큰"이 이 고리를 따라 순환하며, 토큰을 가진 노드만이 데이터를 전송할 수 있는 매체 접근 제어 방식이다.
> 2. **가치**: 토큰 순찰 방식은 충돌(Collision)이 발생하지 않으며, 각 노드에 보장된 전송 기회(Guaranteed Transmission Opportunity)를 제공하여, 정해진 수의 노드에서 최대 지연 시간이 예측 가능한 실시간 통신에 적합한 특성을 가진다.
> 3. **융합**: 토큰 링의 보장된 대역폭과 결정적 지연(Deterministic Delay) 특성은 산업용 이더넷(Industrial Ethernet), TTE (Time-Triggered Ethernet) 등으로 이어졌으며, 자동차 내부 네트워크(CAN, FlexRay)와도相似的概念이다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: 토큰(Token)은 토큰 링 네트워크에서 데이터 전송 권한을代表하는 특수한 控制 프레임이다. 토큰이 네트워크를 순환하며, 노드는 토큰을 보유했을 때만 데이터를 전송할 수 있다. 한 노드가 토큰을 회수하고 자신의 데이터를 목적지 노드와 중간 노드들을 통해 전달하면, 목적지 노드가 데이터를 수신하고 토큰을 재해방한다. 토큰이釋放되면,它是传递给环上的下一个节点. 토큰은 일종의"버스"라고 할 수 있으며, 이 버스를 가지고 있는 사람만이"말할"(데이터 전송) 수 있다. 다른 사람은 모두"듣기"만 가능하다.

- **필요성**: CSMA/CD의 유선 이더넷에서는 충돌이 발생할 수 있어, 고부하 상태에서 채널 이용률(channel utilization)이 급격히 저하되고, 지연 시간의 변동(jitter)이 커지는 문제가 있다. 특히 실시간 음성/影像 통신이나 공장自動化(FA)처럼 일정한 지연이 보장되어야 하는 환경에서는 이러한 unpredictability가致命적일 수 있다. 토큰 링은"토큰이라는 송신 권한凭证"을环上で 순환させる 방식により, 충돌이原理적으로 발생하지 않고, 각 노드에 대해最大待機時間이 보장된다. 이로 인해, 실시간 통신에 적합한 결정적(deterministic) 동작이 가능하다.

- **💡 비유**: 토큰 링은 회重要 회의에서의"발언권 토큰"과 같다. 회의에서 한 사람만이发言권(토큰)을 가지고 있으며,发言권을 가진 사람만이 말씀하 can speak. 다 마치면发言권을 옆 사람에게 건네주며, 이를 순환한다. 따라서 둘이 동시에 말씀하ayer collision이 발생할 수 없으며, 한 사람에게 배정된"대기 시간"이 보장된다.

- **등장 배경 및 발전 과정**:
  1. **1960년대:Token Passing 개념**: MIT의 Fred K. B. Mak과 다른 연구자들이 token passing 방식의 통신망을 제안했다.
  2. **1970년대:IBM의Token Ring 개발**: IBM이 1970년대에 Token Ring을 개발하여, 1985년 IEEE 802.5로 표준화했다.
  3. **1990년대:이더넷과의 경쟁**: 이더넷이 스위치 기반으로 전환하며 성능과 비용 면에서 이점을 갖추자, Token Ring은 시장 점유율을 빠르게 잃었다.
  4. **2000년대:시장 퇴출**: IBM이 2000년대 초 Token Ring 지원을 중단했으며, 현재는歴史적 기술로 취급된다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 토큰 링 동작 단계

토큰(Token)은 프레임이 아닌 약 3바이트의 특수한 控制 프레임이다. 노드는 토큰을 보유했을 때만 전송을 시작할 수 있으며, 전송 완료 후에는 토큰을释累하여 다음 노드에게 전달한다.

```text
  ┌─────────────────────────────────────────────────────────────────────┐
  │                    토큰 링(Token Ring) 동작 단계                        │
  ├─────────────────────────────────────────────────────────────────────┤
  │
  │  [토큰 순환 구조]                                                    │
  │
  │              ┌─────────────────────────────────┐                     │
  │              │                                 │                     │
  │              │    A ──▶ B ──▶ C ──▶ D ──▶ A   │                     │
  │              │    ◀── 토큰 순환 방향 ────────────│                     │
  │              │                                 │                     │
  │              └─────────────────────────────────┘                     │
  │
  │  ※ 논리적 링 (Logical Ring) = 물리적 배선은 스타 (Star)처럼 보일 수 있음  │
  │    하지만 논리적으로는 링으로 동작                                      │
  │
  │  [동작 단계]                                                         │
  │
  │  ① 토큰 보유 (Token Holding):                                          │
  │     - 노드가 토큰을 보유한 상태                                        │
  │     - 다른 노드는 데이터 전송 불가                                      │
  │
  │  ② 데이터 전송 (Frame Transmission):                                   │
  │     - 토큰 보유 노드가 프레임 전송                                      │
  │     - 프레임이 링을 따라 순환                                          │
  │     - 목적지 노드가 프레임 수신 (복사 후 계속 전달)                     │
  │
  │  ③ 토큰 방출 (Token Release):                                          │
  │     - 송신 노드가 데이터 전송 완료 후 토큰 방출                        │
  │     - 토큰이 다음 노드에게 전달                                        │
  │
  │  ④ 토큰 회수 (Token Recirculation):                                   │
  │     - 토큰이 다음 노드에 도달                                          │
  │     - 새 노드가 토큰 보유자가 됨                                        │
  │
  └─────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 토큰 링의 핵심은"논리적 순서"다. 물리적 배선은 스타(중앙 허브/MAU)로 보이더라도, 각 포트에서 들어오고 나가는 순서가 논리적으로 링을 형성한다. 토큰은 3바이트짜리 控制 프레임으로, 프레임과 달리 데이터 없이 네트워크를 순환한다. 노드는 토큰을"보유"했을 때만 프레임을 전송할 수 있다. 프레임이 목적지 노드에 도달하면, 노드는 프레임을 복사하고"TAG(Transmission bit)"를 설정하여 프레임을 계속 전달한다. 원래 송신 노드가 돌아오는 프레임에서 TAG를 확인하여 전송 성공을 확인하고, 토큰을 방출한다. 토큰 방출 후에는 토큰이 링을 따라 순환하며, 다른 노드가 토큰을 획득하여 자신의 전송 기회를 갖는다.

### 토큰 관리를 위한 추가 메커니즘

토큰 링에서는 토큰의 소유권 이전뿐 아니라, 토큰 분실, 다중 토큰, 노드故障 등의特殊情况를 처리하기 위한 메커니즘이 필요하다.

```text
  ┌─────────────────────────────────────────────────────────────────────┐
  │                    토큰 링 관리 메커니즘                                 │
  ├─────────────────────────────────────────────────────────────────────┤
  │
  │  [토큰 관리: ARP (Active Monitor)]                                  │
  │                                                                     │
  │     - 하나의 노드(활성 모니터)가 전체 링을 관리                       │
  │     - 토큰 분실 감시: Monitor_interval 마다 토큰 확인                │
  │     - 다중 토큰 방지: 둘 이상의 토큰 감지 시 하나 회수                 │
  │     - 자기 자신에게서 출발한 프레임이 돌아오는 것 확인                  │
  │
  │  [노드 개입 (Intervention)]                                         │
  │
  │     - 노드가故障 시: ring_error를 신호하여 ARP에게 알림               │
  │     - ARP가故障 노드를 링에서 제거                                   │
  │     - 링이 재구성되어 계속 동작                                       │
  │
  │  [優先순위 (Priority) 메커니즘]                                      │
  │
  │     - 토큰과 프레임에 priority (0~7) 할당                            │
  │     - 높은 우선순위 노드가 낮은 우선순위 노드의 토큰을 뺏을 수 있음    │
  │     - 실시간 트래픽에 우선순위 보장                                    │
  │
  └─────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 토큰 링의安定的运作을 위해 두 가지重要한 메커니즘이 있다. 첫째, ARP (Active Monitor Station)이다. 링 내의 모든 노드 중 하나가 선택되어 전체 링을 관리한다. ARP는 주기적으로(보통 7초) 토큰이 존재하는지 확인하고, 토큰이 없을 경우(분실) 새 토큰을 생성하며, 두 개 이상의 토큰이 탐지되면 이를 회수한다. 둘째, 노드 개입(Intervention)이다. 노드에故障가 발생하면 해당 노드가Electrical적으로 자신을 링에서 분리하여, 나머지 노드들은 계속 통신할 수 있다. 이러한 메커니즘 덕분에 토큰 링은故障시에도部分적 동작이 가능했다.

## Ⅲ. 융합 비교 및 다각도 분석

### 토큰 링 vs 이더넷 vs FDDI 비교

| 비교 항목 | 토큰 링 (IEEE 802.5) | 이더넷 (IEEE 802.3) | FDDI |
|:---|:---|:---|:---|
| **접근 방식** | 토큰 순찰 (Collision 없음) | CSMA/CD (Collision 가능) | 토큰 순찰 |
| **전송 보장** | 결정적 (Deterministic) | 비결정적 (Probabilistic) | 결정적 |
| **최대 지연** | 예측 가능 (노드 수 기반) | 예측 어려움 (경합 기반) | 예측 가능 |
| **실시간성** | 우수 | 보통 (스위치 도입으로改善) | 우수 |
| **채널 이용률** | 부하 높을 때 우수 | 부하 낮을 때 우수 | 부하 높을 때 우수 |
| **비용** | 높음 (토큰 관리 복잡) | 낮음 (단순한 허브/스위치) | 높음 |
| **현재 상태** | 역사적 (사용 중단) | LAN 표준 | 역사적 (FDDI는骨干망에서 사용) |

### 과목 융합 관점

- **자동차 네트워크 (CAN, FlexRay)**: 자동차 내부의 ECU (Electronic Control Unit)들이 CAN (Controller Area Network) 버스에서 메시지를 교환할 때, CAN도 토큰과 유사하게"메시지 우선순위" 기반으로 버스 접근을 제어한다. 단, CAN은 토큰이 아니라"atar 식 접근"(CSMA/CR)이라 불리는 방식을 사용한다.
- **산업용 이더넷 (Industrial Ethernet)**: Profinet, EtherCAT, Time-Sensitive Networking (TSN) 등은 이더넷 기반으로 결정적 동작을 제공하기 위해, 토큰과 유사한 시간 분할(TDMA) 방식을 결합한다.

- **📢 섹션 요약 비유:** 토큰 링은 회重要 회의의"발언권 토큰"을 차례로 돌리는 것과 같습니다.发言권(토큰)을 가진 사람만이 말씀하 can speak. 다 마치고 나면 옆 사람에게 건네주며, 이를 계속 순환합니다.主席(Active Monitor)가 전체 회의를 관리하여, 토큰이 분실되면 새로 만들거나不规矩한 사람를制합니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 토큰 링에서 이더넷으로의 마이그레이션 (1990~2000년대)**: 1990년대까지 토큰 링은 IBM 환경에서 주로 사용되었으며, 금융, 제조 등 대규모 기업에서 네트워크의결정적 동작을 위해 채택했다. 그러나 이더넷 스위치의 성능 향상과 가격 하락으로, 2000년대에 대부분의 기업이 토큰 링에서 이더넷으로 마이그레이션했다. 이더넷 스위치 기반 네트워크에서도全이중 동작으로 충돌이 사라지고, VLAN, QoS 등으로 결정적 동작이 частично 가능해졌기 때문이다.

2. **시나리오 — 산업용 네트워크에서의 토큰 유사와**: 공장自动化(FA) 환경에서는 여전히 제어 시스템의 예측 가능한 동작이 필수적이다. 이를 위해 CAN (Controller Area Network), FlexRay, 그리고 현대의 TSN (Time-Sensitive Networking)이 사용된다. 이들은 엄밀히 말하면 토큰이 아니라priority 기반 arbitration이지만, 기본 원리(결정적 미디어 접근)는 토큰 링과 유사하다.

```text
  │  ※ 2000년대: 이더넷이 LAN 시장 지배                                   │
  │                                                                     │
  │  이더넷이 승리한 이유:                                              │
  │    1. 스위치 도입으로 Collision 문제 해결 (CSMA/CD 불필요)           │

  │    3. 기술 단순성: 프로토콜이 단순하여 관리 용이                     │
  │    4. 벤더 지원: 대부분의厂商가 이더넷 기술 지원                       │
  │
  └─────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 토큰 링이 이더넷에 패배한主要한 이유는 경제성과 확장성이었다. Token Ring은 IBM이 주도하여 가격이 높았고, Token Ring Concentrator(MAU)도 이더넷 허브보다 훨씬 비쌌다. 또한, 이더넷 스위치의 등장은 충돌 문제를 근본적으로 해결했다. 각 포트가 독립 충돌 도메인이 되면서 CSMA/CD의 단점이 사라지고,全이중 통신으로 한 번에 전송과 수신이 가능해졌다. 이에 따라 이더넷은低成本으로高性能을 제공하게 되었고, 토큰 링은 점차 도태되었다. 또한, 토큰 링의 관리 복잡도(Active Monitor, 토큰 분실 처리 등)도 이더넷 대비 부담이었다.

### 도입 체크리스트
- **기술적**: 현대 LAN 설계에서 토큰 링 대신 이더넷 스위치 기반 아키텍처를 선택하고 있는가?
- **운영·보안적**: 실시간성이 중요한 애플리케이션(음성, 영상, 공장 제어)에서는 이더넷 기반 QoS 또는 TSN 도입을 검토하고 있는가?

### 안티패턴
- **토큰 링의 과도한 신뢰**: 과거에는 토큰 링의"결정적 동작"이 과대평가되어, 많은 기업들이 필요 없는 상황에서도 토큰 링을 선택했다.
- **Real-time Ethernet 도입 미숙**: 이더넷 기반 Real-time 확장이 가능함에도, 너무遅くbisiness continuity에 영향을 줄 정도로 기존 토큰 링을 오랫동안 유지했다.

- **📢 섹션 요약 비유:** 토큰 링과 이더넷의 경쟁은大型arinternational 전세버스公司和자유투어택시 competition과 같습니다. 전세버스는 정해진ルート를 순환하며(토큰 순환), 예약된時間에 맞춰乗客을接送하므로predictable하지만, 같은ルート를 맴도는 데 gostosa과 비용이 많이 듭니다. 반면 택시는 아무 경로로든 이동 가능하며,特别是차량이 增加하면(스위치) 서로 충돌할 일 없이 동시에 이동할 수 있어 비용 대效果가 우수합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 토큰 링 (레거시) | 이더넷 스위치 (현대) | 개선 효과 |
|:---|:---|:---|:---|
| **정량** | 네트워크 장비 비용: 高 | 네트워크 장비 비용: 低 | 비용 **70% 절감** |
| **정성** | 관리 복잡도: 高 | 관리 복잡도: 低 | 운영 효율성 향상 |

### 미래 전망
- **TSN (Time-Sensitive Networking)**: 이더넷 기반으로 결정적 동작을 제공하는 차세대 기술로, 산업용, 자동차, 항공など에서 채택이 확산되고 있다.
- **Deterministic Networking**: IETF가 표준화 중인 Deterministic Networking (DetNet)은 토큰 링과 유사하게 보장된 지연과 대역폭을 제공한다.

- **📢 섹션 요약 비유:** 토큰 링의 정신은 자동차의 FlexRay나 산업용 CAN 버스로 이어졌으며, 또한 차세대 TSN에서 부활하고 있습니다. 결국 네트워크 기술은 순환하고 있어, 한 번 잊어버린技術이라도 미래에 다시 필요하다고 판단되면환생할 수 있습니다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **IEEE 802.5** | Token Ring의 표준으로, 현재는 사용되지 않는 역사적 표준이다. |
| **MAU (Multistation Access Unit)** | 토큰 링에서 사용되는 집중 장비로, 각 노드를 논리적 링에 연결한다. |
| **FDDI (Fiber Distributed Data Interface)** | 광섬유 기반의 토큰 전달 LAN으로, 고속 백본망에서 사용되었으나 이더넷에 대체되었다. |
| **TSN (Time-Sensitive Networking)** | 이더넷 기반으로 결정적 동작(보장된 지연, 저지터)을 제공하는 IEEE 표준 집합이다. |
| **CAN (Controller Area Network)** | 자동차 내부 네트워크용 버스 프로토콜로, 토큰 없는 priority 기반 arbitration을使用한다. |

## 👶 어린이를 위한 3줄 비유 설명
1. 토큰 링은 피차트（ кру́г）游戏的"말뚝"과 같습니다. 말뚝을 가진 아이만 말할 수 있고, 다 마르면 옆 아이에게 말뚝을 건네줍니다. 그래서 둘이 동시에 말해서 아무도 못 알아듣는(충돌) 수가 없어요.
2. 하지만 피차트에는主席(Active Monitor)가 있어서, 말뚝이 분실되면 새로 만들고, 놀龄 아이가 있으면 制해줍니다. 그래서 안정적으로 돌아가지만,管理하기가 복잡하죠.
3. 결국 이더넷"아니租马车"가 나오면서 가격이 싸고管理easier 탓에 승차거부권은 사라졌어요. 하지만 오늘날 다시"전세버스"(TSN)가 나타나, 정해진ルート를 정해진 時間에 돌며predictableなサービス를 제공하고 있습니다.
