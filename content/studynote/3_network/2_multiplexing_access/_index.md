# 2. 다중화 및 다중접속 기술

## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 다중화(Multiplexing)는 하나의 고속传输 통로에 여러 저속 신호를 합치는 기술이며, 다중접속(Multiple Access)은 동일한 공유 매체를 여러 사용자가时分/주파수/코드로 구분하여 사용하는 기술이다.
> 2. **가치**: 효율적인 다중화/다중접속은 전송 인프라 비용을 N분의 1로 절감하면서도 각 사용자에게 고품질 서비스를 제공할 수 있게 한다.
> 3. **융합**: 현대 무선 통신(4G/5G/6G)에서 다중화 기술은OFDMA와 대규모 MIMO의 핵심 구성요소로, 주파수 효율 극대화의 열쇠이다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념 정의: 다중화와 다중접속의 본질

다중화(Multiplexing)와 다중접속(Multiple Access)은 흔히 혼동되지만, 개념적으로 명확한 차이가 있다. 다중화는 복수의 신호 источник에서 나온 신호를 하나의 공통传输 매체(Transmission Medium)에 결합하는 기술로, 주로 线路上的 병렬적 전송을 위해 사용된다. 즉, multiplexer(MUX)가 입력侧的 N개 채널을 수신하여 하나의 복합 신호(Composite Signal)를 생성하고, 이 신호가 전송 매체를 통해 이동한 후, demultiplexer(DEMUX)가 복합 신호를 다시 N개의 출력 채널로 분리한다. 다중접속은 다수의 사용자가 동일한 공유 매체(Shared Medium)에 동시에 접근할 수 있도록 하는 기술로, 무선 통신에서 특히 중요하게 활용된다. TDMA, FDMA, CDMA, OFDMA 등이 대표적인 다중접속 방식이다.

### 필요성: 한정된 자원의 효율적 활용

전자파 스펙트럼(Spectrum)은有限自然资源으로, 사용할 수 있는 주파수 범위가 물리적으로 정해져 있다. 세계 각국의 정부 기관(미국의 FCC, 한국의 MSIT 등)이 주파수 대역을分配하고 라이선스를 부여하여 comercial usage과Military/공익用途의 충돌을 방지한다. 그러나 스마트폰과 IoT 기기의 폭발적 증가로 인한 데이터 트래픽 수요는 라이선스 대역만으로는 감당하기 어렵고, 특히 도시 지역에서는 스펙트럼 부족 문제가 심각하다. FCC의 조사 따르면 평균적으로 활용되는 스펙트럼은 전체의 15~85% 수준에 불과하며, 이 비효율성을 해소하는 것이 다중화/다중접속 기술 발전의根本的动力이다.

### 역사적 등장 배경: 전화 교환에서 디지털 다중화로

초기 전화 시스템(1876년 Alexander Graham Bell의 발명 이후)에서는 각 가정에 별도의 전선을引く 방식(직렬 연결)으로 통신했다. 그러나 이 방식은 전선 비용이 用户 수의 제곱에比例하여 증가하는 비효율성이 있었다. 1876년 Charles A. Simple Multiplexing 개념이 처음으로 제시되었고, 1910년대에 FDM (Frequency Division Multiplexing)이 전화 trunk 회선에 적용되어 수십 통화 채널을 하나의 전선으로 동시에 전송할 수 있게 되었다. 1960년대에는 아날로그에서 디지털로의 전환이 시작되어, TDM (Time Division Multiplexing)이主流이 되었고, 이것이 이후 PCM (Pulse Code Modulation)과 Digital Cross Connect 시스템으로 발전했다.

### 스펙트럼 공유의 기본 원리

무선 통신에서 가장 중요한 자원인 주파수 대역을 효과적으로 공유하기 위한 세 가지 기본 원칙이 있다. 첫째, 주파수 분할(Frequency Division)은 전체 대역을 여러 하위 대역으로分割して、各 利用자에게 전용 채널을 부여한다. 둘째, 시간 분할(Time Division)은 모든 이용자가同一 주파수를 사용하지만, 각자에게 특정 시간 슬롯만 할당한다. 셋째, 코드 분할(Code Division)은 모든 이용자가同一 시간과同一 주파수를 사용하지만, 각자에게 고유한 코드 시퀀스를 부여하여 수신侧에서 원하는 신호만 추출할 수 있게 한다. 세 방식 모두 장단점이 있으며, 현대 시스템에서는 이들을Hybrid하여 최적의 효율을 달성한다.

📢 **섹션 비유**: 다중화는高速公路의HOV (High Occupancy Vehicle) 차선과 같다. HOV 차선은 차에 사람 2인 이상만 입장할 수 있어서 한 차선을更多人이 효율적으로 활용하는 것처럼, 다중화도 하나의传输 통로를更多人/신호가 함께 사용하게 한다.

### ASCII: 다중화와 다중접속의概念적 차이

이 도식은 다중화와 다중접속의概念적 차이를 명확히 보여준다.

```
다중화 (Multiplexing): 복수의 신호를 하나의 전송로를 통해 전송
[신호 A] ─┐
[신호 B] ─┼──> [MUX] ──> [전송로 (단일)] ──> [DEMUX] ─┬─> [신호 A]
[신호 C] ─┘                                      └─> [신호 B]
                                                      └─> [신호 C]
예: TDM, FDM, OFDM

다중접속 (Multiple Access): 복수의 사용자가 하나의 공유 매체에 접근
[사용자 1] ─┐
[사용자 2] ─┼──> [다중접속 장치] ──> [공유 매체 (무선)] ──> [기지국]
[사용자 3] ─┘
예: TDMA, FDMA, CDMA, OFDMA
```

**다이어그램 해설**: 다중화는 점대점(Point-to-Point) 통신에서 효율을 높이기 위한 기술로, multiplexer와 demultiplexer가 반드시 쌍으로 존재한다. 반면 다중접속은 점대다점(Point-to-Multipoint) 또는 다점대다점(Multipoint-to-Multipoint) 통신에서 다수의 사용자가 매체를 공유하기 위한 기술이다. 유선 통신에서는 채널 대 여분이 있어 크게 중요하지 않지만, 무선 통신에서는 한정된 스펙트럼 자원을 최대한 효율적으로 활용해야 하므로 다중접속 기술이 핵심적이다. 현대 스마트폰 통신에서 기지국이 수천 명의 사용자를 동시에服务하려면, 이 다중접속 기술의 효율이直接적으로 시스템吞吐量과 사용자待ち時間에 영향을 미친다.

### 시분할 다중화 (TDM)의 동작 원리

시분할 다중화(TDM: Time Division Multiplexing)는 각 입력 신호에게周期性적인 시간 슬롯(Time Slot)을 할당하여, 하나의 전송로를轮流で使用する 방식이다. 기본 개념은 cinema 영화 필름과 유사하다. cinema에서는 24프레임/초의 속도로 정지画像を연속적으로 표시하여 연속된 동작을 느끼게 하지만, 실제로는 각 순간은 개별적인画像이다. TDM도 마찬가지로, N개의 입력 채널이 각각 짧은 시간만 전송 기회를 얻고, 수신側에서는 언제 어떤 채널의 데이터가 오는지를 알 수 있어 복원이 가능하다. TDM의 핵심 요구사항은 엄격한 클록 동기화이다. 송신과 수신의 타이밍이 조금만 어긋나ても 슬롯 경계가 어긋나完整的 신호 복원이 불가능해진다.

### ASCII: TDM (Time Division Multiplexing)의 프레임 구조

이 도식은 4개의 입력 채널이 TDM 방식으로 하나의传输로에 시간적으로 분할되어 전송되는 모습을 보여준다.

```
입력 채널:
채널 1: [A1] [A2] [A3] [A4] [A5] [A6] ...
채널 2: [B1] [B2] [B3] [B4] [B5] [B6] ...
채널 3: [C1] [C2] [C3] [C4] [C5] [C6] ...
채널 4: [D1] [D2] [D3] [D4] [D5] [D6] ...

TDM 프레임 구조 (한 프레임 = 4 슬롯):
┌────────┬────────┬────────┬────────┬────────┬────────┬────────┬────────┐
│  A1    │  B1    │  C1    │  D1    │  A2    │  B2    │  C2    │  D2    │ ...
└────────┴────────┴────────┴────────┴────────┴────────┴────────┴────────┘
슬롯 1    슬롯 2   슬롯 3   슬롯 4   슬롯 1'  슬롯 2'  슬롯 3'  슬롯 4' (다음 프레임)

동기화 비트 (프레임 시작 표시):
┌──┐
│SYNC│ A1   B1   C1   D1   A2   B2   C2   D2
└──┴─────────────────────────────────────────────────> 시간
```

**다이어그램 해설**: TDM의 핵심 설계 문제는 프레임 동기화(Frame Synchronization)이다. 수신 측은 어디서到一个 프레임이 시작되고, 어디서 끝나는지를 정확히 알아야 각 슬롯을 올바른 채널에 배당할 수 있다. SYNC 비트가 바로 이 프레임 경계를 알려주는 역할学이다. 만日 SYNC 비트가 손상되면 전체 프레임의 슬롯이 한 칸씩 밀리는 "프레임 슬립(Frame Slip)" 현상이 발생하여 모든 채널의 데이터가 깨진다. 실무에서 TDM 시스템의可靠性は 이 동기화 메커니즘의 견고성에 크게 좌우된다.

### OFDM (직교 주파수 분할 다중화)의 동작 원리

OFDM (Orthogonal Frequency Division Multiplexing)은 수십에서 수천 개의 직교(Orthogonal) 하위 반송파(Subcarrier)에 데이터를 할당하는 고급 다중화 기술이다. "직교"란 수학적 성질로, 각 하위 반송파의 중심 주파수가 다른 하위 반송파의零点(Zero Point)에 위치하여 서로 간에干渉(Interference)을 최소화한다. 이를 통해 하위 반송파 간에 가드 밴드(Guard Band)를 둘 필요가 없어져 주파수 효율이 크게 향상된다. OFDM은 FFT (Fast Fourier Transform) 알고리즘으로実装되어, 고속 연산이 가능하고, 순환 접두사(CP: Cyclic Prefix)를 통해 다중 경로 페이딩(Multipath Fading)에 강한 내성을 가진다.

### ASCII: OFDM의 직교 하위 반송파 구조

이 도식은 OFDM에서 하위 반송파들의 직교 관계와 FFT 기반 처리를 보여준다.

```
주파수 영역에서 본 OFDM 하위 반송파들:

주파수 ─────────────────────────────────────────────────────────>
            │
    f0      │   f1      f2      f3      f4      f5      f6
    │        │    │       │       │       │       │       │
    ▼        ▼    ▼       ▼       ▼       ▼       ▼       ▼
  ┌──┐  ┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐
  │  │  │  ││  ││  ││  ││  ││  ││  ││  ││  ││  ││  │  <- 각 사각형: 하나의 하위 반송파
  └──┘  └──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘
           │    │       │       │       │       │       │
           ▼    ▼       ▼       ▼       ▼       ▼       ▼
         Sinc  Sinc    Sinc    Sinc    Sinc    Sinc    Sinc   <- 각 주파수의 주파수 응답
                  ↓                    ↓                    ↓
              첫 번째          두 번째           세 번째
              영점           영점 (직교 조건)     영점

핵심: 각 하위 반송파의 중심 주파수가 다른 하위 반송파의 영점에 위치하여 직교성 확보
```

**다이어그램 해설**: OFDM의 직교성은 수학적으로 증명된 성질로, 각 하위 반송파가 다른 하위 반송파와 중첩되어도 상호干渉을 일으키지 않는다. 이것이 가능하려면 하위 반송파 간격(Subcarrier Spacing)이 정확히 FFT bin 폭과 일치해야 하며, 이것은 송신과 수신의 FFT 처리에서 보장된다. 그러나 이것은 이상적인 조건에서であり, 실제 무선 채널에서는 도플러 전移(Doppler Shift)와振荡器 오류(Oscillator Error)로 인해 직교성이 깨져서 ICI (Inter-Carrier Interference)가 발생한다. 5G NR에서는 이 문제를 해결하기 위해 하위 반송파 간격을 유연하게 설정(15kHz, 30kHz, 60kHz, 120kHz, 240kHz)하여 다양한 전이 환경과 서비스 요구사항에 대응한다.

### CDMA (코드 분할 다중접속)의 동작 원리

CDMA (Code Division Multiple Access)는 모든 사용자가同一 시간과同一 주파수를 공유하지만, 각자에게 고유한 코드 시퀀스(Code Sequence)를 부여하여 신호를 구분하는 방식이다. 기본 원리는 각 비트 시간을 더 작은 칩(Chip) 단위로 나누고, 각 사용자에게 고유한 칩 시퀀스를 할당하는 것이다. 예를 들어, 사용자 A의 코드가 [+1, -1, +1, -1]이고 데이터가 "1"이면 [+1, -1, +1, -1]를 보내고, 데이터가 "0"이면 [-1, +1, -1, +1](反転)を送信する。受信侧では、ユーザーが自分のコードで受信信号을 내적하면, 자신의 데이터만抽出되고 다른 사용자의 신호는理论上 0이 된다. 이 원리를 "확산 스펙트럼(Spread Spectrum)"이라고도 하며,Military 통신에서敵의 방해(Jacking)를 견디기 위해 开发되었다.

### ASCII: CDMA의 코드 확산과 복원 과정

이 도식은 CDMA에서 송신侧에서 코드로 신호를 확산하고, 수신侧에서 고유 코드로 신호를 복원하는 과정을 보여준다.

```
송신 과정 (Spread Spectrum):
데이터 비트: 1 → [+, +, +, +] (4 chips로 확산)
사용자 A 코드: [+, +, -, -]
사용자 B 코드: [+, -, +, -]

A의 송신 신호: 데이터 × A코드 = [+, +, -, -] (스프레딩)
B의 송신 신호: 데이터 × B코드 = [+, -, +, -] (스프레딩)

합성 신호 (A+B): [+2, 0, 0, -2]  <-- 실제로는 RF로 변조되어 전송

수신 과정 (De-spreading):
A의 수신: 합성 신호 × A코드 = [+2, 0, 0, -2] · [+, +, -, -]
       = (2×1) + (0×1) + (0×-1) + (-2×-1) = 2 + 0 + 0 + 2 = 4

A의 수신 데이터: 4 > 0 → 1 (성공적 복원)

B의 수신: 합성 신호 × B코드 = [+2, 0, 0, -2] · [+, -, +, -]
       = (2×1) + (0×-1) + (0×1) + (-2×-1) = 2 + 0 + 0 + 2 = 4

B의 수신 데이터: 4 > 0 → 1 (간섭 없이 복원)
```

**다이어그램 해설**: CDMA의 핵심은 직교 코드(Orthogonal Code)를 사용한다는 점이다. 두 코드 [+, +, -, -]와 [+, -, +, -]의 내적(dot product)이 0이 되도록設計되어 있으면, 수신侧에서 자기 코드로 내적할 때 다른 사용자의 신호는完全に 相殺된다. 그러나 실제 환경에서는 다중 경로衰落(Multipath Fading), 비동기 전송, 반송파 주파수 오차 등으로 인해 완전한 직교성이 유지되지 않아 N개의 사용자가 동시에 전송하면 이론 대비 성능이 저하된다. 이것이 CDMA 시스템에서 "능동 사용자 수 제한(Soft Capacity)"이 존재하는 이유이며, 3G UMTS 시스템에서 RRM (Radio Resource Management)이 중요하게 고려하는 사항이다.

📢 **섹션 비유**: OFDM은 오케스트라 연주에 비유할 수 있다. 오케스트라에서 각 악기(하위 반송파)는 서로 다른 멜로디(데이터)를 연주하지만, 모두 동일한 시간에 연주되고(동일 주파수 대역)，观众(수신자)은 특정 악기(특정 하위 반송파)의 소리만 듣고 싶을 때 그 악기의旋律(코드)에 집중할 수 있다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 주요 다중화/다중접속 방식의 정량적 비교

| 방식 | 주파수 효율 | 시간 지연 | 복잡도 | 대표 적용 |
|:---|:---|:---|:---|:---|
| FDM | 낮음 (가드 밴드 필요) | 낮음 | 낮음 | FM 라디오, TV 방송 |
| TDM | 중간 | 중간 | 중간 | PSTN, T1/E1, SDH |
| CDM (CDMA) | 높음 | 높음 (칩 시퀀스 길이) | 높음 | 3G UMTS, GPS |
| OFDM | 매우 높음 | 높음 (CP 오버헤드) | 높음 (FFT) | Wi-Fi, LTE, 5G NR |
| OFDMA | 매우 높음 (동적 할당) | 낮음 (스케줄링) | 매우 높음 | LTE, 5G NR, Wi-Fi 6 |

### OFDM과 OFDMA의 비교

OFDM은 다중화를 위한 기술이고, OFDMA는 OFDM 기반의 다중접속 기술이다. OFDM에서 모든 하위 반송파가 하나의 사용자(또는 스트림)에게 할당되는 반면, OFDMA에서는 하위 반송파들을 여러 사용자 그룹으로 나누어 동시에 서비스한다. OFDMA의 핵심 장점은動的 스케줄링(Dynamic Scheduling)으로, 각 사용자의 트래픽 요구와 채널 상태에 따라 자원(하위 반송파 + 시간 슬롯)을リアルタイムで割り当てることで 전체 시스템呑吐量を 극대화한다. LTE에서는 하위 반송파 간격 15kHz, 1ms 서브프레임 내에서 주파수 도메인(하위 반송파)과 시간 도메인(슬롯)을 2차원으로 스케줄링하며, 5G NR에서는 더 큰 하위 반송파 간격과 다양한 스케줄링 유연성을 지원한다.

### ASCII: OFDM 대 OFDMA 할당 비교

이 도식은 OFDM(단일 사용자)과 OFDMA(다중 사용자)에서의 자원 할당 방식을 비교한다.

```
OFDM (단일 사용자, 전체 대역 사용):
주파수 ─────────────────────────────────────────────────────────>
        ├──Sb0──┤├──Sb1──┤├──Sb2──┤├──Sb3──┤├──Sb4──┤├──Sb5──┤  ... │<- 전체 하위 반송파
        ├────────── 사용자 1 (전체 사용) ─────────────────────────┤

OFDMA (다중 사용자, 주파수/시간 2D 할당):
주파수 ─────────────────────────────────────────────────────────>
        ├──Sb0──┤├──Sb1──┤├──Sb2──┤├──Sb3──┤├──Sb4──┤├──Sb5──┤
시간 ───┬────────┬────────┬────────┬────────┬────────┬────────┤
슬롯 1  │ 사용자1 │ 사용자2 │ 사용자3 │ 사용자1 │ 사용자2 │ 사용자4 │
슬롯 2  │ 사용자2 │ 사용자1 │ 사용자4 │ 사용자3 │ 사용자2 │ 사용자1 │
슬롯 3  │ 사용자3 │ 사용자4 │ 사용자1 │ 사용자2 │ 사용자3 │ 사용자4 │
        └────────┴────────┴────────┴────────┴────────┴────────┘
```

**다이어그램 해설**: OFDMA의 스케줄링은기지국이 매 슬롯(보통 1ms)마다 각 사용자의 CSI (Channel State Information)에 따라 동적으로 자원 할당을 결정한다. 채널 상태가 좋은 사용자에게 더 많은 하위 반송파를 할당하는 것이Adaptive Modulation and Coding (AMC)과 결합되어 시스템 전체 throughput을 극대화한다. 그러나 이 동적 할당에는Scheduling Overhead가 따르며, LTE에서 PDCCH (Physical Downlink Control Channel)가 이 스케줄링 정보를 전송한다. 5G NR에서는 더 유연한 스케줄링과 Grant-Free(스케줄링 없이 사전 설정된 자원으로 데이터 전송) 방식도 지원하여 초저지연 통신(URLLC)을 가능하게 한다.

### 컴퓨터 아키텍처와의 융합: PCIe의 레인 다중화

컴퓨터 내부 버스인 PCIe (Peripheral Component Interconnect Express)도 다중화 기술을 사용한다. PCIe는 레인(Lane) 단위로 동작하며, 각 레인은 송수신 쌍( TX+, TX-, RX+, RX-)으로 구성된다. x4 설정에서는 4개의 레인이 병렬로 사용되어 4배의 대역폭을 제공하며, 이것은 물리적 레인 다중화(Physical Lane Multiplexing)에 해당한다. 상위 레벨에서는 MSI-X (Message Signaled Interrupt) 기술을 통해 interrupt 메시지 자체를 데이터로 패키징하여 별도의 인터럽트 라인을 줄이고, 이것은Modern I/O 시스템의 자원 효율성을 높이는 핵심 기법이다.

### Operating System와의 융합: CPU 시간 분할 스케줄링

다중화 개념은 Operating System의 CPU 스케줄링에도 적용된다. Round-Robin 스케줄링은 TDMA의 개념과 유사하게 각 프로세스에게 동일한 시간 퀀텀(Time Quantum)을轮流 부여하며, Multi-Level Feedback Queue (MLFQ)는 프로세스의 특성에 따라 다른 시간 퀀텀을 부여하여 반응성과 효율성 사이의 트레이드오프를 관리한다.实时 운영체제(Real-Time OS)에서는 TDMA 기반의固定 우선순위 스케줄링이 보장된 응답 시간을 제공하며, 이것은 산업용 제어 시스템과 automobileECU에서 필수적인 요소이다.

### ASCII: OS 스케줄링과 TDMA의 비교

이 도식은 Operating System의 CPU 스케줄링과 TDMA 다중화의相似성을 보여준다.

```
TDMA (통신):
슬롯 1 (사용자 A) ──┐
슬롯 2 (사용자 B) ──┼──> 반복 프레임
슬롯 3 (사용자 C) ──┘

CPU Round-Robin:
프로세스 A (퀀텀 10ms) ─┐
프로세스 B (퀀텀 10ms) ─┼──> 반복 사이클
프로세스 C (퀀텀 10ms) ─┘

동일점:
- 모든 참가자가 동일한 시간 할당
- 순환적 round-trip 구조
- 지연 시간이 참가자 수에 비례

차이점:
- TDMA: 고정 슬롯 크기,專用 채널
- RR: 가변 퀀텀 가능, 공용 CPU 자원
```

**다이어그램 해설**: TDMA와 OS 스케줄링의根本적 동일점은 시간 자원을 여러 참가자 사이에서 분할한다는 것이다. 그러나 중요한 차이점도 있다. TDMA에서는 각 사용자에게 할당된 슬롯이 완료되지 않아도(데이터가 없어도) 슬롯이浪费되고, 이는通信効率의 손실로 이어진다. 반면 OS 스케줄링에서는 프로세스가 I/O 대기로 블록되면 즉시 다른 프로세스가 CPU를 인계받아 효율적인 자원 활용이 가능하다. 이 차이는通信 시스템에서는 슬롯이 시간 대기가 아닌專用 자원이기 때문이며, OS에서는CPU 시간이共有 자원이라는 근본적 차이에서 비롯된다.

📢 **섹션 비유**: FDM은 서로 다른 층에 사는 사람들이 각각 전용 엘리베이터를 사용하는 것과 같다. 각 층(주파수 대역)의住户(신호)는自己的 엘리베이터(반송파)를 사용하므로 충돌이 없지만, 전체 건물의 엘리베이터 수가 줄어든다. OFDM은 여러 층 사람들이 하나의 엘리베이터Shafts를 공유하면서도, 각 층에 도착하는 시간을 다르게 배정하여 충돌을 피하는 것과 같다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오: 5G NR에서 OFDMA 스케줄링 전략

5G NR 네트워크에서 기지국이 대규모 MIMO와 OFDMA를 활용하여 다수의 사용자에게 서비스를 제공하는 상황을 고려해보자. 핵심 문제는 주파수 자원과 시간 자원을 어떻게 최적하게 할당하느냐이다. eMBB (Enhanced Mobile Broadband) 서비스에서는 throughput 극대화가 목표이므로, CSI가 좋은 사용자에게 더 많은 자원 블록(Resource Block)을 할당하는 Frequency Selective Scheduling이 유리하다. 반면 URLLC (Ultra-Reliable Low-Latency Communications) 서비스에서는 지연 시간 최소화가 목표이므로, Grant-Free 방식으로 사전 할당된 자원에 즉시 데이터 전송이 가능해야 하고, Semi-Persistent Scheduling (SPS)을 활용하여 periodic 트래픽에 대한 오버헤드를 줄인다.

### 도입 체크리스트: 다중화/다중접속 기술 도입

| 확인 항목 | 세부 내용 | 판단 기준 |
|:---|:---|:---|
| 서비스 유형 | eMBB, URLLC, mMTC 중 주요 서비스 | 요구 latency, throughput에 따라 다중화 방식 결정 |
| 주파수 대역 | Sub-6GHz vs mmWave | mmWave는 더 넓은 대역폭 but 커버리지 좁음 |
| 사용자 수 | 동시 접속 예상 사용자 수 | 다중접속 용량规划的基础 |
| 이동성 | 정적/보행/차량 속도 | 도플러preading에 따른 하위 반송파 간격 결정 |
| 하드웨어 능력 | FFT 크기, RF 체인 수 | OFDM/OFDMA 구현 가능 여부 |

### 안티패턴: 흔한 설계 실수와 치명적 결함

**안티패턴 1: OFDM에서 사이클릭 prefix 길이 과소평가**
很多 엔지니어가 OFDM 시스템에서 CP (Cyclic Prefix) 길이를 가능한 짧게 설정하여 스펙트럼 효율을 높이려고 하지만, 이것은 다중 경로遅延에 의한 ISI (Inter-Symbol Interference)를 유발한다. CP 길이는 최대 다중 경로 지연spread보다 길어야 하며, LTE의 경우 일반 CP에서 4.69μs, 확장 CP에서 16.67μs로 설정되어 있다. 도시 환경에서는 Building反射으로 인해 지연spread가 클 수 있으므로, CP 길이 선택은 실제 전파 환경 분석을 기반으로 해야 한다.

**안티패턴 2: CDMA에서 소프트 핸드오버 무시**
CDMA 시스템에서 소프트 핸드오버(Soft Handover)는 두 기지국으로부터 동시에 데이터를 수신하여 품질을 향상시키지만, 이로 인해 Resource consumption이 증가한다. 일부 엔지니어가 시스템 용량 부족 문제를 해결하기 위해 소프트 핸드오버 영역을 줄이는데, 이것은 셀 가장자리에서 사용자의服务质量(QoS)을 급격히 저하시킨다.CDMA의 반대급부로, 소프트 섹터링(Soft Sectoring)과 엄격한 Call Admission Control (CAC)이 함께 적용되어야 하며, 이것은 시스템 설계 시 같이 고려되어야 한다.

### ASCII: 무선 시스템 핸드오버 종류 비교

이 도식은 CDMA 시스템에서의 소프트 핸드오버와 LTE/Femtocell 시스템에서의 핸드오버를 비교한다.

```
CDMA 소프트 핸드오버:
        [사용자]
       ╱        ╲
    기지국 A    기지국 B
      │             │
      └──────┬──────┘
             ▼
      합성기 (Rake Receiver)
      - 두 기지국 신호 동시 수신
      - 다이버시티 효과
      - 자원 소비 2배

LTE/Femtocell 래핑 핸드오버:
        [사용자]
           │
       기지국 A  ──X--> 기지국 B
                   (기존 연결 해제 후 새로운 연결 수립)
```

**다이어그램 해설**: 소프트 핸드오버는 CDMA의 가장 큰 장점 중 하나로, 두 기지국으로부터 동시에 데이터를 수신하여 신호 품질을 향상시킨다. 그러나 이것은 동시에 두 기지국의 역방향 링크 자원을 점유하므로 시스템 용량 계획에서 반드시 고려해야 한다. 반면 LTE의 래핑 핸드오버는 이전 연결을 해제하고 새로운 연결을 수립하므로 자원 소비는 적지만, 핸드오버 중短暂的 서비스 중단이 발생할 수 있다. 5G NR에서는 双 연결(Dual Connectivity)과 Master/Slave 기지국 구분을 도입하여 소프트 핸드오버의 장점을 취하면서도 mmWave와 Sub-6GHz의 이종 네트워크를 통합 관리할 수 있게 했다.

📢 **섹션 비유**: 다중화 기술 선택은 식당에서 테이블 예약을 하는 것과 같다. FDM은 각 테이블이 다른 층에 있어服务员이專門으로 관리하지만, 손님이 아무리 와도 이미 차eling된 테이블은浪费된다. OFDMA는 한 층의 테이블을 여러 组이 시간별로轮流 사용하지만, 各 组의 arriv час와滞在 시간에 맞게 효율적으로 배정한다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 정량/정성 기대효과

| 효과 지표 | 도입 전 | 도입 후 | 변화율 |
|:---|:---|:---|:---|
| 주파수 효율 (bps/Hz) | 1 (FDM 기반) | 5~10 (OFDM/OFDMA) | 5~10배 향상 |
| 사용자 동시 접속 수 | 50 (TDMA 기반) | 500+ (OFDMA) | 10배 이상 |
| 평균 지연 (ms) | 50 | 1~5 | 90% 감소 |
| 스펙트럼 활용률 | 15~30% | 60~80% | 2~3배 향상 |

### 미래 전망: 6G에서 비직교 다중접속 (NOMA)

6G 무선통신에서는 NOMA (Non-Orthogonal Multiple Access)가 차세대 다중접속 기술로 연구되고 있다. 기존 OFDMA가 하위 반송파를 직교(排他的)로 할당하여 한 사용자가 한 하위 반송파를 독점하는 반면, NOMA는 복수의 사용자가同一 하위 반송파를 공유한다. 수신側에서는 SIC (Successive Interference Cancellation) 기술을 사용하여强者 신호부터順に 복호화하여 자기 신호를 추출한다. NOMA의 장점은 스펙트럼 효율의 이론적 극대화와, 셀 가장자리 사용자와 근처 사용자에게동시에 서비스를 제공하여公平성을 높일 수 있다는 점이다. 그러나 SIC의 오류 전파(Error Propagation)와 구현 복잡도가 주요 과제이다.

### 관련 표준 및 참조 규격

- 3GPP Release 15/16/17: LTE, 5G NR 표준
- IEEE 802.11ax (Wi-Fi 6): OFDMA 기반 Wi-Fi 표준
- IEEE 802.11be (Wi-Fi 7): 더 넓은 채널 폭과 더 많은 RU (Resource Unit)
- ITU-R M.2083: IMT-2020 (5G) 요구사항
- 3GPP TR 38.912: 5G NR Study Item

### ASCII: 다중접속 기술의 진화 로드맵

이 도식은移动通信에서 다중접속 기술의 발전 과정을 보여준다.

```
1G (1980s): FDMA
  - 아날로그 음성 중심
  - 주파수 하나당 한 통화
  - 낮은 용량, 낮은 품질

2G (1990s): TDMA + FDMA
  - 디지털 음성 (GSM)
  - TDMA로 시간 분할 (8 슬롯/반송파)
  - FDMA로 주파수 분할
  - SMS 등 데이터 서비스 가능

3G (2000s): CDMA
  - 광대역 CDMA (WCDMA)
  - 코드 분할로 동시 사용자 증가
  - 소프트 핸드오버
  - 데이터 서비스 본격화

4G (2010s): OFDMA + SC-FDMA
  - LTE의 하향/상향 다중접속
  - FFT 기반高效 구현
  - IP 기반全 IP 네트워크

5G (2020s): OFDMA + Massive MIMO
  - 5G NR의 유연한 스케줄링
  - Massive MIMO (64~256 안테나)
  - eMBB, URLLC, mMTC 동시 지원
  - mmWave 활용

6G (2030s~): NOMA + RIS
  - 비직교 다중접속 (NOMA)
  - Reconfigurable Intelligent Surface (RIS)
  - Terahertz (THz) 통신
  - AI-native 네트워크
```

**다이어그램 해설**: 각 세대의 다중접속 기술 발전은 궁극적으로 주파수 효율 극대화와 사용자 수 증가를 추구하는 과정이다. FDMA에서 TDMA로의 전환은 시간 자원의 효율적 활용을, TDMA에서 CDMA로의 전환은 역방향 호환성과 소프트 핸드오버 장점을 취한 것이었다. CDMA에서 OFDMA로의 전환은 FFT라는 효율적 알고리즘으로 구현이 가능해지면서 스펙트럼 효율이 획기적으로 향상된 것이다. 6G에서는 직교성 자체를 과감히抛弃하는 NOMA로的方向으로 연구되고 있으며, AI/ML 기반 동적 스케줄링과 Reconfigurable Intelligent Surface (RIS) 기술이 결합되어 궁극적인 스펙트럼 효율 향상이 기대된다.

📢 **섹션 비유**: 다중화 기술의进化는 우편 서비스의 발전과 같다. 처음에는 편지 하나에 전용 우체부가 있었고(FDMA), 나중에는 여러 편지를 하나의快递盒에 담는 방법(TDMA)이 개발되었고, 지금은 여러 사람からの小包을 RFID로 추적しながら 최적의 경로로配送する(OFDMA) 시스템이 되었다.

## 👶 어린이를 위한 3줄 비유 설명

**첫째**, 다중화는 여러 명이 동시에 통화할 수 있는 특별 전화기에 비유할 수 있어요. 보통 전화기는 한 번에 한 사람만 통화할 수 있지만, 회의 전화기는 여러 사람이 동시에 이야기할 수 있죠.

**둘째**, OFDM은 오케스트라 연주에 비유할 수 있어요. 오케스트라에는 비올라, 첼로, 트럼펫 같은 많은 악기들이 있는데, 모두 동시에 다른 음악을 연주하지만 지휘자(수신자)가 들으려는 악기 소리에 집중하면 각 악기의 멜로디를 알아들을 수 있어요.

**셋째**, Wi-Fi나 LTE를 쓰는 우리 스마트폰이 빠르게 인터넷을 쓸 수 있는 것은 바로 이 다중화 기술 덕분이예요. 만약 이 기술이 없으면 우리 모두 하나의 주파수를轮流使用해야 해서 엄청 느려지고,很多人이 동시에 쓸 수 없어요.
