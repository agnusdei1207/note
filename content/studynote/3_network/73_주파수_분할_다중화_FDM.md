# 73. 주파수 분할 다중화 (FDM, Frequency Division Multiplexing)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 주파수 분할 다중화(FDM)는 거대한 전체 전송 대역폭을 아주 얇은 여러 개의 **'주파수 차선(Channel)'으로 쪼개서**, 수많은 저속 신호들을 겹치지 않게 동시에 흘려보내는 전통적인 아날로그 다중화 기술이다.
> 2. **원리 (아날로그 변조)**: 사용자 A의 데이터는 100MHz 반송파에 태우고, 사용자 B의 데이터는 101MHz 반송파에 태우는 식으로 각자 다른 음역대(주파수)를 배정해 허공이나 케이블에 한꺼번에 섞어 쏜다.
> 3. **실무 융합**: 자동차 라디오(FM 89.1, FM 91.9), 아파트 케이블 TV(Ch 7, Ch 11), 초창기 1G 아날로그 휴대전화망의 근간 아키텍처이며, 이 차선 사이에는 충돌을 막기 위해 낭비되는 빈 공간인 **보호 대역(Guard Band)**이 반드시 존재해야 한다.

---

## Ⅰ. FDM의 개요 및 필요성 (Context & Necessity)

- **개념**: 전송 매체(동축케이블, 공기 등)가 제공하는 아주 넓은 총 대역폭을 수많은 좁은 대역폭 채널(Channel)로 논리적으로 분할하여 여러 사용자에게 하나씩 할당하는 기술.
- **필요성**: 
  - 과거 아날로그 시대에는 데이터를 0과 1의 펄스로 쪼개어 번갈아 쏘는 TDM(시분할) 같은 고급 제어 기술이 없었다.
  - 전선 1개로 여러 명의 전화를 보내려면 가장 무식하고 확실한 방법이 필요했다. "남자 목소리, 여자 목소리, 아기 목소리가 겹쳐서 들려도 사람은 다 구별할 수 있잖아? 그럼 **전화 1번은 10kHz 고음으로 쏘고, 전화 2번은 20kHz 초고음으로 쏴서 섞어버리자!**" 이것이 FDM의 탄생 철학이다.

- **💡 비유**: **FDM은 '오케스트라 합주와 귀의 필터링'**과 같다.
  - 콘서트홀(전송 매체)에서 바이올린(고음), 첼로(중음), 콘트라베이스(저음)가 동시에 연주를 한다.
  - 공기 중에는 세 악기의 소리가 완벽하게 하나로 뭉쳐서(다중화) 날아온다.
  - 하지만 내 귀(수신기)는 뇌의 필터(Bandpass Filter)를 통해 섞여 있는 소리 중 바이올린의 고음만 따로 뽑아서 감상할 수 있다. 서로 '음역대(주파수)'가 완전히 달라서 겹치지 않기 때문이다.

## Ⅱ. 핵심 아키텍처 및 메커니즘 (Deep Dive)

### 1. FDM의 송수신 파이프라인 (Modulation & Filtering)
FDM 아키텍처를 구현하려면 반드시 '변조기(Modulator)'와 '필터(Filter)'가 세트로 필요하다.

1. **송신 (MUX)**:
   - 유저 1, 2, 3의 원본 데이터(모두 0~4kHz의 똑같은 목소리 대역)가 들어온다.
   - 변조기가 유저 1은 $f_1$, 유저 2는 $f_2$ 반송파 로켓에 태워 고주파로 이사(Shift) 시킨다.
   - 고주파로 쪼개진 신호들을 아날로그 믹서(가산기)로 그냥 몽땅 더해버려서 하나의 선에 흘려보낸다.
2. **수신 (DeMUX)**:
   - 뭉쳐서 날아온 덩어리 신호를 **대역통과 필터(Bandpass Filter, BPF)**에 통과시킨다.
   - BPF는 특정 주파수(예: $f_1$ 대역)만 통과시키고 나머지는 다 깎아버리는 체(거름망)다. 여기서 분리된 신호를 다시 복조하여 원본 목소리를 끄집어낸다.

### 2. 주파수 도메인 맵핑 시각화 및 보호 대역(Guard Band)

```text
  ┌─────────────────────────────────────────────────────────────┐
  │           [FDM (주파수 분할 다중화) 스펙트럼 배치의 한계 구조]         │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │     전력(Power)                                              │
  │      │   [CH 1]        [CH 2]        [CH 3]                 │
  │      │   /----\        /----\        /----\                 │
  │      │  / USER \      / USER \      / USER \                │
  │      │ /   1    \    /   2    \    /   3    \               │
  │    ──┼─┴────────┴─██─┴────────┴─██─┴────────┴─▶ 주파수(Hz)    │
  │                    ▲             ▲                          │
  │             [가드 밴드 (Guard Band)]                          │
  │                                                             │
  │ * [치명적 낭비]: 채널과 채널 사이가 완벽히 직각으로 떨어지지 않기 때문에,  │
  │   신호가 겹쳐서 간섭(Crosstalk)이 일어나는 걸 막으려면 중간에 데이터를    │
  │   절대 싣지 않는 빈 공간(가드 밴드 █)을 비워두어야 한다. 엄청난 낭비다! │
  └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** FDM의 가장 큰 약점은 차선 도색(가드 밴드)에 땅을 너무 많이 뺏긴다는 것이다. 아날로그 필터는 칼같이 90도로 주파수를 깎아내지 못하고 비스듬하게 깎이기 때문에, 옆 차선의 파동과 살짝 스치기만 해도 혼선이 터진다. 그래서 억지로 빈 공간을 넓게 둬야 했고, 전체 대역폭의 20~30%가 허공에 버려지는 참사가 발생했다.

## Ⅲ. 실무 융합 및 트러블슈팅

### 실무 시나리오

1. **시나리오 — 구형 동축케이블(HFC망) 인터넷의 상향/하향 비대칭 에러**:
   아파트 단지에 들어가는 케이블 모뎀망. 집에서 다운로드(하향)는 기가급으로 엄청 빠른데, 업로드(상향)만 하려고 하면 10Mbps도 안 나오고 핑이 미친 듯이 튄다. 
   **[해결책]** 동축케이블은 철저한 **FDM 아키텍처**다. 전체 1000MHz 파이프 중, 통신사가 TV 방송용으로 50~800MHz를 다 떼어주고, 인터넷 다운로드용으로 800~1000MHz를 크게 떼어줬다. 하지만 인터넷 업로드용 주파수 차선은 가장 노이즈가 심한 바닥 대역인 **5~42MHz의 아주 좁은 공간**만 할당해 두었다. 전형적인 **주파수 분배의 극단적 비대칭(Asymmetric FDM)** 설계 한계다. 이는 장비 고장이 아니라 물리적 차선이 부족한 것이므로, DOCSIS 3.1 이상의 장비로 교체하여 상향 대역 차선을 200MHz까지 논리적으로 확장해 주거나 광랜(FTTH)으로 갈아타야 해결된다.

2. **시나리오 — 사내 아날로그 무전기 시스템(VHF)의 혼선(Crosstalk)**:
   물류센터에서 A조는 1채널, B조는 2채널 무전기를 쓴다. 그런데 A조 반장님이 버튼을 누르고 크게 고함을 지를 때마다, 다른 주파수를 쓰고 있는 B조 무전기 스피커에서도 아주 작게 반장님 목소리가 지지직거리며 새어 나온다.
   **[해결책]** 전형적인 FDM 기반 **인접 채널 간섭(Adjacent Channel Interference)** 에러다. 반장님이 너무 큰 소리로 말해서 A조 송신기의 앰프가 포화(Saturation)되었고, 변조된 전파가 자기 채널을 뚫고 가드 밴드(Guard Band)를 넘어 B조의 주파수 대역까지 피가 튄 것(Splatter)이다. 송신기의 마이크 게인(Gain)을 낮춰서 변조 폭(Deviation)이 옆 채널을 침범하지 못하게 하거나, 무전기 세팅에서 아예 A조와 B조 채널을 1번과 10번처럼 멀리 이격시켜 주파수 사이의 물리적 거리를 벌려주어야 한다.

## Ⅳ. FDM의 몰락과 OFDM으로의 진화

### 도입 한계 (왜 FDM은 버려졌는가?)
- **하드웨어의 무식함**: FDM은 채널이 100개면 송신기에 변조기 100개, 수신기에 아날로그 필터 100개가 물리적으로 땜납 되어 있어야 한다. 부피가 거대해지고 가격이 미친 듯이 비싸진다.
- **가드 밴드의 낭비**: 비싼 라디오 주파수의 30%를 가드 밴드로 버린다는 것은 통신사 입장에선 돈을 길바닥에 뿌리는 짓이었다.

### 후계자: OFDM (직교 주파수 분할 다중화)
- 1990년대 이후 디지털 DSP 기술이 발전하면서, 공학자들은 "가드 밴드 없이 채널들을 아예 겹치게(Overlap) 만들자! 대신 파동의 주기를 수학적으로 완벽하게 맞물리게(직교성, Orthogonality) 짜면 간섭 없이 떼어낼 수 있다!"라는 미친 발상을 해낸다.
- 이것이 바로 **OFDM**이며, 낭비되는 가드 밴드를 없애버린 덕분에 효율이 2배 이상 폭증하여 현대 Wi-Fi와 5G 통신의 절대 군주로 군림하게 되었다. 즉, FDM은 현대 OFDM의 멍청하지만 위대한 아버지 격인 아키텍처다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **반송파 (Carrier Wave)** | FDM에서 각 채널을 구분 짓는 '우주선의 번호판' 같은 역할. 1번 유저는 1번 우주선, 2번 유저는 2번 우주선(반송파)에 태운다. |
| **대역통과 필터 (BPF)** | FDM 수신기의 핵심 부품. 떡져서 날아온 주파수 덩어리에서 내가 원하는 주파수 구역만 가위로 오려내는 아날로그 체(Sieve). |
| **보호 대역 (Guard Band)** | FDM의 아킬레스건. 차선과 차선 사이에 차가 부딪히지 않게 만들어둔 거대한 중앙분리대. 낭비의 주범이다. |
| **OFDM (직교 주파수 분할 다중화)** | 가드 밴드의 낭비가 너무 아까워서 채널들을 아예 겹치게 쑤셔 넣은 FDM의 기적적인 진화형. |

## 👶 어린이를 위한 3줄 비유 설명
1. 고속도로(케이블)에 승용차, 트럭, 오토바이가 엉키면 큰 사고가 나요.
2. 그래서 **FDM** 방식은 1차선은 승용차만, 2차선은 트럭만, 3차선은 오토바이만 다니게 '차선(주파수)'을 엄격하게 그려놓고 동시에 쌩쌩 달리게 하는 방법이에요.
3. 차선끼리 긁히는 걸 막으려고 중간중간 노란색 빗금(보호 대역)을 넓게 칠해뒀는데, 이게 너무 넓어서 막상 차가 다닐 공간이 좁아지는 게 단점이랍니다!
