## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 5G NR(New Radio)는 3GPP R15에서 정의된 5세대移动通信 무선接入 기술로, FR1(0.41~7.125GHz)과 FR2(24.25~52.6GHz, mmWave)라는 두 개의 주파수 범위에서 동작하며, 5G 코어(5GC)의 Service-Based Architecture와 결합하여 超高速, 超低지연, 다중 접속이라는 세 가지 핵심 시나리오를 동시에 지원한다.
> 2. **가치**: 5G NR은 하향 20Gbps, 상향 10Gbps의 이론 속도와 1ms 이하 RTT(Round Trip Time) 지연, 1백만 디바이스/km²의接続密度를 목표로 하며, 이 세 가지 요구사항을 동시에 충족하는 것은 이전 통신 세대에서 불가능했다.
> 3. **융합**: 5G NR은 LTE와 공존하며(NR、LTE 双 연결), mmWave 대역의 짧은 파장을 활용한 Massive MIMO와 Beamforming으로 공간 격리(Spatial Isolation)를 극대화하여 주파수 재사용 효율을 비약적으로 높였다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념

5G NR(New Radio)은 "새로운 무선"이라는 이름 그대로, 4G LTE의 무선接入 기술(eUTRAN)을 기반으로 설계된 것이 아니라, 처음부터 5G의 요구사항(ITU-R IMT-2020)에 부합하도록 설계된全新的 무선 시스템이다. 가장 큰 차별점은 mmWave(FR2) 대역의 도입과, 서비스 요구에 따라 네트워크를 쪼갤 수 있는 Network Slicing의 지원이다.

### 필요성

4G LTE의 한계는 세 가지로 요약된다. 첫째, 주파수 대역이 한정되어(주로 2.6GHz 이하) 대역폭 확보에 한계가 있다. 둘째, 항상 전제 условиях 설계되어 저전력 IoT 센서 연겸には適하지 않았다. 셋째, 네트워크 구조가 유연하지 않아 다양한 서비스 요구사항에 одновременно 대응하기 어려웠다. 5G NR은 이 세 가지 한계를 극복하기 위해 설계되었다.

### 등장 배경

2015년 ITU-R에서 IMT-2020(5G) 요구사항이正式 발표되었고, 3GPP에서 2017년 R15로 첫 5G NR 표준을 완성했다. 2019년 4월, Verizon이 세계 최초 5G 상용 서비스를_mmWave 기반_으로 출시했으며, 이어 2019년 중순 SK Telecom, KT, LGU+ 등 한국 통신사가 5G 상용 서비스를 시작했다. 2020년 R16에서는 URLLC(Ultra-Reliable Low-Latency Communications) 강화와 IIoT(Industrial IoT) 지원이 추가되었고, 2022년 R17에서는 REDCAP(Reduced Capability) 디바이스와 위성 통합이 定义되었다.

### 💡 비유

5G NR은 **"도시 전체를覆盖하는 고속도로網"** 에 비유할 수 있다. 4G는 일반 고속도로 수준이었다면, 5G는 일반 고속도로(FR1, Sub-6GHz) + 초고속 자기부상 철도(FR2, mmWave)를 함께 운영하여, 화물 트럭(대역폭 필요 디바이스)과 경차량(저전력 센서)와 긴급 차량(URLLC)이 모두 자신의 길에서 동시에 최대 속도로 달릴 수 있게 해준다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 주파수 범위 (FR1 vs FR2)

5G NR은 사용 주파수 대역에 따라 FR1(Sub-6GHz)과 FR2(mmWave, 24.25~52.6GHz)로 나뉜다. FR1은 기존 LTE 대역과의 공존이 가능하고 건물 penetration이 양호하여 Coverage 연속성에 유리하고, FR2는 넓은 연속 대역폭을 확보할 수 있어 超고속数据传输에 필수적이지만 커버리지가 짧고障害물에 민감하다는 제한이 있다.

| 구분 | FR1 (Sub-6GHz) | FR2 (mmWave) |
|:---|:---|:---|
| **주파수 범위** | 0.41~7.125 GHz | 24.25~52.6 GHz |
| **대역폭** | 5~100 MHz | 50~400 MHz |
| **커버리지** | 수 km (기지국당) | 100~300m |
| **장애물 취약성** | 강함 | 매우 강함 (침투 감쇠 20~35 dB) |
| **하향 속도** | 2~4 Gbps | 4~20 Gbps |
| **주 용도** | Broad coverage, IoT, 기존 LTE 공존 | 超고속 전송, 고정 무선 접속(FWA) |
| **안테나 기술** | 64T/64R MIMO | 256T/256R Massive MIMO |

### 5G NR 프레임 구조

5G NR의 시간 단위(Ti) 기본은 1ms(Subframe)이고, FR2에서는 더 짧은 슬롯(0.125ms)을 활용한다. 핵심적으로 LTE의 TTI(Transmission Time Interval) 1ms에서 5G NR은 최소 0.125ms(2 slots)까지 축소하여 지연 시간( Latency)을 크게 줄였다. 이는 URLLC 서비스에 필수적인 조건이다.

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │                 5G NR 프레임 및 슬롯 구조 (Numerology)                      │
  ├─────────────────────────────────────────────────────────────────────────┤
  │
  │  [Numerology (μ) — 서브캐리어 간격과 TTI 길이 결정]                           │
  │
  │  ┌───────┬──────────┬──────────────┬─────────────┬───────────────┐     │
  │  │ μ     │ Δf (kHz) │ TTI (slots)  │ slots/frame │ 주 용도        │     │
  │  ├───────┼──────────┼──────────────┼─────────────┼───────────────┤     │
  │  │ 0     │ 15       │ 1 ms (1)     │ 10          │ GSM/LTE 호환  │     │
  │  │ 1     │ 30       │ 0.5 ms (2)    │ 20          │ Broad coverage│     │
  │  │ 2     │ 60       │ 0.25 ms (4)   │ 40          │ 고속 전송     │     │
  │  │ 3     │ 120      │ 0.125 ms (8)  │ 80          │ URLLC, mmWave │     │
  │  │ 4     │ 240      │ 0.0625 ms(16) │ 160         │ future (>52GHz)│     │
  │  └───────┴──────────┴──────────────┴─────────────┴───────────────┘     │
  │
  │  [LTE TTI vs 5G NR TTI 비교]                                            │
  │
  │  LTE:   |←────────── 1 ms TTI ──────────→|  (고정)                       │
  │         ├──0──┬──1──┬──2──┬──3──┬──4──┬──5──┬──6──┬──7──┬──8──┤        │
  │         │     │     │     │     │     │     │     │     │     │        │
  │  5G NR: │←0.5ms→|←0.5ms→|                          (μ=1, 30kHz)     │
  │         ├──slot0─┤──slot1─┤                                        │
  │         │        │        │                                            │
  │  5G NR: │←0.125ms→| (μ=3, 120kHz)                                   │
  │         ├──slot0─┤──...                                              │
  │                                                                     │
  │  [FDD vs TDD 포맷]                                                    │
  │
  │  FDD (Frequency Division Duplex):                                   │
  │   DL 주파수: ─────────────────▶  UL 주파수: ◀────────────────       │
  │         주파수 분리, 동시 전송/수신 가능                               │
  │                                                                     │
  │  TDD (Time Division Duplex):                                        │
  │   시간 ─── DL ──── UL ──── DL ──── UL ────▶                       │
  │         전이구간(Guard Period)로 하향/상향 전환                        │
  │         주파수 대역 효율 높음, 동글본干扰管理 필요                       │
  │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 5G NR의 Numerology(μ)는 0~4까지 정의되며, 서브캐리어 간격(subcarrier spacing)이 15kHz × 2^μ로 증가한다. μ=0은 LTE 호환을 위해 유지되며, μ=1(30kHz)이 FR1의 주류이고, μ=3(120kHz)이 FR2(mmWave)의 표준이다. TTI는 반대로 1ms / 2^μ로 감소하여, μ=3에서는 0.125ms까지 짧아진다. TTI가 짧아지면 스케줄링 주기가 짧아져 지연 시간(Latency)이 감소하지만, 그만큼 제어 영역(Control Region)开销이 증가하는 trade-off가 있다. URLLC 서비스에서는 짧은 TTI가 필수이므로 μ=3이 적합하고, 일반 브로드밴드에는 TTI가 길어도 되므로 μ=1의 효율이 더 높다.

### Massive MIMO와 Beamforming

5G NR의 FR2에서 핵심적인 기술 중 하나가 Massive MIMO와 Beamforming이다. 4G LTE에서 2x2 또는 4x4 MIMO를 사용하지만, 5G NR FR2에서는 256x256 Massive MIMO를 표준으로 지원한다. 또한 mmWave에서는 파장이 짧아(예: 28GHz에서 파장 약 10.7mm) 안테나 배열을小さくても 많은 앵원소를 팩할 수 있어, digital/analog beamforming을 통해 특정 방향으로 信号을 집중시킬 수 있다.

```text

  │              5G NR Massive MIMO & Beamforming 동작 원리                      │
  ├─────────────────────────────────────────────────────────────────────────┤
  │
  │  [Traditional MIMO vs Massive MIMO 비교]                                │
  │
  │  Traditional (4x4 MIMO):           Massive MIMO (256x256):              │
  │                                                                     │
  │       eNB                                                              │
  │      [##]                         [################]                  │
  │      [##] 4x4                      [################] 256x256         │
  │      [##]                          [################]                │
  │      [##]                          [################]                │
  │        │                                │                              │
  │        │  모든 방향으로 균등辐射              │  특정 방향으로 집중 beam    │
  │        ▼                                ▼                              │
  │     ○○○○ (4 Users)              ○○○○○○○○ (Multi-users)               │
  │                                                                     │
  │  [Beamforming 동작 과정]                                              │
  │
  │     eNB / gNB                                                         │
  │   [ ANT ARRAY ]                                                       │
  │       │                                                               │
  │       ├──▶ 빔 1 (사용자 A方向)                                         │
  │       ├──▶ 빔 2 (사용자 B方向)                                         │
  │       ├──▶ 빔 3 (사용자 C方向)                                         │
  │       └──▶ 빔 4 (사용자 D方向)                                         │
  │                                                                     │
  │   각 안테나 앵원소의 위상(phase)과 진폭(amplitude)을 조절하여           │
  │   특정 방향으로干涉パターンを形成 (Arrary Factor)                         │
  │                                                                     │
  │  ┌─────────────────────────────────────────────────────────────────┐  │
  │  │  Beamforming의 핵심 원리 — 위상 배열 (Phased Array)               │  │
  │  │                                                                   │  │
  │  │  각 안테나에서放射된 신호의 위상을 조절하면,                    │  │
  │  │  특정 방향에서는 건설적干涉 (相加強化),                          │  │
  │  │  다른 방향에서는 파괴적干涉 (相殺) 발생                           │  │
  │  │                                                                   │  │
  │  │  θ 방향으로 최대 利得을 위한 위상差: Δφ = (2πd/λ) × sin(θ)       │  │
  │  │  d: 안테나 간격, λ: 파장                                         │  │
  │  │                                                                   │  │
  │  │  예: 28GHz, d=λ/2=5.35mm, 256 안테나 배열                      │  │
  │  │  →理論 최대利得: 10×log10(256) ≈ 24 dBi                         │  │
  │  │  →송신 power 24 dBm + antenna gain 24 dBi = 48 dBm EIRP       │  │
  │  └─────────────────────────────────────────────────────────────────┘  │
  │
  │  [Beam Management 절차 — 3단계]                                        │
  │
  │  ① 빔 검색 (Beam Sweeping):                                           │
  │     gNB가 다양한 방향으로的控制된 빔을 sweep하여                     │
  │     UE가 가장 강한 빔을 발견하게 함                                     │
  │                                                                     │
  │  ② 빔 측정 (Beam Measurement):                                        │
  │     UE가 복수의 빔에 대한 신호 품질(RSRP, SINR)를 측정                  │
  │                                                                     │
  │  ③ 빔 결정/보고 (Beam Determination/Reporting):                        │
  │     UE가 최적 빔 ID와 측정치를 gNB에 보고                             │
  │     gNB가 해당 빔으로 communications 시작                              │
  │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** Massive MIMO의 핵심 효과는 안테나 수(N)가 증가할수록 각 사용자에게 할당할 수 있는 독립적空間 스트림이 증가하여 시스템 용량이 선형적으로 증가한다는 점이다. 이론적으로 N개의 안테나가 있으면 N개의 사용자에게 동시에服务质量을 제공할 수 있다(space division multiple access). Beamforming은 각 사용자에게 필요한 만큼만 신호를 집중投射함으로써, 다른 방향으로 불필요하게 방사되는 에너지를 절약하고,同一 주파수를 공간적으로再利用可能하게 한다. 예를 들어, 28GHz mmWave에서 256 안테나 배열이 만드는 beam의 폭은 약 3~5도 정도로, 원하는 사용자에게 정확히 信号을 전달할 수 있다. FR2의 짧은 커버리지를 보완하기 위해 5G NR은 빔 관리(Beam Management) 절차를 통해 UE와 gNB가 최적의 빔 방향을 실시간으로 검색하고追踪하는 것이 필수적이다.

## Ⅲ. 융합 비교 및 다각도 분석

### 5G 사용 시나리오와 핵심 기술 매핑

ITU-R IMT-2020은 5G를 세 가지 사용 시나리오로 구분하고, 각각에 해당하는 핵심 기술을 명시했다. eMBB(Enhanced Mobile Broadband)는 기존의的高速 데이터服务的 extensión이고, URLLC(Ultra-Reliable Low-Latency Communications)는 산업 자동화, 원격 진료, 자율주행 같은 초低지연·超高可靠性 서비스이고, mMTC(Massive Machine-Type Communications)는 IoT 센서의 超다수 동시 접속을 위한 것이다.

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │              5G 3대 사용 시나리오와 핵심 기술 요구사항                        │
  ├─────────────────────────────────────────────────────────────────────────┤
  │
  │  ┌────────────────┬────────────────┬────────────────┐                  │
  │  │     eMBB       │     URLLC      │     mMTC       │                  │
  │  │ (고속 모바일宽带) │ (초低지연超可靠) │ (대량 IoT)       │                  │
  │  ├────────────────┼────────────────┼────────────────┤                  │
  │  │ Peak: 20 Gbps │ Latency: <1ms  │ Density: 1M/km²│                  │
  │  │ User: 100~    │ Reliability:   │ Battery: 10+年 │                  │
  │  │   1 Gbps     │   99.999%      │   (LPWA 수준)  │                  │
  │  ├────────────────┼────────────────┼────────────────┤                  │
  │  │ FR1: Sub-6GHz │ FR2: mmWave    │ FR1: NB-IoT   │                  │
  │  │ FR2: mmWave   │  Mini-slot     │   LTE-M        │                  │
  │  │ 256QAM        │ (2/4/7 symbols)│                 │                  │
  │  │ Carrier Agg.   │ Grant-free     │ Relaxed numer. │                  │
  │  │ Massive MIMO  │ Preemption     │ Power saving   │                  │
  │  │ Dual Connect.  │ URLLC PUSCH    │   (PSM, eDRX) │                  │
  │  └────────────────┴────────────────┴────────────────┘                  │
  │
  │  [URLLC의 핵심 — Grant-Free vs Grant-Based 비교]                        │
  │
  │  Grant-Based (기존 LTE/5G):                                            │
  │   UE →gNB: Scheduling Request (SR) →                                  │
  │   gNB →UE: UL Grant (자원을 할당해줘) →                                │
  │   UE →gNB: 데이터 전송                                                   │
  │   → 지연: SR(1ms) + Grant(1ms) + 데이터 = 최소 2ms+                     │
  │
  │  Grant-Free (URLLC):                                                   │
  │   UE →gNB: 설정된 자원에 직접 데이터 전송                                │
  │   → 지연: 데이터 전송 자체만 = 0.125~1ms                                 │
  │   → 충돌 위험 증가, but URLLC는 미리 설정된 자원 사용                      │
  │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 5G의 세 가지 시나리오는 각각 상이한 기술적 접근이 필요하다. eMBB는 더 넓은 대역폭(FR2에서 400MHz까지)과 더 많은 안테나(Massive MIMO)로 스루풋을 극대화한다. URLLC는 지연을 줄이기 위해 Grant-Free 방식의 스케줄링을 도입하여 UE가 사전 할당된 자원)에 바로 데이터를 전송할 수 있게 하고,_mini-slot_(2/4/7 OFDM 기호)으로 TTI를 단축한다. mMTC는 대량 디바이스의 접속을 처리하기 위해 LTE-M과 NB-IoT를 5G에서 계속 지원하면서, 전력 절전 모드(PSM, eDRX)로 10년 이상의 배터리 수명을 실현한다. 중요한 점은 하나의 5G 네트워크에서 이 세 가지 시나리오가 **동시에** 제공되어야 한다는 것이다. 이는 네트워크 슬라이싱(Network Slicing)을 통해 실현된다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 5G FWA (Fixed Wireless Access) 배포**: 통신사가 mmWave 기반 5G를 활용하여 가정에 고정 무선 인터넷(FWA)을 제공하려는 상황. FWA는 광-fiber铺设이 어려운 지역에 5G 기지국에서 직접 가정에 전파를照射하여 인터넷을 제공하는 서비스이다. 핵심 이슈는 mmWave의 장애물 취약성으로, 건물이 있거나 나무가 우세한 환경에서는 서비스 품질이 급격히 저하될 수 있다. 해결책으로 FR1(Sub-6GHz, 3.5GHz)를 함께部署하여 coverage를 확보하고, FR2(mmWave)는 throughput이 필수적인 지역에 한정하는 것이 일반적이다.

2. **시나리오 — 5G URLLC 기반 산업 로봇 제어**: 공장 환경에서 5G URLLC를 활용하여 산업 로봇을 실시간 원격 제어하려는 상황. 핵심 요구사항은 99.999% 신뢰성과 1ms 이하 지연이다. 이를 위해 5G NR의 미니슬롯(2 기호, 0.125ms)과 grant-free 스케줄링을 활용하고, preemptive scheduling으로 URLLC 트래픽이 다른 트래픽보다 우선적으로 자원을 점유하도록 설정해야 한다. 또한 무선 환경의変動을 대비하여iversity(다중 경로衰落 방지)를 위한 기술이 추가되어야 한다.

3. **시나리오 — 5G 네트워크 슬라이싱을 활용한 다중 서비스 제공**: 한运营商在同一 5G 네트워크에서 eMBB(일반 소비자), URLLC(자율주행), mMTC(스마트 시티 센서)라는 세 가지 다른 종류의 서비스를 동시에 제공하려는 상황. 네트워크 슬라이싱은 동일 물리 인프라 위에 논리적으로 분리된 네트워크(슬라이스)를 만드는 기술로, 각 슬라이스는 독립된 CN(코어) 구성요소와 무선 자원 할당 정책을 가진다. 핵심 구현은 5GC의 Network Function(NF)들이 RESTful API로 구성되어 있어, 각 서비스 유형에 최적화된 CN 설정을 程序적으로 적용할 수 있다는 점이다.

### 도입 체크리스트

- **기술적**: FR1과 FR2의 배포 전략을 수립했는가? FR2(mmWave) 커버리지가 목표 지역에서 충분한가? UE와 네트워크 간의 EN-DC(EUTRA-NR Dual Connectivity) 구성이 수립되어 있는가?
- **운영·보안적**: 5G 네트워크 슬라이싱에서 슬라이스 간 격리가 제대로 유지되는가? 5G 보안(aka 5G AKA)의 종단 간 인증이 설정되어 있는가?

### 안티패턴

- **FR2 과대평가**: mmWave의 커버리지를 과대평가하여 기지국간 거리를 너무 넓게 배치하면 coverage 공백이 발생한다. FR2는 100~300m Reichweite를 기준으로 planning해야 하고, 장애물과 건물 penetration 손실(20~35 dB)을 반드시 고려해야 한다.
- **URLLC 지연 과대평가**: 1ms 지연 요구사항은 end-to-end에서 무선 구간(radio access network)의 지연만 의미한다. 실제 전체 서비스의 지연에는 전처리/후처리 시간,传输 지연, 코어 네트워크 지연이 합산되므로, 전체 시스템 레벨에서의 지연이 1ms에 근접하게 하려면 코어도 매우 짧은 지연이 필요하다.

- **📢 섹션 요약 비유**: 5G NR의 세 가지 시나리오는 **"같은 도시에 세 가지 다른 도로망을 운영하는 것"** 에 비유할 수 있다. 일반 시민용 도로(eMBB)는 넓고 빨라야 하고, 긴급 차량용 도로(URLLC)는 신호등 없이 거의 즉시 도착해야 하고, 화물 트럭/자전거/人専用도로(mMTC)는 여러 대가 동시에 들어가면서도 서로 interference하지 않아야 한다. 네트워크 슬라이싱은 이 세 가지 도로망을同一 지도 위에 논리적으로 그려各各 운영할 수 있게 해주는 토폴리지에 비유할 수 있다.

### 정량/정성 기대효과

| 구분 | 4G LTE-A | 5G NR eMBB (FR1) | 5G NR URLLC (FR2) | 개선 효과 |
|:---|:---|:---|:---|:---|
| **정량** | 1 Gbps | 2~4 Gbps | 4~20 Gbps | **2~20배 향상** |
| **정량** | 10ms | 4ms | <1ms | **지연 90% 감소** |
| **정량** | 100K/km² | 1M/km² | 1M/km² | **동시 접속 10배** |
| **정성** | 유연성 제한 | 네트워크 슬라이싱 | 초低지연 +超高可靠性 | **차별화 서비스 실현** |

### 미래 전망

- **5G-Advanced (R18/R19)**: 5G NR의 다음 단계인 5G-Advanced는 AI/ML 기반의 무선 자원 관리, XR(Extended Reality) 서비스 최적화, 비地面 네트워크(NTN, Non-Terrestrial Network) 통합, 그리고 통합 접근(Integrated Access and Backup, IAB)등을 목표로 한다.
- **6G와의 연속성**: 5G NR은 6G로 진화하는 과정에서 완전히 폐지되지 않고, 6G의 일부로 통합되거나 6G의 비 terrestre 확장의 기반으로 활용될 전망이다. THz (6G)의 물리적 제한으로 인해 6G도 FR1과 FR2를 계속 활용하게 될 것이다.

**[다이어그램 해설]** 5G NR의 발전은 세 단계로 구분된다. R15는 첫 상용화를 위한 기본 표준으로, eMBB에 초점을 맞추었다. R16은 URLLC를 본격화하고 IIoT(Industrial IoT)과 V2X(Vehicle-to-Everything)를 추가하여 5G의 사업 영역을 확대했다. - **📢 섹션 요약 비유**: 5G NR의 진화는 **"도시 교통 시스템의 진화"** 에 비유할 수 있다. R15는 도시의 기본 도로망(차도, 고속도로)을建设하면, R16은 긴급차량 전용 차선(URLLC)과 공장 도로(V2X)를增设하고, R17은 자전거 도로(REDCap)와 고속도로 위의 하늘 도로(위성 integrated)를增设하는 것에 비유할 수 있다. 5G-Advanced는 모든 도로에 AI 신호 체계(AI-native)를 도입하여 도로 상황에 따라 스스로 차선을 조절하고, 미래 6G는 차도 위를 나는 차(THz)까지 등장하는 완전히 새로운 교통 체계를 꿈꾸는 것과 같다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **FR1/FR2 (주파수 범위)** | 5G NR의 두 가지 주파수 범위로, FR1(Sub-6GHz)은 coverage에 유리하고 FR2(mmWave)는 대역폭에 유리하여 상호 보완적으로部署된다. |
| **mmWave (밀리미터파)** | 24~100 GHz 대역의 초고주파수로, 짧은 파장(1~10mm) 덕분에 huge한 대역폭 확보가 가능하지만障害물에 매우 취약하여 beamforming 기반의 정확한 지향성 전송이 필수이다. |
| **Massive MIMO** | 수십~수백 개의 안테나를 활용한 MIMO 기술로, 공간 다중화(spatial multiplexing)를 통해同一 시간/주파수 자원에서 다중 사용자에게 동시 서비스를 제공한다. |
| **Beamforming** | 안테나 배열의 각 앵원소 위상을 조절하여 특정 방향으로 信号을 집중投射하는 기술로, mmWave에서 커버리지 확대와 간섭 감소에 핵심적이다. |
| **Network Slicing (네트워크 슬라이싱)** | 동일 물리 인프라 위에 논리적으로 분리된 다중 가상 네트워크를作成하는 기술로, 각 슬라이스가 독립된 네트워크 기능과 QoS 정책을 가져 不同 서비스 요구에 одновременно 대응한다. |
| **5GC (5G 코어)** | 3GPP R15에서 정의된 5G 코어 네트워크로, Service-Based Architecture(SBA)에서 네트워크 기능(NF)들이 RESTful API로 상호 통신한다. |

## 👶 어린이를 위한 3줄 비유 설명

1. 5G NR은 **"레이저 조명 시스템"** 에 비유할 수 있어요. 예전 4G는 전구(전파)를広くつけ했지만, 5G는 레이저처럼 특정 방향(Beamforming)으로 조명을集中시켜 더 밝고(Higher Throughput) 멀리까지(Farther Coverage) 빛을 전달할 수 있어요.
2. 5G의 세 가지 기능(eMBB, URLLC, mMTC)은 **"같은 놀이터에서 세 명의 선생님이 각각 다른 게임을 운영하는 것"** 과 같아요.一位 선생님은 레이싱 게임(빠른 데이터)을,另一位 선생님은外科游戏(정확한 작업)을, 세 번째 선생님은 인사 게임(여러 Sensor 연결)을 운영하는 것처럼, 5G도 동시에 세 가지 다른 종류의 서비스를 제공할 수 있어요.
3. 5G가 FR2(mmWave)를 쓰는 것은 **"더 넓은 도로를 만든다"** 는 것과 같아요. 더 넓은 도로(대역폭)에서는 더 많은 차(데이터)가 동시에通行할 수 있지만, 도로가 넓으면(파장이 짧으면) 건물에 가려지는 문제가 생기기 때문에, 레이저 조명(Beamforming)으로 정확히 원하는 곳에 도로를照射하는 기술이 필요한 것이에요.
