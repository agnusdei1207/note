## 핵심 인사이트 (3줄 요약)
> 1. **본질**: WiFi는 IEEE 802.11 표준을 기반으로 2.4GHz 또는 5GHz 주파수 대역에서 작동하는 근거리 무선 LAN (WLAN, Wireless Local Area Network) 기술로, 유선 이더넷의 무선 대체 수단으로 등장했다.
> 2. **가치**: 802.11ax (WiFi 6)는 최대 9.6Gbps throughput과 OFDMA (Orthogonal Frequency Division Multiple Access) 기반 다중 접속으로 지연 시간 75% 감소를 달성하며, IoT (Internet of Things) 디바이스 대량 연결에 최적화되었다.
> 3. **융합**: WiFi 6E는 6GHz 대역 확장으로 160MHz 대역폭 채널을 7개 추가로 확보하여 고밀도 환경에서의 간섭 문제를 해결하고, 5G 셀룰러와 동일한 스펙트럼 공유 정책框架下에서 공존한다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념

WiFi는 IEEE 802.11 표준 규격군을 따르는 무선 매체 접근 방식이다. 유선 LAN (Local Area Network)이 이더넷 케이블로 물리적 연결을 필요로 하는 반면, WiFi는 전파(Radio Wave)를 통해 공간(Space)을 매질(Medium)로 활용하여 디바이스를 네트워크에 연결한다. 이 차이는 "연결의 유연성"이라는 핵심 가치를 만들어내며, 이동 중 데이터 접근과 временно 네트워크 구축이라는 사용 시나리오를 가능하게 했다.

### 필요성

유선 이더넷은 물리적 포트와 케이布이 필요하므로 대규모 배포 시 비용과 복잡성이 증가한다. 특히 사무실 환경에서 직원1인당 평균 3개 이상의 디바이스가 연결되는 현재 상황에서, 각 디바이스마다 유선 포트를 확보하는 것은 비현실적이다. WiFi는 이러한 물리적 제약을 무선이라는 해법으로 대체하여 네트워크 접근 장벽을 크게 낮추었다.

### 등장 배경

1997년 최초의 802.11 표준이 2.4GHz 대역에서 2Mbps 속도로 출시되었지만, 시장의 기대에 미치지 못하여 상용화되지 못했다. 이후 1999년 802.11b (11Mbps)와 802.11a (54Mbps)가 동시에 등장하면서 상용화의 시대를 열었고, 2003년 802.11g가 2.4GHz 대역에서 54Mbps를 지원하여 세계적 보급률 1위를 달성했다.

### 💡 비유

WiFi는 "공기에 레이저를 쏘아서 정보를 전달하는 것"이라고 비유할 수 있다. 레이저는 직진하지만 벽에 막히지만, WiFi 전파는 공간을 투과하여 어디서든 신호를 받을 수 있다. 각 세대의 속도 향상은 레이저의 세기를 높여 더 먼 거리에서 더 빠른 속도를 얻는 것과 같다.

### WiFi 세대별 스펙 비교

802.11 표준의 발전은 크게 세 가지 축으로 진행되었다. 첫째, 변조 방식의 효율성 증가로 주파수 당 데이터 전송량(스펙트럼 효율)이 개선되었다. 둘째, 다중 안테나 기술(MIMO)로 공간을 활용한 병렬 전송이 가능해졌다. 셋째, OFDM(Orthogonal Frequency Division Multiplexing)에서 OFDMA로 전환되어 다중 사용자 동시 접속 효율이 크게 향상되었다.

각 세대의 물리적 특성과 시장 포지셔닝을 정리하면 다음과 같다.

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │                    WiFi 세대별 핵심 파라미터 비교                            │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                         │
  │  ┌───────────┬────────┬──────────────┬─────────────┬────────────────┐ │
  │  │ 세대       │ 연도   │ 최대 속도     │ 주파수 대역   │ 주요 변조 방식  │ │
  │  ├───────────┼────────┼──────────────┼─────────────┼────────────────┤ │
  │  │ 802.11a   │ 1999   │ 54 Mbps      │ 5 GHz       │ OFDM           │ │
  │  │ 802.11b   │ 1999   │ 11 Mbps      │ 2.4 GHz     │ DSSS           │ │
  │  │ 802.11g   │ 2003   │ 54 Mbps      │ 2.4 GHz     │ OFDM           │ │
  │  │ 802.11n   │ 2009   │ 600 Mbps     │ 2.4/5 GHz   │ MIMO-OFDM      │ │
  │  │ 802.11ac  │ 2013   │ 6.9 Gbps     │ 5 GHz       │ MU-MIMO-OFDM   │ │
  │  │ 802.11ax  │ 2019   │ 9.6 Gbps     │ 2.4/5/6 GHz │ OFDMA          │ │
  │  │ 802.11be  │ 2024~  │ 46 Gbps (예) │ 2.4/5/6 GHz │ MLO (Multi-Link)│ │
  │  └───────────┴────────┴──────────────┴─────────────┴────────────────┘ │
  │                                                                         │
  │  DSSS: Direct Sequence Spread Spectrum — 넓은 대역에 신호를 확산        │
  │  OFDM: 직교 부반송파로 대역 분할 다중화                                  │
  │  MIMO: 다중 안테나로 공간 다중화                                          │
  │  OFDMA: 서브캐리어 그룹을 사용자별 할당                                  │
  │  MLO: 여러 링크를 동시에 사용하는 차세대 병렬 전송                         │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 이 비교표는 WiFi의 진화 방향이 단순히 "속도 향상"만이 아니라, 동시 접속 디바이스 밀도 관리와 스펙트럼 효율성 개선에 초점이 맞춰져 있음을 보여준다. 802.11b의 DSSS는 2.4GHz ISM 대역에서 간섭에 강하지만 스펙트럼 효율이 낮아 속도에 한계가 있다. 802.11g의 OFDM 도입으로 주파수 효율이 크게 개선되었지만, 2.4GHz 대역의 혼잡으로 실질 throughput은 표기 속도의 50% 수준에 그쳤다. 802.11n의 MIMO 도입은 공간 다중화(spatial multiplexing)로 이론적 throughput을 4배 이상 끌어올렸지만, 실환경에서 기대 이하의 성능이 보고되었다. 802.11ac는 MU-MIMO(Multi-User MIMO)로 AP가 다중 클라이언트에게 동시 전송 가능하게 했지만, 하행 only의 제한이 있었다. 802.11ax의 OFDMA는 상하행 모두에서 다중 사용자 동시 전송을 지원하여 고밀도 환경의 지연 시간을 획기적으로 개선했다.

---

### OFDM과 OFDMA 동작 원리

OFDM(Orthogonal Frequency Division Multiplexing)은 하나의 넓은 대역폭 채널을 52개의 직교(orthogonal) 서브캐리어(subcarrier)로 분할하는 기술이다. 각 서브캐리어는 서로 직교하므로 간섭 없이 병렬 전송이 가능하다. OFDM의 핵심은 FFT(Fast Fourier Transform) 크기 64에서 52개가 데이터용, 나머지가 파일럿 신호용으로 사용된다는 점이다.

802.11ax의 OFDMA는 이 OFDM을 발전시켜 각 사용자에게 서브캐리어 그룹(Resource Unit, RU)을 할당하는 방식으로 다중 접속을実現한다. 20MHz 채널 기준으로 9개의 RU(각 RU는 26개의 서브캐리어)로 분할되어, 최대 9명의 사용자가 동시에 데이터를 주고받을 수 있다. 이는 AP가 한 명씩轮流 전송하던 방식의 비효율을 획기적으로 줄인다.

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │               OFDM vs OFDMA: 서브캐리어 할당 방식 비교                      │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                         │
  │  [OFDM 방식 — 단일 사용자가 전체 대역 사용]                                │
  │                                                                         │
  │  주파수: ─────────────────────────────────────────────────▶             │
  │         │ P │ D │ D │ D │ D │ D │ D │ D │ D │ D │ D │ D │ P │          │
  │         │ P=파일럿, D=데이터 (52개 서브캐리어 중 48개 사용)               │
  │                     └─────── 사용자 1 독점 ───────┘                     │
  │                                                                         │
  │  [OFDMA 방식 — 다중 사용자가 대역 분할 사용]                              │
  │                                                                         │
  │  주파수: ─────────────────────────────────────────────────▶             │
  │         │ RU1 │ RU2 │ RU3 │ RU4 │ RU5 │ ... │ RU9 │                     │
  │           │      │      │      │      │           │                     │
  │         사용자1  사용자2  사용자3  사용자4  ...    사용자9               │
  │                                                                         │
  │  RU (Resource Unit) 크기: 26, 52, 106, 242, 996 개의 서브캐리어         │
  │  20MHz 채널 → 최대 9개 RU (26-서브캐리어 RU) 동시 할당 가능             │
  │                                                                         │
  │  ┌───────────────────────────────────────────────────────────────────┐   │
  │  │  OFDMA 장점:                                                      │   │
  │  │  ① 지연 시간 감소 — 사용자 대기 없이 동시 전송                      │   │
  │  │  ② 스펙트럼 효율 향상 — 대역 활용률 최대 100%                      │   │
  │  │  ③ 고밀도 환경 최적화 — 다중 IoT 디바이스 동시 접속                  │   │
  │  │  ④ 상하행 효율 개선 — TWT (Target Wake Time) 협회로 IoT 절전      │   │
  │  └───────────────────────────────────────────────────────────────────┘   │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** OFDM 방식에서 사용자 1이 전체 채널을 사용하면 다른 사용자들은 전송 기회를 기다려야 하므로 지연이 누적된다. 반면 OFDMA에서는 RU라는 작은 단위로 채널을 분할하여 여러 사용자가 동시에 데이터를 전송할 수 있다. 802.11ax의 핵심 개선점은 이런 식으로 "시간 분할 다중화(TDMA)"에서 "주파수 분할 다중화(FDMA)"로 전환하여 채널 활용 효율을 극대화했다는 것이다. 특히 IoT 환경에서는 수백 개의 센서가 주기적으로 소량 데이터를 보내는데, 이때 OFDMA는 각 센서에 작은 RU를 할당하여 에너지 효율과 응답성을 동시에 개선한다. TWT(Target Wake Time)는 스케줄러와 연동되어 디바이스가 특정 시간에 깨어나 데이터를 주고받도록 하여 IoT 디바이스의 배터리 수명을 크게 연장한다.

## Ⅲ. 융합 비교 및 다각도 분석

### 세대별 기술적 차이 심층 비교

802.11n과 802.11ac의 차이는 MIMO의 활용 방식에서 드러난다. 802.11n은 SU-MIMO(Single-User MIMO)만 지원하여 한 번에 한 사용자와 통신하지만, 802.11ac는 MU-MIMO(Multi-User MIMO)를 도입하여 AP가 4명, later 8명(802.11ac wave 2)의 사용자에게 동시에 공간 스트림을 할당할 수 있게 되었다. 그러나 MU-MIMO는 하행만 지원하고 상행은 지원하지 않는다는 근본적 제한이 있어 802.11ax에서 상하행 MU-MIMO로 확장되었다.

| 비교 항목 | 802.11n (WiFi 4) | 802.11ac (WiFi 5) | 802.11ax (WiFi 6) |
|:---|:---|:---|:---|
| **최대 MIMO 스트림** | 4×4 | 8×8 (wave 2) | 8×8 |
| **최대 modulation** | 64-QAM | 256-QAM | 1024-QAM |
| **OFDMA** | 미지원 | 미지원 | 상하행 모두 지원 |
| **MU-MIMO** | 미지원 | 하행만 4 users | 상하행 모두 8 users |
| **TWT (절전)** | 미지원 | 미지원 | 지원 |
| **BSS Coloring** | 미지원 | 미지원 | 지원 (간섭 완화) |
| **최대 throughput** | 600 Mbps | 6.9 Gbps | 9.6 Gbps |

### WiFi vs 블루투스 vs 셀룰러 비교

  │  ※ Bluetooth: 저전력, 짧은 데이터, 페어링 기반                           │
  │  ※ LTE: 광범위, 이동성, 통신사 종속                                      │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 세 기술은 서로 다른 니치(niche)를 확보하고 있다. WiFi는办公室/집 환경에서 고대역폭 디바이스(노트북, TV, 게임기) 연결의 표준으로 자리잡았다. 블루투스는웨어러블, 이어폰, 사물인터넷 센서처럼 저전력으로 짧은 데이터를 주고받는 시나리오에 적합하다. LTE는 통신사 네트워크가覆盖하는 곳이라면 어디서든 이동 중에도 데이터 연결이 가능하지만, 모듈 비용과 데이터 요금 문제가 있다. 특히 5G 시대에는 이 경계가 흐려져, WiFi 6와 5G NR이 서로의 영역을 침범하는融合(Convergence) 현상이 가속되고 있다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 고층 사무동 WiFi 커버리지 설계**: 30층 건물에서 각 층당 50명의 사용자가 동시에 WiFi를 사용하는 환경에서, 기존 2.4GHz 단일 채널 AP 배포 시 간섭과 속도 저하 문제가 발생하는 상황. 해결책으로 5GHz 대역 802.11ac AP를 각 층당 3개 이상 배치하고 채널 할당(1, 6, 11 채널)을 최소화하여 셀 간 간섭을 줄이는 설계를 적용해야 한다.

2. **시나리오 — IoT 센서 네트워크의 WiFi 채택 판단**: 공장 환경에서 1,000개 이상의 온도/습도 센서를 WiFi로 연결하여 데이터를 수집하려는 경우. WiFi의 TWT(Target Wake Time) 지원으로 절전 성능이 개선되었지만, 2.4GHz 대역의 간섭과 AP 동시 접속 수 제한(典型的으로 AP당 200여 개)이 문제될 수 있다. 따라서 센서의 데이터 전송 주기와 배터리 수명 요구치를 분석하여, WiFi가 부적합하면 Zigbee나 BLE(Bluetooth Low Energy)로 대체하는 것이 현명하다.

3. **시나리오 — WiFi 6 전환 시机的 판단**: 기존 802.11ac 인프라를 WiFi 6로 전환할지 결정해야 하는 상황에서, 현재 사용자가 AP당 평균 30명이고 주요 트래픽이 웹 브라우징/이메일 수준이라면 WiFi 6의 OFDMA 장점을 활용하지 못한다. 반면 화상 회의, 클라우드 스토리지 대규모 업로드, 실시간 스트리밍이 주流量이라면 WiFi 6 전환으로 사용자당 throughput이 30% 이상 개선될 수 있어 도입 효과가 크다.

### 도입 체크리스트

- **기술적**: AP 배치 전 Site Survey를 수행하여 간섭원과 커버리지 빈틈을 분석했는가? 주파수 대역(2.4/5/6GHz) 선택이 사용 디바이스 호환성과 간섭 환경에 적합한가?
- **운영·보안적**: WPA3 Personal 또는 Enterprise 보안을 적용했는가? 게스트 네트워크와 내부 네트워크를 VLAN(irtual LAN)으로 분리했는가?

### 안티패턴

- **채널 重複配置**: 인접 AP 간 채널 간섭을 고려하지 않고 동일 채널을 중복 배치하면 성능이 반감된다. 반드시 1/6/11(2.4GHz) 또는 인접하지 않은 5GHz 채널을 사용해야 한다.
- **모노밴드 설계**: 2.4GHz에만 의존하면 대역폭 부족과 간섭 문제로 고통받는다. WiFi 6를 도입한다면 반드시 5GHz 또는 6GHz 대역도 함께 활용하는 듀얼밴드 또는 트라이밴드 설계를 해야 한다.

- **📢 섹션 요약 비유**: WiFi 세대 진화는 넓은 도로 위에 들어가는 차선의 수와 신호 제어 방식의 진화로 비유할 수 있다. OFDMA는 톨게이트의 하이패스 차선을 더 세분화하여 동시에 더 많은 차량이 통과하게 하는 것과 같다.

### 미래 전망

- **WiFi 7 (802.11be EHT)**: 320MHz 채널 폭과 MLO(Multi-Link Operation)로 최대 46Gbps throughput 달성을 목표로 한다. 6GHz 대역 중심이며, VR/AR (Virtual Reality/Augmented Reality) 체험과 8K 스트리밍 시나리오에 최적화될 전망이다.
- **WiFi와 5G 공존**: 산업용 IoT 환경에서 5G URLLC(Ultra-Reliable Low-Latency Communications) 요구를 WiFi 6/6E가 도전하는 구도가 형성되고 있으며, 각 기술의 장점وار병 존(적합 영역)이 재정의되고 있다.

```

**[다이어그램 해설]** WiFi의 발전은 크게 세 가지 방향으로 진행된다. 첫째, 스펙트럼 확장(2.4→5→6GHz)으로 더 넓은 대역폭을 확보한다. 둘째, MIMO와 OFDMA의 진화로 공간 효율성을 높인다. 셋째, MLO 같은 기술로 여러 링크를 병렬 활용하여 지연을 최소화한다. WiFi 7 이후에는 5G 셀룰러의 고신뢰低지연 영역과 직접 경쟁하는 영역이 생길 것으로 예상되며, 향후 WiFi 8에서는 THz 대역 활용도 연구되고 있다.

- **📢 섹션 요약 비유**: WiFi 진화는 좁은 골목길(2.4GHz单一车道)에서 넓은 고속도로(6GHz 320MHz 8차로)로 차선이 증가하고, 신호 제어 시스템(OFDMA)이 도입되어 더 많은 차량(디바이스)이 동시에 albeit 빠르게 이동할 수 있게 된 것과 같다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **OFDM (직교 주파수 분할 다중화)** | WiFi의 물리 계층 변조 방식으로, 서브캐리어 분할을 통해 주파수 효율을 높이고 다중 경로 페이딩에 강점을 갖는다. |
| **MIMO (다중 입력 다중 출력)** | 다중 안테나를 통해 공간 다중화를實現하여 throughput을 선형적으로 증가시키며, 802.11n부터 WiFi의 핵심 기술이 되었다. |
| **SSID (서비스 세트 식별자)** | WiFi 네트워크의 논리적 이름으로, AP가 비콘 프레임(Beacon Frame)을 통해 광고하며 클라이언트가 연결 대상을 식별한다. |
| **WPA3 (WiFi Protected Access 3)** | 2018년 도입된 WiFi 보안 프로토콜로, SAE(Simultaneous Authentication of Equals) 기반 비밀번호 방어와 192비트 보안 스위트으로 강화되었다. |
| **OFDMA (직교 주파수 분할 다중 접속)** | 802.11ax의 핵심 기술로, 하나의 채널을 다중 사용자에게 서브캐리어 그룹(RU)으로 할당하여 동시 전송을 가능하게 한다. |
| **TWT (목표 웨이크 타임)** | 802.11ax의 절전 메커니즘으로, AP와 디바이스가 특정 시간에 데이터 교환을 스케줄하여 IoT 디바이스의 배터리 수명을 크게 향상시킨다. |

## 👶 어린이를 위한 3줄 비유 설명

1. WiFi는 **"공기 중의 말풍선"** 과 같아요. 여러 사람이 동시에 공기 중에서 다른 색깔의 말풍선을 서로 던져주면, 각 사람은 자기 색깔의 말풍선만 잡을 수 있듯이 여러 기기가 동시에 데이터를 주고받을 수 있어요.
2. WiFi 세대가 높아질수록 말풍선을 더 많이 동시에 던져주고, 더 빠르게 던질 수 있어서 여러 친구들이 동시에 큰 데이터를 보내 받아도 서로 기다리지 않아도 돼요.
3. 하지만 말풍선은 벽을 통과하지 못하듯이 WiFi 신호도 벽이나 금속에 약해서, 큰 집이나 사무실에서는 여러 개의 WiFi 기지국이 필요한답니다!
