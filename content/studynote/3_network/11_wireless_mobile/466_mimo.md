## 핵심 인사이트 (3줄 요약)
> 1. **본질**: MIMO(Multiple Input Multiple Output)는 다중 안테나를 활용하여同一 시간/주파수 자원에서 복수의 독립적 데이터 스트림을 동시에 전송하는 공간 다중화(Spatial Multiplexing) 기술로, 안테나 수 N개일 때 이론적 throughput이 N배로 증가한다.
> 2. **가치**: LTE-Advanced의 4x4 MIMO에서 5G NR의 256x256 Massive MIMO로 발전하면서, 동일 주파수 자원에서 100배 이상의 throughput 향상이 가능해졌으며, Beamforming을 통해 특정 방향으로 신호를 집중投射하여 커버리지도 개선되었다.
> 3. **융합**: Massive MIMO와 조합된 Beamforming은 5G mmWave의 핵심 기술로, 짧은 파장(28GHz에서 10.7mm)을 활용하여 수십 개의 안테나 앵원소를 소형화된 배열로 팩킹하고, 위상 배열(Phased Array)을 통해 电子적으로 빔 방향을 조절한다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념

MIMO는 송신측(Tx)과 수신측(Rx) 모두에 다중 안테나를 배치하여 복수의 데이터 스트림을 동시에 전송하는 기술이다. 전통적인 SISO(Single Input Single Output) 시스템이 시간이나 주파수라는 1차원적 자원만을 활용했다면, MIMO는 안테나가 만드는 공간(Space)이라는 2차원적 자원을 추가로 활용하여 주파수 효율을 비약적으로 향상시킨다. 핵심 원리는 다중 경로(Multipath)가 존재하는 환경에서 각 안테나 쌍 사이의 채널 응답이 상이하다는 점을 이용하는 것이다.

### 필요성

무선 스펙트럼이 희소한 자원인 이상, 시간이나 주파수만으로는 주파수 효율(spectral efficiency, bps/Hz)을 무한히 높일 수 없다. Shannon의 용량 공식 C = B log₂(1 + SNR)에서 대역폭 B를 늘리는 것에는 한계가 있으므로, 같은 대역에서 더 많은 정보를 보내려면 SNR을 높이거나, 새로운 차원을 추가해야 한다. MIMO는 공간(spatial)이라는 새로운 차원을 추가하여 Shannon 용량의 한계를 극복하는 방법을 제공한다.

### 등장 배경

1990년대 Stanford大学的 Gerard J. Foschini과 Bell Labs의 Alec Goldsmith 등이 MIMO의 이론적 기초를 확립했고, 1998-2001년 사이에 BLAST(Bell Labs Layered Space-Time) 코딩 등의 실용적 알고리즘이 제안되었다. 이후 2007년 IEEE 802.11n(WiFi)에 MIMO가 처음 표준화되었고, 2009년 3GPP LTE R8에서 2x2 MIMO가 도입되었으며, 2011년 LTE-Advanced(R10)에서 8x8 MIMO가 추가되었다.

### 💡 비유

MIMO는 **"입체 교차로"** 에 비유할 수 있다. 기존 도로는平面(시간/주파수 2차원)이라 차선 수(대역폭)가限られていただけだが, MIMO는 입체 교차로(공간 3차원)를 만들어 복수의 차선(공간 스트림)을 동시에利用できるようにした 것이다. Beamforming은 이 교차로의 신호등을 특정 차선(특정 사용자 방향)으로 우세하게 제어하는 것에 해당한다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### MIMO 분류 체계

MIMO는 안테나 구성에 따라 SISO, SIMO, MISO, MIMO, Massive MIMO로 나뉘며, 각각 특성이 다르다.

| 구분 | 안테나 수 (Tx×Rx) | 공간 다중화 | 증명 효과 | 복잡도 |
|:---|:---|:---|:---|:---|
| **SISO** | 1×1 | 없음 | 없음 (기본) |最低 |
| **SIMO** | 1×N | 없음 | Receive diversity (수신 향상) |低 |
| **MISO** | N×1 | 없음 | Transmit diversity (송신 향상) |低 |
| **MIMO** | N×M (N,M ≥ 2) | 가능 | Spatial multiplexing + diversity |中 |
| **Massive MIMO** | N×M (N,M ≫ 1) | 대규모 가능 | huge throughput ↑ + energy efficiency ↑ |高 |

### MIMO 동작 원리

MIMO의 핵심은 복수의 안테나를 통해 서로 다른 데이터 스트림을 동시에 전송하고, 수신 측에서 각 스트림을 분리해내는 것이다. 수신 측에서 복호(decoding)를 하려면 채널 상태 정보(CSI, Channel State Information)가 필요하다. TDD(Time Division Duplex) 시스템에서는 상하행 채널의reciprocity(상호성)을 활용하여 간소화된 CSI 획득이 가능하다.

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │                 MIMO 시스템의 동작 원리: 공간 다중화                               │
  ├─────────────────────────────────────────────────────────────────────────┤
  │
  │  [SISO vs MIMO 비교]                                                  │
  │
  │  SISO (1x1):                                                         │
  │                                                                     │
  │      [TX] ────────────────── [RX]                                    │
  │         1 스트림만 전송 가능                                            │
  │                                                                     │
  │  2x2 MIMO:                                                            │
  │                                                                     │
  │      [TX1] ─── 경로1 ───▶ [RX1]                                      │
  │      [TX2] ─── 경로2 ───▶ [RX2]                                      │
  │      [TX1] ─── 경로3 ───▶ [RX2]                                      │
  │      [TX2] ─── 경로4 ───▶ [RX1]                                      │
  │         (4개 경로 → 채널 행렬 H)                                        │
  │         동시에 2개 스트림 전송 가능                                       │
  │
  │  [MIMO 수학적 모델]                                                   │
  │
  │   y = Hx + n                                                         │
  │                                                                     │
  │   y: 수신 신호 벡터 (M×1)                                            │
  │   H: 채널 행렬 (M×N) — 각 원소는 안테나 쌍 간 채널 이득                  │
  │   x: 송신 신호 벡터 (N×1) — 송신할 데이터 스트림                        │
  │   n: 잡음 벡터 (M×1)                                                 │
  │                                                                     │
  │  Rank(H) = min(N,M) —— 전송 가능한 최대 스트림 수 결정                   │
  │  예: 4x4 MIMO → max rank 4 → 이론상 4배 throughput                     │
  │
  │  [공간 다중화 (Spatial Multiplexing) 동작]                              │
  │
  │   시간/주파수 ─────────────────────────────────────────────────▶       │
  │       │  Layer 1 ───▶  사용자 A (고속 데이터)                            │
  │       │  Layer 2 ───▶  사용자 B (중속 데이터)                            │
  │       │  Layer 3 ───▶  사용자 C (저속 데이터)                            │
  │       │  Layer 4 ───▶  사용자 D (저속 데이터)                            │
  │       │                                                              │
  │       │  동일 시간/주파수에서 4개 스트림 동시 전송                        │
  │
  │  [Diversity vs Spatial Multiplexing 트레이드오프]                        │
  │
  │  ┌─────────────────────────────────────────────────────────────────┐  │
  │  │  Diversity (전송 다중화):                                       │  │
  │  │  • 동일 데이터를 복수의 안테나/경로로 중복 전송                    │  │
  │  │  • SNR 향상,衰落에 강함                                          │  │
  │  │  • 예: LTE의 Transmission Diversity (TxAA)                     │  │
  │  │                                                               │  │
  │  │  Spatial Multiplexing (공간 다중화):                             │  │
  │  │  • 서로 다른 데이터를 복수의 안테나로 동시 전송                     │  │
  │  │  • throughput ↑↑, 단 채널 상황에 민감                           │  │
  │  │  • 예: LTE의 Open/Closed Loop Spatial Multiplexing             │  │
  │  │                                                               │  │
  │  │  트레이드오프: MIMO 모드 적응 (CQI, RI, PMI 기반)                  │  │
  │  │  • 채널 좋을 때 → Spatial Multiplexing优先                      │  │
  │  │  • 채널 나쁠 때 → Diversity mode优先                            │  │
  │  └─────────────────────────────────────────────────────────────────┘  │
  │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** MIMO의 핵심 수학적 모델 y = Hx + n에서, H(채널 행렬)의 rank가 최대 전송 가능 스트림 수를 결정한다. 4x4 MIMO에서 채널이良好하다면 H의 rank는 4가 되어 4개의 스트림을 동시에 전송할 수 있지만, 채널이 나쁘거나 경로 간 상관성이 높으면 rank가 줄어들어 실제 throughput이 떨어진다. LTE에서는 CQI(Channel Quality Indicator), RI(Rank Indicator), PMI(Precoding Matrix Indicator)를 UE가 측정하여 기지국에 보고하고, 기지국이 이를 기반으로 적절한 MIMO 모드와预编码(precoding) 행렬을 선택한다. Diversity 모드는 동일 데이터의 복사본을 여러 경로로 보내어衰落에 강하지만 throughput은 증가하지 않고, Spatial Multiplexing 모드는 다른 데이터를 동시에 보내 throughput을 늘리지만 채널 조건에 민감하다.

### Beamforming 동작 원리

Beamforming은 복수의 안테나에서放射된 신호의 위상(phase)과 진폭(amplitude)을 조절하여 특정 방향으로 건설적干涉(相加強化)을 만들고, 다른 방향으로는 파괴적干涉(相殺)を起こす技術이다. 안테나 배열(Antenna Array)에서 각 앵원소(Element)의 신호 위상을 조금씩 전환(phase shift)하면, 특정 방향으로 신호가 집중된 빔(Beam)이形成된다.

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │                      Beamforming (빔포밍) 동작 원리                            │
  ├─────────────────────────────────────────────────────────────────────────┤
  │
  │  [Analog vs Digital Beamforming]                                       │
  │
  │  Analog Beamforming:                                                   │
  │     각 안테나 앵원소에 RF phase shifter를 연결하여                      │
  │     신호 방향을 전자적으로 조절                                           │
  │     • 구조 간단, 비용 ↓                                                  │
  │     • 한 번에 하나의 빔만 형성 (broadcast에 유리)                        │
  │     • 예: 5G NR FR2의 초기 빔 검색 단계                                 │
  │
  │  Digital Beamforming:                                                  │
  │     각 안테나에 DAC/ADC를 연결하여                                          │
  │     기저대역에서 빔forming 가중치를 조절                                   │
  │     • 복수의 독립적 빔 동시 형성 가능                                    │
  │     • 복잡도 높음, 비용 ↑                                                │
  │     • 예: LTE Advanced의 MU-MIMO                                         │
  │
  │  Hybrid Beamforming (FR2 대부분):                                        │
  │     Analog (RF) + Digital (基带)의 hybrid 형태로                          │
  │     • 수십~수백 개의 안테나에 개별 RF 트랜시버 연결의 비용/복잡도를 줄임     │
  │     • reasonable한数量的 빔 동시 형성                                     │
  │     • 5G NR FR2의 표준 구현                                              │
  │
  │  [Phased Array (위상 배열) 원리]                                         │
  │
  │     ANT 0: │──▶ │──▶ │──▶ │──▶  위상 0°                                  │
  │     ANT 1: │──▶ │──▶ │──▶ │──▶  위상 45° (Δφ)                              │
  │     ANT 2: │──▶ │──▶ │──▶ │──▶  위상 90° (2Δφ)                             │
  │     ANT 3: │──▶ │──▶ │──▶ │──▶  위상 135° (3Δφ)                             │
  │                                                                     │
  │     θ 방향으로 최대利得을 위한 위상差:                                      │
  │     Δφ = (2πd/λ) × sin(θ)                                             │
  │                                                                     │
  │     d: 앵원소 간격, λ: 파장                                              │
  │     θ: 원하는 방향                                                     │
  │
  │  [Beamforming의 효과]                                                  │
  │
  │     Without Beamforming (全方向):                                       │
  │                                                                     │
  │         [eNB]                                                         │
  │          ╱ ╲                                                           │
  │         ╱   ╲                                                          │
  │        ╱     ╲                                                         │
  │       ○       ○  ← 모든 방향에 신호 분산, 수신 신호 세기 저하              │
  │                                                                     │
  │     With Beamforming (지향성):                                         │
  │                                                                     │
  │         [eNB]                                                         │
  │           │                                                            │
  │           │▶ (집중 빔)                                                 │
  │           │                                                            │
  │           ○ (집중된 신호, 더 강한 수신)                                  │
  │                                                                     │
  │     전선 발송 power 동일 → 빔 포밍으로 특정 방향 이득 ↑↑                   │
  │     (이론상 N개 안테나使用时 Array Gain = 10×log10(N) dBi)                │
  │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** Beamforming의 효과는 안테나 수(N)에 따라 이론상 10×log10(N) dBi의 배열 이득(Array Gain)을 제공한다. 예를 들어, 64개 안테나의 Massive MIMO는 이론상 약 18 dBi의 이득을 추가로 얻을 수 있어,同等 송신功率으로 더 먼 거리까지 도달하거나,同等 거리에서 더 낮은功率으로 동작 가능하다. 5G NR FR2(28GHz 등)에서는 Analog beamforming으로 전자적 شع향(beam direction) 제어를 하고, 빔 폭(beamwidth)이 약 3~10° 정도로 매우 좁아精准한 지향성이 가능하지만, 사용자가 빠르게 이동하면 빔追尾(beam tracking)가 필요하다. Hybrid beamforming은 FR2의 표준 방식으로, RF 层에서 coarse한 빔forming을 하고, 基带层에서 finer한 빔forming을 수행한다.

## Ⅲ. 융합 비교 및 다각도 분석

### WiFi vs LTE vs 5G NR의 MIMO 비교

| 구분 | WiFi (802.11n/ac) | LTE (R8/R10) | 5G NR (R15+) |
|:---|:---|:---|:---|
| **최대 MIMO** | 8×8 (ac wave 2) | 8×8 (R10) | 256×256 (FR2) |
| **하향 스트림** | SU-MIMO only | SU-MIMO | SU/MU-MIMO |
| **상향 MIMO** | 4×4 | 1×4 (SIMO) | 4×4 (R15), 2×4 (FR2) |
| **MU-MIMO** | 4×4 DL (ac) | 미지원 (R10), 4×4 DL (R12) | 8×8 UL/DL (R16) |
| **Beamforming** |Explicit (네비게이터) |Implicit (환산) | Explicit + Beamforming Support |
| **채널 추정** | MPEG-20 | CSI-RS | CSI-RS + SRS |

### Massive MIMO의 practical 효과

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │                    Massive MIMO의 Practical 효과 분석                         │
  ├─────────────────────────────────────────────────────────────────────────┤
  │
  │  [ 이론적 효과 vs 실제 효과 ]                                             │
  │
  │  ┌─────────────────────┬──────────────────────────────────────────┐   │
  │  │                     │           Massive MIMO 효과               │   │
  │  │                     ├──────────────────┬───────────────────────┤   │
  │  │                     │ 이론적 (무한대)  │  실제 (100~256 антенна)│   │
  │  ├─────────────────────┼──────────────────┼───────────────────────┤   │
  │  │ Spatial Multiplexing│ N배 throughput↑ │ 3~5배 실질 개선       │   │
  │  │ Array Gain         │ 10log10(N) dB   │ 20~25 dB (100 антенна)│   │
  │  │ Interference suppression│ 간섭 거의消除 │ 10~15 dB 간섭 감소 │   │
  │  │ Energy Efficiency   │ 1/N으로 ↓      │ N이 커질수록 효율 ↑  │   │
  │  └─────────────────────┴──────────────────┴───────────────────────┘   │
  │
  │  [실제 시스템에서 기대할 수 있는 실질적 이점]                                 │
  │
  │  • 동일 coverage을 유지하면서 BS_tx_power를 1/10~1/100로 줄일 수 있음       │
  │  • 동일 power에서 coverage radius를 3~5배 확장할 수 있음                   │
  │  • MU-MIMO로 동일 시간/주파수에 8명 이상의 사용자에게 동시 서비스 가능        │
  │  • 간섭 신호가 상대적으로 작아지므로 SINR이 전반적으로 향상됨                  │
  │
  │  [Massive MIMO의 practical limitation]                                    │
  │
  │  • 안테나 간격: λ/2가理想, 100GHz의 28mm파장에서 14mm간격으로 배치          │
  │    (FR2의 짧은 파장이有利, 하지만 panel 크기 miniaturization 필요)          │
  │  • Mutual coupling: 가까운 안테나 간 전자기 결합으로 인한 효율 저하          │
  │  • Pilot contamination: 동일 시간/주파수의 pilot序列을 많은 셀이 공유하여     │
  │    추정 CSI에 다른 셀의 영향이混合되는 근본적 제한 (TDD 기반)               │
  │  • UE 폼팩터: UE侧 8개 이상의 안테나를 배치하기 어려운小型화 문제            │
  │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** Massive MIMO의 이론적 효과는 매우 크지만, 실제 시스템에서는 여러 제한要因으로 인해 이론 대비 하위 성능을 보인다. 가장 큰 문제는 Pilot Contamination으로, TDD 시스템에서 상하행 채널 reciprocity를 활용하기 위해同一 pilot序列를 인접 셀이 재사용하게 되면, 추정한 CSI에 인접 셀의 신호가 섞여 들어가는 현상이 발생한다. 이 문제는 이론적으로 무한개의 안테나가 있으면 해소되지만, 실용적인 100~256 안테나 구성에서는 完全消除되지 않는다. 또한 UE侧의 폼팩터 제한으로 인해 상향 MIMO는 하향보다 지원 가능한 스트림 수가 적다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — LTE 네트워크에서 4×4 MIMO 성능 최적화**: 도시 지역 LTE 네트워크에서 4×4 MIMO 지원을 위해 기지국을升级하려고 하는 상황. 4×4 MIMO의 성능을最大化하려면 UE도 4×4 MIMO를 지원해야 하고, 채널 상태 정보(PMI, RI)가 정확하게 보고되어야 한다. 그러나 현재 UE의 대부분이 2×2 MIMO만 지원하거나, 4×4 지원 UE라 하더라도 상향은 1×4 SIMO인 경우가 대부분이다. 따라서 하향 4×4 MIMO의 실질적 이점은 제한적이고, MU-MIMO(다중 사용자 MIMO) 같이 여러 사용자에게 동시에 전송하는 방식이 더 현실적인 성능 향상을 줄 수 있다.

2. **시나리오 — 5G FR2(mmWave) 대규모 MIMO 배치**: 경기장이나스타트업 지역에 5G FR2 mmWave 기지국을部署하려고 하는 상황. 28GHz에서 256x256 Massive MIMO를実装하려면, 각 앵원소 간격 λ/2 ≈ 5.35mm로密集 배치해야 하므로, 패널(panel) 크기가 상당히 커진다(256 앵원소 기준 약 8×8 배치 시 약 43×43cm). 또한 각 앵원소에 연결되는 RF 체널(TRX, Transceiver) 비용이 전체 시스템 비용의 대부분을 占め므로, Hybrid Beamforming으로 RF 체널 수를 줄이는 것이 비용 효율적입니다.

3. **시나리오 — TDD 시스템에서 간섭 관리**: TDD 기반 LTE-TDD 또는 5G NR에서 상하행 시간 비율(Uplink/Downlink Ratio)을 설정할 때 발생하는 문제. 상행이 많은 경우(Upload-heavy scenario) 더 많은 시간 자원을 상행에 배분해야 하지만, 그렇게 하면 상행-GAN(Generic Area Network) 간섭이 증가할 수 있다. 또한 TDD에서 가장 중요한 것은 인접 셀과의 상하행 동기화(동일 프레임 구조 사용)이다. 동기화되지 않으면 인접 셀의 상행 신호가 내 셋의 하행 수신에 간섭을 줄 수 있다.

### 도입 체크리스트

- **기술적**: MIMO 등급(Tx/Rx 안테나 수)이 네트워크 성능 목표에 부합하는가? UE 카테고리에 따른 MIMO 지원 수준을 파악하고 있는가? TDD의 경우 인접 셀과의 동기화 구성이 정확한가?
- **운영·보안적**: Massive MIMO의 빔forming 정보(Beamforming State)가適切に管理되어 합법적 도청이나 spoofing에 악용되지 않는가?

### 안티패턴

- **안테나 상관도 과소평가**: 안테나 앵원소 간격이 너무 좁으면 mutual coupling과 상관도가 증가하여 MIMO 성능이 급격히 저하된다. FR1에서는 λ/2 ≈ 10cm(2GHz) 간격이 필요하고, FR2(28GHz)에서는 λ/2 ≈ 5.35mm로密集 배치되어도 된다. 반드시 이 간격을 확보해야 한다.
- **UE MIMO 능력 과대평가**: 네트워크가 8×8 MIMO를 지원하더라도, 대다수 UE가 2×2 MIMO만 지원하면 실제用户体验은 2×2 수준에 머무른다. UE Capability를 기반으로 realistic한 성능 목표를 설정해야 한다.

- **📢 섹션 요약 비유**: MIMO는 **"입체停车场"** 에 비유할 수 있다. 기존平面 주차장은地面(시간/주파수) 면적에限られて 차의 수(데이터)가 제한되지만, MIMO는立体 주차장(공간을 추가)으로 확장하여同一 면적에서 더 많은 차(데이터 스트림)를 주차할 수 있다. Beamforming은立体 주차장의电梯가特定 차(특정 사용자)만 있는 floor로 直行하는 것에 해당한다.

### 미래 전망

- **Full-Digital Beamforming의 보급화**: ADC/DAC 비용이 하락하고, RF 체널 통합(Integrated RF) 기술이 발전함에 따라, 향후 FR1에서도 Full-Digital Beamforming이 가능해져 더 유연한 다중 빔 형성이 예상된다.
- **AI/ML 기반 MIMO 최적화**: CSI 예측, 빔 관리, 스케줄링에 AI/ML을 활용하여, 특히 고속 이동 환경에서 빠른 채널 변화에 대응하는 지능형 MIMO 시스템이 연구되고 있다.

**[다이어그램 해설]** MIMO 기술은 안테나 수의 확장과 함께 진화해왔다. 초기의 MIMO(2×2)는 WiFi n과 LTE R8에서 처음 도입되어throughput 2배 향상을 실현했다. LTE-Advanced(R10)의 8×8 MIMO는 이론적으로 8배 throughput을 제시했지만, 실효는 채널 상태와 UE 능력에 크게 의존했다. Massive MIMO(64×64 이상)는 5G와 함께 등장하여 FR2 mmWave의 짧은 파장을 활용하여 대규모 안테나 배열을 소형화할 수 있었고, Array Gain과 간섭 억제 효과를 통해 기존의 셀룰러를颠覆하는 성능 향상을 이끌어냈다. 향후 AI/ML 기반의 지능형 MIMO가 고속 이동 환경이나 비정형 채널 환경에서 추가적인 성능 개선을 가져올 것으로 기대된다.

- **📢 섹션 요약 비유**: MIMO의进化는 **"소리나는 곳을 동시에 찾는 기술"** 의 진화라고 할 수 있다. 처음에는 소리从一个方向에서만 들었지만(SISO), 이후 여러方向的으로 동시에 들을 수 있게 되었고(MIMO), 거기에 더해 "반향筒"을 특정 방향으로 모아 소리를 크게 하는 기술(Beamforming)까지 더해져, 더 멀리 있고 더 조용한 소리도明瞭하게 들을 수 있게 되었다.

| **Pilot Contamination** | TDD Massive MIMO에서 동일 pilot序列를 인접 셀이 재사용할 때 발생하는 근본적 제한으로, 추정 CSI에 다른 셀의 신호가 섞여 들어와 성능 저하를 유발한다. |

## 👶 어린이를 위한 3줄 비유 설명

1. MIMO는 **"여러 개의 입과 귀를 가진怪物"** 에 비유할 수 있어요.普通 사람은 입이 1개, 귀가 1개라서 한 번에 하나의 말만 할 수 있지만, MIMO怪物은 입이 4개, 귀가 4개라서 네 가지 다른 말을 동시에 할 수 있어요.
2. Beamforming은 **"확성기와 손을 동시에 쓰는 것"** 과 같아요.擴声器는 모든方向に 소리를 보내지만, 손을 모으면 특정 방향으로 소리를集中시켜 더 멀리까지 전달할 수 있어요. MIMO에서도 안테나들의 신호를 특정方向으로 모으면 거기에 있는 사람이 더-strong한 신호를 받을 수 있어요.
3. Massive MIMO는怪物이 엄청나게 많아진 것으로, 입과 귀가 64개 또는 256개씩 있어 더 많은 말을 동시에 할 수 있어요. 하지만 입이 너무 가까이 있으면(안테나 간격 좁으면) 서로干扰하여 제대로 들을 수 없듯이, 적절한 거리를 유지하는 것이 중요해요.
