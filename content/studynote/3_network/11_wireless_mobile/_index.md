## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 무선통신은 전파(Radio Wave)를 매질로 활용하여 공간적 제약 없이 네트워크 접속을 가능하게 하는 기술로, WiFi(802.11), 블루투스(Bluetooth), LTE(4G), 5G NR 등 다양한 표준이 존재하며 각각 다른 주파수 대역과 접근 방식을 채택하고 있다.
> 2. **가치**: WiFi 6는 OFDMA와 MU-MIMO로 고밀도 환경에서 지연 시간 75% 감소를 달성했고, 5G NR은 mmWave와 Massive MIMO로 최대 20Gbps throughput과 1ms 이하 지연이라는 두 가지 상반된 요구사항을 동시에 충족한다.
> 3. **융합**: WiFi 6E와 5G NR은 6GHz 대역 충돌, WiFi Direct와 5G D2D 경쟁, Passpoint와 5G N3IWF 통합 등으로 상호 침범하며融合하고 있으며, 차후 WiFi 7과 6G에서는 이 경계가さらに模糊될 전망이다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 개념

무선통신은 유선 케이블 없이 전파를 통해 데이터를 전송하는 기술이다. 유선 이더넷이 물리적 포트와 케이블을 필요로 하는 반면, 무선통신은 공간 자체를 매질로 활용하여 이동성과 연결의 유연성을 제공한다. IEEE 802.11 표준을 따르는 WiFi, Bluetooth SIG 표준을 따르는 블루투스, 3GPP 표준을 따르는 LTE와 5G NR이 대표적인 무선통신 기술이다. 각 기술은 설계 목표가 달라 서로 다른 니치에서 최적화되어 있다.

### 필요성

현대 사회에서 무선통신의 필요성은 세 가지 측면에서 증가하고 있다. 첫째, 모바일 디바이스의 확산으로いつでもどこでも 네트워크에 접속하려는 수요가 폭발적으로 증가했다. 둘째, IoT 센서와 웨어러블 기기의 대량 연결을 감당할 수 있는 저전력 무선 기술이 필수적이 되었다. 셋째, HD 영상 스트리밍과 화상 회의 등 대역폭 집약적 서비스의 보편화로 인해 이전 세대의 무선 기술로는 감당할 수 없는 데이터 요구량이 발생하고 있다.

### 등장 배경

1997년 최초의 IEEE 802.11 표준이 2.4GHz 대역에서 2Mbps 속도로 등장했으나 상용화에는 실패했다. 1999년 802.11b(11Mbps)와 802.11a(54Mbps)가 동시에 등장하면서 WiFi 상용화의 서막이 열렸고, 2003년 802.11g(54Mbps, 2.4GHz)가 세계적 보급률 1위를 달성했다. 블루투스는 1998년 1.0이 출시되어 케이블 대체 기술로 자리잡았고, 2010년 BLE 4.0의 등장은 저전력 IoT 시장을 열었다. LTE는 2008년 3GPP R8에서 첫 표준이 완성되어 4G 시대를 열었으며, 5G NR은 2018년 3GPP R15에서 첫 표준이 완성되어 2019년 상용화가 시작되었다.

### 💡 비유

무선통신은 **"공기 중의 레이저 통신"** 에 비유할 수 있다. 레이저는 직진하지만 벽에 막히는 반면, WiFi 전파는 공간을 투과하여 어디서든 신호를 받을 수 있다. 각 세대의 속도 향상은 레이저의 세기를 높여 더 먼 거리에서 더 빠른 속도를 얻는 것과 같다. 블루투스는 특정 사람만 듣는 무전기 대화이고, LTE/5G는 도시 전체를覆盖하는广播放送이라고 생각하면 이해하기 쉽다.

### 📢 섹션 요약 비유

무선통신 기술의 진화는 좁은 골목길(2.4GHz 단일 채널)에서 넓은 고속도로(6GHz 320MHz 8차로)로 차선이 증가하고, 신호 제어 시스템(OFDMA)이 도입되어 더 많은 차량(디바이스)이 동시에 빠르게 이동할 수 있게 된 것과 같다.

### WiFi 세대별 진화 과정

WiFi의 발전은 세 가지 축으로 진행된다. 첫째, 변조 방식의 효율성 증가로 스펙트럼 효율이 개선되었다. 둘째, 다중 안테나 기술(MIMO)로 공간을 활용한 병렬 전송이 가능해졌다. 셋째, OFDM에서 OFDMA로 전환되어 다중 사용자 동시 접속 효율이 크게 향상되었다.

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │                    WiFi 세대별 핵심 파라미터 비교                               │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                         │
  │  ┌───────────┬────────┬──────────────┬─────────────┬────────────────┐ │
  │  │ 세대       │ 연도   │ 최대 속도     │ 주파수 대역   │ 주요 변조 방식  │ │
  │  ├───────────┼────────┼──────────────┼─────────────┼────────────────┤ │
  │  │ 802.11a   │ 1999   │ 54 Mbps      │ 5 GHz       │ OFDM           │ │
  │  │ 802.11b   │ 1999   │ 11 Mbps      │ 2.4 GHz     │ DSSS           │ │
  │  │ 802.11g   │ 2003   │ 54 Mbps      │ 2.4 GHz     │ OFDM           │ │
  │  │ 802.11n   │ 2009   │ 600 Mbps     │ 2.4/5 GHz   │ MIMO-OFDM      │ │
  │  │ 802.11ac  │ 2013   │ 6.9 Gbps     │ 5 GHz       │ MU-MIMO-OFDM   │ │
  │  │ 802.11ax  │ 2019   │ 9.6 Gbps     │ 2.4/5/6 GHz │ OFDMA          │ │
  │  │ 802.11be  │ 2024~  │ 46 Gbps (예) │ 2.4/5/6 GHz │ MLO (Multi-Link)│ │
  │  └───────────┴────────┴──────────────┴─────────────┴────────────────┘ │
  │                                                                         │
  │  DSSS: Direct Sequence Spread Spectrum — 넓은 대역에 신호 확산             │
  │  OFDM: 직교 부반송파로 대역 분할 다중화                                     │
  │  MIMO: 다중 안테나로 공간 다중화                                           │
  │  OFDMA: 서브캐리어 그룹을 사용자별 할당                                     │
  │  MLO: 여러 링크를 동시에 사용하는 차세대 병렬 전송                           │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 이 비교표는 WiFi의 진화 방향이 단순히 "속도 향상"만이 아니라 동시 접속 디바이스 밀도 관리와 스펙트럼 효율성 개선에 초점이 맞춰져 있음을 보여준다. 802.11b의 DSSS는 2.4GHz ISM 대역에서 간섭에 강하지만 스펙트럼 효율이 낮아 속도에 한계가 있다. 802.11n의 MIMO 도입은 공간 다중화로 이론적 throughput을 4배 이상 끌어올렸지만, 실환경에서 기대 이하의 성능이 보고되었다. 802.11ax의 OFDMA는 상하행 모두에서 다중 사용자 동시 전송을 지원하여 고밀도 환경의 지연 시간을 획기적으로 개선했다. 802.11be(WiFi 7)의 MLO는 여러 링크를 동시에 사용하여 지연을 최소화하는 차세대 기술이다.

### LTE vs 5G NR 아키텍처 비교

LTE는 무선 접근망(eUTRAN)과 코어 네트워크(EPC)로 구성되며, 5G NR은 NG-RAN과 5GC(5G Core)로 구성된다. 가장 큰 차이점은 5GC가 Service-Based Architecture(SBA)를 채택하여 네트워크 기능들이 RESTful API로 상호 통신한다는 점이다.

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │              LTE EPS vs 5G 시스템 아키텍처 비교                              │
  ├─────────────────────────────────────────────────────────────────────────┤
  │
  │  [LTE EPS (Evolved Packet System)]                                    │
  │
  │    [UE] ─── LTE Uu ─── [eNB] ─── S1 ─── [EPC]                        │
  │                                      │                                 │
  │                              ┌────────┴────────┐                      │
  │                              │                 │                      │
  │                           [MME]           [S-GW]                      │
  │                        (제어 평면)      (사용자 평면)                   │
  │                              │                 │                      │
  │                              └────────┬────────┘                      │
  │                                       │                                │
  │                                    [P-GW]                             │
  │                                       │                                │
  │                                 [Internet]                           │
  │
  │  [5G System (NG-RAN + 5GC)]                                           │
  │
  │    [UE] ─── NR Uu ─── [gNB] ─── N2 ─── [5GC]                         │
  │                                      │                                 │
  │                           ┌──────────┴──────────┐                     │
  │                           │  Service-Based      │                     │
  │                           │  Architecture (SBA)  │                     │
  │                           ├──────────────────────┤                     │
  │                           │ AMF │ SMF │ UPF │  │                     │
  │                           │     │     │     │ ...│                     │
  │                           └──────────────────────┘                     │
  │                                      │                                │
  │                                    [Data Network]                      │
  │
  │  [핵심 차이점]                                                          │
  │
  │  ┌───────────────────────────────────────────────────────────────────┐ │
  │  │  항목            │ LTE                  │ 5G NR                    │ │
  │  ├──────────────────┼──────────────────────┼──────────────────────────┤ │
  │  │ 코어 네트워크    │ EPC (다중 노드)       │ 5GC (SBA, microservices) │ │
  │  │ 제어/사용자 평면 │ 분리 (MME vs S-GW)   │ AMF/SMF 분리 (N4 인터페이스) │ │
  │  │ 인터페이스       │ S1 (TNL), X2 (TNL)   │ N2/N3 (TNL), Xn (TNL)    │ │
  │  │ 음성 처리        │ VoLTE (IMS 필요)     │ VoNR (IMS)               │ │
  │  │ 네트워크 슬라이싱 │ 미지원               │ 네이티브 지원             │ │
  │  │ 지연 시간        │ 10ms RTT            │ 1ms 이하 (URLLC)         │ │
  │  └───────────────────────────────────────────────────────────────────┘ │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** LTE EPC와 5GC의 가장 근본적 차이는 아키텍처 철학에 있다. EPC는 전통적인 노드 기반 설계로, 각 노드(MME, S-GW, P-GW)가 전용 하드웨어에서 실행되고 이기종 인터페이스로 연결된다. 5GC는 SBA(Service-Based Architecture)를 채택하여 네트워크 기능(NF)들이 RESTful API로 상호 통신하며, 각 NF가 독립적으로 스케일링 가능하다. 이는 차세대 네트워크가 다양한 서비스 요구사항에 유연하게 대응할 수 있는 기반을 제공한다. 5GC에서 AMF(Access and Mobility Management Function)는 제어 평면을, UPF(User Plane Function)는 사용자 평면을 담당하며, 이 둘이 N4 인터페이스로 분리된 것이 EPC의 MME/S-GW 분리보다 더 강력한 결합도 유연성을 제공한다.

### OFDM과 OFDMA 동작 원리

OFDM(Orthogonal Frequency Division Multiplexing)은 하나의 넓은 대역폭 채널을 52개의 직교 서브캐리로 분할하는 기술이다. 802.11ax의 OFDMA는 이 OFDM을 발전시켜 각 사용자에게 서브캐리어 그룹(Resource Unit, RU)을 할당하는 방식으로 다중 접속을 실현한다.

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │               OFDM vs OFDMA: 서브캐리어 할당 방식 비교                       │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                         │
  │  [OFDM 방식 — 단일 사용자가 전체 대역 사용]                                │
  │                                                                         │
  │  주파수: ─────────────────────────────────────────────────▶             │
  │         │ P │ D │ D │ D │ D │ D │ D │ D │ D │ D │ D │ D │ P │          │
  │         │ P=파일럿, D=데이터 (52개 서브캐리어 중 48개 사용)               │
  │                     └─────── 사용자 1 독점 ───────┘                     │
  │                                                                         │
  │  [OFDMA 방식 — 다중 사용자가 대역 분할 사용]                              │
  │                                                                         │
  │  주파수: ─────────────────────────────────────────────────▶             │
  │         │ RU1 │ RU2 │ RU3 │ RU4 │ RU5 │ ... │ RU9 │                     │
  │           │      │      │      │      │           │                     │
  │         사용자1  사용자2  사용자3  사용자4  ...    사용자9               │
  │                                                                         │
  │  RU (Resource Unit) 크기: 26, 52, 106, 242, 996 개의 서브캐리어         │
  │  20MHz 채널 → 최대 9개 RU (26-서브캐리어 RU) 동시 할당 가능             │
  │                                                                         │
  │  ┌───────────────────────────────────────────────────────────────────┐ │
  │  │  OFDMA 장점:                                                      │ │
  │  │  ① 지연 시간 감소 — 사용자 대기 없이 동시 전송                      │ │
  │  │  ② 스펙트럼 효율 향상 — 대역 활용률 최대 100%                      │ │
  │  │  ③ 고밀도 환경 최적화 — 다중 IoT 디바이스 동시 접속                  │ │
  │  │  ④ TWT (Target Wake Time)로 IoT 절전 가능                        │ │
  │  └───────────────────────────────────────────────────────────────────┘ │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** OFDM 방식에서 사용자 1이 전체 채널을 사용하면 다른 사용자たちは 전송 기회를 기다려야 하므로 지연이 누적된다. OFDMA에서는 RU라는 작은 단위로 채널을 분할하여 여러 사용자가 동시에 데이터를 전송할 수 있다. 802.11ax의 핵심 개선점은 TDMA에서 FDMA로 전환하여 채널 활용 효율을 극대화했다는 것이다. 특히 IoT 환경에서는 수백 개의 센서가 주기적으로 소량 데이터를 보내는데, 각 센서에 작은 RU를 할당하여 에너지 효율과 응답성을 동시에 개선한다. TWT(Target Wake Time)는 스케줄러와 연동되어 디바이스가 특정 시간에 깨어나 데이터를 주고받도록 하여 IoT 디바이스의 배터리 수명을 크게 연장한다.

### 📢 섹션 요약 비유

LTE의 네트워크 구조는 **"고속철도 운영 시스템"** 에 비유할 수 있다. eNB(기지국)는 역이고, MME는 열차 스케줄 관리자(제어 평면)이며, S-GW/P-GW는 화물 배송 센터(사용자 평면)이다. 모든 열차(데이터/음성)가 표준 궤간(IP 위에서運行하기 때문에 서로 다른 종류의 화물을 적재하더라도 동일 선로 위를走行할 수 있다.

## Ⅲ. 융합 비교 및 다각도 분석

### WiFi vs 블루투스 vs LTE 비교

세 기술은 서로 다른 최적화 목표를 가지고 있으며, 각기 다른 니치에서 작동한다.

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │              근거리 무선 기술 비교 (WiFi / Bluetooth / LTE)                  │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                         │
  │  ┌──────────┬──────────────┬───────────────┬────────────────────────┐    │
  │  │ 항목     │ WiFi (802.11) │ Bluetooth    │ LTE (4G)             │    │
  │  ├──────────┼──────────────┼───────────────┼────────────────────────┤    │
  │  │ 범위     │ 30~100m      │ 10~100m      │ 수 km (광역)          │    │
  │  │ 주파수   │ 2.4/5/6 GHz  │ 2.4/5 GHz    │ 0.7~2.6 GHz (特许)     │    │
  │  │ 속도     │ up to 9.6G   │ up to 54M    │ up to 1G (category)   │    │
  │  │ 전원 소비 │ 높음         │ 낮음          │ 높음 (모듈)            │    │
  │  │ 지연     │ 3~10ms       │ 2~50ms       │ 10~50ms (空中)         │    │
  │  │ 주 용도  │ 대용량 데이터  │ 웨어러블/오디오│ 이동 통신/브로드밴드    │    │
  │  │ 동시 기기 │ 수십 대       │ 7대 (classic) │ 수천 대 (cells)         │    │
  │  └──────────┴──────────────┴───────────────┴────────────────────────┘    │
  │                                                                         │
  │  ※ WiFi: 고대역폭, 항상 전원 필요                                       │
  │  ※ Bluetooth: 저전력, 짧은 데이터, 페어링 기반                           │
  │  ※ LTE: 광범위, 이동성, 통신사 종속                                      │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 세 기술은 서로 다른 니치(niche)를 확보하고 있다. WiFi는 사무실/집 환경에서 고대역폭 디바이스(노트북, TV, 게임기) 연결의 표준으로 자리잡았다. 블루투스는 웨어러블, 이어폰, 사물인터넷 센서처럼 저전력으로 짧은 데이터를 주고받는 시나리오에 적합하다. LTE는 통신사 네트워크가覆盖하는 곳이라면 어디서든 이동 중에도 데이터 연결이 가능하지만, 모듈 비용과 데이터 요금 문제가 있다.

### 4G/5G 세대별 진화 비교

| 비교 항목 | LTE (R8) | LTE-A (R10) | 5G NR (R15+) |
|:---|:---|:---|:---|
| **하향 속도** | 100 Mbps | 1 Gbps | 20 Gbps (NR) |
| **접속 방식** | OFDMA (DL), SC-FDMA (UL) | OFDMA + Carrier Aggregation | OFDMA |
| **대역폭** | 1.4~20 MHz | Up to 100MHz (5x20MHz CA) | 5~100 MHz (FR1), 50~400 MHz (FR2) |
| **코어 네트워크** | EPC (IP 기반) | EPC 강화 | 5GC (Service-based) |
| **지연 시간** | 10ms RTT | 10ms RTT | 1ms (URLLC) |
| **음성 처리** | VoLTE (IMS 필요) | VoLTE | VoNR (IMS) |
| **주요 기술** | OFDMA, SC-FDMA | CA, 8x8 MIMO | mmWave, Massive MIMO, Network Slicing |

### 📢 섹션 요약 비유

5G NR의 세 가지 시나리오(eMBB, URLLC, mMTC)는 **"같은 도시에 세 가지 다른 도로망을 운영하는 것"** 에 비유할 수 있다. 일반 시민용 도로(eMBB)는 넓고 빨라야 하고, 긴급 차량용 도로(URLLC)는 신호등 없이 거의 즉시 도착해야 하고, 화물 트럭/자전거/人 전용도로(mMTC)는 여러 대가 동시에 들어가면서도 서로 interference하지 않아야 한다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 고층 사무동 WiFi 커버리지 설계**: 30층 건물에서 각 층당 50명의 사용자가 동시에 WiFi를 사용하는 환경에서, 기존 2.4GHz 단일 채널 AP 배포 시 간섭과 속도 저하 문제가 발생하는 상황. 해결책으로 5GHz 대역 802.11ac AP를 각 층당 3개 이상 배치하고 채널 할당(1, 6, 11 채널)을 최소화하여 셀 간 간섭을 줄이는 설계를 적용해야 한다.

2. **시나리오 — BLE 기반 자산 추적 시스템**: 병원 환경에서 5,000개의 의료 기기에 BLE 비콘을 부착하여 위치를 실시간 추적하려는 상황. 핵심 이슈는 5,000개 태그의 데이터를 어떻게 효율적으로 수집하느냐이다. 해결책으로 BLE scanning을 수행하는 게이트웨이를 병원 곳곳에 配置하고, 각 태그는 advertisement를 100ms interval로 설정하며, RSSI를 활용하여三角測량으로 위치를 산출한다.

3. **시나리오 — 5G FWA (Fixed Wireless Access) 배포**: 통신사가 mmWave 기반 5G를 활용하여 가정에 고정 무선 인터넷(FWA)을 제공하려는 상황. FR2(mmWave) 커버리지가 목표 지역에서 충분한가? 해결책으로 FR1(Sub-6GHz, 3.5GHz)를 함께 배포하여 coverage를 확보하고, FR2는 throughput이 필수적인 지역에 한정하는 것이 일반적이다.

### 도입 체크리스트

- **기술적**: AP 배치 전 Site Survey를 수행하여 간섭원과 커버리지 빈틈을 분석했는가? 주파수 대역(2.4/5/6GHz) 선택이 사용 디바이스 호환성과 간섭 환경에 적합한가? 5G NR FR2 커버리지 planning 시 건물 penetration 손실(20~35 dB)을 반영했는가?
- **운영·보안적**: WPA3 Personal 또는 Enterprise 보안을 적용했는가? BLE 보안(AES-CCM)이 활성화되어 있는가?

### 안티패턴

- **채널 重複配置**: 인접 AP 간 채널 간섭을 고려하지 않고 동일 채널을 중복 배치하면 성능이 반감된다. 반드시 1/6/11(2.4GHz) 또는 인접하지 않은 5GHz 채널을 사용해야 한다.
- **FR2 과대평가**: mmWave의 커버리지를 과대평가하여 기지국간 거리를 너무 넓게 배치하면 coverage 공백이 발생한다. FR2는 100~300m Reichweite를 기준으로 planning해야 한다.
- **불필요한 장기간 BLE 연결 유지**: 연결이 필요 없는 센서가 지속적인 BLE 연결을 유지하면 Peripheral의 전력이 불필요하게消耗된다.

### 📢 섹션 요약 비유

WiFi 세대 진화는 넓은 도로 위에 들어가는 차선의 수와 신호 제어 방식의 진화로 비유할 수 있다. OFDMA는 톨게이트의 하이패스 차선을 더 세분화하여 동시에 더 많은 차량이 통과하게 하는 것과 같다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 기존 방식 (802.11ac) | WiFi 6 적용 후 | 개선 효과 |
|:---|:---|:---|:---|
| **정량** | 사용자당 200 Mbps | 사용자당 600 Mbps (4x MIMO-OFDMA) | throughput **3배** 향상 |
| **정량** | 지연 시간 30ms | 지연 시간 7.5ms | latency **75%** 감소 |
| **정성** | 고밀도 환경 열화 | 다중 디바이스 동시 서비스 | 스타디움/학원지구 적합 |

### 미래 전망

- **WiFi 7 (802.11be EHT)**: 320MHz 채널 폭과 MLO(Multi-Link Operation)로 최대 46Gbps throughput 달성을 목표로 한다. 6GHz 대역 중심이며, VR/AR 체험과 8K 스트리밍 시나리오에 최적화될 전망이다.
- **6G (2030+)** : THz (>100 GHz) 대역, 자율 AI-native 네트워크, 위성·HAPS 통합을核心으로 할 전망이다.
- **WiFi와 5G 공존**: 산업용 IoT 환경에서 5G URLLC 요구를 WiFi 6/6E가 도전하는 구도가 형성되고 있으며, 각 기술의 적합 영역이 재정의되고 있다.

**[다이어그램 해설]** 무선통신의 발전은 크게 세 가지 방향으로 진행된다. 첫째, 스펙트럼 확장(2.4→5→6GHz)으로 더 넓은 대역폭을 확보한다. 둘째, MIMO와 OFDMA의 진화로 공간 효율성을 높인다. 셋째, MLO 같은 기술로 여러 링크를 병렬 활용하여 지연을 최소화한다. WiFi 7 이후에는 5G 셀룰러의 고신뢰低지연 영역과 직접 경쟁하는 영역이 생길 것으로 예상되며, 향후 WiFi 8에서는 THz 대역 활용도 연구되고 있다.

- **📢 섹션 요약 비유**: 무선통신의 진화는 좁은 골목길(2.4GHz单一车道)에서 넓은 고속도로(6GHz 320MHz 8차로)로 차선이 증가하고, 신호 제어 시스템(OFDMA)이 도입되어 더 많은 차량(디바이스)이 동시에 빠르게 이동할 수 있게 된 것과 같다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **OFDMA (직교 주파수 분할 다중 접속)** | 802.11ax의 핵심 기술로, 하나의 채널을 다중 사용자에게 서브캐리어 그룹(RU)으로 할당하여 동시 전송을 가능하게 한다. |
| **MU-MIMO (다중 사용자 MIMO)** | AP가 다중 클라이언트에게 동시에 공간 스트림을 할당할 수 있어, 전체 시스템 용량을 크게 향상시킨다. |
| **Massive MIMO** | 수십~수백 개의 안테나를 활용한 MIMO 기술로, 공간 다중화를 통해同一 시간/주파수 자원에서 다중 사용자에게 동시 서비스를 제공한다. |
| **Beamforming** | 안테나 배열의 각 앵원소 위상을 조절하여 특정 방향으로 信号을 집중投射하는 기술로, mmWave에서 커버리지 확대와 간섭 감소에 핵심적이다. |
| **mmWave (밀리미터파)** | 24~100 GHz 대역의 초고주파수로, 짧은 파장 덕분에 huge한 대역폭 확보가 가능하지만 장애물에 매우 취약하여 beamforming 기반의 정확한 지향성 전송이 필수이다. |
| **BLE GATT (Generic Attribute Profile)** | BLE에서 데이터를 구조화하는 프로파일로, Service-Characteristic-Descriptor 계층을 통해 센서 데이터를 논리적으로 조직화한다. |

## 👶 어린이를 위한 3줄 비유 설명

1. WiFi는 **"공기 중의 말풍선"** 과 같아요. 여러 사람이 동시에 공기 중에서 다른 색깔의 말풍선을 서로 던져주면, 각 사람은 자기 색깔의 말풍선만 잡을 수 있듯이 여러 기기가 동시에 데이터를 주고받을 수 있어요.

2. WiFi 세대가 높아질수록 말풍선을 더 많이 동시에 던져주고, 더 빠르게 던질 수 있어서 여러 친구들이 동시에 큰 데이터를 보내 받아도 서로 기다리지 않아도 돼요.

3. 하지만 말풍선은 벽을 통과하지 못하듯이 WiFi 신호도 벽이나 금속에 약해서, 큰 집이나 사무실에서는 여러 개의 WiFi 기지국이 필요한답니다!
