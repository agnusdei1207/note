# 562. MIPv4 (Mobile IPv4) - FA, HA, 세모 라우팅 문제

> **핵심 인사이트**: 노트북을 들고 카페에 가서 와이파이를 잡으면 IP 주소가 바뀐다. IP가 바뀌면 다운로드 받던 영화가 끊긴다. 만약 전 세계 어디를 돌아다녀도 내 원래 IP 주소가 바뀌지 않는다면? MIPv4는 우체국(HA)이 내 본가 주소로 온 택배를 내가 머물고 있는 호텔(FA)로 재포장해서(터널링) 보내주는 IP 이동성 보장 기술이다.

## Ⅰ. Mobile IP의 등장 배경
기존 TCP/IP 환경에서는 단말기가 다른 서브넷(다른 공유기/라우터)으로 이동하면 반드시 IP 주소를 새로 할당받아야 하며, 이 과정에서 맺어져 있던 TCP 세션(다운로드, 화상회의 등)이 전부 끊어지게 됩니다. **Mobile IP**는 단말기가 물리적인 망을 옮겨 다녀도 **변하지 않는 고정 IP(Home Address)**를 사용할 수 있게 해주는 IETF 표준 프로토콜(RFC 3344)입니다.

## Ⅱ. MIPv4의 3대 핵심 구성 요소 🌟
1. **MN (Mobile Node)**: 이동하는 단말기 (예: 노트북, 스마트폰)
2. **HA (Home Agent)**: 단말기의 본래 고향 네트워크에 있는 라우터. 단말기가 집을 떠나 있을 때, 단말기의 고정 IP(Home Address)로 날아오는 모든 패킷을 대신 가로채어 현재 위치로 넘겨주는 **'본가 우체국장'** 역할을 합니다.
3. **FA (Foreign Agent)**: 단말기가 새롭게 방문한 타지 네트워크에 있는 라우터. HA가 보내준 패킷을 받아 단말기에게 전달해 주는 **'타지 호텔 프론트 직원'** 역할을 합니다. 단말기가 도착하면 임시 주소인 **COA (Care-of Address)**를 부여합니다.

## Ⅲ. MIPv4의 동작 방식 (IP in IP Tunneling)
1. 노트북(MN)이 타지 망(FA)에 도착하면 임시 주소(COA)를 받고, 이를 고향의 HA에게 신고(Registration)합니다.
2. 외부 서버(CN)가 노트북의 원래 고정 IP로 패킷을 보냅니다.
3. 고향망의 **HA**가 이 패킷을 가로챕니다.
4. HA는 원래 패킷 겉에 **'목적지: 타지 임시 주소(COA)'를 적은 새로운 IP 헤더를 덧씌워(IP-in-IP 터널링)** FA로 쏩니다.
5. FA는 겉포장을 벗겨내고 알맹이 패킷만 노트북에게 전달합니다.

## Ⅳ. 치명적인 단점: 세모 라우팅 문제 (Triangular Routing)
- **문제점**: 외국 서버(CN)가 타지에 있는 노트북에게 패킷을 보낼 때, 바로 직통으로 보내지 못하고 **무조건 고향(HA)을 한 번 거쳐서 와야 합니다.** (CN → HA → FA → MN). 경로가 삼각형 모양이 되어 **극심한 통신 지연(Delay)**과 고향 라우터(HA)의 병목 현상이 발생합니다.
- **해결책 (Route Optimization, RO)**: 처음 한 번만 삼각형으로 통신한 뒤, 노트북이 외부 서버(CN)에게 "나 지금 타지(COA)에 있으니 다음부터는 여기로 직통으로 보내!"라고 알려주어 최적의 경로로 직접 통신하게 만드는 **경로 최적화** 기법이 고안되었습니다.

> 📢 **섹션 요약 비유**: 서울 집 주소(Home Address)를 가진 내가 부산 호텔(FA)로 출장을 갔습니다. 서울 집으로 택배(패킷)가 오면, 서울 집에 있는 아빠(HA)가 택배를 가로채서 겉에 '부산 호텔 주소(COA)'를 덧붙여 다시 택배로 부칩니다(터널링). 이 방식은 편하지만, 부산에 사는 친구가 나한테 물건을 보낼 때도 굳이 서울 집을 거쳐서 부산으로 내려오는 바보 같은 비효율(세모 라우팅)이 발생합니다.
