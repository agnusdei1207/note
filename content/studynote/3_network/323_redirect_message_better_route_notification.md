## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ICMP Type 5 (Redirect) 메시지는, 길을 잘 모르는 멍청한 PC가 엉뚱한 라우터에게 패킷을 보냈을 때, **라우터가 "나한테 주지 말고 저기 옆에 있는 B 라우터한테 바로 주는 게 훨씬 빠르다!"라고 더 좋은 지름길(최적 경로)을 귀띔해 주는 경로 교정 통지서**다.
> 2. **네트워크 대역폭 절약**: 만약 이 메시지가 없다면 PC -> 라우터 A -> 라우터 B로 이어지는 비효율적인 '삼각 라우팅(Triangular Routing)'이 발생하여 같은 동네 안에서 패킷이 쓸데없이 두 번이나 움직이며 LAN 트래픽을 갉아먹는다.
> 3. **보안 취약점 (ICMP Redirect 공격)**: 해커가 가짜 Redirect 메시지를 날려서 "나(해커)한테 패킷을 보내는 게 가장 빠른 길이다!"라고 PC를 속여 모든 트래픽을 가로채는 스니핑 공격에 악용되므로, 현대 기업망에서는 **라우터의 이 오지랖(Redirect 발송) 기능을 강제로 꺼버린다.**

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: 패킷을 수신한 라우터가, 송신자(호스트)에게 "목적지로 가려면 나를 거치는 것보다 나와 같은 서브넷에 있는 다른 라우터를 통하는 것이 더 최적의 경로(Shorter Path)다"라고 알려주는 ICMP 에러 통보 (Type 5).
- **필요성**: PC는 멍청해서 복잡한 라우팅 테이블(OSPF, BGP 등)을 돌리지 못한다. PC는 오직 "내 동네 아니면 묻지도 따지지도 않고 무조건 `기본 게이트웨이(Default Gateway)`로 던진다"라는 1차원적인 논리만 가지고 있다. 만약 회사에 라우터가 2대(A: 일반용, B: 공장 전용) 꽂혀있는데, 내 PC의 게이트웨이가 A로 잡혀있다면, 공장으로 보낼 데이터도 일단 A로 던지는 뻘짓을 하게 된다. 이때 라우터 A가 친절하게 "공장 갈 거면 B한테 직접 줘!"라고 가르쳐주는 기능이 필요했다.

- **💡 비유**: 내가 부산에 가려고 **"동네 파출소(기본 게이트웨이 A)"**에 가서 길을 물었습니다. 파출소 순경 아저씨가 "부산 갈 거면 나한테 오지 말고, 저기 사거리 옆에 있는 **'고속도로 요금소(라우터 B)'**로 바로 직행하세요!"라고 **올바른 약도를 쥐여서 돌려보내는 것(Redirect)**과 같습니다.

> 📢 **섹션 요약 비유:** ICMP Redirect는 내비게이션 없이 무작정 메인 도로로만 직진하려는 초보 운전자(PC)에게, 동네 주민(라우터)이 **"아이고 답답아, 저쪽 샛길로 빠지면 훨씬 빨리 가!"**라고 창문을 내리고 훈수를 두는 훌륭한 길라잡이 서비스입니다.

## Ⅱ. ICMP Redirect의 동작 시나리오와 스니핑 공격 (Deep Dive)

### 1. 비효율적인 핀볼(Ping-Pong) 통신과 교정
같은 192.168.0.0/24 동네에 PC 1대와 2대의 라우터(R1, R2)가 물려있다. 
- R1은 인터넷으로 가는 기본 게이트웨이다.
- R2는 옆 건물 공장 전용망(10.x.x.x)으로 가는 직통 게이트웨이다.
- PC의 게이트웨이는 R1(`192.168.0.1`) 하나만 덜렁 세팅되어 있다.

1. PC가 공장 망(10.x.x.x)으로 핑을 쏜다. PC는 멍청하니까 무조건 R1으로 프레임을 쏜다.
2. R1이 받아보니, "어라? 10.x 망으로 가려면 어차피 내 옆에 있는 R2(192.168.0.2)로 줘야 하네?"라며 R2로 다시 튕겨준다.
3. 이 상황은 PC -> R1 -> R2로 패킷이 한 동네에서 V자 모양으로 꺾이는 완벽한 **대역폭 낭비(Triangular Routing)**다.
4. R1은 R2로 패킷을 넘겨준 직후, PC를 향해 **ICMP Type 5 (Redirect)** 패킷을 쏜다. "야! 공장(10.x)으로 갈 거면 다음부터는 나 거치지 말고 다이렉트로 R2(192.168.0.2)한테 쏴라!"
5. PC는 이 말을 듣고 자신의 메모리(내부 라우팅 테이블)에 `10.x 망 = R2로 보냄`이라는 임시 메모를 적어두고, 다음 패킷부터는 똑똑하게 R2로 직행한다.

```text
 ┌─────────────────────────────────────────────────────────────┐
 │                ICMP Redirect에 의한 경로 최적화 도식             │
 ├─────────────────────────────────────────────────────────────┤
 │                                                             │
 │   [ PC (192.168.0.100) ]                                    │
 │    게이트웨이: R1 (0.1)                                       │
 │       │                                                     │
 │       │ 1. 일단 멍청하게 R1로 던짐                              │
 │       ▼                                                     │
 │   [ 라우터 R1 (192.168.0.1) ] ── 2. R2로 전달 (V자 낭비) ──▶ [ 라우터 R2 ] │
 │       │                            (192.168.0.2)             │
 │       │ 3. ICMP Redirect 발송!                                │
 │       ▼ ("야! 다음부턴 R2로 다이렉트로 쏴라!")                    │
 │                                                             │
 │   [ PC ] "아항! 오케이 메모 완료!"                              │
 │       │                                                     │
 │       │ 4. 두 번째 패킷부터는 똑똑하게 R2로 다이렉트 전송!          │
 │       └───────────────────────────────────────────▶ │
 └─────────────────────────────────────────────────────────────┘
```

### 2. 해커의 악용: ICMP Redirect 공격 (MITM)
마치 ARP Spoofing처럼 너무 사람을 잘 믿어서 발생하는 치명적인 해킹 공격이다.
해커가 같은 네트워크에 숨어서 내 PC로 **가짜 ICMP Redirect(Type 5) 메시지**를 날린다. 
- 해커: "야 PC야, 구글(8.8.8.8)로 가려면 기본 게이트웨이 말고 내 노트북 IP(192.168.0.99)로 보내는 게 제일 빨라! 내 쪽으로 보내라!"
- 멍청한 PC: "어머나 친절하셔라. 알겠습니다!" (라우팅 테이블 오염)
- 결국 내 PC가 구글로 보내는 모든 비밀번호 패킷이 해커 노트북으로 줄줄이 납치당한다.

### 3. 실무적 방어 대책
오늘날 L3 스위치나 기업용 라우터를 구매하면 이 친절한 오지랖 기능이 **기본적으로 강제 종료(Disable)**되어 있다.
- 시스코 명령어: `no ip redirects` (인터페이스 모드)
- 비효율적인 삼각 라우팅이 돌더라도, 해커에게 망 전체를 스니핑(도청) 당하는 것보다 낫다는 철학이다. 어차피 요새는 스위치 속도가 수십 기가비트(Gbps)라 1번 튕겨가는 것쯤은 티도 나지 않는다.

> 📢 **섹션 요약 비유:** ICMP Redirect는 묻지도 않았는데 길을 가르쳐주는 **"오지랖 넓은 동네 아저씨"**입니다. 진짜 경찰(라우터)이 가르쳐주면 고맙지만, 사기꾼(해커)이 샛길로 유인해 강도짓(스니핑)을 할 수 있으므로, 요즘 PC들은 아무리 샛길을 알려줘도 **"그냥 내비게이션(기본 게이트웨이)이 시키는 대로만 우직하게 갈래!"**라며 귀를 막아버립니다.
