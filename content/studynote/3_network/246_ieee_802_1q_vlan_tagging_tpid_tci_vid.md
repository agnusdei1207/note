## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IEEE 802.1Q는 서로 다른 스위치 간에 여러 개의 가상 랜(VLAN) 데이터를 하나의 물리적 케이블(트렁크 포트)로 섞어 보낼 때, 각 프레임이 어느 VLAN 소속인지 구별하기 위해 **이더넷 헤더 중간에 4바이트짜리 꼬리표(Tag)를 끼워 넣는 국제 표준 규격**이다.
> 2. **프레임 구조 변경**: 원래의 출발지 MAC 주소와 Type 필드 사이에 4바이트가 억지로 삽입되기 때문에 전체 이더넷 프레임의 최대 크기가 1518바이트에서 1522바이트(Baby Giant Frame)로 늘어나며, 스위치는 수신 시 이 태그를 읽고 다시 떼어낸 뒤 전달한다.
> 3. **주요 필드 (VID)**: 4바이트 태그 중 가장 중요한 것은 **12비트 길이의 VID(VLAN ID)** 필드이며, 이 12비트 덕분에 하나의 네트워크에서 이론상 최대 4,096개($2^{12}$)의 논리적 VLAN 그룹을 생성할 수 있다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: 802.1Q는 이더넷 프레임에 소속 VLAN 정보를 명시하는 '태깅(Tagging)' 기술의 IEEE 공식 표준이다.
- **필요성**: 만약 A 스위치에 영업부(VLAN 10)와 인사부(VLAN 20)가 있고, B 스위치에도 영업부와 인사부가 있다고 치자. 두 스위치를 1가닥의 선(트렁크)으로 연결하면, B 스위치는 건너온 데이터가 영업부 것인지 인사부 것인지 알 도리가 없다. 따라서 건너가는 택배 상자 겉면에 "이건 10번 방(VLAN 10) 물건이야!"라는 형광색 스티커를 붙여주어야 하는데, 이것이 바로 802.1Q 태그다.

- **💡 비유**: 공항 수하물 컨베이어 벨트(트렁크 선로)에 서울행, 부산행, 제주행 짐들이 섞여서 돌아갑니다. 수하물을 구별하기 위해 각 가방 손잡이에 **"바코드 꼬리표(802.1Q Tag)"**를 매달아 놓는 것과 같습니다. 최종 목적지 공항에 도착하면 직원은 이 꼬리표를 떼어내고(Untag) 승객에게 짐을 돌려줍니다.

> 📢 **섹션 요약 비유:** 802.1Q 태깅은 수많은 사람들이 섞여서 지나가는 하나의 긴 통로(트렁크)에서, 서로 소속을 헷갈리지 않게 **"빨간색(VLAN 10), 파란색(VLAN 20) 이름표(Tag)를 목에 걸어주는 다중화(Multiplexing) 시스템"**입니다.

## Ⅱ. 802.1Q 태그의 구조와 핵심 필드 (Deep Dive)

### 1. 4바이트 태그의 삽입 위치
이더넷 프레임의 앞부분은 `[목적지 MAC] - [출발지 MAC] - [Type] - [Data]` 순으로 되어 있다. 802.1Q 스위치는 프레임을 밖으로 쏠 때, **[출발지 MAC]과 [Type] 사이를 강제로 벌리고 4바이트(32비트)의 태그를 삽입**한다.
- 태그가 삽입되면서 원래 있던 FCS(에러 검출 코드) 값은 틀려지므로, 스위치가 FCS 값을 새롭게 다시 계산하여 맨 꼬리에 덮어쓴다.

```text
 ┌─────────────────────────────────────────────────────────────┐
 │                IEEE 802.1Q 프레임 태깅 구조 도식               │
 ├─────────────────────────────────────────────────────────────┤
 │                                                             │
 │   [ 일반 이더넷 프레임 (Untagged) ]                            │
 │   ┌────────┬────────┬───────┬──────────────────────┬─────┐  │
 │   │목적지MAC│출발지MAC │ Type  │   Payload (IP 패킷)   │ FCS │  │
 │   └────────┴────────┴───────┴──────────────────────┴─────┘  │
 │                       ▲ 여기를 벌리고 끼워 넣음               │
 │                                                             │
 │   [ 802.1Q 태그 이더넷 프레임 (Tagged) ]                       │
 │   ┌────────┬────────┬───────┬───────┬──────────────┬─────┐  │
 │   │목적지MAC│출발지MAC │ 802.1Q│ Type  │ Payload (IP) │ New │  │
 │   │        │        │ TAG(4B│       │              │ FCS │  │
 │   └────────┴────────┴───────┴───────┴──────────────┴─────┘  │
 │                      │       │                              │
 │                      └── ┬ ──┘                              │
 │                           ▼ 4바이트(32비트)의 내부 구조         │
 │                      [ TPID(16) | TCI(16) ]                 │
 │                                                             │
 └─────────────────────────────────────────────────────────────┘
```

### 2. 4바이트(32비트) 태그의 내부 구조
태그는 크게 2바이트의 **TPID**와 2바이트의 **TCI**로 나뉜다.

1. **TPID (Tag Protocol Identifier, 16비트)**:
   - 항상 `0x8100`이라는 고정된 값을 가진다.
   - 수신 스위치가 이 값을 보면 "아, 이 프레임은 일반 프레임이 아니라 802.1Q 태그가 붙은 프레임이구나!"라고 단번에 인식(식별)하게 해주는 마법의 번호다.
2. **TCI (Tag Control Information, 16비트)**:
   - **PRI (Priority, 3비트)**: `IEEE 802.1p`라고 불리는 음성/영상 트래픽 우선순위(QoS)를 부여하는 비트. 0~7까지의 값을 갖는다.
   - **CFI (Canonical Format Indicator, 1비트)**: 토큰링 등 과거 망과의 호환성을 위한 비트 (현재는 거의 `0`으로 고정).
   - **VID (VLAN Identifier, 12비트) ★핵심**: 자신이 속한 VLAN 번호를 나타낸다. 12비트이므로 $2^{12} = 4096$개의 숫자를 표현할 수 있다. (0과 4095는 예약되어 실제 사용 가능 범위는 1~4094번이다.)

> 📢 **섹션 요약 비유:** 4바이트 꼬리표 중 절반(TPID `0x8100`)은 **"이거 꼬리표야! 주의해서 봐!"**라고 외치는 식별자이고, 나머지 절반 중 12비트(VID)가 진짜 **"나 10번 부서(VLAN 10) 소속이야!"**라고 알려주는 핵심 정보칸입니다.
