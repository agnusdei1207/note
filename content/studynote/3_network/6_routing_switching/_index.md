## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 라우팅 (Routing)은 서로 다른 네트워크 간의 패킷 전달 경로를 결정하는 프로세스이며, 스위칭 (Switching)은 동일 네트워크 내에서 목적지 주소에 따라 데이터를 물리적으로 전달하는 메커니즘이다.
> 2. **가치**: 동적 라우팅 프로토콜 (OSPF, BGP)을 통해 네트워크 변화에 실시간으로 대응하는 생존성을 확보하고, 고속 스위칭 기술 (L2/L3/L4/L7)을 통해 대역폭 활용을 최적화하고 병목을 제거한다.
> 3. **융합**: 가상 확장 LAN (VXLAN) 및 세그먼트 라우팅 (SR) 기술이 클라우드 패브릭과 결합되어, 거대한 데이터 센터 내부의 트래픽을 논리적으로 분리하고 초고속으로 전송하는 현대적 망 아키텍처를 완성한다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 데이터의 길 찾기와 전달: 네트워크의 심장

네트워크의 성능과 신뢰성은 결국 **'얼마나 빠르고 정확하게 데이터를 전달하느냐'**에 달려있다. **라우팅**은 복잡한 거미줄 같은 망에서 가장 효율적인 길을 찾는 지능적인 과정이고, **스위칭**은 찾아낸 길을 따라 패킷을 실제로 밀어넣는 고속 엔진의 역할을 한다.

라우팅 및 스위칭 기술이 중요한 이유는 세 가지이다. 첫째, **네트워크의 생존성 (Resilience)**을 위해서이다. 특정 회선이 끊겨도 라우팅 프로토콜이 즉시 우회로를 찾아야 한다. 둘째, **자원 활용의 극대화**를 위해서이며 (예: 부하 분산), 셋째, **논리적 격리 및 보안**을 통해 거대한 물리적 망을 안전한 소규모 망들로 나누어 관리하기 위함이다.

이 그림은 라우팅과 스위칭이 각각 네트워크의 어느 범위에서 동작하는지 비교하여 보여준다.

```text
┌─────────────────────────────────────────────────────────────┐
│                 Routing vs Switching Scope                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [ Global Internet / WAN ]                                 │
│          │                                                  │
│   ┌──────▼──────┐ (L3: Routing)                             │
│   │   Router    │ ──▶ IP 기반, 네트워크 간 경로 결정        │
│   └──────┬──────┘                                           │
│          │                                                  │
│   [ Local Area Network / Data Center ]                      │
│          │                                                  │
│   ┌──────▼──────┐ (L2: Switching)                           │
│   │   Switch    │ ──▶ MAC 기반, 노드 간 데이터 전달         │
│   └──────┬──────┘                                           │
│          │                                                  │
│   [ Host A ] [ Host B ]                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

이 다이어그램의 핵심은 '계층적 연결'이다. 스위치가 건물 내부의 소통을 담당한다면, 라우터는 건물과 건물을 잇는 고속도로와 같다. 실무에서는 이 두 기능이 통합된 **L3 스위치**가 백본 망의 핵심 장비로 활용되어, 내부 스위칭의 속도와 외부 라우팅의 지능을 동시에 제공한다.

### 라우팅 및 스위칭의 주요 지표

1. **Convergence Time (수렴 시간)**: 네트워크 변경 시 모든 라우터가 새 지도를 완성하는 데 걸리는 시간.
2. **Throughput (처리량)**: 단위 시간당 전송 가능한 패킷의 양.
3. **Latency (지연 시간)**: 장비를 통과하는 데 걸리는 물리적/논리적 시간.
4. **AS (Autonomous System)**: 하나의 관리 주체가 운영하는 독립적 라우팅 영역.

📢 **섹션 요약 비유**: 라우팅은 '내비게이션이 길을 찾는 것'과 같고, 스위칭은 '교차로에서 신호등에 따라 차를 보내는 것'과 같습니다. 내비게이션이 아무리 좋아도 교차로가 막히면 소용없고, 교차로가 뻥 뚫려도 엉뚱한 길로 가면 목적지에 갈 수 없습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 동적 라우팅 프로토콜 아키텍처

1. **IGP (Internal Gateway Protocol)**: 기업 내 망 관리.
   - **OSPF**: 링크 상태 알고리즘. 대역폭 기반 최단 경로 산출. 계층적 Area 구조.
   - **EIGRP**: 거리 벡터와 링크 상태의 장점을 결합한 시스코 전용(이었던) 프로토콜.
2. **EGP (External Gateway Protocol)**: 국가 간, ISP 간 망 관리.
   - **BGP**: 경로 벡터 알고리즘. 정책 기반 라우팅. 전 세계 인터넷 백본의 표준.

### 고급 스위칭 및 가상화 기술

물리적 한계를 넘기 위한 현대적 스위칭 기술들이다.

| 기술 명칭 | 핵심 메커니즘 | 효과 |
|:---:|:---|:---|
| **VLAN (Virtual LAN)** | 포트별 논리적 그룹화 | 브로드캐스트 도메인 분리, 보안 강화 |
| **Trunking (802.1Q)** | 하나의 선으로 여러 VLAN 전송 | 스위치 간 효율적 연결 |
| **STP / RSTP** | 루프 경로 논리적 차단 | 네트워크 마비 방지, 이중화 보장 |
| **LACP (EtherChannel)** | 여러 선을 하나로 묶음 | 대역폭 확장 및 장애 복구 (HA) |

이 구조도는 BGP가 인터넷의 거대 노드들(AS) 사이에서 어떻게 경로를 결정하는지 보여준다.

```text
┌─────────────────────────────────────────────────────────────┐
│                 BGP: Path Vector and AS-Path                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [ AS 100 ] ────▶ [ AS 200 ] ────▶ [ AS 300 ]              │
│       │                               ▲                     │
│       └──────────▶ [ AS 400 ] ────────┘                     │
│                                                             │
│   * Path 1: (100, 200, 300) -> 3 Hops                       │
│   * Path 2: (100, 400, 300) -> 3 Hops                       │
│                                                             │
│   * BGP 결정 기준: 1. Local Pref (정책)                     │
│                    2. AS-Path (짧은 경로)                   │
│                    3. MED (진입로 선호도)                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

이 다이어그램의 핵심은 '정책 기반 라우팅'이다. BGP는 단순히 빠른 길만 찾는 게 아니라, "SKT 망은 비싸니까 KT 망을 우선으로 써라" 같은 비즈니스 로직을 경로에 반영할 수 있다. 실무에서는 이 BGP 설정 오류 하나로 한 국가의 인터넷이 마비되는 대형 사고가 발생하기도 한다.

📢 **섹션 요약 비유**: OSPF가 '우리 동네 지도를 빠삭하게 아는 택시 기사'라면, BGP는 '전 세계 국가를 연결하는 외교관'과 같습니다. 기사는 빠른 길을 찾고, 외교관은 국가 간의 조약과 이익에 따라 이동 경로를 결정합니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### L2 vs L3 vs L4 vs L7 스위치 비교

데이터의 어느 깊이까지 뜯어보고 전달하느냐에 따른 분류이다.

| 구분 | 참조 정보 | 주요 기능 | 비유 |
|:---:|:---|:---|:---|
| **L2** | MAC 주소 | 단순 전달, VLAN | 건물 내 우편 배달부 |
| **L3** | IP 주소 | 라우팅, ACL | 지역 우체국 |
| **L4** | Port, TCP/UDP | 부하 분산 (LB) | 아파트 관리실 (택배 분류) |
| **L5~7** | URL, 쿠키, 컨텐츠 | 지능형 LB, 보안 (WAF) | 호텔 컨시어지 (맞춤 서비스) |

### 거리 벡터 (RIP) vs 링크 상태 (OSPF) 비교

| 항목 | RIP (Distance Vector) | OSPF (Link State) |
|:---:|:---|:---|
| **최적 경로 기준** | 홉 (Hop) 수 (최대 15) | **비용 (Cost - 대역폭 기반)** |
| **업데이트 방식** | 전체 테이블 주기적 전송 | **상태 변화 시에만 부분 전송** |
| **수렴 속도** | 느림 | **매우 빠름** |
| **자원 소모** | 낮음 | 높음 (CPU/Mem 부하 존재) |

📢 **섹션 요약 비유**: RIP가 "옆집 아저씨가 그러는데 저기가 가깝대요"라고 소문으로 길을 찾는 것이라면, OSPF는 "내가 헬기를 타고 전체 지도를 그려보니 이 길이 제일 넓어요"라고 과학적으로 길을 찾는 차이입니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단: 백본 망 설계 및 장애 대응 시나리오

**시나리오 1: 데이터 센터 내부의 초고속 가상화 네트워크 구축**
- **판단**: 기존 L2 스위칭은 4,096개의 VLAN 한계에 부딪힌다. 이를 해결하기 위해 L3 기반 위에서 L2 프레임을 터널링하는 **VXLAN (Virtual Extensible LAN)** 기술을 도입한다. 또한 컨트롤러가 모든 스위치의 흐름을 제어하는 **SDN (Software Defined Networking)** 아키텍처를 통해 서버의 위치 이동 (VM Migration) 시에도 네트워크 설정이 자동으로 따라가게 설계한다.

**시나리오 2: 특정 인터넷 회선 정체로 인한 서비스 품질 저하**
- **판단**: BGP의 **Local Preference** 값을 조정하여 트래픽 유출 경로를 수동으로 제어한다. 만약 외부에서 들어오는 트래픽이 문제라면 **AS-Path Prepending** (자신의 AS 번호를 여러 번 붙여 멀어 보이게 함) 기법을 사용하여 상대방 라우터가 다른 경로를 선택하도록 유도한다. 이 과정에서 유입 트래픽과 유출 트래픽의 대칭성 (Symmetry)을 반드시 확인하여 방화벽 차단을 방지한다.

이 도식은 L3 스위치에서 일어나는 **Inter-VLAN 라우팅**의 논리적 구조를 보여준다.

```text
┌─────────────────────────────────────────────────────────────┐
│                 L3 Switching: Router-on-a-Stick vs SVI      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [ VLAN 10 ] ──┐          ┌── [ L3 Engine ] ──┐            │
│                 │          │   (Routing)       │            │
│   [ VLAN 20 ] ──┼──▶ [ L2 Fabric ] ◀───────────┘            │
│                 │                                           │
│   [ VLAN 30 ] ──┘                                           │
│                                                             │
│   * SVI: 가상 인터페이스를 통해 하드웨어 속도로 라우팅      │
│   * 효과: 서브넷 간 통신 시 병목 현상 완벽 제거             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 기술사의 처방은 '교통 설계'와 같습니다. 단순히 도로를 넓히는 게 아니라, 가변 차선(SDN)을 만들고 하이패스(L3 스위칭)를 설치하여 정체 없는 도시를 만드는 설계 능력이 핵심입니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 지능형 전달 체계의 가치

1. **정량적 효과**: 네트워크 장애 수렴 시간 1초 미만 달성, 전체 대역폭 효율 40% 향상.
2. **정성적 효과**: 인프라 운영 자동화 (IaC) 기반 마련, 클라우드 확장을 위한 논리적 유연성 확보.

### 미래 전망: 6G 시대의 테라비트 스위칭과 SRv6

향후 라우팅 기술은 IP 주소를 넘어서는 **SRv6 (Segment Routing over IPv6)**로 수렴할 것이다. 이는 복잡한 레이블 관리 없이 IPv6 헤더만으로 세밀한 경로 제어를 가능하게 한다. 또한 데이터 센터 내부 스위칭은 **광학 스위칭 (Optical Switching)** 기술과 결합하여 전기 신호의 발열 한계를 돌파할 것이다. 기술사는 정적인 라우팅 테이블 관리를 넘어, 의도 기반 네트워크 (IBN)를 통해 비즈니스 목표를 네트워크 정책으로 자동 변환하는 지능형 아키텍처에 대한 전문성을 갖추어야 한다.

📢 **섹션 요약 비유**: 미래의 네트워크는 '스타워즈의 웜홀'과 같아질 것입니다. 목적지만 정하면 중간의 복잡한 길을 거치지 않고도 가장 빠른 차원으로 즉시 이동하는 마법 같은 전송 기술이 완성될 것입니다.

### 📌 관련 개념 맵 (Knowledge Graph)
- **OSPF**: 대규모 내부망을 위한 최단 경로 프로토콜
- **BGP**: 인터넷의 등추를 지탱하는 정책 라우팅
- **VXLAN**: 클라우드 가상화를 위한 L2-over-L3 기술
- **STP**: 네트워크 마비를 막는 수문장
- **L3 Switch**: 하드웨어 가속 라우팅의 핵심 장비
- **Convergence**: 평화로운 네트워크 지도를 완성하는 과정

### 👶 어린이를 위한 3줄 비유 설명
- 라우팅과 스위칭은 전 세계 컴퓨터들이 쓰는 '똑똑한 지도와 표지판'이에요.
- 지도가 있으면 처음 가는 길도 척척 찾아갈 수 있고(라우팅), 표지판이 있으면 복잡한 사거리에서도 사고 없이 쌩쌩 달릴 수 있죠(스위칭).
- 이 친구들 덕분에 우리가 보낸 편지가 지구 반대편 친구에게 눈 깜빡할 사이에 도착하는 거랍니다!
