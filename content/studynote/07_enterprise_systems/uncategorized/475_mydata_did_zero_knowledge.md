## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 마이데이터(MyData)는 개인이 자신의 데이터를 직접 통제하고 원하는 서비스에 제공하는 개인정보 자기결정권 실현 모델이며, DID(Decentralized Identifier, 탈중앙 식별자)는 중앙 기관 없이 블록체인에서 신원을 검증하는 기술 기반이다.
> 2. **가치**: 영지식 증명(ZKP, Zero-Knowledge Proof)과 DID를 결합하면 "나는 만 18세 이상이다"를 생년월일을 공개하지 않고 수학적으로 증명하는 **선택적 공개(Selective Disclosure)** 가 가능하여 개인정보 최소화 원칙을 기술로 구현한다.
> 3. **판단 포인트**: 마이데이터의 성패는 기술이 아닌 생태계 형성에 달려 있다. 데이터를 요청하는 서비스(Verifier)와 데이터를 제공하는 발급자(Issuer)가 동일한 DID 표준(W3C DID, Verifiable Credentials)을 채택해야 실용적 생태계가 형성된다.

## Ⅰ. 개요 및 필요성

현재 인터넷에서 개인 데이터의 흐름은 역설적이다. 내 의료 기록, 금융 거래, 위치 데이터가 네이버·구글·병원·은행에 분산 저장되어 있지만, 정작 나는 이 데이터를 직접 활용하거나 다른 서비스에 제공하기 어렵다.

**마이데이터(MyData)**는 데이터 권력을 플랫폼에서 개인에게로 돌리는 패러다임 전환이다. 금융 마이데이터(2021년 시행)는 은행·카드·보험 데이터를 개인 동의 하에 핀테크 앱으로 전송하는 API 체계를 법으로 구현했다. 의료 마이데이터(PHR)와 공공 마이데이터로 확장 중이다.

**DID(Decentralized Identifier)**는 이 마이데이터 생태계에서 신원 증명의 기술 기반이다. 공무원의 재직증명서가 행정안전부 서버에 의존하는 것과 달리, DID 기반 신원은 블록체인에 기록되어 어떤 중앙 기관도 취소·조작할 수 없다.

📢 **섹션 요약 비유**: 마이데이터는 내 의료 기록을 병원이 아닌 내가 직접 들고 다니는 것처럼, 데이터의 주인이 기관에서 나로 돌아오는 데이터 소유권 혁명이다.

## Ⅱ. 아키텍처 및 핵심 원리

### DID/VC 신뢰 삼각형

```
┌──────────────────────────────────────────────────────────────┐
│          DID/Verifiable Credentials 신뢰 삼각형               │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│         [ Issuer (발급자) ]                                  │
│         예: 행정안전부, 은행, 병원                            │
│              │  1. VC 발급 (서명된 디지털 자격증명)          │
│              ▼                                               │
│         [ Holder (보유자) ] ← 나의 DID 지갑 (스마트폰)       │
│              │  2. VC 제출 (선택적 공개 가능)                 │
│              ▼                                               │
│         [ Verifier (검증자) ]                                │
│         예: 렌터카, 신용대출, 채용 담당자                    │
│              │  3. 발급자 공개키로 서명 검증                  │
│              ▼                                               │
│    [ DID Registry (블록체인) ] ← 공개키·DID Document 저장    │
│                                                              │
│  핵심: Verifier가 Issuer에게 직접 확인하지 않아도 됨         │
│        (발급자 서버 장애 시에도 검증 가능)                   │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 영지식 증명(ZKP)의 마이데이터 적용

| 전통 신원 증명 | ZKP 기반 증명 |
|:---|:---|
| "주민등록증을 보여주세요" → 이름·주소·번호 전부 노출 | "성인임을 증명해주세요" → 만 18세 이상이라는 사실만 수학적 증명 |
| "재직증명서 제출" → 회사명·직위·급여 노출 가능 | "재직 중임을 증명" → 재직 사실만 ZKP로 증명 |
| 과다 공개로 개인정보 침해 | 최소 정보만 공개 (Privacy by Design) |

ZKP 알고리즘: **zk-SNARK** (이더리움 Zcash, 가스 효율), **zk-STARK** (양자 내성, 투명성), **BBS+ Signatures** (W3C VC 선택적 공개 표준)

📢 **섹션 요약 비유**: 영지식 증명은 "내가 보물 지도를 알고 있다"는 것을 지도를 보여주지 않고 증명하는 마술처럼, 비밀(실제 데이터)은 숨기면서 그 비밀에서 나온 사실(나이가 18 이상)만 증명하는 암호 수학이다.

## Ⅲ. 비교 및 연결

### SSI(Self-Sovereign Identity) 성숙도 모델

| 레벨 | 특징 | 현재 상태 |
|:---:|:---|:---|
| 1 | 중앙화 신원 (기관 ID) | 현재 표준 |
| 2 | 연합 신원 (소셜 로그인) | OAuth/OIDC |
| 3 | 사용자 중심 (서드파티 관리) | OpenID |
| 4 | **자기 주권 신원 (SSI/DID)** | 도입 초기 |

### 국내 마이데이터 현황

- **금융 마이데이터**: 2021년 시행, 은행·증권·카드·보험 통합 조회 (토스·카카오페이 활용)
- **의료 마이데이터(PHR)**: 시범 사업 단계 (건강보험공단·병원 진료 기록)
- **모바일 신분증**: 국가 DID 기반 운전면허증·학생증 모바일 발급 (2023년~)
- **분산 ID 연합**: 정부주도 DID 협의체 (통신사·은행·PASS 앱)

📢 **섹션 요약 비유**: 마이데이터 성숙도는 지갑 진화처럼, 현금(중앙화 ID)→신용카드(연합 신원)→모바일 페이(사용자 중심)→암호화폐 지갑(DID/SSI)으로 지불 권력이 은행에서 개인으로 이동하는 여정이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 마이데이터 플랫폼 설계 시 판단 포인트

1. **데이터 표준**: MyData API는 금융위원회 금융 마이데이터 표준 API 또는 HL7 FHIR(의료) 준수 필수
2. **DID 방법론 선택**: `did:web`(단순, HTTP 기반), `did:ethr`(이더리움 기반), `did:gov`(정부 DID 레지스트리)
3. **개인정보보호**: 동의 관리 플랫폼(CMP) 구축 → 목적별 동의·철회·이력 관리
4. **VC 라이프사이클**: 발급→보관(DID 지갑)→제출→검증→폐기 전 과정 자동화
5. **ZKP 성능 고려**: zk-SNARK 증명 생성은 수초 소요 → 모바일 환경 UX 최적화 필요

📢 **섹션 요약 비유**: 마이데이터 플랫폼 설계는 이민국 통관 시스템처럼, 여러 나라(서비스)가 서로 다른 규칙(표준)을 쓰면 사람(데이터)이 자유롭게 이동하지 못하므로, 공통 여권 표준(W3C DID/VC)이 필수이다.

## Ⅴ. 기대효과 및 결론

**기대효과**: 개인 데이터 자기결정권 실현, 중복 데이터 제출 제거(서류 간소화), 신원 사기 방지(위변조 불가 블록체인 VC), 데이터 경제 활성화.

**한계**: DID 지갑 분실 시 신원 복구 절차 복잡, 표준 분열(W3C DID vs 정부 표준), ZKP 연산 부하, 블록체인 거버넌스 문제.

📢 **섹션 요약 비유**: DID 분실은 지갑에 모든 신분증을 넣고 다니다 잃어버리는 것처럼, 회복 방법(Recovery Key, Social Recovery)을 미리 설계하지 않으면 디지털 신원을 완전히 잃을 수 있는 치명적 약점이 있다.

### 📌 관련 개념 맵

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| DID | 기술 기반 | 탈중앙 식별자, 블록체인 기반 신원 |
| VC (Verifiable Credentials) | 신원 증명 형식 | W3C 표준 디지털 자격증명 |
| ZKP | 프라이버시 기술 | 비밀 없이 사실만 수학적 증명 |

| 금융 마이데이터 | 국내 구현 | 2021년 시행 금융 데이터 통합 API |
| FHIR | 의료 표준 | 의료 마이데이터의 데이터 교환 표준 |

### 👶 어린이를 위한 3줄 비유 설명

1. 마이데이터는 내 의사선생님 기록을 내가 직접 들고 다니는 '개인 의료 다이어리'처럼, 내 정보의 주인이 병원이 아닌 나 자신이 되는 거예요.
2. DID는 '내가 직접 만든 디지털 신분증'처럼, 정부나 회사가 발급하는 게 아니라 내가 블록체인에 직접 등록하는 신원이에요.
3. 영지식 증명은 "나는 키가 150cm 이상"임을 실제 키 재지 않고 수학으로 증명하는 것처럼, 비밀은 숨기면서 필요한 사실만 알려줘요.
