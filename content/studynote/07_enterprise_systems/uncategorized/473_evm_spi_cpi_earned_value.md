## 핵심 인사이트 (3줄 요약)

> 1. **본질**: EVM(Earned Value Management)은 프로젝트의 일정(Schedule)과 비용(Cost) 성과를 계획 가치(PV), 획득 가치(EV), 실제 비용(AC)이라는 3개 지표로 통합하여 현재 상태와 완료 시점을 예측하는 객관적 프로젝트 관리 기법이다.
> 2. **가치**: SPI(Schedule Performance Index)와 CPI(Cost Performance Index)가 1.0 미만이면 각각 일정 지연·비용 초과를 의미하여, PM이 조기에 문제를 감지하고 시정 조치를 취할 수 있어 '기 투자 매몰 비용 함정'에서 벗어나게 한다.
> 3. **판단 포인트**: EVM의 EAC(Estimate at Completion, 최종 예상 비용) 계산은 CPI가 향후에도 유지된다는 가정 기반이므로, 초기 단계(< 20% 진행)에서는 신뢰도가 낮아 BAC 기반 재추정과 병행해야 한다.

## Ⅰ. 개요 및 필요성

"프로젝트가 50% 예산을 썼는데 얼마나 진행됐나요?" — 이 질문의 답이 진척도 30%라면 프로젝트는 위기 상황이다. 그러나 전통적인 일정-비용 관리는 "비용을 얼마 썼는가"와 "일정이 어느 단계인가"를 별개로 보아 이러한 통합적 분석이 불가능했다.

EVM은 1960년대 미국 국방부(DOD)가 방산 프로젝트의 비용 초과를 제어하기 위해 도입한 기법으로, 현재 PMI(PMBOK)·ISO 21500·국내 공공 SW 사업의 표준 성과 관리 도구이다.

세 가지 기본값: **PV(Planned Value, 계획 가치)** = 지금까지 계획했던 작업의 예산, **EV(Earned Value, 획득 가치)** = 실제로 완료된 작업의 예산, **AC(Actual Cost, 실제 비용)** = 지금까지 실제 지출한 비용.

📢 **섹션 요약 비유**: EVM은 자동차 계기판처럼, 속도(일정)와 연료(비용)를 동시에 보여줘 "지금 제한 속도보다 빠른데 기름은 덜 쓰고 있다" 또는 "느린데 기름은 과하게 쓰고 있다"는 통합 현황을 즉시 알 수 있게 한다.

## Ⅱ. 아키텍처 및 핵심 원리

### EVM 핵심 공식

| 지표 | 공식 | 의미 | 기준값 |
|:---|:---|:---|:---:|
| **SV** (일정 편차) | SV = EV - PV | (+)선행, (-)지연 | 0 |
| **CV** (비용 편차) | CV = EV - AC | (+)절감, (-)초과 | 0 |
| **SPI** (일정 성과 지수) | SPI = EV / PV | >1 선행, <1 지연 | 1.0 |
| **CPI** (비용 성과 지수) | CPI = EV / AC | >1 절감, <1 초과 | 1.0 |
| **EAC** (최종 예상 비용) | EAC = BAC / CPI | 현재 효율로의 완료 예측 | = BAC |
| **ETC** (잔여 예상 비용) | ETC = EAC - AC | 앞으로 필요한 예산 | — |
| **VAC** (완료 시 편차) | VAC = BAC - EAC | 최종 예산 초과/절감 예측 | 0 |
| **TCPI** (완료 성과 지수) | TCPI = (BAC-EV)/(BAC-AC) | 남은 작업의 필요 효율 | = 1.0 |

### EVM 상태 분류 (S-곡선 분석)

```
┌──────────────────────────────────────────────────────────────┐
│             EVM 상태 분석 매트릭스                            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│         │     SPI > 1        │     SPI < 1       │          │
│  ───────┼────────────────────┼───────────────────┤          │
│  CPI>1  │ ✅ 최상:           │ ⚠️  일정 지연,    │          │
│         │  일정 선행+비용 절감│    비용은 절감     │          │
│  ───────┼────────────────────┼───────────────────┤          │
│  CPI<1  │ ⚠️  일정 선행,    │ ❌ 최악:           │          │
│         │    비용 초과       │  일정 지연+비용 초과│          │
│  ───────┴────────────────────┴───────────────────┘          │
│                                                              │
│  CPI < 0.8 → 프로젝트 종료 검토 (Cancellation Point)        │
│  CPI = 0.85 → 국방부 경험칙: 최종 CPI는 0.85 유지 경향       │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: CPI=0.8은 1만원 일을 1만2천원에 하고 있다는 것으로, 이 효율이 계속되면 1억 프로젝트가 1억2천만원이 될 것을 조기에 경고하는 재정 건강 지표이다.

## Ⅲ. 비교 및 연결

### EVM 계산 예시

- BAC(예산) = 1억 원 / 현재 진행률 계획: 50% / 실제 완료: 40% / 실제 지출: 5,500만 원
- PV = 5,000만 원 (50% × 1억)
- EV = 4,000만 원 (40% × 1억)
- AC = 5,500만 원
- SV = 4,000 - 5,000 = -1,000만 원 (일정 지연)
- CV = 4,000 - 5,500 = -1,500만 원 (비용 초과)
- SPI = 4,000/5,000 = **0.80** (20% 일정 지연)
- CPI = 4,000/5,500 = **0.73** (27% 비용 초과)
- EAC = 1억/0.73 = **1억 3,700만 원** (최종 예상 비용)

### CPM(472)과 EVM의 통합

CPM이 "언제 완료되는가"를 분석한다면, EVM은 "얼마의 비용으로 완료되는가"를 예측한다. 두 기법을 통합하면 임계 경로 상 활동의 비용 성과를 중점 관리하는 일정-비용 통합 관리가 가능하다.

📢 **섹션 요약 비유**: EVM + CPM의 통합은 마라톤 선수의 페이스 관리처럼, 현재 속도(SPI)와 에너지 소비(CPI)를 동시에 보며 완주 시간(EAC)을 실시간으로 예측하는 레이스 전략이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 공공 SW 사업 EVM 적용 시 판단 기준

1. **경보 기준 설정**: SPI < 0.9 또는 CPI < 0.9 → 월간 보고에서 원인 분석 및 시정 계획 수립 의무화
2. **TCPI 활용**: TCPI > 1.1이면 잔여 작업에서 현실적으로 불가능한 효율 요구 → EAC 재산정 및 범위 조정 검토
3. **롤업 분석**: 전체 WBS의 EVM을 임계 경로 활동 레벨까지 분해하여 문제 활동 조기 탐지
4. **발주처 보고**: 월별 EVM 대시보드(S-curve + 주요 지표)를 시각화하여 의사결정권자에게 보고

📢 **섹션 요약 비유**: EVM 경보 시스템은 항공기 연료 경고등처럼, 연료(예산)가 임계치 이하로 떨어지기 전에 조종사(PM)에게 충분한 시간을 주어 착륙지를 변경(범위 조정)하거나 연료를 추가(예산 증액)할 수 있게 한다.

## Ⅴ. 기대효과 및 결론

**기대효과**: 프로젝트 조기 경보로 실패 프로젝트 조기 구제 또는 종료 결정, 예산 초과 예방, 발주처 신뢰 향상.

**한계**: EVM은 완료율(%) 정의가 주관적이어서 진척도 측정 정확도가 핵심 전제, 소규모 프로젝트에서는 관리 오버헤드 과다.

📢 **섹션 요약 비유**: 완료율 측정의 한계는 마치 "90% 완성된 소프트웨어"가 실제로 10%의 기능만 정말 작동하는 경우처럼, EVM 숫자의 신뢰성은 완료율 측정 방법론의 정직함에 달려 있다.
