## 핵심 인사이트 (3줄 요약)

> 1. **본질**: BPR (Business Process Reengineering)은 프로세스를 제로 베이스에서 근본적으로 재설계하는 급진적 혁신이고, PI (Process Innovation)는 기존 프로세스의 지속적·점진적 개선을 추구하는 방법론이다.
> 2. **가치**: BPR은 단기간에 70~80% 효율 향상을 목표로 하지만 70% 실패율이라는 높은 리스크를 수반하며, PI는 안전하게 10~30% 개선을 달성하면서 조직 학습 문화를 구축한다.
> 3. **판단 포인트**: 프로세스의 변화 필요 규모와 조직의 변화 수용 능력을 동시에 평가하여 BPR/PI를 선택해야 하며, 대부분의 현실에서는 핵심 프로세스에 BPR, 지원 프로세스에 PI를 병행한다.

## Ⅰ. 개요 및 필요성

BPR (Business Process Reengineering)은 1993년 Michael Hammer와 James Champy가 "Reengineering the Corporation"에서 체계화한 경영 혁신 방법론으로, "기존 가정과 관행을 버리고 프로세스를 근본적으로 재설계하여 비용·품질·서비스·속도에서 획기적 성과 향상을 달성하는 것"으로 정의된다.

**BPR 4가지 핵심 특성**:
1. **근본성(Fundamental)**: "왜 이 업무를 하는가"부터 재질문 — 기존 가정을 버림
2. **급진성(Radical)**: 점진적 개선이 아닌 프로세스 재창조 (redesign, not improve)
3. **극적 성과(Dramatic)**: 10~15% 개선이 아닌 70~80% 이상의 도약적 향상 목표
4. **프로세스 중심(Process-oriented)**: 기능 부서가 아닌 End-to-End 프로세스 단위로 설계

PI (Process Innovation)는 Davenport(1993)가 IT 활용 기반 프로세스 혁신으로 개념화했으며, TQM (Total Quality Management), Six Sigma, Lean 방법론과 연계하여 지속적·점진적 개선(CI, Continuous Improvement)을 추구한다.

**BPR 실패율이 70%에 달하는 이유** (CSC Index 1994, 497개 기업 조사): ① 변화 저항 관리 실패 62%, ② 경영진 스폰서십 부재 45%, ③ IT 과잉 의존 38%, ④ 단기 성과 압박으로 중도 포기 31%.

📢 **섹션 요약 비유**: BPR은 낡은 건물을 완전히 철거하고 새로 짓는 재건축이고, PI는 내부 인테리어를 조금씩 개선하는 리모델링이다 — 효과는 재건축이 크지만 위험도도 훨씬 높다.

## Ⅱ. 아키텍처 및 핵심 원리

### BPR vs PI 비교표

| 비교 항목 | BPR | PI (Process Innovation) |
|:---|:---|:---|
| 접근 방식 | 제로 베이스 재설계 | 기존 프로세스 개선 |
| 변화 규모 | 급진적 (70~80% 향상 목표) | 점진적 (10~30% 향상) |
| 기간 | 6~18개월 집중 프로젝트 | 지속적 (상시 활동) |
| 리스크 | 매우 높음 (70% 실패율) | 낮음 |
| 주도자 | 경영진·외부 컨설턴트 | 현업 프로세스 오너 |
| 방법론 | Hammer&Champy, Zero-Based | TQM, Six Sigma, Lean, Kaizen |
| IT 역할 | 핵심 Enabler | 지원 도구 |
| 성공률 | ~30% | ~70% 이상 |

### BPR 수행 4단계 프로세스

```
┌──────────┬──────────┬──────────────┬───────────────────┐
│  1단계   │  2단계   │   3단계      │   4단계           │
│ 현황 분석 │ 비전 설계 │  신규 설계   │  이행·변화관리    │
│          │          │              │                   │
│ AS-IS    │ 목표성과  │ TO-BE 설계   │ 파일럿 테스트     │
│ 맵핑     │ 정의     │ IT 아키텍처  │ IT 시스템 구축    │
│ 병목분석  │ KPI 도출  │ 설계         │ 교육 훈련         │
│ 이슈목록  │          │ 파일럿 계획  │ 변화관리 프로그램  │
└──────────┴──────────┴──────────────┴───────────────────┘
```

**Lean과 Six Sigma 통합**:
- **Lean BPR**: BPR 급진성 + Lean 낭비 제거(7 Wastes: 과잉생산·대기·운반·과잉가공·재고·동작·불량) 결합 → 고객 가치 흐름(Value Stream) 기준 재설계
- **Six Sigma DMAIC**: Define-Measure-Analyze-Improve-Control 사이클 → 6σ = 백만 개 중 3.4 DPMO 수준

📢 **섹션 요약 비유**: BPR이 "처음부터 다시 짜는 레시피 개발"이라면, Six Sigma PI는 "기존 레시피에서 실수를 줄이고 일관성을 높이는 품질 관리"다.

## Ⅲ. 비교 및 연결

| 구분 | BPR | TQM | Six Sigma | Lean | Kaizen |
|:---|:---|:---|:---|:---|:---|
| 변화 단위 | 프로세스 전체 | 품질 전반 | 프로세스 품질 | 가치 흐름 | 소규모 개선 |
| 주도 | 탑다운 | 전사 | 프로젝트 팀 | 가치흐름 팀 | 현장 작업자 |
| 속도 | 급진적 | 점진적 | 프로젝트 단위 | 지속적 | 매일 |
| IT 의존도 | 매우 높음 | 중간 | 중간 | 낮음~중간 | 낮음 |
| 원천 | Hammer&Champy | Deming, Juran | Motorola, GE | Toyota TPS | Toyota |

**디지털 BPR**: RPA (Robotic Process Automation), AI/ML, 클라우드를 핵심 IT Enabler로 활용하여 기존 BPR보다 구현 속도를 40% 이상 단축한다.

📢 **섹션 요약 비유**: BPR, TQM, Six Sigma, Lean, Kaizen은 같은 "더 나은 프로세스"를 향한 다른 도구들 — BPR이 불도저라면 Kaizen은 빗자루이고 Six Sigma는 정밀 측정기다.

## Ⅳ. 실무 적용 및 기술사 판단

**BPR 적용 의사결정 체크리스트**:
- [ ] 현재 프로세스가 경쟁 열위의 근본 원인인가? (성과 격차 > 50%)
- [ ] 경영진이 3~5년 장기 투자를 확약하는가?
- [ ] 변화관리(Change Management) 전담 조직이 구성되어 있는가?
- [ ] IT 인프라가 신규 프로세스를 지원할 수 있는가?
- [ ] 파일럿 부서에서 리스크를 먼저 검증할 수 있는가?
- [ ] 실패 시 롤백 계획이 준비되어 있는가?

**안티패턴**:
1. **"자동화 = BPR" 오해**: 비효율 프로세스를 IT로 자동화하면 비효율만 빠르게 실행 → 프로세스 재설계 선행 필수
2. **과도한 범위**: 전사 모든 프로세스를 동시에 BPR → 조직 마비. 핵심 2~3개부터 시작
3. **현업 참여 배제**: 컨설턴트·IT 주도 → 현업 거부감, 이행 후 실용성 부족
4. **변화관리 과소평가**: Kotter 8단계 변화관리 모델 무시 → 저항으로 프로젝트 좌초

**Hammer&Champy BPR 성공 원칙 4가지**:
1. 프로세스 오너(Process Owner) 지정
2. IT를 프로세스 변화의 Enabler로 활용 (not automator)
3. 성과 지표를 결과(Outcome) 중심으로 재정의
4. 조직 문화와 보상 체계를 새 프로세스에 맞게 변경

📢 **섹션 요약 비유**: BPR 실패의 70%는 설계 문제가 아니라 사람 문제다 — 아무리 완벽한 새 레시피도 요리사들이 거부하면 아무 소용이 없다.

## Ⅴ. 기대효과 및 결론

**BPR 성공 시 기대 효과**:
- 프로세스 처리 시간: 50~80% 단축 (예: 보험 청구 30일→2일)
- 인력 소요: 30~50% 절감 (자동화·통합)
- 오류율: 60~80% 감소
- 고객 만족도: NPS 20~40점 향상

**PI 지속 적용 기대 효과**:
- 연간 생산성 향상: 5~15%
- 직원 몰입도 향상 (Kaizen: 직원이 개선의 주체)
- 품질 비용(Cost of Quality) 20~40% 절감

**한계 및 선결 조건**:
- BPR: CEO 스폰서십, 변화관리 예산(총 비용의 15~20%), IT 인프라 준비도
- PI: 프로세스 성숙도 최소 2단계 이상, 데이터 수집 체계, 상시 개선 문화

📢 **섹션 요약 비유**: BPR 기대효과는 낡은 집을 재건축했을 때의 쾌적함이고, PI는 매일 조금씩 청소·수리하여 집이 항상 깔끔하게 유지되는 것이다.

### 📌 관련 개념 맵

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| Lean | PI 방법론 | 낭비 제거를 통한 점진적 개선 |
| Six Sigma | PI 방법론 | DMAIC 기반 통계적 품질 개선 |
| RPA | BPR Enabler | 재설계된 프로세스의 자동화 도구 |
| Change Management | BPR 성공 요건 | Kotter 8단계 변화 관리 |
| ISP | 포함 관계 | ISP 로드맵에 BPR 과제 포함 |
| ERP | BPR IT 인프라 | BPR 후 신프로세스를 ERP에 구현 |

### 👶 어린이를 위한 3줄 비유 설명

1. BPR은 낡고 구불구불한 길을 없애고 직선 고속도로를 새로 놓는 것이고, PI는 기존 길의 신호등을 스마트하게 바꾸고 포트홀을 메우는 것이에요.
2. BPR이 위험한 이유는 고속도로 공사 중에 기존 길도 막혀버릴 수 있기 때문이에요 — 새 길이 완성되기 전까지 출퇴근이 더 힘들어질 수 있어요.
3. PI는 매일 교실을 조금씩 정리하는 것처럼 작지만 꾸준히 쌓이면 결국 훨씬 깔끔한 교실이 되는 거예요.
