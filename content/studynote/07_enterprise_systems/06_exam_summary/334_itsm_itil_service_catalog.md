## 핵심 인사이트 (3줄 요약)

> 1. **본질**: ITSM (IT Service Management) / ITIL (IT Infrastructure Library)의 서비스 카탈로그(Service Catalog)는 IT가 제공하는 모든 서비스를 고객 언어(비즈니스 카탈로그)와 기술 언어(기술 카탈로그)로 이원화하여 문서화한 서비스 메뉴판이다.
> 2. **가치**: 서비스 요청(Request Fulfillment) 표준화, SLA 연계, CMDB 통합을 통해 IT 서비스 가시성을 높이고 서비스 데스크의 처리 효율을 40~60% 향상시킨다.
> 3. **판단 포인트**: 서비스 카탈로그의 성공 여부는 비즈니스 부서(고객)가 IT 용어 없이 원하는 서비스를 찾을 수 있는가, 즉 "고객 관점의 서비스 언어화"에 달려 있다.

## Ⅰ. 개요 및 필요성

ITIL (IT Infrastructure Library)은 영국 정부 CCTA가 1980년대 개발하고 현재 Axelos가 관리하는 IT 서비스 관리 베스트 프랙티스 모음이다. ITIL 4(2019)는 서비스 가치 시스템(SVS, Service Value System)과 4차원 모델을 중심으로 디지털 전환 시대에 맞게 개정되었다.

**ITSM의 핵심 목표**: IT 서비스를 제품이 아닌 비즈니스 서비스로 인식하고, 고객이 원하는 결과(Outcome)를 최저 비용·최소 리스크로 달성하는 것.

**서비스 카탈로그 3가지 구성**:
1. **비즈니스 서비스 카탈로그(Business Service Catalog)**: 현업 고객이 보는 서비스 목록 — IT 용어 없이 비즈니스 언어로 기술 (예: "노트북 신청", "사용자 계정 생성")
2. **기술 서비스 카탈로그(Technical Service Catalog)**: IT 운영팀이 보는 기술 상세 — CI, SLA, 의존성 포함
3. **서비스 파이프라인(Service Pipeline)**: 아직 출시되지 않은 개발 중 서비스

**서비스 포트폴리오 = 파이프라인 + 카탈로그 + 폐기 서비스**

📢 **섹션 요약 비유**: 서비스 카탈로그는 식당 메뉴판 — 손님(현업)은 "파스타"(비즈니스 카탈로그)를 주문하고, 주방(IT)은 "토마토 소스 250g, 파스타 180g, 조리 시간 12분"(기술 카탈로그)으로 처리한다.

## Ⅱ. 아키텍처 및 핵심 원리

### 서비스 카탈로그 관리 구조

| 구성 요소 | 대상 | 내용 | 연계 시스템 |
|:---|:---|:---|:---|
| 비즈니스 서비스 카탈로그 | 현업 고객 | 서비스명, 설명, 요청 방법, SLA | 서비스 포털 |
| 기술 서비스 카탈로그 | IT 운영팀 | CI 목록, 의존성, 기술 스펙 | CMDB |
| 서비스 파이프라인 | IT 기획팀 | 개발 예정 서비스, 출시 일정 | PM 도구 |
| 서비스 폐기 목록 | IT 자산 관리 | 지원 종료 서비스, EOL 일정 | CMDB |

### ITIL 서비스 요청 처리 흐름

```
│  └──────────┘    ┌────▼────┐                    │

│          ┌────────────┼──────────────┐          │

│     │ 표준    │  │ 변경    │  │  인시던트 │     │
│     │ 요청    │  │  요청   │  │  연계     │     │
│     └────┬────┘  └────┬────┘  └─────┬────┘     │

│                       │ 처리 완료                │
│               ┌───────▼───────┐                 │

│               └───────────────┘                 │
└─────────────────────────────────────────────────┘
```

**ITIL 4 주요 관행(Practice) 26개 개요**:
- 일반 관리: 지식 관리, 포트폴리오 관리, 리스크 관리 등 14개
- 서비스 관리: 인시던트 관리, 문제 관리, 변경 통제 등 17개
- 기술 관리: 소프트웨어 개발·관리, 인프라·플랫폼 관리 등 3개

**ServiceNow 구현**: 업계 최대 ITSM 플랫폼으로 서비스 카탈로그, CMDB, 인시던트/문제/변경 관리를 통합하여 제공한다. Gartner Magic Quadrant ITSM 부문 10년 연속 리더.

📢 **섹션 요약 비유**: ITSM 서비스 카탈로그는 앱스토어처럼 — 사용자는 원하는 앱(서비스)을 찾아 설치 신청하고, 뒤에서는 자동으로 프로비저닝되며 SLA(앱 다운로드 시간) 안에 제공된다.

## Ⅲ. 비교 및 연결

| 비교 항목 | 서비스 카탈로그 | 서비스 포트폴리오 | CMDB |
|:---|:---|:---|:---|
| 범위 | 현재 운영 중인 서비스 | 전체 서비스 라이프사이클 | IT 구성 항목(CI) |
| 대상 | 현업 + IT | IT 기획팀 | IT 운영팀 |
| 내용 | 서비스 메뉴 + SLA | 파이프라인+카탈로그+폐기 | 하드웨어·소프트웨어·관계 |
| 업데이트 | 서비스 변경 시 | 분기별 포트폴리오 리뷰 | 변경 관리 연계 자동 |

**SLA (Service Level Agreement) 연계**:
- 서비스 카탈로그 각 항목에 SLA 항목 명시: 가용성(99.9%), 응답 시간(<4시간), 해결 시간(<8시간)
- SLO (Service Level Objective): 내부 팀 목표 (SLA보다 엄격)
- SLI (Service Level Indicator): 실제 측정값 (모니터링 데이터)

📢 **섹션 요약 비유**: 서비스 카탈로그+SLA는 식당 메뉴판에 "주문 후 15분 이내 서빙, 미달 시 10% 할인"이라고 명시한 것 — 고객과 IT의 서비스 수준 계약이다.

## Ⅳ. 실무 적용 및 기술사 판단

**서비스 카탈로그 구축 체크리스트**:
- [ ] 비즈니스 서비스를 현업 언어(IT 용어 없이)로 기술했는가?
- [ ] 각 서비스에 SLA (응답·해결 시간, 가용성)가 연계되어 있는가?
- [ ] 서비스 요청 승인 워크플로우가 정의되어 있는가?
- [ ] CMDB와 연계하여 서비스 구성 항목(CI)이 매핑되어 있는가?
- [ ] 서비스 카탈로그 정기 갱신(신규·변경·폐기) 프로세스가 있는가?
- [ ] 사용자 포털에서 셀프 서비스가 가능한가?

**안티패턴**:
1. **기술 카탈로그만 존재**: IT 내부 용어로만 기술 → 현업이 원하는 서비스를 찾지 못함
2. **SLA 미연계**: 서비스 메뉴는 있지만 이행 기준 없음 → 현업 불만 증가
3. **정기 갱신 없음**: 구축 후 방치 → 현실과 카탈로그 불일치
4. **CMDB 분리**: 서비스 카탈로그와 CMDB가 연계되지 않아 의존성 추적 불가

📢 **섹션 요약 비유**: 서비스 카탈로그 없이 IT 서비스를 요청하는 것은 메뉴판 없는 식당에서 음식을 주문하는 것 — 무엇이 있는지, 얼마나 걸리는지 아무도 모른다.

## Ⅴ. 기대효과 및 결론

**기대 효과**:
- 서비스 데스크 효율: 표준화된 요청으로 처리 시간 40~60% 단축
- 현업 만족도: 셀프 서비스 포털로 24/7 서비스 요청 가능
- 가시성 향상: IT 서비스 범위·비용 투명화 → IT 예산 정당성 확보
- SLA 이행률: 측정 가능한 목표로 IT 서비스 품질 관리

**한계**:
- 초기 구축 공수 높음 (현업 인터뷰 + IT 서비스 목록화)
- 서비스 변경 시 카탈로그 갱신 프로세스 유지 어려움
- 조직 문화: 현업이 직접 담당자에게 전화하는 관행 극복 필요

**선결 조건**: ① ServiceNow 또는 동급 ITSM 플랫폼 ② CMDB 구축 ③ 현업-IT 공동 서비스 언어 정의 워크숍

📢 **섹션 요약 비유**: 서비스 카탈로그는 도시의 버스 노선 안내도 — 어떤 버스(서비스)가 어디로 가는지, 얼마나 걸리는지를 시민(현업)이 한눈에 알 수 있어야 진짜 역할을 한다.

### 👶 어린이를 위한 3줄 비유 설명

1. 서비스 카탈로그는 IT 서비스의 메뉴판이에요 — "노트북이 필요해요", "비밀번호를 잊었어요" 같은 것을 클릭 몇 번으로 신청할 수 있도록 정리한 것이에요.
2. SLA는 "피자 30분 내 배달, 초과 시 무료"처럼 IT가 얼마나 빨리 도와줄지 약속한 것이에요.
3. CMDB는 서비스를 제공하기 위해 필요한 모든 IT 장비와 소프트웨어를 연결 관계까지 기록한 지도예요.
