## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 프로세스 마이닝(Process Mining)은 정보시스템의 이벤트 로그(Event Log)에서 실제 프로세스를 자동으로 발굴·분석하여 현행 프로세스와 설계 프로세스 간의 격차(Gap)를 가시화한다.
> 2. **가치**: 컨설턴트의 인터뷰 기반 프로세스 매핑과 달리 100% 실증 데이터 기반으로 병목·우회·위반을 정확히 파악하여 자동화 및 개선 기회를 객관적으로 발굴한다.
> 3. **판단 포인트**: 이벤트 로그 품질(Case ID·Activity·Timestamp 삼요소 완비)이 분석 가능 여부를 결정하며, 적합성 검사(Conformance Checking)로 규정 준수율을 정량화할 수 있다.

## Ⅰ. 개요 및 필요성

기업의 실제 프로세스는 문서화된 SOP(Standard Operating Procedure)와 다르게 흘러가는 경우가 많다. ERP에서 구매 발주 승인이 역순으로 처리되거나, 병원에서 진료 후 수납이 이중으로 발생하는 등의 '프로세스 이탈(Process Deviation)'이 빈번하다.

프로세스 마이닝은 Wil van der Aalst 교수(아인트호벤 공과대학교)가 2000년대 초 정립한 학문으로, 데이터 마이닝(Data Mining) 기법을 프로세스 분석에 적용한다. ERP, CRM, BPM 등 정보시스템에 자동 생성되는 이벤트 로그를 분석하여 ①실제 프로세스 발굴(Process Discovery), ②설계 vs 실제 적합성 검사(Conformance Checking), ③성과 분석 및 개선(Enhancement)을 수행한다.

| 분석 유형 | 목적 | 질문 |
|:---|:---|:---|
| 발굴(Discovery) | 현행 프로세스 가시화 | "실제로 어떻게 하고 있나?" |
| 적합성 검사(Conformance) | 규정 준수 여부 확인 | "설계대로 하고 있나?" |
| 개선(Enhancement) | 병목·낭비 제거 | "어디서 시간이 낭비되나?" |

📢 **섹션 요약 비유**: 프로세스 마이닝은 CCTV로 공장 작업 흐름을 녹화해서 나중에 분석하는 것처럼, 시스템 로그라는 기록으로 실제 업무 흐름을 재현한다.

## Ⅱ. 아키텍처 및 핵심 원리

### 이벤트 로그 구조 (XES 표준)

```
이벤트 로그 (Event Log)
├── 케이스 1 (Case ID: ORD-001)
│   ├── 이벤트: 주문 접수     (Timestamp: 2024-01-10 09:00)
│   ├── 이벤트: 재고 확인     (Timestamp: 2024-01-10 09:15)
│   ├── 이벤트: 결제 처리     (Timestamp: 2024-01-10 09:30)
│   └── 이벤트: 배송 지시     (Timestamp: 2024-01-10 10:00)
├── 케이스 2 (Case ID: ORD-002)
│   ├── 이벤트: 주문 접수     (Timestamp: 2024-01-10 09:05)
│   ├── 이벤트: 재고 부족     (Timestamp: 2024-01-10 09:20) ← 이탈
│   └── 이벤트: 주문 취소     (Timestamp: 2024-01-10 09:45)
```

### 프로세스 마이닝 아키텍처

```
┌──────────────────────────────────────────────────────┐
│               소스 시스템 (데이터 원천)                │
│    [SAP ERP]  [Salesforce CRM]  [ServiceNow]          │
└──────────────────────┬───────────────────────────────┘
                       │ 이벤트 로그 추출
┌──────────────────────▼───────────────────────────────┐
│         이벤트 로그 전처리 (ETL/ELT)                  │
│     Case ID · Activity · Timestamp 정규화              │
└──────────────────────┬───────────────────────────────┘
                       │
┌──────────────────────▼───────────────────────────────┐
│           프로세스 마이닝 엔진                          │
│  ┌───────────┐ ┌──────────────┐ ┌─────────────┐      │
│  │ Discovery │ │ Conformance  │ │ Enhancement │      │
│  │ Alpha/    │ │ Checking     │ │ 병목 분석    │      │
│  │ Inductive │ │ (준수율 측정) │ │ 대기시간 분석│      │
│  └───────────┘ └──────────────┘ └─────────────┘      │
└──────────────────────┬───────────────────────────────┘
                       │
               대시보드·시각화 (Celonis, ProM)
```

### 주요 프로세스 발굴 알고리즘

| 알고리즘 | 특징 | 적합 상황 |
|:---|:---|:---|
| Alpha Algorithm | 단순, 최초 제안 | 교육·이론 이해 |
| Heuristic Miner | 노이즈에 강건 | 실제 복잡한 로그 |
| Inductive Miner | 완전 적합 모델 보장 | 규정 준수 분석 |
| Fuzzy Miner | 복잡한 프로세스 단순화 | 대규모 변형 프로세스 |

📢 **섹션 요약 비유**: 이벤트 로그 3요소(Case·Activity·Timestamp)는 범인 수사의 누가(누구)·무엇을·언제처럼 사건 재구성의 최소 필수 정보다.

## Ⅲ. 비교 및 연결

### 프로세스 마이닝 vs 전통적 프로세스 분석

| 구분 | 전통 BPR/컨설팅 | 프로세스 마이닝 |
|:---|:---|:---|
| 데이터 기반 | 인터뷰·워크숍 | 100% 실증 이벤트 로그 |
| 객관성 | 주관적 편향 가능 | 객관적 수치 기반 |
| 범위 | 선택적 프로세스 | 전체 케이스 100% 분석 |
| 속도 | 수개월 | 수주 내 분석 가능 |
| 지속성 | 일회성 분석 | 실시간 모니터링 가능 |

### 주요 플랫폼

- **Celonis**: 기업 시장 점유율 1위, SAP 연계 강점, Process Intelligence
- **UiPath Process Mining**: RPA와 통합된 자동화 기회 발굴
- **ProM**: 오픈소스, 연구·학술 표준, Eindhoven 대학 개발
- **Signavio**: SAP 인수, BPM과 통합

📢 **섹션 요약 비유**: 프로세스 마이닝은 GPS 내비게이션의 실시간 교통 정보처럼, 이상적인 경로(설계 프로세스)와 실제 도로 상황(이벤트 로그)을 비교해 최적 경로를 안내한다.

## Ⅳ. 실무 적용 및 기술사 판단

### 도입 적합성 평가

1. **이벤트 로그 가용성**: SAP, Oracle ERP 등에서 이벤트 로그 추출 가능 여부
2. **분석 대상 프로세스**: 주문-배송(OTC), 구매-지불(P2P), 청구-수금(Q2C) 등 표준 프로세스
3. **비즈니스 임팩트**: 처리 건수 大 + 주기 짧음 + 규정 준수 중요 → 우선 적용

### 적합성 검사(Conformance Checking) 활용

```
설계 프로세스 (BPMN 모델)
        +
실제 이벤트 로그
        ↓
적합성 검사 결과:
  - Fitness: 설계 경로 준수율 78%
  - 이탈 패턴: 승인 단계 건너뜀 12%
  - 이탈 케이스: 내부감사 전달 → 자동 경보
```

### RPA 연계 시나리오

프로세스 마이닝으로 자동화 ROI가 높은 프로세스 식별 → RPA 봇 개발 우선순위 결정 → 자동화 후 프로세스 마이닝으로 효과 검증

📢 **섹션 요약 비유**: 적합성 검사는 레시피(설계)와 실제 요리 과정을 비교해 "소금을 2번 넣었네!"를 발견하는 것처럼 숨겨진 이탈을 드러낸다.

## Ⅴ. 기대효과 및 결론

프로세스 마이닝 도입 기업은 평균적으로 ①프로세스 처리 시간 20~30% 단축, ②규정 준수 위반 탐지율 향상, ③RPA/자동화 ROI 향상(자동화 후보 정확도 증가), ④감사(Audit) 비용 절감 등의 효과를 보고하고 있다. BMW, Siemens, 한국 주요 은행들이 SAP 기반 OTC/P2P 프로세스 마이닝을 운영 중이다.

**한계**: 이벤트 로그 품질이 낮으면(Case ID 누락, 타임스탬프 불일치) 의미 있는 분석이 불가능하다. 또한 프로세스 마이닝 결과 해석에는 도메인 전문가의 개입이 필수적이다.

📢 **섹션 요약 비유**: 프로세스 마이닝은 블랙박스 영상처럼, 사고(프로세스 이탈)가 발생했을 때 "언제, 어디서, 왜"를 정확하게 되돌아볼 수 있게 해준다.

### 👶 어린이를 위한 3줄 비유 설명

1. 프로세스 마이닝은 학교에서 학생들이 화장실 갈 때마다 선생님이 "누가, 몇 시에, 얼마나 걸렸나"를 기록해서 나중에 분석하는 것과 같아요.
2. 모든 기록을 보면 "항상 특정 학생이 청소 시간에 갔네!"처럼 숨겨진 패턴을 발견할 수 있어요.
3. 이걸 통해 "청소 전에 미리 화장실 시간을 주면 되겠다"는 개선안을 데이터로 증명할 수 있는 거예요.
