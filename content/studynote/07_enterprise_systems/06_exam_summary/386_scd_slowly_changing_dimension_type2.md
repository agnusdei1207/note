## 핵심 인사이트 (3줄 요약)

> 1. **본질**: SCD(Slowly Changing Dimension)는 DW 차원 테이블에서 속성값이 시간에 따라 변할 때 이력을 어떻게 관리할지를 정의한 패턴으로, Type 2가 가장 널리 사용된다.
> 2. **가치**: Type 2는 변경 발생 시 새 행을 추가하고 유효 기간(Effective/Expiry Date)을 관리하여 과거 임의 시점의 데이터 상태를 완전히 재현할 수 있게 한다.
> 3. **판단 포인트**: 이력 재현 필요성(Type 2), 현재값만 필요(Type 1), 이전·현재값 동시 보존(Type 3) 중 비즈니스 요구에 맞는 타입을 선택해야 한다.

## Ⅰ. 개요 및 필요성

DW에서 고객의 주소, 직원의 부서, 제품의 카테고리 등은 시간이 지나면서 변경된다. 이를 차원 테이블에서 어떻게 처리하느냐에 따라 과거 거래 분석의 정확성이 달라진다. 예를 들어 고객이 서울에서 부산으로 이사한 뒤 발생한 매출과 이사 전 매출을 지역별로 분석할 때, 이력이 없으면 모든 과거 거래가 '부산 고객'으로 집계되는 오류가 발생한다.

Ralph Kimball은 이 문제를 해결하기 위해 SCD Type 1~7을 정의했다. 실무에서는 Type 1(덮어쓰기), Type 2(이력 행 추가), Type 3(컬럼 추가) 세 가지가 주로 사용된다.

📢 **섹션 요약 비유**: SCD는 인물 사진 앨범 관리법 — Type 1은 최신 사진으로 교체, Type 2는 연도별 사진을 모두 보관, Type 3는 최신과 직전 사진 두 장만 보관.

## Ⅱ. 아키텍처 및 핵심 원리

**SCD Type 2 구조:**
```
surrogate_key | natural_key | name   | address | eff_date   | exp_date   | is_current
─────────────────────────────────────────────────────────────────────────────────────
1001          | C-001       | 홍길동 | 서울    | 2022-01-01 | 2024-06-30 | N
1002          | C-001       | 홍길동 | 부산    | 2024-07-01 | 9999-12-31 | Y
```

| SCD 타입 | 처리 방식 | 이력 보존 | 활용 사례 |
|:---|:---|:---|:---|
| Type 0 | 변경 무시 | 없음 | 불변 속성(생년월일) |
| Type 1 | 현재값 덮어쓰기 | 없음 | 오타 수정 |
| Type 2 | 신규 행 추가 | 완전 이력 | 고객 주소, 직원 부서 |
| Type 3 | 컬럼 추가(이전값) | 직전값만 | 조직 개편(현재+이전 부서) |
| Type 6 | 1+2+3 혼합 | 완전 이력 + 현재값 컬럼 | 고성능 이력 쿼리 |

📢 **섹션 요약 비유**: Type 2는 주민등록 이력 등본 — 모든 주소 변경 이력이 날짜와 함께 보존된다.

## Ⅲ. 비교 및 연결

Type 2 구현의 핵심 컬럼: (1) surrogate_key(PK), (2) natural_key(소스 키), (3) eff_date(유효 시작일), (4) exp_date(유효 종료일, 현재 행은 9999-12-31), (5) is_current(현재 행 플래그). 팩트 테이블 조인 시 트랜잭션 날짜가 차원 행의 eff_date~exp_date 범위 안에 있는 서로게이트 키를 사용한다.

📢 **섹션 요약 비유**: 팩트-차원 조인은 타임머신 탑승 — 거래 날짜에 맞는 차원 상태(행)를 찾아 정확한 과거로 이동한다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- 과거 임의 시점 재현 필수(법적 감사, 재무 분석): Type 2 필수
- 현재 상태만 필요(실시간 대시보드): Type 1 충분
- 두 번째 이전 상태까지만 필요(단순 비교): Type 3 고려
- Type 6(Hybrid): 현재값 컬럼 추가로 is_current=Y 필터 없이 현재 조회 성능 향상

📢 **섹션 요약 비유**: Type 선택은 보험 등급 — 보장 범위(이력 깊이)가 넓을수록 보험료(저장 공간·복잡도)가 높아진다.

## Ⅴ. 기대효과 및 결론

SCD Type 2는 DW의 시간적 정확성(Temporal Accuracy)을 보장하는 핵심 메커니즘으로, 과거 거래를 해당 시점의 차원 상태로 정확히 분석할 수 있게 한다. 단, 차원 테이블 크기 증가와 ETL 복잡도 상승이 한계이며, 서로게이트 키 기반 Lookup 로직과 eff_date/exp_date 범위 조인 최적화가 필수 전제 조건이다.

📢 **섹션 요약 비유**: SCD Type 2는 블랙박스 영상 — 언제 무슨 일이 있었는지 정확히 되감아 볼 수 있는 완전한 기록이다.

### 👶 어린이를 위한 3줄 비유 설명

1. SCD는 주소가 바뀐 친구의 연락처를 어떻게 관리할지에 대한 규칙이야.
2. Type 1은 최신 주소로 덮어쓰기(이전 주소 사라짐), Type 2는 이전 주소도 날짜와 함께 보관.
3. Type 2 덕분에 "작년 크리스마스에 그 친구 어디 살았지?"를 정확히 알 수 있어!
