## 핵심 인사이트 (3줄 요약)

> 1. **본질**: PLM(Product Lifecycle Management)은 제품 기획부터 설계·생산·서비스·폐기까지 전 수명 주기의 데이터(BOM·도면·사양)를 통합 관리하며, BOM(Bill of Materials)은 제품 구성 부품 목록으로 설계-생산-서비스의 공통 데이터 언어다.
> 2. **가치**: 설계 변경(ECO, Engineering Change Order)이 BOM→MES→ERP로 자동 전파되면 수작업 재입력 오류와 버전 불일치 문제가 제거되어 제품 개발 기간과 비용이 감소한다.
> 3. **판단 포인트**: EBOM(Engineering BOM)→MBOM(Manufacturing BOM)→SBOM(Service BOM) 전환 프로세스와 ECO(Engineering Change Order) 승인 체계가 PLM 성숙도를 결정한다.

## Ⅰ. 개요 및 필요성

제조업의 경쟁력은 신제품 출시 속도(TTM, Time-to-Market)와 품질에 달려있다. PLM 없이 운영하면 ①설계팀의 CAD 도면과 생산팀의 BOM이 별도 관리 → 버전 불일치, ②설계 변경 시 수작업으로 변경 통보 → 현장에서 구버전으로 생산, ③제품 수명 종료 시 서비스 도면 부재 → 사후 서비스 불가 등의 문제가 발생한다.

PLM은 Windchill(PTC), Teamcenter(Siemens), ENOVIA(Dassault), SAP PLM 등이 대표 솔루션이다. CAD(Computer-Aided Design) 통합 관리, PDM(Product Data Management), 협업 설계, 규정 준수 관리를 포괄한다.

| PLM 주요 영역 | 내용 |
|:---|:---|
| PDM (Product Data Management) | 도면·BOM·문서 버전 관리 |
| CPDM (Collaborative PDM) | 글로벌 팀 협업 설계 |
| 변경 관리 (ECM) | ECR→ECO→ECN 변경 프로세스 |
| 규정 준수 | RoHS·REACH 환경 규정 BOM 연계 |
| 서비스 관리 | After-market 서비스 BOM·매뉴얼 |

📢 **섹션 요약 비유**: PLM은 제품의 출생증명서부터 사망신고까지 모든 서류를 관리하는 호적 시스템처럼, 제품의 전 생애 기록을 관리한다.

## Ⅱ. 아키텍처 및 핵심 원리

### BOM 유형 비교

```
EBOM (Engineering BOM)
  └── 설계 관점: 부품의 기능·규격 중심
      예: "모터 A, 2000RPM 이상, 300W"

MBOM (Manufacturing BOM)
  └── 제조 관점: 실제 생산 공정·순서 반영
      예: "모터 A-001 (공급업체 X), 조립 공정 3단계"

SBOM (Service BOM)
  └── 서비스 관점: 교체·수리 단위 부품 목록
      예: "모터 어셈블리 #3 (현장 교체 가능 최소 단위)"

PBOM (Process BOM)
  └── 공정 BOM: 제조 공정 순서와 BOM 통합
```

### BOM 전환 흐름

```
설계 팀                 생산 기술 팀              서비스 팀
  │                          │                       │
[EBOM]  ──── 전환 ────▶  [MBOM]  ──── 전환 ────▶  [SBOM]
  │                          │                       │
CAD 연계               공정 설계 반영            서비스 묶음 설정
부품 규격              실 사용 부품 코드 매핑      교환 주기 정의
```

### ECO(Engineering Change Order) 프로세스

```
1. ECR (Engineering Change Request): 변경 요청 발의
2. 영향 분석: 변경 부품 사용처 BOM 검색 → 영향 범위 파악
3. ECO 승인: 설계·생산·구매·품질 부서 전자 서명
4. ECN (Engineering Change Notice): 변경 공지 배포
5. 생산 반영: 유효 날짜 기준 MES·ERP BOM 자동 업데이트
6. 이전 재고 처리: 구부품 소진 계획 수립
```

📢 **섹션 요약 비유**: ECO 프로세스는 법 개정처럼, 변경 발의 → 심의 → 공포 → 시행 순서를 밟아야 현장 혼란 없이 설계 변경이 적용된다.

## Ⅲ. 비교 및 연결

### PLM vs ERP의 BOM 관리

| 구분 | PLM | ERP |
|:---|:---|:---|
| BOM 종류 | EBOM (설계) | MBOM (생산 실행) |
| 버전 관리 | 설계 이력 전체 보존 | 현재 유효 버전 중심 |
| 변경 프로세스 | ECO 승인 워크플로우 | MRP 재계산용 BOM 업데이트 |
| 통합 | CAD 직접 연계 | MES·구매·재고와 연계 |
| 사용자 | 설계·생산기술 엔지니어 | 생산관리·구매·재무 담당자 |

### PLM과 디지털 트윈 연계

PLM의 EBOM/MBOM 데이터는 디지털 트윈(Digital Twin) 구축의 핵심 기반이다. 실제 제품의 부품 구성(BOM)과 설계 데이터(CAD)를 디지털 트윈에 반영하면 시뮬레이션·예측 유지보수가 가능하다.

📢 **섹션 요약 비유**: PLM의 EBOM→MBOM 전환은 건축 설계도면(EBOM)이 실제 시공 도면(MBOM)으로 변환되는 것처럼, 이상과 현실을 연결하는 번역 과정이다.

## Ⅳ. 실무 적용 및 기술사 판단

### PLM 도입 핵심 과제

1. **BOM 표준화**: 설계팀마다 다른 BOM 구조를 표준 레벨 수(통상 3~7레벨)로 통일
2. **CAD 통합**: Catia, SOLIDWORKS, NX 등 CAD 시스템과 PLM 양방향 연계
3. **ERP 연계**: MBOM → ERP 자재 마스터 자동 생성, ECO 변경 실시간 반영
4. **글로벌 협업**: 해외 설계 파트너 접근 권한 관리, 동시 설계 충돌 방지
5. **표준·규정 관리**: ISO, 환경 규정(RoHS), 안전 규정(UL/CE) BOM 내 마킹

### 의사결정 포인트

- **EBOM vs MBOM 별도 관리**: 복잡한 다품목 제조업은 EBOM·MBOM 분리 관리 필수
- **PLM 도입 범위**: 전사 일시 도입 vs 신제품 라인 시범 도입 (파일럿 권장)
- **ERP 통합 깊이**: PLM→ERP 단방향 BOM 이관 vs 양방향 실시간 동기화

📢 **섹션 요약 비유**: BOM 표준화는 레고 블록 규격처럼, 모든 팀이 같은 규격(표준 BOM 레벨)으로 부품을 정의해야 설계도를 공유하고 조립할 수 있다.

## Ⅴ. 기대효과 및 결론

PLM 도입 제조 기업은 ①신제품 개발 기간 20~30% 단축(ECO 자동화), ②BOM 오류로 인한 생산 중단 80% 감소, ③설계 변경 비용 30~50% 절감, ④서비스 부품 조달 효율화로 애프터마켓 수익 향상 등의 효과를 거둔다. 항공·자동차·방산 업종에서 PLM은 설계 데이터 추적성 확보를 위한 법적 요건이기도 하다.

**한계**: PLM 도입 프로젝트는 BOM 데이터 마이그레이션(기존 Excel BOM의 PLM 전환)이 가장 어렵고 시간이 많이 걸리는 부분이다. 또한 설계 엔지니어들의 기존 도구(Excel·CAD)에서 PLM으로의 업무 방식 전환 저항이 크다.

📢 **섹션 요약 비유**: PLM은 제품의 유전자 지도(DNA)처럼, 태어나는 순간부터 폐기될 때까지 모든 정보가 기록되어 언제든 참조하고 추적할 수 있는 시스템이다.

### 📌 관련 개념 맵

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| MES | 연계 | MBOM을 MES 작업 지시에 활용 |
| ERP | 연계 | MBOM → ERP 자재 마스터, 생산 오더 |
| CAD | 통합 | 3D 설계 데이터가 PLM EBOM의 원천 |

### 👶 어린이를 위한 3줄 비유 설명

1. PLM은 장난감 설명서처럼, 장난감이 어떤 부품으로 만들어졌고(BOM), 어떻게 조립하는지(공정), 망가지면 어떻게 고치는지(서비스 BOM)가 모두 담겨있어요.
2. 설계가 바뀌면(ECO) 모든 팀이 자동으로 새 설명서를 받아서, 구버전으로 만드는 실수가 없어져요.
3. 자동차 공장에서 나사 하나가 바뀌면 PLM이 "여기 쓰이는 나사야!"라며 다른 100개 제품에도 적용해야 한다고 알려주는 거예요.
