## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 서킷 브레이커(Circuit Breaker) 패턴은 원격 서비스 장애 시 요청을 즉시 실패 처리(Fail Fast)하여 장애의 연쇄 전파(Cascading Failure)를 막고, 시스템이 스스로 회복할 시간을 주는 분산 시스템 복원력 패턴이다.
> 2. **가치**: Closed(정상) → Open(차단) → Half-Open(회복 탐색) 3가지 상태 전이로 장애 서비스로의 요청을 자동 차단하고, 폴백(Fallback) 응답으로 부분 서비스 가용성을 유지한다.
> 3. **판단 포인트**: 임계값(오류율/지연시간)·타임아웃·폴백 로직 설계가 서킷 브레이커의 효과를 결정하며, 너무 민감하면 정상 서비스도 차단하는 오동작이 발생할 수 있다.

## Ⅰ. 개요 및 필요성

분산 시스템에서 한 서비스의 장애가 의존 서비스 전체로 확산되는 "연쇄 장애(Cascading Failure)"는 MSA의 핵심 위험 중 하나다. 예를 들어 결제 서비스가 30초 타임아웃으로 응답하면, 이를 기다리는 주문 서비스의 스레드가 모두 대기 상태가 되어 결국 주문 서비스도 다운되는 상황이 발생한다.

Martin Fowler가 2014년 공식 문서화한 서킷 브레이커 패턴은 전기 회로의 차단기에서 유래했다. 과부하 시 자동으로 회로를 차단(Open)하고, 일정 시간 후 소량의 전류(요청)로 회복 여부를 확인(Half-Open)하는 자동 보호 장치다.

구현 라이브러리: Netflix Hystrix(2012, 유지보수 모드), Resilience4j(Java, 권장), Polly(.NET), py-breaker(Python).

📢 **섹션 요약 비유**: 서킷 브레이커는 전기 차단기처럼, 과부하(장애)가 감지되면 자동으로 회로(연결)를 차단해서 전체 화재(시스템 장애)를 막는다.

## Ⅱ. 아키텍처 및 핵심 원리

### 서킷 브레이커 상태 전이

```
정상 상태
    │
    ▼
┌─────────┐    오류율 > 임계값 (예: 50%)    ┌──────────┐
│  CLOSED  │ ─────────────────────────────▶ │   OPEN   │
│(요청 허용)│                               │(요청 차단)│
└─────────┘                               └─────┬─────┘
    ▲                                           │
    │                                     타임아웃 경과
    │                                     (예: 30초)
    │                                           ▼
    │     성공 임계값 도달 (예: 성공 3회)  ┌──────────────┐
    └──────────────────────────────────── │ HALF-OPEN    │
                                          │(소량 요청 허용)│
                                          └──────────────┘
                                               │
                                    요청 실패 → 다시 OPEN
```

### 핵심 설정 파라미터

| 파라미터 | 설명 | 예시 값 |
|:---|:---|:---|
| failureRateThreshold | Open 전환 오류율 임계값 | 50% |
| slowCallRateThreshold | 느린 호출 비율 임계값 | 100% |
| slowCallDurationThreshold | 느린 호출 기준 시간 | 2000ms |
| waitDurationInOpenState | Open 상태 유지 시간 | 30s |
| permittedNumberOfCallsInHalfOpenState | Half-Open 허용 요청 수 | 5 |
| minimumNumberOfCalls | 통계 수집 최소 요청 수 | 10 |

### 폴백(Fallback) 패턴

```java
// Resilience4j 예시
CircuitBreaker cb = CircuitBreaker.ofDefaults("paymentService");
Supplier<PaymentResponse> supplier = CircuitBreaker
    .decorateSupplier(cb, () -> paymentService.process(request));

// 폴백 적용
Try.ofSupplier(supplier)
    .recover(CallNotPermittedException.class,
        ex -> PaymentResponse.cachedFallback()) // 캐시된 응답 반환
    .recover(Exception.class,
        ex -> PaymentResponse.degraded());      // 저하된 응답 반환
```

📢 **섹션 요약 비유**: Half-Open 상태는 환자가 퇴원 전 시범 외출처럼, 완전히 회복되었는지 소량 테스트 요청으로 확인하는 관찰 기간이다.

## Ⅲ. 비교 및 연결

### 서킷 브레이커 vs 재시도(Retry) 패턴

| 구분 | 서킷 브레이커 | 재시도 (Retry) |
|:---|:---|:---|
| 목적 | 장애 전파 차단 | 일시적 오류 극복 |
| 동작 | 즉시 실패 처리 | 지수 백오프로 재시도 |
| 적합 오류 | 지속적 서비스 장애 | 네트워크 일시 오류 |
| 결합 | 재시도와 함께 사용 가능 | 서킷 브레이커 안에 내포 |

### 관련 패턴

- **벌크헤드(Bulkhead)**: 서비스별 스레드 풀/세마포어 격리로 한 서비스 장애가 다른 서비스 스레드를 잠식하지 않도록 방지
- **타임아웃(Timeout)**: 응답 대기 시간 상한 설정으로 스레드 자원 낭비 방지
- **헬스체크(Health Check)**: 서킷 브레이커와 연계하여 서비스 상태 지속 모니터링

📢 **섹션 요약 비유**: 재시도와 서킷 브레이커의 조합은 문이 잠겼을 때 몇 번 시도해보고(재시도), 계속 안 되면 다른 문을 찾는(폴백) 전략이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 서킷 브레이커 설계 원칙

1. **적절한 임계값**: 너무 낮으면 오동작(False Open), 너무 높으면 장애 전파
2. **폴백 전략 설계**: 캐시 응답·기본값·다운그레이드 서비스 등 사전 정의
3. **모니터링 연계**: Actuator + Prometheus + Grafana로 서킷 상태 실시간 모니터링
4. **Bulkhead 결합**: 서킷 브레이커 + 스레드 풀 격리로 이중 보호

### 의사결정 포인트

- **임계값 설정**: 서비스의 정상 오류율 + 표준편차를 기반으로 임계값 결정
- **Open 유지 시간**: 장애 서비스가 복구되는 평균 시간(MTTR)을 기준으로 설정
- **폴백 품질**: "없음"보다는 "제한된 응답"이 고객 경험 관점에서 항상 우수

📢 **섹션 요약 비유**: 서킷 브레이커 임계값 설정은 체온계 기준처럼, 37.5도(임계값)가 되면 열이 있다고 판단하는 기준이 너무 높거나 낮으면 오진이 생긴다.

## Ⅴ. 기대효과 및 결론

서킷 브레이커 도입으로 ①연쇄 장애 방지(장애 격리), ②폴백으로 부분 서비스 가용성 유지, ③장애 서비스 자동 복구 탐지, ④응답 시간 예측 가능성 향상(빠른 실패) 등의 효과를 거둔다. Netflix의 Hystrix가 초당 수십억 건의 격리된 스레드 실행으로 서비스 가용성 99.99%를 유지하는 데 기여했다.

**한계**: 서킷 브레이커는 데이터의 최종 일관성 문제를 해결하지 못한다. Open 상태 동안 쓰기 요청이 실패하면, 이를 어떻게 처리할지(재시도, 보상 트랜잭션, 이벤트 소싱)에 대한 별도 설계가 필요하다.

📢 **섹션 요약 비유**: 서킷 브레이커는 소방 방화문처럼, 화재(장애)가 옆 건물(서비스)로 번지는 걸 막지만, 화재 자체를 끄는(장애 복구) 역할은 소방서(모니터링·운영팀)가 해야 한다.

### 👶 어린이를 위한 3줄 비유 설명

1. 서킷 브레이커는 집 전기 차단기처럼, 한 방(서비스)에서 과부하가 생기면 자동으로 스위치를 내려서 집 전체(시스템)가 타버리는 걸 막아요.
2. 잠시 후 스위치를 다시 올려봐서(Half-Open) 전기가 잘 들어오면(서비스 회복) 정상 상태로 돌아가요.
3. 전기가 꺼진 동안은 비상 손전등(Fallback 응답)을 사용해서, 어둠 속(전체 서비스 중단)이 되지 않도록 해요.
