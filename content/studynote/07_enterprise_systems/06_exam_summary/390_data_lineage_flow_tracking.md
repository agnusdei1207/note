## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 데이터 리니지(Data Lineage)는 데이터가 소스에서 최종 소비 지점까지 흐르는 경로·변환·의존 관계를 시각적으로 추적하는 데이터 계보(系譜) 관리 기술이다.
> 2. **가치**: 데이터 오류 발생 시 영향 범위(Impact Analysis)를 즉시 파악하고, 규정 준수(GDPR 개인정보 추적) 및 데이터 신뢰성 검증의 핵심 도구가 된다.
> 3. **판단 포인트**: 컬럼 수준(Column-Level) 리니지가 테이블 수준보다 정밀하지만 구현 비용이 높아 규정 준수 요건과 비용을 함께 고려해야 한다.

## Ⅰ. 개요 및 필요성

데이터 파이프라인이 복잡해질수록 "이 대시보드 숫자가 왜 잘못됐지?"를 추적하는 작업이 어려워진다. 데이터 리니지는 소스 테이블→ETL 변환→중간 집계→최종 리포트까지의 전 경로를 그래프로 표현하여 데이터 출처(Provenance), 변환 이력, 하위 영향(Downstream Impact)을 즉시 가시화한다.

GDPR·금융 규정에서는 개인정보가 어느 시스템에 복제·처리됐는지 추적을 요구하므로 리니지가 컴플라이언스 의무가 되기도 한다.

📢 **섹션 요약 비유**: 데이터 리니지는 식품 원산지 추적 — 내 식탁의 음식이 어느 농장, 어느 공장을 거쳐 왔는지 한눈에 볼 수 있다.

## Ⅱ. 아키텍처 및 핵심 원리

```
Source DB          ETL Pipeline       Staging          DW             Dashboard
─────────          ────────────       ───────          ──            ─────────
orders_raw   ──>   join + filter  ──> orders_clean ──> sales_fact ──> Monthly_Sales
customers_raw──>   (customer_id)  ──> cust_clean  ──> dim_customer      ↑
                                                                   [리니지 그래프]
```

**리니지 수준 비교:**

| 수준 | 설명 | 정밀도 | 구현 비용 |
|:---|:---|:---|:---|
| 시스템 리니지 | 시스템 간 흐름 | 낮음 | 낮음 |
| 테이블 리니지 | 테이블 간 흐름 | 중간 | 중간 |
| 컬럼 리니지 | 컬럼 간 변환 추적 | 높음 | 높음 |

**구현 방식:**
- **정적 파싱**: SQL/ETL 코드 파싱으로 의존 관계 추출 (dbt, Apache Atlas)
- **런타임 수집**: 실행 시점 쿼리 로그 파싱 (OpenLineage, Marquez)
- **API 통합**: Airflow, Spark, dbt가 OpenLineage API로 리니지 자동 전송

📢 **섹션 요약 비유**: 컬럼 리니지는 DNA 추적 — "이 숫자의 조상이 어느 컬럼인지"를 세포 수준까지 추적한다.

## Ⅲ. 비교 및 연결

영향 분석(Impact Analysis) vs 근본 원인 분석(Root Cause Analysis): 리니지는 양방향 — 업스트림(원인 추적) 방향과 다운스트림(영향 범위) 방향 모두를 탐색할 수 있어야 한다.

📢 **섹션 요약 비유**: 업스트림은 강의 발원지 찾기, 다운스트림은 강물이 어디까지 흘러가는지 확인하기다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- GDPR 개인정보 처리 추적 의무: 컬럼 수준 리니지 필수
- dbt 기반 ELT: dbt Lineage Graph 자동 생성(dbt docs)
- Apache Airflow + OpenLineage: 런타임 리니지 자동 수집
- 데이터 품질 오류 영향 범위: 다운스트림 리니지로 즉시 파악

📢 **섹션 요약 비유**: OpenLineage는 리니지 공통 언어 — 여러 도구(Airflow, Spark, dbt)가 같은 형식으로 리니지 정보를 공유한다.

## Ⅴ. 기대효과 및 결론

데이터 리니지를 통해 오류 진단 시간 70% 단축, 규정 준수 감사 대응 자동화, 파이프라인 변경 영향 사전 예측이 가능하다. 컬럼 수준 리니지는 정밀하지만 SQL 파싱 복잡도와 성능 오버헤드가 크므로, 규정 준수 요건이 있는 민감 데이터 영역부터 단계적으로 도입하는 것이 현실적이다.

📢 **섹션 요약 비유**: 리니지는 데이터 세계의 GPS 이동 경로 — 어디서 출발해서 어디를 거쳐 여기까지 왔는지 항상 알 수 있다.

### 👶 어린이를 위한 3줄 비유 설명

1. 리니지는 족보 — 이 데이터가 할아버지(소스) → 아버지(ETL) → 나(대시보드)까지 어떻게 내려왔는지 보여줘.
2. 숫자가 잘못됐을 때, 족보(리니지)를 보면 어느 조상(단계)에서 실수가 있었는지 바로 알 수 있어.
3. GDPR은 개인 정보가 어느 집안(시스템)을 거쳐갔는지 증명서를 요구해 — 리니지가 그 증명서야!
