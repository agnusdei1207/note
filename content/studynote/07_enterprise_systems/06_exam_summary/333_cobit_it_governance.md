## 핵심 인사이트 (3줄 요약)

> 1. **본질**: COBIT (Control Objectives for Information and Related Technologies)은 ISACA가 개발한 IT 거버넌스 및 관리 프레임워크로, 40개 거버넌스·관리 목표를 5개 도메인(EDM/APO/BAI/DSS/MEA)으로 구성하여 IT가 비즈니스 가치를 창출하도록 통제한다.
> 2. **가치**: IT 거버넌스의 책임 소재와 의사결정 체계를 명확히 하여 리스크 관리, 규정 준수(SOX, ISO 27001), 자원 최적화를 동시에 달성한다.
> 3. **판단 포인트**: COBIT은 "무엇을(What)" 해야 하는가를 정의하는 프레임워크이며, "어떻게(How)" 구현할지는 ITIL(운영), ISO 27001(보안), TOGAF(아키텍처) 등과 연계하여 보완해야 한다.

## Ⅰ. 개요 및 필요성

COBIT은 1996년 ISACA (Information Systems Audit and Control Association)가 발표하고, COBIT 5(2012) → COBIT 2019(2018)로 진화한 IT 거버넌스 프레임워크다. COBIT 2019는 이전 버전 대비 Design Factors(설계 요인) 11개를 추가하여 조직 맥락에 맞는 맞춤형 거버넌스 시스템을 설계할 수 있도록 했다.

**IT 거버넌스가 필요한 이유**:
1. **가치 창출**: IT 투자가 비즈니스 목표에 기여하는가?
2. **리스크 최적화**: IT 관련 리스크를 허용 가능한 수준으로 관리하는가?
3. **자원 최적화**: IT 자원(인력, 기술, 정보, 인프라)을 효율적으로 활용하는가?
4. **규정 준수**: SOX (Sarbanes-Oxley Act), GDPR, ISO 27001 등 법적·규제 요구사항 충족

**COBIT 2019 5개 도메인**:
1. **EDM (Evaluate, Direct, Monitor)**: 거버넌스 목표 평가·방향 설정·모니터링
2. **APO (Align, Plan, Organise)**: 전략 정렬·계획·조직화
3. **BAI (Build, Acquire, Implement)**: 구축·구매·구현
4. **DSS (Deliver, Service, Support)**: 서비스 제공·지원
5. **MEA (Monitor, Evaluate, Assess)**: 성과 모니터링·평가·검토

📢 **섹션 요약 비유**: COBIT은 기업 IT를 운영하는 헌법 — 무엇이 허용되고 금지되는지, 누가 결정권을 갖는지, 어떻게 감독하는지의 기본 틀을 제공한다.

## Ⅱ. 아키텍처 및 핵심 원리

### COBIT 2019 5개 도메인 구조

| 도메인 | 약자 | 목표 수 | 주요 역할 |
|:---|:---|:---:|:---|
| Evaluate, Direct & Monitor | EDM | 5 | 이사회·경영진 거버넌스 |
| Align, Plan & Organise | APO | 14 | CIO, IT 전략 조직 |
| Build, Acquire & Implement | BAI | 10 | IT 프로젝트·개발팀 |
| Deliver, Service & Support | DSS | 6 | IT 운영·서비스 데스크 |
| Monitor, Evaluate & Assess | MEA | 4 | 내부 감사·컴플라이언스 |

### COBIT 거버넌스 구조 다이어그램

```
┌────────────────────────────────────────────────────┐
│              COBIT 2019 거버넌스 구조               │
│                                                    │
│  ┌──────────────────────────────────────────────┐  │
│  │        EDM (이사회·경영진)                    │  │
│  │   방향 설정 → 위임 → 모니터링                 │  │
│  └──────────────────┬───────────────────────────┘  │
│                     │ 위임                           │
│  ┌──────────────────▼───────────────────────────┐  │
│  │    APO (IT 전략·계획·조직화) = 14개 목표      │  │
│  └──────┬───────────────────────────────┬───────┘  │
│         │ 구축                           │ 서비스    │
│  ┌──────▼──────┐             ┌──────────▼──────┐   │
│  │  BAI        │             │    DSS           │   │
│  │  (구축·구현) │             │  (운영·지원)      │   │
│  │  10개 목표  │             │   6개 목표        │   │
│  └──────┬──────┘             └──────────┬──────┘   │
│         └─────────────┬─────────────────┘           │
│               ┌───────▼──────┐                      │
│               │  MEA (감사)   │                      │
│               │  4개 목표     │                      │
│               └───────────────┘                     │
└────────────────────────────────────────────────────┘
```

**성숙도 모델 (Maturity Model, 0~5)**:
- 0: 불완전 (Incomplete)
- 1: 수행됨 (Performed)
- 2: 관리됨 (Managed)
- 3: 확립됨 (Established)
- 4: 예측 가능 (Predictable)
- 5: 최적화 (Optimising)

📢 **섹션 요약 비유**: COBIT 성숙도 모델은 학생의 학년처럼 — 0학년(아무것도 안 함)에서 5학년(자율적으로 계속 발전)까지 조직의 IT 거버넌스 성숙 수준을 측정한다.

## Ⅲ. 비교 및 연결

| 비교 항목 | COBIT | ITIL | ISO 27001 | TOGAF |
|:---|:---|:---|:---|:---|
| 초점 | IT 거버넌스·통제 | IT 서비스 운영 | 정보 보안 관리 | IT 아키텍처 |
| 범위 | 전사 IT 거버넌스 | IT 서비스 프로세스 | 보안 통제 | 아키텍처 프레임워크 |
| 질문 | What to govern? | How to operate? | How to secure? | How to architect? |
| 산출물 | 거버넌스 정책·목표 | 서비스 카탈로그·SLA | ISMS 인증 | 아키텍처 산출물 |
| 인증 | CGEIT, CISA | ITIL Foundation | ISO 27001 인증 | TOGAF 인증 |
| SOX 연계 | 강함 (IT 통제 감사) | 간접 | 중간 | 없음 |

**SOX (Sarbanes-Oxley Act) IT 컴플라이언스**:
- Section 302: CEO/CFO의 재무 보고 정확성 인증 → IT 통제 관련
- Section 404: 내부 통제 평가 → COBIT EDM·MEA 도메인 활용
- COBIT의 APO12(리스크 관리), MEA02(내부 통제 시스템)가 SOX 감사의 핵심 증거

📢 **섹션 요약 비유**: COBIT은 IT 거버넌스의 헌법, ITIL은 운영 규정, ISO 27001은 보안 법규 — 모두 필요하지만 각자 담당하는 영역이 다르다.

## Ⅳ. 실무 적용 및 기술사 판단

**COBIT 도입 체크리스트**:
- [ ] 조직의 IT 거버넌스 성숙도 현황 평가가 완료되었는가?
- [ ] 11개 Design Factors로 맞춤형 거버넌스 시스템을 설계했는가?
- [ ] 40개 목표 중 조직 우선순위에 따라 10~15개를 선택했는가?
- [ ] RACI 차트로 각 목표의 책임자·협력자를 지정했는가?
- [ ] KGI (Key Goal Indicators) 및 KPI를 목표별로 정의했는가?
- [ ] ITIL, ISO 27001과의 매핑이 되어 있는가?

**안티패턴**:
1. **40개 전체 목표 동시 구현 시도**: 리소스 부족으로 모두 피상적 수준에 머뭄 → 5~10개 우선순위부터 시작
2. **COBIT = ITIL 혼동**: COBIT은 거버넌스(무엇을), ITIL은 운영(어떻게) — 역할 구분 필수
3. **컴플라이언스 체크 리스트화**: 진짜 거버넌스 없이 감사 체크리스트 충족에만 집중
4. **경영진 참여 부재**: IT 부서만 COBIT 적용 → 비즈니스 가치 창출 연계 불가

**COBIT 2019 Design Factors 11가지**:
전략 목표, 엔터프라이즈 리스크 프로파일, IT 관련 이슈, 신뢰·보증 요구사항, 위협 환경, IT 컴플라이언스 요구사항, IT 역할, 소싱 모델, IT 도입 방법, IT 규모, 조직 모델

📢 **섹션 요약 비유**: COBIT의 40개 목표를 모두 동시에 구현하려는 것은 의대 교과서를 하루에 다 외우려는 것 — 환자(조직)에게 가장 시급한 치료부터 시작하는 것이 현명하다.

## Ⅴ. 기대효과 및 결론

**기대 효과**:
- IT 거버넌스 체계화: 이사회~CIO~운영팀까지 명확한 역할·책임 체계
- SOX 컴플라이언스: 상장사 IT 내부 통제 감사 대응 효율화
- 리스크 관리 강화: 리스크 기반 통제로 IT 사고 30~50% 감소
- IT 투자 정당성: IT ROI 측정 체계 구축으로 예산 확보 용이

**한계**:
- 구현 복잡성: 40개 목표 전체 구현은 2~3년 이상 소요
- 중소기업 부담: 대기업 중심 설계로 중소기업 적용 어려움
- 측정 도구 부재: COBIT 자체는 측정 도구가 아니므로 별도 도구 필요

**선결 조건**: ① CGEIT/CISA 자격 보유자 or 컨설턴트 지원 ② 경영진 거버넌스 의지 ③ ITIL·ISO 27001 연계 계획

📢 **섹션 요약 비유**: COBIT은 비행기 조종 매뉴얼 — 모든 체크리스트를 완벽하게 수행하면 안전하게 날 수 있지만, 매뉴얼만 있고 조종사(경영진)가 참여하지 않으면 비행기는 움직이지 않는다.

### 📌 관련 개념 맵

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| ITIL | 운영 보완 | COBIT 거버넌스 + ITIL 운영 = 완전한 IT 관리 |
| ISO 27001 | 보안 보완 | COBIT DSS 도메인과 ISO 27001 통제 연계 |
| TOGAF | 아키텍처 보완 | COBIT APO 도메인과 TOGAF 아키텍처 정렬 |
| SOX | 법적 요구사항 | COBIT EDM·MEA 도메인으로 SOX 대응 |
| ISACA | 개발 기관 | COBIT 발행, CISA·CGEIT 자격증 관리 |
| CMMI | 프로세스 성숙도 | COBIT 성숙도 + CMMI 개발 프로세스 |

### 👶 어린이를 위한 3줄 비유 설명

1. COBIT은 학교의 교칙 같은 것이에요 — 선생님(이사회)이 어떤 규칙을 정하고, 반장(CIO)이 어떻게 실행하고, 교장(감사)이 잘 지키는지 확인하는 체계예요.
2. COBIT의 5단계 성숙도는 자전거 실력 — 처음엔 보조 바퀴로(1단계), 나중엔 혼자 잘 타고(3단계), 최고는 묘기도 부리는(5단계) 것처럼 조직의 IT 수준을 보여줘요.
3. COBIT만 있고 ITIL이 없으면 규칙은 있는데 실제로 어떻게 일할지 모르는 것이에요 — 둘이 함께 있어야 완전한 IT 관리가 돼요.
