## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 데이터 카탈로그(Data Catalog)는 조직의 모든 데이터 자산에 대한 메타데이터를 수집·색인·검색·태깅하여 데이터 발견(Data Discovery)을 가능하게 하는 메타데이터 관리 시스템이다.
> 2. **가치**: "이 데이터가 어디서 왔고, 무엇을 의미하며, 누가 관리하는가?"를 즉시 답할 수 있게 하여 데이터 거버넌스와 자가 서비스 분석(Self-Service Analytics)의 기반을 제공한다.
> 3. **판단 포인트**: 기술 메타데이터(스키마·통계) 자동 수집과 비즈니스 메타데이터(용어·오너·태그) 수동 보강의 균형이 카탈로그 품질의 핵심이다.

## Ⅰ. 개요 및 필요성

데이터 레이크·DW·데이터베이스가 늘어날수록 "분석에 필요한 데이터가 어디 있는가?"를 찾는 데 드는 시간이 전체 데이터 작업의 50~80%를 차지한다고 알려져 있다. 데이터 카탈로그는 이 검색 비용을 줄이고, 데이터의 의미(Semantic)와 품질을 공유하여 중복 작업을 방지한다.

AWS Glue Data Catalog, Apache Atlas, Google Data Catalog, Alation, Collibra 등이 대표적인 구현체다.

📢 **섹션 요약 비유**: 데이터 카탈로그는 도서관 사서 — 어떤 책(데이터)이 어디 있고 무슨 내용인지 빠르게 찾아주는 안내자이다.

## Ⅱ. 아키텍처 및 핵심 원리

```
┌──────────────────────────────────────────────────────────┐
│                   Data Catalog                            │
│  ┌─────────────────────┐   ┌────────────────────────┐    │
│  │ Technical Metadata   │   │ Business Metadata       │    │
│  │ - Schema, Types      │   │ - Glossary Terms        │    │
│  │ - Row Count, Nulls   │   │ - Data Owner            │    │
│  │ - Lineage Graph      │   │ - Tags, Classifications │    │
│  └──────────┬──────────┘   └───────────┬────────────┘    │
│             └──────────────┬───────────┘                  │
│                            ▼                              │
│              ┌─────────────────────────┐                  │
│              │  Search & Discovery UI   │                  │
│              │  (키워드/태그/도메인 검색)│                  │
│              └─────────────────────────┘                  │
└──────────────────────────────────────────────────────────┘
```

| 메타데이터 유형 | 내용 | 수집 방식 |
|:---|:---|:---|
| 기술 메타데이터 | 테이블 스키마, 컬럼 타입, 통계, 파티션 | 자동 크롤링(커넥터) |
| 비즈니스 메타데이터 | 비즈니스 용어, 데이터 오너, 분류, 태그 | 수동 또는 ML 자동화 |
| 운영 메타데이터 | 마지막 갱신, 쿼리 빈도, 접근 로그 | 시스템 자동 수집 |
| 리니지 메타데이터 | 데이터 흐름·변환 계보 | ETL/ELT 파싱 |

📢 **섹션 요약 비유**: 메타데이터는 상품 라벨 — 원산지(출처), 성분(스키마), 유통기한(갱신일), 담당자(오너)를 한 장에 담은 것이다.

## Ⅲ. 비교 및 연결

데이터 카탈로그 vs 데이터 딕셔너리: 딕셔너리는 기술적 스키마 정의에 집중하는 반면, 카탈로그는 비즈니스 맥락·리니지·검색·태깅을 포함한 통합 메타데이터 플랫폼이다.

📢 **섹션 요약 비유**: 딕셔너리는 단어 정의집, 카탈로그는 단어 정의 + 사용 예시 + 관련어 + 저자 정보가 모두 있는 백과사전이다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- 클라우드 AWS 환경: Glue Data Catalog (S3·Athena 연동 내장)
- 오픈소스 온프레미스: Apache Atlas + Ranger (Hadoop 생태계)
- 기업급 거버넌스: Collibra, Alation (비즈니스 용어 집·워크플로우 내장)
- AI 자동 태깅: ML로 컬럼 이름·샘플 데이터 분석 → 민감 데이터 자동 분류

📢 **섹션 요약 비유**: 카탈로그 도구 선택은 사서 고용 — AWS 기본 사서(Glue), 전문 큐레이터(Collibra) 중 조직 규모에 맞게 선택한다.

## Ⅴ. 기대효과 및 결론

데이터 카탈로그 도입으로 데이터 검색 시간 60~70% 단축, 중복 데이터 파이프라인 방지, 규정 준수(GDPR·개인정보보호법) 지원이 가능하다. 단, 메타데이터 품질 유지를 위한 지속적인 큐레이션과 데이터 오너십 문화 정착이 필수 전제 조건이다.

📢 **섹션 요약 비유**: 카탈로그 유지는 정원 관리 — 한 번 만들고 끝이 아니라 꾸준히 가꾸어야 아름다움(데이터 품질)이 유지된다.

### 👶 어린이를 위한 3줄 비유 설명

1. 데이터 카탈로그는 학교 도서관 목록 — 어떤 책이 어디 있는지, 누가 빌려가도 되는지 알 수 있어.
2. 태그는 책에 붙은 스티커 — "어린이용", "과학", "재미있음" 같은 표시로 원하는 책을 빨리 찾아.
3. 오너는 책 관리 선생님 — 이 데이터가 정확한지, 언제 업데이트됐는지 책임지는 사람이야.
