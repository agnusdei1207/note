## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 데이터 패브릭(Data Fabric)은 분산된 이기종 데이터 소스를 메타데이터 지능(AI/ML 기반 자동화)으로 가상적으로 연결하여 물리적 이동 없이 통합 접근을 제공하는 아키텍처 개념이다.
> 2. **가치**: 데이터 이동·복제 없이 실시간 연방 쿼리(Federated Query)를 실현하고, AI가 메타데이터를 분석하여 데이터 파이프라인 자동화·추천을 제공한다.
> 3. **판단 포인트**: 데이터 패브릭은 기술보다 아키텍처 원칙에 가까우며, 데이터 가상화(Data Virtualization)·카탈로그·거버넌스·리니지의 통합으로 실현된다.

## Ⅰ. 개요 및 필요성

Gartner가 주창한 데이터 패브릭은 하이브리드/멀티클라우드 환경에서 온프레미스·클라우드·엣지에 분산된 데이터를 마치 하나의 패브릭(직물)처럼 촘촘히 연결하는 아키텍처다. 핵심 차별점은 "AI/ML 기반 메타데이터 지능" — 활성 메타데이터(Active Metadata)를 분석하여 데이터 통합 패턴을 자동 추천하고 파이프라인을 자동화한다.

📢 **섹션 요약 비유**: 데이터 패브릭은 스마트 도로망 — 각 도시(데이터 소스)를 물리적으로 합치지 않고, 지능형 도로(메타데이터 AI)로 연결하여 어디서든 빠르게 이동한다.

## Ⅱ. 아키텍처 및 핵심 원리

```
┌────────────────────────────────────────────────────────────┐
│                    DATA FABRIC                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │      Active Metadata Layer (AI/ML 지능)               │  │
│  │  자동 데이터 발견 · 품질 추천 · 파이프라인 자동화       │  │
│  └─────────────────┬────────────────────────────────────┘  │
│                    │                                        │
│  ┌─────────────────┼────────────────────────┐              │
│  ▼                 ▼                        ▼              │
│ ┌──────────┐  ┌──────────┐  ┌──────────────────────────┐  │
│ │ On-Prem  │  │ Cloud DW │  │  Data Lake / Edge        │  │
│ │ DB       │  │ (BQ, RS) │  │  IoT, SaaS, APIs         │  │
│ └──────────┘  └──────────┘  └──────────────────────────┘  │
└────────────────────────────────────────────────────────────┘
```

| 구성 요소 | 역할 |
|:---|:---|
| Active Metadata | AI/ML이 메타데이터를 분석하여 자동 추천 |
| Data Virtualization | 물리 이동 없이 가상 통합 접근 제공 |
| Data Catalog | 분산 데이터 자산 발견·태깅 |
| Federated Governance | 전사 정책 + 로컬 자율 거버넌스 |
| Lineage & Quality | 자동 리니지 추적 및 품질 모니터링 |

📢 **섹션 요약 비유**: Active Metadata는 스마트 비서 — 새 데이터가 들어오면 "이건 이전 데이터와 연관성이 있어요"를 자동으로 알려준다.

## Ⅲ. 비교 및 연결

| 항목 | 데이터 메시 | 데이터 패브릭 |
|:---|:---|:---|
| 핵심 접근 | 조직/거버넌스(도메인 분산) | 기술/자동화(메타데이터 AI) |
| 구현 주체 | 도메인 팀 | 중앙 플랫폼 팀 |
| 데이터 이동 | 도메인별 로컬 | 가상화로 이동 최소화 |
| 상호 보완 | 패브릭 기술로 메시 구현 가능 | 메시 조직 위에 패브릭 기술 적용 |

📢 **섹션 요약 비유**: 메시는 조직 설계, 패브릭은 기술 설계 — 두 개를 함께 쓰면 최강 조합이다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- 하이브리드/멀티클라우드 환경 복잡성 해소: 데이터 패브릭 접근
- 데이터 이동 비용/규제 이슈: 가상화(Virtualization) 우선
- AI 기반 자동화 성숙도 요구: Active Metadata 플랫폼(TIBCO, IBM Cloud Pak)
- 소규모 환경: 과잉 투자 위험 — 단계적 도입 권장

📢 **섹션 요약 비유**: 패브릭은 고급 스마트홈 시스템 — 모든 가전을 연결하는 허브가 있으면 편리하지만 설치 비용도 크다.

## Ⅴ. 기대효과 및 결론

데이터 패브릭은 복잡한 하이브리드 환경에서 데이터 통합 비용을 줄이고 AI 기반 자동화로 파이프라인 생산성을 높인다. 메타데이터 지능이 핵심이므로, 카탈로그와 리니지의 품질이 패브릭 효과를 결정한다. 대규모 기업의 데이터 플랫폼 현대화 전략에서 데이터 메시와 상호 보완적으로 적용하는 것이 최적의 접근이다.

📢 **섹션 요약 비유**: 데이터 패브릭은 인터넷 — 모든 컴퓨터(데이터 소스)를 물리적으로 합치지 않고 프로토콜(메타데이터 표준)로 연결한다.

### 📌 관련 개념 맵

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| Active Metadata | 핵심 기술 | AI/ML로 메타데이터를 능동적으로 활용 |
| Data Virtualization | 구현 기술 | 물리 이동 없이 통합 쿼리 |
| Data Mesh | 상호 보완 | 조직 접근 + 기술 접근 결합 |
| Gartner | 개념 출처 | 데이터 패브릭 아키텍처 패턴 정의 |
| Federated Query | 핵심 기능 | 분산 소스 동시 쿼리 |

### 👶 어린이를 위한 3줄 비유 설명

1. 데이터 패브릭은 마법 전화기 — 세계 어디 있는 친구(데이터 소스)와도 직접 연결되지만, 친구를 내 집에 데려올(이동) 필요가 없어.
2. AI 비서가 "이 데이터랑 저 데이터가 연관 있어요"를 자동으로 알려줘 — 내가 직접 찾지 않아도 돼.
3. 여러 나라 언어(다른 DB 형식)를 자동으로 통역해줘서 하나로 볼 수 있어!
