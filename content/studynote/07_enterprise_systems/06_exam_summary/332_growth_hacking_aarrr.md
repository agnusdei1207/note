## 핵심 인사이트 (3줄 요약)

> 1. **본질**: AARRR (Acquisition→Activation→Retention→Revenue→Referral) 프레임워크는 Dave McClure(2007)가 제안한 스타트업 성장 측정 지표 체계로, 고객이 제품을 발견·사용·재방문·결제·추천하는 전체 여정을 5단계 퍼널로 측정한다.
> 2. **가치**: 각 퍼널 단계의 전환율을 측정하여 성장의 병목(Bottleneck)을 식별하고 그로스 해킹 실험의 우선순위를 데이터 기반으로 결정한다.
> 3. **판단 포인트**: 퍼널 전환율 개선의 시작은 "가장 낮은 단계"가 아니라 "개선 임팩트가 가장 큰 단계"를 찾는 것이며, A/B 테스트는 통계적 유의성(p<0.05, 95% 신뢰 수준)을 반드시 확보해야 한다.

## Ⅰ. 개요 및 필요성

Growth Hacking은 2010년 Sean Ellis가 만든 용어로, "제품 성장을 최우선 목표로 하는 실험 기반 마케팅·개발·데이터 통합 접근법"이다. 전통 마케팅이 브랜드 인지도에 집중한다면, 그로스 해킹은 퍼널 각 단계의 수치 개선에 집중한다.

AARRR은 "해적 지표(Pirate Metrics)"라고도 불리며 Dave McClure(500 Startups)가 2007년 SlideShare에 공개한 스타트업 성과 측정 프레임워크다.

**AARRR 5단계**:
1. **Acquisition (획득)**: 고객이 제품을 어떻게 발견하는가? (SEO, SEM, SNS, 바이럴)
2. **Activation (활성화)**: 첫 방문 고객이 핵심 가치를 경험하는가? (Aha! Moment)
3. **Retention (유지)**: 고객이 반복 방문·사용하는가? (Habit Loop)
4. **Revenue (수익)**: 고객이 결제하는가? (전환 최적화, 가격 실험)
5. **Referral (추천)**: 고객이 다른 사람에게 추천하는가? (바이럴 루프)

**업계 평균 전환 벤치마크**:
- Acquisition → Activation: ~20%
- Activation → Retention (Day 7): ~10%
- Retention → Revenue: ~5%
- Revenue → Referral: ~2%

📢 **섹션 요약 비유**: AARRR 퍼널은 마케팅 깔때기 — 100명이 유입되면 20명이 첫 경험, 10명이 재방문, 5명이 결제, 2명이 친구에게 추천하는 수치를 관리하는 것이다.

## Ⅱ. 아키텍처 및 핵심 원리

### AARRR 퍼널 구조

```
┌──────────────────────────────────────────────────┐

│                                                  │

│        │  Acquisition (획득)        │  100명      │
│        └──────────────┬─────────────┘            │
│   20%  ┌──────────────▼─────────────┐            │
│        │  Activation (활성화)       │   20명      │
│        └──────────────┬─────────────┘            │
│   10%  ┌──────────────▼─────────────┐            │
│        │  Retention (유지)          │   10명      │
│        └──────────────┬─────────────┘            │

└──────────────────────────────────────────────────┘
```

### AARRR 단계별 핵심 지표

| 단계 | 핵심 지표 | 개선 실험 |
|:---|:---|:---|
| Acquisition | CAC, 채널별 CVR, CPC | SEO, SEM, 바이럴 캠페인 |
| Activation | Aha! Moment 도달률, 온보딩 완료율 | 온보딩 플로우 A/B 테스트 |
| Retention | DAU/MAU 비율, Day1/7/30 유지율 | 푸시 알림, 이메일 리인게이지먼트 |
| Revenue | ARPU, LTV, 전환율, 업셀 비율 | 가격 실험, 결제 UX 최적화 |
| Referral | 바이럴 계수(k), NPS, 초대 수락률 | 추천 인센티브 A/B 테스트 |

**바이럴 계수(Viral Coefficient, k)**:
k = 초대 발송 수 × 수락 전환율
- k > 1: 기하급수적 성장 (네트워크 효과)
- k = 1: 정체 (현재 사용자 수 유지)
- k < 1: 감소 (신규 획득이 이탈보다 빠르지 않으면 감소)

**A/B 테스트 유의성**: p < 0.05 (95% 신뢰 수준), 최소 샘플 크기 계산 필요 (α=0.05, β=0.20, 효과 크기 기준)

**코호트 분석(Cohort Analysis)**: 동일 시기에 가입한 고객 그룹의 시간별 유지율을 추적하여 유지율 개선 효과를 정확하게 측정한다.

📢 **섹션 요약 비유**: 바이럴 계수 k > 1은 폭탄 세일처럼 한 명이 두 명을 부르고, 두 명이 네 명을 부르는 것 — Dropbox의 "친구 추천 시 무료 용량 증가" 캠페인이 k > 1을 달성했다.

## Ⅲ. 비교 및 연결

| 비교 항목 | AARRR | AIDA | RACE | Customer Journey |
|:---|:---|:---|:---|:---|
| 제안자 | Dave McClure (2007) | St. Elmo Lewis (1898) | Smart Insights | Service Design |
| 단계 수 | 5단계 | 4단계 (Awareness-Interest-Desire-Action) | 4단계 (Reach-Act-Convert-Engage) | 비선형 |
| 초점 | 수치 측정·실험 | 광고·마케팅 설득 | 디지털 마케팅 | 사용자 경험 |
| 데이터 활용 | 핵심 (실험 기반) | 간접적 | 중간 | 정성+정량 |

**NPS와 Referral 연계**:
- NPS (Net Promoter Score) = Promoters(9~10점) % - Detractors(0~6점) %
- NPS > 50: 강력한 추천 기반 → 높은 바이럴 계수 기대
- NPS가 Referral 단계의 선행 지표로 활용

📢 **섹션 요약 비유**: AARRR이 "고객이 들어와서 결제하고 친구를 데려오는 5단계 게임"이라면, AIDA는 "광고를 보고 관심이 생겨 사는 4단계 광고 설득 게임"이다 — 디지털 시대에는 AARRR이 더 정밀한 도구다.

## Ⅳ. 실무 적용 및 기술사 판단

**그로스 해킹 실험 체크리스트**:
- [ ] 현재 AARRR 5단계 전환율이 측정되어 있는가?
- [ ] 퍼널에서 병목(가장 큰 드롭오프) 단계를 식별했는가?
- [ ] A/B 테스트 가설이 "만약 X를 변경하면 Y가 Z% 향상될 것"으로 명확한가?
- [ ] A/B 테스트 샘플 크기와 기간이 통계적으로 충분한가?
- [ ] 코호트 분석으로 유지율 추세를 추적하고 있는가?
- [ ] 허영 지표가 아닌 실행 지표(Actionable Metrics)를 주 측정 도구로 사용하는가?

**그로스 해킹 대표 사례**:
- **Dropbox**: 추천 시 무료 저장 공간 증가 → 가입자 3,900% 증가 (2008~2010)
- **Airbnb**: Craigslist 역통합(역크롤링)으로 숙소 공급자 확보
- **LinkedIn**: "이 사람 알아요?" 기능으로 연결 네트워크 성장

**안티패턴**:
1. **모든 단계 동시 최적화**: 병목 단계 집중 대신 전체 퍼널 동시 실험 → 효과 희석
2. **A/B 테스트 조기 종료**: 통계적 유의성 미확보 시 결과 발표
3. **Retention 무시**: Acquisition에만 집중 → "새는 양동이" (이탈이 획득을 초과)

📢 **섹션 요약 비유**: 새는 양동이에 물을 계속 붓는 것(Acquisition)보다 구멍을 먼저 막는 것(Retention)이 훨씬 효율적이다 — 유지율이 낮은 상태에서 마케팅 예산을 늘리면 돈을 버리는 것이다.

## Ⅴ. 기대효과 및 결론

**그로스 해킹 도입 기대 효과**:
- 성장 효율: 전통 마케팅 대비 동일 예산으로 3~10배 성장 가능
- 데이터 기반 의사결정: 직관 대신 A/B 테스트 결과로 의사결정
- 빠른 실험 사이클: 2주 단위 실험으로 연간 25+번 성장 실험

**한계**:
- B2B 장기 영업 사이클에는 AARRR 단계 측정이 어려움
- 규제 산업(금융, 의료)의 A/B 테스트 제약
- 바이럴 계수 k > 1 지속은 사실상 불가능 → 유지 전략 필수

**선결 조건**: ① 데이터 분석 인프라(GA4, Amplitude, Mixpanel) ② 실험 문화(실패 허용) ③ 교차 기능 그로스 팀(마케팅+개발+데이터)

📢 **섹션 요약 비유**: 그로스 해킹은 농사꾼이 씨앗을 뿌리기 전에 어떤 토양이 가장 잘 자라는지 작은 밭에서 먼저 실험하는 것 — 대규모 투자 전에 작은 실험으로 최적의 방법을 찾는다.

### 👶 어린이를 위한 3줄 비유 설명

1. AARRR 퍼널은 100명이 놀이동산에 들어와서 탈것을 타고(활성화), 다음 주에도 오고(유지), 티켓을 사고(수익), 친구에게 알리는(추천) 흐름을 숫자로 보는 것이에요.
2. 그로스 해킹은 과학 실험처럼 "버튼 색을 빨간색으로 바꾸면 더 많이 클릭할까?"를 두 그룹에 동시에 테스트해서 숫자로 확인하는 것이에요.
3. 유지율을 무시하고 새 고객만 모으는 것은 새는 물통에 계속 물을 붓는 것 — 먼저 구멍(이탈 원인)을 막아야 해요.
