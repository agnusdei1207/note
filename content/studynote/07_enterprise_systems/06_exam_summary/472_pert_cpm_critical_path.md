## 핵심 인사이트 (3줄 요약)

> 1. **본질**: CPM(Critical Path Method)은 프로젝트의 모든 활동과 의존 관계를 네트워크 다이어그램으로 표현하고, 프로젝트 완료 기간을 결정하는 가장 긴 경로(임계 경로, Critical Path)를 식별하는 일정 관리 기법이다.
> 2. **가치**: 임계 경로 상의 활동은 하나라도 지연되면 프로젝트 전체가 지연되므로, 프로젝트 관리자는 자원과 주의를 임계 경로에 집중하여 일정을 통제할 수 있다.
> 3. **판단 포인트**: PERT(Program Evaluation and Review Technique)는 활동 기간의 불확실성을 3점 추정(낙관/정상/비관)으로 처리하여 통계적 완료 가능성을 계산하므로, 불확실성이 높은 R&D·신기술 프로젝트에 CPM보다 적합하다.

## Ⅰ. 개요 및 필요성

"이 프로젝트를 언제까지 완료할 수 있나요?"—PM이 이 질문에 답하려면 수십~수백 개 활동의 의존 관계와 기간을 체계적으로 분석해야 한다. PERT는 1958년 미국 해군 폴라리스 미사일 개발 프로젝트에서, CPM은 1957년 듀퐁의 화학 플랜트 관리에서 각각 독립적으로 개발되어 이후 통합되었다.

두 기법의 핵심 공통 원리: **선행 관계(Precedence)가 있는 활동의 네트워크에서 전체 기간을 최소화하는 계획을 수립**하고, 각 활동의 **여유 시간(Float/Slack)**을 계산하여 어떤 활동이 지연되어도 괜찮은지, 어떤 것이 절대 지연되면 안 되는지를 식별한다.

국내 공공 SW 사업에서는 WBS(Work Breakdown Structure) 기반 일정 계획과 함께 CPM이 표준 일정 관리 기법으로 요구된다.

📢 **섹션 요약 비유**: CPM의 임계 경로는 음악 공연의 지휘자처럼, 오케스트라의 모든 악기(활동) 중 가장 늦게 끝나는 악기가 공연 시간을 결정하므로, 그 악기(임계 활동)에 먼저 집중해야 한다.

## Ⅱ. 아키텍처 및 핵심 원리

### CPM 계산 예시

```
┌──────────────────────────────────────────────────────────────┐
│              CPM 네트워크 다이어그램                          │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│            A(3일)    C(4일)                                  │
│  시작 ───▶ ●────────▶●────────▶ 종료                        │
│            │                   ▲                            │
│            │ B(5일)  D(2일)    │                            │
│            └────────▶●─────────┘                            │
│                                                              │
│  경로 1: 시작→A→C→종료 = 3+4 = 7일                          │
│  경로 2: 시작→B→D→종료 = 5+2 = 7일                          │
│                                                              │
│  ※ 두 경로 모두 임계 경로 (여유 시간 = 0)                    │
│                                                              │
│  전진 계산 (ES/EF):                                          │
│  A: ES=0, EF=3  │  C: ES=3, EF=7                           │
│  B: ES=0, EF=5  │  D: ES=5, EF=7                           │
│                                                              │
│  후진 계산 (LS/LF):                                          │
│  C: LS=3, LF=7  │  A: LS=0, LF=3                           │
│  D: LS=5, LF=7  │  B: LS=0, LF=5                           │
│                                                              │
│  여유(Float) = LS - ES 또는 LF - EF                         │
│  임계 경로: Float = 0 인 활동의 연결                         │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### PERT 3점 추정

| 구분 | 공식 | 설명 |
|:---|:---|:---|
| 기대 기간(te) | te = (O + 4M + P) / 6 | O:낙관, M:최빈, P:비관 |
| 분산(σ²) | σ² = ((P - O) / 6)² | 불확실성 정량화 |
| 표준 편차(σ) | σ = (P - O) / 6 | 기간 변동성 |

- 80% 신뢰수준 완료 시점 = te + 0.84σ
- 95% 신뢰수준 완료 시점 = te + 1.645σ

📢 **섹션 요약 비유**: PERT 3점 추정은 날씨 예보처럼, "최적의 경우 3일, 보통 5일, 최악 10일"을 통계로 결합하여 "80% 확률로 6일 내 완료"라는 확률적 일정을 제시하는 과학적 추정이다.

## Ⅲ. 비교 및 연결

### CPM vs PERT 비교

| 항목 | CPM | PERT |
|:---|:---|:---|
| 기간 추정 | 확정적 (1가지) | 확률적 (3가지) |
| 적합 프로젝트 | 반복적 경험 있는 건설/제조 | 신기술/R&D (불확실성 높음) |
| 결과 | 임계 경로 + Float | 기간 확률 분포 |
| 자원 최적화 | CPM Crashing 기법 지원 | 통계적 버퍼 관리 |

### EVM(473)과의 연계

CPM 일정과 EVM의 계획 가치(PV)를 결합하면 "일정상 지점에서 어느 활동의 비용이 얼마나 사용되어야 하는가"를 계획의 기준으로 삼을 수 있어, 일정-비용 통합 통제가 가능해진다.

📢 **섹션 요약 비유**: CPM + EVM 결합은 고속도로 여행에서 "거리(일정)"와 "기름 소비(비용)"를 동시에 모니터링하여 목적지(프로젝트 완료)까지 안전하게 도착하는 계기판 관리 체계이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 대규모 SI 프로젝트 일정 관리

1. **WBS 작성 → PERT/CPM 연동**: WBS 5수준 이하 활동에 PERT 3점 추정 적용
2. **임계 경로 집중 관리**: 주간 임계 활동 완료 현황 보고 + 리스크 조기 대응
3. **CPM Crashing**: 임계 활동에 자원 추가 투입하여 기간 단축 비용-효과 분석
4. **Fast Tracking**: 순차 활동을 병렬 수행하여 일정 단축 (품질 리스크 수반)
5. **Float 관리**: 비임계 활동의 Float를 리소스 평준화(Resource Leveling)에 활용

📢 **섹션 요약 비유**: CPM Crashing은 배달 속도를 높이기 위해 배달원을 추가 고용하는 것처럼, 시간을 줄이는 대신 비용이 증가하는 비용-일정 트레이드오프 결정이다.

## Ⅴ. 기대효과 및 결론

**기대효과**: 프로젝트 완료 일정 정확도 향상, 자원 집중 효율화, 일정 위험 사전 식별.

**한계**: 활동 기간 추정 오류 시 전체 계획 신뢰성 저하, 외부 리스크(정치적 변경, 기술 장애) 미반영, 수백 개 활동의 수동 관리 비현실적 (MS Project, Primavera 도구 필수).

📢 **섹션 요약 비유**: CPM은 정확한 시공 계획이 있어도 장마(외부 리스크)가 오면 지연되듯, 도구의 한계를 인식하고 리스크 관리와 병행해야 실효성을 발휘하는 일정 관리 기반이다.

### 👶 어린이를 위한 3줄 비유 설명

1. CPM은 요리 순서표처럼, 밥 짓기(20분)→반찬 만들기(15분)→상 차리기(5분)를 그려보고 어떤 순서로 해야 가장 빨리 완성되는지 찾는 방법이에요.
2. 임계 경로는 가장 시간이 오래 걸리는 경로로, 밥 짓기가 늦어지면 전체 식사 준비가 늦어지는 것처럼 절대 늦으면 안 되는 단계예요.
3. Float은 여유 시간으로, "반찬은 밥보다 5분 빨리 끝나니까, 잠깐 쉬어도 돼"라고 알려주는 여유 시간 안내서예요.
