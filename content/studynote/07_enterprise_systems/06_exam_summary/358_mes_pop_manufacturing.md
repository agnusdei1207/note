## 핵심 인사이트 (3줄 요약)

> 1. **본질**: MES(Manufacturing Execution System)는 ERP의 생산 계획을 현장 실행으로 변환하고, 공장 바닥(Shop Floor)의 작업 지시·실적·품질·설비 데이터를 실시간으로 수집하여 ERP에 피드백하는 제조 실행 시스템이다.
> 2. **가치**: ERP-MES-PLC의 수직 통합이 달성되면 생산 계획의 실시간 추적, 불량 발생 시 즉각 대응, OEE(Overall Equipment Effectiveness, 설비 종합 효율) 향상으로 제조 경쟁력이 강화된다.
> 3. **판단 포인트**: MES 도입 전 공장 데이터 수집 인프라(PLC/SCADA 연계, 바코드/RFID 현장 인프라) 구축이 선행되어야 하며, POP(Point of Production) 수집 방식과 데이터 표준화가 핵심이다.

## Ⅰ. 개요 및 필요성

스마트팩토리(Smart Factory) 구현의 핵심은 ERP의 계획 세계와 공장 현장의 실행 세계를 연결하는 것이다. ERP에서 "제품 A를 1,000개 생산하라"는 작업 지시가 내려오면, MES는 이를 공장 작업자·기계·자재에게 세부 작업 지시로 분해하고, 실시간으로 진척·불량·자재 소비를 추적한다.

MESA(Manufacturing Enterprise Solutions Association) International은 MES의 11개 핵심 기능을 정의했다: ①자원 배분 관리, ②작업 일정 수립, ③생산 단위 추적, ④성과 분석, ⑤문서 관리, ⑥품질 관리, ⑦유지보수 관리, ⑧공정 관리, ⑨데이터 수집, ⑩노동 관리, ⑪작업 지시 디스패치.

POP(Point of Production)는 생산 현장의 각 공정(작업 스테이션)에서 작업 시작·완료·불량 발생을 실시간으로 기록하는 데이터 수집 포인트다.

📢 **섹션 요약 비유**: MES는 공장의 신경 시스템처럼, ERP라는 두뇌가 내린 명령을 각 손발(설비·작업자)에 전달하고 피드백을 받아 올리는 역할이다.

## Ⅱ. 아키텍처 및 핵심 원리

### 제조 IT 계층 구조 (Purdue Reference Model)

```
레벨 4 (ERP):
  생산 계획, BOM, MRP, 재무
  └── 계획 지평: 주~월

레벨 3 (MES):
  작업 지시, 생산 추적, 품질, 유지보수
  └── 계획 지평: 시간~일

레벨 2 (SCADA/DCS):
  공정 모니터링, 공정 제어
  └── 계획 지평: 초~분

레벨 1 (PLC/DCS):
  자동화 제어, 센서 신호 처리
  └── 실시간 (밀리초)

레벨 0 (현장 장비):
  센서, 액추에이터, 기계
```

### MES 핵심 기능 및 데이터 흐름

```
ERP (생산 계획·BOM)
     │ 작업 지시 다운로드
     ▼
┌─────────────────────────────┐
│           MES               │
│  ┌─────────┐ ┌───────────┐  │
│  │ 작업지시 │ │  품질 관리 │  │
│  │ 디스패치 │ │ SPC/SQC  │  │
│  └────┬────┘ └─────┬─────┘  │
│  ┌────▼────┐ ┌─────▼─────┐  │
│  │  자재   │ │  설비 관리  │  │
│  │ 추적    │ │ OEE 계산   │  │
│  └─────────┘ └───────────┘  │
└─────────┬───────────────────┘
          │ 실적 업로드
          ▼
POP 단말기 (바코드/RFID 스캔)
SCADA/PLC (자동 데이터 수집)
```

### OEE(Overall Equipment Effectiveness) 산식

```
OEE = 가용률(Availability) × 성능률(Performance) × 품질률(Quality)

가용률 = 실제 가동시간 / 계획 가동시간
성능률 = 실제 생산량 / 이론 생산량
품질률 = 양품 수 / 전체 생산 수

세계 클래스 OEE: ≥ 85%
한국 제조업 평균: 약 60~70%
```

📢 **섹션 요약 비유**: OEE는 공장 기계의 성적표처럼, 얼마나 자주 고장 없이(가용률) 빠르게(성능률) 좋은 품질(품질률)로 생산하는지를 한 숫자로 보여준다.

## Ⅲ. 비교 및 연결

### MES vs ERP vs SCADA

| 구분 | MES | ERP | SCADA |
|:---|:---|:---|:---|
| 역할 | 생산 실행 관리 | 계획·재무·HR | 공정 모니터링·제어 |
| 데이터 단위 | 로트·작업 지시 | 오더·인보이스 | 센서값·공정 파라미터 |
| 시간 지평 | 시간~일 | 일~월 | 초~분 |
| 사용자 | 생산 관리자·QA | 경영진·재무 | 공정 엔지니어 |
| 대표 솔루션 | SAP ME, Siemens Opcenter | SAP ERP | Wonderware, Ignition |

### Industry 4.0과 MES 진화

- **클라우드 MES**: On-premise → SaaS MES (Microsoft Azure IoT + MES)
- **디지털 트윈**: 공장 물리 환경을 디지털로 복제, MES 데이터 활용
- **AI/ML 연계**: 예지 보전(Predictive Maintenance), 실시간 품질 예측

📢 **섹션 요약 비유**: MES와 ERP의 관계는 건물 설계도(ERP)와 현장 시공(MES)의 관계처럼, 계획을 실행으로 연결하는 다리 역할이다.

## Ⅳ. 실무 적용 및 기술사 판단

### MES 도입 성공 요인

1. **공장 현장 데이터 인프라**: PLC 연계, 바코드/RFID 태그, 네트워크 구축 선행
2. **BOM/라우팅 표준화**: ERP와 MES 간 BOM·라우팅 데이터 일치성 확보
3. **작업자 교육**: POP 단말기 사용, 불량 신고 절차 교육 필수
4. **ERP-MES 인터페이스**: SAP PI/PO, 표준 API(ISA-95) 기반 연계 설계
5. **KPI 설정**: OEE, 납기 준수율, 불량율, 작업 지시 완료율

### 의사결정 포인트

- **MES 도입 우선순위**: OEE 개선 필요 공정 → 품질 불량 多 공정 → 추적성(Traceability) 요구 공정
- **자동 수집 vs 수동 입력**: 자동 센서 수집은 정확하지만 초기 비용 高, 수동 입력은 저비용이지만 정확도 낮음
- **패키지 vs 자체 개발**: 다품종 소량 생산은 패키지 MES, 특수 공정은 자체 개발 검토

📢 **섹션 요약 비유**: MES 도입의 데이터 인프라 선행은 수도관 공사처럼, 배관(네트워크·센서)이 없으면 아무리 좋은 수도꼭지(MES 소프트웨어)도 물이 안 나온다.

## Ⅴ. 기대효과 및 결론

MES 도입 기업은 평균 ①OEE 5~15% 향상, ②불량율 20~50% 감소, ③재공품(WIP) 재고 20~30% 감소, ④납기 준수율 10~20% 향상 등의 효과를 거둔다. 자동차·반도체·의약품 제조업에서 MES는 법적 추적성(Traceability) 요건 충족을 위해 필수 도입 시스템이 되었다.

**한계**: MES 구축은 공장 현장 수백~수천 개의 데이터 포인트와 연계가 필요하여 구현 복잡도가 높고, 현장 작업자의 사용성 확보(UX)가 도입 성패를 좌우한다.

📢 **섹션 요약 비유**: MES는 교통 신호 시스템처럼, 공장의 수백 개 기계와 작업자가 충돌 없이 최적 속도로 움직이도록 조율하는 지능형 관제 시스템이다.

### 👶 어린이를 위한 3줄 비유 설명

1. MES는 학교 수업 관리 시스템처럼, 어느 교실에서 무슨 수업을 하는지(작업 지시), 몇 명이 참석했는지(실적), 누가 결석했는지(불량) 실시간으로 알려줘요.
2. 교장 선생님(ERP)이 "이번 주 수학 100시간 가르쳐"라고 지시하면, MES가 각 반(공정)에 나눠 배정하고 결과를 보고해요.
3. 이 덕분에 교장 선생님은 모든 교실을 직접 돌아다니지 않아도 전체 수업 현황을 한눈에 볼 수 있어요.
