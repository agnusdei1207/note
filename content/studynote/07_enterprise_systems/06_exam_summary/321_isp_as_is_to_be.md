## 핵심 인사이트 (3줄 요약)

> 1. **본질**: ISP (Information Strategy Planning)는 조직의 현재 정보화 수준(AS-IS)을 진단하고 목표 상태(TO-BE)를 설계한 뒤 단계적 이행 계획으로 연결하는 IT 전략 수립 방법론이다.
> 2. **가치**: 비즈니스 목표와 IT 역량을 정렬함으로써 IT 투자 ROI를 평균 1.5~2.5배 향상시키고 중복·비효율 시스템을 40% 이상 감축한다.
> 3. **판단 포인트**: ISP 성공은 AS-IS 분석의 객관성(현업 인터뷰 + 데이터 기반), TO-BE 설계의 현실성(6개월 Quick Win 포함), 이행 로드맵의 우선순위 체계화에 달려 있다.

## Ⅰ. 개요 및 필요성

ISP (Information Strategy Planning)는 기업·공공기관이 중장기 정보화 전략을 수립하기 위해 수행하는 체계적 컨설팅 방법론이다. 1990년대 McKinsey와 IBM이 체계화하였으며, 국내에서는 행정안전부 표준 ISP 방법론(2005, 2015 개정)으로 공공기관 도입이 의무화되었다.

ISP가 필요한 이유는 세 가지다. 첫째, 임기응변식 IT 도입은 기술 부채를 누적시키고 데이터 사일로를 심화시킨다. ISP는 3~5년의 그랜드 플랜을 제공하여 단기 투자와 장기 비전을 연결한다. 둘째, 부서별 IT 요구사항이 상충할 때 우선순위 조정 메커니즘이 없으면 예산이 분산된다. ISP의 GAP 분석은 현재-목표 역량 격차를 수치화하여 의사결정 근거를 제공한다. 셋째, EA (Enterprise Architecture)와 연계 없이 추진하는 개별 시스템 구축은 사후 유지보수 비용이 구축비의 3배에 달한다는 Gartner 연구 결과가 있다.

**ISP 3단계 프로세스**:

- **Phase 1 – 현황 분석 (AS-IS)**: 현업 인터뷰(C-레벨 포함), 설문 조사, 시스템 인벤토리 분석. 산출물: 현황 진단 보고서, 이슈 목록
- **Phase 2 – 미래 설계 (TO-BE)**: 비즈니스 목표 재정립, 아키텍처 청사진 설계. 산출물: 목표 아키텍처, 서비스 청사진
- **Phase 3 – 이행 계획 (Roadmap)**: Quick Win(6개월 내), Mid-term(1~2년), Long-term(3~5년) 과제 분류. 산출물: 과제 포트폴리오, ROI 추정서

McKinsey 7-S 모델(Strategy, Structure, Systems, Shared Values, Style, Staff, Skills)은 ISP에서 비IT 요인까지 포함한 전사적 정합성 점검에 활용된다.

📢 **섹션 요약 비유**: ISP는 집을 리모델링하기 전에 현재 구조를 정밀 측량하고(AS-IS), 새 도면을 그린 뒤(TO-BE), 방 하나씩 순서를 정해 고치는 공사 계획서(Roadmap)를 만드는 과정과 같다.

## Ⅱ. 아키텍처 및 핵심 원리

### ISP 방법론 3단계 산출물 구조

| 단계 | 활동 | 주요 도구 | 산출물 |
|:---|:---|:---|:---|
| AS-IS 분석 | 인터뷰·설문·시스템 감사 | McKinsey 7-S, SWOT | 현황 진단서, 이슈 맵 |
| TO-BE 설계 | 아키텍처 청사진, KPI 정의 | EA 프레임워크, BSC | 목표 아키텍처, 로드맵 초안 |
| 이행 계획 | 과제 도출, 우선순위화, ROI 산정 | GAP 분석, 포트폴리오 매트릭스 | 마스터 플랜, 예산 계획서 |

### ISP GAP 분석 흐름도

```
┌─────────────────────────────────────────────────────┐
│               ISP 전체 흐름                         │
├────────────┬─────────────────┬─────────────────────┤
│  AS-IS     │    GAP 분석     │      TO-BE          │
│  (현재)    │                 │     (목표)          │
│            │  ┌───────────┐  │                     │
│ 비즈니스   │  │ 격차 식별  │  │ 비즈니스 목표       │
│ 프로세스   │  │ 우선순위  │  │ 통합 IT 아키텍처    │
│ IT 시스템  ├─▶│ 과제화    ├─▶│ 디지털 역량         │
│ 조직/인력  │  │           │  │ 데이터 플랫폼       │
│ 데이터     │  └───────────┘  │                     │
└────────────┴─────────────────┴─────────────────────┘
        │                                │
        └────────────ROADMAP─────────────┘
            Quick Win │  Mid  │  Long
             (0~6M)   │(1~2Y) │ (3~5Y)
```

### 이행 과제 포트폴리오 (예시)

| 영역 | Quick Win (0~6개월) | Mid-term (1~2년) | Long-term (3~5년) |
|:---|:---|:---|:---|
| 인프라 | 클라우드 이전 POC | 하이브리드 클라우드 | 멀티클라우드 전환 |
| 데이터 | 데이터 품질 점검 | 데이터 레이크 구축 | AI/ML 플랫폼 |
| 프로세스 | BPR 파일럿 | ERP 고도화 | 스마트 자동화 |
| 보안 | 취약점 패치 | Zero Trust 설계 | SASE 전환 |

**ROI 추정**: IT 투자 ROI = (순현금흐름 현재가치 – 초기 투자) / 초기 투자 × 100%. 대형 ISP 과제의 기대 ROI는 3년 기준 150~250%, 회수 기간은 18~30개월이다.

📢 **섹션 요약 비유**: GAP 분석은 현재 내 키(AS-IS)와 롤러코스터 최소 탑승 키(TO-BE)의 차이를 재는 것이고, 로드맵은 "몇 달 동안 우유를 마시면 탈 수 있는지" 계획하는 것이다.

## Ⅲ. 비교 및 연결

| 비교 항목 | ISP | EA (Enterprise Architecture) | BPR |
|:---|:---|:---|:---|
| 목적 | IT 전략 수립 | IT-비즈니스 정렬 아키텍처 | 프로세스 근본 재설계 |
| 기간 | 3~6개월 컨설팅 | 상시 거버넌스 | 6~18개월 혁신 프로젝트 |
| 산출물 | 마스터 플랜 | 아키텍처 레파지토리 | 신규 프로세스 모델 |
| 관계 | ISP 결과가 EA 입력 | EA가 ISP 프레임 제공 | ISP 로드맵에 BPR 포함 |
| 주도 | 외부 컨설턴트 + 내부 TF | 내부 EA 조직 | 프로세스 오너 + 컨설턴트 |
| 리스크 | 실행력 부족 | 관료화·드리프트 | 변화 저항(70% 실패) |

**McKinsey 7-S ISP 활용**:
- Strategy: TO-BE 목표 설정 기준
- Structure: AS-IS 조직 분석
- Systems: 핵심 AS-IS/TO-BE IT 시스템 분석
- Shared Values: 추진 문화 기반 파악
- Style·Staff·Skills: 변화관리 계획 수립

📢 **섹션 요약 비유**: ISP, EA, BPR은 건물 신축의 "시공 전략(ISP)", "설계 도면(EA)", "기존 건물 철거 공법(BPR)"처럼 각자 역할이 다르지만 하나의 프로젝트에서 협력한다.

## Ⅳ. 실무 적용 및 기술사 판단

**ISP 성공 체크리스트**:
- [ ] CEO/CIO 스폰서십 확보 (경영진 참여율 > 80%)
- [ ] 현업 대표 TF 구성 (현업 50% + IT 30% + 외부 컨설턴트 20%)
- [ ] AS-IS 인터뷰 최소 30명 이상 (C-레벨 필수 포함)
- [ ] GAP 분석 수치화 (정성 의견 → 성숙도 점수 1~5점)
- [ ] Quick Win 과제 3개 이상 6개월 내 가시적 성과 보장
- [ ] 로드맵 연간 리뷰 및 갱신 체계 수립

**안티패턴**:
1. **보고서 ISP**: 문서만 작성하고 실행 조직·예산·KPI 없이 방치 → 실행률 20% 미만
2. **기술 편향 ISP**: 비즈니스 전략 없이 최신 기술 도입 로드맵만 나열
3. **현업 배제 ISP**: IT 부서 단독 수행 → 이행 후 사용률 저조
4. **단발성 ISP**: 3~5년 주기 리뷰 없이 초기 계획 고수 → 환경 변화 대응 불가

**ROI 추정 예시 (대형 공공기관)**:
- 초기 컨설팅 비용: 5억 원
- 중복 시스템 통합 절감: 연 8억 원
- 프로세스 자동화 절감: 연 12억 원
- 3년 누적 ROI: (60억 – 5억) / 5억 = **1,100%**

📢 **섹션 요약 비유**: ISP 실패는 헬스장 PT 플랜만 짜고 실제로 운동하지 않는 것과 같다 — 완벽한 계획서가 실행력을 보장하지는 않는다.

## Ⅴ. 기대효과 및 결론

**기대 효과**:
- **비용 절감**: 중복 시스템 통합으로 운영비용 30~40% 절감
- **민첩성 향상**: 표준화된 데이터·서비스 계층으로 신규 서비스 출시 50% 단축
- **리스크 감소**: IT 투자 우선순위 체계화로 불필요한 프로젝트 실패 감소
- **조직 정렬**: 경영진-IT-현업 간 공통 언어·목표 공유

**한계**:
- 이행 조직 지속성 확보 어려움 (컨설팅 종료 후)
- DX 속도 가속으로 3~5년 계획의 유효기간 단축 → 연간 롤링 플랜 전환 추세
- 컨설팅 비용 규모(대기업 10~50억 원)로 중소기업 적용 어려움

**선결 조건**: ① 경영진 스폰서십 ② 내부 협조 체계 ③ ISP 결과를 예산 배분에 직접 반영하는 거버넌스 구조

📢 **섹션 요약 비유**: ISP는 마라톤 대회를 준비하는 훈련 계획서 — 완주의 성패는 계획의 완성도가 아니라 매일의 실행에 달려 있다.

### 👶 어린이를 위한 3줄 비유 설명

1. ISP는 방을 청소하기 전에 "지금 어지러운 것"(AS-IS)을 사진 찍고, "깔끔하게 만들 목표"(TO-BE)를 그린 뒤, "오늘은 책상, 내일은 옷장" 순서(Roadmap)를 정하는 것이에요.
2. GAP 분석은 지금 수학 점수(60점)와 목표 점수(90점)의 차이(30점)를 파악해서, 어떤 단원을 더 공부할지 계획하는 것과 같아요.
3. Quick Win은 큰 목표를 향해 달리면서 중간중간 작은 성공을 맛보는 것 — 마라톤 중에 에너지 음료를 마시며 기운을 얻는 것처럼요.
