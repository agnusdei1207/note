## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 데이터 가상화(Data Virtualization)는 이기종 데이터 소스를 물리적으로 이동·복제하지 않고 가상 레이어를 통해 통합 논리 뷰와 연방 쿼리(Federated Query)를 제공하는 기술이다.
> 2. **가치**: 데이터 복제 비용과 지연 없이 실시간 통합 뷰를 제공하며, 소스 데이터 위치를 사용자에게 투명하게 숨겨 데이터 접근 단순성을 극대화한다.
> 3. **판단 포인트**: 실시간성과 투명성이 강점이지만, 대용량 쿼리 시 소스 시스템 부하와 네트워크 지연이 병목이 되므로 캐싱 전략과 쿼리 최적화가 필수다.

## Ⅰ. 개요 및 필요성

기업 데이터는 RDBMS·NoSQL·데이터 레이크·SaaS API·레거시 시스템 등에 분산되어 있다. 이를 DW로 통합하려면 ETL 파이프라인 구축과 지속적 유지보수가 필요하다. 데이터 가상화는 ETL 없이 가상 레이어(Virtualization Layer)가 각 소스에 실시간으로 쿼리를 위임(Push-Down)하고 결과를 병합하여 단일 논리 뷰를 제공한다.

📢 **섹션 요약 비유**: 데이터 가상화는 배달 앱 — 실제로 식재료를 창고에 모아두지 않고, 주문이 들어오면 각 식당(소스)에서 직접 가져다 준다.

## Ⅱ. 아키텍처 및 핵심 원리

```
사용자/BI 도구
     │ SQL 쿼리
     ▼
┌────────────────────────────────────────────┐
│    Data Virtualization Layer               │
│  (Denodo, Trino, Presto, Apache Drill)     │
│   - 쿼리 파싱 및 최적화                     │
│   - Push-Down 쿼리 분해                    │
│   - 결과 병합 및 캐싱                       │
└────┬──────────┬──────────┬─────────────────┘
     │          │          │
     ▼          ▼          ▼
 Oracle DB   REST API   S3 / Parquet
 (주문)     (SaaS CRM)  (로그 데이터)
```

| 특성 | 물리적 통합(ETL) | 데이터 가상화 |
|:---|:---|:---|
| 데이터 이동 | 필수 | 불필요 |
| 실시간성 | 배치 지연 | 실시간 |
| 소스 부하 | 낮음(적재 후) | 쿼리 시 발생 |
| 구현 속도 | 느림 | 빠름 |
| 대용량 성능 | 높음 | 제한적 |

📢 **섹션 요약 비유**: Push-Down은 쿼리의 다이어트 — 할 수 있는 작업은 소스 가까이에서 처리하여 네트워크로 오가는 데이터를 최소화한다.

## Ⅲ. 비교 및 연결

Trino(구 PrestoSQL)는 클라우드 환경의 대표적 연방 쿼리 엔진으로, S3·Hive·Kafka·RDBMS를 단일 SQL로 쿼리한다. Denodo는 엔터프라이즈급 데이터 가상화 플랫폼으로 캐싱과 거버넌스 기능이 강화되어 있다.

📢 **섹션 요약 비유**: Trino는 만능 번역가 — 어떤 언어(DB 형식)로든 SQL 한 문장으로 대화할 수 있게 해준다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- 소스 데이터 이동 불가(규제, 계약): 가상화 필수
- 실시간 통합 뷰 필요: 가상화 선택
- 대용량 배치 분석: 물리적 ETL/ELT 병용 권장
- 성능 최적화: Materialized View 캐싱, 파티션 프루닝, Push-Down 최대화

📢 **섹션 요약 비유**: 가상화는 패스트푸드, ETL은 정식 요리 — 패스트푸드는 빠르지만 대량 주문엔 정식 주방이 더 효율적이다.

## Ⅴ. 기대효과 및 결론

데이터 가상화는 ETL 파이프라인 없이 빠른 통합을 실현하여 시간 가치(Time-to-Value)를 극대화한다. 특히 규제로 데이터 이동이 제한되거나 실시간 통합이 필요한 환경에서 강력한 도구다. 대용량 분석 성능이 물리적 통합에 비해 제한적이므로, 자주 사용하는 데이터의 캐싱과 물리적 통합의 전략적 조합이 최적의 아키텍처를 만든다.

📢 **섹션 요약 비유**: 데이터 가상화는 도서관 전자 목록 — 책을 직접 들고 오지 않아도 어디 있는지 알고 내용을 미리 볼 수 있다.

### 📌 관련 개념 맵

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| Federated Query | 핵심 기능 | 분산 소스 동시 단일 SQL 쿼리 |
| Push-Down | 최적화 기술 | 쿼리 일부를 소스 DB에서 처리 |
| Trino (Presto) | 구현 엔진 | 오픈소스 분산 SQL 쿼리 엔진 |
| Denodo | 엔터프라이즈 플랫폼 | 데이터 가상화 전용 솔루션 |
| Data Fabric | 상위 개념 | 가상화를 포함한 통합 아키텍처 |

### 👶 어린이를 위한 3줄 비유 설명

1. 데이터 가상화는 마법 유리 — 여러 방(DB)에 있는 물건을 한 유리창(가상 뷰)을 통해 한 번에 볼 수 있어.
2. 물건을 방에서 가져올(ETL) 필요 없이, 보고 싶을 때만 가져다 봐 — 항상 가장 최신 모습이야.
3. 하지만 방이 너무 멀면(소스 부하, 네트워크) 유리창에 나타나는 게 좀 느릴 수도 있어.
