## 핵심 인사이트 (3줄 요약)

> 1. **본질**: BSC (Balanced Scorecard)는 Kaplan & Norton(1992)이 개발한 전략 성과 관리 프레임워크로, 재무(Financial)·고객(Customer)·내부프로세스(Internal Process)·학습성장(Learning & Growth) 4관점의 KPI를 균형 있게 관리한다.
> 2. **가치**: 단기 재무 성과에만 집중하는 기존 성과 관리의 한계를 극복하여 Strategy Map(전략 인과관계 지도)을 통해 장기적 가치 창출 경로를 가시화한다.
> 3. **판단 포인트**: 각 관점의 KPI는 인과관계(cause-effect linkage)로 연결되어야 하며, 학습성장 관점이 내부프로세스→고객→재무 성과로 이어지는 선행 지표 역할을 한다.

## Ⅰ. 개요 및 필요성

BSC (Balanced Scorecard)는 Robert Kaplan (Harvard Business School)과 David Norton이 1992년 Harvard Business Review에 "The Balanced Scorecard — Measures That Drive Performance"를 발표하면서 전략 관리의 혁명을 가져왔다.

**탄생 배경**: 1990년대 기업들은 재무 지표(EPS, ROE, 매출액)만으로 성과를 관리했으나, 이는 후행 지표 편향으로 문제를 사후에 인지하고 혁신·고객·역량 같은 무형 자산을 무시했다. Kaplan&Norton은 재무적 성과가 비재무적 활동의 결과임을 Strategy Map의 인과관계로 증명하고 4관점 균형 관리를 제시했다.

**4관점 요약**:
- **재무(Financial)**: 주주에게 어떻게 보일 것인가? (수익성, 성장성, 현금흐름)
- **고객(Customer)**: 고객에게 어떻게 보일 것인가? (만족도, 충성도, 시장 점유율)
- **내부 프로세스(Internal Process)**: 어떤 프로세스를 탁월하게 수행해야 하는가?
- **학습과 성장(Learning & Growth)**: 혁신과 가치 창출 능력을 어떻게 유지·개선할 것인가?

**IT BSC**: IT 조직 특화 BSC로, 재무 기여→비즈니스 고객→IT 프로세스→IT 역량 4관점으로 재구성하여 IT 부서의 전략 기여도를 측정한다.

📢 **섹션 요약 비유**: BSC는 자동차 계기판처럼 속도(재무), 연료(고객), 엔진 온도(프로세스), 배터리(학습성장) — 한 계기만 보면 위험하고, 네 가지를 동시에 봐야 안전하다.

## Ⅱ. 아키텍처 및 핵심 원리

### BSC 4관점 KPI 구조 (관점별 5개, 총 20개)

| 관점 | 핵심 질문 | 대표 KPI 5개 |
|:---|:---|:---|
| 재무 | 재무 성과가 충족되는가? | 매출 성장률, 영업이익률, ROI, EVA, 현금 전환 주기(CCC) |
| 고객 | 고객이 만족하는가? | NPS, CSAT, 고객 유지율, 시장 점유율, CAC |
| 내부 프로세스 | 핵심 프로세스가 탁월한가? | 사이클 타임, 불량률, 신제품 출시 기간, 주문 이행률, SLA 준수율 |
| 학습과 성장 | 역량·문화·기술이 성장하는가? | 직원 만족도, 교육 시간/인당, 핵심 인재 유지율, IT 역량 성숙도, 혁신 제안 건수 |

### BSC Strategy Map (인과관계 흐름)

```
┌─────────────────────────────────────────────┐
│       BSC Strategy Map (인과관계)            │
│                                             │
│  ┌─────────────────────────────────────┐    │
│  │ 재무 관점  매출 성장 ← 수익성 향상   │    │
│  └──────────────────┬──────────────────┘    │
│                     │ ← 결과                 │
│  ┌──────────────────┴──────────────────┐    │
│  │ 고객 관점  고객 만족 → 고객 유지    │    │
│  └──────────────────┬──────────────────┘    │
│                     │ ← 결과                 │
│  ┌──────────────────┴──────────────────┐    │
│  │ 프로세스   품질 탁월 → 빠른 납기    │    │
│  └──────────────────┬──────────────────┘    │
│                     │ ← 결과                 │
│  ┌──────────────────┴──────────────────┐    │
│  │ 학습성장   역량 향상 → 직원 몰입    │    │
│  └─────────────────────────────────────┘    │
│               ↑ 원인                         │
└─────────────────────────────────────────────┘
```

**BSC 구현 5단계**:
1. 비전/전략 명확화
2. 4관점별 전략 목표 도출
3. Strategy Map 작성 → 인과관계 검증
4. KPI 및 Target 설정 (관점별 5개, 총 20~25개)
5. 이니셔티브(실행 과제) 연계

📢 **섹션 요약 비유**: BSC Strategy Map은 체인처럼 연결된 도미노다 — 직원 역량(학습) → 좋은 프로세스 → 고객 만족 → 재무 성과로 이어지는 연쇄 반응이다.

## Ⅲ. 비교 및 연결

| 비교 항목 | BSC | MBO | OKR | KPI 단독 |
|:---|:---|:---|:---|:---|
| 관점 수 | 4관점 균형 | 재무 중심 | 도전 목표 | 단일/다수 |
| 인과관계 | Strategy Map 있음 | 없음 | 약함 | 없음 |
| 보상 연계 | 중간 | 강함 | 약함 | 강함 |
| 전략 정렬 | 매우 강함 | 보통 | 강함 | 보통 |
| 갱신 주기 | 연간 (분기 리뷰) | 연간 | 분기 | 월/분기 |
| 적합 조직 | 중대형 기업, 공공기관 | 전통 제조업 | 스타트업, 테크 기업 | 모든 규모 |

**IT BSC 4관점 KPI 예시**:

| IT BSC 관점 | 대표 KPI |
|:---|:---|
| 재무 기여 | IT 비용/매출 비율, IT 투자 ROI |
| 비즈니스 고객 | IT 서비스 만족도, SLA 준수율 |
| IT 프로세스 | MTTR (Mean Time To Repair), 변경 성공률 |
| IT 역량 | IT 직원 인증 보유율, 기술 부채 감소율 |

**BSC Cascade (단계적 분해)**: 전사 BSC → 부서 BSC → 팀 KPI → 개인 KPI

📢 **섹션 요약 비유**: BSC Cascade는 회사 목표를 팀장, 팀원 순서로 이어받는 릴레이 바통처럼 — 모든 사람이 같은 방향을 향해 뛰어야 팀이 이긴다.

## Ⅳ. 실무 적용 및 기술사 판단

**BSC 도입 체크리스트**:
- [ ] 경영진이 Strategy Map 작성에 직접 참여하는가?
- [ ] 4관점 KPI 간 인과관계가 검증되었는가?
- [ ] 관점별 KPI 수가 5개 이내로 집중되어 있는가?
- [ ] 각 KPI에 데이터 수집 책임자와 기준이 정의되어 있는가?
- [ ] 분기별 BSC 리뷰 회의가 정례화되어 있는가?
- [ ] 부서/개인 BSC로의 Cascade 계획이 있는가?

**안티패턴**:
1. **KPI 과잉**: 관점당 10개 이상 KPI → "측정을 위한 측정"으로 전락
2. **재무 관점 편중**: 학습성장·프로세스 KPI를 형식적으로 채움
3. **인과관계 미검증**: Strategy Map 없이 KPI만 나열 → BSC의 핵심 가치 상실
4. **과잉 보상 연계**: KPI 100% 달성이 보너스 기준 → KPI 조작(Gaming) 유인 발생

**실제 적용 사례 (글로벌 제조 기업)**:
- 재무: 영업이익률 > 15%, ROIC > 12%
- 고객: NPS > 50, 고객 유지율 > 90%
- 프로세스: 신제품 출시 < 18개월, 불량률 < 100ppm
- 학습성장: R&D/매출 > 8%, 핵심 인재 유지율 > 95%

📢 **섹션 요약 비유**: BSC를 제대로 운영하는 것은 4기통 엔진을 균형 있게 조율하는 것 — 재무 한 실린더만 무리하게 돌리면 엔진이 망가진다.

## Ⅴ. 기대효과 및 결론

**기대 효과**:
- **전략 정렬**: 전사 전략이 현장까지 연결되어 전략 실행률 평균 40% 향상 (Kaplan&Norton 조사)
- **장기 성과**: 재무 후행 지표뿐 아니라 선행 비재무 지표로 미래 성과 예측
- **커뮤니케이션**: Strategy Map으로 전략을 한눈에 공유
- **균형 성장**: 단기 재무 압박으로 장기 역량 훼손 방지

**한계**:
- 구현 복잡성: Strategy Map 완성까지 3~6개월 소요
- 변화 저항: 새 KPI 도입에 대한 현업 저항
- 인과관계 가정의 불확실성: 학습→재무 연결에 수년의 시차 발생

**선결 조건**: ① CEO 직접 스폰서십 ② BI/Analytics 인프라 ③ BSC 전담 PMO (Project Management Office)

📢 **섹션 요약 비유**: BSC의 진짜 가치는 스코어카드 자체가 아니라 그것을 만들기 위해 경영진이 "우리 전략의 인과관계가 정말 맞는가?"를 함께 토론하는 과정에 있다.

### 👶 어린이를 위한 3줄 비유 설명

1. BSC는 건강 성적표처럼 체중(재무), 체력(프로세스), 친구 관계(고객), 공부 습관(학습)을 모두 함께 재는 것이에요 — 체중만 보면 진짜 건강한지 알 수 없어요.
2. Strategy Map은 "매일 운동하면(학습성장) → 체력이 좋아지고(프로세스) → 친구들과 잘 어울리고(고객) → 행복해진다(재무)"처럼 원인과 결과를 그린 그림이에요.
3. BSC를 잘 쓰는 회사는 팀원 모두가 "내가 오늘 한 일이 회사 목표에 어떻게 연결되는지"를 아는 회사예요.
