## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 데이터 스튜어드(Data Steward)는 IT가 아닌 비즈니스 도메인 담당자가 데이터 품질·정의·생명주기의 1차 책임자가 되는 역할이다.
> 2. **가치**: 데이터 거버넌스(Data Governance) 체계에서 CDO(Chief Data Officer)의 정책을 현장에 실행하며, 중복·오류 데이터로 인한 의사결정 오류를 사전 차단한다.
> 3. **판단 포인트**: 스튜어드 지정 범위(도메인별 vs 기능별), 품질 KPI 설정, IT와 비즈니스 간 책임 경계 명확화가 핵심이다.

## Ⅰ. 개요 및 필요성

데이터가 기업의 핵심 자산으로 부상하면서, 데이터 품질(Data Quality)은 IT 부서만의 문제가 아닌 전사 비즈니스 과제가 되었다. 데이터 스튜어드(Data Steward)는 특정 데이터 도메인(고객, 제품, 재무 등)에 대해 **비즈니스 담당자**가 데이터 소유권(Ownership)을 갖고 품질 규칙, 메타데이터(Metadata), 접근 정책을 정의·관리하는 역할이다.

데이터 거버넌스 체계가 없으면 동일 고객 데이터가 CRM, ERP, 마케팅 플랫폼에 각각 다르게 저장되어 360° 고객 뷰가 불가능해진다. 스튜어드는 이러한 데이터 사일로(Data Silo)를 제거하고, 데이터 사전(Data Dictionary)과 메타데이터 카탈로그(Metadata Catalog)를 유지함으로써 전사 데이터 신뢰성을 보장한다.

| 역할 | 책임 범위 | 예시 |
|:---|:---|:---|
| CDO (Chief Data Officer) | 전사 데이터 전략·정책 수립 | 데이터 거버넌스 프레임워크 승인 |
| 데이터 오너 (Data Owner) | 특정 데이터 도메인의 최종 책임자 | 영업 본부장 → 고객 데이터 오너 |
| 데이터 스튜어드 (Data Steward) | 데이터 정의·품질 규칙·메타데이터 관리 | 영업기획팀 → 고객 마스터 스튜어드 |
| 데이터 쿠스토디언 (Data Custodian) | 기술적 데이터 저장·보안·백업 담당 | DBA, 인프라 팀 |

📢 **섹션 요약 비유**: 데이터 스튜어드는 도서관의 사서처럼 책(데이터)의 분류·위치·상태를 책임지는 사람이다.

## Ⅱ. 아키텍처 및 핵심 원리

### 데이터 거버넌스 조직 계층

```
┌─────────────────────────────────────────────────────┐
│                 Data Governance Council               │
│              (CDO + C-Suite 의사결정)                 │
├─────────────────────────────────────────────────────┤
│              Data Stewardship Committee               │
│         (도메인 오너 + 리드 스튜어드 정책 조율)        │
├───────────────┬───────────────┬─────────────────────┤
│  Customer     │   Product     │   Finance           │
│  Steward      │   Steward     │   Steward           │
│  (고객 도메인) │  (제품 도메인) │  (재무 도메인)       │
└───────────────┴───────────────┴─────────────────────┘
         ↓              ↓              ↓
     메타데이터      품질 규칙       데이터 사전
     카탈로그        정의            유지보수
```

### 데이터 스튜어드 핵심 활동

| 활동 영역 | 세부 내용 | 산출물 |
|:---|:---|:---|
| 데이터 정의 | 비즈니스 용어 표준화, 데이터 사전 작성 | Business Glossary |
| 품질 규칙 | 완전성·정확성·일관성·적시성 기준 수립 | DQ Rule Set |
| 메타데이터 관리 | 데이터 계보(Lineage), 출처, 스키마 관리 | Data Catalog |
| 접근 정책 | 개인정보 분류, 마스킹 규칙, 접근 수준 정의 | Data Classification |
| 이슈 해결 | 데이터 품질 이슈 트리아지(Triage) 및 조율 | Issue Log |

### 데이터 품질 6대 차원

```
┌──────────────────────────────────────────────────────┐
│               데이터 품질 측정 차원                    │
├──────────┬───────────────────────────────────────────┤
│ 완전성   │ 필수 필드 누락 비율 (Null Rate < 1%)        │
│ 정확성   │ 실제 값과의 일치도 (오류율 < 0.5%)          │
│ 일관성   │ 시스템 간 동일 데이터 동일 값               │
│ 적시성   │ 데이터 최신성 (업데이트 주기 준수)          │
│ 유일성   │ 중복 레코드 비율 (중복율 < 0.1%)            │
│ 유효성   │ 비즈니스 규칙 부합 여부                     │
└──────────┴───────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 품질 6대 차원은 식재료 검수처럼 신선도(적시성)·무게(정확성)·라벨(완전성)·출처(유효성)를 모두 확인하는 것이다.

## Ⅲ. 비교 및 연결

### 데이터 스튜어드 vs 데이터 엔지니어

| 구분 | 데이터 스튜어드 | 데이터 엔지니어 |
|:---|:---|:---|
| 배경 | 비즈니스 도메인 전문가 | IT/기술 전문가 |
| 관심사 | 데이터 의미, 품질 정책 | 파이프라인, 스키마 구현 |
| 산출물 | Business Glossary, DQ Rule | ETL 파이프라인, 데이터 모델 |
| 권한 | 품질 기준 승인, 정책 결정 | 기술적 구현 실행 |

### 데이터 스튜어드십과 DAMA-DMBOK 연계

DAMA(Data Management Association)의 DMBOK(Data Management Body of Knowledge)은 데이터 거버넌스를 11개 지식 영역으로 구분한다. 데이터 스튜어드는 이 중 데이터 품질 관리(Data Quality Management), 메타데이터 관리(Metadata Management), 마스터 데이터 관리(Master Data Management) 영역에서 핵심 역할을 수행한다.

📢 **섹션 요약 비유**: 스튜어드와 엔지니어의 관계는 식당 메뉴 설계자(요리 콘셉트)와 주방장(실제 조리)의 관계와 같다.

## Ⅳ. 실무 적용 및 기술사 판단

### 데이터 스튜어십 프로그램 도입 로드맵

```
1단계 (착수)     2단계 (확립)    3단계 (성숙)    4단계 (최적화)
    │               │               │               │
현황 진단       스튜어드 지정   품질 KPI 운영   자동화/AI 적용
데이터 도메인   Business       품질 이슈       ML 기반 품질
목록화          Glossary 작성  트리아지 체계   이상 탐지
                               운영
```

### 의사결정 포인트

1. **스튜어드 지정 방식**: 도메인별(고객/제품/재무) 지정이 기능별(마케팅/영업) 지정보다 데이터 중심 조직에 적합
2. **품질 KPI 설정**: 완전성·정확성 등 측정 가능한 지표로 설정하고 월별 대시보드 운영
3. **인센티브 설계**: 스튜어드 활동을 성과 평가에 반영하지 않으면 형식적 운영에 그침
4. **도구 선택**: Collibra, Alation, Informatica IDQ 등 Data Catalog 도구와 연계

📢 **섹션 요약 비유**: 스튜어십 도입은 집 청소처럼 먼저 방 구분(도메인)을 정하고, 청소 담당자(스튜어드)를 지정한 뒤, 청소 기준(품질 KPI)을 만드는 순서로 진행된다.

## Ⅴ. 기대효과 및 결론

데이터 스튜어드 체계를 도입하면 ①데이터 품질 오류로 인한 의사결정 오류 감소, ②규제 컴플라이언스(GDPR, 개인정보보호법) 대응 강화, ③MDM·분석 플랫폼 투자 효과 극대화 등의 효과를 기대할 수 있다. Gartner에 따르면 데이터 품질 문제로 인한 연간 비용은 기업 평균 1,290만 달러에 달한다.

**한계/전제조건**: 스튜어십은 조직 문화 변화를 수반하므로, 경영진의 강력한 후원(Executive Sponsorship)과 스튜어드 역량 강화 교육이 선행되어야 한다. 기술 도구만으로는 효과를 거두기 어렵다.

📢 **섹션 요약 비유**: 데이터 스튜어드는 깨끗한 수돗물처럼, 한 번 정수(품질 확보) 체계를 구축하면 모든 가정(시스템)에서 안전한 물(데이터)을 쓸 수 있게 한다.

### 👶 어린이를 위한 3줄 비유 설명

1. 데이터 스튜어드는 학교 도서관의 사서 선생님처럼, 책(데이터)이 제자리에 있고 낡지 않았는지 확인하는 사람이에요.
2. 스튜어드가 없으면 같은 학생의 이름이 어떤 장부엔 "김민수", 어떤 장부엔 "KIM MINSU"로 달리 적혀서 선생님이 혼란스러워져요.
3. 스튜어드는 "이 데이터는 이렇게 쓰면 돼요"라는 규칙을 만들어서 모든 팀이 같은 언어로 데이터를 이야기할 수 있게 해줘요.
