## 핵심 인사이트 (3줄 요약)

> 1. **본질**: W.H. Inmon의 데이터 웨어하우스(Data Warehouse)는 기업 전체의 데이터를 주제 지향(Subject-oriented)·통합(Integrated)·시간 변이(Time-variant)·비휘발성(Non-volatile) 4대 특성으로 정의하며, 3NF 정규화 기반 EDW(Enterprise Data Warehouse)를 구축하는 하향식(Top-down) 접근법이다.
> 2. **가치**: 전사 데이터의 단일 진실 원천(Single Source of Truth)을 먼저 구축하고 그로부터 부서별 데이터 마트(Data Mart)를 파생시킴으로써 데이터 일관성과 거버넌스를 최우선으로 달성한다.
> 3. **판단 포인트**: Inmon 방식은 EDW 구축에 시간·비용이 많이 드는 대신 데이터 일관성과 유연성이 높으며, Kimball(상향식, 스타 스키마)과 비교하여 프로젝트 규모·기간·팀 역량에 따라 선택한다.

## Ⅰ. 개요 및 필요성

1990년 Bill Inmon이 "Building the Data Warehouse"에서 데이터 웨어하우스의 개념을 처음 공식화했다. 기업의 의사결정 지원 시스템(DSS)에서 온라인 트랜잭션 처리(OLTP)와 분석 처리(OLAP)를 분리해야 한다는 필요성에서 출발했다.

OLTP 시스템에서 직접 분석 쿼리를 실행하면 ①트랜잭션 성능 저하, ②이력 데이터 보관 어려움, ③이종 시스템 데이터 통합 불가 등의 문제가 발생한다. 데이터 웨어하우스는 이를 해결하는 분석 전용 통합 데이터 저장소다.

| Inmon DW 4대 특성 | 설명 |
|:---|:---|
| 주제 지향 (Subject-oriented) | 업무 기능이 아닌 비즈니스 주제(고객, 제품, 판매) 중심으로 구성 |
| 통합 (Integrated) | 이종 소스 시스템의 데이터 코드·형식 표준화 통합 |
| 시간 변이 (Time-variant) | 5~10년 이상 장기 이력 데이터 보관 (OLTP: 최신 데이터만) |
| 비휘발성 (Non-volatile) | 데이터 삭제·수정 없음, 추가(Append) 전용 |

📢 **섹션 요약 비유**: 데이터 웨어하우스는 도서관처럼, 여러 출판사(소스 시스템)의 책을 표준 분류 체계(통합 스키마)로 정리해두고, 오래된 책도 폐기하지 않고(비휘발성) 보관하는 공간이다.

## Ⅱ. 아키텍처 및 핵심 원리

### Inmon 하향식 아키텍처

```
소스 시스템 (OLTP)
  [ERP] [CRM] [SCM] [MES]
         │
         ▼ ETL (추출·변환·적재)
    ┌────────────────────┐
    │  EDW (Enterprise   │
    │  Data Warehouse)   │
    │  3NF 정규화        │
    │  단일 진실 원천    │
    └─────────┬──────────┘
              │ ETL/API
    ┌─────────┼─────────┐
    ▼         ▼         ▼
  [영업      [마케팅   [재무
  Data Mart] Data Mart] Data Mart]
  (Star Schema 비정규화)
              │
              ▼
     BI 도구 / 대시보드
```

### 3NF vs 스타 스키마 비교

| 구분 | 3NF (정규화) — Inmon EDW | 스타 스키마 — Kimball DM |
|:---|:---|:---|
| 목적 | 데이터 중복 최소화, 일관성 | 분석 쿼리 성능 최적화 |
| 구조 | 다수 테이블, 복잡 JOIN | 팩트 + 차원 테이블 단순 구조 |
| 쿼리 성능 | 낮음 (JOIN 多) | 높음 (단순 JOIN) |
| 유연성 | 높음 (새 주제 추가 용이) | 낮음 (스키마 변경 어려움) |
| 구축 시간 | 길다 | 짧다 |

### ETL vs ELT 패러다임

```
ETL (Extract→Transform→Load):
  데이터를 소스에서 추출 → 외부 변환 → DW 적재
  적합: 온프레미스 DW, 구조화 데이터

ELT (Extract→Load→Transform):
  원시 데이터 직접 DW/클라우드 적재 → DW 내부 변환
  적합: 클라우드 DW (BigQuery, Snowflake, Redshift)
  장점: 변환 로직 변경이 쉽고, 원시 데이터 보관
```

📢 **섹션 요약 비유**: Inmon 3NF EDW는 도서관 사서처럼, 모든 책을 표준 분류 체계로 정리해야 하므로 처음 구축에 시간이 걸리지만 나중에 어떤 책이든 찾기 쉽다.

## Ⅲ. 비교 및 연결

### Inmon vs Kimball 방법론 비교

| 구분 | Inmon (Top-down) | Kimball (Bottom-up) |
|:---|:---|:---|
| 접근 방식 | EDW 먼저 → Data Mart 파생 | Data Mart 먼저 → Bus Matrix 통합 |
| 데이터 모델 | 3NF (정규화) | 스타/눈송이 스키마 (비정규화) |
| 구축 기간 | 길다 (1~2년) | 짧다 (수개월) |
| 비즈니스 가치 | 느리게 실현 | 빠르게 실현 |
| 일관성 | 높음 | 버스 매트릭스로 일관성 유지 |
| 적합 기업 | 대형 기업, IT 역량 강한 조직 | 중소기업, 빠른 BI 필요 조직 |

### 현대 클라우드 DW

| 플랫폼 | 특징 |
|:---|:---|
| Snowflake | 스토리지·컴퓨트 분리, 멀티 클라우드 |
| Google BigQuery | 서버리스, SQL, 페타바이트 확장 |
| Amazon Redshift | 열 지향 스토리지, AWS 통합 |
| Azure Synapse | Spark+SQL 통합, Azure 생태계 |

📢 **섹션 요약 비유**: Inmon과 Kimball의 차이는 집 건설처럼, 설계도 완성 후 착공(Inmon, 체계적)과 방 하나씩 먼저 짓고 나중에 연결(Kimball, 빠른 가치)의 차이다.

## Ⅳ. 실무 적용 및 기술사 판단

### DW 구축 방법론 선택 기준

```
Inmon EDW 선택:
  ├── 전사 단일 진실 원천이 최우선
  ├── 장기(2~5년) 투자 가능
  └── 강한 데이터 거버넌스 요구

Kimball Data Mart 선택:
  ├── 빠른 BI 가치 실현 필요 (수개월)
  ├── 특정 부서 분석 요건 명확
  └── 소규모 팀, 빠른 반복 개발 선호

하이브리드 (혼합):
  ├── Data Lakehouse (Delta Lake, Iceberg)
  └── EDW + 부서 Data Mart 혼용
```

### 현대적 DW: 데이터 레이크하우스(Data Lakehouse)

Data Lake(원시 데이터 저장)와 Data Warehouse(구조화 분석)를 통합한 아키텍처. Delta Lake(Databricks), Apache Iceberg가 대표 구현이며 클라우드 오브젝트 스토리지(S3) 위에 ACID 트랜잭션·스키마 진화 기능을 추가한다.

📢 **섹션 요약 비유**: Data Lakehouse는 도서관+창고처럼, 정리된 도서관(DW)의 분석 편의성과 창고(Data Lake)의 방대한 원시 데이터 보관을 동시에 제공한다.

## Ⅴ. 기대효과 및 결론

Inmon EDW 구축으로 ①전사 데이터 단일 진실 원천 확보, ②이종 소스 데이터 통합 및 표준화, ③장기 이력 데이터 분석 가능(5~10년+), ④데이터 거버넌스 강화 등의 효과를 거둔다. 금융·공공 분야처럼 데이터 일관성과 감사 추적이 중요한 조직에서 Inmon EDW는 여전히 핵심 데이터 인프라다.

**한계**: Inmon EDW는 구축 기간이 길고 초기 비용이 높아 빠른 BI 가치 실현을 요구하는 경영진을 설득하기 어렵다. 클라우드 시대에는 서버리스 클라우드 DW(BigQuery, Snowflake)가 대안으로 빠른 구축·탄력적 확장을 제공한다.

📢 **섹션 요약 비유**: Inmon EDW 구축은 국가 지도 제작처럼, 처음 만드는 데 시간이 오래 걸리지만 한 번 완성되면 모든 지역(부서)이 동일한 기준으로 활용할 수 있다.

### 👶 어린이를 위한 3줄 비유 설명

1. 데이터 웨어하우스는 학교 도서관처럼, 모든 반(시스템)의 책(데이터)을 한 곳에 모아 누구나 찾아볼 수 있게 정리한 공간이에요.
2. 오래된 책도 버리지 않고(비휘발성) 날짜별로 정리해두기 때문에, "3년 전 이 시기에 어떤 책이 많이 읽혔나?"라는 질문에도 답할 수 있어요.
3. 처음에 정리(3NF 구조화)하는 데 시간이 걸리지만, 한 번 완성되면 어떤 질문에도 빠르게 답할 수 있는 진짜 지식 창고가 돼요.
