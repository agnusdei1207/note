## 핵심 인사이트 (3줄 요약)

> 1. **본질**: OKR (Objectives & Key Results)은 도전적 목표를 분기 단위로 설정하고 70% 달성을 성공으로 정의하는 반면, MBO (Management by Objectives)는 연간 달성 가능한 목표를 탑다운으로 설정하고 100% 달성을 기준으로 보상에 직접 연계한다.
> 2. **가치**: OKR은 빠른 환경 변화에 애자일하게 대응하고 조직 전체의 방향 정렬(Alignment)과 집중을 촉진하며, MBO는 명확한 책임 소재와 보상 체계를 통해 예측 가능한 성과 관리를 제공한다.
> 3. **판단 포인트**: OKR과 MBO의 핵심 차이는 보상 연계 여부와 달성 목표 수준이다 — 도전과 혁신을 원하면 OKR, 안정적 실행과 보상 명확성을 원하면 MBO를 선택한다.

## Ⅰ. 개요 및 필요성

MBO (Management by Objectives)는 Peter Drucker가 1954년 "The Practice of Management"에서 제시한 목표 관리 방법론으로, 관리자와 직원이 합의한 구체적 목표를 설정하고 연간 달성 결과를 평가·보상에 연계하는 체계다.

OKR (Objectives & Key Results)은 Andy Grove가 Intel에서 개발하고 John Doerr가 1999년 Google에 전파하면서 전 세계 테크 기업의 표준 성과 관리 프레임워크로 확산되었다. "Objective(목표): 무엇을 달성할 것인가?" × "Key Results(핵심 결과): 어떻게 달성했는지 어떻게 알 수 있는가?" 구조로 운영된다.

**OKR 기본 구조**:
- **Objective**: 3~5개, 정성적이고 도전적인 방향성 문장
- **Key Results**: Objective당 3~5개, 정량적·측정 가능한 결과 지표
- **평가 기준**: 0~1.0 스코어, 0.7 = 성공, 1.0 = 목표가 너무 쉬웠음
- **주기**: 분기(3개월), 연간 목표는 별도 Annual OKR

**CFR (Conversations, Feedback, Recognition)**: Google/Intel이 OKR과 함께 운영하는 지속적 성과 관리 방식 — 연간 평가 대신 수시 대화·피드백·인정으로 대체.

📢 **섹션 요약 비유**: MBO는 연말에 성적표를 받는 학교 시스템이고, OKR은 매 분기 스스로 목표를 세우고 70점도 잘한 것으로 인정하는 자기주도 학습 방식이다.

## Ⅱ. 아키텍처 및 핵심 원리

### OKR vs MBO 비교표

| 비교 항목 | OKR | MBO |
|:---|:---|:---|
| 창시자 | Andy Grove → John Doerr (Google) | Peter Drucker (1954) |
| 목표 설정 주기 | 분기(3개월) | 연간(12개월) |
| 방향 | Bottom-up + Top-down 정렬 | Top-down |
| 달성 기준 | 70% = 성공 | 100% = 성공 |
| 보상 연계 | 분리 (보상과 OKR 독립) | 강하게 연계 |
| 목표 수 | 3~5 Objectives × 3~5 KR | 5~10개 MBO 목표 |
| 적합 환경 | 빠른 변화, 혁신 중심 | 안정적 실행, 제조·공공 |
| 주요 활용 기업 | Google, Intel, LinkedIn, Spotify | 전통 대기업, 공공기관 |

### OKR 정렬 구조 (Alignment Tree)

```
┌──────────────────────────────────────────────┐
│           OKR 전사 정렬 구조                  │
│                                              │
│  ┌────────────────────────────────────┐      │
│  │  Company OKR                       │      │
│  │  O: 글로벌 No.1 AI 플랫폼          │      │
│  │  KR1: MAU 1억 돌파 (현재 7,200만)  │      │
│  └────────────┬───────────────────────┘      │
│               │ 정렬 (Aligned)               │
│  ┌────────────┴───────────────────────┐      │
│  │  Division OKR (Product Team)       │      │
│  │  O: AI 추천 품질 혁신              │      │
│  │  KR1: CTR 15% → 22%               │      │
│  └────────────┬───────────────────────┘      │
│               │ 정렬 (Aligned)               │
│  ┌────────────┴───────────────────────┐      │
│  │  Individual OKR                    │      │
│  │  O: 추천 알고리즘 정밀도 개선      │      │
│  │  KR1: 추천 클릭률 오차 < 5%       │      │
│  └────────────────────────────────────┘      │
└──────────────────────────────────────────────┘
```

**OKR 70% 법칙의 의미**:
- 100% 달성 → 목표가 너무 쉬웠음 (도전성 부족)
- 70% 달성 → 최적 도전 수준 (성장과 현실 균형)
- 40% 미만 → 목표 수립 방식 검토 필요

📢 **섹션 요약 비유**: OKR에서 100점을 받으면 오히려 "왜 그 목표가 그렇게 쉬웠나요?"라고 묻는다 — 70점이 A학점인 새로운 평가 문화다.

## Ⅲ. 비교 및 연결

| 구분 | OKR | MBO | BSC | KPI |
|:---|:---|:---|:---|:---|
| 시간 지평 | 분기 | 연간 | 연간(분기 리뷰) | 상시 |
| 목표 성격 | 도전적·방향적 | 달성 가능·구체적 | 균형·전략적 | 운영 지표 |
| 측정 기준 | KR (0.0~1.0) | 달성율(%) | 4관점 KPI | 단위 지표 |
| 주도권 | Bottom-up 허용 | Top-down | 경영진 | 부서 |
| 보상 연계 | 분리 원칙 | 강함 | 중간 | 강함 |
| 혁신 친화성 | 매우 높음 | 낮음 | 중간 | 낮음 |

**CFR 운영 방식**:
- C (Conversations): 주 1회 1:1 미팅 (성과·성장 대화)
- F (Feedback): 수시 360도 피드백
- R (Recognition): 팀 내 공개적 인정·칭찬

**OKR 실패 패턴**:
1. OKR을 업무 To-do List로 오해
2. OKR을 성과 평가·급여에 직접 연계
3. Objective를 정량 지표로 작성 (KR과 혼동)
4. 분기 중 OKR 미리뷰(Check-in) 없이 분기말 평가만

📢 **섹션 요약 비유**: OKR을 보상에 직접 연결하면 직원들이 도전적 목표 대신 무조건 달성할 수 있는 쉬운 목표만 세운다 — 70점이 A학점인 문화가 무너진다.

## Ⅳ. 실무 적용 및 기술사 판단

**OKR 도입 체크리스트**:
- [ ] Objective가 정성적이고 도전적인 문장인가?
- [ ] Key Results가 측정 가능한 숫자로 정의되어 있는가?
- [ ] OKR이 급여·승진 평가와 분리되어 있는가?
- [ ] 분기 중 월별 Check-in 회의가 계획되어 있는가?
- [ ] 회사 OKR → 팀 OKR → 개인 OKR이 정렬되어 있는가?
- [ ] OKR 수가 적절한가? (Objective ≤ 5개, KR ≤ 5개/Objective)

**MBO 적합 상황**:
- 안정적인 운영 환경 (제조, 물류, 공공기관)
- 성과·보상 연계가 문화적으로 중요한 조직
- 연간 예산 계획과 긴밀하게 연계된 목표 관리

**OKR 적합 상황**:
- 빠른 시장 변화 (테크, 스타트업)
- 혁신과 실험 문화를 강화하려는 조직
- 기존 Top-down 목표 설정을 Bottom-up으로 전환하려는 조직

📢 **섹션 요약 비유**: MBO는 "연봉 협상처럼 합의한 것을 100% 이행하는 계약서"이고, OKR은 "팀이 함께 더 멀리 뛰어보자는 도전 선언문"이다 — 두 문서의 목적이 다르다.

## Ⅴ. 기대효과 및 결론

**OKR 도입 기대 효과**:
- 전사 정렬 가속: 분기마다 조직 방향을 재조율 → 전략 민첩성 향상
- 도전 문화 형성: 70% 기준으로 안전한 실패·학습 환경 조성
- 투명성 향상: OKR 공개로 조직 내 목표 가시성 증가
- 집중력 향상: 분기당 Objective 3~5개로 우선순위 집중

**한계**:
- OKR 문화 정착에 2~3년 소요 (초기 혼란)
- 보상 연계 분리에 대한 직원 저항 가능성
- 정성적 Objective 측정의 주관성 문제

**선결 조건**: ① 경영진의 OKR 철학 이해 ② 심리적 안전감(Psychological Safety) 조성 ③ OKR 코치 또는 챔피언 지정

📢 **섹션 요약 비유**: OKR 도입의 가장 큰 장벽은 제도가 아니라 문화다 — "실패해도 괜찮다"는 심리적 안전감 없이 70% 목표 문화는 작동하지 않는다.

### �� 어린이를 위한 3줄 비유 설명

1. MBO는 "100m 달리기에서 10초 이내에 들어오면 상 준다"는 약속이고, OKR은 "9초를 목표로 달리되 9.5초만 나와도 훌륭하다"는 도전 선언이에요.
2. OKR에서 100점을 받으면 "목표가 너무 쉬웠네요"라는 말을 들어요 — 도전적인 목표를 세우는 것 자체가 가치 있는 일이거든요.
3. OKR이 성적표와 다른 점은 매 분기 스스로 목표를 세우고, 실패에서도 배운다는 것이에요 — 점수보다 성장이 더 중요해요.
