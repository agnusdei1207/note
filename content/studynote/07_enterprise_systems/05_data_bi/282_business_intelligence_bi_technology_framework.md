## 핵심 인사이트 (3줄 요약)

> 1. **본질**: BI (Business Intelligence, 비즈니스 인텔리전스)는 조직의 데이터를 수집·통합·분석·시각화하여 비즈니스 의사결정을 지원하는 기술·프로세스·사람의 종합 체계로, "데이터로 더 나은 결정을 내리는" 모든 활동을 포괄한다.
> 2. **가치**: BI는 단순 보고서를 넘어 셀프서비스 분석·예측 분석·임베디드 BI로 진화하여, 조직 내 모든 레벨(경영진·중간관리자·현업 담당자)이 데이터 기반 의사결정을 할 수 있는 환경을 구축한다.
> 3. **판단 포인트**: 성공적인 BI 구현은 기술(DW, OLAP, 시각화)과 함께 데이터 거버넌스·조직 문화·사용자 교육이 균형 있게 갖춰져야 하며, 기술만으로는 BI 가치를 실현할 수 없다.

## Ⅰ. 개요 및 필요성

BI는 1958년 Hans Peter Luhn이 처음 사용한 개념으로, 기업의 과거·현재 데이터를 분석하여 미래 의사결정에 활용하는 기술 체계이다. 현대의 BI는 ETL과 DW에서 수집된 데이터를 OLAP 엔진으로 분석하고, 대시보드와 리포트로 시각화하여 경영 의사결정을 지원하는 전체 파이프라인을 포함한다.

BI가 기업에 필요한 이유는 세 가지이다. 첫째, **데이터 민주화**: 데이터 분석을 IT 전문가의 전유물이 아닌 비즈니스 사용자도 셀프서비스로 수행할 수 있게 한다. 둘째, **의사결정 속도**: 경영진이 직관이나 경험이 아닌 데이터 기반 의사결정을 빠르게 내릴 수 있게 한다. 셋째, **기업 성과 모니터링**: KPI (Key Performance Indicator) 대시보드로 실시간에 가까운 기업 성과를 모니터링하고, 목표 달성 여부를 즉시 파악한다.

BI의 발전 단계는 크게 4세대로 구분된다. 1세대(1970-90s): 정기 보고서(Reporting) 위주. 2세대(2000s): OLAP 기반 자유 탐색 분석. 3세대(2010s): 셀프서비스 BI, 모바일 BI. 4세대(2020s): AI/ML 통합 증강 분석(Augmented Analytics), 자연어 쿼리, 자동 인사이트.

📢 **섹션 요약 비유**: BI는 기업의 '내비게이션 시스템'으로, 과거 주행 기록(데이터)을 분석하여 현재 위치(현황)를 파악하고, 최적 목적지(전략)로 가는 경로(의사결정)를 안내한다.

### BI 분석 유형 4단계 진화

| 분석 유형 | 질문 | 기술 | 가치 수준 |
|:---|:---|:---|:---:|
| **설명적 분석(Descriptive)** | 무슨 일이 있었나? | 대시보드, 보고서 | 기본 |
| **진단적 분석(Diagnostic)** | 왜 그랬나? | OLAP Drill-down | 중간 |
| **예측적 분석(Predictive)** | 어떻게 될 것인가? | ML, 회귀분석 | 높음 |
| **처방적 분석(Prescriptive)** | 어떻게 해야 하나? | AI 최적화, 시뮬레이션 | 최고 |

### BI 도구 비교

| 도구 | 특징 | 강점 | 약점 |
|:---|:---|:---|:---|
| **Tableau** | 시각화 전문 | 풍부한 차트, 드래그앤드롭 | 비용 높음 |

## Ⅲ. 비교 및 연결

### 전통 BI vs 현대 BI

| 항목 | 전통 BI (2000s) | 현대 BI (2020s) |
|:---|:---|:---|
| **데이터 구조** | 정형 데이터만 | 정형+비정형+실시간 |
| **분석 주체** | IT/분석가 | 비즈니스 사용자 (셀프서비스) |
| **배포 방식** | 온프레미스 | 클라우드 SaaS |
| **응답 시간** | 배치 (일/주 기준) | 실시간~준실시간 |
| **분석 깊이** | 설명적 분석 위주 | 예측·처방적 분석 통합 |
| **AI 통합** | 없음 | 자동 인사이트, NLQ |
| **사용자 경험** | 리포트 중심 | 대시보드 + 탐색 + 협업 |

### 셀프서비스 BI (SSBI, Self-Service BI)

```
  전통 BI:
  [비즈니스 요청] → [IT 분석] → [2주 후 리포트 제공]

  셀프서비스 BI:
  [비즈니스 사용자] → [직접 BI 도구] → [즉시 분석]
  → IT 의존도 감소, 분석 민첩성 향상
  → 단점: 데이터 거버넌스 약화 (스프레드마트 문제)
```

📢 **섹션 요약 비유**: 셀프서비스 BI는 ATM기처럼, 은행(IT팀)에 직접 가지 않고도 언제든지 원하는 거래(분석)를 스스로 할 수 있게 해주지만, 보안(거버넌스)은 철저히 관리되어야 한다.

## Ⅴ. 기대효과 및 결론

### 기대효과

BI 구축으로 경영진은 실시간에 가까운 KPI 모니터링으로 시장 변화에 빠르게 대응하고, 현업 담당자는 데이터 기반 일상 업무 의사결정이 가능해진다. 조직 전체의 데이터 리터러시가 향상되어 "데이터로 대화하는" 문화가 정착된다.

### 한계 및 전제조건

- **가비지 인, 가비지 아웃**: 원천 데이터 품질이 낮으면 BI 분석도 신뢰할 수 없다.
- **Shadow BI 문제**: 공식 BI 외에 개인 Excel 분석이 범람하면 KPI 불일치가 발생한다.
- **전제조건**: 데이터 거버넌스 체계, MDM, 명확한 KPI 정의가 BI 구현 전에 갖춰져야 한다.

📢 **섹션 요약 비유**: BI는 자동차처럼, 아무리 좋은 차(BI 도구)가 있어도 도로(데이터 인프라)가 없거나, 목적지(KPI)가 불분명하거나, 운전 기술(데이터 리터러시)이 없으면 제대로 달릴 수 없다.

### 📌 관련 개념 맵

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| DW | BI의 데이터 기반 | 분석 데이터 저장소 |
| OLAP | BI의 분석 엔진 | 다차원 집계 분석 |
| KPI | BI의 목표 기준 | 비즈니스 성과 지표 |
| 셀프서비스 BI | 현대 BI 방향 | 사용자 자율 분석 |
| 데이터 거버넌스 | BI 품질 기반 | 데이터 정책·표준 |
| 증강 분석 | BI 미래 방향 | AI가 자동 인사이트 제공 |

### 👶 어린이를 위한 3줄 비유 설명

1. BI는 학교 성적 관리 시스템처럼, 모든 시험 점수(데이터)를 모아서 "어떤 과목이 약한지, 어느 선생님 수업이 효과적인지"를 한눈에 보여주는 도구예요.
2. 옛날에는 선생님(IT팀)만 성적표를 만들 수 있었지만, 셀프서비스 BI는 학부모(비즈니스 사용자)가 직접 성적 그래프를 볼 수 있게 해줘요.
3. 좋은 BI를 갖추려면 정확한 점수(데이터 품질)와 명확한 목표(KPI)가 먼저 있어야 해요.
