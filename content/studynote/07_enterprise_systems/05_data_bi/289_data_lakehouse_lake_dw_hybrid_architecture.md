## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 데이터 레이크(유연성·저비용)와 데이터 웨어하우스(ACID·성능·거버넌스)의 장점을 하나의 플랫폼에 통합한 아키텍처가 데이터 레이크하우스(Data Lakehouse)다.
> 2. **가치**: 중복 데이터 파이프라인(ETL) 제거, BI·ML·스트리밍을 단일 스토리지에서 처리하므로 데이터 사일로와 레이턴시를 동시에 해소한다.
> 3. **판단 포인트**: Delta Lake, Apache Iceberg, Apache Hudi 같은 오픈 테이블 포맷이 기반 기술이며, 스토리지·컴퓨팅 분리 클라우드 환경에서 가장 효과적이다.

## Ⅰ. 개요 및 필요성

전통적인 데이터 아키텍처는 운영 DB → ETL (Extract-Transform-Load) → 데이터 웨어하우스(DW) 구조였다. DW는 스키마 관리, ACID 트랜잭션, 고속 BI 쿼리를 제공했지만 비용이 높고 비정형 데이터 처리가 어려웠다. 이를 보완하기 위해 데이터 레이크가 도입됐지만, 메타데이터 부재·ACID 미지원·낮은 쿼리 성능으로 Data Swamp가 되는 문제가 발생했다.

데이터 레이크하우스(Data Lakehouse)는 2020년 Databricks가 Delta Lake를 공개하며 본격 등장했다. 오브젝트 스토리지(S3, ADLS) 위에 트랜잭션 메타데이터 레이어(Delta Log)를 추가해 ACID 보장, 스키마 강제, 시간여행(Time Travel) 기능을 구현한다. 결과적으로 BI 분석, ML 학습, 스트리밍 처리를 단일 스토리지에서 수행할 수 있다.

📢 **섹션 요약 비유**: 레이크(창고)와 DW(깔끔한 서랍장)을 합쳐 창고에도 서랍장의 질서를 부여한 것이 레이크하우스다.

### 오픈 테이블 포맷 비교

| 항목 | Delta Lake | Apache Iceberg | Apache Hudi |
|:---|:---|:---|:---|
| 개발사 | Databricks | Netflix | Uber |
| ACID | ✅ | ✅ | ✅ |
| Time Travel | ✅ | ✅ | ✅ |
| 스키마 진화 | ✅ | ✅ | ✅ |
| 스트리밍 | ✅ | ✅ | ✅ (Upsert 특화) |
| 파티션 진화 | 제한 | ✅ (강점) | 제한 |
| 최적화 | Z-Ordering | Hidden Partitioning | Clustering |

📢 **섹션 요약 비유**: 세 가지 테이블 포맷은 모두 창고에 서랍장 시스템을 설치하는 도구인데, 각자의 특기가 조금씩 다른 공구 세트다.

## Ⅲ. 비교 및 연결

### Data Warehouse vs Data Lake vs Data Lakehouse

| 항목 | DW | Data Lake | Data Lakehouse |
|:---|:---|:---|:---|
| 스토리지 비용 | 높음 | 낮음 | 낮음 |
| ACID | ✅ | ❌ | ✅ |
| 지원 데이터 유형 | 정형 | 정형+비정형 | 정형+비정형 |
| SQL 성능 | 최고 | 낮음 | 높음 (향상 중) |
| ML 지원 | 제한 | 높음 | 높음 |
| 스키마 | On Write | On Read | 유연(On Read/Write) |
| 거버넌스 | 강함 | 약함 | 강함 (카탈로그 통합) |

📢 **섹션 요약 비유**: DW는 5성급 호텔 객실, 레이크는 캠핑장, 레이크하우스는 글램핑장—편안함과 자유로움을 동시에 갖췄다.

## Ⅳ. 실무 적용 및 기술사 판단

**도입 전략**:
1. **기존 Data Lake 있는 경우**: Delta Lake/Iceberg 전환으로 ACID 추가 (점진적 마이그레이션)
2. **신규 구축**: 처음부터 Lakehouse 아키텍처로 설계 (DW + Lake 이중 구축 비용 절감)
3. **클라우드별 솔루션**:
   - AWS: Amazon Redshift + S3 + Glue / Databricks on AWS
   - Azure: Azure Synapse Analytics + ADLS
   - GCP: BigLake (BigQuery + GCS 통합)

**기술사 판단 포인트**:
- BI 중심 기업: DW 성능이 중요하므로 Snowflake/BigQuery 우선
- ML+BI 혼합 기업: 레이크하우스가 최적 (Databricks, Synapse)
- 비용 민감 기업: 오픈소스 Delta Lake + Spark on Kubernetes

📢 **섹션 요약 비유**: 레이크하우스는 '올인원 주방'—냉장고·오븐·식기세척기가 하나로 합쳐진 스마트 키친처럼 모든 요리(분석)를 한 공간에서 해결한다.

## Ⅴ. 기대효과 및 결론

**기대효과**:
- 데이터 파이프라인 중복 제거 → ETL 비용 30~50% 절감
- BI 분석과 ML 학습을 단일 플랫폼에서 처리 → 데이터 신선도 향상
- ACID 보장으로 데이터 불일치 오류 제거
- Time Travel 기능으로 규제 감사·롤백 대응 가능

**한계 및 전제조건**:
- 소규모 데이터(~100GB)에서는 전통 DW가 더 빠를 수 있음
- 오픈 테이블 포맷 관리 운영 역량 필요 (메타데이터 유지관리)
- 멀티 엔진 접근 시 엔진별 호환성 검증 필수

📢 **섹션 요약 비유**: 레이크하우스는 만능 선수이지만 진입 장벽이 있어, 처음부터 모든 기능을 다 쓰려 하면 오히려 복잡해진다—단계적 도입이 핵심이다.
