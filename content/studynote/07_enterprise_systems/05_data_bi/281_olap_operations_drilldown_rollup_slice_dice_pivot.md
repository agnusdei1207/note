## 핵심 인사이트 (3줄 요약)

> 1. **본질**: OLAP 연산은 다차원 데이터 큐브를 다양한 방식으로 탐색·조작하는 표준 분석 동작으로, 비즈니스 사용자가 데이터를 "위아래로 이동하고 자르고 회전"하며 인사이트를 발견하는 도구이다.
> 2. **가치**: 5가지 핵심 연산(Drill-down·Roll-up·Slice·Dice·Pivot)이 결합되면 SQL 없이도 복잡한 다차원 탐색이 가능해지며, BI 도구의 인터랙티브 분석 기능의 기반이 된다.
> 3. **판단 포인트**: 각 연산의 정확한 차이(Slice=1차원 고정 vs Dice=2차원+ 고정, Drill-down=세분화 vs Roll-up=요약)를 기술사 논술에서 명확히 구분하여 서술해야 한다.

## Ⅰ. 개요 및 필요성

OLAP 연산은 Edgar Codd의 OLAP 12 규칙과 Ralph Kimball의 다차원 모델링에서 정의된 표준 분석 조작 방법이다. 비즈니스 분석에서 데이터 탐색은 단순히 "얼마나 팔렸는가"를 조회하는 것을 넘어, "어디서? → 어느 지역에서? → 어느 매장에서?"처럼 점점 세밀해지거나 반대로 넓어지는 탐색 과정이 필요하다. OLAP 연산은 이런 탐색 패턴을 표준화한 도구이다.

5가지 핵심 연산은 각각 다른 분석 목적을 가진다. **Drill-down(드릴다운)**은 상위 차원에서 하위 차원으로 세밀하게 파고드는 연산이다. **Roll-up(롤업)**은 반대 방향으로 세부 데이터를 요약·집계한다. **Slice(슬라이스)**는 하나의 차원 값을 고정하여 2차원 뷰를 추출한다. **Dice(다이스)**는 여러 차원의 범위를 지정하여 데이터의 부분집합을 추출한다. **Pivot(피벗)**은 차원 축을 회전시켜 데이터를 재구성한다.

이 연산들은 BI 도구(Tableau, Power BI, MicroStrategy)에서 드래그앤드롭으로 직관적으로 수행되지만, 그 이면에는 MDX (Multi-Dimensional eXpressions)나 SQL 집계 쿼리가 동작한다.

📢 **섹션 요약 비유**: OLAP 연산 5가지는 데이터라는 거대한 보석(큐브)을 다루는 도구로, 확대경(Drill-down), 축소경(Roll-up), 수평 절단(Slice), 입체 절단(Dice), 뒤집기(Pivot)를 통해 보석의 모든 면을 살펴볼 수 있다.

## Ⅱ. 아키텍처 및 핵심 원리

### 5대 OLAP 연산 개요

| 연산 | 이동 방향 | 차원 변화 | 데이터 범위 |
|:---|:---|:---|:---|
| **Drill-down** | 상위→하위 차원 | 세분화 | 더 좁고 상세 |
| **Roll-up** | 하위→상위 차원 | 요약 | 더 넓고 집계 |
| **Slice** | 단일 차원 고정 | 1개 차원 고정 | 2D 단면 |
| **Dice** | 복수 차원 범위 | N개 차원 범위 | N차원 부분집합 |
| **Pivot** | 축 회전 | 행↔열 교환 | 동일 데이터 재배치 |

### 각 연산 상세 구조

```
  ┌─────────────────────────────────────────────────────────┐
  │              OLAP 큐브 연산 시각화                       │
  │                                                         │
  │  초기 뷰: 지역별×연도별 매출 (2D 뷰)                    │
  │        서울   부산   대구   전국합계                     │
  │  2022  500    300    200    1,000                        │
  │  2023  600    350    250    1,200                        │
  │  2024  750    400    300    1,450                        │
  │                                                         │
  │  ① Drill-down (서울 → 강남/강북/강동 세분화):           │
  │        강남   강북   강동   서울합계                     │
  │  2024  350    200    200    750                          │
  │                                                         │
  │  ② Roll-up (연도 → 전체 기간 요약):                     │
  │        서울   부산   대구   전국                         │
  │  전체  1,850  1,050  750    3,650                        │
  │                                                         │
  │  ③ Slice (2024년만 고정):                               │
  │        서울   부산   대구   [제품 차원 자유]              │
  │  전자  350    200    150                                 │
  │  식품  200    100    100                                 │
  │                                                         │
  │  ④ Dice (2024 + 서울/부산 + 전자/식품):                 │
  │  3차원 → 2024년 × {서울,부산} × {전자,식품} 부분집합   │
  │                                                         │
  │  ⑤ Pivot (지역↔제품 축 교환):                          │
  │        전자   식품   잡화                                │
  │  서울  350    200    200                                 │
  │  부산  200    100    100                                 │
  └─────────────────────────────────────────────────────────┘
```

### 연산별 SQL 매핑

```sql
-- ① Drill-down: 지역(서울) → 구(강남/강북/강동)
SELECT district, SUM(sales_amount)
FROM fact_sales f
JOIN dim_store s ON f.store_sk = s.store_sk
WHERE s.city = '서울' AND d.year = 2024
GROUP BY s.district;

-- ② Roll-up: 월별 → 분기별 → 연별
SELECT year, quarter, SUM(sales_amount)
FROM fact_sales f
JOIN dim_date d ON f.date_sk = d.date_sk
GROUP BY ROLLUP(year, quarter);  -- SQL ROLLUP 함수

-- ③ Slice: 특정 차원 값 고정 (2024년 고정)
SELECT city, category, SUM(sales_amount)
FROM fact_sales f
JOIN dim_date    d ON f.date_sk = d.date_sk
JOIN dim_store   s ON f.store_sk = s.store_sk
JOIN dim_product p ON f.prod_sk  = p.prod_sk
WHERE d.year = 2024  -- Slice: 2024년 고정
GROUP BY city, category;

-- ⑤ Pivot (SQL CASE 또는 PIVOT 함수)
SELECT city,
    SUM(CASE WHEN category='전자' THEN sales_amount END) AS 전자,
    SUM(CASE WHEN category='식품' THEN sales_amount END) AS 식품
FROM ...
GROUP BY city;
```

### Drill-through (드릴쓰루) - 추가 연산

집계된 OLAP 셀에서 원본 트랜잭션 데이터로 직접 이동하는 연산이다. "2024년 서울 전자제품 매출 750만원" → 해당 건의 원본 주문 리스트를 조회하는 방식으로, OLAP → OLTP 방향의 상세 조회이다.

📢 **섹션 요약 비유**: Drill-through는 지도의 특정 구역을 클릭하면 해당 건물들의 상세 정보가 팝업되는 것처럼, 집계 수치 뒤에 숨은 원본 거래 내역을 즉시 확인하는 기능이다.

## Ⅲ. 비교 및 연결

### Slice vs Dice 명확한 구분

| 항목 | Slice | Dice |
|:---|:---|:---|
| **차원 고정 수** | 1개 차원 고정 | 2개+ 차원 범위 지정 |
| **결과 형태** | 2D 평면 (단면) | N차원 부분큐브 |
| **SQL 표현** | WHERE 단일 조건 | WHERE 복수 조건 |
| **예시** | year = 2024 | year IN (2023,2024) AND city = '서울' |
| **비유** | 빵을 1번 자르기 | 빵을 여러 방향으로 자르기 |

### Roll-up vs Drill-down 방향성

```
  ──── Drill-down 방향 (세분화) ────▶
  전국 → 지역 → 시/도 → 구 → 동 → 매장

  ◀─── Roll-up 방향 (집계) ────────
  매장 → 동 → 구 → 시/도 → 지역 → 전국
```

📢 **섹션 요약 비유**: Slice는 김밥을 1번 자르는 것(단면 추출), Dice는 주사위(Dice의 어원)처럼 여러 방향으로 자른 육면체 조각을 추출하는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

### BI 도구에서의 OLAP 연산 매핑

| BI 도구 액션 | OLAP 연산 | 예시 |
|:---|:---|:---|
| 차트 클릭 → 세부 | Drill-down | 분기 클릭 → 월별 전환 |
| 상위 레벨 이동 | Roll-up | 월 → 분기 뒤로가기 |
| 필터 적용 | Slice/Dice | 지역 필터 = '서울' |
| 행/열 교환 | Pivot | 행과 열 드래그 교체 |
| 셀 클릭 → 원본 | Drill-through | 원본 거래 리스트 |

### 기술사 논술 작성 요령

OLAP 연산 문제에서 핵심은 각 연산의 **방향성과 차원 변화**를 명확히 서술하는 것이다. 단순 정의만 나열하지 말고, 구체적인 예시(지역별·시간별 매출 분석 시나리오)와 함께 설명하고 각 연산이 어떤 비즈니스 질문에 답하는지를 연결 지어 서술한다.

📢 **섹션 요약 비유**: OLAP 연산을 구사하는 분석가는 요리사처럼, 재료(데이터)를 큼직하게 자르거나(Roll-up), 얇게 슬라이스하거나(Slice), 특정 부위만 도려내거나(Dice), 접시 위에 다시 배치하는(Pivot) 기술로 다양한 요리(인사이트)를 만들어낸다.

## Ⅴ. 기대효과 및 결론

### 기대효과

OLAP 연산의 조합으로 비즈니스 사용자는 "전체 보기 → 이상 징후 발견 → 드릴다운으로 원인 탐색 → 슬라이스로 특정 조건 분석 → 피벗으로 다른 관점 확인" 같은 체계적인 분석 흐름을 SQL 없이 수행할 수 있다. 이는 데이터 기반 의사결정의 속도와 품질을 동시에 향상시킨다.

### 한계 및 전제조건

- **MOLAP 집계 의존성**: Drill-down이 미리 집계되지 않은 레벨까지 내려가면 성능이 저하된다.
- **MDX 복잡성**: 고급 OLAP 연산은 MDX 언어로 구현되며, SQL에 비해 학습 곡선이 가파르다.
- **전제조건**: 차원 계층(Hierarchy)이 올바르게 설계되어야 Drill-down/Roll-up 경로가 정확하게 작동한다.

📢 **섹션 요약 비유**: OLAP 연산 5가지는 망원경의 줌 기능처럼, 전체 밤하늘(Roll-up)에서 특정 별자리(Slice/Dice)로 좁히고, 별 하나에 확대경(Drill-down)을 대거나, 망원경을 돌려 다른 각도(Pivot)로 보는 완전한 탐색 도구이다.

### 👶 어린이를 위한 3줄 비유 설명

1. Drill-down은 지도를 확대하는 것처럼 "전국 → 서울 → 강남구 → 역삼동"으로 점점 자세히 들어가는 거예요.
2. Slice는 케이크를 한 면만 자르는 것처럼 "2024년 케이크 한 층"만 꺼내서 보는 거고, Dice는 여러 방향으로 잘라 작은 조각을 만드는 거예요.
3. Pivot은 레고 블록을 옆으로 돌려보는 것처럼, 같은 데이터를 행과 열을 바꾸어 다른 각도에서 보는 거예요.
