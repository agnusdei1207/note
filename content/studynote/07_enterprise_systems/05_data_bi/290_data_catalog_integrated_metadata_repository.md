## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 데이터 카탈로그(Data Catalog)는 조직 내 모든 데이터 자산의 위치·의미·품질·오너·사용 이력을 체계적으로 등록·검색할 수 있게 하는 '데이터의 도서관 목록 시스템'이다.
> 2. **가치**: 분석가가 필요한 데이터를 수일 걸려 찾던 것을 수분으로 단축하고, 데이터 계보(Lineage) 추적으로 규제 감사·영향도 분석을 자동화한다.
> 3. **판단 포인트**: 카탈로그 도구 선택 시 자동 메타데이터 크롤링, 혈통(Lineage) 시각화, 비즈니스 용어 사전(Business Glossary) 통합 세 가지 기능이 필수다.

## Ⅰ. 개요 및 필요성

데이터 카탈로그(Data Catalog)는 기업 내 데이터베이스, 데이터 레이크, API, BI 리포트 등 모든 데이터 자산에 대한 메타데이터(Metadata)를 중앙에서 관리하는 시스템이다. 쉽게 말해 '데이터의 도서관 목록 카드'다. 누가(오너), 어디에(위치), 어떤 내용이(스키마/설명), 어떤 품질로(품질 지표), 어떻게 변환됐는지(리니지)를 한 곳에서 조회할 수 있다.

현대 기업은 수천 개의 테이블, 수백 개의 데이터 파이프라인을 운영한다. 데이터 거버넌스(GDPR, ISMS-P) 강화로 어떤 데이터가 어디서 왔는지 추적이 의무화됐다. 카탈로그 없이는 새로운 데이터셋을 사용할 때마다 DBA에게 일일이 문의해야 하고, 규제 감사 시 영향도 분석에 수주가 소요된다. 데이터 카탈로그는 이 모든 비효율을 해소하는 핵심 인프라다.

📢 **섹션 요약 비유**: 도서관에 목록 카드가 없으면 사서에게 일일이 물어봐야 하듯, 데이터 카탈로그 없이는 데이터 엔지니어에게 매번 슬랙을 보내야 한다.

### 주요 메타데이터 유형

| 유형 | 설명 | 예시 |
|:---|:---|:---|
| 기술 메타데이터 | 스키마, 데이터 타입, 크기 | 테이블명, 컬럼명, VARCHAR(255) |
| 비즈니스 메타데이터 | 비즈니스 용어, 설명, 오너 | "고객ID: 회원 고유 식별자" |
| 운영 메타데이터 | 갱신 주기, 품질 지표, 사용 빈도 | 일 1회 갱신, NULL률 0.1% |
| 소셜 메타데이터 | 댓글, 별점, 사용 이력 | "신뢰도 높음", 주간 조회 500회 |

📢 **섹션 요약 비유**: 카탈로그는 데이터에 '이름표+설명서+품질 인증서+이용 후기'를 한 번에 붙여주는 종합 정보 태그다.

## Ⅲ. 비교 및 연결

### 주요 데이터 카탈로그 솔루션

| 솔루션 | 특징 | 적합 환경 |
|:---|:---|:---|
| Apache Atlas | 오픈소스, Hadoop 생태계 강점 | 온프레미스 Hadoop |
| AWS Glue Data Catalog | S3·Redshift 자동 통합 | AWS 네이티브 |
| Azure Purview | Microsoft 생태계 전방위 | Azure 중심 |
| Collibra | 엔터프라이즈 거버넌스 강점 | 대형 기업 |
| Datahub (LinkedIn) | 오픈소스, 확장성 우수 | 멀티 클라우드 |
| Alation | AI 기반 추천, 협업 강점 | 분석가 중심 조직 |

📢 **섹션 요약 비유**: 카탈로그 솔루션 선택은 도서관 관리 시스템 선택과 같다—작은 도서관은 무료 소프트웨어로, 대형 국립도서관은 전문 상용 시스템이 필요하다.

## Ⅳ. 실무 적용 및 기술사 판단

**구축 단계**:
1. **자동 크롤링 설정**: 주요 데이터 소스(RDB, S3, Kafka)에 크롤러 연결
2. **비즈니스 용어 사전 구축**: 부서별 용어 통일 (예: '고객' = 회원 테이블의 CUST_ID)
3. **오너십 지정**: 각 데이터셋에 담당 부서·담당자 매핑
4. **품질 대시보드 연동**: Great Expectations, Deequ 등 품질 툴과 통합
5. **리니지 자동 추적**: dbt, Airflow와 연동하여 변환 이력 자동 기록

**기술사 판단 포인트**:
- 카탈로그는 '구축' 프로젝트가 아니라 '운영' 서비스로 접근해야 성공
- 비즈니스 용어 사전이 없는 카탈로그는 반쪽짜리 (기술 메타만 있어서 분석가가 이해 못 함)
- AI 기반 자동 태깅·추천 기능이 있는 도구를 선호 (수동 관리 한계 돌파)

📢 **섹션 요약 비유**: 카탈로그 구축은 도서관 개관이고, 카탈로그 운영은 매일 새 책이 들어올 때마다 목록을 업데이트하는 사서의 일상 업무다.

## Ⅴ. 기대효과 및 결론

**기대효과**:
- 데이터 검색 시간 85% 단축 (수일 → 수분)
- GDPR·ISMS-P 감사 대응 시간 60% 절감 (리니지 자동 추적)
- 데이터 품질 인식률 향상 → 신뢰성 있는 분석 의사결정
- 신규 분석가 온보딩 기간 단축

**한계 및 전제조건**:
- 초기 메타데이터 입력 작업량이 상당 (레거시 시스템은 수동 입력 불가피)
- 조직 전반의 '데이터 문화' 형성 없이는 카탈로그가 형식화
- 카탈로그 데이터 최신성 유지를 위한 자동화 파이프라인 필수

📢 **섹션 요약 비유**: 완벽한 도서관 목록을 만들었어도 새 책이 들어올 때 아무도 등록하지 않으면 금세 쓸모없어지듯, 카탈로그도 지속적인 관리가 없으면 그 가치가 사라진다.

### 👶 어린이를 위한 3줄 비유 설명

1. 데이터 카탈로그는 도서관의 목록 카드예요—어떤 책이(데이터) 어디에 있고, 누가 썼는지(오너) 한눈에 찾을 수 있어요.
2. 마치 마트 앱에서 '우유'를 검색하면 어떤 코너에 있는지 바로 알려주는 것처럼, 데이터 카탈로그도 원하는 데이터를 바로 찾아줘요.
3. 여기에는 '이 데이터는 신선해요(품질)', '이 데이터는 주문 시스템에서 왔어요(리니지)'같은 중요한 정보도 같이 붙어 있어요!
