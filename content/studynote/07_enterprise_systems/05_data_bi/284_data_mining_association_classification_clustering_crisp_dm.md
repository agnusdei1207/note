## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 데이터 마이닝 (Data Mining)은 대규모 데이터에서 통계·머신러닝 기법으로 **기존에 알려지지 않은 숨겨진 패턴과 지식**을 자동으로 발견하는 과정으로, BI의 예측·처방적 분석 단계를 구현한다.
> 2. **가치**: 연관 분석(장바구니)·분류(스팸 필터)·군집(고객 세그먼테이션)·회귀(매출 예측) 등 다양한 기법을 통해 인간의 직관으로는 발견하기 어려운 비즈니스 패턴을 자동으로 추출한다.
> 3. **판단 포인트**: CRISP-DM (Cross Industry Standard Process for Data Mining) 방법론은 데이터 마이닝 프로젝트의 표준 프로세스로, 비즈니스 이해에서 시작하여 배포까지 6단계 순환 구조를 따른다.

## Ⅰ. 개요 및 필요성

데이터 마이닝은 1990년대 대규모 데이터베이스에서 유용한 지식을 추출하는 기술로 등장했다. 통계학, 머신러닝, 데이터베이스 기술이 융합된 학제간 분야로, "데이터에서 광물(유용한 패턴)을 캐낸다"는 의미에서 마이닝(Mining)이라는 이름을 사용한다.

데이터 마이닝이 필요한 이유는 다음과 같다. 첫째, **패턴의 숨김**: 수백만 고객의 구매 데이터에서 "기저귀를 구매한 사람은 맥주도 함께 구매한다"는 패턴은 인간의 직관으로는 발견하기 어렵다. 데이터 마이닝의 연관 분석이 이런 비직관적 패턴을 발견한다. 둘째, **대규모 데이터 처리**: 억 단위 레코드에서 의미 있는 패턴을 수동으로 찾는 것은 불가능하며, 자동화된 알고리즘이 필요하다. 셋째, **예측 모델**: 과거 데이터에서 학습하여 미래를 예측하는 모델(이탈 예측, 수요 예측)을 구축한다.

현대에는 데이터 마이닝이 머신러닝(ML)·딥러닝(DL)과 통합되어 AI 기반 분석 플랫폼으로 진화했다. 그러나 CRISP-DM 방법론은 AI/ML 프로젝트에서도 여전히 유효한 표준 프로세스로 사용된다.

📢 **섹션 요약 비유**: 데이터 마이닝은 거대한 모래사장에서 금을 채굴하는 것처럼, 방대한 데이터(모래) 속에서 가치 있는 패턴(금)을 자동으로 찾아내는 기술이다.

## Ⅱ. 아키텍처 및 핵심 원리

### CRISP-DM 6단계 프로세스

```
  ┌─────────────────────────────────────────────────────────┐
  │                  CRISP-DM 순환 프로세스                  │
  │                                                         │
  │         ┌─────────────────────────────┐                │
  │         │   1. Business              │                │
  │    ┌────▼ Understanding ◄────────────┼────┐           │
  │    │                                 │    │           │
  │    ▼    2. Data Understanding        │    │           │
  │  6. Deployment ◄────────────────────┘    │           │
  │    │                                      │           │
  │    │    3. Data Preparation               │           │
  │    └──────────────────────┐               │           │
  │                           ▼               │           │
  │               4. Modeling ──▶ 5. Evaluation           │
  │                                           │           │
  │                   (반복 순환) ────────────┘           │
  └─────────────────────────────────────────────────────────┘
```

### CRISP-DM 단계별 상세

| 단계 | 영문 명칭 | 주요 활동 | 산출물 |
|:---|:---|:---|:---|
| **1단계** | Business Understanding | 비즈니스 목표, 성공 기준 정의 | 프로젝트 계획서 |
| **2단계** | Data Understanding | EDA (탐색적 데이터 분석), 품질 검사 | 데이터 설명 보고서 |
| **3단계** | Data Preparation | 정제, 특성 엔지니어링, 변환 | 분석용 데이터셋 |
| **4단계** | Modeling | 알고리즘 선택, 모델 학습 | 모델 파라미터 |
| **5단계** | Evaluation | 비즈니스 목표 부합 검증 | 평가 보고서 |
| **6단계** | Deployment | 배포, 모니터링, 유지보수 | 배포 계획서 |

### 4대 데이터 마이닝 기법

```
  ┌─────────────────────────────────────────────────────────┐
  │              데이터 마이닝 주요 기법                     │
  │                                                         │
  │  1. 연관 분석 (Association Analysis)                   │
  │     목적: 동시 발생 패턴 발견 (장바구니 분석)           │
  │     알고리즘: Apriori, FP-Growth                        │
  │     지표: 지지도, 신뢰도, 향상도                        │
  │     예시: 기저귀 → 맥주 (향상도 3.5)                   │
  │                                                         │
  │  2. 분류 (Classification)                               │
  │     목적: 레이블이 있는 클래스로 예측                   │
  │     알고리즘: 의사결정나무, 랜덤포레스트, SVM           │
  │     예시: 이탈 여부 예측 (Yes/No)                       │
  │                                                         │
  │  3. 군집 (Clustering)                                   │
  │     목적: 유사한 그룹 자동 분류 (비지도학습)            │
  │     알고리즘: K-means, DBSCAN, 계층적 군집              │
  │     예시: 고객 세그먼테이션 (4개 그룹 도출)             │
  │                                                         │
  │  4. 회귀 (Regression)                                   │
  │     목적: 연속 수치 예측                                │
  │     알고리즘: 선형회귀, 다항회귀, XGBoost              │
  │     예시: 내달 매출 예측 (금액)                         │
  └─────────────────────────────────────────────────────────┘
```

### 연관 분석 핵심 지표

| 지표 | 계산식 | 해석 |
|:---|:---|:---|
| **지지도(Support)** | P(A∩B) = (A,B 동시 거래 수) / 전체 | A와 B가 함께 나타나는 빈도 |
| **신뢰도(Confidence)** | P(B\|A) = P(A∩B) / P(A) | A 구매 시 B도 구매할 확률 |
| **향상도(Lift)** | Confidence / P(B) | 우연 대비 연관성 강도 |

```
  연관 규칙 예시: {기저귀} → {맥주}
  지지도  = 500/10,000 = 5%    (5%의 거래에서 함께 구매)
  신뢰도  = 500/600   = 83%    (기저귀 구매 고객의 83%가 맥주 구매)
  향상도  = 83% / 20% = 4.15  (랜덤 대비 4.15배 연관성)
```

📢 **섹션 요약 비유**: 연관 분석의 향상도는 "우연히 같이 나타난 건지, 진짜 관련이 있는 건지"를 측정하는 도구로, 향상도=1이면 우연, 4.15이면 "매우 강한 연관성"을 의미한다.

## Ⅲ. 비교 및 연결

### 분류 vs 군집 비교

| 항목 | 분류 (Classification) | 군집 (Clustering) |
|:---|:---|:---|
| **학습 유형** | 지도학습 (레이블 있음) | 비지도학습 (레이블 없음) |
| **사전 지식** | 클래스 수, 레이블 필요 | 클래스 수 미리 정할 수 있음 |
| **예시** | 스팸/정상 메일 분류 | 고객 그룹 자동 발견 |
| **평가 지표** | 정확도, 재현율, F1 | 실루엣 계수, 엘보우 법칙 |
| **알고리즘** | 의사결정나무, RF, SVM | K-means, DBSCAN |

### 데이터 마이닝 vs 전통 통계 분석

| 항목 | 데이터 마이닝 | 전통 통계 분석 |
|:---|:---|:---|
| **접근 방향** | 데이터 주도 (귀납적) | 가설 주도 (연역적) |
| **데이터 크기** | 대규모 | 소규모 샘플 |
| **목적** | 패턴 발견 | 가설 검증 |
| **자동화** | 높음 | 낮음 |
| **해석 가능성** | 낮음 (블랙박스) | 높음 |

📢 **섹션 요약 비유**: 분류 vs 군집은 '학급 편성 방식'의 차이로, 분류는 미리 정해진 반(레이블)에 학생을 배정하고, 군집은 학생들의 특성을 보고 자연스럽게 유사한 그룹을 만드는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 비즈니스 적용 사례

| 산업 | 기법 | 활용 사례 | 비즈니스 가치 |
|:---|:---|:---|:---|
| 유통 | 연관 분석 | 추천 시스템, 함께 사는 제품 | 크로스셀링 매출 증대 |
| 금융 | 분류 | 신용 위험 평가, 사기 탐지 | 부실 채권 감소 |
| 통신 | 분류 | 고객 이탈 예측 | 유지 마케팅 비용 최적화 |
| 마케팅 | 군집 | 고객 세그먼테이션 | 타겟 마케팅 효율 향상 |
| 제조 | 회귀 | 수요 예측, 불량 예측 | 재고 최적화 |

### 기술사 논술 핵심: CRISP-DM 적용 서술

기술사 시험에서 데이터 마이닝 프로젝트를 설명할 때 CRISP-DM 6단계를 구체적 예시와 함께 서술해야 한다:

```
  사례: 통신사 고객 이탈 예측 모델 구축
  1단계: 연간 이탈 고객 15% → 10% 감소 목표
  2단계: 24개월 고객 행동 데이터 탐색, 이탈 비율 분석
  3단계: 결측값 처리, 피처 엔지니어링 (최근성, 빈도, 금액)
  4단계: 랜덤포레스트 모델 학습, 하이퍼파라미터 튜닝
  5단계: F1-Score 0.82, ROI 검증 (예방 비용 < 이탈 손실)
  6단계: CRM 시스템 배포, 월별 재학습 파이프라인 구축
```

📢 **섹션 요약 비유**: CRISP-DM은 집 짓기처럼, 먼저 어떤 집이 필요한지(비즈니스 이해)를 정하고, 땅을 조사하고(데이터 이해), 설계 후(모델링) 입주(배포)까지 체계적으로 진행하는 프로세스이다.

## Ⅴ. 기대효과 및 결론

### 기대효과

데이터 마이닝은 비즈니스에서 직관으로는 발견하기 어려운 패턴을 자동으로 추출하여 경쟁 우위를 제공한다. 이탈 예측 모델로 고객 유지율을 높이고, 연관 분석으로 크로스셀링 매출을 증대하며, 군집 분석으로 개인화 마케팅 효율을 향상시킨다.

### 한계 및 전제조건

- **과적합(Overfitting)**: 학습 데이터에 지나치게 최적화된 모델은 새 데이터에서 성능이 저하된다.
- **해석 가능성 한계**: 복잡한 앙상블 모델은 "왜 이런 결과가 나왔는가"를 설명하기 어렵다.
- **전제조건**: CRISP-DM 1단계(비즈니스 이해)에서 명확한 성공 기준을 정의해야 모델 평가 기준이 생긴다.

📢 **섹션 요약 비유**: 데이터 마이닝 모델은 레시피처럼, 학습 데이터라는 재료로 만들어진 요리이므로 재료(데이터)의 품질이 요리(모델)의 맛을 결정하며, 새로운 레스토랑(실제 환경)에서 제대로 만들어지려면 지속적인 검증과 개선이 필요하다.

### 👶 어린이를 위한 3줄 비유 설명

1. 데이터 마이닝은 탐정 놀이처럼, 수많은 데이터(단서)에서 숨겨진 패턴(범인)을 컴퓨터 알고리즘으로 자동으로 찾아내는 거예요.
2. 연관 분석은 "기저귀 사는 아빠는 맥주도 산다"처럼, 우리가 생각지 못한 관계를 데이터에서 발견해줘요.
3. CRISP-DM은 요리 레시피처럼, 좋은 결과(요리)를 위해 재료 선택(데이터)부터 상에 올리기(배포)까지 체계적인 순서를 정해놓은 거예요.
