> **핵심 인사이트 3줄**
> 1. 맥킨지 7S 모델은 조직 변화를 단순히 전략·구조만이 아닌 7가지 상호 의존적 요소(Strategy·Structure·Systems·Shared Values·Style·Staff·Skills)로 분석하는 프레임워크다.
> 2. 핵심은 Shared Values(공유 가치)가 중심에 위치해 나머지 6개 요소를 연결한다는 점으로, 조직 문화가 전략 실행의 성패를 결정한다.
> 3. IT 시스템 전환·디지털 변혁·M&A 통합 시 7S 정렬 진단이 실패 예방의 핵심 도구로 활용된다.

---

## Ⅰ. 맥킨지 7S 모델의 개요

맥킨지 7S 모델은 **Tom Peters와 Robert Waterman(맥킨지)이 1980년대 제안한 조직 분석·변화 관리 프레임워크**다.

```
            Strategy (전략)
           /               \
     Structure          Systems
      (구조)            (시스템)
         \    Shared    /
          ── Values ──
         /   (공유가치) \
       Staff           Style
      (인력)           (스타일)
           \          /
            Skills
            (기술)
```

### 7S 요소 분류

| 분류       | 요소                  | 설명                         |
|----------|----------------------|-----------------------------|
| 하드 S     | Strategy (전략)      | 경쟁 우위를 위한 행동 계획    |
| 하드 S     | Structure (구조)     | 조직 계층·역할 분담           |
| 하드 S     | Systems (시스템)     | 프로세스·IT 시스템·절차       |
| 소프트 S   | Shared Values        | 핵심 가치·문화·사명          |
| 소프트 S   | Style (스타일)       | 경영진 리더십 방식            |
| 소프트 S   | Staff (인력)         | 인재 채용·육성·동기부여       |
| 소프트 S   | Skills (기술/역량)   | 핵심 역량·차별화 능력         |

📢 **섹션 요약 비유**: 7S 모델은 자전거 7개 부품 점검이다 — 핸들(전략), 프레임(구조), 기어(시스템), 안장(공유 가치)이 모두 맞아야 잘 달린다. 하나만 삐끗해도 넘어진다.

---

## Ⅱ. 하드 S 3요소 — 전략·구조·시스템

### Strategy (전략)

- 경쟁 환경 변화에 대응하는 명확한 방향
- 예: "클라우드 우선" 전략, "아시아 시장 집중"

### Structure (구조)

| 구조 유형    | 특징                     | 적합 상황           |
|-----------|--------------------------|-------------------|
| 기능별 조직  | 전문성 높음, 조율 어려움  | 안정적 환경         |
| 사업부 조직  | 자율성 높음, 중복 발생   | 다각화 기업         |
| 매트릭스 조직 | 유연, 보고 체계 복잡     | 프로젝트 중심       |
| 플랫 조직   | 민첩, 관리 스팬 넓음     | 스타트업·혁신팀      |

### Systems (시스템)

- ERP, CRM, SCM 등 운영 시스템
- 의사결정 프로세스·보고 체계
- **디지털 전환에서 가장 먼저 바뀌는 요소**

📢 **섹션 요약 비유**: 하드 S는 건물의 뼈대다 — 전략(설계도), 구조(철골), 시스템(배관·전기)이 모두 잘 갖춰져야 건물이 선다.

---

## Ⅲ. 소프트 S 4요소 — 가치·스타일·인력·기술

### Shared Values (공유 가치) — 중심 요소

- 조직의 존재 이유·핵심 신념
- 변화 시 가장 마지막까지 남는 요소
- 예: "고객 중심", "혁신과 실험", "투명성"

### Style (리더십 스타일)

- 권위적·민주적·코칭형·변혁적 리더십
- 경영진의 의사결정 방식이 조직 문화를 형성

### Staff / Skills

| 요소   | 관리 포인트                      |
|------|----------------------------------|
| Staff | 핵심 인재 확보·이탈 방지          |
| Skills | 현재 vs 미래 필요 역량 갭 분석   |

📢 **섹션 요약 비유**: 소프트 S는 사람과 문화다 — 설비(하드 S)가 아무리 좋아도 직원들이 목표를 공유하지 않으면 공장은 멈춘다.

## Ⅳ. 7S 정렬 진단 활용

### 디지털 전환 7S 진단 예시

| 요소            | 현재 상태 (AS-IS)        | 목표 상태 (TO-BE)        | 갭                     |
|---------------|------------------------|--------------------------|------------------------|
| Strategy      | 오프라인 중심 판매        | 디지털 채널 50% 이상      | 채널 전략 전환         |
| Structure     | 계층형 4단계             | 스쿼드 기반 애자일 팀     | 조직 구조 재편          |
| Systems       | 레거시 ERP              | 클라우드 SaaS 전환        | 시스템 마이그레이션     |
| Shared Values | "안정·정확"             | "혁신·속도"              | 문화 변화 프로그램      |
| Style         | 지시·통제형             | 임파워먼트·자율           | 리더십 개발             |
| Staff         | IT 인력 부족            | 데이터 엔지니어 10명 확보  | 채용·재교육             |
| Skills        | 운영 역량 강함           | AI/ML 역량 취약           | 교육 프로그램           |

📢 **섹션 요약 비유**: 7S 갭 분석은 건강검진이다 — 7가지 수치(요소)를 측정해 정상 범위를 벗어난 항목(갭)을 집중 치료(변화 관리)한다.

## Ⅴ. M&A 통합과 IT 전환에서의 7S

### M&A 후 통합 실패 원인 (7S 시각)

```
M&A 실패의 70%: 문화(Shared Values·Style) 충돌
  피인수 기업의 핵심 직원(Staff) 이탈
  두 시스템(Systems) 통합 비용 과소평가
  전략(Strategy) 방향 불일치
```

### IT 시스템 전환 체크리스트

- ✅ Strategy: IT 전환이 사업 전략과 연계됐는가?
- ✅ Structure: IT 거버넌스·책임 구조가 명확한가?
- ✅ Systems: 마이그레이션·통합 계획이 있는가?
- ✅ Shared Values: 변화 필요성에 공감대가 있는가?
- ✅ Staff: 변화 주도할 핵심 인력이 있는가?
- ✅ Skills: 필요 기술 역량(클라우드·AI)을 갖추었는가?

📢 **섹션 요약 비유**: M&A 7S 통합은 두 집안 합가이다 — 살림(시스템), 가훈(가치), 생활 방식(스타일)이 모두 달라 충돌하며, 조율하지 않으면 동거가 아닌 분쟁이 된다.

```
맥킨지 7S 모델
├── 하드 S (측정 가능)
│   ├── Strategy (전략)
│   ├── Structure (구조)
│   └── Systems (시스템)
├── 소프트 S (무형)
│   ├── Shared Values (공유 가치) ← 중심
│   ├── Style (리더십 스타일)
│   ├── Staff (인력)
│   └── Skills (역량)
└── 활용 맥락
    ├── 조직 변화 관리
    ├── M&A 통합 진단
    └── 디지털 전환 갭 분석
```

## 👶 어린이를 위한 3줄 비유 설명

1. 7S 모델은 팀 점검 목록이다 — 전략(무엇을 할지), 구조(누가 무엇을 담당하는지), 시스템(어떻게 일하는지)처럼 7가지를 모두 확인한다.
2. Shared Values는 팀의 응원가다 — 아무리 훈련(기술)이 좋아도 같은 노래를 부르지 않으면 팀워크가 무너진다.
3. 7S 정렬은 자전거 정비다 — 페달(전략)이 아무리 세도 체인(시스템)이 끊어지면 앞으로 나가지 못한다.
