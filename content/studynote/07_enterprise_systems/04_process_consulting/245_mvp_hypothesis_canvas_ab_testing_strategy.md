## 핵심 인사이트 (3줄 요약)

> 1. **본질**: MVP 가설 캔버스(Hypothesis Canvas)는 개발 전에 핵심 가정을 명시화하고 검증 조건을 정의하는 도구이며, A/B 테스팅은 실제 사용자 데이터로 가설을 통계적으로 검증하는 실험 방법이다.
> 2. **가치**: 가설을 사전에 명문화하면 팀 내 주관적 판단을 데이터 기반 의사결정으로 전환하고, 고비용 개발 전에 방향 오류를 발견하여 자원 낭비를 방지한다.
> 3. **판단 포인트**: 기술사 서술에서는 가설 캔버스의 구성 요소(신념·지표·최소 기준·실험 설계)와 A/B 테스팅의 통계적 유효성(샘플 크기·신뢰 구간·p-value) 개념을 결합하여 제시해야 한다.

## Ⅰ. 개요 및 필요성

린 스타트업(Lean Startup)의 핵심 원칙은 "만들기 전에 배워라(Learn before Build)"다. 전통적 제품 개발에서는 수개월의 개발 후에야 시장 반응을 확인했지만, 이 방식은 방향이 틀렸을 때의 손실이 너무 크다.

MVP (Minimum Viable Product)는 핵심 가설을 검증하기 위한 최소한의 기능을 갖춘 제품이다. 여기서 "최소(Minimum)"는 기능의 부족이 아니라 **불필요한 기능의 제거**를 의미한다. MVP의 목적은 판매가 아니라 **학습(Learning)**이다.

가설 캔버스(Hypothesis Canvas)는 MVP가 검증해야 할 가설을 구조화하는 도구다. "우리는 [사용자]가 [문제]를 가지고 있고, [솔루션]이 [지표]로 측정되는 [결과]를 달성할 것이라 믿는다"의 형식으로 가설을 명문화한다.

A/B 테스팅은 두 가지 버전(A: 기존, B: 변경)을 동시에 사용자에게 노출하여 통계적으로 우수한 버전을 식별하는 실험 방법이다. 이는 "어떤 디자인이 더 나은가?"라는 주관적 토론을 데이터로 종결하는 수단이다.

📢 **섹션 요약 비유**: MVP 가설 캔버스는 요리 레시피 가설서—"이 재료로 이렇게 만들면 손님이 맛있다고 할 것"을 먼저 쓰고 요리하는 방식이다.

## Ⅱ. 아키텍처 및 핵심 원리

### 가설 캔버스 구조

```
┌──────────────────────────────────────────────────────────────────┐
│                    가설 캔버스 (Hypothesis Canvas)               │
├──────────────────────┬───────────────────────────────────────────┤
│  신념 (Belief)       │  우리는 [사용자]가 [행동]을 할 때         │
│                      │  [문제/니즈]를 경험한다고 믿는다          │
├──────────────────────┼───────────────────────────────────────────┤
│  검증 방법           │  우리는 [실험 설계]를 통해                │
│  (Experiment)        │  [기간] 동안 [샘플 크기]로 테스트한다     │
├──────────────────────┼───────────────────────────────────────────┤
│  성공 지표           │  [핵심 지표]가 [목표 수치] 이상이면       │
│  (Success Metric)    │  가설이 검증된 것으로 판단한다            │
├──────────────────────┼───────────────────────────────────────────┤
│  최소 기준           │  [지표]가 [최소 기준] 미만이면            │
│  (Minimum Criteria)  │  피벗(Pivot) 또는 폐기를 결정한다         │
├──────────────────────┼───────────────────────────────────────────┤
│  학습 결과           │  실험 후 확인된 사실:                     │
│  (Learning)          │  [발견 사항] → 다음 반복 방향             │
└──────────────────────┴───────────────────────────────────────────┘
```

### A/B 테스팅 프로세스

```
┌──────────────────────────────────────────────────────────────────┐
│                   A/B 테스팅 실험 흐름                           │
├─────────────────────────────────────────────────────────────────┤
│  1. 가설 정의                                                    │
│     "B 버전(CTA 버튼 색상 변경)이 A 버전보다 클릭률이 높을 것"   │
│                                                                  │
│  2. 샘플 크기 계산                                               │
│     · 기대 효과(Effect Size): 클릭률 5% → 8% (60% 향상)        │
│     · 신뢰 수준: 95% (α = 0.05)                                 │
│     · 검정력(Power): 80% (β = 0.20)                            │
│     → 샘플: 각 그룹 최소 1,000명                                │
│                                                                  │
│  3. 무작위 분할                                                  │
│     사용자를 A 그룹(50%)과 B 그룹(50%)으로 무작위 배분          │
│                                                                  │
│  4. 데이터 수집                                                  │
│     A: 클릭률 5.2%  │  B: 클릭률 7.8%                          │
│                                                                  │
│  5. 통계 검정                                                    │
│     p-value = 0.003 < 0.05 → 통계적으로 유의미                  │
│                                                                  │
│  6. 의사결정                                                     │
│     B 버전으로 전환 → 다음 가설 설계                             │
└──────────────────────────────────────────────────────────────────┘
```

### A/B 테스팅 핵심 지표

| 지표 | 의미 | 주의사항 |
|:---|:---|:---|
| p-value | 우연히 발생할 확률 | 0.05 미만 = 통계적 유의미 |
| 신뢰 구간 (CI) | 실제 효과 범위 추정 | 95% CI가 0을 포함하면 비유의미 |
| 효과 크기 (Effect Size) | 실질적 의미의 크기 | p-value가 작아도 효과 크기 확인 필수 |
| 샘플 파워 (Power) | 실제 효과 감지 능력 | 80% 이상 권장 |

📢 **섹션 요약 비유**: A/B 테스팅은 두 약의 임상시험처럼—한쪽은 기존 약, 한쪽은 새 약을 투여하고 통계로 어느 쪽이 더 효과적인지 증명한다.

## Ⅲ. 비교 및 연결

### MVP 유형 비교

| MVP 유형 | 설명 | 적합 상황 |
|:---|:---|:---|
| 랜딩 페이지 MVP | 제품 없이 관심도 측정 | 수요 검증 초기 단계 |
| 컨시어지 MVP | 수작업으로 서비스 제공 | 자동화 전 프로세스 검증 |
| 오즈의 마법사 MVP | 뒤에서 인간이 AI처럼 동작 | AI/자동화 기능 검증 |
| 프로토타입 MVP | 클릭 가능한 모형 | UX 흐름 검증 |
| 파일럿 MVP | 한정 지역/그룹 실제 서비스 | 운영 가능성 검증 |

### A/B 테스팅 vs. 다변량 테스팅 (MVT, Multivariate Testing)

| 비교 항목 | A/B 테스팅 | MVT (Multivariate Testing) |
|:---|:---|:---|
| 변수 수 | 1개 요소 변경 | 여러 요소 동시 변경 |
| 필요 샘플 | 상대적으로 적음 | 조합 수 × 샘플 필요 |
| 분석 복잡도 | 낮음 | 높음 (상호작용 효과 분석) |
| 적합 상황 | 단일 가설 검증 | 복수 요소 최적화 조합 탐색 |

📢 **섹션 요약 비유**: A/B는 두 길 중 하나를 선택하는 것이고, MVT는 지도에서 여러 경로 조합을 동시에 탐색하는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 전자상거래 결제 전환율 개선 사례

**가설 캔버스 적용**:
- **신념**: 결제 단계 간소화(3단계 → 1단계)가 전환율을 높일 것이다
- **실험**: 50,000명 방문자를 A/B 50:50 분할, 2주 실험
- **성공 지표**: 결제 완료율 현재 23% → 28% 이상
- **최소 기준**: 25% 미만이면 추가 정성 인터뷰 진행

**A/B 테스팅 결과**:
- A 그룹(기존): 결제 완료율 23.1%
- B 그룹(간소화): 결제 완료율 31.4%
- p-value = 0.001, 효과 크기 = 36% 향상

**의사결정**: B 버전 전체 배포 → 월 매출 약 8.3% 증가 효과.

### 의사결정 포인트
- **가설의 구체성**: "더 좋아질 것"이 아니라 "클릭률이 5%에서 8%로 향상될 것"처럼 수치화된 가설이어야 실험 설계가 가능하다.
- **노벨티 효과(Novelty Effect) 경계**: 새로운 UI에 일시적으로 클릭이 증가하는 현상. 최소 2주 이상 실험 권장.
- **p-hacking 방지**: 중간에 결과를 보고 유리할 때 실험을 중단하는 행위는 통계적 오류—사전에 종료 기준을 정하고 준수해야 한다.

📢 **섹션 요약 비유**: A/B 테스팅에서 중간에 결과 보고 멈추는 것은 동전 던지기를 앞면 나올 때까지만 세는 것처럼 불공정한 실험이다.

## Ⅴ. 기대효과 및 결론

### 기대효과

| 영역 | 효과 |
|:---|:---|
| 의사결정 품질 | 직관 → 데이터 기반 결정으로 전환 |
| 개발 리스크 | 고비용 개발 전 방향 오류 조기 발견 |
| 제품-시장 적합도 | PMF (Product-Market Fit) 탐색 속도 향상 |
| 조직 학습 | 실험 문화 정착으로 혁신 역량 강화 |

### 한계 및 전제 조건

**한계**: ① 트래픽이 적은 서비스는 통계적 유의미성 달성에 수개월 소요. ② A/B 테스팅은 "무엇이 더 좋은지"는 알려주지만 "왜 좋은지"는 알려주지 않음 → 정성 연구 병행 필요. ③ 상호 배타적으로 분리 불가능한 요소(예: 브랜드 이미지)는 A/B 테스팅 불적합.

**전제 조건**: 데이터 수집·분석 인프라(분석 플랫폼), 통계 기초 이해, 실험 결과에 따라 실제로 방향을 바꾸는 의사결정 문화.

📢 **섹션 요약 비유**: MVP와 A/B 테스팅은 바둑에서 정석보다 실전 대국으로 배우는 것—직접 두어보고 결과에서 배우는 것이 가장 빠르다.

### 👶 어린이를 위한 3줄 비유 설명

1. MVP는 새 게임을 만들기 전에 "이런 게임 재미있을 것 같아요"라고 친구들에게 간단한 버전 먼저 해보게 하는 거예요.
2. A/B 테스팅은 두 가지 간식 중 어느 것이 더 맛있는지 반 친구 절반은 A를 먹고 나머지 절반은 B를 먹어서 투표하는 실험이에요.
3. 가장 중요한 것은 미리 "몇 점 이상이면 성공"을 정해두고 실험해야 한다는 거예요—나중에 결과를 보고 기준을 바꾸면 거짓말이 되거든요.
