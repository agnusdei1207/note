## 핵심 인사이트 (3줄 요약)

> 1. **본질**: ISMS-P (Information Security Management System - Personal Information, 정보보호 및 개인정보보호 관리체계) 인증의 위험 평가(Risk Assessment)는 조직의 정보 자산에 대한 위협·취약점·영향도를 체계적으로 분석하여 수용 가능한 보안 수준을 결정하는 핵심 절차다.
> 2. **가치**: 위험 평가는 보안 투자의 우선순위를 결정하는 과학적 근거를 제공하며, "무엇을 먼저 보호해야 하는가?"라는 질문에 데이터 기반의 답을 제시한다.
> 3. **판단 포인트**: 기술사 서술에서는 ISMS-P 위험 평가의 5단계(자산 식별→위협 식별→취약점 분석→위험도 산정→대응 전략 결정), 정성·정량 방법론 비교, 잔여 위험(Residual Risk) 수용 판단을 구체적으로 제시해야 한다.

## Ⅰ. 개요 및 필요성

ISMS-P (Information Security Management System - Personal Information) 인증은 한국인터넷진흥원(KISA, Korea Internet & Security Agency)이 운영하는 국내 정보보호 관리체계 인증제도다. 기존 ISMS와 PIMS (개인정보보호 관리체계)가 2018년 통합된 것으로, 정보보호 102개 항목 + 개인정보 21개 항목으로 구성된다.

위험 평가(Risk Assessment)는 ISMS-P 인증의 핵심 요구사항 중 하나로, 단순 체크리스트 점검이 아니라 **조직 맞춤형 위험 관리 체계**를 갖추었음을 증명하는 과정이다. ISMS-P 심사에서 위험 평가 방법론의 적정성과 결과의 합리성이 집중 심사된다.

위험 평가의 필요성은 두 가지 현실 문제에서 출발한다. 첫째, 보안 예산은 한정되어 있어 모든 위험에 동일한 수준으로 대응할 수 없다. 위험 평가는 "어디에 돈을 써야 하는가"를 결정한다. 둘째, 완벽한 보안은 불가능하므로 어떤 위험은 수용(Accept)해야 한다—이 판단의 근거가 위험 평가다.

ISO 27005 (정보보안 위험 관리 국제 표준)와 NIST SP 800-30은 위험 평가의 국제 표준 프레임워크로, ISMS-P 위험 평가 방법론 설계의 근거가 된다.

📢 **섹션 요약 비유**: 위험 평가는 집 안 점검표처럼—어느 창문이 잠금장치가 약한지, 어느 문이 가장 자주 쓰이는지 파악해서 방범 예산을 어디에 써야 할지 결정하는 과정이다.

## Ⅱ. 아키텍처 및 핵심 원리

### ISMS-P 위험 평가 5단계 절차

```
┌──────────────────────────────────────────────────────────────────────┐
│                   ISMS-P 위험 평가 5단계                             │
├────────┬─────────────────────────────────────────────────────────────┤
│ 단계   │  활동 및 산출물                                             │
├────────┼─────────────────────────────────────────────────────────────┤
│  1     │  자산 식별 (Asset Identification)                           │
│        │  · 정보 자산 목록화: 하드웨어, 소프트웨어, 데이터, 인력    │
│        │  · 자산 가치 산정 (기밀성·무결성·가용성 기준)              │
│        │  · 산출물: 정보 자산 목록, 자산 중요도 등급               │
├────────┼─────────────────────────────────────────────────────────────┤
│  2     │  위협 식별 (Threat Identification)                          │
│        │  · 외부 위협: 해킹, 악성코드, DDoS, 피싱                  │
│        │  · 내부 위협: 내부자 정보 유출, 실수·과실                 │
│        │  · 환경 위협: 자연재해, 전원 장애, 화재                   │
│        │  · 산출물: 위협 목록, 위협 발생 가능성 점수               │
├────────┼─────────────────────────────────────────────────────────────┤
│  3     │  취약점 분석 (Vulnerability Analysis)                       │
│        │  · 기술적: 미패치 취약점, 설정 오류, 암호화 미적용        │
│        │  · 관리적: 접근 통제 미흡, 교육 부재, 정책 노후화         │
│        │  · 물리적: 출입 통제 미흡, CCTV 사각지대                  │
│        │  · 산출물: 취약점 목록, 취약점 심각도 점수                │
├────────┼─────────────────────────────────────────────────────────────┤
│  4     │  위험도 산정 (Risk Calculation)                             │
│        │  위험도 = 자산 가치 × 위협 가능성 × 취약점 심각도         │
│        │  · 5×5 위험 매트릭스 또는 정량적 산정                     │
│        │  · 위험 수준 분류: 高(5~15), 中(3~9), 低(1~6)            │
│        │  · 산출물: 위험 매트릭스, 위험 우선순위 목록              │
├────────┼─────────────────────────────────────────────────────────────┤
│  5     │  대응 전략 결정 (Risk Treatment)                            │
│        │  · 경감(Reduce): 통제 강화                                │
│        │  · 회피(Avoid): 해당 활동 중단                            │
│        │  · 이전(Transfer): 보험·아웃소싱                          │
│        │  · 수용(Accept): 잔여 위험 수용 (경영진 승인)             │
│        │  · 산출물: 위험 처리 계획, 잔여 위험 목록                 │
└────────┴─────────────────────────────────────────────────────────────┘
```

### 위험 매트릭스 (5×5)

| 발생 가능성 \ 영향도 | 매우 낮음(1) | 낮음(2) | 중간(3) | 높음(4) | 매우 높음(5) |
|:---:|:---:|:---:|:---:|:---:|:---:|
| **매우 높음(5)** | 5 | 10 | 15 | 20 | 25 |
| **높음(4)** | 4 | 8 | 12 | 16 | 20 |
| **중간(3)** | 3 | 6 | 9 | 12 | 15 |
| **낮음(2)** | 2 | 4 | 6 | 8 | 10 |
| **매우 낮음(1)** | 1 | 2 | 3 | 4 | 5 |

*15 이상: 高 위험 (즉시 조치), 8~14: 中 위험 (계획적 조치), 1~7: 低 위험 (수용 또는 모니터링)*

📢 **섹션 요약 비유**: 위험 매트릭스는 태풍 예보 지도처럼—강도와 경로 확률을 곱해서 어느 지역이 얼마나 위험한지 색깔로 보여준다.

## Ⅲ. 비교 및 연결

### 정성적 vs. 정량적 위험 평가

| 비교 항목 | 정성적 (Qualitative) | 정량적 (Quantitative) |
|:---|:---|:---|
| 방법 | 5×5 매트릭스, 전문가 판단 | ALE = SLE × ARO (연간 기대 손실) |
| 장점 | 빠르고 실용적, 데이터 부족 시 유용 | 경제적 관점 제공, 투자 대비 효과 계산 |
| 단점 | 주관성, 재현 어려움 | 데이터 수집 어려움, 시간·비용 높음 |
| 적합 상황 | 중소기업, 초기 위험 평가 | 금융기관, 임계 자산 |

*ALE (Annual Loss Expectancy), SLE (Single Loss Expectancy), ARO (Annual Rate of Occurrence)*

### ISMS-P와 ISO 27001 위험 평가 비교

| 비교 항목 | ISMS-P | ISO 27001 |
|:---|:---|:---|
| 적용 범위 | 국내 의무 대상 기관 | 글로벌 자발적 인증 |
| 위험 평가 방법 | 방법론 자율 선택 (결과 검증) | 방법론 자율 선택 (결과 검증) |
| 개인정보 특화 | PIMS 통합 (21개 항목 추가) | GDPR 요구사항과 별도 매핑 필요 |
| 인증 기관 | KISA | BSI, Bureau Veritas 등 |

📢 **섹션 요약 비유**: ISMS-P와 ISO 27001은 같은 보안 목표를 가진 국가대표 기준과 국제 기준의 관계—방향은 같지만 세부 요구사항이 다르다.

## Ⅳ. 실무 적용 및 기술사 판단

### 전자상거래 기업 ISMS-P 위험 평가 사례

**배경**: 회원 1,200만 명 보유 이커머스 플랫폼 ISMS-P 최초 인증.

**자산 분류 결과**: 총 142개 정보 자산 → 중요도 高 37개, 中 65개, 低 40개

**상위 위험 사례 (위험도 20 이상)**:
- **회원 개인정보 DB**: 자산 가치 5 × 해킹 위협 5 × SQL 인젝션 취약점 4 = 위험도 20 → **경감**: WAF 도입, 접근 통제 강화, 암호화 적용
- **결제 서버**: 자산 가치 5 × 내부자 위협 4 × 접근 로그 미흡 4 = 위험도 20 → **경감**: 특권 접근 관리(PAM) 솔루션 도입

**잔여 위험 수용 결정 사례**:
- 오래된 레거시 API 서버 취약점 → 즉시 패치 불가(서비스 영향) → 위험도 12, 임시 보완 통제(네트워크 격리) 적용 후 CTO 승인으로 6개월 수용

### 의사결정 포인트
- **위험 수용(Accept) 판단의 원칙**: 잔여 위험은 반드시 경영진 승인 + 이유 문서화 + 차기 평가 시 재검토 조건 붙여야 한다. "모르고 수용"은 위험 평가의 실패다.
- **위험 평가 주기**: 최초 평가 후 연 1회 이상 정기 재평가 + 중요한 시스템 변경 시 즉시 재평가.

📢 **섹션 요약 비유**: 잔여 위험 수용은 약을 먹지 못하는 환자처럼—완전 치료는 안 되지만 의사의 판단(경영진 승인)과 보완 요법(임시 통제)을 적용하면 관리 가능하다.

## Ⅴ. 기대효과 및 결론

### 기대효과

| 영역 | 효과 |
|:---|:---|
| 보안 투자 최적화 | 위험 우선순위 기반 예산 배분으로 ROI 향상 |
| 규제 준수 | ISMS-P·PIPA·전자금융감독규정 인증 기반 확보 |
| 사고 예방 | 고위험 자산 사전 통제로 침해사고 발생 감소 |
| 의사결정 근거 | 경영진의 보안 투자 승인 객관적 근거 제공 |
| 보험 비용 | 위험 관리 체계 증명으로 사이버 보험 할증 감소 |

### 한계 및 전제 조건

**한계**: ① 위험 평가는 특정 시점의 스냅샷—환경 변화 시 즉시 구식이 됨. ② 정성적 방법의 주관성—평가자 역량에 따라 결과 편차 발생. ③ 위협 인텔리전스 부족으로 신종 위협 반영 어려움.

**전제 조건**: 정보 자산 목록 최신화, 위험 평가 전문 인력(정보보안 관리사 등) 확보, 경영진의 잔여 위험 수용 결정 권한 명확화.

📢 **섹션 요약 비유**: 위험 평가는 자동차 정기검사처럼—한 번 통과했다고 영원히 안전한 게 아니라 정기적으로 다시 점검해야 한다.

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| ISO 27005 | 표준 근거 | 정보보안 위험 관리 국제 표준 |
| KISA (한국인터넷진흥원) | 인증 기관 | ISMS-P 인증 운영 기관 |

| 위협 인텔리전스 (Threat Intelligence) | 입력 데이터 | 최신 위협 동향 반영 위한 외부 데이터 |

### 👶 어린이를 위한 3줄 비유 설명

1. 위험 평가는 소풍 전날 날씨 확인하는 것처럼—비가 올 가능성(위협)이 얼마나 높고, 우산이 있는지(취약점), 소풍이 얼마나 중요한지(자산 가치)를 모두 고려해서 결정해요.
2. 위험도가 높으면 소풍을 미루거나 실내 대안을 준비하고(경감), 위험도가 낮으면 그냥 가면서 우산만 챙겨요(수용).
3. 매번 소풍 전에 다시 날씨를 확인하듯이, 위험 평가도 환경이 바뀔 때마다 다시 해야 해요.
