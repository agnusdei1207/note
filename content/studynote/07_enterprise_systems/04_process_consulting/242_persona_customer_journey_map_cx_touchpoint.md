## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 페르소나(Persona)는 실제 데이터 기반 가상 사용자 프로필이고, 고객 여정 지도(CJM, Customer Journey Map)는 페르소나가 서비스를 경험하는 전 단계를 시각화하여 CX (Customer Experience) 개선 기회를 발굴하는 도구다.
> 2. **가치**: 터치포인트(Touchpoint) 분석을 통해 고객의 감정 저점(Pain Point)과 고점(Gain Point)을 파악하고, 제한된 자원을 가장 큰 영향력을 가진 접점 개선에 집중할 수 있다.
> 3. **판단 포인트**: 기술사 서술에서는 페르소나 작성 원칙(데이터 기반·가상화·대표성)과 CJM의 5개 핵심 행(행동·생각·감정·터치포인트·기회)을 구조적으로 제시해야 한다.

## Ⅰ. 개요 및 필요성

디지털 서비스 설계에서 가장 흔한 오류는 "평균적 사용자"를 가정하는 것이다. 현실에서 평균적 사용자는 존재하지 않으며, 다양한 니즈·맥락·기술 친숙도를 가진 집단이 존재한다. 페르소나(Persona) 기법은 이 다양성을 관리 가능한 3~5개의 대표 사용자 프로필로 구조화한다.

페르소나는 Alan Cooper가 1998년 소개한 이후 UX (User Experience) 디자인의 핵심 도구로 자리잡았다. 핵심은 **실제 인터뷰·관찰·데이터에서 도출**해야 한다는 것이다. 직관적으로 만든 가상 사용자는 팀의 편견을 반영할 뿐이다.

고객 여정 지도(CJM, Customer Journey Map)는 페르소나를 동적으로 만드는 도구다. 정적 프로필인 페르소나에 시간 축을 추가하여, 고객이 인지→탐색→구매→사용→지지하는 전 과정에서 어떤 감정을 느끼는지 시각화한다. 특히 IT 서비스에서는 앱 접속→기능 탐색→오류 경험→지원 요청→이탈이라는 여정 전체를 한 페이지에 담아 팀 간 공유를 가능하게 한다.

CX (Customer Experience) 터치포인트 도출은 이 두 도구의 결합으로 가능하다—어떤 페르소나가, 어떤 단계에서, 어떤 접점(앱·이메일·콜센터·오프라인)을 통해, 어떤 감정을 경험하는지 매핑함으로써 개선 우선순위를 결정한다.

📢 **섹션 요약 비유**: 페르소나는 배우 캐릭터 설정이고, 고객 여정 지도는 그 배우의 영화 스크립트다.

## Ⅱ. 아키텍처 및 핵심 원리

### 페르소나 카드 구조

```
┌─────────────────────────────────────────────────────────────────┐
│                     페르소나 카드 구조                          │
├──────────────────────┬──────────────────────────────────────────┤
│  기본 정보           │  이름: 김민지 (35세, 직장인)             │
│                      │  직업: 중소기업 인사팀장                 │
│                      │  기술 친숙도: 중급 (스마트폰 능숙)       │
├──────────────────────┼──────────────────────────────────────────┤
│  목표 (Goals)        │  · 빠른 결재 처리로 업무 시간 단축       │
│                      │  · 모바일에서도 동일한 편의성            │
├──────────────────────┼──────────────────────────────────────────┤
│  불만 (Frustrations) │  · 첨부파일 업로드 오류 빈발             │
│                      │  · 결재선 설정이 매번 초기화됨           │
├──────────────────────┼──────────────────────────────────────────┤
│  인용문 (Quote)      │  "결재 하나 하려고 왜 이렇게 오래 걸려?" │
├──────────────────────┼──────────────────────────────────────────┤
│  데이터 근거         │  인터뷰 15건 중 8건에서 동일 불만 언급   │
└──────────────────────┴──────────────────────────────────────────┘
```

### 고객 여정 지도 (CJM) 구조

| 여정 단계 | 인지 | 탐색 | 시도 | 사용 | 지지/이탈 |
|:---|:---|:---|:---|:---|:---|
| **행동** | 광고 클릭 | 기능 검색 | 가입·로그인 | 일상 사용 | 리뷰 작성 |
| **생각** | "써볼 만한가?" | "이 기능이 맞나?" | "왜 안 되지?" | "빠르긴 하다" | "추천할까?" |
| **감정** | 😐 중립 | 😕 혼란 | 😠 좌절 | 😊 만족 | 😃 기쁨/이탈 |
| **터치포인트** | SNS 광고 | 도움말 페이지 | 앱 온보딩 | 대시보드 | 이메일 설문 |
| **기회** | A/B 광고 | 검색 개선 | 온보딩 간소화 | UI 개선 | 넛지 알림 |

### 터치포인트 유형 분류

| 유형 | 예시 | 관리 주체 |
|:---|:---|:---|
| 디지털 접점 | 앱·웹·이메일·알림 | IT/UX 팀 |
| 대인 접점 | 콜센터·현장 직원 | CS 팀 |
| 물리적 접점 | 매장·포장·인쇄물 | 마케팅 팀 |
| 간접 접점 | 리뷰·SNS·구전 | 브랜드 팀 |

📢 **섹션 요약 비유**: CJM은 고객이 서비스를 경험하는 여행 일정표—어디서 즐겁고 어디서 지쳤는지 한눈에 보인다.

## Ⅲ. 비교 및 연결

### 페르소나 vs. 세그멘테이션 (Segmentation)

| 비교 항목 | 페르소나 | 시장 세그멘테이션 |
|:---|:---|:---|
| 초점 | 개인의 동기·행동·맥락 | 인구통계·구매행동 집단 |
| 활용 목적 | 설계·개발 결정 가이드 | 마케팅 타기팅 |
| 표현 형식 | 이야기형 서사 프로필 | 통계 데이터 테이블 |
| 팀 공감 유발 | 높음 (이름·얼굴·인용문) | 낮음 (숫자 중심) |

### CJM과 서비스 블루프린트 연계

CJM이 고객 시각에서 경험을 묘사한다면, 서비스 블루프린트(Service Blueprint)는 프론트스테이지(Front Stage, 고객 가시 영역)와 백스테이지(Back Stage, 내부 프로세스)를 함께 매핑한다. CJM → 서비스 블루프린트 순서로 작성하면 고객 경험 개선이 내부 프로세스 어느 지점과 연결되는지 파악할 수 있다.

📢 **섹션 요약 비유**: CJM이 레스토랑 손님의 식사 경험 일지라면, 서비스 블루프린트는 주방과 홀의 운영 매뉴얼이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 전자정부 민원 서비스 CJM 적용 사례

**배경**: 정부24 민원신청 프로세스의 완료율 54% (절반이 중간에 포기).

**페르소나 도출**: 인터뷰 30건 분석 → 3개 페르소나 확정
- P1: 고령 비디지털 민원인 (65세+, 자녀 도움 필요)
- P2: 바쁜 직장인 민원인 (35~45세, 모바일 선호)
- P3: 대리인 처리 민원인 (대표자 위임 업무)

**CJM 터치포인트 감정 분석 (P2 기준)**:
- 검색: 😊 쉽게 접속 → 서류 목록 확인: 😠 복잡한 서류 체계 → 파일 첨부: 😠 반복 오류 → 제출 완료: 😐 확인 불명확

**개선 기회 도출**:
1. 서류 자동 판단 AI 도입 (필요 서류 자동 추천)
2. 첨부파일 오류 즉시 피드백 및 형식 가이드 제공
3. 제출 완료 상태 실시간 알림 강화

**결과**: 개선 후 완료율 54% → 79%, 콜센터 문의 32% 감소.

### 의사결정 포인트
- **페르소나 수**: 3~5개가 적정. 너무 많으면 집중 설계 불가, 너무 적으면 다양성 미반영.
- **CJM 갱신 주기**: 서비스 주요 업데이트마다 재검토. 정적 문서로 방치하면 의미 소실.
- **감정 곡선 낮은 구간**: 이 구간이 최우선 개선 대상—감정 저점은 이탈 트리거다.

📢 **섹션 요약 비유**: 감정 곡선의 최저점을 찾는 것은 배관에서 누수 지점 찾기—거기를 막아야 전체 시스템이 살아난다.

## Ⅴ. 기대효과 및 결론

### 기대효과

| 영역 | 효과 |
|:---|:---|
| 제품 설계 | 사용자 니즈 반영으로 기능 재작업 감소 |
| CS 비용 | 터치포인트 개선으로 문의 건수 감소 |
| 전환율 | 여정 장애 제거로 이탈률 20~40% 개선 |
| 조직 정렬 | 팀 간 고객 이해 공유로 협업 효율 향상 |

### 한계 및 전제 조건

**한계**: ① 데이터 기반 없이 추측으로 만든 페르소나는 편견 강화. ② CJM은 현재 상태(As-Is) 기술이 목적이 아니라 개선(To-Be) 도출이 목적임을 망각하면 분석 과잉. ③ 다채널 환경에서 여정 단절 지점 추적은 데이터 통합 인프라 필요.

**전제 조건**: 실사용자 인터뷰 최소 15~30건 데이터, 다기능 팀(UX·비즈니스·기술·CS) 참여, 경영진 의사결정 반영 채널 확보.

📢 **섹션 요약 비유**: 페르소나 없는 CJM은 배우 없는 영화 대본—내용이 있어도 누가 연기할지 모른다.

### 👶 어린이를 위한 3줄 비유 설명

1. 페르소나는 학교에서 발표할 때 "제 친구 민지는..."처럼 실제 사람처럼 이름을 붙인 상상 속 고객이에요.
2. 고객 여정 지도는 민지가 서비스를 쓸 때 기쁜 순간과 짜증나는 순간을 쭉 이어 그린 그림이에요.
3. 짜증나는 순간을 고치면 민지처럼 비슷한 사람들이 모두 행복해지는 것이 CJM의 목표예요.
