## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 데이터 거버넌스 협의체(DGC, Data Governance Council)는 전사 데이터 자산의 품질·보안·활용을 관리하는 의사결정 기구로, 데이터를 기업의 전략 자산으로 다루는 거버넌스 체계의 핵심이다.
> 2. **가치**: DGC는 부서별로 분산된 데이터 관리 관행을 표준화하고, 데이터 오너십(Data Ownership) 분쟁을 조율하며, 데이터 기반 의사결정(DDDM, Data-Driven Decision Making) 문화를 조직 전체에 확산한다.
> 3. **판단 포인트**: 기술사 서술에서는 DGC의 3계층 구조(전략·전술·운영), 역할별 책임(RACI), 핵심 운영 규칙(메타데이터 표준·데이터 품질 KPI·정책 수립 절차)을 구체적으로 제시해야 한다.

## Ⅰ. 개요 및 필요성

데이터 거버넌스(Data Governance)는 데이터의 수집·저장·처리·활용·폐기 전 생애주기(Data Lifecycle)에 걸쳐 일관된 정책, 표준, 책임 체계를 수립하는 관리 체계다. 이를 실행하는 공식 기구가 DGC (Data Governance Council)이다.

DGC의 필요성은 데이터 경제 시대에 더욱 커졌다. 기업 내 데이터 문제의 대표적 증상은 다음과 같다: ① 부서마다 동일 지표 수치가 다름(데이터 불일치), ② 데이터 사용 전 어느 부서에 허가를 받아야 하는지 불명확(데이터 사일로), ③ 개인정보 처리 기준이 부서마다 달라 규제 위반 위험(컴플라이언스 리스크), ④ AI·분석 프로젝트가 데이터 품질 문제로 지연(데이터 품질 부재).

DGC는 이 문제들을 **조직적으로** 해결하는 기구다. 기술 솔루션(DW, 데이터 카탈로그 등)만으로는 데이터 문제를 해결할 수 없다—사람과 프로세스의 변화가 핵심이기 때문에 협의체 운영이 필수적이다.

국내에서는 PIPA (Personal Information Protection Act, 개인정보보호법) 강화, GDPR (General Data Protection Regulation, EU 일반 데이터 보호 규정) 대응, 금융·의료 데이터 규제 준수를 위해 DGC 운영이 대기업과 공공기관을 중심으로 확산되고 있다.

📢 **섹션 요약 비유**: DGC는 회사의 데이터 국회—데이터를 어떻게 쓰고, 누가 책임지고, 어떤 규칙을 만들지 의결하는 기구다.

## Ⅱ. 아키텍처 및 핵심 원리

### DGC 3계층 구조

```
┌─────────────────────────────────────────────────────────────────┐
│              데이터 거버넌스 협의체 3계층 구조                  │
├──────────────────────────────────────────────────────────────┤
│  계층 1: 전략 (Strategic)                                      │
│  · DGC (Data Governance Council) — C-Level 위원회             │
│  · CDO (Chief Data Officer) 또는 CIO 주관                     │
│  · 데이터 전략·정책·예산·우선순위 결정                         │
├──────────────────────────────────────────────────────────────┤
│  계층 2: 전술 (Tactical)                                       │
│  · 데이터 스튜어드십 위원회 (Data Stewardship Committee)       │
│  · 부서 대표 데이터 스튜어드(Data Steward) 구성               │
│  · 표준 정의·메타데이터 관리·품질 기준 수립                    │
├──────────────────────────────────────────────────────────────┤
│  계층 3: 운영 (Operational)                                    │
│  · 데이터 오너 (Data Owner): 도메인별 책임자                   │
│  · 데이터 스튜어드 (Data Steward): 일상 데이터 관리            │
│  · 데이터 쿠레이터 (Data Curator): 메타데이터 업데이트         │
└─────────────────────────────────────────────────────────────────┘
```

### DGC RACI 역할 정의

| 활동 | CDO | 데이터 오너 | 데이터 스튜어드 | IT/DBA |
|:---|:---:|:---:|:---:|:---:|
| 데이터 정책 수립 | A | C | R | I |
| 메타데이터 표준 승인 | A | C | R | I |
| 데이터 품질 모니터링 | I | A | R | C |
| 데이터 접근 권한 승인 | I | A | C | R |
| 개인정보 처리 기준 | A | R | C | I |

*R=Responsible, A=Accountable, C=Consulted, I=Informed*

### DGC 운영 사이클

| 주기 | 활동 |
|:---|:---|
| 주간 | 데이터 품질 이슈 트래킹, 데이터 스튜어드 오피스아워 |
| 월간 | 데이터 품질 KPI 리뷰, 신규 데이터 정의 검토 |
| 분기 | 데이터 거버넌스 성숙도 평가, 정책 개정 검토 |
| 연간 | 데이터 전략 수립, 거버넌스 프레임워크 재검토 |

📢 **섹션 요약 비유**: DGC 3계층은 국가 거버넌스처럼—국회(전략)가 법을 만들고, 정부(전술)가 규칙을 정하고, 공무원(운영)이 집행한다.

## Ⅲ. 비교 및 연결

### 데이터 거버넌스 성숙도 모델

| 수준 | 특징 | DGC 상태 |
|:---|:---|:---|
| 레벨 1: 혼돈 | 표준 없음, 개인 역량 의존 | DGC 부재 |
| 레벨 2: 인식 | 일부 표준, 비공식 관리 | 임시 협의체 |
| 레벨 3: 관리 | 공식 표준, 메타데이터 관리 시작 | DGC 설립·운영 |
| 레벨 4: 최적화 | 자동화된 품질 모니터링 | DGC + DataOps 통합 |
| 레벨 5: 선도 | 데이터 문화 내재화 | DDDM 완전 정착 |

### DGC와 DAMA (Data Management Association) DMBOK 연계

DAMA (Data Management Association)의 DMBOK (Data Management Body of Knowledge)은 데이터 거버넌스를 11개 지식 영역의 중심에 두고 있다. DGC 운영은 DMBOK의 데이터 거버넌스 영역(지식 영역 1)을 실천하는 조직 구조다.

📢 **섹션 요약 비유**: DMBOK가 데이터 관리의 교과서라면, DGC는 그 교과서를 실행하는 학교 행정 조직이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 금융권 DGC 운영 사례

**배경**: K금융그룹 계열사 7개의 데이터 표준이 제각각이어서 그룹 통합 보고서 작성 시 매월 3일 수작업 정합 필요.

**DGC 설립 단계**:
1. CDO 임명 및 그룹 데이터 거버넌스 기본 정책 수립
2. 계열사별 데이터 오너 지정 (CFO, CMO 등 임원급)
3. 데이터 스튜어드 68명 선발 및 교육
4. 공통 데이터 사전(Business Data Dictionary) 구축

**성과 (12개월 후)**:
- 그룹 공통 지표 정의 표준화: 142개 → 전사 단일 정의
- 월간 통합 보고서 작성 소요일: 3일 → 0.5일
- 데이터 품질 이슈 발생 건수: 월 47건 → 월 12건
- PIPA 감사 대응 소요 시간: 2주 → 3일

### 의사결정 포인트
- **DGC 실패의 주요 원인**: 비즈니스 부서의 참여 부재, IT 주도의 기술 프로젝트로만 인식될 때. DGC는 반드시 비즈니스 리더십이 주도해야 한다.
- **데이터 오너십 분쟁 중재**: 고객 데이터가 영업·마케팅·CS 3개 부서 모두에 걸칠 때 → CDO 주관 중재, 도메인 분할(Master Data Management) 정책 적용.

📢 **섹션 요약 비유**: DGC에서 비즈니스 리더 참여 없이 IT만 운영하면 건축 규정은 있는데 건물주가 없는 아파트와 같다—아무도 실제로 책임지지 않는다.

## Ⅴ. 기대효과 및 결론

### 기대효과

| 영역 | 효과 |
|:---|:---|
| 데이터 품질 | 표준화·모니터링으로 데이터 오류 60~80% 감소 |
| 의사결정 속도 | 신뢰할 수 있는 단일 데이터 소스로 분석 시간 단축 |
| 규제 대응 | PIPA·GDPR 감사 준비 시간 50~70% 절감 |
| AI/분석 가치 | 고품질 데이터 제공으로 AI 모델 정확도 향상 |
| 조직 문화 | 데이터 기반 의사결정(DDDM) 문화 정착 |

### 한계 및 전제 조건

**한계**: ① 초기 설립 비용(인력·도구·교육) 투자 필요. ② 변화 저항(현업 부서의 데이터 통제 거부). ③ CDO·DGC 권한이 약하면 형식적 기구로 전락.

**전제 조건**: CEO·CFO의 DX 전략적 의지, CDO 직위 신설 및 실질적 권한 부여, 데이터 카탈로그·품질 도구 등 기술 인프라 투자.

📢 **섹션 요약 비유**: DGC는 헌법 없이는 작동하지 않는 법원처럼—경영진의 데이터 전략 의지라는 헌법이 먼저 있어야 DGC가 힘을 발휘한다.

### 📌 관련 개념 맵

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| CDO (Chief Data Officer) | 리더십 역할 | DGC를 주관하는 최고 데이터 책임자 |
| MDM (Master Data Management) | 기술 수단 | 핵심 데이터 단일 정의 관리 시스템 |
| DAMA DMBOK | 표준 프레임워크 | 데이터 관리 지식 체계, DGC 설계 근거 |
| 데이터 카탈로그 (Data Catalog) | 운영 도구 | 메타데이터 검색·관리 플랫폼 |
| PIPA / GDPR | 규제 요구 | DGC 필요성의 외부 동인 |
| DataOps | 자동화 연계 | DGC 정책을 데이터 파이프라인에 자동 반영 |

### 👶 어린이를 위한 3줄 비유 설명

1. DGC는 학교 도서관 관리 위원회처럼—책(데이터)을 어디에 두고, 누가 빌릴 수 있고, 어떻게 정리할지 규칙을 만드는 모임이에요.
2. 선생님(CDO)이 전체 규칙을 정하고, 학급 대표(데이터 오너)가 자기 반 책장을 관리하고, 도서 위원(데이터 스튜어드)이 날마다 정리해요.
3. 이 위원회 없이는 모든 책이 뒤섞여서 필요한 책을 찾는 데 하루 종일 걸릴 수도 있어요.
