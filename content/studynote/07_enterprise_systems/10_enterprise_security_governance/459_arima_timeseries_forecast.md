## 핵심 인사이트 (3줄 요약)

> 1. **본질**: ARIMA(AutoRegressive Integrated Moving Average)는 시계열 데이터의 과거 자신(AR), 차분(I), 과거 오차(MA)를 결합하여 미래를 예측하는 통계 모델로, 데이터가 정상성(Stationarity)을 만족할 때 최적 성능을 발휘한다.
> 2. **가치**: 매출 예측·서버 트래픽 예측·이상 탐지 등 비즈니스 시계열 예측에서 딥러닝 모델 대비 훨씬 작은 데이터와 컴퓨팅 자원으로 해석 가능하고 신뢰할 수 있는 예측치를 제공한다.
> 3. **판단 포인트**: ARIMA 적용 전 정상성 검정(ADF Test)과 ACF/PACF 분석으로 p,d,q 파라미터를 결정하는 것이 핵심이며, 계절성이 있으면 SARIMA(Seasonal ARIMA)로 확장해야 한다.

## Ⅰ. 개요 및 필요성

기업의 수많은 의사결정은 "미래에 무슨 일이 일어날까?"에 대한 예측을 기반으로 한다. 재고 수량 결정, 인력 배치 계획, 마케팅 예산 배분 모두 수요 예측의 정확도에 좌우된다.

시계열 예측 모델은 크게 세 가지 접근법이 있다: **통계 기반**(ARIMA, Exponential Smoothing), **머신러닝 기반**(XGBoost, Random Forest), **딥러닝 기반**(LSTM, Transformer). 이 중 ARIMA는 가장 오래된(Box-Jenkins, 1970) 방법이지만, 데이터가 적고 해석 가능성이 중요한 비즈니스 환경에서 여전히 강력한 선택이다.

ARIMA의 핵심 가정은 **정상성(Stationarity)**: 시간에 따라 평균과 분산이 일정한 데이터여야 한다. 주가·매출 같은 비정상 시계열은 차분(Differencing)으로 정상화한 후 ARIMA를 적용한다.

📢 **섹션 요약 비유**: ARIMA는 마치 경험 많은 기상 예보관처럼, 어제의 날씨와 어제 예측의 오차를 학습하여 내일 날씨를 통계적으로 가장 합리적으로 예측하는 전통의 방법론이다.

## Ⅱ. 아키텍처 및 핵심 원리

### ARIMA(p, d, q) 파라미터 의미

| 파라미터 | 명칭 | 의미 | 결정 방법 |
|:---:|:---|:---|:---|
| **p** | AR 차수 | 과거 몇 시점까지의 값을 사용 | PACF 그래프 컷오프 |
| **d** | 차분 차수 | 정상성 확보를 위한 차분 횟수 | ADF 검정 결과 |
| **q** | MA 차수 | 과거 몇 시점까지의 오차를 사용 | ACF 그래프 컷오프 |

### ARIMA 모델링 프로세스

```
┌──────────────────────────────────────────────────────────────┐
│              ARIMA 모델링 Box-Jenkins 방법론                  │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1단계: 데이터 시각화                                         │
│     시계열 플롯 → 추세/계절성/불규칙 성분 확인               │
│                          │                                  │
│  2단계: 정상성 검정                                           │
│     ADF(Augmented Dickey-Fuller) 검정                        │
│     p-value > 0.05 → 비정상 → 차분 적용 (d 결정)           │
│                          │                                  │
│  3단계: ACF/PACF 분석                                         │
│     ACF: MA 차수 q 결정                                      │
│     PACF: AR 차수 p 결정                                     │
│                          │                                  │
│  4단계: 모델 적합                                             │
│     ARIMA(p,d,q) 파라미터로 MLE 추정                         │
│     또는 auto.arima()로 자동 최적 파라미터 선택              │
│                          │                                  │
│  5단계: 진단 및 검증                                          │
│     잔차 분석 (백색 잡음인지 확인)                           │
│     AIC/BIC 기준 모델 선택                                   │
│                          │                                  │
│  6단계: 예측                                                  │
│     미래 n 기간 예측 + 신뢰구간(95%) 제공                    │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 지수 평활(Exponential Smoothing)과의 비교

| 항목 | ARIMA | 지수 평활 (ETS) | Prophet |
|:---|:---|:---|:---|
| 해석 가능성 | 중간 | 높음 | 낮음 |
| 계절성 처리 | SARIMA 확장 | 기본 내장 | 자동 처리 |
| 자동화 | auto.arima | ETS() | 완전 자동 |
| 적합 데이터 | 선형 패턴 | 단순 추세 | 복잡한 계절성 |
| 개발사 | Box-Jenkins | Holt-Winters | Meta (Facebook) |

📢 **섹션 요약 비유**: ARIMA가 통계학자의 정밀 계산기라면, Prophet은 데이터 과학자의 스마트 자동 계산기처럼, 복잡한 계절성을 자동으로 처리하지만 내부 동작을 설명하기 어렵다.

## Ⅲ. 비교 및 연결

### 시계열 모델 선택 기준

| 조건 | 권장 모델 |
|:---|:---|
| 데이터 < 100개, 선형 패턴 | ARIMA |
| 강한 계절성 있음 | SARIMA, ETS |
| 공휴일·이벤트 효과 있음 | Prophet |
| 데이터 > 수만 개, 복잡한 패턴 | LSTM, Transformer |
| 해석 가능성 필수 (규제 보고용) | ARIMA |

### 이상 탐지(Anomaly Detection)에서의 ARIMA 활용

ARIMA 모델의 예측값과 실제값의 차이(잔차)가 3σ를 초과하면 이상으로 판단하는 방식으로, 서버 지연 시간 이상·매출 급감·IoT 센서 오동작 탐지에 활용된다.

📢 **섹션 요약 비유**: ARIMA 이상 탐지는 혈압 측정처럼, "정상 범위(예측 신뢰구간)" 밖으로 수치가 벗어나면 의사(알림 시스템)에게 즉시 보고하는 자동 건강 모니터링이다.

## Ⅳ. 실무 적용 및 기술사 판단

### IT 운영 지표 예측에서의 ARIMA 적용

1. **서버 트래픽 예측**: 일별 DAU를 SARIMA로 모델링하여 주간 계절성 반영, 피크 타임 사전 스케일아웃 자동화
2. **이상 탐지**: 응답 시간(p99) 예측값의 2σ 초과 시 PagerDuty 자동 경보
3. **용량 계획**: 6개월 트래픽 증가 추세 예측으로 IDC 용량 구매 계획 수립

```python
# Python statsmodels ARIMA 예시 (개념 표현)
from statsmodels.tsa.arima.model import ARIMA
from statsmodels.tsa.stattools import adfuller

# 1. 정상성 검정
result = adfuller(time_series_data)
print(f'ADF Statistic: {result[0]:.3f}, p-value: {result[1]:.3f}')

# 2. 모델 적합 (p=2, d=1, q=2)
model = ARIMA(time_series_data, order=(2, 1, 2))
fitted = model.fit()

# 3. 30일 예측 + 신뢰구간
forecast = fitted.get_forecast(steps=30)
pred_ci = forecast.conf_int(alpha=0.05)
```

📢 **섹션 요약 비유**: ARIMA 기반 용량 계획은 기상 예보관이 항공사에 "다음 주 태풍이 온다"고 알려줘 항공편을 미리 조정하는 것처럼, IT 관리자가 트래픽 폭증 전 미리 인프라를 준비하게 하는 예지 정비(Predictive Maintenance) 체계이다.

## Ⅴ. 기대효과 및 결론

**기대효과**:
- 재고 과적/부족 사전 방지 (MAPE 10% 이하 예측 정확도 시)
- IT 인프라 사전 확장으로 장애 예방 및 비용 최적화
- 규제 보고용 예측 모델의 해석 가능성 요건 충족

**한계 및 전제조건**:
- 갑작스러운 외부 충격(COVID, 블랙스완)은 ARIMA로 예측 불가
- 비선형·복잡한 패턴 데이터에서는 딥러닝 대비 성능 열위
- 파라미터 결정(p,d,q)에 도메인 전문성 필요
- 데이터 품질(결측치, 이상치) 전처리가 모델 성능에 직결

📢 **섹션 요약 비유**: ARIMA의 한계는 백미러(과거 데이터)만 보고 미래 도로를 예측하는 것처럼, 과거 패턴이 미래에도 지속된다는 가정이 깨지는 순간(블랙스완) 예측이 무력해진다.

### 👶 어린이를 위한 3줄 비유 설명

1. ARIMA는 "어제 기온이 10도였고, 그 전날 예측 오차가 1도였으니 오늘은 11도쯤 될 것 같아"라고 계산하는 날씨 예측 방법이에요.
2. 먼저 날씨 데이터가 규칙적인지(정상성) 확인하고, 불규칙하면 차분으로 정리한 뒤 예측을 시작해요.
3. 쉬운 날씨 예측에는 딱이지만, 태풍처럼 갑자기 오는 것(블랙스완)은 맞추기 어렵답니다.
