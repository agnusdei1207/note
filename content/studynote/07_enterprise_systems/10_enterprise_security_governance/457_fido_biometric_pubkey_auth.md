## 핵심 인사이트 (3줄 요약)

> 1. **본질**: FIDO(Fast Identity Online) 2.0은 비밀번호를 공개키 암호화 + 생체인증(지문/얼굴)으로 대체하는 표준으로, 생체정보는 절대 서버에 전송되지 않고 디바이스 내 TEE(신뢰 실행 환경)에만 저장되는 '로컬 검증' 아키텍처이다.
> 2. **가치**: 서버에 비밀번호 해시가 없으므로 데이터베이스 유출 시 크리덴셜 탈취가 원천 불가능하며, 피싱 공격도 도메인 바인딩으로 차단하여 비밀번호 기반 계정 탈취의 근본 문제를 해결한다.
> 3. **판단 포인트**: Passkey(패스키)는 FIDO2의 소비자 친화적 구현으로, Apple/Google/Microsoft가 플랫폼 차원에서 지원하면서 비밀번호 없는 미래(Passwordless)의 현실적 표준이 되었다.

## Ⅰ. 개요 및 필요성

매년 기업 보안 사고의 81%는 탈취된 비밀번호가 원인이다(Verizon DBIR). 비밀번호는 근본적으로 취약하다: 기억하기 어려워 동일 비밀번호를 재사용하고, 피싱 사이트에 입력되고, 서버 해킹으로 대량 유출된다.

FIDO Alliance(구글·애플·마이크로소프트·삼성 등 참여)는 비밀번호 없는 인증 표준을 개발했다. FIDO2는 WebAuthn(W3C 표준)과 CTAP(Client-to-Authenticator Protocol)의 조합으로, 웹 브라우저에서 생체인증 기반 로그인을 표준화한다.

핵심 혁신: 인증에 사용되는 개인키는 디바이스를 절대 떠나지 않으며, 서버는 공개키만 보관한다. 서버가 해킹당해도 공개키만 유출될 뿐 로그인에 사용할 수 없다.

📢 **섹션 요약 비유**: FIDO는 열쇠(개인키)를 집에 두고 자물쇠 설계도(공개키)만 은행에 맡기는 것처럼, 은행이 털려도 열쇠가 없어 내 집에는 들어올 수 없는 구조이다.

## Ⅱ. 아키텍처 및 핵심 원리

### FIDO2 인증 흐름

```
┌──────────────────────────────────────────────────────────────┐
│              FIDO2 등록 및 인증 흐름                          │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  [등록 단계]                                                  │
│  서버 ──▶ Challenge 발송 ──▶ 브라우저(WebAuthn API)          │
│                                    │                        │
│                         Authenticator (디바이스)             │
│                         - 지문/얼굴 인증 (로컬)              │
│                         - 공개키/개인키 쌍 생성              │
│                         - 개인키: TEE/Secure Enclave 저장   │
│                    공개키 + Attestation ──▶ 서버             │
│                                    서버: 공개키 저장         │
│                                                              │
│  [인증 단계]                                                  │
│  서버 ──▶ Challenge 발송 ──▶ 브라우저                        │
│                         Authenticator                       │
│                         - 지문 확인 (로컬)                   │
│                         - 개인키로 Challenge 서명            │
│                    서명된 응답 ──▶ 서버                      │
│                         서버: 공개키로 서명 검증 ✅           │
│                                                              │
│  핵심: 생체정보 서버 전송 없음, 개인키 디바이스 이탈 없음    │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### FIDO2 핵심 보안 속성

| 보안 속성 | 메커니즘 | 방어하는 공격 |
|:---|:---|:---|
| 피싱 저항 | 공개키가 특정 도메인에 바인딩 | 피싱 사이트에서 서명 불가 |
| 재전송 공격 방어 | Challenge-Response (1회성) | 캡처한 서명 재사용 불가 |
| 서버 해킹 무력화 | 공개키만 서버에 저장 | 유출된 공개키로 로그인 불가 |
| 생체정보 보호 | TEE/Secure Enclave 로컬 처리 | 생체정보 서버 전송/저장 없음 |

### Passkey(패스키)

FIDO2의 소비자 구현으로, 기기 분실 시에도 iCloud Keychain, Google Password Manager를 통해 동기화된 Passkey로 계속 로그인 가능. 2022년 Apple/Google/Microsoft 동시 발표로 업계 표준화.

📢 **섹션 요약 비유**: Passkey는 자동차 스마트키처럼, 버튼 한 번으로 문이 열리되(간편함) 내 폰(인증기)이 가까이 있을 때만 작동하는(보안성) 비접촉 인증 열쇠이다.

## Ⅲ. 비교 및 연결

### 인증 방식 보안 수준 비교

| 인증 방식 | 피싱 저항 | 서버 해킹 저항 | 사용 편의성 | 비고 |
|:---|:---:|:---:|:---:|:---|
| 패스워드 | ✗ | ✗ | 중간 | 80%+ 침해 원인 |
| OTP SMS | ✗ | ✅ | 낮음 | SIM Swap 공격 취약 |
| OTP TOTP | △ | ✅ | 낮음 | 피싱 실시간 중계 취약 |
| FIDO2 WebAuthn | ✅ | ✅ | 높음 | 최고 수준 보안 |
| Passkey | ✅ | ✅ | 최고 | 동기화 지원 |

### DID(탈중앙 신원)와의 연결

FIDO2의 공개키 인증 구조는 블록체인 기반 DID(Decentralized Identifier)의 기술적 기반과 유사하다. DID는 FIDO의 공개키 개념을 신원 체계 전반으로 확장한다.

📢 **섹션 요약 비유**: 비밀번호 vs FIDO2는 아날로그 열쇠 vs 생체인식 금고처럼, 열쇠(비밀번호)는 복사될 수 있지만 지문(FIDO2)은 복사가 사실상 불가능하다.

## Ⅳ. 실무 적용 및 기술사 판단

### 기업 FIDO2 도입 전략

1. **단계별 전환**: 패스워드 완전 제거 전 FIDO2를 MFA 수단으로 먼저 도입 (Password + FIDO2)
2. **디바이스 관리 통합**: MDM(Mobile Device Management)으로 기업 디바이스의 Authenticator 라이프사이클 관리
3. **FIDO2 서버 구축 vs 외부 IdP**: 내부 FIDO2 서버(예: Yubico, ForgeRock)를 구축하거나 Okta·Azure AD의 Passkey 지원 활용
4. **레거시 시스템 전환**: 웹 기반 시스템은 WebAuthn API로 단기간 전환 가능, 클라이언트-서버 레거시는 CTAP2 기반 하드웨어 키(YubiKey) 활용
5. **사용자 교육**: Passkey 분실 시 복구 절차(백업 인증기, 이메일 복구) 명확화

📢 **섹션 요약 비유**: FIDO2 기업 도입은 회사 출입카드를 생체인식 시스템으로 교체하는 것처럼, 기존 카드(비밀번호) 없이 손가락(지문) 하나로 모든 시스템에 입장하는 제로 마찰 보안 체계이다.

## Ⅴ. 기대효과 및 결론

**기대효과**:
- 피싱·크리덴셜 스터핑 공격 100% 방어 (피싱 저항 내장)
- 비밀번호 관련 헬프데스크 비용 50% 절감
- 로그인 마찰 감소로 이탈률 개선 (패스워드 재설정 제거)
- GDPR/개인정보보호법: 생체정보 비전송으로 규제 준수 용이

**한계 및 전제조건**:
- 디바이스 분실 시 Passkey 복구 절차 명확화 필요
- 레거시 시스템의 WebAuthn 지원 개발 공수
- 모든 직원 디바이스가 생체인증 하드웨어 지원 필요 (구형 기기 교체)
- 기업 환경에서 FIDO2 서버 및 MDM 통합의 초기 구성 복잡도

📢 **섹션 요약 비유**: Passkey의 미래는 마치 현금(비밀번호) 없는 사회처럼, 지갑(기억) 없이도 생체인식 하나로 모든 거래와 접근이 가능한 완전한 디지털 신원 사회를 향한 발걸음이다.

| 개념 | 관계 | 설명 |

### 👶 어린이를 위한 3줄 비유 설명

1. FIDO는 열쇠(비밀번호) 대신 내 지문으로 문을 여는 것처럼, "내가 나임을 증명"하는 새로운 방법이에요.
2. 지문 정보는 절대 서버에 가지 않고 내 폰 안에만 있어서, 서버가 해킹당해도 지문은 안전해요.
3. 패스키(Passkey)는 그 지문을 여러 기기에서 쓸 수 있게 클라우드로 동기화해주는 스마트 열쇠 보관함이에요.
