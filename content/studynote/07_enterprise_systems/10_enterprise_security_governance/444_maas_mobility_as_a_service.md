## 핵심 인사이트 (3줄 요약)

> 1. **본질**: MaaS(Mobility as a Service)는 대중교통, 공유 모빌리티(자전거·킥보드·카셰어링), 택시를 단일 플랫폼에서 계획·예약·결제하는 통합 모빌리티 서비스로, 개인 차량 소유의 대안 패러다임이다.
> 2. **가치**: 도시 교통 혼잡과 탄소 배출을 감소시키면서, 다양한 교통 수단의 최적 조합(멀티모달 라우팅)을 알고리즘이 자동 추천하여 이동 비용과 시간을 동시에 절감한다.
> 3. **판단 포인트**: MaaS 성패는 교통 데이터 개방(GTFS, NeTEx) 수준과 통합 결제(MaaS Operator 면허) 생태계 형성 여부에 달려 있으며, 기술보다 정책·법제 정비가 더 큰 장벽이다.

## Ⅰ. 개요 및 필요성

핀란드 헬싱키의 Whim 앱에서 세계 최초로 상용화된 MaaS는 스마트폰 하나로 지하철 승차권, 따릉이 대여, 카셰어링 예약, 택시 호출을 단일 앱에서 처리하는 서비스이다. 사용자는 이동 경로와 예산에 맞는 최적 수단 조합을 추천받고, 월정액 구독 형태로 무제한 대중교통 + 택시 이용권을 구매하는 것도 가능하다.

국내에서는 카카오T, 티맵(T map) 등이 부분적 MaaS 기능을 제공하고 있으나, 국가 차원의 완전한 통합 MaaS 플랫폼은 아직 구축 중이다. 정부의 '2030 MaaS 로드맵'에 따르면 전국 버스·지하철 데이터를 국가 MaaS 플랫폼에 통합하고, 자율주행 모빌리티(로보택시)까지 연계할 계획이다.

MaaS가 필요한 핵심 이유: ① 도시화 심화로 교통 혼잡비용이 연간 60조 원 이상 발생 ② 1인 1차량 소유 모델의 탄소 배출 한계 ③ 고령화로 인한 자가운전 불가 인구 급증에 대한 모빌리티 접근성 보장

📢 **섹션 요약 비유**: MaaS는 교통의 '넷플릭스'처럼, 월정액을 내면 버스·지하철·킥보드·택시를 마음껏 이용하는 구독 서비스이다.

## Ⅱ. 아키텍처 및 핵심 원리

### MaaS 플랫폼 계층 구조

| 계층 | 구성 요소 | 설명 |
|:---|:---|:---|
| 사용자 인터페이스 | MaaS 앱 | 경로 검색, 예약, 결제, 이용 이력 |
| MaaS 오퍼레이터 | MaaS Operator API | 이종 교통 수단 통합 조율 |
| 교통 서비스 제공자 | TSP (Transport Service Provider) | 버스·지하철·공유 킥보드 등 개별 운영사 |
| 데이터 표준 | GTFS / NeTEx / SIRI | 실시간 도착 정보, 노선 데이터 표준 |
| 결제 인프라 | 통합 결제 게이트웨이 | 단일 토큰으로 다수 TSP에 결제 |

```
┌───────────────────────────────────────────────────────────────┐
│                    MaaS 서비스 아키텍처                        │
├───────────────────────────────────────────────────────────────┤
│                                                               │
│   [ 사용자 ] ──▶ [ MaaS 앱 (멀티모달 라우팅) ]               │
│                         │                                     │
│             ┌───────────┼────────────┐                        │
│             ▼           ▼            ▼                        │
│       [대중교통]   [공유 모빌리티]  [택시/라이드헤일링]        │
│       GTFS/SIRI   API (카카오)    KakaoT/우버                 │
│             │           │            │                        │
│             └───────────┴────────────┘                        │
│                         │                                     │
│              [ 통합 결제 & 정산 레이어 ]                       │
│              [ (MaaS Operator 플랫폼) ]                        │
│                         │                                     │
│              [ 이동 이력 & 탄소 발자국 대시보드 ]              │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

### 멀티모달 라우팅 알고리즘

최적 경로 탐색에는 Dijkstra 또는 A* 알고리즘의 변형인 **CSA(Connection Scan Algorithm)**가 활용되며, 비용·시간·탄소 배출량·접근성(장애인 경사로 유무) 등 다차원 파라미터를 가중합산하여 개인화된 경로를 추천한다.

📢 **섹션 요약 비유**: 멀티모달 라우팅은 '사람 몸에 맞춘 맞춤 의상'처럼, 각자의 가중치(빠름/저렴함/친환경)에 따라 최적 조합을 재단하는 AI 재단사이다.

## Ⅲ. 비교 및 연결

### MaaS 성숙도 모델 (MaaS Alliance Level)

| 레벨 | 명칭 | 특징 | 사례 |
|:---:|:---|:---|:---|
| 0 | 없음 | 통합 없음, 단일 교통수단 앱 | 개별 버스앱 |
| 1 | 정보 통합 | 멀티모달 경로 정보만 제공 | 구글 맵 |
| 2 | 예약 통합 | 경로 + 예약 통합 | 카카오T |
| 3 | 결제 통합 | 단일 결제 + 구독 패키지 | Whim (핀란드) |
| 4 | 정책 통합 | 정부 교통 정책과 MaaS 연동 | 목표 수준 |

### C-ITS(협력 지능형 교통)와의 연결

MaaS는 C-ITS와 결합하면 실시간 교통 흐름 데이터를 활용하여 더 정확한 도착 예측과 동적 가격 책정(수요 기반 가격)이 가능해진다.

📢 **섹션 요약 비유**: MaaS 레벨은 식당의 진화와 같다. 메뉴판(레벨1) → 예약 가능(레벨2) → 단골 멤버십(레벨3) → 도시 전체 음식 배달 정책과 연동(레벨4)으로 발전하는 여정이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 국가 MaaS 플랫폼 구축 시 기술사 판단 포인트

1. **데이터 표준화**: GTFS(General Transit Feed Specification) 기반으로 전국 버스·지하철 노선 데이터 통합 API 구축. 실시간 정보는 GTFS-RT 적용.
2. **ID/인증 통합**: 교통카드, 모바일 신분증, SSO(Single Sign-On)를 연동하는 통합 신원 체계 → DID(Decentralized Identifier) 기반 모빌리티 신원 가능성 검토
3. **결제 정산 복잡도**: 다수의 TSP 간 실시간 정산을 위한 **Clearing House** 구조 설계 필수. 블록체인 스마트 컨트랙트를 활용한 자동 정산 가능성 검토
4. **개인정보 보호**: 이동 패턴 데이터는 생활반경 추론이 가능한 고감도 개인정보 → K-익명화 및 차등 프라이버시(Differential Privacy) 적용 의무

📢 **섹션 요약 비유**: MaaS 구축 책임자(기술사)는 여러 항공사를 연결하는 '공항 운영자'처럼, 각 항공사(TSP)의 이해충돌을 조율하고 공통 표준(IATA)을 적용하는 중립적 플랫폼 설계자 역할이다.

## Ⅴ. 기대효과 및 결론

**기대효과**:
- 도심 승용차 통행량 15~30% 감소 (스톡홀름 사례)
- 교통 탄소 배출 20% 절감
- 교통 취약 계층(고령자·장애인)의 모빌리티 접근성 향상
- 교통 인프라 투자 효율화 (수요 데이터 기반 노선 최적화)

**한계 및 전제조건**:
- 교통 데이터 개방에 대한 민간 사업자 저항
- MaaS Operator 면허 체계 및 법제 미비 (국내)
- 자율주행 통합 시 책임 소재 법적 공백
- 농촌·지방 지역 수익성 문제로 서비스 격차 우려

📢 **섹션 요약 비유**: MaaS의 궁극적 비전은 '도시를 하나의 거대한 교통 운영 체제(OS)'로 만드는 것이다. 각 교통 수단이 앱의 프로세스처럼 자원을 공유하고 협력하여 도시 전체의 이동 효율이 극대화되는 미래이다.
