## 핵심 인사이트 (3줄 요약)

> 1. **본질**: PQC(Post-Quantum Cryptography, 양자 내성 암호)는 양자 컴퓨터가 Shor 알고리즘으로 RSA·ECC 기반 공개키 암호를 수분 내 해독할 수 있게 되는 미래에 대비하여, 격자(Lattice) 기반 등의 수학 난제를 활용한 새로운 암호 체계이다.
> 2. **가치**: NIST가 2024년 발표한 CRYSTALS-Kyber(키 교환)와 CRYSTALS-Dilithium(디지털 서명) PQC 표준은 현재 인터넷 보안의 근간인 TLS, PKI, JWT 서명을 양자 위협에서 보호하는 미래 암호 인프라의 핵심이다.
> 3. **판단 포인트**: "Harvest Now, Decrypt Later" 공격—현재 암호화 트래픽을 저장했다가 향후 양자 컴퓨터로 해독하는 전략—이 이미 진행 중이므로, 장기 기밀 데이터(국가 안보, 의료 기록)는 지금 당장 PQC 전환 계획이 필요하다.

## Ⅰ. 개요 및 필요성

인터넷 보안의 근간인 RSA-2048은 고전 컴퓨터로 해독하려면 수백만 년이 걸린다. 그러나 Peter Shor의 알고리즘은 충분히 강력한 양자 컴퓨터를 사용하면 RSA와 ECC를 다항 시간에 해독한다. IBM, Google, Microsoft가 경쟁하는 1,000 큐비트 이상의 내결함성 양자 컴퓨터가 2030년대에 등장한다면, 현재의 공개키 암호 체계는 무력화된다.

NIST는 2016년부터 PQC 표준화 작업을 시작하여 2024년 최초의 PQC 표준 3개를 공식 발표했다:
- **ML-KEM** (CRYSTALS-Kyber): 키 캡슐화 메커니즘 (TLS 키 교환용)
- **ML-DSA** (CRYSTALS-Dilithium): 디지털 서명 (코드 서명, PKI용)
- **SLH-DSA** (SPHINCS+): 해시 기반 서명 (Kyber 의존성 없는 대안)

한국 국가정보원도 2023년 양자 내성 암호 권고안을 발표하고, 2030년까지 공공 기관의 PQC 전환을 목표로 하고 있다.

📢 **섹션 요약 비유**: PQC는 자물쇠 제조사가 "미래에 마스터키(양자 컴퓨터)가 등장할 것을 미리 알고, 그 마스터키로도 못 여는 새 자물쇠"를 지금 만들어 두는 선제적 보안 혁신이다.

## Ⅱ. 아키텍처 및 핵심 원리

### 고전 암호 vs PQC 비교

| 항목 | RSA-2048 / ECC-256 | PQC (Kyber-768) |
|:---|:---|:---|
| 수학적 기반 | 소인수 분해 / 타원곡선 이산 로그 | 격자(Lattice) 최단벡터 문제 (LWE) |
| 양자 취약성 | Shor 알고리즘으로 해독 가능 | 양자 알고리즘으로 효율적 해독 불가 (현재 기준) |
| 키 크기 | 256바이트 (ECC) | 1,184바이트 (Kyber-768 공개키) |
| 서명 크기 | 64바이트 (ECDSA) | 2,420바이트 (Dilithium3) |
| 성능 | 빠름 | 약간 느림 (키/서명 크기 증가) |
| NIST 표준 상태 | 기존 표준 | 2024년 최종 표준 |

### 격자(Lattice) 기반 암호의 수학적 원리

LWE(Learning With Errors) 문제: 격자에서 약간의 노이즈가 추가된 선형 방정식 시스템에서 비밀 벡터를 복원하는 문제는 고전/양자 컴퓨터 모두에서 계산적으로 어렵다는 수학적 증명에 기반한다.

```
┌───────────────────────────────────────────────────────────────┐
│           PQC 하이브리드 TLS 전환 구조                         │
├───────────────────────────────────────────────────────────────┤
│                                                               │
│  현재 TLS 1.3:                                                │
│  [Client] ────X25519 ECDH 키 교환────▶ [Server]              │
│                                                               │
│  하이브리드 PQC TLS (전환 단계):                              │
│  [Client] ────X25519 + Kyber-768 조합────▶ [Server]          │
│           (고전 + PQC 동시 적용)                              │
│  → 고전 및 양자 컴퓨터 모두에 안전                            │
│                                                               │
│  완전 PQC TLS (2030년대 목표):                                │
│  [Client] ────ML-KEM 키 교환────▶ [Server]                   │
│           + ML-DSA 인증서 서명                                │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

### HARVEST NOW, DECRYPT LATER 공격

적대적 국가 행위자가 현재 암호화된 TLS 트래픽을 수집·저장하고, 미래의 양자 컴퓨터로 해독하는 전략이다. 특히 15~30년 기밀 유지가 필요한 데이터(외교 전문, 방위 계획, 의료 유전체 데이터)는 지금 당장 PQC 전환이 시급하다.

📢 **섹션 요약 비유**: Harvest Now Decrypt Later는 지금 남의 암호 일기를 가져가서 나중에 번역기가 생기면 읽는 것처럼, 현재 보안이 미래에 무력화될 것을 알고 데이터를 선제 수집하는 장기 사이버 전략이다.

## Ⅲ. 비교 및 연결

### 대칭키 암호는 안전한가?

AES-256은 Grover 알고리즘으로 유효 강도가 128비트로 절반 줄지만, 양자 위협에서 여전히 안전한 수준이다. 즉 AES는 PQC 전환이 불필요하지만, **RSA·ECC 기반의 모든 공개키 인프라**는 전환이 필수이다.

### 국내외 PQC 전환 일정

| 기관 | 일정 | 내용 |
|:---|:---|:---|
| NIST | 2024 완료 | ML-KEM, ML-DSA, SLH-DSA 표준 확정 |
| 한국 국가정보원 | 2030년 목표 | 공공 기관 PQC 전환 |
| Google | 2024 적용 | Chrome에 X25519Kyber768 하이브리드 |
| NSA | 2030년 목표 | 미국 국가안보시스템 PQC 전환 |

📢 **섹션 요약 비유**: 대칭키(AES)는 양자 컴퓨터가 자물쇠 따는 속도를 두 배 빠르게 만들 뿐이어서 더 두꺼운 자물쇠(256비트)로 대응 가능하지만, 공개키(RSA)는 자물쇠 구조 자체가 양자로 풀리므로 전혀 다른 방식의 새 자물쇠(PQC)가 필요하다.

## Ⅳ. 실무 적용 및 기술사 판단

### 기업 PQC 전환 로드맵

1. **암호 인벤토리 파악**: RSA·ECC를 사용하는 모든 시스템(TLS 인증서, JWT 서명, 코드 서명, VPN) 목록화
2. **데이터 중요도 분류**: 장기 기밀 데이터 식별 → 우선 전환 대상
3. **하이브리드 PQC 적용**: 전환 기간에 고전 + PQC 알고리즘 동시 적용 (호환성 유지)
4. **키 크기 증가 대응**: PQC 키/서명 크기 증가로 TLS 핸드셰이크 지연 증가 → 프로토콜 최적화 검토
5. **CA/PKI 인프라 갱신**: 인증서 수명 단축(1년 이내) 및 PQC 기반 CA 구축 계획

📢 **섹션 요약 비유**: PQC 전환 로드맵은 도시 전체의 수도관을 교체하는 것처럼, 한 번에 할 수 없어 구역별 점진적으로 교체하되 기존 수도관과 새 수도관이 임시로 병렬 운용(하이브리드)되는 대규모 인프라 혁신이다.

## Ⅴ. 기대효과 및 결론

**기대효과**:
- 양자 컴퓨터 위협 대비 통신 인프라 장기 안전성 확보
- "Harvest Now, Decrypt Later" 공격으로부터 장기 기밀 데이터 보호
- NIST 표준 준수로 글로벌 규제 대응

**한계 및 전제조건**:
- PQC 알고리즘의 키/서명 크기 증가로 대역폭 및 성능 오버헤드
- 양자 위협이 현실화되는 시기의 불확실성 (2030~2040년 예상)
- CRYSTALS-Kyber/Dilithium도 새로운 수학적 공격에 취약할 가능성 잔존
- 전 세계 PKI 인프라의 동시 전환이라는 전례 없는 규모의 마이그레이션

📢 **섹션 요약 비유**: PQC는 지금 당장 집이 불타고 있지 않지만, 10년 후 큰 화재가 예측되어 지금부터 내화 건물로 리모델링하는 예방적 투자이다. 불이 나고 나서는 늦다.
