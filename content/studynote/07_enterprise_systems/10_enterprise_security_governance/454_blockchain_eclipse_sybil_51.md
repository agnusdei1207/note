## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 블록체인의 3대 공격—이클립스(Eclipse), 시빌(Sybil), 51% 공격—은 각각 노드 고립, 신원 위조, 해시/스테이크 다수 장악이라는 서로 다른 벡터로 블록체인의 신뢰 모델을 붕괴시키는 핵심 위협이다.
> 2. **가치**: 이 공격들을 이해하는 것은 단순한 보안 지식이 아니라, 블록체인 기반 시스템의 신뢰 전제 조건을 검증하는 설계 판단력의 핵심이다. 퍼블릭/프라이빗 블록체인의 보안 모델이 근본적으로 다름을 이해해야 한다.
> 3. **판단 포인트**: 51% 공격의 현실적 위협은 해시파워 집중도(마이닝 풀)와 공격 비용(Crypto51.app 기준)으로 정량화되며, 소규모 알트코인은 대형 거래소보다 낮은 비용으로 공격 가능하다는 점이 핵심 취약점이다.

## Ⅰ. 개요 및 필요성

블록체인은 "중앙 기관 없이도 신뢰할 수 있는 분산 원장"을 목표로 한다. 그러나 이 신뢰는 네트워크 참여자의 과반수가 정직하다는 전제(N/2+1 정직 가정)에 기반한다. 이 전제를 위협하는 세 가지 공격 유형을 이해하는 것이 블록체인 시스템 설계의 출발점이다.

2019년 Ethereum Classic은 51% 공격으로 약 110만 달러 상당의 이중 지불(Double Spending) 피해를 입었다. 이 사건은 "탈중앙화"라는 이름 뒤에 숨어있는 집중화 리스크를 세상에 드러냈다.

블록체인 기반 시스템—디지털 화폐, 공급망 추적, DID(탈중앙 신원)—을 기업에 도입할 때, 기술사는 이 위협들이 실제 비즈니스 리스크로 연결되는지를 평가해야 한다.

📢 **섹션 요약 비유**: 블록체인 공격 3종은 선거 시스템 공격처럼, 투표자를 고립(이클립스), 가짜 투표자를 만들거나(시빌), 과반을 장악(51%)하여 민주주의(합의)를 파괴하는 전략이다.

## Ⅱ. 아키텍처 및 핵심 원리

### 3대 공격 비교

| 공격 유형 | 목표 | 방법 | 영향 |
|:---|:---|:---|:---|
| **이클립스(Eclipse)** | 특정 노드 고립 | 피해 노드의 P2P 연결을 모두 공격자 노드로 교체 | 이중 지불, 지갑 트랜잭션 조작 |
| **시빌(Sybil)** | 다수 가짜 신원 생성 | 대량의 가짜 노드로 네트워크 영향력 확보 | 투표 조작, 라우팅 왜곡 |
| **51%** | 합의 과정 장악 | 전체 해시파워/스테이크의 과반 확보 | 이중 지불, 트랜잭션 검열 |

### 이클립스 공격 메커니즘

```
┌──────────────────────────────────────────────────────────────┐
│              이클립스 공격 구조                              │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  정상 네트워크:                                              │
│  [피해 노드] ↔ [정직 노드 1,2,3,4,5,6...]                   │
│                                                              │
│  공격 후:                                                    │
│  [피해 노드] ↔ [공격자 노드 A,B,C,D,E,F...]                 │
│       ↑                                                      │
│  정직 노드들과 연결이 완전히 차단됨                          │
│  → 피해 노드는 공격자가 만든 가짜 블록체인을 신뢰하게 됨    │
│  → 이중 지불 가능: 피해자에게 보냈지만 "취소됨"으로 속임    │
│                                                              │
│  방어: 최소 연결 수 확보, 노드 다양성(ASN) 강제, PeerStrap  │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 51% 공격과 이중 지불(Double Spending)

```
51% 공격 시나리오:
1. 공격자가 거래소에 1,000 BTC를 송금 (공개 체인에 기록)
2. 거래소가 확인 후 달러 출금을 처리
3. 공격자가 비밀 체인에서 동일 코인을 자신에게 다시 송금하는 더 긴 체인 생성
4. 51% 해시파워로 비밀 체인을 공개 체인보다 길게 만들어 공개
5. 비트코인 프로토콜이 더 긴 체인을 정상으로 인정 → 원래 거래소 송금이 취소됨
6. 공격자는 달러도 갖고 BTC도 돌려받는 이중 지불 성공
```

📢 **섹션 요약 비유**: 51% 공격은 시험 채점에서 채점자 51명이 담합하여 오답에 점수를 주는 것처럼, 다수결 합의 시스템에서 다수를 장악하면 규칙 자체를 바꿀 수 있다.

## Ⅲ. 비교 및 연결

### PoW vs PoS 합의 메커니즘에서의 51% 공격 비교

| 항목 | PoW (Proof of Work) | PoS (Proof of Stake) |
|:---|:---|:---|
| 공격 조건 | 전체 해시파워 51% 장악 | 전체 스테이킹 자산 51% 장악 |
| 공격 비용 | 전기·하드웨어 비용 | 해당 암호화폐 구매 비용 |
| 공격 억제 | 경제적 비용 | 공격 시 자신의 자산 가치도 하락 |
| 실현 가능성 | 소규모 코인에서 현실적 위협 | 자산 가치 파괴 역설로 억제 |

### 프라이빗 블록체인에서의 차별화

허가형(Permissioned) 블록체인(Hyperledger Fabric)은 신원 확인된 참여자만 노드를 운영하므로 시빌 공격이 원천 차단된다. 그러나 내부자 결탁이 새로운 위협이 된다.

📢 **섹션 요약 비유**: 퍼블릭 블록체인이 '완전 공개 선거'라면, 프라이빗 블록체인은 '신원 확인된 의원들만의 국회 투표'처럼, 외부 침입은 막지만 내부 담합이라는 새로운 위협이 생긴다.

## Ⅳ. 실무 적용 및 기술사 판단

### 기업 블록체인 도입 시 보안 설계 체크리스트

1. **합의 노드 최소화 금지**: 3개 이하의 검증 노드는 담합에 취약. 최소 7개 이상 권장
2. **노드 지리적/조직적 분산**: 동일 클라우드 프로바이더에 집중 금지 (이클립스 위험)
3. **확인 블록 수 설정**: 이중 지불 방지를 위해 거래 확정 전 대기 블록 수 = 6 이상
4. **허가형 블록체인 채택 기준**: 내부 거래 처리는 Hyperledger, 외부 신뢰 증명은 퍼블릭 연계
5. **감사 로그 외부 앵커링**: 프라이빗 체인 블록 해시를 이더리움 메인넷에 주기적 등록

📢 **섹션 요약 비유**: 블록체인 노드 분산 설계는 핵발전소 안전 설계처럼, 단일 장애점(SPOF)을 없애고 여러 독립적 안전 시스템이 동시에 작동해야 한다는 다중 방어(Defense in Depth) 원칙과 동일하다.

## Ⅴ. 기대효과 및 결론

**방어 전략 요약**:
- **이클립스 방어**: 노드 연결 다양성 강제(최소 20개 피어), Kademlia DHT 강화
- **시빌 방어**: PoW/PoS 합의 메커니즘 자체가 경제적 시빌 방어, 또는 허가형 신원 검증
- **51% 방어**: 합의 참여자 분산화, 마이닝 풀 지배력 모니터링, 체크포인팅

**한계**:
- 소규모 블록체인은 낮은 비용으로 51% 공격에 취약
- 퀀텀 컴퓨팅 발전 시 현재 PoW 해시 알고리즘의 근본적 재설계 필요
- 허가형 블록체인의 내부자 공모는 기술적 방어 한계 (거버넌스 보완 필수)

📢 **섹션 요약 비유**: 블록체인 보안의 역설은 강철 자물쇠(암호학적 안전)를 달았지만, 문을 지키는 경비원(합의 노드)을 매수할 수 있다면 자물쇠가 소용없다는 현실이다.

### 📌 관련 개념 맵

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| PoW/PoS | 방어 기반 | 공격 비용을 높이는 합의 메커니즘 |
| 이중 지불 | 공격 목적 | 51% 공격의 최종 목표 |
| Hyperledger | 대안 구조 | 허가형 블록체인으로 시빌 원천 차단 |
| 체크포인팅 | 방어 기법 | 특정 블록을 되돌릴 수 없게 고정 |
| DID | 연관 기술 | 블록체인 기반 탈중앙 신원 (시빌 관련) |
| 공급망 보안 | 활용 분야 | 블록체인 공급망 추적의 신뢰 전제 |

### 👶 어린이를 위한 3줄 비유 설명

1. 이클립스 공격은 내 친구(노드)의 전화를 차단하고 나쁜 사람들하고만 통화하게 만드는 거예요.
2. 시빌 공격은 학교 투표에서 한 명이 가짜 친구 50명을 데려와 자기가 원하는 결과를 만드는 거예요.
3. 51% 공격은 반 절반이 넘는 친구들이 "이 숙제는 안 해도 된다"고 거짓말로 합의를 바꾸는 거예요.
