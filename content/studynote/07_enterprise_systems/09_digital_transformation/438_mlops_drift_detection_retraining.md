## 핵심 인사이트 (3줄 요약)

> 1. **본질**: MLOps(Machine Learning Operations)는 ML 모델 개발(Dev)과 운영(Ops)을 통합하여 모델 훈련→검증→배포→모니터링→재학습의 생명주기를 자동화하는 방법론으로, 드리프트(Drift) 탐지와 자동 재학습 파이프라인이 핵심이다.
> 2. **가치**: 모델이 프로덕션에서 시간이 지남에 따라 성능이 저하되는 모델 드리프트(데이터·개념 드리프트)를 자동 탐지하고 재학습을 트리거하여 모델 성능의 지속적 유지를 보장한다.
> 3. **판단 포인트**: 데이터 드리프트(입력 분포 변화)와 개념 드리프트(입력-출력 관계 변화)의 구분, PSI(Population Stability Index)·KL 발산·MMD 등 드리프트 탐지 통계 기법의 선택이 실무 핵심이다.

## Ⅰ. 개요 및 필요성

Jupyter Notebook에서 완성된 모델을 프로덕션에 배포하면 "끝"이 아니다. 실제 데이터 패턴은 시간이 지남에 따라 변하고(드리프트), 모델 성능이 서서히 저하된다. 2020년 Algorithmia 조사에 따르면 기업의 56%가 프로덕션 ML 모델 유지보수에 가장 큰 어려움을 겪는다. MLOps는 소프트웨어 DevOps를 ML 생명주기에 적용하여 이 문제를 자동화로 해결한다.

📢 **섹션 요약 비유**: MLOps는 ML 모델의 자동차 정기 검사 — 처음엔 잘 달리던 모델이 길(데이터)이 바뀌면 성능이 떨어지므로, 정기 점검(모니터링)과 수리(재학습)가 필요하다.

## Ⅱ. 아키텍처 및 핵심 원리

```
MLOps 생명주기:
  데이터 수집·검증
       ↓
  피처 엔지니어링 (Feature Store)
       ↓
  모델 훈련 (Kubeflow, MLflow, SageMaker)
       ↓
  검증 (메트릭, 편향 체크)
       ↓
  배포 (Blue/Green, Canary, Shadow)
       ↓
  모니터링 (성능, 드리프트 탐지)
       ↓ 드리프트 감지 시
  자동 재학습 트리거 ←──────────┘

드리프트 유형:
  데이터 드리프트: 입력 분포 P(X) 변화
    예) 코로나 이전 검색 패턴 vs 이후
  개념 드리프트: 입력-출력 관계 P(Y|X) 변화
    예) 스팸 필터: 새로운 스팸 패턴 등장
```

| 드리프트 탐지 기법 | 설명 | 사용 기준 |
|:---|:---|:---|
| PSI (Population Stability Index) | 배포 전후 분포 비교 | >0.25이면 심각 드리프트 |
| KS Test | 두 분포의 최대 거리 | 연속형 변수 |
| KL Divergence | 분포 정보 차이 | 범주형 변수 |
| MMD (Maximum Mean Discrepancy) | 커널 기반 분포 비교 | 고차원 데이터 |

📢 **섹션 요약 비유**: PSI는 모델의 혈압 측정 — PSI > 0.25이면 고혈압(심각한 드리프트) 경보로 즉시 재검진(재학습)이 필요하다.

## Ⅲ. 비교 및 연결

MLflow: 실험 추적(Experiment Tracking)·모델 레지스트리·배포를 통합하는 오픈소스 MLOps 플랫폼. KFP(Kubeflow Pipelines): Kubernetes 기반 ML 워크플로우 오케스트레이션. CI/CD for ML(CI/CD4ML): 모델 코드 변경 시 자동 훈련→테스트→배포 파이프라인.

📢 **섹션 요약 비유**: MLflow는 ML의 GitHub + Jenkins + Docker Hub — 실험 기록(GitHub), 자동 빌드(Jenkins), 모델 저장소(Docker Hub)를 통합한 플랫폼이다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- 드리프트 탐지 자동화: Evidently AI, Fiddler AI, WhyLabs로 실시간 모니터링
- 재학습 트리거 기준: PSI > 0.25 또는 AUC 5% 이상 저하 → 자동 재학습
- A/B 테스트 배포: 새 모델을 10% 트래픽에 먼저 → 성능 확인 후 전환
- 모델 레지스트리: MLflow Model Registry로 버전 관리·승인 워크플로우

📢 **섹션 요약 비유**: 카나리 배포는 탄광의 카나리 새 — 새 모델을 소수 사용자(카나리)에게 먼저 노출하여 문제를 조기에 탐지한다.

## Ⅴ. 기대효과 및 결론

MLOps 자동화로 모델 배포 주기를 수개월에서 수일로 단축하고, 드리프트 자동 탐지로 모델 성능 저하를 조기에 방지할 수 있다. LLMOps는 LLM 특화 MLOps로 프롬프트 버전 관리·RAG 파이프라인 모니터링·파인튜닝 실험 추적을 포함하는 확장 개념이다. 데이터 품질과 피처 일관성이 모든 MLOps의 근본적 전제 조건이다.

📢 **섹션 요약 비유**: MLOps는 ML 공장의 품질 관리 시스템 — 제품(모델)이 생산라인(파이프라인)을 통과하는 모든 단계에서 자동 품질 검사가 이루어진다.

### 👶 어린이를 위한 3줄 비유 설명

1. MLOps는 모델의 건강 검진 — 처음 만든 모델이 시간이 지나면서 세상이 바뀌어 성능이 떨어지면, 자동으로 알려주고 고쳐줘.
2. 드리프트는 길이 바뀐 것 — 예전에 학습한 지도로는 새 도로를 못 찾아서, 지도를 업데이트(재학습)해야 해.
3. A/B 테스트 배포는 시험 운행 — 새 모델을 10명 중 1명에게 먼저 써보고, 잘 되면 모두에게 배포하는 안전한 방법이야!
