## 핵심 인사이트 (3줄 요약)

> 1. **본질**: RAG(Retrieval-Augmented Generation)는 LLM의 할루시네이션(Hallucination: 사실과 다른 자신감 있는 생성)을 외부 지식 베이스에서 실시간 검색한 컨텍스트로 그라운딩(Grounding)하여 제어하는 아키텍처 패턴이다.
> 2. **가치**: 파인튜닝 없이 최신 정보·도메인 지식을 LLM에 주입하고, 인용 소스를 제공하여 답변의 신뢰성과 추적 가능성을 높이며, 지속적 학습 없이 지식을 갱신할 수 있다.
> 3. **판단 포인트**: 벡터 DB(Pinecone, Weaviate, Qdrant)의 임베딩 검색 품질(Recall)과 프롬프트 컨텍스트 윈도우 크기가 RAG 응답 품질의 핵심 결정 요소다.

## Ⅰ. 개요 및 필요성

GPT-4·Claude 같은 LLM은 학습 데이터 컷오프 이후의 정보를 모르고, 도메인 전문 지식을 갖추지 못하며, 훈련 데이터에서 학습한 편향을 반영한다. RAG는 이 문제를 "검색(Retrieve) + 생성(Generate)"의 2단계로 해결한다. Lewis et al.(2020, Meta AI)이 제안한 RAG는 현재 엔터프라이즈 AI 응용의 표준 패턴이 됐다.

📢 **섹션 요약 비유**: RAG는 열린 책 시험 — LLM(수험생)이 교과서(외부 지식 베이스)를 열어보며 답하는 것으로, 기억에만 의존하는 것보다 정확하다.

## Ⅱ. 아키텍처 및 핵심 원리

```
RAG 파이프라인:

  오프라인 인덱싱 단계:
  문서 → 청크 분할 → 임베딩 모델 → 벡터 DB 저장

  온라인 쿼리 단계:
  사용자 쿼리
       │
       ▼
  [Query 임베딩 → 벡터 DB 유사도 검색 → Top-K 문서 검색]
       │ 검색된 컨텍스트
       ▼
  [프롬프트: 시스템 + 컨텍스트 + 사용자 쿼리]
       │
       ▼
  LLM (GPT-4 / Claude / LLaMA)
       │
       ▼
  인용 포함 응답 생성
```

| 구성 요소 | 설명 | 주요 도구 |
|:---|:---|:---|
| 임베딩 모델 | 텍스트→벡터 변환 | OpenAI text-embedding-3, E5 |
| 벡터 DB | 고차원 벡터 유사도 검색 | Pinecone, Weaviate, Qdrant, pgvector |
| 청크 전략 | 문서 분할 방법 | 고정 크기, 시맨틱 청킹 |
| 리랭커 | 검색 결과 재순위 | Cross-Encoder, Cohere Rerank |
| LLM | 최종 생성 | GPT-4, Claude, LLaMA |

📢 **섹션 요약 비유**: 벡터 DB는 의미 기반 도서관 — "사과 주스"를 검색하면 "과일 음료" 책도 찾아주는, 키워드가 아닌 의미로 검색하는 도서관이다.

## Ⅲ. 비교 및 연결

RAG vs 파인튜닝: RAG는 지식 갱신이 즉시(재인덱싱), 파인튜닝은 스타일·태스크 특화에 유리. 실제로 두 방식을 결합하는 것이 최고 성능. Advanced RAG: HyDE(가설적 문서 임베딩), Re-ranking, Multi-Query, Step-back Prompting으로 검색 품질 개선.

📢 **섹션 요약 비유**: RAG vs 파인튜닝은 참고서 vs 암기 — RAG는 시험 볼 때 참고서를 보고, 파인튜닝은 미리 다 외워두는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- 최신 정보 필요: RAG (지식 갱신 용이)
- 특정 도메인 전문성: 파인튜닝 + RAG 결합
- 벡터 DB 선택: 소규모→pgvector, 대규모→Pinecone/Weaviate
- 청크 크기: 너무 작으면 컨텍스트 부족, 너무 크면 노이즈 증가 (512~1024 토큰 권장)
- 평가: RAGAS 프레임워크 (충실도, 답변 관련성, 컨텍스트 재현율 측정)

📢 **섹션 요약 비유**: 청크 크기 설계는 검색 결과 페이지 — 너무 짧으면 정보 부족, 너무 길면 핵심을 찾기 어렵다.

## Ⅴ. 기대효과 및 결론

RAG는 LLM 할루시네이션을 60~80% 감소시키고, 인용 소스 제공으로 AI 응답의 신뢰성과 감사 가능성을 높인다. GraphRAG(Microsoft)는 지식 그래프와 벡터 검색을 결합하여 복잡한 다중 홉 추론을 개선하는 차세대 방향이다. 벡터 DB 유지보수(인덱싱 파이프라인)와 컨텍스트 관련성 품질 관리가 RAG 시스템의 지속적 운영 핵심 과제다.

📢 **섹션 요약 비유**: RAG는 LLM의 메모 패드 — 기억력(파라미터)에만 의존하지 않고, 필요할 때 메모(외부 지식)를 참조하여 더 정확하게 답한다.

### 👶 어린이를 위한 3줄 비유 설명

1. RAG는 열린 책 시험 — LLM이 혼자 기억하는 대신, 인터넷(지식 베이스)을 찾아보며 답해서 훨씬 정확해.
2. 벡터 DB는 의미 기반 검색 — "과일 주스"를 찾으면 "사과 음료", "오렌지 드링크"도 같이 찾아주는 똑똑한 도서관.
3. 인용 소스 제공은 숙제 참고문헌 — "이 답은 이 책 3페이지에서 찾았어요" 라고 알려줘서 믿을 수 있어!
