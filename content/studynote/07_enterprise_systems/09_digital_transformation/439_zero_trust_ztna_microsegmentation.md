## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 제로 트러스트(Zero Trust)는 "절대 신뢰하지 말고, 항상 검증하라(Never Trust, Always Verify)"는 보안 패러다임으로, 네트워크 위치(내부망)에 관계없이 모든 접근 요청을 지속적으로 인증·인가·검증한다.
> 2. **가치**: ZTNA(Zero Trust Network Access)는 VPN의 과도한 네트워크 접근 권한을 대체하여 최소 권한(Least Privilege) 기반 애플리케이션 수준 접근 제어를 제공하고, 마이크로 세그멘테이션은 내부 네트워크를 소규모 격리 구역으로 분할하여 측면 이동(Lateral Movement) 공격을 차단한다.
> 3. **판단 포인트**: NIST SP 800-207 기반의 5대 원칙(명시적 검증·최소 권한·위반 가정)과 PEP(Policy Enforcement Point)·PDP(Policy Decision Point) 아키텍처가 제로 트러스트 구현의 핵심이다.

## Ⅰ. 개요 및 필요성

전통 경계 보안(Perimeter Security)은 "내부망 = 신뢰"를 가정하여 방화벽으로 외부를 차단했다. 그러나 클라우드·원격 근무·BYOD·내부자 위협으로 이 가정이 무너졌다. 2021년 SolarWinds 공격은 신뢰된 내부 소프트웨어 업데이트를 통해 1,800개+ 기관에 침투하여 경계 보안의 근본적 한계를 보여줬다. 바이든 행정부는 2021년 EO 14028으로 연방기관의 제로 트러스트 전환을 의무화했다.

📢 **섹션 요약 비유**: 제로 트러스트는 회사 ID 카드 — 건물(내부망)에 들어왔어도 매 방(리소스)마다 ID 카드(인증·인가)를 다시 확인한다.

## Ⅱ. 아키텍처 및 핵심 원리

```
제로 트러스트 아키텍처 (NIST SP 800-207):

  사용자/기기/앱 (Subject)
          │ 접근 요청
          ▼
  PEP (Policy Enforcement Point)
  (게이트웨이, 프록시, API 게이트웨이)
          │ 정책 질의
          ▼
  PDP (Policy Decision Point)
  ┌─────────────────────────────────┐
  │  PA (Policy Administrator)       │
  │  PE (Policy Engine)              │
  │  ├─ IAM (MFA, RBAC, ABAC)       │
  │  ├─ CDM (기기 상태 모니터링)     │
  │  ├─ SIEM (위협 인텔리전스)       │
  │  └─ PKI (인증서)                 │
  └─────────────────────────────────┘
          │ 허용/거부
          ▼
  리소스 (Resource: 앱, API, DB)

마이크로 세그멘테이션:
  ┌──────────────────────────────┐
  │  Zone A    │  Zone B         │
  │ (DB 서버)  │ (앱 서버)       │
  │            │                 │
  │ 격리 세그먼트: Zone 간 접근   │
  │ 명시적 허용 없이 차단         │
  └──────────────────────────────┘
```

| 원칙 | 내용 | 구현 방법 |
|:---|:---|:---|
| 명시적 검증 | 모든 요청 항상 검증 | MFA, 기기 신뢰, 위치·시간 컨텍스트 |
| 최소 권한 | 필요한 최소 권한만 | RBAC, ABAC, JIT(Just-In-Time) 접근 |
| 위반 가정 | 이미 침해됐다고 가정 | 마이크로 세그멘테이션, 측면 이동 차단 |

�� **섹션 요약 비유**: 마이크로 세그멘테이션은 내부 방화벽 — 같은 건물 안에서도 방마다 자물쇠(방화벽)를 달아 한 방이 뚫려도 다른 방은 안전하게 한다.

## Ⅲ. 비교 및 연결

ZTNA vs VPN: VPN은 전체 네트워크 접근 터널(과도 권한), ZTNA는 특정 앱만 접근 터널(최소 권한). ZTNA는 클라우드·SaaS 접근에도 적합하여 원격 근무 보안의 표준이 됐다. BeyondCorp(Google의 제로 트러스트): 2014년 구글이 내부망 VPN을 제거하고 완전 제로 트러스트로 전환한 최초 사례.

📢 **섹션 요약 비유**: ZTNA vs VPN은 건물 마스터키 vs 방 열쇠 — VPN(마스터키)은 어디든 열지만, ZTNA(방 열쇠)는 허가받은 방만 열린다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- 원격 근무 보안: VPN 대체 ZTNA (Cloudflare Access, Zscaler ZPA)
- 클라우드 다중 환경: SASE(ZTNA+SWG+CASB 통합) 솔루션
- 내부 측면 이동 차단: 마이크로 세그멘테이션(VMware NSX, Illumio)
- JIT 접근: 특권 계정(PAM) 임시 권한 부여 (CyberArk, BeyondTrust)

📢 **섹션 요약 비유**: JIT 접근은 식당 예약제 — 필요할 때만 권한을 주고(예약), 끝나면 즉시 회수(테이블 치우기)하여 항상 열려 있는 권한(상시 접근)을 없앤다.

## Ⅴ. 기대효과 및 결론

제로 트러스트 전환으로 내부자 위협 60% 감소, 측면 이동 공격 차단, 원격 근무 보안 강화, 클라우드·멀티 환경 통합 보안을 달성할 수 있다. 기술 도입보다 아이덴티티(IAM)·기기 신뢰·네트워크 가시성의 데이터 기반 설계가 성공의 핵심이며, 전사적 전환에는 2~3년의 로드맵이 현실적이다.

📢 **섹션 요약 비유**: 제로 트러스트는 디지털 세계의 무한 반복 신원 확인 — 아무리 신뢰받는 사람도 매번 다시 확인하여 보안의 허점을 줄인다.

### 👶 어린이를 위한 3줄 비유 설명

1. 제로 트러스트는 매번 신분증 확인 — 같은 건물 안에 있어도, 방마다 들어갈 때마다 ID 카드를 다시 보여줘야 해.
2. 마이크로 세그멘테이션은 방마다 자물쇠 — 해커가 회사 컴퓨터 한 대를 뚫어도, 다른 방(서버)까지 마음대로 못 가.
3. 제로 트러스트는 의심의 기술 — "안에 있으면 안전"이 아니라, 항상 "당신은 진짜 당신인가요?" 를 물어보는 것이야!
