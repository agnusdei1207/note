## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 프롬프트 엔지니어링(Prompt Engineering)은 파인튜닝 없이 LLM의 입력(프롬프트) 설계만으로 원하는 출력을 이끌어내는 기술로, 제로샷(Zero-Shot)·퓨샷(Few-Shot)·CoT(Chain-of-Thought)·ToT(Tree-of-Thought)가 핵심 기법이다.
> 2. **가치**: CoT는 복잡한 추론 문제를 단계별 중간 과정 서술로 해결률을 대폭 향상시키고, ToT는 여러 사고 경로를 트리로 탐색하여 최적 경로를 선택한다.
> 3. **판단 포인트**: "Let's think step by step"(제로샷 CoT)처럼 단순 지시만으로도 수학·논리 문제 정확도가 2~3배 향상되므로, 고비용 파인튜닝 전에 프롬프트 최적화를 먼저 시도해야 한다.

## Ⅰ. 개요 및 필요성

파인튜닝은 수백만 원의 GPU 비용이 필요하지만, 프롬프트 설계만으로 동등한 성능을 얻을 수 있는 경우가 많다. LLM은 지시(Instruction)에 따라 동작이 달라지는 "지시 기반 범용 엔진"이므로, 좋은 프롬프트 작성이 AI 시스템 품질의 핵심 엔지니어링 과제가 됐다.

📢 **섹션 요약 비유**: 프롬프트 엔지니어링은 직원에게 업무 지시하기 — 지시를 명확하게 할수록 (CoT: "이렇게 단계별로 해줘") 결과가 훨씬 좋아진다.

## Ⅱ. 아키텍처 및 핵심 원리

```
프롬프트 기법 계층:

Zero-Shot: 예시 없이 지시만
  "다음 문장을 긍정/부정으로 분류하세요: ..."

Few-Shot: 2~5개 예시 포함
  "다음은 예시입니다:
   Q: ... A: 긍정
   Q: ... A: 부정
   Q: [새 문장]  A: ?"

CoT (Chain-of-Thought): 단계별 사고 과정 유도
  "단계별로 생각해봅시다(Let's think step by step):
   1단계: ... 2단계: ... 결론: ..."

ToT (Tree-of-Thought): 복수 사고 경로 탐색
  경로 A → 평가 → 유망하면 확장
  경로 B → 평가 → 가지치기(pruning)
  최적 경로 선택 → 최종 답변

ReAct (Reasoning + Acting):
  생각 → 행동(도구 호출) → 관찰 → 반복
```

| 기법 | 장점 | 한계 | 적용 대상 |
|:---|:---|:---|:---|
| Zero-Shot | 간단, 비용 없음 | 복잡한 작업 취약 | 단순 분류·번역 |
| Few-Shot | 성능 안정적 | 토큰 비용 증가 | 형식 일관성 필요 |
| CoT | 복잡 추론 향상 | 긴 출력, 오류 전파 | 수학·논리 문제 |
| ToT | 최적 경로 탐색 | 높은 API 비용 | 창의적·전략적 문제 |

📢 **섹션 요약 비유**: CoT는 수학 풀이 과정 쓰기 — 답만 쓰면 틀릴 수 있지만, 풀이 과정을 단계별로 쓰면 검증이 가능하고 오류도 줄어든다.

## Ⅲ. 비교 및 연결

Self-Consistency: CoT를 여러 번 실행하고 다수결로 최종 답 선택 → 단순 CoT 대비 정확도 향상. APO(Automatic Prompt Optimization): LLM이 자신의 프롬프트를 자동 개선하는 메타 프롬프팅. System Prompt: ChatGPT/Claude의 전역 지시사항으로 페르소나·형식·제약 조건 설정.

📢 **섹션 요약 비유**: Self-Consistency는 여러 전문가 의견 취합 — 한 명(한 번 실행)보다 여러 명(N회 실행)의 다수결이 더 정확하다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- 기술사 핵심: CoT의 원리(중간 추론 단계 명시)와 파인튜닝 대비 비용 효율성 설명
- Few-Shot 예시 선택: 다양성·대표성·순서(최신 예시 마지막)가 중요
- 구조화 출력: JSON 스키마 명시로 파싱 안정성 향상
- 적대적 프롬프팅(Jailbreak) 방지: System Prompt 제약, 입력 검증 필수

📢 **섹션 요약 비유**: 구조화 출력 요청은 표 형식 보고서 지시 — "자유 형식으로 쓰세요" 대신 "이 양식에 맞춰 써주세요"가 훨씬 처리하기 편하다.

## Ⅴ. 기대효과 및 결론

프롬프트 엔지니어링은 LLM 성능을 파인튜닝 없이 극대화하는 저비용 고효율 기술이다. CoT는 복잡한 수학·논리·코딩 문제에서 30~50% 성능 향상을 보이고, ToT는 전략적 계획 문제에서 인간 수준에 근접한다. AI 시스템 설계 시 프롬프트 최적화→RAG→파인튜닝의 점진적 비용 증가 순서로 접근하는 것이 실용적 방법론이다.

📢 **섹션 요약 비유**: 프롬프트 엔지니어링은 LLM의 사용 설명서 숙달 — 같은 AI도 어떻게 지시하느냐에 따라 초보자와 전문가 수준의 차이가 난다.

### 📌 관련 개념 맵

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| Zero-Shot | 기법 | 예시 없는 지시 기반 응답 |
| Few-Shot | 기법 | 소수 예시 포함 프롬프트 |
| CoT (Chain-of-Thought) | 핵심 기법 | 단계별 추론 과정 유도 |
| ToT (Tree-of-Thought) | 고급 기법 | 복수 경로 트리 탐색 |
| ReAct | 에이전트 패턴 | 추론+도구 호출 반복 패턴 |

### 👶 어린이를 위한 3줄 비유 설명

1. Zero-Shot은 예시 없이 시험 — "이거 어떻게 해요?" 를 그냥 물어보는 것, AI가 알아서 하지만 가끔 틀려.
2. CoT는 수학 풀이 보여주기 — "1단계로 이걸 하고, 2단계로 저걸 해서" 라고 하면 AI가 훨씬 정확하게 풀어.
3. Few-Shot은 예제 먼저 보여주기 — 선생님이 문제 3개를 먼저 풀어 보여주면 학생(AI)이 패턴을 배워서 다음 문제를 더 잘 풀어!
