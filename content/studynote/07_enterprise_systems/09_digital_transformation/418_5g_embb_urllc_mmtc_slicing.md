## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 5G는 eMBB(초고속), uRLLC(초저지연·초신뢰), mMTC(대규모 IoT) 세 가지 서비스 클래스를 단일 인프라 위에서 네트워크 슬라이싱(Network Slicing)으로 논리적으로 분리하여 제공하는 5세대 이동통신이다.
> 2. **가치**: 네트워크 슬라이싱은 SDN/NFV 기반으로 물리 인프라를 가상 분리하여, VR 스트리밍·자율주행 제어·스마트 미터 등 전혀 다른 요건의 서비스가 하나의 네트워크를 공유하면서 각자의 SLA를 보장한다.
> 3. **판단 포인트**: 지연·대역폭·신뢰성의 서비스별 SLA 요건을 네트워크 슬라이스 설계에 반영하고, MEC(Multi-access Edge Computing)와 결합하여 지연을 최소화하는 것이 5G 아키텍처 설계의 핵심이다.

## Ⅰ. 개요 및 필요성

4G LTE는 단일 서비스 클래스(모바일 광대역)에 최적화되어 있어 자율주행(1ms 지연 요구)이나 스마트 미터(수십억 기기 동시 접속)에는 부적합하다. 5G는 IMT-2020 요건으로 최고 20Gbps, 지연 1ms, km² 당 100만 기기를 목표로 세 가지 서비스 클래스를 정의했다.

📢 **섹션 요약 비유**: 5G는 멀티레인 고속도로 — 일반 차(eMBB), 구급차(uRLLC), 자전거(mMTC)가 각자 전용 차선에서 방해 없이 달린다.

## Ⅱ. 아키텍처 및 핵심 원리

```
5G 서비스 클래스:
  eMBB (enhanced Mobile Broadband)
    - 최고 20Gbps, 지연 4ms
    - VR/AR, 8K 스트리밍, 모바일 게임

  uRLLC (ultra-Reliable Low Latency Communication)
    - 지연 1ms, 신뢰성 99.9999%
    - 자율주행, 원격 수술, 산업 제어

  mMTC (massive Machine Type Communications)
    - 최고 100만 기기/km²
    - 스마트 시티, 환경 센서, 스마트 그리드

네트워크 슬라이싱:
  ┌──────────────────────────────────────────────┐
  │           물리 네트워크 인프라                 │
  │  ┌──────────┐ ┌──────────┐ ┌──────────────┐  │
  │  │ Slice A  │ │ Slice B  │ │   Slice C    │  │
  │  │ eMBB     │ │ uRLLC    │ │   mMTC       │  │
  │  │ (20Gbps) │ │ (1ms)    │ │ (100만 기기) │  │
  │  └──────────┘ └──────────┘ └──────────────┘  │
  │       SDN/NFV 기반 가상 분리                   │
  └──────────────────────────────────────────────┘
```

| 특성 | 4G LTE | 5G |
|:---|:---|:---|
| 최고 속도 | 1Gbps | 20Gbps |
| 지연 | 10ms | 1ms |
| 연결 밀도 | 10만/km² | 100만/km² |
| 슬라이싱 | 미지원 | 지원 (SDN/NFV) |

📢 **섹션 요약 비유**: 네트워크 슬라이싱은 가상 전용선 — 같은 고속도로지만 구급차(uRLLC)는 항상 비어 있는 전용 차선을 보장받는다.

## Ⅲ. 비교 및 연결

SDN(Software Defined Networking): 제어 평면(Control Plane)과 데이터 평면(Data Plane)을 분리하여 소프트웨어로 네트워크 경로를 동적 제어. NFV(Network Function Virtualization): 라우터·방화벽 등 물리 기기를 가상화(vEPC, vRAN)하여 슬라이스별 배포. MEC(Multi-access Edge Computing): 5G 기지국 근처에 컴퓨팅을 배치하여 uRLLC 지연 최소화.

📢 **섹션 요약 비유**: SDN/NFV는 5G의 레고 시스템 — 물리 장비 대신 소프트웨어 블록으로 원하는 네트워크를 즉시 조립한다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- 기술사 핵심: eMBB/uRLLC/mMTC 세 클래스와 슬라이싱 메커니즘 명확히 설명
- 스마트 팩토리 uRLLC: MEC + 5G Private Network (전용망)
- V2X uRLLC: 1ms 지연 + 99.9999% 신뢰성 → SAE Level 4+ 요건 충족
- Open RAN: 기지국 가상화로 슬라이싱 유연성 향상

📢 **섹션 요약 비유**: 5G Private Network는 회사 전용 도로 — 공공 도로(퍼블릭 5G)보다 빠르고 보안이 높은 기업 전용 네트워크다.

## Ⅴ. 기대효과 및 결론

5G 네트워크 슬라이싱은 단일 인프라에서 완전히 다른 SLA 요건의 서비스를 동시에 제공하여, 이동통신 인프라의 경제성과 유연성을 극대화한다. 자율주행·원격 수술·스마트 팩토리·스마트 시티의 산업 디지털 전환을 가능하게 하는 핵심 통신 인프라다. 슬라이스 간 격리(Isolation) 강도와 동적 자원 재배분이 기술적 핵심 과제로 남아 있다.

📢 **섹션 요약 비유**: 5G는 도시 인프라의 신경망 — 교통(자동차), 의료, 공장, 가정 모두를 하나의 지능형 네트워크로 연결한다.

### 👶 어린이를 위한 3줄 비유 설명

1. 5G는 초고속 셔틀 버스 — 일반 승객(eMBB), 환자(uRLLC), 수백만 센서(mMTC)가 각자 맞는 속도와 안전도로 이동해.
2. 네트워크 슬라이싱은 같은 버스에 VIP 좌석, 일반 좌석, 화물칸을 나누는 것 — 하나의 버스가 세 가지 서비스를 동시에!
3. MEC는 동네 창고 — 중앙 창고(클라우드)까지 가지 않고 동네에서 바로 처리해서 엄청 빠르게!
