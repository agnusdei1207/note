## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 블록체인(Blockchain)은 트랜잭션을 블록으로 연결하고 네트워크 참여자가 복제 보유하는 DLT(Distributed Ledger Technology)로, 머클 트리(Merkle Tree) 해시 구조가 위변조 감지의 수학적 기반이다.
> 2. **가치**: 중앙 신뢰 기관 없이 참여자 간 합의로 거래의 불변성·투명성·추적성을 보장하여 디지털 자산 거래, 공급망 이력 관리, 스마트 컨트랙트 실행의 신뢰 인프라를 제공한다.
> 3. **판단 포인트**: 블록 헤더의 이전 블록 해시 체인과 머클 루트(Merkle Root)가 한 트랜잭션을 변조해도 전체 체인 해시가 무효화되는 원리가 핵심 보안 메커니즘이다.

## Ⅰ. 개요 및 필요성

2008년 Satoshi Nakamoto의 비트코인 논문에서 제안된 블록체인은 분산 원장(Distributed Ledger)의 한 형태로, 거래 기록을 블록에 담아 체인으로 연결하고 P2P 네트워크의 모든 노드가 복제 보유한다. 중앙 기관(은행, 공증 기관) 없이 암호학적 증명과 합의 알고리즘만으로 거래 무결성을 보장한다.

공개형(퍼블릭: Bitcoin, Ethereum), 허가형(프라이빗: Hyperledger Fabric), 컨소시엄형(반허가형: R3 Corda) 세 가지 유형으로 구분된다.

📢 **섹션 요약 비유**: 블록체인은 공개 공증 장부 — 모든 사람이 같은 장부를 가지고, 한 글자라도 바꾸면 모든 사람의 장부와 달라져 즉시 발각된다.

## Ⅱ. 아키텍처 및 핵심 원리

```
Block N-1              Block N               Block N+1
┌────────────┐        ┌────────────┐        ┌────────────┐
│ Header     │        │ Header     │        │ Header     │
│ Prev Hash──┼───────>│ Prev Hash──┼───────>│ Prev Hash  │
│ Merkle Root│        │ Merkle Root│        │ Merkle Root│
│ Timestamp  │        │ Timestamp  │        │ Timestamp  │
│ Nonce      │        │ Nonce      │        │ Nonce      │
├────────────┤        ├────────────┤        ├────────────┤
│ Tx1, Tx2   │        │ Tx3, Tx4   │        │ Tx5, Tx6   │
│ Tx3, ...   │        │ Tx5, ...   │        │ Tx7, ...   │
└────────────┘        └────────────┘        └────────────┘

Merkle Tree 구조:
      Root Hash
     /          \
  Hash(AB)    Hash(CD)
  /    \       /    \
H(A)  H(B)  H(C)  H(D)
 Tx_A  Tx_B  Tx_C  Tx_D
```

| 개념 | 설명 |
|:---|:---|
| 머클 트리 (Merkle Tree) | 트랜잭션 해시를 이진 트리로 구성 — 루트 해시 하나로 전체 검증 |
| 이전 블록 해시 | 체인 연결 — 한 블록 변조 시 이후 모든 블록 해시 무효화 |
| Nonce | PoW에서 목표 해시를 만족하는 임의값 |
| 합의 알고리즘 | PoW, PoS, BFT — 새 블록의 유효성 합의 방법 |

📢 **섹션 요약 비유**: 머클 트리는 블록체인의 지문 — 1만 개 트랜잭션 중 하나가 바뀌면 루트 해시(지문)가 달라져 즉시 탐지된다.

## Ⅲ. 비교 및 연결

| 유형 | 접근 권한 | 합의 | 대표 사례 |
|:---|:---|:---|:---|
| 퍼블릭 | 누구나 | PoW/PoS | Bitcoin, Ethereum |
| 프라이빗 | 특정 기관 | PBFT | Hyperledger Fabric |
| 컨소시엄 | 허가된 그룹 | Raft/BFT | R3 Corda, Quorum |

📢 **섹션 요약 비유**: 퍼블릭은 공원(모두 접근), 프라이빗은 사무실(직원만), 컨소시엄은 회원제 클럽(가입자만)이다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- 금융·공급망 투명성: 퍼블릭 블록체인(Ethereum) 또는 컨소시엄
- 기업 내부 프로세스 자동화: Hyperledger Fabric(프라이빗)
- 성능 vs 탈중앙화: 프라이빗은 TPS 수천, 퍼블릭 PoW는 TPS 7(Bitcoin)~30(ETH)
- 스마트 컨트랙트 가스비: 이더리움 고비용 → Layer 2 솔루션 필요

📢 **섹션 요약 비유**: 블록체인 유형 선택은 회의실 선택 — 공개 발표(퍼블릭), 내부 회의(프라이빗), 파트너 회의(컨소시엄) 각각 다른 방을 쓴다.

## Ⅴ. 기대효과 및 결론

블록체인은 신뢰 제3자 없는 거래와 불변 이력 관리를 가능하게 하지만, 낮은 TPS·높은 가스비·복잡한 거버넌스가 한계다. Layer 2(롤업), 지분증명(PoS) 전환, 샤딩으로 확장성 문제를 해결 중이며, 공급망·디지털 자산·신원 증명 도메인에서 실용화가 가장 빠르게 진행되고 있다.

📢 **섹션 요약 비유**: 블록체인은 강화 유리 — 깨기 어렵지만 두껍고 무거워서(낮은 처리 속도) 창문 전체를 이걸로 만들면 집이 무거워진다.

### 👶 어린이를 위한 3줄 비유 설명

1. 블록체인은 동네 모든 사람이 같은 일기장을 가지고 있는 것 — 누군가 몰래 고치면 다른 사람 일기장과 달라져서 바로 들켜.
2. 머클 트리는 블록 안 거래들의 요약 지문 — 거래 하나를 바꾸면 지문이 달라져서 위변조를 즉시 탐지해.
3. PoW는 수학 문제 경쟁 — 먼저 푼 사람(채굴자)이 새 블록을 추가하고 보상을 받아!
