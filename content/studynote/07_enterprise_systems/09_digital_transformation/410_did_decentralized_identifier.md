## 핵심 인사이트 (3줄 요약)

> 1. **본질**: DID(Decentralized Identifier)는 중앙 기관(정부·기업) 의존 없이 개인이 직접 생성·관리하는 탈중앙 디지털 신원 식별자로, W3C 표준이며 블록체인에 공개 키를 등록하여 신원을 검증한다.
> 2. **가치**: DID Document와 Verifiable Credential(VC)의 조합으로 "자기 주권 신원(SSI: Self-Sovereign Identity)"을 실현하여 개인정보 최소 공개와 선택적 신원 증명이 가능해진다.
> 3. **판단 포인트**: DID는 식별자, VC는 자격 증명, DID Resolver는 공개 키 조회 역할로 각 구성 요소의 역할 분리와 ZKP 결합으로 프라이버시 보호 신원 증명을 완성한다.

## Ⅰ. 개요 및 필요성

현재 디지털 신원은 구글·카카오·NAVER 등 중앙 플랫폼의 OAuth에 의존하여 개인정보가 플랫폼에 귀속된다. DID는 W3C 표준(2022)으로, 개인이 자신의 신원 식별자를 직접 생성·소유하고 블록체인에 공개 키를 등록하여 중앙 기관 없이 신원을 증명한다.

DID + VC(Verifiable Credential) + VP(Verifiable Presentation)의 조합이 완전한 탈중앙 신원 생태계를 형성한다.

📢 **섹션 요약 비유**: DID는 디지털 세계의 여권 발급소 — 정부(중앙 기관) 없이 내가 직접 나만의 여권(DID)을 만들고, 필요할 때만 필요한 정보만 보여준다.

## Ⅱ. 아키텍처 및 핵심 원리

```
SSI (자기 주권 신원) 생태계:
                       
  발급자(Issuer)         소유자(Holder)         검증자(Verifier)
  ─────────────         ─────────────         ───────────────
  대학/정부/기업  ──VC──>  개인 지갑           <──VP── 서비스
  (학위·자격증)   발급     (DID + VC 보관)      요청    (선택 공개)
                                ↑ DID Doc
                           블록체인 (공개 키 등록)
                           DID Resolver (조회)

DID 형식:
  did:method:identifier
  did:ethr:0x1234ABCD...  (Ethereum DID 메서드)
  did:web:example.com     (Web DID 메서드)
```

| 구성 요소 | 설명 | 역할 |
|:---|:---|:---|
| DID | 탈중앙 식별자 | 개인 소유 고유 URI |
| DID Document | 공개 키·서비스 엔드포인트 | DID 메타데이터 |
| VC (Verifiable Credential) | 발급자가 서명한 자격 증명 | 학위·면허·연령 증명 |
| VP (Verifiable Presentation) | 선택적 공개 자격 제출 | 필요한 정보만 제출 |
| SSI (Self-Sovereign Identity) | 자기 주권 신원 | DID 기반 신원 패러다임 |

📢 **섹션 요약 비유**: VC는 디지털 학위증 — 대학(발급자)이 서명하고, 내 지갑(소유자)에 보관하며, 취업(검증자) 시 필요한 것만 꺼내 보인다.

## Ⅲ. 비교 및 연결

| 항목 | 전통 OAuth | DID/SSI |
|:---|:---|:---|
| 신원 소유 | 플랫폼 | 개인 |
| 데이터 공개 | 플랫폼 요청대로 | 선택적 최소 공개 |
| 단일 실패점 | 플랫폼 다운 = 로그인 불가 | 탈중앙 |
| 프라이버시 | 낮음 | 높음 |

ZKP + DID: 영지식 증명으로 나이·자격을 비밀 유지하면서 "성인이다" "자격 보유자다"를 증명.

📢 **섹션 요약 비유**: DID+ZKP는 가면 무도회 신분증 — 가면(익명)을 쓰고도 입장 자격(성인, 회원)을 증명할 수 있다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- 공공 신원 서비스: 정부 DID (MOIS DID, 분산ID 시범사업)
- 기업 임직원 신원: Hyperledger Indy/Aries 기반 엔터프라이즈 SSI
- 교육 자격 증명: W3C VC 표준 + 블록체인 등록 (교육부 DID 학위증)
- DID 방법(Method): `did:ethr`, `did:web`, `did:ion`(Microsoft) 중 인프라에 맞게 선택

📢 **섹션 요약 비유**: DID 방법(Method) 선택은 여권 발행 국가 선택 — 어떤 인프라(블록체인/웹)에 공개 키를 등록하느냐에 따라 결정된다.

## Ⅴ. 기대효과 및 결론

DID/SSI는 개인정보 자기결정권을 기술적으로 실현하는 최선의 아키텍처로, 메타버스·Web 3.0·정부 디지털 서비스에서 채택이 확산되고 있다. 생태계 표준화 미흡과 기존 중앙 인증(OAuth) 대비 복잡한 UX가 현실적 장벽이며, 지갑 분실 시 신원 복구 메커니즘 설계가 필수 전제 조건이다.

📢 **섹션 요약 비유**: DID는 디지털 신분증의 미래 — 지금은 운전면허증(중앙 발급), 미래에는 내가 만든 디지털 신분증으로 어디서든 신원을 증명한다.

### 📌 관련 개념 맵

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| VC (Verifiable Credential) | 자격 증명 | 발급자 서명 검증 가능한 증명서 |
| VP (Verifiable Presentation) | 제출 방식 | 선택적 정보만 제출 |
| SSI (Self-Sovereign Identity) | 신원 패러다임 | 개인 소유 자기 주권 신원 |
| ZKP + DID | 프라이버시 강화 | 비밀 유지 자격 증명 |
| W3C DID 표준 | 국제 표준 | 탈중앙 식별자 W3C 권고안(2022) |

### 👶 어린이를 위한 3줄 비유 설명

1. DID는 내가 만든 디지털 이름표 — 선생님(중앙 기관)이 아니라 내가 직접 만들고, 내가 보관해.
2. VC는 선생님이 도장 찍어준 성적표 — 내 지갑에 넣어두고, 필요할 때만 꺼내서 보여줘.
3. ZKP+DID는 마스크 쓴 신분증 — 얼굴(개인정보)은 숨기고, 나이(자격)만 증명할 수 있어!
