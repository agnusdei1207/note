## 핵심 인사이트 (3줄 요약)

> 1. **본질**: DeFi(Decentralized Finance)는 스마트 컨트랙트 기반으로 은행·거래소 등 중앙 금융 기관 없이 대출·예금·거래·파생상품을 제공하는 탈중앙 금융 프로토콜이다.
> 2. **가치**: 24/7 무허가(Permissionless) 금융 접근, 플래시 론(Flash Loan)을 통한 무담보 즉시 대출, 자동화 시장 조성자(AMM: Automated Market Maker) 기반 탈중앙 거래가 핵심 혁신이다.
> 3. **판단 포인트**: 오라클(Oracle)은 블록체인과 외부 가격 데이터를 연결하는 필수 구성 요소이며, 오라클 조작이 DeFi의 주요 공격 벡터임을 이해해야 한다.

## Ⅰ. 개요 및 필요성

전통 금융은 KYC(고객 신원 확인)·은행 계좌·중앙 거래소가 필요하여 접근성이 제한된다. DeFi는 Ethereum 스마트 컨트랙트로 누구나 지갑만 있으면 대출·예금·거래에 참여할 수 있는 개방형 금융을 구현한다. 2020년 DeFi Summer 이후 TVL(Total Value Locked: 프로토콜에 잠긴 총 자산)이 수백억 달러 규모로 성장했다.

대표 프로토콜: Uniswap(DEX), Aave(대출), Compound(예금), MakerDAO(스테이블코인 DAI).

📢 **섹션 요약 비유**: DeFi는 로봇 은행 — 직원(은행원)이 없고, 규칙(스마트 컨트랙트)이 자동으로 모든 금융 서비스를 처리한다.

## Ⅱ. 아키텍처 및 핵심 원리

```
플래시 론(Flash Loan) 구조:
  한 트랜잭션 안에서:
  1. 무담보 대출 수령 (Aave에서 100만 ETH)
  2. 차익 거래 / 청산 / 담보 교체
  3. 대출 + 수수료 상환
  → 모두 실패 시 트랜잭션 전체 롤백 (아토믹)

AMM(Automated Market Maker) 구조:
  유동성 풀: ETH : USDC = x : y
  가격 공식: x * y = k (상수)
  → 구매 시 x 감소 → 가격(y/x) 상승
```

| 구성 요소 | 설명 | 대표 프로토콜 |
|:---|:---|:---|
| DEX (탈중앙 거래소) | AMM 기반 토큰 스왑 | Uniswap, Curve |
| 대출 프로토콜 | 과담보 대출·예금 이자 | Aave, Compound |
| 스테이블코인 | 가격 안정 토큰 | MakerDAO DAI, USDC |
| 플래시 론 | 무담보 단일 트랜잭션 대출 | Aave, dYdX |
| 오라클 | 외부 가격 데이터 제공 | Chainlink, Band |

📢 **섹션 요약 비유**: 플래시 론은 순간 이동 대출 — 빌렸다가 갚는 것이 1초 안(한 트랜잭션)에 끝나야 하며, 못 갚으면 처음부터 없던 일이 된다.

## Ⅲ. 비교 및 연결

오라클 문제(Oracle Problem): 블록체인은 외부 데이터를 직접 읽을 수 없어 오라클이 필요하다. 단일 오라클 조작(Price Manipulation) 시 DeFi 프로토콜 전체가 공격받을 수 있다. Chainlink는 탈중앙 오라클 네트워크로 복수 데이터 소스를 집계하여 이 문제를 완화한다.

📢 **섹션 요약 비유**: 오라클은 블록체인의 뉴스 앵커 — 외부 세계(가격 데이터)를 블록체인 주민에게 전달하는 역할이지만, 앵커가 거짓말하면 모두가 속는다.

## Ⅳ. 실무 적용 및 기술사 판단

**의사결정 포인트**:
- 차익 거래 자동화: 플래시 론 활용 (MEV, 봇 트레이딩)
- 오라클 보안: Chainlink VWAP + TWAP 방어 (단기 조작 방지)
- 스마트 컨트랙트 감사: Reentrancy, Oracle Manipulation, Flash Loan Attack 취약점 점검
- 규제 대응: 일부 국가 DeFi 규제 강화 추세 — 준법 검토 필수

📢 **섹션 요약 비유**: DeFi 보안은 요새 설계 — 스마트 컨트랙트 버그 하나가 수백억 원의 해킹으로 이어질 수 있다.

## Ⅴ. 기대효과 및 결론

DeFi는 금융 서비스의 접근 평등화와 비용 절감을 실현하는 잠재력을 지니며, 24/7 자동화 금융의 새 패러다임을 제시한다. 단, 스마트 컨트랙트 취약점, 오라클 조작, 규제 불확실성이 현실적 리스크다. 엔터프라이즈 적용 시 규정 준수(KYC/AML)를 갖춘 허가형 DeFi(Permissioned DeFi) 아키텍처가 현실적 경로다.

📢 **섹션 요약 비유**: DeFi는 은행 없는 인터넷 금융 — 초기 인터넷처럼 위험(해킹)도 크지만, 성숙하면 금융을 완전히 바꿀 수 있다.

### 📌 관련 개념 맵

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| AMM (Automated Market Maker) | 핵심 메커니즘 | x*y=k 공식 기반 자동 가격 결정 |
| Flash Loan | 혁신 기능 | 무담보 단일 트랜잭션 대출 |
| Oracle (Chainlink) | 필수 인프라 | 외부 데이터를 블록체인에 공급 |
| TVL (Total Value Locked) | 규모 지표 | 프로토콜에 잠긴 총 자산 가치 |
| MEV (Maximal Extractable Value) | 경제 현상 | 채굴자/검증자의 트랜잭션 순서 조작 |

### 👶 어린이를 위한 3줄 비유 설명

1. DeFi는 로봇 은행 — 직원이 없어도 24시간 내내 돈을 빌려주고, 이자를 받고, 환전해줘.
2. 플래시 론은 마법 대출 — 빌렸다가 갚는 것을 1초 안에 해야 하고, 못 갚으면 처음부터 없던 일!
3. 오라클은 블록체인의 뉴스 앵커 — 현실 세계 가격(주식, 달러)을 블록체인에 알려주는 다리야.
