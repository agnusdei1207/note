## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 행렬 곱셈(Matrix Multiplication)의 나이브 O(n³)를 스트라센(Strassen) 알고리즘이 O(n^2.807)로 개선했으며, 8번의 재귀 곱셈 대신 7번으로 줄이는 분할 정복 아이디어가 핵심이다.
> 2. **가치**: BLAS(Basic Linear Algebra Subprograms) 라이브러리와 GPU 병렬 행렬 곱셈은 머신러닝(딥러닝 추론)·과학 계산·컴퓨터 그래픽스의 성능을 결정하는 핵심 요소다.
> 3. **판단 포인트**: 이론적 최선은 O(n^2.37)(Williams 2023)이지만, 실용 임계점은 n ~ 수백 이상에서 Strassen이 유의미 — 캐시 효율·수치 안정성 고려 시 실용 선택은 BLAS나 cuBLAS가 표준이다.

---

## Ⅰ. 개요 및 필요성

n×n 행렬 두 개를 곱하는 것은 선형 대수의 가장 기본적 연산이지만, 나이브 방법은 n³번의 곱셈·덧셈을 요구한다.

- **나이브(Naive)**: C[i][j] = Σₖ A[i][k] · B[k][j] → O(n³).
- **스트라센(Strassen, 1969)**: O(n^log₂7 ≈ n^2.807).
- **현재 이론적 최선**: O(n^2.37) (Williams-Duan-Wu-Zhou 2023).
- **n=1000 비교**: n³ = 10⁹, n^2.807 ≈ 3.2×10⁸ → 약 3배 빠름.

딥러닝에서 행렬 곱셈(MatMul)은 Transformer·CNN·FC Layer의 대부분을 차지 → 성능 병목.

📢 **섹션 요약 비유**: 스트라센은 "8명이 나눠서 곱하던 계산을 7명이 해결하는 법을 발견" — 1명 줄이는 것이 재귀적으로 적용되면 수백만 번의 연산 차이가 생긴다.

---

## Ⅱ. 아키텍처 및 핵심 원리

### Strassen 블록 분할

n×n 행렬 A, B를 n/2 × n/2 블록으로 분할:

```
A = │ A₁₁  A₁₂ │    B = │ B₁₁  B₁₂ │
    │ A₂₁  A₂₂ │        │ B₂₁  B₂₂ │

나이브: C = A·B → 8번의 (n/2)×(n/2) 행렬 곱셈
T(n) = 8T(n/2) + O(n²) → O(n³)
```

### 스트라센의 7개 곱셈

```
┌──────────────────────────────────────────────────────────┐
│  M₁ = (A₁₁ + A₂₂)(B₁₁ + B₂₂)                          │
│  M₂ = (A₂₁ + A₂₂)B₁₁                                   │
│  M₃ = A₁₁(B₁₂ - B₂₂)                                   │
│  M₄ = A₂₂(B₂₁ - B₁₁)                                   │
│  M₅ = (A₁₁ + A₁₂)B₂₂                                   │
│  M₆ = (A₂₁ - A₁₁)(B₁₁ + B₁₂)                          │
│  M₇ = (A₁₂ - A₂₂)(B₂₁ + B₂₂)                          │
│                                                          │
│  C₁₁ = M₁ + M₄ - M₅ + M₇                               │
│  C₁₂ = M₃ + M₅                                          │
│  C₂₁ = M₂ + M₄                                          │
│  C₂₂ = M₁ - M₂ + M₃ + M₆                               │
│                                                          │
│  7번 곱셈 + 18번 덧셈/뺄셈                               │
│  T(n) = 7T(n/2) + O(n²) → O(n^log₂7) ≈ O(n^2.807)     │
└──────────────────────────────────────────────────────────┘
```

| 알고리즘 | 시간 복잡도 | 곱셈 수 | 비고 |
|:---|:---:|:---:|:---|
| 나이브 | O(n³) = O(n^3.000) | n³ | 단순, 캐시 우호적 |
| Strassen | O(n^2.807) | ~n^2.807 | 분할정복, 수치 오차↑ |
| Pan (1984) | O(n^2.522) | — | 이론 |
| Coppersmith-Winograd | O(n^2.376) | — | 이론 |
| Williams (2023) | O(n^2.372) | — | 현재 이론 최선 |

📢 **섹션 요약 비유**: 스트라센의 7개 곱셈 공식은 "8명 짜리 조 작업을 7명으로 재배치하는 창의적 분업" — 같은 결과를 더 적은 핵심 작업(곱셈)으로 달성한다.

## Ⅲ. 비교 및 연결

### 실용 알고리즘: BLAS와 GPU

이론적 최선 알고리즘과 달리, 실제 라이브러리는 캐시 효율과 SIMD(Single Instruction Multiple Data) 병렬 처리에 초점:

```
┌──────────────────────────────────────────────────────────┐
│  캐시 오블리비어스(Cache-Oblivious) 행렬 곱셈            │
│                                                          │
│  재귀적 분할 → 어떤 캐시 크기에도 자동 최적화           │
│  L1 캐시 크기 M, 블록 크기 √M × √M 자동 선택            │
│                                                          │
│  BLAS Level 3 (DGEMM):                                  │
│    OpenBLAS, MKL → CPU SIMD 최대 활용                   │
│    수백 GB/s 메모리 대역폭 활용                          │
│                                                          │
│  cuBLAS (NVIDIA):                                        │
│    Tensor Core → FP16 행렬 곱 수만 TOPS                 │
│    딥러닝 추론의 핵심 엔진                               │
└──────────────────────────────────────────────────────────┘
```

| 환경 | 라이브러리 | 특징 |
|:---|:---|:---|
| CPU 일반 | OpenBLAS | 오픈소스, SIMD 최적화 |
| CPU Intel | MKL | Intel SIMD 최적화 |
| GPU NVIDIA | cuBLAS | Tensor Core, FP16/BF16 |
| TPU | XLA | 행렬 곱 하드웨어 가속 |
| 양자 내성 암호 | — | NTT 행렬 곱 (모듈러) |

📢 **섹션 요약 비유**: BLAS는 "수학 공식보다 공장 자동화 라인" — 이론적 최선보다 실제 하드웨어에 맞춰 최적화된 생산 라인이 훨씬 빠르다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사 핵심:
1. **스트라센 임계점**: n이 수백 이하에서는 나이브가 실용적으로 더 빠를 수 있음 (오버헤드, 캐시 불리).
2. **딥러닝 성능 병목**: Transformer의 Self-Attention, FC Layer → 행렬 곱셈 → GPU Tensor Core.
3. **수치 안정성**: 스트라센은 덧셈/뺄셈이 많아 나이브보다 수치 오차↑ → 과학 계산에서 주의.
4. **캐시 오블리비어스**: 분할 크기를 런타임에 적응 → 어떤 캐시에도 효율적.

| n 범위 | 권장 알고리즘 | 이유 |
|:---:|:---|:---|
| n ≤ 64 | 나이브 | 캐시 적합, 오버헤드 없음 |
| 64 < n ≤ 수천 | BLAS DGEMM | SIMD 최적화 |
| n > 수천 | cuBLAS / Strassen | GPU 또는 이론 가속 |

📢 **섹션 요약 비유**: 행렬 곱셈 알고리즘 선택은 "공장 규모에 따른 생산 방식" — 소규모(n 작음)는 수작업이 빠르지만, 대규모는 자동화 라인(GPU/BLAS)이 압도적이다.

---

## Ⅴ. 기대효과 및 결론

행렬 곱셈을 이해하면:
- **딥러닝 성능 이해**: MatMul이 AI 추론의 95% 이상을 차지하는 이유와 Tensor Core의 의미 파악.
- **알고리즘 vs. 하드웨어 최적화**: 이론적 O 표기보다 실제 캐시/SIMD 효율이 중요한 경우 인식.
- **스트라센 원리**: "같은 결과를 얻는 더 적은 핵심 연산"을 찾는 분할 정복 사고 체득.

📢 **섹션 요약 비유**: 행렬 곱셈은 현대 AI의 "근육" — ChatGPT부터 이미지 인식까지, 모든 신경망은 본질적으로 빠른 행렬 곱셈 엔진 위에서 작동한다.

### 📌 관련 개념 맵

| 개념 | 관계 |
|:---|:---|
| 나이브 MatMul | O(n³), 단순·안정 |
| Strassen | O(n^2.807), 7 재귀 곱셈 |
| BLAS DGEMM | CPU SIMD 최적화 표준 |
| cuBLAS | GPU Tensor Core, 딥러닝 핵심 |
| 캐시 오블리비어스 | 어떤 캐시에도 자동 최적화 |
| 이론 최선 | O(n^2.372), Williams 2023 |

### 📈 관련 키워드 및 발전 흐름도

```text
[나이브 행렬 곱셈 (Naive MatMul) — O(n³)]
    │
    ▼
[스트라센 알고리즘 (Strassen) — O(n^2.807)]
    │
    ▼
[BLAS / cuBLAS — CPU/GPU SIMD 최적화]
    │
    ▼
[텐서 코어 (Tensor Core) — 딥러닝 행렬 가속]
    │
    ▼
[Flash Attention — 메모리 효율 행렬 연산]
```

행렬 곱셈 알고리즘이 기초 수식에서 하드웨어 최적화와 딥러닝 연산 혁신으로 발전한 흐름이다.
