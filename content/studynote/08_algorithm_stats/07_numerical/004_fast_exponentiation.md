## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 고속 거듭제곱(Fast Exponentiation, 이진 지수법)은 a^n 계산을 분할 정복으로 O(log n) 곱셈에 수행하는 알고리즘으로, n이 클 때 나이브 O(n) 대비 천문학적 차이를 보인다.
> 2. **가치**: 모듈러 지수법(Modular Exponentiation, a^n mod m)은 RSA 암호화·복호화의 핵심 연산이며, 행렬 거듭제곱으로 확장하면 피보나치 수열도 O(log n)에 계산된다.
> 3. **판단 포인트**: 이진 표현(Binary Representation)으로 n의 각 비트를 처리 — 비트가 1이면 곱셈 추가, 매 단계에서 지수를 제곱 → 총 O(log n)번의 곱셈으로 완료.

---

## Ⅰ. 개요 및 필요성

a^1000000000을 계산할 때 단순히 a를 10억 번 곱하는 것은 비현실적이다. 고속 거듭제곱은 이를 단 30번의 곱셈으로 처리한다.

- **분할 정복 원리**:
  - n이 짝수: a^n = (a^(n/2))²
  - n이 홀수: a^n = a · a^(n-1)
- **이진 지수법(Binary Exponentiation)**: n을 이진수로 표현하여 각 비트에 해당하는 제곱값을 조합.
- **모듈러 지수법**: a^n mod m → 암호화(RSA), 나머지 연산 중 오버플로 방지.

📢 **섹션 요약 비유**: 고속 거듭제곱은 "계단을 하나씩 오르지 않고, 1→2→4→8→16층을 점프하는 것" — 2의 배수 점프를 반복해서 O(log n)번만에 꼭대기에 도달한다.

---

## Ⅱ. 아키텍처 및 핵심 원리

### 재귀 구현 및 분석

```python
def fast_pow(a, n, m=None):
    if n == 0:
        return 1
    if n % 2 == 0:
        half = fast_pow(a, n // 2, m)
        result = half * half
    else:
        result = a * fast_pow(a, n - 1, m)
    return result % m if m else result
```

### 이진 지수법 순서도

```
a^13 계산 (13 = 1101₂)

n=13   비트=1: result *= a    → result = a
       a = a²

n=6    비트=0: (스킵)
       a = a⁴

n=3    비트=1: result *= a⁴   → result = a⁵
       a = a⁸

n=1    비트=1: result *= a⁸   → result = a¹³
       완료

┌────────────────────────────────────────────────────┐
│  반복 구현 (Iterative)                             │
│                                                    │
│  result = 1, base = a                              │
│  while n > 0:                                      │
│      if n & 1:     ← 최하위 비트 확인             │
│          result *= base                            │
│      base *= base  ← 제곱 누적                    │
│      n >>= 1       ← 오른쪽 시프트                │
│  return result                                     │
└────────────────────────────────────────────────────┘
```

| 방법 | 시간 복잡도 | 공간 복잡도 |
|:---|:---:|:---:|
| 나이브 (반복 곱) | O(n) | O(1) |
| 고속 거듭제곱 (재귀) | O(log n) | O(log n) 스택 |
| 고속 거듭제곱 (반복) | O(log n) | O(1) |

📢 **섹션 요약 비유**: 이진 지수법은 n의 이진 표현이 "곱할 때 쓸 지시서" — 비트가 1인 자리마다 현재까지의 제곱값을 곱하고, 0이면 그냥 넘어간다.

## Ⅲ. 비교 및 연결

### 행렬 거듭제곱 (Matrix Exponentiation)

행렬도 제곱 가능 → 선형 점화식을 O(log n)에 계산:

피보나치 수열 F(n):

```
┌         ┐ⁿ   ┌         ┐
│  1   1  │  × │  F(1)   │  =  │  F(n+1)  │
│  1   0  │    │  F(0)   │     │  F(n)    │
└         ┘    └         ┘     └          ┘
```

행렬 2×2의 n제곱을 O(log n)에 계산 → F(n)을 O(log n)에 계산 가능 (나이브 O(n) 대비).

### 응용 분야

| 분야 | 활용 |
|:---|:---|
| RSA 암호화 | m^e mod n 계산 (e, n이 수백 비트) |
| 디피-헬만 키 교환 | g^a mod p 계산 |
| 경쟁 프로그래밍 | 큰 수의 나머지 연산 |
| 피보나치/선형 점화식 | 행렬 거듭제곱으로 O(log n) |

📢 **섹션 요약 비유**: 행렬 거듭제곱은 "점화식이라는 피아노 연주를 악보 압축으로 빠르게 연주하는 것" — 100억 번째 음표를 처음부터 칠 필요 없이 O(log n)번에 바로 꺼낸다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사 핵심 포인트:
1. **RSA 연산**: 공개키 e (보통 65537 = 2¹⁶+1)로 m^e mod n 계산 → 고속 거듭제곱 없이는 불가.
2. **모듈러 지수법**: 중간 결과를 mod m으로 줄여 오버플로 방지 — 암호 구현에 필수.
3. **행렬 지수법**: 동적계획법 최적화 — 상태 전이가 선형이면 행렬로 모델링 후 O(log n) 가속.

📢 **섹션 요약 비유**: 고속 거듭제곱은 암호화의 "스타터 모터" — RSA, DH 등 모든 공개키 암호가 이 알고리즘 없이는 실용적으로 동작하지 않는다.

---

## Ⅴ. 기대효과 및 결론

고속 거듭제곱을 이해하면:
- **알고리즘 설계 철학**: 분할 정복 → O(log n)으로 지수 문제 해결하는 패턴 체득.
- **암호 구현**: RSA, ECC 등 공개키 암호의 핵심 연산 구현 이해.
- **행렬 지수법 확장**: DP 최적화, 그래프 경로 수 계산, 선형 점화식 모두 적용.

📢 **섹션 요약 비유**: 고속 거듭제곱은 알고리즘의 "복리 계산기" — 원리는 단순하지만, 암호화부터 피보나치까지 수많은 문제의 핵심 엔진이다.
