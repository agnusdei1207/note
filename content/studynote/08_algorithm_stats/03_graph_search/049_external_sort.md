# 049. 외부 정렬 (External Sort)

## 핵심 인사이트 (3줄 요약)
> 1. 모든 데이터를 RAM에 통째로 올려서 비교하는 기존 수학 잣대(내부 정렬, Internal)를 포기하고, **"메모리가 감당 가능한 청크(블록 단위)만큼만 읽어 올려라"**는 디스크 Paging 회피 기반 합병 프레임워크다.
> 2. 조각난 블록 단위가 메모리에 올라오면 그 안에서 극한으로 퀵소트를 돌려 정렬하고 다시 디스크 임시 파일 찌꺼기로 내려쓴다(Run Generation).
> 3. 그리고 파일 시스템의 수십 개 임시 파일 맨 앞줄을 조금씩 끌어올려 포인터 비교를 하며 최종 1개의 출력 파일로 머지(K-Way Merge 통폐합)시키는 아키텍처망 지능이다.

---
## Ⅰ. 아키텍처 및 원리
```text
  [ External K-Way Merge 로직망 분산 분할 ]
    1단계 (Run 생성) : 100GB 데이터를 1GB(RAM 크기)씩 잘라 올림 -> 내부 정렬 -> 100개의 조각 임시파일 디스크 저장
    2단계 (초병합) : 100개의 조각 임시파일 맨 앞 데이터를 RAM에 조금씩 투입 (K-Way 버퍼망)
                       가장 작은 놈부터 뽑아서 최종 Output 파일에 순차적 Append 쓰기 처리!
```
---
## Ⅱ. 실무 적용 및 결론
거대 Oracle DB 엔진이 인덱스 없는 상태에서 Group by / Order By 풀 스캔 쿼리를 얻어맞아 `Sort/Hash Join` 연산을 돌리다 `Temp Tablespace` 즉 하드디스크의 페이징 공간을 피 터지게 사용하는 방식의 매커니즘이 이것이다. 이 단계로 넘어가면 디스크 I/O 레이턴시에 의해 시스템 응답 속도가 무참히 박살 나므로, 튜닝의 핵심은 이 외부 정렬 스왑을 막아내는 것.
