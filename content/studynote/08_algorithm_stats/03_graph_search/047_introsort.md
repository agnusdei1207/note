# 047. 인트로 정렬 (Introsort)

## 핵심 인사이트 (3줄 요약)
> 1. 캐시 적중률(Cache Hit)이 우월하여 수학적 상수가 무조건 가장 빠른 **퀵 정렬(Quick Sort)을 메인 가속계 엔진**으로 선봉 탑재한다.
> 2. 재귀 호출 뎁스(트리 깊이 깊어짐)를 은밀히 감시하다가, 한계치($2 \times \log N$)를 넘어서며 퀵 정렬이 맛이가기(최악의 $O(N^2)$ 퇴행 징후) 시작하면 지체없이 강제로 브레이크를 걸고 O(N log N)이 절대 보강장치인 **힙 정렬(Heap Sort)** 로 우회 강제 변환한다.
> 3. 배열 구간 파티셔닝 사이즈가 아주 작아지면 함수 호출 오버헤드를 막기 위해 재빠르게 **삽입 정렬(Insert Sort)** 로 마무리 타격을 줘버려 종료 속도를 극대화한다.

---
## Ⅰ. 아키텍처 및 원리
```text
  [ Introsort = Quick + Heap + Insert 하이브리드 엔진 파이플레이어 ]
    START 
     -> 퀵 소트 돌격 시도
     -> [감시망 작동] 재귀 깊이가 이례적으로 너무 깊은가? 
           NO : 그냥 퀵 소트 끝까지 밀어버려라.
           YES : 최악의 케이스 감지! 남은 구간을 전부 힙 소트 트란지션으로 대체 안전 보장!
     -> 마지막으로 자잘한 요소(Length < 16)들은 삽입 정렬로 초고속 먼지 제거. 
    END
```
---
## Ⅱ. 실무 적용 및 결론
현대 C++ STL `std::sort()` 템플릿의 근간이자 마이크로소프트 .NET 프레임워크의 코어 정렬 심장이다. Timsort가 파이썬 등 객체 참조/안정성을 중시하는 도메인을 먹었다면, Introsort는 C/C++ 같은 로우 레벨 환경에서 불안정(Unstable)하지만 1바이트의 메모리 낭비 제로 공간(In-place)과 극한의 절대 속도를 우주 끝까지 튜닝해 내야 할 때 쓰이는 궁극의 로우 레벨 타격기다.
