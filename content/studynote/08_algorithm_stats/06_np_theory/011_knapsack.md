## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 0/1 배낭 문제(0/1 Knapsack)는 무게·가치를 가진 아이템들을 무게 한도 내에서 가치 최대화로 선택하는 NP-Hard 최적화 문제로, 결정 버전은 NP-Complete이다.
> 2. **가치**: 용량 W에 대한 DP(Dynamic Programming)가 O(nW) 의사 다항 시간(Pseudo-Polynomial)으로 풀리며, FPTAS(Fully Polynomial-Time Approximation Scheme)도 존재하여 실용적으로는 효율적이다.
> 3. **판단 포인트**: O(nW)는 입력 비트 수 기준으로 지수이므로 진정한 다항 시간이 아니다 — W가 2ⁿ처럼 크면 여전히 비효율적이며, 이것이 NP-Hard를 벗어나지 못하는 이유다.

---

## Ⅰ. 개요 및 필요성

**배낭 문제(Knapsack Problem)**는 n개의 아이템 각각에 무게 wᵢ와 가치 vᵢ가 주어지고 배낭의 무게 한도 W가 주어질 때, 한도를 초과하지 않으면서 가치를 최대화하는 아이템 선택 문제다.

- **0/1 Knapsack**: 각 아이템을 넣거나(1) 안 넣거나(0). 결정 버전 NP-Complete.
- **Fractional Knapsack**: 아이템을 쪼갤 수 있음 → Greedy O(n log n)로 최적 해 가능.
- **Bounded Knapsack**: 각 아이템이 최대 cᵢ개 → DP 변형.
- **Unbounded Knapsack**: 각 아이템 무한정 사용 가능 → DP 변형.
- **현실 적용**: 자원 배분, 금융 포트폴리오, 화물 적재, 클라우드 VM 패킹.

📢 **섹션 요약 비유**: 배낭 문제는 "한정된 여행 가방에 가장 값어치 있는 짐을 골라 넣는 것" — 쪼갤 수 있으면(Fractional) 쉽지만, 통째로 넣거나 빼야(0/1) 하면 아주 어렵다.

## Ⅱ. 아키텍처 및 핵심 원리

### 0/1 Knapsack DP 테이블

```
dp[i][w] = 처음 i개 아이템, 용량 w 배낭의 최대 가치

점화식:
  dp[i][w] = dp[i-1][w]                              (아이템 i 미선택)
  dp[i][w] = max(dp[i-1][w],  dp[i-1][w-wᵢ] + vᵢ)  (w ≥ wᵢ일 때)
```

```
┌──────────────────────────────────────────────────────┐
│  예시: n=3, W=6                                      │
│  items: (w=2,v=3), (w=3,v=4), (w=4,v=5)             │
│                                                      │
│  w →  0   1   2   3   4   5   6                     │
│  ─────────────────────────────────────               │
│  i=0  0   0   0   0   0   0   0                     │
│  i=1  0   0   3   3   3   3   3  (w=2,v=3)          │
│  i=2  0   0   3   4   4   7   7  (w=3,v=4)          │
│  i=3  0   0   3   4   5   7   8  (w=4,v=5)          │
│                                                      │
│  최적: dp[3][6] = 8  (아이템1+아이템2 선택)         │
└──────────────────────────────────────────────────────┘
```

### 알고리즘 복잡도 비교

| 알고리즘 | 시간 | 공간 | 특징 |
|:---|:---:|:---:|:---|
| 0/1 DP | O(nW) | O(nW) | 의사 다항 시간 |
| 0/1 DP (공간 최적화) | O(nW) | O(W) | 1D 배열 사용 |
| Fractional Greedy | O(n log n) | O(1) | 정확 해, 분수 허용 시 |
| FPTAS | O(n²/ε) | O(n/ε) | (1+ε)-근사 보장 |
| Branch & Bound | O(2ⁿ) 최악 | — | 상한 가지치기 |

### FPTAS 원리

가치를 스케일링: v'ᵢ = ⌊vᵢ / K⌋, K = ε · max(vᵢ) / n  
→ 스케일된 가치로 DP O(n²/ε) 수행  
→ 결과는 OPT의 (1+ε) 이내 보장.

📢 **섹션 요약 비유**: FPTAS는 "아이템 가격표를 대략적으로 반올림해서 계산" — 약간의 오차를 허용하면 훨씬 빠르게 최적에 가까운 답을 얻는다.

## Ⅲ. 비교 및 연결

### 의사 다항 시간 vs. 진정한 다항 시간

| 구분 | 의사 다항 (Pseudo-Poly) | 진정한 다항 (Truly Poly) |
|:---|:---|:---|
| 정의 | 입력 값(W)의 다항식 | 입력 비트 수의 다항식 |
| 예시 | Knapsack O(nW) | Sorting O(n log n) |
| W = 2¹⁰⁰ 이면? | 비현실적 지수 시간 | 무관 |
| 의미 | NP-Hard 해결 아님 | P에 속함 |

W가 입력으로 이진수로 표현될 때 비트 수 = log W이므로, O(nW) = O(n · 2^(log W)) → 지수 시간.

### Subset Sum과의 관계

Subset Sum(부분 합) 문제: 정수 집합에서 합이 T인 부분집합 존재 여부  
→ Knapsack의 특수 케이스 (vᵢ = wᵢ, V = W)  
→ Subset Sum ≤_p Knapsack (Knapsack이 Subset Sum보다 어렵거나 동등)

📢 **섹션 요약 비유**: Knapsack DP는 마치 "각 아이템을 선택할지 말지를 행·열 테이블로 채워나가는 퍼즐" — 테이블이 작으면 빠르지만, 무게 한도가 천문학적으로 크면 테이블 자체가 우주를 넘는다.

## Ⅳ. 실무 적용 및 기술사 판단

| 분야 | 활용 |
|:---|:---|
| 자원 배분 | 프로젝트 예산·인력 최적 배분 |
| 금융 포트폴리오 | 자본 제약 하 수익 최대화 |
| 클라우드 VM 패킹 | 서버 자원 한도 내 최적 VM 배치 |
| 화물 적재 | 항공·선박 무게 제한 내 최대 가치 화물 |

기술사 핵심 포인트:
1. 0/1 Knapsack 결정 버전 = NP-Complete (Subset Sum ≤_p Knapsack).
2. O(nW) DP = 의사 다항 시간 (W가 클 때 비효율적, NP-Hard 상태 유지).
3. FPTAS 존재 = "원하는 정밀도로 다항 시간에 근사 가능"한 최강 근사 보장.
4. Fractional Knapsack은 P (Greedy 최적) — "쪼갤 수 있냐"가 핵심 분기점.

📢 **섹션 요약 비유**: 배낭 문제는 IT 자원 배분의 핵심 — 클라우드 스케줄러, 금융 최적화, 물류 시스템이 모두 이 문제의 변형이다.

---

## Ⅴ. 기대효과 및 결론

배낭 문제를 이해하면:
- **DP 패턴 완성**: 0/1 Knapsack DP는 모든 선택형 DP의 기본 패턴 — 코딩 테스트부터 시스템 설계까지 활용.
- **의사 다항 시간 이해**: W 크기에 따른 실용성 판단 능력.
- **FPTAS 적용**: ε에 따른 시간-정밀도 트레이드오프 설계.

📢 **섹션 요약 비유**: 배낭 문제의 DP 테이블은 알고리즘의 "가격 비교 표" — 각 셀이 "이 아이템까지, 이 무게에서, 최선의 선택"을 기록하는 최적화 지도다.

### 📈 관련 키워드 및 발전 흐름도

```text
[조합 최적화 (Combinatorial Optimization) — 유한 자원·최대 가치 문제 정의]
    │
    ▼
[배낭 문제 (Knapsack Problem) — 0/1·분할·무제한 변형으로 분류]
    │
    ▼
[동적 계획법 (Dynamic Programming) — DP 테이블로 O(nW) 최적 해 도출]
    │
    ▼
[분기 한정 / 탐욕법 (Branch & Bound / Greedy) — NP-Hard 대응 근사 전략]
    │
    ▼
[자원 스케줄링 / 클라우드 빈패킹 (Bin Packing) — 실무 인프라 최적 배치 응용]
```

이 흐름은 고전 배낭 문제의 DP 풀이에서 출발해 클라우드 자원 스케줄링까지 이어지는 조합 최적화의 실무 적용 경로를 나타낸다.
