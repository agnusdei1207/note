## 핵심 인사이트 (3줄 요약)

> 1. **본질**: SAT(Boolean Satisfiability Problem, 불 만족 가능성 문제)는 주어진 불 공식을 참(TRUE)으로 만드는 변수 할당이 존재하는지를 판별하는 문제로, Cook-Levin 정리(1971)에 의해 최초로 NP-Complete임이 증명됐다.
> 2. **가치**: SAT는 모든 NP-Complete 증명의 출발점이자, 현대 SAT 솔버(수백만 변수 처리)를 통해 하드웨어 검증·AI 계획·소프트웨어 분석에 실용적으로 활용된다.
> 3. **판단 포인트**: 3-SAT는 NP-Complete이지만 2-SAT는 SCC(Strongly Connected Component, 강연결 요소) 기반 O(n+m)으로 다항 시간에 풀리는 핵심 차이를 기억해야 한다.

---

## Ⅰ. 개요 및 필요성

**SAT**는 변수 x₁, x₂, ..., xₙ과 AND(∧), OR(∨), NOT(¬) 연산으로 구성된 불 공식(Boolean Formula)이 주어질 때, 모든 변수에 참/거짓을 할당하여 전체 공식이 TRUE가 되도록 하는 할당이 존재하는지를 판별한다.

- **Cook-Levin 정리(1971)**: Stephen Cook(1971)과 독립적으로 Leonid Levin(1973)이 증명. SAT는 NP-Complete.
- **중요성**: 이후 수천 개의 NP-Complete 문제가 SAT(또는 3-SAT)로부터의 환산으로 증명됨.
- **SAT의 변형**:
  - **CNF-SAT (Conjunctive Normal Form)**: AND로 연결된 절(Clause)들의 집합.
  - **3-SAT**: 각 절에 정확히 3개의 리터럴(Literal). 표준 NP-Complete 출발점.
  - **2-SAT**: 각 절에 2개의 리터럴. **P에 속함** — SCC로 O(n+m) 해결.
  - **Horn-SAT**: 각 절에 양(+) 리터럴 최대 1개. **P에 속함** — 단위 전파(Unit Propagation).

📢 **섹션 요약 비유**: SAT는 "다이얼 수십 개가 달린 자물쇠에서 열리는 조합이 있냐"를 묻는 것 — 다이얼 3개짜리(3-SAT)는 어렵지만, 2개짜리(2-SAT)는 규칙이 있어 빠르게 풀린다.

## Ⅱ. 아키텍처 및 핵심 원리

### CNF 공식 예시

```
φ = (x₁ ∨ ¬x₂ ∨ x₃) ∧ (¬x₁ ∨ x₂) ∧ (x₂ ∨ ¬x₃)
```

```
┌────────────────────────────────────────────────────────┐
│  CNF (Conjunctive Normal Form) 구조                    │
│                                                        │
│  φ = C₁  ∧  C₂  ∧  C₃  ∧ ... ∧  Cₖ                  │
│       │                                                │
│       ▼                                                │
│  C₁ = (L₁ ∨ L₂ ∨ L₃)   (절, Clause)                  │
│        │   │   │                                       │
│        ▼   ▼   ▼                                       │
│       xᵢ  ¬xⱼ  xₖ      (리터럴, Literal)              │
│                                                        │
│  3-SAT: 각 절의 리터럴 수 = 정확히 3                   │
│  만족 할당: x₁=T, x₂=T, x₃=F → φ = TRUE ?            │
└────────────────────────────────────────────────────────┘
```

### 알고리즘 비교

| 알고리즘 | 대상 | 시간 복잡도 | 방법 |
|:---|:---|:---:|:---|
| DPLL | CNF-SAT | O(2ⁿ) 최악 | 백트래킹 + 단위 전파 |
| CDCL (Conflict-Driven Clause Learning) | CNF-SAT | 실용적 | 충돌 학습 + 비크로노 백트래킹 |
| SCC (Kosaraju/Tarjan) | 2-SAT | O(n+m) | 함의 그래프 강연결 요소 |
| 단위 전파 | Horn-SAT | O(n·m) | 단위 절 반복 전파 |

### 2-SAT vs. 3-SAT 핵심 차이

```
2-SAT: (x₁ ∨ x₂) 절 → 함의(Implication)로 변환
  ¬x₁ → x₂,  ¬x₂ → x₁  (방향 그래프)
  → SCC로 모순 없으면 만족 가능 (O(n+m))

3-SAT: (x₁ ∨ x₂ ∨ x₃) 절
  → 함의 변환 불가 → NP-Complete
```

📢 **섹션 요약 비유**: 2-SAT는 "두 사람 중 한 명이 진실을 말하면 OK"라서 관계망 분석으로 빠르게 풀리지만, 3-SAT는 "세 사람 중 한 명이 진실"이라서 모든 경우를 따져야 한다.

## Ⅲ. 비교 및 연결

### SAT 솔버의 현실 성능

현대 SAT 솔버(MiniSAT, Z3, CaDiCaL)는 수백만 변수·수천만 절의 산업 인스턴스를 수 초~수 분에 처리한다. 이는 CDCL 알고리즘의 실용적 성능 때문이며, 이론적 지수 시간 최악과는 무관하다.

| 적용 분야 | 구체적 활용 |
|:---|:---|
| 하드웨어 검증 (HW Verification) | 회로 등가성 검사, 버그 탐지 |
| AI 계획 (AI Planning) | 계획 문제를 SAT로 인코딩 |
| 소프트웨어 분석 | 프로그램 취약점 탐지 (기호 실행) |
| 암호 분석 | 해시 역상 탐색, 블록 암호 분석 |
| 생물정보학 | 단백질 구조 예측 |

📢 **섹션 요약 비유**: SAT 솔버는 마치 초고속 자물쇠 해독사 — 이론상 열기 어렵지만, 실제 자물쇠의 패턴을 학습해서 대부분을 빠르게 열어버린다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사 시험에서 SAT 관련 핵심:

1. **Cook-Levin 정리**: SAT가 NP-Complete임의 의의 — 이후 NP-Complete 문제들의 연쇄 증명 기반.
2. **3-SAT의 중요성**: 많은 NP-Complete 증명이 SAT 대신 3-SAT에서 환산 시작 (절 구조가 단순해서 환산이 용이).
3. **2-SAT 구분**: 실무에서 문제가 2-SAT로 모델링 가능한지 확인하면 다항 시간 해법 존재.
4. **SAT 솔버 활용**: 소프트웨어 검증(Formal Verification), 모델 검사(Model Checking)에 SAT/SMT(Satisfiability Modulo Theories) 솔버 사용.

📢 **섹션 요약 비유**: SAT는 계산 이론의 "로제타석(Rosetta Stone)" — 모든 NP 문제를 해독하는 공통 언어로, 실제로는 강력한 SAT 솔버가 현대 산업 현장에서 매일 쓰인다.

---

## Ⅴ. 기대효과 및 결론

SAT를 이해하면:
- **NP 이론의 기초 완성**: Cook-Levin → 환산 체계 → NP-Complete 생태계 전체를 이해하는 핵심.
- **실무 SAT 솔버 활용**: Z3, MiniSAT 등을 이용한 자동화 검증·테스트 케이스 생성.
- **2-SAT 최적화**: 제약 만족 문제(CSP) 중 2-SAT로 환원 가능한 경우 O(n+m) 해법 적용.

SAT는 이론적으로 NP-Complete의 원점이자, 실용적으로 현대 검증 산업의 핵심 엔진이다.

📢 **섹션 요약 비유**: SAT는 수학 이론의 왕(Cook-Levin)이면서 동시에 산업 현장의 일꾼(현대 SAT 솔버) — 이론과 실용의 두 얼굴을 모두 가진 문제다.
