## 핵심 인사이트 (3줄 요약)

> 1. **본질**: TSP(Traveling Salesman Problem, 외판원 문제)는 모든 도시를 정확히 한 번씩 방문하고 출발지로 돌아오는 최단 해밀턴 순환을 찾는 문제로, 최적화 버전은 NP-Hard, 결정 버전은 NP-Complete이다.
> 2. **가치**: 삼각 부등식 TSP에서 크리스토피디스(Christofides) 알고리즘이 1.5-근사를 보장하며, 정확 해법으로는 Held-Karp DP O(2ⁿ·n²)가 최선이다.
> 3. **판단 포인트**: 일반 TSP는 어떤 상수 비율로도 근사 불가능(P ≠ NP 가정)하지만, 삼각 부등식을 만족하는 메트릭 TSP는 1.5-근사(Christofides, 2020년 이후 1.5-ε으로 개선)가 존재한다.

---

## Ⅰ. 개요 및 필요성

**TSP**는 n개의 도시와 각 도시 쌍 사이의 거리가 주어질 때, 모든 도시를 정확히 한 번 방문하고 출발 도시로 돌아오는 최단 경로(Hamiltonian Cycle, 해밀턴 순환)를 찾는 문제다.

- **결정 버전**: 비용 k 이하의 해밀턴 순환이 존재하는가? → NP-Complete.
- **최적화 버전**: 최단 해밀턴 순환 찾기 → NP-Hard.
- **현실 적용**: 택배 경로 최적화, PCB(Printed Circuit Board) 드릴링 경로, DNA 시퀀싱.
- **규모**: 도시 수 n에서 경우의 수 (n-1)!/2 → n=20이면 약 6.08 × 10¹⁶.

📢 **섹션 요약 비유**: TSP는 "한 번씩만 들러 집에 돌아오는 배달부의 최단 경로" — 도시가 20개만 넘어도 모든 경우를 확인하는 것은 우주의 나이보다 오래 걸린다.

## Ⅱ. 아키텍처 및 핵심 원리

### 알고리즘 비교

```
┌──────────────────────────────────────────────────────────┐
│  TSP 알고리즘 스펙트럼                                   │
│                                                          │
│  정확(Exact)                 근사(Approximation)         │
│  ──────────────             ──────────────────           │
│  나이브: O(n!)               그리디 삽입: 실용, 보장 없음 │
│  Held-Karp DP: O(2ⁿ·n²)    2-근사(MST): 메트릭 TSP      │
│  Branch&Bound: 프루닝       Christofides: 1.5-근사        │
│                              LKH: 최강 휴리스틱           │
│                                                          │
│  n ≤ 20      n ≤ 30         n ≤ 수천        n ≤ 수백만  │
└──────────────────────────────────────────────────────────┘
```

### Christofides 알고리즘 (1976, 메트릭 TSP 1.5-근사)

```
┌──────────────────────────────────────────────────────────┐
│  Christofides Algorithm                                  │
│                                                          │
│  1. MST(Minimum Spanning Tree) T 계산       O(E log V)  │
│         │                                               │
│         ▼                                               │
│  2. T에서 홀수 차수(Odd-degree) 정점 집합 O 추출        │
│         │                                               │
│         ▼                                               │
│  3. O에서 최소 완전 매칭(Min Perfect Matching) M 계산   │
│         │                                               │
│         ▼                                               │
│  4. T ∪ M로 오일러 그래프 구성                          │
│         │                                               │
│         ▼                                               │
│  5. 오일러 순환(Eulerian Circuit) 찾기                  │
│         │                                               │
│         ▼                                               │
│  6. 중복 정점 제거 → 해밀턴 순환                        │
│                                                          │
│  근사 비율: ALG ≤ 1.5 · OPT (메트릭 TSP)               │
│  2020: Karlin-Klein-Gharan이 1.5 - ε으로 개선          │
└──────────────────────────────────────────────────────────┘
```

| 알고리즘 | 복잡도 | 근사 비율 | 조건 |
|:---|:---:|:---:|:---|
| Held-Karp DP | O(2ⁿ·n²) | 정확 | 소규모 (n≤25) |
| MST 2-근사 | O(E log V) | 2·OPT | 메트릭 TSP |
| Christofides | O(n³) | 1.5·OPT | 메트릭 TSP |
| LKH 휴리스틱 | 실용 | 보장 없음 | 대규모 현실 |
| 일반 TSP 근사 | — | 불가(P≠NP) | 일반 그래프 |

📢 **섹션 요약 비유**: Christofides 알고리즘은 "배달 경로를 최적보다 최대 50% 더 돌아가도록 보장하는 설계" — 50% 비효율이지만, 수백만 도시도 빠르게 처리할 수 있다.

## Ⅲ. 비교 및 연결

### 일반 TSP vs. 메트릭 TSP

| 구분 | 일반 TSP | 메트릭 TSP (삼각 부등식) |
|:---|:---:|:---:|
| 정확 알고리즘 | NP-Hard | NP-Hard |
| 상수 근사 | 불가능 (P≠NP) | 1.5-근사 (Christofides) |
| 조건 | 없음 | d(u,w) ≤ d(u,v) + d(v,w) |
| 현실 적용 | 비대칭 TSP(ATSP) | 유클리드 TSP, 도로망 |

### Held-Karp DP 구조

- 상태: dp[S][v] = S 집합 방문 후 v에 있을 때 최소 비용.
- 점화식: dp[S][v] = min_{u∈S, u≠v} (dp[S\{v}][u] + dist[u][v]).
- 시간: O(2ⁿ·n²), 공간: O(2ⁿ·n) → n=25 정도까지 현실적.

📢 **섹션 요약 비유**: Held-Karp는 "방문한 도시를 비트마스크로 기억하며 최적 경로를 채워가는 메모리 최적 내비게이션" — 정확하지만 도시가 늘면 메모리가 폭발한다.

## Ⅳ. 실무 적용 및 기술사 판단

| 분야 | 활용 |
|:---|:---|
| 물류·택배 | 배송 경로 최적화 (라스트마일) |
| PCB 제조 | 드릴 헤드 이동 경로 최소화 |
| DNA 시퀀싱 | 단편 조립 순서 최적화 |
| 반도체 | 웨이퍼 검사 경로 최적화 |

기술사 핵심 포인트:
1. 결정 버전 NP-Complete, 최적화 버전 NP-Hard 구분.
2. Christofides 1.5-근사는 메트릭 조건 필수 — 일반 TSP에는 적용 불가.
3. Held-Karp O(2ⁿ·n²)는 n≤25 정도의 정확 해법.
4. LKH(Lin-Kernighan-Helsgott) 휴리스틱이 실제 수십만 도시 경로에서 최강 성능.

📢 **섹션 요약 비유**: TSP 실무는 "완벽한 지도 대신 정밀도 높은 GPS" — 수학적 최적해가 아니어도, 충분히 좋은 해법(LKH, Christofides)이 실용을 지배한다.

---

## Ⅴ. 기대효과 및 결론

TSP를 이해하면:
- **NP-Hard 최적화의 표준 예시**: 이론·실무를 잇는 대표적 NP-Hard 최적화 문제.
- **근사 알고리즘 설계 철학**: 메트릭 조건 활용 → 이론 보장 근사 알고리즘 설계 원리.
- **Held-Karp DP 패턴**: 비트마스크 DP 패턴을 TSP에서 체득하면 다른 순열 최적화에 응용.

📢 **섹션 요약 비유**: TSP는 알고리즘 이론의 "올림픽 마라톤" — 수십 년 동안 최고의 알고리즘 학자들이 달려온 경쟁 무대이자, 실무 물류의 핵심 과제다.
