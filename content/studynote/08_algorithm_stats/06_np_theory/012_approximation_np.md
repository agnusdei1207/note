## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 근사 알고리즘(Approximation Algorithm)은 NP-Hard 최적화 문제에 대해 최적해(OPT) 대비 일정 비율(ρ) 이내의 해를 다항 시간에 보장하는 알고리즘이다.
> 2. **가치**: FPTAS(Fully Polynomial-Time Approximation Scheme)는 임의의 ε > 0에 대해 (1+ε)-근사를 n과 1/ε의 다항식 시간에 보장하는 가장 강력한 근사 보장이다.
> 3. **판단 포인트**: 모든 NP-Hard 문제가 좋은 근사 알고리즘을 허용하지는 않는다 — Max Clique처럼 근사 불가능(Inapproximable)한 문제와 Vertex Cover처럼 2-근사가 존재하는 문제를 구분해야 한다.

---

## Ⅰ. 개요 및 필요성

NP-Hard 최적화 문제는 정확 해법이 지수 시간을 요구하므로, 실용적 대안이 필요하다. 근사 알고리즘은 다음을 보장한다:

- **최소화 문제**: ALG(x) ≤ ρ · OPT(x), ρ ≥ 1.
- **최대화 문제**: ALG(x) ≥ (1/ρ) · OPT(x), ρ ≥ 1.
- ρ를 **근사 비율(Approximation Ratio)** 또는 **근사 인수(Approximation Factor)**라 한다.

이론적 보장이 없는 휴리스틱(Heuristic)과 달리, 근사 알고리즘은 **어떤 입력에서도** ρ 이내임이 수학적으로 증명된다.

📢 **섹션 요약 비유**: 근사 알고리즘은 "최고 셰프 대신 충분히 좋은 요리사를 고용하는 것" — 미슐랭 3스타(완벽)는 아니지만, 2스타(1.5배 내) 보장이 계약서에 적혀 있다.

## Ⅱ. 아키텍처 및 핵심 원리

### 근사 알고리즘 계층

```
┌──────────────────────────────────────────────────────────┐
│  NP-Hard 최적화 문제 해결 전략 계층                      │
│                                                          │
│  FPTAS ── PTAS ── 상수 근사 ── 대수 근사 ── 근사 불가   │
│  ────── ────── ─────────── ──────────── ──────────────  │
│  (1+ε)   (1+ε)  상수 ρ     O(log n)·OPT  어떤 것도 불가 │
│  poly in  poly                                           │
│  n,1/ε   in n                                           │
│                                                          │
│  ◀─────────── 더 강한 보장 ──────────────────────────▶  │
└──────────────────────────────────────────────────────────┘
```

### 주요 근사 알고리즘 예시

| 문제 | 알고리즘 | 근사 비율 | 방법 |
|:---|:---|:---:|:---|
| Vertex Cover | 최대 매칭 기반 | 2 | 에지 선택 후 양 끝점 추가 |
| TSP (메트릭) | Christofides | 1.5 | MST + 최소 매칭 |
| Set Cover | 그리디 | H(n) ≈ ln n | 가장 많이 커버하는 집합 반복 선택 |
| Knapsack | FPTAS | 1+ε | 가치 스케일링 + DP |
| Max-SAT | 랜덤 할당 | 7/8 | 각 변수 독립적 50% 확률 할당 |
| Max Clique | — | 불가능 | n^(1-ε) 이내 불가 (P≠NP) |

### 그리디 Set Cover 분석

```
┌──────────────────────────────────────────────────────────┐
│  Greedy Set Cover                                        │
│                                                          │
│  while 커버 안 된 원소 존재:                             │
│      현재 미커버 원소를 가장 많이 포함하는 집합 S 선택  │
│      S를 커버에 추가, S의 원소들을 커버됨으로 표시       │
│                                                          │
│  근사 비율: H(max set size) ≤ ln n + 1                  │
│  H(k) = 1 + 1/2 + 1/3 + ... + 1/k (조화급수)           │
└──────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 그리디 Set Cover는 "한 번에 가장 많은 구역을 덮는 담요부터 펼치는 방식" — 최적보다 ln n배 더 많이 쓸 수 있지만, 빠르고 이론 보장이 있다.

## Ⅲ. 비교 및 연결

### PTAS vs. FPTAS 차이

| 구분 | PTAS | FPTAS |
|:---|:---|:---|
| 정의 | 임의 ε>0에서 (1+ε)-근사를 n의 다항식에 | 임의 ε>0에서 n과 1/ε 모두의 다항식에 |
| 시간 예시 | O(n^(1/ε)) | O(n²/ε) |
| ε=0.01 이면? | O(n¹⁰⁰) — 매우 느림 | O(100n²) — 현실적 |
| 해당 문제 | Euclidean TSP, Bin Packing 등 | Knapsack, 일부 스케줄링 |

### 근사 불가능성 하한 (Inapproximability)

PCP(Probabilistically Checkable Proof) 정리에 기반해, P ≠ NP 가정 하에 특정 근사 비율보다 좋은 알고리즘이 존재하지 않음을 증명:

- **Max Clique**: n^(1-ε) 이내 근사 불가 (Håstad 1999).
- **Chromatic Number (색칠 문제)**: n^(1-ε) 이내 근사 불가.
- **Set Cover**: (1-ε) ln n 이하 근사 불가 (Dinur-Steurer 2014).

📢 **섹션 요약 비유**: 근사 불가능성은 "이 자물쇠는 완벽한 열쇠뿐 아니라 '반쯤 맞는 열쇠'도 없다"는 수학적 선언이다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 근사 알고리즘 선택 기준:

1. **FPTAS 가능 여부 확인**: Knapsack 등 → FPTAS 사용 (ε 조정으로 시간-정밀도 트레이드오프).
2. **상수 근사 적용**: Vertex Cover(2-근사), TSP(1.5-근사) → 이론 보장 있는 알고리즘 채택.
3. **근사 불가능 문제 인식**: Max Clique → 정확 해법(소규모) 또는 휴리스틱으로 방향 전환.
4. **랜덤화 근사**: Max-SAT의 7/8 랜덤 알고리즘처럼 확률적 보장 활용.

| 근사 보장 | 실용 판단 |
|:---|:---|
| FPTAS | 최강, ε 조정 가능 → 적극 채택 |
| 상수 근사 | 알고리즘 복잡도 낮고 보장 명확 → 표준 채택 |
| 대수 근사 | 대용량 데이터(Set Cover) → 허용 가능 |
| 근사 불가능 | 휴리스틱/FPT/정확 해법 병행 |

📢 **섹션 요약 비유**: 근사 알고리즘 선택은 "계약서에 오차 허용 범위를 명시한 외주 계약" — ρ값이 낮을수록 더 좋은 계약, FPTAS는 원하는 품질을 직접 지정하는 최고 조건이다.

---

## Ⅴ. 기대효과 및 결론

근사 알고리즘을 이해하면:
- **NP-Hard 실용 설계 능력**: 이론적 한계에서 현실적 솔루션으로의 전환 논리 완성.
- **근사 비율의 의미 해석**: ρ=2인 알고리즘이 허용 가능한지 비즈니스 요구사항과 연계.
- **FPTAS 활용**: 자원 배분, 스케줄링, 물류 최적화에서 ε 조정으로 성능-정밀도 균형.

📢 **섹션 요약 비유**: 근사 알고리즘은 "보험이 있는 빠른 결정" — 완벽한 답 대신 "최악이어도 이 정도"라는 보험증서를 들고 빠르게 실행하는 전략이다.

### 📈 관련 키워드 및 발전 흐름도

```text
[P vs NP 문제 (P vs NP Problem) — 계산 가능성의 경계]
    │
    ▼
[NP-완전 (NP-Complete) — 다항 환원으로 대표되는 어려운 결정 문제]
    │
    ▼
[NP-난해 (NP-Hard) — 최적화 포함 더 넓은 난이도]
    │
    ▼
[근사 알고리즘 (Approximation Algorithm) — 해 품질을 보장하는 대안]
    │
    ▼
[랜덤화 알고리즘 (Randomized Algorithm) — 확률을 이용한 근사]
```

이 흐름은 P vs NP의 경계에서 출발해 NP-완전과 NP-난해를 거쳐, 근사와 랜덤화를 통해 실용적 해법으로 내려오는 연구 흐름을 보여준다.
