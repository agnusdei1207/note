> **핵심 인사이트**
> 1. Z 알고리즘은 문자열 S의 각 위치 i에서 S 자체와의 최장 공통 접두사(Longest Common Prefix) 길이를 저장한 Z 배열을 O(n) 시간에 계산하는 알고리즘이다.
> 2. Z 배열을 이용하면 패턴 P를 텍스트 T에서 찾는 문제를 S = P + "$" + T로 구성해 Z 배열을 한 번만 계산함으로써 O(n+m) 패턴 매칭을 달성한다.
> 3. KMP의 실패 함수와 개념적으로 동등하지만, Z 배열은 직관적 구현과 접두사 관련 다양한 문자열 문제(팰린드롬, 반복 패턴 탐지)에 더 직접적으로 활용된다.

---

## I. Z 배열 정의

```
문자열: S = "aabxaa"
인덱스:      0123456 (0-based)

Z 배열 정의:
  Z[i] = S[i..] 와 S[0..] 의 최장 공통 접두사 길이

Z[0] = 전체 길이 (관례상 0 또는 n)
Z[1] = S[1..] = "abxaa" vs "aabxaa" -> 공통 접두사 "a" -> Z[1]=1
Z[2] = S[2..] = "bxaa"  vs "aabxaa" -> 없음 -> Z[2]=0
Z[3] = S[3..] = "xaa"   vs "aabxaa" -> 없음 -> Z[3]=0
Z[4] = S[4..] = "aa"    vs "aabxaa" -> "aa" -> Z[4]=2
Z[5] = S[5..] = "a"     vs "aabxaa" -> "a"  -> Z[5]=1

Z = [-, 1, 0, 0, 2, 1]
```

> 📢 **섹션 요약 비유**: 각 위치에서 문자열 전체와 비교했을 때 "얼마나 닮았는지" 점수표 — Z[i]가 클수록 i 위치가 처음과 많이 닮았다.

---

## II. Z 알고리즘의 핵심 최적화

```
순진한 계산: O(n^2) (각 위치에서 직접 비교)

Z 알고리즘 최적화: O(n)
[l, r] 윈도우: 현재까지 구한 가장 오른쪽 Z 박스

  for i = 1 to n-1:
    if i < r:  # 이전 Z 박스 내부
      Z[i] = min(r - i, Z[i - l])  # 재활용!
    while i + Z[i] < n and S[Z[i]] == S[i + Z[i]]:
      Z[i]++  # 확장
    if i + Z[i] > r:
      l = i; r = i + Z[i]  # 윈도우 갱신
```

| 경우       | 처리 방법                   | 비용       |
|-----------|---------------------------|-----------|
| i < r     | Z[i-l] 재활용 (이전 결과)  | O(1) 조회  |
| i >= r    | 직접 비교로 Z[i] 계산       | 확장 시만  |
| 총 비교   | 각 문자가 최대 2번 비교됨    | O(n) 전체 |

> 📢 **섹션 요약 비유**: 이미 계산한 구간은 "아는 것"으로 처리하고, 모르는 부분만 직접 확인 — 중복 작업을 없애는 DP 정신.

## III. Z 알고리즘으로 패턴 매칭

```
패턴 P = "aab", 텍스트 T = "aabxaab"

합성 문자열: S = P + "$" + T = "aab$aabxaab"
길이: n = 3 + 1 + 7 = 11

Z 배열 계산:
S = a a b $ a a b x a a b
Z = - 1 0 0 3 1 0 0 2 1 0

Z[i] == len(P) == 3 인 위치: i=4
-> T에서 P의 시작 위치 = i - len(P) - 1 = 4 - 3 - 1 = 0

Z[i] == 3 인 위치: i=4, i=8 (?)
실제로 Z[8] = 3 -> 위치 = 8-4 = 4 (T의 인덱스 4)
```

> 📢 **섹션 요약 비유**: "$" 구분자를 중간에 넣어 패턴과 텍스트를 섞은 뒤, Z 배열에서 "패턴 길이만큼 닮은" 위치를 찾는 것.

---

## IV. Python 구현

```python
def z_function(s):
    n = len(s)
    z = [0] * n
    z[0] = n
    l, r = 0, 0
    for i in range(1, n):
        if i < r:
            z[i] = min(r - i, z[i - l])
        while i + z[i] < n and s[z[i]] == s[i + z[i]]:
            z[i] += 1
        if i + z[i] > r:
            l, r = i, i + z[i]
    return z

def pattern_search(text, pattern):
    s = pattern + "$" + text
    z = z_function(s)
    m = len(pattern)
    return [i - m - 1 for i in range(m + 1, len(s)) if z[i] == m]

# 예시
print(pattern_search("aabxaab", "aab"))  # [0, 4]
```

> 📢 **섹션 요약 비유**: "$"로 패턴과 텍스트를 분리해 섞은 뒤 z_function을 한 번만 호출 — 단 한 번의 계산으로 모든 매칭 위치를 찾는다.

## V. Z 배열의 활용 사례

```
1. 패턴 매칭: P + "$" + T
2. 가장 긴 반복 접두사:
   max(Z[i] for i > 0) 값과 위치
3. 주기 찾기:
   S가 길이 p의 주기를 가지면 Z[p] == n-p
4. 문자열 압축:
   주기 p를 찾아 S = (S[:p]) * k
```

| 응용                | 방법                        | 복잡도 |
|--------------------|----------------------------|--------|
| 패턴 매칭            | P + "$" + T Z 배열         | O(n+m) |
| 접두사 포함 위치 탐색 | Z[i] > 0 인 위치            | O(n)   |
| 문자열 주기 탐지     | Z[p] + p == n 조건 확인     | O(n)   |

> 📢 **섹션 요약 비유**: Z 배열 하나로 패턴 매칭, 반복 구조 탐지, 주기 계산을 모두 해결 — 스위스 군용 칼 같은 도구.
