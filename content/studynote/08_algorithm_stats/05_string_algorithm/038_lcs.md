> **핵심 인사이트**
> 1. LCS(Longest Common Subsequence, 최장 공통 부분수열)는 두 수열에서 순서를 유지하면서 공통으로 나타나는 가장 긴 부분수열로, 연속성은 요구하지 않으며 동적 프로그래밍(DP)의 대표적 응용이다.
> 2. DP 점화식의 핵심: `dp[i][j] = dp[i-1][j-1] + 1 (같을 때)` vs `max(dp[i-1][j], dp[i][j-1]) (다를 때)` — 이 두 경우의 분기가 LCS 알고리즘 전체를 구성하며, O(mn) 시간/공간 복잡도를 가진다.
> 3. LCS는 단순한 알고리즘 문제를 넘어 diff 도구(파일 변경 추적), DNA 서열 유사성 비교, 버전 관리 시스템의 merge 기반 기술로 실제 소프트웨어 도처에 활용된다.

## I. LCS vs LCS (부분수열 vs 부분문자열)

```
LCS (Longest Common Subsequence):
  공통 부분수열 (연속 불필요)
  예: ABCBDAB vs BDCABA
  LCS: BCBA (길이 4)
  
  비교: A_CB_DAB
         _BDCABA_
           B  A = BCBA

LCS (Longest Common Substring):
  공통 부분문자열 (연속 필요)
  예: ABCBDAB vs BDCABA
  Longest Common Substring: AB (길이 2)
  
차이:
  Subsequence: 순서는 유지, 연속성 없어도 됨
  Substring: 연속된 문자열이어야 함

중요: 문제에서 "부분수열"인지 "부분문자열"인지 확인!
```

> 📢 **섹션 요약 비유**: 부분수열은 문장에서 단어 선택해 의미 있는 문장 만들기(중간 빼기 가능), 부분문자열은 문장의 연속된 구절만.

## II. DP 점화식

```
dp[i][j] = s1[0..i]와 s2[0..j]의 LCS 길이

점화식:
  if s1[i] == s2[j]:
    dp[i][j] = dp[i-1][j-1] + 1
  else:
    dp[i][j] = max(dp[i-1][j], dp[i][j-1])

예시: s1 = "ABCB", s2 = "BCB"

    ""  B  C  B
  ""  0  0  0  0
  A   0  0  0  0
  B   0  1  1  1
  C   0  1  2  2
  B   0  1  2  3

-> LCS 길이: 3 ("BCB")

역추적 (Backtrack):
  dp[i][j]에서 시작
  s1[i] == s2[j]: 해당 문자 선택, dp[i-1][j-1]로
  else: max(dp[i-1][j], dp[i][j-1]) 방향으로
```

> 📢 **섹션 요약 비유**: DP 테이블은 두 문자열 비교 지도 — 같은 문자 만나면 대각선으로 올라가며 공통 문자 발견.

## III. 시간/공간 복잡도

```
기본 구현:
  시간: O(m * n)
  공간: O(m * n) (DP 테이블)

공간 최적화:
  현재 행과 이전 행만 필요
  -> O(min(m, n)) 공간으로 축소
  (단, 경로 역추적 불가)

구현:
def lcs_length(s1, s2):
    m, n = len(s1), len(s2)
    # 2행만 사용
    prev = [0] * (n + 1)
    curr = [0] * (n + 1)
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                curr[j] = prev[j-1] + 1
            else:
                curr[j] = max(prev[j], curr[j-1])
        prev, curr = curr, [0] * (n + 1)
    return prev[n]

Hirschberg 알고리즘:
  O(m * n) 시간 + O(min(m,n)) 공간으로
  역추적까지 가능 (분할 정복 활용)
```

> 📢 **섹션 요약 비유**: 공간 최적화는 두 줄짜리 체크리스트로 100x100 표 계산하기 — 결과만 필요하면 이전 줄만 기억해도 됨.

## IV. diff와의 연관

```
diff 알고리즘 (Unix diff, Git diff):

핵심 아이디어:
  파일 A의 줄들 = 수열 s1
  파일 B의 줄들 = 수열 s2
  LCS = 변경되지 않은 줄들
  
  LCS 이외의 줄:
    s1에만 있음 -> 삭제 (-)
    s2에만 있음 -> 추가 (+)

Git diff 출력:
  --- a/main.py
  +++ b/main.py
  @@ -1,5 +1,6 @@
   def hello():
  -    print("Hello")
  +    print("Hello, World!")
  +    return "done"

Myers 알고리즘 (1986):
  O((n+m) * d) 시간 (d = 편집 거리)
  변경이 적을수록 빠름
  Git, GNU diff 등에서 사용
```

> 📢 **섹션 요약 비유**: Git diff는 두 파일의 LCS를 찾아 "그대로 있는 줄은 회색, 삭제는 빨강, 추가는 초록"으로 시각화한 것.

## V. 실무 시나리오 — DNA 서열 비교

```
DNA 서열 유사성:
  s1 = "AGGTAB"
  s2 = "GXTXAYB"
  
  LCS = "GTAB" (길이 4)
  유사도: 4/7 ≈ 57%
  
생물정보학 활용:
  두 DNA 서열의 LCS 길이 / 긴 서열 길이
  = 서열 유사도 (Sequence Similarity)
  
  BLAST (생물 서열 검색 도구):
  LCS 기반 알고리즘으로
  수십억 염기 서열 DB에서 유사 서열 탐색
  
실무 최적화:
  길이 1만+ 서열: O(mn) = 10^8 연산
  -> Sparse DP (공통 문자 위치만 처리)
  -> 4-러시아 방법 (블록 단위 처리)
  
코드 표절 탐지:
  두 코드 파일의 토큰 시퀀스 -> LCS
  LCS/전체 길이 > 80% -> 표절 의심
```

> 📢 **섹션 요약 비유**: DNA 서열 비교는 두 언어 문장에서 공통 단어 순서 찾기 — 길이가 길수록 진화적으로 관련성이 높음.

## 📌 관련 개념 맵

```
LCS (최장 공통 부분수열)
+-- DP 점화식
|   +-- 같을 때: dp[i-1][j-1] + 1
|   +-- 다를 때: max(dp[i-1][j], dp[i][j-1])
+-- 관련 알고리즘
|   +-- LCS Substring (연속)
|   +-- 편집 거리 (Edit Distance)
|   +-- Myers diff (파일 비교)
+-- 응용
    +-- Git diff, 파일 비교
    +-- DNA/단백질 서열 유사도
    +-- 코드 표절 탐지
```

## 👶 어린이를 위한 3줄 비유 설명

1. LCS는 두 영화 목록에서 같은 영화를 순서대로 고르되, 중간을 건너뛰어도 되는 가장 긴 공통 목록을 찾는 알고리즘이에요.
2. DP 테이블은 두 문자열 비교 지도처럼, 같은 문자를 만날 때마다 점수가 +1 올라가는 방식으로 가장 긴 공통 경로를 찾아요.
3. Git에서 파일 변경 내역을 빨강/초록으로 보여주는 diff 기능이 바로 LCS 알고리즘으로 "변하지 않은 줄"을 찾아내는 거예요!
