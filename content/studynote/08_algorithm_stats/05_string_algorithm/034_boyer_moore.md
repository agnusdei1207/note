> **핵심 인사이트 3줄**
> 1. 보이어-무어(Boyer-Moore) 알고리즘은 패턴을 오른쪽 끝에서부터 역방향으로 비교하고, 불일치 문자를 분석해 텍스트 포인터를 대폭 전진시키는 패턴 매칭 알고리즘으로, 평균 O(n/m) 시간에 동작한다.
> 2. 두 가지 휴리스틱 규칙(불량 문자 규칙·좋은 접미사 규칙)을 결합해 긴 패턴·자연어 텍스트에서 KMP보다 실용적으로 빠르며, grep·텍스트 편집기 검색 엔진의 기반 알고리즘이다.
> 3. 불량 문자 규칙(Bad Character Rule)만 적용해도 구현이 단순하고 실용적이며, 완전한 좋은 접미사 규칙(Good Suffix Rule)까지 적용하면 최악 O(n+m) 보장이 가능하다.

---

## Ⅰ. 보이어-무어의 핵심 아이디어

보이어-무어는 **패턴을 오른쪽 끝부터 역방향으로 비교**한다.

```
텍스트: ABCBABCABC
패턴:   ABC

KMP: 왼쪽부터 A→B→C 순으로 비교
보이어-무어: 오른쪽 끝 C부터 역방향 비교
  → 불일치 시 "최대한 멀리 점프"해서 빠르게 이동
```

### 두 가지 핵심 규칙

| 규칙                | 핵심 원리                            |
|------------------|-------------------------------------|
| 불량 문자 규칙 (Bad Character) | 불일치 문자를 패턴에서 오른쪽 정렬 |
| 좋은 접미사 규칙 (Good Suffix) | 일치한 접미사를 패턴에서 재활용   |

**실제 이동 거리**: max(불량 문자 이동, 좋은 접미사 이동)

📢 **섹션 요약 비유**: 보이어-무어는 사전 뒤에서 찾기다 — 알파벳 뒤에서부터(역방향) 찾고, 없는 글자가 나오면 그 글자가 있는 페이지까지 한 번에 점프한다.

## Ⅱ. 불량 문자 규칙 (Bad Character Rule)

불일치가 발생한 텍스트 문자를 패턴에서 찾아 정렬한다.

```
패턴: ABCABC
텍스트: ...XBCABC...

역방향 비교:
  패턴[5]=C, 텍스트[i+5]=C → 일치
  패턴[4]=B, 텍스트[i+4]=B → 일치
  패턴[0]=A, 텍스트[i]=X → 불일치!

  X는 패턴에 없음 → 패턴을 m=6칸 전진 (전체 건너뜀)
  X가 패턴에 있다면 → 그 위치에 맞춰 정렬
```

```
def bad_character_table(pattern):
    # 각 문자의 패턴 내 마지막 위치
    bad_char = {}
    for i, c in enumerate(pattern):
        bad_char[c] = i
    return bad_char
```

📢 **섹션 요약 비유**: 불량 문자 규칙은 틀린 단어 점프다 — 찾는 단어(패턴)에 없는 글자가 나오면, 그 글자를 완전히 지나쳐서 다음 시작점으로 이동한다.

## Ⅲ. 간소화 보이어-무어 구현

```
def boyer_moore_search(text, pattern):
    n, m = len(text), len(pattern)
    bad_char = {}
    for i, c in enumerate(pattern):
        bad_char[c] = i
    results = []
    s = 0
    while s <= n - m:
        j = m - 1
        while j >= 0 and pattern[j] == text[s + j]:
            j -= 1
        if j < 0:
            results.append(s)
            s += 1
        else:
            bc_shift = j - bad_char.get(text[s + j], -1)
            s += max(1, bc_shift)
    return results
```

**시간 복잡도**:
- 최선 (패턴 없음): O(n/m) → 텍스트 대부분 건너뜀
- 평균 (영어 텍스트): O(n/m)
- 최악 (반복 패턴): O(n·m) → 좋은 접미사 규칙 추가 시 O(n+m)

📢 **섹션 요약 비유**: 보이어-무어의 최선은 마술 점프다 — 패턴이 긴 영어 단어 검색에서는 텍스트 대부분을 건너뛰어, 실질적으로 O(n/m)의 초고속 탐색이 된다.

## Ⅳ. 좋은 접미사 규칙 (Good Suffix Rule)

불량 문자보다 더 멀리 이동할 수 있는 경우를 처리한다.

```
패턴: ABCABC
비교 중 패턴[3..5]="ABC"가 텍스트와 일치했다가 불일치:
  → "ABC" 접미사가 패턴에 다시 나타나는 위치로 정렬
  → 또는 "BC", "C" 등 부분 접미사 활용

실제 이동 = max(불량 문자 이동, 좋은 접미사 이동)
```

📢 **섹션 요약 비유**: 좋은 접미사 규칙은 이미 맞은 부분을 재활용하는 것이다 — 패턴 끝부분이 텍스트와 맞았다면, 그 패턴 안의 같은 부분에 다시 정렬해서 이동 거리를 더 키운다.

## Ⅴ. KMP vs 보이어-무어 실전 비교

| 비교 항목         | KMP              | 보이어-무어       |
|---------------|-----------------|-----------------|
| 비교 방향       | 왼쪽→오른쪽      | 오른쪽→왼쪽      |
| 최악 복잡도     | O(n+m)          | O(nm) 단순 구현 |
| 평균 복잡도     | O(n+m)          | O(n/m) 더 빠름  |
| 구현 복잡도     | 중간             | 높음             |
| 적합 텍스트     | 바이너리·반복 패턴 | 자연어·긴 패턴  |
| 실사용          | awk, 컴파일러    | grep, 텍스트 에디터|

📢 **섹션 요약 비유**: KMP vs 보이어-무어는 사전 찾기 방식 차이다 — KMP는 앞에서 한 글자씩(안정적), 보이어-무어는 뒤에서 점프하며(평균 빠름).

## 📌 관련 개념 맵

```
보이어-무어 알고리즘
├── 핵심 아이디어
│   ├── 역방향 비교 (오른쪽 끝부터)
│   └── 최대 점프 이동
├── 두 규칙
│   ├── 불량 문자 규칙 (Bad Character)
│   └── 좋은 접미사 규칙 (Good Suffix)
├── 복잡도
│   ├── 평균 O(n/m) 실용적 최적
│   └── 최악 O(nm) → 접미사 규칙 시 O(n+m)
└── 관련 알고리즘
    ├── KMP (왼쪽→오른쪽, O(n+m) 안정)
    ├── Rabin-Karp (해시 기반)
    └── Horspool (BM 단순화 버전)
```

## 👶 어린이를 위한 3줄 비유 설명

1. 보이어-무어는 뒤에서 찾는 책 읽기다 — 단어의 끝 글자부터 확인하고, 없는 글자가 나오면 그 앞부분 전체를 건너뛴다.
2. 불량 문자 규칙은 "없는 글자 만나면 점프"다 — 찾는 단어에 없는 글자가 텍스트에 나오면, 그 글자가 끝날 때까지 한 번에 이동한다.
3. 긴 단어를 찾을수록 더 빠르다 — 100글자 단어를 찾을 때, 보이어-무어는 오히려 더 멀리 점프해서 빨라진다.
