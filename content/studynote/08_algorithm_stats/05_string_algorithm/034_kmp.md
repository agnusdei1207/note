> **핵심 인사이트 3줄**
> 1. KMP(Knuth-Morris-Pratt) 알고리즘은 패턴 매칭 시 이미 비교한 정보를 재활용해 텍스트 포인터를 절대 되돌리지 않음으로써, 브루트 포스의 O(n·m)을 O(n+m)으로 줄인 효율적 문자열 탐색 알고리즘이다.
> 2. KMP의 핵심은 실패 함수(Failure Function, KMP Table)로, 패턴의 각 위치에서 최장 접두사-접미사(LPS: Longest Proper Prefix which is also Suffix)를 사전 계산해 불필요한 재비교를 제거한다.
> 3. 실패 함수 자체가 KMP를 자기 자신에게 적용해 O(m) 시간에 구성되며, 이 자기 참조적 구성이 KMP의 가장 우아한 설계 원리다.

---

## Ⅰ. 브루트 포스 vs KMP

```
텍스트: "AABABCABABABCABAB"
패턴:   "ABABCABAB"

브루트 포스 O(n·m):
  텍스트[0]: A-A 일치, A-B 불일치 → 텍스트[1]로 이동
  텍스트[1]: A-A, B-B, A-A, B-B 일치, C-A 불일치 → 텍스트[2]로 이동
  ... (이미 일치했던 "ABAB"를 다시 비교)

KMP O(n+m):
  실패 함수로 이미 일치한 부분 재활용
  → 텍스트 포인터는 절대 뒤로 이동하지 않음
```

📢 **섹션 요약 비유**: KMP는 책 읽기다 — 브루트 포스는 못 찾으면 첫 페이지부터 다시 읽고, KMP는 읽던 페이지를 기억해서 거기서부터 이어 읽는다.

---

## Ⅱ. 실패 함수 (Failure Function / KMP Table) 구성

LPS(Longest Proper Prefix which is also Suffix) 배열을 계산한다.

```python
def build_kmp_table(pattern):
    m = len(pattern)
    lps = [0] * m
    length = 0  # 현재까지의 LPS 길이
    i = 1

    while i < m:
        if pattern[i] == pattern[length]:
            length += 1
            lps[i] = length
            i += 1
        else:
            if length != 0:
                # 현재 LPS 길이의 이전 LPS로 되돌아감
                length = lps[length - 1]
            else:
                lps[i] = 0
                i += 1
    return lps

# 예시
pattern = "ABABCABAB"
# lps:  [0,0,1,2,0,1,2,3,4]
# 의미: 각 위치까지의 최장 접두사=접미사 길이
```

**시각화**:
```
패턴: A B A B C A B A B
LPS:  0 0 1 2 0 1 2 3 4

LPS[3]=2: "ABAB"의 최장 접두사=접미사 = "AB" (길이 2)
LPS[8]=4: "ABABCABAB"의 최장 접두사=접미사 = "ABAB" (길이 4)
```

📢 **섹션 요약 비유**: LPS 배열은 패턴의 자기 유사성 지도다 — "이 위치에서 불일치가 나면, 얼마만큼 앞으로 돌아가야 하는지"를 미리 계산해 놓은 치트키.

## Ⅲ. KMP 탐색 알고리즘

```python
def kmp_search(text, pattern):
    n, m = len(text), len(pattern)
    lps = build_kmp_table(pattern)
    results = []

    i = 0  # 텍스트 포인터
    j = 0  # 패턴 포인터

    while i < n:
        if text[i] == pattern[j]:
            i += 1
            j += 1
            if j == m:  # 패턴 완전 일치
                results.append(i - j)  # 시작 인덱스
                j = lps[j - 1]  # 다음 탐색 위치
        else:
            if j != 0:
                j = lps[j - 1]  # 패턴 포인터만 되돌림
            else:
                i += 1  # 텍스트 포인터만 전진

    return results
```

**핵심**: 불일치 시 텍스트 포인터(i)는 절대 뒤로 안 감 → O(n) 보장

📢 **섹션 요약 비유**: KMP 탐색은 재사용하는 읽기다 — 불일치가 나면 텍스트 위치를 유지하고, 패턴 위치만 LPS 테이블이 알려주는 곳으로 점프한다.

## Ⅳ. 시간/공간 복잡도

| 단계         | 시간 복잡도 | 공간 복잡도 |
|-----------|-----------|-----------|
| LPS 배열 구성 | O(m)     | O(m)      |
| KMP 탐색    | O(n)      | O(1) (추가)|
| **전체**    | **O(n+m)**| **O(m)**  |

**브루트 포스 비교**:
- 브루트 포스: O(n × m) → 텍스트 1MB, 패턴 1KB면 최악 10⁹ 연산
- KMP: O(n + m) → 동일 입력에서 최악 2×10³ 연산

📢 **섹션 요약 비유**: O(n+m) vs O(n×m)의 차이는 국어사전 찾기다 — 브루트 포스는 모든 페이지를 순서대로(n×m), KMP는 자음·모음 순서(n+m)로 훨씬 빠르게 찾는다.

## Ⅴ. 응용 및 유사 알고리즘

| 알고리즘         | 특성                            | 적합 상황           |
|--------------|--------------------------------|-------------------|
| KMP          | O(n+m), 단순 패턴               | 단일 패턴, 긴 텍스트  |
| Boyer-Moore  | O(n/m) 평균, 역방향 비교        | 영어·긴 패턴         |
| Rabin-Karp   | O(n+m) 평균, 해시 기반          | 다중 패턴 동시 검색  |
| Aho-Corasick | O(n+Σm+k), KMP의 트라이 확장    | 여러 패턴 동시 검색  |
| Z 알고리즘    | O(n+m), Z 배열 기반             | 동일 기능, 단순 구현 |

📢 **섹션 요약 비유**: KMP vs Boyer-Moore는 책 찾기 방법 차이다 — KMP는 앞에서부터 한 글자씩, Boyer-Moore는 끝에서부터 점프하며 찾아 평균적으로 더 빠르다.

## 👶 어린이를 위한 3줄 비유 설명

1. KMP는 이미 읽은 내용을 기억하는 책 읽기다 — 못 찾으면 처음부터 다시 읽지 않고, 기억해둔 곳에서 이어 읽는다.
2. LPS 배열은 패턴의 치트키다 — "이 위치에서 틀리면 몇 칸 앞으로 가야 해"를 미리 계산해 놓아서, 탐색 중에 빠르게 판단할 수 있다.
3. 텍스트 포인터가 절대 안 돌아가는 것이 핵심이다 — 책 페이지를 절대 뒤로 넘기지 않고 앞으로만 가서 O(n)이 보장된다.
