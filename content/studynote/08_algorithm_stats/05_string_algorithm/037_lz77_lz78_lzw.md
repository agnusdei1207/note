> **핵심 인사이트**
> 1. LZ77·LZ78·LZW는 모두 이미 처리한 데이터를 "사전(Dictionary)"으로 삼아 반복 패턴을 참조 포인터로 치환하는 사전 기반 무손실 압축 알고리즘으로, Abraham Lempel과 Jacob Ziv가 1977·1978년에 발표했다.
> 2. LZ77은 슬라이딩 윈도우 방식으로 직전 데이터를 사전으로 사용하고, LZ78/LZW는 명시적 사전을 동적으로 구축한다 — 이 차이가 GIF(LZW), PNG/ZIP(DEFLATE=LZ77+Huffman) 선택의 근거다.
> 3. 현대의 거의 모든 범용 압축 도구(zlib, gzip, ZIP, PNG)는 DEFLATE(LZ77 변형 + Huffman Coding)를 기반으로 하며, zstd·Brotli는 같은 원리에 현대적 최적화를 추가한 후속 알고리즘이다.

---

## I. 사전 기반 압축의 핵심 아이디어

```
관찰: 자연어 텍스트/코드에는 반복 패턴이 많다

예시 문자열:
  "abracadabra"
  
순진한 저장: 8비트 × 11 = 88비트

LZ77 방식:
  "abracad" 이후 "abra"는 7자 전에 길이4로 나옴
  -> "abracad" + (offset=7, length=4)
  
  훨씬 짧게 표현 가능!
```

> 📢 **섹션 요약 비유**: 계약서에서 같은 회사명이 반복될 때 "이하 '갑'"으로 줄여서 참조하는 것 — LZ는 텍스트에서 이 패턴을 자동으로 찾는다.

---

## II. LZ77 — 슬라이딩 윈도우

```
LZ77 구조:
  검색 버퍼 (Search Buffer): 이미 처리한 데이터 (32KB 등)
  전방 버퍼 (Look-ahead Buffer): 앞으로 처리할 데이터

  출력 토큰: (offset, length, next_char)
    offset: 검색 버퍼에서 일치 시작 위치
    length: 일치 길이
    next_char: 다음 불일치 문자

예시:
  입력: "abcabcabc"
  
  처음 "abc": 검색 버퍼 없음 -> (0, 0, 'a'), (0,0,'b'), (0,0,'c')
  다음 "abc": 3자 전에 "abc" 발견 -> (3, 3, 'a')
  -> 원본 9자 -> 압축 후 훨씬 작음

DEFLATE (LZ77 변형):
  슬라이딩 윈도우 32KB
  허프만 코딩으로 토큰 추가 압축
  ZIP, gzip, PNG, zlib 모두 사용
```

| 도구    | 알고리즘           | 특징              |
|--------|------------------|-----------------|
| gzip   | DEFLATE (LZ77+Huffman) | 단일 파일 압축  |
| ZIP    | DEFLATE          | 여러 파일 묶음    |
| PNG    | DEFLATE          | 이미지 무손실     |
| zlib   | DEFLATE 라이브러리| HTTP 압축(gzip) |

> 📢 **섹션 요약 비유**: LZ77의 슬라이딩 윈도우는 30cm 자 — 현재 위치 기준 왼쪽 30cm 이내에서만 참조 가능.

## III. LZ78과 LZW — 명시적 사전

```
LZ78:
  빈 사전으로 시작
  새로운 패턴이 나오면 사전에 추가
  출력: (사전 인덱스, 새 문자)

LZW (Lempel-Ziv-Welch, 1984):
  LZ78의 개선판 (Terry Welch)
  사전에 모든 단일 문자로 초기화 (0-255)
  새 문자열이 사전에 없으면:
    -> 이전까지 매칭된 인덱스 출력
    -> 새 패턴을 사전에 추가
  
  장점: 출력에 새 문자 없음 (사전 인덱스만)
  적용: GIF (특허 문제로 PNG로 대체됨)
        TIFF, PDF
```

```
LZW 예시:
  입력: "ABABABAB"
  사전 초기화: {A:0, B:1}
  
  A -> 사전에 있음
  AB -> 사전에 없음! -> A(0) 출력, AB:2 추가
  B -> 사전에 있음
  BA -> 사전에 없음! -> B(1) 출력, BA:3 추가
  AB -> 사전에 있음 (2)
  ABA -> 사전에 없음! -> AB(2) 출력, ABA:4 추가
  ...
```

> 📢 **섹션 요약 비유**: LZW는 대화하면서 자동으로 공통 약어를 만드는 것 — "ABABABAB"를 반복하다 보면 "AB=2"라는 약어가 생겨서 이후 더 짧게 표현.

## IV. LZ77 vs LZ78/LZW 비교

```
                LZ77              LZ78/LZW
사전 방식       슬라이딩 윈도우    명시적 사전
사전 크기 제한  윈도우 크기(32KB)  이론적 무제한 (메모리 제한)
복잡도          O(n) 탐색          O(n) 탐색
메모리          적음               사전 증가에 따라 증가
압축률          안정적             패턴 반복 많을수록 높음
특허 문제       없음               LZW: 특허 (만료됨, GIF 분쟁)
현재 사용       ZIP, PNG, gzip     GIF, TIFF (일부)
```

> 📢 **섹션 요약 비유**: LZ77은 뒤돌아보는 백미러, LZW는 배운 내용을 사전에 기록하는 단어장 — 접근법은 다르지만 목표는 같다.

## V. 현대 후속 알고리즘

```
zstd (Zstandard, Facebook, 2016):
  LZ77 + ANS (Asymmetric Numeral Systems)
  DEFLATE 대비 3~4배 빠른 압축/해제
  Facebook, Linux 커널 기본 압축
  
Brotli (Google, 2015):
  LZ77 + Huffman + 2차 맥락 모델링
  HTTP/2 웹 압축 표준 (Content-Encoding: br)
  텍스트/HTML에서 gzip보다 20-26% 더 압축
  
LZ4:
  속도 최우선 (압축률 < 속도)
  실시간 데이터 스트림, 게임 에셋
  
비교:
  압축률: zstd > Brotli > zlib > LZ4
  속도:   LZ4 > zstd > zlib > Brotli
```

> 📢 **섹션 요약 비유**: LZ4는 빠른 배달, zstd는 효율적 배달, Brotli는 웹용 최적 포장 — 목적에 따라 다른 현대 알고리즘.

## 📌 관련 개념 맵

```
LZ77 / LZ78 / LZW
+-- LZ77 (1977)
|   +-- 슬라이딩 윈도우 검색
|   +-- (offset, length, char) 토큰
|   +-- DEFLATE = LZ77 + Huffman
+-- LZ78 (1978) / LZW (1984)
|   +-- 명시적 사전 구축
|   +-- GIF (LZW, 특허 만료)
+-- 현대 후속
|   +-- zstd (LZ77+ANS)
|   +-- Brotli (LZ77+Huffman+모델링)
|   +-- LZ4 (속도 최우선)
+-- 응용
    +-- ZIP, gzip, PNG (DEFLATE)
    +-- HTTP 압축 (gzip, Brotli)
```

## 👶 어린이를 위한 3줄 비유 설명

1. LZ 압축은 반복되는 패턴이 나오면 "3칸 전에 4글자랑 같아요"처럼 위치만 저장해서 파일을 작게 만들어요.
2. ZIP 파일, PNG 이미지가 원본보다 작은 이유가 바로 이 LZ77 알고리즘 덕분이에요.
3. 요즘은 더 빠른 zstd, 웹 최적화된 Brotli 같은 현대 버전이 나와서 더 빠르고 작게 압축할 수 있어요!
