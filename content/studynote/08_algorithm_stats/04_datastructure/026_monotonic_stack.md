## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 단조 스택 (Monotonic Stack)은 스택 내 원소가 항상 단조 증가 또는 단조 감소 순서를 유지하도록 삽입 시 조건을 위반하는 원소를 팝(Pop)하는 특수 스택이다.
> 2. **가치**: "다음 더 큰 원소 (Next Greater Element, NGE)", 히스토그램 최대 직사각형, 빗물 고이는 양(Trapping Rain Water) 등 O(n²) 브루트포스 문제를 O(n) 분할상환으로 해결한다.
> 3. **판단 포인트**: 배열에서 각 원소의 "다음/이전 더 크거나 작은 원소"를 구하는 패턴이 보이면 단조 스택, 슬라이딩 윈도우 최댓값이면 단조 덱(Monotonic Deque)을 적용한다.

---

## Ⅰ. 개요 및 필요성

배열 `[2, 1, 5, 6, 2, 3]`에서 각 원소의 다음 더 큰 원소를 O(n²)으로 구하면 느리다. 단조 스택은 스택이 항상 단조 증가(또는 감소) 상태를 유지하도록 관리하여 각 원소가 정확히 한 번씩 push/pop되므로 전체 시간 복잡도가 O(n)이다.

### 단조 스택의 분류

| 종류 | 스택 상태 | 팝 조건 | 사용 시나리오 |
|:---|:---|:---|:---|
| 단조 증가 스택 | 바닥→상단 증가 | 새 원소 ≤ top | NGE, 주식 스팬 |
| 단조 감소 스택 | 바닥→상단 감소 | 새 원소 ≥ top | 다음 작은 원소 |

📢 **섹션 요약 비유**: 단조 스택은 줄 서기의 "키 규칙"—새 사람이 줄 맨 뒤에 서면서 자신보다 키 작은 사람을 줄에서 내보내는 과정이다.

## Ⅱ. 아키텍처 및 핵심 원리

### NGE (Next Greater Element) 동작 예시

```
입력: [2, 1, 5, 6, 2, 3]
출력: [5, 5, 6, -1, 3, -1]  (각 원소의 다음 더 큰 원소)

단조 증가 스택 처리 과정 (인덱스 기준):

처리  스택(인덱스)  팝(pop)된 원소  NGE 결정
────  ──────────  ─────────────  ──────────
 2    [0]          -              -
 1    [0,1]        -              -          1은 push (1<2이므로 증가 유지)
 5    [2]          0→arr[0]=2     NGE[0]=5   
                   1→arr[1]=1     NGE[1]=5   스택에 2보다 작은 것 모두 pop
 6    [3]          2→arr[2]=5     NGE[2]=6
 2    [3,4]        -              -
 3    [3,5]        4→arr[4]=2     NGE[4]=3
끝    []            3→arr[3]=6    NGE[3]=-1
                   5→arr[5]=3    NGE[5]=-1

결과: [5, 5, 6, -1, 3, -1]
```

### 히스토그램 최대 직사각형

```
높이 배열: [2, 1, 5, 6, 2, 3]

   6
   ─
5  █  6
─  █  ─
█  █  █
█  █  █  3
█  █  █  ─
█  █  █  █
2  1  5  6  2  3  ← 막대 높이

단조 증가 스택으로 각 막대의 "왼쪽 경계"와 "오른쪽 경계" 계산
→ 최대 직사각형 넓이 = 10 (5×2, 인덱스 2~3)
시간 O(n)
```

### 슬라이딩 윈도우 최댓값 (단조 덱)

```
입력: [1, 3, -1, -3, 5, 3, 6, 7], 윈도우 k=3

단조 감소 덱(Deque) 사용 (앞이 최댓값):

윈도우  덱(인덱스)  최댓값
───────  ─────────  ─────
[1,3,-1]  [1,2]      3
[3,-1,-3] [1,2,3]    3
[-1,-3,5] [4]        5   (인덱스 4의 5가 이전 모두 제거)
[-3,5,3]  [4,5]      5
[5,3,6]   [6]        6
[3,6,7]   [7]        7

결과: [3, 3, 5, 5, 6, 7]  O(n)
```

📢 **섹션 요약 비유**: 슬라이딩 윈도우 최댓값의 단조 덱은 VIP 대기줄—더 높은 VIP가 오면 앞의 낮은 VIP들이 줄에서 사라지는 방식이다.

## Ⅲ. 비교 및 연결

### 단조 스택 패턴 문제 분류

| 문제 | 스택 유형 | 시간 |
|:---|:---:|:---:|
| Next Greater Element | 단조 증가 | O(n) |
| Next Smaller Element | 단조 감소 | O(n) |
| 주식 스팬 (Stock Span) | 단조 감소 | O(n) |
| 히스토그램 최대 직사각형 | 단조 증가 | O(n) |
| 빗물 고이는 양 (Trapping Rain Water) | 단조 감소 | O(n) |
| 슬라이딩 윈도우 최댓값 | 단조 덱 | O(n) |
| 일일 온도 (Daily Temperatures) | 단조 증가 | O(n) |

### 브루트포스 vs 단조 스택

```
N = 10,000
브루트포스 O(n²) = 100,000,000 연산
단조 스택  O(n)  = 10,000 연산    ← 10,000배 빠름
```

📢 **섹션 요약 비유**: 단조 스택과 브루트포스는 도서관에서 자기 키보다 큰 사람을 찾는 방법—모든 쌍 비교(O(n²)) vs 앞에서부터 키 기록을 유지하며 스택에 쌓는 것(O(n)).

## Ⅳ. 실무 적용 및 기술사 판단

### 주요 활용 사례

- **주식 거래 시스템**: 가격 스팬 계산(Stock Span), 지지선/저항선 분석
- **UI 레이아웃 엔진**: 최대 너비 직사각형 배치(히스토그램 문제)
- **기상 분석**: 일일 기온 변화에서 "며칠 후 더 따뜻한 날" 계산
- **컴파일러 최적화**: 반복 루프 내 피크 탐지
- **게임 AI**: 시야 범위 계산 (Visibility 문제)

### 기술사 판단 기준

```
각 원소의 다음 더 큰/작은 원소     →  단조 스택 O(n)
고정 윈도우 범위 최댓값/최솟값     →  단조 덱 O(n)
히스토그램 최대 직사각형           →  단조 증가 스택
빗물 고이는 양                     →  스택 또는 투포인터
"첫 번째로 ~보다 큰" 패턴 반복    →  단조 스택 강력히 의심
```

📢 **섹션 요약 비유**: 단조 스택은 팀장을 뽑는 면접 줄—새 지원자가 줄에 서면 자신보다 능력치 낮은 앞 사람들이 바로 탈락 처리(pop)되는 시스템이다.

---

## Ⅴ. 기대효과 및 결론

단조 스택은 스택의 불변식 하나(단조성 유지)를 추가함으로써 O(n²)→O(n) 최적화를 달성한다. 각 원소는 최대 1회 push, 1회 pop이므로 분할상환 O(1) 연산이 보장된다. 슬라이딩 윈도우 최댓값 문제에서 단조 덱으로 확장하면 deque의 양쪽 연산을 활용한 O(n) 솔루션이 가능하다.

**결론**: "다음/이전 더 크거나 작은 원소"라는 패턴을 발견하면 단조 스택이 O(n) 최적해이며, 이 패턴의 변형(히스토그램, 빗물, 슬라이딩 윈도우)도 같은 원리로 해결된다.

### 👶 어린이를 위한 3줄 비유 설명

1. 단조 스택은 키 순서 줄 서기야—새 친구가 오면 자신보다 키 작은 친구들이 뒤로 빠지게 돼.
2. 이 규칙 덕분에 "내 앞에 나보다 큰 사람"을 따로 찾지 않아도 스택 맨 위 원소가 항상 그 답이야.
3. 슬라이딩 윈도우 최댓값도 덱으로 같은 방식—범위가 이동할 때마다 낡은 원소와 작은 원소를 자동으로 정리해줘.
