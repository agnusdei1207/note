## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 스파스 테이블 (Sparse Table)은 멱집합(2의 거듭제곱) 단위 구간을 미리 전처리하여, 겹침이 허용되는 연산(min·max·GCD)에 대해 RMQ (Range Minimum Query)를 O(1)에 응답하는 정적 자료구조다.
> 2. **가치**: 전처리 O(n log n), 쿼리 O(1)이라는 조합은 업데이트 없이 쿼리가 극도로 많은 시나리오(수백만 구간 쿼리)에서 세그먼트 트리(O(log n) 쿼리)보다 빠르다.
> 3. **판단 포인트**: 배열이 정적(갱신 없음)이고 구간 min/max/GCD 쿼리가 대량이면 스파스 테이블 O(1), 갱신이 필요하면 세그먼트 트리 O(log n)을 선택한다.

---

## Ⅰ. 개요 및 필요성

구간 최솟값 문제 "배열 arr에서 arr[l..r]의 최솟값 구하기"를 매번 O(n)으로 처리하면 Q개 쿼리에 O(nQ) 시간이 든다. 스파스 테이블은 모든 2의 거듭제곱 길이 구간을 미리 계산해두어, 임의 구간을 두 개의 겹치는 구간으로 분해하여 O(1)에 응답한다. 겹침이 허용되는 이유는 min·max·GCD 같은 **멱등(Idempotent) 연산**의 특성 때문이다.

### 스파스 테이블 vs 다른 구간 쿼리 자료구조

| 자료구조 | 전처리 | 쿼리 | 갱신 | 멱등 연산 한정 |
|:---|:---:|:---:|:---:|:---:|
| 브루트포스 | O(1) | O(n) | O(1) | - |
| 스파스 테이블 | O(n log n) | O(1) | ❌ | ✅ |
| 세그먼트 트리 | O(n) | O(log n) | O(log n) | - |
| 펜윅 트리 (BIT) | O(n) | O(log n) | O(log n) | 부분합 한정 |

📢 **섹션 요약 비유**: 스파스 테이블은 시험 전 모든 연도 범위별 평균을 미리 계산해 표로 만들어두는 것—시험장에서 "2020~2023 평균"을 즉시 읽어낼 수 있다.

## Ⅱ. 아키텍처 및 핵심 원리

### 스파스 테이블 구조

```
배열: arr = [2, 4, 3, 1, 6, 7, 8, 9]
         인덱스:  0  1  2  3  4  5  6  7

dp[j][i] = arr[i..i+2^j-1] 구간의 최솟값

j=0 (길이 1):
  dp[0] = [2, 4, 3, 1, 6, 7, 8, 9]

j=1 (길이 2):
  dp[1][0] = min(arr[0..1]) = min(2,4) = 2
  dp[1][1] = min(arr[1..2]) = min(4,3) = 3
  dp[1][2] = min(arr[2..3]) = min(3,1) = 1
  dp[1][3] = min(arr[3..4]) = min(1,6) = 1
  dp[1] = [2, 3, 1, 1, 6, 7, 8, -]

j=2 (길이 4):
  dp[2][0] = min(dp[1][0], dp[1][2]) = min(2,1) = 1
  dp[2][1] = min(dp[1][1], dp[1][3]) = min(3,1) = 1
  dp[2] = [1, 1, 1, 1, -, -, -, -]

j=3 (길이 8):
  dp[3][0] = min(dp[2][0], dp[2][4]) = 1
```

### 점화식

```
dp[j][i] = min(dp[j-1][i], dp[j-1][i + 2^(j-1)])

시간: O(n log n) 전처리
공간: O(n log n)
```

### O(1) 쿼리 메커니즘

```
query(l, r):
  k = floor(log2(r - l + 1))          ← 구간 길이에 맞는 최대 2^k
  return min(dp[k][l], dp[k][r-2^k+1]) ← 두 겹치는 구간의 min

예시: query(1, 6) → len=6, k=2 (2^2=4)
  min(dp[2][1], dp[2][3]) = min(1, 1) = 1

겹침: [1..4]와 [3..6] → 중간 [3..4]가 겹쳐도 min이므로 결과 정확
```

📢 **섹션 요약 비유**: RMQ의 겹침 허용은 "두 자로 같은 방을 재도 되는" 측정 방식—min(최솟값)은 같은 원소를 두 번 봐도 결과가 변하지 않으므로 겹쳐서 빠르게 계산한다.

## Ⅲ. 비교 및 연결

### 멱등 연산 vs 비멱등 연산

| 연산 | 스파스 테이블 적용 | 이유 |
|:---|:---:|:---|
| min / max | ✅ | min(a,a) = a (겹쳐도 결과 동일) |
| GCD | ✅ | gcd(a,a) = a |
| 구간 합 (Sum) | ❌ | 겹치면 중복 합산 |
| 구간 XOR | ❌ | 겹치면 취소됨 |
| 구간 곱 (Product) | ❌ | 겹치면 중복 곱셈 |

구간 합은 **펜윅 트리 (Fenwick Tree, BIT)** 또는 **세그먼트 트리**를 사용해야 한다.

### LCA (Lowest Common Ancestor)와 스파스 테이블

```
오일러 투어 (Euler Tour) + 스파스 테이블:
  트리의 LCA 문제를 RMQ로 환원
  LCA(u, v) = euler_tour[l..r]의 최소 depth 인덱스
  → 스파스 테이블로 O(1) LCA 쿼리
```

📢 **섹션 요약 비유**: 스파스 테이블로 LCA를 O(1)에 구하는 것은 가계도를 미리 정리해두면 두 사람의 공통 조상을 즉시 찾는 것과 같다.

## Ⅳ. 실무 적용 및 기술사 판단

### 주요 활용 사례

- **구간 최솟값/최댓값 쿼리**: 정적 배열에서 대량 구간 질의
- **LCA (Lowest Common Ancestor)**: 트리에서 두 노드의 최소 공통 조상 O(1)
- **서픽스 배열의 LCP 구간 최솟값**: 패턴 매칭 최적화
- **기상 데이터 분석**: 고정 기간 최저/최고 기온 쿼리
- **경쟁 프로그래밍**: 구간 min/max 쿼리가 10⁶개 이상일 때 표준 선택

### 기술사 판단 기준

```
정적 배열 + 대량 min/max 쿼리         →  스파스 테이블 O(n log n) + O(1)
동적 배열 + 갱신 포함 쿼리            →  세그먼트 트리 O(n) + O(log n)
구간 합/부분합 쿼리                   →  펜윅 트리 O(n) + O(log n)
트리 LCA O(1) 필요                   →  오일러 투어 + 스파스 테이블
겹침 허용 여부 불확실                →  세그먼트 트리 (안전한 범용 선택)
```

📢 **섹션 요약 비유**: 스파스 테이블과 세그먼트 트리의 선택은 책 목차의 두 가지 전략—목차를 한 번 정리하면 다시 고칠 일 없는 교과서(스파스 테이블)냐, 언제든 수정할 수 있는 위키(세그먼트 트리)냐의 차이다.

---

## Ⅴ. 기대효과 및 결론

스파스 테이블은 O(n log n) 전처리를 투자하여 멱등 구간 쿼리를 O(1)로 처리한다. 배열이 정적이고 쿼리 수가 매우 많은 경우(RMQ, LCA)에서 세그먼트 트리를 앞서는 유일한 자료구조다. 갱신이 발생하는 동적 시나리오에서는 세그먼트 트리가 적합하다.

**결론**: 대량의 정적 min/max 쿼리에서 스파스 테이블은 O(1) 응답이라는 독보적 장점을 가지며, 멱등 연산 여부가 적용 가능성의 핵심 판단 기준이다.

### 👶 어린이를 위한 3줄 비유 설명

1. 스파스 테이블은 시험 공부를 미리 다 해두는 것—모든 범위의 답을 표로 만들어두면 시험장에서 즉시 읽어낼 수 있어.
2. "1번~6번 중 최솟값"을 구할 때, 미리 만들어둔 "1~4번 최솟값"과 "3~6번 최솟값" 중 작은 것을 고르면 돼.
3. 덧셈은 겹치면 두 번 세지만, 최솟값은 겹쳐도 괜찮으니까—그게 O(1) 쿼리의 비밀이야!
