# 기술 통계 (Descriptive Statistics) - 데이터의 첫 번째 언어

> ⚠️ 이 문서는 빅데이터 분석의 가장 기본이 되는 기술 통계(Descriptive Statistics)의 핵심 개념인 중심 경향치(평균, 중앙값, 최빈값), 산포도(분산, 표준편차, 범위), 분포(왜도, 첨도)를 탐색적 데이터 분석(EDA) 관점에서 해석하고, 실무 빅데이터 환경에서 수십억 건의 로그와 센서 데이터를 요약하는 데 필수적인 분산 계산 알고리즘(MapReduce 기반 합계, 카운팅)과 한계를 기술사 수준에서 심층 분석합니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 기술 통계는 수백만 건의 원시 데이터를 숫자 몇 개(평균, 분산, 분위수)로 압축하여 "데이터가 전체적으로 어떤 모습인가?"를 한눈에 파악하게 하는 데이터 과학의 가장 기본적인 요약(Summarization) 도구이다.
> 2. **가치**: 기술 통계를 먼저 구하지 않고 고급 분석(회귀, ML)을 돌리면 Garbage In Garbage Out(GIGO)으로 인한 시간 낭비와 잘못된 의사결정이라는 재앙이 발생한다. 데이터의 "체온, 혈압, 맥박"을 측정하지 않고 환자를 진료하는 것과 동일하다.
> 3. **확장**: 전통적인 합계 기반 기술 통계(단순 합산)는 수평 확장(샤딩) 환경에서 정확한 중앙값 산출이 불가능한 데 비해, 수백만 건의 데이터를 분할 집계(Pre-Aggregation)한 뒤 합치는 2단계 방식(Two-Pass Algorithm)이 빅데이터 분산 환경에서의 정확한 기술 통계 산출의 핵심 원리이다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 1. 기술 통계의 정의: "데이터의 체온을 재는 일차원적 스캐너"
기술 통계(Descriptive Statistics)란 주어진 데이터셋의 특징을 수치적 지표(숫자)若干개로 요약하는 통계학의 가장 기본적인 분야입니다. 마치 의사가 환자를 진료할 때 "체온 36.5도, 혈압 120/80, 맥박 72회/분"이라는 숫자若干개만으로 건강 상태를 일차원적으로 파악하는 것처럼, 데이터 과학자는 평균(Average)이나 분산(Variance) 같은 기술 통계量若干개를 계산하는 것으로 수천만 레코드의 대략적인 "체온"을 측정합니다.

### 2. 빅데이터 시대에 기술 통계를 다시 배워야 하는 이유
과거 통계학 교과서는 데이터가 100건, 1,000건인 상황을 전제로 했습니다. 그러나 현대의 기술 통계는 수십억 건의 로그 데이터, 수백만 명의 사용자 행동 데이터를 다루며, "단순 합산으로 평균을 구하면 분산 환경에서 오차가 발생"하거나 "중앙값은 데이터 전체를 정렬해야 하므로 수평 확장 환경에서 계산 비용이 천문학적"이라는 새로운 과제를 안고 있습니다.

- **상황 변화**: 10만 명의 사용자가 있는 플랫폼에서 "평균 구매 금액"을 구한다고 합시다. 단순히 전체 금액을 전체人数으로 나누면 되는데, 문제는 일부 극단적 이상치(구매 금액 10억 원짜리 프리미엄 구독자)가 평균을 크게 왜곡시킬 수 있다는 점입니다. 이 경우 중앙값(Median)이 평균보다 더 정확한 구매 패턴의 대표값이 됩니다.

- **필요성**: 기술 통계를 이해하지 못하면, "고객 평균 체류 시간 5분"이라는 수치만 가지고 "아침 10분에 접속하는 고객에게 Push 알림을 보내자"는 엉터리 전략을 세울 수 있습니다. 실제 데이터의 분포(히스토그램)를 보면 80%의 고객이 30초 안에 앱을 나가는데, 나머지 20%의 전력 사용자(Heavy User)가 평균을 끌어올린 것일 수 있습니다.

- **📢 섹션 요약 비유**: 기술 통계는 "한국 경제의 건강 상태를 국내총생산(GDP) 1개 숫자로 요약하는 것"과 같습니다. GDP 5조 달러라는 숫자 하나가 전체 경제를 대표하지만, 이 숫자背后에는中小企业倒闭(분산),失业率(표준편차),および地域間格差(왜도)라는 무수한 현실이 숨어 있습니다. 경제학자(데이터 과학자)가 GDP라는 기술 통계量를 해석할 때, "이 숫자가 전체를 얼마나 정확하게 대표하는가?"를 항상 의심하고批判적으로 접근해야 합니다.

## Ⅱ. 핵심 아키텍처 및 원리 (Architecture & Mechanism)

```text
┌─────────────────────────────────────────────────────────────────┐
│          [ 기술 통계 (Descriptive Statistics) 핵심 지표 계층 ]         │
│                                                                 │
│  [1. 중심 경향치 (Central Tendency) - "데이터의 중심在哪里?"]          │
│   ├─ 평균 (Mean): 합계 / 개수 → 산술 평균, 이상치에 민감            │
│   ├─ 중앙값 (Median): 데이터를 정렬했을 때 가운데 값 → 이상치 강건     │
│   └─ 최빈값 (Mode): 가장 자주 등장하는 값 → 범주형 데이터에 적합       │
│                                                                 │
│  [2. 산포도 (Dispersion) - "데이터가 중심에서 얼마나 흩어져 있는가?"]  │
│   ├─ 범위 (Range): 최대값 - 최소값 → 극단값에 민감                   │
│   ├─ 분산 (Variance): 각 값과 평균의 편차 제곱 평균                  │
│   ├─ 표준편차 (Standard Deviation): √분산 → 실제偏离幅을 나타냄      │
│   └─ 변동계수 (CV): 표준편차 / 평균 → 단위가 다른 데이터 비교에 유용   │
│                                                                 │
│  [3. 분포 형태 (Distribution Shape) - "데이터의 모양은 어떠한가?"]      │
│   ├─ 왜도 (Skewness): 분포의 좌우 비대칭도                          │
│   │    ▶ 왜도 > 0: 오른쪽 꼬리(양의 왜도) - 극단 큰 값들이 평균 ↑    │
│   │    ▶ 왜도 < 0: 왼쪽 꼬리(음의 왜도) - 극단 작은 값들이 평균 ↓    │
│   └─ 첨도 (Kurtosis): 분포의 뾰족한 정도                           │
│        ▶ 첨도 > 3: 뾰족 (Leptokurtic) - 극단값이 정규분포보다 많음   │
│        ▶ 첨도 < 3: 납작 (Platykurtic) - 극단값이 정규분포보다 적음   │
│                                                                 │
│  [4. 빅데이터 분산 환경에서의 기술 통계 계산]                          │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  [Map Phase]                                            │    │
│  │  - 각 노드가 자신의 파티션 데이터에 대해 기술 통계 계산         │    │
│  │  - Map(k1, v1) → (평균합계, 카운트), (분산합계, 카운트)       │    │
│  │  [Reduce Phase]                                          │    │
│  │  - 각 노드의 부분 합계를 결합하여 전역 통계량 산출             │    │
│  │  - 합산 카운트 = N_total, 평균 = Σ(평균합계) / N_total       │    │
│  └─────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────┘
```

### 1. 평균(Average)의 수학적 정의와 분산 환경 한계
산술 평균(Arithmetic Mean)은 가장 직관적이면서도 가장 많이 사용되는 중심 경향치입니다. 수식으로 표현하면 $\bar{x} = \frac{1}{n}\sum_{i=1}^{n} x_i$ 입니다. 그러나 수평 확장 환경에서 단순 합산 방식에는 큰 문제가 있습니다. 바로 **정밀도 손실(Precision Loss)**과 **이상치 민감성**입니다.

### 2. 분산과 표준편차: 데이터의 "흩어짐"을 숫자로
분산(Variance)은 각 데이터 포인트가 평균에서 얼마나 떨어져 있는지를 제곱한 뒤 평균을 낸 값입니다. $\sigma^2 = \frac{1}{n}\sum_{i=1}^{n}(x_i - \bar{x})^2$ 분산이 크다는 것은 데이터가 평균 주위에 넓게 분포해 있다는 뜻입니다. 표준편차(Standard Deviation)는 이 분산에 제곱근을 취한 것으로, $\sigma$ 단위로 실제 데이터가 평균에서 평균적으로 얼마나 떨어져 있는지를 바로 알 수 있습니다.

### 3. 분위수(Quantile)와 박스플롯: 데이터의 5수 요약
데이터의 전체 분포를 파악하는 데 가장 유용한 도구 중 하나가 **5수 요약(Five-Number Summary)**입니다. 최소값(Min), 제1사분위수(Q1), 중앙값(Median), 제3사분위수(Q3), 최대값(Max)으로 구성됩니다. 이 다섯 숫자로 구성되는 **박스플롯(Box Plot)**을 그리면, 데이터의 중심, 산포, 왜도, 이상치를 한눈에 파악할 수 있습니다.

## Ⅲ. 비교 및 기술적 트레이드오프 (Comparison & Trade-offs)

### 기술 통계 지표별 특성 비교

| 지표 유형 | 계산 복잡도 | 이상치 민감도 | 분산 환경 적합성 | 해석 용이성 |
|:---|:---|:---|:---|:---|
| **산술 평균 (Mean)** | O(n) - 단순 합산 | **매우 높음** | 높음 (맵리듀스 적합) | 높음 (直感的) |
| **중앙값 (Median)** | O(n log n) - 정렬 필요 | 낮음 (강건) | **매우 낮음** (전체 정렬 필요) | 중간 (순서才有意義) |
| **최빈값 (Mode)** | O(n) - 해시 맵 카운팅 | 낮음 | 높음 (맵리듀스 적합) | 낮음 (범주형 한정) |
| **분산 (Variance)** | O(n) - 평균 필요 | 높음 | 높음 (Two-Pass 가능) | 중간 |
| **표준편차 (SD)** | O(n) | 높음 | 높음 | **매우 높음** (단위 복원) |
| **�동계수 (CV)** | O(n) | 높음 | 높음 | **매우 높음** (상대적 비교) |

### 기술 통계의 치명적 함정: 평균의 역설 (Simpson's Paradox)
**가장 위험한 트레이드오프**: 평균을 맹신하다 보면 **심슨의 역설(Simpson's Paradox)**에 빠집니다. 예를 들어, 전체 데이터에서 평균 응답 시간이 120ms로 "양호"해 보이지만, 그룹별로 나누어 분석하면 그룹 A는 100ms, 그룹 B는 200ms로 큰 격차가 있을 수 있습니다. 전체 평균은 그룹 간 구성비의 차이로 인해 잘못된 판단을 유도할 수 있습니다.

### 빅데이터 환경에서의 정확도 문제
**오버플로우(Overflow)와 정밀도 손실**: 수십억 레코드의 합계를 double 타입(64비트 부동소수점)으로 표현할 때, 극단적으로 큰 값과 극단적으로 작은 값을 합산하면 정밀도 손실이 발생할 수 있습니다. 실무에서는 **Kahan 합산 알고리즘**이나 **정밀도가 높은Decimal 타입** 사용을 고려해야 합니다.

- **📢 섹션 요약 비유**: 기술 통계 지표의 선택은 "사람의 체형을 평가하는 방법"과 같습니다. 평균(Mean)은 "한국 남성 평균 키 175cm"라는 숫자로 전체를 대표시키지만, 이 숫자에는 150cm的人和 200cm的人都 포함되어 있어 실제 키 분포를 알 수 없습니다. 반면 중앙값(Median)은 "한국 남성 키 순서를 정렬했을 때 173cm"로, 절반은 이보다 크고 절반은 이보다 작다는 정보를 주어 키 분포를 더 정확히 파악할 수 있습니다. 최빈값(Mode)은 "가장 많이 있는 체형分类"이고, 표준편차(SD)는 "사람들 키가 平均적으로 ±10cm 범위에 분포"라는 정보를 제공합니다. 어떤 지표를 선택하느냐에 따라 같은 데이터에서도 전혀 다른 이야기가 emerges!

## Ⅳ. 실무 판단 기준 (Decision Making)

| 고려 사항 | 세부 내용 | 주요 아키텍처 의사결정 |
|:---|:---|:---|
| **도입 환경** | 정형 데이터(RDBMS) vs 비정형 로그 데이터 | 정형: SQL AVG/Stdev, 비정형: MapReduce 분산 집계 |
| **이상치 존재 여부** | 극단적 이상치가 평균을 왜곡하는 경우 | 중앙값 + IQR(사분위 범위) 조합 사용 |
| **정확도 요구 수준** | 관리 Reports vs 실시간 모니터링 | 관리:全量 정밀 계산, 모니터링: 샘플링 근사치 |
| **분산 환경 적합성** | 수평 확장 환경에서 정확한 중앙값 산출可行性 |tdigest, GK tree 등 근사 분위수 알고리즘 도입 |

*(추가 실무 적용 가이드 - 이상치 탐지에서의 기술 통계 활용)*
- **Z-Score 기반 이상치 탐지**: $Z = (x - \mu) / \sigma$ 계산 시, Z-Score가 3 이상이면 해당 데이터 포인트를 이상치로 판정합니다. 이는 평균에서 표준편차의 3배 이상 벗어난 값은 전체 분포의 0.27%에만 해당하여 통계적으로 매우 드물다는 원리에 기반합니다.
- **IQR(사분위 범위) 방법**: Q1 - 1.5 × IQR 이하 또는 Q3 + 1.5 × IQR 이상을 이상치로 판정합니다. 이 방법은 정규분포를 가정하지 않아 왜도가 있는 분포에도 강건합니다.
- **실무 의사결정**: 서버 응답 시간(_latency_) 모니터링에서 평균이 100ms로 양호해 보여도, P99 지연이 5초인 경우가 있습니다. 따라서 기술 통계 보고서에는 반드시 **평균 + 표준편차 + P50/P95/P99 분위수**를 함께 포함하여 "보통은 빠르지만 가끔 극단적으로 느리다"는 현실을 놓치지 않아야 합니다.

- **📢 섹션 요약 비유**: 기술 통계의 실무 적용은 "의사의 환자 진단 보고서"와 같습니다. 의사가 체온만 측정하고 혈압, 맥박, 혈당 수치 등을 무시하면Patients의 전체 건강 상태를 알 수 없는 것처럼, 데이터 분석가도 평균만 믿고 분산, 왜도, 분위수를 무시하면データの реаль 모습を見誤ります. 훌륭한 의사는 "평균적으로 건강한 것은 맞지만, 이 환자だけは特定指標が平均より著しく偏高しており additional 정밀 검사가 필요하다"고 판단합니다. 이것이 기술 통계를 "단순 계산"이 아닌 "데이터의voice를 듣는 여섯 번째 감각"으로昇華시키는 핵심 관점입니다.

## Ⅴ. 미래 전망 및 발전 방향 (Future Trend)

1. **근사 기술 통계 (Approximate Descriptive Statistics)의 일상화**
   수십억 레코드에서 정확한 중앙값을 구하려면 전체 데이터를 정렬해야 하므로 수평 확장 환경에서도 엄청난 연산 비용이 듭니다. **tdigest 알고리즘(Druid, Elasticsearch 등 실시간 OLAP에서 사용)**나 **GK tree(Quantile digest)** 같은 근사 분위수 알고리즘은 소량의 메모리로 P50~P99 분위수를 99% 이상의 정확도로 근사 산출할 수 있어, 실시간 대시보드의 표준 기술 통계 계산 방식으로 자리잡고 있습니다.

2. **시계열 기술 통계와 동적 임계값报警의 결합**
   전통적인 고정 임계값(평균 + 2σ)을 넘어, 시계열 데이터의 최근 N개 윈도우에서 기술 통계를 동적으로 계산하고, 그 동적 표준편차를 기반으로 실시간으로 이상치를 탐지하는 **적응형 임계값(Adaptive Thresholding)** 시스템이 금융 사기 탐지, 네트워크入侵検知, 공정品質 관리 분야에서 빠르게 확산되고 있습니다.

3. **기술 통계의 자동화 EDA 도구 확산**
   **Pandas Profiling, SweetViz, D-Tale** 같은 자동화 EDA 도구가 기술 통계를 자동으로 계산하고, 왜도와 첨도를 시각화하며, 이상치를 자동으로 표시해 줌으로써 "데이터를 첫눈에 파악"하는 과정이 몇 초 안에 완성되는 시대가 열렸습니다. 그러나 이러한 자동화 도구에도 한계가 있으며, 특히 수십 GB 이상의 큰 데이터에는 **Koalas, PySpark** 등의 분산 환경 EDA 도구 활용이 필수적입니다.

- **📢 섹션 요약 비유**: 기술 통계의 미래는 "단순 체온계로 환자 상태를 파악하던 시절"에서 "실시간 연속 Vital Signs 모니터링 시스템"으로의 진화와 similar 합니다. 과거에는 의사가 수동으로 혈압, 맥박, 체온을 일일이 재였지만(정확하지만低速), 현대 병원에서는 Central Monitoring System이Patients의 수백 개 Vital Signs을 실시간으로 연속 수집·기술 통계 분석하여, patient's 상태가「通常範囲」から 벗어난 찰나에自動报警하는 시스템이 가동됩니다. 데이터 과학 분야에서도 동일한 진화가 진행 중이며, Batch 기반 기술 통계에서 실시간 Streaming 기술 통계(근사 algorithm 기반)로의 전환이 가속화되고 있습니다.

## 🧠 지식 맵 (Knowledge Graph)

*   **기술 통계 핵심 지표 트리 (Descriptive Statistics Taxonomy)**
    *   **중심 경향치 (Central Tendency)**: Mean(산술평균), Median(중앙값), Mode(최빈값)
    *   **산포도 (Dispersion)**: Range(범위), Variance(분산), Standard Deviation(표준편차), IQR(사분위범위), CV(변동계수)
    *   **분포 형태 (Shape)**: Skewness(왜도), Kurtosis(첨도)
    *   **위치 통계 (Quantiles)**: Q1, Q2(Median), Q3, Percentiles(P10, P90, P95, P99)
*   **빅데이터 분산 기술 통계 알고리즘**
    *   **MapReduce Two-Pass**: 1パス目で各パーティションの sum/count を計算, 2パス目で全局平均/分散をaggregate
    *   **근사 알고리즘**: t-digest ( Elasticsearch/Druid ), GK tree ( Pinot ), Quantile Summary
*   **EDA 자동화 도구**
    *   **Pandas Profiling**: Python, 靜的 HTML レポート生成
    *   **SweetViz**: Python, 비교 분석 EDA
    *   **Spark/PySpark**: 大規模 データ向け 기술 통계
