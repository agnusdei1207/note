# 회귀 분석 (Regression Analysis) - 데이터 속 숨은 수학적 관계를 찾는 기술

> ⚠️ 이 문서는 두 변수 이상의 데이터 간 관계를 수학적 함수(모델)로 표현하여, 한 변수의 값으로부터 다른 변수의 값을 예측(Prediction)하거나, 변수 간 인과관계 방향을 추정(Causal Inference)하는 회귀 분석의 핵심 개념인 단순 회귀, 다중 회귀, 정규화(릿지, 라쏘, 엘라스틱넷), 그리고 로지스틱 회귀까지 포괄하는 회귀 분석의 unified된框架을 기술사 수준에서 심층 분석합니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 회귀 분석은 "입력 변수 X가 1만큼 증가할 때 출력 변수 Y는 얼마만큼 증가/감소하는가?"라는 질문에 수학적 방정식(Y = β₀ + β₁X + ε)으로 답하는 것으로, 데이터의 산점도(scatter plot)에 가장 잘 맞는"라인 하나(회귀선)"를 찾는 과정이다.
> 2. **가치**: 이简单的な 회귀 방정식 하나가 있으면, X의 미래 값을 입력하여 Y의 값을 예측할 수 있다. "광고비를 1,000만 원 늘리면 매출이 얼마나 증가하는가?"라는 광고 마케팅 의사결정의 근거가 된다.
> 3. **확장**: 단순 선형 회귀에서 다중 회귀, 비선형 회귀, 정규화(Overfitting 방지), 로지스틱 회귀(0/1 분류)로 확장되면, 현실 세계의 모든 예측 문제에 회귀 분석적 사고를 적용할 수 있는 범용적 도구로 활용된다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 1. 회귀 분석의 기원: Galton의 우연한 발견
회귀(Regression)라는 용어는 영국의Francis Galton(골턴)이 1885년 부모의 키와 자녀의 키 간 관계를 연구하면서 처음 사용했습니다. 그는"부모가平均보다 크면子女도平均보다 크지만,的程度는 조금 더 작다(평균으로의 회귀)"는 것을 발견했고, 이를 **"평균으로의 회귀(Regression to the Mean)"**라고 명명했습니다. 이 우연한 발견이 현대 회귀 분석의 시초가 되었습니다.

### 2. 회귀 분석이 필요한 이유
비즈니스 세계에서"어떤 변수가 어떤 변수에 어떤 영향을 미치는가?"라는 질문은끝없이 반복됩니다:
- "온도가 1도 오르면 아이스크림 매출이 얼마나 오르는가?" (營業 管理)
- "웹페이지 로드 시간이 1초 늘면 사용자 이탈율이 얼마나 증가하는가?" (UX 엔지니어링)
- "운동 시간이 1시간 증가하면 체중이 얼마나 감소하는가?" (헬스케어)

이러한 질문에 답하려면, 데이터 속의 숨겨진 수학적 관계를 찾아내는 회귀 분석이 필수적입니다.

- **📢 섹션 요약 비유**: 회귀 분석은 "투자 은어인" 적정 주식 가격 찾기"와 같습니다. 과거 주가 데이터(입력 변수: 매출, 이익, 금리 등)를 바탕으로"이 기업의 적정 주가는 5만 원이다"라고 数式的으로算出하는 것입니다. 과거 데이터를 통해"수익이 1조 원 증가할 때마다 주가가平均적으로 3,000 원 오른다(β₁ = 0.0003)"라는 일차함수(Y = β₀ + β₁X)를 찾아내는 것이 회귀 분석이며, 이를 통해 미래 수익이 1.5조 원이 되면"적정 주가는 약 4만5천 원"이라는 예측을 산출합니다.

## Ⅱ. 핵심 아키텍처 및 원리 (Architecture & Mechanism)

```text
┌─────────────────────────────────────────────────────────────────┐
│              [ 회귀 분석 (Regression Analysis) 아키텍처 계층 ]           │
│                                                                 │
│  [1. 단순 선형 회귀 (Simple Linear Regression)]                      │
│   수식: Y = β₀ + β₁X + ε                                        │
│   ├─ Y: 종속변수 (예: 매출액) - 우리가 예측したいもの               │
│   ├─ X: 독립변수 (예: 광고비) - 예측에 사용するもの                 │
│   ├─ β₀: 절편 (X=0일 때 Y 값)                                    │
│   ├─ β₁: 회귀계수 (X가 1 증가할 때 Y 변화량) ★ 핵심 ★             │
│   └─ ε: 오차항 (모델이 설명하지 못하는随机적 변동)                  │
│                                                                 │
│  [2. 다중 선형 회귀 (Multiple Linear Regression)]                   │
│   수식: Y = β₀ + β₁X₁ + β₂X₂ + β₃X₃ + ... + ε                  │
│   ├─ X₁, X₂, X₃: 여러 개의 독립변수 (예: 광고비, 날씨, 할인율)     │
│   └─ β₁, β₂, β₃: 각 변수의 순수 기여도 (다른 변수 통제 후)         │
│                                                                 │
│  [3. 정규화 회귀 (Regularized Regression)]                          │
│   ├─ 릿지 (Ridge, L2 정규화): β 계수의 크기를惩戒하여 Overfitting 방지 │
│   │    ▶ 손실함수 + λΣβⱼ²                                          │
│   ├─ 라쏘 (Lasso, L1 정규화): β 계수를 0으로 축소하여 변수 선택     │
│   │    ▶ 손실함수 + λΣ|βⱼ|                                         │
│   └─ 엘라스틱넷 (Elastic Net): L1 + L2 결합                       │
│        ▶ 손실함수 + λ₁Σ|βⱼ| + λ₂Σβⱼ²                            │
│                                                                 │
│  [4. 회귀係数 추정: 최소제곱법 (Ordinary Least Squares, OLS)]        │
│   목적: 잔차²의 합(SSE = Σ(Yᵢ - Ŷᵢ)²)을 최소화하는 β̂를 찾기       │
│   잔차(Residual) = 실제값(Yᵢ) - 예측값(Ŷᵢ)                        │
│   → 미분을 통해 ∂SSE/∂β = 0이 되는 β̂를 해석적으로 구함             │
└─────────────────────────────────────────────────────────────────┘
```

### 1. 최소제곱법(OLS): 회귀선을 찾는 수학적 원리
**Ordinary Least Squares(OLS, 보통최소제곱법)**는 잔차(실제값 - 예측값)의 제곱합을 최소화하는 회귀 계수(β̂)를 찾는 방법입니다. 잔차의 제곱을 사용하는 이유는 (1) 양수/음수 잔차가 서로相殺하는 것을 방지하고, (2) 큰 오차에 더 큰 페널티를 부여하여 outlier에 어느 정도 민감하게 반응하게 만들기 때문입니다.

### 2. 회귀 모델의 성능 지표
| 지표 | 공식 | 해석 |
|:---|:---|:---|
| **R² (결정계수)** | 1 - SSE/SST | 모델이 설명하는 분산 비율 (0~1, 높을수록 좋음) |
| **수정 R² (Adjusted R²)** | 1 - (1-R²)(n-1)/(n-p-1) | 변수 수를 반영한 R² (다중 회귀에서 중요) |
| **RMSE** | √(SSE/n) | 예측 오차의 표준편차 (원래 단위) |
| **MAE** | Σ|Yᵢ - Ŷᵢ|/n | 평균 절대 오차 (이상치에 덜 민감) |
| **AIC/BIC** | -2 log L + p×k | 모델 복잡도까지 고려한 적합도 (낮을수록 좋음) |

### 3. 로지스틱 회귀: 회귀의 분류 문제 확장
**로지스틱 회귀(Logistic Regression)**는 Y가 0 또는 1(예: 구매/비구매, 이탈/유지)인 이항 분류 문제에 회귀 분석을 적용한 것입니다. 직접 Y값을 예측하는 대신, Y=1일 확률 $P(Y=1) = \frac{1}{1 + e^{-(\beta₀ + β₁X)}}$를 예측하며, 이 함수가 S자 곡선(Sigmoid 함수)이어서 확률값을 0~1 사이로 안정적으로 산출합니다.

## Ⅲ. 비교 및 기술적 트레이드오프 (Comparison & Trade-offs)

### 회귀 분석의 주요 변형 비교

| 회귀 유형 | 수식 형태 | 적합한 상황 | 주요 장점 | 주요 단점 |
|:---|:---|:---|:---|:---|
| **단순 선형** | Y = β₀ + β₁X | 변수 1개, 선형 관계 | 해석 용이, 시각화 간편 | 현실 복잡한 관계 포착 어려움 |
| **다중 선형** | Y = β₀ + β₁X₁ + ... | 변수 여러 개, 선형 관계 | 더 많은 설명력 | 다중 공선성 문제 |
| **다항 회귀** | Y = β₀ + β₁X + β₂X² | 비선형 관계 (U자, 포물선) | 곡선 관계 포착 | Overfitting 위험↑ |
| **릿지 (L2)** | 손실 + λΣβⱼ² | 변수 많고 다중 공선성 | 수치적 안정성↑ | 계수를 0으로 만들지 못함 |
| **라쏘 (L1)** | 손실 + λΣ|βⱼ| | 불필요 변수 자동 제거 | 상관 변수 중 하나만 선택 |
| **로지스틱** | Sigmoid(β₀+βX) | 0/1 분류 문제 | 확률값 산출, 해석 용이 | 결정 경계가 선형 |

### 회귀 분석의 치명적 함정
1. **다중 공선성(Multicollinearity)**: 독립 변수 간 높은 상관관계가 있으면 회귀 계수의 분산이 커져 추정이 불안정해집니다. **VIF(분산팽창계수)**로 검출하며, VIF > 10이면 심각한 다중 공선성으로 판단합니다.
2. **이분산성(Heteroscedasticity)**: 잔차의 분산이 일정하지 않고 회귀fitted값에 따라 변하면, OLS 추정이非효율적이 됩니다. White 검정으로 확인하며, 가중最小二剩법(Weighted Least Squares)으로 대응합니다.
3. **자기상관(Autocorrelation)**: 시계열 데이터에서 잔차가 서로 상관관계가 있으면(예: 오늘 양의 잔차 → 내일도 양의 잔차), 표준오차가 过少評価되어 유의성 판단을 그르칩니다.

- **📢 섹션 요약 비유**: 회귀 분석의 various 변형들은 "촬영 렌즈의 종류"에 비유할 수 있습니다. 단순 선형 회귀는 **단일 초점 렌즈**(하나의 각도만 촬영)로 기본적인 풍경 촬영에는 충분하지만, 복잡한 구조물(다중 변수 관계)을 촬영하기엔 부족합니다. 다중 회귀는 **줌 렌즈**(여러 초점 거리를 하나의 렌즈로 통합)로 더 많은Variable을 다루되 렌즈 무게가 무겁고(다중 공선성) 손떨림(불안정 추정)이 심합니다. 릿지는 ** voyager 렌즈 안정기**(손떨림 방지裝置)로 무게는 줄이되 초점이 흐릿해지고, 라쏘는 **hdr 렌즈**(불필요한 영역을 자동으로 잘라냄)로 더 깔끔한构图를 만들지만 때로 필요部分까지 잘라버릴 수 있습니다.

## Ⅳ. 실무 판단 기준 (Decision Making)

| 고려 사항 | 세부 내용 | 주요 아키텍처 의사결정 |
|:---|:---|:---|
| **예측 vs 설명** | 계수 해석이 중요한가? → OLS, 예측 정확도가 중요한가? → 트리/부스팅 | 목적에 따라 모델 선택 |
| **변수 선택** | 변수 많음 → 라쏘(L1) 또는 단계적 선택법 | 불필요 변수는 오히려 Overfitting 유발 |
| **다중 공선성** | VIF > 10 → 릿지 또는 변수 제거 | 상관 변수 제거 또는 PCA |
| **비선형 관계** | 산점도로 비선형 패턴 확인 → 다항 회귀 or GAM | 선형 가정은 심각한 모델 오류 유발 |

*(추가 실무 적용 가이드 - 다중 회귀实战)*
- **변수 선택 전략**:Backward Elimination(전체 변수부터 시작하여 유의하지 않은 변수를 하나씩 제거), Forward Selection(유의한 변수부터 하나씩 추가), Stepwise(양방향) 방법이 있습니다.
- **교차 검증 (Cross-Validation)**: 데이터를 K개 폴드로 나누어 각 폴드에서 회귀 모델을 훈련/검증하여 Out-of-Sample 성능을評価합니다. Overfitting 여부를 확인할 수 있습니다.
- **실무 의사결정**: "매출 = -1000 + 2.5×광고비 + 0.8×할인율 + ε"라는 다중 회귀 결과에서,β₁=2.5는"광고비를 1원 늘릴 때, 할인율을 고정하면 매출이平均적으로 2.5원 증가한다"는 뜻입니다. 이는 인과관계가 아니라 상관관계일 수 있으므로 해석에 주의해야 합니다.

- **📢 섹션 요약 비유**: 회귀 분석의 실무 적용은 "집 값 예측 모델 만들기"와 같습니다. 집 값(종속 변수)에 영향을 미치는因子(독립 변수)로 면적, 위치, 건축연도, 방 수, 학군 등을 넣고"면적이 1평 증가할 때마다 집 값이 얼마씩 오르는지(회귀 계수)"를 분석합니다. 그런데 중요한 것은"인과관계가 아닌 상관관계일 수 있다"는 점입니다. 예컨대"학군 변수의係数가 크다"는 것이"학군이 좋아서 집 값이 오른다"는 의미일 수도 있지만,"좋은 학군 근처의 집들은 면적은 작고 위치는 좋아서" 학군 자체가 원인이지 않을 수도 있습니다. 회귀 계수의 정확한 해석은 비즈니스 의사결정의品质을 좌우합니다.

## Ⅴ. 미래 전망 및 발전 방향 (Future Trend)

1. **AutoML과 회귀 분석의 자동화**
   **AutoML(Auto-sklearn, H2O AutoML, Google Vertex AI)**의 등장으로, 회귀 분석에서 모델 선택, 하이퍼파라미터 튜닝, 변수 선택까지 자동으로 수행하는 시대가 열렸습니다. 그러나 자동화된 모델이 비즈니스 맥락을 이해하지는 못하므로, 분석가는AutoML 결과의Interpretation 능력이 더욱 중요해지고 있습니다.

2. **인과추론으로의 확장: 회귀에서 인과로**
   회귀 분석의 가장 큰 한계는"상관관계는 인과관계를증명하지 못한다"는 것입니다. 더 깊은 질문("광고비를 늘리면 실제로 매출이 증가하는가?")에 답하려면 **랜덤화 실험(RCT)** 또는 **인과추론(Causal Inference)** 방법론( Granger因果性, IV, DID, RDD)이 필요하며, 이는 회귀 분석의 확장으로 위치지고 있습니다.

3. ** Interpretable ML과 회귀 분석의复兴**
   딥러닝 등 블랙박스 모델의Interpreteribility 요구 증가에 따라, **선형 회귀나 로지스틱 회귀와 같은white-box 모델**이 다시 주목받고 있습니다. 회귀 계수의Interpretation 가능성이 규제 금융, 의료 분야에서 매우 중요하게 다뤄지고 있으며, **SHAP(Shapley Additive exPlanations)**와의 결합으로 회귀 분석의Interpretation이 한층 강화되고 있습니다.

- **📢 섹션 요약 비유**: 회귀 분석의 미래는 "단순 계산기에서 슈퍼 컴퓨터로"의 진화라기보다, "단순 해부학 지식에서 정밀 의료로"의深化에 가깝습니다. 과거 의사는"이 환자는 平均적으로 이 정도로Prescribe한다"(단순 회귀)고 판단했지만, 현대 의사는"이 환자의 유전형, 생활습관, 과거 병력을 반영하면 이 환자에게만 정확한 복용량을 Custom"합니다(정밀 의료). 마찬가지로 데이터 과학에서도"전체平均적モデル"에서"각 세그먼트별, 각 상황별_custom Regression Model"로의 진화가 가속화되고 있으며, 이것이 바로**Causal Inference + Machine Learning** 융합의 핵심 방향입니다.

## 🧠 지식 맵 (Knowledge Graph)

*   **회귀 분석 분류 트리 (Regression Taxonomy)**
    *   **선형 회귀**: 단순 선형, 다중 선형, 다항 회귀, 정규화 (릿지/라쏘/엘라스틱넷)
    *   **로지스틱 회귀**: 이항 분류, 다항 로지스틱 (softmax)
    *   **비선형 회귀**: GAM (Generalized Additive Model), 스플라인 회귀
    *   **시계열 회귀**: AR, MA, ARMA, ARIMA
*   **핵심 진단 지표**
    *   VIF (분산팽창계수) - 다중 공선성 검출
    *   White 검정 - 이분산성 검출
    *   Durbin-Watson - 자기상관 검출
    *   Cook's Distance - 이상치/영향점 검출
*   **회귀 vs 분류**
    *   회귀: 연속형 출력 (매출액, 온도, 가격)
    *   분류: 범주형 출력 (이탈/유지, 구매/비구매)
