## 핵심 인사이트 (3줄 요약)

- **본질**: 효과적인 대시보드는 5초 규칙(가장 중요한 인사이트를 5초 안에 파악)을 충족하는 **KPI (Key Performance Indicator) 중심의 계층적 정보 구조**로, F-패턴 시선 흐름과 시각적 계층(Primary KPI → 지원 지표 → 상세 테이블)을 따라 설계된다.
- **가치**: 전략·운영·분석 대시보드의 3유형을 구분하고, 사전 집계(Pre-aggregation)·캐싱·증분 새로고침으로 대규모 데이터셋에서도 서브세컨드 응답을 보장하는 성능 설계가 빅데이터 대시보드의 핵심이다.
- **판단 포인트**: 글로벌 필터·드릴스루·크로스 필터링의 인터랙티비티 패턴을 독자의 분석 목적(탐색 vs 모니터링)에 따라 적절히 제공해야 하며, 과도한 인터랙션은 오히려 목적을 흐린다.

---

## Ⅰ. 개요 및 필요성

### 대시보드란 무엇인가

대시보드(Dashboard)는 **조직의 목표 달성 현황을 한 화면에서 모니터링·분석할 수 있도록 주요 지표를 시각화한 인터페이스**다. 이름은 자동차 계기판에서 유래 — 운전하면서 속도·연료·온도를 한눈에 확인하는 것처럼, 비즈니스 운영 상황을 즉각적으로 파악한다.

### 나쁜 대시보드의 증상

- 화면에 40개의 차트 → 어디를 봐야 할지 모름
- 모든 숫자가 같은 크기 → 중요도 구분 불가
- 필터가 너무 많음 → 분석가만 사용 가능
- 데이터 갱신 5분 소요 → 운영 모니터링 불가
- 데스크탑 전용 레이아웃 → 모바일에서 깨짐

**📢 섹션 요약 비유**: 좋은 대시보드는 **비행기 조종석**과 같다. 조종사(의사결정자)가 즉각적으로 필요한 정보(고도, 속도, 연료)를 파악하도록 최적화되어 있다. 중요한 경보는 크고 빨갛게, 세부 정보는 요청 시 확인한다.

## Ⅱ. 아키텍처 및 핵심 원리

### 대시보드 3유형

```
┌─────────────────────────────────────────────────────────────┐
│                대시보드 3유형 비교                           │
├──────────────┬──────────────────┬───────────────────────────┤
│  전략 대시보드│   운영 대시보드  │   분석 대시보드           │
│  (Strategic) │  (Operational)   │   (Analytical)            │
├──────────────┼──────────────────┼───────────────────────────┤
│ 대상: C-Suite│ 대상: 운영팀     │ 대상: 분석가              │
│ 주기: 월/분기 │ 주기: 실시간/일간│ 주기: 필요 시             │
│              │                  │                           │
│ KPI 방향성,  │ 현재 상태 vs SLA │ 드릴다운, 슬라이스,       │
│ 목표 대비     │ 알림, 임계값 초과 │ 다이스, 필터 탐색         │
│              │                  │                           │
│ 인터랙션: 최소│ 인터랙션: 경보   │ 인터랙션: 최대            │
│ 갱신: 일간+  │ 갱신: 실시간     │ 갱신: 필요 시             │
└──────────────┴──────────────────┴───────────────────────────┘
```

### KPI 계층 설계 (Visual Hierarchy)

```
────── 1차 KPI 카드 (Primary KPI Cards) ──────
  ┌────────────┐  ┌────────────┐  ┌────────────┐
  │  매출 합계  │  │  신규 고객  │  │ 전환율     │
  │  ₩1.2조    │  │  12,450명  │  │  3.8%      │
  │  ▲ +12% YoY│  │  ▲ +8%    │  │ ▼ -0.2%pp  │
  └────────────┘  └────────────┘  └────────────┘
         ↓ 1차 KPI를 설명하는
────── 2차 지원 지표 차트 ──────
  ┌──────────────────────────────┐ ┌──────────┐
  │  월별 매출 추세 (꺾은선)      │ │ 지역별   │
  │                              │ │ 매출 비중│
  └──────────────────────────────┘ └──────────┘
         ↓ 상세 진단을 위한
────── 3차 상세 테이블 ──────
  ┌────────────────────────────────────────────┐
  │  제품별 매출·이익·성장률 상세 테이블        │
  └────────────────────────────────────────────┘
```

### 인터랙티비티 패턴

| 패턴 | 구현 방식 | 활용 |
|:---|:---|:---|
| **글로벌 필터** | 상단 필터 바 → 모든 차트에 동시 적용 | 날짜 범위, 지역 필터 |
| **드릴스루** | 차트 클릭 → 해당 항목 상세 페이지 이동 | 제품 클릭 → 제품 상세 |
| **크로스 필터링** | 차트 A 클릭 → 차트 B 자동 필터 | Power BI 핵심 기능 |
| **툴팁** | 호버 시 상세 값 표시 | 공간 절약, 상세 확인 |
| **파라미터** | 사용자 입력 값으로 계산 동적 변경 | 시나리오 분석 |

**📢 섹션 요약 비유**: 대시보드 KPI 계층은 **뉴스 기사의 역피라미드 구조**와 같다. 헤드라인(1차 KPI)이 가장 중요하고 가장 크게, 그 다음 리드 문단(2차 차트), 상세 내용(3차 테이블) 순으로 계층화된다.

## Ⅲ. 비교 및 연결

### F-패턴 시선 흐름 활용

사용자 시선 연구(Eye Tracking)에서 웹 페이지를 볼 때 F자 형태의 시선 패턴이 관찰된다:

```
시선 흐름:
  ──────────────────── (첫 번째 가로 스캔)
  ──────────
  │
  │ (세로 스캔)
  │
  (두 번째 가로 스캔)
```

대시보드 적용:
- **좌상단**: 가장 중요한 Primary KPI 배치
- **상단 가로**: 전체 KPI 카드 배치
- **좌측 세로**: 주요 필터·네비게이션
- **중앙**: 핵심 차트

### 성능 최적화 설계

| 기법 | 설명 | 적용 도구 |
|:---|:---|:---|
| **사전 집계 (Pre-aggregation)** | 대용량 데이터를 미리 집계·저장 | Druid, ClickHouse, 집계 테이블 |
| **캐싱** | 자주 사용하는 쿼리 결과 캐시 | Redis, 도구 내장 캐시 |
| **증분 새로고침** | 전체가 아닌 새로운 데이터만 갱신 | Power BI Premium, Tableau |
| **WebAssembly** | 클라이언트 측 고성능 처리 | Observable Plot, Perspective |

**📢 섹션 요약 비유**: 대시보드 사전 집계는 **식당의 밀 프레핑(Meal Prep)**과 같다. 손님이 주문하기 전에 식재료를 미리 손질해두면(사전 집계), 주문 후 즉시 요리할 수 있다(서브세컨드 응답).

## Ⅳ. 실무 적용 및 기술사 판단

### 대시보드 설계 프로세스

1. **사용자 인터뷰**: 누가 어떤 결정을 위해 사용하는가?
2. **KPI 정의**: 핵심 질문 3개 이하 → Primary KPI 선정
3. **스케치**: 종이 프로토타입으로 레이아웃 설계
4. **데이터 모델 검토**: 스타 스키마, 집계 레이어 설계
5. **프로토타입 제작**: 실제 도구로 MVP 구현
6. **사용자 테스트**: 5초 테스트 — 화면 보고 5초 후 무엇을 파악했는지 묻기
7. **성능 최적화**: 쿼리 최적화, 캐싱, 사전 집계

### 반응형 대시보드 설계

| 기기 | 설계 원칙 |
|:---|:---|
| **데스크탑** | 전체 레이아웃, 상세 테이블, 모든 필터 노출 |
| **태블릿** | 중요 KPI + 핵심 차트, 일부 필터 |
| **모바일** | Primary KPI 카드만, 스와이프 네비게이션 |

**📢 섹션 요약 비유**: 반응형 대시보드는 **트랜스포머 완구**와 같다. 데스크탑(자동차 형태)에서 태블릿(중간 형태)으로, 모바일(로봇 형태)로 변신하면서 각 상황에 맞는 최적 형태를 제공한다.

## Ⅴ. 기대효과 및 결론

### 효과적인 대시보드의 성과

| 영역 | 효과 |
|:---|:---|
| **의사결정 속도** | KPI 파악 시간 60-80% 단축 |
| **이상 감지** | 임계값 기반 알림으로 문제 즉시 파악 |
| **데이터 민주화** | 분석가 없이 비즈니스 사용자 셀프서비스 |
| **일관성** | 단일 데이터 소스로 팀 간 KPI 불일치 해소 |

### 결론

대시보드 설계는 **데이터 거버넌스의 최전선**이다. 아무리 좋은 데이터와 분석이 있어도, 의사결정자가 쉽고 빠르게 인사이트를 얻지 못하면 가치가 없다. 5초 규칙, F-패턴 레이아웃, KPI 계층 구조, 적절한 인터랙티비티, 성능 최적화의 5가지 요소를 통합적으로 설계할 수 있는 것이 데이터 플랫폼 아키텍트의 역량이다.

**📢 섹션 요약 비유**: 잘 설계된 대시보드는 **응급실 모니터**와 같다. 의사가 복잡한 조작 없이 환자의 생명 신호(KPI)를 즉각 파악하고, 이상이 감지되면 경보가 울리며, 상세 검사는 클릭 한 번으로 확인할 수 있어야 한다.

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| 전략 대시보드 | 유형 분류 | C-Suite 대상, 월간 KPI |
| 운영 대시보드 | 유형 분류 | 실시간 알림, SLA 모니터링 |
| 분석 대시보드 | 유형 분류 | 드릴다운, 탐색적 분석 |
| F-패턴 | 레이아웃 원칙 | 시선 이동 패턴 기반 중요도 배치 |
| 크로스 필터링 | 인터랙션 패턴 | 차트 A 선택 → 차트 B 필터 자동 반영 |
| 사전 집계 | 성능 기법 | 대용량 데이터 미리 집계·저장 |
| KPI 계층 | 정보 구조 | Primary → Supporting → Detail 3단계 |

### 📈 관련 키워드 및 발전 흐름도

```text
[:---]
    │
    ▼
[전략 대시보드]
    │
    ▼
[운영 대시보드]
    │
    ▼
[분석 대시보드]
    │
    ▼
[F-패턴]
    │
    ▼
[크로스 필터링]
```

이 흐름도는 :---에서 출발해 F-패턴까지 이어지며, 중간 단계가 기초 개념을 실무 구조로 발전시키는 과정을 보여준다.

### 👶 어린이를 위한 3줄 비유 설명

- 대시보드는 **자동차 계기판**이에요: 운전할 때 속도·연료·온도를 한눈에 볼 수 있듯, 회사 상황(매출·고객·품질)을 한 화면에서 바로 알 수 있어요.
- 5초 규칙은 "화면을 딱 5초만 봐도 가장 중요한 것을 알 수 있어야 한다"는 원칙이에요 — 5초 안에 이해 못하면 설계가 실패한 거예요.
- 크로스 필터링은 **마법 안경**이에요: 차트 하나를 손가락으로 터치하면 다른 모든 차트가 그것과 관련된 정보만 보여줘요.
