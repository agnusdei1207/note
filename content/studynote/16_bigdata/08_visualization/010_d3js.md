## 핵심 인사이트 (3줄 요약)

- D3.js (Data-Driven Documents)는 데이터를 DOM (Document Object Model)에 직접 바인딩하여 SVG (Scalable Vector Graphics) 기반의 완전 커스텀 인터랙티브 시각화를 구현한다.
- Enter/Update/Exit 패턴으로 데이터 변화에 따른 요소 추가·갱신·제거를 선언적으로 처리하며, 이는 React의 가상 DOM 개념보다 먼저 존재한 데이터 주도 렌더링 패러다임이다.
- 높은 유연성의 대가로 학습 곡선이 가파르며, 빠른 프로토타이핑에는 Observable Plot이나 Vega-Lite 같은 고수준 래퍼를 선택하는 것이 합리적이다.

---

## Ⅰ. 개요 및 필요성

### 1-1. D3.js란?

D3.js는 Mike Bostock이 개발한 JavaScript 라이브러리로, 데이터를 HTML/SVG/Canvas 요소에 바인딩하고 데이터 값에 따라 시각 속성(위치, 크기, 색상)을 변환하는 "데이터 주도 문서(Data-Driven Documents)" 철학을 구현한다.

### 1-2. D3.js가 필요한 이유

| 요구사항 | D3.js 적합도 | 이유 |
|:---|:---:|:---|
| 표준 막대/선 차트 | △ | Highcharts, Chart.js가 더 빠름 |
| 커스텀 네트워크 그래프 | ◎ | 완전 제어 가능 |
| 인터랙티브 지도 (커스텀) | ◎ | GeoJSON + D3-geo 결합 |
| Scrollytelling 시각화 | ◎ | 스크롤 연동 애니메이션 |
| 실시간 데이터 스트리밍 | ○ | 전환(Transition)으로 부드러운 업데이트 |

> 📢 **섹션 요약 비유**: D3.js는 LEGO 블록 세트다. 원하는 무엇이든 만들 수 있지만, 조립 설명서가 없어 직접 설계해야 한다.

## Ⅱ. 아키텍처 및 핵심 원리

### 2-1. Data Binding과 Enter/Update/Exit 패턴

```
       데이터 배열  [A, B, C]
            │
    ┌───────▼────────┐
    │  d3.selectAll  │  ◀── DOM 요소 선택
    │  .data([...])  │  ◀── 데이터 바인딩
    └───────┬────────┘
            │
    ┌───────┴─────────────────────┐
    │                             │
    ▼                             ▼
┌──────────┐              ┌──────────────┐
│  .enter()│              │  .exit()     │
│  데이터는│              │  DOM은 있고  │
│  있으나  │              │  데이터 없음 │
│  DOM 없음│              │  → .remove() │
│  → append│              └──────────────┘
└──────────┘
    ▼
┌──────────┐
│ .merge() │  ← Enter + Update 합쳐서 속성 일괄 설정
└──────────┘
```

### 2-2. 핵심 모듈

| 모듈 | 기능 |
|:---|:---|
| d3-scale | Linear/Log/Ordinal/Time 스케일 (데이터 값 → 픽셀 좌표 매핑) |
| d3-axis | 축 자동 생성 (tick 포맷, 눈금 개수 제어) |
| d3-transition | 애니메이션: `duration()`, `ease()`, `delay()` |
| d3-zoom | 확대/축소, 패닝 제스처 처리 |
| d3-force | 물리 시뮬레이션 기반 네트워크 레이아웃 |
| d3-geo | 지리 투영법 (Mercator, Albers 등) |
| d3-hierarchy | 트리·트리맵·서클패킹 레이아웃 |

### 2-3. Observable Plot — D3 위의 고수준 래퍼

Observable Plot은 D3 창시자 Mike Bostock이 개발한 Grammar-of-Graphics 기반 라이브러리로, `Plot.dot()`, `Plot.line()`, `Plot.barX()` 등 마크(mark) 기반 API로 D3의 30줄 코드를 5줄로 축약한다.

> 📢 **섹션 요약 비유**: Enter/Update/Exit 패턴은 학교 출석부와 같다. 새 학생은 추가(Enter), 기존 학생은 자리 이동(Update), 전학 간 학생은 삭제(Exit)한다.

### D3.js 사용 적합 기준

- 기존 차트 라이브러리로 표현 불가능한 **완전 커스텀** 시각화 필요 시
- 데이터 저널리즘, Scrollytelling 등 **스토리텔링 중심** 시각화
- 팀 내 JavaScript 전문가가 있고 개발 기간이 충분할 때

> 📢 **섹션 요약 비유**: D3.js는 맞춤 정장 제작이고, Highcharts는 기성복이다. 체형이 독특하다면 맞춤 정장이 정답이지만, 시간과 비용이 든다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4-1. 빅데이터와 D3.js 성능 한계

SVG는 DOM 요소 1만 개 이상에서 렌더링 성능이 급격히 저하된다. 대용량 데이터 처리를 위한 대안:

- **Canvas 렌더링**: `d3-canvas-transition`, D3 + WebGL 조합
- **서버사이드 집계**: 원시 데이터 대신 Pre-aggregated 데이터 시각화
- **Datashader**: Python 기반 서버사이드 래스터화 후 이미지로 전송

### 4-2. Scrollytelling 패턴

```javascript
// 스크롤 위치에 따라 시각화 전환 예시
scrollama().setup({ step: ".step" })
  .onStepEnter(({ index }) => {
    chart.transition().duration(500).call(update, steps[index]);
  });
```

### 4-3. 기술사 시험 포인트

- D3.js 선택 근거: 차별화된 인터랙티브 UX, 기성 라이브러리 한계 극복
- 성능 병목 해결: SVG → Canvas/WebGL 전환, 데이터 사전 집계
- 유지보수 리스크: 팀 의존도, Observable Plot 등 래퍼 고려

> 📢 **섹션 요약 비유**: D3.js의 SVG 한계는 "종이 위에 그리는 그림"처럼 복잡해질수록 무거워진다. Canvas는 "디지털 그림판"으로 메모리를 효율적으로 쓴다.

## Ⅴ. 기대효과 및 결론

| 효과 | 내용 |
|:---|:---|
| 차별화된 UX | 타 라이브러리로 구현 불가한 인터랙티브 경험 제공 |
| 데이터 스토리텔링 | 스크롤·애니메이션으로 데이터의 인사이트 전달력 극대화 |
| 오픈소스 생태계 | 수천 개의 D3 예제 재사용, 빠른 프로토타이핑 가능 |

D3.js는 시각화 자유도의 정점에 있는 도구이지만, 모든 프로젝트에 적합하지 않다. 기술사 관점에서 D3.js 채택 결정 시 팀 역량·개발 기간·유지보수 계획을 함께 제시하는 것이 중요하다. 빠른 결과가 필요한 경우 Observable Plot 또는 Vega-Lite를 먼저 검토하는 것이 합리적이다.

> 📢 **섹션 요약 비유**: D3.js는 마스터 셰프의 요리처럼 최고의 결과물을 낼 수 있지만, 그만큼 숙련된 손이 필요하다.

```text
[정적 차트 (Static Chart) — Excel·이미지 기반 시각화]
    │
    ▼
[인터랙티브 차트 라이브러리 — Highcharts·Chart.js]
    │
    ▼
[D3.js — SVG·DOM 직접 조작 기반 커스텀 데이터 시각화]
    │
    ▼
[데이터 바인딩 (Data Binding) — 데이터와 DOM 요소 연결]
    │
    ▼
[인터랙티브 대시보드 — React + D3 통합 실시간 시각화]
    │
    ▼

```
D3.js는 데이터와 DOM을 직접 연결하는 선언적 바인딩 패러다임으로, 정적 차트 라이브러리를 넘어 완전히 커스텀 가능한 인터랙티브 시각화의 표준이 되었다.

### 👶 어린이를 위한 3줄 비유 설명

1. D3.js는 미술 시간에 자유롭게 그림을 그릴 수 있는 빈 도화지예요. 원하는 건 뭐든 그릴 수 있지만, 스스로 다 만들어야 해요.
2. 숫자가 바뀌면 그림도 스르르 바뀌는 마법 같은 기능이 있어요. 이걸 "전환(Transition)"이라고 해요.
3. 그림이 너무 많아지면 느려지기 때문에, 전문가들은 Canvas라는 더 빠른 도화지를 사용해요.
