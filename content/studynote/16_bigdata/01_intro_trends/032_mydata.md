> **핵심 인사이트 3줄**
> 1. 마이데이터(MyData)는 개인이 자신의 데이터 수집·이용·제공을 직접 통제하는 개인정보 자기결정권 기반 데이터 유통 패러다임이다.
> 2. 한국은 2021년 금융 마이데이터를 시작으로 의료·공공·통신 등 전 분야로 확대해 API 기반 데이터 이동권을 법제화하고 있다.
> 3. 마이데이터는 개인에게 데이터 통제권을 부여하는 동시에, 기업에겐 동의 기반 개인화 서비스와 새로운 데이터 비즈니스 모델의 기회를 제공한다.

---

## Ⅰ. 마이데이터의 정의와 배경

마이데이터(MyData)는 **개인이 자신의 데이터를 주도적으로 관리·활용하는 데이터 주권(Data Sovereignty) 개념**이다.

### 기존 데이터 유통 vs 마이데이터

| 특성          | 기존 방식              | 마이데이터              |
|-------------|----------------------|------------------------|
| 데이터 주체   | 기업 (데이터 소유)    | 개인 (데이터 주권)      |
| 수집 동의    | 포괄적·묵시적         | 명시적·세분화된 동의    |
| 데이터 이동   | 기업 간 계약 기반     | 개인 API 요청 기반      |
| 투명성       | 낮음                  | 높음 (이용 내역 조회)   |

### 법적 근거

- **한국**: 신용정보법 개정(2020), 개인정보 보호법 개정(2020) — 데이터 이동권
- **EU**: GDPR 제20조 — 데이터 이동권 (Right to Data Portability)
- **미국**: CCPA — 캘리포니아 소비자 프라이버시법

📢 **섹션 요약 비유**: 마이데이터는 은행 계좌를 다른 앱으로 옮기는 권리다 — 내 돈(데이터)은 내가 원하는 곳에 가져갈 수 있어야 한다는 원칙.

---

## Ⅱ. 한국 금융 마이데이터 서비스

### 서비스 구조 (2022년 1월 정식 출범)

```
개인 ──→ 마이데이터 앱 (토스, 뱅크샐러드, 카카오페이)
   │
   └─ 동의 → 마이데이터 사업자 API 요청
                   │
            금융 API (은행·카드·증권·보험)
                   │
         통합 금융 정보 조회·분석·서비스
```

### 마이데이터 정보 범위

| 분야    | 포함 데이터                           |
|-------|---------------------------------------|
| 은행    | 계좌, 거래 내역, 대출 현황             |
| 카드    | 이용 내역, 청구 정보                   |
| 증권    | 보유 종목, 잔고                        |
| 보험    | 가입 내역, 보험료                      |
| 통신    | 요금 정보 (확대 예정)                   |

**마이데이터 사업자 허가**: 금융위원회 허가 → API 표준 준수 의무

📢 **섹션 요약 비유**: 금융 마이데이터는 여러 은행 통장을 한 앱에서 보는 것이다 — 은행마다 앱을 켜지 않아도 내 모든 자산을 한눈에 볼 수 있다.

## Ⅲ. 마이데이터 API 아키텍처

```
┌───────────────────────────────────────────────────────┐
│                 마이데이터 플랫폼                      │
│                                                       │
│  개인      ──→  마이데이터 사업자  ──→  정보 제공 기관│
│ (본인 인증)    (앱/플랫폼)          (은행·카드사 등) │
│                   │                       │          │
│                OAuth 2.0                 Open API     │
│                /동의 관리                REST/JSON    │
│                   │                                   │
│           마이데이터 종합 포털                        │
│           (금융결제원 중계)                           │
└───────────────────────────────────────────────────────┘
```

### 기술 표준

- **인증**: OAuth 2.0 + PKCE
- **API**: REST/JSON, OpenAPI 3.0 스펙
- **보안**: TLS 1.3, 상호 인증 mTLS
- **중계**: 금융결제원 마이데이터 중계 서버

📢 **섹션 요약 비유**: 마이데이터 API는 통역사가 있는 외교 회담이다 — 각기 다른 언어(은행·카드·보험 데이터 형식)를 공통 언어(API 표준)로 번역해 하나의 앱에 모아준다.

## Ⅳ. 마이데이터 비즈니스 모델

| 모델              | 내용                                |
|-----------------|-------------------------------------|
| 자산 통합 관리    | PFM (Personal Financial Management) |
| 개인화 추천       | 맞춤 금융상품·보험 추천             |
| 신용 평가        | 대안 신용 평가 (통신·쇼핑 데이터)   |
| 데이터 마켓플레이스 | 개인 동의 기반 익명화 데이터 판매  |
| 헬스케어 마이데이터 | 진료·검진 데이터 통합 건강 관리    |

### 마이데이터 3.0 비전

```
1.0: 조회 (내 데이터 한곳에 보기)
2.0: 분석 (AI 기반 맞춤 조언)
3.0: 실행 (자동 금융 최적화·자산 관리 AI)
```

📢 **섹션 요약 비유**: 마이데이터 비즈니스는 개인 비서 서비스다 — 내 모든 금융 정보를 파악한 AI가 "이 대출로 갈아타면 월 3만 원 절약돼요"를 자동으로 알려준다.

## Ⅴ. 마이데이터 확대와 도전 과제

### 분야별 확대 현황

| 분야    | 진행 현황                      |
|-------|-------------------------------|
| 금융    | 2022년 법 시행, 사업자 100개+  |
| 의료    | 마이헬스웨이 시범 (2023~)      |
| 공공    | 행정데이터 이동권 논의 중       |
| 통신    | 통신 마이데이터 추진 중         |

### 도전 과제

| 과제          | 내용                           |
|-------------|-------------------------------|
| 개인정보 보호 | 대규모 집중화 → 해킹 위험       |
| 정보 비대칭   | 개인의 데이터 가치 이해 부족    |
| 플랫폼 독점   | 대형 앱에 데이터 쏠림 현상      |
| 동의 피로     | 과도한 동의 요청 → 묵시적 동의  |

📢 **섹션 요약 비유**: 마이데이터 도전 과제는 집 열쇠를 한 곳에 보관하는 것이다 — 편리하지만, 그 보관함이 털리면 모든 열쇠가 한꺼번에 노출된다.

## 👶 어린이를 위한 3줄 비유 설명

1. 마이데이터는 내 성적표를 내가 관리하는 것이다 — 선생님(은행·병원)이 가지고 있던 성적표를 내가 직접 볼 수 있고, 필요한 곳에 가져갈 수 있다.
2. 금융 마이데이터는 여러 통장을 하나의 앱에서 보는 것이다 — 은행마다 앱을 열지 않아도 내 모든 돈이 어디에 얼마나 있는지 한눈에 알 수 있다.
3. 데이터 이동권은 전학 갈 때 성적표를 가져갈 권리다 — 학교(서비스)를 바꿔도 내 기록(데이터)은 내가 새 학교(앱)에 직접 가져갈 수 있다.
