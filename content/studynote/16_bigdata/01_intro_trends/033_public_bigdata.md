> **핵심 인사이트 3줄**
> 1. 공공 빅데이터는 정부·공공기관이 생산·보유한 데이터를 민간에 개방해 사회·경제적 가치를 창출하는 데이터 민주화의 핵심 인프라다.
> 2. 한국의 공공데이터포털(data.go.kr)은 78,000여 개 데이터셋을 제공하며, 교통·복지·의료·환경 분야 혁신 서비스의 기반이 되고 있다.
> 3. 데이터 품질·개인정보 보호·민감 정보 비식별화가 공공 데이터 활용의 3대 과제이며, FAIR 원칙(Findable·Accessible·Interoperable·Reusable)이 국제 공공 데이터 개방의 표준이다.

---

## Ⅰ. 공공 빅데이터의 정의와 특성

공공 빅데이터(Public Big Data)는 **정부·지자체·공공기관이 행정·사업 과정에서 생산·수집한 데이터를 디지털 형태로 공개한 것**이다.

### 공공 데이터 유형

| 유형          | 예시                               |
|-------------|-----------------------------------|
| 행정 데이터  | 인구 통계, 사업자 등록, 부동산 거래 |
| 의료 데이터  | 건강보험 진료 데이터, 건강 통계     |
| 교통 데이터  | 버스/지하철 운행, 교통량, 주차 정보 |
| 환경 데이터  | 대기질, 수질, 기상 데이터           |
| 재정 데이터  | 국가 예산·결산, 공공 조달           |
| 교육 데이터  | 학교 정보, 교육 통계               |

### 공공 데이터의 특성

- **신뢰성**: 공공기관의 공식 데이터
- **규모**: 국가 단위 대규모 시계열 데이터
- **희소성**: 민간이 수집 불가능한 정보
- **공공성**: 사회 전체의 이익을 위한 개방

📢 **섹션 요약 비유**: 공공 빅데이터는 국가가 운영하는 도서관이다 — 세금으로 만든 책(데이터)을 모든 시민이 무료로 빌려 볼 수 있다.

## Ⅱ. 한국 공공데이터 개방 현황

### 공공데이터포털 (data.go.kr)

```
공공데이터포털 현황 (2024년 기준):
  - 등록 데이터셋: 78,000여 개
  - API 서비스: 8,000여 개
  - 제공 기관: 1,100여 개
  - 월간 이용: 1억 건 이상
```

### 주요 활용 사례

| 분야   | 서비스                         | 사용 데이터             |
|------|-------------------------------|------------------------|
| 교통  | 카카오맵 실시간 버스 정보         | 공공 교통 API           |
| 부동산 | 호갱노노 실거래가 분석           | 국토부 아파트 실거래가   |
| 날씨  | 기상 예보 앱                    | 기상청 API              |
| 의료  | 코로나19 확진자 동선 추적         | 건강보험공단 진료 데이터 |
| 창업  | 상권 분석 서비스                  | 소상공인시장진흥공단     |

📢 **섹션 요약 비유**: 공공데이터포털은 국가 주방이다 — 정부가 키운 식재료(원시 데이터)를 민간 요리사(개발자·기업)가 자유롭게 가져다 요리(서비스)를 만든다.

## Ⅲ. FAIR 원칙 — 공공 데이터 개방 국제 표준

| 원칙             | 의미                          | 실천 방법              |
|--------------|------------------------------|----------------------|
| Findable (발견성) | 데이터를 쉽게 찾을 수 있음   | 메타데이터·검색 인덱스   |
| Accessible (접근성) | 인증·권한에 맞게 접근 가능 | API·개방 프로토콜       |
| Interoperable (상호운용성) | 표준 형식으로 교환 가능 | JSON/CSV·표준 코드  |
| Reusable (재사용성) | 라이선스 명확, 재사용 허용 | CCL·공공누리 라이선스   |

### 5성급 공공 데이터 (Tim Berners-Lee)

```
★     : 어떤 형식이든 오픈 라이선스
★★    : 기계 판독 가능 구조화 형식 (xls → csv)
★★★   : 비독점 형식 (csv, json)
★★★★  : RDF URI 사용 (연결 데이터)
★★★★★ : 다른 데이터와 링크 (Linked Open Data)
```

📢 **섹션 요약 비유**: FAIR 원칙은 요리 레시피 공유 기준이다 — 인터넷에서 찾기 쉽고(Findable), 누구나 볼 수 있고(Accessible), 다른 레시피와 합치기 쉽고(Interoperable), 마음대로 써도 되는(Reusable) 레시피가 좋은 레시피이다.

## Ⅳ. 공공 빅데이터 활용 — 스마트시티·행정 혁신

### 교통 빅데이터 분석

```
공공 교통 데이터 파이프라인:
  버스 GPS → 실시간 수집 → Apache Kafka
                                ↓
                         실시간 처리 (Flink)
                                ↓
                     배차 최적화·혼잡 예측 알림
```

### 의료 공공 데이터 활용

- **건강보험공단 빅데이터**: 2억 건 이상 진료 내역 → 질병 예측 모델
- **DUR (Drug Utilization Review)**: 처방 안전성 실시간 점검
- **국가암데이터**: 암 발생·생존율 분석, 조기 발견 AI 개발

### 사회복지 데이터 매칭

```
복지 사각지대 발굴:
  전기·수도 단수 데이터 + 건강보험 미납 + 복지 수급 여부
  → AI 고위험군 예측 → 담당자 선제 방문
```

📢 **섹션 요약 비유**: 복지 데이터 매칭은 동네 이장이 어려운 이웃을 찾는 것이다 — 전기가 끊기고 연락이 안 되면(데이터 신호) 먼저 찾아가는 능동적 복지를 AI로 구현한다.

## Ⅴ. 공공 데이터 품질과 개인정보 보호

### 공공 데이터 품질 5대 기준

| 기준        | 설명                          |
|-----------|-------------------------------|
| 정확성     | 실제 값과 일치                 |
| 완전성     | 필수 항목 누락 없음             |
| 일관성     | 기관 간 동일 데이터 일관 유지   |
| 적시성     | 최신 데이터 유지                |
| 유효성     | 정해진 형식·범위 내 값          |

### 개인정보 비식별화 기법

| 기법          | 방법                          | 적용 예시              |
|------------|-------------------------------|----------------------|
| 가명처리    | 개인 식별 정보를 가명으로 대체  | 이름 → 홍XX           |
| 총계처리    | 합계·평균으로 개인 정보 제거    | 개별 소득 → 연령대 평균 |
| k-익명성   | k명 이상의 동일 특성 보장       | 주민등록→5세 단위 집계  |
| 범주화      | 정확한 값을 범위로 대체         | 나이 25→20대           |

📢 **섹션 요약 비유**: 비식별화는 사진 속 얼굴 모자이크 처리다 — 모자이크 후에도 사진의 내용(데이터 가치)은 유지되지만, 누가 누구인지(개인 식별)는 알 수 없다.

## 👶 어린이를 위한 3줄 비유 설명

1. 공공 빅데이터는 국가가 운영하는 도서관이다 — 세금으로 만든 데이터 책을 누구나 무료로 읽고 이용할 수 있다.
2. FAIR 원칙은 좋은 도서관 운영 기준이다 — 책을 쉽게 찾고(Findable), 빌리고(Accessible), 다른 책과 합쳐 쓰고(Interoperable), 자유롭게 참고(Reusable)할 수 있어야 한다.
3. 비식별화는 사진 속 얼굴 모자이크다 — 내용(데이터 가치)은 유지하되, 누가 누구인지(개인정보)는 알 수 없게 처리한다.
