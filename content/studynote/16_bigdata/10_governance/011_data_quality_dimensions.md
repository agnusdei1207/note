## 핵심 인사이트 (3줄 요약)

- **본질**: DAMA (Data Management Association)가 정의한 6대 데이터 품질 차원(완전성·정확성·일관성·적시성·유일성·유효성)은 "Garbage In, Garbage Out" 문제를 예방하기 위한 다차원 품질 측정 프레임워크다.
- **가치**: 각 차원별 측정 지표(KPI)를 ETL (Extract Transform Load) 파이프라인에 임베드하면 품질 문제를 발생 시점에 즉시 감지·차단하는 'Fail Fast' 방어선을 구축할 수 있다.
- **판단 포인트**: IBM 연구에 따르면 미국에서 저품질 데이터로 인한 연간 손실이 3.1조 달러에 달하므로, ML(Machine Learning) 시대에 데이터 품질은 모델 정확도와 직결되는 핵심 인프라 지표다.

---

## Ⅰ. 개요 및 필요성

데이터 품질(Data Quality)은 데이터가 의도된 비즈니스 목적에 **얼마나 적합한가(Fitness for Purpose)**를 나타내는 개념이다. 단순한 기술적 정확성을 넘어, 사용자가 신뢰하고 의사결정에 활용할 수 있는 데이터인지가 핵심이다.

### 저품질 데이터의 비즈니스 영향

- **ML 모델**: 편향된 훈련 데이터 → 편향된 예측 → 잘못된 비즈니스 결정
- **CRM (Customer Relationship Management)**: 중복 고객 레코드 → 같은 고객에게 중복 마케팅 발송 → 이탈
- **재무 보고**: 시스템 간 불일치 데이터 → 회계 감사 실패
- **규제 준수**: 부정확한 개인정보 → GDPR 정보 주체 권리 행사 불가

**📢 섹션 요약 비유**: 데이터 품질은 **요리 재료의 신선도**와 같다. 아무리 훌륭한 셰프(데이터 사이언티스트)와 최첨단 주방(AI 플랫폼)이 있어도, 상한 재료(저품질 데이터)로는 맛있는 요리(올바른 인사이트)를 만들 수 없다.

## Ⅱ. 아키텍처 및 핵심 원리

### DAMA 6대 데이터 품질 차원

```
┌─────────────────────────────────────────────────────────────┐
│              DAMA 데이터 품질 6대 차원                       │
├──────────────┬──────────────────────────────────────────────┤
│  차원        │  정의 및 측정 방법                            │
├──────────────┼──────────────────────────────────────────────┤
│ 1. 완전성    │ 필수 항목 값이 존재하는가?                    │
│  Completeness│ 측정: (비결측 레코드 수 / 전체 레코드 수) × 100│
│              │ 예: 고객 이메일 필드 null 비율 < 1%           │
├──────────────┼──────────────────────────────────────────────┤
│ 2. 정확성    │ 실세계 사실과 일치하는가?                     │
│  Accuracy    │ 측정: 샘플링 후 실제 값과 대조                │
│              │ 예: 고객 주소가 실제 거주지와 일치            │
├──────────────┼──────────────────────────────────────────────┤
│ 3. 일관성    │ 여러 시스템·테이블에서 동일한 속성이 일치하는가│
│  Consistency │ 측정: 시스템 간 동일 레코드 속성 불일치율    │
│              │ 예: CRM 나이 vs DW 나이 동일 여부             │
├──────────────┼──────────────────────────────────────────────┤
│ 4. 적시성    │ 필요한 시점에 최신 데이터가 가용한가?         │
│  Timeliness  │ 측정: 데이터 지연 시간(Latency), SLA 준수율   │
│              │ 예: 실시간 재고 데이터 < 5분 지연             │
├──────────────┼──────────────────────────────────────────────┤
│ 5. 유일성    │ 중복 레코드가 없는가?                         │
│  Uniqueness  │ 측정: 중복 제거율, 동일 엔티티 복수 레코드 비율│
│              │ 예: 고객 ID가 유일하게 존재                   │
├──────────────┼──────────────────────────────────────────────┤
│ 6. 유효성    │ 도메인 규칙·형식·범위를 준수하는가?           │
│  Validity    │ 측정: 도메인 규칙 위반율                      │
│              │ 예: 날짜 형식 YYYY-MM-DD 준수, 나이 0-150    │
└──────────────┴──────────────────────────────────────────────┘
```

### 데이터 품질 점수(DQ Score) 계산

각 차원별 점수를 가중 평균하여 종합 품질 점수를 산출한다:

```
DQ Score = w₁ × 완전성점수 + w₂ × 정확성점수 + w₃ × 일관성점수
         + w₄ × 적시성점수 + w₅ × 유일성점수 + w₆ × 유효성점수
```

여기서 가중치(w)는 비즈니스 우선순위에 따라 조정한다 (예: 재무 데이터는 정확성 가중치 높게).

### ETL 파이프라인 내 DQ 체크 위치

| 단계 | DQ 체크 | 실패 시 조치 |
|:---|:---|:---|
| **Extract** | 소스 데이터 완전성·유효성 | 레코드 거부 (Reject), 알림 |
| **Transform** | 변환 후 일관성·정확성 | 변환 오류 격리 (Quarantine) |
| **Load** | 타겟 적재 후 유일성·적시성 | 적재 차단, 품질 리포트 |

**📢 섹션 요약 비유**: ETL 파이프라인의 DQ 체크는 **공장 생산 라인의 품질 검사대**다. 원재료 입고 시(Extract), 가공 중(Transform), 출하 전(Load) 세 단계에서 검사해야 불량품이 최종 제품까지 통과하지 않는다.

## Ⅲ. 비교 및 연결

### 차원별 측정 도구 매핑

| 차원 | 측정 방법 | 적합 도구 |
|:---|:---|:---|
| 완전성 | `COUNT(col IS NULL) / COUNT(*)` | Great Expectations, SQL |
| 정확성 | 외부 참조 데이터와 대조 | 수동 샘플링, MDM |
| 일관성 | 시스템 간 조인 후 차이 집계 | Deequ (Spark), SQL |
| 적시성 | 타임스탬프 기반 지연 측정 | 모니터링 도구, Airflow |
| 유일성 | `COUNT(DISTINCT key) vs COUNT(*)` | SQL, Great Expectations |
| 유효성 | 정규식·범위 체크 | Soda Core, Great Expectations |

### 품질 차원 간 트레이드오프

- **완전성 vs 정확성**: 빠른 수집을 위해 필드를 선택적으로 받으면 완전성은 낮아지지만, 검증 없이 모두 채우면 정확성이 낮아진다.
- **적시성 vs 정확성**: 실시간 데이터(높은 적시성)는 검증 시간 부족으로 정확성이 낮을 수 있다.
- **유일성 vs 완전성**: 중복 제거(유일성 향상) 과정에서 일부 레코드 손실로 완전성이 낮아질 수 있다.

**📢 섹션 요약 비유**: 품질 차원 간 트레이드오프는 **빠른 배송 vs 정확한 배송**의 딜레마다. 당일 배송(적시성)을 강조하면 포장 검수(정확성)가 소홀해질 수 있다. 비즈니스 우선순위에 따라 균형을 잡아야 한다.

## Ⅳ. 실무 적용 및 기술사 판단

### 데이터 품질 관리(DQM) 구현 단계

1. **프로파일링(Profiling)**: 데이터 탐색적 분석 — 결측률, 카디널리티, 통계 분포 파악
2. **규칙 정의(Rule Definition)**: 스튜어드와 협력하여 도메인별 품질 규칙 정의
3. **자동화된 체크(Automated Check)**: ETL/ELT 파이프라인에 DQ 테스트 임베드
4. **모니터링(Monitoring)**: 시계열 품질 지표 추적, 이상 감지 알림
5. **개선(Remediation)**: 품질 이슈 근본 원인 분석(RCA)·수정·재발 방지

### ML에서의 데이터 품질 중요성

```
데이터 품질 → 훈련 데이터 품질 → 모델 성능
┌─────────────────────────────────────────┐
│  완전성 낮음  →  결측값 처리 오버헤드   │
│  정확성 낮음  →  모델 편향(Bias) 발생   │
│  일관성 낮음  →  특성(Feature) 불안정   │
│  적시성 낮음  →  개념 표류(Concept Drift)│
│  유일성 낮음  →  훈련 샘플 편향         │
│  유효성 낮음  →  이상값(Outlier) 오염   │
└─────────────────────────────────────────┘
```

**📢 섹션 요약 비유**: ML에서 데이터 품질은 **학교 성적의 시험지 품질**과 같다. 시험지에 오타가 있거나 문제가 잘못 출제되면(저품질 데이터), 아무리 열심히 공부한 학생(ML 모델)도 좋은 결과를 낼 수 없다.

## Ⅴ. 기대효과 및 결론

### 품질 차원별 개선 효과

| 차원 | 개선 효과 |
|:---|:---|
| **완전성 향상** | 분석 가능 레코드 수 증가, 통계 대표성 확보 |
| **정확성 향상** | 의사결정 오류 감소, 고객 만족도 향상 |
| **일관성 향상** | 단일 진실 공급원(SSOT) 확립, KPI 불일치 해소 |
| **적시성 향상** | 실시간 의사결정 가능, 운영 효율 향상 |
| **유일성 향상** | 중복 마케팅 제거, 비용 절감 |
| **유효성 향상** | 규정 준수 리스크 감소, 처리 오류 최소화 |

### 결론

데이터 품질 6대 차원은 **데이터를 신뢰하고 활용하기 위한 전제 조건**이다. 특히 AI/ML 시대에 훈련 데이터의 품질이 모델 성능을 좌우하므로, 데이터 품질은 단순한 운영 과제가 아니라 **AI 전략의 기반 인프라**다. 정보통신기술사는 각 차원의 측정 방법과 ETL 파이프라인 내 자동화된 품질 체크 설계를 제안할 수 있어야 한다.

**📢 섹션 요약 비유**: 데이터 품질 6차원은 **건강 검진의 6가지 검사 항목**과 같다. 혈압·혈당·콜레스테롤·시력·체중·청력 중 하나라도 나쁘면 건강에 문제가 생기듯, 데이터도 어느 차원이든 낮으면 전체 분석 신뢰성이 무너진다.

### 👶 어린이를 위한 3줄 비유 설명

- 데이터 품질 6차원은 **성적표의 6개 과목**과 같아요: 국어(완전성), 수학(정확성), 사회(일관성), 과학(적시성), 영어(유일성), 음악(유효성) — 모든 과목이 좋아야 우등생(좋은 데이터)이에요.
- 한 과목이라도 0점이면 전체 평균이 낮아지듯, 데이터 품질도 어느 차원이든 나쁘면 분석 결과를 믿을 수 없어요.
- 특히 AI에게 나쁜 데이터를 주면 AI도 나쁜 결과를 내놓아요 — "쓰레기를 넣으면 쓰레기가 나온다(Garbage In, Garbage Out)"는 이 때문이에요.
