## 핵심 인사이트 (3줄 요약)

- 데이터 카탈로그는 조직의 데이터 자산을 검색·이해·신뢰할 수 있게 하는 인프라로, 기술 메타데이터(스키마)·비즈니스 메타데이터(용어·소유자)·운영 메타데이터(품질·신선도)를 통합한다.
- AWS Glue Catalog는 AWS 생태계(Athena, EMR, Glue ETL)와의 완벽한 통합으로 사실상 AWS 표준 메타스토어이며, LinkedIn DataHub는 멀티플랫폼 환경의 오픈소스 기업 카탈로그로 급성장했다.
- 자동 메타데이터 수확(Automated Harvesting)은 운영 부담을 줄이지만 비즈니스 의미·소유자·품질 규칙은 사람의 큐레이션이 필수이며, 이 두 접근의 균형이 카탈로그 성공의 핵심이다.

---

## Ⅰ. 개요 및 필요성

### 1-1. 데이터 카탈로그의 3가지 메타데이터 유형

| 유형 | 내용 | 자동화 가능성 |
|:---|:---|:---:|
| **기술 메타데이터** | 스키마, 데이터 타입, 파티션, 크기 | 높음 |
| **비즈니스 메타데이터** | 용어집, 소유자, 태그, 설명, 정책 | 낮음 (수동 필요) |
| **운영 메타데이터** | 신선도, 품질 점수, 파이프라인 실행 이력 | 중간 |

### 1-2. 주요 카탈로그 도구

| 도구 | 개발사 | 특징 |
|:---|:---|:---|
| **AWS Glue Data Catalog** | AWS | AWS 서비스 기본 통합, Hive 메타스토어 호환 |
| **LinkedIn DataHub** | LinkedIn | 오픈소스, 엔티티 그래프, 멀티소스 |
| **OpenMetadata** | 커뮤니티 | 오픈 API 표준, 플러그인 아키텍처 |
| **Alation** | Alation | 협업·신뢰도 평점, 엔터프라이즈 UI |
| **Collibra** | Collibra | 거버넌스 중심, 비즈니스 사용자 지향 |

> 📢 **섹션 요약 비유**: 데이터 카탈로그는 대형 마트의 상품 관리 시스템처럼, "이 상품은 어디에 있고, 유통기한은 언제이며, 담당자는 누구인지"를 모두 한 곳에서 찾을 수 있게 한다.

## Ⅱ. 아키텍처 및 핵심 원리

### 2-1. AWS Glue Data Catalog 통합 구조

```
┌────────────────────────────────────────────────────────┐
│              AWS Glue Data Catalog                     │
│  (Hive Metastore 호환, Apache Iceberg 지원)            │
│                                                        │
│  ┌───────────────────────────────────────────────┐    │
│  │  Database > Tables > Partitions > Columns     │    │
│  │  (자동 크롤러 or 수동 등록)                   │    │
│  └───────────────────────────────────────────────┘    │
└───────────┬──────────────────────────────┬────────────┘
            │                              │
     ┌──────▼──────┐               ┌──────▼──────┐
     │  AWS Athena │               │  AWS EMR    │
     │  (SQL 쿼리) │               │  (Spark)    │
     └─────────────┘               └─────────────┘
            │                              │
     ┌──────▼──────┐               ┌──────▼──────┐
     │  AWS Glue   │               │  Redshift   │
     │  (ETL Job)  │               │  Spectrum   │
     └─────────────┘               └─────────────┘
```

### 2-2. OpenMetadata API 표준화 모델

OpenMetadata는 OpenAPI 스펙 기반의 표준화된 메타데이터 API를 제공하여, 어떤 데이터 도구도 동일한 방식으로 카탈로그에 메타데이터를 보내고 받을 수 있다.

| API 엔드포인트 | 역할 |
|:---|:---|
| `/api/v1/tables` | 테이블 CRUD |
| `/api/v1/lineage` | 계보 조회·등록 |
| `/api/v1/dataQuality` | 품질 테스트 결과 |
| `/api/v1/glossaryTerms` | 비즈니스 용어 관리 |

### 2-3. 메타데이터 수확 자동화

- **Glue Crawler**: S3/RDS/DynamoDB 스캔 → 스키마 자동 감지
- **DataHub Ingestion**: Spark, dbt, Airflow 실행 시 메타데이터 자동 발행
- **OpenLineage**: Spark/Flink Job에서 표준 계보 이벤트 자동 발행

> 📢 **섹션 요약 비유**: Glue Crawler는 도서관 사서처럼 정기적으로 서가를 돌며 새로 들어온 책(데이터)을 발견하고 목록에 자동으로 추가한다.

## Ⅲ. 비교 및 연결

| 항목 | AWS Glue | LinkedIn DataHub | OpenMetadata |
|:---|:---|:---|:---|
| 클라우드 | AWS 전용 | 멀티클라우드 | 멀티클라우드 |
| 오픈소스 | 부분 | ✅ | ✅ |
| 비즈니스 메타데이터 | 제한적 | 풍부 | 풍부 |
| 계보 시각화 | 기본 | 그래프 시각화 | 그래프 시각화 |
| 학습 곡선 | 낮음 (AWS 사용자) | 중간 | 중간 |

### 카탈로그 거버넌스 프레임워크

- **데이터 소유권**: 각 Dataset의 비즈니스 소유자 지정 (팀/담당자)
- **스튜어드십**: 데이터 품질·문서화 책임자 지정
- **태그 분류 체계**: 보안 등급(기밀/일반), 도메인(마케팅/재무), 데이터 유형(PII/Non-PII)

> 📢 **섹션 요약 비유**: 데이터 소유권은 회사 자산 관리처럼, 각 데이터셋에 "이 데이터는 마케팅팀이 책임진다"는 담당자를 명확히 지정하는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4-1. 카탈로그 도입 성공 요소

1. **경영진 지원**: 데이터 소유권 지정은 조직 문화 변화 — 상위 의사결정 필요
2. **자동화 우선**: 수동 입력 최소화로 최신 상태 유지
3. **사용자 중심 UX**: 검색 편의성이 사용률을 결정
4. **품질 점수 가시화**: "신뢰할 수 있는 데이터"를 점수로 표시

### 4-2. 기술사 시험 포인트

- Glue Catalog vs Hive Metastore: 호환성, AWS 관리형 vs 자체 운영
- DataHub 채택 기업: LinkedIn, Airbnb, Coursera — 실제 대규모 운영 검증
- 메타데이터 품질: 자동 수집된 기술 메타데이터는 70% 수준, 비즈니스 메타데이터는 수동 큐레이션 필수

> 📢 **섹션 요약 비유**: 데이터 품질 점수는 음식점 위생 등급처럼, 데이터를 믿고 먹을 수 있는지 한눈에 알려주는 신뢰 지표다.

## Ⅴ. 기대효과 및 결론

| 효과 | 내용 |
|:---|:---|
| 탐색 효율 | 데이터 검색 시간 평균 50% 절감 |
| 거버넌스 강화 | PII 태깅으로 GDPR 대응 자동화 |
| 신뢰도 향상 | 계보+품질 점수로 의사결정 데이터 신뢰성 향상 |

데이터 카탈로그 통합은 데이터 거버넌스 성숙도의 핵심 지표다. 기술사 관점에서 카탈로그 도입 계획 시, 자동 수집 파이프라인 설계와 조직 내 데이터 소유권 정책 수립을 동시에 진행해야 한다.

> 📢 **섹션 요약 비유**: 데이터 카탈로그 없는 조직은 색인 없는 백과사전을 갖고 있는 것과 같다. 정보는 있지만 찾을 수 없어서 가치를 발휘하지 못한다.

데이터 카탈로그 통합은 데이터 사일로를 해소하고 메타데이터를 중앙화함으로써, 데이터 리니지 추적과 거버넌스 자동화의 기반을 만든다.

### 👶 어린이를 위한 3줄 비유 설명

1. 데이터 카탈로그는 학교 도서관의 책 목록처럼, 어떤 데이터가 있는지, 누가 만들었는지, 어떻게 사용하는지 정리해놓은 목록이에요.
2. Glue Crawler는 로봇 청소기처럼 자동으로 돌아다니며 새로 생긴 데이터를 발견하고 목록에 추가해줘요.
3. 음식에 유통기한이 있듯이, 데이터에도 언제 만들어졌고 얼마나 신선한지 표시해서 믿을 수 있는 데이터인지 알 수 있어요.
