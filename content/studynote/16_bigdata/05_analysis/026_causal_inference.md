## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인과 추론 (Causal Inference)은 관찰 데이터에서 "A가 B를 일으키는가"를 통계적으로 추론하는 방법론으로, 상관관계 (Correlation ≠ Causation)의 함정을 극복하고, 개입 (Intervention)의 효과를 실험 없이도 추정한다.
> 2. **가치**: RCT (Randomized Controlled Trial, 무작위 대조 실험)를 시행할 수 없는 상황—정책 효과 평가, 의료 처방 비교, 광고 효과 측정—에서 관찰 데이터만으로 인과 효과를 추정하여 증거 기반 의사결정을 가능하게 한다.
> 3. **판단 포인트**: Judea Pearl의 SCM (Structural Causal Model)과 Rubin의 잠재적 결과 프레임워크 (Potential Outcomes)는 인과 추론의 두 가지 수학적 기반이며, Microsoft DoWhy 라이브러리는 이를 실무에서 그래프 명세·식별·추정·반박 4단계로 구현한다.

---

## Ⅰ. 개요 및 필요성

"아이스크림 판매량이 증가하면 익사 사고가 증가한다"—이것은 진짜 인과관계인가? 아니다. 여름이라는 공통 원인 (교란변수, Confounder)이 둘 다 올리는 것이다. 상관관계를 인과관계로 오해하는 실수는 수십억 달러의 잘못된 투자 결정으로 이어진다.

빅데이터 시대의 역설은 데이터가 많을수록 가짜 상관관계도 많아진다는 것이다. 수백만 개의 변수 쌍 중 우연히 상관관계를 보이는 것이 반드시 존재한다. 진짜 인과를 데이터에서 찾아내는 인과 추론은 단순 머신러닝 모델의 다음 단계다.

- **📢 섹션 요약 비유**: 상관관계는 두 현상이 함께 변한다는 것이고, 인과관계는 하나가 다른 하나를 만든다는 것이다. 닭이 울면 해가 뜬다—하지만 닭이 울어서 해가 뜨는 것은 아니다.

## Ⅱ. 아키텍처 및 핵심 원리

### 인과 추론의 두 가지 이론적 기반

```text
┌──────────────────────────────────────────────────────────────────────┐
│             인과 추론 프레임워크 비교                                  │
├────────────────────────────────┬─────────────────────────────────────┤
│  잠재적 결과 (Potential         │  구조적 인과 모델                    │
│  Outcomes, Rubin Causal Model) │  (SCM, Pearl)                       │
├────────────────────────────────┼─────────────────────────────────────┤
│  "Y(처리) vs Y(비처리) 비교"    │  DAG (방향 비순환 그래프)로          │
│  개인별 인과 효과 (ITE)         │  인과 구조를 명시적 모델링           │
│  ATE = E[Y(1)] - E[Y(0)]       │                                     │
│                                 │  X → Y (X가 Y를 일으킴)             │
│  반사실 (Counterfactual):       │  Z → X, Z → Y (Z는 교란변수)        │
│  "만약 처방을 받지 않았다면?"    │                                     │
│                                 │  do() 연산자:                        │
│  근본 문제: 같은 사람에게        │  P(Y|do(X=x)) ≠ P(Y|X=x)          │
│  두 가지를 동시에 적용 불가      │  (개입 vs 관찰)                     │
└────────────────────────────────┴─────────────────────────────────────┘
```

### 주요 인과 추론 방법론

| 방법 | 원리 | 전제 | 적용 사례 |
|:---|:---|:---|:---|
| **RCT (Randomized Controlled Trial)** | 무작위 배정으로 교란변수 균형화 | 실험 가능해야 함 | 임상시험, A/B 테스트 |
| **DID (Difference-in-Differences)** | 처리 전후 차이 비교 (병렬 추세 가정) | 시간 연속 관찰 데이터 | 정책 효과 평가 |
| **도구변수 (IV, Instrumental Variables)** | 처리에만 영향을 주는 외생 변수 활용 | 유효한 도구변수 필요 | 교육 효과, 약 효과 |
| **회귀 불연속 (RDD, Regression Discontinuity)** | 임계값 주변 불연속 이용 | 인위적 임계값 존재 | 장학금 효과, 정책 수혜 |
| **성향점수 매칭 (PSM, Propensity Score Matching)** | 처리 확률이 비슷한 단위 매칭 | 측정된 교란변수 | 관찰 연구 |

### DoWhy 4단계 파이프라인

```text
[1단계: 인과 그래프 명세 (Model)]
    → 도메인 지식 기반 DAG 정의
    → 교란변수, 도구변수, 매개변수 명시

[2단계: 인과 효과 식별 (Identify)]
    → 백도어 기준 (Backdoor Criterion)
    → 프론트도어 기준 (Frontdoor Criterion)
    → IV 조건으로 식별 가능성 확인

[3단계: 추정 (Estimate)]
    → 역확률가중치 (IPW, Inverse Probability Weighting)
    → DML (Double Machine Learning)
    → 도구변수 추정량

[4단계: 반박 (Refute)]
    → 랜덤 공통원인 추가 테스트
    → 플라시보 처리 테스트
    → 부분집합 데이터 견고성 검사
```

- **📢 섹션 요약 비유**: DoWhy의 '반박 (Refute)' 단계는 과학자가 실험 결과를 발표하기 전 "이 결과가 가짜일 수 없는지" 스스로 검증하는 과정이다. 인과 추정도 반드시 이 자기 반박 단계를 통과해야 신뢰할 수 있다.

## Ⅲ. 비교 및 연결

| 항목 | 예측 ML 모델 | 인과 추론 |
|:---|:---|:---|
| **목적** | Y 예측 (P(Y\|X)) | 개입 효과 추정 (P(Y\|do(X))) |
| **데이터** | 관찰 데이터 | 관찰 데이터 (또는 실험) |
| **교란변수** | 피처로 포함, 방향 무관 | 명시적 모델링 필수 |
| **결과 해석** | "X와 Y가 관련 있다" | "X가 Y를 n만큼 증가시킨다" |
| **정책 활용** | 예측, 분류 | 정책 설계, 개입 결정 |

인과 추론은 "어떤 처방이 효과 있는가"를 답하므로 정책 결정·의료·마케팅 투자 배분에서 일반 ML 예측 모델보다 직접적인 비즈니스 가치를 제공한다.

- **📢 섹션 요약 비유**: 예측 ML은 "내일 비가 올 확률 80%"를 말하고, 인과 추론은 "인공 강우를 하면 실제로 비가 20% 더 내릴 것"을 말한다. 하나는 예측, 다른 하나는 개입의 효과다.

## Ⅳ. 실무 적용 및 기술사 판단

### 적용 시나리오

1. **마케팅 기여도 분석**: 광고 노출이 실제 구매를 얼마나 증가시켰는가? (PSM 또는 DML 활용)
2. **정책 효과 평가**: 최저임금 인상이 고용에 미치는 효과? (DID 방법론)
3. **의료 처방 비교**: 약 A vs 약 B의 관찰 데이터에서 인과 효과 추정 (도구변수)
4. **알고리즘 공정성**: 특정 속성이 결과에 미치는 직접 인과 효과 측정 (인과 공정성 프레임)

### 주요 도구

| 도구 | 특징 |
|:---|:---|
| **Microsoft DoWhy** | 4단계 파이프라인, Python, 반박 자동화 |
| **EconML (Microsoft)** | DML, 이질적 처리 효과 추정 |
| **CausalML (Uber)** | 마케팅 업리프트 모델링 특화 |
| **dagitty** | R/브라우저, DAG 시각화·식별 분석 |

### 기술사 체크리스트

1. 교란변수를 도메인 지식 기반으로 DAG에 모두 포함했는가?
2. 병렬 추세 가정 (DID), 도구변수 외생성 등 방법별 전제를 검토했는가?
3. DoWhy의 반박 테스트가 모두 통과됐는가?
4. 추정된 인과 효과의 신뢰구간이 비즈니스 결정을 지지하기에 충분히 좁은가?
5. 이질적 처리 효과 (HTE, Heterogeneous Treatment Effect): 사용자 세그먼트마다 효과가 다를 수 있음 → EconML 활용

- **📢 섹션 요약 비유**: 인과 추론에서 교란변수를 빠뜨리는 것은 법정에서 중요한 증거를 숨기는 것과 같다. 누락된 교란변수가 있으면 아무리 정교한 통계도 편향된 결론을 낸다.

## Ⅴ. 기대효과 및 결론

| 효과 | 내용 |
|:---|:---|
| 정책 결정 신뢰성 | "이 정책이 효과 있는가"를 데이터로 증명 |
| 마케팅 낭비 제거 | 실제 효과 있는 채널에만 예산 집중 |
| 의료 결과 개선 | 관찰 연구로 최적 처방 프로토콜 도출 |
| 알고리즘 공정성 | 인과 경로 기반으로 차별 요인 분리 |
| AI 신뢰성 향상 | 예측을 넘어 설명 가능한 인과적 AI |

인과 추론은 머신러닝이 "예측"에서 "결정"으로 진화하는 핵심 기술이다. 관찰 데이터만으로 RCT 수준의 인과 효과를 추정하는 능력은 수조 원 규모의 공공 정책·의료·마케팅 의사결정의 품질을 혁신적으로 높인다. Pearl의 인과 혁명 (Causal Revolution)과 빅데이터의 결합은 AI의 다음 단계를 열고 있다.

- **📢 섹션 요약 비유**: 인과 추론은 데이터 과학의 탐정 기술이다. 상관관계라는 단서에서 진짜 범인(원인)을 찾아내고, 알리바이(교란변수)를 하나씩 배제하여 진실에 도달한다.

### 👶 어린이를 위한 3줄 비유 설명
- 인과 추론은 "A 때문에 B가 생겼다"를 데이터로 증명하는 거예요. 단순히 "A와 B가 함께 일어났다"는 것과 달라요.
- 아이스크림이 많이 팔리는 날 수영장 사고도 많은 건 "더운 날씨" 때문이지, 아이스크림이 사고를 일으키는 게 아니에요.
- DoWhy는 데이터에서 진짜 원인을 찾고, 그 결론이 믿을 수 있는지 자동으로 검증해주는 도구예요!
