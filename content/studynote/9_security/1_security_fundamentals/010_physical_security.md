# 물리적 보안 (Physical Security)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리적 보안은 정보시스템, 네트워크 장비, 데이터센터 등을 자연재해, 절도, 파괴,Unauthorized 접근 등의 물리적 위협으로부터 보호하는 보안 분야다. 기술적 보안이 아무리 강해도 물리적 보안이 부실하면 전체 보안 체계가 無力化된다.
> 2. **가치**: 데이터센터 정전, 장비 절도,肩typoing 등의 물리적 보안 사고는 서비스 중단, 데이터 유출, 규제 위반 등의严重影响을 초래한다. 적절한 물리적 보안은이러한 위험을 방지하고 비즈니스 연속성을 보장한다.
> 3. **융합**: 물리적 보안은 네트워크 보안, 인사보안과 통합적으로 운영되어야 하며, 통합 보안 관리 (ISM: Integrated Security Management)로 발전하고 있다.

---

## Ⅰ. 개요 및 필요성

### 개념 정의

물리적 보안은 시설, 장비, 사람, 정보를 물리적 위협으로부터 보호하기 위한 조치와 시스템을 말한다. 여기에는 건물 및 시설의 접근 통제, 환경 통제 (화재, 홍수, 지진 등), 장비 보안, inúmer監視 시스템,保安 patrol 등이 포함된다. 물리적 보안은 정보보안의 가장 기반이 되는 层으로, 아무리 정교한 암호화나 네트워크 방화벽도 공격자가 물리적으로 서버에 접근하면無意味해진다.

### 필요성

물리적 보안의 중요성은 간과되기 쉽다. 많은 조직이 네트워크 보안과 정보보안에 많은 예산을 투자하지만, 물리적 보안에는 충분한注意를 기울이지 않는다. 그러나 실제 보안 사고의 상당수는 물리적 접근에서 비롯된다. 서버실에Unauthorized 접근하여 직접 데이터를 탈취하거나, 취약한 무선 네트워크에 접근하거나, 방치된 USB 드라이브를 떨어뜨리거나 하는 공격이 있다.

### 비유

물리적 보안은 집의 walls, 문, 창문, 자물쇠와 같다. 어떤 보안을 갖추든 집에 들어올 수 있으면 안에서 무엇이든 할 수 있듯이, 물리적 보안이突破了면 기술적 보안은 무력화된다. 현관문을 잠그더라도 창문을 열면 들어올 수 있듯이, 물리적 보안도 다층 방어 (Defense in Depth) 원칙을 적용해야 한다.

### 등장 배경

물리적 보안의 개념은 컴퓨터 시대 이전부터 존재했다. 군사 시설과 은행 금고가 그最初的 적용事例였다. 컴퓨터 시대 이후에는 데이터센터, 서버실, 통신 Closet 등의 정보基础设施가 물리적 보안의 대상이 되었다. 오늘날에는 biometric 접근 통제, 원격 감시, AI 기반 이상 상황 탐지 등 기술이 발전하여高度화된 물리적 보안이 가능해졌다.

**[다이어그램 해설]** 물리적 보안도 네트워크 보안과 마찬가지로 다층 방어 (Defense in Depth) 원칙을 적용해야 한다. 주변 보안을突破해도 건물 진입이 어려워야 하고, 건물 진입을突破해도 서버실 접근이 어려워야 하며, 서버실 진입을突破해도 장비에 직접 접근하기 어려워야 한다. 각 층마다 서로 다른 종류의 장애물을 배치하여, 공격자가 모든 층을 연속으로突破하기经济적·시간적으로 불가능하도록 해야 한다.

## Ⅱ. 아키텍처 및 핵심 원리

### 접근 통제 시스템

물리적 접근 통제 시스템은 卡키 (Proximity Card, Smart Card), PIN (Personal Identification Number), 生物 인식 (Fingerprint, Face Recognition, Iris Recognition) 등을 利用한다. 이들 방식은単独으로 사용되기보다 조합하여 다단계 인증으로 사용되는 경우가 많다.

| 방식 | 작동 원리 | 장점 | 단점 |
|:---|:---|:---|:---|
| **카드 (Proximity/Smart Card)** | RFID 또는接触式芯片으로 신분 확인 | 사용 편의성, 발급/회수 용이 | 분실/도용 위험 |
| **PIN** | 숫자 코드로 신분 확인 | 비용 低, 변경 용이 | 除き見知り,推測可能 |
| **지문 (Fingerprint)** | 지문 패턴 매칭 | 높은 정확도, 분실 위험 없음 | 위조 가능성 (만들어진 지문) |
| **얼굴 인식 (Face Recognition)** | 얼굴 특징 매칭 | non-contact, 사용 편의 |照明 변화에 민감 |
| **홍채 인식 (Iris Recognition)** | 홍채 패턴 매칭 |最高 정확도, 평생 불변 | 고비용, 사용자抵觸感 |

### 환경 통제 및 재해 복구

물리적 보안은 위협으로 자연재해와 환경적 위협도 포함한다. 이에 대한 방어措施에는**환경 제어** (항온항습, 흡연 탐지, 수분 탐지),**화재 예방 및 진압** (스프링클러, 이산화탄소 소화 시스템,火灾报警器),**UPS 및 발전기** (정전 대비),**홍수 방지** (배수 펌프, 위치 선정) 등이 있다.

## Ⅲ. 융합 비교 및 다각도 분석

### 물리적 보안 vs 논리적 보안

| 구분 | 물리적 보안 | 논리적 (정보) 보안 |
|:---|:---|:---|
| **보호 대상** | 시설, 장비, 사람 | 데이터, 시스템, 네트워크 |
| **보호 수단** | 벽, 자물쇠, 경비원, CCTV | 암호화, 방화벽, 접근 통제 |
| **위협 유형** | 절도, 파괴, 자연재해,Unauthorized 접근 | 해킹, 맬웨어, 데이터 유출 |
| **취약점** | 부적절한 접근 통제, 감시 사각지대 | 소프트웨어 취약점, 설정 오류 |

### 과목 융합 관점

- **건축공학**: 데이터센터 설계 시 Tier 등급, 냉각 효율, 전력 공급体系등을 함께 고려해야 하며,物理적 인프라와 IT 인프라의共同 설계가 필요하다.
- **소방안전**: 데이터센터의 화재 감지 및 진압 체계는消防法과 규제의 적용을 받으며, 적절한 소화 방법 (물 vs. 가스) 선택이 중요하다.
- **네트워크 보안**: 물리적 네트워크 segment와 논리적 VLAN의 매핑, 물리적 포트와 논리적 접근 통제의 연계가 필요하다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 가상화 환경의 물리적 보안**: 서버가 가상화되어 물리적 서버가 적어졌，但仍需旧дата센터 수준의 물리적 보안을 유지해야 하는 상황. 아키텍트는 하이퍼스케일러의 공유 모델을 활용하여物理적 보안은 공급자에게 위탁하고, 논리적 격리에 집중하는 전략을 세운다. 이 경우 공급자의 물리적 보안 인증 (SOC 2, ISO 27001 등)을 확인하고, 자신의 데이터가 위치하는 물리적 영역에 대한 이해관계자 클로즈를 확인해야 한다.

2. **시나리오 — 원격 근무 시대의 물리적 보안**: 직원이 재택근무에서 업무용 노트북을 사용하면서 물리적 보안 위험이 증가했다. 아키텍트는Endpoint 보안 정책에 장치 분실/도난 대응 절차 (원격 잠금, 데이터 초기화) 를 포함하고, 직원에게 장치 보안 강령 (항상 잠금 상태, 주변 노출 방지) 을 교육해야 한다.

### 도입 체크리스트

- **기술적**: 데이터센터/서버실에 다단계 접근 통제가구비되어 있는가? CCTV 및 감시 시스템이24시간 운영되고 있는가?
- **운영·보안적**: 정기적인 물리적 보안 감사 (담장, 문, 잠금장치 등)가 수행되고 있는가? 방문자 출입 기록이 관리되고 있는가?

### 안티패턴

- **가장 안쪽만 철옹**: 서버실 보안을 강화하지만 건물 주변과 건물 자체는 쉽게 진입 가능한 경우. 다층 방비의 원칙을 어긴다.
- **물리적 보안 과소평가**: "우리 시스템은 인터넷에 연결되어 있으므로 물리적 접근은 불필요"라는 잘못된 생각. 인터넷 연결 없이도 내부자 접근, USB 드라이브 등을 통한 공격이 가능하다.
- **외관만 신경**: CCTV 카메라를 설치하지만 녹화 내용을 확인하지 않거나, footage 저장 기간이 짧아 증거 확보가 불가능한 경우.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 물리적 보안 부재 시 | 체계적 물리적 보안 적용 시 | 개선 효과 |

### 미래 전망

- **AI 기반 물리적 보안**: AI 비전 기술로 CCTV 영상을 실시간 분석하여 이상 상황 (방역 구역 진입, 역추적 등) 을 자동 탐지하고报警하는 시스템이 발전하고 있다.
- **통합 보안 플랫폼**: 물리적 보안 (접근 통제, CCTV) 와 논리적 보안 (SIEM, IAM) 을 단일 플랫폼에서 통합 관리하는趋势이 강화되고 있다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **다층 방어 (Defense in Depth)** | 물리적 보안에서도 네트워크 보안과 마찬가지로 다층 방어 원칙을 적용하여, 한 층突破더라도 다음 층으로 보호하는 전략이다. |
| **데이터센터 TIER** | 데이터센터의 가용성 등급 (Tier I~IV)을 정의하는 표준으로, 물리적 인프라의冗長性과 자동화 수준을 규정한다. |
| **물리적 접근 통제 (PAC)** | 생체 인식, 카드, PIN 등을利用하여 시설과 장비에 대한 물리적 접근을 통제하는 시스템이다. |
| **환경 통제** | 데이터센터의 전력, 냉각, 습도 등을 관리하여 장비가 최적의 환경에서 운영되도록保障하는 시스템이다. |
| **CPTED** | 범죄 예방Through environmental design으로, 설계 단계에서 자연적 surveillance와 접근 통제를 고려하는 철학이다. |

## 👶 어린이를 위한 3줄 비유 설명

1. 물리적 보안은**우리 집의 문과 창문, 자물쇠**와 같아. 아무리 좋은 경비 시스템을 가져도 도둑이 쉽게 들어올 수 있으면 무용지물이야.
2. 컴퓨터실 (서버실) 은特別히 중요한 곳이니까**복도, 문, 자물쇠, 카드키, 지문인식**까지 여러 겹으로 지키는 것이야.
3. 그리고 정전이나 화재에도 컴퓨터가 멈추지 않도록**UPS (대용량 배터리)와 발전기**까지 갖추고 있는 거야. 그래서 우리의 정보가 항상 안전하게 보호되는 거야!
