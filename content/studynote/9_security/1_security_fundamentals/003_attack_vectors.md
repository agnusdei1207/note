# 공격 벡터 (Attack Vector)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공격 벡터는 공격자가 정보시스템에 침입하거나 공격을 성공시키기 위해 利用하는 경로, 수단, 방법을 통칭한다. 네트워크, 애플리케이션, 인간 등 다양한层面에서 multiple 공격 벡터가 존재한다.
> 2. **가치**: 공격 벡터를 이해하는 것은 방어 전략의基础이다. 각 공격 벡터의 특징을 알아야 어떤 방어 기술을 어디에 배치해야 하는지 판단할 수 있다.
> 3. **융합**: 단일 공격은越来越少하고, 여러 공격 벡터르 함께 利用하는 다단계 공격 (Multi-Stage Attack)이主流가 되었다. 따라서 방어도 다층 방어 (Defense in Depth)가 필수적이다.

---

## Ⅰ. 개요 및 필요성

### 개념 정의

공격 벡터는 공격자가 타겟 시스템의 취약점에 접근하여 무단 접근, 데이터 훼손, 서비스 중단 등의 악의적 목적을 달성하기 위해 취하는 행동 경로를 말한다. 이는 네트워크를 통한 원격 침입,USB 같은 물리적 매체, 사회공학 (Social Engineering)을 통한 인간의 심리적 약점 利用 등 다양한 형태로 존재한다.

### 필요성

방어자는 시스템 전체를防守해야 하지만, 공격자는 단 하나라도 성공적인 공격 벡터만 발견하면 된다. 이러한 비대칭성으로 인해 보안 담당자는 가능한 공격 벡터全景을 이해하고, 각 벡터에 대한防御 체계를構築해야 한다. 대표적인 공격 벡터인 Phishing, Malware, Exploit, MITM (Man-in-the-Middle) 공격 등의 작전 방식을 알아야 효과적인 방어가 가능하다.

### 비유

공격 벡터는 집에 들어오려는 도적의 진입로와 같다. 문을 통한 진입, 창문을 통한 진입, 지붕을 통한 진입, 심지어 집안에 살고 있는 사람을 속여 열쇠를 받아내는 방법까지 다양하다. 도난 방지 시스템은 모든 가능한 진입로를カバー해야 하며, 하나라도空隙이 있으면 집안의貴重物品이 위험에 처한다.

### 등장 배경

공격 벡터의 유형은 컴퓨터 기술의 발전과 함께 진화해왔다. 1990년대는 주로 버퍼 오버플로우 같은 소프트웨어 취약점을 利用하는 직접적인 코드 실행 공격이主流였고, 2000년대가되면서 스팸 이메일과 웹사이트를 통한 피싱과 맬웨어 배포가 증가했다. 2010년대 이후에는 APT (Advanced Persistent Threat)와 랜섬웨어가 증가하며, 인간의 심리를 利用하는 사회공학과 다단계 공격이 mainstream이 되었다.

```text

  │              주요 공격 벡터 유형 분류                          │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │  ┌────────────────────────────────────────────────────────┐ │
  │  │                   공격 벡터 카테고리                     │ │
  │  ├────────────────────────────────────────────────────────┤ │
  │  │                                                        │ │

  │  │  ├── MITM (Man-in-the-Middle) 공격                    │ │
  │  │  └── 패킷 스니핑, 스푸핑                                │ │
  │  │                                                        │ │

  │  │  ├── 크로스 사이트 스크립팅 (XSS)                       │ │
  │  │  ├── CSRF (Cross-Site Request Forgery)                │ │
  │  │  └── 세션 탈취, 세션 고정                               │ │

  │  │  ├── 발리-whaling (Whaling)                            │ │
  │  │  └── 프리텍싱 (Pretexting), 베이팅 (Baiting)            │ │
  │  │                                                        │ │
  │  │  물리적 공격 벡터                                       │ │
  │  │  ├── USB 드라이브Autorun 악성코드                      │ │
  │  │  ├── 네트워크jack,Eavesdropping                         │ │
  │  │  └── 물리적 접근 후 랩톱 탈취,肩typoing                │ │

  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 공격 벡터는 크게 네트워크, 애플리케이션, 인간, 물리적 4개 영역으로 분류된다. 네트워크 공격 벡터는 인터넷을 통한 원격 침투이고, 애플리케이션 공격 벡터는 웹 애플리케이션의 취약점을 利用한다. 인간 공격 벡터는 기술적 취약점이 아니라 사람의 심리적弱点을 공격하는 것으로, 기술적 방어로는完全 방어가 어렵다. 물리적 공격 벡터는 장치에 직접 접근하여 데이터를 훔치거나 백도어를 설치하는 것이다. 실제 공격에서는 이 여러 벡터르 조합하여 다단계 공격을 수행하는 경우가 많다.

## Ⅱ. 아키텍처 및 핵심 원리

### 네트워크 공격 벡터의 메커니즘

네트워크 공격 벡터는 네트워크 프로토콜의 취약점이나 네트워크 구성의 미흡을 利用한다. **패킷 스니핑 (Packet Sniffing)**은 네트워크상을流れる 패킷을 도청하는 기술로, 암호화되지 않은 평문 데이터 (FTP, HTTP 등)를 쉽게 탈취할 수 있다. **스푸핑 (Spoofing)**은 발신지 IP 주소를 위조하여 신뢰받는 시스템인 척 가장하는 것이다.

**MITM (Man-in-the-Middle) 공격**은 통신 당사자 사이에서 공격자가 끼어들어 양측의 통신을 탈취하거나 변조하는 공격이다. 공격자는 DNS 스푸핑이나 ARP poisoning 등을 통해 트래픽을 자신의 장비로 우회시킨 후, 통신 내용을 복호화하거나 변경하여 다시 전달한다.

```text
  ┌──────────────────────────────────────────────────────────────┐
  │              MITM (Man-in-the-Middle) 공격 구조              │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │  정상 통신:                                                   │
  │  [클라이언트] ◀══════════════════════════════▶ [서버]       │
  │                    암호화된 직접 통신                          │
  │                                                              │
  │  MITM 공격:                                                   │
  │  [클라이언트] ────▶ [공격자] ────▶ [서버]                    │
  │         │           │           │                            │
  │         │      트래픽 가로채기    │                            │
  │         │      복호화/변조       │                            │
  │         │      재암호화 전달     │                            │
  │                                                              │
  │  ┌──────────────────────────────────────────────────────────┐│
  │  │                    공격 절차                              ││
  │  │                                                          ││
  │  │  1. ARP Poisoning:                                        ││
  │  │     공격자가Gateway IP에 자신의 MAC 주소를 mapping하여     ││
  │  │     클라이언트의 Default Gateway 역할을 横領             ││
  │  │                                                          ││
  │  │  2. SSL Stripping:                                        ││
  │  │     HTTPS 연결을 HTTP로 downgrading하여                   ││
  │  │     암호화 트래픽을 평문으로 탈취                          ││
  │  │                                                          ││
  │  │  3. 세션 가로채기:                                        ││
  │  │     평문 쿠키 또는 인증 토큰을 탈취하여                    ││
  │  │     정당한 사용자로 가장                                     ││
  │  └──────────────────────────────────────────────────────────┘│
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** MITM 공격의 핵심은 통신 트래픽을 공격자를 통해 우회시키는 것이다. ARP Poisoning은同一 네트워크 내에서 가장 흔한 기법으로, 공격자가 가짜 ARP 응답을 보내 자신의 MAC 주소를 Gateway IP에 연결한다. 클라이언트는 의식하지 않고 공격자를 통해 통신하게 되고, 공격자는 트래픽을 탈취하거나 변조할 수 있다. SSL/TLS 암호화로 보호되지 않는 통신은 이런 공격에 극도로脆弱하다. 방어 방법으로는 HTTPS强制 사용, HSTS (HTTP Strict Transport Security), 공개 Wi-Fi 사용 자제가 있다.

### 애플리케이션 공격 벡터

웹 애플리케이션의 대표적인 공격 벡터인 **SQL 삽입 (SQL Injection)**은 사용자가 입력한 데이터를 필터링 없이 SQL 쿼리에 직접 삽입하여, 데이터베이스를 비정상적으로 조작하는 공격이다. 관리자 패스워드 우회, 데이터 탈취, 테이블 삭제 등이 가능하다.

**XSS (Cross-Site Scripting)**는 웹 애플리케이션이 사용자 입력을 적절히 검증하지 않고 출력할 때, 악의적인 스크립트가其他 사용자의 브라우저에서 실행되는 공격이다. 세션 쿠키 탈취, 키로거実行, 페이지 변조 등의危害을 줄 수 있다.

```text
  ┌──────────────────────────────────────────────────────────────┐
  │              SQL 삽입 공격 메커니즘                            │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │  정상 쿼리:                                                  │
  │  SELECT * FROM users WHERE id='user' AND pw='pass'           │
  │                                                              │
  │  SQL 삽입 입력 (비밀번호 필드에):                            │
  │  ' OR '1'='1                                                 │
  │                                                              │
  │  결과 쿼리:                                                  │
  │  SELECT * FROM users WHERE id='user' AND pw='' OR '1'='1'   │
  │                                                              │
  │  ┌──────────────────────────────────────────────────────────┐│
  │  │                   SQL 삽입 유형                          ││
  │  │                                                          ││
  │  │  1. In-Band: 결과在同一 채널로 반환                       ││
  │  │     - Error-Based: DB 오류 메시지에서 정보 추출          ││
  │  │     - Union-Based: UNION으로 데이터 끌어올리기            ││
  │  │                                                          ││
  │  │  2. Inferential (Blind): 응답 차이로 정보 추론           ││
  │  │     - Boolean-Based: 참/거짓 응답으로Database 구조 파악   ││
  │  │     - Time-Based: SLEEP() 함수로 응답 지연로 추론        ││
  │  │                                                          ││
  │  │  3. Out-of-Band: 다른 채널로 데이터 획득                  ││
  │  │     - DNS 채널: DNS 요청으로 데이터 외부 전송            ││
  │  └──────────────────────────────────────────────────────────┘│
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** SQL 삽입은 OWASP (Open Web Application Security Project) Top 10에서 항상 상위를 차지하는 대표적인 웹 애플리케이션 공격이다. 기본 원리는非常简单하지만, 방어 부족 시严重影响을 줄 수 있다. 가장 효과적인 방어는 Prepared Statement (파라미터화된 쿼리)를 사용하여 사용자 입력이 쿼리 구조를 변경할 수 없도록 하는 것이다. 추가로 입력 검증 (Input Validation), 최소 권한原则 ( Principle of Least Privilege) 적용, 에러 메시지 숨기기 등이 defense in depth 전략이 된다.

### 사회공학 공격 벡터

기술적 취약점이 아니라 인간의 심리적弱点을 공격하는 것이 사회공학이다. **피싱 (Phishing)**은 가장 흔한 형태로, 신뢰받는 기관을 사칭한 가짜 이메일이나 웹사이트로 사용자를 현혹하여 비밀번호나 금융 정보를 탈취한다. **스피어 피싱 (Spear Phishing)**은 특정 개인이나 조직을 대상으로精心하게 커스터마이즈된 피싱이다.

**발리-웨딩 (Whaling)**은 기업 임원나 정부 고위관을 대상으로 한高级 피싱으로, 더욱 정교한 위장이 필요하다. **프리텍싱 (Pretexting)**은 특정 상황을設定하여 상대방의警戒심을 풀고 정보를 제공하게 만드는 기법이다.

```text
  ┌──────────────────────────────────────────────────────────────┐
  │              사회공학 공격 벡터 — 피싱 워크플로우              │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │  [타겟 조사] ──▶ [攻击策略 수립] ──▶ [위장 이메일 발송]        │
  │      │               │                  │                    │
  │      ▼               ▼                  ▼                    │
  │  ┌──────────┐  ┌──────────┐     ┌──────────┐               │
  │  │ OSINT    │  │ 본뜬      │     │ 가짜 링크 │               │
  │  │ 활용     │  │ 웹사이트  │     │ 배포     │               │
  │  │          │  │ 준비     │     │          │               │
  │  └──────────┘  └──────────┘     └──────────┘               │
  │                                          │                    │
  │                                          ▼                    │
  │                              ┌───────────────────┐            │
  │                              │ 사용자 착각诱导    │            │
  │                              │                    │            │
  │                              │ -紧急感 조성      │            │
  │                              │ -恐惧심리 利用     │            │
  │                              │ -권한 위장        │            │
  │                              └───────────────────┘            │
  │                                          │                    │
  │                                          ▼                    │
  │                              ┌───────────────────┐            │
  │                              │ 정보 탈취 / 악성코드 │            │
  │                              │ 설치              │            │
  │                              └───────────────────┘            │
  │                                                              │
  │  방어 요소:                                                   │
  │  ├─ 이메일 필터링 (스팸 감지)                                 │
  │  ├─ 링크 호버 시 실제 URL 확인                               │
  │  ├─ 다단계 인증 (MFA)                                        │
  │  └─ 정기적인 보안 인식 교육                                   │
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 사회공학 공격의 공통점은 기술적 방어벽이 아니라 인간의 심리적 반응을利用한다는 것이다. 공격자는 긴급성,恐惧, 호기심 등의 감정을刺激하여 피해자가理性的 판단을 하지 못하게 만든다. 피싱 이메일은越来越高 정교해져서 실제 이메일과 거의 구별되지 않는 수준이다. 따라서 가장 효과적인 방어는 기술적 방어와 함께 인간을 대상으로 한 보안 인식 교육이다. 정기적인 피싱 시뮬레이션으로员工的 보안 감수를 높이고,可疑한 이메일은.report하는 문화를 구축해야 한다.

## Ⅲ. 융합 비교 및 다각도 분석

### 주요 공격 벡터 비교

| 공격 벡터 | 공격 난이도 | 탐지 난이도 |主な 대상 | 방어 방법 |
|:---|:---|:---|:---|:---|
| **Phishing** | 낮음 | 중간 | 인간 (End User) | 이메일 필터, 교육, MFA |
| **Ransomware** | 중간 | 낮음 | 시스템, 데이터 | 백업, endpoint 보안, 네트워크 분할 |
| **SQL Injection** | 중간 | 낮음 | 웹 애플리케이션 | Prepared Statement, 입력 검증 |
| **Zero-Day** | 높음 | 매우 높음 | 소프트웨어 | 방화벽, IDS, 네트워크 분할 |
| **Insider Threat** | 낮음 | 매우 높음 | 조직 내부 | 접근 통제, 감사 로깅, DLP |
| **DDoS** | 낮음~중간 | 낮음 | 가용성 | CDN, DDoS 방어 서비스, Anycast |

**[다이어그램 해설]** 공격 벡터의 진화는 기술 환경의 변화와 밀접하다. 1990년대는 개인 컴퓨터와 인터넷의初期로, 기술적 취약점을 直接 利用하는 공격이主流였다. 2000년대에 웹 애플리케이션이 확산되면서 웹 취약점이 주요 공격 벡터가 되었다. 2010년대는 APT와 랜섬웨어가 정점을 찍으며 경제적 이익을 추구하는 공격이 증가했다. 2020년대 이후에는 AI와 딥페이크가 등장하여 피싱 이메일의質이飞跃的に 향상되고, 취약점 발견과 공격 자동화에도 AI가 利用되고 있다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 랜섬웨어 다단계 공격 대응**: 공격자가 피싱 이메일로社内 침투한 후, 내부 네트워크를Laterally Movement하며 취약한 서버를 찾는다. затем 백도어를 설치하고, 결국 랜섬웨어를 배포하여 모든 데이터를 암호화한다. 아키텍트는 네트워크 분할 (Network Segmentation)로 내부 서버 간 접근을制限하고, EDR (Endpoint Detection and Response)으로 의심스러운 행위를탐지하며, 중요한 데이터의 오프라인 백업을 유지해야 한다.

2. **시나리오 —供应链 공격 (Supply Chain Attack)**: 널리 사용되는오픈소스 라이브러리에 악성코드가 포함되어, 해당 라이브러리를 사용하는수천 개의 애플리케이션이 한꺼번에 감염되는 경우다. 아키텍트는 소프트웨어 구성 분석 (SCA: Software Composition Analysis) 도구를 도입하여 사용하는 라이브러리의 취약점과 무결성을 지속적으로 모니터링하고, 신뢰된 출처에서만 라이브러리를 다운로드하는 정책을 수립해야 한다.

```text
  ┌──────────────────────────────────────────────────────────────┐
  │              다단계 공격 (Kill Chain) 단계별 방어 전략         │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │  공격 단계              방어 전략                             │
  │  ─────────              ──────────                            │
  │  1. 정찰 (Recon)        │威胁 인텔리전스, perimeter 강화       │
  │       │                 │                                     │
  │       ▼                 ▼                                     │
  │  2. 무기화 (Weaponize)  │漏洞 관리, 패치 적용                  │
  │       │                 │                                     │
  │       ▼                 ▼                                     │
  │  3. 전달 (Deliver)      │이메일/웹 필터링, sandboxing          │
  │       │                 │                                     │
  │       ▼                 ▼                                     │
  │  4. 악용 (Exploit)      │EDR, IDS/IPS, WAF                    │
  │       │                 │                                     │
  │       ▼                 ▼                                     │
  │  5. 설치 (Install)      │엔드포인트 보호, whitelisting         │
  │       │                 │                                     │
  │       ▼                 ▼                                     │
  │  6. C2 (Command & Control)│네트워크 모니터링, DNS 필터링        │
  │       │                 │                                     │
  │       ▼                 ▼                                     │
  │  7. 행동 (Actions)      │데이터 손실 방지, incident response   │
  │                                                              │
  │  각 단계에서 공격을 차단하면 전체 공격 실패                    │
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** Lockheed Martin이定義한 Kill Chain 모델은 APT 공격의 단계를体系적으로 분석한다. 공격자 입장에서는最初の 단계가 성공해야后续 단계로 진행할 수 있으므로, 방어자 입장에서는 어느 단계에서든 공격을 차단하면 된다. 그러나 후반 단계로 갈수록 탐지하기 어려워지므로,前端에서 최대한 많은 공격을 차단하는 것이 효율적이다. 각 단계에 대응하는 보안 기술을 배치하고, 서로 연결하여 defense in depth를 실현해야 한다.

### 도입 체크리스트

- **기술적**: 네트워크 방화벽과 IDS/IPS가すべての 네트워크_edge에 배치되어 있는가?Endpoint 보안 솔루션 (EDR)이 설치되어 있는가? 취약점 스캐닝이 정기적으로 수행되고 있는가?
- **운영·보안적**:员工 대상 피싱 시뮬레이션과 보안 인식 교육이 정기적으로 실시되고 있는가? Incident Response 계획에 다양한 공격 벡터 시나리오가 반영되어 있는가?

### 안티패턴

- **단일 방어점 의존**: 방화벽 하나만으로 충분한 보안이 가능하다고 믿는 것. 방화벽을突破하면 내부에 아무런 방어 수단이 없다.
- **사용자 교육 무시**: 기술적 방어에만 투자하고 인간을 통한 공격 벡터에 대한 employee 교육은 소홀히 하는 것. 피싱邮件は技術的 방어로는完全 차단이 어렵다.
- **지속적 모니터링 부재**: 보안 장비는 설치하지만 지속적인 로그 분석과 모니터링을 수행하지 않는 것. 보안 장비의 알람을 누계된 채로 두면 의미가 없다.

## 👶 어린이를 위한 3줄 비유 설명
