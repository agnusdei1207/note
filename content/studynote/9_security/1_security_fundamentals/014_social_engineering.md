# 사회공학 (Social Engineering)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사회공학은 기술적 취약점이 아니라 인간의 심리적 약점 (신뢰, 긴급성, 순종 심리 등) 을 利用하여 정보나 접근 권한을 훔치는 공격 기법이다. 기술적 방어벽이 아무리 강해도 인간이突破하면 의미가 없다.
> 2. **가치**: Verizon DBIR에 따르면 보안 사고의 82% 이상에서人的要素가 관여하며, 사회공학은APT, 피싱, 랜섬웨어 등 다양한 공격의 초기 침투 수단으로 사용된다.
> 3. **융합**: 사회공학 방어는 기술적 방어 (이메일 필터링) 와 인간적 방어 (인식 교육) 의 조합으로만 가능하다. 피싱 시뮬레이션과 보안 인식 프로그램이 핵심 방어 수단이다.

---

## Ⅰ. 개요 및 필요성

### 개념 정의

사회공학은 공격자가 표적의 심리적 취약점을 利用하여 비합법적으로 정보나 시스템 접근 권한을 획득하는 공격 기법이다. 전통적 해킹이 기술적 취약점을 찾는 반면, 사회공학은 "인간이라는 취약점"을 공격한다.攻撃者は心理学, 언어학, 사회적-engineering 기법을 적용하여被害者を操纵한다.

### 필요성

기술적 보안은 빠르게 발전하고 있지만, 인간의 심리적弱点は 수천 년 동안 변하지 않았다.authority에 대한 순종, 긴급 상황에서의冲動的 판단, 타인에 대한 신뢰 등은 인간의 본능적 반응이며, 공격자는 이를 Ruth없이 利用한다. 따라서 기술적 방어와 함께 인간을 통한 방어가 필수적이다.

### 비유

사회공학은**변장 사기꾼**의 기술과 같다. 은행 직원을 사칭하여 전화로 비밀번호를 알려달라고 하거나, 택배 기사를 가장하여 아파트에 진입하는 것처럼, 기술이 아닌 "사람을 속이는" 기법이다. 아무리 좋은 자물쇠을 갖추고 있어도, 열쇠를 가진 사람을 속이면 문을 열 수 있듯이.

### 등장 배경

사회공학의 개념은 Kevin Mitnick의 활동으로 대중적으로 알려졌다. Mitnick은 1970~1990년대에 수십 개의 기업과 조직에 전화와 사회적 위장을 利用하여 침투했다. 이후 사회공학은 사이버 범죄의 핵심 도구로 발전했으며, 피싱, 스피어 피싱, 발리 등 다양한 형태로 분화되었다.

```text
  ┌──────────────────────────────────────────────────────────────┐
  │              사회공학 공격 유형 분류                              │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │  ┌──────────────────────────────────────────────────────────┐│
  │  │                                                          ││
  │  │  [기본 유형]                                              ││
  │  │                                                          ││
  ///  │  Phishing (피싱) ──▶ 불특정 다수 대상, 보편적 사기      ││
  ///  │  Spear Phishing ──▶ 특정 개인/조직 대상, 맞춤 공격      ││
  ///  │  Whaling (발리) ──▶ 기업 임원 대상, 고위관 대상        ││
  ///  │  Vishing (보이싱) ──▶ 전화利用, 음성 사기              ││
  ///  │  Smishing (스미싱) ──▶ 문자메시지利用, SMS 사기        ││
  ///  │                                                          ││
  ///  │  [심화 유형]                                              ││
  ///  │                                                          ││
  ///  │  Pretexting ──▶ 특정 상황 설정, 거짓 상황으로 유도      ││
  ///  │  Baiting ──▶ 이익 제공으로 유도, USB 등 물리적 매체   ││
  ///  │  Tailgating ──▶身后跟随, 권한 없는 사람跟随入场        ││
  ///  │  Quid Pro Quo ──▶ 서비스 대가로 정보 요구              ││
  ///  │ watering hole ──▶ 타겟가 자주 방문하는 사이트 감염     ││
  ///  └──────────────────────────────────────────────────────────┘│
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 사회공학은 대상과 수단에 따라 다양하게 분류된다. Phishing은 가장 보편적인 형태로 불특정 다수를 대상으로 하지만, Spear Phishing은 특정 개인이나 조직을 대상으로精心하게 커스터마이즈된다. Whaling은 CEO, CFO 등 기업 임원을 대상으로 하며, 더욱高度화된 정보 수집과 위장이 필요하다. Vishing과 Smishing은 각각 전화와 문자를利用하는 변형이다.

## Ⅱ. 아키텍처 및 핵심 원리

### 사회공학 공격 프레임워크

사회공학 공격은 일반적으로 다음과 같은 프레임워크로 진행된다:

1. **정보 수집 (Information Gathering)**: OSINT (Open Source Intelligence), 소셜 미디어, 회사 website 등 공개 정보 수집
2. **상황 설정 (Pretext Development)**: 공격 대상과 상황에 맞는 믿을 수 있는 시나리오 구축
3. **관계 구축 (Relationship Development)**: 신뢰建立, Rapport 구성
4. **악용 (Exploitation)**: 구축된 관계를 利用하여 정보나 행동 유도
5. **실행 (Execution)**: 획득한 정보나 접근 권한을 利用하여 목표 달성

### 핵심 심리 기법

공격자가 利用하는 주요 심리 기법에는 다음이 포함된다:

```text
  ┌──────────────────────────────────────────────────────────────┐
  │              사회공학에서 利用하는 심리적 취약점                  │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │  1. Authority (권위에의 순종)                                │
  │     └─ "IT 부서에서 보안 업데이트때문에 비밀번호 알려줘야 해"   │
  │     └─上司を名の攻击者がmanager 등을 사칭하는 것이 대표적     │
  │                                                              │
  │  2. Urgency (긴급성)                                         │
  │     └─ "이메일은 오늘까지 확인해야 하고, 그렇지 않으면 계정이 삭제돼요"│
  │     └─ 紧急한 상황을 조성하여对方的批判적 사고를 차단         │
  │                                                              │
  │  3. Social Proof (사회적 증거)                               │
  │     └─ "다른 동료들도 이미 변경했어요"                        │
  │     └─ 他人がすでに従っている情况下では从众心理が働きやすい     │
  │                                                              │
  │  4. Reciprocity (호의의 규범)                                │
  │     └─ "선물을 드리니 도와주세요"                            │
  │     └─ 对方에게 먼저 무언가를 제공하여 은혜를 갚으려는心里を利用│
  │                                                              │
  │  5. Likeness (호감)                                          │
  │     └─ 공통된 관심사, 배경 등을 찾아 친밀감 형성              │
  │     └─  유사성을 찾거나Friendly한 태도로对方의警惕를 풀게함│
  │                                                              │
  │  6. Fear (공포)                                              │
  │     └─ "이 사진을 받았습니다. Bitcoin을不然的话 공개할게요"     │
  │     └─ 恐怖심을 자극하여 理性적 판단을 불가능하게 함           │
  │                                                              │
  │  ※ 이러한 기법들은孤立해서는 작동하지 않으며, 조합되어 사용됨│
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 사회공학 공격의 핵심은 하나의 기법이 아니라 여러 심리 기법의 조합이다. 예를 들어, "IT 부서에서 보낸 것이다" (Authority) + "오늘까지 해야 한다" (Urgency) + "다른 동료들도 했다" (Social Proof)라는 메시지는 세 가지 심리적 pressure를 동시에 가하여,被害者が理性적 판단을 하지 못하고指示に従う可能性을높인다.

## Ⅲ. 융합 비교 및 다각도 분석

### 사회공학 vs 기술적 공격

| 구분 | 사회공학 | 기술적 공격 |
|:---|:---|:---|
| **대상** | 인간 (심리) | 시스템/소프트웨어 |
| **탐지 난이도** | 높음 | 중간 |
| **방어 방법** | 교육/인식 | 기술적 보안 솔루션 |
| **비용** | 매우 낮음 | 높음 |

### 과목 융합 관점

- **심리학**: 사회공학은 심리학의 다양한 이론 (초조함, 인지 편향, 설득 이론 등) 을 利用한다.
- **범죄학**: 사회공학은 사이버 범죄뿐 아니라 오프라인 사기에도 적용되는 일반적 범죄 기법이다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 스피어 피싱 대응**: CFO를 대상으로 한 스피어 피싱 이메일이 발견되었다. 아키니어는 즉시 CFO에게 보고하고, 이메일이 합법적인지 확인하도록 교육한다. 동시에 조직 전체에 해당 시나리오를 공유하여 다른 구성원이 비슷한 이메일을 받았을 때 식별할 수 있도록 한다.

2. **시나리오 — USB 바우팅 공격 대응**: 주차장에 떨어져 있는 USB 드라이브가 발견되었다. 아키니어는 이를 아무도plug in하지 않도록 하고, 발견된 위치에 CCTV 영상을 확인하여攻击자 여부를 분석한다.

### 도입 체크리스트

- **기술적**: 이메일 필터링 및 피싱 탐지 솔루션이 갖추어져 있는가?
- **운영·보안적**: 피싱 시뮬레이션과 보안 인식 교육이 정기적으로 실시되고 있는가?

### 안티패턴

- **기술에만 의존**: 이메일 필터링만으로는 완벽한 차단이 불가능하다.
- **한 번의 교육**: 입사 시 한 번의 교육으로 충분하다고 믿는 것.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 교육 미실시 시 | 체계적 인식 교육 적용 시 | 개선 효과 |
|:---|:---|:---|:---|
| **정량** | 피싱 클릭률 30%+ | 클릭률 5% 이하 | 클릭률 **80% 감소** |
| **정성** | 사고 은폐 가능성 높음 | 보고 문화 조성 | 탐지율 **提升** |

- **AI 생성 피싱**: AI가 개인화된 피싱 이메일을 대규모로 생성하는 것이 가능해지고 있다.
- **딥페이크 음성 사기**: AI 음성 합성 기술로 가족이나上司의 목소리를 가장하여 전화 사기를치는 사례가 증가하고 있다.

## 👶 어린이를 위한 3줄 비유 설명

1. 사회공학은 **"친구가 되고 싶어하는 사기꾼"**과 같아. 낯선 사람이 "나도 너랑 같은 학교 다니는데, 너 번호 알려줄래?" 하고 물어보면 이상하잖아. 그런데 "너랑 같은 반 민수가 같이 가자고 해서" 하면 조금은 믿게 되잖아.
2. 바로 그게 사회공학이야. **"누구세요?"** 에 **"IT부서에서 보냈어요"**, 그리고 **"빨리 해야 해요"** 를 combo로 넣으면 우리脑子는 "뭔가 이상하다"는 생각보다 "따라야지"라는 생각이 먼저 나오는 거야.
3. 그래서 우리 중요한 것은, 아무리 친절해 보여도 **"모르겠으면 항상 부모님이나 선생님께 물어보자"**는 거야. 그것이 바로社会공학から身を守る一番の方法이야!
