# 993. WAF (웹 방화벽) - Web Application Firewall L7 계층 웹 특화 보안 HTTP 트래픽 검사 SQL 인젝션 XSS 차단 암호화 복호화 탐지망 통제 모델

> **핵심 인사이트**: 비싼 돈 주고 샀던 L3/L4 방화벽(992번 SPI)과 탐지기(IDS). 얘네들은 80번(HTTP) 문을 통과하는 패킷은 무조건 합법이라고 생각하고 그냥 다이렉트로 서버까지 쑥쑥 들여보내 준다. 해커는 이 점을 노렸다. 80번 포트를 타고 당당하게 들어와서, 택배 박스(HTTP 본문) 안에 폭탄(SQL 인젝션, 크로스 사이트 스크립팅)을 숨겨서 서버 DB를 싹 다 털어먹었다. "야! 껍데기(IP/포트)만 보지 말고, 택배 상자 안쪽(L7 웹 애플리케이션 계층) 편지 내용을 엑스레이로 뼛속까지 투시해! 그 편지 글씨 안에 `SELECT * FROM` 같은 DB 탈취 마법 주문(SQL 인젝션)이 적혀있으면 그 자리에서 찢어발겨버려!" 웹 애플리케이션 특화 엑스레이 검색대, WAF(웹 방화벽)다.

## Ⅰ. 기존 전통적 방화벽(L3/L4)의 완전한 패배
- 기존 네트워크 방화벽은 우체국 분류원입니다. "도착지 포트가 80번(웹)이네? 정상! 443번이네? 정상!" 
- 웹 서버를 돌리려면 외부에서 손님들이 들어오도록 80번/443번 포트 대문은 무조건 활짝 열어놔야만 합니다. 
- 해커는 해킹 프로그램 포트(22번 등)를 쓰지 않고, 그냥 **평범한 일반인인 척 웹 브라우저를 켜서 80번 포트로 걸어 들어온 뒤, 검색창에 악성 코드(스크립트)를 쓱 적어 넣어서 서버를 파괴합니다(웹 해킹).** 껍데기만 보는 구형 방화벽은 이 웹 해킹을 1%도 탐지하지 못하고 바보가 되었습니다.

## Ⅱ. WAF (Web Application Firewall)의 개념과 등판 🌟
- **개념**: OSI 7계층 중 가장 꼭대기인 **응용 계층(Layer 7)**에서 돌아가는 특수 방화벽입니다. 
- 오직 HTTP와 HTTPS 웹 트래픽의 **'페이로드(본문 내용물)' 전체를 현미경으로 샅샅이 뜯어보고 해독하여, 웹 서버를 노리는 특화된 악성 공격(OWASP Top 10 공격)을 핀셋으로 적발하고 차단하는 웹 전용 엑스레이 방어 시스템**입니다.

## Ⅲ. WAF가 잡아내는 3대 웹 해킹의 주적 (시험 단골) 🌟
WAF는 텍스트(문맥)를 읽을 줄 아는 국어 선생님입니다.

1. **SQL 인젝션 (SQL Injection)**: 
   - 해커가 웹사이트 로그인 창에 아이디 대신 `' OR 1=1 --` 라는 미친 데이터베이스(DB) 파괴 마법 주문을 타이핑해 넣습니다.
   - WAF가 이 HTTP 요청을 뜯어보고 "어? 로그인 창에 왜 SQL 언어가 들어와? DB 털려는 수작이네!" 하고 즉각 차단합니다.
2. **크로스 사이트 스크립팅 (XSS)**: 
   - 해커가 게시판 글쓰기 창에 `<script>해킹코드</script>`를 몰래 적어 넣으려 합니다.
   - WAF가 엑스레이로 "어? 게시판 본문에 자바스크립트 코드가 왜 들어가? 차단!"
3. **웹쉘(Web Shell) 업로드 및 L7 디도스**: 
   - 게시판에 `해킹.jpg.php` 라는 이중 확장자로 바이러스를 올리려는 짓이나, 느릿느릿한 HTTP 요청을 1만 번 보내서 웹서버를 말려 죽이는 Slowloris 공격(L7 DDoS)을 완벽하게 걸러냅니다.

## Ⅳ. HTTPS 암호화 시대, WAF의 진화 (프록시 구조) 🌟
치명적인 문제가 터졌습니다. 요즘 인터넷은 100% **HTTPS(암호화 통신, 982번)**를 씁니다. 해커가 악성코드를 시커먼 암호 캡슐에 넣어서 보내면, WAF도 그 내용을 읽지 못해 장님이 됩니다.

- **해결책 (Reverse Proxy 아키텍처)**: 
  - WAF를 은행 앞의 '대리인(프록시)'으로 세웁니다.
  - 고객(해커)은 WAF와 HTTPS 암호화 통신을 맺습니다. **WAF가 1차로 암호화된 캡슐을 까서(복호화/SSL Offloading) 진짜 쌩얼(텍스트)을 확인합니다.**
  - WAF 뱃속에서 무결성 검사를 마친 깨끗한 패킷만을, 다시 WAF가 뒤쪽에 있는 진짜 웹 서버에게 토스해 줍니다. WAF가 암호 해독 노가다를 대신 다 해주니 뒤쪽의 웹 서버는 CPU 부하가 확 줄어드는 마법(SSL 오프로딩)까지 덤으로 얻게 됩니다.

> 📢 **섹션 요약 비유**: 기존 L3/L4 방화벽은 공항의 '비행기 표 검사원'입니다. 손님이 미국 가는 표(80번 웹 포트)만 있으면 가방 안에 칼이나 폭탄이 들어있는지 확인도 안 하고 무조건 비행기에 태워줘서 테러(웹 해킹)가 터집니다. **WAF(웹 방화벽)**는 그 뒤에 버티고 서 있는 **'초정밀 짐칸 엑스레이 검색대(L7 페이로드 스캔)'**입니다. 표를 보여주고 들어온 손님이라도, WAF는 무조건 가방(HTTP 패킷) 지퍼를 끝까지 열어서 안쪽 내용물을 다 까뒤집어 봅니다. 필통에 볼펜(정상 텍스트)이 있으면 통과시키지만, 화장품 통 안에 몰래 숨겨둔 권총(SQL 인젝션 코드)이나 주머니 속의 수류탄(XSS 스크립트)을 텍스트 엑스레이 스캔으로 귀신같이 찾아내어, 현장에서 테러리스트를 사살하고 은행(웹 서버)을 완벽하게 보호하는 웹 전담 최고존엄 수사관입니다.
