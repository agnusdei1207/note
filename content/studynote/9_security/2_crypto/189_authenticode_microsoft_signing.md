# 189. Microsoft Authenticode (어센티코드)

> ⚠️ 이 문서는 Windows 운영체제 생태계에서 굴러다니는 수백만 개의 윈도우 실행 파일(`.exe`, `.dll`, `.sys`)들이 바이러스가 아님을 입증하기 위해, 마이크로소프트가 독자적으로 설계한 PKCS#7 기반의 코드 서명 기술이자 윈도우 SmartScreen 방어막의 심장인 'Authenticode'를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Authenticode(어센티코드)는 마이크로소프트가 X.509 인증서와 PKCS#7(CMS) 표준을 융합하여 만든 **Windows 전용 코드 서명(Code Signing) 아키텍처**로, 실행 파일의 PE(Portable Executable) 구조 꼬리 부분에 개발자의 디지털 도장(서명)을 밀봉해 넣는 규격이다.
> 2. **가치**: 윈도우 사용자가 인터넷에서 앱을 다운받아 실행할 때 시뻘건 **"게시자를 알 수 없는 앱입니다"라는 공포의 경고창을 없애주고 파란 방패(신뢰받는 게시자)를 띄워주어**, 앱의 무결성(조작 안 됨)과 출처(개발자 신원)를 100% 보증해 주는 유일한 상업적/기술적 여권이다.
> 3. **융합**: 단순한 서명을 넘어, 서명을 찍을 당시의 시간을 영구 박제하는 **타임스탬프(Time Stamping) 서버와 융합**하여, 인증서의 유효기간(1년)이 만료되거나 심지어 취소되더라도 "그 시절엔 정상적으로 만든 프로그램이다"라고 10년 뒤에도 에러 없이 실행시켜 주는 극강의 사용자 경험(UX)을 달성했다.

---

### Ⅰ. 개요 및 왜 '윈도우 전용' 규칙이 필요한가? (Context & Necessity)
우리가 [188번 문서]에서 코드 서명의 위대함을 배웠다. 하지만 추상적인 수학 공식만으로는 윈도우 OS에서 마우스를 더블클릭할 때 어떻게 동작하게 만들지 구현할 수 없다. 

마이크로소프트는 윈도우 실행 파일인 **PE(Portable Executable) 포맷(`.exe`, `.dll`) 구조체 내부의 빈 공간(Data Directory)**에 서명 도장을 예쁘게 박아넣고, 윈도우 커널과 백신(Windows Defender)이 그 도장을 0.1초 만에 파싱(Parsing)해서 검사할 수 있는 구체적이고 현실적인 가이드라인을 세웠다. 이것이 바로 **Authenticode(어센티코드)** 다.

개발자가 Visual Studio 명령 프롬프트에서 `signtool.exe sign /f mycert.pfx mygame.exe` 라는 마법의 명령어 한 줄을 치는 순간, 이 어센티코드 규격에 맞춰 1GB짜리 게임 파일에 넥슨의 도장이 완벽하게 밀봉된다.

> 📢 **섹션 요약 비유**: 코드 서명이라는 게 "소고기에 도장 찍기"라는 개념이라면, Authenticode는 "소고기의 왼쪽 갈비뼈 3번째 마디에 파란색 특수 잉크로 가로 3cm, 세로 3cm 크기로 도장을 꽉 찍어라!" 라고 마이크로소프트(도축장)가 지정해 놓은 아주 구체적이고 완벽한 포장 작업 지시서입니다. 윈도우는 이 규칙대로 찍힌 도장만 돋보기로 들여다보고 통과시켜 줍니다.

### Ⅱ. 어센티코드의 2가지 킬러 기능 (SmartScreen과 타임스탬프)

Authenticode가 평범한 서명과 다른 이유는 윈도우 생태계와 기가 막히게 맞물려 돌아간다는 점이다.

#### 1. UAC와 SmartScreen의 철벽 검문
- 사용자가 인터넷에서 카카오톡 PC 버전을 다운받아 실행한다.
- 윈도우의 UAC(사용자 계정 컨트롤)가 방패 창을 띄운다. 이때 윈도우는 파일 엉덩이에 붙은 Authenticode 서명을 까서 해시 지문을 확인한다.
- **[서명이 진짜일 때]**: 파란색 방패와 함께 "게시자: Kakao Corp"라는 예쁜 글씨가 뜬다. 사용자는 "오 카카오네" 하고 안심하고 [예]를 누른다.
- **[서명이 깨졌거나(바이러스), 아예 없을 때]**: 시뻘건 화면이 화면을 다 덮으며 **"Windows의 PC 보호: 인식할 수 없는 앱의 실행을 차단했습니다."** 라는 무시무시한 경고를 날린다. 웬만한 강심장 고객이 아니고서야 여기서 뒤로 가기를 누르고 앱을 휴지통에 버린다. (상업용 소프트웨어 회사들이 무조건 돈을 주고 어센티코드 인증서를 사야 하는 절대적인 이유다).

#### 2. 시간 여행의 마법: 타임스탬프 (Timestamping)
가장 위대한 아키텍처적 혁신이다.
- 넥슨이 게임을 2020년에 배포했다. 인증서 유효기간은 2021년까지(1년짜리)였다.
- 2025년에 앨리스가 고전 게임이 하고 싶어 그 파일을 다운받아 실행했다.
- 윈도우 OS의 고민: "어? 넥슨 인증서는 2021년에 만료됐는데? 그럼 2025년 지금은 무효(차단) 처리해야 하나?" $\rightarrow$ 이렇게 하면 옛날에 만든 세상 모든 프로그램이 다 막혀버린다!
- **해결책 (타임스탬핑)**: 넥슨 개발자가 2020년에 `signtool`로 도장을 찍을 때, 마이크로소프트의 **타임스탬프 전용 서버(`/tr http://timestamp.digicert.com`)에 파일을 던져서 "내가 2020년 10월 1일에 도장 찍었음!"이라는 시간 증명 도장을 하나 더 받아와서 겹쳐 찍어놨다.**
- **결과**: 2025년에 앨리스가 실행할 때 윈도우는 인증서가 만료되었음에도 불구하고, "아하! 이 프로그램은 인증서가 싱싱하게 살아있던 2020년 10월 당시에 안전하게 만들어진 게 확실하군!" 이라며 100% 무사 통과(실행)를 시켜준다. 시간의 제약을 완벽하게 부숴버린 꼼수다.

```text
┌────────────────────────────────────────────────────────────────────────────────────┐
│           Authenticode의 타임스탬프(Time Stamp) 영구 보존 효과 시각화              │
├────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                    │
│ 💣 [ 타임스탬프 없이 도장 찍은 바보 개발자 ]                                       │
│  2023년 발급받은 인증서 (유효기간: 2024년 12월 31일 만료)                          │
│  2024년 1월 1일: 게임.exe 에 도장 쾅! 배포 완료!                                   │
│                                                                                    │
│  ▶ 2025년에 고객이 실행 ──▶ 윈도우: "인증서 유효기간 지남! 🔴 실행 차단!!"         │
│  (사장님 빡침. 개발자 밤새서 인증서 새로 사서 도장 다시 찍어 패치 배포해야 함 ☠️)  │
│                                                                                    │
│ ────────────────────────────────────────────────────────────                       │
│                                                                                    │
│ 🛡️ [ 타임스탬프 서버 엮어서 도장 찍은 엘리트 개발자 ]                              │
│  2024년 1월 1일: 게임.exe 에 도장 찍을 때, [Time 서버]랑 통신해서                  │
│                 "2024.01.01 정상 시점에 찍음 ⏰" 도장 같이 쾅! 박아버림.           │
│                                                                                    │
│  ▶ 2025년, 2030년에 고객이 실행 ──▶ 윈도우: "인증서는 유통기한 끝났지만,           │
│  저 ⏰ 타임 도장을 보니까 옛날에 정상일 때 만들어진 거네! 영구 통과 🟢!!"          │
│  (개발자는 인증서 만료 신경 안 쓰고 두 다리 뻗고 잔다 🎉)                          │
└────────────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 이 타임스탬프 구조 덕분에, 소프트웨어 회사들은 유효기간 만료 때마다 옛날에 배포한 파일을 일일이 주워 담아 도장을 다시 찍어 배포하는 지옥의 수고로움(Repackaging)에서 완벽하게 해방되었다. 이것은 인증서의 '발급 수명(Validity)'과 소프트웨어의 '생명 주기(Lifecycle)'를 분리(Decoupling)해 낸 우아한 PKI 아키텍처의 표본이다.

### Ⅲ. 실무적 악몽: EV 인증서의 횡포 (SmartScreen 평판)

어센티코드 인증서는 두 종류가 있다. 싼 거(일반 OV, 10만 원)와 비싼 거(EV, 100만 원). [176, 178번 참조]
스타트업 사장님이 돈을 아끼려고 일반 OV 인증서를 사서 앱에 도장을 쾅 찍고 야심 차게 배포했다.
고객이 다운로드하는데, 시뻘건 경고창이 또 뜬다! **"이 앱은 다운로드한 사용자가 많지 않아 PC를 손상시킬 수 있습니다."**

사장님: "아니 나 도장 진짜로 돈 주고 사서 똑바로 찍었는데 왜 또 경고를 띄워 마소 새끼들아!!"

* **SmartScreen 평판(Reputation) 시스템의 폭정**:
  - 마이크로소프트는 일반 도장(OV)을 찍은 앱은 바로 파란 방패를 주지 않는다. 전 세계 사람들이 그 앱을 수만 번 다운로드 받아서 아무 사고가 없다는 게 증명(평판 점수 누적)되어야만 비로소 한 달쯤 뒤에 경고창을 내려준다.
  - 하지만 **EV 인증서 (하드웨어 USB 토큰에 개인키가 들어있고 서류 심사 빡센 100만 원짜리 도장)**로 서명해서 배포하면? **"오! USB 토큰까지 써서 도장 찍은 특A급 회사네! 이놈은 평판 점수 그딴 거 필요 없고 1일 차부터 바로 파란 방패 씌워주고 프리패스 통과!!"**
  - 결국 윈도우 생태계에서 앱 장사를 하려는 모든 기업은 울며 겨자 먹기로 10배 비싼 EV 코드 서명 인증서를 사서 USB를 금고에 모셔야 하는 강제적 컴플라이언스(Compliance)의 포로가 되었다.

### Ⅳ. 결론

"윈도우 생태계를 통치하는 유일한 디지털 국새(國璽)."
Microsoft Authenticode는 해커의 바이러스 파일이 백신을 피해 내 PC에서 실행되려는 마지막 1초의 순간, 파일 구조의 심연에 봉인된 마법의 도장(PKCS#7)을 뜯어내어 원산지와 무결성을 판독해 내는 최후의 철옹성이다. 개발자들에게는 귀찮은 `signtool` 명령어의 암기 대상이자 100만 원짜리 세금(EV 인증서)으로 욕을 먹지만, 이 완고한 서명 정책과 타임스탬프의 융합 아키텍처가 없었다면 지금 이 순간에도 수억 대의 윈도우 PC는 인터넷에서 다운로드한 정체불명의 파일들(Malware)에 의해 처참히 도륙당하고 있었을 것이다.

### 📌 관련 개념 맵
* **기반 뼈대 기술**: Code Signing (코드 서명 본질 로직), PKCS#7 (서명 캡슐화 포맷) [170, 188번 문서]
* **핵심 연계 아키텍처**: Time Stamping (타임스탬프 서버, 인증서 만료 방어)
* **OS 검증 주체**: Windows SmartScreen, UAC (User Account Control)
* **실무 명령어**: `signtool.exe` (Windows SDK 제공)

### 👶 어린이를 위한 3줄 비유 설명
1. 인터넷에서 윈도우 게임을 다운받을 때, 나쁜 해커가 게임 속에 바이러스를 숨겨놨을까 봐 윈도우(경찰)가 엄청 경계하면서 시뻘건 경고창을 띄워요.
2. 하지만 진짜 게임회사 아저씨가 게임 파일 엉덩이에 '어센티코드'라는 윈도우 전용 황금 스티커 도장을 쾅! 붙여놓으면 상황이 달라져요.
3. 윈도우 경찰이 더블 클릭 0.1초 전에 그 황금 스티커를 스캔해 보고 "오! 이건 진짜 넥슨 아저씨가 만든 거고 바이러스도 1도 안 묻었네!" 하고 파란색 안심 마크를 띄우며 게임을 재밌게 켜준답니다!
