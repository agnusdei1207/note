# 188. 코드 서명 (Code Signing)

> ⚠️ 이 문서는 인터넷에서 다운받은 파일(`setup.exe`나 스마트폰 앱)이 네이버나 애플 같은 진짜 회사가 만든 순정품인지, 아니면 해커가 다운로드 중간에 낚아채서 바이러스를 주사기로 찔러넣은 변조된 악성코드인지를 0.1초 만에 판별해 내는 소프트웨어 원산지 보증 시스템, 코드 서명을 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 코드 서명(Code Signing)은 소프트웨어 개발자가 코드를 짜고 컴파일하여 실행 파일(.exe, .apk)을 배포하기 직전에, 자신의 **마스터 개인키를 이용해 파일 전체를 해시(Hash)하고 암호화 도장을 쾅 찍어 겉면에 포장지(PKCS#7)로 묶어두는 기술**이다.
> 2. **가치**: 이 도장 덕분에 사용자의 운영체제(Windows, iOS)는 프로그램을 더블 클릭하는 순간 0.1초 만에 도장을 검사하여 **①이 프로그램을 만든 놈이 진짜 '카카오톡'이 맞는지(인증), ②파일을 다운받는 동안 바이러스가 1바이트라도 침투해 훼손되지 않았는지(무결성)** 완벽하게 확신할 수 있다.
> 3. **한계/융합**: 이 방패는 너무 강력해서 해커들이 아예 게임회사(개발자)의 노트북을 해킹해 '코드 서명용 개인키' 자체를 훔쳐 악성코드에 '합법적인 서명'을 찍어버리는(Stuxnet 사태) 짓을 벌였다. 이에 대응하여 오늘날 코드 서명 키는 절대 하드디스크에 두지 못하게 **물리적 USB 금고(HSM/토큰)에 강제로 박아버리는 융합 규정**으로 진화했다.

---

### Ⅰ. 개요 및 왜 실행 파일에 도장을 찍는가? (Context & Necessity)
당신이 인터넷 자료실에서 `V3_백신_설치.exe`를 다운받아 더블 클릭했다. 
만약 해커(다스)가 다운로드 통신망 중간에 끼어들거나 자료실 서버를 해킹해서, 진짜 V3 파일에 악성 랜섬웨어 코드 100바이트를 주사기로 살짝 찔러넣었다면? 당신의 윈도우 PC는 "주인님이 V3를 켜셨다!"라며 최고 권한(Root)으로 바이러스를 실행하고 컴퓨터는 장렬히 불타오른다.

운영체제(OS)를 만든 마이크로소프트와 애플은 딜레마에 빠졌다. "인터넷에 떠도는 쓰레기 프로그램과 착한 프로그램을 OS가 무슨 수로 구별하지?"
그래서 그들은 암호학자들을 데려와 헌법을 만들었다. **"앞으로 윈도우와 아이폰에서 돌아가는 모든 프로그램은, 개발자가 자기가 만들었다는 암호학적 도장(Code Signing)을 겉면에 쾅 찍어와라! 도장이 없거나 도장이 깨진 프로그램은 무조건 실행을 차단해 버리겠다!"**

> 📢 **섹션 요약 비유**: 시장에서 우유를 샀습니다. 누군가 독약을 탔을까 봐 무섭습니다. 코드 서명은 우유병 뚜껑에 서울우유 공장장님이 진짜 플라스틱 씰링(비닐 띠)을 꽉 붙여놓은 것과 같습니다. 내가 뚜껑을 열 때 비닐 띠가 뜯어지지 않고 짱짱하다면, "아! 공장에서 우리 집 냉장고까지 오는 동안 아무도 뚜껑을 열어 독약을 타지 않았구나(무결성)!" 하고 100% 믿고 원샷할 수 있습니다.

### Ⅱ. 코드 서명의 3단계 완벽 작동 원리

이 메커니즘은 해시(Hash)와 비대칭키(PKI)가 융합된 예술이다.

#### 1. 도장 찍기 (개발자의 작업)
- 넥슨(Nexon) 개발자가 게임 설치 파일(`game.exe`, 1GB)을 다 만들었다.
- 이 1GB짜리 파일을 **SHA-256 믹서기**에 넣고 돌려서 32바이트짜리 아주 작은 지문(Hash)을 뽑아낸다.
- 이 지문을 넥슨의 1급 비밀인 **마스터 개인키(Private Key)**로 쾅! 압축해서 전자서명 도장을 만든다.
- 이 도장과 넥슨의 신분증(인증서)을 예쁜 캡슐[170번 문서 PKCS#7]에 담아 `game.exe` 파일 엉덩이에 찰싹 붙여서(Append) 인터넷에 배포한다.

#### 2. 다운로드 도중의 해커 침투 시도
- 해커가 다운로드 중인 패킷을 낚아채어 바이러스 코드 1줄을 끼워 넣었다. (파일이 변조됨).

#### 3. 운영체제의 검문 (더블 클릭의 순간)
- 앨리스가 윈도우에서 `game.exe`를 더블 클릭한다.
- 윈도우 OS(SmartScreen)는 파일을 당장 실행시키지 않고 잠깐 멈춘다.
- 파일 엉덩이에 붙은 넥슨의 공개키(인증서)를 꺼내서, 넥슨이 찍어둔 도장(서명)을 풀어본다. $\rightarrow$ "넥슨이 이 도장을 찍을 당시의 오리지널 지문이 `1A2B...` 였군."
- 윈도우 OS가 현재 다운받은 1GB짜리 파일 전체를 다시 SHA-256 믹서기에 넣고 갈아본다. $\rightarrow$ "어? 지금 믹서기에서 나온 지문은 `9Z8Y...` 잖아! 넥슨이 처음 찍었던 도장이랑 다르네?!"
- **결과 판정**: "파일이 해커에 의해 조작되었다(무결성 파괴)!!" 윈도우는 즉시 시뻘건 화면을 띄우며 게임 실행을 무자비하게 터뜨려버린다. 해킹 방어 성공!

```text
┌─────────────────────────────────────────────────────────────────────────────────┐
│           소프트웨어 코드 서명(Code Signing)의 무결성 쉴드 작동 시각화          │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│ 💻 [ 넥슨 개발팀 (배포 전) ]                                                    │
│   게임.exe (1GB) ──▶ (해시 믹서기) ──▶ 지문 [A] 추출                            │
│   지문 [A] + [넥슨 마스터 개인키 🔑] ──▶ 🔒 넥슨 도장 쾅! 완성!                 │
│   인터넷 배포: [ 게임.exe 본체 + 🔒 넥슨 도장 ]                                 │
│                                                                                 │
│ ────────────────────────────────────────────────────────────                    │
│ ☠️ [ 해커의 개입 (인터넷 망) ]                                                  │
│   해커가 파일에 악성코드 주입! -> 파일이 "바이러스_게임.exe"로 변조됨.          │
│                                                                                 │
│ ────────────────────────────────────────────────────────────                    │
│ 👨 [ 사용자 PC (더블 클릭 실행 순간) ]                                          │
│   (1) 윈도우 OS가 다운받은 바이러스 파일을 다시 믹서기에 돌림 -> 지문 [B] 나옴  │
│   (2) 파일 엉덩이의 🔒 넥슨 도장을 풀어봄 -> 오리지널 지문 [A] 가 나옴          │
│                                                                                 │
│   ★ 판결: "지문 [A] 와 [B] 가 다르다! 누군가 파일을 뜯어고쳤다!"                │
│   -> 🔴 "이 앱은 손상되었거나 바이러스가 있습니다. 실행을 차단합니다." 컷!      │
└─────────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 코드 서명의 위대함은 "해커가 파일을 변조하고 나서, 자기가 도장(서명)을 다시 찍어버리면 안 되나요?"라는 의문을 완벽히 박살 낸다. 해커가 도장을 다시 찍으려면 **넥슨의 마스터 개인키**를 알고 있어야 하는데, 그건 넥슨의 지하 금고에 있으므로 불가능하다. 해커가 자기 개인키로 도장을 파서 찍으면? 윈도우 OS는 "넌 넥슨이 아니라 해커 다스잖아? 듣보잡 잡상인 차단!" 하고 입구 컷을 시켜버린다. (이것이 공인 인증기관(CA)의 위력이다.)

### Ⅲ. 최악의 악몽: 도난당한 인감도장 (Stuxnet 사태)

2010년 이란 원자력 발전소를 날려버린 전설적인 무기, **스툭스넷(Stuxnet)** 바이러스는 이 완벽한 쉴드를 어떻게 뚫었을까?
수학의 틈을 뚫은 게 아니다. 물리적 현실 세계의 도둑질을 택했다.

- 국가 해커들(미국/이스라엘 추정)은 대만 리얼텍(Realtek) 등 유명 반도체 회사의 본사 서버를 털어서, 그 회사의 **'코드 서명용 마스터 개인키(인감도장)'** 파일을 통째로 훔쳐냈다.
- 해커는 원전 파괴용 악성 바이러스를 짜고, 훔쳐 온 **[리얼텍 진짜 도장]**을 겉면에 쾅 찍어서 윈도우 컴퓨터에 들이밀었다.
- 윈도우 OS의 반응: "오! 리얼텍 님이 팩스로 보내신 합법적이고 안전한 파일이네! 통과 무사 실행 🟢!" 
- 진짜 껍데기를 뒤집어쓴 악성코드는 윈도우의 모든 백신을 합법적으로 패스하며 원심분리기를 파괴했다.

**생태계의 각성 (하드웨어 토큰 의무화)**:
이 사태 이후 CA(인증기관)들은 기겁을 했다. "야! 개발자 놈들이 개인키 파일을 하드디스크 C드라이브에 대충 저장해 두니까 다 털리잖아!"
현재 최상위 등급(EV) 코드 서명 인증서를 사려면, CA는 파일 다운로드 버튼을 안 준다. 대신 며칠 뒤 우체국 택배로 **드릴로 뚫으면 자폭하는 특수 물리 USB(Hardware Token/HSM)**를 보내준다. 개인키는 평생 그 USB 밖으로 나오지 못하며, 넥슨 개발자가 서명을 찍으려면 무조건 컴퓨터에 이 USB를 꽂고 물리적 비밀번호를 눌러야만 도장이 찍히게 법(Compliance)을 통째로 뜯어고쳐 버렸다.

### Ⅳ. 결론

"디지털 공간의 원산지 증명, 해킹과의 끝없는 창방패 싸움."
코드 서명(Code Signing)은 소프트웨어 유통의 신뢰를 담보하는 마지막 마지노선이다. 오픈소스나 이름 모를 파일들이 수억 번씩 복사되어 날아다니는 세상에서, 이 작은 암호학적 캡슐(PKCS#7) 하나가 100만 명의 컴퓨터가 좀비 PC가 되는 것을 막아주고 있다. 해커들이 수학을 포기하고 개발자의 책상 위에 놓인 USB를 훔치는 물리적 첩보전으로 선회할 만큼, 코드 서명 자체의 논리는 완벽에 가깝다. 당신이 아이폰 앱스토어에서 앱을 다운받을 때 아무 의심 없이 손가락을 대는 그 평온함은, 개발자들의 영혼이 담긴 이 치열한 전자 도장(Signature)이 백그라운드에서 수천 번의 검증을 대신 견뎌주고 있기 때문이다.

### 📌 관련 개념 맵
* **전체 도메인**: Application Security, PKI 기반 파일 무결성 및 신뢰성 검증
* **관련 생태계 아키텍처**: Microsoft Authenticode (윈도우 .exe/.dll 서명) [189번 문서], Apple Developer ID (맥/iOS 앱 서명) [190번 문서]
* **핵심 서명 포맷**: PKCS#7 / CMS 캡슐화 [170번 문서]
* **방어하는 공격 모델**: Man-in-the-Middle (다운로드 변조), Malware Injection, 짝퉁 앱 배포 공격
