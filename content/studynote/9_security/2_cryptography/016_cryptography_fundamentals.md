# 암호학 기초 (Cryptography Fundamentals)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 암호학은 정보를 안전한 형태로 변환하여 Unauthorized 접근으로부터 보호하는 기술이다. 기밀성 (Confidentiality), 무결성 (Integrity), 인증 (Authentication), 부인 방지 (Non-repudiation) 를実現한다.
> 2. **가치**: 현대 인터넷 상거래, 온라인 은아크, 메시징 등에서 기반이 되는 기술로, 적절한 암호화 없이는 디지털 시대의 비즈니스가成立할 수 없다.
> 3. **융합**: 암호학은 수론, 정보이론, 컴퓨터과학의 교차점에서 발전했으며, 공개키 infrastructure, TLS, 디지털 서명 등 광범위한 애플리케이션의 기반이다.

---

## Ⅰ. 개요 및 필요성

### 개념 정의

암호학 (Cryptography)은 Greek어 " cryptós" (숨긴) 와 "gráphein" (쓰기) 에서 유래했다. 정보보호를 위해 정보를看不懂하는 형태로 변환하고,正当한 수신자만이 이를 다시看懂할 수 있도록 하는 기술이다.

암호학의 네 가지 핵심 목표는 다음과 같다:
- **기밀성 (Confidentiality)**: 오직 인가된 사람만 정보에 접근 가능
- **무결성 (Integrity)**: 정보가 변경되지 않았음을 보장
- **인증 (Authentication)**: 송신자 또는 수신자의 신원 확인
- **부인 방지 (Non-repudiation)**: 송신자가 보낸 것을 나중에 부정할 수 없음

### 필요성

암호학 없이는 인터넷 통신의安全이 보장되지 않는다. 금융 거래, 의료 기록, 개인 정보 등 모든 중요한 데이터가 노출되고,任何人이 위장하여通信할 수 있게 된다.暗号学的保護는 디지털 시대의 모든 비지니스 활동의基础이다.

### 비유

암호학은**편지 내용을 봉투에 넣고 잠그는 것**과 같다. 편지를 봉투에 넣으면 아무도 내용을 볼 수 없고 (기밀성), 봉투가 뜯어지지 않았다면 내용이 unchanged임을 알 수 있고 (무결성), 보낸 사람의署名으로確認이 가능하며 (인증), 보낸 사람은署名を否認할 수 없다 (부인 방지).

### 등장 배경

암호학은 고대 이집트의 히스로glyph에서도 사용되었으며, Julia Caesar가 사용한 Caesar Cipher, 나치 독일의 Enigma machine까지 역사가 깊다. 1970년대 DES (Data Encryption Standard) 의 탄생과 함께 현대 암호학이 시작되었고, 1976년 Diffie-Hellman 키 교환, 1977년 RSA 알고리즘의 발표로 공개키 암호학时代이 열렸다.

```text
  ┌──────────────────────────────────────────────────────────────┐
  │              암호학 발전 역사                                  │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │  고대 ~ 1949     │  1970년대        │  1990년대 ~ 현재      │
  │  ────────────────┼──────────────────┼─────────────────────  │
  │                  │                  │                       │
  │  수동 암호        │  컴퓨터 암호      │  공개키 암호학        │
  │  - Caesar Cipher │  - DES (1975)   │  - RSA (1977)       │
  │  - Vigenere     │  - CBC 모드     │  - ECC (1985)       │
  │  - Enigma       │  - 키 길이 확대 │  - AES (2001)       │
  │                  │                  │  - SHA-2, SHA-3    │
  │                  │                  │  - 양자 저항 암호   │
  │                                                              │
  │  1949 (Shannon) │  1976 (Diffie-  │  2016~ (NIST       │
  │  정보이론 기반   │  Hellman) 공개키 │  양자후 저항 표준화) │
  │  대칭키 이론     │  키 교환 발표   │                     │
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 암호학은 크게 고대 수동 암호, 1970년대 컴퓨터 암호, 1990년대 이후 공개키 암호학의 3단계로 나뉜다. 고대 암호는 대체 암호 (Substitution) 와 전치 암호 (Transposition) 가 중심이었지만, 컴퓨터의 등장으로 DES와 같은 고급 대칭키 알고리즘이 개발되었다. 1976년 Diffie-Hellman 키 교환의 발표는 암호학의game changer로, 공개키 암호학 시대의 시작을 알렸다. 현재는 양자 컴퓨터의 위협에 대응하기 위한 양자후저항 암호 (Post-Quantum Cryptography) 가 활발히 연구되고 있다.

## Ⅱ. 아키텍처 및 핵심 원리

### 암호화의 기본 구성

암호화 시스템은 다음 요소로 구성된다:

- **평문 (Plaintext)**: 원본 메시지
- **암호문 (Ciphertext)**: 암호화된 메시지
- **키 (Key)**: 암호화/복호화에 사용되는 비밀 정보
- **암호화 알고리즘 (Encryption Algorithm)**: 평문을 암호문으로 변환
- **복호화 알고리즘 (Decryption Algorithm)**: 암호문을 평문으로 복원

### 대칭키 vs 공개키 암호학

암호학의 가장 근본적인 구분은 대칭키 암호학과 공개키 암호학이다:

```text
  ┌──────────────────────────────────────────────────────────────┐
  │         대칭키 vs 공개키 암호학 비교                           │
  ├──────────────────────────────────────────────────────────────┤
  │
  │  【대칭키 암호학】                                             │
  │  ┌──────────────────────────────────────────────────────┐  │
  │  │                                                      │  │
  │  │   [송신자]          [수신자]                          │  │
  │  │      │                 │                             │  │
  │  │      │  비밀 키 (共有)  │                             │  │
  │  │      │◀───────────────▶│                             │  │
  │  │      │                 │                             │  │
  │  │      │  암호화 ──▶    │                             │  │
  │  │      │  평문 → 암호문   │                             │  │
  │  │      │                 │                             │  │
  │  │      │  복호화 ◀──    │                             │  │
  │  │      │  암호문 → 평문   │                             │  │
  │  └──────────────────────────────────────────────────────┘  │
  │
  │  - 같은 키로 암호화/복호화                                   │
  │  - 속도가 빠름 (AES: 수 Gbps 처리 가능)                    │
  │  - 키 분배 문제가 핵심 과제                                 │
  │
  ├─────────────────────────────────────────────────────────────┤
  │
  │  【공개키 암호학】                                            │
  │  ┌──────────────────────────────────────────────────────┐  │
  │  │                                                      │  │
  │  │   [송신자]          [수신자]                          │  │
  │  │      │                 │                             │  │
  │  │      │  공개키 (배포)   │                             │  │
  │  │      │──────────────────│                             │  │
  │  │      │  개인키 (비밀)   │                             │  │
  │  │      │◀────────────────│                             │  │
  │  │      │                 │                             │  │
  │  │      │  암호화 ──▶    │ (수신자 공개키로 암호화)    │  │
  │  │      │  평문 → 암호문   │                             │  │
  │  │      │                 │                             │  │
  │  │      │  복호화 ◀──    │ (수신자 개인키로 복호화)    │  │
  │  │      │  암호문 → 평문   │                             │  │
  │  └──────────────────────────────────────────────────────┘  │
  │
  │  - 공개키로 암호화, 개인키로 복호화                          │
  │  - 키 분배 문제 해결 (공개키는 누구나 알 수 있음)           │
  │  - 속도가 느림 (대칭키比 1000배 이상)                     │
  │  - 주로 키 분배와 인증에 사용                               │
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 대칭키 암호학은 같은 키를 사용하므로 속도가 빠르지만, 송신자와 수신자가事前に 같은 키를共有해야 하는鍵配送問題가 있다. 공개키 암호학은 공개키는 누구나 알 수 있지만 개인키는 수신자만 보유하므로鍵配送問題가 해결된다. 그러나 공개키 암호화는 연산량이 매우 많아 속도가 느리므로, 실무에서는 양쪽의 장점을 활용한 하이브리드方式是 주로 사용한다.

### 혼합 암호 시스템

실제 시스템에서는 대칭키의 속みと 공개키의鍵管理 편의성을 결합한 하이브리드 방식을 사용한다:

1. 송신자가 무작위 세션 키 (대칭키) 생성
2. 세션 키를 수신자의 공개키로 암호화하여 전송
3. 실제 데이터는 세션 키로 암호화하여 전송
4. 수신자는 개인키로 세션 키를 복호화
5. 세션 키로 데이터를 복호화

## Ⅲ. 융합 비교 및 다각도 분석

### 암호 모드 분류

| 모드 | 약자 | 특징 | 용도 |
|:---|:---|:---|:---|
| **Electronic Codebook** | ECB | 같은 평문 = 같은 암호문 (패턴 노출) | 비권장 |
| **Cipher Block Chaining** | CBC | 이전 블록과 XOR 연산 | 일반 데이터 |
| **Counter Mode** | CTR | 카운터 기반 스트림 암호 |高速 통신 |
| **Galois/Counter Mode** | GCM | CTR + 인증 태그 | TLS 1.3 |

### 과목 융합 관점

- **수론 (Number Theory)**: RSA는 소인수분해의 어려움에, ECC는 이산대수의 어려움에 기반한다. These mathematical problems are fundamental to public-key cryptography.
- **정보이론 (Information Theory)**: Shannon의 정보 이론은 perfect secrecy의 개념을定義하고, 암호 시스템의安全性极限를規定한다.
- **네트워크 보안**: TLS/IPsec은 암호학을 활용하여 네트워크 통신을보호하는 핵심 프로토콜이다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 클라우드 데이터 암호화**: 조직의 중요 데이터를 클라우드에 저장해야 하지만, 클라우드 공급업체조차 데이터에 접근할 수 없도록 해야 한다. 아키니어는 클라이언트 측 암호화 (Client-Side Encryption) 방식을採用하여, 데이터는 조직 내부에서 암호화한 후 클라우드에 업로드하고, 키는 별도의 키 관리 서비스 (KMS) 에서管理한다.

2. **시나리오 — 메신저 앱의 종단 간 암호화**: 사용자들이私人 대화를安全하게 주고받을 수 있어야 한다. 아키니어는 Signal 프로토콜을 기반으로 한 종단 간 암호화 (E2EE) 를 구현한다. 이 방식에서는 서버도 암호화된 메시지를看不懂하며, 오직 대화 당사자만 키를 보유한다.

### 도입 체크리스트

- **기술적**: 어떤 암호화 알고리즘을 사용하는가? 키 길이는 충분한가? 키 관리는 어떻게 하는가?
- **운영·보안적**: 암호 키의 백업과 복구 절차가 있는가? 암호화된 데이터의 복호화 권한은 어떻게 관리하는가?

### 안티패턴

- **자체 암호화 알고리즘 개발**: 표준화된 검증된 알고리즘 (AES 등) 을 사용하지 않고 자체 개발한 알고리즘을 사용하는 것. 전문가들도 검증하지 않은 암호화 알고리즘에는 치명적 결함이 있을 가능성이 높다.
- **비밀번호를 키로 직접 사용**: 사용자가 입력한 비밀번호를 그대로 키로 사용하는 것은dictionary 공격에 취약하다. PBKDF2, bcrypt 등으로 키를派生해야 한다.
- **ECB 모드 사용**: ECB 모드에서는 동일한 평문이 동일한 암호문을 생성하므로 패턴이 노출된다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 암호화 미적용 | 암호화 적용 시 | 개선 효과 |
|:---|:---|:---|:---|
| **정량** | 데이터 유출 시 完全 노출 | 데이터 unreadable | 데이터 보호 **100%** |
| **정성** | 고객 신뢰도 낮음 | 보안 인증 획득 | 고객 신뢰 **향상** |

### 미래 전망

- **양자 컴퓨터 위협**: 양자 컴퓨터가 실용화되면 Shor 알고리즘으로 RSA와 ECC가突破될 수 있다. NIST는 이미 양자후저항 암호 표준 (CRYSTALS-Kyber, CRYSTALS-Dilithium) 을 발표했으며, 양자-안전한 암호화への移行が迫切하다.
- **동형 암호화 (Homomorphic Encryption)**: 암호화된 상태로 연산을 수행할 수 있는 기술로, 데이터를 복호화하지 않고도 분석이 가능해 Privacy-Preserving Computing이 실현된다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **AES (Advanced Encryption Standard)** | 현재 표준으로 사용되는 대칭키 암호화 알고리즘으로, 128/192/256비트 키를支持한다. |
| **RSA** | 소인수분해의 어려움에 기반한 공개키 암호화로, 키 분배와 디지털 서명에 사용된다. |
| **ECC (Elliptic Curve Cryptography)** | 타원곡선 이산대수 문제에 기반한 공개키 암호화로, RSA보다 짧은 키로 동일한安全強度を実現한다. |
| **SHA (Secure Hash Algorithm)** | 데이터 무결성 검증에 사용되는 해시 함수로, SHA-2 (SHA-256) 와 SHA-3로 분류된다. |
| **키 관리 (Key Management)** | 암호화 키의 생성, 분배, 저장, 회수, 파기 등의 전체 lifecycle를管理하는 것이다. |
| **양자후저항 암호 (PQC)** | 양자 컴퓨터의 공격에도 안전한 암호화 알고리즘으로, NIST에서 표준화 중이다. |

## 👶 어린이를 위한 3줄 비유 설명

1. 암호화는**글자를 다른 글자로 바꾸는 언어 게임**이야. "가"를 "나"로, "나"를 "다"로 바꾸면 "가나다"가 "나다라"가 되는 것처럼, 올바른 열쇠 없이는 무엇이 적혔는지 알 수 없는 거야.
2. 중요한 건**어떤 글자를 어떤 글자로 바꾸는지에 대한 규칙 (키)**이야. 이 규칙을 아는 사람만이 글을 다시 읽을 수 있거든.
3. 그리고 컴퓨터에서는 "가=1, 나=2" 같은 숫자로 대응하는데, 요즘은 AES라는 가장 좋은 방법을 써서 숫자를 뒤섞는다고 생각하면 돼. 그래서 암호화를 하면 아무리优秀한 도둑이라도 내용을 알 수 없는 거야!
