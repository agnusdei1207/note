## 핵심 인사이트 (3줄 요약)

> 1. **본질**: IoT 및 OT 보안은 사물인터넷 기기와 제조·에너지 등 사회 기반 시설을 제어하는 산업 제어 시스템 (ICS)을 내외부의 사이버 위협으로부터 보호하여 물리적 안전과 비즈니스 가용성을 확보하는 기술이다.
> 2. **가치**: 정보 IT 보안과 달리 물리적 가용성 (Availability)과 안전 (Safety)을 최우선으로 하며, 하드웨어 제약 조건 하에서의 경량 암호화와 망 분리를 통해 국가 핵심 인프라의 마비를 방어한다.
> 3. **융합**: 가시성 확보 (Asset Discovery) 기술과 하이브리드 침입 탐지 시스템이 스마트 팩토리, 스마트 시티의 운영 효율성과 결합되어 초연결 산업 생태계의 신뢰 기반을 형성한다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 사이버 공격이 물리적 파괴로: OT 보안의 경고

과거의 공장이나 발전소는 외부망과 분리되어 안전하다고 믿어왔다. 하지만 '4차 산업혁명'으로 인해 폐쇄망이었던 산업 제어 시스템 (ICS)이 IT 망과 연결되면서 공격의 통로가 열렸다. 이제 사이버 공격은 단순히 데이터를 훔치는 것을 넘어, 터빈을 멈추거나 전력망을 차단하여 **인명 피해와 물리적 재난**을 일으키는 실질적인 위협이 되었다 (예: Stuxnet 사태).

IoT 및 OT 보안이 필요한 이유는 세 가지이다. 첫째, **가용성의 절대적 보장**을 위해서이다. 생산 라인이 1분만 멈춰도 수억 원의 손실이 발생한다. 둘째, **하드웨어 제약 사항 극복**을 위해서이며 (저사양 CPU, 배터리 제한), 셋째, **노후화된 레거시 시스템 보호**를 위함이다 (수십 년 된 OS와 보안 패치 불가 장비).

이 그림은 IT 보안과 OT 보안의 우선순위 및 환경적 차이를 비교한다.

```text
┌─────────────────────────────────────────────────────────────┐
│                 IT Security vs OT Security (CIA)            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [ IT Security ]                 [ OT Security ]           │
│   1. Confidentiality (기밀성)     1. Availability (가용성)  │
│   2. Integrity (무결성)           2. Safety (안전성)        │
│   3. Availability (가용성)        3. Integrity (무결성)     │
│                                                             │
│   - 빠른 패치 주기                - 패치 불가능/매우 신중   │
│   - 데이터 중심                   - 물리적 프로세스 중심    │
│   - 3~5년 교체 주기               - 10~30년 장기 운영       │
│                                                             │
│   * 핵심: "멈추면 죽는다"는 OT의 특성이 보안 설계의 전제    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

이 다이어그램의 핵심은 '우선순위의 역전'이다. IT에서는 비밀을 지키는 게 중요하지만, OT에서는 일단 기계가 돌아가는 것(가용성)과 사람이 다치지 않는 것(안전)이 먼저다. 실무에서는 이 때문에 시스템을 멈춰야 하는 보안 패치보다, 망 분리와 실시간 감시를 통한 '보호' 전략이 우선된다.

### IoT 및 OT 보안의 주요 대상

1. **IoT 단말**: 스마트홈 기기, 웨어러블, 스마트 센서. (보안 자원 극히 제한)
2. **SCADA (Supervisory Control and Data Acquisition)**: 광역 인프라 감시 제어 시스템.
3. **PLC (Programmable Logic Controller)**: 현장 설비를 직접 제어하는 하드웨어.
4. **DCS (Distributed Control System)**: 공정별로 분산된 제어 시스템.

📢 **섹션 요약 비유**: IT 보안이 '비밀 장부를 지키는 금고실 경비'라면, OT 보안은 '멈추면 안 되는 놀이공원 기구의 안전 요원'과 같습니다. 장부가 털리는 것도 문제지만, 기구가 멈춰서 사람이 다치는 것이 훨씬 더 치명적인 상황입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 퍼듀 모델 (Purdue Model)과 망 분리

산업 제어 시스템의 논리적 계층 구조를 정의한 표준이다.

- **Level 4/5 (Enterprise)**: 비즈니스 네트워크 (이메일, ERP).
- **Level 3 (Operations)**: 제어 시스템 관리 및 모니터링.
- **Level 2 (Control)**: 실제 공정 제어 (HMI, PLC).
- **Level 0/1 (Process)**: 물리적 장치 (센서, 모터).
- **보안 원리**: 각 계층 사이를 방화벽으로 차단하고, 특히 IT 망과 OT 망 사이에 **DMZ (산업용 DMZ)**를 두어 접점을 최소화한다.

### IoT 전용 보안 기술: 경량 암호 (Lightweight Crypto)

메모리와 전력이 부족한 IoT 기기를 위한 최적화 암호이다.
- **알고리즘**: LEA (국산 표준), ASCON, PRESENT.
- **특징**: 적은 연산량으로 일정 수준 이상의 보안 강도 유지.

이 구조도는 OT 환경의 침입 탐지를 위한 **한방향 전송 장치 (Data Diode)**의 원리를 보여준다.

```text
┌─────────────────────────────────────────────────────────────┐
│                 Data Diode (한방향 보안 게이트웨이)         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [ OT Network ] ────── ( 물리적 단방향 ) ──────▶ [ IT Network ] │
│   (기밀 시스템)                                   (외부 연결)│
│                                                             │
│   * 원리: 빛(광케이블)이나 하드웨어 회로를 이용해           │
│     데이터가 밖으로 나갈 수는 있지만, 안으로 들어올 수는    │
│     없게 물리적으로 차단함                                  │
│                                                             │
│   * 효과: 해커가 외부에서 제어 명령을 보낼 경로를 원천 봉쇄│
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

이 다이어그램의 핵심은 '물리적 일방통행'이다. 소프트웨어가 아닌 하드웨어 구조로 역방향 통신을 불가능하게 만들어, 설비 데이터는 수집하면서도 외부 침입은 완벽히 차단한다. 실무에서는 국가 주요 기반 시설 (원자력, 수도 등)의 필수 보안 장비로 쓰인다.

📢 **섹션 요약 비유**: 퍼듀 모델은 '잠수함의 격벽'과 같습니다. 한 구역이 물에 잠겨도(해킹당해도) 다른 구역으로 번지지 않게 튼튼한 벽(방화벽)을 세우는 것입니다. 데이터 다이오드는 '창문은 있지만 밖에서만 볼 수 있는 매직 미러'와 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### IoT 보안 3대 이슈 및 대응

| 이슈 | 원인 | 해결책 |
|:---:|:---|:---|
| **계정 탈취** | 초기 설정 비밀번호(Admin) 그대로 사용 | 강력한 비밀번호 설정 강제, 생체 인증 |
| **펌웨어 변조** | 업데이트 경로의 보안 부재 | Secure Boot, 디지털 서명 검증 |
| **DoS 공격** | 기기들을 좀비화하여 대규모 공격 (Mirai) | 비정상 트래픽 모니터링, 포트 제어 |

### 프로토콜 보안 비교: Modbus vs OPC-UA

| 구분 | Modbus (Legacy) | OPC-UA (Modern) |
|:---:|:---|:---|
| **보안성** | 없음 (평문 전송) | **강력함 (인증, 암호화 내장)** |
| **호환성** | 매우 높음 (오래된 표준) | 높음 (이기종 통합 유리) |
| **특징** | 보안 고려 전 설계됨 | 설계 단계부터 보안 내재화 |
| **비유** | 속이 다 보이는 투명 상자 | **비밀번호가 달린 튼튼한 금고** |

📢 **섹션 요약 비유**: Modbus가 '말을 배우기 시작한 아이의 솔직한 대화'라면, OPC-UA는 '외교관들의 암호화된 비밀 전문'과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단: 산업 인프라 자산 보호 및 운영 전략

**시나리오 1: 오래된 PLC 장비와 최신 스마트 팩토리 솔루션의 결합**
- **판단**: 노후 장비에 직접 보안 패치를 하는 것은 위험하다 (시스템 다운 위험). 장비 앞단에 **산업용 보안 게이트웨이**를 배치하여 '가상 패칭'과 '프로토콜 정화'를 수행한다. 또한 폐쇄망 안에서만 동작하던 장비들을 위해 **Micro-segmentation**을 적용하여 각 공정별로 네트워크 보안 경계를 잘게 쪼개는 아키텍처를 수립한다.

**시나리오 2: IoT 기기를 이용한 대규모 DDoS 공격 (Botnet) 징후 포착**
- **판단**: 개별 기기를 하나씩 고치는 것은 불가능하다. 통신사 및 클라우드 게이트웨이 단에서 **공격 트래픽 패턴 (Signature)**을 식별하여 일괄 차단한다. 장기적으로는 제조사에게 **'보안 가이드라인 준수'**를 요구하고, 미인증 기기의 사내 망 접속을 원천 차단하는 **NAC (Network Access Control)** 연동 체계를 강화한다.

이 도식은 OT 보안의 성숙도를 높이기 위한 '단계적 이행 로드맵'을 보여준다.

```text
┌─────────────────────────────────────────────────────────────┐
│               OT Security Maturity Roadmap                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Step 1: Visibility (자산 식별 - 뭐가 있는지 알아야 함)   │
│          │                                                  │
│   Step 2: Segmentation (망 분리 - 격벽 설치)                │
│          │                                                  │
│   Step 3: Detection (모니터링 - 이상 징후 포착)             │
│          │                                                  │
│   Step 4: Response (자동 대응 - 플레이북 실행)              │
│                                                             │
│   * 기술사 가이드: 1단계 가시성 확보 없이 2단계로 가면 안 됨 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 기술사의 보안 판단은 '안전 점검관'의 역할과 같습니다. 새 기계를 들여올 때 성능만 보지 않고 사고 시 위험도(Safety)를 먼저 따지며, 낡은 부품(레거시 시스템)을 새 부품과 안전하게 연결할 수 있는 보호막(게이트웨이)을 설계하는 전문가입니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

### 지능형 산업 보안의 가치

1. **정량적 효과**: 산업 재해 및 설비 중단 비용 80% 절감, 보안 사고 탐지 시간 (MTTD) 90% 단축.
2. **정성적 효과**: 국가 주요 기반 시설의 안정적 운영을 통한 공공 안전 확보, 스마트 제조 경쟁력 강화.

### 미래 전망: CPS 보안과 AI 기반의 자율 방어

향후 보안은 가상 세계와 물리 세계가 완전히 통합된 **CPS (Cyber-Physical Systems) 보안**으로 진화할 것이다. 사람이 일일이 감시하지 않아도 AI가 설비의 미세한 동작 변화를 분석하여 해킹 시도를 물리적 현상으로 포착하는 **AI-anomaly Detection**이 표준이 될 것이다. 또한 양자 컴퓨터 공격에 대비한 물리적 보안 기술 (PUF)이 IoT 기기의 필수 신분증이 될 것이다. 기술사는 사이버 공간의 논리를 넘어, 전기가 흐르고 기계가 움직이는 물리적 현상에 대한 이해를 결합한 '융합 보안 전문가'가 되어야 한다.

📢 **섹션 요약 비유**: 미래의 산업 보안은 '지능형 면역 체계'와 같아질 것입니다. 공장의 모든 로봇과 센서가 스스로 건강을 체크하고, 외부의 침입이 들어오면 실시간으로 항체(보안 정책)를 만들어 스스로를 지켜내는 완벽한 생태계가 완성될 것입니다.
