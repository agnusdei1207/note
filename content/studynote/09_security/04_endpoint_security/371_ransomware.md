## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 랜섬웨어(Ransomware)는 피해자의 파일을 암호화하거나 시스템을 잠금 상태로 만든 후 복호화 키를 대가로 금전(주로 암호화폐)을 요구하는 사이버 갈취 악성코드다.
> 2. **가치**: 2024년 기준 글로벌 사이버 피해 1위 위협으로, RaaS (Ransomware as a Service) 모델로 진입 장벽이 낮아지면서 공격 빈도와 피해 규모가 지속 증가하고 있다.
> 3. **판단 포인트**: 예방(백업·패치·EDR)이 복구보다 압도적으로 효과적이며, 3-2-1 백업 규칙과 네트워크 세그멘테이션이 피해 최소화의 핵심이다.

---

## Ⅰ. 개요 및 필요성

랜섬웨어(Ransomware)는 몸값을 뜻하는 'ransom'과 소프트웨어(software)의 합성어다. 1989년 AIDS Trojan(PC Cyborg)이 최초 사례이지만, 2013년 CryptoLocker 등장 이후 비트코인(Bitcoin)이라는 추적 어려운 결제 수단과 결합하면서 폭발적으로 성장했다.

현대 랜섬웨어의 가장 큰 특징은 이중 갈취(Double Extortion)다. 단순히 파일을 암호화하는 것을 넘어, 암호화 전에 데이터를 미리 유출(exfiltration)해 두고 "몸값을 내지 않으면 데이터를 공개하겠다"고 협박한다. 2020년 Maze 그룹이 이 기법을 대중화했으며, 이후 삼중(Triple) 갈취—피해자 고객/파트너에게도 협박—까지 등장했다.

RaaS (Ransomware as a Service) 모델은 랜섬웨어 코드 개발자와 공격 실행자(affiliate)를 분리해, 기술 수준이 낮은 범죄자도 랜섬웨어 공격을 실행할 수 있게 했다.

📢 **섹션 요약 비유**: 랜섬웨어는 디지털 납치—파일을 인질로 잡고 몸값을 요구한다. 몸값을 내도 돌아온다는 보장이 없다.

## Ⅱ. 아키텍처 및 핵심 원리

| 단계 | 행위 | 주요 기법 |
|:---|:---|:---|
| 초기 침투 | 시스템 접근 | 피싱·RDP 취약점·취약한 VPN |
| 탐색 | 네트워크 정보 수집 | ADRecon, BloodHound |
| 권한 상승 | 도메인 관리자 권한 획득 | Mimikatz, Pass-the-Hash |
| 측면 이동 | 내부 시스템 확산 | PsExec, WMI, SMB |
| 데이터 유출 | 민감 데이터 외부 전송 | rclone, MegaSync |
| 암호화 | 파일 암호화 후 몸값 요구 | AES+RSA 혼합 암호화 |

```text
┌──────────────────────────────────────────────────┐
│            랜섬웨어 암호화 메커니즘              │
├──────────────────────────────────────────────────┤
│                                                  │
│  공격자: RSA 공개키/개인키 쌍 생성               │
│                                                  │
│  피해 시스템:                                    │
│  ┌──────────────────────────────┐               │
│  │ 1. AES 세션키 랜덤 생성       │               │
│  │ 2. 파일을 AES로 암호화        │               │
│  │ 3. AES 키를 RSA 공개키로 암호화│               │
│  │ 4. 암호화된 AES 키 저장       │               │
│  └──────────────────────────────┘               │
│                                                  │
│  복호화: 공격자 RSA 개인키 필요 (C2 서버 보관)   │
│                                                  │
│  → 공개키만으로는 복호화 불가능 (비대칭 암호화)  │
└──────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 파일에 열쇠가 잠겨있고(AES 암호화), 그 열쇠는 해커의 금고 안에 있다(RSA 개인키)—열쇠 없이 금고를 여는 건 수학적으로 불가능하다.

## Ⅲ. 비교 및 연결

| 항목 | 암호화 랜섬웨어 | 잠금 화면 랜섬웨어 | 와이퍼 |
|:---|:---|:---|:---|
| 목적 | 금전 갈취 | 금전 갈취 | 파괴 |
| 파일 복구 | 몸값 지불 시 가능 | 잠금 해제로 가능 | 불가 (삭제) |
| 기술 복잡도 | 높음 | 낮음 | 중간 |
| 대표 사례 | WannaCry, REvil | WinLocker | NotPetya |

📢 **섹션 요약 비유**: 잠금 화면 랜섬웨어는 창문을 막는 것, 암호화 랜섬웨어는 집 안 모든 물건을 잠금 상자에 넣는 것—후자가 훨씬 심각하다.

## Ⅳ. 실무 적용 및 기술사 판단

**3-2-1 백업 규칙**  
- 3개 이상의 데이터 복사본  
- 2가지 다른 저장 매체  
- 1개는 오프사이트(오프라인) 보관  

**기술적 예방**  
1. 정기 패치 관리—EternalBlue(MS17-010) 같은 SMB 취약점 즉시 패치  
2. RDP (Remote Desktop Protocol) 포트 외부 노출 금지  
3. 네트워크 세그멘테이션으로 측면 이동 차단  
4. EDR으로 대용량 파일 암호화 행위 실시간 탐지·차단  
5. 백업 서버를 별도 네트워크에 격리 (랜섬웨어 확산 방지)  

**몸값 지불 여부**: FBI·CISA는 지불 자제 권고—지불해도 복호화 키 미제공 사례 다수, 추가 공격 표적이 될 수 있음.

📢 **섹션 요약 비유**: 백업은 랜섬웨어의 '인질 협상 카드'를 무력화하는 유일한 무기—백업 없이는 협상 테이블에서 불리하다.

---

## Ⅴ. 기대효과 및 결론

랜섬웨어 대응의 핵심은 예방과 복구 능력이다. 공격이 성공하더라도 빠르게 복구할 수 있는 체계(오프라인 백업, 사고 대응 계획)가 갖춰진 조직은 몸값 지불 없이 복구 가능하다. RaaS 생태계의 성장으로 공격 빈도가 계속 증가하는 만큼, 사이버 보험(Cyber Insurance)과 함께 기술적·관리적 통제를 이중으로 강화하는 전략이 요구된다.

📢 **섹션 요약 비유**: 랜섬웨어는 화재와 같다—방화(예방)가 최선이지만, 화재보험(백업+사고대응계획)이 있으면 피해를 최소화할 수 있다.

### 👶 어린이를 위한 3줄 비유 설명
랜섬웨어는 나쁜 사람이 내 컴퓨터 파일을 잠가버리고 돈을 내야 열어준다고 협박하는 프로그램이에요.  
돈을 내도 파일을 안 돌려주는 경우도 있어서 절대 믿으면 안 돼요.  
평소에 파일을 USB나 외장하드에 따로 저장해 두면(백업), 잠가도 피해가 없어요.
