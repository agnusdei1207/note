## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 논리폭탄(Logic Bomb)은 특정 조건(날짜·이벤트·파일 존재 여부 등)이 충족될 때까지 잠복하다가 실행 시 데이터 파괴·시스템 중단·정보 유출 등의 악성 행위를 트리거하는 코드다.
> 2. **가치**: 즉각 탐지가 어렵고 내부자(insider)가 심는 경우가 많아 조직 내부의 신뢰 관계를 악용하는 위협으로, 퇴직 직원 보복 공격의 전형적 수단이다.
> 3. **판단 포인트**: 정적 분석만으로는 트리거 조건이 활성화되지 않으면 탐지가 불가능하므로, 코드 리뷰·동적 분석·소스 코드 버전 관리 감사가 핵심 방어 수단이다.

---

## Ⅰ. 개요 및 필요성

논리폭탄(Logic Bomb)은 악성 코드의 일종으로, 평상시에는 완전히 정상적으로 보이다가 프로그래머가 설정한 특정 트리거 조건이 만족되면 폭발적으로 악성 행위를 실행한다. 시한폭탄처럼 작동하지만 시간만이 아니라 특정 사용자 로그인, 파일 삭제 감지, 특정 프로세스 실행 여부 등 다양한 조건을 트리거로 쓸 수 있다.

내부자 위협(Insider Threat)과의 연관성이 특히 높다. 2008년 미국 팬니 메이(Fannie Mae)의 IT 계약직 직원 Rajendrasinh Makwana는 해고 당일 서버 스크립트에 논리폭탄을 심어 수천 대 서버의 데이터를 삭제하려 했다. 다행히 다른 직원이 사전 발견했지만, 이처럼 조직을 떠나는 직원이 보복 목적으로 삽입하는 경우가 전형적이다.

📢 **섹션 요약 비유**: 논리폭탄은 부비트랩—평소에는 그냥 돌멩이처럼 보이지만, 누군가 밟는 순간(조건 충족) 폭발한다.

---

## Ⅱ. 아키텍처 및 핵심 원리

| 트리거 유형 | 설명 | 예시 |
|:---|:---|:---|
| 시간 기반 | 특정 날짜·시각에 활성화 | 퇴직일 이후 실행 |
| 이벤트 기반 | 특정 이벤트 발생 시 활성화 | 직원명 DB에서 삭제 시 |
| 파일/데이터 기반 | 파일 존재 여부로 활성화 | 특정 파일이 없으면 실행 |
| 네트워크 기반 | 특정 패킷 수신 시 활성화 | C2 신호 수신 시 |
| 로그인 기반 | 특정 계정 마지막 로그인 탐지 | 해고자 계정 로그인 없을 시 |

```text
┌──────────────────────────────────────────────┐
│              논리폭탄 실행 흐름               │
├──────────────────────────────────────────────┤
│                                              │
│  [정상 코드 실행 중]                          │
│         │                                    │
│         ▼                                    │
│  [조건 검사: IF trigger == TRUE]             │
│         │                                    │
│    ┌────┴────────────┐                       │
│    │ FALSE           │ TRUE                  │
│    ▼                 ▼                       │
│  [계속 정상 실행]  [페이로드 실행]            │
│                      │                       │
│               ┌──────┴──────────┐            │
│               │ - 파일 삭제      │            │
│               │ - 데이터 유출    │            │
│               │ - 시스템 다운    │            │
│               │ - 랜섬웨어 드롭  │            │
│               └─────────────────┘            │
└──────────────────────────────────────────────┘
```

**대표적 페이로드 유형**  
- `rm -rf /` 또는 디스크 포맷 명령  
- 데이터베이스 레코드 일괄 삭제  
- 랜섬웨어 드로퍼(dropper) 실행  
- 백도어 설치 트리거  
- 금융 데이터 조작  

📢 **섹션 요약 비유**: 숨겨진 if문 하나가 수천 대 서버를 멈출 수 있다—조용한 코드 한 줄이 시한폭탄이다.

## Ⅲ. 비교 및 연결

| 항목 | 논리폭탄 | 랜섬웨어 | 와이퍼 |
|:---|:---|:---|:---|
| 트리거 | 특정 조건 충족 시 | 즉시 또는 예약 | 즉시 실행 |
| 목적 | 보복·사보타주 | 금전 갈취 | 파괴·혼란 |
| 탐지 난이도 | 매우 어려움 (잠복) | 암호화 행위로 탐지 | 대용량 쓰기로 탐지 |
| 주요 행위자 | 내부자 | 사이버 범죄 조직 | 국가 지원 APT |
| 합법 코드와 구분 | 어려움 | 명확히 악성 | 명확히 악성 |

📢 **섹션 요약 비유**: 랜섬웨어가 무장 강도라면, 논리폭탄은 퇴직한 직원이 두고 간 시한폭탄—강도는 바로 알지만 폭탄은 터지기 전까지 모른다.

## Ⅳ. 실무 적용 및 기술사 판단

**예방 및 탐지 전략**  
1. **소스 코드 리뷰**: 크론탭(crontab), 스케줄러 스크립트, 배치 파일에 숨겨진 시간 조건 탐지  
2. **버전 관리(VCS) 감사**: Git 커밋 히스토리에서 미승인 코드 변경 추적  
3. **최소 권한 원칙(PoLP)**: 코드 배포 권한을 최소화해 삽입 기회 제한  
4. **직원 퇴직 절차**: 퇴직자의 코드 기여분 즉시 감사 및 모든 접근 권한 즉시 회수  
5. **정적·동적 분석**: 특정 날짜 비교, 파일 존재 확인 등 의심 패턴 자동 탐지 도구 적용  
6. **코드 서명 및 무결성 검증**: 배포 전 모든 스크립트 해시 등록 및 검증  

**기술사 포인트**: "내부자 위협 대응 방안"과 함께 서술하면 논리폭탄의 맥락이 명확해진다.

📢 **섹션 요약 비유**: 퇴직자의 사무실 열쇠를 회수하듯, 퇴직자의 코드 기여분도 즉시 감사해야 한다.

---

## Ⅴ. 기대효과 및 결론

논리폭탄은 내부자 위협의 가장 교묘한 형태 중 하나로, 예방이 탐지보다 훨씬 효과적이다. 직원 이직·퇴직 절차에 코드 감사를 필수 단계로 포함하고, 변경 관리(Change Management) 프로세스를 강화하며, 코드 리뷰 문화를 조성하는 것이 핵심 대응 전략이다. SIEM과 UBA (User Behavior Analytics)를 통해 비정상적인 코드 변경 패턴을 조기에 포착하는 것이 최선이다.

📢 **섹션 요약 비유**: 논리폭탄은 예방 접종과 같다—이미 몸속에 들어온 후에는 치료가 어렵고 터지기 전에 백신(코드 리뷰)으로 막아야 한다.

### 👶 어린이를 위한 3줄 비유 설명
논리폭탄은 나쁜 사람이 프로그램 안에 숨겨 놓은 함정이에요.  
평소에는 아무 일도 안 하다가 특정 날짜나 조건이 되면 갑자기 나쁜 일을 해요.  
막으려면 프로그램 코드를 꼼꼼히 검사하고, 회사를 그만두는 직원의 코드도 반드시 확인해야 해요.
