## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 펌웨어 루트킷(Firmware Rootkit)은 BIOS/UEFI·HDD 펌웨어·네트워크 카드 펌웨어 등 하드웨어 초기화 코드에 악성 페이로드를 삽입해 OS 재설치나 디스크 포맷으로도 제거할 수 없는 최심층 백도어다.
> 2. **가치**: OS·하이퍼바이저보다 낮은 링 레벨(Ring -2 이하)에서 실행되므로 모든 보안 소프트웨어를 우회하며, 공급망 공격(Supply Chain Attack)의 핵심 무기로 사용된다.
> 3. **판단 포인트**: UEFI Secure Boot·TPM (Trusted Platform Module) 기반 Measured Boot·펌웨어 서명 검증이 대응책이며, 탐지가 극도로 어려워 하드웨어 교체가 최종 해결책이 되는 경우도 있다.

---

## Ⅰ. 개요 및 필요성

일반적인 루트킷(Rootkit)은 OS 커널이나 사용자 공간에 숨지만, 펌웨어 루트킷은 그보다 훨씬 깊은 계층—BIOS (Basic Input/Output System), UEFI (Unified Extensible Firmware Interface), HDD/SSD 컨트롤러 펌웨어, BMC (Baseboard Management Controller), NIC (Network Interface Card) 펌웨어 등에 코드를 삽입한다. 이 영역은 CPU가 POST (Power-On Self-Test) 단계에서 가장 먼저 실행하는 코드이므로 OS 부팅 전에 백도어가 활성화된다.

2018년 공개된 LoJax(Sednit APT 그룹)는 최초로 야생(in-the-wild)에서 발견된 UEFI 루트킷으로, SPI (Serial Peripheral Interface) 플래시 메모리에 악성 UEFI 모듈을 직접 기록했다. 2022년의 MoonBounce는 UEFI 펌웨어 내 CORE_DXE 컴포넌트를 변조해 감염 방식이 더욱 정교해졌다.

공격자가 이 기법을 선택하는 이유는 영속성(persistence) 극대화다. OS 재설치, 디스크 교체, 심지어 안티바이러스 부팅 디스크 스캔으로도 탐지·제거되지 않으며, 국가 수준 APT (Advanced Persistent Threat) 행위자가 장기 정보 수집에 활용한다.

📢 **섹션 요약 비유**: 건물 보안을 뚫는 것이 아니라 건물 기초공사 단계에 도청 장치를 심는 것처럼, 펌웨어 루트킷은 OS가 뜨기 전에 이미 시스템을 장악한다.

## Ⅱ. 아키텍처 및 핵심 원리

| 삽입 대상 | 실행 시점 | 영속성 강도 |
|:---|:---|:---|
| UEFI/BIOS SPI 플래시 | POST 직후, OS 전 | 매우 강함 (HW 교체 필요) |
| HDD/SSD 펌웨어 | BIOS 이후, 드라이버 로드 시 | 강함 (디스크 교체 필요) |
| BMC 펌웨어 | OOB (Out-of-Band) 관리 상시 | 강함 |
| NIC 펌웨어 | 네트워크 드라이버 초기화 시 | 중간 |
| GPU 펌웨어 | GPU 드라이버 로드 시 | 중간 |

```text
┌─────────────────────────────────────────────────┐
│              부팅 순서 및 감염 계층              │
├─────────────────────────────────────────────────┤
│  전원 ON                                        │
│     │                                           │
│     ▼                                           │
│  [UEFI/BIOS 펌웨어]  ◄── 펌웨어 루트킷 삽입    │
│     │  SPI Flash 실행                           │
│     ▼                                           │
│  [MBR/GPT 부트로더]  ◄── 부트킷 삽입 가능      │
│     │                                           │
│     ▼                                           │
│  [OS 커널 로드]       ◄── 커널 루트킷 삽입 가능 │
│     │                                           │
│     ▼                                           │
│  [사용자 공간]        ◄── 일반 악성코드         │
└─────────────────────────────────────────────────┘
```

펌웨어 루트킷은 감염 후 OS 부팅 과정에 드로퍼(dropper)를 삽입하거나, 네트워크 패킷을 도청하거나, C2 (Command & Control) 서버에 연결하는 백도어 모듈을 주입한다. SPI 플래시 쓰기 보호(write protection) 비트가 꺼져 있거나, 드라이버 취약점을 통해 Ring 0에서 직접 플래시에 접근한다.

📢 **섹션 요약 비유**: BIOS는 건물의 전기 배선판—여기에 숨으면 경비원(OS 보안)이 켜지기 전에 이미 모든 방을 열 수 있다.

## Ⅲ. 비교 및 연결

| 항목 | 펌웨어 루트킷 | 커널 루트킷 | 부트킷 |
|:---|:---|:---|:---|
| 실행 계층 | Ring -2 (펌웨어) | Ring 0 (OS 커널) | MBR/VBR |
| 영속성 | OS 재설치 후에도 생존 | OS 재설치로 제거 | 디스크 포맷으로 제거 |
| 탐지 난이도 | 매우 어려움 | 어려움 | 중간 |
| 대응 방법 | 펌웨어 재플래싱/HW 교체 | 커널 패치 | 부트 영역 복구 |
| 야생 사례 | LoJax, MoonBounce | Necurs | TDL4 |

UEFI Secure Boot는 부트로더와 OS 로더의 서명을 검증하지만, 펌웨어 자체가 변조된 경우에는 Secure Boot도 우회될 수 있다. 이를 막으려면 TPM (Trusted Platform Module)을 사용한 Measured Boot와 펌웨어 업데이트 서명 검증이 함께 필요하다.

📢 **섹션 요약 비유**: Secure Boot는 문에 자물쇠—그런데 열쇠 제조공장(펌웨어)이 이미 해킹됐다면 자물쇠도 무용지물이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

**탐지 기법**  
- CHIPSEC 도구를 이용한 SPI 플래시 무결성 검사  
- TPM PCR (Platform Configuration Register) 값 비교  
- 펌웨어 이미지 해시를 제조사 공식 이미지와 대조  
- UEFI 보안 데이터베이스(db/dbx) 이상 항목 점검  

**예방 및 대응**  
1. UEFI 펌웨어 정기 업데이트 및 서명 검증 활성화  
2. SPI 플래시 쓰기 보호(WP#) 핀 하드웨어 잠금  
3. Secure Boot + TPM Measured Boot 이중 적용  
4. 공급망 단계에서 SBOM (Software Bill of Materials) 기반 펌웨어 검증  
5. 고위험 시스템은 하드웨어 수준 신뢰 루트(RoT) 칩 탑재  

기술사 시험에서는 "루트킷의 계층별 분류"와 "UEFI Secure Boot의 한계"를 함께 서술하면 고득점이다.

📢 **섹션 요약 비유**: 자동차 ECU(제어 모듈)가 해킹된 상태에서는 핸들 잠금장치를 아무리 강화해도 의미가 없다—펌웨어 보안은 그래서 하드웨어부터 시작해야 한다.

---

## Ⅴ. 기대효과 및 결론

펌웨어 루트킷 방어는 단순한 소프트웨어 패치를 넘어 하드웨어 신뢰 루트(Root of Trust) 확보를 요구한다. TPM 2.0 기반 Measured Boot, UEFI Secure Boot, 펌웨어 서명 의무화, 그리고 공급망 보안 강화를 통해 공격 표면을 최소화할 수 있다. 국가 기반 시설과 고보안 환경에서는 FIPS 140-3 인증 HSM (Hardware Security Module)과 연동된 펌웨어 무결성 검증 체계가 필수적으로 요구된다.

📢 **섹션 요약 비유**: 집 전체를 청소해도 벽 안에 숨은 해충은 사라지지 않는다—펌웨어 루트킷은 벽을 뜯어야(HW 교체) 완전히 제거된다.

### 👶 어린이를 위한 3줄 비유 설명
컴퓨터가 켜지는 순간 가장 먼저 실행되는 아주 깊숙한 코드(BIOS/UEFI)에 나쁜 코드를 숨기는 방법이에요.  
운영체제(Windows, Linux)가 시작하기도 전에 이미 컴퓨터를 조종하고 있기 때문에 백신도 못 잡아요.  
마치 자명종 시계 안에 도청기를 심어 놓은 것처럼, 컴퓨터가 켜질 때마다 자동으로 해커에게 연락하는 거예요.
