## 핵심 인사이트 (3줄 요약)
> 1. **본질**: FileVault는 macOS의 기본 FDE (Full Disk Encryption)로, APFS (Apple File System) 볼륨 전체를 암호화하여 로그인 전에는 디스크 내용을 읽지 못하게 한다.
> 2. **가치**: Apple ID/복구 키/기관 복구 키와 연동되어, MacBook 분실 시 저장 데이터 유출을 막으면서도 사용자 로그인 경험을 크게 해치지 않는다.
> 3. **판단 포인트**: Apple Silicon과 T2 탑재 환경에서는 Secure Enclave가 키 보호를 강화하므로, 단순 소프트웨어 암호화가 아니라 하드웨어와 결합된 보호 체계로 봐야 한다.

---

## Ⅰ. 개요 및 필요성

MacBook은 이동성이 높아 분실·도난 위험이 크다. 특히 디자인, 개발, 기획 등 다양한 업무에서 로컬 파일과 브라우저 세션, 인증 토큰이 함께 존재하므로, 디스크만 탈취돼도 피해가 커질 수 있다. FileVault는 이런 위험을 줄이기 위해 macOS에 내장된 전체 드라이브 암호화를 제공한다.

FileVault의 핵심은 사용자가 평소 로그인할 때 쓰는 자격 증명과 디스크 해제 과정을 자연스럽게 연결하는 것이다. 즉 "한 번 더 별도 비밀번호를 외우는 보안"이 아니라, **일상 로그인과 디스크 해제를 묶는 Apple식 UX 보안**이다.

```text
┌──────────────────────────────────────────────────────────────┐
│         FileVault: 로그인 자격 증명과 디스크 해제 결합       │
├──────────────────────────────────────────────────────────────┤
│  로그인 비밀번호 / 복구 키 / 기관 복구 키                    │
│                ↓                                            │
│           FileVault 잠금 해제                               │
│                ↓                                            │
│      APFS 볼륨 전체가 읽기 가능한 상태로 전환               │
└──────────────────────────────────────────────────────────────┘
```

이 구조는 사용자에게 거의 보이지 않지만, 분실된 Mac에서 하드웨어를 분리해 외부 분석하는 공격에는 강하다.

- **📢 섹션 요약 비유**: FileVault는 집 현관 비밀번호와 금고 비밀번호를 같은 카드에 묶어 놓는 것과 같다. 평소엔 편하지만, 카드 없이는 아무것도 열 수 없다.

## Ⅱ. 아키텍처 및 핵심 원리

FileVault는 APFS 볼륨과 키 체계를 중심으로 동작한다. 세부 구현은 macOS 버전과 하드웨어에 따라 다르지만, 실무 관점에서는 "사용자 비밀번호 또는 복구 키 → 볼륨 키 해제 → APFS 전체 암호화"로 이해하면 충분하다.

| 구성 요소 | 역할 | 비고 |
|:---|:---|:---|
| **APFS Volume** | 암호화 대상 | 현대 macOS의 기본 파일 시스템 |
| **Volume Key** | 볼륨 암호화 키 | 로그인/복구 단계에서 해제 |
| **Secure Enclave** | 하드웨어 키 보호 | Apple Silicon/T2 환경 강화 |
| **Recovery Key** | 복구용 키 | 개인/기관 에스크로 가능 |

```text
┌──────────────────────────────────────────────────────────────┐
│                 FileVault 키 해제 흐름                       │
├──────────────────────────────────────────────────────────────┤
│  1. Mac 전원 ON                                             │
│  2. 초기 로그인 화면에서 자격 증명 입력                     │
│  3. Secure Enclave / 보안 하드웨어가 키 검증                │
│  4. Volume Key 해제                                         │
│  5. APFS 볼륨 복호화 후 정상 로그인                         │
└──────────────────────────────────────────────────────────────┘
```

FileVault는 자동으로 시스템을 잠그고 푸는 방식이지만, 계정 비밀번호 분실이나 조직 정책상 복구가 필요할 때를 대비해 복구 키 관리가 필수다. 특히 기업 환경에서는 개인 계정 복구보다 **기관 복구 키(escrow)**가 더 중요하다.

- **📢 섹션 요약 비유**: FileVault는 자동차 키와 시동 키를 하나로 만든 스마트키다. 편하지만, 잃어버리면 복구용 예비키를 반드시 따로 가져야 한다.

## Ⅲ. 비교 및 연결

| 항목 | FileVault | BitLocker | LUKS |
|:---|:---|:---|:---|
| 플랫폼 | macOS | Windows | Linux |
| 하드웨어 결합 | Secure Enclave | TPM | TPM/FIDO2 가능 |
| 사용자 경험 | 로그인과 자연스럽게 결합 | 정책 선택 폭 넓음 | 관리 유연성 높음 |
| 복구 방식 | 복구 키 / 기관 키 | Recovery Key | 복구 헤더/키 |
| 관리 포인트 | Apple ID/MDM 정책 | AD/Azure AD/MDM | crypttab/키 슬롯 |

FileVault는 Apple 생태계에 깊게 통합되어 있다는 점이 강점이다. 반면 범용성은 BitLocker나 LUKS보다 낮다. 하지만 Mac 환경에서는 이 통합성이 오히려 장점이다. 사용자는 별도 보안 절차를 덜 느끼면서도 저장 데이터 보호를 얻는다.

또한 FileVault는 401 FDE의 대표 사례이고, 동시에 834 Encryption at Rest의 macOS 구현 예시다. 따라서 이 문서는 "맥에서 어떻게 FDE가 운영되는가"를 보여 주는 실전 예제로 보면 된다.

- **📢 섹션 요약 비유**: FileVault는 애플 전용 자물쇠다. 다른 회사 자물쇠보다 호환성은 낮지만, 애플 집 안에서는 가장 자연스럽고 잘 맞는다.

## Ⅳ. 실무 적용 및 기술사 판단

### 체크리스트

1. FileVault 복구 키를 개인/기관 에스크로로 분리 보관했는가?
2. 조직 소유 Mac의 경우 MDM으로 활성화 상태를 강제하는가?
3. Apple Silicon/T2 장비의 Secure Enclave 전제 조건을 이해했는가?
4. 사용자 계정 비밀번호 정책과 FileVault 정책이 일치하는가?

### 안티패턴

- 복구 키 없이 무작정 FileVault만 활성화
- 직원이 개인 Apple ID에만 복구 키를 맡기게 두는 운영
- 장비 반납 시 FileVault 해제 절차 없이 초기화
- MDM과 충돌하는 로컬 계정 정책을 방치

### 기술사 판단

FileVault는 "macOS에서는 그냥 켜는 것이 맞다" 수준의 표준 보안이지만, 기업에서는 복구 키와 계정 정책이 핵심이다. 기술사 답안에서는 Apple 생태계 통합과 Secure Enclave의 장점을 언급하되, **에스크로와 운영 절차가 없으면 오히려 장애가 된다**는 점까지 적어야 한다.

- **📢 섹션 요약 비유**: FileVault는 애플 기기의 안전벨트이자 에어백이다. 하지만 에어백이 있어도 정비소가 비상문 키를 잃어버리면 수리할 수 없다.

## Ⅴ. 기대효과 및 결론

FileVault는 맥 사용자에게 거의 마찰 없는 전체 드라이브 보호를 제공한다. 사용자 입장에서는 평소처럼 로그인하지만, 분실·도난·오프라인 분석 상황에서는 저장 데이터가 평문으로 드러나지 않는다.

결론적으로 FileVault는 Apple 생태계에서 가장 자연스러운 FDE이며, 하드웨어·OS·사용자 경험을 동시에 고려한 실용적 저장 보호 방식이다.

- **📢 섹션 요약 비유**: FileVault는 맥북 속 비밀일기를 잠그는 자동 자물쇠다. 평소엔 편리하고, 잃어버리면 강력하다.
