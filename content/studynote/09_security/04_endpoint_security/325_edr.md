## 핵심 인사이트 (3줄 요약)
> 1. **본질**: EDR(Endpoint Detection and Response)은 엔드포인트에서 발생하는 모든 이벤트를 지속 수집·분석하여 위협을 탐지하고, 격리·포렌식·복구까지 자동 대응하는 차세대 보안 플랫폼이다.
> 2. **가치**: AV/EPP가 놓친 침해를 사후에 탐지하고, 전체 공격 타임라인을 재구성하여 SOC 분석가가 신속하게 조치할 수 있는 가시성과 대응 도구를 제공한다.
> 3. **판단 포인트**: MTTD(Mean Time to Detect)·MTTR(Mean Time to Respond) 단축이 핵심 KPI이며, MITRE ATT&CK Evaluations 결과가 EDR 성능 비교의 표준 기준이다.

---

## Ⅰ. 개요 및 필요성

EDR(Endpoint Detection and Response)은 2013년 Gartner의 Anton Chuvakin이 처음 정의한 개념으로, 엔드포인트에서 발생하는 이벤트를 지속적으로 모니터링·기록하고, 위협을 탐지·조사·대응하는 통합 솔루션이다. AV/EPP가 위협을 "예방"하는 데 집중한다면, EDR은 예방을 뚫고 들어온 위협을 "탐지·대응"하는 데 집중한다.

APT(Advanced Persistent Threat) 공격은 수개월에 걸쳐 조용히 환경 내에서 활동한다. 전통적 AV로는 이런 은밀한 활동을 탐지하기 어렵다. EDR은 프로세스 트리, 파일 접근, 레지스트리 변경, 네트워크 연결 등 수천 가지 이벤트를 지속 수집하여 공격자의 행적을 타임라인으로 재구성한다. 이를 통해 SOC(Security Operations Center) 분석가가 침해 범위를 빠르게 파악하고 격리 조치를 취할 수 있다.

EDR의 4대 핵심 기능은 ① 지속적 모니터링(Continuous Monitoring), ② 위협 탐지(Threat Detection), ③ 사고 대응(Incident Response), ④ 위협 헌팅(Threat Hunting)이다. 현대 EDR은 이 기능들을 단일 에이전트와 클라우드 백엔드로 통합하여 제공한다.

📢 **섹션 요약 비유**: EDR은 침입자가 들어왔을 때 발자국을 모두 추적하고, 어디서 무엇을 했는지 CCTV 영상처럼 재생해주는 보안 수사관이다.

## Ⅱ. 아키텍처 및 핵심 원리

### EDR 핵심 기능 구성

| 기능 | 세부 내용 | 비고 |
|:---|:---|:---|
| 이벤트 수집 | 프로세스·파일·레지스트리·네트워크 | 커널 드라이버 기반 |
| 위협 탐지 | 행위 규칙 + ML 모델 | MITRE ATT&CK 매핑 |
| 자동 대응 | 프로세스 종료·네트워크 격리 | Playbook 자동화 |
| 포렌식 수집 | 메모리 덤프·로그·IOC 수집 | 사고 조사 지원 |
| 위협 헌팅 | 쿼리 기반 탐색 | 분석가 능동 조사 |

```
┌───────────────────────────────────────────────────────┐
│                   EDR 아키텍처                        │
├────────────────────┬──────────────────────────────────┤
│   단말 에이전트    │         클라우드 백엔드           │
│                    │                                   │
│  커널 드라이버     │  이벤트 저장소 (Data Lake)        │
│  이벤트 수집       │  위협 분석 엔진 (ML/Rules)        │
│  행위 차단         │  타임라인 재구성                  │
│  포렌식 수집       │  SOC 대시보드                     │
│                    │  자동 대응 Playbook               │
└────────────────────┴──────────────────────────────────┘
```

EDR 에이전트는 운영체제 커널에 드라이버를 설치하여 최저 수준에서 이벤트를 수집한다. 수집된 이벤트는 단말에서 1차 필터링 후 클라우드 백엔드로 전송된다. 클라우드에서는 수개월치 이벤트를 저장하여 사고 발생 후 소급 분석(retrospective analysis)이 가능하다.

📢 **섹션 요약 비유**: EDR의 클라우드 백엔드는 블랙박스처럼, 평소에는 조용히 모든 것을 기록하다가 사고 발생 시 결정적 증거를 제공한다.

## Ⅲ. 비교 및 연결

| 구분 | AV/EPP | EDR | XDR |
|:---|:---|:---|:---|
| 주요 목표 | 예방 | 탐지·대응 | 통합 가시성 |
| 가시성 범위 | 단말(파일) | 단말(모든 이벤트) | 단말+네트워크+클라우드 |
| 포렌식 | 없음 | 상세 타임라인 | 크로스-도메인 타임라인 |
| 위협 헌팅 | 없음 | 가능 | 고급 가능 |
| 자동 대응 | 파일 격리 | 프로세스·호스트 격리 | 오케스트레이션 |

EDR은 SIEM(Security Information and Event Management)과 상호 보완적이다. SIEM이 전사 로그를 통합 분석한다면, EDR은 단말 이벤트에 특화된 고해상도 분석을 제공한다. 최근에는 EDR 데이터를 SIEM에 피드하거나, XDR 플랫폼으로 통합하는 추세다.

📢 **섹션 요약 비유**: AV가 문지기(예방), EDR이 형사(탐지·수사·대응), SIEM이 지휘 본부(통합 분석)라면, XDR은 전 지역 통합 수사팀이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

EDR 도입 시 핵심 고려사항은 ① MITRE ATT&CK Evaluations 성능 점수, ② 에이전트 성능 오버헤드(CPU 사용률 3% 이하 권장), ③ 데이터 보존 기간(90일~1년), ④ SOAR(Security Orchestration, Automation and Response) 연동 수준이다. 특히 자동 격리(auto-isolation) 기능은 랜섬웨어 확산 방지에 결정적이나, 오탐 시 업무 중단 위험이 있어 정책 수립이 중요하다.

기술사 답안에서는 EDR을 "침해 전 예방(EPP)을 보완하는 침해 후 탐지·대응의 핵심 플랫폼"으로 정의하고, MTTD·MTTR 단축 효과와 위협 헌팅을 통한 능동적 보안 운영을 강점으로 서술해야 한다.

📢 **섹션 요약 비유**: EDR 없는 보안팀은 범죄 현장 CCTV 없이 수사하는 형사처럼—단서가 없어 범인을 잡기 어렵다.

---

## Ⅴ. 기대효과 및 결론

EDR 도입으로 평균 탐지 시간(MTTD)을 수개월에서 수분으로 단축하고, 자동 격리를 통해 랜섬웨어·APT의 횡적 이동(lateral movement)을 차단할 수 있다. 포렌식 기능은 사고 발생 후 공격 경로와 범위를 신속하게 파악하여 재발 방지 대책 수립을 지원한다.

MITRE ATT&CK 프레임워크와의 통합은 단순 탐지를 넘어 공격자의 TTP(Tactics, Techniques, Procedures)를 이해하는 위협 인텔리전스로 발전시킨다. 향후 EDR은 AI 기반 자율 대응과 XDR로의 확장을 통해 보안 운영의 지능화·자동화를 주도할 것이다.

📢 **섹션 요약 비유**: EDR은 사고 발생 시 블랙박스 영상을 바로 재생해주는 시스템—무엇이 어떻게 일어났는지 명확히 보여줘 빠른 수습을 가능하게 한다.

### 📌 관련 개념 맵
| 개념 | 관계 | 설명 |
|:---|:---|:---|
| EPP (Endpoint Protection Platform) | 보완 관계 | 예방 담당, EDR은 탐지·대응 담당 |
| MITRE ATT&CK | 평가 기준 | EDR 성능 평가 표준 프레임워크 |
| SOAR (Security Orchestration, Automation and Response) | 연동 기술 | EDR 탐지 후 자동 대응 오케스트레이션 |
| XDR (Extended Detection and Response) | 확장 방향 | EDR의 다영역 통합 확장 |
| 위협 헌팅 (Threat Hunting) | 활용 방법 | EDR 데이터 기반 능동적 위협 탐색 |

### 👶 어린이를 위한 3줄 비유 설명
- EDR은 집 안에 설치된 블랙박스야—누가 들어와서 무엇을 했는지 모두 기록해.
- 나쁜 사람이 들어왔을 때 자동으로 문을 잠그고(격리), 경찰에 신고도 해줘.
- 나중에 수사할 때 "몇 시에 어디서 무엇을 했는지" 영상처럼 보여줄 수 있어!
