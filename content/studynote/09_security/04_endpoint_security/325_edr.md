## 핵심 인사이트 (3줄 요약)
> 1. **본질**: EDR(Endpoint Detection and Response)은 엔드포인트에서 발생하는 모든 이벤트를 지속적으로 수집·기록하고, 위협을 탐지·조사·대응하는 통합 보안 플랫폼이다.
> 2. **가치**: 침해 발생 후 "어떻게, 언제, 무엇이" 일어났는지 타임라인으로 재구성하는 포렌식 능력이 핵심 가치이며, APT(Advanced Persistent Threat) 대응의 필수 도구다.
> 3. **판단 포인트**: EDR의 실용성은 텔레메트리(Telemetry) 수집 범위와 자동화된 대응(SOAR 연동) 품질에 달려 있으며, 솔루션 선택 시 MITRE ATT&CK 평가 결과가 객관적 지표가 된다.

---

## Ⅰ. 개요 및 필요성

EDR(Endpoint Detection and Response)이라는 용어는 2013년 가트너 애널리스트 안톤 추바킨(Anton Chuvakin)이 처음 제안했다. 기존 EPP(Endpoint Protection Platform)가 위협 예방에 집중했다면, EDR은 "침해는 필연적으로 발생한다"는 가정 아래 탐지와 사후 대응을 핵심으로 설계되었다.

EDR이 등장한 배경에는 APT(Advanced Persistent Threat)의 급증이 있다. APT는 수개월~수년에 걸쳐 잠복하며 가치 있는 데이터를 탈취하는 고도화된 공격이다. 기존 AV는 이런 공격의 초기 침투 도구는 막지 못하고, 설령 막더라도 공격자가 이미 내부 시스템에 자리 잡은 후에야 발견되는 경우가 많았다. EDR은 모든 이벤트를 지속적으로 기록(Continuous Recording)함으로써 사후 포렌식을 가능하게 한다.

현대 EDR은 단순 탐지를 넘어 **자동화된 대응(Automated Response)** 까지 수행한다. 감염된 엔드포인트를 네트워크에서 자동 격리(Isolation)하고, 악성 프로세스를 종료하며, 피해 파일을 복구하는 작업이 수 초 내에 자동으로 이루어진다. 이를 통해 MTTR(Mean Time To Respond)을 수일에서 수 분으로 단축할 수 있다.

📢 **섹션 요약 비유**: EDR은 "블랙박스가 달린 스마트 보안관"이다. 24시간 모든 상황을 녹화하고, 이상이 생기면 즉시 출동하며, 사건 이후에는 녹화본으로 범인을 추적한다.

## Ⅱ. 아키텍처 및 핵심 원리

### EDR 핵심 기능 구성

| 기능 | 내용 | 중요도 |
|:----|:-----|:------|
| 지속적 수집 | 프로세스·파일·네트워크·레지스트리 이벤트 기록 | 최상 |
| 위협 탐지 | ML·행위 규칙·IOC(Indicator of Compromise) 매칭 | 최상 |
| 자동 대응 | 격리·프로세스 종료·파일 삭제 | 상 |
| 포렌식/조사 | 타임라인·프로세스 트리·네트워크 흐름 시각화 | 최상 |
| 위협 인텔리전스 | 외부 TI(Threat Intelligence) 피드 연동 | 상 |

```
┌──────────────────────────────────────────────────┐
│                 EDR 아키텍처                     │
│                                                  │
│  엔드포인트 (Endpoint)                           │
│  ┌────────────────────────────────────────────┐  │
│  │  EDR 에이전트                              │  │
│  │  ┌──────────┐  ┌──────────┐  ┌──────────┐ │  │
│  │  │이벤트 수집│  │행위 분석 │  │자동 대응 │ │  │
│  │  └────┬─────┘  └────┬─────┘  └────┬─────┘ │  │
│  └───────┼─────────────┼─────────────┼───────┘  │
│          │ 텔레메트리   │             │ 격리 명령 │
│          ▼             ▼             ▼           │
│  ┌────────────────────────────────────────────┐  │
│  │          EDR 백엔드 (클라우드/온프레미스)   │  │
│  │  데이터 저장소 │ 분석 엔진 │ 관리 콘솔     │  │
│  └────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

EDR의 핵심은 **텔레메트리(Telemetry)** 수집이다. 에이전트는 커널 레벨에서 모든 시스템 이벤트를 후킹하여 중앙 저장소로 전송한다. 이 데이터는 수개월간 보존되어 인시던트 발생 시 타임라인 재구성에 활용된다. 저장되는 데이터 유형에는 프로세스 생성/종료, 파일 읽기/쓰기, 네트워크 연결, 레지스트리 수정, 사용자 로그인 등이 포함된다.

📢 **섹션 요약 비유**: EDR 텔레메트리 수집은 "CCTV + 출입 기록 + 통화 내역을 모두 남기는 건물 보안 시스템"과 같다. 사건 발생 후 언제, 누가, 무엇을 했는지 정확하게 재구성한다.

## Ⅲ. 비교 및 연결

### EDR vs EPP vs XDR 비교

| 특성 | EPP | EDR | XDR |
|:----|:----|:----|:----|
| 초점 | 예방 | 탐지·대응 | 통합 탐지·대응 |
| 커버리지 | 엔드포인트 | 엔드포인트 | 엔드포인트+네트워크+클라우드 |
| 포렌식 | 제한적 | 상세 | 상관 분석 |
| 자동화 | 낮음 | 중간 | 높음 |
| 운영 복잡도 | 낮음 | 중간 | 높음 |

EDR은 SIEM(Security Information and Event Management), SOAR(Security Orchestration, Automation, and Response)와 연동하여 더 넓은 보안 컨텍스트를 활용한다. MDR(Managed Detection and Response) 서비스는 EDR을 외부 보안 전문가가 24/7 운영하는 서비스 모델이다.

📢 **섹션 요약 비유**: EPP가 "방범창"이고 EDR이 "CCTV + 보안 요원"이라면, XDR은 "도시 전체 통합 안전 시스템"이다.

## Ⅳ. 실무 적용 및 기술사 판단

**EDR 도입 및 운영 시 핵심 사항:**
- **MITRE ATT&CK 평가**: 오픈 평가 결과를 참고하여 탐지 커버리지 객관적 비교
- **Hunting 능력**: 알람 없이도 능동적으로 위협을 찾는 Threat Hunting 기능 지원 여부
- **스토리지 설계**: 텔레메트리 데이터 보존 기간(최소 90일, 권고 1년) 및 비용 계획
- **오케스트레이션**: SOAR와 연동하여 반복적인 대응 작업 자동화

**위협 헌팅(Threat Hunting) 활용:**
- IOC(Indicator of Compromise) 및 IOA(Indicator of Attack) 기반 검색
- 의심 프로세스의 부모-자식 관계 트리 분석
- 네트워크 비컨(Beaconing) 패턴 탐지

📢 **섹션 요약 비유**: 위협 헌팅은 "알람이 울리기 전에 먼저 순찰 나가는 보안 요원"이다. 수동적으로 기다리는 것이 아니라 능동적으로 위협을 찾아나선다.

## Ⅴ. 기대효과 및 결론

EDR을 도입하면 MTTD(Mean Time To Detect)를 현재 평균 207일에서 수 시간으로 단축할 수 있으며, 침해 사고의 피해 범위를 90% 이상 축소할 수 있다. 또한 상세한 포렌식 데이터는 법적 대응 및 컴플라이언스 감사에 직접 활용된다.

결론적으로, EDR은 현대 엔드포인트 보안의 중심축이다. 단순 예방을 넘어 지속적인 탐지·대응·조사 능력을 제공하며, APT와 같은 고도화 위협에 대응하는 데 있어 현재까지 가장 실질적인 솔루션이다. XDR로의 확장을 고려한 플랫폼 전략이 장기적으로 바람직하다.

📢 **섹션 요약 비유**: EDR은 "블랙박스 달린 법정 증인"이다. 사건 현장의 모든 것을 기록해 두었다가, 범죄 발생 시 정확한 증언을 제공한다.

### 👶 어린이를 위한 3줄 비유 설명
1. EDR은 학교 복도에 CCTV를 설치하고 모든 움직임을 녹화하는 것과 같아.
2. 누가 나쁜 짓을 하면 바로 감지하고 선생님(보안팀)에게 알려줘.
3. 나중에 "어떻게 일이 일어났는지" 녹화 영상으로 정확하게 확인할 수 있어.
