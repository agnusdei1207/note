## 핵심 인사이트 (3줄 요약)
> 1. **본질**: TTP(Tactics, Techniques, Procedures)는 공격자가 목표를 달성하기 위해 사용하는 전술·기술·세부 절차의 계층적 분류 체계이며, MITRE ATT&CK 프레임워크의 핵심 구성 요소다.
> 2. **가치**: IOC(Indicator of Compromise)는 침해 후 변경되지만 TTP는 공격자의 행동 패턴이므로 쉽게 바뀌지 않아, 위협 인텔리전스(Threat Intelligence)와 탐지 규칙 설계의 가장 지속적인 근거가 된다.
> 3. **판단 포인트**: 단순 IOC 기반 탐지에서 TTP 기반 탐지로 전환하면 파이라미드 오브 페인(Pyramid of Pain) 상단에서 공격자를 압박할 수 있어 방어 효과가 극대화된다.

---

## Ⅰ. 개요 및 필요성

TTP(Tactics, Techniques, Procedures)는 사이버 보안에서 공격자의 행동 방식을 3계층으로 체계화한 개념이다. 군사 전략에서 차용된 용어로, 보안 커뮤니티에서는 주로 MITRE ATT&CK(Adversarial Tactics, Techniques, and Common Knowledge) 프레임워크를 통해 구체화된다.

- **전술(Tactics)**: 공격자의 단기 목표 (예: 초기 접근, 지속성 확보, 데이터 탈취)
- **기술(Techniques)**: 전술을 달성하는 방법 (예: 스피어 피싱, WMI(Windows Management Instrumentation) 실행, LSASS 덤프)
- **절차(Procedures)**: 특정 그룹이나 도구가 기술을 구현하는 구체적 단계 (예: APT29가 Cobalt Strike를 사용하는 특정 방식)

TTP 기반 위협 탐지가 중요한 이유는 **공격자의 변경 비용(Attacker's Cost)**에 있다. 해시값이나 IP 주소 같은 IOC는 쉽게 변경 가능하지만, 공격자의 TTP를 바꾸려면 툴킷 전체를 교체하고 새로운 기술을 습득해야 하므로 훨씬 어렵다. 이를 시각화한 것이 데이비드 비앙코(David Bianco)의 "파이라미드 오브 페인"이다.

📢 **섹션 요약 비유**: TTP는 "범인의 범행 수법 프로파일"이다. 얼굴(IOC)은 바꿀 수 있지만, 범행 수법(TTP)은 쉽게 바꾸지 못해 재범 시 잡을 수 있다.

## Ⅱ. 아키텍처 및 핵심 원리

### MITRE ATT&CK 전술(Tactics) 목록

| 전술 ID | 전술명 | 목표 |
|:--------|:------|:-----|
| TA0001 | Initial Access | 네트워크 최초 침투 |
| TA0002 | Execution | 악성 코드 실행 |
| TA0003 | Persistence | 지속 접근 확보 |
| TA0005 | Defense Evasion | 탐지 우회 |
| TA0007 | Discovery | 내부 환경 정찰 |
| TA0008 | Lateral Movement | 내부 확산 |
| TA0011 | Command and Control | C2 채널 구성 |
| TA0010 | Exfiltration | 데이터 탈취 |

```
┌────────────────────────────────────────────────────┐
│            파이라미드 오브 페인 (Pyramid of Pain)   │
│                                                    │
│      ▲  (탐지 시 공격자 피해 큼)                   │
│      │                                             │
│  ┌───┴────────────────────────────────────────┐   │
│  │   TTP (전술·기술·절차)      ← 최고 가치     │   │
│  ├─────────────────────────────────────────────┤   │
│  │   도구 (Tools) - 악성코드, C2 프레임워크     │   │
│  ├─────────────────────────────────────────────┤   │
│  │   네트워크/호스트 아티팩트                   │   │
│  ├─────────────────────────────────────────────┤   │
│  │   도메인명                                   │   │
│  ├─────────────────────────────────────────────┤   │
│  │   IP 주소                                    │   │
│  ├─────────────────────────────────────────────┤   │
│  │   해시값 (Hash)             ← 최저 가치      │   │
│  └─────────────────────────────────────────────┘   │
│      │                                             │
│      ▼  (탐지 시 공격자 피해 작음)                 │
└────────────────────────────────────────────────────┘
```

TTP 기반 탐지 규칙 설계 시 MITRE ATT&CK의 각 기술(Technique) 번호(T-number)를 탐지 규칙에 태깅한다. 이를 통해 현재 탐지 규칙이 14개 전술 중 어느 영역을 커버하는지, 어떤 공백이 있는지를 히트맵(Heat Map)으로 시각화할 수 있다.

📢 **섹션 요약 비유**: TTP 기반 탐지는 "수법 데이터베이스 구축"이다. 범인이 위장해도 손 쓰는 방식, 도구 사용 습관을 알면 꼬리를 잡을 수 있다.

## Ⅲ. 비교 및 연결

### IOC vs TTP 기반 탐지 비교

| 특성 | IOC 기반 | TTP 기반 |
|:----|:---------|:---------|
| 탐지 대상 | 알려진 악성 지표 | 공격 행위 패턴 |
| 지속성 | 낮음 (쉽게 변경) | 높음 (변경 어려움) |
| 오탐률 | 낮음 | 보통 |
| 미지 위협 대응 | 불가 | 부분 가능 |
| 구현 복잡도 | 낮음 | 높음 |

TTP는 위협 인텔리전스 공유 표준인 STIX(Structured Threat Information Expression)와 TAXII(Trusted Automated eXchange of Intelligence Information)를 통해 조직 간 공유된다. MISP(Malware Information Sharing Platform)는 오픈소스 TI 공유 플랫폼으로, TTP 기반 IOC를 커뮤니티와 공유한다.

📢 **섹션 요약 비유**: TTP 공유는 "경찰서 간 범죄 수법 공유"와 같다. 한 지역에서 발견된 새 수법을 전국 경찰에 알리면 다른 지역도 대비할 수 있다.

## Ⅳ. 실무 적용 및 기술사 판단

**TTP 기반 탐지 규칙 설계 프로세스:**
1. MITRE ATT&CK에서 관련 기술(T-number) 식별
2. 해당 기술의 데이터 소스(Data Source) 확인
3. 탐지 쿼리 작성 (Sigma 규칙 또는 SIEM 쿼리)
4. 테스트 환경에서 오탐 검증
5. 탐지 규칙에 T-number 태깅 후 운영 배포

**활용 도구:**
- **ATT&CK Navigator**: 커버리지 히트맵 시각화
- **Sigma**: 플랫폼 독립적 탐지 규칙 작성 언어
- **MISP**: TI 공유 플랫폼

📢 **섹션 요약 비유**: TTP 탐지 규칙 설계는 "수사 매뉴얼 작성"과 같다. 어떤 상황에서 어떤 단서를 찾을지 사전에 정의해 두면 실제 사건에서 빠르게 대응할 수 있다.

---

## Ⅴ. 기대효과 및 결론

TTP 기반 탐지 체계를 구축하면 제로데이 공격 포함 미지의 위협에 대한 탐지율을 30~50% 향상시킬 수 있으며, 공격자의 피벗(Pivot) 비용을 높여 공격 자체를 억제하는 억지력 효과가 있다. MITRE ATT&CK 평가에서 높은 점수를 받는 EDR 솔루션을 선택하는 것이 TTP 기반 탐지 전략의 시작점이다.

결론적으로, TTP는 현대 위협 인텔리전스와 탐지 체계의 핵심 언어다. IOC를 넘어 TTP로 사고를 확장하는 것이 차세대 보안 운영의 핵심 역량이다.

📢 **섹션 요약 비유**: TTP 기반 보안은 "범죄자의 DNA 프로파일링"이다. 겉모습이 바뀌어도 DNA(TTP)는 변하지 않아 추적이 가능하다.

### 👶 어린이를 위한 3줄 비유 설명
1. TTP는 도둑이 어떻게 문을 따는지, 어디를 먼저 뒤지는지 같은 "범행 수법"이야.
2. 얼굴은 바꿀 수 있지만, 수법은 쉽게 못 바꾸니까 수법을 알면 잡기 쉬워.
3. 전 세계 경찰이 수법을 공유하면 어디서 범행해도 잡힐 확률이 높아져.
