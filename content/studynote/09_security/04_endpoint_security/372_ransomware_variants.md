## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CryptoLocker·WannaCry·Ryuk는 랜섬웨어 진화의 3대 이정표로, 각각 비트코인 결제 시스템, 웜 전파 방식, 표적 기업 침투 전략을 대중화한 분수령 악성코드다.
> 2. **가치**: 이 세 변종의 공격 기법을 이해하면 랜섬웨어 생태계의 기술적·비즈니스적 진화를 총체적으로 파악할 수 있으며, 방어 전략의 기준점이 된다.
> 3. **판단 포인트**: WannaCry는 패치 관리의 중요성, Ryuk는 네트워크 세그멘테이션·백업 격리의 중요성을 각각 증명한 실증 사례다.

---

## Ⅰ. 개요 및 필요성

랜섬웨어는 세대를 거듭하며 진화했다. 초기 랜섬웨어는 단순 잠금 화면 방식이었지만, 2013년 CryptoLocker를 기점으로 강력한 비대칭 암호화를 도입했다. 2017년 WannaCry는 전 세계 150개국을 강타하며 랜섬웨어의 재앙적 잠재력을 보여줬고, 2018년 등장한 Ryuk는 Big Game Hunting(대기업 표적 공격) 전략으로 수십억 달러의 피해를 입혔다.

이 세 변종은 단순한 악성코드가 아닌, 사이버 범죄 생태계의 진화를 보여주는 교과서적 사례다. 각각의 혁신이 이후 수백 개의 후속 랜섬웨어에 영향을 미쳤다.

📢 **섹션 요약 비유**: CryptoLocker는 권총, WannaCry는 자동소총, Ryuk는 저격수—같은 목표지만 전략과 사거리가 완전히 다르다.

---

## Ⅱ. 아키텍처 및 핵심 원리

| 변종 | 등장 연도 | 핵심 혁신 | 피해 규모 |
|:---|:---|:---|:---|
| CryptoLocker | 2013 | RSA-2048 + 비트코인 결제 | 약 $3백만 (게임오버 제우스 봇넷 활용) |
| WannaCry | 2017 | EternalBlue 웜 전파 | 150개국, NHS 마비, $4억+ |
| Ryuk | 2018~ | TrickBot 연계 표적 침투 | 병원·언론사, $100M+ |

```text
┌────────────────────────────────────────────────────┐
│              세 변종의 공격 방식 비교              │
├────────────────────────────────────────────────────┤
│                                                    │
│  CryptoLocker (2013)                               │
│  이메일 피싱 → 실행 → RSA 암호화 → 비트코인 요구  │
│                                                    │
│  WannaCry (2017)                                   │
│  SMB 취약점 자동 스캔 → 네트워크 자동 전파          │
│  EternalBlue(MS17-010) → DoublePulsar 백도어       │
│  → WANNACRY 암호화 실행                            │
│                                                    │
│  Ryuk (2018~)                                      │
│  스피어 피싱 → TrickBot 설치 → Cobalt Strike        │
│  → 내부 망 탐색 → 도메인 관리자 획득               │
│  → GPO(그룹 정책)로 전체 배포 → 암호화             │
└────────────────────────────────────────────────────┘
```

**WannaCry의 킬 스위치(Kill Switch)**  
WannaCry 코드에는 특정 도메인(iuqerfsodp9ifjaposdfjhgosurijfaewrwergwea.com)이 등록되지 않았을 때만 실행되는 체크가 있었다. 보안 연구자 Marcus Hutchins가 해당 도메인을 $10.69에 등록해 전파를 대부분 차단했다.

**Ryuk의 표적화 전략**  
Ryuk는 무차별 전파 대신 TrickBot·Emotet 봇넷으로 사전에 선별한 고가치 표적(병원·지자체·대기업)에만 집중 공격한다. 공격 준비에 수주~수개월이 걸리지만 요구 금액이 수백만 달러에 달한다.

📢 **섹션 요약 비유**: WannaCry는 폭발물을 길거리에 뿌리는 테러리스트, Ryuk는 특정 VIP만 노리는 저격수—WannaCry는 규모, Ryuk는 정밀도가 무기다.

## Ⅲ. 비교 및 연결

| 항목 | CryptoLocker | WannaCry | Ryuk |
|:---|:---|:---|:---|
| 전파 방식 | 이메일 피싱 | SMB 웜 (EternalBlue) | TrickBot 연계 표적 침투 |
| 주요 취약점 | 사용자 실수 | MS17-010 미패치 | 스피어 피싱·AD 취약점 |
| 복호화 제공 여부 | 실제 제공 (신뢰 유지 전략) | 설계 결함으로 불가 | 협상 시 제공 |
| 핵심 교훈 | 비트코인 = 추적 어려운 결제 | 패치 지연의 재앙 | 백업 격리 필수 |

📢 **섹션 요약 비유**: CryptoLocker는 납치 후 협상, WannaCry는 무차별 폭격, Ryuk는 기업 헤드헌터처럼 정밀 표적을 골라 공략한다.

---

## Ⅳ. 실무 적용 및 기술사 판단

**WannaCry 교훈 → 패치 관리**  
- 2017년 3월 MS17-010 패치 배포, 5월 WannaCry 발생—2개월이면 조직 전체 패치 가능했음  
- 자동 패치 업데이트 정책 필수  
- 지원 종료(EoL) OS 격리 또는 교체 계획 수립  

**Ryuk 교훈 → 백업 격리**  
- Ryuk는 백업 서버도 암호화—네트워크에 연결된 백업은 의미 없음  
- 오프라인/오프사이트 불변(immutable) 백업 필수  
- AD (Active Directory) 특권 계정 분리·보호  

📢 **섹션 요약 비유**: WannaCry는 "왜 문을 잠그지 않았냐"고 묻고, Ryuk는 "왜 금고를 인터넷에 연결해 뒀냐"고 묻는다.

---

## Ⅴ. 기대효과 및 결론

CryptoLocker→WannaCry→Ryuk의 진화는 랜섬웨어가 단순 악성코드에서 조직화된 사이버 범죄 비즈니스로 성장했음을 보여준다. 기술사 답안에서는 세 변종의 기술적 혁신을 시대 순으로 서술하고, 각각의 교훈(암호화 강도·패치 관리·백업 격리)을 연결하면 높은 완성도를 얻을 수 있다.

📢 **섹션 요약 비유**: 랜섬웨어 역사는 총→핵폭탄→드론 공습의 진화—각 시대마다 방어 전략도 함께 업그레이드되어야 한다.

### 👶 어린이를 위한 3줄 비유 설명
WannaCry는 컴퓨터가 연결된 네트워크를 통해 저절로 퍼지는 무서운 프로그램이었어요.  
Ryuk는 병원이나 큰 회사만 골라서 공격하는 더 영리한 랜섬웨어예요.  
모두 컴퓨터를 잠가버리고 돈을 요구하지만, 방법과 목표가 달라서 각각 다른 방어가 필요해요.
