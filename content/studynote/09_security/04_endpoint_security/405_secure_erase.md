## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Secure Erase는 단순 삭제가 아니라 저장 매체의 복구 가능성을 낮추기 위해 데이터를 안전하게 소거하거나 키를 파괴하는 절차다.
> 2. **가치**: SSD, 노트북, 폐기 서버, 반납 장비처럼 물리적으로 이동하는 자산의 최종 보안을 책임진다.
> 3. **판단 포인트**: HDD와 SSD는 소거 방식이 다르며, SSD는 덮어쓰기보다 Crypto Erase와 Sanitize 명령이 더 중요하다.

---

## Ⅰ. 개요 및 필요성

파일을 지운다고 데이터가 사라지는 것은 아니다. 운영체제는 보통 메타데이터만 지우고 실제 블록은 남겨 둔다. 포렌식 도구는 그 흔적을 다시 읽어낼 수 있다. Secure Erase는 이런 잔존 데이터를 복구하기 어렵게 만드는 절차다.

특히 SSD는 웨어 레벨링(Wear Leveling) 때문에 "같은 위치를 여러 번 덮어쓰기"가 곧바로 소거를 뜻하지 않는다. 그래서 SSD에서는 단순 overwrite보다 장치 자체의 erase 기능이 더 중요하다.

```text
┌──────────────────────────────────────────────────────────────┐
│                  Secure Erase의 목적                        │
├──────────────────────────────────────────────────────────────┤
│  삭제/포맷 → 흔적 잔존 가능                                  │
│  Secure Erase → 복구 가능성 최소화                           │
└──────────────────────────────────────────────────────────────┘
```

Secure Erase는 폐기 보안의 마지막 단계다. 암호화한 뒤 버리는 것이 아니라, 버리기 직전에 복구가 안 되도록 만드는 것이다.

- **📢 섹션 요약 비유**: 연필로 쓴 글을 지우개로 문지르는 것과, 종이를 갈아버리는 것은 다르다. Secure Erase는 후자에 더 가깝다.

---

## Ⅱ. 아키텍처 및 핵심 원리

Secure Erase는 장치 종류에 따라 방식이 다르다. HDD는 물리적 덮어쓰기나 파괴가 중심이고, SSD는 내장 컨트롤러 명령이나 암호화 키 파괴가 핵심이다.

| 방식 | 대상 | 장점 | 한계 |
|:---|:---|:---|:---|
| **Overwrite** | HDD 중심 | 이해하기 쉬움 | SSD 웨어 레벨링에 약함 |
| **ATA Secure Erase** | SATA SSD/HDD | 표준 명령 사용 | 장치 지원 필요 |
| **NVMe Sanitize** | NVMe SSD | 컨트롤러 수준 소거 | 펌웨어 지원 필요 |
| **Crypto Erase** | SED (Self-Encrypting Drive) | 키 삭제만으로 빠름 | 암호화 구조가 전제 |
| **Physical Destroy** | 모든 매체 | 가장 확실함 | 재사용 불가 |

```text
┌──────────────────────────────────────────────────────────────┐
│                SSD Secure Erase 개념도                      │
├──────────────────────────────────────────────────────────────┤
│  사용자 데이터 ──▶ 내부 컨트롤러 ──▶ 암호화 키 삭제         │
│                         │                                     │
│                         └─ 잔존 블록은 있어도 해독 불가       │
└──────────────────────────────────────────────────────────────┘
```

보안 수준이 높은 조직은 NIST SP 800-88 (Guidelines for Media Sanitization)의 `Clear`, `Purge`, `Destroy` 분류를 기준으로 소거 정책을 정한다.

- **📢 섹션 요약 비유**: 지우개, 파쇄기, 소각로를 상황별로 고르는 것과 같다. 모두 "지운다"는 말이지만 결과는 전혀 다르다.

## Ⅲ. 비교 및 연결

| 항목 | 단순 삭제 | Secure Erase | 물리 파괴 |
|:---|:---|:---|:---|
| 복구 가능성 | 높음 | 낮음 | 거의 없음 |
| 비용 | 가장 낮음 | 중간 | 높음 |
| 재사용 가능성 | 높음 | 높음~중간 | 없음 |
| 감사 증적 | 약함 | 강함 | 강함 |

Secure Erase는 "사용 종료 장비"를 다루는 절차이므로, 암호화와 조합되면 효과가 더 커진다. 예를 들어 SED (Self-Encrypting Drive)는 데이터를 상시 암호화하고, 폐기 시 키를 파괴하는 방식으로 빠른 Crypto Erase가 가능하다.

반대로 단순 overwrite는 SSD에서 신뢰하기 어렵다. 그래서 Secure Erase는 저장 매체의 기술적 특성을 먼저 이해해야 한다.

- **📢 섹션 요약 비유**: 종이를 그냥 버리는 것과 불에 태우는 것은 다르다. 무엇을 버리느냐가 아니라, 다시 읽을 수 없게 만드는 방식이 핵심이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

### 선택 기준

1. 장비를 반납하거나 폐기할 때
2. 민감 정보가 담긴 SSD를 재사용하지 않을 때
3. 리스 장비 회수 후 감사 증적이 필요할 때
4. 암호화된 저장 장치를 키 삭제로 즉시 무력화해야 할 때

### 체크리스트

1. 매체 종류(HDD, SSD, NVMe)를 먼저 분류했는가?
2. NIST SP 800-88 기준에 맞는 Clear/Purge/Destroy를 선택했는가?
3. 소거 결과와 담당자 기록이 남는가?
4. 체인 오브 커스터디 (Chain of Custody)를 유지하는가?

### 안티패턴

- 삭제 명령만 믿고 폐기하는 경우
- SSD에 HDD 방식의 overwrite만 반복하는 경우
- 암호화 키가 남아 있는데 파기 완료라고 오인하는 경우
- 폐기 증적 없이 반출하는 경우

기술사 관점에서 Secure Erase는 "디스크를 버리는 마지막 절차"다. 안전하게 지우는 일과 감사 가능한 증적을 남기는 일이 같이 가야 한다.

- **📢 섹션 요약 비유**: 쓰레기통에 넣는 것으로 끝나지 않는다. 찢고, 태우고, 기록까지 남겨야 진짜 버린 것이다.

---

## Ⅴ. 기대효과 및 결론

Secure Erase는 폐기 장비의 잔존 데이터가 외부로 새지 않게 하는 최종 방어선이다. 삭제와 포맷은 편의 기능이지만, Secure Erase는 보안 통제다.

결론적으로 Secure Erase는 SSD 시대의 필수 데이터 소거 절차다. 암호화, 키 파괴, 장치 특성 이해, 감사 증적이 함께 있어야 안전한 폐기가 완성된다.

- **📢 섹션 요약 비유**: 연필을 지우는 수준이 아니라, 공책 자체를 다시 쓸 수 없게 만드는 과정이다.
