## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널 권한 상승은 커널 공간의 취약점(UAF, 오버플로우, 레이스 조건)을 이용해 커널 자료구조(cred, token)를 조작하여 프로세스에 루트/SYSTEM 권한을 부여하는 가장 강력한 로컬 권한 상승이다.
> 2. **가치**: 커널은 시스템의 모든 권한을 관장하므로, 커널 익스플로잇 성공은 모든 보안 경계(AppArmor, SELinux, 컨테이너)를 무력화한다.
> 3. **판단 포인트**: 방어는 커널 패치 즉시 적용, KASLR(커널 ASLR), SMEP/SMAP(커널←사용자 공간 실행·접근 방지), seccomp-bpf로 시스템 콜 필터링이 핵심 4층이다.

---

## Ⅰ. 개요 및 필요성

커널 권한 상승 공격은 사용자 공간(ring 3)에서 커널 공간(ring 0)의 취약점을 트리거하여 현재 프로세스의 자격증명 구조체(`struct cred`)를 수정한다. `cred->uid/euid`를 0으로 바꾸면 즉시 루트 프로세스가 된다. Windows에서는 `EPROCESS.Token`을 SYSTEM 토큰으로 교체하는 "token stealing" 기법이 동일 목적을 수행한다.

대표 취약점: ① CVE-2022-0847 DirtyPipe(Linux pipe 버퍼 UAF, 임의 파일 쓰기→SUID 바이너리 교체→루트), ② CVE-2021-4034 PwnKit(pkexec BOF), ③ CVE-2023-0179(netfilter UAF), ④ Dirty COW(CVE-2016-5195, copy-on-write 레이스 조건). 이 취약점들은 모두 실제 야생에서 악용됐다.

📢 **섹션 요약 비유**: 커널 익스플로잇은 건물의 전기 패널(커널)을 조작해 모든 방의 잠금을 동시에 해제하는 것—가장 강력하고 영향 범위가 가장 넓다.

---

## Ⅱ. 아키텍처 및 핵심 원리

### 커널 LPE 공격 단계 (Linux cred 조작)

| 단계 | 동작 | 설명 |
|:---|:---|:---|
| 1. 커널 주소 누출 | KASLR 우회 | `/proc/kallsyms`, 사이드 채널, infoleak |
| 2. 취약점 트리거 | 커널 UAF/BOF | 커널 힙 손상 또는 임의 쓰기 프리미티브 |
| 3. cred 주소 획득 | `current->cred` | 커널 심볼 또는 힙 스캔 |
| 4. 권한 조작 | uid=0, gid=0 설정 | `commit_creds(prepare_kernel_cred(NULL))` |
| 5. 사용자 공간 복귀 | IRET 또는 swapgs+sysretq | 루트 쉘 획득 |

```
┌─────────────────────────────────────────────────────┐
│         커널 LPE 권한 조작 구조                      │
├─────────────────────────────────────────────────────┤
│  커널 공간(ring 0)                                  │
│  ┌──────────────────────────────────────┐           │
│  │ task_struct (현재 프로세스)          │           │
│  │  └─ cred: uid=1000, euid=1000       │ ← 조작 전 │
│  │  └─ cred: uid=0,    euid=0          │ ← 조작 후 │
│  └──────────────────────────────────────┘           │
│                                                     │
│  사용자 공간(ring 3): getuid() → 0 (루트!)          │
└─────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: `commit_creds()`는 직원 출입 배지(cred)를 CEO 배지로 교체하는 것—배지 한 장 바꾸면 건물 전체에 접근 가능해진다.

## Ⅲ. 비교 및 연결

| 방어 기술 | 설명 | 우회 방법 |
|:---|:---|:---|
| KASLR | 커널 주소 무작위화 | 정보 누출로 베이스 주소 계산 |
| SMEP | 커널에서 사용자 코드 실행 방지 | ROP 기반 우회 |
| SMAP | 커널에서 사용자 메모리 직접 접근 방지 | STAC/CLAC 명령 악용 |
| seccomp-bpf | 허용 시스템 콜 필터링 | 허용 목록 내 취약 syscall 탐색 |

📢 **섹션 요약 비유**: KASLR은 커널 지도를 무작위로 바꾸고, SMEP은 사용자 구역에서 명령을 못 내리게 하고, SMAP은 커널이 사용자 구역 지갑을 직접 열지 못하게 하는 3중 방어이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

커널 보안 강화 실무: ① `CONFIG_SECURITY_YAMA` + ptrace 스코프 제한으로 KASLR 우회를 위한 `/proc/kallsyms` 제한, ② 컨테이너 환경에서 seccomp-bpf 프로파일 적용, ③ gVisor 같은 커널 격리 VM으로 컨테이너-호스트 커널 공격 표면 제거, ④ LKRG(Linux Kernel Runtime Guard)로 cred 조작 실시간 탐지.

기술사 답안: 커널 LPE를 "시스템 최고 권한 탈취의 가장 강력한 경로"로 정의하고, 방어를 하드웨어 레벨(SMEP/SMAP), OS 레벨(KASLR, seccomp), 탐지 레벨(LKRG, EDR)의 3층으로 서술.

📢 **섹션 요약 비유**: LKRG는 직원 배지 교체를 실시간 감시하는 CCTV—배지가 갑자기 CEO로 바뀌면 즉시 경보를 울린다.

---

## Ⅴ. 기대효과 및 결론

커널 LPE 방어의 핵심은 공격자가 커널 주소를 알 수 없게(KASLR), 설령 안다고 해도 사용자 공간에서 커널로 실행 흐름이 넘어오지 않게(SMEP), 커널이 사용자 메모리를 직접 신뢰하지 않게(SMAP) 하는 3중 구조다. 장기적으로 eBPF 검증기 강화, Rust for Linux의 메모리 안전 커널 드라이버가 커널 취약점 근원을 줄이는 방향이다.

📢 **섹션 요약 비유**: 3중 방어(KASLR+SMEP+SMAP)는 금고 위치를 숨기고, 금고 근처 접근을 차단하고, 금고가 스스로 열리지 않게 하는 것—3개 중 하나가 뚫려도 나머지가 방어선을 유지한다.

### 👶 어린이를 위한 3줄 비유 설명
- 커널 권한 상승은 나쁜 사람이 컴퓨터 운영체제의 핵심(커널)에 있는 배지 보관함을 조작해 최고 권한 배지를 훔치는 거야.
- 배지 한 장만 바꾸면 모든 방(자원)에 들어갈 수 있어서 아주 위험해.
