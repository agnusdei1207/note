## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 백도어(Backdoor)는 정상 인증 절차를 우회해 시스템에 재진입할 수 있는 숨겨진 접근 경로로, 공격자가 초기 침투 후 영속적 접근을 유지하기 위해 설치하는 은닉 메커니즘이다.
> 2. **가치**: APT 킬체인(Kill Chain)의 '설치(Installation)'와 'C2 (Command & Control)' 단계를 연결하는 핵심 요소로, 탐지·제거 후에도 대안 백도어(fallback backdoor)를 통해 재침투한다.
> 3. **판단 포인트**: 백도어는 의도적 삽입(공격자)과 우발적 취약점(개발자 실수)으로 나뉘며, 탐지를 위해서는 네트워크 비정상 트래픽 분석과 파일 무결성 모니터링(FIM)이 병행되어야 한다.

---

## Ⅰ. 개요 및 필요성

백도어(Backdoor)는 시스템 개발자나 공격자가 정상적인 보안 통제를 우회해 접근할 수 있도록 만든 숨겨진 진입점이다. 역사적으로는 개발·유지보수 편의를 위해 의도적으로 삽입된 경우(예: 라우터 기본 관리 계정)도 있지만, 현대 사이버 보안에서는 주로 악성 행위자가 초기 침투 이후 영속성 확보를 위해 설치하는 악성코드 구성 요소를 의미한다.

백도어의 위험성은 "조용함"에 있다. 정상 트래픽으로 위장하거나, 합법적 프로세스에 DLL 인젝션(DLL Injection)을 통해 숨거나, 암호화된 채널(HTTPS, DNS 터널링)을 사용해 C2 통신을 은닉한다. SolarWinds 공급망 공격(2020)에서 SUNBURST 백도어는 9개월 이상 탐지되지 않았다.

📢 **섹션 요약 비유**: 백도어는 건물 설계도에 표시되지 않은 비상구—경비원(보안 솔루션)은 그 문의 존재 자체를 모른다.

---

## Ⅱ. 아키텍처 및 핵심 원리

| 유형 | 특징 | 대표 기법 |
|:---|:---|:---|
| 원격 접속 백도어 | 공격자 명령 수신·실행 | RAT (Remote Access Trojan) |
| 웹셸(Webshell) | 웹 서버에 업로드된 스크립트 | China Chopper, WSO Shell |
| 바인드 셸(Bind Shell) | 포트 열고 연결 대기 | netcat -l -p 4444 |
| 리버스 셸(Reverse Shell) | 피해자가 공격자에게 역접속 | 방화벽 우회 가능 |
| 공급망 백도어 | 소프트웨어 빌드 단계 삽입 | SolarWinds SUNBURST |
| 하드웨어 백도어 | 칩·펌웨어에 내장 | 라우터 기본 계정 |

```text
┌──────────────────────────────────────────────────┐
│               백도어 통신 패턴                   │
├──────────────────────────────────────────────────┤
│                                                  │
│  [공격자 C2 서버]                                │
│       ▲  │                                       │
│       │  │ 암호화된 명령/응답                    │
│       │  ▼                                       │
│  [피해 시스템 백도어]                            │
│       │                                          │
│  ┌────┴─────────────────────────┐               │
│  │ 은닉 기법                    │               │
│  │ - HTTPS/DNS 터널링           │               │
│  │ - 합법 프로세스에 인젝션     │               │
│  │ - 비표준 포트 사용           │               │
│  │ - 트래픽 주기 랜덤화         │               │
│  └──────────────────────────────┘               │
│                                                  │
│  리버스 셸: 피해자 → 공격자 (방화벽 우회)        │
│  바인드 셸: 공격자 → 피해자 (포트 개방)          │
└──────────────────────────────────────────────────┘
```

**리버스 셸이 선호되는 이유**: 대부분의 방화벽은 인바운드 차단이 기본—리버스 셸은 내부에서 외부로 나가는 아웃바운드 연결을 사용해 방화벽을 우회한다.

📢 **섹션 요약 비유**: 바인드 셸은 "내 방에 들어오세요", 리버스 셸은 "내가 당신 방으로 찾아갈게요"—방화벽은 들어오는 손님만 막는다.

## Ⅲ. 비교 및 연결

| 항목 | 백도어 | 트로이목마 | 루트킷 |
|:---|:---|:---|:---|
| 주요 목적 | 지속 접근 유지 | 초기 악성코드 배달 | 존재 은닉 |
| 네트워크 통신 | C2 연결 필수 | 초기 실행 후 선택 | 불필요할 수 있음 |
| 제거 난이도 | 중간 | 낮음 | 높음 |
| 탐지 방법 | 네트워크 트래픽 분석 | 파일 해시 검사 | 커널 무결성 검사 |

📢 **섹션 요약 비유**: 트로이목마는 적군을 성 안으로 들이는 목마, 백도어는 그 적군이 다음에도 들어올 수 있게 만들어 놓은 비밀 통로다.

---

## Ⅳ. 실무 적용 및 기술사 판단

**탐지 전략**  
1. FIM (File Integrity Monitoring)으로 시스템 파일 변조 감지  
2. NDR (Network Detection & Response)으로 비정상 아웃바운드 연결 탐지  
3. DNS 쿼리 로그 분석—DNS 터널링 패턴(긴 서브도메인, 높은 쿼리 빈도) 탐지  
4. 프로세스 트리 분석—비정상적인 자식 프로세스 생성 탐지  
5. SIEM (Security Information and Event Management) 연동 이상 징후 알림  

**사고 대응**  
- 감염 시스템 네트워크 격리(network isolation) 즉시 실행  
- 메모리 덤프를 통한 인메모리 백도어 포렌식 분석  
- YARA 규칙을 이용한 알려진 백도어 시그니처 스캔  

📢 **섹션 요약 비유**: 백도어 탐지는 집 안 CCTV만으로는 부족—집 밖으로 나가는 전화(아웃바운드 트래픽)도 함께 감시해야 한다.

---

## Ⅴ. 기대효과 및 결론

백도어는 공격자가 지속적 접근을 유지하는 핵심 도구로, 단순한 악성코드 제거로는 완전한 대응이 불가능하다. 침해 대응 시에는 모든 가능한 백도어 경로(계정·서비스·예약 작업·레지스트리 등)를 체계적으로 조사하고, 이미지 재설치(bare-metal restore)와 자격증명 전체 교체를 병행해야 완전한 복구가 보장된다.

📢 **섹션 요약 비유**: 침입자가 들어왔다 나간 집은 자물쇠만 바꾸면 되지만, 침입자가 열쇠를 복사해 갔다면 자물쇠 자체를 교체(시스템 재설치)해야 안전하다.

### 👶 어린이를 위한 3줄 비유 설명
백도어는 해커가 처음 집에 들어온 다음, 나중에도 몰래 들어올 수 있게 만들어 놓은 비밀 출입구예요.  
일반 자물쇠(로그인)로는 막을 수 없어요, 왜냐면 그 문이 어디 있는지 집주인도 모르거든요.  
이걸 막으려면 집 안에서 나가는 전화(아웃바운드 트래픽)까지 전부 감시해야 해요.
