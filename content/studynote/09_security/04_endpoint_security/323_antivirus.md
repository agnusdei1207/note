## 핵심 인사이트 (3줄 요약)
> 1. **본질**: AV(Anti-Virus)는 악성코드의 고유 패턴(시그니처)을 데이터베이스와 비교하여 탐지·제거하는 가장 오래된 엔드포인트 보안 기술이다.
> 2. **가치**: 알려진 악성코드의 빠르고 정확한 탐지에 강점이 있으나, 변종·신종·파일리스(fileless) 악성코드에는 한계가 명확하여 현대 EPP/EDR로 보완한다.
> 3. **판단 포인트**: 시그니처 업데이트 주기·오탐율·엔진 성능이 AV 품질의 핵심 지표이며, 기술사 답안에서는 "AV는 EPP의 1차 필터"로 포지셔닝한다.

---

## Ⅰ. 개요 및 필요성

AV(Anti-Virus, 안티바이러스)는 컴퓨터 바이러스·웜·트로이목마 등 악성코드를 탐지하고 제거하는 소프트웨어로, 1980년대 PC 바이러스 등장과 함께 발전했다. 초기에는 부트 섹터 바이러스 등 단순한 형태의 악성코드를 대상으로 했으나, 악성코드가 복잡해짐에 따라 탐지 기술도 발전했다.

AV의 핵심은 시그니처(signature) 데이터베이스다. 보안 업체 연구팀이 새로운 악성코드를 분석하여 고유한 바이트 패턴(시그니처)을 추출하고, 이를 데이터베이스에 등록한다. AV 엔진은 스캔 대상 파일에서 이 패턴을 검색하여 악성코드 여부를 판단한다. 시그니처 업데이트가 생명선이기 때문에 대부분의 AV는 수시간~수일 단위로 자동 업데이트를 수행한다.

현대 AV는 시그니처 탐지에 더해 휴리스틱(heuristic), 클라우드 기반 평판(reputation) 조회, 행위 모니터링 등을 추가하여 EPP(Endpoint Protection Platform)의 핵심 엔진으로 기능한다. 그러나 알려지지 않은 악성코드(zero-day)에는 근본적 한계가 있어 EDR과의 병행이 필수다.

📢 **섹션 요약 비유**: AV는 범죄자 사진첩(시그니처 DB)을 들고 다니는 경비원이다—사진에 있는 사람은 바로 잡지만, 새로운 범죄자는 알아볼 수 없다.

## Ⅱ. 아키텍처 및 핵심 원리

### AV 탐지 기법 비교

| 탐지 기법 | 원리 | 장점 | 단점 |
|:---|:---|:---|:---|
| 시그니처 | 알려진 패턴 비교 | 빠르고 정확 | 신종 미탐지 |
| 휴리스틱 | 의심 코드 패턴 분석 | 변종 탐지 가능 | 오탐 증가 |
| 행위 기반 | 실행 중 행위 모니터링 | 알려지지 않은 악성코드 탐지 | 성능 오버헤드 |
| 클라우드 평판 | 글로벌 해시 DB 조회 | 실시간 최신 정보 | 인터넷 의존 |

```
┌──────────────────────────────────────────────────┐
│                AV 탐지 흐름도                    │
├──────────────────────────────────────────────────┤
│  파일 접근/실행 시도                             │
│        │                                         │
│  ┌─────▼──────────────────────────────────────┐ │
│  │ 1단계: 시그니처 DB 비교       (밀리초)     │ │
│  │ 2단계: 클라우드 해시 평판 조회 (초)        │ │
│  │ 3단계: 휴리스틱 정적 분석     (초)         │ │
│  │ 4단계: 행위 모니터링 (실행 후) (지속)      │ │
│  └─────────────────────────────────────────────┘ │
│        │                                         │
│  허용 / 격리(Quarantine) / 삭제                  │
└──────────────────────────────────────────────────┘
```

AV 엔진은 실시간 보호(on-access scanning)와 정기 검사(on-demand scanning) 두 모드로 동작한다. 실시간 보호는 파일 시스템 드라이버 레벨에서 파일 접근 시 즉시 검사하고, 정기 검사는 스케줄에 따라 전체 시스템을 순차적으로 검사한다.

📢 **섹션 요약 비유**: AV의 실시간 보호는 공항 보안 검색대처럼 모든 파일이 실행되기 전에 검사하고, 정기 검사는 매월 전수 감사하는 것과 같다.

## Ⅲ. 비교 및 연결

| 구분 | 전통적 AV | 차세대 AV (NGAV) |
|:---|:---|:---|
| 탐지 핵심 | 시그니처 DB | ML + 시그니처 |
| 제로데이 대응 | 없음 | 부분 가능 |
| 업데이트 방식 | 주기적 DB 업데이트 | 클라우드 실시간 |
| 성능 | 경량 | 다소 무거움 |
| 대표 제품 | Norton, McAfee | CrowdStrike, SentinelOne |

전통적 AV의 가장 큰 약점은 시그니처 업데이트 이전의 "창"과 패턴 회피 기법(packing, obfuscation)이다. 공격자들은 악성코드를 압축(packing)하거나 코드를 변형(obfuscation)하여 시그니처 탐지를 우회한다. 이에 대응하여 차세대 AV(NGAV: Next-Generation Anti-Virus)는 머신러닝과 행위 분석을 결합한다.

📢 **섹션 요약 비유**: 전통 AV는 주민등록증 확인, NGAV는 생체인식+행동 패턴 분석으로 신원을 확인하는 진화된 시스템이다.

## Ⅳ. 실무 적용 및 기술사 판단

기업 AV 운영 시 핵심 관리 항목은 ① 시그니처 업데이트 성공률(99% 이상 목표), ② 격리(quarantine) 파일의 오탐 검토 프로세스, ③ 정기 검사 스케줄과 업무 시간 분리, ④ 예외(exclusion) 목록 관리다. 예외 목록 과다 등록은 보안 구멍을 만들 수 있어 정기 감사가 필요하다.

기술사 답안에서는 AV를 EPP 내의 "1차 시그니처 필터"로 정의하고, 단독으로는 현대 위협(APT, 랜섬웨어, 파일리스)에 부족하므로 반드시 EDR·행위분석과 병행해야 함을 강조해야 한다.

📢 **섹션 요약 비유**: AV만으로 보안을 완성하는 것은 현관문만 잠그고 창문을 열어둔 것처럼, 기본은 갖췄지만 충분하지 않다.

## Ⅴ. 기대효과 및 결론

AV는 대부분의 알려진 악성코드에 대해 빠르고 저렴하게 대응하는 필수 기반 기술이다. 전 세계 AV 엔진들이 공유하는 EICAR 테스트 파일 표준과 VirusTotal 같은 멀티 엔진 검사 플랫폼은 AV 생태계의 협력 구조를 잘 보여준다.

현대 보안 환경에서 AV는 EPP의 구성 요소로 통합되어, 머신러닝·EDR과 함께 작동하는 레이어 중 하나로 기능한다. 기술사 수험자는 AV가 "필요하지만 충분하지 않은" 기술임을 명확히 이해하고 답안에 반영해야 한다.

📢 **섹션 요약 비유**: AV는 보안의 기초 체력이다—없으면 안 되지만, 그것만으로 완전한 선수가 될 수는 없다.

### 📌 관련 개념 맵
| 개념 | 관계 | 설명 |
|:---|:---|:---|
| 시그니처 DB (Signature Database) | 핵심 구성 | 알려진 악성코드 패턴 저장소 |
| EPP (Endpoint Protection Platform) | 상위 개념 | AV를 포함한 통합 예방 플랫폼 |
| NGAV (Next-Generation Anti-Virus) | 진화 형태 | ML+행위분석 결합 차세대 AV |
| EDR (Endpoint Detection and Response) | 보완 기술 | AV 미탐지 위협의 사후 탐지·대응 |

### 👶 어린이를 위한 3줄 비유 설명
- AV는 나쁜 사람들의 사진이 담긴 책을 보고 비교하는 경비원이야.
- 책에 사진이 있으면 바로 잡지만, 새로운 나쁜 사람은 못 알아볼 수도 있어.
- 그래서 요즘은 행동을 보는 다른 경비원(EDR)도 함께 배치한단다!
