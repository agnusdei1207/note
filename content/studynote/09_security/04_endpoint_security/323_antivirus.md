## 핵심 인사이트 (3줄 요약)
> 1. **본질**: AV(Anti-Virus)는 알려진 악성코드의 시그니처(Signature)를 데이터베이스와 대조하여 탐지·차단·제거하는 가장 기본적인 엔드포인트 보안 솔루션이다.
> 2. **가치**: 시그니처 기반 탐지는 알려진 위협에 대해 빠르고 확실한 대응을 제공하며, 현재도 전체 엔드포인트 위협의 60~70%는 시그니처 매칭으로 처리된다.
> 3. **판단 포인트**: AV의 치명적 약점은 제로데이(Zero-Day)와 다형성(Polymorphic) 악성코드에 대한 무방비 상태이며, 이를 보완하기 위해 ML(Machine Learning)과 행위 분석이 결합된 NGAV(Next-Generation Anti-Virus)로 진화하고 있다.

---

## Ⅰ. 개요 및 필요성

AV(Anti-Virus)는 1980년대 PC 바이러스 확산과 함께 탄생한 최초의 엔드포인트 보안 솔루션이다. 프레드 코헨(Fred Cohen)이 1984년 "바이러스"의 개념을 정의한 직후, 이를 탐지하는 소프트웨어들이 등장했고, 이후 30년간 시그니처 기반 탐지 방식이 업계 표준으로 자리 잡았다.

AV의 기본 원리는 **시그니처 매칭(Signature Matching)** 이다. 악성코드가 발견되면 분석가가 그 고유한 바이트 패턴(Hash값, 특정 문자열 등)을 추출해 시그니처 DB에 등록한다. 이후 시스템에서 파일을 검사할 때 이 DB와 비교하여 일치하는 패턴이 있으면 악성으로 판단한다. 이 방식은 **빠르고 오탐이 낮다**는 장점이 있지만, DB에 없는 신종 악성코드는 탐지할 수 없다는 구조적 한계를 갖는다.

현대 AV는 단순 시그니처 탐지를 넘어 휴리스틱(Heuristic) 분석, 에뮬레이션(Emulation), 클라우드 기반 위협 인텔리전스를 통합하고 있다. 그럼에도 제로데이·다형성·파일리스 공격에 대한 대응 한계가 명확하여, 보안 업계는 AV를 최소 필수 방어선으로 보고 EDR·EPP와의 병용을 권장한다.

📢 **섹션 요약 비유**: AV는 "지명수배 전단지 경비원"이다. 전단지에 있는 얼굴은 100% 잡지만, 전단지에 없는 새 범인은 그냥 통과시킨다.

## Ⅱ. 아키텍처 및 핵심 원리

### AV 탐지 방법 비교

| 탐지 방법 | 원리 | 장점 | 단점 |
|:---------|:-----|:-----|:-----|
| 시그니처 | 알려진 악성코드 패턴 매칭 | 빠르고 정확, 오탐 낮음 | 제로데이 무방비 |
| 휴리스틱 | 코드 구조·특성 기반 추론 | 변종 일부 탐지 | 오탐률 높음 |
| 에뮬레이션 | 가상 환경에서 코드 실행 | 패킹된 악성코드 탐지 | 속도 저하 |
| 클라우드 조회 | 해시값 클라우드 DB 조회 | 최신 위협 즉시 반영 | 네트워크 의존 |

```
┌─────────────────────────────────────────────────┐
│              AV 탐지 파이프라인                 │
│                                                 │
│  파일 접근/실행 시도                            │
│         │                                       │
│         ▼                                       │
│  ┌─────────────┐  일치   ┌──────────────┐      │
│  │ 시그니처 DB  │────────▶│ 악성코드 차단 │      │
│  └──────┬──────┘         └──────────────┘      │
│         │ 불일치                                │
│         ▼                                       │
│  ┌─────────────┐  의심   ┌──────────────┐      │
│  │ 휴리스틱 분석│────────▶│ 샌드박스 실행 │      │
│  └──────┬──────┘         └──────────────┘      │
│         │ 정상                                  │
│         ▼                                       │
│     파일 실행 허용                              │
└─────────────────────────────────────────────────┘
```

시그니처 DB는 벤더(Vendor) 클라우드에서 자동 업데이트된다. 업데이트 주기는 일반적으로 수 시간~1일이며, 이 **업데이트 간격이 바로 취약 창(Vulnerability Window)** 이다. 새로운 악성코드가 발견되고 시그니처가 배포되기 전까지 수천만 대의 PC가 무방비 상태에 놓인다.

📢 **섹션 요약 비유**: AV 탐지 파이프라인은 "세관 검사대"다. 블랙리스트(시그니처) 확인 → X레이 검사(휴리스틱) → 별도 심사실(샌드박스) 순서로 점점 깊은 검사를 수행한다.

## Ⅲ. 비교 및 연결

### AV vs NGAV(Next-Generation Anti-Virus) 비교

| 특성 | 전통 AV | NGAV |
|:----|:--------|:-----|
| 주요 탐지 방식 | 시그니처 | ML + 행위 분석 |
| 제로데이 대응 | 불가 | 부분 가능 |
| 업데이트 의존도 | 높음 | 낮음 |
| 리소스 사용 | 낮음 | 보통~높음 |
| 오탐률 | 낮음 | 보통 |

AV는 현재도 EPP의 핵심 구성 요소로 남아 있으며, NGAV는 AV를 완전히 대체하기보다는 시그니처 탐지의 약점을 ML과 행위 분석으로 보완하는 방향으로 발전하고 있다. 특히 다형성 악성코드(Polymorphic Malware)와 파일리스 공격(Fileless Attack)에 대응하기 위해 메모리 보호와 스크립트 모니터링 기능이 추가되었다.

📢 **섹션 요약 비유**: AV가 "옛날 경비원 (얼굴 외우는 방식)"이라면 NGAV는 "AI CCTV (행동 패턴 분석)"다. 처음 보는 사람도 행동이 수상하면 잡아낸다.

## Ⅳ. 실무 적용 및 기술사 판단

**AV 운영 체크리스트:**
- 시그니처 DB 업데이트 자동화(최대 지연 4시간 이하 권고)
- 전체 시스템 정기 스캔 스케줄 설정(주 1회 이상)
- 검역(Quarantine) 파일 30일 이상 보존(법적 증거 활용)
- 예외 처리 목록(Exclusion List) 문서화 및 주기적 검토

**다형성 악성코드 대응 전략:**
- 시그니처만으로 대응 불가 → NGAV 또는 EDR 병행
- 행위 기반 탐지와 메모리 스캔 활용
- 위협 인텔리전스 피드(Feed) 실시간 구독

📢 **섹션 요약 비유**: AV 관리는 "도감 관리"와 같다. 도감이 최신 버전으로 유지되어야 하고, 도감에 없는 새 종은 전문가(EDR)를 부르는 것이 최선이다.

## Ⅴ. 기대효과 및 결론

AV는 단순하지만 여전히 필수적인 보안 수단이다. 전 세계 맬웨어 샘플의 약 70%는 기존 시그니처로 탐지 가능하며, 구현과 운영 비용이 낮아 중소기업에서도 기본 방어선으로 충분히 활용할 수 있다. 그러나 제로데이·다형성·파일리스 공격이 증가하는 현실에서 AV만으로는 충분하지 않다.

기술사 시험과 실무 모두에서 AV는 "엔드포인트 보안의 시작점"으로 서술하되, 반드시 NGAV·EPP·EDR과의 조합이 필요함을 언급해야 한다.

📢 **섹션 요약 비유**: AV는 "기본 예방 주사"다. 많은 병을 막아주지만, 신종 바이러스에는 추가 처방이 필요하다.

### 👶 어린이를 위한 3줄 비유 설명
1. AV는 "나쁜 사람 사진첩"을 가진 경비원이야. 사진첩에 있는 사람은 바로 알아보고 막아.
2. 하지만 사진첩에 없는 새로운 나쁜 사람은 알아볼 수가 없어.
3. 그래서 사진첩을 매일 업데이트하고, 새로운 수법을 쓰는 사람은 AI 카메라(NGAV)가 잡아내.
