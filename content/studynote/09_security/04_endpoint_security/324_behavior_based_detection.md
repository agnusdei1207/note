## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 행위 기반 탐지(Behavior-Based Detection)는 악성코드의 파일 패턴이 아니라 실행 중 행위(프로세스 생성, 파일 변경, 네트워크 통신)를 분석하여 위협을 식별하는 기술이다.
> 2. **가치**: 시그니처가 없는 신종·변종 악성코드, 파일리스(fileless) 공격, 메모리 기반 공격을 탐지할 수 있어 현대 EDR의 핵심 탐지 엔진이다.
> 3. **판단 포인트**: 정상 행위와 악성 행위의 경계를 정의하는 베이스라인(baseline) 수립과 오탐(False Positive) 관리가 운영의 핵심 과제다.

---

## Ⅰ. 개요 및 필요성

행위 기반 탐지(Behavior-Based Detection)는 악성코드나 공격 행위를 파일의 내용이나 시그니처가 아닌, 실행 중 발생하는 행동 패턴으로 탐지하는 방법론이다. 전통적 AV(Anti-Virus)가 "무엇인가(what it is)"를 판단한다면, 행위 기반 탐지는 "무엇을 하는가(what it does)"를 판단한다.

공격자들은 악성코드를 패킹(packing), 암호화, 코드 난독화(obfuscation)로 시그니처 탐지를 우회해왔다. 파일리스(fileless) 악성코드는 디스크에 파일을 남기지 않고 메모리나 PowerShell·WMI 같은 정상 도구를 악용하여 AV를 완전히 우회한다. 행위 기반 탐지는 이러한 고도화된 공격에 대응하기 위해 필수적이다.

행위 기반 탐지 시스템은 프로세스 실행·종료, 파일 생성·수정·삭제, 레지스트리 변경, 네트워크 연결, API 호출 등 수십~수백 가지 이벤트를 실시간으로 수집하고 분석한다. 이 데이터는 MITRE ATT&CK 프레임워크의 전술(Tactics)·기법(Techniques)에 매핑되어 공격 단계를 파악하는 데 활용된다.

📢 **섹션 요약 비유**: 시그니처 탐지가 범죄자 외모로 판단한다면, 행위 기반 탐지는 수상한 행동(도청, 잠입, 절도 시도)으로 범죄자를 잡는 형사다.

## Ⅱ. 아키텍처 및 핵심 원리

### 행위 기반 탐지 이벤트 유형

| 이벤트 유형 | 감시 항목 | 악성 행위 예시 |
|:---|:---|:---|
| 프로세스 행위 | 생성·종료·권한 변경 | cmd.exe → powershell.exe 자식 프로세스 |
| 파일 시스템 | 대량 파일 암호화 | 랜섬웨어 암호화 패턴 |
| 레지스트리 | 자동 실행 키 등록 | HKCU\Run 항목 추가 |
| 네트워크 | C2 서버 통신 | 비정상 외부 IP 연결 |
| 메모리 | 코드 인젝션 | 프로세스 메모리 쓰기 |

```
┌──────────────────────────────────────────────────────┐
│           행위 기반 탐지 파이프라인                  │
├──────────────────────────────────────────────────────┤
│  이벤트 수집                                         │
│  (커널 드라이버 / ETW / API 후킹)                    │
│          │                                           │
│  ┌───────▼─────────────────────────────────────┐    │
│  │ 행위 분석 엔진                              │    │
│  │ - 규칙 기반 (YARA / Sigma)                 │    │
│  │ - ML 이상 탐지 모델                         │    │
│  │ - MITRE ATT&CK 매핑                         │    │
│  └───────────────────────────────────────────────┘    │
│          │                                           │
│  경보(Alert) → 격리 → 포렌식 수집                    │
└──────────────────────────────────────────────────────┘
```

행위 기반 탐지의 데이터 수집은 커널 레벨 드라이버, ETW(Event Tracing for Windows), API 후킹(API hooking) 등 다양한 방법으로 이루어진다. 수집된 이벤트는 규칙 기반 엔진(YARA, Sigma 규칙)과 머신러닝 이상 탐지 모델로 분석된다.

📢 **섹션 요약 비유**: 행위 기반 탐지 엔진은 CCTV 분석 AI처럼, 모든 움직임을 녹화하고 수상한 패턴이 발견되면 자동으로 경보를 울린다.

## Ⅲ. 비교 및 연결

| 구분 | 시그니처 기반 | 행위 기반 | 머신러닝 기반 |
|:---|:---|:---|:---|
| 탐지 대상 | 알려진 악성코드 | 모든 의심 행위 | 패턴 이상 감지 |
| 제로데이 대응 | 없음 | 가능 | 가능 |
| 오탐율 | 낮음 | 높을 수 있음 | 중간 |
| 분석 단위 | 파일 단위 | 이벤트 시퀀스 | 특징 벡터 |
| 구현 복잡도 | 낮음 | 높음 | 매우 높음 |

행위 기반 탐지는 파일리스 악성코드와 Living-off-the-Land(LOtL) 공격—정상 시스템 도구(PowerShell, WMI, certutil)를 악용하는 공격—탐지에 특히 유효하다. 반면 정상 관리 작업과 악성 행위가 외형상 유사할 때 오탐이 발생할 수 있어 베이스라인 튜닝이 중요하다.

📢 **섹션 요약 비유**: 시그니처는 얼굴 인식, 행위 기반은 걷는 방식·말투·행동 패턴으로 사람을 구별하는 행동 분석이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

행위 기반 탐지 운영의 핵심 과제는 베이스라인(baseline) 수립이다. 조직마다 정상적인 소프트웨어 실행 패턴이 다르므로, 초기 2~4주간 학습 기간을 두어 정상 행위 프로파일을 수집한다. 이후 이 베이스라인에서 벗어나는 행위를 경보로 처리한다.

기술사 답안에서는 행위 기반 탐지를 "알려지지 않은 위협(Unknown Threat) 대응의 핵심 기술"로 정의하고, MITRE ATT&CK 프레임워크와의 연계를 통해 공격 단계(킬 체인) 전반을 가시화하는 효과를 강조해야 한다.

📢 **섹션 요약 비유**: 베이스라인 없는 행위 기반 탐지는 기준 없이 이상한 사람을 잡으려는 것—기준(정상 패턴)이 있어야 이상(악성)을 구분할 수 있다.

---

## Ⅴ. 기대효과 및 결론

행위 기반 탐지는 현대 EDR(Endpoint Detection and Response)의 근간 기술로, 시그니처가 전혀 없는 신종 악성코드, 메모리 기반 공격, 파일리스 위협에 대한 가시성을 제공한다. MITRE ATT&CK 매핑을 통해 단순 탐지를 넘어 공격자의 전술·기법·절차(TTP: Tactics, Techniques, Procedures)를 역추적할 수 있다.

향후 AI/ML 기반 이상 탐지와 결합되어 더욱 정밀한 위협 탐지가 가능해질 것이며, 자동화된 SOAR(Security Orchestration, Automation and Response)와 연동하여 탐지 즉시 대응이 자동화되는 방향으로 발전할 것이다.

📢 **섹션 요약 비유**: 행위 기반 탐지는 보안의 DNA 검사처럼, 외모(시그니처)가 달라도 행동 유전자(패턴)로 위협을 잡아낸다.

### 👶 어린이를 위한 3줄 비유 설명
- 행위 기반 탐지는 나쁜 사람의 외모가 아니라 행동을 보고 잡는 탐정이야.
- 도둑은 모자를 바꿔 써도 걷는 방식(행위)은 숨기기 어렵거든.
