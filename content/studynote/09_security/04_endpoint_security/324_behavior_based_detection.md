## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 행위 기반 탐지(Behavior-Based Detection)는 파일의 시그니처가 아닌 실행 중인 프로세스·시스템 호출·네트워크 동작 **행위 패턴**을 분석하여 악성 여부를 판단하는 탐지 방법이다.
> 2. **가치**: 제로데이(Zero-Day) 익스플로잇, 다형성(Polymorphic) 악성코드, 파일리스(Fileless) 공격처럼 시그니처가 없는 위협을 탐지할 수 있는 사실상 유일한 방법이다.
> 3. **판단 포인트**: 오탐(False Positive)이 높다는 단점이 있어, ML(Machine Learning) 기반 기준선(Baseline) 학습과 컨텍스트 분석을 결합해야 실용적인 탐지 정확도를 달성할 수 있다.

---

## Ⅰ. 개요 및 필요성

행위 기반 탐지(Behavior-Based Detection)는 "무엇인가"가 아니라 "무엇을 하는가"를 보는 탐지 방식이다. 전통적인 시그니처 탐지가 악성코드의 정체(코드 패턴)를 확인하는 것이라면, 행위 기반 탐지는 코드가 실행될 때 발생하는 **시스템 자원 접근, 네트워크 통신, 프로세스 생성, 레지스트리 변경** 등의 동적 행위를 관찰한다.

현대 공격자들은 알려진 악성 파일을 직접 사용하는 대신, 시스템에 이미 설치된 합법적 도구(PowerShell, WMI, certutil 등)를 악용하는 LotL(Living-off-the-Land) 기법을 사용한다. 이런 공격은 파일 기반 시그니처로는 탐지 불가하지만, 행위 패턴(예: PowerShell이 외부 서버에서 코드를 다운로드 후 메모리에서 실행)은 이상하다는 것을 행위 기반 탐지가 포착할 수 있다.

MITRE ATT&CK(Adversarial Tactics, Techniques, and Common Knowledge) 프레임워크는 공격자의 전술·기술·절차(TTP, Tactics, Techniques, and Procedures)를 체계화하며, 이를 기반으로 행위 기반 탐지 규칙을 설계하는 것이 업계 표준이 되었다.

📢 **섹션 요약 비유**: 행위 기반 탐지는 "얼굴을 모르는 소매치기를 행동으로 잡아내는 경찰관"이다. 잡기 전에는 모르지만, 지갑을 더듬는 행동을 보는 순간 알아챈다.

## Ⅱ. 아키텍처 및 핵심 원리

### 행위 기반 탐지 이벤트 유형

| 이벤트 유형 | 예시 | 악성 행위 지표 |
|:-----------|:-----|:-------------|
| 프로세스 생성 | cmd.exe → powershell.exe | 예상치 못한 자식 프로세스 생성 |
| 파일 접근 | lsass.exe 메모리 읽기 | 자격증명 덤프 시도 |
| 네트워크 연결 | 비정상 포트/도메인 연결 | C2(Command and Control) 통신 |
| 레지스트리 변경 | 시작프로그램 키 수정 | 지속성(Persistence) 확보 |
| API 호출 | VirtualAlloc + WriteProcessMemory | 코드 인젝션(Code Injection) 시도 |

```
┌──────────────────────────────────────────────────┐
│           행위 기반 탐지 데이터 흐름             │
│                                                  │
│  OS 커널 이벤트 (시스템 콜, API 후킹)            │
│         │                                        │
│         ▼                                        │
│  ┌──────────────┐                                │
│  │ 이벤트 수집기 │  프로세스·파일·네트워크·레지스트리│
│  └──────┬───────┘                                │
│         │                                        │
│         ▼                                        │
│  ┌──────────────┐     ┌─────────────────┐        │
│  │ 행위 분석 엔진│────▶│ 기준선(Baseline) │        │
│  │ (ML + 규칙)  │     │ 정상 행위 프로파일│        │
│  └──────┬───────┘     └─────────────────┘        │
│         │ 이상 탐지                              │
│         ▼                                        │
│  ┌──────────────┐                                │
│  │ 알림 및 대응  │  격리·차단·SOAR 연동           │
│  └──────────────┘                                │
└──────────────────────────────────────────────────┘
```

핵심 원리는 **기준선(Baseline) 대비 편차 탐지**다. 시스템의 정상 행위 프로파일을 학습하고, 그로부터 통계적으로 유의한 편차가 발생하면 알람을 생성한다. 이 과정에서 ML 모델(이상 탐지, 분류기)이 핵심 역할을 한다.

📢 **섹션 요약 비유**: 행위 기반 탐지는 "음식점 단골손님 패턴 인식"과 같다. 매일 같은 자리에서 같은 메뉴를 먹는 손님이 갑자기 뒷문으로 들어오면 알아챈다.

## Ⅲ. 비교 및 연결

### 시그니처 vs 행위 기반 탐지 비교

| 특성 | 시그니처 기반 | 행위 기반 |
|:----|:-------------|:---------|
| 제로데이 탐지 | 불가 | 가능 |
| 오탐률 | 낮음 | 높음 |
| 리소스 사용 | 낮음 | 높음 |
| 설명 가능성 | 높음 | 보통 |
| LotL 공격 탐지 | 불가 | 가능 |

UEBA(User and Entity Behavior Analytics)는 행위 기반 탐지를 사용자와 엔티티(서버, 네트워크 장비) 레벨로 확장한 것으로, 내부자 위협(Insider Threat) 탐지에 특히 효과적이다. EDR은 단말 레벨의 행위 기반 탐지를 구현하고, SIEM(Security Information and Event Management)은 이를 네트워크·서버 로그와 상관 분석한다.

📢 **섹션 요약 비유**: 행위 기반 탐지가 "개인의 행동 관찰"이라면, UEBA는 "사람들 사이의 이상한 관계까지 분석하는 사회관계 분석관"이다.

## Ⅳ. 실무 적용 및 기술사 판단

**탐지 규칙 설계 원칙:**
- MITRE ATT&CK 기술 번호(T-number)와 매핑하여 커버리지 관리
- 오탐 감소를 위해 단일 이벤트가 아닌 **이벤트 체인(Event Chain)** 기반으로 규칙 설계
- 화이트리스트와 컨텍스트(사용자, 시간대, 위치) 적용

**오탐 관리:**
- 새 소프트웨어 배포 전 탐지 규칙과의 충돌 테스트 필수
- "탐지 전용 모드(Detection-Only Mode)"로 운영 후 튜닝
- 오탐 빈도 KPI(Key Performance Indicator) 설정 및 추적

📢 **섹션 요약 비유**: 행위 기반 탐지 규칙 설계는 "세밀한 레시피 작성"이다. 재료(이벤트) 하나만 보지 않고, 여러 재료의 조합(이벤트 체인)이 완성될 때 판단한다.

## Ⅴ. 기대효과 및 결론

행위 기반 탐지를 도입하면 시그니처 탐지로는 발견할 수 없는 고도화 공격의 50~70%를 추가로 탐지할 수 있다. 특히 APT(Advanced Persistent Threat)나 내부자 위협처럼 장기간 잠복하는 공격에 대해 조기 경보가 가능해진다.

단, 행위 기반 탐지는 단독으로 사용할 경우 오탐이 많아 운영 부담이 크다. 시그니처 기반 탐지와 행위 기반 탐지를 계층화하고, SOAR(Security Orchestration, Automation, and Response) 플랫폼과 연동하여 알람 피로(Alert Fatigue)를 줄이는 운영 체계가 필요하다.

📢 **섹션 요약 비유**: 행위 기반 탐지는 "경험 많은 형사의 직관"이다. 범죄 기록이 없어도 행동이 수상하면 알아채지만, 너무 예민하면 착한 사람도 잡을 수 있다.

### 👶 어린이를 위한 3줄 비유 설명
1. 행위 기반 탐지는 "낯선 사람을 얼굴이 아닌 행동으로 구별하는 탐정"이야.
2. 평소와 다르게 남의 서랍을 만진다거나, 수상한 방향을 계속 살핀다면 의심받게 돼.
3. 이 방법은 새로운 나쁜 사람도 행동이 이상하면 잡아낼 수 있어.
