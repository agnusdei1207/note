## 핵심 인사이트 (3줄 요약)
> 1. **본질**: BitLocker는 Windows의 대표적 FDE (Full Disk Encryption)로, 디스크 전체를 암호화하되 부팅 시점에는 TPM (Trusted Platform Module)과 PIN/USB 키 정책으로만 잠금을 푼다.
> 2. **가치**: 장비 분실이나 오프라인 탈취가 발생해도 저장 매체는 암호문만 남으므로, 노트북 한 대가 곧바로 내부 데이터 유출로 이어지지 않는다.
> 3. **판단 포인트**: TPM만 믿는 모드는 편하지만, 고위험 환경에서는 TPM + PIN 또는 TPM + USB 스타트업 키를 사용해 pre-boot authentication을 강화해야 한다.

---

## Ⅰ. 개요 및 필요성

Windows 노트북은 이동성과 함께 분실 위험도 크다. 로그인 암호만 걸어 두면 사용자가 자리를 비운 상태의 장비를 탈취한 공격자는 디스크를 분리해 오프라인으로 분석할 수 있다. 이때 필요한 것이 전체 드라이브를 잠그는 BitLocker다.

BitLocker는 운영체제가 올라오기 전부터 디스크 잠금을 유지하다가, 부팅 환경이 정상이라고 판단되면 키를 풀어 준다. 즉 "윈도우가 켜진 뒤에 걸리는 보안"이 아니라, **윈도우가 켜지기 전부터 시작되는 보안**이다.

```text
┌──────────────────────────────────────────────────────────────┐
│          BitLocker의 핵심: OS 이전에 디스크를 잠근다         │
├──────────────────────────────────────────────────────────────┤
│  UEFI/BIOS → TPM/PIN 검증 → VMK 해제 → FVEK로 디스크 복호화 │
│  부팅 실패 시: 드라이브는 끝까지 잠김                       │
└──────────────────────────────────────────────────────────────┘
```

이 구조 덕분에 장비를 훔쳐 가도 저장 장치 자체에서 평문을 꺼내기 어렵다. 관리자는 사용 편의성과 보안 강도 사이에서 TPM-only, TPM+PIN, TPM+USB 중 적절한 모드를 골라야 한다.

- **📢 섹션 요약 비유**: BitLocker는 자동차 시동이 걸리기 전까지는 핸들이 잠겨 있는 것과 같다. 열쇠를 꽂아도, 정해진 조건을 만족해야만 차가 움직인다.

## Ⅱ. 아키텍처 및 핵심 원리

BitLocker는 단일 키가 아니라 키 계층과 부팅 정책으로 동작한다.

| 구성 요소 | 역할 | 비고 |
|:---|:---|:---|
| **FVEK (Full Volume Encryption Key)** | 실제 섹터 암호화 | 디스크 데이터를 직접 암호화 |
| **VMK (Volume Master Key)** | FVEK 보호 | TPM/비밀번호/복구키로 해제 |
| **TPM** | 부팅 상태 측정 및 키 봉인 | PCR 값이 바뀌면 해제 거부 |
| **Recovery Key** | 최후의 복구 경로 | 계정/에스크로 보관 필요 |

핵심은 VMK가 곧바로 노출되지 않는다는 점이다. TPM은 PCR (Platform Configuration Register)에 기록된 부팅 상태가 기대값과 일치할 때만 VMK를 풀어 준다. 따라서 Secure Boot, Measured Boot, BitLocker는 서로 분리된 기능이 아니라 한 흐름으로 묶인다.

### BitLocker 시작 모드

| 모드 | 특징 | 적합한 환경 |
|:---|:---|:---|
| TPM only | 편하지만 물리 탈취에 가장 약함 | 일반 사무용 |
| TPM + PIN | 부팅 전 사람이 추가 인증 | 고위험 노트북 |
| TPM + USB key | USB 토큰이 있어야 해제 | 폐쇄망/특수 장비 |
| TPM + PIN + USB | 가장 강함 | 기밀 문서 장비 |

```text
┌──────────────────────────────────────────────────────────────┐
│                BitLocker 키 해제 흐름                        │
├──────────────────────────────────────────────────────────────┤
│  1. 전원 ON                                                │
│  2. Secure Boot / PCR 확인                                 │
│  3. TPM이 VMK 해제 여부 판단                               │
│  4. PIN 또는 USB 키 필요 시 추가 인증                      │
│  5. VMK → FVEK → 디스크 섹터 복호화                        │
└──────────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: BitLocker는 금고 안 금고 구조다. 바깥 상자(FVEK)는 안쪽 열쇠(VMK)로 잠기고, 안쪽 열쇠는 또 TPM과 PIN으로 잠긴다.

## Ⅲ. 비교 및 연결

BitLocker는 Windows 생태계에 가장 깊게 통합된 FDE지만, 같은 FDE라도 FileVault, LUKS, VeraCrypt와 철학이 다르다.

| 항목 | BitLocker | FileVault | LUKS | VeraCrypt |
|:---|:---|:---|:---|:---|
| 플랫폼 | Windows | macOS | Linux | Cross-platform |
| 부팅 통합 | TPM 중심 | Secure Enclave/계정 중심 | dm-crypt + initramfs | 수동 마운트 중심 |
| 사용자 경험 | 매우 쉬움 | 쉬움 | 유연하지만 관리 필요 | 다양하지만 수동 |
| 강점 | 기업 관리, 복구키 에스크로 | Apple 생태계 통합 | 오픈 표준, 세밀 제어 | 이식성, 숨김 볼륨 |
| 약점 | Windows 종속 | Apple 종속 | 운영 복잡도 | 자동화 통합 약함 |

BitLocker의 실제 가치는 "암호화" 자체보다 **기업 운영**에 있다. AD/Azure AD/Intune 같은 관리 체계와 결합해 복구 키를 관리하고, 장비 교체와 정책 배포를 중앙에서 통제할 수 있다. 즉 개인용 암호화 도구가 아니라 **엔드포인트 보안 정책 도구**다.

- **📢 섹션 요약 비유**: BitLocker는 윈도우 전용 출입통제 시스템이다. 기능은 강력하지만, 윈도우 건물 안에서 가장 잘 돌아간다.

## Ⅳ. 실무 적용 및 기술사 판단

### 체크리스트

1. 복구 키를 AD/Azure AD/보안 저장소에 에스크로 했는가?
2. BIOS 업데이트나 TPM 교체 시 PCR 변화를 운영 절차에 반영했는가?
3. TPM-only 대신 TPM + PIN을 써야 하는 장비를 구분했는가?
4. BitLocker 상태를 MDM/그룹 정책으로 모니터링하는가?

### 안티패턴

- 복구 키 없이 TPM-only로 무차별 배포
- BIOS 업데이트 후 대량 복구 키 요구 사태 대응 미흡
- 외장 드라이브/USB 저장소는 암호화 대상에서 빠뜨림
- "암호화했으니 끝"이라고 생각하고 계정 탈취 대응을 생략

### 기술사 판단

BitLocker는 저장 중 암호화의 한 형태지만, 실제로는 **부팅 신뢰 체인과 키 관리 체계**가 더 중요하다. 기술사 답안에서는 "FDE는 디스크 분실을 막고, TPM 기반 부팅 검증은 키 탈취를 막는다"는 두 축을 같이 써야 한다.

- **📢 섹션 요약 비유**: BitLocker를 잘 쓰려면 문만 잠그는 게 아니라 열쇠 예비분도 안전하게 맡겨야 한다. 문은 튼튼해도 비상 열쇠를 잃어버리면 집에 못 들어간다.

---

## Ⅴ. 기대효과 및 결론

BitLocker는 Windows에서 가장 현실적인 기본 FDE다. 사용자 입장에서는 자동으로 켜진 듯 보이지만, 내부적으로는 TPM, PCR, VMK, FVEK가 맞물려 동작한다. 이 구조 덕분에 분실·도난 상황에서 저장 데이터 유출을 크게 줄일 수 있다.

결론적으로 BitLocker는 단순한 "디스크 잠금"이 아니라, **장비 신뢰와 데이터 보호를 연결하는 Windows 엔드포인트 보안의 출발점**이다.

- **📢 섹션 요약 비유**: BitLocker는 노트북의 안전벨트다. 평소엔 의식하지 않지만, 사고가 나면 생존을 가르는 장치다.
