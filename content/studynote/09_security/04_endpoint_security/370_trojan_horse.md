## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트로이목마(Trojan Horse)는 유용한 소프트웨어로 위장해 사용자가 스스로 설치하도록 유도하는 악성코드로, 사용자의 신뢰를 악용하는 사회공학(Social Engineering) 기반 위협이다.
> 2. **가치**: 자기복제 능력이 없어 바이러스·웜과 구분되지만, RAT·키로거·랜섬웨어·백도어 등 다양한 악성 페이로드의 배달 수단(delivery vehicle)으로 활용된다.
> 3. **판단 포인트**: 실행 전 해시 검증·공식 스토어 설치·코드 서명 확인이 핵심 예방책이며, 탐지는 행위 기반 분석(실행 후 비정상 행동 감지)이 시그니처 분석보다 효과적이다.

---

## Ⅰ. 개요 및 필요성

그리스 신화의 트로이 목마처럼, 트로이목마 악성코드는 정상적이고 유용한 프로그램으로 위장해 사용자가 자발적으로 설치하도록 유도한다. 게임 크랙(crack), 무료 소프트웨어, 이메일 첨부파일, 가짜 보안 업데이트 등이 주요 배포 경로다.

트로이목마의 핵심 특징은 **자기복제 불가**다. 바이러스(Virus)는 다른 파일을 감염시키고, 웜(Worm)은 네트워크를 통해 자동 전파하지만, 트로이목마는 혼자서는 퍼지지 않는다. 대신 피싱(Phishing), SEO 포이즈닝(SEO Poisoning), 드라이브-바이 다운로드(Drive-by Download) 같은 사회공학 기법으로 확산된다.

현대의 트로이목마는 단순 백도어를 넘어 RAT (Remote Access Trojan), 뱅킹 트로이목마(Banking Trojan), 스틸러(Stealer) 등으로 전문화·고도화되고 있다.

📢 **섹션 요약 비유**: 선물 상자처럼 예쁘게 포장됐지만 안에는 폭탄이 들어 있다—트로이목마는 신뢰를 무기로 쓴다.

---

## Ⅱ. 아키텍처 및 핵심 원리

| 유형 | 주요 기능 | 대표 악성코드 |
|:---|:---|:---|
| RAT (Remote Access Trojan) | 원격 제어·스파이 | DarkComet, njRAT |
| 뱅킹 트로이목마 | 금융 자격증명 탈취 | Zeus, Emotet, Dridex |
| 다운로더(Downloader) | 추가 악성코드 다운로드 | TrickBot |
| 드로퍼(Dropper) | 디스크에 페이로드 드롭 후 실행 | IcedID |
| 스틸러(Stealer) | 브라우저 쿠키·비밀번호 탈취 | Raccoon, RedLine |
| 클리퍼(Clipper) | 클립보드 암호화폐 주소 변조 | ClipBanker |

```text
┌──────────────────────────────────────────────────┐
│           트로이목마 감염 및 실행 흐름           │
├──────────────────────────────────────────────────┤
│                                                  │
│  [사용자] ← 위장 소프트웨어 다운로드/실행        │
│      │                                           │
│      ▼                                           │
│  [정상 기능 실행]  +  [악성 페이로드 실행]       │
│                          │                       │
│              ┌───────────┼────────────┐          │
│              ▼           ▼            ▼          │
│         [백도어]    [정보 탈취]  [추가 악성코드]  │
│              │           │            │          │
│              └───────────┴────────────┘          │
│                          │                       │
│                    [C2 서버로 전송]               │
└──────────────────────────────────────────────────┘
```

**주요 배포 경로**  
1. 피싱 이메일 첨부파일(Office 매크로 악용)  
2. 크랙 소프트웨어·불법 콘텐츠 배포 사이트  
3. 악성 광고(Malvertising)를 통한 드라이브-바이 다운로드  
4. 가짜 보안 소프트웨어(Scareware)  
5. SEO 포이즈닝—검색 결과 상위 노출 악성 사이트  

📢 **섹션 요약 비유**: 트로이목마는 정문으로 들어오는 택배—배달부가 나쁜 사람인지 포장만 봐서는 알 수 없다.

## Ⅲ. 비교 및 연결

| 항목 | 트로이목마 | 바이러스 | 웜 |
|:---|:---|:---|:---|
| 자기복제 | 없음 | 파일 감염으로 복제 | 네트워크로 자동 전파 |
| 확산 방식 | 사용자 실행 필요 | 파일 열기 | 취약점 자동 악용 |
| 위장 | 정상 프로그램으로 위장 | 실행 파일에 기생 | 독립 실행 |
| 주요 목적 | 악성 페이로드 전달 | 파일 파괴·전파 | 대역폭 소모·전파 |
| 제거 방법 | 파일 삭제 + 페이로드 제거 | 감염 파일 치료 | 패치 + 격리 |

📢 **섹션 요약 비유**: 바이러스는 감기처럼 저절로 퍼지고, 웜은 전화번호부 전체에 메시지를 보내고, 트로이목마는 피해자 스스로 문을 열어준다.

---

## Ⅳ. 실무 적용 및 기술사 판단

**예방 전략**  
1. 소프트웨어는 공식 스토어·제조사 사이트에서만 설치  
2. 실행 파일 다운로드 전 VirusTotal로 해시 검증  
3. 코드 서명(Code Signing) 없는 실행 파일 차단  
4. AppLocker 또는 WDAC (Windows Defender Application Control)으로 허가된 앱만 실행  
5. 사용자 보안 교육—피싱 인식 훈련 정기 실시  
6. 이메일 첨부파일 샌드박스(Sandbox) 분석 후 전달  

**탐지 전략**  
- 행위 기반 EDR: 비정상 프로세스 생성, 레지스트리 변조, 네트워크 연결 패턴 탐지  
- AMSI (Antimalware Scan Interface) 통합 스크립트 실행 시 실시간 분석  
- 네트워크 트래픽에서 알 수 없는 C2 도메인 통신 탐지  

📢 **섹션 요약 비유**: 공식 마트에서 구입한 약과 길거리에서 받은 약—출처(공식 스토어) 확인이 가장 강력한 예방책이다.

---

## Ⅴ. 기대효과 및 결론

트로이목마는 기술적 취약점이 아닌 사람의 신뢰를 공격한다. 따라서 기술적 대응(코드 서명, 앱 허용 목록)과 비기술적 대응(사용자 보안 인식 교육)을 함께 적용해야 효과적이다. 현대 사이버 공격의 상당수가 트로이목마를 초기 침투 벡터로 사용하기 때문에, 이를 막는 것이 전체 보안 체계의 첫 번째 방어선이 된다.

📢 **섹션 요약 비유**: 성벽을 아무리 높게 쌓아도 성 안의 사람이 문을 열면 끝이다—사람이 가장 취약한 링크다.

### 👶 어린이를 위한 3줄 비유 설명
트로이목마는 선물처럼 보이지만 안에 나쁜 프로그램이 숨어 있는 가짜 앱이에요.  
직접 클릭해서 설치해야 실행되기 때문에, 해커는 멋있어 보이는 게임이나 프로그램으로 위장해요.  
공식 앱스토어에서만 설치하고, 모르는 파일은 절대 열지 않는 것이 최고의 방어예요.
