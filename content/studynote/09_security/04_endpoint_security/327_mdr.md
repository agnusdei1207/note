## 핵심 인사이트 (3줄 요약)
> 1. **본질**: MDR(Managed Detection and Response)은 EDR/XDR 기술에 24/7 보안 전문가 서비스를 결합하여, 고도화된 위협 탐지·대응을 외주화(Outsourcing)하는 보안 서비스 모델이다.
> 2. **가치**: 보안 전문 인력이 부족한 중소·중견 기업이 세계 최정상 수준의 위협 탐지·헌팅·사고 대응 역량을 서비스 형태로 즉시 확보할 수 있다.
> 3. **판단 포인트**: MDR 선택 시 SLA(Service Level Agreement) 응답 시간, 원격 대응(Remote Remediation) 권한 범위, 위협 헌팅(Threat Hunting) 포함 여부가 품질을 좌우하는 핵심 지표다.

---

## Ⅰ. 개요 및 필요성

MDR(Managed Detection and Response)은 단순히 보안 도구를 제공하는 것이 아니라, **전문 보안팀이 직접 운영**하는 서비스 형태로 제공된다. 고객사는 MDR 제공업체(Provider)의 에이전트를 설치하고, 이후 24/7 운영은 MDR 팀이 담당한다. SOC(Security Operations Center)를 운영할 예산과 인력이 부족한 조직에게 이상적인 모델이다.

사이버보안 인력 부족은 전 세계적으로 심각하다. ISC²(International Information System Security Certification Consortium)에 따르면 2023년 기준 글로벌 사이버보안 인력 부족이 350만 명에 달한다. 보안 전문가를 채용·유지하는 비용과 24/7 SOC 운영 비용을 감안하면, 중소기업이 자체 운영하는 것은 현실적으로 불가능한 경우가 많다.

MDR 서비스는 위협 탐지(Threat Detection), 조사(Investigation), 위협 헌팅(Threat Hunting), 사고 대응(Incident Response)을 포함한다. 고급 서비스는 악성코드 제거, 격리 해제, 취약점 패치 권고까지 제공한다. MDR은 MSSP(Managed Security Service Provider)와 달리 단순 모니터링이 아닌 **능동적 대응**이 핵심이다.

📢 **섹션 요약 비유**: MDR은 "24시간 운영되는 전문 경비 회사"와 같다. 집에 경보기(EDR)만 달지 않고, 경보기를 전문 경비 회사에 연결하여 실시간 출동까지 보장하는 것이다.

## Ⅱ. 아키텍처 및 핵심 원리

### MDR 서비스 구성 요소

| 구성 요소 | 내용 | 수준 |
|:---------|:-----|:-----|
| 탐지(Detection) | EDR/XDR 기반 24/7 알람 모니터링 | 기본 |
| 조사(Investigation) | 알람 분류·컨텍스트 분석·위협 평가 | 기본 |
| 위협 헌팅(Hunting) | 능동적 IOC/TTP 검색 | 고급 |
| 사고 대응(IR) | 격리·제거·복구 지원 | 고급 |
| 리포팅 | 주간/월간 보안 현황 보고서 | 기본 |

```
┌─────────────────────────────────────────────────────┐
│                MDR 서비스 모델                      │
│                                                     │
│  고객사 환경                                        │
│  ┌──────────────────────────────┐                   │
│  │  엔드포인트/네트워크/클라우드  │                   │
│  │  ┌──────┐ ┌──────┐ ┌──────┐ │                   │
│  │  │ EDR  │ │ NDR  │ │Cloud │ │                   │
│  │  └──────┘ └──────┘ └──────┘ │                   │
│  └─────────────┬────────────────┘                   │
│                │ 텔레메트리 전송 (암호화)             │
│                ▼                                    │
│  MDR 서비스 센터                                    │
│  ┌──────────────────────────────┐                   │
│  │  24/7 SOC 분석가 팀          │                   │
│  │  위협 헌팅 │ IR │ 리포팅     │                   │
│  └──────────────────────────────┘                   │
└─────────────────────────────────────────────────────┘
```

MDR의 핵심 운영 원칙은 **OODA 루프(Observe, Orient, Decide, Act)**의 최적화다. 탐지부터 대응까지 MTTD(Mean Time To Detect)와 MTTR(Mean Time To Respond)을 최소화하는 것이 SLA의 핵심 지표다. 우수 MDR 서비스는 MTTD 15분 이내, MTTR 1시간 이내를 보장한다.

📢 **섹션 요약 비유**: MDR의 OODA 루프는 "응급실 의료팀"처럼 작동한다. 환자 도착(탐지) → 상태 평가(조사) → 처치 결정(대응 계획) → 즉각 처치(대응)가 분 단위로 이루어진다.

## Ⅲ. 비교 및 연결

### MDR vs MSSP vs SOC 비교

| 특성 | MSSP | MDR | 내부 SOC |
|:----|:-----|:----|:---------|
| 서비스 방식 | 모니터링 중심 | 탐지+대응 | 탐지+대응+전략 |
| 기술 수준 | 보통 | 높음 | 높음~매우 높음 |
| 대응 수준 | 알림 전달 | 원격 대응 | 직접 대응 |
| 비용 | 낮음 | 중간 | 높음 |
| 맞춤화 | 낮음 | 중간 | 높음 |

MSSP는 전통적으로 로그 수집·모니터링·컴플라이언스 리포팅에 집중하는 반면, MDR은 능동적 위협 헌팅과 사고 대응이 차별점이다. 내부 SOC는 최고 수준의 맞춤화가 가능하지만 구축·운영 비용이 매우 높다.

📢 **섹션 요약 비유**: MSSP가 "CCTV 화면 보는 아르바이트생"이라면, MDR은 "훈련된 경비원", 내부 SOC는 "자체 경찰서"다.

## Ⅳ. 실무 적용 및 기술사 판단

**MDR 서비스 선택 체크리스트:**
- 대응 SLA: MTTD/MTTR 보장 수준 계약 명시
- 원격 대응 권한: 격리·프로세스 종료 등 원격 조치 포함 여부
- 위협 헌팅: 능동적 헌팅 시간 보장 (월 N시간)
- 리포팅: 주기적 위협 트렌드 분석 및 개선 권고 포함
- 데이터 주권: 텔레메트리 데이터의 저장 위치 및 법적 준수 확인

**MDR 도입 시 주의 사항:**
- 내부 보안 담당자와의 협업 프로세스 명확히 정의
- 에스컬레이션(Escalation) 체계 수립
- 계약 종료 시 데이터 반환 및 삭제 조항 확인

📢 **섹션 요약 비유**: MDR 계약은 "경비 회사 계약서"다. 응답 시간, 어디까지 조치할 수 있는지, 계약 종료 시 열쇠 반납 조항까지 꼼꼼히 확인해야 한다.

---

## Ⅴ. 기대효과 및 결론

MDR 도입 시 기대할 수 있는 주요 효과는 보안 인력 부족 해소, MTTD/MTTR 대폭 단축, 위협 헌팅을 통한 잠복 위협 조기 발견이다. 가트너에 따르면 2025년까지 중소·중견기업의 50% 이상이 MDR 서비스를 활용할 것으로 예측된다.

결론적으로, MDR은 "보안의 민주화"를 실현하는 모델이다. 예산과 인력이 제한된 조직도 대기업 수준의 보안 운영 역량을 확보할 수 있다.

📢 **섹션 요약 비유**: MDR은 "보안계의 클라우드 서비스"다. 서버를 직접 사는 대신 클라우드를 쓰듯, 보안팀을 직접 채용하는 대신 MDR을 구독한다.

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| EDR | 기반 기술 | MDR의 핵심 탐지·대응 도구 |

### 👶 어린이를 위한 3줄 비유 설명
1. MDR은 경비 회사에 우리 집 안전을 맡기는 것과 같아. 경보기만 달지 않고, 전문가들이 24시간 지켜봐.
2. 이상한 게 생기면 바로 출동해서 해결해줘.
3. 보안 전문가를 직접 고용할 돈이 없는 회사도 이렇게 하면 전문가 수준의 보호를 받을 수 있어.
