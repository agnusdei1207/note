## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 힙 펑수이(Heap Feng Shui)는 특정 순서로 메모리 할당·해제를 수행하여 힙의 레이아웃을 공격자가 원하는 형태로 정밀하게 조작하는 기법으로, 확률적 힙 스프레이와 달리 결정론적(deterministic) 공격을 가능하게 한다.
> 2. **가치**: UAF(Use-After-Free), Double Free, Type Confusion 취약점의 신뢰성을 높이기 위해 힙 레이아웃을 예측 가능하게 만드는 전처리 단계로 사용된다.
> 3. **판단 포인트**: 힙 펑수이 방어는 힙 청크 격리(Guard Page), 할당자 무작위화(ASLR heap), 타입 안전 할당자(PartitionAlloc) 도입으로 레이아웃 예측을 어렵게 하는 것이 핵심이다.

---

## Ⅰ. 개요 및 필요성

힙 펑수이(Heap Feng Shui)는 2007년 Alexander Sotirov가 Internet Explorer 브라우저 익스플로잇에서 처음 문서화한 기법이다. "풍수(風水)"라는 이름처럼 힙 메모리 공간을 공격자 의도에 맞게 "배치"하는 기술이다. 특정 크기의 객체들을 전략적으로 할당·해제하여 메모리 조각화를 유도하고, 취약한 객체와 공격 목표 객체를 인접하게 배치한다.

예를 들어 UAF 공격에서 힙 펑수이를 활용하면: ① 크기 A 객체 100개 할당 → ② 짝수 번째 해제 → ③ 취약한 크기 A 객체 할당(비어 있는 짝수 자리 중 하나에 배치됨) → ④ 취약점 트리거로 해제 → ⑤ 공격 페이로드로 같은 크기 블록 재할당. 이 과정에서 힙 펑수이가 없으면 ⑤의 재할당 위치가 예측 불가능하다.

📢 **섹션 요약 비유**: 힙 펑수이는 체스판(힙) 위의 말(객체)을 원하는 위치로 배치한 다음 게임을 시작하는 것—체스를 두기 전에 이미 승리 포지션을 만들어 놓는다.

## Ⅱ. 아키텍처 및 핵심 원리

### 힙 레이아웃 조작 단계

| 단계 | 작업 | 목적 |
|:---|:---|:---|
| 1. 홀 생성 | 특정 크기 객체 할당 후 일부 해제 | 원하는 크기의 빈 청크 생성 |
| 2. 정렬 | 추가 할당으로 힙 단편화 최소화 | 타겟 객체 위치 예측 가능화 |
| 3. 배치 | 취약 객체와 공격 목표 객체 인접 배치 | 오버플로우/UAF 영향 범위 제어 |
| 4. 트리거 | 취약점 발생 | 인접 객체 데이터 조작 |

```
┌─────────────────────────────────────────────────────┐
│          힙 펑수이 레이아웃 조작 예시                │
├─────────────────────────────────────────────────────┤
│  초기:  [A][B][A][B][A][B][A][B]  (A: 타겟 크기)   │
│                                                     │
│  짝수   [A][ ][A][ ][A][ ][A][ ]  (B 해제)         │
│  해제:                                              │
│                                                     │
│  취약   [A][V][A][ ][A][ ][A][ ]  (V: 취약 객체)   │
│  배치:                                              │
│                                                     │
│  재할   [A][V][A][P][A][P][A][P]  (P: 페이로드)     │
│  당:    V 해제 후 P가 V 자리에 배치됨               │
└─────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 빈 주차 공간(해제된 청크)을 미리 특정 위치에 만들어두면, 원하는 차(페이로드)가 정확히 그 자리에 주차된다.

## Ⅲ. 비교 및 연결

| 기법 | 결정론적 여부 | 사전 조작 | 대상 취약점 |
|:---|:---|:---|:---|
| 힙 스프레이 | 확률적 | 대량 할당 | UAF, 제어 흐름 탈취 |
| 힙 펑수이 | 결정론적 | 정밀 배치 | UAF, Double Free, 오버플로우 |
| Grooming | 결정론적 | OS/커널 힙 조작 | 커널 UAF, 커널 오버플로우 |

힙 그루밍(Heap Grooming)은 iOS/macOS 커널 익스플로잇에서 힙 펑수이와 유사하게 커널 힙을 조작하는 기법을 지칭한다.

📢 **섹션 요약 비유**: 힙 스프레이가 "씨앗 뿌리기"라면, 힙 펑수이는 "분재 가꾸기"—정밀한 손길로 원하는 형태를 만든다.

## Ⅳ. 실무 적용 및 기술사 판단

Chrome의 PartitionAlloc은 타입별로 힙을 분리하여 서로 다른 타입의 객체가 인접 배치되는 것을 방지한다. 이는 UAF+힙펑수이 공격에서 "재할당 단계"를 어렵게 만드는 핵심 방어이다. Windows의 Low Fragmentation Heap(LFH)도 무작위 청크 할당 순서를 도입하여 예측 가능한 레이아웃을 방해한다.

기술사 답안에서는 힙 펑수이를 "UAF 취약점의 신뢰성을 높이는 익스플로잇 프리미티브"로 정의하고, PartitionAlloc·타입 격리·힙 무작위화를 방어 삼각형으로 제시한다.

📢 **섹션 요약 비유**: PartitionAlloc은 주차장을 차종별로 분리하는 것—SUV(객체 A)와 세단(객체 B)을 같은 구역에 섞어두지 않으면, 풍수 배치가 불가능해진다.

---

## Ⅴ. 기대효과 및 결론

힙 펑수이 방어 기술의 발전은 현대 브라우저 메모리 안전성의 핵심 토대가 되었다. PartitionAlloc, MiraclePtr, XOR 링크드 리스트 등의 힙 강화 기술은 모두 결정론적 힙 조작을 어렵게 만들기 위한 노력의 산물이다. 궁극적으로 메모리 안전 언어로의 전환이 이 공격 클래스를 근본적으로 차단한다.

📢 **섹션 요약 비유**: 아무리 풍수 전문가(공격자)라도 집(힙)의 구조가 매번 바뀌고, 같은 종류끼리만 모여 살면(타입 격리) 배치를 예측할 수 없다.

### 👶 어린이를 위한 3줄 비유 설명
- 힙 펑수이는 나쁜 사람이 컴퓨터 메모리 방(힙)을 미리 예쁘게 정돈해서 함정을 파는 거야.
- 마치 블록을 특정 순서로 쌓아두면 원하는 블록이 정확히 원하는 자리에 떨어지게 만드는 것처럼.
- 방어하려면 같은 종류끼리만 한 공간에 두고(PartitionAlloc) 공간 위치를 계속 바꿔야(ASLR) 해!
