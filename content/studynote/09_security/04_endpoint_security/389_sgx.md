## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Intel SGX (Software Guard Extensions)는 애플리케이션 코드와 데이터를 엔클레이브(Enclave)라는 암호화된 메모리 영역에 격리·실행해 OS·하이퍼바이저·BIOS 등 더 높은 특권 소프트웨어조차 접근할 수 없도록 하는 하드웨어 기반 신뢰 실행 환경(TEE)이다.
> 2. **가치**: "클라우드 제공자조차 신뢰하지 않는" 제로트러스트 컨피덴셜 컴퓨팅을 가능하게 하며, 암호화 키·머신러닝 모델·개인정보 처리 등 극도로 민감한 연산을 안전하게 위탁 실행할 수 있다.
> 3. **판단 포인트**: SGX는 Spectre·MDS·LVI (Load Value Injection) 같은 마이크로아키텍처 취약점에 취약함이 밝혀졌으며, DCAP (Data Center Attestation Primitives) 기반 원격 증명이 핵심 보안 메커니즘이다.

---

## Ⅰ. 개요 및 필요성

전통적 보안 모델은 OS와 하이퍼바이저를 신뢰의 기반으로 삼는다. 하지만 클라우드 환경에서 테넌트는 호스트 OS·하이퍼바이저가 누구에 의해 어떻게 관리되는지 알 수 없다. Intel SGX는 이 문제를 해결하기 위해 CPU 하드웨어 자체를 신뢰 앵커로 삼고, 엔클레이브 내부 데이터는 CPU 밖으로 나갈 때 항상 암호화되도록 보장한다.

2015년 Skylake CPU부터 상용화된 SGX는 DRM (디지털 저작권 관리)·금융 거래·의료 데이터 처리·비밀 컴퓨팅 등 다양한 분야에서 활용됐다. Azure Confidential Computing, Alibaba Cloud TEE 등이 SGX 기반 서비스를 제공한다.

📢 **섹션 요약 비유**: SGX는 투명한 금고—OS도 하이퍼바이저도 금고 안을 볼 수 없지만, CPU는 금고 안에서 안전하게 연산을 수행한다.

## Ⅱ. 아키텍처 및 핵심 원리

| 구성 요소 | 역할 | 설명 |
|:---|:---|:---|
| EPC (Enclave Page Cache) | 암호화 메모리 | 최대 256MB(SGX1), GB 단위(SGX2) |
| MEE (Memory Encryption Engine) | 메모리 암호화 | AES-CTR로 DRAM 암호화 |
| EPCM (EPC Metadata) | 접근 제어 | 엔클레이브 페이지 허가 기록 |
| 원격 증명 | 무결성 증명 | DCAP/EPID로 외부 검증자에게 증명 |
| SGX SDK | 개발 도구 | 신뢰/비신뢰 코드 분리 개발 |

```text
┌──────────────────────────────────────────────────────┐
│              SGX 메모리 격리 구조                    │
├──────────────────────────────────────────────────────┤
│                                                      │
│  [일반 DRAM]                 [EPC - SGX 메모리]      │
│  OS, 하이퍼바이저,           ┌─────────────────────┐ │
│  다른 프로세스 메모리         │  엔클레이브 코드    │ │
│                              │  엔클레이브 데이터  │ │
│                              │  (CPU 밖 = 암호화)  │ │
│                              └─────────────────────┘ │
│                                        │              │
│                              CPU 내부 복호화          │
│                              (MEE - AES-CTR)          │
│                                                      │
│  ← OS/하이퍼바이저 접근 → [EPCM이 차단]              │
│  ← DMA 접근 → [IOMMU가 차단]                         │
│  ← 물리 메모리 덤프 → [암호화로 보호]                │
└──────────────────────────────────────────────────────┘
```

**원격 증명 흐름 (DCAP 기반)**  
1. 엔클레이브 내부에서 Report 생성 (CPU 서명 포함)  
2. Quote 생성 (Quoting Enclave가 Report를 Quote로 변환)  
3. PCK (Provisioning Certification Key) 인증서로 Quote 서명  
4. 원격 검증자가 Intel PCS에서 인증서 검증 후 신뢰 결정  

📢 **섹션 요약 비유**: SGX 원격 증명은 의사 면허증 확인—"이 의사가 진짜 의사인지(엔클레이브가 진짜 SGX인지)" CPU 제조사(Intel)가 공증한다.

## Ⅲ. 비교 및 연결

| 항목 | Intel SGX | AMD SEV | Intel TXT |
|:---|:---|:---|:---|
| 격리 수준 | 프로세스(엔클레이브) | VM 전체 | 하이퍼바이저/OS |
| 공격자 모델 | 악성 OS/하이퍼바이저 | 악성 하이퍼바이저 | 오염된 부팅 스택 |
| 메모리 암호화 | EPC (선택 메모리) | 전체 VM 메모리 | 없음 |
| 원격 증명 | DCAP/EPID | SEV-SNP 증명 | TPM Quote |

📢 **섹션 요약 비유**: SGX는 은행 금고 안의 개인 사물함(엔클레이브), SEV는 은행 금고실 자체 암호화, TXT는 은행 직원 채용 검증—각각 다른 계층을 보호한다.

## Ⅳ. 실무 적용 및 기술사 판단

**SGX 개발 모델**  
- 애플리케이션을 Trusted(신뢰) + Untrusted(비신뢰) 파트로 분리  
- 민감한 연산(암호화 키 처리, 인증 로직)은 Trusted 엔클레이브에 배치  
- ECALL/OCALL 인터페이스로 두 파트 간 통신  

**보안 한계**  
- Spectre/MDS/LVI: 사이드 채널로 엔클레이브 메모리 유출 가능  
- Foreshadow (L1TF): L1 캐시 통해 엔클레이브 데이터 유출  
- 이에 따라 Intel은 SGX2 + 마이크로코드 완화 지속 업데이트  

**Intel 12세대 이후 SGX 방향**: 데이터센터 Xeon 계열에서는 지속 지원, 일부 소비자 CPU에서는 제거 추세.

📢 **섹션 요약 비유**: SGX는 방탄 금고지만 환기구(사이드 채널)가 취약했다—지속적 패치로 구멍을 막고 있다.

---

## Ⅴ. 기대효과 및 결론

SGX는 컨피덴셜 컴퓨팅(Confidential Computing)의 선구적 기술로, "클라우드 제공자를 신뢰하지 않아도 안전하게 컴퓨팅"하는 패러다임을 실현했다. 다수의 마이크로아키텍처 취약점이 발견됐지만, 지속적인 마이크로코드 업데이트와 프로그래밍 가이드라인 준수로 보안 강도를 유지하고 있다. AMD SEV·Intel TDX와 함께 차세대 클라우드 보안 기반 기술로 발전 중이다.

📢 **섹션 요약 비유**: SGX는 검은 상자(블랙박스) 처리 서비스—내가 직접 서버를 못 보더라도, 연산 결과가 조작되지 않았음을 수학적으로 보장한다.

### 👶 어린이를 위한 3줄 비유 설명
SGX는 컴퓨터 안에 아무도 열어볼 수 없는 특별한 잠금 방(엔클레이브)을 만드는 기술이에요.  
운영체제(Windows, Linux)도 그 방 안을 볼 수 없고, 방 안에서 처리된 결과만 밖으로 나와요.  
클라우드 서버를 빌려 쓸 때 서버 주인도 내 비밀 데이터를 못 보게 하는 데 사용돼요.
