## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 힙 스프레이(Heap Spray)는 공격자가 힙 메모리 대부분을 쉘코드·NOP 슬라이드로 채워, ASLR로 주소를 예측할 수 없어도 무작위로 실행 흐름을 유도하면 명령이 실행될 확률을 높이는 기법이다.
> 2. **가치**: 브라우저·PDF 리더처럼 스크립트로 대량 객체를 할당할 수 있는 환경에서 특히 유효하며, UAF(Use-After-Free)·Type Confusion과 결합하면 신뢰성 높은 익스플로잇이 된다.
> 3. **판단 포인트**: Heap Spray 방어는 힙 무작위화(ASLR 강화)·NX 힙·JIT 컴파일러 격리·메모리 압박 탐지로 다층 구성하며, JavaScript 엔진의 JIT 스프레이 변형에는 JIT 하드닝이 추가로 필요하다.

---

## Ⅰ. 개요 및 필요성

힙 스프레이는 2001년 Neel Mehta, Halvar Flake 등이 처음 문서화한 기법으로, 스크립트 언어(JavaScript, ActionScript)나 OLE 자동화로 수백~수천 개의 대형 문자열·배열을 할당하여 힙을 원하는 패턴으로 가득 채운다. NOP 슬라이드로 둘러싸인 페이로드가 힙의 많은 부분을 차지하면, 제어 흐름이 힙의 어느 곳으로 튀어도 NOP 슬라이드를 타고 내려와 페이로드를 실행하게 된다.

브라우저 UAF(Use-After-Free) 취약점 익스플로잇에서 힙 스프레이는 해제된 객체의 메모리를 공격자 데이터로 빠르게 재점유(reclaim)하는 용도로 사용된다. 특히 Internet Explorer, Adobe Flash, PDF 처리 엔진에서 실제 사고로 이어졌다.

📢 **섹션 요약 비유**: 힙 스프레이는 광활한 들판(힙)에 낙하산 병력(페이로드)을 가득 뿌려—어디에 착지해도 아군이 있어 전투(코드 실행)가 가능하게 하는 전략이다.

## Ⅱ. 아키텍처 및 핵심 원리

### 힙 스프레이 페이로드 구조

| 구성 요소 | 내용 | 크기 |
|:---|:---|:---|
| NOP 슬라이드 | `...` | 수십~수백 KB |
| 쉘코드/페이로드 | 실제 공격 코드 | 수백 바이트~수 KB |
| 반복 패턴 | NOP+페이로드 블록 반복 | 블록 × N회 |
| 목표 주소 | 예측 가능한 힙 영역 | 예: 0x0c0c0c0c |

```
┌─────────────────────────────────────────────────────┐
│             힙 스프레이 메모리 배치                  │
├─────────────────────────────────────────────────────┤
│ [힙 낮은 주소]                                      │
│  NOP NOP NOP ... NOP [쉘코드] NOP NOP ... [쉘코드]  │
│  NOP NOP NOP ... NOP [쉘코드] NOP NOP ... [쉘코드]  │
│  (수백 MB 반복)                                     │
│  ↑ 제어 흐름이 여기로 오면 NOP를 타고 쉘코드 도달  │
│ [힙 높은 주소]                                      │
└─────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: NOP 슬라이드는 활주로—어디서 착륙해도 미끄러져 쉘코드(격납고)에 도착하게 설계됐다.

## Ⅲ. 비교 및 연결

| 기법 | 목적 | ASLR 필요 여부 | 결합 공격 |
|:---|:---|:---|:---|
| 힙 스프레이 | 주소 예측 대신 확률 향상 | 우회 기법 | UAF, Type Confusion |
| 힙 펑수이 | 힙 레이아웃 정밀 제어 | 우회 기법 | Double Free, UAF |
| JIT 스프레이 | JIT 코드 캐시에 페이로드 | JIT 캐시 활용 | JS 엔진 취약점 |

힙 스프레이는 확률적 접근이지만, 힙 펑수이(Heap Feng Shui)는 힙의 구조를 정밀하게 제어하여 결정론적(deterministic) 공격을 가능하게 한다.

📢 **섹션 요약 비유**: 힙 스프레이가 "씨앗을 뿌리는" 전략이라면, 힙 펑수이는 "정원을 설계하는" 전략이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

현대 브라우저는 힙 스프레이 방어를 위해 ① 힙 주소 무작위화(Heap ASLR), ② NX 힙(힙 메모리 실행 방지), ③ JIT 컴파일러 코드 무작위화, ④ 메모리 압박 탐지(할당 폭발 감지)를 구현한다. Chrome의 V8 엔진과 Firefox의 SpiderMonkey는 JIT 페이지를 별도 메모리 영역에 분리하고 W^X(Write XOR Execute) 원칙을 적용한다.

기술사 답안에서는 힙 스프레이를 "ASLR 우회의 확률적 접근"으로 정의하고, 방어 대책을 NX+힙무작위화+JIT격리의 3층으로 서술한다.

📢 **섹션 요약 비유**: 들판에 지뢰(힙 가드 페이지)를 무작위로 심고, 밤에 불을 끄면(힙 ASLR) 씨앗(스프레이)이 어디 떨어질지 예측할 수도 없고, 착지해도 폭발한다.

---

## Ⅴ. 기대효과 및 결론

현대 브라우저 보안의 발전—샌드박스, Site Isolation, JIT 하드닝, MiraclePtr—은 힙 스프레이와 UAF 공격에 대한 대응으로 추진되었다. 힙 스프레이 방어 기술의 발전이 브라우저 보안 전체의 성숙도를 높이는 핵심 동력 중 하나였다.

📢 **섹션 요약 비유**: 힙 스프레이 방어 역사는 "씨 뿌리는 자(공격자)"와 "들판을 계속 바꾸는 농부(방어자)" 사이의 긴 게임으로, 결국 더 스마트한 들판 설계(격리, 샌드박스)가 승리했다.

### 👶 어린이를 위한 3줄 비유 설명
- 힙 스프레이는 나쁜 사람이 컴퓨터 메모리(힙)에 나쁜 코드를 가득 채우는 거야.
- 어디에 코드가 실행되든 나쁜 코드가 나올 만큼 많이 채워두는 방법이야.
- 막으려면 메모리 위치를 계속 바꾸고(ASLR), 거기서 코드 실행 자체를 막아야(NX) 해!
