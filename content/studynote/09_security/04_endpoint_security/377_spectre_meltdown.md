## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Spectre와 Meltdown은 현대 CPU의 성능 최적화 기법인 투기적 실행(Speculative Execution)과 비순서 실행(Out-of-Order Execution)의 부작용을 악용해 다른 프로세스나 커널의 메모리를 사이드 채널(Side Channel)로 읽어내는 하드웨어 취약점이다.
> 2. **가치**: 소프트웨어 패치만으로는 완전 해결이 불가능하며 CPU 성능 저하를 감수해야 하므로, 하드웨어 보안 설계의 한계를 드러낸 역사적 사건이다.
> 3. **판단 포인트**: Meltdown은 커널 메모리 읽기(주로 Intel), Spectre는 다른 프로세스 메모리 읽기(Intel·AMD·ARM 모두 영향)로 영향 범위와 완화 방법이 다르다.

---

## Ⅰ. 개요 및 필요성

2018년 1월 공개된 Spectre(CVE-2017-5753, CVE-2017-5715)와 Meltdown(CVE-2017-5754)은 1990년대부터 설계된 현대 CPU의 근본적인 아키텍처 결함을 드러냈다. Google Project Zero, Graz University of Technology 등 여러 연구팀이 독립적으로 발견했다.

투기적 실행(Speculative Execution)은 CPU가 분기 예측(Branch Prediction)을 이용해 아직 필요하지 않은 연산을 미리 실행해두는 성능 최적화 기법이다. 예측이 틀리면 결과를 버리지만, 이미 캐시에 영향을 남겨두어—이 캐시 타이밍 차이가 바로 사이드 채널 공격의 매개체가 된다.

📢 **섹션 요약 비유**: 투기적 실행은 시험 문제를 보기 전에 미리 예상 답을 적어두는 것—틀려서 지우더라도, 지운 흔적(캐시 타이밍)을 보면 무슨 답을 썼는지 알 수 있다.

## Ⅱ. 아키텍처 및 핵심 원리

| 취약점 | CVE | 메커니즘 | 영향 CPU |
|:---|:---|:---|:---|
| Meltdown | CVE-2017-5754 | 커널 메모리 → 사용자 공간 읽기 | Intel (주로), 일부 ARM |
| Spectre v1 | CVE-2017-5753 | 경계 우회(Bounds Check Bypass) | Intel, AMD, ARM 모두 |
| Spectre v2 | CVE-2017-5715 | 분기 예측 주입(BTI) | Intel, AMD, ARM 모두 |

```text
┌────────────────────────────────────────────────────┐
│           Meltdown 공격 원리                       │
├────────────────────────────────────────────────────┤
│                                                    │
│  1. 공격자 코드가 커널 주소(K) 읽기 시도           │
│     → 정상적으로 권한 거부 예외 발생               │
│                                                    │
│  2. CPU는 예외 처리 전에 투기적으로 K의 값을 읽음  │
│     → 그 값(byte 'B')에 따라 probe_array[B * 4096] │
│        를 캐시에 로드 (투기적 실행)                │
│                                                    │
│  3. 예외 발생, 투기 실행 결과 폐기                 │
│     → 그러나 캐시에는 probe_array[B * 4096] 남아 있음│
│                                                    │
│  4. 공격자가 probe_array의 각 인덱스 접근 시간 측정│
│     → 빠른 접근 = 캐시 히트 = B 값 확인 가능      │
│                                                    │
│  결과: 커널 메모리 1바이트 = 256번 측정으로 추출   │
└────────────────────────────────────────────────────┘
```

**Flush+Reload 사이드 채널**: CPU 캐시를 비운 후(CLFLUSH) 특정 메모리 접근 시간을 나노초 단위로 측정해 캐시 히트/미스를 판별한다. 캐시 히트면 해당 메모리가 최근 접근됐다는 증거—이것이 비밀 정보 추출의 핵심 채널이다.

📢 **섹션 요약 비유**: CPU 캐시는 열람실의 메모—옆 사람이 어떤 책을 꺼내봤는지 메모 속도로 추측할 수 있다.

## Ⅲ. 비교 및 연결

| 항목 | Meltdown | Spectre |
|:---|:---|:---|
| 공격 대상 | 커널 메모리 | 다른 프로세스 메모리 |
| 경계 위반 방식 | 메모리 보호 우회 | 분기 예측 오염 |
| 완화 기법 | KPTI (페이지 테이블 분리) | 리타폴린 (Retpoline), 마이크로코드 업데이트 |
| 성능 저하 | 5-30% (I/O 집약적 워크로드) | 1-10% |
| 패치 가능성 | 소프트웨어 완전 완화 가능 | 부분 완화만 가능 |

📢 **섹션 요약 비유**: Meltdown은 커널의 일기장을 직접 훔쳐보는 것, Spectre는 옆방 사람의 대화를 벽 너머로 엿듣는 것—후자가 더 근본적인 문제다.

---

## Ⅳ. 실무 적용 및 기술사 판단

**완화 조치**  
1. **KPTI (Kernel Page Table Isolation)**: Meltdown 완화—커널과 사용자 페이지 테이블 분리, 성능 저하 동반  
2. **Retpoline**: Spectre v2 완화—컴파일러 기법으로 간접 분기를 무한 루프로 대체  
3. **마이크로코드 업데이트**: CPU 제조사 제공, 분기 예측 버퍼 격리  
4. **IBRS/IBPB/STIBP**: Intel·AMD의 분기 예측 격리 하드웨어 기능  
5. 클라우드 환경에서는 VM 간 격리를 위해 하이퍼바이저 패치 우선 적용 필수  

**성능 영향**: 데이터베이스·파일 서버에서 시스템 콜 빈도가 높은 워크로드는 KPTI 적용 후 최대 30% 성능 저하 가능—패치 vs 성능 트레이드오프를 기술사 답안에서 명시한다.

📢 **섹션 요약 비유**: 은행 금고(커널)를 완전히 다른 건물로 이전(KPTI)하면 안전하지만 출금에 시간이 더 걸린다—보안과 성능의 트레이드오프.

---

## Ⅴ. 기대효과 및 결론

Spectre/Meltdown은 30년간 쌓인 CPU 아키텍처의 숨겨진 가정을 무너뜨린 사건이다. 이후 MDS, SWAPGS, L1TF 등 유사 마이크로아키텍처 취약점이 연속으로 발견됐다. 근본적 해결은 하드웨어 설계 단계에서의 투기적 실행 보안 검증이며, 현재는 완화 기법의 지속적 업데이트와 마이크로코드 패치가 최선이다.

📢 **섹션 요약 비유**: Spectre/Meltdown은 빠른 자동차(고성능 CPU)에 브레이크 설계 결함이 있음을 발견한 것—이미 팔린 자동차를 모두 수리(패치)해야 하지만 완벽한 수리는 불가능하다.
