## 핵심 인사이트 (3줄 요약)
> 1. **본질**: EPP(Endpoint Protection Platform)는 단말에서 악성코드·위협을 사전에 예방하기 위해 시그니처·머신러닝·행위 분석·샌드박스를 통합한 플랫폼이다.
> 2. **가치**: 1세대 AV(Anti-Virus)의 시그니처 한계를 넘어, 알려지지 않은(zero-day) 위협까지 예방 중심으로 대응하며 EDR과 결합 시 탐지·대응까지 완성한다.
> 3. **판단 포인트**: Gartner 기준 EPP는 "예방"에 집중하고 EDR은 "탐지·대응"에 집중한다. 실무에서는 EPP+EDR 통합 플랫폼으로 구현하는 것이 표준이다.

---

## Ⅰ. 개요 및 필요성

EPP(Endpoint Protection Platform)는 엔드포인트에서 발생하는 사이버 위협을 사전에 탐지하고 차단하기 위한 통합 보안 플랫폼이다. 기존 AV(Anti-Virus)가 시그니처(signature) 데이터베이스와의 비교 방식에만 의존하여 알려진 악성코드만 탐지할 수 있었던 한계를 극복하기 위해 등장했다.

EPP는 시그니처 기반 탐지에 더해 휴리스틱(heuristic) 분석, 머신러닝(Machine Learning) 모델, 클라우드 기반 위협 인텔리전스, 샌드박스(sandbox) 동적 분석을 결합하여 신·변종 악성코드에도 대응한다. 또한 방화벽(Personal Firewall), 디바이스 제어(Device Control), 데이터 암호화, 취약점 패치 관리 등 다양한 예방 기능을 단일 에이전트로 통합 제공한다.

현대 EPP는 클라우드 관리 콘솔과 연동되어 전사 단말에 대한 정책 배포·상태 모니터링·인시던트 리포트를 중앙에서 처리한다. EPP는 EDR(Endpoint Detection and Response)과 함께 배포되어 예방(EPP)과 탐지·대응(EDR)의 이중 방어 체계를 구성한다.

📢 **섹션 요약 비유**: EPP는 건물 입구의 보안 검색대처럼, 위협이 안으로 들어오기 전에 다양한 방법으로 검사하고 차단하는 예방 전문가다.

## Ⅱ. 아키텍처 및 핵심 원리

### EPP 핵심 기능 구성

| 기능 모듈 | 역할 | 탐지 방식 |
|:---|:---|:---|
| 시그니처 엔진 | 알려진 악성코드 탐지 | 해시·패턴 비교 |
| 머신러닝 엔진 | 미지 위협 예측 분류 | 정적 분석 모델 |
| 휴리스틱 분석 | 의심 행위 탐지 | 규칙 기반 행위 평가 |
| 샌드박스 | 동적 실행 분석 | 가상 환경 실행 |
| 클라우드 룩업 | 신규 위협 인텔리전스 | TI 피드 연동 |

```
┌─────────────────────────────────────────────────────┐
│              EPP 아키텍처 (단말 에이전트)            │
├──────────────────────────────────────────────────────┤
│  파일 접근  →  시그니처 검사  →  ML 분류             │
│                     │                                │
│             샌드박스 동적 분석                       │
│                     │                                │
│         허용 / 격리(Quarantine) / 차단               │
├──────────────────────────────────────────────────────┤
│     클라우드 관리 콘솔 (정책·리포트·업데이트)        │
└──────────────────────────────────────────────────────┘
```

EPP 에이전트는 파일 실행 전 정적 분석(static analysis)과 실행 중 동적 분석(dynamic analysis)을 모두 수행한다. 정적 분석은 파일 해시, PE(Portable Executable) 구조, 문자열 패턴을 검사하고, 동적 분석은 샌드박스에서 실제 실행하여 레지스트리 변경·네트워크 통신·프로세스 생성 등 행위를 관찰한다.

📢 **섹션 요약 비유**: EPP는 입국 심사관처럼, 여권(시그니처)을 확인하고 얼굴(머신러닝)을 스캔하고 짐(샌드박스)까지 열어보는 다단계 검증 시스템이다.

## Ⅲ. 비교 및 연결

| 구분 | AV (Anti-Virus) | EPP (Endpoint Protection Platform) |
|:---|:---|:---|
| 탐지 방식 | 시그니처 단일 | 시그니처+ML+행위+샌드박스 |
| 제로데이 대응 | 미흡 | 머신러닝으로 부분 대응 |
| 통합 기능 | 없음 | 방화벽·암호화·패치 통합 |
| 관리 방식 | 로컬/서버 | 클라우드 중앙 관리 |
| EDR 연동 | 없음 | 자연스러운 통합 |

EPP와 EDR은 상호 보완적이다. EPP가 위협을 사전에 차단하지 못했을 때 EDR이 사후 탐지·격리·포렌식을 담당한다. 현대 보안 솔루션은 EPP+EDR을 단일 에이전트로 통합(예: CrowdStrike Falcon, SentinelOne)하여 성능 오버헤드를 최소화한다.

📢 **섹션 요약 비유**: AV가 자물쇠라면, EPP는 자물쇠+CCTV+경보기+울타리를 모두 갖춘 스마트 홈 보안 시스템이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기업 EPP 도입 시 핵심 고려사항은 ① 에이전트 성능 오버헤드(CPU·메모리 사용량), ② 오탐율(False Positive Rate), ③ 클라우드 의존도(오프라인 환경 지원 여부), ④ EDR과의 통합 수준이다. 특히 산업 제어 시스템(ICS/OT) 환경에서는 실시간 운영에 영향을 줄 수 있는 샌드박스 분석 시간을 사전 튜닝해야 한다.

기술사 답안에서는 EPP를 "예방 중심의 1차 방어선"으로, EDR을 "탐지·대응 중심의 2차 방어선"으로 포지셔닝하고, 두 기술의 시너지로 Defense-in-Depth(심층 방어)를 구현한다고 서술하면 고점 답안이 된다.

📢 **섹션 요약 비유**: EPP는 병원의 예방 접종처럼 사전에 면역력을 키우고, EDR은 응급실처럼 침입 후 즉각 치료하는 역할이다.

---

## Ⅴ. 기대효과 및 결론

EPP 도입으로 악성코드의 엔드포인트 침투 성공률을 획기적으로 낮출 수 있다. 특히 머신러닝 기반 분류는 새로운 랜섬웨어 변종이 시그니처 업데이트 없이도 차단되는 효과를 제공한다. 클라우드 관리 콘솔을 통한 중앙화된 정책 배포는 수천 대의 단말을 일관성 있게 관리하게 해준다.

향후 EPP는 AI 기반 위협 예측, 제로트러스트 통합, XDR(Extended Detection and Response) 플랫폼으로 발전하며, 엔드포인트 보안의 지능화·자동화를 주도할 것이다.

📢 **섹션 요약 비유**: EPP는 스마트 면역 시스템처럼, 기존에 본 적 없는 바이러스도 패턴을 분석해 미리 막아내는 진화된 방어막이다.

### 👶 어린이를 위한 3줄 비유 설명
- EPP는 학교 입구의 선생님처럼, 나쁜 것(악성코드)이 들어오기 전에 미리 검사해줘.
- 단순히 얼굴(시그니처)만 보는 게 아니라 행동(머신러닝)도 보고 가방(샌드박스)도 열어봐.
- 요즘은 EPP와 EDR이 같이 일해서, 막지 못한 나쁜 것도 바로 잡아낼 수 있어!
