## 핵심 인사이트 (3줄 요약)
> 1. **본질**: EPP(Endpoint Protection Platform)는 AV·방화벽·HIPS·애플리케이션 제어를 단일 플랫폼으로 통합하여 알려진 위협을 예방하는 데 특화된 엔드포인트 보안 솔루션이다.
> 2. **가치**: 단일 에이전트로 다중 보안 기능을 제공함으로써 관리 복잡도를 낮추고 시그니처·휴리스틱·샌드박스 분석을 통해 알려진 위협의 90% 이상을 사전에 차단한다.
> 3. **판단 포인트**: EPP는 예방(Prevention) 중심이라 EDR(Endpoint Detection and Response)이 제공하는 사후 탐지·포렌식 기능이 없어, 고도화 위협에 대응하려면 EDR과 반드시 병행해야 한다.

---

## Ⅰ. 개요 및 필요성

EPP(Endpoint Protection Platform)는 과거 독립적으로 운영되던 AV(Anti-Virus), HIPS(Host Intrusion Prevention System), 방화벽(Host-Based Firewall), 애플리케이션 화이트리스팅(Application Whitelisting), DLP(Data Loss Prevention)를 하나의 에이전트·관리 콘솔로 통합한 차세대 엔드포인트 보안 플랫폼이다. 가트너(Gartner)는 EPP를 "엔드포인트 위협을 예방하는 능력을 갖춘 통합 보안 소프트웨어 스위트"로 정의한다.

기업이 EPP를 도입하는 핵심 이유는 **운영 효율성**이다. 보안 제품이 분산되면 관리 오버헤드, 에이전트 충돌, 정책 불일치가 발생한다. EPP는 이를 단일 플랫폼으로 해결하며, 중앙 관리 콘솔에서 모든 엔드포인트의 보안 상태를 실시간으로 파악할 수 있다.

현대 EPP는 정적 시그니처(Static Signature) 외에도 ML(Machine Learning) 기반 분류기, 행위 분석(Behavioral Analysis), 클라우드 위협 인텔리전스(Cloud Threat Intelligence) 조회를 결합하여 제로데이(Zero-Day) 위협에도 어느 정도 대응한다. 그러나 본질적으로 **예방 중심**이기 때문에 침해 발생 이후의 조사·대응에는 EDR이 필요하다.

📢 **섹션 요약 비유**: EPP는 "올인원 보안 스위트케이스"다. 자물쇠·경보기·방범창을 따로 사는 대신 하나의 스마트 보안 세트로 한꺼번에 구비하는 것이다.

## Ⅱ. 아키텍처 및 핵심 원리

### EPP 핵심 구성 요소

| 구성 요소 | 약어 | 기능 |
|:---------|:-----|:-----|
| 안티바이러스 | AV | 시그니처 기반 악성코드 탐지·제거 |
| 호스트 침입 방지 | HIPS | 시스템 콜 수준 이상 행위 차단 |
| 호스트 방화벽 | HFW | 인바운드/아웃바운드 트래픽 필터링 |
| 애플리케이션 제어 | AC | 미승인 프로그램 실행 차단 |
| 데이터 유출 방지 | DLP | 민감 데이터 외부 전송 차단 |
| 샌드박스 분석 | Sandbox | 의심 파일 격리 실행·행위 검사 |

```
┌──────────────────────────────────────────────┐
│             EPP 플랫폼 구조                  │
│                                              │
│  ┌────────────────────────────────────────┐  │
│  │           단일 에이전트 (Agent)         │  │
│  │  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐  │  │
│  │  │ AV   │ │ HIPS │ │ HFW  │ │ DLP  │  │  │
│  │  └──────┘ └──────┘ └──────┘ └──────┘  │  │
│  └────────────────────────────────────────┘  │
│                    │                         │
│  ┌─────────────────▼──────────────────────┐  │
│  │       중앙 관리 콘솔 (Central Console)  │  │
│  │   정책배포 | 상태모니터링 | 리포트     │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

EPP의 탐지 파이프라인은 파일이 디스크에 기록되거나 프로세스가 실행될 때 순차적으로 작동한다. 먼저 로컬 시그니처 DB를 조회하고, 미지의 파일은 클라우드 인텔리전스에 해시값을 조회한 뒤, 이마저 불명확하면 ML 분류기를 적용한다. 최종적으로 의심 파일은 샌드박스에서 동적 실행되어 최종 판단을 받는다.

📢 **섹션 요약 비유**: EPP의 탐지 파이프라인은 "공항 보안 검색대"다. 여권(시그니처) 확인 → 블랙리스트(클라우드) 조회 → X레이(ML) 투과 → 별도 조사실(샌드박스) 순으로 심층 검사한다.

## Ⅲ. 비교 및 연결

### EPP vs EDR 비교

| 특성 | EPP | EDR |
|:----|:----|:----|
| 초점 | 예방(Prevention) | 탐지·대응(Detection & Response) |
| 탐지 대상 | 알려진 위협 중심 | 알려진·미지 위협 모두 |
| 데이터 수집 | 최소 이벤트 | 전체 텔레메트리(Telemetry) |
| 포렌식 기능 | 제한적 | 타임라인·프로세스 트리 |
| 자원 사용 | 보통 | 높음 |

EPP는 NGAV(Next-Generation Anti-Virus)와 혼용되기도 하는데, NGAV는 EPP의 AV 구성 요소를 ML·행위 분석으로 강화한 것을 의미한다. 현재 시장 트렌드는 EPP+EDR을 통합한 단일 플랫폼으로 수렴하고 있으며, CrowdStrike Falcon, Microsoft Defender, SentinelOne이 대표적이다.

📢 **섹션 요약 비유**: EPP는 "문을 잠그는 것"이고 EDR은 "침입 후 발자국을 추적하는 것"이다. 완전한 보안은 둘 다 필요하다.

## Ⅳ. 실무 적용 및 기술사 판단

**EPP 선택 기준:**
- **탐지율(Detection Rate)**: AV-TEST, AV-Comparatives 인증 성적 확인
- **성능 영향도**: 에이전트로 인한 시스템 지연(부팅 시간, 파일 I/O) 측정
- **클라우드 연동**: 실시간 위협 인텔리전스 피드(Feed) 연동 여부
- **정책 관리**: RBAC(Role-Based Access Control) 기반 관리 콘솔 지원

**운영 시 주의 사항:**
- 정의 파일(Definition File) 업데이트 주기를 자동화하고, 업데이트 실패 시 알림 설정
- 오탐(False Positive)으로 인한 업무 차단을 방지하기 위한 예외 처리 정책 문서화
- 법적 증거로 활용할 수 있도록 검역(Quarantine) 로그 보존

📢 **섹션 요약 비유**: EPP 운영은 "잘 훈련된 경비견 관리"다. 훈련(정책)을 잘 시키지 않으면 손님(정상 파일)도 무는 사고가 발생한다.

## Ⅴ. 기대효과 및 결론

EPP를 도입하면 알려진 악성코드의 90~95%를 사전에 차단할 수 있으며, 단일 에이전트로 다중 보안 기능을 통합 운영함으로써 보안 운영 비용을 40% 이상 절감할 수 있다. 또한 중앙 콘솔에서 모든 엔드포인트의 패치 상태·컴플라이언스를 실시간으로 파악하여 감사(Audit)에 즉시 대응할 수 있다.

그러나 EPP만으로는 파일리스 공격, 제로데이 익스플로잇, LotL(Living-off-the-Land) 기법에 대응하기 어렵다. 따라서 EPP는 반드시 EDR과 병행 운영해야 하며, 장기적으로는 XDR(Extended Detection and Response) 플랫폼으로 통합 진화하는 방향이 권고된다.

📢 **섹션 요약 비유**: EPP는 "독감 예방 주사"다. 맞으면 대부분의 독감을 예방하지만, 신종 바이러스에는 별도 대응이 필요하다.

### 👶 어린이를 위한 3줄 비유 설명
1. EPP는 집에 설치하는 종합 보안 세트야. 자물쇠, 경보기, CCTV가 하나의 세트로 포함되어 있어.
2. 나쁜 사람(악성코드)이 오면 자동으로 문을 잠그고 경보를 울려.
3. 하지만 처음 보는 새로운 나쁜 사람이 오면 놓칠 수 있어서, 형사(EDR)도 함께 필요해.
