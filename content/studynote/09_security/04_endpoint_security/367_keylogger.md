## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 키로거(Keylogger)는 사용자의 키보드 입력을 몰래 기록해 비밀번호·신용카드 번호·개인정보를 탈취하는 정보 수집형 스파이웨어다.
> 2. **가치**: 소프트웨어·하드웨어·커널 세 계층에 모두 구현 가능하며, 정교한 APT 공격의 초기 정보 수집 단계에서 자격증명(credential) 탈취에 핵심적으로 활용된다.
> 3. **판단 포인트**: 탐지는 행위 기반 EDR·커널 API 모니터링으로 가능하지만, 하드웨어 키로거는 소프트웨어 탐지가 불가능해 물리적 보안 점검이 필수다.

---

## Ⅰ. 개요 및 필요성

키로거(Keylogger)는 키보드 입력을 가로채 기록하는 모든 수단을 통칭한다. 원래는 부모 모니터링·기업 직원 감시·법집행 기관의 합법적 감청 도구로 설계됐지만, 오늘날에는 사이버 범죄자들이 크리덴셜 탈취(credential harvesting)에 가장 많이 사용하는 악성코드 중 하나다.

공격자 관점에서 키로거는 단순하지만 강력하다. 사용자가 아무리 강력한 암호를 설정해도, 그 암호를 입력하는 순간을 캡처하면 모든 암호화는 무력화된다. 피싱 이메일, 악성 URL, 드라이브-바이 다운로드(drive-by download) 등 다양한 경로로 배포된다.

정보통신기술사 시험에서는 "소프트웨어 vs 하드웨어 키로거의 탐지 방법 차이"와 "가상 키보드(Virtual Keyboard)의 한계"가 단골 문제다.

📢 **섹션 요약 비유**: 키로거는 당신이 편지를 쓰는 모습을 몰래 훔쳐보는 스파이—뭘 썼는지가 아니라 손가락이 어디서 움직였는지를 기록한다.

## Ⅱ. 아키텍처 및 핵심 원리

| 종류 | 동작 방식 | 탐지 가능 여부 |
|:---|:---|:---|
| API 후킹 키로거 | SetWindowsHookEx 등 Win32 API 후킹 | 가능 (행위 분석) |
| 커널 기반 키로거 | 키보드 드라이버 필터 삽입 | 어려움 |
| 폼 그래버(Form Grabber) | 브라우저 API 후킹으로 폼 데이터 탈취 | 가능 |
| 하이퍼바이저 키로거 | 가상화 계층에서 I/O 캡처 | 매우 어려움 |
| 하드웨어 키로거 | USB/PS2 중간에 물리 장치 삽입 | 소프트웨어 불가 |
| 어쿠스틱 키로거 | 타이핑 음향 분석 (공격 연구 단계) | 소프트웨어 불가 |

```text
┌────────────────────────────────────────────────┐
│               키로거 동작 흐름도               │
├────────────────────────────────────────────────┤
│  사용자 키보드 입력                            │
│       │                                        │
│       ▼                                        │
│  [하드웨어 레이어]  ◄── 하드웨어 키로거 위치   │
│       │                                        │
│       ▼                                        │
│  [키보드 드라이버] ◄── 커널 키로거 위치        │
│       │                                        │
│       ▼                                        │
│  [Win32 메시지 큐] ◄── API 후킹 키로거 위치   │
│       │                                        │
│       ▼                                        │
│  [애플리케이션 수신]                           │
│                                                │
│  [기록된 키 → 암호화 → C2 서버로 전송]         │
└────────────────────────────────────────────────┘
```

**주요 Windows API 남용**  
- `SetWindowsHookEx(WH_KEYBOARD_LL, ...)`: 저수준 키보드 후킹  
- `GetAsyncKeyState()`: 비동기 키 상태 폴링  
- `RegisterRawInputDevices()`: RAW 입력 직접 수신  

📢 **섹션 요약 비유**: 전화 교환기(API 레이어)에서 통화를 녹음하든, 전화선(드라이버)을 직접 탭하든, 결국 같은 정보를 탈취한다—계층만 다를 뿐이다.

## Ⅲ. 비교 및 연결

| 항목 | 소프트웨어 키로거 | 하드웨어 키로거 |
|:---|:---|:---|
| 설치 방법 | 악성코드 감염 | 물리적 접근 필요 |
| 탐지 방법 | EDR·AV 탐지 가능 | 소프트웨어 탐지 불가 |
| 원격 작동 | 가능 (C2 통신) | 물리 회수 필요 |
| 영속성 | OS 재설치로 제거 | 하드웨어 제거 필요 |
| 위협 주체 | 사이버 범죄자, APT | 내부자, 첩보 기관 |

**가상 키보드의 한계**: 화면 클릭 좌표를 캡처하는 스크린샷 키로거에는 가상 키보드도 취약하다.

📢 **섹션 요약 비유**: 소프트웨어 키로거는 내 집 안에 숨은 도둑, 하드웨어 키로거는 현관에 몰래 설치된 CCTV—찾는 방법이 완전히 다르다.

---

## Ⅳ. 실무 적용 및 기술사 판단

**탐지 및 예방 전략**  
1. EDR (Endpoint Detection & Response) 솔루션으로 비정상 API 후킹 탐지  
2. 커널 드라이버 서명 강제화(DSE: Driver Signature Enforcement) 활성화  
3. 중요 업무 PC에 USB 포트 잠금 및 물리적 포트 봉인  
4. MFA (Multi-Factor Authentication) 적용—비밀번호 탈취되어도 인증 보호  
5. 패스워드 관리자 자동완성으로 키 입력 최소화  
6. 안티바이러스 실시간 보호 및 행위 기반 탐지 엔진 활성화  

**사고 대응 시 확인 사항**  
- Windows 이벤트 로그에서 SetWindowsHookEx 호출 추적  
- 네트워크 트래픽에서 주기적 소량 데이터 전송 패턴 탐지  
- USB 연결 이력 및 알 수 없는 HID (Human Interface Device) 장치 점검  

📢 **섹션 요약 비유**: 자물쇠를 10개 달아도 열쇠 복사기(키로거)가 있으면 소용없다—MFA는 열쇠 하나로는 못 여는 2단계 잠금이다.

---

## Ⅴ. 기대효과 및 결론

키로거는 단순한 기법이지만 자격증명 탈취에 있어 여전히 가장 효과적인 공격 벡터 중 하나다. 소프트웨어 키로거는 EDR·MFA·최소 권한 정책으로 대응할 수 있지만, 하드웨어 키로거는 물리적 보안 통제와 정기적인 장비 점검이 반드시 병행되어야 한다. 결국 핵심은 키 입력 자체를 신뢰하지 않는 제로트러스트(Zero Trust) 접근—비밀번호만으로는 충분하지 않으며 다중 인증이 최후 방어선이다.

📢 **섹션 요약 비유**: 일기장을 잠가도 내 손이 쓰는 걸 옆에서 보고 있다면 의미가 없다—MFA는 일기를 봐도 이해 못하게 암호화하는 것이다.

### 👶 어린이를 위한 3줄 비유 설명
키로거는 당신이 키보드로 뭘 치는지 몰래 다 기록하는 나쁜 프로그램이에요.  
비밀번호를 칠 때도 다 기록되기 때문에 해커가 나중에 그 비밀번호로 로그인할 수 있어요.  
막으려면 비밀번호 하나만 말고, 휴대폰 인증도 같이 쓰는 이중 잠금이 필요해요.
