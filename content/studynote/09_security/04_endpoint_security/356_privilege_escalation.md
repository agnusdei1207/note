## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 권한 상승(LPE: Local Privilege Escalation)은 일반 사용자 권한을 가진 공격자가 시스템의 취약점을 이용해 관리자(root/SYSTEM) 권한을 획득하는 공격으로, 원격 코드 실행(RCE) 이후 공격 체인의 필수 후속 단계다.
> 2. **가치**: 내부 위협, 피싱으로 일반 계정 탈취 후 lateral movement·데이터 유출을 위해 반드시 거치는 단계이며, 공급망 공격과 APT의 핵심 과정이다.
> 3. **판단 포인트**: LPE 방어는 최소 권한 원칙(PoLP), 커널 패치 주기, SetUID 최소화, AppArmor/SELinux 강제 접근 제어로 공격 가능한 면적을 줄이는 것이 핵심이다.

---

## Ⅰ. 개요 및 필요성

권한 상승은 수직 권한 상승(Vertical PE: 일반→관리자)과 수평 권한 상승(Horizontal PE: 같은 권한 수준의 다른 사용자 계정 탈취)으로 분류된다. 보안에서 주로 다루는 것은 수직 권한 상승이며, Windows 환경에서는 NT AUTHORITY\SYSTEM, Linux에서는 root(uid=0)를 목표로 한다.

LPE 공격 경로는 다양하다: ① 커널 취약점(CVE-2021-4034 PwnKit, CVE-2022-0847 DirtyPipe), ② SetUID 바이너리 취약점(sudo, pkexec), ③ 약한 파일/레지스트리 권한(DLL hijacking, PATH hijacking), ④ 서비스 구성 오류(WritablePath, UnquotedPath), ⑤ 자격 증명 재사용(Credential Reuse).

📢 **섹션 요약 비유**: 권한 상승은 건물 로비 출입증(일반 사용자)을 갖고 들어온 뒤, 비상구·엘리베이터 취약점을 이용해 서버실 마스터 키(root)를 훔치는 것이다.

---

## Ⅱ. 아키텍처 및 핵심 원리

### LPE 공격 벡터 분류

| 카테고리 | 기법 | 예시 |
|:---|:---|:---|
| 커널 취약점 | UAF, BOF, 레이스 조건 | DirtyPipe, PwnKit |
| SetUID 프로그램 | sudo 취약점, SUID 바이너리 | CVE-2021-3156 (sudo BOF) |
| 서비스 설정 오류 | 쓰기 가능 서비스 경로 | Windows Unquoted Service Path |
| DLL/SO 하이재킹 | PATH/RPATH 우선순위 조작 | 환경변수 LD_PRELOAD |
| 자격 증명 | /etc/shadow, SAM 해시 탈취 | mimikatz, LinPEAS |

```
┌──────────────────────────────────────────────────────┐
│           권한 상승 공격 체인                         │
├──────────────────────────────────────────────────────┤
│  초기 침투 (RCE/피싱) → 일반 사용자 쉘              │
│      │                                              │
│      ├─ 커널 취약점 탐색 (uname -r)                 │
│      ├─ SUID 바이너리 열거 (find / -perm -4000)     │
│      ├─ 서비스 권한 오류 (accesschk, winpeas)       │
│      └─ 자격 증명 파일 탐색 (/etc/shadow, .bash_history) │
│                                                     │
│  성공 → root/SYSTEM 쉘 획득                         │
└──────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: LPE는 "건물 내부에서 약한 고리(취약 설정·취약 코드)를 찾아 관리자 사무실 열쇠를 복사하는 것"이다.

---

## Ⅲ. 비교 및 연결

| 권한 상승 유형 | 목표 | 주요 기법 |
|:---|:---|:---|
| 수직 상승(LPE) | 일반→root/SYSTEM | 커널 익스플로잇, sudo 취약점 |
| 수평 상승 | 사용자A→사용자B | 자격 증명 재사용, 세션 탈취 |
| 컨테이너 탈출 | 컨테이너→호스트 | runc 취약점, 잘못된 권한 |

📢 **섹션 요약 비유**: 수직 상승은 평직원이 CEO 방에 들어가는 것, 수평 상승은 같은 층 다른 직원 사무실에 몰래 들어가는 것이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

LPE 방어 실무 점검 항목: ① `find / -perm -4000 2>/dev/null`로 불필요한 SUID 제거, ② sudo 설정 최소화(`sudoers`), ③ 커널 보안 업데이트 주기 정의(CVSS 7+는 72시간 내 패치), ④ AppArmor/SELinux 프로파일로 특권 프로세스 행위 제한, ⑤ Auditd/EDR로 비정상 특권 취득 이벤트 탐지.

기술사 답안: LPE를 "내부 횡적 이동의 전제 조건"으로 정의하고, 방어를 예방(최소 권한), 탐지(EDR 행위 분석), 격리(컨테이너, MAC)로 3분류.

📢 **섹션 요약 비유**: 최소 권한 원칙은 각 방에 필요한 열쇠만 주는 것—마스터 키가 없으면 방 하나씩 돌파해도 전체를 장악할 수 없다.

---

## Ⅴ. 기대효과 및 결론

LPE 방어 강화는 침해 사고에서 "lateral movement → 전체 장악"으로 이어지는 공격 체인을 끊는 데 결정적이다. 공격자가 일반 계정을 탈취해도 LPE에 실패하면 피해 반경이 해당 계정의 권한으로 제한된다. 이는 Zero Trust "최소 권한"과 "명시적 검증" 원칙의 실천 효과이다.

📢 **섹션 요약 비유**: LPE 방어는 적이 1층(일반 계정)에 침입해도 엘리베이터(특권 승격 경로)를 모두 잠가두면 2층(관리자)에 올라오지 못하게 하는 것이다.

### 👶 어린이를 위한 3줄 비유 설명
- 권한 상승은 학교 학생이 선생님 방(관리자 권한) 열쇠를 훔치는 것과 같아.
- 학교 건물에 들어온(초기 침투) 다음, 약한 잠금장치를 찾아 선생님 열쇠를 복사해.
- 막으려면 필요한 방만 열 수 있는 열쇠를 주고(최소 권한), 불필요한 잠금장치를 없애야(SUID 최소화) 해!
