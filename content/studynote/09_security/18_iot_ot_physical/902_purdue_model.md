## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Purdue 모델은 산업 제어 시스템 (ICS) 아키텍처를 기능적 중요도에 따라 0단계부터 5단계까지 계층화한 전사적 참조 모델이다.
> 2. **가치**: 서로 다른 보안 요구사항을 가진 계층 간의 경계를 정의함으로써, 효율적인 망분리와 심층 방어 (Defense-in-Depth)를 가능하게 한다.
> 3. **판단 포인트**: 현대의 클라우드/IoT 환경에서는 계층 간 경계가 무너지고 있으나, 여전히 '보안 구역(Zone) 분리'의 가장 강력한 논리적 근거가 된다.

---

## Ⅰ. 개요 및 필요성

Purdue 모델 (The Purdue Reference Model)은 1990년대 퍼듀 대학교에서 개발되었으며, 이후 ISA-95 및 IEC 62443 표준의 근간이 되었다. 이 모델은 공장의 말단 장치부터 기업의 경영 시스템까지를 체계적으로 구분하여, "어떤 정보가 어디로 흘러야 하는지"를 정의한다.

보안 관점에서 Purdue 모델은 **망분리(Network Segmentation)**의 표준 설계도다. 계층 간의 직접 통신을 제한하고 DMZ를 통해 데이터를 교환하게 함으로써, IT 영역의 악성코드가 OT 제어 영역으로 침투하는 것을 기술적으로 방지한다.

- **📢 섹션 요약 비유**: Purdue 모델은 성벽(보안 계층)이 겹겹이 쌓인 '거대한 성곽 도시'와 같다. 왕(제어 서버)이 있는 깊숙한 곳까지 가려면 여러 개의 문과 검문소를 거쳐야 한다.

---

## Ⅱ. 아키텍처 및 계층별 역할

Purdue 모델은 크게 IT 영역, 산업용 DMZ, OT 영역으로 나뉜다.

```text
┌──────────────────────────────────────────────────────────────┐
│                  Purdue 모델 계층 구조 (PERA)               │
├──────────────────────────────────────────────────────────────┤
│ [Level 5]  Enterprise Network (인터넷, 메일, ERP)            │
│     │                                                        │
│ [Level 4]  Site Business Planning (사무소, 물류 관리)        │
├─────┼────────────────────────────────────────────────────────┤
│ [Level 3.5] Industrial DMZ (IDMZ - 보안 장벽)                │
├─────┼────────────────────────────────────────────────────────┤
│ [Level 3]  Site Operations (SCADA, HMI, 패치 서버)           │
│     │                                                        │
│ [Level 2]  Area Supervisory Control (HMI, 컨트롤러 서버)     │
│     │                                                        │
│ [Level 1]  Basic Control (PLC, RTU - 제어 로직 실행)         │
│     │                                                        │
│ [Level 0]  Process (센서, 액추에이터, 모터 - 물리 장치)      │
└──────────────────────────────────────────────────────────────┘
```

| 계층 | 명칭 | 주요 구성 요소 | 보안 수준 |
|:---|:---|:---|:---|
| **Level 4-5** | IT 영역 | ERP, 이메일, 인터넷 | 보안 위험 높음, 개방적 |
| **Level 3.5** | **IDMZ** | 프록시, 데이터 소구(SoH) | **접점 통제 핵심 구역** |
| **Level 3** | OT 관리 | SCADA, Historian, AD | 제어망의 관리 계층 |
| **Level 0-2** | OT 제어 | PLC, 센서, 로봇 팔 | **가장 높은 보안성 및 가용성** |

- **📢 섹션 요약 비유**: 0층(현장)에서 일하는 일꾼들이 5층(회장님실)에 직접 갈 수 없도록 층마다 보안 카드를 찍고 다니게 하는 구조다.

## Ⅲ. 비교 및 연결 (과거 vs 현대)

디지털 전환(DX)으로 인해 Purdue 모델의 '엄격한 계층 구조'에 변화가 생기고 있다.

| 항목 | 전통적 Purdue 모델 | 현대적 클라우드/IoT 환경 |
|:---|:---|:---|
| 데이터 흐름 | 하향식/상향식 순차적 흐름 | IoT 기기에서 클라우드로 직접 전송 (Bypass) |
| 네트워크 경계 | 물리적 망분리 및 방화벽 중심 | 소프트웨어 정의 경계 (SDP), Zero Trust |
| 보안 장치 | 계층 간 체크포인트 중심 | 앤드포인트 보안 및 아이덴티티 기반 |
| 연결성 | 폐쇄적, 제한적 연결 | 개방적, API 기반 실시간 연결 |

- **📢 섹션 요약 비유**: 예전엔 성문(방화벽)만 잘 지키면 됐는데, 요즘엔 하늘을 나는 드론(클라우드/IoT)이 성 안으로 바로 들어오는 격이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 Purdue 모델을 기반으로 설계하되, **Level 3.5 (IDMZ)**의 역할을 극대화해야 한다. IT와 OT 계층 사이에는 어떤 직접적인 통신도 허용해서는 안 되며, 반드시 IDMZ의 '데이터 미러링'이나 '점프 호스트'를 거치도록 강제해야 한다.

### 체크리스트
1. Level 4(IT)에서 Level 3(OT)로 직접 접속하는 경로가 완전히 차단되어 있는가?
2. IDMZ 내에 패치 관리 시스템(WSUS 등)과 안티바이러스 업데이트 서버를 별도로 두었는가?
3. 각 계층별로 독립적인 서브넷과 VLAN이 구성되어 있는가?
4. 하위 계층(Level 0-1)의 장비가 인터넷에 직접 노출되어 있지 않은가?

### 안티패턴
- 업무용 PC에서 제어용 PLC에 직접 Ping이 날아가는 평면적 네트워크 구조
- 인터넷 연결이 필요한 OT 장비를 Level 4 망에 직접 연결하는 행위

- **📢 섹션 요약 비유**: 5층(인터넷)에서 0층(기계)까지 한 번에 뚫린 엘리베이터를 만드는 것은 보안 사고로 가는 지름길이다.

---

## Ⅴ. 기대효과 및 결론

Purdue 모델의 적용은 사고 발생 시 피해 범위를 특정 계층으로 한정(Blast Radius 최소화)하는 강력한 억제력을 제공한다. 또한 복잡한 시스템의 통신 흐름을 단순화하여 관리 효율성을 높인다.

결론적으로 Purdue 모델은 '안전한 산업 아키텍처의 문법'이다. 비록 클라우드 도입으로 유연성이 강조되고 있지만, "중요 제어 영역은 보호되어야 한다"는 이 모델의 기본 정신은 변하지 않는 핵심 원칙이다.

- **📢 섹션 요약 비유**: Purdue 모델은 보안의 '족보'다. 이 족보를 무시하고 설계된 시스템은 결국 뿌리 없는 보안이 된다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| IDMZ (Industrial DMZ) | IT와 OT 영역 사이의 완충지대로, 보안 통제의 핵심 |
| ISA-95 | Purdue 모델을 기반으로 엔터프라이즈와 제어 시스템 간 통합을 정의한 표준 |
| 심층 방어 (Defense-in-Depth) | 여러 계층으로 보안을 쌓는 Purdue 모델의 핵심 철학 |
| 에어갭 (Air-gap) | 계층 간 연결을 완전히 끊는 극단적인 보안 형태 |

### 👶 어린이를 위한 3줄 비유 설명

1. 공장의 컴퓨터들을 중요도에 따라 0층부터 5층까지 나누어 놓은 지도예요.
2. 0층은 기계 손발이고, 5층은 사무실 컴퓨터라고 생각하면 돼요.
3. 5층에서 감기가 걸려도 0층 기계까지 옮지 않도록 층마다 튼튼한 문을 달아두는 것이 이 지도의 핵심이랍니다.
