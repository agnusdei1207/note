## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DCS (Distributed Control System)는 대규모 공정 내의 제어 요소를 여러 로컬 컨트롤러에 분산시켜 관리하는 시스템으로, 단일 장애점 (SPOF)을 제거한 구조다.
> 2. **가치**: 한 곳의 제어기가 고장 나도 전체 공정이 멈추지 않게 하여 발전소나 화학 공장처럼 연속성이 생명인 산업에서 고도의 신뢰성을 제공한다.
> 3. **판단 포인트**: 제어 명령이 여러 노드로 분산되어 흐르므로, 노드 간 통신 무결성과 각 컨트롤러의 리소스 보호가 보안의 핵심이다.

---

## Ⅰ. 개요 및 필요성

DCS (Distributed Control System)는 주로 정유, 화학, 발전소 등 '연속 공정 (Continuous Process)'이 이루어지는 시설에서 사용된다. 수천 개의 입출력(I/O) 포인트가 존재하는 복잡한 환경에서 중앙 집중형 제어는 과부하와 위험이 크기 때문에, 기능을 구역별로 나누어 분산 처리한다.

최근의 DCS는 IT 기술을 대거 도입하여 개방형 구조를 띠고 있으며, 이로 인해 제어 노드 간의 통신망이 공격 타깃이 될 확률이 높아졌다. 하나의 노드만 감염되어도 분산된 다른 노드들로 악성코드가 확산될 수 있어 통합적인 방어 체계가 필수적이다.

- **📢 섹션 요약 비유**: DCS는 대형 주방의 '분업 시스템'과 같다. 야채 담당, 고기 담당이 나누어져 있어 한 명이 다쳐도 주방 전체가 멈추지 않고 요리가 계속된다.

---

## Ⅱ. 아키텍처 및 핵심 원리

DCS는 상위 감시 계층과 하위 분산 제어 계층이 고속 네트워크로 연결된 구조다.

| 계층/구성 요소 | 역할 | 특징 |
|:---|:---|:---|
| **Operator Station** | 운영자가 공정 상태를 감시하고 조작 (HMI) | 상위 관리 계층 |
| **Control Station** | 실제 제어 로직을 실행하는 로컬 컨트롤러 | 하위 분산 제어 계층, 실시간성 중요 |
| **I/O Module** | 현장 센서/액추에이터와 직접 연결 | 물리적 신호의 입출력 담당 |
| **Control Network** | 컨트롤러와 서버 간의 고속 통신망 | 이중화(Redundancy) 필수, 고속 통신 |

```text
┌──────────────────────────────────────────────────────────────┐
│                    DCS의 분산 제어 구조                      │
├──────────────────────────────────────────────────────────────┤
│      [중앙 감시 서버 / HMI]  <── (고속 이중화 망) ──┐        │
│             │                                        │       │
│      ┌──────┴──────┬──────────────┬──────────────┐   │       │
│      ▼             ▼              ▼              ▼   │       │
│ [컨트롤러 A]   [컨트롤러 B]   [컨트롤러 C]   [컨트롤러 D]  │       │
│      │             │              │              │   │       │
│   (공정 1)       (공정 2)       (공정 3)       (공정 4)  │       │
└──────────────────────────────────────────────────────────────┘
```

DCS 보안의 핵심은 **이중화된 통신망의 보호**와 **세분화된 권한 관리**이다. 특정 컨트롤러가 장악되더라도 다른 구역의 제어 권한을 획득하지 못하도록 격리해야 한다.

- **📢 섹션 요약 비유**: 큰 배가 구획별로 나누어져 있어 한 곳에 구멍이 나도 배가 가라앉지 않는 '격벽 구조'와 같은 원리다.

## Ⅲ. 비교 및 연결 (DCS vs PLC)

DCS와 PLC는 모두 제어에 쓰이지만, 규모와 목적에서 차이가 있다.

| 항목 | DCS (분산 제어 시스템) | PLC (프로그래머블 로직 컨트롤러) |
|:---|:---|:---|
| 주요 대상 | 복잡한 연속 공정 (예: 화학) | 단순/반복 이산 공정 (예: 조립) |
| 시스템 성격 | 시스템 전체가 하나의 패키지 형태 | 개별 장치 위주의 범용 제어기 |
| 데이터 관리 | 데이터베이스가 중앙에 통합 관리됨 | 각 PLC가 독립적으로 데이터 처리 |
| 가용성 설계 | 시스템 차원의 이중화 기본 내장 | 필요에 따라 외부에서 이중화 구성 |

최근에는 'Hybrid DCS'나 고성능 PLC의 등장으로 두 영역의 기술적 경계가 점차 사라지고 있다.

- **📢 섹션 요약 비유**: DCS가 '거대한 통합 지휘 본부'라면, PLC는 '특수 임무를 수행하는 개별 요원'과 같다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 DCS 보안 설계 시 '네트워크 세분화 (Segmentation)'를 통해 컨트롤러 간의 불필요한 통신을 차단해야 한다. 또한 운영 스테이션(OS)에 대한 앤드포인트 보안과 USB 차단은 기본이며, 컨트롤러의 로직 업데이트 시 '디지털 서명' 검증 절차를 도입해야 한다.

### 체크리스트
1. 제어 네트워크가 이중화되어 있으며, 물리적으로 안전하게 보호되는가?
2. 각 분산 컨트롤러에 대한 접근 권한이 업무별로 최소화되어 설정되었는가?
3. 중앙 엔지니어링 스테이션(ES)에 대한 보안 관리가 철저한가? (로직 변경의 시작점)
4. 공정 데이터의 무결성을 확인하기 위한 별도의 모니터링 시스템이 있는가?

### 안티패턴
- 모든 컨트롤러를 하나의 평면적인 네트워크(Flat Network)에 연결하는 행위
- 제조사에서 제공하는 기본 관리자 패스워드를 변경하지 않고 운영하는 행위

- **📢 섹션 요약 비유**: 주방 구역마다 문을 달아두지 않으면, 한 곳에서 난 불이 주방 전체로 순식간에 번지게 된다.

---

## Ⅴ. 기대효과 및 결론

DCS 보안 강화는 대규모 플랜트의 '제로 다운타임'을 실현하고, 미세한 제어 오류로 인한 대형 폭발 사고 등을 예방한다. 또한 생산 효율의 안정성을 높여 비즈니스 수익성을 극대화한다.

결론적으로 DCS는 분산된 만큼 관리 포인트도 분산된다. 기술사는 분산된 개별 노드의 보안과 중앙의 통합 관제 사이의 균형을 맞추는 '전사적 ICS 보안 거버넌스'를 수립해야 한다.

- **📢 섹션 요약 비유**: DCS 보안은 '분업의 효율성'을 지키면서 '협업의 안전성'을 확보하는 정교한 기술이다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| 이중화 (Redundancy) | DCS의 높은 가용성을 보장하기 위한 하드웨어/네트워크 설계 |
| 엔지니어링 스테이션 (ES) | DCS의 제어 로직을 작성하고 배포하는 가장 중요한 보안 지점 |
| 연속 공정 (Continuous Process) | DCS가 주로 사용되는 산업의 특징 (중단이 매우 치명적임) |
| Fieldbus | 컨트롤러와 하부 센서/액추에이터를 연결하는 통신망 |

### 👶 어린이를 위한 3줄 비유 설명

1. 커다란 공장의 일을 여러 대의 작은 컴퓨터들이 나누어서 관리하는 시스템이에요.
2. 한 대가 고장 나도 다른 컴퓨터들이 일을 대신 해주기 때문에 공장이 멈추지 않아요.
3. 여러 컴퓨터가 서로 이야기를 주고받기 때문에, 중간에 나쁜 소문(잘못된 명령)이 퍼지지 않게 지켜야 해요.
