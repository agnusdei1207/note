## 핵심 인사이트 (3줄 요약)
> 1. **본질**: PROFINET은 표준 이더넷 기술을 기반으로 산업 현장의 장치들을 연결하는 고속·실시간 산업용 이더넷 (Industrial Ethernet) 프로토콜이다.
> 2. **가치**: 일반 IT의 이더넷 인프라를 그대로 쓰면서도, 수 밀리초(ms) 이하의 엄격한 실시간 제어 (IRT)를 가능하게 하여 스마트 팩토리의 중추 역할을 한다.
> 3. **판단 포인트**: 표준 이더넷 기반이므로 일반적인 IT 공격(Dos, ARP 스푸핑 등)에 노출되기 쉬워, 네트워크 분할과 컴포넌트 단위의 보안 설정이 필수적이다.

---

## Ⅰ. 개요 및 필요성

PROFINET은 지멘스(Siemens) 주도로 개발된 PROFIBUS의 후속 표준으로, 현재 전 세계에서 가장 많이 쓰이는 산업용 이더넷 중 하나다. 기존 필드버스(Fieldbus)의 제한된 대역폭과 연결성을 극복하기 위해 TCP/IP를 수용하면서도 산업용 특유의 실시간성을 확보했다.

하지만 이더넷의 개방성은 양날의 검이다. 사무용 네트워크와 연결이 쉬워진 만큼 IT 영역의 악성코드가 공정 라인으로 흘러들어올 위험이 크며, 이를 방지하기 위한 '심층 방어' 기반의 산업 보안 설계가 반드시 병행되어야 한다.

- **📢 섹션 요약 비유**: PROFINET은 공장 안에 깔린 '광속 고속도로'와 같다. 많은 차(데이터)가 빠르게 다닐 수 있지만, 입구 통제를 안 하면 도둑 차들도 쉽게 들어온다.

## Ⅱ. 아키텍처 및 실시간 등급

PROFINET은 통신의 시급성에 따라 세 가지 통신 채널을 제공한다.

| 등급 | 통신 방식 | 특징 | 주요 용도 |
|:---|:---|:---|:---|
| **NRT (Non-Real Time)** | 표준 TCP/IP, UDP/IP 사용 | 지연 가능성 있음, IT 통신과 동일 | 장비 설정, 진단 데이터 전송 |
| **RT (Real Time)** | IP 계층을 생략하고 이더넷 직접 통신 | 1~10ms 수준의 실시간성 확보 | 일반적인 PLC 제어, I/O 데이터 |
| **IRT (Isochronous RT)** | 하드웨어 기반의 예약된 타임슬롯 사용 | 1ms 이하의 정밀한 동기화 | 고속 모터 제어, 로봇 팔 움직임 |

```text
┌──────────────────────────────────────────────────────────────┐
│                  PROFINET 패킷 구조의 계층 분리             │
├──────────────────────────────────────────────────────────────┤
│ [일반 IT 패킷]      [PROFINET RT 패킷]     [PROFINET IRT 패킷] │
│      │                     │                     │           │
│  (TCP/UDP/IP)       (Ethernet / VLAN)     (Hardware Sync)    │
│      │                     │                     │           │
│  [표준 이더넷 하드웨어 환경 (IEEE 802.3)]                     │
└──────────────────────────────────────────────────────────────┘
```

PROFINET 보안의 핵심은 **네트워크 격리**이다. 실시간 제어 패킷(RT/IRT)이 흐르는 구간에 일반 IT 트래픽이 섞여 대역폭을 점유하면 공정 사고로 이어질 수 있다.

- **📢 섹션 요약 비유**: 고속도로에 전용 차로(IRT)를 만들어서 구급차(제어 신호)가 막힘없이 지나가게 하고, 일반 승용차(IT 데이터)는 옆 차로로 다니게 하는 것과 같다.

## Ⅲ. 비교 및 연결 (PROFINET vs PROFIBUS)

과거의 시리얼 방식인 PROFIBUS와 비교하면 유연성과 성능이 비약적으로 향상되었다.

| 항목 | PROFIBUS (Legacy) | PROFINET (Modern) |
|:---|:---|:---|
| 물리 계층 | RS-485 (구리선) | 표준 이더넷 (Cat5e, 광케이블) |
| 통신 속도 | 최대 12Mbps | 최소 100Mbps ~ Gbps |
| 무선 통신 | 지원 어려움 | Wi-Fi, 5G 등 무선 연결 가능 |
| IT 융합 | 게이트웨이 필수 | 직접 TCP/IP 통신 가능 |
| 보안 접근 | 물리적 차단 중심 | 방화벽, 인증, 암호화 등 다각도 접근 |

- **📢 섹션 요약 비유**: PROFIBUS가 '구식 전화선'이라면, PROFINET은 '최신형 기가 인터넷'이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 PROFINET 환경에서 'Security Classes' 개념을 적용해야 한다. 최근 발표된 PROFINET Security 가이드라인에 따라 장치 간 인증과 데이터 무결성을 보장하는 기능을 단계적으로 활성화해야 한다.

### 체크리스트
1. PROFINET 네트워크가 관리형 스위치(Managed Switch)를 통해 VLAN으로 분리되어 있는가?
2. 스위치의 미사용 포트가 차단(Port Security)되어 있는가?
3. 네트워크에 비인가 장비가 연결될 때 이를 탐지하고 차단하는 시스템(NAC)이 있는가?
4. 중요 장치들이 PROFINET Security Class 1 이상의 보안 기능을 지원하는가?

### 안티패턴
- 비용 절감을 위해 일반 가정용 공유기나 허브(Hub)를 산업 현장에 사용하는 행위 (실시간성 파괴)
- 공장 바닥에 노출된 이더넷 케이블을 아무나 노트북을 꽂아 쓸 수 있게 방치하는 행위

- **📢 섹션 요약 비유**: 아무리 빠른 고속도로라도 톨게이트(보안 장비)가 없으면 사고가 났을 때 범인을 잡을 수 없다.

## Ⅴ. 기대효과 및 결론

PROFINET은 산업 현장의 데이터 통합을 가속화하여 생산성을 높이고 유연한 공정 운영을 가능하게 한다. 강력한 보안이 뒷받침된 PROFINET 환경은 '안전한 스마트 제조'의 핵심 기반이 된다.

결론적으로 PROFINET은 IT의 편리함과 OT의 엄격함을 동시에 가진 기술이다. 기술사는 이더넷 기술에 대한 깊은 이해를 바탕으로, 표준 기술이 가지는 취약점을 산업용 보안 솔루션으로 보완하는 입체적 설계를 수행해야 한다.

- **📢 섹션 요약 비유**: PROFINET은 공장을 하나로 묶어주는 '강력한 접착제'이자 '빠른 메신저'이다. 이 메신저가 해킹당하지 않게 관리하는 것이 보안 전문가의 숙명이다.
