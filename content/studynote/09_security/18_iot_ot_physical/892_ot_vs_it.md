## 핵심 인사이트 (3줄 요약)
> 1. **본질**: OT와 IT는 각각 '물리적 프로세스'와 '디지털 데이터'라는 보호 대상을 가지며, 이로 인해 신뢰성, 가용성, 실시간성에 대한 요구 수준이 근본적으로 다르다.
> 2. **가치**: OT의 특성을 무시한 IT 방식의 보안 적용은 공정 마비라는 더 큰 사고를 초래할 수 있으므로, 두 영역의 차이점을 명확히 인지한 융합 보안이 필요하다.
> 3. **판단 포인트**: IT가 지연(Latency)을 허용하며 '정확성'을 챙긴다면, OT는 단 1초의 지연도 허용하지 않는 '결정론적(Deterministic) 통신'이 생명이다.

---

## Ⅰ. 개요 및 필요성

과거 OT 영역은 폐쇄성 덕분에 보안의 불모지였으나, IT 기술이 OT로 유입되면서 두 영역의 충돌이 빈번해지고 있다. IT 보안 전문가가 OT 현장에 가서 시스템을 스캔하다가 PLC가 멈추는 등의 사고가 대표적이다.

따라서 신뢰성 (Reliability), 가용성 (Availability), 실시간성 (Real-time)이라는 세 가지 축을 중심으로 IT와 OT의 기술적·운영적 차이를 분석하는 것은 융합 보안 설계의 선행 과제이다.

- **📢 섹션 요약 비유**: IT 보안은 '은행 전산망'을 지키는 꼼꼼한 회계사와 같고, OT 보안은 '원자력 발전소'를 지키는 엄격한 현장 감독관과 같다.

---

## Ⅱ. 핵심 차이 분석 (RAR 관점)

OT 환경의 핵심 요구사항인 RAR (Reliability, Availability, Real-time)을 중심으로 IT와 비교한다.

| 항목 | IT (Information Technology) | OT (Operational Technology) |
|:---|:---|:---|
| **가용성 (Availability)** | 업무 시간 중심, 계획된 다운타임 가능 | 24/365 중단 불가, 무정지 운영 필수 |
| **신뢰성 (Reliability)** | 데이터 오차 불허, 재전송 가능 | 물리적 동작의 안전성, 예측 가능한 결과 |
| **실시간성 (Real-time)** | 반응 속도 수 초 이내 (Best-effort) | 수 ms 이내의 엄격한 타이밍 (Deterministic) |
| **수명 주기** | 3~5년 (잦은 업그레이드) | 15~30년 (노후 장비 및 레거시 유지) |
| **보안 패치** | 정기적인 리부팅 및 패치 적용 | 가동 중 패치 거의 불가능, 가상 패치 의존 |

```text
┌──────────────────────────────────────────────────────────────┐
│                  보안 우선순위의 역전 (AIC vs CIA)          │
├──────────────────────────────────────────────────────────────┤
│      [IT 보안: CIA]             [OT 보안: AIC]              │
│  1. 기밀성 (C)                  1. 가용성 (A) / Safety       │
│  2. 무결성 (I)                  2. 무결성 (I)                │
│  3. 가용성 (A)                  3. 기밀성 (C)                │
└──────────────────────────────────────────────────────────────┘
```

OT에서는 데이터가 유출되는 것(기밀성 침해)보다, 밸브를 여는 명령이 전달되지 않거나 늦게 전달되는 것(가용성/실시간성 침해)이 인명 사고로 이어질 수 있어 훨씬 위험하다.

- **📢 섹션 요약 비유**: IT는 이메일이 1분 늦게 도착해도 괜찮지만, OT는 자동차 브레이크 신호가 0.1초만 늦어도 대참사가 일어나는 것과 같다.

## Ⅲ. 기술적 연결 및 전이

최근 IT 기술(이더넷, 클라우드, AI)이 OT로 전이되면서 발생하는 변화를 이해해야 한다.

| IT 기술의 OT 전이 | 기대 효과 | 보안 리스크 |
|:---|:---|:---|
| 산업용 이더넷 (Ethernet) | 장치 간 대용량 데이터 전송 | 기존 시리얼 통신보다 공격 경로 확대 |
| 클라우드/빅데이터 분석 | 공정 최적화, 예지 정비 | 외부망 연결에 따른 원격 공격 노출 |
| AI 기반 제어 | 정밀한 자동화 | AI 모델 변조를 통한 오동작 유도 |

- **📢 섹션 요약 비유**: 시골길(폐쇄망)에 고속도로(인터넷)가 뚫리면서 물류(데이터)는 빨라졌지만, 외부의 도둑(해커)들도 빠르게 들어올 수 있게 된 상황이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 OT 현장에 보안 솔루션을 도입할 때 반드시 '수동형(Passive)' 방식을 우선 고려해야 한다. 시스템에 부하를 주거나 패킷을 직접 주입하는 '능동형 스캔'은 OT 장비의 오동작을 유발할 수 있기 때문이다.

### 체크리스트
1. 보안 장비 도입 시 제어 네트워크의 실시간성에 미치는 영향(지연 시간)을 측정했는가?
2. 장애 발생 시 보안 장비가 'Fail-safe' 또는 'Fail-open' 중 어떤 방식으로 작동해야 하는가? (OT는 보통 안전을 위해 Fail-open 선호)
3. 노후화된 레거시 시스템의 경우, 직접 패치 대신 망 격리나 단독 방화벽으로 보호하고 있는가?
4. IT 부서와 OT 부서 간의 책임 소재와 비상시 협력 프로세스가 명확한가?

### 안티패턴
- IT용 통합 백신을 OT 제어 서버에 설치하여 CPU 점유율을 100%로 만드는 행위
- 암호화를 적용하느라 제어 패킷의 지연 시간을 늘려 로봇 팔의 타이밍을 어긋나게 하는 행위

- **📢 섹션 요약 비유**: 환자 수술 중에 보안 검사를 하겠다고 수술실 전원을 끄는 보안 팀이 되어서는 안 된다.

---

## Ⅴ. 기대효과 및 결론

IT와 OT의 차이를 명확히 이해하고 구축된 보안 체계는 산업 현장의 '안전'과 '비즈니스 이익'을 동시에 보장한다. 단순히 방어벽을 쌓는 것을 넘어, 공정의 특성을 반영한 지능형 탐지 체계를 갖추게 된다.

결론적으로 미래의 보안은 IT와 OT를 분리하는 것이 아니라, 두 영역의 가치를 모두 보호하는 '통합 리스크 관리'로 나아가야 한다. 기술사는 두 영역의 가교 역할을 수행하며 '중단 없는 보안'을 실현해야 한다.

- **📢 섹션 요약 비유**: IT와 OT는 서로 다른 언어를 쓰는 두 나라와 같다. 기술사는 두 언어를 모두 완벽히 이해하는 유능한 통역사가 되어야 한다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| AIC 모델 | OT 보안의 우선순위 (Availability, Integrity, Confidentiality) |
| Deterministic Communication | 정해진 시간 내에 반드시 데이터가 도착함을 보장하는 OT의 통신 특성 |
| 가상 패치 (Virtual Patching) | 가용성 때문에 패치를 못 하는 OT 장비를 외부에서 보호하는 기술 |
| Fail-safe | 고장 시 시스템을 안전한 상태(보통 중단)로 만드는 설계 철학 |

### 👶 어린이를 위한 3줄 비유 설명

1. IT는 사무실에서 쓰는 컴퓨터 공부고, OT는 공장에서 기계를 돌리는 공부예요.
2. IT는 비밀이 새나가지 않는 게 중요하고, OT는 기계가 멈추지 않는 게 제일 중요해요.
3. 기계가 0.1초라도 늦게 움직이면 위험하기 때문에, OT는 아주아주 빠른 반응 속도가 필요하답니다.
