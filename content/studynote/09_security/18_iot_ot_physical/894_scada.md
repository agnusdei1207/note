## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SCADA (Supervisory Control and Data Acquisition)는 광범위한 지역에 분산된 산업 장비들을 원격에서 감시하고 데이터를 수집하며 제어하는 시스템이다.
> 2. **가치**: 현장에 직접 가지 않고도 수백 km 떨어진 설비의 상태를 파악하고 제어할 수 있어 기반 시설 운영의 효율성을 극대화한다.
> 3. **판단 포인트**: 통신 매체의 다양성(전용선, 무선, 위성 등)에 따른 데이터 가로채기와 원격 명령 위변조가 최대 보안 리스크이므로 전 구간 암호화와 인증이 핵심이다.

---

## Ⅰ. 개요 및 필요성

SCADA 시스템은 주로 전력망, 송유관, 상하수도 시설 등 국토 전역에 퍼져 있는 기반 시설 관리에 사용된다. 현장의 RTU나 PLC로부터 데이터를 수집하여 중앙의 마스터 스테이션으로 보내면, 운영자가 이를 보고 판단하여 제어 명령을 하달하는 구조다.

통신 지연이 발생할 수 있는 광역 네트워크를 사용하기 때문에 실시간성보다는 '상태 감시'와 '이력 관리'에 더 중점을 둔다. 하지만 이 통신망이 공격자에게 노출될 경우, 국가 전체를 마비시키는 '사이버 테러'의 도구가 될 수 있어 강력한 보안이 요구된다.

- **📢 섹션 요약 비유**: SCADA는 전국 각지에 흩어진 관측소로부터 정보를 보고받고 명령을 내리는 '중앙 관제 센터'와 같다.

---

## Ⅱ. 아키텍처 및 핵심 구성 요소

SCADA는 하부의 현장 장치와 상부의 마스터 터미널로 구성된다.

| 구성 요소 | 역할 | 특징 |
|:---|:---|:---|
| **MTU (Master Terminal Unit)** | 중앙에서 데이터를 수집하고 명령 하달 | 서버급 하드웨어, HMI와 연동 |
| **RTU (Remote Terminal Unit)** | 현장 장치와 통신하여 데이터를 MTU로 전달 | 거친 환경에 강함, 무선 통신 지원 |
| **HMI (Human Machine Interface)** | 운영자에게 시각적인 공정 상태 제공 | 그래픽 기반 모니터링 화면 |
| **통신 인프라** | MTU와 RTU 사이의 데이터 통로 | 전용선, Radio, 위성, 이더넷 등 |

```text
┌──────────────────────────────────────────────────────────────┐
│                    SCADA 시스템의 논리적 구조                 │
├──────────────────────────────────────────────────────────────┤
│    [Control Center (MTU)]  ◀─── (광역 통신망) ───▶ [Remote Site] │
│             │                                        │       │
│      (명령/데이터 수집)                          (현장 제어)   │
│             │                                        │       │
│      [HMI / DB 서버]                          [RTU / PLC]    │
│                                                      │       │
│                                                [센서/액추에이터]│
└──────────────────────────────────────────────────────────────┘
```

SCADA 보안의 아킬레스건은 **프로토콜 취약성**이다. 과거의 SCADA 프로토콜(Modbus, DNP3 등)은 암호화나 인증 기능이 없어 공격자가 중간에서 명령을 조작하기 쉽다.

- **📢 섹션 요약 비유**: SCADA는 사령부(MTU)와 멀리 떨어진 전방 초소(RTU)가 무전기(통신망)로 대화하는 것과 같다. 무전이 도청당하면 작전이 탄로 난다.

## Ⅲ. 비교 및 연결 (SCADA vs IoT)

최근 SCADA와 산업용 IoT (IIoT)가 융합되면서 그 경계가 모호해지고 있다.

| 항목 | SCADA | IIoT (산업용 IoT) |
|:---|:---|:---|
| 주된 목적 | 원격 감시 및 제어 | 데이터 분석 및 예측 |
| 데이터 흐름 | 양방향 (수집 + 제어) | 주로 단방향 (데이터 수집 중심) |
| 연결성 | 전용망 중심 (Closed) | 인터넷/클라우드 중심 (Open) |
| 보안 접근 | 계층적 방어 (Purdue) | 종단간 보안 (End-to-End) |

- **📢 섹션 요약 비유**: SCADA가 '숙련된 관리자가 망원경으로 멀리 보며 조절하는 것'이라면, IIoT는 '수많은 센서가 데이터를 모아 인공지능이 분석하는 것'이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 SCADA 보안을 위해 '통신 구간 암호화'와 '접근 제어 강화'를 최우선으로 해야 한다. 특히 원격 유지보수를 위한 통로가 백도어로 활용되지 않도록 일회용 비밀번호 (OTP)나 인증서 기반의 강력한 인증을 적용해야 한다.

### 체크리스트
1. MTU와 RTU 간의 통신이 암호화되어 있는가? (예: DNP3 Secure Authentication)
2. 외부 인터넷과 SCADA 망 사이에 강력한 에어갭(Air-gap) 또는 단방향 전송 장치가 있는가?
3. RTU 장치에 대한 물리적인 접근 통제(박스 잠금, 센서 등)가 이루어지는가?
4. 비정상적인 데이터 수집 패턴(예: 갑작스러운 밸브 조작)을 탐지하는 SIEM이 연동되어 있는가?

### 안티패턴
- 비용 절감을 위해 보안 기능이 전혀 없는 구형 무선 모뎀을 사용하는 행위
- HMI 화면을 인터넷에 연결된 일반 PC에서 원격으로 보게 만드는 행위

- **📢 섹션 요약 비유**: 무전기로 기밀을 말할 때 암구호(인증)와 암호문(암호화)을 쓰지 않는 것은 적에게 작전 지시서를 주는 것과 같다.

## Ⅴ. 기대효과 및 결론

안전한 SCADA 운영은 국가 에너지·환경 안전의 핵심이다. 스마트 그리드와 같은 차세대 기반 시설로 발전하기 위해서는 SCADA의 고도화된 보안이 전제되어야 한다.

결론적으로 SCADA 보안은 '연결의 가치'를 지키는 일이다. 기술사는 통신 기술의 발전에 발맞추어, 전용망뿐만 아니라 공용망을 사용하더라도 안전하게 데이터를 주고받을 수 있는 '신뢰할 수 있는 원격 제어 체계'를 구축해야 한다.

- **📢 섹션 요약 비유**: SCADA 보안은 도시의 혈관을 지키는 일이다. 혈관이 막히거나 독극물이 들어오지 않게 끊임없이 살피고 관리해야 한다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| RTU (Remote Terminal Unit) | SCADA의 손발 역할을 하는 원격지 핵심 장치 |
| DNP3 (Distributed Network Protocol) | SCADA 시스템에서 가장 널리 쓰이는 표준 프로토콜 |
| 에어갭 (Air-gap) | 보안을 위해 네트워크를 물리적으로 분리하는 방식 |
| 단방향 전송 (Data Diode) | 데이터가 한쪽 방향으로만 흐르게 하여 외부 침입을 원천 차단하는 기술 |

### 👶 어린이를 위한 3줄 비유 설명

1. 아주 먼 곳에 있는 전기 스위치나 수도꼭지를 컴퓨터로 보고 조종하는 시스템이에요.
2. 직접 가지 않아도 되니까 편리하지만, 중간에 나쁜 사람이 명령을 가로챌 수 있어요.
3. 그래서 명령을 보낼 때 암호를 걸어서 우리 편만 알아볼 수 있게 만드는 게 중요해요.
