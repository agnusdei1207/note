## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DNP3는 주로 전력 및 수처리 산업의 SCADA 시스템에서 마스터 스테이션과 원격 장치(RTU) 간의 효율적이고 신뢰성 있는 데이터 교환을 위해 설계된 프로토콜이다.
> 2. **보안 문제**: **레거시 DNP3는 암호화와 강력한 인증이 없어**, 통신 구간에서 데이터 탈취나 가짜 상태 보고, 비인가 제어 명령 주입에 무방비하다.
> 3. **판단 포인트**: 중요 국가 기반 시설에 쓰이는 만큼, TLS 기반의 'DNP3-SAv5 (Secure Authentication)' 적용과 VPN을 통한 전용 통신로 확보가 필수적이다.

---

## Ⅰ. 개요 및 필요성

DNP3 (Distributed Network Protocol)는 북미 전력 산업을 중심으로 표준화되어 전 세계 SCADA 시스템에서 널리 쓰인다. 데이터를 조각내어 효율적으로 전송하고, 통신 장애 시 데이터를 복구하는 능력이 탁월하여 신뢰성이 매우 높다.

그러나 신뢰성(Reliability)에만 집중한 나머지 보안성(Security)은 고려되지 않았다. 전력망 제어 명령이 평문으로 전송되므로, 공격자가 통신망에 침입하여 차단기를 내리거나 전압을 조작하는 등의 공격을 수행할 경우 광범위한 정전(Blackout) 사태를 유발할 수 있다.

- **📢 섹션 요약 비유**: DNP3는 '산속 깊은 곳까지 정확하게 소식을 전달하는 우편 시스템'과 같다. 하지만 편지봉투에 봉인이 없어 누구나 내용을 고칠 수 있는 위험한 상태다.

---

## Ⅱ. 주요 기능 및 보안 취약점

DNP3는 데이터 타입별로 오브젝트를 정의하며, 이벤트 기반 보고 기능을 가진다.

| 주요 특징 | 상세 내용 | **보안 위협 (Critical)** |
|:---|:---|:---|
| **데이터 분할** | 큰 데이터를 작은 조각으로 나누어 전송 | 패킷 재조합 과정에서의 취약점 공격 가능 |
| **비암호화 (Plaintext)** | 모든 관제 데이터가 투명하게 보임 | 전력 사용량, 설비 상태 등 정보 유출 |
| **인증 부재** | 보낸 이가 진짜 마스터인지 확인 안 함 | **비인가 제어 명령(Control Relay Output) 주입** |
| **타임스탬프** | 이벤트 발생 시각을 기록 | 시간 정보를 조작하여 시스템 혼란 유발 |

```text
┌──────────────────────────────────────────────────────────────┐
│                    DNP3 보안 공격 시나리오                   │
├──────────────────────────────────────────────────────────────┤
│ [공격자] ──▶ (가짜 명령 주입) ──▶ [현장 RTU] ──▶ (차단기 개방) │
│     │                                                │       │
│     └─────── (가짜 데이터 보고) ◀── [마스터 서버] ──┘       │
│               "모두 정상입니다" (실제로는 정전 중)           │
└──────────────────────────────────────────────────────────────┘
```

특히 DNP3 패킷을 캡처한 후 나중에 다시 보내는 **리플레이 공격 (Replay Attack)**은 별도의 지식 없이도 시스템을 마비시킬 수 있는 치명적인 위협이다.

- **📢 섹션 요약 비유**: 지휘관의 목소리를 녹음했다가 나중에 다시 틀어서 부대에 잘못된 명령을 내리는 것과 같다.

## Ⅲ. 비교 및 연결 (DNP3 vs Modbus)

둘 다 산업용 프로토콜이지만 사용 환경과 기능에서 차이가 있다.

| 항목 | DNP3 | Modbus |
|:---|:---|:---|
| 주요 산업 | 전력, 수도 등 국가 기반 시설 | 공장 자동화, 빌딩 관리 |
| 효율성 | 이벤트 기반 보고 (변경 시에만 전송) | 폴링 방식 (주기적으로 전체 전송) |
| 신뢰성 | 패킷 조각화 및 복구 기능 강력 | 단순 에러 체크만 수행 |
| 보안 진화 | **DNP3 Secure Authentication (SA)** 표준 존재 | 최근 Modbus TCP Security 등장 |

- **📢 섹션 요약 비유**: Modbus가 '단순한 옆집 대화'라면, DNP3는 '격식을 갖춘 장거리 공문서 전달'과 같다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 DNP3를 사용하는 기반 시설 보안을 위해 반드시 **DNP3 SA (Secure Authentication)** 버전 적용을 검토해야 한다. 기존 인프라 교체가 어렵다면, VPN 장비를 RTU 앞단에 설치하여 통신 구간 전체를 암호화하는 'Secure Tunneling' 전략을 취해야 한다.

### 체크리스트
1. 마스터와 RTU 간의 통신이 전용 회선인가, 아니면 보안 조치가 된 공용망인가?
2. DNP3 SA를 적용하여 메시지 인증 코드(MAC)를 확인하고 있는가?
3. 중요 제어 명령(Select/Operate) 시 챌린지-응답(Challenge-Response) 인증이 이루어지는가?
4. 무선 구간 통신 시 위성이나 LTE 전용 VPN을 거치는가?

### 안티패턴
- 암호화되지 않은 라디오(Radio) 통신망을 통해 DNP3 제어 패킷을 무방비로 날리는 행위
- RTU의 기본 주소(Unit ID)를 변경하지 않고 누구나 추측 가능하게 두는 행위

- **📢 섹션 요약 비유**: 편지를 보낼 때 그냥 보내지 말고, 편지 끝에 서로만 아는 비밀 인장(인증)을 찍고 금고(VPN)에 넣어서 배달해야 한다.

---

## Ⅴ. 기대효과 및 결론

DNP3 보안 강화는 국가 핵심 기반 시설의 가용성을 지키는 보루이다. 전력망과 같은 기간망이 사이버 공격으로부터 안전해질 때 비로소 진정한 의미의 '스마트 시티'가 구현될 수 있다.

결론적으로 DNP3 보안은 '인증된 명령'만이 실행되게 보장하는 것이 핵심이다. 기술사는 레거시 프로토콜의 한계를 보완하기 위해 네트워크 보안 장비와 인증 프로토콜을 계층적으로 배치하는 설계를 완수해야 한다.

- **📢 섹션 요약 비유**: 아무리 소식을 잘 전달해도 가짜 소식이 섞이면 재앙이 된다. DNP3 보안은 '진짜 뉴스'만 골라내는 필터와 같다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| DNP3 SA (Secure Authentication) | 인증 기능을 추가한 DNP3의 보안 확장판 (IEC 62351 기반) |
| IEC 62351 | 전력 시스템 제어 및 통신 보안을 정의하는 상위 표준 |
| 리플레이 공격 (Replay Attack) | DNP3 보안에서 가장 경계해야 할 전형적인 공격 기법 |
| 폴링 (Polling) vs 이벤트 보고 | DNP3의 효율성을 결정짓는 핵심 동작 방식 |
