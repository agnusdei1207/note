## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Zone 맵핑은 식별된 산업 자산들을 기능, 중요도, 보안 요구사항에 따라 적절한 보안 구역(Zone)에 배치하고 논리적 경계를 설정하는 과정이다.
> 2. **가치**: 자산의 성격에 맞는 '맞춤형 보안 정책'을 적용할 수 있게 하여, 불필요한 보안 비용을 줄이고 핵심 자산에 대한 보호력을 집중시킨다.
> 3. **판단 포인트**: 자산의 물리적 위치보다는 **'제어 흐름의 연관성'**과 **'사고 시 피해 규모'**를 우선적인 맵핑 기준으로 삼아야 한다.

---

## Ⅰ. 개요 및 필요성

네트워크를 구역으로 나누기 위해서는 먼저 "무엇을 어디에 넣을 것인가"를 결정해야 한다. 이것이 바로 Zone 맵핑 (Asset Classification & Allocation)이다. 잘못된 맵핑은 보안 구멍을 만들거나, 반대로 업무 효율을 극도로 떨어뜨리는 원인이 된다.

특히 OT 환경에서는 수많은 레거시 장비와 최신 스마트 장비가 혼재되어 있으므로, 이들을 적절히 그룹화하여 보안 수준(SL)을 부여하는 과정이 아키텍처 설계의 성패를 좌우한다.

- **📢 섹션 요약 비유**: Zone 맵핑은 도서관의 책들을 주제별로 분류하여 서가에 꽂는 것과 같다. 귀중한 고서(핵심 제어기)는 별도의 제한 구역에, 잡지(일반 단말)는 열람실에 두는 식이다.

---

## Ⅱ. Zone 맵핑의 기준 및 절차

IEC 62443에 따른 Zone 맵핑은 다음과 같은 기준을 활용한다.

| 매핑 기준 | 설명 | 사례 |
|:---|:---|:---|
| **기능적 유사성** | 수행하는 공정이나 역할이 비슷한 자산들 | 모든 용수 처리 PLC들을 하나의 Zone으로 묶음 |
| **보안 요구사항** | 목표 보안 수준(SL-T)이 동일한 자산들 | SL-T 3가 필요한 핵심 서버들만 별도 Zone 구성 |
| **논리적 연결성** | 서로 빈번하게 통신해야 하는 자산들 | 특정 라인의 HMI와 PLC를 같은 Zone에 배치 |
| **물리적 위치** | 같은 방이나 같은 제어반 내에 있는 자산들 | (차선책) 지리적으로 고립된 현장 RTU들 |

```text
┌──────────────────────────────────────────────────────────────┐
│                    Zone 맵핑 의사결정 트리                   │
├──────────────────────────────────────────────────────────────┤
│ [질문 1] 이 자산이 멈추면 공장 전체가 멈추는가?              │
│      └─▶ Yes: [Critical Zone] 할당                          │
│                                                              │
│ [질문 2] 이 자산은 외부망(인터넷)과 대화해야 하는가?         │
│      └─▶ Yes: [DMZ / Boundary Zone] 할당                    │
│                                                              │
│ [질문 3] 이 자산은 보안 기능이 없는 노후 장비인가?           │
│      └─▶ Yes: [Isolated Legacy Zone] 할당                   │
└──────────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: 학교에서 반을 나눌 때 성적, 성격, 전공 등을 고려하여 가장 공부하기 좋은 조합을 만드는 것과 같다.

## Ⅲ. 비교 및 연결 (물리적 구역 vs 논리적 구역)

Zone 맵핑은 물리적인 위치에 얽매이지 않는 '논리적 설계'가 우선이다.

| 항목 | 물리적 구역 (Physical Zone) | 논리적 구역 (Logical Zone) |
|:---|:---|:---|
| 정의 | 같은 방, 같은 층 등 공간 기반 | 같은 기능, 같은 보안 수준 기반 |
| 장점 | 관리가 직관적이고 시각적임 | 보안 정책 적용이 정교하고 효율적임 |
| 단점 | 멀리 떨어진 장치 간 보안 통일 어려움 | 네트워크 설정(VLAN 등)이 복잡해질 수 있음 |
| 트렌드 | 과거엔 물리적 분리에 의존했으나, 현재는 논리적 맵핑 후 SDN 등으로 구현 |

- **📢 섹션 요약 비유**: 같은 아파트(물리적 구역)에 살아도, 직업이나 관심사(논리적 구역)에 따라 동호회를 만들어 활동하는 것과 같다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 Zone 맵핑 후 반드시 '자산 목록(Asset Inventory)'을 최신으로 유지해야 한다. 새로 도입된 장비가 맵핑되지 않은 채 네트워크에 방치되면, 해당 지점이 전체 구역의 보안 수준을 떨어뜨리는 취약점이 된다.

### 체크리스트
1. 모든 자산이 위험 평가 결과에 따른 적절한 보안 수준(SL)의 Zone에 배정되었는가?
2. 보안성이 없는 레거시 장비들이 보안성이 높은 최신 장비와 같은 Zone에 섞여 있지 않은가?
3. Zone 간의 통신 빈도를 분석하여, 불필요하게 쪼개진 Zone은 없는가? (운영 효율성 고려)
4. 원격 정비용 노트북처럼 이동이 잦은 자산에 대한 동적 Zone 할당 방안이 있는가?

### 안티패턴
- 장비의 중요도를 무시하고 단순히 IP 대역별로 Zone을 나누는 행위
- 한 번 맵핑한 후, 공정 라인이 변경되었음에도 Zone 구성을 그대로 유지하는 행위

- **📢 섹션 요약 비유**: 우등생 반에 사고뭉치를 섞어두거나, 반대로 반을 너무 세세하게 나눠서 조별 과제조차 못 하게 만드는 상황을 경계해야 한다.

---

## Ⅴ. 기대효과 및 결론

체계적인 Zone 맵핑은 보안 관리의 '가시성'을 제공한다. 관리자는 어떤 구역이 위험한지, 어떤 구역이 안전한지 한눈에 파악할 수 있으며, 사고 시 신속하게 해당 구역만 격리하여 대응할 수 있다.

결론적으로 Zone 맵핑은 산업 보안의 '조직도'를 그리는 일이다. 기술사는 공정의 흐름을 꿰뚫어 보는 안목으로, 보안과 생산성이 최적의 조화를 이루는 '황금비율의 맵핑'을 완성해야 한다.

- **📢 섹션 요약 비유**: Zone 맵핑은 복잡한 공장 안에 '질서 있는 칸막이'를 세우는 지적인 작업이다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| 자산 식별 (Asset Discovery) | Zone 맵핑을 하기 위해 선행되어야 하는 자산 찾기 과정 |
| 데이터 흐름 분석 (Data Flow) | Zone 간의 관계(Conduit)를 설정하기 위한 기초 자료 |
| Role-Based Zone | 사용자의 역할(관리자, 운영자 등)에 따라 접근 가능한 Zone을 나누는 방식 |
| 가상랜 (VLAN) | 논리적인 Zone 맵핑을 기술적으로 구현하는 가장 흔한 방법 |

### 👶 어린이를 위한 3줄 비유 설명

1. 공장의 수많은 로봇과 컴퓨터들을 끼리끼리 모아서 같은 방에 넣어주는 거예요.
2. 힘센 로봇들은 튼튼한 방에, 조심해야 할 예민한 컴퓨터들은 조용한 방에 모아요.
3. 이렇게 방을 잘 나눠주면, 한쪽 방에서 싸움(해킹)이 나도 다른 방 친구들은 안전하게 일할 수 있어요.
