## 핵심 인사이트 (3줄 요약)
> 1. **본질**: RTU (Remote Terminal Unit)는 멀리 떨어진 야외 현장에서 센서 데이터를 수집하여 중앙 마스터 스테이션(SCADA)으로 전송하고 제어 명령을 실행하는 장치이다.
> 2. **가치**: 극한의 온도, 습도, 전력 부족 환경에서도 동작하도록 설계되어 송유관, 전신주, 댐 등 광범위한 지역의 기반 시설 관리를 가능하게 한다.
> 3. **판단 포인트**: 야외 노출로 인한 '물리적 침입'과 '무선 통신 가로채기'가 주요 위협이므로, 하드웨어 보안(Tamper-proof)과 통신 구간 보안이 핵심이다.

---

## Ⅰ. 개요 및 필요성

RTU는 SCADA 시스템의 '말단 신경'이다. 공장 내부의 깨끗한 환경에서 동작하는 PLC와 달리, RTU는 산속의 댐, 사막의 송유관, 도심의 변전소 등 가혹한 환경에 배치된다. 따라서 높은 내구성과 저전력 설계가 필수적이다.

물리적으로 관리자의 눈이 닿지 않는 곳에 설치되는 경우가 많아, 공격자가 직접 장비를 뜯어 정보를 추출하거나 통신선을 가로채는 공격에 취약하다. RTU가 장악되면 잘못된 센서 데이터(예: 댐 수위 정상)를 보내 상위 시스템을 속일 수 있어 매우 위험하다.

- **📢 섹션 요약 비유**: RTU는 오지에 파견된 '고독한 관측병'과 같다. 무전기로 본부에 상황을 보고하고 명령을 수행하지만, 적에게 잡히거나 무전기를 뺏길 위험이 늘 있다.

---

## Ⅱ. 아키텍처 및 핵심 기능

RTU는 거친 환경 대응을 위한 특수 설계와 다양한 통신 인터페이스를 가진다.

| 구성 요소 | 역할 | 보안 요구사항 |
|:---|:---|:---|
| **프로세서 & 메모리** | 데이터 로깅 및 통신 프로토콜 처리 | 암호화 연산을 위한 최소 성능 필요 |
| **통신 인터페이스** | 무선(LTE, Sat), 전용선, 시리얼 연결 | 전 구간 TLS 또는 VPN 적용 |
| **입출력 포트 (I/O)** | 현장 계측기 연결 (4-20mA, Modbus 등) | 물리적 단자 보안 및 신호 무결성 |
| **백업 배터리** | 정전 시에도 일정 기간 동작 유지 | 전원 차단 감지 및 알람 전송 |

```text
┌──────────────────────────────────────────────────────────────┐
│                  RTU의 물리적/논리적 보안 경계               │
├──────────────────────────────────────────────────────────────┤
│    [물리적 외함 (Enclosure)]  ──▶ [침입 감지 센서]           │
│             │                                                │
│      ┌──────┴──────┐          ┌──────────────────────┐       │
│      │  [RTU 보드]  │ ◀──────▶ │  [보안 모뎀 / VPN]   │       │
│      └──────┬──────┘          └──────────┬───────────┘       │
│             │                            │                   │
│      [현장 센서 데이터]            [SCADA 센터 통신]         │
└──────────────────────────────────────────────────────────────┘
```

최신 RTU는 단순 전송을 넘어 현장에서 직접 판단을 내리는 로컬 제어 기능도 포함하고 있어, PLC와의 경계가 점점 허물어지고 있다.

- **📢 섹션 요약 비유**: 관측병(RTU)이 무전기뿐만 아니라 스스로를 지킬 수 있는 방패(물리 보안)와 암호 책(VPN)을 가지고 있는 형태다.

## Ⅲ. 비교 및 연결 (RTU vs PLC)

현장 제어 장치라는 점은 같으나 '어디에서 일하느냐'가 가장 큰 차이다.

| 항목 | RTU (원격 터미널 장치) | PLC (프로그래머블 로직 컨트롤러) |
|:---|:---|:---|
| 설치 환경 | 야외, 험지, 분산된 위치 | 실내, 공장, 집중된 위치 |
| 통신 거리 | 수십~수백 km (광역망) | 수십~수백 m (구내망) |
| 환경 내성 | 매우 높음 (영하~고온, 충격) | 보통 (제어반 내 설치) |
| 통신 매체 | Radio, 위성, 무선망 중심 | 고속 이더넷, 광케이블 중심 |

- **📢 섹션 요약 비유**: RTU는 '야전 전투병'이고, PLC는 '본부 내 행정병'과 같은 환경 차이가 있다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 RTU 도입 시 '물리적 위변조 방지 (Tamper-evident)' 기능을 우선 확인해야 한다. 장비 케이스가 열리면 즉시 SCADA 센터로 경보를 보내고 중요 데이터를 삭제하는 기능을 통해 장비 탈취 리스크를 대응해야 한다.

### 체크리스트
1. RTU 장비함이 견고하게 잠겨 있으며, 열림 감지 센서가 연동되어 있는가?
2. 통신망으로 공용망(LTE 등)을 사용할 경우, 전용 APN이나 VPN을 구축했는가?
3. RTU의 펌웨어 업데이트 시 원격에서 무결성 검증을 수행하는가?
4. 미사용 물리 포트(콘솔 포트 등)가 비활성화되거나 물리적으로 차단되었는가?

### 안티패턴
- 장비 유지보수를 위해 외함 열쇠를 장비 주변에 숨겨두는 행위
- 무선 통신 시 암호화 없이 평문으로 데이터를 전송하는 행위 (도청 가능)

- **📢 섹션 요약 비유**: 무전기(RTU)를 산속에 그냥 두고 오면서 비밀번호도 안 걸어놓는 것은 적에게 정보를 가져가라고 하는 것과 같다.

---

## Ⅴ. 기대효과 및 결론

신뢰성 있는 RTU 운영은 국가 에너지·수자원 관리의 효율성을 높이고 사고를 예방한다. 또한 사람이 가기 힘든 곳의 데이터를 실시간으로 수집하여 빅데이터 기반의 인프라 최적화를 가능하게 한다.

결론적으로 RTU는 '물리적 보안'과 '사이버 보안'이 만나는 최전선이다. 기술사는 사이버 공격뿐만 아니라 물리적 파괴나 도난 시나리오까지 고려한 입체적인 방어 전략을 수립해야 한다.

- **📢 섹션 요약 비유**: RTU 보안은 우리 사회의 손발이 되어주는 기특한 로봇들이 납치당하지 않게 잘 지켜주는 것과 같다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| SCADA (Master Station) | RTU가 보고하고 명령을 받는 상위 시스템 |
| 원격 모니터링 (Telemetry) | 멀리 떨어진 장비의 상태를 읽어오는 기술 |
| Tamper-proof | 장비를 강제로 열거나 수정하려는 시도를 방지/감지하는 기술 |
| 저전력 광대역 (LPWAN) | 최신 RTU에서 배터리 수명을 늘리기 위해 사용하는 통신 기술 |

### 👶 어린이를 위한 3줄 비유 설명

1. 아주 먼 산속이나 바다 한가운데서 기계들이 잘 있는지 살펴보고 보고해주는 장치예요.
2. 덥거나 추워도 고장 나지 않게 아주 튼튼하게 만들어졌어요.
3. 아무도 없는 곳에 혼자 있기 때문에, 나쁜 사람이 장치를 가져가거나 고장 내지 못하게 감시카메라처럼 지켜야 해요.
