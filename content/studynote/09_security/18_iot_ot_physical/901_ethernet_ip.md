## 핵심 인사이트 (3줄 요약)
> 1. **본질**: EtherNet/IP는 표준 이더넷(IEEE 802.3)과 TCP/IP 위에서 CIP (Common Industrial Protocol)를 구현한 산업용 네트워크 프로토콜이다.
> 2. **가치**: 일반 상용 IT 인프라와 높은 호환성을 유지하면서도, 산업용 기기 간의 실시간 제어와 정보 교환을 동시에 처리할 수 있는 범용성을 제공한다.
> 3. **판단 포인트**: IT 기반 기술을 적극 활용하므로 'CIP Security'와 같은 확장 표준을 통해 인증, 무결성, 기밀성을 확보하는 것이 보안의 핵심이다.

---

## Ⅰ. 개요 및 필요성

EtherNet/IP는 로크웰 오토메이션(Rockwell Automation) 주도로 개발되어 현재 ODVA에 의해 관리되는 개방형 표준이다. 여기서 'IP'는 Internet Protocol이 아니라 **Industrial Protocol**을 의미하며, 이는 산업용 객체 모델인 CIP를 이더넷 환경에 올렸음을 뜻한다.

전 세계 공장 자동화 시장의 큰 축을 담당하고 있으며, 특히 북미 지역에서 지배적인 위치를 차지한다. 표준 이더넷 장비를 그대로 쓸 수 있어 구축 비용이 저렴하지만, 그만큼 IT 영역의 보안 취약점이 OT 영역으로 그대로 전이될 위험이 크다.

- **📢 섹션 요약 비유**: EtherNet/IP는 일반 도로(이더넷) 위를 달리는 '산업 전용 특수 트럭(CIP)'과 같다. 도로가 익숙해서 편리하지만, 도로 위의 위험(해커)도 똑같이 만날 수 있다.

---

## Ⅱ. 아키텍처 및 통신 모델

EtherNet/IP는 OSI 7계층 중 상위 계층에 CIP를 배치하고 하위 계층에 표준 이더넷/IP를 사용한다.

| 계층 | 사용 기술 | 특징 |
|:---|:---|:---|
| **상위 (Application)** | CIP (Common Industrial Protocol) | 장치 정의, 서비스, 메시징 등 객체 모델 중심 |
| **중간 (Transport/Network)** | TCP / UDP / IP | 명시적(Explicit) 메시지는 TCP, 실시간(Implicit) 메시지는 UDP 사용 |
| **하위 (Data Link/Physical)** | Ethernet (IEEE 802.3) | 표준 NIC 및 스위치 사용 가능 |

```text
┌──────────────────────────────────────────────────────────────┐
│                  EtherNet/IP의 통신 방식 구분               │
├──────────────────────────────────────────────────────────────┤
│ [Explicit Messaging] ──▶ (TCP 기반) ──▶ [장비 설정/진단]      │
│     - 연결 지향적, 비정기적 데이터 전송                      │
│                                                              │
│ [Implicit Messaging] ──▶ (UDP 기반) ──▶ [실시간 제어]         │
│     - 비연결 지향적, 주기적 I/O 데이터 전송 (실시간성)       │
└──────────────────────────────────────────────────────────────┘
```

실시간 성능을 위해 UDP 멀티캐스트를 자주 사용하며, 이때 네트워크 부하를 줄이기 위해 스위치 레벨의 **IGMP Snooping** 설정이 보안 및 성능 면에서 필수적이다.

- **📢 섹션 요약 비유**: 급한 물건은 오토바이(UDP)로 빠르게 보내고, 복잡한 서류는 등기우편(TCP)으로 정확하게 보내는 전략을 쓴다.

## Ⅲ. 비교 및 연결 (EtherNet/IP vs PROFINET)

산업용 이더넷 시장의 양대 산맥인 두 프로토콜을 비교한다.

| 항목 | EtherNet/IP | PROFINET |
|:---|:---|:---|
| 주도 기관 | ODVA | PI (PROFIBUS & PROFINET International) |
| 핵심 프로토콜 | CIP (Common Industrial Protocol) | PROFIdrive, PROFIenergy 등 |
| 실시간성 구현 | 표준 UDP/IP 및 스위치 우선순위 활용 | 전용 하드웨어(IRT) 또는 스택 최적화 |
| IT 친화도 | 매우 높음 (표준 TCP/IP 스택 그대로 사용) | 높음 (NRT/RT 채널은 IT 호환 가능) |
| 보안 솔루션 | CIP Security | PROFINET Security |

- **📢 섹션 요약 비유**: EtherNet/IP가 '기존 도로를 잘 닦아서 쓰는 방식'이라면, PROFINET은 '필요할 때 전용 차선을 따로 만드는 방식'에 가깝다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 EtherNet/IP 환경에서 보안 강화를 위해 **'CIP Security'** 적용을 강력히 권고해야 한다. 이는 TLS와 DTLS를 통해 장치 간 상호 인증을 수행하고 제어 패킷의 변조를 막는 최신 보안 레이어다.

### 체크리스트
1. 네트워크 스위치에서 비정상적인 멀티캐스트 트래픽 폭주(Storm)를 차단하고 있는가?
2. CIP Security를 지원하는 최신 컨트롤러와 통신 모듈을 사용 중인가?
3. 제어망 내에서 EtherNet/IP 장치 외의 불필요한 IP 통신이 차단되어 있는가?
4. 자산 관리 도구를 통해 연결된 EtherNet/IP 장치의 인벤토리가 실시간 관리되는가?

### 안티패턴
- 일반 오피스용 허브를 사용하여 데이터 충돌(Collision)과 패킷 유실을 방치하는 행위
- CIP 통신 포트(44818, 2222)가 외부 인터넷에 노출되어 장치 설정이 변경되는 행위

- **📢 섹션 요약 비유**: 고속도로가 아무리 넓어도 톨게이트(인증)가 없으면 위험한 차들이 마음대로 섞여 들어오게 된다.

## Ⅴ. 기대효과 및 결론

EtherNet/IP의 도입은 공장 하부의 제어 데이터와 상부의 경영 정보 시스템(ERP/MES) 간의 '수직적 통합'을 완성한다. 보안이 확보된 EtherNet/IP는 스마트 제조의 데이터 주권을 지키는 핵심 인프라가 된다.

결론적으로 EtherNet/IP 보안은 '표준 기술의 취약점 보완'과 '산업용 보안 확장'의 조화이다. 기술사는 표준 이더넷 기술에 능숙하면서도 OT 환경의 특수성을 이해하는 융합적 시각으로 아키텍처를 설계해야 한다.

- **📢 섹션 요약 비유**: EtherNet/IP는 공장의 '디지털 척추'이다. 이 척추가 건강하고 안전하게 연결되어야 공장 전체가 유기적으로 움직일 수 있다.
