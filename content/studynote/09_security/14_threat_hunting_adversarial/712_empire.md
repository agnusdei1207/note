## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Empire는 파이썬 기반의 서버와 파워셸(PowerShell) 및 닷넷(.NET) 기반의 에이전트를 사용하는 오픈소스 포스트 익스플로잇(Post-exploitation) 프레임워크이다.
> 2. **가치**: 윈도우 내장 도구인 PowerShell을 극한으로 활용하여 '파일리스(Fileless)' 공격을 수행하며, 자격증명 탈취, 측면 이동, 권한 상승 모듈을 통합 관리한다.
> 3. **판단 포인트**: PowerShell 스크립트 블록 로깅(Event ID 4104)과 AMSI(Antimalware Scan Interface)를 통해 에이전트의 난독화된 실행 행위를 차단하고 모니터링해야 한다.

---

## Ⅰ. 개요 및 필요성

Empire는 'Living off the Land (LotL)' 전략을 충실히 구현한 프레임워크다. 공격자가 타겟 시스템에 별도의 실행 파일(.exe)을 업로드하는 대신, 시스템에 이미 존재하는 `powershell.exe`를 통해 메모리 상에서만 동작하는 에이전트를 실행시킨다.

이러한 접근 방식은 고전적인 안티바이러스(AV) 시그니처 탐지를 손쉽게 우회하게 해준다. 특히 레드팀 작전이나 침투 테스트에서 탐지를 피해 영속성(Persistence)을 유지하고, 복잡한 내부망 침투 시나리오를 자동화하기 위해 매우 높은 필요성을 가진다.

- **📢 섹션 요약 비유**: Empire는 건물에 외부인을 들여보내는 대신, 이미 건물 안에 있는 직원(PowerShell)을 포섭하여 몰래 일을 시키는 '스파이 지휘소'와 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

Empire는 서버(C2)와 에이전트(Agent) 간의 통신을 위해 다양한 리스너(Listener)와 스테이저(Stager)를 사용한다. 초기 실행을 위한 짧은 코드가 실행되면, 메모리 상에서 전체 에이전트 코드를 내려받아 실행하는 '스테이징' 과정을 거친다.

| 구성 요소 | 역할 | 상세 내용 |
|:---|:---|:---|
| Listener | 에이전트의 연결 대기 | HTTP, HTTPS, Dropbox, OneDrive 등 다양한 채널 |
| Stager | 에이전트 초기 설치 코드 | 원라이너(One-liner), 워드 매크로, LNK 등 |
| Agent | 타겟 시스템에서 동작하는 봇 | 메모리 상에서 명령 대기 및 실행 |
| Module | 공격 기능 유닛 | Mimikatz, BloodHound 수집기, 우회 기법 등 |

```text
┌──────────────────────────────────────────────────────────────┐
│                  Empire 프레임워크 동작 구조                 │
├──────────────────────────────────────────────────────────────┤
│ [Empire Server] ◀─── (C2 통신) ───▶ [Target Agent]           │
│       │                                  │                   │
│       ├─▶ [Listener]                     ├─▶ [In-Memory Exec]│
│       └─▶ [Module]                       └─▶ [LotL Tools]    │
└──────────────────────────────────────────────────────────────┘
```

최근에는 `BC-Security`에 의해 유지보수되며 파워셸뿐만 아니라 Python 3 에이전트와 IronPython을 활용한 닷넷 공격 기능이 강화되어, 윈도우뿐만 아니라 리눅스 및 macOS 환경에서도 강력한 위력을 발휘한다.

- **📢 섹션 요약 비유**: 스테이저는 아주 작은 열쇠 구멍으로 실을 집어넣는 것이고, 그 실을 타고 거대한 밧줄(에이전트)이 따라 들어와 내부를 점령하는 방식이다.

## Ⅲ. 비교 및 연결

Empire는 다른 C2 프레임워크들과의 비교를 통해 그 특징이 선명해진다.

| 비교 항목 | Empire | Cobalt Strike | Metasploit |
|:---|:---|:---|:---|
| 주요 언어 | PowerShell / Python | C / Assembly (Beacon) | Ruby |
| 비용 | 오픈소스 (무료) | 상용 (고가) | 오픈소스 / 상용 혼재 |
| 주안점 | 파일리스 포스트 익스플로잇 | 스텔스 통신 및 프로파일링 | 익스플로잇 및 범용성 |
| 탐지 지점 | PowerShell 실행 로그 | 비정상 비콘 통신 패턴 | 익스플로잇 시그니처 |

Empire에서 수집된 정보는 `710. BloodHound`와 연동되어 다음 공격 타겟을 정하는 데 사용되며, `711. CrackMapExec`과 함께 대규모 인프라 공격에 활용되기도 한다.

- **📢 섹션 요약 비유**: Metasploit이 "다양한 도구가 들어있는 맥가이버 칼"이라면, Empire는 "투명 인간처럼 숨어 다니는 닌자"다.

## Ⅳ. 실무 적용 및 기술사 판단

Empire와 같은 PowerShell 기반 공격을 막기 위해서는 '실행 차단'보다 '가시성 확보'가 우선이다. 공격자는 파워셸 실행을 막아도 우회하는 도구를 가져오기 때문이다.

### 체크리스트
1. PowerShell Script Block Logging (4104)이 활성화되어 중앙 로그 서버로 전송되는가?
2. AMSI(Antimalware Scan Interface)가 활성화되어 실시간 메모리 스캐닝을 수행하는가?
3. `powershell.exe`가 외부 인터넷(C2 서버)과 직접 통신하는 비정상적인 행위가 있는가?

### 안티패턴
- PowerShell 실행 권한(Execution Policy)을 변경하는 것만으로 보안이 되었다고 믿는 행위 (너무나 쉽게 우회됨)
- 난독화된 파워셸 코드를 단순한 키워드 기반으로만 탐지하려 하는 행위

- **📢 섹션 요약 비유**: 직원이 일을 못 하게 막는 것보다, 직원이 무슨 말을 하고 어떤 문서를 보는지 일기(로그)를 쓰게 하는 것이 더 효과적이다.

## Ⅴ. 기대효과 및 결론

Empire 대응 체계 구축은 곧 윈도우 환경에서의 LotL 공격에 대한 방어력을 의미한다. 이를 통해 공격자가 시스템 도구를 악용하여 은밀하게 활동하는 공간을 좁히고, 사고 발생 시 빠른 분석 데이터를 확보할 수 있다.

결론적으로 Empire는 강력한 오픈소스 도구로서 공격 기법의 대중화를 이끌었다. 방어자는 이에 대응하기 위해 단순 보안 제품 도입을 넘어, OS 수준의 강화된 로깅 정책과 닷넷 기반의 심층적 행동 분석 역량을 갖추어야 한다.

- **📢 섹션 요약 비유**: 닌자를 잡으려면 담장을 높이기보다 마당에 소리가 나는 모래(로깅)를 깔아두는 것이 좋다.
