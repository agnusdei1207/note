## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Autopsy는 Sleuth Kit(TSK)를 기반으로 하는 오픈소스 디지털 포렌식 플랫폼으로, 직관적인 GUI를 통해 복잡한 증거 분석을 자동화한다.
> 2. **가치**: 상용 도구(EnCase, Axiom 등)에 버금가는 강력한 수집 및 분석 기능을 무료로 제공하며, 타임라인 분석, 키워드 검색, 해시 대조 등을 지원한다.
> 3. **판단 포인트**: 예산이 제한된 환경이나 포렌식 입문 단계, 혹은 상용 도구의 교차 검증(Cross-validation)이 필요할 때 최우선적으로 고려되는 도구다.

---

## Ⅰ. 개요 및 필요성

Autopsy는 디지털 포렌식 수사의 모든 과정을 단일 도구에서 처리할 수 있게 해주는 통합 플랫폼이다. 포렌식 조사관은 수천 개의 파일과 로그 속에서 의미 있는 증거를 찾아내야 하는데, 로우 레벨 명령어를 하나씩 입력하는 방식은 비효율적이다.

Autopsy는 이러한 불편함을 해소하기 위해 파일 시스템 분석 엔진인 Sleuth Kit 위에서 동작하는 GUI 환경을 제공한다. 이를 통해 삭제된 파일 복구, 웹 브라우저 기록 분석, 이메일 조사 등을 자동화하여 조사관의 분석 부하를 크게 줄여준다.

- **📢 섹션 요약 비유**: Autopsy는 수사관에게 돋보기와 핀셋 대신, 범인의 지문과 동선을 자동으로 찾아주는 첨단 수사 본부를 제공하는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

Autopsy는 계층화된 분석 엔진(Ingest Modules) 아키텍처를 채택하고 있다. 데이터 소스를 추가하면 백그라운드에서 여러 분석 모듈이 동시에 돌아가며 인덱싱을 수행한다.

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│                          Autopsy 아키텍처 (Autopsy Architecture)              │
├─────────────────────────────────────────────────────────────────────────────┤
│  [Data Sources] ──▶ [Ingest Modules] ──▶ [Central Database] ──▶ [UI Display] │
│  (Disk Images,       (Email, Keyword,      (SQLite/PostgreSQL)   (Timeline,   │
│   Logical Files)      Hash, Recent Activity)                      Results)     │
│                                                                             │
│  * 하부 엔진: Sleuth Kit (TSK) - 파일 시스템 분석 (FAT, NTFS, EXT, HFS+)        │
└─────────────────────────────────────────────────────────────────────────────┘
```

| 구성 요소 | 설명 | 주요 역할 |
|:---|:---|:---|
| Ingest Modules | 데이터 유입 시 실행되는 분석기 | 해시 값 대조, 키워드 검색, EXIF 추출 등 자동화 |
| Sleuth Kit (TSK) | 하부 커맨드라인 도구 세트 | 볼륨 및 파일 시스템의 로우 레벨 분석 수행 |
| Central Repository | 공통 지식 베이스 | 과거 케이스와의 연관 분석 지원 |
| Case Database | 사건별 데이터 관리 | 분석 결과물과 메타데이터의 영구 저장 |

- **📢 섹션 요약 비유**: Autopsy는 컨베이어 벨트와 같아서, 물건(데이터)을 올려두면 여러 기계(모듈)가 알아서 분류하고 특징을 적어둔다.

## Ⅲ. 비교 및 연결

Autopsy는 상용 포렌식 도구 및 하부 엔진인 Sleuth Kit과 밀접하게 연결된다.

| 항목 | Autopsy | Sleuth Kit (TSK) | 상용 도구 (EnCase/Axiom) |
|:---|:---|:---|:---|
| 사용자 환경 | GUI 기반 | CLI 기반 | 고도화된 GUI |
| 비용 | 무료 (오픈소스) | 무료 (오픈소스) | 매우 고가 (연간 수천 달러) |
| 확장성 | 파이썬/자바 모듈 지원 | 스크립트 연동 가능 | 자체 SDK 제공 |
| 주요 기능 | 통합 분석 플랫폼 | 로우 레벨 수집/분석 | 강력한 분석/보고서 및 기술지원 |

Autopsy는 특히 **타임라인 분석(Timeline Analysis)** 기능이 뛰어나, 파일 생성/수정/접근/삭제(MACB) 시각을 그래프로 보여주어 사건의 전후 맥락을 파악하는 데 매우 유리하다.

- **📢 섹션 요약 비유**: Sleuth Kit이 수동 렌치라면 Autopsy는 전동 드릴이고, 상용 도구는 로봇 팔이 달린 자동 공구함이다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 Autopsy는 초기 분석(Pre-analysis)이나 상용 도구가 지원하지 않는 특정 파일 시스템의 분석을 보완할 때 사용된다. 또한, 솔로 조사관이나 학계에서 포렌식 원리를 학습하고 실습하는 표준 도구로 자리 잡았다.

### 체크리스트
1. **Ingest 순서 조절**: 속도가 빠른 Hash Lookup을 먼저 하고, 시간이 오래 걸리는 Keyword Search는 나중에 실행하도록 설정했는가?
2. **해시 세트 연동**: NIST NSRL(국가 소프트웨어 참조 라이브러리) 해시 세트를 연동하여 알려진 정상 파일을 분석 대상에서 제외했는가?
3. **교차 검증**: Autopsy에서 찾은 증거가 다른 도구에서도 동일하게 나타나는지 확인했는가?

- **📢 섹션 요약 비유**: 유능한 의사가 청진기(무료 도구)와 MRI(유료 도구)를 함께 써서 오진을 막는 것과 같다.

## Ⅴ. 기대효과 및 결론

Autopsy를 활용하면 포렌식 분석의 진입 장벽을 낮추고 조사 비용을 획기적으로 절감할 수 있다. 특히 오픈소스임에도 불구하고 전 세계 수사 기관과 전문가들이 기여한 모듈들이 지속적으로 업데이트되어 최신 침해 사고 패턴에도 대응할 수 있다.

결론적으로 Autopsy는 "기술적 정확성"과 "사용자 편의성"을 동시에 잡은 도구이며, 디지털 포렌식 기술사라면 반드시 숙달해야 할 기본 병기(兵器)다.

- **📢 섹션 요약 비유**: Autopsy는 디지털 수사 세계의 '위키피디아'나 '리눅스'와 같아서, 모두가 함께 만들고 누구나 강력하게 쓸 수 있다.

| 디지털 증거 무결성 | 포렌식 도구가 원본 데이터를 훼손하지 않고 분석해야 하는 원칙 |
