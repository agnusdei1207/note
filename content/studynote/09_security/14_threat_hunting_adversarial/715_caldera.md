## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Caldera는 MITRE ATT&CK 프레임워크를 기반으로 실제 공격자의 전술, 기법, 절차(TTPs)를 자동화하여 테스트하는 사이버 침투 및 지능형 위협 시뮬레이션 플랫폼이다.
> 2. **가치**: 보안 운영 팀(Blue Team)이 자사의 탐지 및 대응 능력을 실제 공격 시나리오와 대조하여 정량적으로 평가하고 개선할 수 있게 한다.
> 3. **판단 포인트**: 단순한 취약점 스캔이 아닌, '성공한 침투 이후의 행동'을 재현하므로 실제 내부망 보안 체계의 실효성을 검증하는 데 최적의 도구다.

---

## Ⅰ. 개요 및 필요성

Caldera는 "공격자가 들어왔을 때, 우리는 이를 정말 탐지할 수 있는가?"라는 질문에 답하기 위해 만들어졌다. 기존 보안 테스트는 알려진 취약점(CVE) 유무를 확인하는 데 그쳤으나, 실제 공격은 정상 도구를 악용하거나 논리적 허점을 찌르는 방식으로 진행된다.

Caldera는 MITRE ATT&CK에 정의된 수백 개의 기법을 '능력(Ability)' 단위로 모듈화하여, 이를 조합한 '작전(Operation)'을 자동으로 수행한다. 이를 통해 기업은 고가의 레드팀 컨설팅 없이도 상시로 공격 시뮬레이션을 수행하고 보안 가시성을 확보할 수 있다.

- **📢 섹션 요약 비유**: Caldera는 화재가 발생했을 때 스프링클러와 대피로가 제대로 작동하는지 확인하기 위해 실시하는 '가상 화재 훈련 로봇'과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

Caldera는 서버(C2)와 에이전트(Sandcat 등), 그리고 공격 시나리오를 정의하는 프로파일(Adversary Profile)로 구성된다. 특히 'Planner' 알고리즘을 통해 현재 시스템 상황에 맞는 최적의 다음 공격 단계를 동적으로 결정한다.

| 구성 요소 | 역할 | 상세 내용 |
|:---|:---|:---|
| Ability | 최소 공격 단위 | 특정한 ATT&CK 기법을 실행하는 코드 (PowerShell, Bash 등) |
| Adversary | 공격자 프로파일 | 특정 APT 그룹(예: APT28)의 기법들을 모아놓은 세트 |
| Operation | 시뮬레이션 실행 | 대상 호스트를 지정하고 특정 Adversary를 적용하여 자동 실행 |
| Agent | 타겟 에이전트 | 명령을 전달받아 실행하는 경량 프로그램 (Sandcat, Manx 등) |

```text
┌──────────────────────────────────────────────────────────────┐
│                    Caldera 자동화 에뮬레이션 구조            │
├──────────────────────────────────────────────────────────────┤
│ [Caldera Server] ──▶ [Planner] ──▶ [Next Best Action 결정]   │
│       ▲                │                   │                 │
│       │          ┌─────┴─────┐       ┌─────┴─────┐           │
│       └──── [Agent 1]     [Agent 2]     [Agent 3]            │
└──────────────────────────────────────────────────────────────┘
```

Caldera의 핵심은 '연쇄 공격'이다. 예를 들어 '자격증명 탈취' 능력이 성공하면, 여기서 얻은 정보를 이용해 '측면 이동' 능력을 자동으로 연계하여 실행한다.

- **📢 섹션 요약 비유**: 로봇 청소기가 집 구조를 파악하며 먼지를 찾듯, Caldera 로봇은 네트워크 구조를 파악하며 보안 구멍을 찾아낸다.

## Ⅲ. 비교 및 연결

Caldera는 다른 보안 진단 도구들과 목적지에서 큰 차이를 보인다.

| 비교 항목 | Caldera | 일반 스캐너 (Nessus) | 713. Cobalt Strike |
|:---|:---|:---|:---|
| 주요 목적 | 방어 체계 검증 (BAS) | 취약점 식별 | 실제 침투 및 작전 수행 |
| 실행 주체 | 내부 보안 팀 (Blue/Purple) | 보안 관리자 | 외부 공격자 / 레드팀 |
| 분석 기준 | MITRE ATT&CK TTPs | CVE 리스트 | 공격자 의도 및 목표 |
| 자동화 수준 | 매우 높음 | 높음 | 낮음 (수동 조작 위주) |

Caldera는 `712. Empire`나 `713. Cobalt Strike`의 실제 공격 코드를 'Ability'로 등록하여 에뮬레이션에 활용할 수 있어, 실전과 매우 유사한 환경을 구축할 수 있다.

- **📢 섹션 요약 비유**: 스캐너가 "문이 잠겼나?"를 보고, Cobalt Strike가 "직접 문을 따고 들어가는 것"이라면, Caldera는 "도둑이 문을 땄을 때 알람이 울리는지 시험하는 것"이다.

## Ⅳ. 실무 적용 및 기술사 판단

Caldera를 효과적으로 활용하려면 시뮬레이션 결과를 EDR/SIEM의 로그와 대조하는 '결과 분석' 과정이 필수적이다.

### 체크리스트
1. 시뮬레이션된 TTPs가 보안 장비에서 경보(Alert)를 발생시켰는가?
2. 경보가 발생하지 않았다면, 로그(Event Log) 자체는 남아있는가?
3. 특정 공격 단계에서 차단되었다면, 그 차단 주체가 무엇인지 파악했는가? (AV, EDR, IPS 등)

### 안티패턴
- 시뮬레이션을 돌려만 보고 실제 탐지 규칙(Rule)을 업데이트하지 않는 행위
- 운영 환경의 가용성을 고려하지 않고 공격적인 파괴 기법을 시뮬레이션하는 행위

- **📢 섹션 요약 비유**: 훈련 후에 "어디가 부족했는지" 반성하지 않으면, 다음 훈련은 시간 낭비일 뿐이다.

---

## Ⅴ. 기대효과 및 결론

Caldera 도입의 가장 큰 기대효과는 '보안의 정량화'다. 우리 회사가 ATT&CK 프레임워크의 몇 %를 탐지할 수 있는지 지표로 산출할 수 있게 된다. 이는 보안 투자의 우선순위를 정하는 데 결정적인 근거가 된다.

결론적으로 Caldera는 단순한 공격 도구가 아니라, 방어자의 역량을 끌어올리는 '학습 도구'다. 지능형 위협이 일상화된 시대에, 가상의 적을 통해 끊임없이 자신을 단련하는 것이 가장 확실한 보안 대책이다.

- **📢 섹션 요약 비유**: 매일 아침 거울(Caldera)을 보며 매무새를 가다듬어야, 밖에서 누군가를 만날 때 당당할 수 있다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| MITRE ATT&CK | Caldera가 사용하는 공격 기법 분류 체계의 표준 |
| BAS (Breach and Attack Simulation) | 침해 및 공격 시뮬레이션을 수행하는 보안 카테고리 |
| Purple Teaming | 공격(Red)과 방어(Blue)가 협력하여 보안을 개선하는 활동 |
| TTPs | 공격자가 사용하는 전술(Tactics), 기법(Techniques), 절차(Procedures) |

### 👶 어린이를 위한 3줄 비유 설명

1. Caldera는 우리 집 보안 시스템이 도둑을 잘 잡는지 시험해주는 '착한 도둑 로봇'이에요.
2. 이 로봇은 진짜 도둑들이 쓰는 방법을 그대로 흉내 내서 우리 집을 여기저기 찔러봐요.
3. 로봇이 지나갔는데 경보기가 안 울리면, 진짜 도둑이 오기 전에 얼른 고쳐야 해요.
