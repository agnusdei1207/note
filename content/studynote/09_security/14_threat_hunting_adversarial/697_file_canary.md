## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 카나리는 중요한 데이터가 포함된 것처럼 위장한 특수 파일로, 공격자가 이 파일을 열거나, 복사하거나, 수정할 때 즉각적인 경보를 발생시키는 탐지 기법이다.
> 2. **가치**: 문서 편집기나 OS의 파일 핸들링 특성을 이용하여, 네트워크 단에서 놓친 데이터 유출(Exfiltration) 시도를 최종 단계에서 포착한다.
> 3. **판단 포인트**: 공격자가 의심 없이 파일을 열어보게 만드는 '사회공학적 유인(Lure)' 요소와, 시스템 성능에 영향을 주지 않는 '경량성'이 핵심이다.

---

## Ⅰ. 개요 및 필요성

파일 카나리 (File Canary)는 데이터 중심 보안의 핵심 도구다. 공격자가 서버를 장악한 뒤 가장 먼저 하는 행위 중 하나는 "중요해 보이는 파일"을 찾아보는 것이다. 파일 카나리는 이런 공격자의 습성을 역이용하여, 파일 자체에 추적 기능을 심어둔 상태를 말한다.

이 기법이 필요한 이유는 '내부 이동 및 데이터 탈취'의 확실한 증거를 잡기 위해서다. 일반적인 파일 접근 로그는 양이 너무 방대하여 진짜 위협을 찾기 어렵지만, 파일 카나리는 "건드려서는 안 될 파일"에 대한 접근이므로 발생하는 즉시 100% 신뢰할 수 있는 침해 사고 지표가 된다.

- **📢 섹션 요약 비유**: 파일 카나리는 금고 안에 넣어둔 '위조지폐 뭉치'와 같다. 도둑이 가져가서 쓰려고 하는 순간 가짜임이 드러나고 위치가 추적된다.

## Ⅱ. 아키텍처 및 핵심 원리

파일 카나리는 주로 오피스 문서(Word, Excel), PDF, 또는 실행 파일 형태로 제작되며, 내부에 보이지 않는 추적 코드가 삽입되어 있다.

| 파일 유형 | 작동 원리 | 탐지 시점 |
|:---|:---|:---|
| MS Office (Docx/Xlsx) | 외부 이미지 링크 또는 매크로 실행 | 파일 오픈 시 외부 비콘 전송 |
| PDF | 외부 자원 요청 (URL) 포함 | 뷰어로 파일을 읽는 순간 |
| 윈도우 바로가기 (LNK) | 아이콘 이미지를 외부 서버에서 호출 | 폴더를 열어 파일을 보기만 해도 탐지 |
| 실행 파일 (EXE) | 실행 시 특정 API 호출 및 알람 | 공격자가 툴을 실행하는 순간 |

```text
┌──────────────────────────────────────────────────────────────┐
│                  파일 카나리 (Word 문서 예시)                 │
├──────────────────────────────────────────────────────────────┤
│ [파일 내부]                                                  │
│  - "2026 경영 전략.docx"                                     │
│  - <img src="http://canary.io/token_id=1234"> (숨겨진 링크)  │
│                                                              │
│ [공격자 행위] : 파일을 열어 내용을 확인하려 함                │
│                                                              │
│ [결과] : 워드 프로그램이 이미지를 불러오기 위해 서버에 접속   │
│         ──▶ 보안팀에 "전략 파일 유출됨" 알람 발생             │
└──────────────────────────────────────────────────────────────┘
```

핵심 원리는 '외부 통신 유발'이다. 파일이 활성화되는 순간, 파일을 열고 있는 기기의 공인 IP 정보와 함께 고유 토큰 값이 관리 서버로 전달된다.

- **📢 섹션 요약 비유**: 편지를 뜯는 순간 보이지 않는 연기가 솟아올라 보낸 사람에게 전달 사실을 알리는 '마법 편지'와 같다.

## Ⅲ. 비교 및 연결

파일 카나리는 시스템 보안의 스택 카나리(Stack Canary)와 이름은 같으나 용도가 다르다.

| 항목 | 스택 카나리 (Stack Canary) | 파일 카나리 (File Canary) |
|:---|:---|:---|
| 대상 | 메모리 스택 프레임 | 저장 장치의 데이터 파일 |
| 목적 | 버퍼 오버플로우 공격 방지 | 데이터 유출 및 비인가 접근 탐지 |
| 작동 위치 | CPU / 프로세스 내부 | 파일 시스템 / 애플리케이션 |
| 결과 | 프로세스 종료 | 보안팀에 경보 전송 |

파일 카나리는 랜섬웨어 탐지 기술과도 연결된다. 랜섬웨어가 파일을 암호화하기 위해 파일 카나리를 건드리는 순간 이를 감지하고 프로세스를 차단하는 방식으로 쓰인다.

- **📢 섹션 요약 비유**: 스택 카나리가 '엔진 과열 방지 장치'라면, 파일 카나리는 '차량 도난 방지 GPS'다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사 관점에서 파일 카나리의 가장 큰 숙제는 '자연스러움'이다. 공격자는 "Password.txt" 같은 너무 뻔한 이름은 의심한다. 조직의 실제 도메인과 직무에 맞는 파일명(예: "2025_급여_조정안.pdf")을 선정하는 전략이 필요하다.

### 체크리스트
1. 파일 카나리가 사내 백신(AV)이나 스캐너에 의해 탐지되어 삭제되지 않도록 예외 처리되었는가?
2. 파일이 열릴 때 발생하는 외부 통신이 내부 프록시(Proxy) 서버를 거쳐도 식별 가능한가?
3. 토큰 서버의 도메인이 공격자에게 "보안용"임을 들키지 않게 위장되어 있는가?

### 안티패턴
- 파일의 용량이 너무 크거나 여는 데 시간이 오래 걸려 공격자가 이상함을 느끼게 만드는 경우
- 퇴사자나 관리자의 정상적인 데이터 이관 작업을 공격으로 오인하여 불필요한 소동을 일으키는 경우

- **📢 섹션 요약 비유**: 낚시 미끼가 너무 크거나 가짜 티가 나면 물고기(공격자)가 물지 않는다. 가장 자연스러운 미끼가 필요하다.

---

## Ⅴ. 기대효과 및 결론

파일 카나리는 공격자에게 '심리적 지뢰밭'을 걷는 듯한 공포를 준다. 성공적으로 데이터를 훔쳐 갔다고 생각하는 순간 자신의 위치를 노출하게 만들기 때문이다. 이는 공격자의 데이터 분석 단계를 방해하고 추적의 결정적 단서를 제공한다.

결론적으로 파일 카나리는 데이터 보호의 '마지막 보루'다. 경계가 뚫리고 시스템이 장악되어도, 데이터 자체에 심어진 지능을 통해 공격자를 역추적할 수 있는 가장 스마트한 기법이다.

- **📢 섹션 요약 비유**: 도둑이 보석을 훔쳐가도 그 보석 안에서 계속 신호가 간다면, 도둑은 보석을 가지고 있을수록 위험해진다.
