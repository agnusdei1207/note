## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Velociraptor는 수천 대의 엔드포인트에서 가시성을 확보하고, 원격으로 디지털 포렌식 아티팩트를 수집 및 분석하는 강력한 오픈소스 엔드포인트 보안 도구이다.
> 2. **가치**: VQL (Velociraptor Query Language)을 통해 파일 검색, 메모리 덤프, 이벤트 모니터링을 코드로 자동화하여 침해사고 대응(IR)의 속도를 극대화한다.
> 3. **판단 포인트**: 정적 수집보다는 '사냥(Hunting)'과 '실시간 감시'에 강점이 있으며, 대규모 환경에서의 빠른 데이터 수집과 중앙 집중식 분석이 필요할 때 최적의 선택이다.

---

## Ⅰ. 개요 및 필요성

침해사고가 발생했을 때 수사관이 수백 대의 PC를 일일이 찾아다니며 데이터를 복사하는 것은 불가능하다. Velociraptor는 이러한 '원격 포렌식'의 한계를 극복하기 위해 등장했다. 중앙 서버에서 클릭 몇 번으로 모든 엔드포인트의 MFT, 레지스트리, 브라우저 기록 등을 동시에 긁어올 수 있다.

기존의 도구들이 정해진 데이터만 수집했다면, Velociraptor는 사용자가 직접 쿼리(VQL)를 작성하여 "지난 1시간 동안 생성된 실행 파일만 찾아라"와 같은 정교한 사냥(Threat Hunting)을 수행할 수 있게 해준다.

- **📢 섹션 요약 비유**: Velociraptor는 모든 건물에 배치되어 본부의 명령에 따라 즉시 증거를 수집하고 보고하는 '초능력 경비원 부대'와 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

Velociraptor는 서버와 에이전트 간의 효율적인 통신을 위해 `gRPC`와 `HTTP/2`를 사용한다. 가장 큰 특징은 모든 동작이 **VQL (Velociraptor Query Language)**이라는 고유한 언어로 정의된다는 점이다.

| 구성 요소 | 역할 | 상세 내용 |
|:---|:---|:---|
| VQL | 프레임워크의 심장 | 데이터 수집, 가공, 필터링을 정의하는 쿼리 언어 |
| Artifact | 재사용 가능한 지식 | 특정 조사 목적(예: Amcache 수집)을 위해 작성된 VQL 세트 |
| Hunt | 대규모 조사 작전 | 수천 대의 에이전트에 동시에 Artifact 수집 명령 하달 |
| Notebook | 협업 분석 도구 | 수집된 데이터를 서버 내에서 SQL로 2차 가공 및 시각화 |

```text
┌──────────────────────────────────────────────────────────────┐
│                  Velociraptor VQL 실행 구조                  │
├──────────────────────────────────────────────────────────────┤
│ [Server UI] ──▶ [VQL 요청 (Hunt)] ──▶ [gRPC Channel]         │
│       │                                     │                │
│ [결과 분석] ◀── [데이터 업로드] ◀──── [Agent VQL 실행]        │
│  (Notebook)                                (File, Reg, Mem)  │
└──────────────────────────────────────────────────────────────┘
```

특히 'Client Side VQL' 방식은 서버가 일일이 지시하지 않아도 에이전트가 스스로 판단하여 특정 이벤트가 발생할 때만 데이터를 서버로 전송하게 하여 네트워크 부하를 줄인다.

- **📢 섹션 요약 비유**: VQL은 경비원에게 주는 '매우 구체적인 수사 지침서'와 같다. 지침서만 바꾸면 경비원은 즉시 새로운 임무를 수행한다.

## Ⅲ. 비교 및 연결

Velociraptor는 유사한 도구인 `717. Osquery`나 상용 EDR과 뚜렷한 차별점을 가진다.

| 비교 항목 | Osquery | Velociraptor | 상용 EDR |
|:---|:---|:---|:---|
| 언어 | 표준 SQL | **VQL (더 강력하고 유연함)** | 자체 비공개 언어 |
| 포렌식 깊이 | 보통 (시스템 상태 위주) | **매우 깊음 (Raw Disk 접근)** | 보통 ~ 깊음 |
| 응답 기능 | 거의 없음 | **강력함 (파일 수집, 프로세스 킬)** | 매우 강력함 (자동 차단) |
| 사용 편의성 | 높음 (SQL 익숙함) | 보통 (VQL 학습 필요) | 매우 높음 (GUI 중심) |

Velociraptor는 `719. KAPE`와 연동하여 표준화된 포렌식 패키지를 수집하거나, 수집된 로그를 `716. Red Canary` 모델처럼 분석하여 위협을 확정하는 데 활용된다.

- **📢 섹션 요약 비유**: Osquery가 "상태를 묻는 설문지"라면, Velociraptor는 "직접 현장을 파헤치는 수사관"이다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 Velociraptor는 특히 '침해사고 초기 대응(Triage)' 단계에서 진가를 발휘한다.

### 체크리스트
1. 조사에 필요한 'Artifact'가 라이브러리에 준비되어 있는가? (예: `Windows.KapeFiles.Targets`)
2. 수집된 대량의 데이터를 저장할 서버의 스토리지가 충분한가?
3. 에이전트와 서버 간의 통신이 보안 채널(TLS)을 통해 안전하게 암호화되어 있는가?

### 안티패턴
- 수천 대의 호스트에서 동시에 메모리 전체를 덤프하는 쿼리를 실행하는 행위 (네트워크 대역폭 고갈)
- VQL 쿼리의 성능(Resource Limit)을 고려하지 않아 엔드포인트의 CPU를 과점하는 행위

- **📢 섹션 요약 비유**: 모든 수사관에게 동시에 무거운 짐을 들고 뛰라고 시키면, 수사는커녕 길만 막히게 된다.

---

## Ⅴ. 기대효과 및 결론

Velociraptor 도입은 기업의 '위협 헌팅(Threat Hunting)' 역량을 내재화하는 가장 빠른 길이다. 상용 솔루션이 제공하지 못하는 로우 레벨 데이터에 대한 접근권을 확보함으로써, 고도화된 위협에 대한 대응력을 갖출 수 있다.

결론적으로 Velociraptor는 단순한 도구가 아니라, 코드로 정의되는 '포렌식 플랫폼'이다. VQL을 얼마나 잘 다루느냐에 따라 보안 팀의 대응 속도와 분석 수준이 결정될 것이다.

- **📢 섹션 요약 비유**: 최고의 수사관에게 최고의 수사 도구와 빠른 이동 수단을 제공하는 것과 같은 효과를 준다.

| 개념 | 연결 포인트 |
|:---|:---|
| VQL | Velociraptor의 모든 것을 제어하는 강력한 전용 쿼리 언어 |

### 👶 어린이를 위한 3줄 비유 설명

1. Velociraptor는 멀리 떨어져 있는 수많은 컴퓨터를 한꺼번에 조사하는 '슈퍼 탐정'이에요.
2. 탐정에게 "나쁜 사람이 남긴 흔적을 다 찾아와!"라고 명령(VQL)만 내리면 돼요.
3. 컴퓨터 하나하나 직접 가서 확인할 필요 없이, 앉은 자리에서 모든 정보를 볼 수 있답니다.
