## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Magnet AXIOM은 PC, 모바일, 클라우드 데이터를 하나의 케이스로 통합하여 분석하는 아티팩트 중심(Artifact-centric) 디지털 포렌식 솔루션이다.
> 2. **가치**: 수천 가지 앱의 데이터를 자동으로 파싱하고 상호 연결하는 강력한 'Artifact Explorer' 기능을 통해 분석가에게 직관적인 타임라인과 관계도를 제공한다.
> 3. **판단 포인트**: 모바일 기기와 클라우드 서비스(Google, iCloud 등)의 비중이 높은 현대적 침해사고 조사에서 기존의 파일 시스템 중심 도구보다 높은 효율을 발휘한다.

---

## Ⅰ. 개요 및 필요성

Magnet AXIOM은 현대 디지털 환경의 복잡성을 해결하기 위해 등장했다. 과거에는 PC 분석 도구와 모바일 분석 도구가 분리되어 있었으나, 범죄자나 공격자는 여러 플랫폼을 교차해서 사용한다. AXIOM은 이러한 '사일로(Silo)' 현상을 제거하고 파편화된 증거를 하나로 묶어준다.

특히 웹 브라우저 기록, 메신저 대화 내용, 소셜 미디어 활동 등 사용자의 행적을 직접적으로 보여주는 아티팩트 추출에 특화되어 있어, 기술적인 파일 시스템 분석보다 '사용자 행위 분석'이 중요한 사건에서 압도적인 필요성을 가진다.

- **📢 섹션 요약 비유**: AXIOM은 여러 CCTV 조각들을 모아 하나의 영화처럼 이어붙여 보여주는 '디지털 편집기'와 같다.

## Ⅱ. 아키텍처 및 핵심 원리

AXIOM은 'Process'와 'Examine'이라는 두 가지 핵심 모듈로 구성된다. Process 모듈은 데이터를 수집하고 수천 개의 파서를 통해 아티팩트를 자동 추출하며, Examine 모듈은 추출된 데이터를 시각화하고 분석한다.

| 모듈 | 역할 | 주요 특징 |
|:---|:---|:---|
| AXIOM Process | 수집 및 인덱싱 | PC/모바일/클라우드 소스 통합 처리 |
| AXIOM Examine | 데이터 시각화 및 필터링 | Timeline, Connections, Artifact Explorer |
| Cloud Plugin | 서비스 인증 정보 활용 수집 | 토큰/비밀번호 기반 클라우드 데이터 획득 |
| AI Classifier | 이미지/비디오 자동 분류 | 유해물, 마약, 무기 등 머신러닝 기반 탐지 |

```text
┌──────────────────────────────────────────────────────────────┐
│             AXIOM의 통합 분석 구조: 플랫폼의 경계 제거       │
├──────────────────────────────────────────────────────────────┤
│ [PC 증거] ─┐                               ┌─▶ [Timeline]    │
│ [Mobile]  ─┼─▶ [AXIOM Process] ─▶ [DB] ───┼─▶ [Connections] │
│ [Cloud]   ─┘                               └─▶ [Explorer]    │
└──────────────────────────────────────────────────────────────┘
```

이 솔루션의 핵심 원리는 'Connections' 기술이다. 예를 들어 특정 이메일 주소가 모바일 앱, PC 브라우저, 클라우드 계정에서 어떻게 공유되고 사용되었는지를 그래프 형태로 시각화하여 공격자의 경로를 추적한다.

- **📢 섹션 요약 비유**: Connections 기능은 흩어진 점들을 연결해 하나의 별자리를 그려내는 것과 같다.

## Ⅲ. 비교 및 연결

AXIOM은 기존의 전통적 포렌식 도구(EnCase, FTK)와 접근 방식에서 차이를 보인다.

| 비교 항목 | 전통적 도구 (EnCase/FTK) | Magnet AXIOM |
|:---|:---|:---|
| 분석 중심 | 파일 시스템 (MFT, Registry 등) | 아티팩트 (App 데이터, SQLite 등) |
| 플랫폼 지원 | PC 위주, 모바일은 별도 도구 필요 | PC, Mobile, Cloud 기본 통합 |
| 시각화 | 트리 구조 및 리스트 위주 | 타임라인, 관계망(Graph) 중심 |
| 주요 용도 | 로우레벨 파일 복구, 법적 표준 절차 | 사용자 행위 추적, 통합 케이스 분석 |

AXIOM은 단순히 포렌식 도구에 그치지 않고, 기업용 솔루션인 'AXIOM Cyber'를 통해 원격 수집 및 침해사고 대응(IR) 워크플로우와 연결되어 엔드포인트 가시성을 확장한다.

- **📢 섹션 요약 비유**: 전통적 도구가 자동차의 엔진 부품을 하나하나 뜯어보는 것이라면, AXIOM은 내비게이션 기록을 보고 차가 어디로 갔는지 확인하는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 AXIOM은 특히 '모바일 기기'가 연루된 사건이나 '클라우드 계정' 탈취 사고에서 최우선으로 선택된다. 다만, 아키텍처상 리소스를 많이 점유하므로 고성능 워크스테이션 환경이 필수적이다.

### 체크리스트
1. 분석 대상에 모바일 기기(iOS/Android)가 포함되어 있는가?
2. 클라우드 계정(Google, Office 365 등)에 대한 법적 접근 권한 및 토큰이 확보되었는가?
3. 수천 개의 아티팩트 중 사건과 직결된 필터링 조건을 설정했는가?

### 안티패턴
- 파일 시스템의 로우레벨 구조 분석이 필요한 상황에서 아티팩트 뷰어 결과만 맹신하는 경우
- 방대한 아티팩트를 필터링 없이 모두 보고서에 포함하여 핵심 증거를 가리는 행위

- **📢 섹션 요약 비유**: 정교한 필터링 없이 모든 정보를 보는 것은, 모래사장에서 바늘을 찾으면서 모래까지 다 집어오는 것과 같다.

## Ⅴ. 기대효과 및 결론

AXIOM 도입은 분석가의 '통찰력'을 극대화한다. 반복적인 데이터 파싱 작업을 도구가 대신해주므로, 분석가는 공격자의 의도와 이동 경로를 파악하는 고차원적 분석에 집중할 수 있다.

미래의 포렌식은 암호화된 기기와 파편화된 클라우드 데이터와의 싸움이 될 것이다. AXIOM은 이러한 환경에서 데이터를 '연결'하고 '시각화'하는 능력을 통해 디지털 수사의 핵심 플랫폼으로 자리 잡을 것이다.

- **📢 섹션 요약 비유**: AXIOM은 복잡한 미로 속에서 공격자가 남긴 발자국을 야광으로 빛나게 해주는 마법의 램프와 같다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| Artifact | 앱이나 시스템이 남긴 사용자 활동의 흔적 (SQLite DB, Plist 등) |
| Mobile Forensics | 스마트폰 데이터 추출 및 분석 (AXIOM의 강력한 기능) |
| Cloud Forensics | API나 토큰을 이용한 서버 데이터 수집 (AXIOM Cloud) |
| IEF (Internet Evidence Finder) | AXIOM의 전신이 된 강력한 웹 아티팩트 분석 도구 |
