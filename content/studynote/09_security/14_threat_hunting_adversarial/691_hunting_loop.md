## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Hunting Loop는 위협 헌팅의 전 과정을 가설 수립, 도구 탐색, 위협 발견, 지식화의 4단계로 구조화하여 반복 수행하는 선순환 방법론이다.
> 2. **가치**: 헌팅 활동이 분석가의 감에 의존하는 일회성 작업이 되지 않도록 체계적인 가이드를 제공하며, 발견된 위협을 조직의 자동화된 탐지 역량으로 자산화한다.
> 3. **판단 포인트**: 루프의 완성은 위협을 '발견'하는 데 그치지 않고, 이를 '정보 공유' 단계를 통해 SIEM/EDR의 영구적인 탐지 룰로 변환하는 데 있다.

---

## Ⅰ. 개요 및 필요성

Hunting Loop는 David Bianco가 제안한 위협 헌팅의 표준 프로세스다. 위협 헌팅은 끝이 없는 과정이며, 방대한 데이터 속에서 목적 없이 방황하기 쉽다. Hunting Loop는 헌팅을 하나의 과학적인 실험 과정으로 정의하여, 분석가가 명확한 목표를 가지고 데이터를 추적할 수 있도록 돕는다.

이 루프가 필요한 이유는 '보안 운영의 내재화'를 위해서다. 외부에서 들어오는 위협 정보(IoC)만으로는 우리 조직만의 특수한 위협을 막을 수 없다. 직접 루프를 돌리며 우리 환경에 최적화된 위협 시나리오를 발굴하고 대응함으로써, 조직 전체의 방어 지능을 높일 수 있다.

- **📢 섹션 요약 비유**: Hunting Loop는 수사관이 사건을 해결하기 위해 가설을 세우고 증거를 수집한 뒤, 범인을 잡고 수사 일지를 남기는 '표준 수사 절차'와 같다.

## Ⅱ. 아키텍처 및 핵심 원리

Hunting Loop는 4가지 핵심 단계가 끊임없이 맞물려 돌아가는 구조다.

| 단계 | 명칭 | 활동 내용 |
|:---|:---|:---|
| 1 | 가설 수립 (Create Hypothesis) | "공격자가 특정 방법(TTP)으로 활동 중일 것이다"라는 가정 설정 |
| 2 | 탐색 및 조사 (Investigate) | 가설을 입증할 데이터를 쿼리, 시각화, 통계 분석 등으로 조사 |
| 3 | 발견 및 대응 (Uncover & Respond) | 가설이 적중하여 위협 흔적 발견 시, 즉각적인 격리 및 근절 수행 |
| 4 | 정보 공유/자산화 (Inform & Enrich) | 성공한 헌팅 기법을 자동 탐지 룰로 전환하고 보고서 작성 |

```text
┌──────────────────────────────────────────────────────────────┐
│                  Hunting Loop (위협 헌팅 루프)                │
├──────────────────────────────────────────────────────────────┤
│          ┌──────────────────────────────────────────┐         │
│          │        [1. 가설 수립 (Hypothesis)]       │         │
│          └────────────────────┬─────────────────────┘         │
│                               ▼                               │
│          ┌──────────────────────────────────────────┐         │
│          │        [2. 탐색 및 조사 (Investigate)]    │         │
│          └────────────────────┬─────────────────────┘         │
│                               ▼                               │
│          ┌──────────────────────────────────────────┐         │
│          │        [3. 발견 및 대응 (Uncover)]        │         │
│          └────────────────────┬─────────────────────┘         │
│                               ▼                               │
│          ┌──────────────────────────────────────────┐         │
│          │        [4. 정보 공유 (Inform/Enrich)]     │         │
│          └────────────────────┬─────────────────────┘         │
│                               ▼                               │
│          (새로운 위협 인텔리전스로 1단계 피드백)              │
└──────────────────────────────────────────────────────────────┘
```

핵심 원리는 '증명과 학습'이다. 가설이 틀렸더라도 "이 방식은 현재 우리 환경에 없다"는 사실을 학습하게 되며, 가설이 맞았다면 그 즉시 방어 체계를 한 단계 업그레이드하는 재료로 삼는다.

- **📢 섹션 요약 비유**: Hunting Loop는 과학자의 실험과 같다. 가설을 세우고 실험(조사)을 통해 결과를 얻고, 이를 논문(지식화)으로 발표하여 모두가 알게 한다.

## Ⅲ. 비교 및 연결

Hunting Loop는 보안 관제(SOC)의 경보 처리와는 다른 호흡을 가진다.

| 항목 | 일반 경보 처리 (Alerting) | Hunting Loop 수행 |
|:---|:---|:---|
| 시작점 | 보안 장비의 알람 (Push) | 분석가의 의문과 가설 (Pull) |
| 주안점 | 신속한 이벤트 처리 | 심층적인 위협 경로 분석 |
| 결과물 | 티켓 클로징, 개별 조치 | 새로운 탐지 규칙, 보안 정책 강화 |
| 반복성 | 이벤트 발생 시마다 반복 | 정기적/상시적 프로세스로 반복 |

발견된 위협 정보는 STIX/TAXII와 같은 표준 형식을 통해 외부 위협 인텔리전스(TI)와 공유되거나, 내부의 SOAR 플레이북으로 자동화된다.

- **📢 섹션 요약 비유**: 경보 처리가 '벨 소리에 문을 열어주는 것'이라면, Hunting Loop는 '집안을 청소하며 구석에 숨은 벌레를 찾아 박멸하는 것'이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사 관점에서 Hunting Loop의 가장 중요한 단계는 4단계 '정보 공유'다. 헌팅은 비용이 많이 드는 수동 작업이므로, 한번 발견한 위협은 반드시 자동화된 룰로 변환하여 '운영 효율성'을 확보해야 한다.

### 체크리스트
1. 가설 수립 시 MITRE ATT&CK의 전술/기법 번호와 매핑하고 있는가?
2. 2단계 조사 시 사용할 수 있는 중앙 집중형 로그 저장소(Data Lake)가 구축되어 있는가?
3. 헌팅 성공 후 작성된 탐지 룰이 운영 환경의 SIEM에 실시간으로 반영되는가?

### 안티패턴
- 3단계에서 위협을 발견하고 조치만 한 뒤, 4단계 지식화를 생략하여 나중에 똑같은 헌팅을 반복하는 경우
- 데이터 조사 과정에서 운영 시스템에 과도한 쿼리 부하를 주어 장애를 유발하는 경우

- **📢 섹션 요약 비유**: 4단계를 생략한 헌팅은 '일기 안 쓰는 여행'과 같다. 시간은 썼지만 남는 지식이 없어 나중에 기억나지 않는다.

---

## Ⅴ. 기대효과 및 결론

Hunting Loop를 꾸준히 돌리는 조직은 공격자보다 앞서 나갈 수 있다. 공격자가 기법을 바꾸더라도 헌팅 가설 또한 계속 진화하기 때문이다. 이는 단순 방어를 넘어 '공격적인 방어(Active Defense)' 체계로의 진화를 의미한다.

결론적으로 Hunting Loop는 위협 헌팅을 '예술'에서 '공학'으로 바꾸는 핵심 프레임워크다. 체계적인 루프 반복만이 지능화된 현대의 침입자를 잡아낼 수 있는 유일한 길이다.

- **📢 섹션 요약 비유**: 운동선수가 매일 루틴을 지키며 근육을 단련하듯이, 보안팀은 Hunting Loop를 돌리며 탐지 근육을 키워야 한다.

### 👶 어린이를 위한 3줄 비유 설명

1. 도둑을 잡기 위해 4단계로 나누어 반복하는 '보안 퀴즈 게임' 같은 거예요.
2. 먼저 도둑이 어디 숨었을지 생각하고(가설), 실제로 찾아보고(탐색), 찾으면 잡고(발견), 어떻게 잡았는지 친구들에게 알려주는(공유) 거예요.
3. 이 게임을 계속하면 우리 집은 세상에서 가장 안전한 집이 된답니다.
