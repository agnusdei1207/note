## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 증거 무결성 해시는 디지털 데이터의 모든 비트를 입력값으로 하여 고유한 고정 길이 값을 추출함으로써, 증거가 수집된 시점부터 법정 제출 시점까지 단 1비트의 변조도 없었음을 증명하는 '디지털 지문'이다.
> 2. **가치**: 보이지 않는 디지털 데이터의 동일성을 수학적으로 입증하여, 수사 결과에 대한 객관적인 신뢰성과 법적 증거 능력(Chain of Custody)을 부여한다.
> 3. **판단 포인트**: MD5의 충돌 위험성을 고려하여 실무에서는 SHA-256 이상의 고강도 알고리즘을 병행하거나 주력으로 사용하여 위변조 주장을 원천 차단해야 한다.

---

## Ⅰ. 개요 및 필요성

증거 무결성 해시 (Integrity Hash)는 디지털 포렌식의 신뢰를 지탱하는 기둥이다. 종이 증거와 달리 디지털 증거는 수정이 너무나 쉽고 흔적이 남지 않는다. 해시는 이러한 디지털 데이터의 특성을 극복하기 위해, 데이터 전체를 대표하는 '단 하나의 숫자'를 생성하여 데이터의 '무결성(Integrity)'을 보장한다.

이 기술이 필요한 이유는 '증거 오염 방지' 때문이다. 수사관이나 제3자가 악의적으로 증거를 조작했는지, 혹은 장비 결함으로 데이터가 깨졌는지를 해시값 비교를 통해 즉각 확인할 수 있다. 해시값이 단 하나라도 다르면 그 증거는 법정에서 "오염된 데이터"로 간주되어 폐기된다.

- **📢 섹션 요약 비유**: 증거 무결성 해시는 증거 봉투에 붙이는 '디지털 봉인 라벨'과 같다. 라벨이 찢어지거나 번호가 다르면 누군가 봉투를 열었다는 뜻이다.

---

## II. 아키텍처 및 핵심 원리

해시 함수는 임의의 길이를 가진 데이터를 고정된 길이의 고유한 값으로 변환하는 '단방향 함수'다.

| 알고리즘 | 해시 길이 | 보안성 및 특징 | 비고 |
|:---|:---|:---|:---|
| MD5 | 128 bit | 빠르지만 충돌 공격 취약점 존재 | 과거 표준, 현재도 빠른 대조용으로 쓰임 |
| SHA-1 | 160 bit | 충돌 가능성 발견됨 | 사용 지양 권고 |
| SHA-256 | 256 bit | 매우 강력한 보안성, 충돌 사례 없음 | 현대 디지털 포렌식의 실질적 표준 |
| SHA-3 | 가변형 | Keccak 알고리즘 기반 차세대 표준 | 하드웨어 구현 효율성 높음 |

```text
┌──────────────────────────────────────────────────────────────┐
│                  해시를 통한 무결성 증명 원리                 │
├──────────────────────────────────────────────────────────────┤
│ [Data A] ──▶ [Hash Function] ──▶ [Value: 4f2e...] (Hash 1)    │
│      │                                                       │
│ (시간 경과 및 증거 이동)                                      │
│      │                                                       │
│ [Data B] ──▶ [Hash Function] ──▶ [Value: 4f2e...] (Hash 2)    │
│                                                              │
│  * 결과: Hash 1 == Hash 2 이면 "무결성 입증 완료"             │
│  * 주의: 데이터 중 한 글자만 바뀌어도 해시값은 완전히 달라짐    │
└──────────────────────────────────────────────────────────────┘
```

핵심 원리는 '눈사태 효과(Avalanche Effect)'다. 원본 데이터에서 아주 미세한 변화(예: 0을 1로 변경)만 있어도 결과 해시값은 완전히 무작위하게 변하므로, 육안으로는 알 수 없는 미세한 조작도 즉시 탄로 난다.

- **📢 섹션 요약 비유**: 해시는 데이터의 '유전자 검사'와 같다. 쌍둥이처럼 보여도 유전자(해시)를 대조하면 진짜인지 가짜인지 확실히 알 수 있다.

## Ⅲ. 비교 및 연결

포렌식에서 해시는 단순히 '보관' 용도뿐만 아니라 '검색' 용도로도 쓰인다.

| 용도 구분 | 설명 | 활용 예시 |
|:---|:---|:---|
| 무결성 입증 | 증거가 변하지 않았음을 증명 | 이미징 직후 해시 기록 및 보고서 첨부 |
| 알려진 파일 식별 | 화이트리스트 기반 불필요 파일 제거 | OS 시스템 파일 해시를 비교하여 분석 대상에서 제외 |
| 악성코드 탐지 | 알려진 악성코드 해시 대조 (NSRL 등) | 바이러스 토탈(VirusTotal) 해시 검색 |
| 동일성 대조 | 서로 다른 매체에 같은 파일이 있는지 확인 | 유출된 파일과 용의자 PC 내 파일의 해시 대조 |

해시는 디지털 증거 관리 체계(Chain of Custody)의 핵심 구성 요소이며, 전자서명(Digital Signature)의 기술적 기반이 되기도 한다.

- **📢 섹션 요약 비유**: 해시는 '범죄자 몽타주(악성코드 탐지)'이자 '인감 도장(무결성 입증)'의 역할을 동시에 수행한다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사 관점에서 가장 중요한 판단은 "어떤 알고리즘을 쓸 것인가"와 "언제 해시를 생성할 것인가"이다. 법적 분쟁을 방지하기 위해 가급적 MD5와 SHA-256을 **동시에 생성**하여 기록하는 '듀얼 해시' 전략이 권장된다.

### 체크리스트
1. 증거 수집 현장에서 이미징 완료 즉시 해시값을 생성하고 봉인지에 기록했는가?
2. 분석 보고서에 사용된 모든 파일의 해시값이 명시되어 재현성을 보장하는가?
3. 해시 생성 시 사용한 도구의 명칭과 버전이 기록되었는가? (도구마다 미세한 차이 방지)

### 안티패턴
- 수사관이 분석을 위해 파일을 열어본 뒤에 나중에 해시를 생성하는 행위 (무효화됨)
- 충돌 공격이 증명된 MD5만 단독으로 사용하여 반대 측 변호인에게 공격의 빌미를 주는 경우

- **📢 섹션 요약 비유**: 해시는 '타임스탬프'와 같다. 시간이 지난 뒤에 찍는 도장은 아무런 의미가 없다.

---

## Ⅴ. 기대효과 및 결론

증거 무결성 해시는 "디지털 데이터는 믿을 수 없다"는 불신을 "수학적으로 입증 가능하다"는 확신으로 바꾼다. 이는 디지털 포렌식이 단순한 기술을 넘어 공신력 있는 과학 수사로 인정받게 된 결정적 계기가 되었다.

결론적으로 해시는 디지털 세계의 '정직한 기록자'다. 인간의 주관이 개입할 틈이 없는 수학적 논리를 통해, 증거의 진실성을 수호하는 최후의 보루 역할을 한다.

- **📢 섹션 요약 비유**: 해시는 디지털 수사의 '눈'이다. 아주 작은 먼지(변조) 하나도 놓치지 않고 끝까지 추적한다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| Collision (충돌) | 서로 다른 데이터가 우연히 같은 해시값을 갖는 현상 (보안 취약점) |
| NSRL (National Software Reference Library) | 알려진 정상 파일들의 해시 데이터베이스 표준 |
| Digital Signature | 해시값에 개인키로 암호화를 더해 부인방지 기능을 추가한 기술 |
| Bit-stream Copy | 해시를 생성하기 위해 물리적 섹터 전체를 복제하는 행위 |

### 👶 어린이를 위한 3줄 비유 설명

1. 컴퓨터 파일의 내용을 아주 짧은 '비밀번호'로 바꾸는 요술이에요.
2. 파일 내용이 점 하나라도 바뀌면 비밀번호가 완전히 엉뚱하게 변해버려요.
3. 이 비밀번호를 미리 적어두면, 나중에 누가 몰래 파일을 고쳤는지 금방 알 수 있답니다.
