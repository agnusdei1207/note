## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가정 침투는 "방어벽은 반드시 뚫린다"는 전제하에, 공격자가 이미 내부 네트워크의 거점을 확보했음을 가정하고 이후의 방어 체계를 검증하는 보안 전략이다.
> 2. **가치**: 경계 보안(Perimeter)의 성공 여부에 집착하기보다, 내부에서의 공격자 행위 탐지 및 피해 확산 차단 역량에 집중하게 하여 실질적인 회복탄력성을 높인다.
> 3. **판단 포인트**: 초기 침투 단계(피싱, 제로데이 등)를 생략함으로써 시간과 비용을 절약하고, 권한 상승이나 내부 이동 같은 고난도 탐지 영역을 집중적으로 테스트할 수 있다.

---

## Ⅰ. 개요 및 필요성

가정 침투 (Assumed Breach)는 현대 보안의 핵심 패러다임인 '제로 트러스트(Zero Trust)'와 궤를 같이한다. 완벽한 방어막은 불가능하며, 공격자는 언젠가 직원의 실수나 알려지지 않은 취약점을 통해 내부로 들어온다는 사실을 인정하는 데서 시작한다.

기존 레드팀 훈련은 초기 침투에 많은 시간을 쏟느라 정작 중요한 내부망 방어 체계 검증을 소홀히 하는 경우가 많았다. 가정 침투 모델을 도입하면, 공격자가 이미 '일반 사용자 계정'이나 '내부 PC 하나'를 장악했다고 치고 훈련을 시작하므로, 내부 보안 관제 시스템의 실효성을 더 깊이 있게 테스트할 수 있다.

- **📢 섹션 요약 비유**: 가정 침투는 도둑이 이미 집 안으로 들어왔다고 생각하고, 금고를 열거나 다른 방으로 가는 것을 어떻게 막을지 고민하는 '내부 수비 전략'과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

가정 침투 전략은 공격자가 이미 내부의 발판(Foothold)을 마련했다는 시나리오에서 출발한다. 따라서 '초기 침투' 단계는 생략되거나 화이트팀에 의해 강제로 권한이 부여된다.

| 단계 | 활동 내용 (가정 침투 모델) | 방어자의 탐지 포인트 |
|:---|:---|:---|
| 발판 마련 | 공격자에게 내부 워크스테이션 권한 부여 | (훈련 시작 지점) |
| 내부 정찰 | AD(Active Directory) 구조 파악, 자산 탐색 | 서비스 스캐닝, 비정상 쿼리 탐지 |
| 권한 상승 | 로컬 관리자 또는 도메인 관리자 권한 획득 | 자격증명 덤프(LSASS), 권한 오남용 |
| 내부 이동 | 타 시스템 접근 및 거점 확산 | 원격 관리 도구(PsExec, WMI) 사용 |
| 목표 달성 | 핵심 데이터 탈취 또는 시스템 파괴 | 대량 데이터 외부 전송, 랜섬웨어 실행 |

```text
┌──────────────────────────────────────────────────────────────┐
│                가정 침투 (Assumed Breach) 모델                │
├──────────────────────────────────────────────────────────────┤
│ [Perimeter Defense] (뚫렸다고 가정)                           │
│          │                                                   │
│          ▼                                                   │
│ [Internal Foothold] ──▶ [Privilege Escalation] ──▶ [Target]  │
│          │                        │                          │
│          └──────▶ [Lateral Movement] ──────┘                  │
│                                                              │
│  * 핵심: 내부 망에서의 "머무르는 시간(Dwell Time)" 단축         │
└──────────────────────────────────────────────────────────────┘
```

핵심 원리는 '방어 심층성'의 검증이다. 외곽 방화벽이 뚫렸을 때, 내부의 세그멘테이션, EDR 탐지, 계정 관리 정책이 공격자를 얼마나 지연시키고 차단할 수 있는지를 확인하는 것이 목표다.

- **📢 섹션 요약 비유**: 성벽(경계)이 무너졌을 때 성 안의 좁은 복도와 함정(내부 보안)이 적군을 얼마나 잘 막아내는지 확인하는 훈련이다.

## Ⅲ. 비교 및 연결

가정 침투는 전통적인 침투 테스트와 목표 지점이 다르다.

| 항목 | 전통적 침투 테스트 (Full Chain) | 가정 침투 (Assumed Breach) |
|:---|:---|:---|
| 시작 지점 | 외부 (Untrusted) | 내부 (Internal/Compromised) |
| 중점 사항 | 침투 경로 발견 | 침해 탐지 및 대응 역량 |
| 시간 효율성 | 낮음 (침투에 시간 소요) | 높음 (핵심 영역 집중) |
| 성과 지표 | 침투 성공 여부 | 공격자 행위 탐지까지의 시간 |

이 모델은 '지능형 지속 위협(APT)' 대응과 'EDR/XDR' 성능 검증에 가장 최적화된 방식이다.

- **📢 섹션 요약 비유**: 전통적 테스트가 '성문에 구멍이 있는지 찾는 것'이라면, 가정 침투는 '성 안에서 스파이를 잡아내는 것'이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사 관점에서 가정 침투는 '리스크 기반 보안'의 실현이다. 모든 것을 막을 수 없다는 비관적 전제가 아니라, '최악의 상황에서도 비즈니스를 지켜낸다'는 현실적인 복원력(Resilience) 전략으로 접근해야 한다.

### 체크리스트
1. 공격자에게 부여한 초기 권한 수준이 실제 발생 가능한 시나리오(예: 신입 사원 PC)와 일치하는가?
2. 내부망 이동 탐지를 위한 '데셉션(Deception)' 기술이나 허니팟이 배치되어 있는가?
3. 훈련 결과 도출된 '탐지 사각지대'가 네트워크 세그멘테이션 정책에 반영되는가?

### 안티패턴
- 외부에서 뚫리지 않았으니 우리는 안전하다고 자만하며 내부 보안 훈련을 소홀히 하는 경우
- 가정 침투 훈련 중 관리자 권한을 너무 쉽게 주어 훈련의 긴장감을 떨어뜨리는 경우

- **📢 섹션 요약 비유**: 가정 침투는 '도어락'만 믿지 말고 '방집기'와 '금고'를 튼튼히 하는 지혜로운 집주인의 마음가짐이다.

---

## Ⅴ. 기대효과 및 결론

가정 침투 모델을 도입하면 조직은 공격자가 내부에서 활동할 수 있는 시간(Dwell Time)을 획기적으로 줄일 수 있다. 이는 곧 침해 사고 발생 시 피해 규모를 결정짓는 핵심 요인이 된다.

결론적으로 가정 침투는 보안의 패러다임을 '완벽한 차단'에서 '신속한 탐지와 격리'로 바꾸는 혁신적인 접근법이다. 뚫려도 죽지 않는 강한 체질을 만드는 것이 현대 보안의 정답이다.

- **📢 섹션 요약 비유**: 예방 주사를 맞는 것은 병균이 들어올 것을 가정하고 몸의 면역 체계(블루팀)를 훈련시키는 최고의 가정 침투 전략이다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| Zero Trust | "아무도 믿지 마라"는 가정 침투의 철학적 기반 |
| Dwell Time | 공격자가 침투 후 발견될 때까지 내부에 머무는 시간 |
| Lateral Movement | 가정 침투 훈련에서 가장 중요하게 다루는 내부 이동 행위 |
| Micro-segmentation | 침투가 발생해도 옆으로 퍼지지 못하게 하는 핵심 방어 기술 |

### 👶 어린이를 위한 3줄 비유 설명

1. 도둑이 이미 집 안으로 들어왔다고 생각하고, 우리 집 강아지와 경보기가 얼마나 빨리 도둑을 찾아내는지 시험하는 거예요.
2. 대문이 열려 있어도 보물상자만큼은 뺏기지 않도록 방 안의 함정을 점검하는 것과 같아요.
3. 이 연습을 하면 진짜 도둑이 들어오더라도 보물을 훔치기 전에 금방 들통나게 된답니다.
