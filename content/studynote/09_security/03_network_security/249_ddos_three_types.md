## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DDoS (Distributed Denial of Service) 공격은 목표 자원을 고갈시키는 계층에 따라 볼류메트릭(Volumetric)·프로토콜(Protocol)·애플리케이션(Application) 3유형으로 구분된다.
> 2. **가치**: 유형을 정확히 분류해야 ISP 스크러빙, 방화벽 튜닝, WAF (Web Application Firewall) 중 올바른 방어 레이어를 선택할 수 있다.
> 3. **판단 포인트**: 공격 트래픽이 대역폭을 포화시키는지(볼류메트릭), 상태 테이블을 고갈시키는지(프로토콜), 정상 요청처럼 서버 자원을 소모시키는지(애플리케이션)를 먼저 확인한다.

---

## Ⅰ. 개요 및 필요성

DDoS (Distributed Denial of Service, 분산 서비스 거부) 공격은 수백~수만 대의 봇넷(Botnet)을 동원해 단일 표적의 서비스 가용성을 파괴하는 공격이다. 단순히 "트래픽이 많다"는 현상만으로는 적절한 방어를 수립하기 어렵다. 공격자가 노리는 병목 자원이 무엇인지—대역폭, 상태 테이블, CPU—를 파악해야 비로소 효과적인 대응이 가능하다.

ICSA (Internet Crime Stoppers Association) 및 Cloudflare 통계에 따르면 2024년 기준 전체 DDoS 중 볼류메트릭이 약 60%, 애플리케이션 계층이 약 25%, 프로토콜 계층이 약 15%를 차지한다. 클라우드 인프라가 일반화되면서 대규모 볼류메트릭 공격보다 탐지가 어려운 애플리케이션 계층 공격 비중이 꾸준히 늘고 있다.

KISA (Korea Internet & Security Agency) DDoS 대응 가이드라인은 공격 유형별로 대응 체계를 분리하여 수립할 것을 권고한다. 한 레이어에만 집중한 방어는 다른 유형의 공격에 속수무책이 되기 때문이다.

📢 **섹션 요약 비유**: 식당이 문을 닫게 만들려면 문 앞에 사람을 가득 채워 입장을 막거나(볼류메트릭), 주방 예약 시스템을 마비시키거나(프로토콜), 주문만 넣고 받지 않아 요리사를 지치게 하는(애플리케이션) 3가지 방법이 있다.

## Ⅱ. 아키텍처 및 핵심 원리

### 3유형 비교 개요

| 구분 | 볼류메트릭 (Volumetric) | 프로토콜 (Protocol) | 애플리케이션 (Application) |
|:---|:---|:---|:---|
| OSI 계층 | 3~4 (네트워크/전송) | 3~4 (네트워크/전송) | 7 (응용) |
| 고갈 자원 | 대역폭 (bps) | 상태 테이블, PPS | CPU, 메모리, DB 커넥션 |
| 대표 공격 | UDP 플러드, ICMP 플러드, DNS Amplification | SYN 플러드, SMURF, Ping of Death | HTTP(S) 플러드, Slowloris, CC 공격 |
| 측정 단위 | Gbps ~ Tbps | Mpps (Million Packets per Second) | RPS (Requests per Second) |
| 주요 방어 | ISP 스크러빙, BGP Anycast | SYN Cookie, 방화벽 상태 테이블 튜닝 | WAF, Rate Limiting, CAPTCHA |

### 공격 흐름 다이어그램

```
[Botnet C2]
     │
     ├──▶ 유형 1: 볼류메트릭 ─────────────────────────────────────────┐
     │        UDP/ICMP 대량 트래픽 → 업스트림 링크 포화               │
     │        Amplification: 소량 쿼리 → 반사 서버 → 대량 응답 증폭   │
     │                                                                 │
     ├──▶ 유형 2: 프로토콜 ────────────────────────────────────────────┤
     │        SYN 플러드: SYN만 전송, ACK 없음                         │
     │        → 방화벽·LB (Load Balancer) 상태 테이블 고갈            │
     │                                                                 │
     └──▶ 유형 3: 애플리케이션 ───────────────────────────────────────┘
              정상 HTTP 요청처럼 보이는 대량 GET/POST
              Slowloris: 헤더를 천천히 전송 → 연결 점유
              → 웹 서버 Worker Thread 고갈
                        │
                        ▼
              [표적 서버/네트워크 서비스 불가]
```

### 각 유형 상세 메커니즘

**① 볼류메트릭 공격** — DNS (Domain Name System) Amplification이 대표적이다. 공격자는 피해자 IP를 출발지로 위조한 소량의 DNS 쿼리를 공개 리졸버(Open Resolver)에 전송한다. 리졸버는 피해자에게 쿼리의 수십~수백 배 크기의 응답을 돌려보낸다. 증폭 배율(Amplification Factor)이 클수록 공격 효율이 높다.

**② 프로토콜 공격** — SYN (Synchronize) 플러드는 TCP (Transmission Control Protocol) 3-way 핸드셰이크의 불완전 상태를 노린다. 서버는 SYN-ACK를 보내고 ACK를 기다리는 반개방(Half-Open) 연결을 상태 테이블에 유지한다. 이 테이블이 가득 차면 정상 사용자도 연결할 수 없다.

**③ 애플리케이션 공격** — CC (Challenge Collapsar) 공격은 정상 HTTP 요청 패턴을 모방하므로 패킷 자체로는 악성 여부 판단이 어렵다. Slowloris는 HTTP 헤더를 극도로 느리게 전송해 서버 연결 소켓을 점유한다. 방화벽은 정상 트래픽으로 판단하는 경우가 많다.

📢 **섹션 요약 비유**: 볼류메트릭은 고속도로에 차를 꽉 채우는 것, 프로토콜은 톨게이트 장부를 가득 채우는 것, 애플리케이션은 계산대 앞에서 잔돈을 세는 척 서 있는 것이다.

## Ⅲ. 비교 및 연결

| 비교 항목 | 볼류메트릭 | 프로토콜 | 애플리케이션 |
|:---|:---|:---|:---|
| 탐지 난이도 | 낮음 (대역폭 급증) | 중간 (상태 테이블 모니터링) | 높음 (정상 요청과 유사) |
| 방어 위치 | 업스트림 ISP, 스크러빙 센터 | 방화벽, L4 스위치 | WAF, 애플리케이션 서버 |
| 필요 장비 | 고대역폭 링크, BGP Anycast | SYN Cookie 지원 방화벽 | WAF, 행동 분석 엔진 |
| 비용 | 대규모 트래픽으로 방어 비용 高 | 중간 | 정밀 분석 필요로 방어 비용 高 |
| 대표 사례 | 2016 Mirai Botnet (1.2 Tbps) | 2013 Spamhaus SYN 플러드 | 2021 HTTP/2 Rapid Reset |

📢 **섹션 요약 비유**: 볼류메트릭은 홍수처럼 넘쳐서 눈에 확 보이지만, 애플리케이션 공격은 조용한 개미 떼처럼 안에서 갉아먹는다.

## Ⅳ. 실무 적용 및 기술사 판단

**단계적 방어 아키텍처** — 실무에서는 단일 솔루션으로 3가지 유형 모두를 방어하기 어렵기 때문에 계층별 방어(Defense in Depth)를 구성한다.

1. **ISP 스크러빙 센터**: 볼류메트릭 공격 트래픽을 업스트림에서 흡수. BGP (Border Gateway Protocol) 블랙홀 라우팅(RTBH, Remote Triggered Black Hole)으로 특정 목적지 IP 트래픽을 드롭.
2. **CDN (Content Delivery Network) + Anycast**: 볼류메트릭 트래픽을 전 세계 POP (Point of Presence)로 분산 흡수.
3. **방화벽 + IPS (Intrusion Prevention System)**: SYN Cookie 활성화, 상태 테이블 한계치 설정으로 프로토콜 공격 방어.
4. **WAF (Web Application Firewall)**: HTTP 요청 패턴 분석, Rate Limiting, 봇 탐지, CAPTCHA (Completely Automated Public Turing test to tell Computers and Humans Apart) 적용.
5. **애플리케이션 레이어 자체 방어**: 연결 타임아웃 단축, 요청별 리소스 쿼터, Async 처리로 Slowloris 영향 최소화.

**기술사 시험 포인트**:
- "DDoS 공격 유형을 계층별로 분류하고 각각의 방어 기술을 설명하라"는 단골 문제다.
- 볼류메트릭↔프로토콜 차이: 둘 다 L3/L4지만 볼류메트릭은 대역폭, 프로토콜은 상태 테이블이 병목이다.
- Amplification 공격의 증폭 배율: DNS 약 28배, NTP (Network Time Protocol) 556배, Memcached 51,000배.

📢 **섹션 요약 비유**: 집 앞에 사람이 몰리면 경찰(ISP), 현관문 잠금(방화벽), CCTV 분석(WAF) 3단계가 모두 있어야 집을 지킬 수 있다.

## Ⅴ. 기대효과 및 결론

DDoS 3유형의 명확한 분류는 방어 예산과 기술 투자를 올바른 레이어에 집중시킨다. 볼류메트릭 공격에 WAF만 투자하거나, 애플리케이션 공격에 대역폭만 늘리는 것은 자원 낭비다.

클라우드 네이티브 환경에서는 Auto Scaling이 일시적으로 애플리케이션 공격을 완화할 수 있으나, 비용 폭증이라는 새로운 리스크를 낳는다. 따라서 방어 설계는 "트래픽 차단"뿐 아니라 "비용 폭발 방지"도 포함해야 한다.

HTTP/2 및 HTTP/3 (QUIC) 환경에서는 멀티플렉싱 특성을 악용한 신종 애플리케이션 공격이 등장하고 있어 지속적인 방어 기술 업데이트가 필수적이다.

📢 **섹션 요약 비유**: 불이 나면 소화기(WAF)·스프링클러(방화벽)·소방차(ISP 스크러빙) 중 맞는 도구를 써야 한다. 불 종류도 모르고 소화기만 들고 있으면 기름불에 물을 끼얹는 실수를 한다.

### 👶 어린이를 위한 3줄 비유 설명
- 볼류메트릭은 친구 100명을 시켜 놀이공원 입구를 꽉 막아 아무도 못 들어가게 하는 것.
- 프로토콜은 놀이공원 예약 시스템 전화를 계속 걸어 전화기를 마비시키는 것.
- 애플리케이션은 놀이공원에 들어와서 매표소 앞에 줄만 서고 표를 안 사는 것.
