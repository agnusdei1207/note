## 핵심 인사이트 (3줄 요약)
> 1. **본질**: uRPF (Unicast Reverse Path Forwarding)는 수신 인터페이스에서 패킷 출발지 IP를 라우팅 테이블로 역방향 검증해 IP 스푸핑(Spoofing)을 차단하는 라우터 수준의 방어 기술이다.
> 2. **가치**: 소스 위조 트래픽이 네트워크 내부로 진입하는 것을 업스트림에서 막아 DDoS Amplification·반사 공격의 연료를 차단한다.
> 3. **판단 포인트**: Strict 모드는 단일 경로 환경에 최적이고, Loose 모드는 비대칭 라우팅 환경에서 False Positive 없이 적용 가능하다.

---

## Ⅰ. 개요 및 필요성

IP 스푸핑(Spoofing)은 패킷의 출발지 IP를 위조해 추적을 어렵게 하거나 반사 공격(Reflection Attack)의 도구로 악용된다. DNS Amplification, NTP (Network Time Protocol) Amplification 등 대규모 볼류메트릭 DDoS 공격은 대부분 소스 IP 위조에 의존한다. 위조된 소스 IP를 가진 패킷이 네트워크에 유입되지 못하도록 막는 것이 근본적인 해결책이다.

uRPF는 IETF RFC 3704에 정의된 표준 기술로, 라우터의 CEF (Cisco Express Forwarding) 또는 동등한 FIB (Forwarding Information Base) 검색 엔진을 활용하여 추가 성능 부하 없이 소스 검증을 수행한다. 인터넷 서비스 제공자(ISP)와 기업 경계 라우터에 배포함으로써 BCP 38 (Best Current Practice 38) 권고 사항을 이행할 수 있다.

BCP 38은 "네트워크 진입 필터링(Network Ingress Filtering)"을 정의한 RFC 2827에 해당하며, 운영자가 자신의 네트워크에서 출발하는 트래픽의 소스 IP를 검증해야 한다고 명시한다. uRPF는 이 정책을 자동화된 방식으로 구현하는 핵심 메커니즘이다.

📢 **섹션 요약 비유**: 편지를 받을 때 보낸 사람 주소가 실제로 그 동네에 존재하는지 우편 지도로 확인한 뒤 없으면 반송하는 것이 uRPF다.

## Ⅱ. 아키텍처 및 핵심 원리

### Strict vs Loose 모드 비교

| 항목 | Strict 모드 | Loose 모드 |
|:---|:---|:---|
| 검증 조건 | 패킷 수신 인터페이스 == 역방향 경로 인터페이스 | 라우팅 테이블에 경로 존재 여부만 확인 |
| 비대칭 라우팅 | 지원 불가 (False Positive 발생) | 지원 가능 |
| 보안 강도 | 높음 | 중간 |
| 적용 위치 | 단일 경로 엣지, 액세스 링크 | 코어/피어링 링크, 비대칭 라우팅 환경 |
| False Positive | 높음 (비대칭 시) | 낮음 |

### 역방향 경로 검증 원리

```
[정상 패킷]                       [스푸핑 패킷]
  SRC: 203.0.113.5                  SRC: 1.2.3.4 (위조)
  수신 IF: eth0                     수신 IF: eth0

  FIB 조회: "203.0.113.5로         FIB 조회: "1.2.3.4로 가려면
  가려면 eth0 경유"                  eth1 경유" → 불일치!
          │                                 │
          ▼                                 ▼
   ┌─────────────┐                  ┌─────────────┐
   │  uRPF 통과  │                  │  uRPF 폐기  │
   └─────────────┘                  └──────────────┘

  [Strict 모드 동작 흐름]
  수신 패킷
    │
    ├── SRC IP로 FIB 역방향 조회
    │         │
    │         ├── 역방향 경로의 출력 IF == 수신 IF?
    │         │     YES → 전달 (Forward)
    │         │     NO  → 폐기 (Drop)
    │         │
    │         └── 경로 없음 → 폐기 (Drop)
    │
    └── [Loose 모드] 경로 존재 여부만 확인
              경로 있음 → 전달
              경로 없음 → 폐기
```

### BCP 38과의 관계

BCP 38 (RFC 2827)은 "내 네트워크에서 나가는 트래픽의 소스 IP가 내 주소 블록에 속하는지 확인하라"는 인터넷 운영 모범 사례다. uRPF는 이를 수신 방향에서 자동으로 구현한다. 모든 ISP가 BCP 38을 준수하면 소스 위조 기반 공격 트래픽이 인터넷에서 사실상 사라진다.

📢 **섹션 요약 비유**: 고속도로 진입로에서 번호판이 이 지역 번호대인지 확인하는 것이 Strict, 번호판이 전국 어딘가에 등록된 번호인지만 확인하는 것이 Loose다.

## Ⅲ. 비교 및 연결

| 비교 항목 | uRPF Strict | uRPF Loose | ACL (Access Control List) 기반 |
|:---|:---|:---|:---|
| 구성 복잡도 | 낮음 | 낮음 | 높음 (IP 범위 수동 관리) |
| 동적 업데이트 | 라우팅 테이블 자동 반영 | 자동 반영 | 수동 업데이트 필요 |
| 비대칭 라우팅 | 미지원 | 지원 | 지원 |
| 성능 영향 | CEF 활용으로 미미 | 미미 | 적음 |
| 정밀도 | 높음 | 낮음 | 매우 높음 |

📢 **섹션 요약 비유**: uRPF는 자동 주소 검증 시스템이고, ACL은 경비원이 직접 명단을 확인하는 방식이다. 명단 관리가 번거롭지만 정밀도는 ACL이 높다.

## Ⅳ. 실무 적용 및 기술사 판단

**배포 권장 위치**:
- **고객 액세스 링크 (Customer-facing Edge)**: Strict 모드로 고객 IP 블록 이외의 소스를 즉시 차단.
- **피어링/트랜짓 링크**: Loose 모드 적용. 비대칭 라우팅이 빈번하므로 Strict는 서비스 장애 위험.
- **데이터센터 서버 팜 진입**: Strict 모드로 내부 IP 위조 차단.

**설정 예시 (Cisco IOS)**:
```
interface GigabitEthernet0/0
  ip verify unicast source reachable-via rx   ! Strict 모드
! 또는
  ip verify unicast source reachable-via any  ! Loose 모드
```

**주의사항**:
- 멀티홈(Multi-homed) 환경에서 Strict 적용 시 트래픽이 들어온 인터페이스와 라우팅 경로가 달라 정상 패킷이 드롭될 수 있다.
- 기본 경로(Default Route, 0.0.0.0/0)가 있으면 Loose 모드에서 모든 소스가 검증 통과해 효과가 없다. 이 경우 ACL과 병행해야 한다.
- IPv6 환경에서는 `ipv6 verify unicast source reachable-via` 명령으로 동일 기능 적용.

**기술사 시험 포인트**:
- uRPF와 BCP 38의 관계를 묻는 문제가 자주 출제된다.
- Strict/Loose 차이를 비대칭 라우팅 맥락에서 설명해야 한다.
- DNS Amplification 방어 수단으로 uRPF를 언급하면 가점 요소가 된다.

📢 **섹션 요약 비유**: 회사 출입증 검사를 건물 입구에서 할지(Strict), 건물 단지 게이트에서만 할지(Loose)는 건물 구조(라우팅 대칭성)에 따라 결정해야 한다.

## Ⅴ. 기대효과 및 결론

uRPF는 라우터의 기존 FIB 인프라를 재활용하므로 추가 하드웨어 없이 소스 위조 트래픽을 대규모로 차단할 수 있다. ISP 레벨에서 전 세계적으로 uRPF + BCP 38을 준수한다면 반사·증폭 공격의 대부분이 원천 차단된다.

실제로 CAIDA (Center for Applied Internet Data Analysis) 연구에 따르면 인터넷 AS (Autonomous System)의 약 25%만이 BCP 38을 준수하며, 이 비율이 낮기 때문에 소스 위조 기반 공격이 여전히 유효하다. 따라서 uRPF 보급 확대는 인터넷 전체의 보안 수준을 높이는 공공재적 성격을 가진다.

📢 **섹션 요약 비유**: 모든 도로에 번호판 위조 감지 카메라를 달면 위조 차량이 도로를 달릴 수 없는 것처럼, ISP 모두가 uRPF를 켜면 위조 패킷이 인터넷을 돌아다닐 수 없다.

| BGP RTBH | 연관 기술 | 블랙홀 라우팅으로 볼류메트릭 차단 |

### 👶 어린이를 위한 3줄 비유 설명
- 편지를 보낼 때 보내는 사람 주소가 가짜인지 확인하려면, 그 주소로 답장이 가는 경로가 실제로 있는지 지도에서 확인하면 된다.
- 지도에 없는 주소에서 온 편지는 가짜니까 바로 버린다.
- 이렇게 하면 가짜 주소로 나쁜 짓을 하는 사람을 막을 수 있다.
