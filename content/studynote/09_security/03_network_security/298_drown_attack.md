## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DROWN(Decrypting RSA with Obsolete and Weakened eNcryption)은 폐기된 SSLv2(Secure Sockets Layer version 2) 서버를 RSA(Rivest–Shamir–Adleman) 복호화 오라클(Decryption Oracle)로 악용하여, 동일 RSA 키를 공유하는 최신 TLS 세션을 복호화하는 크로스 프로토콜 공격이다.
> 2. **가치**: 서버 자체가 SSLv2를 지원하지 않더라도, 같은 RSA 개인 키를 공유하는 다른 서버(이메일 서버, 보조 서버 등)가 SSLv2를 활성화하고 있으면 취약하다 — 이는 "내 서버만 패치했다"는 안일함의 위험을 보여준다.
> 3. **판단 포인트**: 방어의 핵심은 SSLv2의 완전 비활성화와 RSA 키 공유 최소화다. 특히 PFS(Perfect Forward Secrecy)를 지원하지 않는 RSA 키 교환 방식은 DROWN 취약성을 증폭시킨다.

---

## Ⅰ. 개요 및 필요성

2016년 3월 Nimrod Aviram, Sebastian Schinzel 등 국제 공동 연구팀이 공개한 DROWN은 SSLv2가 이미 폐기된 지 오래됐음에도, 수많은 서버가 여전히 활성화 상태로 유지하고 있음을 드러냈다. 발표 당시 전체 HTTPS 서버의 약 33%, 상위 100만 개 웹사이트 중 약 25%가 취약한 것으로 측정됐다.

공격의 핵심은 **크로스 프로토콜(Cross-Protocol)** 특성이다. 공격자는 피해자 브라우저와 안전한 TLS 서버 간의 세션 데이터를 가로챈다. 이 데이터를 SSLv2를 지원하는 서버(같은 RSA 키 공유)에 전송하여 RSA 복호화 연산을 수행하게 한다. SSLv2 서버를 "계산기"로 삼아 TLS 세션의 프리마스터 시크릿(Premaster Secret)을 복원한다.

이 공격이 특히 위험한 이유는 공격자가 **오프라인에서** 대량의 SSL 핸드셰이크를 기록해 두었다가, 나중에 SSLv2 서버에 질의를 반복해 일괄 복호화할 수 있다는 점이다.

📢 **섹션 요약 비유**: 어떤 은행의 신형 금고(TLS)는 아무리 강해도, 같은 마스터 키로 열리는 구형 창고(SSLv2)가 어딘가에 있으면 도둑이 창고를 통해 마스터 키 복사본을 만들어 신형 금고도 열 수 있다.

## Ⅱ. 아키텍처 및 핵심 원리

### 크로스 프로토콜 공격 흐름

```
[1단계] 피해자 TLS 세션 녹화
브라우저                  피해자 TLS 서버 (RSA 키 = K)
    │──── TLS ClientHello ──────────────────────▶│
    │◀─── TLS ServerHello + Certificate (K) ─────│
    │──── ClientKeyExchange (RSA 암호화 PMS) ────▶│
    │                                            │
    └── 공격자가 RSA 암호문 C 캡처 ──────────────┘

[2단계] SSLv2 오라클 질의
공격자                    SSLv2 서버 (같은 RSA 키 = K)
    │──── 변조된 SSLv2 ClientMasterKey ─────────▶│
    │     (C에서 파생된 RSA 암호문)               │
    │◀─── SSLv2 오류 메시지 (내용이 오라클) ──────│
    │                                            │
    반복 수백만 회 → PMS 복원

[3단계] 세션 복호화
    PMS 복원 → 마스터 시크릿 계산 → TLS 세션 키 도출
    → 녹화된 TLS 트래픽 전체 복호화 완료
```

### 공격 유형

| 변형 | 설명 | 계산 비용 |
|:---|:---|:---|
| General DROWN | SSLv2 40비트 EXPORT 암호 사용 | 중간 (클라우드 $440/세션) |
| Special DROWN | 특수 SSLv2 버그(Leaky Export) 악용 | 매우 낮음 (즉시 복호화 가능) |

**Special DROWN**: 일부 SSLv2 구현에서 EXPORT 암호를 사용할 때 키의 일부가 평문으로 노출되는 "Leaky Export" 버그까지 겹치면, 단 1회의 핸드셰이크로도 복호화가 가능했다.

### RSA 키 공유 취약 시나리오

```
HTTPS 서버 (SSLv2 비활성화)
     │
     ├── SMTP 서버 (같은 RSA 인증서, SSLv2 활성화) ← 오라클
     ├── FTP-SSL 서버 (같은 RSA 인증서, SSLv2 활성화) ← 오라클
     └── 보조 HTTPS 서버 (같은 RSA 인증서, SSLv2 활성화) ← 오라클

→ HTTPS 서버가 아무리 잘 설정돼도, 같은 키를 공유하는 서버 중 하나라도 SSLv2를 쓰면 취약!
```

📢 **섹션 요약 비유**: 본사(TLS) 금고는 최신형이지만, 지사(SSLv2) 창고가 낡은 자물쇠를 쓰고 있고 같은 마스터 키로 연결되어 있다. 도둑은 본사가 아닌 지사 창고를 노린다.

## Ⅲ. 비교 및 연결

### SSLv2 기반 취약점 비교

| 항목 | DROWN | POODLE |
|:---|:---|:---|
| 악용 프로토콜 | SSLv2 | SSL 3.0 |
| 공격 유형 | 크로스 프로토콜 RSA 오라클 | 패딩 오라클 |
| 대상 세션 | 현재 TLS 세션의 RSA 암호문 | 동일 세션 |
| MITM 필요 | ✅ (트래픽 캡처 필요) | ✅ |
| 비동기 공격 | ✅ (녹화 후 오프라인 공격 가능) | ❌ |
| 공격 범위 | 타 서버 RSA 키 공유 포함 | 해당 서버만 |
| 방어 | SSLv2 완전 비활성화 | SSL 3.0 비활성화 |

📢 **섹션 요약 비유**: POODLE은 같은 집 안에서 낡은 문을 이용하는 것이고, DROWN은 이웃집(SSLv2 서버)의 낡은 문을 통해 본집(TLS 서버)의 비밀을 알아내는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

**방어 체크리스트**

1. **SSLv2 전면 비활성화**:
   - OpenSSL: `SSL_OP_NO_SSLv2` 플래그 설정
   - nginx: `ssl_protocols TLSv1.2 TLSv1.3;`
   - Apache: `SSLProtocol all -SSLv2 -SSLv3 -TLSv1 -TLSv1.1`

2. **RSA 키 공유 감사**: 동일 인증서/키를 사용하는 모든 서비스(SMTP, IMAP, FTP, VPN 등)를 인벤토리화하고 각각 SSLv2 비활성화 확인

3. **EXPORT 암호 스위트 비활성화**: DROWN, FREAK, Logjam 공통 방어
   - `!EXPORT !eNULL !aNULL !RC4` 암호 스위트 제외

4. **PFS 지원 암호 스위트 우선**: RSA 키 교환 대신 ECDHE/DHE 사용. RSA 개인 키가 유출돼도 과거 세션 복호화 불가

**기술사 논술 포인트**: DROWN 분석 시 ① 크로스 프로토콜 공격의 개념, ② SSLv2 오라클 메커니즘, ③ RSA 키 공유의 위험성, ④ PFS 부재 시 소급 복호화 가능성을 차례로 논한다.

📢 **섹션 요약 비유**: 한 사람이 여러 자물쇠에 같은 마스터 키를 쓴다면, 가장 낡은 자물쇠 하나만 뚫려도 다른 모든 자물쇠가 위험해진다. 각 문마다 다른 키를 쓰거나(키 분리), 오래된 자물쇠를 없애야(SSLv2 비활성화) 한다.

## Ⅴ. 기대효과 및 결론

DROWN이 남긴 핵심 교훈은 **레거시 프로토콜의 잔존 위험**이다. SSLv2는 2011년 RFC 6176으로 이미 공식 폐기됐지만, 여전히 수백만 대 서버에서 활성화 상태였다. 취약한 기능을 "사용하지 않는다"는 것과 "비활성화한다"는 것은 전혀 다른 보안 수준이다.

또한 DROWN은 보안 감사 범위를 확장해야 한다는 교훈을 준다. 단일 서버의 설정이 아니라, **동일 인증서·키를 공유하는 인프라 전체**가 감사 단위가 돼야 한다. 이 원칙은 인증서 수명 주기 관리(CLM, Certificate Lifecycle Management)의 중요성으로 이어진다.

📢 **섹션 요약 비유**: 방화 설비를 강화해도, 같은 수도관을 공유하는 옆 건물에 노후 밸브가 있으면 화재 때 함께 위험해진다. DROWN은 "내 건물"이 아닌 "공유 인프라" 전체를 봐야 한다는 것을 가르쳐 줬다.

### 👶 어린이를 위한 3줄 비유 설명

1. DROWN은 새 집 자물쇠가 아무리 강해도, 같은 마스터 키로 열리는 낡은 창고가 어딘가에 있으면 창고 자물쇠를 통해 마스터 키를 알아낼 수 있는 공격이에요.
2. 도둑은 낡은 창고(SSLv2)에 질문을 수백만 번 해서 마스터 키(RSA 개인 키)를 천천히 파악해요.
3. 해결책은 낡은 창고(SSLv2)를 완전히 잠가버리거나, 각 문마다 다른 열쇠를 쓰는 것(PFS)이에요.
