## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SASE (Secure Access Service Edge)는 Gartner가 2019년 정의한 아키텍처 프레임워크로, 네트워킹(SD-WAN)과 보안(SWG·CASB·ZTNA·FWaaS) 기능을 클라우드 엣지에서 단일 서비스로 통합하여 제공한다.
> 2. **가치**: 사용자가 어디에 있든(본사·재택·카페) 트래픽이 클라우드 엣지를 통과하므로 일관된 보안 정책 적용이 가능하며, 기업 데이터센터로 백홀링(Backhauling)하는 MPLS 비용을 대폭 절감한다.
> 3. **판단 포인트**: 단일 벤더 완전 통합(Cisco+, Palo Alto Prisma) vs 멀티벤더 최적 조합(Best-of-Breed) 전략은 관리 단순성과 기능 최적화의 트레이드오프이며, 기업 규모·기존 투자에 따라 선택해야 한다.

---

## Ⅰ. 개요 및 필요성

2010년대 클라우드 전환이 가속화되면서 기업의 트래픽 패턴이 근본적으로 변했다. 전통적 아키텍처에서는 본사 데이터센터에 방화벽·프록시·DLP(Data Loss Prevention)가 집중되어 있어, 지사나 원격 직원의 모든 트래픽이 MPLS 전용선으로 본사를 거쳐 인터넷에 나갔다(트롬본 라우팅). 클라우드 SaaS (Software as a Service)가 주류가 된 환경에서 이 구조는 지연시간 급증과 MPLS 비용 폭등을 유발한다.

SASE는 보안 기능을 데이터센터에서 분리하여 전 세계 클라우드 POP (Point of Presence)로 분산시킨다. 사용자는 가장 가까운 POP으로 연결하고, 그 POP에서 SWG·CASB·ZTNA 등 보안 검사를 받은 뒤 인터넷 또는 기업 리소스에 접근한다. 물리적 위치에 무관하게 동일한 보안 정책이 적용된다.

📢 **섹션 요약 비유**: 전통적 아키텍처가 "모든 택배가 서울 본사 창고를 거쳐 배송되는 시스템"이라면, SASE는 "전국 주요 도시에 검색대가 있어 가장 가까운 곳에서 검사 후 바로 배송되는 시스템"이다.

## Ⅱ. 아키텍처 및 핵심 원리

### SASE 5대 구성 요소

```
┌─────────────────────────────────────────────────────────────┐
│                  SASE 클라우드 엣지 (POP)                    │
│                                                             │
│  ┌───────────┐  ┌───────────┐  ┌───────────┐               │
│  │ SD-WAN    │  │   SWG     │  │   CASB    │               │
│  │(네트워크) │  │(웹보안)   │  │(클라우드) │               │
│  └───────────┘  └───────────┘  └───────────┘               │
│  ┌───────────┐  ┌───────────┐                               │
│  │  FWaaS    │  │   ZTNA    │                               │
│  │(클라우드  │  │(Zero Trust│                               │
│  │ 방화벽)   │  │ 접근제어) │                               │
│  └───────────┘  └───────────┘                               │
└─────────────────────────────────────────────────────────────┘
         ↑                    ↑                    ↑
   [지사 SD-WAN]       [원격 직원]           [모바일 사용자]
```

### 5대 구성 요소 상세

| 구성 요소 | 풀 네임 | 역할 |
|:---|:---|:---|
| SD-WAN | Software-Defined Wide Area Network | WAN 최적화, 트래픽 분류, 링크 다중화 |
| SWG | Secure Web Gateway | URL 필터링, SSL 검사, 악성코드 탐지 |
| CASB | Cloud Access Security Broker | SaaS 앱 가시성, DLP, 섀도 IT 탐지 |
| FWaaS | Firewall as a Service | L4-L7 클라우드 방화벽, IPS/IDS |
| ZTNA | Zero Trust Network Access | 최소 권한 기반 애플리케이션 접근 제어 |

### SASE vs 기존 아키텍처 비교

```
[기존 MPLS + 온프레미스 보안]
지사 ─── MPLS ───→ 본사 방화벽/프록시 ──→ 인터넷/클라우드
                   (모든 트래픽 백홀링)
                   지연: +30~100ms

[SASE 아키텍처]
지사 ─── SD-WAN → SASE POP(가장 가까운) ──→ 인터넷/클라우드
         (직접 인터넷 분리, 로컬 검사)
         지연: 최소화
```

📢 **섹션 요약 비유**: SWG는 "학교 입구 선생님(위험 웹 차단)", CASB는 "학생 SNS 감시자(클라우드 앱 관리)", FWaaS는 "담장(악성 트래픽 차단)", ZTNA는 "교실마다 다른 열쇠(앱별 접근 제어)", SD-WAN은 "효율적인 통학 버스 배차(네트워크 최적화)"다.

## Ⅲ. 비교 및 연결

| 항목 | 전통 아키텍처 (MPLS+방화벽) | SASE |
|:---|:---|:---|
| 보안 기능 위치 | 데이터센터 집중 | 클라우드 분산(엣지) |
| 원격 접속 | VPN → 백홀링 | ZTNA → 로컬 분기 |
| 클라우드 트래픽 | 백홀 → 데이터센터 경유 | 직접 SaaS 접근 |
| 비용 구조 | CAPEX (장비) + MPLS 회선 | OPEX (구독) |
| 정책 관리 | 지점별 분산 | 중앙 단일 콘솔 |
| 가시성 | 온프레미스 트래픽만 | 모든 엣지 트래픽 |
| 확장성 | 물리 장비 확장 필요 | 클라우드 탄력적 확장 |
| 핵심 벤더 | Cisco, Fortinet | Zscaler, Palo Alto, Cato |

📢 **섹션 요약 비유**: 전통 아키텍처는 "한 도시에 경찰서 하나", SASE는 "모든 동네에 순찰대 배치"다. 어디서 사고가 나든 즉시 대응할 수 있고, 중앙 경찰서(데이터센터)가 혼잡해질 일도 없다.

## Ⅳ. 실무 적용 및 기술사 판단

**도입 단계별 로드맵:**

1. **1단계 - SD-WAN 전환**: MPLS를 인터넷+SD-WAN으로 대체하여 WAN 비용 절감. 보안 기능은 기존 유지하며 네트워크만 현대화.

2. **2단계 - SWG+CASB 클라우드화**: 온프레미스 웹 프록시와 CASB를 클라우드로 이전. 원격 직원이 증가한 환경에서 즉각적 효과.

3. **3단계 - ZTNA 도입**: 레거시 VPN을 ZTNA로 대체. 애플리케이션 단위 접근 제어로 내부망 과도한 접근 제한.

4. **4단계 - FWaaS 통합**: 지점 방화벽을 FWaaS로 교체. 하드웨어 생명주기 관리 부담 제거.

5. **단일 벤더 vs 멀티벤더**: Zscaler(ZIA+ZPA), Palo Alto Prisma, Cato Networks 등 단일 벤더는 통합 관리 편의성. Cisco SD-WAN + Zscaler ZIA 같은 멀티벤더는 기능 최적화.

**주의사항:**
- SaaS 트래픽에 TLS 검사 적용 시 인증서 신뢰 체인 관리 필요
- 일부 국가에서 클라우드 POP 가용성 확인 필요 (Data Residency 요구)
- SASE로의 전환은 최소 12~24개월 프로젝트로 단계적 접근 필수

📢 **섹션 요약 비유**: SASE 도입은 "전통 백화점에서 온라인 쇼핑몰로 전환"과 같다. 한번에 모든 것을 바꾸기보다 물류센터 현대화(SD-WAN) → 앱 런칭(SWG/CASB) → 결제 시스템 교체(ZTNA) 순으로 단계적으로 전환해야 리스크가 낮다.

## Ⅴ. 기대효과 및 결론

SASE 도입으로 기업은 WAN 비용 30~50% 절감, 보안 정책 관리 단일화, 원격 근무 사용자 경험 개선이라는 세 가지 핵심 가치를 얻는다. 특히 COVID-19 이후 원격 근무가 정착된 환경에서 SASE는 선택이 아닌 필수 아키텍처로 자리잡고 있다.

기술사는 SASE를 기술 솔루션이 아닌 아키텍처 원칙으로 이해해야 한다. 모든 기능을 한 번에 도입하는 것보다 기업의 가장 큰 페인포인트(WAN 비용 vs 원격 접속 보안 vs 클라우드 가시성)를 먼저 파악하고, 그에 맞는 구성 요소부터 단계적으로 도입하는 전략이 성공의 핵심이다.

📢 **섹션 요약 비유**: SASE는 "건강검진 종합패키지"다. 혈액검사(SWG), X선(FWaaS), 내시경(CASB), 심전도(ZTNA)를 한 곳에서 받으면 시간도 절약되고 결과를 통합 분석할 수 있다. 단, 어느 것을 먼저 받을지 의사(기술사)의 처방이 필요하다.

### 👶 어린이를 위한 3줄 비유 설명
1. SASE는 학교 건물(본사 데이터센터) 안에서만 검색대를 두는 대신, 학교 버스 안(클라우드 엣지)에서 가방 검사를 하는 방식이야.
2. 어디에 있든 버스를 타면(SASE POP 연결) 위험한 물건은 못 들어오고, 허락된 친구들(ZTNA)만 교실에 입장할 수 있어.
3. 전 세계 어디에 있어도 같은 규칙이 적용되니까, 재택근무하는 친구도 학교 안에 있는 것처럼 안전하게 공부할 수 있어.
