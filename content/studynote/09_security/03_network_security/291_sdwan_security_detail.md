## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SD-WAN 보안의 세부 구현은 오버레이 터널 암호화(IPsec/TLS), 애플리케이션 인식 기반 PBR (Policy-Based Routing), 마이크로 세그멘테이션(Micro-Segmentation) 세 축이 맞물려 동작하며, 이 세 요소가 모두 갖춰져야 진정한 제로 트러스트 WAN이 완성된다.
> 2. **가치**: 트래픽 분류를 통한 PBR은 중요한 업무 트래픽은 최적 경로로, 일반 인터넷 트래픽은 직접 분기하여 비용과 성능을 동시에 최적화하면서 보안 정책을 차별화 적용한다.
> 3. **판단 포인트**: 마이크로 세그멘테이션은 지사 내부에서 IoT, 게스트, 업무 네트워크를 분리하여 랜섬웨어 등 악성코드의 횡적 이동(Lateral Movement)을 차단하는 현대 보안의 핵심 레이어다.

---

## Ⅰ. 개요 및 필요성

SD-WAN의 보안 세부 구현을 논할 때 세 가지 핵심 질문을 답해야 한다. 첫째, WAN 트래픽은 어떻게 암호화되는가? 둘째, 트래픽은 어떤 기준으로 경로가 결정되는가? 셋째, 같은 지사 내 디바이스들은 서로를 얼마나 신뢰하는가?

암호화 측면에서 SD-WAN은 기본적으로 IPsec 오버레이를 사용하지만, 일부 벤더는 TLS 기반 오버레이를 사용하기도 한다. 중요한 것은 단순히 암호화가 '있다'는 것이 아니라, 암호화 키 관리, 인증 방식, 재키잉 주기가 보안 정책 수준에 맞게 설정되어 있는지다.

PBR은 전통적 라우팅의 목적지 기반 결정과 달리, 출발지, 애플리케이션, 사용자 신원, 서비스 수준까지 고려하여 경로를 동적으로 선택한다. 예를 들어 Salesforce(CRM) 트래픽은 직접 인터넷으로, SAP(ERP) 트래픽은 MPLS로, 기타 트래픽은 인터넷 VPN으로 분리하는 세밀한 정책이 가능하다.

마이크로 세그멘테이션은 "한 번 접속에 성공하면 내부망 전체를 이동할 수 있다"는 전통적 보안 모델의 약점을 보완한다. 지사 내부를 업무망/IoT망/게스트망으로 분리하고, 세그먼트 간 통신을 방화벽 정책으로 엄격히 제한한다.

📢 **섹션 요약 비유**: SD-WAN 보안 세부는 "물류 창고 운영"과 같다. 암호화는 "잠긴 화물 컨테이너", PBR은 "화물 종류별 배송 경로 지정", 마이크로 세그멘테이션은 "창고 내부 구역 구분"이다. 세 가지가 모두 있어야 완벽한 물류 보안이 된다.

## Ⅱ. 아키텍처 및 핵심 원리

### 오버레이 터널 암호화 구조

```
지사 A vCPE                                  지사 B vCPE
┌────────────────┐   IPsec/TLS 터널   ┌────────────────┐
│  애플리케이션   │                    │  애플리케이션   │
│  인식 엔진      │                    │  인식 엔진      │
│  ─────────────  │◄── AES-256-GCM ──►│  ─────────────  │
│  PBR 정책 엔진 │     SHA-384 HMAC   │  PBR 정책 엔진 │
│  ─────────────  │                    │  ─────────────  │
│  세그멘테이션   │                    │  세그멘테이션   │
│  엔진(VLAN/VRF)│                    │  엔진(VLAN/VRF)│
└────────────────┘                    └────────────────┘
    │   │   │                               │   │   │
 업무 IoT 게스트                          업무 IoT 게스트
 VLAN VLAN VLAN                          VLAN VLAN VLAN
```

### PBR (Policy-Based Routing) 정책 예시

| 트래픽 유형 | 애플리케이션 인식 | 전송 경로 | 보안 처리 |
|:---|:---|:---|:---|
| ERP (SAP, Oracle) | DPI 시그니처 | MPLS 전용선 | IPsec + QoS 보장 |
| 화상회의 (Zoom, Teams) | DPI + 포트 | 인터넷 직접 | IPsec + 낮은 지연 링크 |
| SaaS (M365, Salesforce) | FQDN + IP | DIA → SSE SWG | TLS 검사 |
| 인터넷 일반 | 미분류 | DIA → NGFW | URL 필터링 |
| 기업 DC 접근 | RFC1918 목적지 | IPsec VPN | AES-256 |
| 게스트 트래픽 | 게스트 VLAN | DIA 격리 | 기업망 접근 차단 |

### 마이크로 세그멘테이션 구현

```
지사 내부 네트워크
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  ┌─────────────────┐   방화벽 정책 (VRF/VLAN 간 제어)   │
│  │   업무망         │──── 엄격한 허용 목록 ────→         │
│  │  10.1.1.0/24    │   기업 서버만 접근 가능             │
│  └─────────────────┘                                    │
│           ↕ 차단                                        │
│  ┌─────────────────┐                                    │
│  │   IoT망          │──── 인터넷만 허용 ────→            │
│  │  10.1.2.0/24    │   업무망·게스트망 접근 차단         │
│  └─────────────────┘                                    │
│           ↕ 차단                                        │
│  ┌─────────────────┐                                    │
│  │   게스트망       │──── 인터넷만 허용 ────→            │
│  │  192.168.100/24 │   기업 리소스 완전 차단             │
│  └─────────────────┘                                    │
└─────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 마이크로 세그멘테이션은 "빌딩 내 방화 구획"과 같다. 한 층에서 불이 나도 방화문이 닫혀 다른 층으로 번지지 않도록, 네트워크 구역 간 악성코드가 번지지 않게 격리한다.

## Ⅲ. 비교 및 연결

| 항목 | 전통 WAN 보안 | SD-WAN 보안 세부 |
|:---|:---|:---|
| 암호화 단위 | 전체 터널 (단일 키) | 애플리케이션별 차별화 가능 |
| 라우팅 기준 | 목적지 IP만 | 앱+사용자+디바이스+SLA |
| 세그멘테이션 | VLAN 물리 분리 | VRF/VLAN 소프트웨어 정의 |
| 정책 배포 | 장비별 CLI | 중앙 컨트롤러 일괄 배포 |
| 가시성 | NetFlow 제한 | DPI 기반 앱 인식 + 통합 로그 |
| Zero Trust 연계 | 어려움 | ✅ 사용자 신원 기반 PBR |
| 랜섬웨어 차단 | 어려움 (플랫 네트워크) | ✅ 세그멘테이션으로 횡적 이동 차단 |

📢 **섹션 요약 비유**: 전통 WAN 보안이 "하나의 큰 자물쇠"라면, SD-WAN 보안 세부는 "모든 방마다 다른 열쇠와 CCTV, 그리고 방화문"이다. 세밀하게 나누면 나눌수록 피해가 한 방에 국한된다.

## Ⅳ. 실무 적용 및 기술사 판단

**암호화 설계 세부 지침:**

1. **키 관리 자동화**: SD-WAN 컨트롤러가 PKI 기반으로 각 엣지 장비의 인증서를 자동 발급·갱신. 인증서 만료로 인한 서비스 중단 방지.

2. **PFS (Perfect Forward Secrecy)**: IPsec 터널에서 DH 그룹 14 이상(2048비트 이상)을 사용하는 PFS 필수 활성화. 터널 키 노출 시 과거 트래픽 보호.

3. **Zero-Touch Provisioning (ZTP) 보안**: 신규 지사 장비가 자동 등록될 때 컨트롤러 인증 없이는 설정 수신 불가하도록 Secure ZTP 구현. 중간자 공격으로 가짜 설정 수신 위험 제거.

4. **PBR 예외 처리**: DPI가 새로운 애플리케이션을 인식하지 못할 경우 기본 경로(Failover Policy) 설정. 미분류 트래픽의 기본값은 '차단' 또는 '보안 검사 후 허용' 권장.

5. **세그멘테이션 레이블**: Cisco SD-WAN(vManage)의 SGT (Scalable Group Tag), VMware SD-WAN의 NSX 연계 등 벤더 고유의 세그멘테이션 레이블을 활용하면 IP 기반 정책보다 유연한 세그멘테이션 구현 가능.

6. **랜섬웨어 대응 시나리오**: 지사 PC가 감염될 경우, IoT망·업무망·게스트망 간 격리 정책으로 횡적 이동 차단. 감염 기기의 트래픽 급증(C2 통신, 파일 암호화 트래픽)을 DPI로 탐지하여 자동 격리.

📢 **섹션 요약 비유**: PFS 적용은 "회의 때마다 다른 암호를 쓰는 것"이다. 오늘 암호가 노출돼도 지난 회의 내용은 다른 암호로 보호되어 있어 과거 기록까지 노출되지 않는다.

## Ⅴ. 기대효과 및 결론

SD-WAN 보안 세부를 올바르게 구현하면, 기업은 WAN 성능 최적화와 보안 강화를 동시에 달성할 수 있다. 오버레이 암호화로 인터넷 WAN의 기밀성을 보장하고, PBR로 트래픽별 최적 경로를 선택하며, 마이크로 세그멘테이션으로 내부 위협의 확산을 차단한다.

기술사는 SD-WAN 설계 검토 시 "터널이 암호화되어 있는가"라는 1차적 질문을 넘어, "PBR 정책이 보안 요구사항을 반영하는가", "세그멘테이션이 랜섬웨어 횡적 이동을 차단할 수 있는가"라는 심층 질문을 던질 수 있어야 한다. 이것이 네트워크 엔지니어와 기술사의 차이다.

📢 **섹션 요약 비유**: SD-WAN 보안 세부 설계는 "은행 설계"와 같다. 금고 잠금장치(암호화)만 있는 게 아니라, 고객·직원·VIP 동선 분리(세그멘테이션), 거래 종류별 창구 분리(PBR), CCTV와 경보(가시성)까지 함께 설계되어야 진짜 안전한 은행이 된다.

### 📌 관련 개념 맵
| 개념 | 관계 | 설명 |
|:---|:---|:---|
| IPsec | 오버레이 암호화 | SD-WAN 터널 기밀성·무결성 제공 |
| PBR | 트래픽 분류 | 애플리케이션별 최적 경로 결정 |
| DPI | 트래픽 인식 | PBR을 위한 L7 애플리케이션 분류 |
| VRF | 세그멘테이션 기술 | 가상 라우팅 테이블로 논리 분리 |
| Micro-Segmentation | 보안 구현 | 네트워크 구역 간 횡적 이동 차단 |
| PFS | 암호 강화 | 키 노출 시 과거 트래픽 보호 |

### 👶 어린이를 위한 3줄 비유 설명
1. SD-WAN 암호화는 모든 편지(트래픽)를 종류에 따라 다른 봉투(암호화 키)에 넣고, 중요한 편지는 특별 배송(MPLS)으로 보내는 거야.
2. 마이크로 세그멘테이션은 교실마다 다른 열쇠가 있어서, 한 교실에서 문제가 생겨도 옆 교실은 안전하게 지키는 방화문이야.
3. 이 세 가지(암호화, 경로 정책, 구역 분리)가 모두 갖춰지면 아무리 나쁜 바이러스도 회사 전체를 못 건드려!
