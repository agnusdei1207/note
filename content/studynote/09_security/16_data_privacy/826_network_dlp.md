## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 DLP는 조직의 네트워크 경계(Gateway)에서 전송되는 데이터를 실시간으로 검사하여 외부로의 민감 정보 유출을 차단하는 보안 기술이다.
> 2. **가치**: 이메일(SMTP), 웹(HTTP/S), 파일 전송(FTP) 등 다양한 프로토콜에 대해 애플리케이션에 독립적인 통합 유출 통제력을 제공한다.
> 3. **판단 포인트**: 암호화된 트래픽(SSL/TLS)을 가시화할 수 있는 복호화 성능과 대용량 트래픽 처리 시의 지연 시간(Latency) 최소화가 핵심이다.

---

## Ⅰ. 개요 및 필요성

네트워크 DLP는 "움직이는 데이터(Data in Motion)"를 지키는 파수꾼이다. 직원이 실수로 민감한 첨부파일을 이메일로 보내거나, 악의적인 사용자가 웹 게시판에 기업 기밀을 올리는 행위를 네트워크 관문에서 원천 봉쇄한다.

개별 PC마다 에이전트를 설치해야 하는 엔드포인트 DLP와 달리, 네트워크 장목(Choke Point)에 한 번만 설치하면 망 전체의 유출 흐름을 파악할 수 있어 초기 도입 및 관리 효율성이 매우 높다.

- **📢 섹션 요약 비유**: 공항 터미널의 공통 검색대와 같다. 승객이 어떤 비행기(프로토콜)를 타든 상관없이 나가는 모든 짐을 한곳에서 검사하는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

네트워크 DLP는 배치 방식에 따라 탐지 위주(Sniffing)와 차단 위주(Inline)로 나뉜다.

### 1. 배치 모델 (Deployment Mode)
| 모드 | 동작 방식 | 장점 | 단점 |
|:---|:---|:---|:---|
| **Monitor (TAP/Mirror)** | 스위치 포트 미러링을 통해 패킷 복사본 분석 | 트래픽 지연 없음, 기존 망 영향 제로 | 실시간 차단 불가 (사후 대응) |
| **In-Line** | 네트워크 경로 중간에 직접 배치 (Proxy) | 실시간 유출 차단 가능 | 장애 시 네트워크 단절 위험 (Bypass 필수) |
| **ICAP (Integrated)** | 웹 프록시(WAF/Proxy)와 연동하여 분석 요청 | 기존 보안 장비와 협업 용이 | 연동 복잡도 증가 |

### 2. 네트워크 DLP 아키텍처 다이어그램
```text
      [ 내부망 ]                [ 보안 관문 ]               [ 외부망 (Internet) ]
┌────────────────────┐      ┌────────────────────┐      ┌────────────────────┐
│  사용자 PC (Email) ├──▶──┤  Network DLP       ├──▶──┤  Web / Mail Server │
│  사용자 PC (Web)   ├──▶──┤  (SSL Inspection)  ├──▶──┤  Cloud Storage     │
└────────────────────┘      └──────────┬─────────┘      └────────────────────┘
                                       │
                         [ Policy & Analysis Engine ]
                         - Regex, Fingerprinting
                         - File Decompression
```

### 3. 주요 분석 기술
- **SSL/TLS Inspection**: 암호화된 HTTPS 트래픽을 복호화하여 내부 콘텐츠를 검사.
- **프로토콜 파싱**: HTTP, SMTP, FTP, IMAP 등 다양한 통신 규약을 해석하여 데이터 추출.
- **파일 압축 해제**: 다중 압축 파일(Zip, 7z 등) 내부를 풀어서 실제 내용을 검사.

- **📢 섹션 요약 비유**: 고속도로 톨게이트에서 모든 차량(패킷)을 멈춰 세우고 짐칸(페이로드)을 확인한 뒤 통과시키는 시스템과 같다.

## Ⅲ. 비교 및 연결

네트워크 DLP는 엔드포인트 DLP와 상호 보완적으로 운영될 때 완벽한 방어가 가능하다.

| 항목 | 네트워크 DLP | 엔드포인트 DLP |
|:---|:---|:---|
| 통제 시점 | 네트워크 전송 시점 | 파일 접근, 매체 복사 시점 |
| 관리 대상 | 네트워크 패킷, 프로토콜 | PC 에이전트, 하드웨어 포트 |
| 암호화 대응 | SSL 복호화 장비 필요 | 암호화 전 평문 단계에서 검사 가능 |
| 사외망 통제 | 불가 (회사 밖에서는 작동 안 함) | 가능 (PC에 상주하며 통제) |
| 인프라 영향 | 네트워크 지연 가능성 | PC 성능 저하 가능성 |

- **📢 섹션 요약 비유**: 네트워크 DLP는 '국경 검문소'이고, 엔드포인트 DLP는 개인의 '가방 자물쇠'와 같다.

---

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 네트워크 DLP 운영의 최대 걸림돌은 **"대용량 트래픽 처리"**다. 특히 동영상 스트리밍이나 대용량 파일 전송이 잦은 환경에서는 분석 엔진이 병목 지점이 될 수 있다.

### 체크리스트
1. **SSL 가시성**: 조직 내 트래픽의 80% 이상인 HTTPS에 대해 복호화 정책이 수립되었는가?
2. **우회 경로 차단**: VPN이나 프록시 서버를 통해 네트워크 DLP를 우회하는 시도를 막을 수 있는가?
3. **가용성 보장**: 장비 장애 시 네트워크를 그대로 통과시키는 하드웨어 바이패스(Bypass) 기능이 있는가?

### 안티패턴
- 모니터링 모드(TAP)로만 운영하면서 유출 사고 발생 시 "탐지는 했지만 차단은 못 했다"고 보고하는 경우
- 모바일 기기(BYOD)가 사용하는 무선망에 대한 네트워크 DLP 적용을 누락하는 경우

- **📢 섹션 요약 비유**: 검문소를 만들었지만, 검문이 너무 오래 걸려 사람들이 옆길(VPN)로 돌아가거나 검문소 문을 그냥 열어두는 상황을 경계해야 한다.

---

## Ⅴ. 기대효과 및 결론

네트워크 DLP는 기업의 보안 정책을 전사적으로 강제하는 강력한 수단이다. 에이전트 설치가 불가능한 IoT 기기나 게스트 PC의 유출 행위까지 잡아낼 수 있다는 점이 큰 장점이다. 최근에는 클라우드 기반의 SASE(Secure Access Service Edge) 환경으로 확장되어 장소에 구애받지 않는 네트워크 통제로 진화하고 있다.

결론적으로 네트워크 DLP는 **'망 보안의 최후 전선'**으로서, 데이터의 흐름을 가시화하고 통제하는 핵심 인프라로 자리 잡아야 한다.

- **📢 섹션 요약 비유**: 성벽을 아무리 높게 쌓아도 성문(네트워크 관문)을 지키지 않으면 소용없다. 네트워크 DLP는 성문을 지키는 가장 똑똑한 문지기다.
