## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 엔드포인트 DLP는 사용자의 PC나 모바일 단말기에 설치된 에이전트를 통해 데이터의 생성, 복사, 수정, 출력 등 모든 로컬 행위를 실시간으로 감시하고 통제하는 기술이다.
> 2. **가치**: 네트워크를 거치지 않는 오프라인 유출 경로(USB, 외장하드, 프린터, 블루투스 등)와 클립보드 복사, 화면 캡처 등 세밀한 행위까지 차단할 수 있다.
> 3. **판단 포인트**: OS 커널 레벨에서 동작하므로 시스템 안정성 및 성능 저하 문제를 최소화하면서, 우회 공격(Safe Mode, 에이전트 강제 종료)에 대한 방어력을 갖추는 것이 핵심이다.

---

## Ⅰ. 개요 및 필요성

엔드포인트 DLP는 데이터 보호의 "최접점 방어" 체계다. 정보 유출의 가장 큰 비중을 차지하는 이동식 저장매체(USB 등)와 출력물에 의한 유출을 막기 위해 필수적이다.

특히 원격 근무가 일상화된 환경에서 직원이 회사망 밖(카페, 집 등)에서 개인용 저장소로 데이터를 옮기거나 화면을 캡처하는 행위는 네트워크 DLP만으로는 잡아낼 수 없다. 엔드포인트 DLP는 장소와 무관하게 데이터가 '사용되는 순간(Data in Use)'과 '저장된 상태(Data at Rest)'를 직접 통제한다.

- **📢 섹션 요약 비유**: 각 방마다 CCTV를 설치하고 문 앞에는 경비원을 두어, 허락 없이 물건을 옮기거나 사진을 찍지 못하게 감시하는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

엔드포인트 DLP는 운영체제(OS)와 긴밀하게 결합하여 동작한다.

### 1. 주요 통제 경로 (Control Channels)
| 경로 | 통제 방식 | 주요 대상 |
|:---|:---|:---|
| **저장 매체** | 입출력 포트 제어 | USB, 외장하드, SD카드, CD-RW |
| **인쇄 및 출력** | 프린터 드라이버 후킹 | 로컬/네트워크 프린터, PDF 가상 프린터 |
| **입출력 행위** | OS 이벤트 가로채기 | 복사/붙여넣기(Clipboard), 화면 캡처(PrtSc) |
| **통신 인터페이스** | 드라이버 수준 차단 | 블루투스, Wi-Fi Direct, 테더링 |
| **콘텐츠 탐지** | 파일 시스템 스캔 | PC 내 방치된 개인정보 파일 검색 및 격리/암호화 |

### 2. 엔드포인트 DLP 동작 아키텍처
```text
┌──────────────────────────────────────────────────────────────┐
│                  사용자 엔드포인트 (Windows/macOS)           │
├──────────────────────────────────────────────────────────────┤
│ [ User App ]   [ File System ]   [ Peripherals ]             │
│      │               │                  │                    │
│      └───────▶ [ DLP Agent (Hooking) ] ◀──────┘              │
│                          │                                   │
│              ┌───────────┴───────────┐                       │
│              │   Policy Enforcement  │                       │
│              └───────────┬───────────┘                       │
└──────────────────────────┼───────────────────────────────────┘
                           ▼
                [ 중앙 관리 서버 (Server) ]
                - 정책 배포, 위반 로그 수집
```

### 3. 핵심 기술: 후킹 (Hooking) 및 드라이버 제어
- **API 후킹**: 애플리케이션이 OS에 요청하는 특정 명령(파일 저장, 클립보드 복사 등)을 중간에서 가로채 정책 위반 여부를 확인.
- **필터 드라이버**: 파일 시스템이나 USB 포트의 드라이버 레벨에서 I/O 트래픽을 직접 제어하여 높은 보안성과 성능을 확보.

- **📢 섹션 요약 비유**: 수도꼭지(USB/프린터) 입구에 거름망(에이전트)을 설치하여, 깨끗한 물(일반 데이터)은 통과시키고 찌꺼기(민감 정보)만 걸러내는 것과 같다.

## Ⅲ. 비교 및 연결

| 항목 | 엔드포인트 DLP | 네트워크 DLP |
|:---|:---|:---|
| 통제 위치 | 사용자 개별 PC (Local) | 네트워크 관문 (Gateway) |
| 오프라인 통제 | 가능 (에이전트 상주) | 불가 (네트워크 연결 필수) |
| 분석 시점 | 데이터가 사용되는 즉시 | 전송 패킷이 발생하는 시점 |
| 배포 부담 | 높음 (모든 PC에 설치 필요) | 낮음 (네트워크 장비 한 대) |
| 우회 난이도 | 어려움 (OS 레벨 통제) | 상대적 쉬움 (VPN, 테더링 등) |

- **📢 섹션 요약 비유**: 엔드포인트 DLP는 개인용 금고를 지키는 경비원이고, 네트워크 DLP는 마을 전체를 둘러싼 성벽의 문지기다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 가장 큰 기술적 도전은 **"OS 업데이트 대응"**과 **"성능 최적화"**다. OS 패치 후 에이전트가 충돌하여 블루스크린이 발생하면 전사적인 업무 마비로 이어진다.

### 체크리스트
1. **OS 호환성**: Windows, macOS뿐만 아니라 다양한 버전의 OS 패치에 대한 신속한 업데이트가 지원되는가?
2. **자가 보호**: 사용자가 에이전트 프로세스를 강제로 죽이거나 삭제하지 못하도록 견고한 보호 장치가 있는가?
3. **오프라인 정책**: 인터넷이 연결되지 않은 출장지 등에서도 기존 정책이 정상 작동하며 로그가 유실되지 않는가?

### 안티패턴
- 모든 파일 스캔 주기를 너무 짧게 설정하여 사용자의 PC가 지속적으로 느려지게 만드는 경우
- 특정 보안 프로그램과의 충돌을 피하기 위해 예외 폴더를 너무 넓게 잡아 유출 통로를 만드는 경우

- **📢 섹션 요약 비유**: 엔드포인트 DLP는 자동차의 안전벨트와 같다. 사고 시 목숨을 구해주지만, 너무 조이면 운전자가 답답해서 벨트를 풀고 싶어 하므로(우회 시도) 착용감을 개선해야 한다.

---

## Ⅴ. 기대효과 및 결론

엔드포인트 DLP는 가장 세밀하고 확실한 데이터 유출 차단 수단이다. 이동식 저장매체 통제부터 화면 캡처 방지까지, 데이터가 디지털 형태에서 아날로그(출력, 캡처)로 변하는 모든 지점을 방어한다. 미래에는 AI 기반의 사용자 행위 분석(UEBA)과 결합하여 '평소와 다른 파일 접근 패턴'을 사전에 차단하는 지능형 엔드포인트 보안으로 발전할 것이다.

결론적으로 엔드포인트 DLP는 **'데이터와 가장 가까운 곳에서 작동하는 최후의 보루'**로서 보안 거버넌스의 핵심 기반이 된다.

- **📢 섹션 요약 비유**: 집안 곳곳에 열쇠를 채우고 창문을 잠그는 것처럼, 가장 꼼꼼하게 우리 집 보물(데이터)을 지켜주는 방법이다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| Filter Driver | 파일 시스템 레벨에서 I/O를 가로채는 엔드포인트 DLP의 핵심 기술 |
| EPP | 백신, EDR, DLP가 통합된 엔드포인트 보호 플랫폼 |
| OCR | 화면 캡처 방지 및 이미지 내 개인정보 탐지를 위한 보조 기술 |
| 커널 모드 (Kernel Mode) | 높은 권한에서 보안 정책을 강제하기 위한 OS 동작 영역 |

### 👶 어린이를 위한 3줄 비유 설명

1. 내 컴퓨터 안을 24시간 지켜주는 로봇 경찰관이 있는 거예요.
2. 내가 중요한 숙제를 USB에 옮기거나 사진을 찍으려고 하면 로봇이 "안 돼!" 하고 막아줘요.
3. 학교 밖으로 나가도 이 로봇은 항상 나를 따라다니며 내 소중한 정보들을 지켜준답니다.
