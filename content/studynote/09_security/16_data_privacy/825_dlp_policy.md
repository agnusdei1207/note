## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DLP 정책은 "무엇이 민감한 데이터인가(Content)"와 "그 데이터가 어떤 상황에서 사용되는가(Context)"를 결합하여 유출 여부를 판단하는 지능형 규칙이다.
> 2. **가치**: 단순 패턴 매칭의 한계를 넘어 사용자, 애플리케이션, 시간, 장소 등 주변 맥락을 고려함으로써 오탐을 줄이고 탐지 정확도를 극대화한다.
> 3. **판단 포인트**: 데이터의 민감도에 따라 콘텐츠 분석 깊이를 조절하고, 업무 프로세스에 맞춘 컨텍스트 예외 처리를 정교하게 설계하는 것이 운영의 성패를 가른다.

---

## Ⅰ. 개요 및 필요성

DLP 시스템의 두뇌는 '정책'이다. 아무리 훌륭한 탐지 엔진이 있어도 "무엇이 유출인가"에 대한 정의가 부실하면 무용지물이다. 초기 DLP는 "주민번호가 들어있으면 차단"과 같은 단순 **콘텐츠(Content)** 기반 정책에 의존했으나, 이는 정상적인 업무(예: 인사팀의 급여 처리)까지 차단하는 부작용을 낳았다.

이를 해결하기 위해 등장한 것이 **컨텍스트(Context)** 기반 정책이다. "누가, 어떤 프로그램으로, 어디로 보내는가"를 함께 따짐으로써 보안성과 편의성 사이의 균형을 맞춘다.

- **📢 섹션 요약 비유**: 검문소에서 가방 내용물(콘텐츠)만 보는 게 아니라, 그 사람이 누구인지와 어디로 가는지(컨텍스트)를 종합해서 통과 여부를 결정하는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

DLP 정책은 크게 두 축으로 구성되며, 이들이 교차(Intersection)할 때 차단이 발생한다.

### 1. 콘텐츠 기반 탐지 (Content-based)
데이터 자체의 속성을 분석하여 민감도를 식별한다.
- **패턴 매칭 (Pattern)**: 주민번호, 카드번호, 계좌번호 등 정형화된 식별자 탐지.
- **키워드 (Keyword)**: '대외비', '기밀', '설계도' 등 특정 단어 포함 여부 확인.
- **지문 인식 (Fingerprinting)**: 원본 문서의 전체 또는 일부와 일치하는지 비교.
- **분류 라벨 (Label/Tag)**: 문서 생성 시 부여된 보안 등급 태그를 읽음.

### 2. 컨텍스트 기반 제어 (Context-based)
데이터를 둘러싼 상황 정보를 분석하여 허용 여부를 결정한다.
| 컨텍스트 요소 | 판단 기준 예시 |
|:---|:---|
| 사용자 (User) | 인사팀은 허용, 일반 직원은 차단 |
| 앱 (Application) | 사내 ERP는 허용, 개인 웹메일은 차단 |
| 목적지 (Destination) | 승인된 USB는 허용, 미승인 저장매체는 차단 |
| 시간/장소 (Environment) | 업무 시간 외 반출 차단, 사외망 접속 시 차단 강화 |

### 3. 정책 판단 로직 다이어그램
```text
┌──────────────────────┐          ┌──────────────────────┐
│  Content Inspection  │          │  Context Awareness   │
│ (What is the data?)  │          │ (Who, How, Where?)   │
└──────────┬───────────┘          └──────────┬───────────┘
           │                                 │
           └───────▶ [ Policy Engine ] ◀───────┘
                          │
            IF (Sensitive Content AND Unsafe Context)
            THEN [ Action: Block / Alert / Encrypt ]
```

- **📢 섹션 요약 비유**: 수술실에 칼(콘텐츠)이 있는 건 정상이지만, 지하철역에서 누군가 칼을 들고 있다면(컨텍스트) 즉시 제지해야 하는 것과 같다.

## Ⅲ. 비교 및 연결

| 항목 | 콘텐츠 기반 (Content) | 컨텍스트 기반 (Context) |
|:---|:---|:---|
| 판단 중심 | 데이터의 '내용' | 데이터의 '흐름과 상황' |
| 분석 부하 | 높음 (전체 파일 스캔 필요) | 낮음 (메타데이터 및 행위 분석) |
| 오탐 가능성 | 높음 (동명이인, 유사 번호 등) | 낮음 (명확한 행위 주체 존재) |
| 우회 가능성 | 있음 (암호화, 이미지 변환 등) | 낮음 (전송 경로 자체를 통제) |
| 주요 용도 | 개인정보 식별, 지적재산 보호 | 유출 경로 차단, 예외 처리 관리 |

- **📢 섹션 요약 비유**: 콘텐츠 기반은 편지 내용을 다 읽어보는 것이고, 컨텍스트 기반은 편지 봉투의 발신인과 수신인만 확인하는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 가장 효과적인 정책은 **"단계적 강화"**다. 처음부터 차단(Block)을 걸지 않고 탐지(Monitor) 기간을 두어 정책을 정교화해야 한다.

### 체크리스트
1. **화이트리스트** 애플리케이션이 명확히 정의되었는가? (정상적인 업무 툴 차단 방지)
2. 대량의 데이터 반출 시에만 경고를 주는 **임계치(Threshold)** 정책이 있는가?
3. 오탐 시 사용자가 직접 소명하고 승인을 요청할 수 있는 **워크플로우**가 마련되어 있는가?

### 안티패턴
- 모든 직원에게 동일한 정책을 적용하여 특정 부서의 업무를 마비시키는 경우
- 암호화된 압축 파일(ZIP)에 대한 대응 정책 없이 단순 파일명 검사만 수행하는 경우

- **📢 섹션 요약 비유**: 정책 설계는 그물을 짜는 것과 같다. 그물코가 너무 작으면 물(업무)이 안 빠지고, 너무 크면 물고기(데이터)를 놓친다.

---

## Ⅴ. 기대효과 및 결론

콘텐츠와 컨텍스트가 결합된 현대적 DLP 정책은 보안의 실효성을 높이는 동시에 임직원의 업무 불편을 최소화한다. 최근에는 사용자 행위 분석(UEBA)과 결합하여 평소와 다른 이례적인 맥락을 스스로 학습하고 정책에 반영하는 인텔리전트 DLP로 발전하고 있다.

결론적으로 DLP 정책은 고정된 규칙이 아니라 **'비즈니스 변화에 따라 진화하는 살아있는 가이드라인'**이 되어야 한다.

- **📢 섹션 요약 비유**: 똑똑한 문지기는 집주인이 들어올 때는 웃으며 문을 열어주고, 낯선 사람이 담을 넘을 때만 호각을 분다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| UEBA | 사용자의 평소 컨텍스트를 학습하여 이상 징후를 탐지하는 기술 |
| 정규표현식 | 콘텐츠 기반 정책에서 패턴을 정의하는 핵심 도구 |
| 데이터 분류 | 정책 적용 전 데이터의 중요도를 먼저 결정하는 선행 작업 |
| 화이트리스트 | 안전하다고 판단되는 컨텍스트(앱, 주소 등)의 예외 목록 |

### 👶 어린이를 위한 3줄 비유 설명

1. "위험한 물건(콘텐츠)"을 가지고 있는지 확인하는 게 첫 번째 규칙이에요.
2. "누가, 왜(컨텍스트)" 가지고 있는지를 보는 게 두 번째 규칙이에요.
3. 요리사 아저씨가 식당에서 칼을 쓰는 건 괜찮지만, 놀이터에서 칼을 들고 있으면 안 된다고 가르쳐주는 것과 같답니다.
