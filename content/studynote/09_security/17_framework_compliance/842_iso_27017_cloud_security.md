## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ISO/IEC 27017은 ISO 27001 및 ISO 27002를 클라우드 환경에 최적화하여 보완한 **클라우드 서비스 보안 통제** 가이드라인이다.
> 2. **가치**: 클라우드 서비스 제공자(CSP)와 클라우드 서비스 이용자(CSC)가 각각 수행해야 할 보안 역할과 책임을 명확히 정의하여 **공유 책임 모델**을 실현하게 한다.
> 3. **판단 포인트**: 기존 보안 통제 외에 클라우드 특화 항목(데이터 삭제, 가상화 보안 등)이 추가되었으므로, 클라우드 도입 시의 보안 가이드라인으로 활용해야 한다.

---

## Ⅰ. 개요 및 필요성

ISO/IEC 27017은 클라우드 컴퓨팅 환경에서의 정보보안을 위한 국제 표준이다. 일반적인 ISMS(ISO 27001)만으로는 해결하기 어려운 클라우드 특유의 위험(멀티테넌시, 가상화 취약점, 데이터 소재지 불분명 등)을 관리하기 위해 제정되었다.

클라우드 환경에서는 하드웨어와 네트워크를 CSP가 관리하므로, 이용자는 가시성(Visibility)과 통제권(Control)을 잃기 쉽다. ISO 27017은 이러한 환경에서도 보안 수준을 일정하게 유지할 수 있도록 CSP와 CSC 간의 합의된 보안 요구사항을 제시한다.

- **📢 섹션 요약 비유**: ISO 27001이 '우리 집을 지키는 보안 규칙'이라면, ISO 27017은 '아파트(클라우드)에 살 때 관리소(CSP)와 입주민(CSC)이 각각 지켜야 할 아파트 전용 보안 수칙'과 같다.

## Ⅱ. 아키텍처 및 핵심 원리

ISO 27017은 기존 ISO 27002의 통제 항목에 클라우드 특화 지침을 추가하고, 7개의 새로운 클라우드 전용 통제를 신설했다.

### 1. 공유 책임 모델 (Shared Responsibility Model) 적용

ISO 27017은 모든 통제 항목에 대해 제공자와 이용자의 역할을 구분한다.

```text
┌─────────────────────────────────────────────────────────────┐
│             ISO 27017 핵심 개념: 역할과 책임 (R&R)          │
├─────────────────────────────────────────────────────────────┤
│  [ Cloud Service Provider ]       [ Cloud Service Customer ]│
│  - 물리적 데이터 센터 보안        - 데이터 암호화 및 관리   │
│  - 가상화 소프트웨어 보안         - 계정 및 권한 관리       │
│  - 서비스 가용성 보장             - 애플리케이션 보안       │
└──────────────────────┬──────────────────────┬───────────────┘
                       │   상호 합의 및 투명성   │
                       └──────────────────────┘
```

### 2. ISO 27017 신설 통제 항목 (CL-Controls)

| 통제 번호 | 핵심 내용 | 설명 |
|:---|:---|:---|
| **CL.6.1.1** | 역할과 책임의 할당 | CSP와 CSC 간의 보안 역할 분담 명시 |
| **CL.9.5.1** | 클라우드 환경 자산 제거 | 이용 중단 시 데이터의 완전하고 안전한 삭제 보장 |
| **CL.12.1.2** | 가상화 환경 격리 | 가상 머신(VM) 간의 상호 영향 차단 및 자원 격리 |
| **CL.12.4.5** | 가용성 모니터링 | 클라우드 자원의 성능과 가용성에 대한 지속적 모니터링 |
| **CL.13.1.4** | 네트워크 가상 구성 | 가상 네트워크 환경에서의 보안 설정 및 분리 |

- **📢 섹션 요약 비유**: 복도 청소는 관리소가 하고, 우리 집 도어락 관리는 내가 하는 것처럼 클라우드에서도 책임을 반으로 나누는 원리다.

## Ⅲ. 비교 및 연결

유사한 클라우드 관련 표준인 **ISO 27018** 및 국내 **CSAP**와 비교된다.

| 항목 | ISO 27017 | ISO 27018 | CSAP (한국) |
|:---|:---|:---|:---|
| **주요 목표** | 클라우드 전반의 보안 강화 | 클라우드 내 개인정보(PII) 보호 | 공공기관용 클라우드 보안 인증 |
| **대상** | CSP 및 CSC 모두 | CSP 중심 (PIMS 보완) | CSP 중심 (국내 공공 납품용) |
| **핵심 내용** | 범용 클라우드 보안 통제 | 개인정보 처리 투명성, 삭제 | 물리적 망 분리 등 강력한 요건 |

ISO 27017 인증을 획득하려면 먼저 **ISO 27001 인증**이 선행되거나 동시에 진행되어야 한다.

- **📢 섹션 요약 비유**: ISO 27017이 클라우드라는 '공간'을 지키는 규칙이라면, ISO 27018은 그 공간에 보관된 '귀중품(개인정보)'만을 전문적으로 지키는 금고 규칙이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 클라우드 도입 초기 단계에서 **'책임 추적성'**과 **'투명성'** 확보를 위해 ISO 27017을 활용해야 한다.

### 실무적 판단 포인트

1.  **서비스 모델(IaaS/PaaS/SaaS)별 차별화**: 모델에 따라 CSP가 책임지는 범위가 다르다. SaaS로 갈수록 CSP의 책임이 커지며, CSC는 설정과 데이터 관리에 집중해야 한다.
2.  **데이터 파기 확인**: 클라우드 계약 종료 시 내 데이터가 디스크에서 완전히 지워졌는지 확인할 수 있는 절차나 증적을 CSP에 요구해야 한다.
3.  **가상화 취약점 대응**: 가상 머신 간의 탈출(VM Escape) 공격 등을 예방하기 위해 CSP가 어떤 격리 기술을 사용하는지 체크리스트로 점검한다.

### 안티패턴
- "클라우드니까 아마존(AWS)이 다 알아서 하겠지"라는 생각 (이용자 책임 영역 방치)
- 내부 물리 보안 지침을 클라우드 가상 환경에 그대로 적용하려는 시도 (비효율 발생)

- **📢 섹션 요약 비유**: 호텔(클라우드)에 묵는다고 해서 내 방의 귀중품(데이터)을 문 열어둔 채 방치하면 안 된다. 호텔은 복도와 로비를 지킬 뿐, 내 방 안의 보안은 내 책임이다.

---

## Ⅴ. 기대효과 및 결론

ISO 27017을 도입하면 조직은 클라우드 전환에 따른 보안 불안감을 해소하고, 사고 발생 시 책임 소재를 명확히 할 수 있다. 또한 글로벌 수준의 보안 체계를 갖춤으로써 해외 고객사에 대한 신뢰를 확보할 수 있다.

결론적으로 클라우드 보안은 '공유(Sharing)'와 '신뢰(Trust)'에 기반한다. ISO 27017은 그 신뢰를 증명하는 구체적인 수단이며, 클라우드 네이티브 보안 아키텍처의 표준 설계도가 된다.

- **📢 섹션 요약 비유**: ISO 27017은 클라우드 시대를 위한 '안전한 공유 경제 가이드라인'이다. 서로의 역할을 지킬 때 모두가 안전해질 수 있다.
