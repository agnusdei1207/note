## 핵심 인사이트 (3줄 요약)
> 1. **본질**: NIST CSF 2.0은 조직의 사이버 보안 위험을 관리하기 위한 프레임워크로, 기존 5개 기능에 **'지배 구조 (GOVERN)'** 기능을 추가하여 전사적 거버넌스를 강화한 최신 표준이다.
> 2. **가치**: 1.0의 핵심 인프라 보호 중심에서 벗어나 모든 규모와 산업의 조직으로 적용 대상을 확대하고, 보안이 기술이 아닌 경영의 영역임을 명확히 했다.
> 3. **판단 포인트**: 새롭게 추가된 'GOVERN' 기능은 다른 5개 기능(ID, PR, DE, RS, RC)의 방향성을 결정하는 중심축이므로, 거버넌스 수립을 최우선 과제로 삼아야 한다.

---

## Ⅰ. 개요 및 필요성

NIST CSF(Cybersecurity Framework)는 미국 국립표준기술연구소(NIST)가 제정한 글로벌 보안 프레임워크다. 2024년 발표된 2.0 버전은 지난 10년간의 위협 환경 변화를 반영하여, 단순한 방어를 넘어 조직의 전략과 보안을 일치시키는 데 중점을 둔다.

복잡해진 클라우드 환경과 공급망 위협 속에서, 기술적 통제만으로는 한계가 있다. 경영진이 보안 위험을 비즈니스 위험으로 인식하고 의사결정에 참여해야만 실질적인 방어가 가능하다는 인식이 CSF 2.0의 탄생 배경이다.

- **📢 섹션 요약 비유**: NIST CSF 1.0이 튼튼한 '성벽과 무기'를 만드는 법이었다면, 2.0은 성을 지키기 위한 '지휘부(거버넌스)'를 세우고 전 국민(모든 조직)이 참여하는 국방 체계를 구축하는 것과 같다.

## Ⅱ. 아키텍처 및 핵심 원리

CSF 2.0은 기존의 5개 핵심 기능(Functions)에 **'GOVERN(지배 구조)'**을 추가하여 6개 기능 체계로 개편되었다.

### 1. NIST CSF 2.0 핵심 기능 다이어그램

```text
              ┌─────────────────────────┐
              │       [ GOVERN ]        │ <─── (New & Central)
              │ (지배 구조 및 전략 수립)│
              └────────────┬────────────┘
                           │
       ┌───────────┬───────┴───────┬───────────┐
       │           │               │           │
┌──────▼───┐  ┌────▼─────┐  ┌──────▼───┐  ┌────▼─────┐  ┌──────▼───┐
│ IDENTIFY │  │ PROTECT  │  │  DETECT  │  │ RESPOND  │  │ RECOVER  │
│  (식별)  │  │  (보호)   │  │  (탐지)  │  │  (대응)   │  │  (복구)   │
└──────────┘  └──────────┘  └──────────┘  └──────────┘  └──────────┘
```

### 2. 주요 기능별 정의 (6 Core Functions)

| 기능 (Function) | 설명 | 주요 카테고리 예시 |
|:---|:---|:---|
| **GOVERN (GV)** | 보안 전략, 역할, 책임, 정책 및 위험 관리 감독 | 조직 맥락, 위험 관리 전략, 공급망 위험 관리 |
| **IDENTIFY (ID)** | 자산, 비즈니스 환경, 취약점 등 현재 상태 파악 | 자산 관리, 위험 평가, 개선 사항 식별 |
| **PROTECT (PR)** | 위험을 방지하거나 제한하기 위한 안전장치 구현 | 데이터 보안, 접근 제어, 인식 교육, 플랫폼 보안 |
| **DETECT (DE)** | 사이버 보안 사건의 발생을 시의적절하게 발견 | 모니터링, 이상 징후 분석, 탐지 프로세스 |
| **RESPOND (RS)** | 탐지된 사건에 대한 적절한 조치 수행 | 사고 분석, 억제, 근절, 소통 |
| **RECOVER (RC)** | 손상된 기능이나 서비스를 정상으로 복구 | 복구 계획 실행, 교훈 반영, 대외 커뮤니케이션 |

- **📢 섹션 요약 비유**: GOVERN은 작전을 짜는 지휘소이고, 나머지는 그 작전에 따라 움직이는 수색(ID), 방어(PR), 초소(DE), 기동대(RS), 공병대(RC)와 같다.

## Ⅲ. 비교 및 연결

NIST CSF 1.1과 2.0의 가장 큰 차이점은 '거버넌스'의 독립과 '적용 범위'의 확장이다.

| 항목 | NIST CSF 1.1 | NIST CSF 2.0 |
|:---|:---|:---|
| **핵심 기능** | 5개 (ID, PR, DE, RS, RC) | 6개 (GV 추가) |
| **적용 대상** | 국가 핵심 기반 시설 중심 | 모든 규모와 유형의 조직 (공공/민간 전체) |
| **거버넌스 위치** | Identify(식별) 기능의 일부 | 독립된 최상위 기능 (모든 활동의 기반) |
| **공급망 관리** | 제한적인 비중 | GV(지배 구조) 내 핵심 요소로 강조 |

CSF 2.0은 ISO 27001과 상호 맵핑이 가능하여, 글로벌 보안 컴플라이언스를 동시에 충족하는 데 유리하다.

- **📢 섹션 요약 비유**: 1.1 버전이 '국가 대표팀 전용 훈련법'이었다면, 2.0 버전은 '동네 조기축구회부터 프로팀까지 모두 쓸 수 있는 표준 가이드'다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 CSF 2.0 도입 시 **'GOVERN'** 기능을 통해 보안과 비즈니스의 정렬(Alignment)을 최우선으로 판단해야 한다.

### 실무적 적용 전략

1.  **GOVERN 중심의 정책 수립**: 보안 사고 발생 시 기술적 대응 이전에, 경영진의 의사결정 체계와 법적 준거성(Compliance)이 GOVERN 카테고리에 명확히 정의되어 있는지 점검한다.
2.  **프로필(Profile) 활용**: 조직의 현재 상태(Current Profile)와 목표 상태(Target Profile)를 작성하고, 그 간극(Gap)을 메우기 위한 로드맵을 6개 기능을 기준으로 수립한다.
3.  **공급망 위험 관리 (C-SCRM)**: 소프트웨어 공급망 공격(SolarWinds 사례 등)에 대비하여, 외부 파트너사의 보안 수준을 GV 기능 내에서 관리하도록 체계를 강화한다.

### 안티패턴
- GOVERN을 무시한 기술 도입: 지휘 계통 없이 비싼 무기(보안 솔루션)만 사들이는 경우
- 1.0 방식의 답습: 거버넌스를 단순히 '자산 식별'의 하위 업무로 처리하는 행위

- **📢 섹션 요약 비유**: 아무리 훌륭한 방패(Protect)가 있어도, 그 방패를 언제 어디서 들어야 할지 결정하는 장군(Govern)이 없으면 전쟁에서 승리할 수 없다.

## Ⅴ. 기대효과 및 결론

NIST CSF 2.0은 보안을 '기술'에서 '문화'와 '경영'으로 격상시켰다. 이를 통해 조직은 단순한 규제 준수를 넘어, 비즈니스 가치를 보호하고 위기 상황에서 빠르게 회복할 수 있는 실질적인 역량을 갖추게 된다.

결론적으로 CSF 2.0은 **'거버넌스가 이끄는 보안'**의 시대를 선언했다. 기술사로서 조직의 보안 아키텍처를 설계할 때, GOVERN이라는 뿌리가 튼튼하게 자리 잡도록 거버넌스 체계를 최우선적으로 설계해야 한다.

- **📢 섹션 요약 비유**: NIST CSF 2.0은 조직이라는 거대한 기계가 보안이라는 윤활유를 바르고 안전하게 돌아가게 만드는 '중앙 제어 장치'와 같다.
