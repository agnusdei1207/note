## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ISO/IEC 27002는 ISO 27001 Annex A의 보안 통제 항목을 실제로 어떻게 구현할 것인지에 대한 상세한 가이드라인을 제공하는 실무 지침서(Code of Practice)이다.
> 2. **가치**: "무엇을 해야 하는가(Requirements)"를 넘어 "어떻게 구현하는가(Implementation Guidance)"를 다루어, 조직이 실무에 즉시 적용 가능한 모범 사례(Best Practices)를 제시한다.
> 3. **판단 포인트**: ISO 27001은 '인증'을 위한 규격이고, ISO 27002는 인증을 준비하기 위한 '참고서'이므로, 조직의 상황에 맞게 유연하게 해석하여 적용해야 한다.

---

## Ⅰ. 개요 및 필요성

ISO/IEC 27002는 정보보호 경영시스템(ISMS)을 구축하는 조직이 보안 통제를 선택하고 실행하는 데 도움을 주기 위해 설계된 국제 표준이다. ISO 27001이 조직이 지켜야 할 '최소한의 법규'라면, ISO 27002는 그 법규를 잘 지키기 위한 '백과사전'과 같다.

보안 담당자 입장에서 "접근 제어를 강화하라"는 ISO 27001의 요구사항은 너무 추상적일 수 있다. 이때 ISO 27002는 사용자 등록, 비밀번호 관리, 특권 권한 제한 등 구체적인 방법론을 제시하여 보안 구축의 일관성과 전문성을 보장한다.

- **📢 섹션 요약 비유**: ISO 27001이 "건강을 위해 규칙적인 운동을 하라"는 지침이라면, ISO 27002는 "팔굽혀펴기는 이렇게 하고, 달리기는 몇 분 동안 하라"고 알려주는 PT 교본과 같다.

## Ⅱ. 아키텍처 및 핵심 원리

ISO 27002의 구조는 ISO 27001 Annex A와 1:1로 대응되며, 각 통제 항목에 대해 상세한 설명과 고려사항을 제공한다.

### 1. ISO 27002:2022 통제 항목 구성 요소

각 통제 항목(Control)은 단순한 설명이 아니라 다음과 같은 구조화된 정보를 포함한다.

| 구성 요소 | 역할 | 설명 |
|:---|:---|:---|
| **통제 명칭** | 항목 식별 | 예: 8.1 사용자 식별 및 인증 |
| **통제 목적** | 목적 정의 | 왜 이 통제가 필요한지 설명 |
| **통제 내용** | 요구사항 요약 | 수행해야 할 핵심 보안 활동 |
| **구현 지침** | 상세 가이드 | 구체적인 프로세스, 기술적 수단, 절차 제시 |
| **기타 정보** | 추가 고려사항 | 법적 요건, 타 표준과의 연계 정보 등 |
| **속성 (Attributes)** | 분류 태그 | #Preventive, #Confidentiality 등 5가지 속성 태그 |

### 2. 2022년판의 핵심: 속성(Attributes) 시스템

ISO 27002:2022는 통제를 다각도에서 분석할 수 있도록 5가지 속성 테이블을 도입했다.

```text
┌─────────────────────────────────────────────────────────────┐
│             Control 8.12: Data Leakage Prevention           │
├─────────────────────────────────────────────────────────────┤
│ [Attributes Table]                                          │
│ - Control Type: #Preventive, #Detective                     │
│ - Information Security Properties: #Confidentiality         │
│ - Cybersecurity Concepts: #Protect, #Detect                 │
│ - Operational Capabilities: #Information_protection         │
│ - Security Domains: #Governance_and_Ecosystem               │
└─────────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: 사전의 단어 설명 뒤에 유의어, 반의어, 예문이 상세히 붙어 있는 것처럼, 보안 항목 하나하나에 풍부한 해설과 태그를 붙여놓은 것이다.

## Ⅲ. 비교 및 연결

가장 흔한 혼동인 ISO 27001과 ISO 27002의 차이를 명확히 해야 한다.

| 항목 | ISO/IEC 27001 | ISO/IEC 27002 |
|:---|:---|:---|
| **성격** | 요구사항 규격 (Requirements) | 지침 및 권고안 (Guideline) |
| **주요 목적** | 제3자 인증 획득 | 보안 통제의 구현 및 관리 |
| **문구 특징** | "~해야 한다 (Shall)" | "~하는 것이 좋다 (Should/May)" |
| **강제성** | 인증을 위해 필수 준수 | 조직의 필요에 따라 선택적 참고 |

ISO 27002는 조직의 내부 보안 규정(Security Policy)을 작성할 때 가장 기초가 되는 자료로 활용된다.

- **📢 섹션 요약 비유**: ISO 27001은 합격해야 하는 '시험지'이고, ISO 27002는 그 시험 문제를 풀기 위해 공부하는 '교과서'다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 ISO 27002를 '정답지'로 보지 말고, 조직의 위험 수준에 맞게 **'가공하여 적용'**하는 능력을 보여야 한다.

### 실무적 활용 전략

1.  **내부 지침 표준화**: ISO 27002의 목차를 기반으로 사내 보안 지침서를 구성하면 글로벌 표준과의 정합성을 쉽게 확보할 수 있다.
2.  **Gap Analysis 수행**: 현재 조직의 보안 현황을 ISO 27002의 모범 사례와 비교하여, 어떤 부분이 부족한지 정량적으로 파악할 수 있다.
3.  **속성 기반 대시보드 구축**: 2022년판의 속성 태그를 활용하여 "우리 회사는 탐지(Detective) 통제가 부족하다"는 식의 가시적인 보안 리포트를 경영진에게 제공할 수 있다.

### 안티패턴
- ISO 27002의 모든 내용을 그대로 복사하여 사내 규정으로 만드는 경우 (조직의 현실과 동떨어진 '장롱 규정'이 됨)
- 최신 개정판(2022)이 나왔음에도 과거(2013) 기준의 구현 지침을 고집하는 경우

- **📢 섹션 요약 비유**: 아무리 좋은 레시피(ISO 27002)가 있어도, 냉장고에 있는 재료와 가족들의 입맛에 맞춰 요리사(기술사)가 적절히 가감해야 최고의 요리가 나온다.

---

## Ⅴ. 기대효과 및 결론

ISO 27002를 활용함으로써 조직은 보안 체계 구축의 시행착오를 줄이고, 전 세계적으로 검증된 보안 수준을 유지할 수 있다. 또한 외부 감사나 파트너사 점검 시 보안 조치의 근거로 제시하기에 가장 강력한 논거가 된다.

결론적으로 ISO 27002는 정보보호 전문가에게는 필수적인 '도구 상자'이다. 이 도구들을 적재적소에 꺼내어 조직의 보안이라는 성을 견고하게 쌓아 올려야 한다.

- **📢 섹션 요약 비유**: ISO 27002는 보안 전문가를 위한 '백과사전'이다. 모르는 문제가 생길 때마다 펼쳐보고 정답에 가까운 길을 찾게 도와준다.
