## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 안티 포렌식(Anti-forensics)은 디지털 증거의 수집, 식별, 분석을 기술적으로 방해하거나 무력화하여 수사 과정을 지연시키고 결과의 신뢰성을 떨어뜨리는 모든 기법을 말한다.
> 2. **가치**: 공격자에게는 자신의 정체를 감추고 법적 처벌을 피하는 수단이 되며, 방어자(분석가)에게는 이러한 기법의 존재 자체가 공격의 고도화를 판단하는 중요한 지표가 된다.
> 3. **판단 포인트**: 데이터 파괴(Wiping), 암호화(Encryption), 데이터 은닉(Steganography) 등 사용된 기법의 특성을 파악하여, 우회 분석 경로(메모리 분석, 키로깅 등)를 신속히 결정해야 한다.

---

## Ⅰ. 개요 및 필요성

안티 포렌식은 포렌식 수사관과의 '지능적 싸움'이다. 포렌식 기술이 발전함에 따라 공격자들도 단순히 파일을 지우는 수준을 넘어, 증거가 생성되는 시점부터 개입하거나 수사 도구의 취약점을 공격하는 방식으로 대응하고 있다.

안티 포렌식 기법이 적용된 침해 사고는 분석 시간이 기하급수적으로 늘어나며, 최악의 경우 "증거 없음"으로 결론지어질 수 있다. 따라서 기술사는 안티 포렌식의 한계를 이해하고, 공격자가 완벽하게 지우지 못한 '그림자 증거'를 찾아내는 통찰력을 가져야 한다.

- **📢 섹션 요약 비유**: 안티 포렌식은 범인이 현장에서 지문을 지우는(Wiping) 것뿐만 아니라, 가짜 지문을 남기거나(Artifact Alteration) 특수 잉크로 비밀 메시지를 적어두는(Steganography) 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

안티 포렌식은 목적에 따라 크게 4가지 범주로 나뉜다.

### 안티 포렌식 기법 분류

| 분류 | 주요 기법 | 상세 내용 |
|:---|:---|:---|
| **데이터 파괴** | Data Wiping / Degaussing | 0과 1을 덮어쓰거나 자기장으로 완전 삭제 (복구 불가) |
| **데이터 은닉** | Steganography / ADS | 이미지/오디오 파일 내부에 데이터를 숨기거나 NTFS ADS 속성 활용 |
| **데이터 암호화** | BitLocker / VeraCrypt | 강력한 암호 알고리즘을 사용하여 분석가의 접근 차단 |
| **흔적 위조/변조** | Timestomping / Log Altering | 파일 시간 정보 수정, 로그의 특정 라인만 교묘히 변경 |

### 데이터 은닉 원리 (NTFS ADS 예시)

```text
  [Standard File Structure]           [ADS Hidden Structure]
┌─────────────────────────┐       ┌─────────────────────────┐
│  Filename: readme.txt   │       │  Filename: readme.txt   │
├─────────────────────────┤       ├─────────────────────────┤
│  Standard $DATA Stream  │       │  Standard $DATA Stream  │
│  (Visible Content)      │       │  (Visible Content)      │
└─────────────────────────┘       ├─────────────────────────┤
                                  │  Hidden $DATA Stream    │ ──▶ (일반 탐색기에서는
                                  │  (readme.txt:hidden.exe)│      보이지 않는 영역)
                                  └─────────────────────────┘
```

- **📢 섹션 요약 비유**: 안티 포렌식 아키텍처는 마술사의 무대 장치와 같다. 관객(분석가)이 보는 곳에는 가짜를 보여주고, 보이지 않는 곳(ADS, 슬랙 공간)에 진실을 숨겨두는 트릭이다.

## Ⅲ. 비교 및 연결

안티 포렌식 기법 간의 효과와 분석가 입장에서의 난이도를 비교해야 한다.

| 항목 | 데이터 파괴 (Wiping) | 데이터 암호화 (Encryption) | 데이터 은닉 (Steganography) |
|:---|:---|:---|:---|
| 복구 가능성 | 거의 불가능 (덮어쓰기 시) | 키 확보 시 가능 | 은닉 위치 발견 시 가능 |
| 탐지 난이도 | 낮음 (0으로 가득 찬 섹터 등) | 낮음 (높은 엔트로피 확인) | 매우 높음 (원본과 대조 필요) |
| 분석가 전략 | 슬랙 공간, 메모리 분석 | 메모리 내 키 추출, 무차별 대입 | 통계적 분석, 시그니처 분석 |
| 연결 개념 | `663. 증거 보전` | `665. 메모리 포렌식` | `671. MFT 분석` |

- **📢 섹션 요약 비유**: Wiping은 문서를 태우는 것이고, Encryption은 금고에 넣는 것이며, Steganography는 투명 잉크로 쓰는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 안티 포렌식 기법이 발견되었을 때 정면 돌파할 것인지 우회할 것인지 판단해야 한다.

### 실무 판단 가이드

1. **암호화 드라이브 발견**: BitLocker나 VeraCrypt가 확인되면 즉시 전원을 차단하지 말고, 메모리 덤프를 통해 '복호화 키'나 '마운트된 볼륨' 정보가 있는지 먼저 확인했는가?
2. **비정상 엔트로피 탐지**: 파일의 내부 구조가 무질서(High Entropy)하다면 암호화나 압축 기법이 사용되었음을 직감하고 전용 분석 프로세스를 가동했는가?
3. **MFT 불일치 분석**: `$SI`와 `$FN` 시간 정보의 미세한 시차를 통해 Timestomping 시도를 입증하고 실제 행위 시각을 보정했는가?
4. **도구 무력화 대응**: 공격자가 특정 포렌식 도구의 취약점을 이용해 분석을 방해하는 'Anti-Analysis' 코드를 심어두었을 가능성에 대비했는가?

### 안티패턴
- 암호화된 파일을 보고 "비밀번호가 없으니 포기한다"고 결론짓는 행위 (메모리 분석 등 우회로 존재)
- 하드디스크가 0으로 덮어쓰여졌다고 해서 모든 증거가 사라졌다고 믿는 행위 (클라우드 로그, 네트워크 패킷 등 외부 증거 잔존 가능성)

- **📢 섹션 요약 비유**: 안티 포렌식 실무는 창과 방패의 대결이다. 방패(안티 포렌식)가 단단하다면 무작정 찌르기보다 방패 옆으로 돌아가거나(메모리 분석) 방패 뒤의 사람을 직접 공략(사용자 수사)해야 한다.

## Ⅴ. 기대효과 및 결론

안티 포렌식 기법에 대한 깊은 이해는 역설적으로 수사의 정밀도를 높인다. 공격자가 무엇을 필사적으로 숨기려 했는지 파악하는 순간, 그것이 곧 사건의 핵심 고리(Critical Path)가 되기 때문이다.

결론적으로 안티 포렌식은 분석가를 괴롭히는 장애물이지만, 동시에 공격자의 수준과 의도를 파악할 수 있는 고급 정보원이다. 기술사는 기술적 한계에 부딪혔을 때 당황하지 않고, 남겨진 파편들을 모아 전체 그림을 완성하는 '디지털 프로파일러'의 자세를 가져야 한다.

- **📢 섹션 요약 비유**: 안티 포렌식은 '완벽한 범죄'를 꿈꾸는 공격자의 환상이다. 하지만 완벽한 범죄는 없으며, 지우려 했던 그 손길 자체가 가장 큰 흔적으로 남게 된다.
