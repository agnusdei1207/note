## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ArcSight는 세계 최초의 SIEM 중 하나로, 이기종 장비의 로그를 표준화(Normalization)하여 실시간 상관분석을 수행하는 강력한 엔진(ESM)을 보유한 정통파 솔루션이다.
> 2. **가치**: 초당 수십만 건(EPS)의 이벤트를 처리하는 성능과 복잡한 논리 구조의 상관분석 룰을 설계할 수 있는 유연성을 제공하여 대규모 보안 관제 센터의 표준이 되어왔다.
> 3. **판단 포인트**: 기술적 깊이와 커스터마이징 자유도는 높지만, 높은 운영 난이도와 인프라 관리 부담으로 인해 숙련된 전문 운영 인력이 필수적으로 요구된다.

---

## Ⅰ. 개요 및 필요성

ArcSight는 HP를 거쳐 현재 Micro Focus(OpenText)에 소속된 SIEM의 조상 격인 솔루션이다. 현대 SIEM의 핵심 기능인 로그 수집, 필터링, 정규화, 상관분석의 표준 공정을 정립했다. 특히 각기 다른 보안 장비의 로그를 ArcSight만의 공통 포맷(CEF: Common Event Format)으로 변환하는 기술은 업계 표준으로 자리 잡았다.

수많은 보안 장비가 쏟아내는 로그는 형식이 제각각이어서 통합 분석이 어렵다. ArcSight는 이를 '정규화'함으로써 "방화벽 로그와 백신 로그를 섞어서" 분석할 수 있게 해준다. 예를 들어 "방화벽에서 차단된 IP가 5분 뒤 웹 서버 로그에 나타나면 경고"와 같은 정교한 시나리오를 가능하게 한다.

- **📢 섹션 요약 비유**: ArcSight는 서로 다른 언어를 쓰는 사람들이 모인 회의장에서 모든 말을 공용어(CEF)로 동시통역해주는 베테랑 통역사와 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

ArcSight의 아키텍처는 SmartConnector(수집/정규화), Logger(저장), ESM(상관분석)의 3단계로 구성된다.

| 구성 요소 | 역할 | 상세 설명 |
|:---|:---|:---|
| SmartConnector | 수집 및 정규화 | 원천 로그 수집, 불필요 로그 필터링, CEF 포맷으로 변환(Normalization) |
| ArcSight Logger | 데이터 저장 | 정규화된 로그를 고속으로 저장 및 검색, 컴플라이언스 대응용 |
| ArcSight ESM | 실시간 상관분석 | 가장 핵심적인 엔진, 메모리 기반 실시간 룰 매칭 및 경보 발생 |

```text
┌──────────────────────────────────────────────────────────────┐
│           ArcSight ESM의 핵심: 상관분석 엔진 (CORR-Engine)      │
├──────────────────────────────────────────────────────────────┤
│ [Raw Logs] ──▶ [SmartConnector] ──▶ [ESM Filter] ──┐         │
│                                                   │         │
│ [Context Data] ───────────────────────────────────┴─▶ [Correlation]
│ (Assets, Zones, Users)                             (Real-time Rule)│
│                                                           │         │
│ [Security Alert] ◀────────────────────────────────────────┘         │
└──────────────────────────────────────────────────────────────┘
```

ArcSight의 정체성은 **ESM (Enterprise Security Manager)**에 있다. ESM은 들어오는 모든 이벤트를 메모리에 로드된 수천 개의 룰과 실시간으로 비교한다. 이때 단순 매칭뿐만 아니라 '상태 기반(Stateful)' 분석을 수행하여, 특정 순서나 시간 간격으로 발생하는 공격 패턴을 정확히 잡아낸다.

- **📢 섹션 요약 비유**: ArcSight 아키텍처는 거대한 필터가 달린 깔때기와 같다. 온갖 쓰레기(Raw Log)를 넣어도 깔때기 끝(SmartConnector)에서는 정제된 보석(CEF)이 나오고, 이를 검사관(ESM)이 실시간으로 확인한다.

## Ⅲ. 비교 및 연결

ArcSight는 최신 빅데이터 SIEM인 Splunk나 Elastic과 비교했을 때 '실시간 상관분석'의 정교함 면에서 여전히 강점을 가진다.

| 항목 | ArcSight (ESM) | Splunk / Elastic (Big Data) |
|:---|:---|:---|
| 분석 방식 | 실시간 스트림 상관분석 중심 | 저장 후 검색/분석 중심 (Search-based) |
| 데이터 포맷 | CEF 기반 엄격한 정규화 | 유연한 비정형 데이터 처리 |
| 룰 설계 | 매우 복잡하고 정교한 논리 가능 | 쿼리 언어 기반의 분석 |
| 주요 용도 | 실시간 위협 탐지 및 관제 | 사고 조사, 위협 헌팅, 비즈니스 분석 |

ArcSight는 최근 'ArcSight Recon'과 같은 차세대 플랫폼을 도입하며 빅데이터 처리 능력을 보강하고 있다. 또한 클라우드 기반 보안 분석 플랫폼인 'ArcSight SaaS'를 통해 현대적인 환경에 적응하고 있다.

- **📢 섹션 요약 비유**: ArcSight가 정해진 규칙에 따라 한 치의 오차 없이 움직이는 '정밀 시계'라면, Splunk는 어떤 질문에도 답할 수 있는 '슈퍼 컴퓨터'와 같다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 ArcSight 운영의 최대 난제는 '커넥터 관리'와 'ESM 성능 최적화'다. 커넥터가 멈추면 해당 장비의 가시성이 즉시 사라지며, 잘못 설계된 룰은 ESM의 메모리를 고갈시켜 시스템을 마비시킬 수 있다.

### 체크리스트
1. 각 장비의 로그 포맷 변경 시 SmartConnector의 파서(Parser)가 즉시 업데이트되는가?
2. ESM의 메모리 부하를 줄이기 위해 룰의 조건(Filter)을 최대한 구체적으로 설정했는가?
3. 로그 저장(Logger)과 실시간 분석(ESM) 사이의 부하 분산 전략이 마련되었는가?

### 안티패턴
- 모든 장비의 모든 로그를 정규화 없이 ESM으로 보내 분석 엔진을 과부하시키는 경우
- 벤더가 제공하는 기본 룰(Default Content)만 사용하고 조직 특유의 위협 시나리오를 반영하지 않는 경우

- **📢 섹션 요약 비유**: ArcSight 운영은 오케스트라 지휘와 같다. 수많은 악기(커넥터)가 제 소리를 내야 하고, 지휘자(운영자)는 ESM이라는 전체 사운드가 흐트러지지 않게 조율해야 한다.

## Ⅴ. 기대효과 및 결론

ArcSight는 검증된 안정성과 정교한 룰 설계 능력을 바탕으로, 국가기관이나 대형 금융사처럼 엄격한 보안 관제가 필요한 환경에서 그 가치를 발휘한다. 특히 CEF라는 표준을 통해 보안 생태계 전반의 통합을 주도해왔다는 점은 부인할 수 없는 공로다.

결론적으로 ArcSight는 '클래식의 힘'을 가진 솔루션이다. 비록 최근의 빅데이터 트렌드에 밀리는 모습도 있지만, 실시간 탐지의 정밀함과 체계적인 정규화 프로세스는 여전히 차세대 보안 관제의 핵심적인 참고 모델이 되고 있다.

- **📢 섹션 요약 비유**: ArcSight는 보안계의 '수동 변속기 스포츠카'다. 운전하기는 까다롭지만, 숙련된 드라이버가 잡으면 어떤 차보다 빠르고 정확하게 트랙(위협 탐지)을 완주한다.
