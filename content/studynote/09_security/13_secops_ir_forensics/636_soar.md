## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SOAR (Security Orchestration, Automation, and Response)는 서로 다른 보안 솔루션들을 하나의 워크플로우로 묶고(Orchestration), 반복적인 작업을 자동화하여(Automation), 사고에 신속하게 대응(Response)하는 기술이다.
> 2. **가치**: 보안 관제(SOC)의 경보 피로를 해소하고, 사람의 개입 없이도 침해 사고의 초기 분석과 차단을 수행하여 위협 대응 시간(MTTR)을 획기적으로 단축한다.
> 3. **판단 포인트**: 단순한 자동화 도구를 넘어, 조직의 보안 성숙도와 프로세스(Playbook)가 잘 정립되어 있어야 비로소 제 성능을 발휘하는 '운영 중심' 플랫폼이다.

---

## Ⅰ. 개요 및 필요성

SOAR (Security Orchestration, Automation, and Response)는 가트너가 제안한 개념으로, 보안 운영의 효율성을 극대화하기 위한 통합 플랫폼이다. 현대의 SOC는 하루에도 수만 건씩 쏟아지는 경보와 제각각인 보안 장비들 사이에서 '분석가 부족'과 '대응 지연'이라는 심각한 문제에 직면해 있다.

SOAR는 이러한 한계를 극복하기 위해 등장했다. SIEM이 위협을 '탐지'하는 데 집중한다면, SOAR는 탐지 이후의 '행동'에 집중한다. 반복적이고 정형화된 대응 업무를 자동화함으로써 분석가가 더 고도화된 위협 헌팅(Threat Hunting)에 집중할 수 있는 환경을 제공한다.

- **📢 섹션 요약 비유**: SOAR는 보안팀의 '지능형 오케스트라 지휘자'와 같다. 제각각 연주하던 악기(보안 장비)들을 하나의 악보(플레이북)에 맞춰 조화롭게 연주하게 만든다.

---

## Ⅱ. 아키텍처 및 핵심 원리

SOAR는 오케스트레이션, 자동화, 그리고 인시던트 관리의 세 가지 핵심 축으로 구성된다.

| 핵심 기능 | 역할 | 상세 설명 |
|:---|:---|:---|
| Orchestration | 이기종 솔루션 통합 | API를 통해 방화벽, EDR, 백신 등 서로 다른 장비를 연결 |
| Automation | 업무 프로세스 자동화 | 플레이북(Playbook)을 실행하여 사람의 개입 없이 작업 수행 |
| Response | 사고 대응 및 관리 | 티켓 생성, 증거 수집, 사고 분석 리포트 자동 생성 |

```text
┌──────────────────────────────────────────────────────────────┐
│             SOAR의 작동 메커니즘: 탐지에서 대응까지            │
├──────────────────────────────────────────────────────────────┤
│ [SIEM Alert] ──▶ [SOAR Platform] ──▶ [Playbook Execution]    │
│ (Threat Detected)│ (Case Open)    │ (Automated Steps)        │
│                        │                  │                  │
│ [Analyst Approval] ◀───┴── [Actions] ◀────┴─▶ [Security Tools]│
│ (Optional)           (Block IP, etc)      (FW, EDR, AD)      │
└──────────────────────────────────────────────────────────────┘
```

SOAR의 심장은 **플레이북 (Playbook)**이다. 이는 특정 보안 이벤트가 발생했을 때 수행해야 할 일련의 과정을 순서도 형태로 정의한 것이다. 예를 들어 '악성 메일 유입' 경보가 뜨면, SOAR는 자동으로 첨부파일의 해시를 확인(TI 연동)하고, 메일 수신자의 PC를 격리(EDR 연동)하며, 해당 메일을 전 직원의 편지함에서 삭제하는 과정을 순식간에 처리한다.

- **📢 섹션 요약 비유**: SOAR 아키텍처는 공장의 '자동화 라인'과 같다. 원재료(경보)가 들어오면 로봇 팔(자동화 엔진)이 정해진 순서(플레이북)대로 가공하고 포장하여 완성품(사고 해결)을 만들어낸다.

## Ⅲ. 비교 및 연결

SOAR를 이해하기 위해서는 기존 SIEM과의 차이점과 연계 방식을 알아야 한다.

| 항목 | SIEM (Security Info & Event Mgmt) | SOAR (Security Orch, Auto, Resp) |
|:---|:---|:---|
| 주 목적 | 위협 탐지 및 로그 통합 분석 | 사고 대응 및 업무 프로세스 효율화 |
| 핵심 기술 | 상관분석 (Correlation) | 플레이북 및 워크플로우 자동화 |
| 주요 데이터 | 로그 및 이벤트 (Logs/Events) | API 및 분석가 액션 (Actions) |
| 역할 비유 | "무슨 일이 일어났나?" 알려주는 감시자 | "어떻게 조치할까?" 수행하는 행동대장 |

SOAR는 SIEM에서 발생한 알람을 입력값으로 받아 실행된다. 또한 위협 인텔리전스(TI) 플랫폼과 연동하여 실시간으로 공격자의 정보를 확인하고, ITSM(예: ServiceNow) 솔루션과 연결하여 장애 티켓을 자동으로 관리하는 등 보안 생태계의 중앙 허브 역할을 수행한다.

- **📢 섹션 요약 비유**: SIEM이 "도둑이야!"라고 외치는 '경보 장치'라면, SOAR는 경찰에 신고하고 문을 잠그고 퇴로를 차단하는 '보안 출동 시스템'이다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 SOAR 도입 시 가장 흔한 실패 원인은 '프로세스 부재'다. 자동화할 업무 절차가 명확하지 않은 상태에서 도구만 도입하면 오히려 관리 부담만 가중된다.

### 체크리스트
1. 자동화할 보안 프로세스가 이미 표준 운영 절차(SOP)로 문서화되어 있는가?
2. 기존에 사용 중인 보안 솔루션들이 API 연동을 원활히 지원하는가?
3. '완전 자동화'와 '분석가 승인 후 실행' 단계 사이의 정책이 명확한가?

### 안티패턴
- 모든 업무를 한 번에 자동화하려는 시도 (단순하고 빈번한 업무부터 단계적 확장 필요)
- 플레이북 업데이트를 소홀히 하여 변화된 위협 환경에 대응하지 못하는 경우

- **📢 섹션 요약 비유**: SOAR 운영은 자율주행차를 모는 것과 같다. 목적지(대응 목표)와 경로(플레이북)가 명확해야 안전하게 도착할 수 있으며, 복잡한 교차로에서는 운전자의 판단이 여전히 중요하다.

---

## Ⅴ. 기대효과 및 결론

SOAR를 성공적으로 도입하면 위협 탐지 후 대응까지 걸리는 시간(MTTR)을 분 단위에서 초 단위로 줄일 수 있다. 이는 공격자가 네트워크 내부에서 활동할 기회를 박탈하는 결정적인 효과를 가진다. 또한 보안 운영의 수준을 일정하게 유지(Standardization)할 수 있다는 장점도 있다.

결론적으로 SOAR는 '사람 중심의 보안'을 '프로세스 중심의 보안'으로 진화시키는 핵심 도구다. 인공지능(AI) 기술이 결합되면서 스스로 플레이북을 추천하거나 상황에 맞는 최적의 대응책을 제시하는 지능형 SOAR로 발전하고 있다.

- **📢 섹션 요약 비유**: SOAR는 보안팀에게 '시간'이라는 가장 소중한 자원을 돌려주는 선물이다. 단순 반복 작업은 기계에게 맡기고, 사람은 더 가치 있는 생각에 집중하게 해준다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| Playbook | SOAR의 자동화 로직을 담은 표준 대응 절차 |
| MTTR | SOAR 도입을 통해 단축시켜야 할 핵심 지표 (Mean Time To Respond) |
| API | 이기종 솔루션 간의 오케스트레이션을 가능하게 하는 기술적 가교 |
| Threat Intelligence | 플레이북 내에서 악성 여부를 판단하는 주요 근거 데이터 |

### 👶 어린이를 위한 3줄 비유 설명

1. SOAR는 우리 집 보안 로봇들을 한꺼번에 조종하는 리모컨과 같아요.
2. 도둑이 오면 강아지 로봇은 짖고, 불은 켜지고, 문은 잠기게 미리 정해두는 거죠.
3. 로봇들이 알아서 착착 움직여주니까 엄마, 아빠가 훨씬 안심할 수 있답니다.
