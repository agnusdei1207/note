## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다이아몬드 모델 (Diamond Model)은 사이버 공격의 4가지 핵심 요소인 침입자(Adversary), 역량(Capability), 인프라(Infrastructure), 피해자(Victim) 사이의 상관관계를 다이아몬드 형태로 도식화한 분석 모델이다.
> 2. **가치**: 파편화된 공격 정보들을 4개의 꼭짓점으로 연결하여 공격자의 캠페인 흐름을 파악하고, 누락된 정보를 추론하거나 위협 인텔리전스를 고도화하는 데 기여한다.
> 3. **판단 포인트**: 사이버 킬 체인이 '공격의 시간적 흐름'에 집중한다면, 다이아몬드 모델은 '공격의 구성 요소와 인과 관계'에 집중하여 입체적인 분석을 가능하게 한다.

---

## Ⅰ. 개요 및 필요성

사이버 공격은 단순한 기술적 사건이 아니라 사회공학적, 인프라적 요소가 결합된 복합체다. 다이아몬드 모델은 이러한 공격의 본질을 4개의 꼭짓점으로 요약하여 보여준다. "누가(Adversary), 무엇을 가지고(Capability), 어디를 통해서(Infrastructure), 누구를(Victim) 공격했는가?"라는 질문에 답하는 과정이다.

보안 분석가는 사고 발생 시 모든 정보를 알 수 없다. 예를 들어, 피해자 정보와 사용된 악성코드(역량)는 알지만 공격자가 누구인지 모를 수 있다. 다이아몬드 모델은 이미 확보된 꼭짓점들 사이의 '축(Axis)'을 분석하여, 보이지 않는 나머지 꼭짓점(공격자 정체)을 찾아내거나 공격자의 다음 행보를 예측하는 데 필수적인 도구다.

- **📢 섹션 요약 비유**: 다이아몬드 모델은 사건 현장의 '수사 지도'와 같다. 범인, 흉기, 도주 경로, 피해자를 연결하여 보이지 않는 공범이나 배후를 찾아내는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

다이아몬드 모델은 4개의 핵심 요소(Node)와 이들을 잇는 관계(Edge)로 구성된다.

| 핵심 요소 | 설명 | 상세 예시 |
|:---|:---|:---|
| **Adversary (침입자)** | 공격을 주도하는 주체 | APT 그룹, 해커 개인, 국가 기관 등 |
| **Capability (역량)** | 공격에 사용된 도구와 기술 | 악성코드, 제로데이 취약점, 스피어피싱 기법 등 |
| **Infrastructure (인프라)** | 공격을 전달하고 제어하는 수단 | C2 서버, 도메인, 봇넷, 이메일 주소 등 |
| **Victim (피해자)** | 공격의 대상이 되는 조직이나 자산 | 특정 기업, 인사 정보 DB, 특정 국가의 기반 시설 등 |

```text
┌──────────────────────────────────────────────────────────────┐
│                Diamond Model of Intrusion Analysis           │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│                      [Adversary]                             │
│                         (Who)                                │
│                          / \                                 │
│                         /   \                                │
│           (Social)     /     \     (Technical)               │
│             Axis      /       \       Axis                   │
│                      /         \                             │
│          [Victim] ───           ─── [Infrastructure]         │
│           (Where)    \         /       (How)                 │
│                       \       /                              │
│                        \     /                               │
│                         \   /                                │
│                          \ /                                 │
│                      [Capability]                            │
│                        (What)                                │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

모델의 핵심 원리는 **'축(Axis)' 분석**이다.
- **기술적 축(Technical Axis)**: 역량과 인프라를 연결하여 공격자가 기술적으로 어떻게 공격을 수행하는지 분석한다.
- **사회적 축(Social Axis)**: 침입자와 피해자를 연결하여 공격의 동기와 목적, 사회적 관계를 분석한다.
이러한 연결을 통해 분석가는 "이 악성코드를 쓰고 이 서버를 사용하는 놈은 과거에 A사를 공격했던 B그룹이다"라는 결론에 도달할 수 있다.

- **📢 섹션 요약 비유**: 다이아몬드 모델은 '범죄의 삼각형'을 사각형(다이아몬드)으로 확장한 것이다. 사람, 도구, 장소, 대상이 연결되어야 비로소 사건의 실체가 드러난다.

## Ⅲ. 비교 및 연결

다이아몬드 모델은 사이버 킬 체인과 결합했을 때 가장 강력한 힘을 발휘한다.

| 항목 | 사이버 킬 체인 | 다이아몬드 모델 |
|:---|:---|:---|
| 분석 관점 | 프로세스 및 시계열 중심 (Linear) | 관계 및 상관관계 중심 (Relational) |
| 장점 | 공격의 진행 단계를 차단하기 좋음 | 공격자의 정체와 의도를 파악하기 좋음 |
| 단점 | 정적인 단계에 얽매일 수 있음 | 시간의 흐름을 표현하기 어려움 |
| 결합 방식 | **Activity Thread**: 킬 체인의 각 단계마다 다이아몬드 모델을 적용하여 분석 |

분석가는 사고가 진행되는 각 단계마다 다이아몬드 모델을 그려본다. 1단계(정찰)에서의 다이아몬드와 7단계(행동)에서의 다이아몬드를 연결하면, 공격자가 시간에 따라 어떻게 인프라를 바꾸고 역량을 진화시켰는지 '스토리라인'을 구성할 수 있다.

- **📢 섹션 요약 비유**: 킬 체인이 "영화의 타임라인"이라면, 다이아몬드 모델은 "장면마다 등장하는 인물 관계도"와 같다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사적 관점에서 다이아몬드 모델은 위협 인텔리전스(TI)의 '구조화'를 돕는다. 단순한 IoC 나열이 아니라, 관계 기반의 데이터를 축적하게 한다.

### 체크리스트
1. 침해 사고 보고서 작성 시 4가지 꼭짓점 중 우리가 놓치고 있는 정보는 무엇인가?
2. 특정 인프라(C2 서버) 차단 시, 공격자가 즉시 바꿀 수 있는 '역량'은 무엇인가?
3. 피해자(Victim)의 특성을 분석하여 공격자가 노리는 다음 타겟(유사 산업군)을 예측할 수 있는가?

### 안티패턴
- 4가지 요소 중 기술적인 부분(역량, 인프라)에만 치우쳐 공격자의 동기(Adversary)를 간과하는 경우
- 다이아몬드 모델을 일회성 사고 분석용으로만 쓰고, 그룹 추적을 위한 데이터베이스로 관리하지 않는 경우

- **📢 섹션 요약 비유**: 다이아몬드 모델 운영은 '점 잇기 게임'과 같다. 흩어진 점들을 연결하다 보면 결국 무서운 괴물(공격자)의 전체 모습이 그려지게 된다.

---

## Ⅴ. 기대효과 및 결론

다이아몬드 모델을 통해 보안 조직은 사고를 '점'이 아닌 '면'으로 이해할 수 있다. 이는 공격자에 대한 깊이 있는 이해를 바탕으로, 단순 차단을 넘어선 전략적 기만(Deception)이나 법적 대응까지 고려할 수 있는 토대가 된다.

결론적으로 다이아몬드 모델은 사이버 위협 분석의 '분석적 틀'이다. 복잡한 침해 사고의 실타래를 4가지 핵심 요소로 풀어냄으로써, 정보 보안을 보다 과학적이고 체계적인 학문의 영역으로 끌어올렸다.

- **📢 섹션 요약 비유**: 다이아몬드 모델은 보안팀에게 '입체 안경'을 주는 것과 같다. 평면적인 로그 데이터 속에서 공격의 입체적인 구조를 보게 해준다.

2. "누가, 무엇으로, 어디서, 누구를" 괴롭혔는지 선으로 연결해서 그림을 그려보는 거죠.
3. 이 그림을 완성하면 나쁜 사람이 어디에 숨어있는지 훨씬 찾기 쉬워진답니다.
