## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ATT&CK 매트릭스 (Matrix)는 MITRE ATT&CK 프레임워크의 전술(Tactics)을 열(Column)로, 기법(Techniques)을 행(Row)으로 배치하여 공격자의 전체 행위를 한눈에 조망할 수 있게 만든 시각화 도표다.
> 2. **가치**: 공격의 시작부터 목표 달성까지의 전 과정을 논리적 흐름에 따라 배치함으로써, 방어 체계의 탐지 가시성(Visibility)과 방어 수준을 히트맵(Heatmap) 형태로 측정하게 돕는다.
> 3. **판단 포인트**: 매트릭스는 단순한 목록이 아니라, 각 조직의 위협 모델링 결과에 따라 우선순위가 높은 기법들을 선별하고 집중 관리하기 위한 '전략적 대시보드'로 활용되어야 한다.

---

## Ⅰ. 개요 및 필요성

ATT&CK 매트릭스 (Matrix)는 방대한 공격 기법 데이터를 보안 전문가들이 실무에서 즉시 활용할 수 있도록 구조화한 결과물이다. 수백 개의 기법이 무작위로 나열되어 있다면 분석가는 어디서부터 손을 대야 할지 알 수 없지만, 매트릭스는 이를 '공격자의 목표(전술)'별로 정렬하여 사고의 체계를 잡아준다.

조직은 매트릭스를 통해 "우리가 초기 침투는 잘 막지만, 내부에서의 권한 상승 탐지는 약하구나"와 같은 객관적인 자기 진단을 할 수 있다. 이는 보안 예산과 인력을 어디에 우선적으로 투입해야 할지에 대한 명확한 근거 데이터가 된다.

- **📢 섹션 요약 비유**: 매트릭스는 보안계의 '주기율표'와 같다. 수많은 원소(공격 기법)를 성질별로 정리하여 화학 반응(공격 흐름)을 예측하고 대비할 수 있게 해준다.

---

## Ⅱ. 매트릭스 구조 및 핵심 원리

매트릭스는 왼쪽(공격 시작)에서 오른쪽(공격 목표 달성)으로 이어지는 시간적/논리적 흐름을 가진다.

| 구성 요소 | 설명 | 특징 |
|:---|:---|:---|
| **Tactics (열)** | 공격자의 상위 목표 | Reconnaissance부터 Impact까지 총 14개 전술 (Enterprise 기준) |
| **Techniques (행)** | 목표 달성을 위한 구체적 기법 | 각 전술 하위에 배치되며, 중복 배치될 수도 있음 |
| **Cells (셀)** | 개별 기법 항목 | 각 셀은 고유 ID(Txxxx)를 가지며 상세 페이지와 연결됨 |

```text
┌──────────────────────────────────────────────────────────────┐
│           Enterprise ATT&CK Matrix (간략화된 구조)             │
├──────────────────────────────────────────────────────────────┤
│ Initial  │ Execution │ Persistence │ ... │ Exfiltration │ Impact │
│ Access   │           │             │     │              │        │
├──────────┼───────────┼─────────────┼─────┼──────────────┼────────┤
│ Phishing │ Command   │ Registry    │ ... │ Over C2      │ Data   │
│ (T1566)  │ Line      │ Keys        │     │ Channel      │ Encrypt│
├──────────┼───────────┼─────────────┼─────┼──────────────┼────────┤
│ Valid    │ Scripting │ Create      │ ... │ ...          │ ...    │
│ Accounts │           │ Account     │     │              │        │
└──────────┴───────────┴─────────────┴─────┴──────────────┴────────┘
```

매트릭스의 핵심 원리는 **'상호 연관성'**이다. 특정 공격 그룹(Group)이나 악성코드(Software)를 선택하면 매트릭스 상에서 그들이 사용하는 기법들이 하이라이트된다. 이를 통해 공격자의 '시그니처 행위'를 시각적으로 파악할 수 있다.

- **📢 섹션 요약 비유**: 매트릭스 구조는 대형 마트의 '진열대'와 같다. 신선식품, 가전, 의류 등으로 코너(전술)가 나뉘어 있고, 각 코너 안에 구체적인 상품(기법)들이 진열되어 있어 필요한 것을 바로 찾을 수 있다.

## Ⅲ. 비교 및 연결

MITRE는 환경에 따라 여러 종류의 매트릭스를 제공하며, 이를 통합하여 보는 것이 중요하다.

| 매트릭스 종류 | 대상 환경 | 주요 특징 |
|:---|:---|:---|
| **Enterprise** | Windows, macOS, Linux, Cloud, Network | 가장 널리 쓰이는 표준 매트릭스 |
| **Mobile** | Android, iOS | 모바일 환경 특화 기법 (권한 요청, 앱 변조 등) |
| **ICS** | 산업 제어 시스템 (OT) | 제어 장치 조작, 물리적 파괴 관련 기법 중심 |

조직은 **ATT&CK Navigator**라는 도구를 사용하여 이 매트릭스 위에 우리 회사의 탐지 룰(SIEM) 적용 현황이나 취약점 점검 결과를 레이어(Layer)로 겹쳐서 분석할 수 있다. 이는 개별 보안 솔루션의 성능을 넘어 조직 전체의 '통합 방어 역량'을 시각화하는 과정이다.

- **📢 섹션 요약 비유**: 여러 매트릭스를 보는 것은 '종합 검진 리포트'를 보는 것과 같다. 내과, 외과, 치과 검진 결과가 다 모여야 내 몸의 전체 상태를 알 수 있는 것과 같다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 매트릭스를 활용할 때 가장 주의할 점은 '양보다 질'이다. 모든 셀을 초록색(탐지 가능)으로 칠하는 것이 목표가 되어서는 안 된다.

### 체크리스트
1. 우리 산업군을 노리는 공격 그룹들이 주로 사용하는 **'Top 10 기법'**을 매트릭스에서 식별했는가?
2. 매트릭스 상의 특정 기법에 대해 '탐지 가능'하다고 판단할 때, 그 근거(로그 소스, 룰 번호)가 명확한가?
3. 새로운 보안 장비를 도입할 때, 매트릭스의 어떤 빈칸(Gap)을 채워줄 수 있는지 평가하는가?

### 안티패턴
- 매트릭스를 단순히 보고용 '색칠 공부' 도구로 전락시켜 실질적인 탐지 로직 개선이 없는 경우
- 매트릭스의 기법 이름만 보고 실제 공격 원리를 이해하지 못한 채 방어 계획을 세우는 경우

- **📢 섹션 요약 비유**: 매트릭스 활용은 '바둑판' 위에서 수 싸움을 하는 것과 같다. 상대방(공격자)이 주로 두는 수(기법)를 읽고, 나의 돌(방어 체계)을 어디에 배치할지 전략적으로 결정해야 승리한다.

## Ⅴ. 기대효과 및 결론

ATT&CK 매트릭스는 보안 관제의 '가시성 혁명'을 가져왔다. 막연하게 "우리는 안전합니다"라고 말하는 대신, 매트릭스를 통해 "우리는 이러한 기법들에 대해 이 정도의 탐지 역량을 갖추고 있습니다"라고 데이터로 증명할 수 있게 되었다.

결론적으로 매트릭스는 보안팀의 '작전 지도'다. 적의 이동 경로를 예측하고 아군의 취약 지점을 보완하게 함으로써, 정보 보안을 예술의 영역에서 데이터 기반의 엔지니어링 영역으로 끌어올렸다.

- **📢 섹션 요약 비유**: 매트릭스는 보안팀에게 '엑스레이 사진'을 찍어주는 것과 같다. 겉으로는 멀쩡해 보여도 어디가 부러졌는지(보안 구멍)를 정확히 짚어내어 수술(보안 강화)하게 해준다.

|:---|:---|

### 👶 어린이를 위한 3줄 비유 설명

1. 매트릭스는 나쁜 사람들이 쓰는 나쁜 짓 기술들을 종류별로 모아놓은 '커다란 표'예요.
2. 어떤 기술은 빨간색, 어떤 기술은 초록색으로 칠해서 우리가 뭘 잘 막는지 한눈에 볼 수 있죠.
3. 이 표를 보고 있으면 어디에 성벽을 더 튼튼하게 쌓아야 할지 똑똑하게 결정할 수 있답니다.
