## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고통의 피라미드 (Pyramid of Pain)는 침해 사고 지표(IoC)의 유형을 6단계로 나누고, 각 지표를 변경할 때 공격자가 느끼는 '고통(난이도)'의 정도를 시각화한 모델이다.
> 2. **가치**: 방어자가 하단 지표(IP, Hash) 차단에만 머물지 않고, 상단 지표(TTP) 탐지에 집중하게 함으로써 공격자의 공격 비용을 극대화하는 전략적 가이드를 제공한다.
> 3. **판단 포인트**: 피라미드의 위로 올라갈수록 방어는 어려워지지만, 일단 성공하면 공격자의 공격 인프라 전체를 무력화할 수 있는 강력한 방어 레버리지를 갖게 된다.

---

## Ⅰ. 개요 및 필요성

보안 대응의 효율성은 "우리가 얼마나 공격자를 힘들게 하느냐"에 달려 있다. 데이비드 비앙코(David Bianco)가 제안한 고통의 피라미드는 방어자가 사용하는 지표들이 공격자에게 실제로 어느 정도의 타격을 주는지 명확히 보여준다.

지금까지 많은 보안 팀은 파일 해시나 IP 주소를 차단하는 데 대부분의 시간을 써왔다. 하지만 공격자는 클릭 한 번으로 해시를 바꾸고, 초 단위로 IP를 변경할 수 있다. 즉, 방어자가 1시간 동안 분석해서 차단한 지표를 공격자는 1초 만에 무력화할 수 있는 구조였다. 고통의 피라미드는 이러한 '방어의 비대칭성'을 깨고, 공격자가 바꾸기 힘든 본질적인 습성(TTP)을 공략해야 한다고 강조한다.

- **📢 섹션 요약 비유**: 고통의 피라미드는 '범인을 막는 방법의 등급'과 같다. 범인의 옷차림(해시)을 외우는 건 쉽지만 범인이 옷을 갈아입으면 끝이다. 하지만 범인의 특이한 걸음걸이(TTP)를 알아내면 어떤 옷을 입어도 잡을 수 있고, 범인은 걸음걸이를 바꾸느라 엄청난 고통을 겪게 된다.

## Ⅱ. 피라미드 계층 구조 및 핵심 원리

피라미드는 하단에서 상단으로 갈수록 공격자가 지표를 수정하는 데 드는 비용과 노력이 기하급수적으로 증가한다.

| 단계 | 지표 유형 | 고통의 정도 | 설명 |
|:---|:---|:---|:---|
| **6 (Top)** | **TTPs** | **가장 높음 (Infectious)** | 공격자의 전술, 기법, 절차. 공격자의 근본적인 행동 양식. |
| **5** | **Tools** | **높음 (Challenging)** | 공격자가 사용하는 소프트웨어 (백도어, 스캐너 등). |
| **4** | **Host Artifacts** | **보통 (Annoying)** | 공격자가 시스템에 남기는 흔적 (레지스트리, 파일명 등). |
| **3** | **Network Artifacts** | **보통 (Annoying)** | 네트워크 트래픽 특성 (User-Agent, 통신 패턴 등). |
| **2** | **IP Addresses** | **낮음 (Simple)** | 공격자가 사용하는 서버의 IP 주소. |
| **1 (Bottom)** | **Hash Values** | **매우 낮음 (Trivial)** | 특정 파일의 고유한 지문 (MD5, SHA-1 등). |

```text
┌──────────────────────────────────────────────────────────────┐
│                    The Pyramid of Pain                       │
├──────────────────────────────────────────────────────────────┤
│                /             \                               │
│               /     TTPs      \  <-- 고통의 정점 (심장 타격)    │
│              /─────────────────\                             │
│             /      Tools        \                            │
│            /─────────────────────\                           │
│           /    Host Artifacts     \                          │
│          /─────────────────────────\                         │
│         /    Network Artifacts      \                        │
│        /─────────────────────────────\                       │
│       /        IP Addresses           \                      │
│      /─────────────────────────────────\                     │
│     /          Hash Values              \ <-- 가장 쉬운 대응    │
│    └─────────────────────────────────────┘                   │
└──────────────────────────────────────────────────────────────┘
```

이 모델의 핵심 원리는 **'공격 비용의 전가'**다.
- 하단(1~2단계) 대응: 방어자도 쉽지만 공격자도 너무 쉽다. 무의미한 소모전이 될 가능성이 크다.
- 상단(5~6단계) 대응: 방어자는 고도의 분석 역량이 필요하지만, 성공 시 공격자는 공격 도구를 새로 개발하거나 공격 전략 자체를 수정해야 하므로 공격을 포기하게 만들 수 있다.

- **📢 섹션 요약 비유**: 피라미드 하단은 '모기 파리채'로 모기를 잡는 것이고, 상단은 '고인 물(서식지)을 없애는 것'과 같다. 파리채는 계속 휘둘러야 하지만, 서식지를 없애면 모기 자체가 사라진다.

## Ⅲ. 비교 및 연결

고통의 피라미드는 위협 인텔리전스(TI)의 성숙도를 측정하는 잣대가 된다.

| 항목 | 낮은 성숙도 (Low Maturity) | 높은 성숙도 (High Maturity) |
|:---|:---|:---|
| 주요 지표 | 해시 리스트, 블랙 IP 리스트 | MITRE ATT&CK 기반 TTP 분석 |
| 대응 방식 | 단순 차단 (Blacklisting) | 행위 기반 탐지 및 헌팅 |
| 도구 활용 | 방화벽, 백신 (AV) | EDR, NDR, 분석 샌드박스 |
| 공격자 반응 | "IP 하나 더 사지 뭐" | "우리의 공격 방식이 다 들켰어, 철수하자" |

최근의 **EDR(Endpoint Detection and Response)** 솔루션들은 단순 파일 검사를 넘어 프로세스의 행위(TTP)를 분석함으로써 고통의 피라미드 상단을 공략하려 노력한다. 또한 **MITRE ATT&CK**은 피라미드의 최상단인 TTP를 표준화된 언어로 설명해주는 훌륭한 파트너다.

- **📢 섹션 요약 비유**: 낮은 성숙도는 '성벽에 붙은 대자보'만 떼어내는 수준이고, 높은 성숙도는 '적군의 암호 체계와 작전 수첩'을 훔쳐보는 수준이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사적 관점에서 피라미드의 교훈은 '리소스 배분의 최적화'다.

### 체크리스트
1. 우리 조직의 보안 관제 인력이 전체 시간 중 몇 %를 IP/해시 차단에 쓰고 있는가?
2. 침해 사고 분석 후 생성되는 보고서에 공격자의 TTP(상단 지표)가 명확히 기술되어 있는가?
3. 보안 장비 도입 시 "이 장비는 피라미드의 어느 단계를 방어해주는가?"를 질문하는가?

### 안티패턴
- 수백만 건의 위협 피드(IoC)를 구독하면서 정작 공격자가 어떤 도구(Tool)를 쓰는지 모르는 경우
- TTP 탐지가 중요하다는 명분하에, 가장 기초적인 해시/IP 차단을 소홀히 하여 쉬운 공격에 뚫리는 경우 (기초 체력은 기본임)

- **📢 섹션 요약 비유**: 피라미드 운영은 '축구 수비'와 같다. 공(IP/해시)만 보고 쫓아다니면 금방 지치지만, 상대 팀의 전술(TTP)을 읽고 길목을 지키면 적은 힘으로도 실점을 막을 수 있다.

---

## Ⅴ. 기대효과 및 결론

고통의 피라미드는 방어자에게 '전략적 우위'를 선사한다. 공격자가 느끼는 고통의 크기가 커질수록 우리의 보안은 견고해진다. 이는 단순히 기술적인 우위를 넘어, 공격자의 경제적 동기(수익성)를 꺾어버리는 가장 강력한 심리적 방어선이 된다.

결론적으로 고통의 피라미드는 보안 팀이 나아가야 할 '북극성'이다. 우리가 오늘 처리한 경보가 피라미드의 어디에 해당하는지 자문하는 습관이, 우리 조직의 보안 수준을 근본적으로 바꾸는 시작점이 될 것이다.

- **📢 섹션 요약 비유**: 고통의 피라미드는 보안팀에게 '진정한 승리의 길'을 알려준다. 적을 단순히 쫓아내는 것이 아니라, 적이 다시는 우리를 넘보지 못하게 만드는 강력한 한 방(TTP 대응)을 준비하게 한다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| IoC | 피라미드를 구성하는 모든 수준의 침해 지표들 |
| TTP | 피라미드의 정점이자 방어자가 도달해야 할 최종 목적지 |
| MTTR | 상단 지표 대응을 통해 궁극적으로 단축시켜야 할 지표 |
| False Positive | 상단으로 올라갈수록 정교해지지만 오탐의 위험도 커짐을 유의 |

### 👶 어린이를 위한 3줄 비유 설명

1. 고통의 피라미드는 나쁜 사람을 얼마나 힘들게 만드는지 순서를 매긴 거예요.
2. 나쁜 사람이 쓴 가면을 뺏는 건 쉽지만(해시), 나쁜 사람의 나쁜 습관을 고치게 하는 건 아주 어렵죠(TTP).
3. 나쁜 사람의 나쁜 습관을 딱 잡아내면, 그 사람은 우리를 괴롭히는 걸 포기하게 된답니다.
