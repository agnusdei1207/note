## 핵심 인사이트 (3줄 요약)
> 1. **본질**: OWASP Testing Guide(WSTG)는 웹 애플리케이션 및 서비스의 보안 취약점을 체계적으로 테스트하기 위한 가장 권위 있는 오픈소스 프레임워크다.
> 2. **가치**: 개발자에게는 안전한 코딩 기준을, 테스터에게는 누락 없는 진단 체크리스트를 제공하여 상호 운용 가능한 보안 수준을 확보한다.
> 3. **판단 포인트**: 단순 취약점 나열이 아닌, 인증, 인가, 데이터 검증 등 12개 카테고리별로 심층적인 테스트 방법론을 제시하여 실무적 대응력을 높인다.

---

## Ⅰ. 개요 및 필요성

OWASP Testing Guide (현재 WSTG - Web Security Testing Guide로 불림)는 웹 애플리케이션의 보안 상태를 평가하는 업계 표준 가이드다. 웹은 공격 표면이 가장 넓고 빈번하게 공격받는 지점이지만, 기술적 변화가 빨라 일관된 보안 기준을 유지하기 어렵다. WSTG는 이러한 환경에서 전문가들이 따라야 할 구체적인 테스트 케이스를 정의한다.

이 가이드가 필요한 이유는 보안 진단의 '누락'을 방지하기 위해서다. 테스터의 역량에 의존하기보다 정형화된 가이드를 따름으로써, 신입 테스터도 일정 수준 이상의 진단 품질을 보장할 수 있게 된다.

- **📢 섹션 요약 비유**: OWASP Testing Guide는 자동차 정비사가 엔진부터 바퀴까지 꼼꼼히 살피기 위해 사용하는 '정밀 검사 항목표'와 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

WSTG는 웹 애플리케이션의 수명 주기(SDLC) 전반에 걸친 테스트를 강조하며, 특히 릴리스 전 단계의 동적 테스트(DAST)에 강점이 있다. 현재 가이드는 약 100개 이상의 테스트 항목을 12개의 논리적 카테고리로 분류한다.

| 카테고리 (주요 예시) | 테스트 목표 | 핵심 점검 항목 |
|:---|:---|:---|
| 정보 수집 (INFO) | 타겟의 아키텍처 파악 | 핑거프린팅, 주석 노출, 경로 추측 |
| 구성 및 배포 (CONFIG) | 서버 및 프레임워크 설정 오류 | 관리자 페이지 노출, SSL 설정 |
| 신원 관리 (IDNT) | 사용자 식별 프로세스 | 계정 열거 가능성, 약한 비밀번호 |
| 인증 (AUTH) | 로그인 메커니즘 검증 | 무차별 대입 공격, 세션 고정 |
| 데이터 검증 (INPVAL) | 비정상 입력 값 처리 | SQLi, XSS, 명령어 인젝션 |

```text
┌──────────────────────────────────────────────────────────────┐
│             WSTG 테스트 프레임워크 (DAST 중심)               │
├──────────────────────────────────────────────────────────────┤
│ [Request] ──▶ [Payload Injection] ──▶ [Response Analysis]    │
│      ▲                  │                      │             │
│      └──────────────────┴── 취약점 판정 ───────┘             │
│                                                              │
│  * 원리: 입력값에 따른 애플리케이션의 행위 변화를 관찰         │
└──────────────────────────────────────────────────────────────┘
```

WSTG의 핵심 원리는 '블랙박스'와 '그레이박스' 테스트의 조합이다. 소스 코드를 보지 않고 외부에서 공격을 시도하는 방식과, 일부 정보를 알고 효율적으로 접근하는 방식을 모두 수용한다.

- **📢 섹션 요약 비유**: WSTG는 요리사가 음식을 만들기 전에 주방의 위생 상태부터 조리 도구의 날카로움까지 하나하나 체크하는 '주방 안전 가이드'와 같다.

## Ⅲ. 비교 및 연결

WSTG는 OWASP Top 10 및 ASVS(Application Security Verification Standard)와 밀접하게 연결된다.

| 항목 | OWASP Top 10 | WSTG (Testing Guide) | ASVS |
|:---|:---|:---|:---|
| 성격 | 인식 제고용 (가장 위험한 10개) | 실무 테스트 방법론 | 요구사항 및 수준 정의 |
| 목적 | "무엇이 위험한가?" | "어떻게 테스트하는가?" | "어떤 수준을 지켜야 하는가?" |
| 활용 | 교육 및 정책 수립 | 실제 진단 및 모의 해킹 | 설계 및 검증 표준 |

WSTG는 Top 10에서 정의한 위험 요소들을 실제로 어떻게 찾아내는지에 대한 기술적 'How-to'를 제공한다.

- **📢 섹션 요약 비유**: Top 10이 '조심해야 할 10대 질병'이라면, WSTG는 그 질병들을 찾아내기 위한 '정밀 검진 방법론'이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사 관점에서 WSTG는 단순한 체크리스트가 아니라 '보안 품질 보증(QA)'의 도구다. 실무에서는 프로젝트의 규모와 중요도에 따라 WSTG 항목 중 필수 항목을 선별하여 테스팅 프로파일을 구성해야 한다.

### 체크리스트
1. 테스트 범위에 API 및 마이크로서비스 엔드포인트가 포함되었는가?
2. 최신 버전의 WSTG(v4.2 등)에서 추가된 신규 공격 기법(예: SSRF)이 반영되었는가?
3. 자동화 도구로 탐지 불가능한 '비즈니스 로직 취약점' 테스트가 계획되었는가?

### 안티패턴
- 가이드의 모든 항목을 기계적으로 테스트하느라 중요한 로직 분석을 놓치는 경우
- 예전 버전 가이드에만 의존하여 최신 웹 기술(SPA, GraphQL) 보안을 무시하는 경우

- **📢 섹션 요약 비유**: WSTG는 내비게이션과 같다. 길을 알려주지만, 실제 운전(판단)은 테스터가 상황에 맞춰 해야 한다.

---

## Ⅴ. 기대효과 및 결론

WSTG를 적용하면 웹 서비스의 보안 가시성이 극대화된다. 정량화된 테스트 결과를 통해 보안 부채를 관리할 수 있으며, 개발 단계에서의 피드백 루프를 강화하여 'Secure SDLC'를 실현할 수 있다.

결론적으로 OWASP Testing Guide는 웹 보안의 '공용 언어'다. 이를 통해 테스터, 개발자, 경영진은 동일한 기준 위에서 보안 리스크를 논의하고 해결할 수 있다.

- **📢 섹션 요약 비유**: 모든 정비사가 같은 매뉴얼을 쓰면 어느 정비소에 가도 차가 안전해지듯이, WSTG는 웹 보안의 신뢰도를 높여준다.
