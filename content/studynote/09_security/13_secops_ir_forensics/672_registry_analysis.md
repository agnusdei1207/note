## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 윈도우 레지스트리 분석은 운영체제와 하이퍼바이저, 어플리케이션의 설정 정보와 사용자 활동 이력이 저장된 계층형 데이터베이스를 조사하여 행위 증거를 추출하는 과정이다.
> 2. **가치**: 시스템 부팅 시 자동 실행되는 프로그램(Run keys), 최근 열어본 파일(MRU), USB 연결 흔적 등 공격자의 영속성 유지(Persistence) 및 행적을 파악할 수 있는 결정적 단서를 제공한다.
> 3. **판단 포인트**: 레지스트리 하이브(Hive) 파일은 시스템 종료 후에도 디스크에 남으므로, 휘발성 메모리 덤프와 디스크 내 하이브 파일을 비교하여 공격자의 레지스트리 조작 여부를 검증해야 한다.

---

## Ⅰ. 개요 및 필요성

윈도우 레지스트리는 시스템의 모든 설정과 사용자 활동이 기록되는 '정보의 보고'다. 윈도우는 부팅 시점부터 종료 시점까지 거의 모든 동작을 레지스트리에 반영한다. 따라서 레지스트리 분석은 사고 발생 시 공격자가 어떤 악성 코드를 자동으로 실행되게 만들었는지, 어떤 파일을 훔쳐보았는지, 어떤 외부 장치를 연결했는지 재구성하는 데 필수적이다.

특히 파일 시스템 분석만으로는 알기 어려운 '사용자의 의도'와 '도구 사용 이력'이 레지스트리 곳곳에 분산되어 저장되어 있어, 이를 통합하여 분석하는 능력이 침해 사고 분석의 성패를 좌우한다.

- **📢 섹션 요약 비유**: 레지스트리는 비행기의 '계기판 설정'과 '승객 명단'이 합쳐진 데이터베이스와 같다. 비행기가 어떻게 설정되어 날았고, 승객이 무엇을 주문했는지 모든 기록이 담겨 있다.

## Ⅱ. 아키텍처 및 핵심 원리

레지스트리는 논리적으로는 트리(Tree) 구조를 가지며, 물리적으로는 하이브(Hive)라고 불리는 파일들에 나뉘어 저장된다.

### 레지스트리 하이브 파일과 경로

| 하이브 파일명 | 물리적 경로 | 논리적 루트 (Root Key) | 주요 정보 |
|:---|:---|:---|:---|
| **SYSTEM** | `%SystemRoot%\System32\config` | HKLM\SYSTEM | 드라이버 설정, 서비스 목록, 시간 설정 |
| **SOFTWARE** | `%SystemRoot%\System32\config` | HKLM\SOFTWARE | 설치된 프로그램 정보, OS 버전 |
| **SAM / SECURITY**| `%SystemRoot%\System32\config` | HKLM\SAM / SECURITY | 계정 정보, 그룹 정책, 권한 설정 |
| **NTUSER.DAT** | `%UserProfile%` | HKCU | 사용자별 MRU, 배경화면, 개인 설정 |

### 주요 포렌식 아티팩트 (Key Artifacts)

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│ [주요 레지스트리 키 및 분석 가치]                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│ 1. Persistence (영속성): Run, RunOnce 키를 통해 악성코드 자동 실행 확인         │
│    - Path: HKLM\Software\Microsoft\Windows\CurrentVersion\Run               │
│                                                                             │
│ 2. Recent Files (MRU): 최근 열어본 문서 및 실행한 명령 추적                   │
│    - Path: HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\RunMRU    │
│                                                                             │
│ 3. USB Devices: 연결되었던 모든 USB 장치의 Serial Number 및 제조사 확인        │
│    - Path: HKLM\System\CurrentControlSet\Enum\USBSTOR                       │
│                                                                             │
│ 4. User Assist: GUI 환경에서 실행된 프로그램의 실행 횟수 및 마지막 시각         │
│    - Path: HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\UserAssist │
└─────────────────────────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: 레지스트리 하이브는 '집안의 각 방'과 같다. 거실(System)에는 공통 가구가 있고, 각자의 방(NTUSER.DAT)에는 개인 소지품이 숨겨져 있는 구조다.

## Ⅲ. 비교 및 연결

레지스트리 분석은 프리페치(Prefetch)나 링크(LNK) 파일 분석과 결합될 때 증거의 신뢰성이 극대화된다.

| 항목 | 레지스트리 분석 | 프리페치 분석 (Prefetch) | 링크 파일 분석 (LNK) |
|:---|:---|:---|:---|
| 증거 성격 | 설정 및 설정된 행위 기록 | 프로그램 실행 성능 향상 기록 | 파일 접근에 대한 바로가기 흔적 |
| 시간 정보 | Last Write Time (키 단위) | 실행 시각 (최대 8회 기록) | 생성/수정/접근 시각 |
| 주요 강점 | 시스템 전반의 상태 파악 | 특정 앱의 실행 여부 입증 | 원본 파일의 위치 및 장치 추적 |
| 연결 개념 | `670. 타임라인 분석` | `671. MFT 분석` | `663. 증거 보전` |

- **📢 섹션 요약 비유**: 레지스트리가 '범행 계획서'와 '영수증'이라면, 프리페치는 'CCTV에 찍힌 범인의 모습'이고, 링크 파일은 '범인이 남긴 발자국'이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 레지스트리 정보가 실시간으로 변한다는 점을 인지하고, 수집 시점의 일관성을 판단해야 한다.

### 실무 판단 가이드

1. **로그인 기록 분석**: `SAM` 하이브와 `SYSTEM` 하이브의 로그 정보를 대조하여 공격자가 원격 데스크톱(RDP)으로 접속한 흔적이 있는지 판단했는가?
2. **악성 서비스 탐지**: `Services` 키 아래에 생소한 이름이나 무작위 문자열로 등록된 서비스가 있는지, 실행 경로가 `%Temp%`와 같은 의심스러운 곳은 아닌지 확인했는가?
3. **사용자 활동 재구성**: `UserAssist` 키의 ROT13 암호화를 해제하여 실제 실행된 프로그램 명칭을 복원했는가?
4. **USB 유출 흔적**: `USBSTOR` 키에서 발견된 장치와 `setupapi.dev.log` 파일을 대조하여 특정 데이터가 유출된 시점을 특정할 수 있는가?

### 안티패턴
- 레지스트리 편집기(regedit)를 실시간 시스템에서 직접 열어 확인하는 행위 (Last Write Time 오염)
- 특정 키의 삭제 흔적(Deleted Key)을 무시하는 행위 (하이브 파일 내부 슬랙 공간에 남아있을 수 있음)

- **📢 섹션 요약 비유**: 레지스트리 분석 실무는 '지문 채취'와 같다. 직접 손을 대지 않고 조심스럽게 하이브 파일을 복제(Imaging)한 뒤 전용 도구(Registry Explorer 등)로 분석해야 한다.

## Ⅴ. 기대효과 및 결론

레지스트리 분석은 윈도우 환경 사고 분석의 '스모킹 건(Smoking Gun)'을 찾아내는 과정이다. 시스템 설정 변경을 통한 백도어 설치나 사용자 계정 탈취의 흔적을 가장 명확하게 보여주며, 복잡한 공격 시나리오를 단순화하여 설명할 수 있게 해준다.

결론적으로 레지스트리는 시스템의 과거와 현재가 공존하는 거대한 기록소다. 기술사는 레지스트리의 구조적 특성과 각 키의 의미를 통달하여, 보이지 않는 공격자의 움직임을 가시화하는 능력을 갖춰야 한다.

- **📢 섹션 요약 비유**: 레지스트리는 컴퓨터의 '기억 장치'다. 지워진 기억이라도 최면(포렌식 분석)을 걸어 다시 떠올리게 하면 사건의 진실이 드러난다.

| 개념 | 연결 포인트 |
|:---|:---|
| Hive File | 레지스트리 데이터를 물리적으로 저장하는 바이너리 파일 포맷 |

| ShimCache | 어플리케이션 호환성을 위해 관리되는 레지스트리 키. 실행 흔적 분석에 유용 |

### 👶 어린이를 위한 3줄 비유 설명

1. 컴퓨터가 자기가 무슨 일을 했는지 적어두는 '비밀 수첩' 같은 곳이에요.
2. 어떤 장난감을 가지고 놀았는지, 어떤 친구(USB)가 놀러 왔었는지 다 적혀 있어요.
3. 이 수첩을 잘 읽어보면 나쁜 사람이 몰래 컴퓨터를 켜서 무엇을 했는지 다 알 수 있답니다.
