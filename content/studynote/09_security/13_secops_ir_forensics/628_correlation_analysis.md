## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 상관 분석 (Correlation Analysis)은 서로 다른 시간, 장소, 장비에서 발생한 개별 보안 이벤트를 논리적 인과 관계로 엮어 복합적인 위협을 탐지하는 기술이다.
> 2. **가치**: 단독으로는 무해해 보이는 활동들의 '패턴'을 인식하여, 고도의 지능형 공격(APT)이나 내부자 위협을 조기에 식별할 수 있게 한다.
> 3. **판단 포인트**: 과도한 상관 분석 규칙은 오탐(False Positive)을 양산하므로, 위협 시나리오 기반의 정교한 튜닝과 임계치(Threshold) 설정이 필수적이다.

---

## Ⅰ. 개요 및 필요성

상관 분석은 SIEM의 '지능'에 해당한다. 개별 보안 장비는 자신이 담당하는 영역(네트워크, 엔드포인트 등)만 볼 수 있는 '터널 시야'를 갖는다. 하지만 현대의 공격은 여러 계층을 넘나들며 긴 시간에 걸쳐 수행된다.

상관 분석은 "외부 IP의 스캔 발생" + "웹 서버의 에러 로그 급증" + "DB 서버의 대량 데이터 외부 전송"이라는 세 가지 사건을 하나의 '데이터 유출 사고'로 연결한다. 이를 통해 수만 건의 단순 알람 중에서 진짜 대응이 필요한 '인시던트(Incident)'를 골라낸다.

- **📢 섹션 요약 비유**: 상관 분석은 '퍼즐 맞추기'와 같다. 조각 하나만 봐서는 무엇인지 알 수 없지만, 조각들을 서로 맞추다 보면 거대한 그림(공격 시나리오)이 드러나는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

상관 분석 엔진은 미리 정의된 규칙(Rule)이나 알고리즘에 따라 들어오는 이벤트를 실시간으로 검사한다.

| 분석 방식 | 특징 | 장점 |
|:---|:---|:---|
| **규칙 기반 (Rule-based)** | If-Then 조건문 형태 | 명확한 탐지 근거, 설정 용이 |
| **통계 기반 (Statistical)** | 평균/표준편차에서 벗어난 수치 탐지 | 미지의 공격(Zero-day) 탐지 가능 |
| **상태 기반 (Stateful)** | 이벤트의 순서와 상태 변화 추적 | 복합 단계 공격(Multi-stage) 탐지 |
| **행위 기반 (Heuristic/ML)** | 머신러닝을 통한 패턴 학습 | 복잡한 상관 관계 자동 발견 |

```text
  [ Event 1 ] ──▶ [ Event 2 ] ──▶ [ Event 3 ]
  (Firewall)       (Server)        (Active Directory)
       │                │                │
       ▼                ▼                ▼
┌─────────────────────────────────────────────────────────────┐
│                 Correlation Engine (SIEM)                   │
│  "IF (Login Fail > 5) AND (Success) AND (New User Created)  │
│   THEN RAISE 'Privilege Escalation Alert'"                  │
└─────────────────────────────┬───────────────────────────────┘
                              │
                              ▼
                      [ High Risk Alert ]
```

- **📢 섹션 요약 비유**: 상관 분석은 '도어록의 비밀번호 확인'과 같다. 숫자 하나하나(이벤트)는 의미가 없지만, 정해진 순서와 조합(상관 규칙)이 맞아야만 비로소 문이 열리는(알람 발생) 원리이다.

---

## Ⅲ. 비교 및 연결

단순 탐지(Detection)와 상관 분석(Correlation)은 정보의 깊이에서 차이가 있다.

| 항목 | 단순 탐지 (Single Event) | 상관 분석 (Multi Event) |
|:---|:---|:---|
| 분석 범위 | 단일 장비, 단일 로그 | 이기종 장비, 시계열 데이터 |
| 복잡도 | 낮음 (시그니처 매칭) | 높음 (조건부 로직) |
| 오탐율 | 높음 (개별 차단 시 발생) | 낮음 (맥락 확인 후 발생) |
| 대응 가치 | 낮음 (단순 차단 보고) | 높음 (사고 대응 시작점) |

- **📢 섹션 요약 비유**: 단순 탐지가 "누군가 벨을 눌렀다"는 사실만 알리는 것이라면, 상관 분석은 "벨을 누른 사람이 5분 전 우리 집 담장을 넘던 사람과 동일하다"는 것을 알려주는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 상관 분석 규칙 설계 시 'MITRE ATT&CK' 프레임워크를 기반으로 공격자의 전술(Tactics)과 기법(Techniques)을 맵핑해야 한다. 또한 '시간 윈도우(Time Window)' 설계를 통해 공격이 발생하는 유효 시간을 정밀하게 조정해야 한다.

### 체크리스트
1. 서로 다른 장비의 시간(NTP)이 동기화되어 이벤트 선후 관계가 명확한가?
2. 공격자가 탐지를 피하기 위해 아주 천천히 수행하는 공격(Low and Slow)을 잡을 수 있는 장기 윈도우 규칙이 있는가?
3. 화이트리스트(정상 행동)를 반영하여 불필요한 알람을 필터링하고 있는가?

### 안티패턴
- 상관 분석 규칙이 너무 복잡하여 SIEM 장비의 CPU 점유율이 100%에 도달하는 경우
- "로그인 실패 3회 시 알람"처럼 너무 단순한 규칙만 설정하여 실제 관제 인력이 알람 홍수에 빠지는 경우

- **📢 섹션 요약 비유**: 상관 분석은 '그물망'과 같다. 그물코가 너무 크면 큰 고기(공격)를 놓치고, 너무 작으면 쓰레기(노이즈)만 잔뜩 걸려 그물이 찢어진다.

---

## Ⅴ. 기대효과 및 결론

상관 분석은 보안 관제의 품질을 결정하는 '핵심 지수'이다. 이를 통해 공격의 전체 경로(Kill Chain)를 파악할 수 있으며, 사고 발생 시 어떤 데이터가 유출되었고 어떤 시스템이 오염되었는지 신속하게 판단할 수 있다.

앞으로는 정적인 규칙(Static Rule)을 넘어, 그래프 DB 기반의 '관계 분석'과 AI를 통한 '자동 시나리오 생성' 기술이 도입될 것이다. 데이터 간의 연결 고리를 시스템이 스스로 찾아내어 분석가에게 제시하는 '가이드형 상관 분석'이 미래의 모습이다.

- **📢 섹션 요약 비유**: 상관 분석은 '별자리 지도'와 같다. 무수히 많은 별(데이터) 중에서 의미 있는 별들을 연결하여 북극성(핵심 위협)을 찾아내고 나아갈 방향(대응 전략)을 정하게 해준다.
