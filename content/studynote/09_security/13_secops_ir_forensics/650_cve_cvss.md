## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CVE (Common Vulnerabilities and Exposures)는 소프트웨어 취약점에 부여된 고유 식별자이며, CVSS (Common Vulnerability Scoring System)는 그 취약점의 심각도를 정량적으로 측정한 점수 체계다.
> 2. **가치**: 파편화된 취약점 정보를 표준화된 언어(CVE)로 통합하고, 위험의 우선순위(CVSS)를 제시함으로써 한정된 보안 리소스를 어디에 먼저 집중할지 결정하게 돕는다.
> 3. **판단 포인트**: CVSS 점수가 높다고 반드시 우리 조직에 치명적인 것은 아니며, 취약점이 실제 노출된 환경(Temporal/Environmental Metrics)을 고려한 맞춤형 대응이 필요하다.

---

## Ⅰ. 개요 및 필요성

소프트웨어의 취약점은 매일 수백 개씩 발견된다. 각 벤더가 자기만의 방식으로 취약점을 부르면 큰 혼란이 발생한다. **CVE (Common Vulnerabilities and Exposures)**는 "이 취약점은 전 세계 공통으로 이 번호(예: CVE-2024-1234)로 부르자"는 약속이다. 이를 통해 서로 다른 보안 장비들이 동일한 위협을 인지하고 협력할 수 있다.

하지만 번호만으로는 "어떤 것부터 고쳐야 할지" 알 수 없다. 이때 등장하는 것이 **CVSS (Common Vulnerability Scoring System)**다. 취약점이 얼마나 공격하기 쉬운지, 성공 시 피해가 얼마나 큰지를 수치화(0.0~10.0)하여 보여준다. 이는 보안 관리자가 '패치 우선순위'를 결정하는 가장 강력한 기준이 된다.

- **📢 섹션 요약 비유**: CVE는 환자에게 부여된 '환자 번호'이고, CVSS는 그 환자의 상태를 나타내는 '응급 등급(Triage)'과 같다. 번호가 있어야 환자를 식별하고, 등급이 있어야 응급 수술 순서를 정할 수 있다.

---

## Ⅱ. 구성 및 핵심 원리

CVE는 식별 번호 체계이며, CVSS는 점수를 산출하는 다차원 알고리즘이다.

| 구분 | 요소 | 설명 |
|:---|:---|:---|
| **CVE** | CVE-YYYY-NNNN | 연도와 순번으로 구성된 고유 번호 (MITRE 관리) |
| **CVSS (Base Score)** | **Exploitability** | 공격 가능성 (경로, 복잡도, 권한 필요 여부 등) |
| | **Impact** | 영향도 (기밀성, 무결성, 가용성 파괴 정도) |

```text
┌──────────────────────────────────────────────────────────────┐
│             CVSS 점수 산출 구조 (v3.x 기준)                    │
├──────────────────────────────────────────────────────────────┤
│ [Base Score] : 취약점 자체의 불변하는 특성 (기본 점수)          │
│      ▲                                                       │
│ [Temporal Score] : 시간 경과에 따른 변화 (익스플로잇 존재 여부 등) │
│      ▲                                                       │
│ [Environmental Score] : 특정 조직의 환경 (보안 통제 유무 등)    │
│                                                              │
│  ===> 최종 결정 점수 (Final Score)                             │
└──────────────────────────────────────────────────────────────┘
```

특히 CVSS 점수 7.0 이상은 'High', 9.0 이상은 'Critical'로 분류된다. 점수가 산출될 때는 **Vector String** (예: `CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H`)이라는 암호 같은 문자열이 함께 생성되는데, 이를 해독하면 "네트워크를 통해(AV:N) 추가 권한 없이(PR:N) 기밀성/무결성/가용성을 모두 파괴(C/I/A:H)할 수 있는 아주 위험한 취약점"임을 알 수 있다.

- **📢 섹션 요약 비유**: CVSS 점수는 '태풍 예보'와 같다. 태풍의 세기(Base Score)는 같아도, 우리 집이 견고한지(Environmental)와 태풍이 지금 얼마나 가까이 왔는지(Temporal)에 따라 내가 느낄 위험은 달라진다.

## Ⅲ. 비교 및 연결

CVE와 CVSS는 취약점 관리 라이프사이클에서 서로 다른 역할을 수행한다.

| 항목 | CVE (식별자) | CVSS (점수) | CWE (약점) |
|:---|:---|:---|:---|
| 역할 | "무엇이 문제인가?" (Name) | "얼마나 위험한가?" (Severity) | "왜 발생했는가?" (Root Cause) |
| 관리 기관 | MITRE | FIRST | MITRE |
| 예시 | CVE-2021-44228 | 10.0 (Critical) | CWE-20 (부적절한 입력 유효성 검사) |
| 비유 | 환자 이름 | 체온/혈압 수치 | 병명 (감기, 폐렴 등) |

이 정보들은 **NVD (National Vulnerability Database)**에 통합되어 관리된다. 보안 담당자는 최신 CVE 정보를 구독하고, 자사 자산 목록(Asset Inventory)과 대조하여 취약한 버전을 사용하는 서버를 찾아낸 뒤, CVSS 점수가 높은 것부터 패치를 적용하는 워크플로우를 가진다.

- **📢 섹션 요약 비유**: CVE가 지문이라면, CVSS는 전과 기록이고, CWE는 범행 수법이다. 이 세 정보가 모여야 한 명의 범죄자(취약점)를 완벽히 이해할 수 있다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사적 관점에서 가장 중요한 실무 판단은 'CVSS 10점 만능주의'를 경계하는 것이다. 점수가 낮더라도 우리 조직의 핵심 서버에 있는 취약점이라면 그것이 우선순위가 되어야 한다.

### 체크리스트
1. 취약점 점검 결과 보고서에 단순 점수뿐만 아니라 Vector String에 대한 해석이 포함되어 있는가?
2. 우리 조직의 환경적 요인을 반영하여 **Environmental Score**를 재산출하고 있는가?
3. 패치 불가능한 환경(Legacy)에서 낮은 CVSS 점수의 취약점을 방치하고 있지는 않은가? (연쇄 공격의 징검다리가 될 수 있음)

### 안티패턴
- CVSS 점수만 보고 기계적으로 패치 순서를 정하는 행위 (비즈니스 중요도 무시)
- CVE 번호가 부여되지 않은 제로데이(0-Day) 위협을 관리 대상에서 제외하는 경우

- **📢 섹션 요약 비유**: CVSS 활용은 '시험 공부'와 같다. 배점이 높은 문제(높은 점수)부터 푸는 것이 유리하지만, 내가 절대 틀리면 안 되는 문제(핵심 자산)가 무엇인지 먼저 파악하는 것이 더 중요하다.

## Ⅴ. 기대효과 및 결론

CVE와 CVSS는 전 세계 보안 커뮤니티가 '하나의 언어'로 소통하게 해준다. 이를 통해 취약점 대응의 자동화가 가능해졌으며, 위협 가시성이 획기적으로 향상되었다. 또한 벤더들이 보안 패치를 신속하게 내놓도록 압박하는 긍정적인 효과도 있다.

결론적으로 CVE/CVSS는 취약점 관리의 '글로벌 표준'이다. 점수라는 숫자에 매몰되지 않고 그 이면의 의미를 해석할 수 있을 때, 비로소 기술사다운 냉철한 보안 의사결정이 가능하다.

- **📢 섹션 요약 비유**: CVE/CVSS는 보안팀에게 '응급실 우선순위 차트'를 선물한다. 아수라장 같은 위협 상황 속에서도 누구부터 살려야 할지(어떤 서버부터 패치할지) 명확하게 가이드해준다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| NVD | CVE 정보와 CVSS 점수를 체계적으로 정리한 데이터베이스 |
| CWE | 소프트웨어 설계/코딩 상의 근본적인 결함 유형 (Common Weakness Enumeration) |
| Exploit | 취약점을 실제로 공격하기 위한 코드나 행위 |
| Zero-Day | CVE 번호가 부여되기도 전에 공격이 시작된 최신 취약점 위협 |

### 👶 어린이를 위한 3줄 비유 설명

1. CVE는 컴퓨터 프로그램의 '아픈 곳'에 붙여준 이름표예요.
2. CVSS는 그 아픈 곳이 얼마나 심각한지 점수로 매긴 '성적표' 같은 거고요.
3. 이름표를 보고 성적표 점수가 높은 곳부터 얼른 치료해주면 컴퓨터가 건강해진답니다.
