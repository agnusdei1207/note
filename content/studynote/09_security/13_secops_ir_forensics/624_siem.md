## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SIEM (Security Information and Event Management)은 다양한 보안 장비와 시스템으로부터 로그와 이벤트를 실시간으로 수집, 정규화, 상관 분석하여 위협을 탐지하는 통합 보안 플랫폼이다.
> 2. **가치**: 파편화된 데이터 사이의 연관 관계를 분석함으로써, 단일 장비로는 발견할 수 없는 복잡한 공격 패턴(APT 등)을 시각화하고 경보를 발령한다.
> 3. **판단 포인트**: 방대한 로그 수집에 따른 스토리지 비용과 탐지 규칙(Rule)의 정확도 사이의 균형을 맞추는 정책 설계가 운영의 핵심이다.

---

## Ⅰ. 개요 및 필요성

SIEM (Security Information and Event Management)은 보안 관제의 '중심 엔진'이다. 기업 내에는 방화벽, IPS, 백신, 서버, DB 등 수많은 장비가 존재하며 각기 다른 형식의 로그를 쏟아낸다.

개별 로그만 보면 정상적인 활동처럼 보일 수 있지만, SIEM은 "방화벽 차단 발생 → 5분 뒤 서버 로그인 실패 → 10분 뒤 DB 계정 생성"과 같은 흐름을 하나로 엮어 공격 시나리오를 완성한다. 이러한 '상관 분석' 기능이 없다면 지능형 위협에 대한 대응은 불가능에 가깝다.

- **📢 섹션 요약 비유**: SIEM은 '유능한 수사 반장'과 같다. 흩어진 목격자들의 진술(로그)을 모아 사건의 전말(공격 시나리오)을 재구성하고 범인을 특정한다.

---

## Ⅱ. 아키텍처 및 핵심 원리

SIEM은 크게 수집(Collector), 정규화(Normalization), 저장(Storage), 상관 분석(Correlation), 가시화(Visualization) 단계로 작동한다.

| 단계 | 주요 기능 | 설명 |
|:---|:---|:---|
| 수집 | Log Collection | syslog, SNMP, API 등을 통해 다양한 장비로부터 데이터 확보 |
| 정규화 | Normalization | 제각각인 로그 형식을 공통 스키마(예: 날짜, IP, 사용자)로 통일 |
| 상관 분석 | Correlation | 미리 정의된 규칙(Rule)이나 알고리즘에 따라 위협 여부 판단 |
| 인덱싱/저장 | Indexing | 빠른 검색을 위한 데이터 구조화 및 장기 보관 |
| 가시화 | Dashboard | 탐지된 위협 상황을 그래프나 대시보드로 요약 표시 |

```text
┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│  Data Source │ ───▶ │ Normalization│ ───▶ │  Correlation │
│ (Router/Web) │      │ (Standardize)│      │   Engine     │
└──────────────┘      └──────────────┘      └──────┬───────┘
                                                   │
                                                   ▼
┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│  Dashboard   │ ◀─── │   Storage    │ ◀─── │    Alerts    │
│  (Visual)    │      │  (Big Data)  │      │ (Real-time)  │
└──────────────┘      └──────────────┘      └──────────────┘
```

- **📢 섹션 요약 비유**: SIEM은 '대형 용광로'와 같다. 제각각인 고철(로그)들을 녹여 정제된 쇳물(정규화 데이터)을 만들고, 이를 통해 튼튼한 철근(위협 인사이트)을 뽑아낸다.

---

## Ⅲ. 비교 및 연결

SIEM은 과거의 SIM(Log Storage)과 SEM(Real-time Monitoring)이 합쳐진 개념이며, 최근에는 UEBA(행위 분석)와 통합되는 추세이다.

| 항목 | SIM (Security Info Mgmt) | SEM (Security Event Mgmt) | SIEM (Integrated) |
|:---|:---|:---|:---|
| 주요 기능 | 로그 수집 및 장기 보관 | 실시간 이벤트 모니터링 | 수집 + 분석 + 대응 지원 |
| 초점 | 분석 및 리포팅 (과거) | 위협 탐지 및 알람 (현재) | 통합 보안 거버넌스 |
| 연동성 | 사후 감사 위주 | 즉각적 조치 위주 | SOAR, EDR 등과 연계 |

- **📢 섹션 요약 비유**: SIM이 '도서관'이고 SEM이 '뉴스 속보'라면, SIEM은 과거의 기록을 참고하여 현재의 속보를 분석하는 '뉴스 분석 센터'이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 SIEM 구축 시 '로그의 양보다 질'을 강조해야 한다. 무분별한 로그 수집은 장비 부하를 초래하고 의미 없는 알람(Noise)만 양산한다. 또한 '탐지 규칙(Rule)의 생명주기 관리'가 필수적이다.

### 체크리스트
1. 로그가 전송 과정에서 위변조되지 않도록 암호화 채널을 사용하는가?
2. 최신 공격 기법(MITRE ATT&CK 등)을 반영한 탐지 규칙이 주기적으로 업데이트되는가?
3. 로그 저장 용량이 법적 준거성(Compliance) 요건을 충족하는가?

### 안티패턴
- 장비 도입 후 기본 제공 규칙(Out-of-the-box Rules)만 쓰고 커스터마이징을 하지 않는 경우
- 로그 검색 속도가 너무 느려 실시간 대응이 불가능한 상태로 방치되는 경우

- **📢 섹션 요약 비유**: SIEM은 '청진기'와 같다. 의사(분석가)가 환자의 몸 구석구석에서 나는 소리(로그)를 잘 들어야 병(공격)을 진단할 수 있지만, 주변이 너무 시끄러우면(노이즈) 아무 소리도 들리지 않는다.

---

## Ⅴ. 기대효과 및 결론

SIEM은 기업 보안 운영의 가시성을 확보하는 가장 강력한 수단이다. 흩어진 보안 이벤트를 하나의 문맥(Context)으로 파악하게 함으로써, 사고 발생 시 원인 분석(Root Cause Analysis) 시간을 획기적으로 단축시킨다.

미래의 SIEM은 '클라우드 네이티브 SIEM'과 'AI 결합'으로 진화하고 있다. 데이터 양에 상관없이 확장 가능한 클라우드 아키텍처와, 인간이 놓치는 미세한 패턴을 찾아내는 머신러닝 기술이 결합되어 '차세대 통합 관제'를 완성할 것이다.

- **📢 섹션 요약 비유**: SIEM은 '밤하늘의 별자리'와 같다. 제각각인 별(이벤트)들을 선으로 이어서 의미 있는 별자리(공격 패턴)로 만들어주는 역할을 한다.
