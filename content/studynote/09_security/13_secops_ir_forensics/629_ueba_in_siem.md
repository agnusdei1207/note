## 핵심 인사이트 (3줄 요약)
> 1. **본질**: UEBA (User and Entity Behavior Analytics)는 시그니처 기반 탐지의 한계를 넘어, AI/ML을 통해 사용자와 엔티티(장비)의 '정상 상태'를 학습하고 이탈하는 이상 행위를 찾아내는 기술이다.
> 2. **가치**: 내부자 위협, 계정 도용, 지능형 지속 위협 (APT) 등 명확한 공격 패턴이 없는 위협을 데이터 프로파일링과 리스크 스코어링으로 가시화한다.
> 3. **판단 포인트**: 단순 SIEM의 상관분석(Correlation)이 '알려진 규칙'에 집중한다면, UEBA는 '알려지지 않은 편차'에 집중하여 보안 탐지의 정밀도를 높인다.

---

## Ⅰ. 개요 및 필요성

UEBA (User and Entity Behavior Analytics)는 사용자(User)뿐만 아니라 IP, 호스트, 애플리케이션 등 엔티티(Entity)의 행위를 분석하여 보안 위협을 탐지하는 보안 기술이다. 기존의 시그니처나 룰 기반 탐지 방식은 이미 알려진 공격 패턴에는 효과적이지만, 정상 계정을 탈취하여 수행하는 은밀한 공격이나 내부자에 의한 정보 유출은 잡아내기 어렵다는 한계가 있었다.

UEBA는 "정상이 무엇인지"를 먼저 정의하는 것에서 시작한다. 평소와 다른 시간에 로그인을 하거나, 평소 접근하지 않던 서버에 접속하거나, 비정상적으로 대량의 데이터를 외부로 전송하는 행위 등을 감지한다. 이는 오탐(False Positive)을 줄이고, 공격자가 보안 인프라 내부에서 활동하는 '체류 시간 (Dwell Time)'을 단축하는 데 필수적이다.

- **📢 섹션 요약 비유**: UEBA는 단골 손님의 얼굴을 기억하는 가게 주인과 같다. 평소와 다른 옷차림이나 말투, 갑작스러운 고액 주문을 보고 "이 사람이 진짜 그 단골이 맞나?"라고 의심해보는 과정이다.

---

## Ⅱ. 아키텍처 및 핵심 원리

UEBA의 핵심은 데이터 수집, 프로파일링, 이상 탐지, 리스크 스코어링의 4단계 프로세스다. SIEM 등으로부터 대량의 로그를 입력받아 머신러닝 알고리즘을 통해 각 개체별 베이스라인(Baseline)을 구축한다.

| 구성 요소 | 역할 | 상세 설명 |
|:---|:---|:---|
| 데이터 소스 | 원천 데이터 제공 | 로그, 패킷, 엔드포인트 행위, AD 데이터 등 |
| 분석 엔진 | 이상 행위 모델링 | 머신러닝(ML), 통계 분석을 통한 패턴 학습 |
| 베이스라인 | 정상 상태 정의 | 특정 기간 동안의 표준 행위 프로파일 생성 |
| 리스크 스코어링 | 우선순위 부여 | 이상 행위의 심각도를 점수화하여 가시화 |

```text
┌──────────────────────────────────────────────────────────────┐
│             UEBA 분석 프로세스: 학습에서 스코어링까지             │
├──────────────────────────────────────────────────────────────┤
│ [로그 데이터] ─▶ [개체 프로파일링] ─▶ [이상 탐지] ─▶ [위험 점수] │
│      │               │               │               │       │
│      └─ SIEM/로그 ───┴─ ML 모델 학습 ──┴─ 편차 분석 ───┴─ 대시보드 │
└──────────────────────────────────────────────────────────────┘
```

분석 엔진은 비지도 학습(Unsupervised Learning)을 주로 사용하여 사전에 정의되지 않은 패턴을 찾아낸다. 예를 들어, 한 사용자가 갑자기 평소보다 10배 많은 파일을 다운로드하면 시스템은 이를 이상 행위로 간주하고 리스크 점수를 높인다.

- **📢 섹션 요약 비유**: UEBA는 평소 걸음걸이와 보폭을 기억했다가, 술에 취해 비틀거리는 것을 보고 상태가 이상함을 알아차리는 웨어러블 기기와 같다.

## Ⅲ. 비교 및 연결

SIEM (Security Information and Event Management)과 UEBA는 상호 보완적인 관계다. 초기 UEBA는 독립적인 솔루션으로 등장했으나, 현재는 대부분의 차세대 SIEM에 통합되어 제공되는 추세다.

| 항목 | 기존 SIEM (Rule-based) | UEBA (Analytics-based) |
|:---|:---|:---|
| 탐지 방식 | 상관분석 및 고정 룰 (If-Then) | 머신러닝 및 행위 프로파일링 |
| 주요 대상 | 알려진 위협 (Known Threats) | 알려지지 않은 위협 (Unknown Threats) |
| 데이터 활용 | 이벤트 중심 (Logs) | 행위 중심 (User/Entity Context) |
| 결과물 | 경보 (Alert) 발생 | 리스크 점수 (Risk Score) 부여 |

UEBA는 탐지된 이상 행위를 SOAR (Security Orchestration, Automation, and Response)와 연결하여 자동 차단하거나, EDR (Endpoint Detection and Response)과 연계하여 엔드포인트 수준의 심층 분석을 수행하는 가교 역할을 한다.

- **📢 섹션 요약 비유**: SIEM이 "빨간불에 건너면 경고"라는 교통 법규라면, UEBA는 "운전자가 졸음운전을 하는 것 같으니 휴식을 권고"하는 지능형 안전 시스템이다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 UEBA 도입 시 가장 큰 고민은 '학습 기간'과 '오탐 관리'다. 모델이 정상 행위를 충분히 학습하기 전에는 많은 경보가 발생할 수 있으며, 직무 변경이나 특수 상황(예: 마감 기한)에 따른 정상적인 행위 변화를 이상 행위로 오인할 수 있다.

### 체크리스트
1. 분석에 필요한 충분한 컨텍스트 데이터(AD, 인사 정보 등)가 확보되었는가?
2. 초기 학습 기간(통상 2~4주) 동안의 노이즈 제거 계획이 있는가?
3. 리스크 점수가 높은 항목에 대해 즉각 대응할 수 있는 SOC 프로세스가 마련되었는가?

### 안티패턴
- 로그 데이터만 넣고 모델이 모든 위협을 자동으로 찾아줄 것이라 기대하는 '블랙박스형' 접근
- 사용자 컨텍스트(직무, 권한 등)를 배제한 채 단순 통계 수치만으로 리스크를 판단하는 경우

- **📢 섹션 요약 비유**: UEBA 운영은 명탐정을 고용하는 것과 같다. 탐정에게 충분한 정보(증거)를 주지 않으면 엉뚱한 사람을 범인으로 지목할 수 있다.

---

## Ⅴ. 기대효과 및 결론

UEBA를 통해 조직은 내부 공모자에 의한 데이터 유출, 은밀하게 진행되는 랜섬웨어의 초기 횡적 이동(Lateral Movement) 등을 효과적으로 방지할 수 있다. 이는 단순한 보안 강화를 넘어, 리스크 기반의 효율적인 SOC 운영을 가능하게 한다.

결론적으로 UEBA는 '규칙' 중심의 보안에서 '상태' 중심의 보안으로 패러다임을 전환하는 핵심 도구다. 미래에는 더 정교한 딥러닝 모델과 결합하여 인간 보안 분석가가 놓치기 쉬운 미세한 징후까지 실시간으로 포착하는 방향으로 진화할 것이다.

- **📢 섹션 요약 비유**: UEBA는 보안팀에게 '투시경'을 주는 것과 같다. 겉으로는 정상적으로 보이는 활동 이면의 위험한 흐름을 꿰뚫어 보게 해준다.

| SIEM | UEBA의 주된 데이터 소스이자 통합 플랫폼 |
| Inside Threat | UEBA가 가장 강력한 탐지 능력을 발휘하는 위협 영역 |
| Machine Learning | UEBA의 행위 분석과 베이스라인 구축을 위한 핵심 기술 |
| SOAR | UEBA에서 탐지된 고위험 행위에 대한 자동 대응 연계 |

### 👶 어린이를 위한 3줄 비유 설명

1. UEBA는 우리 집 강아지가 평소랑 다르게 행동하면 "어디 아픈가?" 하고 알아채는 마음과 같아요.
2. 평소에는 밥도 잘 먹고 잠도 잘 자던 강아지가 갑자기 밥을 안 먹으면 이상하다고 느끼는 거죠.
3. 컴퓨터에서도 나쁜 사람이 착한 사람인 척 행동해도 평소랑 조금만 다르면 딱 잡아낼 수 있어요.
