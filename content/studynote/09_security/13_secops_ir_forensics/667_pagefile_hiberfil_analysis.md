## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페이지 파일(`pagefile.sys`)과 최대 절전 모드 파일(`hiberfil.sys`)은 윈도우 운영체제가 메모리 관리와 상태 보존을 위해 하드디스크에 생성하는 휘발성 메모리의 '확장판'이자 '박제본'이다.
> 2. **가치**: RAM에서 밀려난 데이터나 시스템 종료 직전의 RAM 전체 상태를 담고 있어, 물리 메모리 덤프에서 놓친 과거의 흔적이나 은닉된 악성 코드를 발견할 수 있는 보고(寶庫)다.
> 3. **판단 포인트**: RAM 분석 결과가 불충분할 경우, 이 파일들을 추출하여 문자열 검색(Strings)이나 전용 파싱 도구로 분석함으로써 공격자의 행위를 재구성해야 한다.

---

## Ⅰ. 개요 및 필요성

디지털 포렌식에서 `pagefile.sys`와 `hiberfil.sys`는 RAM의 물리적 한계를 넘어서는 보조 증거물이다. 
- **`pagefile.sys` (Page File)**: 가상 메모리 시스템에서 RAM이 부족할 때 덜 사용하는 데이터를 디스크로 옮겨놓은 공간이다. RAM에는 이미 사라진 과거의 프로세스 흔적이 남아있을 확률이 높다.
- **`hiberfil.sys` (Hibernation File)**: 사용자가 '최대 절전 모드'를 실행할 때 RAM 전체 내용을 디스크에 저장한 파일이다. 시스템이 꺼져 있더라도 사고 당시의 RAM 상태를 고스란히 복원할 수 있게 해준다.

이 파일들은 시스템이 재부팅되더라도 디스크에 남아있는 경우가 많아, 라이브 대응 시 메모리 덤프를 수행하지 못했을 때 최후의 수단으로 활용된다.

- **📢 섹션 요약 비유**: 페이지 파일은 자주 안 쓰는 물건을 넣어둔 '창고'와 같고, 최대 절전 모드 파일은 이사 가기 전 집안을 그대로 찍어둔 '사진'과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

이 파일들은 일반적인 파일 시스템 분석 도구로는 내용을 보기 어려우며, 메모리 구조로 복원하거나 가상 주소를 매핑하는 과정이 필요하다.

### 파일별 특성 및 분석 원리

| 파일명 | 생성 원인 | 데이터 성격 | 분석 포인트 |
|:---|:---|:---|:---|
| **pagefile.sys** | 가상 메모리 페이징 | 조각난 메모리 페이지 | 평문 비밀번호, URL, 채팅 기록, 조각난 코드 |
| **hiberfil.sys** | 최대 절전 모드 (S4) | RAM 전체의 압축본 | 프로세스 목록, 네트워크 연결, 로딩된 드라이버 |

### 분석 워크플로우 (Extraction to Analysis)

```text
  [Evidence Disk]           [Extraction]              [Decompression/Parsing]
┌────────────────┐      ┌──────────────────┐      ┌─────────────────────────┐
│ C:\pagefile.sys│ ──▶  │ FTK Imager 등을  │ ──▶  │ Strings 분석 (Keyword)   │
│ C:\hiberfil.sys│      │ 이용한 파일 추출 │      │ Volatility 이미지 변환   │
└────────────────┘      └──────────────────┘      └─────────────────────────┘
                                                           │
        ┌──────────────────────────────────────────────────┴──────────────────┐
        │ [Analysis Result]                                                   │
        │ - hiberfil.sys: `imagecopy` 플러그인으로 RAW 이미지 변환 후 분석     │
        │ - pagefile.sys: 비연속적 데이터이므로 `yarascan` 등으로 패턴 탐색    │
        └─────────────────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: 압축된 최대 절전 모드 파일을 분석하는 것은 진공 포장된 옷을 뜯어서 다시 입혀보는 것과 같다. 원래 모양대로 펴야(압축 해제) 내용을 확인할 수 있다.

## Ⅲ. 비교 및 연결

물리 메모리(RAM)와 이 파일들은 상호 보완적이지만, 데이터의 연속성 측면에서 차이가 있다.

| 항목 | 물리 메모리 (RAM) | 최대 절전 모드 파일 (Hiberfil) | 페이지 파일 (Pagefile) |
|:---|:---|:---|:---|
| 데이터 연속성 | 매우 높음 (실시간) | 높음 (특정 시점 박제) | 낮음 (조각난 페이지) |
| 수집 난이도 | 높음 (Live 상태 필수) | 낮음 (정적 파일 추출) | 낮음 (정적 파일 추출) |
| 데이터 신선도 | 현재 상태 | 마지막 절전 시점 | 과거~현재 혼재 |
| 분석 도구 | Volatility, MemProcFS | Volatility, Hiber2Raw | Strings, 파싱 도구 |

이 파일들에서 발견된 악성 도메인이나 IP는 `668. 네트워크 포렌식`에서 수집된 트래픽 로그와 대조하여 공격의 유효성을 검증하는 데 사용된다.

- **📢 섹션 요약 비유**: RAM은 지금 흐르는 강물이고, Hiberfil은 그 강물이 얼어붙은 얼음판이며, Pagefile은 강가에 밀려와 쌓인 퇴적물과 같다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 메모리 분석이 불가능한 상황(이미 시스템이 꺼진 경우 등)에서 이 파일들의 존재 여부를 가장 먼저 확인해야 한다.

### 실무 판단 가이드

1. **파일 존재 여부 확인**: 시스템 설정에서 가상 메모리나 최대 절전 모드가 비활성화되어 있지는 않은가?
2. **분석 도구 준비**: `hiberfil.sys`는 압축 방식(XPRESS 등)이 윈도우 버전에 따라 다르므로 적절한 변환 도구를 준비했는가?
3. **키워드 검색 우선**: 페이지 파일은 구조가 파편화되어 있으므로, 무작정 구조 분석을 하기보다 관련 키워드(Email, IP, 계정명)로 검색을 먼저 수행하는 것이 효율적인가?
4. **안티 포렌식 대응**: 공격자가 흔적을 지우기 위해 이 파일들을 초기화(Wiping)했을 가능성은 없는가?

### 안티패턴
- `hiberfil.sys`를 변환 과정 없이 메모리 분석 도구에 바로 넣고 "분석이 안 된다"고 판단하는 행위
- 페이지 파일에 중요한 정보가 없을 것이라 단정하고 수집 대상에서 제외하는 행위

- **📢 섹션 요약 비유**: 페이지 파일 분석은 파쇄된 종이 조각을 맞추는 것과 같다. 비록 조각나 있지만, 그 조각 하나에 결정적인 비밀번호가 적혀 있을 수 있다.

## Ⅴ. 기대효과 및 결론

`pagefile.sys`와 `hiberfil.sys` 분석은 휘발성 증거 수집의 한계를 극복하게 해준다. 특히 공격자가 메모리에서만 동작하는 기법을 썼더라도, 운영체제의 메모리 관리 메커니즘에 의해 의도치 않게 디스크에 흔적이 남게 되는 '기술적 역설'을 이용한 훌륭한 분석 지점이다.

결론적으로 숙련된 포렌식 전문가는 RAM 덤프뿐만 아니라 디스크에 남은 메모리의 그림자인 이 파일들을 능숙하게 다룰 줄 알아야 사고의 전체 맥락을 짚어낼 수 있다.

- **📢 섹션 요약 비유**: 이 파일들은 컴퓨터가 남긴 '무의식의 기록'이다. 사용자는 기억하지 못해도 컴퓨터는 모든 것을 디스크 한구석에 적어두고 있다.
