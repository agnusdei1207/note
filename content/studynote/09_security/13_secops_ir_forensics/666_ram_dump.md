## 핵심 인사이트 (3줄 요약)
> 1. **본질**: RAM Dump는 휘발성 메모리(RAM)의 전체 데이터를 물리적 섹터 단위로 복제하여 파일 형태(.raw, .mem, .img)로 저장하는 증거 수집 과정이다.
> 2. **가치**: 시스템 전원이 차단되면 사라지는 프로세스, 커널 정보, 암호화 키 등을 보존하여 사고 당시의 '생생한 현장'을 사후 분석 가능하게 만든다.
> 3. **판단 포인트**: RAM Dump 수행 자체가 대상 시스템의 메모리를 일부 오염시키므로, 시스템 부하와 증거 보존율 사이의 균형을 고려하여 최적의 도구(DumpIt, LiME 등)를 선택해야 한다.

---

## Ⅰ. 개요 및 필요성

RAM Dump는 디지털 포렌식의 'Order of Volatility(휘발성 순서)' 원칙에 따라 가장 먼저 수행되어야 하는 핵심 작업이다. 하드디스크 이미징이 '과거의 흔적'을 찾는 것이라면, RAM Dump는 '현재 진행 중인 위협'을 고정시키는 역할을 한다.

최근의 공격자들은 디스크에 파일을 남기지 않는 'Fileless Malware'나 메모리 상에서만 동작하는 루트킷을 선호한다. 이러한 위협은 일반적인 안티바이러스나 디스크 포렌식으로는 절대 발견할 수 없기 때문에, 물리 메모리의 내용을 통째로 복제하는 RAM Dump가 사고 분석의 필수 관문이 되었다.

- **📢 섹션 요약 비유**: RAM Dump는 범죄 현장의 '스냅샷' 촬영과 같다. 현장이 정리되기 전(전원 차단 전)에 모든 물건의 배치와 사람들의 대화 내용을 그대로 박제해 두는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

RAM Dump는 운영체제의 아키텍처(Windows, Linux)에 따라 접근 방식이 다르며, 커널 모드 드라이버를 통해 물리 메모리 주소 영역에 직접 접근한다.

### RAM Dump 수행 메커니즘

| 환경 | 주요 도구 | 수집 방식 | 특징 |
|:---|:---|:---|:---|
| **Windows** | DumpIt, Magnet RAM Capture | `\Device\PhysicalMemory` 접근 | 사용자 인터페이스가 간편하고 빠름 |
| **Linux** | LiME (Linux Memory Extractor) | 커널 모듈(LKM) 로드 | 네트워크 전송(Over the network) 지원 |
| **VMware** | `.vmem` 파일 복사 | 가상머신 일시 정지(Suspend) | 시스템 오염 없이 가장 깔끔한 수집 가능 |

### 수집 프로세스 아키텍처

```text
  [Target RAM]             [Dumping Tool]             [Evidence Storage]
┌──────────────┐       ┌──────────────────────┐       ┌────────────────────┐
│ User Space   │ ──┐   │  Kernel Driver Load  │ ──┐   │   External USB     │
├──────────────┤   │   ├──────────────────────┤   │   ├────────────────────┤
│ Kernel Space │ ──┼──▶│  Memory Copy (DMA)   │ ──┼──▶│   MemoryDump.raw   │
├──────────────┤   │   ├──────────────────────┤   │   ├────────────────────┤
│ Free Pages   │ ──┘   │  Hash Calculation    │   │   │   Dump_Hash.txt    │
└──────────────┘       └──────────────────────┘       └────────────────────┘
       ▲                          ▲
       └──────────────────────────┘
          [Tool Occupies Memory]
         (Footprint Minimization)
```

- **📢 섹션 요약 비유**: RAM Dump는 흐르는 강물을 한 국자 떠서 얼음으로 얼리는 것과 같다. 강물의 현재 상태를 그대로 보존하기 위해 아주 조심스럽고 빠르게 떠내야 한다.

## Ⅲ. 비교 및 연결

RAM Dump는 하드디스크 이미징과 비교할 때 수집의 시급성과 휘발성 측면에서 큰 차이를 보인다.

| 항목 | 하드디스크 이미징 (Disk Imaging) | 램 덤프 (RAM Dump) |
|:---|:---|:---|
| 수집 시점 | 전원 차단 후 (Post-mortem) | 전원 차단 전 (Live) |
| 데이터 휘발성 | 낮음 (비휘발성) | 매우 높음 (휘발성) |
| 수집 대상 | 모든 파일, 슬랙 공간 | 프로세스, 네트워크, 암호화 키 |
| 분석 용도 | 과거 행적, 삭제 파일 복구 | 현재 위협, 코드 인젝션 탐지 |
| 시스템 영향 | 없음 (Write Blocker 사용 시) | 있음 (도구 실행으로 인한 메모리 변형) |

RAM Dump 결과는 `665. 메모리 포렌식`에서 다룬 Volatility 도구를 통해 분석되며, 분석 과정에서 발견된 실행 파일 정보는 다시 디스크 포렌식과 연결된다.

- **📢 섹션 요약 비유**: 디스크 이미징이 죽은 사람의 소지품을 조사하는 것이라면, RAM Dump는 살아있는 사람의 기억을 추출하는 것과 같다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 현장에서 RAM Dump를 수행할 때 '시스템 가용성'과 '증거 오염' 사이의 트레이드오프를 결정해야 한다.

### 실무 판단 체크리스트

1. **Order of Volatility**: RAM Dump를 다른 어떤 파일 수집보다 먼저 수행했는가?
2. **도구 선택**: LiME과 같이 네트워크 전송을 지원하는 도구를 써서 로컬 디스크 오염을 방지할 것인가?
3. **Footprint 최소화**: 수집 도구를 대상 PC의 HDD가 아닌 외부 USB에서 직접 실행했는가?
4. **암호화 대응**: 서버가 BitLocker 등으로 암호화되어 있다면, 메모리 덤프에 복호화 키가 포함되도록 주의했는가?

### 안티패턴
- 수집 도구를 분석 대상 PC의 바탕화면에 다운로드하여 실행하는 행위 (디스크와 메모리 대량 오염)
- 메모리 덤프 파일 용량이 RAM 크기와 동일한지 확인하지 않고 철수하는 행위 (불완전한 덤프)

- **📢 섹션 요약 비유**: RAM Dump 실무는 비오는 날 마당에 떨어진 보석을 줍는 것과 같다. 보석을 줍는 동안 내 발자국이 마당을 어지럽히겠지만, 보석(증거)이 빗물(휘발)에 씻겨 내려가기 전에 빨리 주워야 한다.

## Ⅴ. 기대효과 및 결론

RAM Dump는 사고 대응의 '골든 타임'을 확보하는 결정적 수단이다. 이를 통해 공격자의 실시간 활동을 박제하고, 파일리스 공격과 같은 고도화된 위협의 실체를 규명할 수 있다.

결론적으로 RAM Dump는 디지털 포렌식의 시작점이며, 여기서 확보된 데이터의 품질이 전체 사고 분석의 성패를 좌우한다. 기술사는 수집 과정의 오염을 최소화하면서도 필요한 모든 정보를 신속하게 담아내는 전문적인 숙련도를 갖춰야 한다.

- **📢 섹션 요약 비유**: RAM Dump는 사고 현장의 '블랙박스'와 같다. 사고 순간의 모든 정보가 담겨 있어, 나중에 사고 원인을 규명할 때 가장 강력한 증거가 된다.
