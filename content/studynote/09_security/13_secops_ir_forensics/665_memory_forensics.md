## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 포렌식(Memory Forensics)은 휘발성 메모리(RAM)에 남아있는 실행 중인 프로세스, 네트워크 연결, 암호화 키 등 실시간 시스템 상태를 분석하는 기술이다.
> 2. **가치**: 디스크에 흔적을 남기지 않는 파일리스(Fileless) 공격이나 루트킷(Rootkit)을 탐지할 수 있는 유일한 수단이며, 암호화된 드라이브의 복호화 키를 확보할 수 있는 결정적 기회를 제공한다.
> 3. **판단 포인트**: 사고 대응 시 시스템을 종료하기 전, 'Order of Volatility'에 따라 메모리 덤프를 최우선으로 수행하여 휘발성 높은 아티팩트를 보존해야 한다.

---

## Ⅰ. 개요 및 필요성

전통적인 포렌식이 하드디스크의 정적 데이터를 분석하는 데 집중했다면, 메모리 포렌식은 시스템이 동작하는 '동적 상태'를 분석한다. 공격자가 악성 코드를 메모리 상에서만 실행하거나, 실행 후 흔적을 지우는 기법을 사용할 때 메모리 포렌식은 매우 강력한 힘을 발휘한다.

특히 랜섬웨어나 고도화된 APT 공격 시나리오에서 공격자가 사용한 계정 정보, 열려 있는 네트워크 포트, 현재 실행 중인 악성 페이로드 등을 확인하기 위해서는 반드시 메모리 분석이 병행되어야 한다. 메모리는 거짓말을 하지 않는 가장 정직한 목격자이기 때문이다.

- **📢 섹션 요약 비유**: 메모리 포렌식은 범죄 현장의 'CCTV 녹화본'을 보는 것과 같다. 이미 상황이 종료된 후의 현장(디스크)보다, 범인이 움직이고 있던 순간의 기록(메모리)이 훨씬 더 많은 정보를 담고 있다.

---

## Ⅱ. 아키텍처 및 핵심 원리

메모리 포렌식은 크게 '수집(Acquisition)'과 '분석(Analysis)' 단계로 나뉘며, Volatility와 같은 프레임워크를 통해 추상화된 데이터를 해석한다.

### 메모리 포렌식 아키텍처 (Analysis Workflow)

```text
  [Target System]            [Memory Dump]              [Volatility Framework]
┌────────────────┐      ┌────────────────────┐      ┌─────────────────────────┐
│     RAM        │ ───▶ │ Raw Image / Crash  │ ───▶ │ OS Profile / Symbols    │
│ (Live State)   │      │ (DumpIt, LiME)     │      │ (KDBG, Virtual Address) │
└────────────────┘      └────────────────────┘      └─────────────────────────┘
                                                             │
        ┌────────────────────────────────────────────────────┴────────────────┐
        │ [Plugins Analysis]                                                  │
        │ - pslist / psscan : 프로세스 목록 및 은닉 프로세스 탐지             │
        │ - netscan : 네트워크 연결 정보 (IP, Port)                           │
        │ - malfind : 코드 인젝션 및 힙 영역 악성 코드 탐지                   │
        │ - hashdump : 사용자 계정 해시 추출                                  │
        └─────────────────────────────────────────────────────────────────────┘
```

### 주요 분석 대상 (Artifacts)

| 대상 | 설명 | 분석 가치 |
|:---|:---|:---|
| **Process** | 현재 실행 중인 프로그램 목록 | 부모-자식 관계를 통한 비정상 실행 탐지 |
| **Network** | 활성화된 소켓 연결 정보 | C2 서버 통신 및 수평 이동(Lateral Movement) 확인 |
| **Kernel Objects** | 드라이버, 세마포어, 뮤텍스 | 루트킷의 커널 영역 침범 여부 확인 |
| **Cleartext Data** | 평문으로 남은 비밀번호, 메시지 | 암호화된 정보의 원본 확보 |

- **📢 섹션 요약 비유**: 메모리 포렌식 아키텍처는 퍼즐 맞추기와 같다. 흩어진 비트(Bit)들을 모아 운영체제라는 밑그림(Profile) 위에 배치하여 시스템이 어떤 일을 하고 있었는지 복원해내는 과정이다.

## Ⅲ. 비교 및 연결

디스크 포렌식과 메모리 포렌식은 상호 보완적으로 사용되어야 전체 사고의 그림을 완성할 수 있다.

| 항목 | 디스크 포렌식 (Dead) | 메모리 포렌식 (Live) |
|:---|:---|:---|
| 데이터 성격 | 정적 (파일, 메타데이터) | 동적 (프로세스, 연결 정보) |
| 수집 대상 | 비휘발성 저장매체 (HDD, SSD) | 휘발성 저장매체 (RAM) |
| 주요 강점 | 삭제 파일 복구, 긴 시간의 기록 | 실시간 위협 탐지, 파일리스 공격 분석 |
| 약점 | 메모리 상의 악성코드 탐지 불가 | 전원 차단 시 데이터 소멸 |
| 핵심 도구 | EnCase, FTK, Autopsy | Volatility, Rekall, MemProcFS |

메모리 분석 결과 발견된 악성 프로세스의 실행 파일 경로는 다시 디스크 포렌식을 통해 해당 파일의 생성 시점과 유입 경로를 추적하는 '연결 고리'가 된다.

- **📢 섹션 요약 비유**: 디스크 포렌식이 과거의 기록인 '일기장'을 읽는 것이라면, 메모리 포렌식은 현재의 생각인 '뇌 스캔'을 하는 것과 같다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 사고 현장에서 "메모리 덤프의 무결성"과 "시스템 오염 최소화"를 동시에 판단해야 한다.

### 실무 판단 가이드

1. **Order of Volatility 준수**: 디스크를 복제하기 전에 반드시 메모리 덤프를 먼저 수행했는가?
2. **라이브 대응의 흔적**: 메모리 수집 도구 실행으로 인해 파괴된 메모리 영역(Page)이 어느 정도인지 인지하고 있는가?
3. **가상화 환경**: VM 환경이라면 덤프 도구 대신 'Snapshot'이나 'Suspend' 기능을 통해 RAM 상태를 보전할 수 있는가?
4. **암호화 여부**: BitLocker 등이 적용된 시스템이라면 메모리에서 'Recovery Key'를 먼저 추출해야 한다는 판단을 내렸는가?

### 안티패턴
- 악성코드가 실행 중인 것을 알고도 분석을 위해 시스템을 즉시 재부팅하거나 전원을 차단하는 행위
- 덤프 파일을 증거 대상인 동일한 하드디스크에 저장하여 원본을 오염시키는 행위

- **📢 섹션 요약 비유**: 메모리 포렌식 실무는 현장 보존과 같다. 증거를 채취하는 과정에서 내 발자국이 남을 수밖에 없음을 인정하되, 최소한으로 남기려는 노력이 필요하다.

## Ⅴ. 기대효과 및 결론

메모리 포렌식은 현대 사이버 보안 위협의 '마지막 보루'다. 안티 바이러스나 EDR이 놓친 고도의 은닉 위협을 시각화할 수 있으며, 사고 대응의 정밀도를 획기적으로 높여준다.

결론적으로 메모리 분석 능력을 갖춘 DFIR 팀은 공격자의 행위를 단순히 '차단'하는 수준을 넘어 '이해'하는 단계로 나아갈 수 있다. 시스템의 심장부인 RAM을 들여다보는 기술은 디지털 수사에서 가장 강력하고 필수적인 기술로 자리 잡았다.

- **📢 섹션 요약 비유**: 메모리 포렌식은 거짓말 탐지기와 같다. 겉으로 드러난 모습(디스크) 뒤에 숨겨진 진실(메모리)을 찾아내어 사고의 진상을 규명한다.

| Code Injection | 정상 프로세스의 메모리 영역에 악성 코드를 삽입하는 기법. 메모리 분석의 주요 타겟 |
| DMA Attack | 하드웨어 장치를 통해 CPU를 거치지 않고 메모리에 직접 접근하여 데이터를 탈취하는 공격 |

### 👶 어린이를 위한 3줄 비유 설명

1. 컴퓨터가 지금 무슨 생각을 하고 있는지 머릿속(RAM)을 들여다보는 일이에요.
2. 컴퓨터를 끄면 이 생각들이 다 사라지기 때문에, 끄기 전에 얼른 사진을 찍어둬야 해요.
3. 이 사진을 보면 나쁜 프로그램이 몰래 숨어서 무슨 짓을 했는지 다 알 수 있답니다.
