## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 타임라인 분석(Timeline Analysis)은 파일 시스템 메타데이터, 로그, 레지스트리 등 흩어진 아티팩트들의 시간 정보를 추출하여 사건 발생 전후의 흐름을 시계열로 재구성하는 기법이다.
> 2. **가치**: 단일 증거만으로는 알 수 없는 공격자의 '최초 침투 시점', '내부망 이동 경로', '데이터 유출 과정'을 인과관계에 따라 명확히 규명하게 해준다.
> 3. **판단 포인트**: 파일의 시간 정보는 위조(Timestomping)될 수 있으므로, 파일 시스템의 MAC 시간뿐만 아니라 로그와 같은 외부 증거와 교차 검증하여 타임라인의 신뢰성을 판단해야 한다.

---

## Ⅰ. 개요 및 필요성

디지털 포렌식 분석의 궁극적인 목표는 "무슨 일이 일어났는가"를 설명하는 것이다. 타임라인 분석은 수만 개의 파편화된 증거들을 시간이라는 실에 꿰어 하나의 완성된 시나리오를 만드는 핵심 과정이다. 특히 시스템 전체의 모든 시간 정보를 하나로 통합한 것을 '슈퍼 타임라인 (Super Timeline)'이라고 부른다.

공격자는 자신의 흔적을 감추기 위해 파일 생성 시간을 과거로 돌리는 등의 기법을 사용한다. 하지만 시스템 로그, 브라우저 히스토리, 프리페치(Prefetch) 파일 등 시스템 곳곳에 남은 시간 정보까지 모두 일관성 있게 조작하기는 불가능에 가깝다. 따라서 타임라인 분석은 공격자의 거짓말을 밝혀내는 강력한 도구가 된다.

- **📢 섹션 요약 비유**: 타임라인 분석은 흩어진 퍼즐 조각들을 사건 발생 순서대로 나열하여 범죄 영화의 필름을 복원하는 작업과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

타임라인 분석은 데이터 추출, 정규화, 정렬, 분석의 단계를 거치며, 최근에는 Plaso(log2timeline)와 같은 자동화 도구가 널리 쓰인다.

### 타임라인 생성 및 분석 아키텍처 (Plaso Workflow)

```text
  [Evidence Sources]           [Extraction & Parsing]         [Normalization & Sorting]
┌──────────────────┐       ┌──────────────────────┐      ┌──────────────────────────┐
│ File System (MFT)│ ──┐   │                      │      │                          │
├──────────────────┤   │   │  log2timeline.py     │      │     psort.py             │
│ Registry Hives   │ ──┼──▶│ (Artifact Parsing)   │ ──▶  │ (Sorting & Filtering)    │
├──────────────────┤   │   │                      │      │                          │
│ Event Logs, LNK  │ ──┘   └──────────────────────┘      └──────────────────────────┘
└──────────────────┘                                                 │
                                                                     ▼
                                                       ┌──────────────────────────┐
                                                       │  Timeline Analysis       │
                                                       │ (Excel, TimeSketch 등)   │
                                                       └──────────────────────────┘
```

### 주요 분석 시간 정보 (Artifacts)

| 구분 | 시간 유형 | 분석 포인트 |
|:---|:---|:---|
| **파일 시스템** | MACB (Mod, Acc, Change, Birth) | 파일 생성, 수정, 접근 기록 (MFT 분석) |
| **운영체제** | Event Log, Registry | 부팅 시각, 프로그램 실행 시각, USB 연결 기록 |
| **어플리케이션** | Browser History, Email | 외부 통신 시각, 악성 코드 다운로드 시점 |
| **네트워크** | Firewall Log, PCAP | 외부 C2 서버와의 통신 시작 및 종료 시각 |

- **📢 섹션 요약 비유**: 슈퍼 타임라인은 도시 전체의 CCTV를 하나의 큰 화면에 모아놓고 범인의 이동 동선을 1초 단위로 추적하는 통합 관제 시스템과 같다.

## Ⅲ. 비교 및 연결

타임라인 분석은 데이터의 범위에 따라 일반 타임라인과 슈퍼 타임라인으로 구분된다.

| 항목 | 일반 타임라인 (File Timeline) | 슈퍼 타임라인 (Super Timeline) |
|:---|:---|:---|
| 데이터 소스 | 파일 시스템 메타데이터 위주 | 로그, 레지스트리, 아티팩트 전체 포함 |
| 분석 범위 | 특정 폴더/파일 변화 추적 | 시스템 전반의 행위 재구성 |
| 장점 | 생성 속도가 빠름 | 매우 정밀하고 인과관계 규명에 탁월함 |
| 단점 | 정보량이 제한적임 (로그 등 누락) | 데이터 양이 방대하여 분석 부하가 큼 |
| 핵심 도구 | fls, mactime (Sleuthkit) | Plaso (log2timeline) |

슈퍼 타임라인 분석 결과는 `661. DFIR` 보고서의 핵심 뼈대가 되며, 발견된 시간의 불일치는 `674. 안티 포렌식` 기법 적용 여부를 판단하는 근거가 된다.

- **📢 섹션 요약 비유**: 일반 타임라인이 주인공의 행적만 쫓는 카메라라면, 슈퍼 타임라인은 조연과 주변 배경까지 모두 담아내는 풀샷 카메라와 같다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 방대한 타임라인 데이터 중에서 의미 있는 '신호'와 '소음'을 구분하는 능력을 갖춰야 한다.

### 실무 판단 가이드

1. **시간 동기화 확인**: 분석 대상 시스템의 타임존(Timezone) 설정이 UTC인지 로컬 시간인지 확인하고, 로그 간 시차를 보정했는가?
2. **Timestomping 탐지**: MFT의 `$Standard_Information`과 `$File_Name` 속성의 시간 차이를 비교하여 공격자의 시간 조작 여부를 판단했는가?
3. **분석 필터링**: 수백만 개의 라인 중에서 침해 사고 의심 시점 전후 1시간 등으로 범위를 좁혀 효율적인 분석을 수행하고 있는가?
4. **시각화 도구 활용**: 텍스트 형태의 타임라인을 Timesketch와 같은 도구로 시각화하여 공격자의 수평 이동(Lateral Movement) 패턴을 한눈에 파악했는가?

### 안티패턴
- 파일의 생성 시간(Creation Time)만 믿고 공격의 시작 시점을 단정하는 행위 (시간 조작 가능성 간과)
- 모든 데이터를 필터링 없이 엑셀로 열어 분석을 시도하다가 시스템이 멈추는 행위 (데이터 과부하)

- **📢 섹션 요약 비유**: 타임라인 분석 실무는 바다에서 바늘 찾기와 같다. 강력한 자석(Plaso)과 필터(Filter)를 사용하여 의미 있는 바늘(공격 흔적)을 걸러내야 한다.

## Ⅴ. 기대효과 및 결론

타임라인 분석은 디지털 포렌식을 '데이터 수집'에서 '스토리텔링'의 단계로 격상시킨다. 공격자의 침입부터 종료까지의 시나리오를 증거 기반으로 완벽하게 재구성함으로써 법적 증명력을 극대화하고 사고 대응의 교훈(Lessons Learned)을 명확히 도출할 수 있다.

결론적으로 타임라인 분석은 복잡해지는 현대의 보안 사고를 논리적으로 설명할 수 있는 유일한 열쇠다. 데이터의 양이 늘어날수록, 이를 효율적으로 통합하고 분석하는 슈퍼 타임라인 기술의 중요성은 더욱 커질 것이다.

- **📢 섹션 요약 비유**: 타임라인 분석은 디지털 세계의 '역사학'이다. 파편화된 유물들을 시간순으로 정리하여 잃어버린 진실을 찾아내는 숭고한 과정이다.

### 👶 어린이를 위한 3줄 비유 설명

1. 컴퓨터 안에서 일어난 일들을 시간 순서대로 일기장에 정리하는 거예요.
2. 어떤 사람이 몇 시에 들어와서 몇 분 동안 무엇을 했는지 영화처럼 볼 수 있어요.
3. 나쁜 사람이 시간을 속여도, 다른 기록들과 맞춰보면 거짓말인 걸 금방 들통나게 된답니다.
