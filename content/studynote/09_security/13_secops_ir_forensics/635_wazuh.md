## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Wazuh는 호스트 기반의 보안 모니터링(HIDS)을 바탕으로 SIEM과 EDR 기능을 통합 제공하는 강력한 오픈소스 보안 플랫폼이다.
> 2. **가치**: 엔드포인트에 설치된 에이전트를 통해 로그 분석, 무결성 검사(FIM), 루트킷 탐지, 취약점 점검을 수행하며 클라우드 보안 환경(AWS, Azure 등)까지 통합 관리한다.
> 3. **판단 포인트**: 무료로 사용할 수 있는 오픈소스임에도 불구하고 기능의 폭과 깊이가 상용 솔루션에 견줄 만큼 넓어, 가성비 높은 통합 보안 관제 환경 구축에 최적이다.

---

## Ⅰ. 개요 및 필요성

Wazuh는 유명한 오픈소스 HIDS인 OSSEC을 포크(Fork)하여 발전시킨 프로젝트다. 단순한 로그 수집기를 넘어, 호스트 내부에서 어떤 일이 벌어지는지(파일 변경, 설정 유실, 악성 프로세스 등)를 심층적으로 감시한다.

최근의 위협은 네트워크 경계 보안을 뚫고 내부 호스트로 직접 침투한다. Wazuh는 서버나 PC의 "말단(Endpoint)"에서 위협을 잡아내기 때문에, 네트워크 장비만으로는 알 수 없는 상세한 침해 징후를 포착할 수 있다. 또한 규제 준수(PCI DSS, HIPAA, NIST 등)를 위한 감사 보고서를 자동으로 생성해주어 관리자의 부담을 획기적으로 줄여준다.

- **📢 섹션 요약 비유**: Wazuh는 건물 입구의 보안 요원(네트워크 보안)이 아니라, 각 방안에 설치된 지능형 감지기(엔드포인트 보안)와 같다. 문을 열고 들어온 뒤에 방 안에서 벌어지는 수상한 행동을 잡아낸다.

## Ⅱ. 아키텍처 및 핵심 원리

Wazuh의 아키텍처는 Wazuh 에이전트, Wazuh 서버, 그리고 분석 및 시각화를 위한 ELK 스택(또는 Wazuh 인덱서/대시보드)으로 구성된다.

| 구성 요소 | 역할 | 상세 설명 |
|:---|:---|:---|
| Wazuh Agent | 데이터 수집 및 실행 | 호스트에 설치되어 로그 전송, FIM 수행, 대응 스크립트 실행 |
| Wazuh Server | 제어 및 분석 | 에이전트 관리, 분석 엔진(Decoder/Rule)을 통한 위협 탐지 |
| Wazuh Indexer | 데이터 저장 | 탐지된 위협 이벤트와 로그를 저장하는 고성능 검색 엔진 |
| Wazuh Dashboard | 인터페이스 | 시각화 대시보드, 에이전트 상태 모니터링, 취약점 보고서 조회 |

```text
┌──────────────────────────────────────────────────────────────┐
│            Wazuh의 다층 방어 체계: 탐지에서 대응까지           │
├──────────────────────────────────────────────────────────────┤
│ [Endpoint A] ──▶ [Wazuh Agent] ──┐                           │
│ (Logs, FIM)      (Collect/FIM)   │                           │
│                                  │ (Analysis Engine)         │
│ [Endpoint B] ──▶ [Wazuh Agent] ──┴─▶ [Wazuh Server] ───────┐ │
│ (Syscheck, Vuln) (Monitor)          (Decoders/Rules)       │ │
│                                                            │ │
│ [Security Admin] ◀── [Dashboard] ◀── [Wazuh Indexer] ──────┘ │
└──────────────────────────────────────────────────────────────┘
```

Wazuh의 핵심 탐지 원리는 **데코더(Decoder)**와 **룰(Rule)**이다. 수집된 원천 로그를 데코더가 해석하여 의미 있는 필드로 분리하고, 이를 수만 개의 보안 룰과 대조한다. 특히 **FIM (File Integrity Monitoring)** 기능은 중요한 시스템 파일이 변경될 때마다 "누가, 언제, 어떤 명령으로" 바꿨는지 실시간으로 추적하여 백도어 설치나 설정 조작을 원천 차단한다.

- **📢 섹션 요약 비유**: Wazuh 아키텍처는 중앙 관제실(서버)과 현장 요원(에이전트) 시스템이다. 현장 요원이 구석구석을 살피며 무전(로그)을 치면, 관제실에서 상황을 판단해 필요시 현장 요원에게 즉각 조치(Active Response)를 지시한다.

## Ⅲ. 비교 및 연결

Wazuh는 SIEM과 EDR의 경계에 걸쳐 있으며, 특히 오픈소스 보안 도구들 사이에서 '허브' 역할을 한다.

| 항목 | Wazuh | 일반적인 SIEM (Splunk 등) |
|:---|:---|:---|
| 주요 소스 | 호스트 행위 및 시스템 로그 | 네트워크 및 보안 장비 로그 중심 |
| EDR 기능 | 강함 (프로세스 탐지, 원격 대응) | 약함 (별도 EDR 연동 필요) |
| 취약점 관리 | 내장됨 (NVD 연동 자동 스캔) | 보통 별도 솔루션(Nessus 등) 연동 |
| 설치 환경 | 온프레미스/클라우드/컨테이너 | 환경에 따라 다양함 |

Wazuh는 최근 AWS, Azure, GCP 등 주요 퍼블릭 클라우드의 API와 연동하여 클라우드 설정 오류나 비정상적인 리소스 사용을 탐지하는 CSPM (Cloud Security Posture Management) 기능까지 확장되고 있다.

- **📢 섹션 요약 비유**: Wazuh가 모든 기능을 다 갖춘 '맥가이버 칼'이라면, 일반 SIEM은 거대한 '전투 지휘소'와 같다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 Wazuh 운영의 묘미는 '액티브 리스폰스(Active Response)'와 '취약점 탐지'의 활용이다. 위협이 탐지되었을 때 자동으로 방화벽 IP를 차단하거나 계정을 잠그는 등의 시나리오를 구성할 수 있다.

### 체크리스트
1. FIM 설정 시 부하를 줄이기 위해 감시가 불필요한 디렉토리를 제외(Ignore)했는가?
2. 최신 취약점 정보를 반영하기 위해 NVD(National Vulnerability Database) 피드 업데이트가 정상 작동하는가?
3. 에이전트와 서버 간의 통신 구간이 암호화(AES)되어 보호받고 있는가?

### 안티패턴
- 에이전트가 설치된 서버의 자원(CPU/MEM) 상태를 고려하지 않고 너무 잦은 주기(Scan Interval)로 전체 파일 검사를 수행하는 경우
- 탐지된 위협에 대해 알람만 울리고, Wazuh의 강점인 자동 대응 기능을 전혀 쓰지 않는 경우

- **📢 섹션 요약 비유**: Wazuh 운영은 지능형 CCTV 시스템을 관리하는 것과 같다. 단순히 녹화(로그 저장)만 하지 말고, 침입자가 나타나면 자동으로 문을 잠그는(Active Response) 기능을 써야 진짜 효과가 난다.

## Ⅴ. 기대효과 및 결론

Wazuh를 도입하면 추가적인 라이선스 비용 없이도 대규모 인프라에 대한 실시간 위협 탐지 및 대응 체계를 구축할 수 있다. 특히 컨테이너 보안(Docker/K8s) 기능까지 지원하므로, 현대적인 마이크로서비스 아키텍처(MSA) 환경에서도 강력한 보안 방패가 된다.

결론적으로 Wazuh는 "가장 실무적이고 강력한 오픈소스 보안 플랫폼"이다. 복잡한 보안 도구들을 하나씩 따로 운영하는 대신, Wazuh라는 하나의 플랫폼 위에서 통합 보안의 시너지를 낼 수 있다.

- **📢 섹션 요약 비유**: Wazuh는 보안팀에게 '강력한 장갑차'를 선물하는 것과 같다. 거친 위협 현장 속으로 들어가 위협을 직접 맞서 싸우고 방어할 수 있는 힘을 준다.

| 개념 | 연결 포인트 |
|:---|:---|
| FIM | 파일의 변조 여부를 실시간으로 감시하는 Wazuh의 핵심 기능 |

| OSSEC | Wazuh의 뿌리가 되는 오픈소스 HIDS 프로젝트 |
