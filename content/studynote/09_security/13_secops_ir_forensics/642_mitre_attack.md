## 핵심 인사이트 (3줄 요약)
> 1. **본질**: MITRE ATT&CK은 실제 침해 사고 사례를 바탕으로 공격자의 전술(Tactics), 기법(Techniques), 세부 절차(Procedures)를 체계적으로 분류하고 데이터베이스화한 지식 베이스(Knowledge Base)다.
> 2. **가치**: 단순한 파일 해시나 IP 주소 같은 지표(IoC)를 넘어, 공격자가 "어떤 의도로 어떻게 행동하는가"에 대한 공통 언어를 제공하여 공격자 중심의 방어 전략 수립을 가능하게 한다.
> 3. **판단 포인트**: ATT&CK 프레임워크는 단순히 읽는 문서가 아니라, 조직의 탐지 가시성을 측정하고 방어 체계의 빈틈(Gap)을 찾아내기 위한 '보안 지도'로 활용되어야 한다.

---

## Ⅰ. 개요 및 필요성

MITRE ATT&CK (Adversarial Tactics, Techniques, and Common Knowledge)은 2013년 미국의 MITRE 사가 개발한 프레임워크다. 기존의 보안이 "무엇을 차단할 것인가"에 집중했다면, ATT&CK은 "공격자가 지금 무엇을 하고 있는가"를 이해하는 데 집중한다.

사이버 공격은 단발성 이벤트가 아니라 일련의 과정이다. 공격자가 초기 침투에 성공했더라도, 내부에서 권한을 상승시키고 정보를 유출하기까지는 수많은 단계가 필요하다. ATT&CK은 이 과정을 세분화하여 각 단계별 공격 수법을 정의함으로써, 방어자가 공격의 흐름 중 어느 한 지점에서라도 적을 포착하고 차단할 수 있는 기회를 제공한다.

- **📢 섹션 요약 비유**: MITRE ATT&CK은 보안계의 '범죄 수법 대백과사전'과 같다. 범인이 담을 넘는 법, 금고를 따는 법, 흔적을 지우는 법을 모두 정리해두어 경찰이 미리 대비하게 해준다.

## Ⅱ. Tactic, Technique, Procedure (TTP) 관계

ATT&CK의 핵심 구조인 TTP는 공격자의 행위를 입체적으로 분석하게 해준다.

| 구성 요소 | 질문 | 정의 | 예시 |
|:---|:---|:---|:---|
| **Tactic (전술)** | **Why?** | 공격자의 전략적 목표 (의도) | 권한 상승 (Privilege Escalation) |
| **Technique (기법)** | **How?** | 목표를 달성하기 위한 구체적 수단 | 프로세스 주입 (Process Injection) |
| **Procedure (절차)** | **Exact Details** | 기법을 구현하는 세부 실행 단계 | 특정 툴(예: Cobalt Strike)을 이용한 원격 주입 스크립트 실행 |

```text
┌──────────────────────────────────────────────────────────────┐
│           TTP 계층 구조: 의도에서 실행까지의 흐름               │
├──────────────────────────────────────────────────────────────┤
│ [Tactic: Persistence] ─────▶ "공격자는 시스템에 계속 남고 싶다"  │
│          │ (의도)                                            │
│          ▼                                                   │
│ [Technique: Create Account] ──▶ "그래서 가짜 계정을 새로 만든다" │
│          │ (수단)                                            │
│          ▼                                                   │
│ [Procedure: 'net user /add'] ─▶ "실제로 이 명령어를 입력한다"    │
│            (실행)                                            │
└──────────────────────────────────────────────────────────────┘
```

이 관계의 핵심은 **Procedure**를 막는 것은 쉽지만(시그니처 차단), **Tactic**은 공격자의 본질적인 목표이므로 쉽게 변하지 않는다는 점이다. 따라서 방어자는 상위 계층인 Tactic과 Technique을 이해하고 이에 대한 탐지 로직을 갖춰야 한다.

- **📢 섹션 요약 비유**: 전술은 '배고픔을 해결하자'는 목표고, 기법은 '낚시를 하자'는 방법이며, 절차는 '지렁이를 꿰어 3번 던진다'는 구체적인 행동이다.

## Ⅲ. 비교 및 연결

ATT&CK은 다른 보안 프레임워크와 보완적인 관계를 가진다.

| 항목 | Cyber Kill Chain (Lockheed Martin) | MITRE ATT&CK |
|:---|:---|:---|
| 관점 | 공격자의 선형적 단계 (High-level) | 공격자의 행위 중심 (Granular-level) |
| 유연성 | 고정된 7단계 구조 | 유동적인 매트릭스 구조 (비순차적) |
| 상세도 | 단계별 설명 중심 | 수백 개의 세부 기법과 대응 사례 포함 |
| 주요 용도 | 전략적 방어 지점 설정 | 탐지 역량 평가, 레드팀/블루팀 훈련 |

ATT&CK은 **SIEM**에서 탐지된 이벤트에 '공격 맥락'을 부여하는 태그로 쓰이며, **EDR** 솔루션의 탐지 로그를 분류하는 표준 기준으로 자리 잡았다. 또한 **Threat Intelligence** 보고서에서 공격 그룹의 특성을 설명할 때 필수적으로 인용된다.

- **📢 섹션 요약 비유**: 킬 체인이 "범인이 침입해서 도망가기까지의 큰 줄거리"라면, ATT&CK은 "범인이 장갑을 꼈는지, 가짜 신분증을 썼는지"와 같은 세세한 행동 기록부다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 ATT&CK의 가장 강력한 활용법은 **'공격 방어 커버리지(Heatmap)'** 분석이다. 우리 조직의 보안 장비가 ATT&CK의 어떤 기법을 막을 수 있고 어떤 기법에 취약한지 시각화하여 보여주는 것이다.

### 체크리스트
1. 현재 운영 중인 보안 로그(EDR, Sysmon 등)가 ATT&CK의 주요 Technique을 탐지할 수 있는 데이터를 포함하고 있는가?
2. 최신 위협 리포트를 읽을 때, 해당 공격자가 사용한 기법을 ATT&CK ID(예: T1059)로 매핑하여 관리하는가?
3. 레드팀 훈련 시 ATT&CK의 특정 기법을 시나리오에 반영하여 탐지 체계를 검증하는가?

### 안티패턴
- 수백 개의 ATT&CK 기법을 모두 막으려고 시도하는 경우 (우리 산업군을 노리는 주요 기법부터 집중해야 함)
- ATT&CK 매트릭스의 빈칸을 채우는 것에만 급급하여 실제 탐지 성능(정밀도)을 무시하는 경우

- **📢 섹션 요약 비유**: ATT&CK 활용은 건강검진 항목을 체크하는 것과 같다. 모든 항목을 다 검사할 수는 없으므로, 내 나이와 증상(비즈니스 환경)에 맞는 중요한 항목부터 꼼꼼히 체크해야 한다.

## Ⅴ. 기대효과 및 결론

MITRE ATT&CK은 보안 분야에 '표준 언어'를 선사했다. 덕분에 전 세계 보안 전문가들은 서로 다른 솔루션을 쓰더라도 동일한 기준으로 위협을 논의하고 협력할 수 있게 되었다. 이는 방어자의 지식 격차를 줄이고 전체적인 보안 수준을 높이는 데 결정적인 역할을 한다.

결론적으로 MITRE ATT&CK은 보안 운영의 '나침반'이다. 막연한 공포가 아닌 데이터에 기반한 위협 분석을 가능하게 함으로써, 조직이 한정된 자원을 어디에 집중해야 할지 명확한 가이드를 제공한다.

- **📢 섹션 요약 비유**: ATT&CK은 보안팀에게 '입체 전략 지도'를 주는 것과 같다. 적이 어디에 숨어 있고 어떤 경로로 움직일지 예측하게 해주는 가장 믿음직한 도구다.
