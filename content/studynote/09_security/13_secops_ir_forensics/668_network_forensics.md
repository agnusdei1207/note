## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크 포렌식(Network Forensics)은 네트워크 통신 트래픽을 캡처, 기록, 분석하여 침입 탐지, 데이터 유출 경로 확인, 공격자의 의도를 파악하는 디지털 포렌식의 한 분야다.
> 2. **가치**: 호스트 내부 증거(디스크, 메모리)만으로 파악하기 힘든 '공격자의 이동 경로'와 '외부와의 통신 내역'을 입증하며, 휘발성 트래픽을 기록하여 사후 재구성(Traffic Reconstruction)을 가능하게 한다.
> 3. **판단 포인트**: 모든 패킷을 저장(PCAP)할 것인지, 메타데이터만 저장(NetFlow)할 것인지 데이터 용량과 분석의 정밀도 사이의 균형을 고려하여 로깅 정책을 수립해야 한다.

---

## Ⅰ. 개요 및 필요성

네트워크 포렌식은 사고 대응 시 공격자가 시스템에 들어온 '입구'와 나간 '출구'를 찾는 작업이다. 호스트 포렌식이 사건 현장 내부의 지문을 찾는 것이라면, 네트워크 포렌식은 현장 주변의 도로 상황과 범인이 타고 온 차량의 경로를 분석하는 것과 같다.

최근 공격은 내부망 이동(Lateral Movement)과 C2(Command & Control) 서버를 통한 원격 제어가 핵심이다. 이를 규명하기 위해서는 패킷 수준의 정밀 분석이 필수적이다. 특히 로그가 삭제된 상황에서도 네트워크 상에 흐르는 패킷은 속일 수 없기 때문에 객관적인 증거로서 높은 가치를 지닌다.

- **📢 섹션 요약 비유**: 네트워크 포렌식은 고속도로 톨게이트의 '하이패스 기록'과 'CCTV'를 분석하는 것과 같다. 어떤 차가 언제 들어왔고, 그 안에 무엇이 들어있었는지 확인하는 과정이다.

---

## Ⅱ. 아키텍처 및 핵심 원리

네트워크 포렌식은 데이터의 수집 깊이에 따라 Full Packet Capture 방식과 Flow 기반 방식으로 나뉜다.

### 데이터 수집 방식 및 아키텍처

| 방식 | 수집 데이터 | 장점 | 단점 |
|:---|:---|:---|:---|
| **Full Packet Capture (PCAP)** | 페이로드 포함 모든 패킷 | 완벽한 데이터 재구성 가능 | 막대한 저장 공간 필요 |
| **Flow-based (NetFlow)** | 5-tuple (Src/Dst IP, Port, Proto) | 장기간 통계 분석 용이 | 패킷 내부(페이로드) 확인 불가 |

### 분석 시스템 구성 (Network Monitoring Setup)

```text
  [Internal Network]           [Capture Point]            [Analysis System]
┌──────────────────┐      ┌──────────────────────┐      ┌────────────────────┐
│   Host A / B / C │      │   TAP / Span Port    │      │  Snort / Suricata  │
│      Traffic     │ ──▶  │ (Packet Mirroring)   │ ──▶  │  (IDS/IPS Analysis)│
└──────────────────┘      └──────────────────────┘      └────────────────────┘
                                     │                           │
                                     ▼                           ▼
                          ┌──────────────────────┐      ┌────────────────────┐
                          │ PCAP Storage (Arkime)│      │ Wireshark / Zeek   │
                          │ NetFlow (ELK Stack)  │      │ (Packet Inspection)│
                          └──────────────────────┘      └────────────────────┘
```

- **📢 섹션 요약 비유**: PCAP은 통화 내용 전체를 녹음하는 것이고, NetFlow는 통화 일시와 상대방 번호만 기록하는 통화 명세서와 같다.

## Ⅲ. 비교 및 연결

네트워크 포렌식은 호스트 포렌식과 결합될 때 비로소 완전한 시나리오를 구성할 수 있다.

| 항목 | 호스트 포렌식 (Host) | 네트워크 포렌식 (Network) |
|:---|:---|:---|
| 주요 증거 | 파일, 레지스트리, 프로세스 | 패킷, 세션 로그, 흐름 정보 |
| 장점 | 실행된 행위의 상세 결과 확인 | 공격의 시작과 이동 경로 확인 |
| 분석 범위 | 개별 단말 내부 | 망 전체의 통신 흐름 |
| 연결 개념 | `665. 메모리 포렌식` | IDS/IPS, SIEM, NDR |

분석 과정에서 발견된 특정 시간대의 비정상적인 트래픽 증가는 해당 호스트의 `670. 타임라인 분석`과 일치해야 하며, 이 일치 여부가 증거의 신뢰성을 높여준다.

- **📢 섹션 요약 비유**: 호스트 포렌식이 도둑이 훔쳐간 물건 목록을 적는 것이라면, 네트워크 포렌식은 도둑이 담장을 넘는 장면을 포착하는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 암호화 트래픽(HTTPS 등) 비중이 높은 현대 환경에서 어떻게 유의미한 분석 결과를 낼지 판단해야 한다.

### 실무 판단 가이드

1. **암호화 트래픽 대응**: SSL/TLS 가시성 장비(SSL Decryption)를 통해 복호화 후 캡처할 것인가, 아니면 트래픽 패턴(JA3 Hash 등) 분석에 집중할 것인가?
2. **저장 용량 관리**: 중요도가 높은 DMZ 구간은 PCAP을 수행하고, 일반 내부망은 NetFlow만 수행하는 계층적 로깅 전략을 수립했는가?
3. **증거 무결성**: 캡처된 PCAP 파일에 대해 타임스탬프를 부여하고 해시값을 생성하여 변조를 방지했는가?
4. **시간 동기화**: 분석 시스템과 네트워크 장비 간의 NTP(Network Time Protocol) 동기화가 정확하여 로그 간 시차가 없는가?

### 안티패턴
- 암호화된 패킷을 보며 내용이 보이지 않는다고 분석을 포기하는 행위 (패턴과 목적지 IP만으로도 많은 정보 획득 가능)
- 네트워크 장비의 버퍼 용량을 고려하지 않고 무작정 캡처를 시작하여 네트워크 장애를 유발하는 행위

- **📢 섹션 요약 비유**: 네트워크 포렌식 실무는 낚시와 같다. 그물을 어디에 던질지(TAP 위치), 어떤 물고기만 잡을지(Filter 설정)를 잘 정해야 헛수고를 줄일 수 있다.

## Ⅴ. 기대효과 및 결론

네트워크 포렌식은 보안 사고의 '미싱 링크(Missing Link)'를 연결해준다. 공격자가 사용한 취약점(Exploit)을 실시간으로 확인하고, 유출된 데이터의 정확한 양과 종류를 파악함으로써 사고 피해 규모를 정밀하게 산정할 수 있게 한다.

결론적으로 네트워크 포렌식은 경계 보안(Perimeter)이 무너진 제로 트러스트 시대에 모든 통신을 감시하고 기록하는 '디지털 블랙박스'로서 그 역할이 더욱 막중해지고 있다.

- **📢 섹션 요약 비유**: 네트워크 포렌식은 끝없는 추격전의 네비게이션과 같다. 공격자가 어디로 도망가든 그가 남긴 데이터의 흔적을 따라 끝까지 추적한다.
