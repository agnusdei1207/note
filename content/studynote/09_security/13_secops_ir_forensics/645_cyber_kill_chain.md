## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사이버 킬 체인 (Cyber Kill Chain)은 공격자가 목표를 달성하기 위해 거치는 7단계를 군사적 타격 체계 관점에서 정의한 선형적 공격 모델이다.
> 2. **가치**: 공격의 전 과정을 가시화하여, 방어자가 각 단계 중 어느 한 곳만이라도 성공적으로 차단하면 전체 공격을 무력화할 수 있다는 '방어의 기회'를 제시한다.
> 3. **판단 포인트**: 전통적인 외부 침입 방어에는 효과적이지만, 이미 침투한 이후의 내부 횡적 이동(Lateral Movement)이나 클라우드 환경의 복잡한 공격을 설명하기에는 한계가 있음을 인지해야 한다.

---

## Ⅰ. 개요 및 필요성

사이버 킬 체인 (Cyber Kill Chain)은 록히드 마틴(Lockheed Martin)이 제안한 모델로, 지능형 지속 위협(APT)에 대응하기 위한 전략적 프레임워크다. 이 모델의 핵심 아이디어는 "공격자는 모든 단계를 성공시켜야 하지만, 방어자는 단 한 단계만 끊어내면 된다"는 것이다.

현대 보안은 완벽한 차단이 불가능하다는 전제에서 시작한다. 킬 체인은 공격자가 우리 네트워크에 발을 들이기 전(정찰)부터 최종 목적을 달성하기까지(행동)의 경로를 7단계로 쪼개어 보여줌으로써, 보안 팀이 각 단계별로 어떤 방어 도구와 대응 절차를 갖춰야 할지 알려주는 이정표 역할을 한다.

- **📢 섹션 요약 비유**: 사이버 킬 체인은 '은행 강도의 범행 과정'과 같다. 사전 답사부터 도주까지의 과정 중, CCTV에 찍히거나(정찰 차단) 문을 잠그는(침투 차단) 등 어느 한 단계만 막아도 범행은 실패한다.

---

## Ⅱ. 7단계 아키텍처 및 핵심 원리

사이버 킬 체인은 다음과 같은 7단계의 연속된 과정으로 구성된다.

| 단계 | 명칭 | 설명 | 방어 예시 |
|:---|:---|:---|:---|
| 1 | Reconnaissance (정찰) | 대상 탐색, 이메일 수집, 취약점 스캔 | 웹 로그 분석, OSINT 차단 |
| 2 | Weaponization (무기화) | 악성코드와 취약점 공격 도구 결합 (결합물 생성) | 파일 무결성 검사, 위협 분석 |
| 3 | Delivery (전송) | 이메일, USB, 웹 등을 통해 무기 전달 | 이메일 필터링, USB 통제 |
| 4 | Exploitation (취약점 공격) | 대상 시스템의 약점을 이용해 코드 실행 | 패치 관리, DEP/ASLR 적용 |
| 5 | Installation (설치) | 시스템에 백도어 또는 악성코드 설치 (지속성 확보) | 백신(AV), EDR 모니터링 |
| 6 | Command & Control (C2) | 공격자 서버와 원격 제어 채널 구축 | C2 IP/도메인 차단, 트래픽 분석 |
| 7 | Actions on Objectives (행동) | 데이터 유출, 시스템 파괴 등 최종 목적 달성 | DLP, 암호화, 가용성 복구 |

```text
┌──────────────────────────────────────────────────────────────┐
│           Cyber Kill Chain: 7 Steps to a Successful Attack     │
├──────────────────────────────────────────────────────────────┤
│  [1. Recon] ──▶ [2. Weaponize] ──▶ [3. Deliver] ──▶ [4. Exploit] │
│      │               │                │               │      │
│      └─────── (공격 준비 단계) ────────┴───── (공격 실행 단계) ────┘      │
│                                                              │
│  [5. Install] ──▶ [6. C2 Channel] ──▶ [7. Actions on Obj]    │
│      │                 │                    │                │
│      └─────── (지속성 및 제어 단계) ───────┴───── (최종 목표 달성) ────┘    │
└──────────────────────────────────────────────────────────────┘
```

이 모델의 핵심 원리는 **'연쇄 고리의 절단'**이다. 상위 단계(정찰, 전송)에서 막을수록 방어 비용이 저렴하고 피해가 적으며, 하위 단계(행동)로 갈수록 방어 난이도가 높고 피해가 커진다. 따라서 보안 전략은 킬 체인의 최대한 앞 단계에서 위협을 탐지하도록 설계되어야 한다.

- **📢 섹션 요약 비유**: 킬 체인은 '도미노'와 같다. 첫 번째 도미노(정찰)가 쓰러지기 시작하면 끝까지 가려는 성질이 있지만, 중간에 하나만 빼버리면 전체 도미노는 멈추게 된다.

## Ⅲ. 비교 및 연결

사이버 킬 체인은 이후 등장한 MITRE ATT&CK과 자주 비교된다.

| 항목 | Cyber Kill Chain | MITRE ATT&CK |
|:---|:---|:---|
| 구조 | 선형적 (Linear) | 행렬/매트릭스형 (Matrix) |
| 시점 | 외부에서 내부로의 침입 중심 | 침투 이후의 행위 및 기법 중심 |
| 상세도 | 높음 (전략적 단계) | 매우 높음 (기술적 기법) |
| 장점 | 공격 흐름 파악이 직관적임 | 실제 탐지 로직 구현에 유리함 |

최근의 보안 관제는 킬 체인의 **'전략적 흐름'** 위에 ATT&CK의 **'기술적 상세'**를 얹어서 운영한다. 예를 들어 "킬 체인의 6단계(C2)를 수행하기 위해 공격자가 ATT&CK의 'T1071(Application Layer Protocol)' 기법을 사용했다"는 식으로 결합하여 분석한다.

- **📢 섹션 요약 비유**: 킬 체인이 "범죄 영화의 전체 시나리오"라면, ATT&CK은 "장면마다 범인이 사용하는 도구 세트"와 같다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사적 관점에서 킬 체인의 한계를 극복하기 위해서는 '포괄적 방어(Defense in Depth)'가 필요하다. 킬 체인은 외부 침입에는 강하지만, 내부자에 의한 위협이나 계정 도용을 통한 정상적인 접근은 설명하기 어렵다.

### 체크리스트
1. 킬 체인의 각 단계별로 대응하는 보안 솔루션이 누락 없이 배치되어 있는가? (Gap 분석)
2. 각 솔루션이 서로 정보를 공유하여 1단계 탐지 정보가 7단계 방어에 활용되는가?
3. 킬 체인의 앞 단계(1~3단계) 탐지율을 높이기 위해 TI(위협 인텔리전스)를 적극 활용하는가?

### 안티패턴
- 킬 체인의 특정 단계(예: 방화벽/백신)에만 과도하게 투자하고 다른 단계는 방치하는 경우
- 공격자가 단계를 건너뛰거나(예: 공급망 공격) 반복할 수 있다는 점을 간과하는 경우

- **📢 섹션 요약 비유**: 킬 체인 운영은 '성곽 수비'와 같다. 성문(Delivery)만 지키지 말고, 성벽을 타고 넘어오거나(Exploit) 내부에서 반란을 일으키는(Actions) 상황까지 단계별 수비대(보안 장비)를 배치해야 한다.

## Ⅴ. 기대효과 및 결론

사이버 킬 체인은 보안을 막연한 기술의 집합이 아니라, 체계적인 '프로세스'로 이해하게 만들었다. 이를 통해 조직은 공격자의 행동 양식을 이해하고, 한정된 자원을 효율적으로 배치하여 방어의 성공 확률을 높일 수 있다.

결론적으로 사이버 킬 체인은 현대 보안 전략의 '클래식 지침서'다. 비록 매트릭스 형태의 최신 모델들에게 자리를 내주고 있지만, "연결된 고리를 끊는다"는 근본 원리는 여전히 모든 보안 설계의 핵심 철학으로 살아있다.

- **📢 섹션 요약 비유**: 사이버 킬 체인은 보안팀에게 '승리의 공식'을 알려준다. 적의 행로를 알고 길목을 지키면, 어떤 강한 적이라도 이길 수 있다는 믿음을 준다.
