## 핵심 인사이트 (3줄 요약)
1. **본질**: CPU를 거치지 않고 주변 장치가 시스템 메모리(RAM)에 직접 접근할 수 있는 DMA 권한을 악용하여, 외부 포트를 통해 메모리 데이터를 탈취하거나 조작하는 공격이다.
2. **가치**: OS의 권한 제어(Access Control) 체계를 완전히 우회하여 커널 메모리까지 읽고 쓸 수 있다는 점에서 엔드포인트 보안의 치명적인 하이패스 구간이다.
3. **판단 포인트**: Thunderbolt, PCIe 등 고속 인터페이스의 사용이 필수적인 현대 환경에서는 IOMMU(VT-d/AMD-Vi) 기반의 DMA 재매핑과 포트 수준의 보안 정책이 방어의 핵심이다.

---

## Ⅰ. 개요 및 필요성

DMA(Direct Memory Access)는 원래 시스템 성능 향상을 위해 도입되었다. 하드디스크나 네트워크 카드 같은 주변 장치가 CPU의 개입 없이 메모리와 직접 데이터를 주고받음으로써 CPU 부하를 줄이는 기술이다. 하지만 이 '직통 통로'가 보안 관점에서는 거대한 취약점이 된다.

공격자가 Thunderbolt, Firewire, ExpressCard 등 DMA를 지원하는 외부 포트에 악성 장치를 연결하면, OS의 로그인 화면을 우회하여 메모리에 있는 비밀번호, 암호화 키, 개인 정보를 즉시 긁어갈 수 있다. CPU가 개입하지 않으므로 바이러스 백신이나 일반적인 보안 소프트웨어로는 탐지가 불가능하다는 것이 가장 무서운 점이다.

- **📢 섹션 요약 비유**: DMA 공격은 건물의 보안 요원(CPU)이 지키고 있는 정문을 통과하는 대신, 물건 배달용 전용 승강기(DMA)를 타고 비밀 창고로 바로 들어가는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

DMA 공격은 주변 장치가 '버스 마스터(Bus Master)' 권한을 가질 수 있다는 점을 이용한다.

| 구성 요소 | 역할 | 보안 취약점 |
|:---|:---|:---|
| 주변 장치 | 데이터를 메모리로 전송 | 악성 장치가 시스템 버스 장악 가능 |
| DMA 컨트롤러 | CPU 대신 주소 및 제어 신호 발생 | 주소 검증 로직이 결여된 경우 전 영역 접근 가능 |
| 시스템 메모리 | 데이터 저장소 | 커널 영역, 암호 키 등이 평문으로 존재 |

```text
[ DMA Attack Architecture ]
[ Malicious Device ] <---(Thunderbolt/PCIe)---> [ System Bus ]
                                                    |
          (No CPU Intervention)  <------------------┘
          (Direct Access)        |
                                 v
                        [ System Memory (RAM) ]
                        - OS Kernel Memory
                        - Encryption Keys (BitLocker, etc.)
                        - Login Credentials
```

현대적인 공격 도구(예: PCILeech)는 PCIe 인터페이스를 통해 초당 수백 MB 속도로 메모리를 덤프하거나, 실시간으로 커널 코드를 패치하여 암호 입력을 무력화한다.

- **📢 섹션 요약 비유**: CPU라는 꼼꼼한 검사관을 거치지 않고 "옆집에서 보낸 택배예요"라며 메모리 안방으로 바로 들어오는 상황이다.

---

## Ⅲ. 비교 및 연결

DMA 공격은 접근 방식에 따라 여러 변종이 존재한다.

| 구분 | DMA 공격 (일반) | Thunderbolt 공격 | Firewire 공격 |
|:---|:---|:---|:---|
| **물리 포트** | PCIe 내부 슬롯 | Thunderbolt 3/4 (Type-C) | IEEE 1394 |
| **공격 편의성** | 하드웨어 분해 필요 | 외부 포트 연결만으로 가능 | 외부 포트 연결만으로 가능 |
| **방어 기술** | 물리적 잠금, IOMMU | Kernel DMA Protection | 포트 비활성화 |
| **최신 트렌드** | 서버 팜 내 악성 카드 | 노트북 분실/도난 시 공격 | 구형 장비 대상 |

이 공격은 **Kernel DMA Protection** 기술이 등장하게 된 직접적인 계기가 되었다. Windows 10/11과 최신 macOS는 IOMMU를 활용하여 승인되지 않은 주변 장치의 DMA 접근을 원천 차단한다.

- **📢 섹션 요약 비유**: Firewire가 옛날의 허술한 뒷문이었다면, Thunderbolt는 아주 빠르고 편리하지만 관리가 필요한 최신형 화물 통로다.

---

## Ⅳ. 실무 적용 및 기술사 판단

DMA 공격은 물리적 접근이 수반되므로, 실무적으로는 하드웨어와 소프트웨어의 결합 방어가 필수적이다.

### 방어 전략 체크리스트
1. **IOMMU (VT-d / AMD-Vi) 활성화**: 주변 장치가 접근할 수 있는 메모리 주소를 가상화하여 격리한다. (샌드박싱)
2. **Kernel DMA Protection**: OS 수준에서 부팅 전후의 DMA 요청을 검증하고, 잠금 화면 상태에서 신규 DMA 장치 연결을 차단한다.
3. **Thunderbolt Security Level 설정**: BIOS에서 'User Authorization' 모드를 설정하여 사용자가 승인한 장치만 연결되게 한다.
4. **미사용 포트 물리적/논리적 차단**: ExpressCard 슬롯이나 미사용 PCIe 슬롯을 비활성화한다.

### 기술사 판단
"DMA 공격은 하드웨어 편의성이 보안의 역행을 불러온 사례다. 보안 아키텍트는 하드웨어 가상화 기술(IOMMU)을 반드시 활성화하고, Zero Trust 관점에서 주변 장치(Peripheral)의 Identity를 검증하는 체계를 엔드포인트 보안 정책에 반영해야 한다."

- **📢 섹션 요약 비유**: 배달 전용 승강기 앞에 신분 확인기(IOMMU)를 설치하여, 허가받은 배달원만 지정된 층에 갈 수 있게 만드는 것이다.

---

## Ⅴ. 기대효과 및 결론

IOMMU와 Kernel DMA Protection을 적용하면, 외부 포트를 통한 메모리 덤프 위협으로부터 시스템을 안전하게 보호할 수 있다. 이는 특히 이동이 잦은 노트북 사용자나 외부 노출이 많은 키오스크 시스템에서 필수적인 방어선이 된다.

결론적으로, 성능을 위한 '지름길'인 DMA는 보안이라는 '검문소'를 반드시 거쳐야 한다. 하드웨어 수준의 주소 재매핑 기술은 이제 선택이 아닌 현대 컴퓨팅의 기본 인프라다.

- **📢 섹션 요약 비유**: 아무리 빠른 고속도로(DMA)라도 톨게이트(IOMMU)를 만들어 누가 어디로 가는지 통제해야 사고(해킹)를 막을 수 있다.

2. 나쁜 사람이 이 길로 몰래 들어와서 창고에 있는 보물(비밀번호)을 훔쳐가는 게 DMA 공격이에요.
3. 그래서 우리는 창고 앞에 똑똑한 경비원(IOMMU)을 세워서 허락받은 사람만 들어오게 한답니다!
