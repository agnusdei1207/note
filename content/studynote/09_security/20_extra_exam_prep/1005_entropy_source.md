## 핵심 인사이트 (3줄 요약)
1. **본질**: 암호학적으로 안전한 난수를 생성하기 위해 시스템이 외부 또는 내부에서 수집하는 '날것 그대로의 무작위성(Raw Randomness)'의 기원을 의미한다.
2. **가치**: 아무리 훌륭한 암호 알고리즘이라도 엔트로피 소스가 빈약하면 난수의 품질이 낮아져 공격자에게 예측될 수 있으므로, 보안의 신뢰 사슬(Chain of Trust)에서 가장 첫 번째 고리 역할을 한다.
3. **판단 포인트**: 환경적 제약(클라우드, 임베디드)에 따라 사용 가능한 소스가 다르므로, 여러 독립적인 엔트로피 소스를 혼합하여 '단일 장애점(Single Point of Failure)'을 방지하는 설계가 핵심이다.

---

## Ⅰ. 개요 및 필요성

암호학에서의 '안전성'은 공격자가 다음 비트를 예측할 확률이 정확히 $1/2$이어야 한다는 조건에서 출발한다. **Entropy Source**는 이 확률적 불확실성을 제공하는 원재료다. 컴퓨터 시스템은 본질적으로 결정론적(Deterministic)이지만, 물리적 환경이나 사용자의 비결정론적 행위로부터 이 원재료를 조달한다.

엔트로피 소스가 중요해진 이유는 현대의 분산 컴퓨팅 환경 때문이다. 가상화된 서버는 물리 하드웨어와 격리되어 엔트로피 수집이 어렵고, 부팅 직후의 임베디드 기기는 수집할 시간이 부족하다. 이러한 '엔트로피 고갈' 상태에서 만들어진 암호 키는 중복되거나 예측 가능하여, 전 세계적으로 수많은 취약점(예: Debian OpenSSL 예측 가능 키 사건)을 양산해왔다.

- **📢 섹션 요약 비유**: 엔트로피 소스는 맛있는 빵을 만들기 위한 '신선한 밀가루(무작위 비트)'와 같다. 밀가루가 상해 있거나 부족하면 아무리 좋은 오븐(알고리즘)이 있어도 제대로 된 빵(보안)을 만들 수 없다.

## Ⅱ. 아키텍처 및 핵심 원리

엔트로피 소스는 크게 물리적 소스와 비물리적 소스로 나뉜다.

### 1. 엔트로피 소스의 분류
| 분류 | 상세 소스 | 특징 |
|:---|:---|:---|
| **물리적 소스 (Physical)** | 열 잡음, 양자 현상, 클럭 지터(Jitter) | 고품질, 하드웨어 의존적, 재현 불가 |
| **비물리적 소스 (Non-physical)** | 키보드 입력 타이밍, 디스크 I/O 시간, 네트워크 인터럽트 | 소프트웨어적 수집, 사용자 행위 기반 |
| **외부 소스 (External)** | 네트워크 난수 서비스(Beacons), 전용 엔트로피 장치 | 대량 공급 가능, 네트워크 신뢰 필요 |

### 2. 수집 및 정제 아키텍처 (ASCII)
```text
[ Multiple Entropy Sources Architecture ]

[ Source A: Keyboard ] ---┐
[ Source B: Thermal ] ----┤    [ Mixing Function ]    [ Entropy Pool ]
[ Source C: Disk I/O ] ---┼--> (Hash/Conditioning) --> (Accumulate)
[ Source D: RDRAND ] -----┘            |                     |
                                       |                     v
                               [ Quality Check ]      [ Output to CSPRNG ]
```
수집된 날것의 데이터(Raw Digitized Values)는 통계적 편향(Bias)이 있을 수 있다. 이를 'Conditioning' 과정을 통해 엔트로피를 농축하고 고르게 펴서 엔트로피 풀(Pool)에 저장한다.

- **📢 섹션 요약 비유**: 여러 곳에서 흐르는 작은 냇물(소스)을 모아 필터(정제)로 거른 뒤 커다란 저수지(풀)에 가두어 관리하는 과정이다.

## Ⅲ. 비교 및 연결

엔트로피 소스의 품질은 암호 알고리즘의 안전성 수준을 결정한다.

| 비교 항목 | 물리적 엔트로피 소스 (TRNG) | 비물리적 엔트로피 소스 (OS Events) |
|:---|:---|:---|
| **신뢰의 근거** | 물리 법칙 (열역학, 양자역학) | 복잡한 시스템의 비결정성 |
| **품질(Entropy per bit)** | 매우 높음 (1.0에 수렴) | 낮음 (정제가 필수적) |
| **가용성** | 장치 지원 시에만 가능 | 모든 시스템에서 가능 |
| **공격 내성** | 관찰/예측이 원천적으로 불가능 | 상태 추측 공격에 상대적으로 취약 |

엔트로피 소스는 **Seed**와 연결된다. 암호학적 난수 발생기(CSPRNG)는 이 소스들로부터 얻은 짧은 '진짜 난수'를 시드로 삼아, 이를 수학적으로 길게 늘려 사용한다.

- **📢 섹션 요약 비유**: 물리 소스가 '하늘에서 내리는 비'라면, 비물리 소스는 '지나가는 사람들이 쏟은 물'을 모으는 것과 같다. 비가 오는 게 가장 확실하고 깨끗하다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 시스템의 특성에 맞는 엔트로피 소스 확보 전략을 수립해야 한다.

### 실무 전략 체크리스트
1. **임베디드/IoT 장비**: 부팅 시 엔트로피가 극히 부족하므로, `HWRNG`를 반드시 사용하거나 부팅 초기 데이터를 디스크에 저장했다가 재사용하는 'Seed File' 기법을 적용한다.
2. **클라우드 환경**: VM 간 엔트로피 간섭을 방지하기 위해 `VirtIO-RNG` 설정을 통해 호스트의 엔트로피를 게스트에게 직접 주입한다.
3. **건강성 측정 (Health Test)**: 소스가 고장 나거나(예: 센서 고착) 공격자에 의해 조작되는지 감시하기 위해 NIST SP 800-90B 표준에 따른 실시간 품질 테스트를 수행한다.

### 기술사 판단
"엔트로피 소스는 '다중화(Diversity)'가 핵심이다. 단일 소스에만 의존하면 해당 소스가 오염되었을 때 전체 암호 체계가 무너진다. 아키텍트는 하드웨어 소스(RDRAND 등)를 기본으로 하되, 시스템 이벤트를 결합하여 공격자가 모든 소스를 동시에 예측하거나 통제할 수 없도록 '엔트로피 방어선'을 구축해야 한다."

- **📢 섹션 요약 비유**: 요리할 때 한 가지 양념만 쓰지 않고 여러 가지를 섞어야(다중화) 한 가지가 상해도 전체 맛을 망치지 않는 것과 같다.

## Ⅴ. 기대효과 및 결론

고품질의 엔트로피 소스를 확보하면 암호화 키의 추측 공격(Guessing Attack)을 원천 차단하고, 시스템 보안의 '최초 신뢰점'을 견고하게 다질 수 있다. 이는 결과적으로 전체 IT 인프라의 보안 수준을 비약적으로 상향시킨다.

결론적으로 엔트로피 소스는 암호라는 거대한 성을 짓기 위한 '가장 단단한 주춧돌'이다. 이 기초가 부실하면 그 위에 세운 어떤 화려한 암호 알고리즘도 모래성일 뿐이다.

- **📢 섹션 요약 비유**: 엔트로피 소스는 보물 지도를 그릴 때 쓰는 '아무도 모르는 마법의 잉크'다. 이 잉크로 쓴 지도만이 보물을 안전하게 지킬 수 있다.
