## 핵심 인사이트 (3줄 요약)
1. **본질**: 전원이 차단된 후에도 DRAM에 데이터가 즉시 사라지지 않는 잔상 현상(Remanence)을 이용하여, 메모리를 냉각시킨 후 다른 시스템에 연결해 암호화 키 등을 탈취하는 물리적 공격이다.
2. **가치**: 운영체제가 실행 중인 상태에서만 유효하다고 믿었던 메모리 보안이, 전원이 꺼진 상태에서도 물리적 탈취의 대상이 될 수 있음을 보여준다.
3. **판단 포인트**: 소프트웨어적 보안(FDE)만으로는 부족하며, 메모리 암호화(TME/SME) 하드웨어 기술과 빠른 전원 차단 시 메모리 초기화 로직이 결합되어야 한다.

---

## Ⅰ. 개요 및 필요성

Cold Boot Attack(콜드 부트 공격)은 DRAM(Dynamic RAM)의 물리적 특성을 이용한다. 이론적으로 DRAM은 전원이 끊기면 데이터가 사라지지만, 실제로는 수초에서 수분 동안 데이터가 남아있는 '데이터 잔류 현상'이 발생한다. 특히 액체 질소나 에어 스프레이를 이용해 메모리 칩을 영하로 냉각시키면, 전원이 없는 상태에서도 데이터를 수 시간 동안 유지시킬 수 있다.

공격자는 암호화된 노트북의 전원을 강제로 끄거나 재부팅한 뒤, 메모리에 남아있는 디스크 복호화 키(예: BitLocker, FileVault의 키)를 덤프하여 데이터를 탈취한다. 이는 시스템이 '로그아웃'이나 '종료'되기 직전까지 메모리에 키를 올려두어야 하는 현대 OS의 구조적 취약점을 파고든다.

- **📢 섹션 요약 비유**: Cold Boot Attack은 칠판을 지웠지만 분필 가루 자국이 희미하게 남아있는 상태에서, 특수 조명을 비추어 원래 글씨를 읽어내는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

공격의 핵심은 전원 차단 후 데이터 소멸 시간을 물리적으로 지연시키는 것이다.

| 단계 | 행위 | 기술적 원리 |
|:---|:---|:---|
| 1. 기기 확보 | 구동 중이거나 절전 모드인 기기 탈취 | 메모리에 민감 데이터(키)가 로드된 상태 |
| 2. 급속 냉각 | 에어 스프레이 등을 거꾸로 뿌려 냉각 | 낮은 온도에서 캐패시터의 전하 방전 속도 저하 |
| 3. 물리적 이동 | 메모리 칩을 적출하거나 재부팅 유도 | 데이터가 소멸되기 전 공격용 시스템 연결 |
| 4. 메모리 덤프 | 메모리 전체 내용을 이미지로 복사 | 잔상 데이터를 바이너리 형태로 획득 |
| 5. 키 스케줄 복구 | 암호화 알고리즘 특유의 키 구조 검색 | 덤프 파일 내에서 암호화 키 식별 및 추출 |

```text
[ Cold Boot Attack Mechanism ]
[Power ON] -> [Key in RAM] --(Fast Power Off/Reset)--> [Residual Data]
                                    |
                          (Cooling with N2/Air Spray)
                                    |
                    [Frozen Data (Remanence increased)]
                                    |
                    [Extract Key via Scraping Tool]
```

DRAM의 각 셀(Cell)은 전하를 충전하여 데이터를 저장하는데, 온도가 낮아지면 전하가 빠져나가는 속도가 지수함수적으로 느려지는 물리 법칙을 이용한다.

- **📢 섹션 요약 비유**: 뜨거운 커피는 금방 식지만, 얼음 속에 넣은 커피는 오랫동안 차가움을 유지하듯 메모리의 데이터도 얼려서 보존하는 것이다.

## Ⅲ. 비교 및 연결

Cold Boot Attack은 메모리를 직접 공격한다는 점에서 사이드 채널 공격이나 런타임 공격과 비교된다.

| 구분 | Cold Boot Attack | Rowhammer Attack | DMA Attack |
|:---|:---|:---|:---|
| **공격 방식** | 물리적 잔상 취득 | 인접 셀 간섭 유도 (Bit Flip) | 외부 포트를 통한 메모리 직접 읽기 |
| **공격 시점** | 전원 차단 직후 (Off-line) | 시스템 가동 중 (In-use) | 시스템 가동 중 (In-use) |
| **핵심 기술** | 냉각 및 메모리 덤프 | 반복적 메모리 액세스 | 버스 마스터링 권한 획득 |
| **주요 탈취물** | 암호화 키, 로그인 세션 | 권한 상승, 데이터 조작 | 메모리 전 영역 데이터 |

이 공격은 특히 **FDE (Full Disk Encryption)** 시스템에 치명적이다. 디스크는 암호화되어 있어도 이를 푸는 '열쇠'가 메모리에 남아 있기 때문이다.

- **📢 섹션 요약 비유**: Cold Boot는 '남아있는 흔적'을 찾는 것이고, Rowhammer는 '흔적을 조작'하는 것이며, DMA는 '실시간으로 엿보는' 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

Cold Boot Attack은 물리적 접근이 가능한 상황에서 강력하므로, 실무적으로는 하드웨어와 OS 수준의 복합 대응이 필요하다.

### 방어 전략 및 판단
1. **메모리 암호화 (TME/SME)**: Intel TME(Total Memory Encryption)나 AMD SME를 사용하여 RAM에 저장되는 모든 데이터를 실시간 암호화한다. 키가 메모리 밖(CPU 내부)에 있으므로 덤프해도 무용지물이다.
2. **절전 모드 금지 (Hybernation 활용)**: 절전 모드(S3) 대신 최대 절전 모드(S4, 디스크 저장 후 종료)를 사용하면 RAM에서 키가 즉시 소멸된다.
3. **TNC (Trusted Network Connect)**: 기기 재부팅 시 TPM 무결성을 확인하고, 비정상적 부팅 시 키 방출을 차단한다.
4. **빠른 메모리 스크러빙 (Memory Scrubbing)**: BIOS/UEFI 설정에서 재부팅 시 메모리를 0으로 덮어쓰도록 강제한다.

### 기술사 판단
"Cold Boot Attack은 암호학적 알고리즘의 강도가 보안의 전부가 아님을 보여준다. 물리적 보안 영역인 하드웨어 암호화 가속기(Secure Enclave) 내부에만 마스터 키를 보관하고, 일반 RAM 영역에는 세션 키조차 암호화되어 저장되는 '하드웨어 기반 보안 아키텍처'로의 전환이 시급하다."

- **📢 섹션 요약 비유**: 칠판 내용을 다 쓴 뒤에는 물걸레로 깨끗이 닦아내(Scrubbing) 흔적조차 남기지 않는 습관이 필요하다.

## Ⅴ. 기대효과 및 결론

Cold Boot Attack 방어 기술(TME 등)을 도입하면, 기기 분실이나 물리적 탈취 상황에서도 데이터 기밀성을 획기적으로 높일 수 있다. 특히 암호화 화폐 지갑이나 기밀 문서를 다루는 워크스테이션에서 이 방어 체계는 필수적이다.

결론적으로, 메모리는 더 이상 '휘발성'이라는 이름 뒤에 숨어 안전을 보장받을 수 없다. 물리적 잔존 가능성을 상정한 'Zero-Residual' 보안 설계만이 데이터 영속성을 지키는 길이다.

- **📢 섹션 요약 비유**: 도둑이 쓰레기통(버려진 메모리 잔상)을 뒤져서 중요한 영수증을 찾아내지 못하도록, 모든 종이를 파쇄기(메모리 암호화)에 넣어 버리는 것과 같다.
