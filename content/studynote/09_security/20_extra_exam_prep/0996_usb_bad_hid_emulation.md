## 핵심 인사이트 (3줄 요약)
1. **본질**: USB 장치가 단순한 저장 매체가 아니라 키보드나 마우스 같은 HID(Human Interface Device)로 위장하여, 연결 즉시 악성 명령어를 자동 입력하는 공격 기법이다.
2. **가치**: 운영체제가 '키보드'를 신뢰한다는 근본적인 보안 전제를 악용하므로, 바이러스 백신이나 매체 제어 솔루션을 우회하여 시스템 권한을 획득할 수 있다.
3. **판단 포인트**: 'BadUSB' 위협을 방어하기 위해서는 포트 수준의 승인 정책(HID 인증)과 함께, 시스템 권한 상승 시 추가 인증(MFA)을 강제하는 다층 방어가 필수적이다.

---

## Ⅰ. 개요 및 필요성

USB_BAD(또는 BadUSB) 공격은 USB 컨트롤러 펌웨어를 조작하여 장치의 정체성을 속이는 데서 시작된다. 사용자는 단순한 USB 메모리라고 생각하고 꽂았지만, 실제로는 컴퓨터가 이를 '매우 빠른 속도로 타이핑하는 숙련된 해커의 키보드'로 인식하게 만든다.

이 공격이 무서운 이유는 기존의 보안 방식이 통하지 않기 때문이다. 대부분의 기업 보안 솔루션은 USB 저장 장치의 '데이터 유출'을 막는 데 집중하지만, '키보드 연결' 자체를 차단하는 경우는 드물다. 공격자는 이 틈을 타 터미널(CMD/PowerShell)을 열고 백도어를 다운로드하거나 시스템 설정을 변경하는 명령을 불과 수 초 만에 입력 완료한다.

- **📢 섹션 요약 비유**: USB_BAD는 겉모습은 택배 상자(저장 장치)인데, 상자를 열자마자 그 안에서 투명 인간 해커가 튀어나와 내 컴퓨터 키보드를 미친 듯이 두드리는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

USB_BAD는 USB 통신 프로토콜의 유연성을 악용한다.

| 구성 요소 | 역할 및 공격 기법 |
|:---|:---|
| **USB 펌웨어** | 장치의 클래스(HID, Mass Storage 등) 정의 조작 |
| **HID Profile** | 운영체제에 키보드/마우스로 신고하여 신뢰 획득 |
| **Payload** | 실행될 명령어 집합 (예: `Alt+F4`, `powershell.exe ...`) |
| **Injection** | 사람이 입력할 수 없는 속도로 명령어 자동 주입 |

```text
[ USB_BAD Attack Flow ]
1. Plugin: [ Malicious USB ] --(Identify as HID Keyboard)--> [ OS ]
2. Trust:  [ OS ] <--(Accept Input without Verification)---- [ OS ]
3. Attack: [ Malicious USB ] --(Type: Windows+R)-----------> [ OS ]
4. Execute: [ Malicious USB ] --(Type: powershell -encodedCommand ...)
5. Result: [ Backdoor Installed / Data Exfiltrated ]
```

공격자는 장치가 연결되자마자 실행될 'Ducky Script'와 같은 스크립트를 펌웨어에 내장한다. 운영체제는 키보드가 연결되었다고 판단하고, 그 키보드가 보내는 신호를 "사용자의 직접적인 입력"으로 간주하여 실행한다.

- **📢 섹션 요약 비유**: 신분증(펌웨어)을 위조하여 관리직원(OS)에게 "나는 정직한 키보드요"라고 속인 뒤, 사무실에 들어가 맘대로 서류를 고치는 상황이다.

---

## Ⅲ. 비교 및 연결

USB_BAD는 일반적인 USB 악성코드와는 그 궤를 달리한다.

| 구분 | 일반 USB 악성코드 (Autorun) | USB_BAD (BadUSB) |
|:---|:---|:---|
| **공격 위치** | 저장 영역 내 파일 | **USB 컨트롤러 펌웨어** |
| **신뢰 기반** | 파일 실행 권한 | **입력 장치(HID) 신뢰** |
| **백신 탐지** | 파일 스캔으로 탐지 가능 | **탐지 불가** (정상 키보드 입력으로 보임) |
| **방어 기술** | 자동 실행 방지, 백신 | HID 인증, 포트 화이트리스트 |

이 공격은 하드웨어 수준의 신뢰(Root of Trust)가 얼마나 중요한지를 보여준다. 최근에는 **USB-C**와 **Thunderbolt** 포트에서도 동일한 원리의 공격이 가능하며, 충전 케이블에 이 기능을 숨긴 사례(OMG Cable)도 등장했다.

- **📢 섹션 요약 비유**: 일반 악성코드가 '편지에 묻은 독'이라면, USB_BAD는 '편지를 배달하는 집배원이 사실은 강도'인 셈이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

실무적으로 BadUSB를 막는 것은 매우 까다롭지만, 다음과 같은 전략적 판단이 필요하다.

### 방어 전략 체크리스트
1. **HID Filter 드라이버**: 새로운 HID 장치가 연결될 때 사용자 승인을 거치게 하거나, 비정상적인 입력 속도를 차단한다.
2. **물리적 포트 차단**: 보안 구역에서는 USB 포트를 물리적으로 봉인하거나 BIOS에서 비활성화한다.
3. **매체 제어 솔루션 (DLP)**: 단순히 저장 장치 차단을 넘어, USB 클래스(ID) 기반의 화이트리스트 관리를 수행한다.
4. **엔드포인트 행위 분석 (EDR)**: PowerShell이나 CMD가 비정상적인 컨텍스트에서 실행되는지 감시한다.

### 기술사 판단
"USB_BAD는 '입력 장치에 대한 무조건적 신뢰'라는 설계적 결함을 파고든다. 아키텍트는 하드웨어 가용성보다 '신원 확인'을 우선시하는 Zero Trust 원칙을 주변 장치 연결에도 적용해야 하며, 특히 고권한 계정은 키보드 입력만으로 시스템을 변경할 수 없도록 MFA를 반드시 결합해야 한다."

- **📢 섹션 요약 비유**: 아무리 집 열쇠를 가진 사람이라도, 얼굴 확인(추가 인증) 없이는 안방 금고를 열 수 없게 만드는 2중 장치가 필요하다.

---

## Ⅴ. 기대효과 및 결론

USB_BAD 방어 체계를 갖추면, 길거리에서 주운 USB나 선물로 받은 악성 하드웨어를 통한 침해 사고를 효과적으로 막을 수 있다. 이는 특히 사회 공학(Social Engineering) 공격에 취약한 일반 임직원들을 보호하는 강력한 방패가 된다.

결론적으로, 하드웨어는 더 이상 '단순한 도구'가 아니다. 모든 연결된 장치는 스스로의 정체를 입증해야 하며, 시스템은 그 정체가 위조될 수 있음을 항상 경계해야 한다.

- **📢 섹션 요약 비유**: 모르는 사람이 준 사탕(USB)을 함부로 먹지(연결) 않는 교육과, 사탕에 독이 있는지 검사하는 시스템이 조화를 이루어야 한다.

2. 나쁜 사람이 USB 메모리처럼 생겼지만 실제로는 키보드처럼 말하는 장치를 만들어서 컴퓨터를 속이는 게 USB_BAD 공격이에요.
3. 그래서 우리는 모르는 USB를 함부로 꽂지 말고, 컴퓨터가 "진짜 키보드가 맞니?"라고 물어보게 해야 해요!
