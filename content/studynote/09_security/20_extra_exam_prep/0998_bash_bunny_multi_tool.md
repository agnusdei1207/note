## 핵심 인사이트 (3줄 요약)
1. **본질**: 고성능 임베디드 리눅스(Debian) 시스템을 USB 외형에 담아, HID 키보드, 네트워크 어댑터, 저장 장치 등 다양한 장치로 동시에 위장하여 복합적인 공격을 수행하는 도구다.
2. **가치**: 단순한 키보드 입력을 넘어, 가짜 네트워크 게이트웨이를 만들어 트래픽을 가로채거나(MITM), 메모리 덤프를 네트워크로 즉시 전송하는 등 '포켓 사이즈의 해킹 워크스테이션' 역할을 한다.
3. **판단 포인트**: 다단계 페이로드와 복합 인터페이스를 지원하므로, 물리 포트 보안 정책 수립 시 가장 강력한 위협 모델로 상정하고 대응 시나리오를 짜야 한다.

---

## Ⅰ. 개요 및 필요성

Bash Bunny(배쉬 버니)는 Rubber Ducky를 개발한 Hak5의 또 다른 강력한 하드웨어 침투 도구다. Rubber Ducky가 '빠른 키보드'라면, Bash Bunny는 '똑똑한 컴퓨터'다. 쿼드코어 CPU와 RAM을 갖춘 독립된 리눅스 머신이 USB 포트에 꽂히는 순간, 타겟 시스템은 이 장치를 단순한 주변 장치가 아닌 '신뢰할 수 있는 네트워크 어댑터'나 '고성능 스토리지'로 인식하게 된다.

이 도구는 특히 네트워크 보안을 우회하는 데 탁월하다. 예를 들어, 인터넷이 차단된 PC에 꽂아 Bash Bunny를 무선 공유기나 VPN 게이트웨이로 인식시킨 뒤 내부 데이터를 외부로 빼돌릴 수 있다. 물리적 접근이 가능한 상황에서 Bash Bunny는 공격자에게 무한한 가능성을 제공하는 '해킹의 스위스 아미 나이프'다.

- **📢 섹션 요약 비유**: Bash Bunny는 단순히 열쇠(Rubber Ducky)를 가진 사람이 아니라, 아예 가방 속에 온갖 도구를 챙겨온 전문 털이범(임베디드 리눅스)과 같다.

## Ⅱ. 아키텍처 및 핵심 원리

Bash Bunny의 강력함은 여러 가상 장치를 동시에 띄울 수 있는 '멀티 페이로드' 기능에서 나온다.

| 구성 요소 | 기술적 특징 | 역할 |
|:---|:---|:---|
| **임베디드 리눅스** | Debian 기반 OS | 복잡한 로직 및 도구(Nmap, Responder 등) 실행 |
| **페이로드 스위치** | 3단 물리 스위치 | 미리 설정된 공격 모드(Switch 1, 2, Setup) 선택 |
| **가상 인터페이스** | HID / CDC-ECM / MSC | 키보드, 이더넷, 저장소로 동시 위장 |
| **RGB LED** | 상태 인디케이터 | 공격 진행 상황을 시각적으로 알림 |

```text
[ Bash Bunny Complex Attack ]
1. Plugin -> 2. Emulate HID Keyboard -> Opens Terminal
3. Emulate Ethernet Adapter (CDC-ECM) -> Target OS sets it as Default Gateway
4. Bash Bunny runs 'Responder' inside -> Sniffs LLMNR/NBT-NS hashes
5. Bash Bunny Emulates Mass Storage -> Saves captured hashes to internal storage
6. Attack Finished (LED turns Green)
```

이처럼 Bash Bunny는 하드웨어 하나로 **'명령 입력 -> 환경 조작 -> 정보 탈취 -> 저장'**의 전 과정을 자동화한다.

- **📢 섹션 요약 비유**: Bash Bunny는 컴퓨터 포트에 꽂히는 순간, 그 포트를 통해 컴퓨터와 악수를 하는 게 아니라 컴퓨터의 뇌(OS)를 속여서 자기 말을 듣는 노예로 만드는 로봇과 같다.

## Ⅲ. 비교 및 연결

Bash Bunny는 다른 침투 도구들과 확실히 차별화되는 고기능성을 제공한다.

| 비교 항목 | Rubber Ducky | Bash Bunny | LAN Turtle |
|:---|:---|:---|:---|
| **핵심 기능** | 빠른 키보드 입력 | **복합 장치 에뮬레이션** | 네트워크 영구 백도어 |
| **운영체제** | 없음 (펌웨어) | **Full Linux (Debian)** | OpenWRT 기반 |
| **공격 방식** | HID 위주 | HID + Network + Storage | Network 전용 |
| **사용자 숙련도** | 낮음 (간단한 스크립트) | **높음 (리눅스 지식 필요)** | 보통 |
| **주요 용도** | 단순 인증 우회 | **정밀 침투, 데이터 탈취** | 원격 유지 관리, 도청 |

Bash Bunny는 **MITM (Man-In-The-Middle)** 공격과 결합될 때 무서운 위력을 발휘한다. 타겟 컴퓨터의 모든 네트워크 트래픽을 Bash Bunny를 통과하게 만들어 암호화되지 않은 모든 정보를 엿볼 수 있다.

- **📢 섹션 요약 비유**: Rubber Ducky가 '한 가지 마법만 부리는 요정'이라면, Bash Bunny는 '마법책 전체를 들고 다니는 마법사'다.

## Ⅳ. 실무 적용 및 기술사 판단

Bash Bunny와 같은 고성능 도구에 대응하기 위해서는 단순한 포트 차단 이상의 전략이 필요하다.

### 방어 전략 및 판단
1. **네트워크 어댑터 화이트리스트**: 승인되지 않은 새로운 네트워크 인터페이스(특히 CDC-ECM 클래스)가 연결될 때 즉시 차단하거나 알림을 보낸다.
2. **802.1X 인증**: 포트 레벨에서 단말 인증을 강제하여, Bash Bunny가 가짜 네트워크 장치로 동작하더라도 네트워크 접근을 거부한다.
3. **LLMNR/NBT-NS 비활성화**: Bash Bunny가 주로 악용하는 넷바이오스 이름 해석 기능을 꺼서 해시 탈취 공격을 예방한다.
4. **EDR 기반 컨텍스트 분석**: USB 연결 직후 발생하는 네트워크 설정 변경이나 비정상적인 프로세스 생성을 실시간 감시한다.

### 기술사 판단
"Bash Bunny는 물리 포트가 곧 네트워크 침투의 관문이 될 수 있음을 시사한다. 아키텍트는 물리 보안(Physical Lock)을 IT 보안의 시작점으로 삼아야 하며, 특히 서버실이나 중요 단말기에 대해서는 'USB 연결 시 인가된 관리자의 2차 승인'과 같은 강력한 거버넌스를 결합해야 한다."

- **📢 섹션 요약 비유**: 현관문을 잘 잠그는 것도 중요하지만, 창문(USB 포트)으로 들어온 사람이 집안의 전화선(네트워크)을 가로채지 못하게 감시 카메라를 달아야 한다.

## Ⅴ. 기대효과 및 결론

Bash Bunny 대응 체계를 구축하면 물리적 보안 사고로 인한 대규모 정보 유출 리스크를 획기적으로 낮출 수 있다. 이는 단순한 침입 탐지를 넘어, 공격자가 시스템 내부에 발을 붙이지 못하게 하는 '철벽 방어'의 시작이 된다.

결론적으로 Bash Bunny는 하드웨어가 소프트웨어 보안을 어떻게 압도할 수 있는지 보여주는 강력한 사례다. 하드웨어 기반의 보안 신뢰(Hardware-based Trust)를 재구축하는 것이 미래 보안의 핵심 과제가 될 것이다.

- **📢 섹션 요약 비유**: Bash Bunny는 보안 전문가에게는 '가장 까다로운 연습 상대'이며, 기업에게는 '반드시 대비해야 할 실전 위협'이다.
