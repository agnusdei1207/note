## 핵심 인사이트 (3줄 요약)
1. **본질**: 고속 데이터 전송 표준인 Firewire(IEEE 1394) 프로토콜의 하드웨어 사양에 포함된 DMA 기능을 악용하여, 연결된 시스템의 메모리를 직접 읽고 쓰는 공격이다.
2. **가치**: 운영체제의 보안 레이어와 관계없이 하드웨어 수준에서 메모리 전 영역에 접근할 수 있음을 실증하여, 외부 포트 보안의 중요성을 일깨운 고전적이고 강력한 공격이다.
3. **판단 포인트**: 현대의 Thunderbolt 공격의 모태가 된 기술로, 구형 장비 관리 시 해당 포트의 논리적 비활성화나 물리적 차단이 필수적인 보안 항목이다.

---

## Ⅰ. 개요 및 필요성

Firewire(IEEE 1394)는 과거 애플(Apple)이 주도했던 고속 직렬 버스 규격이다. 캠코더, 외장 하드디스크 등 대용량 데이터 전송이 필요한 기기에 널리 쓰였다. 이 규격의 가장 큰 특징은 전송 효율을 극대화하기 위해 '주변 장치가 호스트의 CPU 도움 없이 직접 메모리에 읽고 쓸 수 있는 권한(OHCI)'을 표준 자체에 포함했다는 점이다.

이 설계적 특징은 공격자에게 최고의 선물이 되었다. Firewire 포트가 있는 노트북에 다른 컴퓨터를 연결하고 간단한 공격 소프트웨어(예: WinLockPick)를 실행하면, 대상 컴퓨터의 로그인 비밀번호를 무력화하거나 메모리 전체를 복제할 수 있다. 이는 암호가 걸린 컴퓨터라도 물리적 포트 하나만 열려 있으면 순식간에 뚫릴 수 있다는 사실을 대중에게 각인시킨 사건이었다.

- **📢 섹션 요약 비유**: Firewire 공격은 집주인(CPU)에게 허락을 받는 대신, 가스 점검원용 비상 열쇠(Firewire 표준)를 이용해 거실(메모리)로 바로 들어가는 것과 같다.

## Ⅱ. 아키텍처 및 핵심 원리

Firewire 프로토콜의 SBP-2(Serial Bus Protocol 2)는 주변 장치가 메모리 주소 공간을 직접 지정하여 접근하는 것을 허용한다.

| 단계 | 공격 메커니즘 | 기술적 상세 |
|:---|:---|:---|
| 1. 물리 연결 | 공격자 노트북과 타겟 노트북 연결 | IEEE 1394 케이블 사용 |
| 2. 장치 에뮬레이션 | 공격자 기기를 저장 장치로 위장 | 타겟 시스템이 버스 권한 부여 |
| 3. DMA 요청 | 특정 메모리 주소 범위 읽기/쓰기 | CPU 개입 없이 하드웨어 레벨에서 처리 |
| 4. 페이로드 주입 | 로그인 체크 로직 메모리 패치 | `if (password == correct)`를 `if (true)`로 수정 |
| 5. 인증 우회 | 아무 암호나 입력하여 진입 | 수정된 메모리 로직에 의해 즉시 로그인 성공 |

```text
[ Firewire DMA Attack Flow ]
[ Attacker PC ] <---(IEEE 1394)---> [ Target PC (Locked) ]
       |                                     |
       |------- Direct Memory Access ------->|
       | (Read Memory Scan for Key/Pass)     |
       |                                     |
       |<------ Memory Data Returned --------|
       |                                     |
       |------- Write Patch to Kernel ------>|
       | (Bypass Auth Logic)                 |
```

공격자는 메모리 상에서 암호를 검증하는 특정 어셈블리 명령어를 찾아내어, 이를 무조건 '성공'으로 반환하도록 1~2바이트만 수정(Patch)함으로써 시스템을 장악한다.

- **📢 섹션 요약 비유**: 금고 번호를 맞추는 대신, 금고 내부의 잠금 장치를 막대기로 밀어올려 번호와 상관없이 문이 열리게 만드는 것이다.

## Ⅲ. 비교 및 연결

Firewire 공격은 이후 등장한 다양한 인터페이스 공격의 시초가 되었다.

| 항목 | Firewire (IEEE 1394) | USB (Standard) | Thunderbolt |
|:---|:---|:---|:---|
| **DMA 지원** | 하드웨어 기본 지원 | 기본 지원 안 함 (별도 칩 필요) | 하드웨어 기본 지원 |
| **공격 난이도** | 매우 낮음 (케이블만 필요) | 보통 (악성 칩/펌웨어 필요) | 낮음 (IOMMU 없을 시) |
| **주요 방어** | 포트 드라이버 제거 | 소프트웨어 통제 | Kernel DMA Protection |
| **현재 위상** | 구형 장비 취약점 | HID 에뮬레이션 위주 | 현재 가장 위험한 경로 |

Firewire 공격은 **OS 커널 무결성**의 중요성을 부각시켰다. 런타임에 메모리의 실행 코드가 외부 장치에 의해 변조될 수 있다는 사실은 이후 Secure Boot와 코드 서명 기술 발전에 큰 영향을 주었다.

- **📢 섹션 요약 비유**: Firewire가 '관리가 안 된 뒷문'이었다면, Thunderbolt는 '감시 카메라가 달린 고속 터널'이다.

## Ⅳ. 실무 적용 및 기술사 판단

현재 Firewire는 사양길에 접어들었으나, 오래된 공장 제어 PC, 의료 기기, 구형 워크스테이션에는 여전히 존재한다.

### 방어 전략
1. **드라이버 비활성화**: 사용하지 않는 경우 OS 수준에서 `1394ohci.sys` 등의 드라이버를 삭제하거나 사용 중지한다.
2. **포트 물리적 차단**: 보안이 중요한 장비는 물리적 포트 락(Port Lock)을 사용하여 케이블 연결을 원천 차단한다.
3. **OS 업데이트**: 최신 OS는 잠금 화면 상태에서 SBP-2 장치의 연결을 기본적으로 차단하는 로직을 포함하고 있다.
4. **IOMMU 활용**: 지원되는 경우 BIOS에서 가상화 기반 입출력 격리를 활성화한다.

### 기술사 판단
"Firewire 공격은 '하드웨어 표준의 편의성이 보안 정책을 압도한' 대표적 사례다. 기술사는 레거시 시스템을 실사할 때 미사용 물리 포트의 존재 여부를 반드시 점검해야 하며, 새로운 인터페이스 도입 시 반드시 하드웨어 레벨의 격리 기술(IOMMU 등)이 전제되어 있는지 확인해야 한다."

- **📢 섹션 요약 비유**: 쓰지 않는 오래된 문은 벽돌로 막아버리는 것이 가장 안전한 보안 정책이다.

## Ⅴ. 기대효과 및 결론

Firewire 포트 보안을 강화하면 물리적 접근을 통한 순식간의 인증 우회 사고를 예방할 수 있다. 이는 특히 기업의 자산 관리 실사 시 '잊혀진 구멍'을 메우는 효과를 준다.

결론적으로, Firewire 공격은 보안이 '논리적 암호'뿐만 아니라 '물리적 통로'에 대한 통제가 동반되어야 함을 가르쳐준 역사적 교훈이다. 이러한 철학은 오늘날 Thunderbolt와 USB-C 보안 설계의 근간이 되었다.

- **📢 섹션 요약 비유**: 아무리 튼튼한 방화문이라도, 옆에 작은 환기구(Firewire)가 뚫려 있다면 도둑은 그리로 들어온다.
