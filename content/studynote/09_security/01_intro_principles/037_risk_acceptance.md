> **핵심 인사이트**
> 1. 위험 수용(Risk Acceptance)은 4가지 위험 대응 전략(완화·회피·전가·수용) 중 하나로, 위험 처리 비용이 위험 발생 시 손실보다 크거나 비즈니스 필요에 의해 위험을 의도적으로 감내하는 결정이다.
> 2. 위험 수용은 무기력한 방치가 아니라 임원급 의사결정자(CISO, CRO, CEO)의 명시적 승인을 받고 모니터링 계획을 수립하는 거버넌스 행위다 — 문서화와 주기적 재검토가 필수다.
> 3. ALE(Annual Loss Expectancy = SLE × ARO)로 손실을 수치화하고 대응 비용과 비교해 수용 여부를 결정하는 정량적 위험 분석이 현대 보안 위험 관리의 표준이다.

---

## I. 4가지 위험 대응 전략

```
위험 대응 (Risk Response) 전략:

1. 위험 완화 (Risk Mitigation/Reduction):
   통제 수단 적용으로 위험 가능성·영향 감소
   예: 방화벽 설치, 암호화, 패치 관리

2. 위험 회피 (Risk Avoidance):
   위험 원인 자체를 제거 (비즈니스 중단)
   예: 특정 국가 사업 포기, 위험 기능 제거

3. 위험 전가 (Risk Transfer):
   손실을 제3자에게 이전
   예: 사이버 보험, 아웃소싱 계약 SLA

4. 위험 수용 (Risk Acceptance):
   위험을 알면서 의도적으로 감내
   조건: 처리 비용 > 예상 손실 또는 비즈니스 필수
```

> 📢 **섹션 요약 비유**: 우산 살까? 비를 피해 이사할까? 빗속에서 뛸까? 그냥 맞고 갈까? — 네 가지 선택이 각각 완화·회피·전가·수용이다.

---

## II. 위험 수용의 조건과 요건

```
위험 수용이 적절한 경우:
  1. ALE(연간 예상 손실) < 대응 통제 비용
     예: 손실 예상 100만원 / 방어 비용 500만원
     -> 수용이 경제적 합리

  2. 잔여 위험이 조직 위험 허용 한도 이내
     (Risk Appetite / Risk Tolerance)

  3. 기술적으로 현실적 완화 방법이 없음

  4. 비즈니스 민첩성이 더 중요한 경우
     스타트업의 빠른 출시 vs 보안 완전성

위험 수용의 필수 요건:
  + 임원 서명 (CISO, CRO, CEO 중 한 명)
  + 위험 상세 기술 (위협·취약점·영향·가능성)
  + 재검토 일정 (분기/반기)
  + 모니터링 계획
```

| 수용 조건              | 설명                            |
|---------------------|--------------------------------|
| 경제적 합리성          | 대응 비용 > 예상 손실           |
| 잔여 위험 허용         | Risk Appetite 이내              |
| 기술적 불가능          | 현실적 완화 방법 없음            |
| 명시적 승인           | 임원급 서명 문서화               |

> 📢 **섹션 요약 비유**: 자동차 보험을 들지 않는 것이 아니라, 이미 분석하고 "이 비용에서 보험이 필요 없다"고 서명하는 것 — 무시와 수용은 다르다.

## III. ALE 기반 정량적 위험 분석

```
위험 수용 결정을 위한 수치 계산:

SLE (Single Loss Expectancy, 단일 손실 예상):
  = Asset Value × Exposure Factor
  예: 서버 자산 가치 5,000만원 × 노출 계수 0.4
  = 2,000만원

ARO (Annualized Rate of Occurrence, 연간 발생률):
  예: 보안 침해 연 0.5회 예상

ALE (Annual Loss Expectancy, 연간 손실 예상):
  = SLE × ARO = 2,000만원 × 0.5 = 1,000만원

통제 비용: 연간 500만원
결정:
  ALE 1,000만원 > 통제 비용 500만원
  -> 완화(통제) 선택이 경제적
  
  만약 통제 비용이 1,500만원이면:
  -> 수용이 경제적 (손실 1,000만원 < 비용 1,500만원)
```

> 📢 **섹션 요약 비유**: 홍수 피해 예상액이 100만원인데 방수 공사 비용이 500만원이면 그냥 홍수 대비 저축을 하는 게 낫다 — 이것이 수용의 경제학.

## IV. 위험 수용서 (Risk Acceptance Form)

```
위험 수용서 필수 항목:

1. 위험 식별자 및 설명
   "CVE-2024-XXXX: 웹 서버 원격 코드 실행 취약점"

2. 위험 등급 (높음/중간/낮음 + 수치)
   가능성 3/5, 영향 4/5, 위험 점수 12/25

3. 수용 이유
   "패치 적용 시 비즈니스 크리티컬 레거시 시스템 장애 위험"

4. 보완 통제 (Compensating Controls)
   "WAF 규칙 강화, 이상 접근 모니터링 강화"

5. 재검토 일정
   "2024년 12월 재검토, 패치 테스트 환경 마련"

6. 승인자
   CISO OOO (서명 + 날짜)
```

> 📢 **섹션 요약 비유**: 위험 수용서는 의사의 진단서 — "이 위험을 알고 있고, 이 이유로 지금은 수술하지 않으며, 다음 달에 재검토한다"는 공식 기록.

## V. 실무 시나리오 — 레거시 시스템 취약점 수용

```
상황:
  20년된 COBOL 코어뱅킹 시스템
  취약점: TLS 1.0 미지원, 암호화 취약
  완화 방법: 시스템 교체 (비용 200억원, 기간 3년)
  ALE: 연간 예상 침해 손실 5억원

분석:
  3년간 ALE: 15억원
  교체 비용: 200억원
  -> 수용이 경제적 합리

수용 결정 내용:
  + TLS 1.2 미지원이나 내부 전용망으로 노출 제한
  + 외부 노출 최소화를 위한 추가 방화벽 구성
  + 연 2회 침투 테스트
  + 2027년 차세대 시스템 전환 프로젝트 시작 시 재검토
  + CTO + CISO 공동 서명
```

> 📢 **섹션 요약 비유**: 낡은 은행 건물에 방범 카메라를 더 달고 경비를 늘리는 것 — 즉시 재건축은 비용이 너무 크니까, 보완 통제를 강화하면서 교체를 준비.

## 👶 어린이를 위한 3줄 비유 설명

1. 위험 수용은 모든 문제를 해결하지 않고 "이 위험은 알고 있지만 지금은 그냥 두겠다"고 결정하는 거예요.
2. 방어하는 데 드는 돈이 피해 예상액보다 훨씬 크면 그냥 받아들이는 게 더 합리적이거든요.
3. 하지만 그냥 모르는 척하는 게 아니라, 임원이 서명하고 언제 다시 검토할지 계획을 세워야 해요!
