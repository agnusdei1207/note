## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 포이즈닝 (Data Poisoning)은 AI 모델의 학습 단계에서 훈련 데이터셋의 일부를 의도적으로 왜곡하거나 악성 데이터를 삽입하여 모델의 성능을 떨어뜨리거나 백도어를 설치하는 공격이다.
> 2. **가치**: 모델이 완성된 후에는 탐지하기가 매우 어려운 '근본적인 무결성 훼손'을 유발하며, 특정 조건에서만 발동하는 오작동을 심어 보안 검수를 우회할 수 있다.
> 3. **판단 포인트**: 신뢰할 수 없는 데이터 소스(크라우드 소싱, 웹 크롤링 등)를 사용할 때 가장 위험하며, 데이터 정제(Sanitization)와 견고한 통계적 학습 기법이 필수적인 방어책이다.

---

## Ⅰ. 개요 및 필요성

데이터 포이즈닝은 AI의 '음식(데이터)'에 독을 타는 공격이다. AI 모델은 자신이 학습한 데이터를 진리로 믿고 패턴을 형성한다. 공격자는 이 점을 악용하여, 학습 데이터에 미세하게 조작된 샘플을 섞어 넣음으로써 모델이 편향된 판단을 내리게 하거나 특정 트리거(Trigger)에 반응하도록 세뇌한다.

이것이 심각한 이유는 공급망 공격(Supply Chain Attack)의 성격을 띠기 때문이다. 최근 기업들은 방대한 데이터를 직접 수집하기보다 외부 데이터셋을 활용하는 경우가 많은데, 이 공급망 어딘가에서 데이터가 오염되면 그 데이터를 학습한 모든 모델이 잠재적인 위협을 안게 된다.

```text
[데이터 포이즈닝의 공격 흐름]
오염된 데이터 삽입 ──▶ AI 모델 학습 ──▶ 왜곡된 결정 경계 형성 ──▶ 특정 상황에서 오작동
      ▲                                       │
[공격자의 독 주입]                      [세뇌된 AI 모델]
```

- **📢 섹션 요약 비유**: 데이터 포이즈닝은 시험을 앞둔 학생의 교과서 내용을 몰래 고쳐놓는 것과 같다. 학생은 열심히 공부할수록 오답을 정답으로 믿게 된다.

---

## Ⅱ. 아키텍처 및 핵심 원리

데이터 포이즈닝은 공격의 목적에 따라 **가용성 공격**과 **무결성 공격**으로 나뉜다.

| 공격 유형 | 목표 | 상세 설명 |
|:---|:---|:---|
| **가용성 공격 (Availability)** | 전체 모델 성능 저하 | 모델의 전반적인 정확도를 떨어뜨려 시스템을 쓸모없게 만듦 |
| **무결성 공격 (Integrity)** | 특정 타겟 오분류 | 평소엔 정상이지만, 공격자가 의도한 데이터에만 오작동 (Backdoor) |
| **Label Flipping** | 레이블 변조 | '고양이' 사진의 레이블을 '강아지'로 바꿔서 학습 방해 |
| **Feature 조작** | 특징점 변형 | 데이터의 픽셀이나 수치를 미세하게 조정하여 결정 경계 왜곡 |

```text
┌──────────────────────────────────────────────────────────────┐
│           데이터 포이즈닝 공격 구조 (Flow Chart)             │
├──────────────────────────────────────────────────────────────┤
│  [Trustless Data Sources] ──▶ [Attacker's Poisoning]         │
│             │                         │                      │
│             ▼                         ▼                      │
│  [Clean Dataset] + [Poisoned Dataset] ──▶ [Training Process] │
│                                                  │           │
│  [Result: Poisoned Model] ◀──────────────────────┘           │
│          │                                                   │
│          └─▶ Trigger Input ──▶ [Exploit!]                    │
└──────────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: 데이터 포이즈닝은 스파이를 양성하는 과정과 같다. 평소엔 평범한 시민처럼 행동하게 가르치다가, 특정 암호(Trigger)를 들으면 파괴 활동을 하도록 교육하는 것과 같다.

## Ⅲ. 비교 및 연결

데이터 포이즈닝은 적대적 예제와 자주 비교되지만, 공격의 시점이 완전히 다르다.

| 항목 | 데이터 포이즈닝 (Poisoning) | 적대적 예제 (Adversarial Example) |
|:---|:---|:---|
| **공격 시점** | 학습 단계 (Training Time) | 추론 단계 (Inference Time) |
| **공격 대상** | 학습 데이터셋 (Dataset) | 모델 입력값 (Input) |
| **지속성** | 영구적 (모델 재학습 전까지) | 일시적 (해당 입력에만 반응) |
| **공격 조건** | 학습 데이터에 접근 권한 필요 | 모델 API 또는 입력에 접근 권한 필요 |

이 개념은 **신뢰할 수 있는 AI (Trustworthy AI)** 프레임워크와 연결된다. 데이터의 수집부터 정제, 학습까지의 전 과정이 투명하고 안전해야만 AI의 결과를 믿을 수 있다는 논리다.

- **📢 섹션 요약 비유**: 포이즈닝이 요리 재료 자체에 독을 넣는 것이라면, 적대적 예제는 완성된 요리에 독 가루를 살짝 뿌리는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사적 관점에서 포이즈닝 방어의 핵심은 **데이터 무결성 검증**과 **이상치 탐지**다. 데이터의 양이 너무 방대하여 전수 조사가 어렵다면, 통계적 기법을 동원해야 한다.

### 체크리스트
1. **Data Sanitization**: 학습 전, 다른 데이터들과 통계적으로 너무 동떨어진(Outlier) 샘플을 자동 제거하는가?
2. **Gold Standard Dataset**: 신뢰할 수 있는 깨끗한 데이터로 검증용 셋을 구축하여, 모델의 성능 변화를 상시 모니터링하는가?
3. **Robust Statistics**: 소수의 오염된 데이터가 전체 모델에 미치는 영향을 최소화하는 손실 함수(예: Huber Loss)를 사용하는가?

### 안티패턴
- 인터넷에서 긁어온(Crawling) 데이터를 검증 없이 그대로 학습 서버에 넣는 행위
- 모델의 성능이 갑자기 떨어졌는데, 이를 단순한 '학습 실패'로만 치부하고 원인 분석을 하지 않는 경우

- **📢 섹션 요약 비유**: 포이즈닝 방어 실무는 식재료 검수원과 같다. 신선하지 않거나 수상한 재료가 주방으로 들어오지 못하게 입구에서 꼼꼼히 체크하는 것이 가장 중요하다.

---

## Ⅴ. 기대효과 및 결론

데이터 포이즈닝에 대한 선제적 방어는 **AI 공급망 보안**의 핵심이다. 데이터가 자산인 시대에, 오염된 데이터는 기업의 가장 큰 무형 자산인 'AI 모델'을 한순간에 폐기물로 만들 수 있다.

앞으로는 데이터의 생성 이력을 추적하는 **데이터 리니지(Data Lineage)** 기술과 블록체인을 결합하여 데이터 포이즈닝을 원천 봉쇄하려는 시도가 늘어날 것이다. 결론적으로 데이터 포이즈닝은 "쓰레기를 넣으면 쓰레기가 나온다(GIGO)"는 진리를 보안 관점에서 재확인시켜 주는 가장 고전적이면서도 치명적인 공격이다.

- **📢 섹션 요약 비유**: 깨끗한 물에서만 건강한 물고기가 자랄 수 있듯이, 깨끗한 데이터 환경이 보장될 때만 우리 사회를 이롭게 할 AI가 태어날 수 있다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| 백도어 공격 (Backdoor) | 데이터 포이즈닝을 통해 모델에 숨겨진 오작동 기능을 심는 행위 |
| 트리거 (Trigger) | 포이즈닝된 모델의 악성 행위를 발동시키는 특정 입력 신호 |
| 데이터 리니지 (Data Lineage) | 데이터의 출처와 변경 이력을 추적하여 포이즈닝을 방지하는 관리 체계 |
| 가명화/익명화 | 데이터 포이즈닝 시 민감 정보가 유출되지 않도록 보호하는 기술적 수단 |

### 👶 어린이를 위한 3줄 비유 설명

1. 데이터 포이즈닝은 AI가 공부하는 문제집에 나쁜 사람이 몰래 가짜 정답을 적어놓는 거예요.
2. AI가 가짜 정답을 진짜로 믿고 공부하게 되면, 나중에 큰 실수를 하게 될 수 있어요.
3. 그래서 AI에게 책을 주기 전에 어른들이 가짜 문제가 섞여 있는지 꼼꼼히 검사해야 한답니다.
