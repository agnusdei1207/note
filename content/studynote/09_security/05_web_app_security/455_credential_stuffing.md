## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 크리덴셜 스터핑 (Credential Stuffing)은 다른 서비스에서 유출된 아이디·비밀번호 조합을 자동화 도구로 대량 시도해 계정을 탈취하는 공격이다.
> 2. **가치**: 피해자가 여러 사이트에 동일한 비밀번호를 재사용하는 한, 공격자는 별도의 취약점 없이도 합법적 인증 절차만으로 계정을 점령한다.
> 3. **판단 포인트**: 방어의 핵심은 MFA (Multi-Factor Authentication) 강제 적용과 비정상 로그인 패턴 탐지이며, 비밀번호 재사용을 차단하는 사용자 교육이 병행되어야 한다.

---

## Ⅰ. 개요 및 필요성

크리덴셜 스터핑 (Credential Stuffing)은 브루트 포스 (Brute Force) 공격과 달리 무작위 패스워드를 추측하지 않는다. 대신 다크웹에서 구매하거나 공개적으로 유출된 **콤보리스트 (Combo List)** — 아이디와 비밀번호가 쌍으로 묶인 목록 — 을 이용해, 다른 서비스의 로그인 엔드포인트에 대입하는 방식이다.

전 세계 사용자의 약 65%가 동일한 비밀번호를 여러 서비스에서 재사용한다는 통계가 존재한다. 이 때문에 A서비스에서 유출된 크리덴셜이 B·C·D서비스에도 그대로 작동하며, 공격자는 수억 건의 계정 정보를 저렴한 비용으로 시도할 수 있다. 대규모 크리덴셜 스터핑 캠페인에서는 하루 1억 건 이상의 로그인 요청이 관찰된 사례도 있다.

```text
┌─────────────────────────────────────────────────────────────┐
│             크리덴셜 스터핑 공격 흐름                        │
├─────────────────────────────────────────────────────────────┤
│  1. 다크웹에서 콤보리스트 구매 (유출된 ID + PW 수억 건)       │
│  2. 자동화 봇 (OpenBullet 등)으로 대상 사이트 공격            │
│     ┌────────────┐  POST /login  ┌────────────────┐         │
│     │ 공격자 봇  │ ────────────▶ │ 대상 서버       │         │
│     │ (콤보리스트│               │ (은행, 쇼핑몰 등)│         │
│     │  반복 대입)│ ◀──────────── │ 200 OK / 401    │         │
│     └────────────┘               └────────────────┘         │
│  3. 성공 건 추출 → 계정 탈취 → 금전 피해 또는 재판매          │
└─────────────────────────────────────────────────────────────┘
```

공격 도구는 OpenBullet, SentryMBA 같은 오픈소스 툴이 대표적이며, 프록시 회전 (Proxy Rotation)으로 IP 차단을 우회한다. 공격이 성공하면 계정 내 포인트 탈취, 개인정보 수집, 결제 수단 악용, 계정 판매 등 다양한 2차 피해로 이어진다.

📢 **섹션 요약 비유**: 크리덴셜 스터핑은 마치 마스터키 세트를 들고 아파트 단지를 돌아다니는 것이다. 열쇠 자체는 진짜지만, 자물쇠 주인이 여러 집에 같은 자물쇠를 쓴 게 문제다.

## Ⅱ. 아키텍처 및 핵심 원리

### 공격 인프라 구성

| 구성 요소 | 역할 | 설명 |
|:---|:---|:---|
| 콤보리스트 (Combo List) | 공격 원재료 | 유출된 ID·PW 쌍 |
| 자동화 봇 | 대입 실행 | OpenBullet, SentryMBA |
| 프록시 풀 (Proxy Pool) | IP 우회 | 수천 개 IP 순환 사용 |
| CAPTCHA 우회 서비스 | 탐지 회피 | 사람 풀 또는 ML 솔버 |
| 결과 분석기 | 성공 건 추출 | HTTP 200/302 패턴 필터 |

```text
┌──────────────────────────────────────────────────────────────┐
│               방어 계층 구조                                   │
├──────────────────────────────────────────────────────────────┤
│  레이어 1: 유출 모니터링 (HaveIBeenPwned API 연동)            │
│  레이어 2: 로그인 시도 속도 제한 (Rate Limiting)              │
│  레이어 3: 장치 지문 (Device Fingerprint) 이상 탐지           │
│  레이어 4: MFA (Multi-Factor Authentication) 강제             │
│  레이어 5: 행동 분석 (Behavioral Analytics) 봇 탐지           │
└──────────────────────────────────────────────────────────────┘
```

크리덴셜 스터핑과 브루트 포스는 다르다. 브루트 포스는 하나의 계정에 수많은 패스워드를 시도하므로 계정 잠금 정책으로 방어 가능하다. 반면 크리덴셜 스터핑은 각 계정에 한두 번만 시도하므로 임계값 기반 잠금이 효과적이지 않다.

📢 **섹션 요약 비유**: 방어 계층은 아파트 보안 시스템과 같다. 로비 카메라, 엘리베이터 카드키, 현관 도어락, 방범 창문까지 여러 겹을 두어야 침입자를 막을 수 있다.

## Ⅲ. 비교 및 연결

| 구분 | 크리덴셜 스터핑 | 브루트 포스 | 패스워드 스프레이 |
|:---|:---|:---|:---|
| 시도 방식 | 유출 크리덴셜 대입 | 무작위 패스워드 반복 | 공통 패스워드 다수 계정 |
| 계정당 시도 수 | 1~2회 | 수천~수백만 회 | 1~5회 |
| 탐지 난이도 | 높음 | 낮음 (속도 이상) | 중간 |
| 방어 핵심 | 유출 모니터링 + MFA | 계정 잠금 | 속도 제한 |

HaveIBeenPwned (HIBP)와 같은 서비스는 기업이 자사 사용자의 이메일이 유출된 적 있는지 API로 확인할 수 있게 해준다. 또한 NIST (National Institute of Standards and Technology) SP 800-63B에서는 패스워드 변경 시 알려진 유출 패스워드 목록과 비교하도록 권고한다.

📢 **섹션 요약 비유**: 세 공격은 마치 복사 열쇠(크리덴셜 스터핑), 픽킹 도구(브루트 포스), 공용 마스터키 시도(패스워드 스프레이)처럼 다른 전략이지만 목적은 같다.

## Ⅳ. 실무 적용 및 기술사 판단

**시나리오**: 대형 전자상거래 플랫폼에서 갑자기 새벽 2~4시 사이 로그인 실패율이 평소 대비 30배 상승했다. 특정 IP는 아니지만 수천 개 분산 IP에서 초당 수십 건의 로그인 시도가 감지됐다.

**판단 포인트**:
1. **즉시 대응**: SIEM (Security Information and Event Management) 경보 발동 → 해당 시간대 성공 로그인 계정 목록 추출 → 강제 비밀번호 재설정
2. **중기 대응**: 모든 계정에 MFA 강제 적용, Device Fingerprint 도입
3. **장기 대응**: HIBP API 연동으로 회원가입/로그인 시 유출 여부 실시간 확인, 행동 기반 봇 탐지 (Behavioral Bot Detection) 시스템 도입

**비용 계산**: 계정 탈취 한 건당 평균 피해액 vs. MFA 구현 비용을 비교하면, MFA 도입이 항상 ROI (Return on Investment) 양의 값을 보인다.

📢 **섹션 요약 비유**: 실무 대응은 화재 진압이 아니라 방화 설계다. 불이 난 뒤 끄는 것보다 스프링클러를 미리 설치하는 것이 훨씬 경제적이다.

---

## Ⅴ. 기대효과 및 결론

크리덴셜 스터핑 방어 체계를 갖추면 계정 탈취로 인한 금전 피해, 개인정보 침해, 브랜드 손상을 크게 줄일 수 있다. 핵심은 단일 방어선이 아닌 **다층 방어 (Defense in Depth)** 전략이다.

OWASP (Open Web Application Security Project) Top 10 2021에서 A07 인증 실패 (Identification and Authentication Failures) 항목에 크리덴셜 스터핑이 포함되며, 이는 인증 설계의 취약성을 체계적으로 다루어야 함을 시사한다. 패스키 (Passkey) 및 FIDO2 (Fast IDentity Online 2) 기반 패스워드리스 인증으로 전환하면 크리덴셜 유출 자체의 의미가 없어지므로, 장기적으로는 이 방향이 근본 해결책이다.

📢 **섹션 요약 비유**: 크리덴셜 스터핑 대응의 완성은 자물쇠를 강화하는 것이 아니라, 자물쇠 자체가 필요 없는 생체인식 출입 시스템으로 바꾸는 것이다.
