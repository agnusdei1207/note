## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 브루트 포스 (Brute Force, 무차별 대입 공격)는 가능한 모든 비밀번호·PIN·암호화 키 조합을 순차적으로 시도해 정답을 찾아내는 공격이며, 클라우드 기반 GPU 크래킹으로 짧고 단순한 비밀번호는 수초 내에 해독된다.
> 2. **가치**: MD5·SHA-1로 저장된 비밀번호는 레인보우 테이블 + GPU 가속으로 8자 이하는 수초~수분 내 해독 가능하며, 이것이 bcrypt·Argon2 같은 느린 해시 함수가 표준이 된 이유다.
> 3. **판단 포인트**: 계정 잠금(Account Lockout), Rate Limiting, CAPTCHA, MFA (Multi-Factor Authentication), 느린 해시 함수가 브루트 포스 방어의 5대 핵심 통제다.

---

## Ⅰ. 개요 및 필요성

브루트 포스는 가장 단순한 공격이지만, 현대 하드웨어(GPU, 클라우드)와 자동화 도구의 발전으로 여전히 매우 효과적이다. 공격 유형에 따라 대상과 방법이 달라진다.

**브루트 포스 유형**:
1. **온라인 브루트 포스**: 로그인 폼에 직접 시도 (느리지만 해시 불필요)
2. **오프라인 브루트 포스**: 유출된 해시 파일을 로컬에서 크래킹 (매우 빠름)
3. **비밀번호 스프레잉**: 흔한 비밀번호를 여러 계정에 분산 시도 (잠금 우회)
4. **사전 공격 (Dictionary Attack)**: 흔한 단어 목록으로 시도
5. **레인보우 테이블 공격**: 사전 계산된 해시 역산 테이블 사용

**해시별 초당 시도 횟수 (RTX 4090 기준)**:
- MD5: 164 GH/s (1640억 회/초)
- SHA-1: 72 GH/s (720억 회/초)
- bcrypt(cost=10): 49,000 H/s (4만 9천 회/초)
- Argon2id: 더 느림

MD5와 bcrypt의 속도 차이는 약 330만 배다.

📢 **섹션 요약 비유**: MD5로 비밀번호를 저장하는 것은 도어락 비밀번호를 포스트잇에 써서 문에 붙이는 것이다. bcrypt는 비밀번호를 탱크 금고에 넣는 것이다.

## Ⅱ. 아키텍처 및 핵심 원리

브루트 포스 방어 다계층 구조:

| 계층 | 방어 기술 | 효과 |
|:---|:---|:---|
| 서버 측 | Rate Limiting, 계정 잠금 | 온라인 공격 차단 |
| 프론트엔드 | CAPTCHA, Progressive Delay | 자동화 차단 |
| 저장 | bcrypt/Argon2 (느린 해시) | 오프라인 크래킹 저지 |
| 인증 | MFA | 비밀번호 해독해도 접근 불가 |
| 모니터링 | 비정상 로그인 패턴 탐지 | 공격 감지 |

```
┌──────────────────────────────────────────────────────────┐
│           브루트 포스 vs bcrypt 방어 효과                │
├──────────────────────────────────────────────────────────┤
│  MD5("password123") 크래킹 시간:                         │
│  → GPU 4090으로 약 0.001초 (즉시)                       │
│                                                          │
│  bcrypt(cost=12, "password123") 크래킹 시간:             │
│  → GPU 4090으로 약 2분 (동일 비밀번호!)                  │
│                                                          │
│  bcrypt + 12자 무작위 비밀번호: 수십억 년 이상            │
└──────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: bcrypt의 느린 해시는 자물쇠를 열어보려는 사람에게 시도마다 1분씩 기다리게 만드는 것이다. 같은 시간에 시도할 수 있는 횟수가 330만분의 1로 줄어든다.

## Ⅲ. 비교 및 연결

| 구분 | 온라인 브루트 포스 | 오프라인 브루트 포스 |
|:---|:---|:---|
| 조건 | 로그인 API 직접 접근 | 해시 파일 탈취 필요 |
| 속도 | 매우 느림 (네트워크 제한) | 매우 빠름 (로컬 GPU) |
| 방어 | Rate Limiting, 계정 잠금 | 느린 해시 함수, 솔트 |
| 위협 수준 | 낮음 (탐지 쉬움) | 높음 (DB 탈취 후 발생) |

📢 **섹션 요약 비유**: 온라인은 자물쇠 직접 시도(경비원이 봄), 오프라인은 자물쇠를 가져가서 집에서 복사(아무도 모름). 후자가 훨씬 위험하다.

## Ⅳ. 실무 적용 및 기술사 판단

**계정 잠금 vs 비밀번호 스프레잉 딜레마**:
- 계정 잠금(5회 실패 시 30분)은 브루트 포스를 방어하지만, 비밀번호 스프레잉은 계정당 1~2회만 시도하므로 잠금 없이 진행 가능
- **해결책**: 계정별 잠금 + IP별 Rate Limiting + MFA 조합

**비밀번호 정책 (NIST SP 800-63B 기준)**:
1. **최소 8자** (사용자 생성), 최소 6자 (시스템 생성)
2. **최대 64자** 이상 허용 (길이 제한 완화)
3. **유출 비밀번호 차단**: HaveIBeenPwned API 연동
4. **정기 변경 강제 비추천**: 패턴 있는 약한 비밀번호로 이어짐

📢 **섹션 요약 비유**: "비밀번호 90일마다 변경 강제"는 오히려 "Summer2024! → Fall2024!"처럼 패턴을 만들어 오히려 취약하게 한다.

## Ⅴ. 기대효과 및 결론

느린 해시 함수(bcrypt/Argon2) + MFA + Rate Limiting의 조합은 브루트 포스 공격의 현실적 성공 가능성을 근본적으로 줄인다. 특히 Passkey/FIDO2 기반 패스워드리스 인증으로 전환하면 비밀번호 자체가 없어져 모든 비밀번호 기반 공격이 무의미해진다.

기술사 관점에서 브루트 포스 방어는 암호학적 강도(느린 해시)와 운영 통제(Rate Limiting, MFA)가 함께 작동하는 심층 방어의 대표 사례다. 둘 중 하나만으로는 충분하지 않다.

📢 **섹션 요약 비유**: 브루트 포스 방어는 두꺼운 금고문(bcrypt)과 침입 감지 시스템(Rate Limiting), 그리고 지문 인식(MFA)을 모두 갖추는 것이다.

| 개념 | 관계 | 설명 |
|:---|:---|:---|
| bcrypt/Argon2 | 핵심 방어 | 의도적으로 느린 해시 |
| Rate Limiting | 온라인 방어 | 시도 횟수 제한 |
| MFA | 계층 방어 | 비밀번호 해독해도 접근 불가 |
| CAPTCHA | 자동화 차단 | 봇과 사람 구분 |

### 👶 어린이를 위한 3줄 비유 설명
- 브루트 포스는 금고 비밀번호를 0000부터 9999까지 하나씩 다 눌러보는 것이야.
- bcrypt는 각 번호를 누를 때마다 5초씩 기다리게 해서, 만 번을 다 시도하려면 13시간이 넘게 걸리게 만들어.
- 여기에 MFA를 추가하면 비밀번호를 맞춰도 핸드폰 인증까지 통과해야 하니, 거의 완벽하게 막을 수 있어!
