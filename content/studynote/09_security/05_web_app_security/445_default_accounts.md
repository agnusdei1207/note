## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 기본 계정(Default Accounts) 및 비밀번호 취약점은 제조사나 프레임워크가 초기 설정 편의를 위해 미리 심어둔 `admin/admin`, `root/password` 같은 자격증명을 변경하지 않고 운영 환경에 노출하는 보안 설정 오류(A05)의 대표 사례다.
> 2. **가치**: 미라이(Mirai) 봇넷 사태처럼 수십만 대의 IoT 기기가 단 60여 개의 기본 계정 목록만으로 좀비 PC가 되어 사상 최대 규모의 DDoS 공격을 일으켰을 만큼, 침투 난이도는 낮지만 파급력은 파괴적이다.
> 3. **판단 포인트**: '관리의 편의성'보다 'Hardening(시스템 강화)'을 우선하여, 설치 즉시 강제 비밀번호 변경 정책을 적용하거나 관리자 페이지 자체를 외부 IP로부터 격리하는 것이 실무적 정답이다.

---

## Ⅰ. 개요 및 필요성

소프트웨어나 하드웨어를 처음 설치하면 사용자가 설정을 시작할 수 있도록 공장 출하 시 설정된 기본 계정이 존재한다. 하지만 많은 운영자가 이를 방치하며, 공격자는 구글 해킹(Google Dorking)이나 Shodan 검색을 통해 노출된 기본 계정 페이지를 빛의 속도로 찾아낸다.

**위험성**:
- **Brute-force 불필요**: 이미 알려진 계정 정보를 입력하는 것만으로 최상위 권한(root/admin)을 획득한다.
- **자동화된 공격**: 봇넷 프로그램은 인터넷을 스캔하며 알려진 기본 계정 수만 개를 초당 수천 번씩 대입한다.
- **공급망 전이**: 특정 프레임워크나 라이브러리의 기본 계정이 뚫리면 이를 사용하는 수많은 서비스가 동시에 위협받는다.

📢 **섹션 요약 비유**: 기본 계정 방치는 새 집을 사고 현관문 비밀번호를 '1234'로 둔 채 이사 온 것과 같다. 열쇠가 없어도 누구나 들어올 수 있다.

---

## Ⅱ. 아키텍처 및 핵심 원리

공격자는 주로 다음 경로를 통해 기본 계정 취약점을 악용한다.

1. **지식 베이스 활용**: 제조사 매뉴얼(PDF)에 명시된 기본 ID/PW 리스트 확보
2. **Shodan/Censys 스캔**: 특정 장비의 배너 정보를 통해 모델 식별 후 기본 계정 대입
3. **인프라 관리 페이지**: CMS(WordPress), 데이터베이스(phpMyAdmin), CI/CD(Jenkins) 등의 초기 관리자 계정 노출

| 대상 | 일반적인 기본 계정 | 위험 요인 |
|:---|:---|:---|
| 네트워크 장비 | `admin/admin`, `cisco/cisco` | 네트워크 전체 트래픽 미러링/차단 |
| 데이터베이스 | `root/(empty)`, `sa/password` | 데이터 전체 유출 및 원격 코드 실행 |
| 미들웨어 | `manager/manager`, `admin/admin` | 서버 내 악성 웹쉘(Webshell) 배포 |
| 클라우드 인스턴스 | `ec2-user`, `ubuntu` | 초기 SSH 접근 후 권한 상승 |

```text
┌──────────────────────────────────────────────────────────────┐
│             기본 계정 공격 흐름 (Default Credential Attack)  │
├──────────────────────────────────────────────────────────────┤
│  1. 타겟 식별 (Shodan) ──▶ 2. 모델 매뉴얼 검색 (ID/PW)       │
│                                     │                        │
│  4. 시스템 장악 (Full Access) ◀── 3. 로그인 시도 (Admin UI)   │
└──────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 기본 계정 공격은 자물쇠를 부수는 게 아니라, 자물쇠 옆에 붙어있는 설명서(매뉴얼)를 보고 비밀번호를 그대로 입력하는 것이다.

## Ⅲ. 비교 및 연결

| 구분 | 기본 계정 (Default Password) | 브루트포스 (Brute-force) |
|:---|:---|:---|
| 공격 기법 | 알려진 고정 값 대입 | 가능한 모든 조합 대입 |
| 성공 확률 | 매우 높음 (설정 안 했을 시 100%) | 낮음 (시간 소요) |
| 공격 흔적 | 정상 로그인으로 기록됨 | 수많은 로그인 실패 로그 발생 |
| 예방책 | 최초 접속 시 강제 변경 | 계정 잠금 정책 (Lockout) |

운영체제 강화(OS Hardening) 단계에서 불필요한 기본 계정을 삭제하거나 비활성화하는 것이 정보보안 기사의 핵심 실무이며, 이는 ISMS-P 인증 심사에서도 '인증 및 권한 관리' 항목의 필수 점검 사항이다.

📢 **섹션 요약 비유**: 기본 계정은 '정해진 정답'을 쓰는 것이고, 브루트포스는 '정답이 나올 때까지 찍는 것'이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 실무 대응 전략 (Hardening Checklist)
1. **First-Login Change**: 시스템 설치 후 최초 로그인 시 비밀번호를 변경하지 않으면 다음 단계로 넘어가지 못하게 강제한다.
2. **Default Account Rename/Delete**: `admin`, `guest` 같은 뻔한 이름의 계정은 삭제하거나 이름을 바꾼다.
3. **IP ACL (Access Control List)**: 관리자 페이지는 내부망(VPN)이나 특정 관리자 IP에서만 접근 가능하도록 네트워크 단에서 차단한다.
4. **Credential Scanning**: `Nessus`, `OpenVAS` 등의 스캐너를 돌려 우리 인프라에 기본 계정이 남아있는지 정기적으로 전수 조사한다.

### 안티패턴
- **"나중에 바꿀게요"**: 개발 단계에서 편의를 위해 `admin/admin`으로 두고 실제 운영 서버에 그대로 배포하는 행위.
- **"매뉴얼에 있는 건 아무도 모르겠지"**: 공격자들은 전 세계 모든 장비의 매뉴얼을 데이터베이스화하여 가지고 있다.

📢 **섹션 요약 비유**: 관리자 페이지 격리는 현관문을 바꾸는 게 아니라, 현관으로 가는 길 자체를 비밀 통로(VPN)로 숨기는 것이다.

## Ⅴ. 기대효과 및 결론

기본 계정 폐쇄는 가장 적은 비용으로 가장 확실한 보안 효과를 얻을 수 있는 '가성비 1위' 보안 조치다. 보안은 복잡한 암호학 이전에 '당연히 해야 할 기본 설정'을 지키는 것에서 시작됨을 잊지 말아야 한다.

미래에는 암호 없는(Passwordless) 인증이나 하드웨어 보안 키(FIDO) 도입을 통해 기본 계정이라는 개념 자체를 없애는 방향으로 나아가고 있다.

📢 **섹션 요약 비유**: 기본 계정 차단은 보안의 '기초 공사'다. 기초가 튼튼해야 그 위에 쌓는 화려한 보안 기술이 의미가 있다.
