## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로비저닝 (Provisioning)은 사용자의 디지털 신원 정보를 기반으로 IT 자원(계정, 이메일, 애플리케이션 등)에 대한 접근 권한을 **자동으로 생성 및 배포**하는 프로세스다.
> 2. **가치**: 수동 관리로 인한 인적 오류를 제거하고, 신규 입사자가 업무를 즉시 시작할 수 있도록 '생산성'을 극대화한다.
> 3. **판단 포인트**: 프로비저닝은 단순 계정 생성이 아니라, 조직의 정책(Role, Group)에 따라 적절한 권한을 '할당'하는 동적인 과정이어야 한다.

---

## Ⅰ. 개요 및 필요성

프로비저닝 (Provisioning)은 신원 관리 시스템(IdM)의 결정 사항을 실제 타겟 시스템에 적용하는 실행 단계다. 인사 시스템에 신규 입사자 정보가 입력되면, 이를 바탕으로 AD(Active Directory) 계정을 만들고, 이메일을 생성하며, 업무용 SaaS (Slack, Jira 등)에 계정을 만들어주는 일련의 과정을 의미한다.

프로비저닝이 자동화되지 않으면 IT 관리자는 매번 수동으로 수십 개의 시스템에 들어가 계정을 만들어야 한다. 이는 시간이 오래 걸릴 뿐만 아니라, 특정 시스템의 권한 설정을 누락하거나 오타를 내는 등 보안 취약점을 유발한다. "Day 1 Readiness (출근 첫날 바로 일하기)"는 자동화된 프로비저닝 없이는 불가능하다.

- **📢 섹션 요약 비유**: 프로비저닝은 신병이 입대했을 때 총, 군복, 군화를 치수에 맞춰서 자동으로 지급해 주는 보급 창고 시스템과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

프로비저닝은 중앙의 IdM 시스템이 여러 타겟 시스템(Spoke)에 정보를 밀어넣는(Push) 방식이거나, 타겟 시스템이 정보를 가져오는(Pull) 방식으로 동작한다. 최근에는 표준 프로토콜인 **SCIM (System for Cross-domain Identity Management)**이 널리 쓰인다.

| 구성 요소 | 역할 | 비고 |
|:---|:---|:---|
| Source System | 신원 정보의 원천 (인사 시스템, ERP) | 데이터 마스터 |
| Provisioning Engine | 데이터 매핑 및 워크플로우 제어 | 로직의 중심 |
| Connector / Agent | 타겟 시스템과의 통신 인터페이스 | 시스템별 전용 API 활용 |
| Target System | 계정이 생성될 대상 (AD, Cloud App, DB) | 실제 자원 |

```text
┌──────────────────────────────────────────────────────────────┐
│                    프로비저닝 워크플로우 (Push 방식)            │
├──────────────────────────────────────────────────────────────┤
│ [HR System] ──(신규 데이터)──▶ [IAM/IdM Engine]              │
│                                     │                        │
│          ┌──────────────────────────┼─────────────────────┐  │
│          ▼                          ▼                     ▼  │
│  [Connector A]              [Connector B]         [SCIM API] │
│          │                          │                     │  │
│  [On-premise AD]            [In-house DB]         [SaaS Apps]│
└──────────────────────────────────────────────────────────────┘
```

핵심 원리는 **'속성 매핑 (Attribute Mapping)'**과 **'권한 할당 로직'**이다. 인사 시스템의 '팀명' 속성을 타겟 시스템의 '그룹 권한'으로 어떻게 변환할지 사전에 정의된 규칙에 따라 실행한다.

- **📢 섹션 요약 비유**: 프로비저닝은 아파트 관리 사무소에서 새 입주자가 오면 자동으로 현관 비번을 등록하고, 주차 등록을 해주고, 엘리베이터 권한을 주는 자동화 시스템이다.

## Ⅲ. 비교 및 연결

프로비저닝은 그 적용 범위에 따라 크게 계정 프로비저닝과 자원 프로비저닝으로 나뉘며, 최근에는 클라우드 인프라 자체를 만드는 IaC (Infrastructure as Code)와도 개념적으로 연결된다.

| 구분 | 사용자 프로비저닝 | 인프라 프로비저닝 (IaC) |
|:---|:---|:---|
| 관리 대상 | 디지털 신원 (Identity), 계정 | 서버, 네트워크, 스토리지 |
| 주 목적 | 접근 권한 부여 및 업무 환경 조성 | IT 서비스 구동 환경 구축 |
| 핵심 도구 | Okta, Azure AD, SailPoint | Terraform, Ansible, CloudFormation |
| 관련 보안 | IAM, RBAC | CSPM, 가상화 보안 |

프로비저닝이 완료된 후, 사용자가 실제 로그인할 때 쓰이는 기술이 **SSO (Single Sign-On)**다. 즉, 프로비저닝은 '문을 만드는 과정'이고, SSO는 '한 번의 인증으로 문들을 여는 과정'이다.

- **📢 섹션 요약 비유**: 사용자 프로비저닝은 직원 책상을 세팅하는 것이고, 인프라 프로비저닝은 사무실 건물 자체를 짓는 것이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

실무적으로 프로비저닝 구축 시 가장 어려운 점은 **'예외 처리'**다. 정규직 외에 계약직, 협력사 직원 등 다양한 신분 유형에 따른 프로비저닝 워크플로우를 설계해야 한다.

### 체크리스트
1. **SCIM 지원 여부**: 연동하려는 SaaS가 SCIM 표준을 지원하는가? (미지원 시 커스텀 개발 필요)
2. **데이터 정규화**: 소스 시스템의 데이터 형식이 타겟 시스템과 호환되는가?
3. **승인 워크플로우**: 자동 생성 전, 팀장의 승인 단계가 필요한가?
4. **결과 모니터링**: 프로비저닝 실패 시 알림 및 재시도 로직이 있는가?

### 안티패턴
- 모든 입사자에게 동일한 '최대 권한'을 프로비저닝하는 행위 (최소 권한 원칙 위배)
- 프로비저닝 기록(Log)을 남기지 않아 사후 감사가 불가능한 경우

- **📢 섹션 요약 비유**: 프로비저닝 설계는 피자 주문 시스템과 같다. 토핑(권한)을 정확히 선택하지 않으면 엉뚱한 피자(과도한 권한)가 배달된다.

---

## Ⅴ. 기대효과 및 결론

자동화된 프로비저닝은 조직의 **'민첩성 (Agility)'**을 높인다. 신규 프로젝트 팀 구성이나 조직 개편 시 수백 명의 권한을 몇 분 만에 조정할 수 있다. 또한, 인적 실수를 줄여 보안 사고의 위험을 획기적으로 낮추는 방어 기제이기도 하다.

결론적으로 프로비저닝은 현대적 IAM의 엔진이다. 향후에는 AI가 사용자의 행동 패턴을 분석하여 필요한 권한을 예측하고 선제적으로 제안하는 '인텔리전트 프로비저닝'으로 진화할 전망이다.

- **📢 섹션 요약 비유**: 프로비저닝은 잘 닦인 고속도로와 같다. 차(신원)가 들어오자마자 톨게이트를 지나 목적지(업무 자원)까지 멈춤 없이 달리게 해준다.

### 👶 어린이를 위한 3줄 비유 설명

1. 프로비저닝은 새 친구가 전학 왔을 때 교과서, 책상, 급식판을 한꺼번에 준비해 주는 마법이에요.
2. 이 마법이 없으면 친구는 한참 동안 책도 없이 수업을 들어야 해서 힘들 거예요.
3. 컴퓨터가 알아서 척척 준비해 주니까 선생님도 편하고 새 친구도 신나게 학교생활을 시작할 수 있어요.
