## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Active Directory (AD)는 Windows 도메인 네트워크 환경에서 사용자, 컴퓨터, 그룹 등 모든 자원을 중앙 집중적으로 관리하는 **Microsoft의 디렉터리 서비스 솔루션**이다.
> 2. **가치**: 한 번의 로그인으로 네트워크 내 모든 허용된 자원에 접근하는 SSO를 제공하며, **그룹 정책 (GPO)**을 통해 수천 대의 PC 보안 설정을 일괄 제어한다.
> 3. **판단 포인트**: AD는 기업 보안의 '최종 보스'와 같아, AD 침해는 곧 전사 네트워크 장악을 의미하므로 **Tier 모델 기반의 계층적 보호**가 필수적이다.

---

## Ⅰ. 개요 및 필요성

Active Directory (AD)는 기업 IT 환경의 '신경계'다. 직원이 출근해서 PC를 켜고 로그인하는 순간부터 메일을 보내고 파일 서버에 접근하기까지, 모든 과정의 이면에는 AD가 존재한다. 

AD가 없는 대규모 조직은 상상하기 어렵다. 각 PC마다 아이디를 만들고 보안 패치를 수동으로 돌려야 한다면 관리 효율은 제로에 수렴할 것이다. AD는 수만 명의 신원 정보를 **LDAP**으로 관리하고, 인증은 **Kerberos**로 수행하며, 관리는 **GPO**로 자동화함으로써 "중앙에서 통제되는 안전한 업무 환경"을 실현한다.

- **📢 섹션 요약 비유**: AD는 대형 아파트 단지의 '통합 관리 사무소'와 같다. 모든 세대의 열쇠(계정)를 관리하고, 엘리베이터 권한(인가)을 조정하며, 단지 내 규칙(그룹 정책)을 정해 주민들의 안전을 책임진다.

## Ⅱ. 아키텍처 및 핵심 원리

AD는 논리적으로 **Object(객체), Organizational Unit(OU), Domain, Tree, Forest** 순으로 확장되는 계층 구조를 가진다.

| 계층 요소 | 설명 | 비고 |
|:---|:---|:---|
| **Domain Controller (DC)** | AD 서비스를 실행하고 데이터베이스를 유지하는 서버 | 인증의 핵심 장치 |
| **Global Catalog (GC)** | Forest 내의 모든 객체 정보를 담은 중앙 저장소 | 빠른 검색 지원 |
| **Group Policy (GPO)** | 사용자나 컴퓨터에 적용할 보안/환경 설정 묶음 | 자동화된 정책 시행 |
| **Active Directory DB** | 실제 데이터가 저장되는 파일 (`ntds.dit`) | 가장 민감한 보호 대상 |

### AD의 논리적 계층 구조

```text
┌──────────────────────────────────────────────────────────────┐
│                    Active Directory Forest 아키텍처              │
├──────────────────────────────────────────────────────────────┤
│ [Forest: 전체 조직의 신뢰 경계]                                  │
│    │                                                         │
│    ├─ [Tree A: example.com]          [Tree B: partner.net]   │
│    │     │                                  │                │
│    │     └─ [Domain: sales.example.com]     └─ [Domain: dev] │
│    │           │                                             │
│    │           └─ [OU: Korea] ──▶ [Object: User, Computer]   │
│    │                                                         │
├──────────────────────────────────────────────────────────────┤
│ [Auth Flow: Kerberos] ──▶ TGT/ST 발급 ──▶ 자원 접근            │
└──────────────────────────────────────────────────────────────┘
```

핵심 원리는 **'신뢰의 전이 (Transitive Trust)'**와 **'중앙 집중식 권한 부여'**다. 포리스트 내의 도메인들은 서로 신뢰 관계를 맺어, 한 도메인의 사용자가 다른 도메인의 자원을 이용할 수 있게 한다. 또한 GPO는 하위 계층으로 상속되어, 한 번의 설정으로 전 직원의 화면 잠금 시간을 5분으로 고정하는 식의 강력한 통제를 수행한다.

- **📢 섹션 요약 비유**: AD는 거대한 '국가 행정 시스템'과 같다. 시(Domain)가 다르고 구(OU)가 달라도 대한민국 국민(Forest)이라면 어디서든 주민등록증(인증)을 쓰고 법(GPO)의 적용을 받는 것과 같다.

## Ⅲ. 비교 및 연결

AD는 고전적인 온프레미스 방식이며, 최근에는 클라우드 버전인 Azure AD (Entra ID)와 상호보완적으로 사용된다.

| 항목 | On-premise AD (Active Directory) | Azure AD (Microsoft Entra ID) |
|:---|:---|:---|
| 인증 프로토콜 | Kerberos, NTLM | OIDC, SAML, OAuth 2.0 |
| 관리 대상 | 서버, PC, 프린터, 그룹 정책 | 클라우드 앱, 사용자 신원, 기기 |
| 디바이스 관리 | 도메인 가입 (Domain Join) | 등록 (Registration / Intune) |
| 구조 | 계층적 (Tree/Forest) | 평면적 (Flat) |

AD는 보안 사고의 핵심 타겟으로, **Golden Ticket**이나 **Silver Ticket** 공격과 같은 Kerberos 기반 공격에 취약할 수 있어 상시 모니터링이 필수적이다.

- **📢 섹션 요약 비유**: AD가 '오프라인 본사 사옥'의 출입 통제라면, Azure AD는 '온라인 가상 오피스'의 신분 확인 시스템이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사적 관점에서 AD 보안의 핵심은 **'공격 표면 (Attack Surface) 최소화'**와 **'Tier 모델'** 적용이다. 관리자 계정이 탈취되어 DC가 장악되는 시나리오를 반드시 막아야 한다.

### 체크리스트
1. **Tier 모델 적용**: 도메인 관리자(Tier 0) 계정이 보안이 취약한 일반 PC(Tier 2)에 로그인하지 않도록 격리했는가?
2. **비밀번호 정책**: 복잡성, 길이, 이력 관리가 GPO를 통해 강제되고 있는가?
3. **취약 프로토콜 비활성화**: 보안이 취약한 **NTLMv1**이나 **SMBv1**을 제거했는가?
4. **백업 및 복구**: DC 장애나 랜섬웨어 감염 시 `ntds.dit` 파일을 복구할 수 있는 오프라인 백업이 있는가?

### 안티패턴
- 모든 관리자가 하나의 `Administrator` 공용 계정을 사용하여 책임추적성을 상실하는 경우
- 퇴사자의 계정을 즉시 비활성화하지 않고 방치하여 공격자의 거점으로 제공하는 경우

- **📢 섹션 요약 비유**: AD 보안은 '왕국 지키기'와 같다. 성문(인증)만 잘 지키는 게 아니라, 왕의 침실(DC)로 가는 길을 급에 따라 철저히 나누고(Tier 모델) 감시해야 한다.

## Ⅴ. 기대효과 및 결론

AD의 도입은 IT 자원의 **'가시성'**과 **'통제력'**을 극대화한다. 수만 대의 장비를 단 몇 명의 관리자가 완벽하게 제어할 수 있는 규모의 경제를 실현하며, 이는 기업의 정보보안 수준을 상향 평준화하는 결과를 낳는다.

결론적으로 AD는 현대 기업 인프라의 '뿌리'다. 비록 클라우드로 주도권이 넘어가고 있지만, 하이브리드 환경에서 AD에 대한 깊은 이해는 보안 전문가에게 필수적인 역량이다. AD를 단순히 관리 도구가 아닌, 기업의 신뢰를 유지하는 '보안의 심장'으로 대우해야 한다.

- **📢 섹션 요약 비유**: AD는 오케스트라의 '악보'와 같다. 연주자(사용자)가 바뀌고 악기(기기)가 늘어나도, 하나의 악보(중앙 정책) 아래에서 완벽한 합주(비즈니스)를 가능하게 한다.
