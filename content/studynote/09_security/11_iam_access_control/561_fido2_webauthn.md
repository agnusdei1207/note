## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 공개키 암호화(PKI) 기술을 기반으로, 서버에 비밀번호를 저장하지 않고 사용자의 기기(인증자)에서 본인 확인을 수행하는 글로벌 인증 표준이다.
> 2. **가치**: 비밀번호 자체를 없애(Passwordless) 피싱, 서버 해킹을 통한 대규모 유출 위협을 근본적으로 제거하며, 생체 인증을 통한 높은 편의성을 제공한다.
> 3. **판단 포인트**: WebAuthn(웹 브라우저 표준)과 CTAP2(기기 간 통신 표준)의 결합으로 하드웨어 키, 스마트폰, 노트북 생체 센서를 모두 아우르는 통합 보안 생태계를 구축한다.

---

## Ⅰ. 개요 및 필요성

FIDO2 (Fast IDentity Online 2)는 비밀번호의 한계를 극복하기 위해 FIDO 얼라이언스가 제정한 차세대 인증 표준이다. 이전의 FIDO 1.0(UAF/U2F)이 모바일 앱 중심이었다면, FIDO2는 웹 브라우저에서도 플러그인 없이 작동하도록 확장되었다.

우리가 사용하는 비밀번호는 서버에 저장되므로 서버가 뚫리면 모두 유출된다. 하지만 FIDO2는 **'내 기기에서 내가 나임을 증명'**하고 서버에는 그 결과(서명)만 보낸다. 서버는 내 지문이나 비밀번호를 절대 알 수 없으며, 오직 내 기기에서 온 정당한 서명인지만 확인한다. 이것이 FIDO2가 피싱과 서버 해킹에 무적인 이유다.

- **📢 섹션 요약 비유**: FIDO2는 인감도장을 집(기기)에만 두고, 은행(서버)에는 인감증명서(공개키)만 등록해두는 것과 같다. 도장은 절대 밖으로 나가지 않는다.

---

## Ⅱ. 아키텍처 및 핵심 원리

FIDO2는 **WebAuthn API** (브라우저-서버 통신)와 **CTAP2** (외부 인증키-브라우저 통신)로 구성된다. 핵심 메커니즘은 **'공개키 기반 챌린지-응답(Challenge-Response)'** 흐름이다.

| 구성 요소 | 역할 | 비고 |
|:---|:---|:---|
| Authenticator (인증자) | 지문인식기, 보안 키, TPM 등 | 개인키를 생성하고 안전하게 보관 |
| FIDO Client (브라우저) | WebAuthn API를 통해 인증자와 통신 | 크롬, 사파리, 엣지 등 현대 브라우저 |
| Relying Party (RP) | 구글, 네이버 등 실제 서비스를 제공하는 서버 | 공개키를 저장하고 서명을 검증 |

```text
┌──────────────────────────────────────────────────────────────┐
│             FIDO2/WebAuthn 인증(Assertion) 흐름              │
├──────────────────────────────────────────────────────────────┤
│ [RP 서버] ────(1) 챌린지(난수) 전송──▶ [웹 브라우저]            │
│                                           │                  │
│ [사용자] ◀───(2) "지문 대세요" ────── [인증자]                │
│    │                                      │                  │
│    └──────────(3) 생체 인증 완료 ─────▶ [개인키로 서명]        │
│                                           │                  │
│ [RP 서버] ◀───(4) 서명값 전송 ──────────┘                  │
│    │ (저장된 공개키로 서명 검증)                               │
│    ▼                                                         │
│ [인증 성공!]                                                 │
└──────────────────────────────────────────────────────────────┘
```

**핵심 포인트**:
1. **등록 시**: 인증자가 개인키/공개키 쌍을 생성하고, 공개키만 서버로 보낸다.
2. **인증 시**: 서버가 보낸 난수(Challenge)를 기기 내부의 개인키로 서명하여 돌려보낸다.
3. **보안성**: 개인키는 기기 내 보안 영역(TEE/SE) 밖으로 절대 나가지 않으며, 서버 유출 시에도 개인키는 안전하다.

- **📢 섹션 요약 비유**: FIDO2 아키텍처는 금고 주인이 금고 앞에서 지문을 찍으면, 금고가 자동으로 "주인 맞음"이라는 확인 도장을 찍어 편지를 보내주는 시스템이다.

## Ⅲ. 비교 및 연결

FIDO2는 기존의 모든 인증 수단을 대체하거나 통합할 수 있는 강력한 표준이다.

| 항목 | 일반 비밀번호 | SMS/TOTP MFA | FIDO2 / WebAuthn |
|:---|:---|:---|:---|
| 서버 저장 데이터 | 비밀번호 해시 | 공유 비밀키(Seed) | **공개키 (Public Key)** |
| 피싱 방지 | 취약 | 부분적 방어 | **완벽 방어 (도메인 바인딩)** |
| 편의성 | 낮음 (기억 필요) | 보통 (입력 필요) | **매우 높음 (생체 인식)** |
| 장치 의존성 | 없음 | 기기 필요 | 전용 기기/센서 필요 |

FIDO2는 최근 **Passkey (패스키)** 기술의 근간이 된다. Passkey는 FIDO2의 강력한 보안을 유지하면서, 여러 기기 간에 인증 정보를 동기화하는 편의성을 추가한 개념이다.

- **📢 섹션 요약 비유**: 비밀번호가 종이에 적은 암호라면, FIDO2는 복제가 불가능한 생체 인식 카드키다.

---

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 FIDO2를 도입할 때는 브라우저 호환성과 사용자의 기기 지원 여부를 먼저 파악해야 한다. 모든 사용자가 생체 인식 기기를 가진 것은 아니므로, 하이브리드 전략이 필요하다.

### 체크리스트
1. 브라우저가 WebAuthn API를 지원하는 최신 버전인가?
2. 인증자(Authenticator)가 FIDO2 인증을 받은 신뢰할 수 있는 장치인가? (Attestation 확인)
3. 기기를 잃어버린 사용자를 위한 '복구용 하드웨어 키'나 대체 인증 수단이 마련되어 있는가?

### 안티패턴
- FIDO2를 도입하면서도 보안을 위해 서버에 비밀번호 원문을 병행 저장하는 행위 (도입 의미 퇴색)
- 도메인 검증을 생략하여, 가짜 사이트에서 FIDO2 서명을 요청하게 방치하는 경우

- **📢 섹션 요약 비유**: 실무의 FIDO2 도입은 건물 출입 시스템을 지문 인식으로 바꾸는 것과 같다. 방문객을 위한 임시 카드(대체 인증)를 준비하는 세심함이 성공의 관건이다.

---

## Ⅴ. 기대효과 및 결론

FIDO2는 인증의 역사를 "비밀번호 이전"과 "비밀번호 이후"로 나눌 만큼 혁신적이다. 피싱 사이트가 아무리 똑같이 생겼어도, 브라우저가 도메인을 확인하여 서명을 거부하기 때문에 피싱 자체가 원천 차단된다.

결론적으로, FIDO2/WebAuthn은 현대 웹 보안의 종착역이다. 앞으로 모든 서비스는 비밀번호를 버리고 FIDO2 기반의 Passkey 체제로 전환될 것이며, 이는 인터넷 환경을 훨씬 안전하고 쾌적하게 만들 것이다.

- **📢 섹션 요약 비유**: FIDO2는 보안의 '마법 방패'다. 적이 내 이름을 알아도 방패(개인키)는 뺏을 수 없고, 가짜 성(피싱 사이트)에서는 방패가 스스로 작동하지 않는다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| WebAuthn | 웹 브라우저에서 FIDO2 인증을 사용하기 위한 표준 API |
| CTAP2 | 보안 키(YubiKey 등)와 PC/폰이 통신하기 위한 표준 프로토콜 |
| Attestation | 인증자가 진짜 안전한 기기인지 서버가 확인하는 인증서 |
| Origin Binding | 특정 웹사이트 주소(Domain)에서만 인증이 작동하도록 묶는 보안 장치 |

2. 내 핸드폰이나 노트북이 "이 사람은 우리 주인님 맞아요!"라고 서버에 도장을 찍어 보내주는 거예요.
3. 서버가 해킹당해도 내 지문은 내 물건 속에만 있으니까 절대로 안전하답니다!
