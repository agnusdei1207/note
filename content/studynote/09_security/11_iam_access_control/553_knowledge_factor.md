## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자의 머릿속에 저장된 정보(What you know)를 기반으로 신원을 확인하는 가장 전통적인 인증 요인이다.
> 2. **가치**: 별도의 장치 없이도 어디서나 인증이 가능하며 구현 비용이 매우 낮아 광범위하게 사용된다.
> 3. **판단 포인트**: 망각, 유출, 무차별 대입 공격에 취약하므로 복잡도 정책(Complexity)과 함께 다중 인증(MFA)의 일부로 사용되는 것이 권장된다.

---

## Ⅰ. 개요 및 필요성

지식 요인 (Knowledge Factor)은 오직 정당한 사용자만이 알고 있는 정보를 제시하여 본인임을 증명하는 방식이다. 대표적으로 비밀번호 (Password), PIN (Personal Identification Number), 그리고 "당신의 첫 번째 반려동물 이름은?"과 같은 보안 질문이 여기에 해당한다.

디지털 시스템 초기부터 현재까지 가장 널리 쓰이는 이유는 인프라 구축의 용이성 때문이다. 사용자에게 무언가를 나누어 주거나(소유 요인), 생체 정보를 등록받는(내재 요인) 절차 없이 소프트웨어 설정만으로 즉시 운영이 가능하기 때문이다.

- **📢 섹션 요약 비유**: 지식 요인은 비밀 일기장을 열기 위해 미리 정해둔 '암호'와 같다. 나만 알고 있으면 어디서든 열 수 있다.

---

## Ⅱ. 아키텍처 및 핵심 원리

지식 요인 인증의 핵심은 서버에 저장된 **검증값(해시)**과 사용자가 입력한 **제시값**의 일치 여부를 판단하는 것이다. 보안을 위해 서버는 비밀번호 원문을 저장하지 않고 일방향 해시 함수와 솔트(Salt)를 사용한다.

| 요소 | 설명 | 보안 고려 사항 |
|:---|:---|:---|
| 비밀번호 (Password) | 문자, 숫자, 특수문자의 조합 | 복잡도, 길이, 주기적 변경 |
| PIN | 통상 4~6자리의 숫자 조합 | 짧은 길이로 인한 Brute-force 위협 |
| 보안 질문 | 개인적인 기억 기반 정보 | 사회 공학(SNS 등)을 통한 유추 가능성 |
| 패턴 (Pattern) | 점을 잇는 기하학적 형상 | 화면 지문 잔상(Smudge) 노출 위험 |

```text
┌──────────────────────────────────────────────────────────────┐
│                지식 요인 검증의 기술적 흐름                  │
├──────────────────────────────────────────────────────────────┤
│ [사용자 입력: "P@ss123"] ───▶ [해시 함수 + Salt] ───┐        │
│                                       │              │        │
│                                       ▼              │        │
│ [DB 저장된 해시값] ◀────────── [일치 여부 비교] ◀────┘        │
│                                       │                       │
│                                       └─▶ [성공/실패 응답]    │
└──────────────────────────────────────────────────────────────┘
```

이 방식의 아킬레스건은 '사용자가 외워야 한다'는 점이다. 복잡할수록 보안은 올라가지만 사용자는 잊어버리기 쉽고, 쉬울수록 사용자는 편하지만 공격자가 맞추기 쉬워진다.

- **📢 섹션 요약 비유**: 지식 요인은 금고 번호를 맞추는 것과 같다. 번호가 너무 단순하면 도둑이 맞추고, 너무 어려우면 주인도 못 연다.

## Ⅲ. 비교 및 연결

지식 요인은 다른 요인들과 비교했을 때 가장 확장성이 좋지만 독립적 보안성은 가장 낮다.

| 구분 | 지식 요인 (Knowledge) | 소유 요인 (Possession) | 내재 요인 (Inherence) |
|:---|:---|:---|:---|
| 공유 가능성 | 매우 쉬움 (문자 전달) | 어려움 (실물 전달 필요) | 불가 (신체 일부) |
| 비용 | 거의 없음 | 장치 구매 비용 발생 | 센서 구축 비용 발생 |
| 관리 부담 | 기억의 한계 | 분실/파손 위험 | 변경 불가능성 (유출 시 치명적) |

최근에는 지식 요인의 한계를 극복하기 위해 비밀번호 없는 인증(Passwordless) 기술인 FIDO2나 Passkey가 등장하고 있으며, 지식 요인은 MFA의 첫 번째 관문(1st Factor)으로서 주로 활용된다.

- **📢 섹션 요약 비유**: 지식 요인은 가벼운 천막의 지퍼와 같다. 혼자서는 약하지만, 다른 자물쇠와 합쳐지면 훌륭한 문이 된다.

---

## Ⅳ. 실무 적용 및 기술사 판단

실무에서는 비밀번호 복잡도 정책과 잠금(Lockout) 정책을 적절히 조율해야 한다. 너무 엄격한 정책은 사용자가 비밀번호를 포스트잇에 적어 모니터에 붙이게 만드는 역효과(Shadow Security)를 낳는다.

### 체크리스트
1. 서버 DB에 비밀번호가 평문이 아닌 Salted Hash 형태로 저장되어 있는가?
2. 연속된 실패 시 계정을 일시 잠금하거나 지연 시간(Back-off)을 부여하는가?
3. 흔히 쓰이는 비밀번호(Password123!)를 금지 리스트(Blacklist)로 관리하는가?

### 안티패턴
- 비밀번호 변경을 너무 자주 강제하여(예: 매달), 사용자가 끝자리 숫자만 1씩 올리게 만드는 것
- 보안 질문의 답이 SNS(인스타그램 등)를 통해 쉽게 유추 가능한 정보인 경우

- **📢 섹션 요약 비유**: 실무의 지식 요인 관리는 등산로 안내와 같다. 너무 험하게 만들면 사람들이 길을 벗어나(포스트잇) 엉뚱한 곳으로 간다.

---

## Ⅴ. 기대효과 및 결론

지식 요인은 인류가 발명한 가장 가성비 좋은 인증 수단이다. 완벽하지는 않지만, 모든 사람에게 즉시 적용 가능하다는 점에서 여전히 가치가 크다.

결론적으로, 지식 요인의 미래는 "단독 사용의 지양"에 있다. 비밀번호는 점차 MFA의 한 축으로 밀려나거나, Passkey 같은 기술에 의해 사용자 모르게 뒤에서 작동하는 방식으로 숨겨질 것이다.

- **📢 섹션 요약 비유**: 지식 요인은 낡았지만 튼튼한 캔버스 운동화와 같다. 비싼 구두는 아니어도 가장 대중적이며, 양말(MFA)과 함께 신을 때 가장 안전하다.

|:---|:---|
| 해시 함수 (Hash Function) | 지식 요인을 안전하게 저장하기 위한 필수 수학 도구 |

| 무차별 대입 (Brute-force) | 모든 가능한 조합을 시도하여 지식 요인을 알아내는 공격 |
| 솔트 (Salt) | 같은 비밀번호라도 다른 해시값이 나오게 만드는 무작위 데이터 |
