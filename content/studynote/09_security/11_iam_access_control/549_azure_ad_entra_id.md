## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Microsoft Entra ID (구 Azure AD)는 클라우드 환경에서 사용자 신원과 자원 접근을 관리하는 **IDaaS (Identity as a Service) 플랫폼**이다.
> 2. **가치**: 수천 개의 SaaS 애플리케이션에 대한 통합 SSO를 제공하며, **조건부 액세스 (Conditional Access)**를 통해 실시간 위협 기반의 동적 보안 판단을 수행한다.
> 3. **판단 포인트**: 기존 온프레미스 AD의 단순한 '클라우드 복제본'이 아니라, **제로 트러스트 아키텍처**를 구현하는 클라우드 네이티브 신원 엔진으로 이해해야 한다.

---

## Ⅰ. 개요 및 필요성

비즈니스 무대가 사무실에서 클라우드로 옮겨가면서, 전통적인 Active Directory(AD)는 한계에 부딪혔다. 방화벽 밖의 SaaS 앱(Office 365, Slack, Salesforce 등)에 대한 인증을 AD가 직접 처리하기에는 보안과 기술적 제약이 컸기 때문이다.

이러한 배경에서 탄생한 Microsoft Entra ID는 클라우드 시대의 '새로운 보안 경계'다. 이제 직원은 회사망 밖에서도 자신의 계정 하나로 모든 업무 도구에 안전하게 접속할 수 있다. 특히 머신러닝을 기반으로 사용자의 로그인 위치나 기기 상태를 분석하여 위험을 감지하는 지능형 보안 기능은 현대 기업 보안의 필수 요소가 되었다.

- **📢 섹션 요약 비유**: Entra ID는 전 세계 어디서나 결제와 본인 인증이 가능한 '스마트폰 모바일 신분증'과 같다. 지갑(사내망)을 두고 나와도 스마트폰(클라우드)만 있으면 모든 일을 안전하게 처리할 수 있는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

Entra ID는 평면적인 구조의 **Tenant (테넌트)** 기반으로 동작하며, 표준 웹 프로토콜을 사용하여 서비스를 연결한다.

| 구성 요소 | 역할 | 비고 |
|:---|:---|:---|
| **Directory** | 사용자, 그룹, 기기 정보의 논리적 컨테이너 | Tenant 단위 관리 |
| **Authentication** | OIDC, SAML 2.0, OAuth 2.0 기반 인증 | 최신 웹 표준 준수 |
| **Conditional Access** | 신호(Signal)를 분석하여 접근 허용 여부 결정 | 제로 트러스트의 두뇌 |
| **Identity Protection** | 유출된 자격 증명이나 이상 징후 탐지 | AI 기반 위험 분석 |

### Entra ID 보안 결정 메커니즘 (Conditional Access)

```text
┌──────────────────────────────────────────────────────────────┐
│                  Entra ID 조건부 액세스 동작 원리                │
├──────────────────────────────────────────────────────────────┤
│  [입력 신호 (Signals)]  ──▶  [판단 엔진 (Decision)]  ──▶  [결과 (Enforcement)] │
│  - 사용자 위치 (IP)        - 위험 수위 분석          - 접근 허용          │
│  - 사용 기기 (보안 상태)   - 정책 대조               - MFA 요구 (Step-up) │
│  - 접속 앱 (민감도)        - 기계 학습 (ML)          - 비밀번호 변경 요구 │
│  - 실시간 세션 위험        - 예외 조건 체크          - 접근 차단          │
└──────────────────────────────────────────────────────────────┘
```

핵심 원리는 **'상황 인지 보안 (Context-Aware Security)'**이다. 단순히 ID와 PW가 맞다고 들여보내는 것이 아니라, "평소 한국에서 접속하던 사용자가 10분 만에 미국에서 접속했는가?"와 같은 맥락을 따진다. 모든 접속 요청은 매번 독립적으로 검증되며, 의심스러운 정황이 포착되면 즉시 추가 인증을 요구하거나 차단한다.

- **📢 섹션 요약 비유**: Entra ID는 인공지능이 탑재된 '지능형 보안 검색대'와 같다. 여권이 진짜라도 가방 안의 위험물(보안 위협)이나 여행 경로(접속 맥락)가 수상하면 정밀 검사(MFA)를 하는 것과 같다.

## Ⅲ. 비교 및 연결

가장 많이 혼동되는 온프레미스 AD와의 차이점은 '기술적 언어'와 '관리 범위'에 있다.

| 항목 | Windows Server AD (AD DS) | Microsoft Entra ID (Azure AD) |
|:---|:---|:---|
| **통신 방식** | LDAP, Kerberos, NTLM | HTTP/HTTPS (REST API) |
| **기기 관리** | Group Policy (GPO) | MDM (Intune) |
| **신뢰 경계** | 기업 내부망 (Intranet) | 전 세계 인터넷 (Internet) |
| **확장성** | 서버 추가 필요 (수직적) | SaaS 서비스 (수평적/무제한) |

이 두 세계를 연결하는 다리가 바로 **Azure AD Connect**이며, 이를 통해 기업은 기존 AD의 사용자 정보를 클라우드로 안전하게 동기화하여 하이브리드 환경을 구축한다.

- **📢 섹션 요약 비유**: 서버 AD가 '우리 집 안방 자금'이라면, Entra ID는 '전 세계 ATM에서 꺼내 쓸 수 있는 가상 화폐'와 같다.

## Ⅳ. 실무 적용 및 기술사 판단

실무적으로 Entra ID 도입 시 가장 중요한 의사결정 포인트는 **'라이선스 등급(P1, P2)'**과 **'보안 기본값'** 설정이다.

### 체크리스트
1. **MFA 강제화**: 모든 사용자에 대해, 최소한 관리자 계정에 대해서는 MFA가 활성화되어 있는가?
2. **조건부 액세스 정책**: "회사 기기에서만 특정 데이터 접근 가능"과 같은 제로 트러스트 정책이 수립되었는가?
3. **App Registration 관리**: 외부 앱에 부여된 API 권한이 과도하지 않은지 주기적으로 검토하는가?
4. **셀프 서비스**: 비밀번호 재설정(SSPR) 기능을 활성화하여 헬프데스크 운영 비용을 절감하고 있는가?

### 안티패턴
- 보안 강화를 명목으로 사용자의 접속 환경을 지나치게 제약하여 업무 생산성을 떨어뜨리는 경우
- 레거시 인증(Legacy Auth)을 허용해두어 공격자가 MFA를 우회하도록 방치하는 경우

- **📢 섹션 요약 비유**: Entra ID 운영은 스마트 홈 관리와 같다. 외출 중에도(클라우드) 가전제품(앱)을 제어할 수 있지만, 스마트폰(계정)을 잃어버리면 집 전체가 위험해지므로 이중 잠금(MFA)이 필수다.

## Ⅴ. 기대효과 및 결론

Entra ID는 기업에게 **'글로벌 수준의 보안 인프라'**를 즉시 빌려준다. 중소기업도 Microsoft가 투자하는 거대한 보안 데이터와 AI 탐지 기능을 사용하여 대기업 수준의 방어력을 갖출 수 있다. 또한 사용자는 단 한 번의 로그인으로 모든 업무를 처리하는 극강의 편의성을 누린다.

결론적으로 Entra ID는 **디지털 전환의 관문**이다. 기술사는 단순히 '클라우드 로그인 서비스'로 보지 말고, 기업의 모든 자산과 사용자를 잇는 '신뢰의 허브'로서 아키텍처를 설계하고 보호해야 한다.

- **📢 섹션 요약 비유**: Entra ID는 도시의 '중앙 관제 센터'와 같다. 도시 구석구석(모든 앱)을 실시간으로 감시하고 안전을 확인하여 시민(사용자)들이 안심하고 생활할 수 있게 돕는다.
