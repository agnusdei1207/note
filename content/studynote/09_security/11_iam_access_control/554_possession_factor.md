## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자가 물리적으로 가지고 있는 고유한 매체(What you have)를 통해 신원을 증명하는 인증 요인이다.
> 2. **가치**: 지식 요인(비밀번호)과 달리 복제가 어렵고 물리적 점유가 필수적이어서, 원격지 공격에 대한 강력한 방어력을 제공한다.
> 3. **판단 포인트**: 매체의 분실, 파손, 도난 시 서비스 이용이 불가능해지므로 신속하고 안전한 재발급 및 대체 인증 수단 확보가 필수적이다.

---

## Ⅰ. 개요 및 필요성

소유 요인 (Possession Factor)은 "당신이 무엇을 가지고 있는가?"라는 질문에 답하는 인증 방식이다. 스마트폰 앱(Push 알림), OTP 생성기, 보안 토큰, 스마트카드, 그리고 최근의 하드웨어 보안 키(YubiKey 등)가 여기에 해당한다.

비밀번호(지식 요인)만으로는 피싱이나 대규모 유출 사고를 막기 어렵다는 인식이 퍼지면서 소유 요인의 중요성이 커졌다. 해커가 내 비밀번호를 알아내더라도, 내 손에 들려 있는 스마트폰이나 하드웨어 키까지 뺏어가지 못한다면 계정을 탈취할 수 없기 때문이다.

- **📢 섹션 요약 비유**: 소유 요인은 우리 집 현관문을 열기 위한 '실물 열쇠'와 같다. 비밀번호를 알아도 열쇠가 없으면 들어올 수 없다.

---

## Ⅱ. 아키텍처 및 핵심 원리

소유 요인은 크게 **단방향 생성 방식(OTP)**과 **양방향 챌린지-응답 방식**으로 나뉜다. 최근에는 사용자의 편의성을 위해 스마트폰의 푸시(Push) 기능을 이용하거나 보안 칩(TPM/Secure Enclave)에 저장된 개인키를 활용하는 방식이 주를 이룬다.

| 매체 유형 | 구동 방식 | 특징 |
|:---|:---|:---|
| HW 토큰 (OTP) | 시간/이벤트 기반 번호 생성 | 네트워크 연결 불필요, 배터리 소모 |
| 스마트폰 (App) | Push 알림 또는 소프트 OTP | 범용성 높음, 앱 보안이 관건 |
| 하드웨어 키 | PKI 기반 서명 인증 | 가장 강력한 보안, 별도 구매 필요 |
| SMS / E-mail | 인증 코드 전송 | 가로채기(SIM Swapping) 취약점 존재 |

```text
┌──────────────────────────────────────────────────────────────┐
│                소유 요인 기반의 Push 인증 흐름               │
├──────────────────────────────────────────────────────────────┤
│ [사용자 PC] ─────(1) 로그인 시도────▶ [서비스 서버]           │
│                                           │                  │
│ [사용자 폰] ◀────(2) Push 알림 전송 ─── [인증 서버]           │
│      │                                    ▲                  │
│      └───────(3) '승인' 버튼 클릭 ────────┘                  │
│                                           │                  │
│ <────────────(4) 로그인 완료 알림 ────────┘                  │
└──────────────────────────────────────────────────────────────┘
```

소유 요인의 핵심은 **'고유성'**이다. 특정 장치와 사용자의 계정이 1:1로 묶여(Binding) 있어야 하며, 이를 위해 인증 과정에서 장치의 고유 식별자나 암호화된 인증서가 사용된다.

- **📢 섹션 요약 비유**: 소유 요인 아키텍처는 놀이공원 자유이용권 팔찌와 같다. 내 팔에 채워진 팔찌를 보여줘야만 놀이기구를 탈 수 있다.

## Ⅲ. 비교 및 연결

소유 요인은 지식 요인의 취약성을 보완하며, 내재 요인(생체 인증)을 안전하게 전달하는 통로 역할을 하기도 한다.

| 항목 | 지식 요인 (Knowledge) | 소유 요인 (Possession) | 내재 요인 (Inherence) |
|:---|:---|:---|:---|
| 도난 시 위험 | 즉시 원격 사용 가능 | 실물 탈취 전까지 안전 | 탈취 불가능 (복제 위험) |
| 복제 용이성 | 매우 쉬움 | 하드웨어에 따라 매우 어려움 | 센서 성능에 따라 갈림 |
| 주요 장애 | 망각 (Forget) | 분실 (Lose) | 상해/노화 (Change) |

소유 요인은 특히 **FIDO (Fast IDentity Online)** 프로토콜에서 중요한 역할을 한다. 하드웨어 보안 키나 스마트폰 자체가 인증자(Authenticator)가 되어, 서버로 비밀번호를 보내지 않고도 소유권을 증명한다.

- **📢 섹션 요약 비유**: 소유 요인은 지식 요인이라는 성벽 앞에 세워진 '해자(Moat)'와 같다. 성문을 여는 법을 알아도 물을 건널 배(장치)가 없으면 못 들어온다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 소유 요인을 도입할 때 가장 큰 고민은 '사용자가 장치를 잃어버렸을 때'이다. 이를 대비한 백업 수단(복구 코드 등)이 보안을 해치지 않도록 설계해야 한다.

### 체크리스트
1. SMS 인증을 사용할 경우, SIM Swapping 공격에 대한 방어 대책이 있는가? (가급적 앱 푸시 권장)
2. 하드웨어 토큰 도입 시, 배터리 수명 및 대량 배포/회수 프로세스가 수립되었는가?
3. 기기 변경 시 기존 기기와의 연결을 안전하게 해지하고 새 기기를 등록하는 절차가 있는가?

### 안티패턴
- 스마트폰 분실 시, 이메일로만 본인 확인을 하고 즉시 인증 기기를 변경하게 해주는 경우 (이메일 탈취 시 소유 요인 무력화)
- 소유 요인 장치 자체에 지식 요인(PIN 등) 보호가 없어, 장치만 주우면 누구나 쓸 수 있는 경우

- **📢 섹션 요약 비유**: 소유 요인 운영은 회사 사원증 관리와 같다. 사원증을 잃어버렸을 때 임시 출입증을 주는 절차가 허술하면 도둑이 정규직 행세를 할 수 있다.

---

## Ⅴ. 기대효과 및 결론

소유 요인은 "원격지 공격의 종결자"라 불릴 만큼 계정 보안을 획기적으로 높인다. 사용자가 물리적으로 소지해야 한다는 제약 조건이 공격자의 자동화된 봇 공격을 무력화하기 때문이다.

결론적으로, 소유 요인은 현대 보안 아키텍처에서 **신뢰의 앵커(Trust Anchor)** 역할을 한다. 앞으로는 별도의 하드웨어보다는 스마트폰 내부의 보안 영역(Secure Element)을 활용한 소프트웨어적 소유 인증이 주류가 될 것이다.

- **📢 섹션 요약 비유**: 소유 요인은 성문의 '마지막 빗장'이다. 아무리 화려한 말솜씨(지식 요인)로 문지기를 속여도, 빗장을 풀 실물 열쇠가 없으면 성 안으로 들어갈 수 없다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| YubiKey | 대표적인 하드웨어 기반 소유 요인 보안 키 |
| SIM Swapping | 스마트폰 소유 요인(SMS)을 무력화하는 통신사 가로채기 공격 |
| TPM (Trusted Platform Module) | PC 내부에서 소유 요인(암호 키)을 안전하게 저장하는 하드웨어 칩 |
| OTP (One-Time Password) | 소유한 매체에서 매번 다른 값을 생성해 소유를 증명하는 방식 |

### 👶 어린이를 위한 3줄 비유 설명

1. 소유 요인은 내가 주머니 속에 꼭 가지고 있는 '보물 열쇠'나 '내 핸드폰' 같은 거예요.
2. 도둑이 내 이름을 알아도, 내 주머니 속 열쇠는 뺏어가지 못하니까 안전해요.
3. 하지만 열쇠를 잃어버리면 나도 방에 못 들어가니까 잃어버리지 않게 조심해야 해요!
