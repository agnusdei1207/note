## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IAM (Identity and Access Management)은 적절한 개체(인적/비인적)가 적절한 이유로 적절한 시간에 적절한 자원에 접근할 수 있도록 보장하는 프레임워크다.
> 2. **가치**: 신원 식별부터 권한 부여, 감사까지의 전 과정을 통합 관리하여 보안 위협을 최소화하고 운영 효율성을 극대화한다.
> 3. **판단 포인트**: 현대의 IAM은 단순한 계정 관리를 넘어 제로 트러스트 (Zero Trust) 아키텍처의 핵심 엔진으로서 'Identity is the new perimeter'를 실현하는 도구로 진화하고 있다.

---

## Ⅰ. 개요 및 필요성

IAM (Identity and Access Management)은 조직 내의 개별 사용자가 시스템, 네트워크 또는 데이터에 액세스할 수 있는 권한을 정의하고 관리하는 기술 및 정책의 집합이다. 모든 접근 주체의 디지털 신원을 정의하고, 해당 신원이 수행할 수 있는 작업의 범위를 규정함으로써 정보 자산을 보호한다.

비즈니스 환경이 클라우드와 원격 근무로 확장됨에 따라, 전통적인 네트워크 경계 보안은 한계에 직면했다. 이제는 '어디서 접속하느냐'보다 '누가 무엇을 하느냐'가 더 중요해졌으며, IAM은 이러한 변화된 환경에서 자원을 보호하는 가장 강력한 최후의 방어선 역할을 수행한다.

- **📢 섹션 요약 비유**: IAM은 대형 건물의 통합 보안 시스템과 같다. 출입증을 발급하고(신원), 층별 접근 권한을 설정하며(인가), 누가 언제 들어왔는지 기록(감사)하는 모든 과정을 총괄한다.

---

## Ⅱ. 아키텍처 및 핵심 원리

IAM은 크게 식별 (Identification), 인증 (Authentication), 인가 (Authorization), 책임추적성 (Accountability)의 4가지 핵심 프로세스로 구성된다. 최근에는 여기에 거버넌스 (Governance)와 지능형 분석 기능이 추가된 IGA (Identity Governance and Administration) 형태로 발전하고 있다.

| 구성 요소 | 역할 | 핵심 기술/개념 |
|:---|:---|:---|
| Identity Lifecycle | 신원 생성부터 삭제까지 관리 | Provisioning, JML 프로세스 |
| Authentication | 신원 진위 확인 | MFA, SSO, FIDO, Biometrics |
| Authorization | 권한 수준 결정 | RBAC, ABAC, ACL |
| Audit & Reporting | 활동 기록 및 정책 준수 확인 | 로그 분석, SIEM 연동, 접근 검토 |

```text
┌──────────────────────────────────────────────────────────────┐
│                  IAM 프레임워크 논리 아키텍처                   │
├──────────────────────────────────────────────────────────────┤
│ [주체: Subject]  ──▶ [IAM 엔진: Policy Decision Point] ──▶ [자원: Object] │
│ (사용자, 앱, 기기)          │  (판단 및 제어)  │            (서버, DB, API) │
│                          ▼                │                          │
│                ┌───────────────────┐      │      ┌─────────────────┐ │
│                │   Directory / DB  │ ◀────┘      │  Access Control │ │
│                │ (신원 정보 저장소) │             │     (PEP)       │ │
│                └───────────────────┘             └─────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

IAM의 핵심 원리는 '중앙 집중식 통제'와 '정책 기반 자동화'에 있다. 파편화된 접근 제어를 하나로 묶어 관리 복잡도를 낮추고, 사용자의 직무 변화에 따라 권한을 즉시 업데이트함으로써 권한 남용이나 방치를 방지한다.

- **📢 섹션 요약 비유**: IAM은 공항의 출입국 관리소와 같다. 여권 확인(인증), 비자 확인(인가), 출입국 기록(감사)을 통해 국가 보안을 유지하는 것과 같은 원리다.

## Ⅲ. 비교 및 연결

IAM을 이해하기 위해 과거의 개별 계정 관리 (Siloed Identity) 방식과 현대의 통합 관리 방식을 비교할 필요가 있다. 또한, 인가 방식인 RBAC (Role-Based)와 ABAC (Attribute-Based)의 차이도 중요한 비교 포인트다.

| 항목 | Siloed Identity | 현대적 IAM |
|:---|:---|:---|
| 관리 주체 | 시스템별 개별 관리 | 중앙 집중형 통합 관리 |
| 사용자 경험 | 다수의 ID/PW 필요 | SSO를 통한 단일 진입 |
| 보안성 | 일관성 없는 정책, 퇴사자 방치 위험 | 실시간 권한 회수, 지능형 탐지 |
| 확장성 | 신규 시스템 도입 시 부담 큼 | 표준 프로토콜(SAML, OIDC) 연동 |

네트워크 보안 관점에서는 IAM이 제로 트러스트 아키텍처의 PEP (Policy Enforcement Point)와 PDP (Policy Decision Point)를 구성하는 핵심 컴포넌트로 연결된다.

- **📢 섹션 요약 비유**: Siloed Identity는 집집마다 다른 열쇠를 들고 다니는 것이고, IAM은 카드키 하나로 모든 허용된 문을 여는 스마트 빌딩 시스템이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사 관점에서 IAM은 단순 솔루션 도입이 아닌 '거버넌스 체계 수립'으로 접근해야 한다. 특히 클라우드 전환기에는 온프레미스 AD와 클라우드 IAM (Azure AD, AWS IAM 등) 사이의 하이브리드 동기화 및 신뢰 관계 설정이 가장 큰 실무적 도전 과제다.

### 체크리스트
1. **최소 권한 원칙 (PoLP)**: 사용자가 업무에 꼭 필요한 권한만 가지고 있는가?
2. **신속한 Deprovisioning**: 퇴사나 보직 변경 시 권한이 즉시 회수되는가?
3. **강력한 인증**: 모든 관리자 계정에 MFA가 적용되어 있는가?
4. **접근 검토 (Access Review)**: 정기적으로 불필요한 권한을 전수 조사하는가?

### 안티패턴
- 휴면 계정을 방치하여 공격자의 거점으로 제공하는 경우
- 개인 계정이 아닌 공유 계정 (Shared Account)을 사용하여 책임추적성을 상실하는 경우

- **📢 섹션 요약 비유**: IAM 실무는 정원 관리와 같다. 잡초(불필요한 권한)를 제때 뽑아주지 않으면 정원 전체의 보안이 무너진다.

---

## Ⅴ. 기대효과 및 결론

IAM의 도입은 보안성 향상뿐만 아니라 운영 비용 절감과 사용자 생산성 증대라는 다각적인 이점을 제공한다. 자동화된 프로비저닝은 IT 팀의 반복 업무를 줄여주고, 사용자는 하나의 계정으로 필요한 모든 도구에 접근할 수 있어 업무 몰입도가 높아진다.

결론적으로 IAM은 현대 디지털 비즈니스의 신뢰 기반이다. 인공지능 (AI) 기반의 이상 행위 탐지 기술과 결합하여, 단순히 권한을 주고 뺏는 단계를 넘어 실시간 위협에 대응하는 동적 보안 체계로 진화할 것이다.

- **📢 섹션 요약 비유**: IAM은 비즈니스라는 거대한 오케스트라의 지휘자와 같다. 각 연주자가 제자리에 앉아 제 박자에 맞추게 하여 완벽한 화음을 만들어낸다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| SSO (Single Sign-On) | IAM의 사용자 편의성 및 인증 통합 도구 |
| MFA (Multi-Factor Auth) | 인증 단계의 보안 강화를 위한 필수 요소 |
| RBAC / ABAC | IAM 엔진 내에서 권한을 결정하는 논리 체계 |
| JML 프로세스 | 신원 생애 주기를 관리하는 실무 절차 |

### 👶 어린이를 위한 3줄 비유 설명

1. 우리 학교 도서관에 들어갈 때 이름을 확인하고, 빌릴 수 있는 책을 정해주는 선생님이 IAM이에요.
2. IAM 선생님이 없으면 외부인이 몰래 들어오거나, 소중한 책을 누구나 가져가 버릴 수 있어요.
3. IAM은 정해진 사람만 안전하게 도서관을 이용할 수 있게 도와주는 든든한 가드 아저씨랍니다.
