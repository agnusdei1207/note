## 핵심 인사이트 (3줄 요약)
> 1. **본질**: JML 프로세스는 사용자가 조직에 들어오고(Joiner), 이동하고(Mover), 떠나는(Leaver) 생애 주기별 신원 변화에 따른 **접근 권한 관리의 표준 프레임워크**다.
> 2. **가치**: 인사이동에 따른 보안 공백을 제거하고, 직무에 맞는 최적의 권한(Least Privilege)을 지속적으로 유지하게 한다.
> 3. **판단 포인트**: 특히 'Mover(이동)' 단계에서 발생하는 '권한 축적 (Privilege Creep)' 현상을 방지하는 것이 JML 운영의 성패를 가르는 핵심이다.

---

## Ⅰ. 개요 및 필요성

JML 프로세스는 IAM(Identity and Access Management)의 운영적 실체다. 기술적인 도구보다 중요한 것은 "사람이 변할 때 시스템도 변해야 한다"는 정책적 흐름이다. 

- **Joiner (신규 입사)**: 신규 신원을 생성하고 기본 업무 권한을 부여한다. (Day 1 Productivity)
- **Mover (부서 이동/승진)**: 새로운 직무 권한을 부여하고, **이전 직무 권한을 회수**한다.
- **Leaver (퇴사/계약 종료)**: 모든 접근 권한을 즉시 비활성화하여 보안 위험을 제거한다.

이 프로세스가 무너지면 신규 입사자는 일을 못 해서 놀고, 이동한 직원은 예전 부서 비밀까지 다 알고 있으며, 퇴사자는 회사 자료를 집에서 다운로드하는 아수라장이 펼쳐진다.

- **📢 섹션 요약 비유**: JML은 호텔의 체크인, 방 이동, 체크아웃 서비스와 같다. 각 단계에 맞춰 방 키의 권한이 정확히 바뀌어야 호텔 보안이 유지된다.

---

## Ⅱ. 아키텍처 및 핵심 원리

JML은 인사 시스템(HR)의 이벤트가 IAM 시스템을 거쳐 IT 인프라로 흐르는 워크플로우 구조를 가진다.

| 유형 | 트리거 (Trigger) | 핵심 액션 (Action) | 보안 리스크 |
|:---|:---|:---|:---|
| **Joiner** | 채용 확정 | 계정 생성, 기본 앱 할당 | 부적절한 초기 권한 부여 |
| **Mover** | 인사이동/승진 | 권한 변경, **기존 권한 삭제** | 권한 축적 (Privilege Creep) |
| **Leaver** | 퇴사 발령 | 계정 잠금, 데이터 백업 | 고스트 계정 방치 |

```text
┌──────────────────────────────────────────────────────────────┐
│                    JML 프로세스 운영 메커니즘                     │
├──────────────────────────────────────────────────────────────┤
│ [HR System] ───────(Event)───────▶ [IAM Engine]              │
│      │                                  │                    │
│      ├─▶ 입사(Join) ───────────────▶ Provisioning           │
│      ├─▶ 이동(Move) ───────────────▶ Update & Revoke        │
│      └─▶ 퇴사(Leave) ──────────────▶ Deprovisioning         │
│                                         │                    │
│                                         ▼                    │
│                            [AD, ERP, SaaS, Cloud, etc.]      │
└──────────────────────────────────────────────────────────────┘
```

가장 중요한 원리는 **'권한의 동기화'**다. 인사가 '말'이라면 시스템은 '행동'이다. 말이 떨어지자마자 시스템이 움직여야 하며, 특히 Mover 단계에서는 'Birthright(선천적 권한)' 외의 직무별 권한을 철저히 재산정해야 한다.

- **📢 섹션 요약 비유**: JML은 기차역의 환승 시스템과 같다. 표를 사고(Join), 다른 노선으로 갈아타고(Move), 역을 나가는(Leave) 과정이 매끄러워야 기차가 안전하게 운행된다.

## Ⅲ. 비교 및 연결

JML 프로세스를 효과적으로 구현하기 위해서는 **RBAC (역할 기반 접근 제어)**와 **ABAC (속성 기반 접근 제어)** 기술이 밑받침되어야 한다.

| 구분 | JML 기반 (Lifecycle) | 단순 계정 관리 (Static) |
|:---|:---|:---|
| 관리 관점 | 시간 흐름에 따른 동적 관리 | 특정 시점의 정적 상태 관리 |
| 정책 적용 | 직무(Role) 변화에 따른 자동 변경 | 관리자의 수동 개입 필요 |
| 보안 수준 | 상시 최소 권한 유지 가능 | 권한 남용 및 방치 가능성 높음 |
| 주요 도구 | IGA (Identity Governance) 솔루션 | 단순 Admin 콘솔 |

JML은 또한 **'직무 분리 (SoD, Separation of Duties)'** 원칙과 연결된다. 사용자가 이동할 때(Mover), 서로 상충하는 권한(예: 결제 권한과 감사 권한)을 동시에 갖게 되지 않는지 검증하는 필터 역할을 수행한다.

- **📢 섹션 요약 비유**: JML이 '언제' 바꿀지를 정한다면, RBAC은 '무엇으로' 바꿀지를 정하는 기준표다.

## Ⅳ. 실무 적용 및 기술사 판단

실무에서 가장 실패하기 쉬운 지점은 **'Mover'** 단계다. 승진이나 부서 이동 시 "인수인계 때문에 당분간 이전 권한이 필요해요"라는 요구를 거절하지 못해 권한이 계속 쌓이는 현상이 발생한다.

### 체크리스트
1. **Mover 처리 규칙**: 부서 이동 시 기존 권한을 자동으로 삭제(Revoke)하는 로직이 있는가?
2. **비정규직 관리**: 협력사, 인턴 등 비정규직의 계약 만료일이 시스템에 입력되어 자동 종료되는가?
3. **긴급 퇴사 대응**: 징계 해고 등 긴급 퇴사 시 5분 이내에 모든 접근을 차단할 수 있는 'Kill Switch'가 있는가?
4. **대행 결재**: 휴직 시 권한을 잠시 정지하고 대행자에게 넘기는 프로세스가 있는가?

### 안티패턴
- 퇴사 후 한 달 뒤에야 시스템 계정을 정리하는 경우
- 이동한 직원이 예전 부서의 핵심 기밀 문서에 여전히 접근 가능한 경우

- **📢 섹션 요약 비유**: JML 운영은 계단 청소와 같다. 위(Joiner)에서부터 아래(Leaver)까지 매일 쓸고 닦지 않으면 금방 먼지(보안 위협)가 쌓인다.

---

## Ⅴ. 기대효과 및 결론

JML 프로세스의 완성은 조직의 **'규제 준수 (Compliance)'**를 증명하는 가장 확실한 근거다. 외부 감사 시 "우리는 사람이 바뀔 때마다 이런 절차로 권한을 관리합니다"라고 로그를 보여줄 수 있는 능력은 기업 신뢰도의 척도가 된다.

결론적으로 JML은 단순히 ID를 관리하는 기술이 아니라, 조직의 인사 정책과 보안 기술이 만나는 **'거버넌스의 접점'**이다. 클라우드와 협업 도구가 늘어날수록, 이 자동화된 흐름은 선택이 아닌 필수 생존 전략이 될 것이다.

- **📢 섹션 요약 비유**: JML은 나무의 나이테와 같다. 조직이 성장하고 변화하는 모든 과정을 정교하게 기록하고 관리하여 건강한 숲(기업 보안)을 유지하게 한다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| Privilege Creep | Mover 단계에서 권한을 회수하지 않아 권한이 과도하게 쌓이는 현상 |
| Birthright Access | 모든 직원에게 입사와 동시에 자동으로 주어지는 최소 기본 권한 |
| Zero Day Provisioning | 입사 첫날 오전 9시에 모든 업무 준비가 끝나는 상태 |
| IGA (Identity Governance) | JML 프로세스를 고도화하고 감사 기능을 강화한 통합 체계 |

### 👶 어린이를 위한 3줄 비유 설명

1. JML은 유치원에 처음 왔을 때 가방을 주고(J), 반이 바뀌면 새 교실 열쇠를 주고(M), 졸업하면 가방을 돌려받는(L) 약속이에요.
2. 만약 반이 바뀌었는데 옛날 교실 열쇠를 안 돌려주면, 동생들 공부하는 곳에 가서 장난을 칠 수도 있겠죠?
3. 모두가 자기 자리에 맞는 물건만 가질 수 있게 도와주는 아주 공평한 규칙이랍니다.
