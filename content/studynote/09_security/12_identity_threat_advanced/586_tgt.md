## 핵심 인사이트 (3줄 요약)
> 1. **본질**: TGT (Ticket Granting Ticket)는 사용자가 최초 인증 성공 후 KDC로부터 받는 "티켓 발급용 티켓"으로, 이후 개별 서비스 티켓을 요청할 때 자신의 신원을 증명하는 수단이다.
> 2. **가치**: 한 번 발급받으면 유효 기간(기본 10시간) 동안 비밀번호 재입력 없이도 다양한 서비스 티켓을 획득할 수 있게 하여 SSO(Single Sign-On)를 가능하게 한다.
> 3. **판단 포인트**: TGT는 KDC의 마스터 키(KRBTGT)로 암호화되어 있어 클라이언트는 내용을 수정할 수 없으며, 탈취될 경우 해당 사용자로 위장한 접근이 가능해지므로 메모리 보호가 중요하다.

---

## Ⅰ. 개요 및 필요성

TGT는 Kerberos 인증 체계에서 사용자의 세션을 유지하는 가장 핵심적인 증명서다. 사용자가 아침에 출근하여 로그인을 한 번 하면 퇴근할 때까지 다시 비밀번호를 묻지 않는 이유는 바로 이 TGT를 메모리에 가지고 있기 때문이다.

TGT가 없다면 사용자는 파일 서버에 접근할 때 한 번, 이메일을 확인할 때 또 한 번 비밀번호를 입력해야 한다. TGT는 "이 사용자는 이미 AS(Authentication Server)를 통해 신원이 검증되었다"는 사실을 암호화하여 담고 있으며, 이를 TGS(Ticket Granting Server)에 제시함으로써 개별 서비스 티켓을 즉시 발급받을 수 있다.

- **📢 섹션 요약 비유**: TGT는 놀이공원 입구에서 받은 '자유이용권 손목띠'와 같다. 이것만 있으면 개별 놀이기구를 탈 때마다 매표소에 가서 신분증을 보여줄 필요가 없다.

---

## Ⅱ. 아키텍처 및 핵심 원리

TGT는 AS_REP 단계에서 발급되며, TGS_REQ 단계에서 사용된다.

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│                    TGT의 생성 및 사용 흐름 (Kerberos 6-Step)                 │
├──────────────────────────────────────────────────────────────────────────────┤
│  [Client]              [KDC: AS]            [KDC: TGS]          [Server]     │
│      │                     │                     │                  │        │
│      │── 1. AS_REQ ───────▶│                     │                  │        │
│      │◀─ 2. AS_REP (TGT) ──│                     │                  │        │
│      │      (KRBTGT Key로 암호화됨)              │                  │        │
│      │                     │                     │                  │        │
│      │── 3. TGS_REQ (TGT 제시) ─────────────────▶│                  │        │
│      │◀─ 4. TGS_REP (ST 발급) ──────────────────│                  │        │
└──────────────────────────────────────────────────────────────────────────────┘
```

### TGT의 특징
- **암호화 주체**: KDC의 `KRBTGT` 계정 비밀번호(비밀키)로 암호화된다.
- **내용물**: 사용자의 ID, 네트워크 주소, 유효 기간, 그리고 TGS와 사용할 **로그인 세션 키**가 들어 있다.
- **클라이언트의 역할**: 클라이언트는 TGT를 해독할 수 없다(KRBTGT 키가 없으므로). 단지 TGS에게 전달하는 '암호화된 봉투'로만 취급한다.

- **📢 섹션 요약 비유**: TGT는 보안관이 꽉 밀봉한 봉투와 같다. 나는 그 안을 볼 수 없지만, 다른 가게 주인에게 보여주면 보안관의 인장을 확인하고 나를 믿어준다.

---

## Ⅲ. 비교 및 연결

TGT는 이후 단계에서 발행되는 ST(Service Ticket)와 비교하여 그 성격을 이해해야 한다.

| 항목 | TGT (Ticket Granting Ticket) | ST (Service Ticket) |
|:---|:---|:---|
| **발급 주체** | KDC 내의 AS (인증 서버) | KDC 내의 TGS (티켓 서버) |
| **암호화 키** | KRBTGT 계정의 비밀키 | 대상 서비스 서버의 비밀키 |
| **용도** | 다른 서비스 티켓을 받기 위한 용도 | 특정 서비스(파일, 웹 등) 접근용 |
| **유효 기간** | 상대적 장기 (보통 10시간) | 상대적 단기 (보통 1시간) |

공격자가 TGT를 메모리에서 추출하여 자신의 세션에 주입하는 것을 'Pass-the-Ticket (PtT)' 공격이라고 한다. 만약 공격자가 KRBTGT 키 자체를 획득하여 가짜 TGT를 직접 만들면 그것이 바로 'Golden Ticket' 공격이다.

- **📢 섹션 요약 비유**: TGT가 '백화점 VIP 카드'라면, ST는 그 카드를 제시하고 받은 '특정 매장 입장권'이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사적 관점에서 TGT 관리는 **세션 탈취 방어**와 **KRBTGT 보호**로 귀결된다. TGT는 클라이언트 PC의 LSASS(Local Security Authority Subsystem Service) 메모리에 저장되는데, 이곳이 뚫리면 TGT가 유출된다.

### 운영 및 보안 전략
1. **유효 기간 정책**: TGT의 유효 기간(Maximum ticket lifetime)과 갱신 기간(Maximum renew lifetime)을 조직의 보안 등급에 맞춰 설정해야 한다.
2. **Credential Guard**: 윈도우의 가상화 기반 보안(VBS)을 사용하여 LSASS 메모리를 격리, TGT 추출을 방지해야 한다.
3. **KRBTGT 비밀번호 주기적 변경**: Golden Ticket 공격의 유효성을 끊기 위해 도메인 내 KRBTGT 계정의 비밀번호를 최소 1년에 2회 이상 변경할 것을 권고한다.

- **📢 섹션 요약 비유**: 자유이용권(TGT)이 너무 길면 잃어버렸을 때 위험하므로 적당한 시간을 정해야 한다. 또한 이용권을 발행하는 기계의 비밀번호(KRBTGT)도 자주 바꿔야 한다.

---

## Ⅴ. 기대효과 및 결론

TGT는 Kerberos 프로토콜이 대규모 네트워크에서 효율적으로 동작하게 만드는 일등 공신이다. 사용자에게는 끊김 없는 경험(SSO)을 제공하고, 시스템에는 강력한 중앙 집중형 인증 구조를 제공한다.

결론적으로 TGT는 단순한 데이터 조각이 아니라, 사용자의 디지털 신원을 일정 기간 보증하는 '신뢰의 위임장'이다. 따라서 이를 보호하기 위한 메모리 보안 기술과 마스터 키 관리는 현대 기업 보안의 핵심 과제다.

- **📢 섹션 요약 비유**: TGT는 한 번의 확인으로 하루 종일 건물을 출입하게 해주는 만능 패스와 같다. 이 패스를 잃어버리지 않게 주머니(메모리)를 잘 단속해야 한다.

### 📌 관련 개념 맵
| 개념 | 연결 포인트 |
|:---|:---|
| **KRBTGT** | TGT를 암호화하고 서명하는 데 사용되는 KDC의 핵심 계정 |
| **AS (Authentication Server)** | 사용자의 비밀번호를 확인하고 TGT를 처음 발행해주는 곳 |
| **Pass-the-Ticket (PtT)** | 유효한 TGT를 훔쳐서 다른 컴퓨터에서 사용하는 공격 기법 |

### 👶 어린이를 위한 3줄 비유 설명
1. TGT는 놀이공원 입구에서 "이 친구는 우리 유치원생 맞아요"라고 선생님이 써준 특별한 쪽지예요.
2. 이 쪽지(TGT)만 주머니에 있으면, 공원 안의 어떤 놀이기구라도 표로 바꿀 수 있어요.
3. 하지만 이 쪽지를 다른 친구에게 뺏기면, 그 친구가 나인 척하고 놀이기구를 탈 수 있으니 조심해야 해요.
