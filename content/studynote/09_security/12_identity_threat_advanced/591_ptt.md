## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Pass-the-Ticket (PtT)은 공격자가 사용자의 비밀번호를 몰라도, 메모리(LSASS)에서 추출한 유효한 Kerberos 티켓(TGT/ST)을 자신의 세션에 주입하여 신원을 도용하는 공격이다.
> 2. **가치**: 비밀번호를 크래킹하거나 알아낼 필요 없이 이미 인증된 '결과물'을 재사용하므로, 강력한 비밀번호 정책을 우회할 수 있는 매우 효율적인 공격 수단이다.
> 3. **판단 포인트**: 티켓의 유효 기간 동안만 유효하며, 이를 방어하기 위해 Credential Guard와 같은 메모리 격리 기술과 정기적인 세션 강제 종료 정책이 필요하다.

---

## Ⅰ. 개요 및 필요성

Pass-the-Ticket (PtT)은 Kerberos 인증의 '결과 중심적' 특성을 악용한 공격이다. Kerberos는 일단 티켓을 받으면 그 티켓이 본인임을 증명하는 최종 수단이 된다. 공격자가 사용자 PC의 메모리를 덤프하여 이 티켓(TGT 또는 ST)을 손에 넣는다면, 네트워크상에서 해당 사용자와 동일한 권한을 행사할 수 있다.

이 공격은 특히 비밀번호가 매우 길고 복잡하여 해시를 깨기 힘들 때 강력한 대안이 된다. 공격자는 비밀번호 자체에는 관심이 없으며, 오직 메모리에 상주하는 '인증된 상태'인 티켓만을 목표로 한다.

- **📢 섹션 요약 비유**: PtT는 집 주인의 열쇠를 복사하는 것이 아니라, 주인이 현관문을 열어두고 잠시 자리를 비운 사이 몰래 들어가서 "내가 주인이다"라고 주장하는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

PtT 공격은 Kerberos 흐름에서 AS/TGS 단계를 거쳐 생성된 티켓을 가로채는 방식이다.

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│                  Pass-the-Ticket (PtT) 공격 개념도                           │
├──────────────────────────────────────────────────────────────────────────────┤
│  [Victim PC]              [Attacker PC]               [Server]               │
│      │                           │                       │                   │
│ (1. 티켓 생성: TGT)               │                       │                   │
│ [Memory: LSASS] ◀─ 2. 티켓 추출 ─┤                       │                   │
│      │                           │                       │                   │
│      │                    (3. 티켓 주입)                 │                   │
│      │                           │── 4. AP_REQ (ST) ──▶ │                   │
│      │                           │◀─ 5. Access Granted ─│ (티켓이 맞으니 통과)│
└──────────────────────────────────────────────────────────────────────────────┘
```

### 공격 단계
1. **티켓 추출**: 공격자는 피해자 컴퓨터에서 관리자 권한을 획득한 후, `Mimikatz` 등을 이용해 LSASS 프로세스 메모리에서 유효한 TGT나 ST를 파일(`.kirbi`)로 내보낸다.
2. **티켓 주입**: 공격자는 자신의 컴퓨터로 이 티켓 파일을 가져온 후, 메모리에 다시 로드(pass-the-ticket)한다.
3. **리소스 접근**: 공격자의 운영체제는 이제 메모리에 있는 티켓을 사용하여 대상 서버에 접근을 시도하며, 서버는 티켓이 유효하므로 접근을 허용한다.

- **📢 섹션 요약 비유**: 놀이공원에서 누군가 잃어버린 '자유이용권 손목띠'를 주워서 내 팔에 차고 돌아다니는 것과 같다. 검표원은 손목띠만 보고 나를 입장시켜 준다.

## Ⅲ. 비교 및 연결

PtT는 유사한 공격 기법인 Pass-the-Hash (PtH)와 자주 비교된다.

| 항목 | Pass-the-Ticket (PtT) | Pass-the-Hash (PtH) |
|:---|:---|:---|
| **인증 프로토콜** | Kerberos | NTLM |
| **탈취 대상** | 티켓 (.kirbi 파일 / 메모리 데이터) | NTLM 해시 (MD4) |
| **비밀번호 필요성** | 불필요 | 불필요 |
| **유효 범위** | 티켓 유효 기간 내 (장기 TGT 가능) | 비밀번호 변경 전까지 영구적 |
| **방어 기술** | Credential Guard, 짧은 티켓 TTL | Restricted Admin 모드, PtH 패치 |

PtT는 NTLM 인증이 비활성화된 환경에서도 작동하므로, 현대적인 Active Directory 보안 환경에서 더 위협적이다.

- **📢 섹션 요약 비유**: PtH가 '열쇠의 복사본(해시)'을 들고 다니는 것이라면, PtT는 '이미 승인된 출입증(티켓)'을 들고 다니는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사적 관점에서 PtT 방어는 **"메모리 보호"**와 **"티켓 수명 관리"**에 집중되어야 한다.

### 실무 방어 전략
1. **Windows Credential Guard**: 가상화 기반 보안(VBS)을 사용하여 LSASS를 격리된 컨테이너에서 실행함으로써, 공격자가 메모리를 덤프해도 티켓을 가져가지 못하게 원천 차단한다.
2. **Protected Users 그룹**: 중요한 관리자 계정을 'Protected Users' 그룹에 포함시켜 Kerberos 티켓의 유효 기간을 강제로 짧게 제한하고 취약한 암호화 알고리즘 사용을 금지한다.
3. **네트워크 격리 및 측면 이동 차단**: 공격자가 피해자 PC에 관리자 권한으로 접근하는 것 자체가 PtT의 시작이므로, 엔드포인트 하드닝(Hardening)과 RDP 제한이 필수적이다.

- **📢 섹션 요약 비유**: 손목띠(티켓)를 아무나 풀어갈 수 없게 강력한 접착제를 바르고(Credential Guard), 손목띠의 색깔을 자주 바꿔서(짧은 유효 기간) 훔친 띠를 오래 못 쓰게 하는 전략이다.

---

## Ⅴ. 기대효과 및 결론

Pass-the-Ticket은 Kerberos의 편의성(SSO)을 역이용하는 치명적인 공격이다. 비밀번호를 아무리 길게 설정해도 메모리 관리가 소홀하면 아무런 소용이 없다는 것을 잘 보여준다.

결론적으로 PtT 공격에 대한 완벽한 방어는 존재하지 않으며, '심층 방어(Defense in Depth)' 관점에서 엔드포인트 보안, 메모리 격리, 그리고 이상 징후 모니터링이 유기적으로 결합되어야만 위협을 최소화할 수 있다.

- **📢 섹션 요약 비유**: 지갑(메모리)을 잃어버리면 안의 현금(티켓)을 누구나 쓸 수 있는 것과 같다. 지갑을 몸에 꽉 붙여두거나, 현금의 유통 기한을 짧게 만드는 노력이 필요하다.
