## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 미미카츠 (Mimikatz)는 Windows 메모리에서 평문 비밀번호, 해시, PIN, Kerberos 티켓 등을 추출하기 위해 개발된 강력한 오픈소스 자격 증명 탈취 도구다.
> 2. **가치**: 보안 연구자에게는 인증 아키텍처의 취약성을 증명하는 도구이지만, 공격자에게는 횡적 이동(Lateral Movement)을 위한 필수적인 공격 무기로 활용된다.
> 3. **판단 포인트**: 도구 자체의 시그니처 차단보다는 LSASS 보호, 디버그 권한 제한, 원격 자격 증명 보호 등 윈도우 인증 구조의 강화가 근본적인 해결책이다.

---

## Ⅰ. 개요 및 필요성

미미카츠는 벤자민 델피(Benjamin Delpy)가 Windows 인증 메커니즘의 취약점을 보여주기 위해 C 언어로 개발한 도구다. 단순한 해킹 툴을 넘어, Windows의 자격 증명 관리 시스템인 **LSA (Local Security Authority)**가 메모리에 기밀 정보를 어떻게 유지하는지 적나라하게 드러냄으로써 전 세계 보안 설계에 막대한 영향을 끼쳤다.

이 도구가 등장하기 전까지는 "비밀번호는 암호화되어 저장되므로 안전하다"는 인식이 강했으나, 미미카츠는 관리자 권한만 있다면 메모리를 직접 읽어 평문 비밀번호를 복구할 수 있음을 증명했다. 이는 현대적인 엔드포인트 보안(EDR)과 Windows의 하드웨어 기반 보안(Credential Guard) 발전을 가속화시킨 계기가 되었다.

```text
┌──────────────────────────────────────────────────────────────┐
│             미미카츠의 위상: "Key to the Kingdom"            │
├──────────────────────────────────────────────────────────────┤
│ [인증 취약점 발견] ──▶ [미미카츠 활용] ──▶ [전역 권한 장악]  │
│          │                                        │          │
│          └─▶ [Windows 보안 아키텍처의 대대적 혁신] ◀─┘          │
└──────────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: 미미카츠는 건물의 모든 자물쇠를 무력화할 수 있는 만능 열쇠와 같다. 이 열쇠의 존재가 알려지면서 사람들은 더 튼튼하고 새로운 방식의 자물쇠를 발명하게 되었다.

## Ⅱ. 아키텍처 및 핵심 원리

미미카츠는 주로 `sekurlsa`, `crypto`, `kerberos`, `privilege` 등 다양한 모듈로 구성되어 있으며, 윈도우 내부 API를 직접 호출하여 작동한다.

| 주요 모듈 | 기능 설명 | 주요 대상 |
|:---|:---|:---|
| **sekurlsa** | LSASS 메모리를 덤프하고 자격 증명 추출 | NTLM 해시, 평문 비밀번호, 티켓 |
| **kerberos** | Kerberos 티켓 조작 및 생성 | Golden/Silver Ticket, PTT |
| **crypto** | 인증서 및 개인키 추출 | 암호화된 기밀 정보, 스마트카드 PIN |
| **lsadump** | SAM, LSA Secrets, NTDS 데이터베이스 덤프 | 로컬 및 도메인 해시 정보 |

### 미미카츠 동작 흐름 (Pass-the-Hash 예시)

미미카츠는 공격자가 획득한 관리자 권한을 바탕으로 `SeDebugPrivilege`를 활성화한 후, 다른 프로세스(주로 LSASS)의 메모리 영역에 접근한다.

```text
┌──────────────────────────────────────────────────────────────┐
│           미미카츠를 이용한 공격 시나리오 (PtH)              │
├──────────────────────────────────────────────────────────────┤
│ (1) Privilege::debug   => 프로세스 메모리 접근 권한 획득     │
│          │                                                   │
│ (2) Sekurlsa::logonpasswords => LSASS 메모리 분석            │
│          │                                                   │
│ (3) NTLM Hash 획득     => (예: 59b6...e782)                  │
│          │                                                   │
│ (4) Sekurlsa::pth      => 해시를 이용한 가짜 토큰 생성       │
│          │                                                   │
│ (5) Remote Access      => 다른 서버로 관리자 권한 접속       │
└──────────────────────────────────────────────────────────────┘
```

특히 WDigest나 SSP 등의 레거시 프로토콜이 활성화되어 있을 경우, 메모리 상에 평문 비밀번호가 직접 노출되는 취약점을 즉각적으로 악용한다.

- **📢 섹션 요약 비유**: 미미카츠는 금고를 부수는 망치가 아니라, 금고 주인의 뇌(메모리)를 스캔해서 비밀번호를 직접 알아내는 첨단 장비와 같다.

## Ⅲ. 비교 및 연결

미미카츠는 단독 도구로도 강력하지만, 다른 공격 기법 및 도구들과 연결되어 '킬 체인'을 완성한다.

| 비교 대상 | 미미카츠 (Mimikatz) | Rubeus |
|:---|:---|:---|
| **주요 언어** | C (네이티브 실행) | C# (.NET 기반 실행) |
| **특장점** | LSASS 직접 덤프, 다양한 모듈 | 파일 없이 메모리 상에서 실행(Fileless) 유리 |
| **주력 분야** | 로컬 자격 증명 추출 및 조작 | Kerberos 전용 공격 (AS-REP Roasting 등) |
| **연동성** | Metasploit, Cobalt Strike 탑재 | C# 기반 포스트 익스플로잇 도구와 연동 |

최근에는 미미카츠의 시그니처가 백신(AV)에 의해 쉽게 탐지되므로, 메모리 상에서만 실행되는 파워셸 스크립트 형태(Invoke-Mimikatz)나 탐지를 우회하도록 수정한 변종들이 주로 사용된다.

- **📢 섹션 요약 비유**: 미미카츠가 묵직한 맥가이버 칼이라면, Rubeus는 Kerberos라는 특정 나사만 전문적으로 푸는 전동 드라이버다.

## Ⅳ. 실무 적용 및 기술사 판단

실무적으로 미미카츠를 막는 것은 '숨바꼭질'과 같다. 실행 파일을 차단해도 공격자는 다른 방식을 찾아내기 때문이다. 따라서 시스템 설정을 통해 미미카츠가 작동할 수 있는 '환경'을 없애야 한다.

### 방어 강화 전략

1. **디버그 권한 제한**: 로컬 보안 정책에서 '프로그램 디버그' 권한(SeDebugPrivilege)을 관리자 그룹에서 제거하여 LSASS 접근을 어렵게 만든다.
2. **WDigest 비활성화**: 레지스트리 설정을 통해 평문 비밀번호가 메모리에 남지 않도록 강제한다. (Windows 8.1 이상 기본 적용)
3. **PPL (Protected Process Light)**: LSASS를 보호된 프로세스로 설정하여 관리자라 하더라도 직접적인 메모리 읽기를 차단한다.
4. **Credential Guard**: 가상화 기술을 이용하여 핵심 자격 증명을 별도의 안전한 컨테이너(LSAIso)에 격리한다.

### 기술사 판단 포인트

- 미미카츠 탐지는 파일 시그니처가 아니라 **'동작(Behavior)'**에 집중해야 한다. 비정상적인 프로세스가 LSASS에 핸들을 오픈하거나, `MiniDumpWriteDump` API를 호출하는 행위를 모니터링하는 것이 EDR의 핵심 역량이다.

- **📢 섹션 요약 비유**: 도둑이 든 연장(파일)을 검사하는 것보다, 도둑이 담을 넘거나 자물쇠를 만지는 행위(동작)를 감시 카메라로 포착하는 것이 더 효과적이다.

## Ⅴ. 기대효과 및 결론

미미카츠는 역설적으로 Windows 보안을 가장 강력하게 만든 도구다. 미미카츠에 의해 뚫린 수많은 구멍을 메우는 과정에서 Microsoft는 자격 증명 보호 아키텍처를 근본적으로 재설계했기 때문이다.

앞으로의 인증 보안은 '비밀번호' 중심에서 '티켓'과 '인증서' 중심으로, 다시 '격리된 하드웨어' 중심으로 이동할 것이다. 미미카츠는 그 진화의 과정에서 우리가 잊지 말아야 할 강력한 경고이자, 시스템의 가장 약한 고리를 찾아내는 영리한 나침반으로 기억될 것이다.

- **📢 섹션 요약 비유**: 독을 알아야 해독제를 만들 수 있듯이, 미미카츠라는 강력한 공격 도구에 대한 이해는 강력한 방어 체계 구축의 시작점이다.
