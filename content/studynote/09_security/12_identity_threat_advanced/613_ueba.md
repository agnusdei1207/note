## 핵심 인사이트 (3줄 요약)
> 1. **본질**: UEBA (User Entity Behavior Analytics)는 사용자(User)와 기기(Entity)의 평소 행동 패턴을 학습하여, 정상 범위를 벗어나는 **이상 행위(Anomaly)**를 탐지하는 분석 기술이다.
> 2. **가치**: 단순한 규칙(Rule) 기반 탐지가 놓치는 내부자 위협(Insider Threat)이나 정교한 계정 탈취 공격을 행동의 변화(Baseline deviation)를 통해 포착한다.
> 3. **판단 포인트**: '무엇을 했는가'보다 '평소와 다르게 무엇을 했는가'에 집중하여, 오탐(False Positive)을 줄이고 정교한 위협을 식별하는 것이 핵심 역량이다.

---

## Ⅰ. 개요 및 필요성

전통적인 보안 시스템(SIEM, IDS 등)은 "A라는 행위가 발생하면 경고를 울려라"는 식의 명시적인 규칙에 의존한다. 하지만 공격자가 정상적인 계정 권한을 획득하여 정상적인 도구를 사용한다면 규칙 기반 시스템은 무력화된다. **UEBA**는 이러한 한계를 극복하기 위해 '행동' 그 자체를 분석의 대상으로 삼는다.

UEBA가 필요한 이유는 현대의 위협이 매우 은밀하기 때문이다. 평소 오후 6시에 퇴근하던 직원이 갑자기 새벽 3시에 접속하여 대량의 데이터를 다운로드한다면, 이는 권한상으로는 문제가 없어도 행동상으로는 매우 의심스러운 신호다. UEBA는 이러한 미세한 변화를 감지하여 보안 사고를 미연에 방지한다.

```text
┌──────────────────────────────────────────────────────────────┐
│             UEBA의 핵심 철학: "정상의 정의와 이탈"           │
├──────────────────────────────────────────────────────────────┤
│ [데이터 수집] ──▶ [기준선(Baseline) 수립] ──▶ [편차 분석]    │
│ (로그, 행동,       (평소 습관 학습)           (이상 징후 포착)│
│  접속 시간 등)                                               │
└──────────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: UEBA는 우리 동네를 잘 아는 베테랑 보안관과 같다. 낯선 사람이 나타나서가 아니라, 늘 보던 이웃이 평소와 다르게 가방을 숨기며 걷는 모습에서 위험을 직감하는 방식이다.

## Ⅱ. 아키텍처 및 핵심 원리

UEBA는 머신러닝 알고리즘을 사용하여 사용자 및 엔티티(기기, 애플리케이션 등)의 프로필을 생성한다.

### UEBA 분석 단계

| 단계 | 역할 | 주요 기술 |
|:---|:---|:---|
| **데이터 인입** | 다양한 보안 로그 및 컨텍스트 수집 | SIEM 연동, API 수집 |
| **프로파일링** | 개별 주체별 '정상 행동' 정의 | 베이지안 통계, 클러스터링 |
| **점수 산정** | 이상 행위별 위험도 수치화 (Scoring) | 가중치 부여 알고리즘 |
| **경보 및 대응** | 임계치 초과 시 알림 및 자동 조치 | 조건부 액세스 연동 |

### 행동 분석 모델 (Behavioral Modeling)

UEBA는 단순히 한 가지 사건이 아니라, 여러 사건의 연관성을 분석하여 위험 점수를 쌓아간다.

```text
┌──────────────────────────────────────────────────────────────┐
│               UEBA 리스크 점수 누적 시나리오                 │
├──────────────────────────────────────────────────────────────┤
│ [사건 1] 평소와 다른 시간 로그인 (+10점)                     │
│    │                                                         │
│ [사건 2] 한 번도 안 가본 폴더 접근 (+30점)                   │
│    │                                                         │
│ [사건 3] 대량의 데이터 외부 전송 (+50점)                     │
│    │                                                         │
│ [결과] 총점 90점 => "심각한 위협" 판단 => 계정 즉시 잠금     │
└──────────────────────────────────────────────────────────────┘
```

이처럼 UEBA는 점진적으로 고조되는 공격(Slow and Low attack)을 탐지하는 데 탁월한 성능을 발휘한다.

- **📢 섹션 요약 비유**: UEBA는 건강검진과 같다. 평소 혈압(기준선)을 알기에, 조금만 수치가 올라가도(이상 행위) 몸에 이상이 생겼음을 바로 알아차릴 수 있다.

## Ⅲ. 비교 및 연결

UEBA는 기존 SIEM(Security Information and Event Management)의 진화된 형태 또는 보완재로 평가받는다.

| 비교 항목 | 전통적 SIEM | UEBA |
|:---|:---|:---|
| **분석 중심** | 이벤트 및 로그 데이터 | 사용자 및 기기 행위 |
| **탐지 방식** | 상관관계 규칙 (Rule-based) | 알고리즘 및 통계 (ML-based) |
| **주요 타겟** | 외부 침입 탐지 | 내부자 위협, 계정 사칭 탐지 |
| **결과물** | 개별 알람 (Alert) | 위협 시나리오 및 리스크 점수 |

최근에는 SIEM 내부에 UEBA 기능이 통합되는 추세이며, 이를 통해 '무엇이 일어났는가(SIEM)'와 '누가 왜 그랬는가(UEBA)'를 동시에 파악할 수 있게 되었다.

- **📢 섹션 요약 비유**: SIEM이 "누군가 담을 넘었다"는 경보음이라면, UEBA는 "저 사람은 우리 집 담을 넘을 확률이 높은 행동을 해왔다"는 예측 보고서다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 UEBA 도입 시 초기 학습 기간과 오탐 관리에 대한 전략을 세워야 한다.

### 실무 가이드라인 (체크리스트)

1. **학습 기간 확보**: 정확한 기준선 수립을 위해 최소 2~4주간의 데이터 학습 기간이 필요함을 사용자에게 안내한다.
2. **피어 그룹(Peer Group) 분석**: 동일 부서원들과의 행동 비교를 통해, 부서 전체의 공통된 특이 행위(예: 마감 기한 야근)가 오탐으로 처리되지 않도록 설정한다.
3. **데이터 소스 다양화**: 인증 로그뿐만 아니라 파일 접근, 이메일 전송, 엔드포인트 프로세스 로그 등을 최대한 연동하여 분석의 정밀도를 높인다.
4. **대응 자동화**: 고위험 점수 발생 시 보안 관리자 확인 전에 MFA를 강제하거나 세션을 종료하는 소아(SOAR) 연동을 고려한다.

### 기술사적 판단

- UEBA의 최대 적은 **"알람 피로도(Alert Fatigue)"**다. 모든 이상 징후를 보고하면 운영팀은 마비된다. 따라서 기술사는 단순히 이상함을 찾는 것이 아니라, **"비즈니스 위험과 직결된 이상함"**에 더 높은 가중치를 두도록 알고리즘을 튜닝하는 '보안 거버넌스' 관점의 접근을 강조해야 한다.

- **📢 섹션 요약 비유**: 모든 기침 소리에 반응하는 의사보다, 폐렴의 징후가 보이는 기침 소리를 정확히 가려내는 의사가 유능한 의사다.

## Ⅴ. 기대효과 및 결론

UEBA는 보안의 패러다임을 '방어'에서 '관찰과 예측'으로 바꾼다. 정적인 규칙 뒤에 숨어있던 은밀한 공격자들과 내부의 악의적인 행위자들은 이제 그들의 '행동'을 통해 정체가 드러나게 된다.

결론적으로 UEBA는 지능형 보안 운영(SecOps)의 핵심 엔진이다. 데이터가 늘어나고 공격이 정교해질수록, 행동의 맥락을 읽어내는 UEBA의 중요성은 더욱 커질 것이다.

- **📢 섹션 요약 비유**: 행동은 거짓말을 하지 않는다. UEBA는 시스템 속에 숨겨진 진실을 읽어내는 거짓말 탐지기다.
