## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 조건부 액세스 신호 (Signals)는 사용자 접근 시점에 수집되는 **문맥 정보(Contextual Information)**로, 보안 엔진이 판단을 내리기 위한 핵심 데이터셋이다.
> 2. **가치**: 단일 요소(비밀번호)만 보는 것이 아니라 사용자, 기기, 위치, 앱, 리스크 등 다차원 데이터를 결합하여 접근의 정당성을 입증한다.
> 3. **판단 포인트**: 신호의 신뢰도(예: 관리 기기 여부)에 따라 접근 허용 범위를 동적으로 조절하는 것이 지능형 보안 제어의 핵심이다.

---

## Ⅰ. 개요 및 필요성

조건부 액세스 엔진은 스스로 판단할 수 있는 지능이 없다. 엔진이 올바른 결정을 내리기 위해서는 현재 접근 시도의 상황을 설명해주는 정확한 **신호(Signals)**가 입력되어야 한다. 신호는 접근을 시도하는 주체뿐만 아니라, 그 주체가 처한 환경적 요인까지 모두 포함한다.

신호 기반 접근 제어가 필요한 이유는 '훔친 아이디와 비밀번호'를 걸러내기 위해서다. 공격자가 올바른 자격 증명을 가졌더라도, 평소와 다른 위치(신호)나 회사에서 지급하지 않은 기기(신호)를 사용한다면 엔진은 이를 즉시 비정상으로 간주하고 차단하거나 추가 인증을 요구할 수 있다.

```text
┌──────────────────────────────────────────────────────────────┐
│           신호의 역할: "판단을 위한 다각도 증거"            │
├──────────────────────────────────────────────────────────────┤
│ [접근 시도] ──▶ [신호 1: 사용자] ──▶ [신호 2: 기기] ──▶ [판단]│
│                [신호 3: 위치]   ──▶ [신호 4: 리스크]         │
└──────────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: 신호는 입국 심사관이 확인하는 여권, 비자, 행선지, 가방 내용물과 같다. 어느 하나라도 미심쩍으면(신호 이상) 입국이 거절된다.

---

## Ⅱ. 아키텍처 및 핵심 원리

조건부 액세스에서 활용되는 신호는 크게 5가지 카테고리로 분류된다.

### 5대 핵심 신호 카테고리

| 카테고리 | 주요 신호 항목 | 설명 |
|:---|:---|:---|
| **사용자 (User)** | 그룹 멤버십, 역할(Admin) | 접근 주체가 누구이며 어떤 권한을 가졌는가? |
| **위치 (Location)** | IP 대역, 국가, 신뢰된 지점 | 신뢰할 수 있는 네트워크 또는 지역에서 오는가? |
| **기기 (Device)** | 하이브리드 가입, Intune 준수 | 회사 관리 기기인가? 보안 패치는 최신인가? |
| **애플리케이션** | 클라우드 앱, 레거시 앱 | 어떤 데이터나 서비스에 접근하려고 하는가? |
| **리스크 (Risk)** | 로그인 위험, 사용자 위험 | 위협 인텔리전스 기반의 실시간 위험 점수는? |

### 신호 조합 및 엔진 판단 흐름

```text
┌──────────────────────────────────────────────────────────────┐
│            신호 결합을 통한 지능적 의사결정 예시             │
├──────────────────────────────────────────────────────────────┤
│ [신호 입력]                               [엔진 판단]        │
│ 1. 사용자: 도메인 관리자                      │             │
│ 2. 위치  : 해외 IP (평소와 다름)              │             │
│ 3. 기기  : 개인 노트북 (관리 안 됨)           │             │
│ 4. 리스크: Medium (자격 증명 유출 의심)       │             │
│          │                                    │             │
│          └──────────────────────────────────▶ [ 접근 즉시 차단 ] │
│                                                             │
│ [신호 입력]                                                 │
│ 1. 사용자: 일반 직원                          │             │
│ 2. 위치  : 사내 망 (신뢰된 위치)              │             │
│ 3. 기기  : 회사 PC (Intune 준수)              │             │
│ 4. 리스크: Low (특이사항 없음)                │             │
│          │                                    │             │
│          └──────────────────────────────────▶ [ MFA 없이 허용 ] │
└──────────────────────────────────────────────────────────────┘
```

이처럼 신호의 조합은 "검은색(차단)"과 "흰색(허용)"뿐만 아니라 "회색(추가 인증 후 허용)"의 영역을 만들어낸다.

- **📢 섹션 요약 비유**: 신호는 요리의 재료와 같다. 신선한 재료(안전한 신호)가 모이면 맛있는 요리(허용)가 되지만, 상한 재료(리스크 신호)가 하나라도 섞이면 버려야(차단) 한다.

## Ⅲ. 비교 및 연결

신호는 정적인 신호와 동적인 신호로 구분하여 이해하면 실무 적용이 쉽다.

| 구분 | 정적 신호 (Static Signals) | 동적 신호 (Dynamic Signals) |
|:---|:---|:---|
| **특징** | 고정된 값이나 속성 기반 | 실시간 상태나 행동 분석 기반 |
| **예시** | IP 주소, 그룹 이름, 앱 이름 | 기기 준수 상태, 실시간 리스크 점수 |
| **보안성** | 중간 (우회 가능성 있음) | 높음 (공격자가 조작하기 어려움) |
| **관리 부담** | 낮음 | 중간 (모니터링 도구 연동 필요) |

신호는 **UEBA (User Entity Behavior Analytics)**와 연결된다. UEBA가 사용자의 평소 행동 패턴을 학습하여 '이상 신호'를 생성하면, 조건부 액세스가 이를 받아 즉각적인 통제를 실행하는 구조다.

- **📢 섹션 요약 비유**: 정적 신호는 신분증 사진이고, 동적 신호는 지금 내 눈앞에 서 있는 사람의 움직임과 목소리다. 둘 다 일치해야 진짜다.

## Ⅳ. 실무 적용 및 기술사 판단

기술사는 어떤 신호를 우선적으로 신뢰할 것인지 정책의 우선순위를 결정해야 한다.

### 실무 적용 우선순위 (체크리스트)

1. **기기 준수 신호(Compliance)**: 단순 IP보다 Intune의 '준수 상태' 신호를 더 높게 평가한다. 기기가 안전하면 위치에 상관없이 유연한 접근을 허용할 수 있다.
2. **리스크 기반 신호**: Identity Protection에서 제공하는 High Risk 신호는 모든 정책에 우선하여 즉시 차단하도록 설정한다.
3. **위치 신호 최적화**: VPN 사용자의 공인 IP 대역을 정확히 등록하여 불필요한 정책 충돌을 방지한다.
4. **상호 작용 신호**: 사용자가 직접 MFA를 완료했는지, 또는 생체 인식을 통과했는지의 신호를 다음 접근 결정에 활용한다.

### 기술사적 판단

- "모든 신호를 다 체크하면 보안이 가장 좋지 않은가?"라는 질문에, 기술사는 **"성능과 지연시간(Latency)"**을 언급해야 한다. 너무 많은 외부 신호(Third-party signal)를 연동하면 로그인 시 지연시간이 발생하여 사용자 경험이 악화될 수 있다. 따라서 핵심 신호를 중심으로 최적화된 정책 셋을 구성하는 것이 실력이다.

- **📢 섹션 요약 비유**: 질문이 너무 많은 면접관은 면접 시간을 너무 끌게 된다. 핵심만 짚어서 물어보는 유능한 면접관이 되어야 한다.

## Ⅴ. 기대효과 및 결론

신호 기반 보안은 '맥락을 이해하는 보안'이다. 이는 단순히 문을 잠그는 행위를 넘어, 누가 어떤 의도로 문을 열려 하는지를 파악하여 지능적으로 대응할 수 있게 한다.

결론적으로 신호는 조건부 액세스의 연료다. 더 다양하고 정확한 신호를 확보할수록 기업의 제로 트러스트 아키텍처는 더 정교해지고 안전해진다. 미래에는 AI가 신호 사이의 숨겨진 상관관계를 분석하여, 우리가 미처 설정하지 못한 위험 신호까지도 스스로 찾아낼 것이다.

- **📢 섹션 요약 비유**: 신호는 세상의 모든 정보(Data)를 지혜(Wisdom)로 바꾸어주는 보안의 원재료다.
