## 핵심 인사이트 (3줄 요약)
> 1. **본질**: MS-CHAPv2 (Microsoft Challenge Handshake Authentication Protocol v2)는 주로 VPN(PPTP)이나 무선랜(802.1X/PEAP) 인증에 사용되는 NTLM 기반의 양방향 상호 인증 프로토콜이다.
> 2. **가치**: 클라이언트뿐만 아니라 서버도 신원을 증명하도록 설계되어 초기 버전(v1)의 취약점을 보완했으나, 내부적으로 여전히 약한 DES 암호화 구조를 사용하여 보안 위협이 존재한다.
> 3. **판단 포인트**: 2012년 이후 구조적 결함이 증명되어 단독 사용은 위험하며, 반드시 TLS 터널(PEAP, EAP-TLS) 내부에 캡슐화하여 사용하거나 최신 프로토콜로 대체해야 한다.

---

## Ⅰ. 개요 및 필요성

MS-CHAPv2는 마이크로소프트가 원격 접속 보안을 위해 만든 프로토콜이다. 과거 전화선 연결(PPP)이나 초기 VPN(PPTP) 시절, 사용자가 정말 그 사용자가 맞는지 확인하기 위해 등장했다.

NTLM의 챌린지-응답 방식을 차용하면서도, 서버도 클라이언트에게 암호를 풀어서 보여주게 하는 '상호 인증' 기능을 추가한 것이 특징이다. 하지만 세월이 흐르며 기반 기술인 DES 암호화가 너무 쉽게 깨지게 되면서, 현재는 보안상의 이유로 단독으로 쓰이는 대신 다른 강력한 암호화 터널 안에서만 쓰이는 보조적인 역할을 수행한다.

- **📢 섹션 요약 비유**: MS-CHAPv2는 오래된 비밀 통신 장비와 같다. 서로 누군지 확인하는 기능은 좋지만, 기계 자체가 낡아서 도청당하기 쉽기 때문에 두꺼운 방음벽(TLS) 안에서만 써야 한다.

---

## Ⅱ. 아키텍처 및 핵심 원리

MS-CHAPv2는 챌린지와 응답을 양방향으로 주고받는 4단계 과정을 거친다.

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│                    MS-CHAPv2 4-Step Handshake Flow                           │
├──────────────────────────────────────────────────────────────────────────────┤
│      [Client]                                         [Server]               │
│          │                                               │                   │
│          │◀─ 1. CHALLENGE (Server Nonce) ───────────────│                   │
│          │                                               │                   │
│          │── 2. RESPONSE (Client Nonce + Client Resp) ──▶│                   │
│          │                                               │                   │
│          │◀─ 3. SUCCESS (Server Resp / Mutual Auth) ────│                   │
│          │                                               │                   │
│          │── 4. ACK / FAILURE ──────────────────────────▶│                   │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 동작 특징
1. **Server Challenge**: 서버가 난수를 보낸다.
2. **Client Response**: 클라이언트도 자기만의 난수를 만들고, 사용자 비밀번호 해시로 서버의 난수를 암호화하여 보낸다.
3. **Mutual Authentication (Success Message)**: 서버는 클라이언트의 정답을 확인한 후, 자신도 클라이언트의 난수를 암호화하여 응답한다. 클라이언트는 이를 통해 서버가 진짜인지 확인한다.
4. **결함**: 내부적으로 비밀번호 해시를 3개의 DES 키로 쪼개어 암호화하는데, 이 중 일부가 매우 짧아 무차별 대입 공격에 취약하다.

- **📢 섹션 요약 비유**: 내가 문제를 풀어서 주면, 상대방도 내가 낸 문제를 풀어서 대답해줘야 서로 "진짜구나" 하고 믿는 양방향 퀴즈 쇼와 같다.

## Ⅲ. 비교 및 연결

MS-CHAPv2는 다른 EAP(Extensible Authentication Protocol) 인증 방식과 비교된다.

| 항목 | MS-CHAPv2 (단독) | PEAP-MS-CHAPv2 | EAP-TLS |
|:---|:---|:---|:---|
| **터널 보호** | 없음 (노출됨) | **TLS 터널 내부** | TLS 상호 인증 |
| **보안성** | 낮음 (패킷 가로채기 취약) | 중간 (터널이 지켜줌) | 매우 높음 (인증서 사용) |
| **사용처** | 구형 VPN (PPTP) | 일반적인 Wi-Fi 인증 | 고보안 기업망 Wi-Fi |
| **취약점** | DES 크래킹 (CloudCracker) | 터널 해제 시 위험 | 거의 없음 |

현재 대부분의 기업용 Wi-Fi(WPA2/3 Enterprise)는 MS-CHAPv2를 그대로 쓰지 않고, PEAP(Protected EAP)라는 안전한 터널 속에 감싸서 사용한다.

- **📢 섹션 요약 비유**: MS-CHAPv2가 '벌거벗은 사람'이라면, PEAP는 그 사람에게 '튼튼한 갑옷'을 입혀준 것과 같다.

---

## Ⅳ. 실무 적용 및 기술사 판단

기술사적 관점에서 MS-CHAPv2는 **"단독 사용 금지"**가 핵심 판단 기준이다. 특히 PPTP VPN은 MS-CHAPv2를 기반으로 하는데, 이는 현재 기술력으로 하루 안에 모든 통신 내용을 복호화할 수 있는 수준이다.

### 실무 보안 전략
1. **PPTP VPN 퇴출**: PPTP 대신 L2TP/IPsec이나 SSL VPN, 또는 현대적인 WireGuard/Tailscale로 전환해야 한다.
2. **무선랜 보안 강화**: PEAP-MS-CHAPv2를 사용하더라도, 서버 인증서 검증 기능을 반드시 활성화하여 공격자가 가짜 Wi-Fi(Evil Twin)를 세워 인증 정보를 가로채는 것을 막아야 한다.
3. **EAP-TLS 전환**: 궁극적으로 비밀번호 기반의 MS-CHAPv2를 버리고, 사용자 인증서 기반의 EAP-TLS로 전환하는 것이 가장 안전하다.

- **📢 섹션 요약 비유**: 구멍 난 방패(MS-CHAPv2)를 믿고 전쟁터(인터넷)에 나가지 말아야 한다. 최소한 장갑차(TLS)에 타거나, 아예 투명 망토(인증서)를 써야 한다.

---

## Ⅴ. 기대효과 및 결론

MS-CHAPv2는 윈도우 원격 접속 역사에서 상호 인증의 개념을 대중화한 중요한 프로토콜이다. 하지만 기반 암호 기술의 노후화로 인해 더 이상 단독으로는 신뢰할 수 없는 상태가 되었다.

결론적으로 MS-CHAPv2는 "강력한 외벽(TLS)이 있을 때만 사용할 수 있는 내부 부품"으로 취급되어야 한다. 새로운 시스템을 구축할 때는 이 프로토콜의 의존성을 제거하고, 인증서 기반의 현대적인 인증 체계로 설계하는 것이 기술사적 통찰이다.

- **📢 섹션 요약 비유**: 엔진이 낡은 자동차와 같다. 모양은 멀쩡하지만 고속도로(네트워크)를 달리기엔 위험하므로, 엔진을 바꾸거나 새 차로 갈아타야 한다.

### 📌 관련 개념 맵
| 개념 | 연결 포인트 |
|:---|:---|
| **PEAP** | MS-CHAPv2를 TLS 터널로 보호하여 안전하게 전달하는 프레임워크 |
| **PPTP** | MS-CHAPv2를 기본 인증으로 사용하여 현재는 보안상 퇴출된 VPN 프로토콜 |
| **Mutual Authentication** | 클라이언트와 서버가 서로를 확인하는 MS-CHAPv2의 핵심 기능 |

### 👶 어린이를 위한 3줄 비유 설명
1. MS-CHAPv2는 비밀 편지를 주고받을 때, 나도 상대방을 확인하고 상대방도 나를 확인하는 방법이에요.
2. 서로가 정답을 맞춰야 대화를 시작하지만, 이 방법이 너무 오래되어서 나쁜 사람이 금방 암호를 풀 수 있게 되었어요.
3. 그래서 요즘은 이 편지를 아주 단단한 강철 상자(TLS)에 넣어서 주고받는답니다.
낼 수 있으니, 정답지가 밖으로 안 나가게 조심해야 해요.
