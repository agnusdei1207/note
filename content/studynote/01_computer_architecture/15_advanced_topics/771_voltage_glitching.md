## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 볼티지 글리칭(Voltage Glitching)은 칩의 전원 공급 장치(VCC)에 의도적으로 매우 짧은 순간의 전압 강하(Spike/Brown-out)를 주입하여, CPU가 명령어를 건너뛰거나 잘못된 결과를 계산하게 만드는 물리적 결함 주입 공격이다.
> 2. **가치**: 암호화 키 추출, 부팅 과정의 보안 검사(Secure Boot) 우회, 비밀번호 확인 로직 무력화 등 소프트웨어 보안 계층을 물리적 하위 레벨에서 완전히 붕괴시킬 수 있는 강력한 파괴력을 가진다.
> 3. **판단 포인트**: 이 공격은 물리적 접근(Physical Access)이 전제되므로, 하드웨어 설계 시 전압 모니터링 회로(LVD), 액티브 쉴드, 그리고 글리칭에 강한 소프트웨어 중복 검증 로직을 반드시 포함해야 한다.

---

## Ⅰ. 개요 및 필요성

### 1. 볼티지 글리칭(Voltage Glitching)의 정의
볼티지 글리칭은 임베디드 시스템이나 반도체 칩의 전력 공급 선에 마이크로초(μs) 또는 나노초(ns) 단위의 전압 변동을 일으키는 기술이다. 정상적인 전압 범위 밖으로 아주 잠깐 벗어남으로써, 디지털 회로 내부의 트랜지스터들이 올바른 논리 상태(0 또는 1)를 유지하지 못하게 유도한다.

### 2. 왜 볼티지 글리칭이 위험한가?
소프트웨어가 아무리 완벽해도 하드웨어가 엉뚱한 명령을 실행하면 보안은 무용지물이다.

1.  **조건문 우회**: `if (password == correct) { ... }` 라는 코드에서 비밀번호가 틀렸음에도 불구하고, 글리칭을 통해 CPU가 점프 명령(JNE/JZ)을 건너뛰게 만들어 '참'인 것처럼 속일 수 있다.
2.  **루프 탈출**: 무한 루프나 대기 상태를 강제로 종료시켜 비정상적인 실행 경로로 진입하게 한다.
3.  **암호 키 복원**: 암호화 연산 중 결함을 주입하여 변조된 결과값을 얻어낸 뒤, 이를 차분 결함 분석(DFA)에 사용하여 수 분 내에 마스터 키를 찾아낸다.

### 3. 기술적 배경: 전압과 속도의 상관관계
디지털 소자의 동작 속도는 공급 전압에 정비례한다. 전압이 순간적으로 낮아지면 신호의 전파 지연(Propagation Delay)이 늘어나 클럭 주기 내에 도달하지 못하게 되며, 이는 곧 데이터 파괴나 명령 미실행으로 이어진다.

- **📢 섹션 요약 비유**: 볼티지 글리칭은 복도에서 달려가는 사람(전기 신호)의 발을 아주 잠깐 거는 것과 같다. 넘어지거나 속도가 느려져서 제시간에 목적지(다음 게이트)에 도착하지 못하게 하는 원리다.

## Ⅱ. 아키텍처 및 핵심 원리

### 1. 글리칭 주입 아키텍처
공격자는 대상 칩의 전원 핀을 가로채어 글리치 주입 장치(FPGA, 전용 글리처 등)를 연결한다.

```text
[ Power Supply ] ────▶ [ Glitch Inserter (MOSFET Switch) ] ────▶ [ Target Chip (MCU/CPU) ]
                                ▲
                                │ (Trigger & Timing Control)
                       [ Control Logic (FPGA) ]
                                ▲
                                │ (Instruction Sync)
                       [ Logic Analyzer / Probe ]
```

### 2. 핵심 작동 매커니즘: 전압 강하 (VCC Dip)
1.  **타겟팅**: 공격자는 칩이 보안 명령(예: `CheckSignature`)을 수행하는 정확한 타이밍을 분석한다. (전력 분석 기법 병행)
2.  **트리거**: 특정 신호나 전력 패턴이 감지되면 FPGA가 매우 짧은 시간 동안 전원 스위치를 끄거나 접지로 단락(Short)시킨다.
3.  **임계점 위반**: 전압이 `V_min` 이하로 떨어지면, 내부의 플립플롭이 이전 상태를 유지하지 못하거나 로직 게이트가 잘못된 값을 출력한다.
4.  **결과 확인**: 시스템이 죽지 않고(Reboot 방지) 명령만 건너뛰었는지 확인한다.

### 3. 글리칭의 물리적 영향
| 영향 범위 | 현상 | 보안적 결과 |
|:---|:---|:---|
| **명령어 스킵** | PC(Program Counter)가 다음 명령으로 강제 점프 | 인증 로직 우회 |
| **레지스터 오염** | 레지스터에 저장된 값이 변함 | 연산 결과 조작, 키 유출 |
| **상태 머신 붕괴** | 제어 유닛(CU)의 상태 전이 오류 | 비정상 모드(Debug Mode) 진입 |

### 4. 공격 성공의 핵심: 타이밍과 폭 (Width)
글리치는 너무 길면 칩이 리셋되고, 너무 짧으면 영향이 없다. 수십 나노초 단위의 정밀한 조절이 공격의 성패를 가른다. 이를 위해 공격자는 보통 수천 번의 시행착오를 거쳐 최적의 '글리치 파라미터'를 찾아낸다.

- **📢 섹션 요약 비유**: 볼티지 글리칭은 자물쇠를 돌리는 찰나에 망치로 옆면을 툭 쳐서 잠금 핀이 제자리로 돌아가지 못하게 순간적으로 어긋나게 만드는 기술이다.

## Ⅲ. 비교 및 연결

### 1. Voltage Glitching vs Clock Glitching (경계 비교)
둘 다 결함 주입 공격이지만 조작하는 물리적 대상이 다르다.

| 비교 항목 | 볼티지 글리칭 (Voltage) | 클럭 글리칭 (Clock) |
|:---|:---|:---|
| **조작 대상** | 전원 전압 (VCC) | 시스템 클럭 (CLK) |
| **구현 난이도** | 상대적으로 쉬움 (전원선 노출) | 어려움 (내부 오실레이터 사용 시) |
| **파급 효과** | 데이터/명령어 모두에 영향 | 주로 명령어 실행 타이밍에 영향 |
| **장점** | 칩 외부 전원 핀만으로 공격 가능 | 매우 정밀한 위치 타겟팅 가능 |

### 2. Plundervolt와의 연결
Plundervolt는 소프트웨어로 전압을 조절하는 '원격 글리칭'이라 볼 수 있고, 볼티지 글리칭은 물리적으로 전선을 따서 직접 주입하는 '로컬 글리칭'이다. 원리는 동일하지만 접근 경로가 다르다.

### 3. 시큐어 부트(Secure Boot)와의 연결
임베디드 장비의 가장 큰 위협은 부팅 시 커널의 서명을 검사하는 코드에 글리치를 주입하는 것이다. 서명이 틀렸음에도 "검사 완료"라는 결과값을 내보내게 함으로써, 공격자가 제작한 임의의 펌웨어를 실행하게 만든다.

- **📢 섹션 요약 비유**: 볼티지 글리칭이 밥을 뺏어서 힘을 빼는 것이라면, 클럭 글리칭은 박자를 빠르게 해서 실수를 유도하는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 1. 하드웨어적 방어 전략 (Countermeasures)
1.  **LVD (Low Voltage Detection)**: 전압이 일정 수준 이하로 떨어지면 즉시 시스템을 리셋시키는 하드웨어 감시 회로를 탑재한다.
2.  **Voltage Regulators (LDO)**: 외부 전압이 흔들려도 내부 칩 전압을 일정하게 유지해주는 레귤레이터를 칩 내부에 통합한다.
3.  **Glitch Detectors**: 전압의 급격한 변화(Edge)를 감지하여 보안 경보를 울리는 전용 회로를 설계한다.

### 2. 소프트웨어적 방어 전략
1.  **중복 검증 (Redundancy)**: 보안 검사를 한 번만 하지 않고 두 번 수행하여, 결과가 일치할 때만 통과시킨다. (두 번 모두 글리칭에 성공하기는 기하급수적으로 어려움)
2.  **랜덤 딜레이 (Random Jitter)**: 보안 연산 전후에 무작위 시간을 끌어, 공격자가 정확한 글리칭 타이밍을 잡지 못하게 방해한다.
3.  **상태 변수 반전**: `true/false` 대신 `0x55/0xAA` 같이 비트 차이가 큰 값을 사용하여 비트 하나가 뒤집혀도 오동작으로 간주하게 만든다.

### 3. 기술사적 판단: 물리적 보안의 필요성
기술사는 "물리적 접근이 가능하면 모든 보안은 뚫린다"는 전제하에 설계해야 한다. 특히 ATM, IoT 도어락, 자동차 ECU와 같이 공격자가 장비를 손에 넣을 수 있는 환경에서는, 칩의 핀을 물리적으로 보호하는 에폭시 몰딩(Epoxy Molding)이나 탬퍼 감시(Tamper Detection) 기술을 필수적으로 적용해야 한다.

### 4. 실무 체크리스트
- [ ] 칩의 전원 입력단에 적절한 디커플링 커패시터(Decoupling Cap)가 배치되어 글리치를 흡수하는가?
- [ ] 중요한 보안 로직이 "결과값 반전 검사"와 같은 소프트웨어 방어 기법을 적용했는가?
- [ ] 하드웨어 수준에서 전압 급변 시 인터럽트를 발생시키는 설정이 활성화되어 있는가?
- [ ] 공격자가 쉽게 전원 선을 따지 못하도록 PCB 레이아웃이 설계되었는가?

- **📢 섹션 요약 비유**: 볼티지 글리칭 대응은 정수기에 필터를 달고 압력을 일정하게 유지하는 밸브를 설치하는 것과 같다. 외부에서 오염된 물을 넣거나 압력을 높여도 내부 기계는 안전하게 작동하게 만든다.

## Ⅴ. 기대효과 및 결론

### 1. 도입의 기대효과
- **물리적 무결성 확보**: 장비 탈취 시에도 내부 데이터와 알고리즘을 안전하게 보호한다.
- **제품 신뢰도 향상**: 결함 주입 테스트를 통과한 제품은 높은 수준의 보안 인증(Common Criteria 등)을 획득할 수 있다.

### 2. 한계 및 향후 전망
공격 기술이 발전함에 따라 수 나노초 단위의 미세한 글리치도 주입 가능해지고 있다. 미래의 칩은 내부적으로 클럭과 전원을 완전히 독자적으로 생성(Internal OSC & LDO)하여 외부 조작을 원천 차단하는 방향으로 진화할 것이다. 또한, 머신러닝을 활용하여 전력 소비 패턴과 전압 변화를 실시간 감시하고 이상 징후를 탐지하는 '지능형 보안 프로세서'가 주류가 될 전망이다.

### 3. 최종 결론
볼티지 글리칭은 소프트웨어적 논리가 물리적 현실에 기반하고 있음을 일깨워주는 공격이다. 기술사는 추상적인 코드 뒤에 숨겨진 전압과 전류의 흐름을 이해하고, 가장 낮은 계층(Physical Layer)에서부터 보안의 뿌리를 튼튼히 내려야 한다.

- **📢 섹션 요약 비유**: 볼티지 글리칭은 성벽을 넘는 것이 아니라 성벽 아래의 땅(전압)을 흔들어 성문이 저절로 열리게 만드는 공격이다. 땅이 흔들려도 무너지지 않는 튼튼한 기초 공사만이 성을 지키는 유일한 방법이다.

### 📌 관련 개념 맵

| 개념 | 연결 포인트 |
|:---|:---|
| **Fault Injection** | 볼티지 글리칭을 포함한 의도적 결함 유도 공격의 총칭 |
| **DFA (Differential Fault Analysis)** | 글리칭으로 얻은 오류값으로 암호 키를 푸는 수학적 기법 |
| **LVD (Low Voltage Detection)** | 전압 저하를 감지하여 시스템을 보호하는 하드웨어 장치 |
| **Secure Boot** | 글리칭 공격의 주된 목표가 되는 초기 부팅 보안 검증 과정 |
| **Side-channel Attack** | 글리칭의 타이밍을 잡기 위해 전력 소모를 분석하는 전행 작업 |

### 👶 어린이를 위한 3줄 비유 설명

1. 컴퓨터가 밥(전압)을 먹으면서 일을 하고 있는데, 나쁜 사람이 아주 잠깐 밥그릇을 뺏었다가 다시 돌려줘요.
2. 그러면 컴퓨터가 배가 고파서 깜짝 놀라 문제를 풀다가 한 줄을 건너뛰거나 틀린 답을 적게 돼요.
3. 이를 막으려면 밥이 조금이라도 줄어들면 "누가 밥을 뺏어갔어!"라고 소리치며 일을 멈추게 해야 한답니다.
