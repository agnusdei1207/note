> **핵심 인사이트 3줄**
> 1. D 래치(D Latch)는 SR 래치의 금지 상태(S=R=1)를 제거하기 위해 입력을 단일 D 신호로 단순화한 순차 논리 소자다.
> 2. Enable(EN) 신호가 HIGH일 때만 D 값을 Q에 반영하는 레벨 트리거(Level-Triggered) 방식으로 동작한다.
> 3. D 래치를 두 개 직렬 연결하면 D 플립플롭(D Flip-Flop)이 되어 클록 엣지에서만 상태가 변하는 엣지 트리거 방식을 구현할 수 있다.

---

## Ⅰ. D 래치의 개요

### 1.1 정의와 목적

D 래치(Data Latch)는 SR 래치의 금지 상태를 없애기 위해 입력을 `D` 하나로 줄인 기억 소자다. Enable 신호(EN, 또는 Gate)가 활성화된 동안 D값이 출력 Q에 투명하게 전달된다.

| 소자     | 입력       | 금지 상태 | 특징               |
|----------|------------|-----------|---------------------|
| SR 래치  | S, R       | S=R=1     | 비결정 상태 존재    |
| D 래치   | D, EN      | 없음       | 투명 래치           |
| D 플립플롭 | D, CLK   | 없음       | 엣지 트리거         |

### 1.2 진리표

| EN | D | Q(다음) | 동작          |
|----|---|---------|---------------|
| 0  | X | Q(현재) | 유지(Hold)    |
| 1  | 0 | 0       | 리셋(Reset)   |
| 1  | 1 | 1       | 셋(Set)       |

📢 **섹션 요약 비유**: EN이 수도꼭지, D가 물, Q가 컵 — 꼭지 열릴 때만 물이 들어간다.

## Ⅱ. 내부 구조와 게이트 회로

### 2.1 NAND 게이트 구현

```
D ──┬──[NOT]──┐
    │          ├── NAND ── S'──┐
    └── EN ───┘                ├── SR 래치 → Q
              EN ───┐          │
    D ─────── NAND─ R'──────────┘
```

D 래치는 SR 래치 앞에 NAND 2개와 인버터를 추가해 D→S, D'→R 관계를 자동 생성한다.

### 2.2 투명성(Transparency)

EN=1 구간 동안 D가 변화하면 Q도 즉시 변화한다. 이를 **투명(transparent)** 특성이라 한다.

```
EN: ‾‾‾‾‾‾|_____|‾‾‾‾
D:  0→1→0  (변화 중)
Q:  EN=1 동안 D 추적 → EN=0이면 마지막 D값 유지
```

📢 **섹션 요약 비유**: EN은 CCTV 녹화 버튼 — 버튼 눌린 동안만 실시간 화면, 버튼 떼면 마지막 장면 고정.

## Ⅲ. 레벨 트리거 vs 엣지 트리거

### 3.1 비교표

| 특성          | D 래치 (레벨 트리거)  | D 플립플롭 (엣지 트리거) |
|--------------|----------------------|--------------------------|
| 활성 조건     | EN 레벨 HIGH 전체 구간 | CLK 상승/하강 엣지 순간  |
| 투명성        | 있음                  | 없음                     |
| 글리치 민감성 | 높음                  | 낮음                     |
| 용도          | 레지스터 파일 일부    | 대부분의 플립플롭 설계   |

### 3.2 마스터-슬레이브 D 플립플롭

```
D → [D 래치 M (EN=CLK)] → [D 래치 S (EN=CLK')] → Q
```

EN이 서로 반전된 두 D 래치를 직렬 연결 → CLK 상승 엣지에서만 Q 갱신.

📢 **섹션 요약 비유**: 래치는 문이 열린 동안 누구나 통과, 플립플롭은 문이 열리는 찰나에만 통과 가능.

## Ⅳ. 응용 — 레지스터와 메모리

### 4.1 병렬 레지스터

N비트 데이터를 동시에 저장하기 위해 N개의 D 래치를 공통 EN 신호로 묶는다.

```
EN ─────────────────────
D0 → [D래치] → Q0
D1 → [D래치] → Q1
...
Dn → [D래치] → Qn
```

### 4.2 SRAM 셀과의 관계

SRAM의 6T 셀은 SR 래치 기반이나, 어드레스 디코더가 EN 역할을 해 D 래치와 유사하게 동작한다.

📢 **섹션 요약 비유**: D 래치 N개는 일제히 여닫는 사물함 줄 — 한 명의 관리자(EN)가 모든 칸을 동시에 열고 닫는다.

---

## Ⅴ. 설계 주의사항 — 래치 추론(Latch Inference)

### 5.1 HDL에서의 비의도적 래치

```verilog
// 잘못된 코드 — 래치 생성
always @(*) begin
  if (sel) out = a;  // else 없음 → 래치 추론
end
```

```verilog
// 올바른 코드 — 래치 방지
always @(*) begin
  if (sel) out = a;
  else out = b;  // else 명시
end
```

### 5.2 타이밍 분석 시 주의

EN 구간이 길면 D 입력의 글리치(glitch)가 Q에 직접 전달 → 셋업/홀드 타임 위반 위험.

📢 **섹션 요약 비유**: 소음 많은 교실에서 선생님이 귀를 열고 있으면(EN=1) 아무 잡음도 Q 귀에 전달된다. 귀를 닫아야(EN=0) 조용해진다.

## 👶 어린이를 위한 3줄 비유 설명

1. D 래치는 "열려 있을 때만 메모를 받는 칠판"이야 — 선생님(EN)이 허락할 때만 숫자(D)가 적혀.
2. 허락이 끝나면(EN=0) 칠판은 굳어버려서 아무리 숫자가 바뀌어도 그대로야.
3. 플립플롭은 선생님이 "딱!"(클록 엣지) 하는 순간에만 적는 칠판 — 훨씬 정확해서 컴퓨터 CPU가 주로 써.
