> **핵심 인사이트**
> 1. 아스키코드(ASCII)는 컴퓨터가 0과 1밖에 모르는 한계를 극복하고 인간의 언어인 **문자(Character)를 처리하기 위해 제정한 최초의 7비트 국제 표준 약속**이다.
> 2. 영문 알파벳 대소문자, 숫자 0~9, 특수기호, 그리고 프린터나 통신 제어를 위한 128개의 문자로 구성되어 있다.
> 3. 단순하고 효율적이지만, **7비트(최대 128자)의 크기 한계로 인해 한글이나 한자 같은 다국어 표현이 불가능**하여 이후 유니코드(Unicode) 체계로 발전하는 계기가 되었다.

---

## Ⅰ. 숫자로 문자를 표현하는 약속
초기 컴퓨터는 순수하게 대포알 궤적을 계산하는 수학 기계였습니다. 하지만 사람들은 컴퓨터가 문서 작업이나 통신까지 해주길 바랐습니다. 문제는 컴퓨터가 알파벳 'A'를 이해할 수 없다는 것이었습니다.

그래서 미국 국가표준협회(ANSI)는 전 세계 컴퓨터가 공통으로 사용할 **문자-숫자 치환표(Mapping Table)**를 만들었는데, 이것이 바로 ASCII(American Standard Code for Information Interchange)입니다.
* 대문자 'A'는 십진수 65 (`1000001`)
* 소문자 'a'는 십진수 97 (`1100001`)
* 문자 '0'은 십진수 48 (`0110000`)

```ascii
[ASCII 코드의 기본 구조 매핑 (일부 발췌)]

 십진수 | 이진수 (7비트) | 출력 문자 (Character)
 -------+----------------+--------------------------
   32   |   0100000      |  (공백, Space)
   48   |   0110000      |  '0' (숫자 0 문자)
   65   |   1000001      |  'A' (대문자 A)
   97   |   1100001      |  'a' (소문자 a)
  127   |   1111111      |  (삭제, DEL 제어문자)
```

📢 **섹션 요약 비유**: 아스키코드는 '스파이들의 난수표'입니다. 서로 적의 눈을 피해 "알파 작전(A) 개시!"라는 편지를 보내고 싶을 때, 미리 약속해 둔 난수표 책을 보고 "65번 작전 개시!"라고 숫자로만 적어서 보내면 수신자가 다시 책을 보고 문자로 해석하는 완벽한 치환 암호입니다.

## Ⅱ. 왜 8비트(1바이트)가 아니라 7비트인가?
컴퓨터 메모리는 보통 8비트(1바이트) 단위로 데이터를 저장하는데, 아스키코드는 왜 애매하게 7비트(128개)만 썼을까요?

1960년대 텔레타이프(타자기)로 통신하던 시절, 전선에 노이즈가 껴서 데이터가 깨지는 일이 빈번했습니다. 그래서 8비트 중 **7비트는 실제 문자를 담는 데 쓰고, 남은 1비트는 통신 에러를 검사하는 '패리티 비트(Parity Bit)' 전용 공간으로 비워두었기 때문**입니다.
* 예: 짝수 패리티 규칙일 때, `A(1000001)`는 1의 개수가 두 개(짝수)이므로 패리티 비트를 `0`으로 세팅하여 최종 `0 1000001`로 보냅니다. 만약 도착한 데이터의 1의 개수가 홀수라면? 즉시 "다시 보내라!"고 에러 처리를 합니다.

```ascii
[1바이트(8비트) 내부의 ASCII 코드와 패리티 비트의 동거]

 [ P ] [ 7 ] [ 6 ] [ 5 ] [ 4 ] [ 3 ] [ 2 ] [ 1 ]
  ^      |-----------------------------------|
  |                       |
 에러검사             실제 문자 정보 (ASCII, 최대 128가지)
 전용 1비트
```

📢 **섹션 요약 비유**: 아스키코드의 7비트는 '계란 10구짜리 포장 박스'입니다. 원래 10개를 꽉 채울 수 있지만, 배달 중에 계란이 깨졌는지 확인하기 위해 9개 칸에만 진짜 계란을 넣고 마지막 1칸에는 '안전 검사 확인증'을 꽂아서 배달했던 옛날 방식입니다.

## Ⅲ. 아키텍처 관점: 확장 아스키(Extended ASCII)와 한계
시간이 흘러 통신 기술이 발전하면서 에러 검사를 위한 패리티 비트 1개가 필요 없어졌습니다. 
그러자 IBM이나 마이크로소프트 같은 기업들은 "노는 1비트를 문자로 꽉 채우자!"라며 8비트 전체를 사용하는 **확장 아스키(Extended ASCII)**를 만들었습니다. 128개였던 문자가 256개($2^8$)로 두 배 늘어났고, 남는 자리에 프랑스어, 독일어 악센트나 그래픽 괘선 문자(테두리 그리는 기호)를 우겨넣었습니다.

하지만 한국어, 중국어, 일본어(CJK)처럼 글자 수가 수만 개에 달하는 언어는 256개 공간으로는 어림도 없었습니다. 결국 각 나라가 제멋대로 2바이트짜리 확장 코드를 조합해 쓰다 보니 **'글자가 와장창 깨지는(한글 깨짐 현상)' 글로벌 호환성 재앙**이 발생했습니다. 이 문제를 종식하기 위해 전 세계 모든 문자를 담을 수 있는 거대한 댐인 '유니코드(Unicode)'가 등장하게 됩니다.

```ascii
[문자 표현 공간의 한계와 진화]

 7비트 ASCII (128자) : 영어 알파벳, 숫자, 기호 (미국 중심)
        | (1비트 확장)
 8비트 확장 ASCII (256자) : 유럽어 추가 (여전히 동양권 문자 불가)
        | (바이트 확장)
 16/32비트 유니코드 (수십만 자) : 한글, 한자, 이모티콘(😂)까지 완벽 지원!
```

📢 **섹션 요약 비유**: 아스키코드는 '100페이지짜리 미니 수첩'입니다. 영어 단어만 적을 때는 공간이 널널했지만, 나중에 한국어 단어, 중국어 단어까지 다 적으려고 하니 수첩이 터져버렸습니다. 그래서 결국 전 세계 언어를 다 담을 수 있는 수만 페이지짜리 '거대한 백과사전(유니코드)'을 새로 만들 수밖에 없었습니다.

## 개념 맵 (Concept Map)
```ascii
[아스키코드 (ASCII)]
   │
   ├── 정의: 문자를 7비트 이진수로 치환하는 영문 알파벳 중심의 초기 표준 (총 128자)
   │
   ├── 구조적 특징
   │     ├── 1바이트(8비트) 중 7비트만 데이터로 사용
   │     └── MSB 1비트는 통신 오류 검출용(Parity Bit)으로 예약됨
   │
   └── 시스템 아키텍처 한계
         └── 다국어(한글, 한자) 표현 불가능 -> 확장 아스키(8비트) 시도 -> 유니코드(Unicode)로 패러다임 전환
```

## 어린이 비유
아스키코드는 컴퓨터와 대화하기 위해 만든 **128개의 비밀 번호장부**예요! 컴퓨터는 알파벳 'A'를 못 읽어서, 장부에 "A는 65번이야"라고 적어뒀어요. 그래서 키보드에서 'A'를 톡 누르면, 컴퓨터한테는 65라는 숫자가 슝 날아가요. 하지만 이 장부는 칸이 너무 적어서, 한국말이나 귀여운 웃는 얼굴 이모티콘을 적을 수 없다는 슬픈 단점이 있었답니다.
