# 405. 하드웨어 동기화 (Hardware Synchronization)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 멀티프로세서에서 원자적 연산과 동기화를 지원하는 하드웨어 메커니즘
> 2. **가치**: 효율적인 동기화, 경쟁 조건 방지
> 3. **융합**: CAS, LL/SC, 원자적 명령어, 락 구현

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

하드웨어 동기화(Hardware Synchronization)는 **멀티프로세서 시스템에서 여러 코어 간의 협력과 순서 제어를 위해 CPU가 제공하는 원자적 연산, 배리어, 락 명령어 등의 하드웨어 지원**이다.

### 💡 비유: 토론 회의 진행자

하드웨어 동기화는 **토론 회의 진행자**와 같다. 여러 사람이 발언하려 한다. 동시에 말하면 안 된다. 진행자가 있다. "당신 차례" 말한다. 한 명만 발언. 다 끝나면 다음. 원자적이다. 하드웨어 동기화가 똑같다. 원자적 명령어, 상호 배제, 순서 보장.

### 하드웨어 동기화 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                하드웨어 동기화 개요                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하드웨어 동기화 메커니즘】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 원자적 명령어 (Atomic Instructions):                                  │ │
│  │     • 읽기-수정-쓰기를 한 번에 수행                                    │ │
│  │     • 인터럽트 불가                                                  │ │
│  │                                                               │ │
│  │  2. CAS (Compare-And-Swap):                                              │ │
│  │     • 조건부 원자적 교환                                             │ │
│  │     • x86: CMPXCHG                                                  │ │
│  │                                                               │ │
│  │  3. LL/SC (Load-Linked/Store-Conditional):                               │ │
│  │     • ARM, MIPS, RISC-V                                             │ │
│  │     • 조건부 저장                                                    │ │
│  │                                                               │ │
│  │  4. 메모리 배리어 (Memory Barrier):                                       │ │
│  │     • 순서 강제                                                      │ │
│  │     • x86: MFENCE, ARM: DMB                                         │ │
│  │                                                               │ │
│  │  5. 락 명령어 (Lock Instructions):                                        │ │
│  │     • x86: LOCK 접두사                                               │ │
│  │     • 버스 락킹                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【원자적 연산의 필요성】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  문제: count++ (읽기-증가-쓰기)                                          │ │
│  │                                                               │ │
│  │  비원자적 실행 (경쟁 조건):                                               │ │
│  │  ┌──────────────────────────────────────────────────┐          │ │
│  │  │  시간  Core 0           Core 1                   │          │ │
│  │  │  ────  ──────           ──────                   │          │ │
│  │  │   t1   read count=10                            │          │ │
│  │  │   t2                      read count=10          │          │ │
│  │  │   t3   count=11                                   │          │ │
│  │  │   t4                      count=11               │          │ │
│  │  │   t5   write 11                                  │          │ │
│  │  │   t6                      write 11               │          │ │
│  │  │  → 결과: 11 (12가 되어야 함)                       │          │ │
│  │  └──────────────────────────────────────────────────┘          │ │
│  │                                                               │ │
│  │  원자적 실행:                                                            │ │
│  │  • CAS 또는 atomic increment                                        │ │
│  │  • 한 코어만 성공, 다른 코어 재시도                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 하드웨어 동기화 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                하드웨어 동기화 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CAS 동작】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  CAS(addr, expected, new):                                               │ │
│  │  1. addr의 값 읽기                                                      │ │
│  │  2. expected와 비교                                                     │ │
│  │  3. 같으면 new 저장, 성공 반환                                           │ │
│  │  4. 다르면 실패 반환                                                    │ │
│  │                                                               │ │
│  │  의사코드:                                                                  │ │
│  │  bool CAS(int* addr, int expected, int new) {                            │ │
│  │      if (*addr == expected) {                                            │ │
│  │          *addr = new;                                                    │ │
│  │          return true;                                                    │ │
│  │      }                                                                   │ │
│  │      return false;                                                       │ │
│  │  }  // 원자적으로 실행                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【LL/SC 동작】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  LL (Load-Linked): addr에서 읽기, 링크 레지스터에 저장                    │ │
│  │  SC (Store-Conditional): 링크가 유효하면 저장, 아니면 실패               │ │
│  │                                                               │ │
│  │  장점: ABA 문제 없음 (링크 무효화)                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【x86 LOCK 접두사】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  LOCK INC [addr]    # 원자적 증가                                       │ │
│  │  LOCK XCHG reg, [addr]    # 원자적 교환                                 │ │
│  │  LOCK CMPXCHG [addr], reg    # CAS                                     │ │
│  │                                                               │ │
│  │  구현: 캐시 라인 락킹 (버스 락킹은 예외적)                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 효율적: 하드웨어 수준                                                │ │
│  │  • 스핀락: 대기 중 CPU 사용                                            │ │
│  │  • Lock-free: 진행 보장                                                │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 경합: 캐시 트래픽                                                   │ │
│  │  • ABA: CAS의 문제                                                    │ │
│  │  • 기아: 우선순위 역전                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 하드웨어 동기화 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                하드웨어 동기화 분석                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ atomic 사용】                                                              │)
│  ──────────────────                                                  │
│  #include <atomic>                                                  │
│                                                                     │
│  std::atomic<int> counter{0};                                       │
│                                                                     │
│  // 원자적 증가                                                       │
│  counter.fetch_add(1);                                              │
│                                                                     │
│  // CAS 루프                                                         │
│  int expected = counter.load();                                     │
│  while (!counter.compare_exchange_weak(expected, expected + 1));    │
│                                                                     │
│  【어셈블리 확인】                                                              │)
│  ──────────────────                                                  │
│  $ objdump -d program | grep lock                                   │
│  lock addl $0x1,(%rax)    # 원자적 증가                               │
│                                                                     │
│  【성능 측정】                                                                   │)
│  ──────────────────                                                  │
│  $ perf stat -e instructions,cycles ./program                       │
│  # 경합이 높으면 CPI 증가                                             │
│                                                                     │
│  【하드웨어 동기화 최적화 팁】                                                      │)
│  ──────────────────                                                  │
│  1. 경합 감소: 락 분할                                                │
│  2. 백오프: 지수 백오프                                               │
│  3. 큐 기반: MCS, CLH 락                                             │
│  4. 로컬: 스레드 로컬 우회                                            │
│  5. RCU: 읽기 많은 경우                                               │
│  6. 측정: 프로파일링                                                  │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 하드웨어 동기화: CPU 수준 동기화 지원
• CAS: 비교 후 교환, 원자적
• LL/SC: 조건부 저장, ARM
• x86: LOCK 접두사
• 장점: 효율적, lock-free
• 단점: 경합, ABA 문제
• 활용: 스핀락, 원자적 연산
• 구현: 캐시 라인 락킹
• C++: std::atomic
• 최적화: 백오프, 큐 기반
```
