# 399. 하이퍼스레딩 (Hyper-Threading)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: Intel의 SMT 구현, 단일 코어를 두 개의 논리 프로세서로 표현
> 2. **가치**: 최대 30% 처리량 향상, 5% 미만의 다이 면적 증가
> 3. **융합**: SMT, 논리 프로세서, 리소스 공유, 스케줄링

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

하이퍼스레딩(Hyper-Threading, HT)은 **Intel이 구현한 동시 멀티스레딩(SMT) 기술로, 단일 물리 코어를 두 개의 논리 프로세서로 표현하여 운영체제가 두 개의 CPU로 인식하게 하고, 동시에 두 스레드의 명령어를 실행하는 기술**이다.

### 💡 비유: 한 사람이 두 역할

하이퍼스레딩은 **한 사람이 두 역할**을 하는 것과 같다. 왼손, 오른손이 있다. 왼손은 A를 한다. 오른손은 B를 한다. 한 사람이지만 두 일. 머리는 하나. 하지만 동시 진행. 하이퍼스레딩이 똑같다. 한 코어, 두 논리 프로세서, 동시 실행.

## Ⅱ. 하이퍼스레딩 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                하이퍼스레딩 상세                                          │

│  【중복 vs 공유 리소스】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  중복 (스레드별 별도):                                                   │ │
│  │  • 레지스터 세트 (GPR, XMM/YMM/ZMM)                                  │ │
│  │  • 프로그램 카운터 (IP)                                               │ │
│  │  • APIC ID                                                         │ │
│  │  • 리네임 레지스터 (일부)                                              │ │
│  │  • ROB (Reorder Buffer) 엔트리                                       │ │
│  │                                                               │ │
│  │  공유 (스레드 간 공유):                                                   │ │
│  │  • L1 명령어/데이터 캐시                                              │ │

│  │  • 실행 유닛 (ALU, FPU, AGU)                                         │ │
│  │  • 메모리 서브시스템                                                  │ │
│  │  • 프론트엔드 (Fetch, Decode)                                         │ │
│  │                                                               │ │
│  │  하드웨어 오버헤드: 약 5% 다이 면적 증가                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 특성】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  워크로드별 향상:                                                          │ │
│  │  • 정수 연산: 10-20%                                                │ │
│  │  • 부동소수점: 5-15%                                                │ │
│  │  • 메모리 집약: 20-30%                                              │ │
│  │  • 멀티태스킹: 20-30%                                               │ │
│  │  • 단일 스레드 CPU 집약: 0-5% (오히려 저하 가능)                        │ │
│  │                                                               │ │
│  │  최대 향상: 이론적 30-40%, 실제 15-30%                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 처리량 향상: 최대 30%                                              │ │
│  │  • 낮은 비용: 5% 면적 증가                                            │ │
│  │  • 투명성: OS가 자동 인식                                             │ │
│  │  • 응답성: 멀티태스킹 개선                                             │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 단일 스레드: 저하 가능                                             │ │
│  │  • 캐시 경합: 용량 분할                                               │ │
│  │  • 보안: 부채널 공격 (Spectre 등)                                     │ │
│  │  • 라이선스: 일부 소프트웨어 비용 증가                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
```

### 하이퍼스레딩 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                하이퍼스레딩 분석                                          │

│  ──────────────────                                                  │
│  $ lscpu                                                            │
│  CPU(s): 8                                                          │
│  Core(s) per socket: 4                                              │

│                                                                     │
│  【HT 제어】                                                                   │)
│  ──────────────────                                                  │
│  # HT 비활성화                                                       │
│  $ echo off > /sys/devices/system/cpu/smt/control                   │
│                                                                     │
│  # BIOS에서 HT 끄기 (시스템 재시작 필요)                               │
│                                                                     │
│  【성능 비교】                                                                   │)
│  ──────────────────                                                  │
│  # HT 켜고 성능 측정                                                  │
│  $ perf stat -a ./workload                                          │
│                                                                     │
│  # HT 끄고 성능 측정                                                  │
│  $ echo off > /sys/devices/system/cpu/smt/control                   │
│  $ perf stat -a ./workload                                          │
│                                                                     │
│  【하이퍼스레딩 최적화 팁】                                                        │)
│  ──────────────────                                                  │
│  1. 워크로드 분석: HT 효과 측정                                       │
│  2. 스레드 수: 물리 코어 수로 시작                                    │
│  3. 친화성: 형제 스레드 인식                                           │
│  4. 보안: 클라우드는 HT 끄기                                          │
│  5. 측정: on/off 비교                                                 │
│  6. 라이선스: 코어 기반 라이선스 확인                                  │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 하이퍼스레딩: Intel의 SMT 구현
• 본질: 1 물리 코어 → 2 논리 프로세서
• 중복: 레지스터, PC, APIC
• 공유: 파이프라인, 캐시, 실행 유닛
• 비용: 다이 면적 5% 증가
• 성능: 최대 30% 처리량 향상
• 장점: 낮은 비용, 투명성
• 단점: 캐시 경합, 보안 이슈
• OS: HT 인식 스케줄링
• 완화: 보안 위해 HT 끄기
```
