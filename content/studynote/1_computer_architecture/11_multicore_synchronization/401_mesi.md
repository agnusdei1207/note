# 401. MESI 프로토콜 (MESI Protocol)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 4상태 캐시 일관성 프로토콜 (Modified, Exclusive, Shared, Invalid)
> 2. **가치**: 쓰기 백 최적화, 불필요한 트래픽 감소
> 3. **융합**: 스누핑, 캐시 라인, 버스 트랜잭션

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

MESI 프로토콜은 **캐시 라인의 상태를 Modified, Exclusive, Shared, Invalid의 4가지로 관리하며, 쓰기 백(Write-back) 캐시에서 일관성을 유지하는 가장 널리 사용되는 캐시 일관성 프로토콜**이다. Illinois 프로토콜이라고도 한다.

### 💡 비유: 도서관 책 대출

MESI는 **도서관 책 대출**과 같다. 책이 있다. 한 사람만 빌리면 독점(Exclusive). 여러 사람이 읽으면 공유(Shared). 한 사람이 필기하면 수정(Modified). 반납하면 무효(Invalid). 상태를 추적한다. MESI가 똑같다. 캐시 라인, 4가지 상태, 일관성 관리.

### MESI 프로토콜 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                MESI 프로토콜 개요                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【4가지 상태】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  M (Modified):                                                           │ │
│  │  • 이 캐시만 해당 라인 소유                                            │ │
│  │  • 메모리와 다름 (수정됨)                                             │ │
│  │  • 쓰기 가능, 메모리 쓰기 필요 시 Write-back                          │ │
│  │                                                               │ │
│  │  E (Exclusive):                                                          │ │
│  │  • 이 캐시만 해당 라인 소유                                            │ │
│  │  • 메모리와 동일                                                      │ │
│  │  • 쓰기 시 M로 변경 (버스 트래픽 없음)                                  │ │
│  │                                                               │ │
│  │  S (Shared):                                                              │ │
│  │  • 여러 캐시가 소유 가능                                              │ │
│  │  • 메모리와 동일                                                      │ │
│  │  • 쓰기 시 무효화 브로드캐스트 필요                                    │ │
│  │                                                               │ │
│  │  I (Invalid):                                                             │ │
│  │  • 캐시 라인 무효                                                     │ │
│  │  • 사용 불가, 읽기 시 미스 발생                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상태 다이어그램】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │                    Read Miss (다른 없음)                                   │ │
│  │              ┌──────────────────────────┐                          │ │
│  │              │                          ▼                          │ │
│  │        ┌───────────┐              ┌───────────┐                   │ │
│  │        │           │  Read Hit    │           │                   │ │
│  │   ┌───►│  Invalid  │◄─────────────│ Exclusive │◄───┐              │ │
│  │   │    │    (I)    │              │    (E)    │    │              │ │
│  │   │    └───────────┘              └───────────┘    │              │ │
│  │   │         │                            │         │              │ │
│  │   │         │ Read Miss                  │ Write   │ Read Miss    │ │
│  │   │         │ (다른 있음)                │         │ (다른 있음)  │ │
│  │   │         ▼                            ▼         │              │ │
│  │   │    ┌───────────┐              ┌───────────┐    │              │ │
│  │   │    │           │  Write       │           │    │              │ │
│  │   │    │  Shared   │─────────────►│ Modified  │────┘              │ │
│  │   │    │    (S)    │  (Invalidate)│    (M)    │                   │ │
│  │   └───►└───────────┘              └───────────┘                   │ │
│  │         Other Write               Read/Write Hit                   │ │
│  │         (Invalidate)                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. MESI 프로토콜 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                MESI 프로토콜 상세                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【버스 트랜잭션】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  BusRead (BR):                                                          │ │
│  │  • 캐시 미스 시 메모리/다른 캐시에서 데이터 요청                        │ │
│  │                                                               │ │
│  │  BusReadX (BRX):                                                         │ │
│  │  • 쓰기 미스 시 배타적 읽기 요청                                        │ │
│  │  • 다른 캐시의 S 상태 라인 무효화                                       │ │
│  │                                                               │ │
│  │  BusUpgrade (BU):                                                        │ │
│  │  • S 상태에서 쓰기 시 업그레이드 요청                                   │ │
│  │  • 다른 캐시의 S 상태 라인 무효화                                       │ │
│  │                                                               │ │
│  │  Flush (F):                                                              │ │
│  │  • M 상태 라인을 메모리에 기록                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상태 전이 예시】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  시나리오: Core 0, 1이 변수 X 공유                                       │ │
│  │                                                               │ │
│  │  1. Core 0: X 읽기 → I → E (다른 코어 없음)                            │ │
│  │  2. Core 1: X 읽기 → Core 0: E → S, Core 1: I → S                     │ │
│  │  3. Core 0: X 쓰기 → Core 0: S → M, Core 1: S → I                      │ │
│  │  4. Core 1: X 읽기 → Core 0: M → S (Flush), Core 1: I → S              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • E 상태: 독점 쓰기 시 버스 트래픽 없음                                │ │
│  │  • 쓰기 백: 메모리 쓰기 최소화                                         │ │
│  │  • 널리 사용: Intel, AMD 등                                            │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • S→M: 무효화 트래픽 발생                                             │ │
│  │  • O 상태 없음: 공유 수정 불가 (MOESI는 가능)                           │ │
│  │  • 확장성: 대규모 시스템에서 버스 병목                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
│  4. 읽기: RCU 활용                                                   │
│  5. 측정: coherence 트래픽                                           │
│  6. 캐시 라인: 64바이트 정렬                                          │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• MESI: 4상태 캐시 일관성 프로토콜
• M (Modified): 수정됨, 메모리와 다름
• E (Exclusive): 독점, 쓰기 가능
• S (Shared): 공유, 읽기만 가능
• I (Invalid): 무효
• 장점: E 상태로 버스 트래픽 감소
• 단점: S→M 전환 시 무효화 필요
• 버스: BusRead, BusReadX, BusUpgrade, Flush
• 활용: Intel, AMD 대부분 CPU
• 확장: MOESI (O 상태 추가)
```
