# 397. 멀티스레딩 (Multithreading)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단일 프로세서에서 여러 스레드를 빠르게 전환하거나 동시 실행
> 2. **가치**: 파이프라인 활용률 향상, 처리량 증가
> 3. **융합**: SMT, 하이퍼스레딩, TLP, 하드웨어/소프트웨어

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

멀티스레딩(Multithreading)은 **단일 CPU 코어에서 여러 스레드의 명령어를 실행하여, 메모리 지연 등으로 인한 파이프라인 공회전을 줄이고 명령어 처리량을 높이는 기술**이다. 하드웨어 멀티스레딩과 소프트웨어 멀티스레딩으로 구분된다.

### 💡 비유: 주방의 요리사

멀티스레딩은 **주방의 요리사**와 같다. 요리사가 한 명이다. 하지만 기다리는 동안 다른 요리. 파를 썬다. 끓기를 기다린다. 그 사이에 국을 저음. 한 명이 여러 일. 전환은 빠르다. 멀티스레딩이 똑같다. 한 코어, 여러 스레드, 대기 시간 활용.

### 멀티스레딩 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                멀티스레딩 개요                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【단일 스레드 vs 멀티스레드】                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  단일 스레드:                                                            │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │ T1: [실행][대기][실행][대기][실행][대기]              │    │ │
│  │  │                    ↑                                   │    │ │
│  │  │            캐시 미스 등으로 대기                         │    │ │
│  │  │        → 파이프라인 자원 유휴                            │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  │  멀티스레딩 (SMT):                                                      │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │ T1: [실행][----][실행][----][실행][----]              │    │ │
│  │  │ T2: [----][실행][----][실행][----][실행]              │    │ │
│  │  │            ↑                                           │    │ │
│  │  │    T1 대기 중 T2 실행                                   │    │ │
│  │  │    → 파이프라인 자원 활용                               │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【멀티스레딩 유형】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 세분화 멀티스레딩 (Fine-grained MT):                                  │ │
│  │     • 매 클럭 사이클마다 스레드 전환                                  │ │
│  │     • 라운드 로빈                                                  │ │
│  │     • 대기 시간 숨김                                               │ │
│  │                                                               │ │
│  │  2. 굵은 입자 멀티스레딩 (Coarse-grained MT):                              │ │
│  │     • 긴 지연 이벤트(캐시 미스) 시 전환                               │ │
│  │     • 전환 비용 존재                                               │ │
│  │                                                               │ │
│  │  3. 동시 멀티스레딩 (SMT):                                               │ │
│  │     • 여러 스레드 명령어 동시 발행                                    │ │
│  │     • 최고 효율                                                   │ │
│  │     • Intel Hyper-Threading                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 멀티스레딩 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                멀티스레딩 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하드웨어 멀티스레딩 상세】                                                        │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌───────────────────────────────────────────────────────┐    │ │
│  │  │           SMT 코어 구조 (2-way)                        │    │ │
│  │  │                                                       │    │ │
│  │  │   ┌─────────────┐     ┌─────────────┐                 │    │ │
│  │  │   │   스레드 0   │     │   스레드 1   │                 │    │ │
│  │  │   │  레지스터    │     │  레지스터    │  ← 중복 자원    │    │ │
│  │  │   │  PC         │     │  PC         │                 │    │ │
│  │  │   └─────────────┘     └─────────────┘                 │    │ │
│  │  │            │                 │                        │    │ │
│  │  │            └────────┬────────┘                        │    │ │
│  │  │                     ▼                                 │    │ │
│  │  │   ┌───────────────────────────────────────┐           │    │ │
│  │  │   │           공유 파이프라인              │  ← 공유    │    │ │
│  │  │   │   [Fetch][Decode][Execute][Write]    │           │    │ │
│  │  │   └───────────────────────────────────────┘           │    │ │
│  │  │                     │                                 │    │ │
│  │  │   ┌───────────────────────────────────────┐           │    │ │
│  │  │   │           공유 캐시                    │           │    │ │
│  │  │   └───────────────────────────────────────┘           │    │ │
│  │  └───────────────────────────────────────────────────────┘    │ │
│  │                                                               │ │
│  │  중복 자원: 레지스터, PC, 리오더 버퍼                          │ │
│  │  공유 자원: 파이프라인, 캐시, 실행 유닛                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 향상 요인】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • 메모리 지연 숨김: 캐시 미스 시 다른 스레드 실행                    │ │
│  │  • 파이프라인 활용: 슬롯 낭비 감소                                   │ │
│  │  • 자원 공유: 실행 유닛, 캐시                                        │ │
│  │  • 처리량 증가: 워크로드에 따라 15-40%                                │ │
│  │                                                               │ │
│  │  제한 요인:                                                                  │ │
│  │  • 캐시 경합: 캐시 용량 분할                                        │ │
│  │  • 메모리 대역폭: 대역폭 공유                                        │ │
│  │  • 실행 유닛: 포트 경합                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 처리량 증가: 15-40%                                              │ │
│  │  • 전력 효율: 성능/와트 향상                                         │ │
│  │  • 비용 효율: 코어 추가 없이 성능                                    │ │
│  │  • 응답성: 대기 시간 감소                                           │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 단일 스레드 성능: 약간 저하 가능                                   │ │
│  │  • 캐시 경합: 용량 분할                                              │ │
│  │  • 보안: 스레드 간 공격 (Spectre)                                    │ │
│  │  • 복잡성: OS 스케줄링                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
```

### 멀티스레딩 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                멀티스레딩 분석                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SMT 활성화 확인】                                                              │)
│  ──────────────────                                                  │
│  $ lscpu                                                            │
│  Thread(s) per core: 2    # SMT 활성화                              │
│  Core(s) per socket: 4                                             │
│  Socket(s): 1                                                      │
│  CPU(s): 8               # 4 cores × 2 threads                     │
│                                                                     │
│  【SMT 제어】                                                                   │)
│  ──────────────────                                                  │
│  # SMT 비활성화                                                      │
│  $ echo off > /sys/devices/system/cpu/smt/control                   │
│                                                                     │
│  # SMT 활성화                                                        │
│  $ echo on > /sys/devices/system/cpu/smt/control                    │
│                                                                     │
│  【성능 모니터링】                                                              │)
│  ──────────────────                                                  │
│  # 코어별 활용률                                                     │
│  $ mpstat -P ALL                                                    │
│                                                                     │
│  # 스레드 분석                                                       │
│  $ perf top -t <tid>                                                │
│                                                                     │
│  【멀티스레딩 최적화 팁】                                                          │)
│  ──────────────────                                                  │
│  1. 워크로드: 메모리 바운드 작업에 유리                                │
│  2. 스레드 수: 물리 코어 수 × SMT 스레드                              │
│  3. 친화성: 같은 코어의 스레드 배치                                   │
│  4. 캐시: 공유 데이터 최소화                                          │
│  5. 측정: SMT on/off 비교                                            │
│  6. 보안: 민감한 작업은 SMT 끄기                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 멀티스레딩: 한 코어에서 여러 스레드 실행
• 유형: Fine-grained, Coarse-grained, SMT
• SMT: 동시 멀티스레딩, 최고 효율
• 하이퍼스레딩: Intel의 SMT 구현
• 장점: 처리량 15-40% 증가
• 단점: 캐시 경합, 보안 이슈
• 중복 자원: 레지스터, PC
• 공유 자원: 파이프라인, 캐시
• 활용: 거의 모든 현대 CPU
• 측정: IPC, 처리량
```
