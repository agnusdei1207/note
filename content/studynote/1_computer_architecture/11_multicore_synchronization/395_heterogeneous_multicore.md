# 395. 이종 멀티코어 (Heterogeneous Multicore)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 성능/전력 특성이 다른 코어를 혼합한 아키텍처
> 2. **가치**: 작업 부하별 최적 코어, 에너지 효율
> 3. **융합**: big.LITTLE, DynaMO, 전력 관리, 스케줄링

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

이종 멀티코어(Heterogeneous Multicore)는 **성능, 전력 소모, 명령어 집합 등이 서로 다른 여러 유형의 CPU 코어를 단일 칩에 통합하여, 작업 부하에 따라 최적의 코어를 선택적으로 사용하는 프로세서 아키텍처**다.

### 💡 비유: 스포츠 팀

이종 멀티코어는 **스포츠 팀**과 같다. 선수가 다 똑같지 않다. 키 큰 선수가 있다. 빠른 선수가 있다. 힘 센 선수가 있다. 상황에 따라 교체. 공격, 수비, 특수 팀. 이종 멀티코어가 똑같다. 성능 코어, 효율 코어, 작업별 배치.

## Ⅱ. 이종 멀티코어 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                이종 멀티코어 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【이종 멀티코어 유형】                                                            │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. big.LITTLE (ARM):                                                   │ │
│  │     • Big: 고성능, 높은 전력                                        │ │
│  │     • LITTLE: 저성능, 낮은 전력                                      │ │
│  │     • 클러스터 마이그레이션 / 글로벌 태스킹                           │ │
│  │                                                               │ │
│  │  2. Intel Hybrid (x86):                                                 │ │
│  │     • P-core: Performance                                          │ │
│  │     • E-core: Efficiency                                           │ │
│  │     • Thread Director로 스케줄링                                   │ │
│  │                                                               │ │
│  │  3. GPGPU 통합:                                                         │ │
│  │     • CPU + GPU 코어                                                │ │
│  │     • AMD APU, Intel HD Graphics                                   │ │
│  │                                                               │ │
│  │  4. 가속기 통합:                                                         │ │
│  │     • CPU + 전용 가속기                                             │ │
│  │     • NPU, DSP, FPGA                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스케줄링 방식】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 클러스터 마이그레이션 (Cluster Migration):                             │ │
│  │     • 전체 작업을 big 또는 LITTLE로 이동                              │ │
│  │     • 구현 단순                                                    │ │
│  │     • 전환 비용 존재                                                │ │
│  │                                                               │ │
│  │  2. 글로벌 태스킹 (Global Task Scheduling):                               │ │
│  │     • 각 태스크를 개별적으로 적절한 코어에 배치                         │ │
│  │     • 세밀한 제어                                                  │ │
│  │     • 복잡한 스케줄러                                               │ │
│  │                                                               │ │
│  │  3. HMP (Heterogeneous Multi-Processing):                                │ │
│  │     • 모든 코어 동시 활성화                                          │ │
│  │     • 최대 유연성                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 에너지 효율: 작업별 최적 코어                                      │ │
│  │  • 성능: 높은 처리량                                                │ │
│  │  • 유연성: 다양한 워크로드 대응                                       │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 스케줄링 복잡: OS 변경 필요                                       │ │
│  │  • 캐시 일관성: 코어 간 차이                                         │ │
│  │  • 프로그래밍: 코어 인식 필요                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  【성능】                                                              │

│  │  ────         ───         ──────                                    │ │

│  │  전력         높음         낮음                                      │ │
│  │  클럭         높음         낮음                                      │ │
│  │  파이프라인    깊음         얕음                                      │ │
