# 394. CMP (Chip Multiprocessor)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단일 칩에 여러 프로세서 코어를 통합한 아키텍처
> 2. **가치**: 칩 면적 효율, 전력 효율, 스레드 병렬성
> 3. **융합**: 멀티코어, SMP, 캐시 계층, 인터커넥트

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

CMP(Chip Multiprocessor)는 **단일 실리콘 다이 위에 여러 개의 프로세서 코어를 구현하여, 칩 레벨에서 병렬 처리를 가능하게 하는 프로세서 설계 방식**이다. 멀티코어 프로세서의 학술적 명칭이다.

### 💡 비유: 아파트 단지

CMP는 **아파트 단지**와 같다. 단독주택이 아니다. 한 울타리 안에 여러 세대. 각 세대가 독립적이다. 공용 시설도 있다. 주차장, 놀이터. CMP가 똑같다. 한 칩, 여러 코어, 독립 실행, 공유 자원.

## Ⅱ. CMP 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                CMP 상세                                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CMP 설계 유형】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 동질형 CMP (Homogeneous CMP):                                        │ │
│  │     • 동일한 코어 N개                                                │ │
│  │     • Intel Core i7, AMD Ryzen                                      │ │
│  │     • 프로그래밍 단순                                                │ │
│  │                                                               │ │
│  │  2. 이종 CMP (Heterogeneous CMP):                                        │ │
│  │     • 서로 다른 코어 혼합                                            │ │
│  │     • ARM big.LITTLE, Intel Lakefield                               │ │
│  │     • 전력/성능 최적화                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【캐시 구성】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  구성 방식:                                                                  │ │
│  │  • 사설 L2: 각 코어 전용, 빠른 접근                                    │ │
│  │  • 공유 L2: 코어 쌍 또는 전체 공유                                      │ │
│  │  • 공유 L3: 모든 코어 공유 (LLC)                                       │ │
│  │                                                               │ │
│  │  트레이드오프:                                                                │ │
│  │  • 사설: 낮은 지연, 높은 용량 비효율                                   │ │
│  │  • 공유: 높은 용량 효율, 경합 가능                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 칩 면적 효율: 코어 복제                                       │ │
│  │  • 전력 효율: 낮은 클럭 활용                                      │ │
│  │  • 스레드 병렬성: TLP 활용                                        │ │
│  │  • 설계 복잡도: 단순 코어 복제                                     │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • ILP 제한: 단일 스레드 성능                                      │ │
│  │  • 캐시 일관성: 오버헤드                                          │ │
│  │  • 메모리 대역폭: 병목                                             │ │
│  │  • 소프트웨어: 병렬화 필요                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• CMP: 칩 멀티프로세서
• 본질: 단일 칩에 여러 코어
• 유형: 동질형, 이종형
• 캐시: 사설 L1/L2, 공유 L3
• 인터커넥트: 버스, 링, 메시
• 장점: 면적/전력 효율
• 단점: 병렬 소프트웨어 필요
• 관계: CMP ⊂ Multicore ⊂ SMP
• 설계: 코어 수, 타입, 캐시
• 트렌드: 이종 CMP 증가
```
