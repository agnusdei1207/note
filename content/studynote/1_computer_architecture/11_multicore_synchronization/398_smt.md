# 398. SMT (Simultaneous Multithreading)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단일 코어에서 여러 스레드 명령어를 동시에 발행하는 기술
> 2. **가치**: 파이프라인 활용률 극대화, 처리량 15-40% 향상
> 3. **융합**: 하이퍼스레딩, 발행 슬롯, 레지스터 중복

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

SMT(Simultaneous Multithreading)는 **단일 프로세서 코어에서 여러 스레드의 명령어를 같은 클럭 사이클에 동시에 페치, 발행, 실행하여 파이프라인 자원 활용률을 극대화하는 하드웨어 멀티스레딩 기술**이다.

### 💡 비유: 식당 주방

SMT는 **식당 주방**과 같다. 요리사가 한 명이다. 하지만 여러 요리 동시 진행. 파를 썰면서 국을 끓인다. 프라이팬은 볶는다. 동시에 여러 작업. 자원은 공유. SMT가 똑같다. 한 코어, 여러 스레드, 동시 실행, 자원 공유.

## Ⅳ. 실무 적용

### SMT 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                SMT 분석                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SMT 상태 확인】                                                              │)
│  ──────────────────                                                  │
│  $ lscpu | grep -i thread                                           │
│  Thread(s) per core: 2                                              │
│                                                                     │
│  # 논리 CPU 확인                                                     │
│  $ nproc                                                            │
│  8  # 4 물리 코어 × 2 SMT                                           │
│                                                                     │
│  【SMT 제어】                                                                   │)
│  ──────────────────                                                  │
│  # SMT 끄기                                                          │
│  $ echo off > /sys/devices/system/cpu/smt/control                   │
│                                                                     │
│  # SMT 켜기                                                          │
│  $ echo on > /sys/devices/system/cpu/smt/control                    │
│                                                                     │
│  # SMT 상태 확인                                                     │
│  $ cat /sys/devices/system/cpu/smt/control                          │
│                                                                     │
│  【성능 측정】                                                                   │)
│  ──────────────────                                                  │
│  # SMT on/off 비교                                                   │
│  $ perf stat -a ./workload                                          │
│                                                                     │
│  # 코어별 IPC                                                        │
│  $ perf stat -e instructions,cycles -a                              │
│                                                                     │
│  【SMT 최적화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. 워크로드: 메모리 집약에 유리                                       │
│  2. 스레드 수: SMT 스레드 고려                                        │
│  3. 보안: 민감한 작업은 SMT 끄기                                       │
│  4. 친화성: 형제 스레드 인식                                           │
│  5. 측정: on/off 비교                                                 │
│  6. 캐시: 경합 최소화                                                 │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• SMT: 동시 멀티스레딩
• 동작: 여러 스레드 명령어 동시 발행
• 중복 자원: 레지스터, PC, ROB
• 공유 자원: 파이프라인, 캐시, 실행 유닛
• 성능: 처리량 15-40% 향상
• 장점: 자원 활용, 전력 효율
• 단점: 캐시 경합, 보안 이슈
• 구현: Intel HT, AMD SMT
• 스케줄링: OS SMT 인식
• 완화: 보안을 위해 SMT 끄기
```
