# 404. 메모리 일관성 모델 (Memory Consistency Model)

#### 핵심 인사이트 (3줄 요약)

> 2. **가치**: 프로그래머와 하드웨어 간의 계약, 최적화와 정확성의 균형

## Ⅰ. 개요 (Context & Background)

### 개념 정의

메모리 일관성 모델(Memory Consistency Model)은 **멀티프로세서 시스템에서 여러 코어가 수행한 메모리 연산(읽기/쓰기)이 다른 코어에 어떤 순서로 관찰되는지 정의하는 규칙 집합**이다. 프로그래머와 하드웨어 간의 계약이다.

### 💡 비유: 메시지 전달 순서

메모리 일관성 모델은 **메시지 전달 순서**와 같다. 편지를 보낸다. 순서대로 쓴다. 하지만 받는 사람은? 순서대로 올까? 늦게 보낸 게 먼저? 규칙이 필요하다. "보낸 순서대로 받는다"면 강하다. "아무렇게나"면 약하다. 메모리 모델이 똑같다. 쓰기 순서, 읽기 순서, 가시성 규칙.

### 메모리 일관성 모델 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 일관성 모델 개요                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【일관성 모델 스펙트럼】                                                          │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  강한 모델                                     약한 모델              │ │
│  │  ←─────────────────────────────────────────────────────→      │ │
│  │                                                               │ │
│  │  순차 일관성    TSO        PSO        릴리스      ARM         │ │
│  │  (SC)                              일관성      POWER         │ │
│  │                                                               │ │
│  │  장점:            ↑                                          │ │
│  │  • 이해 쉬움       │                                          │ │
│  │  • 디버깅 쉬움     │    장점:                                 │ │
│  │                   │    • 최적화 자유                          │ │
│  │                   │    • 성능 향상                            │ │
│  │  단점:            │                                          │ │
│  │  • 성능 제한      │    단점:                                  │ │
│  │  • 최적화 어려움  │    • 이해 어려움                          │ │
│  │                   │    • 버그 발생 쉬움                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주요 일관성 모델】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 순차 일관성 (Sequential Consistency, SC):                             │ │
│  │     • 모든 연산이 프로그램 순서대로 실행                                  │ │
│  │     • 가장 직관적, 성능 제한                                             │ │
│  │                                                               │ │
│  │  2. 전체 저장 순서 (Total Store Order, TSO):                              │ │
│  │     • x86, 스토어 버퍼 허용                                             │ │
│  │     • 쓰기-뒤-읽기 재배치만 허용                                         │ │
│  │                                                               │ │
│  │  3. 약한 일관성 (Weak Consistency):                                       │ │
│  │     • ARM, POWER, C++ atomic                                            │ │
│  │     • 명시적 동기화 필요                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 메모리 일관성 모델 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 일관성 모델 상세                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【순차 일관성 (SC)】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  규칙:                                                                   │ │
│  │  1. 각 코어의 연산은 프로그램 순서 유지                                  │ │
│  │  2. 모든 코어가 같은 순서로 관찰                                        │ │
│  │                                                               │ │
│  │  예: Core 0: A=1, B=1                                              │ │
│  │      Core 1: r1=B, r2=A                                            │ │
│  │                                                               │ │
│  │  SC에서는 r1=1, r2=0 불가 (A=1이 먼저여서)                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【TSO (x86)】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  허용되는 재배치:                                                          │ │
│  │  • Store → Load (쓰기 뒤 읽기)                                        │ │
│  │                                                               │ │
│  │  금지되는 재배치:                                                          │ │
│  │  • Store → Store                                                     │ │
│  │  • Load → Load                                                       │ │
│  │  • Load → Store                                                      │ │
│  │                                                               │ │
│  │  이유: 스토어 버퍼 (Store Buffer)                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【약한 일관성 (ARM, C++)】                                                       │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  릴리스-획득 semantics:                                                   │ │
│  │  • Release: 이전 연산이 완료됨을 보장                                    │ │
│  │  • Acquire: 이후 연산이 시작되기 전 완료                                 │ │
│  │                                                               │ │
│  │  C++ 메모리 순서:                                                          │ │
│  │  • memory_order_relaxed: 순서 보장 없음                                 │ │
│  │  • memory_order_acquire: 읽기 획득                                     │ │
│  │  • memory_order_release: 쓰기 릴리스                                   │ │
│  │  • memory_order_seq_cst: 순차 일관성                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 메모리 일관성 모델 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                메모리 일관성 모델 분석                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ atomic 예시】                                                              │)
│  ──────────────────                                                  │
│  // 릴리스-획득                                                      │
│  std::atomic<int> data;                                             │
│  std::atomic<bool> ready{false};                                    │
│                                                                     │
│  // 쓰기 스레드                                                       │
│  data.store(42, std::memory_order_release);                         │
│  ready.store(true, std::memory_order_release);                      │
│                                                                     │
│  // 읽기 스레드                                                       │
│  while (!ready.load(std::memory_order_acquire));                    │
│  int value = data.load(std::memory_order_acquire);                  │
│                                                                     │
│  【메모리 배리어】                                                              │)
│  ──────────────────                                                  │
│  // 전체 배리어                                                       │
│  std::atomic_thread_fence(std::memory_order_seq_cst);               │
│                                                                     │
│  // Linux 커널                                                        │
│  smp_mb();   // 전체 배리어                                           │
│  smp_rmb();  // 읽기 배리어                                           │
│  smp_wmb();  // 쓰기 배리어                                           │
│                                                                     │
│  【아키텍처 확인】                                                              │)
│  ──────────────────                                                  │
│  $ uname -m                                                         │
│  x86_64   # TSO                                                     │
│  aarch64  # Weak                                                    │
│                                                                     │
│  【메모리 모델 최적화 팁】                                                          │)
│  ──────────────────                                                  │
│  1. 이해: 타겟 아키텍처 모델                                           │
│  2. 라이브러리: 검증된 동시성 라이브러리                                 │
│  3. 테스트: TSan, ThreadSanitizer                                    │
│  4. 최소화: 공유 상태                                                 │
│  5. 명시: memory_order 지정                                           │
│  6. 리뷰: 동시성 코드 집중 리뷰                                        │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 메모리 일관성: 연산 순서 가시성 규칙
• SC: 순차 일관성, 가장 강함
• TSO: x86, 스토어 버퍼 허용
• Weak: ARM, 재배치 허용
• 릴리스-획득: 동기화 지점
• 메모리 배리어: 순서 강제
• C++: memory_order 옵션
• 강한 모델: 이해 쉬움, 성능 낮음
• 약한 모델: 성능 높음, 버그 위험
• 선택: 정확성 vs 성능
```
