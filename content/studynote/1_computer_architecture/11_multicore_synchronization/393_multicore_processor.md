# 393. 멀티코어 프로세서 (Multicore Processor)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단일 칩에 여러 CPU 코어를 통합한 프로세서
> 2. **가치**: 전력 효율, 병렬 성능, 프로그램 호환성
> 3. **융합**: CMP, SMP, 캐시 일관성, 스레드 병렬성

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

멀티코어 프로세서(Multicore Processor)는 **단일 실리콘 다이 또는 패키지에 두 개 이상의 독립적인 CPU 코어를 통합하여, 병렬 처리를 통해 성능을 향상시키는 프로세서 아키텍처**다. 전력 효율과 성능의 균형을 추구한다.

### 💡 비유: 협동 가게

멀티코어 프로세서는 **협동 가게**와 같다. 점원 한 명이 아니다. 여러 점원이 있다. 각각 독립적으로 일한다. 동시에 여러 손님 응대. 서로 협력도 한다. 멀티코어가 똑같다. 여러 코어, 독립 실행, 병렬 처리, 협력 가능.

## Ⅱ. 멀티코어 프로세서 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                멀티코어 프로세서 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【캐시 구조】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 사설 캐시 (Private Cache):                                           │ │
│  │     • L1 명령어/데이터 캐시 (각 코어 전용)                          │ │
│  │     • L2 캐시 (코어 전용 또는 코어 쌍 공유)                          │ │
│  │     • 빠른 접근, 낮은 지연                                        │ │
│  │                                                               │ │
│  │  2. 공유 캐시 (Shared Cache):                                            │ │
│  │     • L3 캐시 (Last Level Cache, LLC)                              │ │
│  │     • 모든 코어가 공유                                            │ │
│  │     • 코어 간 데이터 공유                                         │ │
│  │                                                               │ │
│  │  3. 캐시 일관성 (Cache Coherence):                                       │ │
│  │     • 여러 코어의 캐시 내용 일치                                   │ │
│  │     • MESI, MOESI 프로토콜                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【인터커넥트】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  코어 간 통신 구조:                                                        │ │
│  │  • 버스 (Bus): 간단하지만 확장성 제한                               │ │
│  │  • 링 (Ring): Intel, 순차 전달                                    │ │
│  │  • 메시 (Mesh): AMD Zen, 확장성 좋음                               │ │
│  │  • 크로스바: 높은 대역폭, 비용 증가                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 전력 효율: 낮은 클럭으로 동일 성능                                │ │
│  │  • 병렬 성능: 멀티스레드 응용 최적                                   │ │
│  │  • 프로그램 호환: 기존 소프트웨어 실행                               │ │
│  │  • 제조 효율: 코어 복제로 설계 용이                                  │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 병렬 소프트웨어 필요                                             │ │
│  │  • 캐시 일관성 오버헤드                                             │ │
│  │  • 코어 간 통신 비용                                                │ │
│  │  • 메모리 대역폭 경합                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 멀티코어: 단일 칩에 여러 CPU 코어
• 등장 배경: 전력 벽, ILP 한계
• 구조: 사설 L1/L2 + 공유 L3
• 인터커넥트: 버스, 링, 메시
• 장점: 전력 효율, 병렬 성능
• 단점: 병렬 소프트웨어 필요
• 캐시 일관성: MESI, MOESI
• 활용: 데스크톱~슈퍼컴퓨터
• OS: 스케줄링, 동기화
• 트렌드: 코어 수 지속 증가
```
