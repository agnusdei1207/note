# 400. 캐시 일관성 (Cache Coherence)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 멀티코어 시스템에서 모든 코어의 캐시 데이터 일치 보장
> 2. **가치**: 데이터 정합성, 프로그래밍 모델 단순화
> 3. **융합**: MESI, 스누핑, 디렉터리, 일관성 트래픽

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

캐시 일관성(Cache Coherence)은 **멀티프로세서 시스템에서 여러 코어의 캐시에 저장된 동일 메모리 주소의 데이터가 항상 일관된 값을 갖도록 보장하는 하드웨어 메커니즘**이다. 캐시된 데이터의 수정 사항을 모든 코어에 전파한다.

### 💡 비유: 화이트보드 복사

캐시 일관성은 **화이트보드 복사**와 같다. 여러 교실이 있다. 각 교실에 화이트보드. 중앙 내용을 복사했다. 누가 지우고 쓴다. 다른 교실은 모른다. 문제다. 알려줘야 한다. "바뀌었어!" 캐시 일관성이 똑같다. 각 코어 캐시, 수정 전파, 일관성 유지.

### 캐시 일관성 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                캐시 일관성 개요                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【캐시 일관성 문제】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  시나리오: 변수 X = 10 (메모리)                                         │ │
│  │                                                               │ │
│  │  1. Core 0: X를 읽어 캐시에 저장 (X = 10)                              │ │
│  │  2. Core 1: X를 읽어 캐시에 저장 (X = 10)                              │ │
│  │  3. Core 0: X를 20으로 수정 (자신의 캐시만 변경)                         │ │
│  │  4. Core 1: X를 읽음 (여전히 10!) ← 일관성 위반                         │ │
│  │                                                               │ │
│  │       ┌───────────┐         ┌───────────┐                      │ │
│  │       │  Core 0   │         │  Core 1   │                      │ │
│  │       │ ┌───────┐ │         │ ┌───────┐ │                      │ │
│  │       │ │ X=20  │ │         │ │ X=10  │ │  ← 불일치!           │ │
│  │       │ │(Dirty)│ │         │ │(Stale)│ │                      │ │
│  │       │ └───────┘ │         │ └───────┘ │                      │ │
│  │       └───────────┘         └───────────┘                      │ │
│  │              │                    │                            │ │
│  │              └────────┬───────────┘                            │ │
│  │                       ▼                                        │ │
│  │              ┌─────────────────┐                               │ │
│  │              │   메모리: X=10   │                               │ │
│  │              └─────────────────┘                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【일관성 보장 방식】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Write-Through: 쓰기 시 즉시 메모리 반영                            │ │
│  │  2. Write-Back: 캐시에서 수정, 추후 메모리 반영                         │ │
│  │  3. Write-Invalidate: 수정 시 다른 캐시 무효화                         │ │
│  │  4. Write-Update: 수정 시 다른 캐시 갱신                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 캐시 일관성 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                캐시 일관성 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【일관성 프로토콜】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 스누핑 프로토콜 (Snooping Protocol):                                  │ │
│  │     • 모든 코어가 버스 모니터링                                        │ │
│  │     • 쓰기 시 무효화/갱신 브로드캐스트                                  │ │
│  │     • 구현 단순                                                    │ │
│  │     • 확장성 제한 (버스 대역폭)                                        │ │
│  │     • 예: MESI, MOESI, Dragon                                      │ │
│  │                                                               │ │
│  │  2. 디렉터리 기반 프로토콜 (Directory-based):                               │ │
│  │     • 중앙 디렉터리가 캐시 상태 추적                                    │ │
│  │     • 점대점(point-to-point) 통신                                    │ │
│  │     • 확장성 좋음                                                    │ │
│  │     • 구현 복잡                                                    │ │
│  │     • 대규모 시스템에 적합                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MESI 프로토콜 상태】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  상태    의미            특징                                          │ │
│  │  ────    ────            ────                                          │ │
│  │  M       Modified        수정됨, 이 캐시만 소유, 메모리와 다름           │ │
│  │  E       Exclusive       독점, 메모리와 동일, 수정 가능                  │ │
│  │  S       Shared          공유, 여러 캐시, 메모리와 동일                  │ │
│  │  I       Invalid         무효, 사용 불가                                │ │
│  │                                                               │ │
│  │  상태 전이:                                                                   │ │
│  │  • Read Hit: S → S, E → E, M → M                                   │ │
│  │  • Read Miss: I → S (다른 있음) or I → E (없음)                      │ │
│  │  • Write Hit: S → M, E → M, M → M                                  │ │
│  │  • Write Miss: I → M                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 데이터 일관성: 항상 최신 값 보장                                     │ │
│  │  • 프로그래밍: 단순화 (순차 일관성)                                     │ │
│  │  • 투명성: 소프트웨어 변경 불필요                                       │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 오버헤드: 일관성 트래픽                                             │ │
│  │  • 확장성: 코어 증가 시 병목                                            │ │
│  │  • 전력: 트래픽에 따른 소모                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 캐시 일관성 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                캐시 일관성 분석                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【캐시 일관성 미스 확인】                                                          │)
│  ──────────────────                                                  │
│  $ perf stat -e cache-references,cache-misses ./program            │
│  # 캐시 미스율 분석                                                   │

│  $ perf stat -e LLC-loads,LLC-load-misses                          │
│                                                                     │
│  【거짓 공유 확인】                                                              │)
│  ──────────────────                                                  │
│  # perf로 캐시 라인별 미스 분석                                       │
│  $ perf c2c record ./program                                       │
│  $ perf c2c report                                                 │
│                                                                     │
│  【NUMA 시스템】                                                                 │)
│  ──────────────────                                                  │
│  # NUMA 노드별 메모리                                                │
│  $ numactl --hardware                                               │
│                                                                     │
│  # 디렉터리 기반 일관성 (대규모)                                        │
│  # Intel Xeon, AMD EPYC                                             │
│                                                                     │
│  【캐시 일관성 최적화 팁】                                                          │)
│  ──────────────────                                                  │
│  1. 데이터 배치: 캐시 라인 정렬                                       │
│  2. 거짓 공유: 패딩, 분리                                             │
│  3. 공유 최소화: 스레드 로컬                                          │
│  4. 읽기 많음: RCU, 읽기 복사                                         │
│  5. 쓰기 많음: 분산, 파티션                                           │
│  6. 측정: coherence 트래픽                                           │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 캐시 일관성: 여러 캐시의 데이터 일치

• 프로토콜: MESI, MOESI, Dragon
• 상태: Modified, Exclusive, Shared, Invalid
• 무효화: Write-Invalidate
• 갱신: Write-Update
• 장점: 데이터 정합성
• 단점: 일관성 트래픽
• 최적화: 거짓 공유 방지
```
