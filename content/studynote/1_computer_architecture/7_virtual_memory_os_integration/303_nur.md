# 303. NUR (Not Used Recently)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: NUR(Not Used Recently)은 "정확히 가장 오래된 1명을 찾는 것"을 포기하고, "대략적으로 최근에 안 쓴 놈만 걸러내자"는 실용주의적 페이지 교체 철학이다.
> 2. **가치**: 각 페이지에 참조 비트(R)와 수정 비트(M) 2비트만으로 4가지 그룹으로 분류하여, 적은 오버헤드로 LRU에 근접한 성능을 달성한다.
> 3. **융합**: NUR은 이론적 이름이고, 이 철학을 원형 큐로実装したものが Clock Algorithm이며, 모든 현대 OS의 표준 페이지 교체 기반이 되었다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

정확한 LRU는 아름답지만, 수백만 페이지를 관리하는 현실에서는 그 오버헤드가 감당하기 어렵다.

**문제의 소급**: "가장 오래된 1등을 정확히 찾아 퇴학시키려면" 전체를 줄 세워야 한다. 400만 명의 학생을 줄 세우는데 1시간이 걸린다면 그 사이 수업은 진행되지 않는다.

**NUR의 해결**: "그냥 最近使われていない 候補者 중 아무나 잡아 퇴학시키면 되지 않나?"

- **비유**: 전교ept를 찾기 위해 100만 명을 줄 세우기보다는, "50점 이하인 학생들"만抽出してその中에서ランダムに1명を選ぶ方が 빠르다.

- **등장 배경**: 1960년대 가상 메모리 연구에서 등장했으며, 현재는 NUR의 철학을 реализован한 Clock Algorithm의 이론적 기반으로 불린다. 실질적으로 NUR과 Clock은 동의어로 사용된다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 2비트로 만드는 4가지 그룹

```text
  ┌────────────────────────────────────────────────────────────────────────────┐
  │              NUR의 4가지 우선순위 그룹                                             │
  ├────────────────────────────────────────────────────────────────────────────┤
  │
  │  [비트 조합과 의미]                                                         │
  │                                                                            │
  │  (R=0, M=0): 最近未使用 + 改変なし                                        │
  │       → 가장 쓸모없음. 즉석에서 버려도Disk I/O 불필요                          │
  │       → **1순위 Victim**                                                     │
  │                                                                            │
  │  (R=0, M=1): 最近未使用 + 改変あり                                        │
  │       → 버리려면 Disk에 먼저 기록해야 함                                     │
  │       → **2순위 Victim** (I/O 비용 고려)                                    │
  │                                                                            │
  │  (R=1, M=0): 最近使用 + 改変なし                                           │
  │       → 한 번 봐주고 R을 0으로 변경                                          │
  │       → **3순위** (기회 부여)                                               │
  │                                                                            │
  │  (R=1, M=1): 最近使用 + 改変あり                                           │
  │       → 가장 소중한 페이지. 절대轻易에 버리지 않음                              │
  │       → **4순위** (최고 우선순위 유지)                                       │
  └────────────────────────────────────────────────────────────────────────────┘
```

### NUR과 Clock의 관계

```text
  ┌────────────────────────────────────────────────────────────────────────────┐
  │              NUR (理論) → Clock (実装) 의 변환                               │
  ├────────────────────────────────────────────────────────────────────────────┤
  │
  │  [NUR 이론]                                                                │
  │       - "최근에 안 쓴 페이지를 대충 찾아내자"                                 │
  │       - 2비트만으로 판단                                                     │
  │                                                                            │
  │              ▼ 実装変換 ▼                                                   │
  │                                                                            │
  │  [Clock 구현]                                                              │
  │       - 페이지들을 원형 큐로 배치                                             │
  │       - 포인터(시곗바늘)가 빙글빙글 돌며 R 비트를 확인                           │
  │       - R=0인 페이지를 발견하면 M 비트를 확인하여 Victim 결정                   │
  │       - R=1인 페이지는 R=0으로 변경 후 다음으로                               │
  └────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** NUR은 이론적 개념으로 "R과 M 두 비트만으로 最近使われていない ページを 대충 찾자"는 발상이다. Clock은 이 이론을 원형 큐 구조로実現したものである。포인터가 돌아다니며 R 비트를 1→0로 변경하면서 "기회(Second Chance)"를 부여하고, R=0인 페이지를 찾으면 그 페이지의 M 비트에 따라 즉시 버릴지 또는 Disk I/O 후 버릴지를 결정한다.

- **섹션 요약 비유**: NUR은 "50점 이하인 학생을 퇴출시키자"는 학급 경영 전략이고, Clock은 이 전략을 운동장을 돌며 실제로実行하는 방법론이다. 전략(Concept)과 실행(Implementation)의 차이가 있다.

## Ⅲ. 융합 비교 및 다각도 분석

### NUR vs LRU: 정확성 vs 비용

| 구분 | LRU | NUR (Clock) |
|:---|:---|:---|
| **판단 기준** | 정확한 시간 순서 | 대략적인 그룹 분류 |
| **필요 정보** | 모든 페이지의 참조 시간 | R, M 2비트 |
| **구현 비용** | 수백만 시간 기록 + 정렬 | 포인터 이동만 |
| **성능 차이** | 가장 높음 | LRU 대비 95~99% |
| **现实的选择** | 일부 특수 목적 | **OS 표준** |

### 4가지 그룹이 만드는 최적의 균형

```text
  ┌────────────────────────────────────────────────────────────────────────────┐
  │              NUR의 4순위가 만드는 효율성                                          │
  ├────────────────────────────────────────────────────────────────────────────┤
  │
  │  [왜 2비트면 충분한가?]                                                    │
  │                                                                            │
  │  - 1비트면: "썼는지 안 썼는지"만 알 수 있음 (R=0 또는 1)                      │
  │  - 2비트면: "썼는지 + 수정했는지"의 4가지 조합으로 판단 가능                      │
  │                                                                            │
  │  [4가지 조합의 실용성]                                                      │
  │       - (0,0): 即座に排除可能 (最高)                                          │
  │       - (0,1): 記録必要 (I/O 후排除)                                         │
  │       - (1,0) & (1,1): もう一度机会 (保護)                                   │
  │                                                                            │
  │  [결론]                                                                   │
  │       - 이 정도면 "가장 안 좋은 놈"을 高精度에 걸러낼 수 있음                    │
  │       - 그 이상의 정밀도는 성능 향상에 크게 기여하지 않음                         │
  └────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** NUR이 2비트만 사용하는 이유는 "가장 나쁜 놈을 찾는 것"에는 이 정도면 충분하기 때문이다. (0,0) 조합은 即座に排除할 수 있는 최적의 Victim이고, (0,1)은 I/O 비용이 발생하지만 회수 가능한 Victim이다. (1,0)과 (1,1)은 최근에 사용되었으므로 protection受体이다. 이 정도로도 "안 좋은 페이지"를 걸러내는 데 충분한 것이다.

- **섹션 요약 비유**: 100점 만점의 시험을 치르려면 출제, 채점, 분석에 10시간이 걸리지만, 60점 이하만 가려내려면 시험지를 대충 훑어보며 60점 이하만 분류하면 30분이면 충분하다. NUR은 정확히 后者の方法论이다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **현대 OS의 표준**
   Windows, Linux, macOS, Android, iOS 모두ページ交换에 Clock/NUR 계열 알고리즘을使用한다. 2비트만으로 구현 가능하여 하드웨어 부담이 거의 없고, 성능은 완벽한 LRU 대비 95% 이상이다.

2. **TLB와連携**: NUR/Clock은 가상 주소에서 물리 주소로の変換을管理하는 TLB와 밀접한 관련이 있다. TLB 미스 시에도 Clock 계열 알고리즘이 페이지를 회수하여 빈 프레임을 확보한다.

### 도입 체크리스트
- **기술적**: 페이지 폴트율과 구현 복잡도 사이의 트레이드오프를 분석했는가? 대부분의 시스템에서 NUR(Clock)은 최적의 선택이다.

### 안티패턴
- **과도한 Disk I/O**: 수정된 페이지(0,1)가 많을 때, Clock이 Victim을 찾지 못하고 여러 바퀴를 돌면 Disk I/O가 과도하게 발생하여 시스템이 버벅거릴 수 있다.

- **섹션 요약 비유**: "문제아만 퇴출"이라는 규칙(1순위)이 있지만, 问题아 중에서도 짐을 잔뜩 가진 学生(수정된 페이지)를 퇴출시키려면 짐을 정리하는 시간이 필요하므로, 짐 없는 问题아를 먼저 퇴출시키는 것이 효율적이다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | LRU | NUR (Clock) | 개선 효과 |
|:---|:---|:---|:---|
| **정밀도** | 100% (시간 기록) | ~95% (그룹 분류) | 미미한 차이 |
| **实现成本** | 수백만 정렬 | 1비트 + 포인터 | **비용 절감** |
| **적정성** | 과잉 기능 | 적정 기능 | **현실적** |
| **현실 사용** | 특수 목적 | **표준** | - |

### 미래 전망
- **적응형 NUR**: 프로그램의 상태(배치중, I/O 대기 등)에 따라 4순위의 가중치를 동적으로 조절하는 적응형 알고리즘이 연구되고 있다.

- **섹션 요약 비유**: NUR(Clock)은 "完美より실용"의哲学이다. 100점을 위해 100시간投资하는 대신, 95점을 1시간에 달성하는 것이 더 현명하다.
