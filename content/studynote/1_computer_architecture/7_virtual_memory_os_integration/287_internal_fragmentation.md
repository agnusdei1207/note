> **핵심 인사이트**
> 1. 내부 단편화(Internal Fragmentation)는 페이징(Paging) 기법처럼 고정된 크기 단위로 메모리를 할당할 때, **할당된 공간 안에 다 채우지 못한 잉여 공간이 발생하여 버려지는 메모리 낭비 현상**이다.
> 2. 주로 프로세스의 마지막 페이지를 할당할 때 발생하며, 페이지 크기가 클수록 버려지는 공간(내부 단편화)도 커지는 트레이드오프(Trade-off)가 존재한다.
> 3. 과거 메모리가 귀했던 시절에는 큰 골칫거리였으나, 기가바이트(GB) 단위의 **현대 메모리 환경에서는 감수할 만한 아주 미미한 수준의 낭비로 취급**된다.

---

## Ⅰ. 내부 단편화의 발생 원리

페이징 시스템의 핵심 규칙은 "무조건 정해진 빵틀(예: 4KB 페이지) 크기로만 잘라서 준다"는 것입니다. 

- **상황 가정:** 
  어떤 프로그램의 전체 크기가 17KB입니다. OS가 이 프로그램을 메모리에 올리려고 4KB짜리 페이지로 자릅니다.
  - 1번 페이지: 4KB 꽉 참
  - 2번 페이지: 4KB 꽉 참
  - 3번 페이지: 4KB 꽉 참
  - 4번 페이지: 4KB 꽉 참 (여기까지 총 16KB)
  - **5번 페이지:** 남은 1KB를 담기 위해 4KB짜리 페이지 한 장을 온전히 다 씁니다.
- **낭비 발생:** 5번 페이지에는 1KB의 데이터가 있고, **나머지 3KB는 아무것도 없는 텅 빈 공간**이 됩니다. 하지만 이 페이지 전체가 해당 프로그램의 소유로 할당되었기 때문에, 다른 프로그램은 이 3KB의 빈 공간을 절대 쓸 수 없습니다. 이렇게 '내부에서 발생하는 조각'이 바로 내부 단편화입니다.

📢 **섹션 요약 비유:**
피자(메모리)를 무조건 한 판 단위로만 파는데, 내가 1.2판만 먹고 싶다면 어쩔 수 없이 2판을 결제하고 0.8판은 그냥 쓰레기통에 버려야 하는 억울한 상황입니다.

## Ⅱ. 외부 단편화(External)와의 비교

메모리 단편화는 내부와 외부 두 가지로 나뉩니다. 이 둘은 메모리 관리 철학의 양극단입니다.

| 비교 항목 | 내부 단편화 (Internal) | 외부 단편화 (External) |
| :--- | :--- | :--- |
| **발생 원인** | 메모리를 **고정 크기(페이징)**로 할당할 때 | 메모리를 **가변 크기(연속 할당/세그멘테이션)**로 할당할 때 |
| **낭비되는 위치** | 할당된 메모리 덩어리의 **내부** 끝부분 | 할당된 메모리 덩어리들 **사이(외부)**의 빈틈 |
| **낭비의 심각성** | 프로그램당 최대 (페이지 크기 - 1Byte)로 **매우 작음** | 메모리 전체에 구멍이 뚫려 **매우 심각함** |
| **현대의 취급** | **"이 정도 낭비는 쿨하게 무시하자."** | "무조건 없애야만 컴퓨터가 안 멈춘다." |

📢 **섹션 요약 비유:**
내부 단편화는 귤을 껍질째 버리는 작은 낭비라면, 외부 단편화는 냉장고 정리를 너무 못해서 빈자리가 많은데도 큰 수박 한 통을 못 넣는 심각한 공간 낭비입니다.

---

## Ⅲ. 페이지 크기와 내부 단편화의 딜레마

내부 단편화를 줄이려면 어떻게 해야 할까요? 페이지 크기를 1KB, 1Byte 단위로 아주 잘게 쪼개면 됩니다.

- **작은 페이지의 문제:**
  - 1Byte 단위로 쪼개면 내부 단편화는 0%가 되겠지만, 이 조각들이 물리 메모리 어디에 있는지 기록하는 **장부(페이지 테이블)의 두께가 하늘을 찌르게 됩니다.** 메모리를 아끼려다 장부 보관하느라 메모리를 더 쓰는 미친 짓이 됩니다.
- **Huge Page (대형 페이지)의 등장:**
  - 현대 서버 환경에서는 아예 페이지 크기를 2MB나 1GB 단위(Huge Page)로 엄청나게 키워버립니다.
  - 이렇게 하면 프로그램 마지막에 버려지는 내부 단편화 공간이 수백 KB ~ 수백 MB에 달할 정도로 엄청나게 커지지만, 페이지 테이블 크기가 줄어들고 TLB 적중률이 극대화되어 속도가 비교도 안 되게 빨라지기 때문에 **메모리를 버리고 속도를 취하는 사치스러운 선택**을 합니다.

📢 **섹션 요약 비유:**
가방을 살 때 딱 맞는 사이즈를 사려다 가방 개수가 수백 개가 되어 짐 싸다 지치느니, 그냥 엄청나게 큰 이민 가방 몇 개만 사서 안에 공간이 텅텅 비든 말든 대충 쑤셔 넣고 빨리 출발하는 게 낫다는 현대인의 선택입니다.
