# 데이터패스 (Datapath)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터패스(Datapath)는 CPU 내부에서 데이터가 이동하고 가공되는 **물리적인 경로이자 구성 요소들의 집합(ALU, 레지스터, 내부 버스 등)**이다.
> 2. **가치**: 명령어의 '실질적인 일'을 수행하는 공장 라인이며, 데이터패스의 넓이(버스 폭)와 부품의 속도(게이트 지연)가 CPU가 한 사이클에 처리할 수 있는 연산의 한계치를 결정한다.
> 3. **융합**: 제어 유닛(Control Unit)이 내리는 신호에 따라 스위치(MUX)가 열리고 닫히면서, 데이터패스는 명령어마다 요구되는 최적의 연산 회로를 순간적으로 구성한다.

---

## Ⅰ. 데이터패스의 개요

### 1. 정의
- CPU 내부에서 산술 논리 연산을 수행하고 데이터를 저장하거나 전달하기 위해 물리적으로 연결된 하드웨어 요소(ALU, 레지스터 파일, 멀티플렉서, 내부 버스)들의 전체 경로다.

### 2. 등장 배경: '데이터가 흐르는 길'
- 컴퓨터가 계산을 하려면 데이터가 레지스터에서 빠져나와 덧셈기를 거친 뒤 다시 레지스터로 들어가야 한다. 이 과정에서 선(Wire)이 복잡하게 꼬이면 신호 충돌이 발생한다. 따라서 질서정연하게 데이터가 흘러갈 수 있도록 '데이터 전용 고속도로'와 '교차로(MUX)'를 체계적으로 설계한 것이 데이터패스다.

### 3. 💡 비유: '정수장의 파이프망'
- 데이터패스는 **'물을 걸러내는 정수장의 거대한 파이프 시스템'**과 같다.
- 물(데이터)이 파이프(버스)를 타고 흘러가다가 밸브(멀티플렉서)의 조작에 따라 여과기(ALU)를 통과하기도 하고, 저수지(레지스터)에 잠깐 고여 있기도 하면서 최종적으로 깨끗한 물(연산 결과)로 만들어진다.

- **📢 섹션 요약 비유**: 작업자의 지시(제어 신호)에 따라 컨베이어 벨트가 움직이고 기계 팔이 작동하는 실제 조립 공장의 물리적 생산 라인입니다.

---

## Ⅱ. 핵심 구성 요소 및 구조 (Deep Dive)

데이터패스는 데이터를 담는 그릇, 계산하는 기계, 그리고 이들을 연결하는 길로 이루어져 있다.

### 1. 데이터패스의 4대 핵심 소자
| 구성 요소 | 역할 | 상세 구조 및 특징 |
|:---|:---|:---|
| **ALU (산술논리장치)** | 실제 연산(계산) 수행 | 덧셈, 뺄셈, AND, OR, Shift 연산 등 수행 |
| **레지스터 파일 (Register File)** | 데이터의 임시 보관 | 여러 개의 범용 레지스터가 모인 블록. 2개의 Read 포트와 1개의 Write 포트 제공 |
| **멀티플렉서 (MUX)** | 데이터의 이동 경로 선택 (교차로) | 여러 입력 중 제어 신호에 맞는 단 하나만 출력으로 내보냄 |
| **내부 버스 (Internal Bus)** | 소자 간의 물리적 연결 통로 | 데이터가 이동하는 구리선. 32비트 CPU는 버스 폭도 32가닥임 |

### 2. 명령어 실행에 따른 데이터패스 흐름 (ASCII)
```text
    [ R1 = R2 + R3 연산의 데이터 흐름 ]
    
    [ Register File ]
       ├── R2 ──(Data A)───┐
       └── R3 ──(Data B)───┤
                           ▼
                       ┌───┴───┐
      제어 신호 ──────▶│  MUX  │ (상황에 따라 상수(Immediate)가 들어올 수도 있음)
    (ALU_SRC)          └───┬───┘
                           │
             ┌─────────────┴─────────────┐
             │            ALU            │ ◀── 제어 신호 (ALU_OP = ADD)
             └─────────────┬─────────────┘
                           │ Result (A+B)
                           ▼
    [ Register File ] ◀────┘
       └── R1 (Write)
```

### 3. 심층 동작 원리: 단일 사이클 vs 다중 사이클
- **단일 사이클 데이터패스**: 모든 명령어가 무조건 가장 느린 명령어(예: 메모리 로드)의 시간에 맞춰 1클럭 내에 끝남. 자원 낭비가 심함.
- **다중 사이클 데이터패스**: 명령어를 여러 단계로 쪼개어, 단계별로 필요한 데이터패스만 사용. 파이프라이닝(Pipelining)의 근간이 되는 설계다.

- **📢 섹션 요약 비유**: 물건을 옮기고 조립하는 물리적 하드웨어 부품들이 톱니바퀴처럼 맞물려 돌아가는 기계적 실체입니다.

## Ⅲ. 데이터패스와 제어 유닛의 관계 (CPU의 양대 산맥)

| 비교 항목 | 데이터패스 (Datapath) | 제어 유닛 (Control Unit) |
|:---|:---|:---|
| **역할** | 데이터의 **연산과 이동** (근육) | 데이터패스의 **동작 지시** (두뇌) |
| **주요 구성품** | ALU, 레지스터, MUX, Adder | 디코더, 제어 논리 회로, 시퀀서 |
| **입력** | 실제 데이터 (Data) | 명령어의 Opcode (명령어 종류) |
| **출력** | 연산 결과 및 상태 플래그 | 제어 신호 (MUX 선택, ALU 연산 코드 등) |

- **📢 섹션 요약 비유**: 데이터패스가 무거운 짐을 나르는 '손과 발'이라면, 제어 유닛은 그 손발을 언제 어떻게 움직일지 지시하는 '신경망'입니다.

## Ⅳ. 실무 적용 및 최적화 판단 (Strategy)

### 1. 실무 시나리오: 커스텀 데이터패스 설계 (FPGA/ASIC)
- **상황**: 통신 장비에서 특정 암호화 알고리즘(AES)을 소프트웨어로 돌리면 처리량이 턱없이 부족함.
- **판단**: 하드웨어 엔지니어는 Verilog/VHDL을 사용하여 AES 연산에 특화된 **전용 데이터패스**를 설계한다. 일반 CPU는 덧셈기를 반복해서 써야 하지만, 전용 데이터패스는 암호화의 4단계(SubBytes, ShiftRows, MixColumns, AddRoundKey)를 물리적인 회로로 쭉 연결하여 파이프라인화함으로써, 한 클럭 사이클마다 암호화된 데이터를 쏟아내도록(Throughput 극대화) 아키텍처를 최적화한다.

### 2. 기술사적 인사이트: 크리티컬 패스 (Critical Path)와 클럭 한계
- **인사이트**: CPU의 클럭 주파수(예: 5GHz)는 데이터패스 내부에서 '가장 긴 지연 시간'을 가진 경로, 즉 **크리티컬 패스(Critical Path)**에 의해 결정된다. ALU 연산이나 캐시 접근 시간이 클럭 주기보다 길어지면 시스템이 오작동한다. 기술사는 파이프라인 단계를 깊게 쪼개어(Super-pipelining) 크리티컬 패스의 길이를 반으로 줄임으로써 클럭 속도를 높이는 하드웨어적 타협의 원리를 완벽히 이해해야 한다.

- **📢 섹션 요약 비유**: 달리기 릴레이 경주에서 전체 팀의 기록은 가장 느린 주자(크리티컬 패스)의 속도에 맞춰질 수밖에 없으므로, 그 주자의 달리는 구간을 짧게 잘라주는 최적화가 필요합니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **병렬성 확보**: 정수 연산 데이터패스와 부동소수점 데이터패스를 물리적으로 분리하여 동시 다발적 연산(슈퍼스칼라) 가능.
- **병목 지점 도출**: 회로 설계를 통해 시스템의 최대 동작 속도와 전력 소모의 한계점 명확화.

### 2. 미래 전망
과거의 데이터패스는 32비트, 64비트 등 고정된 크기였으나, 최근에는 512비트 폭의 SIMD 데이터패스와 AI 텐서 코어 전용 데이터패스 등 기능별로 거대해지고 다변화되고 있다. 향후에는 실리콘 포토닉스(광반도체) 기술이 도입되어, 구리선 내부 버스의 저항과 발열 문제를 빛(Photon)으로 대체함으로써 데이터패스의 전송 속도를 혁명적으로 끌어올리는 차세대 인터커넥트 혁신이 일어날 것이다.

- **📢 섹션 요약 비유**: 좁은 2차선 도로가 8차선 고속도로로, 나아가 빛의 속도로 달리는 하이퍼루프로 끊임없이 확장되는 데이터의 이동 경로입니다.
