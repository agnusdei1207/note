> **핵심 인사이트**
> 1. 해저드 감지 유닛(Hazard Detection Unit)은 파이프라인 CPU 내부에 하드웨어적으로 심어져, **명령어들 사이의 데이터 꼬임이나 충돌(해저드)을 실시간으로 감시하고 멱살을 잡아 멈춰 세우는 '안전 통제실'**이다.
> 2. 주로 Decode(해독) 단계에 위치하여, 전방 전달(Forwarding) 회로로도 막을 수 없는 최악의 의존성인 **'Load-Use Data Hazard (메모리에서 뽑아오기도 전에 쓰려는 상황)'**를 감지하는 역할을 전담한다.
> 3. 위기가 감지되면 PC와 IF/ID 레지스터의 값을 꽁꽁 얼려(Freeze) 뒤따라오는 명령어들을 멈춰 세우고, 실행부(EX) 쪽으로는 아무것도 하지 마라(NOP)는 **거품(Bubble) 신호를 쏴서 강제로 1클럭을 대기(Stall)시키는 완벽한 안전 장치**이다.

---

## Ⅰ. 지름길(Forwarding)도 소용없는 절대 위기
파이프라인의 웬만한 데이터 꼬임(의존성)은 덧셈기에서 답이 나오자마자 전선을 타고 넘겨주는 '전방 전달(Forwarding)' 기술로 전부 막을 수 있었습니다.

하지만 유일하게 방어가 안 되는 치명적 예외가 바로 `LOAD` 명령어입니다.
* 1번 명령: `LOAD R1, [메모리 100번지]` (R1에 데이터를 가져와라. **이 데이터는 4단계 Memory 구역 끝에 가야 튀어나옴!**)
* 2번 명령: `ADD R2, R1, 5` (R1에 5를 더해라. **이 연산은 3단계 Execute 구역에서 당장 R1이 필요함!**)

2번 명령어는 3구역(E)에서 당장 R1을 썰어야 하는데, 1번 명령어는 아직 4구역(M)에서 밭을 파고(메모리 읽기) 있습니다. 아직 뽑히지도 않은 무(데이터)를 아무리 포워딩 회로로 쏴주려 해도, 미래(4구역 끝)에서 과거(3구역 초입)로 타임머신을 탈 수는 없습니다. 
이 **Load-Use 해저드**가 터지는 것을 막기 위해 '해저드 감지 유닛'이라는 매의 눈이 탄생했습니다.

```ascii
[Load-Use 해저드의 시간적 모순 (타임머신 불가)]

 LOAD : [F] [D] [E] [*M (여기서 비로소 데이터 튀어나옴!)*] [W]
                       \ (이쪽으로 던져주고 싶은데...)
                        X (물리적으로 시간을 거슬러야 하므로 불가능!)
                         \
 ADD  :     [F] [D] [*E (여기서 당장 필요함!)*] [M] [W]
```

📢 **섹션 요약 비유**: Load-Use 해저드는 '식당 주방의 재료 고갈'입니다. 요리사(ADD)가 도마(E 단계) 앞에서 고기를 썰려고 칼을 들었는데, 아직 배달부(LOAD)가 마트(M 단계)에서 고기를 사 오지도 않았습니다. 배달부가 오토바이를 아무리 빨리 몰아도(포워딩), 아직 사지도 않은 고기를 도마 위에 올릴 순 없는 우주의 절대 법칙입니다.

## Ⅱ. 매의 눈과 브레이크 레버: 인터록(Interlock) 발동
해저드 감지 유닛은 파이프라인의 두 번째 방인 **Decode(해독) 구역** 천장에 달려있습니다.
방금 들어온 2번 명령어(ADD)를 해독하면서, 바로 앞방(Execute 구역)으로 넘어간 1번 명령어의 상태를 째려봅니다.

감지 조건은 아주 단순하고 확실합니다.
1. "앞방에 간 놈이 `LOAD` 명령어인가?" (YES)
2. "그놈이 데이터를 넣을 목적지(R1)가, 지금 들어온 놈(ADD)이 썰려고 하는 재료(R1)와 똑같은가?" (YES)

두 조건이 100% 맞아떨어지면, 해저드 감지 유닛은 즉시 파이프라인에 **긴급 정지(인터록, Interlock)** 브레이크를 걸어버립니다.

```ascii
[해저드 감지 유닛의 브레이크 작동 알고리즘]

 IF ( ID/EX.MemRead == 1 )   <-- 앞놈이 LOAD 명령어(메모리 읽기)인지 확인!
 AND
 ( ID/EX.RegisterRt == IF/ID.RegisterRs OR IF/ID.RegisterRt ) <-- 목적지와 내 재료가 겹치는지 확인!
 THEN
   => Stall(멈춤) 발동!! 
```

📢 **섹션 요약 비유**: 해저드 유닛은 지하철 톨게이트의 '부정 승차 센서'입니다. 앞사람(LOAD)이 아직 개찰구를 완전히 빠져나가지도 않았는데, 뒷사람(ADD)이 바짝 붙어서 통과하려고 하면 센서가 매의 눈으로 감지하고 개찰구 바(Bar)를 쾅! 잠가버려(Stall) 사고를 방지합니다.

## Ⅲ. 아키텍처 관점: 앞은 얼리고, 뒤는 거품(Bubble)으로 채워라
긴급 브레이크(Stall)가 발동하면 CPU 내부에서는 아주 정교한 '얼음-땡' 놀이가 시작됩니다.

1. **얼음! (Freeze)**: 
   해저드 유닛은 프로그램 카운터(PC)와 IF/ID 레지스터의 업데이트 전원을 팍 꺼버립니다. (Write Enable = 0). 
   이렇게 되면 2번 명령(ADD)은 Decode 방에서 다음 방으로 못 넘어가고 1클럭 동안 **그대로 얼어붙습니다.** 당연히 PC도 멈췄으니 3번 명령어 도 메모리에서 안 가져옵니다. 
2. **거품 발사! (Bubble)**: 
   2번 명령(ADD)이 얼어붙어서 앞방(Execute)으로 못 갔으니, 앞방은 텅 비게 됩니다. 해저드 유닛은 이 빈방에다가 `Opcode = 00000 (NOP, 아무 짓도 하지 마라)`으로 가득 채운 **거품(Bubble)**을 쏴서 보냅니다. 
   
이 거품이 파이프라인을 타고 뒤로 흘러가면서 완벽하게 1클럭의 시간(여유)을 벌어줍니다. 그 1클럭 동안 앞선 `LOAD` 명령어는 무사히 메모리(M)에서 무를 뽑아냅니다. 이제 시간이 맞았으니 전방 전달(Forwarding) 회로가 무를 낚아채서 2번 명령(ADD)에게 쏴주면, 다음 클럭부터 다시 파이프라인이 쌩쌩 돌아갑니다.

```ascii
[해저드 감지 유닛의 얼음(Freeze)과 거품(Bubble) 투척]

 1. LOAD R1 -> [F] [D] [E] [*M (무 뽑음!)*] [W]
                             \ (1클럭 늦췄더니 이제야 포워딩 가능!)
                              V
 2. ADD  R1 ->     [F] [*D (얼음!)*] [ Bubble (가짜 NOP) ] [E (무 받아서 연산!)] [M] [W]
                                       ^^^^^^
                  해저드 유닛이 E방에 던져버린 1클럭짜리 대기 시간 (거품)
```

📢 **섹션 요약 비유**: 거품(Bubble) 투척은 도로 위의 '세이프티 카(Safety Car)'입니다. 앞쪽 도로에서 공사(LOAD)를 하고 있으면, 경찰차(해저드 유닛)가 세이프티 카(거품)를 경주차들 앞에 투입해서 차들이 1바퀴 동안 천천히 달리며 강제로 대기하게 만듭니다(얼음). 공사가 끝나면 세이프티 카가 싹 빠지고 다시 전속력으로 달립니다. 이 1바퀴의 강제 여유가 바로 파이프라인의 생명을 구하는 Bubble입니다.

## 개념 맵 (Concept Map)
```ascii
[해저드 감지 유닛 (Hazard Detection Unit)]
   │
   ├── 존재 의의: 전방 전달(Forwarding) 회로로 극복 불가능한 'Load-Use Data Hazard' 상황을 실시간 감지하여 H/W 방어
   │
   ├── 위치 및 감지 로직
   │     ├── 위치: Instruction Decode (ID) 단계에 상주
   │     └── 조건: 이전 명령어의 Opcode가 LOAD이고, 그 목적지 레지스터가 현재 명령어의 소스 레지스터와 일치하는지 판별
   │
   └── 제어 액션 (Stall 발생 매커니즘)
         ├── Freeze (동결): PC와 IF/ID 래치의 Write Enable 신호를 0으로 막아 파이프라인 전단부 진행 정지
         └── Bubble (거품): ID/EX 래치로 향하는 제어 신호(Control Signal)를 모두 0(NOP)으로 다중화기(MUX) 조작하여 1 Cycle의 유휴 시간 창출
```

## 어린이 비유
해저드 감지 유닛은 롤러코스터 승강장에 서 있는 **매의 눈 안전요원 아저씨**예요! 출발하려는 뒷차(ADD) 손님이 안전벨트를 매고 있는데, 앞차(LOAD) 손님이 아직 안 내리고 우물쭈물하는 걸 0.1초 만에 싹! 발견해요. 아저씨는 즉시 "스톱! 브레이크 밟아!" 버튼을 눌러 뒷차를 1초 동안 얼음(Freeze)으로 멈춰 세워요. 그사이 앞차가 무사히 내리고 빈 공간(Bubble)이 생기면, "자, 이제 출발~" 하고 안전하게 롤러코스터를 보내주는 최고의 가드랍니다!
