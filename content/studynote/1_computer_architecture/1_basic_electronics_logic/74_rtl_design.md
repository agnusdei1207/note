# RTL (Register Transfer Level) 설계

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: RTL (Register Transfer Level) 설계는 디지털 회로의 동작을 레지스터 (Register) 사이의 신호 흐름과 그 사이의 논리 연산 (Combinational Logic)으로 기술하는 설계 추상화 단계다.
> 2. **가치**: 게이트 (Gate) 수준의 복잡한 물리 회로를 논리적 흐름으로 단순화하여 기술할 수 있게 하며, 논리 합성 (Logic Synthesis) 툴의 직접적인 입력 소스로 활용되어 자동화된 칩 설계를 가능하게 한다.
> 3. **융합**: 현대 SoC (System on Chip) 설계의 표준 공정이며, 상위 단계인 ESL (Electronic System Level) 설계와 하위 단계인 Physical Design (물리 설계)을 잇는 핵심 허브 역할을 수행한다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: RTL (Register Transfer Level) 설계는 디지털 시스템을 **"상태를 저장하는 레지스터"**와 **"상태를 변형하는 조합 논리"**의 조합으로 정의하는 방식이다. 클럭 (Clock)의 매 주기마다 레지스터에 저장된 데이터가 조합 논리를 거쳐 다음 레지스터로 어떻게 전달되는지를 HDL (Hardware Description Language)로 기술한다.

- **💡 비유**: RTL 설계는 "공장의 컨베이어 벨트 시스템"과 같다. 레지스터는 각 공정 사이의 "임시 보관함"이고, 조합 논리는 그 사이에서 부품을 조립하는 "작업 로봇"이다. 벨트가 한 칸 움직일 때마다(클럭 에지), 보관함의 물건이 로봇을 거쳐 다음 보관함으로 정확히 이동하는 과정을 설계하는 것이다.

- **등장 배경**:
  1. **설계 규모의 폭발적 증가**: 수억 개의 트랜지스터를 일일이 배치하는 것이 불가능해지면서, 기능 중심의 추상화된 설계 방식이 필요해졌다.
  2. **EDA (Electronic Design Automation)의 발전**: RTL 코드를 게이트 회로로 자동 변환해주는 합성 기술이 성숙하면서 설계 생산성이 비약적으로 향상되었다.
  3. **재사용성 (IP) 중시**: 검증된 RTL 코드를 IP (Intellectual Property) 형태로 재사용하여 복잡한 시스템을 빠르게 구축하려는 요구가 커졌다.

- **📢 섹션 요약 비유**: 복잡한 미로(게이트)를 직접 그리는 대신, 갈림길(레지스터)과 이동 규칙(조합 논리)만 정해주면 지도가 자동으로 완성되는 스마트 설계 방식과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소 (RTL 아키텍처 모델)

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **레지스터 (Register)** | 상태 유지 및 동기화 | 클럭 에지에 맞춰 입력 값을 캡처 및 출력 | 공정 사이의 임시 저장함 |
| **조합 논리 (Comb. Logic)** | 데이터 변형 및 연산 | 입력 변화에 즉각 반응하여 논리 결과 산출 | 물건을 변형하는 가공 장치 |
| **클럭 (Clock)** | 동작 타이밍 기준 | 시스템 전체의 동기화 신호 제공 | 전체 공정의 시작 종소리 |
| **데이터패스 (Datapath)** | 실제 데이터 처리 통로 | 산술 연산, 시프트, 멀티플렉싱 수행 | 실제 제품이 지나가는 통로 |
| **제어 유닛 (Control Unit)** | 처리 흐름 제어 | 상태 머신 (FSM)을 통해 데이터 흐름 지시 | 공정 전체를 지휘하는 관리자 |

---

### RTL 동작 구조 (Register-to-Register Path)

RTL 설계의 핵심은 클럭 에지 사이의 논리 지연을 관리하는 것이다. 모든 경로는 레지스터에서 시작하여 조합 논리를 거쳐 다음 레지스터에서 끝난다.

```text
 ┌──────────────────────────────────────────────────────────────────┐
 │               RTL 기본 구조 및 타이밍 경로                         │
 ├──────────────────────────────────────────────────────────────────┤
 │                                                                  │
 │    [레지스터 A]        [조합 논리 회로]        [레지스터 B]         │
 │    ┌─────────┐        ┌──────────────┐        ┌─────────┐        │
 │    │         │        │              │        │         │        │
 │  D ┤  Flip-  ├─ Q ────┤  Adder, MUX, ├────────┤  Flip-  ├─ Q    │
 │    │  Flop   │        │  Logic Gates │        │  Flop   │        │
 │    └────┬────┘        └──────────────┘        └────┬────┘        │
 │         ▲                                          ▲             │
 │         │                  Clock                   │             │
 │   ──────┴──────────────────────────────────────────┴──────       │
 │                                                                  │
 │   <--------- 클럭 주기 (T_clk) 내에 완료되어야 함 --------->           │
 │                                                                  │
 │   1. 클럭 상승 에지 발생                                           │
 │   2. 레지스터 A의 값이 Q로 출력됨 (T_co)                            │
 │   3. 조합 논리를 통과하며 연산 수행 (T_comb)                        │
 │   4. 레지스터 B의 입력 D에 도달                                     │
 │   5. 다음 클럭 에지 전까지 안정화 (T_setup)                         │
 └──────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 위 구조는 RTL 설계의 최소 단위인 'Register-to-Register' 경로를 보여준다. 설계자는 조합 논리 (Combinational Logic)가 너무 복잡해져서 지연 시간 (T_comb)이 클럭 주기 (T_clk)를 초과하지 않도록 설계해야 한다. 만약 지연이 너무 길면 파이프라인 (Pipeline)을 추가하여 조합 논리를 여러 단계의 레지스터로 쪼개는 방식이 실무에서 흔히 사용된다. RTL 코딩은 바로 이 구조를 HDL 텍스트로 옮기는 과정이다.

---

### 심층 동작 원리: 상태 머신 (FSM)과 데이터패스 분리

효율적인 RTL 설계는 제어 (Control)와 데이터 처리 (Datapath)를 명확히 분리하는 **FSMD (Finite State Machine with Datapath)** 구조를 따른다.

1. **상태 머신 (FSM)**: 현재 상태를 저장하고, 입력에 따라 다음 상태와 제어 신호 (Control Signal)를 생성한다.
2. **제어 신호**: 데이터패스 내의 멀티플렉서 (MUX) 선택 신호, 연산기 활성화 신호 등을 조절한다.
3. **데이터패스**: 제어 신호에 따라 실제 데이터를 더하고, 빼고, 이동시킨다.
4. **피드백**: 데이터패스의 처리 결과(예: 연산 완료, 비교 결과)를 FSM에 전달하여 다음 상태 결정에 반영한다.

- **📢 섹션 요약 비유**: 지휘자(FSM)가 악보(상태)를 보고 신호를 주면, 연주자(데이터패스)들이 각자의 악기(연산기)로 소리(데이터)를 내는 오케스트라의 협업과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 합성 가능 (Synthesizable) vs 합성 불가 (Non-synthesizable) RTL

| 비교 항목 | 합성 가능 RTL (Synthesizable) | 합성 불가 RTL (Non-synthesizable) |
|:---|:---|:---|
| **목적** | 실제 칩 하드웨어 생성 | 시뮬레이션 및 검증 (Testbench) |
| **주요 문법** | always, assign, if-else, case | #delay, initial, task, function |
| **제약 사항** | 엄격한 하드웨어 매핑 규칙 준수 | 소프트웨어와 유사한 유연한 기술 가능 |
| **결과물** | 게이트 넷리스트 (Gate Netlist) | 시뮬레이션 로그 및 파형 |
| **비유** | 실제 건축이 가능한 도면 | 건축 과정 시뮬레이션용 모형 |

### 과목 융합 관점

- **디지털 논리 회로**: 카르노 맵 (Karnaugh Map) 등을 통한 최적화 지식이 RTL 내 조합 논리 설계의 근간이 되며, 합성 툴이 이를 자동화한다.
- **반도체 공정**: RTL 설계 시 결정된 레지스터의 개수와 조합 논리의 복잡도는 최종 칩의 면적 (Area)과 소비 전력 (Power)에 직접적인 영향을 준다.

- **📢 섹션 요약 비유**: 요리법(RTL)을 쓸 때, 실제로 만들 수 있는 재료(합성 가능 소자)만 사용해야 맛있는 요리(실제 칩)가 완성되는 것과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **타이밍 위반 (Timing Violation) 발생 시**: 특정 조합 논리 경로가 너무 길어 클럭 주기를 맞추지 못할 때, RTL 설계자는 해당 경로 중간에 레지스터를 삽입하는 **파이프라이닝 (Pipelining)** 기술을 적용하여 동작 주파수를 높인다.
2. **저전력 설계 요구 시**: 데이터 변화가 없는 구간에서 클럭이 계속 뛰는 것을 방지하기 위해, 특정 조건에서만 클럭이 레지스터에 전달되게 하는 **클럭 게이팅 (Clock Gating)** 로직을 RTL 수준에서 삽입한다.

### 도입 체크리스트
- 모든 순차 회로 (Sequential Logic)가 단일 클럭 에지에 동기화되어 있는가?
- 조합 논리 루프 (Combinational Loop)가 발생하지 않도록 코딩했는가?
- 의도치 않은 래치 (Latch)가 생성되지 않도록 모든 조건문 (`if-else`, `case-default`)이 완결되었는가?

### 안티패턴
- **거대 조합 논리 (Huge Comb. Logic)**: 한 클럭 주기 안에 너무 많은 연산을 몰아넣으면 지연 시간이 폭증하여 칩의 성능을 떨어뜨린다.
- **비동기 설계 남발**: 클럭이 없는 비동기 신호를 남발하면 메타스테이블 (Metastable) 문제가 발생하여 칩의 안정성을 심각하게 해친다.

- **📢 섹션 요약 비유**: 과도한 욕심으로 한 입에 너무 많은 음식(논리)을 넣으려다 체하는(타이밍 오류) 일을 피하기 위해, 잘게 쪼개어 삼키는(파이프라인) 지혜가 필요합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 도입 전 (Gate Level 설계) | 도입 후 (RTL 설계) | 개선 효과 |
|:---|:---|:---|:---|
| **정량** | 설계 생산성 낮음 (1만 게이트/월) | 설계 생산성 높음 (100만 게이트/월) | 생산성 100배 이상 향상 |
| **정성** | 하드웨어 종속적 설계 | 벤더 독립적, 이식성 우수 | 설계 자산(IP) 보호 및 재사용 가능 |

### 미래 전망
- **AI 기반 RTL 최적화**: 딥러닝 알고리즘이 RTL 코드를 분석하여 전력과 면적을 자동으로 최소화하는 기술이 도입되고 있다.
- **Chisel / SpinalHDL**: 자바나 스칼라 같은 현대적 언어의 강점을 RTL 설계에 도입하여 생산성을 더욱 극대화하려는 시도가 활발하다.

- **📢 섹션 요약 비유**: 수동 타자기에서 워드프로세서로 진화한 것처럼, RTL 설계는 반도체 설계의 민주화와 대형화를 이끈 일등 공신입니다.
