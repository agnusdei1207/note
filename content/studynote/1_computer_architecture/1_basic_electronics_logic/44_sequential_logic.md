> **핵심 인사이트**
> 1. 순차 논리회로(Sequential Logic)는 현재의 입력뿐만 아니라, **과거의 상태(저장된 데이터)까지 반영하여 현재의 출력을 결정**하는 논리 회로이다.
> 2. 데이터를 기억하기 위해 회로의 출력이 다시 입력으로 돌아가는 **피드백 루프(Feedback Loop)와 메모리 소자(플립플롭 등)**를 반드시 포함한다.
> 3. 컴퓨터 아키텍처의 핵심인 CPU의 레지스터(Register), 프로그램 카운터(PC), 그리고 메인 메모리(SRAM)를 구성하는 절대적인 물리적 기반이다.

---

## Ⅰ. 순차 논리회로의 특징: "기억(Memory)의 탄생"
조합 논리회로(가산기, MUX 등)는 자판기처럼 과거를 잊어버리고 현재 누른 버튼(입력)에만 반응했습니다. 하지만 컴퓨터가 복잡한 프로그램(예: 1부터 10까지 더하기)을 실행하려면, 방금 전까지 계산한 덧셈 결과를 머릿속에 '기억'하고 있어야 합니다.

순차 논리회로는 Combinational Logic 블록 옆에 **메모리(저장) 소자**를 달아놓은 형태입니다. 회로가 계산한 출력 결과 중 일부가 다시 뒤로 삥 돌아와서 메모리에 저장되고(Feedback), 다음번 계산을 할 때 새로운 입력과 함께 그 메모리 값이 섞여서 들어갑니다. 이렇게 하여 회로는 '과거의 기억'을 갖게 됩니다.

```ascii
[순차 논리회로의 블랙박스 모델]

 외부 입력 ------------------> +---------------+
 (현재 버튼)                   |               | -----> 외부 출력 (결과)
                               | 조합 논리회로 |
 +-- (과거의 기억) ----------> |               | --+
 |                             +---------------+   |
 |                                                 |
 |                           +-----------------+   | (출력이 되돌아옴)
 +---<----<----<----<----<---| 메모리 (플립플롭)|<--+
        피드백 (Feedback)    +-----------------+
                             (클럭 Clock 에 맞춰 저장)
```

📢 **섹션 요약 비유**: 순차 논리회로는 '끝말잇기 게임'입니다. 내 입에서 나갈 단어(출력)는 단순히 내가 지금 생각나는 단어(현재 입력)가 아니라, 내 앞사람이 방금 전까지 말했던 단어의 끝 글자(과거의 기억, 피드백)에 완벽하게 종속되어 결정됩니다.

## Ⅱ. 상태(State)와 동기/비동기 방식
순차 회로는 기억을 가지고 있기 때문에 시스템의 **'상태(State)'**라는 개념을 가집니다. 예를 들어 선풍기 풍량 조절 버튼을 누를 때, 현재 '정지' 상태라면 '미풍' 상태가 되고, 현재 '강풍' 상태라면 다시 '정지' 상태로 돌아가는 식입니다. (이를 상태도(State Diagram)로 표현합니다.)

순차 논리회로는 메모리 소자가 언제 기억을 갱신하느냐에 따라 두 가지로 나뉩니다.
1. **비동기식 (Asynchronous)**: 외부 입력이 들어오는 즉시(번개 치듯) 상태가 변합니다. 속도는 빠르지만, 입력 신호들이 0.001초라도 어긋나게 도착하면 예측 불가능한 쓰레기 상태에 빠지는 치명적인 버그(Race Condition)가 생깁니다.
2. **동기식 (Synchronous)**: 모든 메모리 소자가 오직 **시스템 클럭(Clock)**이 심장 박동처럼 "쿵!" 하고 뛸 때만 동시에 일제히 상태를 바꿉니다. 현대 컴퓨터(CPU)는 100% 동기식 설계를 채택하여 안정성을 확보합니다.

```ascii
[동기식 순차 회로의 동작 흐름]

 입력 도착! -> (문 닫힘, 대기) -> 입력 도착! -> (대기)
     |
 클럭 펄스 발생 (Tick!) -> 일제히 문이 열림! -> 메모리에 새로운 값 동시 저장!
     |
 새로운 상태(State)로 전이 완료
```

📢 **섹션 요약 비유**: 동기식 순차 회로는 군대의 '제식 훈련'입니다. 병사들(메모리)이 자기 맘대로 발을 내딛는 게 아니라, 조교가 "하나! 둘!(클럭)" 하고 호각을 불 때만 수만 명의 병사들이 한 치의 오차 없이 동시에 상태(자세)를 바꾸는 것입니다.

## Ⅲ. 아키텍처 관점: 순차 회로로 만든 컴퓨터의 심장
컴퓨터 아키텍처에서 조합 회로(ALU)가 '근육'이라면, 순차 논리회로는 데이터를 저장하고 흐름을 제어하는 **'뇌세포'**입니다.

* **플립플롭(Flip-Flop)**: 1비트의 데이터를 기억하는 가장 작은 순차 회로.
* **레지스터(Register)**: 플립플롭을 32개, 64개 묶어서 CPU 안에 만든 초고속 임시 메모리 저장소.
* **카운터(Counter)**: 클럭이 뛸 때마다 1, 2, 3씩 숫자를 세는 순차 회로. CPU가 다음번에 실행할 명령어의 주소를 가리키는 **프로그램 카운터(PC, Program Counter)**가 바로 이 카운터 회로의 집합체입니다.

이들이 없다면 CPU는 단 한 줄의 명령어도 저장하거나 다음 단계로 넘어갈 수 없습니다.

```ascii
[CPU 데이터패스 속의 순차 논리 (레지스터)]

 (메모리: 순차회로)            (ALU: 조합회로)          (메모리: 순차회로)
 레지스터 A (값: 5) ---\                              /---> 레지스터 C (값: 12 저장됨!)
                       |---> 덧셈(5+7) 수행 --------> |     (클럭이 뛸 때 딱 저장)
 레지스터 B (값: 7) ---/     (기억 못함)
```

📢 **섹션 요약 비유**: 조합 회로가 수학 문제를 푸는 '전자계산기'라면, 순차 회로는 그 계산기가 풀어낸 답을 잊어버리지 않게 공책에 꾹꾹 눌러쓰는 '연필과 기억력'입니다. 이 둘이 합쳐져야 비로소 인공지능 같은 거대한 프로그램을 돌릴 수 있습니다.

## 개념 맵 (Concept Map)
```ascii
[순차 논리회로 (Sequential Logic)]
   │
   ├── 정의: 현재 출력 = f(현재 입력 + 과거의 상태)
   │     └── 구성: 조합 논리회로 + 메모리(저장) 소자 + 피드백(Feedback) 루프
   │
   ├── 동작 방식 구분
   │     ├── 동기식(Synchronous): 클럭(Clock) 신호에 맞춰 모든 소자가 동시 전이 (표준)
   │     └── 비동기식(Asynchronous): 입력 변화 즉시 전이 (타이밍 버그 위험)
   │
   └── 시스템 아키텍처 구현
         └── 플립플롭(Flip-Flop), 레지스터(Register), 프로그램 카운터(PC) 구축
```
