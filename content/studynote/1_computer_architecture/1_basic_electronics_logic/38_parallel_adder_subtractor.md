> **핵심 인사이트**
> 1. 병렬 가감산기(Parallel Adder-Subtractor)는 제어 신호(Control Signal, $M$) 하나만으로 덧셈(Addition)과 뺄셈(Subtraction)을 **하나의 회로에서 자유자재로 선택하여 수행할 수 있는 마법의 하드웨어**이다.
> 2. 입력 B에 연결된 **XOR 게이트**가 핵심입니다. 이 XOR 게이트는 제어 신호 $M$의 값에 따라 입력 B를 그대로 통과시키거나 반전(1의 보수)시키는 '선택적 인버터(Programmable Inverter)' 역할을 한다.
> 3. 이 회로는 현대 컴퓨터의 심장인 **산술논리연산장치(ALU, Arithmetic Logic Unit)**의 가장 기초적이고 완벽한 뼈대 모델이다.

---

## Ⅰ. 하드웨어 재사용의 극치: 하나의 회로로 두 가지 연산을
이전 챕터에서 우리는 뺄셈(A - B)을 할 때 전용 감산기를 칩에 따로 만들지 않고, B를 음수(2의 보수)로 바꾼 뒤 가산기(A + (-B))에 집어넣어 면적을 절약한다는 것을 배웠습니다.
그렇다면 덧셈 명령이 들어올 땐 B를 그냥 더하고, 뺄셈 명령이 들어올 땐 B를 뒤집어서(NOT) 더해주는 **스마트한 가산기**를 어떻게 만들 수 있을까요? 

정답은 입력 B의 입구에 **XOR 게이트**를 달아두는 것입니다. 
XOR 게이트는 한쪽 입력(M)을 0으로 주면 다른 쪽 입력(B)이 그대로 통과하고, M을 1로 주면 B가 정반대(NOT)로 뒤집혀서 나옵니다. (1의 보수 완성)

```ascii
[XOR 게이트의 '선택적 반전(Programmable Inverter)' 성질]

 M (제어 신호)  | B (데이터) | 결과 (M ⊕ B) | 설명
 ---------------+------------+--------------+-----------------------
       0        |     0      |      0       | B가 그대로 통과됨 (B)
       0        |     1      |      1       | 
 ---------------+------------+--------------+-----------------------
       1        |     0      |      1       | B가 반대로 뒤집힘 (B')
       1        |     1      |      0       | (1의 보수 완성!)
```

📢 **섹션 요약 비유**: XOR 게이트는 '마법의 거울'입니다. 조종사(M)가 스위치를 0에 두면 거울은 일반 유리창이 되어 내 모습(B)이 그대로 보입니다. 하지만 스위치를 1에 두면 거울은 청개구리 거울이 되어 내 모습이 완벽하게 반대(B')로 찌그러져 보입니다.

## Ⅱ. 병렬 가감산기의 전체 구조와 원리
이 똑똑한 XOR 게이트를 이용해 4비트 병렬 가감산기(Ripple Carry 기반)를 만들어 봅시다.

1. **제어 신호 M (Mode)**: 이 선 하나로 덧셈/뺄셈을 결정합니다. ($M=0$ 이면 덧셈, $M=1$ 이면 뺄셈)
2. **M = 0 (덧셈 모드)**: 
   * XOR 게이트가 $B$를 그대로 통과시킵니다.
   * $M=0$이므로 첫 번째 전가산기(FA0)의 최초 올림수($C_{in}$)도 0이 들어갑니다.
   * 결과: 완벽한 $A + B + 0 = A + B$ (덧셈)
3. **M = 1 (뺄셈 모드)**: 
   * XOR 게이트가 $B$를 완전히 뒤집습니다 ($B'$ 즉, 1의 보수 생성).
   * 2의 보수를 만들려면 +1을 해줘야 하죠? 마침 $M=1$이므로 첫 번째 전가산기(FA0)의 최초 올림수($C_{in}$)로 1이 쏙 들어갑니다!
   * 결과: $A + (B' + 1) = A + (2의 보수) = A - B$ (완벽한 뺄셈!)

```ascii
[4비트 병렬 가감산기 블록도]

 M (제어 신호: 0=Add, 1=Sub) ------------------------------+--------+
 |         |         |         |                           |
 V         V         V         V                           V (맨 처음 C_in으로 들어감!)
(XOR)     (XOR)     (XOR)     (XOR)    <- B입력을 뒤집을지 말지 결정
 ^         ^         ^         ^
 B3        B2        B1        B0   (입력 B)
 |         |         |         |
[FA 3] <- [FA 2] <- [FA 1] <- [FA 0]   (전가산기)
 |         |         |         |
 A3        A2        A1        A0   (입력 A)
 |         |         |         |
 S3        S2        S1        S0   (최종 결과)
```

📢 **섹션 요약 비유**: M(제어 신호)은 놀이공원의 '물대포 레버'입니다. 레버를 당기면(M=1) B 친구들은 물을 맞고 옷이 뒤집혀서(1의 보수) 놀이기구(전가산기)에 타고, 레버의 남은 물 1방울이 보너스로 출발지점(C_in)에 똑 떨어져 완벽한 2의 보수(음수)를 완성해 냅니다.

## Ⅲ. 아키텍처 관점: 오버플로우 (Overflow) 검출
컴퓨터가 지정된 비트(예: 4비트)로 숫자를 표현하다 보면, 덧셈이나 뺄셈 결과가 4비트 그릇에 담을 수 없을 만큼 너무 커지거나 작아져서 부호가 바뀌어버리는 끔찍한 에러가 발생합니다. 이를 **오버플로우(Overflow)**라고 합니다.

병렬 가감산기에서는 오버플로우가 발생했는지 감지하는 특별한 로직을 추가합니다. 
맨 마지막 전가산기(MSB)에서 **나가는 올림수($C_{out}$)와 그 안으로 들어오는 올림수($C_{in}$)가 다르면(즉, XOR 연산 결과가 1이면)** 무조건 오버플로우가 발생한 것입니다. CPU는 이 오버플로우 깃발(Overflow Flag)이 올라가면 즉시 연산 오류를 프로그램에 보고합니다.

```ascii
[오버플로우(V) 검출 회로]

 (MSB 전가산기로 들어오는 올림수 C_{n-1}) ----+
                                            | XOR | ---> V (오버플로우 플래그)
 (MSB 전가산기에서 나가는 올림수 C_n)   ----+
```

📢 **섹션 요약 비유**: 오버플로우 플래그는 자동차의 '엔진 경고등'입니다. 그릇(비트 수)이 넘쳐흘러서 답이 쓰레기가 되어버렸을 때, 계속 주행하면 컴퓨터가 폭발하니까 "삐삐삐! 방금 계산 정답 아닙니다!"라고 번쩍이며 경고해 주는 아주 중요한 안전장치입니다.

## 개념 맵 (Concept Map)
```ascii
[병렬 가감산기 (Parallel Adder-Subtractor)]
   │
   ├── 존재 의의: 전가산기 하나로 덧셈과 뺄셈을 통합 처리 (ALU의 기본)
   │
   ├── 핵심 매커니즘 (제어 신호 M: 0=Add, 1=Sub)
   │     ├── XOR 게이트: M=1일 때 B를 1의 보수(반전)로 변환
   │     └── C_in 연결: M=1일 때 첫 올림수로 1을 더해 2의 보수(-B) 완벽 구현
   │
   └── 안전 장치: 오버플로우 (Overflow, V) 검출
         └── V = C_{in}(MSB) ⊕ C_{out}(MSB) -> 결과가 허용 범위를 초과했는지 CPU에 보고
```

## 어린이 비유
병렬 가감산기는 **더하기와 빼기 버튼이 하나로 합쳐진 변신 로봇**이에요! 조종사가 '더하기 버튼(0)'을 누르면 로봇이 숫자 두 개를 그대로 꿀꺽 삼켜서 더해요. 그런데 조종사가 '빼기 버튼(1)'을 누르면 마법 거울(XOR)이 쓱 나타나서 빼고 싶은 숫자를 정반대 괴물(보수)로 변신시켜서 더해버려요. 그러면 신기하게도 완벽한 빼기 정답이 짜잔! 하고 튀어나온답니다.
