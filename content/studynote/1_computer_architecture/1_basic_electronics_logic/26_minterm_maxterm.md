> **핵심 인사이트**
> 1. 최소항(Minterm)은 입력 변수들이 가질 수 있는 **단 하나의 특정 조합에서만 출력 '1'을 만들어내는 AND 결합식**이다.
> 2. 최대항(Maxterm)은 단 하나의 특정 조합에서만 출력 '0'을 만들어내며, 나머지는 모두 '1'을 출력하는 **OR 결합식**이다.
> 3. 컴퓨터 회로 설계 시, 진리표의 결과를 논리식으로 매핑할 때 이 두 항을 이용해 **곱의 합(SOP)** 또는 **합의 곱(POS)** 형태로 뼈대를 구축한다.

---

## Ⅰ. 최소항(Minterm)의 원리: SOP의 재료
입력 변수가 3개(A, B, C)일 때, 이 변수들이 만들 수 있는 조합은 8가지($2^3$)입니다. 이 8가지 중 **딱 한 줄(Row)만 1을 출력하고 나머지 7줄은 모두 0을 출력하는 아주 까다로운 조건식**이 바로 최소항입니다.
규칙은 간단합니다. 입력이 1이면 그대로(A), 0이면 NOT(A')을 씌워 **AND($\cdot$)로 묶습니다.**

예를 들어 입력이 A=0, B=1, C=0 인 상황만을 저격하는 최소항($m_2$)은 $A' \cdot B \cdot C'$ 가 됩니다. 이 최소항들을 모아서 OR(+)로 묶은 식을 **곱의 합(SOP, Sum of Products)**이라고 하며, 출력 1을 선호하는 인간의 직관과 맞아 가장 많이 사용됩니다.

```ascii
[3변수 최소항(Minterm) 예시]

 A B C | 최소항 기호 | 논리식 (AND 결합)
 ------+-------------+-------------------
 0 0 0 |    m0       | A'B'C' (A=0,B=0,C=0일때만 1)
 0 1 0 |    m2       | A'BC'  (A=0,B=1,C=0일때만 1)
 1 1 1 |    m7       | ABC    (A=1,B=1,C=1일때만 1)
```

📢 **섹션 요약 비유**: 최소항은 '특정 범인 몽타주'입니다. "안경 안 쓰고(A'), 모자 쓰고(B), 수염 없는(C') 사람 잡아와!"라는 식으로 8명의 용의자 중 정확히 딱 1명(출력 1)만 지목해 내는 아주 뾰족하고 구체적인 지시입니다.

---

## Ⅱ. 최대항(Maxterm)의 원리: POS의 재료
최소항이 '1'을 찾는 사냥꾼이라면, 최대항은 **'0'을 찾는 사냥꾼**입니다.
8가지 조합 중 **딱 한 줄에서만 0을 출력하고 나머지 7줄은 모두 1을 출력하는 너그러운 조건식**입니다. 
규칙은 최소항과 완전히 반대입니다. 입력이 0이면 그대로(A), 1이면 NOT(A')을 씌워 **OR($+$)로 묶습니다.**

예를 들어 A=0, B=1, C=0 인 상황을 저격하는 최대항($M_2$)은 $A + B' + C$ 가 됩니다. 이 최대항들을 모아서 AND($\cdot$)로 묶은 식을 **합의 곱(POS, Product of Sums)**이라고 합니다. 드모르간의 법칙에 의해 최소항의 전체를 부정(NOT)하면 최대항이 됩니다.

```ascii
[3변수 최대항(Maxterm) 예시]

 A B C | 최대항 기호 | 논리식 (OR 결합)
 ------+-------------+-------------------
 0 0 0 |    M0       | A+B+C   (A=0,B=0,C=0일때만 0)
 0 1 0 |    M2       | A+B'+C  (A=0,B=1,C=0일때만 0)
 1 1 1 |    M7       | A'+B'+C'(A=1,B=1,C=1일때만 0)
```

📢 **섹션 요약 비유**: 최대항은 '특정 음식 알레르기'입니다. "새우 알레르기(0, 1, 0)만 피하면, 나머지 7가지 메뉴는 뭐든 다 먹을 수 있어(1)!"라는 식으로 딱 하나의 예외(출력 0)만 걸러내는 넓은 그물망입니다.

---

## Ⅲ. 아키텍처 관점: SOP vs POS 선택의 트레이드오프
진리표를 부울 식으로 바꿀 때 설계자는 SOP(최소항 기반)로 할지, POS(최대항 기반)로 할지 결정해야 합니다. 기능은 100% 동일하지만, **어느 쪽이 논리 게이트(트랜지스터)를 더 적게 쓰느냐**가 비용을 결정합니다.

* **SOP가 유리할 때**: 진리표 결과에 '1'이 별로 없고 '0'이 압도적으로 많을 때. (최소항 몇 개만 OR로 묶으면 됨)
* **POS가 유리할 때**: 진리표 결과에 '1'이 꽉 차 있고 '0'이 몇 개 없을 때. (최대항 몇 개만 AND로 묶는 게 칩 면적을 아낌)

이처럼 최소항과 최대항은 하드웨어 설계 시 비용(면적, 전력)을 최적화하기 위한 양면의 칼로 사용됩니다.

```ascii
[진리표 출력 분포에 따른 설계 전략]

 진리표 출력 (F) = {0, 0, 0, 1, 0, 0, 1, 0}
  -> 1이 2개뿐! => 최소항(SOP) 방식으로 설계하는 것이 트랜지스터 낭비가 적음.

 진리표 출력 (F) = {1, 1, 1, 0, 1, 1, 0, 1}
  -> 0이 2개뿐! => 최대항(POS) 방식으로 설계하는 것이 훨씬 이득임.
```

📢 **섹션 요약 비유**: 검은 바둑알 90개와 흰 바둑알 10개가 섞여 있을 때, 바둑알을 다 골라내라는 지시를 받았다면 '흰 바둑알 10개(적은 쪽)'를 골라내는 것이 훨씬 빠릅니다. SOP와 POS 선택도 이처럼 더 적게 힘을 들이는 방향을 고르는 기술입니다.

## 어린이 비유
최소항은 **"딱 이 열쇠만 이 상자를 열 수 있어!"**라는 깐깐한 자물쇠예요. 열쇠 모양이 완벽하게 맞아야만 찰칵(1) 열리죠. 반대로 최대항은 **"이 열쇠 빼고는 다 열 수 있어!"**라는 아주 너그러운 자물쇠예요. 컴퓨터를 만드는 아저씨들은 상자를 1번 열지, 99번 열지에 따라 두 자물쇠 중 하나를 골라 부품값을 절약한답니다!
