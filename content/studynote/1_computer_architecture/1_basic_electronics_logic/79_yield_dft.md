# 수율 (Yield) 및 비검사 설계 (DFT, Design for Testability)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 수율 (Yield)은 투입된 웨이퍼 대비 정상 판정을 받은 칩의 비율이며, 비검사 설계 (DFT, Design for Testability)는 제조 후 발생하는 불량 칩을 빠르고 정확하게 골라내기 위해 칩 내부에 테스트 전용 회로를 삽입하는 설계 기술이다.
> 2. **가치**: DFT는 칩의 관측 가능성 (Observability)과 제어 가능성 (Controllability)을 높여 테스트 비용을 절감하고, 불량 칩이 시장에 유출되는 것을 막아 최종 제품의 신뢰성을 보장한다.
> 3. **융합**: 공정 최적화(수율 향상)와 하드웨어 설계(DFT)가 결합하여 반도체 비즈니스의 수익성을 결정하며, ATPG (Automatic Test Pattern Generation) 및 BIST (Built-In Self Test) 기술을 통해 테스트 자동화를 구현한다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 수율 (Yield)은 반도체 제조 효율의 척도로, 웨이퍼 한 장에서 얻을 수 있는 최대 칩 수 대비 실제 작동하는 칩 수의 비율이다. 비검사 설계 (DFT, Design for Testability)는 이 수율을 확인하고 불량을 진단하기 위해, 설계 단계에서부터 테스트 장비(ATE)가 칩 내부 깊숙한 곳까지 들여다볼 수 있도록 "테스트용 고속도로"를 미리 닦아놓는 작업이다.

- **💡 비유**: 수율은 "사과나무에서 딴 사과 중 상품성이 있는 사과의 비율"과 같고, DFT는 "사과 속이 썩었는지 겉만 보고도 금방 알 수 있게 사과에 특수 센서나 창문을 달아두는 것"과 같다. 창문이 없으면 일일이 다 깎아봐야(파괴 검사) 알 수 있지만, DFT가 잘 된 사과는 엑스레이만 찍어도(비파괴 검사) 불량을 바로 가려낼 수 있다.

- **등장 배경**:
  1. **회로의 복잡도 증가**: 수억 개의 게이트를 가진 칩은 외부 핀(I/O)만으로는 내부 로직의 고장 여부를 확인하기가 수학적으로 불가능해졌다.
  2. **테스트 비용의 폭증**: 반도체 원가에서 테스트가 차지하는 비중이 30~40%에 육박하면서, 테스트 시간을 단축할 자동화 기술이 절실해졌다.
  3. **제로 디펙트 (Zero Defect) 요구**: 자동차나 의료 기기용 칩은 단 하나의 불량도 허용되지 않으므로, 99.9% 이상의 고장 검출률 (Fault Coverage)이 요구된다.

- **📢 섹션 요약 비유**: 수확의 기쁨(수율)을 누리기 위해, 씨앗을 심을 때부터 검사하기 좋은 구조(DFT)를 고민하는 농부의 전략과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### DFT 핵심 구성 요소 및 기술

| 기술명 | 역할 | 상세 내용 | 비유 |
|:---|:---|:---|:---|
| **Scan Chain** | 내부 로직 테스트 | 일반 플립플롭을 Scan FF로 교체하여 거대한 시프트 레지스터 형성 | 모든 방을 한 줄로 잇는 통로 만들기 |
| **BIST (Built-In Self Test)** | 자체 진단 회로 | 칩 내부에 테스트 패턴 생성기(TPG)와 분석기(ORA)를 직접 삽입 | 자가 검진 키트 내장 |
| **MBIST (Memory BIST)** | 메모리 특화 테스트 | SRAM/DRAM의 읽기/쓰기 동작 및 주소선을 고속으로 자동 테스트 | 창고 정리 자동 로봇 |
| **Boundary Scan** | 보드 레벨 테스트 | JTAG (IEEE 1149.1) 표준을 통해 칩 외부 핀의 연결 상태 검사 | 아파트 외벽 및 배관 점검 |
| **ATPG (Auto Test Pattern)** | 테스트 패턴 생성 | 고장 모델(Fault Model)을 바탕으로 불량을 잡을 최소한의 입력값 생성 | 가장 효율적인 범인 취조 문답지 |

### Scan Path (스캔 경로) 아키텍처

Scan 설계는 DFT의 가장 기본으로, 일반적인 동작 모드와 테스트 모드를 선택할 수 있는 Scan Flip-Flop을 사용한다.

```text
 ┌──────────────────────────────────────────────────────────────────┐
 │               Scan Flip-Flop 및 Scan Chain 구조                   │
 ├──────────────────────────────────────────────────────────────────┤
 │                                                                  │
 │    [Scan Flip-Flop 내부]             [Scan Chain 연결 구조]       │
 │                                                                  │
 │   D (Normal) ──┐                    ┌──────┐  ┌──────┐  ┌──────┐ │
 │                │ MUX ┌───┐          │      │  │      │  │      │ │
 │   SI (Scan In) ──┤──▶│D Q├─┬──▶ Q    │  FF1 ├──▶  FF2 ├──▶  FF3 │ │
 │                │     │   │ │        │      │  │      │  │      │ │
 │   SE (Enable)  ──┘   └───┘ │        └──────┘  └──────┘  └──────┘ │
 │                             │           ▲         ▲         ▲    │
 │                             └───────────┴─────────┴─────────┘    │
 │                                         Scan Path (SI -> SO)     │
 │                                                                  │
 │   1. Shift Mode (SE=1): 외부에서 SI를 통해 원하는 값을 FF에 채움    │
 │   2. Capture Mode (SE=0): 내부 조합 논리 연산 결과를 FF에 저장     │
 │   3. Shift Mode (SE=1): 다시 Shift하여 SO(Scan Out)로 결과 확인    │
 └──────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** Scan 설계는 평소에는 각자 따로 노는 플립플롭들을 테스트 모드(`SE=1`)가 되면 하나로 길게 연결된 기차(Shift Register)처럼 만든다. 이를 통해 칩 깊숙한 곳에 있는 플립플롭에 우리가 원하는 값(0 또는 1)을 직접 밀어 넣을 수 있고(제어 가능성), 연산이 끝난 뒤 그 값을 밖으로 꺼내어 확인할 수 있다(관측 가능성). 이 방식은 칩 내부의 모든 조합 논리 회로를 핀 하나로 검사할 수 있게 해주는 혁신적인 구조다.

### 심층 동작 원리: 고장 모델 (Fault Model)

테스트는 "무엇이 고장 났는가"를 정의하는 것부터 시작한다.

1. **Stuck-at Fault (고정 고장)**: 특정 신호선이 물리적 결함으로 인해 항상 0(`Stuck-at-0`) 또는 1(`Stuck-at-1`)에 고정된 상태. 가장 기본적이고 널리 쓰이는 모델이다.
2. **Transition Fault (전이 고장)**: 신호가 바뀌기는 하지만, 규정된 시간(클럭 주기) 내에 바뀌지 않는 속도 관련 고장.
3. **Bridging Fault (가교 고장)**: 인접한 두 선이 서로 붙어서 원치 않는 논리 연산(AND/OR)이 일어나는 고장.
4. **Iddq Testing**: 칩이 정지 상태일 때 흐르는 미세 전류를 측정하여 누설 전류가 큰 불량품을 가려내는 방식.

- **📢 섹션 요약 비유**: 전구가 켜져만 있거나(SA1) 꺼져만 있는(SA0) 불량, 혹은 너무 늦게 켜지는 불량(Transition) 등 고장의 유형별로 맞춤형 검사지를 만드는 과정과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### MBIST (Memory BIST) vs LBIST (Logic BIST)

| 비교 항목 | MBIST (Memory BIST) | LBIST (Logic BIST) |
|:---|:---|:---|
| **검사 대상** | SRAM, DRAM, ROM 등 메모리 | CPU 코어, 로직 게이트 등 논리 회로 |
| **복잡도** | 구조가 정형화되어 있어 구현이 쉬움 | 무작위 패턴 생성이 필요하여 구현이 복잡함 |
| **테스트 패턴** | March Algorithm (읽기/쓰기 반복) | PRPG (Pseudo-Random Pattern Gen.) |
| **점유 면적** | 상대적으로 작음 | 상대적으로 큼 (패턴 생성기 포함) |
| **비유** | 창고의 모든 서랍을 열고 닫아보기 | 공장의 모든 로봇 팔을 움직여보기 |

### 과목 융합 관점

- **컴퓨터 구조**: 최신 프로세서 아키텍처는 전원을 켤 때마다(Booting) MBIST를 수행하여 불량 메모리 셀을 격리하고 여분의 셀(Redundancy)로 교체하는 'Self-Repair' 기능을 포함한다.
- **반도체 공정**: 수율 데이터 분석 (Yield Analysis)은 DFT 테스트 결과와 공정 데이터를 매핑하여, 특정 장비나 가스 라인에서 불량이 발생하는지 찾아내는 빅데이터 분석과 연계된다.

- **📢 섹션 요약 비유**: 내장 메모리를 검사하는 내부 점검반(MBIST)과 전체 로직을 감시하는 보안팀(LBIST)이 칩이라는 요새를 지키고 있습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **Low Yield (저수율) 위기 상황**: 새로운 공정 도입 초기 수율이 10% 미만으로 나올 때, DFT 엔지니어는 'Scan Diagnosis'를 통해 불량이 발생한 정확한 게이트 위치(X, Y 좌표)를 공정 팀에 전달하여 불량의 근본 원인 (Root Cause)을 빠르게 해결한다.
2. **Burn-in Test (가속 수명 테스트)**: 칩에 높은 전압과 열을 가해 잠재적 불량을 조기에 끌어낼 때, DFT의 BIST 기능을 활용하여 외부 장비 없이도 칩 스스로 가혹한 환경에서 무한 반복 테스트를 수행하게 한다.

### 도입 체크리스트
- 테스트 타겟인 고장 검출률 (Fault Coverage)이 99.5% 이상인가?
- DFT 회로 삽입으로 인한 타이밍 페널티가 성능 마진 내에 있는가?
- 파워 도메인별로 테스트 시 소모 전력(Test Power)이 허용치를 넘지 않는가? (테스트 중 칩이 타버리는 현상 방지)

### 안티패턴
- **Neglected Test Power**: 테스트 모드에서는 모든 플립플롭이 동시에 요동치므로 평소보다 수 배의 전력을 소모한다. 이를 고려하지 않고 테스트 패턴을 짜면 전압 강화 (IR-Drop)로 인해 정상 칩을 불량으로 오판하는 'Over-kill'이 발생한다.
- **DFT Logic Security**: 테스트를 위해 만들어둔 통로(JTAG 등)가 해커의 침입 경로로 사용될 수 있다. 보안 칩 설계 시 DFT 포트를 잠그는 'Security Lock'을 반드시 고려해야 한다.

- **📢 섹션 요약 비유**: 검사를 위해 만든 문이 도둑의 침입로가 되지 않도록, 검사가 끝나면 자물쇠를 꼭 잠그는 보안 의식이 필요합니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 도입 전 | 도입 후 (DFT 적용) | 개선 효과 |
|:---|:---|:---|:---|
| **정량** | 칩당 테스트 시간 수 분 | 칩당 테스트 시간 수 초 | 테스트 비용 80% 이상 절감 |
| **정성** | 복합 고장 검출 불가 | 수학적 고장 모델 기반 전수 조사 가능 | 고객 신뢰도 향상 및 브랜드 가치 제고 |

### 미래 전망
- **AI-enhanced ATPG**: 딥러닝을 활용하여 고장 검출률은 유지하면서 테스트 패턴 수는 획기적으로 줄여 테스트 비용을 극단으로 낮추는 기술이 연구되고 있다.
- **IEEE 1687 (IJTAG)**: 칩 내부에 수만 개의 DFT IP들이 거대한 네트워크를 이루어, 마치 인터넷처럼 필요한 곳에 접속해 테스트하는 'Internal JTAG' 기술이 SoC 설계의 표준이 되고 있다.

- **📢 섹션 요약 비유**: 스스로 병을 진단하고 수술 위치까지 알려주는 '스마트 환자'가 된 칩 덕분에, 반도체 의사(엔지니어)들의 업무 효율이 비약적으로 높아지고 있습니다.
