> **핵심 인사이트**
> 1. 디멀티플렉서(DEMUX)는 멀티플렉서(MUX)와 정반대로 작동하는 회로로, **1개의 입력선으로 들어온 데이터를 여러 개의 출력선($2^N$개) 중 하나로 분배해 주는 '디지털 분배기(Distributor)'**이다.
> 2. MUX와 마찬가지로 데이터를 어느 쪽으로 보낼지 결정하는 $N$개의 **선택선(Select Line)**에 의해 경로가 통제된다.
> 3. 컴퓨터 아키텍처에서 직렬로 전송된 데이터를 병렬 장치들로 분배하거나, CPU 연산 결과를 여러 레지스터 중 하나를 골라 저장할 때 쓰인다.

---

## Ⅰ. 디멀티플렉서의 구조: "선택된 길로 가라"
DEMUX는 나팔 모양의 구조를 가집니다. 하나의 거대한 파이프(입력선)로 데이터가 들어오면, 조종사가 레버(선택선)를 조작하여 여러 개의 작은 파이프(출력선) 중 **딱 한 곳으로만 물(데이터)을 흘려보냅니다.** 선택받지 못한 나머지 출력선들은 모두 0 상태를 유지합니다.

예를 들어 1x4 DEMUX는 1개의 데이터 입력(D)과 2개의 선택선(S1, S0), 그리고 4개의 출력선(Y0~Y3)을 가집니다.
* 조종사가 선택선을 `00`으로 맞추면, 데이터 D는 오직 Y0 선으로만 쏙 빠져나갑니다.
* 선택선을 `11`로 맞추면, 데이터 D는 Y3 선으로만 나갑니다.

```ascii
[1x4 디멀티플렉서 (DEMUX)의 구조와 진리표]

             /---> Y0       선택 (S1, S0) | 출력 위치
           / |              --------------+-------------
 데이터 D ---| |---> Y1        0   0   | Y0 = D (나머지 0)
 (입력 1개)  | |---> Y2        0   1   | Y1 = D (나머지 0)
           \ |              1   0   | Y2 = D (나머지 0)
             \---> Y3       1   1   | Y3 = D (나머지 0)
               ^  ^
             S1 S0 (조종석)
```

📢 **섹션 요약 비유**: DEMUX는 우체국의 '택배 분류 컨베이어 벨트'입니다. 하나의 메인 벨트(입력)로 택배가 쏟아져 들어오면, 시스템(선택선)이 택배의 주소를 읽고 서울, 부산, 대구, 광주행 4개의 미끄럼틀(출력) 중 딱 한 곳으로만 상자를 툭 쳐서 떨어뜨려 줍니다.

## Ⅱ. 내부 논리 회로: 디코더(Decoder)와의 놀라운 유사성
DEMUX의 내부를 뜯어보면, 이전에 배운 **디코더(Decoder)와 회로가 99% 똑같이 생겼다**는 놀라운 사실을 알 수 있습니다.
디코더는 입력선을 받아서 해당 출력선을 1로 켜주는 장치였고, 전체 회로를 켜고 끄는 Enable(E) 핀이 있었습니다.

DEMUX는 이 디코더의 **입력선(A, B)을 선택선(S1, S0)으로 이름만 바꾸고, Enable(E) 핀을 데이터 입력선(D)으로 사용**하는 꼼수를 부린 것입니다. 
선택선이 `00`일 때 Y0의 AND 게이트만 켜지는데, 이때 입력선(D)이 1이면 1이 통과하고 0이면 0이 통과하므로 완벽한 분배기 역할을 하게 됩니다.

```ascii
[디코더와 DEMUX의 완벽한 변신 (2x4 Decoder == 1x4 DEMUX)]

 [Decoder 로 쓸 때]                 [DEMUX 로 쓸 때]
 A, B 핀 -> (주소 입력)             A, B 핀 -> (S1, S0 선택선으로 씀)
 E 핀    -> (그냥 항상 1로 켜둠)     E 핀    -> (데이터 D 입력으로 씀)
 출력    -> (해당 번호에 1 출력)      출력    -> (선택된 번호로 D 값 출력)
```

📢 **섹션 요약 비유**: 디코더와 DEMUX는 '양면점퍼'입니다. 겉으로 입으면 조명 스위치(디코더) 역할을 하고, 뒤집어서 입으면 택배 분류기(DEMUX) 역할을 하는 완전히 똑같은 옷(회로)입니다. 설계자는 칩을 아끼기 위해 이 회로 하나를 두 가지 용도로 다 써먹습니다.

## Ⅲ. 아키텍처 관점: 버스(Bus) 분배와 직렬-병렬 변환
CPU 아키텍처와 통신 시스템에서 DEMUX는 MUX와 항상 짝을 지어 다닙니다.

1. **레지스터 쓰기 제어**: ALU가 덧셈을 끝내고 결과(데이터 1개)를 뱉어냈습니다. 이 결과를 32개의 레지스터 중 5번 레지스터에 저장해야 합니다. 이때 1x32 DEMUX가 ALU의 결과를 받아, 선택선(00101)에 맞춰 5번 레지스터로 가는 길만 열어줍니다.
2. **직렬-병렬 변환 (TDM 복원)**: 인터넷 해저 광케이블(1가닥)을 통해 여러 명의 데이터가 섞여서(시분할 다중화) 미친 듯이 들어옵니다. 케이블 끝에 있는 DEMUX는 수신 타이밍(선택선)에 맞춰 1번 사람의 데이터는 1번 컴퓨터로, 2번 사람의 데이터는 2번 컴퓨터로 정확하게 흩뿌려 줍니다.

```ascii
[통신 시스템에서의 MUX와 DEMUX 환상의 짝꿍]

 (송신부: 서울)                     (단일 케이블)                 (수신부: 부산)
 PC 1 ---+                                                    +--- PC 1
 PC 2 ---+--> [ MUX (합치기) ] ==================> [ DEMUX (뿌리기) ] ---+--- PC 2
 PC 3 ---+     시간별로 1-2-3 순서로 쏨              들어온 순서대로 분배  +--- PC 3
```

📢 **섹션 요약 비유**: 송신부의 MUX가 여러 개의 시냇물을 모아 하나의 거대한 강(통신 케이블)으로 합쳐버리는 댐이라면, 수신부의 DEMUX는 그 거대한 강물을 다시 여러 개의 작은 시냇물로 쪼개어 각 농가(컴퓨터)에 골고루 물을 대어주는 수로입니다.

## 개념 맵 (Concept Map)
```ascii
[디멀티플렉서 (Demultiplexer, DEMUX)]
   │
   ├── 정의: 1개의 데이터 입력을 2^N개의 출력 중 1개로 분배 (Distributor)
   │     └── 구성: N개의 선택선(Select Line)으로 출력 경로 결정
   │
   ├── 논리 구조
   │     └── 디코더(Decoder)와 물리적으로 100% 동일한 회로
   │         (Decoder의 Enable 핀을 데이터 입력으로 사용)
   │
   └── 아키텍처 응용
         ├── 데이터 버스 라우팅: CPU 연산 결과를 특정 레지스터에 분배/저장
         └── 통신 네트워크: 단일 케이블로 수신된 시분할 데이터(TDM)를 다중 채널로 복원
```
