## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디멀티플렉서(Demultiplexer, DEMUX)는 멀티플렉서의 역방향 연산을 수행하는 조합 논리회로로, **단 하나의 데이터 입력선으로 들어온 정보를 $2^N$개의 출력선 중 선택된 단 하나로 분배해 주는 '디지털 분배기'**이다.
> 2. **가치**: N개의 선택선(Select Line)을 조절하여 하나의 공통 데이터 스트림을 여러 목적지로 흩뿌려줌으로써, CPU의 연산 결과를 특정 레지스터에 저장하거나 **시분할 다중화(TDM)된 통신 신호를 복원(Demultiplexing)**하는 데 필수적이다.
> 3. **융합**: 내부 논리 구조상 **디코더(Decoder)의 Enable 핀을 데이터 입력으로 사용하는 구조와 물리적으로 100% 일치**하며, 이를 통해 칩 설계 시 하나의 모듈을 용도에 따라 해독기 또는 분배기로 변신시켜 사용하는 아키텍처 재사용성을 제공한다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: 디멀티플렉서는 '데이터 분배기(Data Distributor)'라고도 불리며, 나팔 모양의 구조를 갖는다. 들어오는 입구는 하나(1)인데 나가는 출구는 여러 개($2^N$)다. 어떤 출구로 데이터를 내보낼지는 '선택 신호(Selection)' 조종간이 결정한다.

- **필요성**: 디멀티플렉서는 단일 채널로 전송된 대량의 데이터를 수신측에서 각각의 목적지(레지스터, 출력 장치 등)로 정확히 복원하여 분배하기 위해 반드시 필요하다. 특히 하나의 공통 버스나 통신 회선을 공유하는 멀티플렉싱 환경에서, 시간차를 두고 섞여 들어온 신호들을 원래의 개별 스트림으로 분리해내는 '역다중화(Demultiplexing)' 기능을 수행한다. 이는 CPU 연산 결과의 정밀한 레지스터 저장과 고속 통신 네트워크의 데이터 복원 무결성을 보장하는 시스템 하역 아키텍처의 필수 요소다.
- **💡 비유**: DEMUX는 우체국의 '자동 분류 컨베이어 벨트'와 같다. 하나의 메인 벨트(입력)로 쏟아져 들어오는 택배들을, 시스템(선택선)이 주소를 읽고 서울, 부산, 대구, 광주행 4개의 미끄럼틀(출력) 중 딱 한 곳으로만 상자를 툭 쳐서 떨어뜨려 주는 장치다.
- **등장 배경**: 
  멀티플렉서(MUX)를 통해 여러 데이터를 하나로 묶어 보내는 데 성공한 인류는, 그 도착지에서 다시 데이터를 주인들에게 나눠줘야 하는 과제에 직면했다. 한 가닥의 통신선으로 섞여 들어온 음악, 영상, 텍스트 데이터를 다시 각각의 전용 출력 장치로 분리해 배달하기 위해 탄생한 것이 DEMUX이며, 이는 현대의 복잡한 버스(Bus) 시스템과 네트워크 배분 아키텍처의 근간이 되었다.

단 하나의 데이터가 선택된 목적지로 정확히 배달되는 과정을 시각화하면 다음과 같다.

```text
  ┌──────────────────────────────────────────────────────────────┐
  │         디멀티플렉서(DEMUX)의 1-to-4 분배 원리 (1x4 DEMUX 예시)      │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │   [ 1개의 데이터 입력 D ]         [ 선택 조종간 (Select) ]       │
  │             │                     S1  S0 | 출력 위치          │
  │             │                    ----+----+------------      │
  │             ▼                     0   0  | Y0 = D (나머지 0)  │
  │      ┌─────────────┐              0   1  | Y1 = D (나머지 0)  │
  │      │    DEMUX    │ ──▶ Y0       1   0  | Y2 = D (나머지 0)  │
  │      └─────────────┘ ──▶ Y1       1   1  | Y3 = D (나머지 0)  │
  │             ▲        ──▶ Y2                                  │
  │             │        ──▶ Y3                                  │
  │           S1 S0                                              │
  │                                                              │
  │ * 철학: "나(DEMUX)는 공용 통로를 통해 들어온 소중한 정보를         │
  │   주소가 적힌 단 한 명의 주인에게만 정확히 배달하는 집배원이다!"      │
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** DEMUX는 '권력의 분산'을 담당한다. 1x4 DEMUX의 경우, 입력 D는 단 하나다. 만약 선택 신호가 `01(이진수 1)`로 들어오면, DEMUX 내부의 물리적 전선이 1번 출력선(Y1)에만 철컥 연결된다. 이때 Y0, Y2, Y3는 입력 데이터가 무엇이든 상관없이 무조건 0(Low) 상태로 잠잠하게 유지된다. 이 '스프링클러' 효과 덕분에, CPU가 계산한 단 하나의 결과값을 32개의 레지스터 중 사용자가 지정한 딱 한 곳에만 안전하게 '저장(Write)'할 수 있는 물리적 질서가 유지되는 것이다.

- **📢 섹션 요약 비유**: DEMUX는 아파트 단지의 '수돗물 밸브 계량기'입니다. 수원지에서 내려오는 거대한 단 하나의 파이프(입력)가 아파트 단지 입구에 도착하면, 관리인(선택선)이 밸브를 조작하여 오직 103호(출력) 수돗찌만 콸콸 나오게 하고 나머지 집은 물이 안 나오게 막는 완벽한 배분 시스템입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소 (디코더와의 일란성 쌍둥이 구조)

DEMUX의 가장 충격적인 아키텍처적 진실은, 이전에 배운 '디코더(Decoder)'와 회로 구조가 100% 동일하다는 점이다.

| 요소명 (Decoder 기준) | 요소명 (DEMUX 기준) | 물리적 역할 및 변신 | 비유 |
|:---|:---|:---|:---|
| **입력 핀 (A, B)** | **선택 핀 (S1, S0)** | 어떤 출력을 고를지 결정하는 '주소' 역할로 이름만 바뀜 | 편지 봉투에 적힌 '주소' |
| **Enable (E) 핀** | **데이터 입력 (D)** | 회로를 끄고 켜던 스위치 핀이 '전달할 알맹이 데이터'로 변신 | 편지 봉투 안의 '내용물' |
| **출력 핀 (D0~D3)** | **출력 핀 (Y0~Y3)** | 해독된 번호를 켜던 핀이 '알맹이 데이터를 뱉는 구멍'으로 변신 | 목적지 '우체통' |
| **AND 게이트 배열** | **AND 게이트 배열** | 선택된 주소와 알맹이 데이터를 곱해 최종 분출 | 주소와 내용물이 맞아야 배달 완료 |

### 심층 동작 원리: 디코더를 DEMUX로 부려먹는 '역발상 매핑'

칩 설계자는 칩 면적을 아끼기 위해 디코더와 DEMUX를 따로 설계하지 않는다. 이름표만 바꿔서 돌려쓴다.

```text
  ┌──────────────────────────────────────────────────────────────┐
  │         아키텍처의 둔갑술: Decoder == DEMUX 증명                 │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │   [ 2x4 디코더 회로 ]              [ 1x4 디멀티플렉서 회로 ]       │
  │                                                              │
  │    A ──▶ [입력]                     S1 ──▶ [선택]                │
  │    B ──▶ [입력]                     S0 ──▶ [선택]                │
  │    E ──▶ [활성화(항상 1)]            D  ──▶ [데이터 입력(0 또는 1)] │
  │                                                              │
  │ * 마법: 디코더의 E 핀에 '데이터'를 실어 보내면, 그 데이터가          │
  │   선택된 출력선으로 그대로 복사되어 나간다!                        │
  │   ( E가 0이면 출력도 0, E가 1이면 출력도 1 => 데이터 전달 완료 )     │
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 반도체 라이브러리 설계의 경제학이다. 디코더는 원래 Enable(E) 핀이 1일 때만 작동하는 장치다. 아키텍트는 여기서 무릎을 탁 친다. "E 핀에 1을 넣으면 선택된 선이 1이 되고, E 핀에 0을 넣으면 선택된 선도 0이 되네? 그럼 E 핀이 곧 데이터 입력(D) 아니야?" 
이 위대한 발견 덕분에, 현대의 모든 CPU 안에는 **'디코더/DEMUX 통합 모듈'**이 박혀 있다. 주소를 풀 때는 디코더로 부려먹고, 데이터를 뿌릴 때는 DEMUX로 부려먹는다. 하드웨어 리소스를 100% 재활용하여 칩 면적을 반토막 내는 아키텍처적 연금술의 정수다.

- **📢 섹션 요약 비유**: 이것은 '전등 스위치'를 '전기 콘센트'로 사용하는 것과 같습니다. 스위치(디코더)는 단순히 불을 켜는 용도지만, 그 스위치 뒤에 가전제품(데이터)을 연결하면 전기를 원하는 방으로 전달하는 통로(DEMUX)가 됩니다. 겉모양(회로)은 똑같은데 무엇을 꽂느냐에 따라 용도가 바뀌는 다재다능한 부품입니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 심층 기술 비교: 멀티플렉서 (MUX) vs 디멀티플렉서 (DEMUX) 듀오

통신과 데이터 버스의 양쪽 끝을 책임지는 영혼의 단짝이다.

| 비교 항목 | 멀티플렉서 (MUX) | 디멀티플렉서 (DEMUX) | 아키텍처 판단 포인트 |
|:---|:---|:---|:---|
| **물리적 형상** | 깔때기 (Convergence) | 나팔 (Divergence) | 흐름의 방향성 |
| **논리적 역할** | **데이터 깔때기** | **데이터 스프링클러** | 모으는가 뿌리는가 |
| **선택선(S)의 의미** | "누가 들어올래?" | "누구에게 줄까?" | 타깃의 위치 |
| **통신 융합** | 송신단 (Serializing) | 수신단 (Deserializing) | 데이터의 직렬화/병렬화 |
| **내부 로직 뼈대** | **AND-OR 트리** | **디코더(AND 배열)** | 게이트 뎁스의 차이 |

### 과목 융합 관점

- **네트워크 및 통신 (TDM 복원)**: MUX가 100명의 목소리를 1가닥 선에 섞어서 부산으로 보냈다면, 부산에 도착한 데이터 뭉텅이를 다시 100명의 전화기로 정확히 흩뿌려주는 최종 집배원이 바로 **DEMUX**다. MUX와 DEMUX의 클럭(박자)이 0.000001초라도 어긋나면 아빠 목소리가 옆집 아저씨 전화기로 전달되는 통신 참사가 일어난다. 이 둘의 '동기화(Synchronization)'가 현대 통신 아키텍처의 생명줄이다.
- **CPU 레지스터 파일 (Register Write)**: CPU 연산 장치(ALU)가 덧셈을 마친 결과값(1개)을 다시 32개의 레지스터(저장소) 중 한 곳에 집어넣어야 한다. 이때 ALU의 엉덩이에 **1x32 DEMUX**가 찰싹 붙어 있다. 명령어가 가리키는 주소(선택선)에 맞춰 연산 결과를 정확한 레지스터 방으로 쏘아 넣어 '쓰기(Write)' 작업을 완수한다.

```text
  ┌──────────────────────────────────────────────────────────────┐
  │         아키텍처의 황금 콤비: MUX와 DEMUX를 이용한 자원 공유        │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │   [사용자 A] ──┐                                      ┌──▶ [사용자 A] │
  │   [사용자 B] ──┼─▶ [ MUX ] ───(공용 고속도로)───▶ [ DEMUX ] ─┼──▶ [사용자 B] │
  │   [사용자 C] ──┘      ▲                          ▲      └──▶ [사용자 C] │
  │                      │                          │                    │
  │                    [클럭] ──────────────────────[클럭]                  │
  │                                                              │
  │ * 핵심: 송신측 MUX와 수신측 DEMUX가 똑같은 박자(Clock)로 선택선을      │
  │   돌려야 데이터가 섞이지 않는다. 이를 '다중화(Multiplexing)'라 한다.   │
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 현대 문명을 지탱하는 '공유 경제'의 하드웨어판이다. 비싼 금금속 전선(공용 고속도로)을 사용자마다 하나씩 깔아주는 건 낭비다. 그래서 입구에 MUX를, 출구에 DEMUX를 세워두고 클럭 신호에 맞춰 광속으로 입구와 출구를 동시에 바꿔가며 선을 나눠 쓴다. 이를 **시분할 다중화(Time Division Multiplexing)**라 부른다. 우리가 지하철 한 노선을 수만 명이 시간차를 두고 나눠 타는 것과 완벽히 같은 논리다. 칩 설계자는 이 MUX-DEMUX 쌍을 얼마나 효율적으로 배치하느냐에 따라 칩 내부의 배선 혼잡도(Congestion)를 90% 이상 걷어낼 수 있다.

- **📢 섹션 요약 비유**: MUX와 DEMUX는 '지하철'과 같습니다. 여러 동네 사람들(데이터)이 입구(MUX)에서 기차 한 칸에 옹기종기 모여 타서 이동한 뒤, 목적지 역의 출구(DEMUX)에서 각자의 집으로 뿔뿔이 흩어져 가는 질서 정연한 대중교통 시스템입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 자율주행차 센서 데이터 허브의 데이터 낙수 사고**: 
  **상황**: 자동차 지붕 위 단 하나의 메인 통신선으로 쏟아지는 수만 개의 라이다(LiDAR) 신호를, 차 내부의 제어 칩 10개로 분산시켜야 함. 그런데 DEMUX의 선택 신호 타이밍이 꼬여서 브레이크 제어 칩으로 가야 할 데이터가 오디오 칩으로 가버림.
  **판단**: "DEMUX 선택선의 전파 지연(Propagation Delay) 관리 실패"다. 아키텍트는 DEMUX 입구에 **'버퍼 레지스터(Latch)'**를 융합 설계한다. 선택 신호가 완전히 안정화(Settling)될 때까지 데이터를 잠시 붙잡아 두었다가, 주소가 확실해지는 찰나에만 댐 문을 열어 데이터를 쏟아붓는다. 0.1나노초의 정밀한 타이밍 제어가 자율주행의 생사를 결정짓는 아키텍처적 순간이다.

2. **시나리오 — 고화질 디스플레이 패널의 화소(Pixel) 구동 병목**:
  **상황**: 4K 모니터의 수백만 개 픽셀에 색상 데이터를 보내야 함. CPU에서 수백만 가닥의 전선을 모니터로 보낼 수는 없음.
  **판단**: **'직렬-병렬 변환(SerDes)'** 아키텍처를 도입한다. CPU는 단 1가닥의 초고속 전선으로 데이터를 실어 보낸다. 모니터 패널 끝단에는 수만 개의 **DEMUX**들이 층층이 깔려 있어, 들어온 데이터를 순식간에 수평/수직 전선으로 골고루 뿌려준다. 전선 1가닥을 수만 가닥으로 복사 증폭시켜 화면을 채우는 디스플레이 분배의 마법이다.

```text
  ┌──────────────────────────────────────────────────────────────┐
  │         마이크로아키텍처 합성(Synthesis) 시 DEMUX 설계 의사결정      │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │   [칩 내부의 데이터를 여러 목적지로 분배하는 경로를 설계할 때]        │
  │                │                                             │
  │                ▼                                             │
  │    이미 칩 안에 주소 해독기(Decoder)가 설계되어 있는가?              │
  │          ├─ 예 ─────▶ [새로 설계 금지! 기존 Decoder의 Enable 핀 활용] │
  │          │                     │                            │
  │          │                     └─▶ [면적 절약 및 로직 공유(Logic Reuse)] │
  │          └─ 아니오                                          │
  │                │                                             │
  │                ▼                                             │
  │    분배받는 목적지들의 부하(Load)가 너무 커서 신호가 약해지는가?      │
  │          ├─ 예 ─────▶ [DEMUX 출력단에 고출력 버퍼(Buffer) 트리 삽입]  │
  │          │                                                  │
  │          └─ 아니오 ──▶ [표준 셀 기반 합성 후 최종 라우팅 승인]         │
  │                                                              │
  │  최종 조치: DEMUX는 칩의 '스프링클러'다. 물이 골고루 닿게 압력을 조절하라! │
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 반도체 설계 효율화의 핵심인 '로직 재사용' 전략이다. 유능한 아키텍트는 절대 중복 투자를 하지 않는다. 뺄셈이 필요하면 덧셈기를 속여 쓰고(2의 보수), 분배가 필요하면 디코더를 속여 쓴다(DEMUX 치환). 칩 안에 이미 주소 해독을 위해 깔려있는 수천 개의 디코더들을 째려보며, 그들의 노는 핀(Enable)에 데이터를 실어 보내 분배기 업무까지 '이중 알바'를 시키는 것이 삼성/TSMC의 초미세 공정 원가 절감 비결이다. 하드웨어 설계는 단순히 선을 잇는 것이 아니라, 이미 있는 선을 어떻게 다목적으로 우려먹느냐(Resource Sharing)의 머리 싸움이다.

### 도입 체크리스트
- **글리치 (Glitch) 노이즈 확산 방지**: DEMUX는 입력이 하나라, 입구에서 노이즈가 한 번 발생하면 선택된 모든 출력 경로로 노이즈가 100% 복제되어 퍼져 나간다. 입구 전선에 **'슈미트 트리거(Schmitt Trigger)'**나 저역 통과 필터를 융합하여 깨끗한 데이터만 분배되도록 입구 컷을 철저히 했는가?
- **비활성 출력의 상태 결정**: 선택되지 않은 99%의 전선들을 0(GND)으로 꽉 묶어둘 것인가, 아니면 공중에 띄워둘(Floating) 것인가? 노이즈에 의한 오작동을 막기 위해 반드시 **풀다운(Pull-down) 저항**을 아키텍처에 융합하여 "선택 안 된 놈은 죽은 듯이 가만히 있어라"는 명령을 물리적으로 박아 넣었는가?

### 안티패턴
- **데이터와 선택 신호의 박자 불일치 (Skew 재앙)**: 데이터(D)는 도착했는데, 어디로 보낼지 결정하는 선택선(S) 신호가 0.001초 늦게 도착하는 상황. 데이터가 엉뚱한 옆집 문으로 0.001초 동안 새어 들어갔다가 다시 제자리로 돌아오는 끔찍한 '신호 번짐' 현상이 발생한다. 이를 무시하면 레지스터에 쓰레기 값이 저장되어 시스템이 영구적으로 뻗어버린다. 반드시 데이터와 선택선의 도착 시간을 칼같이 맞추는 타이밍 튜닝(STA)을 거쳐야 한다.

- **📢 섹션 요약 비유**: DEMUX 설계 미숙은 '술 취한 집배원'과 같습니다. 편지(데이터)는 들고 왔는데, 집 번호(선택선)를 헷갈려서 옆집 우체통에 편지를 잘못 넣었다가 다시 꺼내오는 꼴입니다. 찰나의 실수지만 사생활(데이터 무결성)은 이미 박살 났고, 마을(시스템)은 대혼란에 빠지게 됩니다.

### 정량/정성 기대효과

| 구분 | 최적화 전 | 최적화 후 | 개선 효과 |
|:---|:---|:---|:---|
| **정량** | 100개의 장치에 데이터를 보내기 위해 100가닥 전선 필요 | 1x100 DEMUX를 통한 단일 버스 분배 | 칩 내부 라우팅 면적 **90% 이상 극단적 절감** |
| **정량** | 소프트웨어 분배 로직 실행에 100 사이클 소요 | 하드웨어 DEMUX의 즉각적 데이터 흩뿌리기 | 데이터 분배 지연 시간 **100배 단축** |
| **정성** | 데이터가 어디로 갈지 몰라 혼선 발생 | 1:N 확정적 분배 체계 기반의 데이터 무결성 확보 | 칩 내부 통신 아키텍처의 신뢰성 및 보안성(Isolation) 확보 |

### 미래 전망
- **광자 디멀티플렉서 (Photonic DEMUX) - 빛의 분산**: 프리즘이 빛을 무지개색으로 쪼개듯, 하나의 광섬유로 들어온 수만 개의 레이저 신호를 주파수별로 0.1나노초 만에 쪼개어 배달하는 실리콘 포토닉스 DEMUX가 차세대 양자 인터넷의 관문이 될 것이다. 열이 전혀 나지 않는 광속 배달 시대가 열린다.
- **DNA/바이오 분배기**: 살아있는 세포의 단백질 효소 농도를 선택 신호로 삼아, 특정 유전자 정보를 원하는 염기서열로 분배하는 '바이오 DEMUX' 아키텍처가 암 치료 칩의 핵심 기술로 연구되고 있다. 실리콘을 넘어 생명의 정보 흐름까지 지배하는 아키텍처의 확장이다.

### 참고 표준
- **USB (Universal Serial Bus) Protocol**: 단 4가닥의 전선(직렬)으로 들어온 키보드, 마우스, 프린터 데이터를 어떻게 각각의 소프트웨어 드라이버로 DEMUX하여 배달할지 약속한 전 세계 가전기기의 절대 표준.
- **PCI Express (PCIe) Lane Steering**: 그래픽카드와 CPU 사이의 수십 가닥 차선으로 데이터를 광속 분배하기 위해 DEMUX 기술을 어떻게 물리적으로 구현할지 정의한 고성능 아키텍처 규격.

"한 가닥의 좁은 입구"를 "수만 갈래의 풍요로운 출구"로 확장하는, 아키텍처의 위대한 분배자 '디멀티플렉서'의 진화 로드맵은 다음과 같다.

```text
  ┌──────────────────────────────────────────────────────────────┐
  │         디지털 정보의 스프링클러: 디멀티플렉서(DEMUX) 진화 궤적      │
  ├──────────────────────────────────────────────────────────────┤
  │                                                              │
  │   [1단계: 수동 분배]       [2단계: 논리적 융합]     [3단계: 지능형 라우팅]  │
  │                                                              │
  │   전선 바꿔 끼우기 ───▶ 디코더-DEMUX 재사용 ──▶ 가상화 오버레이 네트워크 │
  │  (노가다, 에러 폭발)    (칩 면적 50% 다이어트)    (데이터가 스스로 길을 찾음)│
  │ "주소를 직접 꽂아라!"   "회로 하나로 다 해먹자!"   "목적지가 지능을 갖다!"   │
  └──────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 이 짧은 로드맵은 인류가 '분배의 효율'을 어떻게 극대화해왔는지 보여준다. 1단계: 초기엔 전선을 손으로 옮겨 꽂았다. 2단계: 하지만 칩이 작아지며 "이미 있는 디코더 회로를 거꾸로 돌려서 분배기로 쓰자!"는 기막힌 융합 아이디어가 터졌다. 이 순간 반도체는 덩치를 키우지 않고도 수만 개의 장치를 거느리는 '대인배'가 되었다. 3단계: 이제는 단순한 하드웨어 분배를 넘어, 클라우드 가상화 기술을 통해 수억 개의 가상 주소로 데이터를 빛의 속도로 흩뿌려주는 '보이지 않는 지능형 배달 시대'로 진입하며 문명의 모든 정보 분수를 제어하고 있다.

- **📢 섹션 요약 비유**: 디멀티플렉서의 진화는 '상수도 시스템'의 발전과 같습니다. 옛날엔 우물에서 물을 직접 길어다 날랐지만(수동), 이제는 거대한 메인 관로(입력) 하나만 있으면 거미줄처럼 뻗은 수도관(출력)과 밸브(선택선)를 통해 전 국민의 주방(데이터 목적지)까지 맑은 물을 0.1초 만에 배달해 주는 똑똑한 스프링클러 시스템이 된 셈입니다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **멀티플렉서 (MUX)** | DEMUX의 영혼의 쌍둥이. MUX가 여러 길을 하나로 합쳤다면, DEMUX는 그 종착역에서 다시 보따리를 풀어 주인들에게 나눠주는 환상의 파트너다. |
| **디코더 (Decoder)** | DEMUX와 몸뚱아리(회로)를 공유하는 전생의 형제. 데이터를 싣지 않고 번호만 켜면 디코더, 번호표 자리에 데이터를 실어 보내면 DEMUX가 되는 변신 관계다. |
| **직렬화/병렬화 (SerDes)** | 통신 아키텍처의 핵심. 1가닥으로 줄여 보낸(Serial) 데이터를 다시 여러 가닥(Parallel)으로 펴주는 DEMUX의 마법 같은 데이터 복원 능력이다. |
| **시분할 다중화 (TDM)** | MUX와 DEMUX가 박자를 맞춰 도로를 나눠 쓰는 기술. 1초를 100조각으로 나눠 100명의 데이터를 순서대로 배달하는 현대 통신의 절대 법칙이다. |
| **팬아웃 (Fan-out)** | DEMUX 1가닥 입구 전력이 100가닥 출구로 나뉘면서 약해지는 현상. 이를 보강하기 위해 출구마다 펌프(Buffer)를 다는 노가다 설계가 뒤따른다. |
