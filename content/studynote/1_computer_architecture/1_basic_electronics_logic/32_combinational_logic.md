> **핵심 인사이트**
> 1. 조합 논리회로(Combinational Logic)는 오직 **현재의 입력값(Input)에 의해서만 현재의 출력값(Output)이 결정**되는 논리 회로이다.
> 2. 내부적으로 데이터를 기억하는 메모리(저장) 소자나 피드백 루프(Feedback Loop)가 전혀 없기 때문에 **과거의 상태를 기억하지 못한다.**
> 3. CPU 아키텍처 내에서 데이터의 덧셈, 뺄셈, 비교 등을 수행하는 **산술논리연산장치(ALU)**의 핵심 뼈대를 이룬다.

---

## Ⅰ. 조합 논리회로의 특징: "기억 상실증"
조합 논리회로는 AND, OR, NOT, XOR 게이트들을 거미줄처럼 조합해서 만듭니다. 가장 중요한 특징은 **기억(Memory)이 없다**는 것입니다.

입력 A와 B에 전기 신호가 들어오면, 정해진 게이트의 길을 따라 전기가 흘러 결과를 뱉어내고 끝납니다. 이전에 어떤 값이 들어왔었는지, 방금 전 결과가 무엇이었는지 전혀 알지 못합니다. 마치 자판기에 500원을 넣으면 콜라가 나오고, 다시 500원을 넣으면 또 콜라가 나오는 것처럼, 오직 '지금 이 순간 들어온 돈(입력)'에만 반응합니다.

```ascii
[조합 논리회로의 블랙박스 모델]

 입력 1 (현재) ----> +-------------------+
 입력 2 (현재) ----> |    조합 논리회로  | ----> 출력 (현재 입력에만 의존)
 입력 3 (현재) ----> | (기억 장치 없음)  |
                     +-------------------+
```

📢 **섹션 요약 비유**: 조합 논리회로는 식당의 '주문 키오스크'입니다. 지금 내가 치즈버거 세트를 누르면 정확히 치즈버거 세트 영수증이 나옵니다. 키오스크는 어제 내가 무엇을 먹었는지(과거 상태) 전혀 기억하지 않고 오직 지금 누른 버튼(현재 입력)에만 100% 정직하게 반응합니다.

## Ⅱ. 대표적인 조합 논리회로들
컴퓨터가 연산을 수행하기 위해 사용하는 대부분의 기초 부품들이 바로 조합 논리회로입니다. 이들은 진리표(Truth Table)를 만들고 부울 대수로 최적화하여 칩으로 깎아냅니다.

1. **가산기 (Adder) / 감산기 (Subtractor)**: 두 이진수를 더하거나 빼는 회로. (ALU의 핵심)
2. **비교기 (Comparator)**: 두 숫자가 같은지, 어느 쪽이 더 큰지 비교하는 회로. (XNOR 게이트 등 활용)
3. **디코더 (Decoder)**: 압축된 이진수(예: 3비트 011)를 받아 특정 전구 하나(예: 8개 중 3번째 선)만 켜주는 해독기.
4. **인코더 (Encoder)**: 여러 개의 전구 중 하나에 불이 들어오면, 그걸 짧은 이진수(예: 3비트)로 압축해 주는 장치.
5. **멀티플렉서 (MUX)**: 여러 갈래의 입력길 중 하나만 골라서 출력길로 연결해 주는 데이터 스위치(라우터).

```ascii
[대표적인 조합 논리: 멀티플렉서(MUX)의 예]

 입력 A (0) ------\   +-------+
                  |---|       |
 입력 B (1) ------+---|  MUX  |-----> 출력: 선택 신호(S)가 0이면 A(0), 1이면 B(1) 출력
                  |---|       |
 입력 C (1) ------/   +-------+
                          ^
                          |
                      선택 신호 (S)  <- 이 순간의 S값에 따라 즉시 출력이 바뀜
```

📢 **섹션 요약 비유**: 멀티플렉서(MUX)는 기찻길의 '선로 변경기(스위치)'입니다. 조종사가 현재 레버(선택 신호)를 왼쪽으로 당기고 있으면 무조건 왼쪽 기차가 통과하고, 오른쪽으로 당기면 무조건 오른쪽 기차가 통과합니다. 과거에 어떤 기차가 지나갔는지는 전혀 상관없습니다.

## Ⅲ. 아키텍처 관점: 전파 지연 (Propagation Delay)
조합 논리회로에는 클럭(Clock, 시스템의 심장 박동)이 연결되어 있지 않습니다. 신호가 입력되면 전기가 구리선과 게이트들을 통과하는 물리적인 시간, 즉 **전파 지연(Propagation Delay)**만큼의 시간이 지난 후 물 흐르듯 자연스럽게 결과가 쏟아져 나옵니다.

CPU를 설계할 때 이 전파 지연은 매우 중요합니다. 수백 개의 논리 게이트를 거쳐가는 조합 회로(예: 64비트 곱셈기)는 결과를 내는 데 시간이 꽤 걸립니다. 만약 CPU의 클럭이 이 연산이 끝나는 시간보다 더 빨리 다음 명령을 내려버리면, 연산 도중의 쓰레기 값이 저장되어 시스템이 멈추게(Crash) 됩니다.

```ascii
[조합 회로와 딜레이 타이밍]

 입력 투입 (t=0) -----(게이트 통과 중...)-----> 정확한 결과 출력 (t=5ns)
      ^                                               ^
      |                                               |
  (이 사이에 클럭 펄스가 쳐서 값을 가져가면 오작동 발생!)
```

📢 **섹션 요약 비유**: 전파 지연은 거대한 '핀볼 게임'입니다. 구슬(입력)을 쏘아 올리면 수많은 장애물(게이트)을 튕기며 내려옵니다. 우리는 구슬이 완전히 바닥에 떨어져 점수판(출력)에 점수가 뜰 때까지 조용히 기다려야만 정확한 점수를 알 수 있습니다.

## 개념 맵 (Concept Map)
```ascii
[조합 논리회로 (Combinational Logic)]
   │
   ├── 핵심 정의: 현재 출력 = f(현재 입력)
   │     └── 메모리(저장) 소자 및 피드백 루프 부재
   │
   ├── 주요 응용 컴포넌트
   │     ├── 연산: 가산기(Adder), 비교기(Comparator) -> ALU 구성
   │     └── 데이터 라우팅: 디코더(Decoder), 멀티플렉서(MUX) -> 제어 유닛/메모리 선택
   │
   └── 시스템 설계 고려사항
         └── 전파 지연(Propagation Delay) -> 시스템 최대 클럭 주파수를 제한하는 요인
```

## 어린이 비유
조합 논리회로는 **자판기**와 같아요! 어제 내가 사이다를 뽑았든 환타를 뽑았든, 오늘 내가 다시 '콜라 버튼(현재 입력)'을 누르면 무조건 똑같은 '콜라(현재 출력)'가 덜컹 하고 나와요. 이 자판기는 기억력이 아예 없어서, 오직 내가 방금 꾹 누른 버튼만 보고 일하는 아주 정직한 기계랍니다.
