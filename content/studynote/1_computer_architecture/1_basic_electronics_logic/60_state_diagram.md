> **핵심 인사이트**
> 1. 상태도(State Diagram)는 순차 논리회로(Sequential Logic)가 입력값에 따라 **어떤 상태(State)에서 다음 상태로 어떻게 변해가는지를 원과 화살표로 직관적으로 그려낸 설계 도면**이다.
> 2. 동기식 카운터(Counter)나 신호등 제어기 같은 복잡한 유한 상태 기계(FSM, Finite State Machine)를 하드웨어(플립플롭)로 구현하기 위한 첫 번째 설계 단계이다.
> 3. 컴퓨터 시스템 내부의 모든 제어 유닛(Control Unit) 동작은 이 상태도를 기반으로 논리식으로 변환되어 칩으로 만들어진다.

---

## Ⅰ. 상태도의 구성 요소: 원과 화살표
조합 논리회로는 진리표(Truth Table)로 모든 걸 설명할 수 있지만, 순차 논리회로는 과거의 기억(상태)이 있으므로 단순히 표만으로는 동작을 파악하기 힘듭니다. 그래서 원과 화살표를 이용해 시스템의 흐름을 시각화합니다.

* **상태 (State)**: 원(원형 노드)으로 표시하며, 원 안에 현재 저장된 값(예: `00`, `01`)을 적습니다. 플립플롭이 기억하고 있는 현재의 상태를 의미합니다.
* **전이 (Transition)**: 화살표로 표시하며, 한 상태에서 다른 상태로 넘어가는 길입니다.
* **입력/출력 조건**: 화살표 옆에 `입력값/출력값` 형태로 적습니다. "현재 이 원에 있는데, 입력 1이 들어오면 저 화살표를 타고 다음 원으로 가라"는 뜻입니다.

```ascii
[기본 상태도 (State Diagram) 예시 - 2비트 카운터]

 (입력 X=1일 때 숫자가 증가하는 카운터 상태도)

       +---(X=1)---> ( 01 ) ---+
       |                       |
       | (X=0)           (X=0) | (X=1)
      ( 00 ) <-------+         V
       ^             |      ( 10 )
       |             |         |
       +---(X=1)--- ( 11 ) <---+
```

📢 **섹션 요약 비유**: 상태도는 '보드게임의 말판'입니다. 원은 내가 지금 서 있는 칸(상태)이고, 주사위에서 나온 숫자(입력)가 화살표의 조건입니다. 주사위 숫자대로 화살표를 따라가면 내가 다음 턴에 서 있어야 할 칸(다음 상태)이 정확히 나옵니다.

## Ⅱ. 밀리 머신과 무어 머신 (Mealy vs Moore Machine)
유한 상태 기계(FSM)를 설계할 때, 출력을 결정하는 방식에 따라 크게 두 가지 철학으로 나뉩니다.

1. **무어 머신 (Moore Machine)**:
   * **특징**: 출력이 오직 **'현재 상태(자신이 서 있는 원)'**에 의해서만 결정됩니다. 입력이 1이든 0이든 내가 지금 서 있는 칸이 3번 칸이면 출력은 무조건 정해져 있습니다.
   * **상태도 표현**: 출력값을 아예 원(상태) 안에 적어 넣습니다.
   * **장점**: 설계가 단순하고, 출력이 클럭에 완벽히 동기화되어 에러(글리치)가 적습니다. (대부분의 CPU 설계에 쓰임)

2. **밀리 머신 (Mealy Machine)**:
   * **특징**: 출력이 **'현재 상태'와 '현재 들어온 입력' 모두**에 영향을 받습니다. 3번 칸에 서 있더라도 주사위가 1이 나오냐 2가 나오냐에 따라 즉각적으로 다른 출력을 내뱉습니다.
   * **상태도 표현**: 출력값을 화살표 옆(입력/출력)에 적습니다.
   * **장점**: 무어 머신보다 필요한 상태(원)의 개수를 확 줄일 수 있습니다. 하지만 입력이 변하자마자 출력이 변하므로 타이밍 관리가 꽤 까다롭습니다.

```ascii
[무어 머신과 밀리 머신의 시각적 차이]

 [무어(Moore) 상태도]          [밀리(Mealy) 상태도]
   입력: 1                    입력: 1 / 출력: 0
  -------> (상태 S1 / 출력 0)    -------> (상태 S1)
 (출력이 원 안에 고정됨)        (출력이 화살표를 타는 순간 결정됨)
```

📢 **섹션 요약 비유**: 무어 머신은 '정찰제 자판기'입니다. 콜라 버튼(상태)을 누르면 무조건 콜라 1캔(출력)이 나옵니다. 밀리 머신은 '시장 상인'입니다. 같은 사과(상태)를 팔더라도, 손님이 방긋 웃으면서 사는지 찡그리면서 사는지(입력)에 따라 덤으로 주는 귤의 개수(출력)가 매번 달라집니다.

## Ⅲ. 아키텍처 관점: 상태도에서 하드웨어 칩으로의 변환
엔지니어가 백지에 원과 화살표(상태도)를 그렸다고 해서 컴퓨터가 작동하는 것은 아닙니다. 이 그림을 실제 트랜지스터(하드웨어)로 바꾸는 엄격한 절차가 있습니다.

1. **상태도 작성**: 시스템이 어떻게 움직일지 원과 화살표로 기획합니다.
2. **상태표(State Table) 작성**: 그림을 진리표 같은 표로 옮깁니다. (현재 상태 | 입력 || 다음 상태)
3. **상태 할당(State Assignment)**: S0, S1 같은 문자에 00, 01 같은 이진수를 부여합니다.
4. **플립플롭 선택 및 논리식 도출**: D 플립플롭을 쓸지 T를 쓸지 정하고, 카르노 맵을 이용해 최소화된 논리식을 뽑아냅니다.
5. **회로도(Logic Diagram) 완성**: 논리 게이트와 플립플롭을 선으로 연결하여 칩을 완성합니다.

```ascii
[상태도에서 하드웨어로의 설계 흐름(Design Flow)]

 [1. 기획] 상태도 (원과 화살표)
     |
 [2. 번역] 상태표 (현재 상태 -> 다음 상태 표)
     |
 [3. 수학] 부울 대수 및 카르노 맵 (식 간소화)
     |
 [4. 물리] 실제 플립플롭과 AND/OR 게이트 배치 (칩 완성!)
```

📢 **섹션 요약 비유**: 상태도는 건축가의 '조감도 스케치'입니다. 예쁜 그림(상태도)을 그렸으면, 그걸 수치로 잰 '설계 도면(상태표)'으로 바꾸고, 필요한 '벽돌의 개수(논리식)'를 계산한 다음, 실제 벽돌(플립플롭)을 쌓아 올려야 비로소 진짜 건물(하드웨어)이 완성됩니다.

## 개념 맵 (Concept Map)
```ascii
[상태도 (State Diagram)]
   │
   ├── 정의: 순차 논리회로(FSM)의 상태 전이를 시각화한 설계 도면 (원과 화살표)
   │
   ├── FSM 설계 모델
   │     ├── 무어 머신(Moore): 출력 = f(현재 상태) -> 안정적 타이밍 (원 안에 출력 표기)
   │     └── 밀리 머신(Mealy): 출력 = f(현재 상태, 현재 입력) -> 상태 수 절감 (화살표에 표기)
   │
   └── 하드웨어 설계 워크플로우
         └── 상태도 -> 상태표(State Table) -> 플립플롭 입력식 도출(K-Map) -> 논리 회로도 구현
```

## 어린이 비유
상태도는 **놀이공원의 보물찾기 지도**예요! 동그라미는 우리가 있는 오두막(상태)이고, 화살표는 다음 오두막으로 가는 길이에요. 화살표 옆에는 "빨간 열쇠(입력 1)를 가지면 이 길로 가라"는 힌트가 적혀 있죠. 컴퓨터 만드는 아저씨들은 이 지도를 보면서, 컴퓨터가 길을 잃지 않고 정확하게 다음 오두막으로 찾아가게 해주는 로봇(플립플롭)들을 조립한답니다!
