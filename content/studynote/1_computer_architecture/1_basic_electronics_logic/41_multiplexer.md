> **핵심 인사이트**
> 1. 멀티플렉서(MUX)는 **여러 개의 데이터 입력선 중 단 하나만을 선택(Select)하여 하나의 출력선으로 연결**해 주는 '디지털 데이터 라우터(Router)'이다.
> 2. 입력선이 $2^N$개라면, 어느 선을 고를지 결정하는 **선택선(Select Line)은 $N$개**가 필요하다. (예: 4개의 입력을 고르려면 2개의 선택선이 필요)
> 3. 컴퓨터 아키텍처에서 여러 개의 레지스터 중 하나를 골라 ALU(연산장치)로 보내거나, 여러 프로그램의 데이터를 하나의 통신망으로 묶어 보낼 때(시분할 다중화) 반드시 사용되는 핵심 교통정리 회로이다.

---

## Ⅰ. 멀티플렉서의 구조: "선택받은 자만 지나가라"
멀티플렉서(줄여서 MUX, 먹스)는 깔때기 모양을 하고 있습니다. 들어오는 길은 여러 개($2^N$)인데 나가는 길은 오직 한 개(1)뿐입니다.
어떤 입력길의 데이터를 출력길로 내보낼지를 결정하는 조종간이 바로 **선택선(Select Line, S)**입니다.

예를 들어 4x1 MUX는 4개의 데이터 입력선(D0, D1, D2, D3)과 2개의 선택선(S1, S0)을 가집니다.
* 조종사가 선택선을 `00`으로 맞추면, 나머지 길은 꽉 막히고 오직 D0의 데이터만 출력선(Y)으로 쏙 빠져나갑니다.
* 선택선을 `11`로 맞추면, D3의 데이터만 빠져나갑니다.

```ascii
[4x1 멀티플렉서 (MUX)의 구조와 진리표]

 D0 ---\     선택 (S1, S0) | 출력 (Y)
 D1 ---| \   --------------+----------
 D2 ---|  |---> Y  0   0   |  D0 의 값
 D3 ---| /         0   1   |  D1 의 값
       /           1   0   |  D2 의 값
     ^  ^          1   1   |  D3 의 값
     S1 S0 (조종석)
```

📢 **섹션 요약 비유**: MUX는 기찻길의 '선로 변경 레버'입니다. 북쪽, 남쪽, 동쪽, 서쪽에서 기차가 동시에 달려오더라도, 역장이 레버(선택선)를 '남쪽'으로 당겨놓으면 오직 남쪽에서 온 기차만 중앙역(출력선)으로 들어올 수 있고 나머지는 대기해야 합니다.

## Ⅱ. 내부 논리 회로: AND와 OR의 조합
MUX의 내부를 까보면 디코더(Decoder)와 매우 비슷합니다. 
각각의 데이터 입력선(D0~D3)은 자신만의 **AND 게이트(관문)**를 하나씩 가지고 있습니다. 선택선(S)은 이 관문 중 딱 하나만 열어주는 열쇠 역할을 합니다.

선택선이 `01`이라면, $S1' \cdot S0$ 이라는 조건이 만족되어 1번 관문(D1과 연결된 AND 게이트)만 활성화(1)됩니다. 그럼 D1의 데이터가 AND 게이트를 통과합니다. 마지막으로 이 4개의 관문에서 나오는 길을 커다란 **OR 게이트** 하나로 싹 다 묶어버리면 최종 출력선이 완성됩니다. (통과한 놈 하나만 OR를 거쳐 나감)

```ascii
[MUX의 내부 논리 (부울 식)]

 Y = (D0 · S1'S0') + (D1 · S1'S0) + (D2 · S1S0') + (D3 · S1S0)
      |              |              |              |
    (S가 00일때만) (S가 01일때만) (S가 10일때만) (S가 11일때만)
    D0가 통과      D1이 통과      D2가 통과      D3가 통과
```

📢 **섹션 요약 비유**: MUX 내부는 '지하철 개찰구'와 같습니다. 4개의 개찰구(AND 게이트) 중 승무원(선택선)이 단 하나의 문만 열어줍니다. 승객(데이터)이 그 문을 통과하면, 모두가 결국 하나의 에스컬레이터(OR 게이트)를 타고 위층(출력)으로 올라가게 됩니다.

## Ⅲ. 아키텍처 관점: 데이터 패스(Datapath)의 교차로
컴퓨터 아키텍처에서 MUX는 핏줄을 이어주는 **관절이자 교차로**입니다. 없어서는 안 될 절대적인 부품입니다.

1. **ALU 입력 선택**: CPU의 연산장치(ALU)는 한 번에 두 개의 숫자만 더할 수 있습니다. 그런데 CPU 안에는 32개의 레지스터(저장소)가 있습니다. "3번 레지스터와 7번 레지스터를 더해라"라는 명령이 떨어지면, 32x1 MUX 두 개가 각각 3번 선과 7번 선을 딱 선택해서 ALU의 입에 밥을 넣어줍니다.
2. **시분할 다중화 (TDM)**: 하나의 비싼 인터넷 케이블(해저 광케이블 등)을 수만 명의 사용자가 동시에 쓰고 싶을 때 사용합니다. MUX가 0.001초 단위로 선택선을 1번 사람, 2번 사람, 3번 사람으로 미친 듯이 돌려가며 1가닥의 케이블에 데이터를 순서대로 욱여넣어 보냅니다. (이 선 끝에는 디멀티플렉서가 있어서 다시 사람들에게 나눠줍니다.)

```ascii
[CPU 데이터패스에서의 MUX 활약]

 레지스터 R0 ---+
 레지스터 R1 ---+--> [ MUX ] --(선택된 R1 데이터)--> [ ALU (덧셈기) ]
 레지스터 R2 ---+      ^
                       | (선택 신호: CPU가 "R1 꺼내!" 라고 지시)
```

📢 **섹션 요약 비유**: CPU 안의 MUX는 오케스트라의 '지휘자'입니다. 수십 가지의 악기(레지스터)가 대기하고 있을 때, 지휘자가 지휘봉(선택선)으로 바이올린을 딱 가리키면 오직 바이올린 소리만 스피커(ALU)를 통해 퍼져 나가는 완벽한 통제 시스템입니다.

## 개념 맵 (Concept Map)
```ascii
[멀티플렉서 (Multiplexer, MUX)]
   │
   ├── 정의: 2^N개의 데이터 입력 중 1개를 선택하여 출력 (데이터 라우터)
   │     └── 구성: N개의 선택선(Select Line)으로 제어
   │
   ├── 논리 구조
   │     └── AND-OR 조합 (디코더 로직으로 특정 AND 관문을 열고 OR로 하나로 모음)
   │
   └── 아키텍처 응용
         ├── CPU 데이터패스 (Datapath): 레지스터 버스 선택, ALU 입력 라우팅
         └── 통신 네트워크: 시분할 다중화(TDM)를 통한 단일 선로 전송 효율화
```

## 어린이 비유
멀티플렉서는 **수돗물이 모이는 요술 샤워기**예요! 벽에는 뜨거운 물, 차가운 물, 사과 주스, 포도 주스가 나오는 4개의 파이프가 연결되어 있어요. 하지만 나오는 구멍은 딱 하나죠. 우리가 샤워기 손잡이(선택선)를 포도 주스 쪽으로 딱 돌리면, 다른 파이프는 꽉 잠기고 포도 주스만 콸콸 쏟아져 나오는 거랍니다. 컴퓨터는 이 손잡이를 엄청나게 빨리 돌려서 원하는 데이터를 쏙쏙 빼먹어요!
