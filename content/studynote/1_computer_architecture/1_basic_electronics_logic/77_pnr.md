# 배치 및 배선 (P&R, Place and Route)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 배치 및 배선 (P&R, Place and Route)은 논리 합성 결과물인 넷리스트 (Netlist)를 실제 반도체 기판 위에 물리적인 게이트 위치를 정하고 금속선으로 연결하여 물리적 레이아웃 (Physical Layout)을 완성하는 과정이다.
> 2. **가치**: 설계 추상화 단계의 논리 회로를 실제 제조 가능한 형태(GDSII)로 변환하며, 배선 저항과 캐패시턴스에 의한 실제 지연 시간을 확정하여 칩의 최종 성능과 전력 소모를 결정한다.
> 3. **융합**: 전기적 특성(IR-Drop, EM), 제조 공정 제약(DRC), 논리적 일치성(LVS)을 동시에 만족시켜야 하는 하드웨어 설계의 최종 구현 단계다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 배치 및 배선 (P&R, Place and Route)은 디지털 설계의 마지막 물리적 구현 단계로, 수억 개의 표준 셀 (Standard Cell)을 칩 영역 내에 효율적으로 배치(Placement)하고, 이들 사이의 전기적 연결 통로를 금속 레이어로 연결(Routing)하는 작업이다. 이 과정에서 클럭 신호를 모든 소자에 동시에 전달하기 위한 클럭 트리 합성 (CTS)이 병행된다.

- **💡 비유**: P&R은 "도시 계획 및 인프라 구축"과 같다. 건물의 위치를 잡고(Placement), 전선과 수도관을 매립하며(Power Plan), 도로를 닦아 차들이 다니게 하는(Routing) 과정이다. 모든 집(셀)에 전기가 잘 들어오고 도로가 막히지 않게 설계하는 것이 핵심이다.

- **등장 배경**:
  1. **초미세 공정 진입**: 7nm 이하 공정에서는 배선 지연(Interconnect Delay)이 게이트 지연보다 커져서, 어떻게 배치하느냐가 칩 성능의 80%를 결정하게 되었다.
  2. **복잡도 폭발**: 수작업 레이아웃이 불가능해짐에 따라 EDA (Electronic Design Automation) 툴을 이용한 자동화된 P&R 기술이 필수가 되었다.
  3. **전력 밀도 문제**: 한정된 공간에 너무 많은 셀이 밀집되면 열 발생과 전압 강하 (IR-Drop) 문제가 발생하여 물리적인 분산 배치가 중요해졌다.

- **📢 섹션 요약 비유**: 논리적인 회로도를 바탕으로 실제 전기가 흐르는 미세한 입체 도시를 건설하는 정밀 토목 공사와 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### P&R (Physical Design) 표준 흐름도

| 단계명 | 역할 | 상세 내용 | 비유 |
|:---|:---|:---|:---|
| **Floorplan** | 전체 구조 설계 | I/O 패드 위치, 메모리 매크로 배치, 코어 영역 설정 | 도시 구획 정리 |
| **Power Plan** | 전원망 구축 | VDD/VSS 금속 그물망 (Mesh) 형성 | 전기 간선 매설 |
| **Placement** | 표준 셀 배치 | 수백만 개의 게이트를 최적 위치에 자동 배치 | 건물 세우기 |
| **CTS (Clock Tree Synthesis)** | 클럭 신호망 구축 | 모든 플립플롭에 클럭이 동시 도달하도록 트리 구조 형성 | 수돗물 수압 맞추기 |
| **Routing** | 신호선 연결 | 셀 간 신호선을 금속 레이어로 자동 연결 | 도로 및 통신망 연결 |

---

### 물리 설계 아키텍처 (Physical Layout)

물리 설계는 평면적인 배치를 넘어 여러 층의 금속 레이어를 사용하는 입체적인 구조를 가진다.

```text
 ┌──────────────────────────────────────────────────────────────────┐
 │               P&R 물리적 적층 및 배선 구조                         │
 ├──────────────────────────────────────────────────────────────────┤
 │                                                                  │
 │    [금속 레이어 구조]                    [데이터 흐름]             │
 │  ┌──────────────────┐  Metal 5 (V)   ┌──────────────────────┐    │
 │  │      Power       │  Metal 4 (H)   │  Floorplan (구획)     │    │
 │  ├──────────────────┤  Metal 3 (V)   └──────────┬───────────┘    │
 │  │      Signal      │  Metal 2 (H)              ▼                │
 │  ├──────────────────┤  Metal 1 (V)   ┌──────────────────────┐    │
 │  │   Standard Cell  │  (Silicon)     │  Placement (배치)     │    │
 │  └──────────────────┘                └──────────┬───────────┘    │
 │           ▲                                     ▼                │
 │           │ Via (수직 연결)           ┌──────────────────────┐    │
 │           │                          │      CTS (클럭)       │    │
 │           ▼                          └──────────┬───────────┘    │
 │  ┌──────────────────┐                           ▼                │
 │  │     Substrate    │                ┌──────────────────────┐    │
 │  └──────────────────┘                │    Routing (배선)     │    │
 │                                      └──────────────────────┘    │
 │   * H/V: Horizontal/Vertical (각 층별 배선 방향 교차)             │
 │   * Via: 레이어 간 신호를 수직으로 이어주는 금속 구멍               │
 └──────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** P&R 툴은 실리콘 기판 위에 표준 셀을 배치한 후, 그 위로 10층 이상의 금속 레이어를 쌓으며 배선을 수행한다. 이때 인접한 층끼리는 서로 수직 방향(가로/세로)으로만 선을 긋도록 제한하여 신호 간섭(Crosstalk)을 줄이고 배선 효율을 높인다. CTS (Clock Tree Synthesis) 단계는 클럭 신호가 칩 전체에 퍼지면서 발생하는 스큐 (Clock Skew)를 최소화하기 위해 'H-Tree'나 'Fish-bone' 구조로 정밀하게 설계된다.

---

### 심층 동작 원리: DRC와 LVS 검증

설계가 끝난 후 공장(Foundry)으로 보내기 전, 반드시 거쳐야 하는 물리적 검증 과정이다.

1. **DRC (Design Rule Check)**: 금속 선 사이의 최소 간격, 최소 폭 등 파운드리에서 정한 제조 규칙을 지켰는지 확인한다. (예: "선이 너무 붙으면 합선됨!")
2. **LVS (Layout vs Schematic)**: 완성된 레이아웃에서 역으로 회로를 추출하여, 원래 설계하려던 넷리스트와 논리적으로 100% 일치하는지 비교한다.
3. **ERC (Electrical Rule Check)**: 전원선이 끊기지 않았는지, 과도한 전류가 흐르는 곳은 없는지 전기적 특성을 검증한다.

- **📢 섹션 요약 비유**: 건물을 다 지은 후 소방법(DRC)을 준수했는지, 설계 도면대로 방이 다 만들어졌는지(LVS) 최종 준공 검사를 받는 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### Floorplan vs Placement

| 비교 항목 | Floorplan (플로어플랜) | Placement (배치) |
|:---|:---|:---|
| **대상** | 대형 블록 (SRAM, IP, I/O) | 수백만 개의 작은 표준 셀 (Gates) |
| **결정 방식** | 설계자가 직접 또는 가이드 제공 | EDA 툴의 알고리즘에 의한 자동 배치 |
| **영향** | 칩의 전체 크기와 가로세로비 결정 | 국부적인 혼잡도(Congestion)와 타이밍 결정 |
| **중요성** | 초기 전력망 및 외부 통신 구조 확립 | 최종 타이밍 클로저의 핵심 |
| **비유** | 아파트 단지의 동 배치 | 각 집 안의 가구 배치 |

### 과목 융합 관점

- **운영체제 (OS)**: 메모리 컨트롤러의 물리적 위치가 CPU 코어와 멀어지면 메모리 접근 레이턴시가 증가하며, 이는 OS의 스케줄링 효율에 영향을 준다.
- **반도체 공정**: 배선이 너무 빽빽하면 수율 (Yield)이 떨어지므로, P&R 툴은 배선 혼잡도 (Congestion)를 분석하여 공간을 넓히는 최적화를 수행한다.

- **📢 섹션 요약 비유**: 큰 구역을 나누는 도시 계획(Floorplan)과 세부 필지에 집을 짓는 작업(Placement)이 조화를 이루어야 살기 좋은 도시(고성능 칩)가 됩니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **Congestion (배선 혼잡) 발생 시**: 특정 영역에 셀이 너무 밀집되어 배선할 공간이 부족하면, P&R 툴에 'Placement Density'를 낮추도록 제약을 주거나, 해당 영역에 'Keep-out Zone'을 설정하여 셀들이 넓게 퍼지도록 유도한다.
2. **Clock Skew (클럭 스큐) 문제 시**: 칩의 왼쪽 끝과 오른쪽 끝의 클럭 도착 시간 차이가 크면 홀드 타임 위반이 발생한다. 이때는 CTS 과정에서 클럭 버퍼를 추가하거나 배선 레이어를 높여(저항이 낮은 상위 메탈 사용) 지연 시간을 맞춘다.

### 도입 체크리스트
- 모든 매크로 (SRAM 등) 주변에 충분한 할로 (Halo) 영역이 설정되었는가?
- 전원망 (Power Mesh)의 굵기가 IR-Drop 마진을 충분히 확보할 만큼 두꺼운가?
- 안테나 효과 (Antenna Effect) 방지를 위한 다이오드 삽입이 완료되었는가?

### 안티패턴
- **Dirty Floorplan**: 초기 블록 배치를 엉성하게 하면 나중에 신호선들이 엉키고 꼬여서 (Detour), 타이밍과 면적 모두에서 손해를 보고 설계가 영원히 끝나지 않는 루프에 빠진다.
- **Ignoring Dummy Fill**: 미세 공정에서 금속 패턴 밀도가 일정하지 않으면 제조 공정(CMP) 중 칩이 휘거나 깎일 수 있다. 더미 패턴 삽입을 간과하면 수율이 폭락한다.

- **📢 섹션 요약 비유**: 기초 공사(Floorplan)를 부실하게 하면 아무리 좋은 자재(배선)를 써도 건물이 기울고 물이 새는 부실 공사가 되는 것과 같습니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 도입 전 (Manual Layout) | 도입 후 (Auto P&R) | 개선 효과 |
|:---|:---|:---|:---|
| **정량** | 수백만 게이트 배치에 수년 소요 | 수일 내 최종 레이아웃 완성 | 설계 속도 수백 배 향상 |
| **정성** | 복잡한 디자인 규칙 위반 잦음 | 알고리즘 기반으로 DRC 100% 준수 | 제조 수율 및 신뢰성 극대화 |

### 미래 전망
- **Cloud-based P&R**: 수천 대의 서버를 활용하여 하룻밤 사이에 수천 개의 배치 시나리오를 돌려보고 가장 좋은 결과를 고르는 클라우드 설계 방식이 도입되고 있다.
- **3D-IC 및 TSV**: 칩을 옆으로 늘리는 대신 위로 쌓고 구멍을 뚫어 연결하는 3D 배치 기술이 차세대 고성능 칩의 핵심이 될 것이다.

- **📢 섹션 요약 비유**: 수기로 그리던 설계도에서 AI 기반의 3D 도시 시뮬레이션으로 진화한 것처럼, P&R은 상상을 현실로 바꾸는 반도체 설계의 마지막 마법입니다.
