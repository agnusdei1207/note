> **핵심 인사이트**
> 1. **TPM (Trusted Platform Module)**은 컴퓨터 메인보드에 부착되어 암호화 키 생성, 저장, 플랫폼의 무결성 검증을 수행하는 국제 표준 하드웨어 보안 칩이다.
> 2. 주로 **보안 부팅(Secure Boot)** 과정에서 시스템이 악성코드에 감염되지 않았는지(무결성)를 하드웨어적으로 증명(Attestation)하는 데 사용된다.
> 3. Windows 11의 필수 설치 요구 사항으로 지정되면서 대중적으로 알려졌으며, BitLocker 같은 디스크 암호화 기술의 핵심 하드웨어 기반이 된다.

<br>

## Ⅰ. TPM의 개념과 등장 배경

과거에는 컴퓨터의 비밀번호나 암호화 키를 하드디스크나 소프트웨어 레지스트리에 저장했습니다. 하지만 하드디스크를 떼어내어 다른 컴퓨터에 연결하거나, 악성코드(루트킷)가 운영체제 권한을 탈취하면 속수무책으로 뚫렸습니다.

**TPM(Trusted Platform Module)**은 이러한 문제를 해결하기 위해 고안된 '미니 HSM'입니다. PC 메인보드에 땜질되어(또는 CPU 내부에 통합되어) 제공되는 독립적인 보안 전용 마이크로컨트롤러로, 소프트웨어가 접근할 수 없는 안전한 공간을 제공합니다.

> 📢 **섹션 요약 비유**: 예전에는 집 열쇠를 집 앞 화분 밑(소프트웨어)에 숨겨두었다면, TPM은 벽돌 속에 특수 금고(하드웨어 칩)를 매립하고 거기에 열쇠를 보관하는 방식입니다.

<br>

## Ⅱ. TPM의 핵심 기능

### 1. PCR (Platform Configuration Register)과 무결성 검증
TPM 내부에 있는 특수한 레지스터들(PCR)은 부팅 시 각 단계(BIOS, 부트로더, OS 커널 등)의 해시(Hash) 값을 누적하여 저장합니다.
* 부팅 과정에서 코드가 1비트라도 변조되면(해커가 악성코드를 심어두면) PCR 값이 완전히 달라집니다.
* TPM은 부팅이 끝난 후 이 PCR 값을 자신이 기억하고 있는 '정상 상태의 해시값'과 비교하여, 일치할 때만 디스크 암호화 해제 키 등을 운영체제에 넘겨줍니다.

### 2. 바인딩 (Binding)과 실링 (Sealing)
* **Binding (바인딩)**: 데이터를 암호화할 때 "이 컴퓨터(이 TPM 칩)에서만 복호화할 수 있게 해줘"라고 묶어두는 기능입니다. 하드디스크를 훔쳐서 다른 PC에 꽂아도 절대 열리지 않습니다.
* **Sealing (실링)**: 바인딩보다 한 단계 더 나아가, "이 컴퓨터이고, **동시에 부팅 과정(PCR 값)에 아무런 변조가 없었을 때만** 복호화해 줘"라는 조건을 거는 기능입니다. (BitLocker의 핵심 원리)

### TPM 동작 다이어그램 (BitLocker 예시)

```text
 ┌─────────────── PC ───────────────┐
 │                                  │
 │   [해커가 HDD를 탈취하려 함]     │
 │          │                       │
 │          ▼                       │
 │   ┌─────────────┐                │
 │   │ 암호화된 OS │  ◀─ (복호화 키 필요)
 │   └─────────────┘                │
 │                                  │
 │   ┌─────────────┐                │
 │   │   TPM 칩    │ ─▶ PCR 검증: 정상 부팅인가? (O) ─▶ 복호화 키 제공
 │   └─────────────┘ ─▶ PCR 검증: 변조되었는가? (X) ─▶ 키 제공 거부!
 └──────────────────────────────────┘
```

> 📢 **섹션 요약 비유**: TPM은 매우 깐깐한 경비원입니다. "네가 원래 살던 집(Binding)이 맞고, 문부터 거실까지 들어오는 동안 아무 물건도 건드리지 않았음(Sealing, 무결성)이 확인되어야만 안방 열쇠를 주겠다"고 합니다.

<br>

## Ⅲ. TPM의 종류 (구현 방식)

TPM은 반드시 별도의 물리적 칩이어야만 하는 것은 아니며, 여러 가지 형태로 구현됩니다.

1. **Discrete TPM (dTPM)**: 메인보드에 부착된 완전히 독립된 물리적 반도체 칩입니다. 가장 강력한 보안을 제공하며 기업용 PC에 주로 쓰입니다.
2. **Firmware TPM (fTPM / PTT)**: 물리적 칩 없이, 메인 CPU 내부의 보안 실행 환경(TrustZone 등)을 활용하여 소프트웨어적으로 TPM 기능을 에뮬레이션하는 방식입니다. 일반 소비자용 메인보드(Intel PTT, AMD fTPM)에서 가장 흔히 사용됩니다.
3. **Integrated TPM (iTPM)**: 별도 칩은 아니지만 칩셋이나 CPU 다이(Die) 안에 TPM 하드웨어 로직이 물리적으로 내장된 형태입니다.

> 📢 **섹션 요약 비유**: 경비실 건물을 아예 따로 짓는 것(dTPM), 본관 건물 안에 경비원 전용 방을 만들어주는 것(iTPM), 본관 경비 소프트웨어를 돌리는 것(fTPM)의 차이입니다.
