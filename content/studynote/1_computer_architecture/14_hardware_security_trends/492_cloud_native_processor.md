> **핵심 인사이트**
> 1. **클라우드 네이티브 프로세서**는 데이터센터와 클라우드 서비스(AWS, GCP, Azure 등)의 특성인 '수많은 가상 머신(VM)과 컨테이너의 동시 실행'에 완벽하게 최적화된 새로운 세대의 CPU 아키텍처다.
> 2. 전통적인 x86(Intel, AMD) 서버 CPU는 코어 하나당 성능이 너무 높고 무거워 전기를 많이 먹지만, 클라우드 네이티브 프로세서(주로 ARM 기반)는 **"단일 코어의 절대 성능을 약간 낮추는 대신, 코어 개수를 엄청나게 늘려(다코어) 전력 효율을 극대화"**한다.
> 3. 대표적으로 ARM의 **Neoverse** 아키텍처가 있으며, 이를 기반으로 AWS의 Graviton, 구글의 Axion, MS의 Cobalt 등 클라우드 기업들이 자체 칩을 설계하여 시장을 재편하고 있다.

<br>

## Ⅰ. 전통적 x86 서버의 한계와 클라우드의 요구

기존 데이터센터를 지배하던 인텔 제온(Xeon) 같은 x86 CPU는 원래 소수의 무거운 데이터베이스나 거대한 단일 프로그램(Monolithic)을 가장 빠르게 처리하도록 설계되었습니다. (SMT/하이퍼스레딩 등 복잡한 구조 포함)

하지만 클라우드 시대가 도래하면서 소프트웨어 아키텍처가 변했습니다.
* **마이크로서비스(Microservices)**: 거대한 앱을 아주 작은 기능(컨테이너) 수백 개로 쪼개어 실행합니다.
* **수직 확장(Scale-up) $\rightarrow$ 수평 확장(Scale-out)**: 고성능 CPU 하나보다, 적당한 성능의 CPU 수십 개에 트래픽을 분산시키는 것이 훨씬 유리해졌습니다.

이런 상황에서 x86 코어는 너무 크고 전기를 많이 먹었습니다. 한 서버에 우겨넣을 수 있는 코어 수에 한계가 왔고, 발열을 감당하기 힘들어졌습니다.

> 📢 **섹션 요약 비유**: 거대한 화물(전통적 프로그램)을 옮길 때는 힘센 코끼리(x86 코어) 몇 마리가 필요했지만, 지금의 클라우드는 수백만 개의 쌀알(컨테이너/마이크로서비스)을 동시에 옮겨야 하는 상황입니다. 코끼리보다는 날렵하고 밥을 덜 먹는 개미(ARM 기반 코어) 10만 마리가 훨씬 효율적입니다.

<br>

## Ⅱ. 클라우드 네이티브 프로세서의 특징 (ARM Neoverse 중심)

ARM은 스마트폰 칩셋의 강자였지만, 저전력 설계 기술을 서버용으로 스케일업(Scale-up)하여 **Neoverse**라는 클라우드 전용 아키텍처를 내놓았습니다.

1. **진정한 물리 코어 극대화 (No SMT)**
   * 클라우드는 가상 머신(VM)을 고객에게 쪼개서 팝니다. x86의 논리 코어(하이퍼스레딩)를 쪼개 팔면, 한 고객의 작업이 다른 고객의 성능(캐시)을 깎아먹는 **Noisy Neighbor(시끄러운 이웃)** 문제가 발생합니다.
   * ARM 기반 칩은 코어 1개당 1개의 스레드만 처리하도록 매우 작고 단단하게 만들어, 칩 하나에 64개~128개의 순수 물리 코어를 박아 넣습니다. 서로 간섭이 전혀 없습니다.
2. **압도적인 전력 대비 성능 (Performance per Watt)**
   * 불필요한 레거시 명령어(과거 x86의 잔재)가 없는 RISC 구조를 채택하여, 동일한 성능을 내면서 전기 요금과 냉각 비용을 30~40% 절감합니다.
3. **거대한 캐시 메모리 병렬 구조**
   * 코어가 100개가 넘어가면 데이터를 주고받는 버스(Mesh Network)와 캐시의 설계가 핵심입니다. 클라우드 칩은 L2/L3 캐시를 기가바이트 수준으로 늘리고 대역폭을 극대화합니다.

 │ ┌────┐   ┌────┐ │               │ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐│
 │ │ 거 │   │ 대 │ │ (스레드 공유) │ │작│ │고│ │가│ │벼│ │운│ │코│ │어││

 │ │ 코 │   │ 코 │ │       충돌!   │ (스레드 완전 독립, 캐시 분리 ─▶ 충돌 없음)

## Ⅲ. 클라우드 3대장의 자체 칩 설계 시대

클라우드 네이티브 프로세서의 등장으로 반도체 시장의 생태계가 바뀌었습니다. 
클라우드를 빌려주는 IT 기업들이 인텔/AMD에게서 CPU를 사오는 대신, ARM의 설계도(Neoverse)를 사서 **자신들의 클라우드 데이터센터에 딱 맞는 칩을 직접 만들기 시작**한 것입니다.

* **AWS (아마존)**: Graviton 시리즈 (현재 가장 성공적이며 널리 쓰임)
* **Google Cloud**: Axion 프로세서
* **Microsoft Azure**: Cobalt 100 프로세서

> 📢 **섹션 요약 비유**: 예전에는 택배 회사(클라우드)가 자동차 회사(인텔)에서 기성품 트럭을 사 와서 썼다면, 이제는 부품 설계도(ARM)를 사서 택배 상자 크기에 소름 돋게 딱 맞는 완벽한 택배 전용차(자체 칩)를 스스로 만들어 쓰고 있는 셈입니다.
