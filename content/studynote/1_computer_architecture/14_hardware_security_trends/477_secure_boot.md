> **핵심 인사이트**
> 1. **보안 부팅 (Secure Boot)**은 시스템이 부팅되는 과정에서 실행되는 모든 코드(펌웨어, 부트로더, OS 커널)가 제조사가 디지털 서명한 신뢰할 수 있는 코드인지 검증하는 UEFI(Unified Extensible Firmware Interface)의 표준 기능이다.
> 2. OS가 켜지기 전에 하단(Boot Sector)에서부터 시스템을 장악해버리는 **루트킷(Rootkit)이나 부트킷(Bootkit) 악성코드를 원천적으로 차단**하는 것이 주 목적이다.
> 3. 신뢰의 사슬(Chain of Trust)이라는 개념을 사용하여, 각 단계의 코드가 다음 단계의 코드를 실행하기 전에 서명을 확인하는 릴레이 검증 방식으로 작동한다.

<br>

## Ⅰ. 보안 부팅의 도입 배경: 부트킷의 위협

전통적인 BIOS 기반의 시스템은 전원을 켜면 하드디스크의 첫 번째 섹터(MBR)에 있는 코드를 **묻지도 따지지도 않고** 무조건 실행했습니다. 

해커들은 이를 악용하여, 백신 프로그램이나 운영체제가 켜지기도 전에 먼저 실행되어 시스템 최고 권한을 장악하고 자신을 숨기는 **부트킷(Bootkit)**이라는 악성코드를 만들었습니다. OS 입장에서는 자신이 부팅되기 전에 이미 시스템이 오염되었기 때문에 이를 탐지하거나 치료할 방법이 없었습니다.

이를 막기 위해 "검증되지 않은 코드는 아예 부팅 단계에서 실행조차 하지 못하게 하자"는 아이디어에서 **보안 부팅(Secure Boot)**이 탄생했습니다.

> 📢 **섹션 요약 비유**: 옛날 성문은 문지기 없이 누구나 먼저 들어오는 사람이 성주 행세를 할 수 있었습니다(부트킷). 보안 부팅은 성문에 철저한 문지기를 세워, 왕(제조사)의 인장이 찍힌 마패(디지털 서명)가 없으면 아예 성 안으로 들이지 않는 시스템입니다.

<br>

## Ⅱ. 작동 원리: 신뢰의 사슬 (Chain of Trust)

보안 부팅은 **신뢰의 사슬(Chain of Trust)** 구조로 작동합니다. 하드웨어에 영구적으로 기록된 절대 변하지 않는 가장 기초적인 신뢰점(Root of Trust)에서 시작하여, 릴레이 경주처럼 다음 주자의 신원을 확인한 후 바통을 넘깁니다.

### 부팅 과정 다이어그램

```text
 ┌─────────────────┐  (하드웨어 내장, 변조 불가)
 │ Root of Trust   │  "나는 나를 믿고, 다음 펌웨어를 검증한다"
 └───────┬─────────┘
         │ (서명 확인: OK)
 ┌───────▼─────────┐
 │ UEFI 펌웨어     │  "펌웨어 제조사(예: ASUS) 서명 확인.
 │ (마더보드)      │   다음으로 부트로더를 검증한다"
 └───────┬─────────┘
         │ (서명 확인: OK - 마이크로소프트 서명 확인됨)
 ┌───────▼─────────┐
 │ Windows         │  "Windows 부트로더 확인됨.
 │ 부트로더        │   이제 OS 커널을 검증하고 실행한다"
 └───────┬─────────┘
         │ (서명 확인: 악성코드 감염으로 서명 깨짐!)
 ┌───────▼─────────┐
 │ OS 커널 / 드라이버│ ──▶ [ 부팅 중지! (Red Screen) ]
 └─────────────────┘     "신뢰할 수 없는 코드가 감지되었습니다."
```

1. 마더보드에는 신뢰할 수 있는 인증서(공개키)가 공장 출고 시 저장되어 있습니다.
2. 부팅할 펌웨어나 부트로더의 디지털 서명을 이 공개키로 풀어보고, 일치하면(변조되지 않았으면) 코드를 실행합니다.
3. 서명이 없거나, 해커가 코드를 한 글자라도 수정하여 서명이 깨졌다면 실행을 즉시 차단하고 시스템을 멈춥니다.

> 📢 **섹션 요약 비유**: 릴레이 달리기와 같습니다. 1번 주자가 2번 주자의 얼굴(서명)을 확인하고 바통을 넘기고, 2번 주자는 다시 3번 주자의 얼굴을 확인합니다. 중간에 모르는 사람이 끼어들면 바통을 주지 않고 경기를 중단합니다.

<br>

## Ⅲ. 보안 부팅의 한계와 논란

보안 부팅은 윈도우(Windows) 환경의 보안을 극적으로 끌어올렸지만, 초기에는 많은 논란이 있었습니다.

* **리눅스(Linux) 생태계와의 충돌**: 초기 보안 부팅은 마더보드에 마이크로소프트(MS)의 인증서만 기본 탑재되어 있었습니다. 따라서 개인이 직접 빌드한 리눅스 커널이나 기타 오픈소스 OS를 설치하려 하면 서명이 없어서 부팅이 거부되었습니다.
* **해결책**: 사용자가 BIOS 설정에 들어가 보안 부팅을 수동으로 끄거나(Disable), 심(Shim)이라 불리는 MS 서명이 포함된 1차 부트로더를 리눅스 진영에서 사용하여 이 문제를 우회/해결하고 있습니다.

> 📢 **섹션 요약 비유**: 철저한 문지기(보안 부팅) 때문에 강도는 막아냈지만, 내가 초대한 친구(리눅스)마저 "마패가 없다"며 못 들어가게 막는 상황이 벌어졌던 것입니다. 지금은 친구에게도 임시 출입증(Shim)을 발급해주는 방식으로 해결하고 있습니다.
