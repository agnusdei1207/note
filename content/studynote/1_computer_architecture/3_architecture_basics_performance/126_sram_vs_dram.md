> **핵심 인사이트**
> 1. SRAM(정적 램)과 DRAM(동적 램)은 컴퓨터의 '살아있는 단기 기억'을 담당하는 휘발성 메모리(RAM)를 양분하는 **두 가지 극단적인 반도체 물리 아키텍처**이다.
> 2. **SRAM(Static RAM)**은 전기가 통하는 한 무한히 반사되는 플립플롭 거울 방을 이용해 데이터를 영구 유지하므로 **빛의 속도(초고속)**를 내지만, 트랜지스터가 6개나 필요해 **면적이 거대하고 가격이 미친 듯이 비싸다. (캐시 메모리에 사용)**
> 3. **DRAM(Dynamic RAM)**은 트랜지스터 1개와 작은 물통(커패시터) 1개로 1비트를 저장하여 **면적이 극도로 작아 용량을 테라바이트급으로 늘릴 수 있지만**, 물통의 물이 계속 새어나가 주기적으로 물을 다시 채워줘야 하는 **리프레시(Refresh) 오버헤드 때문에 속도가 느리다. (메인 메모리에 사용)**

---

## Ⅰ. 부자들의 포르쉐: SRAM (정적 램)
**SRAM(Static Random Access Memory)**에서 '정적(Static)'이라는 말은, 전원(전기)만 계속 공급해 주면 안에 들어있는 데이터(1과 0)가 가만히 냅둬도 영원히 지워지지 않고 그 상태를 유지한다는 뜻입니다.

SRAM은 앞서 배운 **래치(Latch)나 플립플롭 회로**를 사용해 만듭니다. 트랜지스터 6개를 교차로 엮어 전기가 무한 루프를 돌게 만듭니다. 물통(커패시터)에 전자를 가두는 게 아니라 전기 스위치 그 자체로 동작하기 때문에, CPU가 "데이터 내놔!" 하면 방문을 여는 즉시(0.001초) 데이터가 빛처럼 튀어나옵니다. 
하지만 1비트(0 하나)를 저장하는 데 방(면적)이 너무 커서, 아무리 미세 공정으로 깎아도 칩 1개에 몇 MB밖에 구겨 넣지 못해 가격이 수백만 원에 달합니다. 

```ascii
[SRAM의 6-TR (트랜지스터 6개) 구조]

   Vdd (전원)
    |
  [TR 1] --+-- [TR 2]  (위아래 크로스 커플링으로 무한 루프 형성)
           |    
  [TR 3] --+-- [TR 4]
    |           |
  [TR 5]      [TR 6]  (문을 열고 닫는 스위치)
  (Wordline)

 -> 복잡하고 방이 크다! 하지만 전기가 통하는 한 절대 지워지지 않고 광속!
```

📢 **섹션 요약 비유**: SRAM은 모터가 계속 돌아가는 '공중부양 자석 팽이'입니다. 전원만 꽂혀 있으면 아무도 건드리지 않아도 팽이가 공중에서 완벽하게 안정적으로 계속 돕니다. 잡고 싶을 때 언제든 0.1초 만에 낚아챌 수(읽기) 있는 완벽한 최고급 장난감입니다.

## Ⅱ. 가성비의 화물차: DRAM (동적 램)
**DRAM(Dynamic Random Access Memory)**에서 '동적(Dynamic)'이라는 말은, 가만히 냅두면 데이터가 사라져버려서 계속해서 엉덩이를 걷어차 주며(동적으로 관리) 살려내야 한다는 슬픈 뜻입니다.

DRAM의 구조는 눈물겹게 단순합니다. 1비트를 저장하는 데 **트랜지스터 1개와 커패시터(물통) 1개**면 끝납니다(1T1C 구조). 물통에 전자가 꽉 차 있으면 '1', 비어있으면 '0'입니다. SRAM보다 방 크기가 6분의 1로 줄어들어, 똑같은 손톱만 한 칩에 수십 기가바이트(GB)의 엄청난 용량을 때려 박을 수 있는 기적의 가성비를 자랑합니다. 

하지만 커패시터(물통)는 밑 빠진 독이라서 시간이 지나면 전자가 줄줄 새어나갑니다(방전). 
그래서 1을 저장했더라도 몇 밀리초(ms)가 지나면 물이 다 빠져나가 0이 되어버리는 대참사가 일어납니다. 이를 막기 위해 메모리 컨트롤러가 0.064초마다 램 전체를 돌아다니며 물이 빠진 통에 다시 물을 콸콸 채워 넣는 막노동을 해야 합니다. 이것이 바로 **리프레시(Refresh)**입니다. 리프레시를 하는 동안에는 CPU가 데이터를 읽을 수 없으므로(병목), SRAM보다 속도가 100배 느려지는 치명적 약점을 가집니다.

```ascii
[DRAM의 1T1C (트랜지스터 1개, 커패시터 1개) 구조]

 Wordline ----[ TR 1 (수도꼭지) ]---- Bitline
                  |
                [ C 1 (물통: 커패시터) ] 
                  |
                 GND

 -> 1T1C의 극강 압축! 싸고 용량 큼!
 -> 하지만 물(전자)이 줄줄 샘... 계속 주기적으로 다시 물을 부어줘야 함 (리프레시 오버헤드 ㅠㅠ)
```

📢 **섹션 요약 비유**: DRAM은 밑이 깨진 '종이컵'입니다. 종이컵은 엄청나게 싸서 천만 개를 쌓아둘 수 있지만, 물(데이터 1)을 담아두면 밑으로 뚝뚝 샙니다. 그래서 주인이 컵이 완전히 마르기 전에 미친 듯이 돌아다니며 계속 물을 다시 채워줘야(리프레시) 하는 고단한 삶을 삽니다. 물을 붓고 있는 순간에는 손님이 그 컵의 물을 마실 수 없어서 속도가 느려집니다.

## Ⅲ. 아키텍처 관점: 철저한 계급 사회 (캐시 vs 메인 메모리)
SRAM과 DRAM은 어느 한쪽이 이긴 것이 아닙니다. 서로의 단점이 상대의 장점이기에, 현대 컴퓨터 아키텍처(Memory Hierarchy)에서 **완벽하게 철저한 계급과 역할 분담**을 이루고 있습니다.

1. **SRAM = 캐시 메모리 (L1, L2, L3)**: 
   * CPU의 연산 속도(수 GHz)를 따라잡을 수 있는 유일한 반도체입니다. 너무 비싸고 열이 나서 많이는 못 달기 때문에, CPU 코어 바로 옆에 찰싹 붙어서 겨우 몇 MB(메가바이트) 크기의 초고속 귀족 대우를 받으며 **캐시 메모리**로 군림합니다.
2. **DRAM = 메인 메모리 (RAM)**: 
   * CPU 칩 바깥에 있는 거대한 녹색 막대기(시금치 램)들입니다. 속도는 느리지만 무식하게 용량이 커서(16GB, 32GB), 윈도우 운영체제와 거대한 3D 게임 데이터를 몽땅 쓸어 담아두는 광활한 평민들의 **주기억장치** 창고 역할을 합니다.

```ascii
[현대 시스템의 철저한 SRAM-DRAM 분업]

 [ CPU 다이(Die) 내부 ]                        [ 메인보드 외부 ]
 
 +-- 연산(ALU) ----------+                     +------------------+
 | + SRAM (L1, L2 캐시)  | <--(느린 버스)----> |  거대한 DRAM     |
 | (용량: 10MB, 속도 1ns) |                     | (용량: 16GB,     |
 +-----------------------+                     |  속도 100ns)     |
    ^                                          +------------------+
   (SRAM은 비싸서 칩 안에 조금만 박음)             (싸서 보드에 잔뜩 꽂음)
```

📢 **섹션 요약 비유**: SRAM은 '천재 비서 1명'이고, DRAM은 '창고 아르바이트생 1만 명'입니다. 천재 비서는 몸값이 10억이라 한 명밖에 고용 못 하지만, 사장님(CPU)이 옆에서 "서류 가져와!" 하면 0.1초 만에 딱 대령합니다. 알바생 1만 명은 최저시급(가성비)이라 엄청난 양의 짐을 들 수 있지만, 물건 하나 찾아오라고 하면 창고를 다 뒤지느라 1분이 걸립니다. 이 둘을 같이 써야만 대기업(컴퓨터)이 돌아갑니다.

## 개념 맵 (Concept Map)
```ascii
[SRAM과 DRAM (휘발성 메모리 아키텍처)]
   │
   ├── SRAM (Static RAM)
   │     ├── 물리 구조: 6개의 트랜지스터(플립플롭)로 1비트 셀 구성 -> 집적도 최하, 면적 큼, 단가 극상
   │     ├── 동작 특성: 전원 유지 시 데이터 영구 보존 (리프레시 불필요) -> 속도 초고속(1~5ns)
   │     └── 시스템 응용: CPU 내부 L1/L2/L3 캐시 메모리
   │
   └── DRAM (Dynamic RAM)
         ├── 물리 구조: 1트랜지스터 + 1커패시터(1T1C) 셀 구성 -> 집적도 극상, 면적 극소, 단가 최하
         ├── 동작 특성: 커패시터 자연 방전 발생 -> 파멸적 성능 저하를 감수하고 주기적 Refresh 강제 (속도 느림 50~100ns)
         └── 시스템 응용: 컴퓨터 메인보드에 장착되는 거대 메인 메모리 (주기억장치)
```

## 어린이 비유
SRAM과 DRAM은 물을 보관하는 완전히 다른 **두 가지 컵**이에요! 
**SRAM 컵**은 100만 원짜리 최고급 티타늄 컵이라 절대 물이 새지 않고 입에 대면 1초 만에 꿀꺽 마실 수 있지만, 너무 비싸서 집에 1개밖에 못 사둬요(캐시).
반면 **DRAM 컵**은 10원짜리 종이컵이에요. 너무 싸서 1억 개를 방에 채울 수 있죠! 하지만 종이가 젖어서 물이 바닥으로 뚝뚝 새기 때문에, 컵이 마르기 전에 엄마가 돌아다니면서 물을 계속 다시 부어줘야(리프레시) 하는 아주 피곤한 컵이랍니다(메인 메모리).
