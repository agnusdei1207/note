> **핵심 인사이트**
> 1. 데이터패스(Datapath)는 CPU 내부에서 명령어(Instruction)를 실행하기 위해 **실제 데이터(숫자)가 물리적으로 흘러가고 가공되는 하드웨어 부품들의 집합과 그 경로**를 의미한다.
> 2. 산술논리연산장치(ALU), 범용 레지스터 파일(Register File), 멀티플렉서(MUX), 데이터 메모리(L1 Cache) 등이 촘촘한 버스(Bus) 전선망으로 거미줄처럼 연결된 거대한 팩토리 라인이다.
> 3. 제어 유닛(Control Unit)이 내리는 신호(0과 1)에 따라 밸브(MUX)가 열리고 닫히면서, 이 데이터패스 위를 달리는 데이터들의 행선지가 결정된다.

---

## Ⅰ. CPU의 '근육'과 '컨베이어 벨트'
CPU 아키텍처 설계도를 펴보면 수많은 네모 박스와 화살표들이 복잡하게 그려져 있습니다. 이 중에서 **"숫자 5나 10 같은 진짜 데이터들이 어디서 나와서 어디로 굴러가는가?"**를 보여주는 길이 바로 데이터패스(데이터 통로)입니다.

명령어 `ADD R1, R2, R3` (R2 + R3 한 값을 R1에 넣어라)가 실행될 때 데이터패스는 아래와 같이 움직입니다.
1. **레지스터 파일**의 R2와 R3 방 문이 열리고, 그 안에 있던 두 숫자가 전선(버스)을 타고 흘러나옵니다.
2. 두 숫자가 **ALU(산술논리연산장치)**의 왼쪽 구멍과 오른쪽 구멍으로 쏙 들어갑니다.
3. ALU 안에서 덧셈이 이루어지고, 새로운 결과 숫자가 아래쪽 구멍으로 튀어나옵니다.
4. 이 결과 숫자가 다시 뒤로 빙 돌아가는 전선을 타고 **레지스터 파일**의 R1 방으로 들어가 찰칵! 하고 저장됩니다.
이 전체 물리적 부품과 배선의 묶음이 데이터패스입니다.

```ascii
[기본 RISC 데이터패스 구조 (ADD 명령어 흐름)]

 [ 레지스터 파일 ] --------(R2 데이터 5)-------> [ ALU ]
  (32개 방 존재)   --------(R3 데이터 7)-------> (덧셈)
       ^                                            |
       |                                            V
       +-----------(결과 데이터 12 되돌아옴)----------+ 
```

📢 **섹션 요약 비유**: 데이터패스는 자동차 공장의 '조립 라인(컨베이어 벨트와 기계팔)'입니다. 철판(데이터)이 창고(레지스터)에서 나와 프레스 기계(ALU)에 들어가서 납작하게 깎인 다음, 다시 컨베이어 벨트를 타고 완성품 창고(레지스터)로 들어가는 모든 물리적인 장비와 길을 뜻합니다.

## Ⅱ. 길을 통제하는 교통순경: 멀티플렉서(MUX)
명령어는 덧셈(ADD)만 있는 게 아닙니다. 메모리에서 데이터를 가져오는 명령어(`LOAD R1, 100번지`)도 있습니다.
이때 데이터는 ALU를 거쳐 나오는 게 아니라, 메모리 칩에서 튀어나와야 합니다. 
결국 레지스터 R1으로 들어가는 전선 골목길에는 **ALU에서 오는 차(데이터)**와 **메모리에서 오는 차(데이터)** 두 대가 동시에 달려오게 됩니다. 충돌이 날 수밖에 없습니다.

이 충돌을 막기 위해 데이터패스의 모든 교차로에는 **멀티플렉서(MUX, 먹스)**라는 신호등 밸브가 반드시 설치되어 있습니다.
제어 유닛(지휘자)이 `MemtoReg=1`이라는 제어 신호를 쏴주면, MUX는 "ALU 차단! 메모리 데이터만 통과시켜!"라며 철도 선로를 휙 바꿔줍니다. 이 MUX들이 없으면 데이터패스는 1초 만에 마비됩니다.

```ascii
[데이터패스 교차로의 지배자: MUX]

 (ALU 연산 결과 12) ----> [ MUX 밸브 ] ----> 레지스터 R1에 도착!
 (메모리 로드 값 99) ----> [    |     ]
                                ^
                           제어 신호 (0이면 위쪽, 1이면 아래쪽 길을 염)
```

📢 **섹션 요약 비유**: MUX는 워터파크 워터슬라이드의 '직원 아저씨(차단기)'입니다. A 터널(ALU)에서 내려오는 사람과 B 터널(메모리)에서 내려오는 사람이 풀장에 동시에 떨어져 머리가 부딪히지 않게, 직원이 한쪽 터널을 막고 교대로 한 명씩만 통과시켜 주는 완벽한 교통정리 시스템입니다.

## Ⅲ. 아키텍처 관점: 파이프라인 데이터패스(Pipelined Datapath)
현대 컴퓨터 아키텍처는 이 데이터패스를 그냥 한 덩어리로 쓰지 않고, 5단계(Fetch, Decode, Execute, Memory, Write-back)로 토막을 내어 **파이프라인 데이터패스**로 만듭니다.

각 단계가 섞이지 않게 하려면 데이터패스의 길목마다 거대한 댐(파이프라인 레지스터/래치)을 세워야 합니다.
예를 들어 Execute(ALU 연산) 구역에서 1번 명령어의 덧셈이 끝나면, 그 결과(12)는 즉시 다음 구역인 Memory 구역으로 슉 넘어가서 파이프라인 래치에 탁 갇힙니다. 그와 동시에 빈 Execute 구역에는 2번 명령어의 데이터가 들어와 새로운 연산을 시작합니다. 
즉, 현대 데이터패스는 **클럭(Clock)이 한 번 칠 때마다 5개의 다른 구역에서 5개의 데이터가 한 칸씩 일제히 옆으로 점프하며 폭포수처럼 쏟아지는 거대한 릴레이 트랙**입니다.

```ascii
[5단계 파이프라인 데이터패스 구조]

 [명령어 메모리] -> (IF/ID 래치) -> [레지스터 파일] -> (ID/EX 래치) -> [ALU 덧셈기] -> (EX/MEM 래치) -> [데이터 메모리] -> (MEM/WB 래치) -> 다시 레지스터로!
 (1구역 Fetch)                   (2구역 Decode)                   (3구역 Execute)                 (4구역 Memory)
```

📢 **섹션 요약 비유**: 파이프라인 데이터패스는 뷔페의 '배식 줄서기'입니다. 접시(데이터)를 든 사람들이 샐러드 코너(IF), 고기 코너(EX), 디저트 코너(MEM)를 차례대로 지나갑니다. 코너 사이에 보이지 않는 줄(래치)이 있어서 앞사람이 고기를 퍼서 옆으로 한 칸 비켜줘야 뒷사람이 고기 코너로 진입할 수 있는 유기적인 흐름입니다.

## 개념 맵 (Concept Map)
```ascii
[데이터패스 (Datapath)]
   │
   ├── 정의: CPU 내에서 데이터가 물리적으로 이동하고 연산되는 하드웨어 회로들의 집합
   │     └── 구성요소: 레지스터 파일, ALU, 시스템 버스, 명령어/데이터 메모리
   │
   ├── 흐름 통제 인터페이스: MUX (멀티플렉서)
   │     └── Control Unit(제어 유닛)의 신호를 받아 여러 경로 중 하나의 데이터만 선택적 통과
   │
   └── 진화: 파이프라인 데이터패스 (Pipelined Datapath)
         └── 단일 경로를 파이프라인 레지스터(래치)로 분할 격리하여 클럭당 다중 명령어 중첩 실행 지원
```

## 어린이 비유
데이터패스는 컴퓨터 공장 안에 복잡하게 얽혀 있는 **구슬 굴리기 미로 트랙**이에요! 빨간 구슬과 파란 구슬(데이터)이 출발선(레지스터)에서 튕겨 나와 롤러코스터 트랙(전선 버스)을 타고 씽쌩씽쌩 달려요. 가다가 망치 기계(ALU)를 만나서 두 구슬이 합쳐져 보라색 구슬이 되기도 하고, 신호등(MUX)을 만나서 왼쪽 길이나 오른쪽 길로 빠지기도 하면서 도착지 창고에 쏙 들어가는 신나는 미로 탐험길이랍니다!
