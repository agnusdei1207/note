# 182. PC 상대 주소 지정 (PC-Relative Addressing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 현재 명령어 위치(PC)에서 상대적 오프셋으로 주소 계산
> 2. **가치**: 위치 독립 코드, 분기, 데이터 접근 최적화
> 3. **융합**: PIC, 공유 라이브러리, JIT, ASLR

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

PC 상대 주소 지정(PC-Relative Addressing)은 **프로그램 카운터(PC)의 현재 값에 오프셋을 더해 유효 주소를 계산**하는 방식이다. 위치 독립 코드(PIC) 작성에 필수적이며, 분기와 지역 데이터 접근에 널리 사용된다.

### 💡 비유: "여기서 오른쪽으로 3걸음"

PC 상대 주소 지정은 **"여기서 3걸음 오른쪽"**과 같다. 절대 위치("서울시 강남구...")가 아니라 현재 위치에서 상대적으로 이동한다. 어디에 서 있든 3걸음 오른쪽은 항상 같은 의미다.

### PC 상대 주소 지정 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                PC 상대 주소 지정 구조                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조】                                                           │
│  ─────────                                                          │
│                                                                     │
│  EA = PC + Offset                                                   │
│  유효 주소 = 현재 PC 값 + 오프셋                                     │
│                                                                     │
│  【동작 원리】                                                       │
│  ─────────────                                                      │
│                                                                     │
│  메모리:                                                            │
│  ┌────────────────────────────────────────────┐                     │
│  │ 0x1000: current instruction    ← PC        │                     │
│  │ 0x1004: next instruction                   │                     │
│  │ ...                                        │                     │
│  │ 0x1100: target data/label    ← EA (PC+256) │                     │
│  └────────────────────────────────────────────┘                     │
│                                                                     │
│  PC = 0x1004 (fetch 시 이미 증가)                                   │
│  Offset = 0xFC (252)                                                │
│  EA = 0x1004 + 0xFC = 0x1100                                        │
│                                                                     │
│  【ISA별 문법】                                                     │
│  ──────────────                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ISA      문법                 예시                             │ │
│  │  ───────  ───────────────────  ───────────────────             │ │
│  │  x86-64   [rip + offset]       LEA RAX, [rip+0x100]            │ │
│  │  ARM      [PC, #offset]        LDR R0, [PC, #0x100]            │ │
│  │  MIPS     beq rs, rt, offset   BEQ $t0, $t1, label             │ │
│  │  RISC-V   auipc + jalr         AUIPC + JALR                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【PC 값 차이】                                                      │
│  ──────────────                                                     │
│  • x86: PC = 현재 명령어 끝                                          │
│  • ARM: PC = 현재 명령어 + 8 (파이프라인)                           │
│  • MIPS: PC = 현재 명령어 + 4                                        │
│  • RISC-V: PC = 현재 명령어                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  PC 상대 주소 장점:                                               │ │
│  │  ──────────────────────                                           │ │
│  │  • 위치 독립 코드 (PIC)                                           │ │
│  │  • 재배치 불필요 (로드 위치 무관)                                 │ │
│  │  • ASLR 호환                                                      │ │
│  │  • 작은 오프셋으로 짧은 명령어                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. PC 상대 주소 지정 활용

### 활용 패턴

```
┌─────────────────────────────────────────────────────────────────────┐
│                PC 상대 주소 지정 활용                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 조건부/무조건 분기】                                           │
│  ──────────────────────────                                         │
│  // C 코드                                                           │
│  if (x > 0) goto positive;                                          │
│                                                                     │
│  // x86-64                                                          │
│  CMP EAX, 0                                                        │
│  JG positive           ; PC 상대 분기                                │
│                                                                     │
│  // ARM                                                             │
│  CMP R0, #0                                                         │
│  BGT positive           ; PC 상대 분기                               │
│                                                                     │
│  【2. 전역 변수 접근 (x86-64)】                                       │
│  ──────────────────────────────                                     │
│  int global_var;                                                    │
│                                                                     │
│  // RIP-relative                                                    │
│  MOV EAX, [rip + global_var]  ; PC 상대                             │
│  LEA RAX, [rip + global_var]  ; 주소 가져오기                        │
│                                                                     │
│  【3. 함수 호출 (PC 상대)】                                          │
│  ──────────────────────────                                         │
│  // 직접 호출                                                        │
│  CALL function          ; PC 상대                                    │
│                                                                     │
│  // x86-64 (32비트 오프셋)                                           │
│  E8 xx xx xx xx         ; CALL rel32                                │
│                                                                     │
│  【4. 문자열 리터럴 접근】                                           │
│  ──────────────────────────                                         │
│  const char *msg = "Hello";                                         │
│                                                                     │
│  LEA RSI, [rip + .Lstr]   ; 문자열 주소                              │
│  ...                                                                │
│  .Lstr:                                                             │
│      .asciz "Hello"                                                 │
│                                                                     │
│  【5. GOT/PLT 접근 (동적 링킹)】                                      │
│  ─────────────────────────────                                      │
│  // 전역 변수 (GOT via RIP-relative)                                │
│  MOV RAX, [rip + global@GOTPCREL]                                   │
│                                                                     │
│  // 함수 호출 (PLT via RIP-relative)                                │
│  CALL [rip + printf@PLT]                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ARM literal pool:                                                │ │
│  │  ──────────────────                                               │ │
│  │  ; 큰 상수/주소는 literal pool에 저장                             │ │
│  │  LDR R0, =0x12345678    ; 어셈블러가 pool 생성                   │ │
│  │  ; 실제로는:                                                      │ │
│  │  LDR R0, [PC, #offset]  ; PC 상대로 pool에서 로드                │ │
│  │  ...                                                             │ │
│  │  .ltorg                    ; literal pool                        │ │
│  │  .word 0x12345678                                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. PC 상대 주소 지정 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                PC 상대 주소 지정 × 타 과목 융합                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【운영체제】                                                        │
│  • PIC (Position Independent Code): 로드 위치 무관                  │
│  • 공유 라이브러리: 임의 주소 로드                                   │
│  • ASLR: 주소 무작위화 호환                                          │
│  • PIE (Position Independent Executable)                           │
│                                                                     │
│  【컴파일러】                                                        │
│  • 지역 분기: if/while/for                                          │
│  • 전역 변수: RIP-relative                                          │
│  • 문자열: literal pool                                              │
│  • PIC 모드: -fPIC 옵션                                              │
│                                                                     │
│  【링커/로더】                                                       │
│  • 재배치: R_X86_64_PC32 (PC 상대)                                  │
│  • GOT: Global Offset Table                                         │
│  • PLT: Procedure Linkage Table                                     │
│  • 심볼 해석: 런타임 바인딩                                          │
│                                                                     │
│  【보안】                                                            │
│  • ASLR: 주소 예측 어렵게                                            │
│  • ROP: 가젯 위치                                                   │
│  • CFI: 제어 흐름 무결성                                             │
│  • PIC: 코드 섹션 읽기 전용                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  x86-64 RIP-relative 인코딩:                                      │ │
│  │  ─────────────────────────────                                    │ │
│  │  MOV EAX, [RIP + 0x1234]                                         │ │
│  │  → 8B 05 34 12 00 00                                              │ │
│  │     │  │  └─────────── offset (32비트)                            │ │
│  │     │  └──────────── ModR/M (RIP-relative = 0x05)                │ │
│  │     └─────────────── opcode                                       │ │
│  │                                                               │ │
│  │  ModR/M = 0x05: 특별한 RIP-relative 인코딩                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### PC 상대 주소 지정 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                PC 상대 주소 지정 분석                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【objdump로 PC 상대 확인】                                         │
│  ──────────────────────────                                         │
│  $ objdump -d program                                               │
│  1000:  48 8b 05 00 10 00 00   mov rax,[rip+0x1000]                 │
│  1007:  e8 f4 0f 00 00         call 2000 <func>                     │
│        └───────────────────    (PC 상대 호출)                        │
│                                                                     │
│  【relocation 확인】                                                 │
│  ──────────────────                                                  │
│  $ readelf -r program                                               │
│  R_X86_64_PC32    .data+0x0    ; PC 상대 재배치                     │
│  R_X86_64_PLT32    printf+0x0  ; PLT 호출                           │
│                                                                     │
│  【PIC vs 비-PIC 비교】                                              │
│  ──────────────────────────                                         │
│  // 비-PIC (절대 주소)                                               │
│  MOV EAX, [0x401000]     ; 고정 주소                                │
│                                                                     │
│  // PIC (RIP-relative)                                              │
│  MOV EAX, [rip + offset] ; 상대 주소                                │
│                                                                     │
│  【컴파일 옵션】                                                     │
│  ──────────────────                                                  │
│  $ gcc -fPIC -c code.c        ; PIC 모드                            │
│  $ gcc -fPIE -pie code.c      ; PIE 실행 파일                       │
│  $ gcc -no-pie code.c         ; PIE 비활성화                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  PC 상대 분기 범위:                                               │ │
│  │  ──────────────────                                               │ │
│  │  x86-64 CALL: ±2GB (32비트 부호있는 오프셋)                       │ │
│  │  ARM B: ±32MB (24비트 × 4)                                       │ │
│  │  MIPS BEQ: ±128KB (16비트 × 4)                                   │ │
│  │  RISC-V JAL: ±1MB (20비트 × 2)                                   │ │
│  │                                                               │ │
│  │  범위 초과 시:                                                    │ │
│  │  • veneer/trampoline 사용                                        │ │
│  │  • 간접 분기                                                      │ │
│  │  • 여러 단계 분기                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• PC 상대 주소 지정: EA = PC + Offset

• 활용: 분기, 전역 변수, 함수 호출, GOT/PLT
• x86-64: RIP-relative 주소 지정
• 컴파일: -fPIC, -fPIE 옵션
```
