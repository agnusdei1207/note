# 196. CISC (Complex Instruction Set Computer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 복잡하고 강력한 명령어 집합, 하드웨어 복잡도
> 2. **가치**: 작은 코드 크기, 이진 호환성, 과거 메모리 제약 해결
> 3. **융합**: x86/x64, 데스크톱/서버 지배, 마이크로코드

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

CISC(Complex Instruction Set Computer)는 **복잡하고 강력한 명령어를 제공하여 적은 명령어로 많은 작업을 수행**하는 아키텍처이다. 메모리가 비싸던 시절 코드 크기를 줄이기 위해 개발되었으며, x86이 대표적이다.

### 💡 비유: 만능 주방

CISC는 **만능 주방**과 같다. "파스타 만들어"라고 한마디만 하면, 반죽부터 소스까지 다 해서 완성된 요리가 나온다. 하지만 주방(하드웨어)은 복잡하고 비싸다. RISC는 재료만 주면 요리사(컴파일러)가 직접 만드는 방식이다.

### CISC 특징

```
┌─────────────────────────────────────────────────────────────────────┐
│                CISC 특징                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 특징】                                                       │
│  ─────────────                                                      │
│  1. 명령어 수: 많음 (수백~수천 개)                                  │
│  2. 명령어 길이: 가변 (1~15바이트)                                  │
│  3. 실행 시간: 가변 (1~수십 클럭)                                   │
│  4. 메모리 연산: 직접 가능                                          │
│ 5. 레지스터: 적음 (8~16개)                                          │
│  6. 마이크로코드: 복잡한 명령어 구현                                │
│                                                                     │
│  【역사적 배경】                                                     │
│  ──────────────────                                                  │
│  • 1960-70년대: 메모리 매우 비쌈                                    │
│  • 목표: 작은 코드 크기 = 메모리 절약                               │
│  • 해결: 복잡한 명령어 하나 = 여러 간단한 명령어                    │
│  • 예: "문자열 복사" 한 명령어로                                    │
│                                                                     │
│  【vs RISC】                                                        │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성            CISC              RISC                         │ │
│  │  ──────────────  ────────────────  ─────────────────            │ │
│  │  명령어 수        많음 (~1000+)     적음 (~100)                  │ │
│  │  명령어 길이      가변 (1~15B)      고정 (32비트)                │ │
│  │  실행 시간        가변 (1~100+)     균일 (1클럭)                 │ │
│  │  메모리 연산      직접 가능         Load/Store만                 │ │
│  │  레지스터         적음 (8~16)       많음 (32)                    │ │
│  │  하드웨어         복잡              단순                          │ │
│  │  컴파일러         단순              복잡                          │ │
│  │  코드 크기        작음              큼                            │ │
│  │  전력 소모        많음              적음                          │ │
│  │  대표 ISA         x86/x64           ARM, MIPS, RISC-V           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【대표 CISC ISA】                                                   │
│  ──────────────────                                                  │
│  • x86/x64: 데스크톱/서버 지배                                      │
│  • VAX: 과거 DEC 미니컴퓨터                                         │
│  • 68k: 과거 매킨토시, 아미가                                        │
│  • z/Architecture: IBM 메인프레임                                   │
│                                                                     │
│  【x86의 복잡한 명령어 예시】                                        │
│  ────────────────────────────────                                   │
│  ; 문자열 복사 (한 명령어)                                           │
│  REP MOVSB           ; ECX 바이트 복사                               │
│                                                                     │
│  ; 문자열 검색                                                       │
│  REPNE SCASB         ; AL 값 검색                                    │
│                                                                     │
│  ; 복잡한 수학                                                        │
│  FSIN                ; 사인 계산                                     │
│  XLAT                ; 테이블 조회                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CISC 철학:                                                         │ │
│  │  ──────────                                                         │ │
│  │  "Make the hardware do more work, simplify the compiler"          │ │
│  │                                                               │ │
│  │  1. 복잡한 하드웨어 → 복잡한 작업 한 명령어로                      │ │
│  │  2. 컴파일러 단순 → 최적화 부담 적음                               │ │
│  │  3. 작은 코드 → 메모리 절약                                        │ │
│  │  4. 이진 호환 → 오랜 기간 유지                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. CISC 상세

### CISC 특징

```
┌─────────────────────────────────────────────────────────────────────┐
│                CISC 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 가변 길이 명령어】                                             │
│  ──────────────────────────                                         │
│  • 1~15바이트                                                        │
│  • 코드 밀도 높음                                                    │
│  • 디코딩 복잡                                                       │
│  • 파이프라인 어려움                                                 │
│                                                                     │
│  ; x86 가변 길이 예시                                                │
│  RET                 ; 1바이트                                       │
│  ADD EAX, 1          ; 3바이트                                       │
│  MOV EAX, [EBX+ECX*4+100] ; 7바이트                                 │
│                                                                     │
│  【2. 메모리-레지스터 연산】                                         │
│  ──────────────────────────────                                     │
│  ; CISC (x86)                                                        │
│  ADD EAX, [mem]      ; 메모리 직접 연산                              │
│  ADD [mem], EAX      ; 메모리에 직접 더하기                          │
│                                                                     │
│  ; RISC는 이렇게 안 됨                                               │
│  LDR R0, [mem]                                                      │
│  ADD R0, R0, R1                                                     │
│  STR R0, [mem]                                                      │
│                                                                     │
│  【3. 마이크로코드】                                                 │
│  ──────────────────                                                  │
│  • 복잡한 명령어 → 내부 RISC 명령어로 변환                          │
│  • ROM에 저장된 마이크로프로그램                                     │
│  • 유연성: 새로운 명령어 추가 가능                                   │
│  • 성능: 복잡한 명령어는 느림                                        │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  복잡한 명령어 (x86)          →    마이크로옵으로 분해              │   │
│  │  REP MOVSB                    →    여러 load/store                  │   │
│  │  STRING ops                   →    루프 + 이동                      │   │
│  │  복잡한 산술                   →    단순 산술 조합                  │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【4. 2-주소 명령어】                                                │
│  ──────────────────────                                             │
│  ADD EAX, EBX        ; EAX = EAX + EBX                              │
│  ; 목적지 = 원본 + 목적지 (파괴적)                                  │
│                                                                     │
│  【5. 다양한 주소 지정】                                             │
│  ──────────────────────                                             │
│  MOV EAX, [EBX+ECX*4+100]  ; 복잡한 주소 계산                       │
│  ; Base + Index * Scale + Displacement                              │
│                                                                     │
│  【6. 이진 호환성】                                                  │
│  ──────────────────                                                  │
│  • 1978년 8086 코드 → 2024년 CPU에서 실행 가능                      │
│  • 누적된 호환성 부담                                                │
│  • 레거시 명령어 유지                                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  현대 x86의 RISC화:                                                 │ │
│  │  ────────────────────                                              │ │
│  │  • 내부적으로 RISC 코어 사용                                       │ │
│  │  • CISC → uOp 변환                                                 │ │
│  │  • uOp 캐시 (L1)                                                   │ │
│  │  • Out-of-order 실행                                               │ │
│  │                                                               │ │
│  │  → 외부는 CISC, 내부는 RISC                                        │ │
│  │  → 복잡하지만 고성능 달성                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. CISC × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                CISC × 타 과목 융합                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                        │
│  • 최적화: 복잡한 명령어 활용                                        │
│  • 코드 선택: 가변 길이 명령어                                      │
│  • 인라인 어셈블리: 쉬움                                             │
│  • 최적화 부담: 적음 (하드웨어가 처리)                              │
│                                                                     │
│  【컴퓨터 구조】                                                     │
│  • 마이크로코드: 복잡한 명령어 구현                                  │
│  • 디코더: 복잡 (길이 파악 필요)                                    │
│  • uOp 캐시: 변환 결과 캐싱                                          │
│  • 파이프라인: 깊고 복잡                                             │
│                                                                     │
│  【운영체제】                                                        │
│  • x86_64: 데스크톱 OS 표준                                         │
│  • 가상화: 하드웨어 지원                                             │
│  • 보안: NX bit, SMEP                                               │
│  • 전력: C-State, P-State                                           │
│                                                                     │
│  【성능】                                                           │
│  • IPC: 2~4 달성 (현대 x86)                                         │
│  • 클럭: 3~5 GHz                                                    │
│  • 코어: 8~64코어                                                   │
│  • TDP: 65~350W                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  x86 역사와 호환성:                                                 │ │
│  │  ────────────────────                                              │ │
│  │  1978: 8086 (16비트)                                               │ │
│  │  1985: 386 (32비트)                                                │ │
│  │  2003: x86-64 (64비트)                                             │ │
│  │  2020s: AVX-512, AMX                                              │ │
│  │                                                               │ │
│  │  모든 시대의 코드가 현대 CPU에서 실행됨!                            │ │
│  │  → 엄청난 소프트웨어 자산                                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### CISC 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                CISC 분석                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【x86-64 어셈블리 분석】                                            │
│  ──────────────────────────                                         │
│  ; int sum_array(int *arr, int n) {                                 │
│  ;     int sum = 0;                                                 │
│  ;     for (int i = 0; i < n; i++) sum += arr[i];                   │
│  ;     return sum;                                                  │
│  ; }                                                                │
│                                                                     │
│  sum_array:                                                         │
│      xor eax, eax           ; sum = 0                               │
│      xor ecx, ecx           ; i = 0                                 │
│      test esi, esi          ; n == 0?                               │
│      je .done                                                        │
│  .loop:                                                             │
│      add eax, [rdi + rcx*4] ; sum += arr[i]                         │
│      inc ecx                 ; i++                                  │
│      cmp ecx, esi            ; i < n?                               │
│      jl .loop                                                        │
│  .done:                                                             │
│      ret                                                           │
│                                                                     │
│  【objdump 분석】                                                    │
│  ──────────────────                                                  │
│  $ objdump -d program                                               │
│  1000:  31 c0           xor eax, eax         ; 2바이트              │
│  1002:  31 c9           xor ecx, ecx         ; 2바이트              │
│  1004:  85 f6           test esi, esi        ; 2바이트              │
│  1006:  74 07           je 100f              ; 2바이트              │
│  1008:  03 04 8f        add eax,[rdi+rcx*4]  ; 3바이트              │
│  100b:  ff c1           inc ecx              ; 2바이트              │
│  100d:  eb f9           jmp 1008             ; 2바이트              │
│  ; 가변 길이 (2~3바이트)                                             │
│                                                                     │
│  【uOp 캐시 분석】                                                   │
│  ──────────────────────                                             │
│  $ perf stat -e uops_retired.any ./program                          │
│  uOps retired: 1,234,567,890                                        │
│  ; CISC 명령어 → 여러 uOp                                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CISC vs RISC 현실:                                                 │ │
│  │  ────────────────────                                              │ │
│  │  1990s: RISC가 성능 우위                                           │ │
│  │  2000s: x86가 클럭 경쟁으로 따라잡음                               │ │
│  │  2010s: 내부 RISC화로 성능 향상                                    │ │
│  │  2020s: ARM가 에너지 효율로 서버 진입                             │ │
│  │                                                               │ │
│  │  → 경계가 모호해짐                                                  │ │
│  │  → CISC는 내부 RISC, RISC는 확장으로 복잡해짐                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```
