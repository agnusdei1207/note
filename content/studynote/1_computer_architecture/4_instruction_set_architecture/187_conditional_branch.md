# 187. 조건부 분기 (Conditional Branch)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 조건이 참일 때만 분기하는 명령어
> 2. **가치**: if/else, while, for 구현의 핵심
> 3. **융합**: 분기 예측, 파이프라인, 알고리즘 최적화

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

조건부 분기(Conditional Branch)는 **플래그 레지스터 상태에 따라 분기 여부를 결정**하는 명령어다. JE, JNE, JG, JL 등의 조건 코드가 있으며, CMP나 TEST 명령어와 함께 사용된다.

### 💡 비유: 신호등에서 방향 결정

조건부 분기는 **신호등에서 방향을 결정**하는 것과 같다. "신호가 빨간 불이면(right turn) 정지하고, 초록 불이면(left turn) 직진"처럼 조건에 따라 다른 길로 간다.

### 조건부 분기 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                조건부 분기 구조                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동작 원리】                                                       │
│  ─────────────                                                      │
│                                                                     │
│  1. 조건 설정: CMP, TEST, 산술 연산                                 │
│  2. 플래그 갱신: Z, N, C, V 플래그                                  │
│  3. 조건 검사: 플래그 조합 확인                                      │
│  4. 분기 결정: 조건이 참이면 PC = 목적지                            │
│                                                                     │
│  【기본 패턴】                                                       │
│  ─────────────                                                      │
│                                                                     │
│  CMP R1, R2           ; R1 - R2, 플래그 설정                        │
│  Jcc label            ; 조건이 참이면 분기                          │
│                                                                     │
│  또는                                                                │
│                                                                     │
│  TEST R1, R1          ; R1 & R1, 플래그 설정                        │
│  Jcc label            ; 조건이 참이면 분기                          │
│                                                                     │
│  【x86 조건부 분기 코드】                                            │
│  ──────────────────────────                                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  코드      조건           의미             부호               │ │
│  │  ────────  ─────────────  ───────────────  ────────────────   │ │
│  │  JE/JZ     ZF=1           같음 / 0         -                  │ │
│  │  JNE/JNZ   ZF=0           같지 않음        -                  │ │
│  │                                                               │ │
│  │  JA/JNBE   CF=0 && ZF=0    초과             부호 없음          │ │
│  │  JAE/JNB   CF=0            이상             부호 없음          │ │
│  │  JB/JNAE   CF=1            미만             부호 없음          │ │
│  │  JBE/JNA   CF=1 || ZF=1    이하             부호 없음          │ │
│  │                                                               │ │
│  │  JG/JNLE   SF=OF && ZF=0   초과             부호 있음          │ │
│  │  JGE/JNL   SF=OF           이상             부호 있음          │ │
│  │  JL/JNGE   SF!=OF          미만             부호 있음          │ │
│  │  JLE/JNG   SF!=OF || ZF=1  이하             부호 있음          │ │
│  │                                                               │ │
│  │  JS        SF=1            음수             -                  │ │
│  │  JNS       SF=0            양수/0           -                  │ │
│  │  JC        CF=1            캐리             -                  │ │
│  │  JNC       CF=0            캐리 없음        -                  │ │
│  │  JO        OF=1            오버플로우       -                  │ │
│  │  JNO       OF=0            오버플로우 없음  -                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ARM 조건부 분기】                                                 │
│  ────────────────────                                               │
│  BEQ (Equal), BNE (Not Equal)                                       │
│  BGT (Greater Than), BLT (Less Than)                                │
│  BGE (Greater or Equal), BLE (Less or Equal)                        │
│  BHI (Higher, unsigned), BLO (Lower, unsigned)                      │
│  BMI (Minus), BPL (Plus)                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  부호 있음 vs 부호 없음 분기:                                      │ │
│  │  ──────────────────────────────                                   │ │
│  │  부호 있음: JG, JGE, JL, JLE (signed comparison)                 │ │
│  │  부호 없음: JA, JAE, JB, JBE (unsigned comparison)               │ │
│  │                                                               │ │
│  │  예: 0xFFFFFFFF                                                  │ │
│  │  • 부호 있음: -1                                                  │ │
│  │  • 부호 없음: 4294967295                                         │ │
│  │  → 같은 값도 비교 결과가 다름!                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 조건부 분기 활용

### 활용 패턴

```
┌─────────────────────────────────────────────────────────────────────┐
│                조건부 분기 활용                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. if 문】                                                        │
│  ─────────────                                                      │
│  // if (a == b) { ... }                                             │
│  CMP EAX, EBX                                                       │
│  JNE skip                ; a != b면 건너뜀                           │
│  ; ... then 블록                                                    │
│  skip:                                                              │
│                                                                     │
│  【2. if-else 문】                                                   │
│  ──────────────────                                                  │
│  // if (a > b) { ... } else { ... }                                 │
│  CMP EAX, EBX                                                       │
│  JLE else                ; a <= b면 else로                          │
│  ; ... then 블록                                                    │
│  JMP done                                                          │
│  else:                                                              │
│  ; ... else 블록                                                    │
│  done:                                                              │
│                                                                     │
│  【3. while 루프】                                                   │
│  ──────────────────                                                  │
│  // while (i < n) { ... i++; }                                      │
│  loop:                                                              │
│  CMP ECX, EDX            ; i < n?                                   │
│  JGE done                ; i >= n이면 종료                          │
│  ; ... 루프 본문                                                    │
│  INC ECX                  ; i++                                     │
│  JMP loop                                                          │
│  done:                                                              │
│                                                                     │
│  【4. for 루프】                                                     │
│  ──────────────────                                                  │
│  // for (i=0; i<n; i++) { ... }                                     │
│  XOR ECX, ECX            ; i = 0                                    │
│  loop:                                                              │
│  CMP ECX, EDX            ; i < n?                                   │
│  JGE done                                                          │
│  ; ... 루프 본문                                                    │
│  INC ECX                  ; i++                                     │
│  JMP loop                                                          │
│  done:                                                              │
│                                                                     │
│  【5. NULL 포인터 검사】                                             │
│  ──────────────────────                                             │
│  TEST RDI, RDI           ; ptr == NULL?                             │
│  JZ null_handler                                                    │
│  ; ptr 사용                                                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  최적화된 조건부 이동 (CMOV):                                      │ │
│  │  ──────────────────────────────────                              │ │
│  │  ; 분기 없이 조건부 실행                                           │ │
│  │  CMP EAX, EBX                                                     │ │
│  │  CMOVGE ECX, EDX    ; EAX >= EBX이면 ECX = EDX                    │ │
│  │                                                               │ │
│  │  장점:                                                            │ │
│  │  • 분기 예측 실패 없음                                             │ │
│  │  • 파이프라인 깨짐 없음                                            │ │
│  │                                                               │ │
│  │  단점:                                                            │ │
│  │  • 두 경로 모두 실행됨 (비용 발생)                                 │ │
│  │  • 모든 조건에 적합하지 않음                                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 조건부 분기 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                조건부 분기 × 타 과목 융합                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                        │
│  • if/switch → 조건부 분기                                          │
│  • 루프 → 조건부 분기 + JMP                                          │
│  • 단락 평가: && || → 조건부 분기 체인                               │
│  • 프로파일 가이드: 분기 확률 기반 배치                              │
│                                                                     │
│  【컴퓨터 구조】                                                     │
│  • 분기 예측: 정적/동적 예측                                         │
│  • 파이프라인: 분기 비용 (버블)                                      │
│  • BTB: 분기 목표 버퍼                                               │
│  • 스페큐레이션: 분기 예측 실패 복구                                 │
│                                                                     │
│  【알고리즘】                                                        │
│  • 이진 검색: 중간 비교 분기                                         │
│  • 정렬: 비교 후 분기                                                │
│  • 상태 머신: 상태 전이 분기                                         │
│  • 트리 순회: 조건부 재귀                                            │
│                                                                     │
│  【운영체제】                                                        │
│  • 스케줄링: 우선순위 비교                                           │
│  • 락: 조건부 대기                                                   │
│  • 에러 처리: 조건부 분기                                            │
│  • 시스템 콜: 조건부 경로                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  분기 예측 성능 영향:                                              │ │
│  │  ────────────────────────                                         │ │
│  │  예측 성공: 1-2 클럭                                              │ │
│  │  예측 실패: 10-20 클럭 (파이프라인 flush)                         │ │
│  │                                                               │ │
│  │  최적화 전략:                                                      │ │
│  │  1. 자주 실행되는 경로를 fall-through                             │ │
│  │  2. __builtin_expect (likely/unlikely)                           │ │
│  │  3. CMOV로 분기 제거                                               │ │
│  │  4. 루프 언롤링                                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 조건부 분기: 플래그에 따라 분기 결정
• 전제: CMP, TEST로 플래그 설정
• x86: JE, JNE, JG, JL, JA, JB, ...
• ARM: BEQ, BNE, BGT, BLT, ...
• 부호: JG/JL (signed) vs JA/JB (unsigned)
• 성능: 분기 예측 실패 시 비용 큼
```
