> **핵심 인사이트**
> 1. 플린의 분류(Flynn's Taxonomy)는 1966년 마이클 플린이 제안한, 세상의 모든 컴퓨터 아키텍처를 **'명령어의 개수(Instruction Stream)'와 '데이터의 개수(Data Stream)'라는 두 가지 축으로 명쾌하게 4등분 한 절대적인 컴퓨터 분류법**이다.
> 2. **SISD (싱글 명령-싱글 데이터)**는 과거 펜티엄 시절까지의 고전적인 단일 코어 컴퓨터 구조를 뜻한다.
> 3. 현대 컴퓨터는 이 분류표를 넘어, **SIMD(GPU의 데이터 폭격)**와 **MIMD(멀티코어 CPU의 스레드 독립 실행)**를 기가 막히게 혼합하여 궁극의 병렬 처리 성능을 달성하고 있다.

---

## Ⅰ. 컴퓨터를 나누는 두 개의 칼날: 명령어와 데이터
마이클 플린은 컴퓨터가 결국 "어떤 지시(명령어)를 받아서, 어떤 재료(데이터)를 요리하느냐?"의 싸움이라고 보았습니다. 
명령어 흐름(Instruction Stream)이 한 가닥(Single)인지 여러 가닥(Multiple)인지, 그리고 데이터 흐름(Data Stream)이 한 가닥(Single)인지 여러 가닥(Multiple)인지에 따라 딱 4가지($2 \times 2$)의 기계로 전 세계 컴퓨터를 분류해 냈습니다.

1. **SISD (Single Instruction, Single Data)**: 한 번에 1개의 명령어로 1개의 데이터만 처리.
2. **SIMD (Single Instruction, Multiple Data)**: 1개의 똑같은 명령어로 수천 개의 데이터를 한방에 처리.
3. **MISD (Multiple Instruction, Single Data)**: 여러 개의 명령어가 1개의 데이터를 집단 구타함. (실제로는 거의 안 쓰임)
4. **MIMD (Multiple Instruction, Multiple Data)**: 여러 개의 명령어가 제각각 여러 개의 데이터를 독립적으로 처리.

```ascii
[플린의 4대 분류 매트릭스 (Flynn's Taxonomy)]

                 Single Data (SD)       |  Multiple Data (MD)
              --------------------------+----------------------------
 Single       |                         |
 Instruction  |        SISD             |         SIMD
 (SI)         | (전통적인 단일 CPU)     | (GPU, 벡터 프로세서)
              |                         |
 -------------+-------------------------+----------------------------
 Multiple     |                         |
 Instruction  |        MISD             |         MIMD
 (MI)         | (우주선 등 특수 목적용) | (멀티코어 CPU, 클러스터)
              |                         |
```

📢 **섹션 요약 비유**: 플린의 분류는 '식당 주방의 4가지 유형'입니다. 명령어는 요리사(지시자)이고 데이터는 화구(프라이팬)입니다. 요리사 1명이 1구에서 요리하는 곳(SISD), 요리사 1명이 100구 화구에 똑같은 계란을 미친 듯이 굽는 곳(SIMD), 요리사 10명이 1개의 프라이팬을 쳐다보며 참견하는 곳(MISD), 10명의 요리사가 각자 자기 프라이팬 10개로 완전히 다른 요리를 만드는 대형 식당(MIMD)으로 세상을 깔끔하게 정리했습니다.

## Ⅱ. SISD, SIMD, MIMD의 실체
각각의 분류가 실제 컴퓨터 역사에서 어떻게 쓰였는지 살펴봅시다.

* **SISD (초기 컴퓨터의 고독)**: 
  우리가 아는 옛날 286, 386 컴퓨터입니다. 파이프라인을 쓰든 수퍼스칼라를 쓰든, 어쨌든 1개의 프로그램 카운터(PC)가 1줄의 명령어를 가져와서 1개의 덧셈을 하고 끝납니다. 직관적이고 코딩하기 쉽지만 속도에 한계가 명확합니다.
* **SIMD (데이터 물량 공세)**:
  `A[0]+B[0]`, `A[1]+B[1]...` 이런 덧셈을 100번 시키면 SISD는 100클럭이 걸립니다. SIMD는 1개의 `ADD` 명령어를 치는 순간, 거대한 하드웨어가 A배열 100개와 B배열 100개를 동시에 빨아들여 1클럭 만에 100개의 덧셈 결과를 토해냅니다. **엔비디아 GPU의 코어 철학**이 바로 이것입니다.
* **MIMD (자유와 혼돈의 멀티코어)**:
  현대의 인텔 Core i9이나 AMD 라이젠 CPU입니다. 코어가 8개라면 PC(프로그램 카운터)도 8개입니다. 1번 코어는 웹 브라우저(명령어 A, 데이터 A)를 돌리고, 2번 코어는 카카오톡(명령어 B, 데이터 B)을 돌립니다. 각자 자기 갈 길을 가는 가장 유연한 병렬 컴퓨터입니다.

```ascii
[SISD vs SIMD vs MIMD 의 시각적 차이]

 [SISD] 명령어 -> (연산기 1) -> 데이터 1 처리

 [SIMD]               /-> (연산기 1) -> 데이터 1
        명령어 (동일) -+-> (연산기 2) -> 데이터 2
                      \-> (연산기 3) -> 데이터 3  (한 방에 쾅!)

 [MIMD] 명령어 A -> (연산기 1) -> 데이터 A
        명령어 B -> (연산기 2) -> 데이터 B
        명령어 C -> (연산기 3) -> 데이터 C  (각자도생!)
```

📢 **섹션 요약 비유**: SISD는 혼자서 빵 1개 굽기, SIMD는 붕어빵 틀에 반죽을 부어 한 번에 10개 찍어내기, MIMD는 셰프 10명이 모여서 각자 피자, 파스타, 스테이크 등 주문 들어오는 대로 맘대로 만들기입니다.

## Ⅲ. 아키텍처 관점: MISD의 미스터리와 결함 허용(Fault Tolerance)
플린의 4가지 분류 중 가장 미스터리한 것이 **MISD (여러 명령어가 1개의 데이터를 팸)**입니다.
"데이터 1개(`A=5`)를 가지고 더하기도 하고 빼기도 하고 곱하기도 동시에 한다고? 그게 무슨 의미가 있어?" 
맞습니다. 일반적인 상업용 컴퓨터에서는 이 방식이 전혀 쓸모가 없어서 MISD 칩은 시장에 존재하지 않습니다.

하지만 **목숨이 걸린 항공우주 시스템이나 미사일 제어 아키텍처**에서는 이 MISD가 구세주가 됩니다.
우주 방사선 때문에 CPU 하나가 미쳐서 "미사일 발사!"라고 잘못된 계산을 내릴 수 있습니다. 그래서 1개의 비행 데이터(고도 10,000m)를 **3개의 완전히 다른 코드로 짜인 독립적인 프로세서(명령어 A, B, C)에 동시에 집어넣습니다.** 
3개의 프로세서가 똑같은 데이터를 가지고 각자 계산을 합니다. 결과가 `[직진], [직진], [추락]`으로 다르게 나왔다면? 시스템은 다수결의 원칙을 따라 '직진'을 선택하고 뻗어버린 1개의 칩을 차단합니다. 이것이 바로 절대 고장 나면 안 되는 **결함 허용 시스템(Fault Tolerant System)**의 궁극적 설계입니다.

```ascii
[MISD 기반의 결함 허용(Redundancy) 시스템 원리]

               /-> [명령어(CPU) A] -> 결과: 100
 데이터 X(1개) -+-> [명령어(CPU) B] -> 결과: 100
               \-> [명령어(CPU) C] -> 결과: 999 (오류 발생!)

 [ 다수결 회로 (Voter) ] -> "100이 두 표니까 100이 정답! C는 고장났으니 무시해!" 
 -> (비행기 추락 방지 완벽 성공!)
```

📢 **섹션 요약 비유**: MISD는 '수학경시대회 교차 채점'입니다. 학생(데이터)이 낸 답안지 1장을 선생님 3명(명령어 3개)이 동시에 각자 채점합니다. 선생님 1명이 졸아서 점수를 잘못 매겨도, 나머지 2명의 선생님이 제대로 매겼다면 그 점수를 진짜 점수로 인정하여 치명적인 채점 실수를 100% 방어하는 완벽한 보안 시스템입니다.

## 개념 맵 (Concept Map)
```ascii
[플린의 분류 (Flynn's Taxonomy)]
   │
   ├── 정의: Instruction Stream(명령어 흐름)과 Data Stream(데이터 흐름)의 개수에 따른 컴퓨터 아키텍처의 4대 분류법
   │
   ├── 병렬성의 진화
   │     ├── SISD (Single I, Single D): 단일 코어 기반의 전통적 폰 노이만 구조 (ILP에 의존)
   │     ├── SIMD (Single I, Multiple D): 하나의 명령으로 다수 데이터를 융단 폭격 -> GPU/벡터 아키텍처 (DLP의 핵심)
   │     └── MIMD (Multiple I, Multiple D): 독립된 멀티코어들이 각자 스레드 처리 -> 현대 CPU 아키텍처 (TLP의 핵심)
   │
   └── 특수 목적 아키텍처
         └── MISD (Multiple I, Single D): 상용으론 부적합하나, 우주/국방 분야의 다수결 기반 결함 허용(Fault Tolerance) 시스템에 활용
```

## 어린이 비유
플린 할아버지가 쓴 분류법은 **학교 청소 시간을 나누는 4가지 방법**이에요!
- **SISD**: 나 혼자서 빗자루 하나 들고 교실 하나 청소하기. (평범하고 느림)
- **SIMD**: 반장 한 명이 "창문 닦아!"라고 소리치면, 반 친구 30명이 동시에 각자 자기 자리 창문을 닦기. (GPU처럼 엄청 빠름!)
- **MIMD**: 5명은 창문 닦고, 5명은 바닥 쓸고, 5명은 쓰레기통 비우기. 각자 할 일이 다 다름! (멀티코어 CPU!)
- **MISD**: 창문 하나(데이터)를 두고 3명의 친구(명령어)가 동시에 걸레질하기. 바보 같아 보이지만, 절대 창문에 얼룩이 안 남게 만드는 특수 청소법이랍니다!
