# 195. RISC (Reduced Instruction Set Computer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단순하고 빠른 명령어 집합, 하드웨어 단순화
> 2. **가치**: 높은 클럭, 파이프라인 효율, 에너지 효율
> 3. **융합**: ARM, MIPS, RISC-V, 모바일/임베디드 지배

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

RISC(Reduced Instruction Set Computer)는 **단순하고 규칙적인 명령어 집합으로 하드웨어를 단순화**하여 높은 성능과 에너지 효율을 달성하는 아키텍처이다. 1980년대 Stanford(ARM 원형)와 Berkeley(RISC-V 원형)에서 개발되었다.

### 💡 비유: 도시락 만들기 vs 레스토랑 요리

RISC는 **도시락 만들기**와 같다. 기본 재료(단순 명령어)만 준비하면, 조리법(소프트웨어)에 따라 무엇이든 만들 수 있다. 하드웨어(주방)는 단순하지만, 요리사(컴파일러)가 효율적으로 활용하면 최고의 요리(성능)가 나온다.

### RISC 특징

```
┌─────────────────────────────────────────────────────────────────────┐
│                RISC 특징                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 원칙】                                                       │
│  ─────────────                                                      │
│  1. 명령어 수: 적음 (50~100개)                                      │
│  2. 명령어 길이: 고정 (보통 32비트)                                  │
│  3. 실행 시간: 균일 (보통 1 클럭)                                   │
│  4. 메모리 접근: Load/Store만                                       │
│  5. 레지스터: 많음 (16~32개)                                        │
│  6. 파이프라인: 깊고 효율적                                         │
│                                                                     │
│  【vs CISC】                                                        │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성            RISC              CISC                         │ │
│  │  ──────────────  ────────────────  ───────────────────────      │ │
│  │  명령어 수        적음 (~100)       많음 (~1000)                 │ │
│  │  명령어 길이      고정 (32비트)     가변 (1~15바이트)             │ │
│  │  실행 시간        균일 (1클럭)      가변 (1~100+클럭)             │ │
│  │  메모리 연산      Load/Store만      직접 연산 가능               │ │
│  │  레지스터         많음 (32개)       적음 (8~16)                  │ │
│  │  파이프라인       깊음              얕음                          │ │
│  │  컴파일러         복잡              단순                          │ │
│  │  코드 크기        큼                작음                          │ │
│  │  전력 소모        적음              많음                          │ │
│  │  대표 ISA         ARM, MIPS, RISC-V  x86                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【대표 RISC ISA】                                                   │
│  ──────────────────                                                  │
│  • ARM: 모바일/임베디드 지배 (아이폰, 안드로이드)                    │
│  • MIPS: 임베디드, 라우터, 게임기                                    │
│  • RISC-V: 오픈소스, 확장성, IoT/서버                                │
│  • PowerPC: 게임기, 임베디드                                        │
│  • SPARC: 서버, 슈퍼컴퓨터 (과거)                                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RISC 철학 (Berkeley RISC):                                        │ │
│  │  ────────────────────────────                                      │ │
│  │  "Keep the hardware simple, let the compiler do the work"         │ │
│  │                                                               │ │
│  │  1. 단순한 하드웨어 → 높은 클럭 달성                              │ │
│  │  2. 컴파일러가 최적화 → 더 나은 코드                               │ │
│  │  3. 레지스터 많이 → 메모리 접근 감소                              │ │
│  │  4. 파이프라인 깊게 → 높은 처리량                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. RISC 상세

### RISC 특징

```
┌─────────────────────────────────────────────────────────────────────┐
│                RISC 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. Load/Store 아키텍처】                                          │
│  ────────────────────────────────                                   │
│  • 메모리 접근: LDR/STR만                                           │
│  • 연산: 레지스터 간만                                               │
│  • 장점: 메모리 지연 숨김, 캐시 최적화                               │
│  • 단점: 코드 크기 증가                                              │
│                                                                     │
│  ; RISC (ARM)                                                        │
│  LDR R0, [a]         ; 로드                                          │
│  LDR R1, [b]         ; 로드                                          │
│  ADD R2, R0, R1      ; 연산                                          │
│  STR R2, [c]         ; 저장                                          │
│                                                                     │
│  ; CISC (x86)                                                        │
│  MOV EAX, [a]        ; 로드+연산+저장 가능                           │
│  ADD EAX, [b]                                                       │
│  MOV [c], EAX                                                       │
│                                                                     │
│  【2. 고정 길이 명령어】                                             │
│  ──────────────────────────                                         │
│  • 32비트 고정 (ARM, MIPS)                                          │
│  • 디코딩 단순                                                       │
│  • 파이프라인 효율                                                   │
│  • 단점: 코드 크기 큼                                                │
│                                                                     │
│  【3. 3-주소 명령어】                                                │
│  ──────────────────────                                             │
│  • 목적지 + 2원본                                                   │
│  • 비파괴적 연산                                                     │
│  • 레지스터 효율                                                     │
│  • 예: ADD R0, R1, R2  ; R0 = R1 + R2                               │
│                                                                     │
│  【4. 대형 레지스터 파일】                                           │
│  ────────────────────────────                                       │
│  • ARM: 16개 (R0-R15) + CPSR                                        │
│  • MIPS: 32개 ($0-$31)                                              │
│  • RISC-V: 32개 (x0-x31)                                            │
│  • 장점: 스플링 감소, 성능 향상                                      │
│                                                                     │
│  【5. 하버드 아키텍처】                                              │
│  ──────────────────────                                             │
│  • 명령어/데이터 분리 캐시                                           │
│  • 동시 fetch/execute                                               │
│  • 파이프라인 효율                                                   │
│                                                                     │
│  【6. 지연 분기 (Delayed Branch)】                                   │
│  ──────────────────────────────────                                 │
│  ; MIPS                                                              │
│  BEQ R1, R2, target    ; 분기                                       │
│  NOP                   ; 지연 슬롯 (항상 실행)                       │
│  ; (분기 후에도 NOP 실행됨)                                          │
│                                                                     │
│  ; ARM은 조건부 실행으로 대체                                        │
│  CMP R0, #0                                                          │
│  ADDNE R1, R2, R3     ; R0!=0이면 실행                              │
│  ; 분기 없이 조건부 실행                                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RISC 성능 이유:                                                    │ │
│  │  ──────────────                                                    │ │
│  │  1. 단순한 디코딩 → 빠른 클럭                                       │ │
│  │  2. 균일한 실행 → 파이프라인 효율                                   │ │
│  │  3. 레지스터 많음 → 메모리 지연 회피                                │ │
│  │  4. Load/Store → 캐시 최적화                                        │ │
│  │  5. 컴파일러 최적화 → 더 나은 코드                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### RISC 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                RISC 분석                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ARM 어셈블리 분석】                                               │
│  ──────────────────────────                                         │
│  ; int factorial(int n) {                                           │
│  ;     if (n <= 1) return 1;                                        │
│  ;     return n * factorial(n-1);                                   │
│  ; }                                                                │
│                                                                     │
│  factorial:                                                         │
│      CMP R0, #1           ; n <= 1?                                 │
│      MOVLE R0, #1         ; return 1                                │
│      BXLE LR              ; 조건부 복귀                              │
│      PUSH {R4, LR}        ; 레지스터 저장                            │
│      MOV R4, R0           ; n 저장                                   │
│      SUB R0, R0, #1       ; n - 1                                    │
│      BL factorial         ; 재귀 호출                                │
│      MUL R0, R4, R0       ; n * factorial(n-1)                       │
│      POP {R4, PC}         ; 복귀                                     │
│                                                                     │
│  【MIPS 어셈블리 분석】                                              │
│  ──────────────────────────                                         │
│  factorial:                                                         │
│      slti $t0, $a0, 2     # n < 2?                                  │
│      bne $t0, $zero, base # yes → base                              │
│      nop                  # 지연 슬롯                                │
│      addi $sp, $sp, -8    # 스택 할당                                │
│      sw $ra, 4($sp)       # 반환 주소 저장                           │
│      sw $a0, 0($sp)       # n 저장                                   │
│      addi $a0, $a0, -1    # n - 1                                    │
│      jal factorial        # 재귀 호출                                │
│      nop                  # 지연 슬롯                                │
│      lw $a0, 0($sp)       # n 복원                                   │
│      lw $ra, 4($sp)       # 반환 주소 복원                           │
│      addi $sp, $sp, 8     # 스택 복원                                │
│      jr $ra               # 복귀                                     │
│      mul $v0, $a0, $v0    # 지연 슬롯 활용                           │
│  base:                                                               │
│      addi $v0, $zero, 1   # return 1                                │
│      jr $ra                                                          │
│                                                                     │
│  【RISC-V 분석】                                                     │
│  ──────────────────────                                             │
│  # 최신 RISC 설계, 오픈소스                                          │
│  # 확장성: RV32I(기본) + M(곱셈) + C(압축) + ...                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RISC 성능 측정:                                                    │ │
│  │  ──────────────                                                    │ │
│  │  $ perf stat ./program                                            │ │
│  │  1,234,567,890 cycles       # 클럭                                 │ │
│  │  987,654,321 instructions   # 명령어                               │ │
│  │  IPC: 0.80                  # 명령어/클럭                          │ │
│  │                                                               │ │
│  │  RISC는 IPC 1.0+ 달성 쉬움                                         │ │
│  │  CISC는 마이크로옵으로 IPC 달성                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• RISC: 단순한 명령어 집합, 빠른 하드웨어
• 핵심: Load/Store, 고정 길이, 3-주소, 많은 레지스터
• 장점: 높은 클럭, 파이프라인 효율, 에너지 효율
• 단점: 코드 크기 큼, 컴파일러 의존
• 대표: ARM, MIPS, RISC-V
• 활용: 모바일, 임베디드, 서버
```
