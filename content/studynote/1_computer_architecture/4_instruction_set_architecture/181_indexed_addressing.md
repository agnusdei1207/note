# 181. 인덱스 주소 지정 (Indexed Addressing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 베이스 주소 + 인덱스×스케일로 배열 요소에 접근
> 2. **가치**: 효율적인 배열/행렬 순회, 루프 최적화
> 3. **융합**: 컴파일러 최적화, 벡터화, 병렬 처리

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

인덱스 주소 지정(Indexed Addressing)은 **베이스 주소에 인덱스 레지스터 값을 더해(스케일 적용 가능) 유효 주소를 계산**하는 방식이다. 배열 요소 접근에 최적화되어 있으며, 인덱스가 실행 중에 결정된다.

### 💡 비유: 아파트 층수

인덱스 주소 지정은 **"1층에서 5층 올라가기"**와 같다. 1층(베이스)에서 시작해서 층수(인덱스)×층높이(스케일)만큼 올라간다. 층수가 바뀌면 다른 집에 도착한다.

### 인덱스 주소 지정 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                인덱스 주소 지정 구조                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조】                                                           │
│  ─────────                                                          │
│                                                                     │
│  EA = [Base] + [Index] × Scale (+ Displacement)                     │
│  유효 주소 = 베이스 + 인덱스 × 스케일 (+ 변위)                       │
│                                                                     │
│  【동작 원리】                                                       │
│  ─────────────                                                      │
│                                                                     │
│  Base:    0x1000   (배열 시작 주소)                                 │
│  Index:   3        (i = 3)                                         │
│  Scale:   4        (sizeof(int) = 4)                                │
│                                                                     │
│  EA = 0x1000 + 3 × 4 = 0x100C  (arr[3]의 주소)                      │
│                                                                     │
│  【메모리 레이아웃】                                                 │
│  ──────────────────                                                 │
│                                                                     │
│        Base=0x1000                                                  │
│           │                                                         │
│           ▼                                                         │
│  ┌────────────────────────────────────────────┐                     │
│  │ arr[0] │ arr[1] │ arr[2] │ arr[3] │ arr[4] │                     │
│  │ 0x1000 │ 0x1004 │ 0x1008 │ 0x100C │ 0x1010 │                     │
│  └────────────────────────────────────────────┘                     │
│          │       │       │       │                                  │
│         i=0    i=1    i=2    i=3   ← Index×Scale                   │
│                                                                     │
│  【ISA별 문법】                                                     │
│  ──────────────                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ISA      문법                       예시                       │ │
│  │  ───────  ─────────────────────────  ────────────────          │ │
│  │  x86      [base+index*scale]         MOV EAX, [RBX+RCX*4]      │ │
│  │  x86      [base+index*scale+disp]    MOV EAX, [RBX+RCX*4+8]    │ │
│  │  ARM      [Rn, Rm, LSL #n]           LDR R0, [R1, R2, LSL#2]   │ │
│  │  MIPS     (rs) + rt*scale            (간접적으로)               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스케일 팩터】                                                     │
│  ──────────────                                                     │
│  x86: 1, 2, 4, 8                                                    │
│  ARM: LSL #0~3 (1, 2, 4, 8)                                         │
│  목적: 요소 크기에 맞춤 (byte, half, word, dword)                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  인덱스 주소 지정 장점:                                           │ │
│  │  ────────────────────────────                                     │ │
│  │  • 루프에서 인덱스만 변경하면 다음 요소                           │ │
│  │  • 별도 주소 계산 명령어 불필요                                   │ │
│  │  • 하드웨어가 자동으로 EA 계산                                    │ │
│  │  • 2차원 배열도 가능 (base + row*col_size + col)                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 인덱스 주소 지정 활용

### 활용 패턴

```
┌─────────────────────────────────────────────────────────────────────┐
│                인덱스 주소 지정 활용                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 1차원 배열 순회】                                              │
│  ──────────────────────                                             │
│  // C 코드                                                           │
│  int arr[100];                                                      │
│  for (int i = 0; i < 100; i++)                                      │
│      sum += arr[i];                                                 │
│                                                                     │
│  // x86-64                                                          │
│  xor eax, eax              ; sum = 0                                │
│  xor rcx, rcx              ; i = 0                                  │
│  loop:                                                              │
│  add eax, [rbx + rcx*4]    ; sum += arr[i]                          │
│  inc rcx                   ; i++                                    │
│  cmp rcx, 100                                                       │
│  jl loop                                                            │
│                                                                     │
│  【2. 구조체 배열】                                                  │
│  ──────────────────                                                  │
│  struct Point { int x, y; };   // 크기 = 8바이트                    │
│  struct Point points[10];                                           │
│                                                                     │
│  // points[i].y 접근                                                │
│  MOV EAX, [RBX + RCX*8 + 4]  ; base + i*8 + offset_of_y             │
│                                                                     │
│  【3. 2차원 배열】                                                   │
│  ──────────────────                                                  │
│  int matrix[4][5];    // row-major                                  │
│  // matrix[i][j] = base + i*20 + j*4                                │
│                                                                     │
│  // x86-64 (i=RCX, j=RDX)                                           │
│  MOV R8, RCX                                                        │
│  IMUL R8, 20              ; i * (5*4)                               │
│  MOV EAX, [RBX + R8 + RDX*4]  ; base + row_offset + j*4             │
│                                                                     │
│  【4. ARM 인덱스 주소 지정】                                         │
│  ──────────────────────────                                         │
│  ; LSL(Logical Shift Left)로 스케일                                 │
│  LDR R0, [R1, R2, LSL#2]  ; R0 = Mem[R1 + R2*4]                     │
│  STR R3, [R4, R5, LSL#3]  ; Mem[R4 + R5*8] = R3                     │
│                                                                     │
│  【5. 문자열/버퍼 처리】                                             │
│  ──────────────────────                                             │
│  char str[100];                                                     │
│  // str[i] 접근                                                     │
│  MOV AL, [RSI + RCX]       ; scale=1 (byte)                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  x86 SIB (Scale-Index-Base) 바이트:                              │ │
│  │  ─────────────────────────────────                                │ │
│  │  ┌─────┬─────────────┬─────────────┐                             │ │
│  │  │Scale│   Index     │    Base     │                             │ │
│  │  │ 2bit│    3bit     │    3bit     │                             │ │
│  │  └─────┴─────────────┴─────────────┘                             │ │
│  │                                                               │ │
│  │  Scale: 00=1, 01=2, 10=4, 11=8                                  │ │
│  │  Index: 레지스터 번호 (SP 제외)                                   │ │
│  │  Base: 레지스터 번호                                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 인덱스 주소 지정 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                인덱스 주소 지정 × 타 과목 융합                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                        │
│  • 배열 인덱싱: arr[i] → [base + i*scale]                           │
│  • 루프 최적화: 인덱스 변수 활용                                     │
│  • 강도 절감: 곱셈 → 시프트                                          │
│  • 벡터화: SIMD 인덱스                                               │
│                                                                     │
│  【자료구조】                                                        │
│  • 배열: 모든 요소 접근                                              │
│  • 행렬: 2차원 배열                                                  │
│  • 텐서: 다차원 배열                                                 │
│  • 해시 테이블: 버킷 인덱싱                                          │
│                                                                     │
│  【알고리즘】                                                        │
│  • 정렬: 비교/교환                                                   │
│  • 검색: 이진 검색 인덱스                                            │
│  • 그래프: 인접 행렬                                                 │
│  • DP: 메모이제이션 테이블                                          │
│                                                                     │
│  【병렬 처리】                                                       │
│  • OpenMP: 인덱스 분할                                               │
│  • SIMD: 벡터 인덱싱                                                 │
│  • GPU: 스레드 인덱스                                                │
│  • 분산: 파티션 인덱스                                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  컴파일러 최적화 예시:                                            │ │
│  │  ────────────────────────────                                     │ │
│  │  // Before: arr[i]                                                │ │
│  │  MOV EAX, [RBX + RCX*4]    ; 인덱스 주소 지정                     │ │
│  │                                                               │ │
│  │  // After strength reduction:                                     │ │
│  │  ; 포인터 증가로 변경                                             │ │
│  │  ADD RDX, 4                 ; ptr += 4                           │ │
│  │  MOV EAX, [RDX]             ; ptr 사용                           │ │
│  │                                                               │ │
│  │  → 곱셈 없이 더 빠름                                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 인덱스 주소 지정 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                인덱스 주소 지정 분석                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【배열 합계 함수】                                                  │
│  ──────────────────                                                  │
│  // C: int sum_array(int *arr, int n);                              │
│  sum_array:                                                         │
│      xor eax, eax            ; sum = 0                              │
│      xor rcx, rcx            ; i = 0                                │
│      test rsi, rsi           ; n == 0?                              │
│      je .done                                                       │
│  .loop:                                                             │
│      add eax, [rdi + rcx*4]  ; sum += arr[i]                        │
│      inc rcx                 ; i++                                  │
│      cmp rcx, rsi            ; i < n?                               │
│      jl .loop                                                       │
│  .done:                                                             │
│      ret                                                           │
│                                                                     │
│  【objdump 분석】                                                    │
│  ──────────────────                                                  │
│  $ objdump -d program                                               │
│  add eax, DWORD PTR [rdi+rcx*4]    ; 인덱스 주소 지정               │
│  mov eax, DWORD PTR [rbx+rax*8+4]  ; 인덱스+변위                    │
│                                                                     │
│  【ARM 어셈블리】                                                    │
│  ──────────────────                                                  │
│  ; arr[i] = arr + i*4                                               │
│  LDR R0, [R1, R2, LSL#2]    ; R0 = arr[i]                           │
│                                                                     │
│  ; 2차원: matrix[row][col]                                          │
│  ; row=R2, col=R3, cols=10                                          │
│  MOV R4, R2, LSL#5          ; row * 32 (10*4 + 패딩)                │
│  ADD R4, R4, R3, LSL#2      ; + col * 4                             │
│  LDR R0, [R1, R4]           ; matrix[row][col]                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  디버깅 팁:                                                        │ │
│  │  ──────────                                                       │ │
│  │  (gdb) p/x $rcx                   ; 인덱스 값                    │ │
│  │  (gdb) x/x $rbx+$rcx*4            ; arr[i] 값                   │ │
│  │  (gdb) watch *(int*)($rbx+$rcx*4) ; arr[i] 변경 감시            │ │
│  │                                                               │ │
│  │  배열 범위 체크:                                                   │ │
│  │  cmp rcx, rsi              ; i < n?                             │ │
│  │  jae out_of_bounds         ; 아니면 에러                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 인덱스 주소 지정: EA = Base + Index×Scale
• 스케일: 1, 2, 4, 8 (요소 크기)
• 활용: 배열 순회, 2차원 배열, 구조체 배열
• 장점: 인덱스만 변경하면 다음 요소
• x86: SIB 바이트로 인코딩
```
