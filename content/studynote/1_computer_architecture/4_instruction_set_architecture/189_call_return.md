# 189. 서브루틴 호출 (Call) 및 복귀 (Return)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 함수를 호출하고 반환하는 메커니즘 (PC 저장/복원)
> 2. **가치**: 코드 재사용, 모듈화, 재귀의 기반
> 3. **융합**: 스택 프레임, 호출 규약, ABI

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

서브루틴 호출(Call)과 복귀(Return)는 **함수 진입 시 반환 주소를 저장하고, 함수 종료 시 저장된 주소로 돌아가는** 메커니즘이다. CALL 명령어는 반환 주소를 스택에 push하고 분기하며, RET 명령어는 스택에서 pop하여 복귀한다.

### 💡 비유: 책갈피를 끼우고 다른 책 읽기

CALL/RET은 **책을 읽다가 책갈피를 끼우고 다른 책을 읽는 것**과 같다. CALL은 "여기까지 읽었어"라고 책갈피(반환 주소)를 끼우고 다른 책(함수)으로 이동한다. RET은 그 책을 다 읽으면 책갈피 위치로 돌아온다.

### CALL/RET 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                CALL/RET 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동작 원리】                                                       │
│  ─────────────                                                      │
│                                                                     │
│  CALL:                                                              │
│  1. PUSH next_instruction_addr  ; 반환 주소 저장                    │
│  2. JMP  target                 ; 함수로 분기                       │
│                                                                     │
│  RET:                                                               │
│  1. POP PC                      ; 반환 주소 복원                    │
│  2. (PC = saved_addr)           ; 이전 위치로 복귀                   │
│                                                                     │
│  【실행 흐름】                                                       │
│  ─────────────                                                      │
│                                                                     │
│  메모리:                                                            │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │  main:                                                        │  │
│  │    1000:  CALL func      ; PUSH 1005, JMP 2000                │  │
│  │    1005:  ...            ; RET 후 여기로                       │  │
│  │                                                               │  │
│  │  func:                                                         │  │
│  │    2000:  ...             ; 함수 본문                          │  │
│  │    2004:  RET             ; POP → PC=1005                     │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  스택 (CALL 직후):                                                  │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │  SP →  |  1005 (반환 주소)  |                                 │  │
│  │        |  이전 프레임...    |                                 │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  【ISA별 문법】                                                     │
│  ──────────────                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ISA      호출            복귀           링크 레지스터          │ │
│  │  ───────  ──────────────  ────────────  ──────────────         │ │
│  │  x86      CALL label      RET           스택 (PUSH/POP)        │ │
│  │  ARM      BL label        BX LR         LR (R14)               │ │
│  │  MIPS     JAL target      JR $ra        $ra ($31)              │ │
│  │  RISC-V   JAL rd, imm     JALR rd, rs   rd (보통 ra=x1)        │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ARM 링크 레지스터 (LR):                                          │ │
│  │  ────────────────────────────                                     │ │
│  │  • BL: LR = PC+4, PC = target                                    │ │
│  │  • BX LR: PC = LR                                                │ │
│  │  • 장점: 스택 접근 없이 빠름                                       │ │
│  │  • 단점: 중첩 호출 시 LR 저장 필요                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. CALL/RET 상세

### 호출 메커니즘

```
┌─────────────────────────────────────────────────────────────────────┐
│                CALL/RET 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【x86-64 CALL/RET】                                                 │
│  ──────────────────────                                             │
│  CALL func                                                          │
│  ; == PUSH next_addr + JMP func                                     │
│  ; SP -= 8                                                          │
│  ; [SP] = next_addr                                                 │
│  ; PC = func                                                        │
│                                                                     │
│  RET                                                                │
│  ; == POP PC                                                        │
│  ; PC = [SP]                                                        │
│  ; SP += 8                                                          │
│                                                                     │
│  RET n (stdcall)                                                    │
│  ; PC = [SP]                                                        │
│  ; SP += 8 + n        ; 인자도 정리                                  │
│                                                                     │
│  【ARM BL/BX】                                                       │
│  ──────────────────                                                  │
│  BL func              ; LR = PC+4, PC = func                        │
│  ; ... 함수 본문                                                     │
│  BX LR                ; PC = LR                                      │
│                                                                     │
│  ; 중첩 호출 시 LR 저장                                              │
│  PUSH {LR}            ; 이전 LR 저장                                 │
│  BL nested_func       ; 새 LR 설정                                   │
│  ; ...                                                              │
│  POP {PC}             ; LR 복원 및 복귀                              │
│                                                                     │
│  【MIPS JAL/JR】                                                     │
│  ──────────────────                                                  │
│  JAL func             ; $ra = PC+8, PC = func                       │
│  ; ... 함수 본문                                                     │
│  JR $ra               ; PC = $ra                                     │
│                                                                     │
│  【간접 호출】                                                       │
│  ──────────────────                                                  │
│  ; 함수 포인터 호출                                                   │
│  CALL [func_ptr]      ; x86                                          │
│  CALL EAX             ; 레지스터 간접                                 │
│  BLX R0               ; ARM                                          │
│  JALR $ra, $t0        ; MIPS                                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스택 프레임 구조 (함수 진입 후):                                   │ │
│  │  ────────────────────────────────                                │ │
│  │  고주소 │  ...                                                    │ │
│  │         │  arg3 (있으면)                                          │ │
│  │         │  arg2                                                   │ │
│  │         │  arg1                                                   │ │
│  │         ├─────────────┬──────────────────────────────            │ │
│  │         │  반환 주소   │ ← CALL이 PUSH                            │ │
│  │         ├─────────────┼──────────────────────────────            │ │
│  │         │  saved RBP  │ ← PUSH RBP                                │ │
│  │         ├─────────────┼──────────────────────────────            │ │
│  │         │  local1     │                                           │ │
│  │         │  local2     │ ← RBP - N                                 │ │
│  │         │  ...        │                                           │ │
│  │  저주소 ├─────────────┴──────────────────────────────            │ │
│  │         │  (빈 공간)   │ ← RSP                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. CALL/RET × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                CALL/RET × 타 과목 융합                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                        │
│  • 함수 호출 → CALL                                                  │
│  • 인자 전달: 레지스터 + 스택                                         │
│  • 반환값: RAX/EAX                                                   │
│  • 호출 규약: cdecl, stdcall, fastcall, System V                    │
│                                                                     │
│  【운영체제】                                                        │
│  • 시스템 콜: SYSCALL/SVC (특수 CALL)                               │
│  • 인터럽트: INT + IRET (특수 CALL/RET)                              │
│  • 컨텍스트 스위치: CALL로 스케줄러 진입                             │
│  • 예외 처리: 예외 핸들러 호출                                       │
│                                                                     │
│  【프로그래밍 언어】                                                 │
│  • 함수: CALL/RET                                                    │
│  • 재귀: 자신을 CALL                                                 │
│  • 메서드: this 포인터 + CALL                                        │
│  • 클로저: 환경 + CALL                                               │
│                                                                     │
│  【컴퓨터 구조】                                                     │
│  • 반환 주소 예측: RAS (Return Address Stack)                        │
│  • 파이프라인: CALL/RET 비용                                         │
│  • 간접 호출: 예측 어려움                                             │
│  • 함수 인라이닝: CALL/RET 제거                                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  호출 규약 (Calling Convention):                                   │ │
│  │  ───────────────────────────────                                  │ │
│  │  x86-64 System V (Linux/macOS):                                   │ │
│  │  • 인자: RDI, RSI, RDX, RCX, R8, R9 (나머지 스택)                  │ │
│  │  • 반환: RAX                                                      │ │
│  │  • Caller-saved: RAX, RCX, RDX, R8-R11                           │ │
│  │  • Callee-saved: RBX, RBP, R12-R15                               │ │
│  │                                                               │ │
│  │  Windows x64:                                                     │ │
│  │  • 인자: RCX, RDX, R8, R9 (나머지 스택)                            │ │
│  │  • 반환: RAX                                                      │ │
│  │  • 섀도 공간: 32바이트 (호출자가 할당)                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### CALL/RET 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                CALL/RET 분석                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【함수 호출 분석】                                                  │
│  ──────────────────                                                  │
│  // int add(int a, int b) { return a + b; }                         │
│  // int result = add(10, 20);                                       │
│                                                                     │
│  ; caller                                                            │
│  MOV EDI, 10            ; arg1                                       │
│  MOV ESI, 20            ; arg2                                       │
│  CALL add               ; 호출                                       │
│  MOV [result], EAX      ; 반환값 사용                                │
│                                                                     │
│  ; callee                                                            │
│  add:                                                               │
│      MOV EAX, EDI       ; a                                          │
│      ADD EAX, ESI       ; a + b                                      │
│      RET                 ; 반환                                      │
│                                                                     │
│  【objdump 분석】                                                    │
│  ──────────────────                                                  │
│  $ objdump -d program                                               │
│  1000:  bf 0a 00 00 00    mov edi, 0xa                              │
│  1005:  be 14 00 00 00    mov esi, 0x14                             │
│  100a:  e8 11 00 00 00    call 1020 <add>                           │
│  100f:  89 05 00 00 00 00 mov [result], eax                         │
│  ...                                                                │
│  1020:  89 f8           mov eax, edi                                │
│  1022:  01 f0           add eax, esi                                │
│  1024:  c3             ret                                          │
│                                                                     │
│  【GDB 디버깅】                                                      │
│  ──────────────────                                                  │
│  (gdb) si                 ; CALL 실행                               │
│  (gdb) backtrace         ; 호출 스택 확인                            │
│  (gdb) p/x $rsp          ; 스택 포인터                               │
│  (gdb) x/x $rsp          ; 반환 주소 확인                            │
│  (gdb) finish             ; 함수 끝까지 실행 (RET까지)               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  꼬리 재귀 최적화 (Tail Call Optimization):                       │ │
│  │  ─────────────────────────────────────────                       │ │
│  │  // 재귀                                                           │ │
│  │  int fact(int n, int acc) {                                       │ │
│  │      if (n == 0) return acc;                                      │ │
│  │      return fact(n-1, n*acc);  // 꼬리 재귀                       │ │
│  │  }                                                                │ │
│  │                                                               │ │
│  │  ; 최적화 전: CALL fact                                           │ │
│  │  ; 최적화 후: JMP fact (새 프레임 없이)                            │ │
│  │                                                               │ │
│  │  장점:                                                             │ │
│  │  • 스택 사용 O(1)                                                 │ │
│  │  • 스택 오버플로우 방지                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• CALL: 반환 주소 저장 + 함수 분기
• RET: 반환 주소 복원 + 복귀
• x86: 스택 사용 (PUSH/POP)
• ARM: LR 레지스터 (R14)
• 호출 규약: 인자/반환값 전달 방식
• 최적화: 인라이닝, 꼬리 재귀
```
