# 172. 가변 길이 명령어 (Variable Length Instruction)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명령어마다 다른 비트 수를 갖는 ISA 설계 방식
> 2. **가치**: 높은 코드 밀도, 풍부한 명령어 표현, 메모리 효율
> 3. **융합**: CISC 철학, x86/x64, 임베디드 메모리 최적화

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

가변 길이 명령어(Variable Length Instruction)는 **명령어마다 다른 비트 수를 갖는 인코딩 방식**이다. 대표적으로 x86/x64(1~15바이트), VAX(1~37바이트)가 있으며, CISC 아키텍처의 핵심 특징이다.

### 💡 비유: 가변 길이 문자 인코딩

가변 길이 명령어는 **UTF-8 인코딩**과 같다. 영어는 1바이트, 한글은 3바이트로 각 문자마다 다른 길이를 갖는다. 자주 쓰는 것은 짧게, 드물게 쓰는 것은 길게 표현해 공간을 절약한다.

### 가변 길이 명령어 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                가변 길이 명령어 구조                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【특징】                                                           │
│  ─────────                                                          │
│  • 명령어마다 다른 길이 (1~15바이트 등)                              │
│  • 복잡한 연산을 한 명령어로 표현                                    │
│  • 높은 코드 밀도                                                    │
│  • 디코딩 복잡도 증가                                                │
│                                                                     │
│  【x86-64 명령어 형식】                                              │
│  ──────────────────────                                             │
│                                                                     │
│  ┌────────┬────────┬────────┬────────┬────────┬────────┬─────────┐ │
│  │ Prefix │ REX    │ Opcode │ ModR/M │  SIB   │ Disp   │ Imm     │ │
│  │ 0-4B   │ 0-1B   │ 1-3B   │ 0-1B   │ 0-1B   │ 0-4B   │ 0-4B    │ │
│  └────────┴────────┴────────┴────────┴────────┴────────┴─────────┘ │
│                                                                     │
│  • 최소: 1바이트 (예: RET)                                          │
│  • 최대: 15바이트 (복잡한 메모리 연산)                               │
│                                                                     │
│  【길이별 예시】                                                     │
│  ──────────────                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어            길이    바이트 인코딩      설명               │ │
│  │  ────────────────  ────    ───────────────  ──────────         │ │
│  │  ret               1B      C3               리턴               │ │
│  │  nop               1B      90               아무것도 안함       │ │
│  │  inc eax           1B      40               eax++              │ │
│  │  push rbx          1B      53               스택에 push        │ │
│  │                                                               │ │
│  │  add al, 5         2B      04 05            al += 5            │ │
│  │  mov eax, 0        2B      31 C0            eax = 0            │ │
│  │                                                               │ │
│  │  add eax, 100      3B      83 C0 64         eax += 100         │ │
│  │  mov eax, ebx      2B      89 D8            eax = ebx          │ │
│  │                                                               │ │
│  │  mov eax, [rbx]    2B      8B 03            eax = *ebx         │ │
│  │  mov eax, [rbx+8]  3B      8B 43 08         eax = *(ebx+8)     │ │
│  │  mov eax, [rbx+rcx]3B      8B 04 0B         eax = *(ebx+ecx)   │ │
│  │                                                               │ │
│  │  mov rax, 0x123456 7B      48 B8 56 34 12   rax = 0x123456     │ │
│  │                            00 00 00 00                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【x86 Prefix 종류】                                                │
│  ────────────────────                                               │
│  • 0x66: Operand size override (16/32비트 전환)                     │
│  • 0x67: Address size override                                      │
│  • 0xF0: LOCK (원자적 연산)                                         │
│  • 0xF2/F3: REP/REPE/REPNE (문자열 반복)                            │
│  • REX (64비트): 0x40~0x4F (64비트 레지스터, 확장)                   │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 가변 길이 명령어 장단점

### 장단점 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                가변 길이 명령어 장단점                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                           │
│  ─────────                                                          │
│                                                                     │
│  1. 높은 코드 밀도                                                  │
│     • 자주 쓰는 명령어는 1~2바이트                                  │
│     • 메모리/캐시 효율                                               │
│     • 작은 바이너리 크기                                             │
│                                                                     │
│  2. 풍부한 명령어 표현                                              │
│     • 복잡한 연산을 한 명령어로                                      │
│     • 다양한 주소 지정 모드                                          │
│     • 강력한 메모리 연산                                             │
│                                                                     │
│  3. 하위 호환성                                                     │
│     • 명령어 추가가 쉬움                                             │
│     • 확장 Prefix 활용                                               │
│     • 오랜 기간 ISA 유지                                             │
│                                                                     │
│  4. 메모리 절약                                                     │
│     • 임베디드 환경에 유리                                           │
│     • 명령어 캐시 히트율                                             │
│                                                                     │
│  【단점】                                                           │
│  ─────────                                                          │
│                                                                     │
│  1. 복잡한 디코딩                                                   │
│     • 명령어 길이 먼저 파악 필요                                     │
│     • 여러 단계 디코딩                                               │
│     • 디코더 하드웨어 복잡                                           │
│                                                                     │
│  2. 파이프라인 비효율                                                │
│     • 명령어 경계 탐색                                               │
│     • 버블 발생 가능                                                 │
│     • 슈퍼스칼라 구현 난이도                                         │
│                                                                     │
│  3. 분기 대상 정렬                                                  │
│     • 점프 대상 계산 복잡                                            │
│     • 정렬 이슈                                                      │
│                                                                     │
│  4. 디버깅 난이도                                                   │
│     • 역어셈블리 복잡                                                │
│     • 명령어 경계 오류                                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  x86 디코딩 단계:                                                │ │
│  │  ──────────────────                                              │ │
│  │  1. Legacy Prefix 확인 (0-4개)                                   │ │
│  │  2. REX Prefix 확인 (64비트)                                     │ │
│  │  3. Opcode 확인 (1-3바이트)                                      │ │
│  │  4. ModR/M 바이트 확인 (있으면)                                  │ │
│  │  5. SIB 바이트 확인 (있으면)                                     │ │
│  │  6. Displacement 확인 (0-4바이트)                                │ │
│  │  7. Immediate 확인 (0-4바이트)                                   │ │
│  │                                                               │ │
│  │  → 최대 15바이트까지 순차 파싱 필요                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 가변 길이 명령어 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                가변 길이 명령어 분석                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【objdump로 길이 분석】                                            │
│  ──────────────────────────                                        │
│  $ objdump -d program | head -20                                   │
│                                                                     │
│  1000:  55                push   rbp              # 1바이트         │
│  1001:  48 89 e5          mov    rbp,rsp          # 3바이트         │
│  1004:  89 7d fc          mov    DWORD PTR [rbp-0x4],edi  # 3B     │
│  1007:  8b 45 fc          mov    eax,DWORD PTR [rbp-0x4]   # 3B     │
│  100a:  83 c0 0a          add    eax,0xa           # 3바이트        │
│  100d:  5d                pop    rbp               # 1바이트        │
│  100e:  c3                ret                      # 1바이트        │
│                                                                     │
│  【코드 크기 비교】                                                  │
│  ──────────────────                                                  │
│  // C 함수                                                           │
│  int add10(int x) { return x + 10; }                               │
│                                                                     │
│  x86-64:  9바이트 (위 예시)                                          │
│  ARM:    12바이트 (3 × 4바이트 고정)                                 │
│  Thumb:   8바이트 (4 × 2바이트)                                      │
│                                                                     │
│  【x86 인코딩 수동 분석】                                            │
│  ──────────────────────────                                         │
│  명령어: add eax, 0xa                                               │
│  바이트: 83 C0 0A                                                    │
│                                                                     │
│  83 = opcode (add r/m32, imm8)                                     │
│  C0 = ModR/M: Mod=11(레지스터), Reg=0(add), R/M=0(eax)             │
│  0A = immediate (10)                                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  가변 길이 명령어 최적화 팁:                                      │ │
│  │  ────────────────────────────                                     │ │
│  │  1. 짧은 인코딩 선호                                              │ │
│  │     xor eax, eax  (2B) > mov eax, 0 (5B)                         │ │
│  │     inc eax (1B, 32비트) > add eax, 1 (3B)                       │ │
│  │                                                               │ │
│  │  2. 레지스터 활용                                                 │ │
│  │     rax-rbx (1B 인코딩) > r8-r15 (2B REX prefix)                 │ │
│  │                                                               │ │
│  │  3. LEA 활용                                                     │ │
│  │     lea eax, [rbx+rcx*4] (복잡한 주소 계산을 1명령어로)          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 가변 길이 명령어: 명령어마다 다른 비트 수
• 대표 ISA: x86/x64 (1~15바이트)
• 장점: 높은 코드 밀도, 풍부한 표현, 하위 호환
• 단점: 복잡한 디코딩, 파이프라인 비효율
• 해결책: uOp 캐시, 디코더 캐시, Macro-op Fusion
```
