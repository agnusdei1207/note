# 160. 피연산자 (Operand)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명령어가 연산을 수행할 대상, 레지스터, 메모리, 상수 가능
> 2. **가치**: 명령어의 유연성과 표현력 결정, 주소 지정 방식과 밀접
> 3. **융합**: 메모리 계층, 레지스터 할당, 컴파일러 최적화의 핵심

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

피연산자(Operand)는 **명령어가 연산을 수행하는 대상**으로, 데이터 값 자체 또는 데이터의 위치(주소)를 지정한다. 명령어는 "무엇을(Opcode)" 할 것인가와 "무엇으로(Operand)" 할 것인가로 구성된다.

### 💡 비유: 요리 재료

피연산자는 **요리 재료**와 같다. "볶아라"(Opcode)라는 지시가 있어도 무엇을 볶을지 재료(Operand)가 있어야 요리가 완성된다. 양파, 당근, 고기 등 다양한 재료가 피연산자다.

### 피연산자 유형

```
┌─────────────────────────────────────────────────────────────────────┐
│                피연산자 유형                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 즉시 피연산자 (Immediate Operand)】                           │
│  ────────────────────────────────────                               │
│                                                                     │
│  • 명령어 내에 데이터 값 직접 포함                                   │
│  • 별도의 메모리/레지스터 접근 없음                                  │
│  • 빠름, but 크기 제한                                               │
│                                                                     │
│  예: MOV R1, #100     ; R1 ← 100 (100이 즉시 피연산자)              │
│      ADD R1, R2, #5   ; R1 ← R2 + 5                                 │
│                                                                     │
│  【2. 레지스터 피연산자 (Register Operand)】                         │
│  ────────────────────────────────────────                           │
│                                                                     │
│  • CPU 내부 레지스터 지정                                           │
│  • 가장 빠른 접근                                                   │
│  • 개수 제한 (16~32개)                                              │
│                                                                     │
│  예: ADD R1, R2, R3   ; R1 ← R2 + R3 (R1, R2, R3가 피연산자)        │
│      MOV R1, R2       ; R1 ← R2                                     │
│                                                                     │
│  【3. 메모리 피연산자 (Memory Operand)】                             │
│  ──────────────────────────────────────                             │
│                                                                     │
│  • 메모리 주소로 데이터 지정                                         │
│  • 레지스터보다 느림                                                │
│  • 큰 데이터 접근 가능                                              │
│                                                                     │
│  예: LOAD R1, [1000]   ; R1 ← Mem[1000]                             │
│      STORE [R2], R1    ; Mem[R2] ← R1                               │
│      ADD R1, [R2]      ; R1 ← R1 + Mem[R2]                          │
│                                                                     │
│  【4. 주소 피연산자 (Address Operand)】                              │
│  ────────────────────────────────────                               │
│                                                                     │
│  • 분기/점프 명령어의 목적지 주소                                   │
│  • 상대 주소 또는 절대 주소                                         │
│                                                                     │
│  예: JMP 1000         ; PC ← 1000                                   │
│      CALL function    ; PC ← function 주소                          │
│      BEQ label        ; if (Z=1) PC ← label                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  접근 속도 비교:                                                 │ │
│  │  ────────────────                                               │ │
│  │  피연산자 타입       접근 시간       상대 속도                   │ │
│  │  ─────────────────  ─────────────   ──────────                  │ │
│  │  즉시 (Immediate)    0 사이클        1× (가장 빠름)             │ │
│  │  레지스터            0 사이클        1×                          │ │
│  │  L1 캐시             1~4 사이클      1~4×                        │ │
│  │  L2/L3 캐시          10~40 사이클    10~40×                      │ │
│  │  메인 메모리         100~300 사이클  100~300×                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 피연산자의 명령어 내 표현

```
┌─────────────────────────────────────────────────────────────────────┐
│                피연산자의 명령어 내 표현                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【MIPS R-type (3 레지스터 피연산자)】                               │
│  ──────────────────────────────────────                             │
│  ┌─────────┬─────────┬─────────┬─────────┬─────────┬─────────┐    │
│  │  opcode │   rs    │   rt    │   rd    │  shamt  │  funct  │    │
│  │  6 bits │ 5 bits  │ 5 bits  │ 5 bits  │ 5 bits  │ 6 bits  │    │
│  └─────────┴─────────┴─────────┴─────────┴─────────┴─────────┘    │
│              │         │         │                                  │
│              ▼         ▼         ▼                                  │
│           소스1     소스2     목적지                                 │
│                                                                     │
│  예: ADD $t0, $t1, $t2  →  rs=$t1, rt=$t2, rd=$t0                   │
│                                                                     │
│  【MIPS I-type (레지스터 + 즉시값)】                                 │
│  ────────────────────────────────────                               │
│  ┌─────────┬─────────┬─────────┬──────────────────────────────┐    │
│  │  opcode │   rs    │   rt    │         immediate            │    │
│  │  6 bits │ 5 bits  │ 5 bits  │          16 bits             │    │
│  └─────────┴─────────┴─────────┴──────────────────────────────┘    │
│              │         │              │                             │
│              ▼         ▼              ▼                             │
│           소스      목적지         상수값                            │
│                                                                     │
│  예: ADDI $t0, $t1, 100  →  rs=$t1, rt=$t0, imm=100                │
│                                                                     │
│  【x86 ModR/M 바이트】                                               │
│  ────────────────────────                                           │
│  ┌─────────┬─────────┬─────────┐                                   │
│  │   Mod   │  Reg/   │   R/M   │                                   │
│  │  2 bits │ Opcode  │ 3 bits  │                                   │
│  │         │ 3 bits  │         │                                   │
│  └─────────┴─────────┴─────────┘                                   │
│                                                                     │
│  Mod 값:                                                            │
│  • 00: 레지스터 간접 (R/M)                                          │
│  • 01: R/M + disp8                                                  │
│  • 10: R/M + disp32                                                 │
│  • 11: 레지스터 직접 (R/M)                                          │
│                                                                     │
│  예: MOV EAX, EBX     →  Mod=11, Reg=EAX, R/M=EBX                   │
│      MOV EAX, [EBX]    →  Mod=00, Reg=EAX, R/M=EBX                  │
│      MOV EAX, [EBX+4]  →  Mod=01, Reg=EAX, R/M=EBX, disp8=4         │
│                                                                     │
│  【ARM 데이터 처리】                                                 │
│  ────────────────────────                                           │
│  ┌───────────┬──────┬──────┬──────┬──────┬──────────────────────┐  │
│  │ Cond      │ Op   │ Rn   │ Rd   │ Sh   │ Operand2            │  │
│  │ 4 bits    │      │      │      │      │ 12 bits             │  │
│  └───────────┴──────┴──────┴──────┴──────┴──────────────────────┘  │
│                                                                     │
│  Operand2:                                                          │
│  • 8비트 즉시값 + 4비트 회전                                        │
│  • 레지스터 + 시프트                                                │
│                                                                     │
│  예: ADD R0, R1, #100      ; R0 ← R1 + 100                         │
│      ADD R0, R1, R2, LSL#3  ; R0 ← R1 + (R2 << 3)                  │
└─────────────────────────────────────────────────────────────────────┘
```

### 피연산자 수에 따른 명령어 분류

```
┌─────────────────────────────────────────────────────────────────────┐
│                피연산자 수에 따른 명령어 분류                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【0-주소 명령어 (0-address)】                                       │
│  ────────────────────────────                                       │
│  • 스택 머신에서 사용                                                │
│  • 피연산자가 스택 top에 암시적                                      │
│                                                                     │
│  예: ADD    ; TOS-1 ← TOS-1 + TOS; TOS ← TOS-1                      │
│      PUSH 100  ; 스택에 100 푸시                                    │
│      POP      ; 스택에서 팝                                          │
│                                                                     │
│  장점: 명령어 크기 작음                                              │
│  단점: 메모리 접근 많음, 성능 낮음                                   │
│  사용: Java VM (일부), Forth                                        │
│                                                                     │
│  【1-주소 명령어 (1-address)】                                       │
│  ────────────────────────────                                       │
│  • 누산기(Accumulator) 머신에서 사용                                 │
│  • 한 피연산자는 누산기에 암시적                                     │
│                                                                     │
│  예: LOAD 1000  ; ACC ← Mem[1000]                                   │
│      ADD 1000   ; ACC ← ACC + Mem[1000]                             │
│      STORE 1000 ; Mem[1000] ← ACC                                   │
│                                                                     │
│  장점: 명령어 길이 짧음                                              │
│  단점: 누산기 병목                                                   │
│  사용: 초기 컴퓨터, 일부 마이크로컨트롤러                           │
│                                                                     │
│  【2-주소 명령어 (2-address)】                                       │
│  ────────────────────────────                                       │
│  • x86 등에서 사용                                                   │
│  • 목적지가 소스 중 하나를 덮어씀                                    │
│                                                                     │
│  예: MOV R1, R2    ; R1 ← R2                                        │
│      ADD R1, R2    ; R1 ← R1 + R2                                   │
│      ADD R1, [R2]  ; R1 ← R1 + Mem[R2]                              │
│                                                                     │
│  장점: 명령어 길이 적당                                              │
│  단점: 원본 데이터 손실                                              │
│  사용: x86, 일부 RISC                                               │
│                                                                     │
│  【3-주소 명령어 (3-address)】                                       │
│  ────────────────────────────                                       │
│  • RISC(RISC-V, ARM, MIPS)에서 사용                                 │
│  • 목적지와 두 소스가 모두 명시적                                    │
│                                                                     │
│  예: ADD R1, R2, R3   ; R1 ← R2 + R3                                │
│      SUB R1, R2, R3   ; R1 ← R2 - R3                                │
│      MUL R1, R2, R3   ; R1 ← R2 × R3                                │
│                                                                     │
│  장점: 데이터 보존, 유연성 높음                                      │
│  단점: 명령어 길이 김                                                │
│  사용: 대부분의 RISC                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  비교 요약:                                                      │ │
│  │  ──────────────                                                  │ │
│  │  타입        명령어 길이   표현력   사용 예                      │ │
│  │  ─────────   ──────────   ──────   ──────────                   │ │
│  │  0-주소      가장 짧음     낮음     스택 머신                    │ │
│  │  1-주소      짧음         낮음     누산기 머신                  │ │
│  │  2-주소      중간         중간     x86                          │ │
│  │  3-주소      김           높음     RISC (MIPS, ARM, RISC-V)    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 융합 비교 및 다각도 분석

### 피연산자 타입별 특성 비교

| 타입 | 크기 | 속도 | 유연성 | 용도 |
|:---|:---|:---|:---|:---|
| **즉시** | 제한 | 빠름 | 낮음 | 상수 |
| **레지스터** | 32/64비트 | 빠름 | 중간 | 임시 데이터 |
| **메모리** | 무제한 | 느림 | 높음 | 큰 데이터 |

```
│  │    MUL R2, R2, R3                                              │ │
│  │    ADD R1, R1, R2                                              │ │
│  │    STORE [a], R1                                               │ │
│  │                                                               │ │
│  │  할당 후 (레지스터):                                            │ │
│  │    ; a,b,c,d 모두 레지스터에 있음                              │ │
│  │    MUL R2, R3, R4   ; R2=c, R3=c, R4=d                        │ │
│  │    ADD R1, R5, R2   ; R1=a, R5=b                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 레지스터 부족 문제**
- 상황: 함수 인자 10개, 레지스터 8개
- 해결: 나머지는 스택에 저장
- 결과: 성능 저하 감수

**시나리오 2: 큰 상수 사용**
- 상황: 32비트 상수, 16비트 즉시 필드
- 해결: 상수를 메모리에 두고 로드
- 결과: 2개 명령어 필요

**시나리오 3: 배열 접근 최적화**
- 상황: 루프 내 배열 접근
- 해결: 베이스 + 인덱스 주소 지정
- 결과: 인덱스 레지스터만 증가

**시나리오 4: SIMD 최적화**
- 상황: 4개 데이터 동시 처리
- 해결: 벡터 레지스터 피연산자
- 결과: 4배 처리량 향상

### 피연산자 최적화 체크리스트

```
┌─────────────────────────────────────────────────────────────────────┐
│                피연산자 최적화 체크리스트                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【레지스터 활용】                                                   │
│  ──────────────────                                                 │
│  □ 자주 쓰는 변수를 레지스터에                                      │
│  □ 레지스터 스풀링 최소화                                            │
│  □ 호출 규약 준수 (callee-saved)                                   │
│  □ 레지스터 윈도우 활용                                              │
│                                                                     │
│  【즉시값 활용】                                                     │
│  ──────────────────                                                 │
│  □ 작은 상수는 즉시 피연산자로                                      │
│  □ 큰 상수는 리터럴 풀                                               │
│  □ 0은 zero 레지스터                                                 │
│  □ 2의 거듭제곱은 시프트로                                          │
│                                                                     │
│  【메모리 접근】                                                     │
│  ──────────────────                                                 │
│  □ 캐시 라인 정렬                                                   │
│  □ 연속 접근 패턴                                                    │
│  □ 프리패치 활용                                                     │
│  □ 불필요한 저장/로드 제거                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  피연산자 최적화 예시:                                           │ │
│  │  ──────────────────────                                         │ │
│  │  Before:                                                        │ │
│  │    LOAD R1, [arr+i]      ; 매번 메모리 접근                     │ │
│  │    ADD R1, R1, #1                                               │ │
│  │    STORE [arr+i], R1                                            │ │
│  │                                                               │ │
│  │  After (레지스터 활용):                                         │ │
│  │    LOAD R1, [arr]        ; 베이스 로드                          │ │
│  │    loop:                                                       │ │
│  │    ADD R1, R1, #1        ; 레지스터만                           │ │
│  │    ...                                                         │ │
│  │    STORE [arr], R1       ; 마지막에만 저장                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 최적화 | 성능 향상 | 적용 난이도 |
|:---|:---:|:---:|
| **레지스터 할당** | 2~5× | 중간 |
| **즉시값 활용** | 1.2~1.5× | 낮음 |
| **메모리 정렬** | 1.5~3× | 낮음 |
| **SIMD** | 4~16× | 높음 |

### 핵심 요약

```
• 피연산자: 명령어가 연산할 대상
• 유형: 즉시, 레지스터, 메모리, 주소
• 주소 수: 0~3-주소 명령어
• 속도: 레지스터 > 캐시 > 메모리
• 최적화: 레지스터 할당, 즉시값, 정렬
```
