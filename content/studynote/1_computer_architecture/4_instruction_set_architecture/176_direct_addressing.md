# 176. 직접 주소 지정 (Direct Addressing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명령어에 메모리 주소를 직접 명시하는 방식
> 2. **가치**: 전역 변수, 정적 데이터 접근에 적합
> 3. **융합**: 절대 주소, PIC, 메모리 매핑 I/O

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

직접 주소 지정(Direct Addressing)은 **명령어 내에 메모리 주소를 직접 포함**하는 방식이다. 유효 주소(EA)가 명령어의 주소 필드와 동일하며, 별도의 주소 계산 없이 지정된 위치에 접근한다.

### 💡 비유: 정확한 사서함 번호

직접 주소 지정은 **"101동 202호"라고 정확히 적힌 주소**와 같다. 물어보거나 계산할 필요 없이 바로 그 위치로 간다. 단, 주소가 길어서(32/64비트) 명령어가 커진다.

### 직접 주소 지정 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                직접 주소 지정 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조】                                                           │
│  ─────────                                                          │
│                                                                     │
│  ┌─────────────┬─────────────────────────────────────┐              │
│  │   Opcode    │         Memory Address              │              │
│  │   연산코드   │           메모리 주소                │              │
│  └─────────────┴─────────────────────────────────────┘              │
│                                                                     │
│  EA = Address Field (유효 주소 = 주소 필드)                         │
│                                                                     │
│  【특징】                                                           │
│  ─────────                                                          │
│  • 주소 계산 없음                                                    │
│  • 절대 주소 사용                                                    │
│  • 명령어 길이 길어짐 (주소 포함)                                    │
│  • 위치 독립 코드 작성 어려움                                        │
│                                                                     │
│  【ISA별 직접 주소 지정】                                            │
│  ──────────────────────────                                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ISA      문법                 예시                            │ │
│  │  ───────  ───────────────────  ───────────────────            │ │
│  │  x86      MOV reg, [addr]      MOV EAX, [0x1000]               │ │
│  │  ARM      LDR Rd, =addr        LDR R0, =0x1000                 │ │
│  │  MIPS     lw rt, addr(rs)      lw $t0, 0x1000($zero)           │ │
│  │  RISC-V   lui + lw             lui + lw 조합                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주소 크기와 명령어 길이】                                         │
│  ────────────────────────────                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  아키텍처   주소 크기    명령어 내 주소    해결책                │ │
│  │  ────────  ──────────  ────────────────  ─────────             │ │
│  │  16비트    16비트       완전 포함        간단                    │ │
│  │  32비트    32비트       부분 (상수풀)    literal pool           │ │
│  │  64비트    64비트       여러 명령어      lui+ori+sllx           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  직접 vs 간접 비교:                                               │ │
│  │  ──────────────────────                                           │ │
│  │  직접: MOV EAX, [0x1000]    ; 0x1000번지의 값                    │ │
│  │  간접: MOV EAX, [EBX]       ; EBX가 가리키는 번지의 값           │ │
│  │                                                               │ │
│  │  직접: 주소가 고정됨                                              │ │
│  │  간접: 실행 중 주소 변경 가능                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 직접 주소 지정 활용

### 활용 패턴

```
┌─────────────────────────────────────────────────────────────────────┐
│                직접 주소 지정 활용                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 전역 변수 접근】                                               │
│  ──────────────────────                                             │
│  int global_var;                   // C 코드                        │
│                                                                     │
│  MOV EAX, [global_var]             ; 전역 변수 읽기                 │
│  MOV [global_var], EBX             ; 전역 변수 쓰기                 │
│                                                                     │
│  【2. 정적 변수 접근】                                               │
│  ──────────────────────                                             │
│  static int counter;               // C 코드                        │
│                                                                     │
│  INC DWORD PTR [counter]           ; counter++                     │
│                                                                     │
│  【3. 메모리 매핑 I/O】                                              │
│  ──────────────────────                                             │
│  #define GPIO_BASE 0x40020000                                       │
│                                                                     │
│  LDR R0, =0x40020000               ; GPIO 베이스 주소               │
│  LDR R1, [R0]                      ; GPIO 레지스터 읽기             │
│  STR R2, [R0]                      ; GPIO 레지스터 쓰기             │
│                                                                     │
│  【4. 상수 풀 (Literal Pool)】                                       │
│  ────────────────────────────                                       │
│  ; ARM에서 큰 상수/주소                                              │
│  LDR R0, =0x12345678               ; 상수 풀에서 로드               │
│  ; 어셈블러가 자동으로 상수 풀 생성                                  │
│                                                                     │
│  【5. 절대 분기】                                                    │
│  ──────────────────                                                  │
│  JMP 0x8000                        ; 절대 주소로 점프               │
│  CALL function_address             ; 절대 주소 함수 호출            │
│                                                                     │
│  【직접 주소 지정 제한점】                                           │
│  ────────────────────────────                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  문제점:                                                          │ │
│  │  ────────                                                         │ │
│  │  1. 주소 고정 → 재배치 필요                                       │ │
│  │  2. ASLR과 충돌                                                   │ │
│  │  3. 위치 독립 코드 불가                                           │ │
│  │  4. 명령어 길이 증가                                              │ │
│  │                                                               │ │
│  │  현대적 대안:                                                      │ │
│  │  • PC 상대 주소 지정 (RIP-relative on x86-64)                    │ │
│  │  • GOT (Global Offset Table)                                     │ │
│  │  • PIC (Position Independent Code)                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 직접 주소 지정 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                직접 주소 지정 분석                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【어셈블리 분석】                                                   │
│  ──────────────────                                                  │
│  // C 코드                                                           │
│  int global_counter = 0;                                            │
│  void increment() { global_counter++; }                             │
│                                                                     │
│  // x86-64 (RIP-relative)                                           │
│  increment:                                                         │
│      add DWORD PTR [rip+0x2000], 1   ; 전역 변수 +1                 │
│      ret                                                            │
│                                                                     │
│  // ARM (literal pool)                                              │
│  increment:                                                         │
│      ldr r0, [pc, #offset]           ; 전역 변수 주소 로드          │
│      ldr r1, [r0]                    ; 값 로드                      │
│      add r1, r1, #1                  ; +1                           │
│      str r1, [r0]                    ; 값 저장                      │
│      bx lr                                                          │
│  .ltorg                               ; literal pool                │
│      .word global_counter            ; 주소 저장                    │
│                                                                     │
│  【objdump로 확인】                                                  │
│  ──────────────────                                                  │
│  $ objdump -d program                                               │
│  1000:  48 8b 05 00 20 00 00   mov rax,[rip+0x2000]                 │
│                               ; RIP-relative 직접 주소              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  직접 주소 지정 디버깅:                                           │ │
│  │  ────────────────────────────                                     │ │
│  │  (gdb) x/x 0x1000            ; 메모리 0x1000 값 확인             │ │
│  │  (gdb) x/10i $pc             ; 현재 위치에서 명령어 확인         │ │
│  │  (gdb) info symbol 0x1000    ; 주소의 심볼 확인                  │ │
│  │                                                               │ │
│  │  재배치 확인:                                                     │ │
│  │  $ readelf -r program        ; 재배치 엔트리 확인                │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 직접 주소 지정: 명령어에 메모리 주소 직접 포함
• EA = Address Field (유효 주소 = 주소 필드)
• 장점: 주소 계산 없음, 접근 간단
• 단점: 주소 고정, PIC 불가, 명령어 길어짐
• 현대적 대안: RIP-relative, GOT, PIC
```
