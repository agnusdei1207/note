# 167. 상태 레지스터 (Status Register / Flag Register)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연산 결과 상태를 저장하는 플래그 비트들의 집합, 조건부 실행의 기준
> 2. **가치**: 조건부 분기, 루프 제어, 예외 처리, OS 모드 관리
> 3. **융합**: 컴파일러 조건부 코드, OS 인터럽트, 보안 권한 레벨

## Ⅰ. 개요 (Context & Background)

### 개념 정의

상태 레지스터(Status Register) 또는 플래그 레지스터(Flag Register)는 **ALU 연산 결과의 상태를 저장하는 특수 레지스터**다. Zero, Carry, Negative, Overflow 등의 플래그 비트로 구성되며, 조건부 분기와 프로그램 흐름 제어의 핵심이다.

### 💡 비유: 시험 결과 표시등

상태 레지스터는 **시험 결과 표시등**과 같다. "100점인가?", "0점인가?", "틀린 문제가 있는가?" 같은 결과를 표시등(플래그)으로 보여준다. 다음에 무엇을 할지 이 표시등을 보고 결정한다.

### 플래그 비트 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                플래그 비트 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【x86 EFLAGS/RFLAGS (64비트)】                                      │
│  ──────────────────────────────────                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  31   28   24   20   16   12    8    4    0                    │ │
│  │  ┌────┬────┬────┬────┬────┬────┬────┬────┬────┐               │ │
│  │  │    │    │    │    │    │    │    │    │    │               │ │
│  │  └────┴────┴────┴────┴────┴────┴────┴────┴────┘               │ │
│  │    │     │     │     │    │                                    │ │
│  │    │     │     │     │    └─ CF (Carry Flag) - 비트 0           │ │
│  │    │     │     │     └────── PF (Parity Flag) - 비트 2          │ │
│  │    │     │     └──────────── AF (Aux Carry) - 비트 4            │ │
│  │    │     └────────────────── ZF (Zero Flag) - 비트 6            │ │
│  │    └──────────────────────── SF (Sign Flag) - 비트 7            │ │
│  │                       ┌──── OF (Overflow Flag) - 비트 11        │ │
│  │                       │    ┌─── IF (Interrupt Flag) - 비트 9    │ │
│  │                       ▼    ▼                                   │ │
│  │  주요 플래그:                                                    │ │
│  │  ────────────────                                               │ │
│  │  비트  이름   의미                   설정 조건                   │ │
│  │  ────  ────   ────────────────────    ────────────────           │ │
│  │  0     CF    캐리/보로우             결과 > 최댓값 or 결과 < 0   │ │
│  │  2     PF    패리티 (짝수)            1의 개수가 짝수            │ │
│  │  4     AF    보조 캐리                하위 니블 캐리             │ │
│  │  6     ZF    결과가 0                 결과 == 0                  │ │
│  │  7     SF    부호 (음수)              결과 < 0 (MSB=1)           │ │
│  │  8     TF    트랩 (단일 스텝)          디버깅용                   │ │
│  │  9     IF    인터럽트 활성화          IF=1이면 인터럽트 허용     │ │
│  │  10    DF    방향 (스트링)            DF=1: 감소, DF=0: 증가     │ │
│  │  11    OF    오버플로우               부호 있는 오버플로우       │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ARM CPSR (Current Program Status Register)】                      │
│  ──────────────────────────────────────────                         │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  31  30  29  28  27  26  ...   7   6   5   4   3   2   1   0    │ │
│  │  ┌───┬───┬───┬───┬───┬───┬─────┬───┬───┬───┬───┬───┬───┬───┐   │ │
│  │  │ N │ Z │ C │ V │ Q │   │     │ I │ F │ T │   Mode    │   │   │ │
│  │  └───┴───┴───┴───┴───┴───┴─────┴───┴───┴───┴───────────┴───┘   │ │
│  │                                                               │ │
│  │  N (Negative): 결과가 음수 (MSB=1)                             │ │
│  │  Z (Zero): 결과가 0                                            │ │
│  │  C (Carry): 캐리 발생 or 보로우 없음                            │ │
│  │  V (Overflow): 부호 있는 오버플로우                             │ │
│  │  Q (Saturation): 포화 연산 발생                                 │ │
│  │  I (IRQ): 인터럽트 마스크                                       │ │
│  │  F (FIQ): 빠른 인터럽트 마스크                                  │ │
│  │  T (Thumb): Thumb 모드                                         │ │
│  │  Mode: 프로세서 모드 (User, FIQ, IRQ, SVC...)                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 조건 코드 플래그 상세

```
┌─────────────────────────────────────────────────────────────────────┐
│                조건 코드 플래그 상세                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Z (Zero) 플래그】                                                 │
│  ──────────────────────                                             │
│  • 설정: 연산 결과 == 0                                              │
│  • 용도: CMP 후 분기 (BEQ, BNE)                                     │
│  • 예: SUB R0, R1, R1 ; R0=0 → Z=1                                  │
│                                                                     │
│  【N (Negative/Signed) 플래그】                                       │
│  ─────────────────────────────────                                   │
│  • 설정: 결과의 MSB == 1 (부호 있는 수에서 음수)                    │
│  • 용도: 부호 있는 비교 (BLT, BGE)                                   │
│  • 예: SUB R0, #1, #2 ; R0=-1 → N=1                                 │
│                                                                     │
│  【C (Carry) 플래그】                                                 │
│  ──────────────────────────                                          │
│  • 덧셈: 결과 > 최댓값 → C=1                                         │
│  • 뺄셈: 결과 >= 0 → C=1 (보로우 없음)                               │
│  • 시프트: 밀려나는 비트                                             │
│  • 용도: 부호 없는 비교, 다중 정밀도 연산                            │
│                                                                     │
│  【V (Overflow) 플래그】                                              │
│  ───────────────────────────                                         │
│  • 설정: 부호 있는 연산에서 결과 범위 초과                          │
│  • 조건: 양수+양수=음수 or 음수+음수=양수                            │
│  • 용도: 부호 있는 오버플로우 검출                                   │
│  • 예: 127+1 (8비트) → -128 (V=1)                                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  플래그 설정 예시 (8비트):                                       │ │
│  │  ────────────────────────                                        │ │
│  │  연산        결과    Z   N   C   V                               │ │
│  │  ──────────  ─────   ─   ─   ─   ─                               │ │
│  │  127 + 1     -128    0   1   0   1   (부호 오버플로우)           │ │
│  │  255 + 1     0       1   0   1   0   (캐리 발생)                 │ │
│  │  0 - 1       255     0   1   0   0   (부호 없는)                  │ │
│  │  128 - 1     127     0   0   1   0   (정상)                       │ │
│  │  0 - 0       0       1   0   1   0   (제로 + 캐리)                │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CMP (Compare)와 플래그】                                          │
│  ──────────────────────────                                          │
│  CMP R1, R2 ; R1 - R2 (결과는 버림, 플래그만 설정)                  │
│                                                                     │
│  조건           플래그 조건              의미                        │
│  ─────────────  ────────────────────    ────────────                 │
│  R1 == R2      Z=1                      같음                        │
│  R1 != R2      Z=0                      다름                        │
│  R1 > R2 (무)  C=1 && Z=0               부호 없는 초과               │
│  R1 < R2 (무)  C=0                      부호 없는 미만               │
│  R1 > R2 (유)  N=V                      부호 있는 초과               │
│  R1 < R2 (유)  N!=V                     부호 있는 미만               │
└─────────────────────────────────────────────────────────────────────┘
```

### ARM 조건부 실행

```
┌─────────────────────────────────────────────────────────────────────┐
│                ARM 조건부 실행                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【조건 코드 (Condition Codes)】                                     │
│  ───────────────────────────────                                     │
│                                                                     │
│  코드  접미사  의미             플래그 조건                          │
│  ────  ──────  ───────────────  ────────────────                     │
│  0000  EQ      Equal            Z=1                                  │
│  0001  NE      Not Equal        Z=0                                  │
│  0010  CS/HS   Carry Set        C=1 (unsigned >=)                    │
│  0011  CC/LO   Carry Clear      C=0 (unsigned <)                     │
│  0100  MI      Minus            N=1                                  │
│  0101  PL      Plus             N=0                                  │
│  0110  VS      Overflow         V=1                                  │
│  0111  VC      No Overflow      V=0                                  │
│  1000  HI      Higher           C=1 && Z=0 (unsigned >)              │
│  1001  LS      Lower/Same       C=0 || Z=1 (unsigned <=)             │
│  1010  GE      Greater/Equal    N=V (signed >=)                      │
│  1011  LT      Less Than        N!=V (signed <)                      │
│  1100  GT      Greater Than     Z=0 && N=V (signed >)                │
│  1101  LE      Less/Equal       Z=1 || N!=V (signed <=)              │
│  1110  AL      Always           (기본)                               │
│                                                                     │
│  【조건부 실행 예시】                                                │
│  ──────────────────────                                              │
│                                                                     │
│  ; 분기 없이 조건부 실행                                             │
│  CMP R1, #0                                                         │
│  ADDEQ R2, R2, #1   ; R1==0이면 R2++                                │
│  ADDNE R2, R2, #2   ; R1!=0이면 R2+=2                               │
│                                                                     │
│  ; 조건부 분기                                                       │
│  CMP R1, R2                                                         │
│  BEQ equal_label    ; R1==R2이면 점프                               │
│  BNE not_equal      ; R1!=R2이면 점프                               │
│  BLT less_than      ; R1<R2 (부호 있음)이면 점프                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  조건부 실행의 장점:                                             │ │
│  │  ────────────────────────                                       │ │
│  │  • 분기 감소 → 파이프라인 버블 감소                              │ │
│  │  • 코드 크기 작아짐                                              │ │
│  │  • 성능 향상 (짧은 조건문에서)                                    │ │
│  │                                                               │ │
│  │  단점:                                                           │ │
│  │  • 모든 명령어가 실행은 아니지만 디코딩됨                        │ │
│  │  • 조건이 복잡하면 분기가 더 나을 수 있음                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: OS 인터럽트 제어**
- 요구: 임계 영역 보호
- 접근: IF 플래그 조작
- 구현: local_irq_save/restore
- 결과: 원자적 실행

**시나리오 2: 조건부 분기 최적화**
- 요구: 분기 오버헤드 감소
- 접근: CMOV, ARM 조건부 실행
- 구현: 분기 없는 코드
- 결과: 파이프라인 효율 향상

**시나리오 3: 다중 정밀도 연산**
- 요구: 큰 수 연산
- 접근: 캐리 플래그 활용
- 구현: ADC/SBB 명령어
- 결과: 정확한 큰 수 연산

**시나리오 4: 디버거 구현**
- 요구: 단일 스텝 실행
- 접근: TF 플래그 설정
- 구현: ptrace 시스템 콜
- 결과: 명령어별 정지

### 플래그 분석 명령어

```
┌─────────────────────────────────────────────────────────────────────┐
│                플래그 분석/디버깅                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【GDB】                                                            │
│  ────────                                                            │
│  # 플래그 확인                                                       │
│  (gdb) info registers eflops                                        │
│  (gdb) p/t $eflops                                                  │
│                                                                     │
│  # 조건부 브레이크포인트                                             │
│  (gdb) break *0x1000 if $eflops & 0x40  # ZF=1이면                  │
│                                                                     │
│  【어셈블리】                                                        │
│  ────────────                                                        │
│  ; 플래그 설정 명령어                                                │
│  CMP R1, R2          ; R1 - R2, 플래그 설정                         │
│  TST R1, #0xFF       ; R1 AND #0xFF, 플래그 설정                    │
│  TEQ R1, R2          ; R1 XOR R2, 플래그 설정                       │
│  ADDS R1, R2, R3     ; ADD + 플래그 설정                            │
│                                                                     │
│  ; 플래그 직접 조작                                                  │
│  SEC                 ; C=1 (Set Carry)                               │
│  CLC                 ; C=0 (Clear Carry)                             │
│  SEI                 ; I=1 (Set Interrupt)                          │
│  CLI                 ; I=0 (Clear Interrupt)                        │
│                                                                     │
│  ; 플래그 저장/복원                                                  │
│  PUSHF               ; 스택에 FLAGS 저장                             │
│  POPF                ; FLAGS 복원                                    │
│  MRS R0, CPSR        ; ARM: CPSR → R0                                │
│  MSR CPSR, R0        ; ARM: R0 → CPSR                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  플래그 기반 분기 코드 생성 (C):                                  │ │
│  │  ────────────────────────────────                                │ │
│  │  if (a > b) {                                                   │ │
│  │      // ...                                                      │ │
│  │  }                                                               │ │
│  │                                                               │ │
│  │  → CMP R0, R1         ; a - b                                   │ │
│  │  → BLE skip           ; a <= b이면 skip                          │ │
│  │  → ...                   ; if 본문                               │ │
│  │  skip:                                                         │ │

```

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 플래그 | 용도 | 중요도 |
|:---|:---|:---:|
| **Z** | 같음, 0 검사 | 필수 |
| **N** | 부호 검사 | 필수 |
| **C** | 캐리, 부호 없는 비교 | 필수 |
| **V** | 오버플로우 | 높음 |
| **I** | 인터럽트 제어 | 높음 |

### 핵심 요약

```
• 상태 레지스터: 연산 결과 상태 저장
• 핵심 플래그: Z, N, C, V
• 조건부 분기: 플래그 기반 결정
• 인터럽트 제어: I/F 플래그
• OS: 모드 비트로 권한 관리
```
