# 173. 주소 지정 방식 (Addressing Mode)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 피연산자의 위치를 찾는 방법을 정의한 규칙
> 2. **가치**: 코드 유연성, 메모리 효율, 컴파일러 최적화의 핵심
> 3. **융합**: 배열 접근, 구조체, 포인터, 함수 호출 구현

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

주소 지정 방식(Addressing Mode)은 **명령어에서 피연산자의 실제 위치(유효 주소)를 결정하는 방법**이다. CPU는 다양한 주소 지정 방식을 제공하여 프로그램이 효율적으로 데이터에 접근할 수 있게 한다.

### 💡 비유: 우편물 배달 방식

주소 지정 방식은 **우편물 배달 방식**과 같다.
- 즉시: 편지 내용에 직접 써있음 (값 자체)
- 레지스터: 사서함 번호 (빠른 접근)
- 직접: 정확한 주소 (번지수)
- 간접: "A씨에게 물어보세요" (포인터)
- 인덱스: "101동 + 호수" (배열)

### 주소 지정 방식 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                주소 지정 방식 개요                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【분류 체계】                                                       │
│  ─────────────                                                      │
│                                                                     │
│  주소 지정 방식                                                      │
│  ├── 즉시 (Immediate)          : 값 자체                             │
│  ├── 레지스터 (Register)       : 레지스터 내용                       │
│  ├── 메모리 직접 (Direct)      : 주소 직접 지정                      │
│  └── 메모리 간접 (Indirect)    : 주소를 통한 접근                    │
│      ├── 레지스터 간접         : 레지스터가 주소 저장                 │
│      ├── 변위 (Displacement)   : 베이스 + 오프셋                     │
│      ├── 인덱스 (Indexed)      : 베이스 + 인덱스×크기                │
│      └── PC 상대 (PC-Relative) : PC + 오프셋                        │
│                                                                     │
│  【유효 주소(Effective Address) 계산】                               │
│  ──────────────────────────────────                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  방식           유효 주소(EA)         예시                      │ │
│  │  ─────────────  ────────────────     ─────────────             │ │
│  │  즉시           없음 (값 자체)        MOV EAX, 100              │ │
│  │  레지스터       없음 (레지스터)       MOV EAX, EBX              │ │
│  │  직접           EA = 주소             MOV EAX, [1000]           │ │
│  │  레지스터 간접   EA = [Rn]            MOV EAX, [EBX]            │ │
│  │  변위           EA = [Rn] + disp     MOV EAX, [EBX+8]          │ │
│  │  인덱스         EA = [Rn] + [Ri]×k   MOV EAX, [EBX+ECX*4]      │ │
│  │  PC 상대        EA = PC + offset     JMP +100                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ISA별 주소 지정 방식 수】                                         │
│  ────────────────────────────                                       │
│  • ARM: 10+ 가지 (Flexible, 강력)                                   │
│  • x86: 20+ 가지 (매우 다양, 복잡)                                   │
│  • MIPS: 5 가지 (단순)                                              │
│  • RISC-V: 7 가지 (모듈형)                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  주소 지정 방식 선택 기준:                                        │ │
│  │  ────────────────────────────                                     │ │
│  │  • 데이터 위치: 레지스터 vs 메모리                                │ │
│  │  • 접근 패턴: 순차, 랜덤, 인덱스                                  │ │
│  │  • 코드 크기: 명령어 길이 고려                                    │ │
│  │  • 성능: 메모리 접근 횟수                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 주소 지정 방식 상세 분류

### 방식별 특징

```
┌─────────────────────────────────────────────────────────────────────┐
│                주소 지정 방식 상세 분류                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 데이터가 어디에 있나?】                                        │
│  ────────────────────────────                                       │
│                                                                     │
│  즉시:    명령어 안에 (opcode 옆)                                   │
│  레지스터: CPU 내부 레지스터                                        │
│  메모리:  메모리 (주소 계산 필요)                                   │
│                                                                     │
│  【2. 메모리 주소 계산 방식】                                        │
│  ────────────────────────────                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  방식           수식                    활용                   │ │
│  │  ─────────────  ────────────────────  ──────────────          │ │
│  │  직접           addr                    전역 변수               │ │
│  │  레지스터 간접   Rn                      동적 할당              │ │
│  │  베이스+변위     Rn + disp              구조체 멤버             │ │
│  │  인덱스          Rn + Ri*scale          배열 요소               │ │
│  │  베이스+인덱스   Rn + Ri*scale + disp   2차원 배열              │ │
│  │  PC 상대        PC + offset            지역 분기               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【3. 접근 속도 비교】                                               │
│  ────────────────────────                                           │
│  레지스터 > 즉시 > 캐시 히트 > 메모리 직접 > 메모리 간접            │
│  (0 클럭)   (0 클럭)  (수 클럭)    (수십 클럭)    (더 느림)         │
│                                                                     │
│  【4. 코드 크기 영향】                                               │
│  ────────────────────────                                           │
│  • 즉시: 큰 값은 명령어 길어짐                                       │
│  • 직접: 전체 주소 필요 (32/64비트)                                  │
│  • 간접: 레지스터 번호만 필요 (3~5비트)                              │
│  • PC 상대: 작은 오프셋만 필요                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ARM 주소 지정 예시:                                              │ │
│  │  ────────────────────                                             │ │
│  │  LDR R0, =1000      ; 직접 (literal pool)                       │ │
│  │  LDR R0, [R1]       ; 레지스터 간접                              │ │
│  │  LDR R0, [R1, #8]   ; 변위                                       │ │
│  │  LDR R0, [R1, R2]   ; 인덱스                                     │ │
│  │  LDR R0, [R1, R2, LSL#2]  ; 인덱스 + 스케일                      │ │
│  │  LDR R0, [R1], #4   ; 후증가 (auto-increment)                    │ │
│  │  LDR R0, [R1, #4]!  ; 전증가 (pre-increment)                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 주소 지정 방식 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                주소 지정 방식 × 타 과목 융합                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                        │
│  • 배열 접근: 인덱스 주소 지정                                       │
│  • 구조체: 변위 주소 지정                                            │
│  • 포인터: 간접 주소 지정                                            │
│  • 지역 변수: 스택 포인터 + 변위                                     │
│  • 전역 변수: 직접 또는 PC 상대                                      │
│                                                                     │
│  【자료구조】                                                        │
│  • 배열: 인덱스 주소 지정                                            │
│  • 연결 리스트: 간접 주소 지정                                       │
│  • 트리: 다중 간접                                                   │
│  • 해시: 인덱스 + 간접                                               │
│                                                                     │
│  【운영체제】                                                        │
│  • 시스템 콜: 간접 호출                                              │
│  • 가상 메모리: 페이지 테이블 간접                                   │
│  • 컨텍스트: 스택 변위 접근                                          │
│  • 공유 메모리: 직접 매핑                                            │
│                                                                     │
│  【네트워크】                                                        │
│  • 패킷 헤더: 변위 접근                                              │
│  • 버퍼 순회: 인덱스                                                 │
│  • 포인터 체인: 간접                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  C 코드와 주소 지정 방식 매핑:                                    │ │
│  │  ─────────────────────────────                                   │ │
│  │  int a = 10;              // 즉시                                │ │
│  │  int b = a;               // 레지스터/직접                       │ │
│  │  int *p = &a;             // 직접                                │ │
│  │  int c = *p;              // 간접                                │ │
│  │  int arr[10]; arr[i]      // 인덱스 (베이스 + i*4)               │ │
│  │  struct { int x, y; } s;  s.y  // 변위 (베이스 + 4)              │ │
│  │  goto label;              // PC 상대                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 주소 지정 방식 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                주소 지정 방식 분석                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【어셈블리 분석】                                                   │
│  ──────────────────                                                  │
│  // C 코드                                                           │
│  int arr[100];                                                      │
│  int sum = 0;                                                       │
│  for (int i = 0; i < 100; i++) sum += arr[i];                       │
│                                                                     │
│  // x86-64 어셈블리                                                 │
│  xor eax, eax              ; sum = 0  (레지스터)                    │
│  xor ecx, ecx              ; i = 0    (레지스터)                    │
│  loop:                                                              │
│  add eax, [arr + rcx*4]    ; arr[i]   (인덱스)                      │
│  inc ecx                   ; i++      (레지스터)                    │
│  cmp ecx, 100              ; i < 100? (즉시)                        │
│  jl loop                   ; 분기     (PC 상대)                      │
│                                                                     │
│  【ARM 주소 지정 활용】                                              │
│  ──────────────────────────                                         │
│  ; 구조체 접근                                                       │
│  struct Point { int x, y; };                                        │
│  LDR R0, [R1, #0]    ; p->x  (변위)                                 │
│  LDR R0, [R1, #4]    ; p->y  (변위)                                 │
│                                                                     │
│  ; 배열 순회 (auto-increment)                                       │
│  loop:                                                              │
│  LDR R0, [R1], #4    ; 로드 후 R1 += 4                              │
│  SUBS R2, R2, #1                                                 │
│  BNE loop                                                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  주소 지정 방식 디버깅 체크리스트:                                 │ │
│  │  ────────────────────────────────                                │ │
│  │  □ EA 계산이 올바른가?                                           │ │
│  │  □ 메모리 정렬이 맞는가?                                          │ │
│  │  □ 포인터가 유효한가?                                             │ │
│  │  □ 배열 인덱스가 범위 내인가?                                     │ │
│  │  □ 변위가 올바른가?                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```
