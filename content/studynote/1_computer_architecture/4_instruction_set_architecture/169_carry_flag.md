# 169. 캐리 플래그 (Carry Flag)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 부호 없는 정수 연산의 오버플로우/언더플로우를 표시하는 플래그
> 2. **가치**: 다중 정밀도 연산, 비트 시프트, 부호 없는 비교의 핵심 지표
> 3. **융합**: 암호학, 빅인티저 연산, 시프트-로테이션 명령어

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

캐리 플래그(CF, Carry Flag)는 **부호 없는 정수 연산에서 자리올림(carry) 또는 빌림(borrow) 발생을 표시하는 플래그**다. 덧셈 시 결과가 레지스터 비트 수를 초과하거나, 뺄셈 시 피감수가 감수보다 작으면 CF=1로 설정된다.

### 💡 비유: 계산기의 자리올림 표시

CF는 **계산기의 자리올림 표시등**과 같다. 999 + 1 = 1000에서 맨 앞의 1은 자리올림이다. 8비트 계산기로 255 + 1을 하면 0이 되고 자리올림 표시등이 켜진다.

### CF 동작 원리

```
┌─────────────────────────────────────────────────────────────────────┐
│                CF 동작 원리                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CF 설정 규칙】                                                    │
│  ────────────────                                                    │
│                                                                     │
│  덧셈: 결과 > 최댓값  →  CF = 1 (자리올림)                           │
│  뺄셈: 피감수 < 감수  →  CF = 1 (빌림)                               │
│  시프트: 밀려나간 비트  →  CF = 밀려난 비트값                         │
│                                                                     │
│  【CF 설정 명령어】                                                  │
│  ──────────────────────                                             │
│  • 덧셈: ADD, ADC (Add with Carry)                                  │
│  • 뺄셈: SUB, SBB (Subtract with Borrow)                            │
│  • 비교: CMP (부호 없는 비교)                                        │
│  • 시프트: SHL, SHR, ROL, ROR, RCL, RCR                             │
│  • 비트: BT (Bit Test)                                              │
│                                                                     │
│  【CF 예시 (8비트 기준, 최댓값=255)】                                 │
│  ─────────────────────────────────                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  연산              결과      CF    설명                         │ │
│  │  ────────────────  ─────    ────  ────────────────────         │ │
│  │  ADD AL, 255+1     0        1     255+1=256 → 0, 캐리 1        │ │
│  │  ADD AL, 200+100   44       1     300=256+44 → 44, 캐리 1      │ │
│  │  ADD AL, 100+50    150      0     150 < 255, 캐리 없음          │ │
│  │                                                               │ │
│  │  SUB AL, 0-1       255      1     0-1: 빌림 발생               │ │
│  │  SUB AL, 50-100    206      1     50<100: 빌림 발생            │ │
│  │  SUB AL, 100-50    50       0     100>50: 빌림 없음            │ │
│  │                                                               │ │
│  │  SHL AL, 1         (AL<<1)   CF=MSB  최상위 비트가 CF로         │ │
│  │  SHR AL, 1         (AL>>1)   CF=LSB  최하위 비트가 CF로         │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CF vs OF 구분】                                                   │
│  ──────────────────                                                  │
│                                                                     │
│        부호 없음      부호 있음                                      │
│  ─────────────────  ─────────────────                               │
│  오버플로우    CF              OF                                    │
│  비교          JC/JNC          JO/JNO (또는 JL/JG 등)               │
│  의미          범위 초과       부호 오류                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CF 위치:                                                        │ │
│  │  ──────────                                                      │ │
│  │  x86: EFLAGS 비트 0 (CF)                                         │ │
│  │  ARM: CPSR 비트 29 (C)                                           │ │
│  │  RISC-V: 없음 (슬롯 기반 조건 분기)                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. CF 활용 패턴

### CF 활용 시나리오

```
┌─────────────────────────────────────────────────────────────────────┐
│                CF 활용 패턴                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 다중 정밀도 연산 (Multi-Precision)】                           │
│  ──────────────────────────────────────────                         │
│  ; 32비트로 64비트 덧셈                                              │
│  ADD  EAX, EBX      ; 하위 32비트 덧셈, CF 설정                      │
│  ADC  EDX, ECX      ; EDX+ECX+CF (캐리 포함)                         │
│                                                                     │
│  ; 32비트로 64비트 뺄셈                                              │
│  SUB  EAX, EBX      ; 하위 32비트 뺄셈, CF 설정                      │
│  SBB  EDX, ECX      ; EDX-ECX-CF (빌림 포함)                         │
│                                                                     │
│  【2. 부호 없는 비교 (Unsigned Compare)】                            │
│  ────────────────────────────────────────                           │
│  CMP  EAX, EBX                                                      │
│  JC   less_than     ; EAX < EBX (부호 없음) → CF=1                  │
│  JNC  greater_equal ; EAX >= EBX (부호 없음) → CF=0                 │
│                                                                     │
│  // C 코드                                                           │
│  if ((unsigned)a < (unsigned)b) { ... }                             │
│                                                                     │
│  【3. 순환 시프트 (Rotate through Carry)】                           │
│  ──────────────────────────────────────                             │
│                                                                     │
│       ┌──CF──┐                                                      │
│       ↓      ↓                                                      │
│  RCL: [b7..b0] → [b7..b0][CF]  ; 왼쪽으로 CF 포함 회전              │
│  RCR: [CF][b7..b0] ← [b7..b0]  ; 오른쪽으로 CF 포함 회전            │
│                                                                     │
│  ; 32비트 왼쪽 회전 (RCL 사용)                                       │
│  RCL  EAX, 1        ; CF가 MSB로, MSB가 CF로                        │
│  ADC  EAX, 0        ; CF를 LSB에 더함                                │
│                                                                     │
│  【4. 비트 조작 (Bit Manipulation)】                                 │
│  ────────────────────────────────                                    │
│  BT   EAX, 5        ; 비트 5를 CF로 복사                             │
│  JC   bit_set       ; CF=1이면 비트 5이 설정됨                       │
│                                                                     │
│  ; CF를 이용한 비트 설정/클리어                                      │
│  BTC  EAX, 5        ; 비트 5를 CF로, 토글                            │
│  BTR  EAX, 5        ; 비트 5를 CF로, 클리어                          │
│  BTS  EAX, 5        ; 비트 5를 CF로, 설정                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CF 기반 조건부 실행 (ARM):                                       │ │
│  │  ────────────────────────────                                     │ │
│  │  MOVCS R0, #1     ; CF=1이면 R0=1                                │ │
│  │  MOVCC R0, #0     ; CF=0이면 R0=0                                │ │
│  │  ADDCS R1, R1, #1 ; CF=1이면 R1++                                │ │
│  │                                                               │ │
│  │  → 분기 없이 부호 없는 조건 처리                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. CF × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                CF × 타 과목 융합                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                        │
│  • unsigned 비교 → CMP + JC/JNC                                     │
│  • long long 연산 → ADC/SBB 활용                                    │
│  • __int128 연산 → 다중 ADC 체인                                    │
│  • 루프 카운터: unsigned 감소 + JNC                                  │
│                                                                     │
│  【운영체제】                                                        │
│  • 메모리 주소 계산: 페이지 경계 검사                                │
│  • 파일 오프셋: 대용량 파일 처리                                     │
│  • 타이머: 64비트 카운터 (HPET)                                      │
│  • 체크섬 계산: 캐리 포함 덧셈                                       │
│                                                                     │
│  【알고리즘】                                                        │
│  • 빅인티저: 다중 정밀도 연산                                        │
│  • 암호학: 모듈러 덧셈/뺄셈                                          │
│  • CRC/해시: 캐리 포함 연산                                          │
│  • 비트 조작: 순환 시프트                                            │
│                                                                     │
│  【네트워크】                                                        │
│  • IP 체크섬: 1의 보수 덧셈 + 캐리                                   │
│  • TCP 시퀀스 번호: 32비트 랩어라운드                                │
│  • 패킷 길이 계산: 오버플로우 검사                                   │
│                                                                     │
│  【보안】                                                            │
│  • 정수 오버플로우: 취약점 탐지                                       │
│  • 암호화: 다중 정밀도 모듈러 연산                                    │
│  • 랜덤 생성: 엔트로피 누적                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CF vs ZF vs OF 활용 구분:                                       │ │
│  │  ────────────────────────────                                     │ │
│  │  • CF: unsigned 오버플로우, 루프 카운터                           │ │
│  │  • ZF: 같음 비교, 0 검사                                          │ │
│  │  • OF: signed 오버플로우                                          │ │
│  │                                                               │ │
│  │  실제 C 코드 비율:                                                │ │
│  │  • unsigned 연산: ~40% (메모리, 인덱스)                           │ │
│  │  • signed 연산: ~60% (일반 산술)                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### CF 디버깅/분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                CF 디버깅/분석                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【GDB】                                                            │
│  ────────                                                            │
│  # CF 확인                                                          │
│  (gdb) p $eflags & 0x1   # CF는 비트 0                              │
│  (gdb) p/t $eflags & 1   # 0 또는 1                                 │
│                                                                     │
│  # CF 기반 브레이크포인트                                            │
│  (gdb) break *0x1000 if ($eflags & 1) != 0                          │
│                                                                     │
│  【어셈블리】                                                        │
│  ────────────                                                        │
│  ; CF 직접 테스트                                                    │
│  CMP  EAX, EBX       ; unsigned 비교                                │
│  JC   carry_set      ; CF=1이면 점프                                 │
│  JNC  no_carry       ; CF=0이면 점프                                 │
│                                                                     │
│  ; SETC/SETNC (x86)                                                 │
│  SETC AL             ; CF=1이면 AL=1, 아니면 AL=0                    │
│  SETNC BL            ; CF=0이면 BL=1, 아니면 BL=0                    │
│                                                                     │
│  ; CMOVC/CMOVNC (x86 조건부 이동)                                   │
│  CMOVC EAX, EBX      ; CF=1이면 EAX=EBX                             │
│  CMOVNC EAX, ECX     ; CF=0이면 EAX=ECX                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CF 활용 최적화 팁:                                               │ │
│  │  ──────────────────────                                           │ │
│  │  1. ADC/SBB로 큰 수 연산 최적화                                    │ │
│  │     ADD+ADC 조합 → 64비트를 32비트 2개로                           │ │
│  │                                                               │ │
│  │  2. unsigned 루프 카운터                                          │ │
│  │     for (i=n; i>0; i--) → SUB+SOMI + BNE                        │ │
│  │                                                               │ │
│  │  3. 비트 테스트 → BT + JC                                         │ │
│  │     AND+TEST → BT (더 직관적)                                     │ │
│  │                                                               │ │
│  │  4. 순환 시프트 → RCL/RCR                                         │ │
│  │     32비트+CF = 33비트 순환                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• CF: 부호 없는 오버플로우/빌림 표시
• 설정: 결과 > 최댓값 또는 피감수 < 감수
• 용도: 다중 정밀도, unsigned 비교, 순환 시프트
• 분기: JC/JNC (x86), BCC/BCS (ARM)
• 연산: ADC (Add with Carry), SBB (Sub with Borrow)
```
