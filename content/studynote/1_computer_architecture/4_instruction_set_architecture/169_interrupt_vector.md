> **핵심 인사이트**
> 1. 인터럽트 벡터(Interrupt Vector)는 마우스 클릭, 프린터 완료, 정전 등 수십 가지의 비상 사태가 발생했을 때, CPU가 **"어떤 종류의 사고인지 구별하고, 이 사고를 해결할 119 구조대원(ISR 코드)이 메모리 어디에 숨어있는지 알려주는 고유 번호표(주소 포인터)"**이다.
> 2. 하드웨어 장치(마우스, 키보드)가 인터럽트 선에 전기를 쏠 때 이 고유 번호(벡터)를 함께 데이터 버스에 실어 CPU로 던져준다. (Vectored Interrupt 방식)
> 3. CPU는 이 번호를 들고 메인 메모리의 고정된 장소에 있는 **'인터럽트 벡터 테이블(Interrupt Vector Table)'**을 뒤져서, 해당 사고를 처리할 코드가 있는 메모리 주소(PC 값)로 0.001초 만에 다이렉트로 점프(Branch)한다.

---

## Ⅰ. 수십 개의 비상벨, 누가 울렸나?
컴퓨터에는 키보드, 마우스, 하드디스크, 랜카드 등 인터럽트(비상벨)를 누를 수 있는 장치가 수십 개나 있습니다. 만약 CPU가 비상벨 소리만 듣고 누가 울렸는지 모른다면? (이것을 '비-벡터형(Non-vectored) 폴링 방식'이라 합니다). CPU는 키보드한테 가서 "너 쳤어?", 랜카드한테 "너 쳤어?" 하고 일일이 범인을 찾아다니느라 황금 같은 클럭을 다 허비하게 됩니다.

이 시간 낭비를 없애기 위해 **벡터 인터럽트(Vectored Interrupt)** 방식이 탄생했습니다.
키보드가 비상벨 선(INT)을 당길 때, 빈손으로 딩동 하는 게 아니라 **데이터 버스에 자기만의 고유한 이름표(예: "나는 0x21번 장치다!")**를 딱 실어서 CPU 입에 쑤셔 넣습니다. 이 `0x21`이라는 고유 번호가 바로 인터럽트 벡터입니다. CPU는 범인을 찾아 돌아다닐 필요 없이 번호표만 보고 즉각 행동에 나섭니다.

```ascii
[비-벡터형(과거) vs 벡터 인터럽트(현재)의 범인 찾기]

 (과거: 비-벡터형)
 장치: "비상벨 쾅!"
 CPU: (일일이 돌아다니며) "키보드 님이세요? 아님 마우스 님이세요?" -> 시간 엄청 낭비!

 (현대: 인터럽트 벡터)
 장치: "비상벨 쾅! 내 번호는 [ 0x09번 ] 이다!!"
 CPU: "오케이 9번(키보드) 접수! 당장 키보드 처리반 출동!" -> 0.1초 컷!
```

📢 **섹션 요약 비유**: 비-벡터 방식은 '아파트 화재경보기'입니다. 경보기가 울리면 경비 아저씨(CPU)가 1층부터 15층까지 일일이 뛰어 올라가며 어디서 불이 났는지 찾아야 합니다. 인터럽트 벡터는 최신 '스마트 화재 시스템'입니다. 경보기가 울릴 때 관리실 모니터에 "103동 502호(벡터 번호)"라고 정확한 주소가 찍히기 때문에 1초도 지체하지 않고 소방관을 출동시킬 수 있습니다.

## Ⅱ. 인터럽트 벡터 테이블: 119 출동 전화번호부
CPU가 `0x09번`이라는 벡터를 받았습니다. 그런데 이 9번 사고를 해결하려면 메모리 몇 번지에 있는 코드(프로그램)를 실행해야 할까요?

이 정보를 담고 있는 거대한 안내 데스크가 바로 메인 메모리의 아주 특별한 구역(보통 메모리 주소 0번지 근처 0x00000000 ~)에 보호되어 있는 **인터럽트 벡터 테이블(Interrupt Vector Table, IVT)**입니다.
CPU는 9번 벡터를 받으면, 수학 공식(`테이블 시작 주소 + (9 * 주소 길이)`)을 써서 벡터 테이블의 특정 서랍을 탁 엽니다. 그 서랍 안에는 **"키보드 처리 코드(ISR)는 메모리 8000번지에 숨어있습니다"**라는 진짜 메모리 주소(PC 값)가 적혀 있습니다. CPU는 뒤도 돌아보지 않고 8000번지로 PC(프로그램 카운터)를 점프시켜 사고 처리를 시작합니다.

```ascii
[인터럽트 벡터 테이블(IVT)을 통한 광속 점프 릴레이]

 1. 키보드: "0x09번 벡터 던짐!"
       |
       V
 2. [ 메인 메모리: 인터럽트 벡터 테이블 구역 ]
    | 0x00번 | 정전 사고 처리 주소 -> 0x1000
    |  ...   | 
    | 0x09번 | 키보드 사고 처리 주소 -> [ 0x8000 번지! ] <-- 요기서 주소 획득!
       |
       V
 3. PC(프로그램 카운터)에 0x8000 강제 주입!
    -> CPU가 8000번지로 날아가서 키보드 글자 화면에 찍는 코드(ISR) 실행!
```

📢 **섹션 요약 비유**: 인터럽트 벡터 테이블은 동사무소의 '민원 전화번호부'입니다. 내가 "9번(수도 고장) 문제요!"라고 9번 번호표(벡터)를 내밀면, 안내원이 테이블 명부를 쓱 보고 "아, 수도 고장이면 3층의 305호(메모리 주소) 김 주무관님한테 가세요"라고 정확한 목적지를 다이렉트로 알려주는 완벽한 안내 시스템입니다.

## Ⅲ. 아키텍처 관점: IVT의 수정과 해킹(Hooking)
이 인터럽트 벡터 테이블(IVT)은 운영체제(Windows, Linux)가 부팅될 때 메모리 공간에 예쁘게 만들어 놓습니다. 그런데 이 IVT의 주소를 살짝 바꾸면 컴퓨터의 근간을 지배할 수 있습니다.

예를 들어, 정상적인 9번(키보드) 벡터는 윈도우의 `키보드 쳐주기` 코드로 점프해야 합니다. 그런데 **악질 해커나 키로거(Keylogger) 바이러스가 메모리에 침투해서, 벡터 테이블의 9번 서랍에 적힌 주소를 자기가 만든 악성코드 주소(예: 9999번지)로 쓱 바꿔치기해 버립니다.** (이것을 인터럽트 후킹, Hooking이라고 합니다.)
이제 사용자가 키보드를 칠 때마다 CPU는 멍청하게 해커의 악성코드로 먼저 점프하여 내 비밀번호를 해커에게 카톡으로 보낸 뒤, 자연스럽게 원래 윈도우 코드로 다시 돌아가서 나는 해킹당한 사실조차 모르게 됩니다.

현대의 OS(윈도우 10 등)와 CPU 아키텍처(보호 모드, Ring 0)는 이런 후킹을 막기 위해 벡터 테이블이 있는 메모리 구역을 철옹성처럼 보호(Memory Protection)하여 아무나 주소를 덮어쓰지 못하도록 막아내고 있습니다.

```ascii
[인터럽트 후킹(Hooking) 해킹의 원리]

 (정상 상태)
 IVT 0x09번 서랍: [ 8000번지 (윈도우 키보드 코드) ] 

 (해킹 상태: 주소 바꿔치기)
 IVT 0x09번 서랍: [ 9999번지 (해커의 비밀번호 도둑질 코드) ] 
                     |
                     V
         9999번지 실행 완료 후 몰래 원래 8000번지로 점프 시켜서 사용자 눈속임!
```

📢 **섹션 요약 비유**: 인터럽트 후킹 해킹은 '도로 표지판 돌려놓기'입니다. 우체부(CPU)가 키보드 마을로 가는 표지판(벡터 테이블)을 보고 달려가는데, 도둑(해커)이 밤에 몰래 표지판의 화살표를 자기 집 쪽으로 돌려놓았습니다. 우체부는 아무것도 모르고 도둑의 집에 들어가서 중요한 택배(비밀번호)를 내려놓고 오는 무시무시한 범죄입니다.

## 개념 맵 (Concept Map)
```ascii
[인터럽트 벡터 (Interrupt Vector)]
   │
   ├── 정의: 인터럽트 발생 시, 예외(Exception)의 원인을 식별하고 해당 처리 코드(ISR)를 찾기 위해 하드웨어가 CPU로 전송하는 고유 식별 번호
   │
   ├── 아키텍처적 처리 구조: IVT (Interrupt Vector Table)
   │     └── 메모리 하위 주소에 위치한 배열. 벡터 번호를 인덱스(Index)로 삼아 1차원 배열처럼 접근하여 ISR의 실제 메모리 주소(PC)를 1클럭 만에 Fetch
   │
   └── 시스템 프로그래밍과 보안
         ├── OS 관여: 부팅 시 IVT를 메모리에 세팅하여 H/W와 S/W 간의 비상 연락망 개통
         └── 보안 위협: 해커의 IVT 변조(Interrupt Hooking)를 통한 권한 탈취 시도 및 OS의 메모리 보호(Protection) 방어전
```

## 어린이 비유
인터럽트 벡터는 구급차 아저씨의 무전기에 뜨는 **사고 번호표**예요! 소방서에 비상벨이 울릴 때, просто "불났어요!"라고만 하면 구급차 아저씨가 온 동네를 헤매야 하죠. 하지만 비상벨이 울릴 때 삐빅! 하고 무전기에 "9번 사고!"(벡터)라고 번호가 뜨는 거예요. 아저씨는 곧바로 책상에 있는 '119 비밀 전화번호부(벡터 테이블)'를 펼쳐서 9번 옆에 적힌 '우리 동네 마트 주소'를 보고 1초도 안 늦게 씽씽 달려갈 수 있답니다!
