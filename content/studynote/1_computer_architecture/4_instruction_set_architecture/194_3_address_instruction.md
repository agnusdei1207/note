# 194. 3-주소 명령어 (3-Address Instruction)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 목적지와 두 원본을 따로 명시하는 명령어
> 2. **가치**: 비파괴적 연산, 최고 성능, RISC 표준
> 3. **융합**: ARM, MIPS, RISC-V, 현대 아키텍처

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

3-주소 명령어(3-Address Instruction)는 **목적지와 두 원본 피연산자를 각각 명시**하는 명령어 형식이다. "ADD rd, rs1, rs2" 형태에서 rd = rs1 + rs2로, 원본이 보존되는 비파괴적 연산이다. RISC 아키텍처의 표준이다.

### 💡 비유: 계산 결과를 새 상자에

3-주소 명령어는 **계산 결과를 새 상자에 담는 것**과 같다. "A 상자와 B 상자를 더해서 C 상자에 넣어"라고 하면, A와 B는 그대로 있고 C에만 결과가 들어간다. 원본이 보존된다.

### 3-주소 명령어 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                3-주소 명령어 구조                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조】                                                           │
│  ─────────                                                          │
│  ┌─────────────────────┬─────────────────────┬────────────────────┬────────────────┐│
│  │       Opcode        │   Destination       │     Source 1       │    Source 2    ││
│  │      (연산코드)      │     (목적지)        │      (원본1)       │     (원본2)    ││
│  └─────────────────────┴─────────────────────┴────────────────────┴────────────────┘│
│                                                                     │
│  【동작 원리】                                                       │
│  ─────────────                                                      │
│  OP dest, src1, src2    →    dest = src1 OP src2                    │
│                                                                     │
│  예: ADD R0, R1, R2  →  R0 = R1 + R2                                │
│  (R1, R2는 변경되지 않음)                                            │
│                                                                     │
│  【특징】                                                           │
│  ─────────                                                          │
│  • 비파괴적 연산: 원본 보존                                          │
│  • 3-피연산자: 목적지 + 2원본                                        │
│  • RISC 표준: ARM, MIPS, RISC-V                                     │
│  • 레지스터만: Load/Store 아키텍처                                   │
│                                                                     │
│  【ISA별 예시】                                                      │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ISA      문법                 예시                    동작     │ │
│  │  ───────  ───────────────────  ─────────────────────  ──────── │ │
│  │  ARM      OP Rd, Rn, Rm       ADD R0, R1, R2         R0=R1+R2 │ │
│  │  MIPS     OP rd, rs, rt       ADD $t0, $t1, $t2      t0=t1+t2 │ │
│  │  RISC-V   OP rd, rs1, rs2     ADD x5, x6, x7         x5=x6+x7 │ │
│  │                                                               │ │
│  │  (모두 비파괴적)                                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예시: a = b + c】                                                 │
│  ──────────────────                                                  │
│  ; ARM                                                               │
│  LDR R0, [b]         ; R0 = b                                       │
│  LDR R1, [c]         ; R1 = c                                       │
│  ADD R2, R0, R1      ; R2 = R0 + R1 = b + c                         │
│  STR R2, [a]         ; a = R2                                       │
│  ; 4개 명령어 (Load/Store 포함)                                      │
│                                                                     │
│  ; 레지스터에 이미 있으면                                             │
│  ADD R2, R0, R1      ; R2 = R0 + R1  ; 1개 명령어                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  3-주소 장단점:                                                    │ │
│  │  ──────────────                                                    │ │
│  │  장점:                                                             │ │
│  │  • 비파괴적 연산 (원본 보존)                                        │ │
│  │  • 명령어 수 최소 (레지스터 기반)                                   │ │
│  │  • 파이프라인 효율                                                 │ │
│  │  • 최고 성능                                                       │ │
│  │                                                               │ │
│  │  단점:                                                             │ │
│  │  • 명령어 길이 김 (32비트 고정)                                     │ │
│  │  • 코드 크기 큼                                                    │ │
│  │  • 레지스터 많이 필요                                               │ │
│  │  • Load/Store 분리 필요                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 3-주소 명령어 활용

### 활용 패턴

```
┌─────────────────────────────────────────────────────────────────────┐
│                3-주소 명령어 활용                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 산술 연산 (ARM)】                                              │
│  ──────────────────────────                                         │
│  // a = b + c - d                                                   │
│  ADD R0, R1, R2      ; R0 = R1 + R2  (b + c)                        │
│  SUB R0, R0, R3      ; R0 = R0 - R3  (b + c - d)                    │
│  ; 2개 명령어 (2-주소는 3개, 1-주소는 4개)                           │
│                                                                     │
│  【2. 비파괴적 연산】                                                │
│  ──────────────────────                                             │
│  // result = a + b (a, b 보존)                                      │
│  ADD R0, R1, R2      ; R0 = R1 + R2                                 │
│  ; R1, R2는 그대로, R0에만 결과                                     │
│                                                                     │
│  【3. 복잡한 수식】                                                  │
│  ──────────────────                                                  │
│  // x = (a + b) * (c - d)                                           │
│  ADD R0, R1, R2      ; R0 = a + b                                   │
│  SUB R3, R4, R5      ; R3 = c - d                                   │
│  MUL R6, R0, R3      ; R6 = R0 * R3                                 │
│  ; 3개 명령어 (원본 모두 보존)                                       │
│                                                                     │
│  【4. 조건부 실행 (ARM)】                                            │
│  ──────────────────────────                                         │
│  CMP R0, #0          ; R0과 0 비교                                   │
│  ADDGT R1, R2, R3    ; if (R0 > 0) R1 = R2 + R3                     │
│  ADDLE R1, R4, R5    ; if (R0 <= 0) R1 = R4 + R5                    │
│  ; 분기 없이 조건부 실행                                             │
│                                                                     │
│  【5. MIPS 예시】                                                    │
│  ──────────────────                                                  │
│  # a = b + c * d                                                    │
│  lw $t0, b           # $t0 = b                                      │
│  lw $t1, c           # $t1 = c                                      │
│  lw $t2, d           # $t2 = d                                      │
│  mul $t3, $t1, $t2   # $t3 = c * d                                  │
│  add $t4, $t0, $t3   # $t4 = b + (c * d)                            │
│  sw $t4, a           # a = $t4                                      │
│                                                                     │
│  【6. RISC-V 예시】                                                  │
│  ──────────────────────                                             │
│  # if (a > b) c = a - b; else c = b - a;                            │
│  beq x5, x6, equal   # if (a == b)                                  │
│  blt x5, x6, less    # if (a < b)                                   │
│  sub x7, x5, x6      # c = a - b (a > b)                            │
│  jal x0, done                                                         │
│  less:                                                               │
│  sub x7, x6, x5      # c = b - a                                    │
│  jal x0, done                                                         │
│  equal:                                                              │
│  addi x7, x0, 0      # c = 0                                        │
│  done:                                                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  3-주소 명령어 장점 활용:                                          │ │
│  │  ────────────────────────────                                      │ │
│  │  1. SSA (Static Single Assignment)와 잘 맞음                       │ │
│  │     x1 = a + b                                                     │ │
│  │     x2 = c + d                                                     │ │
│  │     x3 = x1 * x2                                                   │ │
│  │                                                               │ │
│  │  2. 레지스터 할당 최적화                                            │ │
│  │     • 원본 보존 → 변수가 살아있는 기간 길어짐                       │ │
│  │     • 레지스터 효율 ↑                                               │ │
│  │                                                               │ │
│  │  3. 명령어 스케줄링                                                 │ │
│  │     • 비파괴적 → 의존성 적음                                        │ │
│  │     • 순서 변경 자유로움                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 3-주소 명령어 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                3-주소 명령어 분석                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ARM 어셈블리 분석】                                               │
│  ──────────────────────────                                         │
│  ; int add(int a, int b, int c) { return a + b + c; }               │
│  ; R0=a, R1=b, R2=c                                                 │
│  ADD R0, R0, R1      ; R0 = a + b                                   │
│  ADD R0, R0, R2      ; R0 = (a+b) + c                               │
│  BX LR               ; return R0                                    │
│  ; 2개 명령어로 완료                                                  │
│                                                                     │
│  【MIPS 어셈블리 분석】                                              │
│  ──────────────────────────                                         │
│  # int max(int a, int b) { return (a > b) ? a : b; }                │
│  # $a0=a, $a1=b                                                     │
│  slt $t0, $a1, $a0   # $t0 = (b < a) ? 1 : 0                        │
│  beq $t0, $zero, else  # if (b >= a) goto else                      │
│  add $v0, $a0, $zero # return a                                     │
│  jr $ra                                                              │
│  else:                                                               │
│  add $v0, $a1, $zero # return b                                     │
│  jr $ra                                                              │
│                                                                     │
│  【objdump 분석 (ARM)】                                              │
│  ──────────────────────────                                         │
│  $ arm-linux-gnueabi-objdump -d program                             │
│  1000:  e08 0000 1    add r0, r0, r1                                │
│  1004:  e08 0000 2    add r0, r0, r2                                │
│  1008:  e12 ff1 e     bx lr                                          │
│                                                                     │
│  【RISC-V objdump】                                                  │
│  ────────────────────                                                │
│  $ riscv64-unknown-elf-objdump -d program                           │
│  1000:  00a50533    add a0, a0, a0                                  │
│  1004:  00b505b3    add a1, a0, a1                                  │
│  1008:  00008067    ret                                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  3-주소 최적화 팁:                                                  │ │
│  │  ──────────────                                                    │ │
│  │  1. 레지스터 할당 최적화                                            │ │
│  │     • 자주 쓰는 변수은 레지스터에                                   │ │
│  │     • 레지스터 pressure 줄이기                                      │ │
│  │                                                               │ │
│  │  2. 명령어 스케줄링                                                 │ │
│  │     • 지연 슬롯 활용 (MIPS)                                        │ │
│  │     • 의존성 최소화                                                │ │
│  │                                                               │ │
│  │  3. 조건부 실행 (ARM)                                              │ │
│  │     • 분기 대신 조건부 명령어                                       │ │
│  │     • 파이프라인 버블 감소                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
```
