# 170. 명령어 형식 (Instruction Format)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 명령어의 비트 구조와 필드 배치 방식
> 2. **가치**: 명령어 길이, 피연산자 수, 주소 지정의 기반
> 3. **융합**: ISA 설계, 디코딩 복잡도, 코드 밀도 트레이드오프

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

명령어 형식(Instruction Format)은 **명령어를 구성하는 필드의 배치와 의미를 정의한 구조**다. 일반적으로 연산 코드(opcode), 피연산자(operand), 주소 지정 모드 등의 필드로 구성되며, ISA마다 다양한 형식을 제공한다.

### 💡 비유: 우편 엽서 양식

명령어 형식은 **우편 엽서 양식**과 같다. "누구에게(operand), 무엇을(opcode), 어떻게(addressing mode)"를 적을 칸이 정해져 있다. 양식이 간단하면 쓰기 쉽지만 공간이 낭비되고, 복잡하면 효율적이지만 해석이 어렵다.

### 명령어 형식 기본 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                명령어 형식 기본 구조                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【필드 구성】                                                       │
│  ─────────────                                                      │
│                                                                     │
│  ┌─────────┬──────────────┬──────────────┬──────────────┐          │
│  │ Opcode  │  Operand 1   │  Operand 2   │  Operand 3   │          │
│  │ 연산코드 │   목적지     │    원본 1    │    원본 2    │          │
│  └─────────┴──────────────┴──────────────┴──────────────┘          │
│                                                                     │
│  【필드 종류】                                                       │
│  ─────────────                                                      │
│  • Opcode: 수행할 연산 (ADD, SUB, MOV, ...)                         │
│  • Register 필드: 레지스터 번호 (R0~R15)                             │
│  • Immediate: 상수 값                                                │
│  • Address/Offset: 메모리 주소 또는 변위                             │
│  • Mode: 주소 지정 방식 지정                                         │
│  • Condition: 조건부 실행 조건 (ARM)                                 │
│                                                                     │
│  【주소 수에 따른 분류】                                             │
│  ──────────────────────────                                         │
│                                                                     │
│  0-주소 (스택 머신)    1-주소 (누산기)   2-주소 (GPR)    3-주소     │
│  ┌──────────────┐    ┌─────────────┐  ┌─────────────┐  ┌─────────┐ │
│  │   Opcode     │    │  Opcode     │  │  Opcode     │  │ Opcode  │ │
│  │              │    │  Operand    │  │  Op1   Op2  │  │OP1 OP2  │ │
│  │   (피연산자   │    │             │  │             │  │   OP3   │ │
│  │    스택에서)  │    └─────────────┘  └─────────────┘  └─────────┘ │
│  └──────────────┘                                                   │
│                                                                     │
│  PUSH A            LOAD A          ADD R1, R2      ADD R1, R2, R3  │
│  PUSH B            ADD  B          (R1 = R1 + R2)  (R1 = R2 + R3)  │
│  ADD               STORE C                                        │
│  POP C                                                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어 형식 선택 기준:                                          │ │
│  │  ────────────────────────                                        │ │
│  │  • 코드 크기: 0/1-주소 < 2-주소 < 3-주소                         │ │
│  │  • 레지스터 수: 3-주소 > 2-주소 > 1-주소                         │ │
│  │  • 메모리 접근: 0-주소(많음) > 1-주소 > 2/3-주소(적음)           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 대표적 ISA 명령어 형식

### ISA별 명령어 형식

```
┌─────────────────────────────────────────────────────────────────────┐
│                대표적 ISA 명령어 형식                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ARM (32비트 고정 길이)】                                          │
│  ───────────────────────                                            │
│                                                                     │
│  R-type (레지스터 연산):                                             │
│  ┌──────┬───┬───┬───┬───┬────┬──────┐                              │
│  │ Cond │OP│  S  │ Rn │ Rd │ Sh │ Rm │                              │
│  │ 4bit │4 │ 1  │ 4  │ 4  │ 5  │ 4  │ 32비트                        │
│  └──────┴───┴─────┴────┴────┴────┴──────┘                              │
│  예: ADD R0, R1, R2 (R0 = R1 + R2)                                  │
│                                                                     │
│  I-type (즉시값 연산):                                               │
│  ┌──────┬───┬───┬───┬───┬──────────────┐                            │
│  │ Cond │OP│  S  │ Rn │ Rd │  Imm12      │                            │
│  └──────┴───┴─────┴────┴────┴──────────────┘                            │
│  예: ADD R0, R1, #100                                               │
│                                                                     │
│  【x86 (가변 길이)】                                                 │
│  ──────────────────                                                  │
│  ┌────────┬────────┬────────┬────────┬────────┬────────┬─────────┐ │
│  │ Prefix │ Opcode │ ModR/M │  SIB   │Displace│ Imm    │         │ │
│  │ 0-4B   │  1-3B  │  0-1B  │  0-1B  │ 0-4B   │ 0-4B   │         │ │
│  └────────┴────────┴────────┴────────┴────────┴────────┴─────────┘ │
│  • 최소 1바이트 ~ 최대 15바이트                                       │
│  • 복잡하지만 코드 밀도 높음                                          │
│                                                                     │
│  【RISC-V (가변 길이)】                                              │
│  ──────────────────                                                  │
│  R-type:                                                             │
│  ┌───────┬─────┬─────┬─────┬─────┬───────┐                          │
│  │ funct7│ rs2 │ rs1 │funct3│ rd  │opcode │ 32비트                    │
│  │  7    │  5  │  5  │  3  │  5  │   7   │                          │
│  └───────┴─────┴─────┴─────┴─────┴───────┘                          │
│                                                                     │
│  I-type:                                                             │
│  ┌──────────────┬─────┬─────┬─────┬───────┐                         │
│  │    imm12     │ rs1 │funct3│ rd  │opcode │ 32비트                   │
│  └──────────────┴─────┴─────┴─────┴───────┘                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ISA별 특징 비교:                                                │ │
│  │  ──────────────────                                              │ │
│  │  • ARM: 고정 32비트, 조건부 실행, 정렬 필수                       │ │
│  │  • x86: 가변 길이, 높은 코드 밀도, 복잡한 디코딩                   │ │
│  │  • RISC-V: 기본 32비트 + 16비트 압축, 모듈형 ISA                  │ │
│  │  • MIPS: 고정 32비트, 단순한 3가지 형식                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 명령어 형식 설계 고려사항

### 설계 트레이드오프

```
┌─────────────────────────────────────────────────────────────────────┐
│                명령어 형식 설계 트레이드오프                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【고정 길이 vs 가변 길이】                                          │
│  ──────────────────────────                                         │
│                                                                     │
│  고정 길이 (ARM, MIPS):                                              │
│  ✓ 디코딩 단순, 파이프라인 효율                                      │
│  ✓ 빠른 명령어 fetch                                                 │
│  ✗ 코드 밀도 낮음 (낭비 발생)                                        │
│                                                                     │
│  가변 길이 (x86):                                                    │
│  ✓ 높은 코드 밀도                                                    │
│  ✓ 다양한 연산 표현 가능                                             │
│  ✗ 복잡한 디코딩, 파이프라인 bubble                                  │
│  ✗ 명령어 경계 탐색 필요                                             │
│                                                                     │
│  【레지스터 수 vs 필드 폭】                                          │
│  ──────────────────────────                                         │
│                                                                     │
│  레지스터 8개: 3비트 필드                                            │
│  레지스터 16개: 4비트 필드                                           │
│  레지스터 32개: 5비트 필드                                           │
│                                                                     │
│  → 레지스터 많을수록 명령어 길이 증가 또는                            │
│    피연산자 수 감소                                                  │
│                                                                     │
│  【필드 배치 순서】                                                  │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  효율적 배치:                                                     │ │
│  │  • Opcode: 최상위 비트 (빠른 디코딩)                              │ │
│  │  • 레지스터: 연속 배치 (한 번에 읽기)                             │ │
│  │  • Immediate: 하위 비트 (부호 확장 용이)                         │ │
│  │                                                               │ │
│  │  정렬 고려:                                                       │ │
│  │  • 바이트 경계 맞춤                                               │ │
│  │  • 워드 경계 맞춤 (메모리 접근 효율)                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 명령어 형식 분석/디버깅

```
┌─────────────────────────────────────────────────────────────────────┐
│                명령어 형식 분석/디버깅                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【objdump로 명령어 형식 확인】                                      │
│  ─────────────────────────────                                      │
│  $ objdump -d program                                               │
│  1000:  e08 0000 2   add r0, r0, r2   # ARM                         │
│  1004:  e28 00 00 64  add r0, r0, #100                               │
│                                                                     │
│  $ objdump -d -M intel program                                      │
│  1000:  01 c3        add ebx, eax       # x86                       │
│  1002:  81 c3 64 00 00 00  add ebx, 0x64                             │
│                                                                     │
│  【GDB로 명령어 디코딩】                                             │
│  ─────────────────────────                                          │
│  (gdb) x/i $pc                                                      │
│  => 0x1000 <main>: add r0, r0, r2                                   │
│                                                                     │
│  (gdb) x/1wx $pc                                                    │
│  0x1000: 0xe0800002    # 32비트 16진수                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어 형식 설계 체크리스트:                                     │ │
│  │  ────────────────────────────                                     │ │
│  │  □ 모든 연산을 표현할 수 있는가?                                  │ │
│  │  □ 모든 주소 지정 모드를 지원하는가?                              │ │
│  │  □ 레지스터 수가 충분한가?                                        │ │
│  │  □ 즉시값 범위가 적절한가?                                        │ │
│  │  □ 명령어 정렬이 올바른가?                                        │ │
│  │  □ 확장 가능성이 있는가?                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 명령어 형식: 명령어의 비트 필드 구조
• 구성: Opcode + Operand + Mode + 기타
• 주소 수: 0주소(스택) ~ 3주소(GPR)
• 길이: 고정(ARM/MIPS) vs 가변(x86)
• 트레이드오프: 디코딩 복잡도 vs 코드 밀도
```
