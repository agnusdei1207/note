# 192. 1-주소 명령어 (1-Address Instruction)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 누산기(accumulator)와 한 피연산자 주소를 사용하는 명령어
> 2. **가치**: 간단한 하드웨어, 작은 코드, 초기 컴퓨터 방식
> 3. **융합**: 8비트 마이크로프로세서, 임베디드, 교육용 CPU

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

1-주소 명령어(1-Address Instruction)는 **누산기(accumulator)와 하나의 피연산자 주소**를 사용하는 명령어 형식이다. 한 피연산자는 암묵적으로 누산기(AC)이며, 다른 피연산자만 명령어에 명시된다.

### 💡 비유: 계산기의 메모리 기능

1-주소 명령어는 **계산기의 메모리 기능**과 같다. 계산기의 "현재 값"(누산기)이 항상 있고, "메모리에서 불러오기"나 "메모리에 더하기"처럼 하나의 주소만 지정한다. "AC = AC + Mem[addr]" 형식이다.

### 1-주소 명령어 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                1-주소 명령어 구조                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조】                                                           │
│  ─────────                                                          │
│  ┌─────────────────────┬─────────────────────┐                      │
│  │       Opcode        │       Address       │                      │
│  │      (연산코드)      │      (피연산자)      │                      │
│  └─────────────────────┴─────────────────────┘                      │
│  (한쪽 피연산자는 암묵적으로 누산기)                                  │
│                                                                     │
│  【동작 원리】                                                       │
│  ─────────────                                                      │
│  • 누산기(AC): 항상 한쪽 피연산자                                    │
│  • 명령어: 다른 한쪽 피연산자 주소만 명시                            │
│  • 결과: 누산기에 저장                                               │
│  • 예: ADD addr → AC = AC + Mem[addr]                              │
│                                                                     │
│  【누산기 기반 연산】                                                │
│  ────────────────────────                                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어      동작                   의미                        │ │
│  │  ─────────  ─────────────────────  ───────────────────         │ │
│  │  LOAD addr  AC = Mem[addr]        메모리 → 누산기               │ │
│  │  STORE addr Mem[addr] = AC        누산기 → 메모리               │ │
│  │  ADD addr   AC = AC + Mem[addr]   더하기                        │ │
│  │  SUB addr   AC = AC - Mem[addr]   빼기                          │ │
│  │  AND addr   AC = AC & Mem[addr]   논리곱                        │ │
│  │  OR addr    AC = AC | Mem[addr]   논리합                        │ │
│  │  JUMP addr  PC = addr              분기                         │ │
│  │  JZ addr    if (AC==0) PC = addr  조건부 분기                   │ │
│  │  JNZ addr   if (AC!=0) PC = addr  조건부 분기                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예시: a = b + c】                                                 │
│  ──────────────────                                                  │
│  LOAD b          ; AC = b                                           │
│  ADD c           ; AC = AC + c = b + c                              │
│  STORE a         ; a = AC = b + c                                   │
│                                                                     │
│  【주소 수 비교】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  유형       명령어 수       장점                단점            │ │
│  │  ────────  ──────────────  ─────────────────  ──────────────   │ │
│  │  0-주소    6개 (push/pop)   코드 작음           속도 느림       │ │
│  │  1-주소    3개              단순                누산기 병목     │ │
│  │  2-주소    2개              레지스터 활용       명령어 김        │ │
│  │  3-주소    1개              최고 성능           명령어 가장 김   │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1-주소 장단점:                                                    │ │
│  │  ──────────────                                                    │ │
│  │  장점:                                                             │ │
│  │  • 명령어 길이 적당                                                │ │
│  │  • 하드웨어 단순 (누산기 하나)                                     │ │
│  │  • 레지스터 파일 불필요                                            │ │
│  │  • 초기 컴퓨터에 적합                                              │ │
│  │                                                               │ │
│  │  단점:                                                             │ │
│  │  • 누산기 병목 (한 번에 하나만)                                    │ │
│  │  • 메모리 트래픽 많음                                              │ │
│  │  • 임시 변수 저장 어려움                                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 1-주소 명령어 활용

### 활용 패턴

```
┌─────────────────────────────────────────────────────────────────────┐
│                1-주소 명령어 활용                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 산술 연산】                                                    │
│  ──────────────────                                                  │
│  // a = (b + c) - (d * e)                                           │
│  LOAD b          ; AC = b                                           │
│  ADD c           ; AC = b + c                                       │
│  STORE temp1     ; temp1 = b + c (중간 결과 저장)                   │
│  LOAD d          ; AC = d                                           │
│  MUL e           ; AC = d * e                                       │
│  STORE temp2     ; temp2 = d * e                                    │
│  LOAD temp1      ; AC = b + c                                       │
│  SUB temp2       ; AC = (b+c) - (d*e)                               │
│  STORE a         ; a = 결과                                         │
│  ; 9개 명령어 (중간 저장 필요)                                       │
│                                                                     │
│  【2. 메모리 간 이동】                                               │
│  ──────────────────────                                             │
│  // a = b                                                           │
│  LOAD b          ; AC = b                                           │
│  STORE a         ; a = AC                                           │
│  ; 2개 명령어                                                        │
│                                                                     │
│  【3. 조건부 분기】                                                  │
│  ──────────────────                                                  │
│  // if (a > 0) goto label                                           │
│  LOAD a          ; AC = a                                           │
│  JZ skip         ; if (AC == 0) goto skip                           │
│  JN skip         ; if (AC < 0) goto skip                            │
│  JUMP label      ; AC > 0이면 분기                                  │
│  skip:                                                              │
│                                                                     │
│  【4. 루프】                                                         │
│  ──────────────────                                                  │
│  // for (i=0; i<10; i++) sum += arr[i]                              │
│  LOAD #0                                                            │
│  STORE i         ; i = 0                                            │
│  STORE sum       ; sum = 0                                          │
│  loop:                                                              │
│  LOAD i          ; AC = i                                           │
│  SUB #10        ; AC = i - 10                                       │
│  JGE done        ; if (i >= 10) goto done                           │
│  LOAD sum        ; AC = sum                                         │
│  ADD arr[i]      ; AC = sum + arr[i]  (인덱싱 필요)                 │
│  STORE sum       ; sum = AC                                         │
│  LOAD i          ; AC = i                                           │
│  ADD #1          ; AC = i + 1                                       │
│  STORE i         ; i = AC                                           │
│  JUMP loop                                                          │
│  done:                                                              │
│                                                                     │
│  【8051 (1-주소 기반) 예시】                                         │
│  ────────────────────────────                                       │
│  MOV A, #10       ; A = 10 (누산기 = 10)                            │
│  ADD A, #20       ; A = A + 20                                      │
│  MOV result, A    ; result = A                                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  누산기 머신 (Accumulator Machine):                                │ │
│  │  ──────────────────────────────────                               │ │
│  │  • 대표: 8080, 8086, 6502, 8051                                   │ │
│  │  • 특징: A 레지스터(누산기) 중심                                   │ │
│  │  • 장점: 하드웨어 단순, 비용 저렴                                  │ │
│  │  • 단점: 성능 제한 (누산기 병목)                                   │ │
│  │                                                               │ │
│  │  ; 8086 예시 (AX = 누산기)                                         │ │
│  │  MOV AX, [a]        ; AX = a                                      │ │
│  │  ADD AX, [b]        ; AX = AX + b                                 │ │
│  │  MOV [c], AX        ; c = AX                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 1-주소 명령어 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                1-주소 명령어 분석                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【8051 어셈블리】                                                   │
│  ──────────────────                                                  │
│  ; int sum_array(int *arr, int n)                                   │
│  ; R0 = arr, R1 = n                                                 │
│  MOV A, #0        ; sum = 0                                         │
│  MOV R2, A        ; sum을 R2에                                      │
│  loop:                                                              │
│  MOV A, R1        ; n                                               │
│  JZ done          ; if (n == 0) done                                │
│  MOV A, @R0       ; *arr                                            │
│  ADD A, R2        ; sum += *arr                                     │
│  MOV R2, A        ; R2 = sum                                        │
│  INC R0           ; arr++                                           │
│  DEC R1           ; n--                                             │
│  SJMP loop                                                          │
│  done:                                                              │
│  MOV A, R2        ; 반환값                                          │
│  RET                                                                │
│                                                                     │
│  【6502 어셈블리】                                                   │
│  ──────────────────                                                  │
│  ; A = A + memory                                                   │
│  LDA value1       ; A = value1                                      │
│  CLC              ; 캐리 클리어                                      │
│  ADC value2       ; A = A + value2                                  │
│  STA result       ; result = A                                      │
│                                                                     │
│  【x86 16비트 (1-주소 스타일)】                                       │
│  ────────────────────────────────                                   │
│  MOV AX, [b]      ; AX = b (AX = 누산기)                            │
│  ADD AX, [c]      ; AX += c                                         │
│  MOV [a], AX      ; a = AX                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1-주소 머신 분석 팁:                                               │ │
│  │  ────────────────────────                                          │ │
│  │  1. 누산기가 어디에 있는지 파악                                     │ │
│  │     8051: A, 6502: A, 8086: AX/EAX                               │ │
│  │                                                               │ │
│  │  2. 중간 결과 저장 위치 확인                                       │ │
│  │     메모리, 다른 레지스터, 스택                                    │ │
│  │                                                               │ │
│  │  3. 조건부 분기 조건 확인                                          │ │
│  │     누산기 상태 (0, 음수, 캐리)                                    │ │
│  │                                                               │ │
│  │  4. 인덱싱 방식 확인                                               │ │
│  │     간접 주소, 인덱스 레지스터                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 1-주소 명령어: 누산기 + 한 피연산자 주소
• 구조: Opcode + Address
• 동작: AC = AC op Mem[addr]
• 장점: 하드웨어 단순, 비용 저렴
• 단점: 누산기 병목, 메모리 트래픽 많음
• 활용: 8051, 6502, 초기 마이크로프로세서
```
