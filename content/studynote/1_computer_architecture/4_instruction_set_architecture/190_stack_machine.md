# 190. 스택 머신 (Stack Machine)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연산을 스택 기반으로 수행하는 아키텍처
> 2. **가치**: 간단한 명령어, 작은 코드, JVM/WASM의 기반
> 3. **융합**: 컴파일러 중간 표현, 역폴란드 표기법, 가상 머신

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

스택 머신(Stack Machine)은 **피연산자를 스택에서 관리하고, 연산 결과도 스택에 저장**하는 아키텍처다. 레지스터 머신과 달리 피연산자 주소를 명시하지 않아 명령어가 짧고 단순하다.

### 💡 비유: 접시 쌓기로 계산하기

스택 머신은 **접시 쌓기로 계산하는 것**과 같다. "3 + 4"를 계산하려면: 1) 접시에 3을 적어 쌓고, 2) 접시에 4를 적어 쌓고, 3) 위에서 두 접시를 꺼내 더해서, 4) 결과를 새 접시에 담아 쌓는다. 모든 계산이 접시 더미(스택) 위에서 일어난다.

### 스택 머신 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                스택 머신 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【특징】                                                           │
│  ─────────                                                          │
│  • 피연산자: 스택에서 자동으로 가져옴                                │
│  • 결과: 스택에 자동으로 저장                                        │
│  • 명령어: 0-주소 (피연산자 주소 없음)                               │
│  • 간단함: 레지스터 할당 불필요                                      │
│                                                                     │
│  【기본 연산】                                                       │
│  ─────────────                                                      │
│                                                                     │
│  PUSH value    ; 스택에 값 push                                      │
│  POP           ; 스택에서 값 pop                                     │
│  ADD           ; top 2개를 pop, 더해서 push                          │
│  SUB           ; top 2개를 pop, 빼서 push                            │
│  MUL           ; top 2개를 pop, 곱해서 push                          │
│  DIV           ; top 2개를 pop, 나눠서 push                          │
│                                                                     │
│  【예시: 3 + 4】                                                     │
│  ──────────────────                                                  │
│                                                                     │
│  1. PUSH 3     ; 스택: [3]                                          │
│  2. PUSH 4     ; 스택: [3, 4]                                       │
│  3. ADD        ; 스택: [7]                                          │
│                                                                     │
│  【스택 상태 변화】                                                  │
│  ────────────────────                                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어      스택 변화          설명                              │ │
│  │  ─────────  ─────────────────  ──────────────────────          │ │
│  │  초기       [ ]                 빈 스택                          │ │
│  │  PUSH 3    [3]                  3 push                          │ │
│  │  PUSH 4    [3, 4]               4 push                          │ │
│  │  ADD       [7]                  pop 3,4; push 7                │ │
│  │  PUSH 2    [7, 2]               2 push                          │ │
│  │  MUL       [14]                 pop 7,2; push 14               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【역폴란드 표기법 (RPN)】                                            │
│  ────────────────────────────                                       │
│                                                                     │
│  중위 표기법: 3 + 4 * 2  →  스택 머신 실행 순서:                     │
│  후위 표기법: 3 4 2 * +   →  PUSH 3, PUSH 4, PUSH 2, MUL, ADD       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스택 머신 vs 레지스터 머신:                                       │ │
│  │  ──────────────────────────────────                              │ │
│  │  스택 머신:                                                        │ │
│  │  PUSH 3, PUSH 4, ADD        ; 3개 명령어                         │ │
│  │                                                               │ │
│  │  레지스터 머신:                                                    │ │
│  │  MOV R0, #3                 ; 2개 명령어                         │ │
│  │  ADD R0, R0, #4                                                  │ │
│  │                                                               │ │
│  │  스택: 코드 크기 작음, 속도 느림                                    │ │
│  │  레지스터: 코드 크기 큼, 속도 빠름                                   │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 스택 머신 명령어

### 명령어 상세

```
┌─────────────────────────────────────────────────────────────────────┐
│                스택 머신 명령어                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【0-주소 명령어 (스택 머신)】                                        │
│  ──────────────────────────────                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어      동작                 스택 변화                       │ │
│  │  ─────────  ───────────────────  ─────────────────────         │ │
│  │  PUSH n    push n               [..., n]                       │ │
│  │  POP       pop top              [...]                          │ │
│  │  DUP       duplicate top        [..., a, a]                    │ │
│  │  SWAP      swap top 2           [..., b, a]  ← [..., a, b]     │ │
│  │                                                               │ │
│  │  ADD       pop 2, push sum      [..., a+b]  ← [..., a, b]      │ │
│  │  SUB       pop 2, push diff     [..., a-b]  ← [..., a, b]      │ │
│  │  MUL       pop 2, push prod     [..., a*b]  ← [..., a, b]      │ │
│  │  DIV       pop 2, push quot     [..., a/b]  ← [..., a, b]      │ │
│  │  MOD       pop 2, push rem      [..., a%b]  ← [..., a, b]      │ │
│  │                                                               │ │
│  │  AND       pop 2, push and      [..., a&b]                     │ │
│  │  OR        pop 2, push or       [..., a|b]                     │ │
│  │  XOR       pop 2, push xor      [..., a^b]                     │ │
│  │  NOT       pop 1, push not      [..., ~a]                      │ │
│  │                                                               │ │
│  │  LOAD      pop addr, push mem   [..., mem[addr]]               │ │
│  │  STORE    pop val,addr, store   [...]                          │ │
│  │                                                               │ │
│  │  CALL addr push PC, jump        [..., PC]                      │ │
│  │  RET       pop PC, return       [...]                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【JVM 바이트코드 예시】                                              │
│  ──────────────────────────                                         │
│  ; int add(int a, int b) { return a + b; }                         │
│  ; JVM 바이트코드                                                    │
│  iload_0      ; 지역변수 0 (a)를 스택에 push                         │
│  iload_1      ; 지역변수 1 (b)를 스택에 push                         │
│  iadd         ; top 2개를 pop, 더해서 push                          │
│  ireturn      ; top을 pop해서 반환                                  │
│                                                                     │
│  【WebAssembly 예시】                                                │
│  ────────────────────────                                           │
│  ; (i32.add (local.get 0) (local.get 1))                           │
│  local.get 0      ; 스택에 push                                     │
│  local.get 1      ; 스택에 push                                     │
│  i32.add          ; 더하기                                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Forth 스택 머신 (실제 구현):                                      │ │
│  │  ────────────────────────────                                     │ │
│  │  : SQUARE ( n -- n*n )   DUP * ;                                 │ │
│  │  ; DUP으로 복제, *로 제곱                                          │ │
│  │                                                               │ │
│  │  : FACTORIAL ( n -- n! )                                         │ │
│  │    DUP 1 > IF                                                   │ │
│  │      DUP 1- RECURSE *                                            │ │
│  │    THEN ;                                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 스택 머신 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                스택 머신 분석                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【JVM 바이트코드 분석】                                             │
│  ──────────────────────────                                         │
│  $ javap -c HelloWorld.class                                       │
│  public static void main(java.lang.String[]);                       │
│    Code:                                                            │
│      0: getstatic     #2  // Field java/lang/System.out:L...       │
│      3: ldc           #3  // String Hello                          │
│      5: invokevirtual #4  // Method println:(Ljava/lang/String;)V  │
│      8: return                                                       │
│                                                                     │
│  스택 상태:                                                         │
│  0: [] → [out]                                                      │
│  3: [out] → [out, "Hello"]                                         │
│  5: [] (메서드 호출로 소비)                                          │
│                                                                     │
│  【WebAssembly 텍스트 형식】                                         │
│  ───────────────────────────────                                    │
│  (module                                                            │
│    (func $add (param $a i32) (param $b i32) (result i32)            │
│      local.get $a                                                   │
│      local.get $b                                                   │
│      i32.add)                                                       │
│    (export "add" (func $add)))                                      │
│                                                                     │
│  【C to 스택 머신 변환】                                             │
│  ──────────────────────────                                         │
│  // C: (a + b) * (c - d)                                            │
│  PUSH a      ; [a]                                                  │
│  PUSH b      ; [a, b]                                               │
│  ADD         ; [a+b]                                                │
│  PUSH c      ; [a+b, c]                                             │
│  PUSH d      ; [a+b, c, d]                                          │
│  SUB         ; [a+b, c-d]                                           │
│  MUL         ; [(a+b)*(c-d)]                                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  JIT 컴파일 (스택 → 레지스터):                                      │ │
│  │  ─────────────────────────────────                                │ │
│  │  ; JVM: iload_0, iload_1, iadd                                   │ │
│  │  ; JIT compiled (x86-64):                                         │ │
│  │  MOV EAX, [local_0]                                               │ │
│  │  ADD EAX, [local_1]                                               │ │
│  │                                                               │ │
│  │  → 스택 머신을 레지스터 머신으로 변환                               │ │
│  │  → 실제 하드웨어 성능 활용                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 스택 머신: 스택 기반 연산 아키텍처
• 명령어: 0-주소 (피연산자 주소 없음)
• 장점: 코드 크기 작음, 컴파일러 단순
• 단점: 메모리 접근 많음, 성능 느림
• 활용: JVM, WebAssembly, Forth
• 최적화: JIT로 레지스터 머신 변환
```
