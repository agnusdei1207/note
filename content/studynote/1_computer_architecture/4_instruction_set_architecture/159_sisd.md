> **핵심 인사이트**
> 1. SISD는 플린의 분류(Flynn's Taxonomy) 중 가장 원초적이고 전통적인 형태의 컴퓨터 아키텍처로, **한 번에 딱 1개의 명령어(Instruction)로 딱 1개의 데이터(Data)만을 순차적으로 처리하는 구조**이다.
> 2. 폰 노이만이 처음 제안한 컴퓨터의 본질 그대로이며, 프로그램 카운터(PC)가 1개만 존재하여 코드가 마치 시냇물처럼 위에서 아래로 정직하게 흘러간다.
> 3. 멀티코어나 GPU 같은 거시적 병렬성은 없지만, 이 SISD 구조 내부에서도 파이프라이닝이나 수퍼스칼라를 도입해 눈물겨운 **명령어 수준 병렬성(ILP)**을 쥐어짜 내는 것이 과거 인텔 CPU 진화의 역사였다.

---

## Ⅰ. 가장 정직한 직렬의 미학
"더해라(명령어), 1과 2를(데이터)!"
우리가 프로그래밍을 처음 배울 때 머릿속에 그리는 컴퓨터의 작동 방식이 바로 완벽한 **SISD(Single Instruction Single Data)**입니다.

SISD 아키텍처에는 명령어를 지시하는 **제어 유닛(Control Unit)**이 딱 1개 있습니다. 그리고 실제 계산을 하는 **연산장치(ALU)**도 1개(또는 논리적 1세트)만 있습니다.
CPU는 메모리에서 명령어 하나를 가져오고, 그 명령에 적힌 변수 데이터 하나를 가져와서 덧셈을 끝낸 뒤, 다시 다음 줄로 넘어가는 철저한 **직렬(Serial) 순차 처리**를 수행합니다. 

```ascii
[SISD 아키텍처의 단순 무식한 데이터 흐름]

 [ 제어 유닛(CU) ]
       | (딱 1개의 명령 지시: ADD!)
       V
 [ 연산 장치(ALU) ] <--- [ 데이터 풀(메모리) ] 에서 딱 1개의 데이터 세트(A, B)만 가져옴
       |
       V
 [ 딱 1개의 결과(C) 도출 후 다음 명령어로 얌전히 이동 ]
```

📢 **섹션 요약 비유**: SISD는 '정통 수제 햄버거집 장인'입니다. 주문서(명령어)가 들어오면, 장인 1명(CU)이 빵 1개와 패티 1장(데이터)을 도마(ALU)에 올리고 정성스럽게 햄버거 1개를 완성합니다. 이 햄버거가 다 만들어져 손님에게 나가기 전까지는 절대 다음 햄버거 주문서를 쳐다보지도 않는 아주 고집스럽고 정직한 장인입니다.

## Ⅱ. 폰 노이만 병목의 영원한 희생양
SISD는 설계가 너무나도 아름답고 프로그래머가 코드 짜기(디버깅)가 우주에서 제일 쉽습니다. 그냥 위에서부터 아래로 읽어 내려가면 오류를 금방 찾을 수 있기 때문입니다.

하지만 이 구조는 성능 면에서 끔찍한 저주를 받았습니다.
데이터 100만 개가 들어있는 배열에 +1을 해주는 코드를 짰다고 칩시다. SISD는 `LOAD`, `ADD`, `STORE` 라는 3개의 명령어를 무려 **100만 번이나 반복(Loop)**해야 합니다. 
100만 번 동안 하나뿐인 메모리 버스를 쉴 새 없이 오가야 하므로, CPU가 아무리 5GHz로 빨라도 메모리에서 데이터가 올 때까지 하루 종일 기다려야 하는 **폰 노이만 병목(Von Neumann Bottleneck)의 타격을 가장 직격으로 맞는 아키텍처**입니다. 

```ascii
[SISD의 지옥 같은 반복문(Loop) 노가다]

 For i = 0 to 1,000,000:
    Array[i] = Array[i] + 1;

 -> 컴퓨터: "1번 배열 가져와! 더해! 저장해!" (100클럭 소모)
 -> 컴퓨터: "하아.. 2번 배열 가져와! 더해! 저장해!" (또 100클럭 소모)
 => 100만 번을 1개씩 이 짓거리를 함. 전체 시스템 숨넘어감.
```

📢 **섹션 요약 비유**: SISD의 노가다는 '모래알 100만 개 옮기기'입니다. 아무리 달리기(클럭)가 빠른 사람이라도, 숟가락(1개의 데이터)으로 모래알을 한 번에 한 개씩만 떠서 100만 번을 왕복해서 옮기려면 며칠이 걸립니다. 이것이 단일 명령어 단일 데이터의 태생적 슬픔입니다.

## Ⅲ. 아키텍처 관점: SISD 내부의 피눈물 나는 병렬화 (ILP)
1990년대 인텔 펜티엄이나 초기 ARM 칩들은 모두 이 SISD 구조였습니다. 코어 개수를 늘리는 멀티코어(MIMD)의 개념이 없던 시절, 엔지니어들은 이 **"단 1개의 코어(SISD) 안에서 어떻게든 속도를 쥐어짜 내야 한다!"**는 강박에 시달렸습니다.

그 결과, 겉보기엔 명령어 하나씩 실행하는 SISD인 척하면서, **내부적으로는 몰래 파이프라인을 15단계로 쪼개고(Pipelining), 덧셈기 4개를 병렬로 돌리며(Superscalar), 순서를 제멋대로 섞어서(비순차적 실행, OoOE) 실행하는 어마어마한 꼼수 기술(ILP, 명령어 수준 병렬성)**들을 모조리 개발해 냈습니다.
즉, 현대의 싱글 코어는 완벽히 순진한 고전적 의미의 SISD가 아니라, 마이크로아키텍처 수준에서 극한의 병렬 도핑 약물을 맞은 근육질 SISD라고 볼 수 있습니다.

```ascii
[고전적 SISD vs 현대적 SISD(수퍼스칼라)의 진화]

 [고전적 SISD] (단일 사이클)
 명령 1 끝 -> 명령 2 끝 -> 명령 3 끝 (정직하고 느림)

 [현대적 1코어 SISD] (수퍼스칼라 + 파이프라인)
 겉으론 PC가 1개라 SISD로 분류되지만..
 내부: [명령 1, 2, 3, 4] 를 한 번에 삼켜서 
       ALU 4개에 동시에 던져버림!! (속으론 미친듯이 병렬작업 중)
```

📢 **섹션 요약 비유**: 현대적 SISD는 '겉으론 1명인 척하는 4인조 복면 로봇'입니다. 바깥에서 점장님(분류 기준)이 볼 때는 일꾼이 1명(단일 제어 유닛) 출근한 게 맞습니다. 그런데 이 일꾼이 작업실 안에 들어가면, 몰래 숨겨온 팔 4개(수퍼스칼라)를 쫙 펴고 4인분의 일을 동시에 팍팍 해치우고 퇴근할 때는 다시 팔을 숨기고 모범생처럼 나가는 흉악한 사기 기믹입니다.

## 개념 맵 (Concept Map)
```ascii
[SISD (Single Instruction Single Data)]
   │
   ├── 정의: 1개의 제어 유닛(CU)이 1개의 데이터 스트림을 순차적(Sequential)으로 처리하는 폰 노이만 아키텍처의 원형
   │
   ├── 특징과 한계
   │     ├── 특징: 결정론적(Deterministic) 실행, 프로그래밍 및 디버깅 용이
   │     └── 한계: 대규모 데이터 처리 시 반복문 오버헤드로 인한 극심한 폰 노이만 병목(Bottleneck) 직면
   │
   └── 시스템 진화의 발자취
         └── 거시적 병렬성(멀티코어)이 부재했던 시절, 성능 향상을 위해 파이프라이닝, Superscalar, OoOE 등 ILP(명령어 병렬성) 기술의 극한 발전을 촉발시킨 아키텍처 토대
```

## 어린이 비유
SISD는 가장 정직하고 성실한 **외길 꼬마 우체부**예요! 대장님이 "1번 집, 2번 집, 3번 집에 편지를 주고 와!"라고 명령을 내리면, 꼬마 우체부는 무조건 1번 집에 가서 편지 한 통을 예쁘게 넣고, 다시 돌아와서 2번 집 편지를 꺼내 가는 착한 아이죠. 하지만 너무 정직해서 요령이 없다 보니까, 아파트 100가구에 편지를 다 넣으려면 밤을 꼴딱 새워야 하는 단점이 있었답니다!
