# 188. 무조건 분기 (Unconditional Branch)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 조건 없이 항상 분기하는 명령어
> 2. **가치**: 루프, 함수 호출, 예외 처리의 기반
> 3. **융합**: goto, break, continue, 간접 분기

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

무조건 분기(Unconditional Branch)는 **조건 검사 없이 항상 지정된 주소로 분기**하는 명령어다. JMP, BRA 등이 있으며, 루프 구현, 함수 종료, 예외 처리 등에 사용된다.

### 💡 비유: 일방통행 도로

무조건 분기는 **일방통행 도로**와 같다. 신호등이나 표지판을 확인할 필요 없이 무조건 지정된 방향으로만 간다. 목적지에 도착하면 다시 돌아올 수도 있고(CALL/RET), 아예 새로운 곳으로 갈 수도 있다(JMP).

### 무조건 분기 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                무조건 분기 구조                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동작 원리】                                                       │
│  ─────────────                                                      │
│                                                                     │
│  PC = 목적지 주소 (즉시 변경)                                        │
│                                                                     │
│  【종류】                                                           │
│  ─────────                                                          │
│                                                                     │
│  1. 직접 분기: 목적지 주소가 명령어에 포함                           │
│     JMP label                                                       │
│                                                                     │
│  2. 간접 분기: 목적지 주소가 레지스터/메모리에 저장                   │
│     JMP EAX                                                         │
│     JMP [table + EAX*4]                                             │
│                                                                     │
│  3. PC 상대 분기: 현재 PC에서 상대적 위치                            │
│     JMP +0x100                                                      │
│                                                                     │
│  【ISA별 문법】                                                     │
│  ──────────────                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ISA      직접 분기       간접 분기         특징                 │ │
│  │  ───────  ─────────────  ────────────────  ──────────────      │ │
│  │  x86      JMP label      JMP EAX          다양한 형식           │ │
│  │           JMP 0x1000     JMP [mem]                              │ │
│  │                                                               │ │
│  │  ARM      B label        BX R0            PC 상대, 레지스터    │ │
│  │           BAL label      BLX R0           조건부 가능           │ │
│  │                                                               │ │
│  │  MIPS     J target       JR $t0           26비트 타겟           │ │
│  │           JAL target     JALR $t0         지연 슬롯             │ │
│  │                                                               │ │
│  │  RISC-V   JAL rd, imm    JALR rd, rs1, 0  20비트 오프셋         │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【분기 vs 호출】                                                    │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  JMP                    CALL                                   │ │
│  │  ──────────────────     ──────────────────────                 │ │
│  │  PC = 목적지            PC = 목적지                             │ │
│  │  돌아오지 않음          반환 주소 저장                           │ │
│  │  goto, break            함수 호출                               │ │
│  │                                                               │ │
│  │  JMP label              CALL func                              │ │
│  │  ; 계속 label에서        ; RET으로 돌아옴                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  무조건 분기 활용:                                                 │ │
│  │  ──────────────────────                                           │ │
│  │  • 루프: JMP로 반복                                               │ │
│  │  • switch: jump table로 간접 분기                                │ │
│  │  • break/continue: 루프 탈출/재시작                              │ │
│  │  • goto: 무조건 이동                                             │ │
│  │  • 함수 포인터: 간접 분기                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 무조건 분기 활용

### 활용 패턴

```
┌─────────────────────────────────────────────────────────────────────┐
│                무조건 분기 활용                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 루프 구현】                                                    │
│  ──────────────────                                                  │
│  ; while (1) { ... }   (무한 루프)                                  │
│  loop:                                                              │
│      ; ... 루프 본문                                                │
│      JMP loop                                                       │
│                                                                     │
│  ; for 루프                                                          │
│      XOR ECX, ECX        ; i = 0                                    │
│  loop:                                                              │
│      CMP ECX, 10         ; i < 10?                                  │
│      JGE done                                                         │
│      ; ... 루프 본문                                                │
│      INC ECX            ; i++                                       │
│      JMP loop            ; 반복                                     │
│  done:                                                              │
│                                                                     │
│  【2. break/continue】                                               │
│  ────────────────────────                                           │
│  ; while (i < n) {                                                  │
│  ;     if (arr[i] == 0) break;                                      │
│  ;     if (arr[i] < 0) continue;                                    │
│  ;     process(arr[i]);                                             │
│  ; }                                                                │
│  loop:                                                              │
│      CMP ECX, EDX        ; i < n?                                   │
│      JGE done             ; break (루프 종료)                       │
│      CMP [RBX+RCX*4], 0  ; arr[i] == 0?                             │
│      JE done             ; break                                    │
│      CMP [RBX+RCX*4], 0  ; arr[i] < 0?                              │
│      JL skip             ; continue                                 │
│      ; ... process(arr[i])                                         │
│  skip:                                                              │
│      INC ECX                                                        │
│      JMP loop                                                       │
│  done:                                                              │
│                                                                     │
│  【3. switch-case (jump table)】                                     │
│  ──────────────────────────────────                                 │
│  ; switch (eax) {                                                   │
│  ;     case 0: ... break;                                           │
│  ;     case 1: ... break;                                           │
│  ;     default: ...                                                 │
│  ; }                                                                │
│      CMP EAX, max_case                                               │
│      JA  default                                                     │
│      JMP [jump_table + EAX*4]                                       │
│  .case0:                                                            │
│      ; ...                                                          │
│      JMP done                                                        │
│  .case1:                                                            │
│      ; ...                                                          │
│      JMP done                                                        │
│  default:                                                           │
│      ; ...                                                          │
│  done:                                                              │
│                                                                     │
│  【4. 함수 포인터 (간접 분기)】                                       │
│  ───────────────────────────────                                    │
│  ; void (*func)(int) = handler;                                     │
│  ; func(42);                                                        │
│  CALL [func_ptr]          ; 간접 호출                               │
│  ; 또는                                                             │
│  MOV EAX, [func_ptr]                                                │
│  CALL EAX                 ; 레지스터 간접 호출                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  goto 구현 (드물지만 사용):                                        │ │
│  │  ────────────────────────────                                     │ │
│  │  // C: goto error_handler;                                        │ │
│  │  JMP error_handler                                                │ │
│  │                                                               │ │
│  │  // C: goto *labels[i]; (computed goto)                          │ │
│  │  JMP [label_table + RAX*8]                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 무조건 분기 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                무조건 분기 × 타 과목 융합                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                        │
│  • break/continue → JMP                                             │
│  • goto → JMP                                                        │
│  • switch → jump table (간접 JMP)                                   │
│  • 루프 → 조건부 분기 + JMP                                          │
│                                                                     │
│  【운영체제】                                                        │
│  • 컨텍스트 스위치: JMP로 태스크 전환                                │
│  • 에러 처리: 에러 핸들러로 JMP                                      │
│  • 부트로더: 커널 진입점으로 JMP                                     │
│  • 인터럽트: 핸들러로 분기                                           │
│                                                                     │
│  【컴퓨터 구조】                                                     │
│  • 파이프라인: 무조건 분기도 예측 필요                               │
│  • BTB: 분기 목표 캐시                                               │
│  • 간접 분기: 예측 더 어려움                                         │
│  • 성능: 직접 분기 < 간접 분기 비용                                  │
│                                                                     │
│  【프로그래밍 언어】                                                 │
│  • C goto: JMP                                                       │
│  • computed goto: 간접 JMP                                           │
│  • setjmp/longjmp: 비지역 분기                                       │
│  • 예외: 예외 핸들러로 JMP                                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  직접 분기 vs 간접 분기 성능:                                      │ │
│  │  ───────────────────────────────                                  │ │
│  │  직접 분기:                                                        │ │
│  │  • 목적지가 명령어에 고정                                          │ │
│  │  • BTB에서 잘 예측됨                                               │ │
│  │  • 1-2 클럭                                                        │ │
│  │                                                               │ │
│  │  간접 분기:                                                        │ │
│  │  • 목적지가 런타임에 결정                                          │ │
│  │  • 예측 어려움 (가상 함수, 함수 포인터)                            │ │
│  │  • 5-10+ 클럭                                                     │ │
│  │                                                               │ │
│  │  최적화:                                                           │ │
│  │  • 디버터치(devirtualization): 가상 함수 → 직접 호출              │ │
│  │  • 인라이닝: 함수 호출 제거                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 무조건 분기 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                무조건 분기 분석                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【objdump 분석】                                                    │
│  ──────────────────                                                  │
│  $ objdump -d program                                               │
│  1000:  eb 05           jmp 1007 <.loop>                            │
│  1002:  ...               ; 건너뛴 코드                              │
│  1007:  ...               ; .loop                                   │
│                                                                     │
│  ; 간접 분기                                                         │
│  2000:  ff 20           jmp DWORD PTR [eax]                         │
│  2002:  ff e0           jmp eax                                     │
│                                                                     │
│  【GDB 디버깅】                                                      │
│  ──────────────────                                                  │
│  (gdb) si                 ; JMP 실행                                │
│  (gdb) p/x $pc            ; 분기 후 PC 확인                          │
│  (gdb) break *0x1007      ; 분기 목적지에 브레이크포인트             │
│                                                                     │
│  【jump table 분석】                                                 │
│  ────────────────────                                               │
│  $ objdump -s -j .rodata program                                   │
│  ; jump table 내용 확인                                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  분기 최적화 팁:                                                    │ │
│  │  ──────────────                                                    │ │
│  │  1. 불필요한 JMP 제거 (dead code elimination)                     │ │
│  │  2. JMP 체인 단축 (tail merging)                                   │ │
│  │     JMP a → a: JMP b → JMP b 로 최적화                             │ │
│  │  3. 간접 분기 → 직접 분기 (devirtualization)                       │ │
│  │  4. 루프 언롤링으로 JMP 감소                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 무조건 분기: 조건 없이 항상 분기
• 직접 분기: 목적지가 명령어에 포함
• 간접 분기: 목적지가 레지스터/메모리
• 활용: 루프, break/continue, goto, switch
• 성능: 직접 분기 < 간접 분기
```
