# 168. 제로 플래그 (Zero Flag)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연산 결과가 0인지 여부를 나타내는 상태 플래그
> 2. **가치**: 같음 비교, 루프 종료, 조건부 분기의 핵심 지표
> 3. **융합**: 컴파일러 최적화, OS 스케줄링, 알고리즘 조건 검사

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

제로 플래그(ZF, Zero Flag)는 **연산 결과가 0인지 여부를 나타내는 플래그 비트**다. 결과가 0이면 ZF=1, 0이 아니면 ZF=0으로 설정된다. 조건부 분기에서 가장 자주 사용되는 플래그다.

### 💡 비유: 빈 상자 표시등

ZF는 **빈 상자 표시등**과 같다. 물건을 꺼내서 상자가 비면(결과=0) 표시등이 켜진다(ZF=1). 무언가 남아있으면 표시등이 꺼진다(ZF=0).

### ZF 동작 원리

```
┌─────────────────────────────────────────────────────────────────────┐
│                ZF 동작 원리                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ZF 설정 규칙】                                                    │
│  ────────────────                                                    │
│                                                                     │
│  연산 결과 == 0  →  ZF = 1                                          │
│  연산 결과 != 0  →  ZF = 0                                          │
│                                                                     │
│  【ZF 설정 명령어】                                                  │
│  ──────────────────────                                             │
│  • 산술: ADD, SUB, MUL, DIV (결과에 따라)                            │
│  • 논리: AND, OR, XOR, NOT                                          │
│  • 비교: CMP, TST, TEQ (결과 버림, 플래그만)                         │
│  • 증감: INC, DEC                                                   │
│  • 이동: MOV (일부 아키텍처)                                        │
│                                                                     │
│  【ZF 예시】                                                         │
│  ────────────                                                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  연산              결과    ZF    설명                           │ │
│  │  ────────────────  ─────  ────  ────────────────────           │ │
│  │  SUB R0, R0, R0    0      1     R0 - R0 = 0                    │ │
│  │  SUB R1, #5, #5    0      1     5 - 5 = 0                      │ │
│  │  AND R2, #0, #255  0      1     0 AND 255 = 0                  │ │
│  │  MOV R3, #0        0      1     0 이동 (ARM)                    │ │
│  │                                                               │ │
│  │  ADD R0, #1, #2    3      0     1 + 2 = 3                      │ │
│  │  SUB R1, #5, #3    2      0     5 - 3 = 2                      │ │
│  │  AND R2, #7, #8    0      1     7 AND 8 = 0                    │ │
│  │  OR R3, #0, #1     1      0     0 OR 1 = 1                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CMP와 ZF】                                                        │
│  ──────────────                                                      │
│  CMP R1, R2  ; R1 - R2 (결과 버림)                                  │
│                                                                     │
│  조건           ZF    분기 명령어                                    │
│  ─────────────  ────  ────────────                                  │
│  R1 == R2       1     BEQ (Branch if Equal)                         │
│  R1 != R2       0     BNE (Branch if Not Equal)                     │
│                                                                     │
│  예:                                                                │
│  CMP R0, #10     ; R0 - 10                                          │
│  BEQ equal       ; R0 == 10이면 equal로 점프                        │
│  BNE not_equal   ; R0 != 10이면 not_equal로 점프                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ZF 위치:                                                        │ │
│  │  ──────────                                                      │ │
│  │  x86: EFLAGS 비트 6                                               │ │
│  │  ARM: CPSR 비트 30                                                │ │
│  │  RISC-V: 없음 (비교 결과를 레지스터에 저장)                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. ZF 활용 패턴

### ZF 활용 시나리오

```
┌─────────────────────────────────────────────────────────────────────┐
│                ZF 활용 패턴                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 같음 비교 (Equality Check)】                                   │
│  ──────────────────────────────────                                 │
│  // C 코드                                                          │
│  if (a == b) { ... }                                                │
│                                                                     │
│  // 어셈블리                                                        │
│  CMP R0, R1        ; R0 - R1                                        │
│  BEQ equal_label   ; ZF=1이면 분기                                  │
│                                                                     │
│  【2. 0 검사 (Zero Test)】                                           │
│  ──────────────────────────                                         │
│  // C 코드                                                          │
│  if (ptr == NULL) { ... }                                           │
│                                                                     │
│  // 어셈블리                                                        │
│  CMP R0, #0        ; R0 - 0                                         │
│  BEQ null_handler  ; ZF=1이면 NULL                                  │
│  // 또는                                                            │
│  TST R0, R0        ; R0 AND R0                                      │
│  BEQ null_handler  ; ZF=1이면 0                                     │
│                                                                     │
│  【3. 루프 종료 (Loop Termination)】                                  │
│  ────────────────────────────────                                    │
│  // C 코드                                                          │
│  while (count > 0) { count--; }                                     │
│                                                                     │
│  // 어셈블리                                                        │
│  loop:                                                              │
│      CMP R0, #0       ; count - 0                                   │
│      BEQ done         ; ZF=1이면 종료                               │
│      SUB R0, R0, #1   ; count--                                     │
│      B loop                                                         │
│  done:                                                              │
│                                                                     │
│  【4. 비트 테스트 (Bit Test)】                                        │
│  ────────────────────────────                                        │
│  // C 코드                                                          │
│  if ((flags & MASK) == 0) { ... }                                   │
│                                                                     │
│  // 어셈블리                                                        │
│  TST R0, #MASK     ; R0 AND MASK                                    │
│  BEQ no_bits       ; ZF=1이면 비트 없음                             │
│                                                                     │
│  【5. 문자열 종료 검사 (String End)】                                 │
│  ────────────────────────────────────                                │
│  // C 코드                                                          │
│  while (*str != '\0') { str++; }                                    │
│                                                                     │
│  // 어셈블리                                                        │
│  loop:                                                              │
│      LDRB R1, [R0]    ; 문자 로드                                   │
│      CMP R1, #0       ; '\0'인가?                                   │
│      BEQ done         ; ZF=1이면 종료                               │
│      ADD R0, R0, #1   ; ptr++                                       │
│      B loop                                                         │
│  done:                                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ZF 기반 조건부 실행 (ARM):                                       │ │
│  │  ────────────────────────────                                     │ │
│  │  MOVEQ R0, #1     ; ZF=1이면 R0=1                                │ │
│  │  MOVNE R0, #0     ; ZF=0이면 R0=0                                │ │
│  │  ADDEQ R1, R1, #1 ; ZF=1이면 R1++                                │ │
│  │                                                               │ │
│  │  → 분기 없이 조건부 실행 가능                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. ZF × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                ZF × 타 과목 융합                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                        │
│  • if (a == b) → CMP + BEQ/BNE                                     │
│  • while (n) → SUB + BNE                                            │
│  • 최적화: TST代替 CMP                                              │
│  • 조건부 이동: CMOVZ/CMOVNZ                                        │
│                                                                     │
│  【운영체제】                                                        │
│  • NULL 포인터 검사                                                  │
│  • 에러 코드 검사 (0=성공)                                          │
│  • 스케줄링: 태스크 상태 검사                                       │
│  • 락 획득: 0이면 획득                                              │
│                                                                     │
│  【알고리즘】                                                        │
│  • 검색: 찾음/못찾음                                                 │
│  • 정렬: 교환 필요 여부                                              │
│  • 문자열: 종료 검사                                                 │
│  • 해시: 충돌 검사                                                   │
│                                                                     │
│  【네트워크】                                                        │
│  • 패킷 검사: 길이 0?                                               │
│  • 체크섬: 0이면 정상                                                │
│  • 플래그: 비트 검사                                                 │
│                                                                     │
│  【보안】                                                            │
│  • 권한 검사: 0=권한 없음                                            │
│  • 인증: 0=실패                                                      │
│  • 버퍼: 빈 버퍼 검사                                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ZF 활용 빈도 (C 언어 기준):                                      │ │
│  │  ────────────────────────────                                     │ │
│  │  • == 비교: ~30% of 조건문                                        │ │
│  │  • != 비교: ~25% of 조건문                                        │ │
│  │  • ! 연산: ~10% of 조건문                                        │ │
│  │  • 루프 조건: ~35% of 루프                                        │ │
│  │                                                               │ │
│  │  → ZF는 가장 자주 사용되는 플래그                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### ZF 디버깅/분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                ZF 디버깅/분석                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【GDB】                                                            │
│  ────────                                                            │
│  # ZF 확인                                                          │
│  (gdb) p $eflops & 0x40   # ZF는 비트 6                             │
│  (gdb) p/t ($eflops >> 6) & 1                                       │
│                                                                     │
│  # ZF 기반 브레이크포인트                                            │
│  (gdb) break *0x1000 if ($eflops & 0x40) != 0                       │
│                                                                     │
│  【어셈블리】                                                        │
│  ────────────                                                        │
│  ; ZF 직접 테스트                                                    │
│  TEST RAX, RAX       ; RAX == 0?                                    │
│  JZ is_zero          ; ZF=1이면 점프                                 │
│  JNZ not_zero        ; ZF=0이면 점프                                 │
│                                                                     │
│  ; SETZ/SETNZ (x86)                                                 │
│  SETZ AL             ; ZF=1이면 AL=1, 아니면 AL=0                    │
│  SETNZ BL            ; ZF=0이면 BL=1, 아니면 BL=0                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ZF 활용 최적화 팁:                                               │ │
│  │  ──────────────────────                                           │ │
│  │  1. CMP 대신 TST/TEST 사용 (더 빠름)                              │ │
│  │     CMP R0, #0    →    TST R0, R0                                │ │
│  │                                                               │ │
│  │  2. 분기 대신 조건부 이동                                          │ │
│  │     BEQ skip      →    CMOVZ R1, R2                               │ │
│  │                                                               │ │
│  │  3. 루프 카운터 감소 방식                                          │ │
│  │     CMP R0, #0    →    SUBS R0, R0, #1 (ZF 자동 설정)             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• ZF: 연산 결과가 0인지 표시
• 설정: 결과==0 → ZF=1, 결과!=0 → ZF=0
• 용도: 같음 비교, NULL 검사, 루프 종료
• 분기: BEQ/BNE (ARM), JE/JNE (x86)
• 최적화: 분기 대신 조건부 실행
```
