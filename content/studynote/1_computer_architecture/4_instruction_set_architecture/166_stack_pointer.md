# 166. 스택 포인터 (SP, Stack Pointer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스택의 최상단(Top) 주소를 가리키는 특수 레지스터, LIFO 구조 관리
> 2. **가치**: 함수 호출/복귀, 지역 변수, 인터럽트 처리의 핵심
> 3. **융합**: 운영체제 컨텍스트 스위칭, 컴파일러 스택 프레임, 보안 (스택 오버플로우)

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

스택 포인터(SP, Stack Pointer)는 **스택 메모리의 최상단(Top) 주소를 저장하는 특수 레지스터**다. PUSH/POP 연산으로 스택에 데이터를 저장/인출할 때 SP가 자동으로 조정된다. 함수 호출, 지역 변수, 인터럽트 처리에 필수적이다.

### 💡 비유: 접시 쌓기

SP는 **접시 쌓기의 맨 위 위치**와 같다. 접시(데이터)를 쌓으면(PUSH) 위로 올라가고, 접시를 빼면(POP) 아래로 내려간다. 항상 맨 위 접시의 위치를 가리킨다.

### 스택 구조와 SP

```
┌─────────────────────────────────────────────────────────────────────┐
│                스택 구조와 SP                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스택 메모리 구조】                                                │
│  ──────────────────────                                             │
│                                                                     │
│  주소          내용                SP 방향                          │
│  ──────────   ──────────────────   ──────────────                    │
│  0x8000       ┌────────────────┐                                   │
│               │                │                                   │
│               │  (빈 공간)      │                                   │
│               │                │                                   │
│  0x7FF0       ├────────────────┤ ← SP (초기)                       │
│  0x7FEC       │   Data 1       │                                   │
│  0x7FE8       │   Data 2       │                                   │
│  0x7FE4       │   Data 3       │   SP 감소 방향 (Full Descending)   │
│  0x7FE0       │   Return Addr  │                                   │
│  0x7FDC       │   Local Var 1  │                                   │
│  0x7FD8       │   Local Var 2  │ ← SP (현재)                       │
│  ──────────   └────────────────┘                                   │
│                                                                     │
│  【스택 성장 방향】                                                  │
│  ──────────────────────                                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Full Descending (가장 일반적):                                 │ │
│  │  ────────────────────────────────                               │ │
│  │  • PUSH: SP ← SP - 4; Mem[SP] ← data                           │ │
│  │  • POP:  data ← Mem[SP]; SP ← SP + 4                           │ │
│  │  • SP가 감소하며 스택이 아래로 성장                              │ │
│  │  • SP는 마지막 데이터 가리킴                                    │ │
│  │                                                               │ │
│  │  Empty Descending:                                              │ │
│  │  ──────────────────                                             │ │
│  │  • PUSH: Mem[SP] ← data; SP ← SP - 4                           │ │
│  │  • SP는 다음 빈 위치 가리킴                                      │ │
│  │                                                               │ │
│  │  Full Ascending:                                                │ │
│  │  ────────────────                                               │ │
│  │  • PUSH: SP ← SP + 4; Mem[SP] ← data                           │ │
│  │  • SP가 증가하며 위로 성장                                       │ │
│  │                                                               │ │
│  │  Empty Ascending:                                               │ │
│  │  ──────────────────                                             │ │
│  │  • PUSH: Mem[SP] ← data; SP ← SP + 4                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  SP 관리 방식 (ISA별):                                          │ │
│  │  ────────────────────────                                       │ │
│  │  ISA         레지스터    방식                                     │ │
│  │  ──────────  ──────────  ─────────────────                      │ │
│  │  ARM         R13 (SP)    Full Descending                        │ │
│  │  x86         ESP/RSP     Full Descending                        │ │
│  │  MIPS        $sp ($29)   Full Descending                        │ │
│  │  RISC-V      x2 (sp)     Full Descending                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### SP의 용도

```
┌─────────────────────────────────────────────────────────────────────┐
│                SP의 용도                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 서브루틴 호출/복귀】                                           │
│  ────────────────────────────                                       │
│                                                                     │
│  호출 (CALL/BL):                                                    │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  PUSH LR          ; SP ← SP - 4; Mem[SP] ← LR                  │ │
│  │  or                                                               │ │
│  │  SUB SP, SP, #4   ; SP ← SP - 4                                 │ │
│  │  STR LR, [SP]     ; Mem[SP] ← LR                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  복귀 (RET/POP PC):                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  POP PC           ; PC ← Mem[SP]; SP ← SP + 4                  │ │
│  │  or                                                               │ │
│  │  LDR LR, [SP]     ; LR ← Mem[SP]                                 │ │
│  │  ADD SP, SP, #4   ; SP ← SP + 4                                 │ │
│  │  BX LR            ; PC ← LR                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【2. 지역 변수 저장】                                               │
│  ──────────────────────────                                         │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 함수 진입 (프롤로그)                                         │ │
│  │  PUSH {R4-R7, LR}    ; 저장 레지스터 + 복귀주소                  │ │
│  │  SUB SP, SP, #16     ; 지역 변수 4개 공간                        │ │
│  │                                                               │ │
│  │  // 지역 변수 접근                                               │ │
│  │  STR R0, [SP, #0]    ; local1 = R0                              │ │
│  │  STR R1, [SP, #4]    ; local2 = R1                              │ │
│  │  LDR R2, [SP, #0]    ; R2 = local1                              │ │
│  │                                                               │ │
│  │  // 함수 종료 (에필로그)                                         │ │
│  │  ADD SP, SP, #16     ; 지역 변수 공간 반환                       │ │
│  │  POP {R4-R7, PC}     ; 레지스터 복원 + 복귀                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【3. 함수 인자 전달】                                               │
│  ──────────────────────────                                         │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 인자 4개 초과 시 스택 사용                                   │ │
│  │  // R0~R3: 처음 4개 인자 (ARM EABI)                             │ │
│  │  // SP+: 5번째 이후 인자                                         │ │
│  │                                                               │ │
│  │  // 호출자                                                       │ │
│  │  MOV R0, #1          ; arg1                                     │ │
│  │  MOV R1, #2          ; arg2                                     │ │
│  │  MOV R2, #3          ; arg3                                     │ │
│  │  MOV R3, #4          ; arg4                                     │ │
│  │  PUSH {R4}           ; arg5                                     │ │
│  │  BL func                                                       │ │
│  │  ADD SP, SP, #4      ; 스택 정리                                 │ │
│  │                                                               │ │
│  │  // 피호출자                                                     │ │
│  │  LDR R4, [SP, #4]    ; arg5 접근 (저장된 레지스터 위)            │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【4. 인터럽트/예외 처리】                                           │
│  ──────────────────────────────                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  인터럽트 발생 시:                                               │ │
│  │  1. SP_m ← SP (모드별 SP)                                       │ │
│  │  2. LR_m ← PC                                                   │ │
│  │  3. SPSR_m ← CPSR                                               │ │
│  │  4. PC ← 인터럽트_벡터                                          │ │
│  │                                                               │ │
│  │  핸들러에서:                                                     │ │
│  │  PUSH {R0-R12}      ; 레지스터 저장                              │ │
│  │  ... 인터럽트 처리 ...                                          │ │
│  │  POP {R0-R12}       ; 레지스터 복원                              │ │
│  │  SUBS PC, LR, #4    ; 복귀 (PC ← LR, CPSR ← SPSR)               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스택 프레임 구조:                                               │ │
│  │  ──────────────────                                             │ │
│  │                                                               │ │
│  │  ┌──────────────────────────────────────────────────────────┐ │ │
│  │  │  High Address                                            │ │ │
│  │  │  ────────────                                             │ │ │
│  │  │  ┌─────────────────────────────────────────────────────┐ │ │ │
│  │  │  │  매개변수 (인자)                                     │ │ │ │
│  │  │  ├─────────────────────────────────────────────────────┤ │ │ │
│  │  │  │  복귀 주소 (Return Address / LR)                     │ │ │ │
│  │  │  ├─────────────────────────────────────────────────────┤ │ │ │
│  │  │  │  저장된 레지스터 (Saved Registers)                   │ │ │ │
│  │  │  ├─────────────────────────────────────────────────────┤ │ │ │
│  │  │  │  지역 변수 (Local Variables)                       │ │ │ │
│  │  │  ├─────────────────────────────────────────────────────┤ │ │ │
│  │  │  │  임시 공간 (Temporaries)                             │ │ │ │
│  │  │  └─────────────────────────────────────────────────────┘ │ │ │
│  │  │  Low Address                                             │ │ │
│  │  │               ↑                                           │ │ │
│  │  │               │                                           │ │ │
│  │  │              SP                                           │ │ │
│  │  └──────────────────────────────────────────────────────────┘ │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### SP와 FP의 관계

```
┌─────────────────────────────────────────────────────────────────────┐
│                SP와 FP의 관계                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FP (Frame Pointer) 역할】                                         │
│  ────────────────────────────                                       │
│                                                                     │
│  • 스택 프레임의 고정 기준점                                        │
│  • SP는 실행 중 변하지만 FP는 고정                                  │
│  • 지역 변수 접근: FP + offset                                      │
│  • 디버깅/역추적에 유용                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스택 프레임 예시:                                               │ │
│  │  ──────────────────                                             │ │
│  │                                                               │ │
│  │       주소          내容                 접근                   │ │
│  │  ────────────────  ────────────────────  ──────────             │ │
│  │  FP+16  ┌─────────────────────────────┐                        │ │
│  │         │   매개변수 arg1             │                        │ │
│  │  FP+12  ├─────────────────────────────┤                        │ │
│  │         │   매개변수 arg2             │                        │ │
│  │  FP+8   ├─────────────────────────────┤                        │ │
│  │         │   복귀 주소 (old PC)        │                        │ │
│  │  FP+4   ├─────────────────────────────┤                        │ │
│  │         │   old FP (이전 프레임)       │ ← FP                   │ │
│  │  FP     ├─────────────────────────────┤                        │ │
│  │         │   local1                    │                        │ │
│  │  FP-4   ├─────────────────────────────┤                        │ │
│  │         │   local2                    │                        │ │
│  │  FP-8   ├─────────────────────────────┤                        │ │
│  │         │   saved register            │                        │ │
│  │  FP-12  └─────────────────────────────┘ ← SP                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【프롤로그/에필로그 표준 형식】                                     │
│  ──────────────────────────────────                                 │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  프롤로그:                                                       │ │
│  │  PUSH {FP, LR}       ; FP, LR 저장                              │ │
│  │  MOV FP, SP          ; FP ← SP (프레임 설정)                    │ │
│  │  SUB SP, SP, #N      ; 지역 변수 공간                           │ │
│  │                                                               │ │
│  │  에필로그:                                                       │ │
│  │  MOV SP, FP          ; SP ← FP (프레임 해제)                    │ │
│  │  POP {FP, PC}        ; FP 복원, PC ← LR (복귀)                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  FP 생략 최적화:                                                 │ │
│  │  ────────────────                                               │ │
│  │  • FP 레지스터를 GPR로 활용                                     │ │
│  │  • SP-relative addressing                                       │ │
│  │  • 장점: 레지스터 1개 절약                                       │ │
│  │  • 단점: 디버깅 어려움                                          │ │
│  │  • GCC: -fomit-frame-pointer                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: OS 커널 개발**
- 요구: 컨텍스트 스위칭
- 접근: SP 저장/복원
- 구현: 어셈블리로 직접 제어
- 결과: 정확한 태스크 전환

**시나리오 2: 임베디드 스택 크기 결정**
- 요구: 충분한 스택 크기
- 분석: 호출 깊이, 지역 변수
- 결정: 여유분 포함 (2×)
- 결과: 오버플로우 방지

**시나리오 3: 디버깅/역추적**
- 요구: 콜스택 복원
- 접근: FP 체인 추적
- 구현: FP → old FP → ...
- 결과: 함수 호출 이력 파악

**시나리오 4: 버퍼 오버플로우 방어**
- 요구: 스택 보호
- 접근: 카나리, ASLR
- 구현: 스택 보호 메커니즘
- 결과: 공격 방지

```
│  (gdb) bt full                                                      │
│                                                                     │
│  【어셈블리】                                                        │
│  ────────────                                                        │
│  ; ARM                                                              │
│  MOV R0, SP            ; R0 ← SP                                    │
│  SUB SP, SP, #32       ; SP ← SP - 32                               │
│  ADD SP, SP, #32       ; SP ← SP + 32                               │
│  PUSH {R0-R3}          ; SP 감소, 레지스터 저장                     │
│  POP {R0-R3}           ; 레지스터 복원, SP 증가                     │
│                                                                     │
│  ; x86                                                              │
│  MOV EAX, ESP          ; EAX ← ESP                                  │
│  SUB ESP, 32           ; ESP ← ESP - 32                             │
│  PUSH EAX              ; ESP ← ESP - 4; [ESP] ← EAX                 │
│  POP EBX               ; EBX ← [ESP]; ESP ← ESP + 4                 │
│                                                                     │
│  【스택 사용량 분석】                                                │
│  ──────────────────────                                             │
│  # Linux                                                            │
│  $ cat /proc/<pid>/maps | grep stack                               │
│  $ pmap -x <pid>                                                    │
│                                                                     │
│  # 정적 분석                                                         │
│  $ gcc -fstack-usage -c file.c                                     │
│  $ cat file.su                                                      │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스택 사용량 최적화:                                             │ │
│  │  ────────────────────────                                       │ │
│  │  • 큰 지역 변수 → 힙/정적 할당                                  │ │
│  │  • 깊은 재귀 → 반복문으로 변경                                  │ │
│  │  • VLA (가변 길이 배열) → 동적 할당                             │ │
│  │  • alloca() 사용 자제                                           │ │
│  │  • 인라인 함수로 호출 깊이 감소                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| SP 용도 | 빈도 | 중요도 |
|:---|:---:|:---:|
| **함수 호출** | 매우 높음 | 필수 |
| **지역 변수** | 높음 | 필수 |
| **인터럽트** | 중간 | 필수 |
| **인자 전달** | 중간 | 선택 |

### 핵심 요약

```
• SP: 스택 최상단 주소 저장
• 방향: Full Descending (감소 방향)이 일반적
• 용도: 함수 호출, 지역 변수, 인터럽트
• FP: 스택 프레임 고정 기준점
• 보안: 버퍼 오버플로우 방어 필요
```
