# 191. 0-주소 명령어 (0-Address Instruction)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 피연산자 주소가 없는 명령어 (스택 머신)
> 2. **가치**: 가장 짧은 명령어, 높은 코드 밀도
> 3. **융합**: JVM, WebAssembly, 역폴란드 표기법

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

0-주소 명령어(0-Address Instruction)는 **명령어에 피연산자 주소를 명시하지 않는 명령어**다. 피연산자는 암묵적으로 스택의 top에서 가져오며, 결과도 스택에 저장된다. 스택 머신의 특징이다.

### 💡 비유: 접시 더미에서 자동으로 꺼내기

0-주소 명령어는 **"위에서 두 개 꺼내서 더해"**라고 말하는 것과 같다. 어디서 꺼내는지 말하지 않아도 접시 더미(스택)의 맨 위에서 자동으로 꺼낸다. 주소를 말할 필요가 없다.

### 0-주소 명령어 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                0-주소 명령어 구조                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조】                                                           │
│  ─────────                                                          │
│  ┌─────────────────────┐                                            │
│  │       Opcode        │                                            │
│  │      (연산코드)      │                                            │
│  └─────────────────────┘                                            │
│  (피연산자 주소 없음)                                                │
│                                                                     │
│  【동작 원리】                                                       │
│  ─────────────                                                      │
│  • 피연산자: 스택 top에서 자동 pop                                   │
│  • 결과: 스택에 자동 push                                            │
│  • 예: ADD → pop 2개, 더해서 push                                   │
│                                                                     │
│  【주소 수 비교】                                                    │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  유형       구조              예시 (a+b)                        │ │
│  │  ────────  ────────────────  ───────────────────────           │ │
│  │  0-주소    OP                PUSH a, PUSH b, ADD               │ │
│  │  1-주소    OP A              LOAD a, ADD b                     │ │
│  │  2-주소    OP A1, A2         ADD a, b                          │ │
│  │  3-주소    OP A1, A2, A3     ADD c, a, b                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【0-주소 명령어 종류】                                              │
│  ────────────────────────                                           │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어      동작                     스택 변화                   │ │
│  │  ─────────  ───────────────────────  ────────────────────      │ │
│  │  ADD       pop 2, push sum          [..., a+b]                 │ │
│  │  SUB       pop 2, push diff         [..., a-b]                 │ │
│  │  MUL       pop 2, push prod         [..., a*b]                 │ │
│  │  DIV       pop 2, push quot         [..., a/b]                 │ │
│  │  NEG       pop 1, push neg          [..., -a]                  │ │
│  │  DUP       duplicate top            [..., a, a]                │ │
│  │  POP       remove top               [...]                       │ │
│  │  SWAP      swap top 2               [..., b, a]                │ │
│  │                                                               │ │
│  │  예외 (주소 포함):                                                 │ │
│  │  PUSH addr ; 메모리→스택 (1-주소)                                 │ │
│  │  POP addr  ; 스택→메모리 (1-주소)                                 │ │
│  │  LOAD      ; 주소 스택에서 pop, 값 push                           │ │
│  │  STORE     ; 값,주소 pop, 메모리에 저장                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【코드 크기 비교: a = b + c * d】                                    │
│  ───────────────────────────────────                                │
│                                                                     │
│  0-주소 (스택 머신):                                                 │
│  PUSH b, PUSH c, PUSH d, MUL, ADD, POP a   ; 6개, 각 1바이트        │
│                                                                     │
│  3-주소 (RISC):                                                     │
│  MUL t, c, d                               ; 4바이트                │
│  ADD a, b, t                               ; 4바이트                │
│  ; 총 8바이트 (0-주소는 ~6바이트)                                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  0-주소 장단점:                                                    │ │
│  │  ──────────────                                                    │ │
│  │  장점:                                                             │ │
│  │  • 명령어 길이 짧음 (opcode만)                                     │ │
│  │  • 코드 크기 작음                                                  │ │
│  │  • 컴파일러 단순                                                   │ │
│  │  • 이식성 높음                                                     │ │
│  │                                                               │ │
│  │  단점:                                                             │ │
│  │  • 실행 속도 느림 (스택 접근)                                      │ │
│  │  • 최적화 어려움                                                   │ │
│  │  • 임시 결과 관리 복잡                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 0-주소 명령어 활용

### 활용 패턴

```
┌─────────────────────────────────────────────────────────────────────┐
│                0-주소 명령어 활용                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 산술 연산】                                                    │
│  ──────────────────                                                  │
│  // a * b + c * d                                                   │
│  PUSH a         ; [a]                                               │
│  PUSH b         ; [a, b]                                            │
│  MUL            ; [a*b]                                             │
│  PUSH c         ; [a*b, c]                                          │
│  PUSH d         ; [a*b, c, d]                                       │
│  MUL            ; [a*b, c*d]                                        │
│  ADD            ; [a*b + c*d]                                       │
│                                                                     │
│  【2. 논리 연산】                                                    │
│  ──────────────────                                                  │
│  // (a & b) | (c & d)                                               │
│  PUSH a, PUSH b, AND      ; [a&b]                                   │
│  PUSH c, PUSH d, AND      ; [a&b, c&d]                              │
│  OR                        ; [(a&b)|(c&d)]                           │
│                                                                     │
│  【3. 비교 연산】                                                    │
│  ──────────────────                                                  │
│  // if (a > b) ...                                                  │
│  PUSH a, PUSH b, SUB       ; [a-b]                                  │
│  PUSH 0, GT                ; [a>b ? 1 : 0]                          │
│  JZ skip                   ; 조건부 분기                             │
│  ; ...                                                              │
│  skip:                                                              │
│                                                                     │
│  【4. 함수 호출】                                                    │
│  ──────────────────                                                  │
│  // func(a, b)                                                      │
│  PUSH a         ; 인자 push                                         │
│  PUSH b         ; 인자 push                                         │
│  CALL func      ; PC push, 분기                                     │
│  ; 스택: [result] (함수 반환값)                                      │
│                                                                     │
│  【JVM 예시】                                                        │
│  ──────────────────                                                  │
│  // int max(int a, int b) { return a > b ? a : b; }                 │
│  iload_0        ; [a]                                               │
│  iload_1        ; [a, b]                                            │
│  if_icmple L1   ; pop 2, 비교, 분기                                  │
│  iload_0        ; [a]                                               │
│  goto L2                                                            │
│  L1:                                                                │
│  iload_1        ; [b]                                               │
│  L2:                                                                │
│  ireturn        ; 반환                                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Forth 언어 예시:                                                  │ │
│  │  ──────────────────                                                │ │
│  │  : SQUARE ( n -- n^2 )   DUP * ;                                 │ │
│  │  5 SQUARE .    \ 25 출력                                          │ │
│  │                                                               │ │
│  │  : CUBE ( n -- n^3 )    DUP DUP * * ;                            │ │
│  │  3 CUBE .      \ 27 출력                                          │ │
│  │                                                               │ │
│  │  : SUM-TO ( n -- sum )                                            │ │
│  │    0 SWAP 1+ 0 DO I + LOOP ;                                     │ │
│  │  10 SUM-TO .   \ 55 출력 (1+2+...+10)                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────┐
│                0-주소 명령어 분석                                     │

│                                                                     │

│  public int add(int, int);                                          │

│  【WebAssembly 텍스트】                                              │
│  ──────────────────────────                                         │
│  (module                                                            │
│    (func (export "add") (param $a i32) (param $b i32) (result i32)  │

│      local.get $b     ;; [a, b]                                     │
│      i32.add))         ;; [a+b]                                      │
│                                                                     │
│  【PostScript 예시】                                                 │
│  ──────────────────────                                             │
│  /factorial {        % 스택: n -- n!                                │
│    dup 1 gt {                                                       │
│      dup 1 sub factorial mul                                        │
│    } if                                                             │
│  } def                                                              │
│  5 factorial =       % 120 출력                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스택 추적 (Stack Trace):                                         │ │
│  │  ──────────────────────────                                        │ │
│  │  ; (1 + 2) * (3 - 4)                                              │ │
│  │  PUSH 1      [1]                                                  │ │
│  │  PUSH 2      [1, 2]                                               │ │
│  │  ADD         [3]           ; 1 + 2                                │ │
│  │  PUSH 3      [3, 3]                                               │ │
│  │  PUSH 4      [3, 3, 4]                                            │ │
│  │  SUB         [3, -1]        ; 3 - 4                               │ │
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 0-주소 명령어: 피연산자 주소 없음 (스택)
• 구조: Opcode만 (매우 짧음)
• 동작: 스택 top에서 pop, 결과 push
• 장점: 코드 크기 작음, 컴파일러 단순
• 단점: 실행 속도 느림, 최적화 어려움
• 활용: JVM, WASM, Forth, PostScript
```
