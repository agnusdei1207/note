# 193. 2-주소 명령어 (2-Address Instruction)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 두 피연산자 주소를 명시하는 명령어 (목적지 = 원본)
> 2. **가치**: GPR 기반, x86/CISC 방식, 코드 크기 vs 성능 균형
> 3. **융합**: 메모리-레지스터 연산, 파괴적 연산

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

2-주소 명령어(2-Address Instruction)는 **두 개의 피연산자 주소를 명시**하며, 한 피연산자가 목적지이자 원본인 명령어 형식이다. x86/CISC 아키텍처의 전형적인 방식으로, "ADD dest, src" 형태에서 dest = dest + src다.

### 💡 비유: 상자에 직접 더하기

2-주소 명령어는 **상자에 직접 더하기**와 같다. "A 상자에 B 상자 내용을 더해"라고 하면, A 상자의 원래 내용은 사라지고 A+B 결과가 A 상자에 들어간다. A가 목적지이자 원본이다.

### 2-주소 명령어 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                2-주소 명령어 구조                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조】                                                           │
│  ─────────                                                          │
│  ┌─────────────────────┬─────────────────────┬─────────────────┐    │
│  │       Opcode        │    Destination      │     Source      │    │
│  │      (연산코드)      │     (목적지)        │      (원본)      │    │
│  └─────────────────────┴─────────────────────┴─────────────────┘    │
│                                                                     │
│  【동작 원리】                                                       │
│  ─────────────                                                      │
│  OP dest, src    →    dest = dest OP src                            │
│                                                                     │
│  예: ADD EAX, EBX  →  EAX = EAX + EBX                               │
│                                                                     │
│  【특징】                                                           │
│  ─────────                                                          │
│  • 파괴적 연산: dest가 덮어써짐                                      │
│  • GPR 기반: 범용 레지스터 사용                                      │
│  • 메모리 가능: CISC는 메모리도 피연산자 가능                        │
│  • 명령어 수: 1-주소보다 적음                                        │
│                                                                     │
│  【x86 예시】                                                        │
│  ──────────────────                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어           동작                    설명                  │ │
│  │  ───────────────  ─────────────────────  ───────────────────  │ │
│  │  MOV EAX, EBX     EAX = EBX              데이터 이동           │ │
│  │  ADD EAX, EBX     EAX = EAX + EBX        덧셈                  │ │
│  │  SUB EAX, 10      EAX = EAX - 10         즉시값 뺄셈           │ │
│  │  AND EAX, EBX     EAX = EAX & EBX        논리곱                │ │
│  │  CMP EAX, EBX     (EAX - EBX, 플래그만)  비교                  │ │
│  │  TEST EAX, EBX    (EAX & EBX, 플래그만) 비트 테스트            │ │
│  │                                                               │ │
│  │  MOV EAX, [mem]   EAX = Mem[mem]          메모리 → 레지스터    │ │
│  │  MOV [mem], EAX   Mem[mem] = EAX          레지스터 → 메모리    │ │
│  │  ADD EAX, [mem]   EAX = EAX + Mem[mem]    메모리 포함 연산     │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【예시: a = b + c】                                                 │
│  ──────────────────                                                  │
│  MOV EAX, [b]       ; EAX = b                                       │
│  ADD EAX, [c]       ; EAX = EAX + c = b + c                         │
│  MOV [a], EAX       ; a = EAX                                       │
│  ; 3개 명령어 (1-주소도 3개, 3-주소는 1개)                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  2-주소 장단점:                                                    │ │
│  │  ──────────────                                                    │ │
│  │  장점:                                                             │ │
│  │  • 1-주소보다 명령어 수 적음                                       │ │
│  │  • 메모리 피연산자 가능 (CISC)                                     │ │
│  │  • 레지스터 활용 가능                                              │ │
│  │  • 코드 크기 적당                                                  │ │
│  │                                                               │ │
│  │  단점:                                                             │ │
│  │  • 파괴적 연산 (원본 손실)                                         │ │
│  │  • 3-주소보다 명령어 수 많음                                       │ │
│  │  • 원본 보존 시 이동 필요                                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 2-주소 명령어 활용

### 활용 패턴

```
┌─────────────────────────────────────────────────────────────────────┐
│                2-주소 명령어 활용                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 산술 연산】                                                    │
│  ──────────────────                                                  │
│  // a = b + c - d                                                   │
│  MOV EAX, [b]       ; EAX = b                                       │
│  ADD EAX, [c]       ; EAX = b + c                                   │
│  SUB EAX, [d]       ; EAX = b + c - d                               │
│  MOV [a], EAX       ; a = EAX                                       │
│  ; 4개 명령어                                                        │
│                                                                     │
│  【2. 원본 보존】                                                    │
│  ──────────────────                                                  │
│  // b = a + c (a는 보존)                                            │
│  MOV EAX, [a]       ; EAX = a (a 보존)                              │
│  ADD EAX, [c]       ; EAX = a + c                                   │
│  MOV [b], EAX       ; b = a + c                                     │
│  ; a는 변경 없음                                                     │
│                                                                     │
│  【3. 메모리-메모리 (x86)】                                          │
│  ────────────────────────────────                                   │
│  ; 일부 x86 명령어는 메모리-메모리 가능                              │
│  MOV [dest], [src]  ; (대부분 불가, 레지스터 경유)                   │
│  ; 실제로는:                                                         │
│  MOV EAX, [src]                                                    │
│  MOV [dest], EAX                                                   │
│                                                                     │
│  【4. 비교 및 분기】                                                 │
│  ──────────────────────                                             │
│  // if (a > b) ...                                                  │
│  MOV EAX, [a]                                                       │
│  CMP EAX, [b]       ; EAX - [b], 플래그만                           │
│  JG  greater        ; a > b이면 분기                                 │
│                                                                     │
│  【5. 루프】                                                         │
│  ──────────────────                                                  │
│  // for (i=0; i<n; i++) sum += arr[i]                               │
│  XOR EAX, EAX       ; sum = 0                                       │
│  XOR ECX, ECX       ; i = 0                                         │
│  loop:                                                              │
│  CMP ECX, [n]       ; i < n?                                        │
│  JGE done                                                            │
│  ADD EAX, [arr + ECX*4]  ; sum += arr[i]                            │
│  INC ECX            ; i++                                           │
│  JMP loop                                                           │
│  done:                                                              │
│  MOV [sum], EAX                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  x86 주소 지정 조합:                                                │ │
│  │  ────────────────────────                                          │ │
│  │  ADD reg, reg       ; 레지스터 + 레지스터                          │ │
│  │  ADD reg, mem       ; 레지스터 + 메모리                            │ │
│  │  ADD reg, imm       ; 레지스터 + 즉시값                            │ │
│  │  ADD mem, reg       ; 메모리 + 레지스터                            │ │
│  │  ADD mem, imm       ; 메모리 + 즉시값                              │ │
│  │                                                               │ │
│  │  제약: mem, mem 불가 (메모리 두 개 동시 불가)                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 2-주소 명령어 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                2-주소 명령어 × 타 과목 융합                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                        │
│  • 레지스터 할당: GPR 활용                                           │
│  • 라이브 변수 분석: 파괴적 연산 고려                                │
│  • 복사 전파: MOV 제거                                               │
│  • 데드 코드 제거: 불필요한 MOV 삭제                                 │
│                                                                     │
│  【컴퓨터 구조】                                                     │
│  • CISC: x86, VAX                                                   │
│  • 메모리 피연산자: 직접 접근                                        │
│  • 마이크로코드: 복잡한 명령어                                       │
│  • 성능: 3-주소보다 느림                                             │
│                                                                     │
│  【알고리즘】                                                        │
│  • 교환: MOV + MOV (temp 필요)                                      │
│  • 누적: ADD 반복                                                    │
│  • 복사: MOV                                                         │
│  • XOR 스왑: 3개 XOR로 temp 없이 교환                               │
│                                                                     │
│  【프로그래밍】                                                      │
│  • a += b → ADD a, b                                                │
│  • a = b → MOV a, b                                                 │
│  • a++ → INC a                                                      │
│  • swap: temp=a; a=b; b=temp                                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  XOR 스왑 (임시 변수 없이 교환):                                    │ │
│  │  ──────────────────────────────────                               │ │
│  │  // swap a, b without temp                                         │ │
│  │  XOR EAX, EBX      ; EAX = a ^ b                                   │ │
│  │  XOR EBX, EAX      ; EBX = b ^ (a^b) = a                           │ │
│  │  XOR EAX, EBX      ; EAX = (a^b) ^ a = b                           │ │
│  │  ; 결과: EAX=b, EBX=a (교환됨)                                     │ │
│  │                                                               │ │
│  │  ; 실제로는 temp 사용이 더 빠름 (파이프라인)                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────┐
│                2-주소 명령어 분석                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【objdump 분석】                                                    │
│  ──────────────────                                                  │
│  $ objdump -d program                                               │
│  1000:  89 c3           mov ebx, eax       ; ebx = eax              │
│  1002:  01 d8           add eax, ebx       ; eax += ebx             │
│  1004:  83 c0 0a        add eax, 0xa       ; eax += 10              │
│  1007:  29 c3           sub ebx, eax       ; ebx -= eax             │
│  1009:  83 e3 0f        and ebx, 0xf       ; ebx &= 0x0F            │
│                                                                     │
│  【GDB 디버깅】                                                      │
│  ──────────────────                                                  │
│  (gdb) p/x $eax          ; EAX 확인                                  │
│  (gdb) si                 ; 명령어 실행                              │
│  (gdb) p/x $eax          ; 변경된 EAX 확인                           │
│  (gdb) info registers    ; 모든 레지스터                             │
│                                                                     │
│  【최적화: 불필요한 MOV 제거】                                        │
│  ──────────────────────────────                                     │
│  ; 최적화 전                                                         │
│  MOV EAX, EBX       ; EAX = EBX                                     │
│  ADD EAX, ECX       ; EAX = EAX + ECX                               │
│  MOV [result], EAX  ; result = EAX                                  │
│                                                                     │
│  ; 최적화 후 (EBX가 이후 사용 안 됨)                                 │
│  ADD EBX, ECX       ; EBX = EBX + ECX                               │
│  MOV [result], EBX  ; result = EBX                                  │
│  ; 3개 → 2개로 감소                                                  │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  2-주소 명령어 디자인 팁:                                           │ │
│  │  ────────────────────────────                                      │ │
│  │  1. 원본이 필요하면 먼저 복사                                        │ │
│  │     MOV ECX, EAX   ; EAX 보존                                      │ │
│  │     ADD EAX, EBX   ; EAX 변경                                      │ │
│  │                                                               │ │
│  │  2. LEA로 비파괴적 연산                                             │ │
│  │     LEA EAX, [EBX + ECX]  ; EAX = EBX + ECX                       │ │
│  │     ; EBX, ECX 변경 없음                                           │ │
│  │                                                               │ │
│  │  3. CMOV로 조건부 이동                                              │ │
│  │     CMOVE EAX, EBX  ; if (ZF) EAX = EBX                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 2-주소 명령어: 두 피연산자 주소 명시
• 동작: dest = dest OP src (파괴적)
• 특징: GPR 기반, 메모리 피연산자 가능
• 장점: 1-주소보다 효율, 메모리 직접 연산
• 단점: 원본 손실, 3-주소보다 많은 명령어
• 활용: x86, CISC 아키텍처
```
