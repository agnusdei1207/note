# 183. 데이터 전송 명령어 (Data Transfer Instruction)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터를 한 위치에서 다른 위치로 이동시키는 명령어
> 2. **가치**: 메모리-레지스터, 레지스터-레지스터, 스택 연산
> 3. **융합**: 변수 대입, 함수 인자 전달, 버퍼 복사

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

데이터 전송 명령어(Data Transfer Instruction)는 **데이터를 한 위치에서 다른 위치로 복사/이동**시키는 명령어다. MOV, LOAD, STORE, PUSH, POP 등이 포함되며, 프로그램에서 가장 빈번하게 사용되는 명령어 유형이다.

### 💡 비유: 짐 옮기기

데이터 전송 명령어는 **짐을 옮기는 것**과 같다. 서랍(메모리)에서 책상(레지스터)으로 짐을 옮기거나(MOV), 서랍 위에 잠시 올려두고(PUSH), 다시 꺼내는(POP) 식이다. 짐은 복사되거나 이동된다.

### 데이터 전송 명령어 분류

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터 전송 명령어 분류                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【분류】                                                           │
│  ─────────                                                          │
│                                                                     │
│  데이터 전송                                                        │
│  ├── 레지스터-레지스터: MOV R1, R2                                  │
│  ├── 레지스터-메모리: MOV R1, [addr]  /  MOV [addr], R1             │
│  ├── 즉시값-레지스터: MOV R1, #imm                                  │
│  ├── 스택: PUSH, POP                                               │
│  └── 특수: LEA, XCHG, SWAP                                         │
│                                                                     │
│  【주요 명령어】                                                     │
│  ──────────────                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어    ISA     기능                    예시                 │ │
│  │  ───────  ───────  ─────────────────────  ────────────────     │ │
│  │  MOV      x86     데이터 이동            MOV EAX, EBX          │ │
│  │  MOV      ARM     데이터 이동            MOV R0, R1            │ │
│  │  LDR      ARM     메모리→레지스터        LDR R0, [R1]          │ │
│  │  STR      ARM     레지스터→메모리        STR R0, [R1]          │ │
│  │  lw/sw    MIPS    메모리 로드/스토어     lw $t0, 0($t1)        │ │
│  │  PUSH     x86     스택에 저장            PUSH EAX              │ │
│  │  POP      x86     스택에서 꺼내기        POP EBX               │ │
│  │  LEA      x86     유효 주소 계산         LEA EAX, [EBX+8]      │ │
│  │  XCHG     x86     데이터 교환            XCHG EAX, EBX         │ │
│  │  MOVS     x86     문자열 복사            MOVSB/W/D             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RISC vs CISC】                                                   │
│  ──────────────────                                                  │
│  RISC (ARM, MIPS, RISC-V):                                         │
│  • Load/Store 구조: 메모리 접근은 LDR/STR만                        │
│  • 레지스터-레지스터 연산만                                          │
│  • MOV는 레지스터 간만                                               │
│                                                                     │
│  CISC (x86):                                                        │
│  • 메모리-레지스터 직접 이동 가능                                    │
│  • 다양한 주소 지정 방식                                             │
│  • 복잡한 문자열 명령어                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  데이터 전송 vs 연산:                                             │ │
│  │  ──────────────────────                                           │ │
│  │  • 전송: 데이터 이동만 (상태 변화 없음, 대부분)                    │ │
│  │  • 연산: 데이터 가공 (산술/논리 결과)                             │ │
│  │                                                               │ │
│  │  예외: LEA는 전송처럼 보이지만 주소 계산                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 주요 데이터 전송 명령어

### 명령어 상세

```
┌─────────────────────────────────────────────────────────────────────┐
│                주요 데이터 전송 명령어                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. MOV (Move)】                                                  │
│  ──────────────────                                                  │
│  MOV dest, src       ; dest ← src                                   │
│                                                                     │
│  MOV EAX, EBX        ; 레지스터 → 레지스터                          │
│  MOV EAX, [EBX]      ; 메모리 → 레지스터                            │
│  MOV [EAX], EBX      ; 레지스터 → 메모리                            │
│  MOV EAX, 100        ; 즉시값 → 레지스터                            │
│                                                                     │
│  【2. LOAD/STORE (ARM, RISC)】                                       │
│  ──────────────────────────                                         │
│  LDR R0, [R1]        ; R0 = Mem[R1]                                 │
│  STR R0, [R1]        ; Mem[R1] = R0                                 │
│  LDRB R0, [R1]       ; byte 로드                                    │
│  STRH R0, [R1]       ; halfword 저장                                │
│                                                                     │
│  【3. PUSH/POP (스택)】                                              │
│  ────────────────────────────                                       │
│  PUSH EAX            ; SP -= 4; Mem[SP] = EAX                       │
│  POP EBX             ; EBX = Mem[SP]; SP += 4                       │
│                                                                     │
│  ; 함수 프롤로그                                                     │
│  PUSH RBP                                                          │
│  MOV RBP, RSP                                                       │
│  ; 함수 에필로그                                                     │
│  MOV RSP, RBP                                                       │
│  POP RBP                                                            │
│  RET                                                               │
│                                                                     │
│  【4. LEA (Load Effective Address)】                                 │
│  ─────────────────────────────────                                  │
│  LEA EAX, [EBX+ECX*4+8]  ; EAX = EBX+ECX*4+8 (주소 계산)            │
│                                                                     │
│  ; 용도:                                                            │
│  • 포인터 계산                                                       │
│  • 산술 연산 (곱셈 없이 덧셈)                                        │
│  • 위치 독립 코드                                                    │
│                                                                     │
│  【5. XCHG (Exchange)】                                              │
│  ──────────────────────                                             │
│  XCHG EAX, EBX       ; EAX ↔ EBX                                    │
│                                                                     │
│  ; 스핀락 구현                                                       │
│  lock xchg [mutex], 1   ; 원자적 교환                               │
│                                                                     │
│  【6. MOVS (String Move)】                                          │
│  ─────────────────────────                                          │
│  MOVSB               ; byte: [EDI] = [ESI]; ESI++; EDI++            │
│  MOVSW               ; word                                         │
│  MOVSD               ; dword                                        │
│  REP MOVSB           ; ECX번 반복                                    │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  데이터 크기 접미사 (x86):                                        │ │
│  │  ──────────────────────────                                      │ │
│  │  B = Byte (8비트)                                                │ │
│  │  W = Word (16비트)                                                │ │
│  │  L/D = Dword (32비트)                                             │ │
│  │  Q = Qword (64비트)                                               │ │
│  │                                                               │ │
│  │  MOV AL, BL         ; 8비트                                       │ │
│  │  MOV AX, BX         ; 16비트                                      │ │
│  │  MOV EAX, EBX       ; 32비트                                      │ │
│  │  MOV RAX, RBX       ; 64비트                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 데이터 전송 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터 전송 × 타 과목 융합                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                        │
│  • 변수 대입: a = b → MOV                                           │
│  • 인자 전달: 레지스터 또는 스택                                     │
│  • 반환값: RAX/EAX로 전달                                            │
│  • 레지스터 할당: MOV 최소화                                         │
│                                                                     │
│  【운영체제】                                                        │
│  • 컨텍스트 스위치: 레지스터 저장/복원                               │
│  • 인터럽트: 레지스터 PUSH/POP                                       │
│  • 시스템 콜: 인자 전달                                              │
│  • 버퍼 복사: memcpy 구현                                           │
│                                                                     │
│  【자료구조】                                                        │
│  • 배열: 요소 복사                                                   │
│  • 스택: push/pop                                                   │
│  • 큐: enqueue/dequeue                                              │
│  • 문자열: 복사, 연결                                                │
│                                                                     │
│  【알고리즘】                                                        │
│  • 정렬: swap 구현                                                   │
│  • 검색: 결과 반환                                                   │
│  • 복사: 깊은 복사                                                   │
│  • 병합: 데이터 이동                                                 │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  성능 최적화:                                                      │ │
│  │  ──────────                                                       │ │
│  │  • 불필요한 MOV 제거 (copy propagation)                           │ │
│  │  • XOR로 0 초기화 (MOV보다 짧음)                                  │ │
│  │  • 레지스터 할당으로 메모리 접근 감소                              │ │
│  │  • SIMD로 여러 데이터 동시 전송                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 데이터 전송 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                데이터 전송 분석                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【함수 인자 전달 (x86-64 System V)】                                │
│  ──────────────────────────────────────                             │
│  // int func(int a, int b, int c, int d, int e, int f);             │
│  func:                                                              │
│      ; a=EDI, b=ESI, c=EDX, d=ECX (레지스터)                        │
│      ; e=8[RSP], f=16[RSP] (스택)                                   │
│      MOV EAX, EDI         ; a 사용                                   │
│      MOV EAX, [RSP+8]     ; e 사용                                   │
│      RET                                                           │
│                                                                     │
│  【memcpy 구현】                                                     │
│  ──────────────────                                                  │
│  ; void *memcpy(void *dst, void *src, size_t n);                    │
│  memcpy:                                                            │
│      MOV RAX, RDI         ; 반환값                                   │
│      TEST RDX, RDX        ; n == 0?                                 │
│      JE .done                                                       │
│  .loop:                                                             │
│      MOV R8B, [RSI]       ; src에서 로드                            │
│      MOV [RDI], R8B       ; dst에 저장                              │
│      INC RSI                                                        │
│      INC RDI                                                        │
│      DEC RDX                                                        │
│      JNE .loop                                                      │
│  .done:                                                             │
│      RET                                                           │
│                                                                     │
│  【GDB 디버깅】                                                      │
│  ──────────────────                                                  │
│  (gdb) si                     ; 단일 명령어 실행                     │
│  (gdb) p/x $rax               ; 레지스터 값 확인                     │
│  (gdb) x/10x $rsp             ; 스택 메모리 확인                     │
│  (gdb) watch *(int*)$rdi      ; 메모리 쓰기 감시                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  데이터 전송 체크리스트:                                          │ │
│  │  ────────────────────────                                        │ │
│  │  □ 소스와 목적지 크기가 같은가?                                   │ │
│  │  □ 메모리 주소가 유효한가?                                        │ │
│  │  □ 정렬이 맞는가?                                                  │ │
│  │  □ 스택 오버플로우 위험?                                          │ │
│  │  □ 동시 접근 동기화?                                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```
