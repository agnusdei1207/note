# 171. 고정 길이 명령어 (Fixed Length Instruction)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 명령어가 동일한 비트 수를 갖는 ISA 설계 방식
> 2. **가치**: 단순한 디코딩, 효율적 파이프라인, 빠른 명령어 fetch
> 3. **융합**: RISC 철학, 슈퍼스칼라, VLIW 설계의 기반

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

고정 길이 명령어(Fixed Length Instruction)는 **모든 명령어가 동일한 비트 수를 갖는 명령어 인코딩 방식**이다. 대표적으로 ARM(32비트), MIPS(32비트), RISC-V(기본 32비트)가 있으며, RISC 아키텍처의 핵심 특징이다.

### 💡 비유: 같은 크기의 서랍

고정 길이 명령어는 **모든 칸이 같은 크기인 서랍**과 같다. 어떤 물건이든 같은 크기 칸에 들어간다. 작은 물건은 공간이 남고 큰 물건은 여러 칸이 필요하지만, 어느 칸이든 바로 찾을 수 있다.

### 고정 길이 명령어 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                고정 길이 명령어 구조                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【특징】                                                           │
│  ─────────                                                          │
│  • 모든 명령어 동일 길이 (보통 32비트)                               │
│  • 명령어 경계가 항상 일정                                           │
│  • fetch, decode 로직 단순                                          │
│  • 파이프라인 설계 용이                                              │
│                                                                     │
│  【MIPS 32비트 명령어 형식】                                         │
│  ──────────────────────────                                         │
│                                                                     │
│  R-type (레지스터):                                                  │
│  ┌───────┬─────┬─────┬─────┬─────┬───────┐                          │
│  │ opcode│ rs  │ rt  │ rd  │shamt│funct  │                          │
│  │  6    │  5  │  5  │  5  │  5  │   6   │ = 32비트                 │
│  └───────┴─────┴─────┴─────┴─────┴───────┘                          │
│  예: add $t0, $t1, $t2                                              │
│                                                                     │
│  I-type (즉시값):                                                    │
│  ┌───────┬─────┬─────┬───────────────────┐                          │
│  │ opcode│ rs  │ rt  │    immediate      │                          │
│  │  6    │  5  │  5  │        16         │ = 32비트                 │
│  └───────┴─────┴─────┴───────────────────┘                          │
│  예: addi $t0, $t1, 100                                             │
│                                                                     │
│  J-type (점프):                                                      │
│  ┌───────┬─────────────────────────────────┐                        │
│  │ opcode│            address              │                        │
│  │  6    │               26                │ = 32비트               │
│  └───────┴─────────────────────────────────┘                        │
│  예: j 0x100000                                                     │
│                                                                     │
│  【고정 길이 명령어 ISA 비교】                                       │
│  ────────────────────────────                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ISA      기본 길이   압축 모드    특징                        │ │
│  │  ───────────────────  ───────────  ────────────               │ │
│  │  MIPS     32비트      16비트(MIPS16e) 정렬 중시                 │ │
│  │  ARM      32비트      16비트(Thumb)  조건부 실행                │ │
│  │  RISC-V   32비트      16비트(C)     모듈형 확장                │ │
│  │  PowerPC  32비트      없음           Power 연산                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 고정 길이 명령어 장단점

### 장단점 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                고정 길이 명령어 장단점                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장점】                                                           │
│  ─────────                                                          │
│                                                                     │
│  1. 단순한 명령어 fetch                                             │
│     • PC + 4로 다음 명령어 위치 즉시 계산                           │
│     • 명령어 경계 탐색 불필요                                        │
│                                                                     │
│  2. 효율적 파이프라인                                                │
│     • 모든 단계가 균일한 시간                                        │
│     • 디코드 단계가 단순                                             │
│     • 버블 최소화                                                    │
│                                                                     │
│  3. 슈퍼스칼라 구현 용이                                             │
│     • 여러 명령어 동시 fetch/decode                                 │
│     • 명령어 길이 예측 불필요                                        │
│                                                                     │
│  4. 정렬 최적화                                                      │
│     • 캐시 라인 경계 정렬                                            │
│     • 메모리 접근 효율                                               │
│                                                                     │
│  【단점】                                                           │
│  ─────────                                                          │
│                                                                     │
│  1. 코드 밀도 낮음                                                   │
│     • 단순 명령어도 32비트 사용                                      │
│     • 작은 즉시값도 전체 필드 할당                                   │
│                                                                     │
│  2. 메모리 낭비                                                      │
│     • NOP 패딩 필요                                                  │
│     • 사용하지 않는 필드 발생                                        │
│                                                                     │
│  3. 즉시값 크기 제한                                                 │
│     • 32비트 명령어 내에 32비트 상수 불가                            │
│     • 상수 풀(constant pool) 필요                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  코드 크기 비교 (동일 기능):                                     │ │
│  │  ────────────────────────────                                    │ │
│  │  C 코드: int sum = a + b + c;                                   │ │
│  │                                                               │ │
│  │  x86-64 (가변):  7 바이트                                       │ │
│  │  ARM (고정):    12 바이트 (3×32비트)                            │ │
│  │  Thumb (압축):   6 바이트 (3×16비트)                            │ │
│  │                                                               │ │
│  │  → 고정 길이는 코드 크기가 큼                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 고정 길이 명령어 × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                고정 길이 명령어 × 타 과목 융합                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일러】                                                        │
│  • 명령어 선택 단순화                                                │
│  • 스케줄링: 고정된 레이턴시 활용                                    │
│  • 레지스터 할당: 균일한 명령어 형식                                 │
│  • 코드 생성: 일관된 패턴                                            │
│                                                                     │
│  【운영체제】                                                        │
│  • 컨텍스트 스위치: PC만 +4                                          │
│  • 트랩/인터럽트: 명령어 경계 명확                                   │
│  • JIT 컴파일: 고정 길이 코드 생성 용이                              │
│  • 디버거: 명령어 단위 단계 실행                                     │
│                                                                     │
│  【컴퓨터 구조】                                                     │
│  • 파이프라인: 균일한 단계 설계                                      │
│  • 캐시: 명령어 캐시 라인 정렬                                       │
│  • 분기 예측: PC + 4 기본 예측                                       │
│  • 슈퍼스칼라: 다중 명령어 디스패치                                  │
│                                                                     │
│  【시스템 프로그래밍】                                               │
│  • 디스어셈블리: 간단한 디코딩                                       │
│  • 바이너리 분석: 일정한 간격                                        │
│  • 에뮬레이터: 단순한 인터프리터                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RISC 철학과 고정 길이 명령어:                                   │ │
│  │  ────────────────────────────                                    │ │
│  │  • 단순성: 복잡한 일은 컴파일러가                                 │ │
│  │  • 균일성: 모든 명령어가 1클럭                                   │ │
│  │  • 레지스터: 메모리 접근 최소화                                   │ │
│  │  • 파이프라인: 깊은 파이프라인 가능                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 고정 길이 명령어 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                고정 길이 명령어 분석                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ARM 어셈블리 분석】                                               │
│  ──────────────────────                                             │
│  // C 코드                                                           │
│  int add(int a, int b) { return a + b; }                            │
│                                                                     │
│  // ARM 어셈블리 (32비트 고정)                                       │
│  0000: e0810000  add r0, r1, r0    ; r0 = r1 + r0                   │
│  0004: e12fff1e  bx  lr            ; return                         │
│                                                                     │
│  【MIPS 어셈블리 분석】                                              │
│  ──────────────────────                                             │
│  // C 코드                                                           │
│  int sum = 0;                                                       │
│  for (int i = 0; i < 10; i++) sum += i;                             │
│                                                                     │
│  // MIPS 어셈블리 (32비트 고정)                                      │
│  0000: 20080000  addi $t0, $zero, 0    ; sum = 0                    │
│  0004: 20090000  addi $t1, $zero, 0    ; i = 0                      │
│  0008: 240a000a  addi $t2, $zero, 10   ; limit = 10                 │
│  000c: 01094020  add  $t0, $t0, $t1    ; sum += i                   │
│  0010: 21290001  addi $t1, $t1, 1      ; i++                        │
│  0014: 152afffc  bne  $t1, $t2, -4     ; loop                       │
│                                                                     │
│  【명령어 디코딩 수동 분석】                                         │
│  ──────────────────────────                                         │
│  명령어: 0x00892020 (MIPS)                                          │
│  이진: 000000 00100 01001 00100 00000 100000                        │
│        op     rs    rt    rd   shamt funct                          │
│  해석: add $a0, $a0, $t1                                            │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  고정 길이 명령어 디자인 팁:                                      │ │
│  │  ────────────────────────────                                     │ │
│  │  1. 자주 쓰는 명령어에 작은 opcode 할당                           │ │
│  │  2. 레지스터 필드는 같은 위치 고정                                │ │
│  │  3. 압축 명령어(16비트)로 코드 밀도 개선                          │ │
│  │  4. 확장을 위한 opcode 공간 확보                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 고정 길이 명령어: 모든 명령어 동일 비트 수
• 대표 ISA: ARM, MIPS, RISC-V (32비트)
• 장점: 단순 디코딩, 파이프라인 효율, 슈퍼스칼라 용이
• 단점: 낮은 코드 밀도, 메모리 낭비, 즉시값 제한
• 해결책: Thumb/RVC 등 16비트 압축 모드
```
