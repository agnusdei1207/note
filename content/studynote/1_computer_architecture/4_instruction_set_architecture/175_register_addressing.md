# 175. 레지스터 주소 지정 (Register Addressing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 피연산자가 CPU 레지스터에 있는 주소 지정 방식
> 2. **가치**: 가장 빠른 데이터 접근, 메모리 트래픽 없음
> 3. **융합**: 연산 최적화, 루프 변수, 임시 저장소

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

레지스터 주소 지정(Register Addressing)은 **피연산자가 CPU 내부 레지스터에 저장되어 있는 방식**이다. 메모리 접근 없이 레지스터 간 연산만으로 수행되어 가장 빠른 실행 속도를 제공한다.

### 💡 비유: 책상 위 메모지

레지스터 주소 지정은 **책상 위에 있는 메모지**를 읽는 것과 같다. 서랍(메모리)에서 찾지 않고 바로 눈앞에 있어서 즉시 볼 수 있다. 대신 메모지는 작아서 많은 내용을 담을 수 없다.

### 레지스터 주소 지정 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                레지스터 주소 지정 구조                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구조】                                                           │
│  ─────────                                                          │
│                                                                     │
│  ┌─────────────┬─────────────┬─────────────┐                        │
│  │   Opcode    │   Reg 1     │   Reg 2     │                        │
│  │   연산코드   │   원본       │   목적지     │                        │
│  └─────────────┴─────────────┴─────────────┘                        │
│                                                                     │
│  【특징】                                                           │
│  ─────────                                                          │
│  • 메모리 접근 없음                                                  │
│  • 실행 속도 최고 (0~1 클럭)                                        │
│  • 명령어 길이 짧음 (레지스터 번호만 필요)                           │
│  • 레지스터 수에 제한                                                │
│                                                                     │
│  【ISA별 레지스터 주소 지정】                                        │
│  ──────────────────────────────                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  ISA      문법              예시             레지스터 수       │ │
│  │  ───────  ────────────────  ────────────────  ──────────       │ │
│  │  x86-64   MOV reg1, reg2    MOV EAX, EBX      16개 (R0-R15)    │ │
│  │  ARM      MOV Rd, Rm        MOV R0, R1        16개 (R0-R15)    │ │
│  │  MIPS     add rd, rs, rt    add $t0,$t1,$t2   32개 ($0-$31)    │ │
│  │  RISC-V   add rd, rs1, rs2  add x5,x6,x7      32개 (x0-x31)    │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【레지스터 파일 구조】                                              │
│  ──────────────────────────                                         │
│                                                                     │
│          CPU                                                        │
│     ┌─────────────────┐                                             │
│     │  Register File  │                                             │
│     │  ┌───────────┐  │                                             │
│     │  │ R0: xxxx  │  │ ← 읽기/쓰기 0 클럭                          │
│     │  │ R1: xxxx  │  │                                             │
│     │  │ R2: xxxx  │  │                                             │
│     │  │ ...       │  │                                             │
│     │  │ R15: xxxx │  │                                             │
│     │  └───────────┘  │                                             │
│     └────────┬────────┘                                             │
│              │                                                      │
│     ┌────────┴────────┐                                             │
│     │     Memory      │ ← 읽기/쓰기 수십~수백 클럭                  │
│     └─────────────────┘                                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  레지스터 주소 지정 성능:                                         │ │
│  │  ────────────────────────────                                    │ │
│  │  • 레지스터-레지스터 연산: 1 클럭 (또는 0.5 클럭)                │ │
│  │  • 레지스터-메모리 연산: 3~10+ 클럭                              │ │
│  │  • 메모리-메모리: 대부분 ISA에서 지원 안함                        │ │
│  │                                                               │ │
│  │  → RISC: 모든 연산은 레지스터-레지스터                           │ │
│  │  → CISC: 메모리-레지스터 혼합 허용                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 레지스터 주소 지정 활용

### 활용 패턴

```
┌─────────────────────────────────────────────────────────────────────┐
│                레지스터 주소 지정 활용                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 데이터 이동】                                                  │
│  ──────────────────                                                  │
│  MOV EAX, EBX         ; EAX = EBX                                    │
│  MOV R0, R1           ; R0 = R1                                      │
│                                                                     │
│  【2. 산술 연산】                                                    │
│  ──────────────────                                                  │
│  ADD EAX, EBX         ; EAX = EAX + EBX                              │
│  SUB R0, R1, R2       ; R0 = R1 - R2                                 │
│  MUL R3, R4, R5       ; R3 = R4 * R5                                 │
│                                                                     │
│  【3. 논리 연산】                                                    │
│  ──────────────────                                                  │
│  AND EAX, EBX         ; EAX = EAX & EBX                              │
│  OR  R0, R1, R2       ; R0 = R1 | R2                                 │
│  XOR R3, R3           ; R3 = 0 (자신과 XOR)                          │
│                                                                     │
│  【4. 비교 연산】                                                    │
│  ──────────────────                                                  │
│  CMP EAX, EBX         ; EAX - EBX (플래그만 설정)                    │
│  CMP R0, R1           ; R0 - R1                                      │
│                                                                     │
│  【5. 주소 계산】                                                    │
│  ──────────────────                                                  │
│  LEA EAX, [EBX+ECX]   ; EAX = EBX + ECX                              │
│  ADD R0, R1, R2       ; R0 = R1 + R2 (주소)                          │
│                                                                     │
│  【레지스터 활용 최적화】                                            │
│  ──────────────────────────                                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  나쁨 (메모리 사용):                                              │ │
│  │  MOV [mem1], EAX                                                │ │
│  │  MOV EBX, [mem1]       ; 메모리 2회 접근                        │ │
│  │                                                               │ │
│  │  좋음 (레지스터 사용):                                            │ │
│  │  MOV EBX, EAX          ; 레지스터만 사용                        │ │
│  │                                                               │ │
│  │  컴파일러 최적화:                                                 │ │
│  │  • 레지스터 할당 (Register Allocation)                           │ │
│  │  • 스파illing 최소화                                              │ │
│  │  • 레지스터 coalescing                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  • 포워딩: 레지스터 간 데이터 전달                                   │

│  │  • 메모리: Load/Store만                                          │ │
│  │  • 장점: 단순한 파이프라인                                       │ │
│  │                                                               │ │
│  │  CISC (x86):                                                    │ │
│  │  • 연산: 메모리-레지스터 혼합 허용                                │ │
│  │  • 장점: 코드 밀도                                               │ │
│  │  • 단점: 복잡한 디코딩                                           │ │

## Ⅳ. 실무 적용

### 레지스터 주소 지정 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                레지스터 주소 지정 분석                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【GDB로 레지스터 확인】                                            │
│  ──────────────────────────                                        │
│  (gdb) info registers          ; 모든 레지스터                      │
│  (gdb) p $eax                  ; EAX 값                             │
│  (gdb) p/x $rax                ; RAX 값 (16진수)                    │
│                                                                     │
│  【어셈블리 분석 예시】                                              │
│  ──────────────────────────                                         │
│  // C 코드                                                           │
│  int add(int a, int b) { return a + b; }                            │
│                                                                     │
│  // x86-64                                                          │
│  mov eax, edi          ; a를 eax로 (레지스터-레지스터)               │
│  add eax, esi          ; b를 eax에 더함 (레지스터-레지스터)          │
│  ret                   ; eax 반환                                   │
│                                                                     │
│  // ARM                                                             │
│  ADD R0, R0, R1        ; R0 = R0 + R1                               │
│  BX LR                 ; R0 반환                                    │
│                                                                     │
│  【레지스터 호출 규약 (x86-64 System V)】                            │
│  ──────────────────────────────────────                             │
│  인자 전달: RDI, RSI, RDX, RCX, R8, R9                              │
│  반환값: RAX                                                        │
│  Caller-saved: RAX, RCX, RDX, RSI, RDI, R8-R11                     │
│  Callee-saved: RBX, RBP, R12-R15                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  레지스터 최적화 체크리스트:                                      │ │
│  │  ────────────────────────────                                     │ │
│  │  □ 자주 쓰는 변수를 레지스터에                                    │ │
│  │  □ 메모리 접근 최소화                                             │ │
│  │  □ 레지스터 스파illing 줄이기                                     │ │
│  │  □ 호출 규약 준수                                                  │ │
│  │  □ 레지스터 해저드 고려                                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 레지스터 주소 지정: 피연산자가 레지스터에 있음
• 장점: 가장 빠름, 메모리 트래픽 없음, 짧은 명령어
• 단점: 레지스터 수 제한, 컴파일러 최적화 필요
• RISC: 모든 연산이 레지스터-레지스터
• 활용: 연산, 이동, 비교, 주소 계산
```
