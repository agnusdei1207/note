> **핵심 인사이트**
> 1. 버스트 모드(Burst Mode)는 CPU나 DMA가 메모리, 디스크 등에서 데이터를 읽어올 때, **시작 주소(Starting Address) 한 번만 보내면 그 뒤에 연속된 여러 개의 데이터를 매 클럭마다 쉬지 않고 쏟아내도록(Burst) 하는 고속 데이터 전송 기술**이다.
> 2. 데이터를 1개 요청할 때마다 발생하는 '주소 전송 $\rightarrow$ 대기 $\rightarrow$ 데이터 수신'의 끔찍한 **오버헤드(지연 시간)를 1번으로 압축시켜 전체 시스템 버스의 대역폭(Bandwidth)을 극한으로 끌어올린다.**
> 3. 메모리에 있는 데이터가 연속적으로 뭉쳐있을 것이라는 **공간적 지역성(Spatial Locality)**이 보장될 때만 의미가 있으며, 캐시 라인(Cache Line)을 채우는 가장 핵심적인 하드웨어 동작 방식이다.

---

## Ⅰ. 단일 전송의 비효율성 (Single Transfer)

구형 컴퓨터에서 메모리로부터 데이터를 4번 읽어오려면 아주 피곤한 대화가 4번 반복됩니다.

1. CPU: "100번지 데이터 줘" $\rightarrow$ 메모리: (찾는 중...) $\rightarrow$ "여기 A"
2. CPU: "101번지 데이터 줘" $\rightarrow$ 메모리: (찾는 중...) $\rightarrow$ "여기 B"
3. CPU: "102번지 데이터 줘" $\rightarrow$ 메모리: (찾는 중...) $\rightarrow$ "여기 C"
4. CPU: "103번지 데이터 줘" $\rightarrow$ 메모리: (찾는 중...) $\rightarrow$ "여기 D"

- **문제점:** 데이터를 찾는 시간(Latency)이 10클럭이라면, 고작 4개의 데이터를 받기 위해 **총 40클럭**을 허비해야 합니다. 실제 데이터를 전송하는 시간보다 "요청하고 기다리는 시간"이 훨씬 더 길어지는 심각한 배보다 배꼽이 큰 상황입니다.

📢 **섹션 요약 비유:**
자판기에서 음료수 4개를 뽑을 때, 동전 1개 넣고 기다려서 1개 받고, 다시 동전 1개 넣고 기다려서 1개 받는 답답한 방식입니다.

## Ⅱ. 버스트 모드의 마법 (Burst Transfer)

이 오버헤드를 없애기 위해 현대의 버스(Bus) 프로토콜은 '연속 발사(Burst)' 기능을 추가했습니다.

1. CPU: "100번지부터 **버스트 길이 4(Burst Length 4)**로 줘!"
2. 메모리: (처음 한 번만 10클럭 동안 찾는 중...)
3. 메모리: "여기 A!" (1클럭)
4. 메모리: "여기 B!" (1클럭)
5. 메모리: "여기 C!" (1클럭)
6. 메모리: "여기 D!" (1클럭)

- **결과:** 똑같이 4개의 데이터를 받는데, 처음 대기하는 10클럭을 빼면 나머지 3개는 3클럭 만에 연속으로 튀어나옵니다. 총 40클럭 걸리던 작업이 **13클럭**으로 확 줄어들어 속도가 3배 이상 뻥튀기됩니다.

📢 **섹션 요약 비유:**
동전을 한 번에 4개어치 털어 넣고 "콜라 4개 뽑아줘!"라고 버튼 한 번 누르면, 자판기가 "툭, 툭, 툭, 툭" 하고 1초 간격으로 음료수를 연속해서 토해내는 아주 시원한 방식입니다.

## Ⅲ. 캐시 메모리와 버스트 모드의 찰떡궁합

이 버스트 모드는 CPU 내부의 캐시 메모리(Cache Memory) 시스템이 작동하는 절대적인 기반입니다.

- **캐시 라인(Cache Line):** L1 캐시는 메모리에서 1바이트만 가져오지 않고, 무조건 64바이트 덩어리(Cache Line)를 통째로 가져옵니다. 
- **버스트 전송의 활약:** 만약 64바이트를 1바이트씩 64번 따로 요청했다면 캐시는 오히려 시스템을 느리게 만드는 원흉이 되었을 것입니다. 하지만 버스트 모드를 켜서 "0번지부터 64바이트를 버스트로 쏴!"라고 명령하면, 64개의 데이터가 파도처럼 밀려들어 와 순식간에 캐시를 채워 넣습니다.
- **공간적 지역성:** 어차피 1번 데이터를 불렀으면 바로 옆에 있는 2, 3, 4번 데이터를 부를 확률이 높으므로(공간적 지역성), 이렇게 한 번에 쏟아내는 것이 압도적으로 유리합니다.

📢 **섹션 요약 비유:**
물을 컵으로 64번 퍼 나르는 게 아니라, 아예 굵은 호스를 꽂아놓고 한 번에 물을 콸콸 쏟아부어 거대한 수조(캐시)를 단숨에 채워버리는 원리입니다.

## Ⅳ. DMA 사이클 스틸링과의 관계

DMA(직접 메모리 접근)가 짐을 나를 때도 이 버스트 모드가 중요하게 쓰입니다.

- **Burst Mode DMA:** DMA가 한 번 버스를 차지하면, 1MB어치 데이터를 버스트 모드로 끝까지 다 쏟아낼 때까지 CPU에게 버스를 돌려주지 않습니다. 데이터 전송 속도는 우주 최강이지만, CPU가 너무 오래 굶어 죽는(Stall) 단점이 있습니다.
- **Cycle Stealing DMA:** CPU가 눈치채지 못하게 아주 조금씩만 버스를 훔쳐서 데이터를 전송합니다. CPU는 멈추지 않고 돌아가지만, DMA 입장에서는 버스트 모드의 혜택을 온전히 받지 못해 전송 속도가 느립니다.

📢 **섹션 요약 비유:**
도로를 통제하고 VIP 호송차 100대를 한 번에 신호등 없이 통과시키는 것(버스트 모드)과, 일반 차들 사이에 VIP 차를 한 대씩 눈치껏 끼워 넣는 것(사이클 스틸링)의 차이입니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- `[Spatial Locality]`: 버스트 모드로 왕창 가져온 데이터가 쓰레기가 되지 않고 유용하게 쓰일 수 있는 이유
- `[Cache Line / Block]`: 버스트 모드가 한 번에 가져와야 하는 데이터 전송의 규격화된 덩어리 크기
- `[SDRAM (Synchronous DRAM)]`: 메모리 칩 내부에 클럭 동기화 기술을 넣어 이 버스트 모드를 하드웨어적으로 지원하기 시작한 현대 메모리
- `[DMA]`: CPU를 거치지 않고 I/O 장치와 메모리 간에 버스트 모드로 대용량 데이터를 쏘는 컨트롤러
- `[Cycle Stealing]`: 버스트 모드로 버스를 독점할 때 생기는 CPU 마비 문제를 해결하기 위한 타협적 DMA 전송 기법

1. 버스트 모드는 자판기에서 "콜라 5개 연속으로 뽑아줘!"라고 버튼 한 번만 누르는 마법이에요.
2. 옛날에는 동전 넣고 버튼 누르고, 또 동전 넣고 버튼 누르고를 5번 반복해야 해서 팔이 너무 아팠어요.
3. 하지만 버스트 모드를 쓰면 처음 한 번만 누르고 기다리면 밑에서 콜라가 툭, 툭, 툭 하고 쉬지 않고 떨어져서 아주 편하답니다!
