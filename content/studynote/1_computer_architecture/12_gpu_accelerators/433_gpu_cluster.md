# 433. GPU 클러스터 (GPU Cluster)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다수의 GPU를 네트워크로 연결한 대규모 병렬 컴퓨팅 시스템
> 2. **가치**: 초거대 모델 학습, 분산 처리, 확장성
> 3. **융합**: DGX, HGX, NVLink, InfiniBand, Kubernetes

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

GPU 클러스터는 **다수의 GPU 서버를 고속 네트워크(InfiniBand, NVLink)로 연결하여 대규모 병렬 컴퓨팅을 수행하는 시스템**이다.

### 💡 비유: 슈퍼팀

GPU 클러스터는 **슈퍼팀**과 같다. 혼자서는 한계가 있다. (단일 GPU) 여러 전문가가 모인다. (GPU 클러스터) 각자 역할. 협력. 큰 일 해낸다. GPU 클러스터가 똑같다. 다중 GPU, 분산, 협력, 대규모 학습.

```
┌─────────────────────────────────────────────────────────────────────┐
│                GPU 클러스터 분석                                           │

│                                                                     │

│  ──────────────────                                                  │

│                                                                     │
│  # 모델 래핑                                                           │
│  model = DDP(model.to(device), device_ids=[local_rank])             │

│  ──────────────────                                                  │
│  #!/bin/bash                                                        │
│  #SBATCH --job-name=train                                           │
│  #SBATCH --nodes=4                                                  │
│  #SBATCH --ntasks-per-node=8                                        │
│  #SBATCH --gres=gpu:8                                               │
│                                                                     │
│  srun python train.py                                               │
│                                                                     │
│  【Kubernetes GPU 스케줄링】                                                       │)
│  ──────────────────                                                  │

│    limits:                                                          │
│      nvidia.com/gpu: 8                                              │

│  ──────────────────                                                  │

│  4. 모니터링: DCGM, Prometheus                                       │
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• GPU 클러스터: 다중 GPU 분산 시스템
• 계층: GPU → Node → Rack → Pod
• 통신: NVLink (노드 내), IB (노드 간)
• NCCL: All-Reduce, All-Gather
• DGX/HGX: NVIDIA 시스템
• SuperPOD: 대규모 클러스터
• 장점: 대규모 학습, 확장성
• 단점: 비용, 복잡성, 통신
• 활용: LLM, HPC, AI 연구
• 오케스트레이션: SLURM, K8s
```
