## Ⅰ. 개요 (Context & Background)

### 개념 정의

OpenCL(Open Computing Language)은 **CPU, GPU, DSP, FPGA 등 다양한 이기종 프로세서에서 병렬 프로그램을 실행할 수 있게 하는 개방형 표준 프레임워크**다. Apple이 제안하고 Khronos Group이 관리한다.

### 💡 비유: 만국 공통어

OpenCL은 **만국 공통어**와 같다. 영어가 아니다. 한국어가 아니다. 모두가 이해하는 언어. 어느 나라에서든 통한다. CPU에서도. GPU에서도. FPGA에서도. OpenCL이 똑같다. 표준, 이식성, 다양한 하드웨어.

## Ⅱ. OpenCL 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                OpenCL 상세                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【OpenCL 실행 모델】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  NDRange (N-Dimensional Range):                                           │ │
│  │  • 전역 워크 크기: 전체 스레드 수                                          │ │
│  │  • 로컬 워크 크기: 워크그룹 크기                                           │ │
│  │                                                               │ │
│  │  계층:                                                                    │ │
│  │  NDRange > Work-Group > Work-Item                                         │ │
│  │  (Grid)   > (Block)    > (Thread)    ← CUDA 대응                          │ │
│  │                                                               │ │
│  │  내장 변수:                                                                  │ │
│  │  • get_global_id(dim): 전역 인덱스                                        │ │
│  │  • get_local_id(dim): 로컬 인덱스                                         │ │
│  │  • get_global_size(dim): 전역 크기                                        │ │
│  │  • get_local_size(dim): 로컬 크기                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【OpenCL 커널 예시】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  __kernel void vector_add(                                                │ │
│  │      __global const float *a,                                              │ │
│  │      __global const float *b,                                              │ │
│  │      __global float *c,                                                    │ │
│  │      const unsigned int n)                                                 │ │
│  │  {                                                                         │ │
│  │      int i = get_global_id(0);                                             │ │
│  │      if (i < n) {                                                          │ │
│  │          c[i] = a[i] + b[i];                                               │ │
│  │      }                                                                     │ │
│  │  }                                                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 모델】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • __global: 전역 메모리 (모든 워크아이템)                                │ │
│  │  • __local: 로컬 메모리 (워크그룹 내)                                     │ │
│  │  • __private: 프라이빗 메모리 (워크아이템)                                │ │
│  │  • __constant: 상수 메모리                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• OpenCL: 이기종 병렬 컴퓨팅 표준
• 하드웨어: CPU, GPU, DSP, FPGA
• 이식성: 높음, 다양한 플랫폼
• 성능: CUDA보다 낮음
• 커널: __kernel 함수
• 계층: NDRange > Work-Group > Work-Item
• 메모리: global, local, private, constant
• API: clCreateBuffer, clEnqueueNDRangeKernel
• 장점: 이식성, 개방형
• 단점: 성능, 복잡성
```
