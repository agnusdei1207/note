# 417. GPU 아키텍처 (GPU Architecture)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 대규모 병렬 처리를 위한 SIMD 기반 그래픽 처리 장치
> 2. **가치**: 높은 처리량, 데이터 병렬성, 범용 컴퓨팅(GPGPU)
> 3. **융합**: CUDA, SIMT, 워프, 스트리밍 멀티프로세서

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

GPU 아키텍처(GPU Architecture)는 **수천 개의 작은 코어를 병렬로 배치하여 대규모 데이터 병렬 처리에 최적화된 프로세서 구조**다. 원래 그래픽 렌더링용이었으나 현재는 범용 병렬 컴퓨팅(GPGPU)에도 활용된다.

### 💡 비유: 군인 부대

GPU 아키텍처는 **군인 부대**와 같다. 장군 한 명이 아니다. 수천 명의 군인. 모두 같은 명령. 한 번에 움직임. 개별적이지 않다. 집단 행동. GPU가 똑같다. 수천 코어, 같은 명령, SIMT, 데이터 병렬성.

## Ⅱ. GPU 아키텍처 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                GPU 아키텍처 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【SIMT 실행 모델】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  SIMT (Single Instruction, Multiple Threads):                             │ │
│  │                                                               │ │
│  │  • 워프 (Warp): 32개 스레드가 같은 명령 실행                              │ │
│  │  • 스레드: 개별 데이터 처리                                              │ │
│  │  • 분기 발산: 워프 내 분기 시 순차 실행                                   │ │
│  │                                                               │ │
│  │  워프 실행:                                                                  │ │
│  │  ┌────────────────────────────────────────┐                    │ │
│  │  │  워프 (32 스레드)                        │                    │ │
│  │  │  [T0][T1][T2]...[T31]                   │                    │ │
│  │  │    ↓  ↓  ↓       ↓                      │                    │ │
│  │  │  [ADD][ADD][ADD]...[ADD]  ← 같은 명령   │                    │ │
│  │  │    ↓  ↓  ↓       ↓                      │                    │ │
│  │  │  [D0][D1][D2]...[D31]    ← 다른 데이터   │                    │ │
│  │  └────────────────────────────────────────┘                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 계층】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 레지스터: 스레드 전용, 가장 빠름                                     │ │
│  │  2. 공유 메모리: 블록 내 공유, 낮은 지연                                  │ │
│  │  3. L1/L2 캐시: 하드웨어 관리                                            │ │
│  │  4. 글로벌 메모리: HBM/GDDR, 높은 대역폭                                  │ │
│  │  5. 상수 메모리: 읽기 전용, 캐시됨                                       │ │
│  │  6. 텍스처 메모리: 2D 공간 지역성                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 높은 처리량: TFLOPS급                                               │ │
│  │  • 높은 메모리 대역폭                                                  │ │
│  │  • 데이터 병렬성에 최적                                                │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 분기 발산: 성능 저하                                                │ │
│  │  • 순차 코드: 부적합                                                   │ │
│  │  • PCIe: 호스트 전송 병목                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• GPU: 대규모 병렬 처리 장치
• SIMT: 단일 명령 다중 스레드
• 워프: 32 스레드 그룹
• SM: 스트리밍 멀티프로세서
• 메모리: 레지스터→공유→글로벌
• 장점: 높은 처리량, 대역폭
• 단점: 분기 발산, 순차 코드 부적합
• 활용: 딥러닝, 과학 컴퓨팅
• CUDA: NVIDIA 프로그래밍
• 전력: 와트당 성능 높음
```
