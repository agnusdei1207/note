# 427. 양자화 (Quantization)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 높은 정밀도 숫자를 낮은 정밀도로 변환하여 메모리/연산 효율화
> 2. **가치**: 모델 축소, 추론 가속, 전력 절감
> 3. **융합**: INT8, FP16, PTQ, QAT, 텐서 코어

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

양자화(Quantization)는 **FP32 모델의 가중치와 활성화를 INT8, FP16, INT4 등 저정밀도로 변환하여 모델 크기를 줄이고 추론 속도를 높이는 모델 압축 기법**이다.

### 💡 비유: 픽셀 이미지 축소

양자화는 **픽셀 이미지 축소**와 같다. 원본은 4K. (FP32) 선명하다. 용량이 크다. 1080p로 줄인다. (INT8) 약간 손실. 용량 1/4. 여전히 잘 보인다. 양자화가 똤같다. 정밀도 감소, 크기 감소, 속도 향상.

### 양자화 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                양자화 개요                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【양자화 종류】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 학습 후 양자화 (PTQ: Post-Training Quantization):                       │ │
│  │     • 학습 완료된 모델 양자화                                               │ │
│  │     • 보정 데이터 필요                                                     │ │
│  │     • 빠른 적용                                                           │ │
│  │                                                               │ │
│  │  2. 양자화 인식 학습 (QAT: Quantization-Aware Training):                      │ │
│  │     • 학습 시 양자화 고려                                                   │ │
│  │     • 정확도 손실 최소화                                                    │ │
│  │     • 시간 소요                                                            │ │
│  │                                                               │ │
│  │  3. 동적 양자화 (Dynamic Quantization):                                      │ │
│  │     • 추론 시 활성화 양자화                                                 │ │
│  │     • 가중치는 미리 양자화                                                  │ │
│  │     • 유연성                                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【정밀도 비교】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  타입         비트    범위                    크기       속도    │ │
│  │  ────         ────    ────                    ────       ────    │ │
│  │  FP32         32      ±3.4e38                100%       1x      │ │
│  │  FP16         16      ±65504                 50%        2-4x    │ │
│  │  BF16         16      ±3.4e38 (동적범위)      50%        2-4x    │ │
│  │  INT8         8       -128 ~ 127             25%        4-8x    │ │
│  │  INT4         4       -8 ~ 7                 12.5%      8-16x   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 양자화 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                양자화 상세                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【양자화 공식】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  양자화:                                                                   │ │
│  │  q = round(r / S) + Z                                                     │ │
│  │                                                               │ │
│  │  역양자화:                                                                  │ │
│  │  r ≈ S × (q - Z)                                                          │ │
│  │                                                               │ │
│  │  r: 실수 값 (real)                                                        │ │
│  │  q: 정수 값 (quantized)                                                   │ │
│  │  S: 스케일 (scale)                                                        │ │
│  │  Z: 영점 (zero point)                                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【양자화 방식】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 대칭 양자화 (Symmetric):                                                 │ │
│  │     • -128 ~ 127 (INT8)                                                   │ │
│  │     • Z = 0                                                                │ │
│  │     • 단순, 구현 쉬움                                                      │ │
│  │                                                               │ │
│  │  2. 비대칭 양자화 (Asymmetric):                                              │ │
│  │     • 0 ~ 255 (UINT8)                                                     │ │
│  │     • Z ≠ 0                                                                │ │
│  │     • 더 정확한 범위                                                       │ │
│  │                                                               │ │
│  │  3. 채널별 vs 레이어별:                                                      │ │
│  │     • Per-channel: 채널마다 스케일                                          │ │
│  │     • Per-tensor: 텐서 전체 스케일                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【INT8 vs FP32 성능】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  지표           FP32          INT8          개선               │ │
│  │  ────           ────          ────          ────               │ │
│  │  모델 크기       100%          25%          4x 감소            │ │
│  │  메모리 대역폭   100%          25%          4x 감소            │ │
│  │  연산량         100%          ~25%         4x 감소            │ │
│  │  전력 소비      100%          ~30%         3x 감소            │ │
│  │  지연 시간      100%          ~20%         5x 감소            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 모델 크기 감소                                                        │ │
│  │  • 추론 속도 향상                                                        │ │
│  │  • 전력 소비 감소                                                        │ │
│  │  • 하드웨어 활용                                                         │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 정확도 손실                                                          │ │
│  │  • 복잡한 보정                                                          │ │
│  │  • 하드웨어 의존                                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 양자화 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                양자화 분석                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【PyTorch PTQ】                                                                │)
│  ──────────────────                                                  │
│  import torch                                                       │
│  from torch.quantization import quantize_dynamic                    │
│                                                                     │
│  # 동적 양자화                                                        │
│  model_quantized = quantize_dynamic(                                │
│      model, {torch.nn.Linear}, dtype=torch.qint8                    │
│  )                                                                  │
│                                                                     │
│  【TensorRT INT8】                                                              │)
│  ──────────────────                                                  │
│  import tensorrt as trt                                             │
│                                                                     │
│  # INT8 플래그 활성화                                                 │
│  config.set_flag(trt.BuilderFlag.INT8)                              │
│  config.int8_calibrator = calibrator                                │
│                                                                     │
│  【GPTQ (LLM 양자화)】                                                           │)
│  ──────────────────                                                  │
│  from auto_gptq import AutoGPTQForCausalLM                          │
│                                                                     │
│  model = AutoGPTQForCausalLM.from_quantized(                        │
│      model_path, device_map="auto"                                  │
│  )                                                                  │
│                                                                     │
│  【양자화 최적화 팁】                                                          │)
│  ──────────────────                                                  │
│  1. 보정: 대표 데이터 사용                                            │
│  2. 채널별: 정확도 향상                                               │
│  3. QAT: 정확도 중요 시                                               │
│  4. 하이브리드: 민감 레이어 FP16 유지                                  │
│  5. 프로파일링: 병목 식별                                             │
│  6. 하드웨어: NPU/GPU 활용                                            │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 양자화: FP32 → INT8/FP16 변환
• PTQ: 학습 후 양자화
• QAT: 양자화 인식 학습
• 동적 양자화: 추론 시 활성화
• INT8: 4x 크기 감소, 4x 속도
• INT4: 8x 크기 감소, 8x 속도
• 대칭/비대칭: 영점 유무
• 장점: 크기, 속도, 전력
• 단점: 정확도 손실
• 활용: 엣지 AI, 추론, LLM
```
