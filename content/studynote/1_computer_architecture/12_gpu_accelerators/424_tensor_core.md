# 424. 텐서 코어 (Tensor Core)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 행렬 연산에 특화된 GPU 전용 연산 유닛

> 3. **융합**: AI 가속기, 딥러닝, 혼합 정밀도, Transformer

## Ⅰ. 개요 (Context & Background)

### 개념 정의

텐서 코어(Tensor Core)는 **NVIDIA GPU에 탑재된 행렬 연산 전용 하드웨어로, 혼합 정밀도에서 D = A × B + C 형태의 행렬 곱셈을 한 사이클에 수행하여 딥러닝 연산을 가속한다.**

### 💡 비유: 행렬 계산기

텐서 코어는 **행렬 계산기**와 같다. 일반 계산기가 있다. (CUDA 코어) 하나씩 계산. 행렬 계산기가 있다. (텐서 코어) 행렬 전체를 한 번에. 특화되었다. 빠르다. 텐서 코어가 똑같다. 행렬 곱셈, 전용 하드웨어, 매우 빠름.

│  │                                                               │ │
│  │  특징:                                                                    │ │
│  │  • 1 사이클에 4×4×4 = 64 연산                                             │ │
│  │  • 혼합 정밀도: FP16 입력 → FP32 출력                                    │ │
│  │  • 누적: FP32 정밀도                                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │

│  │  1세대       Volta        FP16               ~125 TFLOPS        │ │
│  │  2세대       Turing       FP16/INT8/INT4     ~130 TFLOPS        │ │
│  │  3세대       Ampere       FP16/BF16/TF32     ~312 TFLOPS        │ │
│  │  4세대       Hopper       FP16/BF16/FP8      ~1000 TFLOPS       │ │
│  │  5세대       Blackwell     FP4/FP8/FP16      ~2500 TFLOPS       │ │
│  │                                                               │ │

└─────────────────────────────────────────────────────────────────────┘

## Ⅱ. 텐서 코어 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                텐서 코어 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【혼합 정밀도 연산】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  FP16 (Half Precision):                                                  │ │
│  │  • 16비트 부동소수점                                                     │ │
│  │  • 입력: A, B                                                           │ │
│  │  • 메모리 절약, 대역폭 효율                                              │ │
│  │                                                               │ │
│  │  FP32 (Single Precision):                                                │ │
│  │  • 32비트 부동소수점                                                     │ │
│  │  • 누적: C, D                                                           │ │
│  │  • 정밀도 유지                                                          │ │
│  │                                                               │ │
│  │  BF16 (BFloat16):                                                        │ │
│  │  • 16비트, FP32와 동일 지수부                                            │ │
│  │  • 동적 범위 유지                                                        │ │
│  │  • LLM 학습에 적합                                                       │ │
│  │                                                               │ │
│  │  TF32 (TensorFloat-32):                                                  │ │
│  │  • 19비트 (10비트 가수, 8비트 지수)                                       │ │
│  │  • Ampere 기본                                                          │ │
│  │                                                               │ │
│  │  INT8/INT4:                                                              │ │
│  │  • 정수 연산                                                             │ │
│  │  • 양자화된 모델                                                         │ │
│  │  • 추론 최적화                                                           │ │
│  │                                                               │ │
│  │  FP8/FP4:                                                                │ │
│  │  • Hopper/Blackwell                                                     │ │
│  │  • 초저정밀도                                                            │ │
│  │  • 대규모 모델 학습                                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【텐서 코어 활용】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. GEMM (General Matrix Multiply):                                      │ │
│  │     • 행렬-행렬 곱셈                                                     │ │
│  │     • 딥러닝의 핵심                                                      │ │
│  │                                                               │ │
│  │  2. 합성곱 (Convolution):                                                 │ │
│  │     • im2col + GEMM                                                     │ │
│  │     • 이미지 처리                                                        │ │
│  │                                                               │ │
│  │  3. Transformer:                                                         │ │
│  │     • Attention: Q × K^T                                                 │ │
│  │     • FFN: 행렬 곱셈                                                     │ │
│  │                                                               │ │
│  │  4. 희소 행렬 (Sparse):                                                   │ │
│  │     • 2:4 희소성                                                         │ │
│  │     • Ampere A100                                                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 매우 높은 처리량                                                     │ │
│  │  • 에너지 효율                                                          │ │
│  │  • 혼합 정밀도                                                          │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 특정 연산만                                                          │ │
│  │  • 행렬 크기 제약                                                       │ │
│  │  • 프로그래밍 복잡성                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 텐서 코어 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                텐서 코어 분석                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【텐서 코어 활성화 확인】                                                            │)
│  ──────────────────                                                  │
│  $ nvidia-smi -a | grep "Tensor Cores"                              │
│  # 텐서 코어 존재 확인                                                 │
│                                                                     │
│  $ ncu --metrics tensor_core_utilization ./program                   │
│  # 텐서 코어 활용률                                                   │
│                                                                     │
│  【PyTorch 혼합 정밀도】                                                            │)
│  ──────────────────                                                  │
│  import torch                                                       │
│  from torch.cuda.amp import autocast, GradScaler                    │
│                                                                     │
│  scaler = GradScaler()                                              │
│  for data, target in dataloader:                                    │
│      with autocast():                                               │
│          output = model(data)                                       │
│          loss = criterion(output, target)                           │
│      scaler.scale(loss).backward()                                  │
│      scaler.step(optimizer)                                         │
│      scaler.update()                                                │
│                                                                     │
│  【CUDA WMMA (Warp Matrix Multiply Accumulate)】                                  │)
│  ──────────────────                                                  │
│  #include <mma.h>                                                   │
│  using namespace nvcuda::wmma;                                      │
│                                                                     │
│  fragment<matrix_a, 16, 16, 16, half, row_major> a_frag;            │
│  fragment<matrix_b, 16, 16, 16, half, col_major> b_frag;            │
│  fragment<accumulator, 16, 16, 16, float> c_frag;                   │
│                                                                     │
│  load_matrix_sync(a_frag, a_ptr, 16);                               │
│  load_matrix_sync(b_frag, b_ptr, 16);                               │
│  fill_fragment(c_frag, 0.0f);                                       │
│  mma_sync(c_frag, a_frag, b_frag, c_frag);                          │
│  store_matrix_sync(c_ptr, c_frag, 16, mem_row_major);               │
│                                                                     │
│  【텐서 코어 최적화 팁】                                                          │)
│  ──────────────────                                                  │
│  1. 행렬 크기: 8/16의 배수                                            │
│  2. 정렬: 128바이트                                                  │
│  3. 혼합 정밀도: AMP 활용                                             │
│  4. 라이브러리: cuBLAS, cuDNN                                        │
│  5. 희소성: 2:4 sparsity                                             │
│  6. 배치: 크기 최적화                                                 │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 텐서 코어: 행렬 연산 전용 유닛
• 연산: D = A × B + C
• 세대: Volta → Turing → Ampere → Hopper → Blackwell
• 정밀도: FP16, BF16, TF32, INT8, FP8, FP4
• 성능: 최대 2500 TFLOPS (FP16)
• 혼합 정밀도: FP16 입력 → FP32 누적
• 활용: GEMM, CNN, Transformer
• WMMA: CUDA 프로그래밍 API
• 장점: 처리량, 에너지 효율
• 단점: 행렬 크기 제약
```
