# 421. GPU 메모리 (GPU Memory)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: GPU의 계층적 메모리 구조 (레지스터-공유-글로벌)
> 2. **가치**: 높은 대역폭, 워프-워프 병렬 접근
> 3. **융합**: HBM, GDDR, 공유 메모리, 메모리 코얼레싱

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

GPU 메모리는 **GPU 내부의 계층적 메모리 구조로, 레지스터, 공유 메모리, L1/L2 캐시, 글로벌 메모리(HBM/GDDR)로 구성되며 높은 대역폭과 병렬 접근을 제공한다.**

### 💡 비유: 회사 건물

GPU 메모리는 **회사 건물**과 같다. 책상이 있다. (레지스터) 팀 회의실이 있다. (공유 메모리) 사무실이 있다. (L1) 건물 전체 창고가 있다. (글로벌 메모리) 가까울수록 빠르다. 멀수록 느리다. GPU 메모리가 똑같다. 계층, 속도, 용량 트레이드오프.

### GPU 메모리 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                GPU 메모리 개요                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메모리 계층 구조】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  속도 ↑                     용량 ↓                            │ │
│  │  ──────────────────────────────────────────────────────→      │ │
│  │                                                               │ │
│  │  ┌──────────────┐                                            │ │
│  │  │   Registers  │  스레드 전용, 가장 빠름                       │ │
│  │  │  (~64KB/SM)  │  지연: 1 사이클                              │ │
│  │  └──────────────┘                                            │ │
│  │         ↓                                                     │ │
│  │  ┌──────────────┐                                            │ │
│  │  │Shared Memory │  블록 전용, 높은 대역폭                       │ │
│  │  │ (~100KB/SM)  │  지연: ~20 사이클                             │ │
│  │  └──────────────┘                                            │ │
│  │         ↓                                                     │ │
│  │  ┌──────────────┐                                            │ │
│  │  │  L1 Cache    │  SM 전용                                     │ │
│  │  │ (~128KB/SM)  │  지연: ~30 사이클                             │ │
│  │  └──────────────┘                                            │ │
│  │         ↓                                                     │ │
│  │  ┌──────────────┐                                            │ │
│  │  │  L2 Cache    │  GPU 전역                                    │ │
│  │  │  (~6MB)      │  지연: ~200 사이클                            │ │
│  │  └──────────────┘                                            │ │
│  │         ↓                                                     │ │
│  │  ┌──────────────┐                                            │ │
│  │  │Global Memory │  HBM/GDDR, 가장 느림                          │ │
│  │  │  (16-80GB)   │  지연: ~400 사이클                            │ │
│  │  │  대역폭: ~TB/s│                                            │ │
│  │  └──────────────┘                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【GPU 메모리 유형】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  HBM (High Bandwidth Memory):                                            │ │
│  │  • 3D 스택 구조                                                         │ │
│  │  • 매우 높은 대역폭 (~2TB/s)                                             │ │
│  │  • 데이터센터 GPU (A100, H100)                                           │ │
│  │                                                               │ │
│  │  GDDR (Graphics DDR):                                                    │ │
│  │  • 전통적 2D 구조                                                        │ │
│  │  • 높은 대역폭 (~1TB/s)                                                  │ │
│  │  • 소비자/게이밍 GPU (RTX 4090)                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. GPU 메모리 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                GPU 메모리 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【공유 메모리 활용】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  __shared__ float tile[16][16];                                           │ │
│  │                                                               │ │
│  │  // 타일 로딩: 글로벌 → 공유                                              │ │
│  │  __syncthreads();                                                          │ │
│  │  // 타일 연산                                                              │ │
│  │  __syncthreads();                                                          │ │
│  │                                                               │ │
│  │  장점:                                                                    │ │
│  │  • 높은 대역폭                                                          │ │
│  │  • 낮은 지연                                                            │ │
│  │  • 데이터 재사용                                                         │ │
│  │                                                               │ │
│  │  용도: 행렬 곱셈, 합성곱, 스텐실                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 코얼레싱】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  문제: 워프 내 스레드가 떨어진 주소 접근                                  │ │
│  │                                                               │ │
│  │  비효율적:                                                                 │ │
│  │  T0: data[0], T1: data[128], T2: data[256]...                            │ │
│  │  → 여러 메모리 트랜잭션                                                 │ │
│  │                                                               │ │
│  │  효율적:                                                                   │ │
│  │  T0: data[0], T1: data[1], T2: data[2]...                                │ │
│  │  → 하나의 코얼레싱된 트랜잭션                                            │ │
│  │                                                               │ │
│  │  32/128바이트 정렬 권장                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 높은 대역폭: TB/s급                                                 │ │
│  │  • 병렬 접근: 워프 단위                                                │ │
│  │  • 유연성: 다양한 유형                                                 │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 용량 제한: 16-80GB                                                  │ │
│  │  • PCIe 병목: 호스트 전송                                              │ │
│  │  • 코얼레싱: 주의 필요                                                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• GPU 메모리: 계층적 구조
• 레지스터: 스레드 전용, 가장 빠름
• 공유 메모리: 블록 공유, 높은 대역폭
• L1/L2: 캐시 계층
• 글로벌: HBM/GDDR, 최대 용량
• HBM: 3D 스택, ~2TB/s
• GDDR: 2D, ~1TB/s
• 코얼레싱: 워프 접근 최적화
• 뱅크 충돌: 공유 메모리 회피
• 최적화: 타일링, 프리페치
```
