# 430. 추론 최적화 (Inference Optimization)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모델 추론 시 지연, 처리량, 메모리 최적화

## Ⅰ. 개요 (Context & Background)

### 개념 정의

추론 최적화(Inference Optimization)는 **학습된 모델의 추론 단계에서 지연 시간, 처리량, 메모리 사용량을 개선하여 실시간 서비스와 비용 효율을 달성하는 기법**이다.

### 💡 비유: 식당 주문 처리 최적화

추론 최적화는 **식당 주문 처리 최적화**와 같다. 주문이 들어온다. (추론 요청) 어떻게 처리할까. 한 번에 하나? (배치 1) 느리다. 여러 개를 함께? (배치) 빠르다. 자주 쓰는 재료는 미리? (캐싱) 빠르다. 추론 최적화가 똑같다. 배치, 캐싱, 병렬, 효율.

## Ⅱ. 추론 최적화 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                추론 최적화 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동적 배칭 (Dynamic Batching)】                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  개념: 여러 요청을 모아서 한 번에 처리                                      │ │
│  │                                                               │ │
│  │  시간 →                                                                    │ │
│  │  ┌────┬────┬────┬────┬────┬────┐                                         │ │
│  │  │ R1 │ R2 │ R3 │ R4 │ R5 │ R6 │  개별 요청                              │ │
│  │  └────┴────┴────┴────┴────┴────┘                                         │ │
│  │        ↓         ↓         ↓                                              │ │
│  │  ┌─────────┬─────────┬─────────┐                                         │ │
│  │  │B1: R1-3 │B2: R4-5 │B3: R6   │  동적 배치                              │ │
│  │  └─────────┴─────────┴─────────┘                                         │ │
│  │                                                               │ │
│  │  장점: 처리량 향상, GPU 활용률 증가                                        │ │
│  │  단점: 지연 증가 (대기 시간)                                               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【KV 캐시 (LLM 추론)】                                                           │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  문제: 자기회귀 생성에서 이전 토큰 재계산                                   │ │
│  │                                                               │ │
│  │  해결: Key, Value 캐싱                                                    │ │
│  │                                                               │ │
│  │  토큰 1: K1, V1 계산 → 캐시                                               │ │
│  │  토큰 2: K2, V2 계산 + K1, V1 로드                                        │ │
│  │  토큰 3: K3, V3 계산 + K1-2, V1-2 로드                                    │ │
│  │                                                               │ │
│  │  메모리: O(n × d × layers)                                                │ │
│  │  효과: 재계산 방지, 속도 향상                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【커널 융합 (Kernel Fusion)】                                                     │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  개념: 여러 커널을 하나로 융합                                              │ │
│  │                                                               │ │
│  │  기존: Conv → ReLU → BN (3개 커널 호출)                                    │ │
│  │  융합: Conv+ReLU+BN (1개 커널)                                             │ │
│  │                                                               │ │
│  │  장점:                                                                    │ │
│  │  • 커널 호출 오버헤드 감소                                                 │ │
│  │  • 메모리 대역폭 절약                                                     │ │
│  │  • 캐시 활용 향상                                                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 지연 감소                                                            │ │
│  │  • 처리량 향상                                                          │ │
│  │  • 비용 절감                                                            │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 복잡성                                                               │ │
│  │  • 하드웨어 의존                                                        │ │
│  │  • 정확도 손실 가능                                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용

### 추론 최적화 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                추론 최적화 분석                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【TensorRT 최적화】                                                              │)
│  ──────────────────                                                  │
│  import tensorrt as trt                                             │
│                                                                     │
│  # INT8 양자화                                                        │
│  config.set_flag(trt.BuilderFlag.INT8)                              │
│  config.set_flag(trt.BuilderFlag.FP16)                              │
│                                                                     │
│  # 커널 융합 자동                                                      │
│  engine = builder.build_engine(network, config)                     │
│                                                                     │
│  【vLLM (LLM 추론)】                                                              │)
│  ──────────────────                                                  │
│  from vllm import LLM, SamplingParams                               │
│                                                                     │
│  llm = LLM(model="meta-llama/Llama-2-7b-hf")                        │
│  outputs = llm.generate(prompts, sampling_params)                   │
│                                                                     │
│  # PagedAttention, 연속 배칭 자동                                     │
│                                                                     │
│  【ONNX Runtime】                                                                │)
│  ──────────────────                                                  │
│  import onnxruntime as ort                                          │
│                                                                     │
│  sess = ort.InferenceSession("model.onnx",                          │
│      providers=['CUDAExecutionProvider'])                           │
│  outputs = sess.run(None, inputs)                                   │
│                                                                     │
│  【추론 최적화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. 프로파일링: 병목 식별                                             │
│  2. 배치: 처리량 향상                                                 │
│  3. 양자화: 메모리/속도                                                │
│  4. 캐싱: 재계산 방지                                                 │
│  5. 융합: 오버헤드 감소                                               │
│  6. 하드웨어: GPU/NPU 활용                                            │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 추론 최적화: 지연, 처리량, 메모리 개선

• 양자화: INT8/FP16
• 프레임워크: TensorRT, vLLM, Triton
• 장점: 속도, 비용, UX
• 단점: 복잡성, 정확도
• 활용: 실시간 서비스, LLM 서빙
```
