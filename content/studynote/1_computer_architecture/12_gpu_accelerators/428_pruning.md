# 428. 가지치기 (Pruning)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 중요도가 낮은 가중치/뉴런을 제거하여 모델 축소
> 2. **가치**: 모델 축소, 연산 감소, 과적합 방지
> 3. **융합**: 희소성, 구조적/비구조적, 정규화, LLM

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

가지치기(Pruning)는 **신경망에서 중요도가 낮은 가중치, 뉴런, 필터 등을 제거하여 모델 크기를 줄이고 연산량을 감소시키는 모델 압축 기법**이다.

### 💡 비유: 나무 가지치기

가지치기는 **나무 가지치기**와 같다. 나무가 자란다. 가지가 많다. (완전 모델) 쓸모없는 가지가 있다. 잘라낸다. (가지치기) 더 튼튼하다. 영양 분배. 신경망도 똑같다. 불필요 가중치 제거, 효율화, 성능 유지.

## Ⅳ. 실무 적용

### 가지치기 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                가지치기 분석                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【PyTorch 가지치기】                                                            │)
│  ──────────────────                                                  │
│  import torch.nn.utils.prune as prune                               │
│                                                                     │
│  # 크기 기반 비구조적 가지치기                                         │
│  prune.l1_unstructured(module, name='weight', amount=0.3)           │
│                                                                     │
│  # 구조적 가지치기                                                    │
│  prune.ln_structured(module, name='weight', amount=0.3, n=2)        │
│                                                                     │
│  【TorchPruning 라이브러리】                                                      │)
│  ──────────────────                                                  │
│  from TorchPruning import DependencyGraph, metapruner               │
│                                                                     │
│  DG = DependencyGraph()                                             │
│  DG.build_dependency(model, example_inputs=torch.randn(1,3,224,224))│
│                                                                     │
│  pruner = metapruner.MetaPruner(                                    │
│      model, example_inputs, pruning_plan=plan                       │
│  )                                                                  │
│  pruner.step()                                                      │
│                                                                     │
│  【NVIDIA 2:4 희소성】                                                           │)
│  ──────────────────                                                  │
│  import torch.sparse                                                │
│                                                                     │
│  # 2:4 희소 패턴 적용                                                 │
│  sparse_weight = torch.sparse.prune_2_4(weight)                     │
│                                                                     │
│  【가지치기 최적화 팁】                                                          │)
│  ──────────────────                                                  │
│  1. 점진적: 천천히 제거                                               │
│  2. 미세 조정: 반드시 필요                                            │
│  3. 구조적: 하드웨어 활용                                             │
│  4. N:M: 텐서 코어 활용                                               │
│  5. 중요도: 종합 평가                                                 │
│  6. 반복: 여러 번 수행                                                │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 가지치기: 불필요 가중치 제거
• 비구조적: 개별 가중치, 높은 희소성
• 구조적: 채널/필터, 하드웨어 친화적
• N:M 희소성: 2:4, 텐서 코어 활용
• 기준: 크기, 그래디언트, 정규화
• SparseGPT: LLM一次性 가지치기
• 장점: 크기, 연산, 과적합 방지
• 단점: 정확도, 미세 조정
• 흐름: 학습 → 평가 → 제거 → 미세 조정
• 활용: 모델 압축, 추론 가속
```
