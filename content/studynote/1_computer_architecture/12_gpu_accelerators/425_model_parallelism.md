# 425. 모델 병렬 (Model Parallelism)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모델을 여러 GPU에 분할하여 단일 GPU 메모리 한계 극복
> 2. **가치**: 대규모 모델 학습, 메모리 확장, 통신 오버헤드
> 3. **융합**: 텐서 병렬, 파이프라인 병렬, 데이터 병렬, ZeRO

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

모델 병렬(Model Parallelism)은 **단일 GPU 메모리에 적재할 수 없는 대규모 모델을 여러 GPU에 분산하여 학습/추론하는 기법으로, 텐서 병렬, 파이프라인 병렬, 데이터 병렬을 조합한다.**

### 💡 비유: 거대 퍼즐 조립

모델 병렬은 **거대 퍼즐 조립**과 같다. 혼자서는 안 된다. 너무 크다. 여러 사람이 필요하다. 각자 일부를 맡는다. 서로 연결한다. 완성한다. 모델 병렬이 똑같다. 모델 분할, GPU 할당, 통신, 완성.

## Ⅱ. 모델 병렬 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                모델 병렬 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【텐서 병렬 (Tensor Parallelism)】                                               │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  행렬 곱셈 분해:                                                          │ │
│  │  Y = XW = X[W₁ | W₂] = [XW₁ | XW₂]                                     │ │
│  │                                                               │ │
│  │  GPU 0: W₁ 연산 → 결과 1                                                │ │
│  │  GPU 1: W₂ 연산 → 결과 2                                                │ │
│  │  All-Reduce: 결과 1 + 결과 2                                            │ │
│  │                                                               │ │
│  │  Transformer MLP:                                                         │ │
│  │  • 첫 번째 선형층: 열 분할                                                │ │
│  │  • 두 번째 선형층: 행 분할                                                │ │
│  │  • All-Reduce로 동기화                                                   │ │
│  │                                                               │ │
│  │  통신: All-Reduce, 높은 빈도                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파이프라인 병렬 (Pipeline Parallelism)】                                         │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  레이어 분할:                                                              │ │
│  │  GPU 0: Layer 0-5                                                        │ │
│  │  GPU 1: Layer 6-11                                                       │ │
│  │  GPU 2: Layer 12-17                                                      │ │
│  │  GPU 3: Layer 18-23                                                      │ │
│  │                                                               │ │
│  │  마이크로배치:                                                              │ │
│  │  • 배치를 M개로 분할                                                     │ │
│  │  • 파이프라인으로 처리                                                   │ │
│  │  • 버블 최소화                                                           │ │
│  │                                                               │ │
│  │  통신: Point-to-Point, 낮은 빈도                                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ZeRO (Zero Redundancy Optimizer)】                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Stage 1: 옵티마이저 상태 분할                                             │ │
│  │  Stage 2: + 그래디언트 분할                                              │ │
│  │  Stage 3: + 파라미터 분할                                                │ │
│  │  Stage 3 + Offload: CPU/NVMe 오프로드                                   │ │
│  │                                                               │ │
│  │  메모리 절약:                                                              │ │
│  │  • 1/N 옵티마이저, 그래디언트, 파라미터                                   │ │
│  │  • N개 GPU 사용 시 N배 절약                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 대규모 모델 학습 가능                                                 │ │
│  │  • 메모리 확장                                                          │ │
│  │  • GPU 활용                                                             │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 통신 오버헤드                                                        │ │
│  │  • 복잡성                                                               │ │
│  │  • 버블 (파이프라인)                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 모델 병렬: 대규모 모델 분산 학습
• 데이터 병렬: 모델 복제, 데이터 분할
• 텐서 병렬: 레이어 내 분할, All-Reduce
• 파이프라인 병렬: 레이어 간 분할, 마이크로배치
• ZeRO: 상태 분할, 메모리 절약
• 3D 병렬: TP + PP + DP
• 통신: NCCL, NVLink, InfiniBand
• Megatron-LM: 텐서 병렬
• DeepSpeed: ZeRO, 오프로드
• 활용: GPT, LLaMA, 초거대 모델
```
