# 418. CUDA (Compute Unified Device Architecture)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: NVIDIA GPU를 위한 병렬 컴퓨팅 플랫폼과 프로그래밍 모델
> 2. **가치**: GPU 병렬 처리를 C/C++로 프로그래밍
> 3. **융합**: 커널, 스레드 계층, 메모리 관리, 스트림

## Ⅰ. 개요 (Context & Background)

### 개념 정의

CUDA(Compute Unified Device Architecture)는 **NVIDIA가 개발한 GPU 병렬 컴퓨팅을 위한 플랫폼과 프로그래밍 모델로, C/C++ 확장을 통해 GPU를 프로그래밍할 수 있게 한다.**

### 💡 비유: 군대 지휘 체계

CUDA는 **군대 지휘 체계**와 같다. 사단이 있다. 연대가 있다. 대대가 있다. 소대가 있다. 계층적 구조. 명령 체계. 그리드는 사단. 블록은 연대. 스레드는 소대. CUDA가 똑같다. Grid-Block-Thread 계층, 명령 실행, 조직적 병렬.

### CUDA 개요

```
┌─────────────────────────────────────────────────────────────────────┐
│                CUDA 개요                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스레드 계층 구조】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────────────────────────────────────────────────┐      │ │
│  │  │                    Grid                              │      │ │
│  │  │  (커널 실행에 의해 생성된 스레드 블록의 집합)           │      │ │
│  │  │                                                     │      │ │
│  │  │  ┌─────────┐  ┌─────────┐  ┌─────────┐             │      │ │
│  │  │  │ Block   │  │ Block   │  │ Block   │             │      │ │
│  │  │  │ (0,0)   │  │ (1,0)   │  │ (2,0)   │             │      │ │
│  │  │  │         │  │         │  │         │             │      │ │
│  │  │  │[T0 T1] │  │[T0 T1] │  │[T0 T1] │             │      │ │
│  │  │  │[T2 T3] │  │[T2 T3] │  │[T2 T3] │             │      │ │
│  │  │  └─────────┘  └─────────┘  └─────────┘             │      │ │
│  │  │                                                     │      │ │
│  │  │  ┌─────────┐  ┌─────────┐  ┌─────────┐             │      │ │
│  │  │  │ Block   │  │ Block   │  │ Block   │             │      │ │
│  │  │  │ (0,1)   │  │ (1,1)   │  │ (2,1)   │             │      │ │
│  │  │  │         │  │         │  │         │             │      │ │
│  │  │  └─────────┘  └─────────┘  └─────────┘             │      │ │
│  │  └─────────────────────────────────────────────────────┘      │ │
│  │                                                               │ │
│  │  Thread: 개별 실행 단위                                               │ │
│  │  Block: 스레드 묶음, 공유 메모리 공유                                   │ │
│  │  Grid: 블록 묶음, 하나의 커널                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CUDA 커널】                                                                   │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  __global__ void vector_add(float *a, float *b, float *c, int n) {         │ │
│  │      int i = blockIdx.x * blockDim.x + threadIdx.x;                        │ │
│  │      if (i < n) {                                                          │ │
│  │          c[i] = a[i] + b[i];                                               │ │
│  │      }                                                                     │ │
│  │  }                                                                         │ │
│  │                                                               │ │
│  │  // 호스트에서 호출                                                    │ │
│  │  int blocks = (n + 255) / 256;                                             │ │
│  │  vector_add<<<blocks, 256>>>(d_a, d_b, d_c, n);                            │ │
│  │           ↑         ↑                                                      │ │
│  │        블록 수   블록당 스레드                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. CUDA 상세

### 상세 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                CUDA 상세                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메모리 관리】                                                                 │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  // 호스트-디바이스 메모리 복사                                           │ │
│  │  cudaMalloc(&d_ptr, size);      // GPU 메모리 할당                          │ │
│  │  cudaMemcpy(d_ptr, h_ptr, size, cudaMemcpyHostToDevice);                   │ │
│  │  // ... 커널 실행 ...                                                      │ │
│  │  cudaMemcpy(h_ptr, d_ptr, size, cudaMemcpyDeviceToHost);                   │ │
│  │  cudaFree(d_ptr);               // GPU 메모리 해제                          │ │
│  │                                                               │ │
│  │  // 통합 메모리 (Pascal 이상)                                            │ │
│  │  cudaMallocManaged(&ptr, size);  // 자동 마이그레이션                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【CUDA 메모리 유형】                                                              │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • __global__: 그리드 전역 (모든 스레드)                                  │ │
│  │  • __device__: 디바이스 메모리                                          │ │
│  │  • __shared__: 블록 내 공유                                              │ │
│  │  • __constant__: 상수 메모리                                             │ │
│  │  • __local__: 스레드 로컬                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동기화】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  • __syncthreads(): 블록 내 모든 스레드 동기화                            │ │
│  │  • cudaDeviceSynchronize(): 호스트-디바이스 동기화                        │ │
│  │  • 스트림: 비동기 실행                                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. CUDA × 타 과목 융합

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                CUDA × 타 과목 융합                                      │

│                                                                     │
```

## Ⅳ. 실무 적용

### CUDA 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                CUDA 분석                                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【컴파일】                                                                     │)
│  ──────────────────                                                  │
│  $ nvcc -o program program.cu                                       │
│  # CUDA 프로그램 컴파일                                               │
│                                                                     │
│  $ nvcc -arch=sm_80 -o program program.cu                           │
│  # 특정 아키텍처 지정                                                  │
│                                                                     │
│  【프로파일링】                                                              │)
│  ──────────────────                                                  │
│  $ nvprof ./program                                                 │
│  # 커널 시간, 메모리 전송                                              │
│                                                                     │
│  $ ncu ./program                                                    │
│  # Nsight Compute (상세 분석)                                         │
│                                                                     │
│  【디버깅】                                                                   │)
│  ──────────────────                                                  │
│  $ cuda-gdb ./program                                               │
│  # CUDA 디버거                                                        │
│                                                                     │
│  $ compute-sanitizer ./program                                      │
│  # 메모리 오류 탐지                                                    │
│                                                                     │
│  【CUDA 최적화 팁】                                                              │)
│  ──────────────────                                                  │
│  1. 메모리 공동alescing: 연속 접근                                    │
│  2. 공유 메모리: 재사용 데이터                                         │
│  3. Occupancy: SM 활용률                                             │
│  4. 스트림: 비동기 실행                                               │
│  5. 분기: 워프 내 분기 최소화                                          │
│  6. 측정: 프로파일링 후 최적화                                         │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• CUDA: NVIDIA GPU 병렬 컴퓨팅 플랫폼

• nvprof, ncu: 프로파일링
• 활용: 딥러닝, 과학 컴퓨팅
• 대안: OpenCL, HIP
```
