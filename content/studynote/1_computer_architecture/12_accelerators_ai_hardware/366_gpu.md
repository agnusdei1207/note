# GPU (Graphics Processing Unit)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: GPU(그래픽 처리 장치)는 화면을 구성하는 수백만 개의 픽셀(점)을 동시에 칠하기 위해, 복잡한 제어 로직(If-Else)을 다 버리고 **단순한 덧셈/곱셈 연산기(ALU) 수천 개를 한 칩에 무식하게 때려 박은 극단적인 대규모 병렬 연산 가속기**다.
> 2. **가치**: 한 명의 똑똑한 교수님(CPU)이 못 푸는 거대한 노가다(행렬 연산)를 1만 명의 초등학생 군단(GPU 코어)이 1초 만에 찢어버림으로써, **3D 게임 산업을 넘어 현대 인공지능(AI)과 딥러닝 혁명을 물리적으로 가능케 한 유일한 구원자**가 되었다.
> 3. **융합**: 초기에는 모니터 출력용 특수 깡통 칩이었으나, 엔비디아(NVIDIA)가 CUDA(쿠다)라는 프로그래밍 플랫폼을 융합시켜 누구나 GPU로 일반 수학 계산(GPGPU)을 짤 수 있게 개방하며 세계 시가총액 1위의 반도체 제국을 건설했다.

---

## Ⅰ. GPU의 개요

### 1. 정의
- 플린의 분류법 중 SIMD(Single Instruction Multiple Data) 또는 자체 규격인 SIMT 구조를 기반으로, 수천 개의 스트리밍 프로세서(코어)를 이용해 대량의 데이터를 동시에 병렬로 연산하는 도메인 특화 반도체 칩이다.

### 2. 등장 배경: '모니터 픽셀은 노가다의 극치다'
- 1990년대, 3D 게임 둠(Doom)이나 퀘이크를 돌리기 위해 CPU는 피눈물을 흘렸다. 1920x1080 모니터의 픽셀은 무려 207만 개다. 이 점들의 색깔과 3D 그림자(벡터 행렬 곱셈)를 초당 60번씩 CPU가 혼자서 순서대로 계산하려니 컴퓨터가 터져나갔다. 엔지니어들은 "어차피 픽셀 A랑 픽셀 B의 색을 칠하는 건 서로 아무 상관 없잖아? 그냥 멍청한 계산기 200만 개를 달아서 한 번에 다 칠해버리자!"라는 발상의 전환을 하였고, 1999년 엔비디아의 '지포스 256'이 최초로 GPU라는 타이틀을 달고 세상을 바꿨다.

### 3. 💡 비유: '초등학생 1만 명의 덧셈 군단'
- CPU는 비행기를 설계하고 철학을 논할 수 있는 **'노벨상 받은 천재 수학자 4명(4코어)'**이다. 똑똑하지만 100만 장의 영수증 덧셈을 시키면 며칠이 걸린다.
- GPU는 덧셈, 뺄셈밖에 모르는 **'평범한 초등학생 10,000명(1만 코어)'**이다. 비행기 설계는 죽어도 못하지만, 각자에게 영수증을 1장씩 나눠주고 "다 같이 더해!"라고 소리치면(SIMD) 단 1초 만에 100만 장의 정답이 우수수 쏟아져 나온다.

- **📢 섹션 요약 비유**: 복잡한 미로를 풀 때는 천재 한 명이 빠르지만, 거대한 운동장의 잡초를 뽑을 때는 생각 없는 알바생 수천 명을 쫙 깔아버리는 게 압도적으로 유리한 자본주의적 물량 공세의 정점입니다.

## Ⅱ. 메커니즘 및 칩 아키텍처 (Deep Dive)

GPU는 CPU가 트랜지스터를 쓰는 방식(철학)을 완벽히 반대로 뒤집었다.

### 1. CPU vs GPU의 트랜지스터 면적 할당 (다이아그램)
```text
  [ CPU (천재 4명) ]                    [ GPU (알바생 수천 명) ]
  ┌──────────┬──────────┐               ┌─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┐
  │ L3 Cache │ L3 Cache │               │A│A│A│A│A│A│A│A│A│A│A│ 
  ├────┬─────┼────┬─────┤               ├─┼─┼─┼─┼─┼─┼─┼─┼─┼─┼─┤
  │ CU │ ALU │ CU │ ALU │               │A│A│A│A│A│A│A│A│A│A│A│
  ├────┴─────┼────┴─────┤               ├─┼─┼─┼─┼─┼─┼─┼─┼─┼─┼─┤
  │ L1/L2    │ L1/L2    │               │A│A│A│A│A│A│A│A│A│A│A│
  └──────────┴──────────┘               └─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┘
  * CPU: 복잡한 조건문(if-else)을 풀고 캐시 미스를 막으려고 
         디코더(CU)와 캐시(Cache)에 면적 80%를 바침. ALU는 20%.
  * GPU: 조건문 개나 주고, 컨트롤러와 캐시를 싹 다 치워버린 뒤, 
         면적의 80%를 오직 무식한 덧셈기(ALU)로만 꽉 채움.
```

### 2. 하드웨어 스레드 스와핑의 미친 은닉술
- GPU는 메모리에서 데이터를 가져오는 속도(VRAM 지연)가 극단적으로 느리다. CPU라면 파이프라인이 멈추고 울었겠지만 GPU는 코웃음 친다.
- GPU 코어 안에는 스레드가 수십 개씩 꽉 차 있다. 1번 스레드가 메모리를 읽으러 가서 400클럭을 멍때리면, 하드웨어가 0클럭 만에 뒤에 서 있던 2번 스레드로 바통을 넘긴다. 2번이 막히면 3번, 4번... 무려 32개의 스레드 뭉치(Warp)가 쉴 새 없이 돌면서 **메모리가 느리다는 사실 자체를 완벽한 계산량으로 덮어버린다 (Latency Hiding).** 

- **📢 섹션 요약 비유**: 물탱크(메모리)에서 물을 푸는 속도가 느리다면, 바가지를 든 사람 한 명이 계속 기다리게 두지 말고 1만 명을 일렬로 세워서 한 방울씩 끊임없이 푸게 만드는 극강의 회전초밥 시스템입니다.

## Ⅲ. GPU가 딥러닝(AI) 시대를 열어젖힌 절대적 이유

왜 AI는 CPU가 아니라 GPU를 선택했을까? 우연이 만든 필연이다.

| 특징 | 3D 게임 그래픽 연산 (본업) | 인공지능 딥러닝 연산 (부업) | 결과 |
|:---|:---|:---|:---|
| **데이터의 형태** | 픽셀들의 3차원 좌표 변환 (**행렬과 벡터**) | 수억 개의 파라미터 가중치 학습 (**거대한 행렬 곱셈**) | **소름 돋게 100% 똑같음** |
| **연산의 성격** | `if-else` 분기문 없음. 무조건 더하고 곱함 | 앞뒤 관계 안 따지고 맹목적으로 더하고 곱함(MAC 연산)| **소름 돋게 100% 똑같음** |
| **정밀도 요구** | 화면 색깔 조금 틀려도 사람 눈엔 안 보임 (FP32/FP16 충분) | AI 확률이 99%나 99.1%나 대세에 지장 없음 (FP16/INT8 가능) | **CPU의 느린 FP64가 굳이 필요 없음** |

- **📢 섹션 요약 비유**: 원래는 벽돌 공장(게임)에서 벽돌을 가장 빨리 찍어내려고 산 바보 로봇(GPU)인데, 알고 보니 새로 유행하는 피자 공장(AI)의 반죽을 찍어내는 데도 모양과 동작이 10,000% 완벽하게 똑같아서 갑자기 몸값이 100배로 뛴 우연의 일치입니다.

## Ⅳ. 실무 적용 및 아키텍처적 판단 (Strategy)

### 1. 기술사적 인사이트: GPGPU와 CUDA의 지배력
- **상황**: 2006년, 과학자들은 GPU의 깡패 같은 연산력이 부러웠지만, 이걸 쓰려면 억지로 3D 게임을 만드는 언어(DirectX, OpenGL)를 배워서 화면의 픽셀 좌표인 척 수학 공식을 쑤셔 넣는 끔찍한 해킹을 해야 했다.
- **판단**: 엔비디아의 CEO 젠슨 황은 인류 역사상 가장 위대한 베팅을 한다. "수천억을 들여 칩을 갈아엎어라! 그래픽 전문가가 아니어도, 누구나 C언어로 GPU의 1만 개 코어를 조종할 수 있게 칩 안에 범용 로직을 깔아줘라!" 이것이 2006년에 발표된 **CUDA(Compute Unified Device Architecture)**다. 즉, GPU를 범용 계산(General-Purpose, GPGPU)에 쓰게 만든 것이다. 이 한 수로 인해 현재 전 세계의 딥러닝 파이토치, 텐서플로우 등 모든 AI 코드는 CUDA 위에서만 돌아가게 되었고, 엔비디아는 생태계를 완전히 록인(Lock-in)시켰다.

### 2. 실무 시나리오: Warp Divergence (분기의 저주)
- **인사이트**: 주니어 개발자가 CUDA로 C언어를 짜다가 루프 안에 무심코 `if (x > 0) { ... } else { ... }` 라는 분기문을 넣었다.
- **결과**: 속도가 박살 났다. GPU는 32개의 스레드가 1개의 묶음(Warp)이 되어 무조건 똑같은 명령어(SIMT)를 실행해야 한다. 32명 중 16명은 `if`를 타고 16명은 `else`를 타면? 끔찍하게도 GPU는 16명이 `if`를 실행할 동안 나머지 16명의 눈을 가려놓고(Masking) 놀게 냅둔다. 끝나면 반대로 16명을 놀게 하고 `else`를 실행한다. 성능이 정확히 반토막 난다. GPU 프로그래머는 하늘이 두 쪽 나도 스레드 그룹 내에서 조건문이 찢어지는 이 **워프 다이버전스(Warp Divergence)**를 피하는 분기 없는(Branchless) 로직을 짜내야 하는 숙명을 진다.

- **📢 섹션 요약 비유**: 32인 33각 달리기(Warp)를 하는데, 중간에 돌멩이(조건문)가 나타나서 누구는 왼쪽, 누구는 오른쪽으로 피하라고 하면 다 같이 엉켜서 자빠지는 끔찍한 병렬의 저주입니다.

## Ⅴ. 기대효과 및 결론

### 1. 정량/정성 기대효과
- **폰 노이만 병목의 부분적 파괴**: CPU가 메인 메모리와 힘겹게 통신하는 동안, GPU는 칩 바로 옆에 HBM(고대역폭 메모리)을 탑재하고 1초에 3TB(테라바이트)의 데이터를 집어삼키며 병목을 물리력으로 터뜨림.
- **인공지능의 특이점 돌파**: 수십 년간 이론으로만 존재하던 딥러닝(신경망) 알고리즘이 GPU의 압도적인 물리적 연산력을 만나 비로소 인류의 곁으로 강림(AlexNet, ChatGPT)하게 된 유일한 인프라.

### 2. 미래 전망
최신 GPU(RTX 4000, H100 등)는 더 이상 단순한 픽셀 덧셈기가 아니다. 화면을 그리는 구형 CUDA 코어 옆에, 아예 AI 행렬 곱셈만 미친 듯이 쳐내는 **텐서 코어(Tensor Core)**를 수천 개 박아 넣었고, 빛의 굴절을 계산하는 레이 트레이싱(RT) 코어까지 구겨 넣었다. 미래의 GPU는 사실상 "그래픽"이라는 단어의 껍데기를 완전히 벗어던지고, 행렬, 벡터, 인공지능, 암호화를 모두 씹어먹는 전 세계 클라우드 데이터센터의 진정한 메인 두뇌(Superchip)로 영구 집권할 것이다.

- **📢 섹션 요약 비유**: 옛날엔 연예인(CPU)의 얼굴에 화장만 이쁘게 칠해주던 무대 뒤의 분장사(GPU)였는데, 분장사가 알고 보니 1만 명의 군대를 거느린 전쟁의 신으로 밝혀지며 무대 위 주연 자리를 발로 차고 올라간 완벽한 쿠데타입니다.
