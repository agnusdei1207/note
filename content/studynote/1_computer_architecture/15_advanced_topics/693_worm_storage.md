> **핵심 인사이트**
> 1. 일반적인 하드디스크나 SSD는 관리자(Root) 권한만 얻으면 언제든 데이터를 수정하거나 지울 수 있어, 내부 직원의 횡령(로그 삭제)이나 랜섬웨어의 암호화 공격에 취약하다.
> 2. **WORM (Write Once Read Many)**은 이름 그대로 **"딱 한 번만 쓸 수 있고, 읽는 건 평생 가능하지만 수정/삭제는 절대 불가능"**한 특수 스토리지 기술이다.
> 3. 과거에는 진짜 CD-ROM처럼 물리적으로 구워버리는 하드웨어 방식을 썼으나, 현재는 오브젝트 스토리지(AWS S3 Object Lock) 수준에서 특정 기간(예: 10년) 동안 시스템 최고 관리자조차 삭제할 수 없게 만드는 소프트웨어적 WORM(Compliance Mode)이 표준으로 자리 잡았다.

<br>

## Ⅰ. 위변조의 공포와 법적 규제 (Compliance)

은행의 계좌 이체 내역이나 병원의 의료 기록, 법원의 소송 증거물 같은 데이터는 절대 위조되어서는 안 됩니다.
만약 해커가 서버 관리자(Root) 비밀번호를 털었다고 가정해 봅시다.

* **일반 스토리지**: 해커가 `rm -rf` 명령어나 디스크 포맷을 날리면 원본과 백업 데이터가 모조리 날아갑니다. 랜섬웨어도 마찬가지로 파일을 자기들만의 암호로 '덮어써버립니다'.
* **WORM 스토리지**: 해커가 포맷 명령을 날려도 디스크가 "응, 거절~" 하고 튕겨냅니다. WORM 스토리지에 한 번 들어간 데이터는 **신(God)이 와도 지울 수 없습니다.** (하드디스크를 물리적으로 망치로 부수지 않는 이상)

> 📢 **섹션 요약 비유**: 연필로 쓴 공책(일반 디스크)은 누구나 지우개로 지우고 위조할 수 있습니다. 하지만 한 번 글씨를 파놓으면 절대 지울 수 없는 **비석에 새긴 글씨(WORM)**는 1,000년이 지나도 조작되지 않은 원본임을 증명합니다.

<br>

## Ⅱ. WORM의 두 가지 구현 방식

### 1. 물리적 WORM (Hardware WORM)
가장 확실한 방식입니다. 
* CD-R, DVD-R, 블루레이(Blu-ray) 같은 광디스크 매체가 대표적입니다. 레이저로 디스크 표면의 염료를 태워서(Burn) 영구적인 구멍을 내버리므로, 물리적으로 두 번 다시 덮어쓸 수 없습니다.
* 일부 특수 하드디스크(Tape, HDD) 펌웨어 자체에 "한 번 쓴 주소는 절대 덮어쓰기 명령을 받지 말라"고 기계적으로 락(Lock)을 걸어둔 장비도 있습니다.

### 2. 논리적 WORM (Software WORM / Object Lock)
광디스크는 너무 느리고 용량이 작아서 현대 데이터센터에는 맞지 않습니다. 그래서 요즘은 일반 HDD 스토리지나 클라우드 위에 **소프트웨어 락**을 겁니다.
* **AWS S3 Object Lock (Compliance Mode)**: 특정 폴더(버킷)에 WORM 정책을 겁니다. "여기에 올라오는 파일은 10년 동안 삭제 금지!"
* 이 모드가 켜지면, **AWS 계정의 주인이 와도, AWS의 사장님인 제프 베조스가 와도 그 파일을 지우거나 덮어쓸 수 없습니다.** AWS 내부 시스템 로직이 파일의 수정 요청 자체를 원천 차단해버리기 때문입니다. 보존 기한이 끝난 뒤에야 비로소 삭제 버튼이 활성화됩니다.

> 📢 **섹션 요약 비유**: 물리적 WORM이 종이에 도장을 찍어 돌이킬 수 없게 만드는 것이라면, 논리적 WORM은 시한장치가 달린 금고에 서류를 넣고 "10년 뒤에 열림" 버튼을 누르는 것입니다. 금고 주인이 변심해도 10년 전에는 절대 문을 열고 서류를 고칠 수 없습니다.

## Ⅲ. 랜섬웨어와의 전쟁 (최후의 방어선)

현대 기업 보안의 트렌드는 방화벽으로 랜섬웨어를 100% 막는 것은 불가능하다고 가정하는 것입니다. (결국 누군가 악성 메일을 누름)

그래서 해커가 회사망 전체를 장악하고 모든 윈도우/리눅스 서버를 암호화시켜버렸을 때, 유일하게 해커의 암호화(덮어쓰기) 공격을 튕겨내고 살아남아 있는 **백업 전용 WORM 스토리지**가 회사의 명운을 가르는 동아줄이 됩니다. 회사는 해커에게 돈을 주지 않고, 포맷 후 WORM에서 깨끗한 원본 데이터를 다시 꺼내와 시스템을 복구합니다.
