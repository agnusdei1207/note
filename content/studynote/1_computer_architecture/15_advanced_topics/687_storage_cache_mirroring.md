> **핵심 인사이트**
> 1. 엔터프라이즈 스토리지는 속도를 높이기 위해, 서버에서 받은 데이터를 굼벵이 같은 물리 디스크에 즉시 쓰지 않고 **스토리지 내부의 초고속 RAM(Write Cache)에 먼저 던져놓고 "저장 완료!"라고 사기(Write-back)**를 친다.
> 2. 하지만 이 상태에서 스토리지가 벼락을 맞아 컨트롤러 보드가 타버리면, RAM에 있던 데이터(디스크에 미처 못 쓴 결제 내역)는 허공으로 영원히 날아가 버리는 대재앙이 터진다.
> 3. 이를 막기 위해 스토리지 박스 안에는 항상 뇌(컨트롤러)가 2개 존재하며, **1번 뇌의 RAM(캐시)에 데이터가 들어오자마자 2번 뇌의 RAM으로 빛의 속도로 복사(Cache Mirroring)**를 해둔다. 1번 뇌가 죽어도 2번 뇌가 기억을 그대로 이어받아 디스크에 무사히 기록한다.

<br>

## Ⅰ. Write-Back 캐시의 무서운 도박

데이터베이스 서버가 스토리지에 "홍길동 잔고 100만 원으로 수정해"라고 씁니다.

* **Write-Through (정직함)**: 스토리지가 물리 디스크(HDD/SSD) 표면에 자기를 띠며 영구적으로 데이터를 꽉 박아 넣을 때까지 기다린 뒤에야 "완료" 응답을 줍니다. 안전하지만 지독하게 느립니다.
* **Write-Back (사기침, 성능 지상주의)**: 스토리지가 자기 보드에 달린 초고속 RAM(캐시)에 데이터를 툭 던져두고, 디스크에는 아직 안 썼으면서도 서버에는 1나노초 만에 **"다 썼음!"**이라고 응답을 쏴버립니다. 나중에 쉴 때 디스크에 내려씁니다(Flush). 엄청 빠릅니다.

하지만 Write-Back 상태에서 찰나의 순간 스토리지 보드가 퍽 하고 고장 나면? 서버는 이미 "저장 완료"를 믿고 고객 화면에 결제 완료 창을 띄웠는데, 진짜 디스크에는 돈이 안 들어있는 끔찍한 데이터 유실(Data Loss)이 발생합니다.

> 📢 **섹션 요약 비유**: 사장님(서버)이 비서(스토리지 캐시)에게 "이거 장부(디스크)에 적어"라고 지시합니다. 비서는 포스트잇에 대충 갈겨쓰고 "넵 장부에 다 적었습니다!"라고 뻥을 칩니다(Write-back). 사장님이 나간 사이 비서가 심장마비(고장)로 쓰러져 포스트잇이 버려지면, 그 장부는 영원히 누락됩니다.

## Ⅱ. 캐시 미러링 (기억의 복제)

억 단위의 가격을 자랑하는 엔터프라이즈 스토리지(EMC, NetApp)는 절대로 이 도박에서 지지 않기 위해 **듀얼 컨트롤러 (Active-Active)** 구조를 가집니다.

스토리지 박스 뒷면을 까보면 뇌(컨트롤러 보드)가 위아래로 2개 꽂혀있습니다.

1. 서버가 컨트롤러 A에게 데이터를 보냅니다.
2. 컨트롤러 A는 자기 RAM(캐시)에 데이터를 올립니다. **하지만 아직 서버에게 "완료"라고 대답하지 않습니다!**
3. 컨트롤러 A는 뒷면에 있는 특수 고속 버스(PCIe NTB, CXL 등)를 타고 **컨트롤러 B의 RAM(캐시)으로 방금 들어온 데이터를 몰래 복사(Mirroring)**해 둡니다.
4. 컨트롤러 B가 "내 머릿속에도 똑같이 적어뒀다"라고 확인(ACK)을 해주는 순간, 그제야 컨트롤러 A는 서버에게 "저장 완료!"라고 외칩니다.

### 컨트롤러 이중화 아키텍처 (ASCII)

```text
  [ 호스트 서버 ] ──(데이터 전송)──┐
                                 ▼ (1. 캐시 적재)
 ╔════════ 스토리지 박스 ══════════════════════════════════════════╗
 ║ ┌── 컨트롤러 A (죽어도 됨) ───┐   (3. 찰나의 복사)   ┌── 컨트롤러 B (백업 뇌) ───┐ ║
 ║ │ Write Cache (RAM)      │ ◀━━━━(PCIe 버스)━━━━▶ │ Write Cache (RAM)      │ ║
 ║ └──────────┬─────────────┘                       └──────────┬─────────────┘ ║
 ║            │ (4. 나중에 한가할 때 디스크에 씀)                 │                      ║
 ║            ▼                                                ▼                      ║
 ║   [ 물리적 디스크(SSD/HDD) 무더기 ] ◀─────────────────────────────────────────┘ ║
 ╚═════════════════════════════════════════════════════════════════════════════╝
```

> 📢 **섹션 요약 비유**: 비서 A가 포스트잇에 메모를 적자마자, 옆에 앉은 비서 B에게 "야, 너도 네 포스트잇에 똑같이 100만 원 적어놔"라고 시킵니다(캐시 미러링). 비서 A가 심장마비로 쓰러져도, 비서 B가 자기 포스트잇을 보고 무사히 장부(디스크)에 글씨를 옮겨 적는 완벽한 이중 방어 시스템입니다.

## Ⅲ. BBU(배터리)와 영구 메모리의 결합

컨트롤러 1개가 죽는 건 미러링으로 막았습니다. 그런데 **데이터센터 건물 자체에 번개가 떨어져서 컨트롤러 A와 B의 전원이 0.1초 만에 동시에 다 나가버리면 어떻게 될까요?** 둘 다 RAM이므로 데이터가 증발합니다.

이를 막기 위해 캐시 메모리 옆에는 무조건 **BBU (Battery Backup Unit, 배터리)**가 찰딱 붙어 있습니다. (다음 챕터에서 계속)
