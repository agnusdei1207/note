## 핵심 인사이트 (3줄 요약)
> 1. **레지스터 파일(Register File)**은 CPU 내부에서 가장 빠른 저장 공간(SRAM 배열)으로, ALU가 계산을 하기 위해 데이터를 꺼내고(Read) 넣는(Write) 핵심 장소다.
> 2. 한 클럭에 여러 명령어를 처리하는 수퍼스칼라 구조에서는 레지스터에 뚫려 있는 **포트(Port, 입출력 통로)의 개수**가 곧 CPU의 동시 실행 능력(IPC)을 결정짓는 물리적 한계선이 된다.
> 3. 포트 수를 늘리면 면적과 소비 전력이 제곱(Quadratic)에 가깝게 폭증하므로, 뱅크 분할이나 다중 복제(Replication) 같은 우회 설계가 필수적이다.



## Ⅰ. 레지스터 포트의 역할과 병목

단일 명령어(스칼라) 처리 CPU를 상상해 봅시다.
명령어 `ADD R1, R2, R3` (R2와 R3를 더해서 R1에 넣어라) 하나를 실행하려면, 레지스터에서 두 개의 값을 읽고(2 Read Ports), 하나의 값을 써야 합니다(1 Write Port). 즉 최소 **2R1W 포트**가 필요합니다.

그런데 현대의 **4-way 수퍼스칼라 프로세서**(한 클럭에 4개의 명령어를 동시 발사)라면 어떨까요?
* 4개의 명령어가 각각 2개를 읽고 1개를 쓴다면, 동시에 **8개의 Read 포트와 4개의 Write 포트(8R4W)**가 필요합니다.

만약 연산 유닛(ALU)이 6개나 있는데 레지스터 포트가 4개밖에 없다면? 데이터를 꺼낼 구멍이 모자라서 ALU 2개는 놀아야 합니다. 즉, **포트의 개수는 곧 공장의 출입문 개수**이며, 동시 작업량의 절대적인 한계가 됩니다.

> 📢 **섹션 요약 비유**: 요리사(ALU) 4명이 동시에 요리를 하려는데, 냉장고(레지스터)의 문(포트)이 1개뿐이면 한 번에 한 명씩만 재료를 꺼낼 수 있습니다. 4명이 동시에 재료를 꺼내려면 냉장고에 문을 8개 달아주어야 합니다.



## Ⅱ. 멀티 포트 설계의 저주 (Area and Power Wall)

"그럼 그냥 포트를 10개, 20개 뚫어주면 되지 않나?" 싶지만, 레지스터 파일의 물리적 특성 때문에 불가능합니다.

SRAM 셀 하나에 포트(트랜지스터와 배선)를 하나 추가할 때마다 셀의 크기가 커집니다.
* 포트의 개수가 $N$배 늘어나면, 배선이 얽히는 복잡성 때문에 전체 레지스터 파일의 **면적은 $N^2$에 비례하여 기하급수적으로 폭발**합니다.
* 면적이 커지면 배선의 길이가 길어지고, 이는 곧 신호 지연(Latency)과 막대한 전력 소모(Dynamic Power)를 유발하여 CPU 클럭 속도를 올릴 수 없게 만듭니다.

### 물리적 면적 폭발 (ASCII 다이어그램)

```text
 [ 2R1W 포트 레지스터 셀 ]    [ 8R4W 포트 레지스터 셀 (면적 폭발!) ]
  ┌─────┐                      ┌───────────────────────┐
 ─┤ 1비트├─                   ─┤                       ├─
 ─┤ 저장 │                    ─┤                       ├─
  └─────┘                     ─┤       1비트 저장       ├─
  (작고 빠름)                  ─┤  (복잡한 12개 배선 엉킴)├─
                               ─┤                       ├─
                               └───────────────────────┘
                               (거대하고 느리며 열이 펄펄 남)
```

> 📢 **섹션 요약 비유**: 방 하나(데이터 1비트)에 문(포트)을 2개 뚫는 건 쉽지만, 문을 12개나 뚫으려면 벽이 모자라서 방 자체를 운동장만하게 키워야 합니다. 방이 커지면 집 전체가 커지고, 이동하는 데 시간이 너무 오래 걸립니다.



## Ⅲ. 현대 아키텍처의 우회 해결책

포트의 저주를 피하기 위해 엔지니어들은 기발한 구조를 사용합니다.

1. **레지스터 파일 뱅킹 (Banking / Clustering)**
   * 거대한 레지스터를 하나의 통짜로 만들지 않고, 홀수 뱅크와 짝수 뱅크(또는 정수부와 부동소수점부)로 분할합니다. 각각은 포트 수를 적게 가져가면서도 전체적으로는 많은 포트가 있는 것처럼 효과를 냅니다.
2. **복제 (Replication)**
   * 쓰기(Write) 포트는 늘리기 어렵지만, 읽기(Read) 포트는 늘리기 쉽습니다. 아예 똑같은 레지스터 파일을 통째로 2개 복사해 두고(예: 데이터 A를 양쪽 다 써놓음), ALU들이 두 곳에서 나누어서 읽어가도록(분산) 설계합니다. (알파 21264 등에서 사용된 기법)
3. **가상 레지스터 맵핑 (Register Renaming)**
   * 물리 레지스터(PRF)를 잘게 쪼개어 특정 연산 유닛 전용으로 할당하여 얽힘을 방지합니다.

> 📢 **섹션 요약 비유**: 문이 12개 달린 거대한 냉장고 1대를 만드는 대신, 문이 2개 달린 작은 냉장고 6대를 사서 요리사들에게 나눠주는 방식(분할과 복제)으로 문제를 해결한 것입니다.
