## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 기능 점수(FP, Function Point)는 SW 규모를 사용자 기능 관점으로 측정하는 표준 단위이며, 공공 SW 대가 산정의 법적 기준이다.
> 2. **가치**: 복잡도 가중치 조작 1점 차이가 수천만 원의 대가 차이를 만들기 때문에, FP 정산 감리는 과대/과소 산정 모두를 막는 양방향 공정성 검증이다.
> 3. **판단 포인트**: IFPUG(International Function Point Users Group) 또는 NESMA(Netherlands Software Metrics Association) 산정 매뉴얼 적용 여부와 복잡도 가중치 근거 문서를 기능 유형별로 교차 검증해야 한다.

## Ⅰ. 개요 및 필요성

### 1.1 기능 점수(FP) 개념

기능 점수는 소프트웨어가 사용자에게 제공하는 기능의 크기를 정량화한다. 국내 공공 SW 사업의 SW 대가 산정 기준(행정안전부 고시)은 IFPUG 방법론 또는 NESMA 방법론 적용을 명시한다.

**기능 유형 5가지**:
- **EI (External Input)**: 외부 입력 — 데이터를 시스템에 입력하는 기능 (예: 회원 가입 화면)
- **EO (External Output)**: 외부 출력 — 처리된 데이터를 출력하는 기능 (예: 통계 보고서)
- **EQ (External Inquiry)**: 외부 조회 — 단순 데이터 조회 기능 (예: 검색 화면)
- **ILF (Internal Logical File)**: 내부 논리 파일 — 시스템이 유지관리하는 데이터 그룹
- **EIF (External Interface File)**: 외부 인터페이스 파일 — 외부 시스템에서 참조하는 데이터

### 1.2 대가 산정 체계

FP 기반 대가 = FP 합계 × 기능 점수 단가 × 보정 계수

단가는 정보통신산업진흥원(NIPA)이 매년 고시하며, 복잡도 가중치(단순/보통/복잡)가 FP 합계를 결정한다.

📢 **섹션 요약 비유**: 기능 점수는 집 평수 계산이다. 방 개수와 크기를 세어 총 면적을 구한 뒤, 거기에 평당 시세를 곱해 가격을 정한다.

## Ⅱ. 아키텍처 및 핵심 원리

### 2.1 IFPUG 기능 점수 산정 구조

```
┌──────────────────────────────────────────────────────────────────┐
│                    기능 점수 산정 흐름                            │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────┐      ┌──────────────────────────────┐  │
│  │   기능 유형 식별     │      │  복잡도 결정 기준             │  │
│  │                     │      │                              │  │
│  │  EI / EO / EQ       │      │  데이터 요소 유형(DET) 수    │  │
│  │  ILF / EIF          │      │  참조 파일 유형(FTR) 수      │  │
│  │                     │      │  레코드 요소 유형(RET) 수    │  │
│  └──────────┬──────────┘      └──────────────┬───────────────┘  │
│             │                                │                  │
│             ▼                                ▼                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │                 복잡도별 FP 가중치 테이블                  │   │
│  │  유형 │ 단순(Simple) │ 보통(Average) │ 복잡(Complex)      │   │
│  │  EI   │      3       │       4       │        6           │   │
│  │  EO   │      4       │       5       │        7           │   │
│  │  EQ   │      3       │       4       │        6           │   │
│  │  ILF  │      7       │      10       │       15           │   │
│  │  EIF  │      5       │       7       │       10           │   │
│  └──────────────────────────────────────────────────────────┘   │
│             │                                                    │
│             ▼                                                    │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  UFP (Unadjusted FP) = Σ (기능 건수 × 복잡도 가중치)     │   │
│  └──────────────────────────────────────────────────────────┘   │
│             │                                                    │
│             ▼                                                    │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  FP = UFP × VAF (Value Adjustment Factor, 가치 조정 계수)│   │
│  │       (IFPUG) 또는 VAF 미적용 (NESMA 추정법)             │   │
│  └──────────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────────┘
```

### 2.2 복잡도 판정 기준 (EI 예시)

| 복잡도 | FTR 수 | DET 수 | FP 가중치 |
|:---:|:---:|:---:|:---:|
| 단순 (Simple) | ≤1 | ≤4, 또는 ≤2이면 1~15 | 3 |
| 보통 (Average) | 2 | 5~15, 또는 1~2에 16+ | 4 |
| 복잡 (Complex) | ≥3 | 16+, 또는 ≥3에 5+ | 6 |

📢 **섹션 요약 비유**: 복잡도 가중치 판정은 이사짐 센터의 짐 부피 측정이다. 작은 박스, 중간 박스, 큰 박스로 분류해서 운반 비용을 정하는 것과 같다.

## Ⅲ. 비교 및 연결

### 3.1 FP 산정 방법론 비교

| 방법론 | 측정 시점 | 정확도 | 활용 용도 | 국내 인정 여부 |
|:---|:---|:---:|:---|:---:|
| IFPUG CPM (상세 기능 점수) | 설계 완료 후 | 높음 | 정산·검수 | O |
| NESMA 추정 기능 점수 | 요구사항 단계 | 중간 | 제안·예산 | O |
| NESMA 지시 기능 점수 | 상세 설계 단계 | 높음 | 계약·정산 | O |
| COSMIC (ISO 19761) | 설계 완료 후 | 높음 | 실시간 SW | 참고 활용 |

### 3.2 FP 조작 패턴과 감리 대응

| 조작 패턴 | 방향 | 탐지 방법 | 감리 조치 |
|:---|:---:|:---|:---|
| 단순 기능을 복잡으로 분류 | 과대 산정 | DET·FTR 수 직접 계산 | 재산정 요구 |
| 유사 기능 중복 계산 | 과대 산정 | 기능 목록 중복 확인 | 중복 항목 제거 |
| ILF를 EIF로 분류 | 과소 산정 (의도적 낮추기) | 데이터 소유권 확인 | 재분류 요구 |
| 산정 근거 문서 미작성 | 검증 불가 | 근거 문서 요청 | 미작성 시 지적 |

📢 **섹션 요약 비유**: FP 조작 탐지는 식당 계산서 검토와 같다. 주문한 메뉴와 수량이 영수증과 일치하는지, 가격이 메뉴판과 맞는지 확인한다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4.1 FP 정산 감리 체크리스트

**산정 근거 문서 검증**
- [ ] 기능 유형(EI/EO/EQ/ILF/EIF)별 산정 근거 문서가 있는가?
- [ ] 각 기능의 DET(Data Element Type) 수와 FTR/RET 수가 명시되어 있는가?
- [ ] 복잡도 판정 기준이 IFPUG CPM 또는 NESMA 기준과 일치하는가?

**실제 기능과 산정 목록 대조**
- [ ] 산정된 기능 목록과 실제 납품 화면 수·테이블 수가 일치하는가?
- [ ] 추가·변경·삭제된 기능이 정산에 반영되었는가?
- [ ] 계약 범위 외 기능이 FP에 포함되어 있지 않은가?

### 4.2 기술사 답안 포인트

기술사 시험의 FP 정산 감리 문제에서 핵심 세 가지:

1. **매뉴얼 적용 충실성**: 산정 기준이 IFPUG/NESMA 공식 매뉴얼 기준을 따르는가?
2. **근거 추적 가능성**: 복잡도 판정 근거를 DET·FTR 수로 역추적할 수 있는가?
3. **과대/과소 균형 검증**: 발주자와 수주자 어느 방향의 조작도 탐지하는 양방향 감리

📢 **섹션 요약 비유**: FP 감리는 아파트 실거래가 검증이다. 면적(평수)이 맞는지, 가격이 공시가 기준에 맞는지, 추가 옵션이 계약서에 있는지 모두 확인한다.

## Ⅴ. 기대효과 및 결론

### 5.1 FP 정산 감리 기대효과

- **공정한 대가 실현**: 과대·과소 산정 없는 정확한 SW 대가로 건전한 SW 산업 생태계 조성
- **분쟁 예방**: 객관적 FP 근거로 발주자·수주자 간 정산 분쟁 사전 차단
- **SW 자산 목록화**: 정확한 FP 데이터가 SW 자산 관리 기반 자료로 활용
- **생산성 벤치마크**: FP당 공수(Effort/FP) 데이터 축적으로 차기 사업 예산 근거 마련

### 5.2 결론

기능 점수 정산 감리는 단순히 숫자를 검증하는 것이 아니라 **SW 규모 측정의 공정성과 투명성을 보장**하는 핵심 전문 활동이다. 감리인은 IFPUG/NESMA 산정 매뉴얼에 대한 깊은 이해를 바탕으로, 기능 유형 분류의 적정성, 복잡도 판정 근거의 충실성, 변경 기능의 정산 반영 여부를 체계적으로 검증해야 한다.

📢 **섹션 요약 비유**: FP 감리 완료는 세금 신고 검증과 같다. 소득을 정직하게 신고했는지, 공제 항목이 적법한지 국세청이 확인하는 과정이다.

### 👶 어린이를 위한 3줄 비유 설명

- 기능 점수는 레고 작품의 블록 개수를 세는 거야. 작은 블록, 중간 블록, 큰 블록으로 나눠서 가격을 정해.
- 복잡도 가중치는 큰 블록이 작은 블록보다 더 비싼 것처럼, 복잡한 기능에 더 높은 점수를 주는 규칙이야.
- 감리는 블록 개수를 정직하게 셌는지, 큰 블록을 더 많이 세거나 작게 세지 않았는지 확인하는 거야.
