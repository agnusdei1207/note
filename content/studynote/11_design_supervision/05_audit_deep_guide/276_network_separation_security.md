## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 망분리(Network Separation) 감리는 외부망(인터넷)과 내부 업무망을 완전히 분리하고, 연계가 불가피한 구간에서 망연계 솔루션(Network Bridging Solution)이 자료 전송의 보안성을 담보하는지 검증한다.
> 2. **가치**: 망분리는 인터넷을 통한 APT (Advanced Persistent Threat, 지능형 지속 위협) 공격이 내부 업무망·DB에 직접 도달하는 경로를 물리적·논리적으로 차단하는 핵심 방어선이다.
> 3. **판단 포인트**: 물리적 망분리 vs. 논리적 망분리 선택의 적정성, 망연계 구간 단방향 전송 여부, 자료 전송 로그의 완전성과 이상 탐지 연계가 핵심 점검 항목이다.

---

## Ⅰ. 개요 및 필요성

### 1.1 망분리 의무화 배경

「개인정보보호법 시행령」 제29조 및 「개인정보의 안전성 확보 조치 기준」 고시는 일정 규모 이상의 개인정보처리자(공공기관 포함)에 대해 인터넷망과 업무망의 분리를 의무화한다.

주요 공공기관 해킹 사건에서 반복적으로 확인된 공격 경로는 "인터넷 연결 PC → 내부망 수평 이동 → 핵심 서버 장악"이었다. 망분리는 이 경로의 최초 진입을 차단한다.

### 1.2 망분리 유형

| 유형 | 방법 | 보안 수준 | 비용 | 대상 |
|:---|:---|:---|:---|:---|
| 물리적 망분리 | 인터넷용 PC와 업무용 PC 완전 분리 | 최높음 | 높음 | 국가기관, 금융 |
| 논리적 망분리 | 가상화(VDI) 기반 망 분리 | 높음 | 중간 | 공공기관 일반 |
| 망연계 솔루션 | 단방향 전송 장치 | 높음 | 중간 | 연계 불가피 구간 |

📢 **섹션 요약 비유**: 망분리는 "은행 직원이 고객 대기실(인터넷망)과 금고실(내부망) 사이를 가로막는 방탄 유리"처럼 — 창구(망연계)를 통해서만 돈(데이터)을 주고받을 수 있고, 직접 통과는 불가능하다.

## Ⅱ. 아키텍처 및 핵심 원리

### 2.1 망분리 보안 아키텍처

```
  ┌──────────────────────────────────────────────────────────┐
  │             망분리 보안 아키텍처 (Network Separation)     │
  ├──────────────────────────────────────────────────────────┤
  │                                                          │
  │  인터넷 구간 (External Zone)                             │
  │  ┌─────────────────────────────────────────────────┐     │
  │  │  인터넷 PC  │  메일 서버  │  웹 서버 (DMZ)       │     │
  │  └─────────────────────────────────────────────────┘     │
  │               │                                          │
  │         ┌─────▼──────────────────────────────┐           │
  │         │       망연계 구간 (Bridge Zone)      │           │
  │         │  ┌────────────────────────────────┐ │           │
  │         │  │  단방향 전송 장치 (Diode)       │ │           │
  │         │  │  (내부 → 외부 단방향만 허용)    │ │           │
  │         │  └────────────────────────────────┘ │           │
  │         │  파일 검역 (Antivirus Scan)          │           │
  │         │  자료 전송 로그 기록                  │           │
  │         └─────┬──────────────────────────────┘           │
  │               │                                          │
  │  내부 업무망 (Internal Zone)                             │
  │  ┌─────────────────────────────────────────────────┐     │
  │  │  업무용 PC  │  내부 서버  │  핵심 DB              │     │
  │  └─────────────────────────────────────────────────┘     │
  └──────────────────────────────────────────────────────────┘
```

### 2.2 망연계 솔루션 유형

| 유형 | 방식 | 특징 | 적용 사례 |
|:---|:---|:---|:---|
| 단방향 전송 장치 (Data Diode) | 물리적 단방향 전송 | 역방향 전송 물리적 불가 | 군·국가기관 |
| 파일 전송 게이트웨이 | 파일 단위 검역 후 전송 | 콘텐츠 필터링 | 공공기관 일반 |
| API 연계 게이트웨이 | 허용된 API만 통과 | 세밀한 통제 | 전자정부 연계 |
| VDI (Virtual Desktop Infrastructure) | 업무 화면만 전송 | 데이터 미이동 | 재택근무 환경 |

### 2.3 자료 전송 보안 통제

```
  ┌──────────────────────────────────────────────────────┐
  │       망연계 자료 전송 보안 통제 항목                  │
  ├──────────────────────────────────────────────────────┤
  │  전송 전 (Pre-Transfer):                             │
  │  - 발신자 인증 (사용자 ID + MFA)                    │
  │  - 파일 악성코드 검사 (AV 스캔)                     │
  │  - 파일 유형·확장자 화이트리스트 검증                │
  │  - DLP (Data Loss Prevention) 콘텐츠 검사           │
  ├──────────────────────────────────────────────────────┤
  │  전송 중 (In-Transfer):                              │
  │  - 암호화 전송 (TLS 1.2+ 또는 자체 암호화)          │
  │  - 전송 용량·속도 제한                               │
  ├──────────────────────────────────────────────────────┤
  │  전송 후 (Post-Transfer):                            │
  │  - 전송 완료 로그 기록 (발신자·파일명·시간·크기)     │
  │  - SIEM 연계 이상 탐지                               │
  │  - 수신자 확인 및 알림                               │
  └──────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 망연계 자료 전송 통제는 "공항 세관 검사"와 같아 — 외부에서 들어오는 물건(파일)은 x-ray(AV 스캔)와 세관 신고(전송 로그)를 반드시 거쳐야 하며, 금지 품목(악성코드)은 통과 불가다.

## Ⅲ. 비교 및 연결

### 3.1 물리적 vs. 논리적 망분리 비교

| 구분 | 물리적 망분리 | 논리적 망분리 (VDI) |
|:---|:---|:---|
| 분리 방식 | PC 2대 운용 또는 네트워크 완전 분리 | 가상화 기반 화면 전송 |
| 보안 수준 | 최높음 | 높음 |
| 구현 비용 | 높음 | 중간 |
| 사용자 편의성 | 낮음 (PC 2대) | 중간 (단일 화면) |
| 적용 대상 | 국가기관, 금융, 1등급 개인정보처리자 | 공공기관 일반 |

### 3.2 망분리 vs. 망연계 필요 구간

| 시나리오 | 망분리 필요 | 망연계 필요 |
|:---|:---|:---|
| 인터넷 서비스 제공 | 외부망-내부망 분리 필수 | 외부→내부 데이터 수신 시 |
| 공공데이터 배포 | 내부망 원본 보호 | 내부→외부 공개 데이터 전송 시 |
| 외부 연계 API | DMZ 구간 처리 | 허용 API만 통과 게이트웨이 |
| 재택근무 | VPN or VDI | 화면만 전송 (VDI 권고) |

📢 **섹션 요약 비유**: 물리적 망분리는 "두 개의 전화기"처럼 완전히 분리된 채널이고, 논리적 망분리는 "화상통화"처럼 화면만 연결되어 실제 파일은 이동하지 않는다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4.1 망분리 보안 감리 체크리스트

```
  ┌──────────────────────────────────────────────────────┐
  │         망분리 보안 감리 체크리스트                    │
  ├──────────────────────────────┬───────────────────────┤
  │ 점검 항목                    │ 기준                  │
  ├──────────────────────────────┼───────────────────────┤
  │ 법적 망분리 의무 해당 여부   │ 법령 기준 확인        │
  │ 망분리 유형 적정성           │ 의무 수준 충족         │
  │ 인터넷망-내부망 라우팅 차단  │ 직접 통신 0건 확인    │
  │ 망연계 솔루션 도입           │ 연계 필요 구간 전체   │
  │ 파일 전송 전 악성코드 검사   │ 100% 적용             │
  │ DLP 콘텐츠 검사              │ 개인정보·기밀 탐지    │
  │ 자료 전송 로그 보존          │ 1년 이상              │
  │ 이상 전송 SIEM 탐지          │ 실시간 경보           │
  └──────────────────────────────┴───────────────────────┘
```

### 4.2 기술사 실무 판단 포인트

- **망분리 의무 해당 여부 선확인**: 개인정보처리자 규모·유형에 따라 의무 여부가 다름 — 법령 기준 정확히 파악
- **우회 경로 탐지**: 관리용 포트, 모뎀 연결, USB 브리지 등 우회 연결 가능성 전수 조사
- **VDI 보안 강화**: 클립보드 복사·화면 캡처·파일 다운로드 차단 여부 확인 (논리적 망분리의 핵심 통제)
- **DLP 효과성 검증**: 실제 테스트 파일(개인정보 패턴)을 이용한 DLP 탐지 실증 확인

### 4.3 망연계 자료 전송 이상 패턴

| 이상 패턴 | 설명 | 대응 방법 |
|:---|:---|:---|
| 대용량 파일 전송 급증 | 대량 데이터 유출 의심 | 용량 임계값 초과 시 차단·경보 |
| 심야·휴일 전송 | 비정상 운영 시간대 활동 | 시간대별 전송 제한 정책 |
| 암호화 파일 전송 | 콘텐츠 검사 우회 시도 | 암호화 파일 전송 금지 또는 복호화 후 검사 |
| 연속 전송 실패 | 공격 탐지 회피 시도 | 실패 임계값 초과 시 계정 차단 |

📢 **섹션 요약 비유**: 기술사는 망분리 감리에서 "국경 검문소장"처럼 — 정식 통로(망연계 솔루션)를 통해서만 자료가 이동하는지, 밀수 경로(우회 연결)는 없는지 철저히 확인한다.

## Ⅴ. 기대효과 및 결론

### 5.1 기대효과

| 효과 | 내용 |
|:---|:---|
| APT 침입 차단 | 인터넷→내부망 직접 경로 물리적 차단 |
| 데이터 유출 방지 | DLP + 단방향 전송으로 기밀 유출 최소화 |
| 법적 의무 충족 | 개인정보보호법 기술적 안전 조치 이행 |
| 보안 사고 책임 한정 | 감사 로그로 사고 원인·경로 명확화 |

### 5.2 결론

망분리 보안 감리는 외부망과 내부망의 완전한 분리, 망연계 구간의 통제 완결성, 자료 전송 로그의 이상 탐지 연계를 3축으로 점검한다. 기술사는 물리적·논리적 망분리의 법적 적합성, 우회 경로 존재 여부, 망연계 솔루션의 보안 통제 수준을 종합적으로 검증하여 공공 정보자산을 보호해야 한다.

📢 **섹션 요약 비유**: 망분리가 완벽한 시스템은 "외부 창구만 열려 있고 내부 금고는 창구를 통해서만 접근 가능한 은행"처럼 — 강도(해커)가 들어와도 창구를 넘어 금고까지 도달할 수 없다.

### 📌 관련 개념 맵

| 관계 | 개념 | 설명 |
|:---|:---|:---|
| 상위 개념 | 네트워크 보안 아키텍처 | 망분리는 심층 방어(Defense-in-Depth)의 핵심 레이어 |
| 하위 개념 | 망연계 솔루션 | 불가피한 연계 구간의 보안 통제 장치 |
| 연관 개념 | DLP (Data Loss Prevention) | 전송 콘텐츠 기반 유출 방지 기술 |
| 연관 개념 | APT (Advanced Persistent Threat) | 망분리가 차단하는 대표 위협 유형 |
| 연관 개념 | SIEM (Security Information and Event Management) | 망연계 로그 기반 이상 탐지 연계 |

### 👶 어린이를 위한 3줄 비유 설명

- 망분리는 학교 교실(내부망)과 교문 밖 거리(인터넷)를 담장(망분리)으로 완전히 나누는 거야.
- 외부에서 교실로 자료를 보내려면 반드시 교무실 검사(망연계 솔루션)를 통과해야 해.
- 감리원은 "담장에 구멍(우회 경로)이 없나요? 교무실 검사가 제대로 작동하나요?"라고 확인해!
