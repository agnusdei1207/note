## 핵심 인사이트 (3줄 요약)

> 1. **본질**: CISA (Certified Information Systems Auditor, 공인 정보시스템 감사인) 자격은 IT 감리·감사의 국제 표준 역량 체계로, 5개 도메인을 통해 정보시스템 전 생명주기를 감리하는 프레임워크를 제공한다.
> 2. **가치**: CISA 5개 도메인(감사 프로세스, IT 거버넌스, 시스템 생명주기, IT 운영, 자산 보호)은 국내 공공 정보시스템 감리와 1:1 매핑이 가능하여, 감리 기준의 국제 정합성을 높인다.
> 3. **판단 포인트**: 각 도메인의 통제 목표(Control Objective)를 실제 감리 점검 항목에 구체화하고, 위험 기반(Risk-Based) 접근으로 감리 범위와 심도를 조정하는 역량이 핵심이다.

---

## Ⅰ. 개요 및 필요성

### 1.1 CISA 자격 개요

CISA (Certified Information Systems Auditor)는 ISACA (Information Systems Audit and Control Association)가 발급하는 정보시스템 감사 분야 세계 최고 권위의 자격증이다. 1978년 도입 이후 전 세계 150여 개국 15만 명 이상이 취득한 국제 표준 자격이다.

CISA는 5개 도메인으로 구성되며, 각 도메인은 감리의 특정 영역을 담당한다.

### 1.2 CISA 5개 도메인 구성

| 도메인 | 명칭 | 비중 | 주요 내용 |
|:---:|:---|:---:|:---|
| 1 | IS 감사 프로세스 (Auditing Process) | 21% | 감사 계획·수행·보고 |
| 2 | IT 거버넌스와 관리 (IT Governance) | 17% | COBIT, 전략 연계 |
| 3 | 정보시스템 획득·개발·구현 (System Lifecycle) | 12% | SDLC, 변경 관리 |
| 4 | IT 운영과 비즈니스 탄력성 (IT Operations) | 23% | 운영 통제, BCP/DRP |
| 5 | 정보자산 보호 (Asset Protection) | 27% | 보안 통제, 암호화 |

📢 **섹션 요약 비유**: CISA 5개 도메인은 "종합병원의 5개 진료과"처럼 — 각 과(도메인)가 신체의 특정 부위(IT 영역)를 전문 진단하며, 종합 검진(감리)은 모든 과를 통과해야 완성된다.

## Ⅱ. 아키텍처 및 핵심 원리

### 2.1 CISA 도메인과 감리 매핑

```
  ┌─────────────────────────────────────────────────────┐
  │          CISA 도메인-감리 영역 매핑                  │
  ├────────────────────────────────────────────────────┤
  │  도메인 1: IS 감사 프로세스                          │
  │  → 감리 계획 수립, 위험 평가, 보고서 작성, 후속 조치  │
  ├────────────────────────────────────────────────────┤
  │  도메인 2: IT 거버넌스                               │
  │  → COBIT 기반 IT 전략-사업 연계 여부 점검            │
  ├────────────────────────────────────────────────────┤
  │  도메인 3: 시스템 생명주기                           │
  │  → SDLC 단계별 산출물, 변경 관리, 테스트 감리        │
  ├────────────────────────────────────────────────────┤
  │  도메인 4: IT 운영                                   │
  │  → 가용성·성능·백업·BCP/DRP 점검                    │
  ├────────────────────────────────────────────────────┤
  │  도메인 5: 자산 보호                                 │
  │  → 접근 통제, 암호화, 취약점 관리, 물리 보안         │
  └─────────────────────────────────────────────────────┘
```

### 2.2 위험 기반 감리 접근법 (Risk-Based Audit Approach)

```
  ┌──────────────────────────────────────────────────┐
  │       위험 기반 감리 접근 (Risk-Based Audit)      │
  ├──────────────────────────────────────────────────┤
  │                                                  │
  │  위험 식별 (Risk Identification)                  │
  │       │                                          │
  │  위험 평가 (Risk Assessment)                     │
  │  → 영향도(Impact) × 발생가능성(Likelihood)        │
  │       │                                          │
  │  위험 등급 설정 (Risk Rating: High/Mid/Low)       │
  │       │                                          │
  │  High Risk 영역 → 심층 감리                       │
  │  Low Risk 영역  → 표본 감리 또는 생략             │
  └──────────────────────────────────────────────────┘
```

### 2.3 COBIT (Control Objectives for Information Technology) 연계

| COBIT 도메인 | CISA 도메인 | 감리 점검 핵심 |
|:---|:---|:---|
| Evaluate, Direct and Monitor (EDM) | IT 거버넌스 | IT 전략 위원회 운영 여부 |
| Align, Plan and Organize (APO) | IT 거버넌스·운영 | IT 계획-사업 전략 연계 |
| Build, Acquire and Implement (BAI) | 시스템 생명주기 | SDLC 단계 통제 |
| Deliver, Service and Support (DSS) | IT 운영 | SLA 준수, 인시던트 관리 |
| Monitor, Evaluate and Assess (MEA) | 감사 프로세스 | KPI 달성 여부 모니터링 |

📢 **섹션 요약 비유**: COBIT-CISA 연계는 "자동차 설계도(COBIT)와 자동차 검사 기준서(CISA)"처럼 — 설계 기준(COBIT)에 맞게 만들어졌는지 검사(CISA)로 확인하는 것이다.

## Ⅲ. 비교 및 연결

### 3.1 CISA vs. 국내 정보시스템 감리 기준 비교

| 구분 | CISA (국제) | 국내 감리 (소프트웨어진흥법) |
|:---|:---|:---|
| 근거 | ISACA 프레임워크 | 소프트웨어진흥법 제46조 |
| 방법론 | 위험 기반 (Risk-Based) | 단계별 산출물 기반 |
| 범위 | 전사 IT 감사 | 사업 단위 감리 |
| 보고 기준 | ISACA 감사 표준 | 행안부 감리 수행 지침 |
| 국제 호환성 | 글로벌 인정 | 국내 한정 |

### 3.2 CISA 도메인별 핵심 통제 목표

| 도메인 | 핵심 통제 목표 | 감리 실무 적용 |
|:---|:---|:---|
| 감사 프로세스 | 독립성, 증거 수집, 문서화 | 감리원 독립성 확인 |
| IT 거버넌스 | 전략 연계, 위험 관리 | IT 전략위원회 운영 확인 |
| 시스템 생명주기 | SDLC 통제, 테스트 | 단계별 산출물 검토 |
| IT 운영 | 연속성, SLA, 변경 관리 | BCP 테스트 결과 확인 |
| 자산 보호 | 접근 통제, 암호화, 물리 보안 | 취약점 스캔 보고서 검토 |

📢 **섹션 요약 비유**: CISA 도메인은 "종합 건강검진 항목표"처럼 — 혈압(거버넌스), 혈당(운영), 심전도(보안) 등 각 항목이 정상 범위인지 체계적으로 점검한다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4.1 CISA 기반 감리 수행 절차

```
  ┌──────────────────────────────────────────────────────┐
  │       CISA 기반 감리 수행 절차                        │
  ├──────────────────────────────────────────────────────┤
  │  1단계: 감리 계획 수립 (Domain 1)                    │
  │     → 위험 식별, 감리 범위, 방법론 확정              │
  ├──────────────────────────────────────────────────────┤
  │  2단계: IT 거버넌스 검토 (Domain 2)                  │
  │     → COBIT 연계 여부, 경영진 책임 확인              │
  ├──────────────────────────────────────────────────────┤
  │  3단계: 시스템 구현 감리 (Domain 3)                  │
  │     → SDLC 산출물, 변경 관리 기록 검토               │
  ├──────────────────────────────────────────────────────┤
  │  4단계: 운영 통제 감리 (Domain 4)                    │
  │     → 가용성 SLA, 백업 테스트, BCP/DRP 검증          │
  ├──────────────────────────────────────────────────────┤
  │  5단계: 자산 보호 감리 (Domain 5)                    │
  │     → 접근 통제, 암호화, 취약점 관리 점검             │
  └──────────────────────────────────────────────────────┘
```

### 4.2 기술사 실무 판단 포인트

- **도메인 간 연계 점검**: 각 도메인은 독립적이지 않으며, 거버넌스(도메인2)가 부실하면 운영(도메인4)·보안(도메인5) 통제도 취약해지는 연쇄 효과 인식
- **증거(Evidence) 기반**: CISA 방법론은 "의혹이 아닌 증거"를 기반으로 감리 결과를 도출
- **독립성 유지**: ISACA IS 감사 표준 1.1 — 감리원은 감리 대상 시스템에 대한 이해관계가 없어야 함
- **CAATs (Computer-Assisted Audit Techniques) 활용**: 대용량 데이터 감사 시 ACL, IDEA 등 자동화 도구 활용

### 4.3 CISA 자격과 국내 기술사 연계 활용

| 항목 | CISA | 기술사 (정보관리) |
|:---|:---|:---|
| 발급 기관 | ISACA | 한국산업인력공단 |
| 갱신 | 연 20 CPE 이상 | 자격 유지 교육 |
| 주요 활용 | IT 감사·감리 국제 업무 | 국내 정보시스템 감리 |
| 상호 보완 | 국제 방법론 | 국내 법령·지침 |

📢 **섹션 요약 비유**: 기술사가 CISA 도메인 지식을 갖추면 "국내 면허(기술사)와 국제 자격증(CISA)을 동시에 가진 의사"처럼 — 국내외 어디서도 전문성을 인정받을 수 있다.

## Ⅴ. 기대효과 및 결론

### 5.1 기대효과

| 효과 | 내용 |
|:---|:---|
| 감리 품질 향상 | 위험 기반 접근으로 High Risk 영역 집중 |
| 국제 정합성 확보 | COBIT-CISA 연계로 글로벌 기준 준수 |
| 감리 효율성 | CAATs 활용으로 대용량 데이터 자동 분석 |
| 전문성 인정 | CISA 자격으로 국제 감리 업무 수행 가능 |

### 5.2 결론

CISA 5개 도메인 지식은 정보시스템 감리의 국제 표준 방법론을 제공한다. 기술사는 CISA 프레임워크를 국내 감리 법령과 결합하여, **위험 기반 접근**과 **증거 기반 판정**의 두 원칙으로 감리 수준을 선진화해야 한다.

📢 **섹션 요약 비유**: CISA 5도메인 감리는 "오케스트라 지휘자"처럼 — 각 섹션(도메인)이 제대로 연주되는지 확인하고, 전체 연주(감리 보고서)가 조화롭게 마무리되도록 이끈다.

### 👶 어린이를 위한 3줄 비유 설명

- CISA는 컴퓨터 시스템 점검사(감리원)가 받는 세계에서 가장 유명한 자격증이야.
- 마치 의사 자격증처럼, CISA가 있으면 전 세계 어디서든 시스템을 안전하게 점검할 수 있어.
- 5개 주요 점검 항목(도메인)을 다 잘 알아야 최고의 시스템 의사(CISA 감리원)가 될 수 있어!
