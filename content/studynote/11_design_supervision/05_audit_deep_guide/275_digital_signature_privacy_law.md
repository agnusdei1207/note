## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 전자서명법(Digital Signature Act)과 개인정보보호법(Personal Information Protection Act) 기반 감리는 온라인 거래·행정에서의 동의 획득 적법성, 전송 암호화 체계, 서명 인증 절차의 법적 충족성을 검증한다.
> 2. **가치**: 전자서명의 법적 유효성과 개인정보의 안전한 전송 보장은 디지털 행정·금융 서비스 신뢰의 근본이며, 위반 시 형사 처벌과 행정 제재가 병행된다.
> 3. **판단 포인트**: 공인 전자서명 또는 동등한 인증 수단 사용 여부, 개인정보 수집·이용 동의 화면의 법정 요건 충족, TLS (Transport Layer Security) 1.2 이상 전송 암호화 적용이 핵심 점검 항목이다.

---

## Ⅰ. 개요 및 필요성

### 1.1 전자서명법 핵심 내용

2020년 개정 「전자서명법」은 기존 공인인증서(공인전자서명) 독점 체계를 폐지하고, 다양한 전자서명 수단이 법적 효력을 가질 수 있도록 개방했다. 이로써 카카오·네이버·PASS 등 민간 인증 수단도 법적 유효성을 획득했다.

| 구분 | 구 전자서명법 (2020년 이전) | 현행 전자서명법 |
|:---|:---|:---|
| 인정 수단 | 공인인증서만 | 모든 검증된 전자서명 |
| 독점 구조 | 한국인터넷진흥원 체계 | 민간 경쟁 체계 |
| 법적 효력 | 공인인증서만 법적 우위 | 동등 법적 효력 |

### 1.2 개인정보보호법 핵심 의무

| 의무 사항 | 내용 | 위반 시 제재 |
|:---|:---|:---|
| 동의 획득 | 명시적·개별적 동의 | 과태료 3,000만 원 이하 |
| 최소 수집 | 서비스 필요 최소한 | 과징금 전년도 매출 3% |
| 전송 보안 | 암호화 전송 의무 | 형사 처벌 가능 |
| 보유 기간 | 동의 기간 또는 법정 기간 | 과태료 |
| 제3자 제공 | 별도 동의 필수 | 형사 처벌 |

📢 **섹션 요약 비유**: 전자서명법은 "도장의 종류는 자유롭게 선택하되, 계약서에 도장을 찍는 행위 자체는 엄격히 규정" — 어떤 도장(인증 수단)이든 쓸 수 있지만, 도장 찍기(전자서명)는 법대로 해야 한다.

## Ⅱ. 아키텍처 및 핵심 원리

### 2.1 전자서명 및 개인정보 처리 아키텍처

```
  ┌────────────────────────────────────────────────────────┐
  │         전자서명 + 개인정보 처리 흐름                    │
  ├────────────────────────────────────────────────────────┤
  │                                                        │
  │  [사용자]                                              │
  │     │                                                  │
  │  ①  동의 획득 화면                                     │
  │     │ (수집 항목, 이용 목적, 보유 기간 명시)            │
  │     │                                                  │
  │  ② 사용자 인증 + 전자서명                              │
  │     │ (공인인증서 / 민간인증서 / 생체인증)              │
  │     │                                                  │
  │  ③ TLS 1.2+ 암호화 전송                                │
  │     │ ──────────────────────────────► [서버]           │
  │     │                                                  │
  │  ④ 서버측 서명 검증                                    │
  │     │ (인증서 유효성, 서명 무결성 확인)                 │
  │     │                                                  │
  │  ⑤ 처리 결과 + 서명 보관                              │
  │     │ (10년 이상 보존 의무 - 전자문서법)               │
  └────────────────────────────────────────────────────────┘
```

### 2.2 개인정보 동의 화면 법정 요건

```
  ┌──────────────────────────────────────────────────────┐
  │       개인정보 동의 화면 법정 요건 (개인정보보호법)    │
  ├──────────────────────────────────────────────────────┤
  │  필수 기재 항목:                                      │
  │  ┌─────────────────────────────────────────────────┐  │
  │  │  1. 수집·이용 목적                               │  │
  │  │  2. 수집 항목 (필수·선택 구분)                   │  │
  │  │  3. 보유 및 이용 기간                            │  │
  │  │  4. 동의 거부 권리 및 불이익 고지                │  │
  │  │  5. 제3자 제공 시 별도 동의 (해당 시)            │  │
  │  └─────────────────────────────────────────────────┘  │
  │  금지 행위:                                           │
  │  - 필수·선택 항목 통합 동의 처리                      │
  │  - 사전 체크 동의 (Pre-checked Checkbox)              │
  │  - 동의 거부 시 서비스 전면 거부                       │
  └──────────────────────────────────────────────────────┘
```

### 2.3 전자서명 인증 수단 비교

| 수단 | 법적 근거 | 보안 수준 | 적합 서비스 |
|:---|:---|:---|:---|
| 공동인증서(구 공인인증서) | 전자서명법 | 높음 | 공공·금융 고위험 거래 |
| 민간 인증서 (카카오·네이버 등) | 전자서명법 (평가 기반) | 중~높음 | 일반 온라인 서비스 |
| FIDO (Fast Identity Online) | 전자서명법 동등 | 높음 | 모바일 생체 인증 |
| OTP (One-Time Password) | 전자금융감독규정 | 중간 | 2차 인증 보조 |

📢 **섹션 요약 비유**: 전자서명 수단 다양화는 "도장 외에 서명·지문·홍채 등 다양한 방법으로 신원을 확인할 수 있게 된 것"처럼 — 방법은 자유롭지만 법적 효력 기준은 동일하게 유지된다.

## Ⅲ. 비교 및 연결

### 3.1 암호화 전송 기술 비교

| 구분 | TLS 1.0/1.1 (구버전) | TLS 1.2 | TLS 1.3 |
|:---|:---|:---|:---|
| 취약점 | POODLE, BEAST | 낮음 | 거의 없음 |
| 성능 | 느림 | 중간 | 빠름 (1-RTT) |
| 사용 여부 | 금지 (Major 결함) | 최소 기준 | 권고 |
| 적용 의무 | 개인정보보호법 제29조 | 기준 | 권고 |

### 3.2 개인정보보호법 vs. EU GDPR 비교

| 구분 | 개인정보보호법 (한국) | GDPR (EU) |
|:---|:---|:---|
| 동의 기준 | 선택적 사항 구분 동의 | 명확하고 적극적인 동의 |
| 잊혀질 권리 | 삭제 요청권 | 잊혀질 권리 (Article 17) |
| 과징금 | 전년도 매출 3% | 전 세계 매출 4% |
| 개인정보 이동권 | 일부 규정 | 완전 보장 (Article 20) |
| DPO (Data Protection Officer) | 의무 아님 (일부 권고) | 고위험 기업 의무 |

📢 **섹션 요약 비유**: 개인정보 동의는 "계약서 조항을 모두 읽고 서명하는 것"처럼 — 작은 글씨로 숨긴 조항(사전 체크)은 법적으로 무효다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4.1 전자서명·개인정보 감리 체크리스트

```
  ┌──────────────────────────────────────────────────────┐
  │     전자서명·개인정보보호법 감리 체크리스트            │
  ├──────────────────────────────┬───────────────────────┤
  │ 점검 항목                    │ 기준                  │
  ├──────────────────────────────┼───────────────────────┤
  │ 전자서명 법정 인증 수단 사용  │ 전자서명법 적합        │
  │ 동의 화면 법정 항목 포함      │ 5개 항목 전부 필수    │
  │ 필수·선택 항목 구분 동의      │ 분리 동의 필수        │
  │ 사전 체크 동의 방식 금지      │ 금지 (Major)          │
  │ TLS 1.2 이상 전송 암호화      │ 필수 (Major)          │
  │ 전자서명 원문 보관            │ 법정 보관 기간 준수   │
  │ 개인정보 처리방침 공개        │ 홈페이지 상시 공개    │
  │ 개인정보 파기 절차 이행       │ 보유 기간 도래 즉시   │
  └──────────────────────────────┴───────────────────────┘
```

### 4.2 기술사 실무 판단 포인트

- **동의 유효성 검증**: 동의 화면 스크린샷·HTML 소스 분석으로 사전 체크 여부, 필수·선택 구분, 거부권 고지 확인
- **개인정보 전송 암호화**: 로그인·회원가입·결제 등 모든 개인정보 전송 구간에 TLS 적용 여부 HTTPS 점검 도구로 실측
- **전자서명 원문 무결성**: 서명 당시의 문서 원본이 변경되지 않았음을 증명할 수 있는 타임스탬프·해시 보관 체계 확인
- **위수탁 개인정보 처리**: 개인정보 처리를 외부에 위탁하는 경우 수탁자 관리 감독 여부, 위탁 계약서 현황 확인

### 4.3 전자문서·전자서명 보존 기간

| 문서 유형 | 보존 기간 | 근거 법령 |
|:---|:---|:---|
| 전자계약서 | 계약 만료 후 5년 | 전자문서법 |
| 공공 전자서명 서류 | 10년 이상 | 전자정부법 |
| 개인정보 처리 동의 기록 | 동의 철회 또는 계약 종료 후 | 개인정보보호법 |
| 금융 거래 기록 | 5년 | 전자금융거래법 |

📢 **섹션 요약 비유**: 기술사는 전자서명·개인정보 감리에서 "계약 공증인"처럼 — 서명이 올바른 방법으로 이루어졌는지, 계약 내용이 안전하게 전달되고 보관되는지를 법 기준으로 확인한다.

## Ⅴ. 기대효과 및 결론

### 5.1 기대효과

| 효과 | 내용 |
|:---|:---|
| 법적 유효성 확보 | 전자서명의 법적 분쟁 가능성 제거 |
| 개인정보 보호 | 전송 암호화·적법 동의로 침해 방지 |
| 이용자 신뢰 확보 | 투명한 동의 절차로 서비스 신뢰도 향상 |
| 규제 리스크 해소 | 과징금·형사 처벌 위험 사전 제거 |

### 5.2 결론

전자서명·개인정보보호법 감리는 법적 요건과 기술적 구현의 정합성을 동시에 검증하는 법-기술 융합 감리 영역이다. 기술사는 동의 화면의 법정 요건 충족, TLS 적용 범위의 완전성, 전자서명 원문 무결성 보존 체계를 핵심 축으로 체계적으로 감리해야 한다.

📢 **섹션 요약 비유**: 전자서명·개인정보 감리를 완료한 서비스는 "공증 도장과 이중 잠금 봉투를 모두 갖춘 중요 계약서"처럼 — 누가 서명했는지, 내용이 바뀌지 않았는지, 안전하게 전달되었는지 모두 보장된다.

### 📌 관련 개념 맵

| 관계 | 개념 | 설명 |
|:---|:---|:---|
| 상위 개념 | 디지털 신뢰 체계 (Digital Trust) | 전자서명은 온라인 신뢰의 기반 |
| 하위 개념 | TLS (Transport Layer Security) | 개인정보 전송 암호화의 표준 |
| 연관 개념 | PKI (Public Key Infrastructure) | 전자서명 인증서 발급·관리 체계 |
| 연관 개념 | FIDO (Fast Identity Online) | 패스워드 없는 생체 인증 표준 |
| 연관 개념 | 개인정보보호법 제29조 | 전송 암호화 의무 조항 |

### 👶 어린이를 위한 3줄 비유 설명

- 전자서명은 컴퓨터에서 도장 찍는 것인데, 어떤 도장이든 쓸 수 있지만 법이 정한 방법으로 해야 해.
- 개인 정보(이름, 주소)를 인터넷으로 보낼 때는 암호 봉투(TLS)에 넣어야 도둑이 못 열어.
- 감리원은 "도장이 올바른 방법으로 찍혔나요? 봉투는 안전하게 잠겼나요?"라고 확인해!
