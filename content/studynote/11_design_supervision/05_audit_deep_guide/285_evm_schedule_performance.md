## 핵심 인사이트 (3줄 요약)

> 1. **본질**: EVM(Earned Value Management, 획득 가치 관리)은 계획 대비 실제 진척을 비용·일정 두 축에서 동시에 측정하는 객관적 PM 기법이며, '진척률 뻥튀기(Over-Reporting)'를 수학적으로 탐지하는 유일한 방법이다.
> 2. **가치**: SPI(Schedule Performance Index, 일정 성과 지수) < 1.0이면 일정 지연, CPI(Cost Performance Index, 비용 성과 지수) < 1.0이면 비용 초과를 의미하므로, 두 지수 조작이 가장 위험한 감리 대상이다.
> 3. **판단 포인트**: EVM 보고서 수치를 현장 방문·체크리스트·완성 산출물과 직접 대조하여, 보고된 EV(Earned Value, 획득 가치)가 실제로 완성된 작업에 기반하는지 검증한다.

## Ⅰ. 개요 및 필요성

### 1.1 EVM의 3대 핵심 지표

EVM은 세 가지 기준값으로 프로젝트 상태를 정량화한다.

- **PV (Planned Value, 계획 가치)**: 특정 시점까지 완성되어야 할 작업의 예산 가치
- **EV (Earned Value, 획득 가치)**: 실제로 완성된 작업의 예산 가치 — **감리 조작의 주요 대상**
- **AC (Actual Cost, 실제 비용)**: 실제로 투입된 비용

### 1.2 파생 지표

| 지표 | 계산식 | 의미 | 정상 범위 |
|:---|:---|:---|:---:|
| CV (Cost Variance, 비용 차이) | EV - AC | 양수: 비용 절감 | ≥ 0 |
| SV (Schedule Variance, 일정 차이) | EV - PV | 양수: 일정 앞서감 | ≥ 0 |
| CPI (Cost Performance Index) | EV / AC | 1.0 미만: 비용 초과 | ≥ 1.0 |
| SPI (Schedule Performance Index) | EV / PV | 1.0 미만: 일정 지연 | ≥ 1.0 |
| EAC (Estimate at Completion) | BAC / CPI | 현 추세 기반 최종 예상 비용 | ≤ BAC |

📢 **섹션 요약 비유**: EVM은 마라톤 GPS 트래커다. 목표 페이스(PV)와 실제 달린 거리(EV), 소비한 체력(AC)을 비교해 완주 예상 시간을 실시간으로 계산한다.

## Ⅱ. 아키텍처 및 핵심 원리

### 2.1 EVM 측정 구조

```
┌──────────────────────────────────────────────────────────────────┐
│                        EVM 측정 구조                              │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  금액                                                            │
│  ▲                                                               │
│  │          ·····BAC (완료 시 총 예산)                           │
│  │    PV ···/                                                    │
│  │       · /   EV (획득 가치선)                                  │
│  │     ·  /  /                                                   │
│  │   ·   / /    AC (실제 비용선)                                 │
│  │  ·   //   /                                                   │
│  │ ·   /·  /                                                     │
│  │·  /  · /                                                      │
│  │ /   ·/                                                        │
│  └────────────────────────────────────────▶ 시간                │
│    │    │                                                        │
│    │    현재 시점                                                 │
│    │                                                             │
│    ┌────────────────────────────────────────────────────────┐   │
│    │  SV = EV - PV  (음수: EV가 PV보다 낮음 = 일정 지연)    │   │
│    │  CV = EV - AC  (음수: AC가 EV보다 높음 = 비용 초과)    │   │
│    │  감리 조작 지점: EV 수치 부풀리기 → SPI/CPI 인위 향상  │   │
│    └────────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────────┘
```

### 2.2 EV 측정 방법별 특성

| 방법 | 적용 WBS 유형 | 조작 위험 | 감리 검증 방법 |
|:---|:---|:---:|:---|
| 0/100% (완료 기준) | 단기 작업 | 낮음 | 산출물 완성 여부 확인 |
| 50/50 (착수/완료) | 중기 작업 | 중간 | 착수 및 완료 시점 문서 확인 |
| 마일스톤 가중치 | 장기 단계 작업 | 높음 | 마일스톤별 완성 산출물 확인 |
| 공정률 (% Complete) | 연속 작업 | 매우 높음 | 현장 실사, 체크리스트 교차 확인 |

📢 **섹션 요약 비유**: EV 측정 방법은 숙제 완성도 측정 방식이다. "다 했어요"(0/100)는 검증하기 쉽지만, "80% 했어요"(공정률)는 직접 봐야 알 수 있다.

## Ⅲ. 비교 및 연결

### 3.1 진척 뻥튀기(Over-Reporting) 패턴과 감리 대응

| 패턴 | 설명 | 탐지 방법 | 감리 판단 |
|:---|:---|:---|:---:|
| 미완성 산출물 완료 처리 | EV 부풀리기 | 산출물 완성도 직접 확인 | 납품 불가 |
| 공정률 임의 상향 | "80% → 90%"로 변경 | 이전 보고 대비 비율 검증 | 중요 지적 |
| 재작업 공수 EV 재계산 제외 | AC 낮추기 | 투입 공수 대장 교차 확인 | 중요 지적 |
| 선행 완료 후 후행 EV 계상 | 시간 왜곡 | WBS 의존 관계 확인 | 지적 |

### 3.2 EVM 성과 지수 해석

| SPI 범위 | CPI 범위 | 상태 | 감리 대응 |
|:---:|:---:|:---|:---|
| ≥ 1.0 | ≥ 1.0 | 계획 이상 진행 | 데이터 신뢰성 재확인 |
| < 1.0 | ≥ 1.0 | 일정 지연, 비용 절감 | 일정 만회 계획 요구 |
| ≥ 1.0 | < 1.0 | 일정 앞서나감, 비용 초과 | 비용 초과 원인 분석 요구 |
| < 1.0 | < 1.0 | 일정 지연 + 비용 초과 | 비상 대응, 에스컬레이션 |

📢 **섹션 요약 비유**: SPI·CPI는 자동차 계기판이다. 속도계(SPI)와 연료 효율(CPI) 두 가지를 동시에 보면 여행이 예정대로 가는지 알 수 있다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4.1 EVM 진척 감리 체크리스트

**EVM 기준선 검증**
- [ ] PMB(Performance Measurement Baseline, 성과 측정 기준선)이 WBS와 연결되어 있는가?
- [ ] BAC(Budget at Completion, 완료 예산)이 계약 금액과 일치하는가?
- [ ] EV 측정 방법(0/100, 마일스톤, 공정률)이 WBS 유형별로 명시되어 있는가?

**현장 대조 검증**
- [ ] 보고서상 EV에 해당하는 산출물이 실제로 완성되어 있는가?
- [ ] 공정률 기반 EV는 현장 방문 또는 체크리스트로 교차 검증되었는가?
- [ ] EAC(최종 예상 비용)가 현재 추세(CPI)와 논리적으로 일치하는가?

### 4.2 기술사 답안 포인트

EVM 감리의 핵심은 **EV 산출 근거의 현장 검증 가능성**이다.

1. **EV 측정 방법 적정성**: 작업 유형에 맞는 EV 측정 방법이 사전에 정의되었는가?
2. **현장 교차 검증**: EVM 보고 수치가 현장 실사·산출물 확인으로 입증되는가?
3. **추세 일관성**: SPI·CPI 추세가 사업 실제 상황과 일치하는가?

📢 **섹션 요약 비유**: EVM 감리는 탐정 조사다. 용의자의 알리바이(보고서)가 현장 증거(실제 완성 산출물)와 일치하는지 직접 확인한다.

## Ⅴ. 기대효과 및 결론

### 5.1 EVM 진척 관리 감리 기대효과

- **조기 경보**: SPI/CPI < 0.9 구간에서 프로젝트 실패 가능성 조기 탐지
- **진척 투명성**: 주관적 진척 보고의 객관화로 이해관계자 신뢰 확보
- **비용 예측**: EAC 추세 분석으로 예산 초과 규모 조기 예측
- **책임 명확화**: WBS 단위 EV 측정으로 지연·비용 초과 책임 소재 명확화

### 5.2 결론

EVM 진척 관리 감리는 **수치의 산술적 검증이 아니라 EV 산출 근거의 현실 부합성**을 검증하는 고난도 전문 활동이다. 감리인은 EVM 보고서를 그대로 수용하지 않고, 핵심 WBS 항목의 완성 산출물을 직접 확인하고, 공정률 기반 EV는 반드시 현장 실사로 교차 검증해야 한다.

📢 **섹션 요약 비유**: EVM 감리는 성적표 검증이다. "90점 받았어요"라는 말을 믿는 게 아니라, 시험지를 직접 보고 점수를 세어보는 것이다.

### 👶 어린이를 위한 3줄 비유 설명

- EVM은 숙제를 얼마나 했는지 점수로 재는 거야. 계획한 것(PV), 실제로 한 것(EV), 쓴 시간(AC)을 비교해.
- SPI는 "계획대로 했어요?" 지수고, CPI는 "돈(시간)을 효율적으로 썼어요?" 지수야.
- 감리는 "정말 80% 했어요?" 하고 숙제를 직접 보여달라고 하는 선생님이야. 말만으로는 안 돼.
