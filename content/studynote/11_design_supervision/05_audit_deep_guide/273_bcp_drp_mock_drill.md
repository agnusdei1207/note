## 핵심 인사이트 (3줄 요약)

> 1. **본질**: BCP (Business Continuity Plan, 업무 연속성 계획) / DRP (Disaster Recovery Plan, 재해 복구 계획)의 실효성은 계획서 존재가 아닌 **모의 훈련 대장 로그 검증**을 통해 확인한다.
> 2. **가치**: 모의 훈련은 이론적 복구 시나리오가 실제 재해 환경에서 작동하는지 검증하며, 미발견 취약점을 사전에 노출시켜 실제 재해 시의 충격을 최소화한다.
> 3. **판단 포인트**: RTO (Recovery Time Objective, 복구 목표 시간) 및 RPO (Recovery Point Objective, 복구 목표 시점) 달성 여부, 모의 훈련 수행 이력(연 1회 이상), 훈련 결과 후속 조치 완료 여부가 핵심 점검 항목이다.

---

## Ⅰ. 개요 및 필요성

### 1.1 BCP/DRP 개요

| 계획 | 정의 | 목적 |
|:---|:---|:---|
| BCP (Business Continuity Plan) | 재해 시 핵심 비즈니스 기능의 연속성 보장 계획 | 비즈니스 중단 최소화 |
| DRP (Disaster Recovery Plan) | IT 시스템·인프라 복구 절차 계획 | 시스템 복구 시간 최소화 |

BCP는 DRP를 포함하는 상위 개념으로, DRP는 IT 인프라 복구에 집중하고 BCP는 전체 사업 연속성을 다룬다.

### 1.2 모의 훈련의 필요성

계획서만 있고 훈련이 없는 BCP/DRP는 실제 재해 시 다음과 같은 문제를 야기한다:

- **인력 혼란**: 훈련되지 않은 직원은 재해 시 역할을 모름
- **절차 오류**: 문서화된 절차가 실제 환경과 불일치
- **기술적 실패**: 백업 시스템이 장기간 미검증으로 복구 불가
- **RTO 초과**: 실제 복구 시간이 계획 대비 수 배 초과

📢 **섹션 요약 비유**: BCP/DRP 모의 훈련은 "소방 훈련"과 완전히 같다 — 화재 대피 계획서가 있어도 실제 훈련 없이는 불이 났을 때 아무도 어디로 도망가야 할지 모른다.

### 2.3 모의 훈련 결과 기록 항목

| 기록 항목 | 내용 | 기준 |
|:---|:---|:---|
| 훈련 일시 | 연 1회 이상 실시 | 법적 의무 |
| 시나리오 내용 | 재해 유형 (화재·지진·사이버 등) | 다양한 시나리오 |
| 실제 RTO 측정값 | 훈련 중 실제 복구 완료 시간 | 계획 RTO 이하 |
| 실제 RPO 달성 여부 | 데이터 손실 범위 확인 | 계획 RPO 이하 |
| 발견된 문제점 | 절차 오류, 기술적 장애 등 | 전수 기록 |
| 후속 조치 완료 | 문제점 시정 여부 | 30일 내 완료 |

📢 **섹션 요약 비유**: RTO는 "불이 나면 몇 분 안에 건물을 떠나야 하는가"이고, RPO는 "마지막으로 복사해 둔 파일이 언제 것인가" — 둘 다 짧을수록 안전하다.

## Ⅲ. 비교 및 연결

### 3.1 훈련 유형별 장단점 비교

| 유형 | 장점 | 단점 | 빈도 권고 |
|:---|:---|:---|:---|
| 계획 검토 (Paper Review) | 저비용, 위험 없음 | 실제 복구 미검증 | 분기 1회 |
| 워크샵 (Tabletop Exercise) | 팀 역할 인지 향상 | 기술적 검증 미흡 | 반기 1회 |
| 시뮬레이션 (Simulation) | 부분 복구 절차 검증 | 전체 복구 미검증 | 연 1회 |
| 실물 훈련 (Live Drill) | 완전한 실효성 검증 | 고비용, 운영 중단 위험 | 2~3년 1회 |

### 3.2 BCP vs. DRP 핵심 차이

| 구분 | BCP (Business Continuity Plan) | DRP (Disaster Recovery Plan) |
|:---|:---|:---|
| 범위 | 전체 사업 기능 | IT 시스템·인프라 |
| 목표 | 비즈니스 연속성 | 시스템 복구 |
| 주체 | 경영진 + IT | IT 부서 |
| RTO 기준 | 비즈니스 단위 | 시스템 단위 |
| 포함 관계 | DRP를 포함 | BCP의 하위 계획 |

📢 **섹션 요약 비유**: BCP는 "회사 전체의 비상 운영 계획"이고, DRP는 "컴퓨터실 복구 계획" — 컴퓨터는 고쳐도 직원들이 출근을 못하면 회사가 안 돌아간다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4.1 BCP/DRP 감리 체크리스트

```
  ┌──────────────────────────────────────────────────────┐
  │       BCP/DRP 모의 훈련 감리 체크리스트               │
  ├──────────────────────────────┬───────────────────────┤
  │ 점검 항목                    │ 기준                  │
  ├──────────────────────────────┼───────────────────────┤
  │ BCP/DRP 문서 존재 여부       │ 필수 (Major)          │
  │ 모의 훈련 수행 이력          │ 연 1회 이상 (Major)   │
  │ 실제 RTO 달성 여부           │ 계획 RTO 이하         │
  │ 실제 RPO 달성 여부           │ 계획 RPO 이하         │
  │ 훈련 결과 보고서 작성        │ 매 훈련 후 필수       │
  │ 발견 문제점 후속 조치        │ 30일 내 완료 권고     │
  │ 최신 시스템 반영 여부        │ 연 1회 이상 업데이트  │
  │ 다양한 재해 시나리오 포함    │ 3개 이상 시나리오     │
  └──────────────────────────────┴───────────────────────┘
```

### 4.2 기술사 실무 판단 포인트

- **훈련 대장 진위 확인**: 모의 훈련 대장이 실제 훈련 수행의 근거인지, 형식적으로 작성된 것인지 구분 (훈련 전후 사진, 참가자 서명, 시간 기록으로 확인)
- **RTO 목표치 적정성**: 업무 중요도 분류(BIA, Business Impact Analysis) 결과에 근거한 RTO 설정 여부 검토
- **Hot/Warm/Cold Site 적합성**: 복구 사이트 유형이 RTO 목표와 일치하는지 확인 (Hot Site: 즉시 전환, Warm: 수시간, Cold: 수일)
- **3-2-1 백업 원칙**: 데이터 3개 복사본, 2개 다른 미디어, 1개 오프사이트 보관 여부 확인

### 4.3 재해 복구 사이트 유형 비교

| 사이트 유형 | 준비 상태 | RTO | 비용 |
|:---|:---|:---|:---|
| Hot Site | 실시간 미러링, 즉시 전환 가능 | 수 분 | 매우 높음 |
| Warm Site | 장비 대기, 데이터 주기적 동기화 | 수 시간 | 높음 |
| Cold Site | 빈 공간만, 장비·데이터 별도 이전 | 수 일 | 낮음 |
| Cloud DR (Disaster Recovery) | 클라우드 기반 탄력적 복구 | 수 분~시간 | 사용량 기반 |

📢 **섹션 요약 비유**: 기술사는 BCP/DRP 감리에서 "소방서 감독관"처럼 — 소화기가 있는지뿐 아니라, 직원들이 소화기 사용법을 실제로 알고 있는지(모의 훈련 이력)를 확인한다.

## Ⅴ. 기대효과 및 결론

### 5.1 기대효과

| 효과 | 내용 |
|:---|:---|
| 복구 시간 단축 | 훈련된 인력의 빠른 역할 수행 |
| 취약점 사전 발견 | 훈련 중 발견된 문제 사전 조치 |
| 비즈니스 신뢰성 | 재해 시 서비스 연속성 보장 |
| 법적 의무 충족 | 전자정부법·개인정보보호법상 연속성 보장 의무 |

### 5.2 결론

BCP/DRP 모의 훈련 감리는 계획서의 존재를 넘어 **실효성 검증**에 집중한다. 기술사는 RTO·RPO 달성 여부와 훈련 대장의 진위성을 중심으로, 모의 훈련이 형식적 행사가 아닌 실질적 위기 대비 도구로 기능하고 있는지를 검증해야 한다.

📢 **섹션 요약 비유**: BCP/DRP 감리가 완료된 시스템은 "매년 소방 훈련을 실시하고, 훈련 시간을 줄여나가는 건물"처럼 — 진짜 불이 나도 모두가 침착하게 대피하고 복구할 수 있다.

### 📌 관련 개념 맵

| 관계 | 개념 | 설명 |
|:---|:---|:---|
| 상위 개념 | 업무 연속성 관리 (BCM, Business Continuity Management) | BCP/DRP는 BCM의 하위 계획 |
| 하위 개념 | RTO/RPO | 복구 목표 시간·시점 핵심 지표 |
| 연관 개념 | BIA (Business Impact Analysis) | 업무 중요도·영향도 분석 |
| 연관 개념 | Hot/Warm/Cold Site | 복구 사이트 유형 분류 |
| 연관 개념 | 3-2-1 백업 원칙 | 데이터 안전한 보존 전략 |

### 👶 어린이를 위한 3줄 비유 설명

- BCP/DRP는 "집에 불이 났을 때 어떻게 탈출할지" 미리 적어둔 계획서야.
- 그런데 그 계획서가 진짜 도움이 되려면 실제로 연습(모의 훈련)을 해봐야 해.
- 감리원은 "그 연습을 정말로 했나요? 기록이 있나요?"라고 확인하는 사람이야!
