## 핵심 인사이트 (3줄 요약)

> 1. **본질**: AI (Artificial Intelligence) 모델 성능 감리는 학습 데이터셋(Dataset) 품질, 모델 평가 지표(Precision, Recall, F1-Score) 적정성, 편향성(Bias) 여부, MLOps (Machine Learning Operations) 배포 절차의 4축을 체계적으로 검증한다.
> 2. **가치**: AI 모델의 편향성이나 과적합(Overfitting)은 공공 의사결정에 직접 영향을 미치므로, 감리를 통해 알고리즘 공정성과 신뢰성을 사전에 보장해야 한다.
> 3. **판단 포인트**: 학습/검증/테스트 데이터 분리 비율, F1-Score 임계값(Threshold), 인구통계학적 동등성(Demographic Parity) 충족 여부가 핵심 점검 항목이다.

---

## Ⅰ. 개요 및 필요성

### 1.1 AI 모델 감리의 배경

공공기관의 AI 시스템이 행정 처분, 복지 수급 결정, 신용 평가 등에 활용되면서, 알고리즘 편향성으로 인한 차별 문제가 전 세계적으로 대두되고 있다. 2021년 EU (European Union) AI Act는 고위험 AI 시스템에 대해 의무적인 검증과 투명성 보고를 요구한다.

국내에서도 「인공지능법」(가칭) 논의가 진행되며, 공공 AI 시스템의 감리 필요성이 법제화 방향으로 가고 있다.

### 1.2 감리 필요 영역

| 감리 영역 | 내용 | 위험 |
|:---|:---|:---|
| 데이터셋 품질 | 학습 데이터 대표성·완전성 | 편향 모델 생성 |
| 모델 성능 지표 | Precision, Recall, F1-Score | 오분류 의사결정 |
| 편향성 감리 | 인종·성별·지역 편향 | 차별적 결과 |
| MLOps 배포 | CI/CD (Continuous Integration/Delivery) 파이프라인 | 미검증 모델 배포 |

📢 **섹션 요약 비유**: AI 모델 감리는 "시험 출제자(데이터)와 채점 기준(지표)이 공정한지 확인하는 것"처럼 — 편향된 문제(데이터)로 학습한 AI는 아무리 성적이 좋아도 공정하지 않다.

## Ⅱ. 아키텍처 및 핵심 원리

### 2.1 AI 모델 성능 감리 체계

```
  ┌──────────────────────────────────────────────────────┐
  │          AI 모델 성능 감리 체계 (Audit Framework)     │
  ├────────────────┬─────────────────┬────────────────────┤
  │  데이터셋 감리 │  모델 성능 감리 │  MLOps 배포 감리   │
  ├────────────────┼─────────────────┼────────────────────┤
  │ 수집 출처 확인 │ 혼동 행렬 분석  │ 버전 관리 확인     │
  │ 레이블 품질    │ Precision 점검  │ A/B 테스트 여부    │
  │ 불균형 비율    │ Recall 점검     │ 모델 롤백 절차     │
  │ 분리 비율      │ F1-Score 기준   │ 드리프트 모니터링  │
  │ 개인정보 처리  │ AUC-ROC 확인    │ 재학습 주기 계획   │
  └────────────────┴─────────────────┴────────────────────┘
```

### 2.2 혼동 행렬 (Confusion Matrix) 기반 성능 지표

```
  ┌────────────────────────────────────────────────────┐
  │        혼동 행렬 (Confusion Matrix)                │
  │                                                    │
  │              예측 양성(Positive) 예측 음성(Negative) │
  │  실제 양성│  TP (True Positive) │ FN (False Neg.) │ │
  │  실제 음성│  FP (False Positive)│ TN (True Neg.)  │ │
  │                                                    │
  │  Precision = TP / (TP + FP)  → 예측 정확도        │
  │  Recall    = TP / (TP + FN)  → 실제 탐지율        │
  │  F1-Score  = 2 × (P × R) / (P + R)  → 조화 평균  │
  └────────────────────────────────────────────────────┘
```

### 2.3 모델 평가 지표별 적용 기준

| 지표 | 의미 | 공공 AI 적용 기준 | 우선 적용 상황 |
|:---|:---|:---|:---|
| Precision (정밀도) | 예측 양성 중 실제 양성 비율 | 0.85 이상 권고 | 오탐 비용 높을 때 |
| Recall (재현율) | 실제 양성 중 예측 양성 비율 | 0.90 이상 권고 | 미탐 비용 높을 때 |
| F1-Score | 정밀도와 재현율의 조화 평균 | 0.87 이상 권고 | 균형 중시 |
| AUC-ROC | 분류 성능 종합 지표 | 0.90 이상 권고 | 이진 분류 전반 |

📢 **섹션 요약 비유**: Precision과 Recall의 트레이드오프는 "그물 코 크기"와 같아 — 그물 코가 작으면(Precision 높음) 작은 물고기도 잡히지만 쓸모없는 것도 많이 걸리고, 크면(Recall 높음) 놓치는 물고기가 없지만 쓸모없는 것도 많이 지나간다.

## Ⅲ. 비교 및 연결

### 3.1 편향성(Bias) 유형 비교

| 편향 유형 | 원인 | 예시 | 감리 대응 |
|:---|:---|:---|:---|
| 샘플 편향 (Sampling Bias) | 특정 집단 과대/과소 대표 | 여성 데이터 부족한 채용 AI | 데이터 분포 분석 |
| 레이블 편향 (Label Bias) | 레이블링 담당자의 주관 | 특정 인종 범죄자 분류 과다 | 크로스 체크 레이블링 |
| 측정 편향 (Measurement Bias) | 특정 집단에 부정확한 특성 | 피부색에 따른 얼굴 인식 오류 | 서브그룹별 성능 분리 측정 |
| 알고리즘 편향 | 모델 최적화 목표 설정 오류 | 주류 집단 최적화 | 공정성 제약 조건 추가 |

### 3.2 공정성 지표 비교

| 지표 | 정의 | 적용 시나리오 |
|:---|:---|:---|
| 인구통계학적 동등성 (Demographic Parity) | 집단 간 양성 예측 비율 동일 | 채용·대출 심사 |
| 균등화 기회 (Equalized Odds) | 집단 간 TP Rate·FP Rate 동일 | 의료 진단 |
| 예측 동등성 (Predictive Parity) | 집단 간 Precision 동일 | 재범 위험 예측 |

📢 **섹션 요약 비유**: AI 편향성 감리는 "달리기 경주에서 출발선이 모든 선수에게 공평한지 확인하는 것"처럼 — 특정 그룹이 불리한 출발선(편향 데이터)에서 시작하면 결과도 불공평하다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4.1 AI 모델 성능 감리 체크리스트

```
  ┌────────────────────────────────────────────────────┐
  │         AI 모델 성능 감리 체크리스트                │
  ├──────────────────────────────┬─────────────────────┤
  │ 점검 항목                    │ 기준                │
  ├──────────────────────────────┼─────────────────────┤
  │ 학습/검증/테스트 분리        │ 6:2:2 또는 7:1.5:1.5│
  │ 데이터 클래스 불균형 비율    │ 1:3 이내 권고       │
  │ F1-Score 기준 충족           │ 0.87 이상           │
  │ 서브그룹별 성능 분리 측정    │ 주요 집단 명시      │
  │ 편향성 평가 수행             │ 정성+정량 평가 필수 │
  │ MLOps 파이프라인 버전 관리   │ Git 기반 관리       │
  │ 모델 드리프트 모니터링       │ 월별 재평가 계획    │
  │ 롤백(Rollback) 절차 문서화  │ 30분 내 복구 계획   │
  └──────────────────────────────┴─────────────────────┘
```

### 4.2 MLOps (Machine Learning Operations) 배포 감리

```
  ┌──────────────────────────────────────────────────┐
  │       MLOps 배포 파이프라인 감리 체크포인트       │
  ├──────────────────────────────────────────────────┤
  │  코드/데이터/모델 버전 관리 (Git + DVC)           │
  │        │                                         │
  │  자동화 테스트 (Unit Test + 성능 Regression)     │
  │        │                                         │
  │  스테이징(Staging) 환경 검증                     │
  │        │                                         │
  │  A/B 테스트 또는 Canary 배포                     │
  │        │                                         │
  │  프로덕션 배포 + 모니터링 활성화                 │
  │        │                                         │
  │  드리프트 탐지 → 재학습 트리거                   │
  └──────────────────────────────────────────────────┘
```

### 4.3 기술사 실무 판단 포인트

- **성능 지표 임계값 명시**: 요구사항 명세서에 F1-Score, Precision, Recall 최솟값 명시 요구
- **블랙박스 모델 설명 가능성**: 공공 의사결정 AI는 XAI (Explainable AI, 설명 가능 AI) 적용 필수
- **데이터 계보(Data Lineage)**: 학습 데이터의 수집 출처, 전처리 이력 추적 가능성 확인
- **배포 후 모니터링**: 데이터 드리프트(Distribution Shift) 탐지 자동화 체계 확인

📢 **섹션 요약 비유**: 기술사는 AI 성능 감리에서 "자율주행차 검사관"처럼 — 시험 코스(테스트 데이터)에서만 잘 달리는 게 아니라, 실제 도로(프로덕션)에서도 공정하고 안전하게 달리는지 확인한다.

## Ⅴ. 기대효과 및 결론

### 5.1 기대효과

| 효과 | 내용 |
|:---|:---|
| 알고리즘 공정성 확보 | 편향성 사전 탐지·제거 |
| 의사결정 신뢰성 향상 | 성능 지표 기반 투명한 검증 |
| 법적 리스크 해소 | EU AI Act, 인공지능법 선제 대응 |
| 운영 안정성 | MLOps 체계로 미검증 모델 배포 방지 |

### 5.2 결론

AI 모델 성능 감리는 데이터·성능·공정성·배포의 4개 축에서 AI 시스템의 신뢰성을 종합 검증하는 고도화된 감리 영역이다. 기술사는 Precision/Recall/F1-Score의 단순 수치를 넘어, 서브그룹별 성능 편차와 설명 가능성(XAI)까지 감리 범위를 확장해야 한다.

📢 **섹션 요약 비유**: AI 모델 감리를 완료한 시스템은 "편향 없이 공정하게 채점하는 AI 심사위원"처럼 — 성별이나 출신지에 상관없이 실력(성능 지표)에만 근거해 공정한 결과를 내린다.

### 👶 어린이를 위한 3줄 비유 설명

- AI가 100번 문제를 맞혔는데, 특정 친구 이름의 문제만 다 틀렸다면 편향(Bias)이 있는 거야.
- F1-Score는 "얼마나 잘 맞히는지"와 "얼마나 놓치지 않는지"를 동시에 재는 점수야.
- 감리원은 AI가 모든 친구를 공평하게 판단하는지, 성적이 진짜인지 확인하는 선생님이야!
