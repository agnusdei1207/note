## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 데이터 거버넌스(Data Governance) 감리는 메타데이터 표준 준수, 공공데이터 개방 규격 적합성, 민감 데이터 비식별화의 세 축을 중심으로 데이터 자산의 신뢰성과 활용성을 검증한다.
> 2. **가치**: 표준화된 메타데이터(Metadata)는 데이터의 검색 가능성·재사용성·상호운용성을 보장하며, 비식별화는 개인정보보호와 데이터 활용의 균형점을 제공한다.
> 3. **판단 포인트**: 메타데이터 등록률, 공공데이터 개방 포맷(JSON, CSV, XML) 준수율, 민감 데이터 마스킹(Masking) 적용 여부가 핵심 KPI (Key Performance Indicator)다.

---

## Ⅰ. 개요 및 필요성

### 1.1 데이터 거버넌스 감리의 배경

공공기관과 기업에서 데이터 자산의 규모가 폭증하면서, 체계적인 관리 없이는 중복·불일치·불완전 데이터가 누적된다. 데이터 거버넌스(Data Governance)는 데이터의 생산·저장·유통·폐기 전 생명주기에 걸쳐 품질·보안·준법을 보장하는 관리 체계다.

「공공데이터의 제공 및 이용 활성화에 관한 법률」(공공데이터법)은 공공기관이 메타데이터 표준을 준수하고, 민감 정보를 적절히 비식별화한 후 데이터를 개방하도록 의무화한다.

### 1.2 감리 필요성

| 필요 이유 | 내용 |
|:---|:---|
| 데이터 품질 보장 | 표준 없이 등록된 데이터는 재사용·통합 불가 |
| 법적 의무 이행 | 공공데이터법·개인정보보호법 준수 여부 확인 |
| 상호운용성 확보 | 기관 간 데이터 연계 시 표준 포맷 필요 |
| 민감 데이터 보호 | 비식별화 미이행 시 개인정보 유출 위험 |

📢 **섹션 요약 비유**: 데이터 거버넌스 감리는 "도서관의 책 분류 시스템"을 점검하는 것 — 모든 책에 정확한 분류 번호(메타데이터)가 붙어야 찾을 수 있고, 개인 일기(민감 데이터)는 잠금 칸에 보관해야 한다.

## Ⅱ. 아키텍처 및 핵심 원리

### 2.1 데이터 거버넌스 감리 체계도

```
  ┌─────────────────────────────────────────────────┐
  │       데이터 거버넌스 감리 영역 (Audit Scope)     │
  ├──────────────┬──────────────┬───────────────────┤
  │  메타데이터  │  공공데이터  │  민감 데이터       │
  │  표준 감리   │  개방 규격   │  비식별화 감리     │
  ├──────────────┼──────────────┼───────────────────┤
  │ 항목명 표준  │ JSON/CSV 포맷│ 가명처리           │
  │ 코드 체계    │ 개방 포털 등록│ 익명처리           │
  │ 데이터 사전  │ 갱신 주기    │ k-익명성           │
  │ 품질 지표    │ API 제공 여부│ l-다양성           │
  └──────────────┴──────────────┴───────────────────┘
```

### 2.2 메타데이터 표준 준수 체계

```
  데이터 생성
      │
  ┌───▼─────────────────────────────────────────┐
  │        메타데이터 등록 (Metadata Registry)    │
  │  ┌─────────┬────────────┬────────────────┐  │
  │  │데이터명  │ 생성기관    │ 갱신일자        │  │
  │  │데이터유형│ 항목정의    │ 공개여부        │  │
  │  │데이터형식│ 주기        │ 민감정보여부    │  │
  │  └─────────┴────────────┴────────────────┘  │
  └───┬─────────────────────────────────────────┘
      │
  ┌───▼──────────────────────────────────────────┐
  │   표준 적합성 검증 엔진 (Validation Engine)   │
  │   → 필수 항목 누락 여부 자동 탐지              │
  │   → 코드 체계 불일치 자동 탐지                │
  └───┬──────────────────────────────────────────┘
      │
  ┌───▼──────────────────────────────────────────┐
  │   공공데이터 포털 개방 (data.go.kr 등)        │
  └──────────────────────────────────────────────┘
```

### 2.3 비식별화 기법 비교

| 기법 | 방법 | 적용 사례 | 재식별 위험 |
|:---|:---|:---|:---|
| 가명처리 | 개인 식별자를 가명으로 대체 | 연구·통계 목적 | 중간 |
| 익명처리 | 식별 가능성 완전 제거 | 공개 데이터 배포 | 낮음 |
| 마스킹(Masking) | 일부 자리 숨김 (예: 주민번호 뒤 7자리) | 업무 화면 표시 | 낮음 |
| 일반화 | 정확한 값 대신 범위 표현 (예: 나이→연령대) | 통계 데이터 | 낮음 |
| 데이터 스와핑 | 레코드 간 값 교환 | 분석 데이터 | 중간 |

📢 **섹션 요약 비유**: 비식별화는 "편지 봉투를 열어도 발신인 주소만 지워지게 하는 것" — 내용(데이터)은 활용할 수 있지만, 누가 보냈는지(개인 정보)는 알 수 없다.

## Ⅲ. 비교 및 연결

### 3.1 메타데이터 등록 방식 비교

| 구분 | 수동 등록 | 자동 수집(Auto Crawling) |
|:---|:---|:---|
| 정확도 | 높음 (담당자 검토) | 중간 (자동화 오류 가능) |
| 적시성 | 낮음 (시간 지연) | 높음 (실시간 수집) |
| 유지보수 | 어려움 | 용이 |
| 적용 범위 | 중요 데이터셋 | 대규모 데이터 레이크 |

### 3.2 공공데이터 개방 규격 준수 체크포인트

| 점검 항목 | 기준 | 비고 |
|:---|:---|:---|
| 파일 포맷 | JSON, CSV, XML, ODS | 독점 포맷 금지 |
| API 표준 | REST API, OpenAPI 3.0 | W3C/IETF 표준 |
| 갱신 주기 | 데이터 특성에 맞게 명시 | 실시간·일·주·월·연 |
| 라이선스 | CCL (Creative Commons License) 명시 | 재활용 범위 표시 |

📢 **섹션 요약 비유**: 공공데이터 개방 규격 준수는 "전 세계 누구나 읽을 수 있는 언어(JSON/CSV)로 책을 써야 한다"는 원칙 — 특정 출판사 전용 형식으로만 읽히면 공공 도서관이 아닌 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4.1 데이터 거버넌스 감리 체크리스트

```
  ┌──────────────────────────────────────────────────┐
  │        데이터 거버넌스 감리 체크리스트              │
  ├──────────────────────────────┬───────────────────┤
  │ 점검 항목                    │ 기준              │
  ├──────────────────────────────┼───────────────────┤
  │ 메타데이터 등록률             │ 95% 이상          │
  │ 필수 항목 완성도              │ 100%              │
  │ 코드 표준 준수율              │ 100%              │
  │ 공개 데이터 포맷 적합성       │ JSON/CSV 필수     │
  │ 개인정보 비식별화 적용        │ 100% (민감 데이터)│
  │ 비식별화 효과성 검증          │ k≥5 이상          │
  │ 데이터 갱신 이행 여부         │ 계획 대비 90% 이상│
  └──────────────────────────────┴───────────────────┘
```

### 4.2 기술사 실무 판단 포인트

- **메타데이터 품질 측정**: 완전성(Completeness), 정확성(Accuracy), 일관성(Consistency) 3개 지표 동시 평가
- **비식별화 적정성 판단**: 단순 마스킹(Masking)만으로는 재식별 위험이 높으므로, 반드시 k-익명성(k-Anonymity) 수준 분석 포함
- **데이터 생명주기 추적**: 생성→저장→가공→공개→폐기 단계별 로그 및 이력 관리 체계 점검
- **데이터 오너십(Ownership)**: 각 데이터셋에 담당자(Data Steward) 지정 여부 확인

### 4.3 비식별화 k-익명성 예시

| 구분 | 원본 데이터 | k-익명성 적용 |
|:---|:---|:---|
| 나이 | 34세 | 30~39세 |
| 지역 | 서울 강남구 | 서울 강남구 (k≥5 충족 시 유지) |
| 직업 | 의사 | 의료직 |

📢 **섹션 요약 비유**: 기술사는 데이터 거버넌스 감리에서 "식품 안전관리원"처럼 — 식재료(데이터)의 원산지(출처), 유통기한(갱신 주기), 알레르기 표시(민감 정보 표시)를 모두 확인해야 한다.

## Ⅴ. 기대효과 및 결론

### 5.1 기대효과

| 효과 | 내용 |
|:---|:---|
| 데이터 활용성 향상 | 표준 메타데이터로 검색·통합·재사용 용이 |
| 법적 리스크 해소 | 개인정보보호법·공공데이터법 준수 |
| 데이터 신뢰성 확보 | 품질 지표 기반 데이터 신뢰도 향상 |
| 데이터 경제 기여 | 개방 데이터 기반 민간 혁신 서비스 창출 |

### 5.2 결론

데이터 거버넌스 감리는 **메타데이터 표준**, **공공데이터 개방 규격**, **민감 데이터 비식별화** 세 축을 통해 데이터 자산의 전체 생명주기를 감독한다. 기술사는 데이터의 수량보다 **품질과 신뢰성**에 집중하며, 비식별화의 적정성을 k-익명성 등 수치 기반으로 검증해야 한다.

📢 **섹션 요약 비유**: 데이터 거버넌스 감리를 완료한 시스템은 "모든 문서에 색인이 붙어 있고, 개인 기록은 금고 안에 있으며, 공개 문서는 누구나 읽을 수 있는 완벽한 도서관" — 데이터의 도서관이다.

### 👶 어린이를 위한 3줄 비유 설명

- 도서관에 책을 꽂을 때 제목·저자·분류번호(메타데이터)를 꼭 붙여야 나중에 찾을 수 있어.
- 일기장처럼 개인적인 책(민감 데이터)은 이름을 지우거나(비식별화) 잠금 칸에 보관해야 해.
- 감리원은 "모든 책에 라벨이 잘 붙어 있나요? 일기장은 안전하게 보관되나요?" 하고 도서관을 점검하는 사람이야!
