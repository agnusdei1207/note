## 핵심 인사이트 (3줄 요약)

> 1. **본질**: CSAP (Cloud Security Assurance Program, 클라우드 보안 인증 프로그램) 감리는 공공기관이 사용하는 클라우드 인프라가 국가 인증 보안 기준을 충족하는지 검증하는 과정이다.
> 2. **가치**: Zone 격리, 가상머신(VM, Virtual Machine) 암호화를 통해 멀티테넌시(Multi-tenancy) 환경에서의 데이터 분리와 무결성을 보장한다.
> 3. **판단 포인트**: CSAP 인증 획득 클라우드 서비스 사용 여부, 보안 존(Zone) 설계 적정성, 저장 데이터·전송 데이터 암호화 여부가 핵심 점검 항목이다.

---

## Ⅰ. 개요 및 필요성

### 1.1 CSAP 개요

CSAP (Cloud Security Assurance Program)는 한국인터넷진흥원(KISA, Korea Internet & Security Agency)이 운영하는 클라우드 서비스 보안 인증 제도다. 공공기관이 클라우드를 도입할 때 보안 적합성이 검증된 서비스만 사용하도록 하여 공공 데이터의 안전성을 보장한다.

「전자정부법」 제54조의2 및 「클라우드컴퓨팅 발전 및 이용자 보호에 관한 법률」에 따라 공공기관은 CSAP 인증 클라우드 서비스를 의무 사용해야 한다.

### 1.2 CSAP 인증 등급

| 등급 | 대상 정보 | 점검 항목 수 |
|:---|:---|:---|
| 표준 등급 | 비공개 공공 데이터 | 78개 |
| 간편 등급 | 공개 가능 공공 데이터 | 30개 (서브셋) |
| 고급 등급 | 국가 기밀·민감 데이터 | 추가 항목 포함 |

### 1.3 감리 필요성

- 공공기관의 클라우드 전환 가속화로 보안 공백 우려 증가
- 멀티테넌시 환경에서 타 기관 데이터와의 물리적·논리적 격리 보장 필요
- CSP (Cloud Service Provider, 클라우드 서비스 제공자)와 공공기관 간 보안 책임 분담 명확화

📢 **섹션 요약 비유**: CSAP는 클라우드 서비스의 "식품 안전 인증 마크(KS 인증)"처럼 — 이 마크가 없는 클라우드는 공공기관 급식(시스템)에 사용할 수 없다.

## Ⅱ. 아키텍처 및 핵심 원리

### 2.1 클라우드 보안 존(Zone) 설계

```
  ┌────────────────────────────────────────────────────┐
  │            클라우드 보안 존 (Security Zone) 설계    │
  ├────────────────────────────────────────────────────┤
  │                                                    │
  │  ┌──────────────┐    방화벽     ┌───────────────┐  │
  │  │  퍼블릭 존   │◄────────────►│  DMZ 존       │  │
  │  │ (Public Zone)│    (FW)      │(Demilitarized │  │
  │  │  외부 접속   │              │  Zone)        │  │
  │  └──────────────┘              └───────┬───────┘  │
  │                                        │           │
  │                                   IPS/IDS          │
  │                                        │           │
  │                               ┌────────▼──────┐   │
  │                               │  내부 업무 존  │   │
  │                               │(Internal Zone)│   │
  │                               └────────┬──────┘   │
  │                                        │           │
  │                               ┌────────▼──────┐   │
  │                               │  DB 보안 존   │   │
  │                               │ (DB Security  │   │
  │                               │    Zone)      │   │
  │                               │ 암호화 저장   │   │
  │                               └───────────────┘   │
  └────────────────────────────────────────────────────┘
```

### 2.2 가상머신(VM) 암호화 체계

| 암호화 대상 | 기술 | 표준 |
|:---|:---|:---|
| 저장 데이터 (Data at Rest) | AES-256 디스크 암호화 | NIST SP 800-111 |
| 전송 데이터 (Data in Transit) | TLS (Transport Layer Security) 1.2 이상 | RFC 8446 |
| 처리 중 데이터 (Data in Use) | TEE (Trusted Execution Environment) | Intel SGX |
| 스냅샷·백업 | 암호화 키 분리 관리 | HSM (Hardware Security Module) 활용 |

### 2.3 CSAP 점검 주요 통제 항목

```
  ┌──────────────────────────────────────────────┐
  │        CSAP 핵심 통제 영역 (78개 항목 중)     │
  ├──────────────┬───────────────────────────────┤
  │  물리 보안   │ 데이터센터 출입 통제, CCTV     │
  ├──────────────┼───────────────────────────────┤
  │  네트워크    │ Zone 격리, DDoS 방어, IPS/IDS  │
  ├──────────────┼───────────────────────────────┤
  │  암호화      │ 저장·전송 암호화, 키 관리       │
  ├──────────────┼───────────────────────────────┤
  │  접근 통제   │ MFA, 특권 계정 관리             │
  ├──────────────┼───────────────────────────────┤
  │  로그·감사   │ 감사 로그 6개월 이상 보존       │
  └──────────────┴───────────────────────────────┘
```

📢 **섹션 요약 비유**: 보안 존 설계는 "은행의 금고 구조"처럼 — 일반 대기실(퍼블릭 존) → 창구(DMZ) → 업무 공간(내부 존) → 금고(DB 존)로 깊어질수록 엄격해진다.

## Ⅲ. 비교 및 연결

### 3.1 CSAP vs. 유사 클라우드 보안 인증 비교

| 구분 | CSAP (한국) | FedRAMP (미국) | ISO 27017 (국제) |
|:---|:---|:---|:---|
| 발급 기관 | KISA | FedCIRC | ISO/IEC |
| 대상 기관 | 공공기관 | 연방정부 기관 | 모든 조직 |
| 의무 여부 | 공공기관 의무 | 연방기관 의무 | 자발적 |
| 갱신 주기 | 3년 (중간 심사 포함) | 3년 | 3년 |
| 점검 항목 수 | 78개 (표준) | 325개 | 37개 추가 통제 |

### 3.2 클라우드 모델별 보안 책임 분담

| 모델 | CSP 책임 | 공공기관 책임 |
|:---|:---|:---|
| IaaS (Infrastructure as a Service) | 물리·네트워크 보안 | OS·미들웨어·데이터 |
| PaaS (Platform as a Service) | OS·런타임 포함 | 애플리케이션·데이터 |
| SaaS (Software as a Service) | 거의 전체 | 접근 권한 관리·데이터 분류 |

📢 **섹션 요약 비유**: 클라우드 보안 책임 분담은 "임대 아파트 계약"과 같아서 — 건물 구조(CSP)는 집주인 책임이지만, 집 안 자물쇠와 귀중품 관리(공공기관)는 세입자 책임이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4.1 클라우드 인프라 CSAP 감리 체크리스트

```
  ┌────────────────────────────────────────────────┐
  │      클라우드 CSAP 감리 체크리스트              │
  ├──────────────────────────────┬─────────────────┤
  │ 점검 항목                    │ 판정 기준        │
  ├──────────────────────────────┼─────────────────┤
  │ CSAP 인증 획득 CSP 사용      │ 필수 (Major)    │
  │ 보안 존 격리 설계            │ 4개 Zone 이상   │
  │ 저장 데이터 AES-256 암호화   │ 100% 적용       │
  │ 전송 TLS 1.2 이상 적용       │ 100% 적용       │
  │ MFA 적용 (특권 계정)         │ 필수 (Major)    │
  │ 감사 로그 보존               │ 6개월 이상      │
  │ VM 이미지 무결성 검증        │ Hash 서명 확인   │
  └──────────────────────────────┴─────────────────┘
```

### 4.2 기술사 실무 판단 포인트

- **CSAP 인증 유효기간 확인**: 계약 기간 내 인증 유효 여부 반드시 확인 (만료 전 갱신 계획 포함)
- **공동 책임 모델 문서화**: CSP와 공공기관 간 보안 책임 경계를 MOU 또는 SLA (Service Level Agreement)에 명시
- **존 격리 실효성 검증**: 논리적 격리(VLAN, VPC)가 물리적 격리 수준으로 동작하는지 네트워크 플로우 로그 분석
- **데이터 잔류 문제**: 클라우드 서비스 해지 시 데이터 완전 삭제 절차(Data Sanitization) 계획 점검

### 4.3 가상머신 암호화 키 관리

| 키 유형 | 관리 방법 | 위험 시나리오 |
|:---|:---|:---|
| DEK (Data Encryption Key) | CSP 자동 관리 | CSP 내부 유출 위험 |
| KEK (Key Encryption Key) | 공공기관 자체 관리 권고 | 공공기관 분실 시 복구 불가 |
| 마스터 키 | HSM (Hardware Security Module) | 물리 보안 필요 |

📢 **섹션 요약 비유**: 가상머신 암호화 키 관리는 "금고 열쇠를 금고 안에 두지 않는 것"처럼 — DEK는 금고 안에 있어도 되지만, 마스터 KEK는 금고 밖 안전한 곳에 따로 보관해야 한다.

## Ⅴ. 기대효과 및 결론

### 5.1 기대효과

| 효과 | 내용 |
|:---|:---|
| 공공 데이터 안전성 확보 | CSAP 기준 Zone 격리·암호화 의무화 |
| 컴플라이언스 (Compliance) | 전자정부법·클라우드법 동시 준수 |
| 사고 대응 능력 향상 | 감사 로그 기반 침해 사고 분석 가능 |
| 벤더 신뢰성 | CSAP 인증 CSP만 사용해 신뢰 확보 |

### 5.2 결론

클라우드 인프라 CSAP 감리는 단순한 체크리스트 확인을 넘어, 공공 데이터의 **Zone 기반 격리**, **전주기 암호화**, **감사 추적 가능성**을 종합 점검하는 고난이도 감리다. 기술사는 CSP와 공공기관 간 공동 책임 모델을 명확히 이해하고, CSAP 인증 유효성과 실제 구현 간의 간극을 철저히 검증해야 한다.

📢 **섹션 요약 비유**: CSAP 감리를 완료한 클라우드는 "소방 안전 검사를 통과한 건물"처럼 — 언제 화재가 나도 스프링클러(보안 통제)가 작동하고, 입주자(데이터)가 안전하게 대피할 수 있다.

### 👶 어린이를 위한 3줄 비유 설명

- 클라우드 창고(클라우드 서비스)를 쓰려면 나라에서 안전하다고 허락한 창고(CSAP 인증)만 써야 해.
- 창고 안은 구역(Zone)으로 나뉘어서 모르는 사람이 마음대로 들어올 수 없어.
- 창고 안 물건(데이터)도 자물쇠(암호화)로 잠겨 있어서 열쇠 없이는 열 수 없어!
