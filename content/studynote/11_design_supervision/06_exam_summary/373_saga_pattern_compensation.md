## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 사가 (Saga) 패턴은 MSA (Microservice Architecture) 환경의 분산 트랜잭션을 일련의 로컬 트랜잭션과 보상 트랜잭션 (Compensating Transaction)으로 대체하여 최종 일관성 (Eventual Consistency)을 달성한다.
> 2. **가치**: 2PC (Two-Phase Commit)의 동기 블로킹 없이 서비스 간 약한 결합을 유지하면서 비즈니스 트랜잭션의 원자성을 보장한다.
> 3. **판단 포인트**: 보상 트랜잭션이 비즈니스적으로 "취소 가능"한지 먼저 분석하고, 멱등성 (Idempotency) 설계가 반드시 수반되어야 한다.

## Ⅰ. 개요 및 필요성

MSA 환경에서 각 서비스가 독립 DB를 가지면, 서비스 A와 B를 하나의 ACID 트랜잭션으로 묶을 수 없다. 2PC는 코디네이터 장애 시 전체 잠금·동기 블로킹으로 성능 저하가 크다.

**사가의 두 구현 방식:**
- **코레오그래피 (Choreography)**: 이벤트로 자율 트리거
- **오케스트레이션 (Orchestration)**: 중앙 오케스트레이터 명령

📢 **섹션 요약 비유**: 사가는 팀 릴레이 달리기다. 누군가 넘어지면 앞선 주자들이 역방향으로 달려 원래 자리로 돌아간다.

## Ⅱ. 아키텍처 및 핵심 원리

```text
┌─────────────────────────────────────────────────────────────────┐
│            Saga 오케스트레이션 - 주문 처리 예시                 │
├─────────────────────────────────────────────────────────────────┤
│  1. 주문 생성 (Order Service)                                   │
│       ▼                                                         │
│  2. 재고 예약 (Inventory Service) → 성공                        │
│       ▼                                                         │
│  3. 결제 처리 (Payment Service) → 실패!                         │
│       ▼ 보상 트랜잭션                                          │
│  3'. 재고 예약 취소 (Inventory Service)                         │
│  2'. 주문 취소 (Order Service)                                  │
└─────────────────────────────────────────────────────────────────┘
```

| 구분 | 코레오그래피 | 오케스트레이션 |
|:---|:---|:---|
| 조율 방식 | 이벤트 기반 자율 | 중앙 오케스트레이터 |
| 가시성 | 낮음 (분산 흐름) | 높음 (중앙 상태) |
| 결합도 | 느슨 | 오케스트레이터 중심 |
| 적합 규모 | 소규모 MSA | 복잡한 워크플로우 |

📢 **섹션 요약 비유**: 코레오그래피는 악보 없는 재즈, 오케스트레이션은 지휘자 있는 클래식이다.

## Ⅲ. 비교 및 연결

| 항목 | Saga | 2PC (Two-Phase Commit) |
|:---|:---|:---|
| 일관성 | 최종 일관성 | 강한 일관성 |
| 결합도 | 느슨 | 강함 (XA 프로토콜) |
| 성능 | 비동기 고성능 | 동기 블로킹 |
| 장애 대응 | 보상 트랜잭션 | 잠금 해제 대기 |

📢 **섹션 요약 비유**: 2PC는 모든 팀원이 동시에 손을 들어야 회의가 진행되는 방식이다.

## Ⅳ. 실무 적용 및 기술사 판단

**보상 트랜잭션 설계 원칙:**
1. **멱등성**: 보상 트랜잭션 중복 실행 시 동일 결과 보장
2. **취소 불가 작업 분리**: 이메일 발송 등은 보상 로직 전에 배치
3. **Outbox 패턴 연계**: 로컬 TX와 이벤트 발행을 원자적 처리

**구현 프레임워크:**
- Axon Framework (Java)
- Apache Camel Saga
- Outbox 패턴 + Debezium CDC (Change Data Capture)

📢 **섹션 요약 비유**: Outbox 패턴은 편지를 바로 보내는 대신 우체통에 넣는 것이다. 우체부가 반드시 배달한다.

## Ⅴ. 기대효과 및 결론

사가 패턴은 MSA에서 2PC 없이 비즈니스 트랜잭션 원자성을 구현하는 핵심 패턴이다. 기술사 시험에서는 코레오그래피/오케스트레이션 비교, 보상 트랜잭션 멱등성, Outbox 패턴 연계를 명확히 서술해야 한다.

📢 **섹션 요약 비유**: 사가는 분산된 팀의 명확한 "되돌리기 프로토콜"이다.

- 실수를 되돌리는 방법(보상)을 미리 정해놔야 혼란이 없어요.
