## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 프로토타입 (Prototype) 패턴은 기존 객체를 복제(Clone)하여 새 인스턴스를 만드는 생성 패턴으로, 객체 초기화 비용이 클 때 성능을 최적화한다.
> 2. **가치**: DB 조회·복잡한 계산으로 초기화되는 객체를 매번 새로 만들지 않고, 기존 객체를 복제하여 수정함으로써 비용을 절감한다.
> 3. **판단 포인트**: 얕은 복사 (Shallow Copy)와 깊은 복사 (Deep Copy)의 차이를 명확히 이해하고, 참조 타입 필드가 있는 경우 반드시 깊은 복사를 구현해야 한다.

## Ⅰ. 개요 및 필요성

복잡한 초기화 과정이 필요한 객체(예: DB에서 대량 데이터를 로드하는 객체)를 매번 `new`로 생성하면 성능이 저하된다. 프로토타입 패턴은 이런 객체를 미리 하나 만들어두고, 필요 시 복제하여 사용한다.

**Java에서의 구현:**
- `Cloneable` 인터페이스 구현 + `clone()` 메서드 오버라이드
- 복사 생성자 (Copy Constructor) 사용
- 직렬화 (Serialization) + 역직렬화 방식

📢 **섹션 요약 비유**: 프로토타입은 문서 양식(Template)을 복사해서 내용을 채우는 것이다. 매번 새 양식을 그리는 것보다 훨씬 빠르다.

## Ⅱ. 아키텍처 및 핵심 원리

```text
┌─────────────────────────────────────────────────────────────────┐
│              Prototype 패턴 구조                                │
├─────────────────────────────────────────────────────────────────┤
│  <<interface>> Prototype                                        │
│  + clone(): Prototype                                           │
│         △                                                       │
│  ConcretePrototype1    ConcretePrototype2                        │
│  + clone()             + clone()                                │
│    → this.copy()         → this.copy()                          │
│                                                                 │
│  Client:                                                        │
│  prototype1 = registry.get("type1")  // 원본 조회               │
│  newObj = prototype1.clone()          // 복제                   │
│  newObj.setName("custom")             // 수정                   │
└─────────────────────────────────────────────────────────────────┘
```

| 복사 유형 | 설명 | 주의사항 |
|:---|:---|:---|
| 얕은 복사 (Shallow Copy) | 기본 타입은 복사, 참조 타입은 참조 공유 | 참조 객체 변경 시 원본 영향 |
| 깊은 복사 (Deep Copy) | 모든 중첩 객체까지 재귀적 복사 | 구현 복잡, 성능 비용 |

📢 **섹션 요약 비유**: 얕은 복사는 집 설계도를 복사했지만 같은 가구를 공유하는 것, 깊은 복사는 가구까지 모두 새로 만드는 것이다.

## Ⅲ. 비교 및 연결

| 항목 | 프로토타입 | 팩토리 메서드 | new 연산자 |
|:---|:---|:---|:---|
| 생성 방식 | 기존 객체 복제 | 서브클래스에 위임 | 직접 생성 |
| 초기화 비용 | 최초 1회만 | 매번 발생 | 매번 발생 |
| 유연성 | 런타임 원형 교체 가능 | 컴파일 타임 결정 | 없음 |

📢 **섹션 요약 비유**: 팩토리는 주문 시마다 요리, 프로토타입은 요리된 음식을 그대로 복사(포장)하는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

**게임 개발에서의 활용:**
몬스터 객체를 DB에서 로드하여 프로토타입으로 등록하고, 게임 진행 중 몬스터 생성 시 복제하여 사용

**프로토타입 레지스트리 (Registry):**
복제할 원형 객체들을 Map으로 관리, `registry.get("key").clone()` 형태로 사용

**주의사항:**
- 순환 참조가 있는 객체의 깊은 복사 처리
- `clone()` 메서드의 가시성 (protected → public)
- 불변 객체와의 결합 (불변 객체는 복제가 불필요)

📢 **섹션 요약 비유**: 프로토타입 레지스트리는 붕어빵 틀 창고다. 틀(원형)을 꺼내 반죽(데이터)만 바꿔 넣으면 된다.

## Ⅴ. 기대효과 및 결론

프로토타입 패턴은 초기화 비용이 큰 객체의 생성을 최적화하는 실용적인 패턴이다. 기술사 시험에서는 얕은 복사 vs 깊은 복사의 차이, Cloneable 인터페이스, 프로토타입 레지스트리 패턴을 명확히 서술해야 한다.

📢 **섹션 요약 비유**: 프로토타입은 마스터 열쇠를 만들어두고 필요한 만큼 복사하는 것이다.

### 👶 어린이를 위한 3줄 비유 설명

- 그림을 그릴 때 매번 처음부터 그리지 않고 이미 그린 그림을 복사하는 것이에요.
- 복사한 그림에 원하는 색을 다시 칠해도 원본은 변하지 않아요 (깊은 복사).
- 그림 복사가 처음부터 그리는 것보다 훨씬 빠르고 쉬워요.
