## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 스트랭글러 피그 (Strangler Fig) 패턴은 레거시를 한 번에 교체하는 빅뱅 방식 대신, 새 기능을 점진적으로 신규 시스템에 구축하면서 레거시를 서서히 대체하는 마이그레이션 전략이다.
> 2. **가치**: 비즈니스 연속성을 유지하면서 수십 년 된 레거시도 운영 중단 없이 MSA (Microservice Architecture)로 전환할 수 있다.
> 3. **판단 포인트**: 교체 경계 정의와 레거시·신규 간 데이터 동기화 전략이 성공의 핵심이다.

## Ⅰ. 개요 및 필요성

레거시 현대화의 최대 위험은 "빅뱅 교체 (Big Bang Replacement)"다. 수년 개발 후 한 번에 전환하면 누락 기능·버그·사용자 저항으로 실패 확률이 높다.

스트랭글러 피그는 열대 무화과나무가 숙주 나무를 서서히 감싸는 자연 현상에서 영감받은 패턴 (Martin Fowler, 2004):
1. **변환 (Transform)**: 레거시 옆에 신규 시스템 구축
2. **공존 (Coexist)**: 레거시·신규 동시 운영, 신규 기능은 신규 시스템에만
3. **제거 (Eliminate)**: 모든 기능 이전 완료 후 레거시 종료

📢 **섹션 요약 비유**: 낡은 건물을 허물지 않고 옆에 새 건물을 지어가면서 방 하나씩 이사하는 방식이다.

## Ⅱ. 아키텍처 및 핵심 원리

```text
┌─────────────────────────────────────────────────────────────────┐
│           Strangler Fig 마이그레이션 아키텍처 (공존 단계)       │
├─────────────────────────────────────────────────────────────────┤
│              [API Gateway / Strangler Facade]                   │
│                        │                                        │
│           ┌────────────┴────────────┐                           │
│           ▼                         ▼                           │
│  ┌─────────────────┐    ┌─────────────────────┐                 │
│  │  레거시 시스템   │    │  신규 마이크로서비스  │                 │
│  │  (기존 기능)    │    │  (이전된 기능)        │                 │
│  └─────────────────┘    └─────────────────────┘                 │
│           │                         │                           │
│           └──────────┬──────────────┘                           │
│                      ▼                                          │
│               데이터 동기화 계층                                 │
└─────────────────────────────────────────────────────────────────┘
```

| 단계 | 활동 | 리스크 관리 |
|:---|:---|:---|
| 변환 | 신규 서비스 구축, 프록시 설치 | 레거시 영향 없음 |
| 공존 | 프록시 트래픽 분기, 이중 기록 | 롤백 가능 상태 유지 |
| 제거 | 레거시 라우팅 제거, 종료 | 점진적 트래픽 전환 (Canary) |

📢 **섹션 요약 비유**: 도로 공사처럼 기존 도로를 닫지 않고 새 도로를 건설하고 차량을 서서히 이전한다.

## Ⅲ. 비교 및 연결

| 마이그레이션 전략 | 위험도 | 비용 | 특징 |
|:---|:---|:---|:---|
| 빅뱅 교체 | 매우 높음 | 높음 | 한 번에 전환 |
| 스트랭글러 피그 | 낮음 | 중간 | 점진적 교체 |
| 리호스팅 (Lift & Shift) | 낮음 | 낮음 | 인프라만 이전 |
| 리팩터링 | 낮음 | 중간 | 코드 개선 |

📢 **섹션 요약 비유**: 빅뱅은 심장 이식 수술, 스트랭글러 피그는 혈관 하나씩 교체하는 치료법이다.

## Ⅳ. 실무 적용 및 기술사 판단

**성공 요인:**
1. **Strangler Facade (API 게이트웨이)**: 트래픽을 레거시/신규로 분기
2. **피처 토글 (Feature Toggle)**: 런타임에 라우팅 제어
3. **데이터 동기화**: 이중 기록 또는 CDC (Change Data Capture)
4. **ACL**: 레거시 데이터 구조 오염 방지

**감리 포인트:**
- 각 단계별 롤백 계획 수립 여부
- 이중 기록 구간의 정합성 검증 방법
- 레거시 종료 기준 및 일정 명세

📢 **섹션 요약 비유**: 핵심은 "언제든 레거시로 돌아갈 수 있는 비상 탈출구"를 항상 열어두는 것이다.

## Ⅴ. 기대효과 및 결론

스트랭글러 피그 패턴은 비즈니스 연속성 유지 속에 레거시를 현대화하는 가장 현실적인 전략이다. 기술사 시험에서는 3단계(변환/공존/제거), Strangler Facade, 데이터 동기화 전략을 구체적으로 서술해야 한다.

📢 **섹션 요약 비유**: 레거시라는 낡은 옷을 양말→바지→상의 순서로 교체하면 추위를 타지 않는다.

### 📌 관련 개념 맵

| 관계 | 개념 | 설명 |
|:---|:---|:---|
| 연관 패턴 | ACL (Anti-Corruption Layer) | 레거시 오염 방지 |
| 연관 패턴 | 피처 토글 | 트래픽 분기 제어 |
| 연관 전략 | 카나리 배포 | 점진적 트래픽 전환 |
| 상위 개념 | 레거시 현대화 | 패턴의 목표 |

### 👶 어린이를 위한 3줄 비유 설명

- 낡은 레고 성을 부수지 않고 옆에 새 성을 지어가요.
- 새 성의 방이 완성될 때마다 낡은 성의 방을 비워요.
- 모든 방이 이사 완료되면 낡은 성을 치워요.
