# 11. 3단계 감리/2단계 감리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 3단계 감리는 要求정의, 설계, 종료的全 단계에 걸쳐 감리를 실시하는 방식이고, 2단계 감리는 要求정의 감리를省略하고 설계와 종료에만 적용하는 방식이다. 2단계 감리는 일정 요건이 충족된 사업에 한하여 적용된다.
> 2. **가치**: 3단계 감리는 全과정에 대한 체계적 관리가 가능하지만 시간과 비용이 많이 들고, 2단계 감리는 비용 대効果比率는 높지만 要求정의 단계의 문제 탐지 기능이欠落하는Risk가 있다.
> 3. **융합**: 감리 단계 수 선택은 프로젝트의規模, 복잡도, 과거 이력, 그리고 法령 요건 등을 종합적으로考慮하여 결정해야 하며, 한 번 결정된 단계를中途에서 변경하는 것은 避すべき이다.

---

### Ⅰ. 개요 및 필요성 (Context & Necessity)

정보시스템 감리의 단계 수를 결정하는 것은 프로젝트 초기에 내려야 할 중요한 전략적決定이다. 3단계 감리는 要求정의(Requirements), 설계(Design), 종료(Final) 全단계에 감리를実施하고, 2단계 감리는 要求정의 단계를省略하고 설계와 종료에만 적용한다.

3단계 감리와 2단계 감리의 구분은 전자정부법 시행령 및行政기관등정보화사업관리규정에 상세히 규정되어 있다. 法律의 관점에서, 의무감리 대상 사업은原则上 3단계 감리를 적용하되, 특정 요건이 충족되면 2단계 감리 적용이 가능하다. 이러한 法律적 근거를 명확히 이해하는 것은 감리 기획 단계에서 필수적이다.

2단계 감리가 존재하는 이유는 完全한 3단계 감리가 항상 실용적이지 않기 때문이다. 만약 모든 사업에 3단계 감리를 강제한다면, 이미 要求정의가 충분히 검증된 사업(예: 이전에 similar한 사업을 성공적으로 수행한 사업자의 사업)까지 불필요하게 要求정의 감리를 반복하여야 하고, 이는 資源의 낭비로 이어진다. 따라서 法律은一定の 요건하에 2단계 감리를 허용함으로써, 감리의 효율성과 적용성을높이고 있다.

다음 다이어그램은 3단계 감리와 2단계 감리의 구조적 차이를 보여준다.

```text
┌─────────────────────────────────────────────────────────┐
│           [3단계 감리 vs 2단계 감리 구조 비교]          │
│                                                         │
│  [3단계 감리]                                           │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐            │
│  │ 요구정의 │───▶│  설계   │───▶│  종료   │            │
│  │  감리   │    │  감리   │    │  감리   │            │
│  └────┬────┘    └────┬────┘    └────┬────┘            │
│       │              │              │                   │
│       ▼              ▼              ▼                   │
│  [RD검토]        [설계검토]      [최종검토]              │
│  - 요구사항      - 아키텍처       - 통합/인수           │
│  - 범위/과업    - 상세설계      - 성능/보안           │
│  - comp/법령    - 시큐어코딩    - 데이터이행          │
│                                                         │
│  ───────────────────────────────────────────────       │
│                                                         │
│  [2단계 감리]                                           │
│  ┌─────────┐    ┌─────────┐    (省略)                  │
│  │ 要求정의 │    │  설계   │───▶│  종료   │            │
│  │  (省略) │    │  감리   │    │  감리   │            │
│  └────┬────┘    └────┬────┘    └────┬────┘            │
│       │              │              │                   │
│       ▼              ▼              ▼                   │
│  [사전검증]      [설계검토]      [최종검토]              │
│  - 선행검증      - 아키텍처       - 통합/인수           │
│    或者省略       - 상세설계      - 성능/보안           │
│                 - 시큐어코딩    - 데이터이행          │
└─────────────────────────────────────────────────────────┘
```

이 도식의 핵심은 2단계 감리에서 要求정의 단계가 完全히 사라지는 것이 아니라, "사전검증"이라는 형태로簡略화되거나, 또는 선행된 검증으로 갈음되는다는 점이다. 즉, 要求정의 자체가省略되는 것이 아니라, 要求정의 감리만省略되는 것이라는 점을 명확히 구분해야 한다.

📢 **섹션 요약 비유**: 3단계와 2단계 감리의 관계는 **'대학 입학 之，前期 과정'**과 같습니다. 전체 Freshman은 1학년前期課程(요구정의)을 거쳐 2학년(설계), 3학년(종료)으로 진행하는데, 편입생(2단계 감리 해당 사업)은 이미 1학년 과정을 이수했으므로, 2학년(설계)부터 다시 시작하여 3학년(종료)까지 진행하면 됩니다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

3단계 감리와 2단계 감리의 적용 요건, 차이점, 그리고選擇 기준을 表로 정리하면 다음과 같다.

**[3단계 vs 2단계 감리 비교표]**

| 구분 | 3단계 감리 | 2단계 감리 |
|:---|:---|:---|
| **법적 근거** | 전자정부법 제57조, 관리규정 제44조 | 관리규정 제44조 3항 |
| **적용 요건** | 의무감리 대상 사업 (전 사업자) | 要求정의 단계가 사전에 충분히 검증된 특정 사업 |
| **省略 조건** | 없음 | (1) Equivalent 시스템 구축 경험, (2) 要求정의 사전검증 완료, (3) 사업複雑도 낮은 경우 등 |
| **감리 비용** | 높음 (3회 감리) | 중간 (2회 감리) |
| **要求정의 관리** | 직접 관리 | 선행 검증으로 갈음 |
| **주요 장점** | 全단계 관리, 체계적 | 비용 절감, 효율적 |
| **주요缺点** | 비용 및 일정 부담 | 要求정의 문제 놓칠 Risk |

2단계 감리 적용 가능성을 判断하는 의사결정 트리를 살펴보면 다음과 같다.

```text
[2단계 감리 적용 判断 트리]

  [시작]
      │
      ├── 要求정의 단계 선행 검증 완료 ?
      │    ├── Yes ──► 2단계 감리 검토
      │    │
      │    └── No ──► 3단계 감리 검토
      │
      └── 사업複雑도 Low ?
           ├── Yes ──► 2단계 감리 검토 (+ 선행 검증 필요)
           │
           └── No ──► 3단계 감리 검토
```

이 의사결정 트리의 핵심은 2단계 감리가 要求정의 단계의 중요성을 完全히 무시하는 것이 아니라, "다른 방식으로 확인"하는 것이다. 要求정의 단계의 선행 검증이 충분하다면, 그것을 要求정의 감리로갈음할 수 있다는 것이다.

📢 **섹션 요약 비유**: 2단계 감리 선택 기준은 **'보험 경력 할인'**과 같습니다. 사고 없이 오래 운전한驱动程序은 보험 할인 대상이듯이, 이전에 Similar한 사업으로 성공적 경험을 보유한 사업자도 要求정의 감리를 部分 경감 받을 수 있습니다.

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

3단계와 2단계 감리의 실무적 차이와 각 유형의적정 상황을 分析해보자.

**[3단계 vs 2단계 감리 적정 상황 분석]**

| 사업 상황 | 권장 유형 | 선택 근거 |
|:---|:---|:---|
| **초대형 신규 사업 (첫 진행)** | 3단계 | 要求정의 단계의 문제이후 전체에 미칠 영향이 매우 큼 |
| **대규모事业, 要求정의 선행 완료** | 2단계 | 이미 要求정의가 검증되었으므로 중복 방지 |
| **복잡한 아키텍처 적용 사업** | 3단계 | 설계 단계에서深度 있는 검토 필수 |
| **표준화된 유지보수 사업** | 2단계 | 要求정의 단계가 단순하므로省略 가능 |
| **보안 민감 정보 처리 사업** | 3단계 | 要求정의 단계에서 보안 요구사항 포함 여부 반드시 확인 |
| **애자일 방법론 적용 사업** | 3단계 or Adaptive | 애자일의 반복적，要求과정이 계속 변하므로 각 iterati에서 검증 |

감리 단계 수에 따른 비용 및 리스크를 비교한 매트릭스이다.

```text
[감리 단계 수별 비용-리스크 分析]

  비용
   │
높음 │  ● 3단계 감리
     │     (고비용-저리스크)
     │
중간 │      ● 2단계 감리
     │        (중비용-중리스크)
     │
낮음 │           ● 감리省略 (위험)
     │             (저비용-고리스크)
     │
     └──────────────────────────────► 리스크
```

이 分析의 핵심은 비용과 리스크가 일반적으로 비례한다는 점이다. 3단계 감리는 비용이 높지만 리스크를最小限화하고, 감리를省略하면 비용은 줄지만 리스크가 크게 增加한다. 2단계 감리는その間の 균형점을 제공한다.

📢 **섹션 요약 비유**: 감리 단계 수 선택은 **'집 짓기 시공 Monitoring'**과 같습니다. 일반 가정집은 현장관리자(2단계 감리)만으로 충분하지만, skyscrapers(초대형 사업)는 각 단계마다 공공进行检查기관(3단계 감리)이 직접 확인해야 안전이担保됩니다.

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

실제 감리 현장에서 3단계와 2단계 감리를 적용할 때의 판단 사례를 살펴보자.

**1. 2단계 감리 전환 판단: "기존 3단계 감리 대상 사업을 2단계로 전환할 수 있는가?"**
*   **상황**: 要求정의 감리가 완료된 후, 설계 단계 진입을 앞두고 있다. 사업자가 要求정의 감리에서 지적된 사항이 모두 조치되었으므로, 설계 감리부터 진행하자는 제안을 했다.
*   **기술사적 판단**: 이것은 이미 3단계 감리 대상으로 사업이 진행되고 있으므로, 要求정의 감리를省略하고 2단계로 전환하는 것은 一般적으로 허용되지 않는다.一旦 출발한 감리 단계 수는途中で 변경할 수 없으며, 만약 전환이 필요하다면 새로운 감리 계획 수립과主管기관의 승인이 필요하다. 단, 要求정의 감리 지적 사항이 모두 조치되었는지의 여부는 설계 감리의 중요한 입력资料으로 활용되어야 한다.

**2. 2단계 감리 선행 검증 인정 판단: "외부 사전검증 결과를 要求정의 감리로 인정받을 수 있는가?"**
*   **상황**: 사업자가 이전에similar한 시스템을 성공적으로 구축한 이력이 있어, 그때 사전검증 결과를 이번 사업의 要求정의 감리 갈음 자료로 활용하고 싶다.
*   **기술사적 판단**: 이전 사업의 사전검증 결과가 이번 사업과 要求정의가实质적으로동일하고, 그 결과가 유효한 경우에 한하여 갈음 인정이 가능하다. 그러나 다음条件을 반드시 확인해야 한다. (1) 이전 사업과 이번 사업의 要求정의内容이 同等 이상인지, (2) 이전 검증 이후 사업 환경의 변화가 없는지, (3) 발주기관이 해당 갈음을승인하는지. 만약 这些条件이 불충분하다면, 要求정의 감리를 별도로実施해야 한다.

**3. 3단계 vs 2단계 최종 선택 판단: "우리 사업에 어느 유형이 적절한가?"**
*   **상황**: 신규 사업 기획 단계에서 감리 유형을 결정해야 한다. 사업자는 2단계 감리를 선호하지만, 발주기관은 3단계 감리를 요구하고 있다.
*   **기술사적 판단**: 다음要素를 综合적으로 고려하여 결정해야 한다. (1) 사업 규모: 50억 원 이상 대규모 사업은 3단계 권고, (2) 사업 복잡도: 신규 기술이나 아키텍처 적용 시 3단계 권고, (3) 사업자 경력: Equivalent 사업 경험이 풍부하면 2단계 고려, (4) 요구사항成熟도: 요구사항이 명학하고 안정적이라면 2단계 고려, (5) 예산 및 일정: 감리 비용과 일정 여유가 있다면 3단계가 안전. 최종적으로는 발주기관의 리스크 허용 수준과 사업의 전략적 중요도를 기준으로 결정해야 한다.

```text
[감리 유형 최종 选择 判斷 Matrix]

          │  要求정의     │ 要求정의
          │ 成熟도 High   │成熟도 Low
  ────────┼───────────────┼───────────────
  대규모/ │   3단계       │    3단계
  복잡事业 │  (안전 우선)  │   (强制)
  ────────┼───────────────┼───────────────
  소규모/ │   2단계       │    3단계
  단순사업 │  (효율 우선)  │   (관리 필요)
```

이 매트릭스의 핵심은 要求정의成熟도가 낮거나 사업이 대규모/복잡하면 반드시 3단계 감리를 선택해야 한다는 점이다. 리스크와 비용 사이에서 安全을 선택하는 것이 장기적으로 더 유리하다.

📢 **섹션 요약 비유**: 감리 유형 선택은 **'장거리 비행기 vs 단거리 비행기'**와 같습니다. 서울에서 부산(단거리/단순)이라면 비행기 대신 KTX(2단계 감리)로 충분하지만, 서울에서 뉴욕(장거리/복잡)이라면 반드시直行噴射機(3단계 감리)로 들어가야 합니다. 거리가 멀고 길이 위험할수록 더 많은检查가필요합니다.

### Ⅴ. 기대효과 및 결론 (Future & Standard)

3단계 및 2단계 감리의 효과적 활용을 통해 기대할 수 있는 효과는 다음과 같다.

| 감리 유형 | 기대 효과 | 측정 지표 |
|:---|:---|:---|
| **3단계 감리** | 全과정 체계적 관리, 리스크 최소화, 프로젝트 성공률 향상 | 프로젝트 최종 성공률 90% 이상 |
| **2단계 감리** | 비용 및 일정 효율화, 불필요한 감리 축소 | 감리 비용 30% 절감, 일정 20% 단축 |
| **유형 选择 적절** | 项目特性에 맞는 적정 리스크 관리 | 문제 조기 발견률 80% 이상 |

**미래 전망:**
미래에는 AI 기반 자동으로 要求정의의成熟도를 分析하고, 그 결과에 따라 감리 단계 수를 권고하는 시스템이 도입될 것으로예상된다. 또한 애자일 방법론의 표준화와 함께, "적응형 감리 단계(Adaptive Audit Phase)"라는 새로운 개념이 등장할 수 있다. 이는 프로젝트의特性과 진행 상황에 따라 단계 수를动态적으로 조정하는 방식이다. 그러나 이러한 발전에도 불구하고, 3단계 감리가_project의 기본骨架를담보하는標準 방식이라는점은 여전히유효할 것이다.

📢 **섹션 요약 비유**: 미래의 적응형 감리 단계는 **'atención 의료 시스템'**과 같습니다. 환자의 상태(사업 진행 상황)에 따라 검사 빈도와 범위를 조절하듯이, 미래에는 프로젝트의 실시간 상태를 分析하여 최적의 감리 단계를 自动 추천하는 시스템이 등장할 것입니다.
