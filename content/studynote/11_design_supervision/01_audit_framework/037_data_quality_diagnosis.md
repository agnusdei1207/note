> **핵심 인사이트**
> 1. 데이터 품질 진단은 정확성·완전성·일관성·유효성·최신성·유일성의 6대 품질 차원을 기준으로 데이터의 결함을 체계적으로 측정하고, 데이터 기반 의사결정의 신뢰성을 보장하는 활동이다.
> 2. "쓰레기 입력, 쓰레기 출력(GIGO: Garbage In, Garbage Out)" — 아무리 정교한 AI·분석 모델도 품질이 낮은 데이터로 학습하면 잘못된 결과를 낸다. 데이터 품질은 디지털 전환의 기반이다.
> 3. 행정안전부 데이터 품질관리 지침(2021)과 ISO 8000 데이터 품질 표준이 공공·민간 분야의 데이터 품질 진단 기준이며, 데이터 감리의 핵심 점검 항목이다.

---

## I. 데이터 품질 6대 차원

```
6대 품질 차원 (ISO 8000 / 행안부 기준):

1. 정확성 (Accuracy):
   데이터가 실제 세계를 올바르게 반영하는가
   예: 고객 주소가 실제 주소와 일치하는가

2. 완전성 (Completeness):
   필수 데이터가 누락 없이 존재하는가
   예: 필수 필드 NULL 비율이 5% 이하인가

3. 일관성 (Consistency):
   동일 데이터가 여러 시스템에서 동일한가
   예: CRM과 ERP의 고객 정보가 일치하는가

4. 유효성 (Validity):
   데이터가 정해진 형식·범위·규칙을 따르는가
   예: 날짜 형식 YYYYMMDD, 나이 0-150

5. 최신성 (Timeliness):
   데이터가 최신 상태를 반영하는가
   예: 고객 주소 변경 후 즉시 반영되는가

6. 유일성 (Uniqueness):
   중복 데이터가 없는가
   예: 고객 ID 중복률이 0%인가
```

> 📢 **섹션 요약 비유**: 데이터는 식재료 — 정확성(신선한가), 완전성(빠진 재료 없는가), 일관성(레시피대로인가), 유효성(먹어도 되는가), 최신성(유통기한 지났나), 유일성(같은 재료 두 번 안 넣었나).

## II. 품질 진단 프로세스

```
데이터 품질 진단 5단계:

1단계: 대상 정의
  진단 범위, 핵심 데이터 선정
  비즈니스 임팩트 높은 데이터 우선

2단계: 품질 규칙 정의
  각 데이터 항목별 품질 기준 수립
  예: 생년월일 - YYYYMMDD, 1900~현재

3단계: 자동/수동 측정
  자동: SQL 쿼리, 데이터 프로파일링 도구
  수동: 샘플 검증, 현장 확인

4단계: 결함 분류 및 원인 분석
  데이터 입력 오류, 시스템 연동 오류
  프로세스 결함, 정의 불명확

5단계: 개선 및 모니터링
  정제(Cleansing), 표준화, 프로세스 개선
  대시보드로 지속 모니터링
```

> 📢 **섹션 요약 비유**: 데이터 품질 진단은 건강검진 — 단순 측정에서 그치지 않고 이상 지표를 원인 분석하고 치료(개선)까지 연결.

## III. 데이터 프로파일링 (Data Profiling)

```
데이터 프로파일링 주요 분석:

열 수준 분석:
  NULL 비율, 고유값 수, 최소/최대/평균
  빈도 분포, 이상값(Outlier)

패턴 분석:
  날짜 형식 일관성
  이메일 형식 준수율
  전화번호 패턴 분포

관계 분석:
  외래키 참조 무결성
  시스템 간 조인 성공률

SQL 예시:
  -- NULL 비율 계산
  SELECT column_name,
    COUNT(*) total,
    SUM(CASE WHEN col IS NULL THEN 1 ELSE 0 END) null_count,
    ROUND(SUM(CASE WHEN col IS NULL THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) null_pct
  FROM table_name
  GROUP BY column_name;
```

| 지표         | 기준          | 조치                    |
|-----------|-------------|------------------------|
| NULL 비율  | 필수 < 1%   | 수집 프로세스 개선       |
| 중복 비율  | 0%          | 중복 제거 배치 수행      |
| 형식 오류  | < 0.1%      | 입력 검증 강화           |
| 참조 무결성| 100%        | FK 제약 조건 추가        |

> 📢 **섹션 요약 비유**: 데이터 프로파일링은 혈액검사 — 표면만 보지 않고 항목별 수치로 정확히 어디가 문제인지 찾아낸다.

## IV. 행안부 데이터 품질관리 지침 핵심

```
대상 기관: 중앙행정기관, 지방자치단체, 공공기관

주요 요건:
  1. 데이터 품질관리 계획 수립 (매년)
  2. 핵심 데이터 품질 측정 (연 1회 이상)
  3. 품질 진단 결과 보고 (기관장 보고)
  4. 개선 조치 이행 및 사후 관리
  5. 데이터 품질 담당자 지정

평가 체계:
  데이터 품질 수준 = (달성 점수 / 만점) × 100
  85점 이상: 우수 / 70~85: 보통 / 70 이하: 미흡

감리 체크포인트:
  + 품질 관리 계획 수립 여부
  + 측정 도구 및 방법의 적절성
  + 결함 발견율 및 조치율
  + 품질 개선 계획의 구체성
```

> 📢 **섹션 요약 비유**: 공공 데이터 품질관리 지침은 식품 위생 기준 — 의무이고, 정기 점검이 있고, 불합격이면 시정 조치를 받는다.

## V. 실무 시나리오 — 금융 대출 심사 데이터 품질 진단

```
문제:
  은행 대출 심사 AI 모델 성능 저하
  원인 불명확 -> 데이터 품질 진단 실시

진단 결과:
  정확성: 고객 소득 데이터 8% 오류 (자동화 수집 오류)
  완전성: 직업 코드 12% NULL (신규 직군 미분류)
  일관성: CRM과 심사 시스템 주소 불일치 15%
  최신성: 신용점수 갱신 주기 90일 → 90일 전 데이터 사용

개선:
  소득 수집 API 검증 로직 강화
  직업 코드 신규 추가 및 매핑 테이블 업데이트
  CRM-심사 시스템 일일 동기화 배치 추가
  신용점수 갱신 주기 30일로 단축

결과:
  AI 모델 F1-Score: 0.72 -> 0.89 (+23.6%)
```

> 📢 **섹션 요약 비유**: AI가 틀린 게 아니라 데이터가 틀렸다 — 품질 진단 후 데이터를 고쳤더니 AI 성능이 극적으로 개선.

## 👶 어린이를 위한 3줄 비유 설명

1. 데이터 품질 진단은 정보가 정확한지, 빠진 것은 없는지, 두 곳에서 다르게 저장되지 않았는지 검사하는 거예요.
2. 나쁜 데이터로 AI를 가르치면 나쁜 AI가 되니까, 데이터 청소가 AI 개발의 첫 번째 단계예요.
3. 은행이나 병원처럼 중요한 곳에서는 데이터 품질이 사람의 생활에 직접 영향을 주기 때문에 법으로 관리해요!
