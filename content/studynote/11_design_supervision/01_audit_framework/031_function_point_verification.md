## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 기능 점수(Function Point, FP) 검증은 소프트웨어 개발 규모 산정 결과의 정확성을 감리 관점에서 검토하는 활동이다. FP 산정 오류는 예산·일정·품질 계획 전체에 연쇄 영향을 주므로 감리의 핵심 검토 항목이다.
> 2. **가치**: IFPUG(International Function Point Users Group) FP 방법론은 외부 입력(EI)·출력(EO)·조회(EQ)·내부 논리 파일(ILF)·외부 인터페이스 파일(EIF) 5가지 컴포넌트를 측정한다. 각 컴포넌트의 복잡도(단순/보통/복잡)에 따라 FP를 산정한다.
> 3. **판단 포인트**: 한국 공공 IT 사업에서 SW 사업 대가 기준(SW진흥법)은 FP 기반 개발비 산정을 규정한다. 감리사는 산정서가 기능 목록과 일치하는지, 복잡도 분류가 적절한지를 검증한다.

---

## Ⅰ. 개요 및 필요성

```text
FP 산정 컴포넌트:

  기능 유형    | 설명                    | 단순 | 보통 | 복잡
  ─────────────┼─────────────────────────┼──────┼──────┼──────
  EI (외부입력) | 사용자→시스템 데이터    |  3   |  4   |  6
  EO (외부출력) | 시스템→사용자 데이터    |  4   |  5   |  7
  EQ (외부조회) | 입출력 합산 없는 조회   |  3   |  4   |  6
  ILF (내부파일) | 내부 논리 데이터 그룹  |  7   | 10   | 15
  EIF (외부파일) | 참조 외부 데이터 그룹  |  5   |  7   | 10

  합산 FP = Σ(각 컴포넌트 수 × FP 가중치)
```

- **📢 섹션 요약 비유**: FP 산정은 건물 평형 측정이다. 방 수(EI/EO/EQ)와 창고(ILF)·주차장(EIF) 면적을 각각 측정하고 복잡도(고급/일반/기본)를 반영해 전체 면적(FP 합계)을 산출한다.

---

## Ⅱ. 아키텍처 및 핵심 원리

### 복잡도 결정 기준

| 컴포넌트 | 단순 기준 | 복잡 기준 |
|:---|:---|:---|
| **EI** | DET ≤ 4, FTR ≤ 1 | DET > 15, FTR > 2 |
| **EO** | DET ≤ 5, FTR ≤ 1 | DET > 19, FTR > 3 |
| **ILF** | DET ≤ 19, RET ≤ 1 | DET > 50, RET > 5 |

*DET: Data Element Type (데이터 항목), FTR: File Type Referenced, RET: Record Element Type

### 감리 FP 검증 절차

```text
1. 요구사항 명세서 → 기능 목록 추출
2. 기능 목록 vs FP 산정서 매핑 검증
   - 누락 기능 확인
   - 과다 산정 기능 확인
3. 복잡도 분류 적절성 검토
   - DET·FTR·RET 카운트 재확인
4. 기술 조정 인자(VAF) 적용 검토
5. FP 환산 공수(MM) 적정성 검토
   - SW진흥원 기능점수당 단가 적용
```

- **📢 섹션 요약 비유**: FP 감리 검증은 건축 면적 재측량이다. 건축주(개발사)가 제출한 설계 도면(FP 산정서)을 감리사가 독립적으로 재측량(재산정)하여 면적(규모) 오류를 확인한다.

## Ⅲ. 비교 및 연결

| 비교 | IFPUG FP | NESMA FP | Use Case Point |
|:---|:---|:---|:---|
| 기관 | 국제 표준 | 유럽 표준 | 대안 방법론 |
| 측정 대상 | 기능 규모 | 기능 규모 | Use Case 규모 |
| 국내 적용 | SW 사업 대가 기준 | 일부 적용 | 병행 적용 |

- **📢 섹션 요약 비유**: FP 방법론들은 면적 단위 체계다. 평방미터(IFPUG), 평방피트(NESMA), 야드(UCP)처럼 단위는 달라도 같은 건물 크기를 측정하는 방법이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 공공 사업 FP 검증 실무

```text
검증 포인트:
  □ FP 산정 기준일(요구사항 기준) 명확화
  □ 재사용 기능 비율 적정성 (재사용 시 할인 적용)
  □ 인터페이스 처리 FP 누락 확인
  □ 배치 프로그램 FP 반영 여부
  □ 단순 반복 CRUD에 복잡도 과다 부여 여부

자동화 도구:
  - ESTI-FP (한국 SW산업협회)
  - 공공 SW 사업 FP 자동 산정 도구
```

- **📢 섹션 요약 비유**: FP 공공 사업 검증은 세금 신고 검증이다. 납세자(개발사)가 제출한 세금 신고서(FP 산정서)를 세무조사(감리)로 검증하고, 과소·과다 신고(누락·과다 산정)를 잡아낸다.

## Ⅴ. 기대효과 및 결론

| 기대효과 | 내용 |
|:---|:---|
| **예산 적정성** | FP 기반 개발비 객관화 |
| **일정 타당성** | 규모-공수 환산의 정확성 |
| **분쟁 예방** | 납품 범위 명확화 |

AI 기반 자동 FP 산정 도구가 연구되고 있다. NLP가 요구사항 명세서 텍스트를 분석하여 EI·EO·EQ·ILF·EIF를 자동 식별하고, FP를 자동 산정하는 기술이 국내 SW 산업 연구소에서 개발 중이다. 감리사는 AI 산정 결과의 비즈니스 맥락 검증으로 역할이 진화하고 있다.

- **📢 섹션 요약 비유**: AI FP 자동 산정은 AI 세무사다. 영수증(요구사항) 사진을 찍으면 AI가 자동으로 세금 계산(FP 산정)을 하고, 세무사(감리사)는 특이 항목만 최종 검토한다.
