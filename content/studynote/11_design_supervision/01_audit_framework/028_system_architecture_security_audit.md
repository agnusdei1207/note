## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 아키텍처 보안 감리는 정보시스템 감리의 보안 영역으로, 아키텍처 설계 단계에서 보안 원칙(Defense in Depth, Least Privilege, Zero Trust 등)이 올바르게 구현됐는지를 독립적으로 검증하는 활동이다.
> 2. **가치**: 보안 취약점은 운영 단계보다 설계 단계에서 발견할 때 수정 비용이 10~100배 저렴하다. 아키텍처 감리는 "설계 시 보안(Security by Design)"을 강제하여 레거시 시스템의 보안 취약점 누적을 예방한다.
> 3. **판단 포인트**: 클라우드 네이티브·마이크로서비스 아키텍처에서 보안 감리의 범위가 확대됐다. 서비스 메시 정책(Istio), API 게이트웨이 접근 제어, 컨테이너 이미지 보안, 서비스 계정 최소 권한이 현대 아키텍처 보안 감리의 핵심 점검 항목이다.

---

## Ⅰ. 개요 및 필요성

```text
┌──────────────────────────────────────────────────────┐
│       아키텍처 보안 감리 체계                         │
├──────────────────────────────────────────────────────┤
│                                                       │
│  감리 범위:                                           │
│  ┌────────────────────────────────────────────────┐  │
│  │ 네트워크 보안 아키텍처 (방화벽·DMZ·세그먼트)   │  │
│  │ 인증·권한 아키텍처 (IAM·SSO·MFA)              │  │
│  │ 데이터 보안 (암호화·키 관리·DLP)               │  │
│  │ 접근 제어 (RBAC·ABAC·Least Privilege)         │  │
│  │ 보안 모니터링 (SIEM·SOAR 통합)                │  │
│  └────────────────────────────────────────────────┘  │
│                                                       │
│  Zero Trust 원칙 검증:                                │
│    "Never Trust, Always Verify" 구현 여부             │
└──────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: 시스템 아키텍처 보안 감리는 건물 준공 전 소방 안전 검사다. 건물이 완공되기 전 설계 단계에서 비상구·스프링클러·방화벽을 제대로 설계했는지 독립적으로 검사한다.

---

## Ⅱ. 아키텍처 및 핵심 원리

### 보안 아키텍처 핵심 원칙 검증

| 원칙 | 감리 점검 항목 |
|:---|:---|
| **Defense in Depth** | 다계층 보안 통제 구현 여부 |
| **Least Privilege** | 서비스·사용자별 최소 권한 부여 |
| **Zero Trust** | 내부 트래픽도 인증·암호화 |
| **Fail Secure** | 장애 시 보안 강화 방향으로 동작 |
| **Separation of Duties** | 권한 분리 (개발·운영·보안 분리) |

### 클라우드 네이티브 보안 감리 항목

```text
컨테이너 보안:
  - 이미지 취약점 스캐닝 (Trivy, Snyk)
  - 특권 컨테이너(Privileged Container) 사용 여부
  - 읽기 전용 루트 파일 시스템

서비스 메시 (Istio):
  - mTLS (상호 TLS) 구현 여부
  - 서비스 간 최소 권한 정책

IAM:
  - 서비스 계정 키 만료 정책
  - MFA 강제 적용 여부
```

- **📢 섹션 요약 비유**: 클라우드 보안 감리는 아파트 단지 보안 점검이다. 출입 카드(IAM), 동별 잠금(Zero Trust), CCTV(모니터링), 화재 경보(SIEM) 등 각 보안 레이어가 제대로 구축됐는지 확인한다.

## Ⅲ. 비교 및 연결

| 비교 | 아키텍처 감리 | 운영 보안 감사 |
|:---|:---|:---|
| 시점 | 설계·구축 단계 | 운영 단계 |
| 목적 | 설계 보안 검증 | 운영 취약점 점검 |
| 효과 | 선제적 예방 | 사후 발견·개선 |
| 비용 효율 | 매우 높음 | 보통 |

- **📢 섹션 요약 비유**: 아키텍처 감리 vs 운영 감사는 건물 설계 검토 vs 건물 안전 점검이다. 설계 단계 검토가 완공 후 보강 공사보다 훨씬 저렴하고 효과적이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

### 공공 정보화 사업 보안 감리 체크리스트
- **개인정보 보호**: 개인정보 암호화(AES-256), 전송 구간 TLS 1.2 이상.
- **접근 제어**: 역할 기반(RBAC) 구현, 관리자 계정 분리.
- **네트워크**: DMZ 구성, 내부망·외부망 분리, 방화벽 정책 문서화.
- **로깅·모니터링**: 감사 로그 6개월 이상 보존, SIEM 연동.
- **취약점 관리**: OWASP Top 10 점검, 정적 분석(SAST) 수행 여부.

- **📢 섹션 요약 비유**: 공공 시스템 보안 감리 체크리스트는 자동차 안전 검사 항목이다. 브레이크(접근 제어), 에어백(암호화), 블랙박스(로깅), 충돌 테스트(취약점 점검) — 각 항목을 빠짐없이 확인한다.

## Ⅴ. 기대효과 및 결론

| 기대효과 | 내용 |
|:---|:---|
| **설계 보안 강화** | Security by Design 실현 |
| **비용 절감** | 운영 단계 보안 패치 비용 감소 |
| **규제 준수** | 개인정보보호법·클라우드 보안 기준 충족 |

AI 기반 자동화 보안 감리(Automated Security Architecture Review)가 등장하고 있다. Infrastructure as Code(IaC) 파일을 자동 분석하여 보안 정책 위반을 실시간 탐지하는 정적 보안 분석 도구(Checkov, tfsec)가 현대 아키텍처 감리의 보조 도구로 활용된다.

- **📢 섹션 요약 비유**: IaC 보안 자동 감리는 코드 보안 맞춤법 검사기다. 인프라 코드(Terraform)를 작성할 때 "이 설정은 보안 취약점이 있어요"라고 실시간으로 알려주는 자동 감리 도구다.

### 👶 어린이를 위한 3줄 비유 설명

1. 시스템 보안 감리는 건물 완공 전 소방 안전 검사예요! 나중에 고치는 것보다 설계 단계에서 잡는 게 훨씬 저렴해요.
2. Zero Trust(절대 믿지 않기) 원칙이 제대로 구현됐는지, 모든 문에 잠금장치가 있는지 확인해요!
3. 요즘은 AI가 인프라 코드를 자동으로 분석해서 보안 취약점을 즉시 알려주는 자동 감리 도구도 있어요!
