## 핵심 인사이트 (3줄 요약)

> 1. **본질**: Blackboard Pattern (블랙보드 패턴)은 여러 전문 모듈(KS, Knowledge Source, 지식 소스)이 공유 데이터 공간(Blackboard, 칠판)에 부분 해법을 올리고 내려받으며 협력해 복잡한 문제를 점진적으로 해결하는 아키텍처이다.
> 2. **가치**: 어떤 알고리즘도 단독으로 최적 해를 보장할 수 없는 **비결정론적 문제**(음성인식, 이미지 분석, 자율주행)에서 다양한 전략을 동적으로 조합할 수 있다.
> 3. **판단 포인트**: 부분 해법이 서로 다른 전문가 모듈에서 나오고, 해법을 통합할 글로벌 알고리즘이 존재하지 않을 때 블랙보드 패턴을 선택한다.

---

## Ⅰ. 개요 및 필요성

1970년대 카네기멜론 대학(Carnegie Mellon University)의 음성인식 프로젝트 HEARSAY-II에서 처음 공식화된 블랙보드 패턴은, 음소(Phoneme)·단어(Word)·문장(Sentence) 각각을 분석하는 전문 모듈들이 공유 칠판을 통해 협력해 발화(Utterance)를 인식하는 구조를 제안했다.

핵심 전제는 **"문제를 단번에 해결하는 단일 알고리즘이 존재하지 않는다"**는 것이다. 대신 여러 전문 지식이 점진적·상호 보완적으로 부분 해법을 축적하면 최종 해답에 도달한다.

### 등장 배경

| 문제 유형 | 이유 | 블랙보드 적합도 |
|:---|:---|:---|
| 음성인식 (Speech Recognition) | 음소→단어→문장 각 레벨의 전문 처리 필요 | ★★★★★ |
| 이미지 분석 (Image Analysis) | 픽셀·엣지·객체 인식 단계별 전문성 필요 | ★★★★☆ |
| 자율주행 판단 | 센서·교통법규·경로 계획 모듈 독립 작동 | ★★★★☆ |
| 단순 CRUD 시스템 | 결정론적 처리 가능 | ★☆☆☆☆ |

📢 **섹션 요약 비유**: 여러 전문의가 한 환자의 차트(칠판)를 번갈아 보며 각자 소견을 적어 나가고, 마지막에 주치의가 종합 진단을 내리는 의료 컨퍼런스가 블랙보드 패턴이다.

## Ⅱ. 아키텍처 및 핵심 원리

### 블랙보드 패턴 구조

```
┌──────────────────────────────────────────────────────────────┐
│                  Blackboard (블랙보드 / 공유 칠판)            │
│                                                              │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐             │
│  │  Level 1   │  │  Level 2   │  │  Level 3   │   ···       │
│  │ 부분 해법  │  │ 부분 해법  │  │ 부분 해법  │             │
│  └────────────┘  └────────────┘  └────────────┘             │
└──────────┬───────────────────────────────────────────────────┘
           │  읽기/쓰기
  ┌────────┼──────────────────────────────────────────┐
  │        ▼                                          │
  │  ┌─────────────┐   ┌─────────────┐               │
  │  │ Knowledge   │   │ Knowledge   │  ···           │
  │  │ Source 1    │   │ Source 2    │                │
  │  │ (지식 소스) │   │ (지식 소스) │                │
  │  └─────────────┘   └─────────────┘               │
  │                                                   │
  │         Control Component (제어 컴포넌트)          │
  │  ┌──────────────────────────────────────────────┐ │
  │  │ 1. 칠판 상태 감시                             │ │
  │  │ 2. 활성화할 KS 선택 (스케줄링)               │ │
  │  │ 3. 종료 조건 판단                             │ │
  │  └──────────────────────────────────────────────┘ │
  └───────────────────────────────────────────────────┘
```

### 구성 요소별 역할

| 구성요소 | 역할 | 비고 |
|:---|:---|:---|
| Blackboard (칠판) | 공유 데이터 구조, 부분 해법 누적 | 계층화된 데이터 구조 |
| Knowledge Source (지식 소스) | 독립적인 전문 처리 모듈 | 서로를 직접 호출하지 않음 |
| Control Component (제어기) | KS 실행 순서 결정, 종료 판단 | 기회주의적(Opportunistic) 스케줄링 |

📢 **섹션 요약 비유**: 화이트보드에 단서를 적어가는 형사팀처럼, 각 형사(KS)가 전문 분야의 단서를 칠판에 추가하고, 팀장(Controller)이 어떤 형사를 다음에 투입할지 결정한다.

## Ⅲ. 비교 및 연결

### 블랙보드 vs 파이프-필터 vs 이벤트 기반

| 항목 | Blackboard | Pipe-Filter | Event-Driven |
|:---|:---|:---|:---|
| 데이터 흐름 | 공유 공간 경유 | 단방향 스트림 | 이벤트 버스 |
| 모듈 간 통신 | 칠판 통한 간접 공유 | 파이프를 통한 직접 전달 | 이벤트 구독/발행 |
| 처리 순서 | 기회주의적, 동적 결정 | 정적 파이프라인 순서 | 이벤트 발생 순서 |
| 상태 공유 | 칠판 = 공유 상태 | 각 필터 무상태 | 이벤트로만 상태 전달 |
| 적합 문제 | 비결정론적, 복잡 추론 | 결정론적 변환 처리 | 느슨한 결합 시스템 |
| 대표 사례 | HEARSAY-II 음성인식 | Unix 파이프, Kafka | 주문 시스템, IoT |

📢 **섹션 요약 비유**: 블랙보드는 여러 전문가가 같은 화이트보드에 의견을 적는 브레인스토밍, 파이프-필터는 릴레이 경주, 이벤트 기반은 누군가 "손 들면" 반응하는 강의실이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 현대 시스템에서의 블랙보드 패턴

**머신러닝 앙상블(ML Ensemble)**: 여러 모델(랜덤 포레스트, XGBoost, 신경망)이 각자 예측 결과를 공유 공간에 제출하고, 메타 러너(Meta-learner)가 최종 예측을 종합하는 스태킹(Stacking)이 블랙보드 패턴의 현대적 구현이다.

**이벤트 소싱(Event Sourcing) 시스템**: 이벤트 스토어(Event Store)가 블랙보드 역할을 하고, 여러 프로젝션(Projection) 핸들러가 각자의 뷰를 생성한다.

**모니터링 이상탐지 시스템**: CPU 사용률 분석기, 메모리 분석기, 네트워크 패킷 분석기가 각각 이상 신호를 공유 칠판에 기록하고, 상관관계 엔진(Correlation Engine)이 복합 이상을 탐지한다.

### 설계 시 주의사항

| 주의사항 | 설명 | 대응 방안 |
|:---|:---|:---|
| 칠판 병목 | 모든 KS가 단일 칠판에 접근 | 분산 칠판(Distributed Blackboard) 설계 |
| 종료 조건 불명확 | 언제 해법이 충분한지 판단 어려움 | 신뢰도 임계값(Confidence Threshold) 설정 |
| 순환 의존 | KS 간 간접 의존성 형성 가능 | 칠판 계층화, 쓰기 권한 제한 |
| 디버깅 어려움 | 실행 순서가 동적으로 결정됨 | 칠판 상태 스냅샷(Snapshot) 로깅 |

### 기술사 판단 기준

- 문제 해결에 **2개 이상의 이질적인 전문 알고리즘이 필요**하고 고정된 실행 순서가 없다면 블랙보드를 검토한다.
- 실시간성이 중요한 경우, 칠판을 **인메모리(In-memory) 저장소(Redis 등)**로 구현한다.
- AI/ML 파이프라인에서 앙상블 전략 설명 시 블랙보드 패턴 관점에서 접근하면 기술사 답안의 깊이가 높아진다.

📢 **섹션 요약 비유**: 퍼즐을 여러 명이 함께 맞추는 것처럼, 각자 자기가 잘 아는 조각을 먼저 놓고, 조각이 쌓이면서 전체 그림이 보이기 시작한다.

## Ⅴ. 기대효과 및 결론

블랙보드 패턴은 단일 알고리즘으로 해결 불가능한 복잡한 추론 문제에서 **전문성의 분산과 협력**을 통해 최적 해를 탐색한다. 현대에는 ML 앙상블, 분산 이벤트 처리, 실시간 이상탐지 시스템 등에서 그 원리가 재발견되고 있다.

| 기대효과 | 설명 |
|:---|:---|
| 전문성 모듈화 | 각 KS가 독립적으로 개발·교체 가능 |
| 동적 확장 | 새로운 KS를 기존 시스템 변경 없이 추가 |
| 복잡 문제 해결 | 단일 알고리즘으로 불가능한 문제 대응 |
| 증분 해법 | 부분 해법을 점진적으로 정제 |

📢 **섹션 요약 비유**: 학교 시험을 여러 선생님이 문제를 나눠 출제하고 채점하듯, 블랙보드는 전문 지식을 모아 최선의 답을 만들어가는 집단지성 아키텍처이다.

### 👶 어린이를 위한 3줄 비유 설명

- 교실 칠판에 선생님들이 돌아가며 자기가 아는 부분을 적어나가면서 어려운 문제의 답을 함께 찾아가는 것이 블랙보드 패턴이에요.
- 수학 선생님, 과학 선생님, 국어 선생님이 각자 잘 아는 부분만 적어도, 모이면 완전한 답이 돼요.
- 반장(Controller)은 누가 다음에 칠판에 나올지 정하고, 답이 완성되면 수업을 마쳐요.
