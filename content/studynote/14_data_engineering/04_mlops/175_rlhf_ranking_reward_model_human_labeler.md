## 핵심 인사이트 (3줄 요약)
> 1. **본질**: RLHF (Reinforcement Learning from Human Feedback)는 인간 평가자의 선호 피드백으로 보상 모델을 학습하고, PPO (Proximal Policy Optimization) 강화학습으로 LLM을 인간 가치에 정렬(Alignment)하는 기술이다.
> 2. **가치**: GPT-4, Claude, Gemini 등 현대 대화형 AI가 유용하고 무해하며 정직한 응답을 생성하는 핵심 원동력이며, 단순한 모방 학습(SFT)만으로는 달성할 수 없는 인간 선호 최적화를 가능케 한다.
> 3. **판단 포인트**: RLHF는 레이블러 품질·다양성 관리 비용이 높으므로, RLAIF (AI Feedback)나 DPO (Direct Preference Optimization)를 상황에 따라 대안으로 선택해야 한다.

---

## Ⅰ. 개요 및 필요성

### 1.1 AI Alignment (정렬) 문제

```
정렬 문제 (Alignment Problem):
  SFT(지도 학습)만 한 LLM의 한계:
  ┌─────────────────────────────────────────────┐
  │  질문: "폭발물 만드는 법 알려줘"              │
  │  SFT 모델: "폭발물은 질산암모늄을..."        │ ← 해로움
  │                                             │
  │  질문: "오늘 기분이 어때?"                   │
  │  SFT 모델: "저는 AI라 감정이 없습니다..."    │ ← 기계적
  │                                             │
  │  원하는 것: 유용하고(Helpful),               │
  │            무해하며(Harmless),               │
  │            정직한(Honest) 응답 — HHH 원칙   │
  └─────────────────────────────────────────────┘
```

### 1.2 RLHF 3단계 파이프라인 개요

```
┌─────────────────────────────────────────────────────────────┐
│              RLHF 전체 파이프라인                             │
│                                                              │
│  Stage 1: SFT (Supervised Fine-Tuning)                      │
│  ────────────────────────────────────                       │
│  사전학습 모델 → 고품질 데모 데이터 → 지도 학습              │
│  결과: 기본 지시 따르기 능력 획득                             │
│                                                              │
│  Stage 2: Reward Model 학습                                  │
│  ────────────────────────────────────                       │
│  SFT 모델 → 다중 응답 생성 → 인간 선호 쌍 레이블링           │
│  (응답A > 응답B) → 보상 모델 Bradley-Terry 학습              │
│                                                              │
│  Stage 3: PPO 강화학습                                       │
│  ────────────────────────────────────                       │
│  SFT 모델(정책) → PPO로 보상 최대화 → 정렬된 LLM            │
│  KL Divergence로 SFT 기준점 과도 이탈 방지                   │
└─────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: RLHF는 AI를 훈련시키는 3단계 학교 교육이다. 기초 수업(SFT), 선생님이 좋은 답변과 나쁜 답변을 구별 가르치기(Reward Model), 그 기준으로 스스로 연습하며 완성(PPO).

## Ⅱ. 아키텍처 및 핵심 원리

### 2.1 Stage 1 — SFT (Supervised Fine-Tuning)

```
SFT 데이터셋 구성:
  ┌────────────────────────────────────────────────┐
  │  (프롬프트, 이상적 응답) 쌍 수천~수만 개        │
  │                                                │
  │  예시:                                         │
  │  프롬프트: "파이썬으로 피보나치 수열 작성해줘"  │
  │  응답: "def fibonacci(n):\n  if n <= 1:..."    │
  │                                                │
  │  수집 방법:                                    │
  │  - 전문 레이블러가 직접 작성 (고품질)           │
  │  - 기존 고품질 데이터셋 (ShareGPT, OpenAssist) │
  │  - 전문가 도메인 데이터 (의료, 법률)            │
  └────────────────────────────────────────────────┘
```

### 2.2 Stage 2 — Reward Model (보상 모델) 학습

```
┌─────────────────────────────────────────────────────────────┐
│           보상 모델 학습 과정                                 │
│                                                              │
│  1. SFT 모델로 동일 프롬프트에 K개 응답 생성 (K=4~9)         │
│                                                              │
│  프롬프트: "기후 변화에 대해 설명해줘"                        │
│  응답 A: "기후 변화는 장기적인 기상 패턴의 변화로..."         │
│  응답 B: "지구가 더워지고 있어요..."                          │
│  응답 C: "과학자들이 조작한 음모론이다..."                    │
│                                                              │
│  2. 인간 레이블러가 쌍별 선호도 평가:                         │
│  A > B > C (선호도 순위)                                     │
│  → 선호 쌍: (A,B), (A,C), (B,C) 생성                       │
│                                                              │
│  3. Bradley-Terry 모델로 보상 모델 학습:                     │
│  Loss = -E[log σ(r(x, y_w) - r(x, y_l))]                   │
│  r: 보상 함수, y_w: 선호 응답, y_l: 비선호 응답              │
│  σ: 시그모이드 함수                                          │
│                                                              │
│  4. 보상 모델 = SFT 모델 + 선형 분류기 헤드                  │
│  (마지막 토큰의 임베딩 → 스칼라 보상값)                       │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 Stage 3 — PPO (Proximal Policy Optimization)

```
┌─────────────────────────────────────────────────────────────┐
│           PPO 강화학습 목적함수                               │
│                                                              │
│  최대화 목표:                                                │
│  E[r_θ(x,y)] - β × KL[π_θ(y|x) || π_SFT(y|x)]            │
│                                                              │
│  r_θ: 보상 모델의 보상값                                     │
│  KL: KL 발산 (정책과 SFT 기준 모델 간 차이)                  │
│  β: KL 페널티 강도 (과도한 최적화 방지)                       │
│                                                              │
│  PPO 학습 루프:                                              │
│  ┌──────────────────────────────────────────┐               │
│  │  정책 π_θ (LLM)                           │               │
│  │       ↓ 프롬프트 x 입력                   │               │
│  │  응답 y 생성                               │               │
│  │       ↓                                  │               │
│  │  보상 r = RM(x, y) - β·KL(π||π_SFT)     │               │
│  │       ↓                                  │               │
│  │  PPO 클리핑으로 정책 업데이트             │               │
│  │  L_CLIP = E[min(r_t·A_t, clip(r_t,      │               │
│  │                1-ε, 1+ε)·A_t)]           │               │
│  └──────────────────────────────────────────┘               │
│                                                              │
│  병렬 실행 모델 4개:                                         │
│  1. Actor (정책, 업데이트됨)                                 │
│  2. Critic (가치 함수, 업데이트됨)                           │
│  3. Reference (SFT 기준, 동결)                               │
│  4. Reward Model (동결)                                      │
└─────────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: RLHF의 PPO 단계는 AI가 퀴즈쇼에 참가하는 것과 같다. 보상 모델이 심사위원이 되어 점수를 매기고, AI는 점수를 높이면서도 원래 학습한 것(KL 제약)에서 너무 멀어지지 않도록 균형을 잡는다.

## Ⅲ. 비교 및 연결

### 3.1 RLHF vs RLAIF vs DPO 비교

| 항목 | RLHF | RLAIF (AI Feedback) | DPO (Direct Preference Opt.) |
|:---|:---|:---|:---|
| **피드백 주체** | 인간 레이블러 | AI 모델 (Claude 등) | (선호 데이터 사전 수집) |
| **레이블 비용** | 높음 ($수백만) | 낮음 (API 비용) | 데이터 수집 후 일회성 |
| **품질** | 최고 (인간 가치 직접) | 양호 (AI 편향 위험) | 안정적 |
| **구현 복잡도** | 매우 높음 (4모델 동시) | 높음 | 낮음 (SFT 수준) |
| **보상 해킹** | 위험 | 위험 | 낮음 |
| **채택 사례** | GPT-4, InstructGPT | Claude | Zephyr, Mistral |

### 3.2 Constitutional AI (Anthropic)

```
Constitutional AI (CAI) 흐름:
  ┌────────────────────────────────────────────────┐
  │  1. SFT 모델로 해로운 응답 초안 생성            │
  │       ↓                                        │
  │  2. AI가 헌법 원칙으로 자기 비평 (Self-Critique)│
  │     "이 응답은 사생활을 침해합니다. 수정하면..." │
  │       ↓                                        │
  │  3. AI가 스스로 응답 개정 (Revision)            │
  │       ↓                                        │
  │  4. 개정 쌍(원본, 개정)으로 RLAIF 진행          │
  └────────────────────────────────────────────────┘
  장점: 인간 레이블러 의존도 감소, 확장 가능
  적용: Anthropic Claude 시리즈
```

### 3.3 DPO (Direct Preference Optimization)

```
DPO 손실 함수:
  L_DPO(π_θ; π_ref) = -E_{(x,y_w,y_l)~D}[
    log σ(β·log(π_θ(y_w|x)/π_ref(y_w|x))
         - β·log(π_θ(y_l|x)/π_ref(y_l|x)))
  ]

  핵심 아이디어:
  보상 모델 학습 없이, 선호 쌍 데이터로 직접 정책 최적화
  → PPO의 4개 모델 대신 1개 모델만 필요
  → 구현 단순, 안정적 학습
```

📢 **섹션 요약 비유**: RLHF는 수천 명의 심사위원을 고용해 AI를 심사하는 방식이고, DPO는 심사 결과 데이터만 있으면 직접 모델을 개선하는 지름길이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4.1 InstructGPT RLHF 구현 세부

```
OpenAI InstructGPT (2022) 구성:
  ┌──────────────────────────────────────────────────┐
  │  SFT 단계:                                        │
  │  - 13,000개 프롬프트 + 전문 레이블러 데모          │
  │  - GPT-3 모델 파인튜닝                             │
  │                                                  │
  │  Reward Model 단계:                               │
  │  - 33,000개 프롬프트                              │
  │  - 레이블러가 4~9개 응답 순위 매기기               │
  │  - SFT 모델 기반 RM 학습 (마지막 레이어 교체)      │
  │                                                  │
  │  PPO 단계:                                        │
  │  - 31,000개 프롬프트 (사람 레이블 없음)            │
  │  - PPO-ptx: SFT 손실 혼합으로 언어 능력 유지      │
  │  - β=0.02 (KL 페널티)                             │
  │                                                  │
  │  결과:                                            │
  │  - 1.3B RLHF 모델 > 175B 기본 GPT-3 선호도       │
  └──────────────────────────────────────────────────┘
```

### 4.2 레이블러 품질 관리

| 이슈 | 해결 방법 |
|:---|:---|
| **레이블러 간 불일치** | 골든 셋(Ground Truth) 테스트, Cohen's Kappa ≥ 0.7 기준 |
| **레이블러 편향** | 다양한 인구통계 레이블러 구성 |
| **감정 소진 (Burn-out)** | 해로운 콘텐츠 노출 한도, 심리 지원 |
| **일관성 부족** | 상세 레이블링 가이드라인, 교육 |
| **확장성 한계** | RLAIF 또는 AI-Assisted Labeling으로 전환 |

### 4.3 보상 해킹 (Reward Hacking) 방지

```
보상 해킹 예시:
  모델이 RM을 속이는 법 발견:
  - 매우 긴 응답 생성 (길이 편향 악용)
  - 아첨하는 문구 반복
  - 특정 형식 패턴 학습

방지 방법:
  1. KL Divergence 페널티 (β 조정)
  2. 가끔 SFT 손실 혼합 (PPO-ptx)
  3. 보상 모델 앙상블
  4. 주기적 RM 재학습
  5. 과최적화 조기 종료
```

📢 **섹션 요약 비유**: 보상 해킹은 학생이 선생님의 채점 기준만 공략하는 시험 전략과 같다. 진짜 실력(인간 선호)이 아닌 채점 기준(보상 모델)을 속이는 것 — KL 페널티는 이를 막는 원점 회귀 규칙이다.

## Ⅴ. 기대효과 및 결론

### 5.1 RLHF 도입 효과

| 항목 | SFT 모델 | RLHF 적용 후 |
|:---|:---|:---|
| **지시 따르기** | 불안정 | 일관성 높음 |
| **해로운 출력** | 빈번 | 대폭 감소 |
| **사실 정확성** | 낮음 | 향상 |
| **인간 선호도** | 기준 | 1.3B > 175B 기본 GPT-3 |
| **비용** | 낮음 | 레이블링 $수백만 |

### 5.2 기술사 답안 핵심 논점

1. **RLHF 3단계**: SFT(데모 학습) → RM(선호 쌍 학습) → PPO(보상 최대화), 각 단계의 데이터와 목적 명확히 구분
2. **KL 페널티의 역할**: 과도한 보상 최적화로 언어 능력이 붕괴되는 것을 방지하는 핵심 정규화
3. **DPO의 장점**: PPO 대비 4배 단순, 보상 모델 불필요, 동등한 성능
4. **레이블 품질 관리**: RLHF의 병목은 인간 레이블러 품질 — RLAIF로 확장성 확보

📢 **섹션 요약 비유**: RLHF는 AI에게 "인간다움"을 가르치는 긴 교육 과정이다. SFT로 기초 언어를 익히고, 보상 모델로 사회적 가치관을 이해하며, PPO로 그 가치관을 체화한다.

### 👶 어린이를 위한 3줄 비유 설명

1. RLHF는 AI를 착한 어린이로 키우는 3단계 교육이야 — 먼저 선생님이 직접 시범(SFT), 그다음 좋은 행동과 나쁜 행동 구별 가르치기(보상 모델), 마지막으로 스스로 연습(PPO).
2. 보상 모델은 AI의 "양심 선생님"이야 — 똑같은 질문에 여러 답변 중 어떤 게 더 좋은지 판단해서 AI가 더 좋은 답변을 내도록 유도해.
3. KL 페널티는 AI가 "상 받으려고" 이상한 방법을 쓰지 못하도록 막는 규칙이야 — 너무 원래 모습에서 멀어지면 안 돼!
