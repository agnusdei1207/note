## 핵심 인사이트 (3줄 요약)

> 1. **본질**: 차분 프라이버시(DP, Differential Privacy)는 "데이터셋에 특정 개인이 포함되든 포함되지 않든 통계 결과가 거의 동일하게" 보장하는 수학적 프레임워크로, **노이즈를 의도적으로 추가하여 개인 기여를 은닉**한다.
> 2. **가치**: ε(엡실론)-DP는 프라이버시 강도를 수치로 표현하여 "데이터 유용성 vs 프라이버시 보호"의 트레이드오프를 **측정 가능하고 증명 가능한 방식**으로 관리할 수 있게 한다.
> 3. **판단 포인트**: ε 값 설정이 핵심—ε이 작을수록 프라이버시 강하지만 데이터 정확도 낮음; 실무에서는 ε=1(강) ~ ε=10(중간)을 기준으로, 글로벌 DP(신뢰 서버) vs 로컬 DP(신뢰 없는 환경) 모델을 선택해야 한다.

---

## Ⅰ. 개요 및 필요성

### 1.1 차분 프라이버시의 등장 배경

2006년 Cynthia Dwork이 제안한 DP는 기존 프라이버시 보호 기법의 약점을 수학적으로 해결했다.

| 기존 기법 | 약점 | DP의 해결 |
|:---|:---|:---|
| 데이터 익명화 | 재식별(Re-identification) 공격 취약 | 개인 기여 자체를 수학적으로 은닉 |
| K-익명성 | 동질성·배경지식 공격 취약 | 배경지식과 무관하게 보호 |
| 집계 통계 공개 | 차분 공격(Differencing Attack)으로 개인 정보 추론 | 쿼리 결과에 노이즈 추가 |

**차분 공격 예시:**

```
쿼리 1: "서울 거주 30대 여성의 평균 연봉" = 4,500만원 (N=1,000명)
쿼리 2: "서울 거주 30대 여성 (김○○ 제외) 평균 연봉" = 4,495만원 (N=999명)
차이 분석: 김○○의 연봉 ≈ 4,500×1,000 - 4,495×999 = 9,505만원 (개인정보 노출!)

DP 적용 시:
쿼리 결과에 노이즈 추가 → 차이를 개인 정보로 활용 불가
```

### 1.2 DP의 수학적 정의

**ε-차분 프라이버시 (ε-Differential Privacy):**

```
랜덤화 알고리즘 M이 모든 인접 데이터셋 D, D'(한 레코드만 다름)와
모든 출력 집합 S에 대해 다음을 만족하면 ε-DP:

  Pr[M(D) ∈ S] ≤ e^ε × Pr[M(D') ∈ S]

  ε (프라이버시 예산): 작을수록 강한 프라이버시
    - ε ≈ 0: 완벽한 프라이버시 (데이터 무용)
    - ε = 1: 강한 프라이버시
    - ε = 10: 중간 수준
    - ε → ∞: 프라이버시 없음
```

**ε, δ-차분 프라이버시 (완화 버전):**

```
  Pr[M(D) ∈ S] ≤ e^ε × Pr[M(D') ∈ S] + δ

  δ: 확률적 허용 오차 (≈ 0에 가깝게 설정, 보통 1/n²)
  → Gaussian Mechanism 사용 시 필요
```

📢 **섹션 요약 비유**: DP는 마치 선거 출구 조사에서 응답자에게 "동전을 던져 앞면이 나오면 실제 투표를 말하고, 뒷면이면 무작위로 아무 후보나 말해라"는 규칙을 준다. 개인의 대답은 확인 불가하지만, 전체 집계는 통계적으로 정확하다.

## Ⅱ. 아키텍처 및 핵심 원리

### 2.1 DP 메커니즘 3종

#### Laplace Mechanism (라플라스 메커니즘)

```
수치형 쿼리에 적용:
  M(D) = f(D) + Lap(Δf / ε)

  Δf (전역 민감도): 한 레코드 변경 시 f 최대 변화량
  Lap(b): 평균 0, 스케일 b인 라플라스 분포 노이즈

예시: 평균 연봉 쿼리
  실제 평균: 4,500만원
  Δf = (최대 연봉 - 최소 연봉) / N = (2억 - 2천만) / 1,000 = 18만원
  ε = 1.0 → 노이즈 = Lap(18만) ≈ 약 ±20만원 오차
  공개 결과: 4,500만원 + 노이즈
```

#### Gaussian Mechanism (가우시안 메커니즘)

```
(ε, δ)-DP에 사용:
  M(D) = f(D) + N(0, σ²)
  σ = (Δ₂f × √(2 ln(1.25/δ))) / ε

  Δ₂f: L2 민감도
  → 딥러닝 그래디언트 클리핑에서 주로 사용
```

#### Exponential Mechanism (지수 메커니즘)

```
범주형 출력에 적용:
  비수치 결과 (예: 추천 아이템) 선택 시 사용
  품질 함수 q(D, r)이 높을수록 r 선택 확률 ↑
  Pr[M(D) = r] ∝ exp(ε × q(D, r) / 2Δq)
```

### 2.2 글로벌 DP vs 로컬 DP 아키텍처

```
┌──────────────────────────────────────────────────────────┐
│               글로벌 DP (Global DP)                        │
│                                                           │
│  사용자 A → 원시 데이터 ─┐                                │
│  사용자 B → 원시 데이터 ─┼→ 신뢰 집계 서버 → DP 노이즈 추가 → 공개 │
│  사용자 C → 원시 데이터 ─┘        ↑                       │
│                           서버를 신뢰해야 함!              │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│               로컬 DP (Local DP)                           │
│                                                           │
│  사용자 A → 개인 DP 노이즈 적용 → 랜덤화된 값 전송 ─┐     │
│  사용자 B → 개인 DP 노이즈 적용 → 랜덤화된 값 전송 ─┼→ 서버 │
│  사용자 C → 개인 DP 노이즈 적용 → 랜덤화된 값 전송 ─┘     │
│                                                           │
│  서버 신뢰 불필요! 하지만 정확도 더 낮음                    │
│  (노이즈를 개인이 추가하므로 집계 노이즈 더 큼)             │
└──────────────────────────────────────────────────────────┘
```

| 항목 | 글로벌 DP | 로컬 DP |
|:---|:---|:---|
| 신뢰 모델 | 중앙 서버 신뢰 필요 | 서버 신뢰 불필요 |
| 정확도 | 높음 (√N 배 더 정확) | 낮음 |
| 사용 예 | 내부 분석, 연구 | 스마트폰 데이터 수집 (Apple, Google) |
| 구현 복잡도 | 낮음 | 높음 |

### 2.3 DP 구현 스택 아키텍처

```
┌──────────────────────────────────────────────────┐
│             DP 데이터 파이프라인 아키텍처            │
│                                                  │
│  원시 데이터 소스                                  │
│       ↓                                          │
│  민감도 분석 (Sensitivity Calibration)             │
│  - 전역 민감도 Δf 계산                             │
│  - 프라이버시 예산 ε 설정                           │
│       ↓                                          │
│  DP 메커니즘 적용 (Laplace / Gaussian)             │
│       ↓                                          │
│  프라이버시 예산 트래커 (Privacy Accountant)         │
│  - Moments Accountant, RDP, zCDP                 │
│  - 누적 ε 추적 (composition theorem)              │
│       ↓                                          │
│  DP 처리된 데이터 공개 / 모델 학습                   │
└──────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 라플라스 메커니즘은 마치 성적표를 공개할 때 "실제 점수 ± 무작위 오차"를 공개하는 것이다. 개인의 점수는 정확히 알 수 없지만, 전체 평균은 여전히 유용하게 계산된다.

## Ⅲ. 비교 및 연결

### 3.1 DP 구현 사례 비교

| 기관 | 구현 | 메커니즘 | ε 값 |
|:---|:---|:---|:---|
| 구글 RAPPOR | Chrome 사용 통계 | 로컬 DP + 무작위 응답 | ε ≈ ln(3) ≈ 1.1 |
| Apple iOS | 이모지·단어 사용 통계 | 로컬 DP | ε = 4 ~ 8 |
| Facebook | 광고 측정 | 글로벌 DP | ε = 3 |
| 미국 Census Bureau | 2020 인구조사 | 글로벌 DP | ε = 17.14 |
| OpenDP (Harvard) | 오픈소스 DP 라이브러리 | 글로벌·로컬 DP | 사용자 설정 |

### 3.2 DP와 다른 프라이버시 기법 비교

| 기법 | 방법 | 강점 | 약점 |
|:---|:---|:---|:---|
| K-익명성 | 준식별자 일반화 | 직관적, 구현 쉬움 | 동질성·배경지식 공격 취약 |
| 차분 프라이버시 (DP) | 수학적 노이즈 추가 | 수학적 증명 가능 | 정확도 손실 |
| 동형 암호화 (HE) | 암호화 상태로 연산 | 데이터 노출 없음 | 속도 매우 느림 |
| 연방 학습 (FL) | 데이터 비이동 | 원본 데이터 중앙화 없음 | 그래디언트 공격 취약 |
| FL + DP | 가중치에 노이즈 추가 | 프라이버시 수학적 보장 | 정확도 vs 프라이버시 균형 |

### 3.3 프라이버시 예산 관리 (Privacy Budget Management)

```
여러 쿼리를 연속으로 수행할 때 ε 누적:

  단순 합성: ε_total = ε_1 + ε_2 + ... + ε_k
  → 비효율적 (빠르게 예산 소진)

  고급 합성 (Advanced Composition):
  ε_total = ε × √(2k × ln(1/δ))  (k번 쿼리 시)

  모먼츠 어카운턴트 (Moments Accountant):
  → DP-SGD(딥러닝 학습)에서 정밀한 ε 추적
  → 더 작은 ε 소비로 더 많은 학습 가능

실무 예산 전략:
  1. 데이터 탐색용: ε = 5~10 (낮은 프라이버시)
  2. 공개 통계용: ε = 1~3 (중간 프라이버시)
  3. 의료·민감 데이터: ε = 0.1~1 (높은 프라이버시)
```

📢 **섹션 요약 비유**: ε 프라이버시 예산은 마치 일일 통화 허용량과 같다. 전화를 많이 할수록(쿼리 많음) 하루 통화량(ε)이 소진되고, 다 쓰면 더 이상 통계를 공개할 수 없다. 예산을 아껴 쓰는 것이 관건이다.

## Ⅳ. 실무 적용 및 기술사 판단

### 4.1 DP-SGD (차분 프라이버시 확률적 경사하강법) - 딥러닝 적용

```
DP-SGD 알고리즘 (딥러닝 학습에 DP 적용):

  각 미니배치 학습 단계:
  1. 각 샘플의 그래디언트 g_i 계산
  2. 그래디언트 클리핑: g_i ← g_i / max(1, ||g_i||₂/C)
     (C: 클리핑 임계값, 전역 민감도 = C)
  3. 노이즈 추가: g_noisy = Σg_i + N(0, σ²C²I)
  4. 일반 SGD 업데이트 수행

프라이버시 회계:
  각 배치마다 소비된 ε를 Moments Accountant로 추적
  → 전체 학습 완료 시 총 (ε, δ) 계산·공개
```

### 4.2 실무 ε 설정 가이드라인

| 도메인 | 권장 ε | 이유 |
|:---|:---|:---|
| 의료 (진단 데이터) | 0.1 ~ 1.0 | HIPAA 수준 민감도 |
| 금융 (거래 기록) | 0.5 ~ 2.0 | 금융 규제 요건 |
| 모바일 텔레메트리 | 4 ~ 8 | 대규모 데이터로 노이즈 상쇄 |
| 통계 공개 (인구조사) | 10 ~ 20 | 집계 데이터, 정확도 중요 |
| 내부 분석 | 10 이상 | 내부 신뢰 환경 |

### 4.3 기술사 답안 핵심 포인트

```
DP 시스템 설계 시 필수 언급:
  ✓ ε-DP 수학적 정의 및 프라이버시 예산 개념
  ✓ 메커니즘 선택: 수치형 → Laplace, (ε,δ)-DP → Gaussian,
                   범주형 → Exponential
  ✓ 글로벌 DP vs 로컬 DP 선택 기준 (신뢰 모델)
  ✓ DP-SGD로 ML 모델 자체에 DP 적용 (FL + DP 연계)
  ✓ 프라이버시 예산 관리: Moments Accountant 사용
  ✓ ε vs 모델 정확도 트레이드오프 실험 결과 제시
  ✓ GDPR Art.89 (연구 목적 처리) 와의 연계
```

📢 **섹션 요약 비유**: DP-SGD는 마치 수학 시험 채점 결과를 발표할 때, 실제 점수에 무작위 오차를 넣어 공개하는 것과 같다. 개인 점수는 보호되지만, 반 전체의 학습 수준 파악(모델 학습)은 여전히 가능하다.

## Ⅴ. 기대효과 및 결론

### 5.1 DP 도입의 가치

| 효과 | 내용 |
|:---|:---|
| 수학적 프라이버시 보장 | 공격 모델에 무관한 증명 가능한 보호 |
| 데이터 공개 가능 | 민감 데이터도 DP 처리 후 공개·공유 가능 |
| 규제 강화 대응 | GDPR, CCPA 등 개인정보 처리 근거 제공 |
| 재식별 불가 | K-익명성과 달리 배경지식 공격 방어 |

### 5.2 DP vs 프라이버시 기법 종합 비교

```
┌────────────────────────────────────────────────────┐
│     프라이버시 보호 기법 선택 트리                    │
│                                                    │
│  데이터 공개/공유 필요한가?                          │
│    │                                               │
│    ├─→ 수치 통계 공개: Laplace / Gaussian DP        │
│    │                                               │
│    ├─→ ML 모델 학습: DP-SGD (+ FL 연계 고려)         │
│    │                                               │
│    ├─→ 범주형 공개: Exponential Mechanism           │
│    │                                               │
│    └─→ 모바일 데이터 수집: 로컬 DP (RAPPOR 류)       │
│                                                    │
│  신뢰 서버 없음 → 로컬 DP                           │
│  신뢰 서버 있음 → 글로벌 DP (더 높은 정확도)          │
└────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 차분 프라이버시는 마치 전국 시험 통계를 공개할 때, 각 학생 점수에 "약간의 운 요소"를 넣어 발표하는 방식이다. 개인을 특정할 수 없지만 전체 성적 분포는 충분히 파악할 수 있어, 교육 정책 수립에 활용할 수 있다.

### 👶 어린이를 위한 3줄 비유 설명

1. **차분 프라이버시**는 마치 "우리 반에서 안경 쓴 사람이 몇 명이에요?" 물음에 선생님이 정확한 숫자가 아닌 "아마도 5명쯤?"처럼 약간 틀린 숫자를 말해서, 특정 학생이 안경을 쓰는지 알 수 없게 하는 방법이에요.
2. **ε(엡실론)**은 "얼마나 많이 틀린 숫자를 말할 것인지"를 정하는 기준이에요—숫자가 작을수록 더 많이 틀리게 말해서 프라이버시를 더 잘 지키지만, 통계가 덜 유용해져요.
3. **로컬 DP**는 마치 선생님에게 대답하기 전에 각 학생이 스스로 동전을 던져 앞면이면 사실을, 뒷면이면 거짓을 말하게 해서, 선생님도 개인의 실제 대답을 알 수 없게 하는 방법이에요.
