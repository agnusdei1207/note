## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 옵티마이저(Optimizer)는 손실 함수(Loss Function)의 기울기(Gradient)를 이용해 모델 파라미터를 반복 갱신하는 알고리즘으로, 수렴 속도와 안정성의 균형이 핵심이다.
> 2. **가치**: Adam(Adaptive Moment Estimation)은 모멘텀(Momentum)의 방향 관성과 RMSProp의 적응 학습률을 결합하여 대부분의 딥러닝 작업에서 뛰어난 기본 선택지가 된다.
> 3. **판단 포인트**: 배치 크기(Batch Size)와 학습률(Learning Rate) 스케줄링 전략의 조합이 훈련 안정성과 일반화 성능을 결정짓는 실무의 핵심 변수이다.

---

## Ⅰ. 개요 및 필요성

딥러닝 모델 훈련은 손실 함수 L(θ)를 최소화하는 파라미터 θ를 찾는 최적화 문제다. 이를 위해 경사 하강법(Gradient Descent)이 사용된다.

**경사 하강법의 기본 갱신 규칙**

```
θ_{t+1} = θ_t - η · ∇L(θ_t)
```

여기서 η는 학습률(Learning Rate), ∇L은 손실 함수의 기울기다.

### 배치 유형 비교

| 구분 | 배치 GD (Batch GD) | SGD (Stochastic GD) | 미니배치 SGD (Mini-batch SGD) |
|:---|:---|:---|:---|
| 데이터 사용량 | 전체 데이터셋 | 샘플 1개 | N개 (32~512) |
| 기울기 정확도 | 높음 | 낮음(노이즈 큼) | 중간 |
| 메모리 | 많이 필요 | 적음 | 적당 |
| 수렴 속도 | 느림 | 빠르나 불안정 | 균형 우수 |
| GPU 효율 | 낮음 | 낮음 | 높음(벡터화) |

**핵심**: 실무에서는 미니배치 SGD가 표준이며, 배치 크기는 하드웨어 메모리와 일반화 성능을 고려해 선택한다.

📢 **섹션 요약 비유**: 경사 하강법은 안개 낀 산을 내려올 때 발밑 경사만 보고 걷는 것이다. 배치 GD는 GPS로 전체 지형을 파악하지만 느리고, SGD는 눈 감고 뛰어 빠르지만 위험하며, 미니배치는 짧게 살펴보고 빠르게 걷는 현실적 타협이다.

## Ⅱ. 아키텍처 및 핵심 원리

### 모멘텀(Momentum) — 관성 기반 최적화

이전 기울기 방향의 관성을 유지해 진동을 줄이고 수렴을 가속한다.

```
v_t   = β·v_{t-1} + (1-β)·∇L(θ_t)
θ_t+1 = θ_t - η·v_t
```

- β (모멘텀 계수): 보통 0.9
- 평탄한 방향은 가속, 진동 방향은 감쇠

### RMSProp — 적응 학습률(Adaptive Learning Rate)

각 파라미터별 최근 기울기 크기의 제곱 이동 평균으로 학습률을 조정한다.

```
s_t   = ρ·s_{t-1} + (1-ρ)·(∇L)²
θ_t+1 = θ_t - (η / √(s_t + ε)) · ∇L
```

- 자주 업데이트되는 파라미터: 학습률 ↓
- 드물게 업데이트되는 파라미터: 학습률 ↑

### Adam (Adaptive Moment Estimation) — 모멘텀 + RMSProp

```
m_t = β₁·m_{t-1} + (1-β₁)·∇L          ← 1차 모멘텀 (방향)
v_t = β₂·v_{t-1} + (1-β₂)·(∇L)²       ← 2차 모멘텀 (크기)

m̂_t = m_t / (1 - β₁ᵗ)                  ← 편향 보정 (Bias Correction)
v̂_t = v_t / (1 - β₂ᵗ)                  ← 편향 보정

θ_t+1 = θ_t - η · m̂_t / (√v̂_t + ε)
```

기본 하이퍼파라미터: η=0.001, β₁=0.9, β₂=0.999, ε=1e-8

### 옵티마이저 비교 다이어그램

```
손실 곡면 (Loss Landscape)
     ┌─────────────────────────────────────────┐
     │ ↘↘↘  SGD: 노이즈 많은 지그재그 경로  ↙↙ │
     │  ↘──────────────────────────────→  글로벌 최솟값 │
     │   모멘텀: 관성으로 진동 감소              │
     │    ↘═══════════════════════→           │
     │     Adam: 적응적, 빠르고 안정적 수렴     │
     │      ↘━━━━━━━━━━━━━━━━━━━→ ★          │
     └─────────────────────────────────────────┘
     
옵티마이저 성능 요약
┌──────────────┬───────────┬───────────┬───────────┐
│  옵티마이저  │  수렴속도 │  안정성   │  메모리   │
├──────────────┼───────────┼───────────┼───────────┤
│ SGD          │   느림    │   낮음    │   낮음    │
│ SGD+Momentum │   중간    │   중간    │   낮음    │

│ Adam         │   빠름    │   높음    │   높음    │

└──────────────┴───────────┴───────────┴───────────┘
```

📢 **섹션 요약 비유**: Adam은 경험 많은 등산가다. 모멘텀으로 발걸음의 방향 관성을 유지하고(어디로 가던 중이었는지 기억), RMSProp으로 가파른 곳은 조심스럽게 평탄한 곳은 빠르게 걷는다. 두 전략을 동시에 쓰니 대부분의 지형에서 최적이다.

## Ⅲ. 비교 및 연결

### 학습률 스케줄링(Learning Rate Scheduling) 전략

| 전략 | 방법 | 적용 시점 |
|:---|:---|:---|
| Step Decay | 일정 에폭마다 학습률 감소 | 단순 이미지 분류 |
| Cosine Annealing | 코사인 함수로 주기적 감소 | Vision Transformer |
| Warm-up + Decay | 초반 증가 후 점진적 감소 | BERT, GPT 사전 학습 |
| Cyclical LR | 주기적 증감 반복 | 안장점(Saddle Point) 탈출 |
| One-Cycle Policy | 단일 주기 급격한 증가 후 감소 | 빠른 수렴 필요 시 |

### 학습률과 배치 크기의 관계 (Linear Scaling Rule)

```
배치 크기를 k배 키우면 → 학습률도 k배 증가 (선형 스케일링)
이유: 기울기 추정 분산이 1/k 감소 → 더 큰 보폭 가능
```

**주의**: 매우 큰 배치(>8K)는 일반화 성능 저하 → Warm-up 필수

📢 **섹션 요약 비유**: 학습률 스케줄링은 자동차 속도 조절과 같다. 처음 도로(Warm-up)는 천천히, 고속도로 구간은 빠르게, 복잡한 교차로(수렴 근처)에서는 다시 천천히 접근한다.

## Ⅳ. 실무 적용 및 기술사 판단

### AdamW — 가중치 감쇠(Weight Decay) 분리

일반 Adam은 L2 정규화를 기울기에 흡수하지만, AdamW는 가중치 감쇠를 파라미터 갱신에 직접 적용한다.

```
θ_t+1 = θ_t - η·(m̂_t / (√v̂_t + ε)) - η·λ·θ_t
```

- 트랜스포머(Transformer) 계열 모델에서 표준 옵티마이저
- λ (가중치 감쇠): 0.01~0.1

### 실무 선택 가이드

| 상황 | 추천 옵티마이저 | 이유 |
|:---|:---|:---|
| 일반 딥러닝 | Adam / AdamW | 기본 성능 안정적 |
| 이미지 분류(대규모) | SGD + Momentum + LR Decay | 최종 정확도 우수 |
| NLP / Transformer | AdamW + Warm-up | L2 정규화 효과 분리 |
| 강화 학습 | Adam / RMSProp | 비정상 데이터에 강인 |
| 초경량 모델 | SGD | 메모리 절약 |

### 기울기 클리핑(Gradient Clipping)

```python
# 기울기 폭발 방지
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

RNN(Recurrent Neural Network)·트랜스포머 학습 시 필수적으로 적용한다.

📢 **섹션 요약 비유**: 옵티마이저 선택은 공사 현장 장비 선택과 같다. 정밀 작업(NLP)에는 다기능 공구(AdamW)를, 오래된 표준 작업(이미지 분류)에는 검증된 단순 공구(SGD+Momentum)를, 빠른 실험에는 만능 드릴(Adam)을 쓴다.

## Ⅴ. 기대효과 및 결론

### 최적화 알고리즘 발전사

```
Gradient Descent (1847)
      ↓
SGD (Stochastic GD) — 노이즈를 정규화 효과로 활용
      ↓
Momentum (1986) — Polyak, 관성 도입
      ↓
AdaGrad (2011) — 적응 학습률 최초
      ↓
RMSProp (2012) — Hinton, 이동 평균 도입
      ↓
Adam (2014) — Kingma & Ba, 모멘텀+RMSProp
      ↓
AdamW (2019) — Loshchilov, 가중치 감쇠 분리
      ↓
Lion (2023) — Google, 사인 기반 업데이트
```

### 기술사 시험 핵심 포인트

1. **Adam 수식** 완전히 기술: 1차·2차 모멘텀, 편향 보정
2. **배치 크기-학습률 관계**: Linear Scaling Rule 명시
3. **옵티마이저 선택 근거**: 태스크별 추천과 이유
4. **학습률 스케줄링** 종류와 적용 시점
5. **기울기 클리핑** 필요성 (RNN, 트랜스포머)

📢 **섹션 요약 비유**: 좋은 옵티마이저는 마라톤 코치와 같다. 선수(파라미터)가 어디로 달려야 하는지 방향을 잡고(모멘텀), 오르막과 내리막에 따라 페이스를 조절하며(적응 학습률), 무리하지 않도록 에너지를 관리한다(가중치 감쇠). 최종 목표는 최단 시간에 결승선(전역 최솟값)에 도달하는 것이다.

### 👶 어린이를 위한 3줄 비유 설명
1. 경사 하강법은 눈 감고 언덕을 내려올 때 발이 닿는 기울기를 느끼며 한 걸음씩 내딛는 것이야.
2. Adam 옵티마이저는 "어디로 가던 중이었지?"(모멘텀)와 "이 길이 얼마나 가파르지?"(RMSProp)를 동시에 기억하는 똑똑한 탐험가야.
3. 학습률 스케줄링은 처음엔 천천히 주변을 살피고, 익숙해지면 빠르게 달리다가, 목적지 근처에서 다시 천천히 걷는 여행 계획이야.
