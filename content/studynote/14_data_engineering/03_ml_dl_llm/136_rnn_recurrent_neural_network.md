## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이전 시점의 은닉 상태(Hidden State)를 다음 시점 입력으로 재사용하는 순환 구조로, 시퀀스 데이터의 문맥을 유지한다.
> 2. **가치**: 자연어, 음성, 주가 같은 순서 의존적 데이터를 처리하는 최초의 효과적인 딥러닝 구조이다.
> 3. **판단 포인트**: 장기 의존성(Long-term Dependency) 학습이 어렵고 기울기 소실(Vanishing Gradient)이 심각하다 — 이를 해결하기 위해 LSTM (Long Short-Term Memory)과 GRU (Gated Recurrent Unit)가 등장했다.

## Ⅰ. 개요 및 필요성

일반 신경망(MLP, CNN)은 각 입력을 독립적으로 처리해 순서 정보를 무시한다. "나는 밥을 먹는다"와 "밥은 나를 먹는다"는 단어가 같아도 의미가 다르듯, 순서 정보는 언어·음성·시계열에서 핵심이다.

RNN (Recurrent Neural Network)은 각 시점(Time Step)에서 현재 입력 xₜ와 이전 은닉 상태 hₜ₋₁을 함께 처리해 순서 문맥을 누적한다.

**핵심 수식**
- hₜ = tanh(Wₕ · hₜ₋₁ + Wₓ · xₜ + b)
- yₜ = Wᵧ · hₜ + bᵧ

📢 **섹션 요약 비유**: RNN은 일기를 쓸 때 어제 일을 기억하면서 오늘 일을 추가하는 연속 기록 방식이다.

## Ⅱ. 아키텍처 및 핵심 원리

| 구성 요소 | 설명 | 역할 |
|:---|:---|:---|
| 입력 xₜ | 현재 시점 입력 | 단어 임베딩, 센서 값 등 |
| 은닉 상태 hₜ | 현재 문맥 벡터 | 이전 시점 정보 누적 |
| 출력 yₜ | 현재 시점 예측값 | 다음 단어, 감정 분류 등 |
| Wₕ, Wₓ, Wᵧ | 학습 가중치 행렬 | 모든 시점에서 공유 |
| BPTT | Backpropagation Through Time | 시간 축으로 역전파 |

```
[RNN 펼친 구조 (Unrolled RNN)]

       x₁          x₂          x₃          x₄
       │            │            │            │
  h₀  ▼       h₁  ▼       h₂  ▼       h₃  ▼
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│  RNN 셀 │─▶│  RNN 셀 │─▶│  RNN 셀 │─▶│  RNN 셀 │
└─────────┘  └─────────┘  └─────────┘  └─────────┘
       │            │            │            │
       y₁           y₂           y₃           y₄

역전파 방향: y₄ → y₃ → y₂ → y₁ (BPTT)
기울기가 역방향으로 전달될수록 소실 위험 증가
```

**BPTT (Backpropagation Through Time)**
- 시간 축을 따라 역전파 수행
- 시퀀스 길이 T만큼 기울기를 역방향으로 전달
- T가 크면 기울기 소실 or 폭발 심화

📢 **섹션 요약 비유**: RNN은 전화 통화 내용을 메모하면서 대화를 이어가는 비서인데, 통화가 길어질수록 초반 내용을 잊어버린다.

## Ⅲ. 비교 및 연결

| 항목 | RNN | LSTM | Transformer |
|:---|:---|:---|:---|
| 장기 의존성 | ❌ 취약 | ✅ 강함 | ✅ 매우 강함 |
| 병렬 처리 | ❌ 순차 처리 | ❌ 순차 처리 | ✅ 완전 병렬 |
| 파라미터 수 | 적음 | 4배 | 매우 많음 |
| 실무 사용 | 단순 시계열 | 음성 인식, NLP | 대형 언어 모델 |

**RNN 변형 구조**
- 양방향 RNN (Bidirectional RNN): 정방향 + 역방향 은닉 상태 결합 → 문장 전체 문맥 활용
- 다층 RNN (Stacked RNN): 여러 층 쌓아 표현력 향상
- Encoder-Decoder RNN: Seq2Seq 번역 모델의 기반

📢 **섹션 요약 비유**: RNN은 단기 기억만 있는 사람처럼 방금 전 일은 기억하지만, 지난달 일은 잊어버린다.

## Ⅳ. 실무 적용 및 기술사 판단

**적용 사례**
- 주가 예측, 이상 탐지 (단기 시계열)
- 간단한 감정 분석 (Short Sequence NLP)
- 음성 인식 전처리 단계 (LSTM으로 대체 추세)

**한계 및 대안 선택 기준**
- 시퀀스 길이 > 50: LSTM 또는 GRU 사용
- 대용량 텍스트: Transformer 계열 사용
- 실시간 스트리밍: RNN/LSTM이 추론 지연 낮음

**기술사 출제 포인트**
- "RNN의 기울기 소실 문제 원인과 해결 방안을 설명하시오"
- "BPTT와 일반 역전파의 차이점을 설명하시오"

📢 **섹션 요약 비유**: RNN은 GPS 없이 이전 도로 기억에만 의존해 운전하는 것인데, 길이 길어질수록 길을 잃기 쉽다.

## Ⅴ. 기대효과 및 결론

RNN은 시퀀스 처리 딥러닝의 출발점이었다. 기울기 소실 문제로 긴 시퀀스에는 한계가 있지만, 짧은 시계열이나 임베딩 처리에는 여전히 활용된다. LSTM → Attention → Transformer로 이어지는 발전의 역사적 기반이다.

📢 **섹션 요약 비유**: RNN은 최초의 '기억하는 신경망'이었다. 완벽하지 않지만, 이 아이디어가 없었다면 Transformer도 없었다.

### 📌 관련 개념 맵
| 관계 | 개념 | 설명 |
|:---|:---|:---|
| 핵심 구조 | 은닉 상태 (Hidden State) | 이전 시점 정보 보존 |
| 학습 알고리즘 | BPTT | 시간 축 역전파 |
| 문제 | 기울기 소실 (Vanishing Gradient) | 긴 시퀀스 학습 불가 |
| 발전 | LSTM | 게이트로 장기 의존성 해결 |
| 발전 | GRU (Gated Recurrent Unit) | LSTM 경량화 |
| 발전 | Transformer | 어텐션으로 병렬 처리 |

### 👶 어린이를 위한 3줄 비유 설명
1. RNN은 이야기를 들으면서 앞 내용을 기억해 다음 줄을 예측하는 스토리텔러예요.
2. 하지만 이야기가 너무 길어지면 초반 내용을 잊어버려요 — 그게 기울기 소실이에요.
3. 그래서 LSTM이라는 더 좋은 기억력을 가진 스토리텔러가 등장했어요.
