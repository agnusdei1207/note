## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 할루시네이션(Hallucination)은 LLM이 사실이 아닌 내용을 자신감 있게 생성하는 현상이며, RAG (Retrieval Augmented Generation, 검색 증강 생성)는 실시간 외부 지식 검색으로 이를 억제한다.
> 2. **가치**: RAG는 모델 재학습 없이 최신 정보와 도메인 지식을 주입해 환각을 30~60% 감소시키고, 생성 근거를 검색 문서로 추적 가능하게 한다.
> 3. **판단 포인트**: RAG는 최신성·변경 빈도 높은 지식에 효과적이고, 파인튜닝은 스타일·포맷·깊은 도메인 지식 내재화에 적합하다 — 두 가지를 결합하는 것이 최선이다.

## Ⅰ. 개요 및 필요성

LLM은 학습 데이터의 패턴으로 텍스트를 생성한다. 알지 못하는 내용을 "모른다"고 말하기보다 그럴듯한 내용을 지어내는 경향이 있다 — 이것이 할루시네이션이다.

**할루시네이션 유형**
- 사실 오류: "세종대왕은 1443년에 한글을 창제했다" → 맞음 / "아인슈타인은 1922년 노벨 물리학상" → 틀림 (1921년)
- 존재하지 않는 출처: 가상의 논문·뉴스 기사 인용
- 지식 단절(Knowledge Cutoff): 학습 이후 사건 미반영
- 내적 불일치: 같은 대화에서 모순된 답변

📢 **섹션 요약 비유**: 할루시네이션은 시험지를 모두 자신 있게 채웠지만, 모르는 문제를 그럴싸하게 지어낸 학생의 답안이다.

## Ⅱ. 아키텍처 및 핵심 원리

### RAG 구조

| 컴포넌트 | 역할 | 기술 |
|:---|:---|:---|
| 문서 청킹 | 문서를 검색 가능한 단위로 분할 | 고정 크기, 의미 단위 분할 |
| 임베딩 모델 | 텍스트 → 벡터 변환 | text-embedding-3, BGE, E5 |
| 벡터 데이터베이스 | 벡터 저장 및 유사도 검색 | Pinecone, Weaviate, Chroma |
| 리트리버 (Retriever) | 쿼리 관련 문서 검색 | Top-K 유사도 검색 |
| 리랭커 (Reranker) | 검색 결과 재순위 | Cross-Encoder, Cohere Rerank |
| LLM 생성기 (Generator) | 검색 결과 기반 답변 생성 | GPT-4, LLaMA |

```
[RAG 파이프라인]

[인덱싱 단계 - Offline]
문서(PDF, DB, 웹)
      │
  청킹 (Chunking)
  텍스트 조각 512~1024 토큰
      │
  임베딩 모델
  (텍스트 → 벡터)
      │
  벡터 DB에 저장
  (Pinecone, Chroma 등)

[쿼리 단계 - Online]
사용자 질문
      │
  쿼리 임베딩
      │
  ┌───▼───────────────────┐
  │  벡터 DB 유사도 검색   │
  │  Top-K 관련 청크 반환  │
  └───────────────────────┘
      │
  (선택) 리랭킹
      │
  ┌───▼───────────────────────┐
  │  LLM 프롬프트 구성         │
  │  "다음 문서를 참고하여...  │
  │  [검색된 문서 청크들]      │
  │  질문: [사용자 질문]"      │
  └───────────────────────────┘
      │
  LLM 생성 → 근거 있는 답변
```

**RAG 고급 패턴**

| 패턴 | 설명 |
|:---|:---|
| Naive RAG | 기본 검색-생성 |
| Advanced RAG | 쿼리 재작성, 청크 최적화, 리랭킹 |
| Modular RAG | 각 단계를 모듈로 교체 가능 |
| Self-RAG | LLM이 검색 필요 여부 스스로 판단 |
| HyDE (Hypothetical Document Embeddings) | 가상 문서 생성 후 검색 |

📢 **섹션 요약 비유**: RAG는 시험 전날 교과서를 암기하는 대신, 시험 중에 참고서를 찾아보는 오픈북 시험 방식이다.

## Ⅲ. 비교 및 연결

| 항목 | 파인튜닝 | RAG | RAG + 파인튜닝 |
|:---|:---|:---|:---|
| 최신 정보 | ❌ 재학습 필요 | ✅ 실시간 | ✅ |
| 도메인 깊이 | ✅ 높음 | 보통 | ✅ 높음 |
| 환각 감소 | 보통 | ✅ 높음 | ✅ 최고 |
| 추적 가능성 | ❌ | ✅ 출처 제공 | ✅ |
| 비용 | 중~높음 (초기) | 중간 (운영) | 높음 |

**RAG 실패 원인**
1. 검색 실패: 관련 문서가 검색에 포함되지 않음 → 임베딩 품질, 청킹 전략 개선
2. 컨텍스트 품질: 검색된 문서가 질문과 무관 → 리랭킹 추가
3. LLM 무시: 검색 결과를 무시하고 기존 지식 사용 → 프롬프트 강화

📢 **섹션 요약 비유**: RAG에서 검색 품질이 나쁘면, 오답 참고서를 보면서 시험을 치르는 것과 같다.

## Ⅳ. 실무 적용 및 기술사 판단

**RAG 구현 스택**
- Python: LangChain, LlamaIndex
- 임베딩: OpenAI text-embedding-3-small, BGE-M3
- 벡터 DB: Chroma (로컬), Pinecone (클라우드), PostgreSQL + pgvector
- LLM: GPT-4o, LLaMA-3, Claude-3

**청킹 전략**
- 고정 크기 (512 토큰, 100 오버랩): 단순, 적용 쉬움
- 의미 단위 청킹: 문단, 섹션 경계 기준
- 계층적 청킹: 요약 + 세부 내용 인덱싱

**기술사 출제 포인트**
- "LLM 할루시네이션의 원인과 RAG가 이를 완화하는 메커니즘을 설명하시오"
- "RAG의 인덱싱 파이프라인과 쿼리 파이프라인을 구분해 설명하시오"

📢 **섹션 요약 비유**: RAG는 AI에게 "모를 때는 도서관에서 찾아봐"라고 가르치는 것이다. 찾아본 근거가 있으니 훨씬 신뢰할 수 있다.

## Ⅴ. 기대효과 및 결론

RAG는 기업 환경에서 가장 빠르게 채택되는 LLM 활용 패턴이다. 내부 문서, 제품 매뉴얼, 법규집을 인덱싱해 정확한 답변을 제공하는 AI 어시스턴트를 구축할 수 있다. RAG는 환각 감소, 최신 정보 반영, 출처 제공이라는 세 가지 실용적 이점으로 기업 AI 신뢰성을 높인다.

📢 **섹션 요약 비유**: RAG는 AI의 거짓말쟁이 습관을 교정하는 가장 실용적인 방법이다. "모르면 찾아봐"라는 단순한 원칙이 AI를 신뢰할 수 있게 만든다.

### 👶 어린이를 위한 3줄 비유 설명
1. 할루시네이션은 AI가 모르는 답을 그럴듯하게 꾸며내는 것 — 자신 있게 틀리는 거예요.
2. RAG는 모를 때 도서관에서 관련 책을 찾아 읽고 답하도록 해서, 근거 있는 답변을 만들어요.
3. 덕분에 AI가 틀릴 때도 "어느 책에서 봤는지" 알 수 있어서 확인이 가능해요.
