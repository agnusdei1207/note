## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프롬프트 엔지니어링(Prompt Engineering)은 모델 가중치를 변경하지 않고 입력 지시 설계만으로 LLM 출력 품질을 극대화하는 기술이다.
> 2. **가치**: CoT (Chain of Thought)은 중간 추론 과정을 명시해 수학 문제·논리 추론 정확도를 2~4배 높이고, Few-Shot은 예시 2~10개로 태스크를 즉시 학습시킨다.
> 3. **판단 포인트**: 파인튜닝이 필요 없어 비용이 매우 낮지만, 모델 능력의 한계를 넘지 못하므로 복잡한 전문 태스크에는 RAG (Retrieval Augmented Generation)나 파인튜닝이 필요하다.

## Ⅰ. 개요 및 필요성

동일한 모델도 프롬프트에 따라 결과가 극적으로 다르다. "5더하기 3은?"과 "수학 선생님처럼 풀이 과정을 보여주면서 5더하기 3을 설명해주세요"는 완전히 다른 출력을 만든다.

프롬프트 엔지니어링은 파인튜닝 없이 기존 LLM에서 최대 성능을 이끌어내는 기술로, API 비용 절감과 빠른 프로토타이핑에 핵심이다.

**프롬프트 요소**
- 역할 지정 (Role): "당신은 X 전문가입니다"
- 지시 (Instruction): 구체적인 작업 명세
- 맥락 (Context): 배경 정보, 제약 조건
- 예시 (Examples): Few-Shot 사례
- 출력 형식 (Format): JSON, 마크다운 등

📢 **섹션 요약 비유**: 프롬프트 엔지니어링은 같은 요리사(모델)에게 어떻게 주문하느냐에 따라 전혀 다른 요리가 나오게 하는 주문 기술이다.

## Ⅱ. 아키텍처 및 핵심 원리

| 기법 | 설명 | 효과 | 적합 태스크 |
|:---|:---|:---|:---|
| Zero-Shot | 예시 없이 지시만 | 기본 | 단순 태스크 |
| Few-Shot | 2~10개 예시 포함 | 높음 | 특정 포맷/스타일 |
| CoT (Chain of Thought) | 추론 과정 명시 | 매우 높음 | 수학, 논리, 복잡 추론 |
| Zero-Shot CoT | "단계적으로 생각해봐" | 높음 | CoT 데이터 없을 때 |
| Self-Consistency | CoT 여러 번 실행 후 다수결 | 최고 | 정확도 최우선 |
| Tree of Thought (ToT) | 추론 경로 트리 탐색 | 최고 | 복잡 문제 해결 |
| ReAct | 추론 + 행동(도구 호출) | 높음 | AI 에이전트 |

```
[CoT vs Zero-Shot CoT 비교]

일반 프롬프트:
Q: 10명이 5분에 5개를 만들면, 100명이 100개를 만드는 데 몇 분?
A: 100분 (❌ 오답)

CoT (Few-Shot):
예시) Q: 3명이 3분에 3개 → 5명이 5개 만드는 데 몇 분?
      풀이) 1명이 3분에 1개 → 5명이 3분에 5개 → 3분 (✅)
Q: 10명이 5분에 5개...
A: 1명이 5분에 0.5개 → 10명이 5분에 5개이므로
   100개를 100명이 만들면 1명당 1개, 5분에 0.5개이므로 10분 (✅)

Zero-Shot CoT:
"단계적으로 생각해봐 (Let's think step by step)"
→ 모델이 자동으로 추론 단계 생성

[고급 프롬프트 기법 - ToT]

        문제
          │
    ┌─────┼─────┐
 경로A    경로B   경로C
   │       │       │
  평가    평가    평가
  (OK)   (❌)   (OK)
   │               │
  확장             확장
   │               │
  결론1           결론2
          │
      최선 선택
```

📢 **섹션 요약 비유**: CoT는 수학 시험에서 답만 쓰지 않고 풀이 과정을 보여주게 하는 것이다. AI도 과정을 쓰면서 실수가 줄어든다.

## Ⅲ. 비교 및 연결

| 항목 | 프롬프트 엔지니어링 | 파인튜닝 | RAG |
|:---|:---|:---|:---|
| 비용 | 매우 낮음 | 중~높음 | 중간 |
| 속도 | 즉시 | 수 시간~일 | 즉시 (인덱싱 후) |
| 최신 지식 | ❌ | ❌ | ✅ |
| 도메인 특화 | 제한적 | 높음 | 높음 |
| 모델 수정 | ❌ 없음 | ✅ 가중치 변경 | ❌ 없음 |

**시스템 프롬프트 (System Prompt)**
- ChatGPT API의 역할 정의
- 전체 대화에 적용되는 영구 지시
- 예: "당신은 친절한 한국어 고객 서비스 담당자입니다. 항상 존댓말을 사용하세요."

📢 **섹션 요약 비유**: 프롬프트 엔지니어링은 AI에게 새 스킬을 가르치는 게 아니라, 이미 가진 스킬을 올바르게 발휘하도록 안내하는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

**프롬프트 작성 베스트 프랙티스**
1. 구체적인 역할 부여: "당신은 10년 경력의 데이터 엔지니어입니다"
2. 명확한 출력 형식: "JSON 형식으로 반환하세요: {key: value}"
3. 제약 조건 명시: "3문장 이내로", "전문 용어 없이"
4. 예시 포함 (Few-Shot): 원하는 입출력 형태 2~3개 제시
5. CoT 유도: "단계적으로 생각하여 답변하세요"

**프롬프트 인젝션 보안 주의**
- 악의적 사용자가 프롬프트를 통해 시스템 프롬프트를 무효화하거나 유해 콘텐츠를 유도하는 공격
- 입력 검증, 출력 필터링으로 방어

**기술사 출제 포인트**
- "Chain of Thought 프롬프팅이 LLM의 추론 능력을 향상시키는 원리를 설명하시오"
- "Few-Shot Learning과 Zero-Shot Learning의 차이를 설명하시오"

📢 **섹션 요약 비유**: 좋은 프롬프트는 직원에게 업무 지시서를 잘 쓰는 것과 같다. 명확할수록, 예시가 있을수록 결과가 좋다.

## Ⅴ. 기대효과 및 결론

프롬프트 엔지니어링은 파인튜닝 없이 LLM 성능을 극대화하는 저비용 고효율 방법이다. CoT, Self-Consistency, ToT 등 고급 기법으로 GPT-4 수준 모델에서 전문가에 근접한 추론이 가능하다. 프롬프트 최적화만으로 기업 생산성을 30~50% 향상시킨 사례들이 보고되고 있다.

📢 **섹션 요약 비유**: 프롬프트 엔지니어링은 같은 도구를 더 잘 사용하는 기술이다. 도구를 바꾸지 않고도 전문가와 초보자의 결과 차이를 만든다.

### 👶 어린이를 위한 3줄 비유 설명
1. 프롬프트 엔지니어링은 AI에게 "이렇게 이렇게 해줘"라고 잘 부탁하는 방법이에요.
2. CoT는 "답만 말하지 말고 어떻게 풀었는지 설명하면서 답해"라고 하면 AI가 더 정확하게 풀어요.
3. Few-Shot은 "이런 문제는 이렇게 답하더라"는 예시를 2~3개 보여주면, AI가 그 방식을 따라 해요.
