## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 양자화(Quantization)는 모델 가중치/활성화를 고정밀도(FP32/FP16)에서 저정밀도(INT8, INT4)로 변환해 메모리와 연산 비용을 4~8배 절감한다.
> 2. **가치**: 70B LLM을 FP16으로 추론하면 A100 2개가 필요하지만, 4bit 양자화 시 단일 RTX 3090에서 가능해 온디바이스(On-Device) AI를 실현한다.
> 3. **판단 포인트**: PTQ (Post-Training Quantization)는 재학습 없이 즉시 적용 가능하지만 정확도 손실이 있고, QAT (Quantization-Aware Training)는 성능을 유지하지만 재학습이 필요하다.

## Ⅰ. 개요 및 필요성

LLM의 폭발적 성장으로 추론 비용과 메모리가 문제가 되었다. GPT-3(175B)의 FP32 가중치는 700GB — 이를 4bit로 양자화하면 87.5GB로 줄어든다.

양자화는 딥러닝 모델을 엣지 디바이스(스마트폰, IoT, 임베디드)에 배포하거나, 단일 GPU로 대형 모델을 추론하는 핵심 기술이다.

**메모리 계산 (LLM)**
- FP32: 파라미터 수 × 4 bytes
- FP16/BF16: 파라미터 수 × 2 bytes
- INT8: 파라미터 수 × 1 byte
- INT4/NF4: 파라미터 수 × 0.5 bytes

📢 **섹션 요약 비유**: 양자화는 고화질 사진을 적당한 화질로 압축하는 것이다. 용량은 크게 줄지만, 내용은 거의 그대로다.

## Ⅱ. 아키텍처 및 핵심 원리

| 양자화 방식 | 비트 수 | 메모리 절감 | 정확도 손실 | 사용 시나리오 |
|:---|:---|:---|:---|:---|
| FP32 → FP16 | 16bit | 50% | 거의 없음 | 기본 추론 |
| FP32 → INT8 | 8bit | 75% | 소량 | 서버 추론 최적화 |
| FP32 → INT4 (GPTQ) | 4bit | 87.5% | 중간 | 소비자 GPU |
| FP32 → NF4 (QLoRA) | 4bit | 87.5% | 최소화 | 파인튜닝 |
| FP32 → INT2 | 2bit | 93.75% | 큰 손실 | 연구 단계 |

```
[양자화 과정]

FP32 가중치 벡터:
[0.731, -0.124, 0.892, -0.467, ...]

INT8 양자화:
  scale = (max - min) / 255
  zero_point = -min / scale
  q(w) = round(w / scale) + zero_point

양자화 결과:
[187, 95, 228, 60, ...]  (INT8)

역양자화 (추론 시):
  w_approx = (q - zero_point) × scale
→ [0.730, -0.125, 0.891, -0.468, ...]  (근사값)

[QLoRA 동작 구조]

4bit NF4 양자화 베이스 모델
┌────────────────────────────────┐
│  W₀ (NF4, 동결, 4bit)          │
│  + BnB(bitsandbytes) 역양자화  │
└────────────────────────────────┘
          +
LoRA 어댑터 (BF16, 학습 가능)
┌────────────────────────────────┐
│  A (r×k), B (d×r)             │
│  학습 시 BF16 계산             │
└────────────────────────────────┘
→ 70B 모델 파인튜닝 = 48GB GPU 1개
```

**양자화 알고리즘**

| 알고리즘 | 설명 | 주요 라이브러리 |
|:---|:---|:---|
| GPTQ | 가중치 단위 최적 양자화, Post-Training | AutoGPTQ |
| AWQ (Activation-aware Weight Quantization) | 활성화 크기 고려 중요 채널 보호 | AutoAWQ |
| bitsandbytes | QLoRA 파인튜닝 지원, NF4 형식 | HuggingFace 통합 |
| GGUF (GGML) | llama.cpp 포맷, CPU 추론 지원 | Ollama, LM Studio |

📢 **섹션 요약 비유**: INT8 양자화는 소수점 대신 정수로만 계산하는 것이다. 약간 덜 정확하지만 8배 빠른 계산이 가능하다.

## Ⅲ. 비교 및 연결

| 항목 | PTQ (사후 양자화) | QAT (양자화 인식 학습) |
|:---|:---|:---|
| 학습 필요 | ❌ 없음 | ✅ 재학습 |
| 구현 복잡도 | 낮음 | 높음 |
| 정확도 | 약간 손실 | 높음 (FP32 근접) |
| 적용 시점 | 학습 후 | 학습 중 |
| 적합 모델 | LLM (GPTQ, AWQ) | CNN, 소형 모델 |

**온디바이스 AI 추론 스택**
- 스마트폰: ONNX Runtime, TensorFlow Lite, CoreML (Apple)
- PC/노트북: llama.cpp + GGUF (CPU), Ollama
- IoT/임베디드: TensorRT, ONNX INT8

📢 **섹션 요약 비유**: PTQ는 완성된 케이크를 작은 접시로 나누는 것, QAT는 처음부터 작은 접시에 맞게 구운 케이크다. 맛은 후자가 더 좋다.

## Ⅳ. 실무 적용 및 기술사 판단

**양자화 선택 가이드**
- 서버 배포, 성능 우선: AWQ INT4 + vLLM 서빙
- 단일 GPU 파인튜닝: QLoRA (NF4 + LoRA)
- 엣지/CPU 추론: GGUF Q4_K_M (llama.cpp)
- 모바일 온디바이스: TFLite INT8 또는 CoreML

**정확도 영향 실측 (LLaMA-2 7B 기준)**
- FP16: MMLU 45.3
- GPTQ INT4: MMLU 44.8 (1% 손실)
- GGUF Q4_K_M: MMLU 44.5 (2% 손실)

**기술사 출제 포인트**
- "양자화의 원리와 PTQ/QAT의 차이를 설명하시오"
- "QLoRA가 메모리를 절감하는 방법과 파인튜닝 정확도를 유지하는 방식을 설명하시오"

📢 **섹션 요약 비유**: 양자화는 AI 모델을 스마트폰에 탑재하는 '압축 기술'이다. 잘 압축하면 성능 손실 없이 어디서나 사용할 수 있다.

## Ⅴ. 기대효과 및 결론

양자화는 LLM의 에너지 소비와 배포 비용을 획기적으로 낮춘다. 4bit 양자화를 통해 추론 속도는 2~4배 향상되고, 메모리는 75% 이상 절감된다. 온디바이스 AI(스마트폰, 노트북에서 실행되는 로컬 LLM)는 개인 정보 보호와 오프라인 활용을 가능하게 한다.

📢 **섹션 요약 비유**: 양자화는 AI를 슈퍼컴퓨터에서 해방시켜 손 안의 스마트폰에서 실행할 수 있게 하는 기술이다.
