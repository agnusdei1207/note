## 핵심 인사이트 (3줄 요약)
> 1. **본질**: PEFT (Parameter-Efficient Fine-Tuning)는 전체 모델 파라미터의 0.1~1%만 학습해 전체 파인튜닝에 근접한 성능을 달성하는 기법의 총칭이다.
> 2. **가치**: LoRA (Low-Rank Adaptation)는 가중치 행렬의 업데이트를 저랭크 행렬 곱으로 근사해, VRAM을 80% 이상 절감하면서 소비자급 GPU로 대형 모델 파인튜닝을 가능하게 한다.
> 3. **판단 포인트**: LoRA rank(r)는 작을수록 효율적이고 클수록 표현력이 높다; r=8~16이 대부분의 실무에서 최적이며, QLoRA는 양자화(Quantization)와 결합해 4bit에서 동작한다.

## Ⅰ. 개요 및 필요성

GPT-3(175B) 전체 파인튜닝 시 A100 GPU 256개 이상이 필요하다. 반면 LoRA를 사용하면 동일 모델을 단일 A100 하나로 파인튜닝할 수 있다. PEFT 기법은 이처럼 대형 모델의 민주적 활용을 가능하게 한다.

전체 파인튜닝의 핵심 문제:
1. 메모리: 모든 파라미터의 기울기와 옵티마이저 상태 저장 필요
2. 비용: GPU 시간이 모델 크기에 선형 비례
3. 파국적 망각(Catastrophic Forgetting): 사전 학습 지식 손실 위험

📢 **섹션 요약 비유**: 건물 전체를 재건축하지 않고, 필요한 방만 인테리어 공사하는 것이 PEFT이다.

## Ⅱ. 아키텍처 및 핵심 원리

### LoRA (Low-Rank Adaptation) 핵심 원리

사전 학습된 가중치 행렬 W₀ (d×k)를 동결하고, 저랭크 행렬 ΔW = BA를 추가한다.

| 항목 | 수식/값 | 설명 |
|:---|:---|:---|
| 원래 순전파 | h = W₀x | 동결된 사전 학습 가중치 |
| LoRA 추가 | h = W₀x + BAx | 저랭크 업데이트 추가 |
| A 초기화 | 정규 분포 (N(0,σ²)) | 랜덤 초기화 |
| B 초기화 | 0 행렬 | 학습 시작 시 ΔW=0 |
| 랭크 r | 보통 4~16 | r << min(d,k) |
| 스케일 | α/r | 학습률 안정화 |
| 파라미터 절감 | r(d+k)/(d×k) × 100% | r=8, d=k=1024 → 1.6% |

```
[LoRA 구조]

원래 레이어 (동결):
  x ──▶ [W₀ (d×k, 동결)] ──▶ h₀

LoRA 추가 경로:
  x ──▶ [A (k×r)] ──▶ [B (r×d)] ──▶ ΔW·x

결합:
  x ──┬──▶ [W₀ (동결)] ──────────┬──▶ h = h₀ + (α/r)·BAx
      └──▶ [A (k×r)] ──▶ [B (r×d)]┘

학습 대상: A, B만 (파라미터 수 = r(d+k))
동결 대상: W₀ (파라미터 수 = d×k)

[추론 시 병합]
W_merged = W₀ + (α/r)·BA
→ 추가 지연 없이 원래 레이어처럼 동작
```

**PEFT 기법 비교**

| 기법 | 방식 | 파라미터 비율 | 특징 |
|:---|:---|:---|:---|
| LoRA | 저랭크 행렬 삽입 | 0.1~1% | 추론 시 병합 가능 |
| Adapter | 병목 레이어 삽입 | 0.5~3% | 추론 지연 발생 |
| Prefix Tuning | 소프트 프롬프트 학습 | 0.1% | 생성 태스크 적합 |
| Prompt Tuning | 입력 임베딩 학습 | < 0.01% | 매우 경량 |
| (IA)³ | 스케일/시프트 벡터 | < 0.01% | 극도 경량 |

📢 **섹션 요약 비유**: LoRA는 두꺼운 백과사전(원본 모델)에 메모지를 붙이는 것이다. 책은 그대로이고, 메모지만 바꾸면 다른 전문가로 변신한다.

## Ⅲ. 비교 및 연결

| 항목 | Full Fine-Tuning | LoRA | QLoRA |
|:---|:---|:---|:---|
| 학습 파라미터 | 100% | ~1% | ~1% |
| VRAM (7B) | ~60GB | ~14GB | ~6GB |
| 성능 | 최고 | 근접 | 약간 열세 |
| 병합 가능 | N/A | ✅ | ✅ (역양자화 필요) |
| GPU 접근성 | H100 필요 | A100 가능 | RTX 3090 가능 |

**QLoRA (Quantized LoRA)**
- 기본 모델을 4bit NF4 양자화로 로드
- LoRA 어댑터는 BF16/FP16으로 학습
- 메모리: 65B 모델을 단일 48GB GPU로 파인튜닝 가능

📢 **섹션 요약 비유**: LoRA가 요약본을 추가하는 것이라면, QLoRA는 그 요약본도 압축해서 더 작은 노트북에 적는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

**LoRA 적용 대상 레이어**
- 일반적으로 어텐션 Q, V 행렬에 적용
- 성능 향상: Q, K, V, O + FFN 전체에 적용 시

**LoRA 하이퍼파라미터 가이드**
- rank r = 8 (기본), 16 (복잡 태스크), 4 (경량)
- alpha = rank × 2 (일반), rank (보수적)
- target_modules: q_proj, v_proj (최소), 전체 어텐션 (성능 극대화)
- dropout = 0.05~0.1

**Hugging Face PEFT 예시**
```python
from peft import LoraConfig, get_peft_model
config = LoraConfig(
    r=8, lora_alpha=16,
    target_modules=["q_proj","v_proj"],
    lora_dropout=0.1,
    task_type="CAUSAL_LM"
)
model = get_peft_model(base_model, config)
```

**기술사 출제 포인트**
- "LoRA가 저랭크 행렬로 파인튜닝하는 원리와 파라미터 효율의 근거를 설명하시오"
- "QLoRA가 메모리를 절감하는 메커니즘을 설명하시오"

📢 **섹션 요약 비유**: LoRA rank는 포스트잇 크기와 같다. 크면 더 많이 적을 수 있지만, 작아도 핵심은 충분히 담을 수 있다.

## Ⅴ. 기대효과 및 결론

PEFT, 특히 LoRA와 QLoRA는 LLM 파인튜닝의 민주화를 이끌었다. 2023년 이후 오픈소스 생태계에서 수천 개의 LoRA 어댑터가 공유되었으며, Hugging Face Hub에서 원클릭으로 적용 가능하다. 기업들은 기밀 데이터를 클라우드 전송 없이 로컬에서 파인튜닝하는 데 PEFT를 적극 활용한다.

📢 **섹션 요약 비유**: PEFT는 AI 파인튜닝의 '저가 항공권'이다. 전체 파인튜닝보다 좌석이 좁을 수 있지만, 같은 목적지에 훨씬 싸게 갈 수 있다.
