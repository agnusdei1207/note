# 167. BFF (Backend For Frontend)

> **핵심 인사이트**: 모바일 앱, 웹 브라우저, 스마트 워치는 화면 크기도 다르고 보여줄 데이터 양도 다르다. API 게이트웨이 하나에 이 모든 짐을 지우지 말고, 프론트엔드 환경(기기)별로 "너네 전용 맞춤형 심부름꾼(BFF)"을 각각 붙여서 가공된 데이터만 딱딱 떠먹여 주는 우아한 아키텍처 패턴이다.

## Ⅰ. BFF (Backend For Frontend)의 개념
마이크로서비스 아키텍처(MSA)에서, 다양한 종류의 프론트엔드 클라이언트(모바일 앱, 데스크톱 웹, 스마트 TV 등)가 백엔드 마이크로서비스들과 직접 통신하지 않도록, **각 클라이언트 환경(Frontend)에 특화된 전용 중간 백엔드(Backend) 서버를 따로 구성하는 설계 패턴**입니다.

## Ⅱ. API 게이트웨이의 한계와 BFF의 등장

만약 거대한 통합 API 게이트웨이(Single API Gateway) 하나만 두고 모바일과 웹을 모두 응대한다면?
모바일 앱은 데이터 요금이 비싸므로 상품의 '이름과 가격'만 필요하고, 데스크톱 웹은 넓은 화면을 채우기 위해 '상세 설명과 리뷰 100개'가 필요합니다. 하지만 범용 게이트웨이는 무조건 전체 데이터를 다 줘버려서 모바일의 메모리를 터지게 하거나(Over-fetching), 반대로 여러 번 API를 부르게 만듭니다(Under-fetching).

```text
[ BFF 패턴 적용 구조 ]

┌─────────┐   ┌───────────────────────────┐   ┌──────────────────┐
│ Mobile  │ ─▶│ BFF for Mobile (모바일용) │ ─▶│ 상품 MS, 결제 MS │
│ App     │   │ (딱 이름과 가격만 조립)   │   │                  │
└─────────┘   └───────────────────────────┘   │                  │
                                              │                  │
┌─────────┐   ┌───────────────────────────┐   │                  │
│ Desktop │ ─▶│ BFF for Web (웹 전용)     │ ─▶│ 상품 MS, 리뷰 MS │
│ Web     │   │ (리뷰 100개까지 화려하게) │   └──────────────────┘
└─────────┘   └───────────────────────────┘
```

## Ⅲ. BFF의 주요 역할과 장점

| 특징 | 설명 |
|:---|:---|
| **맞춤형 데이터 가공 (Aggregation & Formatting)** | 각 클라이언트의 UI 화면에 딱 맞게 여러 백엔드 서비스의 데이터를 조합하고, 불필요한 필드는 잘라낸 뒤 완벽한 형태의 JSON으로 포장해서 내려줍니다. |
| **프론트엔드-백엔드 간 결합도 감소** | 백엔드 개발자는 순수 비즈니스 로직(상품 생성, 결제)에만 집중하고, BFF 서버는 프론트엔드 개발자(보통 Node.js 등으로 작성)가 관리하여 UI 요구사항 변경에 빠르게 대응합니다. |
| **단일 장애점(SPOF) 완화** | 범용 API 게이트웨이가 죽으면 모든 클라이언트가 죽지만, 모바일 BFF가 죽어도 웹 사용자는 정상적으로 서비스를 이용할 수 있습니다. |

## Ⅳ. GraphQL과의 관계
BFF 패턴을 대체하거나 보완하는 가장 강력한 무기가 **GraphQL**입니다. GraphQL을 도입하면 클라이언트(프론트엔드)가 "나는 상품명과 가격만 줘"라고 쿼리(Query)를 마음대로 조작할 수 있으므로, 굳이 기기별로 멍청한 BFF 서버를 여러 대 띄울 필요 없이 하나의 GraphQL 서버로 완벽한 BFF 효과를 낼 수 있습니다.

> 📢 **섹션 요약 비유**: 대형 식당(백엔드) 주방에서 음식을 똑같이 만들어 내보내면, 어른용 상(웹)에는 그대로 나가고 아기용 상(모바일)에는 전용 도우미(BFF)가 달라붙어 가위로 작게 고기를 잘라주고 매운 양념은 덜어내서 아기가 바로 숟가락으로 떠먹기 좋게(UI 최적화) 세팅해 주는 친절한 맞춤형 서빙 시스템입니다.
