# 서비스 디스커버리 (Service Discovery)

> **Service Discovery**: 동적으로 확장/축소되는 마이크로서비스 환경에서, 서비스 인스턴스의 IP 주소와 포트를 자동으로 등록하고 검색할 수 있게 하는 메커니즘으로 하드코딩된 종속성을 제거하고 클라우드 네이티브 환경의 유연성을 보장하는 핵심 인프라

## 핵심 인사이트

마이크로서비스 환경에서 **서비스의 위치(IP:Port)는 고정되지 않습니다**. 오토스케일링으로 새로운 파드가 뜨고, 롤링 업데이트로 IP가 변경되며, 장애 복구로 인스턴스가 재배치됩니다. **Service Discovery**는 이러한 동적 환경에서 **"서비스 A가 서비스 B를 호출할 때 B의 현재 위치를 어떻게 아는가?"**라는 근본적인 문제를 해결합니다. 서비스 인스턴스가 시작될 때 자신의 위치를 **등록(Register)**하고, 클라이언트가 **질(Query)**하여 현재 활성화된 인스턴스 목록을 받아 **부하 분산(Load Balancing)**까지 수행하는 생명주기 관리 시스템입니다.

---

## Ⅰ. 개념 정의 및 등장 배경

### 1. 서비스 디스커버리의 정의

**서비스 디스커버리**는 분산 시스템에서 서비스 인스턴스의 **네트워크 위치 정보(IP 주소, 포트)**를 동적으로 관리하고 검색할 수 있는 메커니즘입니다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    서비스 디스커버리 기본 개념                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  문제: "주문 서비스가 사용자 서비스를 호출하고 싶은데, 어디에 있지?"           │
│                                                                             │
│  ❌ 전통적 방식 (하드코딩)                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                      │   │
│  │   class OrderService {                                              │   │
│  │     private static final USER_SERVICE_URL =                         │   │
│  │       "http://192.168.1.100:8080";  // ← 하드코딩된 IP!              │   │
│  │                                                                      │   │
│  │     public User getUser(String id) {                                │   │
│  │         return restTemplate.getForObject(                          │   │
│  │           USER_SERVICE_URL + "/users/" + id, User.class);           │   │
│  │     }                                                               │   │
│  │   }                                                                 │   │
│  │                                                                      │   │
│  │   문제점:                                                            │   │
│  │   - 서버가 죽으면? → 연결 실패                                      │   │
│  │   - 오토스케일링로 새 인스턴스가 뜨면? → 알 수 없음                  │   │
│  │   - IP가 변경되면? → 코드 재컴파일/배포 필요                         │   │
│  │                                                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ✅ 서비스 디스커버리 방식                                                  │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                      │   │
│  │   class OrderService {                                              │   │
│  │     private DiscoveryClient discoveryClient;  // 디스커버리 클라이언트│   │
│  │                                                                      │   │
│  │     public User getUser(String id) {                                │   │
│  │       // 1. 등록된 사용자 서비스 인스턴스 목록 조회                  │   │
│  │       List<ServiceInstance> instances =                             │   │
│  │         discoveryClient.getInstances("user-service");               │   │
│  │                                                                      │   │
│  │       // 2. 로드 밸런싱으로 하나 선택                                │   │
│  │       ServiceInstance instance = loadBalancer.choose(instances);    │   │
│  │                                                                      │   │
│  │       // 3. 동적 URL로 호출                                          │   │
│  │       String url = instance.getUri() + "/users/" + id;              │   │
│  │       return restTemplate.getForObject(url, User.class);            │   │
│  │     }                                                               │   │
│  │   }                                                                 │   │
│  │                                                                      │   │
│  │   장점:                                                              │   │
│  │   - 인스턴스가 추가/제거되어도 자동으로 반영                         │   │
│  │   - IP 변경에 코드 수정 불필요                                      │   │
│  │   - 장애 인스턴스 자동 제외 (Health Check)                          │   │
│  │                                                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2. 등장 배경

| 환경 | 특징 | 서비스 디스커버리 필요성 |
|:-----|:-----|:------------------------|
| **온프레미스** | 고정 IP, 수동 배포 | 낮음 (하드코딩 가능) |
| **가상화** | 가상 IP, 제한적 동적 확장 | 중간 |
| **컨테이너/쿠버네티스** | IP 자주 변경, 초단위 오토스케일 | **필수** |
| **서버리스** | 완전 동적, 함수 단위 | **필수** |

@EnableDiscoveryClient
public class UserServiceApplication {
    public static void main(String[] args) {

    }

// application.yml
/*
spring:
  application:
    name: user-service  # 서비스 이름 (디스커버리용)

      defaultZone: http://localhost:8761/eureka/  # Eureka Server 주소

  instance:
    lease-renewal-interval-in-seconds: 30  # 30초마다 Heartbeat
    lease-expiration-duration-in-seconds: 90  # 90초 동안 없으면 만료
    prefer-ip-address: true  # IP 주소 사용
    metadata-map:
      zone: us-east-1  # 가용 존 메타데이터
*/

// ==========

// 서비스 디스커버리 사용 예시
@Service
public class OrderService {

    @Autowired
    private DiscoveryClient discoveryClient;

    @Autowired
    private RestTemplate restTemplate;

    public User getUser(String userId) {
        // 1. user-service의 인스턴스 목록 조회
        List<ServiceInstance> instances =
            discoveryClient.getInstances("user-service");

        if (instances.isEmpty()) {
            throw new ServiceUnavailableException(
                "user-service를 찾을 수 없습니다.");
        }

        // 2. 로드 밸런싱 (간단한 Round-Robin)
        ServiceInstance instance = instances.get(
            ThreadLocalRandom.current().nextInt(instances.size()));

        // 3. 동적 URL로 호출
        String url = instance.getUri() + "/api/users/" + userId;
        return restTemplate.getForObject(url, User.class);
    }
}

// ==========

// LoadBalanced RestTemplate (Spring Cloud LoadBalancer)
@Configuration
public class RestTemplateConfig {

    @Bean
    @LoadBalanced  // 자동 로드 밸런싱
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }
}

@Service
public class OrderService {

    @Autowired
    private RestTemplate restTemplate;

    public User getUser(String userId) {
        // 서비스 이름으로 직접 호출! (디스커버리 + 로드 밸런싱 자동)
        return restTemplate.getForObject(
            "http://user-service/api/users/" + userId,
            User.class);
    }
}

### 📞 서비스 디스커버리는 "전화번호부 자동 갱신 시스템"

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   🏢 회사에 직원들이 계속 이사를 다닙니다:                                   │
│                                                                             │
│   - 김 대리는 이번 주에 3번 이사 갔어요 (IP 변동)                           │
│   - 박 과장은 지금 휴가 중이에요 (장애)                                    │
│   - 신입 사원이 매일 입사해요 (오토스케일링)                                 │
│                                                                             │
│   ─────────────────────────────────────────────────────────────────        │
│                                                                             │
│   ❌ 나쁜 방법 (전화번호부 하드코딩)                                        │
│   ─────────────────────────────────────────────────────────────────        │
│                                                                             │
│         나의 전화번호부                                                    │
│      ┌─────────────────────────┐                                          │
│      │ 김 대리: 010-1111-1111   │  ◀─ 하드코딩!                           │
│      │ 박 과장: 010-2222-2222   │                                         │
│      │ 이 사원: 010-3333-3333   │                                         │
│      └─────────────────────────┘                                          │
│                                                                             │
│      문제:                                                                  │
│      - 김 대리가 이사 가면 번호가 바뀌는데...                               │
│      - 내 전화번호부는 여전히 옛날 번호! 😱                                │
│      - 신입 사원은 전화번호부에 없어서 연결 안 돼요!                         │
│      - 박 과장이 휴가 간 줄도 몰라서 계속 전화!                            │
│                                                                             │
│   ─────────────────────────────────────────────────────────────────        │
│                                                                             │
│   ✅ 좋은 방법 (서비스 디스커버리)                                           │
│   ─────────────────────────────────────────────────────────────────        │
│                                                                             │
│         📋 "전화번호부 자동 갱신 센터"                                      │
│      ┌───────────────────────────────────────────────────────┐            │
│      │                                                        │            │
│      │   등록 시스템                                           │            │
│      │   ┌────────────────────────────────────────────────┐  │            │
│      │   │ "저기요, 제 전화번호는 010-1111-1111이에요!"   │  │            │
│      │   │ (김 대리가 자신의 번호를 등록)                   │  │            │
│      │   └────────────────────────────────────────────────┘  │            │
│      │                         │                             │            │
│      │                         ▼                             │            │
│      │   ┌────────────────────────────────────────────────┐  │            │
│      │   │             전화번호부 DB                       │  │            │
│      │   │  ┌──────────────────────────────────────────┐  │  │            │
│      │   │  │ 김 대리: 010-1111-1111 (살아있음 ✓)       │  │  │            │
│      │   │  │ 박 과장: 010-2222-2222 (휴가중 ⊗)         │  │  │            │
│      │   │  │ 이 사원: 010-3333-3333 (살아있음 ✓)       │  │  │            │
│      │   │  │ 최 신입: 010-4444-4444 (방금 등록!)       │  │  │            │
│      │   │  └──────────────────────────────────────────┘  │  │            │
│      │   └────────────────────────────────────────────────┘  │            │
│      │                         ▲                             │            │
│      │   조회 시스템           │                             │            │

│      │   │ → 항상 최신 정보를 제공!                       │  │            │
│      │   └────────────────────────────────────────────────┘  │            │
│      │                                                        │            │

│      │   │ 응답 없으면...                                  │  │            │
│      │   │ → 전화번호부에서 자동 제거!                    │  │            │

│      │                                                        │            │
│      └───────────────────────────────────────────────────────┘            │

│   "누가 이사 가든, 누가 쉬든, 신입이 들어오든...                               │
│    항상 최신 전화번호를 자동으로 찾아주는 게 서비스 디스커버리예요!"          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**핵심**: 서비스 디스커버리는 전화번호(서비스 위치)가 계속 바뀌는 환경에서, **누가 어디에 있는지 자동으로 추적하고 알려주는 시스템**입니다. 인스턴스가 자신의 위치를 등록하고, 살아있는지 계속 확인(Health Check)하며, 클라이언트가 항상 최신 정보를 얻을 수 있게 해줍니다!
