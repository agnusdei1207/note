# 158. REST API 성숙도 Level 0

## 핵심 인사이트 (3줄 요약)
> 1. REST의 개념이 전혀 없고 그저 HTTP 터널을 뚫는 깡통 전송관(TCP 소켓)처럼 사용하는 원시적 형태(RPC 스타일)다.
> 2. 웹소켓이나 레거시 SOAP 연동 시스템에서 단일 엔드포인트(예: `/api/endpoint`) 하나만을 열어놓고 함수명(`"action": "deleteUser"`)을 Payload 본체에 심어 전송하는 최악의 병목 캡슐화다.
> 3. HTTP 인프라가 제공하는 캐싱, 에러코드(404, 201) 체계를 모조리 무시하고 무조건 `200 OK`만 때리므로 라우팅 스케일링이 사실상 폐쇄형 블랙박스가 된다.

---
## Ⅰ. 아키텍처 및 원리
```text
  [ Level 0 호출 구조 ]
   POST /centralAPI HTTP/1.1
   { "Target": "User", "ID": 7, "Action": "Delete" }
   -> HTTP 레벨과 자원 레벨의 분리가 전혀 없고 엉망으로 섞임 (RPC의 유산)
```
---
## Ⅱ. 실무 적용 및 결론
과거 C 언어 소켓 기반 연동이나 금융권 레거시에서 급하게 웹 포팅을 할 때 가장 많이 발견되는 적폐 구조다. 아키텍트는 차세대(MSA)망 도입 설계 시 Level 0 구조를 파헤쳐 도메인별 Resource 분할(Level 1)과 Verb 분할(Level 2)의 메스 칼을 대어 클라우드 네이티브로 이주시켜야 한다.
