# RPA (Robotic Process Automation)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: RPA는 사람이 컴퓨터에서 수행하는 정형화되고 반복적인 업무 프로세스를 소프트웨어 로봇(Bot)이 자동으로 수행하게 하는 기술로, 기존 시스템의 수정 없이 사용자 인터페이스(UI) 레벨에서 작동하는 비침습적(Non-invasive) 자동화 방식이다.
> 2. **가치**: 단순 반복 업무에서의 휴먼 에러를 제로화하고 처리 속도를 획기적으로 높여 운영 비용을 절감하며, 인적 자원을 고부가가치의 창의적 업무에 집중할 수 있게 하는 '디지털 노동력(Digital Workforce)' 역할을 수행한다.
> 3. **융합**: 초기에는 단순 규칙 기반(Rule-based)이었으나, 현재는 AI, OCR, ML 기술이 결합된 지능형 자동화(IPA) 및 전사적 자동화 체계인 **초자동화(Hyperautomation)**의 핵심 수단으로 진화하고 있다.

+++

## Ⅰ. 개요 (Context & Background)

1. **개념**: 로봇 프로세스 자동화(RPA, Robotic Process Automation)는 공장의 로봇 팔이 물건을 조립하듯, 소프트웨어 로봇이 사무실의 마우스와 키보드를 대신 조작하는 것이다. 엑셀 데이터를 ERP에 입력하거나, 웹사이트의 정보를 긁어와(Scraping) 보고서를 만드는 등 "규칙이 명확하고 반복적인" 업무를 자동화하는 데 최적화되어 있다.

2. **💡 비유**: RPA는 사무실의 **'매크로 장인'**과 같습니다. 매일 아침 수십 개의 파일을 열고 복사-붙여넣기를 하던 사람의 손을 대신하여, 한치의 오차도 없이 24시간 365일 지치지 않고 일하는 '디지털 비서'를 채용하는 것과 같습니다.

3. **등장 배경**:
    - **인건비 상승과 주 52시간 근무**: 제한된 시간 내에 더 많은 성과를 내야 하는 생산성 압박 증대.
    - **레거시 시스템 통합의 한계**: 시스템 간 API 연동이 힘들거나 비용이 너무 많이 들 때, UI를 통해 데이터를 연동하는 경제적 대안으로 부상.
    - **데이터 기반 행정의 가속**: 수많은 원천 데이터를 수집하고 정제하는 단순 반복 작업이 DX의 병목 현상으로 작용.

📢 섹션 요약 비유: RPA는 복잡한 수술(시스템 통합) 대신 간편한 '파스(UI 자동화)'를 붙여 업무의 통증(비효율)을 즉시 가라앉히는 실용적인 기술입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

RPA의 기술적 작동 방식과 봇의 유형을 분석한다.

### 1. RPA 작동 아키텍처 (ASCII)

```text
    ┌─────────────────────────────────────────────────────────────┐
    │          [ RPA 제어 센터 : Orchestrator / Manager ]         │
    │  봇 배포, 스케줄링, 권한 관리, 로그 모니터링, 성과 분석     │
    └───────────────┬─────────────────────────────▲───────────────┘
                    │ (작업 지시)                 │ (실행 결과 피드백)
                    ▼                             │
    ┌─────────────────────────────────────────────┴───────────────┐
    │          [ 소프트웨어 로봇 : Digital Worker ]               │
    ├──────────────────┬──────────────────────┬───────────────────┤
    │  1. 화면 인식    │  2. 로직 처리        │  3. 데이터 입출력 │
    │  - UI 셀렉터     │  - 조건문 (If/Then)  │  - ERP, Excel, Web│
    │  - 이미지 / OCR  │  - 반복문 (Loop)     │  - Email, DB      │
    └──────────────────┴──────────────────────┴───────────────────┘
```

**[다이어그램 해설]**  
RPA는 중앙에서 명령을 내리는 **Orchestrator**와 실제 현장에서 일을 하는 **Bot**으로 나뉜다. 봇은 화면의 버튼이나 텍스트 입력창(Selector)을 인식하여 사람처럼 클릭하고 글자를 타이핑한다. 이 과정에서 ERP나 엑셀 파일 등 서로 다른 시스템 사이를 종횡무진하며 데이터를 실어나른다.

### 2. RPA 봇의 두 가지 유형

| 구분 | 유인 봇 (Attended RPA) | 무인 봇 (Unattended RPA) |
|:---|:---|:---|
| **작동 방식** | 사람의 PC에서 사용자가 실행 | 중앙 서버에서 스스로 실행 |
| **개입 여부** | 사람이 시작/종료 제어 | 사람의 개입 없이 스케줄에 따라 작동 |
| **적합 업무** | 판단이 필요한 중간 단계 자동화 | 대량의 정형 데이터 처리 (야간 배치 등) |
| **비유** | 조수석의 내비게이션 | 자율주행 자동차 |

### 3. RPA 구축 5단계 프로세스

1. **대상 발굴 (Discovery)**: 자동화 시 효과가 큰 업무(High ROI) 선정.
2. **프로세스 설계 (PDD)**: 로봇이 따라 할 수 있도록 업무를 0.1초 단위로 쪼개어 정의.
3. **개발 및 테스트 (SDD)**: RPA 툴을 사용하여 로직 구현 및 예외 처리(Exception Handling).
4. **배포 (Deployment)**: 실제 운영 환경에 봇 탑재.
5. **모니터링 및 유지보수**: 봇의 가동 상태 감시 및 시스템 UI 변경 시 로직 수정.

📢 섹션 요약 비유: 유인 봇은 요리사를 돕는 '주방 보조'이고, 무인 봇은 밤새 빵을 굽는 '자동 오븐기'와 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

전통적 자동화와의 차이 및 지능형 자동화로의 진화다.

### 1. 전통적 시스템 통합(EAI/API) vs RPA 비교

| 구분 | 시스템 통합 (Integration) | RPA 자동화 (Automation) |
|:---|:---|:---|
| **수준** | 백엔드(Back-end) 연동 | 프론트엔드(Front-end) 연동 |
| **침습성** | 높음 (DB/코드 수정 필요) | **낮음** (기존 시스템 그대로 사용) |
| **구축 비용** | 높음 (장기 프로젝트) | **낮음** (단기 구축 가능) |
| **안정성** | 매우 높음 | 중간 (UI 변경 시 오류 가능성) |
| **비유** | 건물의 '수도 파이프' 매설 | 이동식 '호스' 연결 |

### 2. 과목 융합: AI와 결합된 IPA (Intelligent Process Automation)

- **문제점**: 기존 RPA는 "글자가 조금만 비뚤어져도" 못 읽는 한계가 있음.
- **해결 (IPA)**:
  - **AI-OCR**: 손글씨나 비정형 영수증에서 텍스트를 정확히 추출.
  - **NLP**: 이메일의 내용을 읽고 작성자의 의도(불만, 주문 등)를 파악.
  - **ML**: 과거 데이터를 학습하여 "이 결재는 승인해도 되는 리스크 수준인가?"를 스스로 판단.

📢 섹션 요약 비유: 전통적 자동화가 건물을 지을 때부터 설계된 '엘리베이터'라면, RPA는 나중에 필요해서 설치한 '에스컬레이터'입니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

실무에서 RPA 도입 시 실패하지 않기 위한 전략적 제언이다.

### 1. 자동화 대상 선정 (Decision)

기술사적 관점에서 모든 일을 자동화하려는 욕심은 금물이다. **'RPA 적합성 매트릭스'**를 활용해야 한다.
- **최적 대상**: 
  1. 반복 횟수가 많은가? (High Frequency)
  2. 예외 상황이 적은가? (Rule-based)
  3. 데이터가 정형화되었는가? (Digital Input)
  4. 기존 시스템 변경이 어려운가? (Legacy System)
- **전략**: ROI가 확실한 'Quick-Win' 과제부터 시작하여 조직의 신뢰 확보.

### 2. RPA 운영 및 보안 체크리스트

| 점검 항목 | 확인 질문 | 기술사적 제언 |
|:---|:---|:---|
| **Bot 권한** | 봇이 관리자 권한을 남용하고 있지는 않은가? | 봇 전용 사번 부여 및 최소 권한(PoLP) 할당 |
| **UI 변경 대응** | ERP 화면이 바뀌면 봇이 멈추는가? | 이미지 기반이 아닌 객체(Object) 기반 인식 설계 |
| **거버넌스** | 부서마다 제각각 봇을 만들어 관리 사각지대가 생겼나? | CoE(전문 역량 센터)를 통한 중앙 집중 관리 |
| **데이터 보안** | 봇이 다루는 고객 비밀번호가 로그에 노출되는가? | Credential Store를 통한 민감 정보 암호화 저장 |

### 3. 안티패턴
- **"The Fragile Bot"**: 예외 처리를 안 해서 조금만 오류가 나도 멈춰버리는 봇.
- **"Robot for Bad Process"**: 엉망인 업무 프로세스를 고치지 않고 그대로 자동화하는 것. (GIGO 가속화)

📢 섹션 요약 비유: RPA 봇에게 사번을 주는 것은 신입 사원을 뽑는 것과 같아서, 명확한 업무 기술서(PDD)와 철저한 근태 관리(모니터링)가 필요합니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

RPA는 기업의 운영 효율성을 한 단계 격상시키는 '디지털 혁신의 촉매제'다.

| 기대효과 | 설명 |
|:---|:---|
| **비용 절감** | 단순 반복 업무의 인건비 및 처리 비용 50~80% 절감 |
| **품질 향상** | 사람의 부주의로 인한 오입력, 누락 사고 원천 차단 |
| **직원 가치 증대** | 지루한 노가다(?)에서 해방되어 창의적 기획 및 분석에 집중 |

미래의 RPA는 **"Hyperautomation"**의 일원으로 통합될 것이다. 단순히 업무 하나를 자동화하는 것이 아니라, AI가 전사 업무를 관찰하여 자동으로 RPA 시나리오를 만들고 배포하는 '자동화의 자동화' 단계에 진입할 전망이다. 또한 사람이 봇과 대화하며 실시간으로 업무를 지시하는 **'Conversational RPA'**가 보편화될 것이다. 이제 RPA는 도구를 넘어, 인간과 협업하는 동료로서 기업의 아키텍처에 완전히 내재화될 것이다.

📢 섹션 요약 비유: 미래의 RPA는 내 책상 옆에서 내가 하는 일을 가만히 지켜보다가 "그 일은 이제 제가 할게요"라고 말하며 내 업무의 짐을 덜어주는 '지능형 쉐도우 워커'가 될 것입니다.

### 📌 관련 개념 맵
| 관련 개념 | 관계 및 시너지 설명 |
|:---|:---|
| Hyperautomation | RPA를 중심으로 AI, 마이닝 등 모든 기술을 동원한 전사 자동화 |
| Process Mining | RPA 개발 전, 실제 업무 흐름을 데이터로 분석하여 대상을 찾는 기술 |
| IPA (Intelligent PA) | RPA에 인공지능(AI)을 결합하여 비정형 업무까지 처리하는 지능형 자동화 |
| CoE (Center of Excellence) | 기업 내 RPA 전략과 표준, 봇 관리를 총괄하는 전문 조직 |
| OCR (Optical Character Recog.) | 종이 문서를 봇이 읽을 수 있는 데이터로 바꿔주는 RPA의 단짝 기술 |

### 👶 어린이를 위한 3줄 비유 설명
1. 매일 아침 숙제를 똑같이 베껴 적는 일을 나 대신 해주는 "똑똑한 로봇 손" 같은 거예요.
2. 로봇은 잠도 안 자고 실수도 안 해서, 아주 빠르고 정확하게 일을 끝내준답니다.
3. 로봇에게 힘든 일을 맡기고, 우리는 더 재미있는 놀이를 생각할 시간을 갖게 되는 아주 고마운 친구예요!
