# DRM (Data Reference Model, 데이터 참조 모델)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DRM은 범정부 EA(GEA)의 5대 참조 모델 중 하나로, 부처 및 기관 간의 데이터 공유와 재사용을 촉진하기 위해 데이터의 표준 분류, 구조, 교환 방식을 정의한 전사적 데이터 표준 프레임워크다.
> 2. **구조**: 데이터의 의미를 정의하는 **데이터 설명(Description)**, 비즈니스 맥락을 분류하는 **데이터 맥락(Context)**, 기관 간 연계 방식을 정의하는 **데이터 공유(Sharing)**의 3대 요소로 구성된다.
> 3. **가치**: 데이터의 중복 구축을 방지하여 예산을 절감하고, 국가 차원의 데이터 정합성을 확보함으로써 디지털 플랫폼 정부 구현을 위한 '데이터 칸막이 해소'의 핵심 도구로 작동한다.

+++

## Ⅰ. 개요 (Context & Background)

1. **개념**: 데이터 참조 모델(DRM, Data Reference Model)은 공공기관들이 사용하는 수많은 데이터에 대한 **'공용 사전'**이자 **'분류표'**다. 각 기관이 '주소'나 '이름'이라는 데이터를 서로 다른 규격으로 관리하면 부처 간 정보 연계가 불가능해진다. DRM은 이러한 혼란을 막기 위해 국가 전체가 동일한 언어로 소통할 수 있는 표준 체계를 제공한다.

2. **💡 비유**: DRM은 대형 마트의 **'상품 분류 및 라벨링 시스템'**과 같습니다. 전국의 모든 매장(기관)이 동일한 카테고리(맥락)와 바코드 규격(설명), 그리고 물류 이동 방식(공유)을 공유해야만 고객(국민)이 어디서든 원하는 물건을 쉽게 찾고 배송받을 수 있는 것과 같습니다.

3. **등장 배경**:
    - **행정 정보 공유의 장벽**: 기관별로 독자적인 데이터 모델을 구축하여 시스템 간 데이터 불일치 및 연계 비용 과다 발생.
    - **범정부 아키텍처(GEA) 수립**: 전 국가적 IT 자산을 효율적으로 관리하기 위한 5대 참조 모델의 필요성 대두.
    - **데이터 기반 행정 요구**: 과학적 의사결정을 위해 부처를 넘나드는 통합 데이터 분석 인프라 구축 필수.

📢 섹션 요약 비유: DRM은 대한민국이라는 거대한 스마트 시티의 '데이터 도로 규격'이자 '공용 표지판'입니다. 이 규격이 맞아야만 정보라는 자동차들이 사고 없이 자유롭게 다닐 수 있습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

DRM 2.0 표준을 중심으로 3대 핵심 구성 요소를 분석한다.

### 1. DRM 3대 구성 요소 및 구조 (ASCII)

```text
    ┌─────────────────────────────────────────────────────────────┐
    │              [ DRM : 3 Core Components ]                    │
    ├─────────────────────────────────────────────────────────────┤
    │                                                             │
    │  1. 데이터 설명 (Data Description)                          │
    │     - 데이터 요소, 메타데이터, 구조 정의                    │
    │     - "데이터가 무엇을 의미하는가?" (What)                  │
    │                                                             │
    │  2. 데이터 맥락 (Data Context)                              │
    │     - 비즈니스 기반 분류, 주제 영역 정의                    │
    │     - "데이터가 어디서 쓰이는가?" (Where/Why)               │
    │                                                             │
    │  3. 데이터 공유 (Data Sharing)                              │
    │     - 교환 규격, 메시지 포맷, 서비스 인터페이스             │
    │     - "데이터를 어떻게 주고받는가?" (How)                   │
    │                                                             │
    └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]**  
DRM은 데이터를 입체적으로 관리한다. 단순히 이름만 정의하는 것이 아니라, 그 데이터가 어떤 업무(Context)에서 사용되는지, 그리고 타 기관에 줄 때는 어떤 포맷(Sharing)으로 줘야 하는지까지 패키지로 관리하는 아키텍처를 갖는다.

### 2. DRM 구성 요소 상세 (Table)

| 구성 요소 | 핵심 항목 | 상세 설명 | 기술사적 포인트 |
|:---|:---|:---|:---|
| **데이터 설명** | 메타데이터, 엔티티, 속성 | 데이터의 명칭, 타입, 길이, 제약 조건 표준화 | 데이터 표준화, 품질 관리 |
| **데이터 맥락** | 주제 영역 (Subject Area) | 비즈니스 업무(BRM)와 데이터의 매핑 분류 | 데이터 거버넌스, 분류 체계 |
| **데이터 공유** | 교환 패키지, 데이터 스키마 | XML/JSON 기반 메시지 정의, API 규격 | 상호운용성, Open API |

### 3. DRM의 데이터 분류 체계 (Taxonomy)

- **최상위 (주제 영역)**: 예) 고용, 보건복지, 산업진흥.
- **중간 (데이터 그룹)**: 예) 구직자 정보, 병원 위치 정보.
- **최하위 (데이터 요소)**: 예) 주민등록번호, 성명, 휴대전화번호.

📢 섹션 요약 비유: 데이터 설명은 상품의 '성분표'이고, 데이터 맥락은 상품이 놓인 '진열대 위치'이며, 데이터 공유는 상품을 담아 배달하는 '택배 상자'와 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

DRM과 다른 데이터 관리 기법과의 관계다.

### 1. MDM(기준 데이터 관리) vs DRM 비교

| 구분 | MDM (Master Data Mgmt) | DRM (Data Reference Model) |
|:---|:---|:---|
| **목적** | 단일 데이터 소스(Gold Copy) 확보 | 전사적 데이터 표준 및 공유 기준 수립 |
| **범위** | 핵심 기준 정보 (고객, 상품, 자산) | 비즈니스 전체 데이터 자산 |
| **성격** | 실행/운영 중심 (Operational) | 참조/설계 중심 (Strategic) |
| **관계** | DRM의 표준을 따라 MDM을 구축함 | 범정부 DRM은 각 기관 MDM의 가이드가 됨 |
| **비유** | 집 안의 공용 열쇠 관리 | 열쇠의 규격과 보관함 표준 설계 |

### 2. 과목 융합: 데이터 레이크와 DRM의 역할

현대적인 빅데이터 환경(Data Lake)에서 DRM은 'Schema-on-Read'의 기준이 된다.
- **Data Discovery**: DRM의 맥락(Context) 정보를 통해 데이터 레이크 내의 수많은 데이터 중 필요한 것을 쉽게 검색.
- **Data Catalog 연동**: DRM의 메타데이터 정보가 자동화된 데이터 카탈로그 시스템의 기초 입력값으로 활용됨.

📢 섹션 요약 비유: MDM이 '진짜 보석 하나'를 잘 관리하는 법이라면, DRM은 전국의 모든 보석함이 '어떤 규격으로 만들어져야 하는지'를 정하는 국가 표준 설계법입니다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

실무에서 DRM을 활용하여 데이터 거버넌스를 강화하는 전략이다.

### 1. 데이터 연계 사업에서의 DRM 적용 (Decision)

공공 사업에서 'A기관 데이터와 B기관 데이터를 통합 분석'하라는 과제가 주어졌을 때:
- **잘못된 접근**: 각 기관의 DB를 통째로 부어 넣고 나중에 매핑함. (데이터 정합성 오류 폭증)
- **기술사적 제언**: 범정부 DRM의 **'공용 메시지 표준'**을 기반으로 기관 간 인터페이스 스키마를 선 정의. 
- **효과**: 개발 단계에서의 데이터 변환(ETL) 로직 단순화 및 향후 확장이 용이한 유연한 구조 확보.

### 2. DRM 관리 및 활용 체크리스트

| 점검 항목 | 확인 질문 | 기술사적 제언 |
|:---|:---|:---|
| **표준 준수성** | 우리 기관의 데이터 사전이 범정부 표준과 일치하는가? | 범정부 표준 메타데이터 도구와 연동 |
| **공유 가능성** | 민감 정보 제외 후 타 기관에 API로 제공 가능한 구조인가? | 공유 컴포넌트(SRM)와 데이터(DRM)의 결합 설계 |
| **품질 연계** | DRM 표준을 어긴 데이터가 DB에 쌓이고 있지는 않은가? | 실시간 데이터 프로파일링을 통한 표준 준수 체크 |
| **비즈니스 정렬** | 업무 프로세스(BRM) 변경 시 데이터 구조도 바뀌는가? | BRM-DRM 간의 매핑 매트릭스 최신화 |

### 3. 안티패턴
- **"The Ivory Tower Standard"**: 실무 DB 설계와는 전혀 상관없는 문서용 DRM 구축.
- **"Metadata Overload"**: 너무 세세한 항목까지 표준화하려다 정작 중요한 데이터의 공유 속도를 늦추는 경우. (핵심 데이터 위주의 표준화 우선순위 설정 필요)

📢 섹션 요약 비유: DRM은 '국제 공용어'와 같아서, 공부하기는 힘들지만 한 번 익혀두면 전 세계 누구와도(어떤 시스템과도) 막힘없이 대화할 수 있는 강력한 무기가 됩니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

DRM은 데이터 기반의 지능형 정부와 기업을 만드는 가장 견고한 주춧돌이다.

| 기대효과 | 설명 |
|:---|:---|
| **상호운용성 보장** | 시스템 간 데이터 전송 시 변동 비용 50% 이상 절감 |
| **데이터 품질 향상** | 표준화된 설명을 통해 데이터 오독 및 가공 오류 최소화 |
| **신속한 DX 추진** | 잘 정의된 데이터 맥락을 바탕으로 신규 비즈니스 서비스 조기 발굴 |

미래의 DRM은 **"Semantic Web & Linked Data"** 체계로 진화할 것이다. 사람이 일일이 매핑하지 않아도 AI가 데이터의 의미를 추론하여 자동으로 DRM 표준과 연결하고, '데이터 메시(Data Mesh)' 환경에서 각 도메인이 자율적으로 관리하면서도 전사 표준을 유지하는 지능형 연합 모델이 주류가 될 것이다. DRM은 이제 정적인 분류표를 넘어, 데이터가 스스로의 가치를 증명하게 만드는 '지능형 데이터 지도'로 진화하고 있다.

📢 섹션 요약 비유: 미래의 DRM은 모든 데이터에 이름표와 GPS를 달아주는 것과 같아서, 어떤 데이터가 어디서 태어나서 어디에 쓰이는지 실시간으로 알려주는 '디지털 데이터 족보'가 될 것입니다.

### 📌 관련 개념 맵
| 관련 개념 | 관계 및 시너지 설명 |
|:---|:---|
| Metadata (메타데이터) | 데이터 설명을 구성하는 '데이터에 대한 데이터' |
| BRM (Business Reference Model) | 데이터 맥락의 기준이 되는 비즈니스 기능 분류 모델 |
| SRM (Service Reference Model) | 데이터 공유를 실제 웹 서비스나 API로 구현하는 모델 |
| Data Governance | DRM을 수립하고 준수하도록 통제하는 전사적 관리 체계 |
| Interoperability (상호운용성) | DRM의 궁극적인 목표인 시스템 간 원활한 정보 교환 능력 |

2. 이 약속이 있으면 내가 어느 도서관에 가더라도 원하는 책을 금방 찾을 수 있고, 도서관끼리 책을 빌려주기도 쉬워져요.
3. 데이터들이 서로 친하게 지내고 잘 도와줄 수 있게 만든 "공용 이름표 규칙"이랍니다!
