# 493. IAST (Interactive Application Security Testing)

> ⚠️ 이 문서는 코드의 줄 번호는 알지만 진짜로 해킹되는진 모르는 '정적 분석(SAST)'과, 진짜 뚫리는 건 알지만 코드 몇 번째 줄인지 모르는 '동적 분석(DAST)'의 치명적 단점들을 완벽하게 융합하여, **앱의 심장(메모리) 안에 에이전트(Agent)를 박아두고 외부 공격과 내부 코드의 반응을 동시에 실시간(Interactive)으로 분석하는 궁극의 3세대 보안 테스팅, 'IAST'**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: IAST(대화형 보안 테스팅)는 애플리케이션 '내부(Inside)'에 스파이 칩(Agent)을 심어두고, 외부에서 들어오는 트래픽이 어떤 소스 코드 경로를 타고 흐르는지를 현미경처럼 실시간 모니터링하는 기술이다. (SAST + DAST의 장점 결합)
> 2. **가치**: DAST처럼 실제로 공격을 던지면서, SAST처럼 **"이 공격이 142번째 줄 코드를 뚫고 들어왔습니다!"**라고 소스 코드 레벨의 정확한 위치(Line Number)까지 짚어주어 오탐(False Positive)이 0에 수렴한다.
> 3. **기술 체계**: 자바(Java)의 JVM이나 .NET CLR 같은 가상 머신(VM)에 바이트코드 인스트루멘테이션(Instrumentation, 계측) 기술로 기생하여 작동하므로, 테스트를 위해 코드를 수정할 필요가 없다. 주로 QA 통합 테스트(Integration Test) 단계에서 백그라운드로 돌아간다.

---

### Ⅰ. 개요: 장님과 벙어리의 만남 (Context & Necessity)
SAST(정적 분석)는 시력이 좋아서(코드 100% 열람) 해킹 구멍 1만 개를 찾아냈다. 하지만 그중 9,990개는 진짜 해킹이 불가능한 가짜 경고(오탐)였다. (장님)
DAST(동적 분석)는 확실하게 뚫리는 10개(진탐)만 정확하게 찾아냈다. 하지만 "화면 뚫렸어!"라고 소리칠 뿐, 개발자에게 "그래서 코드 어디를 고쳐야 하는데?"라는 질문에는 꿀 먹은 벙어리가 된다.

보안 업계는 답답했다. **"밖에서 때리는 놈(DAST)이랑, 안에서 쳐다보는 놈(SAST)이 실시간으로 대화(Interactive)하게 만들면 안 될까?"**

이 마법을 실현한 것이 **IAST**다.
QA 엔지니어가 평소처럼 브라우저로 게시판 글쓰기 테스트를 한다. 이때 앱 내부에 심어진 IAST 에이전트가 그 트래픽을 가로채어 "어? 글쓰기 할 때 방금 `SQL` 쿼리가 위험하게 날아갔어! `BoardService.java` 50번 줄이야!"라고 개발자에게 알람을 쏜다. 
따로 해킹 툴(DAST)을 돌리느라 시간을 낭비할 필요도 없고, 소스코드 스캔(SAST)의 멍청한 가짜 알람에 시달릴 필요도 없는 혁명이 시작된 것이다.

> 📢 **섹션 요약 비유**: 도둑(해커)을 잡기 위해 SAST는 '도면'만 보고, DAST는 밖에서 문만 흔들어봅니다. IAST는 집 '안방'에 투명 인간 경찰(에이전트)을 숨겨놓고, 도둑이 문을 흔들 때 안에서 "방금 2번 창문으로 도둑이 손을 집어넣었습니다!"라고 정확한 위치를 실시간 무전(Interactive)으로 쏴주는 최고의 보안 감시망입니다.

---

### Ⅱ. IAST의 작동 원리 (Instrumentation)

IAST가 어떻게 소스 코드의 줄 번호까지 아는 걸까? 핵심은 **바이트코드 계측(Bytecode Instrumentation)**이라는 고도의 기술이다.

#### 1. 에이전트 삽입 (Passive / Active)
- 톰캣(Tomcat) 서버를 띄울 때 `-javaagent:iast.jar` 옵션 한 줄만 주면 끝이다. 소스 코드를 건드릴 필요가 없다.
- 이 에이전트는 앱이 메모리에 올라갈 때, 핵심 함수들(`DB 실행`, `파일 읽기` 등)의 껍데기에 몰래 '센서'를 덕지덕지 붙여놓는다 (Instrumentation).

#### 2. 실시간 모니터링 (Taint Tracking)
- 정상적인 QA 테스트 트래픽이나 DAST의 공격 트래픽이 웹 서버로 들어온다.
- 센서가 작동한다! "오, 방금 유저가 친 비밀번호 데이터가 `login()` 함수를 지나서, 암호화도 안 거치고(Taint Flow), `query()` 함수로 바로 꽂혔어! SQL 인젝션 100% 확정이다!"
- 에이전트는 공격의 흐름, 실행된 소스코드의 파일명, 라인 넘버를 통째로 묶어서 보안 대시보드로 쏴준다.

#### 3. Active IAST (능동형)
- 센서가 지켜만 보는 것(Passive)을 넘어, QA가 평범하게 글쓰기를 누르면 IAST 에이전트가 그 트래픽을 가로채서 스스로 악성코드(`' OR 1=1`)로 변형한 뒤(Mutate) 앱 내부로 쏴버린다. 
- 별도의 DAST 해킹 툴을 돌릴 필요 없이, QA의 기능 테스트와 보안 테스트가 한 번의 클릭으로 100% 동시에 끝난다.

```text
┌────────────────────────────────────────────────────────────────────────────────┐
│           IAST (Interactive) 에이전트 기반의 실시간 엑스레이 방어 시각화       │
├────────────────────────────────────────────────────────────────────────────────┤
│                                                                                │
│ 👨‍🦰 [ QA 테스터 / DAST ] ──(클릭)──▶ 🏢 [ 웹 애플리케이션 (JVM/런타임) ]     │
│                                       │                                        │
│     (밖에서 bình범하게 기능 테스트 중)      │ 🕵️‍♂️ [ IAST 에이전트 ] 기생중!  │
│                                       │   1. 트래픽 흐름 감시                  │
│                                       │   2. 취약 코드 도달 확인               │
```

### Ⅲ. IAST의 한계와 포지셔닝

IAST는 이론상 완벽하지만, 실무에서 SAST/DAST를 완전히 멸종시키지는 못했다.

* **언어와 플랫폼 종속성**: IAST는 JVM(Java), .NET, Node.js 처럼 '가상 머신이나 런타임 해석기'가 있는 언어에서만 센서를 붙일 수 있다. C나 C++로 깡통 컴파일(바이너리)된 레거시 시스템에는 에이전트를 심을 수가 없다.
* **테스트 커버리지의 한계**: SAST는 클릭 안 해도 코드를 다 읽지만, IAST는 QA나 로봇이 '실제로 그 화면을 클릭해서 코드가 실행되어야만' 취약점을 찾을 수 있다. 즉, 테스트 안 한 코드는 평생 검사할 수 없다.
* 그래서 실무의 DevSecOps 파이프라인은 `(빌드 시점: SAST) -> (QA 통합 테스트 시점: IAST) -> (오픈 직전: DAST)`의 삼각 편대로 무장한다.

### Ⅳ. 결론

"어둠 속의 더듬거림(DAST)과 쏟아지는 소음(SAST) 속에서, 마침내 타겟을 정확히 조준하다."
IAST(Interactive Application Security Testing)는 보안 진단의 패러다임을 '사후 결과 보고'에서 '실시간 내시경 검사'로 진화시킨 걸작이다. 이 기술의 가장 큰 위대함은 보안팀과 개발팀 사이의 지긋지긋한 감정싸움(False Positive 핑퐁)을 종식시켰다는 데 있다. "코드 몇 번째 줄이 뚫렸다"라는 빼박불가한 재현 경로와 콜스택(Call Stack) 증거 앞에서는 어떤 개발자도 변명할 수 없으며 즉각적인 수정에 돌입하게 된다. IAST는 QA의 기능 테스트(Functional Test) 활동을 그대로 보안 테스트(Security Test)로 치환해 버림으로써, 애자일(Agile) 속도를 1밀리초도 깎아먹지 않고 보안성을 극대화한 현대 시프트-레프트(Shift-Left) 사상의 가장 우아한 기술적 완성이다.

### 📌 관련 개념 맵
* **전제 기술**: 바이트코드 계측 (Bytecode Instrumentation), Taint Analysis (오염 데이터 추적)
* **비교/결합 도구**: SAST (정적 분석 - 소스코드), DAST (동적 분석 - 해킹 툴)
* **확장 기술**: RASP (Runtime Application Self-Protection - 494번 문서, IAST가 진화하여 아예 방어까지 함)
