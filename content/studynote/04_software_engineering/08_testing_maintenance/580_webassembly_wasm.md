# 580. 웹어셈블리 (WebAssembly, WASM)

> ⚠️ 이 문서는 웹 브라우저에서 오직 자바스크립트(JavaScript)만 실행할 수 있었던 오랜 독점 체제를 깨고, **C/C++, Rust, Go 같은 강력한 백엔드 언어들을 브라우저 위에서 네이티브(Native)에 가까운 엄청난 속도로 실행시킬 수 있게 만든 웹의 새로운 표준, '웹어셈블리'**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: WebAssembly(WASM)는 자바스크립트를 대체하는 것이 아니라, 자바스크립트가 하기 힘든 '무거운 계산'을 돕기 위해 브라우저 엔진에 탑재된 **바이너리(0과 1) 실행 포맷**이다.
> 2. **가치**: 기존에 포토샵이나 프리미어 프로 같은 무거운 PC용 프로그램을 브라우저 위에서 똑같은 속도로 돌릴 수 있게 해준다. 별도의 플러그인(ActiveX, Flash) 없이 모든 최신 브라우저가 기본 지원한다.
> 3. **기술 체계**: C++이나 Rust로 짠 코드를 컴파일러(Emscripten 등)를 통해 `.wasm` 파일로 변환하고, 이를 브라우저의 자바스크립트가 불러와서 실행하는 구조를 가진다.

---

### Ⅰ. 개요: 자바스크립트의 한계를 넘다 (Context & Necessity)
웹 브라우저는 태생적으로 가벼운 문서를 보여주기 위해 만들어졌다.
자바스크립트(JS)는 인터프리터 언어라 읽기 쉽고 유연하지만, **태생적으로 너무 느리다.**
- 웹에서 3D 게임을 돌리거나, 4K 동영상을 편집하려고 하면 JS는 계산을 감당하지 못하고 브라우저를 뻗게 만든다.

이 문제를 해결하기 위해 구글, 애플, 마이크로소프트, 모질라(파이어폭스)가 손을 잡고 만든 것이 **WebAssembly**다.
"C++ 같은 무거운 언어로 코드를 짜서, 브라우저가 바로 읽을 수 있는 기계어(바이너리)로 압축해서 던져주자!"

> 📢 **섹션 요약 비유**: 자바스크립트가 요리법을 **'한국어(사람의 언어)'**로 읽으면서 그때그때 요리하는 거라면, WASM은 요리법을 아예 **'기계어 펀치 카드(바이너리)'**로 찍어와서 로봇 팔에 꽂아버리는 것과 같습니다. 로봇 팔이 펀치 카드를 읽고 요리하는 속도는 사람이 글씨를 읽으며 요리하는 속도와 비교할 수 없이 빠릅니다.

---

### Ⅱ. WASM의 3대 핵심 특징

#### 1. 네이티브에 가까운 성능 (Near-Native Performance)
- 코드가 이미 컴파일(기계어 변환)된 상태로 브라우저에 다운로드된다. 
- 파싱하고 컴파일하는 시간이 생략되어 다운로드 즉시 초고속으로 실행된다. (JS보다 수배~수십 배 빠름)

#### 2. 다국어 지원 (Polyglot)
- 자바스크립트를 배울 필요 없이, 내가 원래 잘하던 C, C++, Rust, Go, Python 등으로 코드를 짜서 웹에 올릴 수 있다. (특히 메모리 관리가 완벽한 Rust와 궁합이 최고다.)

#### 3. 샌드박스 보안 (Sandbox Security)
- 브라우저 밖으로 나가서 내 컴퓨터의 하드디스크나 마이크를 맘대로 건드릴 수 없다. 철저하게 브라우저의 보안 규칙(동일 출처 정책 등) 안에서만 안전하게 실행된다.

---

### Ⅲ. WASM은 어떻게 작동하는가?

WASM은 절대 혼자서 UI(버튼, 화면)를 그리지 못한다. 반드시 **자바스크립트라는 비서**가 필요하다.

1. **개발 (Rust)**: 영상 필터를 씌우는 무거운 함수를 Rust로 개발한다.
2. **컴파일 (WASM)**: 컴파일러가 이를 `filter.wasm` 파일로 압축한다.
3. **통합 (JS)**: 웹페이지의 자바스크립트가 `filter.wasm`을 다운받아 로드한다.
4. **실행**: 사용자가 '필터 적용' 버튼을 누르면, JS가 WASM에 있는 함수를 호출해서 0.1초 만에 결과를 받아 화면에 그린다.

```text
┌──────────────────────────────────────────────────────────────┐
│           JavaScript와 WebAssembly의 협력 작동 구조 시각화            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ [ 👨‍💻 웹 브라우저 (V8 엔진) ]                                     │
│   ┌────────────────────┐          ┌────────────────────┐     │
│   │ 📜 JavaScript 로직 │          │ ⚙️ WebAssembly 모듈 │     │
│   │ - 버튼 클릭 이벤트   │ ──(호출)─▶ │ - 3D 렌더링 계산     │     │
│   │ - 화면 그리기 (DOM) │ ◀─(결과)── │ - 영상 압축 알고리즘 │     │
│   └────────────────────┘          └────────────────────┘     │
│                                                              │
│ ★ 특징: JS는 UI와 이벤트를 담당하고, WASM은 무거운 수학 계산만 전담한다. │
└──────────────────────────────────────────────────────────────┘
```

### Ⅳ. 결론

"PC 프로그램의 종말을 앞당기는 기술."
Figma, AutoCAD, Adobe Photoshop 같은 엄청난 무도구들이 모두 WASM을 통해 웹 브라우저 안으로 들어왔다. "프로그램은 다운로드해서 설치해야 한다"는 과거의 상식이 깨지고 있는 것이다. WASM은 자바스크립트를 죽이러 온 암살자가 아니다. 오히려 자바스크립트의 한계를 완벽하게 보완해 줌으로써, 브라우저를 단순한 문서 뷰어에서 '보편적인 운영체제(Universal OS)'로 진화시킨 웹 생태계 역사상 가장 위대한 발명품 중 하나다.

### 📌 관련 개념 맵
* **관련 언어**: C/C++, Rust, Go, AssemblyScript
* **기반 환경**: Web Browser V8 Engine (크롬), SpiderMonkey (파이어폭스)
* **프레임워크**: Blazor (.NET C#을 웹으로), Yew (Rust를 웹으로)
* **확장 기술**: WASI (WebAssembly System Interface) - 브라우저 밖(서버/도커)에서도 WASM을 돌리려는 표준

---

### 👶 어린이를 위한 3줄 비유 설명
1. 자바스크립트는 친절하지만 힘이 좀 약한 도서관 사서 선생님이에요. 무거운 책 수백 권을 옮기라고 하면 쓰러져버리죠.
2. WASM은 터미네이터 같은 천하장사 로봇이에요. 사서 선생님이 "저 무거운 책 좀 다 옮겨줘!"라고 버튼만 누르면, 로봇이 1초 만에 책을 다 옮겨주죠.
3. 로봇(WASM)은 사람들과 직접 대화할 줄 몰라서, 항상 사서 선생님(JS)이 명령을 내려줘야만 완벽한 팀워크가 된답니다!
