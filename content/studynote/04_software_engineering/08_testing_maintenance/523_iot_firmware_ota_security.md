# 523. IoT 펌웨어 무결성 및 OTA (Over-The-Air) 안전 배포

> ⚠️ 이 문서는 스마트 홈 카메라, 테슬라 자율주행차, 공장의 로봇 팔처럼 전 세계에 수백만 대가 흩어져 있는 쪼그만 기기들(IoT)에 새로운 소프트웨어(펌웨어)를 업데이트해 줄 때, 해커가 중간에서 가짜 바이러스 파일을 날려보내어 **수백만 대의 자동차와 카메라를 동시에 살인 병기나 좀비 PC(Mirai 봇넷)로 만들어버리는 대참사를 막기 위한 궁극의 무선 방어망, '안전한 OTA 배포 구조'**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: OTA(Over-The-Air)는 USB 선을 꽂지 않고 무선 인터넷(Wi-Fi, 5G)으로 기기의 두뇌(펌웨어)를 원격 업데이트하는 기술이다. 편리하지만, 중간에 해커가 펌웨어를 가로채어 악성코드로 변조하면 기기 전체가 적의 조종을 받게 된다.
> 2. **가치**: 1대의 웹 서버 해킹과 달리, 수백만 대의 IoT 기기가 악성 펌웨어에 감염되면 디도스(DDoS) 공격의 미사일 포대(봇넷)가 되거나, 달리던 자동차의 브레이크가 먹통이 되는 등 물리적인 인명 피해(Cyber-Physical)로 직결된다.
> 3. **기술 체계**: 이를 막으려면 제조사 서버에서 펌웨어에 **'디지털 서명(RSA/ECC)'** 도장을 꽉 찍어 날려 보내고, 쪼그만 IoT 기기 내부에 심어진 안전한 칩(Secure Boot, TPM)이 도장(무결성)을 완벽히 검사한 뒤에만 업데이트를 허락하는 철통 아키텍처가 필수적이다.

---

### Ⅰ. 개요: 허공을 날아가는 뇌수술 (Context & Necessity)
테슬라(Tesla) 자동차는 서비스 센터에 갈 필요 없이 밤에 주차장에 세워두면 Wi-Fi를 타고 차가 알아서 최신 소프트웨어로 업데이트된다. 이것이 **OTA (Over-The-Air)** 기술이다.

정말 편리하지만, 해커의 눈에는 완벽한 사냥감이다.
테슬라 서버가 자동차로 업데이트 파일(`firmware.bin`)을 무선으로 쏜다.
해커가 허공을 날아가는 그 파일을 가로채어(Man-in-the-Middle), `브레이크_고장.bin` 파일로 스윽 바꿔치기해서 자동차에 밀어 넣는다.
자동차가 멍청하게 "어? 본사에서 온 새 업데이트네? 설치!" 하고 다음 날 고속도로에서 브레이크를 꺼버린다.

단순한 컴퓨터 고장이 아니라 생명이 날아가는 문제다. 
그래서 IoT 공학자들은 "허공으로 펌웨어를 쏠 때(OTA), 이 파일이 **본사가 보낸 진짜 파일이 맞는지(인증)**, 중간에 **1바이트도 조작되지 않았는지(무결성)**를 기계가 100% 확실하게 검증하고 나서야 두뇌를 갈아 끼우는 방탄 시스템"을 설계했다. 이것이 **펌웨어 무결성 검증망**이다.

> 📢 **섹션 요약 비유**: 왕(본사 서버)이 멀리 변방에 있는 장군(IoT 기기)에게 "내일 아침 성문을 열어라"라는 명령서(업데이트)를 비둘기(OTA) 편에 보냈습니다. 장군은 편지를 받자마자 성문을 여는 바보가 아닙니다. 편지 겉봉에 찍힌 왕의 고유한 '빨간 밀랍 도장(디지털 서명)'이 깨지지 않고 100% 똑같은지(무결성) 돋보기로 확인한 뒤에야 편지를 뜯고 명령을 따릅니다.

### Ⅱ. OTA 안전 배포를 위한 3중 아키텍처

IoT 기기는 메모리도 작고 CPU도 멍청하다. 이 작은 녀석이 해커를 막으려면 인프라 차원의 설계가 필요하다.

#### 1. 서버 단: 펌웨어 암호화 및 디지털 서명 (Sign)
- 제조사 서버는 펌웨어 파일 전체를 암호화(AES)하여 해커가 뜯어보지 못하게(기밀성) 감춘다.
- 가장 중요한 건 **디지털 서명(Digital Signature)**이다. 제조사가 꽁꽁 숨겨둔 '마스터 개인키(Private Key)'로 펌웨어 파일에 도장을 찍는다.

#### 2. 네트워크 단: TLS 기반 전송 (Transport)
- 펌웨어 파일을 그냥 HTTP 평문으로 쏘지 않는다. 무조건 **상호 인증 TLS(mTLS - 512번 문서)** 터널을 뚫어, 엉뚱한 해커 서버가 주는 파일을 다운받지 않도록 기기와 서버가 서로 신분증(인증서)을 교차 검증하며 다운로드한다.

#### 3. 기기 단: 보안 부팅 (Secure Boot)과 무결성 검사 ★가장 중요★
- IoT 기기(자동차, 카메라) 내부에는 공장에서 출고될 때 절대 수정할 수 없는 부위에 제조사의 '마스터 공개키(Public Key)'가 하드코딩(eFuse, TPM 칩)되어 있다.
- 다운로드받은 펌웨어를 실행(Boot)하기 전, 기기는 자기 뱃속에 있는 '공개키'를 꺼내 펌웨어에 찍힌 서명(도장)을 검사한다.
- 해커가 중간에 파일을 조작했다면 서명이 100% 깨진다. 기기는 "삐빅! 서명 불일치! 이 펌웨어는 가짜(독극물)다!"라며 업데이트를 거부하고, 예전에 쓰던 안전한 구버전 펌웨어로 다시 부팅(Rollback)해 버린다.

```text
┌───────────────────────────────────────────────────────────────────────────────┐
│           해커의 공격을 막아내는 OTA 무결성 펌웨어 업데이트 시각화            │
├───────────────────────────────────────────────────────────────────────────────┤
│                                                                               │
│ 🏢 [ 테슬라 본사 서버 ]                                                       │
│   새 펌웨어(V2) + [ 👑 마스터 개인키로 전자서명(도장) 쾅! ]                   │
│          │                                                                    │
│          │ (인터넷 / 무선 OTA 전송) ──────────┐                               │
│          ▼                                  │                                 │
│ 😈 [ 해커의 중간 가로채기 ]                         │                         │
│   "내가 악성 바이러스 파일로 바꿔치기 해야지 ㅋ"      │                       │
│          │                                  │                                 │
│          ▼ (가짜 펌웨어 도착)                    ▼ (진짜 펌웨어 도착)         │
│                                                                               │
│ 🚗 [ 테슬라 자동차 (내부 TPM 보안 칩) ]                                       │
│                                                                               │
│  ❌ 해커 파일: "내 뱃속의 공개키로 도장을 맞춰보니... 도장 안 맞잖아! 꺼져!"  │
│             (업데이트 거부, 생존 성공 🟢)                                     │
│                                                                               │
│  ✅ 진짜 파일: "오! 본사 도장이 완벽하게 맞네 (무결성 통과). 설치 삐빅!"      │
└───────────────────────────────────────────────────────────────────────────────┘
```

### Ⅲ. 실무적 딜레마: 롤백(Rollback) 방지 공격

해커가 진짜 펌웨어를 변조하는 걸 포기했다 (서명을 위조할 수 없으니까).
대신 영악한 해커는 **"예전에 취약점이 있어서 폐기된 1년 전 구버전 진짜 펌웨어(V1)"**를 어디서 주워 왔다. 이 V1 펌웨어는 과거에 제조사가 진짜로 만든 거라 **'도장(서명)'이 완벽하게 찍혀있다.**

해커는 이 옛날 V1 파일을 자동차에 밀어 넣는다 (Downgrade Attack / Anti-Rollback 취약점 공격).
자동차는 도장을 검사하고 "어? 진짜 본사 도장 맞네!" 하고 1년 전 취약한 펌웨어로 자기 뇌를 되돌려버린다. 그리고 해커는 그 옛날 취약점으로 차를 해킹해 버린다.

* **방어 설계 (Anti-Rollback 퓨즈)**: 이를 막기 위해 IoT 기기의 칩(eFuse) 내부에는 현재 설치된 펌웨어의 버전 번호(예: V2)가 물리적으로 타버려서(Burn) 기록된다. 기기는 업데이트를 할 때 "나한테 들어온 파일이 V2보다 높은 버전(V3)인지"를 검사하여, 버전이 낮으면 진짜 도장이 찍혀있어도 설치를 튕겨내 버린다.

---

### Ⅳ. 결론

"디지털의 가벼운 날갯짓이 현실의 거대한 철덩어리를 움직인다."
IoT 기기의 OTA(Over-The-Air) 배포는 물리적 세상(Physical)과 사이버 세상(Cyber)의 경계를 허문 가장 거대한 축복이자 재앙의 불씨다. 웹 서버 해킹은 DB가 털리고 끝이지만, 100만 대의 커넥티드 카나 발전소 센서가 악성 펌웨어에 장악당하는 순간 소프트웨어 버그는 인간의 생명을 위협하는 흉기로 돌변한다. 펌웨어가 허공을 가로질러 작은 기기의 심장부로 이식되는 그 위태로운 찰나의 순간을 지키는 것은, 절대 위조할 수 없는 비대칭키 암호학(RSA/ECC)과 하드웨어 칩(Secure Boot)에 깊숙이 뿌리내린 '무결성 검증(Integrity Check)' 아키텍처뿐이다. 이 지독한 검증망이 없다면, 다가올 초연결 5G 시대의 인프라는 언제든 해커의 스위치 하나로 멈춰버리는 거대한 시한폭탄이 될 것이다.

### 📌 관련 개념 맵
* **전제되는 암호 기술**: 디지털 서명(Digital Signature - 504번 문서), 해시 무결성 검증
* **관련 해킹 기술**: Mirai 봇넷 (IoT 기기 좀비화 DDoS 공격), 롤백 공격(Downgrade)
* **보안 아키텍처/하드웨어**: Secure Boot(보안 부팅), TPM (신뢰할 수 있는 플랫폼 모듈), eFuse
* **보호 목표**: 데이터 무결성 (Integrity) 및 가용성 (Availability)

### 👶 어린이를 위한 3줄 비유 설명
1. 로봇 장난감에 매달 날아오는 '새로운 기능 테이프(펌웨어)'를 틀어주면 로봇이 똑똑해져요(OTA).
2. 근데 나쁜 악당이 허공을 날아가는 테이프를 낚아채서 '나를 때려라'라고 녹음된 미친 테이프(악성 펌웨어)로 몰래 바꿔치기하면 로봇이 나를 공격하겠죠?
3. 그래서 공장에서는 진짜 테이프 겉면에 절대 흉내 낼 수 없는 '마법의 투명 도장(디지털 서명)'을 찍어 보내요! 로봇은 테이프를 틀기 전에 무조건 자기 눈(보안 칩)으로 이 마법 도장을 확인하고, 도장이 없으면 절대 테이프를 재생하지 않는답니다!
