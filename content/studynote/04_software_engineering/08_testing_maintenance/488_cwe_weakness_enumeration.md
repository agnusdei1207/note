# 488. CWE (Common Weakness Enumeration)

> ⚠️ 이 문서는 해커들이 공격할 수 있는 수만 가지의 잡다한 취약점들을 개별적으로 부르지 않고, **"이 버그는 'SQL 인젝션 계열' 버그야", "이건 '버퍼 오버플로우 계열' 버그야"라며 소프트웨어의 근본적인 '설계 및 코딩의 약점(Weakness)'들을 체계적으로 분류하고 고유 번호를 매겨놓은 전 세계 보안의 뿌리 백과사전, 'CWE'**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CWE(보안 약점 사전)는 특정 회사의 프로그램에 난 구멍(CVE)이 아니라, 전 세계 모든 소프트웨어가 공통으로 앓고 있는 **'근본적인 질병(약점)의 종류'**를 분류해 놓은 카테고리(분류 체계)다.
> 2. **가치**: "우리 회사 코딩 가이드라인은 뭘 기준으로 만들지?"라는 고민을 없애준다. 정적 분석 툴(SAST)이나 시큐어 코딩 규정은 모두 이 CWE 번호(예: `CWE-89` = SQL 인젝션)를 기준으로 탐지 룰(Rule)을 세팅한다.
> 3. **비교 개념**: CWE가 '질병의 종류(예: 감기, 암)'라면, 뒤이어 나올 CVE(489번)는 '특정 환자의 진단서(예: 구글 크롬 90버전에서 터진 감기)'를 의미한다. CWE는 예방(Prevention)을 위한 설계 단계의 뼈대다.

---

### Ⅰ. 개요: 족보 없는 버그들의 족보 만들기 (Context & Necessity)
전 세계 해커들이 매일 새로운 해킹 기법을 쏟아낸다.
- "A 은행에서 쿼리문에 따옴표를 안 막아서 털렸대!"
- "B 쇼핑몰은 아이디 칸에 특수문자 넣어서 털렸대!"

보안 학자들(MITRE 코퍼레이션)이 이 수백만 건의 해킹 사례를 모아놓고 분석해 보니 충격적인 사실을 깨달았다.
**"야, 겉모습만 다르지 결국 까고 보면 전부 다 개발자가 '입력값 검증(Input Validation)'을 제대로 안 해서 터진 똑같은 근본 원인(Weakness)이잖아?!"**

학자들은 이 수만 개의 잡다한 버그들을 족보(Tree) 구조로 깔끔하게 묶어버리기로 결심했다.
가장 큰 뿌리에 'CWE-20: 부적절한 입력값 검증'을 박아놓고, 그 가지 밑에 'CWE-89: SQL 인젝션', 'CWE-79: 크로스 사이트 스크립팅(XSS)' 등 **전 세계 모든 코딩 실수의 원인들을 번호를 매겨 딕셔너리로 편찬**했다. 이것이 바로 보안 업계의 대백과사전, **CWE(Common Weakness Enumeration)**다.

> 📢 **섹션 요약 비유**: 병원에 수만 명의 환자가 옵니다. 누구는 콧물이 나고, 누구는 기침을 하고, 누구는 열이 납니다. 의학자들은 이 잡다한 증상들을 묶어서 "이것들의 근본 원인은 모두 '감기 바이러스(CWE)'라는 족보에 속한다!"라고 책에 질병 코드를 매겨 정리했습니다. CWE는 의사(보안 전문가)들이 진단을 내리기 위해 펼쳐보는 '질병 분류 사전'입니다.

---

### Ⅱ. CWE의 계층 구조와 Top 25

CWE는 단순히 리스트를 나열한 게 아니라 부모-자식 간의 트리(Tree) 구조를 가진다.

1. **최상위 클래스 (Class)**: 아주 거시적인 개념. (예: `CWE-707`: 부적절한 구조적 처리)
2. **중간 베이스 (Base)**: 일반적인 코딩 약점. (예: `CWE-20`: 부적절한 입력 확인)
3. **최하위 변형 (Variant)**: 개발자가 실제로 저지르는 아주 구체적인 실수. (예: `CWE-89`: SQL 구문 삽입)

#### 🏆 CWE Top 25 (가장 치명적인 25대 질병)
OWASP Top 10이 웹사이트에 특화된 차트라면, 미국 국토안보부 산하의 CISA와 MITRE가 매년 발표하는 **CWE Top 25 Most Dangerous Software Weaknesses**는 웹, 앱, C언어, 하드웨어를 가리지 않고 "전 세계 소프트웨어를 파괴하는 가장 무서운 코딩 결함 25개"의 랭킹이다.
여기에도 어김없이 1위는 'Out-of-bounds Write (버퍼 오버플로우)', 2위는 'XSS', 3위는 'SQL Injection' 같은 고전적인 약점들이 수십 년째 철밥통을 유지하고 있다.

---

### Ⅲ. 실무 활용 (시큐어 코딩과 정적 분석의 뇌)

CWE는 보안 문서 작업이나 솔루션(Tool)을 만들 때 절대적인 척도가 된다.

* **정적 분석기(SAST)의 탐지 룰**: SonarQube나 Fortify 같은 코딩 검사 툴이 빨간 줄을 띄울 때, 무조건 `[CWE-89 위반]`이라는 꼬리표를 달아서 알려준다. 이 툴들의 내부 AI 엔진 자체가 CWE 백과사전을 통째로 학습한 뇌(Brain)이기 때문이다.
* **행정안전부 시큐어 코딩 가이드**: 대한민국 정부나 KISA에서 배포하는 "소프트웨어 보안 약점 47개 항목" 같은 모든 공공기관 문서의 뿌리(맵핑 대상)가 바로 이 글로벌 CWE 번호다. 정부 사업을 수주하려면 코드를 스캔해서 "우리 코드는 CWE 주요 항목에 위배되지 않았습니다"라는 증명서를 내야 한다.

```text
┌───────────────────────────────────────────────────────────────────────────────┐
│           CWE (약점 족보)와 CVE (실제 발생 사건)의 패러다임 비교 시각화       │
├───────────────────────────────────────────────────────────────────────────────┤
│                                                                               │
│ 📘 [ CWE (Common Weakness Enumeration) : 의학 백과사전 ]                      │
│   분류번호: CWE-89 (질병 이름)                                                │
│   개념: SQL 인젝션 (개발자가 쿼리에 파라미터를 안 묶었을 때 생기는 고질병)    │
│   방어책: Prepared Statement를 쓰시오. (예방주사 가이드)                      │
│                                                                               │
│            ⬆️ (이 족보에 속하는 실제 터진 사고들) ⬇️                          │
│                                                                               │
│ 💥 [ CVE (Common Vulnerabilities) : 실제 환자 진단서 ]                        │
│   - 사건 1: CVE-2023-1234 (구글 크롬 114버전에서 터진 SQL 인젝션 사고)        │
│   - 사건 2: CVE-2021-9999 (오라클 DB 12버전에서 터진 SQL 인젝션 사고)         │
│                                                                               │
│ ★ 결론: 수십만 개의 CVE(실제 사고)들의 근본 뼈대를 추려보면, 결국 몇 백 개의  │
│         CWE(코딩 약점) 족보 안으로 다 귀결된다!                               │
└───────────────────────────────────────────────────────────────────────────────┘
```

### Ⅳ. 결론

"모든 혁신적인 해킹은 결국 고전적인 인간의 실수에서 출발한다."
CWE(Common Weakness Enumeration)는 해커들의 끝없는 창의성 앞에서 패배주의에 빠질 뻔했던 방어자(엔지니어)들에게, 적의 공격 패턴이 유한하며 구조화될 수 있다는 희망을 안겨준 위대한 분류학(Taxonomy)적 성취다. 해커가 아무리 현란한 우회 스크립트를 짜와도, 아키텍트는 CWE라는 엑스레이를 통해 그 본질이 결국 'CWE-20(입력값 검증 부재)'이라는 낡은 질병임을 꿰뚫어 볼 수 있다. 소프트웨어 설계자(Architect)가 시프트 레프트(Shift-Left) 보안을 실현하기 위해 책상에 꽂아두고 평생을 음미해야 할 궁극의 오답 노트가 바로 이 CWE 사전이다.

### 📌 관련 개념 맵
* **비교 개념**: CVE (공개된 실제 취약점 번호 - 489번 문서 참조)
* **점수화 모델**: CVSS (취약점의 심각도를 점수 매기는 계산식 - 490번 문서 참조)
* **실무 활용 도구**: SAST (정적 코드 분석 도구의 탐지 규칙 기반)
* **보조 차트**: OWASP Top 10 (웹에 특화된 취약점 차트), CWE Top 25 (전체 SW 약점 차트)

### 👶 어린이를 위한 3줄 비유 설명
1. 세상에 교통사고가 하루에도 수만 번씩 일어나요. 트럭이랑 부딪히고, 오토바이랑 부딪히고 종류도 다양하죠.
2. 경찰 아저씨들이 이 수많은 사고를 일일이 부르기 힘드니까, 공통된 원인을 묶어서 '1번: 졸음운전', '2번: 신호위반', '3번: 과속'이라고 딱 **원인 족보(분류표)**를 만들었어요.
3. 이렇게 전 세상의 컴퓨터 프로그램들이 망가지는 수만 가지 이유를, "비밀번호를 너무 쉽게 만들었네(1번)", "입력 확인을 안 했네(2번)"처럼 딱딱 분류해서 책으로 엮어놓은 코딩 약점 백과사전이 **CWE**랍니다!
