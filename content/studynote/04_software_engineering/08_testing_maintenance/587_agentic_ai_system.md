# 587. 에이전틱 AI (Agentic AI) 시스템

> ⚠️ 이 문서는 인간이 시키는 대로 텍스트만 뱉어내는 수동적인 AI를 넘어, **목표를 달성하기 위해 AI 스스로 계획을 세우고, 필요한 도구(인터넷 검색, DB 조회, 이메일 발송)를 직접 호출하며 자율적으로 행동하는 '에이전틱 AI' 아키텍처**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 에이전트(Agent)는 **'LLM(두뇌) + 도구(손발) + 메모리(기억)'**의 결합체다. 단순히 대답만 하는 것이 아니라 현실 세계(시스템)의 상태를 변경(Action)할 수 있는 주도권을 가진다.
> 2. **가치**: "다음 주 부산 출장 일정 잡아줘"라고 하면, AI가 스스로 KTX 시간표를 검색하고, 내 캘린더를 확인해서 빈 시간을 찾은 뒤, 법인 카드로 예매하고 나에게 확인 메일을 보내는 수준의 완전 자동화를 이룬다.
> 3. **기술 체계**: AI가 어떤 도구를 언제 써야 할지 스스로 판단하게 만드는 **ReAct (Reasoning and Acting)** 프롬프팅 기법과 **Function Calling(함수 호출)** 기술이 핵심 엔진이다.

---

### Ⅰ. 개요: 생각하는 기계에서 '행동하는 기계'로 (Context & Necessity)
기존의 RAG(585번 문서)나 단순 챗봇은 결정적인 한계가 있다. **'읽기(Read)'만 가능하고 '쓰기(Write/Action)'를 못 한다는 것이다.**
- 사용자: "내일 서울 날씨 어때?"
- RAG 챗봇: "저희 사내 문서에는 날씨 정보가 없습니다." (실패)

**에이전틱 AI**는 다르다.
에이전트에게는 '구글 검색', '사내 DB', '사내 메신저'라는 도구(Tool)들을 쥐여준다.
- 사용자: "내일 서울 날씨 어때?"
- 에이전트: (생각 중...) '날씨는 내가 모르니까 [구글 검색 도구]를 써야겠다.' $\rightarrow$ 검색 실행 $\rightarrow$ (결과 확인 중...) '비가 오네.' $\rightarrow$ "내일 서울에 비가 옵니다."

> 📢 **섹션 요약 비유**: 기존 AI가 도서관에 묶여서 질문에 대답만 해주는 **'사서 선생님'**이라면, 에이전트는 바깥세상으로 나가서 필요한 물건을 사 오고 심부름까지 완벽하게 해내는 **'자율 주행 로봇 비서'**와 같습니다.

---

### Ⅱ. 에이전트의 3대 핵심 메커니즘 ★

에이전트가 자율적으로 움직일 수 있는 이유는 다음 3단계를 스스로 반복하기 때문이다.

#### 1. 생각 (Thought / Reasoning)
- 사용자의 목표를 달성하기 위해 **"내가 지금 무엇을 해야 하는가?"**를 스스로 계획한다.
- "서울 날씨를 알아내려면 먼저 인터넷 검색이 필요해."

#### 2. 행동 (Action / Tool Use)
- 계획을 바탕으로 개발자가 쥐여준 도구(Tools) 중 하나를 선택해서 실행한다.
- **Function Calling**: GPT-4 같은 최신 LLM은 "내가 이 함수(도구)를 실행할게"라고 JSON 형태로 정확하게 함수 이름과 파라미터를 뱉어내는 능력이 있다.

#### 3. 관찰 (Observation)
- 도구를 실행한 결과값을 읽어본다.
- "검색 결과에 강수량 80%라고 적혀있네. 충분한 정보가 모였으니 이제 사용자에게 대답하자." (부족하면 1번 '생각'으로 다시 돌아간다 - ReAct Loop)

---

### Ⅲ. 에이전트 시스템 설계의 위험성 (Risk)

에이전트에게 '쓰기' 권한을 주면 심각한 보안 사고가 터질 수 있다.
- 에이전트가 "이 유저 데이터는 필요 없는 것 같군"이라며 스스로 `DB DELETE` 도구를 실행해 버린다면?
- 에이전트가 해커의 프롬프트 인젝션(Prompt Injection)에 속아서 사내 기밀 문서를 외부 이메일로 전송해 버린다면?

**해결책 (Human-in-the-Loop, HITL)**
위험한 행동(예: 결제, 삭제, 이메일 발송)을 할 때는 에이전트가 혼자 결정하게 두지 말고, **"사장님, 제가 이메일을 보내려고 하는데 승인(Approve)해 주시겠어요?"**라며 반드시 인간의 버튼 클릭(개입)을 거치도록 아키텍처를 설계해야 한다.

```text
┌──────────────────────────────────────────────────────────────┐
│           에이전틱 AI (ReAct 프롬프팅) 작동 루프 시각화               │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ [ 👨‍💻 사용자 ] "오늘 우리 쇼핑몰 매출액을 팀장님께 메일로 보내"           │
│       │                                                      │
│       ▼                                                      │
│ ┌──────────────────────────────────────────────────────────┐ │
│ │ 🤖 AI 에이전트 루프 (ReAct)                                  │ │
│ │                                                          │ │
│ │ 1. Thought: 매출을 알려면 DB를 뒤져야겠다.                     │ │
│ │ 2. Action : [DB 쿼리 Tool] 실행 -> "SELECT * FROM sales" │ │
│ │ 3. Observe: "결과: 500만 원"                               │ │
│ │                                                          │ │
│ │ 4. Thought: 이제 팀장님 이메일 주소를 찾아야겠다.                 │ │
│ │ 5. Action : [사내 직원 검색 Tool] 실행 -> "팀장"             │ │
│ │ 6. Observe: "결과: boss@company.com"                       │ │
│ │                                                          │ │
│ │ 7. Thought: 정보가 다 모였다. 메일을 보내자.                    │ │
│ │ 8. Action : [이메일 전송 Tool] 실행 -> "매출: 500만 원"        │ │
│ └──────────────────────────────────────────────────────────┘ │
│                                                              │
│ ★ 특징: 개발자가 순서를 짜준 게 아니라, AI가 스스로 판단해서 움직인다.    │
└──────────────────────────────────────────────────────────────┘
```

### Ⅳ. 결론

"소프트웨어는 명령을 받는 존재에서, 스스로 목표를 달성하는 존재로 진화한다."
에이전틱 AI는 RPA(Robotic Process Automation)의 한계를 부수고 진정한 지능형 자동화 시대를 열었다. 여러 명의 전문 에이전트(코딩 에이전트, 리뷰 에이전트, 배포 에이전트)가 서로 토론하며 거대한 소프트웨어를 완성하는 '멀티 에이전트(Multi-Agent)' 시스템은 이미 실무에 도입되고 있다. 개발자는 이제 함수를 짜는 사람이 아니라, 에이전트들이 사용할 '도구(Tools)'를 정교하게 깎아주고 이들을 지휘하는 역할을 맡게 될 것이다.

### 📌 관련 개념 맵
* **관련 프레임워크**: LangChain (586번), AutoGPT, CrewAI (멀티 에이전트)
* **핵심 프롬프팅**: ReAct (Reasoning and Acting)
* **LLM 필수 기능**: Function Calling (Tool Calling)
* **보안 아키텍처**: Human-in-the-Loop (HITL)

---

### 👶 어린이를 위한 3줄 비유 설명
1. 기존 AI는 백과사전과 같아요. 물어보면 대답은 잘하지만 손발이 없어서 심부름은 못 하죠.
2. 에이전틱 AI는 심부름꾼 로봇이에요. "마트 가서 두부 사 와!"라고 하면 스스로 지갑(도구)을 챙기고, 마트 가는 길(검색)을 찾아서 출발하죠.
3. 두부가 다 팔렸으면 그냥 돌아오는 게 아니라, 스스로 생각해서 옆 가게로 가서 두부를 사 오는 진짜 똑똑한 로봇이랍니다!
