# 564. 헬름 (Helm) 차트를 이용한 K8s 패키지 관리

> ⚠️ 이 문서는 쿠버네티스(K8s)에 앱 하나를 띄우기 위해 작성해야 하는 수십 장의 복잡한 YAML 설정 파일들을, **마치 스마트폰의 앱스토어나 리눅스의 `apt-get`처럼 클릭(명령어) 한 번에 설치하고 삭제할 수 있게 만들어주는 패키지 매니저 '헬름(Helm)'**을 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 쿠버네티스용 패키지 매니저(Package Manager)다. `Pod`, `Service`, `Ingress` 등 여러 개의 YAML 파일들을 하나의 묶음인 **'차트(Chart)'**로 만들어서 관리한다.
> 2. **가치**: 복잡한 K8s 설정을 매번 하드코딩할 필요 없이, 템플릿(빈칸 뚫기) 기능을 이용해 개발(Dev), 검증(QA), 운영(Prod) 환경에 맞춰 변수값만 쏙쏙 바꿔서 재사용할 수 있다.
> 3. **기술 체계**: 뼈대가 되는 `deployment.yaml` 같은 템플릿 파일과, 그 빈칸을 채워줄 `values.yaml` 파일이 결합되어 최종 K8s 설정 파일을 동적으로 생성해 낸다.

---

### Ⅰ. 개요: 지옥의 YAML 복사 붙여넣기 (Context & Necessity)
쿠버네티스에 '워드프레스' 블로그를 하나 띄우고 싶다고 치자.
K8s를 맨땅에서 쓰려면 다음 작업이 필요하다.
1. `wordpress-deployment.yaml` 작성 (100줄)
2. `wordpress-service.yaml` 작성 (30줄)
3. `mysql-deployment.yaml` 작성 (100줄)
4. `mysql-service.yaml` 작성 (30줄)
... 이걸 일일이 타이핑하고 하나씩 `kubectl apply`를 쳐야 한다.

**Helm**은 이 모든 과정을 명령어 한 줄로 끝낸다.
```bash
helm install my-blog bitnami/wordpress
```
끝이다. Helm이 알아서 필요한 수십 개의 YAML 파일을 K8s에 한꺼번에 던져주고 세팅을 끝낸다.

> 📢 **섹션 요약 비유**: K8s YAML 파일들을 일일이 짜는 것이 **'이케아 가구 부품을 하나하나 주문해서 설명서 보고 조립하는 것'**이라면, Helm은 **'완성된 가구를 박스째 배달받아서(설치) 포장만 뜯어 쓰는 것'**과 같습니다. 원한다면 박스에서 '색깔(Value)' 정도만 주문할 때 바꿀 수 있죠.

---

### Ⅱ. 헬름 차트(Chart)의 핵심 구조

헬름이 패키지를 관리하는 단위를 **차트(Chart)**라고 부른다. 차트 폴더를 열어보면 다음과 같은 마법의 2가지 파일이 있다.

#### 1. 템플릿 (Templates / 빈칸 뚫린 문서)
- K8s YAML 파일인데, 곳곳에 구멍(`{{ }}`)이 뚫려있다.
- 예: `replicas: {{ .Values.replicaCount }}` (서버 개수는 나중에 채울게!)

#### 2. Values.yaml (빈칸 채우기 답안지)
- 템플릿의 구멍을 채워줄 실제 값들이 적혀있는 파일이다.
- 예: `replicaCount: 3`
- 개발자는 템플릿(뼈대)은 건드리지 않고, 이 `values.yaml` 파일만 운영 환경용, 개발 환경용으로 따로 만들어서 적용한다.

---

### Ⅲ. 헬름의 3대 장점

#### 1. 환경별 재사용성 (Reusability)
- 템플릿 하나를 만들어두고, `values-dev.yaml`, `values-prod.yaml`만 교체해서 배포하면 되므로 코드 중복이 획기적으로 줄어든다.

#### 2. 강력한 버전 관리와 롤백 (Rollback)
- Helm은 설치할 때마다 버전(Revision)을 매긴다.
- 배포를 했는데 서버가 터졌다면? `helm rollback my-app 1` 명령어 하나면 이전 상태로 관련된 모든 K8s 객체들이 일제히 타임머신을 타고 되돌아간다.

#### 3. 생태계 (Helm Hub / Artifact Hub)
- 세상에 존재하는 거의 모든 유명한 소프트웨어(MySQL, Redis, Nginx, Prometheus 등)는 이미 똑똑한 사람들이 Helm 차트로 예쁘게 포장해서 인터넷에 올려두었다. 우리는 그냥 다운받아 쓰기만 하면 된다.

```text
┌──────────────────────────────────────────────────────────────┐
│           Helm 템플릿 렌더링(Rendering) 과정 시각화                 │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ [ 📜 Templates (뼈대) ]         [ 📝 Values.yaml (값) ]       │
│  replicas: {{ .Values.rep }}     rep: 5                      │
│  image: {{ .Values.img }}        img: "nginx:latest"         │
│             │                            │                   │
│             └─────────────┬──────────────┘                   │
│                           ▼ (Helm Install)                   │
│               ┌───────────────────────┐                      │
│               │ replicas: 5           │  (완성된 YAML)         │
│               │ image: "nginx:latest" │ ──▶ [ ☸️ 쿠버네티스 ]   │
│               └───────────────────────┘                      │
│                                                              │
│ ★ 장점: 개발서버 배포 시엔 Values.yaml 내용만 'rep: 1'로 바꿔서 쓰면 됨! │
└──────────────────────────────────────────────────────────────┘
```

---

### Ⅳ. 결론

"쿠버네티스를 인간의 언어로 번역하는 도구."
쿠버네티스가 클라우드의 운영체제(OS)라면, 헬름은 그 운영체제 위에서 돌아가는 앱스토어(App Store)다. K8s의 폭발적인 복잡성을 템플릿이라는 캡슐 안에 안전하게 숨겨주기 때문에, 인프라 엔지니어(DevOps)가 예쁘게 차트를 빚어 놓으면 일반 개발자들은 `helm install` 명령어 하나만으로도 거대한 시스템을 쉽게 다룰 수 있게 된다.

### 📌 관련 개념 맵
* **관련 기술**: Kubernetes (K8s), YAML, Go Template
* **동의어/유사 도구**: Kustomize, Kustomize (K8s 기본 기능)
* **패키지 저장소**: Artifact Hub
* **활용 파이프라인**: GitOps (ArgoCD, FluxCD와 결합하여 사용)

### 👶 어린이를 위한 3줄 비유 설명
1. 쿠버네티스를 그냥 쓰는 건 레고 블록 수만 개를 바닥에 쏟아놓고 설명서 없이 알아서 비행기를 조립하는 것과 같아요.
2. 헬름(Helm)은 '비행기 조립 세트 박스'와 같아요. 그 안에 블록과 설명서가 완벽하게 세트로 포장되어 있죠.
3. 우리는 그냥 "비행기 세트 하나 주세요!"(Helm install)라고 말하기만 하면, 눈앞에 멋진 비행기가 뚝딱 완성된답니다!
