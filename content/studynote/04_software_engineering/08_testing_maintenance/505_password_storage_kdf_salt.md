# 505. 비밀번호 저장 방식 - KDF 및 솔트(Salt) 적용

> ⚠️ 이 문서는 사용자의 비밀번호를 보호하기 위해 단순한 해시(SHA-256) 함수를 쓰는 것만으로는 엄청나게 빨라진 해커의 그래픽카드(GPU) 연산 속도를 막아낼 수 없음을 깨닫고, **소금(Salt)을 쳐서 해커의 사전을 무력화시키고 일부러 연산 속도를 끔찍하게 느리게 만드는 KDF(키 파생 함수)를 통해 비밀번호를 완벽하게 굳혀버리는 최신 패스워드 방어 아키텍처**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비밀번호를 평문으로 저장하면 안 된다고 해서 SHA-256 같은 빠른 해시 함수를 썼더니, 해커들이 1초에 수십억 번의 해시를 계산해 버려 암호가 순식간에 뚫리는 치명적 약점(속도의 저주)이 발견되었다.
> 2. **가치**: 이 속도의 저주를 막기 위해, 해시를 계산할 때 의도적으로 CPU나 메모리를 혹사시켜서(Work Factor) 1번 계산하는 데 0.5초가 걸리도록 **'지독하게 느려 터진 암호화 함수(KDF)'**를 사용하는 것이 현대 보안의 핵심이다.
> 3. **기술 체계**: 비밀번호 앞에 무작위 쓰레기값을 붙이는 **솔트(Salt)**를 기본으로 깔고, 그 값을 **PBKDF2, bcrypt, scrypt, Argon2** 같은 KDF 함수 믹서기에 넣고 수만 번 돌려서 궁극의 비밀번호 다짐육(해시)을 만들어낸다.

---

### Ⅰ. 개요: 너무 빨라서 망해버린 암호 (Context & Necessity)
우리는 504번 문서에서 비밀번호는 복구 불가능한 '일방향 해시(SHA)'를 써야 한다고 배웠다.
개발자가 `1234`를 SHA-256으로 해시해서 `8d96...` 이라는 문자로 DB에 저장했다. 완벽해 보인다.

하지만 해커들은 똑똑했다. 전 세계 사람들이 자주 쓰는 비밀번호 수억 개(1234, password, qwerty 등)를 자기들 컴퓨터에서 미리 SHA-256으로 돌려서 매핑 테이블(이른바 **레인보우 테이블, Rainbow Table**)을 만들어버린 것이다! 
이제 DB가 털리면 해커는 해시값을 자기 표에서 `Ctrl+F`로 검색만 하면 0.1초 만에 원본 비밀번호(`1234`)를 쏙 빼갈 수 있다.

더 큰 문제는 SHA-256이 **'너무 빠르다'**는 것이다. 원래 파일 무결성을 빨리 검사하려고 만든 알고리즘이라, 해커가 RTX 4090 그래픽카드 몇 장 꽂고 무차별 대입 공격(Brute-force)을 돌리면 1초에 100억 개의 비밀번호를 찍어 맞춰버린다. 빠름이 곧 독이 된 것이다.

> 📢 **섹션 요약 비유**: 해커가 열쇠 구멍에 맞는 열쇠를 찾으려고 1초에 100억 번씩 찰칵찰칵 열쇠를 꽂아보는 상황입니다. 방어자는 자물쇠 입구에 끈적한 꿀(고의적인 지연 시간, Work Factor)을 잔뜩 발라놔서, 열쇠를 한 번 꽂고 돌리는 데 무조건 0.5초가 걸리게 뻑뻑하게 만들어버립니다. 해커는 100억 번을 시도하려면 평생이 걸리게 되어 스스로 포기하게 됩니다.

---

### Ⅱ. 완벽한 패스워드를 굽는 2가지 재료

#### 1. 솔트 (Salt, 소금치기) - 레인보우 테이블 무력화
- **원리**: 사용자가 `1234`를 쳤을 때, 서버가 몰래 무작위 문자열(`@X9z`)을 하나 만들어서 붙인다. $\rightarrow$ `1234@X9z`
- **효과**: 이걸 해시 믹서기에 갈아버리면, 해커가 미리 만들어둔 `1234` 전용 해킹 사전(레인보우 테이블)이랑 값이 아예 안 맞아서 해독표가 완벽하게 휴지통행이 된다. 유저마다 소금(Salt)의 값이 다 다르므로 해커는 유저 1만 명을 털려면 해킹 사전을 1만 번 처음부터 새로 만들어야 하는 지옥에 빠진다.

#### 2. KDF (Key Derivation Function) - 브루트포스 무력화
- **원리**: 소금을 쳐서 해시한 결과물을 다시 믹서기에 넣고, 그걸 또 믹서기에 넣고... 이렇게 수학적 믹서기질을 **1만 번 연속으로 반복(Iteration)**하게 만든다.
- **효과**: 해커가 암호를 1번 찍어 맞추려면 연산을 1만 번 해야 하므로 컴퓨터가 터질 듯이 뜨거워지고 속도가 1만 배 느려진다. 정상적인 유저는 로그인할 때 0.5초만 기다리면 되니까 아무 불편함이 없지만, 1억 번을 시도해야 하는 해커에게 0.5초는 1년의 지연 시간(Delay)이라는 치명타가 된다.

```text
┌────────────────────────────────────────────────────────────────────────────┐
│           KDF(Bcrypt 등)와 Salt를 활용한 패스워드 방어 아키텍처 시각화     │
├────────────────────────────────────────────────────────────────────────────┤
│                                                                            │
│ 👨‍🦰 유저 입력: "qwer12"                                                  │
│                                                                            │
│ 1️⃣ [ 소금(Salt) 생성 ] : 무작위 랜덤 문자열 "X7$m" 생성                   │
│       합체 ──▶ "qwer12X7$m"                                                │
│                                                                            │
│ 2️⃣ [ KDF 지연 연산 (Work Factor = 12번 꼬기) ]                            │
│       믹서기 빙빙빙! (일부러 CPU를 혹사시키며 0.3초 동안 연산 반복)        │
│                                                                            │
│ 3️⃣ [ DB 최종 저장 형태 ]                                                  │
│     $2b$12$X7$m...(Salt정보)...$8J9s2L...(최종 해시값)                     │
│                                                                            │
│ 😈 [ 해커가 털어갔을 때 ]                                                  │
│   - "어? 소금이 다 달라서 내 해킹사전(Rainbow Table) 못 쓰네?" (1차 좌절)  │
│   - "직접 연산해서 찍어맞추자! ..근데 1번 맞추는데 0.3초나 걸려?           │
│      10억 번 찍어보려면 10년 걸리네 ㅅㅂ 포기!" (2차 완벽 방어 🟢)         │
└────────────────────────────────────────────────────────────────────────────┘
```

### Ⅲ. 면접 필수! KDF 알고리즘 4대장 진화사

단순한 SHA-256을 버리고, 업계가 어떤 '고의적 지연 알고리즘(KDF)'으로 진화해 왔는지 외워야 한다.

1. **PBKDF2**: 미국 정부(NIST)의 공식 인정을 받은 근본 알고리즘. SHA 해시를 정해진 횟수만큼 미친 듯이 반복(Iteration)하게 해서 **CPU를 혹사**시킨다. 하지만 해커들이 그래픽카드(GPU)를 수백 개 연결해서 병렬 연산으로 깨부수기 시작했다.
2. **bcrypt (비크립트)**: 브루스 슈나이어가 만들었으며 현재 가장 대중적으로 쓰인다. 이놈은 연산할 때 메모리(RAM)를 꽤 많이 잡아먹게 설계되어서, 해커가 GPU 코어를 수천 개 달아도 메모리 병목 때문에 GPU 해킹이 깡통이 된다.
3. **scrypt (에스크립트)**: bcrypt보다 훨씬 독종이다. "아예 램(RAM)을 미친 듯이 갉아먹게 만들자!" 해커가 이걸 뚫으려고 해킹 전용 칩(ASIC)을 만들려면 메모리 칩을 엄청나게 달아야 해서 칩 제작 비용만 수십억 원이 들게 만들어버렸다 (비용 파괴 방어).
4. **Argon2 (아르곤2)**: 2015년 전 세계 패스워드 해싱 대회(PHC)에서 우승한 **최종 보스이자 가장 현대적인 권장 표준**. CPU, 메모리, 심지어 병렬 처리(Threads) 정도까지 개발자가 자유자재로 세팅할 수 있는 궁극의 방어 무기다. OWASP가 1순위로 추천한다.

### Ⅳ. 결론

"가장 훌륭한 방어는 성벽을 높이는 것이 아니라, 공격자의 뼈를 깎고 지치게 만드는 것이다."
비밀번호 저장 아키텍처는 창과 방패의 모순을 가장 극적으로 보여준다. 빠르고 효율적인 것이 미덕인 소프트웨어 공학의 세계에서, 유일하게 **"어떻게 하면 일부러 더 느리고, 더 무겁고, 더 비효율적인 연산을 하게 만들까?"**를 고민하는 이질적인 영역이기 때문이다. Salt를 쳐서 공간의 우위(Rainbow Table)를 무력화하고, Argon2 같은 KDF를 도입하여 시간의 우위(Brute-force)를 뺏어버리는 이 설계는 단순한 암호학을 넘어선 해커와의 치열한 심리전이자 경제학적 비용 소모전의 정점이다.

### 📌 관련 개념 맵
* **전제되는 취약점**: Cryptographic Failures (OWASP 2위 - 479번 문서)
* **보안 기초**: 일방향 해시 함수 (One-way Hash - 504번 문서)
* **해커의 공격 기법**: 레인보우 테이블 공격 (Rainbow Table), 무차별 대입 공격 (Brute-force)
* **주요 KDF 알고리즘**: PBKDF2 $\rightarrow$ bcrypt $\rightarrow$ scrypt $\rightarrow$ Argon2

### 👶 어린이를 위한 3줄 비유 설명
1. 해커가 우리 집 자물쇠(비밀번호)를 풀려고 열쇠 1억 개를 엄청나게 빨리 찰칵찰칵 꽂아보고 있어요. 이대로면 10분 만에 뚫리겠죠.
2. 그래서 자물쇠 구멍 안에 찐득찐득한 꿀이랑 껌(KDF 지연 기술)을 잔뜩 발라놨어요! 이제 열쇠를 한 번 꽂고 돌리는 데 1초씩이나 엄청 오래 걸리게 된 거예요.
3. 원래 주인이 집에 들어갈 때는 1초만 기다리면 되니까 아무 상관 없지만, 열쇠 1억 개를 꽂아봐야 하는 해커는 1억 초(약 3년)나 걸리니까 너무 힘들어서 결국 포기하고 도망가게 만드는 멋진 함정이랍니다!
