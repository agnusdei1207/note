# 508. 인증 (Authentication) 트렌드 - 패스워드의 종말

> ⚠️ 이 문서는 해커들이 유저가 머리로 외우는 지식 기반의 비밀번호(Password)를 무차별 대입이나 피싱으로 너무 쉽게 박살 내고 있는 현실을 타파하기 위해, **인간의 뇌(기억)에서 비밀번호를 아예 지워버리고 생체 인식과 하드웨어 암호 키를 결합하여 해킹이 불가능한 구조를 짠 현대 인증 기술(MFA, FIDO, 패스워드리스)**을 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 지난 50년간 인터넷을 지배했던 "아이디와 텍스트 비밀번호" 체계는 완전히 실패했다. 현재의 보안 트렌드는 유저가 비밀번호를 타이핑하는 행위 자체를 없애는 **패스워드리스(Passwordless)**로 향하고 있다.
> 2. **과도기 (MFA)**: 비밀번호가 털릴 것에 대비해, 내가 알고 있는 것(비번) 외에 내가 가지고 있는 것(핸드폰 OTP)이나 내 몸(지문)을 두 개 이상 섞어서 확인하는 **다중 인증(MFA)**이 현재 보안 업계의 법적 강제 사항이 되었다.
> 3. **미래 (FIDO & WebAuthn)**: 비밀번호를 아예 폐기하고, 내 스마트폰 기기 자체에 해커가 절대 훔쳐 갈 수 없는 마스터키(개인키)를 박아두고 지문/얼굴 인식으로만 승인하는 FIDO(Fast IDentity Online) 기술이 글로벌 1순위 표준으로 등극했다.

---

### Ⅰ. 개요: 123456의 저주 (Context & Necessity)
해커가 시스템을 뚫는 가장 쉬운 방법은 뭘까? SQL 인젝션? 
아니다. 그냥 구글에 널려있는, 다른 사이트에서 털린 고객들의 `아이디/비밀번호` 명부를 다운받아서 우리 사이트 로그인 칸에 봇(Bot)으로 1초에 1,000번씩 쑤셔 넣는 것(크레덴셜 스터핑)이다. 인간은 게으르기 때문에 모든 사이트의 비밀번호를 똑같이 쓴다.

보안 업계는 절망했다. "사용자에게 '대문자, 소문자, 특수문자 섞어서 12자리로 만드세요!'라고 강요해 봤자, 다들 수첩이나 포스트잇에 적어두고 다니잖아? **결국 인간의 뇌(기억)에 의존하는 인증(Knowledge-based)은 무조건 실패한다!**"

여기서 두 가지 혁명이 시작되었다.
하나는 "비밀번호 털려도 한 번 더 검사하자"는 **MFA(다중 인증)**이고,
다른 하나는 "아예 비밀번호를 없애버리자"는 **패스워드리스(FIDO)**다.

> 📢 **섹션 요약 비유**: 옛날엔 은행 금고를 열 때 '숫자 다이얼(비밀번호)'만 돌리면 열렸습니다. 도둑이 뒤에서 숫자만 훔쳐보면 금고가 털리죠. 그래서 지금은 다이얼을 맞춘 다음(1차), 은행원에게 내 얼굴을 보여주고 지문을 찍어야(2차, 생체 인식) 문이 열립니다. 나아가 미래의 금고는 아예 다이얼이 없고 내 지문과 망막만으로 문이 쓱 열리는 마법의 문(패스워드리스)으로 바뀌고 있습니다.

---

### Ⅱ. MFA (Multi-Factor Authentication) - 다중 인증

인증의 3대 요소(Factor) 중 서로 다른 종류의 2개 이상을 조합해야만 문을 열어주는 시스템이다.

1. **지식 기반 (Knowledge)**: 내가 알고 있는 것 (예: 아이디/비밀번호, 핀 번호)
2. **소유 기반 (Possession)**: 내가 가지고 있는 것 (예: 핸드폰 문자 SMS, OTP 앱, 스마트 카드)
3. **생체/특성 기반 (Inherence)**: 나 자신인 것 (예: 지문, 홍채, 정맥 인식)

* **방어 원리**: 해커가 피싱 사이트로 내 `비밀번호(지식)`를 훔쳐 갔다고 치자. 해커가 러시아에서 로그인을 시도해도, 내 핸드폰(소유)으로 날아온 `6자리 OTP 번호`를 해커는 절대 알 수 없으므로 1차 방어선이 뚫려도 2차 방어선에서 해킹이 100% 무산된다.

---

### Ⅲ. 패스워드의 종말: FIDO와 WebAuthn

MFA도 귀찮다. 매번 핸드폰 찾아서 6자리 번호를 타이핑해야 한다. 그래서 애플, 구글, MS가 연합하여 **FIDO (Fast IDentity Online)**라는 혁명적인 글로벌 동맹을 맺었다.

#### 1. FIDO의 작동 원리 (비대칭키 암호화 마술)
- FIDO의 핵심은 **"비밀번호를 서버로 전송하지 않는다"**는 것이다.
- 내가 회원가입할 때 내 핸드폰 안의 칩(안전한 공간, TEE/Secure Enclave)이 몰래 자물쇠(공개키)와 열쇠(개인키) 쌍을 만든다.
- 핸드폰은 자물쇠(공개키)만 휙 던져서 서버에 저장시킨다. **진짜 열쇠(개인키)는 핸드폰 밖으로 평생 1cm도 나가지 않는다.**
- 나중에 로그인할 때, 서버가 나한테 "너 진짜 너 맞아? 이 편지 방금 준 자물쇠로 잠갔으니까 네 열쇠로 열어봐"라며 퀴즈(Challenge)를 보낸다.
- 나는 핸드폰의 지문 인식기에 손가락을 댄다. 지문이 맞으면, 폰 내부 칩이 열쇠(개인키)를 꺼내서 편지를 푼 다음, 정답(서명)만 서버로 딱 돌려보낸다. 로그인 성공!

#### 2. 해킹이 수학적으로 불가능한 이유
- 서버가 해킹당해서 DB가 다 털려도 해커가 가져간 건 빈 '자물쇠(공개키)'뿐이다.
- 해커가 중간에서 인터넷 통신망(패킷)을 엿들어도 열쇠(비밀번호)가 날아다니질 않으니 훔칠 게 없다.
- 해커가 지문 화면을 가짜 피싱 사이트로 띄워도 폰 내부 칩이 속지 않는다. (해커 멸망)

#### 3. WebAuthn (웹 인증 표준)
- 원래 FIDO는 앱(App)이나 USB 보안키에서만 쓸 수 있었다. 
- 이 멋진 기술을 웹 브라우저(크롬, 사파리)에서도 `navigator.credentials.create()` 자바스크립트 코드 한 줄로 웹사이트에서 지문/얼굴 인식 로그인을 띄울 수 있게 만든 글로벌 W3C 웹 표준 규격이 바로 **WebAuthn**이다.

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│           기존 비밀번호 방식 vs FIDO(패스워드리스) 아키텍처 보안 비교       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│ ❌ [ 기존 비밀번호 방식 (털리기 딱 좋음) ]                                  │
│   유저 (비번: 1234) ────(인터넷 전송)────▶ 서버 DB (해시: 8d96..)           │
│   ▶ 해커 1: 전송 중에 패킷을 엿듣고 "1234" 탈취!                            │
│   ▶ 해커 2: 서버 DB를 털고 레인보우 테이블로 돌려서 "1234" 해독!            │
│                                                                             │
│ 🛡️ [ FIDO 기반 패스워드리스 방식 (해킹 불가) ]                              │
│   유저 스마트폰 내부 칩: [개인키(절대 안나감)]                              │
│                                                                             │
│   1️⃣ 서버: "너 맞으면 내가 주는 문장에 전자서명 쾅 찍어서 보내봐!"         │
│   2️⃣ 폰: (지문 인식 통과) -> 개인키로 서명(도장)만 찍어서 인터넷 전송      │
│   3️⃣ 서버: DB에 저장해둔 [공개키]로 도장 모양 검사 -> "오! 맞네 통과!"     │
│                                                                             │
│   ▶ 해커: 인터넷을 백날 훔쳐봐도 '도장 찍힌 종이'만 날아다니지,             │
│           진짜 도장(개인키/비밀번호)은 인터넷에 안 나와서 훔칠 게 없음! 😭  │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Ⅳ. 결론

"비밀번호를 외우게 하는 시스템은 설계부터 잘못된 폭력이다."
인증(Authentication) 시스템의 진화는, 보안 사고의 책임을 '비밀번호를 쉽게 만든 사용자'에게서 '근본적인 암호학을 시스템에 깔아둔 기업'으로 옮겨가는 위대한 과정이다. 해커가 AI와 GPU 연산을 무기로 브루트포스(Brute-force)의 극한을 달리는 21세기에, 인간의 나약한 기억력(지식 기반 인증)에 의존하는 것은 서버의 대문을 활짝 열어놓은 것과 같다. OTP와 SMS를 거치는 MFA 과도기를 지나, 이제 지문 한 번 터치로 모든 복잡한 비대칭키 암호화(RSA/ECC) 수학 연산을 백그라운드에서 숨겨버리는 FIDO와 WebAuthn이야말로, '편리함과 보안은 반비례한다'는 수십 년간의 IT 징크스를 깨부순 가장 완벽한 사용자 경험(UX)이자 궁극의 방패다.

### 📌 관련 개념 맵
* **해결하려는 취약점**: Identification and Authentication Failures (OWASP 7위 - 484번 문서)
* **인증 3요소**: 지식(What you know), 소유(What you have), 생체(What you are)
* **기반 암호 기술**: 공개키/비대칭키 암호 (RSA, ECC - 504번 문서 참조)
* **관련 표준 체계**: FIDO Alliance, W3C WebAuthn, 패스키(Passkey - 애플/구글/MS 연합)

### 👶 어린이를 위한 3줄 비유 설명
1. 옛날엔 집 문을 열 때 '비밀번호(1234)'를 외워서 눌렀어요. 근데 도둑이 뒤에서 몰래 보고 외워서 우리 집을 털어갔죠.
2. 그래서 도둑을 막으려고 비밀번호 누르고 나서, 내 주머니에 있는 '핸드폰으로 온 6자리 번호'까지 쳐야 문이 열리게 바꿨어요. 도둑은 내 폰이 없으니 절대 못 들어오죠! (이게 **MFA 다중 인증**이에요)
3. 근데 이것도 매번 번호 누르기 귀찮아요! 그래서 아예 번호판을 부숴버리고, 내 '손가락 지문'을 대면 철컥! 하고 자동으로 열리는 마법의 스마트 문(패스워드리스, **FIDO**)을 만들었답니다. 내 손가락은 도둑이 훔쳐 갈 수 없으니 100% 안전하죠!
