> **핵심 인사이트**
> 1. PERT(Program Evaluation and Review Technique)는 3점 추정(낙관·비관·최빈)을 베타 분포로 모델링해 불확실한 프로젝트의 기간 기대치와 표준편차를 계산하고, 완료 확률을 정량적으로 추정하는 확률적 일정 관리 기법이다.
> 2. PERT의 핵심 수식: 기대치 μ = (o + 4m + p) / 6, 분산 σ² = ((p-o)/6)², 표준편차 σ = (p-o)/6 — 최빈치(m)에 4배 가중치를 부여해 현실적 추정치에 집중하면서도 불확실성(분산)을 표현한다.
> 3. PERT는 CPM과 결합해 임계 경로의 확률적 완료 기간을 계산하는 데 사용되며, R&D·신제품 개발처럼 과거 경험이 없어 확정적 기간 추정이 불가능한 프로젝트에서 필수적이다.

## I. PERT 3점 추정

```
3가지 추정치:
  o (Optimistic): 모든 것이 잘 될 때 최단 기간
  m (Most Likely): 정상 조건에서 가장 가능성 높은 기간
  p (Pessimistic): 최악의 상황에서 최장 기간

기대치 (Expected Time, TE):
  TE = (o + 4m + p) / 6
  
  최빈치 m에 4배 가중치:
    베타 분포의 평균 근사식
    평균이 최빈치 쪽으로 편향되도록

표준편차 (Standard Deviation, σ):
  σ = (p - o) / 6
  
분산 (Variance):
  σ² = ((p - o) / 6)²
```

| 예시 작업    | o(낙관) | m(최빈) | p(비관) | TE=(o+4m+p)/6 | σ=(p-o)/6 |
|-----------|-------|-------|-------|------------|---------|
| DB 설계    | 5일   | 8일   | 11일  | 8.0일      | 1.0일   |
| API 개발   | 3일   | 7일   | 21일  | 8.0일      | 3.0일   |
| UI 구현    | 4일   | 6일   | 8일   | 6.0일      | 0.67일  |

> 📢 **섹션 요약 비유**: DB 설계와 API 개발의 기대치는 같은 8일이지만, API의 표준편차(3일)가 DB(1일)의 3배 — 더 불확실하고 리스크가 높다.

---

## II. PERT vs CPM 비교

```
PERT:
  기간: 3점 추정 (확률적)
  결과: 기대 기간 + 완료 확률
  목적: 일정 불확실성 관리
  사용: R&D, 신제품 개발, 소프트웨어

CPM:
  기간: 고정 1개 값 (확정적)
  결과: 임계 경로 기간 + 비용 최적화
  목적: 비용-시간 트레이드오프
  사용: 건설, 제조, 반복 프로젝트

실무에서 PERT-CPM 통합:
  임계 경로 = CPM으로 찾기
  기간 추정 = PERT 3점 추정 적용
  완료 확률 = 정규분포 Z 통계 계산
```

> 📢 **섹션 요약 비유**: CPM은 베테랑 건설업자의 정밀 견적, PERT는 연구자의 "아마 3개월이지만 최악엔 6개월"같은 범위 추정.

---

## III. 임계 경로 완료 확률 계산

```
임계 경로 기대치와 분산:
  여러 작업의 합계 -> 기대치와 분산 모두 합산

  TE_프로젝트 = ΣTE_i  (임계 경로 작업들)
  σ²_프로젝트 = Σσ²_i

완료 확률 계산 (정규분포 근사):
  D = 목표 완료 기간
  Z = (D - TE_프로젝트) / σ_프로젝트
  
  Z 통계표에서 확률 조회:
    Z=0:    50% 확률
    Z=1.0:  84% 확률
    Z=1.65: 95% 확률
    Z=2.0:  97.7% 확률
```

```
예시:
  임계 경로: A-B-D-F (TE=25일, σ²=4, σ=2)
  목표: 28일 내 완료
  
  Z = (28 - 25) / 2 = 1.5
  P(Z ≤ 1.5) ≈ 93.3%
  
  -> 28일 안에 완료할 확률 93.3%
```

> 📢 **섹션 요약 비유**: PERT로 "25일에 완료 예상, 28일이면 93% 확률"처럼 기간과 확률을 함께 고객에게 제시할 수 있다.

## IV. 몬테카를로 시뮬레이션 응용

```
PERT 한계:
  임계 경로만 분석 -> 비임계 경로의 불확실성 무시
  분포를 정규분포로 가정 -> 실제와 다를 수 있음

몬테카를로 시뮬레이션:
  각 작업 기간을 분포에서 무작위 샘플링
  전체 프로젝트 기간 계산
  10,000번 반복 -> 히스토그램으로 완료 분포 도출
  
  결과: "30일 내 완료 확률 80%, 35일 내 99%" 같은
        정밀 확률 도출
  
  도구: Oracle Primavera Risk Analysis, @RISK
```

> 📢 **섹션 요약 비유**: PERT는 한 가지 경우의 수 계산, 몬테카를로는 1만 가지 상황을 시뮬레이션해서 가장 현실적인 확률 분포를 찾는 것.

## V. 실무 시나리오 — 소프트웨어 출시 PERT

```
스타트업 신제품 개발 PERT:

작업별 3점 추정:
  요구사항 (o=1, m=2, p=3주): TE=2.0, σ=0.33
  설계 (o=2, m=4, p=12주):    TE=4.67, σ=1.67
  개발 (o=6, m=10, p=20주):   TE=10.67, σ=2.33
  테스트 (o=2, m=4, p=6주):   TE=4.0, σ=0.67
  출시 (o=1, m=2, p=3주):     TE=2.0, σ=0.33

임계 경로 합계:
  TE = 2.0+4.67+10.67+4.0+2.0 = 23.34주
  σ² = 0.11+2.79+5.43+0.45+0.11 = 8.89
  σ = √8.89 = 2.98주

완료 확률:
  24주: Z=(24-23.34)/2.98=0.22 -> 59%
  28주: Z=(28-23.34)/2.98=1.56 -> 94%
  -> VC에게 "28주 출시 94% 확률"로 제안!
```

> 📢 **섹션 요약 비유**: 투자자에게 "개발 기간 6개월"이 아니라 "6.5개월에 94% 확률로 완료"라고 말할 수 있다 — 훨씬 신뢰도 높은 일정 제시.

## 👶 어린이를 위한 3줄 비유 설명

1. PERT는 "가장 빠를 때, 보통, 가장 늦을 때"의 세 가지 경우를 평균 내서 현실적인 일정을 만드는 방법이에요.
2. 수학 공식으로 "30일 안에 완료할 확률이 95%"처럼 확률도 알 수 있어요.
3. 우주선 발사나 신약 개발처럼 한 번도 해보지 않은 프로젝트에서 일정을 잡을 때 특히 유용하답니다!
