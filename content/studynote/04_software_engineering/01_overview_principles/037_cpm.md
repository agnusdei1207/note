> **핵심 인사이트**
> 1. CPM(Critical Path Method)은 프로젝트의 모든 작업 간 의존 관계를 네트워크로 표현하고, 전체 프로젝트 기간을 결정하는 가장 긴 경로(임계 경로, Critical Path)를 찾는 일정 관리 기법이다.
> 2. 임계 경로 상의 작업은 여유 시간(Float/Slack)이 0이므로 한 작업이 지연되면 프로젝트 전체가 지연된다 — 프로젝트 관리자는 임계 경로에 가장 많은 자원을 집중해야 한다.
> 3. CPM과 PERT(Program Evaluation and Review Technique)의 핵심 차이는 작업 기간의 확실성 — CPM은 확정적 기간을, PERT는 낙관·비관·최빈 3가지 추정치의 가중 평균을 사용한다.

---

## I. CPM 기본 개념

```
네트워크 다이어그램 (AOA 방식):
  Activity On Arrow — 화살표가 작업, 원이 이벤트

  [시작]--A(3일)-->[이벤트1]--C(4일)-->[이벤트3]
                           |                    |
                       B(2일)              D(1일)
                           |                    |
                     [이벤트2]--E(5일)-->[이벤트4]-->[완료]

AON (Activity On Node) 방식:
  노드가 작업, 화살표가 의존 관계
  더 직관적, MS Project 등에서 사용
```

| 용어       | 설명                                    |
|----------|----------------------------------------|
| ES(Early Start) | 가장 빠른 시작 시점              |
| EF(Early Finish)| 가장 빠른 완료 시점 = ES + Duration|
| LS(Late Start)  | 가장 늦은 시작 시점               |
| LF(Late Finish) | 가장 늦은 완료 시점               |
| Float/Slack | 지연 가능한 여유 시간 = LS - ES   |

> 📢 **섹션 요약 비유**: 공항 환승 시간표 — 비행기 출발까지 여유 시간이 없는 환승(임계 경로)은 절대 지연하면 안 된다.

## II. 전진 계산 (Forward Pass) / 후진 계산 (Backward Pass)

```
전진 계산 (Early 시간 계산):
  ES[시작] = 0
  EF[작업] = ES[작업] + Duration[작업]
  ES[후속] = max(EF[선행들])

후진 계산 (Late 시간 계산):
  LF[완료] = EF[완료]  (프로젝트 마감일)
  LS[작업] = LF[작업] - Duration[작업]
  LF[선행] = min(LS[후속들])

Float 계산:
  Total Float = LS - ES = LF - EF
  
임계 경로 조건:
  Float = 0인 작업들의 연결
```

> 📢 **섹션 요약 비유**: 전진 계산은 "최대한 빨리 시작"을 계산하고, 후진 계산은 "최대한 늦게 시작해도 마감을 지키는 시점"을 거꾸로 계산하는 것.

---

## III. CPM vs PERT 비교

```
CPM:
  작업 기간: 고정 1개 값 (확정적)
  용도: 반복적 프로젝트 (건설, 제조)
  초점: 비용-시간 트레이드오프 최적화

PERT:
  작업 기간: 3점 추정
    낙관적(o) + 4×최빈(m) + 비관적(p)
    기대치 = (o + 4m + p) / 6
    분산 = ((p-o)/6)^2
  용도: 불확실한 프로젝트 (R&D, 신기술)
  초점: 일정 완료 확률 추정 (Z-통계)
```

| 비교    | CPM              | PERT               |
|-------|------------------|--------------------|
| 기간  | 확정적 1개 값     | 3점 추정 가중 평균   |
| 특징  | 비용 최적화       | 확률적 일정 분석     |
| 사용처| 건설, 제조       | R&D, 신제품 개발    |

> 📢 **섹션 요약 비유**: CPM은 경험 있는 건설업자의 정확한 견적, PERT는 연구자의 낙관/비관 시나리오를 모두 고려한 범위 추정.

## IV. 임계 경로 단축 기법

```
CPM 비용-시간 트레이드오프:

정상 작업 기간 -> 단축 시 비용 증가
  크래싱 (Crashing): 자원 추가 투입으로 기간 단축
  패스트 트래킹 (Fast Tracking): 원래 순차 작업을 병렬 수행
  
  단축 결정 기준:
    단축 비용 < 지연 비용(페널티)이면 단축

  한계:
    임계 경로 작업만 단축해도 의미 있음
    비임계 경로가 새 임계 경로가 될 수 있음
```

> 📢 **섹션 요약 비유**: 임계 경로 단축은 고속도로 차선 늘리기 — 이미 막히지 않는 국도(비임계 경로)는 아무리 넓혀도 도착 시간이 줄지 않는다.

## V. 실무 시나리오 — 소프트웨어 릴리스 CPM

```
6개월 소프트웨어 릴리스 프로젝트 CPM:

작업 목록:
  A: 요구사항 분석 (2주)
  B: UI 설계 (3주, A 후)
  C: DB 설계 (2주, A 후)
  D: 프론트엔드 개발 (6주, B 후)
  E: 백엔드 개발 (8주, C 후)
  F: 통합 테스트 (3주, D, E 후)
  G: QA 검증 (2주, F 후)

경로별 기간:
  A-B-D-F-G: 2+3+6+3+2 = 16주
  A-C-E-F-G: 2+2+8+3+2 = 17주 <- 임계 경로!

결론: 백엔드(E)가 1주 늦으면 전체 1주 지연
      -> 백엔드 개발에 인력 집중
```

> 📢 **섹션 요약 비유**: 소프트웨어 프로젝트에서 백엔드 개발자가 임계 경로에 있다면, 그 팀의 생산성이 전체 출시 일정을 결정한다.

## 📌 관련 개념 맵

```
CPM (임계 경로 기법)
+-- 핵심 계산
|   +-- 전진 계산 (ES, EF)
|   +-- 후진 계산 (LS, LF)
|   +-- Float = LS - ES
+-- 임계 경로
|   +-- Float = 0인 작업들의 연결
|   +-- 프로젝트 최소 완료 기간 결정
+-- 관련 기법
|   +-- PERT (3점 추정)
|   +-- 간트 차트 (시각화)
|   +-- WBS (작업 분류 체계)
+-- 일정 단축
    +-- 크래싱 (비용-시간 트레이드오프)
    +-- 패스트 트래킹 (병렬화)
```

## 👶 어린이를 위한 3줄 비유 설명

1. CPM은 집 짓기 순서처럼 "기초 공사 → 벽 세우기 → 지붕 올리기"의 의존 관계를 따라 가장 오래 걸리는 길을 찾는 방법이에요.
2. 그 가장 긴 길(임계 경로)에 있는 작업이 하루 늦으면 집 완성이 하루 늦어져요.
3. 그래서 프로젝트 매니저는 임계 경로에 있는 팀에게 가장 많은 도움을 줘야 해요!
