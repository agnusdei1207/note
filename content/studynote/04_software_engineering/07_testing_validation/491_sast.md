## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SAST (Static Application Security Testing)는 실행 전 코드 분석이다.
> 2. **가치**: 개발 초기에 보안 약점을 잡아 비용을 낮춘다.
> 3. **판단 포인트**: 소스코드의 패턴과 데이터 흐름을 본다.

---

## Ⅰ. 개요 및 필요성

SAST는 코드를 실행하지 않고 분석한다. 그래서 빌드 단계나 코드 리뷰 단계에 넣기 좋다.

보안 규칙을 자동화해 반복적인 실수를 줄일 수 있다.

- **📢 섹션 요약 비유**: 시험지를 제출하기 전에 오탈자를 먼저 찾는 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

SAST는 소스코드, 바이트코드, AST (Abstract Syntax Tree) 등을 분석할 수 있다.

```text
소스코드 -> 정적 분석 -> 규칙/패턴 매칭 -> 경고
```

| 특징 | 의미 |
|:---|:---|
| 비실행 | 코드만 검사 |
| 빠른 피드백 | 개발 단계 적용 |
| 규칙 기반 | 취약 패턴 탐지 |

- **📢 섹션 요약 비유**: 요리를 하지 않아도 재료 목록만 보고 상한 걸 찾는 것과 같다.

---

## Ⅲ. 비교 및 연결

SAST는 DAST보다 빠르지만, 런타임 문제를 직접 보지는 못한다.

| 구분 | SAST | DAST |
|:---|:---|:---|
| 시점 | 개발/빌드 전 | 실행 중 |
| 대상 | 코드 구조 | 동작 결과 |
| 강점 | 빠른 탐지 | 실제 공격 재현 |

그래서 둘을 함께 쓰면 보완적이다.

- **📢 섹션 요약 비유**: 설계도 검토와 완공 후 점검은 서로 다른 문제를 잡아낸다.

---

## Ⅳ. 실무 적용 및 기술사 판단

실무에서는 PR 검사, CI (Continuous Integration), 보안 기준선 적용에 유용하다.

점검 포인트는 다음과 같다.
1. 경고가 너무 많아 무시되지는 않는가?
2. 우선순위를 조정해 실제 위험부터 보는가?
3. 오탐을 줄이기 위한 규칙 관리가 있는가?

- **📢 섹션 요약 비유**: 경보기는 많을수록 좋지만, 계속 울리면 아무도 안 듣게 된다.

---

## Ⅴ. 기대효과 및 결론

SAST는 개발 초기에 취약점을 줄여 전체 보안 품질을 끌어올린다.

결론적으로 이 항목은 "실행 전 코드 보안 분석"이다.

- **📢 섹션 요약 비유**: 집을 짓기 전에 기둥이 잘못됐는지 보는 검사다.
