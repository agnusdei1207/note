## Ⅰ. 개요 및 필요성

행정안전부와 KISA (Korea Internet & Security Agency)는 안전한 소프트웨어 개발을 위한 가이드를 제시한다.

현장에서 가장 중요한 건 체크리스트를 외우는 게 아니라, 개발 과정에 녹여 넣는 것이다.

- **📢 섹션 요약 비유**: 학교에서 안전수칙을 붙여 두는 것처럼, 개발에도 기본 규칙이 필요하다.

---

## Ⅱ. 아키텍처 및 핵심 원리

가이드는 입력 검증, 인증, 세션, 접근 제어, 오류 처리, 암호화 같은 분야를 포괄한다.

```text
요구사항 -> 설계 -> 구현 -> 검토 -> 배포
         ^ 보안 약점 점검
```

| 범주 | 의미 |
|:---|:---|
| 입력 | 외부 데이터 통제 |
| 인증/세션 | 사용자 확인 |
| 암호화 | 민감 정보 보호 |

- **📢 섹션 요약 비유**: 집을 지을 때 전기, 수도, 잠금장치를 설계도에 같이 넣는 것과 같다.
