## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 이미지는 여러 계층으로 쌓이므로 불필요한 계층을 줄이는 것이 중요하다.
> 2. **가치**: 빌드 속도와 전송 크기를 줄인다.
> 3. **판단 포인트**: 캐시 효율, 레이어 수, 보안 표면을 본다.

---

## Ⅰ. 개요 및 필요성

Docker 이미지는 명령 하나마다 계층이 쌓일 수 있다. 계층이 많고 크면 배포가 느려진다.

- **📢 섹션 요약 비유**: 옷을 너무 많이 겹쳐 입으면 무겁고 움직이기 불편한 것과 같다.

---

## Ⅱ. 아키텍처 및 핵심 원리

이미지 계층은 변경된 부분만 다시 저장한다. 그래서 레이어 정리가 잘 되면 캐시와 전송 효율이 좋아진다.

```text
Base -> Dependencies -> App -> Config
```

| 방법 | 효과 |
|:---|:---|
| Multi-stage build | 빌드 도구 제거 |
| Layer merge | 계층 감소 |
| .dockerignore | 불필요 파일 제외 |

- **📢 섹션 요약 비유**: 짐을 상자에 나눠 넣되, 꼭 필요한 것만 담는 방식이다.

---

## Ⅲ. 비교 및 연결

작은 이미지일수록 배포와 롤백이 빠르다. 캐시를 잘 쓰면 반복 빌드가 빨라진다.

| 구분 | 최적화 전 | 최적화 후 |
|:---|:---|:---|
| 크기 | 큼 | 작음 |
| 전송 시간 | 김 | 짧음 |
| 공격 면적 | 큼 | 작음 |

- **📢 섹션 요약 비유**: 큰 여행가방을 줄이면 이동이 훨씬 편해진다.

---

## Ⅳ. 실무 적용 및 기술사 판단

실무에서는 빌드 스테이지 분리, 의존성 고정, 불필요 파일 제거를 병행한다.

점검 포인트는 다음과 같다.
1. 런타임 이미지에 빌드 도구가 남아 있지 않은가?
2. 자주 바뀌는 것과 잘 안 바뀌는 것이 분리되는가?
3. 캐시를 해치지 않는 순서인가?

- **📢 섹션 요약 비유**: 창고 정리처럼 자주 쓰는 것과 안 쓰는 것을 나눠야 한다.

---

## Ⅴ. 기대효과 및 결론

레이어 최소화는 배포 성능과 보안을 동시에 개선한다.
