## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 경로 커버리지는 프로그램의 시작부터 끝까지 가능한 실행 경로를 모두 테스트하는 방식이다.
> 2. **가치**: 분기만 보는 것보다 흐름 전체를 확인할 수 있어 복잡한 로직 검증에 강하다.
> 3. **판단 포인트**: 반복문이 있으면 경로 수가 무한해질 수 있어, 현실적으로는 제한이 필요하다.

---

## Ⅰ. 개요 및 필요성

경로 커버리지(Path Coverage)는 제어 흐름 그래프(Control Flow Graph)의 모든 실행 경로를 확인하는 기법이다. 분기 하나만 보는 것이 아니라, 어떤 순서로 분기가 이어지는지까지 본다.

이 기법이 중요한 이유는 같은 분기라도 순서가 달라지면 결과가 달라질 수 있기 때문이다. 중첩 조건과 예외 처리가 많을수록 경로 기반 검증의 필요성이 커진다.

- **📢 섹션 요약 비유**: 목적지에 가는 길이 여러 개면, 어떤 길이 막히는지도 함께 확인해야 한다.

---

## Ⅱ. 아키텍처 및 핵심 원리

경로 커버리지는 시작 노드에서 종료 노드까지의 모든 가능한 경로를 의미한다. 순환이 없는 코드에서는 경로를 전부 나열할 수 있지만, 반복문이 있으면 경로 수가 급증한다.

```text
Start -> A? -> B? -> End
경로1: T -> T
경로2: T -> F
경로3: F
```

| 항목 | 설명 |
|:---|:---|
| 경로 | 시작에서 종료까지의 실행 순서 |
| 제어 흐름 그래프 | 프로그램의 흐름을 노드와 간선으로 표현 |
| 순환 복잡도 | 독립 경로 수를 계산하는 기준 |

경로 수가 많아질수록 테스트가 폭발하므로, 실무에서는 독립 경로 중심으로 접근한다.

- **📢 섹션 요약 비유**: 미로는 출구가 아니라 지나온 길까지 기억해야 제대로 검사된다.

---

## Ⅲ. 비교 및 연결

경로 커버리지는 결정 커버리지보다 넓은 관점을 제공한다. 결정 커버리지가 각 분기의 참/거짓이라면, 경로 커버리지는 그 분기들이 이어진 전체 흐름이다.

| 구분 | 본다 | 한계 |
|:---|:---|:---|
| 결정 커버리지 | 분기 결과 | 경로 조합은 놓칠 수 있음 |
| 경로 커버리지 | 실행 순서 전체 | 반복문에서 무한해질 수 있음 |

이 때문에 경로 커버리지는 기초 경로 테스팅(Basis Path Testing)과 함께 자주 언급된다.

- **📢 섹션 요약 비유**: 한 문만 보는 것은 문검사이고, 집 전체 동선을 보는 것은 입주 점검이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

실무에서는 분기 구조가 복잡한 승인 흐름, 예외 처리, 상태 전이 로직에 쓰인다. 다만 전체 경로를 다 보는 것은 비현실적일 수 있으므로, 중요한 독립 경로부터 우선한다.

체크 포인트는 다음과 같다.
1. 반복문이 있는지 먼저 확인한다.
2. 종료 조건이 잘 닫히는지 본다.
3. 독립 경로 수를 기준으로 테스트 우선순위를 정한다.

- **📢 섹션 요약 비유**: 모든 길을 다 걷는 대신, 가장 중요한 갈림길부터 확인한다.

---

## Ⅴ. 기대효과 및 결론

경로 커버리지는 흐름의 맥락까지 검증해 주기 때문에 복잡한 로직에서 유용하다. 하지만 경로 수가 늘어나면 유지 비용이 커진다.

결론적으로 이 기법은 "흐름 전체를 보는 강한 검증"이다. 그러나 실무에서는 독립 경로 중심으로 줄여서 써야 한다.

- **📢 섹션 요약 비유**: 여행 일정을 짤 때 모든 우회로를 다 넣지 않고 핵심 경로부터 확인하는 것과 같다.

| Basis Path Testing | 현실적인 경로 테스트 전략 |

### 👶 어린이를 위한 3줄 비유 설명

1. 미로에서 한 번만 들어가면 안 되고, 여러 길을 다 가봐야 해요.
2. 길이 많아지면 다 가기 어려워요.
3. 그래서 중요한 길부터 먼저 확인해요.
