# 402. 하향식 통합 (Top-down Integration)

## 핵심 인사이트 (3줄 요약)
> 1. 시스템의 큰 그림(UI 흐름, 메인 컨트롤러 로직)을 가장 초기에 검증하여 **사용자 눈에 보이는 뼈대를 우선 확보**하는 데 특화되어 있다.
> 2. 아직 개발되지 않은 하위 모듈 파트를 임시로 가짜 반환값을 던져주는 **스텁(Stub, 껍데기 함수)** 으로 대체하여 상위 모듈이 터지지 않도록 지탱한다.
> 3. 하위 코어 로직(DB 처리, 계산 등)의 결함이 너무 늦게 발견되면 상위 아키텍처 전체를 갈아엎어야 하는 리스크를 내포한다.

---
## Ⅰ. 아키텍처 및 원리
```text
  [ 하향식 뼈대 조립망 ]
        [ 메인 UI (Controller) ]
           |              |
      [ 결제(Stub) ]  [ 인증(Stub) ] <-- 아직 코딩 안 된 부분
```
---
## Ⅱ. 실무 적용 및 결론
화면(Frontend) 중심의 애자일 개발이나 사용자 피드백을 빨리 받아야 하는 모바일 앱 환경에서 유리하다. 하지만 금융처럼 하위 계층(DB 연동) 처리가 전체 설계의 사활을 쥐고 있는 시스템에서는 최악의 접근법이다.
