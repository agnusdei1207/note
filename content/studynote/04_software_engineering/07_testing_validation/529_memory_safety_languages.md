## 핵심 인사이트 (3줄 요약)
> 1. **본질**: Memory Safety는 버퍼 오버플로우와 use-after-free 같은 문제를 줄이는 것이다.
> 2. **가치**: Rust와 Go는 안전성 중심 언어로 주목받는다.
> 3. **판단 포인트**: 개발 편의와 안전성의 균형을 본다.

---

## Ⅰ. 개요 및 필요성

메모리 오류는 치명적 취약점으로 이어질 수 있다. 그래서 안전한 언어와 설계 방식이 중요해졌다.

Rust와 Go는 이런 흐름에서 주목받는다.

- **📢 섹션 요약 비유**: 장난감을 만들 때 못이 튀어나오지 않게 미리 설계하는 것이다.

---

## Ⅱ. 아키텍처 및 핵심 원리

메모리 안전성은 소유권, 가비지 컬렉션, 경계 검사 같은 방법으로 확보된다.

```text
안전한 언어 기능 -> 메모리 오류 감소 -> 보안 취약점 감소
```

| 언어 | 특징 |
|:---|:---|
| Rust | 소유권/빌림 검사 |
| Go | 가비지 컬렉션 |
| 공통 | 안전성 우선 |

- **📢 섹션 요약 비유**: 물건을 빌려 주고 돌려받는 규칙이 있으면 잃어버리기 어렵다.

---

## Ⅲ. 비교 및 연결

메모리 안전성은 성능과 개발 생산성과의 균형이 필요하다.

| 구분 | 전통 언어 | Rust/Go |
|:---|:---|:---|
| 안전성 | 개발자 책임 | 언어 지원 |
| 성능 | 높음 | 상황에 따라 다양 |
| 생산성 | 익숙함 | 학습 필요 |

보안 사고 예방 관점에서 채택이 늘고 있다.

- **📢 섹션 요약 비유**: 칼을 잘 쓰는 것도 중요하지만, 손 베지 않게 손잡이가 있는 게 더 안전하다.

---

## Ⅳ. 실무 적용 및 기술사 판단

실무에서는 새로운 서비스, 보안 민감 모듈, 시스템 도구에 점진 적용한다.

점검 포인트는 다음과 같다.
1. 메모리 오류가 줄어드는가?
2. 기존 시스템과 통합 가능한가?
3. 성능과 운영 복잡성이 허용되는가?

- **📢 섹션 요약 비유**: 새 안전벨트를 달 때도 차에 잘 맞아야 한다.

---

## Ⅴ. 기대효과 및 결론

메모리 안전성은 보안 결함의 큰 원인을 줄인다.

결론적으로 이 항목은 "안전한 언어 도입을 통한 취약점 감소"다.

- **📢 섹션 요약 비유**: 튼튼한 재료로 만들면 고장도 줄어든다.
