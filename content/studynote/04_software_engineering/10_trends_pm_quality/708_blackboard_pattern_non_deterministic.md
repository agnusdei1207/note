# 708. 블랙보드 패턴 비결정적 문제 해결

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 블랙보드 패턴(Blackboard Pattern)은 명확한 순서나 알고리즘으로 풀 수 없는 '비결정적(Non-deterministic)' 문제를 해결하기 위해, 여러 전문가(모듈)들이 하나의 공용 칠판(블랙보드)에 각자의 지식을 적어가며 점진적으로 정답을 만들어내는 소프트웨어 아키텍처다.
> 2. **가치**: 음성 인식, 영상 분석처럼 정해진 순서 없이 여러 인공지능(AI)이나 휴리스틱 모듈이 동시다발적으로 협력해야 하는 복잡한 인지(Cognitive) 문제를 유연하게 해결할 수 있다.
> 3. **판단 포인트**: 이 패턴은 제어 흐름이 예측 불가능하므로, 일반적인 업무용 시스템(결제, 주문)에 적용하면 시스템이 스파게티가 된다. "정해진 공식은 없지만, 여러 전문가가 머리를 맞대면 풀 수 있는" AI나 로보틱스 도메인에만 제한적으로 도입해야 한다.

---

## Ⅰ. 개요 및 필요성

소프트웨어의 99%는 '결정적(Deterministic)'이다. 입력 A가 들어오면, B 함수를 거쳐 C를 출력한다는 명확한 순서도(Flowchart)가 존재한다. 

하지만 인간의 지능을 흉내 내는 문제는 이 방식으로 풀 수 없다. 예를 들어, 로봇이 "저기 있는 빨간 사과 좀 집어줘"라는 말을 들었을 때, 음성 인식 모듈, 색깔 인식 모듈, 거리 측정 모듈이 어떤 순서로 동작해야 할까? 음성 인식이 완벽하지 않으면 카메라가 과일 바구니를 보고 "아, 사과를 말했구나!"라고 추론해서 빈칸을 메워야 한다.

이렇게 **미리 정해진 실행 순서(Control Flow) 없이, 여러 모듈이 각자 자기가 아는 정보만 조금씩 보태서 점진적으로 퍼즐을 완성해 나가는 구조**를 만들기 위해 고안된 것이 **블랙보드 패턴(Blackboard Pattern)**이다.

- **📢 섹션 요약 비유**: 어려운 수학 문제를 풀 때, 선생님이 칠판(Blackboard)에 문제를 적어두면, 기하학 천재, 대수학 천재, 산수 천재(전문가 모듈들)가 순서에 상관없이 칠판으로 뛰어나와 자기가 아는 공식을 적고 들어가면서 결국 정답을 찾아내는 과정이다.

---

## Ⅱ. 아키텍처 및 핵심 원리

블랙보드 아키텍처는 크게 세 가지 핵심 컴포넌트로 구성된다.

### 1. 3대 핵심 컴포넌트
| 컴포넌트 | 영문 명칭 | 역할 |
|:---|:---|:---|
| **블랙보드** | Blackboard | 모든 데이터, 가설, 부분적인 결과물이 기록되는 **중앙 전역 메모리 공간**. |
| **지식 소스** | Knowledge Source (KS) | 자기만의 전문 지식(알고리즘, 휴리스틱)을 가진 **독립적인 전문가 모듈들**. 칠판을 계속 쳐다보고 있다. |
| **제어 컴포넌트** | Control Component | 칠판의 상태를 보고 "지금은 A 전문가가 나설 타이밍이야!"라고 지시하는 **진행자(감독관)**. |

```text
┌──────────────────────────────────────────────────────────────┐
│                  블랙보드 패턴 아키텍처                      │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ ┌──────────────── [ 제어 컴포넌트 (Control) ] ─────────────┐ │
│ │ "지금 칠판에 소리 데이터가 올라왔으니 음성팀이 분석하세요!"│ │
│ └──────┬──────────────────────┬──────────────────────┬───────┘ │
│        │ (지시)               │ (지시)               │ (지시)  │
│        ▼                      ▼                      ▼        │
│ ┌─────────────┐        ┌─────────────┐        ┌─────────────┐│
│ │ 지식 소스 A │        │ 지식 소스 B │        │ 지식 소스 C ││
│ │ (음성 인식) │        │ (문맥 분석) │        │ (오타 교정) ││
│ └──────┬──────┘        └──────┬──────┘        └──────┬──────┘│
│        │ (단어 기록)          │ (의미 기록)          │ (수정)  │
│ ┌──────▼──────────────────────▼──────────────────────▼───────┐ │
│ │                 [ 블랙보드 (Blackboard) ]                  │ │
│ │  가설 1: "사과를 줘" (확률 80%)                            │ │
│ │  가설 2: "상자를 줘" (확률 20%) ➔ C가 오타 교정 후 삭제   │ │
│ └────────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

- **작동 방식**: 지식 소스들은 서로 직접 대화하지 않는다. 오직 블랙보드를 통해서만 소통한다. A가 블랙보드에 무언가를 적으면, 그걸 본 B가 뛰어나와 살을 붙이는 방식이다.

- **📢 섹션 요약 비유**: 강력계 형사팀의 '수사 본부 칠판'과 같다. 지문 감식반(KS 1), CCTV 분석반(KS 2), 프로파일러(KS 3)가 서로 회의할 필요 없이, 자기가 찾은 단서를 칠판(블랙보드)에 계속 붙여나가면 반장(제어)이 그걸 보고 범인을 추론해 낸다.

## Ⅲ. 비교 및 연결

블랙보드 패턴은 다른 이벤트 기반 아키텍처와 비슷해 보이지만, '문제 해결의 불확실성'에서 차이가 난다.

| 비교 항목 | 파이프라인 (Pipeline) | 발행/구독 (Pub/Sub) | 블랙보드 (Blackboard) |
|:---|:---|:---|:---|
| **실행 순서** | A $\rightarrow$ B $\rightarrow$ C (고정됨) | 이벤트 발생 시 즉시 실행 | **순서 없음 (제어자가 결정)** |
| **결과 예측성** | 100% 결정적 | 이벤트 흐름에 따라 다름 | **비결정적 (가설과 확률로 접근)** |
| **주요 사용처** | 컴파일러, 데이터 전처리 | MSA 비동기 통신 | **AI, 로보틱스, 자연어 처리** |
| **모듈 간 결합도**| 중간 | 매우 낮음 | 낮음 (블랙보드에만 의존) |

- **📢 섹션 요약 비유**: 파이프라인이 컨베이어 벨트에서 '자동차'를 조립하는 정해진 공정이라면, 블랙보드 패턴은 레고 블록을 바닥에 다 쏟아놓고 친구들이 모여서 '멋진 우주선'을 창의적으로 만들어가는 과정이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

과거에는 블랙보드 패턴이 너무 느리고 추적하기 어려워 학계(AI 연구)에서만 쓰였으나, 최근 자율주행과 복합 AI(Multimodal)의 발전으로 다시 조명받고 있다.

### 실무 의사결정 및 판단 포인트
1. **타겟 도메인의 복잡성**: 알고리즘으로 풀 수 있는 문제(예: 최단 경로 찾기, 정렬)에 블랙보드를 쓰면 속도만 느려지는 재앙이 된다. 지문 인식, 자율주행차의 센서 퓨전(Sensor Fusion, 레이더+카메라+라이다)처럼 **정답이 없고 확률적인 가설을 좁혀가야 하는 문제**에만 제한적으로 도입해야 한다.
2. **제어 컴포넌트의 병목(Bottleneck)**: 여러 전문가가 동시에 칠판에 글을 쓰려다 보면 칠판(메모리)에 락(Lock)이 걸리고, 진행자(제어 컴포넌트)가 너무 바빠져서 시스템 전체가 멈춘다. 고성능 병렬 처리 아키텍처와 효율적인 스케줄링 알고리즘이 필수적이다.
3. **디버깅의 극악한 난이도**: 블랙보드는 "왜 이런 결과가 나왔는지" 역추적하기가 지옥 같다. 실행할 때마다 지식 소스들이 칠판에 접근하는 타이밍이 달라져서 버그 재현이 안 되기 때문이다. 칠판의 모든 상태 변화를 스냅샷으로 기록하는 로깅(Logging) 체계가 반드시 수반되어야 한다.

### 안티패턴
- **일반 업무 시스템에 도입하기**: 일반적인 쇼핑몰의 '주문-결제-배송' 로직을 파이프라인이 아닌 블랙보드로 설계하는 것. "결제 모듈이 칠판을 보고 결제를 시도하고, 배송 모듈이 눈치껏 배달을 시작한다"는 식의 설계는 시스템을 완전히 통제 불능으로 만든다.

- **📢 섹션 요약 비유**: 블랙보드는 천재들의 브레인스토밍 회의다. 신제품 아이디어를 낼 때는 최고지만, '우편물 봉투 붙이기' 같은 단순 반복 노동을 천재들의 브레인스토밍 회의 방식으로 처리하면 회사가 망한다.

## Ⅴ. 기대효과 및 결론

블랙보드 패턴을 활용하면, 새로운 전문가 모듈(예: 새로운 AI 인식 엔진)이 개발되었을 때 기존 코드를 수정할 필요 없이 그냥 칠판 앞에 세워두기만 하면 된다. 즉, 시스템의 확장성과 유연성이 극대화된다.

결론적으로 블랙보드 패턴은 전통적인 소프트웨어 공학의 '명확함'을 포기하는 대신, 인간 지능의 '직관과 협력'을 소프트웨어로 모사한 아키텍처다. 다가오는 멀티모달(Multimodal) AI 시대에, 눈(Vision)과 귀(Audio)와 입(Text) 역할을 하는 여러 AI 모델을 하나로 엮어내는 마에스트로(진행자)의 역할로 블랙보드 패턴은 다시 화려하게 부활하고 있다.

- **📢 섹션 요약 비유**: 환자가 복통으로 쓰러졌을 때(문제), 컴퓨터처럼 "1번 검사, 2번 검사" 순서대로 하지 않고, 내과, 외과, 신경과 의사들이 빙 둘러서서(블랙보드) 각자의 지식으로 확률을 좁혀나가며 희귀병을 찾아내는 '하우스 박사'의 진단법이다.

### 📌 관련 개념 맵
- `[옵저버 패턴 (Observer Pattern)]`: 지식 소스(KS)들이 블랙보드의 상태 변화를 감지할 때 주로 사용하는 소프트웨어 디자인 패턴.
- `[이벤트 주도 아키텍처 (EDA)]`: 블랙보드 패턴을 네트워크 기반의 분산 시스템으로 확장할 때 뼈대가 되는 아키텍처 철학.
- `[센서 퓨전 (Sensor Fusion)]`: 자율주행 자동차에서 라이다, 레이더, 카메라 등 여러 센서(지식 소스)의 정보를 중앙에서 합쳐 장애물을 확률적으로 인지하는 블랙보드 패턴의 대표적 실무 응용.
- `[휴리스틱 (Heuristic)]`: 완벽한 정답을 보장할 수는 없지만, 실용적으로 쓸 만한 해답을 찾아내는 경험적 알고리즘. 블랙보드의 지식 소스들이 주로 사용하는 접근법.

### 👶 어린이를 위한 3줄 비유 설명
1. 커다란 도화지(블랙보드) 하나를 펴놓고 친구들이 모였어요. 무엇을 그릴지는 아무도 몰라요.
2. 한 친구가 '동그라미'를 그리니까, 다른 친구가 뛰어나와서 '귀'를 달고, 또 다른 친구가 '수염'을 그렸어요.
3. 정해진 순서 없이 서로 돌아가면서 그림을 조금씩 덧그렸더니, 어느새 칠판에 아주 멋진 고양이 그림이 완성되는 신기한 방법이랍니다!
