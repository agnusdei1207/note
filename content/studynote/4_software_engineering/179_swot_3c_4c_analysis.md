# 179. SWOT 분석, 3C/4C 분석 연계 요구 도출

> **핵심 인사이트**: 훌륭한 IT 시스템은 단순한 기능의 집합이 아니라 비즈니스 전략의 결과물이어야 한다. 기업의 강점과 약점, 시장의 기회와 위협(SWOT), 그리고 자사·고객·경쟁사(3C)를 분석해야 진정으로 비즈니스 가치를 창출하는 IT 요구사항을 도출할 수 있다.

## Ⅰ. 비즈니스 전략 연계 요구 도출의 개념
단순히 현업 담당자가 "이 기능 만들어주세요"라고 말하는 것을 넘어, 기업의 전략적 목표를 달성하기 위해 IT 시스템이 무엇을 지원해야 하는지 하향식(Top-Down)으로 요구사항을 발굴하는 기법입니다. 이를 위해 대표적인 경영 전략 프레임워크인 SWOT 분석과 3C/4C 분석을 활용합니다.

## Ⅱ. SWOT 분석을 통한 요구사항 도출
기업 내부의 환경(강점, 약점)과 외부 환경(기회, 위협)을 교차 분석하여 IT 시스템이 지원해야 할 전략 과제를 도출합니다.

```text
┌─────────────────┬──────────────────────────────────────────┐
│                 │           내부 환경 (Internal)           │
│                 ├────────────────────┬─────────────────────┤
│                 │ 강점 (Strength)    │ 약점 (Weakness)     │
├────────┬────────┼────────────────────┼─────────────────────┤
│ 외부   │ 기회   │ [ SO 전략 ]        │ [ WO 전략 ]         │
│ 환경   │(Oppor- │ 강점을 살려 기회   │ 약점을 보완하여 기회│
│(Exter- │tunity) │ 선점 (시스템 확대) │ 포착 (IT 자동화)    │
│ nal)   ├────────┼────────────────────┼─────────────────────┤
│        │ 위협   │ [ ST 전략 ]        │ [ WT 전략 ]         │
│        │(Threat)│ 강점을 통해 위협   │ 약점을 극복하고 위협│
│        │        │ 방어 (보안 강화)   │ 회피 (클라우드 전환)│
└────────┴────────┴────────────────────┴─────────────────────┘
```
- **요구사항 연계 예시**: 오프라인 매장이 많다는 강점(S)과 모바일 쇼핑 증가라는 기회(O)를 결합하여, **"O2O(Online to Offline) 기반 픽업 시스템 구축"** 이라는 핵심 IT 요구사항을 도출합니다.

## Ⅲ. 3C / 4C 분석 연계
거시적인 시장 환경을 분석하기 위해 고객, 경쟁사, 자사의 역량을 분석하는 프레임워크입니다.

| 구분 | 주요 분석 내용 | IT 요구사항 도출 예시 |
|:---|:---|:---|
| **Customer (고객)** | 고객의 숨은 니즈(Needs)와 Pain Point 분석 | 맞춤형 상품 추천 AI 알고리즘 도입 요구 |
| **Competitor (경쟁사)** | 경쟁사의 차별화 전략 및 약점 분석 | 경쟁사 앱 대비 결제 단계를 3단계 축소하는 간편 결제 요구 |
| **Company (자사)** | 자사의 핵심 역량과 가용 자원 분석 | 기존 방대한 고객 데이터를 활용한 데이터 웨어하우스 구축 요구 |
| **+ Channel / Circumstance** | 유통 채널 또는 법/제도적 거시 환경 분석 | 개인정보보호법 개정에 따른 암호화 시스템 요건 도출 |

## Ⅳ. 전략적 요구 도출의 효과
경영 전략 프레임워크를 IT 요구공학에 접목하면, 시스템 구축의 **ROI(투자 대비 효과)** 와 **비즈니스 케이스(Business Case)** 가 명확해집니다. 이는 프로젝트 스폰서(경영진)를 설득하고 프로젝트의 방향성을 잃지 않게 하는 강력한 무기가 됩니다.

> 📢 **섹션 요약 비유**: 집을 지을 때 "방 3개짜리 집 지어주세요"라고 무작정 삽을 뜨는 대신, "우리 가족은 손님이 자주 오고(강점/기회), 겨울에 추위를 많이 타니까(약점/위협), 1층에 게스트룸을 크게 빼고 단열에 예산을 집중하자"고 전체적인 설계도를 기획하는 작업입니다.
