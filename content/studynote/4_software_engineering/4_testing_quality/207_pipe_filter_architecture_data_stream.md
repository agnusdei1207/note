# 207. 파이프-필터 아키텍처 (Pipe-Filter) - 데이터 스트림 처리 필터 재사용성 배치 처리 오디오 인코딩 유닉스 쉘 

> **핵심 인사이트**: 비디오 인코딩 공장을 생각해보자. 거대한 비디오 원본이 들어오면, 1번 기계가 화질을 깎고(Filter), 그 결과물이 긴 컨베이어 벨트 파이프(Pipe)를 타고 2번 기계로 가서 자막을 입히고, 다시 파이프를 타고 3번 기계에서 압축해서 완성본이 나온다. "야! 이 기계(필터) 3대는 서로 이름도 성도 몰라! 그냥 자기 왼쪽 파이프에서 물이 들어오면 묵묵히 깎아서 오른쪽 파이프로 뱉어내는 역할만 죽어라 하는 거야! 만약 자막 기계가 고장 나면? 그냥 그 기계만 쏙 빼고 새로운 자막 기계(필터)를 파이프에 꽂아 넣으면 공장은 1초도 안 멈추고 굴러가잖아!!" 데이터를 찰흙처럼 깎아내는 무한 조립식 컨베이어 벨트, 파이프-필터 아키텍처다.

## Ⅰ. 데이터 흐름(Data-Flow)의 제왕: 파이프-필터
- **개념**: 데이터가 시스템의 한쪽 끝(Source)으로 들어와서 다른 쪽 끝(Sink)으로 나갈 때까지, **독립적인 여러 개의 데이터 가공 모듈(필터)들을 데이터 전달 통로(파이프)로 줄줄이 엮어서 순차적으로 처리하는 아키텍처 패턴**입니다.

## Ⅱ. 핵심 구성 요소 2가지 🌟
너무 단순해서 더 강력합니다.

### 1. 필터 (Filter) - "독립적인 찰흙 가공 기계"
- 데이터 스트림(찰흙)을 받아서, 필터링, 압축, 암호화, 정렬 등의 **가공(Processing)을 수행하고 다음 놈에게 뱉어냅니다.**
- **핵심 원칙**: 필터들은 **서로의 존재를 100% 모릅니다(완벽한 독립성, 무상태 Stateless).** 앞 필터가 누구고 뒷 필터가 누군지 관심 없고, 오직 "파이프를 타고 내게 들어온 데이터를 가공해서 내보낸다"는 단 하나의 책임만 가집니다.

### 2. 파이프 (Pipe) - "컨베이어 벨트"
- 앞 필터의 출력(Output)을 뒤 필터의 입력(Input)으로 꽂아주는 **일방향 데이터 통로**입니다. 데이터를 중간에 보관하는 버퍼 역할을 하기도 합니다.

## Ⅲ. 어디에 쓰이는가? (킬러 유스케이스) 🌟 핵심 기출 🌟
- **유닉스/리눅스 쉘 명령어**: `ls -l | grep "txt" | sort` (이 수직선 `|` 기호가 바로 파이프입니다! 앞 명령어의 결과를 뒤 명령어가 받아 깎아내는 완벽한 파이프-필터 아키텍처)
- **컴파일러**: 소스코드 ➜ [어휘 분석기] ➜ [구문 분석기] ➜ [의미 분석기] ➜ 기계어
- **오디오/비디오 스트리밍 처리 (FFmpeg)**: 영상 프레임을 받아서 노이즈 제거, 색상 보정, 인코딩을 순차적으로 수행하는 멀티미디어 공장.

## Ⅳ. 장점과 치명적 단점
- **장점 (레고 조립)**: 새로운 기능이 필요하면 그냥 새 필터를 하나 파이프 중간에 찰칵 끼워 넣으면 끝입니다(극강의 재사용성과 확장성). 또한 각 필터가 별도의 스레드나 서버에서 **병렬 처리(동시 작업)**가 가능해 속도를 미친 듯이 올릴 수 있습니다.
- **치명적 단점 (대화 단절)**: 데이터가 한 방향으로만 무식하게 흘러가기 때문에, 1066번처럼 "야! 이거 에러 났는데 어떡해? 다시 보낼까?" 같은 **복잡한 양방향 대화(상호작용)나 피드백 제어가 끔찍하게 어렵습니다.** 또한 각 필터가 데이터를 자기 입맛에 맞게 포맷 변환(Parsing)하느라 CPU 오버헤드가 발생합니다.

> 📢 **섹션 요약 비유**: **파이프-필터(Pipe-Filter) 아키텍처**는 거대한 **'정수기 필터 시스템'**입니다. 더러운 흙물(원본 데이터)이 파이프를 타고 들어오면, 1번 부직포 필터가 굵은 모래를 거릅니다. 1번 필터는 자기가 정수기 1번인지 2번인지 모릅니다. 그냥 구멍으로 물이 들어오니 걸러서 뒤로 뱉을 뿐입니다. 그 물은 파이프를 타고 2번 활성탄 필터로 가고, 3번 역삼투압 필터를 거쳐 맑은 물(최종 데이터)로 컵에 떨어집니다. 만약 활성탄 필터가 막히면? 시스템 전체를 고칠 필요 없이, 정수기 뚜껑을 열어 2번 필터만 쏙 뽑아내고 새 필터를 '찰칵' 끼워 넣기만 하면 정수기는 다시 1초 만에 완벽하게 돌아가는(플러그 앤 플레이) 데이터 컨베이어 벨트의 교과서입니다.
