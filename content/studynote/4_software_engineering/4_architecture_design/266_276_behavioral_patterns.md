# 266-276. GoF 디자인 패턴: 행위 패턴

> **핵심 인사이트**: 객체는 혼자 존재하지 않는다. 행위 패턴은 객체들이 서로 소통하며 복잡한 알고리즘을 수행하는 '대화의 규칙'을 정의한다. 변화하는 전략을 즉시 교체하고(Strategy), 상태 변화를 자동으로 알리며(Observer), 요청을 객체화하여 기록하는(Command) 과정은 시스템에 지능과 유연성을 부여한다.

---

## Ⅰ. 핵심 행위 패턴 (11종 중 주요 10종)

1. **옵저버 (Observer)**: 상태 변화 시 관찰자들에게 자동으로 알림을 보냄. (구독 모델)
2. **전략 (Strategy)**: 알고리즘을 캡슐화하여 런타임에 동적으로 교체 가능하게 함.
3. **템플릿 메서드 (Template Method)**: 상위 클래스에서 골격을 정의하고, 하위 클래스에서 세부 구현을 수행.
4. **이터레이터 (Iterator)**: 내부 구조를 노출하지 않고 요소들을 순차적으로 접근.
5. **커맨드 (Command)**: 요청을 객체로 캡슐화하여 매개변수화하거나 실행 취소(Undo) 지원.
6. **스테이트 (State)**: 객체의 상태에 따라 행동을 다르게 수행하도록 함.
7. **중재자 (Mediator)**: 객체 간의 복잡한 통신을 중재자 객체 하나로 집중시켜 결합도 감소.
8. **메멘토 (Memento)**: 객체의 상태를 저장해두었다가 나중에 복원. (타임머신)
9. **방문자 (Visitor)**: 객체 구조를 변경하지 않고 새로운 연산을 추가.
10. **책임 연쇄 (Chain of Responsibility)**: 요청을 처리할 수 있는 객체를 만날 때까지 고리를 따라 전달.

---

## Ⅱ. 행위 패턴의 가치
* **유연한 알고리즘**: 코드 수정 없이 동작을 바꿀 수 있음.
* **상호작용 최적화**: 객체 간의 복잡한 호출 관계를 정돈하여 이해하기 쉬운 구조로 만듦.

---

## Ⅲ. 개념 맵 및 요약

```ascii
[행위 패턴의 성격별 분류]

  "알림과 관찰" ──> [ Observer ]
  "알고리즘 교체" ──> [ Strategy / Template Method ]
  "요청과 전달"   ──> [ Command / Chain of Responsibility ]
  "상태와 기록"   ──> [ State / Memento ]
```

📢 **섹션 요약 비유**: **옵저버**는 유튜브 채널의 '구독과 알림 설정'입니다. **전략**은 게임 캐릭터가 무기를 칼에서 총으로 실시간으로 바꾸는 것이고, **템플릿 메서드**는 표준 요리 레시피에서 핵심 재료만 바꾸어 나만의 요리를 만드는 것입니다. **커맨드**는 리모컨 버튼 하나에 복잡한 명령(전원 켜기 + 채널 이동)을 입력해두는 것이며, **메멘토**는 게임 중간에 '저장(Save)' 포인트로 되돌아가는 것과 같습니다.
