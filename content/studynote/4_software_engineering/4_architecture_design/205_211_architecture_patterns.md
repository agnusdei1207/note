# 205-211. 주요 아키텍처 패턴 (Layered, MVC, MVVM)

> **핵심 인사이트**: 복잡한 시스템을 한 덩어리로 만들면 고칠 때마다 전체가 흔들린다. 계층형 아키텍처는 역할을 수평으로 나누고, MVC/MVVM은 사용자 화면과 비즈니스 로직을 수직으로 분리한다. 이 '격리'와 '분리'의 기술이 현대 소프트웨어의 유지보수성을 지탱하는 근간이다.

---

## Ⅰ. 기반 아키텍처 스타일
1. **계층형 아키텍처 (Layered Architecture)**: 시스템을 하위 계층이 상위 계층에 서비스를 제공하는 수평적인 층으로 나눕니다. (예: Presentation - Business - Data)
2. **클라이언트-서버 (Client-Server)**: 서비스를 요청하는 자와 제공하는 자를 명확히 분리합니다.
3. **파이프-필터 (Pipe-Filter)**: 데이터 스트림을 일련의 필터를 거쳐 순차적으로 처리합니다. (유닉스 쉘 커맨드 등)

---

## Ⅱ. UI 아키텍처 패턴 (MVC 계열)

### 1. MVC (Model-View-Controller)
* **Model**: 데이터와 비즈니스 로직 처리.
* **View**: 사용자에게 보여지는 화면.
* **Controller**: 사용자의 입력을 받아 모델을 업데이트하고 뷰를 제어.
* **특징**: 가장 고전적인 패턴이나, 뷰와 모델 사이의 의존성이 완전히 제거되지는 않습니다.

### 2. MVP (Model-View-Presenter)
* **Presenter**: 뷰와 모델 사이에서 모든 중계를 담당. 뷰는 직접 모델을 보지 못하게 차단.

### 3. MVVM (Model-View-ViewModel)
* **ViewModel**: 뷰의 상태를 데이터 형태로 가지고 있음.
* **Data Binding**: 뷰와 뷰모델 사이의 상태를 자동으로 동기화. 
* **장점**: 현대 웹 프레임워크(React, Vue)나 모바일(Android) 개발의 대세 패턴입니다.

---

## Ⅲ. 특수 패턴
* **브로커 패턴**: 분산 시스템에서 컴포넌트 간 통신을 중계.
* **블랙보드 패턴**: 음성 인식처럼 여러 지식 모듈이 공용 데이터베이스(Blackboard)를 보고 답을 찾아가는 방식.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[UI 패턴의 진화 흐름]

  (전통) [ MVC ] : Controller가 중간에서 조율 (느슨한 연결)
           │
  (차단) [ MVP ] : Presenter가 뷰와 모델을 완벽히 분리
           │
  (현대) [ MVVM ] : 데이터 바인딩으로 뷰와 모델을 자동 연결
```

📢 **섹션 요약 비유**: **계층형 아키텍처**는 고층 빌딩의 층별 구성(식당가, 사무실, 주차장)과 같습니다. **MVC**는 식당에서 손님(View), 웨이트리스(Controller), 주방(Model)이 분업하는 체계입니다. **MVVM**은 손님이 메뉴판(ViewModel)에 체크만 하면 주방 음식이 자동으로 나오고 상이 차려지는 '최첨단 스마트 테이블' 시스템과 같습니다. **파이프-필터**는 정수기 필터를 통과하며 물이 점점 깨끗해지는 과정과 같습니다.
