# 251-257. GoF 디자인 패턴: 생성 패턴

> **핵심 인사이트**: 객체를 만드는 과정도 '설계'가 필요하다. 무분별한 `new` 연산자 사용은 결합도를 높인다. 생성 패턴은 객체가 생성되는 방식과 시점을 캡슐화하여, 시스템이 어떤 객체를 어떻게 만드는지 몰라도 그 객체를 사용할 수 있게 하는 '유연한 공장' 역할을 한다.

---

## Ⅰ. GoF (Gang of Four) 디자인 패턴 개요
에릭 감마 등 4명의 전문가가 정리한 23가지 소프트웨어 설계 패턴입니다.
* **분류**: 생성(Creational), 구조(Structural), 행위(Behavioral).

---

## Ⅱ. 핵심 생성 패턴 (5종)

1. **싱글톤 (Singleton)**: 클래스의 인스턴스를 단 하나만 생성하고 어디서든 접근 가능하게 함. (설정 정보, 로그 기록 등)
2. **팩토리 메서드 (Factory Method)**: 객체 생성을 직접 하지 않고, 서브클래스가 어떤 클래스의 인스턴스를 만들지 결정하게 함. (상속 이용)
3. **추상 팩토리 (Abstract Factory)**: 구체적인 클래스를 지정하지 않고, 연관된 객체들의 **제품군**을 생성하기 위한 인터페이스 제공. (테마별 UI 부품 생성 등)
4. **빌더 (Builder)**: 복잡한 객체의 **생성 과정과 표현 방법을 분리**하여, 동일한 생성 절차에서 서로 다른 표현 결과물을 만듦.
5. **프로토타입 (Prototype)**: 원본 객체를 복사(Clone)하여 새로운 객체를 생성. 생성 비용이 큰 경우 유리.

---

## Ⅲ. 생성 패턴의 장점
* **결합도 감소**: 사용하는 쪽에서 구체적인 클래스 명을 알 필요가 없음.
* **일관성 유지**: 객체 생성 로직이 한곳에 집중되어 관리가 쉬움.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[생성 패턴의 목적별 분류]

  "단 하나만 존재해야 해" ──> [ Singleton ]
  "객체군을 세트로 만들어" ──> [ Abstract Factory ]
  "복잡한 조립이 필요해"   ──> [ Builder ]
  "복사해서 새로 만들자"   ──> [ Prototype ]
```

📢 **섹션 요약 비유**: **싱글톤**은 마을에 딱 하나뿐인 '이장님'과 같습니다. **팩토리 메서드**는 주문이 들어오면 주방장이 아닌 '피자 전문 셰프'가 피자를 굽는 분업이고, **추상 팩토리**는 '삼성 세트(폰, 워치, 버즈)'나 '애플 세트'처럼 브랜드별 패키지를 생산하는 시스템입니다. **빌더**는 서브웨이 샌드위치처럼 빵, 소스, 토핑을 순서대로 골라 나만의 메뉴를 완성하는 과정이며, **프로토타입**은 고품질의 도자기 원형을 보고 '틀'을 떠서 대량 복제하는 것과 같습니다.
