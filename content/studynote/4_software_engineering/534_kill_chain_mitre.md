# 534. 사이버 킬 체인 및 MITRE ATT&CK 프레임워크

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사이버 킬 체인 (Cyber Kill Chain)은 해커의 공격 과정을 7단계의 선형적 흐름으로 정립한 모델이며, MITRE ATT&CK은 실제 공격 전술(Tactics)과 기술(Techniques)을 매트릭스 형태로 체계화한 지식 베이스다.
> 2. **가치**: 파편화된 보안 이벤트를 공격자의 거시적 의도(사이버 킬 체인)와 미시적 기술(ATT&CK) 관점에서 해석하게 함으로써, 탐지 지연 시간(Dwell Time)을 단축하고 지능형 지속 위협(APT)에 대한 입체적 방어 체계를 구축한다.
> 3. **융합**: 침해 사고 대응(IR), 위협 헌팅(Threat Hunting), 보안 관제 자동화와 결합되며, 현대 아키텍처에서는 공격자의 행동 패턴을 실시간 매핑하여 '능동적이고 예측 가능한 보안 운영'을 실현한다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 
  - **사이버 킬 체인 (Cyber Kill Chain)**: 록히드 마틴이 제안한 모델로, 정찰부터 목적 달성까지의 단계를 정의한다. 이 중 한 단계만 끊어도 전체 공격을 무력화할 수 있다는 원리다.
  - **MITRE ATT&CK**: 실제 발생한 수만 건의 해킹 사례를 분석하여 공격자의 행동 양식을 '전술-기술-절차(TTP)'로 DB화한 것으로, 킬 체인보다 훨씬 구체적이고 실전적이다.

- **💡 비유**: 
  - **사이버 킬 체인**: 도둑이 집을 털기 위해 세운 **"큰 그림의 작전 계획"** (사전 조사 ──▶ 담 넘기 ──▶ 금고 찾기 ──▶ 도망)과 같습니다.
  - **MITRE ATT&CK**: 도둑이 담을 넘을 때 사용한 **"구체적인 연장과 기술 목록"** (사다리 타기, 유리 절단기, 도어락 해킹 등)을 일목요연하게 정리한 백과사전과 같습니다.

- **등장 배경**:
  1. **APT 공격의 정교화**: 단순 바이러스 차단 방식으로는 수개월간 치밀하게 진행되는 APT 공격을 막을 수 없게 되어, 공격의 '흐름'을 이해해야 했다.
  2. **표준 언어의 부재**: "공격자가 권한을 뺏으려 한다"는 상황을 보안 전문가마다 다르게 설명하던 문제를 해결하기 위해 전 세계 공용의 '공격 기술 식별자'가 필요해졌다.
  3. **데이터 기반 보안**: 축적된 공격 데이터를 분석하여 미래의 공격을 예측하고 방어 전략을 고도화하려는 요구가 커졌다.

- **📢 섹션 요약 비유**: 적군이 우리 성을 공격해올 때, 그들이 거쳐야 할 경로(킬 체인)를 파악하고 그들이 손에 든 무기(ATT&CK)가 무엇인지 미리 안다면 전쟁의 승패는 이미 결정된 것과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 사이버 킬 체인 7단계 (Lockheed Martin)

공격의 연쇄 고리를 끊는 지점을 식별하는 모델이다.

| 단계 | 명칭 | 핵심 활동 | 비유 |
|:---|:---|:---|:---|
| **1단계** | **정찰 (Reconnaissance)** | 타겟 조사, 이메일 수집, 포트 스캔 | 집 주변 배회 및 사전 조사 |
| **2단계** | **무기화 (Weaponization)** | 악성코드 제작 및 문서 결합 | 총과 총알 준비 |
| **3단계** | **전달 (Delivery)** | 이메일 첨부, USB, 피싱 사이트 유도 | 택배 상자로 위장 전달 |
| **4단계** | **취약점 공격 (Exploitation)** | 소프트웨어 버그를 이용한 코드 실행 | 열려 있는 창문으로 침입 |
| **5단계** | **설치 (Installation)** | 백도어 설치, 지속성 유지 장치 마련 | 집 안에 몰래 숨을 곳 마련 |
| **6단계** | **명령 및 제어 (C2)** | 해커 서버와 연결하여 원격 조종 수행 | 무전기로 외부 지령 받기 |
| **7단계** | **행동 (Actions on Obj.)** | 데이터 유출, 시스템 파괴, 랜섬웨어 실행 | 보물 훔치기 및 방화 |

---

### 2. MITRE ATT&CK 매트릭스 아키텍처

공격자의 전술(Tactics)과 기술(Techniques)을 입체적으로 매핑한 구조다.

```text
 ┌─────────────────────────────────────────────────────────────────────────────┐
 │                      MITRE ATT&CK 엔터프라이즈 매트릭스 (요약)                   │
 ├─────────────────────────────────────────────────────────────────────────────┤
 │                                                                             │
 │ [Tactics (전술)] : 공격자의 목적 (Why?)                                        │
 │  ┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐        │
 │  │ 초기 침투 │  실행    │ 지속성 유지│ 권한 상승 │ 방어 회피 │ 데이터 유출│ ...    │
 │  └────┬─────┴────┬─────┴────┬─────┴────┬─────┴────┬─────┴────┬─────┘        │
 │       │          │          │          │          │          │              │
 │ [Techniques (기술)] : 구체적 방법 (How?)                                       │
 │  ┌────┴────┐ ┌───┴────┐ ┌────┴────┐ ┌────┴────┐ ┌────┴────┐ ┌────┴────┐       │
 │  │ 피싱 (T1566) │  Power- │ 레지스트리  │ Sudo     │ 난독화    │ C2 통신   │       │
 │  │          │  shell   │ 런키 등록   │ (T1548)  │ (T1027)  │ (T1071)  │       │
 │  └──────────┘ └────────┘ └──────────┘ └──────────┘ └────────┘ └─────────┘       │
 │                                                                             │
 │  [Procedures (절차)] : 실제 사용된 공격 도구 및 사례 (예: APT28, Lazarus)         │
 │                                                                             │
 │  ※ 가치: 우리 방화벽이 T1566(피싱)은 막지만 T1071(C2)은 못 막음을 수치화 함         │
 └─────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** MITRE ATT&CK은 단순한 리스트가 아니라 **매트릭스**다. 가로축인 **전술(Tactics)**은 공격자가 달성하려는 14가지 목표(초기 침투, 권한 상승 등)를 나타내며, 세로축인 **기술(Techniques)**은 그 목표를 이루기 위해 사용하는 수백 가지의 구체적인 방법들이다. 각 기술에는 `T1566`과 같은 고유 ID가 부여되어 전 세계 보안 장비와 위협 인텔리전스에서 공통 식별자로 쓰인다. 이를 통해 기업은 자사의 보안 장비가 어떤 기술들을 방어할 수 있는지 '히트맵(Heatmap)'으로 그려 보안의 사각지대를 직관적으로 파악할 수 있다.

---

### 심층 동작 원리: TTP (Tactics, Techniques, Procedures) 분석

공격자의 유전 정보를 분석하는 현대 보안 분석의 핵심이다.
1. **Tactics (전술)**: "왜 하는가?" (예: 정보 유출)
2. **Techniques (기술)**: "어떻게 하는가?" (예: SQL 인젝션)
3. **Procedures (절차)**: "실제로 무엇을 썼는가?" (예: sqlmap 도구를 이용한 특정 시퀀스)
4. **가치**: IP나 파일 해시(IOC)는 금방 바꿀 수 있지만, 공격자의 습관인 TTP는 바꾸기 매우 어렵기 때문에 TTP 기반 분석이 탐지에 더 효과적이다.

- **📢 섹션 요약 비유**: 범인의 이름이나 옷차림(IOC)은 바뀔 수 있지만, 항상 밤 12시에 뒷문을 통해 들어오는 범행 수법(TTP)은 변하지 않는다는 점을 노린 수사 기법입니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 사이버 킬 체인 vs MITRE ATT&CK 비교

| 비교 항목 | 사이버 킬 체인 (Cyber Kill Chain) | MITRE ATT&CK |
|:---|:---|:---|
| **모델 구조** | 선형적 흐름 (Linear) | **매트릭스/네트워크 구조** |
| **상세도** | 거시적 (7단계) | **미시적 (수백 개의 기술)** |
| **공격자 관점** | 외부로부터의 침투 과정 강조 | **내부 침투 이후의 모든 행위 강조** |
| **주요 용도** | 전략적 방어 지점 설정 | 실전 탐지 룰(Rule) 생성, 레드팀 훈련 |
| **비유** | 전쟁의 주요 승부처 | 적군의 무기 도감 및 전술 교본 |

---

### 기술 융합: 위협 헌팅 (Threat Hunting)과 ATT&CK

공격이 발생하기 전 미리 숨은 해커를 찾아내는 활동이다.
- **가설 수립**: "공격자가 우리 서버의 권한 상승을 위해 T1548(Sudo) 기술을 쓸 것이다"라는 가설을 세운다.
- **데이터 분석**: EDR/SIEM 로그에서 해당 기술의 징후를 검색한다.
- **가치**: ATT&CK 프레임워크를 나침반 삼아 망망대해 같은 로그 데이터 속에서 해커의 흔적을 효율적으로 찾을 수 있다.

- **📢 섹션 요약 비유**: 도둑이 들었을 때 경보가 울리길 기다리는 것이 아니라, 도둑이 사다리(T1543)를 쓸 것 같은 곳에 미리 매복해 있는 선제적 방범 활동입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: APT 공격 그룹 '라자루스' 대응 체계 구축

- **문제 상황**: 최근 특정 국가의 공격 그룹이 우리 산업군을 타겟으로 공격을 개시했다는 정보 입수.
- **의사결정 (프레임워크 기반 대응)**:
  1. **Profile Mapping**: MITRE ATT&CK 사이트에서 'Lazarus Group'이 주로 사용하는 기술(T1566 피싱, T1059 명령 실행 등)을 추출한다.
  2. **Gap Analysis**: 추출된 기술 목록을 우리 회사의 보안 솔루션 탐지 범위와 대조하여, 못 막는 기술(Gap)이 무엇인지 파악한다.
  3. **탐지 룰 생성**: 킬 체인의 '명령 및 제어(C2)' 단계를 끊기 위해, 해당 그룹이 쓰는 통신 패턴(T1071)을 기반으로 SIEM에 실시간 탐지 룰을 적용한다.
  4. **시뮬레이션**: 레드팀(공격팀)이 ATT&CK에 정의된 해당 그룹의 기술을 재현하여 우리 방어 체계가 실제 작동하는지 검증한다.

### 도입 체크리스트
- **Completeness**: 내부망 침투 이후의 '횡적 이동(Lateral Movement)' 단계에 대한 탐지 시나리오가 마련되어 있는가?
- **Mapping**: 우리 회사 보안 도구(EDR, WAF 등)의 결과 리포트가 ATT&CK ID와 연결되어 표시되는가?
- **Freshness**: 최신 공격 기술이 반영된 ATT&CK 데이터베이스(현재 v14 이상)를 참조하고 있는가?

### 안티패턴
- **앞 단계만 차단**: 킬 체인의 1~3단계(정찰, 전달) 방어에만 예산을 쏟고, 일단 뚫린 후(4~7단계)의 감시를 소홀히 하는 경우.
- **번호 지상주의**: ATT&CK 기술 번호를 다 안다고 해서 보안이 된 것으로 착각하는 것. 기술의 원리를 이해하고 실제 로그와 매핑하는 노력이 필수다.

- **📢 섹션 요약 비유**: 대문에 자물쇠만 수십 개 달고(앞 단계 방어), 정작 창문으로 들어온 도둑이 거실에서 파티를 벌이는데도(뒷 단계 감시 부재) 모르는 어리석음을 범해서는 안 됩니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 기대효과 | 세부 내용 |
|:---|:---|:---|
| **정량적 (Detection)** | 위협 탐지 범위 2배 확장 | 알려진 모든 공격 기법에 대한 체크리스트 확보로 보안 사각지대 제거 |
| **정량적 (Time)** | 평균 분석 시간(MTTA) 단축 | 표준화된 공격 기술 코드를 통해 사고의 성격을 즉시 파악 |
| **정성적 (Interoperability)** | 글로벌 정보 공유 강화 | 해외 보안 기관과의 공통 언어(ATT&CK ID)를 통한 신속한 정보 교환 |
| **정성적 (Strategy)** | 체계적 보안 투자 근거 마련 | 히트맵 분석을 통해 부족한 보안 영역에 대한 예산 투입의 명분 확보 |

### 미래 전망
- **AI-Generated TTPs**: 공격자가 AI를 이용해 매번 새로운 변종 기술을 만들어낼 것에 대비하여, AI가 실시간으로 ATT&CK 기술을 확장하고 탐지 코드를 생성하는 체계가 도입될 것이다.
- **Unified Attack Knowledge**: IT를 넘어 OT(산업제어), Cloud, Mobile, Space(우주) 인프라까지 모든 영역의 공격 기법이 하나의 거대한 지식 체계로 통합될 것이다.

### 참고 표준
- **MITRE ATT&CK Matrix**: 전 세계 사실상의 표준
- **Lockheed Martin Cyber Kill Chain**: 전략적 공격 흐름 모델
- **Diamond Model of Intrusion Analysis**: 공격자, 인프라, 능력, 희생자 관계 분석 모델

- **📢 섹션 요약 비유**: 이제 보안은 운에 맡기는 도박이 아니라, 적의 일거수일투족을 데이터로 읽어내고 대응하는 '데이터 사이언스 기반의 지능형 전쟁'으로 진화하였습니다.

## 👶 어린이를 위한 3줄 비유 설명
1. **사이버 킬 체인**은 나쁜 해커가 우리 컴퓨터에 쳐들어올 때 거쳐야 하는 **"7개의 징검다리"**와 같아요. 이 중 하나만 슥 치워버리면 해커는 물에 빠져서 공격에 실패한답니다.
2. **MITRE ATT&CK**은 해커들이 쓰는 **"비밀 무기 가방"**을 몰래 들여다보고 정리해놓은 수첩이에요.
3. 이 수첩과 징검다리를 잘 공부하면, 해커가 어떤 나쁜 짓을 하려는지 미리 알고 우리 보물을 아주 안전하게 지킬 수 있어요!
