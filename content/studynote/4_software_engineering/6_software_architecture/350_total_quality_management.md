# 350. 전사적 품질 관리 (TQM, Total Quality Management)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전사적 품질 관리 (TQM, Total Quality Management)는 조직 전체가 참여하여 품질을 지속적으로 개선하고, 고객 만족을 달성하는 품질 경영 철학 및 시스템이다.
> 2. **가치**: TQM을 적용하면 품질 관련 비용이 절감되고, 고객 만족도가 향상되며, 조직의 지속적인 개선 문화가 형성되어 장기적인 경쟁력 확보에 기여한다.
> 3. **융합**: TQM의 원칙은 소프트웨어 개발 조직에서도 애자일 방법론, Six Sigma, Lean 등 다양한 품질 관리 기법과 결합되어 활용되고 있다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: 전사적 품질 관리 (TQM, Total Quality Management)는 조직의 모든 구성원이 품질 개선에 참여하여, 고객 만족을 지속적으로 달성하는 것을 목표로 하는 품질 경영 철학이다. TQM은 1950년대 미국에서 시작되어, 1980년대에 일본의 제조업 경쟁력의 배경으로 주목받으며世界的に普及した. TQM의 핵심 원칙으로는 고객 중심, 전사적 참여, 지속적인 개선, 의사결정의 사실 기반, 프로세스 중심, 품질 우선이 있다.

- **필요성**: 소프트웨어 개발에서 품질 관리는 개발팀만의 문제가 아니라 조직 전체의 문제이다. 품질 목표를 달성하려면 경영진의 리더십, 개발팀의 실천, 영업/고객지원 등의 협력이 필요하다. TQM은 이러한 조직 전체의 노력을 체계적으로統合하여品質改善を実現する方法론을 제공한다.

- **💡 비유**: TQM은 "건강을 위한 생활 습관"에 비유할 수 있다. 건강유지를 위해 운동(개발), 식단(요구사항), 수면(휴식) 등을 모두 신경 써야 하듯이, 조직에서도 모두가 품질 개선에 참여해야 한다.

- **등장 배경**: TQM은 1951년 미국 품질관리협회(AQCA)가 발전시킨 개념으로, 1980년대 일본 제조업의 품질 경쟁력을 분석하면서世界的に注目された。主要企業であるToyota, Motorola 등이 TQM 기반의品質経営를 실천하여成功を収めた。

- **📢 섹션 요약 비유**: TQM은 "가족 전체의健康管理"와 같다. 한 사람만 건강한 생활습관을 유지해도 전체 건강이 좋아지듯이, 조직에서도全体가 참여하여 품질 개선에 힘써야 한다.

**[다이어그램 해설]** TQM의 8대 원칙은 품질 경영의 핵심 가치을 제공한다. 고객 중심은 모든 활동의 최종 목표가 고객 만족임을 강조한다. 전사적 참여는 품질 개선이 특정 부서의 문제가 아니라 조직 전체의 책임임을 강조한다. 프로세스 중심은 결과를 만들기 위한活動의連鎖としてプロセスを管理することを意味する。 지속적인 개선은 일회성이 아닌 끊임없는 노력을 강조한다. 의사결정의 사실 기반은 감이 아닌 데이터에 기반한 의사결정의 중요성을 강조한다. 품질 우선은 품질을 비용, 일정보다 우선시하는 문화적 전환을 의미한다. 상호 호혜적 공급업체 관계는 품질이 공급망 전체에 걸쳐 있어야 함을 강조한다. 사회적 책임은 조직의 존재理由が社会貢献にあることを認識する。

### PDCA 사이클

```text
┌─────────────────────────────────────────────────────────────────┐
│              PDCA (Plan-Do-Check-Act) 사이클                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                        ┌─────────┐                              │
│                        │   Act   │                              │
│                        │ (개선)  │                              │
│                        └────┬────┘                              │
│                             │                                    │
│              ┌─────────────┼─────────────┐                      │
│              │             │             │                      │
│        ┌────┴────┐  ┌────┴────┐  ┌────┴────┐                │
│        │  Plan   │  │   Do    │  │  Check   │                │
│        │ (계획)  │─▶│ (실행)  │─▶│ (점검)  │                │
│        └─────────┘  └─────────┘  └────┬────┘                │
│                                        │                        │
│                                        └──────────────────────▶  │
│                                                                 │
│  [Plan (계획)]                                                 │
│  • 문제점 분석 및 원인 파악                                       │
│  • 개선 목표 설정                                               │
│  • 개선 방안 수립                                               │
│                                                                 │
│  [Do (실행)]                                                  │
│  • 계획된 개선 활동 실행                                         │
│  • 교육 및 훈련 실시                                            │
│  • 프로토타입/파일럿 실행                                       │
│                                                                 │
│  [Check (점검)]                                               │
│  • 실행 결과 측정 및 분석                                        │
│  • 계획 대비 성과 비교                                           │
│  •偏差分析 및 원인 파악                                         │
│                                                                 │
│  [Act (개선)]                                                  │
│  • 개선 조치 결정 및 표준화                                       │
│  • 성공 사례 확산                                               │
│  • 다음 PDCA를 위한 계획 수립                                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** PDCA 사이클은 TQM의 핵심 개선 방법론이다. Plan 단계에서는 문제점을 분석하고 원인를 파악한 후 개선 목표를 설정하고 개선 방안을 수립한다. Do 단계에서는 계획된 개선 활동을 실행하고, 필요한 교육을 실시하며, 프로토타입이나 파일럿을 통해试点한다. Check 단계에서는 실행 결과를 측정하고 계획 대비 성과를 비교하며,偏差를 분석한다. Act 단계에서는 개선 조치를 결정하고 성공 사레를 표준화하여 확산하며, 다음 PDCA를 위한 계획을 수립한다. PDCA 사이클은 지속적으로回転하여組織의品質改善を驅動する。

### TQM 도구 (QC 7가지 도구)

| 도구 | 용도 | 활용 예시 |
|:---|:---|:---|
| **흐름도 (Flowchart)** | 프로세스 시각화 | 개발 프로세스 단계별 분석 |
| **체크시트 (Check Sheet)** | 데이터 수집 | 결함 유형별 빈도 수집 |
| **히스토그램** | 데이터 분포 파악 | 응답 시간 분포 분석 |

| **Cause-and-Effect 图** | 원인 분석 | 魚骨图로 결함 원인 분석 |
| **산점도 (Scatter Diagram)** | 상관관계 분석 | 코드 복잡도와 결함 수 관계 |
| **관리도 (Control Chart)** | 프로세스 안정성监控 | 빌드 성공률 추적 |

## Ⅳ. 품질 관리 및 테스트 (Quality & Testing)

### TQM 적용의 효과

| 구분 | TQM 적용 전 | TQM 적용 후 |
|:---|:---|:---|
| **품질 비용** | 높은 실패 비용 | 예방 비용 비율 증가, 실패 비용 감소 |
| **고객 만족** | 보통 | 향상 |
| **직원 참여** | 부분적 | 전사적 참여 |
| **개선 활동** | 일회성 | 지속적인 PDCA |
| **품질 문화** | 규정 준수 중심 | 자율적 품질 관리 |

- **📢 섹션 요약 비유**: TQM은 "건강管理 프로그램"과 같다. 운동(품질 개선), 식단(요구사항 관리), 수면(휴식/회고) 등을 모두 체계적으로管理하여全身の健康을維持する。

## Ⅴ. 최신 트렌드 및 결론 (Trends & Conclusion)

### 소프트웨어 개발에서의 TQM 적용

1. **애자일과의 융합**: 애자일의 retrospective, continuous improvement가 TQM의 PDCA와 일치
2. **DevOps와의 결합**: CAMS (Culture, Automation, Measurement, Sharing)가 TQM 원칙을 실현
3. **Six Sigma**: 통계적 품질 관리 기법과 TQM의 결합

### TQM 성공 요소

1. **경영진의 진정한 리더십**:言葉だけでなく 실천으로 보여주는 리더십
2. **전사적 참여 문화**:品質改善が全員参加的活动である文化
3. **지속적인 교육**:品質 管理スキル向上를 위한 투자
4. **데이터 기반 의사결정**:感情ではなくデータ에 기반한 판단

- **📢 섹션 요약 비유**: TQM은 "가족 全員参加의 집안 관리"와 같다. 집안 전체가 정리정돈(품질 개선)에 참여하고, 모두가 편안하게 살 수 있는環境을維持한다.
