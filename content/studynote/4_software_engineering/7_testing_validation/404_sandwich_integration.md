# 404. 샌드위치 통합 (Sandwich / Hybrid Integration)

## 핵심 인사이트 (3줄 요약)
> 1. 대규모 시스템에서 메인 UI(상향)와 DB 코어(하향)가 각기 팽개쳐질 때 발생하는 일정 지연을 막기 위해 **동시다발적으로 조립 파이프라인을 두 갈래로 돌리는 하이브리드** 방식이다.
> 2. 스텁(Stub)과 드라이버(Driver)가 동시에 많이 필요하며, 정중앙의 핵심 비즈니스 로직 층(Logic Layer) 파트에서 상/하 결합이 동시에 미팅 포인트를 이룬다.
> 3. 개발 인력이 풍부한 대형 엔터프라이즈 프로젝트 병렬 추진 시 공기(Time-to-market) 단축에 비약적인 이점을 얻는다.

---
## Ⅰ. 아키텍처 및 원리
```text
  [ 샌드위치 병합 미팅망 ]
   (Frontend 팀) : UI -> 하향식 전개 (Stub 필요)
                     | (융합 계층: 샌드위치 결합부)
   (Backend 팀)  : DB -> 상향식 전개 (Driver 필요)
```
---
## Ⅱ. 실무 적용 및 결론
오늘날 React 프론트 팀과 Spring Boot 백엔드 팀이 각자 API 명세서(Swagger) 하나만 들고 양쪽 끝에서 개발 조립을 시작하여, 마지막 릴리스 직전 API 게이트웨이 중간 층에서 만나 샌드위치로 도킹하는 것이 바로 이 결합 방식의 현대적 투영 체계다.
