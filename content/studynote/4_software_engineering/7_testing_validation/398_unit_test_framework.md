# 398. 단위 테스트 프레임워크 (xUnit)

## 핵심 인사이트 (3줄 요약)
> 1. Java의 JUnit, Python의 pytest 등은 **Arrange-Act-Assert(AAA)** 패턴 규격을 양산하여 테스터의 생산성을 극대화한다.
> 2. 테스트 스위트(Suite) 단위의 일괄 자동 실행과 `setUp()`, `tearDown()` 라이프사이클을 통해 메모리 오염 없는 멱등성 보장 검증을 지원한다.
> 3. 소나큐브(SonarQube) 등의 커버리지 측정 도구와 맞물려 빌드 통과의 품질 메트릭(게이트키퍼) 기준선으로 작용한다.

---
## Ⅰ. 아키텍처 및 원리
```text
  [ xUnit 라이프사이클 런타임 콜백 ]
  BeforeAll -> [ BeforeEach -> TEST -> AfterEach ] (반복) -> AfterAll
```
---
## Ⅱ. 실무 적용 및 결론
테스트는 사람이 수동으로 하는 것이 아니라 "프레임워크가 코드로 짜인 스크립트를 트리거"하는 구동계다. 이것이 없으면 TDD나 애자일의 근간인 '회귀 테스트의 고속 검증'이 불가능하다.
