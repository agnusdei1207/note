# 420-429. 화이트박스 테스트와 커버리지 (MC/DC)

> **핵심 인사이트**: 화이트박스 테스트는 소스코드라는 '지도'를 샅낱이 훑는 작업이다. 내가 짠 코드의 모든 줄(Statement)과 모든 갈림길(Decision)을 한 번씩은 밟아보았는지 확인해야 한다. 특히 항공기나 자율주행차처럼 안전이 직결된 분야에서는 각 조건이 결과에 미치는 영향까지 입증하는 MC/DC 수준의 정교한 검증이 필수다.

---

## Ⅰ. 화이트박스 테스트 (White-box Test) 개요
프로그램의 내부 구조와 논리 흐름을 직접 보면서 수행하는 테스트입니다. (구조 기반 테스트)

---

## Ⅱ. 코드 커버리지 (Code Coverage) 단계
테스트가 코드의 어느 정도를 수행했는지 나타내는 지표입니다.

1. **구문 커버리지 (Statement)**: 코드의 모든 문장을 최소 한 번 실행. (가장 낮은 수준)
2. **결정 커버리지 (Decision / Branch)**: 모든 분기문의 참(True)과 거짓(False)을 최소 한 번씩 실행.
3. **조건 커버리지 (Condition)**: 분기문 내의 각 개별 조건식(예: `A > 0`)이 참/거짓을 한 번씩 가짐.
4. **조건/결정 커버리지**: 개별 조건과 전체 결정이 모두 참/거짓을 만족하게 함.
5. **MC/DC (Modified Condition/Decision Coverage)**:
   - 각 개별 조건이 **다른 조건에 상관없이** 전체 결정 결과에 독립적으로 영향을 미침을 증명.
   - 예: `(A or B)`에서 A가 바뀔 때 결과가 바뀌고, B가 바뀔 때 결과가 바뀌는 케이스 확보.
6. **경로 커버리지 (Path)**: 가능한 모든 실행 경로를 테스트. (가장 강력하지만 조합이 많아 어려움)

---

## Ⅲ. 데이터 흐름 테스팅 (Data Flow Testing)
* 변수의 **정의(Define)**와 **사용(Use)** 경로를 추적하여 검증하는 기법입니다. (DU 경로 분석)

---

## Ⅳ. 개념 맵 및 요약

```ascii
[커버리지 수준의 계층]

  ( 최상위 )  [ Path Coverage ] (모든 길)
      │
  ( 고안전 )  [ MC/DC ] (개별 조건의 독립적 영향)
      │
  ( 일  반 )  [ Decision / Condition Coverage ] (분기/조건)
      │
  ( 최하위 )  [ Statement Coverage ] (모든 줄 실행)
```

📢 **섹션 요약 비유**: **화이트박스 테스트**는 투명한 상자 속의 톱니바퀴가 어떻게 맞물려 도는지 보는 것입니다. **구문 커버리지**는 지도의 모든 도로를 한 번씩 지나가는 것이고, **결정 커버리지**는 모든 교차로에서 좌회전과 우회전을 다 해보는 것입니다. **MC/DC**는 "신호등이 빨간불로 바뀌었을 때(조건), 다른 차들과 상관없이 내 차가 멈추는지(결과)"를 확인하여 신호등의 독립적인 효력을 입증하는 아주 깐깐한 안전 검사입니다.
