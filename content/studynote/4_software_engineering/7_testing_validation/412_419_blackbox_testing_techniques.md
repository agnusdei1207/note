# 412-419. 블랙박스 테스트 기법 (Boundary, Pairwise)

> **핵심 인사이트**: 블랙박스 테스트는 '입력'과 '출력'의 마법이다. 모든 데이터를 넣을 수 없다면, 결함이 숨어있을 만한 대푯값(Equivalence)과 아슬아슬한 경계선(Boundary)을 공략해야 한다. 또한 수억 개의 조합 중 핵심 쌍(Pair)만 골라내는 페어와이즈 기법은 테스트의 가성비를 극대화하는 영리한 전략이다.

---

## Ⅰ. 블랙박스 테스트 (Black-box Test) 개요
프로그램의 내부 구조를 보지 않고, **명세(Specification)**를 기반으로 입력값에 대한 올바른 출력이 나오는지 검증하는 기법입니다.

---

## Ⅱ. 주요 테스트 데이터 선정 기법

1. **동등 분할 (Equivalence Partitioning)**: 입력 영역을 유효/무효 클래스로 나누고, 각 클래스에서 하나의 대푯값을 뽑아 테스트.
2. **경계값 분석 (Boundary Value Analysis)**: 결함이 가장 많이 발생하는 **경계 지점(최솟값, 최댓값 주변)**을 집중 테스트. (예: 1~100 입력 시 0, 1, 100, 101 테스트)
3. **의사 결정 테이블 (Decision Table)**: 복잡한 조건들의 조합(And, Or 등)을 표로 만들어 논리적 누락 없이 테스트.
4. **상태 전이 테스트 (State Transition)**: 객체의 상태가 이벤트에 따라 올바르게 변하는지 시나리오 검증.
5. **페어와이즈 (Pairwise)**: 수많은 입력 변수들의 조합 중, **두 개의 변수 간의 모든 조합**이 최소 한 번은 테스트되도록 최적화. (조합 폭발 방지)

---

## Ⅲ. 고급 블랙박스 기법
* **원인-결과 그래프**: 입력 조건(원인)과 출력(결과)의 관계를 논리 게이트로 분석하여 테스트 케이스 도출.
* **유스케이스 테스팅**: 사용자의 실제 서비스 이용 흐름(Scenario)을 기반으로 테스트.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[블랙박스 기법의 공략 포인트]

  [ Input ] ──> |  Black  | ──> [ Output ]
                |   Box   |
      │             │             │
      ├─ 동등분할 (대푯값)         └─ 결과값이 맞나?
      ├─ 경계값 (끝부분)
      └─ 페어와이즈 (핵심조합)
```

📢 **섹션 요약 비유**: **블랙박스 테스트**는 상자 안에 뭐가 들었는지 모르고 밖에서 두드려보는 것입니다. **동등 분할**은 수박 한 트럭 중 하나만 먹어보고 당도를 판단하는 것이고, **경계값 분석**은 옷 치수가 '95'인지 '100'인지 경계선에 있는 옷을 입어보며 꽉 끼는지 확인하는 것입니다. **페어와이즈**는 수천 가지 반찬 조합 중 궁합이 안 맞는 핵심 '단짝'들만 골라 먹어보며 배탈(오류)이 나는지 보는 것과 같습니다.
