# 408. 베타 테스트 (Beta Test)

## 핵심 인사이트 (3줄 요약)
> 1. 알파 테스트와 정반대로, **개발자가 도저히 어찌할 수 없는 "통제 불가능한 환경(고객의 옛날 폰 환경 등)"** 에서 프로그램이 정상 작동하는지를 검증한다.
> 2. 주로 상용 제품(B2C)이나 게임 등에서 출시 직전에 수만 명의 트래픽 서버 부하 테스트(스트레스)를 공짜로 얻어내기 위해 클로즈/오픈 베타의 방식을 띤다.
> 3. 여기서 피드백된 버그 리포트(Bug Report)는 출시 후 핫픽스(Hotfix)를 위한 생명선이 되며 노이즈 마케팅의 효과까지 겸비한다.

---
## Ⅰ. 아키텍처 및 원리
```text
  [ 베타 테스트 환경의 확산망 ]
    [ 운영 스펙 임시 서버 (Live Proxy) ] <=== (인터넷) ===> [ 테스터 A (윈도우7) ]
           |                                             [ 테스터 B (맥북) ]
       버그 리포팅 / 텔레메트리 자동 수집                      [ 테스터 C (모바일 3G망) ]
```
---
## Ⅱ. 실무 적용 및 결론
"고객의 실 환경은 개발 환경과 무조건 다르다." 아무리 사내에서 완벽했어도 낡은 백신 프로그램과 충돌이 나서 앱이 죽어버리는 현상은 베타 테스트를 통해서만 검출(필드 테스트 방어) 가능하다. 오늘날 얼리 액세스(Early Access)라는 이름으로 유료로 베타를 파는 상술적 비즈니스 모델로 변질되기도 했다.
