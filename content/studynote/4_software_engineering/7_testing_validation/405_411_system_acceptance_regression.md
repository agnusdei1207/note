# 405-411. 시스템, 인수 및 회귀 테스트

> **핵심 인사이트**: 개발이 끝났다고 테스트가 끝난 것은 아니다. 전체 시스템의 조화를 보고(시스템), 주인의 맘에 드는지 확인하며(인수), 하나를 고쳤을 때 열이 망가지지 않았는지(회귀)를 끊임없이 확인해야 한다. 특히 회귀 테스트는 소프트웨어의 진화 과정에서 '어제의 성공'을 보장하는 신뢰의 기반이다.

---

## Ⅰ. 시스템 테스트 (System Test)
통합된 전체 시스템이 요구사항(기능 및 비기능)을 모두 만족하는지 검증하는 단계입니다.
* **항목**: 성능, 보안, 회복, 강도(Stress) 테스트 등.

---

## Ⅱ. 인수 테스트 (Acceptance Test)
사용자(고객)가 시스템을 최종적으로 인수할 것인지를 결정하기 위해 수행하는 테스트입니다.

1. **알파 테스트 (Alpha Test)**: 개발자의 장소에서 사용자가 통제된 환경 하에 수행.
2. **베타 테스트 (Beta Test)**: 실제 사용 환경(필드)에서 다수의 사용자가 자유롭게 수행.
3. **규정 준수 테스트**: 법적 규제나 표준을 만족하는지 확인.

---

## Ⅲ. 회귀 테스트 (Regression Test)
소프트웨어 수정(기능 추가, 버그 수정 등) 이후, **기존에 잘 작동하던 기능에 부작용(Side Effect)이 없는지** 확인하기 위해 다시 수행하는 테스트입니다.

* **전략**:
  - **Retest All**: 전체 테스트 케이스 다시 수행. (안전하지만 비용 큼)
  - **Selective**: 변경된 모듈과 연관된 부분만 골라서 수행.
* **가치**: 배포 후 발생할 수 있는 '황당한 버그'를 막아주는 가장 중요한 안전망입니다.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[테스트 단계별 목적 분류]

  [ System Test ] ────> "기술적으로 완벽한가?" (개발자 완료)
          │
  [ Acceptance Test ] ──> "사용하기에 만족한가?" (고객 승인)
          │
  [ Regression Test ] ──> "고친 뒤에 딴 데 안 터졌나?" (지속적 신뢰)
```

📢 **섹션 요약 비유**: **시스템 테스트**는 자동차 조립 후 시운전을 해보며 가속과 제동이 잘 되는지 확인하는 전수 검사입니다. **인수 테스트**는 차를 사러 온 고객이 직접 앉아보고 "시트가 편하네"라며 계약서에 사인하는 과정입니다. **알파 테스트**는 본사 전용 트랙 시승, **베타 테스트**는 일반 도로 시승입니다. **회귀 테스트**는 타이어 하나 갈았는데 브레이크가 안 듣는 불상사가 없는지 확인하는 '정기 정비'와 같습니다.
