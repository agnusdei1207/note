# 403. 상향식 통합 (Bottom-up Integration)

## 핵심 인사이트 (3줄 요약)
> 1. 하향식의 반대 개념으로, 하위 단위 모듈들이 완전히 검증된 상태에서 결합되므로 **시스템의 코어 로직(DB I/O, 보안)의 결함을 초기에 확실히 사살**한다.
> 2. 윗단의 호출 함수가 아직 안 만들어진 상태이므로, 하위 모듈에 테스트 데이터를 꽂아줄 가짜 제어 모듈인 **테스트 드라이버(Test Driver)** 가 필수적으로 요구된다.
> 3. 전체 UI 시스템(화면)을 프로젝트 맨 마지막에나 볼 수 있기 때문에 경영진이나 사용자가 피드백을 적시에 주지 못하고 기다려야 하는 소통 단절 약점이 존재한다.

---
## Ⅰ. 아키텍처 및 원리
```text
  [ 상향식 뼈대 조립망 ]
        [ 메인 제어(Driver 임시 껍데기) ] <-- 아직 코드 없는 구역
           |              |
      [ DB 접근 ]    [ 로깅 모듈 ] <-- 가장 먼저 조립/검증 완료
```
---
## Ⅱ. 실무 적용 및 결론
객체 지향 시스템(OOP)이나 MSA 환경에서 공통 라이브러리(Utility, Core)를 개발할 때 완벽하게 맞아떨어진다. 밑바닥부터 탄탄하게 벽돌을 굳히고 올라가므로, 전체 구조가 무너질 일은 없으나 프로토타이핑(Prototyping) 전개 관점에서는 비효율적이다.
