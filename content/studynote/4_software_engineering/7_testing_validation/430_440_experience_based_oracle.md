# 430-440. 경험 기반 테스팅과 테스트 오라클

> **핵심 인사이트**: 명세서에 적힌 대로만 테스트해서는 '의외의 버그'를 잡을 수 없다. 테스터의 노련한 직관을 발휘하는 탐색적 테스팅은 문서화의 부담을 줄이고 결함 발견의 속도를 높인다. 또한, 테스트 결과가 참인지 거짓인지 판별해 주는 '오라클'을 명확히 세워야만 자동화와 객관적 검증이 가능해진다.

---

## Ⅰ. 경험 기반 테스팅 (Experience-based)
명세서보다는 테스터의 기술, 지식, 직관에 의존하는 방식입니다.

1. **탐색적 테스팅 (Exploratory Testing)**: 테스트 설계와 수행을 동시에 진행.
   - **구성**: 차터(테스트 목표), 타임박스(시간 제한), 회고.
2. **오류 추정 (Error Guessing)**: "여기쯤에 버그가 있겠지?"라고 과거 경험을 살려 추측.
3. **리스크 기반 테스팅**: 비즈니스 영향도가 높거나 고장 확률이 높은 곳에 자원을 집중.

---

## Ⅱ. 테스트 오라클 (Test Oracle)
테스트를 수행한 결과가 **옳은지 그른지 판단**하기 위해 사전에 정의된 참값입니다.

* **4대 유형**:
  1. **참 오라클 (True)**: 모든 입력에 대해 정답을 알고 있는 경우. (현실적으로 어려움)
  2. **샘플링 오라클**: 특정 몇몇 입력에 대해서만 정답 제공.
  3. **휴리스틱 오라클 (Heuristic)**: 샘플링 오라클 + 추정 및 직관적 판단.
  4. **일관성 오라클 (Consistent)**: 이전 버전의 실행 결과와 현재 결과가 같은지 비교. (회귀 테스트에 유용)

---

## Ⅲ. 정적 테스팅 vs 동적 테스팅
1. **정적 테스팅 (Static)**: 코드를 실행하지 않음. (리뷰, 인스펙션, 정적 분석 도구)
2. **동적 테스팅 (Dynamic)**: 코드를 직접 실행하여 결함 발견.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[테스트 수행의 두 축]

  ( 명세 기반 ) [ Black-box ] <───> [ Experience-based ] ( 경험 기반 )
                                            │
                                            ▼
                                [ Test Oracle (판단 기준) ]
```

📢 **섹션 요약 비유**: **경험 기반 테스팅**은 베테랑 요리사가 레시피 없이도 간을 보며 맛을 맞추는 것이고, **탐색적 테스팅**은 처음 가보는 미지의 숲을 지도 없이 일단 발길 닿는 대로 가보며 지형을 파악하는 탐험입니다. **테스트 오라클**은 요리 대회의 '심사위원'입니다. 모든 요리에 점수를 매기는 완벽한 심사위원(참 오라클)부터, 특정 요리만 맛보는 심사위원(샘플링)까지 다양하게 존재합니다.
