# 464. 서비스 가상화 (Service Virtualization)

> ⚠️ 이 문서는 한두 개의 단위 테스트용 목(Mock) 객체를 넘어서, 수십 개의 컨테이너가 거미줄처럼 통신하는 현대의 MSA(마이크로서비스 아키텍처) 환경에서, **내 서비스와 연동되어야 하지만 아직 개발되지 않았거나 점검 중인 '외부 API 서버 통째로'를 완벽하게 모방하여 띄워놓는 거대한 가짜 서버, '서비스 가상화'**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단위 테스트의 `Stub`이나 `Mock`이 코드 레벨(메모리)에서 노는 작은 가짜라면, 서비스 가상화(Service Virtualization)는 아예 독립된 네트워크 포트(HTTP/REST)를 열고 응답하는 **'거대한 통 스텁 서버(Stub Server)'**다.
> 2. **가치**: 카카오페이 결제망, 은행망 등 내가 마음대로 껐다 켰다 통제할 수 없거나 너무 비싼 외부 3rd-party 시스템들을 가상화시켜버림으로써, 수백 명의 개발자가 외부 환경의 눈치를 보지 않고 병렬로 E2E(End-to-End) 테스트를 달릴 수 있게 해준다.
> 3. **기술 체계**: Mountebank, WireMock 같은 전용 도구를 사용하며, 단순히 고정된 텍스트 응답을 넘어서서 트래픽 딜레이(네트워크 랙)를 고의로 유발하거나 복잡한 상태 머신(State Machine)까지 흉내 내는 등 차원이 다른 모사(Simulation) 능력을 자랑한다.

---

### Ⅰ. 개요: Mock 객체만으로는 버틸 수 없는 세상 (Context & Necessity)
거대한 쇼핑몰 플랫폼(MSA)을 생각해보자. 내가 만든 '주문 서비스' 컨테이너가 정상 동작하려면 다음과 통신해야 한다.
1. `카카오페이 외부 API` (돈 빠져나감, 1분에 100건 제한 있음)
2. `사내 배송 서비스 API` (다른 팀이 만들고 있는데 2주 뒤에나 완성됨)
3. `국세청 국세망 API` (새벽 1시부터 5시까지 매일 점검함)

내일 '주문 서비스' 통합 테스트를 돌려야 한다. 
코드 안에 Mock 객체를 끼워 넣는 건 한계가 있다. 나는 진짜 HTTP REST API로 쏘고 응답받는 전체 흐름을 검증하고 싶은데, 카카오페이는 돈이 나가고 배송팀은 완성도 안 됐고 국세청은 점검 중이다. 개발이 멈췄다(Block).

이때 인프라 엔지니어가 나선다. **"까짓것, 카카오페이랑 국세청이랑 똑같이 응답하는 가짜 서버(가상 머신) 3대를 클라우드에 새로 띄워줄게! 내일부터 이 가짜 IP 주소로 통신해!"**
이것이 외부 환경의 불확실성을 완벽하게 썰어버리는 **서비스 가상화(Service Virtualization)**의 등장이다.

> 📢 **섹션 요약 비유**: 단위 테스트의 Mock이 영화 세트장의 '가짜 소품(모형 권총)'이라면, 서비스 가상화는 아예 '뉴욕 한복판 세트장을 통째로 할리우드에 똑같이 지어버리는 것'입니다. 배우(내 시스템)는 비행기를 타고 뉴욕(진짜 외부망)에 갈 필요 없이 세트장에서 편하게 폭파 씬을 마음껏 찍을 수 있습니다.

### Ⅱ. 서비스 가상화의 3대 초격차 특징

서비스 가상화는 껍데기뿐인 스텁(Stub Server)의 진화형으로, 단순히 `200 OK`를 뱉는 것을 넘어선다.

#### 1. 프로토콜과 네트워크 흉내 내기 (Protocol Simulation)
- HTTP/REST API뿐만 아니라, TCP, UDP, JMS, gRPC 등 거의 모든 네트워크 프로토콜을 받아내고 흉내 낸다. 내 시스템 입장에서는 진짜 네트워크 너머의 실존하는 서버와 통신하는 것과 100% 똑같은 착각을 일으킨다.

#### 2. 네트워크 결함/성능의 의도적 모사 (Fault Simulation)
- 단순히 긍정적인 대답만 흉내 내지 않는다. "이 가짜 국세청 서버는 앞으로 3초 동안 타임아웃(Timeout) 랙을 유발해라", "10번 호출하면 1번은 고의로 `500 에러`를 던져라!"라고 세팅해 둘 수 있다.
- 덕분에 외부 서버가 미쳐 돌아갈 때 우리 쪽 시스템의 '서킷 브레이커(Circuit Breaker)'가 훌륭하게 방어해 내는지를 극한으로 테스트할 수 있다.

#### 3. 기록 및 재생 (Record & Playback)
- 가짜 서버 대본을 일일이 짜기 귀찮은가? 가상화 도구를 '진짜 카카오페이 망' 중간에 스니퍼처럼 며칠 끼워 둔다(Record).
- 며칠 동안 왔다 갔다 한 수백만 개의 트래픽 흐름을 자기가 알아서 녹음하고 학습한다. 그리고 진짜 망을 끊어버리면, 자기가 카카오페이인 척 그 대본 그대로 리플레이(Playback)를 해준다. 마법 같은 기술이다.

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│           서비스 가상화(Service Virtualization) 아키텍처 도입 전/후 비교     │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│ ❌ [ 도입 전 : 막혀버린 테스트 파이프라인 ]                                  │
│   [ 나의 시스템 ] ──(호출)──▶ [ 진짜 카카오 결제망 ] (과금폭탄/접근제한)     │
│   [ 나의 시스템 ] ──(호출)──▶ [ 옆팀 배송 시스템 ] (아직 개발 안됨 🚫)       │
│                                                                              │
│ ────────────────────────────────────────────────────────────                 │
│ 🛡️ [ 도입 후 : WireMock 등을 통한 가상화 천국 ]                              │
│   [ 나의 시스템 ] ──(네트워크)──▶ ☁️ [ 가상화 서버 (WireMock) ]              │
│                                ├── "나 카카오 결제망이야 결제 성공!"         │
│                                └── "나 배송 시스템이야 배송 중!"             │
│                                                                              │
│ ★ 결론: 수백 명의 개발자가 외부 눈치, 요금 걱정 1도 없이 무제한 24시간 통신  │
│         시뮬레이션과 통합 테스트(Integration)를 빵빵 터뜨릴 수 있다!         │
└──────────────────────────────────────────────────────────────────────────────┘
```

### Ⅲ. 실무 툴과 모의 서버의 위상

대표적인 오픈소스 서비스 가상화 툴은 **WireMock**과 **Mountebank**다.
- JSON이나 YAML 파일로 대본(Contract)을 짜두면 즉시 웹서버로 변신하여 포트를 열어준다.
- 최신 MSA 트렌드인 **소비자 주도 계약 테스트(Consumer-Driven Contract Testing, Pact)**의 핵심 인프라로 쓰인다. "야 옆팀! 너네 API 아직 다 안 만들었지? 일단 이 '가짜 계약서 서버(가상화)' 먼저 띄워놔. 우리 프론트팀은 이거 찔러보면서 화면 먼저 만들고 있을게!" (병렬 개발의 극대화)

---

### Ⅳ. 결론

"거대한 사일로(Silo)를 박살 내는 가상화의 힘."
마이크로서비스 아키텍처(MSA)는 시스템을 수백 개로 쪼개어 민첩성을 얻었지만, 반대급부로 "다른 놈이 살아있지 않으면 나도 테스트를 못 하는" 거미줄 같은 의존성의 늪을 낳았다. 서비스 가상화(Service Virtualization)는 이 복잡성의 늪에 빠진 대기업 IT 인프라를 구원한 거대한 방주다. 이는 단순한 테스트 도구를 넘어 개발, QA, 운영 팀이 서로의 일정을 갉아먹지 않고 완전히 독립적인(Decoupled) 우주에서 최대 속도로 질주할 수 있게 만들어 주는, 현대 IT 조립 공정의 가장 위대한 가속 장치(Accelerator)라 할 수 있다.

### 📌 관련 개념 맵
* **전제되는 환경**: 마이크로서비스 아키텍처 (MSA), 분산 시스템
* **상위 개념**: 테스트 더블 (Test Double) $\rightarrow$ 그중 스텁(Stub)이 서버 단위로 커진 형태
* **활용 테스트**: 통합 테스트 (Integration Testing), E2E 테스트, 계약 테스트 (Contract Testing)
* **대표 오픈소스 툴**: WireMock, Mountebank, MockServer
