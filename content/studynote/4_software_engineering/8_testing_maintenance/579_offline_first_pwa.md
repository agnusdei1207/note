# 579. 오프라인 우선 (Offline-first) 아키텍처

> ⚠️ 이 문서는 웹 브라우저는 인터넷이 끊기면 공룡 게임이나 해야 한다는 편견을 깨고, **비행기 모드나 터널 안에서도 웹 앱이 정상적으로 작동하고 데이터를 저장하게 만들어주는 PWA(Progressive Web App)의 오프라인 우선 아키텍처**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: "네트워크는 항상 끊긴다"고 가정한다. 사용자가 무언가를 클릭했을 때 가장 먼저 클라우드 서버에 물어보는 것이 아니라, 브라우저 내부의 로컬 DB에 먼저 물어보고 화면을 보여주는 방식이다.
> 2. **가치**: 모바일 앱(Native App)과 동일한 수준의 부드러운 사용자 경험을 제공한다. 인터넷이 끊긴 상태에서 이메일을 작성하고 전송 버튼을 누르면, 인터넷이 연결되는 순간 백그라운드에서 자동으로 서버로 전송해 준다.
> 3. **기술 체계**: 네트워크와 브라우저 사이에 문지기로 서 있는 **서비스 워커(Service Worker)**와 브라우저 내부의 데이터베이스인 **IndexedDB**가 핵심 기술이다.

---

### Ⅰ. 개요: 터널에서 글이 날아갔다 (Context & Necessity)
지하철을 타고 출근하며 브라우저로 열심히 블로그 글을 쓰고 있었다.
터널에 진입해서 인터넷이 끊겼는데, 실수로 '저장' 버튼을 눌렀다.
- 일반 웹사이트: `인터넷 연결 없음` 화면이 뜨고, 30분 동안 쓴 글이 허공으로 날아간다. (사용자 대분노)

**오프라인 우선(Offline-first) 앱**은 다르다.
인터넷이 끊긴 상태에서 '저장'을 누르면, 일단 내 폰(브라우저) 안에 있는 미니 데이터베이스에 글을 몰래 저장해 둔다. 그리고 화면에는 "저장되었습니다!"라고 띄워준다. 나중에 지하철에서 내려서 인터넷이 연결되면, 내 폰이 알아서 클라우드 서버로 글을 쏴준다.

> 📢 **섹션 요약 비유**: 일반 웹사이트가 **'실시간 인터넷 뱅킹'**이라면, 오프라인 우선 웹 앱은 **'가계부 수첩'**과 같습니다. 은행 전산망(인터넷)이 끊겨도 내 수첩에 일단 지출 내역을 다 적어둘 수 있고, 나중에 은행 문을 열면 수첩을 보고 은행 장부(서버)를 한 번에 업데이트하는 것과 같습니다.

---

### Ⅱ. 오프라인 아키텍처의 2대 핵심 기술

웹 브라우저를 모바일 앱처럼 강력하게 만들어주는 HTML5 표준 기술들이다.

#### 1. 서비스 워커 (Service Worker) - 똑똑한 문지기
- 브라우저와 인터넷망 사이에 서 있는 투명한 프록시(Proxy) 서버다.
- 브라우저가 이미지를 요청하면 서비스 워커가 낚아챈다. 
- "어? 인터넷 끊겼네? 괜찮아, 어제 받아둔 이미지 캐시가 내 주머니에 있으니까 이거 보여줘!" (오프라인 화면 표시 가능)

#### 2. IndexedDB - 브라우저 속의 진짜 DB
- 단순한 텍스트 쪼가리를 저장하는 쿠키(Cookie)나 로컬 스토리지(LocalStorage)와 다르다.
- 수백 메가바이트(MB)의 진짜 객체 데이터(JSON)를 저장하고 검색할 수 있는 브라우저 내장 NoSQL 데이터베이스다.

---

### Ⅲ. Background Sync (백그라운드 동기화)

사용자가 인터넷이 끊겼을 때 누른 '좋아요' 버튼은 어떻게 서버로 전달될까?

1. 사용자가 오프라인에서 '좋아요' 클릭.
2. 서비스 워커가 이 요청을 가로채서 IndexedDB의 `[Sync Queue]`라는 폴더에 저장함.
3. 사용자가 브라우저를 끄고 주머니에 폰을 넣음.
4. 나중에 와이파이가 연결됨.
5. 브라우저가 꺼져 있어도 **서비스 워커가 백그라운드에서 조용히 깨어나** 큐에 있던 '좋아요' 요청을 서버로 날림.

```text
┌──────────────────────────────────────────────────────────────┐
│           오프라인 우선 (Offline-first) 작동 구조 시각화              │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ [ 📱 브라우저 화면 ]                                             │
│       │ (게시글 저장 요청)                                       │
│       ▼                                                      │
│ ┌──────────────────┐  (인터넷 연결됨?) ──▶ [ ☁️ 클라우드 서버 ]  │
│ │ 🤖 Service Worker│                                         │
│ └──────────────────┘  (인터넷 끊김!) ───▶ [ 🗄️ IndexedDB ]    │
│                                           (로컬에 임시 저장)     │
│                                                              │
│ ★ 특징: 화면은 인터넷 상태와 무관하게 로컬 DB의 응답을 받고 바로 넘어감.  │
└──────────────────────────────────────────────────────────────┘
```

### Ⅳ. 결론

"웹의 미래는 브라우저 안에 갇혀있지 않다."
오프라인 우선 아키텍처는 스마트폰 시대에 웹(Web)이 네이티브 앱(Native App)을 이기기 위해 꺼내 든 강력한 무기다. 이를 적용한 웹사이트(PWA)는 홈 화면에 아이콘으로 설치할 수 있고, 푸시 알림도 보낼 수 있으며, 인터넷 없이도 작동한다. 개발 난이도는 크게 올라가고 캐시 무효화(Cache Invalidation) 같은 골치 아픈 문제가 생기지만, 아프리카처럼 네트워크가 불안정한 국가를 타겟팅하거나 스타벅스 앱처럼 매끄러운 UX가 생명인 서비스에서는 필수적인 아키텍처다.

---

### 📌 관련 개념 맵
* **관련 기술**: PWA (Progressive Web Apps), Web App Manifest
* **브라우저 저장소**: IndexedDB, Cache API, LocalStorage
* **핵심 스크립트**: Service Worker (메인 스레드와 별개로 백그라운드에서 동작)
* **네트워크 전략**: Cache First, Network First, Stale-while-revalidate

---

### 👶 어린이를 위한 3줄 비유 설명
1. 일반 웹사이트는 텔레비전 생방송과 같아요. 안테나(인터넷)가 뽑히면 화면이 치지직거리며 아무것도 안 보이죠.
2. 오프라인 우선 앱은 '넷플릭스 저장하기' 기능과 같아요. 인터넷이 끊겨도 비행기 안에서 미리 받아둔 만화영화를 볼 수 있죠.
3. 그리고 비행기 안에서 '다음 화 보고 싶어요' 버튼을 눌러두면, 비행기에서 내려서 와이파이가 터질 때 폰이 알아서 몰래 다음 화를 다운받아 놓는답니다!
