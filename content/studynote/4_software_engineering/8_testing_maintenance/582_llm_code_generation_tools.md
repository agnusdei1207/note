# 582. LLM 기반 코드 생성 도구 (Copilot, Cursor 등)

> ⚠️ 이 문서는 구글을 검색하고 Stack Overflow를 뒤적이며 남의 코드를 복사해 오던 개발자들의 일상을 완전히 바꿔버린, **에디터 안에서 내 마음을 읽고 코드를 자동으로 짜주는 '초거대 언어 모델(LLM) 기반의 코드 어시스턴트'**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 수십억 줄의 오픈소스 코드를 학습한 AI(주로 OpenAI의 GPT 계열)가 개발자가 지금 치고 있는 코드의 '문맥(Context)'을 파악하여 다음 줄에 올 코드를 1초 만에 예측하고 제안해 주는 도구다.
> 2. **가치**: 보일러플레이트(Boilerplate, 반복되는 뻔한 코드)를 치는 시간을 완전히 없애주고, 처음 써보는 낯선 언어나 라이브러리도 공식 문서를 읽을 필요 없이 바로 코딩할 수 있게 해준다.
> 3. **기술 체계**: 에디터 플러그인 형태의 **GitHub Copilot**이 1세대를 열었고, 아예 에디터 전체가 AI와 유기적으로 결합된 **Cursor(커서)** 편집기가 2세대를 이끌고 있다.

---

### Ⅰ. 개요: 스택오버플로우의 몰락 (Context & Necessity)
"자바스크립트로 날짜 계산하는 함수 어떻게 만들더라?"
과거의 개발자는 브라우저를 열고, 구글에 검색하고, 스택오버플로우에 올라온 답변을 긁어서 내 에디터에 붙여넣고, 내 변수명에 맞게 다시 고쳐야 했다. (최소 3분 소요)

**LLM 코드 생성 도구**는 이 과정을 1초로 단축했다.
에디터에 주석으로 `// 오늘부터 3일 뒤 날짜를 YYYY-MM-DD로 반환`이라고 치고 엔터를 누르면, AI가 흐릿한 회색 글씨로 완벽한 함수를 제안한다. 개발자는 `Tab` 키만 누르면 끝이다.

> 📢 **섹션 요약 비유**: 과거의 개발 방식이 **'모르는 게 생길 때마다 도서관에 가서 백과사전을 찾아보는 것'**이었다면, LLM 도구는 내 옆자리에 **'세상의 모든 책을 다 외우고 있는 천재 조수'**를 앉혀두고 일하는 것과 같습니다.

---

### Ⅱ. 대표적인 AI 코드 생성 도구들

#### 1. GitHub Copilot (깃허브 코파일럿) - 1세대 선구자
- **특징**: VS Code, IntelliJ 같은 기존 에디터에 '플러그인' 형태로 설치해서 쓴다.
- **방식**: 내가 지금 열어놓고 있는 파일의 앞뒤 코드를 읽고 다음 코드를 예측(Auto-complete)하는 데 특화되어 있다.

#### 2. Cursor (커서) - 2세대 혁명
- **특징**: VS Code를 포크(Fork)해서 아예 처음부터 AI 전용으로 마개조한 '새로운 에디터'다.
- **강점 (코드베이스 인지)**: Copilot이 내가 연 파일만 본다면, Cursor는 **"내 프로젝트 폴더 전체(수백 개의 파일)"를 다 읽고 이해한다.**
- **예시**: "방금 추가한 DB 컬럼을 프론트엔드 화면까지 다 연결해 줘"라고 치면, Cursor가 백엔드 컨트롤러, 프론트엔드 API, UI 파일 3개를 동시에 알아서 수정해 버린다.

#### 3. Devin, AutoGPT - 3세대 자율 에이전트 (미래)
- **특징**: "이런 웹사이트 만들어줘"라고 던져주면, AI 혼자서 브라우저를 열어 문서를 검색하고, 에러가 나면 스스로 디버깅하며 앱 전체를 완성해 내는 자율 로봇이다.

---

### Ⅲ. 실무 활용 팁: AI를 100% 활용하는 법

AI는 독심술사가 아니다. 좋은 코드를 얻으려면 좋은 힌트(Context)를 줘야 한다.

1. **명확한 주석 쓰기**: `// 데이터 가져오기` (X) $\rightarrow$ `// URL /api/users에서 데이터를 가져와서 나이순으로 내림차순 정렬 후 반환` (O)
2. **네이밍 컨벤션 지키기**: 함수 이름을 `doSomething()`이라고 지으면 AI가 바보가 된다. `fetchActiveUsers()`라고 지어두면 엔터만 쳐도 안의 로직을 완벽하게 짜준다.
3. **테스트 코드 짜달라고 하기**: 인간이 가장 귀찮아하는 `Jest`나 `JUnit` 테스트 코드는 AI가 가장 잘 짜는 분야다. 함수를 블록 지정하고 `/test`라고 치면 경계값 테스트까지 완벽하게 짜준다.

```text
┌──────────────────────────────────────────────────────────────┐
│           GitHub Copilot의 코드 생성 작동 원리 시각화                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ [ 👨‍💻 개발자 (VS Code) ]                                      │
│  // 이메일 유효성을 검사하는 함수를 만들어줘 (주석 작성)                │
│  function validateEmail(email) {                             │
│                                                              │
│        (1. 이 주석과 앞뒤 코드를 OpenAI 서버로 전송)                 │
│        ────────────────────────────────────▶                 │
│                                            [ 🧠 OpenAI LLM ] │
│        ◀────────────────────────────────────                 │
│        (2. 예측된 코드를 반환)                                    │
│                                                              │
│      const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/; (회색 글씨로 뜸)│
│      return regex.test(email);                               │
│  }                                                           │
│  (3. 개발자가 'Tab' 키를 누르면 코드 완성!)                         │
└──────────────────────────────────────────────────────────────┘
```

### Ⅳ. 결론

"도구를 거부하는 자는, 도구를 쓰는 자에게 대체된다."
LLM 기반 코드 생성 도구는 단순한 유행이 아니라 프로그래밍의 표준(Default)이 되었다. 이것을 쓰지 않고 코딩하는 것은 포크레인을 놔두고 삽으로 땅을 파는 것과 같은 기행이다. 하지만 주의할 점이 있다. AI가 짜준 코드를 '내가 완벽하게 이해하고 리뷰할 수 있는 실력'이 없다면, 내 프로젝트는 언제 터질지 모르는 시한폭탄(Hallucination)으로 가득 차게 될 것이다. (이러한 부작용은 583번 문서에서 다룬다.)

### 👶 어린이를 위한 3줄 비유 설명
1. 코딩은 블록 장난감을 조립하는 것과 같아요. 예전엔 내가 설명서를 보고 수백 개의 블록을 직접 다 끼워 맞춰야 했죠.
2. Copilot은 내 옆에 앉은 똑똑한 조수예요. 내가 "자동차 바퀴를 만들 거야"라고 하면, 조수가 바퀴 블록 세트를 찾아서 내 손에 쥐여주죠.
3. 나는 그 바퀴가 내 자동차에 맞는지 확인하고 "오케이!"(Tab 키) 하고 끼우기만 하면 아주 빨리 장난감을 완성할 수 있답니다!
