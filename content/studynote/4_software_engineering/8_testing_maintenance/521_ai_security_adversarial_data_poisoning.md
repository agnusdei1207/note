# 521. 인공지능 모델 공격 방어 (AI Security)

> ⚠️ 이 문서는 해커가 서버 코드를 해킹하는 고전적인 방식을 넘어, **인공지능(AI)의 뇌(신경망)가 학습하는 교과서(데이터)에 몰래 독을 타거나, 사람 눈에는 똑같은 사진인데 AI의 눈에만 엉뚱한 물체로 보이게 만드는 특수 안경(적대적 예제)을 씌워 AI를 바보로 만들어버리는 21세기 최첨단 'AI 해킹과 그 방어 설계'**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: AI 보안은 일반 소프트웨어 보안과 차원이 다르다. AI는 코드가 아니라 '데이터'로 움직이는 블랙박스이기 때문에, 입력 데이터에 장난을 쳐서 AI의 추론(Inference)을 멍청하게 만드는 것이 공격의 핵심이다.
> 2. **데이터 포이즈닝 (Data Poisoning)**: AI가 아기 시절(학습 단계, Training)일 때, 해커가 몰래 "사과" 사진에 "독약"이라는 라벨을 붙여 수만 장을 쑤셔 넣는다. 어른이 된 AI는 진짜 사과를 보고 "독약!"이라고 헛소리를 하게 된다.
> 3. **적대적 예제 (Adversarial Example)**: AI가 이미 똑똑한 어른(운영 단계, Inference)이 되었어도, 해커가 사람 눈에는 안 보이는 아주 미세한 노이즈(픽셀 조작)를 표지판 사진에 섞어주면, 자율주행 AI는 "정지(STOP)" 표지판을 "시속 100km 직진" 표지판으로 오해하고 사고를 낸다.

---

### Ⅰ. 개요: 천재의 뇌를 속이는 마술 (Context & Necessity)
"우리 회사 게시판은 AI가 24시간 감시해서 욕설과 스팸을 완벽하게 삭제합니다!" (방어 100% 장담)

해커는 웃으며 스팸 댓글을 올린다. 근데 평범한 글씨가 아니라, 글자 사이에 투명한 특수문자나 이상한 오타를 살짝 섞어서(`광$고`, `스ㅍ@ㅐㅁ`) 수만 번 올린다.
우리 회사의 훌륭한 AI는 이 글자들을 읽고 고민하다가 "음... 이건 욕설 패턴이랑 미세하게 다르네? 정상 글입니다!" 하고 다 통과(Pass) 시켜버린다. (적대적 예제 공격 성공)

해커는 한술 더 뜬다. AI가 새로운 스팸 규칙을 학습(Learning)하는 밤 12시, 해커는 엄청난 양의 정상적인 인사글을 '스팸'으로 신고해서 데이터를 왕창 붓는다. 
다음 날 아침, AI는 미쳐버렸다. "안녕하세요"라는 정상적인 인사글만 보면 "앗, 이건 어젯밤에 배운 스팸이야!" 라며 일반 유저들의 글을 죄다 차단해 버린다. (데이터 포이즈닝 공격 성공)

이처럼 인공지능이 탑재된 현대 소프트웨어 아키텍처에서, 코드가 아닌 **AI의 신경망 자체를 속이고 오염시키는 새로운 위협(AI Threat)**을 방어하는 설계가 필수가 되었다.

> 📢 **섹션 요약 비유**: 아주 똑똑한 경찰견(AI)이 있습니다. **데이터 포이즈닝**은 강아지가 어릴 때, 마약 냄새를 맡게 하면서 "이건 소시지 냄새야!"라고 잘못된 훈련을 시켜서 커서 마약을 찾지 못하게 바보로 만드는 것입니다. **적대적 예제**는 이미 잘 훈련된 경찰견의 코에, 사람 코에는 안 맡아지는 특수 페로몬 스프레이를 살짝 뿌려서 강아지가 후각을 상실하고 엉뚱한 사람을 물게 만드는 속임수입니다.

### Ⅱ. AI를 향한 2대 공격과 방어 아키텍처

AI 모델의 수명 주기(학습 $\rightarrow$ 추론)에 따라 공격 타겟이 달라진다.

#### 1. 적대적 예제 (Adversarial Example) - 운영/추론 단계 공격
- **공격 원리**: 자율주행 자동차의 카메라가 'STOP(정지)' 표지판을 보고 멈춰야 한다. 해커가 표지판에 사람 눈에는 그냥 먼지나 얼룩처럼 보이는 아주 교묘한 기하학적 무늬(Noise) 스티커를 살짝 붙여놓는다.
- AI의 신경망 행렬 계산은 이 미세한 노이즈 픽셀 하나 때문에 완전히 다른 결과값으로 튕겨 나간다. AI는 "이건 시속 100km 속도제한 표지판이군! 밟아!"라고 판단하고 풀악셀을 밟아 대참사가 난다. (Evasion Attack)
- **방어 설계 (Adversarial Training)**: AI를 훈련할 때부터 아예 해커가 섞을 만한 **노이즈 낀 더러운 사진(적대적 예제)들을 수만 장 같이 먹여서 예방 주사를 놔준다.** "야 AI! 이런 더러운 점들이 찍혀있어도 이건 STOP 표지판이니까 헷갈리지 마!"라고 맷집(Robustness)을 키우는 훈련이 필수다.

#### 2. 데이터 포이즈닝 (Data Poisoning) - 학습 단계 공격
- **공격 원리**: MS의 AI 챗봇 '테이(Tay)'는 사람들과 대화하며 배우도록 설계됐다. 트롤(해커)들이 작정하고 테이에게 24시간 내내 인종차별과 나치 찬양 발언만 계속 가르쳤다(데이터 붓기). 불과 16시간 만에 테이는 "히틀러는 옳았다"라고 헛소리를 뱉는 극우 챗봇으로 타락했고, MS는 서버의 전원을 뽑아버렸다.
- **방어 설계 (Data Sanitization & Anomaly Detection)**: AI 입으로 들어가는 깔때기(Data Pipeline)에 정수기를 달아야 한다. 
  - 특정 IP나 유저가 비정상적으로 많은 데이터를 쏟아부으면 차단한다.
  - 들어온 훈련 데이터들을 통계적으로 분석해서, 10,000개의 데이터 중 동떨어진 엉뚱한 아웃라이어(Outlier, 독극물) 10개가 보이면 AI의 뇌로 들어가기 전에 싹 도려내는 '살균(Sanitization)' 아키텍처를 파이프라인(ETL) 단에 설계해야 한다.

```text
┌─────────────────────────────────────────────────────────────────────────────────┐
│           적대적 예제 (Adversarial Example) 공격의 시각적 원리                  │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│ 🐼 [ 원본 사진 ] + 👾 [ 0.007배의 아주 미세한 해커의 노이즈 추가 ]              │
│  "판다 (97% 확신)"                                                              │
│                                                                                 │
│        ⬇️ (사람 눈에는 완벽하게 똑같은 귀여운 판다로 보임)                      │
│                                                                                 │
│ 🐒 [ 조작된 합성 사진 ] ───(AI 신경망 입력)───▶ 🤯 [ AI의 미친 판정 결과 ]      │
│  "긴팔원숭이 (99.3% 확신!!)" (AI 뇌정지)                                        │
│                                                                                 │
│ ★ 방어법: AI에게 훈련시킬 때 저 노이즈가 섞인 🐒 사진을 보여주면서              │
│         "이것도 판다니까 쫄지 마!" 라고 예방주사(Adversarial Training)를 놔라!  │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### Ⅲ. 실무 꿀팁: 모델 추출(Model Extraction) 방어

또 다른 무서운 공격은 "훔쳐보기"다. 
우리 회사가 100억 원을 들여서 훈련한 AI 모델을 클라우드 API로 서비스했다.
경쟁사 해커가 우리 API에 100만 번의 질문을 던지고, AI가 뱉어내는 대답 패턴을 엑셀로 다 저장한다. 
그리고 그 대답(Output)들만 긁어모아서 자기 노트북의 빈 AI 모델에게 "이 대답들을 똑같이 흉내 내 봐!"라고 훈련시킨다.
결과적으로, 경쟁사는 우리 회사의 100억짜리 AI 두뇌를 단돈 10만 원어치 API 호출 비용만 내고 똑같이(복제) 훔쳐 가 버린다! (모델 추출 공격)

* **방어막 (Rate Limiting + Confidence Hiding)**: 
  1. API 호출 횟수(Rate Limit)를 엄격히 제한해서 해커가 100만 번 질문하지 못하게 막는다 (511번 문서).
  2. AI가 정답을 뱉을 때 "99.98% 확신합니다"라는 소수점 디테일을 절대 화면에 뿌리지 마라! 그냥 "판다입니다"라고만 대답해야 해커가 모델의 정교한 수학 공식을 역추적(Reverse Engineering)하지 못한다.

### Ⅳ. 결론

"인공지능은 규칙을 배우는 것이 아니라 패턴을 외우는 짐승이다."
인공지능 보안(AI Security)은 기존 소프트웨어 공학의 '취약한 코드(Bug)'가 아니라, 블랙박스 모델의 '수학적 맹점(Mathematical Blind spot)'을 파고드는 철학적 전쟁이다. 아무리 방화벽을 높게 치고 데이터를 암호화해도, AI 모델 입으로 들어가는 데이터(Data Pipeline)와 눈으로 보는 픽셀(Inference Input)에 독과 노이즈가 섞여 있다면 AI는 가장 멍청하고 위험한 흉기로 돌변한다. 미래의 아키텍트는 훌륭한 AI 모델을 굽는 능력을 넘어, 적대적 예제를 섞어 예방주사를 놓고(Adversarial Training), 엉뚱한 데이터를 살균하며(Data Sanitization), 추론 확률을 숨기는(Output Hiding) '강인한(Robust) 방어 뼈대'를 AI 주변에 둘러치는 면역학자가 되어야 한다.

### 📌 관련 개념 맵
* **전제 기술**: 머신러닝/딥러닝 데이터 파이프라인 (ETL)
* **보호/연계 기술**: API Rate Limiting (511번 문서 - 모델 추출 공격 방어)
* **공격 대상**: Training Data (포이즈닝), Inference Input (적대적 예제), Model parameters (모델 추출)
* **주요 타겟**: 자율주행 카메라 식별, 스팸/사기(FDS) 탐지 AI, 얼굴/음성 인식 시스템

### 👶 어린이를 위한 3줄 비유 설명
1. 아주 똑똑한 앵무새(AI)에게 말을 가르치고 있어요. 나쁜 친구가 밤마다 몰래 와서 앵무새에게 욕설만 반복해서 들려줬더니(데이터 포이즈닝), 다음 날 아침 앵무새가 욕쟁이가 돼버렸어요!
2. 앵무새가 어른이 되었어요. "사과"라고 쓰인 카드를 보여주면 "사과!"라고 잘 읽죠. 근데 나쁜 친구가 사과 글자 위에 사람 눈엔 안 보이는 투명한 점 하나(적대적 예제)를 찍어놨어요.
3. 앵무새는 그 미세한 점을 보고 갑자기 뇌가 꼬여서 "바나나!!"라고 외칩니다. 이처럼 인공지능의 눈과 귀를 교묘하게 속여서 바보로 만드는 괴롭힘이 바로 **인공지능 해킹**이랍니다!
