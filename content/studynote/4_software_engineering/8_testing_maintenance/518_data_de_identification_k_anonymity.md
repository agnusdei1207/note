# 518. 비식별화 기술 (K-익명성, L-다양성, T-근접성)

> ⚠️ 이 문서는 빅데이터나 인공지능(AI)을 훈련시키기 위해 엄청나게 많은 고객 데이터가 필요하지만, 생얼(원본) 그대로의 데이터를 쓰면 개인정보보호법(징역/벌금)에 걸리는 딜레마를 타파하기 위해, **데이터의 쓸모(통계적 가치)는 살려두면서도 "이 데이터가 홍길동의 것인지 절대 역추적할 수 없게" 치밀하게 뭉개고 암호화하는 '가명 처리 및 비식별화 3대 수학적 모델'**을 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **K-익명성 (K-Anonymity)**: "숨고 싶으면 군중 속으로 숨어라!" 특정 사람의 데이터를 공개할 때, 나와 똑같은 조건(나이, 지역, 성별)을 가진 사람이 최소 K명 이상 무더기로 존재하게 만들어서 "이 데이터가 누구 건지" 콕 집어낼 수 없게 뭉개버리는(범위화) 기초 기술이다.
> 2. **L-다양성 (L-Diversity)**: K-익명성으로 3명을 묶어놨더니, 맙소사 3명이 전부 '암 환자'다. 누가 누군진 몰라도 셋 다 암 환자라는 치명적 사실이 뽀록나는(동질성 공격) 약점을 막기 위해, 묶여있는 사람들의 민감한 질병 정보(종류)가 최소 L개 이상 섞여 다양해야 한다고 강제하는 방어 기술이다.
> 3. **T-근접성 (T-Closeness)**: L-다양성으로 [암, 감기, 위염]으로 섞어놨어도 결국 '전부 아프네?'라는 유추가 가능한 약점(쏠림 현상)을 막기 위해, 묶여있는 사람들의 병 분포(비율)가 전체 국민의 병 비율(분포도)과 T만큼 비슷해야 한다고 깐깐하게 맞추는 궁극의 방어막이다.

---

### Ⅰ. 개요: 지우면 쓰레기, 안 지우면 범죄자 (Context & Necessity)
병원에서 AI에게 "어떤 사람이 암에 잘 걸리나"를 학습시키려 한다.
`홍길동, 32세, 서울 역삼동, 위암` 이라는 원본 DB가 있다.

* **보안팀**: "미쳤습니까? 개인정보보호법 위반입니다! 이름이랑 주소 다 지우세요!"
* **데이터팀**: "이름(홍길동)을 지우는 것까진 오케이. 그런데 '32세, 역삼동'까지 싹 다 별표(`***`)로 지워버리면, AI가 지역별/나이별 암 발생률을 어떻게 학습합니까? 다 지울 거면 그냥 데이터 갖다 버리세요!"

이 '개인정보 보호'와 '빅데이터 활용' 사이의 피 튀기는 싸움(Trade-off)에서 탄생한 타협점이 바로 **비식별화(De-identification)** 기술이다.
"이름은 지우고(삭제), 나이 32세는 `30대`로 뭉뚱그리고(범위화), 역삼동은 `서울시`로 범위를 넓히자."
이런 식으로 수학적 조작을 가해 원래 누구인지 못 찾게 만드는 규칙들을 **K, L, T 모델**이라고 부른다.

> 📢 **섹션 요약 비유**: 모자이크 처리와 같습니다. 범죄자의 얼굴을 100% 까만색 펜으로 칠해버리면(완전 삭제) 키나 성별조차 알 수 없습니다. 비식별화는 블러(모자이크)를 살짝 줘서 "키 큰 남자"라는 통계적 정보(데이터 가치)는 남기면서도, 그게 정확히 '누구'인지는 절대 못 알아보게(프라이버시) 얼굴을 뭉개는 예술적인 편집 작업입니다.

### Ⅱ. 비식별화의 3대 수학적 모델 (면접 필수 암기)

이 3가지는 꼬리에 꼬리를 무는 약점 극복의 진화 과정을 거친다.

#### 1. K-익명성 (K-Anonymity) : 군중 속에 숨기
- **원리**: 주어진 데이터 집합에서 같은 속성(나이, 지역, 성별)을 가진 레코드가 무조건 **최소 K개 이상** 존재하도록 데이터를 뭉개는(범위화/Generalization) 기술.
- **예시 (K=3 적용)**:
  `[서울 역삼동, 32세, 남자]` $\rightarrow$ `[서울시, 30대, 남자]` 로 값을 뭉뚱그린다. 이렇게 뭉뚱그린 데이터와 똑같은 데이터를 가진 사람이 표 안에 무조건 3명 이상이 되게 만든다. 해커가 이 표를 봐도 "서울 30대 남자 3명 중에 누가 홍길동이지?" 하고 헷갈리게(특정 불가) 만든다.
- **치명적 약점 (동질성 공격)**: K=3으로 3명을 묶었는데, 하필 그 3명의 병명이 전부 `[에이즈]`였다. 해커는 "홍길동이 이 3명 중 누군진 모르겠지만, 어쨌든 셋 다 에이즈네? 그럼 홍길동도 100% 에이즈 당첨!"이라며 민감 정보를 털어버린다.

#### 2. L-다양성 (L-Diversity) : 섞어서 헷갈리게 하기
- **원리**: K-익명성의 약점을 보완. 묶인 K명의 그룹 안에 있는 **'민감 정보(질병, 연봉 등)'의 종류가 무조건 최소 L개 이상 서로 다르게 섞여 있도록** 그룹을 강제로 찢어놓는 기술.
- **예시 (L=3 적용)**:
  아까 그 3명을 묶을 때, 병명이 `[에이즈, 감기, 위염]` 이렇게 3개(L) 이상 섞이도록 그룹핑을 한다. 
  해커가 "홍길동이 이 3명 중 하나네?"까지는 알아내도, "홍길동이 에이즈인지, 감기인지, 위염인지"는 절대 특정할 수가 없다.
- **치명적 약점 (쏠림 공격/유사성 공격)**: 병을 다르게 섞어놓긴 했는데 하필 `[위암, 폐암, 간암]`으로 묶어놨다. 해커는 "무슨 암인진 몰라도 어쨌든 암 환자네?"라고 뭉뚱그려 유추해 낸다.

#### 3. T-근접성 (T-Closeness) : 궁극의 밸런스 패치
- **원리**: L-다양성의 약점마저 보완하는 끝판왕. 특정 그룹 안에 묶여있는 민감 정보의 분포(비율)가, 아예 **전체 데이터(전 국민)의 비율 분포와 T수치 이내로 비슷하게 유지되도록** 조작하는 끔찍하게 엄격한 기술.
- **효과**: 그룹 안에 암 환자가 몰려있는 것을 수학적으로 금지하고, 전국 평균 비율처럼 [암 10%, 감기 50%, 콧물 40%] 비율을 유지하며 묶이게 만든다. 해커는 표를 백날 들여다봐도 일반 국민 평균 통계밖에 못 보므로, 개인의 특성을 완벽히 숨길 수 있다.

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│           K-익명성과 동질성 공격(약점) 붕괴 시나리오 시각화                  │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│ 🛡️ [ K=3 으로 묶어놓은 병원 데이터 ]                                         │
│   이름(삭제됨) │  지역 (범위화) │ 나이 (범위화)  │ 병명 (민감정보)           │
│   ──────────┼──────────────┼──────────────┼──────────────                    │
│     ***     │  서울 강남구   │  30대        │  위암 💥                       │
│     ***     │  서울 강남구   │  30대        │  위암 💥                       │
│     ***     │  서울 강남구   │  30대        │  위암 💥                       │
│                                                                              │
│ 😈 [ 해커 (동질성 공격) ]                                                    │
│   "나는 홍길동이 32살이고 강남구에 사는 걸 이미 알고 있지. 표를 볼까?        │
│   강남구 30대는 저 3명 묶음 중 한 명이군 (누군진 모름 K-익명성 방어 성공).   │
│   어? 근데 3명이 전부 다 위암이네? 그럼 홍길동 넌 무조건 위암이구나!" 뚫림!  │
│                                                                              │
│ 🛠️ 해결책: 병명을 [위암, 감기, 비염]으로 다양하게(L-다양성) 섞어서 묶어라!   │
└──────────────────────────────────────────────────────────────────────────────┘
```

### Ⅲ. 실무 활용 (데이터 3법과 가명 정보)

대한민국 '데이터 3법'이 개정되면서, 이 비식별화 기술은 법적인 요건이 되었다.
- **가명 정보**: 추가 정보(마스터키)가 없으면 누구인지 알아볼 수 없게 뭉갠 정보. (예: 홍길동 $\rightarrow$ 홍*동). 이 정보는 **정보 주체의 동의 없이도 AI 학습이나 통계 작성**에 막 가져다 쓸 수 있는 합법적 특권을 얻는다.
- 데이터 엔지니어들은 원본 DB를 AI 서버로 넘길 때, 중간에 `파이썬 비식별화 파이프라인`을 달아놓고 K-익명성 수치를 K=5 정도로 팍팍 뭉개버린 다음에야 안심하고 AI 팀에 데이터를 넘겨준다.

### Ⅳ. 결론

"완벽한 익명화는 완벽한 쓸모없음을 의미한다. 우리는 그 사이의 외줄 타기를 한다."
비식별화 기술(De-identification)은 프라이버시(Privacy)와 데이터 유용성(Utility)이라는 영원히 양립할 수 없는 두 가치를 하나의 아키텍처에 욱여넣으려는 인류의 수학적 몸부림이다. 데이터를 뭉개면 뭉갤수록 해커로부터 안전해지지만(K, L, T 상승), AI가 패턴을 학습할 통계적 뾰족함은 둔탁해져 결국 쓰레기 데이터가 되고 만다. 이 피 말리는 제로섬 게임 속에서, 비즈니스 가치를 창출하면서도 개인정보보호법이라는 감옥에 가지 않을 최적의 K값을 찾아내 파이프라인(ETL)에 녹여 넣는 데이터 엔지니어의 곡예야말로 4차 산업혁명의 가장 위태로운 윤리적 최전선이다.

### 📌 관련 개념 맵
* **전제되는 사상**: 개인정보 보호 중심 설계 (Privacy by Design - 516번 문서)
* **법적 근거**: GDPR, 대한민국 데이터 3법 (가명 정보 활용 조항)
* **핵심 처리 기법**: 마스킹(Masking, `***`), 범위화(범주화, `30대`), 삭제(Deletion)
* **데이터 생명주기**: ETL(추출, 변환, 적재) 파이프라인 중간에서 비식별화 수행

### 👶 어린이를 위한 3줄 비유 설명
1. **K-익명성**: 나 혼자 빨간 옷을 입으면 튀어서 잡혀요. 그래서 나와 똑같은 빨간 옷을 입은 사람 최소 K명(예: 3명)을 내 주변에 우르르 세워놔서 "누가 나인지" 헷갈리게 숨는 거예요!
2. **L-다양성**: 빨간 옷 3명이 모였는데 하필 3명 다 콧물을 흘리고 있어요! 그럼 누가 누군진 몰라도 "저 그룹은 다 감기 걸렸네" 하고 털리죠. 그래서 콧물, 배탈, 기침 등 아픈 증상(L)을 여러 개 다양하게 섞어서 묶는 거랍니다.
3. **T-근접성**: 콧물, 배탈을 섞어 놨어도 결국 "다 아픈 애들이네" 하고 걸려요. 그래서 우리나라 전체의 아픈 사람, 건강한 사람 비율표(T)랑 똑같이 맞춰서 그룹을 묶어버리는 가장 깐깐하고 완벽한 변장술이랍니다!
