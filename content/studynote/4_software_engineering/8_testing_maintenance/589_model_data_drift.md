# 589. 모델 드리프트 (Model/Data Drift) 모니터링

> ⚠️ 이 문서는 한 번 똑똑하게 학습된 AI 모델이라도 시간이 지나면 현실 세계의 변화를 따라가지 못하고 바보가 되어버리는 현상인 **'드리프트(Drift)'의 종류와, 이를 조기에 감지하여 스스로 재학습(Retraining)하게 만드는 MLOps의 핵심 방어 기제**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: "세상은 변하는데 AI는 과거에 머물러 있다"는 것이 드리프트의 핵심이다. 들어오는 데이터의 성격이 변하면(Data Drift), AI의 정답률도 떨어지게 된다(Model Drift).
> 2. **가치**: 이 현상을 방치하면 추천 시스템이 엉뚱한 상품을 추천하여 회사 매출이 폭락한다. 24시간 드리프트를 모니터링하면 모델이 썩어버리기 전에 골든타임을 확보할 수 있다.
> 3. **기술 체계**: 과거 학습 데이터의 통계적 분포와 현재 실시간으로 들어오는 데이터의 통계적 분포를 비교(K-S 검정 등)하여, 오차가 임계치를 넘으면 MLOps 파이프라인에 **재학습(Retrain) 트리거**를 쏜다.

---

### Ⅰ. 개요: 썩어가는 AI (Context & Necessity)
2019년에 완벽한 '마스크 수요 예측 AI'를 만들었다.
하지만 2020년 코로나19가 터지자, 이 AI는 완전히 바보가 되었다. 과거의 데이터(코로나 이전) 패턴으로는 폭증하는 현재의 수요를 예측할 수 없었기 때문이다.

이처럼 **소프트웨어의 코드(Code)는 썩지 않지만, AI 모델(Model)은 시간이 지나면 반드시 썩는다.**
이를 방지하기 위해 데이터 과학자들은 AI가 뱉어내는 결괏값과 현실 세계의 정답을 끊임없이 비교하며, AI가 치매에 걸렸는지 실시간으로 모니터링해야 한다.

> 📢 **섹션 요약 비유**: 모델 드리프트는 **'유행이 지난 패션 잡지'**와 같습니다. 작년 잡지(과거 데이터)를 아무리 열심히 공부한 패션 테러리스트(AI)라도, 올해 길거리(현재 데이터)에 나가면 사람들에게 촌스럽다고 비웃음을 당하게 됩니다.

---

### Ⅱ. 드리프트의 2가지 종류 ★

#### 1. 데이터 드리프트 (Data Drift)
- **정의**: 모델에 입력되는 X값(Input)의 성격 자체가 변하는 현상이다.
- **예시**: 사과만 인식하던 과일 분류 AI에게, 어느 날 갑자기 사용자들이 '샤인머스캣' 사진을 올리기 시작했다. AI는 한 번도 본 적 없는 데이터를 강제로 사과라고 분류하며 에러를 낸다.

#### 2. 컨셉 드리프트 (Concept Drift / Model Drift)
- **정의**: 세상의 규칙(X와 Y의 관계)이 변해버리는 현상이다. 입력 데이터는 똑같은데, 정답이 바뀌었다.
- **예시**: 과거에는 '재택근무'라는 단어가 긍정적인 메일이었지만, 코로나 이후에는 '재택근무'가 평범한 일상이 되어 중립적인 메일로 변했다. AI는 계속 긍정이라고 우긴다.

---

### Ⅲ. 방어 전략: 재학습 루프 (Retraining Loop) 설계

드리프트를 감지하면 AI를 병원에 보내야 한다.

1. **지속적 모니터링**: 
   - 현재 유저들이 입력하는 데이터의 통계(평균, 분산)와 과거 학습 데이터의 통계를 실시간으로 비교한다.
2. **트리거(Trigger) 발생**: 
   - 두 데이터의 분포 차이가 10% 이상 벌어지면 알람(Alert)을 울린다.
3. **자동 재학습 (Automated Retraining)**: 
   - 알람이 울리면 MLOps(588번 문서) 파이프라인이 즉시 가동된다.
   - 최근 3개월 치의 싱싱한 데이터를 새로 긁어와서 모델을 자동으로 깎아낸다.
4. **섀도우 배포 (Shadow Deployment)**: 
   - 새로 만든 모델이 멍청할 수도 있으니, 575번 문서에서 배운 섀도우 배포로 운영 트래픽을 슬쩍 흘려보내 성능을 검증한 뒤 기존 모델과 교체한다.

```text
┌──────────────────────────────────────────────────────────────┐
│           데이터 드리프트 감지 및 자동 재학습 파이프라인 시각화           │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ [ 📊 실시간 입력 데이터 ] ──┐                                   │
│                        ▼                                     │
│                 ┌───────────────┐ ──(경고! 분포가 달라짐!)──┐    │
│                 │ 🔍 드리프트 감지 │                         │    │
│                 └───────────────┘                         ▼    │
│                        ▲                           ┌───────────┐│
│ [ 📚 과거 학습 데이터 ] ───┘                           │ ⚙️ 재학습 ││
│                                                    │ 파이프라인 ││
│                                                    └─────┬─────┘│
│                                                          │      │
│ [ 🌐 프로덕션 환경 ] ◀──────(새로운 모델(v2.0) 자동 배포)──────┘      │
└──────────────────────────────────────────────────────────────┘
```

### Ⅳ. 결론

"AI 모델의 배포는 끝이 아니라 유지보수의 시작이다."
전통적인 소프트웨어는 배포하고 나면 개발자가 쉴 수 있었지만, AI 모델은 배포되는 순간부터 서서히 성능이 떨어지기 시작한다. 모델 드리프트 모니터링은 이 하락 곡선을 브레이크 잡고 다시 우상향으로 끌어올리는 유일한 방법이다. 아무리 똑똑한 알고리즘으로 만든 AI라도, 변화하는 세상의 데이터를 주기적으로 먹여주지 않으면 결국 쓰레기(Garbage)를 뱉어내는 고철 덩어리가 될 뿐이다.

### 👶 어린이를 위한 3줄 비유 설명
1. 모델 드리프트는 작년 수학 시험 족보만 달달 외운 학생(AI)이 올해 시험을 치는 것과 같아요.
2. 선생님이 올해부터 문제 스타일을 완전히 바꿨다면(데이터 드리프트), 이 학생은 시험을 망치게 되겠죠?
3. 그래서 우리는 학생이 매일매일 '최신 모의고사'를 풀어보게 만들어서(재학습), 선생님의 출제 스타일 변화를 바로바로 따라가게 도와줘야 한답니다!
