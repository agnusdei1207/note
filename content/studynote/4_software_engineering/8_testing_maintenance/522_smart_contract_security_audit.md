# 522. 블록체인/스마트 컨트랙트 (Smart Contract) 보안 감사

> ⚠️ 이 문서는 한 번 네트워크(블록체인)에 배포되면 지구상 그 누구도 코드를 수정(Patch)하거나 삭제할 수 없는 불변의 세계에서, **단 한 줄의 코딩 실수(예: 재진입 공격)가 발견되는 순간 수백억 원의 고객 돈이 해커의 지갑으로 1초 만에 증발해 버리는 끔찍한 재앙을 막기 위해, 배포 전 목숨을 걸고 수행해야 하는 '스마트 컨트랙트 보안 감사(Audit)'**를 다룹니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스마트 컨트랙트(Smart Contract)는 이더리움 블록체인 위에서 자동으로 돈(코인)을 주고받게 짜놓은 '자판기 코드(Solidity)'다. 이 코드는 한 번 블록체인에 올라가면 **수정(Update)이 100% 불가능**하므로, 배포 전 단 1개의 버그도 남기지 않는 극한의 정적/동적 보안 검수(Audit)가 필요하다.
> 2. **가치 (가장 치명적인 버그)**: DAO 사태(600억 원 탈취)로 유명한 **재진입 공격(Reentrancy Attack)**이 끝판왕이다. 출금 코드가 실행되는 그 찰나의 순간에 해커가 미친 듯이 출금 함수를 1,000번 반복 호출하여, 내 잔고가 깎이기 전에 금고의 돈을 다 털어가는 끔찍한 논리 오류다.
> 3. **기술 체계**: 이를 막으려면 무조건 돈을 먼저 보내지 말고 **잔고를 먼저 깎은 뒤에 돈을 보내는(CEI 패턴: Checks-Effects-Interactions)** 코딩 철칙을 지켜야 하며, 배포 전에 외부 전문가를 불러 수천만 원을 주고 코드 감사(Audit)를 받아야 한다.

---

### Ⅰ. 개요: 패치가 불가능한 무결성의 저주 (Context & Necessity)
일반적인 웹 개발자(Web2)가 쇼핑몰을 만들다 결제 버그를 냈다.
"앗! 돈이 복사되잖아! 당장 서버 내리고 코드 수정(Patch)해서 10분 만에 재배포할게요!" (해결 완료)

하지만 블록체인 개발자(Web3)가 스마트 컨트랙트로 이더리움에 '은행 자판기' 코드를 올렸다.
오픈 첫날 버그가 터졌다. 해커가 100억 원의 이더리움을 복사해서 자기 지갑으로 빼가고 있다.
개발자: "앗! 당장 서버 끄고 코드 뜯어고쳐!!"
블록체인: **"어? 나는 블록체인이라서 서버의 전원 버튼도 없고, 한 번 올라간 코드는 절대 삭제하거나 수정(Patch)할 수 없는데? 영원히 그 코드로 돌아갈 거야 ㅋㅋㅋ"**
개발자: (100억이 눈앞에서 0원 될 때까지 멍하니 구경하며 오열함)

이것이 스마트 컨트랙트 개발의 무서움이다. **무결성(Immutability)**이라는 블록체인의 장점이, 버그가 났을 때는 **패치 불가**라는 끔찍한 사형 선고가 된다. 그래서 블록체인 개발에서는 코드를 메인넷에 올리기 전(Shift-Left), 돋보기를 들고 한 줄 한 줄 악마처럼 검증하는 **보안 감사(Audit)**가 프로젝트 성공의 전부라 해도 과언이 아니다.

> 📢 **섹션 요약 비유**: 일반 프로그램은 나사로 조립한 레고(Web2)라서 잘못 만들면 나사를 풀고 다시 조립하면 됩니다. 하지만 스마트 컨트랙트는 한 번 모양을 잡고 쇳물을 부어 단단하게 굳혀버린 강철 검(Web3)입니다. 굳은 뒤에 흠집을 발견해도 깎아낼 수가 없으므로, 쇳물을 붓기 전 설계도 단계에서 완벽하게 모양을 검사(보안 Audit)해야만 합니다.

### Ⅱ. 최악의 폭탄: 재진입 공격 (Reentrancy Attack) ★

면접에서 "스마트 컨트랙트 보안"을 물으면 0순위로 튀어나와야 하는 단어다. 2016년 이더리움 생태계를 쪼개버린 'The DAO' 해킹 사건의 주범이다.

#### [ 바보 같은 출금 코드 (취약점) ]
```solidity
function withdraw() public {
    // 1. 유저 잔고 확인
    uint bal = balances[msg.sender];
    require(bal > 0);

    // 2. 돈을 보냄 (★ 여기서 지옥이 열린다!)
    (bool sent, ) = msg.sender.call{value: bal}(""); 
    
    // 3. 다 보냈으니 유저 잔고를 0으로 깎음
    balances[msg.sender] = 0; 
}
```
* **문제점**: 2번 줄에서 해커 지갑으로 돈이 들어가는 순간, 해커의 지갑에 숨겨진 악성 코드(Fallback 함수)가 자동으로 실행된다. 이 악성 코드가 3번 줄(잔고 깎기)이 실행되기도 전에, 다시 1번 줄(`withdraw`)을 빛의 속도로 1,000번 호출(재진입, Reentrancy)해 버린다!
* **결과**: 내 잔고는 아직 안 깎였으므로 계속 통과된다. 해커는 1만 원을 넣고, 은행 금고에 있는 1,000만 원이 바닥날 때까지 돈을 무한 복사해서 뽑아간다.

#### [ CEI 패턴 (Checks-Effects-Interactions) 방어법 ]
단순하다. 순서를 바꾸면 된다. **돈부터 주지 말고, 장부(잔고)부터 깎아라!**
```solidity
function withdrawSafe() public {
    uint bal = balances[msg.sender];
    require(bal > 0);                  // 1. 확인 (Checks)

    balances[msg.sender] = 0;          // 2. 내 잔고 0원으로 깎아버림 (Effects) ★

    (bool sent, ) = msg.sender.call{value: bal}(""); // 3. 마지막으로 돈을 줌 (Interactions)
}
```
* **결과**: 돈을 받는 순간 해커가 다시 1번으로 재진입해 봤자, 이미 2번에서 내 잔고를 0원으로 깎아버렸기 때문에 1번 조건문(`bal > 0`)에 걸려서 튕겨 나간다. (완벽 방어)

### Ⅲ. 스마트 컨트랙트의 다른 흔한 취약점들

재진입 말고도, 블록체인이라는 특수한 환경 때문에 생기는 독특한 버그들이 있다.

#### 1. 프론트 러닝 (Front-Running / 샌드위치 공격)
- 내가 주식(토큰)을 사려고 100만 원어치 매수 버튼을 누른다. 이 주문은 잠시 대기열(Mempool)에 머문다.
- 해커가 대기열을 훔쳐보고 "오, 쟤가 크게 사서 가격 오르겠네?" 하고, 나보다 비싼 수수료(Gas Fee)를 내서 나보다 0.1초 먼저 새치기해서 똑같은 주식을 사버린다! 그리고 가격이 오르면 나에게 비싸게 팔고 튄다. (트랜잭션 순서 조작 공격)

#### 2. 오버플로우 / 언더플로우 (Integer Overflow/Underflow)
- `uint8`(최대 255) 변수에 잔고가 0원인데, 1원을 빼면 `-1원`이 아니라 한 바퀴를 빙 돌아 `255원`의 부자가 되어버리는 버그다. (이더리움 최신 버전인 Solidity 0.8부터는 언어 차원에서 에러를 내며 자동으로 막아준다.)

#### 3. 난수 생성 불가 (Bad Randomness)
- 가위바위보 게임을 짜려고 `랜덤숫자 = block.timestamp(현재 시간)`을 썼다. 
- 해커는 채굴자(Miner)다. 자기가 블록을 만들 때 채굴 시간을 맘대로 조작해서 무조건 가위가 나오게 조작해 버린다! 블록체인 안에서는 절대 내부 변수를 써서 랜더(Random)를 만들면 안 되고, 외부 세계의 신뢰할 수 있는 데이터(Chainlink Oracle 등)를 가져와야 한다.

### Ⅳ. 결론

"코드가 곧 법(Code is Law)이라면, 버그는 곧 재앙이다."
스마트 컨트랙트(Smart Contract)는 중앙 서버의 신뢰를 블록체인 네트워크의 수학적 무결성으로 치환한 위대한 발명이다. 하지만 '수정 불가능(Immutable)'이라는 이 오만한 아키텍처는 인간이 짜는 코드의 불완전성을 전혀 고려하지 않았다. 일반 소프트웨어가 런타임 환경에서 어떻게든 버그를 수습하며 진화(Agile)한다면, 스마트 컨트랙트는 런칭 버튼을 누르기 전 수십 번의 정적 분석(Slither), 동적 퍼징(Echidna), 그리고 전문가의 뼈를 깎는 수동 코드 리뷰(Audit)를 통과해야만 생존할 수 있는 가장 가혹한 워터폴(Waterfall)적 징벌의 세계다. 여기서 CEI 패턴(상태 변경 후 통신)은 단순한 코딩 스타일을 넘어, 돈이 오가는 이 신뢰 없는 세계에서 자산을 지키는 최후의 철갑옷이다.

1. 내가 로봇 은행원한테 "내 통장에 1만 원 있으니까 돈 줘!"라고 했어요.
2. 멍청한 로봇 은행원은 1만 원 지폐를 내 손에 쥐여주고 나서, 장부에서 내 잔고를 0원으로 고치려고 몸을 돌립니다. (취약한 코드)
3. 근데 내가 로봇보다 훨씬 빨라서, 로봇이 장부를 고치기 1초 전에 다시 "나 아직 장부에 1만 원 남았으니 돈 또 줘!"라고 1,000번 소리쳤어요. 로봇은 헷갈려서 은행 금고 돈이 거덜 날 때까지 1만 원씩 1,000번을 뱉어냅니다! 이게 무시무시한 **재진입 공격**이랍니다!
