# 로직 밤 (Logic Bomb) / 타이머 밤 (Time Bomb)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 로직 밤은 인가를 받은 정상 소프트웨어 내부에 심어진 악의적 코드 조각으로, 특정 조건(날짜, 시간, 특정 파일 존재, 특정 사용자 로그인 등)이 충족되면 백도어를 열거나 데이터를 파괴하는 지연 실행형 악성코드다.
> 2. **가치**: 로직 밤은 외부 방화벽이나 안티바이러스 같은 경계 방어 체계를 우회하는 것이 아니라, 이미 내부에 침투하여 인가된 권한을 보유한 상태에서 "정당한 업무 소프트웨어인 척" 행동하는 것이므로 탐지 난이도가 극도로 높다.
> 3. **융합**: 로직 밤은 소프트웨어 개발 라이프사이클에서 내부자 위협이나 서드파티 공급망 공격과 결합되어, 개발자나 유지보수 업체가 배포한 정상 업데이트에 몰래 삽입되는 공급망 공격의 핵심 수단으로 활용된다.

---

## 1. 개요 및 필요성

### 개념 및 정의
로직 밤은 컴퓨터 보안 분야에서 악성 소프트웨어의 한 종류로, 기본이 되는 호스트 프로그램에 내장되어 특정 조건이 달성될 때까지待机 상태를 유지하는 악성 코드 모듈이다. 영어로는 Logic Bomb, 한국어로는 "로직 밤" 또는 "논리 폭탄"이라고도 부른다. 이 조건은 다양한 트리거가 될 수 있는데, 가장 흔한 것에는 특정 날짜나 시간(타임 bomb), 특정 파일의 존재/부재, 특정 사용자의 로그인, 특정 명령어 실행, 또는 프로그램 내 특정 횟수 실행 등이 있다. 트리거 조건이 충족되면 즉시 악성 payload가 실행되어 데이터를 삭제하거나, 네트워크 외부로 정보를 외부 유출하거나, 또는 시스템에 백도어를 열어두는 등의 second strike를 실행한다.

**필요성 및 등장 배경**
1980년대 초반, 컴퓨터 프로그래머들이 고용주와의 분쟁이 발생할 경우에 대한 보복 수단으로 로직 밤을 사용하는 사례가 처음 reported 되었다. 특히 금융권, 군사시설, 중요 인프라에서는 시스템 관리자나 감리 담당자가 은밀히仕組む 로직 밤이 사고 직후에 발견되기 어렵고, 증거도湮滅하기 쉬워 법적 증거 확보가 극히 어려운 점에서 고의적 파괴 도구로 악용되어 왔다. 대표적인 사례로 1988년 Marcus J. Ranum이라는 연구자가 설계한 "Trusted Systems" 모델에서 외부 침입보다 내부자의 의도적 saboteur 코드에 대한 방어 없이는 아무리 외부 방어를 튼튼히 해도 소용이 없다는 점을 처음으로系统적으로 논증했다.

```
[로직 밤의 공격 경로 vs 기존 보안 체계의 탐지 공백]

[기존 Perimeter Security의 탐지 영역]
  방화벽 --> 네트워크 침입 탐지 (IDS)
       |
       +--> 외부 해킹 트래픽/패턴 차단
                    |
         Perimeter Security는 여기서 끝!
                    |
                    v
[내부: 이미 인가된 사용자의 정당한 프로세스 내부]
  [정상 업무 소프트웨어 (Word, SAP, 내부 프로그램)]
       |
       +--> 정상 기능 실행 (문서 작성, 회계 처리 등)
       |
       +--> [Embedded 로직 밤] --> 조건 충족 시 발동!
                    |
         탐지 불가! 안티바이러스도 정상 SW로 인식

핵심 문제:
- 방화벽: 네트워크 경계를 넘은 트래픽은 볼 수 없음
- 안티바이러스: 악성 시그니처 데이터베이스에 없으면 탐지 못함
- IDS/IPS: 이미 허용된 내부 프로세서의 행위 패턴은 알 수 없음
```

**[다이어그램 해설]** 이 구조는 로직 밤이 왜 "내부자 위협의 정점"으로 불리는지를 명확히 보여준다. 방화벽은 국경수의 역할을 하는데, 로직 밤은 이미 국경 안쪽(내부 네트워크, 인가된 시스템)에서 활동하는 "공인 출입증을 가진 내부자"이기 때문에 방화벽의 통찰을 받을 수가 없다. 더욱이 안티바이러스 소프트웨어도 의심의 눈으로 보는 것은 "악성 시그니처가 데이터베이스에 등록된已知 악성코드"뿐이다. 로직 밤은 특정 고객환경에만埋头 위해专门히 개발되어 시그니처가 존재하지 않는 0-day 악성코드인 경우가 대부분이므로, 전통적 안티바이러스의 시그니처 기반 탐지 방식으로는 100% 탐지되지 않는다.

- **요약 비유**: 로직 밤은 미술관 경비원이 무점검 통과증을 가진 순찰원에게만 출입을 허용하는 상황에서, 그 순찰원 costumes 안底에 단도를 숨기고 들어온刺客와 같다. 미술관 안에서는 "나 경비원이야" 하고 아무 데나 돌아다닐 수 있으며, 특정 시각이 되면 갑자기 단도를 빼들어 중요한 그림을 찌르거나 방화선을 off하는 것이다.

## 2. 아키텍처 및 핵심 원리

### 로직 밤의 내부 구조 및 구성 요소

로직 밤은 단순해 보이지만, 은폐성과 신뢰성 측면에서 매우 정교하게 설계되는 경우가 많다.

| 요소명 | 역할 | 내부 동작 | 비유 |
|---|---|---|---|
| **트리거 조건 (Trigger)** | 로직 밤이 발동할 조건을 판단하는 코드 | 시간/파일/이벤트/카운터 등을 지속적으로 Monitor |炸탄의 압력 설정 장치 |
| **비활성화 루틴 (Arm/Disarm)** | 특정 조건에서 자신을 무력화하는 코드 | 내부자 구별 코드, 특정 IP에서만 발동 등 |炸탄의 remote kill switch |
| **악意 payload** | 실제 피해를 주는 악성 코드 | 데이터 파괴, 백도어 개방, 데이터 외부 전송 |炸탄의 폭발물 |
| **은폐 메커니즘 (Obfuscation)** | 코드 분석을困難하게 하는 기술 | 암호화, 난독화, 코드 은밀 삽입 |炸탄의 위장 코팅 |
| **반감기 (Self-Destruct)** | 증거를 인멸하는 코드 | 로그 삭제, 파일 복제 교체 |炸탄의 미립자 살포 장치 |

### 로직 밤의 실행 메커니즘: 타이머 기반 트리거 분석

타임 bomb은 로직 밤의 가장 흔한 형태로,特定 시각이나日期에 맞춰 자동으로 발동한다.

```
[타임 bomb 로직 밤의 내부 발동 메커니즘]

[호스트 프로세스 (정상 업무 소프트웨어) 내부]

① 트리거 체크 루틴 (매 실행 시 또는 타이머 인터럽트 시)
   if (current_date >= target_date &&
       current_time >= target_time) {
       goto PAYLOAD;
   }

   [아래는 정상 업무 코드...]
   document.save();
   calculate_financial_report();

② PAYLOAD 실행 (트리거 조건 충족 시)
   - OS 명령 실행 (system("rm -rf /critical/data"));
   - 파일 시스템의 핵심 데이터抹消
   - 또는 백도어 포트 개방 (nc -l -p 31337)

③ 로그 삭제 (자기 자신을 위한 로그 인멸)
   - audit logs에서 해당 프로세스 흔적 제거

[핵심 특성]
- PAYLOAD 발동 전까지는 정상 SW와 100% 동일한 동작
- 메모리 내에서만 payload가 존재, 디스크에는 정상 데이터로 위장
```

**[다이어그램 해설]** 타임 bomb의恐怖는 "发砲 전까지의 완전한 무죄"에 있다. 로직 밤은 발동 조건이 충족되기 전까지는 일반적인 업무 소프트웨어와 완벽하게 동일한 메모리 패턴, CPU 사용량, 네트워크 연결을 보인다. 더욱이 타임 bomb의 경우 midnight이나 주말, 연초처럼 관리자가比较少 확인하는 시간대에 맞춰仕組まれる 경우가 많아, 발동初期 단계의 이상 징후를 발견할機会을逸하기 쉽다.

### 로직 밤의 종류별 분류

| 유형 | 트리거 조건 | 실제 사례 |
|---|---|---|
| **타임 bomb** | 특정 날짜/시간 도달 | 퇴사일에 발동하는 Sabotage 코드 |
| **이벤트 bomb** | 특정 이벤트 발생 | "/etc/shadow 파일 접근 시" 발동 |
| **카운터 bomb** | 특정 횟수 실행 후 | SW가 1000번째 실행될 때 payload 발동 |
| **앙환 bomb** | 특정 파일/사용자 패턴 감지 | "사장 이름이 들어간 파일 열릴 때" 발동 |
| **네트워크 bomb** | 원격的命令 수신 | C2 서버로부터 "activate" 명령 수신 시 발동 |

- **요약 비유**: 로직 밤은 영화에서 범죄자가 상대방의 핸드폰에仕組한 위치 추적 미니 장치와 같다. 평소에는 아무 소리도 내지 않고そっと 보내지만, 목표 인물이 특정 장소에 도착하거나 특정 단어를 말하면 그제야 장치가 작동하여 위치를 알려주거나 백도어 개방, 심장 박동 중지 장치를 가동한다.

## 3. 융합 비교 및 다각도 분석

### 로직 밤 vs 기타 악성코드의 비교

| 비교 항목 | 로직 밤 | 바이러스 | 웜 | 백도어 |
|---|---|---|---|---|
| **자기 복제 능력** | 없음 | 있음 (호스트에寄生) | 있음 (자체 전파) | 없음 |
| **트리거 조건** | 반드시 존재 | 선택적 | 선택적 | 일반적으로 없음 |
| **주요 목적** | 파괴/데이터 유출 | 분산/감염 확대 | 시스템 장악/분산 | 지속적 접근 확보 |
| **탐지 난이도** | **极其 높음** | 보통 | 보통 | 보통 |
| **주요 감염 경로** | 내부자 삽입, 서드파티 공급망 | 파일 공유, 이메일 | 네트워크 취약점 | 주로 외부 해킹 |

로직 밤의 가장 큰 차별점은 자기 복제 능력이 없음에도 불구하고 탐지 난이도가 가장 높다는 것이다. 바이러스와 웜은 자기 복제 과정에서 필연적으로 비정상적인 파일 생성이나 네트워크 트래픽 패턴을 만들어 내지만, 로직 밤은 이미 승인되고 배치된 정상 소프트웨어 내부에서 조용히 기다리기 때문에 파일系统中에 별도의 악성 artifact를 남기지 않는다.

```
[악성코드 類別별 탐지 난이도 vs 피해 범위]

피해 범위:
  높음
   |
   |      +-----------------+
   |      |      웜         |
   |      +-----------------+
   |      |   로직 밤       |
   |      +-----------------+
   |      | 백도어 | 바이러스 |
   +------+----+--+---------+
           낮음        높음
         탐지 난이도

핵심 통찰:
로직 밤은 피해 범위는 중간~높음이지만
탐지 난이도가 가장 높아 실질적 위험이 가장 크다!
```

**[다이어그램 해설]** 웜과 바이러스는 피해 범위가 광범위하지만, 그만큼 노이즈가 많이 발생하여 IDS나 안티바이러스로 탐지될 확률이 높다. 반면 로직 밤은 노이즈가 거의 없고 "트리거 조건이 충족될 때까지安静히潜伏"하므로 탐지율이 극히 낮다. 이래서 실전 보안에서는 로직 밤 방어를 위해 서드파티 코드 감사, 행위 기반 탐지, 그리고 개발 라이프사이클 내의 코드 출처 추적과 서명이 필수적인 방어 수단으로 자리 잡고 있다.

- **요약 비유**: 로직 밤은 잠복력이极强的 적군 첩보원과 같다. 웜은大喊大叫しながら城門を突破して城を佔領하지만, 防壁가 대응하면たちまち捕まる. 그러나 첩보원은城の内部にいる中使用人身份으로潜伏し、特定の時機が到来するまですべての城門を開帳して内部から城を攻略한다. 발견难度가极高하기 때문에 가장怖い相手이다.

## 4. 실무 적용 및 기술사적 판단

### 실무 시나리오: 퇴사 예정 개발자가仕組んだ 로직 밤

**상황**:某 SI 기업에서 개발자 A씨가 사직서를 제출했다. 2주 후 퇴사했다. 1개월 후,该公司의 주요 사업용 프로그램에서 대규모 일괄 배치 작업이 실행되었고, 결과 데이터가 전부 空파일로 출력되는事故가 발생했다.

**분석**: 확인 결과,A씨가 개발에 참여했던 배치 프로그램 내에 로직 밤이仕組まれて 있었음이 파악되었다. 트리거 조건은 "배치 프로그램 실행 횟수가 30회를 넘을 것"이었고, 퇴사 전에는 테스트 iterations때문에 횟수가 아직 25회에 불과했지만, 사후 1개월간正常使用으로 인해 30회가 채워졌다. 31회차 실행부터는 계산 결과가 空로 반환되도록仕組まれた payload가 발동하여 데이터가 空파일로 처리된 것이었다.

**기술적 교훈**: 로직 밤의 트리거가 "횟수 기반"이었고, 그 횟수가 테스트 단계에서 아직 도달하지 않은 수치였기 때문에, 事前のコードレビューでも 발견되지 않았다. 또한 배치 프로그램의 단위 테스트는 항상 성공했고, Integration Test에서도 정상 동작했기에, Safe Quality Assurance 프로세스를 完全적으로 우회했다.

### 도입 체크리스트 (로직 밤 방어 체계)

- **코드 감사 및 서드파티 리뷰**: 외부에서 도입하는 소프트웨어나 OSS 라이브러리에 대해서는 반드시 이원화 코드 감리를 실시하고, 감사 담당자는 해당 코드의 개발사와 상이한 별도의 보안 전문 기관을 활용해야 한다.
- **행위 기반 침입 탐지 (EDR)**: 로직 밤은 파일系统中에 악성 artifact를 남기지 않으므로, 전통적 안티바이러스로는 탐지 불가하다. 대신 프로세스의 파일 시스템 접근 패턴, 레지스트리 수정 패턴, 네트워크 연결 패턴을 지속적으로 모니터링하는 EDR 솔루션이 필수적이다.
- **무결성 모니터링**: Tripwire 같은 파일 무결성监控系统를 통해 중요 시스템의 바이너리 및 설정 파일의 SHA-256 해시를 사전에 등록해 두고, 해당 파일이 변경되면即座에 알람을 발생시키는 체계를 운영해야 한다.

### 안티패턴

- **시그니처 기반 안티바이러스만 신뢰**: 로직 밤은 본질이 "특정 환경에서만 작동하는 Custom 악성코드"이므로 시그니처 데이터베이스에 존재할리가 없다.
- **퇴직자 코드에 대한 사후 감사 미실시**: 퇴직자의 코드는 퇴직 前至少 3개월 전부터 감사 범위에 포함시켜야 한다. 회사의 핵심 코드에 접근 가능한 직원의 퇴직 은퇴 예정은 곧바로 Insider Threat 레벨을 상향 조정해야 한다.

- **요약 비유**: 로직 밤은 말벌과 같다. 평소에는 꿀을 따러 다니는 건강한 벌로 보이다가, 특정 자극(트리거 조건)이 발생하면 그제야针을 꽂아 대거 살상 피해를 입힌다. 말벌이 꿀을 따러 올 때마다殺虫剂를 쓰는 것이 아니라, 벌이本身的 특성을 미리 파악해두는 것이 最善의 방어이다.

## 5. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 방어 체계 미도입 | 방어 체계 도입 | 기술적 함의 |
|---|---|---|---|
| **로직 밤 탐지율** | < 5% | 60~80% | 0-day 로직 밤이라도 비정상 행위 패턴으로 탐지 가능 |
| **인시던트 대응 시간** | 사고 발생 후 平均 45~90일 후 발견 | 사고 前 조기 경보 | 데이터 유출/파괴 피해 최소 70% 이상 감소 |
| **법적 증거 확보** | 로그 인멸로 대부분 불가능 | EDR telemetry + FIM 로그로 완전한 타임라인 재구성 | Forensic 분석 및 법적 소송 가능성 확보 |

### 미래 전망

로직 밤의 진화 방향은 두 가지다. 첫째, 머신러닝을 利用한 지능화다. 트리거 조건을 "비정상 행위 패턴"으로까지 확장하여, AI가 시스템 관리자의 평소 행위 패턴을 학습한 후 "이 사용자의 행동이 평소와 다르다"는抽象적 조건으로 발동하는 로직 밤이 등장할 것으로 예측된다. 둘째, 암호화폐 맥락에서의 변종이다. Cryptocurrency 채굴 봇이 감염 호스트의 CPU/GPU 자원을 利用하여蓦字을 채굴하도록仕組まれた 로직 밤은, 발동 初부터 비정상적 고CPU 사용률이라는 확실한 행위 패턴을 보여 탐지되기가 상대적으로 쉽다.

### 참고 표준

- **NIST SP 800-53 (SA-12)**: Supply Chain Threat countermeasures
- **ISO/IEC 27001**: 정보보안관리시스템 내의 내부자 위협 관리 통제 항목
- **CIS Controls v8 (Control 14)**: Security Awareness and Skills Training

- **요약 비유**: 로직 밤은 디지털 세계의 고무 Laser와 같다. 평소에는 совершенно 보이지 않고 가만히 있지만, 특정 조건이 되면 作動하여目标에 가장 큰 해를 끼친다. 防護 위해서는 레이저 탐지기(EDR)뿐 아니라, 내부자에 忍び寄る 것을事前에 감지하는 thermal camera(행위 패턴 분석)와 금고 안의 물체가 원래 있어야 할 자리에 있는지 확인하는 periodic inventory check(파일 무결성 모니터링)가 함께 필요하다.

## 관련 개념 맵

| 개념 명칭 | 관계 및 시너지 설명 |
|---|---|
| **백도어** | 로직 밤의 주요 payload 중 하나로, 특정 조건 충족 시 시스템에 지속적 인가 접속 경로를 확보하는 악성 코드이다. |
| **Insider Threat** | 로직 밤이 가장 흔하게利用되는威胁 벡터로, 조직 내부자가 권한을 利用하여仕組む 경우가 대부분이다. |
| **EDR** | 로직 밤의 비정상 행위 패턴을 탐지하는 핵심 기술이다. |
| **FIM** | 중요 시스템 파일의 SHA-256 해시를 사전 등록하고, 파일 변경 시即座에 알람을 발생시켜 로직 만에 의한 시스템 바이너리 변조企图를 포착한다. |
| **Supply Chain Attack** | 로직 밤이 가장怖い感染 경로 중 하나로, 서드파티 라이브러리나 개발도구를 통해 大衆적으로 배포되어 다수의 조직에 동시 침투하는 대규모 공격이다. |

## 어린이를 위한 3줄 비유 설명
1. 로직 밤은 비 오는 날 학교의 컴퓨터 안에 숨어 있는定时폭탄과 같아요. 평소에는 컴퓨터가 딱히 아프거나 이상한 소리를 내지 않고 정상적으로 작동하지만, 학기말 시험 치는 날(특정 조건)이 오면 갑자기 모든 저장된 시험 점수를 사라지게 해버려요.
2. 마치 kitchen timer를 맞춰두고 취침한 후, 특정 시간이 되면おkick零食 제조기계가 알아서 돌아가면서 문제를 일으키는 것과 같아요. timer가 돌아가는 동안은 아무도 기계가 이상한 것을 몰라요.
3. 선생님이 시험 당일 아침에 모든 게 이미 Occurred된 뒤에야 "어머, 시험 점수가 다去哪했어?" 하고 발견을 하는 거예요.事後發見이 핵심 특징인 가장 느리지만 가장阴险한 고무력 있는 고장 도구예요.
