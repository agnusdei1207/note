# 바이러스 (Virus) - 호스트 프로그램 기생

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컴퓨터 바이러스는 인간 세포의바이러스처럼 자기 혼자서는生殖不可能하며, 반드시 다른 정상 호스트 프로그램에 기생하여 그 실행에 맞춰 스스로를 복제하고 전파하는 self-replicating 악성 소프트웨어다.
> 2. **가치**: 바이러스는宿主に寄生하여 움직인다는 점에서 웜과根本적으로 다르며, 이 기생 특성으로 인해 네트워크를 통한自動 전파가 어렵고 사용자의어떤 행위(파일 실행, USB 삽입 등)에 의해 manual하게 전파되어 탐지 범위가 상대적으로 제한적이지만, 한번 감염되면 호스트의 모든 권한을 그대로借用하여 피해 규모가 걷잡을 수 없이 커질 수 있다.
> 3. **융합**: 바이러스는 운영체제의 VFS 후킹, 시스템 콜 가로채기, 브라우저 확장 프로그램 삽입 등 다양한 기생 서식지를利用하며, 랜섬웨어, 키로거, Botnet 에이전트 등 다른 악성코드 유형의 초기 감염 벡터로 가장 많이 활용된다.

## 1. 개요 및 필요성

### 개념 및 정의
컴퓨터 바이러스는 1970년대 말~1980년대 초 Arpanet 환경에서 처음 보고된 자체 복제 악성코드로, 생물학적 바이러스가 살아 있는 세포 없이는繁殖할 수 없는 것처럼, 컴퓨터 바이러스 역시 반드시 호스트 프로그램이나 문서 파일에 기생하여 그 실행에 붙어서만 복제되고 전파될 수 있다. 이 기생이라는 속성이 바이러스의 가장 근본적인 특성이며, 이로 인해 웜과의根本적差異가 생긴다.

바이러스의 구성 요소는 일반적으로 다음 세 부분으로 나뉜다. 첫째, **감염 루틴**: 자기 자신의 복제본을 다른 정상 호스트 프로그램에注入하는 메커니즘으로, 보통 대상 프로그램의 끝에 자신의 코드를 추가하거나, 혹은 빈 공간에分散적으로注入하는 방식이 있다. 둘째, **트리거 조건**: 특정 조건(날짜, 시간, 사용자 행위, 시스템 상태)이 충족될 때까지危害を加える行為를 지연시키는 로직으로, 사전 감염 시 기간 동안은 탐지 가능성이 낮다. 셋째, **payload**: 실질적인 피해를 주는 코드 세그먼트로, 데이터 파괴, 백도어 개방, 랜섬웨어暗号化, 키로깅, DDoS 에이전트 설치 등 다양한 Maldns를 포함한다.

**필요성 및 등장 배경**
바이러스가 필요한 위협 벡터인 이유는 자기 만으로 전파할 수 없는이라는 자기 제한적 특성에도 불구하고, 이로 인해 오히려 "사용자의 암묵적授權"을 획득한다는 점에 있다. 웜은 네트워크 취약점만을 利用하여 사용자의知道的 관여 없이勝手的に 전파되지만, 바이러스는 사용자가 의도적으로 infected 파일을 실행하거나 USB를 통해 infected 파일을共有할 때 비로소 전파가 이루어진다.

```
[바이러스 vs 웜: 전파 메커니즘의根本적 차이]

[바이러스 - 기생 전파]

정상 프로그램 실행 --> 바이러스가 호스트에寄生 -->
   |
   +--> 사용자가 infected 파일을 이메일 첨부/USB共有
   |         |
   |         +--> 다른 피해자의 PC에서 실행 --> 감염 확대
   |
전제: "사용자의 의도적 행위" 필요 (파일 열기, 실행 등)


[웜 - 자율 전파]

네트워크 취약점 악용 --> 네트워크 통해勝手적으로 전파
   |
   +--> 이메일, USB 등 매개 매체 불필요

전제: "사용자 행위 불필요" (취약점만 있으면 자동 감염)

핵심 difference 요약:
- Virus = 유인 전파 (사용자 행위 필요, 기생)
- Worm = 자율 전파 (사용자 행위 불필요, 독립 실행)
```

**[다이어그램 해설]** 이 비교는 바이러스와 웜의 가장 핵심적인 분류 기준인 "전파 매커니즘"의 차이를 명확히 보여준다. 바이러스는 반드시 누군가 infected 파일을 실행해줘야만 전파가 이루어지므로, 이론적으로는 사용자의セキュリティ意識向上 교육과 메일 필터링 등으로感染율을大幅히 줄일 수 있다. 반면 웜은 ETERNALBLUE 같이 네트워크 취약점을 exploiting하는 것만으로勝手的に蔓延하므로, 패치 미실치 시스템이 한 대라도 있으면 그것만으로 네트워크 전체로瞬時に 전파될 수 있다.

- **요약 비유**: 컴퓨터 바이러스는 딸기잠옷과 같다. 딸기잠옷은 저기러버 한 마리만으로는 어디 갈 수 없고 반드시 딸기(호스트 프로그램)에 붙어서 딸기를 옮기며, 딸기를 옮기는 것이 아니라 딸기에 스스로를感染시켜 똑같은 옷을 입은 딸기를 하나 더 만들어낸다. 반면 웜은 여러 다리를 타고勝手的に 돌아다니는 autonomouse小偷 로봇이다.

## 2. 아키텍처 및 핵심 원리

### 바이러스의 분류 체계

바이러스는宿主に 붙는 방식, 암호화 여부, 은폐 메커니즘 등 다양한 기준에 따라 여러 类型으로 분류된다.

| 유형 | 감염 방식 | 특징 | 탐지 난이도 |
|---|---|---|---|
| **파일 감염 바이러스** | 실행 파일에 직접 코드 추가 | 가장 전통적, 직관적 감염 | 보통 |
| **부트 섹터 바이러스** | 디스크의 부트 섹터나 MBR에 기생 | 부팅 시早期活性 hook, OS보다 먼저 실행 | 보통~높음 |
| **매크로 바이러스** | Word/Excel 문서 내 VBA 매크로에感染 | 문서를 열기만 하면 발동, 크로스 플랫폼 | 높음 |
| **스크립트 바이러스** | 배치 파일, PowerShell, JavaScript 등 | 발밑 기반, 래핑easy, rapid变异 | 보통~높음 |
| **다형성 바이러스** | 감염 시마다 암호화된 페이로드 복호화 키를 변경 |同一个病毒도 각 감염마다 다른 암호화 | 매우 높음 |
| **멀티파트病毒** | 파일 감염 + 부트 섹터 감염 동시 수행 | 높은 확산 가능성 | 보통~높음 |
| **루트킷型病毒** | 시스템 심층에 침투하여 탐지 코드 자체를 무력화 | 가장恐い, 탐지 자체를 원천 봉쇄 | 극도로 높음 |

### 바이러스 감염 메커니즘: 파일 감염 과정 상세 분석

```
[파일 감염型 바이러스의感染 메커니즘 상세 흐름]

[Phase 1: 감염 탐색 단계]
① 바이러스가 호스트 프로세스로 실행됨
② 현재 디렉터리의 실행 파일 목록 열거

[Phase 2: 감염 실행 단계]
③ 대상 파일의 끝 부분에 바이러스 코드 붙이기

기존 실행 파일 구조:
  [Host Code (원본 코드)] [Padding Area (여유 공간)]

감염 후 실행 파일 구조:
  [Host Code (원본 코드)] [JMP Virus (전이 명령)] [Virus Code (악성 코드)]

실행 시 흐름:
Host 실행 --> JMP Virus로 이동 --> Virus Code 실행
                |
                +--> 감염 루틴 실행 (다른 파일 감염)
                +--> 원본 Host Code으로 복귀 (조용히 수행)

[Phase 3: 지속潜伏 단계]
④ 원본 호스트 코드를 조용히 실행 (사용자 미발견)
⑤ 트리거 조건 충족 시 payload 발동 (데이터 파괴, 백도어 등)
```

**[다이어그램 해설]** 파일 감염型 바이러스의 핵심 트릭은 "원본 코드의 실행 흐름을 자신을 향해 변경(JMP)시켜, 감염 루틴을 실행한 뒤 원래 코드로 자연스럽게 복귀"하는 데 있다. 사용자가感染된 실행 파일을 실행하면, 바이러스 코드가 먼저 실행되어 디렉터리 내 다른 실행 파일을 순차적으로 감염시키면서 동시에 원본 호스트 코드를 메모리에 로드하여 정상 실행시킨다.

### 부트 섹터 바이러스의 특수성

부트 섹터 바이러스는 특히危险的이다. Because it infects the disk's boot sector or MBR, it executes before the operating system even loads, giving it maximum privilege and stealth.

```
[부트 섹터 바이러스의 earliest activeness 메커니즘]

[정상 부팅 순서]
BIOS/UEFI POST --> MBR 읽기 --> Boot Sector 로드
      |
      +--> Operating System 로드 --> OS 정상 가동

[부트 섹터 바이러스 감염 시]
BIOS/UEFI POST --> MBR 읽기 --> [infected Boot Sector]
      |
      +--> 바이러스가 메모리에 먼저 로드 -->
           |
           +--> BOOTkit 기능 설치 (OS 커널에 후킹)
           +--> 원본 Boot Sector을 다른 섹터로 도망시킴
           +--> OS 정상 부팅 (사용자는 모름)

부트킷이 설치되면:
- OS 커널 레벨에서 안티바이러스 조차 탐지 불가
- 키로깅, 네트워크 가로채기 등이 커널 모드에서 작동
- 시스템 재부팅 후에도 지속 존재 (硬盘 포맷에서도 살언)
```

- **요약 비유**: 부트 섹터 바이러스는脳外科 sebelum 의식이 있는 상태에서 작동하는 악마의 Microchip과 같다.人的大脑가 깨어나기 전(OS 부팅 전)에 먼저 chip이埋め込まれ, brain이 켜지자마자 chip이 brain의 모든 명령을 먼저 받으면서brain가 걸어내는 것을 모두 가로채고, brain itself는 "나는的功能正常稼働"이라고 믿게 된다.

## 4. 실무 적용 및 기술사적 판단

### 실무 시나리오: 매크로 바이러스에 의한 대규모 정보 유출

**상황**:某 대기업 임원이 외부 파트너에게 Word 문서를 이메일로 보내工作任务을 할당했다. 이 Word 문서에는 VBA 매크로에仕組まれた 매크로 바이러스가Embedding되어 있었다. 받는이가 Word로 문서를 열자, 매크로가自動的に 실행되어 주소록의 모든 연락처 이메일을 탈취하여 같은 바이러스를 모든 연락처에게 순차적으로 발송했다.Within 3 hours,全社の 주소록이外部流出하고,約 200명의 직원이感染되었다.

**기술적 분석**: 이 사건의恐怖은 전통적 파일 감염 바이러스와는 달리, 매크로 바이러스가 Word/Excel/PowerPoint의macro 엔진에서 직접 실행된다는 점이다. Corporate 이메일 게이트웨이在去来는 exe, .dll 같은 실행 파일은 차단하지만, .docm, .xlsm 같은macro 문서에는 거의 무방비로 통과시켰다.

### 도입 체크리스트 (바이러스 방어 체계)

- **애플리케이션 화이트리스트**: AppLocker 또는 Windows Defender Application Control을 활용하여, 사전 승인된 Hash 기반 실행 파일만 허용하고, 그 외 모든 신규 실행을 차단한다.
- **매크로 보안 강화**: 조직 내 Office 매크로 설정을 "인터넷에서 다운로드된 매크로는全部 차단"으로 설정하고, VBA 매크로 실행 시 반드시 "디지털 서명된 매크로만 허용"으로 격상시킨다.
- **USBAutorun 차단**: Autorun.inf를 통한 자동 감염 경로를 원천 차단하고, USB 연결 시 자동 실행 옵션을 그룹ポリシー로 비활성화한다.
- **파일 무결성 모니터링**: 중요 시스템의 실행 파일에 대해 SHA-256 해시를 사전 등록하고, 파일 변경 시即座에 Security team에 알람을 발생시킨다.

### 안티패턴

- **시그니처 기반 안티바이러스만 신뢰**: Polymorphic, Metamorphic 바이러스를 포함한 未確認 바이러스를 시그니처 데이터베이스에서 찾을 수 없으므로, 100% 탐지를 보장하지 않는다.
- **macro 문서 자동 실행 허용**: 매크로 바이러スの最も 흔한感染 경로가 이메일로 전달되는 .docm/.xlsm 파일이다.

- **요약 비유**: 바이러스 방어는 단순히감기바이러스 예방접종과 같다. 매년 유행하는 독감 바이러스株에 대한 예방접종을 맞아두는 것도 중요하지만, 예측 불가능한 新種 바이러스에 대해서는 기본 면역력(행위 기반 탐지, 화이트리스트)이 핵심 방어선이 된다.

## 어린이를 위한 3줄 비유 설명
1. 컴퓨터 바이러스는 선물 상자 속の 숨겨진 나쁜 귀신과 같아요. 선물 상자를 열면可愛い 인형(정상 프로그램)이 줄을 당기면「삭제할까요?」라고 물어보는데, 줄 중 하나에 나쁜 귀신이 숨어 있어서 인형을 만지는 순간 당신의 방에 들어와 여러 선물 상자(다른 파일)를 찾아 그 중에도 똑같은 나쁜 귀신을 심어버려요.
2. 나쁜 귀신이 한두 개 심어진 정도로는 아직 아무런 문제가 없어요. 하지만 선물 상자를 여러 번 열고 닫다 보면 방 안에서 점점 더 많은 나쁜 귀신이 모이고, 어느 날 집에 아무도 없을 때(트리거 조건)一斉に家中の 식량을 없애거나(데이터 파괴), kunci를 바꿔서 가족을 밖에 가둡니다(랜섬웨어).
3. 예방법은很简单해요: 알지 못하는 사람에게서 받은 선물 상자(실행 파일)를 함부로 열지 말고, 열어야 한다면病毒가 숨어 있을지 몰라서gift를 opened한那一刻就算中毒해도即座 격리할 수 있는 준비를 해두는 것이예요!
