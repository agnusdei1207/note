# 웜 (Worm) - 자가 복제 네트워크 전파 독자 실행

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 웜은 네트워크 취약점을 利用하여 사용자의 いかなる类型的 개입 없이도 자기 혼자서 자율적으로 복제되고 인터넷/인트라넷을 통해 전파되는 self-replicating 악성 소프트웨어로, 컴퓨터바이러스와는 달리 호스트 프로그램에 기생할 필요가 없이 독립적인 실행 파일로 존재한다.
> 2. **가치**: 웜은 단일 초기 감염만으로 네트워크상에 존재하는 모든 취약한 시스템에 Simultaneous에 전파될 수 있는爆炸적 확산 능력을 갖추고 있어, WannaCry, NotPetya처럼数時間内に数万 대를 감염시키는毁灭적 규모의 캠페인이可能하며, 이 속도 때문에 전통적 보안 대응으로는追いつかない.
> 3. **융합**: 웜은 네트워크 프로토콜 스택, 원격 서비스, Zer-day 취약점 등 OS와 네트워크의 복합 영역에서 활동하며, 특히 네트워크 세그먼트간 격리가 되어 있지 않은企业内部망에서横向移動으로瞬時に全军 진군할 수 있다.

## 1. 개요 및 필요성

### 개념 및 정의
웜은 1988년 Robert Morris가 만든 인터넷 역사상 첫번째大规模 웜인 Morris Worm에서 그名が 유래했다. Worm은 한국어 뜻 그대로 "벌레"인데, 이 벌레가 네트워크라는 그물눈을 利用하여 스스로 복제하며 퍼져나가는 모습에서 유래했다. 컴퓨터바이러스와의最大 차이는 "기생 여부"이다. 바이러스는 반드시 정상 호스트 프로그램에 붙어서 그 프로그램을 실행해야만 복제가 시작되지만, 웜은宿主に依赖 없이독립적인 실행 파일로 자기 혼자서 존재하며, 네트워크 취약점만을 利用하여自律적으로扩散한다. 이 특성으로 인해 웜은 "使用자의 어떤 행동도 필요 없이" 전파가 가능하여, 2001년 Code Red는 IIS 웹服务器的 취약점을 利用하여 6시간内に20만 대를 감염시켰다.

웜의 내부 구조는 크게 세 부분으로 구성된다. 첫째, **전파 엔진**: SMTP, SMB, SSH, HTTP 등 다양한 네트워크 프로토콜을 利用하여 다음 감염 대상을探索하고 취약점에 접근하는 모듈이다. 둘째, **복제 모듈**: 자신을 다음 대상 호스트에 복사하고 원격에서 실행시키는 모듈로, 일반적으로 TFTP, HTTP, SMB 파일 공유 등을 통해 자신의 복제본을 전송한다. 셋째, **payload**: 실질적인 피해를 주는恶意 code로, DDoS 에이전트 설치, 백도어 개방, 데이터 훼손, 랜섬웨어暗号化 등 다양하다.

**필요성 및 등장 배경**
웜이 등장한 역사적 배경에는 "모리스 웜(Morris Worm)" 사변이 있다. 1988년 11월 2일, 코넬 대학의学生 Robert Morris가 자신의研究 목적으로 만든 웜이 버그로 인해 의도치 않게 인터넷 전체로扩散되어, 당시 인터넷에 연결된 약 6천 대 중 약 10%인 6천 대를 감염시키고 수백만 달러의 피해를 냈다. 이것이 internet worm의恐怖을世人에게 각인시킨第一个 사례이며, 이후 지속적으로 Zer-day 취약점을 利用하는新型 웜이 등장하면서 네트워크 방화벽, IPS/IDS, 취약점 관리, 네트워크 분할 등의 보안 체계가 본격적으로 발전하게 된 계기가 되었다.

- **요약 비유**: 웜은STFloored 자동複製细菌와 같다. 일반적인细菌는宿主に寄生하여 번식하지만, 이 특수한细菌는 영양분(네트워크 취약점)만 있으면 스스로分裂하며, 一度体内に入ると exponential的速度で全身に扩散합니다.

### 웜의 전파 단계별 동작 분석

```
[웜의 4단계 전파 메커니즘 상세 분석]

[Step 1: 정찰 (Reconnaissance)]
- 랜덤 IP 대역에 TCP SYN 스캔 (포트 445, 22, 80 등)
- 병렬 스캔으로 수천 IP를 동시에 확인
- 열린 포트에 해당 서비스 배너 확인

[Step 2: 취약점 利用 (Exploitation)]
- 발견된 서비스의 버전을 식별
- 해당 버전의 Zer-day 또는 已知 취약점에匹配的 익스플로잇 전송
- 예: EternalBlue (SMBv1 취약점) --> RCE

[Step 3: 복제 및 실행 (Replication & Execution)]
- 타겟 시스템에서 wget/curl/TFTP로 웜 복제본 다운로드
- chmod +x 로 실행 권한 부여
- 백그라운드에서 웜 프로세스 실행
- 원래 웜은 parent로 계속 스캔+전파 시도

[Step 4: 잔존 및 PAYLOAD 발동]
- crontab, 레지스트리 Run key, init script 등에 등록
- 시스템 재부팅 후에도 재감염 방지
- 사전仕組된 payload 발동 (DDoS, 랜섬, 백도어 등)

[전파 속도 비교]

Morris Worm (1988): 感染 6,000대 / 수시간
Code Red (2001):     感染 359,000대 / 14시간

=> 취약점 하나 + 네트워크만 있으면 数시간 内全球 감염 가능!
```

## 4. 실무 적용 및 기술사적 판단

### 실무 시나리오: WannaCry/EternalBlue에 의한企业内部全校感染

**상황**:某 제조기업의 본사는qdm에서 주요 사업장 3곳이 동시에 Production 라인이 멈추는 큰事故가 발생했다. 事後 分析하면, 本사의 한 임원이 받은 이메일의 압축 파일을】解凍하고 그 안에 있던 파일을 클릭해버린 것이 발단이였다. 그 文件은 "주문서.xlsx.exe"로 위장된 WannaCry 웜이었다. 클릭 즉시 내부 네트워크를 통해 全사 서버 1,200 대가 4시간内に 전부 감염되었다.

**기술적 分析**: 이事故의悲惨은 클릭 하나가 全社的生产라인을 멈추게 만든 점이다. Click한 임원의 PC는社内局域网에 연결되어 있었고, WannaCry는 ETERNALBLUE를 利用하여社内LAN 내의 모든 Windows SMB 서버를 자동으로探索하고 있었다. 임원이 클릭한 순간, SMB 통신을 위해 Port 445가 열려 있던社内全 서버가 即時 감염되었다.

### 도입 체크리스트 (웜 방어 체계)

- **네트워크 분할**: 중요 자산을 서로 다른 VLAN에 분리하고, VLAN 간 연결에 엄격한 ACL을 적용한다. 그래야万一 한 구역이 웜에 감염되어도 全社扩散를 원천 차단한다.
- **취약점 즉시 패치 자동화**: WSUS, SCCM, Qualys, Tenable 등의Patch Management 도구를 利用하여, 윈도우 보안 패치를48시간 이내에 자동 배포하는 체계를 수립한다.
- **SMBv1 비활성화**: ETERNALBLUE 계열 웜은 전부 SMBv1을 利用한다. 그룹ポリシー로 SMBv1을 차단하면 이 계열 웜의 99%를 원천 차단할 수 있다.
- **네트워크 레벨 IPS/IDS**: Snort, Suricata, Darktrace 등의 네트워크 IDS/IPS를 운영하여, 봇넷 C2 통신 및 웜의 Lateral Movement 패턴을 즉각 탐지하고 차단한다.

### 안티패턴

- **네트워크 분할 없이 плоская 네트워크 운영**: 全社가同一 네트워크 브로드캐스트 도메인에 있으면, 웜이一台에만 감염되면 Gratuitous ARP와 SMB 재밍을 통해瞬時に 全네트워크로扩散한다.
- **패치 적용 지연**: "다음维护window까지 패치 적용을 미루자"는 사고 직전의 最悪 판단이다. Zer-day 취약점 공개 후 24~48시간이Critical Window이며, 이 기간 내에 패치하지 않으면 웜이 全球에서流行하는 것을防げない.

- **요약 비유**: 웜 방어는STFloorless 선풍기故障와 같다. 선풍기 모터에 문제가 생겨 불꽃이 나면, 그 불꽃이 바로 옆 선풍기로 옮겨붙는 것이 아니라,Electrical wiring을 타고 집안의 모든 선풍기로 옮겨붙고, 한 선풍기에서 불꽃이 나면 그 집의 wiring을 타고隣집 선풍기로 옮겨붙는다. 共用地線的全部를 끊어야(네트워크 분할)하거나, 元불꽃을 낸 모터 자체를 即時に 교체해야(即时 패치)한다.
