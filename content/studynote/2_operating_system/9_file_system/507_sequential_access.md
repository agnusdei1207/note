## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 순차 접근(Sequential Access)은 파일의 레코드를 처음부터 끝까지 정해진 순서대로 하나씩 처리하는 가장 고전적이고 단순한 데이터 접근 방식이다.
> 2. **가치**: 카세트 테이프나 자기 테이프(Magnetic Tape)와 같은 순차적 매체에 최적화되어 있으며, 대량의 데이터를 일괄 처리(Batch Processing)할 때 뛰어난 예측 가능성을 제공한다.
> 3. **융합**: 현대 시스템에서는 로그 파일(Log File) 기록이나 미디어 스트리밍(Streaming) 등 연속적인 데이터 흐름을 다루는 영역에서 여전히 핵심적인 메커니즘으로 활용된다.

---

## Ⅰ. 순차 접근의 정의와 매커니즘 (Definition & Mechanism)

- **정의**: 데이터를 읽거나 쓸 때 이전 레코드의 위치 정보를 바탕으로 다음 레코드를 찾아가는 방식이다.
- **동작 원리**:
  - `read next`: 현재 위치에서 다음 데이터를 읽고 포인터를 자동으로 1단위 전진시킨다.
  - `write next`: 현재 위치 다음에 데이터를 기록하고 포인터를 전진시킨다.
  - `reset / rewind`: 포인터를 파일의 맨 처음으로 되돌린다.
- **특징**: 특정 중간 지점으로 즉시 점프(Jump)하는 기능이 제한적이며, n번째 레코드를 보려면 앞의 n-1개를 거쳐야 한다.

📢 **섹션 요약 비유**: 순차 접근은 "카세트 테이프"와 같다. 3번째 노래를 들으려면 1번과 2번 노래를 빠르게 감아서(Rewind/Forward) 지나가야 하는 것과 같다.

---

## Ⅱ. 순차 접근의 논리적 구조 (Logical Architecture)

순차 접근 파일은 레코드의 연속으로 구성된다.

### 1. 포인터 기반 이동
OS는 내부적으로 `Current Position Pointer`를 유지하여 다음 연산의 대상 바이트를 가리킨다.

### 2. 순차 접근 프로세스 (ASCII Diagram)

```text
 [ Start ]                                        [ End ]
    ▼                                                ▼
 ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
 │ Rec1 │ Rec2 │ Rec3 │ Rec4 │ Rec5 │ Rec6 │ ...  │ EOF  │
 └──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
           ▲      │
           └──────┘
        Current Pos (Pointer)
        - After read: Pointer moves to Rec3
        - After reset: Pointer moves to Rec1
```

📢 **섹션 요약 비유**: 순차 접근은 "한 줄로 늘어선 기차의 칸"과 같다. 5호차에 가려면 반드시 1, 2, 3, 4호차를 통과해서 걸어가야 하는 구조다.

## Ⅲ. 주요 활용 사례 (Use Cases)

- **컴파일러 (Compilers)**: 소스 코드를 처음부터 끝까지 읽으며 문법을 분석하는 과정은 전형적인 순차 접근이다.
- **로그 파일 (Log Files)**: 시스템 장애나 접근 기록을 시간 순서대로 덧붙여(Append) 저장할 때 사용한다.
- **데이터 백업 (Backups)**: 자기 테이프 장치에 대용량 데이터를 저장할 때 비용 대비 가장 효율적인 방식이다.
- **미디어 스트리밍**: 비디오나 오디오 파일을 재생할 때 시간에 맞춰 순차적으로 데이터를 버퍼링한다.

📢 **섹션 요약 비유**: 활용 사례는 "소설책 읽기"와 같다. 이야기의 흐름을 이해하려면 첫 페이지부터 차례대로 읽어야지, 중간부터 읽으면 내용을 이해하기 어려운 것과 같다.

## Ⅳ. 순차 접근의 장단점 분석 (Pros & Cons)

### 1. 장점
- **단순성**: 별도의 인덱스나 복잡한 주소 변환 테이블이 필요 없어 구현이 쉽다.
- **공간 효율성**: 데이터 사이에 빈틈(Gap) 없이 빽빽하게 저장할 수 있어 저장 공간 낭비가 적다.
- **예측 가능성**: 다음 읽을 위치가 명확하므로 미리 읽기(Pre-fetching)를 통한 성능 최적화가 용이하다.

### 2. 단점
- **검색 속도**: 특정 데이터를 찾기 위한 탐색 시간(Search Time)이 데이터 양에 비례하여 늘어난다 ($O(n)$).
- **수정의 어려움**: 중간에 있는 데이터를 삭제하거나 길이를 변경하면 뒤에 오는 모든 데이터를 밀어내거나 당겨야 한다.

📢 **섹션 요약 비유**: 순차 접근은 "줄 서기"와 같다. 내 차례를 기다리는 것은 정직하고 쉽지만(장점), 맨 뒷사람이 맨 앞사람을 만나려면 모두가 비켜줘야 한다(단점).

---

## Ⅴ. 현대 파일 시스템에서의 위상 (Modern Status)

- **추상화**: 현대의 HDD/SSD는 물리적으로 직접 접근(Direct Access)이 가능하지만, OS는 파일 연산의 기본 인터페이스로 순차적 인터페이스(`read/write` 시스템 콜)를 제공하여 사용성을 높인다.
- **VFS 계층**: 가상 파일 시스템(VFS)은 파이프(Pipe)나 소켓(Socket)과 같이 근본적으로 순차적인 입출력 장치를 파일과 동일한 순차 접근 방식으로 다룰 수 있게 해준다.

📢 **섹션 요약 비유**: 현대적 위상은 "에스컬레이터"와 같다. 원한다면 뛰어갈 수도 있지만(Direct), 대부분의 사용자는 가만히 서서 차례대로 이동하는 방식(Sequential Interface)을 편안하게 이용한다.
