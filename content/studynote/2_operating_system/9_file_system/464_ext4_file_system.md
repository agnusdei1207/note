# 464. ext4 파일 시스템

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 리눅스 표준 저널링 파일 시스템
> 2. **가치**: 안정성과 성능의 균형
> 3. **융합**: ext3, 저널링, 아이노드와 연관

---

## Ⅰ. 개요

### 개념 정의

ext4(Fourth Extended File System)는 **리눅스에서 가장 널리 사용되는 저널링 파일 시스템으로, ext3의 개선판**이다.

### 💡 비유: 건물 리모델링
ext4는 **건물 리모델링**과 같다. 기존 구조(ext3)를 유지하면서 내부를 개선했다.

### ext4 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                ext4 파일 시스템                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ext4 계층 구조】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │              슈퍼블록 (Superblock)                         │    │ │
│  │  │  • 파일 시스템 전체 정보                                    │    │ │
│  │  │  • 블록 크기, 전체 블록 수, 여유 블록                        │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │

│  │  │  ┌───────────────────┬───────────────────┬───┐             │    │ │
│  │  │  │ 슈퍼블록 복사본      │ 슈퍼블록 복사본    │   │             │    │ │
│  │  │  │ 그룹 디스크립터      │ 그룹 디스크립터    │   │             │    │ │

│  │  │  │ 아이노드 테이블      │ 아이노드 테이블    │   │             │    │ │
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                ext4 상세                                              │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ext4 주요 기능】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 익스텐트 (Extents):                                               │ │
│  │     • 연속된 블록을 하나의 단위로 관리                                   │ │
│  │     • 기존: 블록마다 포인터 → 익스텐트: 시작+길이                        │ │
│  │     • 대용량 파일 성능 향상                                            │ │
│  │                                                             │ │
│  │  2. 지연 할당 (Delayed Allocation):                                   │ │
│  │     • 쓰기 시점이 아닌 flush 시점에 블록 할당                            │ │
│  │     • 단편화 감소                                                      │ │
│  │     • 성능 향상                                                        │ │
│  │                                                             │ │
│  │  3. 다중 블록 할당 (Multiblock Allocation):                            │ │
│  │     • 한 번에 여러 블록 할당                                            │ │
│  │     • 할당 오버헤드 감소                                                │ │
│  │                                                             │ │
│  │  4. 저널 체크섬:                                                      │ │
│  │     • 저널 데이터 무결성 검증                                           │ │
│  │     • 손상 감지 향상                                                    │ │
│  │                                                             │ │
│  │  5. 빠른 fsck:                                                        │ │
│  │     • 미사용 아이노드 스킵                                              │ │
│  │     • fsck 시간 대폭 단축                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ext4 아이노드 구조】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ext4 아이노드 (256 bytes):                                           │ │
│  │  • 기본 메타데이터 (권한, 소유자, 크기 등)                                │ │
│  │  • 60바이트 익스텐트 트리 (4 익스텐트 × 15바이트)                          │ │
│  │  • 확장 속성 (xattr)                                                   │ │
│  │  • 나노초 타임스탬프                                                   │ │
│  │                                                             │ │
│  │  익스텐트 트리:                                                         │
│  │  • 직접 익스텐트 (0-3): 파일 데이터 직접                                │ │
│  │  • 간접 노드: 대용량 파일 확장                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【ext4 저널 모드】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  data=ordered (기본): 메타데이터만 저널링, 데이터 먼저 쓰기             │ │
│  │  data=writeback: 메타데이터만 저널링, 순서 보장 없음                   │ │
│  │  data=journal: 메타데이터 + 데이터 모두 저널링                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ext4 파일 시스템 생성】                                            │
│  ──────────────────                                                  │
│  // 기본 생성                                                          │
│  $ sudo mkfs.ext4 /dev/sda1                                         │
│                                                                     │
│  // 익스텐트 크기 지정                                                  │
│  $ sudo mkfs.ext4 -E stride=16,stripe-width=64 /dev/sda1           │
│                                                                     │
│  // 저널 없이 생성 (성능 우선)                                           │
│  $ sudo mkfs.ext4 -O ^has_journal /dev/sda1                        │
│                                                                     │
│  // 아이노드 크기 지정                                                  │
│  $ sudo mkfs.ext4 -I 256 /dev/sda1                                 │
│                                                                     │
│  【ext4 튜닝】                                                          │
│  ──────────────────                                                  │
│  // 파일 시스템 정보 확인                                                │
│  $ sudo tune2fs -l /dev/sda1                                        │
│                                                                     │
│  // 저널 모드 변경                                                      │
│  $ sudo tune2fs -o journal_data_writeback /dev/sda1                │
│                                                                     │
│  // 예약 블록 비율 변경 (기본 5%)                                        │
│  $ sudo tune2fs -m 1 /dev/sda1  // 1%로 변경                         │
│                                                                     │
│  // 마운트 옵션                                                         │
│  $ sudo mount -o noatime,data=writeback /dev/sda1 /mnt             │
│                                                                     │
│  【ext4 마운트 옵션】                                                  │
│  ──────────────────                                                  │
│  // /etc/fstab 예시                                                   │
│  /dev/sda1  /  ext4  defaults,noatime,data=ordered  0  1           │
│                                                                     │
│  주요 옵션:                                                             │
│  • noatime: 접근 시간 갱신 안 함 (성능 향상)                              │
│  • relatime: 접근 시간 선택적 갱신                                        │
│  • data=ordered: 기본 저널 모드                                        │
│  • barrier=1: 쓰기 배리어 활성화 (안전)                                   │
│  • discard: TRIM 활성화 (SSD)                                          │
│  • user_xattr: 확장 속성 지원                                           │
│  • acl: ACL 지원                                                       │
│                                                                     │
│  【ext4 상태 확인】                                                    │
│  ──────────────────                                                  │
│  // 파일 시스템 사용량                                                    │
│  $ df -Th /                                                          │
│  Filesystem     Type  Size  Used Avail Use% Mounted on              │
│  /dev/sda1      ext4   50G   20G   28G  42% /                       │
│                                                                     │
│  // 아이노드 사용량                                                       │
│  $ df -iTh /                                                         │
│  Filesystem     Type  Inodes  IUsed  IFree IUse% Mounted on         │
│  /dev/sda1      ext4  3.2M    100K   3.1M   4% /                    │
│                                                                     │
│  // 상세 통계                                                           │
│  $ sudo dumpe2fs /dev/sda1 | head -30                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 리눅스 표준 저널링 파일 시스템
• 최대: 볼륨 1 EB, 파일 16 TB
• 특징: 익스텐트, 지연 할당, 다중 블록 할당
• 저널: ordered(기본), writeback, journal
• 마운트: noatime, relatime, barrier
• 생성: mkfs.ext4
• 튜닝: tune2fs
• 확인: df, dumpe2fs
```
