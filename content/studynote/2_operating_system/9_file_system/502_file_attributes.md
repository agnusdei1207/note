## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 속성(File Attributes)은 파일의 실제 데이터 내용이 아닌, 운영체제(OS)가 파일을 관리하기 위해 유지하는 보조 정보인 메타데이터(Metadata)의 집합이다.
> 2. **가치**: 속성은 파일 시스템의 무결성을 보장하고, 접근 제어(Access Control) 및 효율적인 검색, 백업 전략 수립을 위한 핵심 파라미터로 활용된다.
> 3. **융합**: 현대 저널링 파일 시스템(Journaling FS)은 속성 변경 이력까지 관리하여 시스템 충돌 시 복구 지점을 제공하고 보안 감사(Audit)의 근거 자료로 사용된다.

---

## Ⅰ. 파일 속성의 개념과 역할 (Definition & Purpose)

- **정의**: **파일 속성(File Attributes)**은 파일의 물리적 데이터와는 별개로 OS의 파일 제어 블록(FCB, File Control Block) 또는 아이노드(inode)에 저장되는 관리용 정보다.
- **역할**:
  1. **식별**: 시스템 내에서 파일을 유일하게 구분.
  2. **관리**: 파일의 물리적 위치와 크기를 추적하여 데이터 입출력(I/O) 지원.
  3. **보안**: 권한 및 소유권 확인을 통해 불법적인 접근 차단.
  4. **최적화**: 생성 및 수정 시간을 기반으로 캐싱(Caching) 및 증분 백업 수행.

📢 **섹션 요약 비유**: 파일 속성은 "도서관 책의 대출 카드"와 같다. 책의 내용(데이터)을 읽지 않아도 카드(속성)만 보면 누가 썼는지, 언제 들어왔는지, 누구에게 빌려줄 수 있는지 알 수 있다.

---

## Ⅱ. 주요 파일 속성 체계 (Core Attributes Hierarchy)

OS는 다양한 종류의 속성을 관리하며, 이는 크게 식별, 위치, 보호, 시간 속성으로 분류된다.

### 1. 속성 분류 및 상세
- **이름 (Name)**: 인간이 이해할 수 있는 유일한 문자열.
- **식별자 (Identifier)**: 시스템 내부의 고유 번호(예: inode 번호).
- **타입 (Type)**: 파일의 형식(Regular, Directory, FIFO, Socket 등).
- **소유자 (Owner/UID)**: 파일을 생성한 사용자 또는 소유 그룹의 ID.
- **크기 (Size)**: 파일의 현재 바이트 수 및 블록 점유 수.
- **시간 정보 (Timestamp)**: 생성 시간(Birth), 최근 수정 시간(mtime), 최근 접근 시간(atime).

### 2. 파일 메타데이터 구조 (ASCII Diagram)

```text
       ┌────────────────────────────────────────────────────────┐
       │             File Control Block (FCB / inode)           │
       ├────────────────────────────────────────────────────────┤
       │ [Identification]       Name: "system_config.conf"      │
       │                        ID: 0x8A2C (Unique Inode #)     │
       ├────────────────────────────────────────────────────────┤
       │ [Security/Auth]        Owner UID: 1001 (Admin)         │
       │                        Group GID: 100 (Staff)          │
       │                        Permissions: 644 (rw-r--r--)    │
       ├────────────────────────────────────────────────────────┤
       │ [Storage Info]         File Size: 2,048 Bytes          │
       │                        Block Count: 4 Blocks           │
       │                        Address Pointers: [B10, B11...] │
       ├────────────────────────────────────────────────────────┤
       │ [Temporal Data]        Creation: 2023-10-27 10:00:00   │
       │                        Last Mod: 2023-10-28 14:20:05   │
       │                        Last Acc: 2023-10-29 09:12:33   │
       └────────────────────────────────────────────────────────┘
```

📢 **섹션 요약 비유**: 파일 속성 체계는 "여권의 정보 면"과 같다. 이름, 국적, 생년월일, 만료일 등을 한눈에 확인하여 국가 시스템이 사람을 관리하게 돕는다.

## Ⅲ. 시간 속성과 데이터 무결성 (Time Attributes & Integrity)

- **생성 시간 (Creation Time)**: 파일이 처음 디스크에 할당된 시각.
- **최근 수정 시간 (mtime, Modification Time)**: 파일의 **데이터 내용**이 마지막으로 변경된 시각.
- **최근 속성 변경 시간 (ctime, Change Time)**: 파일의 **메타데이터(권한, 이름 등)**가 변경된 시각.
- **최근 접근 시간 (atime, Access Time)**: 파일 데이터를 마지막으로 읽거나 실행한 시각.
  - *참고*: 성능 향상을 위해 현대 리눅스(Linux) 등에서는 `noatime` 또는 `relatime` 옵션을 사용하여 atime 업데이트를 제한하기도 한다.

📢 **섹션 요약 비유**: 시간 속성은 "음식의 유통기한 및 제조일자"와 같다. 언제 만들어졌고 언제 마지막으로 검사했는지에 따라 신선도(무결성)를 판단한다.

---

## Ⅳ. 소유권과 보호 속성 (Ownership & Protection)

- **사용자 식별자 (UID, User Identifier)**: 파일의 소유자를 정의하며, 해당 사용자는 파일의 권한을 변경할 권리를 갖는다.
- **그룹 식별자 (GID, Group Identifier)**: 특정 사용자 그룹에게 공동의 권한을 부여하기 위해 사용된다.
- **접근 제어 목록 (ACL, Access Control List)**: 전통적인 UID/GID 구조를 넘어, 특정 사용자별로 세밀한 권한(Read/Write/Execute)을 지정한다.
- **특수 비트 (Sticky Bit, SetUID, SetGID)**: 파일 실행 시 일시적으로 권한을 격상시키거나 공유 디렉터리 내 보안을 강화하는 용도로 사용된다.

📢 **섹션 요약 비유**: 보호 속성은 "회사의 직급과 출입 카드 권한"과 같다. 사장님(소유자)은 모든 방에 들어갈 수 있지만, 외부 방문자(Other)는 로비만 이용 가능하다.

---

## Ⅴ. 속성의 물리적 저장과 관리 (Storage & Persistence)

- **디렉터리 항목 (Directory Entry)**: 파일 이름과 inode 번호를 매핑하여 저장한다.
- **inode 테이블**: 실제 속성 정보(이름 제외)를 저장하는 전용 영역이다.
- **비휘발성 (Non-volatility)**: 모든 속성은 시스템 전원이 꺼져도 유지되어야 하므로 디스크의 특정 시스템 영역에 기록된다.
- **확장 속성 (Extended Attributes, xattr)**: 사용자가 정의하거나 OS 기능 확장을 위해 추가적으로 붙이는 메타데이터(예: 보안 레이블, 체크섬 등).

📢 **섹션 요약 비유**: 속성의 저장은 "병원의 차트실"과 같다. 환자(데이터)는 병실에 있지만, 그들의 상태 정보(속성)는 차트실에 따로 보관되어 의사(OS)가 효율적으로 관리하게 한다.
