## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일(File)은 운영체제(OS, Operating System)가 사용자에게 제공하는 논리적 저장 단위의 추상화(Abstraction)이자 비휘발성 저장 장치에 기록된 정보의 집합이다.
> 2. **가치**: 물리적 장치의 복잡성(섹터, 트랙 등)을 은폐하고 이름(Name)과 속성(Attributes)을 가진 일관된 논리적 인터페이스를 제공함으로써 데이터 관리의 효율성을 극대화한다.
> 3. **융합**: 현대 OS는 VFS(Virtual File System)를 통해 디스크 파일뿐만 아니라 장치, 파이프, 소켓 등을 모두 '파일'로 취급하는 'Everything is a file' 철학을 구현한다.

---

## Ⅰ. 파일의 정의와 추상화 (Concept & Abstraction)

- **정의**: **파일(File)**은 보조기억장치에 저장된 연관된 정보들의 논리적 집합체다. 운영체제는 물리적 저장 매체의 물리적 특성을 추상화하여, 사용자가 비트(Bit)나 바이트(Byte)의 연속체로 데이터를 바라볼 수 있게 한다.
- **추상화의 목적**: 하드디스크(HDD, Hard Disk Drive)나 SSD(Solid State Drive)의 물리적 주소(LBA, Logical Block Address)를 직접 다루지 않고, 의미 있는 단위인 '파일'로 데이터를 관리하기 위함이다.
- **💡 비유**: 파일은 "서류 봉투"와 같다. 봉투 안에는 문서, 사진, 도면 등 다양한 형태의 정보가 담길 수 있지만, 우리는 외부에서 '보고서.docx'라는 이름표(Name)만 보고 그 안의 내용을 식별하고 관리한다. 서류함의 어느 칸(물리적 위치)에 있는지 몰라도 이름만 알면 찾을 수 있는 것과 같다.

📢 **섹션 요약 비유**: 파일은 거대한 도서관의 책과 같아서, 독자(사용자)는 서가 번호(물리적 주소) 대신 책 제목(파일명)으로 지식을 소비한다.

## Ⅱ. 파일 속성 (File Attributes) 및 구조 (Structure)

파일은 데이터 자체뿐만 아니라 이를 관리하기 위한 **메타데이터(Metadata)**를 포함한다.

### 1. 주요 파일 속성
- **이름 (Name)**: 사용자가 식별할 수 있는 유일한 문자열.
- **식별자 (Identifier)**: 시스템 내부에서 파일을 구분하는 고유 번호 (예: inode 번호).
- **타입 (Type)**: 파일의 종류 (예: 실행 파일, 데이터 파일, 디렉터리).
- **위치 (Location)**: 장치 내 파일 데이터의 시작 포인터.
- **크기 (Size)**: 파일의 현재 크기 및 최대 허용 크기.
- **보호 (Protection)**: 읽기, 쓰기, 실행 권한(rwx) 정보.
- **시간 및 사용자 식별 (Time & User ID)**: 생성, 수정, 최근 접근 시간 및 소유자 정보.

### 2. 파일 개념도 (ASCII Diagram)

```text
 [ Logical View (User) ]          [ System View (OS/Metadata) ]
 ┌──────────────────────┐        ┌──────────────────────────────────┐
 │    MyDocument.txt    │──────▶ │ File Control Block (FCB/inode)   │
 └──────────────────────┘        ├──────────────────────────────────┤
            │                    │ - Name: MyDocument.txt           │
            ▼                    │ - ID: 10245                      │
 ┌──────────────────────┐        │ - Type: Regular File             │
 │ "Hello, World!..."   │        │ - Permissions: rw-r--r--         │
 │ (Byte Stream Data)   │        │ - Owner: user01                  │
 └──────────────────────┘        │ - Data Pointers: [P1, P2, ...]   │
                                 └──────────────────────────────────┘
                                                │
                                                ▼
                                [ Physical View (Storage Device) ]
                                ┌───────┬───────┬───────┬───────┐
                                │ Block │ Block │ Block │ Block │
                                │  #45  │  #46  │  #88  │  #90  │
                                └───────┴───────┴───────┴───────┘
```

📢 **섹션 요약 비유**: 파일 속성은 "신분증"과 같다. 이름, 주민번호(식별자), 주소(위치), 직업(타입) 등을 통해 국가(OS)가 국민(파일)을 체계적으로 관리하는 것과 동일하다.

## Ⅲ. 파일의 논리적/물리적 구조 (Logical vs Physical)

- **논리적 구조**: 사용자가 보는 관점으로, 레코드(Record)의 집합이나 단순한 바이트 스트림(Byte Stream)으로 정의된다. 현대 범용 OS는 대부분 구조가 없는 바이트 스트림 방식을 선호하여 유연성을 확보한다.
- **물리적 구조**: 저장 장치의 블록(Block) 단위로 나뉘어 저장되는 형태다. 연속 할당, 연결 할당, 인덱스 할당 등의 기법을 통해 논리적 데이터를 물리적 블록에 매핑한다.
- **관리 주체**: OS의 파일 시스템(FS, File System)이 논리적 주소를 물리적 주소로 변환(Mapping)하는 역할을 수행한다.

📢 **섹션 요약 비유**: 논리적 구조가 "소설의 줄거리"라면, 물리적 구조는 "책의 종이와 잉크 배치"다. 독자는 줄거리를 읽지만, 실제로는 종이 위에 찍힌 잉크를 보는 것이다.

## Ⅳ. 파일 타입과 확장자 (Types & Extensions)

- **타입의 중요성**: OS는 파일 타입을 통해 적절한 응용 프로그램과 연결하거나 시스템의 오작동을 방지한다.
- **확장자 (Extension)**: 파일 이름의 끝에 붙는 문자열(예: .exe, .sh, .jpg)로 타입을 암시한다.
- **매직 넘버 (Magic Number)**: 파일의 최상단에 기록된 고유 식별 바이트로, 확장자와 무관하게 실제 파일 형식을 판단하는 근거가 된다.
- **시스템 파일 vs 사용자 파일**: 커널(Kernel)이 관리하는 특수 파일(장치 파일 등)과 사용자가 생성한 일반 데이터 파일을 구분한다.

📢 **섹션 요약 비유**: 확장자는 "옷차림"이고 매직 넘버는 "DNA"다. 옷을 갈아입어도(확장자 변경) 근본적인 정체성(매직 넘버)은 변하지 않는다.

---

## Ⅴ. 파일 보호 및 보안 (Protection & Security)

- **접근 제어 (Access Control)**: 특정 사용자가 파일에 대해 수행할 수 있는 연산(읽기, 쓰기, 실행, 삭제, 목록 보기 등)을 제한한다.
- **ACL (Access Control List)**: 각 파일에 대해 접근 가능한 사용자 목록과 권한을 명시한 리스트다.
- **권한 비트 (Permission Bits)**: Unix/Linux 계열에서 사용하는 rwxrwxrwx 형태의 비트 마스크로 소유자, 그룹, 기타 사용자의 권한을 관리한다.
- **공유 (Sharing)**: 다중 사용자 시스템에서 파일 공유는 필수적이며, 이를 위해 소유권(Ownership)과 보호 기법이 정밀하게 설계되어야 한다.

📢 **섹션 요약 비유**: 파일 보호는 "금고의 비밀번호"와 같다. 금고 주인만 열 수 있게 하거나, 특정인에게만 내용물을 보여줄 수 있도록 통제하는 장치다.

## 👶 아이를 위한 비유 (Child Analogy)
> "파일은 네가 그린 그림을 넣어두는 **'마법의 상자'**야. 상자 겉면에는 '내 그림'이라는 **이름**이 써 있고, 언제 만들었는지 **날짜**도 적혀 있지. 이 상자는 엄마(OS)가 창고(하드디스크) 깊숙한 곳 어디에 숨겨놨는지 다 기억하고 있어서, 네가 이름만 부르면 바로 가져다준단다. 하지만 다른 사람이 네 그림을 망치지 못하게 **자물쇠(보호)**를 채워둘 수도 있어!"
