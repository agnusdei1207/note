# 465. NTFS (New Technology File System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 윈도우 표준 파일 시스템
> 2. **가치**: 보안, 복구, 대용량 지원
> 3. **융합**: 저널링, ACL, 압축과 연관

---

## Ⅰ. 개요

### 개념 정의

NTFS(New Technology File System)는 **마이크로소프트가 개발한 윈도우 운영체제의 표준 파일 시스템**이다.

### 💡 비유: 스마트 오피스
NTFS는 **스마트 오피스**와 같다. 보안 카드(ACL), 감시 카메라(저널링), 압축 수납(압축) 기능이 있다.

### NTFS 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                NTFS 구조                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NTFS 아키텍처】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                   $MFT (Master File Table)               │    │ │
│  │  │  • 모든 파일/디렉토리 메타데이터                             │    │ │
│  │  │  • 파일 하나당 하나의 MFT 엔트리                             │    │ │
│  │  │  • 파일 속성: $STANDARD_INFORMATION, $FILE_NAME,          │    │ │
│  │  │             $DATA, $ATTRIBUTE_LIST, ...                   │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                   데이터 영역                              │    │ │
│  │  │  • 파일 데이터                                            │    │ │
│  │  │  • B+ 트리 구조로 인덱싱                                    │    │ │
│  │  │  • 클러스터 단위 할당                                      │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MFT 구조】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  MFT 엔트리 (1KB):                                                    │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  헤더        │ 속성 리스트       │ 리즈던트             │    │ │
│  │  │  (Header)    │ (Attributes)      │ (Resident)           │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  주요 속성:                                                           │ │
│  │  • $STANDARD_INFORMATION: 타임스탬프, 속성                        │ │
│  │  • $FILE_NAME: 파일 이름, 부모 디렉토리                            │ │
│  │  • $DATA: 파일 데이터 (리즈던트 또는 논리즈던트)                    │ │
│  │  • $SECURITY_DESCRIPTOR: 보안 설정                                │ │
│  │  • $OBJECT_ID: 고유 식별자                                        │ │
│  │                                                             │ │
│  │  작은 파일: MFT 내에 직접 저장 (리즈던트)                             │ │
│  │  큰 파일: MFT 외부에 저장 (논리즈던트)                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                NTFS 상세                                              │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NTFS 주요 기능】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 저널링 ($LogFile):                                               │ │
│  │     • 메타데이터 변경 사항 기록                                         │ │
│  │     • 시스템 충돌 시 빠른 복구                                          │ │
│  │                                                             │ │
│  │  2. 보안 (ACL):                                                       │ │
│  │     • 파일/디렉토리별 세밀한 권한                                        │ │
│  │     • 사용자/그룹별 읽기/쓰기/실행/삭제                                  │ │
│  │                                                             │ │
│  │  3. 압축:                                                             │ │
│  │     • 파일/디렉토리 단위 압축                                           │ │
│  │     • 투명한 압축/해제                                                  │ │
│  │                                                             │ │
│  │  4. 암호화 (EFS):                                                     │ │
│  │     • 파일 시스템 레벨 암호화                                           │ │
│  │     • 사용자 인증서 기반                                                │ │
│  │                                                             │ │
│  │  5. 하드 링크:                                                        │ │
│  │     • 여러 파일명이 같은 데이터 참조                                     │ │
│  │                                                             │ │
│  │  6. 심볼릭 링크:                                                       │ │
│  │     • 파일/디렉토리에 대한 링크                                         │ │
│  │                                                             │ │
│  │  7. 디스크 쿼터:                                                      │ │
│  │     • 사용자별 디스크 사용량 제한                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NTFS vs FAT32】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  특성           FAT32             NTFS                             │ │
│  │  ────           ─────             ────                              │ │
│  │  최대 볼륨       2 TB (실제 32GB)   256 TB (이론상 16 EB)           │ │
│  │  최대 파일       4 GB              16 TB (이론상 16 EB)             │ │
│  │  보안           없음               ACL                              │
│  │  저널링         없음               있음                              │
│  │  압축           없음               있음                              │
│  │  암호화         없음               EFS                              │
│  │  호환성         높음               윈도우/일부 리눅스                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NTFS 시스템 파일】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  $MFT         마스터 파일 테이블                                    │ │
│  │  $MFTMirr     MFT 백업 (처음 4개 엔트리)                             │ │
│  │  $LogFile     저널 파일                                              │ │
│  │  $Volume      볼륨 정보                                              │ │
│  │  $AttrDef     속성 정의                                              │ │
│  │  $Bitmap      할당 비트맵                                            │ │
│  │  $Boot        부트 섹터                                              │ │
│  │  $BadClus     배드 섹터 리스트                                        │ │
│  │  $Secure      보안 설명자                                            │ │
│  │  $Upcase      대소문자 변환 테이블                                    │ │
│  │  $Extend      확장 속성 디렉토리                                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【윈도우에서 NTFS 관리】                                            │
│  ──────────────────                                                  │
│  // 파일 시스템 확인                                                    │
│  > fsutil fsinfo volumeinfo C:                                      │
│  파일 시스템 이름 : NTFS                                               │
│  최대 구성 요소 길이 : 255                                             │
│                                                                     │
│  // 압축 설정                                                          │
│  > compact /c /s:C:\Data                                            │
│  > compact /u /s:C:\Data  // 압축 해제                                │
│                                                                     │
│  // 디스크 쿼터 설정                                                    │
│  > fsutil quota enforce C:                                          │
│  > fsutil quota modify C: 1000 2000 username                        │
│                                                                     │
│  【리눅스에서 NTFS 사용】                                              │
│  ──────────────────                                                  │
│  // ntfs-3g로 마운트                                                   │
│  $ sudo apt install ntfs-3g                                         │
│  $ sudo mount -t ntfs-3g /dev/sdb1 /mnt/ntfs                       │
│                                                                     │
│  // 읽기/쓰기 마운트                                                    │
│  $ sudo mount -t ntfs-3g -o rw /dev/sdb1 /mnt/ntfs                 │
│                                                                     │
│  // ntfsfix로 간단한 복구                                              │
│  $ sudo ntfsfix /dev/sdb1                                           │
│                                                                     │
│  // NTFS 파티션 생성                                                   │
│  $ sudo mkfs.ntfs /dev/sdb1                                         │
│                                                                     │
│  【NTFS 정보 확인】                                                    │
│  ──────────────────                                                  │
│  // ntfsinfo로 상세 정보                                               │
│  $ ntfsinfo -m /dev/sdb1                                            │
│  Cluster Size: 4096                                                 │
│  Volume Size: 500 GB                                                │
│  MFT Size: 256 MB                                                   │
│                                                                     │
│  // 윈도우에서 MFT 크기 확인                                            │
│  > fsutil usn queryjournal C:                                       │
│                                                                     │
│  【NTFS 최적화】                                                      │
│  ──────────────────                                                  │
│  // 8.3 파일명 생성 비활성화 (성능 향상)                                  │
│  > fsutil 8dot3name strip C:                                        │
│                                                                     │
│  // 마지막 접근 시간 갱신 비활성화                                        │
│  > fsutil behavior set disablelastaccess 1                         │
│                                                                     │
│  // 디스크 조각 모음                                                     │
│  > defrag C: /O  // 최적화 포함                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 윈도우 표준 파일 시스템
• 구조: MFT (Master File Table) 중심
• 기능: 저널링, ACL, 압축, EFS, 쿼터
• 최대: 볼륨 256 TB, 파일 16 TB
• MFT: 파일당 1KB 엔트리, 속성 기반
• 리눅스: ntfs-3g로 읽기/쓰기
• 관리: fsutil, compact, defrag
• 시스템 파일: $MFT, $LogFile, $Boot 등
```
