# 463. 저널링 파일 시스템 (Journaling File System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 쓰기 작업을 저널에 먼저 기록
> 2. **가치**: 빠른 복구와 데이터 무결성
> 3. **융합**: ext4, XFS, fsck와 연관

---

## Ⅰ. 개요

### 개념 정의

저널링 파일 시스템(Journaling File System)은 **모든 변경 사항을 저널(로그)에 먼저 기록한 후 실제 데이터에 적용하는 파일 시스템**이다.

### 💡 비유: 장부
저널링 파일 시스템은 **이중 장부**와 같다. 거래를 임시 장부(저널)에 먼저 적고, 확정 후 정식 장부(파일 시스템)에 옮긴다.

### 저널링 파일 시스템 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                저널링 파일 시스템                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【저널링 동작 과정】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  전통적 파일 시스템 (비저널링):                                          │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  쓰기 요청 → 메타데이터 변경 → 데이터 쓰기                       │    │ │
│  │  │                    ↓                                     │    │ │
│  │  │           [중단 시 손상 위험]                                 │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  저널링 파일 시스템:                                                    │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                                                         │    │ │
│  │  │  1. 시작    2. 저널 기록    3. 실제 쓰기    4. 커밋          │    │ │
│  │  │     │           │              │             │           │    │ │
│  │  │     ▼           ▼              ▼             ▼           │    │ │
│  │  │  ┌─────┐    ┌──────┐      ┌──────┐     ┌──────┐        │    │ │
│  │  │  │요청 │ → │저널에 │  →  │데이터 │  → │저널  │        │    │ │
│  │  │  │    │    │기록   │      │쓰기   │     │커밋  │        │    │ │
│  │  │  └─────┘    └──────┘      └──────┘     └──────┘        │    │ │
│  │  │                                                         │    │ │
│  │  │  [중단 시 저널로 복구 가능]                                   │    │ │
│  │  │                                                         │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【저널 구조】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │                    저널 (Journal)                         │    │ │
│  │  │  ┌─────────┬─────────┬─────────┬─────────┐               │    │ │
│  │  │  │트랜잭션 1│트랜잭션 2│트랜잭션 3│   ...   │               │    │ │
│  │  │  │헤더      │헤더      │헤더      │         │               │    │ │
│  │  │  │데이터    │데이터    │데이터    │         │               │    │ │
│  │  │  │커밋      │커밋      │커밋      │         │               │    │ │
│  │  │  └─────────┴─────────┴─────────┴─────────┘               │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  각 트랜잭션: 헤더 + 데이터 + 커밋                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                저널링 상세                                             │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【저널링 모드】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. data=ordered (기본):                                             │ │
│  │     • 메타데이터만 저널링                                               │ │
│  │     • 데이터는 먼저 쓰고 메타데이터 저널링                                │ │
│  │     • 성능과 안정성 균형                                                │ │
│  │                                                             │ │
│  │  2. data=writeback:                                                  │ │
│  │     • 메타데이터만 저널링                                               │ │
│  │     • 데이터 쓰기 순서 보장 없음                                        │ │
│  │     • 가장 빠름, 약간 덜 안전                                           │ │
│  │                                                             │ │
│  │  3. data=journal:                                                    │ │
│  │     • 메타데이터 + 데이터 모두 저널링                                    │ │
│  │     • 가장 안전하지만 가장 느림                                          │ │
│  │     • 중요 데이터에 적합                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주요 저널링 파일 시스템】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  파일 시스템   개발사      특징                                          │ │
│  │  ──────────   ──────      ────                                          │ │
│  │  ext3/ext4    Linux       ext2 + 저널링, 널리 사용                      │ │
│  │  XFS          SGI         고성능, 대용량 파일 지원                        │ │
│  │  JFS          IBM         저널링, 고성능                                 │ │
│  │  ReiserFS     Namesys     저널링, 효율적 공간 사용                       │ │
│  │  btrfs        Linux       COW, 스냅샷, 체크섬                            │ │
│  │  NTFS         Microsoft   Windows, 저널링 지원                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【저널링 장단점】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  장점:                                                               │
│  │  • 빠른 복구: 저널 리플레이만으로 복구                                   │ │
│  │  • 데이터 무결성: 일관성 보장                                           │ │
│  │  • fsck 시간 단축: 전체 스캔 불필요                                      │ │
│  │                                                             │ │
│  │  단점:                                                               │
│  │  • 약간의 성능 저하: 중복 쓰기                                          │ │
│  │  • 공간 오버헤드: 저널 공간 필요                                        │ │
│  │  • 복잡도 증가: 구현이 복잡                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【저널링 설정 확인】                                                │
│  ──────────────────                                                  │
│  // ext4 저널 모드 확인                                               │
│  $ cat /proc/mounts | grep ext4                                     │
│  /dev/sda1 / ext4 rw,relatime,data=ordered 0 0                      │
│                                                                     │
│  // tune2fs로 저널 정보 확인                                           │
│  $ sudo tune2fs -l /dev/sda1 | grep -i journal                      │
│  Filesystem features:      has_journal                              │
│  Journal inode:            8                                        │
│  Journal backup:           inode blocks                            │
│                                                                     │
│  【저널 모드 변경】                                                    │
│  ──────────────────                                                  │
│  // data=writeback으로 변경 (성능 우선)                                 │
│  $ sudo mount -o remount,data=writeback /                           │
│                                                                     │
│  // data=journal로 변경 (안전 우선)                                      │
│  $ sudo mount -o remount,data=journal /                             │
│                                                                     │
│  // /etc/fstab에 영구 설정                                              │
│  /dev/sda1  /  ext4  defaults,data=writeback  0  1                  │
│                                                                     │
│  【저널 크기 조정】                                                    │
│  ──────────────────                                                  │
│  // 저널 크기 확인                                                      │
│  $ sudo tune2fs -l /dev/sda1 | grep "Journal size"                  │
│  Journal size:             128M                                    │
│                                                                     │
│  // 저널 크기 변경 (파일시스템 재생성 필요)                               │
│  $ sudo mkfs.ext4 -J size=256 /dev/sda1                            │
│                                                                     │
│  【저널 비활성화】                                                      │
│  ──────────────────                                                  │
│  // 저널 제거 (성능 향상, 위험)                                           │
│  $ sudo tune2fs -O ^has_journal /dev/sda1                           │
│                                                                     │
│  // 저널 다시 추가                                                       │
│  $ sudo tune2fs -O has_journal /dev/sda1                           │
│  $ sudo e2fsck -f /dev/sda1                                        │
│                                                                     │
│  【XFS 저널 확인】                                                      │
│  ──────────────────                                                  │
│  $ xfs_info /dev/sda2                                                │
│  log      =internal log           bsize=4096   blocks=2560           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 변경 사항을 저널에 먼저 기록
• 구조: 트랜잭션 = 헤더 + 데이터 + 커밋
• 모드: ordered(기본), writeback(빠름), journal(안전)
• 장점: 빠른 복구, 무결성, fsck 단축
• 단점: 성능 오버헤드, 공간 필요
• 종류: ext4, XFS, JFS, btrfs, NTFS
• 확인: tune2fs -l, /proc/mounts
```
