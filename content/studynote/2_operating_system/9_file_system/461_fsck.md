# 461. fsck (File System Check)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 시스템 무결성 검사 도구
> 2. **가치**: 파일 시스템 오류 탐지와 복구
> 3. **융합**: 슈퍼블록, 아이노드와 연관

---

## Ⅰ. 개요

### 개념 정의

fsck(File System Check)은 **파일 시스템의 무결성을 검사하고 오류를 복구하는 시스템 도구**다.

### 💡 비유: 건물 안전 점검
fsck는 **건물 안전 점검**과 같다. 구조(슈퍼블록), 방(아이노드), 내용물(데이터)을 확인하고 문제를 수정한다.

### fsck 구조

```

│                fsck 개요                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 슈퍼블록 검사:                                                   │ │
│  │     • 파일 시스템 크기, 블록 수 검증                                   │ │
│  │     • 손상 시 백업 슈퍼블록으로 복구                                   │ │
│  │                                                             │ │
│  │  2. 아이노드 검사:                                                   │ │
│  │     • 아이노드 타입, 모드, 크기 검증                                   │ │
│  │     • 고아 아이노드 탐지                                              │ │
│  │     • 손상된 아이노드 표시                                            │ │
│  │                                                             │ │
│  │  3. 블록 검사:                                                       │ │
│  │     • 할당된 블록 추적                                                │ │
│  │     • 중복 할당 탐지                                                  │ │
│  │     • 미할당 블록 식별                                                │ │
│  │                                                             │ │
│  │  4. 디렉토리 검사:                                                   │ │
│  │     • 디렉토리 엔트리 무결성                                           │ │
│  │     • 순환 참조 탐지                                                  │ │
│  │     • 고아 파일 탐지                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【fsck 실행 시점】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  시스템 부팅                                            │    │ │
│  │  │  • 비정상 종료 후 자동 실행                                 │    │ │
│  │  │  • /etc/fstab의 pass 옵션에 따라                              │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  수동 실행                                              │    │ │
│  │  │  • 관리자가 명시적으로 실행                                 │    │ │
│  │  │  • 파일 시스템 손상 의심 시                                 │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  주의: 반드시 언마운트 상태에서 실행!                                   │ │
│  │                                                             │ │

└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                fsck 상세                                              │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【fsck 단계별 실행】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Pass 1: 블록과 크기 검사                                             │ │
│  │  • 모든 아이노드 순회                                                  │ │
│  │  • 블록 주소 범위 검증                                                │ │
│  │  • 파일 크기 일치 확인                                                │ │
│  │                                                             │ │
│  │  Pass 2: 디렉토리 구조 검사                                           │ │
│  │  • 디렉토리 엔트리 형식 확인                                           │ │
│  │  • 아이노드 번호 유효성                                                │ │
│  │  • 순환 참조 탐지                                                     │ │
│  │                                                             │ │
│  │  Pass 3: 디렉토리 연결 검사                                           │ │
│  │  • 모든 디렉토리가 연결됐는지 확인                                      │ │
│  │  • lost+found 디렉토리 생성                                           │ │
│  │                                                             │ │
│  │  Pass 4: 참조 수 검사                                                │ │
│  │  • 링크 수와 실제 참조 일치 확인                                       │ │
│  │  • 불일치 시 수정                                                     │ │
│  │                                                             │ │
│  │  Pass 5: 비트맵 검사                                                  │ │
│  │  • 블록 비트맵과 실제 비교                                             │ │
│  │  • 아이노드 비트맵 검사                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【fsck 발견 문제】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  문제 유형           설명                    조치                       │ │
│  │  ──────────         ────                    ────                      │ │
│  │  고아 아이노드       참조 없는 아이노드        lost+found로 이동         │ │
│  │  중복 블록          여러 파일의 같은 블록     복사 또는 삭제             │ │
│  │  잘못된 블록 수      실제와 불일치            수정                       │ │
│  │  디렉토리 손상       엔트리 오류              수정 또는 삭제             │ │
│  │  슈퍼블록 손상       메타데이터 오류          백업에서 복구             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【저널링 파일 시스템과 fsck】                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ext3/ext4 (저널링):                                                 │ │
│  │  • fsck 시간 대폭 단축                                                │ │
│  │  • 저널 리플레이만으로 복구                                            │ │
│  │  • 전체 검사는 옵션                                                    │ │
│  │                                                             │ │
│  │  ext2 (비저널링):                                                     │ │
│  │  • 전체 파일 시스템 스캔 필요                                          │ │
│  │  • 큰 파일 시스템에서 오래 걸림                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【fsck 기본 사용】                                                  │
│  ──────────────────                                                  │
│  // 읽기 전용 검사 (변경 없음)                                         │
│  $ sudo fsck -n /dev/sda1                                          │
│                                                                     │
│  // 자동 복구 (묻지 않고 수정)                                         │
│  $ sudo fsck -y /dev/sda1                                          │
│                                                                     │
│  // 미리보기 (어떤 수정이 필요한지)                                      │
│  $ sudo fsck -N /dev/sda1                                          │
│                                                                     │
│  // 강제 검사 (마운트된 파일시스템도)                                    │
│  $ sudo fsck -f /dev/sda1                                          │
│                                                                     │
│  【ext4 전용 e2fsck】                                                │
│  ──────────────────                                                  │
│  // 기본 검사                                                          │
│  $ sudo e2fsck /dev/sda1                                           │
│                                                                     │
│  // 백업 슈퍼블록으로 복구                                               │
│  $ sudo e2fsck -b 32768 /dev/sda1                                  │
│                                                                     │
│  // 백업 슈퍼블록 위치 확인                                               │
│  $ sudo dumpe2fs /dev/sda1 | grep -i "backup superblock"           │
│                                                                     │
│  // 손상 의심 시 심층 검사                                               │
│  $ sudo e2fsck -cc /dev/sda1  // 배드 블록 검사 포함                    │
│                                                                     │
│  【XFS 파일 시스템】                                                    │
│  ──────────────────                                                  │
│  // XFS 복구                                                          │
│  $ sudo xfs_repair /dev/sda2                                       │
│                                                                     │
│  // 읽기 전용 검사                                                       │
│  $ sudo xfs_repair -n /dev/sda2                                    │
│                                                                     │
│  // 강제 로그 제거                                                      │
│  $ sudo xfs_repair -L /dev/sda2                                    │
│                                                                     │
│  【부팅 시 fsck 설정】                                                │
│  ──────────────────                                                  │
│  // /etc/fstab의 pass 필드                                            │
│  # <device>  <mount>  <type>  <options>  <dump>  <pass>            │
│  /dev/sda1   /       ext4    defaults    0       1                 │
│  /dev/sda2   /home   ext4    defaults    0       2                 │
│  // pass=0: fsck 안 함                                                 │
│  │  // pass=1: 루트, 먼저 검사                                           │
│  // pass=2: 루트 이후 검사                                               │
│                                                                     │
│  // 최대 마운트 횟수 후 자동 fsck                                         │
│  $ sudo tune2fs -c 30 /dev/sda1   // 30회 마운트마다                   │
│  $ sudo tune2fs -i 1m /dev/sda1   // 1개월마다                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일 시스템 무결성 검사 도구
• 검사: 슈퍼블록, 아이노드, 블록, 디렉토리
• 단계: 5 Pass로 구성
• 문제: 고아 아이노드, 중복 블록, 손상
• 실행: 언마운트 상태에서 수행
• 옵션: -n(읽기), -y(자동), -f(강제)
• ext4: e2fsck 사용, 저널링으로 빠름
• XFS: xfs_repair 사용
```
