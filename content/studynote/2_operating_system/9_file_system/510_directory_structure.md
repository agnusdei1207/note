## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디렉터리(Directory)는 수많은 파일들을 체계적으로 분류하고 관리하기 위해 이름과 실제 파일 정보를 매핑해놓은 특수 파일이자 관리용 데이터 구조다.
> 2. **가치**: 단일 레벨에서 시작하여 계층적 트리 및 비순환 그래프 구조로 발전하며, 파일 이름 중복 해결(Naming), 효율적인 그룹화(Grouping), 고속 검색(Searching) 기능을 제공한다.
> 3. **융합**: 현대 OS는 디렉터리를 '경로명(Pathname)'이라는 논리적 주소로 추상화하여, 물리적 저장 위치에 관계없이 일관된 접근 인터페이스를 사용자에게 제공한다.

---

## Ⅰ. 디렉터리의 역할과 연산 (Role & Operations)

- **정의**: 심볼릭 파일 이름(Symbolic Name)을 해당 파일의 FCB(File Control Block)나 inode 번호로 변환해주는 '심볼 테이블'이다.
- **핵심 연산**:
  - **Search**: 특정 파일명을 찾아 해당 위치 정보 획득.
  - **Create**: 새 파일을 만들고 디렉터리에 항목 추가.
  - **Delete**: 디렉터리 항목 제거.
  - **List**: 디렉터리 내의 모든 파일 목록 출력.
  - **Rename**: 파일 이름 변경 및 메타데이터 갱신.

📢 **섹션 요약 비유**: 디렉터리는 "스마트폰의 연락처 앱"과 같다. 이름만 누르면 실제 전화번호(물리 주소)를 연결해주고, 그룹별로 나누어 관리하게 해주는 장치다.

---

## Ⅱ. 디렉터리 논리적 구조의 진화 (Evolution)

효율성과 유연성을 위해 디렉터리 구조는 점진적으로 발전해왔다.

### 1. 단일 레벨 디렉터리 (Single-level Directory)
- 모든 파일이 하나의 디렉터리에 존재. 파일명 중복이 불가능하며 관리가 어렵다.

### 2. 2단계 디렉터리 (Two-level Directory)
- 사용자별로 마스터 디렉터리(MFD)를 두고 그 아래 사용자 파일 디렉터리(UFD)를 구성. 사용자 간 이름 충돌은 없으나 공유가 어렵다.

### 3. 트리 구조 디렉터리 (Tree-structured Directory)
- 현대 OS의 표준. 임의의 깊이로 하위 디렉터리(Subdirectory)를 가질 수 있어 가장 유연하다.

### 4. 구조 시각화 (ASCII Diagram)

```text
 [ Root (/) ]
      │
      ├──── [ bin ] ─── ls, cd, cp (System Executables)
      │
      ├──── [ home ] ───┬─── [ user1 ] ─── a.txt, b.jpg
      │                 │
      │                 └─── [ user2 ] ─── [ project ] ─── src.c
      │
      └──── [ etc ] ─── config.sys
```

📢 **섹션 요약 비유**: 트리 구조는 "나무의 줄기와 가지"와 같다. 뿌리(Root)에서 시작해 큰 가지(폴더)를 치고, 그 끝에 나뭇잎(파일)이 달리는 자연스러운 분류 방식이다.

---

## Ⅲ. 경로명과 탐색 방식 (Pathnames & Search)

- **절대 경로 (Absolute Path)**: 루트(/)부터 시작하여 파일에 이르는 전체 경로 (예: `/home/user1/a.txt`).
- **상대 경로 (Relative Path)**: 현재 작업 디렉터리(CWD)를 기준으로 한 경로 (예: `./a.txt`).
- **탐색 최적화**: 경로가 길어지면 디스크 I/O가 늘어나므로, 자주 접근하는 디렉터리 정보는 메모리의 **디렉터리 항목 캐시(dentry cache)**에 저장하여 속도를 높인다.

📢 **섹션 요약 비유**: 경로는 "편지 봉투의 주소"와 같다. '서울특별시...'로 시작하는 절대 주소와, 옆집 친구에게 보내는 '옆집'이라는 상대 주소의 차이와 같다.

---

## Ⅳ. 비순환 그래프 구조와 공유 (Acyclic-graph Directory)

- **개념**: 트리 구조를 확장하여, 서로 다른 디렉터리가 동일한 파일이나 디렉터리를 공유할 수 있게 허용하는 구조다.
- **공유 방식**:
  - **하드 링크 (Hard Link)**: 여러 디렉터리 항목이 동일한 inode를 직접 가리킴.
  - **심볼릭 링크 (Symbolic Link)**: 파일의 경로명을 담고 있는 특수 파일을 만들어 간접적으로 가리킴.
- **주의점**: 순환(Cycle)이 발생하면 삭제 시 자원 회수가 영원히 안 될 수 있으므로, 링크 생성 시 순환 탐색 알고리즘을 사용한다.

📢 **섹션 요약 비유**: 비순환 그래프는 "공유 폴더의 바로가기"와 같다. 원본은 하나지만 여러 곳에서 접근할 수 있어 협업과 자원 절약에 유리하다.

---

## Ⅴ. 디렉터리의 물리적 구현 (Physical Implementation)

- **선형 리스트 (Linear List)**: 파일 이름을 순차적으로 저장. 구현은 쉽지만 검색 속도가 느림($O(n)$).
- **해시 테이블 (Hash Table)**: 파일 이름을 해시 값으로 변환하여 인덱스로 사용. 검색 속도가 매우 빠름($O(1)$).
- **B-Tree 계층**: 디렉터리 내 파일이 수만 개 이상일 때 성능을 유지하기 위해 사용된다(예: NTFS의 B-tree 인덱스).

📢 **섹션 요약 비유**: 물리적 구현은 "물류 창고의 선반 배치"와 같다. 물건을 그냥 쌓아두느냐(리스트), 번호를 매겨 최적의 동선으로 배치하느냐(해시)의 차이다.
