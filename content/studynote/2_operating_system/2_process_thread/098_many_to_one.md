# 098. 다대일 스레드 모델 (Many-to-One)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 사용자 스레드가 하나의 커널 스레드에 매핑
> 2. **가치**: 효율적 전환, 커널 독립, 낮은 오버헤드
> 3. **융합**: 사용자 수준 스레드, Green Threads와 연관

---

## Ⅰ. 개요

### 개념 정의

다대일(Many-to-One) 스레드 모델은 **여러 사용자 수준 스레드가 하나의 커널 스레드에 매핑되는 구조**다. 스레드 관리가 사용자 공간에서 이루어져 효율적이지만, 한 스레드 블록 시 전체가 블록된다.

### 💡 비유: 한 팀의 계약자
다대일 모델은 **한 팀의 계약자**와 같다. 팀원들(사용자 스레드) 여러 명이 하나의 계약(커널 스레드)으로 일한다. 한 팀원이 아프면(블록) 계약 자체가 멈춘다.

## Ⅲ. 기대효과 및 결론

### 핵심 요약

```
• 모델: Many 사용자 스레드 → One 커널 스레드
• 장점: 빠른 전환, 커널 독립, 자원 절약
• 단점: 블록 시 전체 블록, 멀티코어 미활용
• 관리: 사용자 공간 라이브러리
• 예시: Green Threads, Python GIL
```
