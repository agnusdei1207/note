## Ⅰ. 개요

### 개념 정의

프로세스 상태(Process State)는 **프로세스가 실행되는 동안 거치는 여러 단계**로, New, Ready, Running, Waiting, Terminated 등이 있다.

### 💡 비유: 은행 업무
프로세스 상태는 **은행 업무**와 같다. 대기(New) → 번호표(Ready) → 창구(Running) → 서류 대기(Waiting) → 완료(Terminated).

```
│  │                     ┌─────────┐                               │ │
│  │                     │   New   │                               │ │
│  │                     │ (생성)  │                               │ │
│  │                     └────┬────┘                               │ │
│  │                          │ admit                              │ │
│  │                          ▼                                    │ │
│  │  ┌─────────┐  dispatch  ┌─────────┐  timeout  ┌─────────┐   │ │
│  │  │ Ready   │ ─────────► │ Running │ ────────► │ Ready   │   │ │
│  │  │ (준비)  │ ◄───────── │ (실행)  │ ◄──────── │ (준비)  │   │ │
│  │  └────┬────┘            └────┬────┘           └─────────┘   │ │
│  │       │                      │                               │ │
│  │       │                      │ I/O 요청                       │ │
│  │       │                      ▼                               │ │
│  │       │                 ┌─────────┐                           │ │
│  │       │                 │ Waiting │                           │ │
│  │       │                 │ (대기)  │                           │ │
│  │       │                 └────┬────┘                           │ │
│  │       │                      │ I/O 완료                        │ │
│  │       │                      ▼                                │ │
│  │       └────────────────► ┌─────────┐                          │ │
│  │                          │   Exit  │                          │ │
│  │                          │(종료)   │                          │ │
│  │                          └─────────┘                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상태별 설명】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  상태              설명                    큐                  │ │
│  │  ────                ────                    ────              │ │
│  │  New (생성)        프로세스 생성 중          -                 │ │
│  │  Ready (준비)      CPU 할당 대기             Ready Queue      │ │
│  │  Running (실행)    CPU에서 실행 중           -                 │ │
│  │  Waiting (대기)    I/O/이벤트 대기           Wait Queue       │ │
│  │  Terminated (종료) 실행 완료                -                 │ │
│  │                                                               │ │
```

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                프로세스 상태 상세                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【리눅스 프로세스 상태】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  상태              코드          설명                          │ │
│  │  ────                ────          ────                          │ │
│  │  RUNNING           R           실행 중/실행 가능                │ │
│  │  INTERRUPTIBLE     S           인터럽트 가능 대기               │ │
│  │  UNINTERRUPTIBLE   D           인터럽트 불가 대기               │ │
│  │  STOPPED           T           일시 중지                        │ │
│  │  TRACING           t           추적 중                          │ │
│  │  ZOMBIE            Z           좀비 (종료됨, 부모 wait 안 함)     │ │
│  │  DEAD              X           종료                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상태 전이 트리거】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  전이              트리거                                      │ │
│  │  ────                ────                                        │ │
│  │  Ready→Running    스케줄러 디스패치                              │ │
│  │  Running→Ready    타임 슬라이스 만료, 선점                        │ │
│  │  Running→Waiting  I/O 요청, 이벤트 대기                          │ │
│  │  Waiting→Ready    I/O 완료, 이벤트 발생                          │ │
│  │  Running→Terminated exit(), 정상/비정상 종료                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │

│  # STAT 컬럼 해석                                                    │
│  # R: Running, S: Sleeping, D: Disk sleep                          │
│  # T: Stopped, Z: Zombie, <: High priority                         │
│  # N: Low priority, L: Locked pages, s: Session leader             │
│  # +: Foreground, l: Multithreaded                                │

│  # top에서 상태 확인                                                 │
│  $ top                                                             │
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• New → Ready → Running → Waiting → Terminated
• Ready: CPU 할당 대기, Ready Queue
• Running: CPU 실행 중
• Waiting: I/O/이벤트 대기, Wait Queue
• Terminated: 실행 완료
• 리눅스: R, S, D, T, Z, X 상태
```
