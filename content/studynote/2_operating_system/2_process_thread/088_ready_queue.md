# 088. 준비 큐 (Ready Queue)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 할당을 대기하는 프로세스들이 모인 큐
> 2. **가치**: 스케줄링 기반, 우선순위, 공정성
> 3. **융합**: 스케줄러, 문맥 교환, 멀티 레벨 큐와 연관

---

## Ⅰ. 개요

### 개념 정의

준비 큐(Ready Queue)는 **CPU 할당을 기다리는 Ready 상태 프로세스들이 모인 큐**다. 스케줄러가 이 큐에서 다음 실행할 프로세스를 선택한다.

### 💡 비유: 은행 대기열
준비 큐는 **은행 대기열**과 같다. 번호표를 뽑은 사람(프로세스)들이 줄을 서서 창구(CPU)가 비기를 기다린다. 우대 고객(높은 우선순위)은 먼저 갈 수 있다.

### 준비 큐 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                준비 큐 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【단일 준비 큐】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  Ready Queue                                                  │ │
│  │  ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐           │ │
│  │  │ P1  │──►│ P2  │──►│ P3  │──►│ P4  │──►│ P5  │           │ │
│  │  └─────┘   └─────┘   └─────┘   └─────┘   └─────┘           │ │
│  │     │                                                    │ │
│  │     ▼                                                    │ │
│  │  ┌─────────┐                                             │ │
│  │  │   CPU   │ ◄── Scheduler dispatch                      │ │
│  │  └─────────┘                                             │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【멀티 레벨 큐】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────────┐   High Priority Queue                        │ │
│  │  │ Q0 (RT)     │   실시간 프로세스                              │ │
│  │  └──────┬──────┘                                              │ │
│  │         │                                                      │ │
│  │  ┌─────────────┐   Interactive Queue                          │ │
│  │  │ Q1 (Inter)  │   대화형 프로세스                              │ │
│  │  └──────┬──────┘                                              │ │
│  │         │                                                      │ │
│  │  ┌─────────────┐   Batch Queue                                │ │
│  │  │ Q2 (Batch)  │   일괄 처리 프로세스                            │ │
│  │  └──────┬──────┘                                              │ │
│  │         │                                                      │ │
│  │  ┌─────────────┐   Low Priority Queue                         │ │
│  │  │ Q3 (Low)    │   백그라운드 프로세스                          │ │
│  │  └─────────────┘                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                준비 큐 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【큐 구현 방식】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구현              설명                    복잡도              │ │
│  │  ────                ────                    ────              │ │
│  │  연결 리스트        단순, 유연             O(n) 검색         │ │
│  │  우선순위 큐        힙 기반                 O(log n) 삽입/삭제│ │
│  │  비트맵 + 리스트    O(1) 우선순위 찾기      Linux CFS         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux CFS 런큐】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct cfs_rq {                                               │ │
│  │      unsigned int nr_running;    // 실행 가능 태스크 수        │ │
│  │      u64 min_vruntime;           // 최소 가상 런타임           │ │
│  │      struct rb_root tasks_timeline;  // Red-Black 트리        │ │
│  │      struct rb_node *rb_leftmost;    // 가장 왼쪽 노드        │ │
│  │  };                                                            │ │
│  │                                                               │ │
│  │  • Red-Black 트리로 vruntime 기준 정렬                          │ │
│  │  • 가장 작은 vruntime이 leftmost                               │ │
│  │  • O(log n) 삽입/삭제                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 준비 큐: Ready 상태 프로세스 대기열
• 구현: 연결 리스트, 우선순위 큐, R-B 트리
• Linux CFS: Red-Black 트리 기반
• 멀티 레벨: 우선순위별 별도 큐
• 스케줄러: 큐에서 다음 프로세스 선택
```
