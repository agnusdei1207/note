# 112. 취소점 (Cancellation Point)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 지연식 스레드 취소가 실제로 수행되는 지점
> 2. **가치**: 안전한 취소, 자원 정리 보장
> 3. **융합**: 스레드 취소, 지연식 취소, POSIX와 연관

---

## Ⅰ. 개요

### 개념 정의

취소점(Cancellation Point)은 **지연식 취소(Deferred Cancellation)에서 스레드 취소가 실제로 처리되는 코드 지점**이다. 취소 요청이 들어와도 취소점에서만 실제 취소가 발생한다.

### 💡 비유: 정거장
취소점은 **버스 정거장**과 같다. 내리고 싶어도(취소 요청) 정거장(취소점)에서만 내릴 수 있다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                취소점 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【취소점 동작 과정】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. pthread_cancel() 호출                                      │ │
│  │  2. 취소 요청이 스레드에 전달                                   │ │
│  │  3. 스레드가 취소점 함수 호출                                   │ │
│  │  4. 취소 요청 확인                                              │ │
│  │  5. 정리 핸들러 실행 (등록된 순서 역순)                          │ │
│  │  6. 스레드 종료 (PTHREAD_CANCELED 반환)                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【취소점이 아닌 함수】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  함수                      이유                                │ │
│  │  ────                            ────                            │ │
│  │  pthread_mutex_lock()    대부분 구현에서 취소점 아님            │ │
│  │  malloc()/free()         구현 따라 다름                         │ │
│  │  순수 계산 코드           시스템 콜 없음                         │ │
│  │                                                               │ │
│  │  → 이런 경우 pthread_testcancel() 명시적 호출 필요              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【안전한 취소를 위한 팁】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 락 획득 전 취소 비활성화                                     │ │
│  │  • 락 해제 후 취소 재활성화                                     │ │
│  │  • cleanup handler로 락 해제 보장                               │ │
│  │  • 장시간 루프에 testcancel() 추가                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【명시적 취소점 추가】                                                │
│  ──────────────────                                                  │
│  void* compute_intensive(void* arg) {                               │
│      while (working) {                                              │
│          // 긴 계산 작업                                             │
│          for (int i = 0; i < 1000000; i++) {                       │
│              result += heavy_computation(i);                        │
│          }                                                          │
│                                                                     │
│          // 명시적 취소점                                            │
│          pthread_testcancel();                                      │
│      }                                                              │
│      return NULL;                                                   │
│  }                                                                  │
│                                                                     │
│  【락과 취소점 안전 패턴】                                              │
│  ──────────────────                                                  │
│  void safe_operation() {                                            │
│      // 취소 비활성화                                                │
│      pthread_setcancelstate(PTHREAD_CANCEL_DISABLE, &oldstate);    │
│                                                                     │
│      pthread_mutex_lock(&mutex);                                    │
│                                                                     │
│      // cleanup 핸들러 등록                                          │
│      pthread_cleanup_push(unlock_mutex, &mutex);                   │
│                                                                     │
│      // 취소 재활성화                                                │
│      pthread_setcancelstate(PTHREAD_CANCEL_ENABLE, NULL);          │
│                                                                     │
│      // 실제 작업 (취소점 포함 가능)                                  │
│      do_io_operation();                                             │
│                                                                     │
│      // cleanup 핸들러 제거 (실행 O)                                  │
│      pthread_cleanup_pop(1);                                        │
│                                                                     │
│      // 원래 상태 복원                                               │
│      pthread_setcancelstate(oldstate, NULL);                        │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 지연식 취소가 처리되는 지점
• 명시적: pthread_testcancel()
• 암시적: I/O, sleep, wait 함수들
• 안전: 자원 정리 보장
• 패턴: 락 전후로 취소 비활성화
• cleanup: 핸들러로 정리 보장
```
