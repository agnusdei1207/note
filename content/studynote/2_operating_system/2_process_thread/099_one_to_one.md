# 099. 일대일 스레드 모델 (One-to-One)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 각 사용자 스레드가 하나의 커널 스레드에 매핑
> 2. **가치**: 진정한 병렬성, 독립 블록, 멀티코어 활용
> 3. **융합**: 커널 수준 스레드, Linux NPTL, pthread와 연관

---

## Ⅰ. 개요

### 개념 정의

일대일(One-to-One) 스레드 모델은 **각 사용자 수준 스레드가 별도의 커널 스레드에 매핑되는 구조**다. 한 스레드가 블록되어도 다른 스레드가 계속 실행될 수 있으며, 멀티코어에서 진정한 병렬 실행이 가능하다.

### 💡 비유: 개별 계약자
일대일 모델은 **개별 계약자**와 같다. 각 계약자(사용자 스레드)가 별도의 계약(커널 스레드)을 맺는다. 한 계약자가 아파도(블록) 다른 계약자는 계속 일한다.

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  OS                  구현                    특징                │ │
│  │  ────                ────                    ────              │ │
│  │  Linux              NPTL                    1:1, 고성능        │ │
│  │  Windows            Windows Threads         1:1                │ │
│  │  macOS              pthreads                1:1                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux NPTL (Native POSIX Thread Library)】                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • Linux 2.6부터 기본 스레드 구현                                │ │
│  │  • clone() 시스템 콜 기반                                        │ │
│  │  • 각 스레드가 별도의 PID (TID)                                   │ │
│  │  • getpid()는 같은 값 반환 (프로세스 ID)                          │ │
│  │  • gettid()로 스레드 ID 확인                                     │ │
│  │                                                               │ │
│  │  # NPTL 버전 확인                                                │ │
│  │  $ getconf GNU_LIBPTHREAD_VERSION                              │ │
│  │  $ /lib/x86_64-linux-gnu/libc.so.6                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스레드 확인】                                                      │
│  ──────────────────                                                  │
│  # 프로세스의 스레드 목록                                              │
│  $ ps -T -p <pid>                                                   │
│  $ ls /proc/<pid>/task                                             │
│                                                                     │
│  # 스레드 ID 확인                                                     │
│  $ cat /proc/<pid>/stat | awk '{print $1}'                          │
│                                                                     │
│  # top에서 스레드 보기                                                │
│  $ top -H -p <pid>                                                  │
│                                                                     │
│  【pthread 사용】                                                     │
│  ──────────────────                                                  │
│  # Linux pthread은 1:1 모델                                          │
│  pthread_create() → clone() 시스템 콜 → 새 커널 스레드              │
│                                                                     │
│  # 스레드 수 확인                                                     │
│  $ cat /proc/<pid>/status | grep Threads                          │
│                                                                     │
│  # 스레드별 CPU 사용량                                                 │
│  $ top -H                                                          │
│  $ pidstat -t -p <pid> 1                                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 모델: One 사용자 스레드 → One 커널 스레드
• 장점: 병렬성, 독립 블록, 멀티코어
• 단점: 생성/전환 오버헤드
• Linux: NPTL (Native POSIX Thread Library)
• Windows: Windows Threads
• macOS: pthreads
```
