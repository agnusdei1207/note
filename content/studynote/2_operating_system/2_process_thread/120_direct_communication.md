# 120. 직접 통신 (Direct Communication)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 수신자/송신자를 명시하여 직접 메시지 전달
> 2. **가치**: 명확한 통신 대상, 자동 링크 생성
> 3. **융합**: 간접 통신, 메시지 전달, 소켓과 연관

---

## Ⅰ. 개요

### 개념 정의

직접 통신(Direct Communication)은 **메시지를 보내거나 받을 상대방 프로세스를 명시적으로 지정**하는 통신 방식이다. 송신자는 수신자를 지정하고, 수신자는 송신자를 지정한다.

### 💡 비유: 직접 전화
직접 통신은 **직접 전화**와 같다. 전화를 걸 상대방(수신자)의 전화번호(프로세스 ID)를 직접 입력한다.

### 직접 통신 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                직접 통신 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【직접 통신 동작】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  Process A (P)                    Process B (Q)              │ │
│  │  ┌─────────────┐                 ┌─────────────┐             │ │
│  │  │             │                 │             │             │ │
│  │  │ send(Q, msg)│ ──────────────► │ receive(P,  │             │ │
│  │  │             │                 │    msg)     │             │ │
│  │  │ 수신자 지정 │                 │ 송신자 지정 │             │ │
│  │  └─────────────┘                 └─────────────┘             │ │
│  │                                                             │ │
│  │  링크 특성:                                                    │ │
│  │  • 자동 생성 (프로세스 쌍마다)                                  │ │
│  │  • 정확히 하나의 링크                                          │ │
│  │  • 양방향 통신 가능                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【직접 통신 프리미티브】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  대칭형:                                                        │ │
│  │  send(P, message)     - 프로세스 P에게 메시지 전송               │ │
│  │  receive(Q, message)  - 프로세스 Q로부터 메시지 수신             │ │
│  │                                                               │ │
│  │  비대칭형:                                                      │ │
│  │  send(P, message)     - 프로세스 P에게 메시지 전송               │ │
│  │  receive(id, message) - 아무 프로세스로부터 수신, id 반환        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 구현 단순               • 프로세스 ID 알아야 함               │ │
│  │  • 명확한 통신 대상        • 모듈화 제한                        │ │
│  │  • 자동 링크 생성          • 확장성 낮음                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│                                                                     │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 자동으로 링크 생성                                           │ │

│  │                                                               │ │

│  │  • 소켓 통신: connect()로 상대 지정                             │ │
│  │  • MPI: MPI_Send(dest, ...)                                   │ │
│  │  • Unix Domain Socket: 경로로 상대 지정                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【소켓을 이용한 직접 통신】                                            │
│  ──────────────────                                                  │
│  // 송신측 (클라이언트)                                                │
│  int sock = socket(AF_INET, SOCK_STREAM, 0);                       │
│  struct sockaddr_in server;                                         │
│  server.sin_addr.s_addr = inet_addr("192.168.1.1");               │
│  server.sin_port = htons(8080);                                    │
│                                                                     │
│  connect(sock, &server, sizeof(server));  // 직접 상대 지정          │
│  send(sock, message, len, 0);                                       │
│                                                                     │
│  // 수신측 (서버)                                                      │
│  struct sockaddr_in client;                                         │
│  int client_sock = accept(sock, &client, &len);                    │
│  recv(client_sock, buffer, len, 0);                                │
│                                                                     │
│  【MPI 직접 통신】                                                      │
│  ──────────────────                                                  │
│  // 프로세스 0에서 프로세스 1로 전송                                   │
│  MPI_Send(buffer, count, MPI_INT, 1, tag, comm);                   │
│                                                                     │
│  // 프로세스 1에서 프로세스 0으로부터 수신                              │
│  MPI_Recv(buffer, count, MPI_INT, 0, tag, comm, &status);          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 상대방 프로세스 명시적 지정

```
