# 116. 협력적/독립적 프로세스 (Cooperating/Independent Process)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다른 프로세스와 상호작용 여부에 따른 분류
> 2. **가치**: IPC 필요성, 동기화 요구사항 결정
> 3. **융합**: IPC, 공유 메모리, 메시지 전달과 연관

---

## Ⅰ. 개요

### 개념 정의

협력적 프로세스(Cooperating Process)는 **다른 프로세스와 데이터를 교환하거나 영향을 주고받는 프로세스**다. 독립적 프로세스(Independent Process)는 **다른 프로세스와 상호작용하지 않고 독립적으로 실행**되는 프로세스다.

### 💡 비유: 혼자 일하기 vs 팀워크
독립적 프로세스는 **혼자 일하는 직원**, 협력적 프로세스는 **팀워크로 일하는 직원**과 같다. 혼자 일하면 소통이 필요 없지만, 팀이면 소통이 필수다.

│  │                                                             │ │

│  │  ┌─────────────────┐        ┌─────────────────┐              │ │
│  │  │                 │        │ Process A       │              │ │
│  │  │   Process       │        │      │          │              │ │
│  │  │   (격리)        │        │      ▼          │              │ │

│  │  │  • 자원 독립    │        │  │   IPC   │    │              │ │

│  │  │                 │        │       │         │              │ │
│  │  │                 │        │       ▼         │              │ │

│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【특징 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  항목              독립적 프로세스    협력적 프로세스           │ │
│  │  ────                    ────                  ────             │ │
│  │  데이터 공유       없음                있음 (IPC 필요)          │ │
│  │  실행 영향         없음                서로 영향                 │ │
│  │  동기화            불필요              필요                      │ │
│  │  IPC               불필요              필요                      │ │
│  │  예시              계산 작업           클라이언트-서버           │ │

│                                                                     │
│  【협력적 프로세스의 이유】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  이유                      예시                                │ │
│  │  ────                        ────                                │ │
│  │  정보 공유                파일 공유, DB                        │ │
│  │  계산 속도 향상           병렬 처리                            │ │
│  │  모듈성                  마이크로서비스                        │ │
│  │  편의성                  파이프, 필터                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

## Ⅲ. 실무 적용

### 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【협력적 프로세스: 쉘 파이프라인】                                      │
│  ──────────────────                                                  │
│  # 세 개의 협력적 프로세스                                            │
│  $ ls -la | grep ".txt" | wc -l                                    │
│                                                                     │
│  Process 1 (ls) → Process 2 (grep) → Process 3 (wc)                │
│       │                  │                  │                       │
│       └──── 파이프 ──────┘                  │                       │
│                            └──── 파이프 ─────┘                       │
│                                                                     │
│  【협력적 프로세스: 생산자-소비자】                                      │
│  ──────────────────                                                  │
│  ┌─────────────┐         ┌─────────────┐                           │
│  │  Producer   │ ──────► │  Consumer   │                           │
│  │             │  공유    │             │                           │
│  │  데이터 생성 │  버퍼    │  데이터 소비 │                           │
│  └─────────────┘         └─────────────┘                           │
│                                                                     │
│  【독립적 프로세스 확인】                                               │
│  ──────────────────                                                  │
│  # IPC 사용 없는 프로세스                                             │
│  $ lsof -p <pid> | grep -E "socket|pipe|shm"                        │
│                                                                     │
│  # 격리된 프로세스                                                    │
│  $ ls /proc/<pid>/fd                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 독립적: 다른 프로세스와 상호작용 없음
• 협력적: 데이터 교환, 상호 영향
• 독립적: IPC 불필요, 동기화 불필요
• 협력적: IPC 필요, 동기화 필요
• 협력 이유: 공유, 속도, 모듈성, 편의성
• 예시: 파이프, 클라이언트-서버
```
