# 096. 사용자 수준 스레드 (User-level Thread)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자 공간 라이브러리에서 관리되는 스레드
> 2. **가치**: 빠른 전환, 커널 독립, 유연한 스케줄링
> 3. **융합**: 커널 수준 스레드, Many-to-One, LWP와 연관

---

## Ⅰ. 개요

### 개념 정의

사용자 수준 스레드(User-level Thread)는 **커널의 지원 없이 사용자 공간의 스레드 라이브러리가 관리하는 스레드**다. 커널은 스레드의 존재를 모르고, 하나의 커널 스레드에 여러 사용자 스레드가 매핑된다.

### 💡 비유: 매니저와 직원
사용자 수준 스레드는 **매니저가 직원을 관리**하는 것과 같다. 회사(커널)는 매니저(커널 스레드)만 알고, 직원들(사용자 스레드)은 모른다. 매니저가 직원을 배치한다.

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【사용자 수준 스레드 사용 예시】                                      │
│  ──────────────────                                                  │
│  # Python (GIL - Global Interpreter Lock)                           │
│  # Python 스레드는 사용자 수준에서 관리                               │
│  import threading                                                   │
│  t = threading.Thread(target=func)                                 │
│  t.start()                                                          │
│                                                                     │
│  # Node.js (이벤트 루프 - 단일 스레드)                                │
│  # 비동기 I/O로 사용자 수준에서 태스크 전환                            │
│  async function task() { ... }                                      │
│                                                                     │
│  # Go (Goroutine - M:N 모델)                                        │
│  # Go 런타임이 사용자 수준에서 스케줄링                               │
│  go func() { ... }()                                                │
│                                                                     │
│  【스레드 모델 확인】                                                 │
│  ──────────────────                                                  │
│  # pthreads 설정 확인                                                │
│  $ getconf GNU_LIBPTHREAD_VERSION                                  │
│                                                                     │
│  # 스레드 라이브러리 확인                                             │
│  $ ldd /bin/ls | grep pthread                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
