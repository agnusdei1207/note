# 107. 프로세스 종료 (exit, wait)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: exit()로 종료, wait()로 동기화하는 정리 과정
> 2. **가치**: 자원 회수, 상태 전달, 좀비 방지
> 3. **융합**: 부모/자식 관계, 좀비 프로세스, 시그널과 연관

---

## Ⅰ. 개요

### 개념 정의

프로세스 종료는 **exit() 시스템 콜로 프로세스가 종료하고, 부모가 wait()로 종료 상태를 회수**하는 과정이다. 자원 정리와 부모-자식 동기화를 수행한다.

### 💡 비유: 퇴근과 인계
프로세스 종료는 **퇴근과 인계**와 같다. 직원(exit)이 퇴근하며 업무 보고(종료 상태)를 남기고, 관리자(wait)가 보고를 확인(회수)한다.

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【exit() 사용】                                                      │
│  ──────────────────                                                  │
│  #include <stdlib.h>                                                │
│                                                                     │
│  void cleanup() {                                                   │
│      printf("정리 작업 수행\n");                                      │
│  }                                                                  │
│                                                                     │
│  int main() {                                                       │
│      atexit(cleanup);  // 종료 시 호출                              │
│      exit(0);          // 정상 종료                                  │
│  }                                                                  │
│                                                                     │
│  【waitpid() 사용】                                                   │
│  ──────────────────                                                  │
│  pid_t pid = fork();                                                │
│  if (pid == 0) {                                                    │
│      // 자식                                                         │
│      exit(42);                                                      │
│  } else {                                                           │
│      // 부모                                                         │
│      int status;                                                    │
│      waitpid(pid, &status, 0);                                      │
│                                                                     │
│      if (WIFEXITED(status)) {                                       │
│          printf("종료 코드: %d\n", WEXITSTATUS(status));             │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【비차단 대기】                                                       │
│  ──────────────────                                                  │
│  // WNOHANG: 종료 안 했으면 즉시 반환                                 │
│  if (waitpid(pid, &status, WNOHANG) == 0) {                        │
│      printf("자식 아직 실행 중\n");                                   │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• exit(): 프로세스 종료 요청
• wait(): 자식 종료 대기 및 상태 회수
• 상태: 0-255 (정상), 시그널 (비정상)
• 과정: 자원 해제 → 좀비 → wait() → PCB 삭제
• 매크로: WIFEXITED, WEXITSTATUS 등
• 좀비: wait() 안 하면 좀비 프로세스
```
