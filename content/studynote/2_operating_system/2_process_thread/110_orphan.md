# 110. 고아 프로세스 (Orphan Process)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 부모가 먼저 종료된 자식 프로세스
> 2. **가치**: init에 입양되어 정상 종료 가능
> 3. **융합**: 좀비 프로세스, 부모/자식, init과 연관

---

## Ⅰ. 개요

### 개념 정의

고아 프로세스(Orphan Process)는 **부모 프로세스가 먼저 종료되어 부모가 없는 자식 프로세스**다. init 프로세스(PID 1) 또는 systemd가 입양하여 관리한다.

### 💡 비유: 고아
고아 프로세스는 **부모를 잃은 아이**와 같다. 보호자(init)가 새로 입양하여 돌본다.

### 고아 프로세스 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                고아 프로세스 구조                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【고아 프로세스 발생 전】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  부모 프로세스 (PID 100)         자식 프로세스 (PID 101)       │ │
│  │  ┌─────────────┐                ┌─────────────┐             │ │
│  │  │ Running     │                │ Running     │             │ │
│  │  │ PPID = 50   │                │ PPID = 100  │             │ │
│  │  └─────────────┘                └─────────────┘             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【고아 프로세스 발생 후】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  부모 (종료됨)                    자식 → 고아                 │ │
│  │  ┌─────────────┐                ┌─────────────┐             │ │
│  │  │   (없음)    │                │ Running     │             │ │
│  │  │             │   입양         │ PPID = 1    │             │ │
│  │  │             │ ◄───────────── │ (init)      │             │ │
│  │  └─────────────┘                └─────────────┘             │ │
│  │                                                             │ │
│  │  init/systemd (PID 1)                                        │ │
│  │  ┌─────────────┐                                            │ │
│  │  │ 새 부모     │                                            │ │
│  │  │ wait() 호출 │                                            │ │
│  │  └─────────────┘                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【고아 vs 좀비】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분            고아 프로세스         좀비 프로세스            │ │
│  │  ────                    ────                   ────            │ │
│  │  상태            실행 중                종료됨                  │ │
│  │  부모            종료됨                  실행 중                │ │
│  │  PPID            1 (init)               원래 부모              │ │
│  │  CPU 사용        계속 사용              0%                     │ │
│  │  문제            없음 (정상)            PID 낭비               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                고아 프로세스 상세                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【입양 과정】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 부모 프로세스 종료                                         │ │
│  │  2. 커널가 자식 프로세스들의 PPID를 1로 변경                    │ │
│  │  3. init/systemd가 새 부모가 됨                                │ │
│  │  4. 자식 종료 시 init가 wait() 호출                            │ │
│  │  5. 좀비 상태 방지                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【고아 프로세스 발생 시나리오】                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시나리오                    설명                               │ │
│  │  ────                        ────                                │ │
│  │  부모 비정상 종료           segfault, SIGKILL                   │ │
│  │  부모 정상 종료             exit() 호출, wait() 안 함            │ │
│  │  데몬 생성                  fork() 후 부모 exit()               │ │
│  │  백그라운드 실행            &                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【의도적 고아 생성 (데몬화)】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  pid_t pid = fork();                                          │ │
│  │  if (pid > 0) {                                                │ │
│  │      exit(0);  // 부모 종료 → 자식은 고아                       │ │
│  │  }                                                             │ │
│  │  // 자식은 init에 입양되어 데몬으로 실행                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【고아 프로세스 확인】                                                │
│  ──────────────────                                                  │
│  # PPID가 1인 실행 중인 프로세스                                      │
│  $ ps -ef | awk '$3 == 1 {print}'                                   │
│  $ ps -o pid,ppid,cmd --ppid 1                                      │
│                                                                     │
│  # 고아 프로세스 찾기 (실행 중 + PPID=1)                              │
│  $ ps -eo pid,ppid,stat,cmd | awk '$2 == 1 && $3 !~ /Z/'           │
│                                                                     │
│  【데몬화 절차】                                                        │
│  ──────────────────                                                  │
│  void daemonize() {                                                 │
│      // 1. fork 후 부모 종료                                         │
│      if (fork() > 0) exit(0);                                       │
│                                                                     │
│      // 2. 새 세션 생성                                              │
│      setsid();                                                      │
│                                                                     │
│      // 3. 작업 디렉터리 변경                                         │
│      chdir("/");                                                    │
│                                                                     │
│      // 4. umask 설정                                                │
│      umask(0);                                                      │
│                                                                     │
│      // 5. 파일 디스크립터 닫기                                       │
│      close(STDIN_FILENO);                                           │
│      close(STDOUT_FILENO);                                          │
│      close(STDERR_FILENO);                                          │
│  }                                                                  │
│                                                                     │
│  【init vs systemd】                                                  │
│  ──────────────────                                                  │
│  # init (전통적)                                                      │
│  $ ps -p 1 -o comm  # "init"                                        │
│                                                                     │
│  # systemd (현대적)                                                   │
│  $ ps -p 1 -o comm  # "systemd"                                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 상태: 부모 종료, 자식 실행 중
• PPID: 1 (init/systemd)
• 입양: init이 새 부모가 됨
• vs 좀비: 고아는 실행 중, 좀비는 종료
• 용도: 데몬 생성에 의도적 활용
• 문제: 없음 (정상적인 상태)
```
