# 109. 좀비 프로세스 (Zombie Process)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 종료되었으나 부모가 wait()하지 않은 프로세스
> 2. **가치**: 종료 상태 보존, 부모 알림 역할
> 3. **융합**: exit/wait, 고아 프로세스, PCB와 연관

---

## Ⅰ. 개요

### 개념 정의

좀비 프로세스(Zombie Process)는 **자식 프로세스가 exit()로 종료되었으나, 부모 프로세스가 wait()를 호출하지 않아 PCB가 남아있는 상태**다. 좀비는 실제로 실행되지 않고 PCB만 차지한다.

### 💡 비유: 퇴사 처리 안 된 직원
좀비 프로세스는 **퇴사했는데 인사 기록에 남아있는 직원**과 같다. 실제로는 회사에 없지만(실행 안 함), 기록(PCB)은 남아있다.

### 좀비 프로세스 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                좀비 프로세스 구조                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【좀비 프로세스 생성 과정】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  부모 프로세스               자식 프로세스                    │ │
│  │  ┌─────────────┐            ┌─────────────┐                 │ │
│  │  │ Running     │            │ Running     │                 │ │
│  │  │             │   fork()   │             │                 │ │
│  │  │ (wait 안 함)│ ◄───────── │             │                 │ │
│  │  └─────────────┘            └──────┬──────┘                 │ │
│  │                                    │                         │ │
│  │                                    │ exit()                  │ │
│  │                                    ▼                         │ │
│  │                             ┌─────────────┐                  │ │
│  │                             │   Zombie    │                  │ │
│  │                             │ (defunct)   │                  │ │
│  │                             │             │                  │ │
│  │                             │ • PCB 유지  │                  │ │
│  │                             │ • 종료 상태 │                  │ │
│  │                             │ • 실행 X    │                  │ │
│  │                             │ • 메모리 X   │                  │ │
│  │                             └─────────────┘                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【좀비 상태 특징】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  항목                      상태                                │ │
│  │  ────                        ────                                │ │
│  │  CPU 사용                 0%                                   │ │
│  │  메모리                   해제됨                                │ │
│  │  파일 디스크립터          닫힘                                  │ │
│  │  PCB                     유지 (종료 상태 포함)                  │ │
│  │  ps 출력                 <defunct> 또는 Z                      │ │
│  │  kill 영향               없음 (이미 종료)                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【좀비 해결 방법】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 부모가 wait() 호출                                        │ │
│  │  2. 부모가 SIGCHLD 처리                                       │ │
│  │  3. 부모 종료 → init이 입양 후 wait()                          │ │
│  │  4. SIGCHLD를 SIG_IGN으로 설정 (자동 제거)                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
│  ──────────────────                                                  │

│  │  • PID (다른 프로세스가 사용 불가)                              │ │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • PID 고갈 (좀비가 많으면 새 프로세스 생성 불가)               │ │
│  │  • 프로세스 테이블 슬롯 낭비                                     │ │
│  │  • 시스템 리소스 누수                                            │ │

│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【좀비 방지 패턴】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 방법 1: SIGCHLD 무시                                       │ │
│  │  signal(SIGCHLD, SIG_IGN);                                     │ │
│  │                                                               │ │
│  │  // 방법 2: SIGCHLD 핸들러에서 wait                             │ │
│  │  void sigchld_handler(int sig) {                               │ │
│  │      while (waitpid(-1, NULL, WNOHANG) > 0);                  │ │
│  │  }                                                             │ │
│  │  signal(SIGCHLD, sigchld_handler);                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【좀비 프로세스 확인】                                                │
│  ──────────────────                                                  │
│  # 좀비 프로세스 찾기                                                 │
│  $ ps aux | grep Z                                                   │
│  $ ps -eo pid,ppid,stat,cmd | grep Z                                │
│                                                                     │
│  # 좀비 프로세스 개수                                                  │
│  $ ps aux | awk '$8 ~ /Z/ {print}' | wc -l                         │
│                                                                     │
│  # top에서 좀비 보기                                                   │
│  $ top  # zombie count 확인                                          │
│                                                                     │
│  【좀비 제거 방법】                                                     │
│  ──────────────────                                                  │
│  # 좀비은 kill로 죽일 수 없음 (이미 죽음)                              │
│  # 방법 1: 부모에게 SIGCHLD 전송                                      │
│  $ kill -CHLD <parent_pid>                                          │
│                                                                     │
│  # 방법 2: 부모 프로세스 종료                                         │
│  $ kill <parent_pid>                                                │
│  # → init이 입양 후 wait() 호출                                      │
│                                                                     │
│  【좀비 사냥 스크립트】                                                  │
│  ──────────────────                                                  │
│  # 좀비의 부모 찾기                                                   │
│  $ ps -o ppid= -p <zombie_pid>                                      │
│                                                                     │
│  # 부모에게 SIGCHLD 보내기                                            │
│  $ for z in $(ps aux | awk '$8 ~ /Z/ {print $2}'); do              │
│        ppid=$(ps -o ppid= -p $z)                                    │
│        kill -CHLD $ppid                                             │
│    done                                                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 상태: 종료 + PCB 유지
• 원인: 부모가 wait() 안 함
• 특징: CPU 0%, 메모리 해제, 실행 X
• 문제: PID 고갈, 슬롯 낭비
• 해결: wait(), SIGCHLD, 부모 종료
• 방지: signal(SIGCHLD, SIG_IGN)
```
