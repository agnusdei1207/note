# 097. 커널 수준 스레드 (Kernel-level Thread)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 운영체제 커널이 직접 관리하는 스레드
> 2. **가치**: 진정한 병렬성, 블록 독립성, 멀티코어 활용
> 3. **융합**: 사용자 수준 스레드, One-to-One, SMP와 연관

---

## Ⅰ. 개요

### 개념 정의

커널 수준 스레드(Kernel-level Thread)는 **운영체제 커널이 직접 생성, 관리, 스케줄링하는 스레드**다. 각 커널 스레드는 독립적으로 스케줄링되며, 멀티코어에서 진정한 병렬 실행이 가능하다.

### 💡 비유: 독립 계약자
커널 수준 스레드는 **독립 계약자**와 같다. 회사(커널)가 각 계약자(스레드)를 직접 고용하고 관리한다. 한 계약자가 일을 못해도 다른 계약자는 계속 일한다.

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【커널 스레드 확인】                                                  │

│  # 커널 스레드 목록                                                    │
│  $ ps aux | grep '\['                                                │
│  $ top -H                                                             │
│                                                                     │
│  # 특정 커널 스레드                                                    │
│  $ ps -C kswapd, ksoftirqd -o pid,comm                              │
│                                                                     │
│  # 스레드별 CPU 사용량                                                 │
│  $ top -H -p <pid>                                                   │
│                                                                     │
│  【pthread으로 커널 스레드 생성】                                       │
│  ──────────────────                                                  │
│  # Linux pthread는 1:1 모델 사용                                      │
│  pthread_t tid;                                                       │
│  pthread_create(&tid, NULL, func, NULL);                            │
│                                                                     │
│  # 생성된 커널 스레드 확인                                             │
│  $ ls /proc/<pid>/task                                               │
│  $ cat /proc/<pid>/status | grep Threads                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 커널 수준: OS가 직접 관리하는 스레드
• 모델: One-to-One (사용자:커널 = 1:1)
• 장점: 병렬성, 독립 블록, 멀티코어
• 단점: 생성/전환 오버헤드, 자원 소모
• Linux: NPTL (Native POSIX Thread Library)
• 예시: kswapd, ksoftirqd, migration
```
