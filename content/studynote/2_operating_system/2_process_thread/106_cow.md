# 106. Copy-on-Write (COW)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 쓰기 시점까지 메모리 복사를 지연하는 최적화 기법
> 2. **가치**: fork() 성능 향상, 메모리 절약, 효율적 자원 사용
> 3. **융합**: fork/exec, 가상 메모리, 페이지 보호와 연관

---

## Ⅰ. 개요

### 개념 정의

Copy-on-Write(COW)는 **여러 프로세스가 동일한 물리 메모리를 공유하다가, 쓰기 작업이 발생할 때만 복사**하는 최적화 기법이다. fork() 시스템 콜의 성능을 획기적으로 개선한다.

### 💡 비유: 공문서
COW는 **공문서**와 같다. 모든 사람이 원본을 읽을 수 있다. 누군가 수정하려면 그때 복사본을 만들어서 수정한다.

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. fork() 호출                                               │ │
│  │     - 부모의 페이지 테이블 복사                                 │ │

│  │  3. 쓰기 접근                                                  │ │
│  │     - 페이지 폴트 발생                                         │ │

│  │     - 원본 내용 복사                                           │ │

│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  항목                      전통적 fork    COW fork             │ │

│  │  메모리 복사              전체 복사       필요시만 복사          │ │
│  │  fork() 시간              O(n)           O(1)                  │ │
│  │  메모리 사용              2배            필요시만 증가           │ │
│  │  exec() 즉시 호출         낭비           최소화                 │ │

## Ⅲ. 실무 적용

### 시스템 확인
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【COW 확인】                                                         │
│  ──────────────────                                                  │
│  # fork() 전후 메모리 비교                                           │
│  $ cat /proc/<pid>/smaps | grep -E "^(Private|Shared)"              │
│                                                                     │
│  # 페이지 공유 상태 확인                                              │
│  $ cat /proc/<pid>/maps                                             │
│  $ pmap -x <pid>                                                    │
│                                                                     │
│  【COW 활용 사례】                                                     │
│  ──────────────────                                                  │
│  • fork() + exec(): 자식이 exec() 호출 시 복사 불필요                 │
│  • 스레드: 스레드는 주소 공간 공유 (COW 불필요)                        │
│  • 파일 시스템: Btrfs, ZFS 스냅샷                                    │
│  • 메모리 맵 파일: 읽기 전용 공유                                     │
│                                                                     │
│  【관련 커널 파라미터】                                                │
│  ──────────────────                                                  │
│  # COW 관련 설정                                                     │
│  $ cat /proc/sys/vm/overcommit_memory                               │
│  # 0: 휴리스틱, 1: 항상 허용, 2: 엄격                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 쓰기 시점까지 복사 지연
• 동작: Read-Only 공유 → 쓰기 시 복사
• 장점: fork() 빠름, 메모리 절약
• 효과: fork()+exec() 패턴 최적화
• 구현: 페이지 테이블 + 페이지 폴트
• 활용: ZFS, Btrfs 스냅샷
```
