# 081. 프로그램 vs 프로세스

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로그램은 정적 코드, 프로세스는 실행 중인 인스턴스
> 2. **가치**: 메모리 적재, 실행 상태, 자원 할당의 차이
> 3. **융합**: PCB, 메모리 구조, 스케줄링과 연관

---

## Ⅰ. 개요

### 개념 정의

**프로그램(Program)**은 디스크에 저장된 정적인 실행 가능 코드와 데이터다. **프로세스(Process)**는 메모리에 적재되어 실행 중인 프로그램 인스턴스다.

### 💡 비유: 요리법 vs 요리 중
프로그램은 **요리책에 적힌 레시피**다. 프로세스는 **실제 요리 중인 냄비**다. 레시피는 하나지만 여러 번 요리할 수 있다.

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프로세스 확인】                                                    │
│  ──────────────────                                                  │

│  $ ps aux                                                          │
│  $ ps -ef                                                          │

│  # 프로세스 트리                                                    │
│  $ pstree                                                          │
│                                                                     │

│  $ top                                                             │

│  $ ./program &                                                     │
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 프로그램: 정적 코드, 디스크 저장, 자원 불필요
• 프로세스: 동적 인스턴스, 메모리 적재, 자원 할당
• 구조: Text, Data, BSS, Heap, Stack
• PCB: PID, 상태, 레지스터, 스케줄링 정보
• 1개 프로그램 → 여러 프로세스 가능
```
