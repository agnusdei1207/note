# 087. 프로세스 상태 전이 (State Transition)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스가 한 상태에서 다른 상태로 변화하는 과정
> 2. **가치**: 스케줄링 기반, 자원 관리, 이벤트 기반 전이
> 3. **융합**: 스케줄러, 인터럽트, I/O와 연관

---

## Ⅰ. 개요

### 개념 정의

프로세스 상태 전이(State Transition)는 **프로세스가 실행 중 한 상태에서 다른 상태로 변화하는 과정**이다. 스케줄러, 인터럽트, I/O 완료 등이 전이를 유발한다.

### 💡 비유: 신호등
상태 전이는 **신호등**과 같다. 빨강(Ready) → 초록(Running) → 노랑(Waiting)으로 상태가 바뀐다. 타이머와 센서가 전이를 결정한다.

### 상태 전이 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                상태 전이 구조                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【전이 유형】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  ┌─────────┐                                                   │ │
│  │  │  New    │                                                   │ │
│  │  └────┬────┘                                                   │ │
│  │       │ ① fork() → admit                                       │ │
│  │       ▼                                                        │ │
│  │  ┌─────────┐  ② scheduler dispatch  ┌─────────┐               │ │
│  │  │  Ready  │ ──────────────────────► │ Running │               │ │
│  │  │         │ ◄───────────────────── │         │               │ │
│  │  └────┬────┘  ③ timeout/preemption  └────┬────┘               │ │
│  │       │                                  │                     │ │
│  │       │                                  │ ④ I/O request       │ │
│  │       │                                  ▼                     │ │
│  │       │                             ┌─────────┐                │ │
│  │       │                             │ Waiting │                │ │
│  │       │                             │         │                │ │
│  │       │                             └────┬────┘                │ │
│  │       │                                  │                     │ │
│  │       │  ⑤ I/O completion               │                     │ │
│  │       ◄──────────────────────────────────┘                     │ │
│  │       │                                                        │ │
│  │       │  ⑥ exit()                                              │ │
│  │       ▼                                                        │ │
│  │  ┌─────────┐                                                   │ │
│  │  │Terminated│                                                  │ │
│  │  └─────────┘                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【전이 원인】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  전이              원인                                        │ │
│  │  ────                ────                                        │ │
│  │  New→Ready        프로세스 생성 완료                              │ │
│  │  Ready→Running    CPU 스케줄링 (dispatch)                        │ │
│  │  Running→Ready    타임 슬라이스 만료, 선점                        │ │
│  │  Running→Waiting  I/O 요청, wait(), sleep()                      │ │
│  │  Waiting→Ready    I/O 완료, signal, 이벤트                       │ │
│  │  Running→Terminated exit(), 정상/비정상 종료                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                상태 전이 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【전이와 스케줄러】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  스케줄러           관여 전이                                    │ │
│  │  ────                ────                                        │ │
│  │  장기 스케줄러     New → Ready (다중 프로그래밍 정도)             │ │
│  │  단기 스케줄러     Ready ↔ Running (CPU 스케줄링)               │ │
│  │  중기 스케줄러     Ready ↔ Suspended (스와핑)                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【선점형 vs 비선점형】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분              선점형                비선점형                │ │
│  │  ────                ────                ────                    │ │
│  │  Running→Ready    강제 가능           불가능                    │ │
│  │  전이 시점        타이머, 우선순위      자발적 yield만           │ │
│  │  응답성            좋음                 나쁨                      │ │
│  │  구현 복잡도       높음                 낮음                      │ │
│  │  예시              Linux, Windows       초기 UNIX               │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• New→Ready: 프로세스 생성
• Ready→Running: 스케줄러 디스패치
• Running→Ready: 선점, 타임아웃
• Running→Waiting: I/O 요청
• Waiting→Ready: I/O 완료
• Running→Terminated: exit()
```
