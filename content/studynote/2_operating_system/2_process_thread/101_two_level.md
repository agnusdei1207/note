# 101. Two-level 스레드 모델 (Two-level Model)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다대다 모델과 일대일 모델의 혼합 구조
> 2. **가치**: 유연성과 성능의 최적 균형, 바인딩 옵션 제공
> 3. **융합**: Many-to-Many, LWP, Solaris와 연관

---

## Ⅰ. 개요

### 개념 정의

Two-level 스레드 모델은 **다대다(Many-to-Many) 매핑과 일대일(One-to-One) 바인딩을 혼합한 구조**다. 일반적인 사용자 스레드는 여러 커널 스레드를 공유하지만, 특정 스레드는 전용 커널 스레드에 바인딩할 수 있다.

### 💡 비유: 혼합 계약
Two-level 모델은 **혼합 계약**과 같다. 대부분의 팀원(사용자 스레드)은 공유 계약(커널 스레드)으로 일하지만, 중요한 팀원은 전용 계약(바인딩)을 맺는다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                Two-level 모델 상세                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【운영체제별 구현】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  OS                  구현                    특징                │ │
│  │  ────                ────                    ────              │ │
│  │  Solaris            LWP + 바인딩             Two-level         │ │
│  │  IRIX               sprocs                   Two-level         │ │
│  │  Tru64 UNIX         pthreads                 Two-level         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【바인딩(Bound) 스레드】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 특정 사용자 스레드 → 전용 커널 스레드                         │ │
│  │  • 실시간 처리, 높은 우선순위 작업에 적합                        │ │
│  │  • 다른 스레드의 블록에 영향 없음                                │ │
│  │  • Solaris: pthread_attr_setscope(PTHREAD_SCOPE_SYSTEM)       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【언바운드(Unbound) 스레드】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 여러 사용자 스레드 → 공유 커널 스레드 풀                      │ │
│  │  • 경량, 효율적인 스레드 생성                                   │ │
│  │  • 일반적인 작업에 적합                                         │ │
│  │  • Solaris: pthread_attr_setscope(PTHREAD_SCOPE_PROCESS)      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### Solaris 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Solaris Two-level 모델】                                          │
│  ──────────────────                                                  │
│  # 바인딩 스레드 생성 (1:1)                                          │
│  pthread_attr_t attr;                                               │
│  pthread_attr_init(&attr);                                          │
│  pthread_attr_setscope(&attr, PTHREAD_SCOPE_SYSTEM);               │
│  pthread_create(&tid, &attr, func, NULL);                          │
│                                                                     │
│  # 언바운드 스레드 생성 (M:N)                                        │
│  pthread_attr_setscope(&attr, PTHREAD_SCOPE_PROCESS);              │
│  pthread_create(&tid, &attr, func, NULL);                          │
│                                                                     │
│  【사용 시나리오】                                                    │
│  ──────────────────                                                  │
│  바인딩 스레드:                                                      │
│    • 실시간 오디오/비디오 처리                                       │
│    • 높은 우선순위 인터럽트 처리                                     │
│    • CPU 집약적 계산 작업                                            │
│                                                                     │
│  언바운드 스레드:                                                    │
│    • 일반적인 I/O 작업                                               │
│    • 대량의 경량 태스크                                              │
│    • 블로킹 가능성이 높은 작업                                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 모델: Many-to-Many + One-to-One 혼합
• 장점: 유연한 바인딩, 성능 최적화
• 단점: 구현 복잡, 관리 오버헤드
• 바인딩: 전용 커널 스레드 (실시간)
• 언바운드: 공유 커널 스레드 (일반)
• 예시: Solaris LWP
```
