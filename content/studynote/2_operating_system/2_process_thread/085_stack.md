# 085. 스택 (Stack) 영역

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 함수 호출 시 지역 변수와 복귀 주소를 저장하는 LIFO 메모리
> 2. **가치**: 빠른 할당/해제, 자동 관리, 함수 호출 지원
> 3. **융합**: 스택 프레임, 호출 규약, 버퍼 오버플로우와 연관

---

## Ⅰ. 개요

### 개념 정의

스택(Stack) 영역은 **함수 호출 시 지역 변수, 매개변수, 복귀 주소를 저장하는 LIFO(Last-In First-Out) 메모리 영역**이다. 함수 진입 시 자동 할당, 반환 시 자동 해제된다.

### 💡 비유: 접시 쌓기
스택은 **접시 쌓기**와 같다. 나중에 쌓은 접시(함수 호출)가 먼저 나간다. 접시를 꺼내면(함수 반환) 아래 접시가 보인다.

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스택 크기 확인/변경】                                              │
│  ──────────────────                                                  │
│  # 스택 크기 제한 확인                                                │
│  $ ulimit -s                                                        │
│  $ ulimit -a                                                        │
│                                                                     │
│  # 스택 크기 변경                                                    │
│  $ ulimit -s 8192                                                   │
│                                                                     │
│  【스택 사용 확인】                                                   │
│  ──────────────────                                                  │
│  # 스택 주소 확인                                                     │
│  $ cat /proc/<pid>/maps | grep stack                               │
│                                                                     │
│  # 백트레이스                                                        │
│  $ gdb -p <pid>                                                     │
│  (gdb) bt                                                           │
│                                                                     │
│  【스택 오버플로우 디버깅】                                           │
│  ──────────────────                                                  │
│  # AddressSanitizer                                                 │
│  $ gcc -fsanitize=address -fstack-protector-all program.c          │
│                                                                     │
│  # Stack Protector                                                  │
│  $ gcc -fstack-protector-strong program.c                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 스택: LIFO 구조의 함수 호출 메모리

• 성장: 하향 (↓) 방향
• 관리: 자동 할당/해제
• 레지스터: SP, BP, IP
• 문제: 스택 오버플로우
```
