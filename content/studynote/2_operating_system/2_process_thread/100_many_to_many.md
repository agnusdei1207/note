# 100. 다대다 스레드 모델 (Many-to-Many)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 사용자 스레드가 여러 커널 스레드에 매핑
> 2. **가치**: 유연성, 병렬성, 효율성의 균형
> 3. **융합**: Two-level, LWP, 스케줄러 액티베이션과 연관

---

## Ⅰ. 개요

### 개념 정의

다대다(Many-to-Many) 스레드 모델은 **여러 사용자 수준 스레드가 여러 커널 스레드에 유연하게 매핑되는 구조**다. 다대일의 효율성과 일대일의 병렬성을 결합한 하이브리드 모델이다.

### 💡 비유: 팀과 계약
다대다 모델은 **여러 팀이 여러 계약**을 맺는 것과 같다. 팀원(사용자 스레드)들이 여러 계약(커널 스레드)을 나눠서 수행한다. 한 팀원이 아파도 다른 계약은 계속 진행된다.

## Ⅲ. 기대효과 및 결론

### 핵심 요약

```
• 모델: Many 사용자 스레드 → Many 커널 스레드
• 장점: 병렬성과 효율성 균형, 유연성
• 단점: 구현 복잡, 커널 수정 필요
• 예시: Solaris LWP, Go Goroutines
• 변형: Two-level (M:N + 1:1 혼합)
```
