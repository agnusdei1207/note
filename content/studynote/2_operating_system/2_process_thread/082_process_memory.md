# 082. 프로세스 메모리 구조

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스의 논리적 메모리 영역 (Text, Data, BSS, Heap, Stack)
> 2. **가치**: 메모리 보호, 효율적 할당, 세그먼트 관리
> 3. **융합**: 가상 메모리, 페이지, 세그먼트와 연관

---

## Ⅰ. 개요

### 개념 정의

프로세스 메모리 구조는 **프로세스가 사용하는 논리적 메모리 영역**으로, Text(Code), Data, BSS, Heap, Stack으로 구성된다.

### 💡 비유: 건물 층별 용도
프로세스 메모리는 **건물의 층별 용도**와 같다. 지하(BSS), 1층(Text), 2층(Data), 옥탕(Heap, Stack) 등 용도별로 구역이 나뉘어 있다.

### 프로세스 메모리 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                프로세스 메모리 구조                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메모리 레이아웃】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  높은 주소                                                    │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  Stack (스택)                                           │ │ │
│  │  │  • 지역 변수, 매개변수, 리턴 주소                          │ │ │
│  │  │  • 함수 호출 시 생성, 반환 시 소멸                          │ │ │
│  │  │  • ↓ 성장 (하향)                                        │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                           │                                   │ │
│  │                           ▼                                   │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  ↑                                                      │ │ │
│  │  │  Heap (힙)                                              │ │ │
│  │  │  • 동적 메모리 할당 (malloc, new)                          │ │ │
│  │  │  • ↑ 성장 (상향)                                        │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                           │                                   │ │
│  │                           ▼                                   │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  BSS (Block Started by Symbol)                          │ │ │
│  │  │  • 초기화되지 않은 전역/정적 변수                          │ │ │
│  │  │  • 0으로 초기화                                          │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                           │                                   │ │
│  │                           ▼                                   │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  Data (데이터)                                           │ │ │
│  │  │  • 초기화된 전역/정적 변수                                 │ │ │
│  │  │  • 읽기/쓰기 가능                                        │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                           │                                   │ │
│  │                           ▼                                   │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  Text (코드)                                             │ │ │
│  │  │  • 실행 가능 기계어 코드                                  │ │ │
│  │  │  • 읽기 전용, 공유 가능                                    │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │  낮은 주소                                                    │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• Text: 실행 코드, 읽기 전용, 공유 가능
• Data: 초기화된 전역/정적 변수
• BSS: 초기화되지 않은 변수, 0으로 초기화
• Heap: 동적 할당, 상향 성장
• Stack: 지역 변수, 함수 호출, 하향 성장
• Stack과 Heap은 반대 방향으로 성장
```
