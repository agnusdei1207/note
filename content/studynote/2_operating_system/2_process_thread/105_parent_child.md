# 105. 부모/자식 프로세스 (Parent/Child Process)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: fork()로 생성된 프로세스 간 계층적 관계
> 2. **가치**: 자원 상속, 종료 동기화, 프로세스 트리
> 3. **융합**: fork/exec, 좀비/고아 프로세스, wait()와 연관

---

## Ⅰ. 개요

### 개념 정의

부모/자식 프로세스는 **fork() 시스템 콜로 생성된 프로세스 간의 계층적 관계**다. 부모 프로세스가 자식 프로세스를 생성하며, 자식은 부모의 PPID(Parent Process ID)를 가진다.

### 💡 비유: 가족 관계
부모/자식 프로세스는 **가족 관계**와 같다. 부모(부모 프로세스)가 자식(자식 프로세스)을 낳고, 자식은 부모의 성(PPID)을 따른다.

### 부모/자식 프로세스 관계

```
┌─────────────────────────────────────────────────────────────────────┐
│                부모/자식 프로세스 관계                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프로세스 트리】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │                      init (PID 1)                            │ │
│  │                         │                                    │ │
│  │            ┌────────────┼────────────┐                      │ │
│  │            │            │            │                      │ │
│  │         systemd      sshd        bash                      │ │
│  │            │            │            │                      │ │
│  │            │            │      ┌────┴────┐                  │ │
│  │            │            │      │         │                  │ │
│  │         daemon      sshd      ls       grep                 │ │
│  │                     세션      (자식)   (자식)                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【부모-자식 관계】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  부모 프로세스               자식 프로세스                     │ │
│  │  ────────────               ────────────                      │ │
│  │  • fork() 호출자            • fork()로 생성                   │ │
│  │  • 자식 PID 반환            • 0 반환                          │ │
│  │  • wait()로 종료 대기       • exit()로 종료                   │ │
│  │  • 자원 상속源              • 자원 상속받음                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【관계 특성】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성                      설명                               │ │
│  │  ────                        ────                               │ │
│  │  PPID                     부모의 PID                           │ │
│  │  자원 공유                 파일, 메모리 공유 가능               │ │
│  │  독립 실행                 별도의 주소 공간                     │ │
│  │  종료 동기화               wait()로 자식 종료 확인              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【프로세스 트리 확인】                                                │
│  ──────────────────                                                  │
│  # pstree로 프로세스 트리 보기                                        │
│  $ pstree -p                                                        │
│  $ pstree -p $$      # 현재 쉘 기준                                  │
│                                                                     │
│  # PPID 확인                                                         │
│  $ ps -ef | grep <process>                                          │
│  $ ps -o pid,ppid,comm -p <pid>                                     │
│                                                                     │
│  【부모-자식 관계 확인】                                               │
│  ──────────────────                                                  │
│  # 자식 프로세스 찾기                                                 │
│  $ pgrep -P <parent_pid>                                            │
│                                                                     │
│  # 부모 프로세스 찾기                                                 │
│  $ ps -o ppid= -p <child_pid>                                       │
│                                                                     │
│  【init/systemd 프로세스】                                            │
│  ──────────────────                                                  │
│  # PID 1 확인                                                        │
│  $ ps -p 1 -o pid,comm                                              │
│  # 고아 프로세스는 init이 입양                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 관계: fork()로 생성된 계층 구조
• 부모: fork() 호출자, 자식 PID 반환
• 자식: fork() 결과, 0 반환
• 상속: 파일, 환경변수, UID/GID
• 독립: PID, 메모리, 파일 오프셋
• 루트: init (PID 1)
```
