# 093. 스레드 자원 공유

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 같은 프로세스의 스레드들이 주소 공간을 공유
> 2. **가치**: 메모리 절약, 빠른 통신, 효율적 자원 사용
> 3. **융합**: 동기화, 락, 경합 조건과 연관

---

## Ⅰ. 개요

### 개념 정의

스레드 자원 공유는 **같은 프로세스에 속한 스레드들이 코드, 데이터, 힙, 열린 파일 등의 자원을 공유**하는 것을 의미한다. 이는 메모리 효율성을 높이지만 동기화 문제를 야기한다.

### 💡 비유: 룸메이트
스레드 자원 공유는 **룸메이트 생활**과 같다. 거실, 주방, 화장실(공유 자원)은 같이 쓴다. 각자 방(스택)은 따로 쓴다. 누가 화장실을 쓰는지 확인해야 한다.

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【공유 메모리 확인】                                                 │
│  ──────────────────                                                  │
│  # 프로세스 메모리 맵                                                 │
│  $ cat /proc/<pid>/maps                                            │
│                                                                     │
│  # 스레드별 메모리                                                    │
│  $ pmap -x <pid>                                                    │

│                                                                     │
│  # Helgrind (valgrind)                                              │
│  $ valgrind --tool=helgrind ./program                              │
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 공유: Code, Data, Heap, Files, Signals
• 독립: Stack, Registers, PC, TLS, errno

• 문제: 경합 조건, 데드락, 가시성
• 해결: Mutex, Semaphore, Atomic
```
