# 102. 암묵적 스레딩 (Implicit Threading)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 스레드 생성과 관리를 컴파일러/런타임이 자동 처리
> 2. **가치**: 개발자 부담 감소, 동시성 버그 최소화
> 3. **융합**: Thread Pool, OpenMP, GCD와 연관

---

## Ⅰ. 개요

### 개념 정의

암묵적 스레딩(Implicit Threading)은 **스레드 생성, 관리, 동기화를 개발자가 아닌 컴파일러나 런타임 라이브러리가 자동으로 처리**하는 방식이다. 명시적 스레딩의 복잡성을 추상화하여 동시성 프로그래밍을 단순화한다.

### 💡 비유: 자동 기어
암묵적 스레딩은 **자동 기어 자동차**와 같다. 운전자(개발자)가 기어를 직접 조작하지 않아도, 자동으로 최적의 기어(스레드)가 선택된다.

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【OpenMP 예시】                                                     │
│  ──────────────────                                                  │
│  #pragma omp parallel for                                          │
│  for (int i = 0; i < n; i++) {                                     │
│      array[i] = compute(i);                                        │
│  }                                                                  │
│  // 컴파일러가 자동으로 스레드 분할                                   │
│                                                                     │
│  【GCD (Grand Central Dispatch)】                                    │
│  ──────────────────                                                  │
│  // Swift                                                           │
│  DispatchQueue.global().async {                                     │
│      // 백그라운드 작업                                              │
│      let result = heavyComputation()                               │
│      DispatchQueue.main.async {                                     │
│          // UI 업데이트                                              │
│          updateUI(result)                                           │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Java Fork-Join】                                                  │
│  ──────────────────                                                  │
│  ForkJoinPool pool = new ForkJoinPool();                           │
│  pool.submit(() -> {                                                │
│      // 병렬 스트림 처리                                              │
│      list.parallelStream().forEach(this::process);                 │
│  });                                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
