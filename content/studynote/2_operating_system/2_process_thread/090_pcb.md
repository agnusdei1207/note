# 090. PCB (Process Control Block)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스의 모든 정보를 담은 커널 데이터 구조
> 2. **가치**: 문맥 교환, 스케줄링, 자원 관리의 핵심
> 3. **융합**: task_struct, 문맥 교환, 스케줄러와 연관

---

## Ⅰ. 개요

### 개념 정의

PCB(Process Control Block)는 **운영체제가 프로세스를 관리하기 위해 유지하는 프로세스의 모든 정보를 담은 데이터 구조**다. 리눅스에서는 task_struct로 구현된다.

### 💡 비유: 환자 차트
PCB는 **병원 환자 차트**와 같다. 환자(프로세스)의 이름, 나이, 상태, 약물, 검사 결과(모든 정보)가 기록된다. 의사(OS)가 차트를 보고 진료한다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                PCB 상세                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【리눅스 task_struct 주요 필드】                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  필드              설명                                        │ │
│  │  ────                ────                                        │ │
│  │  pid               프로세스 ID                                   │ │
│  │  state             프로세스 상태                                  │ │
│  │  prio              우선순위                                      │ │
│  │  mm                메모리 관리 정보                               │ │
│  │  fs                파일시스템 정보                                │ │
│  │  files             열린 파일 테이블                               │ │
│  │  signal            시그널 핸들러                                  │ │
│  │  thread_info       스레드 정보 (SP, PC 등)                        │ │
│  │  cpus_mask         CPU 친화성                                    │ │
│  │  utime/stime       사용자/커널 CPU 시간                           │ │
│  │  start_time        시작 시간                                      │ │
│  │  real_parent       실제 부모 프로세스                              │ │
│  │  children          자식 프로세스 리스트                           │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【PCB vs TCB】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분              PCB                   TCB                   │ │
│  │  ────                ────                  ────                 │ │
│  │  대상              프로세스              스레드                  │ │
│  │  리눅스            task_struct           task_struct (동일)     │ │
│  │  공유              독립 자원              Code, Data, Files     │ │
│  │  독립              -                    SP, PC, Registers     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【PCB 정보 확인】                                                   │
│  ──────────────────                                                  │
│  # /proc에서 PCB 정보                                                │
│  $ cat /proc/<pid>/status                                          │
│  $ cat /proc/<pid>/stat                                            │
│  $ cat /proc/<pid>/statm                                           │
│                                                                     │
│  # 특정 정보만 추출                                                  │
│  $ cat /proc/<pid>/status | grep -E "Pid|State|Threads"           │
│                                                                     │
│  # 메모리 정보                                                       │
│  $ cat /proc/<pid>/maps                                            │
│  $ cat /proc/<pid>/smaps                                           │
│                                                                     │
│  # 파일 디스크립터                                                   │
│  $ ls -la /proc/<pid>/fd                                           │
│  $ ls -la /proc/<pid>/fd | wc -l                                   │
│                                                                     │
│  【PCB 크기】                                                        │
│  ──────────────────                                                  │
│  # task_struct 크기 (커널 버전마다 다름)                              │
│  # 대략 5-10KB 정도                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• PCB: 프로세스의 모든 정보를 담은 구조체
• 구성: PID, 상태, 레지스터, 메모리, I/O, 회계
• 리눅스: task_struct로 구현
• TCB: 스레드용 PCB (리눅스에서는 동일)
• 위치: 커널 메모리 영역
```
