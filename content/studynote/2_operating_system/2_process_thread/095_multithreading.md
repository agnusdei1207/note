# 095. 멀티스레딩 장점

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 스레드로 응답성, 자원 공유, 경제성, 확장성 확보
> 2. **가치**: 성능 향상, 자원 효율, 병렬 처리
> 3. **융합**: SMP, 병렬 처리, 비동기 I/O와 연관

---

## Ⅰ. 개요

### 개념 정의

멀티스레딩 장점은 **하나의 프로세스에서 여러 스레드를 실행하여 얻는 이점**으로, 응답성, 자원 공유, 경제성, 다중 처리기 활용이 있다.

### 💡 비유: 요리 팀
멀티스레딩은 **요리 팀**과 같다. 요리사 혼자(단일 스레드) 하면 느리지만, 요리사 여러 명(멀티스레드)이 나눠서 하면 빠르다. 같은 주방(메모리)을 쓴다.

### 멀티스레딩 장점 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                멀티스레딩 장점 구조                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【4대 장점】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. 응답성 (Responsiveness)                                    │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 한 스레드가 블록되어도 다른 스레드 계속 실행            │ │ │
│  │  │  • UI 스레드와 작업 스레드 분리                           │ │ │
│  │  │  • 사용자 경험 향상                                        │ │ │
│  │  │                                                           │ │ │
│  │  │  예: 웹 브라우저                                           │ │ │
│  │  │  Thread 1: UI 렌더링                                      │ │ │
│  │  │  Thread 2: 네트워크 다운로드                               │ │ │
│  │  │  Thread 3: JavaScript 실행                                 │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  │  2. 자원 공유 (Resource Sharing)                               │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 같은 주소 공간 공유                                     │ │ │
│  │  │  • 프로세스 간 통신(IPC) 불필요                            │ │ │
│  │  │  • 메모리 절약                                             │ │ │
│  │  │                                                           │ │ │
│  │  │  비교:                                                     │ │ │
│  │  │  멀티프로세스: 각각 독립 메모리 → 메모리 낭비              │ │ │
│  │  │  멀티스레드: 메모리 공유 → 효율적                          │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  │  3. 경제성 (Economy)                                          │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 스레드 생성이 프로세스 생성보다 빠름                     │ │ │
│  │  │  • 스레드 전환이 프로세스 전환보다 빠름                     │ │ │
│  │  │  • 더 적은 시스템 자원 사용                                 │ │ │
│  │  │                                                           │ │ │
│  │  │  비교:                                                     │ │ │
│  │  │  프로세스 생성: 수 ms                                       │ │ │
│  │  │  스레드 생성: 수 μs (마이크로초)                            │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  │  4. 다중 처리기 활용 (Scalability)                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  • 멀티코어 CPU에서 병렬 처리                               │ │ │
│  │  │  • CPU 활용률 극대화                                        │ │ │
│  │  │  • 처리량 증가                                              │ │ │
│  │  │                                                           │ │ │
│  │  │  8코어 시스템:                                             │ │ │
│  │  │  단일 스레드: 1코어만 사용 (12.5% 활용)                     │ │ │
│  │  │  8 스레드: 8코어 모두 사용 (100% 활용)                       │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 응답성: 스레드 블록 시에도 계속 실행
• 자원 공유: 메모리 공유, IPC 불필요
• 경제성: 빠른 생성/전환, 적은 자원
• 확장성: 멀티코어 활용, 병렬 처리
• 비교: 멀티스레드 > 멀티프로세스 (효율)
```
