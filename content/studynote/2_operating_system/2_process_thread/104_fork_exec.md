# 104. 프로세스 생성 (fork, exec)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: fork()로 복제, exec()로 교체하는 두 단계 생성
> 2. **가치**: 유연한 프로세스 생성, 코드 재사용
> 3. **융합**: 부모/자식 프로세스, COW, 쉘 구현과 연관

## Ⅰ. 개요

### 개념 정의

프로세스 생성은 **fork() 시스템 콜로 부모 프로세스를 복제하고, exec()로 새로운 프로그램으로 교체**하는 두 단계 과정이다. Unix/Linux의 핵심 프로세스 생성 메커니즘이다.

### 💡 비유: 복사와 바꾸기
프로세스 생성은 **문서 복사 후 내용 바꾸기**와 같다. 먼저 원본 문서(부모)를 복사(fork)한 다음, 새 내용(exec)으로 덮어쓴다.

### fork()와 exec() 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                fork()와 exec() 구조                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【fork() 시스템 콜】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  부모 프로세스                자식 프로세스                   │ │
│  │  ────────────                ────────────                    │ │
│  │  ┌─────────────┐            ┌─────────────┐                 │ │
│  │  │ PID = 100   │            │ PID = 101   │                 │ │
│  │  │ PPID = 50   │   fork()   │ PPID = 100  │                 │ │
│  │  │ 코드        │ ─────────► │ 코드 (복사) │                 │ │
│  │  │ 데이터      │            │ 데이터 (복사)│                 │ │
│  │  │ 스택        │            │ 스택 (복사)  │                 │ │
│  │  └─────────────┘            └─────────────┘                 │ │
│  │                                                             │ │
│  │  반환값: 101 (자식 PID)       반환값: 0                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【exec() 시스템 콜】                                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  exec() 전                   exec() 후                       │ │
│  │  ────────                   ────────                         │ │
│  │  ┌─────────────┐            ┌─────────────┐                 │ │
│  │  │ 기존 코드    │           │ 새 프로그램 │                 │ │
│  │  │ 기존 데이터  │ exec()    │ 코드        │                 │ │
│  │  │             │ ─────────► │ 새 데이터   │                 │ │
│  │  │ PID 유지    │            │ PID 유지    │                 │ │
│  │  └─────────────┘            └─────────────┘                 │ │
│  │                                                             │ │
│  │  새 프로그램으로 완전히 교체 (PID는 동일)                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【fork() + exec() 조합】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  부모                    자식                                │ │
│  │  ────                    ────                                │ │
│  │  PID: 100               PID: 101                             │ │
│  │  코드: shell            코드: /bin/ls                        │ │
│  │                          ↑                                   │ │
│  │  fork() ──────────────► 복제                                 │ │
│  │  │                         │                                 │ │
│  │  │                      exec("/bin/ls")                      │ │
│  │  │                         │                                 │ │
│  │  ◄──────── wait() ◄───────┘                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 fork() 사용】                                                 │
│  ──────────────────                                                  │
│  pid_t pid = fork();                                                │
│  if (pid < 0) {                                                     │
│      perror("fork failed");                                         │
│  } else if (pid == 0) {                                             │
│      // 자식 프로세스                                                │
│      printf("Child: PID = %d\n", getpid());                        │
│      exit(0);                                                       │
│  } else {                                                           │
│      // 부모 프로세스                                                │
│      printf("Parent: Child PID = %d\n", pid);                      │
│      wait(NULL);                                                    │
│  }                                                                  │
│                                                                     │
│  【fork() + exec() 조합 (쉘 구현)】                                    │
│  ──────────────────                                                  │
│  pid_t pid = fork();                                                │
│  if (pid == 0) {                                                    │
│      // 자식: 새 프로그램 실행                                        │
│      execlp("ls", "ls", "-la", NULL);                              │
│      perror("exec failed");                                         │
│      exit(1);                                                       │
│  } else {                                                           │
│      // 부모: 자식 종료 대기                                          │
│      int status;                                                    │
│      waitpid(pid, &status, 0);                                      │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• fork(): 부모 프로세스 복제
• exec(): 새 프로그램으로 교체
• 반환: 부모는 자식 PID, 자식은 0
• COW: 메모리 최적화
• 상속: 파일 디스크립터, 환경변수
• 비상속: PID, 타이머, 락
```
