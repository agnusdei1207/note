# 114. LWP (경량 프로세스, Lightweight Process)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사용자 스레드와 커널 스레드 사이의 중간 계층
> 2. **가치**: 다대다 모델 구현, 유연한 스케줄링
> 3. **융합**: 스케줄러 액티베이션, Solaris, 상향 호출과 연관

---

## Ⅰ. 개요

### 개념 정의

LWP(Lightweight Process)는 **사용자 수준 스레드와 커널 스레드 사이의 중간 추상화 계층**이다. 다대다 스레드 모델에서 사용자 스레드를 커널 스레드에 매핑하는 가상 실행 단위다.

### 💡 비유: 팀장
LWP는 **팀장**과 같다. 직원들(사용자 스레드)을 관리하고, 회사(커널)와 소통한다. 여러 직원이 한 팀장을 공유할 수 있다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                LWP 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LWP vs 프로세스 vs 스레드】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분              프로세스         LWP           스레드        │ │
│  │  ────                    ────            ────           ────    │ │
│  │  주소 공간          독립            공유          공유          │ │
│  │  생성 비용          높음            중간          낮음          │ │
│  │  커널 관리          O              O             선택적         │ │
│  │  독립 스케줄링      O              O             X (LWP 통해)   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Solaris LWP 구현】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 사용자 스레드: 사용자 공간 라이브러리 관리                    │ │
│  │  • LWP: 커널이 스케줄링                                        │ │
│  │  • 바인딩: 특정 사용자 스레드를 LWP에 고정 가능                  │ │
│  │  • 동적 생성: 필요에 따라 LWP 생성/삭제                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스케줄러 액티베이션과 연동】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 커널가 사용자 스레드 라이브러리에 이벤트 통지                │ │
│  │  2. 상향 호출(Upcall)로 스케줄링 정보 전달                      │ │
│  │  3. 라이브러리가 LWP에 스레드 재배치                            │ │
│  │  4. 효율적인 다대다 매핑 유지                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Solaris LWP 확인】                                                 │
│  ──────────────────                                                  │
│  # LWP 수 확인                                                        │
│  $ ps -o pid,nlwp,comm -p <pid>                                     │
│                                                                     │
│  # 스레드/LWP 상세                                                    │
│  $ pstack <pid>                                                     │
│  $ prstat -L -p <pid>                                               │
│                                                                     │
│  【Linux에서의 유사 개념】                                              │
│  ──────────────────                                                  │
│  # Linux은 1:1 모델 사용                                              │
│  # 스레드 = 커널 스레드 (LWP 개념 없음)                                │
│                                                                     │
│  # 스레드 확인                                                        │
│  $ ps -L -p <pid>                                                   │
│  $ ls /proc/<pid>/task                                              │
│                                                                     │
│  # NLWP (Number of LWPs) 확인                                        │
│  $ cat /proc/<pid>/status | grep Threads                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
