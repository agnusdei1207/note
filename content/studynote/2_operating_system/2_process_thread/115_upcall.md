# 115. 상향 호출 (Upcall)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널이 사용자 공간으로 이벤트를 통지하는 메커니즘
> 2. **가치**: 효율적인 스레드 스케줄링, 이벤트 기반 통신
> 3. **융합**: LWP, 스케줄러 액티베이션, 다대다 모델과 연관

---

## Ⅰ. 개요

### 개념 정의

상향 호출(Upcall)은 **커널이 사용자 공간의 스레드 라이브러리에게 이벤트를 알리는 메커니즘**이다. 일반적인 시스템 콜(하향 호출)과 반대 방향의 통신이다.

### 💡 비유: 비상 연락
상향 호출은 **회사가 직원에게 비상 연락**하는 것과 같다. 평소엔 직원이 회사에 연락하지만, 중요한 일이 있으면 회사가 직원에게 연락한다.

│  │                                                             │ │
│  │  ┌─────────────┐              ┌─────────────┐               │ │

│  │  │             │              │             │               │ │
│  │  │ Application │              │ Thread      │               │ │
│  │  │      │      │              │ Library     │               │ │
│  │  │      ▼      │              │      ▲      │               │ │
│  │  │  syscall()  │              │   handler   │               │ │
│  │  └──────┼──────┘              └──────┼──────┘               │ │
│  │         │                            │                       │ │
│  │ ────────┼────────────────────────────┼─────────────────      │ │

│  │  │             │   upcall()   │             │               │ │
│  │  │ 처리        │ ───────────► │ 이벤트      │               │ │

│  │  이벤트                    설명                                │ │

└─────────────────────────────────────────────────────────────────────┘

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                상향 호출 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스케줄러 액티베이션과 연동】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 사용자 스레드가 블록 (예: I/O 대기)                         │ │
│  │  2. 커널가 이를 감지                                            │ │
│  │  3. 커널가 새 LWP 생성 후 상향 호출                              │ │
│  │  4. 사용자 스레드 라이브러리가 핸들러 실행                       │ │
│  │  5. 다른 사용자 스레드를 새 LWP에 스케줄                         │ │
│  │  6. CPU 활용도 유지                                              │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 효율적인 스레드 관리      • 구현 복잡                        │ │
│  │  • 빠른 이벤트 통지          • 커널 수정 필요                    │ │
│  │  • CPU 활용도 향상           • 오버헤드 가능                    │ │
│  │  • 다대다 모델 최적화                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【유사 개념】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 시그널: 커널이 프로세스에 이벤트 통지                         │ │
│  │  • 콜백: 비동기 이벤트 처리                                     │ │
│  │  • 인터럽트: 하드웨어가 CPU에 이벤트 통지                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 개념적 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【상향 호출 핸들러 예시】                                              │
│  ──────────────────                                                  │
│  void upcall_handler(upcall_event_t event) {                        │
│      switch (event.type) {                                          │
│          case UPEVENT_LWP_CREATE:                                   │
│              // 새 LWP 사용 가능                                     │
│              schedule_ready_thread(event.lwp);                      │
│              break;                                                 │
│                                                                     │
│          case UPEVENT_THREAD_BLOCK:                                 │
│              // 스레드 블록됨                                        │
│              mark_thread_blocked(event.thread);                     │
│              schedule_another_thread(event.lwp);                    │
│              break;                                                 │
│                                                                     │
│          case UPEVENT_THREAD_WAKEUP:                                │
│              // 스레드 깨어남                                        │
│              mark_thread_ready(event.thread);                       │
│              break;                                                 │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Linux에서의 대안】                                                  │
│  ──────────────────                                                  │
│  # Linux은 1:1 모델 사용으로 상향 호출 불필요                          │
│  # 대신 다음 메커니즘 활용:                                           │
│                                                                     │
│  • 시그널: SIGCHLD, SIGIO 등                                         │
│  • epoll/inotify: 이벤트 통지                                        │
│  • futex: 빠른 사용자 공간 동기화                                     │
│  • eventfd: 스레드 간 이벤트 통지                                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 커널→사용자 공간 이벤트 통지
• 방향: 하향 호출(시스템 콜)의 반대
• 용도: 스케줄러 액티베이션, LWP 관리
• 이벤트: 블록, 깨움, 선점, 생성, 파괴
• 장점: 효율적 스레드 관리
• Linux: 1:1 모델로 미사용
```
