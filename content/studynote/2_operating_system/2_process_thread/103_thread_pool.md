# 103. 스레드 풀 (Thread Pool)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 미리 생성된 스레드 집합을 재사용하는 패턴
> 2. **가치**: 생성 오버헤드 감소, 자원 관리, 응답성 향상
> 3. **융합**: 암묵적 스레딩, 비동기 처리, 작업 큐와 연관

---

## Ⅰ. 개요

### 개념 정의

스레드 풀(Thread Pool)은 **작업 처리를 위해 미리 생성된 스레드 집합**이다. 새로운 작업이 들어오면 풀에서 대기 중인 스레드에 할당하고, 작업 완료 후 스레드를 풀로 반환하여 재사용한다.

### 💡 비유: 콜센터 직원
스레드 풀은 **콜센터 직원들**과 같다. 미리 고용된 직원들(스레드)이 대기하다가 전화(작업)가 오면 처리한다. 전화가 끝나면 다음 전화를 기다린다.

### 스레드 풀 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                스레드 풀 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구성 요소】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  작업 제출                 스레드 풀                         │ │
│  │  ────────                 ────────                           │ │
│  │  ┌─────┐                 ┌───────────────────────────────┐  │ │
│  │  │Task1│ ────────────►  │  작업 큐 (Task Queue)          │  │ │
│  │  └─────┘                 │  ┌─────┬─────┬─────┬─────┐    │  │ │
│  │  ┌─────┐                 │  │ T1  │ T2  │ T3  │ T4  │    │  │ │
│  │  │Task2│ ────────────►  │  └─────┴─────┴─────┴─────┘    │  │ │
│  │  └─────┘                 │         ▼                      │  │ │
│  │  ┌─────┐                 │  ┌──────────────────────────┐ │  │ │
│  │  │Task3│ ────────────►  │  │   스레드 풀 (Workers)     │ │  │ │
│  │  └─────┘                 │  │  ┌────┐ ┌────┐ ┌────┐   │ │  │ │
│  │                          │  │  │ W1 │ │ W2 │ │ W3 │   │ │  │ │
│  │                          │  │  └────┘ └────┘ └────┘   │ │  │ │
│  │                          │  │   ▼      ▼      ▼       │ │  │ │
│  │                          │  │  실행   대기   실행      │ │  │ │
│  │                          │  └──────────────────────────┘ │  │ │
│  │                          └───────────────────────────────┘  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 스레드 생성 비용 절감    • 풀 크기 설정 어려움                │ │
│  │  • 자원 사용량 제한 가능    • 작업 대기 시간 발생                │ │
│  │  • 빠른 응답 시간          • 데드락 가능성                      │ │
│  │  • 스레드 수 관리 용이                                        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                스레드 풀 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【풀 크기 결정】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  작업 유형          권장 스레드 수                              │ │
│  │  ────                    ────                                    │ │
│  │  CPU 바운드          CPU 코어 수 + 1                             │ │
│  │  I/O 바운드          CPU 코어 수 × (1 + 대기시간/계산시간)        │ │
│  │  혼합형              모니터링 후 동적 조절                        │ │
│  │                                                               │ │
│  │  공식: N = N_cpu × U_cpu × (1 + W/C)                           │ │
│  │  - N_cpu: CPU 코어 수                                          │ │
│  │  - U_cpu: 목표 CPU 사용률                                      │ │
│  │  - W: 대기 시간, C: 계산 시간                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주요 구현】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  언어/플랫폼         구현                                       │ │
│  │  ────                    ────                                    │ │
│  │  Java               ExecutorService, ThreadPoolExecutor        │ │
│  │  Python             concurrent.futures.ThreadPoolExecutor     │ │
│  │  .NET               ThreadPool, Task Parallel Library         │ │
│  │  C++                std::thread + 작업 큐                      │ │
│  │  Go                 Goroutine + Work Stealing                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 코드 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Java ThreadPoolExecutor】                                         │
│  ──────────────────                                                  │
│  ThreadPoolExecutor executor = new ThreadPoolExecutor(             │
│      4,                              // 코어 스레드 수               │
│      10,                             // 최대 스레드 수               │
│      60L, TimeUnit.SECONDS,          // 유휴 스레드 유지 시간        │
│      new ArrayBlockingQueue<>(100)   // 작업 큐                     │
│  );                                                                 │
│                                                                     │
│  executor.submit(() -> {                                            │
│      // 작업 수행                                                    │
│  });                                                                │
│                                                                     │
│  【Python ThreadPoolExecutor】                                       │
│  ──────────────────                                                  │
│  from concurrent.futures import ThreadPoolExecutor                 │
│                                                                     │
│  with ThreadPoolExecutor(max_workers=4) as executor:               │
│      results = executor.map(process_item, items)                   │
│                                                                     │
│  【Linux 커널 Workqueue】                                            │
│  ──────────────────                                                  │
│  // 커널 스레드 풀                                                   │
│  struct workqueue_struct *wq;                                       │
│  wq = create_workqueue("my_wq");                                   │
│  queue_work(wq, &my_work);                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
