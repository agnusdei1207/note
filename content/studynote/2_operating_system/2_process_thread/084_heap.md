# 084. 힙 (Heap) 영역

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 실행 시간에 동적으로 할당되는 메모리 영역
> 2. **가치**: 유연한 메모리 관리, 가변 크기 데이터
> 3. **융합**: malloc/free, 단편화, 메모리 누수와 연관

---

## Ⅰ. 개요

### 개념 정의

힙(Heap) 영역은 **실행 시간에 프로그래머가 동적으로 할당하고 해제하는 메모리 영역**이다. malloc, free, new, delete 등으로 관리한다.

### 💡 비유: 창고
힙은 **창고**와 같다. 필요할 때마다 공간을 빌리고(할당), 다 쓰면 반납한다(해제). 공간이 부족하면 확장할 수 있다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                힙 상세                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【메모리 단편화】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  외부 단편화 (External Fragmentation)                          │ │
│  │  • 할당된 블록 사이의 작은 빈 공간                               │ │
│  │  • 총 여유 공간은 충분하나 연속적이지 않음                        │ │
│  │                                                               │ │
│  │  내부 단편화 (Internal Fragmentation)                          │ │
│  │  • 할당된 블록 내부의 사용되지 않는 공간                          │ │
│  │  • 요청 크기 > 실제 할당 크기                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 할당 알고리즘】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  알고리즘          설명                    특징                  │ │
│  │  ────                ────                    ────              │ │
│  │  First-Fit        첫 번째 적합 블록         빠름, 단편화       │ │
│  │  Best-Fit         가장 작은 적합 블록       느림, 단편화 적음   │ │
│  │  Worst-Fit        가장 큰 블록              느림, 단편화 많음   │ │
│  │  Buddy System     2의 거듭제곱 단위         단편화, 빠른 병합   │ │
│  │  Slab Allocator   객체 캐싱                 커널용 최적화      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 명령어 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【힙 메모리 확인】                                                   │
│  ──────────────────                                                  │
│  # 힙 크기 확인                                                       │
│  $ cat /proc/<pid>/maps | grep heap                                │
│                                                                     │
│  # 메모리 사용량                                                     │
│  $ cat /proc/<pid>/status | grep -E "VmData|VmStk"                │
│                                                                     │
│  【메모리 누수 감지】                                                 │
│  ──────────────────                                                  │
│  # valgrind로 메모리 누수 검사                                       │
│  $ valgrind --leak-check=full ./program                            │
│                                                                     │
│  # AddressSanitizer                                                 │
│  $ gcc -fsanitize=address -g program.c                             │
│  $ ./a.out                                                          │
│                                                                     │
│  # mtrace (glibc)                                                   │
│  $ export MALLOC_TRACE=./mtrace.log                                │
│  $ ./program                                                        │
│  $ mtrace ./program ./mtrace.log                                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
