# 108. 연쇄적 종료 (Cascading Termination)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 부모 종료 시 자식들도 함께 종료되는 현상
> 2. **가치**: 자원 누수 방지, 프로세스 트리 정리
> 3. **융합**: 부모/자식 관계, 고아 프로세스, init과 연관

---

## Ⅰ. 개요

### 개념 정의

연쇄적 종료(Cascading Termination)는 **부모 프로세스가 종료될 때 모든 자식 프로세스들도 함께 종료되는 현상**이다. 일부 운영체제에서 기본 동작으로 구현된다.

### 💡 비유: 회사 폐업
연쇄적 종료는 **회사 폐업**과 같다. 본사(부모)가 문을 닫으면 지사들(자식)도 모두 문을 닫는다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                연쇄적 종료 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【발생 상황】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 부모가 exit() 호출                                          │ │
│  │  • 부모가 시그널로 종료 (SIGKILL, SIGTERM)                      │ │
│  │  • 부모가 비정상 종료 (세그먼트 오류 등)                         │ │
│  │  • 프로세스 그룹/세션 종료                                      │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Linux에서의 구현】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 기본: 연쇄적 종료 없음 → 고아 프로세스 발생                   │ │
│  │  • 해결 1: SIGHUP 시그널 처리                                   │ │
│  │  • 해결 2: prctl(PR_SET_PDEATHSIG, SIGTERM)                   │ │
│  │  • 해결 3: 프로세스 그룹 사용                                   │ │
│  │                                                               │ │
│  │  // 부모 종료 시 자식도 종료                                    │ │
│  │  prctl(PR_SET_PDEATHSIG, SIGTERM);                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 자원 누수 방지          • 의도치 않은 종료                   │ │
│  │  • 고아 프로세스 방지      • 데이터 손실 가능                   │ │
│  │  • 시스템 정리             • 정상 종료 절차 생략                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【prctl을 사용한 연쇄 종료 구현】                                     │
│  ──────────────────                                                  │
│  #include <sys/prctl.h>                                             │
│                                                                     │
│  pid_t pid = fork();                                                │
│  if (pid == 0) {                                                    │
│      // 자식: 부모 종료 시 SIGTERM 받음                              │
│      prctl(PR_SET_PDEATHSIG, SIGTERM);                             │
│                                                                     │
│      // 부모가 이미 종료되었는지 확인                                 │
│      if (getppid() == 1) {                                          │
│          exit(1);  // init이 부모면 이미 고아                        │
│      }                                                              │
│                                                                     │
│      // 자식 작업 수행                                                │
│      while (1) { sleep(1); }                                        │
│  }                                                                  │
│                                                                     │
│  【프로세스 그룹 종료】                                                │
│  ──────────────────                                                  │
│  // 새 프로세스 그룹 생성                                             │
│  setpgid(0, 0);  // 자신이 리더                                      │
│                                                                     │
│  // 그룹 전체에 시그널 전송                                            │
│  killpg(getpgrp(), SIGTERM);                                        │
│                                                                     │
│  【프로세스 트리 종료 확인】                                            │
│  ──────────────────                                                  │
│  $ pstree -p <pid>                                                  │
│  $ kill -TERM -<pgid>  # 프로세스 그룹 종료                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 부모 종료 시 자식들도 종료
• 발생: exit(), 시그널, 비정상 종료
• Linux: 기본 지원 X, prctl로 구현
• 장점: 자원 누수 방지, 정리
• 단점: 의도치 않은 종료 가능
• 대안: 프로세스 그룹, 세션
```
