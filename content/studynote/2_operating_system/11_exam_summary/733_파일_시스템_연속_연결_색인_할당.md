# 파일 시스템 (연속/연결/색인 할당)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디스크에 파일을 저장하는 세 가지 방식 - 연속 할당, 연결 할당, 색인 할당
> 2. **가치**: 각 방식의 장단점, 외부/내부 단편화, 직접/순차 접근
> 3. **융합**: FAT, i-node, ext4, NTFS와 통합된 파일 시스템 구현

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

파일 할당 방식은 디스크에 파일을 저장하는 방법으로, 연속/연결/색인 할당이 있습니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    파일 시스템 할당 방식                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              1. 연속 할당 (Contiguous Allocation)              │ │
│   │                                                              │ │
│   │   ┌───────────────────────────────────────────────────────┐ │ │
│   │   │ File A (연속)                                           │ │ │
│   │   │ ┌─────┬─────┬─────┬─────┐                             │ │ │
│   │   │ │ Blk │ Blk │ Blk │ Blk │                             │ │ │
│   │   │ │  1  │  2  │  3  │  4  │                             │ │ │
│   │   │ └─────┴─────┴─────┴─────┘                             │ │ │
│   │   │                                                       │ │ │
│   │   │ 시작 블록: 1, 길이: 4                                  │ │ │
│   │   └───────────────────────────────────────────────────────┘ │ │
│   │                                                              │ │
│   │   장점: 빠른 순차/직접 접근, 단순                            │ │
│   │   단점: 외부 단편화, 파일 크기 증가 어려움                   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              2. 연결 할당 (Linked Allocation)                  │ │
│   │                                                              │ │
│   │   ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐                 │ │
│   │   │ Blk │ ─→ │ Blk │ ─→ │ Blk │ ─→ │ Blk │                 │ │
│   │   │  1  │    │  5  │    │  9  │    │  3  │                 │ │
│   │   └─────┘    └─────┘    └─────┘    └─────┘                 │ │
│   │      │                                              │        │ │
│   │   시작: 1, 끝: 3                                    NULL     │ │
│   │                                                              │ │
│   │   장점: 외부 단편화 없음, 파일 크기 증가 용이                │ │
│   │   단점: 직접 접근 느림, 포인터 오버헤드                      │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              3. 색인 할당 (Indexed Allocation)                 │ │
│   │                                                              │ │
│   │   색인 블록 (i-node):                                         │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │ [0] → Blk 1   [1] → Blk 5   [2] → Blk 9   [3] → Blk 3│   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │         │            │            │            │              │ │
│   │         ↓            ↓            ↓            ↓              │ │
│   │       ┌─────┐     ┌─────┐     ┌─────┐     ┌─────┐          │ │
│   │       │ Blk │     │ Blk │     │ Blk │     │ Blk │          │ │
│   │       │  1  │     │  5  │     │  9  │     │  3  │          │ │
│   │       └─────┘     └─────┘     └─────┘     └─────┘          │ │
│   │                                                              │ │
│   │   장점: 직접 접근 빠름, 외부 단편화 없음                      │ │
│   │   단점: 색인 블록 오버헤드                                   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 색인 할당이 현대 파일 시스템의 표준입니다.

**💡 비유**: 파일 할당은 책장에 책을 꽂는 방식과 같습니다.

**📢 섹션 요약 비유**: 색인 할당이 최고예요!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **연속 할당** | 시작+길이 | 연속 배치 | 책 한 권 |
| **연결 할당** | 포인터 | 연결 리스트 | 책 시리즈 |
| **색인 할당** | 색인 블록 | 직접 포인터 | 목차 |
| **FAT** | 테이블 | 연결 최적화 | 색인표 |

**할당 방식 비교**

| 방식 | 직접 접근 | 외부 단편화 | 파일 크기 |
|:---|:---:|:---:|:---:|
| **연속** | 빠름 | 있음 | 고정 |
| **연결** | 느림 | 없음 | 가변 |
| **색인** | 빠름 | 없음 | 가변 |

**핵심 알고리즘**: 파일 생성 → 할당 방식 선택 → 블록 배치 → 메타데이터 갱신

**📢 섹션 요약 비유**: 색인이 직접 접근을 빠르게 해요!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 실제 파일 시스템**

| FS | 할당 방식 | 특징 |
|:---|:---|:---|
| **FAT** | 연결 (테이블) | 단순 |
| **ext4** | 색인 (i-node) | 저널링 |
| **NTFS** | 색인 (MFT) | 윈도우 |
| **XFS** | 색인 (익스텐트) | 대용량 |

**📢 섹션 요약 비유**: 익스텐트가 효율적이에요!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: CD/DVD** → 연속 할당 (읽기 전용)
**시나리오 2: 일반 OS** → 색인 할당
**시나리오 3: USB** → FAT (호환성)

**도입 체크리스트**: 접근 패턴, 파일 크기, 호환성

**📢 섹션 요약 비유**: 용도에 맞게 선택해요!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)
