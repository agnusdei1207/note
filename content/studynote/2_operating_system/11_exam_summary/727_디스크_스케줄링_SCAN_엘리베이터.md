# 디스크 스케줄링 (SCAN, 엘리베이터)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디스크 헤드가 한 방향으로 이동하며 요청을 처리하고 끝에서 반대 방향으로 전환하는 스케줄링
> 2. **가치**: 공평한 대기 시간, 균등한 서비스, 호위 효과 감소
> 3. **융합**: SSTF, C-SCAN, LOOK과 통합된 디스크 스케줄링

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SCAN(엘리베이터) 디스크 스케줄링은 디스크 헤드가 한 방향으로 이동하며 요청을 처리하고 끝에서 반대 방향으로 전환하는 알고리즘입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SCAN (엘리베이터) 디스크 스케줄링                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SCAN 동작 원리                                   │ │
│   │                                                              │ │
│   │   트랙:  0 ────────────────────────────────────────── 199     │ │
│   │                           ↑                                  │ │
│   │          ┌────────────────┴────────────────┐                │ │
│   │          │         헤드 이동                │                │ │
│   │          │   ────────────────────→         │ (오른쪽)       │ │
│   │          │   ←────────────────────         │ (왼쪽)         │ │
│   │          └─────────────────────────────────┘                │ │
│   │                                                              │ │
│   │   요청: 23, 67, 89, 132, 45, 178, 12                        │ │
│   │   현재: 50, 방향: 오른쪽                                      │ │
│   │                                                              │ │
│   │   처리 순서: 50 → 67 → 89 → 132 → 178 → 끝 → 45 → 23 → 12   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              엘리베이터 비유                                   │ │
│   │                                                              │ │
│   │   ┌───┐                                                      │ │
│   │   │ 1 │ ← 요청                                                │ │
│   │   ├───┤                                                      │ │
│   │   │   │                                                      │ │
│   │   │ ↑ │ ← 헤드 (엘리베이터)                                   │ │
│   │   │   │                                                      │ │
│   │   ├───┤                                                      │ │
│   │   │ 2 │ ← 요청                                                │ │
│   │   └───┘                                                      │ │
│   │                                                              │ │
│   │   엘리베이터처럼 한 방향으로 이동하며 승객(요청)을 태움        │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: SCAN은 엘리베이터처럼 동작하여 공평한 서비스를 제공합니다.

**💡 비유**: SCAN은 엘리베이터가 위아래로 움직이며 승객을 태우는 것과 같습니다.

**📢 섹션 요약 비유**: 엘리베이터처럼 왕복해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **헤드** | 이동 주체 | 트랙 간 이동 | 엘리베이터 |
| **방향** | 이동 방향 | 양방향 전환 | 상/하 |
| **끝점** | 전환 위치 | 0 또는 끝 | 최상/최하 |
| **대기열** | 요청 목록 | 큐 관리 | 승객 |

**알고리즘 비교**

| 알고리즘 | 탐색 시간 | 공평성 | 특징 |
|:---|:---:|:---:|:---|
| **FCFS** | 나쁨 | 좋음 | 단순 |
| **SSTF** | 좋음 | 나쁨 | 기아 |
| **SCAN** | 좋음 | 좋음 | 공평 |

**핵심 알고리즘**: 요청 → 방향별 정렬 → 이동 처리 → 끝에서 전환

**📢 섹션 요약 비유**: SCAN이 공평해요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: SCAN vs C-SCAN vs LOOK**

| 비교 항목 | SCAN | C-SCAN | LOOK |
|:---|:---:|:---:|:---:|
| **전환** | 양 끝 | 한쪽만 | 마지막 요청 |
| **균등성** | 보통 | 좋음 | 좋음 |
| **효율** | 보통 | 좋음 | 최고 |

**📢 섹션 요약 비유**: LOOK이 더 효율적이에요!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: HDD** → SCAN/C-SCAN 사용
**시나리오 2: SSD** → 스케줄링 불필요 (랜덤 액세스)
**시나리오 3: 데이터베이스** → Deadline 스케줄러

**도입 체크리스트**: 디스크 타입, 워크로드, 지연 요구

**📢 섹션 요약 비유**: SSD는 스케줄링이 달라요!

### Ⅴ. 기대효과 및 결론 (Future & Standard)
