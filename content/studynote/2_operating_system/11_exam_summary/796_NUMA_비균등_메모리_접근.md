# NUMA (비균등 메모리 접근)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU-메모리 거리에 따른 접근 속도 차이
> 2. **가치**: 스케일업 성능, 대규모 시스템, 메모리 대역폭
> 3. **융합**: 메모리 정책, affinity, 마이그레이션과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

NUMA(Non-Uniform Memory Access)는 각 CPU가 로컬 메모리에 더 빠르게 접근하는 멀티프로세서 아키텍처입니다.

```

│                    NUMA (비균등 메모리 접근)                          │

│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              NUMA 아키텍처                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Node 0                    Node 1                    │   │ │
│   │   │  ┌─────────────────┐      ┌─────────────────┐       │   │ │
│   │   │  │ CPU 0, 1, 2, 3  │      │ CPU 4, 5, 6, 7  │       │   │ │
│   │   │  │    Local MEM    │      │    Local MEM    │       │   │ │
│   │   │  │    (128GB)      │      │    (128GB)      │       │   │ │

│   │   │                                                      │   │ │

│   │   │  • Local: ~100ns                                     │   │ │
│   │   │  • Remote: ~150-300ns (1.5-3배 느림)                  │   │ │

│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
```

> **해설**: NUMA는 로컬 메모리 접근 최적화가 핵심입니다.

**💡 비유**: NUMA는 집 안의 냉장고와 창고의 냉장고를 사용하는 차이입니다.

**📢 섹션 요약 비유**: 로컬 메모리가 빠르다!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Node** | 노드 | CPU+MEM | 방 |
| **Distance** | 거리 | 지연 시간 | 이동 |
| **Policy** | 정책 | 할당 방식 | 규칙 |
| **Autonuma** | 자동 | 마이그레이션 | 정리 |

**주요 API**

| API | 설명 |
|:---|:---|
| **numa_alloc_onnode()** | 노드 할당 |
| **mbind()** | 메모리 바인드 |
| **set_mempolicy()** | 정책 설정 |
| **move_pages()** | 페이지 이동 |

**핵심 알고리즘**: 노드 감지 → 정책 설정 → 로컬 할당 → 필요 시 마이그레이션

**📢 섹션 요약 비유**: numactl로 확인!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 메모리 아키텍처**

| 유형 | 접근 | 확장성 |
|:---|:---:|:---:|
| **UMA** | 균등 | 낮음 |
| **NUMA** | 비균등 | 높음 |
| **CC-NUMA** | 캐시 일관성 | 높음 |

**📢 섹션 요약 비유**: 서버는 대부분 NUMA!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: DB** → 노드별 인스턴스
**시나리오 2: VM** → NUMA aware 배치
**시나리오 3: HPC** → INTERLEAVE

**도입 체크리스트**: 노드 토폴로지, 워크로드 분석

**📢 섹션 요약 비유**: DB는 노드 분리!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 메모리 대역폭 2-4배, 지연 30-50% 감소

**미래 전망**: CXL, 분산 NUMA

**참고 표준**: ACPI NUMA, Linux NUMA API

**📢 섹션 요약 비유**: NUMA는 대형 서버의 필수!
