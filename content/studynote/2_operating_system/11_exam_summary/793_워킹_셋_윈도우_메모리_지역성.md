# 워킹 셋 윈도우 (메모리 지역성)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 일정 시간 동안 프로세스가 참조하는 페이지 집합
> 2. **가치**: 스래싱 방지, 메모리 할당, 성능 최적화
> 3. **융합**: 페이지 교체, 스와핑, 캐시와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

워킹 셋(Working Set)은 일정 시간 윈도우(Δ) 동안 프로세스가 참조하는 페이지 집합으로, 메모리 요구량을 나타냅니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    워킹 셋 윈도우                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              워킹 셋 개념                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  W(t, Δ): 시각 t에서 윈도우 Δ 동안 참조된 페이지 집합  │   │ │
│   │   │                                                      │   │ │
│   │   │  시간축:                                               │   │ │
│   │   │  ──────────────────────────────────────────────────→ │   │ │
│   │   │          ┌─────── Δ ───────┐                         │   │ │
│   │   │          │                 │                         │   │ │
│   │   │    t-Δ   t               t+Δ                         │   │ │
│   │   │          ↑                                             │   │ │
│   │   │       시각 t                                           │   │ │
│   │   │                                                      │   │ │
│   │   │  워킹 셋 크기 (WSS):                                    │   │ │
│   │   │  • 작은 WSS: 적은 메모리로 실행 가능                   │   │ │
│   │   │  • 큰 WSS: 많은 메모리 필요                            │   │ │
│   │   │                                                      │   │ │
│   │   │  평균 워킹 셋: Σ WSS_i / n                            │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스래싱과 워킹 셋                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  스래싱 발생 조건:                                     │   │ │
│   │   │  Σ WSS_i > 가용 메모리                                │   │ │
│   │   │                                                      │   │ │
│   │   │  결과:                                                 │   │ │
│   │   │  • 빈번한 페이지 폴트                                  │   │ │
│   │   │  • 스와핑 급증                                        │   │ │
│   │   │  • CPU 유휴, I/O 급증                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  해결:                                                 │   │ │
│   │   │  • 프로세스 수 제한                                    │   │ │
│   │   │  • 메모리 증설                                        │   │ │
│   │   │  • 워킹 셋 기반 할당                                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 워킹 셋 이상의 메모리를 할당하면 페이지 폴트가 감소합니다.

**💡 비유**: 워킹 셋은 요리할 때 필요한 도구의 집합입니다.

**📢 섹션 요약 비유**: WSS 합 > 메모리 = 스래싱!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Δ** | 윈도우 | 시간 범위 | 기간 |
| **WSS** | 크기 | 페이지 수 | 필요량 |
| **D** | 평균 | ΣWSS/n | 부하 |
| **M** | 가용 | 프레임 수 | 자원 |

**워킹 셋 모델**

| 파라미터 | 설명 |
|:---|:---|
| **Δ** | 윈도우 크기 |
| **W(t,Δ)** | 시각 t의 워킹 셋 |
| **|W|** | 워킹 셋 크기 |

**핵심 알고리즘**: Δ 동안 참조 → 페이지 집합 → WSS 계산 → 할당

**📢 섹션 요약 비유**: Δ가 크면 WSS 증가!
