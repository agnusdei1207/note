# RAID 0, 1, 5, 6 (성능/신뢰성)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 디스크를 조합하여 성능 향상과 신뢰성 확보를 동시에 달성하는 기술
> 2. **가치**: 성능, 중복성, 비용의 균형, 용도별 최적 레벨 선택
> 3. **융합**: 디스크 스케줄링, 스토리지, 고가용성과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RAID(Redundant Array of Independent Disks)는 여러 디스크를 조합하여 성능 향상과 신뢰성 확보를 동시에 달성하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RAID (Redundant Array of Independent Disks)       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RAID 레벨 비교                                   │ │
│   │                                                              │ │
│   │   RAID 0 (스트라이핑):                                        │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  D1  │  D2  │  D3  │  D4  │  D5  │  D6  │          │   │ │
│   │   │ Disk1│ Disk2│ Disk3│ Disk1│ Disk2│ Disk3│          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │   • 성능: 최고 (N배)     • 신뢰성: 없음 (1개 고장 = 전체 손실)│   │
│   │   • 용량 효율: 100%                                        │ │
│   │                                                              │ │
│   │   RAID 1 (미러링):                                            │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  D1  │  D2  │  D3  │          D1  │  D2  │  D3  │   │   │ │
│   │   │ Disk1│ Disk2│ Disk3│   ←미러→  Disk4│ Disk5│ Disk6│   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │   • 성능: 읽기 좋음      • 신뢰성: 최고 (N-1개 고장 허용)   │ │
│   │   • 용량 효율: 50%                                          │ │
│   │                                                              │ │
│   │   RAID 5 (분산 패리티):                                       │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  D1  │  D2  │  D3  │  P1  │                          │   │ │
│   │   │  D4  │  D5  │  P2  │  D6  │                          │   │ │
│   │   │  D7  │  P3  │  D8  │  D9  │                          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │   • 성능: 좋음           • 신뢰성: 1개 고장 허용            │ │
│   │   • 용량 효율: (N-1)/N                                      │ │
│   │                                                              │ │
│   │   RAID 6 (이중 패리티):                                       │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  D1  │  D2  │  D3  │  P1  │  Q1  │                  │   │ │
│   │   │  D4  │  D5  │  P2  │  Q2  │  D6  │                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │   • 성능: 보통           • 신뢰성: 2개 고장 허용            │ │
│   │   • 용량 효율: (N-2)/N                                      │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: RAID는 성능과 신뢰성의 균형을 선택하는 기술입니다.

**💡 비유**: RAID는 여러 바구니에 나눠 담거나 복사해 두는 것과 같습니다.

**📢 섹션 요약 비유**: 나눠 담거나 복사해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **스트라이핑** | 분산 저장 | 블록 단위 분할 | 나눠 담기 |
| **미러링** | 중복 저장 | 복사본 유지 | 복사 |
| **패리티** | 오류 복구 | XOR 연산 | 체크섬 |
| **재구성** | 고장 복구 | 패리티로 복원 | 수리 |

**RAID 레벨 선택 기준**

| 용도 | 추천 RAID | 이유 |
|:---|:---|:---|
| **고성능** | RAID 0 | 최대 속도 |
| **신뢰성** | RAID 1 | 완전 복제 |
| **균형** | RAID 5 | 성능+신뢰성 |
| **대용량** | RAID 6 | 2개 고장 허용 |

**핵심 알고리즘**: 데이터 분할 → 분산 저장 → 패리티 계산 → 재구성

**📢 섹션 요약 비유**: RAID 5가 가장 인기 있어요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: RAID 레벨**

| 레벨 | 읽기 | 쓰기 | 신뢰성 | 비용 |
|:---|:---:|:---:|:---:|:---:|
| **0** | 최고 | 최고 | 최악 | 최저 |
| **1** | 좋음 | 보통 | 최고 | 높음 |
| **5** | 좋음 | 보통 | 좋음 | 중간 |
| **6** | 좋음 | 낮음 | 최고 | 높음 |

**📢 섹션 요약 비유**: RAID 10도 좋아요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 데이터베이스** → RAID 10 (RAID 1+0)
**시나리오 2: 파일 서버** → RAID 5/6
**시나리오 3: 캐시** → RAID 0

**도입 체크리스트**: 신뢰성 요구, 성능 요구, 예산

**📢 섹션 요약 비유**: 상황에 맞게 선택해요!
