# 버퍼 오버플로우 공격 (스택)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 버퍼 크기를 초과하는 데이터를 입력해 반환 주소를 덮어쓰는 공격
> 2. **가치**: 메모리 보안 이해, 방어 기법(ASLR, Stack Canary) 습득
> 3. **융합**: 스택 프레임, 함수 호출, 메모리 보호와 통합된 보안

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

버퍼 오버플로우 공격(Buffer Overflow Attack)은 버퍼 크기를 초과하는 데이터를 입력해 반환 주소를 덮어써 악성 코드를 실행하는 공격입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    버퍼 오버플로우 공격 (Stack Buffer Overflow)       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              정상 스택 프레임                                  │ │
│   │                                                              │ │
│   │   높은 주소                                                   │ │
│   │   ┌─────────────────────┐                                    │ │
│   │   │   Return Address    │ ← 함수 복귀 주소                   │ │
│   │   ├─────────────────────┤                                    │ │
│   │   │   Saved EBP         │ ← 이전 스택 프레임 포인터           │ │
│   │   ├─────────────────────┤                                    │ │
│   │   │   Local Variables   │                                    │ │
│   │   │   (buffer[8])       │ ← 버퍼 (8바이트)                   │ │
│   │   └─────────────────────┘                                    │ │
│   │   낮은 주소                                                   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              버퍼 오버플로우 발생                              │ │
│   │                                                              │ │
│   │   높은 주소                                                   │ │
│   │   ┌─────────────────────┐                                    │ │
│   │   │ 0xdeadbeef          │ ← 악성 코드 주소로 덮어씀!         │ │
│   │   ├─────────────────────┤                                    │ │
│   │   │ AAAABBBBCCCCDDDD    │ ← 버퍼 overflow                    │ │
│   │   │ AAAABBBBCCCCDDDD    │                                    │ │
│   │   └─────────────────────┘                                    │ │
│   │   낮은 주소                                                   │ │
│   │                                                              │ │
│   │   공격 코드: strcpy(buffer, "AAAA...恶意주소");              │ │
│   │   → 버퍼 크기 검증 없이 복사 → 오버플로우 발생               │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 버퍼 오버플로우는 입력 검증 부재 시 발생합니다.

**💡 비유**: 버퍼 오버플로우는 컵에 물을 넘치게 부어 옆 컵까지 망치는 것과 같습니다.

**📢 섹션 요약 비유**: 넘치면 돼요!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Buffer** | 데이터 저장 | 고정 크기 | 컵 |
| **Return Addr** | 복귀 주소 | 함수 종료 시 점프 | 목적지 |
| **Shellcode** | 악성 코드 | 공격자 코드 | 바이러스 |
| **NOP Sled** | 슬라이드 | 실행 위치 완화 | 미끄럼틀 |

**공격 유형**

| 유형 | 설명 | 대상 |
|:---|:---|:---|
| **Stack Overflow** | 지역 변수 오버플로우 | 스택 |
| **Heap Overflow** | 동적 할당 오버플로우 | 힙 |
| **Return-to-libc** | 라이브러리 함수 호출 | libc |

**핵심 알고리즘**: 버퍼 초과 → 반환 주소 덮어쓰기 → 악성 코드 실행

**📢 섹션 요약 비유**: 스택이 가장 흔해요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 방어 기법**

| 기법 | 설명 | 효과 |
|:---|:---|:---|
| **ASLR** | 주소 난수화 | 높음 |
| **Stack Canary** | 스택 감시값 | 높음 |
| **DEP/NX** | 실행 방지 | 높음 |
| **StackGuard** | 컴파일러 보호 | 중간 |

**📢 섹션 요약 비유**: ASLR이 주소를 숨겨요!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 웹 서버** → ASLR, DEP 활성화
**시나리오 2: 임베디드** → Stack Canary
**시나리오 3: 컨테이너** → seccomp 필터

**도입 체크리스트**: -fstack-protector, NX 비트

**📢 섹션 요약 비유**: 방어는 다층으로!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**미래 전망**: Control-Flow Integrity (CFI)

**참고 표준**: CVE, CWE-121

**📢 섹션 요약 비유**: 오버플로우는 보안의 적!
