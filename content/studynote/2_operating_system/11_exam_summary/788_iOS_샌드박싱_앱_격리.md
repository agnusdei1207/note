# iOS 샌드박싱 (앱 격리)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 앱을 격리된 환경에서 실행하여 접근 제한
> 2. **가치**: 보안, 프라이버시, 시스템 안정성
> 3. **융합**: 코드 사이닝, 권한, Entitlements와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

iOS 샌드박싱은 각 앱을 격리된 환경(sandbox)에서 실행하여 다른 앱이나 시스템 자원에 대한 접근을 제한합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    iOS 샌드박싱                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              샌드박스 구조                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  App Sandbox:                                          │   │ │
│   │   │  ┌─────────────────────────────────────────────────┐ │   │ │
│   │   │  │  App Bundle        ← 읽기 전용                   │ │   │ │
│   │   │  │  Documents/        ← 사용자 데이터               │ │   │ │
│   │   │  │  Library/          ← 설정, 캐시                  │ │   │ │
│   │   │  │  tmp/              ← 임시 파일                   │ │   │ │
│   │   │  └─────────────────────────────────────────────────┘ │   │ │
│   │   │                                                      │   │ │
│   │   │  접근 불가:                                            │   │ │
│   │   │  • 다른 앱의 데이터                                    │   │ │
│   │   │  • 시스템 파일                                         │   │ │
│   │   │  • 하드웨어 (권한 없이)                                │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              권한 및 Entitlements                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Info.plist 권한:                                     │   │ │
│   │   │  • NSCameraUsageDescription                          │   │ │
│   │   │  • NSMicrophoneUsageDescription                      │   │ │
│   │   │  • NSLocationWhenInUseUsageDescription               │   │ │
│   │   │  • NSPhotoLibraryUsageDescription                    │   │ │
│   │   │  • NSContactsUsageDescription                        │   │ │
│   │   │                                                      │   │ │
│   │   │  Entitlements (개발자 계정 필요):                      │   │ │
│   │   │  • App Groups (앱 간 데이터 공유)                      │   │ │
│   │   │  • Keychain Sharing                                  │   │ │
│   │   │  • iCloud                                            │   │ │
│   │   │  • Push Notifications                                │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: iOS 샌드박스는 기본적으로 모든 접근을 차단하고 필요한 것만 허용합니다.

**💡 비유**: 샌드박스는 각 앱에게 전용 방을 주는 것과 같습니다.

**📢 섹션 요약 비유**: 기본 차단, 필요 시 허용!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Sandbox** | 격리 | 접근 제어 | 방 |
| **Entitlement** | 권한 | 특수 접근 | 출입증 |
| **Code Signing** | 무결성 | 서명 검증 | 도장 |
| **TCC** | 사용자 동의 | 권한 요청 | 허락 |

**주요 메커니즘**

| 메커니즘 | 설명 |
|:---|:---|
| **Trusted BSD** | 커널 정책 |
| **AMFI** | 코드 서명 검증 |
| **TCC DB** | 권한 데이터베이스 |

**핵심 알고리즘**: 앱 실행 → 샌드박스 적용 → 권한 확인 → 접근 제어

**📢 섹션 요약 비유**: AMFI가 게이트키퍼!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 모바일 격리**

| 항목 | iOS | Android |
|:---|:---|:---|
| **격리** | 강함 | 중간 |
| **권한** | 런타임 | 설치+런타임 |
| **앱 공유** | 제한적 | Intent |

**📢 섹션 요약 비유**: iOS가 더 강력한 격리!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 일반 앱** → 기본 샌드박스

**📢 섹션 요약 비유**: App Groups로 제한적 공유!
