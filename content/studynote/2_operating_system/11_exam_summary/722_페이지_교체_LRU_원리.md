# 페이지 교체 (LRU 원리)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가장 오랫동안 사용되지 않은 페이지를 교체하는 최적 근사 알고리즘
> 2. **가치**: 지역성 기반, 좋은 성능, 널리 사용되는 교체 정책
> 3. **융합**: 클럭 알고리즘, 참조 비트, 페이지 폴트와 통합된 교체

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

LRU(Least Recently Used) 페이지 교체는 가장 오랫동안 사용되지 않은 페이지를 교체하는 알고리즘입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    LRU (Least Recently Used) 페이지 교체             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              LRU 원리                                         │ │
│   │                                                              │ │
│   │   "과거에 가장 적게 사용된 페이지가                            │ │
│   │    미래에도 가장 적게 사용될 것이다"                           │ │
│   │                                                              │ │
│   │   지역성(Locality) 기반:                                      │ │
│   │   • 시간적 지역성: 최근 사용한 것은 다시 사용                  │ │
│   │   • 공간적 지역성: 인접한 것도 사용                           │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              LRU 예시                                         │ │
│   │                                                              │ │
│   │   페이지 참조 순서: 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5        │ │
│   │   프레임 수: 3                                                │ │
│   │                                                              │ │
│   │   ┌─────┬────────────────────────────────────────────────┐   │ │
│   │   │ 시간 │  1   2   3   4   1   2   5   1   2   3   4   5 │   │ │
│   │   ├─────┼────────────────────────────────────────────────┤   │ │
│   │   │ F1  │  1   1   1   4   4   4   5   5   5   3   3   3 │   │ │
│   │   │ F2  │  -   2   2   2   1   1   1   1   1   1   4   4 │   │ │
│   │   │ F3  │  -   -   3   3   3   2   2   2   2   2   2   5 │   │ │
│   │   ├─────┼────────────────────────────────────────────────┤   │ │
│   │   │폴트 │  X   X   X   X   X   X   X   -   -   X   X   X │   │ │
│   │   └─────┴────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   총 폴트: 10회                                               │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              LRU 구현 방법                                    │ │
│   │                                                              │ │
│   │   1. 카운터: 각 페이지에 타임스탬프                          │ │
│   │   2. 스택: 사용 시 맨 위로 이동                               │ │
│   │   3. 참조 비트: 근사 구현 (Clock 알고리즘)                    │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: LRU는 실용적인 페이지 교체 알고리즘입니다.

**💡 비유**: LRU는 자주 안 쓰는 물건을 정리하는 것과 같습니다.

**📢 섹션 요약 비유**: 옛날 것은 나가세요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **참조 순서** | 사용 기록 | 시간 순서 | 방명록 |
| **카운터** | 타임스탬프 | 시간 기록 | 도장 |
| **스택** | LRU 순서 | 맨 아래 = 교체 | 책더미 |
| **참조 비트** | 근사 구현 | 0/1 표시 | 간편법 |

**교체 알고리즘 비교**

| 알고리즘 | 복잡도 | 성능 | 구현 |
|:---|:---:|:---:|:---:|
| **OPT** | 불가 | 최고 | X |
| **LRU** | 높음 | 좋음 | 어려움 |
| **Clock** | 낮음 | 좋음 | 쉬움 |
| **FIFO** | 낮음 | 나쁨 | 쉬움 |

**핵심 알고리즘**: 폴트 발생 → LRU 페이지 찾기 → 교체

**📢 섹션 요약 비유**: Clock이 실용적이에요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 구현 오버헤드**

| 방식 | 하드웨어 | 오버헤드 |
|:---|:---:|:---:|
| **정확한 LRU** | 필요 | 높음 |
| **참조 비트** | 선택 | 낮음 |
| **소프트웨어** | 불필요 | 높음 |

**📢 섹션 요약 비유**: 참조 비트로 근사해요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 일반 OS** → Clock 알고리즘 (LRU 근사)
**시나리오 2: 실시간** → 고정 할당
**시나리오 3: DB** → LRU 변형 (LRU-K)

**도입 체크리스트**: 참조 비트, 교체 대상 선정, 오버헤드

**📢 섹션 요약 비유**: Clock이 실제로 많이 쓰여요!
