# POSIX 스레드 (Pthread API)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: POSIX 표준 스레드 라이브러리 인터페이스
> 2. **가치**: 이식성, 멀티스레딩, 동기화
> 3. **융합**: 뮤텍스, 조건 변수, 배리어와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

POSIX 스레드(Pthread)는 IEEE POSIX.1c 표준에 정의된 스레드 프로그래밍 API입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    POSIX 스레드 (Pthread)                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              핵심 API                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  스레드 관리:                                          │   │ │
│   │   │  pthread_create()    - 스레드 생성                    │   │ │
│   │   │  pthread_exit()      - 스레드 종료                    │   │ │
│   │   │  pthread_join()      - 스레드 대기                    │   │ │
│   │   │  pthread_detach()    - 분리                          │   │ │
│   │   │  pthread_self()      - ID 조회                       │   │ │
│   │   │  pthread_equal()     - ID 비교                       │   │ │
│   │   │                                                      │   │ │
│   │   │  동기화:                                               │   │ │
│   │   │  pthread_mutex_lock/unlock()    - 뮤텍스              │   │ │
│   │   │  pthread_cond_wait/signal()     - 조건 변수           │   │ │
│   │   │  pthread_rwlock_rdlock/wrlock() - 읽기/쓰기 락        │   │ │
│   │   │  pthread_barrier_wait()         - 배리어              │   │ │
│   │   │  sem_wait/post()                - 세마포어            │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              스레드 생성 예제                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  #include <pthread.h>                                 │   │ │
│   │   │                                                      │   │ │
│   │   │  void* thread_func(void* arg) {                      │   │ │
│   │   │      // 스레드 작업                                   │   │ │
│   │   │      return NULL;                                    │   │ │
│   │   │  }                                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  int main() {                                        │   │ │
│   │   │      pthread_t tid;                                  │   │ │
│   │   │      pthread_create(&tid, NULL, thread_func, NULL); │   │ │
│   │   │      pthread_join(tid, NULL);                        │   │ │
│   │   │  }                                                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: Pthread는 스레드 생성, 동기화, 종료의 표준 인터페이스입니다.

**💡 비유**: Pthread는 스레드용 만능 도구 세트입니다.

**📢 섹션 요약 비유**: -lpthread 링크!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **pthread_t** | ID | 스레드 식별자 | 이름표 |
| **pthread_attr_t** | 속성 | 설정 | 옵션 |
| **pthread_mutex_t** | 뮤텍스 | 상호 배제 | 자물쇠 |
| **pthread_cond_t** | 조건 | 대기/신호 | 알림 |

**속성 설정**

| 속성 | 설명 |
|:---|:---|
| **detachstate** | 분리 상태 |
| **schedpolicy** | 스케줄링 |
| **stacksize** | 스택 크기 |

**핵심 알고리즘**: create → start routine → join/detach

**📢 섹션 요약 비유**: join은 자원 회수!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 동기화**

| 도구 | 용도 | 타입 |
|:---|:---|:---|
| **Mutex** | 상호 배제 | 이진 |
| **RWLock** | 읽기/쓰기 | 다중 |
| **Condvar** | 대기/신호 | 이벤트 |
| **Barrier** | 동기점 | 집합 |

**📢 섹션 요약 비유**: RWLock이 읽기 많을 때!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 병렬 처리** → 스레드 풀 + Pthread
**시나리오 2: 생산자-소비자** → Condvar
**시나리오 3: 읽기 많은 작업** → RWLock

**도입 체크리스트**: -pthread 플래그, 에러 처리

**📢 섹션 요약 비유**: 에러 코드 반환!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 이식성, 표준 준수

**참고 표준**: IEEE POSIX.1c, SUS
