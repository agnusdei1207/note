# 버퍼 캐시 (파일 입출력 지연)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디스크 I/O를 줄이기 위해 메모리에 자주 사용하는 데이터를 캐싱하는 기법
> 2. **가치**: I/O 성능 향상, 지연 쓰기, 미리 읽기
> 3. **융합**: 페이지 캐시, 파일 시스템, 가상 메모리와 통합된 캐싱

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

버퍼 캐시(Buffer Cache)는 디스크 I/O를 줄이기 위해 메모리에 자주 사용하는 데이터를 캐싱하는 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    버퍼 캐시 (Buffer Cache)                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              캐시 없이 vs 캐시 사용                           │ │
│   │                                                              │ │
│   │   캐시 없이:                                                  │ │
│   │   ┌────────┐     ┌────────┐     ┌────────┐                  │ │
│   │   │  App   │ ──→ │  Disk  │ ──→ │  App   │                  │ │
│   │   │(read)  │     │(10ms)  │     │(data)  │                  │ │
│   │   └────────┘     └────────┘     └────────┘                  │ │
│   │   매번 디스크 접근 → 느림                                      │ │
│   │                                                              │ │
│   │   캐시 사용:                                                  │ │
│   │   ┌────────┐     ┌────────┐     ┌────────┐                  │ │
│   │   │  App   │ ──→ │ Cache  │ ──→ │  App   │                  │ │
│   │   │(read)  │     │(0.1ms) │     │(data)  │                  │ │
│   │   └────────┘     └────────┘     └────────┘                  │ │
│   │   메모리에서 바로 → 빠름!                                      │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              캐싱 기법                                         │ │
│   │                                                              │ │
│   │   1. 미리 읽기 (Read-ahead):                                   │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  요청: 블록 1                                         │   │ │
│   │   │  실제: 블록 1, 2, 3, 4 미리 읽기                      │   │ │
│   │   │  지역성 활용                                          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   2. 지연 쓰기 (Delayed Write / Write-back):                  │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  쓰기 요청 → 캐시에만 쓰기 → 나중에 디스크 반영       │   │ │
│   │   │  장점: 쓰기 모아서 한 번에 (성능)                     │   │ │
│   │   │  단점: 크래시 시 데이터 손실 가능                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   3. 동기화 쓰기 (O_SYNC, fsync):                             │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  즉시 디스크에 쓰기                                   │   │ │
│   │   │  데이터 무결성 보장                                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 버퍼 캐시는 디스크 I/O 병목을 크게 줄여줍니다.

**💡 비유**: 버퍼 캐시는 자주 쓰는 책을 책상 위에 두는 것과 같습니다.

**📢 섹션 요약 비유**: 읽기는 미리, 쓰기는 나중에!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Read-ahead** | 미리 읽기 | 연속 블록 | 도란도란 |
| **Write-back** | 지연 쓰기 | 모아서 쓰기 | 나중에 |
| **Dirty page** | 수정 페이지 | 쓰기 필요 | 변경됨 |
| **Flush** | 디스크 반영 | 일괄 쓰기 | 정리 |

**캐시 정책**

| 정책 | 설명 | 용도 |
|:---|:---|:---|
| **LRU** | 오래된 것 교체 | 일반 |
| **LFU** | 적게 쓴 것 교체 | 특수 |
| **Clock** | 참조 비트 기반 | 실용 |

**핵심 알고리즘**: 요청 → 캐시 조회 → Hit: 반환, Miss: 디스크 읽기

**📢 섹션 요약 비유**: Hit는 100배 빨라요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

|:---|:---|:---|
| **버퍼 캐시** | 별도 | 통합 |

**📢 섹션 요약 비유**: 현대는 통합 캐시!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: DB 로그** → O_SYNC (안전)
**시나리오 2: 웹 서버** → 캐시 활용
**시나리오 3: 크래시 복구** → fsck, 저널링

**도입 체크리스트**: 안전 vs 성능, 캐시 크기

**📢 섹션 요약 비유**: 중요한 건 sync!
