# 분산 락 (Distributed Lock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 시스템에서 상호 배제를 보장하는 락
> 2. **가치**: 일관성, 동시성 제어, 리더 선출
> 3. **융합**: Redis, ZooKeeper, etcd와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

분산 락(Distributed Lock)은 분산 시스템의 여러 노드 간 상호 배제를 보장하는 동기화 메커니즘입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    분산 락 (Distributed Lock)                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              분산 락 시나리오                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Node 1          Node 2          Node 3               │   │ │
│   │   │    │               │               │                  │   │ │
│   │   │    │  Lock 요청    │               │                  │   │ │
│   │   │    ├──────────────→│               │                  │   │ │
│   │   │    │               │               │                  │   │ │
│   │   │    │         코디네이터 (Redis/ZK)                    │   │ │
│   │   │    │               │               │                  │   │ │
│   │   │    │   Lock 획득   │               │                  │   │ │
│   │   │    │←──────────────┤               │                  │   │ │
│   │   │    │               │               │                  │   │ │
│   │   │  [락 보유]        대기            대기                │   │ │
│   │   │    │               │               │                  │   │ │
│   │   │    │   Lock 해제   │               │                  │   │ │
│   │   │    ├──────────────→│               │                  │   │ │
│   │   │    │               │               │                  │   │ │
│   │   │    │         다음 대기자에게                        │   │ │
│   │   │    │               │               │                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │

│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Redis:                                                 │   │ │
│   │   │  SET lock_key value NX PX 30000                       │   │ │
│   │   │  (NX: 없을 때만 생성, PX: 만료 시간)                    │   │ │
│   │   │                                                      │   │ │
│   │   │  ZooKeeper:                                            │   │ │
│   │   │  /locks/lock- 에 ephemeral 노드 생성                   │   │ │
│   │   │  가장 작은 시퀀스 노드가 락 획득                        │   │ │
│   │   │                                                      │   │ │
│   │   │  etcd:                                                 │   │ │
│   │   │  lease + revision 기반 락                              │   │ │

│   └──────────────────────────────────────────────────────────────┘ │

└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 분산 락은 코디네이터 서비스가 중앙에서 관리합니다.

**💡 비유**: 분산 락은 여러 지점의 은행이 중앙 시스템을 통해 계좌 잠금을 관리하는 것과 같습니다.

**📢 섹션 요약 비유**: 만료 시간 필수!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Lock Key** | 키 | 고유 식별자 | 번호표 |
| **Lease** | 만료 | TTL | 시간 제한 |
| **Token** | 식별 | 소유자 ID | 이름 |
| **Queue** | 대기 | FIFO | 줄 |

**주요 고려사항**

| 이슈 | 설명 |
|:---|:---|
| **Deadlock** | 만료로 방지 |
| **Split Brain** | 쿼럼 사용 |
| **Clock Skew** | 벡터 클럭 |
| **Failure** | 타임아웃 |

**핵심 알고리즘**: SET NX → 작업 → DELETE (token 확인)

**📢 섹션 요약 비유**: Redlock이 안전!
