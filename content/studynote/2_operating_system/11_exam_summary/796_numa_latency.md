# NUMA 아키텍처의 지연 시간 및 성능 최적화

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 멀티 소켓 서버 환경에서 각 CPU가 자신에게 물리적으로 가까운 **로컬 메모리**와 다른 CPU에 연결된 **원격 메모리**를 가짐으로써, 메모리 접근 위치에 따라 지연 시간(Latency)이 달라지는 비균등 메모리 접근 방식.
> 2. **가치**: 중앙 버스 병목을 해결하여 수천 개의 코어로 확장 가능한 시스템을 구축할 수 있게 하며, 데이터와 프로세스를 동일 노드에 배치하는 **NUMA-aware** 스케줄링을 통해 최고의 성능을 인출한다.
> 3. **융합**: 하드웨어의 인터커넥트(QPI, UPI 등) 기술과 운영체제의 메모리 할당 정책이 결합되어, 클라우드 서버와 고성능 컴퓨팅(HPC) 성능의 핵심 지표가 된다.

---

### Ⅰ. NUMA 아키텍처의 구조 및 지연 시간

전통적인 UMA(Uniform Memory Access) 방식의 한계를 극복하기 위해 등장했다.

#### 1. 구조적 특징 (ASCII)
```ascii
    [ Node 0 ]                        [ Node 1 ]
    +-----------------------+         +-----------------------+
    | [ CPU 0 ] [ CPU 1 ]   |         | [ CPU 2 ] [ CPU 3 ]   |
    |          |            |         |          |            |
    |   [ Local Memory ]    | <-----> |   [ Local Memory ]    |
    +-----------------------+ (Inter- | +-----------------------+
                               connect)
```

#### 2. 지연 시간(Latency) 차이
- **Local Access**: 자신이 속한 노드의 메모리에 접근. (Fast, Low Latency)
- **Remote Access**: 인터커넥트를 통해 다른 노드의 메모리에 접근. (Slow, High Latency)
- **NUMA Ratio**: 원격 접근 시간과 로컬 접근 시간의 비율. 이 비율이 클수록 최적화의 중요성이 커진다.

---

### Ⅱ. 운영체제의 NUMA 인지형 최적화 전략

운영체제는 성능 저하를 막기 위해 아래와 같은 정책을 사용한다.

| 전략 | 설명 | 기대 효과 |
|:---|:---|:---|
| **First-touch Policy** | 페이지를 실제로 처음 사용하는 CPU의 로컬 메모리에 프레임을 할당. | 로컬 히트율 극대화 |
| **NUMA Scheduling** | 프로세스를 가급적 해당 메모리가 있는 노드의 CPU로 스케줄링. | 원격 접근 오버헤드 방지 |
| **Page Migration** | 참조 패턴을 분석하여 자주 쓰이는 페이지를 원격에서 로컬로 자동 이주. | 런타임 성능 보정 |
| **Memory Interleaving** | 데이터를 모든 노드에 골고루 분산하여 대역폭(Bandwidth) 극대화. | 특정 노드 뱅크 병목 해소 |

---

### Ⅲ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 가상화 환경에서의 NUMA (vNUMA)
- **현상**: 거대한 VM(가상 머신)이 여러 물리 소켓에 걸쳐 생성될 때 성능이 들쭉날쭉함.
- **기술사적 결단**: 
  - 하이퍼바이저에서 **vNUMA** 설정을 활성화하여 게스트 OS에게 하부 하드웨어의 NUMA 구조를 알려주어야 한다. 
  - 이를 통해 게스트 OS 내부의 스케줄러가 스스로 최적의 배치를 수행하게 유도한다.

#### 2. 기술사적 인사이트: "Locality is King"
- 현대의 멀티 테라바이트 서버에서 NUMA를 무시한 프로그래밍은 시스템 성능의 50% 이상을 낭비하는 결과를 초래한다. 특히 데이터베이스 엔진 설계 시 **노드별 메모리 할당(numactl 활용)**은 필수적인 성능 튜닝 공정이다.

---

### Ⅳ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **확장성 확보**: 수십 개의 소켓을 가진 슈퍼컴퓨터급 서버 구축 가능.
- **예측 가능한 성능**: 워크로드 격리를 통한 지연 시간의 안정적 관리.

#### 2. 미래 전망
최근에는 메모리와 CPU뿐만 아니라 가속기(GPU)와 스토리지(NVMe)까지 NUMA 구조에 편입되고 있다. 앞으로는 CXL(Compute Express Link) 기술을 통해 노드 간의 메모리 경계를 허물면서도 지연 시간을 나노초 단위로 관리하는 **컴포저블 인프라 (Composable Infrastructure)** 가 차세대 데이터센터의 표준이 될 것이다.
