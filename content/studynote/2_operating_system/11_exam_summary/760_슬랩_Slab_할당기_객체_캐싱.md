# 슬랩 (Slab) 할당기 / 객체 캐싱

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 커널 객체를 효율적으로 할당/해제하는 메모리 관리자
> 2. **가치**: 단편화 감소, 캐싱, 빠른 할당
> 3. **융합**: kmalloc, kmem_cache, 커널 메모리와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

슬랩 할당기(Slab Allocator)는 커널 객체를 효율적으로 할당/해제하는 메모리 관리자입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    슬랩 (Slab) 할당기                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              문제: 버디 시스템의 한계                          │ │
│   │                                                              │ │
│   │   버디 시스템:                                                │ │
│   │   • 페이지 단위 할당 (4KB 배수)                               │ │
│   │   • 작은 객체(예: 100B)도 4KB 필요 → 내부 단편화              │ │
│   │   • 자주 할당/해제 → 오버헤드                                 │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              슬랩 할당기 구조                                  │ │

│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │              Cache (객체 캐시)                        │   │ │

│   │   │        │                    │                │       │   │ │

│   │   │   │ └─────┘ │         │ └─────┘ │      │ └─────┘ │  │   │ │
│   │   │   └─────────┘         └─────────┘      └─────────┘  │   │ │
│   │   │   (페이지에서 할당)    (페이지에서 할당)              │   │ │

│   │              장점                                              │ │
│   │                                                              │ │
│   │   1. 단편화 감소: 같은 크기 객체를 한 슬랩에 모음              │ │
│   │   2. 빠른 할당: free 리스트에서 즉시 꺼냄                      │ │
│   │   3. 캐싱: 자주 쓰는 객체를 재사용                            │ │
│   │   4. 초기화 생략: 생성자/소멸자 패턴                          │ │
```

> **해설**: 슬랩 할당기는 커널 객체 할당을 최적화합니다.

**💡 비유**: 슬랩 할당기는 같은 크기의 서랍에 같은 물건을 정리하는 것과 같습니다.

**📢 섹션 요약 비유**: 같은 크기는 같은 슬랩에!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Cache** | 객체 타입별 | kmem_cache | 서랍장 |
| **Slab** | 페이지 그룹 | 1+ 페이지 | 서랍 |
| **Object** | 실제 데이터 | 할당 단위 | 물건 |
| **Free List** | 빈 객체 | 연결 리스트 | 빈 칸 |

**슬랩 상태**

| 상태 | 설명 |
|:---|:---|
| **Full** | 모든 객체 사용 중 |
| **Partial** | 일부 사용 |
| **Empty** | 모두 비어있음 |

**핵심 알고리즘**: 캐시 조회 → 슬랩 선택 → 객체 반환

**📢 섹션 요약 비유**: kmalloc이 범용 캐시!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 할당기**

| 할당기 | 용도 | 특징 |
|:---|:---|:---|
| **Buddy** | 페이지 | 외부 단편화 해결 |

| **SLOB** | 임베디드 | 단순 |
| **SLUB** | 현대 | 성능 |

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: PCB 할당** → task_struct 캐시
**시나리오 2: inode** → inode_cachep
**시나리오 3: 네트워크 버퍼** → skbuff 캐시

**도입 체크리스트**: /proc/slabinfo 확인

**📢 섹션 요약 비유**: slabtop으로 확인!

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 할당 속도 10배 향상

**미래 전망**: per-CPU 슬랩, lockless

**참고 표준**: Linux Slab, SLUB, SLOB

**📢 섹션 요약 비유**: 슬랩은 커널 메모리 효율의 핵심!
