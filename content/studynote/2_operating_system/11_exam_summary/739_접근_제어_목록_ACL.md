# 접근 제어 목록 (ACL)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일/자원에 대해 특정 사용자나 그룹별로 세밀한 권한을 지정하는 목록
> 2. **가치**: 세밀한 권한 제어, 유연한 보안, 유닉스 권한 확장
> 3. **융합**: DAC, MAC, 파일 시스템, 보안 모델과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

접근 제어 목록(ACL, Access Control List)은 파일/자원에 대해 특정 사용자나 그룹별로 세밀한 권한을 지정하는 목록입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    접근 제어 목록 (ACL)                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              기본 유닉스 권한 vs ACL                           │ │
│   │                                                              │ │
│   │   기본 유닉스 권한 (3종류만):                                  │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Owner (소유자) │ Group (그룹) │ Other (기타)        │   │ │
│   │   │    rwx          │    rwx       │    rwx             │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │   한계: 사용자별 세밀한 권한 불가                              │ │
│   │                                                              │ │
│   │   ACL (세밀한 권한):                                          │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Entry         │  권한                              │   │ │
│   │   │  ──────────────────────────────────────────────     │   │ │
│   │   │  user:alice    │  rwx   (사용자 alice)              │   │ │
│   │   │  user:bob      │  r-x   (사용자 bob)                │   │ │
│   │   │  group:dev     │  rwx   (그룹 dev)                  │   │ │
│   │   │  group:qa      │  r--   (그룹 qa)                   │   │ │
│   │   │  other         │  ---   (기타)                      │   │ │
│   │   │  mask          │  rwx   (최대 권한 제한)             │   │ │
│   │   │  default:user:alice │ rwx  (새 파일 상속)           │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              ACL 엔트리 유형                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  user   : 사용자별 권한                              │   │ │
│   │   │  group  : 그룹별 권한                                │   │ │
│   │   │  other  : 기타 사용자 권한                           │   │ │
│   │   │  mask   : 최대 허용 권한 (제한)                      │   │ │
│   │   │  default: 새 파일/디렉터리에 상속                    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: ACL은 기본 유닉스 권한의 한계를 극복합니다.

**💡 비유**: ACL은 건물 출입카드에 사람별로 다른 권한을 부여하는 것과 같습니다.

**📢 섹션 요약 비유**: 사람별로 다르게 권한을 줘요!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **user entry** | 사용자 권한 | 특정 사용자 | 개인 카드 |
| **group entry** | 그룹 권한 | 특정 그룹 | 팀 카드 |
| **mask** | 권한 제한 | 최대 한도 | 상한선 |
| **default** | 상속 | 새 파일 | 기본값 |

**ACL 명령어**

| 명령 | 기능 |
|:---|:---|
| **getfacl** | ACL 조회 |
| **setfacl** | ACL 설정 |
| **chacl** | ACL 변경 |

**핵심 알고리즘**: 접근 요청 → ACL 조회 → 권한 확인 → 허용/거부

**📢 섹션 요약 비유**: mask가 최대 권한이에요!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 권한 모델**

| 모델 | 유연성 | 복잡도 |
|:---|:---:|:---:|
| **기본 유닉스** | 낮음 | 낮음 |
| **ACL** | 높음 | 중간 |
| **RBAC** | 높음 | 높음 |
| **MAC** | 제한적 | 높음 |

**📢 섹션 요약 비유**: ACL이 실용적이에요!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 프로젝트 공유** → ACL로 사용자별 권한
**시나리오 2: 웹 서버** → ACL로 접근 제어
**시나리오 3: 규정 준수** → 감사 로그

**도입 체크리스트**: FS 지원(acl), 관리 오버헤드

**📢 섹션 요약 비유**: ext4, XFS가 ACL 지원!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 세밀한 보안, 유연성

**미래 전망**: RBAC 통합, 자동화

**참고 표준**: POSIX ACL, NFSv4 ACL

**📢 섹션 요약 비유**: ACL은 정밀 보안의 도구!
