# 컨테이너 (네임스페이스 격리)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: OS 커널을 공유하며 프로세스 수준 격리를 제공하는 경량 가상화
> 2. **가치**: 빠른 시작, 낮은 오버헤드, 이식성, 클라우드 네이티브
> 3. **융합**: Namespace, Cgroups, Union FS와 통합된 격리

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

컨테이너(Container)는 OS 커널을 공유하며 네임스페이스로 격리된 프로세스 그룹입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    컨테이너 (Container)                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              VM vs 컨테이너                                   │ │
│   │                                                              │ │
│   │   VM (가상 머신):                                             │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  App 1  │  App 2  │  App 3  │  App 4                │   │ │
│   │   │  Bins   │  Bins   │  Bins   │  Bins                 │   │ │
│   │   │Guest OS │Guest OS │Guest OS │Guest OS               │   │ │
│   │   └─────────┴─────────┴─────────┴─────────┘             │   │ │
│   │                   Hypervisor                             │   │ │
│   │                   Host OS / Hardware                     │   │ │
│   │   크기: GB 단위, 시작: 분 단위                            │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   Container (컨테이너):                                      │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  App 1  │  App 2  │  App 3  │  App 4                │   │ │
│   │   │  Bins   │  Bins   │  Bins   │  Bins                 │   │ │
│   │   │ Libs    │ Libs    │ Libs    │ Libs                  │   │ │
│   │   └─────────┴─────────┴─────────┴─────────┘             │   │ │
│   │            Container Engine (Docker)                     │   │ │
│   │                   Host OS Kernel                         │   │ │
│   │                   Hardware                               │   │ │
│   │   크기: MB 단위, 시작: 초 단위                            │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              네임스페이스 (Namespace)                         │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  PID   : 프로세스 ID 격리                            │   │ │
│   │   │  NET   : 네트워크 인터페이스 격리                    │   │ │
│   │   │  MNT   : 마운트 포인트 격리                          │   │ │
│   │   │  IPC   : IPC 리소스 격리                             │   │ │
│   │   │  UTS   : 호스트명 격리                               │   │ │
│   │   │  USER  : 사용자/그룹 ID 격리                         │   │ │
│   │   │  CGROUP: Cgroup 루트 디렉터리 격리                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 컨테이너는 VM보다 가볍고 빠릅니다.

**💡 비유**: 컨테이너는 아파트에서 각 가구가 독립된 공간을 쓰는 것과 같습니다.

**📢 섹션 요약 비유**: 컨테이너는 초 단위 시작!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Namespace** | 격리 | 커널 리소스 분리 | 방 |
| **Cgroups** | 제한 | 자원 사용량 제한 | 계량기 |
| **Union FS** | 레이어 | 이미지 계층화 | 레이어 |
| **Runtime** | 실행 | 컨테이너 라이프사이클 | 관리자 |

**Docker 구조**

| 구성 | 설명 |
|:---|:---|
| **Image** | 읽기 전용 템플릿 |
| **Container** | 실행 중인 인스턴스 |
| **Registry** | 이미지 저장소 |
| **Dockerfile** | 빌드 스크립트 |

**핵심 알고리즘**: unshare → 격리된 공간 생성 → 프로세스 실행

**📢 섹션 요약 비유**: Cgroups가 자원을 제한!
