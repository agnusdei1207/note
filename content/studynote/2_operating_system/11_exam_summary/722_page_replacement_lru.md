# 페이지 교체 LRU (Least Recently Used) 알고리즘

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페이지 폴트 발생 시 빈 프레임이 없다면, 과거의 참조 이력을 바탕으로 **가장 오랫동안 사용되지 않은 페이지**를 교체 대상으로 선정하는 기법.
> 2. **가치**: "최근에 참조된 페이지는 가까운 미래에 다시 참조될 가능성이 높다"는 **시간 국부성(Temporal Locality)** 원리를 가장 잘 활용하여, 최적(Optimal) 알고리즘에 근접한 성능을 낸다.
> 3. **융합**: 하드웨어적인 참조 비트(Reference Bit)와 소프트웨어적인 큐(Queue) 또는 카운터(Counter) 관리가 결합되어 실질적인 메모리 관리의 핵심 로직을 구성한다.

---

### Ⅰ. LRU (Least Recently Used)의 동작 원리

- **개념**: 각 페이지마다 마지막 사용 시간을 기록하고, 가장 과거에 사용된 페이지를 추출한다.
- **가정**: 과거의 행동이 미래를 예측하는 가장 좋은 지표라는 믿음에 기반한다.
- **💡 비유**: **책상 위에 쌓인 책들 중 맨 아래에 깔린 책(가장 오랫동안 안 본 책)을 치우고** 새 책을 놓는 것과 같다.

---

### Ⅱ. LRU 실행 과정 시뮬레이션 (ASCII)

3개의 프레임이 있고 페이지 참조 순서가 `1, 2, 3, 4, 1, 2`인 경우다.

```ascii
    Ref: [ 1 ] [ 2 ] [ 3 ] [ 4 ] [ 1 ] [ 2 ]
    
    F1:  [ 1 ] [ 1 ] [ 1 ] [ 4 ] [ 4 ] [ 4 ]  <-- (1) 4 replaces 1 (LRU)
    F2:  [   ] [ 2 ] [ 2 ] [ 2 ] [ 1 ] [ 1 ]  <-- (2) 1 replaces 2 (LRU)
    F3:  [   ] [   ] [ 3 ] [ 3 ] [ 3 ] [ 2 ]  <-- (3) 2 replaces 3 (LRU)
    
    Fault:  Y     Y     Y     Y     Y     Y
```

**[분석]**
- 페이지 4가 들어올 때, 프레임에 있는 1, 2, 3 중 가장 오래전에 참조된 것은 1이므로 1이 쫓겨난다.
- 이후 1이 다시 들어올 때는 현재 프레임(4, 2, 3) 중 2가 가장 오래되었으므로 2를 교체한다.

---

### Ⅲ. LRU 구현 방식 (Implementation)

순수 LRU는 하드웨어 지원 없이는 구현 비용이 매우 비싸다.

| 방식 | 설명 | 오버헤드 |
|:---|:---|:---:|
| **계수기 (Counter)** | 각 페이지에 논리적 시계를 기록. | 사용 시마다 시간 쓰기 발생 (높음) |
| **스택 (Stack)** | 참조된 페이지를 스택 맨 위로 올림. | 포인터 수정 비용 (중간) |
| **근사 LRU (Clock)** | 참조 비트(Reference Bit)를 사용하여 시계 방향으로 탐색. | **낮음 (실무 표준)** |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. Belady의 모순 방지
- **특징**: LRU는 **스택 알고리즘**의 일종으로, 프레임 수가 늘어나면 페이지 폴트가 줄어들거나 같음이 수학적으로 보장된다. FIFO와 달리 Belady의 모순이 발생하지 않는다.

#### 2. 기술사적 인사이트
- **Clock 알고리즘 (Second Chance)**: 순수 LRU는 모든 메모리 참조마다 업데이트가 필요하여 너무 느리다. 따라서 대부분의 현대 운영체제는 참조 비트가 0인 페이지를 찾는 **Clock 알고리즘**을 통해 LRU를 흉내 낸다.
- **Working Set과의 관계**: LRU 성능을 극대화하려면 프로세스의 워킹 셋 크기만큼 프레임을 확보해 주는 것이 전제되어야 한다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **페이지 폴트 최소화**: 국부성 원리를 활용한 효율적인 메모리 교체.
- **시스템 처리량 안정화**: 불필요한 디스크 I/O 감소를 통한 CPU 이용률 유지.

#### 2. 미래 전망
최근에는 단순히 '시간'뿐만 아니라 '참조 빈도'를 결합한 **LRU-K**나 **ARC (Adaptive Replacement Cache)** 알고리즘이 고성능 파일 시스템이나 데이터베이스 캐시 계층에서 LRU를 대체하고 있다. 운영체제 레벨에서도 워크로드의 특성을 실시간으로 학습하여 LRU와 LFU(Least Frequently Used) 사이를 동적으로 오가는 적응형 모델로 진화 중이다.
