# 스래싱 (Thrashing, CPU 이용률 저하)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스가 실행보다 페이징에 더 많은 시간을 보내는 극단적 성능 저하 현상
> 2. **가치**: 메모리 부족의 증상, 워킹 셋/프레임 할당의 중요성
> 3. **융합**: 가상 메모리, 페이지 폴트, 다중 프로그래밍과 통합된 메모리 관리

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

스래싱(Thrashing)은 프로세스가 실행보다 페이징에 더 많은 시간을 보내는 극단적 성능 저하 현상입니다.

> **해설**: 스래싱은 메모리 부족의 위험한 증상입니다.

**💡 비유**: 스래싱은 너무 많은 그릇을 닦느라 요리를 못 하는 주방과 같습니다.

**📢 섹션 요약 비유**: 페이지만 옮기고 일은 안 해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **다중 프로그래밍** | 원인 | 프로세스 과다 | 과적 |
| **워킹 셋** | 기준 | 필요 프레임 | 최소량 |
| **페이지 폴트** | 증상 | 잦은 발생 | 신호 |
| **CPU 이용률** | 지표 | 급격히 하락 | 온도 |

**스래싱 징후**

| 징후 | 정상 | 스래싱 |
|:---|:---:|:---:|
| **폴트율** | 낮음 | 매우 높음 |
| **CPU** | 높음 | 낮음 |
| **디스크 I/O** | 중간 | 매우 높음 |
| **반응 시간** | 빠름 | 느림 |

**핵심 알고리즘**: CPU↓ 감지 → 원인 분석 → 프로세스 축소

**📢 섹션 요약 비유**: 디스크가 바쁘면 의심해요!
