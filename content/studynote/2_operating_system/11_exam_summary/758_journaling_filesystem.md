# 저널링 파일 시스템 (Journaling File System) 및 복구 원리

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일의 실제 데이터를 수정하기 전, 수행할 변경 사항을 별도의 **로그(Journal) 영역에 먼저 기록**하여 시스템 급중단 시 데이터 불일치를 방지하는 기술.
> 2. **가치**: 갑작스러운 전원 차단이나 시스템 크래시 발생 시, 전체 디스크를 전수 검사(fsck)하는 대신 **로그만 확인하여 수초 내에 일관성을 복구**할 수 있게 한다.
> 3. **융합**: 데이터베이스의 트랜잭션(ACID) 개념을 파일 시스템에 이식한 결과물이며, 현대 리눅스(ext3/4, XFS)와 윈도우(NTFS)의 표준 신뢰성 보장 메커니즘이다.

---

### Ⅰ. 저널링(Journaling)의 필요성: 비일관성 문제

- **문제 상황**: 파일을 저장하는 도중 정전이 발생하면, '파일 이름'은 생성되었는데 '데이터 블록'은 연결되지 않는 등의 **메타데이터 불일치**가 발생한다.
- **과거 해결책 (fsck)**: 부팅 시 전체 디스크를 훑으며 오류를 수정함. 테라바이트급 디스크에서는 복구에 수 시간이 소요되어 가용성이 심각하게 저하됨.

---

### Ⅱ. 저널링 동작 단계 (ASCII)

데이터 쓰기 작업의 3단계 흐름이다.

```ascii
    [ Step 1: Journal Write ]
    Write intended changes to the LOG area. (Metadata only or Data+Metadata)
    
    [ Step 2: Commit ]
    Wait for log to be safely stored. Add a "Commit" record.
    
    [ Step 3: Checkpoint (Actual Write) ]
    Update the real filesystem structures (i-nodes, Data blocks).
```

- **복구 시나리오 (Crash Recovery)**:
  - **로그에 기록 중 사고**: 로그가 미완성임 $\to$ 작업 무시 (Rollback 효과).
  - **로그 기록 완료 후 사고**: 로그 내용을 바탕으로 실제 파일 시스템에 반영 (**Redo / Playback**).

---

### Ⅲ. 저널링 모드 분류 (Linux ext4 기준)

| 모드 | 설명 | 성능 | 안정성 |
|:---|:---|:---:|:---:|
| **Ordered** | **(기본)** 메타데이터만 저널링하되, 데이터 쓰기 완료 후 저널을 커밋. | 보통 | 우수 |
| **Writeback** | 메타데이터만 저널링. 데이터 쓰기 순서는 보장 안 함. | **최상** | 보통 |
| **Journal** | 데이터와 메타데이터를 모두 저널링. 모든 것을 두 번 씀. | 최악 | **최상** |

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 대용량 스토리지와 복구 시간 (MTTR)
- **현상**: 서버 재부팅 시마다 파일 시스템 체크로 인해 서비스 복구가 늦어짐.
- **기술사적 결단**: 
  - 반드시 저널링이 지원되는 **ext4**나 **XFS**를 사용한다. 
  - 저널 영역(Log Device)을 별도의 고속 NVMe SSD에 배치하는 **External Journal** 기법을 통해 성능 저하를 방지한다.

#### 2. 기술사적 인사이트
- **Soft Updates vs Journaling**: FreeBSD 등에서 쓰이는 Soft Updates는 쓰기 순서를 논리적으로 맞춰 일관성을 유지한다. 저널링보다 성능은 좋으나 구현이 매우 복잡하여, 현대의 대중적인 OS는 관리 편의성이 높은 저널링을 선호한다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **신속한 장애 복구**: 디스크 크기와 상관없이 일정한 복구 시간(수 초 이내) 보장.
- **데이터 정합성**: 복잡한 파일 시스템 구조의 논리적 파손 원천 차단.

#### 2. 미래 전망
최근에는 저널링을 넘어, 원본을 건드리지 않고 항상 새 공간에 쓰는 **Log-structured File System (LFS)**이나 **Copy-on-Write (COW)** 기반의 파일 시스템(Btrfs, ZFS)이 주목받고 있다. 이는 지우기 전 쓰기가 안 되는 SSD의 물리적 특성과 완벽히 결합하여, 저널링 오버헤드 없이도 더 높은 수준의 일관성과 스냅샷 기능을 제공하는 차세대 표준으로 자리 잡고 있다.
