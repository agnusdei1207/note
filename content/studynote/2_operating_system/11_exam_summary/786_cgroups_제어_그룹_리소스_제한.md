# cgroups (제어 그룹 리소스 제한)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 그룹별 자원 사용량 제한, 계측, 격리
> 2. **가치**: 컨테이너, QoS, 멀티테넌시, 공정한 분배
> 3. **융합**: namespaces, Docker, Kubernetes와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

cgroups(Control Groups)는 프로세스를 그룹으로 묶어 자원(CPU, 메모리, I/O) 사용을 제한하고 모니터링하는 리눅스 커널 기능입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    cgroups (제어 그룹)                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              cgroups 아키텍처                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  계층 구조:                                            │   │ │
│   │   │                                                      │   │ │
│   │   │  /sys/fs/cgroup/                                     │   │ │
│   │   │    ├─ cpu/                                           │   │ │
│   │   │    │   ├─ container1/     ← 그룹                     │   │ │
│   │   │    │   │   ├─ cpu.shares                           │   │ │
│   │   │    │   │   ├─ cpu.cfs_quota_us                     │   │ │
│   │   │    │   │   └─ cgroup.procs                          │   │ │
│   │   │    │   └─ container2/                               │   │ │
│   │   │    ├─ memory/                                       │   │ │
│   │   │    │   ├─ memory.limit_in_bytes                     │   │ │
│   │   │    │   └─ memory.usage_in_bytes                     │   │ │
│   │   │    ├─ blkio/                                        │   │ │
│   │   │    │   └─ blkio.throttle.read_bps_device            │   │ │
│   │   │    └─ pids/                                         │   │ │
│   │   │        └─ pids.max                                  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              cgroups v1 vs v2                                 │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  cgroups v1:                                          │   │ │
│   │   │  • 각 컨트롤러별 별도 계층                             │   │ │
│   │   │  • /sys/fs/cgroup/cpu, memory, blkio 등              │   │ │
│   │   │  • 복잡한 계층 구조                                    │   │ │
│   │   │                                                      │   │ │
│   │   │  cgroups v2:                                          │   │ │
│   │   │  • 통합 계층 (단일 트리)                               │   │ │
│   │   │  • /sys/fs/cgroup/ (unified)                         │   │ │
│   │   │  • 스레드 모드, pressure 정보                         │   │ │
│   │   │  • 커널 4.5+                                          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: cgroups는 자원을 그룹 단위로 제한합니다.

**💡 비유**: cgroups는 각 팀에 예산을 배정하는 것과 같습니다.

**📢 섹션 요약 비유**: 컨테이너 자원 제한의 기반!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **cpu** | CPU 제한 | shares, quota | 시간 |
| **memory** | 메모리 제한 | limit, OOM | 공간 |
| **blkio** | I/O 제한 | throttle | 대역폭 |
| **pids** | 프로세스 수 | max | 개수 |

**주요 컨트롤러**

| 컨트롤러 | 파일 | 설명 |
|:---|:---|:---|
| **cpu.shares** | 상대적 비중 | 1024 기본 |
| **cpu.max** | 절대적 한도 | quota/period |
| **memory.max** | 한도 | 바이트 |
| **io.max** | IOPS/BPS | 장치별 |

**핵심 알고리즘**: 프로세스 → cgroup 등록 → 컨트롤러 적용 → 제한 실행

**📢 섹션 요약 비유**: v2가 단순!
