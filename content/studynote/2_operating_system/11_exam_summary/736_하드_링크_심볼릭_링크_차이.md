# 하드 링크 / 심볼릭 링크 차이

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하드 링크는 동일한 i-node를 공유, 심볼릭 링크는 경로를 가리키는 별도 파일
> 2. **가치**: 파일 공유, 공간 절약, 파일 시스템 이해
> 3. **융합**: i-node, 파일 시스템, 디렉터리와 통합된 링크 관리

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

하드 링크는 동일한 i-node를 공유하는 파일이고, 심볼릭 링크는 경로를 가리키는 별도의 파일입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    하드 링크 vs 심볼릭 링크                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              하드 링크 (Hard Link)                             │ │
│   │                                                              │ │
│   │   ┌─────────────┐     ┌─────────────┐                       │ │
│   │   │ file1.txt   │     │ file2.txt   │                       │ │
│   │   │ (하드 링크) │     │ (하드 링크) │                       │ │
│   │   └──────┬──────┘     └──────┬──────┘                       │ │
│   │          │                   │                               │ │
│   │          └───────┬───────────┘                               │ │
│   │                  ↓                                           │ │
│   │          ┌─────────────┐                                     │ │
│   │          │   i-node    │                                     │ │
│   │          │  (동일)     │                                     │ │
│   │          │  link=2     │                                     │ │
│   │          └──────┬──────┘                                     │ │
│   │                 │                                            │ │
│   │                 ↓                                            │ │
│   │          ┌─────────────┐                                     │ │
│   │          │  데이터 블록 │                                     │ │
│   │          └─────────────┘                                     │ │
│   │                                                              │ │
│   │   특징:                                                       │ │
│   │   • 동일한 i-node 번호                                       │ │
│   │   • link count 증가                                          │ │
│   │   • 원본 삭제해도 데이터 유지                                 │ │
│   │   • 같은 파일 시스템 내에서만 가능                            │ │
│   │   • 디렉터리 링크 불가                                       │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              심볼릭 링크 (Symbolic Link / Soft Link)          │ │
│   │                                                              │ │
│   │   ┌─────────────┐     ┌─────────────┐                       │ │
│   │   │ link.txt    │     │ target.txt  │                       │ │
│   │   │ (심볼릭)    │ ──→ │ (원본)      │                       │ │
│   │   │ i-node: 100 │     │ i-node: 200 │                       │ │
│   │   └─────────────┘     └─────────────┘                       │ │
│   │        │                                                    │ │
│   │        │ 경로 정보 저장                                      │ │
│   │        │ "/path/to/target.txt"                              │ │
│   │        ↓                                                    │ │
│   │   ┌─────────────┐                                           │ │
│   │   │  별도 파일   │                                           │ │
│   │   │ (경로만 저장)│                                           │ │
│   │   └─────────────┘                                           │ │
│   │                                                              │ │
│   │   특징:                                                       │ │
│   │   • 별도의 i-node                                            │ │
│   │   • 원본 경로만 저장                                         │ │
│   │   • 원본 삭제 시 링크 끊김 (dangling link)                   │ │
│   │   • 다른 파일 시스템 가능                                    │ │
│   │   • 디렉터리 링크 가능                                       │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 하드 링크는 실제 파일, 심볼릭 링크는 바로가기입니다.

**💡 비유**: 하드 링크는 같은 문서의 복사본이 아니라 같은 문서, 심볼릭 링크는 바로가기입니다.

**📢 섹션 요약 비유**: 하드는 본체, 심볼릭은 바로가기!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **i-node 공유** | 하드 링크 | link count++ | 같은 책 |
| **경로 저장** | 심볼릭 | 경로 문자열 | 주소 |
| **link count** | 참조 수 | 0이면 삭제 | 대여 수 |
| **dangling** | 끊긴 링크 | 원본 없음 | 끊긴 전화 |

**비교 표**

| 특성 | 하드 링크 | 심볼릭 링크 |
|:---|:---:|:---:|
| **i-node** | 동일 | 별도 |
| **원본 삭제** | 유지 | 끊김 |
| **파일 시스템** | 동일만 | 상관없음 |
| **디렉터리** | 불가 | 가능 |

**핵심 알고리즘**: 링크 생성 → i-node/link 파일 생성 → 참조

**📢 섹션 요약 비유**: 심볼릭이 더 유연해요!

**📢 섹션 요약 비유**: 백업엔 하드, 라이브러리엔 심볼릭!
