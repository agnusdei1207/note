# 유니커널 (Unikernel 경량 OS)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단일 애플리케이션만 실행하는 초경량 OS
> 2. **가치**: 빠른 부팅, 작은 크기, 높은 보안, 클라우드 최적
> 3. **융합**: 라이브러리 OS, VM, 컨테이너와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

유니커널(Unikernel)은 단일 애플리케이션만 실행하기 위해 필요한 기능만 포함하는 초경량 운영체제입니다.


> **해설**: 유니커널은 필요한 라이브러리만 링크하여 단일 이미지로 빌드합니다.

**💡 비유**: 유니커널은 특수 목적의 맞춤 정장입니다.

**📢 섹션 요약 비유**: 필요한 것만!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **LibOS** | 라이브러리 | OS 서비스 | 도구 |
| **Hypervisor** | 하이퍼바이저 | 실행 환경 | 무대 |
| **Solo5** | 추상화 | 하드웨어 인터페이스 | 기반 |
| **Build** | 빌드 | 링크 | 조립 |

**주요 특징**

| 특징 | 설명 |
|:---|:---|
| **단일 주소 공간** | 커널/사용자 분리 없음 |
| **단일 프로세스** | 멀티태스킹 없음 |
| **정적 링크** | 필요한 것만 |
| **직접 하드웨어** | 최소 추상화 |

**핵심 알고리즘**: 앱 + LibOS → 링크 → 단일 이미지 → VM 실행

**📢 섹션 요약 비유**: 부팅이 ms!
