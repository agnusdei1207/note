# I/O 직접 메모리 접근 (DMA)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU 개입 없이 장치가 메모리에 직접 접근하는 기술
> 2. **가치**: CPU 부하 감소, 고속 데이터 전송, 버스 활용 효율
> 3. **융합**: 버스, 인터럽트, 캐시 일관성과 통합된 I/O

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

DMA(Direct Memory Access)는 CPU 개입 없이 장치가 메모리에 직접 접근하여 데이터를 전송하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    I/O 직접 메모리 접근 (DMA)                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              PIO vs DMA                                       │ │
│   │                                                              │ │
│   │   PIO (Programmed I/O):                                      │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │   CPU         장치           메모리                 │   │ │
│   │   │    │           │               │                    │   │ │
│   │   │    │←─read─────┤               │                    │   │ │
│   │   │    │           │               │                    │   │ │
│   │   │    ├────write────────────────→│                    │   │ │
│   │   │    │           │               │                    │   │ │
│   │   │   CPU가 모든 데이터를 직접 이동                     │   │ │
│   │   │   → CPU 사용률 높음, 느림                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   DMA (Direct Memory Access):                                │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │   CPU        DMA 컨트롤러     장치      메모리      │   │ │
│   │   │    │              │           │          │          │   │ │
│   │   │    │──설정──────→│           │          │          │   │ │
│   │   │    │              │←─transfer─┤          │          │   │ │
│   │   │    │              │                      │          │   │ │
│   │   │    │←──완료 인터럽트──────────────────┤          │   │ │
│   │   │   CPU는 설정만, DMA가 직접 전송                      │   │ │
│   │   │   → CPU 사용률 낮음, 빠름                           │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              DMA 전송 과정                                    │ │
│   │                                                              │ │
│   │   1. CPU가 DMA 컨트롤러에 설정:                              │ │
│   │      - 소스 주소 (장치/메모리)                               │ │
│   │      - 목적지 주소 (메모리/장치)                             │ │
│   │      - 전송 크기 (byte count)                                │ │
│   │                                                              │ │
│   │   2. DMA 컨트롤러가 버스 제어권 획득 (BR/BG)                 │ │
│   │                                                              │ │
│   │   3. 데이터 전송 (Cycle Stealing / Burst)                    │ │
│   │                                                              │ │
│   │   4. 완료 후 CPU에 인터럽트                                  │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: DMA는 CPU를 데이터 전송에서 해방시킵니다.

**💡 비유**: DMA는 배달 서비스, PIO는 직접 픽업과 같습니다.

**📢 섹션 요약 비유**: CPU는 감독만 해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **DMA Controller** | 전송 관리 | 주소, 카운트 | 배달원 |
| **Bus Request** | 버스 요청 | BR 신호 | 예약 |
| **Bus Grant** | 버스 승인 | BG 신호 | 승인 |
| **Interrupt** | 완료 알림 | IRQ | 알림 |

**전송 모드**

| 모드 | 설명 | 특징 |
|:---|:---|:---|
| **Burst** | 연속 전송 | 빠름, 버스 독점 |
| **Cycle Steal** | 한 번에 일부 | 공정, 느림 |

**핵심 알고리즘**: 설정 → BR → BG → 전송 → 완료 IRQ

**📢 섹션 요약 비유**: Burst가 빠르고 Cycle Steal이 공정!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: I/O 방식**

| 방식 | CPU 부하 | 속도 |
|:---|:---:|:---:|
| **PIO** | 높음 | 느림 |
| **Interrupt** | 중간 | 중간 |
| **DMA** | 낮음 | 빠름 |

**📢 섹션 요약 비유**: 대량 전송엔 DMA!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 디스크 I/O** → DMA 필수
**시나리오 2: 네트워크** → DMA + Ring Buffer
**시나리오 3: GPU** → Bus Master DMA

**도입 체크리스트**: 버스 마스터링, 캐시 일관성

**📢 섹션 요약 비유**: 캐시 무효화가 중요!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: CPU 사용률 90% 감소

**미래 전망**: IOMMU, RDMA

**참고 표준**: PCI DMA, IOMMU

**📢 섹션 요약 비유**: DMA는 고속 I/O의 핵심!
