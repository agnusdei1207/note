# 블로킹 / 논블로킹 / 비동기 I/O

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: I/O 완료를 기다리는 방식의 차이 (블로킹 vs 논블로킹 vs 비동기)
> 2. **가치**: 응답성 향상, 리소스 효율, 확장성
> 3. **융합**: epoll, kqueue, io_uring, 이벤트 루프와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

블로킹/논블로킹/비동기 I/O는 I/O 작업 완료를 기다리는 방식의 차이입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    블로킹 / 논블로킹 / 비동기 I/O                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              블로킹 I/O (Blocking)                            │ │
│   │                                                              │ │
│   │   Thread                    Kernel                           │ │
│   │     │                         │                               │ │
│   │     │──read()────────────────→│                               │ │
│   │     │                         │                               │ │
│   │     │    [스레드 대기]        │   I/O 진행                   │ │
│   │     │                         │                               │ │
│   │     │←────데이터 반환─────────│                               │ │
│   │     │                         │                               │ │
│   │   특징: 간단하지만 스레드가 블록됨                            │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              논블로킹 I/O (Non-blocking)                      │ │
│   │                                                              │ │
│   │   Thread                    Kernel                           │ │
│   │     │                         │                               │ │
│   │     │──read() (O_NONBLOCK)───→│                               │ │
│   │     │←─EAGAIN (준비 안 됨)────│                               │ │
│   │     │                         │                               │ │
│   │     │──다른 작업──             │                               │ │
│   │     │                         │                               │ │
│   │     │──read() (poll)─────────→│                               │ │
│   │     │←────데이터 반환─────────│                               │ │
│   │     │                         │                               │ │
│   │   특징: 스레드가 계속 실행, 폴링 필요                        │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              비동기 I/O (Asynchronous / AIO)                  │ │
│   │                                                              │ │
│   │   Thread                    Kernel                           │ │
│   │     │                         │                               │ │
│   │     │──aio_read() (요청만)───→│                               │ │
│   │     │←─즉시 반환 (요청 ID)────│                               │ │
│   │     │                         │                               │ │
│   │     │──다른 작업 계속──        │   I/O 진행                   │ │
│   │     │                         │                               │ │
│   │     │                         │──완료 시그널/콜백──┐         │ │
│   │     │←─────────────────────────────────────────────┘         │ │
│   │     │                         │                               │ │
│   │   특징: 완료 알림, 진정한 비동기                              │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              비교 표                                          │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  방식      │  반환    │  대기  │  복잡도  │  확장성 │   │ │
│   │   │  ─────────────────────────────────────────────────  │   │ │
│   │   │  Blocking  │  늦음   │  O     │  낮음    │  낮음   │   │ │
│   │   │  Non-block │  즉시   │  폴링  │  중간    │  중간   │   │ │
│   │   │  Async     │  즉시   │  X     │  높음    │  높음   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 비동기 I/O가 확장성이 가장 좋습니다.

**💡 비유**: 블로킹은 줄서서 기다리기, 비동기는 번호표 뽑고 다른 일 하기입니다.

**📢 섹션 요약 비유**: 비동기가 최고!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Blocking** | 대기 | 스레드 슬립 | 줄서기 |
| **O_NONBLOCK** | 즉시 반환 | EAGAIN | 확인만 |
| **epoll** | 이벤트 통지 | 준비된 FD | 알림 |
| **io_uring** | 링 버퍼 | SQ/CQ | 큐 |

**I/O 멀티플렉싱**

| API | OS | 특징 |
|:---|:---|:---|
| **select** | POSIX | 1024 한계 |
| **poll** | POSIX | 무제한 |
| **epoll** | Linux | O(1) |
| **kqueue** | BSD | O(1) |
| **IOCP** | Windows | 비동기 |

**핵심 알고리즘**: 요청 → 이벤트 통지 → 처리

**📢 섹션 요약 비유**: epoll이 리눅스 표준!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 이벤트 모델**

| 모델 | 확장성 | 복잡도 |
|:---|:---:|:---:|
| **Thread per Connection** | 낮음 | 낮음 |
| **Event Loop** | 높음 | 중간 |
| **io_uring** | 매우 높음 | 높음 |

**📢 섹션 요약 비유**: Node.js가 이벤트 루프!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 웹 서버** → epoll + event loop (nginx)
**시나리오 2: DB** → io_uring
**시나리오 3: 파일 복사** → blocking (간단)

**도입 체크리스트**: 연결 수, 응답 시간 요구

**📢 섹션 요약 비유**: C10K 문제 해결!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 동시 연결 100배 증가

**미래 전망**: io_uring, DPDK

**참고 표준**: POSIX AIO, io_uring

**📢 섹션 요약 비유**: 비동기는 확장성의 핵심!
