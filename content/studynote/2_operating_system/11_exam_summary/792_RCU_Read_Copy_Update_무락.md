# RCU (Read Copy Update 무락)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 읽기는 락 없이, 쓰기는 복사 후 교체
> 2. **가치**: 읽기 성능 극대화, 커널 동기화
> 3. **융합**: Grace Period, 콜백, SMP와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

RCU(Read-Copy-Update)는 읽기는 락 없이 수행하고, 쓰기는 복사본 수정 후 포인터 교체로 수행하는 동기화 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    RCU (Read Copy Update)                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              RCU 동작 원리                                     │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Reader (락 없이):                                    │   │ │
│   │   │  rcu_read_lock();                                     │   │ │
│   │   │  p = rcu_dereference(ptr);   // 포인터 읽기          │   │ │
│   │   │  // p 사용                                            │   │ │
│   │   │  rcu_read_unlock();                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  Updater (복사 후 교체):                               │   │ │
│   │   │  new = copy(old);              // 복사                │   │ │
│   │   │  modify(new);                  // 수정                │   │ │
│   │   │  rcu_assign_pointer(ptr, new); // 원자적 교체        │   │ │
│   │   │  synchronize_rcu();            // Grace Period 대기   │   │ │
│   │   │  free(old);                    // 이전 버전 해제      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              Grace Period                                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Timeline:                                              │   │ │
│   │   │                                                      │   │ │
│   │   │  Reader 1: ──────[rcu_read_lock ... unlock]──────    │   │ │
│   │   │  Reader 2: ──────────[rcu_read_lock ... unlock]──    │   │ │
│   │   │  Reader 3: ─────────────────[rcu_read_lock ...       │   │ │
│   │   │                          ↑                            │   │ │
│   │   │                    ptr 교체                           │   │ │
│   │   │                          │← Grace Period →|           │   │ │
│   │   │                                             ↑         │   │ │
│   │   │                                    모든 Reader 완료   │   │ │
│   │   │                                    → old 해제 가능    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: RCU는 읽기 락 없이, 쓰기는 지연 해제로 동시성을 보장합니다.

**💡 비유**: RCU는 책을 읽는 동안 새 판을 준비해 교체하는 것과 같습니다.

**📢 섹션 요약 비유**: 읽기 락 없이!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **rcu_read_lock** | 진입 | 선점 비활성화 | 입장 |
| **rcu_dereference** | 읽기 | 메모리 배리어 | 조회 |
| **rcu_assign_pointer** | 쓰기 | 원자 교체 | 등록 |
| **synchronize_rcu** | 대기 | Grace Period | 완료 |

**주요 API**

| API | 설명 |
|:---|:---|
| **call_rcu()** | 비동기 콜백 |
| **synchronize_rcu()** | 동기 대기 |
| **kfree_rcu()** | RCU 후 해제 |

**핵심 알고리즘**: read_lock → 읽기 → unlock || write → 교체 → grace period → free

**📢 섹션 요약 비유**: synchronize_rcu가 핵심!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 동기화**

| 기법 | 읽기 | 쓰기 | 용도 |
|:---|:---:|:---:|:---|
| **Spinlock** | 락 | 락 | 짧음 |
| **RWLock** | 공유 | 배타 | 균등 |
| **RCU** | 없음 | 지연 | 읽기 많음 |

**📢 섹션 요약 비유**: RCU가 읽기에 최고!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 커널 리스트** → RCU
**시나리오 2: 설정 데이터** → RCU
**시나리오 3: 쓰기 많음** → 부적합

**도입 체크리스트**: 읽기/쓰기 비율, 메모리 비용

**📢 섹션 요약 비유**: 읽기 90% 이상에 RCU!
