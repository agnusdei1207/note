# FAT 방식 (연결 할당 최적화)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연결 할당의 포인터들을 별도 테이블(FAT)에 모아 캐싱하여 직접 접근을 가능하게 한 방식
> 2. **가치**: 연결 할당의 장점 유지, 직접 접근 개선, 단순한 구조
> 3. **융합**: 파일 할당, USB/SD카드, MS-DOS와 통합된 파일 시스템

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

FAT(File Allocation Table)는 연결 할당의 포인터들을 별도 테이블에 모아 캐싱하여 직접 접근을 가능하게 한 파일 시스템입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    FAT (File Allocation Table)                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              기본 연결 할당 문제                               │ │
│   │                                                              │ │
│   │   ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐                 │ │
│   │   │  1  │ ─→ │  5  │ ─→ │  9  │ ─→ │  3  │ NULL            │ │
│   │   └─────┘    └─────┘    └─────┘    └─────┘                 │ │
│   │   포인터가 각 블록에 있어서 직접 접근 시 처음부터 순회 필요    │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              FAT 해결책                                       │ │
│   │                                                              │ │
│   │   FAT 테이블 (메모리에 캐싱됨):                                │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Entry │ 0  │ 1  │ 2  │ 3  │ 4  │ 5  │ 6  │ 7  │...│   │ │
│   │   │────────┼────┼────┼────┼────┼────┼────┼────┼────┼───│   │ │
│   │   │ Value  │ -  │ 5  │ -  │EOF │ -  │ 9  │ -  │ -  │...│   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                    ↑        ↑              ↑                 │ │
│   │                  File A   File B         File A              │ │
│   │                  시작=1   시작=3         다음=5              │ │
│   │                                                              │ │
│   │   File A 체인: 1 → 5 → 9 → 3 (EOF)                          │ │
│   │   File B 체인: 3 (EOF)                                       │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              FAT 버전                                         │ │
│   │                                                              │ │
│   │   ┌───────────────┬───────────────┬─────────────────────┐   │ │
│   │   │ FAT12         │ FAT16         │ FAT32               │   │ │
│   │   │ 12-bit entry  │ 16-bit entry  │ 32-bit entry        │   │ │
│   │   │ 최대 4,096 클 │ 최대 65,536  │ 최대 2^28 클러스터   │   │ │
│   │   │ 러스터        │ 클러스터     │                      │   │ │
│   │   │ 플로피        │ 소용량       │ 대용량              │   │ │
│   │   └───────────────┴───────────────┴─────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: FAT는 연결 할당의 단점을 테이블 캐싱으로 해결합니다.

**💡 비유**: FAT는 각 책의 다음 권 위치를 적어둔 목록과 같습니다.

**📢 섹션 요약 비유**: 테이블에 다음 위치를 적어둬요!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **FAT 테이블** | 포인터 배열 | 클러스터 체인 | 목록 |
| **클러스터** | 할당 단위 | 여러 섹터 | 상자 |
| **디렉터리** | 파일 시작 | 시작 클러스터 | 색인 |
| **EOF** | 종료 표시 | 0xFFFFFFFF | 끝 |

**FAT 장단점**

| 장점 | 단점 |
|:---|:---|
| 단순한 구조 | 대용량 비효율 |
| 직접 접근 가능 | 단편화 |
| 호환성 좋음 | 보안 부족 |

**핵심 알고리즘**: 시작 클러스터 → FAT 조회 → 체인 따라가기 → EOF

**📢 섹션 요약 비유**: USB/SD카드에 많이 써요!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: FAT vs i-node**

| 비교 항목 | FAT | i-node |
|:---|:---:|:---:|
| **직접 접근** | 가능 | 빠름 |
| **대용량** | 비효율 | 효율 |
| **단편화** | 있음 | 적음 |
| **호환성** | 최고 | OS 의존 |

**📢 섹션 요약 비유**: FAT은 호환성의 왕이에요!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: USB/SD** → FAT32/exFAT (호환성)
**시나리오 2: 윈도우 시스템** → NTFS
**시나리오 3: 리눅스** → ext4

**도입 체크리스트**: 호환성, 용량, 보안

**📢 섹션 요약 비유**: 이동식 미디어엔 FAT!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 호환성, 단순함

**미래 전망**: exFAT 확대, 대용량 지원

**참고 표준**: Microsoft FAT, exFAT

**📢 섹션 요약 비유**: FAT은 역사적인 파일 시스템이에요!
