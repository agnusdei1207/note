# 페이지 컬러링 (캐시 색상)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 물리 페이지를 캐시 색상으로 분류하여 충돌 감소
> 2. **가치**: 캐시 성능 향상, 폴스 셰어링 방지
> 3. **융합**: 페이지 할당, NUMA, 멀티코어와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

페이지 컬러링(Page Coloring)은 물리 페이지를 캐시 인덱스 기준으로 색상으로 분류하여 캐시 충돌을 줄이는 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    페이지 컬러링                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              문제: 캐시 충돌                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Set-Associative Cache:                               │   │ │
│   │   │                                                      │   │ │
│   │   │  VPN 비트: [Tag | Index | Offset]                    │   │ │
│   │   │                       ↑                               │   │ │
│   │   │              캐시 인덱스                              │   │ │
│   │   │                                                      │   │ │
│   │   │  문제:                                                │   │ │
│   │   │  • 다른 VPN이 같은 Index 사용                         │   │ │
│   │   │  • 캐시 충돌 증가                                     │   │ │
│   │   │  • 성능 저하                                          │   │ │
│   │   │                                                      │   │ │
│   │   │  예: VPN 0x0, 0x100, 0x200 → 같은 캐시 라인 매핑      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              페이지 컬러링                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  색상 = VPN mod 색상_수                               │   │ │
│   │   │                                                      │   │ │
│   │   │  색상 수 = 캐시 크기 / (페이지 크기 × associativity)  │   │ │
│   │   │                                                      │   │ │
│   │   │  예: 4MB 캐시, 4KB 페이지, 4-way                      │   │ │
│   │   │  색상 수 = 4MB / (4KB × 4) = 256 색상                 │   │ │
│   │   │                                                      │   │ │
│   │   │  할당:                                                │   │ │
│   │   │  • 같은 색상 페이지끼리 충돌 가능                      │   │ │
│   │   │  • 다른 색상은 충돌 없음                               │   │ │
│   │   │  • 컬러링으로 색상 분배 최적화                          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 컬러링은 물리 페이지 할당 시 캐시 인덱스를 고려합니다.

**💡 비유**: 페이지 컬러링은 도서관에서 책을 주제별로 분류하는 것과 같습니다.

**📢 섹션 요약 비유**: 색상 = 캐시 인덱스!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Color** | 색상 | 캐시 인덱스 | 주제 |
| **Page** | 페이지 | 4KB 단위 | 책 |
| **Allocator** | 할당자 | 색상 분배 | 사서 |
| **Bin** | 버킷 | 색상별 풀 | 선반 |

**주요 기법**

| 기법 | 설명 |
|:---|:---|
| **Round-robin** | 색상 순환 |
| **Bin filling** | 버킷 채우기 |
| **Best-fit** | 최적 색상 |

**핵심 알고리즘**: VPN → 색상 계산 → 해당 색상에서 PFN 할당

**📢 섹션 요약 비유**: PFN low bits가 색상!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 대형 페이지** → 컬러링 불필요
**시나리오 2: 4KB 페이지** → 컬러링 효과적
**시나리오 3: NUMA** → NUMA + 컬러링

**도입 체크리스트**: 캐시 토폴로지, 페이지 크기

**📢 섹션 요약 비유**: ARM, MIPS에서 많이 사용!
