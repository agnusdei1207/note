# 람포트 타임스탬프 (논리적 시계)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분산 시스템에서 이벤트 순서를 결정하는 논리적 시계
> 2. **가치**: 인과 관계 파악, 일관성, 순서 보장
> 3. **융합**: 벡터 클럭, 분산 합의, 이벤트 소싱과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

람포트 타임스탬프(Lamport Timestamp)는 Leslie Lamport가 제안한 분산 시스템에서 이벤트 순서를 결정하는 논리적 시계입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    람포트 타임스탬프                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              논리적 시계 규칙                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  규칙:                                                  │   │ │
│   │   │                                                      │   │ │
│   │   │  1. 각 프로세스는 이벤트 발생 시 자신의 시계 증가        │   │ │
│   │   │     C_i = C_i + 1                                     │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 메시지 전송 시 현재 시계 포함                       │   │ │
│   │   │     send(m, C_i)                                      │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 메시지 수신 시:                                    │   │ │
│   │   │     C_i = max(C_i, timestamp(m)) + 1                  │   │ │
│   │   │                                                      │   │ │
│   │   │  happened-before (→):                                 │   │ │
│   │   │  A → B 이면 C(A) < C(B)                               │   │ │
│   │   │  (역은 성립 안 함)                                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              예시                                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  P1: 1───2───3───4                                  │   │ │
│   │   │       │       │                                      │   │ │
│   │   │       │m1     │m2                                    │   │ │
│   │   │       ↓       ↓                                      │   │ │
│   │   │  P2: 1───3───4───5                                   │   │ │
│   │   │           │       │                                  │   │ │
│   │   │           │m3     │                                  │   │ │
│   │   │           ↓       │                                  │   │ │
│   │   │  P3: 1───4───5───6                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  순서: P1:1 < P1:2 < P2:1 < P3:1 < P1:3 < P2:3 ...   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 람포트 시계는 인과 관계를 부분적으로만 파악합니다.

**💡 비유**: 람포트 시계는 각자 시계를 가진 여행자가 소통하며 시간을 맞추는 것과 같습니다.

**📢 섹션 요약 비유**: 수신 시 max + 1!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **C_i** | 시계 | 카운터 | 시간 |
| **Send** | 전송 | 타임스탬프 포함 | 발신 |
| **Receive** | 수신 | 시계 동기화 | 수신 |
| **happened-before** | 순서 | 인과 관계 | 선후 |

**총 순서 (Total Order)**

| 조건 | 설명 |
|:---|:---|
| **C(A) < C(B)** | 타임스탬프 비교 |
| **C(A) = C(B)** | 프로세스 ID로 결정 |

**핵심 알고리즘**: 이벤트 → C++ → 메시지 시 수신자 max(C, msg.ts) + 1

**📢 섹션 요약 비유**: 벡터 클럭이 더 정확!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 순서 보장** → Lamport
**시나리오 2: 충돌 해결** → Vector Clock
**시나리오 3: 실시간** → Hybrid Logical Clock

**도입 체크리스트**: 인과 관계 필요성, 오버헤드

**📢 섹션 요약 비유**: 간단하면 Lamport!
