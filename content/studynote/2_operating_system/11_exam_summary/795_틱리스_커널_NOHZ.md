# 틱리스 커널 (NOHZ)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 불필요한 타이머 틱을 제거하여 절전 및 성능 향상
> 2. **가치**: 배터리 수명 연장, 지터 감소, 실시간성
> 3. **융합**: idle, full, adaptive 모드와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

틱리스 커널(Tickless Kernel, NOHZ)은 일정한 주기의 타이머 인터럽트(tick)를 제거하여 전력 소모를 줄이는 기술입니다.

```

│                    틱리스 커널 (NOHZ)                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              기존 HZ vs NOHZ                                   │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  기존 (HZ=1000):                                      │   │ │
│   │   │  ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐           │   │ │
│   │   │  │tick│tick│tick│tick│tick│tick│tick│tick│ 1ms 간격  │   │ │
│   │   │  └──┘ └──┘ └──┘ └──┘ └──┘ └──┘ └──┘ └──┘           │   │ │
│   │   │                                                      │   │ │
│   │   │  문제:                                                 │   │ │
│   │   │  • 유휴 시에도 인터럽트                                 │   │ │
│   │   │  • 절전 방해                                           │   │ │
│   │   │  • 불필요한 웨이크업                                   │   │ │
│   │   │                                                      │   │ │
│   │   │  NOHZ (Tickless):                                     │   │ │
│   │   │  ┌──┐                 ┌──┐                           │   │ │

│   │   │  └──┘                 └──┘                           │   │ │
│   │   │                                                      │   │ │

│   │   │  • 딥 슬립 가능                                        │   │ │

│   │   │  • 유휴 시 틱 비활성화                                  │   │ │
│   │   │  • 배터리 절약                                         │   │ │
│   │   │  • 노트북, 모바일                                      │   │ │
```

> **해설**: NOHZ는 필요한 시점에만 틱을 발생시킵니다.

**💡 비유**: 틱리스는 필요할 때만 알람을 맞추는 것과 같습니다.

**📢 섹션 요약 비유**: 유휴 시 틱 없음!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **tick_sched** | 스케줄러 | 틱 관리 | 관리자 |
| **hrtimer** | 타이머 | 다음 이벤트 | 알람 |
| **clockevent** | 하드웨어 | 프로그래밍 | 시계 |
| **next_event** | 다음 | 예정 시간 | 예약 |

**주요 함수**

| 함수 | 설명 |
|:---|:---|
| **tick_nohz_idle_enter()** | 유휴 진입 |
| **tick_nohz_stop_sched_tick()** | 틱 정지 |
| **tick_nohz_restart()** | 틱 재시작 |

**핵심 알고리즘**: 다음 이벤트 계산 → 타이머 프로그래밍 → 슬립 → 이벤트 시 웨이크업

**📢 섹션 요약 비유**: hrtimer가 다음 틱 예약!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 절전 기술**

| 기술 | 수준 | 효과 |
|:---|:---:|:---:|
| **HZ** | 커널 | 없음 |
| **NOHZ_IDLE** | 유휴 | 높음 |
| **NOHZ_FULL** | 실행 | 최고 |
| **C-state** | CPU | 높음 |

**📢 섹션 요약 비유**: NOHZ_FULL이 최고 절전!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 모바일** → NOHZ_IDLE
**시나리오 2: 실시간** → NOHZ_FULL + isolcpus
**시나리오 3: 서버** → NOHZ_IDLE

**도입 체크리스트**: 워크로드, 코어 격리

**📢 섹션 요약 비유**: HPC는 NOHZ_FULL!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 배터리 10-20% 연장, 지터 90% 감소

**미래 전망**: 완전 틱리스, virtualization 지원

**참고 표준**: Linux Kernel NOHZ

**📢 섹션 요약 비유**: 틱리스는 현대 커널의 표준!
