# 가비지 컬렉션 (블록 지우기)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: SSD에서 유효하지 않은 페이지를 정리하여 새로운 쓰기 공간을 확보하는 과정
> 2. **가치**: 쓰기 공간 확보, 성능 유지, 쓰기 증폭 발생
> 3. **융합**: FTL, TRIM, 마모 평준화와 통합된 플래시 관리

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

가비지 컬렉션(GC)은 SSD에서 유효하지 않은 페이지를 정리하여 새로운 쓰기 공간을 확보하는 과정입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    가비지 컬렉션 (Garbage Collection)                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              GC 필요 이유                                     │ │
│   │                                                              │ │
│   │   플래시 메모리 특성:                                          │ │
│   │   • 덮어쓰기 불가: 쓰려면 먼저 지워야 함                        │ │
│   │   • 지우기 단위: 블록 (여러 페이지)                            │ │
│   │   • 삭제된 파일: OS는 삭제, SSD는 모름                        │ │
│   │                                                              │ │
│   │   결과: 유효하지 않은 페이지가 늘어남 → GC로 정리               │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              GC 과정                                          │ │
│   │                                                              │ │
│   │   GC 전 (블록 A):                                              │ │
│   │   ┌─────┬─────┬─────┬─────┐                                 │ │
│   │   │유효 │무효 │유효 │무효 │                                 │ │
│   │   │ P1  │(삭제)│ P3  │(삭제)│                                 │ │
│   │   └─────┴─────┴─────┴─────┘                                 │ │
│   │                                                              │ │
│   │   GC 과정:                                                    │ │
│   │   1. 유효 페이지 복사 (P1, P3 → 새 블록)                       │ │
│   │   2. 기존 블록 지우기                                          │ │
│   │   3. 빈 블록 확보                                              │ │
│   │                                                              │ │
│   │   GC 후:                                                      │ │
│   │   ┌─────┬─────┬─────┬─────┐    ┌─────┬─────┬─────┬─────┐   │ │
│   │   │ P1  │ P3  │     │     │    │     │     │     │     │   │ │
│   │   │(복사)│(복사)│ 빈  │ 빈  │    │ 빈  │ 빈  │ 빈  │ 빈  │   │ │
│   │   └─────┴─────┴─────┴─────┘    └─────┴─────┴─────┴─────┘   │ │
│   │       새 블록                     지워진 블록                │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              쓰기 증폭 (Write Amplification)                   │ │
│   │                                                              │ │
│   │   WA = 실제 플래시 쓰기량 / Host 요청 쓰기량                   │ │
│   │                                                              │ │
│   │   예: 4KB 쓰기 요청 → GC로 16KB 쓰기 발생 → WA = 4            │ │
│   │                                                              │ │
│   │   WA ↑ → 수명 ↓, 성능 ↓                                      │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: GC는 SSD 성능과 수명에 큰 영향을 미칩니다.

**💡 비유**: GC는 책장에서 쓸모없는 책을 치우고 새 책을 위한 공간을 만드는 것과 같습니다.

**📢 섹션 요약 비유**: 유효한 건 옮기고 나머지는 지워요!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **유효 페이지** | 보존 대상 | 복사 | 살릴 책 |
| **무효 페이지** | 삭제 대상 | 버림 | 버릴 책 |
| **빈 블록** | 새 공간 | GC 결과 | 빈 책장 |
| **쓰기 증폭** | 부작용 | 추가 쓰기 | 낭비 |

**GC 유형**

| 유형 | 시점 | 영향 |
|:---|:---|:---|
| **Background** | idle 시 | 최소 |
| **Foreground** | 쓰기 필요 시 | 지연 |
| **Idle** | 대기 시 | 효율 |

**핵심 알고리즘**: 블록 선택 → 유효 복사 → 블록 지우기 → 매핑 갱신

**📢 섹션 요약 비유**: TRIM이 GC를 도와줘요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 데이터베이스** → TRIM 활성화
**시나리오 2: 로그 서버** → 높은 OP
**시나리오 3: 읽기 위주** → GC 영향 적음

**도입 체크리스트**: TRIM, OP, WA 모니터링

**📢 섹션 요약 비유**: TRIM을 꼭 켜세요!
