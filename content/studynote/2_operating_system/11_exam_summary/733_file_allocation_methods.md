# 파일 할당 방법 비교: 연속, 연결, 색인 할당

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 보조 기억 장치(Disk)의 빈 블록에 파일을 효율적으로 배치하고 관리하기 위한 **디스크 공간 할당 전략**.
> 2. **가치**: 데이터 접근 속도(Sequential/Random Access)와 공간 활용 효율(단편화) 사이의 Trade-off를 해결하여 최적의 파일 시스템 성능을 확보한다.
> 3. **융합**: 고전적인 연속 할당부터 현대적 파일 시스템의 표준인 **색인 할당(Indexed Allocation)**까지, 운영체제의 발전과 함께 저장 장치의 물리적 특성에 맞춰 진화해온 기술이다.

---

### Ⅰ. 파일 할당의 3대 방식 비교

| 구분 | 연속 할당 (Contiguous) | 연결 할당 (Linked) | 색인 할당 (Indexed) |
|:---|:---|:---|:---|
| **원리** | 인접한 블록에 연속 저장 | 포인터로 다음 블록 연결 | 인덱스 블록에 주소 모음 |
| **접근 방식** | **순차/직접 접근** 가능 | **순차 접근**만 가능 | **순차/직접 접근** 가능 |
| **공간 효율** | 외부 단편화 발생 | 단편화 없음 (포인터 낭비) | 단편화 없음 (인덱스 낭비) |
| **특징** | 최고의 읽기 속도 | 파일 크기 확장이 용이함 | 현대 파일 시스템의 근간 |
| **비유** | 일렬로 주차된 기차 | 보물 찾기 지도 (다음 장소) | 책의 목차 (Index) |

---

### Ⅱ. 할당 방식별 구조도 (ASCII)

#### 1. 연속 할당 (Contiguous)
- 시작 블록 번호와 길이만 알면 됨.

```ascii
    [ Directory ] : File A (Start: 2, Len: 3)
    [ Disk ] : [ 0 ] [ 1 ] [ A ] [ A ] [ A ] [ 5 ]
                             ^2    ^3    ^4
```

#### 2. 연결 할당 (Linked)
- 각 블록이 다음 블록의 주소를 가짐.

```ascii
    [ Directory ] : File B (Start: 1)
    [ Disk ] : [ B ] -> [ 5 ] -> [ 3 ] -> [ -1 ]
                 ^1      ^5      ^3
```

#### 3. 색인 할당 (Indexed)
- 모든 블록 주소를 하나의 '인덱스 블록'에 저장.

```ascii
    [ Directory ] : File C (Index Block: 10)
    [ Index Block 10 ] : { 2, 5, 8, 12, ... }
    [ Data Blocks ] : [ 2 ], [ 5 ], [ 8 ], [ 12 ]
```

### Ⅲ. 상세 성능 분석 및 기술사적 판단 (Strategy)

#### 1. 어떤 방식이 실무에 적합한가?
- **연속 할당**: CD-ROM이나 읽기 전용 미디어, 또는 파일 크기가 고정된 시스템에서 성능이 극대화된다.
- **연결 할당**: FAT(File Allocation Table) 파일 시스템의 모태가 되었다. 직접 접근이 불가능한 치명적 단점이 있다.
- **색인 할당**: 현대의 유닉스/리눅스(i-node), 윈도우(NTFS) 시스템의 표준이다. 인덱스 블록 하나가 담을 수 있는 크기를 넘는 대용량 파일을 위해 **다단계 인덱스** 구조를 병행한다.

#### 2. 기술사적 인사이트
- **외부 단편화의 습격**: 연속 할당은 메모리 관리와 마찬가지로 외부 단편화 문제를 해결하기 위해 주기적인 **디스크 조각 모음 (Defragmentation)**이 필요하다.
- **신뢰성 문제**: 연결 할당은 중간 포인터 하나만 깨져도 뒤쪽 데이터를 모두 잃어버리는 취약점이 있다.

---

### Ⅳ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **데이터 검색 속도 향상**: 색인 할당을 통한 빠른 랜덤 액세스 보장.
- **디스크 공간 가용성**: 조각난 빈 블록을 남김없이 사용하여 저장 효율 극대화.

#### 2. 미래 전망
최근에는 **Extent (범위)** 기반 할당이 대세다. 이는 연속 할당과 색인 할당의 장점을 섞은 것으로, 가능한 한 연속적으로 할당하되(Extent), 파일이 커지면 불연속적인 Extent를 추가하여 인덱스로 관리하는 방식이다. 이를 통해 I/O 성능과 공간 효율을 동시에 잡는 하이브리드 아키텍처로 진화하고 있다.

### 👶 어린이를 위한 3줄 비유 설명
1. **연속 할당**은 운동장에 친구들이 손을 잡고 일렬로 쭉 서 있는 거예요. 찾기 쉽고 빠르지만, 중간에 빈자리가 생기면 들어가기 힘들죠.
2. **연결 할당**은 보물찾기 쪽지를 하나씩 찾는 거예요. 쪽지에 "다음 보물은 나무 밑에 있어"라고 적힌 식이죠.
3. **색인 할당**은 책 맨 앞에 있는 '차례' 페이지와 같아요. "1장은 10쪽, 2장은 50쪽"이라고 적혀 있어서 내가 가고 싶은 곳을 바로 찾아갈 수 있답니다!
