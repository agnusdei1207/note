# SSTF 기아 현상 (가운데 편중)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 현재 헤드 위치에서 가장 가까운 요청을 처리하여 기아 현상과 중앙 편중 발생
> 2. **가치**: 평균 탐색 시간 최소, 처리량 향상, 불공평한 서비스
> 3. **융합**: FCFS, SCAN, C-SCAN과 비교되는 디스크 스케줄링

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SSTF(Shortest Seek Time First)는 현재 헤드 위치에서 가장 가까운 요청을 처리하는 디스크 스케줄링 알고리즘입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SSTF (Shortest Seek Time First)                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SSTF 동작 원리                                   │ │
│   │                                                              │ │
│   │   트랙:  0 ────────────────────────────────────────── 199     │ │
│   │                           ↑                                  │ │
│   │          요청: 23, 67, 89, 132, 45, 178, 12                  │ │
│   │          현재: 50                                             │ │
│   │                                                              │ │
│   │   거리 계산:                                                  │ │
│   │   23: |50-23| = 27                                           │ │
│   │   67: |50-67| = 17  ← 최소!                                  │ │
│   │   89: |50-89| = 39                                           │ │
│   │   ...                                                        │ │
│   │                                                              │ │
│   │   처리 순서: 50 → 67 → 45 → 23 → 12 → 89 → 132 → 178        │ │
│   │               (항상 가장 가까운 것 선택)                       │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              기아 현상 & 가운데 편중                          │ │
│   │                                                              │ │
│   │   기아 현상:                                                  │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  끝 근처 요청: 178                                   │   │ │
│   │   │  새 요청 계속 들어옴: 50, 55, 60...                  │   │ │
│   │   │  → 178은 계속 밀려남 → 무한 대기 (기아)              │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   가운데 편중:                                                │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  디스크 중간 부분에 요청이 많아짐                     │   │ │
│   │   │  헤드가 중간에서만 왔다 갔다                          │   │ │
│   │   │  끝 부분은 서비스 안 됨                               │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: SSTF는 효율적이지만 기아 현상과 가운데 편중 문제가 있습니다.

**💡 비유**: SSTF는 가까운 심부름만 먼저 하는 것과 같습니다.

**📢 섹션 요약 비유**: 가까운 것만 계속 가요!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **최단 거리** | 선택 기준 | |거리| 최소 | 가까운 것 |
| **기아** | 부작용 | 무한 대기 | 굶주림 |
| **편중** | 부작용 | 중앙 집중 | 쏠림 |
| **효율** | 장점 | 탐색 최소 | 빠름 |

**알고리즘 비교**

| 알고리즘 | 탐색 시간 | 기아 | 공평 |
|:---|:---:|:---:|:---:|
| **FCFS** | 나쁨 | 없음 | 좋음 |
| **SSTF** | 좋음 | 있음 | 나쁨 |
| **SCAN** | 중간 | 없음 | 좋음 |

**핵심 알고리즘**: 요청 목록 → 거리 계산 → 최소 선택 → 이동

**📢 섹션 요약 비유**: 빠르지만 불공평해요!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 성능 vs 공평성**

| 지표 | SSTF | SCAN | C-SCAN |
|:---|:---:|:---:|:---:|
| **평균 탐색** | 최소 | 중간 | 중간 |
| **공평성** | 최악 | 좋음 | 최고 |
| **기아** | 있음 | 없음 | 없음 |

**📢 섹션 요약 비유**: 균형이 필요해요!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 일반 PC** → SSTF 사용 (공평성 덜 중요)
**시나리오 2: 서버** → SCAN/C-SCAN (공평성 중요)
**시나리오 3: 실시간** → Deadline

**도입 체크리스트**: 공평성 요구, 워크로드

**📢 섹션 요약 비유**: 상황에 맞게 선택해요!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 처리량 향상, 불공평

**미래 전망**: SSD로 스케줄링 중요성 감소

**참고 표준**: 운영체제, 디스크 스케줄링

**📢 섹션 요약 비유**: SSTF는 효율 vs 공평의 딜레마예요!
