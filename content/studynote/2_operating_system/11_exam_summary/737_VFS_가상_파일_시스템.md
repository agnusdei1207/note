# VFS (가상 파일 시스템)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다양한 파일 시스템을 통합된 인터페이스로 추상화하는 커널 계층
> 2. **가치**: 파일 시스템 독립성, 마운트 유연성, 공통 API
> 3. **융합**: ext4, NTFS, FAT, NFS와 통합된 파일 시스템 인터페이스

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

VFS(Virtual File System)는 다양한 파일 시스템을 통합된 인터페이스로 추상화하는 커널 계층입니다.


> **해설**: VFS 덕분에 응용 프로그램은 어떤 파일 시스템인지 알 필요가 없습니다.

**💡 비유**: VFS는 다양한 전자기기에 공통으로 쓰이는 USB 포트와 같습니다.

**📢 섹션 요약 비유**: 파일 시스템의 통역사예요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **superblock** | FS 메타정보 | 전체 정보 | 도서관 |
| **inode** | 파일 정보 | 메타데이터 | 책 정보 |
| **dentry** | 경로 구성 | 디렉터리 | 색인 |
| **file** | 열린 파일 | fd 관리 | 책 대출 |

**VFS 작업**

| 작업 | 설명 |
|:---|:---|
| **마운트** | FS를 디렉터리에 연결 |
| **경로 해석** | dentry 캐시 사용 |
| **파일 접근** | inode → file |

**핵심 알고리즘**: open → VFS → FS별 함수 → 결과 반환

**📢 섹션 요약 비유**: 같은 API로 다양한 FS 접근!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: VFS 지원 파일 시스템**

| 유형 | 예시 | 특징 |
|:---|:---|:---|
| **디스크** | ext4, XFS | 영구 저장 |
| **네트워크** | NFS, SMB | 원격 |
| **가상** | /proc, /sys | 커널 정보 |
| **메모리** | tmpfs | RAM |

**📢 섹션 요약 비유**: proc, sys는 가상 파일 시스템!
