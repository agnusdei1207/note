# 시스템 클럭 / 타이머 / 틱

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하드웨어 클럭 기반 시간 측정 및 인터럽트 생성 메커니즘
> 2. **가치**: 스케줄링, 타이머, 시간 동기화의 기반
> 3. **융합**: 스케줄러, 인터럽트, RTOS와 통합된 시간 관리

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

시스템 클럭(System Clock)은 하드웨어 기반 시간 소스이고, 타이머(Timer)는 특정 시간 후 인터럽트를 발생시키며, 틱(Tick)은 OS의 시간 단위입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    시스템 클럭 / 타이머 / 틱                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              클럭 소스 계층                                    │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  실시간 클럭 (RTC, Real-Time Clock)                  │   │ │
│   │   │  - 배터리 백업                                       │   │ │
│   │   │  - CMOS에 저장                                       │   │ │
│   │   │  - 시스템 꺼져도 유지                                 │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                     ↓                                       │   │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  TSC (Time Stamp Counter)                           │   │ │
│   │   │  - CPU 클럭마다 증가                                 │   │ │
│   │   │  - 높은 해상도 (나노초)                              │   │ │
│   │   │  - rdtsc 명령으로 읽기                               │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                     ↓                                       │   │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  OS 타이머 (jiffies, tick)                          │   │ │
│   │   │  - CONFIG_HZ (보통 100~1000Hz)                       │   │ │
│   │   │  - 1 tick = 1/HZ 초                                  │   │ │
│   │   │  - 스케줄링 단위                                     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              타이머 인터럽트                                  │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Timer Interrupt (매 tick마다 발생)                  │   │ │
│   │   │                                                      │   │ │
│   │   │  1. 현재 시간 업데이트 (jiffies++)                   │   │ │
│   │   │  2. 스케줄러 호출 (need_resched 확인)                │   │ │
│   │   │  3. 타임아웃 만료 확인 (timer wheel)                 │   │ │
│   │   │  4. 통계 업데이트                                    │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 클럭은 시스템의 심장 박동과 같습니다.

**💡 비유**: 클럭은 시스템의 심장 박동, 틱은 그 박동의 단위입니다.

**📢 섹션 요약 비유**: 틱은 스케줄링의 기본 단위!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **RTC** | 실시간 | 배터리 유지 | 시계 |
| **TSC** | 고해상도 | CPU 사이클 | 초시계 |
| **Tick** | OS 단위 | HZ 기반 | 메트로놈 |
| **Timer** | 알람 | 만료 인터럽트 | 알람시계 |

**타이머 종류**

| 타이머 | 해상도 | 용도 |
|:---|:---|:---|
| **PIT** | 밀리초 | 구형 |
| **HPET** | 나노초 | 고해상도 |
| **APIC Timer** | 나노초 | per-CPU |

**핵심 알고리즘**: tick 발생 → 인터럽트 → 스케줄링 체크

**📢 섹션 요약 비유**: HZ=1000이면 1ms마다 틱!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: Tick 모드**

| 모드 | 설명 | 용도 |
|:---|:---|:---|
| **Periodic** | 일정 간격 | 일반 |
| **Tickless** | 필요시만 | 절전 |
| **NO_HZ** | 동적 | 서버/모바일 |

**📢 섹션 요약 비유**: Tickless가 배터리 절약!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 실시간 OS** → 고해상도 타이머
**시나리오 2: 모바일** → Tickless
**시나리오 3: 서버** → NO_HZ_FULL

**도입 체크리스트**: HZ 설정, NO_HZ 활성화

**📢 섹션 요약 비유**: RTOS는 정밀 타이머 필요!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 정확한 스케줄링, 절전

**미래 전망**: 더 높은 해상도, 에너지 효율

**참고 표준**: POSIX clock, hrtimer

**📢 섹션 요약 비유**: 클럭은 시스템의 심장!
