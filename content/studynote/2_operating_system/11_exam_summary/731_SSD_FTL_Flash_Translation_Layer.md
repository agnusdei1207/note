# SSD FTL (Flash Translation Layer)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 논리 블록 주소(LBA)를 플래시 물리 주소(PBA)로 매핑하고 마모 평준화를 관리하는 펌웨어
> 2. **가치**: 덮어쓰기 불가 문제 해결, 마모 평준화, 가비지 컬렉션
> 3. **융합**: 플래시 메모리, TRIM, 마모 평준화, RAID와 통합된 스토리지 관리

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

FTL(Flash Translation Layer)은 논리 블록 주소(LBA)를 플래시 물리 주소(PBA)로 매핑하고 마모 평준화를 관리하는 펌웨어입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    FTL (Flash Translation Layer)                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              FTL 역할                                         │ │
│   │                                                              │ │
│   │   ┌───────────────┐      ┌───────────────┐                  │ │
│   │   │   Host (OS)   │      │  SSD (Flash)  │                  │ │
│   │   │               │      │               │                  │ │
│   │   │ LBA: 0,1,2... │ ───→ │    FTL        │ ───→ PBA        │ │
│   │   │ (논리 주소)   │      │  (매핑 테이블)│    (물리 주소)  │ │
│   │   └───────────────┘      └───────────────┘                  │ │
│   │                                                              │ │
│   │   FTL 기능:                                                   │ │
│   │   • 주소 매핑 (LBA → PBA)                                    │ │
│   │   • 마모 평준화 (Wear Leveling)                              │ │
│   │   • 가비지 컬렉션 (Garbage Collection)                       │ │
│   │   • 배드 블록 관리                                           │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              플래시 메모리 특성                                │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  1. 덮어쓰기 불가 (Erase-before-write)               │   │ │
│   │   │     → 쓰려면 먼저 지워야 함                           │   │ │
│   │   │                                                      │   │ │
│   │   │  2. 지우기 단위: 블록 (수백 KB~수 MB)                 │   │ │
│   │   │     쓰기 단위: 페이지 (4~16 KB)                       │   │ │
│   │   │                                                      │   │ │
│   │   │  3. 쓰기 횟수 제한 (Wear-out)                         │   │ │
│   │   │     SLC: ~10만회, MLC: ~3천회, TLC: ~1천회            │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: FTL은 SSD가 HDD처럼 보이게 만드는 핵심 펌웨어입니다.

**💡 비유**: FTL은 새 책장에 책을 옮기고 오래된 책장을 비우는 도서관 사서와 같습니다.

**📢 섹션 요약 비유**: 주소를 바꿔서 플래시를 관리해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **매핑 테이블** | 주소 변환 | LBA→PBA | 목차 |
| **마모 평준화** | 수명 연장 | 쓰기 분산 | 교체 |
| **GC** | 공간 확보 | 유효 이동 | 청소 |
| **배드 블록** | 결함 관리 | 제외 처리 | 수리 |

**매핑 방식 비교**

| 방식 | 단위 | 오버헤드 | 용도 |
|:---|:---:|:---:|:---|
| **페이지** | 4KB | 높음 | 일반 |
| **블록** | 수MB | 낮음 | 대용량 |
| **하이브리드** | 혼합 | 중간 | 최신 |

**핵심 알고리즘**: 쓰기 요청 → 새 페이지 할당 → 매핑 갱신 → GC 필요 시 실행

**📢 섹션 요약 비유**: 페이지 매핑이 정밀해요!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: HDD vs SSD**

| 특성 | HDD | SSD |
|:---|:---|:---|
| **접근** | 기계적 | 전자적 |
| **지연** | ~10ms | ~0.1ms |
| **내구성** | 높음 | 쓰기 제한 |
| **FTL** | 없음 | 필수 |

**📢 섹션 요약 비유**: SSD가 100배 빨라요!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 데이터베이스** → SSD + TRIM
**시나리오 2: 로그 서버** → 쓰기 내구성 고려
**시나리오 3: 캐시** → SSD 티어

**도입 체크리스트**: DWPD(내구성), TRIM, 오버프로비저닝

**📢 섹션 요약 비유**: TRIM이 GC를 도와줘요!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 빠른 I/O, 전력 절감

**미래 전망**: ZNS SSD, 컴퓨테이셔널 스토리지

**참고 표준**: NVM Express, ONFI

**📢 섹션 요약 비유**: FTL은 SSD의 뇌예요!
