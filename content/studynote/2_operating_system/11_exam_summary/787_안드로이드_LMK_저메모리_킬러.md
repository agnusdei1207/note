# 안드로이드 LMK (저메모리 킬러)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리 부족 시 중요도 기반으로 앱 종료
> 2. **가치**: 시스템 안정성, 포그라운드 보호, 사용자 경험
> 3. **융합**: oom_adj, ActivityManager, 프로세스 우선순위와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

LMK(Low Memory Killer)는 안드로이드에서 메모리 부족 시 프로세스 중요도에 따라 종료하는 커널 메커니즘입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    안드로이드 LMK                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              프로세스 중요도 (ADJ)                              │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  oom_adj 값 (낮을수록 중요):                           │   │ │
│   │   │                                                      │   │ │
│   │   │  -16 (NATIVE)      : 네이티브 서비스                  │   │ │
│   │   │  -12 (SYSTEM)      : 시스템 프로세스                  │   │ │
│   │   │    0 (FOREGROUND)  : 포그라운드 앱 (현재 사용)         │   │ │
│   │   │    1 (VISIBLE)     : 가시적 백그라운드                │   │ │
│   │   │    2 (PERCEPTIBLE) : 인지 가능 (음악 재생)            │   │ │
│   │   │    3 (BACKUP)      : 백업 에이전트                    │   │ │
│   │   │    4 (HEAVY_WEIGHT): 무거운 백그라운드                │   │ │
│   │   │    5 (SERVICE)     : 서비스                           │   │ │
│   │   │    6 (HOME)        : 런처                             │   │ │
│   │   │    7 (PREVIOUS)    : 이전 앱                          │   │ │
│   │   │    8 (SERVICE_B)   : B 서비스                         │   │ │
│   │   │    9 (CACHED)      : 캐시된 프로세스                  │   │ │
│   │   │   15 (UNKNOWN)     : 알 수 없음                       │   │ │
│   │   │                                                      │   │ │
│   │   │  LMK는 높은 ADJ 값부터 종료                            │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              LMK 임계값                                        │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  /sys/module/lowmemorykiller/parameters/             │   │ │
│   │   │                                                      │   │ │
│   │   │  adj:   0, 1, 2, 4, 6, 9, 12, 15                     │   │ │
│   │   │  minfree: 8GB, 12GB, 16GB, 20GB, 24GB, 28GB...       │   │ │
│   │   │                                                      │   │ │
│   │   │  예: 사용 가능 메모리 < 8GB → oom_adj >= 15 종료      │   │ │
│   │   │      사용 가능 메모리 < 12GB → oom_adj >= 12 종료     │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: LMK는 캐시된 앱부터 시작해 포그라운드까지 단계적 종료합니다.

**💡 비유**: LMK는 비상시 불필요한 짐부터 버리는 것과 같습니다.

**📢 섹션 요약 비유**: 캐시 → 서비스 → 포그라운드 순 보호!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **oom_adj** | 중요도 | 0-15 | 우선순위 |
| **minfree** | 임계값 | 페이지 수 | 경고선 |
| **LMK Driver** | 커널 | 종료 실행 | 처형자 |
| **AMS** | 관리 | ADJ 조정 | 관리자 |

**LMK 버전**

| 버전 | 설명 |
|:---|:---|
| **LMK** | 기본 커널 구현 |
| **LMKD** | 사용자 공간 데몬 |
| **PSI** | 최신 Pressure Stall |

**핵심 알고리즘**: 메모리 부족 → minfree 확인 → 해당 adj 프로세스 종료

**📢 섹션 요약 비유**: LMKD가 최신!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 메모리 관리**

| 방식 | OS | 기준 |
|:---|:---:|:---|
| **OOM Killer** | Linux | oom_score |
| **LMK** | Android | oom_adj |
| **Jetsam** | iOS | 부족 시 |

**📢 섹션 요약 비유**: LMK는 OOM의 모바일 버전!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 포그라운드 서비스** → 낮은 adj
**시나리오 2: 캐시된 앱** → 높은 adj
**시나리오 3: 서비스** → 중간 adj

**도입 체크리스트**: minfree 튜닝, PSI 활성화

**📢 섹션 요약 비유**: 포그라운드는 adj=0!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 포그라운드 보호, 시스템 안정성

**미래 전망**: PSI 기반, R ssize 예측

**참고 표준**: Android AOSP

**📢 섹션 요약 비유**: LMK는 안드로이드의 메모리 수호자!
