# C-SCAN (단방향 회전)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: SCAN을 개선하여 한 방향으로만 서비스하고 끝에서 즉시 시작점으로 복귀하는 스케줄링
> 2. **가치**: 더 균등한 대기 시간, 마지막 요청 공평성
> 3. **융합**: SCAN, LOOK, 디스크 스케줄링과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

C-SCAN(Circular SCAN)은 한 방향으로만 서비스하고 끝에서 즉시 시작점으로 복귀하는 디스크 스케줄링입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    C-SCAN (Circular SCAN)                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              C-SCAN 동작 원리                                 │ │
│   │                                                              │ │
│   │   트랙:  0 ────────────────────────────────────────── 199     │ │
│   │                           │                                  │ │
│   │          ┌────────────────┴────────────────┐                │ │
│   │          │    서비스 방향 (오른쪽만)         │                │ │
│   │          │    ────────────────────→        │                │ │
│   │          │    ←═══════════════════         │ (즉시 복귀)    │ │
│   │          └─────────────────────────────────┘                │ │
│   │                           ↑                                  │ │
│   │                     빠른 복귀 (서비스 없음)                   │ │
│   │                                                              │ │
│   │   요청: 23, 67, 89, 132, 45, 178, 12                        │ │
│   │   현재: 50                                                   │ │
│   │                                                              │ │
│   │   처리 순서: 50 → 67 → 89 → 132 → 178 → 0 → 12 → 23 → 45   │ │
│   │                           (끝에서 0으로 즉시 복귀)            │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              SCAN vs C-SCAN                                   │ │
│   │                                                              │ │
│   │   SCAN:                                                       │ │
│   │   0 ←──────────────────────→ 199 ←──────────────────────→ 0  │ │
│   │   서비스      서비스       서비스      서비스                 │ │
│   │                                                              │ │
│   │   C-SCAN:                                                     │ │
│   │   0 ←═════════════════════→ 199 ←═════════════════════→ 0   │ │
│   │   복귀(서비스X)   서비스    복귀(서비스X)   서비스            │ │
│   │                                                              │ │
│   │   장점: 끝 근처 요청도 빠른 서비스                            │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: C-SCAN은 SCAN보다 더 공평한 대기 시간을 제공합니다.

**💡 비유**: C-SCAN은 컨베이어 벨트처럼 한 방향으로만 흐르는 것과 같습니다.

**📢 섹션 요약 비유**: 한 방향으로만 서비스해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **단방향** | 서비스 방향 | 오른쪽만 | 컨베이어 |
| **빠른 복귀** | 위치 초기화 | 서비스 없이 | 리셋 |
| **원형** | 논리적 구조 | 0과 끝 연결 | 고리 |
| **균등성** | 대기 시간 | 모든 위치 동일 | 공평 |

**알고리즘 비교**

| 알고리즘 | 끝 근처 | 중간 | 특징 |
|:---|:---:|:---:|:---|
| **SCAN** | 느림 | 빠름 | 양방향 |
| **C-SCAN** | 균등 | 균등 | 단방향 |
| **LOOK** | 빠름 | 빠름 | 최적화 |

**핵심 알고리즘**: 오른쪽 이동 → 끝 도달 → 0으로 복귀 → 다시 오른쪽

**📢 섹션 요약 비유**: C-SCAN이 가장 공평해요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 서버** → C-SCAN으로 공평성
**시나리오 2: 실시간** → Deadline 스케줄러

**도입 체크리스트**: 공평성 요구, 워크로드 분석

**📢 섹션 요약 비유**: 공평성이 중요하면 C-SCAN!
