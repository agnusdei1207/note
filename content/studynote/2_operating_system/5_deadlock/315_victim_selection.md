# 315. 희생자 선택 (Victim Selection)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 교착상태 해소를 위해 희생할 프로세스 선택
> 2. **가치**: 교착상태 복구의 핵심 결정
> 3. **융합**: 교착상태 복구, 롤백과 연관

---

## Ⅰ. 개요

### 개념 정의

희생자 선택(Victim Selection)은 **교착상태 해소를 위해 중단할 프로세스를 선택하는 과정**이다. 선택된 희생자는 롤백되고 자원이 해제된다.

### 💡 비유: 구명보트
희생자 선택은 **구명보트**와 같다. 보트가 가라앉으면 누군가 내려야 한다. 누구를 내릴지 결정하는 것이다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                희생자 선택 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【비용 함수】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  비용 = w1 × 우선순위                                               │ │
│  │       + w2 × 진행 시간                                              │ │
│  │       + w3 × 사용 자원                                              │ │
│  │       + w4 × 롤백 비용                                              │ │
│  │       - w5 × 희생 횟수 (기아 방지)                                   │ │
│  │                                                             │ │
│  │  가장 낮은 비용의 프로세스가 희생자                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【선택 전략】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 최소 비용:                                                       │ │
│  │     • 비용 함수로 계산하여 최소 선택                                  │ │
│  │                                                             │ │
│  │  2. 최소 자원:                                                       │ │
│  │     • 가장 적은 자원을 보유한 프로세스                                 │ │
│  │                                                             │ │
│  │  3. 최소 진행:                                                       │ │
│  │     • 가장 적게 실행된 프로세스                                       │ │
│  │                                                             │ │
│  │  4. 최소 우선순위:                                                    │ │
│  │     • 가장 낮은 우선순위 프로세스                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【기아 문제】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  문제: 동일한 프로세스가 계속 희생될 수 있음                           │ │
│  │                                                             │ │
│  │  해결:                                                                │ │
│  │  • 희생 횟수를 비용 함수에 포함                                       │ │
│  │  • 한 번 희생된 프로세스는 일정 기간 보호                              │ │
│  │  • aging 기법 적용                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【부분 vs 전체 롤백】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  부분 롤백:                                                           │ │
│  │  • 교착상태 해소에 필요한 만큼만 롤백                                  │ │
│  │  • 최소한의 손실                                                      │ │
│  │  • 구현 복잡                                                          │ │
│  │                                                             │ │
│  │  전체 롤백:                                                           │ │
│  │  • 프로세스 전체를 재시작                                             │ │
│  │  • 큰 손실                                                            │ │
│  │  • 구현 단순                                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【희생자 선택 구현】                                                   │
│  ──────────────────                                                  │
│  class VictimSelector {                                             │
│      Map<Integer, Integer> sacrificeCount = new HashMap<>();        │
│                                                                     │
│      Process selectVictim(Set<Process> deadlockedProcesses) {       │
│          Process victim = null;                                     │
│          int minCost = Integer.MAX_VALUE;                           │
│                                                                     │
│          for (Process p : deadlockedProcesses) {                    │
│              int cost = calculateCost(p);                           │
│              if (cost < minCost) {                                  │
│                  minCost = cost;                                    │
│                  victim = p;                                        │
│              }                                                      │
│          }                                                          │
│                                                                     │
│          // 희생 횟수 기록 (기아 방지)                                   │ │
│          sacrificeCount.merge(victim.getId(), 1, Integer::sum);     │
│                                                                     │
│          return victim;                                             │
│      }                                                              │
│                                                                     │
│      int calculateCost(Process p) {                                 │
│          int priorityCost = p.getPriority() * 100;                  │
│          int progressCost = (int) (p.getElapsedTime() / 1000);      │
│          int resourceCost = p.getHeldResources().size() * 10;       │
│          int starvationBonus = sacrificeCount.getOrDefault(         │
│              p.getId(), 0) * 50;  // 기아 방지                       │ │
│                                                                     │
│          return priorityCost + progressCost + resourceCost          │
│               - starvationBonus;                                    │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【데이터베이스 희생자 선택】                                            │
│  ──────────────────                                                  │
│  // MySQL: innodb_deadlock_detect = ON                              │
│  // 자동으로 희생자 선택하여 롤백                                        │
│                                                                     │
│  // PostgreSQL: deadlock_timeout 후 자동 감지                        │
│  // 희생자 선택 기준:                                                  │ │
│  // 1. 트랜잭션 시작 시간                                               │
│  // 2. 락 모드                                                        │
│  // 3. 쿼리 유형                                                       │ │
│                                                                     │
│  【분산 시스템 희생자 선택】                                             │
│  ──────────────────                                                  │
│  class DistributedVictimSelector {                                  │
│      // 전역 순서 부여                                                  │ │
│      Process selectVictimGlobal(List<Process> candidates) {         │
│          // 타임스탬프 기반 선택                                        │ │
│          return candidates.stream()                                 │
│              .min(Comparator.comparing(Process::getTimestamp))      │
│              .orElse(null);                                         │
│      }                                                              │
│                                                                     │
│      // Wound-Wait: 오래된 것이 젊은 것을 선점                          │ │
│      boolean shouldPreempt(Process older, Process younger) {        │
│          return older.getTimestamp() < younger.getTimestamp();      │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【롤백 비용 추정】                                                      │
│  ──────────────────                                                  │
│  class RollbackCostEstimator {                                      │
│      int estimateRollbackCost(Process p) {                          │
│          int cost = 0;                                              │
│                                                                     │
│          // 수행한 작업 수                                             │ │
│          cost += p.getOperationsCount() * 10;                       │
│                                                                     │
│          // 할당받은 자원 수                                            │ │
│          cost += p.getHeldResources().size() * 20;                  │
│                                                                     │
│          // 변경한 데이터 양                                            │ │
│          cost += p.getModifiedDataSize() / 1024;  // KB              │ │
│                                                                     │
│          // 체크포인트 이후 진행 거리                                    │ │
│          cost += p.getDistanceFromCheckpoint() * 5;                 │
│                                                                     │
│          return cost;                                               │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【희생자 선택 로깅】                                                   │
│  ──────────────────                                                  │
│  void logVictimSelection(Process victim, Set<Process> deadlocked) { │
│      log.warn("Deadlock detected. Processes involved: {}",          │
│               deadlocked.stream()                                   │
│                   .map(p -> "P" + p.getId())                        │
│                   .collect(Collectors.joining(", ")));              │
│      log.warn("Selected victim: P{} (cost={})",                     │
│               victim.getId(), calculateCost(victim));               │
│  }                                                                  │
│                                                                     │
│  【희생자 선택 후 복구】                                                │
│  ──────────────────                                                  │
│  class DeadlockRecovery {                                           │
│      VictimSelector selector = new VictimSelector();                │
│      RollbackManager rollbackManager = new RollbackManager();       │
│                                                                     │
│      void recover(Set<Process> deadlockedProcesses) {               │
│          Process victim = selector.selectVictim(deadlockedProcesses);│
│                                                                     │
│          // 희생자 롤백                                                 │ │
│          rollbackManager.rollback(victim);                          │
│                                                                     │
│          // 자원 해제                                                   │ │
│          victim.releaseAllResources();                              │
│                                                                     │
│          // 대기 중인 프로세스 깨우기                                     │ │
│          wakeupWaitingProcesses(victim.getReleasedResources());     │
│                                                                     │
│          // 희생자 재시작 (선택적)                                        │ │
│          restartProcess(victim);                                    │
│      }                                                              │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
