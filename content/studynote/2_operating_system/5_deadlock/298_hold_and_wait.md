# 298. 보유 및 대기 (Hold and Wait)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자원을 보유하면서 다른 자원 대기
> 2. **가치**: 교착상태 조건 2
> 3. **융합**: 두 단계 락킹, 일괄 요청과 연관

---

## Ⅰ. 개요

### 개념 정의

보유 및 대기(Hold and Wait)는 **프로세스가 최소한 하나의 자원을 보유하면서 다른 자원의 할당을 대기하는 상태**다. 교착상태 발생의 4가지 필요조건 중 두 번째다.

### 💡 비유: 쇼핑카트
보유 및 대기는 **쇼핑카트**와 같다. 카트에 물건을 담아두고(보유), 계산대에서 다른 물건을 기다린다(대기).

### 보유 및 대기 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                보유 및 대기 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【상황 시각화】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  프로세스 P1:                                                        │ │
│  │  ┌───────────────────────────────────────────────────────────┐│ │
│  │  │  • 자원 A 보유 중 (locked)                                    ││ │
│  │  │  • 자원 B 요청 (waiting)                                     ││ │
│  │  │  • 자원 B가 P2에게 할당됨                                     ││ │
│  │  │  → P1은 A를 보유하며 B를 대기                                   ││ │
│  │  └───────────────────────────────────────────────────────────┘│ │
│  │                                                             │ │
│  │  프로세스 P2:                                                        │ │
│  │  ┌───────────────────────────────────────────────────────────┐│ │
│  │  │  • 자원 B 보유 중 (locked)                                    ││ │
│  │  │  • 자원 A 요청 (waiting)                                     ││ │
│  │  │  • 자원 A가 P1에게 할당됨                                     ││ │
│  │  │  → P2는 B를 보유하며 A를 대기                                   ││ │
│  │  └───────────────────────────────────────────────────────────┘│ │
│  │                                                             │ │
│  │  결과: P1 ↔ P2 서로 보유한 것을 대기 → 교착상태 가능                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【코드 예시】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 프로세스 1                                                       │ │
│  │  lock(A);            // A 보유                                       │ │
│  │  // ... 작업 ...                                                     │ │
│  │  lock(B);            // B 요청, 대기!                                  │ │
│  │                                                             │ │
│  │  // 프로세스 2 (동시 실행)                                             │ │
│  │  lock(B);            // B 보유                                        │ │
│  │  // ... 작업 ...                                                     │ │
│  │  lock(A);            // A 요청, 대기! → 보유 및 대기 발생                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                보유 및 대기 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【무력화 방법】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 일괄 요청 (Request All at Once):                                 │ │
│  │     • 모든 필요 자원을 한 번에 요청                                     │ │
│  │     • 하나라도 불가능하면 아무것도 획득 안 함                              │ │
│  │     • 단점: 자원 낭비, 기아 가능                                       │ │
│  │                                                             │ │
│  │  2. 두 단계 락킹 (Two-Phase Locking):                                │ │
│  │     • 1단계: 락 획득만 (해제 없음)                                      │ │
│  │     • 2단계: 락 해제만 (획득 없음)                                      │ │
│  │     • DB 트랜잭션에서 사용                                             │ │
│  │                                                             │ │
│  │  3. 자원 보유 없이 대기:                                               │ │
│  │     • 자원 요청 전 보유 자원 모두 해제                                   │ │
│  │     • 요청 실패 시 처음부터                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【일괄 요청 구현】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 방법 1: 전역 락으로 일괄 요청                                        │ │
│  │  lock(&global_resource_lock);  // 전역 락 획득                        │ │
│  │  lock(&resourceA);                                                 │ │
│  │  lock(&resourceB);                                                 │ │
│  │  lock(&resourceC);                                                 │ │
│  │  unlock(&global_resource_lock);  // 전역 락 해제                      │ │
│  │                                                             │ │
│  │  // 이제 세 자원 모두 보유                                              │ │
│  │  // 추가 요청 없음 → 보유 및 대기 조건 위반                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점 (무력화 시):                                                    │ │
│  │  • 교착상태 예방                                                      │ │
│  │  • 구현 비교적 단순                                                    │ │
│  │                                                             │ │
│  │  단점 (무력화 시):                                                    │ │
│  │  • 자원 낭비 (필요 전에 획득)                                          │ │
│  │  • 기아 가능성 (계속 실패)                                            │ │
│  │  • 미리 알아야 함 (어떤 자원이 필요한지)                                 │ │
│  │  • 동시성 감소                                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【일괄 요청 구현】                                                      │
│  ──────────────────                                                  │
│  // Java: tryLock으로 일괄 획득 시도                                     │
│  public boolean acquireAll(Lock[] locks) {                          │
│      // 먼저 모든 락 획득 시도                                             │
│      for (Lock lock : locks) {                                       │
│          if (!lock.tryLock()) {                                      │
│              // 하나라도 실패하면 모두 해제                                   │
│              releaseAll(locks);                                      │
│              return false;                                           │
│          }                                                          │
│      }                                                              │
│      return true;  // 모두 획득 성공                                     │
│  }                                                                  │
│                                                                     │
│  public void releaseAll(Lock[] locks) {                             │
│      for (Lock lock : locks) {                                       │
│          try {                                                       │
│              lock.unlock();                                          │
│          } catch (Exception e) {                                     │
│              // 이미 해제됨                                              │
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  // 사용                                                               │
│  Lock[] locks = { lockA, lockB, lockC };                            │
│  while (!acquireAll(locks)) {                                        │
│      Thread.sleep(backoff);  // 백오프 후 재시도                         │
│  }                                                                  │
│  try {                                                              │
│      // 작업 수행                                                       │
│  } finally {                                                         │
│      releaseAll(locks);                                              │
│  }                                                                  │
│                                                                     │
│  【두 단계 락킹 (데이터베이스)】                                           │
│  ──────────────────                                                  │
│  // 트랜잭션 예시                                                        │
│  BEGIN TRANSACTION;                                                  │
│                                                                     │
│  // 1단계 (Growing Phase): 락 획득만                                     │
│  SELECT * FROM accounts WHERE id = 1 FOR UPDATE;  // 락 획득          │
│  SELECT * FROM accounts WHERE id = 2 FOR UPDATE;  // 락 획득          │
│                                                                     │
│  // 2단계 (Shrinking Phase): 락 해제만                                   │
│  UPDATE accounts SET balance = balance - 100 WHERE id = 1;          │
│  UPDATE accounts SET balance = balance + 100 WHERE id = 2;          │
│                                                                     │
│  COMMIT;  // 모든 락 해제                                               │
│                                                                     │
│  【C++ RAII로 일괄 락 관리】                                             │
│  ──────────────────                                                  │
│  #include <mutex>                                                    │
│  #include <memory>                                                   │
│                                                                     │
│  class ScopedLocks {                                                 │
│      std::vector<std::mutex*> mutexes;                              │
│      bool acquired = false;                                         │
│                                                                     │
│  public:                                                            │
│      bool tryAcquireAll(std::vector<std::mutex*>& mtxs) {           │
│          for (auto* mtx : mtxs) {                                    │
│              if (!mtx->try_lock()) {                                 │
│                  releaseAll();                                       │
│                  return false;                                       │
│              }                                                      │
│              mutexes.push_back(mtx);                                │
│          }                                                          │
│          acquired = true;                                            │
│          return true;                                                │
│      }                                                              │
│                                                                     │
│      void releaseAll() {                                            │
│          for (auto* mtx : mutexes) {                                 │
│              mtx->unlock();                                          │
│          }                                                          │
│          mutexes.clear();                                            │
│          acquired = false;                                           │
│      }                                                              │
│                                                                     │
│      ~ScopedLocks() {                                               │
│          if (acquired) releaseAll();                                │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【실제 사용 사례】                                                      │
│  ──────────────────                                                  │
│  • 데이터베이스: 트랜잭션의 Two-Phase Locking                              │
│  • 파일 시스템: 다중 파일 잠금                                              │
│  • 네트워크: 다중 연결 관리                                                │
│  • 메모리: 다중 버퍼 접근                                                 │
│                                                                     │
│  【성능 영향】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  전략             동시성    자원 활용    기아 가능성                  │ │
│  │  ────────         ──────   ────────     ────────                   │ │
│  │  보유 및 대기 허용   높음     높음         없음                       │ │
│  │  일괄 요청          낮음     낮음         있음                       │ │
│  │  두 단계 락킹        중간     중간         없음                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
