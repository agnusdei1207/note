# 방화벽 (Firewall)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 방화벽은 **네트워크 트래픽을 필터링**하여 **허용된 연결만 통과**시키는 보안 시스템으로, **Stateful Packet Inspection**가 핵심이다.
> 2. **가치**: "모든 포트를 열면 안 되니, 필요한 것만 열자"는 **최소 권한 원칙**을 네트워크에 적용하며, DDoS, 포트 스캔 방어에 효과적이다.
> 3. **융합**: **iptables**(Legacy), **nftables**(Modern), **UFW**(Ubuntu), **firewalld**(RHEL) 등 다양한 도구가 존재하며, **하드웨어 방화벽**과 결합한다.

---

## Ⅰ. 방화벽의 개요

### 1. 정의
- 방화벽은 네트워크 경계에 설치하여 **트래픽을 제어**하는 보안 장치/소프트웨어이다.

### 2. 등장 배경
- 1980년대에 처음 등장
- "불이 건물로 번지지 막는 벽"에서 유래

### 3. 💡 비유: '건물 경비실'
- 방화벽은 **"건물 출입구의 경비원"**과 같다.
- 출입증(패킷)을 확인하고 허가된 사람만 통과시킨다.

## Ⅲ. iptables/nftables

### 1. iptables
```bash
# 규칙 확인
iptables -L -n

# SSH 허용
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# 기본 거부
iptables -P INPUT DROP
```

### 2. nftables
```bash
nft add rule ip filter input tcp dport 22 accept
```
