# TPM (Trusted Platform Module)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: TPM은 **암호화 코프로세서가 내장된 하드웨어 칩**으로, **보안 키 생성/저장, 디지털 서명, 플랫폼 무결성 측정**을 수행한다.
> 2. **가치**: 소프트웨어 공격에도 **키 추출 불가**하며, **BitLocker, LUKS**와 결합하여 디스크 암호화의 키를 안전하게 보호한다.
> 3. **융합**: TPM 1.2/2.0 표준, TPM 2.0은 **ECC, SHA-256**을 지원하며, Intel PTT, AMD fTPM으로 CPU에 내장되어 있다.

---

## Ⅰ. TPM의 개요

### 1. 정의
- TPM은 ISO/IEC 11889 표준에 따르는 **보안 하드웨어 칩**으로, 신뢰할 수 있는 플랫폼의 루트 오브 트러스트이다.

### 2. 등장 배경
- 소프트웨어만으로는 키 보안 불가
- **"하드웨어로 키를 보호해야 한다"**

### 3. 💡 비유: '금고'
- TPM은 **"벽에 박힌 금고"**와 같다.
- 키를 넣으면 소유자만 꺼낼 수 있다.

---

## Ⅱ. TPM 기능 (Deep Dive)

### 1. 주요 기능
- **키 생성/저장**: 키를 칩 내에서 생성, 절대 추출 불가
- **디지털 서명**: 문서 서명
- **난수 생성**: 진성 난수
- **PCR**: 부팅 무결성 측정
- **Attestation Identity Key (AIK)**: 익명 증명

---

## Ⅲ. TPM 버전

| 버전 | 특징 |
|:---|:---|
| **TPM 1.2** | RSA 2048, SHA-1 (취약) |
| **TPM 2.0** | RSA/ECC, SHA-256, 다중 알고리즘 |

---

## Ⅳ. TPM 활용

### 1. 디스크 암호화
```bash
# LUKS + TPM
cryptsetup luksFormat /dev/sda1 --tpm2-object=...
```

### 2. SSH 키
```bash
# TPM으로 SSH 키 생성
ssh-keygen -t ecdsa-sk
```

---

## Ⅴ. TPM 확인

### 1. Linux
```bash
# TPM 확인
dmesg | grep -i tpm

# tpm2-tools
tpm2_startup
```
