### 💡 핵심 인사이트 (Insight)
1. **물리적 자원의 정량적 제한**: 리눅스 컨트롤 그룹(Cgroups)은 프로세스 그룹별로 CPU, 메모리, I/O 등 시스템 자원 사용량을 제한, 우선순위 지정, 격리, 모니터링하는 커널 기술입니다.
2. **"Noisy Neighbor" 문제 해결**: 특정 컨테이너가 시스템의 모든 자원을 독점하지 못하게 막아, 동일 호스트 내 다른 프로세스들의 성능을 보장하는 핵심 메커니즘입니다.
3. **계층 구조의 관리**: 자원을 계층적으로 관리하여 부모 그룹의 자원 한도 내에서 자식 그룹들이 자원을 나누어 쓰도록 설계할 수 있습니다.

---

## Ⅰ. 리눅스 컨트롤 그룹(Cgroups)의 정의 및 주요 기능
### 1. 정의
프로세스들을 그룹으로 묶어 시스템 자원(CPU, Memory, Disk I/O, Network 등)의 가용량을 정교하게 제어하는 리눅스 커널의 기능입니다.

### 2. 4대 주요 기능
- **Resource Limiting (자원 제한)**: 그룹이 사용할 수 있는 최대 메모리나 CPU 점유율을 설정합니다.
- **Prioritization (우선순위)**: 특정 그룹에 더 많은 CPU 시간이나 I/O 대역폭을 할당합니다.
- **Accounting (측정)**: 그룹별 자원 사용량을 모니터링하고 과금이나 분석에 활용합니다.
- **Control (제어)**: 그룹 내 모든 프로세스를 한꺼번에 중지(Freeze)하거나 재개(Resume)합니다.

📢 **섹션 요약 비유**: Cgroups는 '각 가구마다 한 달 전기 사용량을 제한하고, 요금을 따로 매기는 계량기 및 차단기'와 같습니다.

---

## Ⅱ. Cgroups의 아키텍처 및 작동 방식
### 1. 계층 구조와 컨트롤러 (ASCII Diagram)

```text
    [ Root Cgroup (/) ]
           |
    +------+------+
    |             |
 [ cpu ]      [ memory ]  <-- 서브시스템 (Subsystems/Controllers)
    |             |
 +--+--+       +--+--+
 |     |       |     |
[G1]  [G2]    [G3]  [G4]  <-- 계층적 프로세스 그룹
 |     |       |     |
(P1)  (P2)    (P3)  (P4)  <-- 실제 프로세스 (Tasks)
```

### 2. 주요 서브시스템(Subsystems)
- **cpu**: CPU 사용 시간(Weight) 및 사용량(Quota) 제한.
- **cpuset**: 특정 프로세스 그룹이 실행될 CPU 코어 지정.
- **memory**: 메모리 사용 한도 설정 및 OOM(Out of Memory) 처리.
- **blkio**: 블록 장치(Disk)에 대한 읽기/쓰기 속도 및 IOPS 제한.
- **pids**: 그룹 내에서 생성할 수 있는 프로세스 개수 제한 (Fork Bomb 방지).

📢 **섹션 요약 비유**: 아키텍처는 '회사의 부서별 예산 관리 시스템'과 같습니다. 총무팀, 개발팀 등 부서별로 예산(자원)을 나눠주고, 그 안에서 팀원들이 나눠 쓰는 구조입니다.

---

## Ⅲ. Cgroups v1 vs v2
### 1. Cgroups v1
- 각 서브시스템(CPU, Memory 등)이 독립적인 계층 구조를 가짐.
- 관리의 유연성은 높으나 구성이 복잡하고 자원 간 상호작용 분석이 어려움.

### 2. Cgroups v2 (Unified Hierarchy)
- 모든 서브시스템이 단일한 계층 구조를 공유함.
- 설정이 간소화되고 I/O와 메모리 제어 간의 협업(예: Writeback 제어)이 강화됨.
- 최신 배포판(Fedora, Ubuntu 22.04+) 및 쿠버네티스의 표준으로 자리 잡음.

📢 **섹션 요약 비유**: v1이 '각 공과금(전기, 수도, 가스)을 따로 관리하는 고지서'라면, v2는 '모든 공과금을 통합 관리하는 스마트 홈 시스템'입니다.

## Ⅳ. 컨트롤 그룹의 실제 설정 (Virtual File System)
### 1. cgroupfs interface
리눅스는 `/sys/fs/cgroup/` 경로에 가상 파일 시스템을 통해 Cgroups를 노출합니다. 사용자는 파일을 쓰고 읽는 것만으로 설정을 변경할 수 있습니다.

### 2. 예시 (Memory 제한)
```bash
# 새로운 그룹 생성
mkdir /sys/fs/cgroup/memory/my_app
# 500MB로 메모리 제한 설정
echo 500M > /sys/fs/cgroup/memory/my_app/memory.limit_in_bytes
# 특정 프로세스(PID)를 그룹에 할당
echo 1234 > /sys/fs/cgroup/memory/my_app/tasks
```

📢 **섹션 요약 비유**: 설정 방식은 '메모장에 적는 것만으로 마법이 일어나는 주문서'와 같습니다. 파일에 숫자를 적으면 즉시 물리적 제약이 걸립니다.

---

## Ⅴ. 현대 인프라에서의 핵심 역할
### 1. Docker/K8s의 Resource Quota
사용자가 YAML 파일에 정의한 `limits`와 `requests`는 내부적으로 Cgroups 설정으로 변환되어 실행 중인 컨테이너를 제어합니다.

### 2. 시스템 안정성 확보
특정 프로세스의 버그로 인한 메모리 누수가 발생해도, Cgroups 제한에 걸려 해당 프로세스만 종료될 뿐 호스트 전체의 가용성은 유지됩니다.

📢 **섹션 요약 비유**: Cgroups는 클라우드라는 거대한 배에서 '물이 새도 특정 칸만 잠가 배 전체가 가라앉지 않게 하는 격벽'과 같은 역할을 합니다.
