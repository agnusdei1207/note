# 사이드 채널 공격 (Side-Channel Attacks)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사이드 채널 공격은 **시스템의 직접적인 암호문이 아닌 부수적 정보(시간, 전력, 소음, 캐시)**를 분석하여 비밀 정보를 유출하는 공격이다.
> 2. **가치**: "강력한 암호라도 구현이 취약하면 무용"하며, **Spectre, Meltdown**가 보여주듯 CPU 아키텍처 자체의 근본적 취약점이다.
> 3. **융합**:** 캐시 타이밍, 전력 분석, 전자기파 분석** 등 다양한 벡터가 존재하며, **상수 시간 알고리즘, 캐시 플러시**로 완화한다.

---

## Ⅰ. 사이드 채널의 개요

### 1. 정의
- 사이드 채널은 시스템의 **의도하지 않은 정보 누출 경로**(시간, 전력, 소음 등)을 이용한 공격이다.

### 2. 등장 배경
- 1996년 Paul Kocher: **타이밍 공격**으로 암호 키 복구
- 2018년 Spectre/Meltdown: CPU 추측 실행 활용

### 3. 💡 비유: '금고 두드리기'
- 사이드 채널은 **"금고의 다이얼을 돌리는 소리"**를 듣고 자릿수 추측하는 것과 같다.
- 금고(암호)가 강해도 소리(부수 정보)로 정보 유출.

---

## Ⅱ. 주요 사이드 채널 (Deep Dive)

### 1. 타이밍 공격
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                    타이밍 공격 원리                            │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │   [일반적인 경우]                                               │
    │    올바른 비밀번호: 1234 ──▶ 즉시 접근 ⏱️ 0.1ms               │
    │    틀린 비밀번호:    0000 ──▶ 거부   ⏱️ 0.1ms               │
    │                                                                 │
    │   [취약한 구현]                                                │
    │    첫 자: 1인 비번 ──▶ 접근 성공 시도만 ⏱️ 0.1ms               │
    │    첫 자: 9인 비번 ──▶ 즉시 거부        ⏱️ 0.1ms               │
    │                                                                 │
    │   * 응답 시간 차이로 올바른 자릿수 추측                        │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. 캐시 공격
- **Flush+Reload**: 캐시 라인 플러시 후 접근 시간 측정
- **Prime+Probe**: 캐시 프라임 후 타겟이 사용했는지 확인
- **Spectre**: 추측 실행으로 캐시에 데이터 남기기

### 3. 전력 분석
- 전력 소비 패턴으로 연산 추론

## Ⅲ. Spectre & Meltdown

### 1. Meltdown
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                       Meltdown (CVE-2017-5754)                    │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [원인]                                                        │
    │   - 예외 처리 중 추측 실행이 계속됨                            │
    │   - 커널 메모리를 사용자 공간에서 읽을 수 있음                  │
    │                                                                 │
    │  [영향]                                                        │
    │   - Intel, ARM 프로세서                                        │
    │   - KAISER (Kernel Page Table Isolation)로 완화                │
    │                                                                 │
    │  [완화] KPTI                                                   │
    │   - 커널과 사용자 페이지 테이블 분리                            │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. Spectre
- **Branch Prediction Poisoning**: 분기 예측 오도
- **Bounds Check Bypass**: 배열 경계 검증 우회
- **Variant 1, 2**: 더 넓은 범위

## Ⅳ. 완화 기법

### 1. 소프트웨어
- **상수 시간 알고리즘**: 입력 크기无关 동일 시간
- **캐시 플러시**: 사용 후 캐시 정리
- **mfence/lfence**: 명령어 순서 강제

### 2. 하드웨어
- **Microcode 업데이트**: CPU 펌웨어 패치
- **RDRAND/CPUID**: 하드웨어 난수

## 👶 어린이를 위한 3줄 비유 설명
1. 사이드 채널은 **"문 밖에서 소리 듣고 방 안을 추측"** 같아요.
2. 문이 열리는 소리, 사람 걸음 소리로 누가 방에 있는지 알 수 있죠.
3. 아무리 비밀번호가 강해도 소리로 정보를 빼내는 해커가 있어서 조심해야 해요!
