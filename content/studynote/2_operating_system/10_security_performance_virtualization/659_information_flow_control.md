# 정보 흐름 제어 (Information Flow Control)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 정보 흐름 제어는 **데이터의 기밀성 라벨(비밀 등급)을 추적**하여 **비밀 정보가 공개 경로로 유출되는 것을 방지**하는 기술이다.
> 2. **가치**: "암호화는 있지만 정보 흐름은 통제 안 함" 문제를 해결하며, **악의적 프로그램, 우회 가능한 DAC**의 한계를 극복한다.
> 3. **융합**:** Non-interference**, **Declassification**, **Taint Analysis**와 연계되며, SELinux의 MLS(Multi-Level Security)가 구현 예시이다.

---

## Ⅰ. 정보 흐름 제어의 개요

### 1. 정의
- 정보 흐름 제어는 데이터의 **기밀성 라벨**을 보존하며 정보 흐름을 통제한다.

### 2. 등장 배경
- 1970년대 Bell-LaPadula 이론: 정보 흐름 보안

### 3. 💡 비유: '기밀 문서 봉투'
- 정보 흐름 제어는 **"기밀 문서는 봉투에 넣어 공개 장소에 놓지 않음"**과 같다.
- 비밀 문서가 공부방(비밀)에서 복도(공개)로 나가지 못하게 막는다.

---

## Ⅱ. 기본 개념 (Deep Dive)

### 1. 라벨링
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                   정보 라벨 예시                                │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [데이터 객체]                                                  │
    │   - password: ★★★ (Top Secret)                                │
    │   - config:   ★★☆ (Secret)                                   │
    │   - public:   ☆☆☆ (Unclassified)                             │
    │                                                                 │
    │  [프로그램 변수]                                                │
    │   int x; // ★★★                                               │
    │   int y; // ☆☆☆                                               │
    │                                                                 │
    │  [흐름 규칙]                                                   │
    │   x = y;  // ✅ 낮음→높음 (가능)                               │
    │   y = x;  // ❌ 높음→낮음 (금지, declassify 필요)              │
    └─────────────────────────────────────────────────────────────────┘
```

### 2. Non-interference
- 보안 레벨이 낮은 입력이 높은 레벨 출력에 영향을 주지 않아야 함
