# 제로 트러스트 (Zero Trust)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제로 트러스트는 **"신뢰하지 않고 항상 검증(Verify, Then Trust)"**하는 보안 모델로, 내부/외부 구분 없이 **모든 접근을 검증**한다.
> 2. **가치**: "내부망은 안전하다"는 잘못된 가정을 깨고 **내부자 공격, 랜섬웨어 확산**을 방지하며, **최소 권한** 원칙을 강제한다.
> 3. **융합**:** ID(MFA)**, **Device(상태)**, **Network(위치)**, **Data(분류)**를 종합적으로 검증하는 **Policy Decision Point**와 **Policy Enforcement Point**로 구현된다.

---

## Ⅰ. 제로 트러스트의 개요

### 1. 정의
- 제로 트러스트는 **네트워크 경계(Perimeter)가 사라진 환경**에서 모든 접근을 검증하는 보안 프레임워크이다.

### 2. 등장 배경
- 2010년 Forrester, John Kindervag가 개념화

### 3. 💡 비유: '모든 손님 검문'
- 제로 트러스트는 **"호텔 출입구에서 모든 손님을 검문"**하는 것과 같다.
- VIP든 직원이든 신분증을 확인하고 목적지를 등록해야 한다.

## Ⅱ. 제로 트러스트 원칙 (Deep Dive)

### 1. NIST SP 800-207
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                 Zero Trust 핵심 원칙                            │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  [1] Never Trust, Always Verify                               │
    │   - 내부망 신뢰 X                                              │
    │   - 모든 요청 검증                                             │
    │                                                                 │
    │  [2] Least Privilege Access                                   │
    │   - 필요할 때만, 필요한 만큼만 접근                             │
    │   - Just-In-Time, Just-Enough                                │
    │                                                                 │
    │  [3] Assume Breach                                            │
    │   - 이미 침해되었다고 가정                                     │
    │   - Lateral Movement 방지                                     │
    │                                                                 │
    │  [4] Micro-Segmentation                                       │
  │   - 세분화된 격리                                               │
    │   - 자산별로 보안 정책                                         │
    └─────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 아키텍처

### 1. PDP & PEP
```text
    ┌─────────────────────────────────────────────────────────────────┐
    │                Zero Trust 아키텍처                              │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │   User/Device                                                  │
    │      │                                                         │
    │      │ 접근 요청                                               │
    │      ▼                                                         │
    │   ┌───────────────────────────────────────────────┐           │
    │   │         Policy Enforcement Point          │           │
    │   │              (PEP)                           │           │
    │   │  - 요청 가로채기                                   │           │
    │   │  - 검증 거버킬 첨부                                 │           │
    │   └───────────────────┬───────────────────────────┘           │
    │                       │                                       │
    │                       ▼                                       │
    │   ┌───────────────────────────────────────────────┐           │
    │   │       Policy Decision Point                  │           │
    │   │          (PDP)                               │           │
    │   │                                               │           │
    │   │  [검증 요소]                                   │           │
    │   │   - Identity: MFA 인증 여부                    │           │
    │   │   - Device: 상태(패치, jailbreak)              │           │
    │   │   - Network: 위치, 신뢰 여부                  │           │
    │   │   - Data: 분류, 사용자 역할                   │           │
    │   │   - Behavior: 이상 행동?                      │           │
    │   │                                               │           │
    │   │  결정: Allow/Deny/Limit                       │           │
    │   └───────────────────┬───────────────────────────┘           │
    │                       │                                       │
    │                       ▼                                       │
    │   Resource (App/Data)                                          │
    │                                                                 │
    │  * 모든 요청은 PDP를 거쳐야 함                               │
    └─────────────────────────────────────────────────────────────────┘
```
