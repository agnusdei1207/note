# 501. I/O 시스템 개요 (I/O System Overview)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU와 외부 장치 간 데이터 전송
> 2. **가치**: 하드웨어 추상화와 효율적 통신
> 3. **융합**: 인터럽트, DMA, 버퍼링과 연관

---

## Ⅰ. 개요

### 개념 정의

I/O 시스템(Input/Output System)은 **CPU와 외부 장치(입출력 장치) 간의 데이터 전송을 관리하는 하드웨어 및 소프트웨어 체계**이다.

### 💡 비유: 배달 서비스
I/O 시스템은 **배달 서비스**와 같다. 주문자(CPU)와 수신자(장치) 사이에서 물건(데이터)을 전달한다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                I/O 시스템 상세                                         │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【I/O 장치 분류】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  분류 기준        장치                       특징                        │ │
│  │  ────────        ────                       ────                        │ │
│  │  블록 장치       디스크, SSD               블록 단위 접근               │ │
│  │  캐릭터 장치     키보드, 마우스, 직렬 포트    바이트 단위 순차 접근         │ │
│  │  네트워크 장치   NIC, 무선 카드              패킷 단위 전송               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【I/O 접근 방식】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 프로그램 방식 I/O (Programmed I/O):                                 │ │
│  │     • CPU가 직접 I/O 명령 실행                                          │ │
│  │     • 폴링으로 상태 확인                                                │ │
│  │     • CPU 낭비 심함                                                     │ │
│  │                                                             │ │
│  │  2. 인터럽트 기반 I/O:                                                  │ │
│  │     • 장치가 준비되면 인터럽트                                           │ │
│  │     • CPU는 다른 작업 수행 가능                                          │ │
│  │     • 일반적인 방식                                                      │ │
│  │                                                             │ │
│  │  3. DMA (Direct Memory Access):                                        │ │
│  │     • DMA 컨트롤러가 직접 메모리 전송                                     │ │
│  │     • CPU 개입 최소화                                                   │ │
│  │     • 대용량 전송에 적합                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주요 성능 지표】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  지표          설명                    단위                          │ │
│  │  ────          ────                    ────                          │ │
│  │  대역폭        단위 시간 전송량          MB/s, GB/s                    │ │
│  │  지연 시간     요청부터 완료까지          ms, μs                        │ │
│  │  IOPS         초당 I/O 작업 수          ops/sec                       │ │
│  │  CPU 이용률    I/O에 사용된 CPU 시간      %                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux I/O 서브시스템 확인】                                          │
│  ──────────────────                                                  │
│  // 장치 목록                                                          │
│  $ ls -la /dev/                                                      │
│  $ lsblk                                                             │
│  $ cat /proc/devices                                                 │
│                                                                     │
│  // I/O 통계                                                          │
│  $ cat /proc/diskstats                                               │
│  $ iostat -x 1                                                       │
│                                                                     │
│  // 인터럽트 확인                                                      │
│  $ cat /proc/interrupts                                              │
│  $ watch -n 1 cat /proc/interrupts                                   │
│                                                                     │
│  【DMA 정보 확인】                                                      │
│  ──────────────────                                                  │
│  // DMA 채널                                                          │
│  $ cat /proc/dma                                                     │
│                                                                     │
│  // I/O 포트                                                          │
│  $ cat /proc/ioports                                                 │
│                                                                     │
│  // 메모리 맵핑 I/O                                                    │
│  $ cat /proc/iomem                                                   │
│                                                                     │
│  【장치 드라이버 확인】                                                  │
│  ──────────────────                                                  │
│  // 로드된 모듈                                                        │
│  $ lsmod                                                             │
│  $ modinfo <module_name>                                             │
│                                                                     │
│  // 장치 정보                                                          │
│  $ lspci -v                                                          │
│  $ lsusb -v                                                          │
│                                                                     │
│  // 커널 로그                                                          │
│  $ dmesg | grep -i "dma\|interrupt\|io"                             │
│                                                                     │
│  【I/O 스케줄러 확인】                                                  │
│  ──────────────────                                                  │
│  // 현재 스케줄러                                                       │
│  $ cat /sys/block/sda/queue/scheduler                               │
│  [mq-deadline] none                                                  │
│                                                                     │
│  // 스케줄러 변경                                                      │
│  $ echo noop > /sys/block/sda/queue/scheduler                       │
│                                                                     │
│  【I/O 성능 테스트】                                                    │
│  ──────────────────                                                  │
│  // 간단한 쓰기 테스트                                                  │
│  $ dd if=/dev/zero of=/tmp/test bs=1M count=1024 conv=fdatasync    │
│                                                                     │
│  // 읽기 테스트                                                        │
│  $ dd if=/tmp/test of=/dev/null bs=1M                              │
│                                                                     │
│  // fio로 상세 테스트                                                   │
│  $ fio --name=randread --ioengine=libaio --iodepth=16 \             │
│        --rw=randread --bs=4k --direct=1 --size=1G /dev/sdb1        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: CPU와 외부 장치 간 데이터 전송 관리
• 계층: 응용 → 시스템콜 → 장치독립 → 드라이버 → 컨트롤러 → 장치

• 접근: 프로그램 I/O, 인터럽트, DMA
• 성능: 대역폭, 지연, IOPS
• 목적: 추상화, 효율성, 표준화
```
