# 248. Per-CPU 변수 (Per-CPU Variables)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 각 CPU가 독립적인 변수 복사본 사용
> 2. **가치**: 락 없는 동기화, 캐시 친화적
> 3. **융합**: 커널 동기화, NUMA, 통계 수집과 연관

---

## Ⅰ. 개요

### 개념 정의

Per-CPU 변수(Per-CPU Variables)는 **각 CPU 코어가 독립적인 변수 복사본을 가지는 데이터 구조**다. 공유 데이터에 대한 락을 피하고 캐시 효율성을 높인다.

### 💡 비유: 각 교실의 칠판
Per-CPU 변수는 **각 교실의 칠판**과 같다. 1반은 1반 칠판에, 2반은 2반 칠판에 쓴다. 서로 방해하지 않는다. 나중에 모아서 합계를 구한다.

### Per-CPU 변수 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                Per-CPU 변수 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 개념】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  일반 변수 (공유):                                               │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  int counter;  // 모든 CPU가 공유                             │ │ │
│  │  │                                                         │ │ │
│  │  │  CPU 0 ─┐                                                │ │ │
│  │  │  CPU 1 ─┼── lock() → counter++ → unlock()  // 락 필요      │ │ │
│  │  │  CPU 2 ─┤                                                │ │ │
│  │  │  CPU 3 ─┘                                                │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  Per-CPU 변수:                                                   │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  DEFINE_PER_CPU(int, counter);                             │ │ │
│  │  │                                                         │ │ │
│  │  │  CPU 0: counter[0]++  // 락 없음, 캐시 친화적               │ │ │
│  │  │  CPU 1: counter[1]++  // 락 없음, 캐시 친화적               │ │ │
│  │  │  CPU 2: counter[2]++  // 락 없음, 캐시 친화적               │ │ │
│  │  │  CPU 3: counter[3]++  // 락 없음, 캐시 친화적               │ │ │
│  │  │                                                         │ │ │
│  │  │  합계: sum = counter[0] + counter[1] + ...                │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 레이아웃】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌──────────┬──────────┬──────────┬──────────┐               │ │
│  │  │ Per-CPU  │ Per-CPU  │ Per-CPU  │ Per-CPU  │               │ │
│  │  │ 영역 0   │ 영역 1   │ 영역 2   │ 영역 3   │               │ │
│  │  │ (CPU 0)  │ (CPU 1)  │ (CPU 2)  │ (CPU 3)  │               │ │
│  │  ├──────────┼──────────┼──────────┼──────────┤               │ │
│  │  │ var_a[0] │ var_a[1] │ var_a[2] │ var_a[3] │               │ │
│  │  │ var_b[0] │ var_b[1] │ var_b[2] │ var_b[3] │               │ │
│  │  │ var_c[0] │ var_c[1] │ var_c[2] │ var_c[3] │               │ │
│  │  └──────────┴──────────┴──────────┴──────────┘               │ │
│  │                                                             │ │
│  │  각 CPU는 자신 영역만 접근 → 캐시 충돌 없음                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                           │ │
│  │  • 락 없이 접근 가능                                              │ │
│  │  • 캐시 false sharing 없음                                       │ │
│  │  • 높은 확장성                                                    │ │
│  │  • 컨텍스트 스위치 비용 절감                                       │ │
│  │                                                             │ │
│  │  단점:                                                           │ │
│  │  • 전체 합계 계산 필요 시 비용                                     │ │
│  │  • CPU 간 마이그레이션 시 주의                                     │ │
│  │  • 메모리 사용량 증가                                             │ │
│  │  • 선점 가능한 환경에서 주의                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                Per-CPU 변수 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 커널 활용】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // Per-CPU 변수 정의                                             │ │
│  │  DEFINE_PER_CPU(int, process_count);                             │ │
│  │  DEFINE_PER_CPU(unsigned long, context_switches);                │ │
│  │  DEFINE_PER_CPU(struct rq, runqueue);  // 스케줄러 큐             │ │
│  │                                                             │ │
│  │  // 접근 매크로                                                     │ │
│  │  this_cpu_read(var);       // 현재 CPU 값 읽기                    │ │
│  │  this_cpu_write(var, val); // 현재 CPU 값 쓰기                    │ │
│  │  this_cpu_add(var, n);     // 현재 CPU 값 더하기                   │ │
│  │  this_cpu_inc(var);        // 현재 CPU 값 증가                     │ │
│  │                                                             │ │
│  │  // 특정 CPU 값 접근                                               │ │
│  │  per_cpu(var, cpu_id);                                            │ │
│  │                                                             │ │
│  │  // 전체 합계                                                       │ │
│  │  int total = 0;                                                   │ │
│  │  for_each_possible_cpu(cpu) {                                     │ │
│  │      total += per_cpu(process_count, cpu);                        │ │
│  │  }                                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【활용 사례】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 스케줄러 런큐                                                   │ │
│  │     • 각 CPU가 자신의 런큐 관리                                      │ │
│  │     • 락 없이 로컬 태스크 스케줄링                                   │ │
│  │                                                             │ │
│  │  2. 네트워크 통계                                                   │ │
│  │     • 패킷 카운터                                                   │ │
│  │     • 바이트 카운터                                                 │ │
│  │     • 읽기 시 락 없음, 쓰기 시 락 없음                               │ │
│  │                                                             │ │
│  │  3. 메모리 할당자 (per-cpu slab)                                   │ │
│  │     • 각 CPU가 로컬 캐시 가짐                                        │ │
│  │     • 락 없이 빠른 할당                                              │ │
│  │                                                             │ │
│  │  4. 인터럽트 카운터                                                 │ │
│  │     • 각 CPU 인터럽트 통계                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【선점 주의사항】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 안전한 접근 (선점 비활성화)                                      │ │
│  │  int cpu = get_cpu();  // 선점 비활성화, CPU ID 획득               │ │
│  │  per_cpu(var, cpu)++;                                             │ │
│  │  put_cpu();  // 선점 재활성화                                       │ │
│  │                                                             │ │
│  │  // 또는 this_cpu 매크로 (자동 처리)                                 │ │
│  │  this_cpu_inc(var);  // 내부적으로 선점 보장                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【사용자 공간 Per-CPU (Thread-Local Storage)】                      │
│  ──────────────────                                                  │
│  // C++ thread_local                                                 │
│  thread_local int counter = 0;                                      │
│                                                                     │
│  void increment() {                                                 │
│      counter++;  // 락 없음, 각 스레드 자체 복사본                      │
│  }                                                                  │
│                                                                     │
│  int get_total() {                                                  │
│      // 전체 합계는 별도 메커니즘 필요                                  │
│      // (atomic sum 등)                                              │
│  }                                                                  │
│                                                                     │
│  【Java ThreadLocal】                                                │
│  ──────────────────                                                  │
│  ThreadLocal<Integer> counter = ThreadLocal.withInitial(() -> 0);  │
│                                                                     │
│  void increment() {                                                 │
│      counter.set(counter.get() + 1);                                │
│  }                                                                  │
│                                                                     │
│  【Go goroutine-local (표준 없음, 패턴으로 구현)】                      │
│  ──────────────────                                                  │
│  // 각 고루틴이 자체 카운터                                             │
│  // Map으로 관리하거나 채널로 집계                                       │
│                                                                     │
│  【통계 수집 패턴】                                                      │
│  ──────────────────                                                  │
│  // 각 스레드가 로컬 통계 누적                                           │
│  thread_local struct Stats {                                        │
│      uint64_t requests;                                              │
│      uint64_t bytes;                                                 │
│      uint64_t errors;                                                │
│  } local_stats;                                                      │
│                                                                     │
│  void record_request(size_t size) {                                 │
│      local_stats.requests++;                                         │
│      local_stats.bytes += size;                                      │
│  }                                                                  │
│                                                                     │
│  // 주기적으로 전역 집계                                                │
│  atomic<uint64_t> global_requests;                                  │
│  atomic<uint64_t> global_bytes;                                     │
│                                                                     │
│  void flush_stats() {                                               │
│      global_requests.fetch_add(local_stats.requests);               │
│      global_bytes.fetch_add(local_stats.bytes);                     │
│      local_stats = {};  // 리셋                                      │
│  }                                                                  │
│                                                                     │
│  【NUMA 고려사항】                                                      │
│  ──────────────────                                                  │
│  • NUMA 노드별 Per-CPU 할당 고려                                      │
│  • 메모리 할당 시 numa_node_id() 사용                                 │
│  • Linux: alloc_percpu() vs __alloc_percpu()                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 각 CPU 독립 변수 복사본
• 장점: 락 없음, 캐시 친화적, 확장성
• 접근: this_cpu_*, per_cpu()
• 활용: 스케줄러, 통계, 메모리 할당
• 주의: 선점, 합계 비용, 마이그레이션
• 유사: Thread-Local Storage (TLS)
```
