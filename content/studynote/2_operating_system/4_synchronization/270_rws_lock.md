# 270. RWS 락 (Read-Write-Seq Lock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 읽기-쓰기-순차 락의 조합
> 2. **가치**: 읽기 락 없이 빠른 읽기, 쓰기 보호
> 3. **융합**: RW 락, 순차 락, 버전 카운터와 연관

---

## Ⅰ. 개요

### 개념 정의

RWS 락(Read-Write-Seq Lock)은 **읽기-쓰기 락과 순차 락의 특징을 결합한 동기화 메커니즘**이다. 읽기는 버전 확인만으로 락 없이 수행하고, 쓰기는 배타적 보호를 받는다.

### 💡 비유: 신문 구독
RWS 락은 **신문 구독**과 같다. 읽을 때는 그냥 펴본다. 도중에 바뀌면 다시 읽는다. 쓸 때만(기사 작성)은 아무도 못 보게 한다.

### RWS 락 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                RWS 락 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct rws_lock {                                                 │ │
│  │      unsigned seq;          // 순차 번호 (짝수: 안정, 홀수: 쓰기 중)    │ │
│  │      rwlock_t rwlock;       // 쓰기 보호용 RW 락                    │ │
│  │  };                                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동작 모드】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 빠른 읽기 (seqlock 모드):                                        │ │
│  │     do {                                                            │ │
│  │         seq1 = lock.seq;                                            │ │
│  │         if (seq1 & 1) continue;  // 쓰기 중                          │ │
│  │         data = read();                                              │ │
│  │         seq2 = lock.seq;                                            │ │
│  │     } while (seq1 != seq2);                                         │ │
│  │     // 락 없음!                                                      │ │
│  │                                                             │ │
│  │  2. 보호 읽기 (rwlock 모드):                                         │ │
│  │     read_lock(&lock.rwlock);                                        │ │
│  │     data = read();                                                  │ │
│  │     read_unlock(&lock.rwlock);                                      │ │
│  │     // 읽기 락 있음                                                   │ │
│  │                                                             │ │
│  │  3. 쓰기 (배타적):                                                   │ │
│  │     write_lock(&lock.rwlock);                                       │ │
│  │     lock.seq++;  // 홀수                                             │ │
│  │     write();                                                        │ │
│  │     lock.seq++;  // 짝수                                             │ │
│  │     write_unlock(&lock.rwlock);                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【세 모드 비교】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  모드          락 유무      동시성      일관성                      │ │
│  │  ────          ──────        ──────      ──────                    │ │
│  │  빠른 읽기      없음         최대        버전 확인                   │ │
│  │  보호 읽기      공유 락      중간        보장                        │ │
│  │  쓰기          배타 락      단독        보장                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                RWS 락 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【언제 어떤 모드 사용?】                                                │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  빠른 읽기:                                                          │ │
│  │  • 데이터 불일치 허용 가능                                            │ │
│  │  • 재시도 비용 낮음                                                   │ │
│  │  • 읽기 빈도 매우 높음                                                │ │
│  │  • 예: 통계 카운터, 상태 플래그                                       │ │
│  │                                                             │ │
│  │  보호 읽기:                                                          │ │
│  │  • 반드시 일관된 데이터 필요                                          │ │
│  │  • 포인터 역참조 (널 체크)                                            │ │
│  │  • 복잡한 읽기 작업                                                  │ │
│  │  • 예: 리스트 순회, 포인터 접근                                       │ │
│  │                                                             │ │
│  │  쓰기:                                                               │ │
│  │  • 데이터 수정                                                       │ │
│  │  • 배타적 접근 필요                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【vs 순수 Seqlock】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  순수 Seqlock:                                                      │ │
│  │  • 읽기: 버전 확인만                                                  │ │
│  │  • 쓰기: 스핀락                                                       │ │
│  │  • 쓰기 중 읽기: 버전이 홀수면 재시도                                  │ │
│  │  • 문제: 쓰기 중 읽기 반복 재시도 → 기아                               │ │
│  │                                                             │ │
│  │  RWS 락:                                                            │ │
│  │  • 읽기: 선택 가능 (빠른/보호)                                        │ │
│  │  • 쓰기: RW 락                                                       │ │
│  │  • 보호 읽기 사용 시 쓰기 중 읽기 블록                                 │ │
│  │  • 장점: 기아 방지, 유연성                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                              │ │
│  │  • 빠른 읽기: 락 없음                                                │ │
│  │  • 유연성: 두 가지 읽기 모드                                          │ │
│  │  • 쓰기 보호: 배타적                                                 │ │
│  │  • 기아 방지: 보호 읽기 가능                                          │ │
│  │                                                             │ │
│  │  단점:                                                               │ │
│  │  • 복잡도: 두 메커니즘 결합                                           │ │
│  │  • 메모리: 두 락 구조                                                │ │
│  │  • 빠른 읽기: 재시도 필요 가능                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 커널 seqcount + rwlock】                                     │
│  ──────────────────                                                  │
│  // include/linux/seqlock.h                                          │
│  typedef struct {                                                    │
│      struct seqcount seqcount;                                       │
│      seqcount_spinlock_t lock;                                       │
│  } seqlock_t;                                                        │
│                                                                     │
│  // 빠른 읽기                                                          │
│  unsigned int seq;                                                   │
│  do {                                                                │
│      seq = read_seqcount_begin(&s->seqcount);                        │
│      // 데이터 읽기                                                    │
│  } while (read_seqcount_retry(&s->seqcount, seq));                   │
│                                                                     │
│  // 쓰기                                                              │
│  write_seqlock(&s->lock);                                            │
│  // 데이터 쓰기                                                        │
│  write_sequnlock(&s->lock);                                          │
│                                                                     │
│  【사용 사례: /proc/stat】                                             │
│  ──────────────────                                                  │
│  // Linux fs/proc/stat.c                                             │
│  // 통계 읽기: seqlock 사용 (빠른 읽기)                                  │
│  // 쓰기: write_seqlock                                               │
│                                                                     │
│  int show_stat(struct seq_file *p, void *v)                          │
│  {                                                                   │
│      u64 user, nice, system, idle;                                   │
│      unsigned int seq;                                               │
│                                                                     │
│      do {                                                            │
│          seq = read_seqcount_begin(&s->seqcount);                    │
│          user = s->user;                                             │
│          nice = s->nice;                                             │
│          system = s->system;                                         │
│          idle = s->idle;                                             │
│      } while (read_seqcount_retry(&s->seqcount, seq));               │
│                                                                     │
│      // 출력                                                          │
│  }                                                                   │
│                                                                     │
│  【사용 사례: 네트워크 통계】                                            │
│  ──────────────────                                                  │
│  // net/core/dev.c                                                   │
│  struct rtnl_link_stats64 *dev_get_stats(struct net_device *dev)    │
│  {                                                                   │
│      // 빠른 읽기로 통계 수집                                           │
│      // 쓰기는 패킷 처리 시                                             │
│  }                                                                   │
│                                                                     │
│  【C++ 구현 예시】                                                      │
│  ──────────────────                                                  │
│  #include <atomic>                                                   │
│  #include <shared_mutex>                                            │
│                                                                     │
│  class RWSLock {                                                     │
│      std::atomic<unsigned> seq{0};                                  │
│      std::shared_mutex rwlock;                                       │
│                                                                     │
│  public:                                                            │
│      // 빠른 읽기                                                      │
│      template<typename F>                                           │
│      auto fast_read(F&& func) {                                     │
│          unsigned s1, s2;                                            │
│          decltype(func()) result;                                    │
│          do {                                                        │
│              s1 = seq.load(std::memory_order_acquire);              │
│              if (s1 & 1) continue;  // 쓰기 중                        │
│              result = func();                                        │
│              s2 = seq.load(std::memory_order_acquire);              │
│          } while (s1 != s2);                                         │
│          return result;                                              │
│      }                                                              │
│                                                                     │
│      // 보호 읽기                                                      │
│      template<typename F>                                           │
│      auto protected_read(F&& func) {                                │
│          std::shared_lock lock(rwlock);                             │
│          return func();                                              │
│      }                                                              │
│                                                                     │
│      // 쓰기                                                          │
│      template<typename F>                                           │
│      void write(F&& func) {                                          │
│          std::unique_lock lock(rwlock);                             │
│          seq.fetch_add(1, std::memory_order_release);  // 홀수        │
│          func();                                                      │
│          seq.fetch_add(1, std::memory_order_release);  // 짝수        │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  // 사용 예시                                                         │
│  RWSLock lock;                                                       │
│  int data;                                                           │
│                                                                     │
│  // 빠른 읽기                                                          │
│  int val = lock.fast_read([&] { return data; });                    │
│                                                                     │
│  // 보호 읽기                                                          │
│  int val = lock.protected_read([&] {                                │
│      // 복잡한 연산, 포인터 접근 등                                      │
│      return complex_calc(data);                                     │
│  });                                                                 │
│                                                                     │
│  // 쓰기                                                              │
│  lock.write([&] { data = new_value; });                              │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  시나리오              RW 락      Seqlock     RWS 락               │ │
│  │  ────                  ─────       ────────     ──────               │ │
│  │  읽기 90%, 쓰기 10%     1000        1500        1400               │ │
│  │  읽기 99%, 쓰기 1%      1500        5000        4800               │ │
│  │  읽기 중 쓰기 긴 경우    800         300 (기아)   900                │ │
│  │                                                             │ │
│  │  RWS: 유연성 + 기아 방지                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 읽기-쓰기 + 순차 락 조합
• 모드: 빠른 읽기, 보호 읽기, 쓰기
• 빠른 읽기: 버전 확인만, 락 없음
• 보호 읽기: 공유 락, 일관성 보장
• 쓰기: 배타 락, 버전 증가
• 장점: 유연성, 기아 방지
• 활용: Linux 커널 통계
```
