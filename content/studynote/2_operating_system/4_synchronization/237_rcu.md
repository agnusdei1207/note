# 237. RCU (Read-Copy-Update)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 읽기는 무잠금, 쓰기는 복사 후 교체
> 2. **가치**: 읽기 제로 오버헤드, Linux 커널 핵심 동기화
> 3. **융합**: Grace Period, 메모리 배리어, Seqlock과 연관

---

## Ⅰ. 개요

### 개념 정의

RCU(Read-Copy-Update)는 **읽기는 락 없이 수행하고, 쓰기는 데이터를 복사하여 수정한 후 포인터를 원자적으로 교체하는 동기화 기법**이다. Linux 커널에서广泛하게 사용된다.

### 💡 비유: 도서관 책 교체
RCU는 **도서관 책 교체**와 같다. 기존 책을 빼지 않고 새 책을 준비한다. 다 준비되면 책장의 책을 교체한다. 기존 책을 읽던 사람은 그대로 읽는다.

### RCU 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                RCU 구조                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 원리】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 읽기: 락 없이 직접 접근                                       │ │
│  │     rcu_read_lock();   // 선점 비활성화                         │ │
│  │     p = rcu_dereference(ptr);  // 포인터 읽기                   │ │
│  │     // 데이터 읽기                                               │
│  │     rcu_read_unlock(); // 선점 활성화                           │ │
│  │                                                             │ │
│  │  2. 쓰기: 복사-수정-교체                                          │ │
│  │     new = kmalloc(sizeof(*old));  // 복사                       │ │
│  │     *new = *old;                                                 │ │
│  │     new->field = new_value;  // 수정                            │ │
│  │     rcu_assign_pointer(ptr, new);  // 교체                       │ │
│  │     synchronize_rcu();  // Grace Period 대기                    │ │
│  │     kfree(old);  // 이전 데이터 해제                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시각화】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  초기 상태:                                                       │
│  │  ptr ──────► [데이터 A]                                         │ │
│  │                                                             │ │
│  │  쓰기 시작 (복사):                                                 │
│  │  ptr ──────► [데이터 A]    [데이터 A'] ← 새 복사본               │ │
│  │                                                             │ │
│  │  쓰기 완료 (포인터 교체):                                           │
│  │  ptr ──────► [데이터 A']   [데이터 A] ← 여전히 참조 가능          │ │
│  │                                                             │ │
│  │  Grace Period 종료:                                               │
│  │  ptr ──────► [데이터 A']                                   │ │
│  │                          [데이터 A] ← 해제됨                      │ │
│  │                                                             │ │
│  │  Grace Period: 모든 기존 읽기 완료할 때까지 대기                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Grace Period】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 모든 기존 읽기 스레드가 rcu_read_unlock() 할 때까지            │ │
│  │  • synchronize_rcu(): 블로킹 대기                               │ │
│  │  • call_rcu(): 콜백으로 비동기 처리                              │ │
│  │  • 커널은 주기적으로 GP 완료 확인                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

│  │  • rcu_assign_pointer(p, v): 안전한 포인터 할당                  │ │

│  │  • kfree_rcu(ptr, offset): RCU 후 kfree                         │ │

│  │  ────            ────         ────               ────             │ │
│  │  Mutex      락        락             일반적                       │ │

│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【사용 사례】                                                         │

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 커널 예시】                                                   │
│  ──────────────────                                                  │
│  #include <linux/rcupdate.h>                                         │
│                                                                     │
│  struct data {                                                      │
│      int value;                                                     │
│      struct rcu_head rcu;                                           │
│  };                                                                 │
│  struct data __rcu *global_ptr;                                     │
│                                                                     │
│  // 읽기 (락 없음)                                                    │
│  int read_value(void) {                                             │
│      struct data *p;                                                │
│      int value;                                                     │
│                                                                     │
│      rcu_read_lock();                                               │
│      p = rcu_dereference(global_ptr);                               │
│      value = p->value;                                              │
│      rcu_read_unlock();                                             │
│      return value;                                                  │
│  }                                                                  │
│                                                                     │
│  // 쓰기 (동기)                                                       │
│  void update_value(int new_val) {                                   │
│      struct data *old, *new;                                        │
│                                                                     │
│      old = rcu_dereference(global_ptr);                             │
│      new = kmalloc(sizeof(*new), GFP_KERNEL);                       │
│      new->value = new_val;                                          │
│                                                                     │
│      rcu_assign_pointer(global_ptr, new);                           │
│      synchronize_rcu();  // GP 대기                                  │
│      kfree(old);                                                     │
│  }                                                                  │
│                                                                     │
│  // 쓰기 (비동기 콜백)                                                │
│  static void free_callback(struct rcu_head *head) {                 │
│      struct data *p = container_of(head, struct data, rcu);         │
│      kfree(p);                                                       │
│  }                                                                  │
│                                                                     │
│  void update_value_async(int new_val) {                             │
│      struct data *old, *new;                                        │
│      old = rcu_dereference(global_ptr);                             │
│      new = kmalloc(sizeof(*new), GFP_KERNEL);                       │
│      new->value = new_val;                                          │
│      rcu_assign_pointer(global_ptr, new);                           │
│      call_rcu(&old->rcu, free_callback);                            │
│  }                                                                  │
│                                                                     │
│  【Userspace RCU (liburcu)】                                          │
│  ──────────────────                                                  │
│  #include <urcu.h>                                                   │
│  #include <urcu-pointer.h>                                           │
│                                                                     │
│  // 읽기                                                             │
│  rcu_read_lock();                                                    │
│  p = rcu_dereference(ptr);                                          │
│  // 데이터 사용                                                       │
│  rcu_read_unlock();                                                  │
│                                                                     │
│  // 쓰기                                                             │
│  new = malloc(sizeof(*new));                                         │
│  *new = *old;                                                        │
│  new->field = value;                                                 │
│  rcu_assign_pointer(ptr, new);                                       │
│  synchronize_rcu();                                                  │
│  free(old);                                                          │
│                                                                     │
│  【디버깅】                                                            │
│  ──────────────────                                                  │
│  # RCU 스톨 감지                                                      │
│  $ cat /sys/module/rcupdate/parameters/rcu_cpu_stall_suppress      │
│                                                                     │
│  # RCU 콜백 상태                                                      │
│  $ cat /debug/rcu/rcu_gp_callbacks                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 읽기 무잠금, 쓰기 복사-수정-교체
• Grace Period: 기존 읽기 완료 대기
• API: rcu_read_lock/unlock, synchronize_rcu
• 장점: 읽기 제로 오버헤드
• 단점: 쓰기 메모리 오버헤드, 지연
• 용도: 읽기 압도적, 포인터 기반 자료구조
```
