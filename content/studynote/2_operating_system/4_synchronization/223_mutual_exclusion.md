# 223. 상호 배제 (Mutual Exclusion)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 한 번에 하나의 스레드만 자원 사용
> 2. **가치**: 동시 접근으로 인한 데이터 손상 방지
> 3. **융합**: 임계 영역, 락, 동기화와 연관

---

## Ⅰ. 개요

### 개념 정의

상호 배제(Mutual Exclusion, Mutex)는 **여러 스레드가 동시에 같은 자원에 접근하지 못하도록 보장하는 속성**이다. 동기화의 가장 기본적인 요구조건이다.

### 💡 비유: 일인용 화장실
상호 배제는 **일인용 화장실**과 같다. 안에 사람이 있으면 잠금장치가 걸리고, 밖에서는 대기한다. 한 사람만 사용한다.

### 상호 배제 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                상호 배제 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【상호 배제 없이】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  스레드 A ─────────────────────────────────────►             │ │
│  │            │                                                 │ │
│  │            ▼                                                 │ │
│  │  공유 자원 ┌───────────────────────────────────┐              │ │
│  │           │  data = 10                        │              │ │
│  │           │  A: data = 20                     │              │ │
│  │           │  B: data = 30  ◄─ 동시 수정!        │              │ │
│  │           └───────────────────────────────────┘              │ │
│  │            ▲                                                 │ │
│  │            │                                                 │ │
│  │  스레드 B ─────────────────────────────────────►             │ │
│  │                                                             │ │
│  │  문제: 데이터 손상, 예측 불가능한 결과                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상호 배제 있이】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  스레드 A ──┬── lock() ──┬── 접근 ──┬── unlock() ──►         │ │
│  │            │            │          │                         │ │
│  │            ▼            ▼          ▼                         │ │
│  │  공유 자원 ┌───────────────────────────────────┐              │ │
│  │           │  data = 10                        │              │ │
│  │           │  A: data = 20 (단독)               │              │ │
│  │           │  B: data = 30 (단독)               │              │ │
│  │           └───────────────────────────────────┘              │ │
│  │            ▲            ▲          ▲                         │ │
│  │            │            │          │                         │ │
│  │  스레드 B ──┴── 대기... ──┴── lock() ── unlock() ──►         │ │
│  │                                                             │ │
│  │  결과: 데이터 일관성 보장                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상호 배제 보장 방법】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  방법                설명                    성능               │ │
│  │  ────                    ────                        ────       │ │
│  │  인터럽트 금지        단일 CPU만              낮음              │ │
│  │  소프트웨어 알고리즘    Peterson 등             낮음              │ │
│  │  하드웨어 지원        TAS, CAS                 높음              │ │
│  │  OS 동기화 객체        Mutex, Semaphore         중간              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                상호 배제 상세                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【소프트웨어 해법: Peterson 알고리즘】                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  int turn = 0;                                                  │ │
│  │  bool flag[2] = {false, false};                                │ │
│  │                                                             │ │
│  │  // 스레드 i의 진입 코드                                        │ │
│  │  void enter_critical(int i) {                                  │ │
│  │      int j = 1 - i;  // 상대방                                 │ │
│  │      flag[i] = true;   // 진입 의사 표시                       │ │
│  │      turn = j;         // 상대방에게 차례 양보                  │ │
│  │      while (flag[j] && turn == j) { /* 대기 */ }              │ │
│  │  }                                                             │ │
│  │                                                             │ │
│  │  void exit_critical(int i) {                                   │ │
│  │      flag[i] = false;                                          │ │
│  │  }                                                             │ │
│  │                                                             │ │
│  │  단점: 바쁜 대기(busy waiting), 2개 스레드만                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【하드웨어 해법: Test-and-Set】                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 원자적 명령어                                                │ │
│  │  bool test_and_set(bool* lock) {                               │ │
│  │      bool old = *lock;                                         │ │
│  │      *lock = true;                                             │ │
│  │      return old;  // 이전 값 반환                               │ │
│  │  }                                                             │ │
│  │                                                             │ │
│  │  void lock(bool* lock) {                                       │ │
│  │      while (test_and_set(lock)) { /* 스핀 */ }                │ │
│  │  }                                                             │ │
│  │                                                             │ │
│  │  void unlock(bool* lock) {                                     │ │
│  │      *lock = false;                                            │ │
│  │  }                                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상호 배제 속성】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. Safety (안전성): 동시 진입 없음                             │ │
│  │  2. Liveness (활성): 진입 보장                                  │ │
│  │  3. Fairness (공정성): 기아 없음                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【POSIX Mutex】                                                      │
│  ──────────────────                                                  │
│  pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;                 │
│                                                                     │
│  void protected_access() {                                          │
│      pthread_mutex_lock(&mutex);     // 상호 배제 시작               │
│      // 임계 영역                                                    │
│      shared_data++;                                                 │
│      pthread_mutex_unlock(&mutex);   // 상호 배제 종료               │
│  }                                                                  │
│                                                                     │
│  【C++ std::atomic_flag】                                             │
│  ──────────────────                                                  │
│  #include <atomic>                                                   │
│  std::atomic_flag lock = ATOMIC_FLAG_INIT;                          │
│                                                                     │
│  void spinlock_lock() {                                             │
│      while (lock.test_and_set(std::memory_order_acquire)) {        │
│          /* 스핀 */                                                │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  void spinlock_unlock() {                                           │
│      lock.clear(std::memory_order_release);                         │
│  }                                                                  │
│                                                                     │
│  【Linux 커널 spinlock】                                              │
│  ──────────────────                                                  │
│  spinlock_t my_lock;                                                │
│  spin_lock_init(&my_lock);                                          │
│                                                                     │
│  void critical_section(void) {                                      │
│      spin_lock(&my_lock);       // 상호 배제 획득                    │
│      // 임계 영역                                                    │
│      spin_unlock(&my_lock);     // 상호 배제 해제                    │
│  }                                                                  │
│                                                                     │
│  【성능 측정】                                                         │
│  ──────────────────                                                  │
│  # 락 경합 측정                                                       │
│  $ perf stat -e lock:lock_acquire,lock:lock_conflict ./program     │
│                                                                     │
│  # 락 대기 시간                                                       │
│  $ perf record -e lock:lock_wait ./program                         │
│  $ perf report                                                      │
│                                                                     │
│  # ftrace로 락 추적                                                  │
│  $ trace-cmd record -e lock:lock_acquire ./program                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
