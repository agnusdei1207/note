# 238. Grace Period (유예 기간)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모든 기존 읽기가 완료될 때까지의 대기 기간
> 2. **가치**: RCU에서 안전한 메모리 해제 보장
> 3. **융합**: RCU, 메모리 관리, 동기화와 연관

---

## Ⅰ. 개요

### 개념 정의

Grace Period(GP)는 **RCU에서 포인터 교체 후, 해당 포인터를 참조하던 모든 읽기 작업이 완료될 때까지의 대기 기간**이다. 이 기간이 지나야 이전 데이터를 안전하게 해제할 수 있다.

### 💡 비유: 회의 종료 대기
Grace Period는 **회의 종료 대기**와 같다. "회의 끝났습니다"라고 말한 후, 모든 참석자가 회의실을 나갈 때까지 기다린다. 다 나가면 방을 치운다.

### Grace Period 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                Grace Period 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Grace Period 타임라인】                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  시간 ───────────────────────────────────────────────────►    │ │
│  │                                                             │ │
│  │         GP 시작                    GP 종료                    │ │
│  │            │                          │                        │ │
│  │            ▼                          ▼                        │ │
│  │  ──────────┼──────────────────────────┼───────────────────    │ │
│  │            │                          │                        │ │
│  │  R1: ──────┼────[읽기 중]─────────────┼──────────────────────  │ │
│  │            │                          │                        │ │
│  │  R2: ──────┼────[읽기 중]─────────────┼──────────────────────  │ │
│  │            │                          │                        │ │
│  │  W:  ──────┼──[교체]──[GP 대기]───────┼──[해제]─────────────   │ │
│  │            │                          │                        │ │
│  │  R3: ──────┼──────────────────────────┼──[읽기]────────────── │ │
│  │            │         (새 데이터)         │                        │ │
│  │  R4: ──────┼──────────────────────────┼──[읽기]────────────── │ │
│  │            │         (새 데이터)         │                        │ │
│  │                                                             │ │
│  │  GP 종료 = GP 시작 전에 시작된 모든 읽기 완료                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【GP 완료 조건】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 모든 CPU가 최소 한 번 컨텍스트 스위치 수행                    │ │
│  │  2. 또는 모든 CPU가 최소 한 번 idle 진입                         │ │
│  │  3. 또는 강제적인 퀴즈(quip) 수행                                 │ │
│  │                                                             │ │
│  │  CPU가 컨텍스트 스위치하면:                                        │ │
│  │  • rcu_read_lock() 보유 중이 아님을 의미                         │ │
│  │  • 기존 포인터 참조 종료                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【GP 감지 방식】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • Tree RCU: 계층적 gp state 관리                               │ │
│  │  • Tiny RCU: 단일 CPU용 간단한 구현                             │ │
│  │  • GP kthread: 전용 커널 스레드가 GP 관리                        │ │
│  │  • FQS (Force-Quiescent-State): 주기적 강제 체크                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                Grace Period 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Quiescent State】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  정의: CPU가 RCU 읽기 영역에 있지 않은 상태                        │ │
│  │                                                             │ │
│  │  예:                                                              │
│  │  • 컨텍스트 스위치                                                │ │
│  │  • idle 상태 진입                                                 │ │
│  │  • 사용자 모드 진입                                               │ │
│  │  • nohz_full 모드                                                 │ │
│  │                                                             │ │
│  │  모든 CPU가 QS를 보고하면 GP 완료                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【synchronize_rcu() 동작】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 새 GP 시작 요청                                               │ │
│  │  2. GP 번호 할당                                                   │ │
│  │  3. 모든 CPU의 QS 대기                                             │ │
│  │  4. QS 보고 완료 시 GP 종료                                        │ │
│  │  5. 대기 중인 스레드 깨움                                          │ │
│  │                                                             │ │
│  │  오버헤드:                                                         │ │
│  │  • 수십 마이크로초 ~ 수백 밀리초                                   │ │
│  │  • CPU 수, 로드에 따라 다름                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【call_rcu() vs synchronize_rcu()】                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  synchronize_rcu():                                               │ │
│  │  • 블로킹                                                          │ │
│  │  • GP 완료까지 대기                                                │ │
│  │  • 사용하기 쉬움                                                   │ │
│  │                                                             │ │
│  │  call_rcu():                                                       │ │
│  │  • 논블로킹                                                         │ │
│  │  • 콜백 등록 후 즉시 반환                                          │ │
│  │  • GP 후 콜백 실행                                                 │ │
│  │  • 대량 작업에 효율적                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 모니터링 및 튜닝
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RCU 상태 모니터링】                                                 │
│  ──────────────────                                                  │
│  # RCU 통계                                                           │
│  $ cat /proc/rcu/rcu_exp/rcudata                                    │
│  $ cat /debug/rcu/rcu_gp                                            │
│                                                                     │
│  # GP 완료 시간                                                       │
│  $ cat /sys/kernel/debug/rcu/rcu_gp                                 │
│                                                                     │
│  # 콜백 큐 길이                                                        │
│  $ cat /debug/rcu/*/*/rcu_data                                      │
│                                                                     │
│  【RCU 스톨 감지】                                                      │
│  ──────────────────                                                  │
│  # 스톨 경고 (dmesg)                                                  │
│  rcu_sched self-detected stall on CPU                               │
│  rcu_preempt blocked for 12345 ms                                   │
│                                                                     │
│  # 스톨 원인                                                           │
│  • 너무 긴 RCU 읽기 영역                                               │
│  • preempt disable 너무 긺                                            │
│  • CPU이 하이퍼바이저에서 멈춤                                         │
│  • 실시간 태스크가 CPU 독점                                            │
│                                                                     │
│  【성능 튜닝】                                                          │
│  ──────────────────                                                  │
│  # expedited GP (빠르지만 비용 높음)                                   │
│  synchronize_rcu_expedited();                                        │
│                                                                     │
│  # lazy GP (지연되지만 효율적)                                        │
│  synchronize_rcu();  // 기본                                         │
│                                                                     │
│  # NOHZ 영향                                                         │
│  # nohz_full CPU는 QS 보고 지연될 수 있음                              │
│                                                                     │
│  【디버깅 옵션】                                                        │
│  ──────────────────                                                  │
│  CONFIG_RCU_TRACE=y           # RCU 추적                             │
│  CONFIG_RCU_STALL_COMMON=y    # 스톨 감지                             │
│  CONFIG_DEBUG_LOCK_ALLOC=y    # 락 디버깅                             │
│  CONFIG_PROVE_RCU=y            # RCU 검증                             │
│                                                                     │
│  # RCU 엄격 모드                                                      │
│  CONFIG_RCU_STRICT_GRACE_PERIOD=y                                   │
│                                                                     │
│  【GP 대기 시간 최소화】                                                │
│  ──────────────────                                                  │
│  • RCU 읽기 영역 최소화                                                │
│  • preempt_disable 구간 최소화                                        │
│  • 긴 작업은 workqueue로 이동                                         │
│  • 콜백 방식 고려 (call_rcu)                                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
