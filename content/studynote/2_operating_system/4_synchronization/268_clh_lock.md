# 268. CLH 락 (CLH Lock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연결 리스트 기반 스핀락
> 2. **가치**: 캐시 효율적, NUMA 친화적
> 3. **융합**: MCS 락, 스핀락, FIFO 락과 연관

---

## Ⅰ. 개요

### 개념 정의

CLH 락(Craig, Landin, and Hagersten Lock)은 **각 스레드가 연결 리스트의 자신 노드에서 스핀하는 큐 기반 스핀락**이다. 캐시 효율성과 공정성을 제공한다.

### 💡 비유: 줄서기
CLH 락은 **줄서기**와 같다. 각자 자신의 자리에 선다. 앞 사람이 끝나면 내 차례다. 앞 사람만 보면 된다.

### CLH 락 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                CLH 락 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct clh_node {                                                 │ │
│  │      atomic_bool locked;  // true: 대기, false: 진입               │ │
│  │  };                                                              │ │
│  │                                                             │ │
│  │  struct clh_lock {                                                 │ │
│  │      atomic<clh_node*> tail;  // 리스트 끝                          │ │
│  │  };                                                              │ │
│  │                                                             │ │
│  │  각 스레드:                                                         │ │
│  │  • my_node: 자신의 노드                                              │ │
│  │  • pred_node: 앞 노드 (여기서 스핀)                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【동작 원리】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  lock():                                                           │ │
│  │  1. my_node->locked = true  (대기 상태)                            │ │
│  │  2. pred_node = tail.exchange(my_node)  // 이전 tail 저장          │ │
│  │  3. while (pred_node->locked) spin();  // 이전 노드에서 스핀         │ │
│  │                                                             │ │
│  │  unlock():                                                         │ │
│  │  1. my_node->locked = false  // 다음 스레드 진입 허용               │ │
│  │  // 다음 lock 호출 시 새 노드 사용 권장                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【큐 상태 변화】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  초기: tail → [node0(locked=false)]                                │ │
│  │                                                             │ │
│  │  T1 lock:                                                          │ │
│  │  tail → [node1(locked=true)]                                       │ │
│  │           ↑                                                         │ │
│  │      node0(locked=false)                                           │ │
│  │      (T1은 node0에서 스핀)                                           │ │
│  │                                                             │ │
│  │  T2 lock:                                                          │ │
│  │  tail → [node2(locked=true)]                                       │ │
│  │           ↑                                                         │ │
│  │      node1(locked=true)                                            │ │
│  │           ↑                                                         │ │
│  │      node0(locked=false)                                           │ │
│  │      (T2은 node1에서 스핀)                                           │ │
│  │                                                             │ │
│  │  T0 unlock: node0.locked = false                                   │ │
│  │  → T1 진입!                                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【캐시 장점】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  일반 스핀락: 모든 코어가 같은 변수 스핀                                  │ │
│  │  → 캐시 라인 바운싱 (MESI 트래픽 폭증)                                  │ │
│  │                                                             │ │
│  │  CLH 락: 각 코어가 자신의 pred_node에서만 스핀                          │ │
│  │  → pred_node는 지역적으로 캐시                                        │ │
│  │  → 캐시 라인 바운싱 최소화                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                CLH 락 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CLH vs MCS】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CLH:                                                              │ │
│  │  • 스핀 위치: 이전 노드 (pred_node->locked)                           │ │
│  │  • 노드 방향: 이전 → 현재 (backward)                                 │ │
│  │  • unlock: 자신의 locked = false                                    │ │
│  │  • 장점: 구현 단순                                                    │ │
│  │  • 단점: NUMA에서 이전 노드가 원격일 수 있음                            │ │
│  │                                                             │ │
│  │  MCS:                                                              │ │
│  │  • 스핀 위치: 자신의 노드 (my_node->locked)                           │ │
│  │  • 노드 방향: 현재 → 다음 (forward)                                  │ │
│  │  • unlock: 다음 노드의 locked = false                               │ │
│  │  • 장점: NUMA 친화적                                                 │ │
│  │  • 단점: unlock가 다음 노드 찾아야 함                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NUMA 고려사항】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CLH:                                                              │ │
│  │  • 스핀 위치 = 이전 스레드의 노드                                      │ │
│  │  • 이전 스레드가 다른 NUMA 노드 → 원격 메모리 스핀                       │ │
│  │  • NUMA 비효율                                                      │ │
│  │                                                             │ │
│  │  MCS:                                                              │ │
│  │  • 스핀 위치 = 자신의 노드 (지역 메모리)                                │ │
│  │  • NUMA 친화적                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                              │ │
│  │  • 캐시 효율적 (각자 자신의 pred에서 스핀)                              │ │
│  │  • FIFO 공정성                                                      │ │
│  │  • 기아 상태 없음                                                    │ │
│  │  • 구현 비교적 단순                                                   │ │
│  │  • O(1) 공간 (스레드당 1노드)                                         │ │
│  │                                                             │ │
│  │  단점:                                                               │ │
│  │  • NUMA 비효율 가능                                                  │ │
│  │  • 노드 재사용 주의                                                   │ │
│  │  • 일반 스핀락보다 오버헤드                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Java CLH 락 구현】                                                  │
│  ──────────────────                                                  │
│  import java.util.concurrent.atomic.AtomicReference;                │
│  import java.util.concurrent.atomic.AtomicBoolean;                  │
│                                                                     │
│  class CLHLock {                                                     │
│      static class Node {                                             │
│          final AtomicBoolean locked = new AtomicBoolean(false);      │
│      }                                                              │
│                                                                     │
│      private final AtomicReference<Node> tail;                       │
│      private final ThreadLocal<Node> myNode;                        │
│      private final ThreadLocal<Node> predNode;                      │
│                                                                     │
│      public CLHLock() {                                              │
│          tail = new AtomicReference<>(new Node());                  │
│          myNode = ThreadLocal.withInitial(() -> new Node());        │
│          predNode = new ThreadLocal<>();                             │
│      }                                                              │
│                                                                     │
│      public void lock() {                                            │
│          Node node = myNode.get();                                  │
│          node.locked.set(true);                                     │
│          Node pred = tail.getAndSet(node);                           │
│          predNode.set(pred);                                        │
│          while (pred.locked.get()) {                                 │
│              Thread.onSpinWait();                                    │
│          }                                                          │
│      }                                                              │
│                                                                     │
│      public void unlock() {                                          │
│          Node node = myNode.get();                                  │
│          node.locked.set(false);                                    │
│          myNode.set(predNode.get());  // 노드 재사용                   │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【C++ CLH 락】                                                       │
│  ──────────────────                                                  │
│  #include <atomic>                                                   │
│  #include <memory>                                                  │
│                                                                     │
│  class CLHLock {                                                     │
│      struct Node {                                                   │
│          std::atomic<bool> locked{false};                            │
│      };                                                              │
│                                                                     │
│      std::atomic<Node*> tail;                                        │
│      static thread_local Node* my_node;                              │
│      static thread_local Node* pred_node;                            │
│                                                                     │
│  public:                                                            │
│      CLHLock() {                                                     │
│          tail.store(new Node());                                     │
│      }                                                              │
│                                                                     │
│      void lock() {                                                  │
│          my_node = new Node();                                      │
│          my_node->locked.store(true);                                │
│          pred_node = tail.exchange(my_node);                         │
│          while (pred_node->locked.load()) {                          │
│              cpu_relax();                                            │
│          }                                                          │
│      }                                                              │
│                                                                     │
│      void unlock() {                                                │
│          my_node->locked.store(false);                               │
│          // 다음 lock에서 pred_node 재사용                              │
│          my_node = pred_node;                                        │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  thread_local CLHLock::Node* CLHLock::my_node = nullptr;             │
│  thread_local CLHLock::Node* CLHLock::pred_node = nullptr;          │
│                                                                     │
│  【Linux 커널 qspinlock】                                              │
│  ──────────────────                                                  │
│  // CLH/MCS 변형, 4바이트 큐 기반 스핀락                                │
│  // 3개 스레드 이하에서 일반 스핀락처럼 동작                                │
│  // 그 이상에서 큐 모드로 전환                                          │
│                                                                     │
│  // include/asm-generic/qspinlock.h                                 │
│  typedef struct {                                                    │
│      atomic_t val;                                                   │
│  } arch_spinlock_t;                                                  │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  64 코어, 높은 경합                                                  │ │
│  │  ──────────────────                                                │ │
│  │  락 타입              처리량 (ops)    캐시 미스율                     │ │
│  │  ────────              ─────────        ───────────                 │ │
│  │  스핀락 (CAS)           50,000         80%                          │ │
│  │  티켓 락                80,000         40%                          │ │
│  │  CLH 락                200,000        15%                          │ │
│  │  MCS 락                220,000        10%                          │ │
│  │                                                             │ │
│  │  CLH/MCS: 캐시 미스 크게 감소                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【언제 사용할까?】                                                      │
│  ──────────────────                                                  │
│  적합:                                                                 │
│  • 높은 경합, 다중 코어                                                 │
│  • 캐시 효율 중요                                                       │
│  • 공정성 필요                                                         │
│  • 단일 NUMA 노드                                                      │
│                                                                     │
│  부적합:                                                               │
│  • NUMA 시스템 (MCS 권장)                                              │
│  • 낮은 경합 (일반 락 충분)                                              │
│  • 짧은 임계 영역 (오버헤드)                                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
