# 256. 브래킷 (Bracket)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 임계 영역 진입/퇴출을 자동화하는 RAII 패턴
> 2. **가치**: 예외 안전, 락 누수 방지
> 3. **융합**: 스마트 포인터, RAII, 뮤텍스와 연관

---

## Ⅰ. 개요

### 개념 정의

브래킷(Bracket)은 **락 획득과 해제를 자동으로 처리하는 RAII(Resource Acquisition Is Initialization) 기법**이다. 생성자에서 락을 획득하고 소멸자에서 해제하여 예외 안전성을 보장한다.

### 💡 비유: 자동 문 잠금장치
브래킷은 **자동 문 잠금장치**와 같다. 문을 닫으면 자동으로 잠긴다. 열쇠를 안 잃어버려도 안전하다. 나갈 때 자동으로 잠긴다.

### 브래킷 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                브래킷 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 패턴】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  // 수동 락 관리 (위험)                                            │ │
│  │  lock();                                                            │ │
│  │  // 작업...                                                         │ │
│  │  if (error) {                                                       │ │
│  │      unlock();  // 실수로 누락 가능                                  │ │
│  │      return;                                                        │ │
│  │  }                                                                  │ │
│  │  // 더 많은 작업...                                                  │ │
│  │  unlock();  // 실수로 누락 가능                                      │ │
│  │                                                             │ │
│  │  문제:                                                               │ │
│  │  • 예외 발생 시 unlock 누락 → 락 누수                                │ │
│  │  • 여러 return 경로                                                  │ │
│  │  • 코드 복잡                                                        │ │
│  │                                                             │ │
│  │  // 브래킷 패턴 (안전)                                               │ │
│  │  {                                                                  │ │
│  │      LockGuard guard(lock);  // 생성자: lock 획득                   │ │
│  │      // 작업...                                                     │ │
│  │      if (error) {                                                   │ │
│  │          return;  // 소멸자 자동 호출 → unlock                       │ │
│  │      }                                                              │ │
│  │      // 더 많은 작업...                                              │ │
│  │  }  // 소멸자 자동 호출 → unlock                                     │ │
│  │                                                             │ │
│  │  장점:                                                               │ │
│  │  • 예외 안전                                                        │ │
│  │  • 락 누수 방지                                                      │ │
│  │  • 코드 단순화                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【구현 예시】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  template<typename Lock>                                           │ │
│  │  class LockGuard {                                                 │ │
│  │      Lock& lock_;                                                   │ │
│  │  public:                                                            │ │
│  │      explicit LockGuard(Lock& lock) : lock_(lock) {                 │ │
│  │          lock_.acquire();  // 또는 lock_.lock()                     │ │
│  │      }                                                              │ │
│  │      ~LockGuard() {                                                 │ │
│  │          lock_.release();  // 또는 lock_.unlock()                   │ │
│  │      }                                                              │ │
│  │      // 복사 금지                                                    │ │
│  │      LockGuard(const LockGuard&) = delete;                          │ │
│  │      LockGuard& operator=(const LockGuard&) = delete;               │ │
│  │  };                                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RAII 원칙】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  Resource Acquisition Is Initialization:                           │ │
│  │  • 자원 획득 = 객체 생성                                             │ │
│  │  • 자원 해제 = 객체 소멸                                             │ │
│  │  • 스코프 기반 자동 관리                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                브래킷 상세                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ 표준 라이브러리】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. std::lock_guard (C++11)                                        │ │
│  │     • 가장 단순                                                      │ │
│  │     • 생성 시 락 획득, 소멸 시 해제                                   │ │
│  │     • 추가 기능 없음                                                 │ │
│  │                                                             │ │
│  │  2. std::unique_lock (C++11)                                       │ │
│  │     • 유연함                                                        │ │
│  │     • 지연 락, 조건적 락, 소유권 이전 가능                             │ │
│  │     • condition_variable과 함께 사용                                 │ │
│  │                                                             │ │
│  │  3. std::shared_lock (C++14)                                       │ │
│  │     • 공유 락 (읽기 락)                                              │ │
│  │     • shared_mutex와 함께 사용                                       │ │
│  │                                                             │ │
│  │  4. std::scoped_lock (C++17)                                       │ │
│  │     • 다중 락 교착상태 없이 획득                                       │ │
│  │     • 가변 인자 템플릿                                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【다중 락 문제】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  문제: 교착상태 위험                                                  │ │
│  │  lock(mutex_a);                                                     │ │
│  │  lock(mutex_b);  // 다른 스레드가 반대 순서면 교착상태                  │ │
│  │                                                             │ │
│  │  해결: std::lock (C++11)                                            │ │
│  │  std::lock(mutex_a, mutex_b);  // 교착상태 없이 모두 획득             │ │
│  │  std::lock_guard<std::mutex> lock_a(mutex_a, std::adopt_lock);     │ │
│  │  std::lock_guard<std::mutex> lock_b(mutex_b, std::adopt_lock);     │ │
│  │                                                             │ │
│  │  또는 scoped_lock (C++17):                                           │ │
│  │  std::scoped_lock lock(mutex_a, mutex_b);                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ std::lock_guard】                                              │
│  ──────────────────                                                  │
│  #include <mutex>                                                    │
│                                                                     │
│  std::mutex mtx;                                                     │
│  int shared_data;                                                    │
│                                                                     │
│  void safe_increment() {                                            │
│      std::lock_guard<std::mutex> lock(mtx);                          │
│      shared_data++;                                                  │
│      // 예외 발생해도 소멸자가 unlock 호출                               │
│  }                                                                  │
│                                                                     │
│  【C++ std::unique_lock】                                             │
│  ──────────────────                                                  │
│  #include <mutex>                                                    │
│  #include <condition_variable>                                       │
│                                                                     │
│  std::mutex mtx;                                                     │
│  std::condition_variable cv;                                         │
│  bool ready = false;                                                 │
│                                                                     │
│  void wait_for_signal() {                                           │
│      std::unique_lock<std::mutex> lock(mtx);                         │
│      cv.wait(lock, []{ return ready; });  // lock/unlock 자동        │
│      // 작업...                                                       │
│  }                                                                  │
│                                                                     │
│  【C++ std::scoped_lock (다중 락)】                                   │
│  ──────────────────                                                  │
│  #include <mutex>                                                    │
│                                                                     │
│  std::mutex mtx1, mtx2;                                              │
│                                                                     │
│  void transfer_data() {                                             │
│      std::scoped_lock lock(mtx1, mtx2);  // 교착상태 없이 모두 획득     │
│      // 안전하게 두 리소스 접근                                        │
│  }                                                                  │
│                                                                     │
│  【Java synchronized 블록】                                           │
│  ──────────────────                                                  │
│  private final Object lock = new Object();                          │
│                                                                     │
│  public void safeMethod() {                                         │
│      synchronized (lock) {                                           │
│          // 임계 영역                                                 │
│          // 예외 발생해도 자동 해제                                     │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【Python with 문】                                                   │
│  ──────────────────                                                  │
│  import threading                                                   │
│                                                                     │
│  lock = threading.Lock()                                            │
│                                                                     │
│  def safe_increment():                                              │
│      with lock:                                                     │
│          # 임계 영역                                                 │
│          pass                                                        │
│      # with 블록 종료 시 자동 해제                                     │
│                                                                     │
│  【Rust MutexGuard】                                                  │
│  ──────────────────                                                  │
│  use std::sync::Mutex;                                              │
│                                                                     │
│  let mutex = Mutex::new(0);                                         │
│                                                                     │
│  {                                                                  │
│      let mut guard = mutex.lock().unwrap();                          │
│      *guard += 1;                                                    │
│  }  // guard 드롭 시 자동 unlock                                      │
│                                                                     │
│  【Go defer】                                                         │
│  ──────────────────                                                  │
│  var mu sync.Mutex                                                   │
│                                                                     │
│  func safeIncrement() {                                             │
│      mu.Lock()                                                       │
│      defer mu.Unlock()  // 함수 종료 시 실행                            │
│      // 임계 영역                                                    │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 락 획득/해제 자동화 (RAII)

• 다중 락: std::lock, scoped_lock으로 교착상태 방지
• 언어: C++, Java, Python, Rust, Go
```
