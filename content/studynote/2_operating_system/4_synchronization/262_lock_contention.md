# 262. 락 경합 (Lock Contention)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 스레드가 동시에 같은 락 요청
> 2. **가치**: 성능 저하의 주요 원인 식별
> 3. **융합**: 스핀락, 스케줄러, 캐시 일관성과 연관

---

## Ⅰ. 개요

### 개념 정의

락 경합(Lock Contention)은 **여러 스레드가 동시에 같은 락을 획득하려고 경쟁하는 상황**이다. 높은 경합은 대기 시간 증가와 성능 저하를 유발한다.

### 💡 비유: 단일 출구
락 경합은 **단일 출구**와 같다. 100명이 동시에 나가려 한다. 한 번에 한 명만. 줄이 길어진다. 출구가 병목이다.

### 락 경합 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                락 경합 구조                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【경합 시나리오】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  낮은 경합 (Low Contention):                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  T1: lock() ──[작업]── unlock()                           │ │ │
│  │  │  T2:          lock() ──[작업]── unlock()                  │ │ │
│  │  │  T3:                   lock() ──[작업]── unlock()         │ │ │
│  │  │                                                         │ │ │
│  │  │  특징: 거의 대기 없음, 높은 처리량                               │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  높은 경합 (High Contention):                                      │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  T1: lock() ──[작업]── unlock()                           │ │ │
│  │  │  T2: lock() ─X─[대기]─X─────[작업]── unlock()              │ │ │
│  │  │  T3: lock() ─X─[대기]─X──[대기]─X─────[작업]── unlock()     │ │ │
│  │  │  T4: lock() ─X─[대기]─X──[대기]─X──[대기]─X──[작업]──►      │ │ │
│  │  │                    ↑                                       │ │ │
│  │  │              긴 대기 시간, 낮은 처리량                              │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【경합 측정 지표】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 락 대기 시간 (Wait Time):                                       │ │
│  │     • 평균 대기 시간                                                 │ │
│  │     • 최대 대기 시간                                                 │ │
│  │                                                             │ │
│  │  2. 락 획득 비율 (Acquisition Rate):                                │ │
│  │     • 성공 횟수 / 전체 시도                                          │ │
│  │                                                             │ │
│  │  3. 경합률 (Contention Ratio):                                     │ │
│  │     • 충돌 횟수 / 전체 시도                                          │ │
│  │                                                             │ │
│  │  4. 큐 길이 (Queue Length):                                        │ │
│  │     • 대기 중인 스레드 수                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【영향】                                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 처리량 감소                                                      │ │
│  │  • 응답 시간 증가                                                    │ │
│  │  • CPU 사용률 변동 (스핀락 시 높음)                                   │ │
│  │  • 컨텍스트 스위치 증가 (블로킹 락 시)                                  │ │
│  │  • 캐시 효율 저하                                                    │ │
│  │  • 스케일링 한계                                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                락 경합 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【경합 완화 전략】                                                     │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 임계 영역 최소화                                                 │ │
│  │     • 락 내부 코드 줄이기                                             │ │
│  │     • 락 밖에서 할 수 있는 작업 분리                                   │ │
│  │                                                             │ │
│  │  2. 락 세분성 향상                                                   │ │
│  │     • 굵은 락 → 세밀한 락                                             │ │
│  │     • 락 스트라이핑                                                  │ │
│  │                                                             │ │
│  │  3. 읽기-쓰기 분리                                                   │ │
│  │     • 읽기는 공유 락                                                 │ │
│  │     • 쓰기만 배타 락                                                 │ │
│  │                                                             │ │
│  │  4. 락 프리 알고리즘                                                  │ │
│  │     • CAS, 원자적 연산                                               │ │
│  │     • RCU                                                          │ │
│  │                                                             │ │
│  │  5. 지역성 향상                                                      │ │
│  │     • Per-CPU 변수                                                  │ │
│  │     • 스레드 로컬 저장소                                              │ │
│  │                                                             │ │
│  │  6. 적응적 락                                                       │ │
│  │     • 스핀 + 블로킹 조합                                              │ │
│  │     • 경합 수준에 따른 전환                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【경합 vs 스레드 수】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  처리량                                                              │ │
│  │    ↑                                                                   │ │
│  │    │     _________ (이상적)                                         │ │
│  │    │    /                                                          │ │
│  │    │   /  ________ (낮은 경합)                                      │ │
│  │    │  /  /                                                         │ │
│  │    │ /  /   ______ (중간 경합)                                      │ │
│  │    │/  /   /                                                       │ │
│  │    └──/───/─────── (높은 경합)                                       │ │
│  │      └────┴────┴────┴────┴────→ 스레드 수                            │ │
│  │                                                             │ │
│  │  아메달의 법칙 (Amdahl's Law):                                       │ │
│  │  순차적 부분(락)이 병렬 성능을 제한                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【경합 모니터링 (Java)】                                              │
│  ──────────────────                                                  │
│  // JMX으로 락 경합 모니터링                                            │
│  import java.lang.management.*;                                     │
│                                                                     │
│  ThreadMXBean threadBean = ManagementFactory.getThreadMXBean();    │
│  long contentionTime = threadBean.getThreadContentionTime();       │
│  long blockedCount = threadBean.getThreadBlockedCount();           │
│                                                                     │
│  【Linux futex 통계】                                                 │
│  ──────────────────                                                  │
│  cat /proc/<pid>/status | grep voluntary_ctxt_switches              │
│  cat /proc/<pid>/status | grep nonvoluntary_ctxt_switches           │
│                                                                     │
│  perf lock record -p <pid>                                          │
│  perf lock report                                                   │
│                                                                     │
│  【경합 완화 예시】                                                     │
│  ──────────────────                                                  │
│  // Before: 높은 경합                                                  │
│  synchronized (globalLock) {                                        │
│      // 모든 작업이 락 내부                                              │ │
│      data = compute();                                              │
│      result = process(data);                                        │
│      write(result);                                                 │
│  }                                                                  │
│                                                                     │
│  // After: 임계 영역 최소화                                             │
│  data = compute();  // 락 밖                                          │
│  synchronized (globalLock) {                                        │
│      result = process(data);  // 락 내부                              │
│  }                                                                  │
│  write(result);  // 락 밖                                             │
│                                                                     │
│  // 락 스트라이핑                                                       │
│  StripedLock stripedLock = new StripedLock(16);                    │
│  stripedLock.lock(key);                                             │
│  // 작업                                                             │
│  stripedLock.unlock(key);                                           │
│                                                                     │
│  【ConcurrentHashMap으로 경합 완화】                                   │
│  ──────────────────                                                  │
│  // Before: 단일 락                                                   │
│  Map<K, V> map = Collections.synchronizedMap(new HashMap<>());     │
│                                                                     │
│  // After: 스트라이핑 락                                                │
│  Map<K, V> map = new ConcurrentHashMap<>();                        │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  전략              4 스레드    16 스레드   64 스레드                 │ │
│  │  ────              ──────        ───────     ──────                 │ │
│  │  단일 락            1000 ops    400 ops    100 ops                  │ │
│  │  RW 락             1000 ops    800 ops    300 ops                  │ │
│  │  스트라이핑           1000 ops    900 ops    700 ops                  │ │
│  │  락 프리             1000 ops    950 ops    850 ops                  │ │
│  │                                                             │ │
│  │  높은 스레드 수에서 락 경합 완화 효과 큼                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【락 경합 프로파일링 (Linux)】                                        │
│  ──────────────────                                                  │
│  # ftrace로 락 대기 추적                                               │
│  echo 1 > /sys/kernel/debug/tracing/events/lock/lock_acquire/enable │
│  echo 1 > /sys/kernel/debug/tracing/events/lock/lock_contended/enable │
│  cat /sys/kernel/debug/tracing/trace                                │
│                                                                     │
│  # eBPF로 상세 분석                                                    │
│  bpftrace -e 'kprobe:mutex_lock { @start[tid] = nsecs; }           │
│              kretprobe:mutex_lock /@start[tid]/ {                    │
│                  @wait[comm] = hist(nsecs - @start[tid]);            │
│                  delete(@start[tid]);                                │
│              }'                                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
