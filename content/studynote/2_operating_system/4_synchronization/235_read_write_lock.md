# 235. 읽기-쓰기 락 (Read-Write Lock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 읽기는 병렬, 쓰기는 배타적
> 2. **가치**: 읽기 많은 워크로드에서 높은 동시성
> 3. **융합**: 뮤텍스, 공정성, 기아와 연관

---

## Ⅰ. 개요

### 개념 정의

읽기-쓰기 락(Read-Write Lock, rwlock)은 **여러 읽기 스레드는 동시에 접근할 수 있지만, 쓰기 스레드는 배타적으로 접근하는 락**이다. 읽기 많은 시나리오에서 높은 동시성을 제공한다.

### 💡 비유: 도서관
읽기-쓰기 락은 **도서관**과 같다. 여러 사람이 동시에 책을 읽을 수 있다(읽기). 하지만 책을 고치려면 다른 사람이 다 나가야 한다(쓰기).

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                읽기-쓰기 락 상세                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【구현 방식】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct rwlock {                                                 │
│  │      int readers;        // 현재 읽기 중인 스레드 수              │ │
│  │      bool writer;        // 쓰기 중인지                           │ │
│  │      mutex lock;         // 내부 상태 보호                        │ │
│  │      condition_variable read_cond;                              │ │
│  │      condition_variable write_cond;                             │ │
│  │  };                                                             │ │
│  │                                                             │ │
│  │  read_lock():                                                    │ │
│  │      mutex_lock(&lock);                                          │ │
│  │      while (writer || write_waiters > 0)  // 쓰기 우선 정책       │ │
│  │          cond_wait(&read_cond, &lock);                           │ │
│  │      readers++;                                                  │ │
│  │      mutex_unlock(&lock);                                        │ │
│  │                                                             │ │
│  │  read_unlock():                                                  │ │
│  │      mutex_lock(&lock);                                          │ │
│  │      readers--;                                                  │ │
│  │      if (readers == 0)                                           │ │
│  │          cond_signal(&write_cond);                               │ │
│  │      mutex_unlock(&lock);                                        │ │
│  │                                                             │ │
│  │  write_lock():                                                   │ │
│  │      mutex_lock(&lock);                                          │ │
│  │      write_waiters++;                                            │ │
│  │      while (readers > 0 || writer)                               │ │
│  │          cond_wait(&write_cond, &lock);                          │ │
│  │      write_waiters--;                                            │ │
│  │      writer = true;                                              │ │
│  │      mutex_unlock(&lock);                                        │ │
│  │                                                             │ │
│  │  write_unlock():                                                 │ │
│  │      mutex_lock(&lock);                                          │ │
│  │      writer = false;                                             │ │
│  │      cond_broadcast(&read_cond);  // 또는 signal                 │ │
│  │      cond_signal(&write_cond);                                   │ │
│  │      mutex_unlock(&lock);                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【적합한 상황】                                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 캐시: 읽기 많고 쓰기 적음                                       │ │
│  │  • 데이터베이스: 조회 많고 수정 적음                               │ │
│  │  • 설정: 읽기 많고 변경 드묾                                       │ │
│  │  • 파일 시스템: 읽기 많고 쓰기 적음                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【POSIX rwlock】                                                     │
│  ──────────────────                                                  │
│  #include <pthread.h>                                                │
│                                                                     │
│  pthread_rwlock_t rwlock = PTHREAD_RWLOCK_INITIALIZER;              │
│                                                                     │
│  // 읽기                                                             │
│  void read_data() {                                                 │
│      pthread_rwlock_rdlock(&rwlock);                                │
│      // 데이터 읽기                                                  │
│      pthread_rwlock_unlock(&rwlock);                                │
│  }                                                                  │
│                                                                     │
│  // 쓰기                                                             │
│  void write_data() {                                                │
│      pthread_rwlock_wrlock(&rwlock);                                │
│      // 데이터 쓰기                                                  │
│      pthread_rwlock_unlock(&rwlock);                                │
│  }                                                                  │
│                                                                     │
│  【C++17 shared_mutex】                                               │
│  ──────────────────                                                  │
│  #include <shared_mutex>                                             │
│                                                                     │
│  class ThreadSafeCache {                                            │
│      mutable std::shared_mutex mutex;                               │
│      std::map<int, std::string> cache;                              │
│  public:                                                            │
│      std::string get(int key) const {                               │
│          std::shared_lock<std::shared_mutex> lock(mutex);           │
│          auto it = cache.find(key);                                 │
│          return it != cache.end() ? it->second : "";                │
│      }                                                              │
│      void set(int key, const std::string& value) {                  │
│          std::unique_lock<std::shared_mutex> lock(mutex);           │
│          cache[key] = value;                                        │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【Java ReentrantReadWriteLock】                                      │
│  ──────────────────                                                  │
│  import java.util.concurrent.locks.*;                               │
│                                                                     │
│  ReadWriteLock rwlock = new ReentrantReadWriteLock();               │
│                                                                     │
│  void readData() {                                                  │
│      rwlock.readLock().lock();                                      │
│      try {                                                          │
│          // 데이터 읽기                                               │
│      } finally {                                                    │
│          rwlock.readLock().unlock();                                │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  void writeData() {                                                 │
│      rwlock.writeLock().lock();                                     │
│      try {                                                          │
│          // 데이터 쓰기                                               │
│      } finally {                                                    │
│          rwlock.writeLock().unlock();                               │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【성능 고려사항】                                                     │
│  ──────────────────                                                  │
│  • 읽기가 쓰기보다 현저히 많을 때만 유리                               │
│  • 읽기/쓰기 비슷하면 일반 뮤텍스가 더 나을 수 있음                     │
│  • 공정성 정책 선택 중요                                              │
│  • 쓰기 기아 방지 위해 타임아웃 사용 고려                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 읽기 병렬, 쓰기 배타적 락
• 규칙: 여러 읽기 가능, 쓰기는 단독
• 공정성: 읽기 우선, 쓰기 우선, FIFO
• 적합: 읽기 많고 쓰기 적은 경우
• 구현: pthread_rwlock, shared_mutex
• 주의: 쓰기 기아 가능성
```
