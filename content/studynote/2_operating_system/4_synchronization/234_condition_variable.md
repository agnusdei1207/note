# 234. 조건 변수 (Condition Variable)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 조건이 만족될 때까지 대기하는 동기화 객체
> 2. **가치**: 효율적인 이벤트 대기, 생산자-소비자 패턴
> 3. **융합**: 뮤텍스, 모니터, 신호와 연관

---

## Ⅰ. 개요

### 개념 정의

조건 변수(Condition Variable)는 **특정 조건이 만족될 때까지 스레드를 대기시키고, 다른 스레드가 조건을 만족시키면 깨우는 동기화 객체**다. 뮤텍스와 함께 사용해야 한다.

### 💡 비유: 식당 호출기
조건 변수는 **식당 대기 호출기**와 같다. 자리가 나면 호출기가 울린다. 그 전까지는 다른 일을 하거나 편하게 대기한다.

### 조건 변수 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                조건 변수 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 연산】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  wait(mutex): 조건 대기 (락 해제 + 대기 + 깨어날 때 락 재획득)    │ │
│  │  signal(): 하나의 대기 스레드 깨움                               │ │
│  │  broadcast(): 모든 대기 스레드 깨움                              │ │
│  │                                                             │ │
│  │  스레드 A (대기)               스레드 B (깨우기)                  │ │
│  │  ──────────────                ──────────────                  │ │
│  │                                                             │ │
│  │  lock(mutex)                                                 │
│  │  while (!condition)                                          │
│  │      wait(cv, mutex) ────────────────────┐                   │ │
│  │      (락 해제, 대기)                     │                    │ │
│  │      (깨어남, 락 재획득)◄─────────────────┤                   │ │
│  │                                          │                    │ │
│  │                                          signal(cv)           │ │
│  │                                          (또는 broadcast)      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【wait() 동작 상세】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. mutex 락 해제                                                │ │
│  │  2. 조건 변수 대기 큐에 추가                                      │ │
│  │  3. 스레드 슬립 (CPU 반납)                                        │ │
│  │  4. signal/broadcast 시 깨어남                                   │ │
│  │  5. mutex 락 재획득 (다른 스레드가 쓰면 대기)                     │ │
│  │  6. wait() 반환                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Spurious Wakeup】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  이유: signal 없이도 깨어날 수 있음 (구현상 최적화)                │ │
│  │  해결: 항상 while 루프에서 조건 재확인                             │ │
│  │                                                             │ │
│  │  // 잘못된 방식                                                  │ │
│  │  if (!condition)                                                │ │
│  │      wait(cv, mutex);  // 스퓨리어스 웨이크업 가능               │ │
│  │  // condition이 여전히 false일 수 있음                           │ │
│  │                                                             │ │
│  │  // 올바른 방식                                                  │ │
│  │  while (!condition)                                             │ │
│  │      wait(cv, mutex);  // 깨어나도 다시 확인                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                조건 변수 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【signal vs broadcast】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  signal():                                                        │ │
│  │  • 하나의 대기 스레드만 깨움                                      │ │
│  │  • 대부분의 경우 충분                                              │ │
│  │  • 효율적                                                          │ │
│  │                                                             │ │
│  │  broadcast():                                                     │ │
│  │  • 모든 대기 스레드 깨움                                          │ │
│  │  • 여러 조건이 있을 때 필요                                        │ │
│  │  • 우선순위 스케줄링 시 필요할 수 있음                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【사용 패턴】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 대기 측                                                        │ │
│  │  pthread_mutex_lock(&mutex);                                     │
│  │  while (condition_is_false) {                                    │ │
│  │      pthread_cond_wait(&cond, &mutex);                           │
│  │  }                                                               │
│  │  // 조건이 참일 때 수행                                            │ │
│  │  pthread_mutex_unlock(&mutex);                                   │
│  │                                                             │ │
│  │  // 신호 측                                                        │ │
│  │  pthread_mutex_lock(&mutex);                                     │
│  │  // 조건을 참으로 만듦                                             │ │
│  │  pthread_cond_signal(&cond);  // 또는 broadcast                  │
│  │  pthread_mutex_unlock(&mutex);                                   │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주의사항】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 반드시 뮤텍스와 함께 사용                                       │ │
│  │  • while 루프로 조건 확인                                         │ │
│  │  • signal 전에 조건 변경                                          │ │
│  │  • 잊지 말고 뮤텍스 unlock                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【POSIX 조건 변수】                                                  │
│  ──────────────────                                                  │
│  #include <pthread.h>                                                │
│                                                                     │
│  pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;                 │
│  pthread_cond_t cond = PTHREAD_COND_INITIALIZER;                    │
│  int ready = 0;                                                     │
│                                                                     │
│  // 대기 스레드                                                       │
│  void* waiter(void* arg) {                                          │
│      pthread_mutex_lock(&mutex);                                    │
│      while (!ready) {                                               │
│          pthread_cond_wait(&cond, &mutex);                          │
│      }                                                              │
│      // ready == true일 때 수행                                      │
│      pthread_mutex_unlock(&mutex);                                  │
│      return NULL;                                                   │
│  }                                                                  │
│                                                                     │
│  // 신호 스레드                                                       │
│  void signaler() {                                                  │
│      pthread_mutex_lock(&mutex);                                    │
│      ready = 1;                                                     │
│      pthread_cond_signal(&cond);                                    │
│      pthread_mutex_unlock(&mutex);                                  │
│  }                                                                  │
│                                                                     │
│  【C++ std::condition_variable】                                      │
│  ──────────────────                                                  │
│  #include <condition_variable>                                       │
│  #include <mutex>                                                    │
│                                                                     │
│  std::mutex mtx;                                                    │
│  std::condition_variable cv;                                        │
│  bool ready = false;                                                │
│                                                                     │
│  void waiter() {                                                    │
│      std::unique_lock<std::mutex> lock(mtx);                        │
│      cv.wait(lock, []{ return ready; });  // 조건 포함               │
│      // ready == true일 때 수행                                      │
│  }                                                                  │
│                                                                     │
│  void signaler() {                                                  │
│      std::lock_guard<std::mutex> lock(mtx);                         │
│      ready = true;                                                  │
│      cv.notify_one();  // 또는 notify_all()                         │
│  }                                                                  │
│                                                                     │
│  【타임아웃 대기】                                                     │
│  ──────────────────                                                  │
│  std::unique_lock<std::mutex> lock(mtx);                            │
│  if (cv.wait_for(lock, std::chrono::seconds(5), []{ return ready; })) {
│      // 조건 만족                                                    │
│  } else {                                                           │
│      // 타임아웃                                                     │
│  }                                                                  │
│                                                                     │
│  【Java Object.wait/notify】                                          │
│  ──────────────────                                                  │
│  synchronized (lock) {                                              │
│      while (!ready) {                                               │
│          lock.wait();                                               │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  synchronized (lock) {                                              │
│      ready = true;                                                  │
│      lock.notify();  // 또는 notifyAll()                            │
│  }                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
