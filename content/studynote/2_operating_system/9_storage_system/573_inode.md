# 573. inode (Index Node)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 메타데이터를 저장하는 자료구조
> 2. **가치**: 파일 식별, 블록 포인터, 권한 관리
> 3. **융합**: 파일 시스템, 하드 링크, 블록 할당과 연관

---

## Ⅰ. 개요

### 개념 정의
**inode(Index Node)**는 **파일의 메타데이터와 데이터 블록 위치를 저장하는 자료구조**입니다.

### 💡 비유: 도서관의 도서 카드
inode는 **도서관의 도서 카드**와 같습니다. 책의 위치와 정보를 담고 있습니다.

│  │  │  atime              (접근 시간)                        │    │ │   │

│  │  │  블록 크기           (I/O 블록 크기)                     │    │ │   │

│  │  └─────────────────────────────────────────────────────┘    │ │   │

│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【inode 포인터 구조】                                                │ |

│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │

│  │    ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼            │ │   │
│  │  [데이터 블록들]                                                  │ │   │
│  │                                                             │ │   │

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                inode 상세                                           │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【inode 번호】                                                        │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  • 파일시스템 내에서 고유한 번호                                 │ │   │
│  │  • 파일 식별의 실제 키                                          │ │   │
│  │  • 파일 이름은 디렉토리 엔트리일 뿐                               │ │   │
│  │                                                             │ │   │
│  │  하드 링크:                                                     │ │   │
│  │  • 같은 inode 번호를 가진 여러 파일명                              │ │   │
│  │  • 파일 이동/이름 변경은 inode 번호 유지                           │ │   │
│  │  • 링크 카운트로 참조 수 관리                                      │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【타임스탬프】                                                        │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  타입              변경 시점                  명령어               │ │   │
│  │  ────              ────────                  ──────               │ │   │
│  │  atime             파일 읽기                  cat, less           │ │   │
│  │  mtime             내용 수정                  echo >, nano        │ │   │
│  │  ctime             속성 변경                  chmod, chown        │ │   │
│  │                                                             │ │   │
│  │  주의: mtime 변경 시 ctime도 함께 변경됨                           │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【inode와 파일 삭제】                                                │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  파일 삭제 과정:                                                  │ │   │
│  │  1. 디렉토리에서 파일명 제거                                       │ │   │
│  │  2. 링크 카운트 감소                                              │ │   │
│  │  3. 링크 카운트 = 0이면:                                          │ │   │
│  │     - inode 해제                                                 │ │   │
│  │     - 데이터 블록 해제                                             │ │   │
│  │                                                             │ │   │
│  │  복구: 삭제 직후 inode/블록 덮어쓰기 전까지만 가능                   │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【inode 정보 확인】                                                  │
│  ──────────────────                                                │
│  // inode 번호 확인                                                    │
│  $ ls -i file.txt                                                   │
│  1234567 file.txt                                                   │
│                                                                     │
│  // 상세 정보                                                         │
│  $ stat file.txt                                                    │
│  File: file.txt                                                     │
│  Size: 1024          Blocks: 8          IO Block: 4096   regular file│
│  Inode: 1234567       Links: 1                                      │
│  Access: (0644/-rw-r--r--)  Uid: (1000/user)  Gid: (1000/group)   │
│  Access: 2024-01-15 12:00:00.000000000 +0900                       │
│  Modify: 2024-01-15 12:00:00.000000000 +0900                       │
│  Change: 2024-01-15 12:00:00.000000000 +0900                       │
│                                                                     │
│  // inode 포인터 정보 (debugfs)                                       │
│  $ sudo debugfs /dev/sda1 -R "stat <1234567>"                      │
│                                                                     │
│  【inode 사용량 확인】                                                 │ |
│  ──────────────────                                                │
│  // 파일시스템 inode 정보                                               │
│  $ df -i                                                            │
│  Filesystem      Inodes   IUsed    IFree IUse% Mounted on           │
│  /dev/sda1      6553600  245760  6307840    4% /                    │
│                                                                     │
│  // inode 사용량이 많은 디렉토리                                        │
│  $ find /path -xdev -printf '%h\n' | sort | uniq -c | sort -k1 -n | tail│
│                                                                     │
│  // 특정 디렉토리 inode 사용량                                          │
│  $ ls -la /path | wc -l                                             │
│                                                                     │
│  【하드 링크】                                                          │ |
│  ──────────────────                                                │
│  // 하드 링크 생성                                                    │
│  $ ln original.txt hardlink.txt                                     │
│                                                                     │
│  // inode 확인 (동일함)                                               │
│  $ ls -i original.txt hardlink.txt                                  │
│  1234567 hardlink.txt                                               │
│  1234567 original.txt                                               │
│                                                                     │
│  // 링크 카운트 확인                                                   │
│  $ stat original.txt | grep Links                                   │
│  Links: 2                                                           │
│                                                                     │
│  【심볼릭 링크 (다른 inode)】                                          │ |
│  ──────────────────                                                │
│  // 심볼릭 링크 생성                                                  │
│  $ ln -s original.txt symlink.txt                                   │
│                                                                     │
│  // inode 확인 (다름)                                                 │
│  $ ls -i original.txt symlink.txt                                   │
│  1234567 original.txt                                               │
│  1234568 symlink.txt                                                │
│                                                                     │
│  【inode 문제 해결】                                                   │ |
│  ──────────────────                                                │
│  // inode 고갈 시                                                     │
│  $ df -i                       // inode 사용량 확인                   │
│  $ find /path -type f -delete  // 작은 파일 삭제                      │
│                                                                     │
│  // 파일시스템 재생성 (inode 수 늘림)                                   │
│  $ mkfs.ext4 -N 1000000 /dev/sda1  // inode 수 지정                  │
│                                                                     │
│  // 디렉토리 너무 깊을 때                                               │
│  $ find /path -type d | wc -l    // 디렉토리 수 확인                   │
│                                                                     │
│  【Python inode 접근】                                                 │ |
│  ──────────────────                                                │
│  import os                                                           │
│                                                                     │
│  # inode 번호                                                         │
│  inode = os.stat('file.txt').st_ino                                 │
│  print(f"Inode: {inode}")                                           │
│                                                                     │
│  # 타임스탬프                                                        │
│  stat = os.stat('file.txt')                                         │
│  print(f"atime: {stat.st_atime}")                                   │
│  print(f"mtime: {stat.st_mtime}")                                   │
│  print(f"ctime: {stat.st_ctime}")                                   │
│                                                                     │
│  # 링크 생성                                                         │
│  os.link('original.txt', 'hardlink.txt')  # 하드 링크                 │
│  os.symlink('original.txt', 'symlink.txt')  # 심볼릭 링크             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 파일 메타데이터와 블록 포인터 저장
• 구성: 모드, 링크수, UID/GID, 크기, 타임스탬프, 포인터
• 포인터: 직접 12개 + 간접 + 이중간접 + 삼중간접
• 특징: 파일 이름 없음 (디렉토리가 관리)
• 타임스탬프: atime, mtime, ctime
• 하드 링크: 같은 inode 번호 공유
• 링크 카운트: 0이면 파일 삭제
• inode 고갈: 디스크 공간 있어도 파일 생성 불가
• 확인: stat, ls -i, df -i
• ext4: inode 크기 256 bytes
```
