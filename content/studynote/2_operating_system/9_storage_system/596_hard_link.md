# 596. 하드 링크 (Hard Link)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 같은 inode를 가리키는 또 다른 파일명
> 2. **가치**: 공간 절약, 데이터 동기화
> 3. **융합**: inode, 링크 카운트, 파일 삭제와 연관

---

## Ⅰ. 개요

### 개념 정의
**하드 링크(Hard Link)**는 **기존 파일과 같은 inode를 가리키는 또 다른 파일명**입니다.

### 💡 비유: 같은 책의 또 다른 제목
하드 링크는 **같은 책의 또 다른 제목**과 같습니다. 내용은 같고 이름만 다릅니다.

### 하드 링크 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                하드 링크 구조                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하드 링크 개념】                                                     │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  디렉토리 엔트리                 inode                      │ │   │
│  │  ┌─────────────────┐         ┌─────────────────┐           │ │   │
│  │  │ original.txt    │─────────▶│ inode #12345    │           │ │   │
│  │  │ hardlink.txt    │─────────▶│ • 크기: 1KB      │           │ │   │
│  │  │                 │         │ • 권한: 644      │           │ │   │
│  │  │                 │         │ • 링크수: 2      │           │ │   │
│  │  │                 │         │ • 데이터 포인터   │           │ │   │
│  │  └─────────────────┘         └────────┬────────┘           │ │   │
│  │                                        │                       │ │   │
│  │                                        ▼                       │ │   │
│  │                              ┌─────────────────┐              │ │   │
│  │                              │   데이터 블록    │              │ │   │
│  │                              │   (내용 동일)    │              │ │   │
│  │                              └─────────────────┘              │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【하드 링크 vs 심볼릭 링크】                                           │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  항목            하드 링크           심볼릭 링크               │ │   │
│  │  ────            ─────────           ─────────               │ │   │
│  │  inode           동일함              다름                     │ │   │
│  │  데이터          공유                가리킴                    │ │   │
│  │  원본 삭제       영향 없음            끊김 (dangling)           │ │   │
│  │  파일시스템       같은 파일시스템만     다른 파일시스템 가능       │ │   │
│  │  디렉토리        불가능               가능                      │ │   │
│  │  크기            원본과 같음          경로 길이                  │ │   │
│  │  링크 카운트      증가                 변화 없음                  │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【링크 카운트】                                                        │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  링크 카운트 = 해당 inode를 가리키는 디렉토리 엔트리 수             │ │   │
│  │                                                             │ │   │
│  │  파일 생성: 링크 카운트 = 1                                       │ │   │
│  │  하드 링크 생성: 링크 카운트++                                     │ │   │
│  │  하드 링크 삭제: 링크 카운트--                                     │ │   │
│  │  링크 카운트 = 0이면: inode와 데이터 블록 해제                      │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                하드 링크 상세                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하드 링크 특성】                                                    │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  장점:                                                           │ │   │
│  │  • 디스크 공간 절약 (데이터 복사 없음)                              │ │   │
│  │  • 원본과 동일한 데이터 (동기화 불필요)                              │ │   │
│  │  • 원본 삭제해도 데이터 유지                                        │ │   │
│  │  • 성능 오버헤드 없음                                              │ │   │
│  │                                                             │ │   │
│  │  제한:                                                           │ │   │
│  │  • 같은 파일시스템 내에서만 가능                                    │ │   │
│  │  • 디렉토리에는 하드 링크 불가 (root만 가능)                         │ │   │
│  │  • 파일시스템이 inode 기반이어야 함                                 │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【파일 삭제 과정】                                                    │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  rm file.txt:                                                     │ │   │
│  │  1. 디렉토리에서 파일명 제거                                        │ │   │
│  │  2. inode 링크 카운트 감소                                         │ │   │
│  │  3. 링크 카운트 > 0: inode/데이터 유지                              │ │   │
│  │  4. 링크 카운트 = 0: inode 해제, 데이터 블록 해제                    │ │   │
│  │                                                             │ │   │
│  │  ※ unlink() 시스템 콜이 실제 수행                                  │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【사용 사례】                                                         │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  • 백업: rsync --link-dest (증분 백업)                            │ │   │
│  │  • 버전 관리: 파일 변경 없으면 하드 링크                            │ │   │
│  │  • 공유 데이터: 여러 위치에서 동일 파일 접근                         │ │   │
│  │  • chroot 환경: 필수 파일 복사 대신 링크                           │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하드 링크 생성】                                                    │ |
│  ──────────────────                                                │
│  // 하드 링크 생성                                                      │
│  $ ln original.txt hardlink.txt                                     │
│                                                                     │
│  // 확인                                                             │
│  $ ls -li original.txt hardlink.txt                                 │
│  1234567 -rw-r--r-- 2 user user 100 Jan 15 original.txt            │
│  1234567 -rw-r--r-- 2 user user 100 Jan 15 hardlink.txt            │
│  //      ↑ 동일한 inode 번호    ↑ 링크 카운트 = 2                       │
│                                                                     │
│  // stat으로 확인                                                      │
│  $ stat original.txt                                               │
│  Inode: 1234567  Links: 2                                          │
│                                                                     │
│  // 파일 내용 확인 (동일)                                               │
│  $ diff original.txt hardlink.txt                                  │
│  // 출력 없음 = 동일                                                   │
│                                                                     │
│  【하드 링크 삭제】                                                    │ |
│  ──────────────────                                                │
│  // 원본 삭제                                                          │
│  $ rm original.txt                                                  │
│                                                                     │
│  // 하드 링크로 데이터 여전히 접근 가능                                  │
│  $ cat hardlink.txt                                                 │
│                                                                     │
│  // 링크 카운트 감소                                                    │
│  $ stat hardlink.txt                                               │
│  Links: 1                                                          │
│                                                                     │
│  // 하드 링크도 삭제하면 데이터 삭제                                      │
│  $ rm hardlink.txt                                                  │
│                                                                     │
│  【파일시스템 제한 확인】                                               │ |
│  ──────────────────                                                │
│  // 다른 파일시스템에 하드 링크 시도                                      │
│  $ ln /home/user/file.txt /mnt/external/hardlink.txt               │
│  ln: failed to create hard link ... Invalid cross-device link      │
│                                                                     │
│  // 심볼릭 링크는 가능                                                   │
│  $ ln -s /home/user/file.txt /mnt/external/symlink.txt             │
│                                                                     │
│  // 파일시스템 확인                                                      │
│  $ df -T /home/user /mnt/external                                  │
│                                                                     │
│  【증분 백업 (rsync)】                                                 │ |
│  ──────────────────                                                │
│  // rsync --link-dest로 하드 링크 기반 증분 백업                        │
│  $ rsync -avz --link-dest=/backup/current /data/ /backup/$(date +%F)│
│                                                                     │
│  // 구조:                                                            │
│  // /backup/2024-01-15/file.txt (새로운 데이터)                        │
│  // /backup/2024-01-16/file.txt (변경 없으면 하드 링크)                 │
│  // /backup/2024-01-17/file.txt (변경 없으면 하드 링크)                 │
│                                                                     │
│  // 공간 절약 확인                                                      │
│  $ du -sh /backup/*                                                 │
│  $ du -shl /backup/*    // 링크 고려                                  │
│                                                                     │
│  【Python으로 하드 링크】                                              │ |
│  ──────────────────                                                │
│  import os                                                           │
│                                                                     │
│  # 하드 링크 생성                                                      │
│  os.link('original.txt', 'hardlink.txt')                            │
│                                                                     │
│  # inode 확인                                                         │
│  orig_stat = os.stat('original.txt')                                │
│  link_stat = os.stat('hardlink.txt')                                │
│  print(f"Same inode: {orig_stat.st_ino == link_stat.st_ino}")       │
│  print(f"Link count: {orig_stat.st_nlink}")                         │
│                                                                     │
│  # 링크 삭제 (unlink)                                                 │
│  os.unlink('hardlink.txt')                                          │
│                                                                     │
│  【C 언어로 하드 링크】                                                 │ |
│  ──────────────────                                                │
│  #include <unistd.h>                                                 │
│  #include <stdio.h>                                                  │
│                                                                     │
│  int main() {                                                        │
│      if (link("original.txt", "hardlink.txt") == 0) {                │
│          printf("Hard link created\n");                              │
│      } else {                                                        │
│          perror("link");                                             │
│      }                                                               │
│                                                                     │
│      if (unlink("hardlink.txt") == 0) {                              │
│          printf("Link removed\n");                                   │
│      }                                                               │
│      return 0;                                                       │
│  }                                                                   │
│                                                                     │
│  【링크 카운트가 높은 파일 찾기】                                         │ |
│  ──────────────────                                                │
│  // 링크 카운트 > 2인 파일                                              │
│  $ find /path -type f -links +2                                    │
│                                                                     │
│  // 특정 inode를 가진 모든 하드 링크 찾기                                 │
│  $ find /path -inum 1234567                                        │
│                                                                     │
│  // 같은 inode를 가진 파일 확인                                          │
│  $ ls -li /path | awk '$1 == 1234567'                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 같은 inode를 가리키는 또 다른 파일명
• 생성: ln 원본 하드링크
• inode: 원본과 동일
• 링크 카운트: 하드 링크 수 + 1
• 데이터: 공유 (복사 없음)
• 삭제: 링크 카운트 감소, 0이면 데이터 삭제
• 제한: 같은 파일시스템만, 디렉토리 불가
• 심볼릭 링크와 차이: inode 동일, 원본 삭제 시 유지
• 확인: ls -li, stat
• 백업: rsync --link-dest
• Python: os.link()
• C: link(), unlink()
```
