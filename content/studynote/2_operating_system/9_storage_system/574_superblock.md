# 574. 슈퍼블록 (Superblock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일시스템 전체 메타데이터를 저장하는 자료구조
> 2. **가치**: 파일시스템 상태, 크기, 사용량 관리
> 3. **융합**: 파일 시스템, 마운트, fsck와 연관

---

## Ⅰ. 개요

### 개념 정의
**슈퍼블록(Superblock)**은 **파일시스템의 전체 정보와 상태를 저장하는 핵심 자료구조**입니다.

### 💡 비유: 도서관 안내판
슈퍼블록은 **도서관 안내판**과 같습니다. 전체 구조와 사용 현황을 보여줍니다.

### 슈퍼블록 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                슈퍼블록 구조                                         │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【슈퍼블록 위치】                                                     │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  [부트 블록] [슈퍼블록] [블록 그룹 0] [블록 그룹 1] ...          │ │   │
│  │     1KB       1KB                                              │ │   │
│  │                                                             │ │   │
│  │  • 블록 그룹마다 슈퍼블록 백업 존재                              │ │   │
│  │  • 손상 시 백업에서 복구 가능                                    │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【슈퍼블록 구성 요소】                                                │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  필드                    설명                                    │ │   │
│  │  ─────                   ────                                    │ │   │
│  │  s_inodes_count         전체 inode 수                            │ │   │
│  │  s_blocks_count         전체 블록 수                              │ │   │
│  │  s_r_blocks_count       예약 블록 수 (root용)                      │ │   │
│  │  s_free_blocks_count    사용 가능한 블록 수                        │ │   │
│  │  s_free_inodes_count    사용 가능한 inode 수                       │ │   │
│  │  s_first_data_block     첫 데이터 블록 번호                        │ │   │
│  │  s_log_block_size       블록 크기 (log2)                          │ │   │
│  │  s_blocks_per_group     그룹당 블록 수                            │ │   │
│  │  s_inodes_per_group     그룹당 inode 수                           │ │   │
│  │  s_mtime                마운트 시간                               │ │   │
│  │  s_wtime                쓰기 시간                                 │ │   │
│  │  s_mnt_count            마운트 횟수                               │ │   │
│  │  s_state                파일시스템 상태                            │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【ext4 블록 그룹 구조】                                               │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  ┌─────────────────────────────────────────────────────┐    │ │   │
│  │  │                  블록 그룹 0                          │    │ │   │
│  │  ├─────────────────────────────────────────────────────┤    │ │   │
│  │  │ [슈퍼블록][그룹 기술자][비트맵][inode 비트맵][inode 테이블][데이터]│ │   │
│  │  └─────────────────────────────────────────────────────┘    │ │   │
│  │                                                             │ │   │
│  │  그룹 기술자 (Group Descriptor):                                  │ │   │
│  │  • 블록 비트맵 위치                                              │ │   │
│  │  • inode 비트맵 위치                                             │ │   │
│  │  • inode 테이블 위치                                             │ │   │
│  │  • 사용 가능한 블록/inode 수                                      │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                슈퍼블록 상세                                         │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파일시스템 상태】                                                   │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  상태                  설명                                     │ │   │
│  │  ────                  ────                                     │ │   │
│  │  EXT4_VALID_FS         정상                                     │ │   │
│  │  EXT4_ERROR_FS         오류 발생                                 │ │   │
│  │  EXT4_ORPHAN_FS        고아 inode 존재                            │ │   │
│  │                                                             │ │   │
│  │  비정상 마운트 해제 시:                                           │ │   │
│  │  • dirty 플래그 설정                                             │ │   │
│  │  • 다음 마운트 시 fsck 필요                                       │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【예약 블록】                                                          │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  목적:                                                           │ │   │
│  │  • root가 로그인할 공간 확보                                      │ │   │
│  │  • 디스크가 꽉 차도 시스템 관리 가능                               │ │   │
│  │                                                             │ │   │
│  │  기본: 전체 블록의 5%                                              │ │   │
│  │                                                             │ │   │
│  │  // 예약 블록 비율 변경                                            │ │   │
│  │  $ tune2fs -m 1 /dev/sda1     // 1%로 변경                        │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【슈퍼블록 백업】                                                      │ |
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  백업 위치:                                                      │ │   │
│  │  • 그룹 0 (필수)                                                 │ │   │
│  │  • 그룹 1 (선택)                                                  │ │   │
│  │  • 그룹 3, 5, 7, 9, 25, 27, ... (sparse)                        │ │   │
│  │                                                             │ │   │
│  │  복구 방법:                                                      │ │   │
│  │  $ e2fsck -b 32768 /dev/sda1   // 백업 슈퍼블록 사용               │ │   │
│  │  $ e2fsck -b 98304 /dev/sda1                                       │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 파일시스템 전체 메타데이터 저장
• 위치: 부트 블록 다음, 그룹마다 백업
• 구성: 블록/inode 수, 사용량, 크기, 상태
• 상태: clean, error, orphan
• 예약 블록: 기본 5% (root용)
• 백업: 그룹 0, 1, 3, 5, 7, 9, ...
• 복구: e2fsck -b 백업위치
• 확인: tune2fs -l, dumpe2fs, df
• 변경: tune2fs (라벨, 예약, fsck 간격)
• ext4: 블록 그룹 구조
```
