# 보안 위협 유형

## 핵심 인사이트

**시스템 보안 위협은 정보 보안의 3대 요소인 기밀성(Confidentiality), 무결성(Integrity), 가용성(Availability)을 침해하는 행위들이다.** 운영체제 관점에서 이 위협들은 **트로이 목마, 백도어, 바이러스, 웜, 버퍼 오버플로우** 등 다양한 형태로 나타나며, 각각의 공격 벡터와 방어 기법이 존재한다. 이러한 위협을 이해하는 것은 시스템 보안 설계의 첫 단계다.

---

## Ⅰ. CIA 트라이어드

### 1. 정보 보안의 3대 요소

```
┌─────────────────────────────────────────────────────────────┐
│                  CIA 트라이어드 (Security Triad)             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                    ┌─────────────┐                          │
│                    │   기밀성    │                          │
│                    │Confidentiality│                         │
│                    │  승인된     │                          │
│                    │  사용자만   │                          │
│                    │  접근 가능   │                          │
│                    └──────┬──────┘                          │
│                           │                                 │
│          ┌────────────────┼────────────────┐                │
│          │                │                │                │
│    ┌─────▼─────┐    ┌─────▼─────┐    ┌─────▼─────┐         │
│    │  무결성    │    │  가용성    │    │   인증     │         │
│    │ Integrity │    │ Availability│   │Authentication│       │
│    │ 데이터     │    │ 필요할     │    │ 신원       │         │
│    │ 변경 불가  │    │ 때 사용    │    │ 확인       │         │
│    │           │    │ 가능       │    │            │         │
│    └───────────┘    └────────────┘    └────────────┘       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 위협 유형 분류

| 위협 | 침해 요소 | 예시 |
|:---|:---|:---|
| **스니핑** | 기밀성 | 패킷 캡처, 키로깅 |
| **변조** | 무결성 | 데이터 변경, 메시지 재생 |
| **DoS/DDoS** | 가용성 | 서비스 과부하, 리소스 고갈 |
| **스푸핑** | 인증 | IP 위장, 피싱 |
| **랜섬웨어** | 가용성+기밀성 | 파일 암호화 |

## Ⅱ. 주요 보안 위협 유형

### 1. 트로이 목마 (Trojan Horse)

```
┌─────────────────────────────────────────────────────────────┐
│                  트로이 목마 동작 원리                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  사용자가 "유용한 프로그램"으로 착각                         │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  게임/유틸리티 설치 파일                            │   │
│  │  ┌───────────────────────────────────────────────┐ │   │
│  │  │ 정상 기능 (게임 실행)                         │ │   │
│  │  └───────────────────────────────────────────────┘ │   │
│  │  ┌───────────────────────────────────────────────┐ │   │
│  │  │ 악성 기능 (비밀번호 탈취, 백도어 생성)        │ │   │
│  │  └───────────────────────────────────────────────┘ │   │
│  └─────────────────────────────────────────────────────┘   │
│                          │                                   │
│                          ▼                                   │
│  실행 시 정상 기능은 보이지만                               │
│  백그라운드에서 악성 코드 실행                              │
│                                                             │
│  예: free_game.exe                                         │
│    ├── 게임은 정상 작동 ✓                                  │
│    └── 동시에 C&C 서버에 정보 전송 ✗                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 트랩 도어 / 백도어 (Trap Door / Backdoor)

| 유형 | 설명 | 예시 |
|:---|:---|:---|
| **하드코딩된 비밀번호** | 개발자가 남긴 특수 비밀번호 | `/bin/passwd`의 debug 모드 |
| **숨겨진 포트** | 일반 스캔으로 발견 안 됨 | 31337 포트 |
| **신분 우회** | 인증 절차 건너뛰기 | `if (user == "root")` |
| **시스템 콜 후킹** | 커널 함수 조작 |

### 3. 로직 밤 (Logic Bomb / Timer Bomb)

```
┌─────────────────────────────────────────────────────────────┐
│              로직 밤 (Logic Bomb) 구조                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  일반 코드                                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                                                      │   │
│  │   if (current_date > "2025-01-01") {                │   │
│  │       delete_all_files();  // ★ 폭탄!              │   │
│   }                                                    │   │
│   │                                                    │   │
│   │   if (!license_key_valid()) {                      │   │
│   │       system_crash();      // ★ 폭탄!             │   │
│   │   }                                                 │   │
│   │                                                    │   │
│   │   if (programmer_name != payroll) {                │   │
│   │       skip_programmer_pay();  // ★ 폭탄!          │   │
│   │   }                                                 │   │
│   │                                                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  발동 조건: 날짜, 조건, 특정 이벤트                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4. 바이러스 (Virus) vs 웜 (Worm)

```
┌─────────────────────────────────────────────────────────────┐
│               바이러스 vs 웜 비교                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  바이러스 (Virus)                 웜 (Worm)                  │
│  ┌─────────────────┐             ┌─────────────────┐       │
│  │ 호스트 필요     │             │ 독립 실행       │       │
│  │                 │             │                 │       │
│  │ .exe, .doc      │             │ 자가 복제       │       │
│  │ 파일에 기생     │             │                 │       │
│  │                 │             │ 네트워크 전파   │       │
│  │ 사용자 개입     │             │                 │       │
│  │ 필요 (실행)     │             │ 자동 확산       │       │
│  └─────────────────┘             └─────────────────┘       │
│                                                             │
│  예:                              예:                       │
│  • Macro Virus (Word)            • Code Red (2001)         │
│  • Boot Sector Virus             • SQL Slammer (2003)      │
│  • File Infector                 • WannaCry (2017)         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅲ. 시스템 취약점 및 공격

### 1. 버퍼 오버플로우 (Buffer Overflow)

```
┌─────────────────────────────────────────────────────────────┐
│           버퍼 오버플로우 공격 원리                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  취약한 코드:                                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ void vulnerable() {                                │   │
│  │     char buffer[64];                               │   │
│  │     gets(buffer);  // ★ 경계 검사 없음!            │   │
│  │ }                                                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  정상 입력: "Hello"                                         │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ [H][e][l][l][o][\0][  ][  ]...[  ]                 │   │
│  │  ↑buffer                              ↑64바이트      │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  공격 입력: "A" × 100 + 쉸코드 주소                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ [A][A]...[A][A][RET_ADDR][  ]...[  ][쉸코드]       │   │
│  │  ↑buffer      ↑64바이트      ↑리턴 주소          │   │
│  │                              (덮어쓰기!)           │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  함수 복귀 시 덮어쓴 주소로 점프 → 쉸코드 실행!            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 스푸핑 (Spoofing)

| 유형 | 설명 | 방어 |
|:---|:---|:---|
| **IP 스푸핑** | 발신자 IP 위조 | SPF/DKIM, 필터링 |
| **MAC 스푸핑** | MAC 주소 위조 | 포트 보안 |
| **ARP 스푸핑** | ARP 캐시 포이즈닝 | 정적 ARP 테이블 |
| **DNS 스푸핑** | 가짜 DNS 응답 | DNSSEC |

### 3. DoS/DDoS 공격

```
┌─────────────────────────────────────────────────────────────┐
│              DoS vs DDoS 비교                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  DoS (Denial of Service)          DDoS (Distributed)        │
│  ┌─────────────────┐             ┌─────────────────┐       │
│  │ 단일 공격자     │             │ 다중 공격자     │       │
│  │                 │             │ (봇넷)          │       │
│  │ ┌───────────┐   │             │ ┌───┐ ┌───┐ ┌───┐│       │
│  │ │ 공격자    │   │             │ │봇1│ │봇2│ │봇3││       │
│  │ └─────┬─────┘   │             │ └─┬─┘ └─┬─┘ └─┬─┘│       │
│  │       │         │             │   │    │    │  │       │
│  │       ▼         │             │   └──┬─┴──┬─┘  │       │
│  │   ┌─────────┐  │             │      ▼    ▼    │       │
│  │   │ 타겟    │  │             │   ┌─────────┐   │       │
│  │   │ 서버    │  │             │   │ 타겟    │   │       │
│  │   │ 과부하  │  │             │   │ 서버    │   │       │
│  │   └─────────┘  │             │   │ 붕괴   │   │       │
│  └─────────────────┘             └─────────────────┘       │
│                                                             │
│  공격 기법:                                                │
│  • SYN Flood: 핸드셰이크 완료하지 않고 연속 요청           │
│  • UDP Flood: 무상태 UDP 프로토콜 악용                     │
│  • HTTP Flood: GET 요청 폭주                               │
│  • Slowloris: 헤더를 천천히 보내 연결 유지                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅳ. 방어 기법

### 1. 다계층 방어 (Defense in Depth)

```
┌─────────────────────────────────────────────────────────────┐
│            다계층 보안 방어 체계                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌───────────────────────────────────────────────────────┐ │
│  │                 정책 및 규정                          │ │
│  │  (보안 정책, 사용자 교육, 컴플라이언스)               │ │
│  └───────────────────────────────────────────────────────┘ │
│                           │                                 │
│  ┌───────────────────────────────────────────────────────┐ │
│  │              외부 경계 (Perimeter)                     │ │
│  │  (방화벽, IDS/IPS, DDoS 방지, WAF)                    │ │
│  └───────────────────────────────────────────────────────┘ │
│                           │                                 │
│  ┌───────────────────────────────────────────────────────┐ │
│  │            네트워크 내부 (Internal)                    │ │
│  │  (내부 방화벽, VLAN, 네트워크 세그먼트이션)            │ │
│  └───────────────────────────────────────────────────────┘ │
│                           │                                 │
│  ┌───────────────────────────────────────────────────────┐ │
│  │              호스트 기반 (Host-based)                  │ │
│  │  (SELinux/AppArmor, 백신, HIDS)                       │ │
│  └───────────────────────────────────────────────────────┘ │
│                           │                                 │
│  ┌───────────────────────────────────────────────────────┐ │
│  │            애플리케이션 (Application)                 │ │
│  │  (입력 검증, SQL 인젝션 방지, CSRF 토큰)             │ │
│  └───────────────────────────────────────────────────────┘ │
│                           │                                 │
│  ┌───────────────────────────────────────────────────────┐ │
│  │                 데이터 (Data)                         │ │
│  │  (암호화, 백업, 무결성 검증)                         │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. OS 레벨 방어

| 기법 | 설명 |
|:---|:---|
| **ASLR** | 주소 공간 무작위화 |
| **DEP/NX** | 데이터 실행 방지 |
| **Stack Canary** | 스택 보호 변수 |
| **SELinux/AppArmor** | 강제적 접근 제어 |
| **Capabilities** | 세분화된 권한 |

## 어린이를 위한 비유

**보안 위협은 "집의 보안"**과 같습니다.

**CIA 트라이어드**는 집의 3가지 핵심 요소입니다:
- **기밀성**: "내 방은 **나만 들어갈 수 있어**"
- **무결성**: "내 물건은 **아무도 바꿀 수 없어**"
- **가용성**: "내 집은 **언제든 사용할 수 있어**"

**위협 유형**들은 침입하는 방법입니다:
- **트로이 목마**: "선물로 받은 인형 안에 **도둑이 숨어있어**"
- **백도어**: "문 잠금 장치를 **만든 사람이 몰래 키**를 가지고 있어"
- **바이러스**: "옷에 달라붙은 **씨앗**이 옷장 전체를 퍼뜨려"
- **웜**: "스스로 이동하면서 **집을 복제**하는 로봇"

**방어**는 이렇게 합니다:
- **ASLR**: "매번 집 주소를 **바꿔서** 도둑이 찾지 못하게"
- **Canary**: "함정 **카나리아**를 설치해 도둑이 밟으면 알람이 울려"
- **다계층 방어**: "울타리, 경비견, CCTV, **여러 겹의 보안**"

보안은 **한 번의 실수**로 모든 것을 잃을 수 있으므로, **여러 겹의 방어**가 필요합니다!
