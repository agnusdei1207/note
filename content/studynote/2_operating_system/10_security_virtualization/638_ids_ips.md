# IDS (Intrusion Detection System) / IPS (Intrusion Prevention System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 네트워크/시스템 트래픽을 실시간 모니터링하여 공격을 탐지(IDS)하고 차단(IPS)하는 보안 시스템
> 2. **가치**: 방화벽의 수동 룰벅 리스를 자동화하고 침투를 실시간으로 대응
> 3. **융합**: 시그니처 기반, 이상 탐지 기반, 정책 위반 기반 등 다양한 탐지 기법

---

### Ⅰ. 개요 (Context & Background)

**개념**
IDS(Intrusion Detection System)는 **네트워크나 시스템의 활동을 모니터링하여 비정상적이거나 악의적인 행위를 탐지**하는 시스템이다. IPS(Intrusion Prevention System)는 IDS의 탐지 기능에 **실시간 차단(In-line) 기능을 추가**하여 공격을 자동으로 막는다. NIDS(Network IDS), HIDS(Host IDS), NBA(Network Behavior Analysis) 등이 있다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         IDS vs IPS 비교                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   IDS (탐지만)                                                             │
│   ────────────────                                                          │
│   인터넷 ──▶ [IDS: 탐지] ──▶ 방화벽 ──▶ 내부 네트워크 ──▶ 서버        │
│                │                                                           │
│                │알림: 관리자에게                                      │
│                │                                                         │
│                ▼                                                        │
│              로그 분석                                                │
│              수동 차단                                                │
│                                                                             │
│   IPS (탐지+차단)                                                         │
│   ───────────────────                                                       │
│   인터넷 ──▶ [IPS: 탐지+차단] ──▶ 내부 네트워크 ──▶ 서버            │
│                │                                                           │
│                │즉시 차단                                            │
│                ▼                                                        │
│              정상 트래픽만 통과                                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**💡 비유**
IDS/IPS는 **보안 경비원(CCTV)과 자동 방벽창**와 같다.
- IDS: CCTV로 감시, 경비원에게 알림
- IPS: 자동으로 문을 닫고 침입자 차단
- 목표: 침입자 조기 발견, 차단

**등장 배경**
① **1980대**: 초기 IDS 개발
② **1990대**: 상용 IDS 등장 (Snort 등)
③ **2000년대**: IDS → IPS 발전
④ **현재**: AI 기반 탐지, 자동화 대응

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소**

| 구분 | 배치 방식 | 장점 | 단점 | 대표 도구 |
|:---|:---|:---|:---|:---|
| **NIDS** | 네트워크 경로 | 전체 트래픽 감시 | 암호화된 트래픽 분석 어려움 | Snort, Suricata |
| **HIDS** | 개별 호스트 | 정확한 정보 수집 | 설치/관리 복잡 | OSSEC, Wazuh |
| **NBA** | 네트워크 경로 | 미지 공격 탐지 | 높은 오탐지 | Lancope, Darktrace |
| **Hybrid** | 복합 | 종합적 보안 | 복잡함 | 다양 |

**해설**
1. **NIDS**: 네트워크 트래픽 실시간 분석

3. **NBA**: 네트워크 행태 기반 탐지

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 상용 IDS/IPS**

| 제품 | 유형 | 장점 | 단점 | 적합 환경 |
|:---|:---|:---|:---|:---|
| **Snort/Suricata** | NIDS | 오픈소스, 커뮤니티 | 설정 복잡 | 중소기업 |
| **OSSEC** | HIDS | 무료, 강력함 | 설치 복잡 | 온프레미스 |
| **Splunk** | SIEM + IDS | 분석 강력 | 비용 비쌈 | 대기업 |
| **Darktrace** | NBA | AI 기반, 미지 탐지 | 비용 비쌈 | 엔터프라이즈 |
| **Palo Alto** | NGFW | 통합 방화벽 | 비용 비쌈 | 중소기업 |

**과목 융합 관점**

1. **네트워크**: 프로토콜 분석, 트래픽 처리
2. **운영체제**: 시스템 콜, 파일 시스템
3. **보안**: 탐지, 대응, 포렌식

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나리오 1: IDS 오탐지 최소화**
- **튜닝**: 정상 트래픽 학습
- **Baseline**: 비지니스 시간대 파악
- **정책 구체화**: 명확한 규칙

**시나리오 2: IPS 배치**
- **투명 모드**: 로그만 수집
- **차단 모드**: 신뢰도 높은 규칙만 차단
- **검역 모드**: 샌드박스에서 테스트

**도입 체크리스트**

**기술적 측면**
- [ ] IDS/IPS 배치
- [ ] 룰벤 업데이트
- [ ] 로그 관리 시스템
- [ ] 모니터링 대시표
- [ ] 백업 및 복구

**운영·보안적 측면**
- [ ] 분석가 양성
- [ ] 오탐지 대응 절차
- [ ] 정기적 훈련
- [ ] 법적 준비

**안티패턴**
1. **"IDS면 돼"**: IPS 필요, 빠른 대응
2. **"오탐지 줄여"**: 관리자 피로 초래
3. **"장비만 하면 돼"**: 실제 대응 능력 필요
