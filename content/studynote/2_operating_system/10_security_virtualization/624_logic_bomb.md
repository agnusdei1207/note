# 로직 밤

## 핵심 인사이트

**로직 밤(Logic Bomb)은 특정 조건, 시간, 이벤트가 발생할 때만 악성 코드를 실행하도록 설계된 프로그램 내부의 악성 코드다.** 정상적인 상태에서는 아무것도 하지 않다가, **미리 정해진 트리거(Trigger)**가 발동하면 시스템을 파괴하거나 데이터를 삭제한다. 종종 불만을 품은 내부자(직원)가 회사를 보복하기 위해 심어 놓는 경우가 많다.

---

## Ⅰ. 정의 및 특징

### 1. 정의

| 요소 | 설명 |
|:---|:---|
| **로직 밤** | 특정 조건에서만 발동하는 악성 코드 |
| **타이머 밤** | 시간 기반으로 발동하는 로직 밤 |
| **트리거** | 폭탄을 터뜨리는 조건 (날짜, 파일, 이벤트) |
| **휴지 시간** | 심어진 시점부터 발동까지의 시간 |

### 2. 특징

```
┌─────────────────────────────────────────────────────────────┐
│               로직 밤의 특징                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                                                     │   │
│  │   정상 코드                                          │   │
│  │   ┌─────────────────────────────────────────────┐   │   │
│   │   │ 정상적인 비즈니스 로직                     │   │   │
│   │   │ (알기 쉽게, 투명하게)                      │   │   │
│   │   └─────────────────────────────────────────────┘   │   │
│   │                                                     │   │
│   │   숨겨진 코드                                       │   │
│   │   ┌─────────────────────────────────────────────┐   │   │
│   │   │ if (trigger_condition) {                    │   │   │
│   │   │     // ★ 폭탄!                              │   │   │
│   │   │     delete_all_data();                      │   │   │
│   │   │ }                                            │   │   │
│   │   └─────────────────────────────────────────────┘   │   │
│   │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  • 트리거 전: 탐지 어려움                                  │
│  • 트리거 후: 이미 늦음                                     │
│  • 코드 리뷰에서 발견 가능하지만 복잡한 코드에 숨기기 쉬움 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅱ. 트리거 유형

### 1. 트리거 분류

```
┌─────────────────────────────────────────────────────────────┐
│                  로직 밤 트리거 유형                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 시간 기반 (Time-based)                                  │
│     ┌───────────────────────────────────────────────────┐  │
│     │ if (time() > EXPIRY_DATE) {                       │  │
│     │     system("rm -rf /");                           │  │
│     │ }                                                  │  │
│     │                                                    │  │
│     │ 예: 계약 만료일, 해고일, 퇴직일                    │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  2. 논리 기반 (Logic-based)                                │
│     ┌───────────────────────────────────────────────────┐  │
│     │ if (!license_valid()) {                           │  │
│     │     corrupt_database();                           │  │
│     │ }                                                  │  │
│     │                                                    │  │
│     │ 예: 라이선스 만료, 특정 계정 부재                  │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  3. 이벤트 기반 (Event-based)                              │
│     ┌───────────────────────────────────────────────────┐  │
│     │ if (user_deleted_from_payroll()) {                │  │
│     │     send_confidential_data_to_press();            │  │
│     │ }                                                  │  │
│     │                                                    │  │
│     │ 예: 해고, 부서 이동, 권한 박탈                     │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
│  4. 파일 기반 (File-based)                                 │
│     ┌───────────────────────────────────────────────────┐  │
│     │ if (!exists("/tmp/keep_alive.flag")) {             │  │
│     │     shutdown_system();                             │  │
│     │ }                                                  │  │
│     │                                                    │  │
│     │ 예: 특정 파일 삭제 시                              │  │
│     └───────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 구체적 예시

| 트리거 | 실제 사례 |
|:---|:---|
| **날짜** | "2000년 1월 1일이 되면 시스템 다운" (Y2K 밤) |
| **해고** | 급여 시스템에서 내 이름이 사라지면 폭발 |
| **특정 사용자** | CEO가 로그인하지 않으면 작동 안 함 |
| **파일 존재** | `/tmp/I_AM_HERE` 파일이 없으면 파괴 |

## Ⅳ. 탐지 및 방어

### 1. 코드 리뷰 관점

```c
// ❌ 의심스러운 코드 (로직 밤 의심)
void process_payroll() {
    // ... 정상 급여 처리 ...

    // ★ 왜 이 코드가 여기 있지?
    if (current_date >= EXPIRY_DATE) {
        format_hard_drives();
    }
}

// ✅ 리팩토링된 코드
void process_payroll() {
    // ... 급여 처리만 담당 ...
}

// 별도의 라이선스 관리 모듈
void check_license() {
    if (license_expired()) {
        notify_admin();
        // 포맷 같은 파괴적 행위 금지!
    }
}
```

### 2. 탐지 방법

| 방법 | 설명 |
|:---|:---|
| **정적 분석** | 의심스러운 조건문 검색 (date, time, file check) |
| **동적 분석** | 샌드박스에서 시간 조작하며 테스트 |
| **코드 리뷰** | 퇴직/해고 직원의 코드 집중 검사 |
| **무결성 검사** | 주기적 체크섬 비교 |
| **행태 분석** | 비정상적인 시스템 셧다운 모니터링 |

### 3. 정적 분석 도구

```bash
# grep으로 의심스러운 패턴 검색
$ grep -r "time()" --include="*.c" | grep -E "(delete|remove|shutdown)"
$ grep -r "system(" --include="*.c" | grep -v "print\|log"

# cppcheck 정적 분석
$ cppcheck --enable=all source_code/

# SonarQube 코드 품질 분석
# 로직 밤 관련 룰 설정 가능
```

```c
// ✅ 안전한 날짜 검증
int check_license_expiry() {
    time_t now = time(NULL);
    struct tm *tm_now = localtime(&now);

    // 서버에서 확인 (조작 불가)
    if (!verify_license_with_server()) {
        log_warning("License verification failed");
        return 0;

    }

    return 1;
}
```

## 어린이를 위한 비유

**로직 밤은 "시한 폭탄"**과 같습니다.

상상해 보세요:
- **평소**: 폭탄은 주머니에 조용히 있어요 (아무도 몰라요)
- **트리거**: "12월 31일 자정"이 되거나, "내가 회사에서 해고"되면
- **폭발**: 펑! 하고 데이터가 사라져요

실제로 있었던 일:
- 어떤 프로그래머가 회사에서 해고될 뻔했어요
- 그래서 급여 프로그램에 **"내 이름이 급여 명단에서 사라지면 회사 컴퓨터 전부 다운!"**이라는 코드를 심었어요
- 나중에 정말로 해고되자... 펑!

트리거의 종류:
- **타이머**: "2025년 1월 1일에 터져"
- **파일 확인**: "`/tmp/still_employed` 파일이 없으면 터져"
- **계정 확인**: "급여 명단에 내 이름이 없으면 터져"

방어하는 방법:
- **코드 검사**: 프로그래머가 퇴사할 때 코드를 꼼꼼히 읽어봐요
- **4눈 검토**: 코드는 최소 2명이 검토해야 해요
- **감시**: 이상한 작업이 있으면 바로 알람이 울려야 해요

로직 밤은 보통 **"속이 쓰린 내부자"**가 만들기 때문에 방어가 정말 어려워요. 신뢰할 수 있어 보이는 사람일수록 더 확인해야 해요!
