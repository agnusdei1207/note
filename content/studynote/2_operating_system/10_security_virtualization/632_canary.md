# 카나리 (Canary) / 스택 스매싱 가드 (Stack Smashing Protector)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 반환 주소 바로 앞에 무작위 값(카나리)을 삽입하여 버퍼 오버플로우 탐지
> 2. **가치**: 함수 반환 전 카나리 변경 확인으로 스택 파괴 실시간 탐지
> 3. **융합**: 컴파일러(gcc -fstack-protector), OS 수준(SSP)에서 구현

---

### Ⅰ. 개요 (Context & Background)

**개념**
카나리(Canary)는 **석탄 광산의 카나리**에서 유래한 용어로, 위험 탐지를 위해 **스택의 반환 주소 바로 앞에 무작위 값을 삽입**하는 보안 기술이다. 버퍼 오버플로우가 발생하면 카나리가 먼저 손상되므로, 함수가 반환되기 전에 카나리 값을 검사하여 공격을 탐지할 수 있다. 스택 스매싱 가드(Stack Smashing Protector)라고도 한다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        카나리 기본 작동 원리                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   카나리가 없는 경우 (취약)                                                │
│   ─────────────────────────────────                                         │
│   ┌─────────────────────────────────────────────────────────────────┐      │
│  │                    취약 함수 스택 프레임                          │      │
│  │                                                                   │      │
│  │   High Address                                                    │      │
│  │      │                                                            │      │
│  │      ▼                                                            │      │
│  │   ┌───────────────────┐                                           │      │
│  │   │   반환 주소       │  ←─ 공격자가 덮어쓰기 목표                │      │
│  │   ├───────────────────┤                                           │      │
│  │   │   저장된 EBP       │                                           │      │
│  │   ├───────────────────┤                                           │      │
│  │   │                   │                                           │      │
│  │   │   버퍼 [16바이트] │  ←─ 오버플로우로 여기까지 덮어씀          │      │
│  │   │                   │      반환 주소도 같이 변함!               │      │
│  │   └───────────────────┘                                           │      │
│  │                   │                                                │      │
│  │                   ▼                                                │      │
│  │              Low Address                                           │      │
│  │                                                                   │      │
│  │   → 공격자가 원하는 주소로 점프!                                  │      │
│  │                                                                   │      │
│  └───────────────────────────────────────────────────────────────────┘      │
│                                                                             │
│   카나리가 있는 경우 (보안)                                                │
│   ─────────────────────────────────                                         │
│   ┌─────────────────────────────────────────────────────────────────┐      │
│  │                    보호된 함수 스택 프레임                         │      │
│  │                                                                   │      │
│  │   High Address                                                    │      │
│  │      │                                                            │      │
│  │      ▼                                                            │      │
│  │   ┌───────────────────┐                                           │      │
│  │   │   반환 주소       │  ←─ 카나리가 보호                          │      │
│  │   ├───────────────────┤                                           │      │
│  │   │   저장된 EBP       │                                           │      │
│  │   ├───────────────────┤                                           │      │
│  │   │   카나리          │  ←─ 무작위 값 (예: 0xdeadbeef)           │      │
│  │   ├───────────────────┤                                           │      │
│  │   │                   │                                           │      │
│  │   │   버퍼 [16바이트] │                                           │      │
│  │   │                   │                                           │      │
│  │   └───────────────────┘                                           │      │
│  │                   │                                                │      │
│  │                   ▼                                                │      │
│  │              Low Address                                           │      │
│  │                                                                   │      │
│  │   오버플로우 발생 시:                                               │      │
│  │   ┌───────────────────┐                                           │      │
│  │   │ AAAA (변경됨)     │  ←─ 카나리가 변조됨!                      │      │
│  │   ├───────────────────┤                                           │      │
│  │   │ AAAA              │  ←─ EBP 덮어씀                             │      │
│  │   ├───────────────────┤                                           │      │
│  │   │ AAAA...AAAA       │  ←─ 버퍼 전체 덮어씀                      │      │
│  │   └───────────────────┘                                           │      │
│  │                                                                   │      │
│  │   함수 반환 전:                                                     │      │
│  │   → 카나리 검사: 0xdeadbeef != AAAA                                │      │
│  │   → __stack_chk_fail() 호출                                        │      │
│  │   → 프로그램 종료 (공격 차단!)                                     │      │
│  │                                                                   │      │
│  └───────────────────────────────────────────────────────────────────┘      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**💡 비유**
카나리는 **광산의 카나리**와 같다.
- 광산 가스: 보이지 않는 위험 (버퍼 오버플로우)
- 카나리: 가스에 먼저 반응 (반환 주소보다 먼저 손상)
- 광부: 카나리 죽음을 보고 즉시 대피 (프로그램 종료)

**등장 배경**
① **1997년 StackGuard**: 최초의 카나리 구현
② **2000대**: gcc에 -fstack-protector 도입
③ **현재**: 거의 모든 현대 컴파일러, OS 표준 기능

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소**

| 요소 | 역할 | 생성 방법 | 검사 위치 | 비유 |
|:---|:---|:---|:---|:---|
| **카나리 값** | 무작위 정수 | TLS, 전역 변수 | 함수 반환 전 | 경보기 |
| **카나리 위치** | 반환 주소 앞 | 컴파일러 자동 | 프롤로그/에필로그 | 센서 |
| **검사 함수** | 위반 시 처리 | __stack_chk_fail | libc에 구현 | 대응팀 |
| **종류** | 랜덤, null, terminator | 목적에 따라 다름 | 컴파일 옵션 | 카나리 종류 |

**ASCII 구조 다이어그램**

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                         카나리 종류별 특징                                   │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │                  1. 랜덤 카나리 (Random Canary)                     │    │
│  │                                                                     │    │
│  │   • 프로세스 시작 시 랜덤 생성                                      │    │
│  │   • 각 스레드별로 다른 값 (TLS에 저장)                             │    │
│  │   • NULL 바이트 포함 (문자열 함수 보호)                              │    │
│  │   • 가장 강력한 보호                                                  │    │
│  │                                                                     │    │
│  │   예시: 0xaabbccdd, 0x12ab34cd 등                                    │    │
│  │                                                                     │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │                  2. NULL 카나리 (Null Canary)                        │    │
│  │                                                                     │    │
│  │   • 모두 0x00으로 구성                                                │    │
│  │   • strcpy, gets 등 NULL 종료 문자열 함수에서 보호                    │    │
│  │   • 공격자가 0으로 덮어쓰기 어려움                                   │    │
│  │   • 보안성 낮음, 성능 우회                                            │    │
│  │                                                                     │    │
│  │   예시: 0x00000000                                                    │    │
│  │                                                                     │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │                  3. Terminator Canary                                 │    │
│  │                                                                     │    │
│  │   • 0x000000ff, 0xffffffff 등                                        │    │
│  │   • 특정 조건에서 유용                                                │    │
│  │                                                                     │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │                  카나리 검사 과정                                     │    │
│  ├─────────────────────────────────────────────────────────────────────┤    │
│  │                                                                     │    │
│  │  함수 프롤로그 (진입 시)                                             │    │
│  │  ┌───────────────────────────────────────────────────────────────┐  │    │
│  │  │  sub rsp, 0x10           ; 버퍼 공간 할당                     │  │    │
│  │  │  mov rax, [fs:0x28]      ; 카나리 값 로드 (TLS에서)            │  │    │
│  │  │  mov [rsp+0x08], rax    ; 카나리 스택에 저장                   │  │    │
│  │  │  ...                   ; 함수 본체                            │  │    │
│  │  └───────────────────────────────────────────────────────────────┘  │    │
│  │                                                                     │    │
│  │  함수 에필로그 (종료 시)                                             │    │
│  │  ┌───────────────────────────────────────────────────────────────┐  │    │
│  │  │  call __stack_chk_fail  ; 카나리 검사 (아래에서 설명)         │  │    │
│  │  │  mov rax, [rsp+0x08]    ; 스택에서 카나리 로드                │  │    │
│  │  │  cmp rax, [fs:0x28]      ; TLS의 원본과 비교                  │  │    │
│  │  │  jne .L_stack_chk_fail  ; 다르면 실패                         │  │    │
│  │  │  ...                   ; 같으면 정상 반환                      │  │    │
│  │  │                                                             │  │    │
│  │  │  .L_stack_chk_fail:                                       │  │    │
│  │  │      call __stack_chk_fail  ; 위반 시 호출                   │  │    │
│  │  └───────────────────────────────────────────────────────────────┘  │    │
│  │                                                                     │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

**해설**
1. **프롤로그**: 함수 진입 시 카나리를 스택에 삽입
2. **에필로그**: 함수 반환 전 카나리 검사
3. **위반 시**: __stack_chk_fail()로 프로그램 종료

**심층 동작 원리: gcc -fstack-protector**

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                       컴파일러 보호 옵션별 동작                               │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  gcc -fstack-protector 옵션                                                │
│  ────────────────────────────────────────────────────────────────────────   │
│  • char buffer[8];와 같은 버퍼가 있는 함수에만 카나리 추가                 │
│  • 버퍼가 없는 함수는 보호하지 않음                                        │
│                                                                              │
│  gcc -fstack-protector-strong 옵션 (권장)                                  │
│  ────────────────────────────────────────────────────────────────────────   │
│  • 다음 조건 중 하나 이상이면 카나리 추가                                    │
│  • 배열을 할당하는 함수                                                   │
│  • 지역 변수의 주소를 호출하는 함수                                        │
│  • alloca를 사용하는 함수                                                 │
│                                                                              │
│  gcc -fstack-protector-all 옵션                                           │
│  ────────────────────────────────────────────────────────────────────────   │
│  • 모든 함수에 카나리 추가                                                  │
│  • 성능 저하 큼                                                           │
│                                                                              │
│  ┌──────────────────────────────────────────────────────────────────────┐  │
│  │                   __stack_chk_fail 동작                             │  │    │
│  ├──────────────────────────────────────────────────────────────────────┤  │    │
│  │                                                                      │  │    │
│  │  1. stderr에 "stack smashing detected" 메시지 출력                  │  │    │
│  │  2. SIGABRT 시그널로 프로세스 종료                                   │  │    │
│  │  3. 코어 덤프 생성 (coredump)                                        │  │    │
│  │                                                                      │  │    │
│  │  목적:                                                              │  │    │
│  │  • 공격을 즉시 차단                                                 │  │    │
│  │  • 포렌식 분석을 위한 기록 남김                                     │  │    │
│  │                                                                      │  │    │
```

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 스택 보호 기법**

| 기법 | 탐지 대상 | 탐지 시점 | 우회 가능성 | 오버헤드 |
|:---|:---|:---|:---|:---|:---|
| **Stack Canary** | 스택 버퍼 오버플로우 | 반환 전 | 정보 누출로 우회 | 미미 |
| **SafeStack** | 스택 분리 | 실행 시 | 어려움 | 중간 |
| **StackCookies** | Microsoft 구현 | 반환 전 | 정보 누출로 우회 | 미미 |
| **Shadow Stack** | 반환 주소 복사 | 실행 시 | 매우 어려움 | 미미 |

**과목 융합 관점**

1. **컴파일러**: 프로즘/에필로그 삽입, 최적화

2. **운영체제**: TLS, 스레드, 시그널 처리

3. **보안**: Defense in Depth, NX/DEP와 결합

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나리오 1: 카나리 활성화 확인**
```bash
# gcc 컴파일러
gcc -fstack-protector-strong -Wstack-protector ...

# 바이너리 확인
readelf -s a.out | grep stack_chk
```

**시나시오 2: 우회 공격**
- 정보 누출로 카나리 획득
- __stack_chk_fail GOT 오버라이드

**도입 체크리스트**

**기술적 측면**
- [ ] -fstack-protector-strong 활성화
- [ ] 정적 분석 도구 확인
- [ ] 런타임 보호 확인
- [ ] 컴파일러 최신 버전

**운영·보안적 측면**
- [ ] 보안 코딩 교육
- [ ] 정기적 보안 평가
- [ ] 코어 덤프 분석

**안티패턴**
1. **"성능 문제"**: 오버헤드 미미
2. **"false positive"": 실제 오버플로우만 탐지
3. **"우회 가능하니까 쓸모 없어"**: NX/DEP와 결합하면 강력

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**

| 지표 | 카나리 비활성 | 카나리 활성 | 개선 효과 |
|:---|:---|:---|:---|
| **스택 오버플로우 탐지율** | 0% | ~99% | 탐지 가능 |
| **익스플로이팅 난이도** | 쉬움 | 어려움 | 90% 증가 |
| **성능 오버헤드** | 0% | <1% | 미미 |
| **false positive** | N/A | 실제 공격만 | 정확 |

**미래 전망**
1. **Shadow Stack**: Intel CET
2. **SafeStack**: LLVM 구현
3. **ARM Pointer Authentication**

### 👶 어린이를 위한 3줄 비유 설명
1. **개념**: 카나리는 깃발 같은 경보기를 스택에 꽂아두는 거예요
2. **원리**: 나쁜 사람이 깃발(반환 주소)을 꺾으려고 하면 경보기(카나리)가 먼저 울려요
3. **효과**: 경보가 울리면 컴퓨터가 즉시 멈추서 공격을 막아요
