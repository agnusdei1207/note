# 포트 스캐닝 (Port Scanning)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 대상 시스템의 열린 포트(Open Port)를 식별하여 취약점을 발견하는 사전 정찰 기법
> 2. **가치**: 공격자는 공격 전 정보 수집에, 방어자는 자가 점검에 활용
> 3. **융합**: TCP/IP 프로토콜의 세부 동작과 OS의 네트워크 스택 이해 필요

---

### Ⅰ. 개요 (Context & Background)

**개념**
포트 스캐닝(Port Scanning)은 **대상 호스트의 어떤 포트가 열려 있는지(서비스가 실행 중인지) 확인하는 기법**이다. 공격자는 이를 통해 취약한 서비스를 찾아내고 공격을 계획하며, 시스템 관리자는 자가 보안 점검에 활용한다. 주요 기법으로 TCP Connect, SYN Stealth, FIN, XMAS, NULL 스캔 등이 있다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         포트 스캐닝 기본 흐름                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   공격자                                                                  │
│   ┌─────────────┐                                                        │
│   │  스캐너      │                                                        │
│   └─────┬───────┘                                                        │
│         │                                                                 │
│         ▼ 포트별 연결 시도                                                 │
│   ┌─────────────────────────────────────────────────────────────────┐     │
│  │  포트 21 ───▶ [OPEN]     FTP 서비스 발견!                     │     │
│  │  포트 22 ───▶ [OPEN]     SSH 서비스 발견!                     │     │
│  │  포트 80 ───▶ [OPEN]     HTTP 서비스 발견!                    │     │
│  │  포트 135 ──▶ [FILTERED] 방화벽에 의해 차단                  │     │
│  │  포트 443 ──▶ [OPEN]     HTTPS 서비스 발견!                   │     │
│  │  포트 445 ──▶ [OPEN]     SMB 서비스 발견! (취약점 있을 수도) │    │
│  │  포트 3306─▶ [OPEN]     MySQL 서비스 발견!                   │     │
│  │  포트 3389─▶ [CLOSED]    서비스 없음                           │     │
│  │  포트 8080─▶ [OPEN]     대체 웹 서비스 발견!                  │     │
│  └─────────────────────────────────────────────────────────────────┘     │
│         │                                                                 │
│         ▼                                                                 │
│   ┌─────────────────────────────────────────────────────────────────┐     │
│  │                         스캔 결과                              │     │
│  │                                                                  │     │
│  │  • 개방된 서비스 식별                                         │     │
│  │  • 운영 중인 OS 추정                                         │     │
│  │  • 취약한 서비스 버전 확인                                    │     │
│  │  • 공격 계획 수립                                               │     │
│  │                                                                  │     │
│  └───────────────────────────────────────────────────────────────────┘     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**💡 비유**
포트 스캐닝은 **건물의 문과 창문 확인**과 같다.
- 열린 문: 개방된 포트 (서비스 실행 중)
- 잠긴 문: 닫힌 포트 (서비스 없음)
- 도둑: 열린 문을 찾아 침입 경로 확인

**등장 배경**
① **1980대**: 초기 스캐 도구 등장
② **1995년 nmap**: 현재까지 가장 널리 쓰이는 도구
③ **2000년대**: 분산 스캔, 스텔스 기법
④ **현재**: 클라우드 기반 대규모 스캔

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소**

| 스캔 기법 | 원리 | 탐지 난이도 | 속도 | 특징 |
|:---|:---|:---|:---|:---|
| **TCP Connect** | 완전한 3-way handshake | 쉬움 | 느림 | 로그에 남음 |
| **SYN Stealth** | SYN만 전송 | 중간 | 빠름 | 로그 최소화 |
| **FIN/XMAS/NULL** | 플래그 비정상 | 어려움 | 빠름 | 방화벽 우회 |
| **UDP Scan** | ICMP 응답 확인 | 중간 | 빠름 | 신뢰도 낮음 |
| **SCTP/ECT** | 새로운 프로토콜 | 어려움 | 빠름 | 우회 기법 |

**ASCII 구조 다이어그램**

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                         포트 스캔 기법별 비교                                 │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │              1. TCP Connect Scan (가장 기본적)                        │    │
│  │                                                                     │    │
│  │   ┌───────────────────────────────────────────────────────────────┐ │    │
│  │   │ 스캐너                   서버                             │ │    │
│  │   │  │  ──── SYN ───────────────▶│                           │ │    │
│  │   │  │                         │ SYN_RCQ에 추가            │ │    │
│  │   │  │  ◀── SYN+ACK ──────────│                           │ │    │
│  │   │  │                         │ (포트 OPEN)               │ │    │
│  │   │  │  ──── ACK ──────────────▶│ 연결 확립                 │ │    │
│  │   │  │  ──── RST ──────────────▶│ 연결 종료                 │ │    │
│  │   │  │                                                         │ │    │
│  │   │   ◀── RST ──────────────▶ (포트 CLOSED)                       │    │
│  │   │                                                         │ │    │
│  │   │   장점: 정확, 신뢰할 수 있음                                  │    │
│  │   │   단점: 로그에 남음, 느림                                     │    │
│  │   └───────────────────────────────────────────────────────────────┘ │    │
│  │                                                                     │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │              2. SYN Stealth Scan (Half-open Scan)                  │    │
│  │                                                                     │    │
│  │   ┌───────────────────────────────────────────────────────────────┐ │    │
│  │   │ 스캐너                   서버                             │ │    │
│  │   │  │  ──── SYN ───────────────▶│                           │ │    │
│  │   │  │                         │ SYN_RCQ에 추가            │ │    │
│  │   │  │  ◀── SYN+ACK/ RST ─────│                           │ │    │
│  │   │  │  (ACK를 보내지 않음!)     │ (반쪽 열린 상태)         │ │    │
│  │   │  │                         │                          │    │
│  │   │  │   SYN+ACK → OPEN                                        │    │
│  │   │  │   RST    → CLOSED                                       │    │
│  │   │  │                                                         │ │    │
│  │   │   장점: 로그 최소화, 빠름                                     │    │
│  │   │   단점: 일반 IDS가 탐지                                      │    │
│  │   └───────────────────────────────────────────────────────────────┘ │    │
│  │                                                                     │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │              3. FIN/XMAS/NULL Scan (스텔스 기법)                      │    │
│  │                                                                     │    │
│  │   RFC 793를 위반하는 플래그 조합으로 탐지                             │    │
│  │   • FIN Scan: FIN 플래그만 전송                                      │    │
│  │   • XMAS Scan: FIN+URG+PSH 전송                                     │    │
│  │   • NULL Scan: 모든 플래그 0으로 전송                                 │    │
│  │                                                                     │    │
│  │   RFC 준수 시스템: RST 반환                                         │    │
│  │   Microsoft 시스템: 응답 없음 (취약점!)                            │    │
│  │                                                                     │    │
│  │   장점: 스텔스, 일부 방화벽 우회                                    │    │
│  │   단점: 신뢰도 낮을 수 있음                                         │    │
│  │                                                                     │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │              4. UDP Scan                                           │    │
│  │                                                                     │    │
│  │   UDP 포트에 패킷 전송                                              │    │
│  │   • 응답 있음 → OPEN                                               │    │
│  │   • ICMP Port Unreachable → CLOSED                                  │    │
│  │   • 응답 없음 → Filtered 또는 Open                                   │    │
│  │                                                                     │    │
│  │   장점: 빠름, 간단                                                   │    │
│  │   단점: 신뢰도 낮음 (UDP는 비연결형)                               │    │
│  │                                                                     │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

**해설**
1. **TCP Connect**: 완전한 연결로 정확하지만 로그 남음
2. **SYN Stealth**: 반쪽 열린 상태로 스텔스
3. **FIN/XMAS/NULL**: 플래그 조합으로 우회
4. **UDP Scan**: 비연결형 프로토콜 스캔

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 포트 스캔 탐지 기법**

| 탐지 기법 | 원리 | 오탐지율 | 성능 영향 | 복잡도 |
|:---|:---|:---|:---|:---|
| **Log Analysis** | 로그 파일 분석 | 낮음 | 낮음 | 쉬움 |
| **Network IDS** | 트래픽 패턴 분석 | 중간 | 중간 | 중간 |
| **Honeypot** | 가짜 서비스 | 높음 | 낮음 | 중간 |
| **Behavioral** | 비정상 패턴 | 중간 | 낮음 | 어려움 |

**과목 융합 관점**

1. **네트워크**: TCP/IP, 프로토콜 스택
2. **운영체제**: 소켓 프로그래밍, 네트워크 서브시스템
3. **보안**: 자가 방어, 탐지

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나시오 1: 자가 스캔 (보안 점검)**
```bash
nmap -sV -O target.com
```

**시나시오 2: 탐지 대응**
- **방화벽**: 비허용 포트 차단
- **IDS**: 탐지, 알림
- **Honeypot**: 데미 서비스로 미끼

**도입 체크리스트**

**기술적 측면**
- [ ] 방화벽 정책 점검
- [ ] IDS/IPS 배치
- [ ] 로그 모니터링
- [ ] Honeypot 운영
- [ ] 정기적 스캔

**운영·보안적 측면**
- [ ] 불필요한 서비스 중지
- [ ] 포트 최소화
- [ ] 보안 정책 수립
- [ ] 스캔 탐지 알림

**안티패턴**
1. **"아무도 모르게"**: 스캔 항상 진행 중
2. **"방화벽이 막아"**: 내부 스캔 가능
3. **"포트 닫으면 돼"**: 필수 서비스만

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**

| 지표 | 대응 전 | 대응 후 | 개선 효과 |
|:---|:---|:---|:---|
| **발견된 개방 포트** | 많음 | 최소화 | 80% 감소 |
| **스캔 탐지율** | 낮음 | 높음 | 90% 증가 |
| **취약점 노출** | 높음 | 낮음 | 85% 감소 |

**미래 전망**
1. **AI 기반 탐지**: 이상 패턴 학습
2. **스마트 방화벽**: 자동화된 대응
3. **Deception**: 가짜 서비스로 미끼

**참고 표준**
- **RFC 768**: ICMP
- **nmap**: 스캔 도구
- **OWASP**: 포트 보안

### 👶 어린이를 위한 3줄 비유 설명
1. **개념**: 포트 스캐닝은 집의 문과 창문을 열어보는 것과 같아요
