# umask

## 핵심 인사이트

**umask(user mask)는 새로 생성되는 파일과 디렉터리의 기본 권한을 제어하는 프로세스별 비트 마스크다.** 시스템의 기본 권한(파일 666, 디렉터리 777)에서 umask 값이 **제거(mask)**되어 최종 권한이 결정된다. 이는 보안의 기본 원칙인 "최소 권한의 원칙"을 실현하는 핵심 메커니즘이다.

---

## Ⅰ. 개요 및 정의

### 1. 정의

| 요소 | 설명 |
|:---|:---|
| **umask** | 새 파일/디렉터리 생성 시 **제거될 권한 비트**를 지정하는 마스크 |
| **기본값** | 일반적으로 `022` (그룹/other의 쓰기 권한 제거) |
| **범위** | 프로세스별로 설정되어, 자식 프로세스에게 **상속**됨 |

### 2. 동작 원리

```
┌─────────────────────────────────────────────────────────────┐
│                   umask 동작 원리                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   최종 권한 = 기본 권한 & ~umask                           │
│                                                             │
│   파일:     0666 & ~022 = 0666 & 755 = 0644 (rw-r--r--)   │
│   디렉터리: 0777 & ~022 = 0777 & 755 = 0755 (rwxr-xr-x)  │
│                                                             │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐               │
│   │ 기본값  │ →  │ umask   │ →  │ 최종값  │               │
│   ├─────────┤    ├─────────┤    ├─────────┤               │
│   │ 파일    │    │ 022     │    │ 644     │               │
│   │  666    │    │(비트제거)│    │         │               │
│   │ 디렉터리 │    │         │    │ 755     │               │
│   │  777    │    │         │    │         │               │
│   └─────────┘    └─────────┘    └─────────┘               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅱ. 비트 연산 및 계산

### 1. 비트별 계산

```
┌─────────────────────────────────────────────────────────────┐
│                umask = 022 일 때 권한 계산                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   파일 권한 (666 = rw-rw-rw-)                             │
│   ┌───┬───┬───┬───┬───┬───┬───┬───┬───┐                     │
│   │ r │ w │ x │ r │ w │ x │ r │ w │ x │  (기본)          │
│   │ 1 │ 1 │ 0 │ 1 │ 1 │ 0 │ 1 │ 1 │ 0 │  = 666            │
│   └───┴───┴───┴───┴───┴───┴───┴───┴───┘                     │
│             ─       ─       ─                              │
│   ┌───┬───┬───┬───┬───┬───┬───┬───┬───┐                     │
│   │   │   │   │ 0 │ 0 │ 0 │ 0 │ 0 │ 0 │  (umask 022)      │
│   │ 0 │ 0 │ 0 │ 1 │ 0 │ 0 │ 1 │ 0 │ 0 │  = 022            │
│   └───┴───┴───┴───┴───┴───┴───┴───┴───┘                     │
│         ─       ─       ─                                  │
│   ┌───┬───┬───┬───┬───┬───┬───┬───┬───┐                     │
│   │ r │ w │ - │ r │ - │ - │ r │ - │ - │  (결과)          │
│   │ 1 │ 1 │ 0 │ 1 │ 0 │ 0 │ 1 │ 0 │ 0 │  = 644            │
│   └───┴───┴───┴───┴───┴───┴───┴───┴───┘                     │
│                                                             │
│   디렉터리 권한 (777 = rwxrwxrwx)                         │
│   ┌───┬───┬───┬───┬───┬───┬───┬───┬───┐                     │
│   │ r │ w │ x │ r │ w │ x │ r │ w │ x │  = 777            │
│   │ 1 │ 1 │ 1 │ 1 │ 1 │ 1 │ 1 │ 1 │ 1 │                   │
│   └───┴───┴───┴───┴───┴───┴───┴───┴───┘                     │
│             ─       ─       ─                              │
│   AND  ~022 (755 = rwxr-xr-x)                             │
│   ┌───┬───┬───┬───┬───┬───┬───┬───┬───┐                     │
│   │ r │ w │ x │ r │ - │ x │ r │ - │ x │  = 755            │
│   │ 1 │ 1 │ 1 │ 1 │ 0 │ 1 │ 1 │ 0 │ 1 │                   │
│   └───┴───┴───┴───┴───┴───┴───┴───┴───┘                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 일반적인 umask 값

| umask | 파일 | 디렉터리 | 용도 |
|:---:|:---:|:---:|:---|
| **022** | 644 (rw-r--r--) | 755 (rwxr-xr-x) | **표준** (그룹/other 쓰기 제거) |
| **002** | 664 (rw-rw-r--) | 775 (rwxrwxr-x) | 팀 협업 (그룹 쓰기 허용) |
| **027** | 640 (rw-r-----) | 750 (rwxr-x---) | 보안 중요 (그룹 읽기만) |
| **077** | 600 (rw-------) | 700 (rwx------) | **최대 보안** (개인 전용) |
| **000** | 666 (rw-rw-rw-) | 777 (rwxrwxrwx) | 권장하지 않음 |

## Ⅲ. 시스템 동작 및 설정

### 1. 파일 생성 과정에서의 umask

```
┌─────────────────────────────────────────────────────────────┐
│            리눅스 커널 파일 생성 시 umask 적용              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1) 사용자 공간                                            │
│     fd = open("newfile.txt", O_CREAT | O_WRONLY, 0666);   │
│                                                         │
│  2) 시스템 콜 진입 (fs/open.c)                             │
│     ┌─────────────────────────────────────────────────┐   │
│     │ long do_sys_open(int dfd, const char *filename, │   │
│     │                  int flags, umode_t mode)      │   │
│     │ {                                              │   │
│     │     if (!IS_PRIVATE(inode))                    │   │
│     │         mode &= ~current_umask();  // ★ umask  │   │
│     │     ...                                         │   │
│     │ }                                              │   │
│     └─────────────────────────────────────────────────┘   │
│                                                             │
│  3) 최종 권한                                              │
│     requested: 0666                                        │
│     umask:       022                                        │
│     actual:      0666 & ~022 = 0644                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 쉘별 umask 설정

```bash
# 현재 umask 확인
$ umask
022

# 일시적 변경
$ umask 077
$ touch test.txt
$ ls -l test.txt
-rw------- 1 user group 0 ...  # 600 권한

# 영구적 설정 (셸 설정 파일)
~/.bashrc    : umask 022       # bash 사용자
~/.zshrc     : umask 022       # zsh 사용자
/etc/profile : umask 022       # 전역 설정
/etc/login.defs: UMASK 022     # PAM 시스템 기본값
```

### 3. 프로세스 상속

```
┌─────────────────────────────────────────────────────────────┐
│                 umask의 프로세스 상속                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  부모 프로세스 (umask=022)                                  │
│       │                                                     │
│       │ fork() + exec()                                    │
│       ├─────────────────────────────────────┐              │
│       ▼                                     ▼              │
│  자식 프로세스 A                  자식 프로세스 B          │
│  umask = 022 (상속)                umask = 077 (변경)      │
│       │                                     │              │
│       │                                     │ fork()       │
│       ▼                                     ▼              │
│  손자 A-1                         손자 B-1                    │
│  umask = 022                     umask = 077 (상속)        │
│                                                             │
│   ⚠️ exec()은 umask를 유지함                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Ⅳ. 보안 및 활용

### 1. 보안 관점에서의 umask

| 시나리오 | 권장 umask | 이유 |
|:---|:---:|:---|
| 일반 사용자 | **022** | 개인 파일 보호 + 공유 가능 |
| 웹 서버 | **027** | 로그, 설정 파일 보호 |
| 데이터베이스 | **007** 또는 **077** | 중요 데이터 격리 |
| 개발 팀 공유 | **002** | 협업을 위한 그룹 쓰기 허용 |
| 임시 디렉터리 | **077** | `/tmp` 대신 사용자 전용 임시 공간 |

### 2. 웹 서버 보강 예시

```bash
# 웹 서버 (www-data)의 umask 설정
$ cat /etc/apache2/envvars
export APACHE_RUN_USER=www-data
export APACHE_RUN_GROUP=www-data
umask 027  # 그룹은 읽기만, other는 완전 차단

# 결과: 생성되는 파일
# -rw-r-----  www-data www-data  index.html  (640)
# drwxr-x---  www-data www-data  uploads/    (750)
```

### 3. 프로그래밍에서의 umask 제어

```c
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>

// umask 변경 후 파일 생성
mode_t old_umask = umask(077);  // 기존 값 저장
int fd = open("secret.txt", O_CREAT | O_WRONLY, 0666);
umask(old_umask);                // 복원
// 결과: secret.txt은 600 (rw-------)

// 스레드 환경에서의 주의점
// umask는 프로세스 전역 상태이므로
// 스레드 간 경쟁 조건 발생 가능!
pthread_mutex_lock(&umask_mutex);
mode_t old = umask(new_mask);
// ... 파일 생성 ...
umask(old);
pthread_mutex_unlock(&umask_mutex);
```

### 4. PAM을 통한 시스템 관리

```bash
# /etc/login.defs
UMASK           022
# 또는 그룹별 다른 umask
USERGROUPS_ENAB yes

# /etc/pam.d/common-session
session optional pam_umask.so umask=022
```

## Ⅴ. 문제 및 디버깅

### 1. 흔한 문제

```bash
# 문제: 파일에 실행 권한이 없다!
$ gcc hello.c -o hello
$ ./hello
bash: ./hello: Permission denied

$ ls -l hello
-rw-r--r-- 1 user group 16456 ...  # 644 권한

# 원인: umask가 022라 실행 비트(x)가 제거됨
$ umask
022

# 해결: 명시적으로 실행 권한 부여
$ chmod +x hello
# 또는 umask 변경 (권장하지 않음)
$ umask 0  # 임시로만 사용
```

### 2. 디버깅 명령어

```bash
# 현재 umask 확인 (8진수)
$ umask
022

# 기호 형식으로 확인
$ umask -S
u=rwx,g=rx,o=rx

# 특정 디렉터리에서의 umask 시뮬레이션
$ python3 -c "print(oct(0o666 & ~0o022))"
0o644

# 모든 setuid/setgid 파일 찾기 (보안 점검)
$ find / -type f \( -perm -4000 -o -perm -2000 \) 2>/dev/null
```

## 어린이를 위한 비유

**umask는 "기본 규칙 정하기"**와 같습니다.

학교에 새로 전학 온 친구가 있다고 상상해 보세요:

- **기본 설정 (umask=022)**: "누구나 내 물건을 **볼 수는** 있지만, **함께 쓰거나** 수정하려면 **내 허락**을 받아야 해"
  - 파일은 모두가 읽을 수 있지만(644), 내 물건을 망치지 못하게 쓰기는 제한

- **협업 모드 (umask=002)**: "우리 **반 친구들**은 **함께 사용**해도 돼"
  - 그룹(반 친구들)은 읽고 쓰기 가능(664)

- **비밀 모드 (umask=077)**: "이건 **나만의 비밀 일기**야"
  - 나만 읽고 쓸 수 있어(600)

umask는 매번 파일을 만들 때마다 "이건 누가 써도 돼?"라고 물어보는 대신, **미리 정해둔 기본 규칙**을 자동으로 적용해 주는 편리한 시스템입니다!
