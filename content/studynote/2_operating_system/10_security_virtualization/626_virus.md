# 바이러스 (Virus)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 자가 복제 능력을 가진 악성코드로 정상 파일(실행 파일, 문서)에 기생하여 전파
> 2. **가치**: 생물학적 바이러스와 유사하게 숙주 없이는 독립적으로 존재할 수 없으며, 사용자 개입 필요
> 3. **융합**: 부트 섹터, 파일 바이러스, 매크로 바이러스 등 다양한 형태로 진화

---

### Ⅰ. 개요 (Context & Background)

**개념**
컴퓨터 바이러스(Virus)는 **자신의 코드를 다른 정상 프로그램이나 파일에 삽입하여 복제하는 악성코드**를 말한다. 생물학적 바이러스가 세포에 기생하듯, 컴퓨터 바이러스도 실행 파일, 문서, 부트 섹터 등에 기생하며, 감염된 파일이 실행될 때마다 다른 파일로 전파된다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         바이러스 기본 감염 프로세스                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   정상 파일 (health.exe)                                                    │
│   ┌───────────────────┐                                                    │
│   │  PE 헤더          │                                                    │
│   │  .text 섹션       │                                                    │
│   │  .data 섹션       │                                                    │
│   └───────────────────┘                                                    │
│           │                                                                │
│           │ 바이러스 감염!                                                  │
│           ▼                                                                │
│   감염 파일 (infected.exe)                                                  │
│   ┌───────────────────┐                                                    │
│   │  바이러스 코드    │  ←─── 앞이나 뒤에 삽입                             │
│   ├───────────────────┤                                                    │
│   │  원본 PE 헤더     │                                                    │
│   │  원본 .text 섹션  │                                                    │
│   │  원본 .data 섹션  │                                                    │
│   └───────────────────┘                                                    │
│           │                                                                │
│           │ 실행 시                                                        │
│           ▼                                                                │
│   ┌───────────────────────────────────────────────────────────────────┐    │
│   │                   바이러스 실행 흐름                             │    │
│   │                                                                   │    │
│   │   1. 바이러스 코드 먼저 실행                                      │    │
│   │      └─ 메모리에 상주 (Resident)                                  │    │
│   │      └─ 다른 파일 감염 시도                                      │    │
│   │                                                                   │    │
│   │   2. 원본 프로그램으로 제어 이동 (JMP)                             │    │
│   │      └─ 사용자는 정상 동작인 줄 착각                               │    │
│   │                                                                   │    │
│   │   3. 백그라운드에서 감염 활동 계속                               │    │
│   │                                                                   │    │
│   └───────────────────────────────────────────────────────────────────┘    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**💡 비유**
바이러스는 **생물학적 바이러스**와 유사하다.
- 기생: 정상 세포(파일)에 침투
- 복제: 세포 분열 시 바이러스도 함께 복제
- 전파: 다른 세포로 계속 확산
- 증상: 숙주(시스템) 기능 저하, 파괴

**등장 배경**
① **1983년 Fred Cohen**: 최초로 바이러스 개념 정의 및 실험
② **1987년 Lehigh**: 최초의 파일 바이러스 (도스 환경)
③ **1990대 매크로 바이러스**: Word/Excel 매크로 악용
④ **2000년 이후**: 이메일, P2P, 웹을 통한 확산

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소**

| 구성 요소 | 역할 | 내부 동작 | 기법 | 비유 |
|:---|:---|:---|:---|:---|
| **감염 모듈** | 다른 파일로 전파 | 파일 검색, 쓰기, 코드 삽입 | APPEND, PREPEND, Cavity | 침투 |
| **페이로드** | 악성 기능 | 데이터 파괴, 메시지 표시, 백도어 | 시간 조건부, 확률적 | 증상 |
| **트리거** | 실행 타이밍 | 날짜, 실행 횟수, 키 입력 | 로직 밤 | 활성화 |
| **보호 메커니즘** | 감지 회피 | 난독화, 암호화, 변형 | Polymorphic, Metamorphic | 위장 |
| **상주 모듈** | 메모리 유지 | TSR, 후킹, 핸들러 설치 | Interrupt hooking | 잠복 |

**해설**
1. **파일 바이러스**: 가장 흔한 형태, 실행 파일에 기생
2. **부트 섹터 바이러스**: 시스템 부팅 시 가장 먼저 실행, 메모리 상주
3. **매크로 바이러스**: Office 문서의 VBA 매크로 악용
4. **다형성 바이러스**: 난독화로 매번 다른 시그니처 생성
5. **다형태 바이러스**: 코드 자체를 재작성하여 완전히 다른 바이너리

**심층 동작 원리: 바이러스 감염 알고리즘**

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                         바이러스 감염 알고리즘 상세                            │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  ① 초기화 (Initialization)                                                   │
│  ────────────────────────────────────────────────────────────────────────   │
│  • 자신이 이미 메모리에 상주하는지 확인                                     │
│  • 아직 아니면 상주 모듈 실행                                               │
│                                                                              │
│  ② 상주 (Residency)                                                          │
│  ────────────────────────────────────────────────────────────────────────   │
│  • 인터럽트 후킹 (INT 21h, INT 13h 등)                                     │
│  • 시스템 서비스 후킹 (SSDT)                                                │
│  • 파일 시스템 후킹                                                         │
│                                                                              │
│  ③ 감염 (Infection)                                                          │
│  ────────────────────────────────────────────────────────────────────────   │
│                                                                              │
│  감염 루틴 의사 코드:                                                        │
│  ┌───────────────────────────────────────────────────────────────────┐    │
│  │                                                                   │    │
│  │  foreach file in find_files("*.exe", current_dir, recursive) {   │    │
│  │                                                                   │    │
│  │      // 1. 이미 감염되었는지 확인                                 │    │
│  │      if (is_infected(file)) continue;                            │    │
│  │                                                                   │    │
│  │      // 2. 파일 크기 제한 (너무 크면 생략)                        │    │
│  │      if (file.size > MAX_SIZE) continue;                         │    │
│  │                                                                   │    │
│  │      // 3. 읽기 전용/시스템 속성 해제                            │    │
│  │      remove_readonly(file);                                      │    │
│  │                                                                   │    │
│  │      // 4. 파일 읽기                                               │    │
│  │      original = read_file(file);                                 │    │
│  │                                                                   │    │
│  │      // 5. 바이러스 복제                                         │    │
│  │      virus_copy = mutate(virus_code);  // 다형성                 │    │
│  │                                                                   │    │
│  │      // 6. 파일에 쓰기                                            │    │
│  │      write_file(file, virus_copy + original);                    │    │
│  │                                                                   │    │
│  │      // 7. 파일 타임스탬프 유지                                  │    │
│  │      set_timestamp(file, original.time);                         │    │
│  │                                                                   │    │
│  │      // 8. 감염 표식 남기기                                       │    │
│  │      mark_infected(file);                                        │    │
│  │  }                                                               │    │
│  │                                                                   │    │
│  └───────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  ④ 페이로드 실행 (Payload Execution)                                          │
│  ────────────────────────────────────────────────────────────────────────   │
│  • 트리거 조건 확인 (날짜, 실행 횟수)                                      │
│  • 조건 만족 시 악성 기능 실행                                               │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 악성코드 유형별 차이**

| 특성 | 바이러스 | 웜 | 트로이 목마 | 랜섬웨어 |
|:---|:---|:---|:---|:---|
| **자가 복제** | 있음 (기생) | 있음 (독립) | 없음 | 선택적 |
| **숙주 필요** | 필수 | 불필요 | 없음 | 없음 |
| **전파 속도** | 느림 | 매우 빠름 | 없음 | 느림 |
| **사용자 개입** | 필요 | 불필요 | 필수 | 필요 |
| **주요 피해** | 데이터 손상 | 네트워크 마비 | 권한 탈취 | 데이터 암호화 |
| **대표 사례** | ILOVEYOU, Melissa | WannaCry, Code Red | Emotet, Zeus | Ryuk, Conti |

**과목 융합 관점**

1. **운영체제(파일 시스템)**: 바이러스는 파일 시스템 구조(PE 포맷, i-node)를 악용 → 무결성 검사로 방어

2. **네트워크**: 이메일, P2P, 공유 폴더로 전파 → 메일 필터, 네트워크 세그먼트 분리

3. **컴퓨터 바이러스학**: 생물학적 바이러스 모델링 기법 적용

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나리오 1: 바이러스 감염 대응 절차**
```
1. 격리: 감염 시스템 네트워크 분리
2. 진단: 백신으로 전체 스캔
3. 치료/삭제: 백신이 치료 불가능하면 파일 삭제
4. 복구: 백업에서 정상 파일 복원
5. 사후: 감염 경로 분석, 예방 조치
```

**시나리오 2: 예방 방법**
- **백신 실시간 보안**: 실시간 스캔 활성화
- **최신 定义 업데이트**: 최소 일일 업데이트
- **이메일 필터**: 첨부파일(.exe, .docm) 차단
- **사용자 교육**: 알 수 없는 첨부파일 실행 금지
- **최소 권한**: 관리자 권한으로 일반 작업 금지

**도입 체크리스트**

**기술적 측면**
- [ ] 백신/EDR 설치 및 실시간 보호
- [ ] 정기적 전체 시스템 스캔
- [ ] OS/앱 패치 최신 상태 유지
- [ ] 이메일 보안 게이트웨이
- [ ] Application Whitelisting

**운영·보안적 측면**
- [ ] 보안 인식 교육
- [ ] 이상 활동 보고 체계
- [ ] 백업 및 복구 훈련
- [ ] 인시던트 대응 계획

**안티패턴**
1. **"무료 백신 충분"**: 기업용 백신과 기능 차이 큼
2. **"안 웃겨서 안 걸려"**: 행동 기반 탐지 우회
3. **백업 없음**: 유일한 복구 수단
4. **백신 끄기**: 성능 이유로 끄면 무방비

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**

| 지표 | 방어 전 | 방어 후 | 개선 효과 |
|:---|:---|:---|:---|
| **감염율** | 높음 (20~30%) | 낮음 (<1%) | 95% 감소 |
| **탐지 시간** | 수주~수개월 | 실시간~수시간 | 99% 단축 |
| **파일 손실율** | 높음 | 최소화 | 90% 감소 |
| **업무 중단** | 수일~수주 | 수시간 | 95% 감소 |

**미래 전망**
1. **AI 기반 탐지**: 머신러닝으로 제로데이 탐지
2. **행동 기반 방어**: EDR, XDR로 실시간 탐지
3. **클라우드 백신**: 샌드박스, 정적/동적 분석

**참고 표준**
- **AV-TEST**: 백신 성능 벤치마크
- **MITRE ATT&CK**: T1201 (System Binary Proxy Execution)
- **CIS Controls**: Critical Security Controls
