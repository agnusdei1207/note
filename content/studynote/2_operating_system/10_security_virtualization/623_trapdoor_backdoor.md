# 트랩 도어 / 백도어

## 핵심 인사이트

**백도어(Backdoor)는 정상적인 인증 절차를 우회하여 시스템에 접근할 수 있는 비밀 통로다.** 개발자가 디버깅용으로 의도적으로 남기기도 하지만, 공격자가 시스템을 장악한 후 영구적인 접근 경로를 확보하기 위해 설치하기도 한다. 백도어는 일반적인 보안 검사에서 발견되지 않도록 **은밀하게 작동**하며, OS의 다양 계층(애플리케이션, 시스템 콜, 커널)에 존재할 수 있다.

---

## Ⅰ. 정의 및 특징

### 1. 백도어의 특징

| 특징 | 설명 |
|:---|:---|
| **비밀성** | 존재가 알려지지 않아야 함 |
| **우회성** | 정상 인증 절차를 건너뜀 |
| **영구성** | 재부팅 후에도 유지되도록 설계 |
| **은밀성** | 탐지되지 않도록 숨김 |
| **편의성 vs 위험** | 개발자 편의 vs 보안 리스크 |

| 유형 | 예시 | 코드 예시 |

## Ⅲ. 백도어 기법

### 1. 하드코딩된 비밀번호

```c
// 취약한 코드 예시
int authenticate(char *username, char *password) {
    // 정상 인증
    if (check_database(username, password))
        return 1;

    // ★ 백도어: 하드코딩된 마스터 키
    if (strcmp(password, "MaStEr_KeY_2025!") == 0) {
        syslog(LOG_AUTH, "Admin access via backdoor");
        return 1;
    }

    return 0;
}
```

### 3. LD_PRELOAD 후킹

```c
// backdoor.c - 라이브러리 백도어
#define _GNU_SOURCE
#include <dlfcn.h>
#include <unistd.h>
#include <sys/types.h>

// 원본 함수 포인터
static uid_t (*real_getuid)(void) = NULL;

uid_t getuid(void) {
    // 원본 함수 처음 호출 시 로드
    if (!real_getuid) {
        real_getuid = dlsym(RTLD_NEXT, "getuid");
    }

    // 백도어 트리거: 특정 환경 변수 확인
    if (getenv("GIVE_ME_ROOT")) {
        return 0;  // 루트 UID 반환
    }

    return real_getuid();  // 정상 동작
}
```

```bash
# 컴파일 및 사용
$ gcc -shared -fPIC -o libbackdoor.so backdoor.c -ldl
$ LD_PRELOAD=./libbackdoor.so su
# 환경 변수만 있으면 루트 권한 획득!
```

## 어린이를 위한 비유

**백도어는 "비밀 문"**과 같습니다.

상상해 보세요:
- **정문**: "비밀번호 입력하고 출입하면 돼" - 이게 정상적인 로그인
- **백도어**: "건물 뒤쪽에 아무도 모르는 비밀 문이 있어"

백도어의 종류:
- **개발자가 남긴 문**: "나중가서 작업하려고" 만든 문 (잊어버리면 위험!)
- **도둑이 만든 문**: "언제든 다시 들어올 수 있게" 몰래 만든 문

트랩 도어는 **"비상 탈출구"**와 같아요:
- 건물이 불이 났을 때 쓰는 비상문처럼
- 개발 중 급하게 디버깅할 때 쓰는 비밀 통로

위험한 점:
- **나쁜 사람이 문 위치를 알면**? 누구나 들어올 수 있어
- **문이 있는지 모르면**? 보안을 믿다가 해킹당해

그래서 보안 전문가는 **벽을 두드려** 보면서 비밀 문이 없는지 확인하고, **항상 경비**를 서는 거예요!

Ken Thompson은 **"자신이 컴파일러까지 수정해서"** 아무도 못 알아보게 백도어를 만들었죠. "트러스트 트러스트" 논문으로 유명해졌어요.
