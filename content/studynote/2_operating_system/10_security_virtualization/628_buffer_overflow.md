# 버퍼 오버플로우 (Buffer Overflow)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 할당된 메모리 경계를 초과하는 데이터 쓰기로 인해 인접 메모리(반환 주소, 변수)를 덮어쓰는 취약점
> 2. **가치**: 가장 오래되고 치명적인 보안 취약점으로 권한 상승, 코드 실행의 핵심 공격 기법
> 3. **융합**: 메모리 관리, 스택/힙 구조, 보안 기술(DEP, ASLR, Canary)과 직접적 연관

---

### Ⅰ. 개요 (Context & Background)

**개념**
버퍼 오버플로우(Buffer Overflow)는 프로그램이 할당된 버퍼(메모리 공간)의 크기를 초과하여 데이터를 쓸 때 발생하는 취약점이다. 주로 C/C++ 언어의 안전하지 않은 함수(strcpy, gets, sprintf 등)에서 발생하며, 공격자는 이를 악용하여 **반환 주소(Return Address)를 덮어써 원하는 코드를 실행**하거나 **프로그램의 실행 흐름을 조작**할 수 있다.

```

│                         버퍼 오버플로우 기본 원리                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   정상 실행 시                                                              │
│   ────────────────                                                          │

│  │                   │                                                │      │

│   ───────────────────────────                                               │
│   ┌─────────────────────────────────────────────────────────────────┐      │

│  │   ┌───────────────────┐                                           │      │
│  │   │ BBBB             │  ←─ 공격자가 원하는 주소로 덮어씀!        │      │
│  │   ├───────────────────┤                                           │      │
│  │   │ AAAA             │  ←─ EBP 덮어씀                             │      │

│  │   │   (오버플로우)    │                                           │      │

│  │                                                                   │      │
│  │   결과: 함수가 끝날 때 BBBB 주소로 점프!                           │      │
│  │        → 공격자의 쉘코드 실행 또는 원하는 위치로 이동              │      │
│  │                                                                   │      │
│  └───────────────────────────────────────────────────────────────────┘      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**💡 비유**
버퍼 오버플로우는 **물통 넘치기**와 같다.
- 물통: 정해진 크기의 버퍼
- 물: 입력 데이터
- 넘치는 물: 인접한 공간(반환 주소)을 적시는 데이터
- 위험: 넘친 물이 전자제품(반환 주소)에 닿아 합선(악성 코드 실행)

**등장 배경**
① **1970대**: 최초의 버퍼 오버플로우 공격 보고
② **1988년 Morris Worm**: fingerd 취약점 악용
③ **1990대**: 다양한 기법 정립, Shellcode 인젝션
④ **2000대 이후**: 방어 기술(DEP, ASLR, Stack Canary) 등장

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소**

| 요소 | 설명 | 내부 동작 | 공격 기법 | 방어 기술 |
|:---|:---|:---|:---|:---|
| **스택 버퍼** | 함수의 지역 변수 저장 | 고정 크기 배열 | Return Address 덮어쓰기 | Stack Canary |
| **힙 버퍼** | 동적 할당 메모리 | malloc/free 관리 | 함수 포인터 덮어쓰기 | Safe Linking |
| **취약 함수** | 경계 검사 없음 | strcpy, gets, sprintf | 입력 조작 to Overflow | 안전한 함수 사용 |
| **반환 주소** | 복귀 위치 | 스택에 저장 | 쉘코드 주소로 변경 | ASLR |
| **셸코드** | 실행할 악성 코드 | NOP Sled + Payload | 버퍼에 삽입 | DEP/NX Bit |

**해설**
1. **스택 기반**: 가장 흔한 형태, 반환 주소 조작
2. **힙 기반**: 메타데이터 조작으로 임의 위치 쓰기
3. **정수 오버플로우**: 산술 연산 오버플로우로 버퍼 크기 조작
4. **포맷 스트링**: printf 계열 함수의 취약점 악용
5. **ROP**: DEP/NX 우회, 기존 코드 재사용

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 방어 기술별 특징**

| 방어 기술 | 원리 | 보호 대상 | 우회 방법 | 효과 |
|:---|:---|:---|:---|:---|
| **Stack Canary** | 반환 주소 앞에 무작위 값 삽입 | 스택 버퍼 오버플로우 | 정보 누출로 우회 가능 | 높음 |
| **DEP/NX Bit** | 스택/힙 실행 불가 | 셸코드 실행 방지 | ROP로 우회 | 높음 |
| **ASLR** | 주소 공간 랜덤화 | 반환 주소 예측 방지 | 정보 누출 + ROP | 중간 |
| **Safe SEH** | 예외 처리 체인 보호 | SEH 우회 공격 | 메모리 누출 | 중간 |

**과목 융합 관점**

1. **컴퓨터 구조**: 메모리 계층, 스택 프레임 구조 이해 필수

2. **프로그래밍 언어**: C/C++의 메모리 관리, 안전한 함수 선택

3. **운영체제**: 메모리 보호, 가상 주소 공간, 시스템 호출

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나리오 1: 취약점 발견 시 대응**
```
1. 식별: 정적 분석 도구로 취약 코드 탐지
2. 평가: 익스플로이 가능성, 영향 평가
3. 수정: 안전한 함수로 교체 (strcpy → strncpy)
```

- **코드 검증**: Coverity, Fortify 등 정적 분석
- **런타임 보호**: ASLR, DEP 활성화

1. **" strncpy는 안전해"**: 버퍼 크기 확인 안 하면 여전히 취약

4. **"성능이 중요해"**: 안전을 희생할 만큼 성능 이득은 크지 않음
