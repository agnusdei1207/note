# 376. 캐시 인식 데이터 구조 (Cache-aware Data Structures)

> **Insight: 하드웨어의 속도에 맞춘 지능적인 데이터 설계**
> - **캐시 인식 데이터 구조(Cache-aware Data Structures)**는 현대 CPU의 L1/L2/L3 캐시 라인(주로 64바이트) 크기와 계층 구조를 명시적으로 고려하여 데이터 배치와 접근 패턴을 최적화하는 설계 기법입니다.
> - 알고리즘의 복잡도(O-notation)가 낮더라도 캐시 미스가 빈번하면 성능이 급격히 저하되는 'Memory Wall' 문제를 해결하기 위해, 공간 지역성(Spatial Locality)을 극대화합니다.
> - 이는 단순한 소프트웨어 최적화를 넘어, 하드웨어의 물리적 특성을 소프트웨어 아키텍처의 핵심 요소로 편입시키는 현대적 성능 공학의 정수입니다.

---

## Ⅰ. 캐시 인식 설계의 개념 및 필요성

### 1. 정의
- **Cache-aware**: CPU 캐시의 세부 파라미터(캐시 라인 크기, 캐시 용량)를 알고리즘 설계 시 상수로 활용하여 최적의 성능을 내는 방식입니다.

### 2. 필요성
- **Memory Wall**: CPU 연산 속도는 비약적으로 발전한 반면, 메인 메모리(DRAM) 접근 속도는 그에 못 미쳐 발생하는 병목 현상을 해결해야 합니다.
- **Cache Line Utilization**: 메모리에서 데이터를 가져올 때 64바이트 단위로 가져오므로, 이 공간을 얼마나 알차게 쓰느냐가 성능을 결정합니다.

📢 **섹션 요약 비유**: 시장 바구니(캐시 라인)에 물건을 담아올 때, 필요한 물건들을 한데 모아 담아서 시장에 가는 횟수를 최소화하는 것과 같습니다.

---

## Ⅱ. 주요 설계 전략 및 구조

### 1. 데이터 정렬 및 패딩 (Alignment & Padding)
```
[Unoptimized Structure]           [Cache-aware Structure]
+-------+-------+-------+         +-------+-------+-------+
| Flag  | Data  | Pad   |         | Flag  | Data  | (Align) |
+-------+-------+-------+         +-------+-------+-------+
(캐시 라인 경계에 걸침)             (캐시 라인 한 칸에 쏙 들어감)
```

### 2. B-Tree vs B-Tree+ (캐시 친화적 탐색)
- **B-Tree**: 노드 하나가 캐시 라인 크기에 딱 맞도록 설계하여, 한 번의 메모리 로드로 여러 키 값을 동시에 비교할 수 있게 합니다.

### 3. 구조체 배열(AoS) vs 배열 구조체(SoA)
- **AoS (Array of Structures)**: 객체 지향적이지만 특정 필드만 탐색할 때 캐시 효율이 낮음.
- **SoA (Structure of Arrays)**: 동일한 속성끼리 모여 있어 벡터 연산(SIMD) 및 캐시 효율이 극대화됨.

📢 **섹션 요약 비유**: 서랍장(캐시) 칸막이 크기에 딱 맞는 상자들을 준비해서 공간을 낭비 없이 꽉 채우고, 비슷한 물건끼리 한 서랍에 모아두는 정리법입니다.

---

## Ⅲ. 실무 최적화 기법 (C/C++ 기준)

### 1. 캐시 라인 정렬 (Alignment)
```cpp
// 64바이트 캐시 라인 경계에 정렬
struct alignas(64) HighPerformanceData {
    int id;
    float values[15]; // 합쳐서 정확히 64바이트
};
```

### 2. False Sharing 방지
- 서로 다른 스레드가 같은 캐시 라인에 있는 서로 다른 변수를 수정할 때 발생하는 성능 저하를 방지하기 위해 강제로 패딩을 삽입합니다.
```cpp
struct Counter {
    atomic<int> val;
    char padding[60]; // 다른 스레드의 캐시 라인 침범 방지
};
```

📢 **섹션 요약 비유**: 옆 사람과 같은 접시(캐시 라인)를 쓰면 서로 포크가 부딪혀서 불편하니까, 개인 접시를 따로 쓰도록 간격(패딩)을 띄워주는 배려입니다.

## Ⅳ. Cache-oblivious (캐시 망각) 알고리즘과의 비교

| 구분 | Cache-aware (캐시 인식) | Cache-oblivious (캐시 망각) |
| :--- | :--- | :--- |
| **파라미터** | 캐시 크기를 알고 있어야 함 | 캐시 크기 몰라도 최적화됨 |
| **최적화 수준** | 특정 시스템에서 최고 성능 | 모든 시스템에서 준수한 성능 |
| **구현 난이도** | 상대적으로 직관적 | 재귀적 구조 등 수학적 복잡도 높음 |
| **대표 기술** | 정렬된 B-Tree, SIMD 최적화 | Van Emde Boas 레이아웃 |

📢 **섹션 요약 비유**: 캐시 인식은 특정 트럭 크기에 맞춰 짐을 싸는 것이고, 캐시 망각은 어떤 크기의 상자가 오더라도 효율적으로 담기도록 짐을 작게 쪼개서 분할 정복하는 방식입니다.

## Ⅴ. 기술적 이점 및 적용 결과

### 1. 성능 지표 변화
- **Latency**: 메모리 접근 횟수 감소로 인한 지연 시간 단축.
- **Throughput**: 초당 처리 가능한 데이터량 급증 (최대 수십 배 성능 향상 가능).

### 2. 현대적 응용
- 게임 엔진의 ECS (Entity Component System) 아키텍처.
- 인메모리 DB의 컬럼 지향 저장소 최적화.

📢 **섹션 요약 비유**: 똑같은 재료로 요리해도 조리 도구(CPU)의 특성을 잘 아는 요리사가 훨씬 빠르게 맛있는 음식을 내놓는 것과 같습니다.

### 📌 지식 그래프 (Knowledge Graph)
- **Spatial Locality**: 캐시 인식 구조가 활용하는 핵심 원리.
- **False Sharing**: 멀티코어 환경에서 주의해야 할 부작용.
- **SIMD (Single Instruction Multiple Data)**: 캐시 효율과 결합되어 성능을 폭발시키는 기술.
