# 374. 주소 공간 무작위 배치 (ASLR)

> **Insight: 예측 불가능성을 통한 메모리 보안의 요새**
> - **ASLR(Address Space Layout Randomization)**은 프로세스의 주요 데이터 영역(스택, 힙, 라이브러리 등)이 메모리에 로드되는 주소를 매번 무작위로 변경하여, 공격자가 특정 코드나 데이터의 위치를 예측하지 못하게 방어하는 핵심 보안 기술입니다.
> - 이 기술은 버퍼 오버플로우와 같은 메모리 취약점을 이용한 **Return-to-libc** 또는 **ROP(Return-Oriented Programming)** 공격의 성공 확률을 획기적으로 낮춥니다.
> - 완벽한 방패는 아니지만, DEP(데이터 실행 방지)와 결합되어 현대 운영체제의 메모리 보호 시스템에서 가장 기초적이고 강력한 방어선 역할을 수행합니다.

---

## Ⅰ. ASLR의 개념 및 도입 배경

### 1. 정의
- **ASLR**: 프로세스가 실행될 때마다 메모리 주소 공간의 레이아웃을 무작위로 배치하는 운영체제의 보안 기능입니다.

### 2. 도입 배경
- 과거에는 실행 파일과 라이브러리가 항상 고정된 주소에 로드되었습니다.
- 공격자는 시스템 라이브러리(libc)의 특정 함수(예: `system()`) 주소를 미리 알아내어, 버퍼 오버플로우 취약점을 통해 프로그램 흐름을 해당 주소로 돌리는 공격을 수행했습니다.
- 이를 막기 위해 "공격자가 주소를 알 수 없게 만들자"는 철학에서 ASLR이 탄생했습니다.

📢 **섹션 요약 비유**: 보물 지도에 보물의 위치가 항상 (10, 10)으로 적혀 있다면 누구나 찾을 수 있겠지만, 보물 사냥꾼이 올 때마다 보물을 다른 곳으로 옮겨 숨기는 것과 같습니다.

---

## Ⅱ. ASLR의 동작 메커니즘 및 대상

### 1. 무작위 배치 대상 영역
```
[Process Memory Layout with ASLR]
+---------------------------+  <-- High Address
| Stack (Random Offset)     |  -- 매 실행마다 시작 주소 변경
+---------------------------+
|           v               |
:                           :
+---------------------------+
| Shared Libraries (DLL/so) |  -- libc 등의 로드 위치 무작위화
+---------------------------+
:                           :
|           ^               |
+---------------------------+
| Heap (Random Offset)      |  -- 동적 할당 영역의 시작점 변경
+---------------------------+
| Data/BSS (Randomized)     |
+---------------------------+
| Code/Text (PIE 필요)      |  -- Position Independent Executable
+---------------------------+  <-- Low Address
```

### 2. PIE (Position Independent Executable)
- 실행 파일 자체의 코드 영역까지 무작위화하기 위해서는 **PIE**로 컴파일되어야 합니다. 그렇지 않으면 코드 영역은 고정된 주소에 머물게 되어 ASLR의 효과가 반감됩니다.

📢 **섹션 요약 비유**: 집 안의 가구(스택, 힙) 위치만 바꾸는 것이 아니라, 집 자체(코드 영역)의 주소까지 매일 바뀌는 동네를 만드는 것입니다.

## Ⅲ. 실무 적용 및 설정 (Linux 기준)

### 1. 커널 설정 확인 및 제어
```bash
# ASLR 상태 확인 (0: 비활성, 1: 부분 활성, 2: 완전 활성)
cat /proc/sys/kernel/randomize_va_space

# ASLR 활성화 (스택, 라이브러리, 힙 모두 무작위화)
echo 2 > /proc/sys/kernel/randomize_va_space
```

### 2. 프로그램별 PIE 확인
```bash
# checksec 도구를 사용한 확인
checksec --file=/usr/bin/ls
# 결과: PIE enabled 표시 확인
```

📢 **섹션 요약 비유**: 성벽의 높이를 높이는(암호화) 것이 아니라, 성의 입구를 안개 속에 숨겨서 적군이 사다리를 어디에 놓아야 할지 모르게 만드는 설정입니다.

## Ⅳ. ASLR의 한계 및 공격 기법

### 1. Entropy (엔트로피)의 한계
- 32-bit 시스템에서는 주소 공간이 좁아 무작위화할 수 있는 경우의 수가 적습니다. 공격자가 **Brute-force (무차별 대입)** 공격을 통해 주소를 맞출 확률이 존재합니다.

### 2. Information Leak (정보 유출)
- 프로그램의 취약점을 통해 단 하나의 주소값(예: 특정 함수의 실제 주소)만이라도 유출되면, 공격자는 오프셋(Offset) 계산을 통해 다른 모든 함수의 위치를 계산해낼 수 있습니다.

### 3. JIT Spraying
- JIT(Just-In-Time) 컴파일러를 이용해 메모리에 대량의 실행 가능한 코드를 뿌려, ASLR을 무력화하고 실행 권한을 획득하는 기법입니다.

📢 **섹션 요약 비유**: 보물을 매번 다른 곳에 숨겨도, 숨기는 장소 후보가 10군데뿐이라면 언젠가 걸리게 되고, 보물 지도의 한 조각이라도 들키면 전체 위치가 탄로 나는 것과 같습니다.

---

## Ⅴ. 현대적 발전 및 보완 기술

### 1. KASLR (Kernel ASLR)
- 사용자 공간뿐만 아니라 **운영체제 커널** 자체의 로드 주소도 무작위화하여 커널 대상 공격을 방어합니다.

### 2. Fine-grained ASLR
- 함수 단위나 기본 블록(Basic Block) 단위로 더욱 세밀하게 순서를 섞어, 정보 유출이 발생하더라도 오프셋 계산을 불가능하게 만드는 기술입니다.

📢 **섹션 요약 비유**: 이제는 집의 주소만 바꾸는 게 아니라, 집 안의 방 순서와 가구 배치까지 전부 뒤죽박죽으로 섞어놓는 고도의 보안 시스템으로 진화하고 있습니다.

- **Entropy**: ASLR의 보안 강도를 결정짓는 무작위성의 척도.
