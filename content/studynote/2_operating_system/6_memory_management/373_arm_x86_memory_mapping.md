# 373. ARM / x86의 메모리 매핑 아키텍처 차이

> **Insight: 서로 다른 철학이 빚어낸 두 세계의 주소 변환**
> - **x86 아키텍처**는 하위 호환성과 복잡한 CISC 구조를 기반으로 강력한 하드웨어 자동화 기능을 제공하며, 세그먼테이션과 페이징이 결합된 역사적 유산을 현대적 계층 구조로 발전시켰습니다.
> - **ARM 아키텍처**는 전력 효율과 유연한 임베디드 대응을 위해 소프트웨어적인 제어 여지를 더 많이 남겨두었으며, 사용자 공간과 커널 공간을 물리적으로 분리하는 전용 베이스 레지스터 구조를 채택했습니다.
> - 두 아키텍처의 차이를 이해하는 것은 이기종 컴퓨팅 환경에서 메모리 정렬, 배리어 설정, 그리고 커널 포팅을 위한 필수적인 지식입니다.

---

## Ⅰ. 메모리 매핑의 기본 구조 차이

### 1. 하드웨어 vs 소프트웨어 제어 비중
- **x86 (Hardware Centric)**: 페이지 테이블 워크(Page Table Walk)가 하드웨어에 의해 고도로 자동화되어 있으며, OS는 테이블의 시작점(CR3)만 설정하면 됩니다.
- **ARM (Flexible Control)**: 고성능 코어(Cortex-A)는 하드웨어 워크를 지원하지만, 시스템 설정(SCTLR)을 통해 캐시 가능 여부나 메모리 타입(Device, Normal) 등을 매우 정교하게 소프트웨어가 제어해야 합니다.

### 2. 주소 공간 분리 모델
- **x86**: 단일 CR3 레지스터가 전체 가상 주소 공간을 가리키며, 상위 주소(Kernel)와 하위 주소(User) 구분을 페이지 권한 비트로 수행합니다.
- **ARM**: `TTBR0` (Lower VA - User)와 `TTBR1` (Upper VA - Kernel)로 베이스 레지스터가 이원화되어 있어 컨텍스트 스위칭 시 커널 매핑을 유지하기 유리합니다.

📢 **섹션 요약 비유**: x86은 한 권의 커다란 지도책을 쓰면서 뒷부분은 관계자 외 출입금지 표시를 해둔 것이고, ARM은 손님용 지도와 관리자용 지도를 아예 따로 들고 있는 셈입니다.

---

## Ⅱ. 페이지 테이블 계층 및 엔트리 포맷

### 1. 계층 구조 비교 (64-bit 기준)
```
[x86-64: 4-Level Paging]            [ARMv8-A: 4-Level Lookup]
+-----------------------+           +-----------------------+
| PML4 (Page Map L4)    |           | L0 Index (GD)         |
+-----------------------+           +-----------------------+
| PDPT (Dir Pointer)    |           | L1 Index (PUD)        |
+-----------------------+           +-----------------------+
| PD (Page Directory)   |           | L2 Index (PMD)        |
+-----------------------+           +-----------------------+
| PT (Page Table)       |           | L3 Index (PTE)        |
+-----------------------+           +-----------------------+
(Fixed 4KB, 2MB, 1GB)               (Flexible Granule: 4K, 16K, 64K)
```

### 2. 페이지 엔트리(PTE) 속성 비트
- **x86**: `P`(Present), `R/W`, `U/S`(User/Supervisor), `NX`(No-Execute) 등 고전적인 비트 구조.
- **ARM**: `AF`(Access Flag - 소프트웨어 관리 필요), `SH`(Shareability), `AttrIndx`(MAIR 레지스터 참조) 등 임베디드 및 멀티코어 최적화 비트가 발달했습니다.

📢 **섹션 요약 비유**: x86의 주소 카드는 정해진 칸에 체크만 하면 되는 양식이라면, ARM의 주소 카드는 상황에 따라 별도의 규칙표(MAIR)를 보고 내용을 해석해야 하는 커스터마이징 양식입니다.

---

## Ⅲ. 엔디언(Endian) 및 메모리 정렬(Alignment)

### 1. 엔디언 방식
- **x86**: 철저한 **리틀 엔디언(Little-Endian)** 아키텍처입니다.
- **ARM**: **바이 엔디언(Bi-Endian)**을 지원하여 소프트웨어 설정에 따라 리틀 엔디언과 빅 엔디언을 선택할 수 있습니다 (현대 OS는 주로 리틀 엔디언 사용).

### 2. 정렬 제약 (Alignment)
- **x86**: 비정렬 접근(Unaligned Access)을 하드웨어가 비교적 관대하게 처리해주지만 성능 손실이 발생합니다.
- **ARM**: 엄격한 정렬을 요구하는 경우가 많으며, 설정에 따라 비정렬 접근 시 `Alignment Fault` 예외를 발생시킵니다.

📢 **섹션 요약 비유**: x86은 책을 거꾸로 꽂아도 대충 읽어주는 관대한 사서이고, ARM은 책을 정해진 방향과 위치에 딱 맞춰 꽂지 않으면 경고를 울리는 깐깐한 사서입니다.

---

## Ⅳ. 캐시 관리 및 메모리 배리어 (Memory Barrier)

### 1. 캐시 일관성 (Cache Coherency)
- **x86**: 하드웨어가 강력한 캐시 일관성을 보장하며, 쓰기 시 `Strong Ordering`을 선호합니다.
- **ARM**: 약한 메모리 모델(`Weakly Ordered Memory`)을 채택하여 성능을 높이는 대신, 프로그래머가 명시적인 배리어 명령어(`DMB`, `DSB`, `ISB`)를 사용하여 동기화를 제어해야 합니다.

### 2. TLB 관리 방식
- **x86**: CR3 쓰기 시 비전역 TLB가 자동 플러시되는 등의 하드웨어 자동화가 강합니다.
- **ARM**: 특정 주소 범위나 ASID별로 소프트웨어가 명시적으로 `TLBI` 명령을 내려야 하는 경우가 많습니다.

📢 **섹션 요약 비유**: x86은 주방장이 알아서 재료의 신선도를 체크하는 자동 주방이고, ARM은 요리사가 요리 단계마다 "재료 확인했음"이라고 신호를 줘야 사고가 안 나는 수동 주방입니다.

---

## Ⅴ. 가상화 지원 아키텍처 (EPT vs Stage-2)

### 1. x86: EPT (Extended Page Table)
- 게스트 물리 주소(GPA)에서 호스트 물리 주소(HPA)로의 변환을 MMU가 2차 테이블을 통해 직접 수행합니다.

### 2. ARM: Stage-2 Translation
- VTTBR(Virtualization Translation Table Base Register)을 사용하여 게스트와 호스트 사이의 주소 공간을 분리하고 관리합니다.

📢 **섹션 요약 비유**: x86은 손님이 주문하면 주방장이 직접 재료를 가져오는 방식이고, ARM은 서빙 로봇이 재료를 중간 창고까지 가져다주면 주방장이 거기서 다시 요리하는 2단계 구조입니다.

---

### 📌 지식 그래프 (Knowledge Graph)
- **CISC vs RISC**: 두 아키텍처의 철학적 뿌리.
- **Little-Endian**: 공통적으로 사용하는 데이터 저장 방식.
- **Memory Barrier**: ARM에서 특히 중요하게 다뤄지는 동기화 기법.

### 👶 어린이를 위한 3줄 비유 설명
1. x86은 모든 기능이 다 들어있는 '자동 변신 로봇' 같아서 조종하기 쉽지만 덩치가 커요.
2. ARM은 내가 조립하고 설정해야 하는 '레고 블록' 같아서 손이 많이 가지만 에너지를 적게 써요.
3. 길 찾기(메모리 매핑)를 할 때, x86은 내비게이션이 다 해주지만 ARM은 지도를 보며 직접 표지판을 세워야 할 때가 많답니다.
