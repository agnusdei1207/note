# 375. 메모리 보호 키 (Memory Protection Keys)

> **Insight: 사용자 공간에서 휘두르는 정교한 메모리 통제권**
> - **MPK(Memory Protection Keys)**는 CPU 하드웨어 기능을 활용하여, 커널의 개입 없이(시스템 콜 없이) 사용자 공간에서 직접 특정 메모리 페이지 영역의 읽기/쓰기 권한을 실시간으로 제어하는 고성능 메모리 보호 기술입니다.
> - 페이지 테이블의 속성을 바꾸기 위해 발생하던 막대한 성능 저하(TLB 플러시 등)를 해결하며, 프로세스 내부의 보안 격리(Sandboxing)를 극대화할 수 있습니다.
> - 현대의 보안 강화형 라이브러리나 인메모리 데이터베이스에서 데이터의 무단 수정이나 유출을 막는 핵심적인 초고속 방어 기제로 주목받고 있습니다.

---

## Ⅰ. MPK의 개념 및 특징

### 1. 정의
- **Memory Protection Keys (MPK/PKEYS)**: 물리적인 페이지 테이블 엔트리에 '키(Key)'를 부여하고, 스레드별 레지스터 설정을 통해 해당 키를 가진 페이지들에 대한 접근 권한을 즉각적으로 변경하는 기술입니다.

### 2. 주요 특징
- **고속 동작**: 시스템 콜(mprotect 등)을 호출하지 않고, 레지스터 쓰기 한 번으로 권한을 바꿉니다.
- **스레드별 제어**: 동일한 주소 공간을 공유하는 스레드들이라도 서로 다른 권한을 가질 수 있습니다.
- **세분화된 격리**: 최대 16개의 도메인(키)을 정의하여 메모리를 논리적으로 분리할 수 있습니다.

📢 **섹션 요약 비유**: 방의 문을 바꾸는 대신(페이지 테이블 수정), 모든 방문을 열 수 있는 마스터키 레지스터를 내가 들고 있어서 즉시 문을 잠그거나 열 수 있는 것과 같습니다.

---

## Ⅱ. 하드웨어 아키텍처 및 동작 원리

### 1. 페이지 테이블 엔트리(PTE) 확장
```
[x86-64 PTE Layout]
63        51      12 11   9 8       5 4 3 2 1 0
+---------+---------+-----+---------+---------+
| Protection Key (4-bit)  | ... | 속성 비트들 |
+---------+---------+-----+---------+---------+
              |
              v (0~15번 키 중 하나 할당)
```

### 2. PKRU (Protection Key Register User)
- CPU 내부에 위치한 32비트 레지스터입니다.
- 각 키(2비트씩 할당)에 대해 **AD (Access Disable)** 및 **WD (Write Disable)** 비트를 설정합니다.
- **WRPKRU** 명령어로 즉시 갱신됩니다.

📢 **섹션 요약 비유**: 지도는 그대로 두고(페이지 테이블), 내 주머니에 있는 보안 카드(PKRU) 설정만 바꿔서 "지금부터 5번 구역은 출입 금지!"라고 선포하는 것과 같습니다.

---

## Ⅲ. 실무 적용 및 코드 예시 (C/Linux)

### 1. MPK 사용 흐름
1. `pkey_alloc()`: 새로운 키를 할당받습니다.
2. `pkey_mprotect()`: 특정 메모리 영역에 해당 키를 부여합니다.
3. `pkey_set()`: 현재 스레드의 권한 레지스터(PKRU)를 설정합니다.

### 2. 예제 코드
```c
int pkey = pkey_alloc(0, 0); // 키 할당
pkey_mprotect(ptr, size, PROT_READ|PROT_WRITE, pkey); // 영역 지정

// 쓰기 금지 설정 (사용자 공간 명령어)
pkey_set(pkey, PKEY_DISABLE_WRITE); 

// ... 이 시점에서 ptr에 쓰기를 시도하면 SIGSEGV 발생 ...

// 쓰기 허용 설정
pkey_set(pkey, 0); 
```

📢 **섹션 요약 비유**: 복잡한 행정 절차(시스템 콜) 없이, 현장에서 즉시 보안 스티커를 붙이고 떼는 것만큼이나 빠른 속도로 메모리를 보호합니다.

---

## Ⅳ. 주요 활용 사례 및 보안 효과

### 1. In-process Sandboxing
- 브라우저의 렌더링 엔진이나 자바스크립트 가상 머신(VM)에서, 민감한 데이터 영역을 코드 실행 영역으로부터 격리하여 JIT 공격 등을 방어합니다.

### 2. Secure Logging 및 Metadata Protection
- 로그 버퍼나 데이터베이스 인덱스를 평소에는 읽기 전용으로 두었다가, 데이터를 업데이트하는 아주 짧은 순간에만 쓰기 권한을 열어 우발적인 데이터 오염을 방지합니다.

📢 **섹션 요약 비유**: 금고 문을 열어두는 것이 아니라, 돈을 넣을 때만 1초 정도 열었다가 바로 닫는 '순발력 있는 경비원'의 역할을 수행합니다.

---

## Ⅴ. 기술적 한계 및 주의사항

### 1. 키 개수의 제한
- 현재 x86 아키텍처는 최대 16개의 키만 지원하므로, 매우 복잡한 계층 구조를 가진 메모리 격리에는 한계가 있습니다.

### 2. 스펙터(Spectre) 스타일 공격
- 추측 실행(Speculative Execution) 중에 MPK 검사를 우회하여 데이터를 읽어낼 수 있는 취약점이 보고된 바 있어, 소프트웨어적인 보완이 필요합니다.

📢 **섹션 요약 비유**: 열쇠 꾸러미에 열쇠가 16개뿐이라 수천 개의 방을 하나하나 따로 잠글 수는 없으며, 유령(취약점)이 벽을 통과해 훔쳐보는 것까지는 완벽히 막지 못할 수 있습니다.

- **mprotect()**: MPK와 비교되는 전통적인 메모리 보호 시스템 콜.
- **Sandboxing**: MPK의 가장 대표적인 응용 분야.
