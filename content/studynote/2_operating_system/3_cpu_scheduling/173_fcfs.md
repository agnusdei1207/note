# 173. FCFS (선입선출, First-Come First-Served)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 먼저 도착한 프로세스가 먼저 실행
> 2. **가치**: 가장 단순한 스케줄링 알고리즘
> 3. **융합**: 비선점형, 호위 효과, 대기 시간과 연관

---

## Ⅰ. 개요

### 개념 정의

FCFS(First-Come First-Served)는 **준비 큐에 먼저 도착한 프로세스가 먼저 CPU를 할당받는 비선점형 스케줄링 알고리즘**이다. FIFO 큐로 구현된다.

### 💡 비유: 식당 줄서기
FCFS는 **식당 줄서기**와 같다. 먼저 온 사람이 먼저 들어가고, 식사가 끝날 때까지 다음 사람은 기다린다.

### FCFS 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                FCFS 구조                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동작 방식】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  도착 순서              준비 큐 (FIFO)             CPU         │ │
│  │                                                             │ │
│  │  P1 (t=0) ────────►  ┌─────┬─────┬─────┐    ┌───────────┐   │ │
│  │  P2 (t=0)           │ P1  │ P2  │ P3  │───►│  P1 실행   │   │ │
│  │  P3 (t=0)           │ 24ms│  3ms│  3ms│    │  (24ms)    │   │ │
│  │                     └─────┴─────┴─────┘    └───────────┘   │ │
│  │                                                            ▼  │ │
│  │                                          ┌───────────────────┐│ │
│  │                                          │ P1 완료 → P2 실행 ││ │
│  │                                          │ P2 완료 → P3 실행 ││ │
│  │                                          └───────────────────┘│ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【간트 차트 예시】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  P1 (24ms)         P2 (3ms)    P3 (3ms)                       │ │
│  │  ├──────────────────┤├────┤├────┤                             │ │
│  │  0                 24          27      30                     │ │
│  │                                                             │ │
│  │  대기 시간: P1=0, P2=24, P3=27                                │ │
│  │  평균 대기 시간: (0+24+27)/3 = 17ms                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【특징】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 비선점형: 실행 중인 프로세스 종료/I/O까지 계속                │ │
│  │  • FIFO 큐로 구현                                              │ │
│  │  • 구현 가장 간단                                               │ │
│  │  • 호위 효과 발생 가능                                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                FCFS 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【장단점】                                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 구현 간단              • 호위 효과                          │ │
│  │  • 공정 (도착 순서)        • 평균 대기 시간 길다                 │ │
│  │  • 문맥 교환 적음          • I/O 바운드 불리                    │ │
│  │  • 기아 없음              • 대화형 부적합                       │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【호위 효과 (Convoy Effect)】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  긴 프로세스가 먼저 오면:                                        │ │
│  │                                                             │ │
│  │  P1 (24ms)         P2 (3ms)    P3 (3ms)                       │ │
│  │  ├──────────────────┤├────┤├────┤                             │ │
│  │  0                 24          27      30                     │ │
│  │                                                             │ │
│  │  P2, P3는 P1 때문에 긴 대기 → 호위 효과                         │ │
│  │                                                             │ │
│  │  짧은 프로세스가 먼저 오면:                                      │ │
│  │  P2 (3ms)    P3 (3ms)    P1 (24ms)                            │ │
│  │  ├────┤├────┤├──────────────────┤                             │ │
│  │  0     3       6              30                              │ │
│  │                                                             │ │
│  │  평균 대기 시간: (0+3+6)/3 = 3ms → 훨씬 좋음                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【성능 분석】                                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 반환 시간: 도착 순서에 따라 크게 변화                         │ │
│  │  • 대기 시간: 긴 프로세스 뒤에 오면 길어짐                        │ │
│  │  • 처리량: 긴 프로세스 비율에 따라 다름                          │ │
│  │  • 응답 시간: 비선점형이라 대화형 부적합                         │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FCFS 구현 (의사코드)】                                                │
│  ──────────────────                                                  │
│  struct Process {                                                    │
│      int pid;                                                        │
│      int arrival_time;                                               │
│      int burst_time;                                                 │
│      int waiting_time;                                               │
│      int turnaround_time;                                            │
│  };                                                                  │
│                                                                     │
│  void FCFS(Process p[], int n) {                                     │
│      // 도착 시간순 정렬                                              │
│      sortByArrival(p, n);                                            │
│                                                                     │
│      int current_time = 0;                                           │
│      for (int i = 0; i < n; i++) {                                   │
│          if (current_time < p[i].arrival_time)                       │
│              current_time = p[i].arrival_time;                       │
│                                                                     │
│          p[i].waiting_time = current_time - p[i].arrival_time;       │
│          current_time += p[i].burst_time;                            │
│          p[i].turnaround_time = p[i].waiting_time + p[i].burst_time; │
│      }                                                               │
│  }                                                                   │
│                                                                     │
│  【Linux 프로세스 순서 확인】                                            │
│  ──────────────────                                                  │
│  # 프로세스 시작 시간                                                  │
│  $ ps -eo pid,lstart,cmd --sort=lstart                              │
│                                                                     │
│  # FCFS와 유사한 작동 (기본적으로 CFS 사용)                            │
│  $ cat /proc/sched_debug | grep -A 5 "cfs_rq"                       │
│                                                                     │
│  【실제 활용 사례】                                                      │
│  ──────────────────                                                  │
│  // 스레드 풀에서 FIFO 작업 큐                                         │
│  #include <queue>                                                    │
│                                                                     │
│  class FIFOTaskQueue {                                               │
│      std::queue<Task> task_queue;                                    │
│      std::mutex mtx;                                                 │
│  public:                                                             │
│      void addTask(const Task& task) {                                │
│          std::lock_guard<std::mutex> lock(mtx);                      │
│          task_queue.push(task);  // FCFS: 먼저 온 것이 먼저 실행       │
│      }                                                               │
│      Task getNextTask() {                                            │
│          std::lock_guard<std::mutex> lock(mtx);                      │
│          if (task_queue.empty()) return Task();                      │
│          Task task = task_queue.front();                             │
│          task_queue.pop();                                           │
│          return task;                                                │
│      }                                                               │
│  };                                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
