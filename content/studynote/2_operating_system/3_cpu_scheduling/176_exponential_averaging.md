# 지수 평균법 (Exponential Averaging)

## 1. 개요: CPU 버스트 시간의 예측
- 최적 스케줄링인 SJF(Shortest Job First)를 구현하기 위해, 다음 프로세스의 CPU 버스트 시간을 예측하는 통계적 기법
- 과거의 이력 데이터에 가중치를 두어 미래의 시간을 추정함

## 2. 지수 평균법의 수식 및 메커니즘

### 2.1 수식 (Mathematical Representation)
- `τ_{n+1} = α · t_n + (1 - α) · τ_n`
  - **τ_{n+1}:** 다음 CPU 버스트 예측값
  - **t_n:** 가장 최근에 측정한 실제 CPU 버스트 값
  - **τ_n:** 이전까지의 예측값
  - **α:** 가중치 계수 (0 ≤ α ≤ 1)

### 2.2 가중치 설정의 의미 (ASCII)
```text
α = 0 : 과거의 예측값이 곧 다음 예측값 (최근 이력 무시)
α = 1 : 가장 최근의 실제 값이 다음 예측값 (과거 전체 이력 무시)
α = 0.5 : 최근 정보와 과거 이력을 동등하게 반영 (가장 일반적)

[ 가중치 반영 개념 ]
t_n -> t_{n-1} -> t_{n-2} ...
(최근) <----------- (과거)
(높은 가중치) <----- (낮은 가중치)
```

## 3. 지수 평균법의 특징 및 효과

### 3.1 특징
- **지수적 감소:** 과거로 갈수록 가중치가 지수적으로 감소하여 최근 데이터를 더 중요하게 반영함
- **데이터 저장 최적화:** 모든 과거 데이터를 저장할 필요 없이 이전 예측값 하나만 기억하면 됨

### 3.2 효과
- 프로세스의 최근 행태(I/O 바운드 vs CPU 바운드) 변화를 기민하게 반영 가능

## 4. 기술사적 견해: 예측 기반 스케줄링의 한계
- 지수 평균법은 예측의 정확성을 높여주지만, 여전히 미래를 확정적으로 알 수 없는 '불확실성'이 존재함
- 따라서 현대 운영체제는 단순 지수 평균보다는 MLFQ(Multi-Level Feedback Queue)를 통해 실행 중인 프로세스의 행동을 관찰하며 동적으로 우선순위를 조정하는 방식을 병행함
