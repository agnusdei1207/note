# 보장 스케줄링 (Guaranteed Scheduling)

## 1. 보장 스케줄링 (Guaranteed Scheduling)의 정의
- 시스템에 N개의 사용자가 있을 때, 각 사용자에게 CPU 시간의 1/N을 균등하게 할당하겠다고 약속(Guarantee)하는 방식
- 시스템이 약속을 지키기 위해 실제 사용된 CPU 시간을 추적하고 보정함

## 2. 동작 메커니즘

### 2.1 주요 추적 지표
1. **보장된 시간:** (프로세스 생성 이후 경과 시간) / (시스템 총 프로세스 수)
2. **실제 할당된 시간:** 프로세스가 실제 실행된 시간의 누적 합

### 2.2 스케줄링 결정 규칙 (ASCII)
```text
[ 계산식 ]
Ratio = (실제 할당된 시간) / (보장된 시간)

[ 결정 ]
1. Ratio가 가장 작은 프로세스를 다음에 실행 (약속보다 덜 할당받은 프로세스 우선)
2. Ratio가 1에 도달할 때까지 충분히 CPU를 할당함
```

## 3. 특징 및 장점
- **공정성 보장:** 특정 사용자가 시스템 자원을 독점하는 것을 방지함
- **사용자 경험 개선:** 자신의 지분만큼은 확실히 보장받는다는 예측 가능성 제공

## 4. 기술사적 견해: 현대 스케줄러와의 비교
- 보장 스케줄링은 개념적으로 리눅스의 CFS와 유사함 (CFS는 vruntime을 통해 누적 실행 시간을 관리함)
- 단, 단순 1/N 배분보다는 사용자별 가중치(nice값)를 적용한 가중 보장 방식이 실제 시스템 환경에는 더 적합함
