# 188. nice 값 (Nice Value)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스 우선순위 조정을 위한 사용자 레벨 값
> 2. **가치**: CPU 시간 분배 비율 조절
> 3. **융합**: CFS, vruntime, 우선순위 스케줄링과 연관

---

## Ⅰ. 개요

### 개념 정의

nice 값은 **프로세스의 "친절함" 정도를 나타내는 값으로, 다른 프로세스에게 양보할 의향을 표시**한다. -20(가장 높은 우선순위)에서 +19(가장 낮은 우선순위) 범위다.

### 💡 비유: 줄서기 예절
nice 값은 **줄서기 예절**과 같다. nice가 높으면(+값) "먼저 하세요"라고 양보하고, nice가 낮으면(-값) "급해요"라고 먼저 가려고 한다.

## Ⅲ. 실무 적용

### 사용 방법
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【nice 명령어】                                                         │
│  ──────────────────                                                  │
│  # nice 값 10으로 실행 (낮은 우선순위)                                  │
│  $ nice -n 10 ./background_task                                     │
│                                                                     │
│  # nice 값 -5로 실행 (높은 우선순위, root 필요)                         │
│  $ sudo nice -n -5 ./important_task                                 │
│                                                                     │
│  # 기본 nice 값 (0)으로 실행                                           │
│  $ ./normal_task                                                    │
│                                                                     │
│  【renice 명령어】                                                       │
│  ──────────────────                                                  │
│  # 실행 중인 프로세스 nice 값 변경                                      │
│  $ renice -n 5 -p 1234               # PID 1234의 nice를 +5로        │
│  $ renice -n -10 -p 1234             # PID 1234의 nice를 -10로       │
│                                                                     │
│  # 사용자별 변경                                                      │
│  $ renice -n 10 -u username          # 해당 사용자의 모든 프로세스    │
│                                                                     │
│  # 프로세스 그룹별 변경                                                │
│  $ renice -n 5 -g groupname          # 해당 그룹의 모든 프로세스      │
│                                                                     │
│  【nice 값 확인】                                                        │
│  ──────────────────                                                  │
│  # ps로 확인                                                          │
│  $ ps -eo pid,ni,comm               # ni 컬럼이 nice 값              │
│  $ ps -l                                                            │
│                                                                     │
│  # top으로 확인                                                        │
│  $ top                                                              │
│  # NI 컬럼 확인                                                       │
│                                                                     │
│  # 특정 프로세스                                                       │
│  $ cat /proc/<pid>/stat | awk '{print $19}'                         │
│                                                                     │
│  【프로그래밍에서의 nice】                                                │
│  ──────────────────                                                  │
│  #include <sys/resource.h>                                           │
│  #include <unistd.h>                                                 │
│                                                                     │

│  int ret = nice(-5);               // 높은 우선순위                   │

│  ──────────────────                                                  │

│  # 백업 작업                                                          │
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 프로세스 친절함/우선순위 조정 값
• 범위: -20(높음) ~ +19(낮음)
• 기본값: 0
• 효과: nice -1 = CPU 시간 1.25배
• 권한: 일반 사용자는 증가만, root는 감소 가능
• 용도: 백그라운드(+), 실시간/중요(-)
```
