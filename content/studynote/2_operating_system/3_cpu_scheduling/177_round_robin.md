# 177. 라운드 로빈 (RR, Round Robin)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 각 프로세스에 동일한 타임 퀀텀 할당, 순환 실행
> 2. **가치**: 시분할 시스템의 핵심, 공정한 CPU 분배
> 3. **융합**: 선점형, 타임 퀀텀, 응답 시간과 연관

---

## Ⅰ. 개요

### 개념 정의

라운드 로빈(Round Robin)은 **각 프로세스에 동일한 타임 퀀텀(time quantum)을 할당하고, 시간이 만료되면 다음 프로세스로 전환하는 선점형 스케줄링 알고리즘**이다. 시분할 시스템의 기본 알고리즘이다.

### 💡 비유: 카드 게임 턴
라운드 로빈은 **카드 게임 턴**과 같다. 각 플레이어가 동일한 시간(또는 기회)을 갖고, 순서대로 돌아간다.

## Ⅲ. 실무 적용

### 구현 및 설정
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RR 구현 (의사코드)】                                                  │
│  ──────────────────                                                  │
│  struct Process {                                                    │
│      int pid, burst, remaining;                                      │
│  };                                                                  │
│                                                                     │
│  void RoundRobin(Process p[], int n, int quantum) {                  │
│      Queue ready_queue;                                              │
│      int time = 0;                                                   │
│                                                                     │
│      // 모든 프로세스를 큐에 추가                                      │
│      for (int i = 0; i < n; i++) {                                   │
│          ready_queue.push(p[i]);                                     │
│      }                                                               │
│                                                                     │
│      while (!ready_queue.empty()) {                                  │
│          Process current = ready_queue.front();                      │
│          ready_queue.pop();                                          │
│                                                                     │
│          if (current.remaining > quantum) {                          │
│              // 타임 퀀텀만큼 실행                                     │
│              time += quantum;                                        │
│              current.remaining -= quantum;                           │
│              ready_queue.push(current);  // 큐 뒤로                  │
│          } else {                                                    │
│              // 남은 시간 모두 실행 후 완료                            │
│              time += current.remaining;                              │
│              current.remaining = 0;                                  │
│          }                                                           │
│      }                                                               │
│  }                                                                   │
│                                                                     │
│  【Linux 타임 슬라이스 확인】                                            │
│  ──────────────────                                                  │
│  # 최소 타임 슬라이스                                                  │
│  $ cat /proc/sys/kernel/sched_min_granularity_ns                     │
│  2250000    # 2.25ms                                                 │
│                                                                     │
│  # CFS는 동적 타임 슬라이스 사용                                       │
│  # 프로세스 수와 nice 값에 따라 조정                                    │
│                                                                     │
│  【RR 스케줄링 클래스】                                                  │
│  ──────────────────                                                  │
│  # SCHED_RR (실시간 Round Robin)                                      │
│  $ chrt -r -p 50 <pid>                                              │
│                                                                     │
│  # RR 타임 퀀텀 확인                                                   │
│  $ cat /proc/sys/kernel/sched_rr_timeslice_ms                        │
│                                                                     │
│  【성능 측정】                                                           │
│  ──────────────────                                                  │
│  # 문맥 교환 횟수                                                      │
│  $ vmstat 1                                                          │
│  # cs 컬럼                                                            │
│                                                                     │
│  # 응답 시간 측정                                                       │
│  $ latencytop                                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 동일한 타임 퀀텀 할당, 순환 실행
• 유형: 선점형 스케줄링
• 구현: 원형 큐 + 타이머 인터럽트
• 장점: 공정성, 기아 없음, 응답성
• 단점: 문맥 교환 오버헤드
• 타임 퀀텀: 10-100ms 권장
• 적용: 시분할 시스템, 대화형
```
