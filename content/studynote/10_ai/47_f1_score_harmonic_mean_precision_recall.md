# 평가 지표 - F1-Score (정밀도와 재현율의 조화 평균)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: F1-Score는 머신러닝 분류 모델을 평가할 때, 서로 상충하는 관계(Trade-off)에 있는 **정밀도(Precision)**와 **재현율(Recall)**을 곱하고 더하여 산출하는 **조화 평균(Harmonic Mean)** 기반의 단일(Single) 성능 지표다.
> 2. **가치**: 한쪽 지표만 극단적으로 높은 '속임수 모델(예: 전부 다 양성이라고 예측해서 재현율 100% 만들기)'의 점수를 가차 없이 깎아내어, 정밀도와 재현율이 골고루(균형 있게) 높을 때만 높은 점수를 부여하는 매우 보수적이고 신뢰할 수 있는 잣대다.
> 3. **융합**: 정상 데이터가 99%, 사기/불량 데이터가 1%인 **극단적 불균형 데이터(Imbalanced Data)** 환경에서는 정확도(Accuracy)가 99%라도 아무 쓸모가 없으므로, 실무 모델 배포(MLOps) 시에는 무조건 F1-Score를 최우선 기준으로 삼아 알고리즘 성능과 하이퍼파라미터를 평가하고 튜닝한다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: F1-Score는 0부터 1(또는 0%~100%) 사이의 값을 가진다. 1에 가까울수록 오탐지(False Positive)와 미탐지(False Negative)가 모두 적은, 즉 헛발질도 안 하고 놓치는 것도 없는 완벽한 모델임을 뜻한다. 단순 산술 평균(Arithmetic Mean)이 아닌 조화 평균을 사용하는 것이 핵심이다.

- **필요성**: 데이터 사이언티스트가 AI를 만들었는데 정밀도는 10%, 재현율은 100%가 나왔다. 이 두 숫자를 합쳐서 상사에게 "우리 모델 점수는 몇 점입니다"라고 단일 보고서로 제출해야 한다. 만약 그냥 둘을 더해 반으로 나누는 산술평균 `(10+100)/2 = 55점`을 주면, 모델의 심각한 불균형(정밀도 폭망)이 가려진 채 그럭저간 괜찮은 점수로 왜곡된다. 이 꼼수를 막고 둘 중 하나라도 망가졌을 때 전체 점수를 바닥으로 끌어내려 모델의 민낯을 드러낼 강력한 '평균' 공식이 필요했다.

- **💡 비유**: 국영수 시험을 봤는데 어떤 학생이 [국어 100점, 수학 0점]을 맞았고, 다른 학생은 [국어 50점, 수학 50점]을 맞았습니다.
  - 산술 평균(일반 평균)으로 계산하면 두 학생 모두 똑같이 평균 **50점**입니다. (편식하는 학생과 골고루 하는 학생을 구별 못 함)
  - 조화 평균(F1-Score 방식)으로 계산하면, 수학 0점인 첫 번째 학생은 평균 **0점**으로 추락하고, 두 번째 학생은 **50점**을 받습니다. 조화 평균은 과목 편식을 절대 용납하지 않고, 모든 과목(정밀도와 재현율)을 골고루 잘해야만 좋은 점수를 줍니다!

- **등장 배경 및 발전 과정**:
  1. **IR (정보 검색) 분야의 고민**: 1990년대 문서 검색 엔진 개발 시, 사용자가 검색한 결과에 '정확한 문서(Precision)'만 나오는 것과 '관련 문서 전부(Recall)'가 나오는 것 사이의 균형을 측정할 단일 지표가 필요했다.
  2. **F-Measure의 도입**: 통계학자 C.J. van Rijsbergen은 가중치를 두어 조화 평균을 구하는 F-Measure 공식을 제안했다.
  3. **F1-Score의 정착**: 둘 다 똑같은 가중치(1:1)로 중요하게 본다는 의미의 F1-Score (F-beta에서 beta=1)가 현대 머신러닝의 이진 분류 및 불균형 데이터 챌린지의 사실상 표준 평가지표로 확고히 자리 잡았다.

- **📢 섹션 요약 비유**: 오디션에서 노래는 100점인데 춤이 0점인 참가자와, 노래와 춤 모두 80점인 참가자가 있을 때, "가수는 둘 다 잘해야 진짜 아이돌이지!"라며 한쪽만 파고든 참가자를 무자비하게 탈락시키는 가장 엄격한 오디션 심사위원과 같습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 조화 평균의 수학적 원리와 F1-Score 공식

F1-Score는 왜 산술 평균이 아닌 조화 평균을 사용하는가? 그것이 핵심이다.

```text
  ┌───────────────────────────────────────────────────────────────┐
  │         F1-Score의 공식: 산술 평균 vs 조화 평균의 충격적 차이        │
  ├───────────────────────────────────────────────────────────────┤
  │                                                               │
  │  ▶ F1-Score 공식 = 2 * (Precision * Recall) / (Precision + Recall) │
  │                                                               │
  │  [ 상황: 꼼수 부리는 AI 모델 A ]                                    │
  │   - 정밀도(Precision): 0.1 (10%)  <-- 엉뚱한 거 다 잡아서 망함.       │
  │   - 재현율(Recall): 1.0 (100%) <-- 무조건 잡는다고 해서 100% 만점 받음.│
  │                                                               │
  │  [ 1. 만약 산술 평균(일반 평균)을 쓴다면? ]                           │
  │     (0.1 + 1.0) / 2 = 0.55 (55점)                             │
  │     ▶ 결론: "음.. 55점이면 절반은 했네. 나쁘지 않아." (왜곡 발생!)       │
  │                                                               │
  │  [ 2. 조화 평균(F1-Score)을 쓴다면? ]                             │
  │     2 * (0.1 * 1.0) / (0.1 + 1.0) = (0.2) / (1.1) = 0.18 (18점!)│
  │     ▶ 결론: "18점짜리 쓰레기 모델이네! 당장 폐기해!" (진실 폭로!)         │
  │                                                               │
  │  ※ 핵심 원리: 조화 평균은 두 값 중 '더 작은 값' 쪽으로 맹렬하게 끌려 내려가는 │
  │              수학적 성질이 있다. 따라서 밸런스가 무너지면 패널티가 막대하다! │
  └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 산술 평균은 극단적인 값(아웃라이어)에 영향을 많이 받아 둘 중 하나만 만점을 받아도 평균이 쑥 올라간다. 하지만 조화 평균(역수의 평균의 역수)은 두 숫자의 밸런스가 맞을 때 가장 높은 값을 낸다. 예를 들어 둘 다 0.5일 때 F1-Score는 똑같이 0.5지만, 하나가 0.1이고 하나가 0.9이면 0.18로 바닥을 친다. 머신러닝 모델은 임계값(Threshold) 꼼수를 부리면 재현율을 100%로 만들거나 정밀도를 100%로 만들기 매우 쉽다(트레이드오프). F1-Score는 이 꼼수를 원천 차단하고, 진정한 의미의 '균형 잡힌 성능 향상'이 일어났을 때만 점수를 올려주는 방어막 역할을 한다.

---

### F-Beta Score (F1의 확장)

모든 비즈니스가 정밀도와 재현율을 1:1로 동일하게 중요시하지는 않는다. 의료처럼 미탐지(재현율 저하)가 더 위험할 때는 재현율에 가중치를 두는 **F-Beta Score**를 사용한다.

- **F2-Score** (Beta=2): 재현율(Recall)을 정밀도보다 **2배** 더 중요하게 생각하는 조화 평균. (암 진단 등)
- **F0.5-Score** (Beta=0.5): 정밀도(Precision)를 재현율보다 **2배** 더 중요하게 생각하는 조화 평균. (스팸 필터 등)
- **F1-Score** (Beta=1): 둘의 가중치를 똑같이 1:1로 둔다.

## Ⅲ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 극단적 데이터 불균형에서의 평가 오류**: 신용카드 이상 거래 탐지(FDS) 대회를 열었는데, A팀은 Accuracy(정확도) 99% 모델을 냈고, B팀은 Accuracy 90% 모델을 냈다. 경영진은 A팀 모델을 도입하려 한다. (사기 거래 비율은 전체 데이터의 1%다)
   - **판단**: 정상 99%, 사기 1%인 데이터에서 무조건 "정상"이라고 찍은 깡통 모델(A팀)이 정확도 99%를 받은 것이다.
   - **해결책**: 데이터 엔지니어(기술사)는 즉각 A팀 모델과 B팀 모델의 **F1-Score를 대조**하여 경영진에게 보고해야 한다. A팀은 사기 거래를 하나도 못 맞혀 재현율이 0%이므로 F1-Score가 0점이다. 반면 B팀은 정상 거래 오탐이 약간 있어 정확도는 90%로 떨어졌지만, 사기 거래를 70% 찾아내었고 정밀도도 높아 F1-Score가 0.65점이다. B팀 모델만이 실전에 투입 가능한 진정한 모델임을 증명해야 한다.

2. **시나리오 — 다중 클래스(Multi-class) 분류에서의 F1-Score 집계**: 사진을 보고 '고양이, 강아지, 사자, 토끼' 4가지를 분류하는 AI를 만들었다. 그런데 각각의 클래스마다 F1-Score가 따로따로 나오는데, 최종 모델 점수는 1개로 어떻게 뽑아야 할지 의견이 분분한 상황.
   - **판단**: 클래스가 3개 이상일 때는 개별 F1-Score를 평균 내는 방식에 따라 점수가 완전히 달라진다 (Macro vs Micro).
   - **해결책**: 
     - **Macro F1**: 각 클래스의 F1 점수를 그냥 산술 평균 낸다. 클래스별 데이터 개수에 상관없이 모든 동물(소수 클래스 포함)을 공평하게 똑같은 비중으로 대우할 때 쓴다.
     - **Micro F1**: 전체 TP, FP, FN을 몽땅 한 바구니에 다 더한 뒤 F1을 계산한다. 데이터가 압도적으로 많은 클래스(예: 강아지 90%)의 결과에 점수가 크게 휘둘린다. 
     - 데이터 불균형이 심한데 소수 클래스(사자)를 잘 맞히는 게 중요하다면 **Macro F1**을 최종 지표로 선택하는 아키텍처적 결단을 내려야 한다.

### 도입 체크리스트
- **비즈니스적**: 머신러닝 프로젝트 킥오프(Kick-off) 단계에서, 비즈니스 스폰서와 "우리는 정확도(Accuracy)를 버리고, F1-Score를 모델 인수 기준(Acceptance Criteria)으로 삼겠습니다"라고 명확히 합의 및 서명(Sign-off)을 마쳤는가? (나중에 정확도가 왜 이리 낮냐는 태클을 방지하기 위함)

## Ⅳ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | Accuracy(정확도) 단일 평가지표 | F1-Score 평가지표 (불균형 데이터) | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (탐지 신뢰도)**| 불량/사기 클래스 탐지율 0~10% (무시됨) | 꼼수 방지로 인해 실질 타겟 탐지율 확보 | 소수 핵심 데이터의 실질 탐지 성능 **극한 보장** |
| **정량 (임계값 튜닝)**| 임계값 조절 가이드 부재 | F1 점수가 최고점을 찍는 임계값 명확히 도출| AI 모델의 예측 임계값 최적화 시간 **대폭 단축** |
| **정성 (운영 리스크)**| 현장에 쓸 수 없는 깡통 모델 배포 위험 | 성능의 민낯이 드러나 불량 모델 배포 사전 차단 | **MLOps 파이프라인의 배포 승인 신뢰성 100% 확보** |

F1-Score는 데이터의 불균형이 만들어내는 착시 현상을 꿰뚫어 보는 엑스레이(X-Ray)이자, "한 놈만 팬다"는 머신러닝의 꼼수를 용납하지 않는 가장 엄격한 판사다. 기술사는 AI 시스템을 아키텍처링하고 운영할 때 겉보기 화려한 정확도 숫자에 매몰되지 않고, 혼동 행렬의 밑바닥을 긁어올려 만든 F1-Score를 통해 진정으로 비즈니스를 구원할 '균형 잡힌 모델'을 감별해 내는 통찰력을 발휘해야 한다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **정밀도 (Precision) & 재현율 (Recall)** | F1-Score를 탄생시킨 아버지와 어머니. 이 둘은 시소처럼 한쪽이 오르면 한쪽이 내려가는 Trade-off 관계에 있어 조화 평균이 필수다. |
| **혼동 행렬 (Confusion Matrix)** | TP, FP, FN 값을 도출하여 평가지표들의 원자재를 제공하는 2x2 형태의 가장 근원적인 분류 평가 매트릭스다. |
| **조화 평균 (Harmonic Mean)** | 평균을 구하는 수학적 방법의 하나로, 두 숫자 중 작은 숫자에 가중치를 주어 밸런스가 붕괴되었을 때 가차 없이 점수를 깎는 F1의 핵심 원리다. |
| **불균형 데이터 (Imbalanced Data)** | 전체 데이터 중 1% 미만이 정답(타겟)인 악조건의 데이터셋으로, 여기서 깡통 모델의 정확도 사기를 막기 위해 F1-Score가 존재한다. |
| **ROC-AUC** | F1-Score와 함께 불균형 데이터 분류 모델 평가의 양대 산맥으로 쓰이는 곡선 면적 그래프로, 전체 임계값에 대한 성능을 종합 평가한다. |

## 👶 어린이를 위한 3줄 비유 설명
1. 체육대회에서 '달리기'와 '매달리기' 두 종목을 봤어요. A 친구는 달리기 100점, 매달리기 0점을 받았고, B 친구는 둘 다 80점을 받았어요.
2. 둘을 그냥 더해서 반으로 나누면 평균 점수는 비슷해 보이지만, 진짜 훌륭한 체육인은 편식하지 않고 골고루 잘하는 B 친구잖아요?
3. 그래서 한쪽만 극단적으로 잘하고 다른 쪽은 아예 포기해 버린 친구의 점수는 바닥으로 깎아버리고, 골고루 잘한 친구에게만 좋은 점수를 주는 특별하고 공평한 평균 점수 계산법을 'F1-Score'라고 한답니다!
