## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 소프트맥스(Softmax) 함수는 K개의 임의 실수 벡터를 합이 1인 확률 분포로 변환하며, 다중 클래스 분류(Multi-class Classification)의 출력층에서 "각 클래스에 속할 확률"을 반환한다.
> 2. **가치**: 크로스 엔트로피(Cross-Entropy) 손실 함수와 결합할 때 수학적으로 매우 효율적인 기울기를 생성하며, 출력 확률 분포를 해석 가능한 신뢰도(Confidence)로 활용할 수 있다.
> 3. **판단 포인트**: Softmax는 온도 매개변수(Temperature Parameter, T)로 출력의 날카로움(Sharpness)을 조절할 수 있어 지식 증류(Knowledge Distillation)와 샘플링(Sampling)에서 핵심 역할을 한다.

---

## Ⅰ. 개요 및 필요성

### 소프트맥스 함수 정의

```
소프트맥스 수식:

         e^(zᵢ)
yᵢ = ─────────────    (i = 1, 2, ..., K)
       K
      Σ e^(zⱼ)
      j=1

특성:
  - 모든 yᵢ ∈ (0, 1)   : 각 출력은 양수
  - Σ yᵢ = 1            : 모든 출력의 합 = 1 (확률 분포)
  - argmax(z) = argmax(y): 가장 큰 로짓 = 가장 큰 확률
```

### 필요성: 왜 단순 정규화가 아닌가?

단순 정규화 (Linear Normalization):
```
yᵢ = zᵢ / Σzⱼ  ← 음수 처리 불가, 기울기 균일
```

소프트맥스 (지수 함수 활용):
```
yᵢ = e^(zᵢ) / Σe^(zⱼ)  ← 음수 처리 가능, 큰 값 강조, 미분 용이
```

지수 함수 e^z를 사용함으로써:
1. 음수 로짓(Logit, z)도 양수로 변환
2. 큰 값과 작은 값의 차이를 증폭 → 승자 독식(Winner-takes-more) 효과
3. 연속 미분 가능 → 역전파 적용 가능

> 📢 **섹션 요약 비유**: 소프트맥스는 시험 점수를 "반에서 몇 등인지 확률"로 변환하는 것 — 100점은 80점보다 훨씬 높은 1등 확률을 가지고, 모든 학생의 확률 합은 항상 100%가 된다.

## Ⅱ. 아키텍처 및 핵심 원리

### 다중 분류 출력층에서 Softmax 동작

```
┌───────────────────────────────────────────────────────────────────┐
│               Softmax 다중 클래스 분류 흐름                        │
│                                                                   │
│   은닉층 출력    로짓 (Logit, z)    소프트맥스 출력     예측 클래스 │
│                                                                   │
│              z₁ = 2.0  ──► e^2.0 = 7.39                          │
│                                      ÷ 합계          ──► 개 : 70% │
│   [FC Layer] z₂ = 1.0  ──► e^1.0 = 2.72 = 10.52  ──► 고양이: 26%│
│                                                    ──► 새  :  4%  │
│              z₃ = 0.1  ──► e^0.1 = 1.11                          │
│                              합: 11.22                             │
│                                                                   │
│  확인: 7.39/11.22 = 0.659 ≈ 66%                                   │
│        2.72/11.22 = 0.242 ≈ 24%                                   │
│        1.11/11.22 = 0.099 ≈ 10%                                   │
│        합계 = 100% ✓                                               │
└───────────────────────────────────────────────────────────────────┘
```

### 수치 안정성: 오버플로우 방지

e^z 계산 시 z가 크면 오버플로우(Overflow) 발생:

```
z = [1000, 1001, 999] → e^1000 = 수치 폭발!

해결: 최대값 빼기 (Numerically Stable Softmax)
c = max(z)
yᵢ = e^(zᵢ-c) / Σe^(zⱼ-c)

동치 증명: e^(zᵢ-c) / Σe^(zⱼ-c) = e^(zᵢ)/Σe^(zⱼ) (분모분자에 e^(-c) 약분)
```

### 온도 매개변수 (Temperature Parameter)

```
소프트맥스 with 온도:  yᵢ = e^(zᵢ/T) / Σe^(zⱼ/T)

T → 0 (낮은 온도): 분포가 날카로워짐 → argmax에 집중 (결정적)
T = 1 (기본값):    표준 소프트맥스
T → ∞ (높은 온도): 분포가 균등해짐 → 랜덤 샘플링 (불확실)
```

```
┌───────────────────────────────────────────────────┐
│        온도에 따른 소프트맥스 출력 변화             │
│                                                   │
│  로짓: z = [2.0, 1.0, 0.1]                        │
│                                                   │
│  T=0.5 (날카로움):  [0.92, 0.08, 0.00]  ← 확신 강 │
│  T=1.0 (기본):      [0.66, 0.24, 0.10]  ← 기본    │
│  T=2.0 (부드러움):  [0.46, 0.34, 0.20]  ← 불확실  │
│  T=10  (균등화):    [0.36, 0.33, 0.31]  ← 거의 균등│
└───────────────────────────────────────────────────┘
```

### 소프트맥스 기울기 (야코비안 행렬)

```
∂yᵢ/∂zⱼ = yᵢ(1 - yᵢ)   (i = j)
         = -yᵢyⱼ         (i ≠ j)
```

크로스 엔트로피 손실 L = -Σ tᵢ log(yᵢ)와 결합 시:

```
∂L/∂zᵢ = yᵢ - tᵢ  (예측 확률 - 정답 원핫벡터)
```

이 단순한 기울기가 Softmax + Cross-Entropy 조합이 표준이 된 이유다.

> 📢 **섹션 요약 비유**: 소프트맥스는 투표 시스템 — 각 후보(클래스)가 얻은 점수(로짓)를 지수 변환해 모든 표(확률)의 합이 100%가 되게 만들며, 점수 차이가 클수록 표 쏠림 현상이 심해진다.

## Ⅲ. 비교 및 연결

### Softmax vs 관련 개념 비교

| 개념 | 설명 | 차이점 |
|:---|:---|:---|
| **Softmax** | K개 클래스 확률 분포 | 모든 출력 고려한 정규화 |
| **Sigmoid** | 1개 이진 확률 | 독립적 이진 판단 |
| **argmax** | 가장 큰 로짓의 인덱스 | 미분 불가, 확률 아님 |
| **Sparsemax** | Softmax의 희소 버전 | 일부 클래스만 0이 아닌 확률 |
| **Gumbel-Softmax** | 이산 샘플링 미분 가능화 | 강화학습·VAE에서 사용 |

### 소프트맥스와 로지스틱 회귀 관계

이진 분류에서 Softmax(K=2) = Sigmoid:

```
K=2 소프트맥스:
y₁ = e^z₁ / (e^z₁ + e^z₂) = 1 / (1 + e^(z₂-z₁)) = σ(z₁ - z₂) ← Sigmoid
```

따라서 소프트맥스는 로지스틱 회귀(Logistic Regression)의 다중 클래스 일반화다.

> 📢 **섹션 요약 비유**: Softmax와 argmax의 차이는 "1등 뽑기"와 "1등 뽑기 + 점수 차이 알기" — argmax는 우승자만 알고, Softmax는 각 후보가 얼마나 경쟁력 있는지까지 보여준다.

## Ⅳ. 실무 적용 및 기술사 판단

### 기술사 시험 핵심 논점

1. **왜 Softmax인가**: 다중 클래스에서 "모든 클래스에 대한 확률의 합=1" 보장 필요 → 상호 배타적 클래스 분류에 필수
2. **크로스 엔트로피와 결합 이유**: Softmax + CE 기울기 = yᵢ - tᵢ (단순, 수치 안정)
3. **온도 매개변수 활용**: 지식 증류(Knowledge Distillation)에서 교사 모델의 "소프트 레이블"을 높은 온도로 생성
4. **과신뢰(Overconfidence) 문제**: Softmax 출력이 항상 합=1이라 틀려도 높은 확률 → 교정(Calibration) 기법 필요

### 실무 활용 예시

```
이미지 분류 (1,000개 클래스, ImageNet):
로짓 z [1000차원] → Softmax → 확률 분포 → argmax → 예측 클래스

자연어 처리 (다음 단어 예측):
로짓 [어휘 크기=50,000] → Softmax(T) → 확률 분포 → 샘플링 → 다음 토큰

지식 증류 (Knowledge Distillation):
교사 로짓 → Softmax(T=4) → 소프트 레이블 → 학생 모델 학습
```

> 📢 **섹션 요약 비유**: 소프트맥스는 포털 사이트의 "검색 결과 순위" — 검색어와의 관련성(로짓) 점수를 확률로 변환해 각 결과가 "정답일 가능성"을 퍼센트로 보여주는 것과 같다.

## Ⅴ. 기대효과 및 결론

### Softmax 특성 종합

| 특성 | 값 / 설명 |
|:---|:---|
| **수식** | yᵢ = e^(zᵢ) / Σe^(zⱼ) |
| **출력 범위** | 각 yᵢ ∈ (0, 1), Σyᵢ = 1 |
| **클래스 수** | K ≥ 2 (K=2 시 Sigmoid와 동등) |
| **사용 위치** | 다중 분류 출력층 |
| **결합 손실** | 범주형 크로스 엔트로피 |
| **기울기** | yᵢ - tᵢ (CE 결합 시) |
| **온도 조절** | T<1 날카롭게, T>1 부드럽게 |

### 기대효과

| 효과 | 내용 |
|:---|:---|
| **확률 해석** | 모델 출력을 신뢰도(Confidence)로 해석 가능 |
| **다중 클래스** | K개 클래스에 자연스럽게 확장 |
| **최적화 효율** | CE와 결합 시 단순하고 안정적인 기울기 |
| **소프트 레이블** | 온도 조절로 지식 증류·샘플링 활용 |

### 결론

소프트맥스는 다중 클래스 분류의 출력층 표준 활성화 함수로, 임의의 실수 벡터를 확률 분포로 변환하는 핵심 역할을 한다. 크로스 엔트로피 손실과의 결합으로 효율적인 역전파가 가능하며, 온도 매개변수를 통해 지식 증류와 언어 모델 샘플링에서도 폭넓게 활용된다. 기술사 시험에서는 수식, 확률 분포 특성, CE와의 결합 이유, 온도 매개변수가 핵심이다.

> 📢 **섹션 요약 비유**: 소프트맥스는 "파이 나누기" — 어떤 크기의 파이(로짓 벡터)든 K개 조각으로 나누어 모든 조각의 합이 100%가 되게 만들며, 지수 함수로 큰 조각을 더 크게 강조한다.

| 개념 | 연관 키워드 | 관계 |
|:---|:---|:---|

### 👶 어린이를 위한 3줄 비유 설명

🍕 **"피자 나눠 먹기"**

1. 친구들(클래스)에게 점수(로짓)에 따라 피자를 나눠주는데, 모든 친구 몫의 합은 항상 피자 한 판(확률 합=1)이에요.
2. 점수가 제일 높은 친구가 가장 큰 조각을 받고 — 이게 소프트맥스가 "확률이 제일 높은 클래스를 선택"하는 방식이에요.
3. 온도를 높이면 점수 차이를 줄여서 비슷하게 나눠주고, 낮추면 1등이 거의 다 가져가요 — 이게 온도 매개변수예요!
