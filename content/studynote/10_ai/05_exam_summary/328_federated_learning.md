## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 연합 학습 (Federated Learning)은 원시 데이터를 중앙 서버로 전송하지 않고, 각 분산 디바이스(스마트폰, 병원, 기업)에서 로컬로 모델을 학습한 뒤 **가중치 업데이트(Gradient 또는 모델 파라미터)만 서버로 전송**하여 전역 모델을 집계하는 프라이버시 보존 분산 ML 기법이다.
> 2. **가치**: 개인 건강 기록·금융 거래·메시지처럼 중앙 집중화가 불가능하거나 불법인 민감 데이터를 활용하여 AI를 학습할 수 있어, 개인정보보호법(GDPR, PIPA)을 준수하면서도 다수 데이터에서 AI 성능을 확보하는 핵심 프라이버시 기술이다.
> 3. **판단 포인트**: 연합 학습의 핵심 가정인 **데이터 비IID(Non-IID) 문제** — 각 디바이스의 데이터 분포가 서로 달라 국소 학습이 전역 모델을 왜곡하는 현상 — 와 이를 해결하는 FedProx, FedNova 등의 알고리즘이 기술사 설계의 핵심이다.

---

## Ⅰ. 개요 및 필요성

100개 병원의 환자 X선 이미지로 폐암 진단 AI를 학습하고 싶다. 하지만 각 병원의 환자 데이터는 개인정보보호법상 다른 병원과 공유 불가능하다. 전통적 ML은 중앙 서버에 데이터를 모아야 하므로 불가능하다.

**연합 학습(Federated Learning)**은 이 문제를 우아하게 해결한다:
1. 중앙 서버가 초기 전역 모델을 각 병원으로 배포
2. 각 병원은 자체 데이터로 모델을 로컬 학습
3. 학습된 가중치(Gradient) 업데이트만 서버로 전송 (원시 데이터 비전송!)
4. 서버가 모든 병원의 업데이트를 집계하여 전역 모델 갱신
5. 1~4 반복

- **📢 섹션 요약 비유**: 연합 학습은 비밀 레시피 공유 없는 요리 대회다. 100명의 요리사가 각자 집에서 자신만의 비밀 재료(개인 데이터)로 요리를 연습하고, "레시피 발전 방향(가중치 업데이트)"만 주최사에 보낸다. 주최사가 모든 방향을 평균내서 "다음 연습 방향"을 다시 배포한다. 어떤 요리사도 다른 요리사의 비밀 재료를 알 수 없지만, 전체적으로 요리 수준이 높아진다.

## Ⅱ. 아키텍처 및 핵심 원리

```text
┌──────────────────────────────────────────────────────────────────┐
│         연합 학습 (Federated Learning) FedAvg 알고리즘 구조          │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │            중앙 집계 서버 (Aggregation Server)            │   │
│  │            전역 모델 W_global 관리                         │   │
│  └──────┬────────────────────┬──────────────────────────────┘   │
│         │ 전역 모델 배포       │ 가중치 업데이트 수신             │
│    ┌────▼────┐           ┌────▼────┐           ┌────▼────┐      │
│    │ 병원 A  │           │ 병원 B  │    ...     │ 기업 N  │      │
│    │ 로컬학습│           │ 로컬학습│           │ 로컬학습│      │
│    │ ΔW_A   │           │ ΔW_B   │           │ ΔW_N   │      │
│    └─────────┘           └─────────┘           └─────────┘      │
│                                                                  │
│  FedAvg 집계:                                                     │
│  W_global ← Σᵢ (nᵢ/n) × W_local_i  (데이터 수 가중 평균)         │
│                                                                  │
│  Non-IID 문제:                                                    │
│  병원 A (폐암 케이스 많음) ≠ 병원 B (심장 질환 케이스 많음)           │
│  → 국소 학습이 전역 모델에서 멀어져 집계 후 성능 저하 (Drift)          │
│  → FedProx: 국소 손실에 정규화 추가하여 전역 모델 근방에 고정          │
└──────────────────────────────────────────────────────────────────┘
```

| 구성 요소 | 역할 | 핵심 기술 |
|:---|:---|:---|
| 클라이언트 | 로컬 데이터로 모델 학습, 업데이트 전송 | SGD, 국소 에폭 |
| 집계 서버 | 업데이트 수집 및 전역 모델 갱신 | FedAvg, FedProx |
| 보안 집계 | 업데이트 중에도 개별 기여 비노출 | 안전한 다자 계산(SMC) |
| 차등 프라이버시 | 노이즈 추가로 역추론 방지 | DP-SGD |

- **📢 섹션 요약 비유**: FedAvg는 국회 투표 평균이다. 100개 선거구(디바이스) 의원들이 각자 지역 민심(로컬 데이터)을 반영해 투표(로컬 학습)하고, 투표 결과만 국회에 제출(가중치 전송). 국회가 모든 투표를 인구 비례로 집계(FedAvg)하여 전국 정책(전역 모델)을 결정한다. 의원 지역구 민심(원시 데이터)은 외부에 공개 안 된다.

## Ⅲ. 비교 및 연결

**프라이버시 강화 기술 스택**:
1. **연합 학습**: 원시 데이터를 서버로 전송하지 않음
2. **차등 프라이버시 (DP)**: 가중치 업데이트에 노이즈 추가 → 역추론(Model Inversion) 공격 방지
3. **안전한 다자 계산 (SMC)**: 서버가 개별 클라이언트 업데이트를 보지 못하게 암호화 집계
4. **동형 암호 (HE)**: 암호화 상태에서 모델 집계 수행

세 가지를 조합하면 강력한 프라이버시 보존 ML 시스템이 된다.

- **📢 섹션 요약 비유**: 연합 학습 + DP + SMC는 금고 속에서 투표하는 시스템이다. 각 사람이 금고(로컬 디바이스) 안에서 투표하고(연합 학습), 투표지에 무작위 낙서를 추가해(DP) 특정 표가 누구 것인지 모르게 하고, 금고들이 자동으로 합산해서(SMC) 서버에 결과만 전달한다. 완벽한 익명 선거가 AI 학습에 적용된 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

**실제 연합 학습 배포 사례**:
- **Google Gboard**: 스마트폰에서 다음 단어 예측 모델을 연합 학습. 타이핑 내용이 Google 서버에 전송되지 않음
- **Apple iOS**: 이모지 제안, Siri 발음 개선을 연합 학습
- **의료 분야**: NVIDIA FLARE 플랫폼으로 여러 병원의 데이터 없이 AI 학습

**연합 학습 한계**:
- 통신 비용: 다수 라운드의 모델 업데이트 전송 → 네트워크 대역폭 부담
- 계산 비용: 클라이언트 측 학습 연산 필요 → 배터리 소모
- 무임승차(Free Rider): 학습에 기여하지 않고 전역 모델만 사용하는 악의적 클라이언트
- 데이터 포이즈닝 공격: 악의적 클라이언트가 모델을 오염시키는 업데이트 전송

- **📢 섹션 요약 비유**: 연합 학습의 무임승차 문제는 팀 프로젝트의 무임승차 학생과 같다. 다른 학생들이 열심히 기여(로컬 학습)하는 동안, 일부는 기여 없이 결과물(전역 모델)만 가져간다. 인센티브 메커니즘(기여도 기반 보상)과 평판 시스템(기여 이력 추적)으로 이 문제를 완화한다.

## Ⅴ. 기대효과 및 결론

연합 학습은 프라이버시 보존 AI의 핵심 패러다임으로, "데이터 주권(Data Sovereignty)"을 지키면서 AI 성능을 높이는 유일한 방법이다. GDPR·PIPA(개인정보보호법) 강화로 중앙 집중 학습이 어려워지는 세상에서, 연합 학습은 의료·금융·법률·제조 등 규제 산업 AI의 필수 인프라가 될 것이다. EU의 GAIA-X, 미국의 NIST AI 프레임워크, 한국의 마이데이터 정책 모두 연합 학습과 같은 프라이버시 보존 AI 기술의 확산을 지원하고 있다.

- **📢 섹션 요약 비유**: 연합 학습은 AI의 민주적 참여 모델이다. 중앙 빅테크가 모든 데이터를 독점하는 대신, 각 개인·기업·기관이 자신의 데이터 주권을 지키면서도 집단 지성에 기여한다. 데이터 없는 AI 학습이라는 역설을 해결하며 AI 민주화와 프라이버시를 동시에 달성하는 21세기 핵심 기술이다.

### 👶 어린이를 위한 3줄 비유 설명

1. **연합 학습**은 100명의 학생이 각자 집에서 **자기 노트를 바탕으로 공부**하고, "공부 방법의 개선점만" 선생님에게 알려주는 것이에요 — 개인 노트 내용은 절대 공유하지 않아요!
2. 선생님이 모든 학생의 개선점을 합쳐서 **"더 나은 공부법"을 다시 알려주고**, 이 과정을 반복하면 전체 학습 효율이 올라가요.
3. 덕분에 병원들이 **환자 정보 공유 없이** AI 진단을 함께 학습하거나, 스마트폰이 **타이핑 내용을 서버에 보내지 않고** AI를 개선할 수 있어요!
