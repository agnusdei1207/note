## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 모델 양자화 (Quantization)는 신경망 가중치와 활성화값의 수치 정밀도를 FP32(32비트 부동소수점)에서 INT8(8비트 정수) 또는 INT4로 낮춰 모델 크기를 4배 축소하고 추론 속도를 높이는 모델 압축 기법이다.
> 2. **가치**: GPU/NPU가 정수 연산(INT8)을 부동소수점(FP32) 대비 2~4배 빠르게 처리하는 하드웨어 특성을 활용하여, 같은 비용으로 더 많은 추론 요청을 처리하거나 온디바이스(스마트폰, 엣지) 환경에 거대 모델을 배포하는 것이 가능해진다.
> 3. **판단 포인트**: PTQ (Post-Training Quantization, 훈련 후 양자화)는 이미 학습된 모델에 즉시 적용 가능하고, QAT (Quantization-Aware Training, 양자화 인식 학습)는 훈련 중 양자화를 시뮬레이션하여 성능 저하를 최소화하는 더 정교한 방법이다.

---

## Ⅰ. 개요 및 필요성

GPT-3(175B 파라미터)를 FP32로 저장하면 700GB, FP16이면 350GB의 VRAM이 필요하다. 이는 고가의 A100 GPU 8~16장이 필요한 수준이다. **양자화(Quantization)**는 이 거대한 모델의 가중치를 INT8(8비트 정수) 또는 INT4(4비트)로 낮춰 용량을 획기적으로 줄인다.

FP32 → INT8로 변환 시 용량이 4배 축소(700GB → 175GB)되고, INT4로 변환하면 8배 축소(87.5GB)된다. 단, 정밀도가 낮아지므로 정확도 손실(Accuracy Loss)이 발생하며, 이를 최소화하는 것이 양자화 기술의 핵심이다.

- **📢 섹션 요약 비유**: 양자화는 고해상도 사진(FP32, 24MB)을 압축 저장(INT8, 6MB)하는 것이다. 용량은 4배 줄지만 화질(정확도)이 약간 떨어진다. 잘 압축하면 육안으로 거의 차이를 모르고(QAT), 허술하게 압축하면 화질이 눈에 띄게 나빠진다(PTQ 부주의). 화질 손실 없는 압축이 양자화 기술의 목표다.

## Ⅱ. 아키텍처 및 핵심 원리

```text
┌──────────────────────────────────────────────────────────────────┐
│         양자화 (Quantization) 수식 및 변환 구조                      │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  FP32 → INT8 변환 수식:                                           │
│  Q(x) = Round(x / scale + zero_point)                           │
│  Dequantize: x̂ = (Q(x) - zero_point) × scale                   │
│                                                                  │
│  예시: x = 1.234 (FP32)                                          │
│  scale = 0.01, zero_point = 0                                    │
│  Q(1.234) = Round(1.234/0.01) = Round(123.4) = 123 (INT8)       │
│  Dequantize: 123 × 0.01 = 1.23 (≈ 1.234, 정밀도 약간 손실)        │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  정밀도별 표현 범위:                                       │    │
│  │  FP32: ±3.4×10^38, 소수점 7자리 정밀도 (4 bytes/값)      │    │
│  │  FP16: ±65,504,   소수점 3자리 정밀도 (2 bytes/값)        │    │
│  │  BF16: ±3.4×10^38, 소수점 2자리 정밀도 (2 bytes/값)      │    │
│  │  INT8: -128~127,  정수만 표현 가능   (1 byte/값)          │    │
│  │  INT4: -8~7,     정수 16개만 표현   (0.5 bytes/값)        │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
│  PTQ (Post-Training Quantization):                              │
│  학습 완료 모델 → 보정 데이터셋(Calibration) → 스케일 계산 → 양자화   │
│                                                                  │
│  QAT (Quantization-Aware Training):                             │
│  학습 중 양자화 시뮬레이션(Fake Quantization) → 양자화 오류 반영 학습  │
│  순전파: FP32, 역전파: 양자화 오류 기울기 포함                       │
└──────────────────────────────────────────────────────────────────┘
```

| 방법 | 적용 시점 | 성능 | 구현 복잡도 | 사용 사례 |
|:---|:---|:---|:---|:---|
| PTQ (W8A8) | 학습 후 | 중간 | 낮음 | 빠른 배포, 대부분 모델 |
| QAT | 학습 중 | 높음 | 높음 | 엣지 디바이스 고성능 요구 |
| GPTQ | 학습 후, LLM 특화 | 높음 | 중간 | LLM 4비트 양자화 표준 |
| AWQ | 학습 후, 활성화 인식 | 매우 높음 | 중간 | 온디바이스 LLM |

- **📢 섹션 요약 비유**: PTQ는 이미 찍어둔 사진을 나중에 압축 소프트웨어로 변환하는 것이고, QAT는 사진을 찍을 때부터 "이 사진은 나중에 압축될 거야"를 고려해 노출·초점을 최적화하는 것이다. 나중에 압축해도 손실이 최소인 사진이 된다.

## Ⅲ. 비교 및 연결

**QLoRA (Quantized LoRA)**: 기반 모델을 4비트로 양자화하고, 그 위에 16비트 LoRA 어댑터를 추가하는 기법이다. 70B 모델을 단일 48GB GPU에서 파인 튜닝 가능하게 해 오픈소스 LLM 파인 튜닝의 혁명을 일으켰다.

**혼합 정밀도 (Mixed Precision)**: 모든 연산을 같은 정밀도로 하지 않고, 계산(FP16/BF16)과 가중치 업데이트(FP32)를 다른 정밀도로 처리하여 속도와 정밀도를 동시에 확보하는 학습 기법.

- **📢 섹션 요약 비유**: QLoRA는 냉동 창고에 얼린(4비트 양자화) 재료를 보관하면서, 요리(LoRA 파인 튜닝)할 때만 해당 부분을 해동(16비트 LoRA)하는 전략이다. 창고 공간(GPU 메모리)을 최대 절약하면서 요리 품질(파인 튜닝 성능)은 유지한다.

## Ⅳ. 실무 적용 및 기술사 판단

**하드웨어별 최적 양자화 전략**:
- **NVIDIA GPU (Tensor Core)**: INT8, TensorRT INT8 양자화, GPTQ 적용
- **Apple Silicon (Neural Engine)**: ANE에서 INT8 최적화, CoreML 양자화 지원
- **ARM CPU (스마트폰)**: ARM NEON INT8, XNNPACK 양자화 라이브러리
- **전용 NPU**: INT4 지원 칩 (퀄컴 스냅드래곤 NPU 등) 활용

**정확도 손실 허용 기준 (규제 관점)**: 의료·금융 AI에서 양자화로 인한 정확도 저하가 실제 의사 결정에 미치는 영향을 사전 평가하고, EU AI Act 등 규제에서 요구하는 성능 기준을 충족하는지 검증해야 한다.

- **📢 섹션 요약 비유**: 의료 AI 양자화 검증은 비행기 경량화 테스트와 같다. 기체를 10kg 가볍게 만들면(양자화) 연료 효율이 오르지만(속도↑), 구조 강도가 0.1%라도 약해지면 안 된다(의료 정확도 절대 유지). 엔지니어(개발자)는 경량화 전후 성능 차이를 법적 기준에 맞게 문서화해야 한다.

## Ⅴ. 기대효과 및 결론

양자화는 AI 모델의 "다이어트 기술"이다. 거대 언어 모델이 클라우드 서버에서만 동작하던 시대를 끝내고, 스마트폰·엣지 서버·IoT 기기에서도 강력한 AI가 동작하는 온디바이스 AI 시대를 열고 있다. GPTQ, AWQ, LLM.int8() 등 LLM 특화 양자화 방법의 빠른 발전으로 INT4 양자화에서도 FP16과 거의 같은 성능을 달성하는 수준에 이르렀다.

- **📢 섹션 요약 비유**: 양자화는 AI를 창고형 마트(클라우드 서버)에서 편의점(스마트폰)으로 분산시키는 기술이다. 마트의 모든 상품을 편의점 크기에 맞게 소포장하면, 소비자가 창고까지 가지 않아도 집 근처에서 원하는 것을 즉시 살 수 있다. AI의 접근성이 창고에서 편의점 수준으로 민주화된다.
