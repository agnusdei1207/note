## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 트랜스포머의 어텐션 메커니즘은 위치 정보가 없으므로, 포지셔널 인코딩 (Positional Encoding)을 토큰 임베딩에 더해 시퀀스 내 위치 정보를 주입한다.
> 2. **가치**: 사인/코사인 삼각함수 PE (Positional Encoding)는 학습 없이도 작동하며, 훈련보다 긴 시퀀스로 외삽(Extrapolation)이 가능하고, 상대 위치 정보를 내적으로 표현할 수 있다.
> 3. **판단 포인트**: 고정 PE (삼각함수) vs 학습 가능 PE (Learned PE) vs 상대 PE (RoPE, ALiBi) 비교와 각각의 외삽 성능 트레이드오프를 이해하는 것이 핵심이다.

---

## Ⅰ. 개요 및 필요성

트랜스포머는 시퀀스를 병렬로 처리하므로 토큰 순서 정보가 자동으로 반영되지 않는다. CNN은 지역 수용 영역 (Receptive Field), RNN은 순차 처리로 위치 정보를 내재하지만, 어텐션은 집합(Set) 연산과 본질적으로 같다.

이를 해결하기 위해 각 위치 pos에 대한 벡터를 토큰 임베딩에 더한다:
```
입력 = 토큰 임베딩 + 포지셔널 인코딩
```

📢 **섹션 요약 비유**: 포지셔널 인코딩은 줄을 서있는 사람들에게 번호표를 나눠주는 것이다. 트랜스포머는 사람들의 얼굴(임베딩)만 보므로 누가 몇 번째인지 별도로 알려줘야 한다.

---

## Ⅱ. 아키텍처 및 핵심 원리

### 삼각함수 포지셔널 인코딩 수식

```
PE(pos, 2i)   = sin( pos / 10000^(2i/dₘₒₐₑₗ) )
PE(pos, 2i+1) = cos( pos / 10000^(2i/dₘₒₐₑₗ) )

pos  : 시퀀스 내 위치 (0, 1, 2, ...)
i    : 임베딩 차원 인덱스 (0 ~ dₘₒₐₑₗ/2 - 1)
dₘₒₐₑₗ : 임베딩 차원 (예: 512)
```

### 주파수 해석

i가 작을수록 고주파(빠른 변화), i가 클수록 저주파(느린 변화):

```
i=0: sin(pos / 10000⁰) = sin(pos)         ← 주기 2π ≈ 6.28
i=128: sin(pos / 10000^(256/512))          ← 주기 ≈ 2π × 100
i=255: sin(pos / 10000^(510/512))          ← 주기 ≈ 2π × 10000
```

```
┌──────────────────────────────────────────────────────┐
│  PE 행렬 (pos × dₘₒₐₑₗ)                              │
│                                                      │
│  pos=0  [sin₀,  cos₀,  sin₁,  cos₁, ...]            │
│  pos=1  [sin₁,  cos₁,  sin₁', cos₁'...]             │
│  pos=2  [sin₂,  cos₂,  ...]                          │
│  ...                                                 │
│                                                      │
│  낮은 차원: 고주파 (미세 위치 구분)                    │
│  높은 차원: 저주파 (대략적 위치 구분)                  │
└──────────────────────────────────────────────────────┘
```

### 상대 위치 표현 (내적 성질)

```
PE(pos) · PE(pos+k) = f(k)  (pos에 독립적)
→ 두 위치의 내적이 상대 거리 k에만 의존
→ 어텐션 스코어에 상대 위치 정보 암묵적 반영
```

| PE 방식 | 외삽 | 학습 파라미터 | 상대 위치 | 채택 모델 |
|:---|:---|:---|:---|:---|
| 삼각함수 PE | 가능 | 없음 | 암묵적 | 원래 트랜스포머 |
| Learned PE | 어려움 | 있음 (pos×d) | 없음 | BERT, GPT-2 |
| RoPE | 우수 | 없음 | 명시적 | LLaMA, GPT-NeoX |
| ALiBi | 우수 | 없음 | 선형 패널티 | BLOOM |

📢 **섹션 요약 비유**: 삼각함수 PE는 "위치마다 고유한 음악 코드(주파수 조합)"를 부여하는 것이다. 낮은 음(저주파)은 큰 틀의 위치를, 높은 음(고주파)은 세밀한 위치를 표현한다.

---

## Ⅲ. 비교 및 연결

RoPE (Rotary Position Embedding)는 쿼리/키 벡터를 회전 행렬로 변환해 상대 위치를 정확히 반영:
```
RoPE: q' = q · R(pos_q), k' = k · R(pos_k)
q'·k' = q · R(pos_q - pos_k) · kᵀ  ← 상대 위치만 의존
```

ALiBi는 어텐션 스코어에 거리에 비례한 패널티를 직접 더함:
```
Attention score = QKᵀ/√dₖ - m·|pos_q - pos_k|  (m: 헤드별 기울기)
```

📢 **섹션 요약 비유**: RoPE는 "위치마다 다른 각도로 시계를 돌려놓는 것", ALiBi는 "멀리 있는 단어에 거리 벌금을 매기는 것"이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

- BERT-base: 512 위치까지 학습 PE (Learned) → 512 초과 시 성능 저하
- LLaMA 2: RoPE → 4096→32768 컨텍스트 외삽 가능
- RoPE의 NTK-aware 스케일링: 기반 값 10000을 늘려 외삽 성능 향상

기술사 포인트: 삼각함수 PE의 주파수 해석 + 상대 위치 내적 성질 + RoPE 외삽 장점을 연결해 설명하면 고득점 가능.

📢 **섹션 요약 비유**: RoPE의 외삽 능력은 "훈련 때 10km까지만 달렸지만 마라톤 42km도 같은 방식으로 달릴 수 있는" 범용성이다.

---

## Ⅴ. 기대효과 및 결론

포지셔널 인코딩은 트랜스포머가 위치 불변(Position-invariant)인 어텐션의 단점을 보완하는 핵심 설계다. 삼각함수 PE는 우아한 수학적 성질로 상대 위치를 암묵적으로 반영하며, RoPE와 ALiBi는 이를 발전시켜 더 긴 컨텍스트 처리를 가능하게 한다. LLM의 컨텍스트 길이 경쟁에서 PE 설계는 핵심 기술 요소다.

📢 **섹션 요약 비유**: 포지셔널 인코딩은 트랜스포머에게 "시간 감각"을 주는 시계다. 이 시계가 없으면 모든 단어가 동시에 도착한 것처럼 보인다.

### 👶 어린이를 위한 3줄 비유 설명
1. 트랜스포머는 단어들을 동시에 보는데, 어떤 단어가 몇 번째인지 모르니까 번호표(포지셔널 인코딩)를 달아줘야 해.
2. 삼각함수 PE는 위치마다 다른 주파수의 음악 코드를 부여해서, 어떤 두 위치든 고유하게 구별할 수 있게 해.
3. RoPE는 각 위치마다 시계를 다른 각도로 돌려서, 두 단어의 거리(상대 위치)를 자동으로 계산할 수 있게 하는 더 똑똑한 번호표야.
