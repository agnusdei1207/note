## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DTW (Dynamic Time Warping, 동적 시간 워핑)는 두 시계열이 시간축 상에서 속도가 다르더라도 형태적 유사도를 측정하는 비선형 정렬 기법으로, 유클리드 거리의 위상 오정렬 (Phase Misalignment) 한계를 극복한다.
> 2. **가치**: DP (Dynamic Programming) 기반 비용 누적 행렬로 O(n²) 복잡도에 최적 워핑 경로 (Warping Path)를 찾으며, 음성인식·제스처 인식·시계열 분류에 광범위하게 활용된다.
> 3. **판단 포인트**: Sakoe-Chiba Band 등 워핑 윈도우 제약으로 복잡도를 줄이고 과적합 방지가 가능하며, Fast DTW는 선형 근사로 대규모 시계열에 적용 가능하다.

---

## Ⅰ. 개요 및 필요성

유클리드 거리는 두 시계열의 시간 인덱스를 1:1로 매칭한다. 같은 신호가 빨리 발음된 경우와 느리게 발음된 경우는 매우 다른 거리가 계산된다. DTW는 **시간 탄성 (Time Elasticity)**을 허용해 1:다 또는 다:1 매칭을 통해 진짜 유사도를 측정한다.

예시: 두 심전도(ECG) 파형이 심박 주기만 다를 때 DTW는 같은 패턴으로 인식, 유클리드 거리는 다른 파형으로 오인식한다.

📢 **섹션 요약 비유**: DTW는 "노래를 빠르게 부른 버전과 느리게 부른 버전이 같은 노래인지 확인하는 방법"이다. 박자가 달라도 음정 패턴이 같으면 같다고 판단한다.

---

## Ⅱ. 아키텍처 및 핵심 원리

### 비용 누적 행렬 (Cost Accumulated Matrix)

시계열 Q = [q₁, …, qₙ], C = [c₁, …, cₘ]에 대해:

```
d(i,j) = |qᵢ - cⱼ|  (점별 거리, 유클리드)

누적 비용 D(i,j):
D(i,j) = d(i,j) + min{ D(i-1,j), D(i,j-1), D(i-1,j-1) }
```

```
┌─────────────────────────────────────────────┐
│   j→  1    2    3    4    5                  │
│ i↓                                          │
│  1  [0.1] [0.4] [0.9] [1.5] [2.1]           │
│  2  [0.3] [0.2] [0.5] [1.0] [1.6]  ← DP 전진│
│  3  [0.8] [0.4] [0.3] [0.4] [0.9]           │
│  4  [1.2] [0.6] [0.4] [0.2] [0.5]           │
│  5  [1.8] [1.0] [0.7] [0.3] [0.1] ← 최종값  │
│                                              │
│  최적 워핑 경로: D(n,m)에서 역추적 ↙↙←↙     │
└─────────────────────────────────────────────┘
```

### 워핑 경로 제약 (Warping Constraints)

| 제약 | 설명 | 효과 |
|:---|:---|:---|
| Sakoe-Chiba Band | 대각선에서 ±w 이내만 허용 | O(nw) 복잡도 감소 |
| Itakura Parallelogram | 평행사변형 영역 허용 | 물리적 의미 있는 워핑 |
| 제약 없음 | 전체 행렬 탐색 | O(n²) 최대 유연성 |

### DTW 복잡도

- 기본 DTW: O(n²) 시간, O(n²) 공간
- Sakoe-Chiba Band (w): O(nw)
- FastDTW: O(n) 선형 근사

📢 **섹션 요약 비유**: 비용 행렬은 "두 산을 잇는 다리를 놓을 때 가장 낮은 고도를 따라가는 최적 경로 찾기"다. DP는 매 걸음마다 왼쪽·위·대각 중 가장 싼 길을 선택한다.

## Ⅲ. 비교 및 연결

| 거리 | 시간 탄성 | 복잡도 | 특징 |
|:---|:---|:---|:---|
| 유클리드 거리 | ✗ | O(n) | 빠르나 위상 민감 |
| DTW | ✓ | O(n²) | 위상 불변, 느림 |
| EDR (Edit Distance) | ✓ | O(n²) | 노이즈 강인 |
| LCSS | ✓ | O(n²) | 부분 매칭 허용 |

DTW + k-NN (k-최근접 이웃) 조합은 시계열 분류 벤치마크에서 딥러닝과 경쟁하는 강력한 기준선(Baseline)이다.

📢 **섹션 요약 비유**: DTW vs 유클리드 거리는 "음악을 감상할 때 박자까지 맞춰보는 전문가(DTW) vs 그냥 파형만 겹쳐보는 초보(유클리드)"의 차이다.

---

## Ⅳ. 실무 적용 및 기술사 판단

**음성인식**: 발화 속도가 다른 음성 신호 유사도 측정 (DTW 원 용도)
**금융 시계열**: 상관도가 높은 주식 군집화 (DTW 기반 k-Means)
**의료 신호**: ECG 이상 탐지 (정상 템플릿과 DTW 거리 임계값)
**제스처 인식**: 손 동작 분류 (가속도 센서 시계열)

기술사 포인트:
1. DTW는 같은 길이가 아닌 시계열도 비교 가능 → 유클리드 거리의 한계 극복
2. 워핑 경로 제약 없으면 과도한 시간 압축/팽창 → Sakoe-Chiba Band 필수 고려
3. 대규모 데이터셋 → FastDTW 또는 딥러닝 기반 임베딩으로 대체 고려

📢 **섹션 요약 비유**: Sakoe-Chiba Band는 "악보를 2배 빠르게 치는 건 OK, 10배 빠르게 치는 건 다른 곡"이라는 합리적 제한을 거는 것이다.

---

## Ⅴ. 기대효과 및 결론

DTW는 시계열 분석에서 유클리드 거리가 처리하지 못하는 위상 차이 문제를 DP의 최적 부분 구조로 해결한다. FastDTW와 GPU 병렬화로 대규모 데이터에도 적용 가능하며, 딥러닝 시대에도 해석 가능하고 데이터 효율적인 방법으로 여전히 가치 있다.

📢 **섹션 요약 비유**: DTW는 "시간이 고무줄인 세계"의 거리 자다. 잡아 늘려도 형태가 같으면 가깝다고 말해주는 유연한 도구다.
