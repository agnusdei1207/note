## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 배치 정규화(Batch Normalization, BN)는 미니배치의 각 특성을 평균 0, 분산 1로 표준화한 뒤 학습 가능한 스케일(γ)과 이동(β) 파라미터로 재조정하여, 각 층의 입력 분포를 안정화시키는 정규화 기법이다.
> 2. **가치**: 내부 공변량 이동(Internal Covariate Shift) 현상을 억제해 더 큰 학습률 사용을 가능하게 하고, 드롭아웃(Dropout) 없이도 정규화 효과를 내어 깊은 신경망(Deep Network)의 훈련을 극적으로 가속한다.
> 3. **판단 포인트**: 훈련(Training) 시에는 미니배치의 μ_B, σ²_B를 사용하고, 추론(Inference) 시에는 훈련 중 계산한 이동 평균(Moving Average) μ̄, σ̄²를 사용한다.

---

## Ⅰ. 개요 및 필요성

심층 신경망(Deep Neural Network)에서 각 층의 파라미터가 업데이트될 때마다 이전 층의 출력 분포가 변한다. 이를 내부 공변량 이동(ICS, Internal Covariate Shift)이라 한다. ICS가 심하면 학습률을 작게 설정해야 하고 초기화에 민감해져 훈련이 느리고 불안정해진다. BN은 각 미니배치에서 활성화(Activation) 이전 또는 이후에 정규화를 수행해 분포를 안정시킨다.

📢 **섹션 요약 비유**: BN이 없는 깊은 신경망은 "전화 게임(끝말잇기)"이다. 100명이 속삭이면 첫 메시지가 왜곡되어 마지막엔 완전히 다른 메시지가 된다(ICS). BN은 매 10명마다 "원본 메시지를 다시 확인(정규화)"하는 체크포인트다.

---

## Ⅱ. 아키텍처 및 핵심 원리

```
┌──────────────────────────────────────────────────────────┐
│           배치 정규화 (Batch Normalization) 수식          │
├──────────────────────────────────────────────────────────┤
│  미니배치 B = {x₁, ..., xₘ}                             │
│                                                          │
│  1. 배치 평균:  μ_B = (1/m) Σᵢ xᵢ                     │
│  2. 배치 분산:  σ²_B = (1/m) Σᵢ (xᵢ - μ_B)²          │
│  3. 정규화:     x̂ᵢ = (xᵢ - μ_B) / √(σ²_B + ε)       │
│  4. 스케일·이동: yᵢ = γ · x̂ᵢ + β                     │
│     (γ, β: 학습 가능 파라미터)                          │
│                                                          │
│  추론 시:                                               │
│  μ̄ = EMA(μ_B), σ̄² = EMA(σ²_B) 사용                  │
│  (이동 지수 평균, 훈련 중 누적)                         │
└──────────────────────────────────────────────────────────┘
```

| 정규화 방법 | 정규화 축 | 사용 상황 |
|:---|:---|:---|
| Batch Norm (BN) | 배치 × 공간 | CNN, 대배치 |
| Layer Norm (LN) | 특성 차원 | Transformer, NLP |
| Instance Norm | 각 샘플 개별 | 스타일 변환 |
| Group Norm | 특성 그룹 | 소배치, 탐지 |

📢 **섹션 요약 비유**: γ와 β 파라미터는 "표준화 후 원래대로 되돌릴 수 있는 스위치"다. 무조건 평균 0, 분산 1이 최선이 아닐 수 있다. γ와 β가 있으면 네트워크가 "이 층에서는 분산이 2인 분포가 최적"이라는 것을 스스로 학습할 수 있다.

---

## Ⅲ. 비교 및 연결

Layer Normalization(레이어 정규화): 배치 방향이 아닌 특성 차원으로 정규화. 시퀀스 길이가 가변적인 NLP(배치 크기가 작거나 1인 경우)에서 BN은 불안정하지만 LN은 각 샘플 독립적으로 정규화하므로 안정적이다. Transformer 아키텍처는 LN을 기본으로 사용한다. Pre-LN(레이어 전 정규화)과 Post-LN(레이어 후 정규화)의 수렴 특성 차이도 중요한 아키텍처 선택이다.

📢 **섹션 요약 비유**: BN vs LN은 "가로 줄 맞추기 vs 세로 줄 맞추기"다. BN은 배치의 같은 특성을 세로(배치 방향)로 정규화하고, LN은 한 샘플의 모든 특성을 가로(특성 방향)로 정규화한다. CNN은 가로 줄, Transformer는 세로 줄을 선호한다.

---

## Ⅳ. 실무 적용 및 기술사 판단

BN 적용 위치: Conv → BN → ReLU 순서가 일반적이나, 최근 연구에서 Conv → ReLU → BN이 더 나은 경우도 있다. 배치 크기가 너무 작을 때(배치 크기 < 8) BN 통계 추정이 부정확해지므로 Group Norm 또는 Layer Norm으로 대체한다. 전이 학습(Transfer Learning)에서 사전 훈련된 모델의 BN 레이어를 고정(freeze)할지 여부가 성능에 큰 영향을 준다.

📢 **섹션 요약 비유**: 작은 배치에서 BN이 불안정한 것은 "3명을 보고 전 국민 평균 키를 추정"하는 것과 같다. 표본이 너무 작으면 통계가 왜곡된다. Group Norm은 "키를 연령대별로 따로 평균 내는" 접근으로 작은 배치에서도 안정적이다.

---

## Ⅴ. 기대효과 및 결론

BN은 2015년 도입 이후 깊은 신경망 훈련의 표준 도구가 됐다. 더 큰 학습률 허용, 초기화 민감도 감소, Dropout 대체 등으로 훈련 속도와 성능을 동시에 개선한다. 현대 모델에서는 Transformer의 Layer Norm, Vision Transformer의 Pre-LN이 주류가 되었으나, CNN 아키텍처에서 BN은 여전히 기본값이다.

📢 **섹션 요약 비유**: BN은 신경망의 "교정기"다. 층을 거칠수록 신호가 왜곡되는 것을 매 층마다 교정해 신경망이 깊어져도 안정적으로 신호를 전달한다. 이 교정기 덕분에 100층 이상의 초깊은 신경망(ResNet-152)이 처음으로 실용화됐다.
