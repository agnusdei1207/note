## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 코사인 유사도(Cosine Similarity)는 두 벡터 사이의 각도(θ)의 코사인 값으로 방향적 유사성을 측정하며, 벡터의 크기(magnitude)를 정규화하므로 문서 길이에 무관한 텍스트 유사도 측정에 최적화된 거리 척도다.
> 2. **가치**: "AI"와 "인공지능"처럼 의미는 같지만 하나는 2자, 다른 하나는 4자인 경우 TF-IDF 벡터의 L2 정규화 후 코사인 유사도를 계산하면 크기 차이를 무시하고 의미적 방향만 비교해 높은 유사도를 얻는다.
> 3. **판단 포인트**: cos(θ) = (A·B)/(|A||B|) = ΣAᵢBᵢ/√(ΣAᵢ²·ΣBᵢ²) 범위는 [-1, 1]이며, L2 정규화된 벡터 간의 내적(Dot Product)이 코사인 유사도와 동치다.

---

## Ⅰ. 개요 및 필요성

검색 엔진에서 "기계 학습"이라는 쿼리에 가장 유사한 문서를 찾을 때, 짧은 문서(100단어)와 긴 문서(1,000단어)를 유클리드 거리로 비교하면 긴 문서의 단어 빈도가 절대적으로 크므로 불공평하다. 코사인 유사도는 두 벡터의 방향만 비교하므로 문서 길이(크기)에 무관하게 의미적 유사도를 측정한다. Transformer 기반 모델의 임베딩 공간에서 의미론적 유사도 검색(Semantic Similarity Search)의 핵심 지표다.

📢 **섹션 요약 비유**: 코사인 유사도는 "나침반 방향 비교"다. 두 사람이 걷는 거리(벡터 크기)는 달라도, 같은 방향(북쪽)을 향하면 코사인 유사도 = 1(동일 방향)이다. AI는 문서의 "방향성(주제)"만 보고 유사도를 결정한다.

---

## Ⅱ. 아키텍처 및 핵심 원리

```
┌─────────────────────────────────────────────────────────┐
│           코사인 유사도 수식 및 기하학적 의미            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  cos(θ) = (A·B) / (|A|·|B|)                           │
│         = ΣᵢAᵢBᵢ / √(ΣᵢAᵢ²) · √(ΣᵢBᵢ²)             │
│                                                         │
│  θ=0°  → cos=1.0  → 완전 동일 방향 (최고 유사)        │
│  θ=90° → cos=0.0  → 직교 (무관)                       │
│  θ=180°→ cos=-1.0 → 반대 방향 (최저 유사)             │
│                                                         │
│  L2 정규화 후 내적:                                    │
│  â = A/|A|,  b̂ = B/|B|   (단위 벡터)                  │
│  cos(θ) = â · b̂          (정규화된 내적)              │
│                                                         │
│  ANN 검색: FAISS/Milvus에서 코사인 거리 =1-cos(θ)     │
└─────────────────────────────────────────────────────────┘
```

| 유사도/거리 | 수식 | 크기 영향 | 주요 용도 |
|:---|:---|:---|:---|
| 코사인 유사도 | A·B/\|A\|\|B\| | ❌ 무관 | 텍스트, 임베딩 |
| 유클리드 거리 | √Σ(Aᵢ-Bᵢ)² | ✅ 영향 있음 | K-Means, KNN |
| 내적 (Dot Product) | Σ AᵢBᵢ | ✅ 영향 있음 | Attention Score |
| 맨해튼 거리 (L1) | Σ\|Aᵢ-Bᵢ\| | ✅ 영향 있음 | 희소 데이터 |

📢 **섹션 요약 비유**: L2 정규화 후 내적 = 코사인 유사도는 "모든 선수를 같은 체급(단위 벡터)으로 맞춘 뒤 방향 비교"다. 체급(크기)을 통일하면 내적만으로 방향 유사도를 공정하게 측정할 수 있다.

## Ⅲ. 비교 및 연결

PPMI(Positive Pointwise Mutual Information, 양의 점별 상호정보)와 코사인 유사도의 결합: TF-IDF 벡터에 PPMI 가중치를 적용하면 희귀 단어의 공동 등장 패턴을 더 잘 포착한다. 벡터 데이터베이스(Vector Database, Milvus, Pinecone, Weaviate)에서 ANN(Approximate Nearest Neighbor, 근사 최근접 이웃) 검색은 FAISS의 HNSW(Hierarchical Navigable Small World) 인덱스로 수십억 벡터에서 밀리초 내 코사인 유사도 검색을 지원한다.

📢 **섹션 요약 비유**: 벡터 DB에서 코사인 유사도 검색은 "도서관 사서 없는 초고속 도서 추천"이다. 수십억 권의 책(벡터) 중에서 "이 책과 방향이 비슷한 책 10권"을 밀리초 만에 찾아주는 FAISS가 ChatGPT의 RAG(Retrieval-Augmented Generation) 시스템 뒤에서 돌아간다.

---

## Ⅳ. 실무 적용 및 기술사 판단

추천 시스템에서 사용자 임베딩 벡터와 아이템 임베딩 벡터의 코사인 유사도 상위 K개를 추천한다(Item-based CF). Sentence-BERT(SBERT) 같은 문장 임베딩 모델은 코사인 유사도로 의미론적 문장 검색(Semantic Search)을 구현한다. 주의: 이진 특성 데이터(0/1 벡터)에서는 자카드 유사도(Jaccard Similarity)가 더 적합하고, 고차원 희소 데이터(TF-IDF)에서는 코사인이 적합하며 밀집 임베딩(BERT 출력)도 코사인이 적합하다.

📢 **섹션 요약 비유**: 코사인 유사도는 "유튜브 추천 알고리즘의 심장"이다. 내가 본 영상의 임베딩 벡터와 방향이 가장 비슷한 임베딩을 가진 영상 10개를 추천한다. 영상 길이(크기)가 달라도 주제 방향이 비슷하면 추천된다.

---

## Ⅴ. 기대효과 및 결론

코사인 유사도는 자연어 처리(NLP)와 정보 검색(IR) 분야의 표준 유사도 척도로, Transformer 임베딩 공간에서 의미론적 검색의 핵심 수식이다. L2 정규화 벡터에서 내적이 코사인과 동치라는 사실은 GPU에서 행렬 곱으로 수십억 쌍의 유사도를 효율적으로 계산할 수 있게 해준다.

📢 **섹션 요약 비유**: 코사인 유사도는 AI 세계의 "나침반 거리"다. 얼마나 멀리 걸어왔는지(벡터 크기)가 아니라, 같은 방향(주제/의미)을 향하고 있는지만 측정한다. ChatGPT의 RAG부터 유튜브 추천까지, 이 단순한 삼각함수 공식이 현대 AI 검색의 기반이다.
