## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 온도 (Temperature) 파라미터 T는 언어 모델의 로짓 (Logit) 분포를 재조정해 출력의 무작위성을 제어하며, T→0은 결정론적(그리디), T→∞는 균등 분포(완전 무작위)에 수렴한다.
> 2. **가치**: 창의적 글쓰기에는 T=0.8~1.2, 사실 기반 답변에는 T=0.0~0.3을 사용하는 등 태스크에 맞는 온도 설정이 LLM 응용의 품질을 결정한다.
> 3. **판단 포인트**: 온도는 확률 분포의 "뾰족함(sharpness)"을 조절하며, Top-K/Top-P 샘플링과 함께 디코딩 전략을 구성한다.

---

## Ⅰ. 개요 및 필요성

LLM은 각 토큰 위치에서 어휘 전체에 대한 로짓 벡터를 출력한다. 이 로짓을 소프트맥스로 확률로 변환할 때 온도 T를 적용해 분포의 형태를 조절한다.

온도 개념은 열역학에서 온도가 높을수록 입자 운동이 무질서해지는 것과 동일한 직관이다. 통계 역학의 볼츠만 분포 (Boltzmann Distribution)에서 유래했다.

📢 **섹션 요약 비유**: 온도는 주사위의 "공정함 조절기"다. 온도를 낮추면 가장 무거운 면이 항상 나오는 부정 주사위, 온도를 높이면 모든 면이 똑같이 나오는 공정 주사위가 된다.

---

## Ⅱ. 아키텍처 및 핵심 원리

### 온도 적용 소프트맥스

```
P(xₜ = wᵢ | context) = exp(zᵢ / T) / Σⱼ exp(zⱼ / T)

zᵢ : i번째 어휘에 대한 로짓
T  : 온도 파라미터 (T > 0)
```

### T 값에 따른 분포 변화

```
원래 로짓: z = [3.0, 1.0, 0.5, -1.0]  (어휘 4개)

T=0.1 (낮음):  P ≈ [0.998, 0.002, 0.000, 0.000]  ← 거의 결정론적
T=1.0 (기본):  P ≈ [0.780, 0.105, 0.064, 0.051]  ← 표준
T=2.0 (높음):  P ≈ [0.512, 0.236, 0.168, 0.084]  ← 다양성 증가

┌──────────────────────────────────────────────────┐
│  T=0.1  ███████████████████████████████▌         │
│         ▌                                        │
│  T=1.0  ████████████████▌                        │
│              ██▌  █▌ █▌                          │
│  T=2.0  ██████████▌                              │
│              █████▌  ████▌  ██▌                  │
└──────────────────────────────────────────────────┘
```

### T=0 극한: 그리디 디코딩 (Greedy Decoding)

```
T → 0: argmax P(xₜ | context) = argmax zᵢ
→ 항상 가장 높은 로짓 토큰 선택 (결정론적)
```

| T 범위 | 출력 특성 | 적합 태스크 |
|:---|:---|:---|
| 0.0 ~ 0.3 | 결정론적, 반복적 | 코드 생성, 사실 Q&A |
| 0.5 ~ 0.8 | 균형 잡힌 다양성 | 요약, 번역 |
| 0.8 ~ 1.2 | 창의적 | 글쓰기, 브레인스토밍 |
| 1.5 이상 | 일관성 저하, 헛소리 | (권장 안함) |

📢 **섹션 요약 비유**: T=0은 "항상 가장 좋아하는 메뉴를 시키는 사람", T=1.5는 "메뉴판을 눈 감고 찍는 사람"이다.

## Ⅲ. 비교 및 연결

온도는 Top-K, Top-P와 함께 디코딩 파이프라인을 구성한다:

```
로짓 → [온도 적용] → [Top-K 필터] → [Top-P 필터] → 샘플링
```

- 온도 적용 후 Top-K/P 적용: 분포 조정 후 저확률 토큰 제거
- 온도 낮음 + Top-P 1.0: 결정론적에 가까움
- 온도 높음 + Top-P 0.9: 창의적이되 완전한 무작위는 방지

📢 **섹션 요약 비유**: 온도·Top-K·Top-P는 "삼중 필터 커피"다. 온도는 원두 농도, Top-K는 선택 원두 가짓수, Top-P는 사용할 원두 비율을 조절한다.

## Ⅳ. 실무 적용 및 기술사 판단

**OpenAI API**: `temperature` 파라미터 (0~2 범위)
**Hugging Face**: `do_sample=True`, `temperature=0.8`
**온도와 반복 패널티 (Repetition Penalty)**: 높은 온도에서 반복 토큰 억제 추가 필요

기술사 포인트: 온도 수식(로짓/T 후 소프트맥스)과 T→0/T→∞의 극한 동작을 수식으로 설명. 태스크별 권장 범위 제시.

📢 **섹션 요약 비유**: 온도 튜닝은 조리 온도 조절과 같다. 너무 낮으면(낮은 T) 날것으로 먹는 수준, 너무 높으면(높은 T) 타버리는 수준이다.

## Ⅴ. 기대효과 및 결론

온도 파라미터는 단 하나의 스칼라로 LLM의 창의성-신뢰성 트레이드오프를 제어하는 직관적인 도구다. 볼츠만 분포에서 영감을 받은 이 메커니즘은 오늘날 모든 생성 AI 서비스의 핵심 파라미터가 됐다. 올바른 온도 설정은 프롬프트 엔지니어링과 함께 LLM 응용의 품질을 결정한다.

📢 **섹션 요약 비유**: 온도 파라미터는 AI의 "대담함 슬라이더"다. 왼쪽으로 밀면 안전하고 예측 가능한 답, 오른쪽으로 밀면 놀랍고 새로운 아이디어가 나온다.

### 👶 어린이를 위한 3줄 비유 설명
1. 온도는 주사위의 공정함 조절기야. 낮추면 항상 같은 숫자가 나오고, 높이면 어떤 숫자든 골고루 나와.
2. AI가 다음 단어를 고를 때 온도가 낮으면 "가장 그럴듯한 단어만" 고르고, 높으면 "어떤 단어든 도전"해.
3. 정확한 답이 필요하면 온도를 낮추고, 재미있는 이야기를 짓고 싶으면 온도를 높여봐!
