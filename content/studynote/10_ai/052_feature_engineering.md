# 특성 공학 (Feature Engineering) - 데이터의 숨은 패턴을掘り당기는 기술

> ⚠️ 이 문서는 머신러닝 모델의 성능을 결정하는 가장 중요한 단계 중 하나인 '특성 공학'의 개념,/raw data를 모델이 학습하기 쉽게 변환하는 파이프라인, 그리고 도메인 지식 기반의 피처 설계 전략과 자동화 기법들을 심층 분석합니다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 특성 공학은 머신러닝 모델이 데이터 속에서 '핵심 규칙'을 찾는 데 필요한 입력 변수(피처)를 도메인 지식과 통계 기법을 활용하여 설계, 추출, 변형하는 과정이다. Garbage In, Garbage Out(GIGO) 원칙에 따르면, 아무리 강력한 알고리즘이라도 먹히지 않는다고 주장한다.
> 2. **가치**: Kaggle 경진대회 분석 결과, 상위 입상자들의 남들보다 뛰어난 성능의 80% 이상이 하이퍼파라미터 튜닝이 아닌 더 나은 특성 공학에서 비롯된다. 이는 알고리즘 선택보다 데이터 표현(Raw Data → Meaningful Features)의 문제가 압도적으로 중요하다는 실증적 증거다.
> 3. **융합**: 전통적인 통계 기반 수작업 특성 설계에서 AutoML의 자동 피처 생성, 그리고 딥러닝의 자동 피처 학습(Representation Learning)으로 진화하고 있으나, 특정 도메인(의료, 금융)에서는 여전히 인간 전문가의 도메인 지식이 딥러닝을 능역할 수 있다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 1. 데이터가 곧 지혜다: 특성이 성능을 결정하는 원리
"좋은 특성을 가진 평범한 알고리즘이, 나쁜 특성을 가진 좋은 알고리즘보다 항상 이긴다."
- **현대적 함의**: 2010년대 이후 딥러닝의 부상은 "end-to-end 학습"을 통해 수작업 특성 공학을 대체할 수 있다는 점이었다. 그러나 그럼에도 불구하고 특성 공학은 여전히 실무에서 결정적 역할을 한다. 그 이유는 세 가지다:
  1. **제한된 데이터**: 수집 가능한 데이터가 제한적일 때, 기존 데이터로부터 더 많은 정보를 뽑아내는 것이 필수적이다.
  2. **도메인 지식**: 의료, 금융 등 전문가의 지식은 데이터에 내재된 인과 관계를 피처로 변환할 수 있다.
  3. **모델 효율성**: 잘 설계된 특성은 모델의 수렴 속도와 일반화 성능을 비약적으로 향상시킨다.

### 2. 특성 공학의 분류 체계
```
Raw Data (원시 데이터)
  │
  ├─ 피처 선택 (Feature Selection)    ← 불필요한 특성을 제거
  │   ├─ 필터 방법 (Filter): 상관분석, 卡方검정
  │   ├─ 래퍼 방법 (Wrapper): 전진선택법, 후진제거법
  │   └─ 임베디드 방법 (Embedded): L1 규제, 트리 기반 중요도
  │
  ├─ 피처 추출 (Feature Extraction)  ← 기존 특성에서 새로운 특성을 도출
  │   ├─ 차원 축소 (PCA, LDA, t-SNE)
  │   ├─ 텍스트 (TF-IDF, Word2Vec, 임베딩)
  │   └─ 시계열 (移动平均, 차분, 주파수 성분)
  │
  └─ 피처 생성 (Feature Construction) ← 도메인 지식 기반 새 특성 창조
      ├─ 수학적 변환 (로그, 제곱, sqrt)
      ├─ 비법칙 상호작용 (x1 * x2, x1/x2)
      └─ 시간 기반 (년/월/일, 요일, 공휴일)
```

- **📢 섹션 요약 비유**: 특성 공학은 "요리사에게 좋은 재료(原材料)를 주는 것"과 같다. 신선한 야채(데이터)를 아무런 손질 없이(전처리 없음) 내면 실력 좋은 요리사도 요리를 망친다. 야채를 손질하고(특성 추출), 양념을 하고(특성 변환), 재료 간의 궁합을 고려하여 조합하는(특성 상호작용) 과정이 특성 공학이다. 같은 재료로도 요리사의 실력에 따라 미쉐린 스타도 되고,路边摊도 된다.

## Ⅱ. 핵심 아키텍처 및 원리 (Architecture & Mechanism)

### 1. 피처 선택 (Feature Selection) - 불필요한 특성을 제거하는 기술

```text
┌─────────────────────────────────────────────────────────────────────┐
│           [ 피처 선택 전략 비교: 필터 vs 래퍼 vs 임베디드 ]                    │
│                                                                         │
│  ① 필터 방법 (Filter)          ② 래퍼 방법 (Wrapper)        ③ 임베디드 방법 (Embedded)   │
│  ───────────────             ─────────────────            ─────────────────────    │
│                                                                         │
│  [Raw Data]                   [Raw Data]                       [Raw Data]            │
│      │                            │                               │              │
│      ▼                            ▼                               ▼              │
│  통계 테스트                      모델 학습                        모델 학습               │
│  (상관계수, 卡方)                 (전진선택, 후진제거)                  (L1 규제, 트리 중요도) │
│      │                            │                               │              │
│      ▼                            ▼                               ▼              │
│  순위화 → 선택                검색 → 평가 → 반복              자동 선택 (비교)         │
│  속도 빠름, 독립적            정확도 높지만, 계산 비용 ↑         정확도 + 속도 균형       │
│  상관관계만 고려              모델 특정性强                      모델非특化通用            │
└─────────────────────────────────────────────────────────────────────┘
```

#### 피처 선택의 대표적 기법 3가지
1. **전진 선별법 (Forward Selection)**: 빈 모델에서 시작하여, 각 단계에서 가장 성능 향상에 기여하는 특성을 추가. Stopping criterion에 도달할 때까지 반복.
2. **후진 제거법 (Backward Elimination)**: 전체 특성으로 시작하여, 가장 影响가 작은 특성을 하나씩 제거.
3. **랜덤 포레스트 기반 중요도 (Feature Importance)**: 트리 기반 모델의 불순도 감소량을 기반으로 특성 중요도 측정.

### 2. 피처 생성 (Feature Construction) - 도메인 지식으로新특성 만들기

**① 수학적 변환 (Mathematical Transformations)**
```python
# 로그 변환: 왜곡된 분포를 정규분포에 가깝게
df['log_price'] = np.log1p(df['price'])  # price가 극단적으로 치우친 경우

# 제곱근 변환: 분산 안정화
df['sqrt_area'] = np.sqrt(df['area'])

# 박스-칵스 변환: 분포 정규화
from scipy import stats
df['boxcox_area'] = stats.boxcox(df['area'])[0]
```

**② 상호작용 특성 (Interaction Features)**: 두 특성의 곱하거나 나눈 결과를 새 특성으로 생성.
```python
# 도메인 지식 기반 상호작용
df['room_per_floor'] = df['num_rooms'] / (df['floor'] + 1)
df['price_per_area'] = df['price'] / (df['area'] + 1)
```

**③ 시간 기반 특성 (Temporal Features)**: 시계열 데이터에서 시간 패턴을 포착.
```python
df['hour'] = df['timestamp'].dt.hour
df['dayofweek'] = df['timestamp'].dt.dayofweek  # 요일 (0=월, 6=일)
df['is_weekend'] = df['dayofweek'].isin([5, 6]).astype(int)
df['is_holiday'] = df['date'].isin(holidays)
```

- **📢 섹션 요약 비유**: 피처 선택은 "책상 위 불필요한 물건을 버리는 정리"라고 할 수 있다. 피처 추출은 "책을 여러 권 겹쳐서 한 권으로 만드는 압축"이라고 할 수 있다. 피처 생성은 "없던 새 물건(예: 냉장고에 재료 간의 궁합도)을 창조해내는 창작"이다. 이 세 가지가 조화롭게 이루어질 때, 모델은 데이터의 숨은 금을发掘할 수 있다.

## Ⅲ. 비교 및 기술적 트레이드오프 (Comparison & Trade-offs)

### 대표적 차원 축소 기법 비교 (Dimensionality Reduction)

| 기법 | 유형 | 원리 | 강점 | 약점 |
| :--- | :--- | :--- | :--- | :--- |
| **PCA** | 비지도 | 분산이 最大方向으로 투영 | 선형 데이터에 효과적, 해석 용이 | 비선형 구조 포착 불가 |
| **LDA** | 지도 | 클래스 분리 最大方向으로 투영 | 분류 문제에 특화 | 클래스 수-1 차원으로 축소 |
| **t-SNE** | 비지도 | 이웃 간 확률 분포 보존 | 비선형 구조 시각화에 우수 | 계산 비용 높음, 확정적 아님 |
| **UMAP** | 비지도 | 위상 구조 보존, 속도 빠름 | 대규모 데이터 대응, 일관된 결과 | 하이퍼파라미터 민감 |

### AutoML 기반 자동 특성 공학 도구 비교

| 도구 | 자동 특성 생성 | 시간 기반 특성 | 상호작용 특성 | 비선형 변환 |
| :--- | :--- | :--- | :--- | :--- |
| **Featuretools** | Deep Feature Synthesis (DFS) | 자동 파생 | 자동 생성 | 제한적 |
| **Auto-sklearn** | 알고리즘 자동 선택 | 제한적 | 제한적 | 있음 |
| **H2O AutoML** | 자동 피처 처리 | 제한적 | 제한적 | 있음 |
| **TPOT** | 유전 프로그래밍 기반 | 제한적 | 제한적 | 제한적 |

### 기술적 딜레마: 수작업 vs 자동

**수작업 특성 공학의 강점**:
- 도메인 전문가의 지식 활용 가능 (의료, 금융 등)
- 해석 가능성이 높음 (특성의 의미 파악 가능)
- 제한된 데이터에서 효율적

**자동 특성 공학의 강점**:
- 대량 데이터에서 인간의 한계를 초월
- 비선형 관계 자동 포착
- 시간 효율성 (전문가 없이 특성 생성 가능)

- **📢 섹션 요약 비유**: 수작업 특성 공학은 "셰프가 직접 재료를 손질하는 것"이고, 자동 특성 공학은 "전자레인지에 재료를 넣고 버튼 하나를 누르는 것"이다. 전자레인지는 빠르고 편리하지만, 셰프의 감과 경험으로 만들어지는 고품질 요리를 능가하지는 못한다. 결국 최적의 결과는 두 접근법의 적절한 조화에서 나온다.

## Ⅳ. 실무 판단 기준 (Decision Making)

| 고려 사항 | 세부 내용 | 주요 아키텍처 의사결정 |
|:---|:---|:---|
| **도입 환경** | 기존 레거시 시스템과의 호환성 분석 | 마이그레이션 전략 및 단계별 전환 계획 수립 |
| **비용(ROI)** | 초기 구축 비용(CAPEX) 및 운영 비용(OPEX) | TCO 관점의 장기적 효율성 검증 |
| **보안/위험** | 컴플라이언스 준수 및 데이터 무결성 보장 | 제로 트러스트 기반 인증/인가 체계 연계 |

*(추가 실무 적용 가이드 - 금융 신용评估 모델 개발)*
- **상황**: 은행에서 개인신용평가(PCA) 모델을 개발 중이다. 기존 데이터는年收入, 年齢, 就業期間 등 20개의 원시 특성뿐이다.
- **실무 의사결정**:
  1. **도메인 지식 기반 특성 생성**: 신용 전문가의 지식으로 "부채 비율(DTI = 총 부채/연간 소득)", "신용利用率(카드 사용액/카드 한도)", "응답률(납부 예정일 기준 실제 납부 비율)" 등을 계산하는 파생 변수를 생성.
  2. **시계열 특성 추출**: 과거 12개월간의还款 패턴을 기반으로 "평균 연체 횟수", "최장 연체 기간", "최근 3개월 급여 변동성" 등의 시간 축적 특성 생성.
  3. **비선형 변환**:收入分布가 왼쪽으로 치우친 경우, 로그 변환 적용.
  4. **자동 피처 선택**: 인간 전문가가 만든 30개의 특성 중, 랜덤 포레스트 기반 중요도로 상위 15개만 선별하여 모델 단순화 + 과적합 방지.

- **📢 섹션 요약 비유**: 금융 특성 공학은 "신용카드 거래 명세서(원시 데이터)에서 카드사의 수익성과 고객의 상환 능력을 판단하는회계 분석 보고서(의미 있는 특성)를 작성하는 것"과 같다. 거래 금액(원시 데이터)만으로는 알 수 없는 "지출 패턴의 불안정성"이나 "신용 Utilization의 미세한 변화"가 파생 특성을 통해 포착되고, 이것이 부실 예측의 핵심 변수가 된다.

## Ⅴ. 미래 전망 및 발전 방향 (Future Trend)

1. **AutoML 기반 자동 특성 공학의崛起**
   전통적인 수작업 특성 공학은 데이터 과학자의 시간과 도메인 지식에 크게 의존했다. 그러나 AutoML(自动머신러닝)의 발전으로 특성 공학까지 자동화하는の方向으로 진화하고 있다. Google's AutoML Tables, H2O AutoML 등은 자동으로 피처를 생성, 선택, 변환하는 파이프라인을 구축한다. 특히 Featuretools의 Deep Feature Synthesis(DFS)는 관계형 数据库에서 자동으로 피처를 생성하는 기법으로 주목받고 있다.

2. **시각 그래프 신경망(Graph Neural Network) 기반 피처 생성**
   비정형 데이터(그래프, 시퀀스)에서도 특성 공학이 자동화되고 있다. GNN(Graph Neural Network)을 활용하여 노드의 구조적 위치, 이웃 노드와의 관계 등을 자동으로 피처로 변환한다. 이는 소셜 네트워크 분석, 추천 시스템, 약물 발견等领域에서 기존의 수작업 특성 공학을 크게上まわる 성능을 보이고 있다.

- **📢 섹션 요약 비유**: 특성 공학의 미래는 "인간 셰프와人工智能 로봇 셰프의 협력"과 같다. 인간 셰프는食材의 본질적 가치(도메인 지식)를 파악하고, 로봇 셰프는瞬時に 수천 가지의食材 조합(자동 피처 생성)을 시도한다. 둘이 협력할 때, 어떤 셰프도 만들지 못한 미쉐린 별 요리가 탄생할 수 있다.

## 🧠 지식 맵 (Knowledge Graph)

*   **특성 공학 파이프라인**
    *   Raw Data → 피처 선택 → 피처 추출 → 피처 생성 → 모델 학습
*   **대표적 기법**
