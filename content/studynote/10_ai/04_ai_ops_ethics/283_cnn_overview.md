## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CNN(Convolutional Neural Network, 합성곱 신경망)은 합성곱 연산(Convolution)으로 이미지의 **지역적 공간 패턴**을 추출하고, **가중치 공유(Weight Sharing)**와 **이동 불변성(Translation Invariance)**으로 파라미터를 대폭 줄이면서 이미지 분류·탐지·분할에 탁월한 성능을 발휘한다.
> 2. **가치**: FC(Fully Connected Layer) 대비 가중치 수를 수천 배 이상 줄이고, 이미지 내 위치가 달라도 동일한 특성(엣지, 텍스처, 객체 부분)을 인식하는 이동 불변성으로 이미지 인식의 혁명을 이끌었다.
> 3. **판단 포인트**: 기술사 시험에서 합성곱의 출력 크기 계산 공식, 풀링의 역할, 특성 맵(Feature Map)의 의미, 그리고 LeNet→AlexNet→VGG→ResNet의 발전 계보를 묻는 문제가 출제된다.

---

## Ⅰ. 개요 및 필요성

### FC 레이어의 한계

전통적인 완전 연결층(FC, Fully Connected Layer)은 이미지를 1차원 벡터로 펼쳐 처리한다. 224×224×3 이미지라면 **150,528개의 입력 노드**가 필요하고, 첫 은닉층에 1,000개 뉴런만 있어도 **1억 5천만 개의 가중치**가 필요해 학습 불가능에 가깝다.

또한 FC는 **공간 구조(Spatial Structure)를 무시**한다. 이미지에서 근접한 픽셀들은 높은 상관관계를 가지는데, FC는 이를 전혀 활용하지 못한다.

CNN은 다음 핵심 아이디어로 이를 해결한다:
1. **지역 수용 영역(Local Receptive Field)**: 한 번에 작은 영역(예: 3×3)만 처리
2. **가중치 공유(Weight Sharing)**: 동일한 필터를 이미지 전체에 반복 적용
3. **풀링(Pooling)**: 공간 해상도 축소로 이동 불변성 확보

📢 **섹션 요약 비유**: CNN은 이미지를 한꺼번에 전체를 보는 것이 아니라, 돋보기로 작은 부분씩 훑으며(합성곱) 패턴을 찾고, 같은 돋보기(가중치 공유)를 이미지 전체에 재사용하는 효율적인 탐정 방법이다.

## Ⅱ. 아키텍처 및 핵심 원리

### CNN 전체 아키텍처

```
입력 이미지 (H × W × C)
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│  합성곱 블록 (Conv Block)                                    │
│  ┌────────────────────────────────────────────────────────┐  │
│  │ Conv(3×3, F filters) → BN → ReLU                      │  │
│  │ 출력: H' × W' × F  (특성 맵, Feature Map)             │  │
│  └────────────────────────────────────────────────────────┘  │
│                  × 여러 회 반복                              │
├──────────────────────────────────────────────────────────────┤
│  풀링 레이어 (Pooling Layer)                                 │
│  MaxPooling(2×2, stride=2) → H/2 × W/2 × F (공간 축소)     │
├──────────────────────────────────────────────────────────────┤
│  깊어질수록: 공간 크기(H,W)↓, 채널 수(F)↑                 │
│  저수준 특성(엣지) → 중수준(텍스처) → 고수준(객체 부분)   │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
Flatten → FC Layer → Softmax → 분류 결과
```

### 합성곱 연산(Convolution)

```
입력 (5×5)         필터 (3×3)         출력 특성 맵 (3×3)
┌─────────────┐   ┌─────────┐        ┌─────────┐
│ 1  2  3  0  1│  │ 1  0 -1 │        │  ●  ●  ● │
│ 4  5  6  1  2│× │ 1  0 -1 │   →    │  ●  ●  ● │
│ 7  8  9  2  1│  │ 1  0 -1 │        │  ●  ●  ● │
│ 0  1  2  3  0│  └─────────┘        └─────────┘
│ 1  0  1  2  3│
└─────────────┘
출력 크기 = (N - F + 2P) / S + 1
  N: 입력 크기, F: 필터 크기, P: 패딩, S: 스트라이드
예) (5 - 3 + 0) / 1 + 1 = 3
```

### 출력 크기 계산 공식

| 파라미터 | 의미 | 예시 |
|:---|:---|:---:|
| N (Input Size) | 입력 특성 맵의 크기 | 32 |
| F (Filter Size) | 필터(커널)의 크기 | 3 |
| P (Padding) | 입력 주변 패딩 크기 | 1 |
| S (Stride) | 필터 이동 간격 | 1 |
| **출력 크기** | **(N - F + 2P) / S + 1** | **(32-3+2)/1+1=32** |

패딩 P=1로 Same Padding을 적용하면 입력과 출력 크기가 같다 (32→32).

### 풀링(Pooling) 레이어

```
Max Pooling (2×2, stride=2):
┌───────────────────────────────────┐
│  입력 4×4  →  출력 2×2           │
│  ┌───┬───┐                        │
│  │ 1  3  │  max→ 9               │
│  │ 2  9  │                        │
│  ├───┼───┤                        │
│  │ 5  2  │  max→ 7               │
│  │ 7  1  │                        │
│  └───┴───┘                        │
│  이동 불변성: 패턴이 조금 이동해도 │
│  동일한 최대값이 출력됨            │
└───────────────────────────────────┘
```

### 대표 CNN 아키텍처 계보

| 아키텍처 | 연도 | 핵심 기여 | 파라미터 수 |
|:---|:---:|:---|:---:|
| LeNet-5 | 1998 | CNN의 시작, 필기체 인식 | ~60K |
| AlexNet | 2012 | ReLU, Dropout, GPU 학습 | ~62M |
| VGGNet | 2014 | 3×3 필터 깊은 스택 | ~138M |
| GoogLeNet | 2014 | Inception 모듈, 1×1 Conv | ~6.8M |
| ResNet | 2015 | 잔차 연결(Skip Connection) | ~25M(50층) |
| EfficientNet | 2019 | 복합 스케일링(Width/Depth/Resolution) | ~5.3M(B0) |

📢 **섹션 요약 비유**: CNN의 발전은 도시가 성장하는 것과 같다. LeNet이 작은 마을이라면, AlexNet은 고속도로(GPU 병렬), ResNet은 다리(Skip Connection)로 교통 체증(그래디언트 소실)을 해결한 현대 도시다.

## Ⅲ. 비교 및 연결

### 가중치 공유(Weight Sharing)의 의미

FC 레이어: 각 연결마다 별도의 가중치 → 파라미터 수 = 입력 × 출력  
CNN: 동일한 필터가 전체 이미지를 슬라이딩 → 파라미터 수 = F × F × C_in × C_out

예) 224×224×3 입력, 첫 레이어:
- FC: 150,528 × 1,000 = **1억 5천만 개**
- CNN (3×3×3 필터, 64개): 3×3×3×64 = **1,728개** (87,000배 절감)

### 이동 불변성(Translation Invariance) vs 이동 등변성(Translation Equivariance)

- **합성곱 레이어**: **이동 등변성(Equivariance)** - 입력이 이동하면 특성 맵도 같이 이동
- **풀링 레이어**: **이동 불변성(Invariance)** - 입력이 조금 이동해도 출력이 동일

두 개념의 결합으로 "고양이가 이미지의 어느 위치에 있든 고양이로 인식"이 가능하다.

### CNN vs Transformer

| 항목 | CNN | Transformer (ViT) |
|:---|:---|:---|
| 귀납적 편향 | 지역성, 이동 불변성 (강함) | 없음 (데이터로 학습) |
| 장거리 의존성 | 깊은 레이어 필요 | Self-Attention으로 직접 |
| 데이터 효율성 | 적은 데이터에서 우수 | 많은 데이터 필요 |
| 현재 트렌드 | 하이브리드 (ConvNeXt) | ViT, Swin Transformer |

📢 **섹션 요약 비유**: CNN은 오랜 경험(귀납적 편향)이 쌓인 전문 화가가 그림의 부분부분을 보며 그리는 방식이다. Transformer는 그림 전체를 한눈에 보는 능력이 있지만 처음엔 경험이 없어 많은 그림을 봐야 한다. 최신 아키텍처는 두 방법을 결합한다.

## Ⅳ. 실무 적용 및 기술사 판단

### 기술사 시험 판단 포인트

1. **출력 크기 계산**: (N - F + 2P) / S + 1 공식 암기
2. **가중치 공유 효과**: 파라미터 수 대폭 절감 이유
3. **풀링의 역할**: 공간 크기 축소 + 이동 불변성 + 과적합 방지
4. **ResNet의 잔차 연결**: 그래디언트 소실 방지, H(x) = F(x) + x

### 잔차 연결(Residual Connection, Skip Connection)

```
입력 x
  │   └──────────────────────────┐
  ▼                              │ (지름길)
Conv → BN → ReLU → Conv → BN    │
  ▼                              │
  (+)◄─────────────────────────-┘
  ▼
ReLU → 출력 H(x) = F(x) + x
```

잔차 연결로 그래디언트가 곱셈 없이 덧셈으로 직접 전달 → 100층 이상의 네트워크 학습 가능.

### 주요 응용 분야

- **이미지 분류(Image Classification)**: ResNet, EfficientNet
- **객체 탐지(Object Detection)**: YOLO, Faster R-CNN (CNN 백본 사용)
- **이미지 분할(Image Segmentation)**: U-Net (인코더-디코더, Skip Connection)
- **의료 영상**: X-ray 진단, MRI 분석 (소량 데이터 + 전이 학습)

📢 **섹션 요약 비유**: ResNet의 잔차 연결은 100층짜리 건물에 엘리베이터를 설치하는 것과 같다. 계단(순방향 합성곱)으로만 올라가면 너무 힘들어 그래디언트가 사라지지만(그래디언트 소실), 엘리베이터(잔차 경로)로 정보가 직접 전달된다.

## Ⅴ. 기대효과 및 결론

CNN이 이미지 인식 혁명을 이끈 이유:

1. **파라미터 효율성**: 가중치 공유로 FC 대비 수만 배 파라미터 절감
2. **이동 불변성**: 객체 위치에 무관한 강건한 인식
3. **계층적 특성 추출**: 엣지 → 텍스처 → 부분 → 객체의 계층적 표현 학습
4. **전이 학습(Transfer Learning)**: ImageNet 사전 학습 모델을 다양한 도메인에 적용

AlexNet이 2012년 ImageNet에서 압도적 1위를 기록한 이후, CNN은 컴퓨터 비전(Computer Vision)의 표준이 되었다. 현재는 Vision Transformer(ViT)와의 하이브리드 형태(ConvNeXt, Swin Transformer 등)로 발전하고 있다.

📢 **섹션 요약 비유**: CNN은 동물의 시각 피질처럼, 간단한 엣지(V1)에서 복잡한 형태(상위 피질)까지 계층적으로 처리하는 생물학적 영감을 받은 구조다. 자연의 설계를 모방했기 때문에 이미지 인식에서 이렇게 강력한 성능을 보이는 것이다.

### 📌 관련 개념 맵

| 개념 | 연관 키워드 | 관계 |
|:---|:---|:---|
| CNN (Conv Net) | 합성곱, 풀링, 특성 맵 | 이미지 처리 특화 신경망 |
| 합성곱 (Convolution) | 필터, 스트라이드, 패딩 | 지역 패턴 추출 연산 |
| 가중치 공유 (Weight Sharing) | 파라미터 절감, 필터 재사용 | CNN의 핵심 효율화 기법 |
| 특성 맵 (Feature Map) | 채널, 활성화, 응답 | 합성곱 출력 표현 |
| 풀링 (Pooling) | MaxPool, 이동 불변성 | 공간 축소 + 이동 불변성 |
| 이동 불변성 (Translation Invariance) | 풀링, 위치 무관 인식 | CNN의 핵심 특성 |
| ResNet | 잔차 연결, Skip Connection | 깊은 네트워크 그래디언트 소실 해결 |
| 전이 학습 (Transfer Learning) | ImageNet, 사전 학습 | CNN 사전 학습 모델 재활용 |

### 👶 어린이를 위한 3줄 비유 설명

- CNN은 그림의 작은 부분씩 같은 돋보기로 훑어보며 패턴을 찾는 방법이에요. 같은 돋보기를 모든 곳에 쓰니까 가중치를 아주 적게 쓸 수 있어요.
- 풀링은 "어디쯤에 고양이가 있네"를 알면 정확히 어느 픽셀인지 몰라도 되는 것처럼, 위치가 조금 달라도 같은 패턴으로 인식하게 해줘요.
- ResNet은 100층짜리 건물에 엘리베이터를 설치한 것처럼, 정보가 직접 뛰어넘어 전달되어 아무리 깊어도 잘 학습할 수 있어요.
