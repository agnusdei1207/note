## 핵심 인사이트 (3줄 요약)
> 1. **본질**: XAI (eXplainable AI, 설명 가능한 AI)는 블랙박스(Black-Box) AI 모델의 예측 근거를 인간이 이해 가능한 방식으로 설명하는 기술로, "왜 이런 예측을 했는가"에 답함으로써 AI 시스템의 신뢰성·공정성·책임성을 확보하는 연구·기술 분야다.
> 2. **가치**: EU AI Act, 금융권 신용 평가 규제, 의료 AI 허가 기준 등에서 고위험 AI에 대한 설명 의무가 법제화되어, XAI는 이제 AI 비즈니스 컴플라이언스의 핵심 요소이며 AI에 대한 사회적 신뢰 구축의 기술적 기반이다.
> 3. **판단 포인트**: XAI 방법론은 **모델 특화(Model-Specific)** vs **모델 불가지론적(Model-Agnostic)**, **전역적(Global)** vs **국소적(Local)** 설명으로 분류하며, 실무에서는 모델 불가지론적 국소 설명(LIME, SHAP)이 어떤 모델에도 적용 가능해 가장 널리 사용된다.

---

## Ⅰ. 개요 및 필요성

"당신의 주택 담보 대출이 거절됐습니다." AI가 이 결정을 내렸는데 왜 거절됐는지 설명하지 못한다면, 해당 고객은 부당한 차별에 대응할 권리를 박탈당한 것이다. 딥러닝 블랙박스가 도처에 배치되는 시대에 **XAI**는 법적·윤리적 필수 요건이 됐다.

XAI는 세 가지 이해관계자에게 서로 다른 수준의 설명을 제공한다:
- **모델 개발자**: 어떤 특징이 모델에 얼마나 영향을 주는가 (피처 중요도)
- **비즈니스 의사결정자**: 왜 이 예측이 합리적인가 (직관적 근거)
- **최종 사용자**: 왜 나에 대해 이런 결정을 내렸는가 (개인 설명)

- **📢 섹션 요약 비유**: XAI 없는 AI는 이유를 모르고 결정을 내리는 판사다. "피고인 유죄"라고 판결했지만 "왜 유죄인가"를 설명하지 못한다면 법치주의(AI 거버넌스) 위반이다. XAI는 AI 판사에게 "판결문 작성 의무"를 부여한다 — 증거(특징), 논리(모델 추론), 결론(예측)을 명확히 서술해야 한다.

## Ⅱ. 아키텍처 및 핵심 원리

```text
┌──────────────────────────────────────────────────────────────────┐
│         XAI 방법론 분류 체계 및 주요 기법                            │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  분류 기준 1: 설명 범위                                             │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  전역적(Global): 모델 전체 행동 설명                      │    │
│  │  예: 피처 중요도, PDP(부분 의존성 그래프), 결정 트리 요약   │    │
│  │  vs.                                                    │    │
│  │  국소적(Local): 특정 예측 하나에 대한 설명               │    │
│  │  예: LIME, SHAP 값, 반사실적 설명(Counterfactual)        │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
│  분류 기준 2: 모델 의존도                                           │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  모델 특화(Model-Specific):                              │    │
│  │  - 의사결정 트리: 규칙 자체가 설명 (화이트박스)            │    │
│  │  - 선형 회귀: 계수가 특징 중요도                          │    │
│  │  - CNN 어텐션 맵: 그래디언트 기반 시각화(Grad-CAM)        │    │
│  │  vs.                                                    │    │
│  │  모델 불가지론적(Model-Agnostic):                         │    │
│  │  - LIME: 로컬 서로게이트 모델                             │    │
│  │  - SHAP: 샤플리 값 기반 공정한 기여도                     │    │
│  │  - PFI(Permutation Feature Importance): 특징 순열 교란  │    │
│  └─────────────────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────────────────┘
```

| XAI 기법 | 설명 범위 | 모델 의존 | 특징 | 대표 도구 |
|:---|:---|:---|:---|:---|
| SHAP | 국소/전역 | 불가지론 | 공정한 기여도 분배 | shap 라이브러리 |
| LIME | 국소 | 불가지론 | 국소 대리 모델 | lime 라이브러리 |
| Grad-CAM | 국소 (이미지) | CNN 특화 | 히트맵 시각화 | PyTorch hooks |
| PDP | 전역 | 불가지론 | 특징-예측 관계 시각화 | scikit-learn |
| ICE | 국소/전역 | 불가지론 | 개별 예측 의존성 | scikit-learn |

- **📢 섹션 요약 비유**: Grad-CAM (Gradient-weighted Class Activation Mapping)은 AI가 사진을 보고 "고양이"라고 분류할 때, "어느 부분을 보고 그 결론을 내렸나"를 열지도(Heatmap)로 표시하는 것이다. 귀 부분이 빨갛게(활성화) 표시되면 AI가 귀를 보고 고양이라고 판단했다는 것이다. 사람이 "맞아, 저 귀가 고양이 귀야"라고 직관적으로 검증할 수 있다.

## Ⅲ. 비교 및 연결

**공정성 (Fairness)와 XAI의 관계**: AI 모델이 특정 인종·성별·나이에 대해 편향된 예측을 할 때, XAI 도구가 어떤 특징이 편향의 원인인지 밝혀낼 수 있다. SHAP 값으로 "이 모델은 성별 특징이 신용 평가에 과도하게 영향을 준다"는 것을 발견하고 수정할 수 있다.

**EU AI Act XAI 요구사항**: 고위험 AI 시스템(의료, 신용, 고용, 법집행)은 사용자에게 설명 제공 의무. AI 생성 결정에 대한 인간 검토 권리 보장. 투명성(Transparency) 원칙이 법적 의무로 강제됐다.

- **📢 섹션 요약 비유**: XAI와 AI 공정성의 관계는 법정의 증거 개시(Discovery) 원칙이다. 재판에서 원고는 피고가 어떤 근거로 결정을 내렸는지 알 권리가 있다. AI가 대출을 거절했다면, 고객은 어떤 정보(특징)가 어떻게 영향을 미쳤는지(SHAP 값) 알 권리가 있다. 이를 거부하면 법적 제재를 받는다.

## Ⅳ. 실무 적용 및 기술사 판단

**XAI 구현 단계**:
1. **탐색적 XAI**: 모델 개발 중 어떤 특징이 중요한지 확인(전역 SHAP)
2. **디버깅 XAI**: 잘못된 예측 케이스 분석(국소 LIME/SHAP)
3. **거버넌스 XAI**: 규제 감사 대응용 설명 자동 생성 파이프라인
4. **사용자 XAI**: 최종 고객에게 "왜 이 추천을 했는가" 자연어 설명 제공

**한국 금융권 적용**: 2023년 금융위원회 "AI 활용 금융서비스 가이드라인"에서 신용 평가 AI는 거절 이유를 구체적으로 설명해야 한다고 명시. SHAP 기반 상위 3개 부정적 요인을 자연어 메시지로 제공하는 구현이 표준화되고 있다.

- **📢 섹션 요약 비유**: "대출 거절 이유 3가지(SHAP 상위 3 음의 기여 특징)"를 고객에게 제공하는 것은 의사가 "혈압이 높아서, 당뇨 가족력이 있어서, 체중이 과다해서 보험 가입이 어렵습니다"라고 구체적 이유를 말하는 것이다. 이유 없는 거절("AI가 거절함")은 의료 과실(AI 법 위반)에 해당한다.

## Ⅴ. 기대효과 및 결론

XAI는 AI가 사회에 수용되기 위한 신뢰 기반 기술이다. "왜"라는 질문에 답하지 못하는 AI는 규제와 사회 저항에 부딪혀 배포가 제한된다. SHAP, LIME의 등장으로 어떤 복잡한 블랙박스 모델도 사후적으로 설명 가능하게 됐으며, Grad-CAM이 의료 영상 AI의 임상 신뢰를 높이고 있다. XAI는 AI 성능과 AI 신뢰의 교차점으로, 기술사가 AI 시스템 설계 시 반드시 고려해야 할 필수 요소다.

- **📢 섹션 요약 비유**: XAI는 AI의 "양심 회로"다. 고성능 AI가 양심(설명)이 없으면 전능한 독재자가 된다. XAI는 AI의 모든 결정에 "왜 이 결정을 내렸는가"를 반드시 설명하게 하는 민주적 제어 장치로, AI와 인간 사회의 공존을 위한 기술적 계약서다.

| 개념 | 연관 키워드 | 관계 |
|:---|:---|:---|
| SHAP | 샤플리 값, 공정한 기여도 | XAI의 가장 수학적으로 엄격한 방법 |
| LIME | 국소 대리 모델, 모델 불가지론 | XAI의 직관적 국소 설명 방법 |

| EU AI Act | 고위험 AI, 설명 의무 | XAI 법제화의 핵심 규제 프레임워크 |

### 👶 어린이를 위한 3줄 비유 설명

1. **XAI(설명 가능한 AI)**는 AI가 "왜 이런 결정을 내렸는지"를 사람이 알 수 있게 **이유를 설명해주는** 기술이에요!
2. "대출 거절됐어요" 대신 **"소득이 낮아서 30점, 연체 이력이 있어서 20점 감점됐어요"**처럼 구체적 이유를 알려줘요.
3. EU AI Act 같은 법규에서 의료·금융 AI는 **반드시 설명을 제공**해야 한다고 의무화해서, XAI가 AI 사업의 필수 기술이 됐어요!
