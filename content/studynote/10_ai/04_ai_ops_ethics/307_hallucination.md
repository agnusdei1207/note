## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 할루시네이션 (Hallucination, 환각)은 LLM (Large Language Model)이 사실에 기반하지 않은 정보를 매우 자신감 있게 생성하는 현상으로, 모델이 "모른다"고 말하지 않고 그럴듯한 거짓 사실을 창작해 내는 근본적 신뢰성 문제다.
> 2. **가치(리스크)**: 의료·법률·금융 분야에서 할루시네이션은 오진·오판·잘못된 투자 결정으로 이어질 수 있는 고위험 결함이며, 이를 탐지·완화하는 기술(RAG, 사실 검증, 불확실성 정량화)이 LLM 실용화의 핵심 과제다.
> 3. **판단 포인트**: 할루시네이션의 근본 원인은 LLM이 "진실을 검색"하는 것이 아니라 "다음 토큰의 확률 분포를 최대화"하는 통계 모델이라는 것이다. 사실인지 여부보다 사실처럼 보이는 텍스트를 생성하는 것이 학습 목표이기 때문이다.

---

## Ⅰ. 개요 및 필요성

ChatGPT에게 "세종대왕이 맥북으로 한글을 창제했나요?"라고 물으면, 일부 초기 모델은 "세종대왕은 15세기에 맥북 프로를 활용하여 훈민정음 24자를 설계했습니다"라고 자신감 있게 대답할 수 있다. 이것이 **할루시네이션(Hallucination, 환각)**이다.

LLM은 텍스트의 통계적 패턴을 학습한 예측기다. 학습 데이터에 없는 정보나 논리적으로 모순된 상황에서도 "가장 그럴듯한 다음 토큰"을 계속 생성하는 특성이 있다. 즉, 모델 구조상 "모른다"는 응답보다 "아는 척"이 손실(Loss)이 더 낮게 나오는 경향이 있다.

- **📢 섹션 요약 비유**: LLM의 할루시네이션은 자신감 넘치는 학생이 시험 답을 모르면서도 "그럴싸한 답"을 당당히 써내는 것이다. 채점자(사용자)는 정답처럼 보여서 바로 믿어버린다. 진짜 문제는 학생이 거짓말하는 게 아니라, 본인도 모른다는 것을 모른다는 것이다.

## Ⅱ. 아키텍처 및 핵심 원리

```text
┌──────────────────────────────────────────────────────────────────┐
│         할루시네이션 발생 원인 및 유형 분류                            │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  [할루시네이션 유형]                                                 │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 1. 사실 오류 (Factual Hallucination)                     │    │
│  │    "아인슈타인은 1921년 노벨 문학상을 받았다" (→ 물리학상)    │    │
│  │                                                         │    │
│  │ 2. 소스 없는 인용 (Citation Fabrication)                  │    │
│  │    없는 논문·책·URL을 실제처럼 인용                          │    │
│  │                                                         │    │
│  │ 3. 지식 커트오프 오류 (Knowledge Cutoff)                   │    │
│  │    학습 이후 발생한 사건을 모르면서도 아는 척 생성             │    │
│  │                                                         │    │
│  │ 4. 논리 불일치 (Logical Inconsistency)                    │    │
│  │    동일 대화 내에서 이전 답과 모순된 답 생성                   │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
│  [근본 원인]                                                       │
│  LLM 학습 목표: P(next token | context) 최대화                    │
│  → 진실 여부와 무관하게 "언어적으로 자연스러운" 출력 생성               │
│  → 학습 데이터의 오류·편향·누락이 그대로 학습됨                       │
│  → 드문 사실은 학습 데이터 부족으로 잘못 기억                         │
└──────────────────────────────────────────────────────────────────┘
```

| 완화 기법 | 방법 | 효과 |
|:---|:---|:---|
| RAG (검색 증강 생성) | 실시간 외부 DB 검색 결과를 컨텍스트에 주입 | 사실 기반 생성, 최신 정보 반영 |
| 사실 검증 파이프라인 | 생성 출력을 외부 KB와 교차 검증 | 오류 탐지 및 수정 |
| 불확실성 표현 유도 | "모르면 '모른다'고 말해" 지시 프롬프트 | 자신감 점수 보정 |
| RLHF | 인간 피드백으로 정직성 강화 | 사실 정확도 향상 |
| 온도(Temperature) 조정 | Temperature 낮춤 | 더 결정론적 응답, 창작성 희생 |

- **📢 섹션 요약 비유**: RAG는 기억력이 나쁜(할루시네이션) 전문가에게 시험 중 참고서(외부 지식 DB)를 허용하는 것이다. 암기에 의존하지 않고 책을 찾아보게 하면(검색) 정확도가 극적으로 올라간다. 단, 책에 없는 내용은 여전히 모른다고 말해야 한다.

## Ⅲ. 비교 및 연결

- **할루시네이션 vs 편향(Bias)**: 할루시네이션은 사실 자체를 틀리게 생성하는 것이고, 편향(Bias)은 특정 집단에 대한 편향된 서술을 하는 것이다. 예를 들어 "남성 엔지니어, 여성 간호사"처럼 성별과 직업을 편향적으로 연결하는 것은 할루시네이션이 아닌 편향이다. 하지만 둘 다 AI 신뢰성 문제의 핵심 구성 요소다.
- **일관성 (Consistency) 검사**: 동일 프롬프트를 N회 실행하여 답변이 일관되는지 확인. 일관성이 낮으면 할루시네이션 위험 신호.

- **📢 섹션 요약 비유**: 할루시네이션과 편향의 차이는 지도의 두 종류 오류와 같다. 할루시네이션은 실제 없는 강을 그리는 것(사실 오류), 편향은 강이 실제로 있지만 특정 마을만 크게 그리는 것(과장·왜곡)이다. 두 오류 모두 사용자를 잘못된 길로 안내하지만 원인과 해결책이 다르다.

## Ⅳ. 실무 적용 및 기술사 판단

**고위험 도메인 배포 시 필수 방어 체계**:
1. **RAG + 출처 인용 강제**: 모든 사실 주장에 참조 문서 출처를 표시
2. **신뢰도 점수(Confidence Score)**: 생성 답변의 자신감 수준 정량화 및 임계값 아래면 "불확실합니다" 표시
3. **Human-in-the-Loop**: 고위험 결정(의료 진단, 법률 판단)은 반드시 전문가 최종 검토
4. **사실 검증 API 연동**: WolframAlpha, Google Knowledge Graph 등 외부 사실 검증 서비스와 출력 후처리 연동
5. **Hallucination 벤치마크**: TruthfulQA, HaluEval 등 전용 벤치마크로 배포 전 할루시네이션 발생률 측정

- **📢 섹션 요약 비유**: 의료 AI에서 할루시네이션 방어는 비행기 조종사의 이중 점검 의무와 같다. 자동 조종장치(AI)가 "활주로 37L로 착륙"이라고 판단해도, 조종사(의사)가 반드시 직접 확인하고 최종 결정을 내린다. AI 출력은 제안이지 판결이 아니다.

---

## Ⅴ. 기대효과 및 결론

할루시네이션은 LLM의 구조적 한계이자 현재 AI 신뢰성의 가장 큰 도전이다. 완전한 제거는 불가능하지만, RAG·RLHF·사실 검증 파이프라인·불확실성 정량화 등을 조합하여 실용 가능한 수준으로 완화할 수 있다. EU AI Act와 같은 규정에서 고위험 AI 시스템은 할루시네이션 발생률 공개와 완화 조치를 요구하며, AI 신뢰성 엔지니어링(AI Safety Engineering)이 독립적 전문 분야로 급부상하고 있다.

- **📢 섹션 요약 비유**: 할루시네이션 완화 기술 발전은 자동차 안전벨트 의무화 역사와 같다. 자동차(LLM)가 아무리 빠르고 편리해도 사고(할루시네이션)가 날 수 있으니, 안전벨트(RAG, 사실 검증)를 의무적으로 장착해야 운행 허가(규제 승인)를 준다. AI도 성능과 안전성이 함께 발전해야만 사회에 배포될 수 있다.

### 👶 어린이를 위한 3줄 비유 설명

1. **할루시네이션**은 AI가 "모른다"고 말하는 대신 **그럴싸한 거짓말을 자신 있게 하는** 것이에요 — 마치 시험에서 모르는 문제에 아무 답이나 당당히 쓰는 것처럼요!
2. AI는 진실을 찾는 게 아니라 **"자연스럽게 이어지는 글자"**를 만들어내는 기계라서, 사실이 아닌 것도 그럴듯하면 써버려요.
3. 이걸 막으려면 **외부 지식 검색(RAG)**이나 **전문가 확인(Human-in-the-Loop)**을 반드시 함께 사용해야 해요!
