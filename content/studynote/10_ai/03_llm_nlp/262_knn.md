## 핵심 인사이트 (3줄 요약)
> 1. **본질**: K-최근접 이웃(K-NN, K-Nearest Neighbors)은 "유사한 것은 유사한 레이블을 가진다"는 가정 아래, 예측 시점에 가장 가까운 K개 이웃의 다수결/평균으로 결과를 내는 게으른 학습기(Lazy Learner)다.
> 2. **가치**: 별도 훈련 없이 즉시 사용 가능하고 비선형 경계를 자연스럽게 표현하지만, 차원의 저주(Curse of Dimensionality)와 예측 시 높은 계산 비용이 단점이다.
> 3. **판단 포인트**: K값 선택(홀수 권장), 거리 척도 선택, 특성 스케일링이 K-NN 성능을 좌우하는 세 가지 핵심 요소다.

---

## Ⅰ. 개요 및 필요성

K-NN은 가장 단순하고 직관적인 지도 학습 알고리즘이다. "이 과일은 어떤 종류인가?"라는 질문에 "가장 비슷하게 생긴 K개의 과일을 보면 안다"는 논리다.

**K-NN의 특이한 특성 — 게으른 학습기(Lazy Learner)**:
- 훈련(Training): 단순히 데이터를 메모리에 저장 (O(1))
- 예측(Prediction): 쿼리와 모든 훈련 샘플 간 거리 계산 (O(n·d))
- 대부분의 알고리즘과 반대로, 계산 비용이 예측 시점에 집중된다.

| 구분 | 분류 (Classification) | 회귀 (Regression) |
|:---|:---|:---|
| 집계 방법 | K개 이웃의 다수결 | K개 이웃의 평균/가중 평균 |
| 출력 | 클래스 레이블 | 연속값 |
| K=1 | 가장 가까운 1개 이웃 클래스 | 가장 가까운 이웃 값 그대로 |
| K=N | 전체 데이터 다수결 | 전체 평균 (상수) |

📢 **섹션 요약 비유**: K-NN은 "처음 보는 식당에서 주변 테이블 손님들이 먹는 음식을 보고 주문"하는 것이다. 주변 사람들이 다 스테이크를 먹으면(K 이웃이 같은 클래스) 스테이크가 맛있다고 판단한다.

## Ⅱ. 아키텍처 및 핵심 원리

### K-NN 예측 과정

```
  쿼리 포인트 x_q (?) 에 대한 K-NN 분류 (K=5)
  ┌──────────────────────────────────────────────┐
  │                                              │
  │  ○  ○    ←── 클래스 A (Circle)              │
  │     ○                                        │
  │         [x_q]  ← 쿼리 포인트               │
  │        ╱ ↑ ╲                                │
  │       ╱  │  ╲  K=5 최근접 이웃:            │
  │      ●   ○   ●  ┌─────────────────────────┐ │
  │       ╲     ╱   │ ○○○●● → A:3, B:2        │ │
  │        ●───●    │ → 다수결: 클래스 A       │ │
  │                 └─────────────────────────┘ │
  │  ●  ●    ←── 클래스 B (Square)              │
  └──────────────────────────────────────────────┘
```

### 거리 척도 (Distance Metric)

| 거리 | 수식 | 특징 |
|:---|:---|:---|
| 유클리드 (Euclidean) | √Σ(x_i - x_j)² | 직선 거리, 가장 일반적 |
| 맨해튼 (Manhattan) | Σ\|x_i - x_j\| | 격자 이동 거리, 이상값 강건 |
| 민코프스키 (Minkowski) | (Σ\|x_i-x_j\|^p)^(1/p) | p=1:맨해튼, p=2:유클리드 일반화 |
| 코사인 유사도 (Cosine) | 1 - cos(θ) | 방향 유사성, 텍스트에 적합 |
| 해밍 (Hamming) | 다른 비트 수 / 전체 | 범주형/이진 데이터 |

### K 값의 영향

```
  K=1 (과적합 위험):         K=N (과소적합):
  ┌────────────────────┐     ┌────────────────────┐
  │  ● ●               │     │                    │
  │ ● ●  ●●○          │     │  ──────────────    │
  │ ● ●  ○●○○         │     │  (단일 경계선)     │
  │      ○ ○ ○        │     │                    │
  │  (복잡한 경계)     │     │                    │
  └────────────────────┘     └────────────────────┘

  최적 K 선택: 교차 검증(Cross Validation)으로 결정
              일반적으로 홀수 권장 (동점 방지)
              경험칙: K ≈ √n (n=훈련 샘플 수)
```

### 차원의 저주 (Curse of Dimensionality)

고차원에서는 모든 포인트가 서로 비슷하게 멀어져, 거리 기반 판단이 무의미해진다.

```
  1D: 100개 점 중 10%를 커버 → 이웃 10개
  2D: 같은 커버리지 → 한 면의 31.6% 필요
  nD: n차원 → 전체의 10^(n/2)% 필요 → 이웃 = 전체 데이터

  → 고차원에서 K-NN은 사실상 모든 점이 이웃
    → 거리 차이가 무의미 → 성능 급락
```

📢 **섹션 요약 비유**: 차원의 저주는 "도시가 너무 커지면 가까운 이웃의 개념이 사라지는 것"이다. 서울에서 가장 가까운 집과 1km 떨어진 집의 차이가 무의미해지듯, 고차원에서는 모든 거리가 비슷해진다.

## Ⅲ. 비교 및 연결

### K-NN vs 다른 분류기 비교

| 특성 | K-NN | SVM | 결정 트리 | 신경망 |
|:---|:---|:---|:---|:---|
| 훈련 시간 | O(1) | O(n²~n³) | O(n log n) | O(epoch × n) |
| 예측 시간 | O(n·d) | O(sv·d) | O(depth) | O(layers) |
| 해석 가능성 | 높음 | 낮음 | 높음 | 낮음 |
| 고차원 | 약함 | 강함 | 중간 | 강함 |
| 스케일링 필요 | 필수 | 필수 | 불필요 | 권장 |

### K-NN의 변형들

```
  ┌────────────────────────────────────────────┐
  │ K-NN 변형                                  │
  ├────────────────┬───────────────────────────┤
  │ 가중 K-NN      │ 가까운 이웃에 더 큰 가중치│
  │ (Weighted)     │ w = 1/distance²            │
  ├────────────────┼───────────────────────────┤
  │ 반경 이웃      │ K 고정 대신 반경 r 내 모든│
  │ (Radius-based) │ 이웃 사용                  │
  ├────────────────┼───────────────────────────┤
  │ 근사 최근접    │ 정확도를 포기하고 속도를   │
  │ (ANN, Approx.) │ 높임 (FAISS, HNSW)         │
  └────────────────┴───────────────────────────┘
```

📢 **섹션 요약 비유**: K-NN은 "실시간 여론조사"다. 매번 새로운 유권자(쿼리)가 나타날 때마다 모든 유권자의 의견(훈련 데이터)을 다시 집계한다. 의회(모델)를 구성하지 않고 매번 직접 민주주의를 실행하는 것이다.

## Ⅳ. 실무 적용 및 기술사 판단

### K-NN 최적화 전략

1. **전처리 필수**: 특성 스케일링(StandardScaler 또는 MinMaxScaler) — 거리 계산에 스케일이 직접 영향
2. **K 선택**: 교차 검증으로 최적 K 탐색, 홀수 사용 (동점 방지)
3. **차원 축소**: PCA(Principal Component Analysis) 또는 특성 선택으로 차원 축소
4. **인덱싱 구조**: KD-Tree(낮은 차원), Ball-Tree(중간 차원), FAISS(고차원 근사)

### 대규모 데이터에서의 K-NN 활용

현대에는 K-NN이 직접 분류기로 쓰이기보다 **벡터 데이터베이스(Vector DB)**와 결합하여 추천 시스템, 의미 검색(Semantic Search)에 활용된다.

```
  현대 K-NN 활용 패턴:
  ┌────────────────────────────────────────────┐
  │  임베딩 벡터 (Word2Vec, BERT 등) 생성      │
  │              │                             │
  │  FAISS/HNSW 인덱스 구축 (ANN 근사)         │
  │              │                             │
  │  쿼리 벡터 → 유사 벡터 K개 검색           │
  │              │                             │
  │  추천/검색 결과 반환                       │
  └────────────────────────────────────────────┘
```

### 기술사 답안 포인트

- **"K-NN이 게으른 학습기인 이유"**: 훈련 단계가 없음 → 데이터를 통째로 메모리에 유지 → 예측 시 O(n·d)
- **"차원의 저주 해결책"**: 특성 선택, PCA, 임베딩으로 차원 축소 후 K-NN 적용
- **"K 값이 클수록 좋은가"**: K가 크면 편향 증가, 작으면 분산 증가 → 트레이드오프

📢 **섹션 요약 비유**: K-NN 스케일링은 키와 몸무게로 사람을 분류할 때, 키는 cm단위(수백), 몸무게는 kg단위(수십)면 키가 지배적이다. 같은 단위로 정규화해야 두 특성이 동등하게 거리에 반영된다.

## Ⅴ. 기대효과 및 결론

K-NN을 올바르게 활용하면:

1. **빠른 프로토타이핑**: 훈련 없이 즉시 성능 확인 → 데이터 품질 빠른 검증
2. **비모수적 유연성**: 분포 가정 없이 임의 형태의 결정 경계 표현
3. **이상값 탐지**: 이웃 거리가 비정상적으로 먼 포인트 = 이상값
4. **추천 시스템**: 사용자 임베딩의 K-NN으로 협업 필터링(Collaborative Filtering) 구현

딥러닝 시대에도 K-NN은 **임베딩 기반 유사도 검색**이라는 새로운 형태로 대규모 추천/검색 시스템의 핵심 구성요소로 살아있다.

📢 **섹션 요약 비유**: K-NN은 "족보 없이 직관으로 판단하는 원로"다. 복잡한 이론(모델 구조)은 없지만, 과거 경험(훈련 데이터) 전체를 기억해서 비슷한 상황을 찾아내는 순수한 경험적 지혜다.

### 📌 관련 개념 맵

| 개념 | 연관 키워드 | 관계 |
|:---|:---|:---|
| K-NN (K-Nearest Neighbors) | 거리 기반, 게으른 학습기 | 인스턴스 기반 분류/회귀 |
| 유클리드 거리 (Euclidean Distance) | 직선 거리, L2 노름 | K-NN 기본 거리 척도 |
| 맨해튼 거리 (Manhattan Distance) | 격자 이동, L1 노름 | 이상값 강건 거리 |
| 차원의 저주 (Curse of Dimensionality) | 고차원, 거리 무의미화 | K-NN 최대 약점 |
| 게으른 학습기 (Lazy Learner) | 훈련 O(1), 예측 O(n) | 학습 비용 없는 알고리즘 |
| 특성 스케일링 (Feature Scaling) | StandardScaler, 필수 전처리 | 거리 왜곡 방지 |
| KD-Tree / Ball-Tree | 공간 인덱싱, 검색 가속 | K-NN 예측 속도 향상 |
| ANN (Approximate Nearest Neighbor) | FAISS, HNSW | 대규모 고차원 근사 탐색 |
