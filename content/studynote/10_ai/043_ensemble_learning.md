# 앙상블 학습 (Ensemble Learning) - 다수의 약한 분류기를 결합하여 강력한 성능을 달성하는 기법

> ⚠️ 이 문서는 머신러닝에서 예측 성능을 극대화하기 위해 여러 개의 모델을 결합하는 '앙상블 학습(Ensemble Learning)'의 개념, 다수결 투표(Voting), 평균화(Averaging), 스태킹(Stacking) 등의 결합 방법, 그리고 앙상블이 왜 성능을 향상시키는지에 대한 이론적 배경에 대해 심층 분석한다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 앙상블 학습은 동일한 훈련 데이터로 훈련된 여러 모델(基础 학습기, Base Learner)의 예측을 결합하여 단일 모델의 예측보다 더 나은 성능을 달성하는 기법이다. 각 모델이 다른 종류의 오차를냈다면,它们的 오차가 서로 상쇄되어 전체 성능이 향상된다.
> 2. **가치**: 1990년대以降, 앙상블 방법은 Kaggle, Netflix Prize 등 실전 대회与合作사에서 압도적으로 우승했다. 실제로 최고 성능 시스템은 대부분 단일 모델이 아닌 앙상블이다.
> 3. **융합**: 앙상블은 동일 알고리즘의 여러 인스턴스(예: Random Forest)를 결합하는 Homogeneous 앙상블과, 다른 알고리즘(예: SVM + 결정 트리 + 로지스틱 회귀)을 결합하는 Heterogeneous 앙상블로 나뉘며, 다양한 상황에서 필수적인 기법이 되었다.

## Ⅰ. 개안 및 필요성 (Context & Necessity)

### 1.众人的Clinton은女神보다 낫다: 합의 힘 (Pain Point)
학생 A가 수학 문제를 혼자 풀었는데, 자신이 틀렸을까봐不安했다.
- **문제 발생**: 혼자 풀면 틀릴 가능성이 있다. 그러나 친구 10명이 각각 풀어 Majority Voting하면, 대부분 맞는 답을 찾을 수 있다.
- **앙상블의 해결책**: 여러 명의 지식을 종합하면, 각자의오류를 상쇄하고より 정확한 판단을 달성할 수 있다.

### 2. 앙상블 학습의 필요성
앙상블은 분산(Variance)과 편향(Bias) 사이의 트레이드오프를 활용한다.
- **편향 감소**: 여러 모델의 예측을 평균하면 예측값이 더 진짜 값에 가까워진다
- **분산 감소**: 단일 모델 대비 예측의 Variation이 줄어든다
- **정리**: 앙상블은 여러 모델의弱点を互いに補い合い、全体としてより强大的预测能力を実現します.

- **📢 섹션 요약 비유**: 앙상블 학습은 "역사 학회 토론회"와 같다. 단일 연구자(단일 모델)의 발표는 그 사람의 분석 틀에限制되어偏見이 발생할 수 있다. 그러나 여러 분야 연구자(다양한 모델)가 각자의 分析道具으로 같은 주제를探讨하면, 서로의 Limitationsを補い合い、より包括적이고 정확한結論에 도달할 수 있다.

## Ⅱ. 핵심 아키텍처 및 원리 (Architecture & Mechanism)

### 1. 앙상블 학습 흐름 다이어그램

```text
┌─────────────────────────────────────────────────────────────────────┐
│                      [ 앙상블 학습 흐름 ]                                           │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │                      입력: 훈련 데이터 D                              │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                              │                                             │
│          ┌──────────────────┼──────────────────┐                      │
│          ↓                  ↓                  ↓                      │
│    ┌──────────┐        ┌──────────┐        ┌──────────┐               │
│    │ 모델 M_1 │        │ 모델 M_2 │        │ 모델 M_n │               │
│    │ 학습기 1  │        │ 학습기 2  │   ...   │ 학습기 n  │               │
│    └────┬─────┘        └────┬─────┘        └────┬─────┘               │
│         ↓                  ↓                  ↓                      │
│    예측 P_1            예측 P_2            예측 P_n                    │
│         └──────────────────┼──────────────────┘                      │
│                            ↓                                           │
│              ┌─────────────────────────────┐                          │
│              │       결합 전략 (Combination)      │                      │
│              │  • 다수결 투표 (Voting)            │                      │
│              │  • 평균화 (Averaging)              │                      │
│              │  • 가중 평균 (Weighted Averaging) │                      │
│              │  • 스태킹 (Stacking)              │                      │
│              └─────────────────────────────┘                          │
│                            ↓                                           │
│                      최종 예측                                          │
└─────────────────────────────────────────────────────────────────────┘
```

### 2. 앙상블 결합 방법 유형

| 결합 방법 | 분류 (Class) 예측 | 회귀 (Continuous) 예측 | 설명 |
|:---|:---|:---|:---|
| **Hard Voting** | 다수결 | - | 가장 많은 표를 받은 클래스 선택 |
| **Soft Voting** |_argmax of 평균 확률_ | - | 각 모델의 클래스 확률 平均 |
| **Simple Averaging** | - | 평균 | 모든 모델 예측의 平均 |
| **Weighted Averaging** | - | 가중 평균 | 모델별 가중치 부여 平均 |

### 3. Soft Voting vs Hard Voting 수식

**Hard Voting:**
$$\hat{y} = \text{mode}\{M_1(x), M_2(x), ..., M_n(x)\}$$

**Soft Voting (확률 기반):**
$$\hat{y} = \arg\max_j \frac{1}{N} \sum_{i=1}^{N} P_i(y=j | x)$$

### 4. 앙상블의 오차 분해 이론

**편향-분산 분해:**
$$\text{Expected Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Error}$$

**앙상블의 효과:**

| 상황 | 단일 모델 | 앙상블 |
|:---|:---|:---|
| **모든 모델이同一 오차** | 편향 = b, 분산 = v | 개선 없음 |
| **모델 오차가 독립적** | 편향 = b, 분산 = v | 분산 ≈ v/n으로 감소 |
| **모델 오차가 음의 상관관계** | 편향 = b, 분산 = v | 분산大幅 감소, 편향도 감소 가능 |

### 5. Diversity (다양성)의 중요성

앙상블의 성능을 결정하는 핵심 요인은 模型間の 多様性(Diversity)이다.

| 다양성 확보 방법 | 설명 |
|:---|:---|
| **데이터 샘플링 차이** | 부트스트랩으로 서로 다른 훈련 데이터 사용 |
| **Feature 차이** | 각 모델이 다른 Feature 부분집합 사용 |
| **모델 아키텍처 차이** | 다른 알고리즘 (트리, SVM, 신경망 등) 사용 |
| **랜덤 시드 차이** | 다른 초기값으로 훈련 |
| **하이퍼파라미터 차이** | 다른 하이퍼파라미터로 훈련 |

- **📢 섹션 요약 비유**: 앙상블의 다양성은 "취향이 다른 맛 평가단 구성"과 같다. 全 평가원이 같은 취향(예: 한국 음식만 좋아하는 전문가)로 구성되면, 이탈리안 음식 평가에서偏見이 발생할 수 있다. 그러나 한국, 이탈리안, 프랑스, 일본 등 다양한 취향의 평가원으로 구성되면, 음식의多면적 측면을 평가할 수 있어より公正な評価가 가능해진다.

## Ⅲ. 비교 및 기술적 트레이드오프 (Comparison & Trade-offs)

### 앙상블 기법 비교

| 기법 | 결합 방식 | 훈련 방식 | 장점 | 단점 |
|:---|:---|:---|:---|:---|
| **Voting** | 예측 집계 | 병렬 | 단순, 효과적 | 동일 가중치 |
| **Bagging** | 평균/다수결 | 병렬 | 분산 감소 | 편향 감소 없음 |
| **Boosting** | 순차 가중 합산 | 순차 | 편향/분산 모두 감소 | 과적합 가능성 |
| **Stacking** | 메타 모델 | 2단계 | 다양한 모델 조합 | 복잡, 과적합 위험 |

### Homogeneous vs Heterogeneous 앙상블

| 유형 | 설명 | 예시 |
|:---|:---|:---|
| **Homogeneous** | 동일 알고리즘, 다른 인스턴스 | Random Forest (결정 트리 × N) |
| **Heterogeneous** | 서로 다른 알고리즘 | SVM + 트리 + 로지스틱 회귀 앙상블 |

### Voting 앙상블 상세 비교

```text
┌─────────────────────────────────────────────────────────────────────┐
│                  [ Hard Voting vs Soft Voting ]                                   │
│                                                                         │
│  ▷ Hard Voting (다수결)                                                │
│  ──────────────────────                                               │
│      모델 1 예측: Class A      → 표 1장                                  │
│      모델 2 예측: Class B      → 표 1장                                  │
│      모델 3 예측: Class A      → 표 1장                                  │
│      모델 4 예측: Class A      → 표 1장                                  │
│      모델 5 예측: Class B      → 표 1장                                  │
│                        ↓                                               │
│      최종 예측: Class A (3표 vs 2표)                                   │
│                                                                         │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ │
│                                                                         │
│  ▷ Soft Voting (확률 평균)                                             │
│  ──────────────────────                                                │
│      모델 1: P(A)=0.9, P(B)=0.1                                        │
│      모델 2: P(A)=0.4, P(B)=0.6                                        │
│      모델 3: P(A)=0.8, P(B)=0.2                                        │
│      모델 4: P(A)=0.7, P(B)=0.3                                        │
│      모델 5: P(A)=0.3, P(B)=0.7                                        │
│                        ↓                                               │
│      평균: P(A) = (0.9+0.4+0.8+0.7+0.3)/5 = 0.62                      │
│            P(B) = (0.1+0.6+0.2+0.3+0.7)/5 = 0.38                      │
│      최종 예측: Class A (0.62 > 0.38)                                   │
│                                                                         │
│      ※ Soft Voting이 일반적으로 더 나음 (확률 정보 활용)                   │
└─────────────────────────────────────────────────────────────────────┘
```

- **📢 섹션 요약 비유**: Hard Voting은 "총선에서의 단순 다수결 투표"와 같다.各지역구에서 후보 별로 得票 수만 비교하여 winner-take-all한다. Soft Voting은 "비례대표制国会 분석"과 같다.各당 repous의比 例만을 비교하여より细致한 比 例分配를 반영한다. Therefore, Soft Voting은 각 후보의 "지지 세력의 크기"를 반영하여 보다 정밀한 분석을 가능하게 한다.

## Ⅳ. 실무 판단 기준 (Decision Making)

| 고려 사항 | 세부 내용 | 주요 아키텍처 의사결정 |
|:---|:---|:---|
| **도입 환경** | 기존 레거시 시스템과의 호환성 분석 | 마이그레이션 전략 및 단계별 전환 계획 수립 |
| **비용(ROI)** | 초기 구축 비용(CAPEX) 및 운영 비용(OPEX) | TCO 관점의 장기적 효율성 검증 |
| **보안/위험** | 컴플라이언스 준수 및 데이터 무결성 보장 | 제로 트러스트 기반 인증/인가 체계 연계 |

*(추가 실무 적용 가이드 - 주가 예측 앙상블)*
- **상황**: 투자은행에서 단기 주가 방향(상승/하락)을 예측하는 모델을 개발 중. 다양한 모델을 결합하여 예측 안정성을 높이려고 한다.
- **실무 의사결정**:
  1. **모델 다양성 확보**: LSTM (시계열 패턴), XGBoost (규칙 기반), RF (비선형 패턴)를 함께 사용
  2. **결합 방식**: Soft Voting 선택 (확률 기반 예측이므로)
  3. **가중치 설정**: 각 모델의 검증 성능 기반으로 가중치 조절
  4. **Stacking 검토**: 기본 모델 예측을 입력으로 하는 메타 모델(로지스틱 회귀) 추가로 고려
  5. **리스크 관리**: 최종 예측을 그대로 사용하지 않고, 앙상블 예측의 불확실성(확률 분산)도 함께 고려

- **📢 섹션 요약 비유**: 주가 예측 앙상블은 "투자 리스크 Committee"와 같다. 각 리스크 관리자(모델)가 다른 分析框架로 주가를 분석한다: 재무제표 기반 분석가는 우상향, 기술적 분석가는 단기 하락, 거시경제 분석가는中立을 제시한다. Committee는これらの意见을 종합하여 최종 투자 전략을 결정한다. 특히 분석가들意见이 크게 엇갈릴 때(앙상블의 불확실성 높음)에는より konservativ한 투자 전략을 채택한다.

## Ⅴ. 미래 전망 및 발전 방향 (Future Trend)

1. **딥러닝과 앙상블의 결합 (Deep Ensemble)**
   최근 연구에서 동일한 신경망을 다른 초기값으로 여러 번 훈련하여 앙상블하는 'Deep Ensemble'이 제안되었다. 이 방법은 Monte Carlo Dropout과 함께 불확실성 추정에 효과적이며, Bayesian Neural Network 대안으로 주목받고 있다.

2. **AutoML과 앙상블의自動化**
   AutoML系统中, 모델 선택과 앙상블을 자동으로 최적화하는 연구가 진행되고 있다. 예를 들어, Evolutionary Algorithm을 활용하여 최적의 모델 조합과 가중치를 자동 탐색한다.

- **📢 섹션 요약 비유**: 앙상블의 미래는 "AI 거버넌스 시스템"과 같다. 과거에는単一AI 시스템이 모든 판단을 내렸지만, 향후에는 여러 AI 시스템이 각각 다른角度에서 분석하고, 그 결과물을 综合하여より革新的이고 공정한 판단을 내리는 거버넌스 시스템이 구축될 것이다.

    *   **다수결 투표 (Voting)**: 다수 모델의 예측 중 최빈값 선택
