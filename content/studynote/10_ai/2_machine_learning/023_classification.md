## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 분류 (Classification)는 입력 데이터가 어떤 범주(Category)에 속하는지 예측하는 지도 학습 태스크로, 이진 분류(Binary)와 다중 분류(Multi-class)로 나뉜다.
> 2. **가치**: 스팸 이메일 필터링, 질병 진단, 사기 탐지, 이미지 분류 등"사전 정의된 범주 중 하나를 선택"하는 모든实际问题에 필수적이다.
> 3. **융합**: 전통 ML(KNN, SVM, Decision Tree)에서 딥러닝(CNN, Transformer)까지 다양한 알고리즘이 적용되며, 문제 특성(데이터 규모,Interpretability 요구, 클래스 불균형)에 따라 적절한 알고리즘과 평가 지표 선택이 중요하다.

---

## Ⅰ. 개요 및 필요성

### 개념 정의

분류 (Classification)는 입력 특성(feature) 벡터 x가事前에 정의된 K개의 클래스(범주) C = {c₁, c₂, ..., cₖ} 중 어느 하나에 속하는지를预测하는 작업이다. 수식으로 표현하면 다음과 같다: ŷ = argmax_k P(y=k | x) 또는 ŷ = f(x; θ) where f outputs class label.

이진 분류 (Binary Classification)는 클래스가 2개(양성/음성, 예/아니오)인 가장 단순한 경우이며, 다중 분류 (Multi-class Classification)는 클래스가 3개 이상인 경우다. 다중 레이블 분류 (Multi-label Classification)는 하나의 입력에 여러 클래스가 동시에 할당될 수 있는 경우다.

### 비유

분류는"선거용지 읽기"와 같다. 유권자가 찍은 표(입력)를"후보 A/B/C 중 누구?"로 읽어 판단하는 과정이다.

### 필요성

분류가 필요한 이유는"자동화된 의사결정의 기초"이기 때문이다. 사람이 직접 분류하면 시간과 비용이 많이 들고 일관성을 유지하기 어렵지만, 분류 모델은 대규모 데이터를 빠르게, 일관되게 분류할 수 있다.

### 섹션 요약 비유

분류는"자동 분류대(自動仕分け)"와 같다. 물건을 특성별로 살펴보고"여기는 과일, 저기는 채소"라고 자동으로 구분하는 시스템이다.

## Ⅱ. 아키텍처 및 핵심 원리

### 이진 분류의 평가 지표: Confusion Matrix

```
┌──────────────────────────────────────────────────────────────────────┐
│                    Confusion Matrix와 평가 지표                            │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  【Confusion Matrix (혼동 행렬)】                                     │
│                                                                      │
│                          예측 결과                                    │
│                     Positive    Negative                            │
│                 ┌────────────┬────────────┐                        │
│  실제  Positive │     TP     │     FN     │  ← True/False는        │
│       결과       │ (참양성)   │ (거짓음성)  │    예측의 正誤        │
│                 ├────────────┼────────────┤                        │
│        Negative │     FP     │     TN     │  ← Positive/Negative는  │
│                 │ (거짓양성)  │ (참음성)   │    예측 결과의 분류      │
│                 └────────────┴────────────┘                        │
│                                                                      │
│  【주요 평가 지표】                                                 │
│                                                                      │
│  Accuracy (정확도) = (TP + TN) / (TP + TN + FP + FN)               │
│                    = 예측 일치 수 / 전체 수                           │
│                                                                      │
│  Precision (정밀도) = TP / (TP + FP)                               │
│                    = 양성으로 예측한 것 중 실제 양성의 비율            │
│                                                                      │
│  Recall (재현율) = TP / (TP + FN)                                    │
│                  = 실제 양성 중 양성이라고 예측한 비율                 │
│                  = Sensitivity, True Positive Rate (TPR)              │
│                                                                      │
│  Specificity = TN / (TN + FP)                                        │
│              = 실제 음성 중 음성이라고 예측한 비율                     │
│                                                                      │
│  F1 Score = 2 × (Precision × Recall) / (Precision + Recall)        │
│           = Precision과 Recall의 조화 평균                           │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** Confusion Matrix는 분류 모델의 예측 결과를 4가지 경우로 분류한 표다. TP(참양성)는 실제 양성을 양성으로 정확히 예측, FN(거짓음성)은 실제 양성을 음성으로 잘못 예측, FP(거짓양성)은 실제 음성을 양성으로 잘못 예측, TN(참음성)은 실제 음성을 음성으로 정확히 예측한 경우다. Accuracy는 전체 중 정확한 예측의 비율이지만, 클래스 불균형 시 misleading할 수 있다. Precision은"예측한 양성 중 실제 양성의 비율"으로, 거짓양성(FP)이 많으면 낮아진다. Recall은"실제 양성 중 양성으로 예측한 비율"으로, 거짓음성(FN)이 많으면 낮아진다. F1 Score는 Precision과 Recall의 조화 평균으로, 둘 사이의 균형을 나타낸다.

### Precision-Recall Tradeoff와阈值

```
┌──────────────────────────────────────────────────────────────────────┐
│                    Precision-Recall Tradeoff                              │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  【확률적 분류기 (로지스틱 회귀 등)】                               │
│                                                                      │
│  많은 분류기는 클래스 소속 확률 P(y=1|x)를 출력한다.                   │
│  이 확률과 Threshold(임계값)를 비교하여 최종 분류를 결정:             │
│                                                                      │
│  if P(y=1|x) ≥ threshold: → Positive로 분류                         │
│  else: → Negative로 분류                                            │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐     │
│  │                                                             │     │
│  │   확률 분포                    Threshold의 영향:            │     │
│  │                                                             │     │
│  │   [양성 클래스] ████████████                          │     │
│  │                  ████████████████                       │     │
│  │   [음성 클래스]        ████████████████████            │     │
│  │                        ████████████████████████          │     │
│  │                                                             │     │
│  │            임계값 0.3              임계값 0.7               │     │
│  │            → Precision 낮음,      → Precision 높음,     │     │
│  │              Recall 높음            Recall 낮음           │     │
│  │                                                             │     │
│  │   Threshold ↑ → Positive 예측 적어짐 → Precision ↑, Recall ↓    │     │
│  │   Threshold ↓ → Positive 예측 많아짐 → Precision ↓, Recall ↑      │     │
│  │                                                             │     │
│  └─────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  【ROC 곡선과 AUC】                                                  │
│                                                                      │
│  ROC (Receiver Operating Characteristic):                            │
│  - X축: FPR (False Positive Rate) = FP / (FP + TN)                  │
│  - Y축: TPR (True Positive Rate) = TP / (TP + FN) = Recall         │
│  - 모든 가능한 Threshold에 대한 TPR-FPR 쌍을 플롯한 것                │
│                                                                      │
│  AUC (Area Under Curve):                                             │
│  - ROC 곡선 아래 면적 (0.5 = 무작위, 1.0 = 완벽)                     │
│  - 클래스 불균형에 robust한 평가 지표                                 │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │                                                             │    │
│  │   TPR (Recall) │                                    /       │    │
│  │              1.0 │                                 /         │    │
│  │                  │                               /           │    │
│  │                  │                            /              │    │
│  │                  │                         /                 │    │
│  │                  │                      /                    │    │
│  │                  │                   /                       │    │
│  │                  │                /                          │    │
│  │                0 │ ──────────────/─────────────────────   │    │
│  │                  │              /                            │    │
│  │                  │            /                               │    │
│  │                  │          /                                  │    │
│  │                  │       /                                    │    │
│  │                  │     / (AUC = 0.9 → 매우 좋은 분류기)      │    │
│  │                  │   /                                       │    │
│  │                0 │──────────────────────────────────────     │    │
│  │                  │              FPR = 0                      │    │
│  │                                                             │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** Precision-Recall Tradeoff는 임계값(threshold)에 따라 Precision과 Recall이 반비례하는 관계를 보여준다. Spam filter에서 threshold를 낮추면(확률 0.3 이상이면 스팸으로 분류) 더 많은 이메일을 스팸으로 분류하므로 Recall은 높지만, 실제로 정상인 이메일도 스팸으로 오인할 수 있어 Precision은 낮아진다. 반대로 threshold를 높이면(0.7 이상) Precision은 높아지지만 실제 스팸을 놓칠 수 있어 Recall이 낮아진다. ROC 곡선은 모든 임계값에 대해 TPR과 FPR의 관계를 시각화하며, AUC는 그 곡선 아래 면적으로"분류기의 Overall 성능"을 하나의 수치로 요약한다.

### 섹션 요약 비유

Precision-Recall Tradeoff는"안전 vs 편의를 동시에 만족시킬 수 없는 상황"과 같다. 완벽한 안보는 불편하고, 완벽한 편的安全性이 낮다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교: 이진 분류 vs 다중 분류

| 구분 | 이진 분류 (Binary) | 다중 분류 (Multi-class) |
|:---|:---|:---|
| **출력** | 0 또는 1 | {1, 2, ..., K} |
| **예시** | 스팸/정상, 악성/양성 | Iris 종 분류, MNIST 숫자 |
| **전략** | 하나의 분류 경계면 | K개의 분류 경계면 or 일대일 |
| **활성화** | Sigmoid | Softmax |
| **평가** | Confusion Matrix (2×2) | Confusion Matrix (K×K) |

### 섹션 요약 비유

이진 분류는"양자택일", 다중 분류는"여러 선택지 중 결정"에 해당한다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 — 질병 선별 검사**: 암 선별에서"거짓양성(FP)"은 불필요한 검사·불안 유발, 반면"거짓음성(FN)"은 진단 지체로 생명 위험이라는根本적으로 다른 비용을 가진다. 따라서 암 선별에서는 Recall을 극대화하는 것이 중요하며, 이는 Precision-Recall tradeoff에서 Recall 측으로 치우친 threshold 선택을 정당화한다.

### 섹션 요약 비유

분류의 평가 지표 선택은"어떤 실수를 더容認할 수 있는가?"에 대한 판단이다. 스팸 필터에서는 정상이 스팸으로 분류(FP)가 치명적일 수 있고, 악성 종양 탐지에서는 놓침(FN)이 치명적이다.

## Ⅴ. 기대효과 및 결론

### 미래 전망

- **AutoML**: 분류 알고리즘 선택과 하이퍼파라미터 튜닝 자동화로 ML 진입 장벽 하락
- **설명 가능 분류 (Explainable Classification)**: 분류 결과에 대한 해석 제공 (SHAP, LIME) 요구 증가
- **Class-Imbalanced Learning**: SMOTE, 클래스 가중치 조정 등으로 불균형 데이터 대응 기술 발전

### 섹션 요약 비유

분류 기술의 발전은"더 정확하고, 더 빠르고, 더 훑 understandable한 자동 분류 시스템"으로의 진화다.

## 👶 어린이를 위한 3줄 비유 설명

1. 분류는"选择题을 푸는 것"과 같아요. 스팸 이메일 필터는"이 이메일은 스팸이냐? 정상izh?"라고 답하는 것이고, 질병 진단은"이 종양은 악성이냐? 양성냐?"라고 답하는 것이에요.
2. Precision은"스팸이라고 답한 것 중 진짜 스팸의 비율"이고, Recall은"실제 스팸 중 스팸이라고 한 비율"이에요. 둘 다 높으면 좋은데, 보통은 하나를 높이면 다른 하나가 낮아져서 Tradeoff가 있어요.
3. F1 Score는 Precision과 Recall의"조화 평균"이라고 생각하면 돼요. 둘 다 적당히 높을 때 F1도 높아지거든요!
