## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 회귀 (Regression)는 입력 변수(들)로부터 연속적数值 출력(dependent variable)을 예측하는 지도 학습 태스크로, 주택 가격 예측, 온도 예측, 매출 예측 등에 활용된다.
> 2. **가치**: 인과관계 분석,趋势 예측, resource planning 등"얼마인가?"를 묻는 모든業務 의사결정에 필수적인基礎 제공한다.
> 3. **융합**: 선형 회귀에서 출발하여 릿지/라쏘(정규화), SVM 회귀(SVR), 의사결정 트리 회귀,神经网络 회귀 등 다양한 확장 모델로 발전했다.

## Ⅰ. 개요 및 필요성

### 개념 정의

회귀 (Regression)는 하나 이상의 입력 변수(독립 변수, X)로부터 연속적 출력 변수(종속 변수, Y)를 예측하는 통계적/기계学习方法이다. 분류가"어떤 category?"(이산적 출력)를 묻는다면, 회귀는"얼마인가?"(연속적 출력)를 묻는다.

수식으로는 Y = f(X) + ε로 표현한다. f는 알 수 없는 진정한 함수, ε는 노이즈(불규칙한 오차)다. 머신러닝의 목표는訓練 데이터를 통해 f를 근사하는 함수 f̂을 찾는 것이다.

### 비유

회귀는"수학 문제의 답을 대입하여 直線또는 곡선으로 예측하는 것"과 같다. 면적이 클수록 집값이 높아진다는 패턴을 데이터에서 학습하여, 새로운 집의 면적만 알면 예상 가격을 예측하는 것이 회귀다.

### 필요성

회귀가 필요한 이유는"연속적 의사결정"이 实世界에서 매우 흔하기 때문이다. 매장의 면적, 위치, 주변 상권 등의 입력에서 적절한 商品価格을 결정하거나, 과거 판매 데이터에서 미래 매출을 예측하는 것은企业经营의核心이다.

### 섹션 요약 비유

회귀는"숫자로 대답하는 시험"과 같다. 선택지가 있는 시험(분류)과 달리, 정확한 숫자(연속값)를 맞혀야 한다.

## Ⅱ. 아키텍처 및 핵심 원리

### 선형 회귀 (Linear Regression)

```
┌──────────────────────────────────────────────────────────────────────┐
│                    선형 회귀:原理과최적화                                │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  【단순 선형 회귀 (Simple Linear Regression)】                       │
│                                                                      │
│  Y = β₀ + β₁X + ε                                                   │
│                                                                      │
│  - β₀ (절편): X=0일 때의 Y값                                        │
│  - β₁ (기울기): X가 1단위 증가할 때 Y의 변화량                       │
│  - ε: 오차항 (평균 0, 분산 σ²)                                       │
│                                                                      │
│  【최소제곱법 (Ordinary Least Squares, OLS)】                        │
│                                                                      │
│  목표: 잔차 제곱 합 (Sum of Squared Residuals, RSS)을 최소화          │
│                                                                      │
│  RSS = Σ(yᵢ - ŷᵢ)² = Σ(yᵢ - (β₀ + β₁xᵢ))²                      │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐     │
│  │                                                             │     │
│  │   실제 데이터 점 (xᵢ, yᵢ)           예측 직선 ŷ = β₀ + β₁x   │     │
│  │           ●                                           /      │     │
│  │        ●●●●                                      /         │     │
│  │      ●●●●●●●                                  /            │     │
│  │     ●●●●●●●●●●                            /               │     │
│  │       ●●●●●●                            /                 │     │
│  │          ●●                          /                     │     │
│  │                                /  ← 잔차 (실제 - 예측)        │     │
│  │        ┌──────────────────────────────────────┐            │     │
│  │        │  RSS = Σ(잔차²)를 최소화하는 β₀, β₁ 탐색 │            │     │
│  │        └──────────────────────────────────────┘            │     │
│  │                                                             │     │
│  └─────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  【다중 선형 회귀 (Multiple Linear Regression)】                      │
│                                                                      │
│  Y = β₀ + β₁X₁ + β₂X₂ + ... + βₚXₚ + ε                            │
│                                                                      │
│  - p개의 입력 변수 (X₁, X₂, ..., Xₚ)                                 │
│  - 행렬 표기: Y = Xβ + ε                                            │
│  - 해: β̂ = (XᵀX)⁻¹Xᵀy (정규 방정식)                                │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 선형 회귀는"데이터를 관통하는 最 적합한直線(또는 평면/초평면)"을 찾는 것이다. OLS는 예측값과 실제값 사이의 차이(잔차)의 제곱 합을 최소화하는 파라미터를 찾는다. 이때"잔차를 제곱하는 이유"는 (1) 양수/음수 차이를统合하기 위해, (2) 큰 오차에 더 많은 페널티를 부과하기 위함이다. 정규 방정식 (XᵀX)⁻¹Xᵀy는 역행렬을 통해 해석적으로 최적 파라미터를 계산할 수 있게 해주지만, 역행렬 계산이 실용적으로는Computationally expensive한 경우가 있어 경사 하강법이 대신 사용되기도 한다.

### 손실 함수와 경사 하강법

```
┌──────────────────────────────────────────────────────────────────────┐
│                    회귀 손실 함수와최적화                               │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  【주요 손실 함수】                                                 │
│                                                                      │
│  1. MSE (Mean Squared Error) — 가장 범용적                          │
│     MSE = (1/n) Σ(yᵢ - ŷᵢ)²                                       │
│     → 큰 오차에 크게 페널티 (제곱效果)                               │
│                                                                      │
│  2. MAE (Mean Absolute Error)                                      │
│     MAE = (1/n) Σ|yᵢ - ŷᵢ|                                        │
│     → outliers에 덜 민감                                            │
│                                                                      │
│  3. Huber Loss — MSE와 MAE의 hybrid                                 │
│     δ 이하면 MSE, 초과이면 MAE                                      │
│     → outliers에도鲁棒하면서도 미분 가능!                            │
│                                                                      │
│  【경사 하강법 (Gradient Descent)】                                  │
│                                                                      │
│  파라미터 업데이트 규칙:                                            │
│  θ ← θ - α × ∇_θ Loss(θ)                                          │
│                                                                      │
│  α: 학습률 (learning rate)                                          │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐     │
│  │                                                             │     │
│  │   Loss Landscape            경사 하강법 과정:                │     │
│  │                             θ₁ ← θ₀ - α∇Loss              │     │
│  │        ╲        ● ← 현재 위치                                   │     │
│  │         ╲      ╱                                            │     │
│  │          ╲    ╱         ← 경사의 반대 방향으로 이동            │     │
│  │           ╲  ╱                                              │     │
│  │  Minimum ╲╱                     전역 최솟값(global minimum) │     │
│  │            ●                                                   │     │
│  │                                                             │     │
│  │  α (학습률)가 너무 크면 → 진동/발산                           │     │
│  │  α (학습률)가 너무 작으면 → 수렴 太 늦음                      │     │
│  │                                                             │     │
│  └─────────────────────────────────────────────────────────────┘     │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** MSE는 예측 오차의 제곱的平均으로, 큰 오차에 큰 페널티를 부여한다. 그러나 outliers(극단적 이상치)에 매우 민감하다. MAE는 절대값의 평균으로 outliers에 덜 민감하지만, 미분이 불가능한 점(0에서 kink)이 있어 경사 하강법 적용이 어렵다. Huber Loss는 이 두 방식의 장점을 결합하여, 작은 오차에는 MSE를, 큰 오차에는 MAE를 적용한다. 경사 하강법은 손실 함수의 기울기(gradient)의 반대 방향으로 파라미터를 업데이트하여Minimum에 도달하는 算法이다. 학습률(α)의 선택이 매우重要으로, 너무 크면Minimum을 넘나들어 발산하고, 너무 작으면 수렴까지 오래 걸린다.

### 과적합과 정규화 (Ridge & Lasso)

```
┌──────────────────────────────────────────────────────────────────────┐
│                    과적합 vs 정규화                                       │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  【과적합 (Overfitting)】                                          │
│                                                                      │
│  - 훈련 데이터에 대한 오차는 낮지만, 테스트 데이터에서 오차가 높음      │
│  - 모델이 너무 복잡하여 노이즈까지 학습한 상태                          │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐     │
│  │                                                             │     │
│  │   과소적합          적합            과적합                  │     │
│  │    (High Bias)   (적당)        (High Variance)            │     │
│  │                                                             │     │
│  │     ───            ╭──╮          ───╮╮╮╮╮            │     │
│  │   ────           ─╯  ╰─╯       ─╯  ╰─╯╰╮╮            │     │
│  │  ────          ──          ─╯        ╰─╯╰╮            │     │
│  │                                                             │     │
│  │  훈련: 높음        적당          낮음                      │     │
│  │  테스트: 높음       적당          높음                      │     │
│  │                                                             │     │
│  └─────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  【정규화 (Regularization)】                                        │
│                                                                      │
│  목적: 모델 복잡도에 페널티를 부과하여 과적합 방지                      │
│                                                                      │
│  Ridge (L2 정규화):                                                  │
│  Loss = MSE + λ Σβⱼ²                                               │
│  → 모든 가중치를 작게 만듦 (，但不是 0)                              │
│  → 변수 간 다중 공선성에 효과적                                       │
│                                                                      │
│  Lasso (L1 정규화):                                                  │
│  Loss = MSE + λ Σ|βⱼ|                                               │
│  → 일부 가중치를 정확히 0으로 만듦 (특징 선택 효과)                  │
│  → 불필요한 특성을 자동으로 제거                                      │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐     │
│  │                                                             │     │
│  │   Ridge: β̂ = (XᵀX + λI)⁻¹Xᵀy  (L2)                       │     │
│  │   Lasso: L1 페널티로 일부 β = 0 (sparse 모델)               │     │
│  │                                                             │     │
│  │   λ (람다): 정규화 강도                                   │     │
│  │   λ ↑ → Regularization ↑ → 계수 ↓ (더 간단한 모델)          │     │
│  │   λ ↓ → Original OLS에 가까워짐 (과적합 위험)               │     │
│  │                                                             │     │
│  └─────────────────────────────────────────────────────────────┘     │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 과적합은 모델이 훈련 데이터의 노이즈까지 학습하여 새로운 데이터에 대한 성능이 저하되는 현상이다. 정규화는 손실 함수에 모델 복잡도에 대한 페널티를 추가하여 이 문제를 해결한다. Ridge(L2)는 모든 가중치의 제곱합에 페널티를 부여하여 모든 가중치를 작게 만들지만 정확히 0으로 만들지는 않는다. Lasso(L1)은 가중치의 절대값 합에 페널티를 부여하여 일부 가중치를 정확히 0으로 만들어 자동적인 특징 선택 효과를 보인다. 정규화 강도를 조절하는 λ가 크면 클수록 더 간단한(과적합에 덜 취약한) 모델이 된다.

### 섹션 요약 비유

정규화는"너무 복잡한 답안을 쓰면 감점하는 시험 평가 방식"과 같다. 어떤学生은 수백 개의細항목을 다暗記하여 시험에 다 맞지만(과적합), 선생님이"핵심 개념만 서술하라"(정규화)고 하면 실제 적용能力更强的 답안을 쓰게 된다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교: MSE vs MAE vs Huber

| 손실 함수 | 수식 | 장점 | 단점 |
|:---|:---|:---|:---|
| **MSE** | (1/n)Σ(y-ŷ)² | 미분 가능, 큰 오차에 강함 | outliers에 민감 |
| **MAE** | (1/n)Σ|y-ŷ| | outliers에 강함 | 미분 불가능, 학습 불안정 |
| **Huber** | MSE (작은 오차) / MAE (큰 오차) | 둘 다的长处 | δ 설정 필요 |

### 섹션 요약 비유

손실 함수의 선택은"학생 평가 방식의 선택"과 같다. MSE는"세밀한评価"(큰 실수에게 더 큰 감점), MAE는"同等한 평가"(모든 실수에 동일 감점), Huber는"조별 종합"이다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 — 주택 가격 회귀 예측**: 면적, 방 개수, 위치, 건축 연도 등을 입력받아 주택 가격(연속적 값)을 예측한다. Ridge 회귀를 적용하여 다중 공선성(면적과 방 개수가 동시에 增加など)을 자동处理하고, λ를 교차 검증으로 최적화하여 과적합을 방지한다.

### 섹션 요약 비유

회귀 예측은"데이터에서 추세를 읽어 미래를 예측하는 것"이다.

## Ⅴ. 기대효과 및 결론

### 미래 전망

- **비선형 회귀의 확산**: Kernel Ridge Regression, Gaussian Process 등 비선형 관계를 처리하는 회귀 기법 발전
- **강화 학습 결합**: 회귀 모델이 환경 예측 모델로 활용되어 RL의 sample efficiency 향상
- **Explainable Regression**:係数の解釈가능성이 중요한 의료, 금융 분야에서 Shapley Values 등 설명 기법 적용 확대

### 섹션 요약 비유

회귀의 미래는"より正確な予測と解釈可能性の両立"이다.

|:---|:---|

## 👶 어린이를 위한 3줄 비유 설명

1. 회귀는"몇 점 받을까?"와 같이数字로 답하는 시험과 같아요. 면적, 방 개수, 위치などを入れたら"집값는 5억원"이라고予測するのが回帰です.
2. 그런데 너무 많은 조건을 세밀하게 fitting하면(과적합), 훈련 데이터에서는 잘 맞지만, 새로운 집(테스트)에서는 엉뚱한 값이 나올 수 있어요.
3. 그래서"너무 복잡한 답은Simplify해라"(정규화)라는 규칙을 적용해요. Ridge는 모든 가중치를 작게 만들면서 Mulligator들이 비슷한 수준으로 참여하게 하고, Lasso는 아예 일부 Mulligator을"무시"(가중치 0)해버리는 거예요!
