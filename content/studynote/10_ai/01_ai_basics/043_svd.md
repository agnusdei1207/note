> **핵심 인사이트**
> 1. SVD(Singular Value Decomposition)는 임의의 m×n 행렬 A를 A = U Σ V^T 세 행렬의 곱으로 분해하는 선형대수 기법으로 — 정방행렬에만 적용되는 고유값 분해(Eigendecomposition)의 일반화이며, 협업 필터링 추천 시스템, PCA, 이미지 압축, LSA(잠재 의미 분석)의 수학적 기반이다.
> 2. Σ의 대각 원소인 특이값(Singular Value)은 내림차순으로 정렬되며, 상위 k개만 보존하는 Truncated SVD는 원 행렬의 가장 중요한 정보를 최소 차원으로 압축 — 이것이 노이즈 제거와 차원 축소의 원리이다.
> 3. 현대 LLM(대규모 언어 모델)의 LoRA(Low-Rank Adaptation) 파인튜닝은 SVD에서 영감을 받은 저랭크 행렬 분해를 활용하여 수십억 파라미터 모델을 소량의 데이터로 효율적으로 파인튜닝한다.

---

## Ⅰ. SVD 수학적 정의

```
SVD (Singular Value Decomposition):

임의 m×n 행렬 A에 대해:
  A = U Σ V^T
  
  U: m×m 직교행렬 (Left Singular Vectors)
     U^T U = I (직교 정규)
     
  Σ: m×n 대각행렬 (Singular Values)
     대각 원소 σ₁ ≥ σ₂ ≥ ... ≥ σᵣ ≥ 0
     r = rank(A)
     
  V: n×n 직교행렬 (Right Singular Vectors)
     V^T V = I (직교 정규)

기하학적 해석:
  어떤 선형 변환 A도:
    회전(V^T) → 스케일링(Σ) → 회전(U) 순서로 분해
  
  데이터 변환: Ax = U Σ V^T x
    1. V^T로 데이터 회전
    2. Σ로 각 차원 스케일링
    3. U로 다시 회전

관계:
  A^T A = V Σ^T Σ V^T → V = A^T A의 고유벡터
  A A^T = U Σ Σ^T U^T → U = A A^T의 고유벡터
  특이값 σᵢ = √(A^T A의 i번째 고유값)

계산 복잡도:
  전체 SVD: O(mn·min(m,n))
  Truncated SVD (상위 k): O(mnk)
```

> 📢 **섹션 요약 비유**: SVD는 악보 분해 — 복잡한 음악(행렬)을 "리듬 패턴(V^T) × 음량(Σ) × 음색(U)"으로 분리해서 가장 중요한 악기(큰 특이값)만 남기면 여전히 원래 곡처럼 들려요.

## Ⅱ. Truncated SVD와 차원 축소

```
Truncated SVD (절단 SVD):
  상위 k개 특이값만 보존
  Aₖ = U_k Σ_k V_k^T
  
  U_k: m×k (상위 k개 좌특이벡터)
  Σ_k: k×k (상위 k개 특이값)
  V_k: n×k (상위 k개 우특이벡터)
  
  Aₖ = 원 행렬의 Rank-k 근사 (최적 근사, Eckart-Young 정리)

정보 보존율:
  보존율 = Σᵢ₌₁^k σᵢ² / Σᵢ₌₁^r σᵢ²
  
  예: σ = [10, 8, 3, 1, 0.5, ...]
  k=2 보존율 = (100+64)/(100+64+9+1+0.25...) = 164/175 ≈ 94%

이미지 압축 예시:
  원 이미지: 512×512 픽셀 행렬
  전체 SVD: 512×512 + 512×512 + 512×512 = 786,432 원소 저장
  k=50 Truncated:
    U_50: 512×50 = 25,600
    Σ_50: 50 = 50
    V_50: 512×50 = 25,600
    저장 원소: 51,250 (원본의 6.5%)
    육안: 거의 동일한 이미지 품질

노이즈 제거 원리:
  신호: 큰 특이값 (상위 k개)
  노이즈: 작은 특이값 (하위)
  → k개만 보존 = 노이즈 제거 + 핵심 구조 유지

Python 구현:
  from sklearn.decomposition import TruncatedSVD
  svd = TruncatedSVD(n_components=50)
  X_reduced = svd.fit_transform(X)  # (n_samples, 50)
```

> 📢 **섹션 요약 비유**: Truncated SVD는 사진 압축 — 100가지 색상 정보 중 중요한 50가지만 남겨도 사진이 충분히 잘 보여요. 나머지 50가지는 노이즈에 가까워서 버려도 돼요.

## Ⅲ. SVD 기반 추천 시스템

```
협업 필터링 (Collaborative Filtering) + SVD:

사용자-아이템 평점 행렬:
        영화A  영화B  영화C  영화D
사용자1 [  5     4     ?     1  ]
사용자2 [  4     ?     1     2  ]
사용자3 [  1     1     5     ?  ]
사용자4 [  ?     2     4     5  ]

문제: ? 값 예측 (추천)

SVD 적용:
  R ≈ U Σ V^T (k개 특이값 사용)
  
  R[i,j] ≈ U_i · Σ · V_j^T
  = 사용자 i의 잠재 선호도 · 영화 j의 잠재 특성

잠재 요인 의미:
  k=2 가정:
  잠재 요인 1: "액션 vs 로맨스" 선호도
  잠재 요인 2: "고전 vs 현대" 선호도

행렬 분해로 예측:
  사용자1의 영화C 평점 = U₁ · Σ · V_C^T
  
  계산 결과: 예측 평점 3.8
  → 추천 여부 결정 (임계값 예: 4.0)

SVD++ (개선):
  암묵적 피드백 (클릭, 시청 시간) 포함
  Netflix Prize에서 SVD++ 모델 우승

현재 더 많이 사용: Matrix Factorization (SGD 기반)
  sklearn.decomposition.NMF
  implicit 라이브러리 (ALS)
```

> 📢 **섹션 요약 비유**: SVD 추천 시스템은 영화 취향 분석기 — "이 사람은 액션+현대 영화 좋아함" 잠재 특성을 발견해서 아직 안 본 영화 중 취향에 맞는 것 추천.

## Ⅳ. LSA와 LoRA

```
LSA (Latent Semantic Analysis):
  텍스트 분석에 SVD 적용
  
  단어-문서 행렬 (TF-IDF):
           문서1  문서2  문서3
  단어1  [ 0.8   0.1   0.0 ]
  단어2  [ 0.6   0.0   0.2 ]
  단어3  [ 0.0   0.9   0.1 ]
  
  SVD → 잠재 의미 공간 (k차원)
  잠재 의미 1: "AI/기술" 주제
  잠재 의미 2: "의학" 주제
  
  유사도 계산: cosine similarity in k-space
  "car" ≈ "automobile" (단어는 달라도 잠재 의미 동일)

LoRA (Low-Rank Adaptation):
  LLM 파인튜닝 기법 (2021, Microsoft)
  
  아이디어: 사전 훈련 가중치 W를 동결
           작은 저랭크 행렬 ΔW로 조정
  
  W' = W + ΔW
  ΔW = B × A (where B: d×r, A: r×d, r << d)
  
  예: d=4096 (LLaMA 레이어 크기), r=8
  기존 파라미터: 4096 × 4096 = 16.7M
  LoRA 파라미터: 4096×8 + 8×4096 = 65,536 (0.4%)
  
  SVD 연결:
    B × A는 rank-r 행렬 분해 = Truncated SVD와 동일 구조
    파인튜닝 = 저랭크 방향 업데이트만
  
  실용 결과:
    GPT-4 수준 모델을 8GB VRAM GPU에서 파인튜닝
    성능: Full Fine-tuning과 거의 동일 (95%+ 수준)
```

> 📢 **섹션 요약 비유**: LoRA는 교과서에 포스트잇 붙이기 — 두꺼운 교과서(LLM 가중치)를 새로 쓰지 않고, 핵심 페이지에 메모(저랭크 행렬)만 추가해서 새로운 지식 학습.

## Ⅴ. 실무 시나리오 — 뉴스 추천 시스템

```
뉴스 플랫폼 SVD 기반 추천 시스템 구축:

데이터:
  사용자: 100만 명
  기사: 50만 개
  클릭 이력: 5,000만 건 (희소 행렬)

행렬 구성:
  R: 100만 × 50만 사용자-기사 행렬
  값: 클릭=1, 비클릭=0 (암묵적 피드백)
  희소도: 0.01% (매우 희소)

Truncated SVD 적용:
  k=200 잠재 요인
  sklearn TruncatedSVD (ARPACK 알고리즘)
  
  학습 시간: ~30분 (배치)
  결과:
    U: 100만 × 200 (사용자 잠재 특성)
    V: 50만 × 200 (기사 잠재 특성)

온라인 추천:
  특정 사용자 u의 추천:
    scores = U[u] · V^T  → 50만 × 1 스코어 벡터
    top-10 = argsort(scores)[-10:]
    
  ANN(근사 최근접 이웃) 사용:
    Faiss(Meta) 또는 Annoy(Spotify)
    정확한 최근접보다 1000배 빠른 근사 탐색

성능 지표:
  CTR(클릭률): 2.1% → 3.8% (+81%)
  뉴스 체류 시간: 2.3분 → 3.7분 (+61%)
  콘텐츠 다양성: 지니 계수 0.72 → 0.61 (개선)

한계:
  콜드 스타트: 신규 사용자/기사 잠재 벡터 없음
  → 해결: 콘텐츠 기반 필터링으로 보완
  
  실시간성: 배치 SVD 학습 → 실시간 클릭 반영 지연
  → 해결: 온라인 Matrix Factorization (SGD)
```

> 📢 **섹션 요약 비유**: SVD 추천은 도서관 사서 AI — 사용자의 과거 독서 패턴(행렬)에서 "이 사람의 취향"을 수학적으로 파악하고, 아직 안 읽은 책 중 가장 맞는 것을 추천.

## 👶 어린이를 위한 3줄 비유 설명

1. SVD는 복잡한 행렬을 세 개의 단순한 행렬로 분해 — 복잡한 악보를 "리듬 × 음량 × 음색"으로 쪼개는 것처럼요!
2. Truncated SVD는 사진 압축 — 100가지 색상 중 중요한 50가지만 남겨도 사진이 거의 똑같이 보여요. 불필요한 노이즈가 제거돼요!
3. LoRA는 교과서에 포스트잇 붙이기 — ChatGPT 같은 큰 모델을 새로 만들지 않고, 얇은 메모(저랭크 행렬)만 추가해서 새 임무 학습!
