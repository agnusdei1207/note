> **핵심 인사이트**
> 1. 그리드 서치(Grid Search)·랜덤 서치(Random Search)·베이지안 최적화(Bayesian Optimization)는 하이퍼파라미터 공간 탐색 전략의 3세대로, 그리드(완전 탐색)→랜덤(균일 샘플링)→베이지안(이전 결과 활용 지능 탐색) 순으로 효율이 높아진다.
> 2. Bergstra & Bengio(2012) 연구는 하이퍼파라미터 중 일부는 성능에 거의 영향을 미치지 않아 그 차원을 낭비하는 그리드 서치와 달리, 랜덤 서치는 같은 횟수의 평가로 더 많은 "중요한 차원"을 탐색한다는 수학적 근거를 제시했다.
> 3. 베이지안 최적화(Optuna 등)는 Gaussian Process나 TPE(Tree-structured Parzen Estimator)로 사후분포를 업데이트하며 탐색하여, 적은 평가 횟수(20~50회)로도 그리드 서치 수백 회에 근접한 성능을 달성한다.

---

## Ⅰ. 하이퍼파라미터 튜닝 방법 비교

```
세 가지 튜닝 방법 개요:

그리드 서치 (Grid Search):
  모든 조합 완전 탐색
  예: lr=[0.001,0.01,0.1], depth=[3,5,7]
      → 3×3 = 9가지 조합 모두 시도
  장점: 완전 탐색, 재현 가능
  단점: 조합 수 지수적 증가

랜덤 서치 (Random Search):
  정해진 횟수 내 무작위 조합 탐색
  동일 시도 횟수에서 그리드보다 효율적
  Bergstra & Bengio (2012) 증명
  장점: 같은 비용에 더 넓은 탐색 범위

베이지안 최적화:
  이전 평가 결과로 다음 탐색 위치 예측
  Gaussian Process + Expected Improvement
  TPE (Tree-structured Parzen Estimator)
  도구: Optuna, Hyperopt, Ax
  장점: 적은 평가로 최적값 근접 가능

비교:
  방법         | 평가 횟수 | 최적 탐색 | 구현 복잡도
  -------------|---------|---------|----------
  그리드 서치  | 지수적   | 완전     | 낮음
  랜덤 서치    | 선형     | 부분     | 낮음
  베이지안     | 선형     | 지능적   | 중간
```

> 📢 **섹션 요약 비유**: 그리드=모든 메뉴 다 먹기, 랜덤=무작위 3개 시식, 베이지안=이전 맛집 경험 기반 다음 선택.

## Ⅱ. 그리드 서치 상세

```
그리드 서치 (Grid Search) 상세:

작동 방식:
  파라미터 그리드 정의
  모든 조합에 대해 교차 검증 실행
  최고 성능 조합 선택

sklearn 예시:
  from sklearn.model_selection import GridSearchCV
  
  param_grid = {
      'max_depth': [3, 5, 7, 10],
      'n_estimators': [100, 200, 500],
      'learning_rate': [0.01, 0.1, 0.3]
  }
  # 4 x 3 x 3 = 36가지 조합
  # 5-Fold CV × 36 = 180번 학습
  
  grid_search = GridSearchCV(
      XGBClassifier(), param_grid, cv=5
  )

적합한 상황:
  파라미터 수 적음 (3개 이하)
  각 파라미터 범위 좁음
  재현성이 중요한 공개 벤치마크

문제: 차원의 저주
  파라미터 4개, 각 5가지 = 5^4 = 625 조합
  파라미터 10개, 각 5가지 = 5^10 ≈ 1,000만 조합
```

> 📢 **섹션 요약 비유**: 그리드 서치는 지도 전체를 격자로 나눠서 한 칸씩 다 뒤지기 — 확실하지만 지역이 클수록 시간이 기하급수적으로 늘어난다.

## Ⅲ. 랜덤 서치와 베이지안 비교

```
랜덤 서치 (Random Search):

핵심 인사이트 (Bergstra & Bengio 2012):
  "HP 공간에서 일부 HP는 성능에 무관"
  그리드: 무관한 차원도 모두 탐색 (비효율)
  랜덤: 동일 횟수에서 더 다양한 중요 HP 탐색

예시:
  hp1 (중요) × hp2 (무관)
  그리드(3×3=9): hp1을 3가지만 탐색
  랜덤(9회):     hp1을 최대 9가지 탐색 (중요 차원)

베이지안 최적화 (Bayesian Optimization):

Optuna TPE 작동 방식:
  1. 초기: 랜덤 탐색 10~20회
  2. 이전 결과 분석:
     good HP 분포 (l(x)) vs bad HP 분포 (g(x))
  3. EI(Expected Improvement) 최대화 지점 탐색:
     EI = P(better) × E(improvement)
  4. 그 지점 평가 후 분포 업데이트 반복

Optuna 예시:
  import optuna
  
  def objective(trial):
      lr = trial.suggest_float('lr', 1e-4, 1e-1, log=True)
      depth = trial.suggest_int('max_depth', 3, 10)
      return cross_val_score(model(lr, depth), X, y).mean()
  
  study = optuna.create_study(direction='maximize')
  study.optimize(objective, n_trials=50)
```

> 📢 **섹션 요약 비유**: 베이지안은 경험 많은 음식 평론가 — "이전에 맛있었던 재료들"을 기반으로 다음 레스토랑을 골라 효율적으로 최고의 식당 발견.

## Ⅳ. 실전 튜닝 전략

```
실전 HP 튜닝 전략:

단계별 추천:

1단계: 중요 HP 식별 (전문 지식)
  XGBoost 핵심: learning_rate, n_estimators, max_depth
  신경망 핵심: learning_rate, batch_size, dropout

2단계: 넓은 범위 랜덤 서치 (50회)
  중요 HP 범위 좁히기

3단계: 베이지안 최적화 (30회)
  좁혀진 범위에서 정밀 탐색

4단계: 최종 튜닝
  학습률 파인튜닝
  조기 종료 설정

조기 종료 (Early Stopping):
  Pruner: 나쁜 HP 빠르게 제거 (Optuna MedianPruner)
  비용: 전체 계산량 50~80% 절감

검증 원칙:
  Nested CV: 외부 CV(성능) + 내부 CV(HP 튜닝)
  테스트 세트는 최종 1회만 사용
  데이터 누수 없이 독립적 검증
```

> 📢 **섹션 요약 비유**: 실전 HP 튜닝은 부동산 탐색 — 넓은 도시(랜덤)로 동네 후보 좁히고, 그 동네 집들(베이지안)을 정밀 탐색.

## Ⅴ. 실무 시나리오 — 금융 이상 탐지

```
금융 이상 탐지 모델 HP 튜닝:

데이터: 거래 1,000만 건, 특성 80개
기본 모델: XGBoost (기본 HP)
기본 AUROC: 0.87

튜닝 전략 (총 80회 평가):

1단계: 랜덤 서치 50회
  범위: n_estimators=[100~2000], max_depth=[3~10],
        lr=[0.01~0.3], subsample=[0.6~1.0]
  5-Fold CV 각 평가
  결과: 최적 범위 식별, AUROC 0.905

2단계: Optuna TPE 30회
  범위 좁혀서 정밀 탐색
  MedianPruner로 조기 종료 활용
  결과: AUROC 0.921

개선: 0.87 → 0.921 (+5.9%)
  이상 탐지 1% 개선 = 연 수십억 사기 방지

최종 파이프라인:
  월별 리트레이닝 + 자동 HP 재튜닝 (Optuna)
  데이터 분포 변화(Drift) 감지 시 전체 재튜닝
```

> 📢 **섹션 요약 비유**: 금융 이상 탐지 HP 튜닝은 레이더 감도 조정 — 너무 민감(과적합)하면 오경보, 너무 둔감하면 탐지 실패, 최적 감도를 베이지안으로 찾기.

## 👶 어린이를 위한 3줄 비유 설명

1. 그리드 서치는 모든 조합을 다 시도하고(느리지만 완전), 랜덤 서치는 무작위로 시도해서(빠르지만 부분), 베이지안은 이전 결과를 보고 다음을 골라요(가장 똑똑)!
2. 수학적으로 랜덤 서치가 그리드보다 효율적인 이유는 — 일부 설정값은 성능과 관계없어서 랜덤이 오히려 더 다양한 "중요한 값"을 탐색하기 때문이에요.
3. Optuna(베이지안)는 "이전 맛집 경험"을 기억하고 다음에 더 좋을 것 같은 식당을 골라요 — 50번 가도 좋은 식당을 효율적으로 찾아요!
