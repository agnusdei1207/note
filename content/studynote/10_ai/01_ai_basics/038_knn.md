> **핵심 인사이트**
> 1. k-NN은 "같은 특성을 가진 것들은 가까이 있다"는 직관적 원리로 동작하는 게으른 학습(Lazy Learning) 알고리즘으로, 별도의 모델 학습 없이 예측 시점에 k개의 가장 가까운 이웃을 찾아 분류/회귀를 수행한다.
> 2. k 값 선택이 핵심 하이퍼파라미터 — k가 너무 작으면 과적합(노이즈에 민감), 너무 크면 과소적합(경계가 과도하게 부드러워짐)하며, 일반적으로 교차 검증으로 최적 k를 탐색한다.
> 3. k-NN의 가장 큰 한계는 O(n) 예측 시간과 O(n) 메모리 요구 — 데이터가 수백만 건이 되면 실시간 예측이 불가하여 KD-Tree, Ball-Tree, Approximate Nearest Neighbor(ANN) 등 최적화가 필요하다.

---

## I. 동작 원리

```
k-NN 분류 (Classification):

학습 단계:
  데이터를 그냥 메모리에 저장 (학습 없음!)
  
예측 단계:
  1. 새 데이터 포인트 x 입력
  2. 훈련 데이터 모든 점과 거리 계산
  3. 가장 가까운 k개 선택
  4. k개 중 가장 많은 클래스로 분류 (다수결)

예시 (k=3):
  새 데이터: (2.5, 3.0)
  가장 가까운 3개: 클래스A, 클래스A, 클래스B
  -> 예측: 클래스A (2:1 다수결)

k-NN 회귀 (Regression):
  k개 이웃의 평균값으로 예측
```

> 📢 **섹션 요약 비유**: 새 동네에서 맛집 찾기 — 주변 k개 식당 중 가장 많이 추천받은 곳(분류) 또는 평점 평균(회귀)으로 결정.

---

## II. 거리 지표

```
주요 거리 지표:

유클리드 거리 (Euclidean Distance):
  d = sqrt( sum((xi - yi)^2) )
  연속형 특성에 가장 일반적
  
맨하탄 거리 (Manhattan Distance):
  d = sum( |xi - yi| )
  격자 이동, 고차원에서 유용
  
민코프스키 거리 (Minkowski Distance):
  일반화: p=2 -> 유클리드, p=1 -> 맨하탄
  
코사인 유사도 (Cosine Similarity):
  벡터 간 각도 기반
  텍스트/문서 k-NN에 적합
  
해밍 거리 (Hamming Distance):
  범주형 데이터 (이진/문자열 비교)
  
주의: 특성 스케일 통일 필수!
  특성A: [0, 100], 특성B: [0, 1]
  -> 특성A가 거리를 지배 -> 정규화 필요
```

> 📢 **섹션 요약 비유**: 직선 거리(유클리드)와 블록 이동 거리(맨하탄) — 같은 두 점이라도 상황에 따라 적합한 거리 기준이 다름.

---

## III. k 값과 과적합/과소적합

```
k=1: 과적합 (Overfitting)
  가장 가까운 1개만 -> 노이즈에 민감
  훈련 정확도 100%, 테스트 낮음
  
k=n (전체 데이터): 과소적합 (Underfitting)
  모든 데이터 다수결 -> 항상 같은 클래스
  
최적 k 선택:
  일반적: sqrt(n) 시작점
  교차 검증 (Cross-Validation)으로 탐색
  홀수 k (이진 분류에서 동점 방지)

k 값에 따른 결정 경계:
  작은 k: 복잡한 경계 (구불구불)
  큰 k: 단순한 경계 (부드러움)
```

| k 값 | 편향(Bias) | 분산(Variance) | 경계 복잡도 |
|-----|----------|-------------|---------|
| 작음 | 낮음      | 높음          | 복잡      |
| 큰  | 높음      | 낮음          | 단순      |

> 📢 **섹션 요약 비유**: k=1은 옆 한 명 의견만 듣는 것(변덕스러움), k=전체는 1000명 여론조사(항상 같은 결과) — 적당한 k가 핵심.

## IV. k-NN 가속 구조

```
브루트 포스 한계:
  O(n * d) 예측 시간 (n=데이터, d=차원)
  데이터 100만 건: 예측 1회 = 수십 ms

KD-Tree:
  공간을 재귀적으로 이분
  O(log n) 평균 쿼리 시간
  단점: 고차원(>20)에서 성능 급감 (차원의 저주)

Ball-Tree:
  구(Ball) 구조로 공간 분할
  고차원에서 KD-Tree보다 나음

ANN (Approximate Nearest Neighbor):
  정확한 이웃 대신 근사 이웃 탐색
  FAISS (Meta), Annoy (Spotify), HNSW
  대규모 벡터 검색 (추천 시스템, RAG)에 사용

FAISS:
  10억 벡터도 ms 단위 검색
  LLM의 Retrieval-Augmented Generation 핵심
```

> 📢 **섹션 요약 비유**: 브루트 포스는 도서관 전체를 뒤지기, KD-Tree는 서가 번호로 좁히기, FAISS는 AI 사서가 가장 비슷한 책 즉시 찾기.

## V. 실무 시나리오 — 추천 시스템

```
콘텐츠 기반 추천 (k-NN 활용):

사용자-영화 벡터:
  사용자 A의 평점 벡터:
  [액션:5, 로맨스:2, SF:4, 공포:1, 코미디:3]

k-NN 추천:
  1. 사용자 A와 가장 유사한 k=5명 찾기
     (코사인 유사도로 취향 벡터 거리)
  2. 5명이 5점 준 영화 중 A가 안 본 것
  3. 추천 목록 생성

현대적 구현 (FAISS + 임베딩):
  사용자/영화를 128차원 임베딩 벡터로
  FAISS HNSW Index로 실시간 ANN 검색
  초당 수백만 쿼리 처리 가능

Netflix/YouTube:
  수억 사용자 임베딩을 ANN으로 검색
  -> k-NN 원리 + 현대적 최적화
```

> 📢 **섹션 요약 비유**: Netflix 추천은 "나와 비슷한 취향의 사람들이 좋아하는 것" — k-NN의 핵심 원리를 초대형 규모로 구현한 것.

## 📌 관련 개념 맵

```
k-NN (k-최근접 이웃)
+-- 동작 원리
|   +-- 게으른 학습 (인스턴스 기반)
|   +-- 거리 기반 투표/평균
+-- 핵심 하이퍼파라미터
|   +-- k 값 (편향-분산 트레이드오프)
|   +-- 거리 지표 (유클리드, 맨하탄, 코사인)
+-- 최적화
|   +-- KD-Tree, Ball-Tree
|   +-- ANN: FAISS, Annoy, HNSW
+-- 응용
    +-- 분류, 회귀, 이상 탐지
    +-- 추천 시스템 (임베딩 + ANN)
    +-- RAG (LLM 벡터 검색)
```

## 👶 어린이를 위한 3줄 비유 설명

1. k-NN은 새 친구를 만들 때 "나와 제일 비슷한 k명은 어떤 그룹 소속인지" 보고 그 그룹으로 분류하는 방법이에요.
2. 학습이 따로 없어서 "게으른 알고리즘"이라 불리지만, 그래서 새 데이터가 들어올 때마다 모든 이웃을 찾아야 해서 느려질 수 있어요.
3. Netflix가 "당신과 비슷한 취향의 사람들이 좋아한 영화"를 추천하는 것이 k-NN의 핵심 원리예요!
